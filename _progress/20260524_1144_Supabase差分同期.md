# Supabase同期を差分方式に変更（statement timeout対策）

日時: 2026-05-24 11:44 (JST)

## 背景・目的

- 当日のルーティン報告で、Supabase同期の batch 9 のみ PostgreSQL statement timeout（57014）が発生していた。
- 原因調査の結果、`sync_supabase.py` は **毎回 index.json 全件（7,618件）を upsert** する設計で、件数増加に伴い重いバッチが制限時間を超えていた。
- ユーザー希望は「ムダを省く＝差分同期化」。新着・変更分だけ送る方式に作り替えた。

## 備考・気づき

- 「失敗バッチを state 未反映にする」だけで、明示的なリトライ実装なしに次回自動再送が成立する（差分同期の副次効果）。
- state ファイルはローカル限定。別環境/初回実行時は自動的に全件同期になる（=安全側に倒れる設計）。
- Supabase側を手で消して不整合が出たら `python scripts/sync_supabase.py --full` で全件再送できる。
