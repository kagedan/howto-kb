## 備考・気づき（lessons 候補）

1. **Workflow Haiku low の id 漏れ**: 200バッチ規模でも 4-150 件程度の採点漏れが発生する。`inventory_review.py` が「⚠ 未採点 id」を報告するので、補完 agent で別途採点 → workflow_*scores-{tag}.json に追記 → 再 review で対応
2. **Workflow の id 重複出力**: 採点バッチで稀に id 重複（Haiku の出力）。`inventory_shortlist.py` は id ベースで dedup するので件数は正しい
3. **Supabase DELETE in.(...) フィルタ + 日本語 id**: 100件バッチで稀に 400 Bad Request が出る（id 内の特殊文字エンコード問題と推測）。エラー時に半分割再帰する `_delete_batch` 関数で 100% 削除を保証
4. **Python print バッファリング**: `run_in_background` で起動した python スクリプトの進捗ログが出力ファイルに反映されない（バッファリング）。完了通知まで進捗不可視。次回からは `PYTHONUNBUFFERED=1` を付けるか、`sys.stdout.flush()` 明示が必要
5. **archive は .gitignore で除外済み**: 単純な「articles → archive」move でリポジトリは軽量化される（commit には articles 側の delete だけが反映される）
