## 備考・気づき

### 実装上の判断
- `refetch_rss_fulltext.py` はワンショット用として残した。毎朝の新規分は `crawl_rss.py` 内で
  処理されるので、このスクリプトは過去分の棚卸し・障害復旧用途で将来また使う可能性あり
- スクリプトを `_generate_mds.py` 側ではなく `crawl_rss.py` 側に組み込んだのは、
  手順3が既に要約・MD生成・X有益判定・RT Clippings出力と責務過多でリスクが高かったため
  (ユーザーからの明確なガイダンスに従った)

### 注意点・学び
- **GitHub Secret Scanning**: 記事本文に含まれるダミーの Slack Webhook 文字列
  (`XXXXXXXXX/XXXXXXXXX/XXXXXXXXXXXXXXXXXXXXXXXX`) がパターンマッチでヒットする。
  プレースホルダは `YOUR_TEAM_ID/YOUR_CHANNEL_ID/YOUR_WEBHOOK_TOKEN` 形式が安全
- **カテゴリ移動時のファイル名衝突**: 同じIDパターン (`YYYY-MM-DD-article-NN.md`) が
  複数カテゴリに存在し得るため、カテゴリ移動時は衝突チェック必須
- **SKILL.md と実装の乖離**: スケジューラ履歴の時刻表示はUTC、SKILL.md説明の時刻はJSTで、
  一見ズレて見えるが整合している(10:00 JST = 01:00 UTC)
- **Claude Code の Bashツール制約**: timeout指定は最大10分だが、run_in_background で
  30分・60分指定しても効いた実績あり(ドキュメントより寛容)
