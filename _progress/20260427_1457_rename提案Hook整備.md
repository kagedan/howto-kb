## 備考・気づき

- **`/rename` をはじめCLI内蔵スラッシュコマンドはツール／Hook経由で自動実行できない**。Skill toolドキュメントにも "Do not use this tool for built-in CLI commands" と明記。今後同様の質問が来たら即座に「不可。提案＋ユーザーコピー実行が公式手段」と答える
- Windows でPython Hookが日本語を含むJSONを stdout に出すとき、パイプは cp932 デフォルトのため文字化けする。`sys.stdout.reconfigure(encoding="utf-8")` ＋ `ensure_ascii=True`（デフォルト）の二重対策が無難
- ユーザー級 `~/.claude/settings.json` の SessionStart は既存の "startup" matcher の hooks 配列に追記すれば既存フックと並列実行される（matcher を分けると2エントリになるが動作は同じ）
- `.claude/` はリポジトリの `.gitignore` 対象なのでプロジェクト級設定はマシン固有。ユーザー級配置に移したことで、別マシンでも同じ動作を期待できる（ただし `~/.claude/` 配下を別マシンへ持ち出す場合は手動コピー必須）
- 提示フォーマットを「コードブロック非使用・前後空行・1行プレーンテキスト」に固定したのは、コードブロックで囲むとコピー時にバッククォートが混入してそのまま `/rename` が失敗するため
- 日付フォーマットを `MMDD` から `YYMMDD` へ変更したのは、年をまたぐ過去セッションを区別しやすくするユーザー要望
