## 備考・気づき

- **インライン `python -c` のJSON→shell→Python 3重エスケープは破綻しやすい**。Hookロジックは独立Pythonスクリプトに切り出して `python .claude/hooks/xxx.py` で呼ぶのが圧倒的にシンプル＆テスト可能。今後Hook追加するときも同じパターンで。
- `.claude/` は `.gitignore` 対象なので、Hookスクリプトと settings.json はローカルマシン専用。別マシンや worktree で同じ挙動を期待するなら手動コピー or `.gitignore` の例外設定が必要。
- IDE で `.env` を開いただけでもClaude Codeに通知が飛ぶ仕組みが今回判明（`<system-reminder>The user opened the file ...`）。プライバシー観点では中身が送られているわけではなく「開いた事実」のみ。
- claude-automation-recommender は read-only で、勝手にファイルを作らない。提案を見て採用→実装の2段階フロー。
