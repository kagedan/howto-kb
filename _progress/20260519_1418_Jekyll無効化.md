# GitHub Pages の Jekyll ビルド無効化

日時: 2026-05-19 14:18 (JST)

## 背景・目的

- howto-kb は GitHub Pages 経由で `index-latest.json` などの JSON を配信している
- 記事の蓄積で Jekyll ビルドが重くなり、日次の複数 push の間隔より長くなって「higher priority waiting request」で連続キャンセルされていた
- JSON 配信が目的で HTML 生成は不要なため、`.nojekyll` を置いて Jekyll ビルドを無効化する

## 備考・気づき

- `.nojekyll` を消すと GitHub Pages が Jekyll ビルドを再開し、記事蓄積に比例してビルド時間が伸び、日次 push 間隔を超えて連続キャンセルする状態に戻る（JSON 配信が一時停止する恐れあり）。**将来 `.nojekyll` を消さないこと**
- Actions 画面の Annotations に「Node.js 20 actions が deprecated」という warning が出ているが、これは GitHub 側の pages 関連 action 内部の話で、howto-kb 側で対応する必要は無い
- 原因特定はユーザー側（chat.claude.ai に Actions のスクショを貼って分析）で済んでいたため、Claude Code 側は `.nojekyll` を置くだけの単発作業で完了した
