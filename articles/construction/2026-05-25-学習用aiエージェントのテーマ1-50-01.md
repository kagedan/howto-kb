# ブロック詳細説明書（テーマ 1〜50）

---

## 共通ブロック用語集

| ブロック名 | 意味 |
| --- | --- |
| `LLM Agent` | OpenAI API (gpt-4o / gpt-4o-mini) または Azure OpenAI を呼び出す中核処理。LangChain `AgentExecutor` または LangGraph の `Node` として実装 |
| `Vector DB` | テキストをベクトル化して保存・類似検索するDB。ローカルはChromaDB/FAISS、クラウドはAzure AI Search |
| `Embedding Model` | テキストをベクトルに変換するモデル。`text-embedding-3-small`（OpenAI）または `sentence-transformers/all-MiniLM-L6-v2`（ローカル無料） |
| `Orchestrator` | 複数エージェントに仕事を割り振る司令塔。LangGraph `StateGraph` で実装 |
| `Supervisor` | Orchestratorと同義。どのサブエージェントを次に呼ぶか判断する |
| `Tool` | エージェントが呼び出せる外部機能（DB・API・ファイル操作等）。LangChain `@tool` デコレータで定義 |
| `Graph API` | Microsoft 365 (Outlook/Teams/SharePoint等) にアクセスするREST API |
| `Human-in-the-loop` | エージェントの処理途中で人間の確認・承認を挟む仕組み |

### Theme 1: SQL Agent

**設計変更**: なし ✅

| ブロック | 実装技術 | 役割 | 入力 | 出力 |
| --- | --- | --- | --- | --- |
| `ユーザー` | Streamlit UI / CLI | 自然言語クエリを入力するインターフェース | テキスト入力 | クエリ文字列 |
| `LLM Agent` | LangChain `create_sql_agent` + gpt-4o | 自然言語をSQLに変換し、実行・エラー修正・結果説明を行う中核 | 自然言語クエリ + DBスキーマ | SQL文 / 最終回答 |

**実装ポイント**:

* LangChain の `SQLDatabaseToolkit` を使うと DB スキーマ自動読み込み + SQL実行ツールが一式セットアップされる
* エラー時の自動修正は `handle_parsing_errors=True` で有効化
