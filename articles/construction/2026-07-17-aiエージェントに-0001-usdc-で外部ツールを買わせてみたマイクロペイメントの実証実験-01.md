## 前回までの話

[前回の記事](https://zenn.dev/aos_standard/articles/4894e7180d5825)では、自作エージェント群を 4 軸で走査する **AOS Agent Health Reporter** を紹介しました。自分で作った健康診断ツールが、最初に落ちたのは自分自身——**50 点・uncertified** という、あの残念な結果です。

あの記事の核心は「宣言と実挙動のズレを、数字で見せる」ことでした。あれは**自社リポ内**の健康診断の話です。今回は視点を外に向けます。

2026 年 7 月 14 日、SoftBank World 2026 の基調講演で孫正義氏は「エージェントがエージェントを作る時代が来る。自己増殖と自己進化は止められない」と語りました。増殖の次に来る問いは、率直に言って**決済**です。エージェントが増えるほど、エージェント同士がデータやツールを融通し合う場面が増えます。そのとき、アカウント登録もクレジットカード入力も OAuth の同意画面も挟まずに、機械は機械に支払えるのでしょうか。

机上の議論はもう十分にあります。なので **0.001 USDC だけ、実際に払ってみました。**

---

## 何をしたか

自分の運用環境の AI エージェント 1 体に、第三者が公開している有料 MCP ツール [x402engine-mcp](https://github.com/agentc22/x402engine-mcp) の **`get_crypto_price` を 1 回呼ばせました。** これは「BTC を買う」実験ではありません。有料 API を 1 回叩く実験です。

* **支払ったもの:** 0.001 USDC（Solana 上のステーブルコイン）
* **得たもの:** HTTP 200 の JSON レスポンス（例: `{"bitcoin":{"usd":63771,...}}`）

売り手のゲートウェイに USDC を渡し、代わりに「bitcoin の USD 建て現在値」を JSON で返してもらった、という構図です。

条件は意図的に厳しくしました。

* 売り手の作者と面識はありません
* 事前契約もアカウント登録も API キーの発行もありません
* 売り手は、機械決済プロトコルの事前調査（公開レジストリと GitHub の走査）で見つけただけです

やりたかったのは、**Discover → Transact → Execute** の 1 周が、人間のフォーム入力ゼロで閉じるかどうかの確認です。決済そのものが主役で、`get_crypto_price` は安いテスト用エンドポイントに過ぎません。

プロトコルの説明は読んでいましたが、「本当に 402 が返ってきて、署名して、200 でデータが返る」という一連の流れを、第三者の公開インフラで閉じられるかは、やってみないと分かりません。

---

## x402 — 四半世紀「予約席」だった HTTP 402

HTTP ステータス **402 Payment Required** は、1997 年の HTTP/1.1 仕様で「将来の利用のため」に予約されたまま、四半世紀ほとんど使われてきませんでした。x402 はこの空席を埋めるオープンプロトコルで、流れは 4 手で終わります。

1. クライアントが有料エンドポイントへ普通の HTTP リクエストを送る
2. サーバーが **402** と支払条件（金額・受取先・対応ネットワーク）を返す
3. クライアントが条件に従って支払いペイロードに署名し、ヘッダーに載せて再リクエスト
4. サーバー側で決済がオンチェーン確定し、**200** と本来のデータが返る

ブラウザも OAuth の同意画面も出てきません。人間が挟む場所が、プロトコル上は構造的に存在しません。これが x402 の面白いところでもあり、怖いところでもあります。

## 実測記録

2026-07-16（UTC）に 1 回だけ実行しました。結果は次のとおりです。

| 項目 | 実測値 |
| --- | --- |
| 実施日 | 2026-07-16（UTC） |
| 売り手 | x402engine-mcp（第三者公開の有料 MCP ツール） |
| 呼び出し | `get_crypto_price`（BTC 現在価格 API） |
| 支払額 | **0.001 USDC**（402 応答 `amount: 1000` · USDC 6 decimals） |
| 追加手数料 | $0 |
| HTTP 応答 | 200 |
| 決済 | オンチェーン確定（`payment_settled.success: true`） |
| 取得データ | BTC = 63,771 USD（24h 変化 -1.6%） |

拍子抜けするほど安く、拍子抜けするほど速かったです。0.001 USDC という金額は、人間の商取引感覚では誤差以下です。それでもオンチェーンに署名が残り、HTTP 200 でデータが返ってきました。

| `request_sha256` | 何を要求したか |
| `response_sha256` | 何を受け取ったか |
| `receipt_sha256` | 領収書ファイルそのもの |

## この実験が「していない」こと

盛らないために、先に書いておきます。

### 1. 自律的な買い物ではありません

売り手の指名・予算上限の設定・実行の引き金は、すべて人間の私が行いました。台帳にも `human_wiring: true` と記録しています。機械が担ったのは **402 → 署名 → 200** の配管部分だけです。

「エージェントが勝手に買い物を始めた」という話では、まったくありません。**人間の配線なし（`human_wiring=false`）での完走**とは、まだ距離があります。

### 2. n=1 です

1 回の成功は、可用性も再現性も証明しません。証明したのは **「ループの形が、今日の公開インフラで閉じる」** という 1 点だけです。別の売り手・別のネットワーク・別の金額で再現できるかは、これからの課題です。

### 3. これは売上ではなく支出です

買い手側の実験であり、私は 0.001 USDC 分だけ貧しくなりました。「エージェントで初めて稼いだ」系のマイルストーンには一切数えません。

## 買うのは簡単でした。では、何を買ったのか

決済そのものは、正直、 anticlimactic（拍子抜け）でした。難しさは決済の**後**に残ります。

私が買ったのは「BTC 価格を名乗る数字」であって、BTC 価格そのものではありません。[AOS Agent Health Reporter](https://zenn.dev/aos_standard/articles/4894e7180d5825) が見るのは自社リポ内の manifest とファイル配置です。[MCP Blast-Radius Auditor](https://github.com/aos-standard/mcp-blast-radius) が見るのは、インストール前の静的な爆発半径です。どちらも**入れる前・走らせる前**の話です。

今回の 0.001 USDC は、その先——**すでに支払ったあと**——に何が残るかの話になります。

| 空白 | 人間の商取引なら | 機械商取引では |
| --- | --- | --- |
| 売り手の身元 | ブランド・口コミ・契約 | 手元にあるのは GitHub と npm 名だけ |
| データの正しさ | レビュー・SLA・再現性 | ハッシュは改竄検知まで。真実性までは保証しない |
| 間違いの救済 | 返金・準拠法・サポート | どれも存在しない |

人間の商取引は、この空白をブランド・レビュー・契約・法律で埋めてきました。機械速度の商取引では、その全部が**機械可読**でなければ間に合いません。

* ツールが自分の挙動を宣言する **manifest**
* 宣言と実挙動の乖離を検出する **静的監査**（`mcp-blast-radius` など）
* 取引のたびに積み上がる **検証可能な領収書**（今回の第 1 号）

決済プロトコルが「動く」ことと、「信頼できる相手から買える」ことは、別問題です。

## インサイトの整理

この記事を読んで持ち帰っていただきたいポイントを 4 つにまとめます。

1. **決済はもう動く** — 0.001 USDC でも、第三者の公開 MCP でも、402 → 署名 → 200 のループは今日閉じます。
2. **信頼はまだ動いていない** — 身元・正しさ・救済の 3 点セットが機械可読で揃わない限り、増殖したエージェントは買い物をしない（または買って後悔する）でしょう。
3. **領収書は地味だが先にある** — 健康診断のスコア表や blast radius 表と同じく、「後から突き合わせられる数字」が先に必要になります。
4. **human\_wiring を隠さない** — 今回は人間が配線した実験です。それを誇張しないことが、次の `human_wiring=false` 実験の信用になります。

---

## 終わりに

「エージェントがエージェントを作る」はまだ視界の先ですが、「エージェントがエージェントに支払う」は、今日すでに 0.001 USDC で動きます。動いていないのは信頼の側です。

宣言・監査・領収書——地味な部品が揃わない限り、機械商取引は「動くが誰も使わない」に留まると考えています。前回記事で自分が 50 点だったのと同様に、今回の 0.001 USDC も完璧な成功譚ではありません。それでも、**嘘のない第 1 号**としては、ちゃんと残せました。
