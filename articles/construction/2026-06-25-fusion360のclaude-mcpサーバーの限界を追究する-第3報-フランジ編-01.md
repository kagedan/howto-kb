# 設計順序

着手前に依存グラフ（誰が誰の寸法・位置・個数を決めるか）を確定し、上流→下流で構築。独立変数のみ userParameters 化。派生値はパラメータ化せず、参照・式・"up to" で導出。

**パラメータ命名は予約語との衝突を構造的に回避する。** 用途・コンポーネント名を前置した接頭辞付き命名を既定とし（例: `pcd` は予約されうるため `bolt_pcd`）、汎用的な短い語の単独使用を避ける。

envelope（最上位の参照専用ジオメトリ）  
ソリッドを持たない基準を最上位コンポーネントに最初に構築：

* 軸方向：積層 ConstructionPlanes（setByOffset のオフセットを式で：base\_thk, base\_thk + top\_thk…）
* 周方向：共有ボルト円スケッチ（PCD基準円＋基準角）と共有中心軸
* 全子コンポーネントはこの最上位ジオメトリを参照（PCD等を各部品に分散させない）

# 環境クセの事前確定

本格着手の前に、**多重生成系API（パラメータ追加・フィーチャ連続生成など）が1スクリプト内で複数回呼べるか**を最小プローブで一度だけ確認する。複数回呼で失敗するなら、その操作種別は「1スクリプト1操作」に固定する。クセは最初に確定し、以後は固定パターンで実行する（この挙動はトランザクションのコミット境界に依存し環境ごとに異なるため、汎用ルールに特定の値を直書きせず、毎回プローブで判定する）。

# スケッチ

寸法記載・完全拘束（sketch.isFullyConstrained で確認）

addTwoPointRectangle/sketchCircles.add\* は自動拘束を付けない → geometricConstraints で水平/垂直/中点/一致を明示。過剰拘束に注意。**既に寸法で位置が確定した点へ追加の coincident を重ねると solver が破綻しうる（過剰拘束）。拘束は重複させず、寸法で固定済みの要素には幾何拘束を足さない。**

中心同定は補助線（isConstruction=True）。**補助線はジオメトリに無関係でも、それ自身の長さ・角度が拘束対象に含まれる → 完全拘束を狙うなら補助線にも寸法を与える。** B-Rep面を sketches.add() に渡さず ConstructionPlane を使う。他スケッチ参照は sketch.project()。

**同種スケッチが複数あるときは、最初の1要素で isFullyConstrained==True の定型を完成させてから複製する（試行錯誤を1要素に閉じ込め、残りは使い回す）。**

# フィーチャ

押し出しは距離値でなく ToEntityExtentDefinition.create(plane) でenvelope平面まで（"up to"）。**ただし下流依存を持たないリーフ寸法（ボルト頭高さ等）は、参照先となる平面を増やすより、パラメータ式による距離押し出しでよい。**  
参照先はB-Rep面より ConstructionPlane を優先。

**パラメータ式には、その環境の式エンジンがサポートする関数しか使えない。`min`/`max`/条件分岐等を使う前に1式で評価可能性を確認する。退化（0度角度寸法など）を式の小細工で回避せず、初期ジオメトリを非退化な位置に置いて solver を収束させる。**

# 周方向の繰り返し（収束依存）

円周配置は手動複数配置を禁止。1要素を作り circularPatternFeatures で展開し、quantity を個数パラメータで駆動。**quantity は数値でなくパラメータ名の式文字列で与える（パターンが computeAll で個数追従するため）。**  
角度のハードコード禁止。ピッチは 360 deg / qty の式で導出。  
パターンの基準角は明示（回転対称で縮退するため）。

# アセンブリ

1コンポーネント1責務。共有ジオメトリは最上位定義。位置はenvelope参照で埋め込み、剛体締結に joints を使わない（相対運動の自由度がある時のみ）。

# スクリプト実行

例外で全フィーチャがロールバック → 関連操作を1スクリプトに集約。itemByName で既存チェックし冪等化。空コンポーネント殻は bRepBodies.count==0 で掃除。

**例外発生時、メッセージの文言を真因と決めつけない。まず「単独で再現するか／文脈（直前操作・実行回数）に依存するか」を切り分けてから対処する（文言が指す対象＝名前・値・拘束と、真因が乖離するケースを前提とする）。**

**生成系APIの直後に同系APIを呼ぶと collection が無効化される環境がある → 削除と追加、追加と追加を同一スクリプトに混在させず分離する。**

# 合格条件（満たすまで未完成）
