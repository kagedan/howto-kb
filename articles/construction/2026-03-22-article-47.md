Simple &gt; Complex

When building LLM agents, the most successful implementations use basic composable patterns.

My take: agentic frameworks are great for not needing to reinvent the wheel while building agent patterns. https://t.co/SMU63RTcI8
🔄 Two main types of agentic systems:

• Workflows: Predefined paths
• Agents: Dynamic, self-directed systems

🛠️ Start simple! Only add complexity when needed. Many applications work fine with single LLM calls + retrieval https://t.co/lI3hfjzDs0
🔍 Key Workflow Patterns:

• Prompt chaining
• Routing
• Parallelization
• Orchestrator-workers
• Evaluator-optimizer

Explained below 👇
💡 Prompt chaining

Sequential LLM calls where output of one feeds into another - like writing content then translating it

Best for tasks with clear subtasks, like:

• Writing + translating content
• Creating outlines then full documents https://t.co/b8uss5FTUX
🔀 Routing

An initial LLM determines which specialized model handles the task, perfect for sorting queries by complexity

Shines when handling different types of inputs:

• Customer service queries
• Difficulty-based task distribution https://t.co/SNDr7ZcdSG
⚡️ Parallelization

Breaking tasks into parallel subtasks or using multiple LLMs to vote on answers

This has two key forms:

• Sectioning: Breaking into subtasks
• Voting: Multiple attempts for confidence https://t.co/yVAM0XKDKI
