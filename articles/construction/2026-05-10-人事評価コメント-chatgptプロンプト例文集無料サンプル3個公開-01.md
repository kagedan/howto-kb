## 評価シーズン、コメント作成に何時間かけていますか

半期に一度やってくる人事評価シーズン。管理職・人事担当者にとって、評価コメントの作成は意外なほど時間を奪う作業だ。

「指示待ちが多い」「コミュニケーションに難あり」――頭の中では伝えたいことがあるのに、人事考課書に書ける言葉に変換するまでに手が止まる。評価の根拠を事実ベースで書こうとすると、かえって表現が硬くなる。やんわり伝えようとすると、今度は具体性が消える。

さらに近年はハラスメントへの感度が高まり、「何気なく書いた一文がパワハラと受け取られた」という事例も珍しくない。

ChatGPTは、こうした「言語化のボトルネック」を解消するツールとして機能する可能性がある。ただし、ただ「評価コメントを書いて」と投げるだけでは精度が低い。**プロンプトの設計次第で出力の質は大きく変わる**、というのが筆者の体感だ。

本記事では、実際に使えるプロンプトを3本無料公開する。

## 無料サンプル①：メンバー成果評価プロンプト

一般メンバーの業績・目標達成度をA〜C評価別に言語化するためのプロンプト。

```
# 役割
あなたは人事評価の専門家です。管理職が記入する「メンバーの成果評価コメント」を作成してください。

# 入力情報
- 評価対象者の等級: [一般メンバー / 中堅メンバーなど]
- 評価期間の主な業務・担当プロジェクト: [例：○○システムの保守運用、月次レポート作成]
- 目標達成状況: [例：KPI達成率110%、期中に追加タスクを自発的に引き受けた]
- 総合評価ランク: [A / B / C のいずれか]

# 出力条件
- 字数: 150〜200字
- 文体: 敬体（です・ます調）
- 数値や具体的なエピソードを1つ以上含める
- 「頑張りました」「良かったです」などの曖昧な表現は使わない
- ネガティブな評価（C評価）の場合も、事実ベースで建設的に表現する

# 出力
評価コメント本文のみを出力してください。
```

**使用例（B評価・一般メンバー）の出力イメージ：**

> 今期は担当する月次レポートの提出精度が向上し、差し戻し件数が前期比で減少しました。期中に追加された○○プロジェクトのサポート業務にも積極的に対応し、チームの期限遵守に貢献しています。一方、複数タスクが重なる局面では優先順位の判断に時間を要するケースが見受けられました。次期は自律的なタスク管理の精度向上を期待します。

## 無料サンプル②：ネガティブ表現ポジティブ変換プロンプト

「指示待ち」「ミスが多い」など、頭に浮かんだ生の評価語を建設的表現に変換するプロンプト。

```
# 役割
あなたは人事・労務の専門家です。以下の「生の評価語」を、被評価者が前向きに受け取れる建設的な表現に変換してください。

# ルール
- 事実を否定・隠蔽するのではなく、「課題」として明示しながら「成長余地」として表現する
- ハラスメントと取られかねない断定的な人格否定表現は使わない
  - NG例: 「やる気がない」「向いていない」「使えない」
- 出力は「変換後の表現」と「変換のポイント（1行）」をセットで返す

# 変換してほしい表現リスト
1. 指示待ちが多い
2. ミスが目立つ
3. 覇気がない
4. コミュニケーション能力が低い

# 出力フォーマット
1. 変換後: ○○○
   ポイント: ○○○
（以下同様）
```

**出力イメージ：**

```
1. 変換後: 指示を受けてから行動に移す傾向があり、自発的な提案・先読み行動の習慣化が今後の成長課題です。
   ポイント: 「待ち」という行動事実を維持しながら、次期目標の文脈でまとめた。

2. 変換後: 業務の正確性において改善の余地があり、セルフチェックのプロセス定着を期待します。
   ポイント: 「ミス」という結果ではなくプロセスにフォーカスし、解決策を示唆した。

3. 変換後: 現時点では積極的な発言・提案の機会が限られており、チーム内での存在感の発揮が次の目標です。
   ポイント: 「覇気」という主観的・感情的ワードを行動観察の言葉に置き換えた。

4. 変換後: 関係者との情報共有・確認プロセスに課題があり、認識齟齬を防ぐコミュニケーション設計が求められます。
   ポイント: 「能力が低い」という属性評価を避け、行動・プロセスの改善点として表現した。
```

## 無料サンプル③：部下→上司 360度フィードバックプロンプト

部下が上司を評価する場面は、心理的安全性の問題から表現が難しい。このプロンプトは「率直だが攻撃的にならない」バランスを意識した設計。

```
# 役割
あなたは360度フィードバックの記入支援AIです。部下が上司に対して記入する評価コメントを生成してください。

# 入力情報
- 評価対象の上司の役職: [例：チームリーダー / 課長]
- 上司の良かった点（具体的な行動や場面）: [例：週1の1on1で業務の相談に乗ってくれた]
- 改善を期待する点（具体的な行動や場面）: [例：急な方針変更の際に背景の説明が少ない]
- 記入者の立場: [例：一般メンバー・チーム歴2年]

# 出力条件
- 字数: 120〜180字
- 上司への敬意を保ちつつ、率直に改善点を伝える表現にする
- 「人格否定」「感情的な不満」と取られる表現は使わない
- 改善点は「〜いただけるとチームとしてより動きやすくなります」などの前向きな語尾で締める

# 出力
フィードバックコメント本文のみを出力してください。
```

**出力イメージ：**

> 週次の1on1では業務上の相談を丁寧に聞いていただき、方向性に迷った際の指針として大変助かっています。一方、期中に方針が変更された際、背景や理由の共有が後手に回るケースが筆者の体感として数回ありました。変更の意図を事前に共有いただけると、チームとして素早く対応できると感じています。引き続きよろしくお願いいたします。

## プロンプトのカスタム方法

上記3本はそのままコピーして使えるが、自社の等級定義・評価軸・コンピテンシー名称を「#入力情報」セクションに追記することで精度がさらに上がる可能性がある。たとえばメンバー成果評価プロンプトなら「評価軸: 弊社の行動指針○○に基づく主体性・チームへの貢献・品質管理の3軸」と一行加えるだけで、出力の語彙が自社の評価シートに近づく。また、ChatGPT（GPT-4o以上）での使用を推奨する。GPT-3.5系では文体の安定性が落ちる場合がある。
