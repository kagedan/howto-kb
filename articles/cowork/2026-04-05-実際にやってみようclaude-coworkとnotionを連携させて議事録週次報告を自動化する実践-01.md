【学習目標】この記事を読むと何ができるようになるか

この記事を読むと、以下の3つができるようになります。

* Claude CoworkにNotionプラグインを接続する手順がわかる
* Notionのページを読み込んで文書を自動生成するプロンプトが書ける
* 議事録作成・週次報告・タスク更新の3つの業務を自動化するイメージが持てる

入門編でご説明した「Notionという倉庫」を、いよいよClaude Coworkに「開けてもらう」具体的な方法を解説します。

---

## 1. 準備：NotionプラグインをClaude Coworkに接続する

### そもそも「プラグイン」って何？

Claude Coworkにおけるプラグインとは、\*\*AIと外部サービスをつなぐ「橋渡し役」\*\*です。技術的にはMCP（Model Context Protocol）という仕組みを使っていますが、難しく考える必要はありません。「ClaudeにNotionへのアクセス権を渡す設定」と覚えておいてください。

プラグインを接続すると、Claude CoworkがあなたのNotionのページやデータベースを読み書きできるようになります。

### 接続手順（概要）

```
【Notionプラグインの接続手順】

Step 1: Claude Coworkのプラグイン設定を開く
  → サイドバーまたはメニューから「プラグイン」「連携」を選択

Step 2: Notionを選択してインストール
  → プラグイン一覧からNotionを見つけ、「接続」ボタンをクリック

Step 3: Notionの認証を行う
  → ブラウザでNotionのログイン画面が開くので、許可するページ・DBを選んで承認

Step 4: Claude Coworkに戻って接続確認
  → 「Notionに接続しました」と表示されれば完了
```

> **ポイント**: 接続時に「どのページ・データベースへのアクセスを許可するか」を選べます。最初はよく使うプロジェクトページに絞って接続するとシンプルです。

## 2. 実践①：Notionの議事録ページから要点を抽出する

### シナリオ

あなたは毎週、Notionに会議の議事録を記録しています。しかし、後から「あの件、どこに書いたっけ？」と探し回ることが多い。Claude Coworkに要点を抽出・整理してもらいましょう。

### Before（連携なし）の状況

```
【Notionの議事録ページ（例）】
---
日時：2026年4月1日
参加者：山田、鈴木、田中、佐藤

議題1：Q2マーケティング施策について
  - 田中：SNS広告の予算を先月比120%に増額したい
  - 鈴木：ROIの数字を先に確認すべきでは？来週までにデータ提出
  - 山田：承認。鈴木さん4/8までにレポート共有をお願い

議題2：新サービスのLP作成
  - 佐藤：外注か内製かで議論
  - 結論：今回は内製。田中・佐藤が担当。4/15完成目標
（以下続く…）
```

この議事録から「次のアクション」と「担当者」だけ抽出しようとすると、読んで手作業でまとめる必要があります。

### After（Claude Cowork + Notion連携）のやり方

Claude Coworkに以下のように話しかけます。

```
【Claude Coworkへの指示文（プロンプト）例】

Notionの「2026年4月1日 定例会議 議事録」ページを読んで、
以下の形式でアクションアイテムをまとめてください。

| アクション | 担当者 | 期日 |
|---|---|---|
|（ここに内容を入れる）|（担当者）|（期日）|

期日が明記されていないものは「要確認」と記入してください。
```

### Claudeが返してくれる回答の例

```
【Claude Coworkの出力例】

Notionの議事録ページを確認しました。アクションアイテムは以下の通りです。

| アクション | 担当者 | 期日 |
|---|---|---|
| SNS広告ROIレポートの作成・共有 | 鈴木 | 4月8日 |
| 新サービスLPの内製制作 | 田中・佐藤 | 4月15日 |
```

手動では5〜10分かかっていた作業が、プロンプトを入力してから30秒で完了します。

## 3. 実践②：週次報告書を自動生成する

### シナリオ

毎週月曜日、先週の進捗をまとめた週次報告書を上司に提出しています。Notionにはプロジェクトページと日々のメモが蓄積されているので、それを使って自動生成してみましょう。

### 使えるプロンプトテンプレート

```
【週次報告書の自動生成プロンプト】

Notionの「[プロジェクト名]」フォルダにある先週（[日付範囲]）の
ページをすべて確認して、以下の形式で週次報告書を作成してください。

---
■ 今週の成果（完了したこと）
・

■ 課題・懸念事項
・

■ 来週の予定（アクションアイテム）
・
---

箇条書きは各項目3つ以内にまとめ、ビジネス文書として
上司に送れる形式で出力してください。
```

> **コツ**: プロンプトに「上司に送れる形式で」「3つ以内で」などの制約を入れると、出力がコンパクトに整います。

## 4. 実践③：NotionのタスクをAIに更新してもらう

### シナリオ

会議でタスクの担当者や期日が変更になったとき、Notionのデータベースを更新するのが面倒。Claude Coworkに言葉で伝えるだけで更新してもらいましょう。

### 使えるプロンプトテンプレート

```
【Notionタスク更新プロンプト】

Notionの「タスク管理DB」で、以下の変更を行ってください。

・「LP制作」のタスクの担当者を「田中」→「佐藤」に変更
・期日を「4月15日」→「4月20日」に変更
・ステータスを「進行中」のままにしておく
```

これだけで、Claude CoworkがNotionのデータベースにアクセスして値を書き換えてくれます。

---

## 5. 上手に使うためのプロンプトの3原則

Claude CoworkでNotionを操作するとき、以下の3点を意識するとスムーズです。

### 原則1：「どのページか」を明確に伝える

```
❌ 曖昧な指示
「Notionを読んで報告書を作って」

✅ 明確な指示
「Notionの『2026年Q1 マーケティングプロジェクト』ページを読んで報告書を作って」
```

### 原則2：「出力形式」を先に指定する

何も指定しないと長文が返ってきがちです。「表形式で」「箇条書きで3点」など形式を先に伝えましょう。

### 原則3：「誰向けの文書か」を伝える

```
「上司への報告書として、簡潔にまとめて」
「チームメンバーへのSlackメッセージとして、カジュアルな文体で」
```

## 【まとめ】重要ポイントのおさらい

* **接続は4ステップ**: プラグイン設定→Notionを選択→認証→完了
* **議事録の要点抽出**: 「◯◯ページを読んで、アクションアイテムを表形式で」
* **週次報告の自動生成**: 指定期間のページをまとめて「報告書フォーマットで」出力
* **タスクの更新**: 変更内容を言葉で伝えるだけでNotionを書き換えられる
* **プロンプトの3原則**: ①ページ名を明確に ②出力形式を指定 ③誰向けかを伝える

応用・事例編では、これをチーム全体で活用するための設計方法と、さらに高度なワークフローを紹介します。
