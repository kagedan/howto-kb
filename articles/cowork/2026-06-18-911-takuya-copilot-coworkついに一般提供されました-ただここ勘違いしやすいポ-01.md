Copilot Coworkついに一般提供されました！

ただここ、勘違いしやすいポイントがあって、結構大事なので共有しておきます。

『Microsoft 365 Copilot の月額ライセンスを払えば Cowork も使い放題』ではないんですよね。Cowork の実行分は『Copilot Credits』による従量課金になります。

そもそも Cowork は普通のチャット型Copilotとはちょっと違います。

メール送信、会議調整、Word・Excel・PowerPoint・PDF作成、Teamsへの投稿、組織内検索みたいな作業を、複数ステップでまたいで実行してくれる『作業実行型エージェント』なんですよね。各アクションは承認してから実行される設計になっています。

構造をシンプルにするとこうです。

・Microsoft 365 Copilot の月額ライセンス
→ Copilot Chat、各アプリ内のCopilot、Researcher、Analyst、Agent Builder などの基本体験を使うための固定費

・Copilot Cowork
→ 複雑で長時間・複数ツールにまたがる作業を実行する追加機能。こちらは Copilot Credits を消費する従量課金

Cowork のタスク料金は『モデル利用』『コンテキスト取得』『ツール呼び出し』『実行時間』の4要素で計算されます。

そしてこの Copilot Credits、Cowork だけじゃなく Copilot Studio、Dynamics 365 agents、Power Platform、Work IQ APIs などでも使われる共通の利用単位なんですよね。

価格は Pay-as-you-go で 1 Copilot Credit あたり $0.01。100クレジットで1ドル、1,000クレジットで10ドルというイメージです。とはいえ1回のタスクが何クレジットになるかは、作業の重さで変わります。

Microsoft はタスクを light / medium / heavy の3段階で捉えていて、
・軽いタスク → 少ない情報源・限定的な推論・出力1つ程度
・中程度 → 複数ソース・構造化された推論・複数出力
・重いタスク → 広範囲の情報収集・深い推論・多数の出力

という分類です。つまり「使うほど高くなる」構造です。

ただ、ここがよくできていて、ちゃんと管理機能も用意されているんですよね。

Microsoft 365 管理センターの『Cost Management』ダッシュボードで、Copilot Credits の消費状況を確認したり、ユーザー・グループ・テナント単位で利用制限・予算・アラート・上限を設定できます。現時点でこの使用量ベース課金の管理対象は Cowork と Work IQ API が中心です。

支払い方法は2種類。
・PayGo → 使った分だけ後払い（$0.01 / Copilot Credit）
・P3 → 一定量のCopilot Creditsを前払い購入して割引を受ける方式（1年単位。未使用分は年次期間終了時に失効、購入分を超えたら追加購入かPayGo課金）

課金開始は一般提供日から。とはいえ Frontier期間中（2026年3月30日〜6月16日）に Cowork を使っていたテナントは移行猶予があって、2026年7月1日までは利用分が課金されません。Frontier利用中の顧客は2026年6月30日までに Microsoft 365管理センターで従量課金設定を行うよう案内されています。

実務的にどう捉えるべきか、ここがほんとに大事です。

Cowork は「月額Copilotを契約したら無制限で何でも自動実行できる」世界ではなく、高度なエージェント作業はAIの実行コストに応じて追加課金される、という設計なんですよね。

特にコストが増えやすいのは、こういう使い方です。

複数のメール、会議、ファイル、SharePoint、Teams、Excel、外部プラグインをまたいで調査・判断・作成までやらせるタスク。たとえば「今週の商談状況を確認して、失注リスクを洗い出して、フォローアップメールを作って、Teamsに報告して」みたいな依頼ですね。単なるチャット回答ではなく、複数の情報取得・推論・ツール操作が走るので、軽いタスクよりクレジット消費は大きくなりやすいはずです。

一方で管理者側は「誰に使わせるか」「いくらまで使わせるか」「どのグループに予算を割り当てるか」「上限に達したら止めるか」を管理できます。だから企業導入では、野放しにすると怖いけど、管理設定をすれば運用できる、という位置づけですね。
