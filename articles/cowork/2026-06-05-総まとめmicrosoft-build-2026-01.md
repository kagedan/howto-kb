## 筆者の感想

今年のBuildを一言でいうなら『 **「自分専用に育て続けるAI」を、誰もが手元から作れる時代の幕開け** 』です。

特定の1つのモデルやプラットフォームではなく、「その上で開発者が価値を複利的に積み上げられるか」が、終始一貫したテーマでした。

まず結論として、今年のBuildは次の3つのキーワードで読み解けます。

1. **unmetered intelligence**（従量課金を気にせず使える、ローカルの知能）
2. **agent-first**（エージェントを前提に、OS・デバイス・クラウドを設計し直す）
3. **hill-climbing machine**（自社データで継続的に賢くなり続ける、改善マシン）

本記事では、この3つを軸に、発表内容をわかりやすく整理していきます！

## キーノートの全体像：フロンティア・インテリジェンス・エコシステム

冒頭でSatya Nadellaさんが掲げたメッセージは、とてもシンプルでした。

それは **「どうすれば、あなた自身がこのフロンティア・インテリジェンス・エコシステムにフルに参加できるか？」** という問いです。

主役は、特定の技術やプラットフォームそのものではありません。その上で、開発者が価値を構築し、複利的に積み上げていけるか——ここに焦点が当てられました。

会場で示されたAIスタックは、次の5層で整理できます。

1. ユビキタスな計算ファブリック（エッジ＋クラウド）
2. モデル / コンテキスト / ツール（エマージング層）
3. エージェントやアプリを動かすランタイム
4. 開発ツーリング
5. セキュリティ・コンプライアンス・ガバナンス

## エッジとWindows：「unmetered intelligence」をローカルに

最初のテーマは、エッジ——つまり手元のPCです。

ここでのキーワードが **unmetered intelligence** です。日本語にすると「従量課金を気にせず、使い放題の知能」というニュアンスになります。

### Windowsに載るローカルモデル「Aion 1.0」

Microsoftは、Windows ML / Windows AIの対象を、Windowsのインストールベース全体のGPU（さらにNPU・CPU）へと拡大しました。

すでにOutlookの要約、PowerPointの代替テキスト生成、Teamsの超解像などは、オンデバイスのAIで動いています。AdobeのAfter EffectsやPremiereもWindows MLを活用しているとのことです。

そして目玉が、Windows標準搭載（in-box）の新しいローカルモデル2種です。

* **Aion 1.0 Instruct**：要約・リライト・意図抽出・アクセシビリティ向けの小型モデル
* **Aion 1.0 Plan**：14Bパラメータ・32Kコンテキストの推論＋ツールコール（プランニング）モデル

たとえばAion 1.0 Instructは、従来比でメモリ使用量を約3.4分の1に抑えつつ、要約を6倍、応答を2倍に高速化したと紹介されました。

この2つを使うと、クラウドへの往復なしで、完全にローカルなエージェンティックループを組めます。これが、unmetered intelligenceの実体です。

*Windows標準搭載の新ローカルモデル「Aion 1.0 Instruct」「Aion 1.0 Plan」*

### 「ドリームマシン」Surface RTX Spark Dev Box

ハードウェアの発表も、とても豪華でした。

シリコンでは、Intelの新製品、Qualcomm（ハイエンドのSnapdragon X系と、500ドル未満PC向けの2系統）、そしてNVIDIAの新しいPC向けSoC「N1X」が登場します。N1Xは、CPU・GPU・AIを単一のSoCに統合し、ユニファイドメモリを備えます。

注目の製品が、開発者向けの「ドリームマシン」 **Surface RTX Spark Dev Box** です。

* 最大 **1ペタFLOP** のAI演算性能
* 20個のCPUコア
* **128GB** のユニファイドメモリ
* 最大 **120Bパラメータ** のLLM（大規模言語モデル）を、最大100万トークンのコンテキストでローカル実行

「データセンター級の計算が、デスクトップに来た」と言っても、大げさではありません。年内に、米国のMicrosoft.comで発売予定です。

ノート型の **Surface Laptop Ultra**（NVIDIA RTX Spark搭載・128GBメモリ・終日バッテリー）も今秋発売予定で、OEM各社からも同SoC搭載機が登場します。

さらにJensen Huangさんは、NVIDIA DGX Station（GB300）にWindowsが来ることも明かしました。1兆パラメータ級のモデルをローカルで動かせる、「デスクトップのデータセンター」です。

### 開発者のためのWindows体験

デモでは、「開発者のためのWindows」への進化が示されました。

* **Windows Development Configurations（GA）**：WinGetの構成ファイル1つで、Python・Nodeなどの開発環境を一括セットアップ
* **インテリジェントターミナル**：GitHub Copilotなど好みのエージェントを組み込み、エラーを自動検出して修正案を提示
* **UNIX系コマンドのネイティブ対応**：grep・head・tail・touchなど70超のコマンドを追加。Homebrewもネイティブ対応
* そのほか、縦型タスクバー、PowerToysの新機能「Grab and Move」、WSLコンテナのGPU活用など

デモ機は、通知もニュースフィードもない「静かな」デフォルト設定でした。メインのエージェントが、適切な複雑度のサブタスクをローカルモデルへ委譲する構成も披露され、ローカル消費なら従量課金を気にしなくてよい点が、繰り返し強調されました。

## クラウドインフラ：tokens per dollar per watt

エッジの次は、クラウドです。

Microsoftが一貫して掲げる最適化の方程式が **tokens per dollar per watt**（電力あたり・コストあたりのトークン生成量）です。電子が入り、トークンが出るまでを、端から端まで最適化するという発想です。

![Microsoft Build 2026の基調講演スライド。Datacenter・CPUs・Storage・GPUs・Networking・Memoryの6つのインフラ構成要素をアイコン付きの横並びブロックで示す図](https://static.zenn.studio/user-upload/deployed-images/2e047213aa69df99a48ddb18.png?sha=df2128ec6d285b0ccbd24b658cc60babffd796b0)  
*データセンターは、CPU・ストレージ・GPU・ネットワーク・メモリを端から端まで最適化する対象になる*

興味深いのは、技術の前に「コミュニティからの許し（permission）を得る」原則が示された点です。地域の電力価格を上げない、水利用に配慮する、地元の雇用や税収に貢献する——こうした前提があってはじめて、建設に踏み出すという姿勢でした。

Azureはすでに500を超えるデータセンターを展開し、ハイパースケーラー最大級のフットプリントを持ちます。直近18ヶ月で、最初の10年分を上回る容量を追加したそうです。

*「コミュニティ・ファースト」のインフラ原則。電気料金・水・雇用・税収・地域投資へのコミットメントを掲げる*

### Fairwater、Maia 200、Cobalt 200

象徴的な発表が、初の「AIスーパーファクトリー」 **Fairwater**（ジョージア州＋ウィスコンシン州）です。

AI前提のグラウンドアップ設計で、2階建て構造によりGPUを高密度に配置します。閉ループ冷却により、実質的な水消費はゼロ。年間の水使用量は、レストラン1軒分に相当するとのことです。

シリコン面では、自社製チップの新世代が登場しました。

* **Maia 200**（自社AIアクセラレータ第2世代）：現行の主力GPU比で、tokens per dollarが30%優位。GPT-5.5で検証済みで、Microsoft 365 Copilotを駆動予定
* **Cobalt 200 VM**（プレビュー）：クラウドネイティブ＆エージェント向けの次世代Arm CPU。エージェント呼び出しで33%の低レイテンシ、14%の高速化、23%の高スループットを達成

エージェント時代には、AIアクセラレータとCPUの比率が **「1：1」** に近づくという見立ても語られました。だからこそ、両者の協調設計が重要になります。

## Jensen Huangさん対談：PCが「パーソナルAI」へ

ここで、NVIDIA CEOのJensen Huangさんが、台北からリモートで登場しました。

Jensen Huangさんは、RTX Sparkを「約3年前の、2人の会話から始まったプロジェクト」と振り返ります。そして、印象的なフレーズを残しました。

それは **「PCが、パーソナルコンピュータからパーソナルAIへ進化した」** という言葉です。

外出中にPCへテキストで指示すれば、PC上のツールをAIが自律的に操作し、戻る頃には作業が進んでいる——そんな世界観です。RTX Sparkは、NVFP4という数値フォーマット（両社共同開発）と128GBメモリにより、数千億パラメータ級のモデルをローカルで動かせます。

GPU世代の役割分担も、わかりやすく整理されました。

| 世代 | 主な役割 |
| --- | --- |
| Vera Rubin | エージェント向けに設計された次世代。超低レイテンシに特化 |
| Grace Blackwell | ポストトレーニング・強化学習・推論。NVL72でラック全体が1台のコンピュータに |
| Hopper | 事前学習の世代 |

Microsoftは世界最大のGrace Blackwell導入企業であり、Fairwaterでは、トークン生成コストをHopper比で約30分の1に削減したとのことです。

また、GitHubへのコミット数が「完全に放物線」を描き、直近数ヶ月で3倍に増えたという話も飛び出しました。エージェンティックシステムが、実際に生産的で利益も生んでいる証拠だと語られました。

## Project Solara：エージェントファーストデバイスという新発想

ここからが、agent-firstを象徴する発表です。

既存のフォームファクタ（ノートPCやデスクトップ）に新機能を載せるだけでなく、 **エージェント時代のために、新しいデバイスそのものを作れないか？** ——その問いから生まれたのが **Project Solara** です。

発想の核心は、「1つの形を選ぶ」ことではありません。 **エージェントが、デバイス群（constellation）をまたいで、1つのシステムとして動く** という考え方です。

Project Solaraは、チップからクラウドまでをカバーするターンキー基盤で、3つの柱を持ちます。

1. エンタープライズ対応（AOSP：Androidのオープンソース版をベースにしたデバイスプラットフォーム）
2. エージェント駆動のインタラクション（フォームファクタに適応するjust-in-time UI）
3. 拡張性（自社エージェントの持ち込み）

![「Project Solara」のタイトルと、Enterprise readiness・Just in time UI・Bring your own agentの3本柱とアイコン列を示すスライド、中央に登壇者](https://static.zenn.studio/user-upload/deployed-images/7c5bb0f7e9d4b1c53904e92e.png?sha=743d7a1e9b4bb312a3b0a65908d14b4a70d0c74a)  
*Project Solaraの3本柱（エンタープライズ対応 / just-in-time UI / 自社エージェントの持ち込み）*

会場では、2種類の試作デバイスが初公開されました。

* **据置型（MediaTekシリコン）**：デスク常駐のコンパニオン。Windows Hello for Businessで、歩み寄るだけでセキュアにサインインし、M365 Copilotへ即アクセス
* **携帯型「アクセスバッジ」（Qualcommシリコン）**：指紋でアンロックし、音声やカメラでエージェントを操作。医療現場では、ハンズフリーの音声documentation（話者分離つき）やバイタル確認などを支援

![横長スクリーンに医療現場のワークフロー（患者受付、記録、バイタル取得・検証、服薬管理）を示す4パネルが並び、中央のステージに登壇者が立つ](https://static.zenn.studio/user-upload/deployed-images/c597dc6448f5e5d09835776b.png?sha=15f8d00b2bdde03d2808d8cf94180aab3aa65607)  
*アクセスバッジ型デバイスによる医療ワークフロー（受付・記録・バイタル・服薬管理）*

同じハード・ソフト基盤のまま、エージェントや形状、センサーを差し替えれば、小売・産業・金融・法務などへ展開できます。AccuWeatherやBest Buy、Targetなどが、活用を検討中とのことです。

Qualcomm CEOのCristiano Amonさんとの対談では、スマートフォン中心の垂直統合から、 **エージェントがデジタル体験の中心になる、オープンで水平なプラットフォーム** への移行が語られました。

![大型スクリーンに『Project Solara』のタイトルとaka.ms/ProjectSolaraのURLが表示され、ステージ上に試作デバイス群が並び中央に登壇者が立つ](https://static.zenn.studio/user-upload/deployed-images/c5b4768915214971e6f4a24d.png?sha=183fb94428a1b62a814613b1571a56bb40ffd10e)  
*ステージに並ぶProject Solaraの試作デバイス群。形やセンサーの異なるデバイスが「1つのシステム」として描かれた*

視点を広げると、Project SolaraはRabbit R1のような「スマートフォンを代替するAIデバイス」の系譜とも見ることができます。さらに遡れば、1987年にAppleが構想した **「Knowledge Navigator」** ——音声で情報を探し、AIエージェントが代わりに動いてくれるパーソナルアシスタントのビジョン——にじわじわ近づいているとも言えます。OSにWindowsではなくAndroid（AOSP）ベースを採用しているのも興味深く、デバイス販売よりも「その上で消費されるトークンや関連サービスで収益を上げる」プラットフォーム戦略と捉える見方もあります。

## データ層とMicrosoft IQ：エージェントに「正しいコンテキスト」を

スタックを一段上がると、モデルとコンテキストの層です。

エージェントは「保存・検索・推論・行動・学習」を、連続的なループで回します。この呼び出しパターンは、従来のユーザー向けアプリとは大きく異なります。

なお、Foundryのモデルカタログは、11,000を超えるモデルをそろえる業界最大級の規模です。OpenAIからAnthropic、自社のMAIまでが並びます。先週には、OpenAIのリアルタイム音声モデルと **Claude Opus 4.8** が追加されました。

### 4つの「IQ」でエージェントを接地する

今年の主役級の発表が **Microsoft IQ** です。

Microsoft IQは、4つのIQで構成されます。

## エージェントランタイムとセキュリティ：MXCとOpenClaw

### OSが封じ込める「MXC」

そこで導入されたのが **MXC（Microsoft Execution Containers）** です。

OSネイティブの仕組みで、エージェントの隔離・封じ込めを、ポリシーとして強制します。軽量な処理向けの「プロセスレベル分離」から、ユーザー分離の「セッションレベル分離」、最大隔離の「Windows 365 for agents」まで、用途に応じて選べます。

ポイントは、 **誰が作ったエージェントであっても、OSが強制する** という点です。

## 量子：Majorana 2

そして、もう1つの「次なるフロンティア」が、量子コンピューティングです。

昨年のMajorana 1（初のトポロジカルQPU：量子演算ユニット）に続き、第2世代の **Majorana 2** が発表されました。量子研究そのものにも、DiscoveryのエージェンティックループでAIを活用し、「数年分の研究を、直近1年に圧縮した」と語られました。

Majorana 2の要点は、次の通りです。

* キュービット寿命は **平均20秒、最大約1分**。一般的な方式のマイクロ秒級に対し、前世代比で約1000倍の信頼性
* Majorana 1と同じフォームファクタ（100分の1ミリ）で、デジタル制御を維持
* クレジットカードより小さいチップに、100万キュービットを載せる道筋

位置づけは「Majorana 1＝基礎物理の実証、Majorana 2＝スケールのエンジニアリングの開始」です。公式ブログでは、エージェンティックAIの力も借りて、2029年までにスケーラブルな量子マシンを実現すると明言されています。

## まとめ：Build 2026は「フロンティアを自分のものにする」場

最後に、今年のBuildを振り返りましょう。

冒頭の問いは **「どうすれば、あなた自身がフロンティア・インテリジェンス・エコシステムにフルに参加できるか？」** でした。その答えが、3つのキーワードに凝縮されています。

* **unmetered intelligence**：Aion 1.0やSurface RTX Spark Dev Boxで、ローカルの知能を使い放題に
* **agent-first**：MXCやProject Solara、Agent 365で、OS・デバイス・運用をエージェント前提に作り直す
* **hill-climbing machine**：Frontier TuningとRLEで、自社データから継続的に賢くなり続けるモデルを所有する

特に印象的だったのは、「知能を消費する」段階から、 **「知能を所有し、自分のコントロール下でフロンティアに到達する」** 段階への転換です。

Satya Nadellaさんは、最後にこう締めくくりました。技術には、2つの物語がある、と。1つは「技術が力を集中させ、人間の主体性を奪う」物語。もう1つは「次の波で、開発者・科学者・企業・コミュニティに、機会を解き放つ」物語です。

後者を本物にするのが、私たちの仕事——。この言葉に、開発者向けカンファレンスとしてのBuildの矜持を感じました。

膨大な発表でしたが、軸は明快です。あなたなら、この「自分専用に育てられるAI」を、どんな価値づくりに使ってみたいでしょうか？
