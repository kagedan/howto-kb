## はじめに

`GitHub Copilot` `ClaudeCode` `Codex`  
各社は、これまで定額制のサブスクサービスを提供してきました。  
しかし、各社軒並み「使用量ベース」に移行しています。  
それでも、`Gemini Code Assist`はずっと無料で使えていました。  
それも、2026年6月16日にサービス提供が終了します。

そこで、私は完全無料のIDEを開発することにしました。  
その名も「**MicroCode**」です。  
当初は**最小構成**で拡張性の高い、ユーザーが育てていくIDEとしてコンセプトを打ち出しました。  
現状は、ClaudeCode、Codex、Antigravityなどの便利機能の全部盛りなIDEとなりつつあります。

例えば、Codexのペット機能、ClaudeCodeのCowork機能、一気通貫の自律型エージェント、それらがてんこ盛りの内容をなっています。

しかも、拡張機能とは別に、IDEで使用するツールは、ユーザーが追加することで、無限大の拡張性を実現できます。  
もう、開発も大詰めの段階です。  
すでに、簡単なプロダクトは一気通貫で完成される域まで到達しています。

Electronで開発しているので、見た目や使用感はVSCodeに寄せいています。  
ちなみに、`Cursor`や`Antigravity`のようなVSCodeクローンではなく、ゼロから構築しています。

使用感が似ているので`VS Code`、`Antigravity`、`Cursor`などのIDEを使用してきた方は、すぐに使い慣れると思います。

## デスクトップマスコット

作業中は、一生懸命キーボードを叩き、エラーが出たら頭を抱え、タスクがすべて完了したら通知音を鳴らし、手を振って知らせます。  
IDEを最小化していても、マスコットを表示していれば進捗を確認できます。更に、ドラッグで任意の場所に移動でき、大きさの変更や最前面表示の有無も切り替えられます。頭上のバルーンをクリックすれば、ミニチャットが開いてIDEを最小化したまま会話や指示出しもできます。  
将来的には、このマスコットをモバイルアプリ化して、外出先でも進捗を確認したり追加の指示を出せるように計画中です。  
（作業していない間は、歩き回ったり、座って休んだり、踊ったりします）

このIDEの最大の特徴は、拡張性です。  
標準装備されているツールでは物足りないと感じればツールを追加したり、他にはないニッチなツールを追加したり、ご自身のプロダクトに合わせたツールを作成・追加し、ユーザーが育てて行くことも可能です。その、ツールもエージェントに依頼して追加することができます。  
つまり、最小構成ながら、カスタマイズによって、ユーザー好みにどんどん成長していく成長型IDEとなっています。

# MicroCode

## ローカル LLM と育てる、自己拡張型 IDE

**MicroCode** は、ローカル LLM を「質問に答えるチャット」から「プロジェクトを理解し、計画を立て、実装し、検証する開発パートナー」へ引き上げるための IDE です。

コードを書く。ファイルを読む。コマンドを実行する。ブラウザで結果を見る。失敗したら原因を探し、修正し、もう一度検証する。

MicroCode は、その一連の作業をエージェント自身が進められる環境を目指しています。

---

## なぜ MicroCode なのか

一般的な AI コーディング支援は、会話が長くなるほど弱点が見えてきます。

* 何を終え、何が残っているのかを見失う
* 同じ調査や同じ失敗を繰り返す
* コードを書いた後の検証が人間任せになる
* ツールが増えるほど使い方を誤りやすくなる
* クラウドへ送れないコードや資料を扱いにくい

MicroCode は、これらを IDE の内部構造から解決します。

単に回答を生成するのではなく、**計画、実行、検証、記憶**を明示的な仕組みとして持たせる。それが MicroCode の設計思想です。

## 4つの柱

### 1. Plan: 作業の現在地を失わない

MicroCode のエージェントは、複雑な作業を始める前に計画を作ります。

`plan_and_track` は、会話履歴とは独立した永続的なタスク台帳です。セッションが長くなっても、「何を終えたか」「今どこにいるか」「次に何をするか」が残ります。

大きな依頼には `hierarchical_task_orchestrator` が使えます。親タスク、子タスク、実行可能な葉タスクへ分解し、PLAN パネルで階層的に追跡できます。

主な特徴:

* Plan-first Gate により、計画なしの見切り発車を防止
* タスクの `pending` / `in_progress` / `completed` / `blocked` / `failed` を管理
* 多段の親子タスク
* 葉タスク単位の進捗集計
* 親タスクの自動同期
* 折りたたみ可能な PLAN パネル

### 2. Act: IDE と OS を操作できる

MicroCode は、エージェントに「手」を与えます。

Monaco Editor を中心とした IDE 上で、ファイル操作、検索、ターミナル、ブラウザ確認、Git、静的解析、テスト実行を一つの自律ループへ接続しています。

代表的な操作:

* ファイルの読み取り、作成、部分編集、削除、リネーム
* PowerShell / CMD を考慮したコマンド実行
* ワークスペース検索とシンボル検索
* Git status、diff、log、commit
* 内部ブラウザの操作とコンソール確認
* API 呼び出し、ネットワークログ確認
* 依存関係の解析

エージェントの操作はチャット上の細いツールバッジとして表示されます。実行中、成功、失敗を確認でき、必要に応じて詳細を展開できます。

### 3. Verify: 書いて終わりにしない

MicroCode は、生成コードを無条件に正しいものとして扱いません。

`verification_cycle` は、静的解析とテスト実行を完了ゲートとして扱います。問題が残っていれば、タスクを完了扱いにせず、修正と再検証を続けます。

さらに、同じ失敗が繰り返された場合は `self_healing_escalator` が検出します。一定回数以上の失敗後は、ユーザーへ選択肢を提示するか、設定に応じて作業を継続できます。

品質を支えるツール:

* `code_analyzer`
* `test_runner`
* `file_diff`
* `code_formatter`
* `ai_code_review`
* `architecture_guard`
* `performance_profiler`
* `verification_cycle`
* `self_healing_escalator`

### 4. Learn: 経験を次の作業へ残す

MicroCode は、毎回ゼロから考え直すエージェントで終わりません。

`knowledge_search` は、過去の失敗、成功パターン、設計判断、ツール利用メモを必要な時だけ検索します。全情報を毎回プロンプトへ詰め込まず、必要な知識だけを取り出します。

`knowledge_capture` は、再利用価値のある学びを Markdown Wiki として保存します。`context_compactor` は、長い履歴やツール結果を整理し、次の判断に必要な要点と保存候補を抽出します。

現在の基礎:

* 必要時だけ使う長期記憶検索
* 再利用可能な学びの保存
* セッション復帰ログ
* 会話とツール結果の圧縮
* `knowledge/index.md` と `knowledge/log.md`

今後は、セッション履歴から上限制御された `MEMORY.md` を生成し、原文ログを残したまま、エージェント向けの圧縮済み長期記憶として活用します。

## ローカルで動く意味

MicroCode は、ローカル LLM を前提に設計されています。

* コードや資料を外部サービスへ送らずに扱える
* モデル、ツール、ルールを自分の環境に合わせて調整できる
* ネットワークに依存せず、開発環境の内側で完結できる
* ツール定義や Wiki を Markdown / JSON / Python として監査できる

ローカルであることは制限ではありません。自分の開発環境を、自分の手元で育てられるという選択肢です。

## ツールを後から増やせる

MicroCode の Tool Registry は、ツール定義と実装を分離しています。

```
tools/
  defines/
    my_tool.json
  implementations/
    my_tool.py
```

JSON で入出力を定義し、Python で `execute(**kwargs)` を実装する。バックエンド起動時に動的ロードされ、エージェントが利用できるようになります。

この構造により、MicroCode は完成品を一方的に受け取る IDE ではなく、利用者の目的に合わせて成長する IDE になります。

## 使い方のイメージ

たとえば、「設定画面へ新しい機能を追加し、動作確認まで行って」と依頼した場合。

```
User request
  -> 計画を作る
  -> 関連ファイルを調べる
  -> 必要なコードを編集する
  -> 静的解析を実行する
  -> テストする
  -> ブラウザで確認する
  -> Diff を確認する
  -> PLAN を完了する
  -> 再利用可能な学びを Knowledge に残す
```

ユーザーは、エージェントが何をしているのかを PLAN パネルとツールログで追跡できます。

## Pure Utility

MicroCode の UI は、開発者がコードとエージェントの作業に集中できるよう設計されています。

* 高コントラストなダークテーマ
* Monaco Editor を中心とした実用的なレイアウト
* 情報量を保ちながら、視覚的なノイズを抑えたツールログ
* 思考プロセスのストリーミング表示
* バックエンド異常終了時の自動復旧と状態通知
* ファイル読み込み中の視覚表示
* デスクトップマスコットによる視覚的な状況確認

派手さより、使い続けられること。  
装飾より、開発の流れを止めないこと。

MicroCode は、そのための IDE です。

## これから

MicroCode は現在も成長中です。次の大きな開発テーマは、以下の3つです。

### サブエージェント

調査、レビュー、デバッグを独立したコンテキストへ委譲します。メインエージェントは計画を維持しながら、専門エージェントの結果を統合します。

### MICROCO MOBILE

外出先から PC 上の MicroCode を監視し、追加指示、判断回答、緊急停止を行う Android コンパニオンアプリです。PC が停止中の場合は、次回 IDE 起動時に受信する「予約」を作成します。

### Knowledge System

一次資料、LLM が維持する Wiki、運用 Schema を分離し、Ingest / Query / Lint のサイクルを構築します。セッション復帰ログからは、文字数上限を持つ圧縮済み長期記憶を生成します。
