毎週木曜日は「経営判断・データ分析の自動化」をテーマにお届けしています。

「売上データを集めて、前年比を出して、傾向を分析して、対策を考える」——この一連の流れに、毎月どれくらいの時間を使っていますか？多くの経営者・管理職の方が「丸1日かかる」と答えます。しかし、Claude Coworkを使えば、この作業の大部分を自動化できます。データを入れるだけで、分析レポートが出てくる「仕組み」を一緒に作っていきましょう。

## Claude Coworkとは？

Claude Coworkは、Claudeデスクトップアプリに搭載されている「作業を丸ごと任せるモード」です。普段のチャット（Chat）とは違い、フォルダとファイルを使って「毎回同じ品質で仕事をしてくれる環境」を作ることができます。プログラミングの知識は一切不要です。

## 月次売上レポートの自動生成｜Excel集計→分析→報告書が30分で完成

経営者にとって毎月の売上レポート作成は欠かせない業務ですが、「数字を集める→比較する→傾向を読む→文章にまとめる」という作業は時間がかかります。noteでも「データ起点で勝ち筋を描く」ことの重要性が指摘されていますが、その前段階の「データ整理と分析」に時間を取られてしまっては本末転倒です。

> ポイント：「データを入れたら分析レポートが出てくる」仕組みを一度作れば、毎月使い回せます

**手作業の場合：** 半日〜1日（データ収集・整理・分析・レポート作成）

**Cowork自動化の場合：** 30分（データ貼り付け→確認→完成）

**Cowork用のフォルダ構成：**

```
月次売上分析/
├── CLAUDE.md
├── 入力/
│   └── 2026年3月_売上データ.txt
├── 出力/
│   └── （ここにレポートが生成されます）
└── テンプレート/
    └── レポート形式.txt
```

**CLAUDE.mdファイルの中身（そのままコピーして使えます）：**

```
# 月次売上レポート自動生成
## あなたの役割
あなたは経営分析の専門家です。入力フォルダの売上データを読み取り、経営者向けのわかりやすいレポートを作成してください。
## 処理ルール
- 入力/フォルダにある最新の売上データファイルを読み取る
- 前月比・前年同月比を計算する
- 好調な部門と課題のある部門を特定する
- 数字の羅列ではなく「何が起きているか」「なぜそうなったか」を言葉で説明する
- 専門用語は使わず、経営者が読んですぐ理解できる表現にする
## 出力フォーマット
以下の構成でレポートを作成し、出力/フォルダに保存してください。
ファイル名：「YYYY年MM月_月次売上レポート.md」
■ 今月のサマリー（3行以内）
■ 売上の全体像（前月比・前年同月比）
■ 部門別の分析（好調な点・課題）
■ 注目すべき変化（トレンドや異常値）
■ 来月に向けた提案（3つまで）
## 注意事項
- 推測や憶測は入れない。データに基づいた分析のみ行う
- 悲観的すぎる表現は避け、改善の方向性を示す
- グラフは作成しない（テキストベースのレポートに集中する）
```

**Coworkに入力する指示文（そのままコピペで使えます）：**

```
入力フォルダの売上データを読み取って、月次売上レポートを作成してください。
テンプレートフォルダのレポート形式も参考にしてください。
出力フォルダに保存してください。
```

YouTubeでも「ChatGPTがExcelファイルを読み込んで分析・グラフを作成する」動画が多数公開されており、AIによるデータ分析は実用段階に入っています。Coworkなら、この流れを「毎月ボタンひとつ」で再現できます。

## KPIモニタリングの自動化｜「数字を見る」から「異変に気づく」へ

「毎週KPIを確認しているが、結局"見ているだけ"になっている」——そんな声をよく聞きます。noteの記事でも「AI分析でビジネス課題を瞬時に解決する」方法が紹介されていますが、大切なのは「数字を見る」ことではなく「異変に気づく」ことです。

> ポイント：Coworkに「前週との差が10%以上ある項目だけ報告して」と指示すれば、見るべきポイントが自動で絞り込まれます

**Cowork用のフォルダ構成：**

```
KPIモニタリング/
├── CLAUDE.md
├── 入力/
│   ├── 今週のKPI.txt
│   └── 先週のKPI.txt
├── 出力/
│   └── （ここにアラートレポートが生成されます）
└── 参考/
    └── KPI定義と目標値.txt
```

**CLAUDE.mdファイルの中身：**

```
# KPIモニタリング・異変検知
## あなたの役割
あなたは経営ダッシュボードの分析担当です。毎週のKPIデータを比較し、経営者が注目すべきポイントだけを簡潔に報告してください。
## 処理ルール
- 入力/フォルダの「今週のKPI」と「先週のKPI」を比較する
- 参考/フォルダの目標値と照らし合わせる
- 前週比で10%以上の変化がある項目をピックアップする
- 目標値に対して未達の項目を優先的に報告する
- 「順調」な項目は1行で済ませ、「要注意」な項目を詳しく解説する
## 出力フォーマット
出力/フォルダに以下の形式で保存してください。
ファイル名：「YYYY年MM月DD日_KPIアラート.md」
🟢 順調な指標（箇条書きで簡潔に）
🟡 注意が必要な指標（変化の内容と考えられる要因）
🔴 対応が必要な指標（数値の変化、影響範囲、推奨アクション）
## 注意事項
- すべての数字を羅列しない。経営者の判断に必要な情報だけに絞る
- 「なぜ変化したか」の仮説は、データから読み取れる範囲にとどめる
- やらないこと：グラフ作成、予測モデルの構築
```

**Coworkに入力する指示文：**

```
今週と先週のKPIデータを比較して、注意すべき変化をまとめてください。
目標値との乖離も確認してください。
```

Redditのr/ClaudeAIでは「AI関連の支出の73%がAnthropicに集中している」という興味深い投稿がありました。ビジネスの現場でもClaude活用が急速に広がっていることの表れです。

## 競合動向レポートの自動収集｜情報収集に2時間→確認に10分

「競合の動向を把握したいが、調べる時間がない」——これは多くの経営者が抱える課題です。Webサイトのチェック、ニュース検索、SNSの確認を手作業で行うと2時間以上かかりますが、Coworkなら情報の整理と要約を自動化できます。

> ポイント：「何を知りたいか」をCLAUDE.mdに書いておけば、毎回同じ視点で情報が整理されます

**Cowork用のフォルダ構成：**

```
競合分析/
├── CLAUDE.md
├── 入力/
│   └── 今週の収集情報.txt
├── 出力/
│   └── （ここに競合レポートが生成されます）
└── 参考/
    ├── 競合企業リスト.txt
    └── 注目ポイント.txt
```

**CLAUDE.mdファイルの中身：**

```
# 競合動向レポート作成
## あなたの役割
あなたは市場調査のアナリストです。入力フォルダの情報をもとに、経営者が5分で読める競合動向レポートを作成してください。
## 処理ルール
- 入力/フォルダの収集情報を読み取る
- 参考/フォルダの競合企業リストに記載された企業に関する情報を優先的に抽出する
- 注目ポイント.txtの観点（価格変更、新製品、提携、人事異動など）で情報を整理する
- 自社への影響度が高い情報を上位に配置する
## 出力フォーマット
出力/フォルダに保存してください。
ファイル名：「YYYY年MM月DD日_競合動向レポート.md」
■ 今週の重要ニュース（3件以内、影響度順）
■ 各競合の動き（企業ごとに2〜3行でまとめる）
■ 自社への示唆（「だから当社はこうすべき」のヒント）
## 注意事項
- 情報の出典を明記する
- 不確かな情報には「未確認」と注記する
- 5分以内で読み切れる分量に収める
```

**Coworkに入力する指示文：**

```
入力フォルダの情報から、今週の競合動向レポートを作成してください。
自社への影響が大きいものを優先してまとめてください。
```

Qiitaでは「Snowflake × Databricksの次世代データ基盤PoC実践」やZennでは「自律AIが車輪の再発明をする——ツール管理の設計パターン」など、データ分析基盤のAI活用が活発に議論されています。エンジニアの間で話題になっているこうした技術は、やがてCoworkのような「ノーコードツール」に降りてきます。

## CLAUDE.mdを上手く書くコツ

**1. 「やらないこと」を明記する**

データ分析系のCLAUDE.mdでは特に重要です。「予測はしない」「グラフは作らない」「推測を書かない」と書いておくと、AIが余計な作業で暴走することを防げます。

**2. 出力のファイル名ルールを決める**

「YYYY年MM月\_月次売上レポート.md」のように、日付入りのファイル名ルールを決めておくと、出力フォルダにレポートが時系列で蓄積されます。3か月分たまると、それ自体がトレンド分析の素材になります。

**3. 「参考」フォルダで文脈を共有する**

KPIの目標値や競合リストなど「毎回同じ前提条件」は、参考フォルダに入れておきましょう。毎回の指示文に書く必要がなくなり、指示がシンプルになります。

## 安全に使うための注意点

* **機密データの取り扱い**：売上データやKPIなどの社内機密情報をCoworkに入力する際は、社内のAI利用ポリシーを確認してください。Claude Teamプランでは入力データがモデルのトレーニングに使われないことが保証されています
* **数字の確認は必ず人間が行う**：AIが計算した前年比や前月比は、必ず人間の目で検算してください。AIは計算を間違えることがあります
* **バックアップを取る**：入力フォルダのデータは別の場所にもバックアップを保存しておきましょう

## まとめ｜経営データ分析を自動化する3つのポイント

**1. 「毎回同じ分析」はCoworkに任せる**

月次売上レポート、週次KPI確認、競合動向チェック——これらは「やること」が毎回ほぼ同じです。CLAUDE.mdに分析の手順とフォーマットを書いておけば、データを入れるだけで同じ品質のレポートが出てきます。

**2. 「全部見る」から「異変だけ見る」に切り替える**

数字を全部眺めるのではなく、「前週比10%以上の変化」「目標未達の項目」だけをAIに拾わせましょう。経営者の時間は「判断」に使うべきで、「データ整理」に使うべきではありません。

**3. まず1つの定型レポートから始める**

いきなり全社のデータ分析を自動化しようとせず、まずは「月次売上レポート」ひとつだけCoworkで作ってみてください。CLAUDE.mdの書き方に慣れたら、KPIモニタリング、競合分析と徐々に範囲を広げていくのがおすすめです。
