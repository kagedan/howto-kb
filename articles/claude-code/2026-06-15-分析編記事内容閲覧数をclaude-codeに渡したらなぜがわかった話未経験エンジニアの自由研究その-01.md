👇キャリア探究シリーズ

👇完全未経験データエンジニア奮闘記

拙い文章ですが、読んでいただいているみなさま、本当にありがとうございます☺️  
完全未経験でデータエンジニアの私だからこその視点・言葉で、少しでも発信を続けていきたいと思う、今日この頃。

そんなある日、ふと思ったのです。

> 「この記事、なんで伸びてるんだろう？🤔」

Zennの統計ページやGA4を見れば「いつ・どの記事が伸びたか」はわかります。  
でも、**「なぜ伸びたのか？」** は、数字だけを見ていてもわかりません。

そこで思いついたのが、

**『記事の内容と閲覧数データを組み合わせて、Claude Codeに"なぜ？"を分析してもらおう！』**

そこで今回の記事では、その仕組みを作って実際に分析してみた結果を、自由研究と題してまとめます。

## Zennの統計ページやGA4だけでは「なぜ？」はわからない

Zennの統計ページやGA4を使えば、以下のことはわかります。

| わかること | 例 |
| --- | --- |
| どの記事が伸びたか | 奮闘記その１が今週+17回 |
| いつ伸びたか | 06-08以降に急増 |
| どこから来たか | Zenn内の回遊 or 検索流入 |

でも、肝心の **「なぜ伸びたのか？」** はわかりません。

> * タイトルが良かったから？
> * トピックの需要が高かったから？
> * 別の記事からの流入があったから？

数字だけを眺めていても、答えは出てきません。

**「なぜ」を知るには、記事の中身と数字を同時に見る必要があります。**

## 解決策｜記事内容 × 閲覧数を Claude Code に渡す

そこで思いついたのが、以下の構成です。

#### リポジトリ構成

```
YurieShoji/zenn-articles/
├── articles/               # 記事の内容置き場
│   ├── article-slug-1.md
│   ├── article-slug-2.md
│   └── ...
├── stats/                   # 閲覧数データ置き場
│   ├── 2026-06-07.json
│   └── ...
```

「記事内容」と「閲覧数」。  
この2つを**同じリポジトリに置く**ことで、Claude Codeが両方を参照した上で **「なぜ？〇〇したのか？」** を分析できます。

## 実際に Claude Code で分析してみた

### 投げたプロンプト

以下のようなプロンプトを Claude Code に渡しました。

> articles/ と stats/ を参照して、閲覧数が100回を超えている記事について、  
> 「なぜ伸びているのか」を分析してください。

### 分析対象

閲覧数100回超えの2記事が対象になりました。（※閲覧数は、本分析実施時点の数値）

| No. | 記事 | 閲覧数 |
| --- | --- | --- |
| 1 | 【What?】「ビジネス」×「データエンジニア」とは？｜キャリア探求シリーズ１ | 117回 |
| 2 | さっそくの"絶望"。救ってくれたのは『伴走支援型AI🐧』だった｜奮闘記その１ | 118回 |

### 返ってきた分析結果

---

#### ① No.1(キャリアシリーズWhat編)が伸びている理由

---

#### ② No.2(奮闘記その1)が伸びている理由

* **AI/ChatGPTトピックの爆発的需要**

  + topics に `ai` `chatgpt` が含まれる。Zennの `ai` タグは現在最も流入の多いカテゴリの一つ。
* **タイトルの情緒的引力**

  + 「さっそくの"絶望"」「救ってくれたのは」という感情の弧がタイトルに凝縮されている。技術記事としては異例の物語性で、クリック率を引き上げやすい。
* **奮闘記その２との連動効果**

  + その２が公開された直後の06-07以降、その１の伸びが加速している。
  + その２を読んだ人がその１に遡る **「シリーズ内逆流動線」** が発生していると考えられる。

|  | 奮闘記その２ diff | 奮闘記その１ diff |
| --- | --- | --- |
| 06-08 | +31 | +17 |
| 06-09 | +9 | +8 |
| 06-10 | +12 | +14 |

## 分析して気づいたこと

Claude Codeに分析してもらって、3つの大きな気づきがありました。

### 気づき①｜「シリーズ内逆流動線」がうまく機能していたこと

シリーズものについて、個人的にあまり意図せず関連記事のリンクを置いていました。  
その方が気になってくれた読者が回遊しやすいよなと考えていたためです。  
それが想定以上に機能していたことがわかりました。

### 気づき②｜タイトルの「感情の弧」がうまく機能していたこと

「さっそくの"絶望"」「救ってくれたのは」というタイトルは、  
当時の自分の心情を赤裸々に伝えられるワードとして、練りに練って設計したものでした。

Claude Codeに分析してもらうことで、  
**「これが情緒的引力になってクリック率を上げている」** と言語化されて、  
自身のつまづきをこのワードに込めることができていたのかなと、振り返ることができました。

また **「自身の感情を投影した言葉は、相手に伝わる」** ということを学びました。

### 気づき③｜「ビジネス×データエンジニア」が差別化キーワードになっていたこと

最近「FDE」のワードも流行ってきていますが、  
私はあえて「ビジネス×データエンジニア」と表現していました。

なぜなら、FDEがこれだけ話題になる前から、私の所属する株式会社ギックスでは、  
**「ビジネス×データエンジニア」という言葉を使って、データエンジニアのキャリアや働き方を表現してきていたから**です。

それが「希少なポジション」という形で差別化キーワードとなっていたのかなと思います。  
※単純にFDEが脚光を浴びているからこそ、「ビジネス」×「エンジニア」というワードが目を引いたのかな？とも思います。

## 一方で、「本当にそうかな？」と感じた観点

一方で、「本当にそうと言えるかな？」と感じたAIの分析結果について。

> * **読者設定が明確**
>   + 「こんな方に読んでほしい」セクションで対象読者を絞ることで、共感した人がシェアしやすい。

この点については、**「読者設定が明確だから伸びた」と言い切れるのか？** という疑問が残りました。  
なぜなら、読者設定が明確な記事もあれば、そうでない記事もあるからです。  
また読者設定は記事内に書いてあり、もうすでに閲覧してくださっている状態だから、  
そもそも関係ないのでは？という疑問を感じました。

## 結論｜「なぜ？」は記事の中身を見ないとわからない。ただし、鵜呑みにしない。

GA4や統計ページだけでは絶対に辿り着けない分析結果でした。

記事の構成・タイトル・topicsといった、  
**定性情報と数字を掛け合わせて初めて、「なぜ伸びたのか」が見えてきます。**

一方で、AIの分析結果をそのまま鵜呑みにするのは危険だとも感じました。  
「読者設定が明確だから伸びた」のように、**因果関係が曖昧な分析**も混じっています。

大事なのは、**AIの分析を「仮説」として受け取り、自分の頭で検証すること**。  
それこそがまさに、ギックスが掲げている**AI-Informed** でもあります。  
<https://www.gixo.jp/news-press/30720/>

今回の分析はあくまでスタート地点であり、これをもとに記事を改善して、  
また数字で検証するというサイクルを回していきたいと思います。

## おわりに

GA4や統計ページは「何が起きているか」を教えてくれます。  
でも「"なぜ"起きているか」はわかりません。

今回、Claude Code に「記事内容 × 閲覧数」を渡して分析してみた結果、  
**「数字だけでは絶対にわからなかった "なぜ？"」** が見えてきました。

そして、完全未経験エンジニアとしてまだまだ試行錯誤の毎日ですが、  
**「自分が作ったものを、自分で分析して、次に活かす」** というサイクルが回り始めた気がして、  
素直に嬉しかったです😊🍀
