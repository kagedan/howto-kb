# ■ AIに仕事を丸投げして"想定外の結果"が返ってきた経験、ありませんか

![](https://pbs.twimg.com/media/HMTocYabAAAaLYZ.jpg)

## ▍ 「ちょっと直して」が大惨事になる

Codexに「整理して」と指示したことはありますか。
**返ってきた結果に血の気が引いた人、多いはずです。**

僕も最初のころ何回もやらかしました。

- 頼んだのは1箇所なのに**関係ないファイルまで10個書き換えられた**
- AIの方針がブレて**何をやったのか追えなくなった**
- 「小さな変更だけ」なのに**プロジェクト全体をリニューアルされた**
Codexの能力が低いわけじゃない。
**指示の出し方に問題があるんです。**

## ▍ この"事故"には名前がある——「AIドリフト」

「AIが意図からズレていく現象」を**AIドリフト**と呼びます。
人間のチームだって同じですよね。

方針を決めずに作業を始めると完成品がズレる。
AIも全く同じなんです。

- **計画なしで作業させると、AIは自分の解釈で突っ走る**
- **指示が曖昧なほど、AIドリフトのリスクが上がる**
- **ファイル数が多いほど、事故の被害が拡大する**
僕は500時間これでクレジットを溶かしました。
でもある機能で**この問題がほぼゼロになったんです。**
それがPlan Modeです。

# ■ Plan Modeは「まず計画、次に実行」——ただそれだけ

![](https://pbs.twimg.com/media/HMToed1bUAAaif9.jpg)

## ▍ 料理で例えるとわかりやすい

仕組みはめちゃくちゃシンプル。
**「計画」と「作業」を完全に分ける。** これだけなんですよ。

- ❌ **Plan Modeなし**: 「カレー作って」→ AIが勝手に料理開始。出てくるのはなぜかシチュー
- ✅ **Plan Modeあり**: 「カレー作って」→ 「材料と手順はこれ。OKですか？」→ 確認してからスタート
**たったこれだけで成果物の精度が劇的に変わります。**

## ▍ 「読み取り専用の安全装置」がかかる

Plan Mode中、AIは**ファイルを一切編集しません。**
コードを読んで分析するだけです。

この「読み取り専用」状態で——

- **影響範囲を洗い出してもらえる**
- **作業ステップを分解してもらえる**
- **リスクを先に指摘してもらえる**
何も壊す心配なしにできます。
僕はこれを「AIの安全装置」って呼んでるんですよ。

## ▍ 計画がOKなら実行に移る

計画に「ここ変えて」とフィードバックも出せます。
「これでOK」なら実行に移る。

- **人間が常に主導権を握るワークフロー**
- **AIが勝手に暴走するリスクをゼロにできる**
- **大企業のAI導入でも前提になっている仕組み**
僕らのチームではこの流れを徹底しています。

# ■ 起動は一瞬。/planと打つだけでAIが「考える」モードに変わる

![](https://pbs.twimg.com/media/HMTohtcbYAAak-o.jpg)

## ▍ 覚えることは5つだけ

僕がいつも使ってる流れを説明します。

- **①切り替え**: /plan と入力するだけ。Shift+Tabでも切替OK
- **②指示を出す**: 専門用語は不要。やりたいことを自然な言葉で伝える
- **③計画が出る**: AIがコードを読んで計画を提案。何も編集されない
- **④確認・修正**: 納得で
