## はじめに

AIエージェント（Claude Code など）に開発を丸投げしようとしたとき、以下のような問題に直面したことはありませんか？

* コンテキストがすぐに肥大化してトークン制限に引っかかる
* 深いリサーチが必要なタスクで「それっぽい嘘（ハルシネーション）」をつく
* エージェントが独自に学習した不要なルール（AI Slop）が蓄積して身動きが取れなくなる

これらの課題を解決するため、役割の異なる3つのAIエージェントを連携させ、互いの弱点を補完・監査しあう「Agent Triad（三者連携）アーキテクチャ」を構築しました。

## ベースとなるアーキテクチャ

このアーキテクチャでは、単一のルールファイル（`AGENTS.md`）を Single Source of Truth とし、以下の3つのエージェントが協調して動作します。

1. **Antigravity CLI (AGY)**: ターミナルでの広範なリサーチ、ブラウザ操作（Playwright）、ドキュメントの収集を担う「上流のリサーチャー」。広く浅く探索する。
2. **Claude Code**: AGYが収集した知識を基に、実際のファイル操作やコーディングを高速で行う「実装ハブ」。
3. **Codex CLI**: 実装完了後、別ベンダーのAIモデルという視点からクロスレビューやセキュリティ監査を行う「下流のオーディター」。深く厳密に検証する。

## 改善1: コンテキスト限界を突破する NotebookLM + Headroom

### 問題

大量の公式ドキュメントや仕様書をそのままプロンプトに渡すと、コンテキストウィンドウがすぐに枯渇し、APIコストが跳ね上がります。

### 解決策

これを防ぐため、**Google NotebookLM** と **Headroom** を連携させたゼロコスト・パイプラインを構築しました。

1. **AGYによる全自動インジェスト**  
   `notebooklm-py` と Playwright を活用し、収集したドキュメント群を NotebookLM に全自動でアップロードします。NotebookLM がプロジェクトの「外部Brain」となります。
2. **必要な時だけクエリ**  
   Claude や Codex が開発中に仕様で迷った場合、直接 Web 検索をするのではなく `python -m notebooklm ask` で NotebookLM に質問を投げます。NotebookLM はアップロードされたソース（グラウンディング）にのみ基づいて回答するため、ハルシネーションが排除されます。
3. **Headroom によるトークン圧縮**  
   NotebookLM から返ってきた詳細な長文回答は、**Headroom**（`HEADROOM_MODE = "token"`）を通すことで、必要な意味を保ったままキャッシュを最大化・圧縮され、各エージェントのコンテキスト寿命を約30%延ばします。

## 改善2: 安全な YOLO モード（全自動実行）の確立

### 問題

各エージェントがファイル操作やシェル実行のたびにユーザーの承認（y/n）を求めてくると、パイプラインが止まってしまい完全自動化（Headless）が実現できません。

### 解決策

安全なサンドボックス（Dockerコンテナ等）環境を前提とし、全エージェントで自動承認モード（YOLOモード）を標準化しました。

* **Claude**: `~/.claude/settings.json` で `"defaultMode": "bypassPermissions"` を設定。
* **Codex**: `~/.codex/config.toml` で `approval_policy = "never"` を設定。
* **AGY**: セッション開始時に `/session-yolo-confirm` を実行。

## 改善3: 独自の自己進化（RSI）の汚染を防ぐピアレビュー

### 問題

エージェントがエラーから自律的に改善策（ルール）を学習する Recursive Self-Improvement (RSI) は強力ですが、1つのエージェントが誤ったルールを学習するとシステム全体が汚染されてしまいます。

### 解決策

ルールの自動追加（Autonomous rule accumulation）を禁止し、\*\*証拠に基づくピアレビュー（Evidence-backed changes）\*\*を必須としました。

例えば、Claude がエラーから新しいルールの候補を作成した場合、それをそのまま `AGENTS.md` に追記するのではなく、必ず **Codex** が「構文に問題はないか」「セキュリティ境界を越えていないか」をクロスレビューしてから昇格（Promote）させます。

## まとめ

| 改善項目 | 変更内容 | 効果 |
| --- | --- | --- |
| **役割分担** | AGY(調査) / Claude(実装) / Codex(監査) の3層構造 | 各AIの得意分野に集中し、ハルシネーションとAI Slopを排除 |
| **コンテキスト管理** | NotebookLM (Brain) + Headroom (Token圧縮) | トークン消費を抑えつつ、膨大なドキュメントを正確に参照 |
| **自律性の確保** | 各ツールのYOLOモード（自動承認）の統一 | ユーザーの介入なしでパイプラインが完走 |
| **自己改善(RSI)** | ピアレビュー（他エージェントによる監査）の必須化 | 誤ったルールの学習（システム汚染）を防止 |

この Agent Triad アーキテクチャにより、複雑な要件定義から実装、テスト、監査までのフローを、極めて高い精度で完全自動化することが可能になります。
