長時間駆動が、ガチでヤバい。
OpenAIのAIコーディングツール「Codex」に新機能「/goal」が追加された。

「ゴールを渡したら、達成するまで自律で動き続ける」という機能で、
試したら1時間以上稼働してシューティングゲームを1本まるごと作ってくれた。

しかも、画像生成スキルをONにしておくと
・キャラクター素材
・背景グラフィック
・ゲーム内アイコン

全部AIが自動生成。手動で作った素材、1つもなし👇

これ何がすごいかというと、「1回の指示で完了する単発タスク」ではなく「ゴールが完成するまで自律的にループし続けるモード」という点。

プロンプトを投げるたびに人間が次の指示を出す必要がなく、
/goal はゴールを達成するまで計画・実行・検証を繰り返し続ける。

長時間・複数ターンにわたって自律で走り続けることができ、長期タスクに対応できる。
これって本質的には、「人間が作業を続ける」から「AIがプロジェクトを完走する」へのシフト。

今まで:
・コードを少し書く → AIに聞く → 修正する → また聞く

/goal 以降:
・詳細なゴールを渡す → 放置 → 完成品を受け取る

「人間は要件を定義する人」に役割が変わる。
実際に試したプロンプトはこういう内容。

「ジャングルを舞台にしたトップダウン視点のシューティングゲームを作って。ドッジロール・スプリント・カバーアクションあり。アイテムドロップで経験値を稼いでスキルアップできる設計で、グラフィックはImageGen 2 APIで自動生成」

① AIが自分でプロンプトを大量生成して素材を作る
② 素材をゲームエンジン（Godot）に組み込む
③ ゲームロジックを自律的に実装する

を組み合わせて実行できる。
