僕はベッドに寝っ転がって本を読むのが好きです。数学書を読むときなどはさすがに机に向かいますが、軽い読み物や雑誌を読むぐらいのときは寝転がって読みます。

ただ、これには読書ノートが取りにくいという問題があります。昨年から読書ノートをスマホで取るために、Obsidian Syncを契約したのですが、フリック入力で文を書くのはともかく、マークダウン記法を入力するのが面倒でやめてしまいました。

そんな中、Claude CodeにObsidianのドキュメントを整理させるという使い方を知り、そこから発想を得ました。

## 使い方の概略

1. （寝転がって）本を読みながら、スマートフォンの音声入力でObsidianのデイリーノートに読書メモを書き込む
   * 文章の整形はしない（Claudeがやってくれるため）
   * 誤字・脱字があってもよほど酷くない限りは修正しない（Claudeが修正してくれるため）
2. Obsidian SyncでVaultがPCと同期される
3. PC上のClaude Desktopのスケジュールタスクで、前日のデイリーノートをVaultの既存記事に反映させたり、新規記事を作らせたりする

スマートフォンの音声入力を使うのは、そちらの方がフリック入力よりも速いと思ったからです。ある程度の誤字・脱字はClaudeが修正してくれますが、音声入力の調子によっては文字が全く入力されなかったりするので、実はフリック入力のほうが速かったりするのかもしれません。

## 設定

前提：以下のものを導入済みとします

* スマートフォン
* PC
  + Obsidianアプリ
  + Claude Desktop
    - Claude Codeでは、起動していないと定期タスクが走らないため、Claude Desktopが必要になります
* Obsidian Sync

スマートフォンに追加の設定は必要ありませんが、PCのClaude Desktopにスキルを追加し、スケジュールタスクを設定する必要があります。

1. Claude Desktopに自作の`reflect-journal`スキルを追加します。
   * ディレクトリ構成が僕のVaultの構成になっているので、必要があれば修正してください。
2. ObsidianのVaultのルートディレクトリに`CLAUDE.md`を配置します。
3. Claude Desktopのスケジュールタスクを設定します。
   * 僕は毎朝5時に走らせるようにしました。

### 解釈が不確かだった箇所

| 箇所 | 推測した内容 |
| --- | --- |
| 「本素数判定法集覧」 | ノートタイトルまたは冒頭の区切り表現。「素数判定法集覧」として解釈 |
| 「小さな椅子を持たないか」 | 「小さな因数を持たないか」の誤変換 |
| 「71。13の倍数判定方法」 | 「7・11・13の倍数判定方法」の誤変換 |
| 「その偶数は目通し、奇数番目、どうしを足した結果が7…フォートして合同なら電話」 | 奇数番目と偶数番目のブロックを mod m で比較するルールの説明 |
| 「ニトさんの倍数を無視すると、6 K + 16 K + 5」 | 「2と3の倍数を無視すると、6k+1と6k+5」の誤変換 |
| 「造文は周期的に変化する」 | 「増分は周期的に変化する」の誤変換 |
| 「映画に N が3。41の時」 | 「例えば N が 341 の時」の誤変換 |
| 「メールセンです。MP = 2の P 上-1」 | 「メルセンヌ数 M\_P = 2^P - 1」の誤変換 |
| 「軽の会場 MOD 入れが効率的に…gcd 軽の会場をもっと NN」 | 「(N-1)! mod N が効率的に…gcd(k!, N) > 1」の誤変換 |

## 感想、妥協したところと今後の展望

面倒くさがりな自分でも、これぐらいの労力なら、ちゃんと読書ノートを取るようになるのではないかと思います。

本当はCI(GitHub Actions)やClaudeのルーティンなどのクラウド環境で動かしたかったのですが、以下の理由から難しそうだということで、自分のPCで動かすことで妥協しました。

* Obsidian Syncをインストール&ログインする術がなさそう
* ObsidianのGitプラグインを使うという手もあるが、スマートフォン環境では極めて不安定らしい

ObsidianとClaude Desktopを常時起動しておいて、メモをとった翌日にはPCを立ち上げないといけないですが、これぐらいならいいでしょう。

あらためて考えてみると、スマートフォンでメモを取る時にObsidianを使う理由は特にないので、Google Docとかに書いてしまうのでもいいのかもしれません。しかし、コンテンツの同期をObsidian Syncに依存しているので・・・やっぱりObsidian SyncがAPIを公開してくれるのを待つしかないのでしょうか？
