# 1 Claude codeのセットアップ

![](https://static.zenn.studio/user-upload/4e392339fc9e-20260326.png)  
組み合わせは**WSL2 + Git Bash CLI**で実施する。日常使いとしてVSCodeの拡張機能も設定する。  
マルチエージェントで一人会社をしていきたい。

## 1.1 各種インストールと設定

### 1.1.1 Windows用のインストールとパス設定

| 使いたいこと | 使く環境 |
| --- | --- |
| VSCode拡張機能を使う | Git Bash か AlmaLinux のどちらか1つ |
| マルチエージェント・本格利用 | AlmaLinux（WSL2）一択 |
| Windowsのコマンドプロンプトから使う | PowerShell / CMD |

Windows用をインストール

```
# PowerShellまたはCMDで実行
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```

`claude.exe`を指定したディレクトリに移動してから、環境変数を設定する。

```
claude --version
claude doctor
```

**Git Bash CLI**

```
#パスを通す
echo 'export PATH="$PATH:/c/Users/<username>/ClaudeCode"' >> ~/.bashrc
source ~/.bashrc
claude --version
```

### 1.1.2 Linux(WSL)用のインストール

**WSL2のセットアップ**

```
# まずWSL2本体を有効化（まだの場合）
wsl --install --no-distribution
```

OS再起動

```
# AlmaLinuxをインストール
wsl --install -d AlmaLinux-9
```

AlmaLinuxでLinux用のclaudeのバイナリファイルをインストールが必要。  
AlmaLinuxで下記を実行。

```
#AlmaLinuxを起動
> WSL
$ curl -fsSL https://claude.ai/install.sh | bash
Setting up Claude Code...
✔ Claude Code successfully installed!
  Version: 2.1.83
  Location: ~/.local/bin/claude
  Next: Run claude --help to get started
✅ Installation complete!

[User1@DESKTOP-M7K4QV7 ***]$ source ~/.bashrc
[User1@DESKTOP-M7K4QV7 ***]$ claude --version

```

WSLを開くとカレントディレクトリがWIndowsパスになっている。

```
C:\Users\username>wsl
[User1@DESKTOP-M7K4QV7 User1]$ pwd
/mnt/c/Users/<Username>
```

下記を実行して、wslコマンドでホームディレクトリに移動するようにする。  
AlmaLinux上で下記を実行

### 1.1.3 日常使い用としてVSCodeの拡張機能設定

**VSCodeでCluade Codeの拡張機能インストール**  
VS Codeを開いて`Ctrl+Shift+X`で拡張機能パネルを開き、「Claude Code」で検索  
Anthropic製のものをインストール

**WSL拡張機能のインストール**  
VS Codeの拡張機能パネル（Ctrl+Shift+X）で「WSL」と検索 → Microsoft製の「WSL」をインストール。

**WSL接続モードで開く**  
AlmaLinuxのターミナルを開いて

```
mkdir -p ~/Project
cd ~/Project
#ここでVSCodeを起動
code .
```

これでVS CodeがAlmaLinux接続モードで開く。左下に`WSL:AlmaLinux-9`と表示されていれｗば成功  
**Claude Codeにログイン**  
VSCode内のターミナルで"""Ctrl+`"""で

ブラウザが開くのでClaude.aiのアカウントでログイン。一度ログインすれば以降は不要。

画面にURLが表示されるのでそれをブラウザに張り付けて実行すると認証トークンが表示されるので、それをVSCodeのターミナルに入力する。

## 1.2 エージェントチームのセットアップ

Agent TeamsはClaude Code v2.1.32以降が必要。

### 1.2.1 settings.json を編集して有効化

| 場所 | 役割 | 優先度 |
| --- | --- | --- |
| `~/.claude/settings.json`（AlmaLinux内） | AlmaLinux環境全体に適用 | 低 |
| `~/projects/myapp/.claude/settings.json` | そのプロジェクトだけに適用 | 高 |
| `C:\Users\<Username>\.claude\settings.json` | Windowsのデスクトップアプリに適用 | Windows側のみ |

AlmaLinuxの`~/.claude/settings.json`を編集して下記を書き込んで保存。

```
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

Claude CCodeを再起動して有効化

**tmuxのインストール（任意だが推奨）**  
split-paneモードはVS Codeの統合ターミナルでは動きません。AlmaLinuxのターミナルで使う場合はtmuxが必要です

Claude Codeを再度起動して`/status`を打ち、`Setting sources: User settings`となっていればOK。

### 1.2.2 CLAUDE.mdを書いてエージェントに役割設定

AlmaLinux上でプロジェクトディレクトリに移動してから`vim CLAUDE.md`をして下記のように設定。下記はラジオをやる場合のサンプル

```
# ラジオ番組制作プロジェクト

## 概要
20代向けビジネス・キャリア系ラジオの企画・制作を行う。
市場調査から話題作成まで、複数のエージェントが分担して作業する。

## エージェント構成

### リサーチAgent
- 役割：市場調査・トレンド収集
- 担当：SNS・ニュース・YouTube・Podcastの動向調査
- 出力：調査レポート（research_report.md）

### 企画Agent
- 役割：話題・コーナー企画の立案
- 担当：リサーチ結果をもとにトーク台本の骨子を作成
- 出力：企画書（plan.md）

### ライターAgent
- 役割：台本・原稿の執筆
- 担当：企画をもとに実際のトーク原稿を作成
- 出力：台本（script.md）

### レビューAgent
- 役割：品質チェック
- 担当：20代に刺さるか・情報の正確性・話しやすさを確認
- 出力：レビューコメント（review.md）

## ターゲット
- 年齢：20代
- 関心：就活・転職・副業・スキルアップ・お金・キャリア設計

## トンマナ
- 堅すぎず、フランクで親しみやすい口調
- 難しい経済用語は使わず、具体的な事例で説明
- 1トピック5〜10分で話せる分量

## ワークフロー
1. リサーチAgentが今週のトレンドを調査
2. 企画Agentが3〜5つの話題候補を作成
3. ライターAgentが選ばれた話題の台本を執筆
4. レビューAgentが確認してフィードバック

## 出力ファイル
- research_report.md：調査レポート
- plan.md：企画書
- script.md：台本
- review.md：レビュー
```

保存できたら その場で`code .` でVSCodeを起動する。  
"""Ctrl+`"""で`pwd`をしてプロジェクトフォルダにいることを確認。異なる場合は対象ディレクトリまで移動してから`claude`を実行。  
その後、下記を入力して実行させる。

```
CLAUDE.mdを読んで、ラジオ番組制作のエージェントチームを作成してください。
共有タスクリストを使って順番に進めてください。
```
