## 3日や。たった3日で、ぜんぶひっくり返った

2026年5月18日から19日。**72時間**のあいだに、AIに仕事を頼む値段が1桁変わってもうた。

「また新モデルが出たんやろ、よくある話やん」——そう思うやろ？ 実は今回、ちゃう。**性能はそのままで、値札だけが10分の1になった**。これ、AIの世界では「また一段速くなりました」より、よっぽどデカい事件やねん。

おっちゃん、長いこと社内SEやっとるけど、こういう「値段が桁で落ちる瞬間」を現場で見たことが何回かある。サーバが自前ラックからクラウドに移ったとき。電話回線がIP電話になったとき。あのときと同じ匂いがする。**安なった瞬間に、世界の前提がこっそり書き換わる**んや。

今日はその「72時間」で何が起きたんか、数字を並べて、最後に「で、あんたはどうする？」まで一緒に考えよ。

![5月18〜19日の72時間で起きたコーディングエージェント連続リリースのタイムライン](https://static.zenn.studio/user-upload/deployed-images/d008f049ae87f4dc326739b8.png?sha=c98a210663359826d7e01f3ca9ee04ee060a82c0)

## まず、何が起きたんか

時系列で並べるとこうや。

| 日付 | 誰が | 何を出した |
| --- | --- | --- |
| 5/18 | **Cursor** | Composer 2.5(自社コーディングモデル) |
| 5/19 | **Anthropic** | Claude の Managed Agents 強化(自ホストサンドボックス・MCPトンネル) |
| 5/19 | **Alibaba** | Qwen 3.7 Max API |
| 5/19 | **Google** | Antigravity 2.0(マルチエージェント開発環境) |

たった2日や。示し合わせたわけでもないやろに、全員がほぼ同時に手札を切った。**それぞれ別の戦い方やのに、向かっとる方向は一つ**——「フロンティア級の賢さを、もっと安く、もっと自律的に」。

この足並みの揃いっぷりが、今回いちばん不気味なとこやねん。1社だけなら「がんばったな」で済む。でも4社が同じ週に同じ方向を向いたら、それはもう**業界全体の地殻変動**や。

## 数字で見たら、ゾッとする

口で「安なった」言うても伝わらん。数字で見よ。Cursor の Composer 2.5 と、いまの王者 Claude Opus 4.7 を、同じ土俵(100万トークンあたりの単価)で並べる。

| モデル | 入力(/Mtok) | 出力(/Mtok) | コーディング性能 |
| --- | --- | --- | --- |
| **Cursor Composer 2.5**(標準) | $0.50 | $2.50 | SWE-Bench Multilingual 79.8% |
| **Claude Opus 4.7** | $5.00 | $25.00 | Composer 2.5 と同等域 |

見てくれ。**入力で10倍、出力で10倍**。きれいに1桁ちゃう。しかも Cursor 公式いわく、性能は Opus 4.7 や GPT-5.5 と「同等」やと。SWE-Bench Multilingual で 79.8%、CursorBench v3.1 で 63.2% 出しとる。

「安かろう悪かろうちゃうんか？」と疑うやろ。おっちゃんも疑った。でも Composer 2.5 のタネを知って納得した。**Moonshot が公開した Kimi K2.5 という"土台"の上に、25倍の合成タスクで鍛え直した**もんやねん。つまりゼロから巨大モデルを焼く必要がなくなった。土台はオープンで配られとるから、その上で味付けするだけでフロンティア級に届く。**自前で小麦から育てる時代から、ええ生地を仕入れて自分の店の味にする時代**になったんや。

## で、月いくら変わるんや

ここが、あんたの財布に効く話や。ベンチマークの%は正直ピンとこん。「で、月いくら違うん？」が本音やろ。

仮に、1日にエージェントへ入力300万トークン・出力100万トークン投げる開発者がおったとする(エージェント使い倒すと普通にこれくらいいく)。月20営業日で計算したろ。

* **Opus 4.7**: 入力 (3×$5)＋出力 (1×$25) ＝ 1日$40 → **月$800**
* **Composer 2.5**: 入力 (3×$0.50)＋出力 (1×$2.50) ＝ 1日$4 → **月$80**

**月$800 が月$80**。年で言うたら $9,600 が $960 や。浮いた差額で、もう一人ぶんのツール契約が丸ごと買える。**ラーメン屋で言うたら、いつもの一杯が急に1コインで食えるようになった感じ**やな。味は据え置きで。

ただし——ここは正直に言う。**この皮算用には罠がある**。

Alibaba の Qwen 3.7 Max は表向き $2.50/$7.50 と安いけど、「拡張思考(extended thinking)」がデフォルトでオンになっとって、実運用では3〜4倍のコストに膨らむっていう注記が付いとる。Opus 4.7 も、価格自体は据え置きやのに**新しいトークナイザーで同じ文章が最大35%多いトークンに化ける**。つまり「表の単価」と「請求書の金額」は別モノや。**メニューの値段とレジの金額がちゃう居酒屋**、あるやろ。あれと同じ。安さに飛びつく前に、自分の使い方で一回実測せなあかん。

![1日の入出力トークン量を仮定したOpus 4.7とComposer 2.5の月額コスト試算バーチャート](https://static.zenn.studio/user-upload/deployed-images/7ac069ce975259a636f831aa.png?sha=c2f089851c0f80ba1dedbe48fc7aa47fb75902f6)

## なんで、急に落ちたんや

「企業が善意で値下げした」——んなわけない。値段が落ちるには、ちゃんと理由がある。今回は3つ重なった。

**1つ目、土台がタダで配られ始めた。** さっきの Kimi K2.5 がそうや。中国勢を中心に、フロンティアに肉薄するモデルが Apache ライセンスでばら撒かれとる。土台がオープンなら、各社は「最後の味付け」だけにコストを集中できる。焼き直しのコストが消えるんや。

**2つ目、競争の階層が変わった。** いままでは「いちばん賢いモデルは誰や」の一騎打ちやった。それが今は **Cursor・Claude Code・Codex が"重なり合う1つのスタック"** になりつつある。誰か1人が勝つんやのうて、層になって共存する。共存するなら、差別化のポイントは「賢さ」から「安さと使い勝手」に移る。そら値段で殴り合うことになるわな。

**3つ目、巨大化競争が一服した。** 5月のフロンティアは、面白いことに「もっと大きく」やのうて「もっと賢い設計」に舵を切った。subquadratic attention みたいな、計算の仕組みそのものを効率化する研究が前に出てきた。**パラメータを増やして殴る筋トレから、技で省エネする柔道へ**——力点が移ったんや。デカさで勝負せんようになったら、コストは自然に下がる。

![基盤モデルのオープン化・スタックの重層化・アーキテクチャ革新の3要因が価格下落を生む構造図](https://static.zenn.studio/user-upload/deployed-images/011d115b266b15a31b5698af.png?sha=91635f94970258b497814e939872800619a0e10e)

## 立ち位置が変わったのは、全員や

この72時間、主役は Cursor やったけど、地味に全プレイヤーの足元が動いとる。

* **Google** は Antigravity 2.0 で「マルチエージェント」に全張り。動的サブエージェント、スケジュール実行、Go で書かれた CLI、公開 SDK、そして Gemini 3.5 Flash。開発環境ごと囲いにきた。一方で消費者向けの Gemini CLI と Code Assist 拡張は6月18日で畳む。**選択と集中を、同じ月にやっとる**。
* **OpenAI** は5月初頭に GPT-5.5 Instant を ChatGPT の標準に据えた。年換算売上は$25Bを超え、年内IPOに向けて動いとるという報道もある。製品より「会社の体力」を見せにきとる。
* **中国勢**(Alibaba・DeepSeek・GLM ほか)は、オープンと低価格でフロンティアの足首を掴みにきた。Qwen 3.7 Max が Anthropic のメッセージ仕様に乗せてきたあたり、**"乗り換えやすさ"まで設計に入れとる**のがしたたかや。
* **Anthropic** は値段の殴り合いには乗らず、「企業のデータを外に出さんで済む」方向(自ホストサンドボックス・MCPトンネル)に振った。安さで勝てんとこは、安心で勝つ。ちなみに今年2月の Pragmatic Engineer の調査(エンジニア906人)では、Claude Code が"いちばん愛されとるツール"で46%の支持やった——ただしこれは2月の数字。この3日でその地図がどう塗り替わるかは、まだ誰にもわからん。

つまりや。**「安さ」「自律性」「データ安全」「会社の体力」——殴り方は全員バラバラ**。でも全員が「次の主導権」を取りにきとる。

## さて、あんたはどれを選ぶ

ここまで読んで「で、結局どれ使えばええねん」と思とるやろ。おっちゃんの整理はこうや。

* **コスト最優先・大量に回す**なら → Composer 2.5 みたいな低単価エージェント。ただし自分の使い方で月額を**実測**してから。
* **難しい一発勝負・品質が命**なら → Opus 4.7 級。高いけど、外せん仕事はここ。
* **会社のデータを外に出せん**なら → 自ホスト・トンネル系。値段やのうて安全で選ぶ。
* **エージェントを何体も連携させたい**なら → Antigravity みたいなマルチエージェント環境。

大事なのは「1個に決め打ちせんこと」やと思う。**料理によって包丁を使い分けるみたいに**、タスクによってエージェントを替える。幸い、各社が"乗り換えやすさ"を競い始めたおかげで、使い分けのコストはどんどん下がっとる。

## おっちゃんの見立て

今回いちばん怖いのは、値下げそのものやない。**「賢さは、もう希少品やない」と全員が認め始めた**ことや。

賢さが安なったら、価値はどこに移るか。たぶん「賢さをどう組み合わせて、どう仕事に流し込むか」——**設計と段取り**のほうや。包丁が安なったら、勝負は腕に移る。あんたの現場で、その腕、磨けとるか？

72時間で値段が1桁落ちる世界や。半年後、今日の「安い」がまた「高い」になっとるかもしれん。**安くなったAIに何をさせるか決めるのは、結局あんたや。** さて、あんたなら、この浮いた9割の予算、何に使う？

---

📎 **おっちゃんの現場から**

実はうち(CCC)でも、ちょうどこの"使い分け"を地で行っとる。社内に建てたAIの会社で、軽い調べ物は安いモデル、設計の肝や最終判断は賢いモデル、と役職ごとにモデルを割り当てとるんや。最初は「全部いちばん賢いやつでええやん」と思てた。でも実際は、賢いモデルを雑用に使うのは**タクシーで隣のコンビニ行くようなもん**やった。今回の価格破壊で、この「適材適所」がますます効いてくる。安いカードが増えるほど、配り方のセンスが問われる——現場の手触りとして、そう感じとる。
