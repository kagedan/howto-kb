あなたは今、無料でAIを使い続けられると思っているだろうか。

2026年5月9日現在、OpenAIは「全ユーザーにChatGPTをもっと使わせる」と「その代わり広告を見せる」の両方を同時に動かしている。ChatGPT for Excel・Google Sheetsが全プランで解放され、広告モデルが世界5カ国に拡大された。一方、Anthropicは開発者向けに静かに重要バグを修正し、GoogleはI/O 2026まで残り10日で最大の発表に備えている。

今日の記事では、この「AIサービスの収益化フェーズ移行」と、開発者が今すぐ恩恵を受けられる技術アップデートの両方を整理する。

## この記事でわかること

* ChatGPTがスプレッドシートに本格参入した意味と、今日から使える方法
* OpenAIが「無料AI」を広告モデルで支える戦略の全貌
* Claude Code Opus 4.7の1Mコンテキスト修正で何が変わるか

## 今日の3行サマリ

* ChatGPT for Excel・Google Sheetsが全プラン（Free含む）で全世界解放。GPT-5.5搭載でスプレッドシート操作をAIに任せられる時代が本格到来
* ChatGPT広告がUK・日本・メキシコ・ブラジル・韓国の5カ国に拡大。Free/Goユーザーには広告、Plus以上は引き続き広告なし
* Claude Code: Opus 4.7が「1Mトークンコンテキスト」を正しく活用できるバグ修正が公開。MCP auto-retry（最大3回）も追加

## ChatGPTがスプレッドシートの「新しい同僚」になった

5月5日、OpenAIはChatGPT for ExcelとChatGPT for Google Sheetsをベータ終了・全世界全プラン解放した。Free・Go・Plus・Pro・Business・Enterprise・Edu、すべてのユーザーが対象だ。

何ができるかは明確だ。テキスト1文でワークブックを一から生成し、汚いデータを整理し、複数シートをまたいで数式の繋がりを説明し、なぜ出力が変わったかを日本語で解説させられる。GPT-5.5搭載なので、スプレッドシート固有の文脈も理解精度が上がっている。

他社との比較で言えば、Claudeにもスプレッドシートへの連携試験はあるがOpenAIほどの公式ネイティブ統合には至っていない。GoogleはGeminiをSheetsに組み込み済みだが、「ChatGPTで使い慣れたプロンプト感覚のまま」Sheetsを操作できる点がOpenAIの差別化だ。

**今すぐ試せること：** Google WorkspaceのマーケットプレイスまたはExcelアドインストアで「ChatGPT for Sheets」を検索してインストールする。まず手元の実務シートで「このシートの数式のエラーを直して」と1文投げてみるだけでいい。

## ChatGPT広告、世界5カ国へ──「無料AIを維持するコスト」を誰が払うか

2026年2月に米国で始まったChatGPT広告テストが、いよいよUK・日本・メキシコ・ブラジル・韓国に展開される。Free・Goティアのログイン済み成人ユーザーが対象で、広告はChatGPTの生成コンテンツと「明確に区別」されたラベル付きで表示される。

選択肢は3つだ。広告を受け入れて無料で使い続けるか、有料プランに移行して広告なしを得るか、Free tierで「広告を減らす代わりに1日のメッセージ数を減らす」オプトアウトを選ぶか。

Claudeの無料プランには現時点で広告はない。GeminiのFree tierにも広告はない。OpenAIが「広告付き無料AI」を選んだのは業界初の実験だ。Freeユーザーを抱えたまま収益化する道として、有料移行を促さずとも収益が入る仕組みを作ろうとしている。

ビジネスパーソンへの示唆は一つ：Freeティアで仕事に組み込んでいるならば、広告が入り始める前に「月いくらなら有料に移行するか」の自分なりの基準を決めておくことだ。Plus（月$20）で広告なし・メッセージ制限緩和が維持されるかどうかは費用対効果計算のベースになる。

## Claude Code: Opus 4.7が「本物の1Mトークン」を使えるようになった

開発者には見逃せないバグ修正だ。Claude CodeのOpus 4.7セッションで、`/context`コマンドが200Kウィンドウ基準で誤って計算されていた問題が修正された。実際にはOpus 4.7は1Mトークンのコンテキストウィンドウを持つのに、200K基準で「コンテキストが埋まりそう」と判断してauto-compactが早期発動していたのだ。

影響は大きい。大規模コードベースの解析、複数ファイルにまたがるリファクタリング、長期セッションでの継続作業──どれも本来の1Mトークン容量でこなせるようになる。比較するなら、OpenAIのGPT-5.4も1Mトークンコンテキストを売りにしているが、Claude Code環境でOpus 4.7を使う場合は今回の修正後に初めて対等条件で戦える。

同時に、MCP（Model Context Protocol）サーバーが起動時の一時エラーで切断されていた問題も改善された。最大3回の自動リトライが走るようになり、開発環境の安定性が上がる。`CLAUDE\_CODE\_SESSION\_ID`環境変数もBashサブプロセスに渡されるようになり、セッション管理の細かい制御が可能になった。

**今すぐ試せること：** `claude update`で最新版に更新し、`/context`コマンドで表示されるコンテキスト使用量が正しく反映されているか確認する。

## OpenAI「ChatGPT Futures: Class of 2026」──AIネイティブ世代の最初の出口

OpenAIが発表した「ChatGPT Futures: Class of 2026」は、2026年に大学を卒業する26名の学生を選抜するプログラムだ。各メンバーに$10,000のグラントとフロンティアモデルへのアクセスが提供される。

このプログラムが示唆することは数字だけではない。2022年入学の学生は「ChatGPTが登場した秋に大学に入り、4年間AIと共に過ごして卒業する」最初の世代だ。彼らが何をAIで作ったか──障害者向けアクセシビリティツール、医療分野の研究加速、メンタルヘルスの多言語翻訳、地域課題解決──が、今後の「AIを使いこなす人材」の標準像を決める。

Anthropic・Googleもそれぞれ学術プログラムを持つが、$10Kグラントという金額は業界水準でも高い。OpenAIが「次世代のAIユーザー像」をブランド戦略の一部として積極的に定義しようとしている姿勢が見える。

## Google TurboQuant──KVキャッシュを「6倍圧縮」して大規模LLMの記憶コストを激減させる

ICLR 2026でGoogleが発表したTurboQuantは、LLMのKVキャッシュを3bitで圧縮することでメモリ使用量を最大6倍削減するアルゴリズムだ。2段階で処理する。

まず「PolarQuant」がランダム回転行列を各キー・バリューベクトルに適用して分散を均一化し、スカラー量子化の誤差を最小化する。次に「QJL（1bit Quantized Johnson-Lindenstrauss変換）」で残差誤差を追加圧縮する。

特筆すべきは「データ非依存」という特性だ。訓練不要・キャリブレーション不要・モデル非依存で動作するため、どのLLMにも原則として適用できる。NVIDIA H100での実測ではアテンション計算が最大8倍高速化された。

実際のインパクトは大きい。1Mトークン超のコンテキストウィンドウをGPU1枚で扱える可能性が高まる。MetaやOpenAIも独自のKV圧縮手法を持つが、「データ非依存・訓練ゼロ」で同等以上の圧縮率を出せる手法はこれまでなかった。エッジでのLLM推論が現実に近づく技術的転換点だ。

## Google I/O 2026まで10日──Gemini 4と「Remy」が勢力図を塗り替える

5月19〜20日のGoogle I/O 2026まで残り10日だ。リークと事前情報が積み上がっており、期待値は過去最高水準にある。

中心はGemini 4だ。画像・動画生成機能をネイティブに統合し、テキスト・マルチモーダル推論・コンテンツ生成を単一モデルで完結させる設計とされる。これが本当に実現すれば、画像生成にDALL-Eを、動画にSoraを使い分けているOpenAIの構造に対して明確な差別化になる。

「Remy」は24時間対応のパーソナルアシスタントエージェントとして噂されており、ChatGPTのメモリ機能やClaudeのプロジェクト機能と正面からぶつかる設計とみられる。すでにGoogle AI ProユーザーはAI Studioの利用制限拡大を受けており、I/Oに向けた布石が静かに打たれている。

5月19日の発表内容は今週後半に各メディアが事前リポートを出し始める。次の記事ではGoogle I/Oの注目発表を詳報する予定なので、ぜひフォローしてお待ちください。
