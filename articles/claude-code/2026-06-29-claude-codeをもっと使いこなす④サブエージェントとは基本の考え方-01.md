# Claude Codeをもっと使いこなす④｜サブエージェントとは？基本の考え方

こんにちは、開発日和です。

Claude Codeを使い始めて「なんか裏でいろいろ動いてるな」と感じることはありませんか？それがサブエージェントの仕業です。

今回はサブエージェントの基本的な考え方を解説します。「難しそう」と感じるかもしれませんが、概念を掴んでしまえばシンプルです。

---

## サブエージェントとは

一言で言うと、**親のエージェントのClaudeCodeセッションから派生される子どものAIタスク**です。

通常のClaude Codeはひとつの会話が一本線で進みますが、サブエージェントを使うと「別の作業を別のAIに任せて並行して進める」ことができます。

```
ユーザー ──→ Claudeエージェント（メイン）
                │
                ├──→ サブエージェントA（コードを実装する）
                ├──→ サブエージェントB（テストコードを書く）
                └──→ サブエージェントC（ドキュメントを更新する）
```

---

## なぜサブエージェントが必要なのか

### コンテキスト汚染を防ぐ

ひとつのセッションで大量の作業をこなすと、会話が長くなりClaudeが過去の文脈に引きずられてしまいます。サブエージェントは**独立したクリーンなコンテキスト**で動くので、この問題を回避できます。  
それぞれのサブエージェントに、必要な情報だけに限定して会話ができます。

### 並行作業が可能になる

「Aが終わったらBをやる」という直列作業を「AとBを同時にやる」に変えられます。大量のドキュメント解析やシーダー生成、本丸のプログラム作成など、時間のかかる作業に効果的です。

### 専門的なタスクに集中させる

コードの調査専門のエージェント・テスト生成専門のエージェント・ドキュメント更新専門のエージェント、というように役割を分けることができます。

---

## サブエージェントの種類

Claude Codeには用途ごとにあらかじめ定義されたエージェントタイプがあります。

| エージェントタイプ | 得意なこと |
| --- | --- |
| `claude` | 汎用（なんでもいける） |
| `Explore` | ファイルやコード、シンボルの検索・調査するための高速な読み取り専用エージェント |
| `Plan` | 実装計画を設計担当。重要ファイルの特定やステップバイステップの実装計画を作成する |
| `claude-code-guide` | Claude Code、Claude Agent SDK、Claude APIの使い方などの質問対応 |
| `general-purpose` | 複雑な質問の調査やマルチステップタスクに使う汎用エージェント |

プロンプトを工夫することで、独自の役割を持つカスタムエージェントを作ることもできます。  
`/agents`コマンドから、カスタムエージェントを作成できます。

---

## サブエージェントの基本的な動き方

Claude Codeがサブエージェントを使う場面は主に2パターンあります。

### 1. Claudeが自動的に判断して使う

複雑なタスクを依頼すると、Claudeが「これはサブエージェントで並行処理したほうが速い」と判断して自動的に動かすことがあります。  
けっこう勝手に判断して、意識せずとも動くことが多いです。

```
> このプロジェクト全体のリファクタリング計画を立てつつ、
  各モジュールのコードを調査報告書を作成しつつ、
  新機能Aの実装を仕様書に沿って進めて。
```

このような大きなタスクでは、Claudeが自律的にサブエージェントを使い始めることがあります。

### 2. 明示的に指示して使う

`/background` コマンドや特定の指示を使って、意図的にサブエージェントを起動することもできます。

```
> /background テストファイルを全部見て、失敗しているものをリストアップして
```

---

## サブエージェントを使うときの注意点

### コストが増える

サブエージェントはそれぞれ独立してAPIを呼び出すため、複数立ち上げるとコストが比例して増えます。必要な場面で使うのがポイントです。

### コンテキストは引き継がない

サブエージェントは独立しているため、メインセッションの会話の流れを知りません。「さっきの会話でAと決めた件」はサブエージェントには通じないので、必要な情報は明示的に渡す必要があります。

### 状態確認は `/agents` で

動いているサブエージェントの一覧と状態をここで確認できます。意図せず動き続けているエージェントはここで止めましょう。

とは言いつつも、勝手に動き出すので、最初のうちは自動でなんかサブエージェントが動いているな〜くらいでいいと思います。

---

## 「エージェント」という言葉の整理

Claude Code周辺では「エージェント」という言葉がいくつかの意味で使われます：

| 文脈 | 意味 |
| --- | --- |
| Claude Code自体 | AIエージェント（自律的にツールを使って作業するAI） |
| サブエージェント | Claude Codeが起動する子タスク |
| Claude Agent SDK | エージェントを自分で作るためのSDK（Part16で解説） |

混乱しやすいので、「どのレベルの話か」を意識しながら読むとスッキリします。

---

## まとめ

* サブエージェント = メインのClaudeから生まれる子どものAIタスク
* 並行処理・クリーンなコンテキスト・役割分担が主なメリット
* コストは増えるので使い所を選ぶ
* `/agents` で状態確認ができる
* 結局、最初は気づいたら勝手に動いてくる
