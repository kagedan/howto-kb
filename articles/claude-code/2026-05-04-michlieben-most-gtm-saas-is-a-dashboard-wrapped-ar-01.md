most gtm saas is a dashboard wrapped around an api the agent can just call directly. strip them out and campaigns drop from days to minutes.

here's a simple workflow to try:

> install claude code. one project folder, one CLAUDE.md, an empty .claude/skills/ directory, a .env for api keys.
> spend the afternoon on CLAUDE.md. ICP definition, scoring rules, tool preferences, voice guidelines. keep it under 200 lines. the agent reads it on every run.
> connect apollo. write one skill that scores a csv of leads against your ICP and returns the top 50%. run it on a real export. that's your first deliverable.
> run the same skill on a second export. if the tiering shifts, the policy is in the prompt, not the markdown. the workflow won't survive handoff. move it until both runs are identical.

every workflow after that is the same pattern: one markdown file, one api, one skill. once you master this, you can keep creating more complex workflows and weave together multiple APIs for each function of your GTM motion.
