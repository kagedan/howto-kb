# 2. Claude.ai(チャット)の機能

## 2-1. Projects(プロジェクト)

**何ができる：** 特定の仕事・テーマ専用の作業スペースを作り、カスタム指示や参考資料をまとめて常駐させられる機能。

**使い方：**

1. サイドバーの「Projects」→ 新規作成
2. 「プロジェクトの指示」に役割・前提・トーンなどを記述
3. 「ナレッジ」に資料(PDF/テキスト/コード等)をアップロード
4. そのプロジェクト内で会話すると、毎回説明しなくても指示と資料を踏まえて回答してくれる

**ポイント：**

* 資料が大量になり通常のコンテキスト上限に近づくと、自動的にRAG(検索拡張)モードに切り替わり、実質的な参照可能量が拡張される
* Team/Enterpriseではプロジェクトをメンバー間で共有可能

## 2-2. Artifacts(アーティファクト)

**何ができる：** コード・HTML・SVG図・Mermaid図・スプレッドシートなど、チャットとは別のプレビューウィンドウにその場で生成・編集・ダウンロードできる機能。

**使い方：** 「〇〇のツールを作って」「これをグラフにして」のように依頼すると自動的に右側にウィンドウが開く。

**活用例：** 簡易Webアプリのプロトタイプ、インタラクティブな学習教材、データ可視化ダッシュボード

## 2-3. メモリー(Memory)

**何ができる：** 過去の会話内容を記憶し、次回以降の会話で自然に踏まえて応答する機能。

**使い方：** 設定からメモリーをON。会話するたびに自動で蓄積される(手動でメモを追加することも可能)。

**注意点：** メモリーが未使用の場合、過去チャットの内容は引き継がれない。

## 2-4. Research(リサーチ)

**何ができる：** Claudeが自律的に何度もWeb検索を重ね、複数ソースを統合して出典付きレポートを作成する機能。

**使い方：** チャット入力欄下の「Research」ボタンをON →「〇〇について調べてレポートにまとめて」と依頼。

**利用可能プラン：** Pro以上(Freeは不可)。

## 2-5. Skills

**何ができる：** よく使う指示パターンをテンプレート化し、再利用できる機能。

**使い方：** 設定からSkillsを作成、もしくは公式が用意した定型スキル(ドキュメント作成の型など)を有効化。

**利用可能プラン：** Freeプランでも利用可能な点が他社との差別化ポイント。

## 2-6. ファイル生成(Word/Excel/PowerPoint)

**何ができる：** チャット内の依頼だけでWord・Excel・PowerPoint形式の完成ファイルを生成し、ダウンロードできる。

**使い方：** 「これをWordの報告書にして」「この数値をExcelの表にまとめて」と依頼。

## 2-7. 外部コネクタ

**何ができる：** Google Drive/Docs、Slack、Gmail、社内ツールなどと連携し、実際のデータを参照して回答させられる機能。

**使い方：** 設定 → コネクタから対象サービスを接続(OAuth認証)。

# 3. Claude Code 解説

Claude Codeはターミナル(CLI)・VS Code/JetBrains拡張・デスクトップアプリから使える、**開発者向けのエージェント型コーディングツール**です。Pro以上のプランに含まれています。

## 3-0. Claude Codeでできること

「チャットにコードを貼り付けて質問する」のとは根本的に違い、Claude Codeは**あなたのリポジトリの中で実際に手を動かします**。

**コードベースの理解・調査**

* リポジトリ全体を横断的に読み、「この認証処理はどこで行われている?」のような質問に答える
* `@ファイル名`でファイルを、`@ディレクトリ/`でフォルダ全体を、`@URL`でWebページを文脈に取り込める

**実装・修正**

* 複数ファイルにまたがる機能追加・修正を一括で実行
* バグの原因調査→修正→テストで検証、まで一連で完遂
* エラーメッセージを渡すだけでスタックトレースを追って根本原因を特定

**テスト**

* テストコードの新規作成、既存テストのカバレッジ拡大
* テスト実行→失敗の分析→修正→再実行のループを自律的に回す

**Git操作**

* 変更内容を踏まえたコミットメッセージの作成・コミット実行
* PRの作成、GitHub上のPRレビュー(`/review`)
* `git worktree`を使った並列作業(`/batch`で複数エージェントが同時にPRを作る)

**品質管理**

* コードレビュー(`/code-review`)、セキュリティ監査(`/security-review`)
* リファクタリング・コードの簡素化(`/simplify`)

**自動化・長時間タスク**

* `/goal`で完了条件を設定し、達成まで自律的に作業を継続
* `/loop`で定期チェック(CI監視など)をセッション内で繰り返し実行
* `/schedule`でクラウド上のスケジュールタスクを設定
* Hooksでツール実行前後に自動処理(保存時の自動フォーマットなど)

**外部連携**

* MCPサーバー経由でGitHub・Slack・DB・社内APIなどを直接操作
* GitHub Actionsに組み込んでCI内でClaudeを動かす(`/install-github-app`)

## 3-1. インストール

```
# ネイティブバイナリ(推奨)
curl -fsSL https://claude.ai/install.sh | bash

# Homebrew(macOS)
brew install --cask claude-code

# npm(任意OS)
npm install -g @anthropic-ai/claude-code

# バージョン確認
claude --version

# プロジェクトディレクトリでセッション開始
cd your-project
claude
```

## 3-2. 設定ファイルの全体像

Claude Codeは複数の設定ファイルを階層的に読み込み、マージします(深い階層＝プロジェクト側が優先)。

```
~/.claude/                      # ユーザー全体(グローバル)
├── CLAUDE.md                   # 全プロジェクト共通の指示
├── settings.json               # 個人グローバル設定(権限・hooks等)
├── commands/                   # 個人用スラッシュコマンド(レガシー)
├── skills/                     # 個人用スキル(推奨形式)
└── claude.json                 # Claudeが自動管理(MCP・キャッシュ等)

<プロジェクトルート>/
├── CLAUDE.md                   # プロジェクト全体の指示(git管理・チーム共有)
├── CLAUDE.local.md             # 個人用の上書き指示(.gitignore推奨)
└── .claude/
    ├── settings.json           # チーム共有設定(git管理)
    ├── settings.local.json     # 個人上書き設定(.gitignore推奨)
    ├── .mcp.json                # MCPサーバー接続設定
    ├── commands/                # プロジェクト用スラッシュコマンド(レガシー)
    │   └── <名前>.md
    └── skills/                  # プロジェクト用スキル(現在の推奨形式)
        └── <名前>/SKILL.md
```

### CLAUDE.md ─ Claudeの「頭脳」

プロジェクトの文脈・規約をClaudeに常時読ませておくためのファイルです。書く内容の例：

```
## Common Commands
npm run dev       # 開発サーバー起動
npm test          # テスト実行
npm run build     # ビルド

## Directory Structure
app/models/   - DBモデル
app/api/      - ルートハンドラ
app/core/     - 設定とユーティリティ

## Tech Stack
- TypeScript(JavaScriptは使わない)
- React + Vite
- Prisma(ORMはSequelizeは使わない)

## Code Style
- インデント: 2スペース
- 命名規則: camelCase(変数)、PascalCase(コンポーネント)
- コミット: feat/fix/docsプレフィックスを使うこと

## Forbidden
- any型の使用禁止
- console.logを本番コードに残さない
```

**コツ：** 最初から全部書こうとせず、`/init`コマンドでコードベースを解析させ自動生成した上で、Claudeが間違えるたびに追記していくのが効率的です。「短い方が良い」とされ、300行より30行のCLAUDE.mdの方が実際には機能します。

### CLAUDE.local.md ─ 自分専用の上書き設定

チームには共有したくない、自分だけの環境設定・好みを書く場所。`.gitignore`に追加して管理し、CLAUDE.mdより後に読み込まれるため内容が優先されます。

```
## My Local Environment
- エディタ: VS Code(`code`コマンドで開く)
- Node.js: nvmで管理、`nvm use`を先に実行

## My Preferences
- コミットメッセージは日本語で書く
- コードコメントは英語で書く

## My Overrides
- チームはJestだが、自分のローカルはVitestで実行
```

### settings.json ─ Claudeの「柵」(権限制御)

CLAUDE.mdが「知識・文脈」なら、settings.jsonは「何を許可・禁止するか」を決めるファイルです。3箇所に分散して管理されます。

| ファイル | スコープ | git管理 |
| --- | --- | --- |
| `~/.claude/settings.json` | 個人・全プロジェクト共通 | しない |
| `.claude/settings.json` | プロジェクト・チーム共有 | する |
| `.claude/settings.local.json` | プロジェクト・個人上書き | しない(.gitignore) |

```
{
  "permissions": {
    "allow": [
      "Read", "Edit", "Write",
      "Bash(npm run *:*)",
      "Bash(git status:*)",
      "Bash(git diff:*)"
    ],
    "deny": [
      "Bash(rm -rf *)"
    ]
  },
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          { "type": "command", "command": "npx prettier --write $CLAUDE_FILE_PATH" }
        ]
      }
    ]
  }
}
```

許可(allow)にも禁止(deny)にも入っていないコマンドは、実行前に毎回確認を求められます。denyルールは常にallowより優先されます。

### .mcp.json ─ 外部ツール接続設定

MCP(Model Context Protocol)サーバーへの接続情報。GitHub・Slack・社内APIなどをツールとして使わせられます。

```
# MCPサーバーを追加
claude mcp add github -- npx -y @modelcontextprotocol/server-github

# 一覧確認
claude mcp list

# 削除
claude mcp remove github
```

### スラッシュコマンド(独自コマンド)の作り方

自分専用のショートカットコマンドを作れます。以前は`.claude/commands/`形式でしたが、現在は`.claude/skills/<名前>/SKILL.md`形式が推奨です(旧形式も引き続き動作)。

```
---
description: PRの差分をレビューする
allowed-tools: Read, Grep, Glob, Bash(git diff:*)
model: claude-opus-4-8
---

## Changed Files
!`git diff --name-only HEAD~1`

## Review Checklist
1. ロジックのバグ・エッジケース
2. セキュリティ上の懸念
3. パフォーマンス
4. エラーハンドリング漏れ
```

これを`.claude/skills/review/SKILL.md`として保存すると、セッション中に`/review`で呼び出せます。`$ARGUMENTS`で引数を受け取ることも可能です。

## 3-3. 組み込みスラッシュコマンド完全リファレンス

Claude Code内で`/`と入力すると一覧が出ます。**60以上の組み込みコマンド+5つのバンドルスキル**があり、カテゴリ別に整理すると以下の通りです。

### コンテキスト・セッション管理

| コマンド | 何をするか |
| --- | --- |
| `/clear` | 会話履歴を完全リセット(エイリアス： `/reset` `/new`)。タスク切り替え時に推奨 |
| `/compact [指示]` | 会話を要約して圧縮。「〇〇に焦点を当てて」と指示を添えることも可能 |
| `/context` | コンテキスト使用状況をカラーグリッドで可視化 |
| `/resume [セッション]` | 過去の会話を再開(エイリアス： `/continue`) |
| `/rewind` | 会話・コード変更を巻き戻し(エイリアス： `/checkpoint` `/undo`) |
| `/branch [名前]` | 会話を新しいセッションに分岐(旧`/fork`、エイリアスとして残存) |
| `/export [ファイル名]` | 会話をファイルまたはクリップボードに書き出し |
| `/rename [名前]` | セッション名を変更 |
| `/recap` | セッション復帰時の要約を表示 |
| `/btw <質問>` | メインタスクを中断せず横道の質問をする(本筋の文脈を汚さない) |
| `/goal <条件>` | セッションの完了条件を登録。達成までClaudeが自律的に作業を継続。`/goal clear`で解除 |
| `/copy [N]` | 直近(またはN番目)の応答をクリップボードにコピー |

| `/permissions` | ツールの許可・拒否設定を確認・変更(エイリアス： `/allowed-tools`) |

| `/config` | 設定画面を開く(エイリアス： `/settings`) |

| `/cd <パス>` | プロンプトキャッシュを保ったまま作業ディレクトリを移動 |

## 3-4. Hooks(フック)

特定のイベント発生時にシェルコマンドを自動実行する仕組みです。

| イベント | タイミング |
| --- | --- |
| `PreToolUse` | ツール実行前(危険なコマンドを止めるガードレールに使う) |
| `PostToolUse` | ツール実行後(自動フォーマットなどに使う) |
| `PreCompact` / `PostCompact` | コンテキスト圧縮の前後 |
| `PermissionRequest` | 権限確認ダイアログ表示時 |

`PreToolUse`で終了コード2を返すとそのツール呼び出しを拒否できるため、「危険なBashコマンドを弾く」「機密情報を書き込む前に止める」といったガードレールに使われます。

## 3-5. サブエージェント(Subagents)

独立したコンテキストウィンドウを持つ別セッションを、タスクの一部として並列に立ち上げられる機能。大規模な作業を分割し、コンテキストを汚さずに並行処理したい場合に使います。

```
claude --agents '{
  "code-reviewer": {
    "description": "コード変更後に自動レビュー",
    "prompt": "セキュリティ・パフォーマンス・ベストプラクティスに焦点を当ててレビューする",
    "tools": ["Read", "Grep", "Glob", "Bash"],
    "model": "sonnet"
  }
}'
```

## 3-6. スキル・プラグイン・スラッシュコマンドの違い

| 概念 | 正体 | 使い所 |
| --- | --- | --- |
| スラッシュコマンド | `/名前`で呼び出すプロンプトテンプレート | 定型作業のショートカット |
| スキル | 指示+補助ファイルをまとめた名前付きバンドル。自律的にも呼ばれる | ドメイン知識・手順の共有 |
| サブエージェント | 独立したコンテキストを持つ別セッション | 並列処理・文脈分離 |
| プラグイン | スキル・フック・MCP設定などをまとめた配布単位 | チーム/公開マーケットプレイスでの共有 |

# 4. Claude Cowork 解説

Coworkは、**ターミナル操作をしない人向けの「Claude Codeのエージェント能力」** です。Claude Desktopアプリ(Web/モバイルにも展開中)上で動作し、ファイル操作・ブラウザ操作を自律的に行います。

## 4-1. 通常チャットとの決定的な違い

「チャットでもファイルをアップロードして資料は作れるのに、何が違うの?」という疑問はもっともです。違いを一言でいうと、**チャットは「答えを返す」、Coworkは「仕事を完遂する」** です。

|  | Claude.ai チャット | Cowork |
| --- | --- | --- |
| 基本動作 | 1問1答。自分が対話をリードする | タスクを渡すと計画→分解→実行→成果物まで自走 |
| ファイルの扱い | 手動でアップロード(点数・容量に制限) | フォルダを接続すれば**直接読み書き**。アップロード不要、大量ファイルも一括処理 |
| 作業の粒度 | 1回の応答で完結する範囲 | 複数ステップ・数十分〜数時間の作業を分解して継続 |
| 並列処理 | なし | 大きなタスクをサブタスクに分割し**並列実行**(調査しながら下書きも進める等) |
| 自分の関与 | ずっと画面の前にいる必要がある | 席を離れてOK。リモートセッションならPCを閉じても継続 |
| 定期実行 | 不可 | スケジュールタスクで無人実行可能 |
| PC操作 | 不可 | ブラウザ操作・デスクトップアプリ操作まで可能 |

**具体例で比べると：**

* チャット：「この領収書PDF(1枚アップロード)の内容を表にして」→ その1枚分の表が返る
* Cowork：「経費フォルダの領収書200枚を全部読んで、月別・費目別に集計したExcelを作って」→ フォルダを直接スキャンし、完成したExcelファイルがフォルダに保存される

## 4-2. Claude Codeとの違い

|  | Claude Code | Cowork |
| --- | --- | --- |
| 対象ユーザー | エンジニア | 非エンジニア含む全員 |
| 操作方法 | ターミナル/IDE | チャットのような対話UI |
| 主な用途 | コード生成・デバッグ | ファイル整理・資料作成・調査 |
| 得意分野 | リポジトリ・git・CI | ドキュメント・スプレッドシート・ブラウザ業務 |

Anthropic自身が「Claude Codeのエージェント能力を知識労働向けに展開したもの」と位置づけており、**開発はCode、それ以外のデスクワークはCowork**という棲み分けが基本です。

## 4-3. Coworkでできること(詳細)

### ① ローカルファイルの直接操作

* 接続を許可したフォルダ内のファイルを、アップロード不要で直接読み込み・編集・新規作成
* PDF・Excel・Word・PowerPoint・画像など幅広い形式に対応
* 数式入りExcelや整形済みドキュメントなど、そのまま提出できる品質の成果物を生成
* 例：「ダウンロードフォルダを種類別に整理して」「この10個のPDFから要点を抽出して1本のレポートに」

### ② タスクの自律実行・並列処理

* 依頼を分析→計画を提示→承認後に実行、という流れで進む
* 複雑な作業は自動でサブタスクに分解し、複数のワークストリームを**並列で**進める
* 作業中も「今何をしているか」が可視化され、途中で介入することも放置することもできる

### ③ リモート実行(ベータ)

* 処理はAnthropicのサーバー上の隔離環境で実行され、セッションとファイルはアカウントに保存
* **ラップトップを閉じても作業は継続**。デスクトップ・Web・モバイルのどこからでも同じセッションを開ける
* ローカルファイルやブラウザを使うタスクだけは、そのPCでClaude Desktopアプリを開いておく必要がある

### ④ スケジュールタスク(定期実行)

* 「毎朝8時に業界ニュースをまとめてレポート化」のような定期タスクを設定可能
* リモート実行のため、PCがオフラインでも実行される
* 実行履歴は「スケジュール済み」ページから確認可能

### ⑤ Dispatch(スマホから母艦PCに指示)

* スマホからメッセージを送ると、デスクトップPC側のClaudeがローカルファイル・コネクタ・アプリを使って作業
* 外出先から「机の上のPC」に仕事を投げておき、帰ってから成果物を確認する使い方ができる

### ⑥ ブラウザ操作(Claude in Chrome)

* Chrome拡張機能経由で、Webページの閲覧・情報抽出・フォーム入力・クリック操作を自動化
* 例：「競合5社の最新プレスリリースを収集し、要点をExcelにまとめて」→各社サイトを巡回して表に整理

### ⑦ コンピュータ操作(Computer Use)

* デスクトップ上の実アプリを直接操作(Pro/Maxで研究プレビュー)
* 例： Excelアプリでスプレッドシートを更新させる、社内ダッシュボードを操作させる

### ⑧ プロジェクト

* 関連タスクを1つのワークスペースにまとめ、ファイル・指示・メモリを保持
* 「四半期レポート作成」プロジェクトに過去のテンプレートを入れておけば、毎回の説明が不要に

### ⑨ Live Artifacts(デスクトップのみ)

* 接続アプリやローカルファイルの最新データを反映する、**開くたびに自動更新されるHTMLダッシュボード**を作成できる

### ⑩ プラグイン・コネクタ

* Sales・Finance・Marketing・Legal・HRなど業務領域別のプラグイン(スキル+コネクタ+コマンドのパッケージ)を導入可能
* MCPコネクタでGmail・カレンダー・Notion・Slackなどと接続(設定は数クリック、コード不要)

## 4-4. 使い方

1. Claude Desktopアプリをインストール(claude.com/download)。Web/モバイルでもメッセージボックスで「Cowork」を選択して開始できる(ベータ、段階展開中)
2. Pro以上のプランでログイン
3. メッセージボックス左下で「Cowork」を選択
4. やってほしいタスクを説明する
5. Claudeの実行計画を確認してから実行を許可する

## 4-5. 安全に使うための注意点

* **フォルダ・ツールへのアクセスは自分で選択**でき、それ以外にはClaudeは触れない
* 削除など重要な操作には承認が必要な設定にできる
* ブラウザ操作はサンドボックス外で直接画面操作するため、機密情報を扱うサイトでの利用は避けることが推奨されている
* スケジュールタスクはリアルタイム監視できないため、まずは低リスクな要約・整理タスクから始めるのが安全

## 4-6. プランごとの違い

| プラン | Coworkの位置づけ |
| --- | --- |
| Free | 利用不可 |
| Pro | 含まれる。フォルダ整理や短いレポート作成など軽いタスク向け |
| Max | より複雑・長時間のタスクに対応する枠が多い |
| Team | Slackコネクタも含む。シート課金制 |
| Enterprise | 管理者による利用制御・監査対応あり |

# 5. その他の周辺機能

## 5-1. Claude Design(AIデザインツール)

Anthropic Labs(実験的プロダクト部門)が2026年4月17日にリリースした、**会話だけでデザイン成果物を作れるツール**です。リサーチプレビュー(ベータ)段階ですが、Pro以上の有料プランに追加料金なしで含まれています(Freeでは利用不可、Enterpriseはデフォルト無効で管理者の有効化が必要)。

**何ができる：**

* **動くプロトタイプ**：「予約システムのモバイルUIを5画面で」→ボタンを押すと実際に画面遷移するプロトタイプを生成
* **スライド・ピッチデック**：「この提案書PDFを15枚のピッチデックにして」のような変換も可能
* **LP(ランディングページ)・ワンページャー・広告素材・ドキュメント・ワイヤーフレーム・アニメーション**などテンプレートを選んで生成

**画面と使い方：**

1. `claude.ai/design` にアクセス(またはデスクトップアプリのHomeタブ左下で「Design」を選択)
2. 画面は「左にチャット、右にライブキャンバス」の2ペイン構成
3. 日本語で「AI研修の提案書を作って」のように指示→数秒〜数分で初稿が生成される
4. 修正は4つの方法から： **チャットで指示**/**インラインコメント**/**要素の直接編集**/**AIが自動生成する調整スライダー(Tweaks)**

**デザインシステムの取り込み：**

* GitHubリポジトリ・デザインファイル・アップロード・Claude Codeの`/design-sync`コマンドから、自社のブランドカラー・フォント・コンポーネントを取り込める
* 以降の生成物はすべて自社デザインシステム準拠でチェックされてから表示される
* 大規模リポジトリは全体でなく`src/components/`などデザイン関連フォルダだけ接続するのが推奨

**エクスポート・連携：**

* PDF / PPTX / スタンドアロンHTML / ZIP形式で書き出し可能
* **Canvaへ直接送信**して仕上げ・共同編集に引き継げる(公式連携)
* **Claude Codeへのハンドオフ**： ZIPをCodeのプロジェクトに渡して実装を継続。逆にCode側からMCPサーバー(`claude mcp add claude-design`)を接続してターミナルからデザイン操作も可能
* URLでの共有・コメント・共同編集に対応(非公開/閲覧のみ/編集可の3段階)

**注意点：**

* 利用量はチャット・Code・Coworkと**共通のプラン上限**にカウントされる(以前は別枠だったが統合された)
* PCブラウザ前提で、iOS/Androidアプリからは利用不可
* FigmaやCanvaの「代替」ではなく「たたき台を高速に作る補完ツール」という位置づけ。精密な仕上げは既存ツールに渡すのが現実的

## 5-2. Claude Tag(Slack常駐AIエージェント) ※Team/Enterprise限定

2026年6月23日に発表された、**SlackチャンネルにClaudeを「AIの同僚」として常駐させる機能**です。旧「Claude in Slack」の後継で、2026年8月3日に自動移行されます。現状Team/Enterpriseプランのベータで、**個人のProプランでは使えません**が、チーム導入を検討する際の参考として詳しく書いておきます。

**基本の使い方(3つの入り口)：**

| 入り口 | 動作 | 課金 |
| --- | --- | --- |
| チャンネルで`@Claude`をタグ付け | **組織のアイデンティティ**で動作。管理者が設定したツール・アクセス権を使用 | 組織に課金 |
| `@Claude`へのDM | **自分のClaudeアカウント**の機能(Web検索・コネクタ等)をそのまま使用 | 個人アカウントに課金 |
| AIアシスタントパネル | Slack画面右側のパネルからどこでもClaudeにアクセス | 個人アカウントに課金 |

**従来のチャットボットとの違い：**

* **マルチプレイヤー**： チャンネル内のClaudeは1人で、全員が同じClaudeとやりとりする。誰かが始めた作業の途中経過が全員に見え、別のメンバーが途中から引き継げる
* **継続学習**： チャンネルの会話を追い続けて文脈を蓄積。月曜のスタンドアップで共有した情報が木曜にも残っている。許可すれば他チャンネルやデータソースからも知識を集める(プライベートチャンネルは対象外)
* **自律実行**： タスクを任せると数時間〜数日にわたり自律的に作業。依頼者がオフラインでも進む
* **アンビエントモード**： 有効化すると、Claudeが能動的に関連情報をフラグ立てし、完了を通知し、スレッドが停滞したら担当者にメンションする。「指示待ち」ではなく自分から動く

**できることの例：**

* 「@Claude このPRを作って」「先週の売上をまとめて」「このデータを分析して」→タスクを分解し、許可されたリポジトリ・コネクタを使って実行、結果をスレッドに投稿
* Anthropic社内では、プロダクトチームのコードの約65%が社内版Claude Tagによって書かれていると公表されています。コード生成のほか、プロダクト指標の追跡、サポートチケットのトリアージ、バグ原因調査にも使われているとのこと

**管理・ガバナンス(導入検討者向け)：**

* セットアップできるのはPrimary Owner / Owner権限のみ
* メンバーアクセスは3階層(Slack全ユーザー/Claude組織メンバー/ロール許可者のみ ※3つ目はEnterprise)
* 支出管理： 組織全体のハードキャップ+チャンネルごとの個別上限。75%/95%到達でアラート、上限超過の作業は「静かに短縮」されず明示的に拒否される
* 監査ビュー:：投稿・コミット・PRを起点のSlackスレッドまで追跡可能

## 5-3. Claude in Chrome

Chrome拡張機能としてインストールし、Claudeがブラウザ上で実際にクリック・入力・ナビゲーションを行う。Coworkから呼び出されるほか、単独でも利用可能。

## 5-4. Claude for Excel / Word / PowerPoint / Outlook

Microsoft 365内で直接Claudeを呼び出せるアドイン。4アプリ間で同じ文脈を共有できるため、「Excelで分析→PowerPointで資料化→Wordでレポート化→Outlookで送信」という一連の作業を同じデータを覚えたまま進められる。Pro以上で利用可能。

# まとめ

* 個人のProプランでも「Projects・Artifacts・Research・メモリー・Cowork・Claude Code・Microsoft 365連携」までかなり広くカバーされている

* **Cowork**は「非エンジニア版Claude Code」というポジションで、ファイル操作やブラウザ操作を丸ごと任せられる
* Claude Tagのようなチーム常駐型の機能はTeam/Enterprise限定のものもあるため、個人利用の場合は事前にプラン対応を確認する
