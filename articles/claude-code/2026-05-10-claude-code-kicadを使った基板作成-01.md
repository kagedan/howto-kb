# きっかけ

* これまでは電子工作のマイコンに、ESP32やラズパイPicoを使ってきた。理由はmicropythonで記述できるから。
* 今回、熱帯魚の照明や家庭菜園の水やりのために、毎日決まった時間にオンにするというシンプルなデバイスを作成しようと思い立った
* 私は、C++は書けないし基板の作成も素人。やりたいのは開発・工作であってC++や電子部品の勉強ではない。そういった背景もあり、これまで基板作成のハードルを高く感じていた。
* 昨今、アプリ開発もclaude codeに指示を出すだけでコードを書くことがめっきり減っている。
* 同じようにハードウェア開発もclaude codeを使ってできないかを試した

# 使ったもの

* claude code

* マルツオンライン

# 最終的にこれがベスト

### 1. 要件や設計を決める

claudeチャットでもchatGPTでも良いですが、壁打ち形式で要件を決めていきます。

実は私は当初、シンプルなマイコンとしてPicマイコンしか知りませんでしたが、AIに「Macのcursorでコードを書くことができ、書き込みも容易なもの」という要望で相談したところ、ATtinyを教えてもらいました。

また、要望（毎日午前9時にオンする動作を）に対して、何を使って実現していくか、という話も大まかに決めていきます。

ただし、セカンドオピニオン的にclaudeに聞いたことをchatGPTに聞くという作業はしたほうがいいです。

webアプリ作成のことならAIはかなり正確な回答を返してくれる昨今ですが、ハード系となると思い込みや憶測で回答してくることが多いです。

### 2. モノレポでプロジェクトを作成する

これは今回のプロジェクトのためというより、モノレポのほうがライブラリや部品表を管理しやすいという理由です。

私の場合は、pico、pico W、ESP32、M5stackというようにハードウェアごとにリポジトリを作成しています。  
そして、各デバイスごとのスタブやライブラリを共通化させることで、コード補完ができるようにしています。

このやり方は、claude codeを使う上でもものすごく便利なので、ぜひともモノレポ管理をおすすめします

### 3. 部品在庫表を作る

私の場合は、家に秋月電子で購入したNch MOSFETやダイオードなどの部品はありました。

回路設計においてなるべく家にあるものを使ってほしいので、部品在庫表を作ります。

部品在庫表は、ATtinyだけでなくラズパイやESP32でも使うので、プロジェクトのリポジトリとは独立させて、スプレッドシートに保存しておくといいでしょう

### 4. claude codeに要件やプロジェクトを読み込ませる

ここからはお馴染みの手法です。

リポジトリにdocs/定期実行デバイス仕様書.mdというファイルを作成し、その中に1で作成した設計やら要件を書きます。

そこまで正確に書かなくても構いません。  
claude codeと対話する中で要件を擦り合わせていきます。

そして、このリポジトリでclaude /initを実行して内容を読み取らせます。  
（この後は、基本的claude codeでの作業です）

### 5. 部品の接続表を作成する

3で作った部品在庫表をソースに、プロジェクト内に今回の工作に使用する部品と、その部品の各端子がどの部品の端子と接続されているかを一覧にした部品接続表を作成します。  
（本当は直接回路図を書いてほしいところですが、今のところうまくいかないです）

素人で恐縮ですが、私はダイオードとかコンデンサって用途名と材質名が混在していて苦手です。デカップリングコンデンサと言われてもピンと来ず、結局セラミックコンデンサを使えばいいという結論に至るまで何度もAIに聞き返しました。

そういったメモも書いておくといいでしょう

### 6.手動で回路図を作成

実はここはまだAIでうまくいかない部分です。この後でも述べますが、AIにschファイルを読ませたり作成させても、うまくいかないので、基本的な編集は手でやったほうが早いです。

※ AIは間違いチェックにも使えません。ダイオードの向きを逆と思い込んで修正を求めてきます

ただし、フットプリントの割り当て作業という地味に面倒な作業は前述の部品接続表を使ってできるので、claude codeには「フットプリントを割り当てて」と指示を出すだけで全部やってくれます

### 7. 基板を作成

ここも手動でやるほうが早いです

スクショを撮ってAIに聞くという動作を繰り返してます

2026年時点のKiCad（バージョン10）はAIとのデータやり取りはテキストベースでしかできていないっぽいですが、来年リリースされるバージョン11ではMCPが実装されるので、そこは期待しましょう

# うまくいかなかったこと

## claude codeに回路図を作成させる

schファイルやpcbファイルをcursorやVSCodeで開けるの、知ってましたか

バイナリ形式かと思いきや、実はテキストで書かれています。

ということはclaude codeが直接SCHやPCBを作成できるのでは?と期待して作成させましたが、claude codeが作成したschファイルをKicadで開こうとするとエラーが出て、まともに開けません

一応、エラーをclaude codeに言うと訂正してきますが、ちゃんと開けるものにするために10回くらいやり取りしなければなりませんでした

さらに、開けたとて配線がつながっていなかったりとボロボロな内容です。

そのわりには回路図をスクショしてAIに送ると適切な回答を返してきたりもするので、AIは回路を理解できないのではなく、schファイルのコンテキスト（1万行くらいの文字列）を理解しづらいのでは、と考えています
