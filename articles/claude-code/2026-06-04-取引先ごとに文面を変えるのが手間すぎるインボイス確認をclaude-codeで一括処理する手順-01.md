インボイス制度（適格請求書等保存方式）が始まって以来、「取引先が適格請求書発行事業者かどうか確認したいが、文面を1社ずつ書くのが追いつかない」という状況が経理担当のあるあるになっています。

確認すべき取引先が多いほど、作業はシンプルに積み上がります。取引先の名前を変えて、担当者名を変えて、送付経緯の1行を調整して——内容は同じなのに、1社ずつメール文面を組み立てて送る。50社あれば4〜8時間。インボイス登録の有無を確認するためだけに、半日が消えます。

これ、**Claude Codeで取引先リストを渡すだけで一括生成できます**。Day3（請求書PDF）・Day8（レシート仕訳）・Day11（銀行明細整形）・Day16（月次チェックリスト）に続く経理シリーズ5本目。今回は「取引先への定型文を量産する」作業を自動化します。

## このワークフローで何ができるか

**ビフォー: 1社ずつ文面を作成・送付に4〜8時間**（取引先50社×5〜10分/社）  
**アフター: 20分以内**（リスト整備5分→CLAUDE.md設定5分→一括生成10分→確認・送付準備5分）

確認文面の他に「返答がない取引先への督促文面」「登録なしと判明した場合の社内報告文」も同じワークフローで生成できます。

向いている人

* 個人事業主・フリーランスで複数の取引先と継続取引があり、インボイス登録状況の整理が後回しになっている
* 購買・経理担当で、仕入先のインボイス登録番号を取引先マスタに反映しきれていない
* 会計事務所のスタッフで、複数クライアントのインボイス対応を代行している

向いていない人

## 手順

### ステップ1: CLAUDE.mdに「自社情報」と「文面テンプレの設計ルール」を書く

新規プロジェクトを作り、CLAUDE.mdに自社の基本情報と生成する文面のルールを書きます。

```
あなたは経理担当として、インボイス制度に関する取引先確認文面を作成します。
取引先リストを渡されたら、以下のルールに従って各社宛の確認文面を一括生成してください。

【自社情報】
- 社名: 株式会社○○（以下、架空）
- 適格請求書発行事業者登録番号: T1234567890123（架空）
- 担当者名: 経理部 山田（架空）
- 問合せ先メールアドレス: accounting@example.com（架空）

【確認文面の設計ルール】
1. 宛名は「{会社名} {担当者名}様」の形式
2. 書き出しは「平素より大変お世話になっております」の定型
3. 本文で確認する内容:
   ① 適格請求書発行事業者の登録の有無
   ② 登録済みの場合は登録番号（T＋13桁）
   ③ 登録予定がない場合はその旨の確認
4. 返答期限は送付日から10営業日以内を設定する（日付は「{返答期限}」のプレースホルダで出力）
5. 件名: 【ご確認のお願い】インボイス制度 適格請求書発行事業者登録番号のご確認について
6. 結びは「お手数をおかけしますが、何卒よろしくお願い申し上げます。」で統一

【文面のトーン】
- 丁寧・簡潔。押しつけや催促の印象を与えない
- 箇条書きで確認事項を明示し、先方が回答しやすい構成にする
- 返答フォーマット（はい/いいえ形式）を文末に付ける

【NG事項】
- 「登録しなければ取引を打ち切ります」等の圧力表現は使わない
- 実在の会社名・個人名を文面に出力しない（架空のサンプル用途）
- 国税庁の確認サイトのURLは掲載しない（先方への作業押し付けになるため）
```

一度設定すれば、取引先リストを差し替えるだけで何度でも使えます。

### ステップ2: 取引先リストをCSV形式で整備する

確認を送りたい取引先を、最小限の情報でリスト化します。Excelの取引先マスタがあればそこから貼り付けるだけです。

```
以下の取引先リストから、CLAUDE.mdのルールに従って各社宛の確認文面を一括生成してください。
返答期限のプレースホルダ「{返答期限}」は 2026-07-18（10営業日後）で埋めてください。

取引先リスト:
No. | 会社名               | 担当者名   | 取引内容     | 優先度
1   | 田中デザイン事務所   | 田中様     | デザイン外注 | 高（月次請求あり）
2   | 株式会社山本印刷     | 山本様     | 印刷費       | 高（月次請求あり）
3   | フリーランス 鈴木様  | 鈴木様     | 撮影業務     | 中（都度依頼）
4   | 合同会社ABC物流      | 佐藤様     | 配送委託     | 中（都度依頼）
5   | 佐々木社労士事務所   | 佐々木様   | 顧問契約     | 高（月次顧問料）
（以下、残り45社分）
```

「優先度」列を入れておくと、「高」の取引先を先に処理するよう指示できます。月次請求が発生している取引先から優先して確認を取るのが、仕入税額控除の観点から合理的です。

### ステップ3: 返答管理シートを作り、状況を追跡する

文面を送付したら、返答状況を管理するシートをClaude Codeに生成させます。

```
上記の取引先リスト（5社）から、返答管理シートをMarkdown表形式で出力してください。
カラム: No. / 会社名 / 送付日 / 返答期限 / 返答状況（未返答/登録済み/登録なし/確認中）/ 登録番号 / 備考
初期値はすべて「未返答」で出力してください。
```

出力例：

```
| No. | 会社名             | 送付日   | 返答期限  | 返答状況 | 登録番号         | 備考 |
|-----|-------------------|----------|-----------|----------|-----------------|------|
| 1   | 田中デザイン事務所 | 2026-06-06 | 2026-07-18 | 未返答 | —          |      |
| 2   | 株式会社山本印刷   | 2026-06-06 | 2026-07-18 | 未返答 | —          |      |
| 3   | フリーランス 鈴木様 | 2026-06-06 | 2026-07-18 | 未返答 | —         |      |
| 4   | 合同会社ABC物流    | 2026-06-06 | 2026-07-18 | 未返答 | —          |      |
| 5   | 佐々木社労士事務所 | 2026-06-06 | 2026-07-18 | 未返答 | —         |      |
```

返答が来たら「返答状況」と「登録番号」を更新するだけ。常に「残り何社が未返答か」が一目でわかります。

### ステップ4: 返答期限を過ぎた取引先への督促文面を生成する

返答期限を過ぎても連絡がない取引先には、督促の文面が必要です。

```
以下の未返答取引先に対して、督促文面を生成してください。
初回確認から2週間が経過した2回目の連絡です。
圧力にならない丁寧な督促で、返答しやすい構成にしてください。

未返答取引先:
- 田中デザイン事務所 / 田中様
- フリーランス 鈴木様
```

初回確認文面とは異なるトーンで、「再確認のお願い」として送付できる文面が出ます。督促まで含めてワークフローに組み込めば、「送りっぱなしで返答が追いきれない」状態を防げます。

### ステップ5: 登録なしと判明した取引先の社内報告文を生成する

取引先から「インボイス登録の予定はない」と返答があった場合、経理責任者・上長への社内報告が必要になります。

```
以下の取引先からインボイス登録なし（予定なし）の回答を受けました。
社内の経理責任者向けに、今後の取引方針について確認を仰ぐ報告文を生成してください。

取引先: フリーランス 鈴木様（撮影業務・都度依頼）
直近の取引実績: 2026年1〜5月で計6回・合計報酬額 約38万円

報告内容に含めるべきポイント:
1. 当該取引先のインボイス未登録の事実
2. 今後の仕入税額控除への影響（経過措置の適用有無）
3. 取引継続 or 見直しの判断を経理責任者・上長に仰ぐ旨
```

確認文面・督促文面・社内報告文の3種類をすべて同じプロジェクト内で生成できるため、インボイス対応の一連の作業がClaude Code 1つで完結します。

## 応用と注意点

**応用1: 新規取引先の開始時に都度チェックする仕組みを作る**  
取引先マスタへの新規登録時に「インボイス登録確認文面を生成する」ルーティンをCLAUDE.mdに追加しておくと、既存取引先の対応が終わった後も新規取引先への確認が漏れません。「新規取引先を登録したら必ず確認文面を出す」習慣が定着します。

**応用2: 定期的な登録番号の有効性確認**  
一度確認して登録番号を取得しても、取引先が後日インボイス登録を失効・取り消しするケースがあります。半年に1回、取引先リストを更新して「登録番号の確認依頼」を再送する仕組みを組み込んでおくと、仕入税額控除の根拠が常に最新状態に保てます。

**注意点1: 確認文面はそのまま送付前に内容確認を行う**  
Claude Codeが生成する文面は自社情報・取引先情報のプレースホルダが正しく埋まっているか、返答期限の日付が正確かを必ず確認してください。特に返答期限の計算（10営業日後）は休日を考慮するため、カレンダーで実際の日付を確認してから送付してください。

**注意点2: 取引先への圧力にならない文面の確認**  
インボイス未登録の取引先に対して「登録しなければ取引を継続しない」という実質的な強制は、下請法・独占禁止法の観点から問題になるケースがあります。確認文面のトーンは「確認のお願い」の範囲に留め、登録の強要に当たらないよう最終確認してください。疑問がある場合は顧問弁護士・税理士に相談してください。

**注意点3: 個人情報の取扱い**  
取引先リストには会社名・担当者名・取引金額等の情報が含まれます。Claude Codeへの入力前に、自社の個人情報管理規程および取引先との機密保持契約との整合を確認してください。

## まとめ

* インボイス取引先確認文面は **取引先リストを渡すだけで50社分が20分以内に一括生成**できる
* 確認文面・督促文面・社内報告文の3種類をすべて同じワークフローで作れる
* 返答管理シートをセットで生成すると「未返答が何社残っているか」が常に見える
* 送付前に返答期限の日付・自社情報のプレースホルダが正しく埋まっているか必ず確認する

インボイス対応は「一度終わらせたら終わり」ではなく、新規取引先の追加・登録番号の有効性確認が定期的に発生します。Claude Codeを使って「リストを渡すだけで文面が出る」仕組みを一度作っておくと、次回以降の確認作業が毎回ゼロスタートにならずに済みます。

経理特化シリーズ（Day3=請求書PDF・Day8=レシート仕訳・Day11=銀行明細整形・Day16=月次チェックリスト・Day21=インボイス確認文面）の5本をセットで使うと、月末経理の全工程——受領・精算・取込・締め・対外確認——がClaude Code 1本で回ります。
