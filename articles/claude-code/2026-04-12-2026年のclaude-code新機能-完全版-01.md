## 1月から4月まで、新規追加だけを全部並べる

## はじめに

Claude Code の changelog は、新規追加・改善・バグ修正・セキュリティ修正が全部混ざっています。「結局、新機能だけで見ると何が増えたのか」が見えにくい。

この記事では、**2026年1月〜4月に確認できる新規追加機能だけ**に絞って、Claude Code の進化を完全版として並べ直します。改善・不具合修正・モデル更新は原則入れない。「新しく追加された機能」だけを拾う、という方針です。

ただし今回は、Claude Code 本体だけに狭く閉じません。**Claude Code本体**、**実質的に連動する Claude Apps 側**、**Claude Platform 側の隣接機能**を分けつつ、Claude Code の進化に直結するものだけを扱います。Cowork、Dispatch、Channels、Managed Agents、Advisor tool まで含めて見た方が、2026年の全体像は正確に見えるからです。

---

## まず結論：2026年のClaude Codeは何を手に入れたのか

一言で並べると、こうです。

* Cowork で作業領域がコードの外へ広がった
* Agent Teams でマルチエージェント協調が始まった
* Automatic Memories でセッション横断の記憶を持った
* MCP connectors で外部知識や業務システムとつながった
* Plugins / admin controls / scheduled tasks で組織運用基盤に近づいた
* Dispatch で非同期作業委任ができるようになった
* Channels で実行中セッションに外からイベントを入れられるようになった
* Computer Use で実際にUIやPCを触れるようになった
* Auto Mode で承認の自動化レイヤーを持った
* Monitor tool で可観測性を持った
* Advisor tool / Managed Agents でより大きな agent runtime に接続し始めた
* voice dictation で音声入力が実用化された
* /powerup で学習コストを下げた

つまり 2026年の Claude Code は、対話型コーディング補助から、**接続され、非同期で動き、環境を操作し、監視されるエージェント運用基盤**へ変わりました。

## 第3章：Agent Teams / Automatic Memories / Fast Mode（2026年2月5日〜7日）

### 革命度：8.5 / 10

### 何が追加されたのか

Claude Code v2.1.32〜36 周辺で、3つの重要な新機能が同時に見えてきます。

### Agent Teams（research preview）

`CLAUDE\_CODE\_EXPERIMENTAL\_AGENT\_TEAMS=1` で有効になるマルチエージェント協調機能。複数のエージェントが並行して異なるコンポーネントを担当できます。

### Automatic Memories

Claude が作業中に有用な文脈を自動的にメモリへ記録し、セッションをまたいで呼び出せるようになりました。`/memory` で管理可能です。

### Fast Mode（Opus 4.6対応）

Opus 4.6 系で fast mode が使えるようになり、速度と品質のバランスを取る新しい推論運用が可能になりました。

### なぜ重要か

Agent Teams はマルチエージェント時代への入口です。Automatic Memories は「毎回ゼロから始める」問題への直接的な回答。この2つが揃ったことで、Claude Code は「1回使い切り」のツールから「使うほど賢くなるシステム」へ近づきました。Fast mode は派手ではありませんが、その運用を現実速度にする土台です。

## 第14章：Auto Mode（2026年春に前面化）

### 革命度：9.0 / 10

### 何が追加されたのか

Auto Mode は、危険な操作だけを止め、それ以外はより自動で進める Claude Code の承認自動化レイヤーです。

### 仕組み

Anthropic の説明では、権限プロンプトの大半は結局承認されている。Auto Mode はその無駄を減らすため、二段階の安全確認で安全な操作を通し、危険な操作だけをブロックします。

* 入力層 — プロンプトインジェクション検知
* 出力層 — トランスクリプト分類器

### なぜ重要か

エージェント運用の最大の摩擦は承認コストです。Auto Mode はその摩擦を大幅に減らす。`--dangerously-skip-permissions` と標準承認フローの間の、実用的な中間地点です。

### 注記

ここは少し慎重に書くべきです。Jan–Apr の公式資料だけだと、「3月25日に突然生まれた」と一本で固定するより、**2026年春に前面化した承認自動化レイヤー** と書く方が安全です。

## 第20章：Advisor tool（2026年4月9日）

### 革命度：8.5 / 10

### 何が追加されたのか

Advisor tool は、Claude Platform 側で public beta 公開された機能です。高速な executor モデルが生成する途中で、より高知性な advisor モデルが戦略的助言を返します。

### 仕組み

* 安価で速い executor model がコードを生成

### なぜ重要か

Advisor tool は Claude Code v2.1.98 の changelog には記載されていません。**Platform 側の機能**です。同日に Claude Code 側では Monitor tool が追加された。つまり4月9日は、**Platform 側に「軍師」、Claude Code 側に「監視」** が揃った日でした。

## 結論：2026年のClaude Codeに何が起きたのか

2026年の新機能だけを並べると、Anthropic が足していたのは細かい便利機能ではありません。足していたのは、

* 外部接続（MCP connectors、Channels）
* 非同期委任（Dispatch、Scheduled tasks）
* 環境操作（Computer Use）
* 自律承認（Auto Mode）
* 協調（Agent Teams）
* 記憶（Automatic Memories）
* 監視（Monitor tool）
* 軍師（Advisor tool）
* 組織運用（RBAC、plugins、admin controls、OpenTelemetry）

です。

つまり 2026年の新規追加はすべて、Claude Code を「使うツール」から「運用するエージェント」へ近づける方向に揃っています。

これを進化の3段階で言い換えると、

* 第1段階（1月） — IDEの外へ出た（Cowork）
* 第2段階（2〜3月） — 接続し、非同期で動き、環境を操作するようになった（MCP、Dispatch、Channels、Computer Use、Auto Mode）
* 第3段階（4月） — 監視され、組織で運用される基盤になった（Monitor、RBAC、GA、Advisor）

Claude Code は 2026年に、**コーディング補助ツールからエージェント運用基盤へ変わった**。これが今年前半の本質です。
