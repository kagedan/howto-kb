CodexやClaude Codeに、少し長い修正や調査を任せることが増えた。  
しかし、作業時間のながさに呼応して「もうタスクが完了したのではないか」「承認待ちで止まっているのではないか」という心配が脳のリソースを奪っていることを感じていた。

そこで、AIエージェントからLINEへ通知を送れるサービス`piko`をつくり、[公開した](https://piko.t-oto.dev/)。

この記事では、`piko`開発に至った"地味だが無視できない苦しみ"と、あなたが5分足らずでそれを解消する方法を紹介する。

## なぜAIエージェントにLINE通知がほしかったのか

AIエージェントは、すでに30分単位の作業を任せられる道具になっている。  
軽い修正なら数分で終わるし、少し大きな調査やリファクタなら、こちらが別の作業をしている間に進めてくれる。

問題は、その終わり方である。

作業が完了したのか、途中で失敗したのか、こちらの判断待ちなのか。  
そこに気づけないと、AIに任せたはずの作業を、人間がずっと気にかける。  
輪をかけるように、ChatGPT/Claudeアプリからの通知挙動は不安定。

**いいから、我々のアテンションが最も高いLINEに送ってくれればいいのだ。**

と、意気込み調べてみたら意外と導入が面倒くさい。LINE API、Webhook、トークン管理、......誰かもっと手軽に導入できるようにはしてくれないだろうか...

という心の声への救世主、それがpikoである。

## 導入はわずか3ステップ

### Step1: pikoのLINE公式アカウントを追加する

まず、pikoのLINE公式アカウント[piko｜AI通知エージェント](https://lin.ee/zbfq1Ab)を追加する。

### Step2: 導入プロンプトをコピーする

案内された導入プロンプトをコピーし、CodexやClaude CodeなどのAIエージェントに貼り付ける。

すると、そのプロンプトを受け取ったAIエージェントは、通知に必要な小さな設定ファイルと送信用スクリプトを、対象リポジトリ内に自ら用意することで無事セットアップが完了。  
もし不安なことがあれば、事前にAIに聞いても良いだろう。

### Step3: 日々のプロンプトに通知条件を書く

セットアップが終わったら、あとは日々の依頼文に一言添えるだけ。

長い修正や調査を任せるときは

仕組み化したければ`AGENTS.md`/`CLAUDE.md`に

```
## Notification policy

- 長時間の修正、調査、テスト実行では、完了・失敗・確認待ちだけをpikoでLINE通知する
- 通知本文には、状態、要約、次に見るべきファイルやコマンドを含める
- 人間の判断が必要な場合は、選択肢と推奨案を短く添える
```

さらに、定期実行タスクで使うと、AIがさらに強力な相棒に！

```
毎日8時と12時に、@googlecalendar、@slack、daily-notes/、@todoistを確認してください。

8時には、今日の予定、午前中に進めるべきこと、忘れていそうな準備をまとめてください。
12時には、午前中のSlackやメモの変化を見て、午後の予定に影響しそうなことを教えてください。

特に、次の4つを見てください。
- 予定の前に準備しておくべき資料やメモ
- Slackで自分の返信や判断が止めていそうな話
- Obsidianの日報やメモに残っている未完了タスク
- 今日やるはずなのに、予定にもタスクにも落ちていない作業

特に重要な情報についてはpikoでLINEしてください。
```
