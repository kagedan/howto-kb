## 1. 音声ファイルからテキスト化される仕組み

議事録自動作成には大きく「文字起こし」と「要約・整形」の2ステップがあります。

**文字起こし（Speech-to-Text）**には、OpenAIが開発した「**Whisper**」が最も実績があります。Whisperは多言語対応・話者識別に優れており、方言混じりの日本語も高精度でテキスト化します。精度は条件次第ですが、クリアな録音では**98%超**の認識率が報告されています。

**要約・議事録整形**には「**Claude**」を使います。2026年現在、Anthropicが提供する最新モデル「Claude Sonnet 4.6」は、長文のテキストを構造的に整理する能力が大幅に向上しており、AI議事録サービス「Rimo Voice」でも採用されています。

つまり、流れはシンプルです：

```
録音ファイル（.mp4 / .m4a / .wav）
    ↓ Whisper（文字起こし）
生テキスト
    ↓ Claude（要約・整形）
完成した議事録
```

この2段階のパイプラインを自動実行する仕組みを、Claude Codeで構築します。

## 2. 実際のプロンプト（コピペOK）

Claude Codeでこのワークフローを動かすために必要なプロンプトをそのまま公開します。

### ステップ①：Whisperで文字起こし

Claude Codeのチャット欄に以下を入力します：

```
以下のコマンドを実行して、meeting_recording.mp4をテキストに変換してください。

uv init whisper-project
cd whisper-project
uv add openai-whisper
uv run python -c "
import whisper
model = whisper.load_model('medium')
result = model.transcribe('meeting_recording.mp4', language='ja')
with open('transcript.txt', 'w') as f:
    f.write(result['text'])
print('文字起こし完了: transcript.txt')
"
```

ポイントはmodel='medium'の指定です。smallは速いですが精度が落ち、largeは精度は最高ですが処理時間がかかります。日本語の通常会議ならmediumがベストバランスです。

### ステップ②：Claude で議事録に整形

文字起こしが完了したら、以下のプロンプトでClaudeに議事録生成を依頼します：

```
transcript.txtを読み込んで、以下のフォーマットで議事録を作成してください。

## 会議概要
- 日時：
- 参加者：（文字起こしから推測）
- 会議目的：

## 決定事項
- （箇条書きで3〜5点）

## アクションアイテム
| タスク | 担当者 | 期限 |
|-------|-------|------|

## 議論の要点
（見出しごとに200〜300文字で整理）

## 次回予定
```

このプロンプト1つで、散らかった生テキストが構造化された議事録に変換されます。実際に試したところ、1時間の会議録音が**約4分**で整形された議事録になりました。

## 3. 議事録フォーマットのカスタマイズ

デフォルトの議事録フォーマットは万能ではありません。会議の種類に応じてカスタマイズすることで、実用性が格段に上がります。

### プロジェクト定例ミーティング向け

```
以下の形式でまとめてください：
- 前回アクションアイテムの進捗確認
- 今週の課題と対策
- 来週までのタスクリスト
- リスク・懸念事項
```

### 1on1ミーティング向け

```
以下の形式でまとめてください：
- メンバーの状況・気持ち（ポジティブ/ネガティブ要因）
- 今週の成果
- 困っていること
- マネージャーのサポート事項
```

### 経営会議向け

```
以下の形式でまとめてください：
- KPI達成状況
- 主要な意思決定事項
- 部門別課題
- 次回会議までの優先アクション
```

自社の議事録テンプレートをClaude Codeに一度覚えさせれば、以降は「いつものフォーマットで議事録を作って」と言うだけで対応してくれます。Claude Codeのプロジェクトメモリ機能を活用するのがコツです。

## 4. Slack・Notionへの自動送信連携

議事録が完成したら、次は**自動配信**の仕組みを作ります。

### Slack連携

```
作成した議事録を、Slack APIを使って #general チャンネルに投稿してください。
メッセージには「📝 本日の議事録」というヘッダーを付けてください。
SLACK_BOT_TOKEN は環境変数から読み取ってください。
```

Claude Codeがコードを書いて実行してくれます。Slack Appのトークンを事前に取得しておくだけで、残りはClaudeが全自動でやってくれます。

### Notion連携

```
作成した議事録を、Notion APIを使って「議事録データベース」に新規ページとして追加してください。
タイトルは「YYYY-MM-DD 会議名」の形式にしてください。
NOTION_API_KEY と NOTION_DATABASE_ID は環境変数から読み取ってください。
```

このSlack→Notion連携まで自動化すると、**会議終了から5分後には全員のSlackに通知が届き、Notionにも自動保存されている**状態になります。私のチームではこの仕組みを導入してから、「議事録の送付忘れ」という問題が完全になくなりました。

## 5. 精度を上げる3つのコツ

最後に、文字起こし・議事録の精度を上げるために特に効果的だった3つのコツを共有します。

### コツ①：録音環境の改善が最大の投資

どれだけ優れたAIを使っても、録音品質が悪ければ精度は上がりません。**マイク付きイヤホン（1,000〜3,000円程度）を使うだけで認識率が劇的に改善**します。ノイズキャンセリング機能付きであればさらに効果的です。

対面会議の場合は、会議室の中央にスマートフォンを置いて録音するより、各参加者がPCのマイクで話した方が精度が出ます。

### コツ②：専門用語の辞書を用意する

業界特有の用語・社内略語・人名は誤認識されやすいです。Claudeへのプロンプトに以下を追記するだけで精度が大幅に改善します：

```
※以下の用語・変換規則に注意してください：
- 「きーえー」→「KA（Key Account）」
- 「まとりょーしか」→「マトリョーシカ構造」（社内用語）
- 人名：田中部長、鈴木さん、Johnさん
```

### コツ③：Claudeに「文脈」を与える

議事録の整形精度は、会議の背景情報を与えることで大きく変わります：

```
この会議は、新製品ローンチに向けた週次進捗確認です。
主な議題は「マーケティング予算配分」と「発売日の最終決定」です。
上記の文脈を踏まえて議事録を作成してください。
```

たった3行の背景情報を加えるだけで、Claudeは重要な発言を適切に強調し、無関係な雑談を適度に省略してくれます。
