検索エージェントの「脳」と「メモ帳」を分離したら、20Bモデルがフロンティアモデルに肉薄する性能を叩き出しました。

タイトル: Harness-1: Reinforcement Learning for Search Agents with State-Externalizing Harnesses
Paper 📄: https://t.co/RYgI57f0l6
Code 💻: https://t.co/3hOahvYFoy
Model 🤗: https://t.co/MksLAEAAOW
HF Paper: https://t.co/6jf7qSWkEK

🔍 概要
Harness-1は、強化学習で訓練された20Bパラメータの検索エージェントです。従来の検索エージェントが「何を検索するか」と「何を見たか覚えておく」を同時にモデルに負わせていたのに対し、状態管理を環境側の「ハーネス」に外部化するという新しいアーキテクチャを提案しています。

❓ 解決する課題
従来の検索エージェントは、追記型のトランスクリプト上でポリシーを訓練するため、モデルが意味的な検索判断と帳簿管理を同時に学習する必要がありました。これにより、RLの報酬信号がまばらになり、学習が非効率化。特にハードなクエリでは空の結果セットで終わるロールアウトが多発し、有効な学習信号がほとんど得られない問題がありました。

💡 方法論と提案手法
「認知的オフローディング」の原則に基づき、候補プール、重要度タグ付きキュレーションセット（最大30文書）、エビデンスグラフ、検証キャッシュ、BM25文圧縮、コンテキストバジェット管理といった状態管理をすべてハーネス側に移譲。ポリシーは「何を検索し、どの文書を残し、何を検証し、いつ終了するか」という意味的判断のみに集中します。訓練はGPT-5.4をティーチャーとしたSFTの後、CISPO RLで最適化しています。

📊 実験結果
・8つの検索ベンチマークで平均キュレーションリコール0.730を達成
・次点のオープンソースエージェントに+11.4ポイントの差
・GPT-5.4やSonnet-4.6よりも高い平均リコールを記録し、Opus-4.6のみが上回る
・訓練に使っていないheld-outベンチマークでは、ソースファミリーの2.2倍のゲイン（平均+17.0pt）を達成し、汎化性能の高さを実証

🌍 ユースケース
法的文書や特許の調査、金融レポートの証拠収集、学術文献のマルチホップ検索など、複数文書にまたがる複雑な情報検索タスクに幅広く活用できます。ハーネスの設計原則は、検索以外のエージェント（コード生成、データ分析等）にも応用可能です。
