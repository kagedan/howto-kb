**Claude CodeやCodexで仕事を進めることが経営層や非エンジニアのビジネスパーソンでも一般的になりつつある中で、上手く使えている人とそうでない人の差が大きく開いてきている**、と感じています。

しかもその差は、派手なプロンプトのテクニックや開発知識の有無ではなく、使う上での意識や、タスクの渡し方、あるいは自分のナレッジをどう設計しておくかという、**少しの意識や使い方の違いでついている**ように思います。

今回のnoteでは、私自身が仕事のほとんどをClaude CodeとCodexで進める中で掴んだ、AIエージェントを「仕事のOS」として使えるようになるための12のポイントをまとめました。

## 1. 「方針を考える」「人と会う」「しゃべる」以外は、まずClaude CodeかCodexに投げる

![](https://assets.st-note.com/img/1781085788-XgnNjWaxhYk5umceME2P3JOr.png?width=1200)

自分は**「方針を考える」「人と会う」「しゃべる」の3つ以外は、とりあえずまずClaude CodeかCodexに仕事を投げています**。
