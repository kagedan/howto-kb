■ はじめに

このYouTube動画を見たのがきっかけでした。  
内容を要約すると、こうです。

「Claude Codeとffmpeg・Whisperを使えば、無音カット・文字起こし・字幕追加・音声最適化が全自動になる」

コードを自分で書く必要はない。  
Claudeが全部やってくれる。

「本当に？」と思いながら試してみました。  
今回はその全記録です。

---

■ 作ったもの

完成した仕組みはこうです。

動画ファイルを ~/video-edit/ フォルダに入れる  
　↓  
Claude Codeに「無音カットして字幕もつけて」と入力する  
　↓  
① 無音部分を自動検出・削除  
② Whisperで日本語文字起こし  
③ 字幕を動画に埋め込み  
　↓  
完成動画が output/ フォルダに保存される

必要なのはこれだけ：  
・MacBook（Apple Silicon）  
・Claude Pro（有料プラン）  
・ffmpeg  
・Whisper

■ 環境のセットアップ

【ffmpegをインストール】

ターミナルでこのコマンドを実行するだけです：  
　brew install ffmpeg

HomebrewがMacに入っていれば1コマンドで完了です。

【Whisperをインストール】

　pip3 install openai-whisper

インストール後、「PATHに含まれていない」という警告が出ました。  
こう対処しました：

　echo 'export PATH="$HOME/Library/Python/3.9/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc

動作確認：  
　whisper --version  
　→ 使い方が表示される ✅

【Claude Codeを起動】

　mkdir -p ~/video-edit && cd ~/video-edit  
　claude

ダークモードを選んで、Googleアカウントでログイン。  
「素晴らしいものを構築しましょう」と表示されたら完了です。

■ スキルファイルを作る

Claude Codeには「スキルファイル」という仕組みがあります。  
マークダウンファイルに指示を書いておくと、Claudeがそれを読んで動きます。

~/video-edit/ フォルダに [CLAUDE.md](http://CLAUDE.md) という指示書を作りました。  
内容は「使えるスキル一覧」と「元ファイルを上書きしない」などの基本ルールです。

スキルファイルは4つ用意しました：  
・01\_silence\_removal.md → 無音部分のカット  
・02\_transcribe.md → 文字起こし（Whisper）  
・03\_captions.md → 字幕の追加  
・04\_audio\_master.md → 音声の最適化

Claude Codeはこのファイルを読んで、何をすべきか判断します。

■ 実際に動かしてみた

動画ファイルを用意して、Claude Codeにこう入力しました：

「input.mp4 の無音をカットして、字幕もつけて」

Claudeが自動でやったこと：

① まず動画の情報を確認  
　フルHD・約6分43秒の動画と認識。

② Pythonスクリプトを自動生成  
　以下の処理を含む [process.py](http://process.py) を自分で書きました：  
　1. 無音検出（ffmpeg silencedetect）  
　2. 無音カット  
　3. Whisper文字起こし（日本語）  
　4. 字幕を動画に埋め込み

③ バックグラウンドで実行開始  
　約10分待機。

■ エラーが出た──でもClaude Codeが自分で直した

【エラー①：字幕の書式問題】

「output.mp4: No such file or directory」というエラーが出ました。  
原因はffmpegのオプションのカンマが区切り文字と誤認識されたこと。

Claude Codeが自動修正：  
SRTファイルをASS形式に変換してリトライしました。

【エラー②：libassが入っていない】

「この ffmpeg は libass なし（字幕フィルタ未サポート）でビルドされています」

今度はffmpeg自体の問題です。  
字幕を映像に直接焼き込むには libass というライブラリが必要で、  
インストールされているffmpegにそれが含まれていなかった。

Claude Codeが自動対応：  
　1. brew install libass を実行  
　2. brew reinstall ffmpeg でffmpegを再インストール

それでも解決しなかったため、方針を変えました。  
「映像に焼き込む」のではなく「トラックとして埋め込む（ソフト字幕）」方式に切り替え。

ここが印象的でした。自分で修正方法を考えて、自分で試して、自分で解決した。  
私がやったことは「待つ」だけでした。

■ 使いやすくする設定

【起動を1コマンドに】

ターミナルで「videoedit」と打つだけで起動できるようにしました：  
　echo 'alias videoedit="cd ~/video-edit && claude"' >> ~/.zshrc

【ダブルクリックで起動するファイルを作成】

デスクトップに「動画編集を起動.command」を作成。  
アイコンをダブルクリックするだけで Claude Code が開きます。

【確認ダイアログを減らす】

~/.claude/settings.json に許可コマンドを追加しました。  
ffmpeg・whisper・python3・brew の操作は確認なしで実行できるようになります。

■ ソフト字幕 vs 焼き込み字幕

今回はソフト字幕（トラック埋め込み）方式を採用しました。

　　　　　　　　ソフト字幕（今回）　焼き込み字幕  
YouTube　　　　　　◎　　　　　　　　○  
SNS投稿　　　　　　△　　　　　　　　◎  
あとから修正　　　　簡単　　　　　　要再編集  
処理速度　　　　　　速い　　　　　　　遅い

YouTubeメインなら今回の方法で十分です。  
SNS（InstagramやTikTok）に使いたい場合は焼き込みが必要です。

■ つまずきポイントまとめ

→ プレイヤーの字幕トラックをONにする

→ ソフト字幕方式に切り替え
