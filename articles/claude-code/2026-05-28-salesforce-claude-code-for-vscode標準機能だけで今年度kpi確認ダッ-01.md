## はじめに

「ダッシュボード作りたいけど、どんなレポートを用意してどう組み合わせればいいか迷って時間がかかる・・・」  
Salesforceを使用している人ならば一度は経験したことがあるのではないでしょうか。

本記事では、VSCodeの拡張機能である「Claude Code for VSCode」を使って、レポート・ダッシュボードを作ってみたいと思います。

### この記事のターゲット

・Salesforceシステム管理者 （レポートは作れるが設計に迷う・時間がかかる）  
・社内企画担当者（KPIは決まっているがレポートやダッシュボード作成が苦手）

### なぜClaude Code for VSCodeなのか

最大の理由は、1つの画面でやり取りするだけで実装の基盤が完成するためです。

**実装の作業時間が体感で約1/3** になり、削減できた時間を実装以外の部分（実際に動作するかの確認やさらにデータを活用するための思考など）に使えるようになりました。

## KPIの決め方

まずはじめに、**KPIはAIに決めてもらうものではありません。**

「いい感じの営業KPIを出して」とAIに丸投げすることは技術的には可能ですが、出てきた指標が「自分たちが見たい数字」かどうかは、業務を理解している人間にしか判断できません。

KPIに限らず、「〇〇の状況を確認したい」という背景があってレポートやダッシュボードを実装するはずです。「〇〇」の部分は実装依頼前に明確にしておきましょう。

### KPI設計の4ステップ

```
Step 1：ゴールを1行で書く
　「誰が、何を判断するための数字か」を明確にする
 例：「各チームリーダーが、今月の営業目標達成率で残り営業日のチームのアクションを検討する」

Step 2：今見えていないことを言語化する
　「理想の状態」と「現在の状態」のギャップが見える指標か
 例：「最終的な売上額は見えているが、進行中の商談がどれくらい停滞しているかが見えていない」

Step 3：計測可能な形に落とす
　「〇〇率」「〇〇金額」「〇〇件数」の形式で定義する
例：「最終活動日から14日以上経過している商談の件数」や「年間売上目標金額に対する現時点での達成率」

Step 4：実際にSalesforceで実装する
　← ここからClaude Codeの出番
```

AIが真価を発揮するのは **Step 4以降** です。「決まった定義を正確に・素早く実装すること」に使います。

## Claudeへの効果的なプロンプト

Claudeとのやり取りで時間とトークン消費を無駄にしないコツは、**「目的・制約・出力形式をセットで最初にまとめて渡す」** ことです。

今回の全体の流れはこれだけです。

```
① KPIの概要をまとめてClaudeに渡す（この時に制約や補足もまとめて渡す）
　　↓
② Claudeが実装方法の検討〜実装を行い・デプロイのコマンド（GOサイン）を出す手前まですべてを出力
　　↓
③ デプロイ（作成した機能を環境に反映）
```

### 実際に使ったプロンプト

※プロンプト内に記載のある（JbltGJHWqnxvAeqiKlGbynyoCfiQsr）のようなランダム英数字は一意の名前です。ご自身が指定したいものに変換してください。

```
【背景】
- あなたはSalesforce環境のシステム管理者です。
- 全ての質問や依頼には根拠に基づき回答を生成してください。
- 根拠なしでの憶測で生成せずに、不明点や疑問点は回答生成前に質問してください。

【依頼】
- 画像に記載されている6指標を含む今年度KPI確認用ダッシュボードとそれに必要なレポートSalesforce内で生成してください。
- できるだけ既存項目（Amount, CreatedDate, CloseDate, StageName）を活用し、レポート内の数式やバケット列を駆使してください。
- 違和感を感じた際や項目が不明な場合は必ず質問してください。

【補足】
- ダッシュボードは「今年度KPI（JbltGJHWqnxvAeqiKlGbynyoCfiQsr）」を編集してください。　
- ダッシュボードに使用するレポートを生成した場合は全てレポートフォルダ「ブログ用レポートフォルダ（ReportForBlog）」に格納してください。

【画像内の補足】
- 年度ごとの目標は期初に会社として決まっています。今年度は1000万円です。
- 小規模商談は300万円以下、中規模商談は300〜700万円、大規模商談は700万円以上　です。
- 「担当者別達成状況」レポートは、必ず「所有者」でグルーピングしてください。

【出力形式】
- まずは作成予定のレポートとその構成、ダッシュボードコンポーネントの設定をテキストで出力してください。
- 合意後にこれらを配置するメタデータXMLと、sf project deploy用のコマンドを出力してください。
```

実際上記のプロンプトで送信しても質問や確認はありますし、100点のダッシュボードが作成できたわけではないです。ただ、「〇〇が確認できるレポートとダッシュボード作って」というアバウトな依頼よりは遥かにまともな成果物ができます。

### ほぼ確実に遭遇するメッセージ

Claudeから以下のような英語だらけのメッセージが出ても焦らないでOKです！  
![](https://static.zenn.studio/user-upload/c12d5cc8b9ef-20260522.png)  
*よく出現する確認メッセージ*

これはClaudeが「あなたの代わりに、Salesforceの設定（項目名やフェーズ名など）を自動で下調べしていいですか？」という確認をしてくれています。

| 選択肢 | 解説 |
| --- | --- |
| Yes | 今回だけ実行する |
| Yes, allow … for this project | 同じコマンドパターンをプロジェクト限定で以降もスキップ（自分だけに適用） |
| No | 実行しない |
| Tell Claude what to do instead | テキスト欄で別指示も入力可 |

迷ったら Yes で1回ずつ確認しながら進めるのが無難です。  
慣れてきたら2番目の選択肢で作業を効率化できます。

![](https://static.zenn.studio/user-upload/bc54b8f4f141-20260522.png)  
👆️プロンプト内で依頼しているので、回答を生成するために必要な情報が欠損していたら「逆質問」をしてくれます。

### デプロイ（機能を反映）後に私たちがやるべきこと

メタデータのデプロイが通っても・・  
Salesforce環境で確認するとコンポーネントがエラー（「データが表示できません」等）になっていることがあります。  
これは環境固有のフェーズ名や集計単位の微妙なズレが原因の場合が多いです。

**デプロイ後に管理者が実施すべきことは確認＆修正**

* 確認・微修正は自分でやる：レポート画面でサクッと条件を直す（一番早いし確実です）
* 自身で確認・Claudeに修正を依頼する：エラー内容を貼り付けて修正するよう指示する（細かい確認のやり取りが発生する可能性があります）

**エラーの確認・修正依頼のプロンプト例**

```
【背景】
先ほどデプロイしたダッシュボードの「〇〇の達成率」というコンポーネントで「このレポートはこのコンポーネントのソースとして使用できません。サマリーレポートかマトリックスレポートの場合、1 つ以上のグループ化をレポートに追加してください。行制限のある表形式レポートの場合、レポートの [ダッシュボードの設定] を指定してください。」というエラーが発生しています。

【目的】
- 最終的には現在の実行ユーザーで同じコンポーネントを確認した時にエラーが出ない状態に修正することが目的です。
  
【依頼】
- このエラーを解決するための原因調査を行ってください。
- 権限不足が原因ならば「どのユーザーに対して何の権限が不足しているか」を教えてください。
- 権限以外が原因の場合は原因の詳細と解決方法を提示してください。
- エラー解消だけでなく「〇〇の達成率」が正しく確認できるレポート・ダッシュボードである必要があります。

【出力形式】
- エラーの内容と修正方法の提案をテキストベースで回答してください
- 合意後にこれらを配置するメタデータXMLと、sf project deploy用のコマンドを出力してください。
```

## おわりに

昨今「AI」がみるみるうちに進化していて、実際今まで実装者が時間をかけて実施していた業務を「指示するだけ」でAIがこなしてくれることも増えましたね。  
そんな時代の中で、「Salesforceシステム管理者としての自分の価値はどこにあるのか？」  
私は日々、それを模索しながら業務に向き合っています。

前述の通り、今回このClaudeの回答が一発で100点満点だったわけではありません。  
ただ「何のためにこの数字を見るのか」という本質を問い続け、その数字をもとにどのようなアクションを起こすかという「意思決定」を行い、最後にアウトプットの精度を担保するのは、どこまでいっても私たち人間だと思っています。

何度も言いますが、AIはまだまだ完璧ではありません。こちらの使い方次第で100点に近づけることはできても、最後の最後で「これじゃない」という答えが返ってくることだってあります。

だからこそ、任せられる作業はAIを最大限活用して、そこで生まれた時間を「現場の声」に耳を傾けたり、数字の裏側にある本質を汲み取って次のアクションにつなげたり…そんな、私たち人間にしかできないところに、もっともっと時間を使っていきたいと思っています。
