起動コマンド

一番基本的なことほど、意外と見落とされがちです。これらの起動パラメータを使うと、後々の操作をかなり省けます。  
以下のコマンドは、PowerShell または CMD のターミナルで入力します。

claude  
そのまま対話に入り、現在のディレクトリを開きます。毎日いちばんよく使うコマンドで、特に説明はいらないでしょう。

claude ディレクトリパス  
指定したディレクトリを直接開きます。事前に cd で移動する必要はありません。例えば：

claude ~/projects/my-app

異なるプロジェクトにすばやく入るのに便利です。

claude --resume  
前回の対話を再開します。ターミナルを閉じてしまったり、別の作業に切り替えたりしたあとでも、このコマンドですぐに元の状態に戻れます。改めて要件を説明し直す必要はありません。

claude --dangerously-skip-permissions  
権限チェックをスキップして、すべての権限を直接付与します。素早く開発したいときには便利ですが、リスクは自己責任です。Claude を信頼していて、自由にコードを触らせたいなら、このパラメータを付けてもよいでしょう。

初心者は慎重に使ってください！！！

会話内でよく使うスラッシュコマンド

対話に入ったあと、/ で始まるこれらのコマンドこそ本当の効率化ツールです。覚えておけば、毎回自然言語で説明しなくて済みます。  
以下のコマンドは、Claude Code の作業画面に入ってから使います。

/help  
利用可能なコマンドをすべて表示します。忘れたらこれを打てば、公式ドキュメントのように一覧で確認できます。

/clear  
対話履歴を消去します。話が脱線したり、コンテキストが散らかってきたりしたときに、一度クリアしてやり直すと、Token の無駄も減り、考え方も整理しやすくなります。

/exit または /quit  
Claude Code を終了します。ショートカットの Ctrl + D でもできます。使いやすい方で大丈夫です。

/model  
AI モデルを切り替えます。入力すると選択肢が表示されるので、異なるモデルの性能を手早く試したいときにとても便利です。

Claude 4.6 Sonnet：デフォルト。速度と性能のバランスが良い  
Claude 4.6 Opus：最も高性能。複雑な問題向きだが、その分コストは高め  
Claude 4.6 Haiku：最速。シンプルな質疑応答向き

/context  
現在のコンテキスト使用量を確認します。Token が上限に近づいてきたら、早めに確認して、必要なら整理しましょう。

/cost  
現在のセッションでどれくらい Token を使ったか、いくらかかったかを確認します。細かく把握したい人には必須です。

/review  
コードレビューを依頼します。現在の変更内容を Claude にレビューしてもらえます。人力より細かく見てくれることも多いです。

/compact  
対話履歴を圧縮し、それまでの内容を要約して、より多くのコンテキスト領域を空けます。何ラウンドもやり取りしたあとに使うと、かなり助かります。

/btw（2026年3月追加）  
「ちなみに聞くけど」という感じで、一時的なサイド会話を開くコマンドです。今のメインタスクを中断せずに済み、Token も節約できます。

大きな作業を進めている途中で、ふと小さな疑問が出てきたとき、わざわざ最初からやり直したり、メインの流れを崩したりする必要はありません。  
そのまま /btw この API パラメータの意味は？ のように聞けば OK です。

特徴：

メインの会話には影響しない。主線のコンテキストはそのまま  
主線の Token を消費しないので、かなり節約できる  
既存のコンテキストに基づいてしか回答できず、ツール呼び出しはできない  
「ついでに聞きたい」タイプの質問に向いていて、新しい会話を開く必要がない

実際、Token を半分くらい節約できることもあり、この機能は本当に開発者のことをよく分かっています。

/fork  
現在の会話を分岐して、コピーを別ルートとして処理します。サイドの話が広がって、そのまま新しい主線にしたくなったときに便利です。  
また、元の会話を土台にしつつ、別の案を試したいときにも使えます。お互いに干渉しません。

モード切り替え

Claude Code にはいくつかの隠れたモードがあります。うまく使えば、効率がかなり上がります。

Shift + Tab  
これは Plan モードです。読み取り専用権限になり、Claude は案を出すだけで、コードには触りません。新しいプロジェクトを受け取った直後に、まず全体構造を整理してもらいたいときに向いています。勝手にコードを変更されないので、とても安全です。

Shift + Tab を2回押す  
これは思考モードです。深い思考を有効にし、Claude がより時間をかけて問題を分析し、より周到な案を出してくれます。複雑なバグやアーキテクチャ設計のときには、ぜひ使うべきです。

ちょっとしたコツ：プロンプトに ultrathink を加えると、さらに効果的です。

プロジェクトの初期化

/init  
対話形式でプロジェクトを初期化し、CLAUDE.md ファイルを生成します。  
このファイルは、Claude にとっての「プロジェクト仕様書」のようなもので、例えば次のような内容を書けます。

プロジェクト全体の説明  
コードスタイルのルール  
ビルドやテストのコマンド  
開発フロー上の取り決め

今後 Claude がコードを変更するときも、ここで決めたルールに沿って動くようになります。好き勝手に書かなくなるので、これこそ本当の意味での「Claude のローカライズ」です。この機能を知らない人は意外と多いです。

よく使うショートカットキー

こうしたターミナル内のショートカットは、慣れるとマウス操作より何倍も速くなります。

応用テクニック

ファイルの参照

@ファイル名 と入力するだけで、そのファイル内容をコンテキストに追加できます。例えば：

@src/App.js のどこでエラーが出ているか見てください

自分でコピー＆ペーストするより、ずっと速いです。

画像対応

画像をそのままターミナルにドラッグすれば、Claude はスクリーンショット内のエラー情報や UI デザイン案も理解できます。  
この機能は本当に便利で、エラー画面をそのまま投げるだけで問題点を探してくれます。

Skills スキル

Claude Code には、いくつかのプリセット AI ワークフローも用意されていて、これも / から呼び出せます。

/pr-description：PR の説明文を自動生成し、変更内容を分かりやすくまとめる  
/test：単体テストを生成する  
/doc：ドキュメントを生成する

こうしたプリセットスキルを使えば、毎回プロンプトを書く手間がかなり減ります。

私がよく使うコマンドの組み合わせ

日常開発で何度も使っている、かなり実用的な組み合わせをいくつか紹介します。

新しいパソコンでプロジェクトをクローンした直後の最初の一手  
cd my-project  
claude  
/init

いくつかの質問に答えるだけで CLAUDE.md ができ、その後は Claude がずっとあなたのルールを理解した状態で動いてくれます。

新しいプロジェクトをすばやく分析する  
claude

Shift + Tab を押して Plan モードを有効にしてから：

このプロジェクトの構造を整理して、それぞれのモジュールが何をしているのか説明して

読み取り専用で変更はせず、1分ほどで全体の構造を把握できます。

コードを直して、コミット前に確認したいとき  
/review

Claude にもう一度チェックしてもらうと、自分が見落としていた問題が見つかることがよくあります。

コンテキストが散らかって、やり直したいとき  
/clear  
さっきの要件は違いました。最初からやり直します：...

Claude Code のすごいところは、単にコードが書けることではありません。  
本当に、自分を開発フローの中の「ペアプログラミングの相棒」として振る舞ってくれることです。  
これらのコマンドは、あなたと Claude が連携するための「共通言語」のようなものです。慣れれば慣れるほど、連携はどんどんスムーズになります。
