「AIに割安株を探してもらえないか」と思って、ChatGPTやClaudeのWeb版に「割安株を探して」と頼んだことがある人は多いはずだ。それなりの答えは返ってくる。ただし毎回結果が微妙に違う、10銘柄以上を頼むと途中で止まる、セッションをまたぐとコンテキストがリセットされる。投資分析のワークフローに組み込もうとすると、この「再現性のなさ」がネックになる。

この記事では、再現性のある投資分析エージェントをPythonで構築する方法をまとめる。Claude APIを軸に、株価データ取得・財務スクリーニング・バックテストまでを自動化する構成を扱う。

-----

## 全体アーキテクチャ

今回構築するエージェントの全体像はこうだ。

```
[ユーザー入力] 「日本株で割安な高配当銘柄を探して」
         ↓
[オーケストレーター] Claude API（判断・ルーティング）
         ↓
  ┌──────────────────────────┐
  │          ツール群              │
  │  ① yfinance（株価・財務データ）  │
  │  ② スクリーニングロジック        │
  │  ③ バックテストエンジン          │
  │  ④ ニュース取得（Web検索）       │
  └──────────────────────────┘
         ↓
[出力] スクリーニング結果・分析レポート・バックテスト結果
```

重要な設計原則が1つある。スクリーニングのロジックはPythonスクリプトとして固定する。AIに毎回判断させるのではなく、「どう探すか」はコードで決め、「何を意味するか」「次に何をするか」はAIに判断させる。これが再現性の確保に直結する。

## ステップ①：株価・財務データ取得ツール

まずyfinanceで株価と財務指標を取得するツールを作る。

```python
import yfinance as yf
import pandas as pd
from datetime import datetime, timedelta
import json

def get_stock_data(ticker: str, period: str = "1y") -> dict:
    """
    指定した銘柄の株価と主要財務指標を取得する。
    
    Args:
        ticker: 銘柄コード（例: "7203.T"=トヨタ, "AAPL"=Apple）
        period: 取得期間（"1mo", "3mo", "6mo", "1y", "2y"）
    """
    stock = yf.Ticker(ticker)
    
    # 株価履歴
    hist = stock.history(period=period)
    if hist.empty:
        return {"error": f"{ticker} のデータが取得できませんでした"}
    
    # 財務指標
    info = stock.info
    
    return {
        "ticker": ticker,
        "company_name": info.get("longName", "N/A"),
        "current_price": info.get("currentPrice", hist["Close"].iloc[-1]),
        "52w_high": info.get("fiftyTwoWeekHigh"),
        "52w_low": info.get("fiftyTwoWeekLow"),
        # バリュー指標
        "per": info.get("trailingPE"),           # PER（株価収益率）
        "pbr": info.get("priceToBook"),           # PBR（株価純資産倍率）
        "psr": info.get("priceToSalesTrailing12Months"),  # PSR
        "ev_ebitda": info.get("enterpriseToEbitda"),
        # 配当
        "dividend_yield": info.get("dividendYield"),
        "dividend_rate": info.get("dividendRate"),
        # 成長性
        "revenue_growth": info.get("revenueGrowth"),
        "earnings_growth": info.get("earningsGrowth"),
        # アナリスト情報
        "analyst_target_price": info.get("targetMeanPrice"),
        "recommendation": info.get("recommendationMean"),  # 1=強買, 5=強売
        "num_analysts": info.get("numberOfAnalystOpinions"),
        # 直近の価格変動
        "price_1mo_change": (
            (hist["Close"].iloc[-1] / hist["Close"].iloc[-21] - 1) * 100
            if len(hist) >= 21 else None
        )
    }

def get_financial_statements(ticker: str) -> dict:
    """損益計算書・貸借対照表の主要項目を取得する"""
    stock = yf.Ticker(ticker)
    
    try:
        income_stmt = stock.income_stmt
        balance_sheet = stock.balance_sheet
        
        result = {}
        
        # 直近の売上・利益
        if not income_stmt.empty:
            result["total_revenue"] = income_stmt.loc["Total Revenue"].iloc[0] if "Total Revenue" in income_stmt.index else None
            result["net_income"] = income_stmt.loc["Net Income"].iloc[0] if "Net Income" in income_stmt.index else None
            result["operating_income"] = income_stmt.loc["Operating Income"].iloc[0] if "Operating Income" in income_stmt.index else None
        
        # 自己資本比率
        if not balance_sheet.empty:
            total_assets = balance_sheet.loc["Total Assets"].iloc[0] if "Total Assets" in balance_sheet.index else None
            equity = balance_sheet.loc["Stockholders Equity"].iloc[0] if "Stockholders Equity" in balance_sheet.index else None
            if total_assets and equity:
                result["equity_ratio"] = equity / total_assets * 100
        
        return result
    except Exception as e:
        return {"error": str(e)}
```

## ステップ②：スクリーニングロジック

閾値はYAMLで管理する。コードに埋め込まず、設定ファイルで制御することで再現性が保たれる。

```yaml
# screening_config.yaml
japan_value_dividend:
  name: "日本バリュー高配当"
  per_max: 15          # PER 15倍以下
  pbr_max: 1.5         # PBR 1.5倍以下
  dividend_yield_min: 0.03  # 配当利回り 3%以上
  equity_ratio_min: 40      # 自己資本比率 40%以上

us_growth:
  name: "米国成長株"
  revenue_growth_min: 0.15  # 売上成長率 15%以上
  recommendation_max: 2.5   # アナリスト推奨 2.5以下（1=強買）
  per_max: 35

undervalued_consensus:
  name: "コンセンサス乖離割安株"
  # 現在株価がアナリスト目標株価より20%以上低い
  price_to_target_max: 0.80
  num_analysts_min: 3
```

```python
import yaml

def load_screening_config(config_path: str = "screening_config.yaml") -> dict:
    with open(config_path) as f:
        return yaml.safe_load(f)

def screen_stock(stock_data: dict, financial_data: dict, strategy: str, config: dict) -> dict:
    """
    指定した戦略で銘柄をスクリーニングする。
    
    Returns:
        {"pass": bool, "score": float, "reasons": list}
    """
    criteria = config.get(strategy, {})
    reasons = []
    score = 0
    checks_passed = 0
    total_checks = 0
    
    def check(condition, reason_pass, reason_fail, weight=1):
        nonlocal score, checks_passed, total_checks
        total_checks += 1
        if condition:
            reasons.append(f"✓ {reason_pass}")
            score += weight
            checks_passed += 1
        else:
            reasons.append(f"✗ {reason_fail}")
    
    per = stock_data.get("per")
    pbr = stock_data.get("pbr")
    div_yield = stock_data.get("dividend_yield")
    equity_ratio = financial_data.get("equity_ratio")
    
    if strategy == "japan_value_dividend":
        if per:
            check(per <= criteria["per_max"], 
                  f"PER {per:.1f}倍（{criteria['per_max']}倍以下）",
                  f"PER {per:.1f}倍（基準: {criteria['per_max']}倍以下）")
        if pbr:
            check(pbr <= criteria["pbr_max"],
                  f"PBR {pbr:.2f}倍（{criteria['pbr_max']}倍以下）",
                  f"PBR {pbr:.2f}倍（基準: {criteria['pbr_max']}倍以下）")
        if div_yield:
            check(div_yield >= criteria["dividend_yield_min"],
                  f"配当利回り {div_yield*100:.1f}%（{criteria['dividend_yield_min']*100}%以上）",
                  f"配当利回り {div_yield*100:.1f}%（基準: {criteria['dividend_yield_min']*100}%以上）")
        if equity_ratio:
            check(equity_ratio >= criteria["equity_ratio_min"],
                  f"自己資本比率 {equity_ratio:.1f}%（{criteria['equity_ratio_min']}%以上）",
                  f"自己資本比率 {equity_ratio:.1f}%（基準: {criteria['equity_ratio_min']}%以上）")
    
    return {
        "pass": checks_passed == total_checks and total_checks > 0,
        "score": score / total_checks if total_checks > 0 else 0,
        "reasons": reasons
    }
```

## ステップ③：Claude APIとツールをつなぐエージェント

ここがエージェントの中枢だ。Claudeのtool_use機能を使って、「何の情報を取るか」「どう分析するか」の判断をLLMに委ねる。

```python
import anthropic

client = anthropic.Anthropic()

# Claudeに渡すツール定義
tools = [
    {
        "name": "get_stock_data",
        "description": "指定した銘柄の株価と財務指標（PER、PBR、配当利回り等）を取得します。",
        "input_schema": {
            "type": "object",
            "properties": {
                "ticker": {
                    "type": "string",
                    "description": "銘柄コード（例: '7203.T'=トヨタ、'AAPL'=Apple）"
                },
                "period": {
                    "type": "string",
                    "description": "株価履歴の取得期間",
                    "enum": ["1mo", "3mo", "6mo", "1y", "2y"]
                }
            },
            "required": ["ticker"]
        }
    },
    {
        "name": "get_financial_statements",
        "description": "指定した銘柄の損益計算書・貸借対照表データを取得します。",
        "input_schema": {
            "type": "object",
            "properties": {
                "ticker": {"type": "string"}
            },
            "required": ["ticker"]
        }
    },
    {
        "name": "screen_stocks",
        "description": "指定した銘柄リストをスクリーニング戦略でフィルタリングします。",
        "input_schema": {
            "type": "object",
            "properties": {
                "tickers": {
                    "type": "array",
                    "items": {"type": "string"},
                    "description": "スクリーニング対象の銘柄コードリスト"
                },
                "strategy": {
                    "type": "string",
                    "description": "スクリーニング戦略",
                    "enum": ["japan_value_dividend", "us_growth", "undervalued_consensus"]
                }
            },
            "required": ["tickers", "strategy"]
        }
    }
]

def run_investment_agent(user_query: str, max_iterations: int = 10) -> str:
    """
    投資分析エージェントのメインループ。
    Claudeが必要に応じてツールを呼び出しながら分析を進める。
    """
    config = load_screening_config()
    
    messages = [{"role": "user", "content": user_query}]
    
    system_prompt = """あなたは株式投資分析の専門アシスタントです。
ユーザーの依頼に応じて、株価データ取得・財務分析・スクリーニングツールを活用して分析を行います。

重要な注意事項：
- この分析は情報提供目的であり、投資推奨ではありません
- 分析結果には必ず「投資は自己責任で」という注意を添えてください
- 不確実な情報は不確実と明記してください
- バックテスト結果は過去のパフォーマンスであり、将来を保証しません"""

    for _ in range(max_iterations):
        response = client.messages.create(
            model="claude-sonnet-4-6",
            max_tokens=4096,
            system=system_prompt,
            tools=tools,
            messages=messages
        )
        
        # ツール呼び出しがなければ最終回答
        if response.stop_reason == "end_turn":
            return response.content[0].text
        
        # ツール呼び出しを処理
        tool_results = []
        for content_block in response.content:
            if content_block.type == "tool_use":
                tool_name = content_block.name
                tool_input = content_block.input
                
                # ツールを実行
                if tool_name == "get_stock_data":
                    result = get_stock_data(**tool_input)
                elif tool_name == "get_financial_statements":
                    result = get_financial_statements(**tool_input)
                elif tool_name == "screen_stocks":
                    # 複数銘柄を一括スクリーニング
                    screening_results = []
                    for ticker in tool_input["tickers"]:
                        stock_data = get_stock_data(ticker)
                        fin_data = get_financial_statements(ticker)
                        screen_result = screen_stock(
                            stock_data, fin_data, 
                            tool_input["strategy"], config
                        )
                        screening_results.append({
                            "ticker": ticker,
                            "company": stock_data.get("company_name"),
                            **screen_result
                        })
                    result = screening_results
                else:
                    result = {"error": f"未定義のツール: {tool_name}"}
                
                tool_results.append({
                    "type": "tool_result",
                    "tool_use_id": content_block.id,
                    "content": json.dumps(result, ensure_ascii=False)
                })
        
        # アシスタントの応答とツール結果を会話に追加
        messages.append({"role": "assistant", "content": response.content})
        messages.append({"role": "user", "content": tool_results})
    
    return "最大イテレーション数に達しました。処理を中断します。"
```

## ステップ④：簡易バックテスト

スクリーニングで選ばれた銘柄が過去にどのパフォーマンスだったかを検証する。

```python
import numpy as np

def simple_backtest(
    tickers: list, 
    start_date: str, 
    end_date: str,
    initial_capital: float = 1_000_000,
    rebalance_period: str = "Q"  # Q=四半期, M=月次
) -> dict:
    """
    シンプルな等重ポートフォリオのバックテスト。
    
    注意: これは過去データに基づく検証です。将来のパフォーマンスを保証しません。
    """
    import yfinance as yf
    
    prices = yf.download(tickers, start=start_date, end=end_date)["Close"]
    if prices.empty:
        return {"error": "データが取得できませんでした"}
    
    # 等重リバランス
    weights = {ticker: 1/len(tickers) for ticker in tickers}
    
    # 日次リターン
    returns = prices.pct_change().dropna()
    
    # ポートフォリオリターン
    portfolio_returns = sum(
        returns[ticker] * weight 
        for ticker, weight in weights.items() 
        if ticker in returns.columns
    )
    
    # 累積リターン
    cumulative_return = (1 + portfolio_returns).cumprod()
    total_return = cumulative_return.iloc[-1] - 1
    
    # 年率リターン（252営業日換算）
    n_days = len(portfolio_returns)
    annual_return = (1 + total_return) ** (252 / n_days) - 1
    
    # ボラティリティ（年率）
    annual_vol = portfolio_returns.std() * np.sqrt(252)
    
    # シャープレシオ（無リスク金利0.5%想定）
    risk_free_rate = 0.005
    sharpe = (annual_return - risk_free_rate) / annual_vol if annual_vol > 0 else 0
    
    # 最大ドローダウン
    cummax = cumulative_return.cummax()
    drawdown = (cumulative_return - cummax) / cummax
    max_drawdown = drawdown.min()
    
    return {
        "period": f"{start_date} ～ {end_date}",
        "tickers": tickers,
        "total_return": f"{total_return*100:.1f}%",
        "annual_return": f"{annual_return*100:.1f}%",
        "annual_volatility": f"{annual_vol*100:.1f}%",
        "sharpe_ratio": f"{sharpe:.2f}",
        "max_drawdown": f"{max_drawdown*100:.1f}%",
        "final_value": f"¥{initial_capital * (1 + total_return):,.0f}",
        "warning": "⚠️ 過去のパフォーマンスは将来を保証しません"
    }
```

## 再現性を保つための3つのファイル管理

再現性のない仕組みで投資判断は積み重ねられないという問題意識から、設定と状態を3つのファイルで管理する構成にしている。

```
investment-agent/
├── screening_config.yaml   ← 閾値の定義（変更履歴をgit管理）
├── portfolio.csv           ← 保有銘柄リストと取得単価
└── CLAUDE.md               ← エージェントへの永続的な指示
                              （例：「日本株の分析では東証プライム銘柄を優先」）
```

この3ファイルがあることで、セッションが切れても「どう分析するか」「何を保有しているか」「どんなルールで動くか」が引き継がれる。

## Claude Code Teams機能との組み合わせ

Claude Code の Teams 機能を使えば、4つのエージェントがそれぞれ別の地域を同時に探索し、結果を集約して返してくれる。

「日本・米国・ASEAN・香港の4地域で割安株を同時に探したい」ときに特に効果が大きい。逐次実行だと各地域に数十秒かかるが、並列実行なら1/4の時間で完了する。

## できること・できないことの整理

|できること           |できないこと             |
|----------------|-------------------|
|過去データに基づくスクリーニング|将来の株価予測            |
|財務指標の定量比較       |インサイダー情報の取得        |
|バックテスト（過去実績検証）  |リアルタイムの板情報         |
|ニュースの要約・感情分析    |自動売買（証券口座との連携は別途必要）|
|複数銘柄の一括分析       |短期トレーディングの精度保証     |

特に重要な点として、このシステムは「分析補助ツール」だ。投資判断の最終責任は人間が持つ。AIが「通過」と判定した銘柄でも、定性的な判断（経営陣の質、業界の将来性）は人間が加える必要がある。

## まとめ

今回のエージェントの設計を3行で表すとこうなる。

「ロジックはコードで固定、判断はAIに委ねる、状態はファイルで永続化する」

ただしどれだけ精緻なエージェントを作っても、市場の動きを完全に予測するシステムは存在しない。バックテストで良い結果が出た戦略が、未来でも通用するとは限らない。このエージェントは「同じ条件で繰り返し分析できる環境を作る」ツールだ。投資判断そのものを代替するものではない。
