## はじめに

毎月いくつかのサブスクを契約していると、「あれ、いつ解約しなきゃいけないんだっけ？」「合計いくら払ってるんだろう」という場面が増えてきた。

既存のアプリを使う選択肢もあったが、せっかくなので **Claude Code を使って自分専用のサブスク管理アプリを作ってみた**。プログラミングの経験はほぼなし、ビルドツールも使わず、HTML1ファイルで完結するアプリを目指した。

この記事では、Claude Codeを使ってどのように開発を進めたか、どんなやり取りをしたか、どんな知見が得られたかをまとめる。

**`subsuku.html` をダブルクリックするだけで動くアプリ。インストール不要。**

### 主な機能

* サブスクの追加・編集・削除
* 月額合計・年間合計の自動計算
* 解約予定日までのカウントダウン（7日以内→オレンジ、3日以内→赤）
* 解約リマインダーと次回決済の通知パネル
* クイック選択（Netflix・Spotify・ChatGPTなど主要サービスをワンタップ入力）
* プラン選択（Netflixのスタンダード/プレミアムなど複数プランに対応）
* 無料期間機能（無料期間中は合計から除外）
* 解約済みフラグ（解約済みはグレーアウト・合計除外）
* データはlocalStorageに自動保存（ブラウザを閉じてもデータが残る）

### 技術スタック

* HTML / CSS / JavaScript（バニラ）
* ビルドツールなし、フレームワークなし
* 外部ライブラリなし
* localStorageでデータ永続化

## 開発の流れ

### 1. 要件をClaude Codeと整理する

最初にやったのは「何を作るか」を Claude と一緒に整理することだった。

```
私：サブスクを管理するアプリを作りたい。
　　月の決済日と、解約する日程を管理したい。

Claude：要件を整理しましょう。
　　　　・支払いサイクルは月額だけ？年額も？
　　　　・合計金額は表示したい？
　　　　・スマホでも使いたい？
　　　　・通知機能は必要？
```

「何となく作って」ではなく、**Claude が足りていない要件を質問で引き出してくれた**。これが最初の気づきだった。

決まった要件：

| 項目 | 内容 |
| --- | --- |
| 登録データ | サービス名・月額料金・決済日・解約予定日 |
| 支払いサイクル | 月額のみ |
| 合計表示 | 月額合計・年間合計 |
| 通知 | アプリ内表示のみ（ブラウザ通知なし） |
| 対応端末 | スマホメイン・PC対応 |
| データ保存 | localStorage |

### 2. まず動くものを作ってもらう

要件が固まったら「作って」と伝えるだけで、Claude がフォルダ作成からHTMLファイルの生成まで一気にやってくれた。

最初に生成されたコードですでに：

* 基本的なCRUD（追加・編集・削除）
* 解約予定日のカウントダウンと色分け
* レスポンシブデザイン

が動いていた。**「まず動かしてみる」が圧倒的に速い。**

### 3. 使いながら機能を追加していく

実際に開いてみると「もっとこうしたい」が出てきた。それをそのまま伝えていくスタイルで開発を進めた。

**追加した機能と、Claude とのやり取りの例：**

#### クイック選択ボタン

```
私：毎回サービス名と料金を手入力するのが面倒。
　　よく使うサービスをボタンで選べるようにしたい。

Claude：追加ボタンを押すとサービスボタンが並ぶUIを実装します。
　　　　タップすると名前と料金が自動入力されます。
```

#### 解約日の自動計算

```
私：解約予定日を毎回計算して入力するのが面倒。

Claude：決済日を入力したら「前日」「1週間前」ボタンを表示して、
　　　　押すと自動で日付を入れる機能にします。
```

#### 無料期間対応

```
私：無料期間中のサービスは合計から除きたい。

Claude：「無料期間終了日」欄を追加して、
　　　　期間中は合計除外・打ち消し線バッジ表示にします。
```

こんな感じで、**「こうしたい」→「実装します」→「F5でリロード」** のサイクルを繰り返すだけで機能が増えていった。

### 4. バグが出ても Claude が直してくれる

開発中に「クイック選択を押しても料金が入らない」というバグが発生した。

```
私：ボタンを押しても料金が自動入力されない。

Claude：原因判明です。
　　　　applySuggest 関数が2箇所に定義されていて、
　　　　古いバージョンが新しいバージョンを上書きしていました。
　　　　削除します。
```

自分でデバッグしたら相当時間がかかっていたはず。**エラーの内容を伝えるだけで原因特定から修正まで完結した。**

## Claude Codeを使って得られた知見

### 知見1：「要件を一緒に詰める」ところから始める

最初から「作って」と言うより、要件整理から一緒にやったほうが精度が上がる。Claude が「こういう場合はどうする？」と聞いてくれるので、自分では気づいていなかった仕様の穴が埋まっていく。

### 知見2：「とにかく動くものを出してもらう」が最速

完璧な仕様を決めてから作るより、まず動くものを出してもらって「ここ違う」「これ追加したい」を繰り返すほうが圧倒的に速い。アジャイル開発を自然とやれる。

### 知見3：「面倒だな」という感覚が機能追加のトリガーになる

使いながら「毎回手入力が面倒」「計算が面倒」という感覚が出てきたら、それがそのまま要件になる。ユーザー視点（自分）と開発視点（Claude）が分離できているので、機能の優先順位が自然と決まる。

### 知見4：技術的な理解がなくてもできるが、少し知っておくと会話がスムーズ

`localStorage` が何なのか、なぜHTMLだけで動くのかを少し理解しておくと、Claude の説明が腑に落ちやすくなる。完全にブラックボックスでも動くが、「なんで？」を1回聞いておくと次のアプリ開発でも活きる。

### 知見5：会話履歴がそのまま開発ドキュメントになる

Claude Code との会話には、「なぜこの機能を追加したか」「どんな問題が起きてどう直したか」が全部残っている。振り返りや記事作成のときにそのまま素材として使えた（この記事もそれで書いた）。
