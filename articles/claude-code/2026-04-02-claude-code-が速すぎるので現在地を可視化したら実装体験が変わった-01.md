# AI が速すぎて、ついていけない

Claude Code に「この機能を実装して」と頼むと、一瞬でソースコードを生成してくれます。

便利になったと思う反面、自分の理解が追いついていない...そんな状態ではレビューもデバッグもできません。

仕様駆動開発などのアプローチもありますが、クリーンアーキテクチャのような複数レイヤーのシステムでは、**テキストベースでは全体像を直感的に把握しづらいと感じていました。**

# シーケンス図で「現在地」を示してみた

そこで思いついたのが、 **シーケンス図にレイヤーごとの実装状態を色で表現する** ことです。

各レイヤーを `box` でグループ化し、実装済みを緑、未実装を黄色にする。1レイヤー終わるたびに色を更新していきます。

![](https://static.zenn.studio/user-upload/4def9d686f2b-20260402.png)

内側のレイヤー（Repository）が緑に変わり、外側（UseCase, Handler）はまだ黄色。「次はここを実装します」とAIと認識を合わせながら実装を進めることで、**現在地が常に分かる**ようになりました。

その結果、認知負荷が下がり、効率的に実装を進められるようになりました。

この体験が良かったので、Claude Code の Skill を作成してみました。

# スキルの構成

ワークフローは 3 Phase です。

SKILL.md

```
---
name: implementation-guide
description: >-
  実装の伴走スキル。計画書に基づく段階的な実装をナビゲートし、
  コンポーネント図・シーケンス図で進捗をリアルタイムに可視化する。
  「伴走して」「一緒に実装」「ステップバイステップで進めて」
  「実装を開始」「計画書に沿って実装」などの指示で発動する。
metadata:
  author: PenPeen
  version: 1.0.0
---

# 実装伴走ガイド

実装の計画から完了まで、ユーザーと並走しながら段階的に進めるスキル。

## コア原則

1. **許可なくファイルを書かない** — 説明 → 承認 → 書き込みの 3 ステップを必ず踏む
2. **説明は抽象度高めから** — 詳細は聞かれたときだけ。デフォルトは「何を、なぜ」の 2 行
3. **今どこにいるかを常に示す** — シーケンス図上の現在位置を伝えてから作業に入る

## ワークフロー

### Phase 0: 準備

1. **計画書の確認**: ユーザーが計画書を渡した場合は Read する。なければ作成を提案する
2. **進捗ファイルの生成**: 以下を含む進捗ファイルを作成する
   - コンポーネント図（全体像）
   - シーケンス図（レイヤー別の処理フロー）
   - ファイル別チェックリスト

### Phase 1: 実装ループ（レイヤーごとに繰り返す）

各レイヤー（ファイル群）に対して以下を回す:

#### Step 1: ナビゲーション

シーケンス図上の現在位置をユーザーに伝える。

「次は [レイヤー名]（シーケンス図の [色] 背景の部分）です。
 [このレイヤーが何をする層か — 1 文で]」

変更するファイルの一覧と、各ファイルの操作（新規 / 追記）を提示する。

#### Step 2: ファイル単位の実装

各ファイルに対して:

1. **説明**（2-3 文）: 何を書くか、なぜ必要か
2. **承認待ち**: ユーザーの OK を待つ
3. **書き込み**: 承認後にファイルを作成/編集

説明時のガイドライン:
- interface と implementation が分かれる場合は「interface = 宣言、implementation = 中身」と簡潔に区別する
- 既存ファイルへの追記の場合は、既存コードとの関係を一言添える
- 既存パターンがある場合は「○○と同じパターン」で済ませる

#### Step 3: 進捗更新

レイヤーの全ファイルが完了したら:

1. シーケンス図の該当レイヤーの色を**黄 → 緑**に更新する
2. チェックリストの該当項目にチェックを入れる
3. 更新後の進捗をユーザーに短く報告する

### Phase 2: 完了

全レイヤーの実装が完了したら:

1. 進捗ファイルを最終更新（全レイヤー緑）
2. 実装サマリを生成:
   - 作成/変更したファイル一覧
   - テスト結果（実行した場合）
3. ビルド・テストの実行を提案する

## 進捗ファイルの色規約

ダークモード・ライトモード両対応の色を使用する。

| 状態 | シーケンス図の box 色 | 意味 |
|------|---------------------|------|
| 実装済み | `rgb(30, 100, 50)` — 暗めの緑 | このレイヤーは完了 |
| 未実装 | `rgb(120, 90, 20)` — 暗めの黄 | これから実装する |
```

### **Phase 1（準備）**:

計画書を読み込み、コンポーネント図 + シーケンス図 + チェックリストを含む進捗ファイルを生成する。

### **Phase 2（実装ループ）**:

レイヤーごとに以下を繰り返します。

1. **ファイル単位の実装** — 説明 → 承認 → 書き込みの 3 ステップ

![](https://static.zenn.studio/user-upload/a95a507061a1-20260402.png)

2. **進捗更新** — シーケンス図の色を黄→緑に更新

![](https://static.zenn.studio/user-upload/1fb6d764f628-20260402.png)

### **Phase 3（完了）**:

全レイヤーが緑になったら、実装サマリを生成して終了です。

![](https://static.zenn.studio/user-upload/0e1c3120ae12-20260402.png)
