KARPATHY'S ONE-NOTE METHOD GETS MUCH MORE POWERFUL WHEN YOU ADD A MAINTENANCE LOOP.

The original idea was simple: append every thought to one running note, then periodically scroll through it and pull useful ideas back to the top.

Great for capturing.

Terrible for manually processing five years of accumulated thinking.

Claude Code changes the second half.

The setup is three layers:

1. An append-only stream for raw thoughts, links, and half-formed ideas.

2. An Obsidian vault of plain Markdown files that hold durable notes, projects, and past decisions.

3. A CLAUDE.md file that tells the agent how the vault works and what it is allowed to change.

Then give the agent a repeatable job:

Read today's appends.

Find the relevant old notes.

Extract decisions and action items.

Draft possible links.

Flag contradictions.

Leave a short review report.

The key is not letting an LLM freely rewrite your memory.

It is giving it a clear inbox, explicit rules, and a reviewable output.

Your messy note is no longer the destination.

It becomes the input stream for a personal knowledge system that improves every time you think.
