新しいリポジトリを開くと、最初の問いはほぼ決まっています。

「どこから読めばいい?」

ファイルツリーを眺めて `src/` を開く。`index.ts` に import が20行。`auth/` を開いても import が20行。30分後もエントリーポイントを探している。

**[Understand Anything](https://github.com/Lum1104/Understand-Anything)**（MIT）はこの問題に直接答えるツールです。

---

## 使い始め方

Claude Code か Cursor を使っているなら、数コマンドで動きます。

**Claude Code:**

```
/plugin marketplace add Lum1104/Understand-Anything
/plugin install understand-anything
```

**Cursor:**  
[Understand-Anything リポジトリ](https://github.com/Lum1104/Understand-Anything) を clone して Cursor で開くと、`.cursor-plugin/plugin.json` を自動検出してプラグインが有効になります。

**その他のプラットフォーム:**  
Codex・OpenCode・OpenClaw・Antigravity にも対応しています。各プラットフォームのインストール手順は [README](https://github.com/Lum1104/Understand-Anything) をご覧ください。

---

## コマンド一発、コードベースがグラフになる

5つの専門エージェントが並列で走り、ファイル検出・関数抽出・アーキテクチャ分類・学習順序の決定をこなします。解析結果は `.understand-anything/knowledge-graph.json` に保存され、次回以降は変更されたファイルだけ再解析します。終わるとブラウザが開きます。

各アーキテクチャレイヤーに色が自動割り当てされたグラフが描かれています。ノードをクリックすると、そのファイルや関数の役割が自然言語で出ます。「この関数は認証トークンの検証を担当します」という具合にです。

![skills CLI のナレッジグラフ — 192ノード、351エッジ、7層](https://static.zenn.studio/user-upload/22e6cb0e1f61-20260322.png)

30分かかっていた「ここか」という確信が、30秒で得られます。

## 実在の TypeScript プロジェクトで試した結果

エージェントスキルを 40 以上の AI コーディングエージェントへ配布するパッケージマネージャーである vercel-labs の [`skills` CLI](https://github.com/vercel-labs/skills) を `/understand` にかけてみました。Understand Anything 自体もスキルとして配布される側なので、多少メタな実験ですね。

55ファイル、**192ノード、351エッジ、8ステップのガイドツアー**が生成されました。アーキテクチャは7層に分類されています。

生成されたガイドツアー（8ステップ）

1. `src/cli.ts` — コマンドディスパッチャー。ここを読むだけでコマンドセット全体が把握できる
2. `src/add.ts` — `skills add` の実装。fan-out が最大で、最も多くのモジュールを呼び出す司令塔
3. `src/installer.ts` — 942行の実際のインストールエンジン。シンボリックリンク作成、パストラバーサル防止、エージェントごとの配置先決定まで
4. `src/agents.ts` — 40+ エージェントのレジストリ。コードベース全体で fan-in が最大、あらゆるコマンドが依存する中枢
5. `src/providers/` — スキルの取得元を抽象化するプロバイダーシステム。GitHub, GitLab, ローカルパスを統一インターフェースで扱う
6. `src/skills.ts` / `src/source-parser.ts` / `src/git.ts` — スキル解決・構文解析・Git オペレーションのコアサービス群
7. `src/skill-lock.ts` / `src/local-lock.ts` — インストール状態の永続化とロックファイル管理
8. テスト戦略 — `src/` 内のユニットテストと `tests/` の結合テストで 26 ファイル、19本の tested\_by エッジ

インストールエンジンへのエッジを確認すると、呼び出し元は `add` / `sync` / `remove` / `list` の4コマンドです。`install` は `add` 経由のため、エッジには直接現れません。

![agents.ts を選択した状態 — Connections(23) が右パネルに列挙される](https://static.zenn.studio/user-upload/068b2f2b3e69-20260322.png)

`src/agents.ts` の Connections=23。inbound のエッジを数えると、あらゆるコマンドがここに依存していることが一目でわかります。コードを読む前に構造が確認できます。

## コードベースに「質問」できる

ナレッジグラフが手に入ると、自然言語で質問できます。

```
/understand-chat 支払いフローはどのように機能しますか？
```

「認証はどこで処理されてる?」「このエラーはどのモジュールから来てる?」  
grep の代わりに聞けばいい。グラフを参照した答えが返ります。

他にも使えるコマンドがあります。

```
# 変更の影響範囲を解析
/understand-diff

# 特定ファイル・関数を深堀り
/understand-explain src/auth/login.ts

# 新メンバー向けオンボーディングガイドを生成
/understand-onboard
```

`/understand-diff` は特に実用的です。PR を出す前に「このファイルを変えると、どこまで波及するか」をグラフで確認できます。レビューで「ここも影響するのでは?」と指摘される前に把握できる。

## なぜこれが必要か

コードが変わるたびにドキュメントを更新する人はほぼいません。[Understand Anything](https://github.com/Lum1104/Understand-Anything) はコードから直接グラフを生成するので、常に現在の状態を反映します。

入社初日でも引き継ぎでも、「どこから読めばいい?」に30秒で答えが出ます。

## 参考: 技術的な仕組みが気になる方へ

[Understand Anything](https://github.com/Lum1104/Understand-Anything) は、5つのエージェントがパイプラインで協調します。

コアは TypeScript + web-tree-sitter で AST を解析しています。ダッシュボードは React 18 + React Flow + Dagre で描画。検索は Fuse.js によるファジー検索に加え、意味検索にも対応しています。
