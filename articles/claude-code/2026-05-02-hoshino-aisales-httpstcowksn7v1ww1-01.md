## 第1章：まず整理｜Claudeの「4つの顔」を区別する

Claudeは1つの製品ではありません。

正直、僕も最初はぐちゃぐちゃでした。

「Claude」「Claude Code」「Claude Cowork」、名前が似すぎていてXで見るたび混乱していました。

整理するとこうなります。

Claude.ai → ブラウザで使うチャット版。スキル機能・プロジェクト機能はここ

Claude Cowork → デスクトップアプリで動くAIエージェント。ファイル操作や定期実行を担当

Claude Code → ターミナルで動く開発者向けツール。プログラミングが必要

コネクタ → 外部サービス連携の仕組み。Claude.ai・Cowork全てから呼び出せる

非エンジニアの結論を先に言います。

Claude.ai＋Cowork＋コネクタの3つで、業務自動化はほぼ完結します。

Claude Codeは不要です。

僕も一時期Claude Codeに挑戦しましたが、ターミナルでフォルダを移動する操作でつまずきました。

「これ、非エンジニアには無理じゃないか？」が正直な感想でした。

でもClaude.aiとCoworkを使い始めてから、自動化の景色が一気に変わったんです。

ここから紹介する7つの仕組みは、全部Claude Codeなしで動きます。

## 第2章：仕組み①｜スキル機能で「毎回同じプロンプト」を終わらせる

スキル機能は2025年10月にリリースされた、業務手順の再利用パッケージです。

正式名称は「Agent Skills」。

オープンスタンダードとして公開されており、Claude.ai・Cowork・Code・APIの全てで動きます。

無料プランでも2026年1月から開放されているので、誰でも今日から使えます。

例えば「週報スキル」を1つ作っておけば、次から「週報作って」の一言で起動します。

毎週金曜30分かけていた週報作成が、5分で終わるようになりました。

年間で換算すると約26時間、ほぼ丸3日分の時間が浮きます。

スキルの作り方はめちゃくちゃ簡単です。

カスタマイズで＋をオンにして、Claudeに作りたいスキルを伝えるだけ。

プロンプト例

```
週報自動作成スキルを作りたいです。

条件： ・入力：今週やったことを箇条書きで伝える ・出力：実績／来週の計画／課題と対策／上長への相談事項 ・トーン：丁寧語、具体的な数字を必ず入れる ・分量：500〜800文字
トリガーキーワード：「週報作って」「レポート」
```

## 第3章：仕組み②｜プロジェクト機能で「自分専用ワークスペース」を作る

スキル機能と並ぶClaude.aiの主力機能が、プロジェクト機能です。

スキルとプロジェクトは似ているようで役割が違います。

スキル → 必要なときだけ呼び出される「業務手順」

プロジェクト → そのワークスペース内で常に読み込まれる「背景知識」

最大200Kトークンのナレッジを保持できます。

僕は今、X記事執筆用のプロジェクトに自分の文体ルール、過去記事、職務経歴を全部入れています。

ベンチャー企業で15年間働いていた時代の経験を元に、自分の文脈をClaudeが完全に理解した状態で文章を生成してくれます。

「あなたは誰で、何をしていて、どういうスタイルで書くか」を毎回説明する手間がゼロになりました。

プロンプト例です。プロジェクト立ち上げ直後に投げるテンプレです。

```

確認項目： ・各ファイルの役割 ・ファイル間の優先順位（どれを一番参照したいか） ・このプロジェクトで取り組むべき最初のタスク3つ。
```
