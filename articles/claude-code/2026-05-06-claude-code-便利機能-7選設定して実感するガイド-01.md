Claude Code は **VSCode 拡張機能・ブラウザ（claude.ai/code）・デスクトップアプリ** のどれからでも使えます。  
  
この記事で紹介する設定は、チャット欄にコマンドを入力するか、設定ファイルを作成するだけで完結します。ターミナルの知識は不要です。導入前後の変化を全機能に掲載しているので、読みながらそのまま実機で試してみてください。

## 1.CLAUDE.md — プロジェクトの「記憶」を作る

毎回「npm じゃなく pnpm を使って」「インデントは2スペース」と言い直す必要がなくなります。

**CLAUDE.md とは**

プロジェクトのルートに置く Markdown ファイルです。Claude Code を開くたびに自動で読み込まれ、書いたルール・コマンド・注意事項がそのセッションに最初から反映されます。

![](https://assets.st-note.com/img/1778026790-V76CsgxKN8vlMtbfc5Ww9eLG.png?width=1200)

**設定手順**

1. チャット欄に/initと入力して送信する  
   → Claude がコードを自動解析し、CLAUDE.md の雛形を提案してくれます
2. 提案内容をチャット上で確認し「OK」または「〇〇を追加して」と返信する  
   → プロジェクトルートに CLAUDE.md が自動作成されます
3. VSCode の場合：エクスプローラーに表示された CLAUDE.md をクリックして直接編集できます

![](https://assets.st-note.com/img/1778026867-qgJUp2DZrwin5uF3eScAsj7R.png?width=1200)

### 書く場所と用途

![](https://assets.st-note.com/img/1778026966-1NVhoH6mzytkeP0jROBJ4r3g.png?width=1200)

### 記述例（コピペして使えます）

```
# プロジェクト規約

## よく使うコマンド
- パッケージ管理: pnpm（npm は使わない）
- テスト実行: pnpm test
- ビルド: pnpm build
- リント: pnpm lint

## コード規約
- インデント: 2スペース
- 言語: TypeScript（any 禁止）
- コメント: 日本語で書く
- API ルート: src/api/handlers/ 以下に配置

## 注意事項
- .env ファイルは絶対に編集しないこと
- package-lock.json は触らない
```

**ポイント**

200行以内に収めると効果が高いです。「具体的なコマンド名」を書くのがコツ。「きれいにフォーマットして」ではなく「インデントは2スペース」のように書きましょう。

**対応環境：🔵 VSCode🚫 ブラウザ（非対応）🟣 デスクトップ**

## 2.Hooks — 編集後の自動フォーマット＆通知

Claude がファイルを編集するたびに自動でコマンドを実行できます。フォーマット・通知・ファイル保護の3つが特に便利。

**設定ファイルの場所**

VSCode・デスクトップアプリの場合、プロジェクト内に .claude/settings.json というファイルを作成します。  
VSCode ではエクスプローラーから新規ファイルとして作成できます：  
プロジェクトフォルダ右クリック → **「新しいフォルダ」→ .claude** → その中に **「新しいファイル」→ settings.json**

**Hooks とは**

Claude がツールを使うタイミングに合わせて**コマンドを自動実行する仕組み**です。設定ファイルに JSON で書くだけで動きます。

![](https://assets.st-note.com/img/1778027079-UTSrJZ6ontXcxNQdCGL1R4qy.png?width=1200)

**実用例① — ファイル編集後に Prettier を自動実行（Mac / Linux）**

![](https://assets.st-note.com/img/1778027130-6AgyE5DzYax3mrK0PkcXuqMV.png?width=1200)

.claude/settings.json に以下を記述：

```
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "cat | jq -r '.tool_input.file_path' | xargs npx prettier --write 2>/dev/null || true"
          }
        ]
      }
    ]
  }
}
```

**実用例② — Claude が入力待ちになったらデスクトップ通知**

![](https://assets.st-note.com/img/1778027168-eU7CdX5zvt3WbwL0kEBgcTpV.png?width=1200)

Mac の場合：

```
{
  "hooks": {
    "Notification": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "osascript -e 'display notification \"返答待ちです\" with title \"Claude Code\"'"
          }
        ]
      }
    ]
  }
}
```

Windows の場合：

```
{
  "hooks": {
    "Notification": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "powershell -Command \"Add-Type -AssemblyName System.Windows.Forms; [System.Windows.Forms.MessageBox]::Show('返答待ちです','Claude Code')\""
          }
        ]
      }
    ]
  }
}
```

**実用例③ — .env ファイルへの書き込みをブロック（Mac / Linux）**

```
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "INPUT=$(cat); FILE=$(echo \"$INPUT\" | jq -r '.tool_input.file_path // empty'); if [[ \"$FILE\" == *.env* ]]; then echo \"保護されたファイルです\" >&2; exit 2; fi"
          }
        ]
      }
    ]
  }
}
```

**実際の効果**

Claude が .env ファイルを書き換えようとすると自動でキャンセルされ、チャット上に「保護されたファイルです」と表示されます。秘密情報の誤書き換えリスクがゼロになります。

**対応環境：🔵 VSCode🟢 ブラウザ🟣 デスクトップ**

## 3.カスタムスラッシュコマンド — よく使う手順を一発呼び出し

「差分を要約して」「コミットメッセージを作って」など繰り返す指示を /コマンド名 で実行できます。

ファイルの作成場所

![](https://assets.st-note.com/img/1778027242-V7zYyDMLBI0SxZmnhUQ4Wlc1.png?width=1200)

**VSCode での作り方**

エクスプローラーで **.claude フォルダ** を右クリック →「新しいフォルダ」で **commands** フォルダを作成 → その中に **コマンド名.md** ファイルを新規作成します。

**実用例① — コミットメッセージ自動生成**

ファイル名：.claude/commands/commit.md

```
---
description: ステージング済みの変更からコミットメッセージを生成する
---

## 変更内容

!`git diff --staged`

## タスク

上記の差分をもとに、以下の形式でコミットメッセージを提案してください：

- 1行目: feat / fix / chore / docs + 簡潔な概要（50文字以内）
- 3行目以降: 変更の背景・理由（必要な場合のみ）
- 日本語で書く
```

![](https://assets.st-note.com/img/1778027292-vN67iPJR8pjeAXh4gGbWIuaB.png?width=1200)

![](https://assets.st-note.com/img/1778027323-WQJfoDGR05gzVL3HyvkZETAI.png?width=1200)

**実用例② — PR の説明文ドラフト生成**

ファイル名：.claude/commands/pr-desc.md

```
---
description: PR の説明文（日本語）をドラフトする
---

## 変更差分

!`git diff main...HEAD`

## コミット一覧

!`git log main...HEAD --oneline`

## タスク

GitHub PR の説明文を日本語でドラフトしてください。
構成: 変更概要 / 変更理由 / テスト方法 / スクリーンショット欄
```

**ポイント — !`コマンド` の記法**

!`git diff` のように書くと、スラッシュコマンドを実行したときにシェルコマンドの出力が自動で埋め込まれます。毎回コピペ不要で最新の情報が反映されます。

**対応環境：🔵 VSCode🟢 ブラウザ🟣 デスクトップ**

## 4.パーミッションモード — 確認ダイアログを減らす

Shift + Tab を押すだけで切り替えられます。作業内容に合わせてモードを変えると承認の手間が激減します。

**モード一覧**

![](https://assets.st-note.com/img/1778027421-3yMmdqbA52JIPrnekhVtETcz.png?width=1200)

### 切り替え方法

1. Claude Code のチャット画面が開いている状態でShift+Tabを押す
2. 画面上部または入力欄の付近にモード名が表示されて切り替わる（循環式）
3. 作業に合わせてその都度切り替えてOK

![](https://assets.st-note.com/img/1778027540-fKntQiNzRoYlJqF74HL5vgr9.png?width=1200)

### デフォルトモードを固定する（設定ファイルで変更）

毎回切り替えるのが面倒な場合は、.claude/settings.json に書いておくと起動時から指定モードになります：

```
{
  "permissions": {
    "defaultMode": "acceptEdits"
  }
}
```

**注意**

auto モードは Claude が自動実行するため、重要なリポジトリでは慎重に使いましょう。初めてのプロジェクトは default から始めるのがおすすめです。

**対応環境：🔵 VSCode🟢 ブラウザ🟣 デスクトップ**

## 5.メモリ機能 — Claude に習慣を覚えさせる

「次から〇〇で書いて」と一度言うと、Claude が自動でメモを取り、次のセッションでも守ってくれます。

![](https://assets.st-note.com/img/1778027572-f6VDptk8eI5AlZ7i09SCRnEq.png?width=1200)

### 使い方

1. チャットで「次から〇〇にして」「〇〇は覚えておいて」と伝える
2. Claude がメモリファイルに保存（保存しましたと表示される）
3. 次回のセッションから自動で反映される

![](https://assets.st-note.com/img/1778027598-h2ZIEWS0XadMcC6x1mUqAr5V.png?width=1200)

### 保存されたメモリを確認・編集する

1. チャット欄に/memoryと入力して送信
2. 保存されているメモリの一覧が表示される
3. 不要なものは「〇〇の記憶を削除して」とチャットで伝えるか、表示されたファイルを直接編集

**活用例**

「このプロジェクトのAPIキーは .env.local に書く」「テストファイルは \_\_tests\_\_ フォルダに置く」など、プロジェクト固有のルールを覚えさせると特に効果的です。

**対応環境：🔵 VSCode🟢 ブラウザ🟣 デスクトップ**

## 6. /compact — コンテキスト切れを防ぐ

長時間作業していると「コンテキストが満杯です」と表示されます。/compact を使えばセッションを維持したまま続けられます。

**コンテキストウィンドウとは**

Claude が一度のセッションで覚えられる会話量の上限です。大きなプロジェクトや長時間の作業でこの上限に近づくと、パフォーマンスが落ちたり途中で止まったりします。

/compact は、これまでの会話を**重要な情報だけに要約・圧縮**して、続きから作業できるようにするコマンドです。

![](https://assets.st-note.com/img/1778027683-L9EB3vlgU8wMi71YZyNWk4fm.png?width=1200)

**使い方**

1. チャット欄に/compactと入力して送信
2. Claude がこれまでの会話を要約して圧縮する（数秒で完了）
3. 「〇〇について作業中でした」という要約が表示され、そのまま続けられる

![](https://assets.st-note.com/img/1778027710-V0TPvbFCDdIoWcaArN2Lj594.png?width=1200)

**ポイント**

コンテキスト警告が出る前に /compact を使うのが効果的です。また、CLAUDE.md に書いたルールは圧縮後も読み込まれるため、大事な指示はここに書いておきましょう。

**対応環境：🔵 VSCode🟢 ブラウザ🟣 デスクトップ**

![](https://assets.st-note.com/img/1778027745-8EwVOvT2lmeuKBzNLkSWgx35.png?width=1200)

### その他の便利ショートカット

![](https://assets.st-note.com/img/1778027767-c03ORbNQ2Feoh5JrLq8DHZYg.png?width=1200)

### ショートカットをカスタマイズする（VSCode・デスクトップ）

**設定ファイルの場所**

ホームフォルダ内の .claude/keybindings.json を作成します。  
Windowsの場合：C:\Users\ユーザー名\.claude\keybindings.json  
Macの場合：/Users/ユーザー名/.claude/keybindings.json

```
{
  "bindings": [
    {
      "context": "Chat",
      "bindings": {
        "ctrl+e": "chat:externalEditor",  // Ctrl+E で外部エディタ起動
        "ctrl+shift+s": "chat:submit"       // Ctrl+Shift+S で送信
      }
    }
  ]
}
```

**ポイント**

ファイルを保存するだけで即反映されます。Claude Code の再起動は不要です。
