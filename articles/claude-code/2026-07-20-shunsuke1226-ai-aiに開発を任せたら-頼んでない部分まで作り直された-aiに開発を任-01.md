『AIに開発を任せたら、
頼んでない部分まで作り直された』

AIに開発を任せて起きるこれ、原因はプロンプトの書き方ではありません。作る前に「何を作るか」が決まっていないだけです。

GitHubが出した Spec Kit という無料ツールは、そこを機械的に潰してくれます。

いきなりコードを書かせない。先にAI自身に仕様書を書かせます。

1. 何を作りたいのかを整理させる
2. 決まっていない部分を先に質問させる
3. 設計と作業の順番を並べさせる
4. そこでようやく着手

GitHub公式のオープンソースで、12万スター。
Claude Code でもそのまま動きます。

プロの開発者が着工前に仕様を固めるのと同じ手順を、ツール側が強制してくれる形ですね。

非エンジニアほど効きます。曖昧なまま走らせて手戻りする時間が、いちばん高くつくので。

まずは /specify だけ試すのがおすすめです。
