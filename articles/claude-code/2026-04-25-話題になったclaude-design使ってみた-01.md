# はじめに

Claude Designへのアクセスは以下でアクセスできます。  
Claude Pro、Max、Team、Enterpriseプランのいずれかに加入していれば利用可能です。  
<https://claude.ai/design/>

思えば初投稿になります「いむにだ」です。  
これから色々書ける時書こうかなと思ってます（時間あれば...）。

## 今回やったこと

現在個人開発でナイトワーク向けの収支管理アプリをバイブコーディングで作っているので、アプリを紹介するLPも作らないとなーと思っていた矢先に、**Claude Design**が発表されたので試しに使ってみました。

### プロジェクトの作成

まずはプロジェクトを作成していきます。  
自分の場合は骨組みから決めていき、後からデザインを仕上げたかったのと、既存のテンプレートなどはまだ持ってないので今回は**PrototypeのWireFrame**を選択します。  
![はじめ](https://static.zenn.studio/user-upload/5294869c7aa8-20260419.png)

### ※プロジェクト作成の違いについて

perplexityに聞いた内容だと以下になります。  
最初から骨組みなしでLPを作りたい場合はHigh-fidelityを選択すれば良さそうですね。

* **Prototype**: ワイヤーフレームやモックアップ向けです。アプリ画面、ダッシュボード、SaaSのUIなど、インタラクティブな画面設計に向いています。
  + **Wireframe（ローファイ）**: 画面のレイアウトや基本構造、ユーザー操作の流れを素早く確認するモードです。 線やボックスで要素を表現し、色や画像は最小限なので、アイデア検証やチームレビューに適しています。
  + **High-fidelity（ハイファイ）**:最終製品に近いビジュアルとインタラクションを作れるモードです。 本物の色、フォント、画像、アニメーションを入れて、ユーザー操作感や細部までテストできます。
* **Slide deck**: ピッチ資料、提案書、社内発表、説明会資料などのスライド作成向けです。発表用に整理されたページ構成を作りやすいです。
* **From template**: 既存のテンプレートを使って作り始めるモードです。デザインの土台が決まっているので、ブランドや既定レイアウトに合わせたいときに便利です。
* **Other**: 上の3つに当てはまらないもの、たとえばランディングページ、1ページもの、インフォグラフィック、マーケ素材などを作る入口です。

これを見る限り**Other**を選択しても良かったのですが、何も決めず進めるとどんなページ構成とデザインのLPになってしまうんだ...？と思ったので選択しませんでした。

### ワイヤーフレーム作成の指示

まだざっくりの想像でしかページ構成を考えてなかったので、一旦チャット投げてみてから決めようとしたのでmdファイルを添付し指示してみます。  
mdファイルの中身は以下のように書いています。

* アプリ名
* 誰向けか
* アプリの主な機能
* 作成しているアプリのデザインコンセプト

### High-fidelityで仕上げる

骨組みは出来上がったので、最終段階へ仕上げていきます。  
チャットの左下にある**import**から「Skills and design systems」を選択する。  
以下項目の内容です。

* Animated video: タイムラインや動きのある映像デザイン向け。
* Interactive prototype: 実際に操作できるアプリや画面の試作向け。
* Make a deck: スライド資料の作成向け。
* Make tweakable: 後から調整しやすい、編集しやすい構成にする。
* Frontend design: 既存のブランドに寄せすぎず、見た目重視でフロントエンドを作る用途。
* Wireframe: 画面の骨組みを素早く作る用途。
* Export as PPTX (editable): PowerPointで編集できる形式で出力。
* Export as PPTX (screenshots): 画像ベースのPPTXで、見た目重視ですが編集性は低い。
* Save as PDF: 印刷や共有向けのPDF出力。
* Save as standalone HTML: 1つのHTMLファイルとして保存し、単体で動かしやすくする。
* Send to Canva: Canvaで編集できる形で渡す。

今回は「Make tweakable・Frontend design」を選択。  
![](https://static.zenn.studio/user-upload/0d87621243d3-20260419.png)

### チャット

以下はワイヤーフレームからHigh-fidelityで仕上げる時のテンプレ？みたいです。

```
Wireframeをもとに、High-fidelityで仕上げてください。
見た目は完成版に近い品質で、色、余白、フォント、写真、ボタン、影、質感まで整えてください。
```

自分は少し変えた内容でチャットを投げます。

### この後

一旦ある程度のものは出来上がったけど、やはり修正点は多い（Liquid Glass UIとかのアピールいらんでしょ）のでzipでダウンロードして、残りはClaude codeや、自分で細かい点を修正することにした。

### 感想

一通り使ってみましたが、使った感じデザイン仕様をもう少し細かく決めるとか、Hi-fiで作る際に細かく指定してあげるとLPデザインの完成度も高くなるんだろうなと思った（あとアップロードした画像も足りなかった今回）。  
ただ何もない状態から**Claude code**でDESIGN.mdを使いclaude.mdなどを使用しテキストで指定して作るのと、**Claude Design**でワイヤーフレームから作り、そこからさらに仕上げて作るのとでは最終的な完成度がまた違う。  
実際**Claude Design**で作られたものを、後から**Claude code**でルールや細かい部分などを決めて修正すれば完成度は高まるので非Webデザイナーにとっては便利だな~と感じた。  
最近は「Codex GPT5.5 × GPT Image2」の組み合わせも凄そうだったのでそれも触ってみて色々模索してみようと思います！
