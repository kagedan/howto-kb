**残り11日。あなたの自動化、このまま動かし続けて大丈夫ですか？**

Anthropicが2026年6月15日に実施するClaudeのプラン変更が、開発者・AI活用者の間で話題になっている。変更の核心は「自動化処理の使用量をサブスクリプションから切り離す」というもの。  
普通にターミナルで会話しながら使う分には影響ゼロ。でも私のように、定期自動実行やAgent SDKで複数のClaudeを毎日回している場合は、話が大きく変わってくる。

この記事では、**実際に複数エージェントを業務自動化に使っている現場**から、変更内容の要点と実測コスト試算（一次情報）、そして6月15日前にやるべき対策5選をまとめた。

---

## 何が変わるのか（要点3行）

「インタラクティブ利用（ターミナルで直接会話しながら使うClaude Code）」は今まで通りサブスクリプション枠内。影響があるのは、**cronやLaunchd（UNIX系OSの定期実行エンジン）などで自動実行している処理だけ**だ。

※Anthropic公式ヘルプより：6月8日にクレジット有効化メールが届く予定。届いたら必ず有効化すること。

  

---

## 私の実際の運用状況（一次情報Type A）

イザークコンサルティングでは、MacのLaunchd（定期実行システム）を使って以下の自動化タスクを毎日実行している。

* CrowdWorks提案文生成：日2回 × 最大25件 → 約150,000トークン/日
* ブログ記事下書き（TechBoost/note）：日1回 → 約80,000トークン/日
* SNS投稿スケジュール生成：日1回 → 約30,000トークン/日
* issue自動解決ループ：日2回 → 約100,000トークン/日
* **合計：約360,000トークン/日**

月30日換算で**月間約1,080万トークン**の自動化処理になる。

Anthropicの公式情報によれば、Max 5xの月額クレジット$100は、Sonnet 4.6のoutput換算で約3〜4百万トークン分に相当する。私の使い方だと**約2〜3倍の超過**が発生しうる計算だ。追加請求が発生した場合、月$150〜$200のコスト増になりうる。

AIでの業務自動化を構築するなら、最初から適切なプロンプト設計を意識しておくことが重要だ。私のエージェント設計を支えているプロンプトの設計指針は、BOOTHの「AI開発プロンプト完全パック」にまとめてある。

## 6月15日前にやるべき5つのアクション

### アクション1：自動化タスクのトークン消費量を計測する（今週中）

まず現状把握から。Claudeのターミナル利用ログは自動では残らないため、呼び出しをラップする計測スクリプトを仕込む。

```
#!/bin/bash
# scripts/lib/token-meter.sh
LOG_FILE="$HOME/.claude/token-usage.jsonl"
RESPONSE=$(claude -p "$@" --output-format json 2>&1)
echo "$RESPONSE" | jq -c '{ts: now|todate, input: .usage.input_tokens, output: .usage.output_tokens}' >> "$LOG_FILE"
echo "$RESPONSE" | jq -r '.result'
```

1週間回すだけで、どのタスクが最もトークンを消費しているかが見えてくる。

### アクション2：軽量タスクのモデルを格下げする（来週・一次情報Type B）

すべての処理に高性能モデル（Sonnet 4.6）を使う必要はない。実際に試した結果を公開する。

* SNS投稿の下書き：Sonnet 4.6 → Haiku 4.5 に変更 → **コスト削減率 約95%**
* 提案文の初稿作成：Sonnet 4.6 → Haiku 4.5 に変更 → **コスト削減率 約95%**
* 提案文の最終品質チェック：Sonnet 4.6 を維持（品質が収益直結のため）
* ブログ記事執筆：Sonnet 4.6 を維持（深い思考が必要なため）

「最終的な品質判断」だけ上位モデルを使い、「量をこなす前処理」を軽量モデルに任せる設計で、全体コストを60〜70%削減できた。

大量の処理を効率化するには、業務フロー別にカスタマイズされたプロンプトが鍵になる。61本の業務効率化プロンプトをまとめたパックで、まず「どのタスクを軽量化できるか」を棚卸しすると早い。

### アクション3：バッチ実行を日次1回に集約する（6/10まで）

細かい頻度で動かしていた処理を日次1回のまとめ実行に変えると、コンテキスト再読み込みなどのオーバーヘッドが減り、トークン消費が下がる。CrowdWorks提案文生成を日2回から日1回に統合しただけで、トークン消費が約15%減少した。

GitHub Actionsを使っている場合は、トリガーをpush毎から日次スケジュールに変更するだけで効果が出る。CI/CDワークフローのテンプレートはBOOTHにまとめてある。

### アクション4：プロンプトキャッシングを設定する（6/12まで）

Claude APIには、同じシステムプロンプト（ルール定義やガイドライン文書）を何度も送る処理を自動的に省略する「プロンプトキャッシング」機能がある。claude -p ではなくAPI直接呼び出しに切り替え、リクエストに特定のフラグを付けるだけで有効になる。毎回長いルール文書を読み込ませるような処理では、入力トークンを最大90%削減できる。

### アクション5：6月8日のメールを見逃さない

Anthropicは6月8日に、プログラマティック枠のクレジットを「有効化」するためのメールを送ると発表している（Anthropic公式ヘルプ参照）。このメールが来たら即座に有効化しよう。有効化しないまま6月15日を迎えると、$0のクレジット上限から即座に従量課金に切り替わる可能性がある。
