前回の「Claude Code 完全ガイド」で主要なスラッシュコマンドを紹介しましたが、実は**重要なコマンドが4つ抜けていました**。この記事では補完します。

## 抜けていた4つのコマンド

1. `/model` - モデル切り替え
2. `/fast` - 高速モード
3. `/plan` - プランニングモード
4. `/bug` - バグ報告

それぞれ詳しく解説します。

---

## `/model` - モデル切り替え

**概要:** 使用するClaudeモデルを切り替えます。タスクに応じて最適なモデルを選択可能。

### 使用例

```
User: /model
Claude: 現在のモデル: claude-sonnet-4-20250514

利用可能なモデル:
1. claude-sonnet-4-20250514 (現在)
2. claude-opus-4-20250514
3. claude-haiku

切り替えるモデルを選択:
```

### 各モデルの特徴

| モデル | 特徴 | 向いているタスク |
| --- | --- | --- |
| **Sonnet** | バランス型 | 通常の開発、コード生成 |
| **Opus** | 最高性能 | 複雑な設計、難しい問題 |
| **Haiku** | 高速・低コスト | 単純なタスク、素早い確認 |

### 活用シーン

```
# 複雑なリファクタリングはOpusで
/model opus
> このレガシーコードをモダンなアーキテクチャに書き換えて

# 単純な修正はHaikuで（コスト削減）
/model haiku
> タイプミスを修正して
```

### コツ

* **Opus**は本当に難しい問題のために取っておく（コストが高い）
* **Haiku**はファイル読み込みや簡単な質問に最適
* セッションごとに切り替えてコスト最適化

---

## `/fast` - 高速モード

**概要:** 同じモデルでより高速な応答を実現。ストリーミングが高速化されます。

### 使用例

```
User: /fast
Claude: 高速モードを有効にしました。
- 応答速度が向上します
- モデルは変更されません
```

### 仕組み

* モデルを変えずに最適化
* 内部処理の調整により高速化
* 品質は維持

### 活用シーン

* すぐに結果が欲しい時
* 長いセッションで待ち時間を減らしたい時
* 対話的な開発フロー

---

## `/plan` - プランニングモード

**概要:** 実装前に詳細な計画を立てるモード。大きな変更前に設計を確認できます。

### 使用例

```
User: /plan
Claude: プランニングモードに入りました。
実装前に計画を提示します。実際のコード変更は行いません。

> 認証システムを実装して

Claude: ## 実装プラン

### フェーズ1: 基盤構築
1. Userモデルの作成
2. パスワードハッシュ化の実装
3. セッション管理の設定

### フェーズ2: エンドポイント
1. POST /auth/register
2. POST /auth/login
3. POST /auth/logout

### フェーズ3: ミドルウェア
1. 認証チェックミドルウェア
2. ロールベースアクセス制御

この計画で進めてよろしいですか？
```

### 活用シーン

* 大規模な機能追加
* アーキテクチャの変更
* チーム開発での設計共有
* レビュー前に方針確認

### メリット

* 思考停止でコードを書かせない
* 設計の問題を早期発見
* チームとの合意形成

---

## `/bug` - バグ報告

**概要:** Claude Codeのバグや問題をAnthropicに報告します。

### 使用例

```
User: /bug
Claude: バグ報告フォームを開きます。

報告内容を入力してください:
- 何が起きましたか？
- 期待する動作は？
- 再現手順は？

Claude Codeが問題の詳細を収集し、報告します。
```

### 活用シーン

* Claude Code自体の不具合を見つけた時
* 予期しない動作を報告したい時
* 機能リクエスト

## まとめ

見落としがちな4つのコマンド：

1. **`/model`** - タスクに応じたモデル選択でコスト最適化
2. **`/fast`** - 待ち時間を減らしてサクサク開発
3. **`/plan`** - 大きな変更前に設計確認
4. **`/bug`** - 問題報告でClaude Codeを改善

これで本当の「完全版」です！
