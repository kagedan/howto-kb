## 1. 専用の空フォルダを先に作ろう

ホームディレクトリ直下や、既存の大事なプロジェクトの中でいきなり動かし始めるのは避けましょう。作業用の空フォルダを新規に作ってから、そこで起動しましょう。

Claude Codeはファイル操作が直接的です。専用フォルダを用意しないまま動かすと、試行錯誤の途中のうっかりした変更が、大事なファイルを巻き込んで広がります。専用フォルダの外で作業していたばかりに、後片付けに時間を取られることになります。専用フォルダの中に閉じていれば、そもそも被害が広がりません。

手順1: ターミナルに、以下をまとめて貼り付けてEnterを押してください。

```powershell
mkdir ~/projects/my-first-claude && cd ~/projects/my-first-claude && claude
```

フォルダを作り、その中に移動して、Claude Codeを起動するところまでを、1行でまとめて実行しています

注意

フォルダ名は好きな名前に変えて構いません。大事なのは「ここで何が起きても取り返しがつく」と思える場所を選ぶことです。

コマンドを打つのに抵抗があれば、次の手順でも同じことができます。

1. エクスプローラーやFinderで先にフォルダを作る
1. そのフォルダの中で右クリックして「ターミナルで開く」を選ぶ
1. 最後に claude とだけ入力してEnterを押す
> Tips
新しいプロジェクトを始めるたびに、このフォルダ作成から起動までの流れはそのまま使い回せます。一度慣れてしまえば、次からはほとんど考えずに打てるようになります。

> Tips
慣れてきたら、既存プロジェクトの中でも同じ考え方で臨めます。作業前にgitでコミットしておく、ブランチ（作業用の複製）を切っておくといった一手間が、専用フォルダと同じ役割を果たします。

安心して試行錯誤できる場所を、先に用意しておきましょう。最初の数日にありがちな「やらかした」を、この一手間が防いでくれるはずです。
