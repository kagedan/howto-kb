## この記事で分かること

2026年3月24日、AnthropicがClaude CodeとClaude Coworkに「Computer Use」機能を正式追加しました。Xでは**5.9万いいね・1,560万ビュー**を記録し、開発者コミュニティを震撼させた注目機能です。

この記事では、Computer Useの概要、技術的な仕組み、実際のユースケースまで一気に解説します。

## Computer Useとは

**Claude CodeのCLIセッションから、macOS/Windowsのデスクトップを直接操作できる機能**です。

| 操作 | 詳細 |
| --- | --- |
| スクリーンショット | 画面の現在の状態をClaudeが視覚的に認識 |
| マウス操作 | 左クリック、右クリック、ダブルクリック、ドラッグ |
| キーボード入力 | テキスト入力、ショートカットキー実行 |
| アプリ起動 | Finder、ブラウザ、各種GUIアプリを起動・操作 |

従来のClaude Codeが「ターミナルの中の世界」で完結していたのに対し、Computer UseはGUIの世界まで操作範囲を拡張します。

## 動作原理

Computer Useは以下のループで自律的に動作します。

```
1. スクリーンショット撮影（画面認識）
   ↓
2. 次のアクション決定（クリック位置、入力内容をClaudeが判断）
   ↓
3. マウス/キーボード操作を実行
   ↓
4. 結果確認（再度スクリーンショット）
   ↓
5. タスク完了まで1〜4を繰り返し
```

この「観察→判断→実行→確認」のサイクルを高速に回すことで、人間が手動で行う複雑なGUI操作を段階的に自動化しています。

## 実践ユースケース

### 1. 開発→UI確認の一気通貫

```
# Claude Codeでの一連の流れ（イメージ）
> コードを修正して、ブラウザでlocalhost:3000を開いて、
  ログインフォームに「test@example.com」と入力して、
  動作を確認してスクリーンショットを見せて
```

これまで手動で行っていたブラウザ確認が、Claude Codeのセッション内で完結します。

### 2. E2Eテストの即席実行

Cypress/Playwrightの設定なしで、「このフォームをテストして」と指示するだけで実際のブラウザ操作によるテストが可能になります。

### 3. スプレッドシート操作

Excelやスプレッドシートを開いて、データの入力・集計・グラフ作成まで自動化。非エンジニアの業務自動化にも応用できます。

### 4. ドキュメント自動生成

アプリの各画面をスクリーンショットで撮影し、操作マニュアルを自動生成するワークフローも実現可能です。

## 利用条件（2026年4月時点）

| 項目 | 内容 |
| --- | --- |
| 対応プラン | Claude Pro / Claude Max |
| 対応OS | macOS（3/24〜）、Windows（4/3〜） |
| ステータス | Research Preview（正式版ではない） |
| 利用ツール | Claude Code / Claude Cowork |

### セキュリティ上の制約

* **アプリアクセス許可制**: 新規アプリへのアクセス時にユーザー承認が必要
* **ブラウザ操作制限**: ブラウザは「読み取り」ティアで、クリック/タイピングは制限
* **ターミナル保護**: ターミナル/IDEへのキーボード入力はブロック（Bashツール経由で操作）
* **金融操作禁止**: 取引実行、送金、注文はClaudeが自主的に拒否

Anthropicは「コンピューター使用はClaudeのコーディングやテキスト対話と比較するとまだ初期段階」と公式に述べています。

## まとめ

Claude Code Computer Useは、AIエージェントの操作範囲をCLIからGUIへ拡張する画期的な機能です。

* **開発効率**: コード修正→ブラウザ確認→スクショ報告が1セッションで完結
* **自動化範囲**: GUI操作が必要な業務も自動化の対象に
* **個人開発**: 一人で開発・テスト・ドキュメント作成まで回せる生産性

5.9万いいねの反響が示す通り、AIエージェント開発の次のフロンティアがここにあります。
