Claude Codeに動的ワークフローが実装されました。AIが自分でオーケストレーション台本を書き、最大1,000体のサブエージェントを並列で走らせ、互いの成果を検証し合ってから結果を返す仕組みです。数週間かかる規模の移行や監査が数時間で終わるようになります。仕組みと使い方を全部まとめました。

────────────────

▎NEW │ dynamic workflowsとは

5月28日、Opus 4.8と同時にリサーチプレビューとして公開されました。

仕組みはこうです。大きなタスクを頼むと、ClaudeがJavaScriptの「指揮台本」を自分で書きます。その台本を専用の実行エンジンが裏で動かし、サブエージェントを同時に16体、1回の実行で最大1,000体まで並列で走らせます。

その間もチャット画面は普通に使えます。途中経過は台本側が抱えて、あなたに届くのは最終的な答えだけです。

▎POINT │ 何が新しいのか

ポイントは「計画を会話からコードに移した」ことです。

これまでのAIエージェントは、手順も途中結果も全部ひとつの記憶（コンテキスト）に抱えて進めていました。だから長い作業ほど道に迷いやすかったわけです。

動的ワークフローでは、ループや分岐や中間結果を台本が保持します。Claudeの記憶に残るのは最終回答だけなので、何時間・何日かかる仕事でも計画が脱線しません。

しかも台本は毎回タスク専用に書き下ろされます。Agent SDKなどで事前に組む固定型の自動化は汎用的になりがちでしたが、Opus 4.8は「この仕事のためだけの仕組み」をその場で設計できる賢さに到達した、というのがAnthropicの説明です。

▎PATTERN │ 台本に組み込まれる戦い方

Anthropicが挙げる代表的なパターンがこちらです。

・分割統治: タスクを細かく割り、1体ずつクリーンな記憶で担当させ、最後に統合する
・敵対的検証: あるエージェントの成果を、別のエージェントが粗探しする。答えが一致するまで反復
・トーナメント: 複数体に同じ課題を別アプローチで競わせ、審判役が勝者を選ぶ
・完了まで回す: 「新しい発見がゼロになるまで」探索エージェントを出し続ける

一発勝負のAIでは届かない精度に到達できる理由は、この「複数の目で潰し合う」構造にあります。

▎CASE │ 実例

・Bun（人気のJavaScript実行環境）は、この仕組みでZig言語からRust言語への書き換えを進めたと紹介されています
・コードベース全体のセキュリティ監査: カテゴリ別に独立エージェントが調査→別働隊が各指摘に反論→構造化レポート
・数十ファイル規模のライブラリ移行や、原因不明バグの調査（ログ・ファイル・データを別々の担当が調べ、仮説を独立に検証）

「修正は1行、でも影響範囲の確認が膨大」みたいな仕事ほど効きます。

▎HOW │ 使い方

・Claude Code v2.1.154以降が必要です
・プロンプトに「ultracode」と入れるか、「ワークフロー作って」と頼むだけで起動します
・/effort ultracode にすると、タスクごとにAIが自動で判断してくれます
・まず試すなら同梱の /deep-research が手軽です。複数の情報源を横断調査して、検証済みのレポートを1本返してくれます
・/workflows で進捗・エージェント数・トークン消費をリアルタイム監視できます

▎TIPS │ 保存と再利用

気に入った台本はワークフロー画面でsキーを押すと保存でき、次回からスラッシュコマンド1発で同じ段取りを再実行できます。

「毎ブランチで回すレビュー」「定例の調査」のような繰り返し仕事は、/loop（定期実行）や /goal（完了条件の指定）と組み合わせるのが公式推奨です。中断しても完了済みの結果はキャッシュされ、途中から再開できます。

▎WARN │ 注意点

・トークン消費は通常セッションよりかなり多くなります。公式も「まず小さいタスクで感覚を掴んで」と案内しています
・「10kトークンで」のように予算を直接指定するのも有効です
・実行中はユーザーの途中介入ができません。段階ごとに確認したい場合は、段階ごとに別のワークフローとして回します
・対象はMax/Team/Enterpriseプラン＋API（Bedrock/Vertex AI/Foundry含む）。Enterpriseは管理者の有効化が必要です

大規模な移行、コードベース全体の監査、絶対に間違えたくない変更。そういう「1人のAIでは手に余る仕事」の答えがこれです。
