## nix環境でclaude codeをバージョンアップしたい

どうも、レオナールです。  
突然ですが、私はいまMacOSでnixの仮想環境を使いつつも  
その中でclaude codeを活用して  
AIエージェントの活用をしている状況ではあります。

環境情報としては  
nix : 2.34.5  
claude code : 2.1.123  
と言ったものです。

今回、claude codeをバージョンアップしたいと思って  
nix内のshellにあるclaude codeをバージョンアップしようとしたところ、  
なぜかバージョンアップできませんでした。

ちなみに、なぜclaude codeのバージョンアップをしたかったのかというと  
fable5を使いたかったからです。

fabel5は2.1.170からしかモデルが使えなかったので  
バージョンアップをしようとしたところ  
なぜかアップデートされず、変なことになっていたということです。

## 何をどうしたか

ちょっとエラーが意味がわからなくて、  
一旦claudeに聞きました。  
すると、nix環境でのdevenv.nixの中のclaude-codeのバージョンが固定されてしまっているというのが原因ということがわかりました。

正直いうとnixのことはあんまり知らず、  
今のMacOS内で仮想環境を別に分けながら  
コマンドのバージョンを環境別に定義できると思ったので  
使っていたので中身までそこまで知りませんでした。

具体的にどうやって解決したかというのは結構説明が必要なので概要だけ記載しておきます。  
・claudeのバージョンアップ  
・devenv shellの環境でclaude --version、whitch claudeでバージョン確認  
→claudeがバージョンアップされていない

```
$ which claude                                                         
/nix/store/[uniqeな文字列]-claude-code-2.1.123/bin/claude
```

・claude codeのdevenv.nixのバージョンが固定されている  
→claudeに質問して真偽の確認  
・devenv updeteが効かない  
→claude codeでunfree対応になっているのでyamlの設定ファイルを書き換える必要がある  
・devenv.yamlのinputにallow: Unfreeを追加  
・devenv updeteが通る

と言った形で対応しました。

## 対応して思ったこと

nixというdevenvという環境は  
便利だから使っていたんですけど  
仮想環境内で少し変更する時には環境内の設定値に関しての知識が多少必要と感じました。

でも、claudeに聞くと一瞬で回答が返ってきて大した問題でもなくなったというのが  
これまでのデバッグ対応と違うところですかね笑

devenv環境というのはnix専用のコマンドだったりツールが環境から独立して配置されている仕様となります。

fableを使いたくてnix環境を頑張ってデバッグしようとしたんですけど  
なかなか癖があるものだなと思いました。  
devenv自体もnix環境の一種のようなので使い分けが必要という認識です。

## 何がやりたかったのか

claude codeをnix環境内で一旦バージョンアップしてその先何がやりたいかというと  
fableでUnityを動かすにあたって何ができて何ができていないかを明確にすることです。

半分好奇心で半分は検証がてらやってまして、  
今後活用の範疇としてはインフラの要件定義や基本設計や詳細設計に関しても  
活用する想定なので設計について勉強中です。

ゲームの構成でもインフラの要件定義や基本設計の考え方が使える部分があるので  
なるべく知識の転用をしつつ、AIエージェントの活用範囲を明確にしながら  
自動化するような方針で作業を進めたいというのが今の私のやりたいことですね。
