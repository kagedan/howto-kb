【冒頭フック】

claude って打つ前に、毎回 cd ~/projects/myapp ってやってませんか？

自分はずっとそうしてたんですが、ある日「これ、まとめて1コマンドにできるんじゃ」と思って設定してみたら、思った以上に快適になりました。

内容は単純で、ターミナルに cc と打つだけで、指定したディレクトリに移動してから Claude Code が起動するようにする、というものです。たった数行の設定で終わります。

【なぜこれをやろうと思ったか】

Claude Code を使い始めて最初に感じた地味なストレスが、「起動するまでの手数の多さ」でした。

やること自体は2ステップで

・cd でプロジェクトのディレクトリに移動する

・claude と打って起動する

これだけなんですが、毎日何度もやっているとじわじわ面倒になってきます。しかも複数のプロジェクトを掛け持ちしていると、どのディレクトリで起動すればいいか毎回考え直すことになる。

そこで「ディレクトリ指定 + claude 起動」を1コマンドにまとめる cc 関数を作ることにしました。cc は claude code の略です。

【準備するもの】

・Claude Code がインストール済みであること（claude コマンドが使える状態）

・bash か zsh が使えること（Windows の場合は Git Bash や WSL でも OK）

・テキストエディタ（設定ファイルを編集するだけなので何でも可）

以上です。特別なツールは不要です。

【手順】

■ ステップ1：シェルの設定ファイルを開く

bash を使っている場合は ~/.bashrc、zsh の場合は ~/.zshrc を開きます。

nano ~/.bashrc # bash の場合

nano ~/.zshrc # zsh の場合

Windows の PowerShell を使っている場合は、以下のコマンドでプロファイルを開きます。

notepad $PROFILE

プロファイルファイルが存在しない場合は New-Item -Path $PROFILE -Type File -Force で作成してから開いてください。

■ ステップ2：cc 関数を追加する

設定ファイルの末尾に以下を追加します。

【bash / zsh の場合】

cc() {

  local dir="${1:-$HOME/claude-workspace}"

  cd "$dir" && claude

}

引数を渡すとそのディレクトリで起動し、引数なしで cc と打つだけだとデフォルトのディレクトリ（ここでは ~/claude-workspace）で起動します。$HOME/claude-workspace の部分は自分がよく使うプロジェクトのパスに書き換えてください。

【PowerShell の場合】

function cc {

    param([string]$dir = "$HOME\claude-workspace")

    Set-Location $dir

    claude

}

やっていることはどちらも同じで、①ディレクトリに移動してから②claude を実行する、という2段階の処理を1つの関数にまとめているだけです。

■ ステップ3：設定を反映する

設定ファイルを保存したら、変更を反映させます。

source ~/.bashrc # bash の場合

source ~/.zshrc # zsh の場合

PowerShell の場合はターミナルを再起動するか、. $PROFILE を実行します。

■ ステップ4：動作確認

cc と打ってみてください。指定したディレクトリに移動してから Claude Code が起動すれば成功です。

別のプロジェクトで作業したいときは cc ~/projects/other-project のようにパスを引数で渡すこともできます。

【ハマりポイント・注意点】

ひとつ注意したいのが、Claude Code はその起動ディレクトリを「作業の基点」として扱うという点です。

たとえばプロジェクトの途中のサブディレクトリで claude を起動すると、プロジェクトルートにある CLAUDE.md や .claude/ の設定が読み込まれないことがあります。自分も最初これで「あれ、設定が反映されてない？」と混乱しました。

cc 関数を使うとデフォルトで決まったディレクトリから起動するようになるので、この問題を自然に回避できます。

また、Windows 環境で Git Bash と PowerShell を両方使っている場合は、それぞれの設定ファイル（~/.bashrc と $PROFILE）に同じ関数を追加しておくと、どちらのターミナルからでも cc が使えて便利です。

【実際に使ってみての感想】

設定してから1週間ほど経ちますが、Claude Code を開くときのストレスがほぼゼロになりました。作業フォルダで cc と打つだけで始められるので、「あ、ちょっと Claude に聞いてみようかな」という軽いノリで起動できるようになった気がします。

こういう「地味だけど毎日やってること」を1ステップでも減らすと、思ったより作業のテンポが変わるんだな、と実感しています。難しい設定じゃないので、Claude Code ユーザーは一度やってみることをおすすめします。

【まとめ】

・cc 関数を作れば、ディレクトリ移動 + Claude Code 起動を1コマンドにできる

・設定ファイル（.bashrc / .zshrc / $PROFILE）に数行追加するだけ

・引数なしでデフォルトディレクトリ起動、引数ありで任意のディレクトリ起動に対応

・プロジェクトルートから起動するので CLAUDE.md の設定も正しく読み込まれる

地味な改善ですが、毎日使うツールだからこそ効いてきます。もしこの記事が参考になったら、スキを押してもらえると次の記事を書く励みになります。
