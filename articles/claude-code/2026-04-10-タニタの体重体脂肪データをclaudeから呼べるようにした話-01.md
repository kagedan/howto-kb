## はじめに ― なぜ作ったのか

健康診断の血液検査で肝臓の数値がよろしくなかったので、ダイエットをしていますが、ここ最近は運動をしても体重が減りづらくなってきました。モチベーション維持のためにも視点を変えたいのと、LLMでも誰でもいいので「頑張っているね！」って言ってもらいたかった。そんな動機でLLMに私の機密情報を教える決意をしました。

リポジトリ: [personal-health-mcp](https://github.com/okizuka-hitts/personal-health-mcp)

---

## MCPサーバーの実装自体は難しくない

最初に言ってしまうと、Claude Code を使えば MCP サーバーの実装は難しくないです。特に今回はローカル動作なので、セキュリティ等もIDやSECRETの管理ぐらいの最低限だけです。  
肝心のデータについても、私の使っているタニタの体組成計[BC-768](https://shop.tanita.co.jp/shop/products/BC768WH)はBluetoothでモバイルアプリ経由で[HealthPlanet](https://www.healthplanet.jp/)にデータを保持してくれるし、OAuth2認証のAPIも提供されているので、あとは、FastMCPフレームワークを使ってMCPツールにするだけです。  
キャッシュ・差分同期設計は効率化のために必要ですが、設計方針さえ固まっていれば、Claude Codeが実装まで持っていってくれました。

---

## ユースケースからClaude Codeに渡す要件定義を行う

ツール仕様の検討は、要件定義書をClaudeと対話しながら作るスタイルで進めました。当初はMCPができればいい程度の要求から要件定義を作りだしたのですが、ユースケースがはっきりしていなかったためか、最初の要件は曖昧すぎて、次のような課題を見つけて対処しました。

#### 廃止されたAPI Tagを使ってる

当初は[HealthPlanetのAPI仕様](https://www.healthplanet.jp/apis/api.html)で提供の終わってるタグのデータまで要件定義に入ってました。API仕様書のURLはプロンプトに含めたので安心していましたが、Claudeは誤認していたようです。廃止されたタグはHTML上は<S></S>で消されていたけど、Claudeに画面キャプチャも渡して明確に対象外にしました。

#### 体重と体脂肪率しかわからない設計になっていた

HealthPlanetのAPIは基本的で体重と体脂肪率だけが取得できる仕様です。Claudeの提案では、当初MCPはこのAPIのラッパーのような仕様になっていたので、体重と体脂肪率を聞くぐらいしかできないMCPツールになってしまっていました。これだと何も分からない。  
Claudeと相談。![](https://static.zenn.studio/user-upload/75d18d8c919b-20260410.png)  
最終的には「BMIで肥満度を判定したい」とか、「基礎代謝（Harris-Benedict式など）も計算し目標体重までの食事や運動のサポートをLLMにお願いしたい」、など具体的なユースケースと実現性を踏まえ、Claudeの提案通りに「身長」「性別」「年齢（生年月日）」の基礎データをHealthPlanetのAPIのデータから収集し、DBに保存、`get_profile`でLLMに情報を提供できるようにしました。

### 最終的なMCPツール一覧

| ツール名 | 概要 |
| --- | --- |
| `get_latest_body_composition` | 直近の体重・体脂肪率を1件返す |
| `get_body_composition` | 指定日付範囲のデータ一覧を返す |
| `get_measurements_range` | データが存在する日付範囲と件数を返す |
| `get_profile` | 生年月日・身長・性別を返す |
| `list_available_metrics` | 取得可能なメトリクス一覧を返す |

## 実際に使ってみた ― LLMならではの体験

MCPをClaude Desktopに接続し、実際にいくつかプロンプトを試してみました。

### 「除脂肪体重で変動見せて」

タニタのヘルスプラネットのモバイルアプリだと、「筋肉量」とか「推定骨量」を見せてくれるけど、APIではそれらのデータは意図的な(?)制限なのか取得できない。そこで一般的なデータで置き換え。除脂肪体重の変化を見ることで、体重変化の質が分かります。  
Claudeに問い合わせると除脂肪体重を計算し、グラフと考察まで出してくれた。  
![](https://static.zenn.studio/user-upload/d958068a5ec3-20260410.png)

### 「月に1kg体重を落としたい。運動は週5日、1日の運動の消費が350kcalとして、摂取カロリーの理想教えて」

Claudeは想定通り基礎代謝を計算し、必要な消費カロリーを運動分を考慮した上で回答した。MCPサーバー経由だとプロフィール情報が推論の前提として使われる。  
![](https://static.zenn.studio/user-upload/e149f052c665-20260410.png)

またMCPにしたことで、ダイエットを意識した消費カロリーを考慮した会話ができるようになった。

```
今日、どうしても夜に厚切りトンカツ定食食べたｓいい？
朝はピーナツバター塗ったトーストとブラックコーヒーでした。
```

Claudeの回答が

```
昼食の具体的な提案はこんなイメージです。
しっかり食べたい場合（〜800kcal）
夜にトンカツを控えているので、昼は脂質を抑えた和定食系が相性よいです。
例えば焼き魚定食（鮭・さば）、豚しゃぶ定食、蕎麦＋小鉢1品あたりが700〜800kcalに
収まりやすいです。
軽めに済ませたい場合（〜500kcal）
夜を思いきり楽しみたいなら昼を絞る手もあります。
そばや冷しゃぶサラダ、おにぎり1〜2個＋味噌汁くらいで500kcal以内に抑えると、
夜の罪悪感がゼロになります。
```

## まとめ・所感

AI Agentに実装を任すのであれば、ローカル動作のMCPサーバーの実装は難しくないし、すぐに実現できる。データ量やAIの利用制限が許す限り色々試してみたい。

実はClaudeからは当初期待した「頑張ってるね！」と言ってもらえていない。次はBLE心拍計のデータもMCPで取れるようにし、運動の消費カロリーや運動強度からプラン作ってもらえるように、そして次こそ「頑張ってるね！」と言ってもらえるようにしたいです。
