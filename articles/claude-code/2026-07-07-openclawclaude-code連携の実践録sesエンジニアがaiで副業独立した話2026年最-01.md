## SESやめたい、でも何から始めればいい？

「SES 1年目で転職を考えている」「客先常駐に疲れた」「フリーランスになりたいけど何のスキルが足りないか分からない」

2026年現在、こうした相談がエンジニアコミュニティに溢れている。筆者も数年前はSESエンジニアとして客先を転々としていた一人だ。

この記事では、**OpenClaw（AIの思考・記憶・指示管理）とClaude Code（コード生成・実行）を組み合わせた実践的な開発ワークフロー**を、実際のコマンドと得られた結果とともに紹介する。SESで消耗しながらも「次のステップ」を模索しているエンジニアにとって、このAI連携ワークフローはスキルギャップを埋めるための現実的な武器になる。

## OpenClawとClaude Codeとは何か

まず基本的な整理から入ろう。

### OpenClaw

OpenClawは、AIエージェントに**長期記憶・経営判断・タスク指示**をシステム的に与えるためのOS的フレームワークだ。Claude Codeなどのコーディングエージェントに「会社の方針」「過去の意思決定」「役割定義」を持たせることができる。

構成は大きく3レイヤー:

```
openClaw/
├── skills/          # 役割別スキル定義（CEO, CTO, CMO...）
├── memory/          # 長期記憶ストア（プロジェクト文脈・教訓）
└── projects/        # プロジェクト別設定・状態管理
```

### Claude Code

Anthropic製のコーディングエージェント。ターミナルから`claude`コマンドで起動し、自然言語で「このバグを直して」「APIを追加して」などの指示を出すと、ファイル読み書き・テスト実行・Gitコミットまで自律的にやってくれる。

**2つを組み合わせると何が起きるか**:

- OpenClawが「何を・なぜ・どの優先度で」を判断（思考と記憶）
- Claude Codeが「どうやって」を実装（実行）

経営と開発の分業が、AIレイヤーで完結する。

## 実践ユースケース1：フリーランス案件調査の自動化

### 課題

SES 1年目で転職や独立を検討し始めると、まず「自分のスキルの市場価値」が分からない壁にぶつかる。Forkwellやレバテックを見ても、単価相場の幅が広すぎて自分がどこに位置するか見えない。

### ソリューション

OpenClawのCMOスキルにリサーチ機能を持たせ、Claude Codeで結果をMarkdownに整形するパイプラインを組んだ。

```bash
# OpenClaw CMOスキルで案件リサーチを指示
claude --skill openclaw-keiei-os:cmo \
  "Pythonフリーランス案件の単価相場を調査して。\
   スキルセット: Python, FastAPI, AWS, 3年経験。\
   想定市場: 東京リモート案件。\
   レバテックとCrowdWorksの相場感を比較して"
```

**実際に得られた出力（抜粋）**:

```markdown
## Python/FastAPI フリーランス単価相場（2026年7月時点）

### レバテックフリーランス傾向
- Python全般: 月60〜90万円帯が主流
- FastAPI + クラウド経験3年: 月75〜95万円が中央値
- AWSアーキテクチャ設計ができる場合: 月90〜120万円超も

### SES正社員との比較
- SES平均年収（3年目）: 420〜500万円/年
- 同スキルのフリーランス: 年収換算900〜1,080万円
（社保自己負担・非稼働月を考慮しても1.5〜1.8倍）

### 注意点
- 出典が存在する最新の統計ではなく市場観測ベース。
  契約前に複数エージェントへ確認を推奨。
```

この調査を手動でやると半日かかっていた。プロンプト1本で5分以内に構造化されたサマリが返ってくる。

## 実践ユースケース3：副業プロダクト開発の自律実行

ここが一番インパクトが大きかった部分だ。OpenClawのCTOスキルで技術判断を委ね、Claude Codeで実装を回すことで、**SESの業務時間外（夜間・休日）に副業プロダクトを自律的に開発**できるようになった。

### 実際のコマンドフロー

```bash
# 1. プロジェクト仕様をOpenClawに登録
claude --skill openclaw-keiei-os:cc-products \
  "新プロダクト登録:\
   名前: SES単価シミュレーター\
   概要: スキルセットを入力すると市場単価を推定するWebApp\
   技術スタック: Next.js, Python(FastAPI), Vercel\
   MVP期限: 2026年8月末"

# 2. CTOスキルで技術設計レビュー
claude --skill openclaw-keiei-os:cto \
  "SES単価シミュレーターのシステム設計をレビューして。\
   DB設計案: スキルマスタをJSONで持つFlat構造 vs PostgreSQLリレーション。\
   トレードオフを分析して推奨案を出して"

# 3. Claude Codeで実装実行
claude "CTOの推奨に従いPostgreSQL設計で実装開始。\
  まずスキルマスタテーブルのマイグレーションファイルを作成して。\
  Prisma使用、TypeScript型定義も同時生成して"
```

### 実際に生成されたコード（抜粋）

```typescript
// prisma/schema.prisma (Claude Codeが生成)
model Skill {
  id          String   @id @default(cuid())
  name        String
  category    SkillCategory
  baseRate    Int      // 基本月単価（万円）
  premiumRate Int      // 上位25%単価（万円）
  demandScore Float    // 需要スコア（0-1）
  updatedAt   DateTime @updatedAt
  
  skillCombinations SkillCombination[] @relation("primarySkill")
}

model SkillCombination {
  id            String @id @default(cuid())
  primarySkill  Skill  @relation("primarySkill", fields: [primarySkillId], references: [id])
  primarySkillId String
  bonusRate     Int    // 組み合わせボーナス（万円）
}

enum SkillCategory {
  LANGUAGE
  FRAMEWORK
  CLOUD
  DATABASE
  DEVOPS
}
```

このスキーマ設計の確認から実装まで、手動なら2〜3時間かかるタスクが約15分で完了した。

## 実践ユースケース4：SES契約条件の交渉資料自動生成

SESエンジニアが転職・フリーランス化を検討するとき、「自分の市場価値を数字で説明できない」という壁がある。OpenClawのCFOスキルを使うと、交渉用の根拠資料を自動生成できる。

```bash
claude --skill openclaw-keiei-os:cfo \
  "フリーランス転向の収支シミュレーションを作成して:\
   現状: SES正社員 年収480万円（東京）\
   想定フリーランス単価: 月70万円\
   稼働見込み: 年10ヶ月\
   経費概算: 月15万円（健保, 年金, 交通費, 機材）\
   比較表と損益分岐点を計算して"
```

**出力されたシミュレーション**:

```markdown
## フリーランス転向 収支シミュレーション（2026年）

| 項目              | SES正社員       | フリーランス       |
|-------------------|-----------------|--------------------|
| 年間売上           | 480万円         | 700万円（70万×10月）|
| 社会保険料（自己負担）| 約67万円（会社折半）| 約130万円（全額）   |
| その他経費         | 実質ほぼ0       | 約50万円           |
| **手取り推計**     | **約380万円**   | **約520万円**       |
| 差額               | -               | **+140万円/年**     |

### 損益分岐点
- 稼働月6.9ヶ月以上でSES正社員の年収を超える
- 月単価60万円以上で実現（スキルセット上は達成可能水準）

### リスク要因
- 非稼働月の発生リスク（エージェント紹介の安定性に依存）
- 確定申告・帳簿管理の工数（freeeで月2〜3時間程度）
```

この種の計算資料を作るのに、以前は表計算ソフトと格闘していた。プロンプト1本で10分以内に交渉用の根拠資料が完成する。

## ワークフロー全体像：OpenClaw × Claude Code の構成

```
┌─────────────────────────────────────────────┐
│               OpenClaw OS                    │
│  ┌──────────┐ ┌──────────┐ ┌──────────────┐ │
│  │  記憶層   │ │  思考層   │ │   指示層     │ │
│  │(memory/) │ │(skills/) │ │ (tasks/)     │ │
│  │プロジェクト│ │CEO/CTO   │ │ 優先度      │ │
│  │文脈/教訓  │ │CMO/CFO   │ │ スケジュール │ │
│  └──────────┘ └────┬─────┘ └──────────────┘ │
└───────────────────┼─────────────────────────┘
                    │ 判断・指示
                    ▼
┌─────────────────────────────────────────────┐
│             Claude Code                      │
│  ┌──────────┐ ┌──────────┐ ┌──────────────┐ │
│  │ファイル   │ │テスト    │ │  Git/Deploy  │ │
│  │読み書き   │ │実行      │ │  自動コミット │ │
│  └──────────┘ └──────────┘ └──────────────┘ │
└─────────────────────────────────────────────┘
```

### 設定ファイルの実例

OpenClawとClaude Codeの連携を最大化するには、`.claude/CLAUDE.md`にプロジェクト文脈を埋め込む。

```markdown
# プロジェクト: SES単価シミュレーター

## コンテキスト
- オーナー: フリーランスPythonエンジニア（独立1年目）
- 目的: SES市場の単価相場を可視化するWebサービス
- フェーズ: MVP開発中（2026年7月〜）

## 技術スタック
- Frontend: Next.js 15 + TypeScript + Tailwind v4
- Backend: FastAPI + PostgreSQL（Neon）
- Deploy: Vercel（Frontend）+ Railway（Backend）

## 開発ルール
- コミット前にESLint + Prettierを必ず通す
- 本番pushは差分確認後のみ
- APIキーはVercel環境変数に分離（ハードコード禁止）
```

このCLAUDE.mdがあることで、毎回「このプロジェクトは何か」を説明する必要がなくなり、Claude Codeが文脈を引き継いで作業を続けられる。

## SESエンジニアへの現実的なアドバイス

### 「SESやめたい」と思ったときの選択肢

SES 1年目で転職を考えるのは珍しくない。ただし選択肢は「転職」だけではない。

**選択肢A: 社内SE・自社開発企業へ転職**
- メリット: 安定した環境、チームでの開発経験
- デメリット: 転職活動の工数、スキルセットの証明が必要

**選択肢B: SESのまま単価交渉 or 上位案件へ移行**
- メリット: リスクが低い、現職の経験を積みながら市場価値を上げる
- デメリット: 構造的な問題（多重下請け）は解決しない

**選択肢C: 副業→フリーランス転向（本記事の主テーマ）**
- メリット: 単価相場が大幅に上がる可能性、自律性が高い
- デメリット: 収入の波、営業工数、社会保険の全額自己負担

どの選択が正解かは個人の状況による。ただし2026年現在、**AIツールを使いこなせるエンジニアの市場価値は明確に上がっている**。OpenClaw + Claude Codeのようなスタックを実務で使える人材は、採用市場でも差別化要因になる。

### フリーランス移行前に確認すべきチェックリスト

```bash
# OpenClawで移行準備チェックリストを生成
claude --skill openclaw-keiei-os:coo \
  "フリーランス独立前のチェックリストを作成して。\
   スキル面・財務面・契約面・メンタル面の4軸で整理して"
```

生成されたチェックリスト（要約）:

- **スキル面**: GitHub公開ポートフォリオ3件以上、技術ブログまたは登壇実績
- **財務面**: 生活費6ヶ月分の貯蓄、freeeまたはマネーフォワード導入
- **契約面**: 電子契約（CloudSign等）の準備、NDA・業務委託契約書のひな形
- **メンタル面**: 非稼働月0収入の現実を想定済み、家族・パートナーの理解

## まとめ：AIは「次のキャリアへの橋」になる

OpenClawとClaude Codeの連携は、単なる「便利なツールの組み合わせ」ではない。**AIに思考・記憶・実行を委ねることで、一人のエンジニアが小さなチームと同等のアウトプットを出せる**ようになる。

SES 1年目で転職を考えているなら、まず副業でこのワークフローを使ってポートフォリオを作ることを勧める。フリーランスへの転向を考えているなら、単価相場の調査から収支シミュレーションまでをAIに任せ、判断の精度を上げることができる。

「SESやめたい」という感情は出口を持っている。その出口を開ける鍵のひとつが、今のAIツールスタックだ。
