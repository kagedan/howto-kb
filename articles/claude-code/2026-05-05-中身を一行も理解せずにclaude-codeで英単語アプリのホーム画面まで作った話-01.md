# 中身を一行も理解せずにClaude Codeで英単語アプリのホーム画面まで作った話

![](https://static.zenn.studio/user-upload/8395ce021fa7-20260505.png)  
これ、自分はコードを一行も理解していません。

ついでに言うと、Next.jsもPrismaもSQLiteも、何ひとつ「これは何か」を説明できません。動いてるけど、中で何が起きてるかは謎のまま。

そんな状態でClaude Codeを触ってみたら、休日の数時間で英単語アプリのホーム画面までできてしまった、という話です。

## 自分について

普段はセキュリティ系の業務をやってます。インフラ寄りの現場経験あるけど、**プログラミング経験はゼロ**。コードを書く側の仕事はしたことがない、という前提で読んでもらえると。

## きっかけ：Claudeに「なんか作りたい」と相談した

Claude Codeっていうコーディング用のツールがあるらしい、と聞いて触ってみたくなった。とはいえ何を作ればいいか分からない。なのでまずChatのClaude（つまりこの記事を一緒に書いてる相棒）に「テスト的にClaude Code使ってアプリ的なの作りたい、案出して」と丸投げ。

返ってきたアイデアの中から「英単語フラッシュカード（間隔反復付き）」を選択。せっかくなので音声付きで、AIで例文も自動生成、みたいなところまで欲張った。

## 設計も全部おまかせ

技術スタックは全部Claudeが決めた。

* Next.js + TypeScript
* Tailwind CSS
* Prisma + SQLite
* Leitnerシステム（間隔反復のアルゴリズム）

…と言われても、この時点で自分が知ってるのはTypeScriptって名前だけ。あとは「ふーん、それでいいよ」と全肯定。

さらにClaudeが「Phase 1〜4に分割して、各Phaseごとに自分（Claude）がClaude Codeに投げるプロンプト出すよ」と言ってくれた。至れり尽くせり。**ありがたく頼ることにする**。

## Phase 1：とりあえずYesを押し続ける時間

ChatのClaudeから「Phase 1のプロンプトです、これコピペしてClaude Codeに貼って」と渡された。素直に貼る。

Claude Codeが何やら考え始めて、「このコマンド実行していい？」と聞いてくる。

```
npx create-next-app@latest flashcard-app --typescript --eslint --tailwind ...
```

…と言われても分からない。分からないので、その都度ChatのClaudeにスクショを送りつけて「これ何？Yes押していい？」と聞く。

「これは新規プロジェクト作るコマンド、Yesで大丈夫」

「Yes」

「これはNode.jsの場所探してるだけ、安全」

「Yes」

「これは型チェック、読み取り専用」

「Yes」

…という流れで、画面の右と左を行き来しながらYesを連打する時間がしばらく続いた。**自分でやってる感ゼロ。** だが進む。

## ハマりポイント①：Node.jsが入っていない

数分後、Claude Codeが何度もNode.jsを探し始めた。色んな場所をしらみつぶしに。

ChatのClaudeに状況を見せたら「これ、たぶん入ってないですね」と。試しにターミナルで `node -v` を叩く。

```
zsh: command not found: node
```

うん、入ってない。

そもそもNode.jsって何？という疑問は飲み込んで、公式サイトからインストーラー落としてポチポチ。`v24.15.0` がインストールされた。何かは分からないが入った。

## ハマりポイント②：Prismaがv7すぎて沼る

Node.js問題が解決したら、今度はPrisma周りでClaude Codeの様子がおかしくなった。

「`@prisma/adapter-libsql` のバージョン確認」「`@libsql/client` のバージョン確認」「Homebrewにnodeあるか確認」…と、**何かを延々と調査し続ける**。

明らかにおかしい。ChatのClaudeに泣きつく。

Claudeが画面の上の方を見るように指示してくれて、エラーメッセージが見つかった。

> Prisma 7はスキーマでのURL設定を廃止した破壊的変更があります。Next.js 15対応のPrisma 5（安定版・標準構成）にダウングレードする方が最もシンプルです。

要するに「`create-next-app` が最新すぎて、安定版より新しいPrismaを連れてきちゃった」。Claude Codeは「v5に下げよう」と提案してきていた。

**自分にはこれが正しい判断なのか、まったく分からない。**

ChatのClaudeに「これ受け入れていい？」と聞いたら「v7は新しすぎて情報少ない、v5は枯れてる安定版、ダウングレードは正当な判断、Yesでいい」と。

「Yes」

数分後、`Prisma 5.22.0` で全部やり直した結果、エラーが消えた。直ったらしい。**何が起きたかは正直よく分かっていない。**

## ハマりポイント③：Claude Codeの暴走を止める判断

このハマりの過程で一番勉強になったのが、**「Claude Codeは放っておくと環境いじりで解決しようとし始める」** という現象。

`@prisma/adapter-libsql` を調べる、別のクライアントを調べる、Homebrewを調べる、…と、何かを足したり差し替えたりすることで解決を試みる方向に走り出した。

これをChatのClaudeが察知して、「No」を選んで一回止めるべきと教えてくれた。

> 「環境いじりで逃げ」モードに入ってる。これは典型的な迷走パターンで、放置すると泥沼化します。

素人目には「Claude Codeすごいから任せておけば全部解決してくれる」と思いがちだけど、**要所で『止めて事情を聞く』判断が必要**らしい。

これは自分がやったというより、ChatのClaudeに教えてもらった学び。実態としては自分が判断してるんじゃなくて、AIが別のAIを監督してる状態。**その横で自分はYes/Noボタンを押す係**。

## 完成：ホーム画面が出た

Phase 1のすべてが終わったあと、別ターミナルで `npm run dev` を叩いて、ブラウザで `localhost:3000` を開いた。

![](https://static.zenn.studio/user-upload/d7ae9b2af5ee-20260505.png)

**素直に「うわ、すごい」と思った。**

中央寄せのレイアウト、4枚のカード、アイコン付き、説明文付き、ホバーしたらちょっと動く。自分が指示した内容は「ホーム画面と4つのリンクを作って」程度だったのに、ここまで整ったものが出てきた。

しかも自分はこの画面のコードを **一行も書いていないし、一行も読んでいない**。

## 振り返り：これってどうなのか

楽しかった。動きそうなものが手元にある喜び。

ただ、自分は普段セキュリティ系の業務をやっている。「**中身を理解しないまま動かすな**」と人に言う側の人間である。

そんな自分が今日やったのは、

* 知らないランタイムを入れる
* 知らないフレームワークでプロジェクトを作る
* 知らない理由でバージョンを下げる
* ファイルを中身を見ずにコミットする

**完全に普段「やるな」と言われている側の動きである。**

今回はローカル遊びだから実害ゼロだけど、次は **既に動いてるものの中身を一回開けてみる** くらいから、ちゃんと向き合っていきたい。  
なおセキュリティ職として、今回の作業を業務環境や本番で同じノリでやるのは論外だと自負しております。

## これから始める人へ（軽め）

もし「自分もClaude Codeで何か作ってみたい」という人がいたら、雑に共有しておくこと。

* **ChatのClaudeとClaude Codeを併用するのが最強**：Claude Codeが何やってるか分からない時、スクショ送ると優しく教えてくれる。
* **Yesを押す前に一回聞く癖をつけるといい**：特に「インストール」「ダウングレード」「設定変更」系は要注意。自分はこれをChatのClaudeに丸投げしてた（ただし別AIも誤判断するため、業務マシンや共有環境では注意が必要）
* **詰まったら『なぜそうしようとしてるか』を聞く**：Claude Codeが暴走しかけた時、理由を聞くと冷静になれる
* **スクショを撮りまくる**：あとで記事を書くときに役立つ（これがまさにそう）
* **課金は今のところゼロ（2026/05/05執筆時点では）**：Claude.aiの月額サブスクの中で完結。API連携を始めると別途課金になるけど、今回はそこまで行ってない
