おはようございます、って言おうとしたら今日ももうコンバンワの時間になってたと思ったら、翌日になってたイイ加減おじさんです。

最近、Antigravityに続いてようわからんのがClaude Codeです。いつも通りイイ加減に調べてみました。調べたよと言っても調べたのはAIだけどね。そもそもよくわかって無いんで😅

## chatGPTが調べたよ

結論から言うと、  
  
・出来ることはかなり被っている  
・ただし思想がまったく違う  
・結果として“使い方”も全然変わる  
  
この3点。  
  
この記事では、Antigravityとの違いを軸に  
Claude Codeの正体を整理していきます。いや、もうめんどくさいから整理するのはＡＩくん達ね。

### Claude Codeとは

一言で言うと、  
  
「ターミナルに住んでるAIエンジニア」  
  
です。  
  
黒い画面（CLI）から日本語で指示を出すと、  
  
・コード生成  
・既存コードの修正  
・ファイル操作  
・テスト実行  
・Git操作  
  
まで全部やってくれます。  
  
つまり、  
  
👉 エディタの中にAIがいるのではなく  
👉 OSレベルで“開発を代行する存在”  
  
これがポイントです。  
  
⸻

### できること（結論：ほぼ同じ）

まず気になるのはここ。  
  
「Antigravityと同じこと出来るの？」  
  
結論：  
  
👉 ほぼ全部できる  
  
⸻

### Claude Codeで出来ること

・アプリ開発（フロント〜バックまで）  
・既存プロジェクトの改修  
・バグ修正  
・テスト自動化  
・複数ファイルの一括編集  
・Git操作（commit / push など）  
・外部ツール連携（Slack・Driveなど） (ぽんずテックブログ￼)  
  
⸻

### Antigravityと被る領域

・自然言語 → コード生成  
・エージェントによる自動実行  
・マルチファイル編集  
・開発の自動化  
  
⸻  
  
つまり、  
  
👉 **“出来ること”で見るとほぼ同じ**  
  
⸻

### じゃあ何が違うのか（ここが本質）

ここが一番重要です。  
  
違いは機能ではなく、  
  
「設計思想」  
  
です。  
  
⸻  
  
**① 主導権が違う**  
  
Antigravity  
  
👉 AIが主導（丸投げ型）  
  
・タスクを投げる  
・AIが計画  
・AIが実装  
・AIがテスト  
  
👉 人間はマネージャー  
  
⸻  
  
Claude Code  
  
👉 人間が主導（共同作業型）  
  
・実行前に確認が入る  
・細かく指示できる  
・逐次コントロール可能  
  
👉 人間がハンドルを握る (ぽんずテックブログ￼)  
  
⸻  
  
この違いはかなり大きいです。  
  
⸻  
  
**② UIが違う（ここで好みが分かれる）**  
  
Antigravity  
  
👉 IDE（VS Code型）  
  
・画面が視覚的  
・エージェント管理画面あり  
・進行状況が見える  
  
⸻  
  
Claude Code  
  
👉 ターミナル  
  
・黒画面  
・ログベース  
・操作はコマンド中心  
  
⸻  
  
つまり、  
  
👉 Antigravity＝直感的  
👉 Claude Code＝職人向け  
  
⸻

**③ 得意領域が違う**

ここはかなり重要。  
  
⸻  
  
Antigravityが強い  
  
・0→1開発  
・丸ごとアプリ生成  
・マルチエージェント並列実行  
  
👉 「全部任せたい人」向け  
  
⸻  
  
Claude Codeが強い  
  
・既存コードの修正  
・精密なリファクタリング  
・品質重視の開発  
  
👉 「細かく制御したい人」向け (うるチカラ￼)  
  
⸻  
  
イメージで理解する  
  
これが一番分かりやすいです。  
  
⸻  
  
Antigravity  
  
👉 「AI開発チームの監督」  
  
・部下（AI）が勝手に動く  
・あなたは指示出しだけ  
  
⸻  
  
Claude Code  
  
👉 「優秀なエンジニア1人」  
  
・横にいる  
・一緒に作る  
・細かい相談ができる  
  
⸻  
  
実は“併用”が最強  
  
ここはかなり重要な視点。  
  
実際は対立ではなく、  
  
👉 役割分担ができる  
  
⸻  
  
役割分担  
  
・Antigravity → 設計・全体構築  
・Claude Code → 実装・細部調整  
  
⸻  
  
この組み合わせは、  
  
👉 開発スピード  
👉 品質  
  
両方を引き上げます。 (ASIに仕事を奪われたい￼)  
  
⸻  
  
最近の進化（かなり重要）  
  
Claude Codeは今かなり進化しています。  
  
Anthropicが出した新機能で、  
  
・自動許可モード（auto mode）  
・PC操作（ファイル・ブラウザ操作）  
  
など、  
  
👉 より“エージェント化”が進行中 。  
  
⸻  
  
つまり、  
  
👉 Antigravity寄りに進化している  
  
とも言えます。  
  
⸻

### 結論

最後にまとめます。  
  
⸻  
  
■ 出来ること  
  
👉 ほぼ同じ  
  
⸻  
  
■ 本質的な違い  
  
👉 操作思想が違う  
  
⸻  
  
■ 向いている人  
  
Antigravity  
  
・初心者  
・丸投げしたい  
・爆速で作りたい  
  
⸻  
  
Claude Code  
  
・開発経験あり  
・細かく制御したい  
・品質を重視したい  
  
⸻  
  
■ 一番強い使い方  
  
👉 両方使う  
  
⸻

### 最後に

今のAI開発は、  
  
👉 「どっちが上か」ではなく  
👉 「どう組み合わせるか」  
  
のフェーズに入っています。  
  
Antigravityだけでも強いですが、  
Claude Codeを組み合わせると  
  
👉 “AI開発チーム”が完成する  
  
そんな位置づけです。

らしいです。うーん、取り敢えず使ってみんとわからんな🙁
