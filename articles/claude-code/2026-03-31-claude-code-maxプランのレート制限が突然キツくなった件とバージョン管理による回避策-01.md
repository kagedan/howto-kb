## 何が起きているのか

2026年3月23日頃から、Claude Code Maxプラン（5x / 20x）のセッション制限が異常に早く消費される問題が発生しています。以前は5時間持ったセッションが1〜2時間で枯渇し、[GitHub Issue #38335](https://github.com/anthropics/claude-code/issues/38335) には229件超のコメントが集まっています。

**主な症状：**

* 同じ作業量なのにセッションが1〜2時間で枯渇
* 1回のプロンプトで使用量が21%→100%に跳ね上がる
* 使っていないのに使用量が増える「ゴーストトークン」現象

## 原因（推定）

3つの要因が重なっていると思われます。

1. **3/28で2倍キャンペーンが終了** — 平日オフピーク・週末に使用量2倍だったのが元に戻った
2. **Anthropicによるピーク時間帯の制限強化** — 太平洋時間 5:00〜11:00 AM（日本時間 22:00〜翌6:00）が対象、約7%のユーザーに影響（公式発表）
3. **Opusの1Mコンテキストがデフォルトに** — コンテキストが大きい分、1リクエストあたりのトークン消費が増加

## 回避策

### バージョンダウン（最も効果的）

```
# とりあえず使えるバージョン（automodeは使えない）
npm install -g @anthropic-ai/claude-code@2.1.68
```

設定ファイルやプロジェクトデータはバージョン変更の影響を受けません。

### CVM（Claude Version Manager）で複数バージョン管理

[CVM](https://github.com/clawdcc/cvm) を使えば、nvm のようにバージョンを切り替えられます。macOS専用。

```
npm install -g @clawd/cvm
cvm install 2.1.68
cvm use 2.1.68
```

### その他

* **ピーク時間帯（22:00〜6:00 JST）を避ける**
* **会話を定期的にリセット**してコンテキスト肥大を防ぐ

## 所感

月額$100〜$200払っているヘビーユーザーに対して、事前通知なしのレート制限強化はかなりキツい。Anthropicが具体的なトークン割当量を公開していない以上、バージョンダウンで凌ぎつつ公式の改善を待つしかなさそうです。
