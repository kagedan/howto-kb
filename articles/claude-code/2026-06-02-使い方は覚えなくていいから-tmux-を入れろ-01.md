Claude Code や Codex を使っていると、AI にコマンドを実行させる場面が増えます。その中で困ることが 2 つあります。

1. 時間がかかるコマンドを background task で実行したとき、エージェントごとに挙動が違う
2. AI が対話型コマンドを実行できない

どちらも tmux で解決できます。

tmux といえば「ターミナルを分割してショートカットで操作する玄人向けツール、自分には関係ない」と思っている人も多いかもしれません。**しかし、tmux の操作を覚える必要はありません。** tmux を使うのは AI エージェントなので、インストールさえしておけば OK です。  
Mac であれば、今すぐ `brew install tmux` を実行しましょう。必要なことはそれだけです。

## 課題1: background task の挙動、把握していますか？

長いテストや dev server を background task として実行したとき、以下の挙動を把握していますか。

* `Ctrl+C` で止めたとき、background taskも止まるのか
* セッションを閉じたとき、プロセスは残るのか
* 再開したとき、そのタスクを参照できるのか
* ログはどこに残るのか
* 別のエージェントからそのタスクを見られるのか

Codex にも同じ疑問が当てはまります。新しいエージェントが次々と登場する中、それぞれの挙動を把握するのは大変ですよね。

## 課題2: AI は対話型の CLI が苦手

Claude Code にこんなことを言われたことはありませんか。

```
このコマンドは対話型のCLIのため、私には実行できません。
代わりに ! npx create-next-app を実行してください。
```

`npx create-next-app` や `npm init` のような対話型コマンドは、AI が直接実行できません。AI は Bash コマンドを非対話的に実行するため、途中でプロンプトが出ると処理が止まってしまいます。

これで、tmux session を使って `pnpm test` を実行してくれます。

tmux のセッションは親プロセスとは独立しているので、`Ctrl+C` で止めても background task は残りますし、セッションを閉じてもプロセスは生き続けます。`tmux ls` で一覧を確認でき、別のエージェントからも参照できます。つまり、冒頭で挙げたような疑問をエージェントごとに調べる必要はありません。tmux のセッションの挙動さえ把握しておけば、Claude Code でも Codex でも同じように扱えます。

```
さっきのテストの tmux session 監視して
```

このような指示で確認できます。Claude Code で起動した tmux session を Codex から参照するなど、エージェント間で自由に共有できます。

dev server も tmux で起動しておけば、エージェントを切り替えた後も状態を確認できます。「ログもファイルに残して」と頼めば保存してくれるので、tmux session を閉じた後でも AI にログを読ませられます。

人間の介入が必要な場面では、自分で session に入って操作できます。`tmux attach -t <session名>` で入れますが、こんなことも覚える必要はありません。都度 AI に聞きましょう。

```
さっきの tmux session に自分で入りたいからコマンド教えて
```

と言えばOKです。

### 「課題2: AI は対話型の CLI が苦手」 の解決

AI に対話型コマンドを頼んで断られても、こう返せば実行してくれます。

```
AI: npx create-next-app は対話型のCLIのため、私には実行できません。
あなた: tmux 使ってやって
```

tmux にはキー入力を送る機能があるので、AI が対話型プロンプトへの逐次回答を入力できます。  
対話型コマンドだと判れば、AI が自ら tmux で実行してくれることもあります。

## まとめ

tmux は、AI に長時間コマンドや対話型コマンドを任せたいときに役立ちます。

筆者は Codex と Claude Code を併用していますが、tmux のおかげでエージェント間の引き継ぎがかなり楽になりました。Claude Code で起動した tmux session を Codex に監視させたり、途中で自分が session に入って確認したりできます。よく使うパターンは CLAUDE.md（AGENTS.md）や Skills に書いておけば、毎回説明する必要もありません。まだ使っていない方は、まずインストールだけでも済ませておくと役立つはずです。
