Claude wrote the PR. Codex reviewed it. Claude rewrote based on the feedback.

The full workflow from r/ClaudeAI:

1. Install Codex CLI, run it in a tmux pane
2. Claude Code writes the feature, opens a PR
3. Claude pings Codex via shell, you approve permissions in real time
4. Codex reviews the PR and drops comments
5. Claude wakes up, validates each comment before editing

The catch:

Overkill for shipping experiments. Single-agent with a good /review skill handles most solo founder work.

Worth it when you ship to production daily, when a bug costs revenue, when stakes justify the setup tax.

Multi-model review might be the next leg of dev leverage. Just not for everyone yet.
