① Subagents

・作業を役割ごとに分ける
・調査、レビュー、実装を分担
・1つのチャットに全部抱え込まない

これだけでClaude Codeの使い方が変わるんだよね。

② Skills

・繰り返す手順を固定
・毎回同じ説明をしなくていい
・自分のworkflowをClaudeに覚えさせる

ここ、マジで一番大事すぎる。

③ parallel sessions

・同時に複数方向で試せる
・片方で実装、片方で検証
・人間は判断に集中できる

「1会話で全部やる」から卒業するやつ。

④ Hooks

・作業後のチェックを自動化
・formatやtestを挟める
・事故りやすい手順を先に潰せる

Claude Codeは、ただのチャットじゃないんだよね。
workflowを組んで初めて本領発揮する道具なんだよね。

creator本人の使い方を見ると、
「何を足せばいいか」がかなり見えるの助かる。
