ものづくり補助金（正式名称：ものづくり・商業・サービス生産性向上促進補助金）の申請で、最初につまずくのが「事業計画書の革新性をどう書けばいいか分からない」という壁です。

補助金の審査員が見ているのは「この事業者が本当に新しいことをやろうとしているか」という1点です。「設備を新しくしたい」「生産効率を上げたい」だけでは採択されません。「どんな課題を抱えていて、この補助事業でどう解決し、市場にどんな価値をもたらすか」——この一連のストーリーが論理的に繋がっていることが求められます。

事業計画書のノウハウがない状態で書き始めると、「革新性」「優位性」「実現可能性」の各セクションがバラバラになり、書き直しを繰り返すうちに公募締め切りが迫る——という展開は補助金申請あるあるです。

これ、**Claude Codeで「審査項目ごとの評価視点」をCLAUDE.mdに固定しておけば、事業の概要を投入するだけで骨子が揃います**。審査員の視点で「ここが弱い」を事前に指摘させることもできるため、提出直前の大幅修正を大幅に減らせます。

## このワークフローで何ができるか

**ビフォー: 事業計画書の初稿に1週間**（審査項目の調査2日・骨子作成2日・肉付けと整合性確認3日）  
**アフター: 1日**（CLAUDE.md設定30分・骨子生成1時間・審査視点でのチェック・修正3時間・スコアリング確認1時間）

向いている人

* ものづくり補助金（一般型・省力化（オーダーメイド）枠・グローバル展開型）の申請を初めて検討している製造業・サービス業の中小企業経営者
* 過去に申請して不採択になり、事業計画書の何が弱かったのか分からない経営者
* 補助金申請支援（事業計画書のドラフト作成・添削）を顧問メニューにしている中小企業診断士・行政書士

向いていない人

## 手順

### ステップ1: CLAUDE.mdに「ものづくり補助金の審査項目と評価視点」を書く

新規プロジェクトを作り、CLAUDE.mdに今年度の公募要領から取り出した審査項目と評価視点を書きます。ものづくり補助金は年度・枠によって審査項目が変わるため、**必ず最新の公募要領を確認してCLAUDE.mdを更新する**のが原則です。

```
あなたはものづくり補助金の事業計画書ドラフトを作成する専門担当です。
申請者の事業概要を受け取ったら、下記の審査項目に沿った骨子を作成してください。

【対象補助金】
ものづくり・商業・サービス生産性向上促進補助金（一般型）
補助上限: 最大1,250万円（補助率1/2、小規模事業者2/3）
※ 申請する枠・年度の公募要領で必ず最新情報を確認すること

【審査項目と評価視点（2026年度版・概要）】

■ 技術面
- 補助事業の具体的内容: 「何を・どのように・なぜ」が明確か
- 革新性: 従来手法との違いが具体的に示されているか（「業界初」「自社初」のレベルで可）
- 優位性: 競合と比較したときの強みが1〜2行で説明できるか

■ 事業化面
- 事業化見通し: 補助事業終了後2〜3年以内に収益化できるか、数字で示せるか
- 市場の規模・競合: ターゲット市場の規模と競合状況が示されているか
- 実施体制: 誰が・いつまでに・何をするかのスケジュールが現実的か

■ 政策面
- 賃上げ・雇用: 補助事業を通じて賃上げ・雇用拡大に繋がるか
- 付加価値額・生産性向上: 補助事業後の付加価値額増加目標が示されているか

【骨子の構成（セクション別）】
1. 補助事業の概要（3〜5行）: 何を・誰に・どんな技術/設備で実現するか
2. 解決する課題と現状（5〜8行）: 今どんな問題を抱えていて、なぜ解決が必要か
3. 補助事業の内容と革新性（5〜8行）: 具体的な取り組みと、従来手法との違い
4. 市場分析と事業化見通し（5〜8行）: ターゲット市場・競合・収益化のシナリオ
5. 実施スケジュール（箇条書き）: 月単位で「何をいつまでに」
6. 付加価値額・賃上げ目標（数字）: 補助事業終了後3年間の目標値

【NG表現（採択率を下げる代表例）】
- 「革新的な取り組みを行います」（革新の中身がない）
- 「業務効率化を実現します」（何をどの程度効率化するか数字なし）
- 「市場ニーズが高い」（根拠なし）
- 「品質向上が見込めます」（比較基準と向上幅がない）
```

![](https://assets.st-note.com/img/1780877884-wZOJYUghSl1M5Ckqry6pxj8K.jpg?width=1200)

[画像: VS Codeで開いたCLAUDE.md（審査項目・NG表現）のスクショ]

### ステップ2: 事業の概要を箇条書きで投入してドラフトを生成する

CLAUDE.mdが設定された状態で、自社の事業概要を箇条書きで渡します。完璧な情報でなくてもよく、「あるだけ全部出す」ことを優先します。

```
以下の事業概要をもとに、CLAUDE.mdの審査項目・構成に沿って
ものづくり補助金（一般型）の事業計画書骨子を作成してください。

【申請者の事業概要】
- 会社名: 株式会社○○製作所（架空）
- 業種: 金属加工業（プレス加工・板金溶接）、従業員20名
- 補助事業の内容: AI搭載の画像検査装置を導入し、不良品検査を自動化する
- 課題: 目視検査に熟練工2名が1日4時間を費やしており、検査精度にばらつきがある。熟練工の高齢化で後継者不足が深刻
- 革新性: 従来は目視のみだったが、AIカメラ＋自社開発の判定アルゴリズムで「不良率0.3%未満」の精度を目指す
- 投資対象: AI画像検査システム（機器費用700万円・ソフト開発費200万円）
- 期待効果: 検査工数50%削減・不良品流出率を現行の1.2%→0.3%未満に改善
- 目標市場: 自動車部品メーカー（既存取引先3社・新規開拓5社を3年以内に目標）
- 賃上げ計画: 補助事業終了後に検査担当2名の業務転換＋賃上げ（年収5%増）

骨子なので各セクションは200字以内のプレースホルダー形式で出してください。
肉付けが必要な箇所は [要追記: ○○] と明示してください。
```

### ステップ3: 「革新性」セクションを審査員視点で強化する

事業計画書の採否を分ける最大のポイントが「革新性」セクションです。骨子で出た内容を、審査員が「採択に値する」と判断する強度まで引き上げます。

```
ステップ2で作成した骨子の「補助事業の内容と革新性」セクションを
以下の観点で強化してください。

【革新性を強化する4つの視点】
1. 業界比較: 「業界標準は○○だが、本事業では△△を行う」の形式で、従来との差を1行で示す
2. 自社比較: 「自社では過去に○○方式をとっていたが、今回は△△に変える」の形式で変化を示す
3. 数値の対比: 現状（不良率1.2%・検査工数4時間/日）→目標（不良率0.3%・工数2時間/日）を表形式で示す
4. 実現可能性の根拠: なぜこの数値が達成できると考えているか（過去の試験データ・導入事例等）を1〜2行で添える

出力形式: 書き換えた「革新性」セクション（400字以内）＋「採択リスクがある弱点とその補強案」を3点箇条書き
```

「採択リスクがある弱点」を事前に出させることで、申請前に潰せる穴が見えます。審査員が指摘する前に自分で発見できる状態になります。

### ステップ4: 付加価値額・スケジュールを数字で固める

ものづくり補助金では「補助事業後の付加価値額」が採択条件の一つになります。骨子に数字を入れる段階で、条件を満たしているかClaude Codeに検算させます。

```
下記の数値前提で、付加価値額の3年間試算とスケジュールを作成してください。

【数値前提】
- 現状の年商: 8,000万円・営業利益率8%
- 補助事業後の目標: 検査工数削減で原価率を3%改善、新規受注5社×単価500万円/年を追加
- 設備投資総額: 900万円（補助金450万円・自己負担450万円）
- 従業員数・賃上げ計画: 検査担当2名を品質保証業務に転換、年収5%増（1人当たり約20万円増）

出力形式:
1. 付加価値額（売上－外注費－材料費）の1〜3年目の試算（表形式）
2. 付加価値額増加率（補助金申請の要件：年率3%以上が目安）の確認
3. 実施スケジュール（月単位・12ヶ月）：設備発注→導入→試運転→本稼働→効果測定
```

付加価値額の増加率が要件を下回っている場合、ここで気づいて計画を修正できます。提出直前に数値が合わないと発覚するよりはるかに安全です。

## 応用と注意点

**応用1: 枠が変わってもCLAUDE.mdの差し替えで対応**  
ものづくり補助金には一般型のほか「省力化（オーダーメイド）枠」「グローバル展開型」など枠があり、審査項目と補助上限が異なります。CLAUDE.mdの【審査項目】ブロックを各枠の公募要領に合わせて書き換えるだけで、同じワークフローで対応できます。

**応用2: 不採択になった事業計画書を次回申請用に改善する**  
過去の不採択事業計画書をClaude Codeに読ませ、「採択リスクがある箇所を3点指摘して改善案を出して」と依頼すると、何が弱かったかの分析が出ます。審査コメント（不採択通知に記載される場合がある）を合わせて渡すと、より精度の高い改善案が出ます。

**注意点1: 公募要領は毎回必ず最新版を確認する**  
ものづくり補助金の審査項目・補助上限・補助率・申請要件は、年度・枠・回次によって変わります。CLAUDE.mdに書いた情報が古い場合、NG要件を満たさない申請書になるリスクがあります。公募開始のたびに中小企業庁・ものづくり補助事業公式サイトで最新の公募要領を確認してからCLAUDE.mdを更新してください。

**注意点2: 提出前に認定支援機関のレビューが必須**  
ものづくり補助金の申請には、原則として**認定経営革新等支援機関（認定支援機関）による確認書**が必要です。Claude Codeで事業計画書のドラフトを仕上げた後、税理士・中小企業診断士・金融機関など認定支援機関のレビューを受けてから最終提出してください。AIが作ったドラフトをそのまま提出することは避けてください。

**注意点3: 採択を保証するものではない**  
Claude Codeは事業計画書の「構成・表現の整理」を行うツールです。採否は申請者の事業内容・競合する申請件数・審査員の評価など、書類以外の要素も含めた総合判断です。「採択が保証される」ものではありません。

## まとめ

* ものづくり補助金の事業計画書は **CLAUDE.mdに審査項目・NG表現・構成テンプレを固定**してから骨子を生成すると、1日でドラフトが揃う
* 「革新性」セクションは「業界比較×自社比較×数値対比×実現可能性」の4視点で強化する
* 付加価値額の増加率（年率3%以上が目安）はClaude Codeに事前に検算させて、提出前に要件確認を完了させる
* 公募要領の確認・CLAUDE.mdの更新・認定支援機関のレビューは必須ステップ

「何から書けばいいか分からない」で止まっている事業計画書は、**Claude Codeで骨子の型を先に作ってしまう**と動き出します。骨子が揃ったあとの肉付けは、自社の強みと数字を入れるだけです。補助金の締め切りが迫ってから慌てないために、骨子だけでも早めに作っておくことをおすすめします。
