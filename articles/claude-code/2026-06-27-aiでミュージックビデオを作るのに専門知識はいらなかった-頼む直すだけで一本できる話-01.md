最初に、いちばん大事なことを書きます。

私がやったことは、**「Higgsfield を使ってMVを作りたい」と Claude に頼んだ。それだけ**です。

キャラクターの設定画、各カットの絵、動画化、編集ソフトでの字幕・トランジション・色補正・演出 ── こういう"いかにも難しそうな工程"は、全部 Claude の方から「次はこれをやりましょう」と提案してきました。私がやったのは、出てきたものに「いいね」「ここが違う」と返すこと。つまり**壁打ち**だけです。

この記事は、その手軽さと、なぜそれで良いものができるのかを伝えるために書きました。専門用語もいくつか出てきますが、**それらは全部 Claude が勝手にやってくれる**ので、あなたが覚える必要はありません。「そういう名前のものがあるんだな」くらいで読み流して大丈夫です。

---

## 半分は「驚くほど簡単」、もう半分は「だから高品質」

この記事で伝えたいことは2つあり、半々です。

**前半:こんなに簡単だった。** 専門知識ゼロで、頼むだけで一本できる。 **後半:でも"全自動"ではダメだった。** 一手ごとに確認して直す「壁打ち」があるから、見られる品質になる。

実は私、以前に**全自動でMVを作る仕組み**を組んだことがあります。曲を入れたら最後まで一気に完成する、理想的に見えるやつです。でも、結局**使い物にならない品質**でした。理由ははっきりしていて、**「途中で確認して、違ったら直す」工程が無かったから**です。AIは一発で完璧を出しません。今のAIのレベルでは、**ステップごとに人間が確認して直していく流れが、最適解**なんです。

だからこの記事は、「簡単だよ」と「ちゃんと関われば高品質になるよ」を、両方伝えます。

---

## 実際の流れ:私は何を「頼んだ」だけだったか

順番に、私が実際にやったことを書きます。専門的に見える工程も、私の側の行動は「頼む」と「返事する」だけだった、というのが分かると思います。

### 1. 最初に渡すのは、4つだけ

スタートは驚くほど身軽です。私が Claude に渡したのは、この4つだけでした。

私の場合は曲を Suno(AIで作曲できるサービス)で作っているので、曲名・スタイル・歌詞は**もともと手元にあるもの**。それを貼るだけです。

そしてもう一つ、**歌詞のタイミングデータ**（どの歌詞が何秒に出るか）も一緒に渡しました。これは字幕を曲にピッタリ合わせるためのものです。

ここで一つ、私なりのこだわりを共有します。音声を読み込んで**タイミングを自動で判定してくれる仕組み**もあります。便利なのですが、私は個人的に、たまに変なタイミングになって、その**手直しで泥沼にハマった**経験が何度かありました。だから今は、歌詞タイミングを作るツールで**自分でタイミングを用意して渡す**ようにしています。

とはいえ、ここで身構える必要はありません。タイミングデータが無ければ「無いので音声から推定して」と頼めば Claude が割り出してくれます。自分で用意したいなら専用のツールで作ればいい。**どちらでも進められます。** 私は「ズレを後で直すストレスが嫌だから自分で作る派」というだけです。

要するに、最初に渡すのは **曲名・スタイル・歌詞・音源**（＋あれば歌詞タイミング）。**Sunoで曲を作った人なら、ほぼ手元にあるものばかり**です。

### 2.「Higgsfieldを使ってMVを作りたい」と頼んだ

渡したうえで、私が言ったのはシンプルにこれです。「この曲でMVを作りたい。9:16の縦型で」。それだけです。

Higgsfield(ヒッグスフィールド)は、画像や動画を生成してくれるサービスです。Claude にこれをつなぐと、Claude が直接、絵や映像を作れるようになります。

つなぎ方も、難しいことはありません。Claude Code(あとで説明します)を使うなら、表示された通りの一行をターミナルに貼るだけ。

```
claude mcp add --transport http --scope user higgsfield https://mcp.higgsfield.ai/mcp
```

※この「MCP」というのは、要するに**Claudeと外部サービスをつなぐ接続口**のこと。意味は分からなくて大丈夫です。「これを貼ればHiggsfieldが使えるようになる」とだけ知っていれば十分です。

### 3. 企画は、Claudeと一緒に「壁打ち」して決めた

すると Claude が、曲を解析して(テンポや盛り上がりの位置を勝手に調べてくれます)、**企画案を出してきました**。

ここが、いちばん人間が関わるべきところです。私は最初に出てきた案に何度も「ピンと来ない」と返しました。Claudeも何度か的外れな案を出しました。でも、やり取りを続けるうちに、**この曲の本当の芯**にたどり着けたんです。

正直に言うと、これは一発では決まりませんでした。むしろ何度もぶつかった。でも、**そのぶつかり合い(壁打ち)こそが、企画を良くした**んです。最初の案で「まあいいか」と進めていたら、ありきたりなものになっていました。

「いいね」「違う」「もっとこういう感じ」── この往復を恐れないこと。それが品質の9割を決めます。

### 4. キャラクター設定画も、頼んで・見て・直しただけ

企画が決まると、Claudeが「まずキャラクターの設定画を作って、全カットで同じ顔・同じ服が出るようにしましょう」と提案してきました。

ここで「Element(エレメント)」という仕組みが出てきます。**一度キャラを登録すると、以降ずっと同じ見た目で出せる**機能です。これも Claude が勝手にやってくれます。私は出てきた設定画を見て、「この感じでOK」と返しただけ。

「主人公はこういう雰囲気で」と注文すれば、その通りに作り直してくれます。

### 5. 各カットの絵も、難しいものから順に確認した

次に、Claudeが各カットの「1枚目の絵」を作っていきます。私はここでも、出てきた絵を見て当たり外れを伝えるだけ。

賢いな、と思ったのは、**難しい絵から先に作って確認させてくれた**ことです。いきなり全部作るのではなく、いちばん表現が難しいカットを2枚ずつ出して、「どっちがいい?」と聞いてくる。OKが出てから、残りを一気に作る。これだと、方向がズレていても無駄になりません。

実際、ある重要なカットは一発で決まらず、私が「ここがイメージと違う」と言うと、Claudeが「では構図を変えて」と作り直してくれました。**この"違うと言える"のが、全自動には無かったものです。**

### 6. 動画化も、おまかせ

絵が揃うと、Claudeがそれを動画にしていきます(静止画を動かす、と思ってください)。

ここでも私の作業はありません。たまに「二人が手をつないでしまっている」みたいな失敗が出るので、それを指摘すると直してくれる。指摘して直す、の繰り返しです。

### 7.「Claude Codeで、Remotionで編集して」と頼んだ

最後の仕上げです。ここで私が言ったのは、ほぼこれだけ。

**「Claude Code で、Remotion を使って編集して。字幕・字幕アニメーション・トランジション・カラーグレーディング・CTA(最後に曲名とアーティスト名)を入れて」**

用語を一応説明すると ──

…と書きましたが、**これも全部 Claude がやってくれます**。私は「編集して」と頼んで、出てきたものを見て、「この字幕の出方いいね」「ここの色もう少し」と返しただけ。Remotionのコードを自分で書いた、なんてことは一切ありません。

「Claude Code で Remotion で編集して」── 本当に、この一言で編集工程が動き出します。

### 8. 書き出して、完成

最後に「書き出して」と頼めば、9:16のフル尺MVが一本、出来上がります。SNSにもYouTubeにも出せる形で。

## つまり、あなたがやることはこれだけ

工程は7つありましたが、**あなたの行動は2種類しかありません**。

1. **頼む**(「MV作りたい」「キャラ設定画作って」「Claude Codeで編集して」)
2. **返事する**(「いいね」「これは違う」「もっとこう」)

絵を描く技術も、動画編集ソフトの知識も、プログラミングも要りません。難しそうな部分は、**頼めばClaudeが提案してきて、勝手にやってくれる**。あなたは作品の"監督"として、出てきたものに反応するだけです。

---

## なぜ「全自動」ではダメで、「壁打ち」が必要なのか

ここが、もう半分の大事な話です。

「そんなに簡単なら、全部自動でやればいいじゃないか」と思うかもしれません。私もそう思って、以前に**完全自動の仕組み**を作りました。曲を入れたら、企画もキャラも絵も動画も編集も、人間が一度も口を挟まずに完成する。

結果、**品質は使い物になりませんでした。**

理由は、今ならはっきり分かります。**AIは一発で完璧を出さないから**です。企画は最初の案がベストとは限らない。絵も動画も、何枚かに一枚は失敗が混じる。全自動だと、その失敗をそのまま積み上げて完成にしてしまう。誰も「これは違う」と言わないから。

逆に、**一手ごとに確認して、違ったら直す**。この壁打ちを挟むだけで、品質が段違いになります。企画でぶつかったからこそ芯が見つかり、絵を「違う」と言えたからこそ世界観が揃い、動画の失敗を指摘できたからこそ破綻が消えた。

**確認して直す。これが、今のAIで良いものを作るための、ほぼ唯一にして最強のコツです。** 難しい技術ではなく、ただ「気になったら言う」だけのこと。それができる人間が一人いるだけで、AIの出力は作品になります。

## まとめ

* AIでMVを作るのに、専門知識は**いらない**。Claudeに「Higgsfieldを使ってMVを作りたい」と頼むところから始まる
* キャラ・絵・動画・編集といった難しそうな工程は、**Claudeが提案して、勝手にやってくれる**。あなたは「いいね/違う」と返すだけ
* 「Claude Codeで、Remotionで編集して」と頼めば、字幕も色も演出も組み上がる
* ただし**全自動はダメ**。一手ごとに確認して直す「壁打ち」があるから、見られる品質になる
* あなたの役割は、技術者ではなく**監督**。気になったら「違う」と言う。それが品質を作る

道具は驚くほど簡単になりました。でも、最後に作品を決めるのは、出てきたものに「これは違う」と言えるあなたの目です。そこだけは、AIに任せず、あなたが握っていてください。

難しく考えず、まずは曲を一つ、Claudeに渡してみてください。「これでMV作りたい」と。そこから先は、対話しながら、一緒に作っていけます。
