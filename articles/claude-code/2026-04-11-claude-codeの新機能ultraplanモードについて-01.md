Claude Codeに「ultraplan」という新しいプランニングモードが追加されたので、概要・使い方・通常のPlanモードとの違い・注意点をまとめます。

なお、ultraplanは現在 research preview であり、Claude Code v2.1.91 以降で利用できます。仕様は今後のフィードバックで変わる可能性があります。

## ultraplan とは何か

ひとことで言うと「ローカルCLIから計画タスクを、クラウド上で動く Claude Code on the web のセッションに丸投げする機能」です。Plan mode で走るクラウド側の Claude がリポジトリを調べて計画ドラフトを書いている間、自分は手元のターミナルで別の作業を続けられます。

計画ができあがったらブラウザで開き、セクション単位でコメントや修正依頼を出し、最終的に「そのままクラウドで実装させる」か「ターミナルに戻してローカルで実装する」かを選べます。

## 通常のローカルPlanモードとの大きな違い

- 計画の生成場所：ローカルではなくクラウド（自分のターミナルは自由）  
- レビュー体験：ターミナルのチャットではなく、ブラウザ上のドキュメント的UIでインラインコメント・絵文字リアクション・アウトラインサイドバーが使える  
- 実行先の柔軟さ：計画を承認した後、Webで実装してPR作成するか、ターミナルにテレポートして手元で実装するかを選べる

「ターミナル上で長い計画を一気にレビューして返事する」体験に比べ、「気になった箇所だけにコメントを残してAIに直させる」ドキュメントレビューに近い感覚になります。

## 前提と制約

- Claude Code v2.1.91 以降  
- Claude Code on the web のアカウント  
- GitHub リポジトリと紐づいていること  
- Anthropic のクラウド基盤上で動くため、Amazon Bedrock / Google Cloud Vertex AI / Microsoft Foundry 経由では利用不可  
- クラウドセッションはアカウントのデフォルトの cloud environment で起動する

Remote Control を使っている場合、ultraplan 起動時に自動で切断されます。両方とも claude.ai/code の同じUI面を使うので、同時接続はできません。

## 起動方法は3つ

ローカルCLIセッションから、次の3通りで起動できます。

1. コマンド：/ultraplan の後にプロンプトを書く  
   例：/ultraplan migrate the auth service from sessions to JWTs  
2. キーワード：通常のプロンプトの中のどこかに ultraplan という単語を含める  
3. ローカルPlanからの昇格：Claude がローカルで計画を作り終えて承認ダイアログを出したとき、「No, refine with Ultraplan on Claude Code on the web」を選ぶとそのドラフトをクラウドに送り、そこからさらに練り込める

コマンドとキーワード起動は、実行前に確認ダイアログが出ます。Plan 昇格ルートは「その選択自体が確認行為」なのでダイアログは省略されます。

### ステータスインジケータを読む

ultraplan を起動すると、CLIのプロンプト入力欄に小さなステータスが表示されます。

- ◇ ultraplan ：Claudeがコードベースを調べて計画を書いている最中  
- ◇ ultraplan needs your input ：確認したいことがある。セッションリンクを開いて回答する  
- ◆ ultraplan ready ：計画ができあがった。ブラウザで開いてレビューする

「今クラウド側で何が起きているのか」を、ターミナルを離れずに把握できる作りになっています。

/tasks を実行して ultraplan のエントリを選ぶと、セッションリンク・エージェントの活動状況・「Stop ultraplan」アクションが入った詳細ビューが開きます。Stop するとクラウドセッションはアーカイブされ、インジケータもクリアされます。ターミナルには何も保存されません。

## ブラウザでのレビュー体験

◆ ultraplan ready になったらセッションリンクをクリックして claude.ai に移動します。専用のレビュービューに計画が表示され、次の操作ができます。

- インラインコメント：任意の部分をハイライトしてコメントを残すと、Claude がその箇所だけに対応してくれる  
- 絵文字リアクション：長文を書かずに「OK」「気になる」を伝えられる  
- アウトラインサイドバー：セクション間のジャンプ

コメントをまとめて送ると Claude が計画を改訂し、新しいドラフトを提示してきます。これを好きなだけ繰り返してから、実行方式を決めます。

実行場所を選ぶ：Webか、ターミナルか

計画が固まったら、ブラウザ側でどちらの実行方法を使うかを選びます。

クラウドで実行する場合は「Approve Claude's plan and start coding」を選びます。同じ Claude Code on the web のセッションの中でそのまま実装が始まります。ターミナルには確認メッセージが出てインジケータはクリアされ、作業はクラウド側で続行されます。完了したら Web UI 上で diff をレビューし、そのまま PR を作成できます。

ターミナルに戻して実行する場合は「Approve plan and teleport back to terminal」を選びます。待機していた手元のターミナルで実装できます。このオプションはCLI発信のセッションで、かつターミナルがまだポーリング中のときだけ現れます。選択するとWeb側のセッションはアーカイブされ、並行して走り続けることはありません。

ターミナルには「Ultraplan approved」というダイアログが出て、3つの選択肢から選びます。

- Implement here：今の会話に計画を注入して、直前の続きから実装に入る  
- Start new session：今の会話をクリアして、計画だけを文脈にまっさらな状態で始める。上部に claude --resume コマンドが印字されるので、元の会話にも戻れる  
- Cancel：実装はせず、計画をファイルに保存する。保存先のパスが出力されるので後で戻れる

ローカル環境への完全なアクセスが必要な実装には「ターミナルに戻す」、そのままブランチを切って PR まで出したいなら「クラウド実行」と使い分けられます。

## 向いているケース

### 設計判断が重い・分岐が多いタスク

大規模リファクタ、アーキテクチャ変更、新機能の技術選定 など、トレードオフを複数比較したい場合

### コードベース横断の影響範囲調査が必要

- 「この変更はどこに波及する？」を広く読ませて整理させたい時

### 外部情報との突き合わせが必要

最新ライブラリのドキュメントや記事を Web から引いて設計に反映させたい時（ローカル/planより Web 調査が得意）

### まとまった思考時間が欲しい調査・企画

競合調査、技術書の章構成、プロジェクトの長期ロードマッ プなど、じっくり考えさせたい非コーディング作業 .

### 手元のマシンを占有したくない長時間タスク

クラウド側で走るので、ローカル Claude Code は別作業に使える

### 向いていないケース（ローカル/planで十分）

数ファイルで完結する小さな修正の計画   
「やり方を相談したい」程度の軽い壁打ち   
機密度が高くクラウドに載せたくないリポジトリ

## 実際に触ってみて

実際に自分で触ってみたのですが、

```
ultra plan: session creation failed — Repo is too large
  to
     teleport. Please setup GitHub on
  https://claude.ai/code
```

というエラーがでて自分の環境では使えませんでした。

ターミナルを起動できない環境でレビューができるのは便利そうなので、使ってみたかったですが、Gihubリポジトリの内容を転送する仕組みのようで、容量が大きいと上手く動かないようです。

またクラウドにリポジトリを乗せることになるので、機密性のある情報を乗せたくない時には使わない方がいいでしょう。

まだリリースされたばかりのresearch preview版なので仕方ないですね。今後のアップデートに期待。
