## はじめに

2026年6月15日から、Claude のサブスクリプション（Pro / Max）における **Claude Agent SDK・`claude -p`・GitHub Actions などプログラム経由の利用** が、従来のサブスク利用枠とは別の **月次クレジット制** に移行します。

この変更については情報が錯綜しやすく、特に **「ターミナルで普通に Claude Code を使う分も課金されるのか？」** という点で誤解が生まれやすくなっています。

そこで本記事では、**Anthropic 公式ヘルプセンターの一次情報のみ**を根拠に、

- **何が対象で、何が対象外なのか** を正確に切り分け、
- **「あなたの使い方は対象か？」を判定するフロー**

を提供します。推測は一切書きません。各主張の出典は末尾に記載した公式ページです。

> **本記事はこんな方向け**
> - Claude Code を Max / Pro サブスクで使っている
> - `claude -p` や Agent SDK で自動化・定期実行スクリプト・Bot を動かしている
> - 6/15 から自分が「課金対象」になるのか確認したい

---

## 結論（先に表で）

| 使い方 | 6/15 以降の扱い |
|--------|----------------|
| **ターミナル / IDE で対話的に Claude Code を使う** | **対象外。今まで通りサブスク利用枠** |
| Web / デスクトップ / モバイルの Claude チャット | 対象外。今まで通り |
| Claude Cowork | 対象外。今まで通り |
| **`claude -p`（非対話モード）** | **対象。別枠の月次クレジットを消費** |
| **Agent SDK（Python / TypeScript）で自作プロジェクト** | **対象。月次クレジットを消費** |
| **Claude Code GitHub Actions 連携** | **対象。月次クレジットを消費** |
| Agent SDK 経由で認証する第三者アプリ | 対象。月次クレジットを消費 |

ポイントは **「対話利用かどうか」ではなく「プログラム経由の利用かどうか」** で線が引かれていることです。

## 公式の文言（一次情報）

Anthropic のヘルプセンター記事 [Use the Claude Agent SDK with your Claude plan](https://support.claude.com/en/articles/15036540) に、対象・対象外が明記されています。

**クレジットを消費するもの（対象）:**
- Claude Agent SDK usage in your own projects (Python or TypeScript)
- The `claude -p` command in Claude Code (non-interactive mode)
- The Claude Code GitHub Actions integration
- Third-party apps that authenticate with your Claude subscription through the Agent SDK

**対象外（従来通りサブスク利用枠が適用されるもの）:**
- Interactive Claude Code in the terminal or IDE
- Claude conversations on the web, desktop, or mobile apps
- Claude Cowork

つまり、**ターミナルや IDE で対話的に使う Claude Code は明示的に除外**されています。普段の対話セッションだけを使っている人は、6/15 以降も挙動は変わりません。

## 判定フロー：あなたの使い方は対象か？

```
あなたの Claude Code / Claude の使い方は？
│
├─ ターミナル/IDE で対話しながら使うだけ
│     → ✅ 対象外。今まで通り。何もしなくてよい
│
├─ `claude -p "..."` を cron / スクリプト / Bot から呼んでいる
│     → ⚠️ 対象。クレジットを消費する
│
├─ Agent SDK (Python / TypeScript) で自作アプリを動かしている
│     → ⚠️ 対象。クレジットを消費する
│
└─ GitHub Actions で Claude Code を回している
      → ⚠️ 対象。クレジットを消費する
```

**「対象」に1つでも当てはまる人**は、6/15 までに後述の準備をしておくと安全です。

## クレジットの金額（プラン別）

公式ヘルプセンターに記載された月次クレジット額は以下の通りです。

| プラン | 月次クレジット |
|--------|---------------|
| Pro | $20 |
| Max 5x | $100 |
| Max 20x | $200 |
| Team（Standard seats） | $20 |
| Team（Premium seats） | $100 |
| Enterprise（usage-based） | $20 |
| Enterprise（seat-based Premium seats） | $200 |

このクレジットは、従来のサブスク利用枠とは **別枠** として管理されます。

---

## クレジットを使い切ったらどうなるか

ここが運用上もっとも重要です。公式の記載は次の通りです。

- **`usage credits` を有効化している場合**：月次クレジットを超えた分は標準 API レートで課金され、処理は継続します。
- **`usage credits` を有効化していない場合**：クレジットが切れた時点で **Agent SDK のリクエストは停止**し、次回のクレジット更新まで再開しません。

> When your monthly credit runs out, additional Agent SDK usage flows to usage credits at standard API rates—but only if you've enabled usage credits. If usage credits aren't enabled, Agent SDK requests stop until your credit refreshes.
> （出典：[Anthropic Help Center](https://support.claude.com/en/articles/15036540)）

つまり**デフォルト（usage credits 無効）のままだと、クレジット切れで自動化が止まる**可能性があります。逆に言えば、勝手に高額請求されることはありません。

そのほかの公式記載：
- **繰り越しなし**：未使用クレジットは翌請求サイクルに繰り越されません（"Unused credits don't roll over to the next billing cycle."）
- **個人アカウント単位**：クレジットはチームで共有・プールできません（"Credits belong to individual accounts. They can't be shared or pooled across teammates."）

## 自動化を運用している人が 6/15 までにやること

`claude -p` や Agent SDK で自動化・Bot・定期実行をしている場合、6/15 までに以下を確認しておくと安全です。

1. **自分の使い方が「対象」か判定する**（上のフロー参照）。対話だけなら何もしなくてよい。
2. **Anthropic からの案内を確認する**：6/15 前に届く Anthropic からの案内メールやアカウント設定で、クレジットの扱いに関する最新の手続きを確認しておく。
3. **`usage credits` を有効化するか決める**：
   - 有効化する → クレジット超過後も止まらないが、超過分は API レートで課金される
   - 有効化しない（デフォルト） → 超過すると自動化が止まる（想定外の請求は発生しない）
   - **「止まると困る自動化」か「課金されると困る自動化」か** で選ぶ。
4. **自分の月間消費量を見積もる**：対象の処理がクレジット枠（Max 5x なら $100）に収まるか、API レート換算で概算しておく。

## まとめ

- 6/15 の変更は **プログラム経由の利用（Agent SDK / `claude -p` / GitHub Actions）** を別枠クレジット制にするもの。
- **ターミナル / IDE での対話的な Claude Code は対象外**。普段対話でしか使っていない人は影響なし。
- **自動化・Bot・定期実行をしている人は対象**。6/15 までにクレジットの確認と `usage credits` の方針決定をしておくと安全。
- デフォルトでは超過時に**停止**するので、勝手に高額請求はされない。止まると困るなら `usage credits` を有効化する。

本記事の主張はすべて Anthropic 公式ヘルプセンターの記載に基づいています。仕様は変更される可能性があるため、最終的には必ず公式ページで最新の内容をご確認ください。
