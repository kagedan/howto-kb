## はじめに — なぜ1日27件なのか

僕は一人会社を経営している。プロダクトは9つ。SNSチャネルはX・LinkedIn・Facebookの3つ。単純計算で **9 × 3 = 27件/日** の投稿が必要になる。

これを手動でやるのは不可能だ。1件5分としても2時間以上かかる。しかも毎日。

結論から言うと、今は **朝の5分で全部門を把握し、承認ボタンを押すだけ** の状態になっている。この記事では、そこに至るまでの設計と失敗を共有する。

## 全体アーキテクチャ

パイプラインは3層構造になっている。

```
[コンテンツ生成層]
  Claude Code → プロダクト別投稿文を生成

[スケジューリング層]
  launchd (macOS) → 時間帯・曜日を分散配置

[配信層]
  Autonomos (Sales Cruise) → 各チャネルへ自動投稿
```

### なぜこの構成か

最初はcronを使っていた。しかし **macOSではcronがフルディスクアクセスの問題で全滅する** という罠にハマった。17個のジョブが一晩で全部止まっていた
