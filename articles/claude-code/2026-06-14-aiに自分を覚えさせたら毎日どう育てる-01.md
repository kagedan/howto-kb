# AIに自分を覚えさせたら、毎日どう育てる？ — そして、リポジトリはだんだん散らかっていく

> 連載「外部コンテキストで記憶を補う試み」の2本目。  
> [1本目](https://zenn.dev/hobomokha/articles/9381d939aef2bd)では、AIに自分のことを覚えさせる「秘書リポジトリ」の作り方を紹介した。  
> でも、作っただけでは育たない。この記事では、**毎日どう使って育てるのか**を、実際の流れに沿って紹介する。  
> そして最後に、育てていくと必ずぶつかる「ある問題」の話をする。

## はじめに：箱は作った。で、どう使うの？

[前回の記事](https://zenn.dev/hobomokha/articles/9381d939aef2bd)で、AIに自分のことを覚えさせる「引き出し」を作った。GitHubという場所に、自分の情報を書いたファイルを置いて、Claude Code というツールに読み書きしてもらう仕組みだ。

でも、正直なところ、作った直後はこう思うはずだ。

> 「箱はできた。……で、これ毎日どう使うの？」

ここがいちばん大事なのに、意外と語られない。立派な手帳を買っても、書き方が分からなければ三日で引き出しの奥に消える。あれと同じだ。

この記事では、特別なスキルは一切いらない、**毎日の使い方**だけを紹介する。プログラミングの知識も、難しい用語も出てこない。やることは「話す」と「覚えておいてと頼む」の2つだけだ。

---

## 1. 育て方は、たった2ステップ

AIに自分を覚えさせる作業は、拍子抜けするほど単純だ。

1. **いつも通り、AIに相談する**
2. **話が一区切りしたら「いまの話、覚えておいて」と頼む**

これだけ。すると、AIが会話の内容を整理して、GitHubのファイルに書き残してくれる。次に話すとき、AIはその記録を読んでから答えるので、「昨日の続き」ができる。

たとえば、こんな感じだ。

> **あなた：** 最近、仕事で後輩の指導を任されたんだけど、どう接していいか分からなくて。つい自分でやった方が早いと思っちゃう。
>
> **Claude：** （相談に乗ってくれる。あなたは普通に会話する）
>
> **あなた：** ありがとう。いまの話、覚えておいて。
>
> **Claude：** では `domains/work.md` に、こんな形で残しておきますね。  
> 「後輩指導を任され始めた。『自分でやった方が早い』と感じる傾向がある——任せる練習が今後のテーマ」。これでよければ反映します。

「これでいい」と返すと、AIがファイルに保存する。次に仕事の相談をしたとき、AIは「前に『任せるのが苦手』って言ってましたよね」と覚えた状態で話してくれる。

保存された `domains/work.md` の中身は、こんなふうになっている。

```
## 後輩指導（2024-07〜）

- 指導を任され始めた。「自分でやった方が早い」という傾向を自覚している。
  任せる練習が今後のテーマ `[FLOW]`
- 2024-07-10: プレゼンを任せたら、本人が自分で修正してきた。
  「任せる感触」を初めて掴めた気がする
- 2024-07-15: 本番プレゼン当日。代わりに話してしまった。
  次は本人に最初から話させてみる
```

一言の相談が、こういう「自分の記録」として積み上がっていく。

## 2. 1日の中で、どう使うか

もう少し具体的に、ある人の「ふつうの1日」を覗いてみよう。

### 朝、通勤中に

スマホから、頭の中のモヤモヤを吐き出す。

> 今日のプレゼン、緊張する。去年も似た場面で早口になって失敗した気がする。

AIは過去の記録を読んで、「前にも『緊張すると早口になる』とメモしてましたね。今日は最初の一文だけゆっくり、を意識してみては」と返す。**自分の過去のパターンを、自分より覚えているAI**になっている。

### 昼休みに

ふと思いついたアイデアや、読んだ本の感想を放り込む。

> さっき読んだ記事、『管理しないマネジメント』って考え方が刺さった。覚えておいて。

AIが `domains/reading.md`（読書メモ）に残す。塵も積もれば、自分だけの「気づき集」になる。

### 夜、寝る前に

一日を振り返る。

> 今日のプレゼン、最初ゆっくり話せた。前より落ち着いてた。覚えておいて。

これが翌朝の「去年は失敗した」と繋がっていく。**昨日の自分と今日の自分が、ちゃんと地続きになる。** これが、AIに記憶を持たせることの気持ちよさだ。

## 3. 1週間で、ファイルはこう増えていく

最初は空っぽだったリポジトリ（情報の入れ物）が、1週間も使うとこうなる。

**初日**

```
my-ai-secretary/
├── master_profile.md   ← 自分の基本情報（まだスカスカ）
└── domains/
    ├── work.md          ← 空
    └── life.md          ← 空
```

**1週間後**

```
my-ai-secretary/
├── master_profile.md   ← 価値観や性格が少しずつ書かれてきた
└── domains/
    ├── work.md          ← 後輩指導、プレゼンの話で埋まってきた
    ├── life.md          ← 生活リズム、気分の波のメモ
    ├── reading.md       ← 読書メモ（自分で増やした）
    └── health.md        ← 睡眠や運動の記録（これも増やした）
```

中身も見てみよう。`master_profile.md`（自己理解のファイル）は、1週間でこんな内容が書かれていく。

```
## 仕事・キャリア `[FIXED]`
- IT企業でデータ活用とAI導入を担当するプロジェクトマネジャー
- 技術そのものより「誰かの役に立つ形にする」観点を好む

## 価値観・行動スタイル `[FIXED]`
- 成長の実感を常に求める。停滞と「なんとなく過ごすこと」が苦手
- 物事をグラデーションで捉える。白黒つけるより構造を理解してから動く
- コミュニケーション能力を「意図的に」鍛えてきた自覚がある

## 今のテーマ `[FLOW]`
- 後輩指導：「自分でやった方が早い」という癖を自覚して直している最中
- 関心：生成AIを日常と仕事に組み込む実験を継続中
```

`[FIXED]`（しばらく変わらない情報）と `[FLOW]`（今の状況・定期更新が必要なもの）が混在しているのが分かるだろうか。これがあるだけで、AIが「この情報は今も有効か？」を判断しやすくなる。

ファイルが増えるのは、悪いことではない。**むしろ育っている証拠**だ。話すテーマが増えるたびに、「これは新しいファイルに分けよう」とAIに頼めば、勝手に作ってくれる。

最初の1週間は、たいてい楽しい。話すたびにAIが自分を理解していくのが分かる。「昨日より、ずっと的確なアドバイスが返ってくる」という手ごたえが、毎日ある。

## 4. ……でも、ある日気づく

1〜2週間も経つと、多くの人がある違和感にぶつかる。

> あれ、この話、どのファイルに書いたっけ？

たとえば「転職」について。最初は `work.md` に書いた。でも別の日、気分の話の流れで `life.md` にも書いた。さらに別の日、AIが気を利かせて新しく `career.md` を作った。

**同じテーマが、3つのファイルに散らばっている。** どれが最新で、どれが正しいのか、自分でも分からなくなる。

あるいは、こんなことも起きる。

* 昔「週2で運動する」と書いたのに、今は週4に増えた。でも古い記述が残ったまま
* 似たようなメモが、微妙に違う表現であちこちにある
* 新しいことを書こうとして、「これ、どこに書けばいいんだ？」と毎回迷う

そして、散らかると困るのはあなただけではない。**AIも困る。**

AIは、あなたのファイルを読んで答えてくれる。でも、同じテーマが3つのファイルにあって、しかも内容が食い違っていたら? AIはどれを信じればいいか分からない。古い情報を読んで「週2で運動してますよね」と的外れなことを言い出す。

つまり、散らかったリポジトリは、**せっかくの記憶を、逆に使えなくする。**

---

## 5. 散らかりの正体

なぜ散らかるのか。整理して考えると、原因は3つだ。

1. **同じ話が複数の場所に書かれる**（どこに書くか決まっていないから）
2. **古い情報が消されずに残る**（更新したつもりが、上書きでなく追記になっている）
3. **「正解の置き場所」が無い**（だから毎回迷うし、毎回違う場所に書く）

逆に言えば、この3つを解決する「仕組み」を作れば、散らからなくなる。

!

**散らかりを防ぐ3つの手**

* 「このテーマはこのファイル」という**地図**を作る
* 「これが最新の正解」と決める**1か所**を用意する
* 増えすぎたら**片付ける**ルールを持つ

これが、次回からの話だ。素朴に育ててきたリポジトリを、**散らからない・腐らない仕組みに育て直していく。**

---

## まとめ

AIに記憶を持たせる仕組みは、作って終わりではない。**毎日「話して、覚えておいてと頼む」だけで育つ。** 特別なスキルはいらない。

* 朝のモヤモヤ、昼の気づき、夜の振り返り——ひとこと投げるだけでいい
* 続けるコツは、頑張らないこと
* 1〜2週間で、リポジトリは目に見えて育つ

そして、育てば必ず**散らかる**。同じ話があちこちに、古い情報が残ったまま、置き場所に毎回迷う。でもこれは失敗ではなく、**次の段階に進んだ合図**だ。

次回は、この散らかったリポジトリを、どうやって「整理された、信頼できる記憶」に変えていくか——その第一歩を紹介する。まずは「どのテーマをどこに書くか」の地図を作るところから始めよう。

---

*この記事の「1日の使い方」「散らかっていく様子」は、筆者が実際に体験した過程をもとにしています。最初はみんな散らかります。安心してください。*
