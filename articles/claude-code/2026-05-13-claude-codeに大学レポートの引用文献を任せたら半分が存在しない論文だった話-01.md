## 「あれ、この論文どこにも出てこない」と気づいた深夜

「Claude Codeに大学レポートの引用文献を任せたら、半分が存在しない論文でした」と書くと、SNSの煽りタイトルに見えるんですが、本当にそうだったので、その夜の話を書きます。

CiNiiで検索しても出てこない。Google Scholarで検索しても出てこない。著者名で叩いてもヒットしない。深夜2時、提出は翌日の朝9時。背筋が冷えました。

自分は21歳の理工系大学生で、AIに月$230 払って個人開発のメイン環境にしています。コードを書かせるのは慣れた頃で、レポートの引用集めも軽い気持ちでClaude Codeに振りました。AIに何を任せて、何を渡してはいけないかの、自分の境界線が一段はっきりした夜の話です。

## 整いすぎたリストを、自分は信じた

数日前、理工系の専門科目のレポートを書いていました。特定テーマの先行研究を5本ほど引用してまとめる、よくある形式です。

自分はClaude Codeに「このテーマで、関連論文を5本、タイトル・著者・掲載誌・年号付きでリストアップしてほしい」と頼みました。

返ってきたリストはきれいでした。タイトルはそれっぽい、著者名は実在感のある日本人名、掲載誌も知っている学会誌の名前、年号もちゃんと並んでいる。画面の上で見るぶんには違和感がありません。

このとき自分は、コード生成と同じ感覚でリストを受け取っていました。Plan→Apply→テスト通過、で大体動くClaude Codeに、無意識に同じ信頼を引用文献にも振っていたんです。コードと引用文献は本来、扱いがぜんぜん違うはずなのに。

レポートの本文は、その5本を中心に組み立てて、引用箇所も決め終わっていました。

## 1本目で「あれ」、3本目で「これは違う」

提出前のセルフチェックで、引用元を再確認しようとしたのが救いでした。

1本目をCiNiiで叩く。出てこない。著者名で検索。それらしい論文なし。「あれ」と思って、Google Scholarに切り替える。やっぱり出てこない。

最初は自分の検索の問題だと思いました。タイトルの一語を変えてみる、英語で叩いてみる、著者の所属を推測して大学のリポジトリを覗いてみる。それでも出てこなかったとき、背筋が冷えました。

残りの4本も全部検索しに行きます。3本目で「あ、これも出ない」。4本目で「これは半分以上ダメだ」。5本中、確認できた実在論文は2本だけでした。

椅子の背もたれに体を預けて、しばらく天井を見ていました。このまま提出していたら、捏造の論文を3本、教員に渡していたことになります。学業倫理上、軽い話ではありません。

## なぜ自分は「ある」と思い込んだか

ここから先は、自分の弱さの話です。

Claude Codeのコード生成は、毎日触っていて、Plan→Apply→テスト通過の流れに体が慣れています。テストが通ったという観察装置が常に後ろにあるから、生成物にOKを出すのが軽くなる。その軽さを、自分は引用文献にも持ち込みました。文献は本来CiNii/Scholar/論文DBで検証できるのに、コードでいうテストに相当するチェックを、習慣として自分はやっていなかった。

もう一つ、リストの整い方が罠でした。タイトル・著者・掲載誌・年号がきれいに並んだ形式は、それっぽさで脳が「ある」と判定します。Claude は嘘をつくつもりはなくて、整った形式で出してきただけです。それを実在の証拠と取り違えたのは、こちら側の癖でした。

コードと引用文献は、AIに渡すと挙動が似ているのに、検証する習慣の有無で結果が180度違う。

## 火傷してから引いた、自分の3つのルール

あの夜のあと、自分は3つのルールを置きました。

ひとつ、AIに引用文献を出させたら、例外なく1本ずつCiNii/Google Scholar で検索して実在確認する。抜け道は作らない。

ふたつ、AIに頼むときに「実在を保証しないから、自分で確認する前提で出して」と明示する。AIの態度が変わるわけではないんですが、自分の側のスイッチが入る。

みっつ、コードと引用文献は別物として扱う。コード生成の信頼感を、検証習慣のない領域に持ち込まない。

自分はAIに月$230 払っています。それでも、こういう失敗を踏んでルールを足していく日があります。課金の内訳と「警戒しながらすぐ触れ」のロジックは、別の記事に書きました。AIに任せていい/任せたらダメの境界線は、たぶん全員、いちど火傷してから引き直すものだと思っています。

レポート提出の前に、引用元の検索ボタンを押す癖。今のところ、自分にとってはこれが一番安いリスクヘッジです。
