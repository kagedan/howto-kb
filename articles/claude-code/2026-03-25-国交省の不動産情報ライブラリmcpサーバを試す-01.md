### 概要

利用可能なAPIの詳細は README に記載されているので割愛。  
サーバーは5つのツールを公開しています。

1. `get_land_price_point_by_location` — 地価公示・地価調査の専用ツール
2. `get_urban_planning` — 都市計画区域・区域区分の専用ツール
3. `get_zoning_district` — 用途地域の専用ツール
4. `get_multi_api` — 複数APIを統合的に呼び出すツール（全25種対応）
5. `plateau_space_id` — PLATEAU空間ID計算ツール

`claude_desktop_config.json` に MCP の設定を追加

```
  }

```

### 内部処理

興味のある方向け。MCP サーバーが質問を受け取ってからデータを返すまでの処理フローを追ってみます。

内部処理

#### 全体フロー

```
handle_call_tool (server.py)
  ↓ ツール名からAPI仕様を取得
build_payload (utils/payload.py)
  ↓ 引数を内部payload形式に変換
handle_request (request_processor/handler.py)
  ↓ RequestModelにバリデーション
GeospatialService.process_request (service/geospatial_service.py)
  ├── converted_coordinate: 緯度経度 → XYZタイル座標に変換
  ├── APIEnum.from_code → get_instance: 各APIクラスをインスタンス化
  ├── asyncio.gather: 複数APIを並列実行
  ├── build_map_url: 不動産情報ライブラリ地図URLを生成
  └── 結果を返却
```

#### 緯度経度 -> XYZ タイル座標変換

この MCP の中核は緯度経度を XYZ タイル座標に変換する処理だと思います。

src/utils/coordinates\_conversion.py

```
def latlon_to_tile_fraction(lat: float, lon: float, zoom: int):
    n = 2**zoom
    lat_rad = radians(lat)
    x_float = n * ((lon + 180) / 360)
    y_float = n * (1 - (log(tan(lat_rad) + 1 / cos(lat_rad)) / pi)) / 2

    x = floor(x_float)
    y = floor(y_float)

    # タイル内での相対位置（小数部分）
    x_frac = x_float - x
    y_frac = y_float - y

    return x, y, x_frac, y_frac
```

ズームレベル 15 固定で計算 (const.py ZOOM)、タイル内の相対位置 `x_frac` と `y_frac`は周辺タイルの取得方向を決定するために使われている様子。

#### ポイントデータ: 周辺4タイル取得 -> 距離フィルタ

ポイントデータは周辺4つのタイルを取得。

![](https://static.zenn.studio/user-upload/3153ab6db767-20260324.png)

指定座標がタイル内のどの象限にあるかで、2×2 の取得方向が決まります。座標に最も近い角の方向に広げることで、タイル境界付近のデータも漏れなく取得します。

src/request\_processor/common/point\_filter.py

```
def get_surrounding_tiles(x, y, x_frac, y_frac):
    if x_frac < 0.5 and y_frac < 0.5:
        tiles = [(x, y), (x - 1, y), (x - 1, y - 1), (x, y - 1)]
    elif x_frac >= 0.5 and y_frac < 0.5:
        tiles = [(x, y), (x + 1, y), (x + 1, y - 1), (x, y - 1)]
    elif x_frac >= 0.5 and y_frac >= 0.5:
        tiles = [(x, y), (x + 1, y), (x + 1, y + 1), (x, y + 1)]
    else:
        tiles = [(x, y), (x - 1, y), (x - 1, y + 1), (x, y + 1)]
    return tiles
```

タイル内の相対位置に応じて、中心タイル＋隣接3タイルの計4タイルを選択します。タイル境界付近のデータも漏れなく取得するための工夫です。

取得後は WGS84楕円体距離（pyproj の `Geod`）で実距離を計算し、指定距離以内のポイントのみを返します。

#### ポリゴンデータ: 1タイル取得 -> 座標重なり判定

ポリゴンは中心1タイルのみを取得。

src/request\_processor/common/polygon\_filter.py

```
def overlap_judge(features, latlon):
    search_lat, search_lon = latlon
    search_geom = f"POINT({search_lon} {search_lat})"
    filtered = []
    for feature in features:
        geom = feature.get("geometry", {})
        multipolygon = geojson_coords_to_multipolygon_wkt(geom)
        result = judge_intersects(search_geom, multipolygon)
        if result:
            filtered.append(feature)
    return filtered
```

Shapely の `intersects` を使って指定座標とポリゴン/マルチポリゴンの重なりを判定し、その地点が用途地域や防火地域など、「どの地域に含まれるか」の判定に使用されている。

### ユースケース

地理空間MCPサーバー（不動産情報ライブラリAPI）で何ができるか。

ここまでのデモで取得したデータは、不動産・都市計画の専門家だけが使うものではありません。聞き方を変えるだけで、日常の判断に役立つ情報として活用できます。

#### 地価公示データ: 街の相場感を知りたい

地価公示は国土交通省が毎年1月1日時点の標準地の価格を公示するもので、不動産取引の指標になるデータです。

活用例:

* 引っ越し先の家賃相場の目安: 地価が高いエリア ≒ 家賃も高い。デモ6のように東京・大阪・福岡・札幌を比較すれば、どの都市のどのエリアが自分の予算に合うか一目でわかる
* 再開発エリアの発見: 前年比の上昇率が高い地点は、再開発やインフラ整備が進んでいるサイン。札幌の北2条東（+14.4%）や福岡の博多駅前（+12.9%）など
* マイホーム購入の比較検討: 同じ予算で「東京の郊外」と「札幌の中心部」のどちらが広い土地を買えるか、といった数字での比較が可能

#### 都市計画区域・区域区分: 住環境の判断材料に

* 都市計画区域:「計画的にまちづくりを進める区域」
* 市街化区域:「積極的に市街化を図るべき区域」
* 市街化調整区域:「市街化を抑制すべき区域」

活用例:

* 土地購入前の基本チェック: 市街化調整区域では原則として建物が建てられない。購入前に「この土地は市街化区域か？」と聞くだけで大きなリスクを回避できる
* 将来性の判断: 都市計画区域に指定されていれば、行政がインフラ整備を計画的に行うため、長期的な住環境の安定が期待できる

#### 用途地域: どんな建物を建てられるか

* 用途地域: 「この土地でどんな建物を建てられるか」を定めた区分

活用例:

* 飲食店の出店判断: 「この場所でカフェを開けるか？」→ 商業地域なら OK、第一種低層住居専用地域だと制限あり
* 住環境の確認: 「隣に工場や大型商業施設が建つリスクはあるか？」→ 用途地域の区分で判断できる
* 建蔽率・容積率の確認: 「どのくらいの大きさの建物が建てられるか」がわかる。デモ5の結果では建蔽率80%・容積率800%で、高層ビルも建てられるエリアだった

#### 防災データ: 災害へ備えるために

防火地域、災害危険区域、液状化傾向図、大規模盛土造成地、地すべり防止地区、急傾斜地崩壊危険区域などを一括取得

活用例:

* 引っ越し先のリスクチェック: デモ1のように「地震に強い？」と聞くだけで7種類の防災データが返ってくる。不動産屋では教えてくれない「液状化しやすい地盤」も確認できる
* 保険の検討材料: 災害危険区域に指定されている場合、火災保険・地震保険の必要性が高まる
* 子どもの通学路の安全確認: 急傾斜地崩壊危険区域や地すべり防止地区を避けた通学路かどうか

### 強み

MCP サーバーの強みは、専門知識や専門用語を使わなくてもこれらのデータを横断的に取得することができる点です。また、従来なら不動産情報ライブラリで1つずつレイヤーを切り替えて確認していた作業が「引っ越し先の安全性と利便性を全部教えて」などユースケースを元に必要なデータを取得できるようになります。

### おわりに

正直、不動産情報ライブラリのデータは専門家向けと考えていましたが、いくつかのデモを通して自分が住んでいる地域や他地域との比較など、ふとしたきっかけで気軽に調べることが出来るツールとして想像していたよりも活用できるシーンがありそうだと感じました。

以前紹介した `mlit-dpf-mcp`（PLATEAU 3Dデータ・橋梁点検等）と合わせると、国交省の地理空間データへのアクセスが MCP で広くカバーされつつあります。`plateau_space_id` ツールも搭載済みで、PLATEAU との連携強化も期待されますね。

また新しいツールが登場したら試してみたいと思います。それでは。
