「Claude の自動化スクリプト、6/15 にちゃんと動き続けるか不安…」

そう思ったあなたへ。

この記事で5分後に、2026-06-15 の Claude 旧モデル廃止で何が起きるか、自分のスクリプトに何をすればいいか、新しい `ant` CLI を試すべきかが分かります。

**結論、2026-06-15 は Claude Sonnet 4 と Claude Opus 4 の旧モデル ID（`claude-sonnet-4-20250514` / `claude-opus-4-20250514`）が API 上で廃止される日です。** 公式告知の対象は API のモデル ID であり、サブスクプラン構造や [Claude.ai](http://Claude.ai) Web の表示については別管理です。やることは1つ — 自動化スクリプトの中でこれらのモデル ID を呼んでいたら、それぞれ `claude-sonnet-4-6` / `claude-opus-4-7` に書き換えてください。

**旧モデルIDとは、API リクエストで指定する `model` 名のことです。** [Claude.ai](http://Claude.ai) Web で見える「Sonnet 4」「Opus 4」というラベルとは別レイヤで、廃止予定の対象は API レイヤの ID（`claude-sonnet-4-20250514` など）です。

## この記事で得られる3つのポイント

* 2026-06-15 に廃止される正確なモデル ID と、推奨移行先

## 4. `ant` CLI 登場の位置付け — `claude-p` ユーザーは移行すべきか

2026-04-08 に Anthropic は新しい **`ant` CLI** をリリースしました。Claude API の公式コマンドラインクライアントで、Claude API との高速インタラクション・Claude Code とのネイティブ統合・YAML での API リソース管理を備えます。

`claude-p` ユーザーが知っておくべきポイントは3つです。

* **強制ではなく選択肢の追加**：既存 `claude-p` 構成のままでも、モデル ID さえ置換すれば動き続けます
* **設定の宣言的管理**：`ant` CLI は YAML で API リソースを管理できるので、cron スクリプト群と相性が良い
* **Claude Code とのネイティブ統合**：開発環境を Claude Code 中心で揃えているなら、シームレスに繋がる

副業ライター視点での判断軸はシンプルで、**既存スクリプトの規模が小さければ `claude-p` 継続でOK、自動化が複数ファイル・YAML 管理したいフェーズに来ているなら `ant` CLI 移行を検討**するのが現実的です。

## 5. Managed Agents は何のためにあるか（深追いはしないコーナー）

同じく 2026-04-08 に**Claude Managed Agents** がパブリックベータでリリースされました。Claude を自律エージェントとして実行するための完全管理ハーネスで、セキュアなサンドボックス、組み込みツール、サーバー送信イベントストリーミングを備えています。

ベータヘッダー `managed-agents-2026-04-01` が必要で、料金体系は既存のサブスクとは別枠。

副業勢の現実的な距離感は次の通りです。

* **現時点では試す必要なし**：個人副業の自動化に必要な機能は `claude-p` / `ant` CLI で大抵カバーできる
* **方向性は知っておく価値あり**：「Claude をエージェントとして長時間走らせる」前提のインフラが整いつつある、というシグナル
* **Enterprise 寄り**：Managed Agents は社内導入・大規模運用前提の機能群
