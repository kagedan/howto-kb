## テクニック1：「Chain of Draft（CoD）」— 思考を段階的に展開させる

通常のプロンプトでは一発で答えを求めますが、CoD（Chain of Draft）は答えを出す前に複数の「草稿」を生成させます。

\*\*プロンプト例\*\*:

```

以下の問いに答える前に、3つの異なる視点からの草稿（Draft 1〜3）を先に作ってください。

その後、最も優れた要素を組み合わせた最終回答を出してください。

問い：[ここに質問を書く]

```

これを使うと、AIが複数の視点を検討したうえで回答するため、単純な「答えを返す」より深い洞察が得られます。ビジネス戦略の検討、記事のアングル選定、採用判断など、重要な意思決定に特に効果的です。

## テクニック2：「Persona Injection」— 専門家の思考回路を注入する

「〇〇の専門家として回答して」という指示は多くの人が知っています。しかしパワーユーザーはさらに詳細なペルソナ設定をします。

\*\*プロンプト例\*\*:

```

あなたはシリコンバレーのスタートアップで10年の経験を持つCMOです。

B2Bマーケティングを専門とし、特にコンテンツマーケティングとSEOが得意です。

過去に3社をシリーズBまで成長させた経験があります。

この前提で、以下のマーケティング課題について助言してください：

[課題を書く]

```

具体的な経歴・実績・専門性を書くほど、回答の質が上がります。

## テクニック3：「Socratic Questioning」— AIに自分を問い詰めさせる

自分のアイデアや計画をAIに批判させることで、盲点を発見できます。

\*\*プロンプト例\*\*:

```

私が今から話す計画について、ソクラテス的問答法で質問してください。

私の思い込みや矛盾を突く質問を5つ挙げてから、それぞれに答えるよう促してください。

計画：[ここに書く]

```

自分では気づかない前提条件や論理の穴が浮き彫りになります。

---

## テクニック4：「Output Calibration」— 出力品質を数値で制御する

\*\*プロンプト例\*\*:

```

以下の文章を書いてください。品質スケールは1〜10で、9以上を目指してください。

まず7の草稿を書き、次にそれを批評し、9以上にするための改善点を指摘してから最終版を出力してください。

テーマ：[ここに書く]

```

この「自己評価→改善」のループを一回のプロンプトに組み込むことで、出力の完成度が大幅に上がります。

---

## テクニック5：「Constraint Forcing」— 制約で創造性を引き出す

意図的に制約を課すことでAIの創造性が高まります。

\*\*プロンプト例\*\*:

```

以下のコンテンツを作ってください。ただし：

- 「AI」という単語を一切使わない

- 冒頭の一文は5語以内

- 全体で使える形容詞は3つまで

テーマ：[ここに書く]

```

制約があると、AIは通り一遍の表現を避けて独創的な言葉を選びます。広告コピーやSNS投稿の作成に特に効果的です。
