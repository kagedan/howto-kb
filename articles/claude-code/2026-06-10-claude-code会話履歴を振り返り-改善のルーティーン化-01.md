Claude Codeを利用しており、日々CLAUDE.mdを更新したり、スキル化したり、サブエージェント化したり、改善しています。

# ワークフロー振り返り

前回レビュー以降のClaude Code会話履歴を分析し、エージェントがより円滑に動くための改善提案を出力してください。

## 手順

1. **対象期間を決定する**
   ```bash
   # 前回レビューのタイムスタンプを取得
   LAST_REPORT=$(ls -t ~/.claude/scripts/reports/review_*.md 2>/dev/null | head -1)
   if [ -n "$LAST_REPORT" ]; then
       SINCE=$(date -r "$LAST_REPORT" -u +%Y-%m-%dT%H:%M:%SZ)
       echo "前回レビュー: $LAST_REPORT"
       echo "対象期間: $SINCE 以降"
   else
       SINCE=$(date -u -v-1d +%Y-%m-%dT00:00:00Z)
       echo "前回レビューなし。昨日以降を対象"
   fi
   ```

2. **対象の会話ファイルを特定する**
   ```bash
   # SINCEより新しいメッセージを含むJSONLファイルを収集
   python3 -c "
   import json, os, glob, sys
   since = '$SINCE'
   targets = []
   for path in glob.glob(os.path.expanduser('~/.claude/projects/**/*.jsonl'), recursive=True):
       try:
           with open(path) as f:
               for line in f:
                   obj = json.loads(line)
                   ts = obj.get('timestamp', '')
                   if ts and ts >= since:
                       targets.append(path)
                       break
       except:
           pass
   for t in sorted(targets):
       print(t)
   "
   ```
   最大15ファイルまで対象にする。15件を超えた場合はその旨をレポートに記載する。

3. **各ファイルの内容を把握する**
   - 対象ファイルから `timestamp >= SINCE` のメッセージのみ読む
   - `user` のメッセージ（何を依頼したか）と `assistant` のメッセージ（何をしたか）を確認する
   - `cwd` フィールドでどのプロジェクトの会話かを把握する

4. **以下の観点で分析する**

   ### スキル化候補
   繰り返し行われるワークフローや定型コマンドパターン。`/skill-name` として標準化できるもの。

   ### agents.md 更新候補
   新しいエージェントの振る舞い・ツールの使い方・サブエージェントとの連携パターンで、記録しておく価値があるもの。

   ### サブエージェント化候補
   独立したサブエージェントに委譲すればメインの会話コンテキストを節約できるタスク。

   ### CLAUDE.md 改善候補
   プロジェクトごとの指示が不足している・不明確な部分。

   ### permissions["allow"] 追加候補
   会話中に繰り返し実行されたBashコマンドのうち、まだ allow リストに入っていないもの。以下の手順で確認する：

   ```bash
   # グローバルと各プロジェクトの現在の allow リストを確認
   cat ~/.claude/settings.json | python3 -c "import json,sys; s=json.load(sys.stdin); [print(x) for x in s.get('permissions',{}).get('allow',[])]" 2>/dev/null
   for f in $(find ~/git -name "settings.json" -path "*/.claude/*" 2>/dev/null); do
     echo "=== $f ==="; cat "$f" | python3 -c "import json,sys; s=json.load(sys.stdin); [print(x) for x in s.get('permissions',{}).get('allow',[])]" 2>/dev/null
   done
   ```

   会話ファイルのアシスタントメッセージから Bash tool_use ブロックのコマンドを抽出し、allow リストに含まれていない・繰り返し登場するコマンドをピックアップする。提案する際は：
   - 追加先（グローバル `~/.claude/settings.json` / プロジェクト `.claude/settings.json`）も明示する
   - ワイルドカード（`*`）で汎用化できるパターンは汎用化して提案する

5. **構造化されたMarkdownレポートを出力する**

## 出力フォーマット

```markdown
# 振り返りレポート - YYYY-MM-DD

## 対象期間
SINCE 〜 現在（対象ファイル数: N 件 / 実際に存在したファイル数: M 件）
※ M > 15 の場合は「⚠️ 上限超過: M 件中 15 件のみ分析」と記載する

## サマリー

## スキル化候補
- 🔴/🟡/🟢 **候補名**: 説明
  - 根拠: 会話中の具体的なパターン

## agents.md 更新候補
- ...

## サブエージェント化候補
- ...

## CLAUDE.md 改善候補
- ...

## permissions["allow"] 追加候補
| コマンドパターン | 追加先 | 根拠 |
|---|---|---|
| `Bash(...)` | global / project | 繰り返し回数・承認が必要だった文脈 |

分析した会話の一覧（プロジェクト名・概要）
```

## 注意

- ファイルの作成・変更は行わない（分析・レポート出力のみ）
- 会話ファイルが見つからない場合は「対象期間の会話履歴なし」と出力する
```

# ルーティーン化

毎朝自分で実行するというのは地味に面倒なので、MacのAutomatorの機能で日次で自動実行するようにしました。

`カレンダーアラーム`でカレンダーのイベントに紐づけてワークフローを実行することができます。ワークフローは、`シェルスクリプトの実行`でClaudeで作成した振り返りスキルを呼び出す用のシェルスクリプトを実行する用に設定するだけです。

# 実際に運用してみて

毎朝レポートを確認してClaudeを改善してから、作業をするということが習慣化できています。日々Claudeを育てている実感があり、作業も円滑になったと実感することも多いです。  
また、私は作業しながらClaude改善も考えて、、と並行で作業することが苦手なので、１日で区切って振り返りをし、改善をするというフローが合っていたなと思います。

## 今後

日々改善していくうちに、おそらく指摘の件数が減るはずなので、指摘件数を減らして、週１振り返りくらいにできればいいなと思っています。そうなると、指摘の件数の遷移を何処かに記録しておくと、Claudeの成長を視覚化できて良さそうですね。

また、今回はClaudeの改善を目的としましたが、会話履歴をもとに人間の改善も目指せるのではないかなと思っています。例えば、会話履歴から人間が理解していなさそう、理解が困難な部分を取り出して、ブログ化候補として提案させたり、しっかり理解できているか問題を作成させるなどです。Claudeと共に人間も成長していきたいですね！
