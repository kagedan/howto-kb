【Claude / Claude Code / Cowork の使い分け】 ※ブクマ推奨

経営者からよく聞かれるので、保存版でまとめます。

Anthropicは、Claudeに触れる方法を3つ用意している。
それぞれ、狙っている仕事がまったく違う。

▶︎Claude（チャット）＝「考える」
▶︎Claude Code＝「作る」
▶︎Cowork＝「やる」

3つとも、月20ドルのプランに全部込み。
いま、最もレバレッジの高いサブスクの一つ。

それぞれ経営者が何にどう使うべきか、続けて深掘る👇

①Claude（チャット）＝「考える」

最も多くの人が知っている、会話型のAI。
プロンプトを打つ → Claudeが応える → 一緒に練り上げる。

経営者の使いどころ：
▸ ラフな構想を、構造化された計画に変える
▸ メール・報告書・提案文の作成
▸ 複雑なテーマのリサーチと要約
▸ 契約書・PDF・資料の分析
▸ Artifactsで、その場で動く試作ツールを作る

ポイントは、すべてが"会話"で進むこと。
AIに作業を渡すのではなく、AIと一緒に思考する場所。

弱点：ローカルファイルへの直接アクセスはない（アップロードが必要）。

②Claude Code＝「作る」

ターミナルで動く、コーディングエージェント。
やりたいことを日本語で伝えると、コードを読み・書き・テストし・エラーを直し・完成まで持っていく。

経営者の使いどころ：
▸ 機能の構築・デバッグを、コードベース全体で
▸ テストの作成・実行・修正を自動で
▸ git操作・PR作成まで
▸ 複数のエージェントを並列で走らせる

CLAUDE.md という設定ファイルに、自社の規約・パターン・制約を書いておけば、"自社のやり方"でコードを書く。

非エンジニアでも、業務システムや自動化の構築に使える。

弱点：チャットより学習コストが高い。重い作業はトークン消費も増える。

③Cowork＝「やる」

最も新しい。「開発以外の、あらゆる仕事のためのClaude Code」。
成果物のイメージを伝えると、計画・実行・納品まで自律でやる。

経営者の使いどころ：
▸ ローカルファイルを直接編集（アップロード不要）
▸ 定型タスクをスケジュールで自動実行
▸ スマホから遠隔でタスクを指示
▸ 画面を直接操作させることも可能

サンドボックスで動き、許可したフォルダしか触れない。コード不要。

弱点：PCを起動したままにする必要がある。

―――

選び方：
▸ 考えたい・書きたい → Claude
▸ 作りたい・自動化したい → Claude Code
▸ 成果物を仕上げてほしい → Cowork

道具を選ぶ前に、"今は考えたいのか・作りたいのか・やってほしいのか"を見極める。それだけ。
