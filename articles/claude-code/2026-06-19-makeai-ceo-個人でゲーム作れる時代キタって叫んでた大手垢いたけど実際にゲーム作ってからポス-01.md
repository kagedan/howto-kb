個人でゲーム作れる時代キタ！って叫んでた大手垢いたけど、実際にゲーム作ってからポストしなさい。僕はただの驚き屋でないから、ど素人ですがUreal Engine 5.8でゲーム作ってみた。これはガチでやべぇぇぇ！！！

Claude CodeのMCP連携だけでここまでいけるとは！

個人ゲーム開発者の逆襲、本日始まった。

Unreal Engine 5.8がMCPサーバー対応で、Claude Code・Codexから直接Editor操作可能に。Blueprints編集、ライティング設定、プロシージャル都市生成、全部AIエージェントに丸投げ。ターミナルで「街灯置いて、キーライト3200Kに」って言うだけで完成。

Unreal Engine × AIの革命を全部解説する↓

まず、何が起きたのか。

【2026年6月17日】
Unreal Fest Chicagoで「Unreal Engine 5.8」が本日リリース。
UE5シリーズの最終メジャーアップデート。

【衝撃の中身】
Unreal Editorに公式MCPサーバーが埋め込まれた。
つまり、Claude Code・Codex・Cursor・Geminiが、Editorを直接操作できるようになった。

ゲーム開発の「人間が画面でクリックする時代」が、構造ごと崩れた瞬間。
ど素人の僕でも、その日のうちにゲーム作れた。

ど素人がやった手順、ガチで簡単すぎ。

【ステップ1：プラグイン有効化】
Edit → Plugins → "Unreal MCP" 検索 → Enable → 再起動
Toolset Registryプラグインも自動でON。

【ステップ2：自動起動設定】
Edit → Editor Preferences → General → Model Context Protocol
「Auto Start Server」をON。
http://127.0.0.1:8000/mcp に自動バインド。

【ステップ3：Claude Code接続】
クライアント設定ファイル生成→プロジェクトルートでclaude起動。
これだけ。所要時間5分。

実際にClaude Codeに投げた指示、これだけ。

【プロンプト1】
「夜の街の3Dシーン作って。中央に大通り、街灯沿いに配置、キーライトを3200Kの暖色で」

【裏で動いたこと】
・Claude CodeがMCP経由でEditorのアクター操作API呼び出し
・街灯メッシュをスポーン→スプライン沿いに配置
・ディレクショナルライトの色温度を3200Kに変更
・全部Editor内部状態を確認しながら実行

【結果】
画面クリック0回、約3分で完成。
従来なら30分以上かかる作業。

PCG Primitive Pluginと組み合わせると、都市1つ生成可能。

【何ができるか】
PCG = Procedural Content Generation（プロシージャル生成）。
MCP経由でClaude Codeが指示すれば、建物・道路・植生・街灯を自動配置。

【僕が試した指示】
「中世風の街を1km四方で生成、城を中心に住宅街・市場・城壁を配置」

【裏で起きたこと】
・PCGノードグラフが自動構築
・Blueprintsで配置ルールを生成
・草・木・小物が自動でばら撒かれる
・ライティングも自動調整
