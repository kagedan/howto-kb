## ブログ・note記事作成での3つの活用パターン

**パターン①：記事のAI臭さチェックスクリプトを作らせた**  
私のブログ・noteには「AI臭い表現を使わない」というルールがあります。このルールをもとにPythonのチェックスクリプトを作らせました。

**パターン②：記事ファイルの管理コマンドを作らせた**  
`ready/`フォルダに投稿待ち記事が溜まってきたとき、ファイル名・タイトル・投稿状況をまとめて表示するスクリプトをClaude Codeに書かせました。

**パターン③：記事構成の叩き台を作らせた**  
「こういう体験を記事にしたい」と伝えると、見出し構成の案を出してくれます。ChatGPTとの違いは、プロジェクトフォルダ内の既存記事を参照しながら「この記事と似た構成にして」という指示ができることです。
