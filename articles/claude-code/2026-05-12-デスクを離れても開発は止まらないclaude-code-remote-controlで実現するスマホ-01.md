「長時間かかるタスクを走らせたまま外出したけれど、進捗が気になる。スマホで確認できたら……」「子どもを寝かしつけながら、手元のスマホで追加の指示を出せたら……」。開発者なら誰しも一度は感じたことのある「デスクから離れると作業が止まる」問題が、Claude CodeのRemote Control機能によってついに解決されました。

本記事では、2026年2月にResearch Previewとして公開され、同年3月6日に全プランへ正式解禁されたClaude Code Remote Controlの仕組みから実践的な使い方、セキュリティモデル、ユースケースまでを徹底的に解説します。公式ドキュメント（英語・日本語）や複数の実践レポートをもとに、この機能を余すところなく掘り下げていきます。

## Remote Controlとは何か——基本概念の整理

### 一言で言うと

**Remote Controlは、ローカルマシンで動いているClaude Codeのセッションを、スマートフォン・タブレット・別PCのブラウザから操作できる機能です。**

claude.ai/codeにアクセスするか、ClaudeのiOS/Androidアプリから接続するだけで、デスクで始めた作業をどこからでも継続できます。

重要なのは「実行場所がローカルのまま変わらない」という点です。スマホから指示を送っても、実際にコードを書いたりファイルを読み書きしたりするのはあなたのローカルマシンです。ファイルシステム、MCPサーバー、プロジェクト設定、CLIにチューニングしたカスタム設定——これらすべてがそのまま活きます。

### Remote Controlが登場した背景

Claude Codeはもともとターミナルで動くCLIツールです。それだけに「PCのそばにいないと何もできない」制約がありました。

このギャップを埋めようとしたサードパーティ製ツールや独自のSSHトンネル設定が存在してはいましたが、設定が複雑だったり、セキュリティ上の懸念があったりしました。Remote Controlはその課題をAnthropicが公式に解決した機能です。

### Remote Control と Claude Code on the web の違い

Remote Controlと似た機能として**Claude Code on the web（ウェブ版Claude Code）**があります。どちらもブラウザやモバイルアプリから利用できますが、根本的な違いがあります。

![](https://assets.st-note.com/img/1778543192-58YvmMTwO2GNCkoWartVg1Al.png?width=1200)

ローカルの開発環境をそのまま活かしたいならRemote Control、リポジトリをクローンせず手軽にクラウドで始めたい・複数タスクを並列実行したいならClaude Code on the webという使い分けが基本です。

### Remote Controlとその他の遠隔操作方法の比較

Claude Codeにはリモートから操作するための手段が複数あります。

![](https://assets.st-note.com/img/1778543245-7AmKhaXT0n1Ux2YOFLPcjdkv.png?width=1200)

Remote Controlは「走行中のセッションをリアルタイムで操縦する」ことに特化しています。Dispatchは「新しいタスクを依頼して自動実行させる」イメージ、ChannelsはTelegramやDiscordからのメッセージに反応させる仕組みです。目的に合わせて使い分けましょう。

## 仕組みとセキュリティ——「PC が丸ごとインターネットに晒されない」安心感

### 接続アーキテクチャ

Remote Controlの接続は以下のように動作します。

![](https://assets.st-note.com/img/1778543347-RYjoH8Vvky3Ap4IP6nQhWw2N.png?width=1200)

1. ローカルのClaude Codeが起動すると、**アウトバウンドHTTPS（ポート443）でAnthropicのAPIサーバーに登録**し、コマンドのポーリング（問い合わせ）を開始します。
2. スマホやブラウザから接続すると、Anthropicのサーバーがその接続リクエストを受け取ります。
3. AnthropicサーバーはWebクライアント（スマホ/ブラウザ）とローカルセッション間で**メッセージをルーティング**します。
4. ローカルマシンは外部からの接続を「受け付ける」ポートを一切開きません。

### なぜ安全なのか

このアーキテクチャには重要なセキュリティ上の特性があります。

**インバウンドポートを開かない設計**が最大の安心ポイントです。通常のSSHサーバーやngrokなどのトンネルツールでは「外部からPCへの接続」を許可するポートが必要ですが、Remote Controlはそれをしません。PCは外部に向かって問い合わせるだけです。

**TLS暗号化**によるすべての通信保護、**短命な認証情報**（用途ごとに異なるトークンで、一定時間で自動失効）、**同時接続は1セッションにつき1リモートのみ**といった多層的な防護があります。

実際に流れるデータはチャットメッセージとツールの実行結果のみで、ファイルの中身やMCPサーバーの設定そのものが外部に送出されることはありません。

## 利用要件——使い始める前に確認すること

### 必要なバージョン

Remote Control機能にはClaude Code **v2.1.51以降**が必要です（モバイルプッシュ通知には**v2.1.110以降**が必要）。

まずバージョンを確認しましょう。

```
$ claude --version
```

古い場合はアップデートします。

```
$ claude update
```

### プラン別の利用可否

![](https://assets.st-note.com/img/1778543518-DVEw5HLnXjPyC1NthR9s0ldf.png?width=1200)

2026年3月6日以降、全プランで利用可能になりました。ただし**TeamおよびEnterpriseプランでは管理者による有効化が必要**です。[管理者は`claude.ai/admin-settings/claude-code`にアクセスし、](http://xn--v9j184r40erye%60claude.ai/admin-settings/claude-code%60%E3%81%AB%E3%82%A2%E3%82%AF%E3%82%BB%E3%82%B9%E3%81%97%E3%80%81)**Remote Control**トグルをオンにします。

APIキーでの利用はサポートされていません。Remote [Controlにはclaude.ai](http://xn--Controlclaude-9g4l3b.ai) OAuthによる認証が必須です。

### 認証の確認

claude.aiアカウントでサインイン済みかどうかを確認します。

```
$ claude
```

起動後、まだサインインしていなければ以下を実行します。

```
/login
```

`claude.ai`オプションを選択して、ブラウザでOAuth認証を完了します。

**注意点**: 環境変数`ANTHROPIC\_API\_KEY`が設定されている場合、APIキー認証が優先されてRemote Controlが使えません。あらかじめ設定を解除するか、`/login`で明示的にclaude.aiでサインインし直してください。

### ワークスペース信頼の承認

プロジェクトディレクトリで`claude`を一度起動し、ワークスペース信頼のダイアログを承認する必要があります。一度承認すれば次回以降は不要です。

## Remote Controlの起動方法——4つのパターンを使いこなす

Remote Controlには用途に応じた複数の起動方法があります。

### パターン1：サーバーモード（`claude remote-control`）

最もシンプルな起動方法です。プロジェクトディレクトリで以下を実行します。

```
$ claude remote-control
```

短縮形は`claude rc`です。

ターミナルにはセッションURLが表示され、スペースキーを押すとQRコードが現れます。このモードでは、ターミナルが「サーバー」として動作し、**リモートデバイスからの接続を待ち受け**ます。

### サーバーモードのオプション

`--name`フラグでセッションに名前をつけられます。

```
$ claude remote-control --name "会員機能リファクタリング"
```

`--spawn`フラグでセッション生成方式を変更できます。

![](https://assets.st-note.com/img/1778543729-Orujwt9XHpELK8Z3ceanS7FY.png?width=1200)

たとえば複数の機能開発を並行したい場合は以下のようにします。

```
$ claude remote-control --spawn worktree
```

`--capacity`で同時セッション数の上限（デフォルト32）も設定できます。

### パターン2：対話型セッション（`claude --remote-control`）

```
$ claude --remote-control
```

または名前付きで起動します。

```
$ claude --remote-control "認証基盤の設計"
```

サーバーモードとの違いは、ターミナルでも通常通りClaude Codeを操作できる点です。**ローカルとリモートの両方から同時に操作可能**な形態です。

### パターン3：既存セッションから切り替え（`/remote-control`コマンド）

作業の途中でRemote Controlを有効にしたい場合、セッション内で以下を実行します。

```
/remote-control
```

短縮形は`/rc`です。名前を付ける場合は引数で渡します。

```
/remote-control 機能Xの実装
```

これにより、それまでの会話履歴を引き継いだままリモートアクセスが有効になります。外出前の最後にサッと実行するのに最適です。

### パターン4：VS Code拡張機能から

Claude CodeのVS Code拡張機能を使っている場合は、プロンプトボックスに`/remote-control`または`/rc`と入力するか、`/`でコマンドメニューを開きます。接続後はエディタ上部のバナーに接続ステータスが表示され、**Open in browser**ボタンから直接セッションへ移動できます。

### 全セッションで自動有効化する方法

毎回コマンドを打つのが面倒な場合は、全セッション自動有効化がおすすめです。

```
/config
```

を実行し、**Enable Remote Control for all sessions**を`true`に設定します。以後、Claude Codeを起動するたびに自動でRemote Controlが有効になります。

## スマホ・別デバイスから接続する——3つの方法

### 方法1：セッションURLをブラウザで開く

ターミナルに表示された以下のようなURLをコピーしてブラウザで開きます。

```
https://claude.ai/code/session_XXXXXX?bridge=env_XXXXXX
```

### 方法2：QRコードをスマホでスキャン

`claude remote-control`起動後にスペースキーを押すとQRコードが表示されます。スマホのカメラでスキャンするとClaudeアプリが起動してセッションに直接接続されます。**最も素早く繋げる方法です。**

### 方法3：セッション一覧から選ぶ

claude.ai/codeまたはClaudeモバイルアプリを開くと、アクティブなRemote Controlセッションが一覧表示されます。コンピュータアイコンと緑のステータスドットがついているものがRemote Controlセッションです。

### Claudeアプリのインストール

スマホにClaudeアプリをまだインストールしていない場合は、Claude Code内で以下を実行するとダウンロード用のQRコードが表示されます。

```
/mobile
```

## モバイルプッシュ通知——「終わったら教えて」が実現

### 概要

Claude Code v2.1.110以降では、Remote Controlがアクティブな状態でスマホへのプッシュ通知が届くようになりました。長時間かかるテストやビルドが完了したタイミングで通知を受け取れるため、頻繁に画面を確認する必要がなくなります。

通知タイミングはClaudeが自動で判断しますが、プロンプトで明示的に指定することもできます。

```
テストが終わったらプッシュ通知で教えて
```

### セットアップ手順

1. ClaudeアプリをiOSまたはAndroidにインストールしてサインイン
2. OSの通知許可プロンプトを承認
3. ターミナルで`/config`を実行し、**Push when Claude decides**を有効化

### トラブルシューティング

`/config`が「No mobile registered」と表示する場合は、Claudeアプリをスマホで一度開いてプッシュトークンを更新します。iOSではフォーカスモードやサマリー設定が通知を遅らせることがあるため、設定→通知→Claudeで確認してください。Androidでは電池最適化がバックグラウンド配信を妨げることがあるため、Claude アプリをバッテリー最適化の除外リストに追加するのがおすすめです。

## 実践ユースケース——こんなシーンで使える

### ユースケース1：長時間タスクを走らせたまま外出する

ビルド、テストスイート、データ処理など数十分かかる処理を依頼してから外出するシナリオです。従来は終わるまで待つかPCのそばにいるしかありませんでしたが、Remote Control＋プッシュ通知を使えば外出中でも完了を把握できます。

```
$ claude --remote-control
> 全テストを実行して、失敗したケースがあれば修正してください。終わったらプッシュ通知で教えて
```

完了通知がスマホに届いたら、アプリから結果を確認して次の指示を送れます。

### ユースケース2：電車の中でコードレビューと追加指示

移動中はPCを開けなくても、スマホのClaude アプリからセッションにアクセスして結果を確認し、追加の指示を送れます。「あのファイルの処理、もう少し丁寧にエラーハンドリングして」といった細かな指示も、テキスト入力だけで完結します。

### ユースケース3：子育て中の隙間時間を活用する

子どもの寝かしつけ中や入浴中など、PCから離れざるを得ない時間でも、スマホ片手にセッションの進捗確認と追加指示が可能です。ソファに座ったまま、あるいは横になりながら開発を継続できます。

### ユースケース4：サーバーモードで複数タスクを並行管理する

`--spawn worktree`オプションを使うと、1台のPCで複数の独立したgit worktreeを展開して並行開発できます。たとえば機能A・バグ修正B・リファクタリングCを同時に進め、それぞれのセッションをスマホやブラウザから管理するといった使い方が可能です。

```
$ claude remote-control --spawn worktree --capacity 5
```

## 制限事項——知っておくべき現在の制約

![](https://assets.st-note.com/img/1778544746-svLGW4IXjTP7ORfKJVg2iwx8.png?width=1200)

特に注意が必要なのは以下の点です。

**ターミナルを閉じるとセッション終了**という点は、最初に戸惑う人が多い制限です。Remote Controlはあくまでローカルプロセスへの「窓」であり、ローカルプロセス自体が止まるとリモートからも何もできなくなります。外出先から長時間操作したい場合は、出発前にローカルのターミナルが開いたままであることを確認してください。

**Ultraplan実行中はRemote Controlが切断される**点も覚えておきましょう。両機能がclaude.ai/codeのインターフェースを占有するため、同時には使用できません。

**一部コマンドはローカルのみ対応**です。インタラクティブなピッカーを使うコマンド（`/mcp`、`/plugin`、`/resume`など）はローカルターミナルからしか機能しません。ただし`/compact`、`/clear`、`/context`などテキスト出力を返すコマンドはスマホやウェブからも使えます。

## トラブルシューティング——よくあるエラーと解決策

APIキーでサインインしています。`claude auth login`を実行してclaude.aiオプションで認証し直してください。`ANTHROPIC\_API\_KEY`環境変数が設定されている場合は先に解除します。

### 「完全スコープのログイントークンが必要です」

`claude setup-token`や`CLAUDE\_CODE\_OAUTH\_TOKEN`の長命トークンで認証されています。`claude auth login`で通常のOAuthセッションに切り替えてください。

### 「組織のポリシーで無効になっています」

チームや企業ユーザーの場合、[管理者が`claude.ai/admin-settings/claude-code`でRemote](http://xn--v8j325r40erye%60claude.ai/admin-settings/claude-code%60%E3%81%A7Remote) Controlトグルをオンにしていない可能性があります。管理者に依頼してください。APIキー認証でもこのエラーが出ることがあります。

### 「リモート認証情報の取得に失敗しました」

`--verbose`フラグをつけて詳細ログを確認します。

```
$ claude remote-control --verbose
```

ファイアウォールやプロキシがポート443のアウトバウンドHTTPSをブロックしている場合に起きやすいエラーです。ネットワーク管理者に確認してください。

## まとめ——「デスクに縛られない開発」の実現へ

Claude CodeのRemote Controlは、単なる「遠隔操作ツール」にとどまりません。ローカル環境の強力さを保ちながら、場所の制約を取り除くという点で、開発者のワークスタイルを根本から変える可能性を持っています。

これまで外出中は「PCが終わるのを待つだけ」だったところが、スマホから状況を確認して次の指示を出せるようになります。通勤時間、育児の合間、夜中のふとした思いつきも、すぐに実行に移せます。

セキュリティ面でもインバウンドポートを一切開かない設計は、ファイアウォール下の開発環境でも安心して使えるポイントです。

現在は全プランで利用可能ですが、機能は活発に進化しています。モバイルプッシュ通知、VS Code拡張機能サポート、サーバーモードの高度なオプションなど、リリースのたびに新しい使い方が生まれています。

まずは`claude remote-control`コマンドを一度試してみてください。スマホとPCを並べてリアルタイム同期を体験すると、きっと「これがなかった頃はどうしていたんだろう」と感じるはずです。
