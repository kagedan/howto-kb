「Claude Code、気になってるけど環境構築でつまずきそう」——そう思って後回しにしている人、けっこう多いと思います。

実際、ターミナル系のツールは最初の30分でつまずくと、そのまま放置になりがちです。この記事では、**インストールから最初のコマンドが動くまで**を、つまずきポイントごと潰しながら進めます。所要時間はだいたい15分です。

## 結論（先に全体像）

* インストールは**公式のネイティブインストーラ**が一番ラク（`npm` ではなくなりました）
* 必要なのは「対応OS」と「Pro / Max などの対象プラン」だけ
* 動かすのは `claude` の一行。最初にやるべきは `/init`
* つまずきの9割は「PATHが通ってない」「プランが対象外」の2つ

逆に言うと、ここさえ押さえれば詰まりません。順番に行きます。

## こんな人向け

* Claude Code をこれから初めて入れる
* 過去に `npm install` 版を試して、なんか古い情報だった気がする
* インストールでエラーが出て放置している

## 事前に確認すること

### 対応OS（公式要件）

| OS | 要件 |
| --- | --- |
| macOS | 13.0 以上 |
| Windows | 10 (1809+) / Server 2019+ |
| Linux | Ubuntu 20.04+ / Debian 10+ / Alpine 3.19+ |

ハードは 4GB RAM 以上、x64 または ARM64。シェルは Bash / Zsh / PowerShell / CMD のいずれかでOKです。

### プランの確認（ここ重要）⚠️

Claude Code は **Pro / Max / Team / Enterprise / Console アカウント**が必要です。**無料の Claude.ai プランでは使えません**。ここを知らずに進めて、ログイン段階で弾かれる人が多いので先に確認しておきます。

## インストール手順

### Step 1. インストーラを実行する

公式の推奨は**ネイティブインストーラ**です。OSごとにコマンドが違います。

**macOS / Linux / WSL:**

```
curl -fsSL https://claude.ai/install.sh | bash
```

**Windows PowerShell:**

```
irm https://claude.ai/install.ps1 | iex
```

> 💡 以前は `npm install -g @anthropic-ai/claude-code` が主流でしたが、現在はネイティブインストーラが推奨です（こちらは自動アップデートが効きます）。Homebrew 派は `brew install --cask claude-code` でもOK。

### Step 2. インストールを確認する

入ったか確認します。

バージョン番号が出れば成功です。もし `command not found` が出たら、PATHの問題です（後述の「つまずき」で対処します）。

もう一段詳しくチェックしたいときは:

これはインストール状態や設定をまとめて診断してくれるコマンドです。後々トラブったときにも使うので覚えておくと便利です。

### Step 3. プロジェクトで起動してログインする

**作業したいプロジェクトのディレクトリに移動してから**起動します。

```
cd path/to/your-project
claude
```

初回はログインを求められます。`/login` を実行するとブラウザが開くので、そこで承認すればOKです（多くの組織ではこれが標準）。

Console の API キーを使う場合は、起動前に環境変数で渡す方法もあります。

```
export ANTHROPIC_API_KEY=sk-ant-...
claude
```

### Step 4. 最初にやるべき `/init`

起動したら、まず実行してほしいのが `/init` です。

これはプロジェクト直下に `CLAUDE.md` を生成するコマンドです。コードベースの構成や規約を読み取って、\*\*毎回のセッション開始時に自動で読み込まれる「指示書」\*\*を作ってくれます。公式も「最も価値の高いセットアップ手順」と書いているくらいで、ここをやるかどうかで体験がかなり変わります。

`CLAUDE.md` の中身の書き方は奥が深いので、別記事で詳しく扱います。今日はまず「生成する」ところまでで十分です。

## ハマったポイント（実際に詰まる箇所）

### `command not found: claude`

インストールは成功しているのに `claude` が見つからない、というやつです。原因はほぼPATH。新しいターミナルを開き直すと直ることが多いです。それでもダメなら、インストーラが入れた `~/.local/bin` がPATHに含まれているか確認します。

```
echo $PATH | tr ':' '\n' | grep '.local/bin'
```

何も出てこなければ、シェルの設定ファイル（`.zshrc` や `.bashrc`）に追記します。

```
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

### `sudo` で入れてしまった

`sudo npm install -g ...` で入れると、後からファイル権限の問題が出ます。公式も非推奨です。やってしまった場合は一度アンインストールして、`sudo` なしで入れ直すのが結局いちばん早いです。

### ログインで弾かれる

前述のとおり、無料プランだと使えません。「設定をいくら見直しても進めない」ときは、だいたいプランが対象外です。アカウントのプランを確認してください。

## つまずいたら `claude doctor`

何かおかしいと思ったら、まずこれです。

インストール経路の競合（古いインストーラの残りなど）や、アップデートの失敗もここで見えます。エラーメッセージで検索する前に、一度叩いてみると原因が早く分かります。

## ここまでで何ができるようになったか

* `claude --version` でインストール確認
* `cd` して `claude` で起動、`/login` でログイン
* `/init` で `CLAUDE.md` を生成

ここまで来れば、コードベースの要約をお願いしたり、テストの失敗を貼って直してもらったり、という「本番の使い方」に進めます。

## 次の一歩：コストとモデルの話

使い始めて多くの人が次にぶつかるのが、**コストとレート制限**です。長いセッションを回していると、思ったより消費が早い。ここで「公式アカウント以外の選択肢ってあるの？」という疑問が出てきます。

実は Claude Code は `ANTHROPIC_BASE_URL` を差し替えることで、**OpenAI互換のサードパーティAPI**経由でも動かせます。私自身は、コストを抑えたりモデルを切り替えたりする目的で、第三者APIゲートウェイ（[EvoLink](https://docs.evolink.ai/en/api-manual/language-series/claude/claude-messages-api?utm_source=community&utm_medium=qiita&utm_campaign=cc_getting_started) など）を併用しています。設定は環境変数1〜2個なので、慣れてきたら試す価値があります。

ただ、これは「入門」の次のステップなので、まずは公式アカウントで一通り触ってからで十分です。第三者API経由のセットアップ手順は別記事で詳しく書きます。

## まとめ

最初に書いたとおり、Claude Code のインストールでつまずく原因はだいたい「PATH」と「プラン」の2つだけです。

1. ネイティブインストーラで入れる
2. `claude --version` で確認、ダメなら `claude doctor`
3. `cd` して `claude`、`/login`、最後に `/init`

この3ステップさえ通れば、もう環境構築で悩むことはありません。あとは実際にコードを触らせてみるフェーズです。
