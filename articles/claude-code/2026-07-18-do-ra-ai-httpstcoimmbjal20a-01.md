# Claude Codeはもうコーディングツールではない

Claude Codeはターミナルでもデスクトップでも動く。ファイルを読み内容を理解しコマンドを実行し外部ツールとまで連携する。

土台には6つの拡張の仕組みがある。CLAUDE.mdでルールを渡す。Skillsで手順を教える。Subagentsで作業を分担する。Slash commandsでワンタッチ実行する。Hooksで自動発火させる。MCPで外部サービスとつなぐ。

この仕組みが揃った結果起きた変化がある。「指示されたことをやる道具」から「自分で仕事を見つけて終わらせる道具」への移行だ。

海外ではこの移行がすでに家庭にも経営の現場にも広がっている。ここから紹介する5つの事例はすべて出典付きの実例だ。

# 事例1: 息子の滑り台タワーを設計した父親

John Conneely氏がこの日Claude Codeに頼んだのはコードではなかった。息子のための滑り台付きタワーの設計図だ。

寸法を決める。材料リストを出す。組み立て手順を段階ごとに分解する。木工DIYの設計は本来構造計算と採寸の知識が要る作業だ。Conneely氏はその工程をそのままClaude Codeに任せた。

出てきたのはコードではなく図面と手順書だった。ここが決定的に違う。

プログラミングツールという看板を持ちながら中身は「複雑な設計を素人でも形にする道具」に変わっている。子どもに何か作ってあげたいが技術がないという悩みは もう技術の問題ではなくなった。

今日下せる判断は一つ。作りたいものがあるならまず寸法と用途をそのままClaude Codeに話してみることだ。

# 事例2: 会議録音から「対立を避けた瞬間」を検出する経営者

Every創業者のDan Shipper氏はさらに踏み込んだ使い方をしている。自分が出席した会議の録音データをClaude Codeに読み込ませ「言うべきことを言わず対立から逃げた瞬間」を洗い出させているのだ。

検出対象は曖昧な言い回しだ。「maybe」「potentially」のようなヘッジ表現。「if that's okay」のような許可を求める言葉。「whatever you think」のような判断の丸投げ。これらが出た箇所を時間スタンプ付きで一覧化する。

複数回の会議を横断して分析すれば自分がどのパターンで逃げているかが実例で見えてくる。時短のためではない。自分を映す鏡としてAIを使っている。

今日下せる判断は一つ。次の重要な会議を録音しその文字起こしをClaude Codeに渡して「自分が言い淀んだ場面」を聞いてみることだ。

# 事例3: 喋るだけで完成する「今日の作戦盤」

XDAのジャーナリストMahnoor Faisal氏は毎朝コーヒーを飲む前にある儀式を続けている。Claude Codeを開きスペースキーを押しっぱなしにして頭の中にあるものをそのまま喋る。

仕事の締め切り。大学の課題。返さなきゃいけないメール。予定を組む電話。細かい用事。順番はバラバラでいい。喋り終えてキーを離すとClaude Codeがその内容をカテゴリ分けし優先度タグを付けたインタラクティブなHTMLページに変換
