岩崎彰憲です。  
AI活用や起業準備、日々の学びを「AIメモ」として記録しています。

今回は、Claude Codeの設定方法についてまとめます。

Claude Codeは、簡単に言うと「パソコンの中のファイルを見ながら、コードを書いたり直したりしてくれるAI」です。

普通のClaudeやChatGPTは、ブラウザで質問して回答をもらう使い方が中心です。

一方でClaude Codeは、ターミナルやVS Codeの中で動き、プロジェクトのファイルを読みながら、コード修正やエラー調査を手伝ってくれます。

ただ、最初に使おうとすると、

「ターミナルって何？」  
「どのコマンドを入れればいいの？」  
「WindowsとMacで何が違うの？」  
「Node.jsは必要なの？」  
「インストールできた後、何をすればいいの？」

となりやすいです。

僕自身も、こういう開発ツールは最初の設定でつまずきやすいと感じています。

この記事では、ITに詳しくない方でもClaude Codeを始められるように、できるだけ順番に整理します。

  

## Claude Codeとは

Claude Codeは、Anthropicが提供しているAIコーディングツールです。

コーディングツールというと難しく聞こえますが、イメージとしては「開発作業を一緒に進めてくれるAIアシスタント」です。

できることは、たとえば以下のようなものです。

・プロジェクトの中身を説明してもらう  
・どのファイルが何をしているか教えてもらう  
・エラーの原因を探してもらう  
・小さな修正をしてもらう  
・READMEなどの説明文を書いてもらう  
・テストコードを書いてもらう  
・Gitの変更内容を確認してもらう

普通のAIチャットと違うのは、Claude Codeがパソコン上のプロジェクトフォルダを見ながら作業できる点です。

つまり、コードをコピーして貼り付けなくても、Claude Codeが必要なファイルを探しながら回答してくれます。

## まず知っておきたい専門用語

設定手順に入る前に、出てきやすい言葉を簡単に整理します。

## ターミナル

ターミナルとは、文字でパソコンに命令を出す画面です。

Windowsでは「PowerShell」や「コマンドプロンプト」、Macでは「ターミナル」という名前で使います。

黒い画面に文字を入力するので最初は怖く見えますが、やっていることは「アプリを起動する」「フォルダを移動する」「インストールする」といった操作です。

## コマンド

コマンドとは、ターミナルに入力する命令です。

たとえば、

```
claude
```

と入力すると、Claude Codeを起動する命令になります。

## プロジェクトフォルダ

プロジェクトフォルダとは、アプリやWebサイトのファイルがまとまっているフォルダです。

たとえば、Webサイトを作る場合、HTML、CSS、JavaScriptなどのファイルが入っているフォルダがプロジェクトフォルダになります。

Claude Codeは、このフォルダの中で起動して使います。

## CLI

CLIは「Command Line Interface」の略です。

難しく聞こえますが、意味としては「ターミナルで操作するツール」です。

Claude CodeはCLIとして使うことができます。

## 事前に必要なもの

Claude Codeを使う前に、以下を用意します。

・Claudeのアカウント  
・パソコン  
・ターミナル  
・作業するフォルダ  
・必要に応じてVS Code

Claude Codeを使うには、Claudeの有料プラン、またはAnthropic Consoleのアカウントが必要になります。

まずは、Claudeにログインできる状態にしておくとスムーズです。

VS Codeは必須ではありません。

ただ、初心者の場合は、コードや変更差分が見やすいので、あとからVS Codeも入れておくと便利だと思います。

## インストール方法はどれを選べばいいか

Claude Codeには複数のインストール方法があります。

代表的なのは以下です。

・Native Install  
・Homebrew  
・WinGet  
・npm

ITに詳しくない方は、まずNative Installを選ぶのが一番分かりやすいです。

Native Installは、Claude Codeを直接パソコンに入れる方法です。

以前はnpmでインストールする説明を見かけることも多かったのですが、今は公式ドキュメントでもNative Installが推奨されています。

npmを使う方法もありますが、その場合はNode.jsが必要になります。

初心者の方は、まずNative Installで進めるのが安全です。

## WindowsでClaude Codeをインストールする方法

Windowsの場合は、PowerShellを使います。

PowerShellとは、Windowsに最初から入っているターミナルのようなものです。

## 1. PowerShellを開く

画面左下の検索欄で、

```
PowerShell
```

と検索します。

「Windows PowerShell」が出てきたら開きます。

## 2. インストールコマンドを貼り付ける

PowerShellに以下を貼り付けて、Enterを押します。

```
irm https://claude.ai/install.ps1 | iex
```

これでClaude Codeのインストールが始まります。

しばらく待つと完了します。

## 3. インストールできたか確認する

インストールが終わったら、PowerShellで以下を入力します。

```
claude --version
```

バージョン番号が表示されれば、インストールはできています。

もし「claudeが見つかりません」のような表示が出た場合は、一度PowerShellを閉じて、もう一度開いてから試してください。

それでも動かない場合は、以下も試します。

```
claude doctor
```

これはClaude Codeの状態を確認するための診断コマンドです。

## MacでClaude Codeをインストールする方法

Macの場合は、ターミナルを使います。

## 1. ターミナルを開く

Launchpad、またはSpotlight検索で、

```
ターミナル
```

と検索して開きます。

## 2. インストールコマンドを貼り付ける

ターミナルに以下を貼り付けて、Enterを押します。

```
curl -fsSL https://claude.ai/install.sh | bash
```

これでClaude Codeがインストールされます。

## 3. インストールできたか確認する

インストール後に、以下を入力します。

```
claude --version
```

バージョン番号が出れば成功です。

もし動かない場合は、ターミナルを一度閉じて、もう一度開いてから試します。

## npmでインストールする方法

Native Installでうまくいかない場合や、すでにNode.jsを使っている人は、npmでもインストールできます。

npmを使う場合は、Node.js 18以上が必要です。

まず、Node.jsが入っているか確認します。

```
node --version
```

数字が表示されればNode.jsは入っています。

次に、Claude Codeをインストールします。

```
npm install -g @anthropic-ai/claude-code
```

ただし、初心者の方は最初からnpmで入れようとしなくていいと思います。

npmは権限エラーが出ることもあるので、まずはNative Installを試す方が分かりやすいです。

## Claude Codeにログインする

インストールができたら、Claude Codeを起動します。

ターミナルまたはPowerShellで以下を入力します。

```
claude
```

初回起動時は、ブラウザが開いてログインを求められます。

Claudeのアカウントでログインします。

ログインが完了すると、ターミナル側に戻ってClaude Codeが使える状態になります。

一度ログインすれば、毎回ログインし直す必要は基本的にありません。

アカウントを切り替えたい場合は、Claude Codeの中で以下を入力します。

```
/login
```

## 最初にテスト用フォルダを作る

いきなり大事なプロジェクトで使うのは少し怖いと思います。

最初は、テスト用のフォルダを作って試すのがおすすめです。

WindowsでもMacでも、ターミナルに以下を入力します。

```
mkdir claude-test
cd claude-test
```

これは、

```
claude-test というフォルダを作る
そのフォルダに移動する
```

という意味です。

このフォルダの中でClaude Codeを起動します。

```
claude
```

これで、Claude Codeがテスト用フォルダの中で起動します。

## 最初に聞くべきこと

Claude Codeを起動したら、いきなり「アプリを作って」と頼むより、まずは簡単な質問から始めるのが良いです。

たとえば、以下のように聞きます。

```
このフォルダで、簡単なHTMLページを作りたいです。
まだファイルは作らず、必要な手順だけ初心者向けに説明してください。
```

ポイントは、

```
まだファイルは作らず
```

と入れることです。

最初から自動でファイルを作らせるのではなく、まず説明だけしてもらうと安全です。

## 最初のファイル作成を試す

説明を読んで問題なさそうなら、次にファイル作成を頼みます。

```
index.htmlを作って、自己紹介ページのサンプルを作成してください。
作る前に、変更内容を簡単に説明してください。
```

Claude Codeは、どのファイルを作るか、どんな内容にするかを提案します。

ファイルを変更する前には確認を求められます。

内容を見て問題なければ承認します。

## 初心者はPlan Modeを使うと安心

Claude Codeには、作業前に計画を立ててもらう使い方があります。

初心者の場合、いきなり編集させるより、まず計画だけ出してもらう方が安全です。

たとえば、こう依頼します。

```
まずPlan Modeで進めてください。
まだファイルは編集せず、どのファイルを作るか、どんな手順で進めるかだけ説明してください。
```

これを入れると、Claude Codeがいきなりコードを変更するのを防ぎやすくなります。

慣れるまでは、

```
まだ修正しないでください
まず方針だけ教えてください
変更前に必ず確認してください
```

という言い方を使うのがおすすめです。

## VS Codeで使う方法

ターミナルだけでもClaude Codeは使えます。

ただ、ITに詳しくない方は、VS Codeで使った方が分かりやすい場合があります。

VS Codeは、コードを書くための無料エディタです。

VS CodeにClaude Code拡張機能を入れると、エディタの中でClaudeと会話しながら作業できます。

## VS Codeでの設定手順

1. VS Codeを開く
2. 左側の拡張機能アイコンをクリックする
3. 検索欄に「Claude Code」と入力する
4. Claude Code拡張機能をインストールする
5. VS Codeを再起動する
6. Claude Codeのパネルを開く
7. Claudeアカウントでサインインする

VS Code版では、Claudeが変更したい内容を差分で確認できます。

差分とは、変更前と変更後の違いのことです。

初心者にとっては、「どこが変わるのか」を目で見て確認できるので安心です。

## CLAUDE.mdを作っておくと便利

Claude Codeを継続して使うなら、プロジェクトの中にCLAUDE.mdというファイルを作ると便利です。

CLAUDE.mdは、Claude Codeへの説明書のようなものです。

たとえば、以下のような内容を書いておけます。

```
# Claude Codeへの指示

・回答は日本語でお願いします
・初心者にも分かるように説明してください
・ファイルを変更する前に、必ず変更内容を説明してください
・大きな変更をする前に、まず計画を出してください
・不要なファイル削除はしないでください
・コマンドを実行する前に、その意味を説明してください
```

これをプロジェクトの一番上の階層に置いておくと、Claude Codeが毎回読み込んでくれます。

初心者の方は、まずこのCLAUDE.mdを作ることをおすすめします。

## 最初に設定しておきたい安全ルール

Claude Codeは便利ですが、ファイルを編集したり、コマンドを実行したりできます。

そのため、最初は慎重に使った方がいいです。

初心者の方は、以下を守ると安全です。

・いきなり本番環境で使わない  
・まずテスト用フォルダで試す  
・「まだ修正しないで」と伝えてから相談する  
・ファイル変更前に説明してもらう  
・よく分からないコマンドは実行しない  
・削除系の操作は必ず確認する  
・大事なファイルはバックアップしておく

特に注意したいのは、削除系のコマンドです。

たとえば、

```
rm
del
remove
```

のような言葉が出てきた場合は、何を消すのか必ず確認した方がいいです。

## よく使うコマンド

最低限覚えるなら、以下だけで十分です。

```
claude
```

Claude Codeを起動します。

```
claude --version
```

バージョンを確認します。

```
claude doctor
```

問題がないか診断します。

```
/login
```

Claude Codeの中でログインし直します。

```
/help
```

Claude Codeの中で使えるコマンドを確認します。

```
/exit
```

Claude Codeを終了します。

## よくあるつまずき

## claudeと入力しても動かない

一度ターミナルやPowerShellを閉じて、もう一度開いてください。

それでも動かない場合は、以下を試します。

```
claude doctor
```

または、もう一度インストールコマンドを実行します。

## PowerShellとCMDを間違える

Windowsには、PowerShellとコマンドプロンプトがあります。

PowerShellの場合は、画面に

```
PS C:\
```

のように表示されます。

PowerShellでは以下を使います。

```
irm https://claude.ai/install.ps1 | iex
```

CMDの場合は別のコマンドになります。

初心者の方は、PowerShellを使う方が分かりやすいと思います。

## ログイン画面が出ない

`claude` と入力してもログインできない場合は、以下を試します。

```
/login
```

それでも難しい場合は、一度Claude Codeを終了して、もう一度 `claude` を実行します。

## 動作が重い

大きなプロジェクトで使うと、動作が重くなることがあります。

その場合は、まず小さいフォルダで試す方がよいです。

また、長く会話しすぎた場合は、

```
/clear
```

で会話をクリアすることもできます。

## 最初に試すおすすめプロンプト

Claude Codeを入れたら、最初は以下のような依頼から始めると分かりやすいです。

```
このフォルダの中身を確認して、初心者にも分かるように説明してください。
まだファイルは変更しないでください。
```

```
このプロジェクトで使われている技術を説明してください。
専門用語は補足してください。
```

```
エラーが出ています。
まだ修正せず、原因として考えられることを順番に説明してください。
```

```
この機能を追加したいです。
まず、どのファイルを変更する必要があるかだけ教えてください。
```

```
変更前に必ず方針を説明し、承認を取ってから作業してください。
```

初心者の場合は、最初から「作ってください」よりも、「説明してください」「方針を出してください」から始めるのが安全です。

## まとめ

Claude Codeは、コードを書ける人だけのツールではなく、これから開発を学びたい人にとってもかなり役立つツールだと思います。

ただし、最初の設定ではターミナルやコマンドが出てくるため、ITに慣れていない人は少し戸惑いやすいです。

まずは、以下の順番で進めるのが良さそうです。

1. Claudeアカウントを用意する
2. Native InstallでClaude Codeを入れる
3. `claude --version` で確認する
4. `claude` で起動する
5. ブラウザでログインする
6. テスト用フォルダを作る
7. まずは「まだ変更しないで」と伝えて相談する
8. 慣れてきたらVS Codeでも使ってみる
9. CLAUDE.mdで安全ルールを書く

Claude Codeは便利ですが、最初から全部任せるよりも、自分が理解しながら少しずつ使う方が安全です。

特に初心者のうちは、

「まだ修正しないで」  
「まず方針だけ教えて」  
「変更前に説明して」

この3つを入れるだけで、かなり安心して使えると思います。

AIに任せるというより、AIと一緒に確認しながら進める。

Claude Codeは、そのためのかなり実用的な道具だと感じました。
