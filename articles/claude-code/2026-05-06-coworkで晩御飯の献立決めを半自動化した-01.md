## はじめに

「今日の晩御飯どうしよう」って毎週悩みませんか。私は悩みます。

一時期はレシピをいくつか確保しておいて、そこからランダムにともおもっていたのですが。家の食材の残りとか、家族の好き嫌いも考慮しながらとなるとかなり実装が大変になるので、いつかはやりたいと思いつつも手付かずでした。

そこで面倒な部分はAIに任せながら、ある程度はロジックに基づいて献立が決まるようにAnthropicのデスクトップAIツール「**Cowork**」を使って、平日4日分の献立決めを半自動化してみました。（うちは月から木まで作り置き、金はその日の気分で作るというスタンスです）  
レシピはデリッシュキッチンから頂いております。基本的にどのメニューも美味しいイメージ

Coworkは Claude Code をベースにしたデスクトップアプリで、ファイル操作・ブラウザ内JavaScriptの実行・MCPプラグイン呼び出しなどを Claude が直接やってくれます。プログラミング不要で、実際のファイルを読み書きしながら AI が動いてくれる、というイメージです。

## どういうロジックにしたか

delishkitchen.tv からランダムにレシピを取得し、**月〜木の4日分の主菜候補を3択ずつ提示 → ユーザーが選ぶ → 買い物リスト生成 → 履歴に記録** というフローを毎週回せるようにしました。後述していますがCOWORKは画面内にHTMLを表示することができるので、そこでこちらが決めた選択肢などを出して指示しやすくしています。

またfetchについてはClaude in Chromeを利用し、ブラウザから実行しています。基本的に人が利用する速度ぐらいのアクセスになるので、BOTのようにサーバーに負荷をかけないようになっています。

```
[1] 設定フォームウィジェット（チャット内に表示）
    └ 直近履歴・肉種選択・使いたい食材・NG解除・追加指示
         ↓  sendPrompt() でClaudeに設定を渡す
[2] Claude が fetch() でレシピを並列取得・フィルタリング
         ↓
[3] 候補ウィジェット（各曜日3択 + 再抽選ボタン）
         ↓  sendPrompt() で選択結果をClaudeに渡す
[4] 最終チェック → 買い物リスト生成 → menu_history.json に記録
```

## delishkitchen.tv の利用規約について

スクレイピングまわりは事前に規約を確認しています。もし問題等あればコメントで教えていただけますと幸いです。

以上を踏まえ、**個人利用・低負荷・非商用** という条件で利用しています。大量アクセスや商用利用の場合は必ず公式の利用規約をご確認ください。  
値が取得できるURLについてはブラウザ上で確認できるネットワーク情報で把握しており、リバースエンジニアリングを活用して得た知識ではありません。

## プロジェクト構成

```
晩御飯決め/
├── CLAUDE.md          # Claudeへの指示書（プロジェクトルール）
├── site_structure.md  # delishkitchen.tv の構造メモ（技術仕様）
├── dish_dislikes.json # 苦手料理・NG食材リスト
└── menu_history.json  # 承認済み献立の履歴
```

Coworkでは `CLAUDE.md` をプロジェクトルートに置くと、Claudeが自動で読み込んで毎回参照してくれます。これがこの仕組みの核心です。ルールはMDに書いてあるので、毎週同じフローが再現できる、というのがミソ。

## HTMLウィジェットの仕組み

Coworkの面白いところは、**HTMLをチャット内にインラインレンダリングして、ボタンやフォームの操作結果をそのまま Claude に送り返せる** ことです。これはブラウザやレシピサイトとは無関係で、Coworkのチャット画面内だけで完結します。

ClaudeがHTML+JSのコードを生成してチャット内に描画すると、そのウィジェットの中から `sendPrompt()` というグローバル関数を呼び出せます。これを呼ぶと、ユーザーが入力したかのようにメッセージがチャットに投稿され、Claudeがそれを受け取って次の処理に進みます。

```
// Cowork のウィジェット内で使えるグローバル関数
sendPrompt("【献立選択結果】\n月曜：豚もやしのオイスター炒め ...");
```

ウィジェットのHTMLコード自体は毎回 Claude が生成しますが、「何を・どの順で・どんな形で出すか」はCLAUDE.mdに書いてあるので、結果として毎回同じUIが出てきます。

## 実際の利用イメージ

実際のCowork上での画面はこんな感じです。

### ① 設定フォーム

「来週の晩御飯の献立」と投げると、まず設定フォームが出てきます。

![](https://static.zenn.studio/user-upload/324dec6155e7-20260506.png)

フォームには5つのエリアがあります。

**直近の献立履歴**  
`menu_history.json` から直近2週分を自動表示。曜日・料理名・肉種バッジで一覧できます。「先週これ食べたな」と確認しながら設定できる。

**肉種チェックボックス**  
鶏もも肉・豚肉・牛肉・ひき肉の4種。各肉種の直近使用回数も表示されます。直近1ヶ月で3回以上使った肉種は「多め」バッジがついてデフォルトでチェックが外れる仕組み（ルール違反を自動防止）。スクショだと豚肉が「直近3回・多め」になってチェックが外れています。  
魚がないのは魚料理面倒だからというのは内緒、、

**使いたい食材（自由記入）**  
「なす」「きのこ」など、今週使いたい食材を自由に入力できます。入力すると、その食材を含むレシピを優先的に検索してくれます。

**NG食材チェックボックス**  
`dish_dislikes.json` に登録されたNG食材が全チェック済みで表示されます。「今週だけポン酢OKにしたい」みたいな場合はチェックを外せばその週だけ解除。

**追加指示（自由記入）**  
「中華系を混ぜて」「さっぱり系で」など自由に書けます。

送信すると `sendPrompt()` でこういうメッセージがClaudeに飛びます。

```
【来週の献立検索】
肉: 鶏もも肉, 牛肉, ひき肉
使いたい食材: なす
NG有効: レタス, カリフラワー, ポン酢, チャーシュー, むね肉, ささみ, 手羽
NG解除: なし
追加指示: 中華系を混ぜて
```

Claudeはこの構造化されたメッセージを受け取って検索処理に入ります。

### ② 候補選択 → 確定 → 買い物リスト

設定を送ると、Claudeが裏で `fetch()` を叩いてレシピを集めて、月〜木それぞれ3択の候補ウィジェットを出してくれます。各曜日で1つ選んで送信すると、最終チェックが走って献立確定 → 買い物リストまで一気に出てきます。

![](https://static.zenn.studio/user-upload/afb1e326cb89-20260506.png)

![](https://static.zenn.studio/user-upload/62bdd5a50fe6-20260506.png)

確定と同時に `menu_history.json` にも追記されるので、次週フォームを開くと自動で履歴が更新されている、という流れです。

## CLAUDE.md の全文

```
# 晩御飯決めプロジェクト

## 概要
平日（月〜木）4日分の晩御飯（主菜のみ）を考えるプロジェクト。
※日数は指示によって変動することがある。

## レシピ取得先
- https://delishkitchen.tv/

## レシピ選定ルール

1. **揚げ物は選ばない**
   - フライパンで完結する程度の調理ならOK（例：から揚げ風炒め、竜田揚げ風など）
   - 大量の油を使う本格的な揚げ物はNG

2. **肉のバラエティを重視する（2回まで同じ肉OKだが3回以上は不可）**
   - 同じ肉の種類は1ヶ月に最大2回まで
   - 例：鶏が2回 → OK、鶏が3回以上 → NG

3. **ひき肉・合いびき肉の出現頻度を下げる**
   - 1ヶ月4週のうち1〜2週のみ選ばれるようにする（約25〜50%の確率）
   - ランダム選定時に他の肉（鶏・豚・牛）を優先し、ひき肉は補助的に使う

4. **魚介系は除外する**
   - 魚・貝・エビ・イカ・タコ等の魚介類を使ったメインは選ばない

5. **主菜のみを選ぶ**
   - 副菜・汁物・デザートは選ばない
   - メインのおかず1品のみをレシピとして提示する

6. **プレミアム会員限定レシピは選ばない**
   - APIレスポンスの `data.state_premium === 0` のものだけを採用
   - DOMクラス（`.item-card__premium-ribbon`）での判定は誤検知があるため使用しない

7. **苦手料理・NG食材は選ばない**
   - dish_dislikes.json を参照。食材制限（ingredient_restrictions）も必ず確認する
   - 鶏肉は「鶏もも肉」のみOK。鶏むね肉・ささみ・手羽・鶏ひき肉・鶏レバー・砂肝は除外

## ランダム選定の方法

**基本方針: fetch() でHTMLとAPIを並列取得する（高速）。**
**⚠️ サーバー側（bash/Python）からはアクセス制限で403になる。必ず `javascript_tool`
（Claude in Chrome）でブラウザのコンテキスト内で実行すること。**

### 食材指定なし → カテゴリページを使用
- 豚肉: 458 / 鶏肉: 430 / 牛肉: 821 / ひき肉: 9479
- page=1 で総件数を取得 → 総ページ数を計算 → ランダムページを選ぶ
- ページ内のレシピIDを抽出し、APIで一括フェッチ（Promise.all）

### 食材指定あり → 検索ページを使用（フォールバックあり）
- キーワード候補を順番に試す（例: 「豚肉 なす」→「豚 なす」→「なす」）
- ヒット0件なら次のキーワードへ

### 主菜・フリー判定（API）
- `data.recipe_category.kind_id === 2` → 主菜のみ採用
- `data.state_premium === 0` → 無料レシピのみ採用

### 再抽選のルール
- valid が1件以上あればそのまま最大3件選ぶ
- valid が0件の場合のみ別ページで1回だけ再試行する

## 履歴管理
- 承認された献立は menu_history.json に記録する
- 同じ献立名は直近1ヶ月（約4週）は選ばない

## ウィジェットによる操作フロー

「来週の献立を決めたい」と言われたら、以下の順でHTMLウィジェットを使って進める。

### ステップ1：設定フォームウィジェットを表示する

以下の要素を含むHTMLフォームをチャット内（ブラウザタブ）に表示する：

- **直近の献立履歴**：menu_history.json の直近2週分を曜日・料理名・肉種バッジで表示
- **肉種チェックボックス**：鶏もも肉・豚肉・牛肉・ひき肉の4種。直近回数を表示し、
  直近1ヶ月3回以上の肉種は「多め」バッジをつけてデフォルトでチェックを外す
- **使いたい食材（自由記入）**：任意のテキスト入力欄
- **NG食材チェックボックス**：dish_dislikes.json の内容を全チェック済みで表示。
  チェックを外すと今週だけNG解除
- **追加指示（自由記入）**：任意のテキスト入力欄

送信すると sendPrompt() で以下のフォーマットのメッセージをチャットに送信する：

【来週の献立検索】
肉: 鶏もも肉, 牛肉, ひき肉
使いたい食材: なす
NG有効: レタス, カリフラワー, ポン酢, チャーシュー, むね肉, ささみ, 手羽
NG解除: なし
追加指示: 中華系を混ぜて

### ステップ2：レシピを検索して候補ウィジェットを表示する

【来週の献立検索】メッセージを受け取ったら、site_structure.md の手順でレシピを取得し、
候補選択ウィジェットをチャット内に表示する：
- 月〜木の4曜日それぞれについて、レシピ候補を3択で表示（タイトル＋URLリンク付き）
- 各曜日に「再抽選する」の選択肢を4つ目として用意する
- 送信すると sendPrompt() で【献立選択結果】または【再抽選リクエスト】を送信する

【献立選択結果】
月曜：豚もやしのオイスター炒め https://delishkitchen.tv/recipes/...
火曜：鶏肉と里芋の照り煮 https://delishkitchen.tv/recipes/...
水曜：里芋と牛肉のピリ辛炒め https://delishkitchen.tv/recipes/...
木曜：たけのこの回鍋肉 https://delishkitchen.tv/recipes/...

【再抽選リクエスト】
再抽選: 月,水
確定済み: 火曜 鶏肉と里芋の照り煮 https://...
          木曜 たけのこの回鍋肉 https://...

### ステップ3：確定・買い物リストの提示

【献立選択結果】を受け取ったら：
1. 4品のAPIから材料データを取得して最終確認（プレミアム・NG食材チェック）
2. 献立一覧をテーブル形式で提示
3. 全日分の材料をまとめた買い物リストを材料名・分量つきで提示（コピペできる形式）
4. menu_history.json に承認済み献立として追記する

【再抽選リクエスト】を受け取ったら：
- 指定された曜日だけ再検索してステップ2のウィジェットを再表示する
```

## fetch ベースの高速アクセス手順

⚠️ bash/Python からは403で弾かれる。必ず `javascript_tool`（Claude in Chrome）で
ブラウザのコンテキスト内で fetch() を実行すること。

### ケース1：食材指定なし → カテゴリページ

const html1 = await fetch(`https://delishkitchen.tv/categories/${catId}`).then(r=>r.text());
const total = parseInt((html1.match(/(\d[\d,]*)品/) || ['','68'])[1].replace(',',''));
const idsOnPage1 = [...new Set([...html1.matchAll(/\/recipes\/(\d+)/g)].map(m=>m[1]))];
const totalPages = Math.max(1, Math.floor(total / Math.max(12, idsOnPage1.length)));
const randPage = Math.floor(Math.random() * totalPages) + 1;
const html = randPage === 1 ? html1
  : await fetch(`https://delishkitchen.tv/categories/${catId}?page=${randPage}`).then(r=>r.text());
const ids = [...new Set([...html.matchAll(/\/recipes\/(\d+)/g)].map(m=>m[1]))];

### ケース2：食材指定あり → 検索ページ（フォールバックあり）

const searchTerms = [`${meatName} ${ingredient}`, `${meatKey} ${ingredient}`, ingredient];
for (const term of searchTerms) {
  const html1 = await fetch(
    `https://delishkitchen.tv/search?q=${encodeURIComponent(term)}&page=1`
  ).then(r=>r.text());
  const total = parseInt((html1.match(/(\d[\d,]*)品/) || ['','0'])[1].replace(',',''));
  if (total === 0) continue;
  const totalPages = Math.max(1, Math.floor(total / 12));
  const randPage = Math.floor(Math.random() * totalPages) + 1;
  const html = randPage === 1 ? html1
    : await fetch(
        `https://delishkitchen.tv/search?q=${encodeURIComponent(term)}&page=${randPage}`
      ).then(r=>r.text());
  ids = [...new Set([...html.matchAll(/\/recipes\/(\d+)/g)].map(m=>m[1]))];
  break;
}

### 全IDのAPIを並列フェッチ（共通）

const recipes = (await Promise.all(ids.map(id =>
  fetch(`https://delishkitchen.tv/api/recipes/${id}?include_maintenance=true`)
    .then(r=>r.json()).then(d=>({ id, data: d.data })).catch(()=>null)
))).filter(Boolean);

## APIレスポンスの主要フィールド

d.title              // レシピ名
d.state_premium      // 0=無料, 1以上=プレミアム
d.recipe_category    // { id, kind_id, name }
d.ingredient_groups  // [{ name, recipe_ingredients: [{name, servings1}] }]

## 主菜判定（★重要）

| kind_id | 意味               | 採否   |
|---------|--------------------|--------|
| 2       | 主菜               | ✅ 採用 |
| 1       | 主食(丼・パスタ等)| ❌ 除外 |
| その他  | 副菜・汁物等        | ❌ 除外 |

## フィルタリングの全体ロジック

// isChicken=true のとき鶏むね肉等を材料レベルでも除外
// isMince=false のときひき肉を使う料理を材料レベルで除外
const valid = recipes.filter(r => {
  if (!r.data) return false;
  if (r.data.state_premium !== 0) return false;
  if (r.data.recipe_category?.kind_id !== 2) return false;
  if (NG_TITLE.some(w => r.data.title.includes(w))) return false;
  if (HISTORY.some(h => r.data.title.includes(h))) return false;
  const allIngs = r.data.ingredient_groups.flatMap(g =>
    g.recipe_ingredients.map(i => i.name)
  ).join(' ');
  if (NG_INGS.some(w => allIngs.includes(w))) return false;
  if (isChicken && ['むね肉','胸肉','ささみ','手羽'].some(w => allIngs.includes(w))) return false;
  if (!isMince && ['ひき肉','挽き肉','合いびき','合挽'].some(w => allIngs.includes(w))) return false;
  return true;
});

## dish\_dislikes.json の全文

## まとめ

晩御飯を決める作業ならClaudeDecktopのプロジェクト機能でも出来そうですが、Coworkはこちらが指定したフォームを出せるのがそれがかなりありがたいですね。作ったばかりなので毎週使いながらチューニングしていこうと思います。

ルールのチューニングは会話の中でそのまま伝えると反映してくれるので、「やっぱりグラタンはやめて」「ポン酢もNGにして」みたいに口頭感覚でメンテできるのも便利です。ただし**会話でのみ覚えた内容はセッションをまたぐと忘れる**ので、定着させたいルールは必ずMDに書き直す、というのが大事なポイント。

仕事を便利にするのもいいですが、こういった日常のタスクをCoworkで解決するのいいですよね。皆さんも毎週やってる作業などをCoworkに代用させてはいかがでしょうか。
