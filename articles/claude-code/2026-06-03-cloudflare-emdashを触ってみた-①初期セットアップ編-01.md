自社サービスのブログ構築に向けて、最近ベータ版がリリースされたCloudflareが開発したCMS「EmDash」を触ってみました。今回はセットアップから実際に記事を公開するところまで。

## EmDashってなに？

2026年4月にCloudflareがリリースした「WordPressの現代版」を目指したCMSです。管理画面はWordPressにかなり近く、サーバーレス構成・AI前提の設計など、今の時代に合わせて作り直されています。

特に大きく違うのがプラグインのセキュリティです。WordPressはプラグインを入れるとDB・ファイルシステム・外部への通信も自由にできる状態になっていて、脆弱性の90%以上がプラグイン由来と言われています。EmDashはプラグインをサンドボックス内で隔離して実行する設計で、どのテーブル・どの機能・どの外部ホストにアクセスするかを事前に宣言する仕組みになっており、宣言外のことは技術的にできないのでセキュリティ面で優れた設計になっています。

## セットアップの前に

Cloudflareの管理画面で2つ準備が必要です。これをやっておかないとデプロイ時にエラーになるので先に済ませておくのをおすすめします。

## セットアップが本当に簡単だった

Node.jsが入っていれば、これだけです。

```
npm create emdash@latest
cd my-blog
npm run dev
```

数分でローカルに管理画面付きのブログが立ち上がります。

デプロイもコマンド一発。ただし初回は先にCloudflareアカウントと連携が必要です。実行するとブラウザが開いてログインできます。

```
npx wrangler login
npm run deploy
```

Cloudflare Workers・D1・R2が全部自動でセットアップされて公開完了です。初めて触った状態でその日のうちに公開まで持っていけました。

デプロイ後に発行されたURLへアクセスし、管理画面でメールアドレスやブログ名、サンプル記事の有無などを設定するとブログと管理画面が作成されます。

## 触ってみて良かったところ

* **セットアップが速い**：インフラの知識がなくても公開まで持っていけるのは本当に楽でした。
* **CDNの設定が不要**：デプロイ後に自動的にエッジ配信が効いた状態になります。
* **AIが実装する前提で設計されている**：リポジトリのプロジェクトルートに `AGENTS.md` というファイルが用意されていて、起動コマンド・触っていいファイルの一覧・EmDash固有のルール・やってはいけないことなど、「このテンプレートの設計思想を守りながらカスタマイズしてね」という内容がAI向けにまとめられています。`AGENTS.md` が実体ファイルで、OpenAI Codex CLI はこれを直接読みます。Claude Code 向けには `CLAUDE.md` というシンボリックリンクが貼られていて、同じファイルを参照する仕組みになっています。

| 項目 | 実体 | シンボリックリンク |
| --- | --- | --- |
| 指示書 | `AGENTS.md` | `CLAUDE.md` |
| スキル | `.agents/skills/` | `.claude/skills/` |

スキルファイルには「特定の作業をするときはこのスキルをロードして」という指示が `AGENTS.md` に書かれていて、AIが必要に応じて参照します。

| スキル | 内容 |
| --- | --- |
| `building-emdash-site` | コンテンツ取得・Portable Text・スキーマ設計など、サイト構築全般 |
| `creating-plugins` | EmDash プラグインの作り方 |
| `emdash-cli` | CLI コマンドの使い方 |

## 気になったところ

* **基本的な機能以外は自分で実装する必要がある**：WordPressなら「SEO対策したければYoast入れる」みたいに探してインストールできますが、EmDashはまだエコシステムが育っていないのでそうはいきません。ただEmDash自体がAIに実装させる前提の設計になっているので、次回はそこを試してみようと思っています。
* **SEO設定が反映されない**：管理画面にはSEOタイトル・メタディスクリプション・OG画像・canonical URLの入力欄があるんですが、入力しても実際には反映されません。バグなのか自分で実装する前提なのかわかりませんが、中身を触っていくうちに他にも同じようなことが出てきそうなので、テンプレートのカスタマイズは必須だと思っておいた方が良さそうです。

## まとめ

まだベータ版ということもあり、SEO設定など少し触っただけでも動かない箇所がありました。それでもかなり面白いCMSだと感じています。

セットアップやカスタマイズ方法などはCodexに手順を聞きながら進めたのですが、それにしてもスムーズで。あとから `AGENTS.md` の存在に気づいて、なるほどと思いました。Codexが「このプロジェクトでは何を触っていいか・どう動かすか」を最初から把握した状態でサポートしてくれていたからだったんですね。

セキュリティ周りの設計はもちろん、AIによる実装やコード生成を前提にしたような考え方も興味深く、WordPressからのインポート機能も管理画面についているのでどこまで使えるのかも気になっています。

まだ分からないことだらけですが、とりあえずもっと触ってみたい！まずは自社サービスのブログ開設に必要な機能を整えながら、引き続き検証していこうと思います。
