# 非プログラマです

なんか最近流行ってるらしいじゃん？Claude Code？ナウな業界関係者としては使ってないとイケてないと聞いたので使ってみるよ。

ということで、プロダクトオーナーロールなわたしもClaudeCodeで業務刷新してみました。

# 何をしたかったか

今業務で取り扱っているプロダクトが多方面へのカスタマイズ要望に応えていたら大きく３つの亜種に分離してしまいました。基本機能は同じものの、こっちにはこの画面が無くて、そっちではあの文言が違う。一番厄介なのが見た目同じだけど動作の仕組みが違ったりする。  
これはマズい。何がマズいってこの３つのプロダクトの違いをわたし以外誰も説明できない。下手したらわたしもとっさには回答できないこともあり、個人への依存が激しい。  
休めない。困る。俺。働く。（原始人モード

ということで、まずは仕様をまとめておきたいなーと軽い気持ちで着手しました。

# やったこと

どうやらClaudeCodeはskillっちゅーのがイケてるらしい。なのでまずはskillを作ってみた。  
結論から言うと、この順番が良かった。一説によるとCLAUDE.mdを先になんとかしろと言うのがあるけど、何ができるかわからんのにここをいじるってなかなか難しいなあ、と。

### なんか保存されないぞ？

skillには結果を同skill配下の指定フォルダに保存、としているのに何か保存されない。正しくは２回目以降チャットを新規にはじめたらされなくなりました。軽く原因調べてるとDesktopにはそんな不具合があるらしい？とか。

なので急遽ターミナルからClaudeを導入・実行。これはHOMEに作られる.claude配下にスキルを置くと動くらしいので、Desktop版で作ったのをダウンロードして指定通りに置いてみる。すると、おお動いた。こっちではちゃんと保存されるので解決。

Desktop版の方が初期の導入やビジュアライゼーションはいい感じでしたが、ターミナルも作業完了後や確認の際にポコポコなるので終わったのに気づけてこれはこれでいいですね。

# 結果

着手は水曜夕方で、金曜には使えてたので２日程度で完成してました。  
さらに言うと、機能差分をまとめるとついでに画面や遷移差分も作りたくなってこちらも並行でぽちぽち。２つ合わせても２日。作業としてはすごく効率いい気がします。

効率よくできた理由としては、時間空いた合間に都度チャットで修正指示を出せたことですね。Confluenceに自分で表を作る場合は途中で作業分断されてしまうと思い出しに時間がかかり大変でしたが、これならまとまった時間が無くても資料作りができるのでいい感じです。表の構造でまとめ方少し違うなーと思った時も、こう言う感じのまとめ方の方がいいかも、とチャットで改善方針投げるといい感じに作り直してくれる。すごいねぇ。

今後は仕様変更や差分が発生したらこのスキルでどんどん追加していけばいいし、何より課題だったこのプロダクトの場合ここの仕様ってどうなってたっけ？をチャットで確認できるのはすんごく便利。あとついでに作った画面仕様や遷移はもう少し頑張れば実際のキャプチャ付き画面遷移図を作ったりテストの自動化に役立てられるかもしれない。久々に（？）いい仕事した感がありました。

# そして今…

大体これを4月頭に実施して、大体４週間弱が経過。今はもっぱらClaude Codeをターミナルから動かしています。

開発チームでTeamプランを契約していますが、上記のskillsはGHEで共有して開発の皆様にも活用いただきつつ、さらなる更新を継続中。今のマイブームはPlayWright。  
それとは別に企画側の仕事もぼちぼちCC化をしています。特に、アウトプットの形式ともかく、インプットの原稿は全てmdファイルで作成して、skillで適切なアウトプット形式にするのがいい感じですね。

ところでまだまだ利用始めの初心者ですが、今のところSKILL.mdやCLAUDE.mdを直接いじるケースはないですね。初期セットアップがそこそこ心理的な障壁になってるような気もしたのですが、特に何もいじらなくても馴染んでよしなに動いてくれているので困ってません。ですので、個人的意見でのおすすめですが、難しい参考書は読まずにまずは使う、スキルを作ってみるのがおすすめです。
