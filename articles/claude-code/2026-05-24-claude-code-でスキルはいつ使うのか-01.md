Claude Code のスキルについての考え方などをご紹介します。  
あくまで事例の一つとして、参考にしていただけると幸いです。

## 1. 使う必要がなければ使わなくてOK

最新のモデルは賢いです。幅広く学習したデータが含まれているので、何も困っていなければスキルは使わなくてOK。  
ファイルの中身や構造も理解してくれます。  
期待通りの結果が返ってこない場合は、Claude Code が知らないことは教えてあげましょう。賢くても知らないことはエスパーではないので出来ないのです。

* 繰り返し同じことを入力している
* 実行するたびに同じ結果が返ってこない

という状況が気になってきたら、`CLAUDE.md` の設定を試してみましょう。

## 2. CLAUDE.md を設定する

繰り返し同じことを入力したり、実行するたびに結果が異なるのなら、CLAUDE.md を設定しましょう。

それで困らなくなったら、スキルは使わなくてOK。

* CLAUDE.md が大きくなりすぎて人間が読むのが辛い。Claude Code が守ってくれない。
* もっと効率いい方法を知りたい
* トークンを節約したい
* 決まった手順を一言で呼び出したい

という状況が気になってきたら、スキルを試してみましょう。

## 3. スキルがよくわからない

初めは仕方がないです。なら、Claude Codeに聞いてみましょう。

* さっきの内容をスキルにして
* さっきの内容をスキルにできそう？
* もっと効率よくできるスキルはない？

だいたい Claude Code がやってくれます。ただ、せっかくならスキルを使うためにスキルを使ってみましょう。

skills.sh という、スキルを探せるサービスがあります。

```
npx skills add https://github.com/vercel-labs/skills --skill find-skills -a claude-code
npx skills add https://github.com/anthropics/skills --skill skill-creator -a claude-code
```

`Installation scope` は `Project` をまずはお勧めします。 Claude Code を実行しているディレクトリに `.claude/skills` が作られて確認しやすいからです。  
また、`-a claude-code`（`--agent claude-code`）オプションは、他のAIエージェント（Copilot Workspace等）向けの設定ファイルが生成されるのを防ぎ、Claude Code専用に最適化してインストールするためのものです。Claude Codeで利用する場合は、このオプションを指定することをお勧めします。

## 4. スキルを使ってみよう

Claude Code で、`/skill-creator` や `/find-skills` を入れてみてください。

```
/find-skills                  Helps users discover and install agent skills when they ask questions like "how do I do X", "find a skill for X", "is there a skill that can...", or
                              express interest in extending capabilities. This skill should be used when the user is looking for functionality that might exist as an installable skil…
/skill-creator                Create new skills, modify and improve existing skills, and measure skill performance. Use when users want to create a skill from scratch, edit, or
                              optimize an existing skill, run evals to test a skill, benchmark skill performance with variance analysis, or optimize a skill's description for better …
```

のように、補完で出ればスキルインストール成功です。出てない場合は Claude Code を再起動がいるかもしれません。

使えるようなら、以下のように試してみてください。自分で考えるより、まずは Claude Code に丸投げです。

```
/find-skills さっき行った内容を効率よくできるスキルない？
/skill-creator さっきの内容をスキルにして
```

そうすると、Claude Code がスキルをインストールしてくれたり、スキルを作ってくれたりしているはずです。  
Claude Code が出力した内容をよく読んでみましょう。インストールされた `スキル名` が出ているはず。

`/スキル名` で呼び出せるようになっているはず。スキルは Claude Code が必要と思えば勝手に使ってくれます。  
明示的に使いたいときは `/スキル名 やりたい内容` を入力して明示的にスキルを呼んでみましょう。

## 5. スキルを読んでみよう

スキルはただのテキストです。英語が読めなかったら Claude Code に翻訳してもらいましょう。分からないところがあったら Claude Code に聞いてみましょう。だいたい答えてくれます。

## 6. スキルの構造は？スキルのルールは？

[Agent Skills オープン標準仕様](https://agentskills.io/home) というものがあります。Claude Code もこの仕様に沿って作られています。読んでみましょう。面倒だったら Claude Code に URL を渡して聞いてみましょう。解説してくれます。

スキルの本体は `SKILL.md` です。まずはこれを読めばどんなスキルかわかります。  
`SKILL.md` 以外には、スキルが使用するプログラム `scripts` や スキルが参照するファイル(テンプレートとか)や知識とかも含めることができます。

```
my-skill/
├── SKILL.md          # Required: metadata + instructions
├── scripts/          # Optional: executable code
├── references/       # Optional: documentation
├── assets/           # Optional: templates, resources
└── ...               # Any additional files or directories
```

## 7. スキルを活用するには？

スキルの内容を読んで、スキルのフォルダに含まれているものを見ると、何に使うかが見えてきます。  
そのスキルが必要ない・使用されていないと思えば、そのスキルは削除してしまいましょう。

スキルの内容が使われていそうだけどよく分からないなら。Claude Code に聞いたり改善してもらいましょう。

だいたい Claude Code に聞けばいいです。やって貰えばいいです。  
人間は、Claude Code が何をやったかだけを把握できていればOK。分からなければ、Claude Code に聞けばいいです。  
人間に聞くのと違って Claude Code は嫌な顔はしません。トークン(お金)は消費しますが。

色々聞いてみた結果、それでも不要と思うのなら、スキルは使用しなくてもいいでしょう。  
スキルは無くても出来ることは沢山あります。  
それでも、 `skills.sh` のように、いろいろな人やAIが便利だと思うスキルを共有してくれてます。

せっかくならスキルを活用してみてはどうでしょうか？

## 8. 他のエージェントでも使える？

`Agent Skills オープン標準仕様` に準拠しているため、実は Claude Code 以外のAIエージェントでも、基本的には全く同じようにスキルを活用できます。

例えば、**Antigravity** などのエージェントでは、スキルの置き場所（例: `.agents/skills`）やエージェント設定ファイル（例: `AGENTS.md`）などの仕様に細かな違いはあります。しかし、標準仕様に沿って作られているため、一度作成したカスタムスキルや習得した概念は、そのまま他のエージェントでも使い回すことが可能です。
