# まえがき

Anthropic Claude Code(Opus/Sonnet)、OpenAI Codex、GitHub Copilot、Google Gemini などコーディングエージェントで指示したりドキュメントを読み込ませたりするとあっという間にトークンを消費してしまいます。原始人など日本語の冗長な言い回しを削るといった使い方もありますが、そもそもトークンの消費についてしらべてみました。

# モデルのほとんどは英語でできている

日本語のトークンを分割しているため、日本語→英語のコストが大きいです。  
日本語はそもそも「分かち書き」がないため、文章をどこで区切るかわかりません。  
また、「こんにちは」でも、

「今日は、docsの中の仕様のフェーズ1を実装しましょう。」

の「今日は」は「きょうは」なのか「こんにちは」なのかどちらとも読めるため、指示には含めないほうがいいです。ここをなくすだけで3~4トークンを節約できます。

上記例だと、

「docsの中のファイル名.mdのフェーズ1(#見出し名)を実装」

が僕の中ではパフォーマンスもよく、かつトークン数も抑えられました。  
見出し名を指示すればエージェントが grep してそこだけを読むようになるので、全部読ませることにはならないのでおすすめです。

# じゃあ、英語で書いたらいいのか？

答えはイエスです。

モデルは英語でできているので、たとえば「こんにちは」で3~4トークンを消費するのに対して、「Hello」は1トークンしか消費しないので、間違いなく英語で書いたほうがいいです。DeepL や Gemini で英語にしてもらってそれをプロンプトするという組み合わせもおすすめです。

# じゃあ、中国語は？

これも日本語と比べると、トークン消費量は少ないです。「你好」は1~2トークンしか消費しません。日本語と比べたら約半分です。僕は英語、日本語どちらも使えるので、「英語使える人は英語で、中国語使える人は中国語で」と変えたほうがいいかもしれません。

中国語は漢字一字には動詞が含まれているので、「送」は「send」の意味で一字に圧縮されています。みなさんもよく「中国語だと短いな」と感じるのはそういうことです。トークン消費の少なさもあって、中国系の AI オープンソース(DeepSeek・Qwenなど)がどんどん出てきている背景もありそうです。

※実際はもっと根深いところで、米国の AI ビジネスが伸びてきているなかで、中国はあえてどんどんオープンソース化し米国一強というモデルを崩しに行っているという政治的戦略もありそうですが、自信をもって言えないのでここまでにしておきます

# 最近は各社ともに多言語に対応する辞書が強化されてトークン消費量(input)は抑えられている傾向にある

最近のモデルでは、多言語対応の辞書が強化されており、以前のモデルに比べると日本語のトークン効率は劇的に向上しています。ただし、出力(output)は英語/中国語が強い印象がします。

これはおそらく英語、中国語ともに「文法」が決まっていて、基本的に SVO の形をしているので、日本語より優位なところがあります。だから、原始人も削れば削るほど「偽中国語」になり「それでも意味は通じる」という結果なので、日本語でプロンプト書いても「冗長な文章になっていないか」一呼吸をおいて削れる部分は削るとよいでしょう。

# 僕がやっていること

原始人は使ってはいませんが、プロンプトの最初にある約束をしています。

```
チャットで敬語・丁寧語など日本語特有の冗長な不要。あなたの考え事・独り言は不要。最低限の言葉と会話にする。記憶して。
```

これだけでも Claude Code の5時間ウィンドウを1時間半で消費せず、3時間半は使えるようになりました。かなり効きます。

すこしでも無駄に感じたらその場で矯正すると、

まで減らしてくれます。注意はここで「実装してください」や「お願いします」と返さないことです。

でよいです。これで黙々と淡々とこなしてくれますし、トークン消費量も劇的に抑えることができるので試してみてください。「あえて丁寧に話さない」ことが結果的に消費電力を抑えられるので、みんなハッピーです。

この記事でも丁寧であるほど無駄に電力を使うことも言及がありました。

1000万ドル  
→ 1,589,375,090円(4/20 15:00時点の為替で算出)

数千万ドルが具体的ではないので、1,000万ドルで計算すると約16億円です。全世界の人が「ありがとう」や丁寧な返しをするだけで軽く十数億円は蒸発し、専用データセンターが必要だやら電気を賄うために発電所をつくらないといけないやら、深刻な状況なので今回の記事のように「あえて丁寧な言葉はやめる」にシフトしてもいいかもしれません。
