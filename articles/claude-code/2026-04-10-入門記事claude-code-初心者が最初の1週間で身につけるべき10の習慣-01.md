Claude Code を入れたばかりの1週間は、伸び方が一生を決める。最初に雑な使い方の癖がつくと後から矯正するのが大変で、逆に最初の7日間で正しい型を身につければ、翌週から目に見えて生産性が跳ね上がる。

この記事では、俺が新しい同僚に Claude Code を渡すときに必ず教える「最初の7日間で身につける10の習慣」を Day1〜Day7 の形式で全公開する。

## Day1: CLAUDE.md を書く

**やること**: プロジェクトのルートに `CLAUDE.md` を作り、最低限以下を書く。

```markdown
# プロジェクト指示

## このプロジェクトは
Next.js 14 + TypeScript + Prisma のECサイト。

## 禁止事項
- any 型の使用
- 変更対象外のファイルへの勝手なリファクタ
- 明示的指示なしのコミット

## コミット前チェック
1. npx tsc --noEmit
2. npm run build
3. npm test
```

**なぜ必要か**: Claude は起動時にこれを読む。毎回口頭で伝え
