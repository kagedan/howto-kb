以前、「ChatGPTにコーディングを手伝わせたら作業時間が半分になった」という記事を書いた。

あの記事の結論は「ChatGPTは80点のたたき台を5分で出してくれる優秀なアシスタント」だった。  
今もその認識は変わっていない。

でも最近、Claude Codeを使ってランディングページ（LP）を1本作ってみたら、ちょっと認識が揺らいだ。

「…これ、ChatGPTより良くない？」

今日は、前回のChatGPTでのコーディング体験と比較しながら、Claude CodeでLP制作をやってみた正直な感想を書く。

---

## そもそもClaude Codeって何？

知らない人のために簡単に説明すると、Claude CodeはAnthropicが出しているAIコーディングツール。  
ターミナル（コマンドライン）から起動して、自然言語で指示を出すとコードを書いてくれる。

ChatGPTとの最大の違いは、**自分のPC上のファイルを直接操作できる**こと。

ChatGPTの場合、「このコードを書いて」→ コードが画面に表示される → 自分でコピーしてファイルに貼り付ける、という手順が必要だった。

Claude Codeは、指示を出すと勝手にファイルを作成・編集してくれる。  
コピペ作業がゼロ。これだけで作業の手触りが全然違う。

---

## 今回やったこと：LPを1本作る

クライアント案件ではなく、自分の練習用としてLPを1本作ってみた。

ChatGPTでコーディングを手伝わせた前回の記事では「部分的な作業」を依頼した。  
カードレイアウトを書かせたり、CSSのアニメーションを書かせたり、エラーの原因を聞いたり。

今回は**ページ丸ごと**をClaude Codeに作らせてみた。  
ヘッダーからフッターまで、レスポンシブ対応込みで。

---

## ChatGPTとClaude Code、何が違ったか

### 違い①：ファイル操作の有無

**ChatGPTの場合**  
1. 「こういうコードを書いて」とプロンプトを投げる  
2. ChatGPTがコードを画面に表示する  
3. 自分でコードをコピーする  
4. エディタを開いてファイルに貼り付ける  
5. ブラウザで確認する  
6. 修正があればまたChatGPTに聞く → コピー → 貼り付け → 確認…

この「コピー→貼り付け→確認」のループが地味に面倒だった。

**Claude Codeの場合**  
1. 「こういうLPを作って」と指示する  
2. Claude Codeが直接ファイルを作成する  
3. ブラウザで確認する  
4. 「ここをこう直して」と指示する → Claude Codeが直接ファイルを修正する

コピペ作業がゼロ。  
指示を出して、確認して、修正を伝える。この繰り返しだけ。

Webデザイナーがコーダーに指示を出す感覚にすごく近い。  
「ここの余白をもう少し広げて」「フォントサイズを16pxに変えて」と言えば、Claude Codeがコードを直接書き換えてくれる。

### 違い②：コードの品質

**ChatGPTのコード**  
前回の記事でも書いたけど、ChatGPTのコードは「動くけど、ちょっと荒い」ことが多かった。  
クラス名が雑だったり、不要なコードが混じってたり。  
80点のたたき台としては十分だけど、そのまま納品レベルにするには手直しが必要だった。

**Claude Codeのコード**  
ここが一番驚いたポイント。  
Claude Codeが書くHTML/CSSは、構造がかなりきれい。  
セマンティックなHTMLタグの使い方、CSSのクラス名の命名規則、コードのコメント。  
Webデザインをやってきた目から見ても「これ、ちゃんとわかってる人が書いたコードだ」と思えるレベルだった。

特にCSSの書き方が丁寧で、後から自分が修正する時にも読みやすい。ChatGPTのコードは動くけど「なんでこの書き方？」と思うことがあったのに対して、Claude Codeは「あ、こう書くよね」と納得感がある。

### 違い③：デザインのセンス

ここは正直、どっちもまだ課題がある。

ChatGPTもClaude Codeも、何も指定せずに「LPを作って」と投げると、「AIが作りました感」が出るデザインになる。  
色使いが無難すぎたり、余白のバランスが微妙だったり。

ただ、Claude Codeのほうが微調整の指示に対する反応が良かった。  
「もっとミニマルに」「余白を広めに取って」「この配色に変えて」と伝えた時の修正精度がChatGPTより高い印象。  
会話のキャッチボールでデザインを追い込んでいく作業が、Claude Codeのほうがスムーズだった。

### 違い④：レスポンシブ対応

LPなので当然スマホ対応が必要。

ChatGPTに「レスポンシブ対応して」と頼んだ前回は、基本的なブレイクポイントは設定してくれたけど、スマホでの表示崩れをいくつか手動で直す必要があった。

Claude Codeは、最初から「PC、タブレット、スマホの3段階でレスポンシブ対応してください」と指示したら、かなり精度の高いレスポンシブCSSを書いてくれた。  
もちろん細かい調整は必要だったけど、手直しの量はChatGPTの時より明らかに少なかった。

## 作業時間の比較

同じ規模のLPを作ると仮定して、体感の比較。

**手動でゼロから作る場合**  
コーディングだけで6〜8時間

**ChatGPTを使う場合（前回の記事の方法）**  
ChatGPTに部分的にコードを書かせる＋自分でコピペ＋調整  
→ 3〜4時間（約半分）

**Claude Codeを使う場合（今回）**  
Claude Codeにページ丸ごと作らせる＋指示で修正＋最終調整  
→ 2〜3時間（3分の1くらい）

ChatGPTとClaude Codeの差は「コピペ作業の有無」と「修正指示の精度」で生まれてる。  
Claude Codeは直接ファイルを触ってくれるのでコピペ時間がゼロだし、修正指示に対する反応が良いので「指示→確認→再指示」のループが少なくて済む。

---

## じゃあChatGPTはもう使わないのか？

そんなことはない。使い分けがある。

**Claude Codeが向いてる場面**  
- ページ丸ごと、またはセクション単位でコードを書く時  
- 「ここをこう直して」の修正を何度も繰り返す時  
- ファイル構成が複数にまたがる時（HTML/CSS/JSを同時に扱う）

**ChatGPTが向いてる場面**  
- 「このCSSの書き方を教えて」のような質問・学習  
- エラーの原因を聞く（コードを貼り付けて「なぜ動かない？」）  
- 企画書やメール作成など、コーディング以外の業務

つまり「コードを書く作業」はClaude Code、「考える・調べる・書く（文章）」はChatGPT。  
この使い分けが今のところのベスト。

---

## これまでの経験はClaude Codeでも武器になる

前回のChatGPT記事で「経験があるほどAIを使いこなせる」と書いた。  
これはClaude Codeでも同じ。

Claude Codeに「LPを作って」と丸投げしても、一般的なテンプレートみたいなものが出てくるだけ。  
でもそれなりの経験があると、指示の出し方が違う。

「ファーストビューにCTAを置いて、スクロールしなくても問い合わせできる構成にして」  
「セクション間の余白は80pxで統一、モバイルでは40pxに」  
「フォントはNoto Sans JPで、見出しは700、本文は400」

こういう具体的な指示が出せるのは、いくつもWebサイトを作ってきた経験があるから。  
AIは「指示の精度」で出力が変わる。  
この原則は、どのツールを使っても同じ。

---

## まとめ：コーディングのAI活用は次のフェーズに入った

ChatGPTで「コードを書いてもらう」時代から、Claude Codeで「コードを書かせる」時代に変わりつつある。

「書いてもらう」と「書かせる」の違いは、コピペの有無。  
たった1ステップの差だけど、作業の体感が全然違う。  
ディレクターがコーダーに指示を出す感覚で、AIにコーディングを任せられる。

乗り換えかけてる、と書いたけど、正確には「使い分けが明確になった」。  
コーディングはClaude Code、それ以外はChatGPT。  
Webデザインをやってきた人間として、今はこの組み合わせがいちばんしっくりきてる。
