Claude Codeのベストプラクティスを調べると、大抵は「具体的な指示を出しましょう」「CLAUDE.mdを活用しましょう」という話になります。それ自体は正しいのですが、なぜそれが必要なのかを知らないと、どこまでやればいいか分からなくなりがちです。

公式ドキュメントを読んで気づくのは、ほぼすべての推奨事項が1つの制約から来ているということです。

**コンテキストウィンドウは想像より早く詰まる。**

それだけです。これを理解すれば、なぜCLAUDE.mdが必要なのか、なぜ探索→計画→実装のステップを踏むのか、なぜサブエージェントが有効なのか、全部つながります。この記事では実際に手を動かしながら、その流れを体験できるように整理しました。

## 環境を整える前に：パーミッション設定

最初に片付けておきたいのがパーミッションです。デフォルトのClaude Codeは、ファイル書き込みやBashコマンドのたびに確認を求めてきます。最初は安心感がありますが、10回クリックした頃には惰性でOKを押しているだけです。

`/permissions` を実行して、安全なコマンドをホワイトリ
