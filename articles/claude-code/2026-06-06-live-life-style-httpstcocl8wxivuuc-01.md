# 「うちの業界は特殊だから、AIには分からない」

https://x.com/arceyul/status/2062847275471175976

整理してみた

そう思ったことはありませんか？

正直に言うと、その感覚は半分正しいです。

AIは、あなたの会社の事情を最初から知っているわけではありません。

- 業界特有の商習慣
- 顧客の警戒心
- 使ってはいけない表現
- 法務・規制上の注意点
- 社内だけで通じる略語
- ベテランが感覚で判断している基準
- 顧客ごとの温度感
- 商品ごとの提案順序
こうしたものは、AIには見えません。

しかし、もう半分は違います。

AIに分からないのではなく、
こちらがAIに分かる形で説明していないだけです。

今回、Claude Codeを構築したエンジニアが語っていた「実際に機能するプロンプトの書き方」は、開発者だけの話ではありません。

むしろ、特殊な業界で働く営業、企画、広告、採用、管理職にこそ必要な考え方です。

なぜなら、特殊業界の最大の課題は、業務が属人化しやすいことだからです。

この記事では、Claude Codeの考え方を、非エンジニアでも分かるように翻訳します。

そして、特殊な業界でAIを実務に使うための具体的なプロンプトまで紹介します。

# 結論：特殊業界でAIを使う鍵は、プロンプトではなく“文脈設計”である

AIに成果を出させるには、
「良い感じに作って」
では足りません。

必要なのは、文脈設計です。

文脈設計とは、簡単に言えば、AIに仕事を頼む前に、次の情報を渡すことです。

- 自社は何をしている会社なのか
- 顧客はどんな人たちなのか
- 何を大切にしている業界なのか
- どんな表現を避けるべきなのか
- どの順番で提案すべきなのか
- 何をもって良い成果物とするのか
- 最後に何をチェックすべきなのか
この文脈を渡さないままAIに頼むと、一般論が返ってきます。

一般論は綺麗です。

でも、現場では使えません。

特殊業界で必要なのは、綺麗な文章ではなく、
顧客の温度感に合った、実務で使える文章です。

# まず定義：Claude Codeの考え方とは何か

Claude Codeは、本来はエンジニアがコードを書くためのAIツールです。

ただし、今回重要なのは、プログラミングの話ではありません。

重要なのは、AIの使い方の型です。

Claude Code的な使い方を、普通の業務に置き換えるとこうなります。

1. 現状を読ませる
2. 前提を理解させる
3. 作業前に計画させる
4. 実行させる
5. 結果を検証させる
6. 良かったルールを記憶させる
7. 次回以降に使い回す

多くの人は、いきなり4番から始めます。

メールを書いて。
記事を書いて。
提案文を作って。
資料を作って。

だから、薄いものが出てきます。

本来は、1番から始めるべきです。

まず顧客情報を読んで。
業界前提を理解して。
提案前に不足情報を出して。
作業計画を作って。
そのうえで文章を作って。
最後にチェックして。

この違いが、AI活用の成果を分けます。

# 特殊業界でAIがズレる理由

## 理由1：業界の常識を説明していない

人間の社内では当たり前でも、AIには当たり前ではありません。

たとえば、特殊業界では次のような暗黙知があります。

- この言い方は顧客に失礼
- この単語は使うと警戒される
- この順番で提案すると断られやすい
- この数字は外部に出してはいけない
- この商材は新規より既存向け
- 決裁者には数字、現場担当者には負担軽減を伝える
- いきなり売り込むと関係が壊れる
- 相手の表面的な課題と本音は違う
こうした情報を渡さずに、
「いい感じに作って」
と言っても、AIは一般的な回答しか出せません。

## 理由2：判断基準が人の頭の中にある

特殊業界では、ベテランが無意識に判断していることが多いです。

たとえば、営業であればこうです。

- この顧客は今は攻めるべきではない
- この担当者は短く話した方がいい
- この会社は価格より安心感を重視する
- この案件は今月ではなく来月追うべき
- この顧客は資料より事例を見せた方が刺さる
- この表現は少し強すぎる
- この書き方だと押し売りに見える
これを人間が言語化していなければ、AIには再現できません。

つまり、AI活用は、
ベテランの暗黙知を言語化する作業でもあります。
