## Claude Codeとは？Claude.aiとの決定的な違い

Claude Codeは、Anthropicが提供すエージェント型AIコーディングツーです。ターミナル（黒い画面）、VS Code、デスクトップアプリ、ブラウザの4つの環境から使えます。

ルる

### ブラウザのAIと「エージェント型AI」の違い

普段使っているClaude.aiやChatGPTは「チャット型」です。質問→回答のやり取りで、実際の作業はあなた自身が行います。

Claude Codeは「エージェント型」。あなたが「〇〇して」と伝えると、AIが自分でファイルを開き、コードを書き、コマンドを実行し、結果を確認するま自律的に動きま。  
すでClaude.ai（チャット型）とClaude Codeの違い  
- Claude.ai：質問に答えてくれる。作業はあなたがやる  
- Claude Code：指示を出すとAIが実際に作業してくれる  
- Claude Code：複数ファイルを横断して理解・編集できる  
- Claude Code：ターミナル操作・Git操作まで自動でやってくれる

公式ドキュメントより：https://code.claude.com/docs/ja/overview

### どんな人に向いている？

コードを書く機会があるエンジニア（初心者〜上級者）  
繰り返しの定型作業を自動化したい非エンジニア  
「AIに丸投げして仕事を減らしたい」すべての人

## Claude Codeでできること7選

実際に私が使ってみて「これは便利すぎる」と感じた機能を7つ厳選しました。

### ①コード生成・バグ修正

一番基本的な機能ですが、Claude Codeが優れているのコードベース全体を理解した上で作業してくれ点です。

るは

```
claude "ログイン機能のバグを修正して。エラーメッセージはこれ: [エラー貼り付け]"
```

これだけで、関連するファイルを自動的に読み込み、原因を特定し、修正し、動作確認まで行います。「どのファイルを編集すればいい？」と悩む必要がありません。

### ②Git操作の自動化

コードを書いた後のコミット・プルリクエスト作成まで全部やってくれます。

```
claude "今の変更をコミットして。意味のあるメッセージを付けて"
```

CI/CDと連携すれば、Slackで「@Claude このバグ直して」とメンションするだけでプルリクエストが作成される、という夢のようなワークフローも実現できます。

### ③複数ファイルの一括処理

「プロジェクト全体のコメントを英語から日本語に変換して」「全ファイルのインデントを統一して」といった大量作業も一発です。

```
git diff main --name-only | claude -p "変更されたファイルのセキュリティ問題をチェックして"
```

### ④MCP連携で外部サービスと繋ぐ

MCP（Model Context Protocol）という仕組みを使うと、Claude CodeをSlack・Jira・Notion・Google Driveなど300以上のサービスと繋げられます。

「JiraのTODOチケットを読んで、対応するコードを書いてPRを出して」が1つのコマンドで完結します。非エンジニアでも使えるレベルの自動化です。

### ⑤エージェントチームで並列作業

Claude Codeの中でも特に先進的な機能です。複数のClaude Codeを同時に起動して、異なるタスクを並行処理させられます。

エージェントA：フロントエンドのUI修正  
エージェントB：バックエンドのAPI実装  
エージェントC：テストコードの作成  
この3つを同時並行で走らせ、リードエージェントが調整・統合するイメージです。大規模なプロジェクトで力を発揮します。

### ⑥スケジュール実行・定期自動化

毎日・毎週・特定のタイミングに自動実行する設定ができます。

毎朝のPRレビュー自動化  
夜間のCI失敗分析  
週次の依存関係セキュリティチェック  
「ルーティン」機能を使えば、コンピューターがオフの状態でもAnthropicのサーバー上で処理が走ります。

### ⑦CLAUDE.mdで行動をカスタマイズ

プロジェクトルートにCLAUDE.mdというファイルを置くと、そこに書いたルールをClaude Codeが毎回読み込んで動作します。

```
# このプロジェクトのルール

これを書いておくだけで、毎回同じ指示を繰り返す必要がなくなります。チームで使う場合にも特に効果的です。

- Pythonはバージョン3.11以上を使う
- コメントは必ず日本語で書く
- テストなしのPRは作らない
```
