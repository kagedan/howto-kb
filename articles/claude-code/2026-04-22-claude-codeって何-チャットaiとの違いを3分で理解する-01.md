## 背景

「Claude Codeって名前は聞いたことある。でも普通のChatGPTやClaudeと何が違うの？」

そういう疑問を持っている人が多い。2026年現在、Claude Codeは急速に普及しているが、「エージェント型」という言葉の意味がわからず、触れずにいる人も少なくない。この記事では、その疑問に答えることを目的とする。

---

## 解説

### チャットAIとClaude Codeの根本的な違い

ざっくり言うと、こうだ。

* **Claude.aiチャット**：「会話する AI」。質問すると答えてくれる。
* **Claude Code**：「実際に作業するAI」。指示すると、ファイルを開いて編集し、コマンドを実行して、結果を確認するまでやってくれる。

チャットAIは「答えを返す」だけだ。Claude Codeは「手を動かす」。この差が大きい。

### 「エージェント型」って何か

エージェント型とは、AIが自分でタスクを分解して実行するタイプのことだ。

従来のAIとの違いを整理するとこうなる。

* **従来のチャットAI**：聞いたことに答える。次のステップは人間が判断する。
* **エージェント型AI（Claude Code）**：タスクを受け取る → 手順を考える → 実行する → 結果を確認する → 次のステップを判断する。このループを自律的に繰り返す。

具体例で考えると分かりやすい。「このバグを直して」と頼んだ場合、こうなる。

```
チャットAI：「原因はおそらく〜です。修正するには〜してください。」（答えるだけ）
Claude Code：ファイルを開く → バグ箇所を特定する → コードを修正する → テストを実行する → 完了を報告する（全部やる）
```

Claude Codeには2つの使い方がある。

**方法1：デスクトップアプリ（初心者向け）**

公式サイト（[https://claude.ai/download）からダウンロードするだけだ。ターミナル操作が不要で、Windows](https://claude.ai/download%EF%BC%89%E3%81%8B%E3%82%89%E3%83%80%E3%82%A6%E3%83%B3%E3%83%AD%E3%83%BC%E3%83%89%E3%81%99%E3%82%8B%E3%81%A0%E3%81%91%E3%81%A0%E3%80%82%E3%82%BF%E3%83%BC%E3%83%9F%E3%83%8A%E3%83%AB%E6%93%8D%E4%BD%9C%E3%81%8C%E4%B8%8D%E8%A6%81%E3%81%A7%E3%80%81Windows) / Mac 両対応。

**方法2：CLIツール（ターミナルから使う）**

Node.jsがインストールされている環境で以下を実行する。

```

```

インストール後、プロジェクトのフォルダで `claude` と打てば起動する。

## 実務への落とし込み

初心者が最初に試すべき使い方は3つだ。

**1. 既存のコードを説明させる**

プロジェクトのフォルダで起動して、こう頼む。

Claude Codeはファイルを読み込み、構成を把握した上で解説してくれる。引き継いだコードの把握に便利だ。

**2. エラーメッセージをそのまま渡す**

エラーが出たら、スタックトレースをそのまま貼り付けて頼む。

```
このエラーを直して：[エラーメッセージをここに貼る]
```

原因の特定から修正まで自動でやってくれる。

**3. コメントやドキュメントを書かせる**

コードを読んで、適切なコメントを書いてファイルを保存するところまでやってくれる。

---

## 注意点 / 限界 / 誤解されやすい点

**「全部任せれば動くコードができる」は過信だ。**

Claude Codeは指示通りに動く。しかし「何を作るか」「どう設計するか」の判断は人間がやる必要がある。曖昧な指示を出すと、曖昧な結果が返ってくる。

**ターミナルが苦手でも使える。**

デスクトップアプリ版があるため、ターミナル操作の知識は必須ではない。まずはアプリ版から試すのがおすすめだ。

**実行権限に注意する。**

Claude Codeはファイルの読み書きやコマンドの実行が可能だ。本番環境やデータが消えると困るフォルダで実行するときは注意が必要だ。
