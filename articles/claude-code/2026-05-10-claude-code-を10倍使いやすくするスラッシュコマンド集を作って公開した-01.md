Claude Code を使い始めて気づいたことがある。

毎回同じプロンプトを書いている。

```
「このコードをレビューして。バグ・セキュリティ・パフォーマンスの観点で、重要度別に整理して返して」
「テストを書いて。エッジケースも含めて網羅的に」
「PR の説明文を書いて。変更理由と破壊的変更を含めて」
```

これを毎回手で打つのは無駄だし、プロンプトの質がその日の気分で変わる。

そこで Claude Code のスラッシュコマンド機能を使って、よく使うプロンプトを **11個のコマンド** にパッケージ化した。これが **Claude Code Power Pack** だ。

---

## Claude Code のスラッシュコマンドとは

Claude Code では `~/.claude/commands/` にマークダウンファイルを置くと、それがスラッシュコマンドとして使えるようになる。

```
~/.claude/commands/pp/review.md
→ /pp:review として使える
```

ファイルの中身がそのままプロンプトになる。つまり **よく使うプロンプトをコマンド化して、毎回呼び出せる** 仕組みだ。

---

## Power Pack に入っている11コマンド

### `/pp:review` — コードレビュー

```
/pp:review src/auth/login.py
```

バグ・セキュリティ・パフォーマンス・保守性の4軸でレビューし、重要度別に構造化して返す。

**出力例:**

```
## 📋 Summary
全体的に堅実な実装。セキュリティに1件の重要な問題あり。

## 🔴 Critical (must fix)
1. 42行目: パスワードが平文でログに出力されている
   → `logger.info(f"Login attempt: {password}")` を削除

## 🟡 Warnings
- 78行目: SQLクエリが文字列結合でSQLインジェクションのリスク
  → パラメータ化クエリに変更推奨

## ✅ Well done
- 入力バリデーションが徹底されている
- エラーレスポンスに詳細情報を含めていない（セキュリティ上正しい）
```

---

### `/pp:test` — テスト生成

```
/pp:test src/utils/parser.py
```

ハッピーパスだけでなく、空入力・境界値・エラーケースまで網羅したテストコードを生成する。

---

### `/pp:pr` — PR説明文

現在のブランチの git diff を読んで、PR説明文とチェックリストを自動生成する。

---

### その他8コマンド

| コマンド | 何をするか |
| --- | --- |
| `/pp:security` | OWASP準拠のセキュリティ監査 |
| `/pp:debug` | バグの原因を仮説立てて体系的に診断 |
| `/pp:explain` | 3段階（誰でも／初級／上級）で解説 |
| `/pp:refactor` | リファクタリング案を理由付きで提示 |
| `/pp:docs` | 言語に合わせたdocstring生成 |
| `/pp:optimize` | 計算量分析 + 具体的な最適化案 |
| `/pp:arch` | アーキテクチャの設計問題を指摘 |
| `/pp:standup` | git履歴から朝会報告文を生成 |

## インストール方法

**30秒で完了する。**

```
# Mac / Linux
git clone https://github.com/JunCola1017/claude-code-power-pack
cd claude-code-power-pack
bash install.sh

# Windows
git clone https://github.com/JunCola1017/claude-code-power-pack
cd claude-code-power-pack
.\install.ps1
```

インストール後、Claude Code を再起動すれば `/pp:` で始まるコマンドが使えるようになる。

---

## なぜ「自作」したのか

既存のプロンプト集との違いは2点ある。

1. **出力フォーマットが固定されている**  
   「コードレビューして」と打つと、毎回違う形式で返ってくる。Power Pack のコマンドは出力構造を明示的に指定しているので、いつ使っても同じフォーマットで返ってくる。
2. **実際に毎日使うものだけ入っている**  
   「あったら便利かも」ではなく「毎日使ってて面倒だったもの」だけを厳選した。
