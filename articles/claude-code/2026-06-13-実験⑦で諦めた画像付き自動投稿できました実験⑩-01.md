## 結果から先に言います

**毎朝の投稿作業が、11〜16分から4分になりました。**

**Before（実験⑦まで）**

* 投稿文を生成：1分
* 画像を作る：5〜10分（手動）
* Bufferに予約：3分（1本ずつコピペ）
* スプシに記録：2分（手動）
* **合計：約11〜16分**

* 投稿文を生成：1分
* `y`を押して待つ：3分
* **合計：約4分**

差は7〜12分。毎朝続ければ**月に3〜4時間が浮く計算**です。

## 何を変えたか

実験⑦時点では「テキストのみ自動予約」が動いた状態でした。今回は3箇所を改修しました。

`/post`の末尾に確認プロンプトを追加しました。

```
画像生成・Buffer予約・スプシ追記まで自動実行しますか？[y/N]
```

`y`を押すと：

1. gpt-image-2でキャラのイラストを3枚生成
2. Bufferに画像付きで予約
3. スプシに3行自動追記

人間がやることは確認だけです。

## ⚠️ やる前に知っておいてほしいこと：サードパーティツールのリスク

BufferのようなサードパーティツールからX（Twitter）に投稿を予約する方法は便利ですが、**Xからスパム・自動化アカウントと判定されるリスクがあります。**

実際、僕のアカウントは2026年5月30日に凍結されました（番外編②参照）。Xからは「偽装アカウントの疑い」と通知されましたが、\*\*本当の理由はわかりません。\*\*外部ツールの使用が関係していたかも不明です。ただ、自動化ツールを使っていたタイミングで凍結が起きたことは事実なので、リスクとして共有しておきます。

使うなら知っておきたいこと：

* **Xの公式パートナーツール**（Buffer・Hootsuite等）はX APIの利用規約内で動作しているが、投稿頻度や操作パターン次第でフラグが立つことがある
* **1日あたりの投稿数を無理に増やさない**（僕は3本/日を上限にしている）
* **急に大量の予約を入れない**（スパムフィルターに引っかかりやすくなる）
* **アカウントが育っていない初期ほどリスクが高い**（フォロワーが少ない＝信頼スコアが低い）

自動化は便利ですが、ゼロリスクではありません。「リスクを理解した上で使う」が前提です。

## 仕組みの詳細（読み飛ばし可）

技術的に興味がある方向けです。仕組みに興味がなければ読み飛ばしてもらって大丈夫です。

### ① 画像をBufferに自動送信できるようにした

実験⑦で400エラーが出て諦めた画像付き投稿。

直接バイナリを送るのではなく、**無料の画像ホスティングサービスにアップロードしてURLを取得し、そのURLをBufferに渡す**方法で解決しました。

### ② スプシへの自動追記

毎回手動でコピペしていたスプレッドシートへの記録も自動化しました。

スプレッドシート側に受け口を作っておき、PythonからデータをPOSTするだけで書き込まれる仕組みです。**APIキーも不要、OAuth設定も不要**です。

### ③ 詰まったこと3つ

今回も一発では動きませんでした。

**詰まり①：画像生成APIのパラメータエラー**

新しいモデルに切り替えた直後、旧モデル専用のパラメータが残っていてエラーが出ました。  
→ 該当の1行を削除するだけで解決。

**詰まり②：スプシへの外部書き込みが失敗した**

原因：「ブラウザから開いたときしか動かない」関数を外部からの通信に使っていた。  
→ ID指定で直接ファイルを開く方法に変更して解決。

**詰まり③：Windows日本語文字化け**

Windows環境でPythonを実行すると日本語が全文字化けしました。  
→ スクリプト先頭に文字コードの設定を2行追加して解決。
