こんにちは、kote2(こてつ)です。

非エンジニア向けの **Claude Code 初期環境構築ビルダー(kote2 Builder for Claude Code)** を作りました。現在noteで無料版と有料版を配布中です。

じゃあ、宣伝記事をZennでも書こうか、というわけではありません。このビルダーの開発秘話を書こうと思った次第。

# 非エンジニア向けClaude Code開発環境ビルダーとは？

まず軽くこのkote2 Builder for Claude Code(以下 kote2式ビルダー)の説明をしますね。

「作りたいものはなんですか？」  
↓  
「それではこの内容(設計)でどうですか？」  
↓  
「では、作成します」  
という、  
ヒアリング → 設計 → 内容確認 → 実装  
という流れでプロダクトが作れます。

## もちろん自分流にカスタマイズできます

このkote2式ビルダーは主に3つのファイルで構成されています。

### builder.md

kote2式ビルダーのコアファイル。ディレクトリ構成とか自分流にしたければ、この中のファイルを書き換えます。  
![](https://static.zenn.studio/user-upload/a09489924b45-20260606.png)

ちなみに有料版は、開発者モードというのが備わっています。以下のコマンドを打つと、ビルダーモードと開発者モードを切り替えられます。

```
cp CLAUDE.dev.md CLAUDE.md   # 開発者モードに切り替え
cp CLAUDE.live.md CLAUDE.md  # ビルダーモードに切り替え
```

何のことはなく、開発者モード用のCLAUDE.mdを作ってメインのCLAUDE.mdに上書きしてるだけですが、この切替方法意外と便利です。特にこのようなプロダクトを開発している場合は。

### aboutme.md

自分の情報をあらかじめここに書いておくと、「はじめまして」と毎回自己紹介する必要がなくなります。このファイルはビルド後も残るので、書き換えたい時は書き換えればいいのでビルドし直す必要はありません。  
![](https://static.zenn.studio/user-upload/0da84db0564d-20260606.png)  
技術スタックなんかもここに書いておくと設計時に考慮してくれます。  
![](https://static.zenn.studio/user-upload/6dea244a4623-20260606.png)  
目標やゴールも書いておくとロードマップを作成する時なんかもいちいち説明する必要がなくなります。  
![](https://static.zenn.studio/user-upload/93ac1a70b8b6-20260606.png)

### ai-behavior.md

AIへのお願いを事前に伝える設定ファイル。書き出したプロジェクトに吹き込まれます。

![](https://static.zenn.studio/user-upload/c39ec4b72491-20260606.png)

このように、ファイル構成は単純なので、簡単にカスタマイズすることが出来ます。

## リビジョン機能(セーブ機能)

ビルドしたプロジェクトを更に修正したい時、変更が伴ったら「リビジョンしておいて」「セーブしておいて」と言うとこれまでの変更履歴をリビジョンファイルで管理する設計になっており、会話を新規から始めてもゲームで言う「続きから始める」状態を維持できます。

## 自己修復機能

エラーが出たらAIが学習する仕組みを実装しています。  
![](https://static.zenn.studio/user-upload/d249692dc646-20260606.png)

※これは Anthropic 公式記事 [How Claude Code works in large codebases](https://claude.com/blog/how-claude-code-works-in-large-codebases-best-practices-and-where-to-start) で推奨されている Stop hook で自己改善の設計パターンを、非エンジニアのプロジェクトでも動くよう最小構成で組み込んだものです。

以上のような機能を持つのがkote2式ビルダーです。

# 実は1度作り直した

このkote2式ビルダーですが、実は一度作り直しています。最初作ったのはコンセプトや機能は似ているんですが、自分が使うにはいいけど配布して使ってもらうには、今のようにカスタマイズに使うところを切り出してなかったり、ファイル名が直感的でなかったり、余計な機能がモリモリだったり。  
あとフリー版と有料版を別々に作っていたので、もう自分もフリー版と有料版の違いがよくわからなくなったりしていました。

で、最初から作り直したほうがいいんじゃないかという事になり、1度通ってきた開発の流れを改善すべく、フリーと有料は両方管理するようにして開発、出来たファイルのチェックやテストを全て自動化。かなり時間短縮してスピーディーに開発できる体制が作れるようになりました。  
![](https://static.zenn.studio/user-upload/13e245132c3b-20260606.png)

ちなみに有料と言っても現時点であまり違いはなく、ただこのビルダーを使っている人達で情報共有しあえる場をnoteコミュニティに持つということにしました。

## リリースして思ったこと

まず、早く公開したいというジレンマ。実は、このビルダーは非エンジニア向けのClaude Code講座の付属品として開発したものです。これを使えばClaude Codeのお作法的なところは全部やってくれるから作りたいものに集中できるよー、みたいな感じで。なのでClaude Code講座を書き終えたら同時に出そうと思ってました。

ところが、似たようなものを公式が出してきました。

<https://claude.com/plugins/claude-code-setup>

似てると言っても別物なんですけど、このようにAIが絡むと他が出してしまう焦りが常につきまといます。ただ、自分のは一度作り直しているほどなので一朝一夕で作れるものではないと思いつつ、いやAIがあれば作れるかもとも思いつつ。ね？ジレンマです。

という事で、Claude Code講座は基礎編と上級編で分けて基礎編が出来た時点で公開と、実は公開を早めました。

## 作ったものをお金に変えるには

これプロ版はnoteのメンバーシップ、しかも5000円/月という値段をつけています。恐らく高い部類だと思いますが、しっかりバージョンアップし、コミュニティ内で活用事例とか出し合って盛り上がってくれるといいと思ってます。まだ出したばかりでPRもしてないので閑散としていますが。

最近益々AIによって仕事を奪われるみたいな話が現実味を帯びてますが、このようなオンラインコミュニティを作るというのは海外で新しいビジネスとして注目されているようです。日本だとどうしてもオンラインサロンみたいな胡散臭いものを想像してしまいますが、しっかりした人徳で集めるという感じのものです。

私自身、一次情報はXじゃなく海外のオンラインのコミュニティに入って収集しています。そこは月15,000円です。ただ共有される情報は実際AIで稼いでる人達で、話されるテーマは「AI活用してどうお金に変えていくか」を真剣に考える人が集まったコミュニティです。（3000人以上会員がいるので少なく見積もって・・・そういう世界です）

でも、新しい形でお金に変えていかないとほんと生き残れないと思うので、早いうちからこういう形を作りたかったという裏話でした。
