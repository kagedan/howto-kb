# Pythonほぼ未経験の私がClaudeと一緒に電子書籍キャプチャツールを作った話

## はじめに

プログラミングはほぼ未経験です。  
でも「自分で買った電子書籍を自分のパソコンに保存したい」という気持ちだけで、AIと一緒にPythonツールを作りました。

結果、**ちゃんと動くツールが完成しました。**

この記事では「どうやってAIに指示を出したか」「どんな壁にぶつかったか」を正直に書きます。同じように「作りたいものはあるけどコードが書けない」という人の参考になれば嬉しいです。

---

## 作ったもの

**汎用電子書籍キャプチャツール**

* 電子書籍ビューアを自動でページ送りしながらスクリーンショットを保存
* 最後にPDFにまとめる
* GUIで操作できる（ボタンを押すだけ）

GitHubで公開しています  
<https://github.com/mamez31/ebook-capture-tool>

---

## やったこと

### Step1：まず仕様書を作った

いきなりコードを書いてもらうのではなく、**「何を作りたいか」をまとめた仕様書を先に作りました。**

Claudeと壁打ちしながら、こんな内容を詰めました。

* どんな機能が必要か
* GUIで何を設定できるようにするか
* ファイルの保存先はどうするか
* 終了検知はどうするか

仕様書をしっかり作ったことで、AIへの指示がブレなくなりました。これが一番大事だったと思います。

### Step2：Phase分けで少しずつ作った

一度に全部作ってもらおうとすると、コードが複雑になってバグだらけになります。

なので**5つのPhaseに分けて、動作確認しながら進めました。**

| Phase | 内容 |
| --- | --- |
| Phase1 | GUI・キャプチャ・保存・テストモード |
| Phase2 | PDF生成・見開き分割 |
| Phase3 | image終了検知（OpenCV） |
| Phase4 | テキスト終了検知（OCR） |
| Phase5 | ヘルプ画面・ログ改善 |

各Phaseごとに「動いた！」を確認してから次に進むので、どこで問題が起きたかすぐわかります。

### Step3：Claude Codeに指示を出した

実際のコード生成はClaude Codeを使いました。

指示の出し方はシンプルです。

```
Phase1を実装してください。

【実装範囲】
・tkinter GUI
・スクリーンショット取得
・画像保存
・停止ボタン
・テストモード

【まだ実装しないもの】
・PDF生成
・OCR
・終了検知

【重要】
まずは「確実に動く最小構成」を優先してください。
```

ポイントは\*\*「まだ実装しないもの」を明示すること。\*\*  
書かないとAIが全部一度に実装しようとして、収拾がつかなくなります。

## 詰まったところ

### Tesseractのインストール

OCR機能を使うためにTesseractというソフトが必要だったのですが、インストール後にPATHが通っておらずエラーになりました。

解決策：PowerShellで環境変数を設定して解決。

### 見開き表示の扱い

見開きで分割すると1ページ目だけズレる問題が発生。電子書籍ビューア側の仕様が原因でした。

解決策：見開きオフで撮影する運用に変更。
