この記事を読めば、Web制作のどの工程が、どこまで自動化できるのかが分かります。

Claude Codeって、コーディングを自動化する道具だと思われがちです。

僕も最初はそういう認識でした。

でも実際に毎日の制作で使い込んでいくと、自動化できるのはコーディングだけじゃなかったんですよ。

ヒアリングの議事録、制作前のチェック、ワイヤーフレーム作成、見積もり、納品前の品質チェック、WordPress化、保守まで。制作のほぼ全工程で使えます。

コーディングだけAIに任せて、それ以外の業務にはそれほど使ってないって人もいるんじゃないかと。

でも正直、コード生成だけに使うのは宝の持ち腐れなんですよね。

むしろ、その前後にある細かい作業を全部AIに渡せるのが大きいです。

なので今回は僕がWeb制作の中で実際に自動化してる工程を何を・どうやって・どれくらい変わったか、上から順に全部書き出します。

どれも、特別なスキルがなくてもできることです。

# ① ヒアリングと議事録づくりはAIに任せている

案件が動き出すとまず打ち合わせがありますよね。

あの議事録づくり、割としんどい作業です。

今は打ち合わせを録画しておいて、その音声をAIに渡すだけ。

文字起こしから議事録の整形まで自動でやってくれます。

決定事項と次回やること、共有事項に分けてまとめてくれるので、人がやると1時間かかってたのが5分で終わります。

リサーチもここに含めてます。

競合サイトの分析とか、ターゲットの調査とか。

自分1人の目で見るより抜け漏れが少ないんですよね。

半日かかってた準備が1時間もかからない。

僕が教えた方だと、このリサーチを8割くらい自動化して、その分の時間をディレクションやお客さんのAI相談に回せるようになってました。

# ② 制作着手前のチェックはコマンド一発で37項目見てくれる

目立たないけど、かなりラクになったやつです。

デザインカンプを渡すだけで、コーディングに入る前に37項目を自動でチェックしてくれるスキルを作りました。

ブレイクポイント、使われてるフォント、カラーパレット、共通パーツ、表記の揺れ（2026年と2022年が混ざってる、みたいなやつ）、PCとSPで違う箇所、実装が難しそうなところ。

これを5〜6分で洗い出してくれます。

なんでこれが大事かというと、制作してて「想定より工程増えたな」ってなるとき、原因の7割くらいは制作前のチェックの甘さだからです。

着手前にここを潰しておくと、後で慌てないし、見積もりのミスも減ります。

チェック項目は自分でファイルを書き換えれば増やせるので、案件で踏んだ地雷をどんどん足していけます。

# ③ ワイヤーフレームは議事録を渡すだけで出てくる

ワイヤー作成って、本来コーダーの仕事じゃないと思ってる人が多いと思います。

でもここをAIに任せられると、コーダーのままディレクションの領域に入れます。

やってることはシンプルで、議事録やヒアリング内容を渡すと、競合リサーチ→ページ構成の提案→承認→そのままFigmaに反映までを一気にやってくれます。

カード型とかCTAとかヘッダーとか、よく使うパーツを30種類くらいテンプレ化してあるので、構成案の精度もそこそこ高いです。

とはいえ出てくるのは構成のたたき台です。

業種ごとの定番セクション、たとえば士業なら料金表や実績、店舗なら地図やメニューが抜けてないかは自分で確認します。

そこを足す前提でいうと、ゼロから構成を考える時間がまるごと消えます。

これで丸1日かかってたワイヤー作成が、最終調整まで含めて30分〜1時間で終わります。

# ④ 見積もりも算出から提案文まで自動化した

見積もりって、毎回「これでいいのかな」と不安になる作業じゃないですか。

ここはステップ式で回答してくれるAIに任せてます。

サイトの種類、ページ数、必要なパーツ、業務範囲を順番に聞かれて、料金表を参照しながら金額を出して、最後はクライアントに送る見積書つきのメール文面まで作ってくれる。

料金表は別ファイルにしてあるので、値上げしたらそこを直すだけで全部に反映されます。

抜け漏れって人間だから起きるんですよね。

工数を1個見積もり忘れる、みたいな。これを定型化しておくと、その事故が減ります。

改修案件のときは、受注前にAIにコードを解析させる感じです。

中身の設計を理解した上で「これは受けて大丈夫か」「工数はどれくらいか」を判断できるので、

見積もりの精度が上がるので蓋を空けたら地獄なようなコードで想定より遥かに工数が多かったみ
