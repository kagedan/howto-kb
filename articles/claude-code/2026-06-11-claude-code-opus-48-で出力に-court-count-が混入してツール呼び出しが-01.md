## 症状

Claude Code を使っていると、ある時点から出力に `court` や `count` といった脈絡のない英単語が混ざり込み、

* ツール（コマンド）が実行されないままターンが終わる
* 生のツール呼び出しらしきテキストが出力欄に漏れて表示が汚れる

という状態になる。一度始まると、同じセッション内ではリトライしてもほぼほぼ直らない。

長時間立ち上げっぱなしのセッションや、日本語を多く扱うセッションで踏みやすい。X でも同様の報告が見られる。

## 原因

ターミナルの文字化け（グリフ崩れ）でも、API の汎用的な配信不具合でもなく、**Claude Opus 4.8 に固有のリグレッション**である。

ツール呼び出しは本来、構造化された `tool_use` ブロック（内部的には `<invoke>` 等の制御マーカー）として出力される。Opus 4.8 では、この **ツール呼び出しの開始マーカーが `court` / `count` などの実在英単語トークンに化ける** ことがある。結果として、

1. パーサーがツール呼び出しを認識できず、`tool_use` がただのテキストとして表示される（＝出力欄の汚染）
2. コマンドが実行されないままターンが終わる（＝実行できずに止まる）

ランダムな文字化けではなく "それっぽい単語" になるのは、ツール呼び出しの境界制御トークンのサンプリングが破綻し、その近傍にある実在語トークンが漏れ出ているためと見られる（正確な機構は未確認）。

### 厄介な点：自己強化ループ

一度壊れた出力が会話履歴に残ると、モデルがそれを「正しい出力フォーマット」と誤認して模倣し続ける。このため **リトライしても自己修復しない**。`--continue` は壊れた履歴を読み戻すので、再発の引き金になる。

## 発火条件

報告で共通している条件：

* **Opus 4.8**（4.7 では起きない。4.7 の類似バグは修正済み）
* **長時間・大きく蓄積した文脈**（特に 1M context の常時起動セッション）
* **非 ASCII＝日本語の引数**（ツール引数に日本語が混ざると踏みやすい）

GitHub の報告では `platform:windows` ラベルが付いているが、トリガーの本体は「Opus 4.8 × 長文脈 × 日本語引数」なので、WSL などでも条件が揃えば発生し得る。日本語ユーザーが特に踏みやすいわりに、英語圏の GitHub では過少報告気味。

## 回避策

### 発生してしまったら（履歴の汚染を断つ）

* `Esc` ×2 または `/rewind` で、壊れる前のチェックポイントに戻す
* `/compact` で履歴を要約し、模倣ループを断ち切る
* 重症なら `/clear` でフルリセット
* `--continue` は使わない（壊れた例を読み戻すため）

### そもそも踏まないために

1. **Opus 4.7 に下げる**（最も確実。能力差はごく僅か）
2. **タスクを分解** してターンを短く保つ（文脈が軽くなりトリガーを踏みにくい）
3. **`/compact` をこまめに** ＋ 守らせたい制約は `CLAUDE.md` に逃がし、揮発的な履歴の外に置く
