## 🔓 Claude Mythos / Capybara — 「次のClaude」がリークで判明

3月26日、[Fortune](https://fortune.com/2026/03/26/anthropic-says-testing-mythos-powerful-new-ai-model-after-data-leak-reveals-its-existence-step-change-in-capabilities/)がスクープした。Anthropicの未保護データストアから、未発表モデルのブログ記事ドラフトが丸ごと見つかった。

### 何がわかったか

* モデル名: **Claude Mythos**（別バージョンでは **Capybara**）
* ポジション: **Opusの上位ティア** — これまでの最上位を超える新カテゴリ
* 性能: コーディング・学術推論・サイバーセキュリティで「劇的に高いスコア」
* 自社評価: 「**前例のないサイバーセキュリティリスク**」「他のどのAIモデルよりも大幅に先行」
* 状況: 早期アクセス顧客によるトライアル中。リリース時期は未定

ドラフトが2つあって、v1が「Mythos」、v2が「Capybara」。Anthropicは「公開を検討中だった初期ドラフト」と[説明](https://the-decoder.com/anthropic-leak-reveals-new-model-claude-mythos-with-dramatically-higher-scores-on-tests-than-any-previous-model/)しとる。

### サイバーセキュリティ能力の実績

先週の話やけど、Opus 4.6がMozillaと協力してFirefoxの脆弱性を**2週間で22件**発見した実績がある。その上位モデルとなると、脆弱性発見の速度と精度がさらに上がる。攻撃にも防御にも使える諸刃の剣や。

### Claude Code ユーザーへの影響

MythosがClaude Codeで使えるようになったら、コーディング能力のジャンプが期待できる。Opusでも十分すごいのに、その上があるっていうのは純粋にワクワクする。ただ、レートリミットがどうなるかは未知数。Opusでもカツカツの人がMythosに行ったら、もっとキツくなるんちゃうかな。

### おっちゃんの一言

自分とこのブログ原稿がダダ漏れになるっていう、**セキュリティ最強モデル作った会社がセキュリティ最弱のミスやらかした**のが最高に皮肉。「教訓はいつも足元にある」ってやつや。

## 📊 3月のClaude Code — 1年分を1ヶ月でやった月

![3月のClaude Code 9機能タイムライン](https://static.zenn.studio/user-upload/deployed-images/7b441459e43013dabd398d76.png?sha=39f1d9c38f5b79429db054876fe99fbc5deb506d)

3月だけで追加された主要機能を並べると、ちょっと引くくらいの量やねん。まるで引っ越しの日に段ボール全部一気に開けたみたいな状況や。

| 機能 | 概要 | おっちゃんの一言 |
| --- | --- | --- |
| **Computer Use** | macOS GUI操作（Pro/Max） | ターミナルの外に出た。怖さもある |
| **Voice Mode** | `/voice` で音声入力、push-to-talk | 手が塞がってるとき助かる |
| **Auto Mode** | 許可確認の自動判定 | 新入社員が判断できるようになった |
| **/loop** | 定期監視コマンド | 放置運用の入り口 |
| **Channels** | Telegram/Discord連携 | モバイルからタスク投げ |
| **--bare** | CI/CD向け高速起動 | スクリプト組む人向け |
| **出力 64K→128K** | Opus 4.6の最大出力倍増 | 長いファイル一発生成 |
| **Opus 4.6 移行** | 旧Opus 4/4.1 API廃止 | 自動移行されとる |
| **1Mコンテキスト** | 標準価格で一般提供 | でかいコードベースが全部入る |

[Builder.ioのまとめ](https://www.builder.io/blog/claude-code-updates)を見ると、普通のツールなら四半期ごとに1つずつ出しそうな機能が全部3月に来てる。追いかけるだけで息切れするわ。

そしてこの上にMythosが控えてる。4月、どうなるんやろな。

> 出典: [Builder.io](https://www.builder.io/blog/claude-code-updates), [Claude Code Changelog](https://code.claude.com/docs/en/changelog)
