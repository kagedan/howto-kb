朝、無人で回しているジョブが結果ログだけ残してプロセスが返ってこない。claude -p は最終結果を出力しているのに終わらない。終了コードが立たないから cron からは成功とも失敗とも見えず、次の起動が前のインスタンスと重なる。失敗より、この「黙って終わらない」状態のほうが無人運用ではずっと怖い。

Claude Code 2.1.163 で、この症状の一種が直った。バックグラウンドで起動したコマンドが終わらないと claude -p が最終結果後に永久ハングしていた不具合が修正され、stdin を閉じたあと約5秒でバックグラウンドシェルを止めてプロセスが抜けるようになった。「いつか終わるかもしれない」が「5秒後には必ず畳む」に変わった、というのがこの修正の本質だ。無限のハングは検知も回収もできないが、有限のタイムアウトなら検知して、記録して、リトライに回せる。

ただし注意したいのは、アップデートで直るのは「終わらない系」だけだということ。大量並列で 429（レート制限）が返る、長時間処理が利用枠の上限で止まる、クライアント再起動でジョブごと消える——こうした別系統の止まり方は、このハング修正とは無関係で、運用側の手当てが要る。「2.1.163 で全部解決した」と思って無対策で並列を流すと、別の死因で同じように詰まる。修正が境界を引いたのは、あくまで「最終結果後にバックグラウンドシェルが居座る」という一つの止まり方だけだ。
