## 「AIで仕事が変わる」と言われても、自分の業務は変わらなかった

ChatGPTが出てから数年。書店に行けばAI活用本が並び、社内でも「AIで効率化」という言葉が飛び交います。にもかかわらず、多くのビジネス職の方から同じ声を聞きます。

これは使う側のリテラシーの問題ではありません。**ツールと業務のミスマッチ**が大きい、というのが結論です。

そしてここ1年、この状況を変えつつあるのが **Claude** と、その派生である **Claude Code** です。本記事では、エンジニアではないビジネス職の方がClaudeを業務で使いこなすための入り口を整理します。

## Claude / Claude Codeとは(30秒で)

* **Claude**: Anthropic社が開発するAIアシスタント。ChatGPTと同じく、チャット形式で対話する生成AIです。
* **Claude Code**: もともとエンジニア向けに提供された「ターミナルで動くClaude」ですが、**ファイルを直接読ませたり、フォルダ単位で作業させられる**点が強力で、非エンジニアでも使える場面が一気に増えました。
* **料金**: 無料プランのほか、個人向けに Pro ($20/月)、Max ($100 または $200/月)、法人向けのTeam / Enterprise プランがあります(2026年4月時点)。Claude Codeは基本的にProプラン以上で使えます。

重要なのは、「ChatGPTの上位互換」ではなく、**得意分野が違うAI** という点です。どちらか片方だけ使うよりも、用途で使い分けるのが現実的な落とし所になります。

## なぜChatGPTでは詰まりやすかったのか

非エンジニアの業務で頻出するのは、「**長い文章や資料を相手にする仕事**」です。議事録、契約書、提案資料、報告書、調査レポート。これらに対して、従来のAIツールには2つの壁がありました。

### 壁1: 文脈ウィンドウの狭さ

ChatGPTの標準的なモデルは一度に扱える文字数に限界があり、長い議事録を貼り付けると途中から読まれない・要約が浅くなる、といった現象が起きがちです。

Claude(特に2026年4月に公開された**Claude Opus 4.7**)は、標準で **100万トークン**(日本語換算でおおよそ70〜80万字)の文脈を扱えます。これは、中編小説まるごと・数時間分の議事録・数十ページのPDFを**そのまま丸投げできる**レベルです。

### 壁2: 丁寧さと率直さのバランス

Claudeはもともと「**helpful, harmless, honest**(役に立つ・害がない・誠実)」を設計思想に据えたモデルで、実務の観点で言うと以下の挙動が出やすい傾向があります。

もちろんClaudeも万能ではなく、ハルシネーション(事実でないことを自信満々に書く)はゼロではありません。ただ、長文処理と「盛りすぎない文章生成」の組み合わせは、ビジネス職の日常業務と相性が良いのです。

## 非エンジニアが今日から使える5つのユースケース

ここからは具体的なシーンです。いずれも、コードは一切書きません。

### ユースケース1: 1時間の会議議事録を、構造化された議事録と宿題リストに変換する

**シーン**: 週次の営業戦略会議。録音データを書き起こした7,000字のテキストが手元にある。これを「議事メモ」と「次週までのアクション一覧」にまとめたい。

**やり方**: 書き起こしテキストを全文貼り付け、以下のように指示します。

```
以下は弊社の営業戦略会議の書き起こしです。
1. 議題ごとに「決まったこと」「保留・持ち越し」に分けて箇条書きにしてください。
2. 発言者ごとに、次週までの宿題(担当・期限・内容)を抽出し、表形式でまとめてください。
3. 読み手は会議に参加していない役員です。事実ベースで、推測は含めないでください。
```

Claudeは7,000字程度であれば軽々と処理します。ChatGPTでも同じことはできますが、**「推測を含めない」の指示への忠実度**が高く、元文書にない情報が混じりにくいのがポイントです。

### ユースケース2: 社外向けメールの「角度違い3パターン」を一瞬で出す

**シーン**: 値上げのお願いメールを送りたい。関係性の濃さや相手の立場によって、強め・標準・柔らかめの書き分けが必要。

**やり方**:

```
添付の過去メールを参考に、文体を合わせてください。
以下の要件で、同じ内容のメール文を3パターン書いてください。
- パターンA: 長年の取引先向け、関係性重視、柔らかめ
- パターンB: 新規取引先向け、ビジネスライクに事実ベース
- パターンC: 担当者が強く抵抗しそうな相手向け、根拠強め

内容: 2026年7月から原材料高騰に伴い、〇〇商品の価格を5%改定させていただきたい。協議の場を設けたい。
```

**ポイント**: 過去メールのスタイルを学習させてから書かせると、「なんかAIっぽい文」を避けやすくなります。

### ユースケース3: 提案資料の骨子を、ヒアリングメモだけから作る

**シーン**: 商談ヒアリング後、1週間以内に提案資料を持っていく必要がある。PowerPointの構成から考えるのが毎回重い。

**やり方**:

```
以下はクライアントへのヒアリングメモです。
この内容をもとに、提案資料(全体で15スライド以内)の骨子を作ってください。

構成は以下の形で出してください:
- スライド番号 / タイトル / このスライドで伝える1メッセージ / 載せるべき要素(図表・データ・事例など)

制約:
- 最初の3スライドで「相手の課題」「なぜ今か」を示すこと
- 価格の話は後半(12スライド目以降)に置くこと
```

提案の「流れ」をゼロから考える時間が最も重いので、ここを2割まで圧縮できるだけで効きます。Claudeは長いヒアリングメモでも文脈を失わないため、「ヒアリングで聞いた相手の言葉をスライドに反映する」精度が高めです。

### ユースケース4: Excel / スプレッドシートの下ごしらえ

**シーン**: 部門別の売上データ(CSV)を、地域 × 製品カテゴリのクロス集計に整えたい。ピボットテーブルが毎回うまく組めない。

**やり方**: CSVファイルをそのまま添付し、以下のように指示します。

```
添付のCSVを読み込み、以下の形で整えてください。
1. 行: 地域(北海道〜沖縄の9ブロック)
2. 列: 製品カテゴリ
3. 値: 売上合計(千円単位)
4. 空白セルは0埋め、合計行と合計列を追加

整えた結果を、コピペでスプレッドシートに貼れるタブ区切りテキストで出力してください。
```

Claude Codeを使えばさらに一歩進んで、**ローカルのExcelファイルを直接書き換える**ことも可能です。「コーディングじゃん」と思われるかもしれませんが、ユーザーが書くのは日本語の指示だけです。

### ユースケース5: 競合リサーチの要約と比較表

**シーン**: 新規サービスの企画中。競合3社のサイトやプレスリリースをざっと把握したい。

**やり方**: 競合3社の公式サイトURL・製品ページ・直近のプレスリリースURLを一覧で渡し、

```
以下3社の製品ページとプレスリリースを読み、以下の観点で比較表を作ってください。
- ターゲット顧客 / 主要機能 / 価格モデル / 差別化キーワード / 最近1年の動き

情報が公開情報から読み取れない項目は「記載なし」と明記してください。
推測や一般論で埋めないでください。
```

「**情報がない項目は無理に埋めない**」と明示するのがコツ。これを書かないと、AIはそれっぽい一般論で穴埋めしがちです。

## やりがちな失敗と、その回避法

### 失敗1: プロンプトを「丸投げ」してしまう

「いい感じにまとめて」「うまく書いて」は、AIに対しては曖昧すぎる指示です。最低限、「誰向け」「何のため」「どんな形式で」の3点を言語化するだけで、出力品質は大きく変わります。

### 失敗2: 機密情報をうっかり貼る

ClaudeもChatGPTも、入力した内容がプラン・設定によっては学習に使われる可能性があります。個人情報・顧客企業名・契約金額・未公表情報を扱うときは、

この3点を習慣化するのが安全です。

### 失敗3: AIの出力をノーチェックで出す

AIは自信満々に間違えます。特に数字・固有名詞・法的表現は、**必ず人間が裏を取る**工程を残してください。むしろAIを使うことで、「チェックする時間が浮いた分を、本当に重要な検証に回せる」と捉えるのが現実的です。

### 失敗4: 出力の「最初の答え」で満足する

Claudeに限らず、生成AIは対話しながら磨き込むツールです。最初の出力に対して「もう少し柔らかく」「この部分を厚めに」と追加指示を重ねていくと、5回目くらいで実用品質になります。**1発で完璧を期待しない**のが結果的に早い。

## 「もう一段深く使いたい」と思った方へ

ここまで読んで、「自分の仕事のこの場面で使えそう」と具体的なシーンが1つでも浮かんだなら、導入の準備はできています。

ただ、実際にやってみると次の壁が来ます。

* 「毎回プロンプトを考えるのが面倒」
* 「同僚が使っているプロンプトが気になる」
* 「出力の質が日によってブレる」

これを解決するのが、**自分専用のプロンプトテンプレートを持つ**ことです。使うシーンごとに「型」を持っておけば、毎回ゼロから考える必要がなくなります。

## まとめ

* ChatGPTで手応えがなかったのは、多くの場合「長文処理」と「丁寧な指示への追従」のミスマッチ
* Claudeは長文・資料系の業務と相性が良く、非エンジニアの日常業務で差が出やすい
* 議事録・メール・提案骨子・Excel・リサーチの5パターンから試すと入りやすい
* プロンプトを「丸投げしない」「機密を入れない」「ノーチェックで出さない」「1発で満足しない」の4原則を守る

AIを「すごいらしいツール」から「自分の道具」に変える最初の一歩は、**具体的なシーンで一度手応えを感じること**です。今日の業務で1つだけ、試してみてください。
