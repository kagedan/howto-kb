Claude Codeで成果が出ない原因、ほぼここにある😳
AIに渡す「コンテキストの質と速度」が出力品質を決める最重要要素。
プロンプトの書き方よりも、こっちの方が本質だ。

https://t.co/H6PTRiVarW

・Claude Codeはセッションをまたいで記憶しない。毎回ゼロから説明するか、コンテキストファイルで注入するかで、出力品質に大きな差が出る
・ObsidianのVaultはMarkdownファイル群に双方向リンクを持つ構造。通常のフォルダと違い、思考の関係性が保存される
・ObsidianをClaude Codeに参照させると、蓄積した思考・プロジェクト情報をセッション開始時に一気に注入できる

Obsidianで記録し続けることで、Claude Codeに渡せるコンテキストが育っていく。

使い込むほど出力が良くなる構造が、手元に作れる。
