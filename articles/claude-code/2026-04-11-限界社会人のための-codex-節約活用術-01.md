## はじめに

### Codexのコスパが良すぎる

こんにちは、限界社会人です。

突然ですが、私はもっぱらCodexをメインに使用してます。  
というのも、Claude Code は

* Proプラン（$17）では、 5h の Rate Limitが厳しすぎてほぼ使い物にならない
* とはいえMaxプラン（$100）を課金するほど使い倒せない
* 週末しか使わないので、5h の Rate Limit が厳しい Claude Code は相性が悪い

という背景で、ChatGPT Plus（$20）を使用しています。  
体感、ChatGPT Plus でCodexを利用するときは Rate Limit がClaude CodeのMax（$100）プランと同程度の水準で、かなり使い倒せる印象でした。

月3,000円程度でChatGPTのThinkingモデルも使える上、Claude Codeの1.5万円クラスとほぼ同等のコーディングエージェントを使えるなんて、限界社会人の私にとっては夢のような生活です。

### 限界社会人がコスパよくCodexを使うために

おそらくこれから先、OpenAIやAnthropicの各モデルのAPI利用料は上がっていくのではないかと思っています。

長らく「安すぎる」と言われており、ようやく適正価格に近づく動きが見え始めたと思っています。

とはいえ私は限界社会人。  
今までと同じ金額感で使い倒すための救済策を絞り出したので、同胞の皆さんのためにもご紹介できればと思います。

## Codex 節約活用術

ここまでは限界社会人の悲痛な叫び()をダラダラと書いてきましたが、ここからは実践的な **Codexを今まで通り安く使い倒す方法** をご紹介します。

### いったん３行で結論

これで理解して実現できる人はこれ以上読まなくてOKです。

* CodexにOpenRouterを繋いで使用する
* OpenRouterは `gpt-oss-120b:free` といった無料モデルを、条件を満たすと 1,000リクエスト/日 まで無料で行える（詳しくは [Limits](https://openrouter.ai/docs/api/reference/limits) 参照）
* サブエージェントで使用するモデルを一部OpenRouter経由にすることで、Codexのサブスク枠の消費をかなり抑えることができる

### そもそもOpenRouterとは

OpenRouter を雑に一言でいうと、複数のLLMをまとめて叩けるモデル用のルーター兼プロキシです。  
OpenAI 互換の API 形式で、OpenAI 系モデルだけでなく、Qwen や Gemma、NVIDIA 系なども同じようなインターフェースで呼び分けできます。

嬉しいポイントは、単にモデルが多いだけではありません。  
OpenRouter 側で利用量を一元管理できたり、プロバイダ障害時のフェイルオーバーが効いたり、モデルごとのコストやレイテンシを見ながら「どの用途に何を使うか」を切り分けやすい、という運用上のメリットがあります。“普段はCodexのサブスク枠を使い、重くない処理や補助的な処理だけ別ルートに逃がす” という節約運用と非常に相性が良いです。

### OpenRouterには無料モデルが存在する

OpenRouterには、以下のような無料で使えるモデルが存在します。  
モデル検索で `free` と入れると、

* OpenAI: gpt-oss-120b (free)
* Qwen: Qwen3 Coder 480B A35B (free)
* NVIDIA: Nemotron 3 Super (free)
* Google: Gemma 4 31B (free)

といった無料モデルが表示されるかと思います。

<https://openrouter.ai/models?q=free>

無料モデルの使用方法は、以下の記事がとてもわかり易かったのでご参照ください。  
<https://note.com/lumidina/n/n5e0781a954f0>

### CodexをOpenRouterに接続して使用できる

ここが本題です。  
Codex CLI は OpenRouter を model provider として接続できます。 OpenRouter 公式にも Codex CLI 向けのガイドがあり、 `~/.codex/config.toml` に model\_provider と model\_providers.openrouter を定義する形で使えます。Codex 側も config.toml でモデルや provider を切り替える前提になっているので、公式に沿った素直な構成です。

最小構成はこんな感じです。

~/.codex/config.toml

```
[profiles.openrouter]
model_provider = "openrouter"
model = "openai/gpt-oss-120b:free"

[model_providers.openrouter]
name = "openrouter"
base_url = "https://openrouter.ai/api/v1"
env_key = "OPENROUTER_API_KEY"
```

OpenRouterでAPI Keyを発行し、環境変数に登録したうえで

```
export OPENROUTER_API_KEY="sk-or-..."
```

プロファイルを指定してcodexを起動します。

```
codex --profile openrouter
```

起動時のダイアログにて以下のようにモデルが切り替わっていればOK。  
![Codex起動時のメッセージ](https://static.zenn.studio/user-upload/0e91c641002f-20260412.png)

これでOpenRouterの無料モデルを使ってCodexを動かせるようになりました。

※正式な/最新の情報は公式ページをご確認ください。  
<https://openrouter.ai/docs/guides/coding-agents/codex-cli>  
<https://developers.openai.com/codex/config-advanced#custom-model-providers>

### サブエージェントを無料のモデルに切り替える

さらなるチューニングのため、サブエージェントの構成をいじります。  
当然ですが gpt-oss-120b 等のfreeモデルは、 gpt-5.4 などの高性能モデルと比べて精度は低いです。

<https://artificialanalysis.ai/models/comparisons/gpt-5-4-vs-gpt-oss-120b>

そのため、

* メインのエージェントは `gpt-5.4`
* サブエージェントは `gpt-oss-120b:free`

という感じで使い分けてみましょう。  
Planning等のコアタスクは高性能モデル、調査や実装などの付随タスクは無料モデルを使用します。

.codex/agents/reviewer.toml （例）

```
name = "reviewer"
sandbox_mode = "read-only"
description = "Final reviewer focused on correctness, regressions, and test gaps."
developer_instructions = """
Review the final diff like an owner.
Prioritize correctness, regressions, security concerns, and missing test coverage.
Lead with concrete findings.
Avoid style-only comments unless they hide a real risk.
"""

model_provider = "openrouter"
model = "openai/gpt-oss-120b:free"
```

※サブエージェントの使い方は別記事をご参照ください。

### 動作を見る

まずは通常プロファイルで起動。モデルは `gpt-5.4 medium` になっているのでOK。

![](https://static.zenn.studio/user-upload/75e15994ea76-20260412.png)

しばらく見ていると、 `openai/gpt-oss-120b:free medium` でサブエージェントが起動しました。

これで、リーダーだけに給料を払いつつ末端の従業員は格安で利用するというウルトラブラック企業(?)の完成です。まるで限界社会人ですね。

## おわりに

ここまで色々と書きましたが、ヘビーに使うのであれば普通に$100プランへ課金するのも十分合理的だと思います。  
実際、世間で言われている通り今までが安すぎただけで、毎日のようにガッツリ使う人にとっては素直に上位プランへ行くのが一番わかりやすいです。

とはいえ、全員がそこまで使い倒すわけではありません。  
特に私のような「週末中心で使う」「普段はそこまで重くないが、使うときはしっかり使いたい」タイプにとっては、いきなり$100はやや重いのも事実です。

だからこそ、メインは高性能モデルを使いつつ、調査やコード探索、影響範囲の洗い出しといった補助タスクを低コストなモデルに逃がす、という使い分けにはかなり価値があると思っています。

どれくらいコスパ良く使えるかは使い方次第。  
同胞である限界社会人の皆さんは是非一度試してみてください。
