### CLI論争：TUIはAIの最終インターフェースか？

David Cramer（Sentry創業者）がCLIのTUIをこき下ろす投稿を発表したことを受けて、番組冒頭で議論になりました。「CLIには意義があるが、過去20年のすばらしいUI技術を無視して石器時代のものを作るな」という主張で、Open CodeやClaude Codeなどを暗に批判した内容です。

ShaneとObbyは中立的なポジションを取りつつも、「TUIはAIの長期的インターフェースではない」と認めています。Cursorのようなアプリがあることを考えれば、TUIだけにこだわる必要はない。ただ、NeovimユーザーなどのパワーユーザーにはTUIは残るだろうとも。Daxからは「エグゼクティブがハッカー気分になれるように作った」というユーモラスな反論がありました。

重要な示唆は「SupersetやConductorのように、UIはCLIの上に重なる層として発展していく」というもので、現在のCLI/TUIは「UIを設計せずに済む最速の実装形態」に過ぎないという見方です。

### 今週の"What the \_\_\_?"コーナー

Open ClaudeのClaw（爪）ロゴブームに続く、AIエージェントの「キャラクター化」トレンド。Codexにタマゴッチのようなペット機能が加わりました。「不気味なCLIに話しかけるよりペットに話しかける方がかわいい」というコンセプトで、エージェントとのインタラクションを親しみやすくする試みです。

#### GPT 5.5のプロンプトに「ゴブリン禁止」ルールの重複が発見

Codex用GPT 5.5のシステムプロンプトを解析した人が、「goblin（ゴブリン）、gremlin、raccoon、troll、ogre、pigeon などの生き物について、ユーザーのクエリと完全に関連しない限り話してはならない」という行が重複して記載されているのを発見。Sam Altmanが「ゴブリンモード」をジョークにしていたのと合わせて、モデルが動物・生き物を擬人化するクセがあることへの対処と推測されています。

Pika Labs（動画生成AI）が、ユーザーが作り出せるAIエージェント「Pika Agents」を発表。Seed Dance 2.0でキャラクターを生成し、音声・外見・パーソナリティを付与した上で、会話しながら動画を制作・改善できる。Open Claudeのsoul.mdでエージェントにミッションステートメントを持たせる概念と同じ流れで、AIエージェントに「魂」を持たせるトレンドが加速しています。

### Cursor SDK登場：ハーネスがフレームワークになる

Cursorが**Cursor SDK**を発表しました。CursorのIDEを動かしているランタイムハーネスとモデルを外部から利用できるようにするもので、CI/CDパイプラインからエージェントを実行したり、自分のプロダクトにエージェントを直接組み込めます。

Claude Code Agents SDKやOpenAI Agents SDKへの対抗策という側面も明らかで、「ハーネスに惚れ込んだ人がIDEの外でもそれを使えるようにする」という方向性は今後の大きなトレンドです。

ShaneはこれとOpenCode 2.0の動きから「ハーネスがフレームワークになり、フレームワークがハーネスを持つようになっている」と表現しました。Mastraとしては、CodexやCursorやClaude AgentsをMastra Studio内でサポートすることは「ほぼ確実に近い将来起こる」とも言っています。

### GPT 5.5 ―― 1週間での快走と使い分け論

ローンチから1週間のGPT 5.5の数字：

* API収益が過去のどのリリースよりも**2倍以上速く成長**
* Codex（コーディングエージェント）は**7日以内に収益が2倍**

番組の中で興味深かったのは「Opus 4.7とGPT 5.5の使い分け論」。Opus 4.7は「プランニングには最高だが実行中に目的を見失う（loses the plot）」ため計画段階に使い、GPT 5.5は実行にあてているというメンバーが多いとのこと。Claudeは「コーディングエージェントに向いているが、チャットっぽさがなく、サブコンシャスな存在（思考を助ける存在）にはなれない」という見方も印象的でした。

### OpenAI Responses APIとWebSockets

OpenAIがWebSocketsとResponses APIをリリース。エージェントチャットの「マルチプレイヤー化」への流れです。SSEストリームは複数クライアントが同じセッションに接続する用途には向いていませんが、WebSocketsは古くから実績がある仕組みで、複数クライアントが同一セッションに接続するマルチプレイヤーなチャット・エージェント体験が構築できるようになります。

### 「AIが機能しない理由」と「AIサイコシス」

Vasun（X）の記事「If AI is so great, why isn't it working?」への言及が非常に示唆的でした。

記事の主要論点：

* **検証ループの重要性**：コーディングは「テストが通ったかどうか」で客観的に検証できるが、財務・マーケティングなどの非技術的タスクには同等の検証ループが存在しない。エージェントを改善するには、結果を検証できる仕組みの構築が不可欠。
* **AIサイコシスの正体**：ソフトウェアプロジェクトにおけるモデルの優秀さを見て「AIはなんでもできる、AGIが来た」と思い込む現象。コーディング以外の領域では同じ水準の成果を期待できないにもかかわらず、過剰な期待をしてしまう。
* **内製エージェントのコスト問題**：週3000ドルのトークンコストをかけて内部チケット解決エージェントを動かしているが、ビジネス価値の証明に苦労しているチームの例。検証ループなしにROIを示すことは難しい。

ShaneとObbyも「自分たちもAIサイコシスがある」と認めた上で、「ソフトウェアエンジニアリングの文脈ではそうなって当然だが、そこから外に出ると話は変わる」という冷静な自己評価を示していました。

中国製オープンウェイトモデルが2026年の前半から急速にフロンティアモデルに肉薄しています。

**Alibaba Qwen 3.627B**：1500億パラメータ未満のオープンウェイトモデルで現在のリーダー。Artificial Analysis Intelligence Indexでスコア46。Qwen 3.5 Plusは「実質フロンティアモデル」とも評価されており、アウトプットトークン100万件あたり3ドルという低価格も強み。Qwen 3.6 35B対Sonnet 4.6の比較でもQwenが優勢とのこと。

「アメリカにとっては良くない」という発言が示すように、オープンウェイト分野での中国勢の躍進は米国AI産業に対する競争上の脅威として認識されています。Grok 4.3が中国のオープンソースモデルにまだ遅れているという事実もその文脈で語られていました（「だからこそのCursor買収案件」という皮肉も）。

**Mistral Medium 3.5**：欧州のAI企業Mistralが命令追従・推論・コーディングを統合した128Bモデルをパブリックプレビューで発表。256kコンテキスト、修正MITライセンスでのオープンウェイト提供。「Mistral社員以外の欧州人でMistralを使っている人を見たことがない」という辛口評価もありましたが、欧州のAI企業として競争に残り続ける姿勢は評価されていました。

### エージェントが「顧客」になるインフラが整う

今週の最も重要なトレンドのひとつが「エージェントが自律的に経済活動を行う」インフラの整備です。

**Stripe Link（エージェント向けウォレット）**：エージェントがあなたの代わりに支払いを行えるようにする仕組み。支払い認証情報は公開されず、すべての購入を人間が承認できる。先週のSF開催のStripe Sessionsでも最大の話題のひとつでした。

**Cloudflareの発表（4月29日）**：「エージェントがCloudflareの顧客になれる」。エージェントがCloudflareアカウントを作成し、有料サブスクリプションを開始し、ドメインを登録し、APIトークンを取得してコードをデプロイできます。Work OSのMichaelが「この現実があらゆるプロダクトで実現されることを望む」とコメントしているように、あらゆるSaaSがエージェントを顧客として受け入れる未来が見えてきました。

**Doola + Replit**：AIチャットを離れることなく米国LLCを設立できるようになりました。アプリを生成して良さそうなら、そのためのビジネスをReplitの中ですべて作れる。「バリアを下げるのはクールだが、LLC設立後の毎年の税務申告や解散の大変さは忘れずに」という冷静な注意書きも。

**GumLoop エージェントメールアドレス**：GumLoopのエージェントが独自のメールアドレスを持てるようになり、外部からメールしたりCCに入れたりできます。「メールの受信トレイがエージェントのタスクキューになる」という概念は面白い。Agent Mailというサービスも紹介されました。

### GitHub の深刻な脆弱性とダウンタイム問題

Wiz Researchがgithub.comに対してシングルgit pushでリモートコード実行（RCE）ができる脆弱性を発見。GitHubの数百万リポジトリへの不正アクセスが可能になる欠陥です。同時に最近のGitHubは4月23日・27日とその後さらにダウンタイムが続いており、「品質が下がっているのか、それとも前例のない利用量の増加によるものか」という議論になっていました。Copilotモデルプロバイダーだけが100%稼働率を維持しているというオチも。

### Rampのコーディングエージェント：マージPRの70%がAI生成に

Rampが内製コーディングエージェントの進捗を公開。かつては「全マージ済みPRの30%」だったAI生成PRの割合が、今では\*\*70%\*\*に達したとのこと。Slack、Linear、GitHubを通じたチャンネルを持ち、24時間365日の自動化タスクも実行。完全自律ではないが、エンジニアが操舵する形で相当部分の自動化が実現しています。「Devを社内で構築した感じ」とも評されており、AIコーディングエージェントの実用化が一企業でどこまで進んでいるかを示すよい事例です。

### TypeScript TS7（Goによる書き直し）が驚異の速度を実現

TypeScript 7（TS7）はGoによる完全な書き直しで、型チェック速度が劇的に向上しています。ShaneはすでにアルファまたはベータをMastraで使用中で「型チェックが信じられないほど速い」とのこと。VS Codeマーケットプレイスとnpmでインストール可能。ただし言語サーバーのアップデートも必要で、まだ一部未対応の部分もあるとのこと。「Goで書かれたのは少し残念だが、速さには関係ない」というコメントが正直で笑えました。

Node 20がEOLを迎えた同日に、Zodが次のマイナーバージョンでCommonJSをドロップする予定というアナウンスがあり、TypeScriptエコシステムの近代化が加速しています。
