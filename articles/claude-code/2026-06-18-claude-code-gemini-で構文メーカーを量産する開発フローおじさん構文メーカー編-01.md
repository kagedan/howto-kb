## はじめに

入力文を絵文字たっぷりの“おじさん構文”に変換する、おじさん構文メーカーを作りました。

<https://hone-gumi.com/ojisan-koubun-maker/>

ただ、この記事で共有したいのは「おじさん構文メーカー単体の作り方」ではありません。**小泉構文メーカー → タゴサク構文メーカー → おじさん構文メーカー と、同じ型のAIツールを Claude Code × Gemini でどう“量産”しているか**という開発フローです。

### 自己紹介

ホネグミ代表、応用情報技術者の資格を持つエンジニア×マーケターです。会社役員を4年、独立して4年目。フリーランスとして、ネタ系のAIツールを量産しながら、思想駆動型サービス開発を続けています。「個人がAIで小さなツールを量産する」立場からの記録として読んでもらえればと思います。

## 量産を支える「テンプレ化された構成」

3つの構文メーカーは、すべて同じ構成を共有しています。

```
WordPress固定ページ（カスタムHTML：00_code.html）
  ↓ fetch()
/wp-admin/admin-ajax.php
  ↓
function.php（共通のAJAXハンドラー群）
  ↓
Gemini API（ツールごとの専用 or 共用キー）
```

新しい構文メーカーを足すときに変えるのは、基本的に次の3つだけです。

| 変える部分 | 変えない部分 |
| --- | --- |
| プロンプト（構文の型） | admin-ajax.php 経由のAPIプロキシ構成 |
| UI（色・絵文字・例文） | カウンター（DiagnosisCounterSystem） |
| APIキー戦略 | IPベースの日次制限（16:00リセット） |

「毎回ゼロから作らない」ことが、量産の最大のコツです。

## Claude Code に渡す“差分”の作り方

実際の制作では、既存の `00_code.html` と `function.php` を正として、Claude Code に「タゴサクと同じ構成で、おじさん構文版を作って」と差分ベースで依頼します。

これにより、Claude Codeは既存の勝ちパターン（命名規則・AJAX action名・カウンター登録）を踏襲したまま、プロンプトとUIだけを差し替えたコードを生成できます。人間がやるのは、構文の型の定義と最終確認です。

## ツールごとの「APIキー戦略」を設計に組み込む

量産時に地味に重要なのが、APIキーの分け方です。Gemini APIの無料枠はGCPプロジェクト単位なので、ツールが増えるほど枠の食い合いが起きます。

そこで、アクセス量の見込みでキー戦略を変えています。

* 利用が少ないツール（例：タゴサク）→ 既存キーを共用
* 伸びる見込みのツール（例：おじさん）→ **専用GCPプロジェクト＋専用キー `GEMINI_API_KEY_OJISAN`**

コードは「専用キー優先・なければ共通キーにフォールバック」で書いておくと、どちらの戦略でも同じ実装で回せます。

function.php

```
$api_key = defined('GEMINI_API_KEY_OJISAN') && GEMINI_API_KEY_OJISAN
    ? GEMINI_API_KEY_OJISAN
    : (defined('GEMINI_API_KEY') ? GEMINI_API_KEY : '');
```

「量産する」と決めた瞬間から、キー設計はアーキテクチャの一部になります。

## プロンプトは「意味を保って見た目だけ変える」

おじさん構文メーカーは、返信を生成するのではなく、\*\*入力文の意味を保ったまま見た目をおじさん構文化する“変換型”\*\*にしました。

```
元の文の意味は保ったまま、次の特徴を加える：
- 絵文字・顔文字を複数
- 語尾や一部をカタカナに（〜ネ、〜ヨ）
- 気遣いの一言や質問を追加
- 最後に軽い自分語り
出力は本文のみ。
```

温度は0.9で毎回違う出力にし、出力トークンはLINE一通分に絞ります。構文の「型」を言語化できれば、別の構文メーカーにも同じ枠組みで横展開できます。

## 公開後のSEO/LLMO定型ルーティン

量産で見落としがちなのが、公開後の導線整備です。ホネグミでは新ツールごとに「3点セット」を必ず回しています。

1. 構文メーカー総合ガイド（ハブページ）に1ブロック追加して内部リンク
2. 新ツールページにFAQ構造化データ（FAQPage JSON-LD）を追記
3. `llms.txt` に1行追加して、LLMからの参照経路を確保

ツールを作るたびにこの3点を回すことで、各構文メーカーが相互にリンクし合うトピッククラスターが育ちます。

## まとめ

おじさん構文メーカーは、単体としては小さなネタツールです。ですが、Claude Code × Gemini を使うと、こうしたツールは「テンプレ化された構成＋差分プロンプト＋キー戦略＋公開ルーティン」のセットで量産できます。

個人開発でAIツールを継続的に増やしたいなら、1個を丁寧に作るより、**2個目以降を速く作れる“型”を先に整える**方が効きます。
