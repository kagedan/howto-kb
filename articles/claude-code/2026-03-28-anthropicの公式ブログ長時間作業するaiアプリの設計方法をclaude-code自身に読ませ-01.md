## ここがかわったよ！ 〜Claude Code 改善まとめ〜

> 2026年3月28日に行った設定の改善を、わかりやすく説明します。

---

## そもそも何をしたの？

Anthropic（Claudeを作っている会社）が「AIをもっと賢く使う方法」について書いた記事を参考にして、  
Claude Code の設定を全部見直しました。

たとえるなら、**新しいアルバイトスタッフに、会社のマニュアルを渡した**イメージです。  
今まではマニュアルなしで働かせていたのを、ちゃんとした手順書を作って渡した、という感じです。

---

## 変更① 「作業マニュアル」を作った

### 何を作ったか

`CLAUDE.md` というファイルを新しく作りました。

### どんな内容？

Claude が毎回の作業の最初に読む「指示書」です。こんなことが書いてあります：

```
✅ 日本語で答えること
✅ 作業を始める前に「何をもって完成とするか」を決めること
✅ コードを書いたあとは必ず自分でチェックすること
✅ 「うまくできました！」と安易に言わないこと
```

### なぜ改善されるの？

**Before（改善前）**

> Claude:「はい、完成しました！完璧です！」  
> 実際：バグがある、要件を満たしていない...

**After（改善後）**

> Claude:「完成しました。自己チェックの結果、〇〇の部分に問題があります。こう修正しました。」

実は AI は自分が作ったものを自分で評価すると、**甘い点数をつけがち**です。  
「自分の作文を自分で採点したら甘くなる」のと同じですね。  
それを防ぐために、具体的なチェック項目を4つ設けました：

| チェック項目 | 内容 |  
|---|---|  
| 🔧 機能性 | ちゃんと動くか？ |  
| ✨ 品質 | 読みやすいコードか？ |  
| 🛡️ 堅牢性 | 予期しない入力でも壊れないか？ |  
| 🎯 忠実性 | 頼んだことと違うことをしていないか？ |

---

## 変更② 「許可証」をまとめた

### 何をしたか

`settings.json` という設定ファイルを整理しました。

### どんな内容？

Claude が作業中に使えるコマンド（道具）を事前にまとめて許可しました。

### なぜ改善されるの？

**Before（改善前）**

```
Claude：「Python を実行してもいいですか？」
あなた：「はい」
Claude：「このフォルダを作ってもいいですか？」
あなた：「はい」
Claude：「このサイトにアクセスしてもいいですか？」
あなた：「はい」
...（毎回聞かれる）
```

**After（改善後）**

```
Claude：（許可済みの操作はそのまま実行）
あなた：（作業に集中できる）
```

よく使う操作（Python実行、Webスクレイピング、フォルダ作成など）を  
あらかじめ「これはOKですよ」と登録しておいたので、毎回確認が来なくなります。

---

## 変更③ ゴミ箱の整理をした

### 何をしたか

`settings.local.json` の中にたまっていた古い「一時許可」を削除しました。

### どんな内容？

過去のセッションで「この1回だけ許可して」と積み重なった設定が **42個** もありました。  
その中には、古いバージョン（2.1.78）専用の解析コマンドなど、もう使わないものがたくさん。  
それを **12個** に絞り込みました。

### なぜ改善されるの？

散らかった部屋より、整理された部屋の方が何がどこにあるかわかりやすいですよね。  
設定ファイルも同じで、スッキリさせることで管理しやすくなります。

---

## 変更④ 「完了通知」の仕組みを作った

### 何をしたか

`hooks`（フック）という機能を使って、作業が終わったときに**ビープ音**が鳴るようにしました。

### なぜ改善されるの？

Claude に長い作業（たとえば30分かかるコード生成など）を頼んだとき、  
終わったかどうかをずっと画面を見て確認する必要がなくなります。  
「ピッ」と音が鳴ったら完了、という感じです。

---

## 変更⑤ 「自己紹介ファイル」を作った

### 何をしたか

メモリファイルに、あなたのことを記録しました。

### どんな内容？

```
・日本語で話す人
・Python と HTML をよく使う
・Windows 11 環境
・スクレイピングや業務アプリをよく作る
・出力ファイルは output/ フォルダに入れるスタイル
```

### なぜ改善されるの？

今まで Claude は毎回「はじめまして、あなたはどんな人ですか？」という状態でした。  
このメモリがあることで、新しいセッションを始めても**あなたのことを覚えた状態**でスタートできます。

---

## まとめ：何がどう変わったか

```
改善前
─────────────────────────────────────────────
❌ マニュアルなし → 毎回ゼロスタート
❌ 自己評価が甘い → 「完璧です！」が口癖
❌ 毎回「許可しますか？」 → 作業がブツブツ中断
❌ 設定が42個の雑多なゴミ → 管理しにくい
❌ 完了してもわからない → ずっと画面を見てる
❌ 毎回あなたのことを忘れる → 毎回説明が必要

改善後
─────────────────────────────────────────────
✅ CLAUDE.md で毎回同じ基準で作業
✅ 4項目チェックで正直な自己評価
✅ よく使う操作は事前許可済み → スムーズ
✅ 設定がスッキリ12個 → 見通しがいい
✅ 完了したらビープ音で通知
✅ メモリがあなたのことを覚えている
```

Anthropicの公式ブログ「[長時間作業するAIアプリの設計方法](https://www.anthropic.com/engineering/harness-design-long-running-apps)」(2026-03-24)
