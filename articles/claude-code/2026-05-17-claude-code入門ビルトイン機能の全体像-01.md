# はじめに

**Claude Code**は、Anthropic社が提供するAI支援開発ツールです。CLI、デスクトップアプリ、Web版（claude.ai/code）、IDE拡張機能として利用でき、コーディング、デバッグ、リファクタリングなど、開発の様々な場面でAIがサポートしてくれます。

この記事では、Claude Codeの**ビルトイン機能の全体像**を入門者向けに解説します。

**対象読者**:

* Claude Codeを使い始めたばかり
* どんな機能があるか知りたい
* 基本的な使い方を理解したい

**この記事で分かること**:

* Claude Codeの基本的な使い方
* 主要なビルトイン機能（Read、Edit、CLAUDE.md、Agent、Plan等）
* 初期セットアップで設定すべき機能
* 次に学ぶべきこと

# 1. Claude Codeとは

## 1.1 Claude Codeの特徴

* **AIアシスタント**: Claude（Sonnet 4.5等）がコードを読み、編集し、実行を支援
* **マルチプラットフォーム**: CLI、デスクトップ、Web、IDE拡張
* **ファイル操作**: ローカルファイルの読み書き、検索、編集
* **コマンド実行**: シェルコマンド、スクリプトの実行
* **Git統合**: ブランチ作成、コミット、PR作成

## 1.2 基本的な使い方

### CLI版の起動

### 対話例

```
あなた: 「このプロジェクトの構造を教えて」
Claude: プロジェクトを探索し、ファイル構成を説明

あなた: 「READMEにインストール手順を追加して」
Claude: README.mdを読み込み、内容を編集

あなた: 「テストを実行して」
Claude: pytest等を実行し、結果を報告
```

# 2. 基本的なツール

Claude Codeは、以下のツールを使ってタスクを実行します。

## 2.1 Read：ファイル読み込み

```
あなた: 「src/main.pyの内容を確認して」
Claude: Read(src/main.py) → ファイル内容を表示
```

**特徴**:

* 大きなファイルは範囲指定で読み込み（offset, limit）
* PDFファイルも読み込み可能（ページ指定）
* Jupyter Notebookも対応

## 2.2 Edit：ファイル編集

```
あなた: 「この関数にDocstringを追加して」
Claude: Edit(src/main.py, old_string, new_string)
```

**特徴**:

* 差分のみを送信（効率的）
* 編集前に必ずReadでファイルを確認
* 複数箇所を同時に編集可能

## 2.3 Write：新規ファイル作成

```
あなた: 「tests/test_main.pyを作成して」
Claude: Write(tests/test_main.py, content)
```

**注意**:

* 既存ファイルの編集にはEditを使用
* Writeは上書きするため、既存ファイルには使わない

## 2.4 Bash：コマンド実行

```
あなた: 「テストを実行して」
Claude: Bash(pytest tests/ -v)
```

**特徴**:

* シェルコマンドを実行
* git、npm、pytest等のツールを利用可能
* `run_in_background`でバックグラウンド実行も可能

# 3. CLAUDE.md：最初に設定すべきプロジェクトコンテキスト

## 3.1 CLAUDE.mdとは

プロジェクトルートに配置するMarkdownファイルで、Claude Codeが会話開始時に自動的に読み込みます。

**役割**:

* プロジェクトの全体像をAIに伝える
* コーディング規約・開発フローを定義
* よく使うコマンドを記録

## 3.2 基本的な書き方

```
# プロジェクト名

## 技術スタック
- Python 3.11
- FastAPI
- PostgreSQL

## コーディング規約
- Linter: ruff
- Formatter: black
- 型ヒント必須

## よく使うコマンド
- テスト: `pytest tests/`
- サーバー起動: `uvicorn src.main:app --reload`
```

## 3.3 なぜ最初に設定すべきか

CLAUDE.mdがあると、Claude Codeは：

* プロジェクトの技術スタックを理解
* コーディング規約に従った提案
* よく使うコマンドを把握

**推奨**: Claude Codeを使い始めたら、まずCLAUDE.mdを作成しましょう。

**詳細な書き方**: 実践的な書き方やコンテキスト劣化問題については、[ハーネス機能詳解](./e82bf4b89552ae.md#1-claudemd%E3%83%97%E3%83%AD%E3%82%B8%E3%82%A7%E3%82%AF%E3%83%88%E3%82%B3%E3%83%B3%E3%83%86%E3%82%AD%E3%82%B9%E3%83%88%E3%81%AE%E4%B8%AD%E6%A0%B8)で解説しています。

# 4. settings.json：権限管理と環境設定

## 4.1 settings.jsonとは

`.claude/settings.json` でプロジェクト固有の設定を定義します。

**主な用途**:

* よく使うコマンドの自動許可
* 環境変数の設定
* HOOK（自動実行コマンド）の定義

## 4.2 権限管理の設定

頻繁に使うコマンドを自動許可することで、毎回の承認を省略できます：

```
{
  "permissions": {
    "allow": [
      "Bash(pytest tests/*)",
      "Bash(npm run *)",
      "Read(src/**/*.py)"
    ]
  }
}
```

## 4.3 環境変数の設定

プロジェクトモードや設定を定義：

```
{
  "env": {
    "PROJECT_MODE": "development",
    "TEST_COVERAGE_TARGET": "80"
  }
}
```

## 4.4 初期セットアップ時の推奨設定

```
{
  "permissions": {
    "allow": [
      "Bash(pytest *)",
      "Bash(npm *)",
      "Read(src/**/*)",
      "Read(tests/**/*)"
    ]
  }
}
```

**詳細**: [ハーネス機能詳解](./e82bf4b89552ae.md#5-settingsjson%E3%83%97%E3%83%AD%E3%82%B8%E3%82%A7%E3%82%AF%E3%83%88%E5%9B%BA%E6%9C%89%E8%A8%AD%E5%AE%9A)で詳しく解説しています。

# 5. Agent：サブエージェントによる並行作業

## 3.1 Agentとは

複雑なタスクや探索的な作業を、**サブエージェント**に委任できます。

**メリット**:

* メインの会話コンテキストを節約
* 並行して複数の調査を実行
* 専門的なタスクに特化したエージェント

## 3.2 Agentの種類

### Explore：コードベース探索

```
あなた: 「認証関連のコードを全て調査して」
Claude: Exploreエージェントを起動
→ 複数ファイルを並行探索し、結果を報告
```

**用途**:

* 大規模なコードベースの探索
* 特定の機能がどこにあるか調査
* 依存関係の把握

### Plan：実装計画立案

```
あなた: 「新機能の実装プランを作成して」
Claude: Planエージェントを起動
→ アーキテクチャを検討し、詳細なプランを提示
```

**用途**:

* 大規模な変更の事前計画
* 複数のアプローチを比較
* リスクの洗い出し

### general-purpose：汎用タスク

```
あなた: 「複数のAPIドキュメントを調査して」
Claude: 汎用エージェントを起動
```

## 5.3 Agentの使いどころ

* **使う**: 大規模な探索、独立した調査タスク
* **使わない**: 簡単なファイル読み込み、単一ファイルの編集

# 6. Plan mode：実装前の計画立案

## 6.1 Plan modeとは

大きな変更を実装する前に、**詳細な計画を立ててユーザーの承認を得る**モードです。

## 6.2 Plan modeの流れ

```
1. 探索フェーズ
   Exploreエージェントでコードベースを調査

2. 設計フェーズ
   Planエージェントでアプローチを検討

3. プラン作成
   .claude/plans/ にプランファイルを保存

4. ユーザー承認
   プランを確認し、承認または修正

5. 実装
   承認されたプランに従って実装
```

## 6.3 Plan modeが役立つ場面

* **複数ファイルにまたがる変更**: 影響範囲が広い
* **アーキテクチャ変更**: 設計判断が必要
* **不明点が多い**: まず調査・検討が必要

## 6.4 Plan modeの例

```
あなた: 「認証システムをJWTからOAuthに移行したい」

Claude: Plan modeに入ります
→ Phase 1: 認証関連のコードを全て探索（Exploreエージェント）
→ Phase 2: 移行アプローチを検討（Planエージェント）
→ Phase 3: 詳細プランを作成（ファイルリスト、手順、リスク）
→ Phase 4: あなたの承認を待つ

あなた: 承認

Claude: プランに従って実装開始
```

# 7. Task管理：進捗の可視化

## 7.1 Task機能とは

複数ステップのタスクを分解し、進捗を追跡する機能です。

## 7.2 基本的な使い方

```
あなた: 「新機能を実装して」

Claude: タスクを作成します
  Task 1: 設計 [not_started]
  Task 2: 実装 [not_started]
  Task 3: テスト [not_started]

→ 作業を進めると自動的にステータスが更新
  Task 1: 設計 [completed] ✓
  Task 2: 実装 [in_progress] ⏳
  Task 3: テスト [not_started]
```

## 7.3 Taskの活用シーン

* 大きなタスクを小さく分解
* 進捗を一目で把握
* セッションをまたいでも状態を保持

# 8. Memory：会話から学習する永続的な記憶

## 8.1 Memoryとは

Claude Codeは会話から重要な情報を自動的に抽出し、`.claude/projects/[project]/memory/` に永続的に保存します。

**CLAUDE.mdとの違い**:

* **CLAUDE.md**: 手動で書くプロジェクト情報
* **Memory**: 会話から自動的に学習

## 8.2 Memoryの種類

* **user**: ユーザーの役割・好み・スキルレベル
* **feedback**: 過去のフィードバック・失敗事例
* **project**: プロジェクトの進行状況・決定事項
* **reference**: 外部リソースへのポインタ

## 8.3 基本的な使い方

### 明示的な記録

```
あなた: 「プロジェクトではPytestを使うことを覚えておいて」
Claude: Memoryに保存しました
```

### 自動学習

会話中に重要な情報を自動検出して保存します。

## 8.4 注意点

Memoryは時間とともに古くなります。定期的に整理が必要です（月1回推奨）。

**詳細**: [ハーネス機能詳解](./e82bf4b89552ae.md#2-memory%E6%B0%B8%E7%B6%9A%E7%9A%84%E3%81%AA%E8%A8%98%E6%86%B6%E3%81%AE%E7%AE%A1%E7%90%86)で詳しく解説しています。

# 9. SKILLS：定型タスクをコマンド化

## 9.1 SKILLSとは

よく使う定型タスクを `/skill-name` コマンドとして登録できます。

**配置場所**: `.claude/skills/[skill-name].md`

## 9.2 基本的な作成方法

```
<!-- .claude/skills/review.md -->
---
name: review
description: コードレビューを実施
---

# Instructions

1. git diffで変更確認
2. コーディング規約チェック
3. セキュリティレビュー
4. パフォーマンス確認
```

呼び出し方：

## 9.3 よく使うSKILLS例

* `/review`: コードレビュー
* `/test`: テスト実行・結果分析
* `/deploy`: デプロイ手順の実行

## 9.4 いつ使うべきか

* 繰り返し実行するタスク
* チェックリストが明確なタスク
* チームで手順を統一したいタスク

**詳細**: [ハーネス機能詳解](./e82bf4b89552ae.md#3-skills%E5%AE%9A%E5%9E%8B%E3%82%BF%E3%82%B9%E3%82%AF%E3%81%AE%E3%82%B3%E3%83%9E%E3%83%B3%E3%83%89%E5%8C%96)で実践例を紹介しています。

# 10. HOOK：イベント駆動の自動実行

## 10.1 HOOKとは

特定のイベント時に自動実行されるコマンドを `settings.json` で定義できます。

**主なイベント**:

* `session-start-hook`: セッション開始時
* `user-prompt-submit-hook`: プロンプト送信時
* `tool-result-hook`: ツール実行後

## 10.2 基本的な設定例

```
{
  "hooks": {
    "session-start-hook": "echo '開発モード: テストカバレッジ80%以上'",
    "tool-result-hook": "[ \"$TOOL_NAME\" = \"Edit\" ] && echo '✅ 編集完了'"
  }
}
```

## 10.3 活用例

* **注意喚起**: プロジェクト固有の制約を表示
* **自動チェック**: 編集後にLint実行
* **統計記録**: セッション情報をログに記録

## 10.4 初心者への推奨

HOOKは高度な機能です。まずはCLAUDE.md、settings.json、SKILLSを使いこなしてから導入しましょう。

**詳細**: [ハーネス機能詳解](./e82bf4b89552ae.md#4-hook%E3%82%A4%E3%83%99%E3%83%B3%E3%83%88%E9%A7%86%E5%8B%95%E3%81%AE%E8%87%AA%E5%8B%95%E5%AE%9F%E8%A1%8C)で実践パターンを解説しています。

# 11. MCP（Model Context Protocol）：外部サービス連携

## 11.1 MCPとは

Claude Codeが外部サービスと連携するためのプロトコルです。

**例**:

* **GitHub MCP**: GitHubリポジトリへのアクセス
* **Filesystem MCP**: 特定ディレクトリへのアクセス
* **Slack MCP**: Slackへのメッセージ送信

## 11.2 MCP設定例

```
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "${GITHUB_TOKEN}"
      }
    }
  }
}
```

## 11.3 MCPの活用例

```
あなた: 「GitHubのissue #123を確認して」
Claude: GitHub MCP経由でissueを取得
→ issueの内容を表示

あなた: 「このissueをクローズして」
Claude: GitHub MCP経由でissueをクローズ
```

# 13. Claude Codeの使いこなしステップ

## Step 1: 初期セットアップ（1日目）

# 14. よくある質問

## Q1. Claude Codeは無料で使える？

A. Claude Codeは無料で使えますが、以下のプランがあります：

* **Free**: 制限あり
* **Pro**: 月額$20（トークン制限緩和）
* **Team**: チーム向けプラン

## Q2. どのファイルまでアクセスできる？

A. プロジェクトディレクトリ内のファイルにアクセスできます。設定で許可したコマンドのみ実行可能です。

## Q3. プライバシーは大丈夫？

A. ローカルで実行され、明示的に送信しない限り、コードはAnthropicのサーバーに送られません。設定で制御できます。

## Q4. 他のAIツールとの違いは？

* **GitHub Copilot**: コード補完に特化
* **Cursor**: エディタ統合
* **Claude Code**: プロジェクト全体の理解・ファイル操作・コマンド実行に強い

## Q5. チームで使える？

A. はい。CLAUDE.md、SKILLS、settings.jsonをGitで共有することで、チーム全体で統一された開発体験を実現できます。
