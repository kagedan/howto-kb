Claude Code を触っていると、こんな話が出てくる。

「MCP を有効化すれば、AI が直接メールを下書きしたり、Slack に投稿したり、Google カレンダーを覗いたりできる」

教科書の Ch7 では、MCP そのものの仕組みを説明した。「AI が外部サービスに繋がる道具」だ。でも、\*\*「実際にどのサービスが繋がるの？繋がるとどんなことができるの？」\*\*という部分は、ざっくり流してしまった。

ここで、そのカタログをまとめる。MCP で繋がる定番サービスを一覧にして、それぞれ「繋ぐと何ができるか」を具体例で見ていく。これさえ知っていれば、自分の業務に「AI が何を手伝うか」が一気に見える。

## MCP の動作原理—— AI に「道具袋」を持たせる

MCP はシンプルだ。AI が何かやりたいとき、**パソコンやクラウドサービスに「あの情報を持ってきて」「これを実行して」と指示する仕組み**。

スマホとアプリの関係をイメージしてほしい。カレンダーアプリが連絡先にアクセスしたり、地図アプリが現在地を使ったり、メモアプリがクラウドに自動保存したりする。スマホは、それぞれのアプリに「この情報、使っていい」という権限を与えている。

AI と外部サービスも同じ。AI が Gmail に繋がるには、あらかじめ許可を出しておいた Gmail が、AI に情報を渡す。これが「繋がっている」という状態だ。

## メール・チャット—— 毎日のやり取りを AI で効率化

もっとも使われているのが**メール と チャット**の連携だ。

Gmail に繋げば、AI は「メール一覧を取得」「下書きを作成」「ラベルを付ける」ができる。営業リストのメール企画を AI に考えさせて、下書きを見て「いいね」と承認してから送る—— こういった流れが可能になる。Slack に繋げば「チャンネルを読んで要約」「特定の投稿を探して」「指定時刻に投稿」といった指示が出る。

毎朝の日報を AI に書かせたり、営業日誌を Slack ボットが自動投稿したり、月次レポートをメール配信する——これらはすべて MCP で繋がったからできることだ。

## スケジュール・ファイル—— 情報の「入り口」と「保管庫」

次が**Google カレンダーと、Google ドライブや Notion といったクラウドストレージ**だ。

AI が Google カレンダーに繋がると「今週の予定」を読める。それをもとに「金曜日が空いているから、営業活動に充てよう」といった提案ができる。逆に、AI が会議の日程を提案して、人間が「OK」と言ったら Google カレンダーに自動登録—— これも可能だ。

Google ドライブや Notion に繋げば、AI は「営業資料を探して」「データシートに書き込んで」「共有フォルダに保存」といった指示に応える。つまり、AI が単に提案するだけでなく、**情報を組織内に実際に保管**できるわけだ。

## 会計・業務システム—— 事業の裏側を AI に任せる

もう少し専門的だが、MCP は\*\*freee（クラウド会計）や GitHub（コード管理）、Figma（デザインツール）\*\*にも繋がる。

freee に繋げば、AI は「先月の売上はいくらか」「経費をカテゴリ分けして」「請求書を作成」といった指示に応える。単なる計算ではなく、**実際に会計データに手を加える**ことができるわけだ。GitHub なら、AI がコードのエラーを見つけたときに「修正提案を出して、その場で PR を作成」といったことも可能。Figma なら、デザイン案を AI に作らせてファイルに保存——こんな流れが生まれる。

## よく聞く用語ミニ辞典

**MCP** —— Model Context Protocol。AI が外部サービスに繋がるための標準仕様。Anthropic（Claude の開発元）が定めた。

**認証** —— AI がサービスにアクセスするときの「身分確認」。パスワードや API キーを使う。

**スコープ**（権限）—— 「何を AI に許すか」を制限する機能。「メール閲覧は OK、但し送信は禁止」といった指定ができる。

**下書き** —— AI が実際の操作を実行する前に、「こんなメールを送ろう」と人間に提示して確認を取る仕組み。

## 繋がるサービスは増え続けている—— 今後の広がり

この記事を書いている時点では、Gmail や Slack、Google カレンダーがメジャーだ。でも、MCP はまだ発展途上。新しいサービスが次々と対応を始めている。

Stripe（決済）、Zapier（自動化プラットフォーム）、さらには**社内システムを MCP 対応にするコンサル**も出始めている。つまり、今後は「自分たちの業務システムに AI を直結する」という組織が増えていくということだ。

## まとめ

| サービス | 繋ぐとできること | よく使う人 |
| --- | --- | --- |
| Gmail | メール下書き、メール一覧取得、ラベル管理 | 営業、事務 |
| Slack | チャンネル要約、投稿検索、自動投稿 | マネージャー、チームリード |
| Google カレンダー | 予定確認、スケジュール提案、自動登録 | マネージャー、営業 |
| Google ドライブ・Notion | 資料検索、データシート編集、ファイル保存 | 企画、営業事務 |
| freee | 売上確認、経費分類、請求書作成 | 経理、経営者 |
| GitHub | コード検索、バグ修正提案、PR 作成 | エンジニア |
| Figma | デザイン案作成、ファイル保存、公開 | デザイナー、PM |

繋がるサービスが増えるほど、AI はあなたの仕事の「相棒」になっていく。大事なのは、何でもかんでも AI に任せるのではなく、**読み取りと提案は AI に、実行は人間が確認する**というバランスだ。これさえ守れば、MCP は強力な武器になる。
