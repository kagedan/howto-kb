## Claude Codeのサブエージェント、思ったより使いどころが限られた

最近Claude Code（AnthropicのCLI版Claude）をメインで使っているんだけど、サブエージェント機能をちゃんと試したのは先週が初めてだった。

公式ドキュメントを読むと、Explore・Plan・general-purposeの3種類があって、並列で最大3つまで起動できるらしい。「メインのコンテキストを消費せずに調査を委任できます」みたいなことが書いてあって、これは良さそうだなと。自分のプロジェクト（ファイル数それなりにあるTypeScript製のCLIツール）で実際に回してみた記録を残しておく。

## Exploreエージェントが一番わかりやすく便利だった

やったこと自体はシンプルで、「このプロジェクトの認証周りの実装を調べて、どのファイルがどう絡んでるか整理して」という指示をExploreエージェントに投げた。

体感としては、メインのセッションでGrepやReadを何度も叩いて調査するのとやってること自体は同じ。ただ、調査の過程でコンテキストウィンドウがどんどん埋まっていく問題がサブエージェントだと起きない。これが地味にでかい。

メインセッションで大きめのファイルを5〜6個読み込むと、それだけで結構なトークンを消費する。その後に「じゃあここをリファクタして」と頼むと、コンテキストの後ろのほうが押し出されて、さっき読んだファイルの内容を忘れてたりする。サブエージェントに調査をやらせると、返ってくるのは要約だけなので、メインのコンテキストはきれいなまま。

```
# メインセッションでの指示イメージ
> @explore このプロジェクトの src/auth/ 配下を調査して、
>   認証フローの全体像と各ファイルの責務をまとめて

# → Exploreエージェントがバックグラウンドで
#    Grep, Read, Glob を駆使して調査
# → 結果だけがメインに返ってくる
```

返ってきたまとめは8割くらい正確だった。残り2割は「このファイルはミドルウェアとして使われている」みたいな推測が混じっていて、実際にはもう使われてない古いコードだったりした。まあ人間がやっても初見のコードベースで同じミスはする。

## Planエージェントは正直まだ微妙

Planエージェントにも「認証をJWTからセッションベースに変更する計画を立てて」と投げてみた。

返ってきた計画は……教科書的だった。「1. セッションストアを選定する 2. ミドルウェアを差し替える 3. テストを更新する」みたいな。いや、それはそう。知ってる。

自分が欲しかったのは「このプロジェクト固有の事情を踏まえた計画」で、たとえば「`src/middleware/auth.ts`の`verifyToken`関数は3箇所から呼ばれてるから、まずそこのインターフェースを変えないといけない」みたいな具体性。Planエージェントは一応コードも読んでくれるんだけど、Exploreほど深く潜らない印象がある。

結局Exploreで調査 → その結果を自分で読んで → メインセッションに直接「こういう方針でリファクタして」と指示する、というのが一番うまく回った。Planエージェントの出番はなかった。

## 並列実行は便利だけど待ち時間がある

3つまで並列で走らせられるので、「認証周り調べて」「DB接続周り調べて」「テストのカバレッジ状況見て」を同時に投げてみた。

全部返ってくるまでに40秒くらい。1個ずつ順番にやるよりは明らかに早い。ただ、3つ同時に走らせるとレスポンスが若干遅くなる気がした（計測はしてないので体感の話）。

あと、サブエージェント同士は情報を共有しない。認証の調査結果を踏まえてDB周りも調べてほしい、みたいな依存関係がある場合は順番に投げるしかない。

## 「サブエージェントに投げる / 自分でやる」の境界

1週間くらい使ってみて、なんとなく見えてきた使い分け。

**サブエージェントに投げたほうがいいやつ:** 知らないコードベースの全体像把握。ファイルが多くて、Grepの結果を何回も追いかけないといけないような調査作業。これはExploreエージェントの独壇場。メインのコンテキストを汚さないメリットがでかい。

**自分でやったほうが早いやつ:** 「この関数の引数を変えて」くらいの、もうどこを触るかわかってる作業。サブエージェントに委任するオーバーヘッド（指示を書く → 待つ → 結果を読む）のほうが、直接やるより時間かかる。

あと、コンテキストウィンドウにまだ余裕があるときは、わざわざサブエージェントを使う理由が薄い。セッションの後半、すでにいろいろ読み込んだ後で「もう1個調べたいことがあるんだけどコンテキストがきつい」ってなったときに真価を発揮する感じ。

## 雑感

正直、サブエージェントを常時使いこなすというよりは、「あ、今コンテキスト節約したいな」ってときにExploreを呼ぶ、くらいの使い方に落ち着いた。Planは今のところ出番なし。general-purposeは汎用すぎて逆に指示が難しい。

マルチエージェントって言葉の響きはかっこいいんだけど、実態は「調査を別スレッドに逃がせる」くらいの理解でいいと思う。それだけでも十分便利なんだけど、なんかもっとすごいことができそうな期待をしてると肩透かしを食らう。

Claude Code自体はかなり気に入ってるので、サブエージェントの精度が上がってくれたらまた試す。特にPlanエージェントがプロジェクト固有の文脈をもっと拾えるようになったら化けそうな気配はある。
