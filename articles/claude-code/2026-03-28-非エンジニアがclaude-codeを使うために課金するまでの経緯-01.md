今回は、非エンジニアの私が、Claude Codeを使い始めた経緯をお話しします。

一番大きなきっかけは、「Everything-Claude-Code」のディレクトリ構成を見て**既視感**を覚えた、という体験でした。

## 過去：Claude codeの知識は、YouTubeで名前を聞く程度

私の生成AIに関する主な情報収集源は、通勤途中に聞き流すYouTube動画です。

非エンジニアと言えど、私もChatGPTやCopilotなどの生成AIチャットツールは日常的に利用していますので、新しい機能の使い方や活用事例などYouTube動画から学んでいます。

なにせラジオ感覚ですので、色んな動画を流しっぱなしにしていると、それほど関心のない情報も流れ込んできます。  
Claude Codeもその一つでした。

最初に興味を持ったのはSkillsの機能。AIに作業手順書のようなファイルを読み込ませることで、特定タスクの再現性を高める機能です。  
次にAgent Teamsという、親エージェントと子エージェントでチームを作って並行作業を行う機能、が実装されたあたりで「なにこれおもしろそう！試してみたい！」と思うようになりました。

ですがClaude Codeについて調べてみると、非エンジニアの天敵とも言える、**黒背景＋白英字の画面**が出てくるわ、Proプラン以上の契約が必要だわで、怖気づいて行動に移すことができませんでした。

---

## 転機：「Everything Claude Code」のディレクトリ構成に既視感を覚えた

2026年1月、「Everything Claude Code」という言葉が話題になっていました。調べてみれば、ハッカソン優勝者のClaude Code設定集が公開されたとのことで、Claude Codeに触れていないながらも興味は沸きます。

興味本位でEverything Claude Codeを解説しているテック記事を読んだら、こんなディレクトリ構成になっているとのこと。

```
everything-claude-code/
├── CLAUDE.md          # プロジェクト全体の指示書
├── agents/            # 28の専門エージェント
├── skills/            # 125のワークフロー定義
├── commands/          # 60のスラッシュコマンド
├── rules/             # 常時適用のガイドライン
├── hooks/             # 自動トリガー設定
├── contexts/          # 動的コンテキスト
├── mcp-configs/       # 外部ツール連携設定
└── docs/              # ドキュメント
```

参考：[everything-claude-code](https://github.com/affaan-m/everything-claude-code)

エージェントの役割を定義したAgents。手順書であるSkills。規範となるRules。必要な外部リソースを示したMCP-Configsなどなど。

このディレクトリ構成を見たときに、私は**既視感**を覚えました。

**私「ん？これってなんか、QMS（品質マネジメントシステム）の管理に似てる？」**

Claude Codeの最強設定集が、抽象的な目線でみると、すごく馴染みのあるものに見えてきたのです。

### 「AIエージェントのためのQMS」という考え方

QMSは我々人間のために作られた管理システムですが、同じことをAIエージェントのために作ったものが「Everything Claude Code」と捉えることもできるんじゃないだろうか？

そう考えたとき私の中で、逆の発想が生まれました。

「会社のQMS文書をMarkdownに変換し、各部署の役割を与えたAIエージェントに読ませれば、バーチャル組織でQMSを回せるってこと？」

日々、本職でQMSのPDCAサイクル（Plan→Do→Check→Act）の回らなさっぷりに絶望している私にとって、こんなに楽しそうな題材はありません。

やりたいことを見つけ、それがClaude Codeならできそうという気づきを得たことで、一気にClaude Codeを使いたい意欲が高まってきました。

---

## やりたいことは見つかったが、一歩目を踏み出す足は重い

月20ドルという金額はまだしも、黒背景＋白英字と対峙する勇気がまだ私には出てきていませんでした。

そこで私は、「無料で試せるものから慣れていこう！」という考えに至ります。

当時盛り上がっていたGoogle Antigravityを触ってみたり、Google AI StudioのBuildでアプリを作ってみたり、Gemini CLIをインストールしてみたり。

どれも「すごいな〜」と感心はしましたが、これを使って何かやってみようという所までは行けませんでした。

今にして思えば、Everything Claude Codeを見てしまったので、頭の中にはClaude Codeしかなかったんでしょうね。  
Gemini CLIに触れたことでターミナルも少し慣れましたので、いよいよClaude Codeを使ってみようと決意しました。

## Claudeの有料プランを契約する前も、Claude無料版と一緒にプランニング

有料プランを契約する前もClaudeの無料版と対話して、どのプランがいいか、契約したら何から始めるか、念入りにプランニングを行った上で、Proプランを契約しました。

我ながら今回は慎重過ぎたなと思っていますが、それくらい、私にとってClaude Codeというのはハードルが高く見えていたんだと思います。

さっさと有料プランを契約した方が早かったかもしれません。  
それでも、先に無料ツールに触れたことで、やりたいことの具体性が増し、意思を持ってProプラン契約に進めたので、結果的に良かったなと思っています。

## おわりに

私のケースは、先に無料ツールをいろいろ触ったことでAIエージェントの感触を掴むことができましたし、情報を受信する感度も高まったと思います。

「自分にできるか自信がない」と感じている人も、無料でできることからトライしてみましょう。  
試した経験がきっかけとなって、いきなり視界が開けるかもしれませんよ。
