「最新AIを入れたのに、仕事があまり変わらない」。この原因の多くは、モデル性能ではなく**仕事の渡し方**にあります。

海外の先行企業は、AIを「質問に答えるチャット」から「資料を集め、複数工程を進め、成果物を検証して返す実務エージェント」へ移し始めています。NVIDIAのイベント準備、Australian Payments Plusの照合作業、Deutsche Telekomの顧客対応、Rampの障害調査、Harveyの法務分析、Lokaliseの多言語運用など、業界は違っても成功パターンには驚くほど共通点があります。

本稿は、2026年7月時点の公式情報と海外事例を基に、**AnthropicのFable 5とOpenAIのGPT-5.6を、読者が自分の仕事へ移植できる粒度**まで分解した実践ガイドです。

単なる機能比較ではありません。次のものを持ち帰れるように設計しました。

* どの仕事をFable 5、GPT-5.6、または両方へ任せるべきか
* 海外企業の成功事例を、自社サイズの手順へ縮小する方法
* 開発、営業、金融、法務、調査、翻訳、経営報告の実践ワークフロー
* そのまま使える長文プロンプト、品質評価表、30日導入計画
* 長時間エージェント、マルチエージェント、ツール接続、承認境界の設計
* 情報漏えい、誤引用、過剰自動化を防ぐガバナンス

なお、Fable 5とGPT-5.6は登場して間もないモデルです。そこで本稿では、**最新モデル固有の公式機能**と、**Claude／ChatGPT／Codexの製品群で既に実証された海外の運用パターン**を明確に分けます。「ある企業が最新モデルだけでこの成果を出した」と誤読させず、再現可能な設計だけを抽出します。

※本稿の「Fable」は物語生成サービス等ではなく、Anthropicが2026年7月に発表したAIモデル「Claude Fable 5」を指します。

## 1. 結論：海外では「モデル選び」より「仕事の設計」が差になっている

Fable 5とGPT-5.6の優劣を一言で決めるのは実務的ではありません。先に仕事を分類すると、選択は簡単になります。

**Fable 5を第一候補にしやすい仕事**

* 数時間から数日にまたがる大規模な実装、移行、調査
* 長い資料群を読み続け、途中経過を記憶しながら進める仕事
* 自律的なサブエージェント分担と、繰り返し検証が効く仕事
* 大きなコードベース、PDF、表、画像を横断する複雑な案件

**GPT-5.6を第一候補にしやすい仕事**

* 営業、金融、運用などで多くのツールを呼び、最終成果物まで仕上げる仕事
* Sol／Terra／Lunaを使い分け、品質・速度・費用を細かく制御する運用
* 独立した調査を並列化し、集約役が矛盾を解消する仕事
* ChatGPT WorkやCodexを使い、表計算、文書、スライド、Web成果物を作る仕事

**両方を組み合わせる価値が高い仕事**

* 高リスクな法務・金融・公開文書で、作成者と検証者を分離したい
* 長時間の実装と、関係者向け資料・承認フローの両方が必要
* 片方の推論癖に依存せず、重要な主張を別モデルで反証したい

モデル名ではなく、次の5軸で決めてください。

1. **仕事の長さ**：5分か、5時間か、数日か
2. **必要な文脈**：1ファイルか、社内システム横断か
3. **失敗コスト**：やり直せるか、契約・送金・公開に影響するか
4. **成果物**：回答文か、検証済みのファイルや更新操作か
5. **承認境界**：どこまで自動、どこから人間承認か

## 2. 海外で起きている5つの変化

### 変化1：回答ではなく「完成物」を受け取る

従来は「メール案を書いて」「この表を要約して」が中心でした。現在の先進運用は、CRM、会議記録、メール、スプレッドシート、社内文書をAIに読ませ、優先順位付きの営業ブリーフ、月次報告、再現可能な分析、修正済みコードなどを返させます。

成果物には、結論だけでなく**出典、未確認事項、仮定、次の承認者**を含めます。これにより、人間はゼロから作るのではなく、根拠を確認して決める役割へ移れます。

### 変化2：プロンプトより「コンテキスト」が重要になる

賢いモデルでも、古いファイル、欠けた顧客情報、定義の違うKPIを渡せば誤ります。海外の実装で効いているのは、CRM、データ基盤、チケット、ログ、文書庫などを、権限付きコネクタで安全につなぐことです。

長い指示文より、次の4点をそろえる方が品質を上げます。

* 正本となる情報源
* 指標・用語・顧客名の定義
* 更新日時と責任者
* 参照してよい範囲、書き込んでよい範囲

### 変化3：一人のAIから「役割分担するチーム」へ

調査、反証、計算、文章化、最終監査を一つの会話で順番に行うと、途中の思い込みが最後まで残ります。そこで、独立性の高い仕事はサブエージェントへ分け、最後に統合役が矛盾を解消します。

ただし、何でも並列にすればよいわけではありません。前工程の答えがないと次へ進めない仕事は直列、地域別調査や複数仮説の検証は並列が向きます。

### 変化4：一つのモデルを全件に使わない

高性能モデルは高価で遅くなりやすい一方、分類、抽出、形式変換の多くは小型モデルで足ります。GPT-5.6ではSol／Terra／Luna、Fable 5ではeffortやタスク予算を使い、難易度に応じて計算量を変えられます。

海外運用の要点は「最高性能を常用する」ことではなく、**難しい判断だけ上位モデルへ上げるルーティング**です。

### 変化5：デモより評価とガバナンスを先に作る

法務、金融、通信の導入事例に共通するのは、現場の専門家が評価基準を作っていることです。文章が自然かではなく、正確性、重要事項の抜け、引用の正しさ、規定違反、修正時間を測ります。

「AIができるか」ではなく、**人間の現行工程より安全・速いか**を比較するのが実務導入です。

## 3. 海外事例の早見図──成果の裏にある共通設計

![](https://assets.st-note.com/img/1784203209-tL0oEBVZs247QkPbuNlqf8Gn.jpg?width=1200)

代表例を短く整理します。

**NVIDIA：大規模イベント準備と事後分析**

ChatGPT Workの公式紹介では、GTCの準備で登録アカウント、面談、営業準備をまとめ、従来Excelで行っていた作業の大きな部分を自動化。終了後は数百件の文字起こしと会議メモを分析し、チームが「収集」ではなく「発見」に時間を使えるようにしたと紹介されています。

**Australian Payments Plus：照合調査と安全な全社展開**

AP+では、複雑な照合問題の調査が4時間から30分になった例が報告されています。価値があるのは数字だけではありません。管理されたツール、プライバシー、ガバナンス、社内チャンピオン制度を同時に整え、300以上のカスタムGPTと1,000以上のProjectsへ広げています。

**Deutsche Telekom：業務再設計としてのAI**

月間アクティブユーザーは5万人超。顧客対応、通話中支援、通話後要約、翻訳、ネットワーク運用まで広げています。公式事例が強調するのは、AIを追加機能として配るだけでなく、仕事そのものを再設計することです。

**Ramp：開発・障害対応の並列化**

これはFable 5以前のClaude Code事例ですが、長時間・サブエージェント型のFable 5に引き継げる運用パターンです。複数セッション、テストループ、DatadogやSentryとの接続により、障害調査時間を最大80%短縮したとされています。

**Harvey／Quillit／Lokalise：専門品質は専門家の評価で作る**

法務のHarveyは、AIエンジニアと法律専門家が共同で、正確性、重要性、幻覚、文体、形式を評価。定性調査のQuillitは引用精度を重視し、ローカライズのLokaliseは言語ペアごとのモデル選択、翻訳メモリ、品質評価を組み合わせました。

成功企業は「万能AI」を探しているのではありません。**対象業務、正本データ、評価基準、承認者を一つの運用システムにしています。**

## 4. 今日から試せる最小ワークフロー：「資料束→意思決定パック」

まずは、送信・公開・更新をAIに許可しない読み取り専用の仕事から始めます。会議メモ、週次資料、表計算、顧客メールなど5〜20点を一つのフォルダへ集め、次の指示をFable 5またはGPT-5.6へ渡してください。

```
あなたは意思決定支援担当です。添付資料だけを根拠に、責任者が15分で判断できる「意思決定パック」を作ってください。

目的：［今回決めたいこと］
読者：［役職／前提知識］
期限：［日付］

出力：
1. 3行の結論
2. 確認できた事実（各項目にファイル名・該当箇所）
3. 推論（事実と明確に分ける）
4. 重要な不一致・欠損・古い情報
5. 選択肢A/B/Cと、費用・効果・リスク
6. 推奨案と反対意見
7. 決定後48時間の行動計画（担当・期限）

禁止：
- 資料にない数値や発言を補わない
- 出典がない主張を事実として書かない
- 外部への送信、ファイル更新、公開操作をしない

完成条件：
- すべての重要数値に出典がある
- 事実／推論／提案が区別されている
- 不明点を「不明」と表示する
- 最後に、自分の結論を崩す反証を3つ挙げる
```

この一回で見るべきKPIは、文章の美しさではありません。

* 人間が資料を探す時間
* 出典の正解率
* 重要な抜けの数
* 人間が修正した時間
* 意思決定までのリードタイム

この最小実験が安定したら、次に「情報の収集」「定型ファイルの作成」「承認後の更新」を一段ずつ足します。最初から全自動にしないことが、結果的に最短です。

## この先で扱う内容

有料部分では、海外事例を単に紹介するのではなく、**読者が同じ構造を再現するための手順**へ落とし込みます。

* NVIDIA型：イベント前後の営業インテリジェンス
* AP+型：照合エラーの証拠付き調査
* Ramp型：コード・ログ・監視を横断する障害対応
* Harvey型：法務レビューと二重検証
* Quillit型：インタビューの引用付き定性分析
* Lokalise型：多言語コンテンツのルーティングと品質評価
* Fable 5向け：数時間動く長期タスクの設計テンプレート
* GPT-5.6向け：Sol／Terra／Luna、PTC、マルチエージェントの使い分け
* 8職種の実践プロンプト集
* コネクタ設計、承認境界、データ保護、費用管理
* 失敗しやすい設計と修正方法
* 30日導入ロードマップと評価シート

ここから先は「読んで分かる」ではなく、**コピーして小さく実装できる**ことを目標にします。
