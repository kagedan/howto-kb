普段から Claude Code を開発でフルに活用していますが、僕はコードを読んで細かめにフィードバックする方で、ふと「Claude Code へのフィードバックに、自分のソフトウェアエンジニアとしての考え方がにじみ出ているのでは？」と思いました。

他の人のエンジニアリング哲学みたいなのを聞くのが結構好きなのですが、自分がどう考えているかちゃんと文字にする機会がないので、Claude Code とのやり取りの中でにじみ出ている自分のソフトウェア開発の考え方みたいなものを抽出してみることにしました。

# Claude Code の会話ログ

Claude Code は、セッションのやり取りを `~/.claude/projects/` 以下に JSONL 形式で保存しています。プロジェクトごとにディレクトリが切られて、その中にセッション単位のファイルが溜まっていきます。

このログの保存期間は `settings.json` の `cleanupPeriodDays` に設定されていて、デフォルトで 30 になっています。そのため、何も設定しなければログは30日で消えます。これを修正することで、保存期間を延ばすこともできます。

```
// ~/.claude/settings.json
{
  "cleanupPeriodDays": 36500  // 100年。実質消えない
}
```

保存期間を延ばして3ヶ月分のログを集めたところ、溜まっていたログの量は以下のとおりです。

| 項目 | 値 |
| --- | --- |
| プロジェクト数 | 49 |
| セッションファイル総数 | 717 |
| 合計サイズ | 356MB |

テキストファイルですが、意外とサイズが大きいですね。

# 抽出方法

やったことはシンプルで、2 ステップです。

1. **各セッションから、ソフトウェアエンジニアリングの考え方を抽出する:** 各セッションのログをそれぞれ独立したエージェントに読ませて、「このフィードバックをしている理由として、こういう考え方がある」という、転用できる原則をまとめさせました。そのセッションの具体的な内容ではなく、背景にある考え方をまとめるよう指示しました。
2. **抽出結果を1つのドキュメントに統合する:** 700セッション分が出てくるので、これを1つのドキュメントにまとめさせました。

# 出てきたもの

ここから出てきたものを抜粋します。

## 出力は狭く、入力は広く

> * 調査は徹底的、出力は絞る
> * 文書から強調記号を消す
> * 形骸化したルールを削除する
> * 「全部書いたほうが親切」は錯覚

AI が書いたコメントとかドキュメントに、「冗長すぎる」「これは自明なので不要」みたいなことを言いまくっている記憶はあります。

## 仕組み > 注意

> * 自動化で済むことをチェックリストに書かない
> * 注意力で済まないことを型・スキーマ・マクロで強制
> * 「気を付ける」より「構造的に不可能」
> * レビューは構造的に独立した文脈で

必ずやってほしいこと・絶対にやらないでほしいことを CLAUDE.md に書いて何とかなるとは思っていなくて、静的チェックが全てだと思っているので、それがにじみ出ていそうです。

## 動かしてから決める

> * 試作 → 判断 → 書き換えの循環
> * 検証環境に観察用の計測装置を投げ込む
> * 試作で議論を終わらせる
> * B → A の二段昇格
> * 文書化されていない挙動は試して確かめる

「本当にそれでいける？その処理だけスクリプト書いて動かしてみて」みたいなのを言いまくった結果っぽいです。

## 共有面と個人面を分ける

> * 共有された場所に出るのは完成した成果物のみ
> * 起稿と公開を分ける
> * 出荷用の差分と調査用の差分を分離
> * 試作環境と整備された場所を分ける

雑にローカルで試行錯誤をよくやらせる一方で、プルリクをレビューに出すときは「読み手のことを考えて」みたいなダメ出しをいっぱいした結果、明示的に指示しなくても試作とプルリクを分けるようになったみたいです。

## 受け手のための設計

> * 文書は読み手の道具
> * メトリクスは判断のための情報
> * PR はレビュアーへの引き継ぎ書
> * コミットメッセージは将来の自分への手紙
> * エラーメッセージはデバッグ可能な情報量で

個人的な感覚ですが、AI はコードベースをメッチャ読んでいるので、「コードベースをメッチャ読めば分かるかもしれないざっくりしたエラーメッセージ/ドキュメント/コメント」を出力する印象があります。これはよくダメ出ししていた記憶があります。

# まとめ

自分の言動から抽出された Principles みたいなの、書いてあることに全て納得感があります（それはそう）。エコーチェンバー的な危険も感じますが、自分がぼんやり考えていたことに名前がついて整理されるのは気持ちが良いので、ぜひお試しください。
