## はじめに

最近、AIコーディングを使う機会が増え、コードを書く速度はかなり上がりました。一方で、使われなくなったコードや似たような実装、複雑になった関数をどう継続的に見つけるかは、以前より重要になっていると感じています。  
今回は TypeScript / JavaScript 向けツールである [fallow-rs/fallow](https://github.com/fallow-rs/fallow) を使って、デッドコード、コード重複などのチェックを試してみます。

## fallowとは

fallowはTypeScriptおよびJavaScript向けの**決定論的**なコードベース分析ツールです。デッドコード、コード重複などをチェックしてくれます。

> fallowはAIを用いて新たな発見を生成するのではなく、人間やAIエージェントが検証可能な決定論的なエビデンスを提供します。

分析エンジンにはAIを内蔵していないようです。

## クイックスタートガイドにそって早速試す

ではデッドコード、コード重複などを特定してもらいましょう。  
クリーンアップ対象としては、未使用のファイル、未使用のエクスポート、未使用の依存関係などが含まれるようです。

* `audit`: 現在の変更ファイルをベースブランチと比較して監査する
* `--base`: 比較元を指定（例: `--base staging`）

### サマリ

![](https://static.zenn.studio/user-upload/673c75ea91fc-20260604.png)  
実行すると、まず複雑度チェックのサマリが表示されました。

`✗ 206 above threshold · 4087 analyzed (0.12s)`

これは、解析対象となった関数などが 4,087 件あり、そのうち設定された複雑度のしきい値を超えたものが 206 件あったようです。

続いて、audit 全体の結果として以下のようなサマリが表示されました。

`✗ dead code: 95 issues · complexity: 206 findings · duplication: 305 clone groups · 210 changed files (1.85s)`

今回の結果では、デットコード系の指摘が 95 件、複雑度の指摘が 206 件、重複コードのまとまりが 305 件見つかりました。また、ベースブランチとの差分として 210 ファイルが変更対象として見られています。

すでにベースブランチ側に存在していた問題は inherited findings として扱われ、今回の結果では 406 件が判定対象から除外されていました。  
既存の問題も含めてすべて失敗条件にしたい場合は、表示にもある通り `--gate all` を付けて実行します。

```
npx fallow audit --gate all
```

## エージェントにfallowでセルフチェックさせる

fallowは人間がCLIで確認するだけでなく、エージェントに使わせることも想定されています。

たとえば、エージェントが新機能を実装した後に以下を実行します。

```
fallow dead-code --changed-since [ブランチ名] --format json
```

これにより、今回の変更によって未使用になったファイルやexportを検出できます。  
特に良いなと思ったのは、`--format json` の出力がエージェントフレンドリーになっている点です。  
`--format json` の各issueに、修正提案を含む `actions` 配列と `auto_fixable` フラグが含まれると説明されています。つまり、エージェント側で「自動修正できそうなもの」と「人間が確認した方がよいもの」を分けて扱いやすくなります。

```
"actions": [
        {
          "type": "remove-export",
          "auto_fixable": true,
          "description": "Remove the `export` (or `export type`) keyword from the type declaration"
        },
```

ちなみに、エージェント向けには [fallow-rs/fallow-skills](https://github.com/fallow-rs/fallow-skills) も用意されています。  
Claude Code、Codexなどに、fallowのコマンドの使い分けや注意点を教えるためのスキル集のようです。今回は深掘りしませんが、エージェントに継続的にfallowを使わせるなら、こちらも合わせて入れておくと便利そうです。

## まとめ

今回は、TypeScript / JavaScript向けのコードベース分析ツールであるfallowを試してみました。

`npx fallow audit` を実行するだけで、デッドコード、複雑度、コード重複をまとめて確認できるのは便利でした。特に、AIコーディングで変更量が増えているコードベースでは、レビューやリファクタの優先順位を決める材料として使いやすそうです。

また、`--format json` の出力には `actions` 配列や `auto_fixable` フラグが含まれており、エージェントに修正候補を渡しやすい点も良さそうでした。MCP連携やfallow-skillsも用意されているので、エージェントにサポートしてもらう用途にも使えそうです。

一方で、デッドコード系の指摘は静的解析だけで判断しきれないケースもあるため、最終的には人間orエージェントが確認する前提で使うのが良さそうです。  
今回はローカル実行とエージェント連携を中心に試しましたが、次は[GitHub Actions](https://github.com/fallow-rs/fallow#ci-integration)に導入して、PR作成時に自動で `fallow audit` を実行するところまで試してみたいです。
