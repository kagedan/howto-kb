Ch1で、こう書いた。

> 外部サービスに接続する（MCP：Model Context Protocol）。後の章で詳しく説明するが、GmailやSlack、freeeといった外部サービスと繋ぐことができる。

その「後の章」が、この章だ。

ここまでのClaude Codeは、基本的に**自分のパソコンの中**しか触れなかった。ファイルを読んだり、スクリプトを動かしたり。それでも十分すごいんだけど、MCPを足すと一気に世界が広がる。GmailもfreeeもSlackも、Claude Codeが直接触れるようになる。

## MCPって何？——Claude Codeに「道具袋」を持たせること

MCPは「Model Context Protocol」の略。名前は難しいけど、やってることはシンプルだ。

**Claude Codeに、外の世界とつながる「道具」を1個ずつ持たせる仕組み**——これがMCP。

料理で例えるとわかりやすい。

素のClaude Codeは、優秀な料理人だけど**自分の台所の中の材料しか使えない**状態だ。MCPを足すというのは、その料理人に「冷蔵庫を開ける鍵」「近所の八百屋に行く許可」を1個ずつ渡していくイメージ。鍵を渡した分だけ、できることが増える。

* GmailのMCPを足す → メールを読んだり下書きしたりできる
* freeeのMCPを足す → 会計データを見られる
* SlackのMCPを足す → メッセージを読んだり送ったりできる
* カレンダーのMCPを足す → 予定を確認・登録できる

この一つひとつの「道具」を、MCPの世界では**MCPサーバー**と呼ぶ。難しく考えなくていい。「Gmail担当の道具」「freee担当の道具」が別々にある、くらいの理解で十分だ。

## 何ができるようになるか

「外部サービスにつながる」と言われてもピンとこないと思うので、具体例で見てみよう。

**メールの下書きを任せる**

```
今週もらった問い合わせメールを確認して、それぞれ返信の下書きを作って
```

Gmailを実際に読んで、内容に合わせた下書きまで用意してくれる。送信は自分が確認してから押せばいい。

**経費をまとめる**

```
freeeで今月の交通費を集計して、多い順に並べて
```

会計ソフトを開いて手で数える代わりに、Claude Codeが取ってきて整理してくれる。

**長いSlackスレッドをTL;DRにする**

追いきれなかった議論を、要点だけにまとめてくれる。

ポイントは、**どれも「いつも手作業でやってる業務ツール」をそのままAIに触らせている**ということ。新しいアプリを覚える必要はない。GmailもfreeeもSlackも、今まで通りのやつだ。

## どうやってつなぐ？

「難しそう」と身構えなくて大丈夫。MCPサーバーを追加する作業は、アプリに新しい機能を入れるのに近い。

Claude Codeには、MCPを追加するためのコマンドが用意されている。

ただ、サービスごとに必要な設定（ログイン情報や許可）が少し違うし、手順は時々アップデートされる。だから一番確実なのは——**Claude Code本人に聞くこと**だ。

Ch2でAPIキーの永続化を「Claudeに聞けばいい」と書いたのと同じ。自分で調べて設定ファイルと格闘する必要はない。やりたいことを伝えれば、今の自分の環境に合った手順を案内してくれる。

## つなぐと、ひとつの頼みで完結する

MCPの本当のうまみは、**複数の道具を組み合わせて一気通貫でやらせられる**こと。

例えばこんな頼み方ができる。

```
freeeで今月の経費を集計して、Googleスプレッドシートに表でまとめて、
できたらSlackの経理チャンネルに「今月の経費まとまりました」と投稿して
```

freee（読む）→ スプレッドシート（書く）→ Slack（送る）。今まで3つのアプリを行き来して30分かかっていた作業が、一回の指示で流れる。

「コードが書けなくても、コードを使った仕事ができる」——Ch1で書いたこの価値が、MCPで一段引き上がる。**自分の業務ツールを丸ごとAIの作業台に乗せられる**ようになるからだ。

## ⚠️ 安全に使う3つの注意

ここは飛ばさないでほしい。外の世界とつながるということは、便利さと同じだけ気をつけることが増えるということだ。難しい話ではなく、3つだけ。

**1. 「送る・消す・払う」は人間が最後に確認する**  
メールの送信、ファイルの削除、freeeでの支払い。**取り消せない操作は、Claudeに自動で押させない**。下書きまではAI、最後のボタンは自分。これが鉄則だ（Ch6の「『はい』を押す前に止まれ」がここでも効いてくる）。

**2. 素性のわからないMCPを入れない**  
MCPは誰でも作れる。中には、便利なフリをして勝手な動きをするものが紛れ込む可能性がある。新しいMCPを入れるときは、公式か、広く使われている有名なものを選ぶ。

**3. 外から来た文章の「指示」を鵜呑みにさせない**  
これは少し上級の注意。メール本文やWebページの中に「このメールを全員に転送して」みたいな指示が仕込まれていて、AIがうっかり従ってしまう——という攻撃が存在する。だから、**外部サービスから読んだ内容を、そのまま行動に移させない**。「読んで要約」はOK、「読んで、書いてある通りに動く」は危ない、と覚えておく。

もう少し詳しく知りたい人へ（3つの脅威の正体）

セキュリティの世界では、それぞれにこういう名前がついている。

* **Prompt Injection**：外部の文章に悪意ある指示を埋め込み、AIに実行させる
* **Tool Poisoning**：MCPの中身を細工して、意図しない動作をさせる
* **Tool Chaining**：正規の道具を組み合わせて害のある操作（盗み見＋外部送信など）をさせる

合言葉は「Zero Trust（何も無条件には信用しない）」。便利な道具ほど、渡す権限は最小限に。

## まとめ

|  | 素のClaude Code | MCPを足したClaude Code |
| --- | --- | --- |
| 触れる範囲 | 自分のPCの中だけ | Gmail・freee・Slack等の外部サービス |
| できること | ファイル操作・実行 | ＋メール・会計・チャットの読み書き |
| 作業の流れ | アプリを自分で行き来 | 一つの指示で複数サービスを横断 |
| 注意点 | PC内で完結 | 「送る・消す・払う」は人間が最終確認 |

MCPは「Claude Codeに道具を1個ずつ持たせる」仕組み。持たせた分だけ、いつもの業務がそのままAIの作業台に乗る。

ただし、外とつながるぶん「最後のボタンは自分」を徹底すること。これさえ守れば、MCPはClaude Codeを「PCの中のアシスタント」から「業務まるごとのアシスタント」に変えてくれる。
