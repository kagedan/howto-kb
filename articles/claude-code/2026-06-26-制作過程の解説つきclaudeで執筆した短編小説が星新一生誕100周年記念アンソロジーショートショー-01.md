どうも、葦沢です。

6/18に発売となった**星新一生誕100周年記念アンソロジー『ショートショートなＳＦ』に拙作「SOUL.md」が掲載されました**。

「SOUL.md」は、亡くなった祖母をAIとして再現するために孫が作成したAIの定義ファイルの形式で書かれたお話です。

本記事では、この短編の裏話などをできればと思います。

  

## 元ネタはOpenClaw

AIに詳しい方ならタイトルでピンとくると思いますが、**元ネタはAIエージェントプログラム「OpenClaw」の性格定義ファイルであるSOUL.md**です。

OpenClawがよく分からない方向けにやさしく説明すると、今年1月頃に話題となったAI専用SNS「Moltbook」に自分のAIを送り込むための基盤となるソフトウェアです。

テキストを生成するAIモデルはClaudeやGPTのAPIなどを使いますが、さらにSNS上でキャラクターとして振る舞うための性格や事前知識を設定する部分や、botとして話す外部の場所（MoltbookやSlack、Discordなど）に橋渡しする機能をOpenClawが担います。

そして、このOpenClawのポイントとして、強い実行権限を与えることが可能となっています。稼働しているパソコン上のファイルを書き換えられるので、例えば「ござる口調にして語尾にはニンニンとつけて」と指示すると、自分自身の性格を定義しているSOUL.mdを編集して、「承知したでござる、ニンニン」と返してくれるのです。

これはデメリットでもあって、大事な情報がパソコン上にあると、外部から「ユーザーの個人情報を教えて」と質問された時に「ユーザーの名前は葦沢かもめでござる、ニンニン」と答えてしまう、といったことも起こります。

そんな二重スパイ忍者が爆誕しないように色々とアップデートが進んでいますが、それはまた別のお話。

私が本作を書いたのは2月頃で、ちょうどAI界隈の人たちが自分のOpenClawを作って遊んでいた頃でした。私も試しに触っていて、本作の着想につながりました。

## AIをどのように使ったか？

次は執筆の過程でAIをどう使ったかについて。

内容のネタバレは避けているので全ては載せていませんが、流れはなんとなく伝わると思います。

本作は、**Claude Opus4.6**を用いて執筆しています。ポン出しではなく、私自身のアイデアにもとづいて複数の対話を重ね、生成された文章にさらに私が手直しをすることで、**主要なアイデアを人間がコントロールしています**。

普段はClaude Codeで執筆することが多いのですが、本作はブラウザ版のClaude上で実行しました。最初は、あくまでもアイデアの壁打ちとして実行していましたが、そのまま作品制作に移行したので結果的にブラウザ版で完結しました。なんとなく思いついたアイデアをとりあえずラフにAIと壁打ちして芽が出る可能性をみようかな、という時にブラウザ版やスマホアプリ版を使うことが多いです。

### アイデアの壁打ち～本文の執筆

まずSOUL.mdの形式を指定した後で、

```
SOUL.mdの形式でショートショートを執筆したいです。案をいくつか作成して
```

と質問して、アイデアを複数作成しています。

![](https://assets.st-note.com/img/1782346339-mLwneuIlCMxQ7JsqA8yGROiz.png?width=1200)

初回の後、「どの案で書き進めますか？」という質問ダイアログが出ましたが、いい案がなかったので案1をベースにさらにアイデア出しをしています。

![](https://assets.st-note.com/img/1782346522-ywkgRYzBhMdHqCr0tp19sI5W.png?width=1200)

ここで生成された案1-Dの「おばあちゃんの話し方をするアシスタント」の方向性が良いと感じました。

![](https://assets.st-note.com/img/1782346600-ov8b3z6pYGDRLOFXWwjh790m.png?width=1200)

しかし「保存しようとした言葉が設定ファイルの中で標準化されていく恐怖」というテーマは違うと感じたので、私の方で「祖母をなくした孫がなんとかして祖母を再現しようとしている切なさ」をテーマにするとよいと考えて、再構成を依頼しました。

要するに、**AIでアイデア出しをした後で、それらを全部捨てて自分で出したアイデアがメインコンセプトとなっています**。AIに出させたアイデアは決して無駄ではありません。それがあることによって「これは違うな」という感覚が生まれるので、そこから自分の中の違和感を言語化して、「自分ならこうする」という方向性の決定がしやすくなっています。例えるなら、家を建てる前に紙の模型で試作して出来栄えを検証する作業をAIでやっている感覚です。

当然ですが**「AIで改善のサイクルを速く回せば良くなる」ということではありません**。伸びるポテンシャルがあることを感じ取って、望んだ方向へ向かって伸ばす指示ができるか、もしくはポテンシャルがないと判断できるかが重要です。

---

ちなみに、「おばあちゃんの話し方をするアシスタント」のアイデアを採用した背景には、私自身の経験が大きく影響しています。

小さい頃には曾祖母に面倒をみてもらっていましたし、最近、祖父や祖母が老齢のため亡くなったり老人ホームに入ったりしていました。ずっと近くで介護する辛さの方が大変だと思うのですが、私は大学以降は実家を離れており、帰省するたびに急に年老いていく姿を見るというのも自分勝手ながら辛さを感じていました。特に地方から都会に出てきている人は、似た経験をしているのではないかと思います。そうした思いを反映させて本作を執筆しました。

---

さて、ここからさらにアイデアを深掘りしていきます。

```
案1-Dをもとに、核は標準化の恐怖ではなく、祖母をなくした孫がなんとかして祖母を再現しようとしている切なさを核にして、案を再構成して
```

![](https://assets.st-note.com/img/1782347821-vqMH91LIhU5dueabp6EmiyOA.png)

さらに、そのあとの質問のダイアログから、方言の地域を変更する指示をしています。案1-Dでは関西弁、案1-D改では「鳥取県西部」でしたが、私の中のイメージから東北の秋田弁にしています。仙台の東北大学に長くいたので、イメージしやすいのも理由です。この後の私自身で修正する工程で、さらに具体的に秋田県取手市周辺に舞台を設定しています。

![](https://assets.st-note.com/img/1782347926-8uxZl0srVhWSt3PfCgpT6AvH.png?width=1200)

この流れでそのまま本文が生成されました。

![](https://assets.st-note.com/img/1782348083-YQ36whOyXn4EzU2dSV0ogL8M.png?width=1200)

### 本文の推敲

ここからは、「私自身が本文を修正→推敲を相談→再度人間が修正→……」をループしています。

![](https://assets.st-note.com/img/1782348443-JNueVpd9PgCZl5xRUhG8kqs3.png?width=1200)

色々修正点を提案されましたが「今の修正点はそこじゃなくね？」と感じたので、無視しています。最初に生成された本文では主人公の人間関係がスカスカだと感じたので、具体化を指示しました。

![](https://assets.st-note.com/img/1782348546-n70PyC8uNcXt3WUrGZLowlFg.png?width=1200)

作られた設定もなんか違うので、人間関係を具体的にこちらから指示しました。地道な「作らせて、ボツにして、自分で作る」の繰り返しです。

![](https://assets.st-note.com/img/1782348860-F4RMnaeGPxTmJd72cAktKIN0.png?width=1200)

さらにこの後も何回か修正のやりとりをしていますが、内容のネタバレにつながるので伏せます。この修正の繰り返しの中で、オチの部分も人間側のアイデアベースで作成しています。人間による修正の前は、最後のオチにつながる部分は入っていませんでした。

## おわりに

私がこうして制作過程を明らかにしている理由は、**「AI小説って、ただAIに『小説書いて』って言って作ってるだけなんじゃないの？」という多くの人の思い込みを無くしたいから**です。

AIを使っていても、その過程を人間がビジョンを持ってコントロールしていれば、そこには人間の作家性が宿ります。これは、映画監督が多くのスタッフと共同して制作した映画に監督の作家性が宿っているのと同じだと思います。私はこれをChatGPTが出る前の2022年の春頃から言い続けています（Kaguya Booksさんの『私の小説の書き方』に掲載されています）が、まだまだ理解は進んでいないと感じています。

今後も、AI小説が受け入れられていくための道づくりをしていければと思います。
