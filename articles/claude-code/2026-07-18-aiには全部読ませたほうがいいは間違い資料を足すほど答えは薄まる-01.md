「あなたはプロの編集者です」と役割を与えた直後は、切れ味のいいレビューが返ってきた。それが20往復もやり取りを重ねた頃には「とても素敵な文章ですね!」という骨抜きの返事に、覚えはないでしょうか。あるいは、会議の資料一式をぜんぶ読み込ませて要点を聞いたら、どの資料にも書いていないような、当たり障りのない一般論が返ってきたことは。

![](https://assets.st-note.com/img/1783852108-X8fsVIzuSMLlH7QaxJv3qYEZ.png?width=1200)

「AIにはとにかく資料を食わせろ」「たくさん読ませるほど賢くなる」「全部渡せばよしなにやってくれる」。この手の言説は、はっきり言って間違っています。読ませるほど賢くなるどころか、**読ませるほど答えは薄まる**のです。

先に正確なところも言っておくと、間違っているのは「読ませること」ではありません。「無差別に読ませること」です。関連する情報は、AIを確かに賢くします。ではその境目はどこにあるのか。この記事では、AIの「賢さ」と「劣化」が実は同じ1つの仕組みから来ていることを説明します。読み終わる頃には、冒頭の2つの症状に同じ1つの答えが出せるようになっているはずです。

  

## 前提: AIは毎回、台本の全文を読み直している

この連載の大前提を1行だけ復習します。AIは会話を覚えておらず、メッセージを送るたびに「これまでの会話+渡した資料+今回の発言」の全文を1つの台本として読み直し、続きを生成しています。詳しくは前回の記事で書きました。

今回はその続きとして、読み直される台本が「どう読まれるか」の話をします。

![](https://assets.st-note.com/img/1783852121-byLihOqAJcfMsdR12UnVWmB4.png?width=1200)

## AIは、台本の全体を見ながら「次の一語」を選んでいる

台本を読み直したAIがやっていることは、実はひとつしかありません。「次に来る言葉(トークン=AIが読む文字のかたまり)を1つ予測して、置く」。これを高速に繰り返しているだけです。

ポイントは、その予測のしかたにあります。AIは次の一語を選ぶとき、台本に載っているすべてのトークンを参照して、候補の確率を上げ下げしています。この働きはアテンション(attention=注意)機構と呼ばれていて、いまのAIの心臓部にあたります。

たとえば「このゲームは\_\_」に続く言葉を予測させると、候補の確率は手前の文脈だけで入れ替わります。

![](https://assets.st-note.com/img/1783852149-Kzq16fpeDBd9ZiPTmY2gWCMo.png?width=1200)

指示を1文字も変えていないのに、文脈が変われば答えの傾向が変わる。台本に置かれたすべてが、次の一語の確率に効いている。これがこの記事の主役になる仕組みです。

## AIの賢さの正体は、信号の「混合」

この「全部を参照して確率を上げ下げする」働きを、思い切って単純化したモデルで見てみます。AIの内部では膨大な数値が動いていますが、ここでは1本の軸(ネガティブ −1 からポジティブ +1)だけで考えます。

あるゲームについてAIに質問したとします。AIは学習した知識から、このゲームに +0.5(まあまあ好評)くらいの信号を持っている。ここで会話に「Amazonのレビューでは低評価だった」という一文を足すと、答えのトーンは中立寄りに変わります。さらに「Steamでは絶賛されている」と足すと、また持ち上がる。

![](https://assets.st-note.com/img/1783852169-H1xotD4ph78IWj5kmiUcgZ2F.png?width=1200)

つまりAIは、文脈に置かれた信号を混ぜて答えを作っています。

これは劣化ではありません。むしろAIの賢さの源泉です。あなたの状況や条件に合わせて答えが変わるのは、この混合が働いているから。プロンプトに例を2〜3個添えると出力の形が揃う、few-shot(少数の例示)と呼ばれる定番テクニックも同じ原理で、例が信号として答えを引っ張っています。

## 同じ仕組みが、裏返る

では、その +0.5 という大事な信号を、数千トークンの無関係な雑談や資料の中に埋めたら、どうなるでしょうか。

AIのアテンションには限りがあります。Anthropic はこれを attention budget(アテンション予算)と呼んでいます。新しいトークンを読ませるたびに、この予算は少しずつ削られていきます。技術的にも、トークンは文脈内の他のすべてのトークンと参照し合う構造なので、文脈が長くなるほどアテンションは薄く引き伸ばされていきます。結果として、どのモデルでも、コンテキストが長くなるほどその中の情報を正確に思い出す能力は落ちます。この現象には context rot(コンテキストの腐敗)という名前が付いていて、崖のように急に壊れるのではなく、坂道のようにじわじわ劣化するのが厄介なところです。

1軸モデルで言えばこうなります。+0.5 の高信号が、ほぼ 0 の無関係トークン数千個と一緒に混ぜられ、平均されて、答えは 0 付近、つまり当たり障りのない一般論に落ちる。

![](https://assets.st-note.com/img/1783852191-nNTC2JF8BdIoxGMYbjcE7Ot9.png?width=1200)

これで冒頭の2つの症状の答え合わせができます。

賢さ(信号の混合)と劣化(信号の希釈)は、**同じ1つの仕組みの表と裏**なのです。「読ませるほど賢くなる」は、関連する情報を足している間は本当で、無関係なものを足し始めた瞬間に嘘になります。悪いのは量ではなく、無差別さ。ここが、冒頭で「間違っているのは無差別に読ませることだ」と言った理由です。

## ついでに、「学習させたのに」も間違っている

この言説とセットでよく聞くのが、「うちの資料をたくさん学習させたから、AIはもう詳しいはずだ」という言い方です。これも仕組みの上で間違っています。

![](https://assets.st-note.com/img/1783852210-Mkt92iQ7NA0KDerpxIfJ1CGl.png?width=1200)

チャットに何万文字貼り付けても、**AIのモデル本体は1ビットも学習しません**。学習(訓練)とはモデル内部のパラメータを更新するプロセスで、開発元が行うものです。あなたが貼り付けた資料は台本に載っただけで、その台本の外には何も残りません。「読ませる」と「学習させる」はまったくの別物です。しかも読ませた効果はその会話限りで、無差別に読ませればその場ですら薄まる。二重に割に合わないのです。

## 畳めば直る。ただし、タダではない

「じゃあ、長くなってしまった会話はどうすれば?」への標準の答えは、要約して畳むことです。コンパクションと呼ばれ、Claude Code のようなツールは自動でやってくれます。手動で「新しいチャットに要約を渡す」のも原理は同じで、台本が短くなるぶん、薄まりは回復します。

ただし代償があります。要約は取捨選択であり、要約する時点では、何が後で大事になるかは分からないのです。たとえば会話の序盤で「予算は5万円まで」と決めていたとしましょう。話題がとっくに別へ移っていれば、要約の目にはただの済んだ話に見えて、削られます。そして畳んだ後のAIは、予算を知らないまま提案を始める。些細に見えた一言ほど、こうして落ちていきます。

![](https://assets.st-note.com/img/1783852231-M3gX85w14ATVd06zr9yGjNxL.png?width=1200)

だから機械任せの要約より、自分で選んで引き継ぐほうが強い。何が大事かを知っているのは、あなたのほうだからです。

## 仕事は「食わせる」ことではなく、「選ぶ」こと

仕組みが分かれば、対処はそのまま導けます。

1. 資料は、答えに効く部分だけを選んで渡す。「どれが効くか分からないから全部」は、効く部分を自分の手で薄めています
2. 大事な条件が効かなくなってきたら、最新のメッセージで言い直す。埋もれた1行を掘り当ててもらうことを祈るより、高信号を手前に置き直すほうが確実です
3. 話題が変わったら、新しいチャットに必要な前提だけを要約して引き継ぐ。毎回守らせたいルールは、カスタム指示(設定)の側へ

原則は1つにまとまります。**価値は量ではなく、信号の密度で決まる。**あなたの仕事はAIに食わせることではなく、選ぶことなのです。

![](https://assets.st-note.com/img/1783852254-avJfIgDN1KHRwTSqG6ApUuMP.png?width=1200)

ちなみにこの「何を文脈に載せ、何を載せないか」は、AIを組み込んだ製品を作る開発の世界では「コンテキストエンジニアリング」という設計分野になっていて、Anthropic が公式ガイドを出すほどの本丸の技術になっています。その話は、次の記事でじっくり書きました。

## まとめ

要点を言い切ってしまいます。

「AIには全部読ませたほうがいい」は間違っている。関連する情報は答えを賢くするが、無関係なトークンは信号を薄めるだけ。賢さと劣化は、同じ仕組みの表と裏。そして何を貼っても、AIは1ビットも学習しない。

最後に、今日からできる小さな行動提案を。次に資料を貼り付けようとしたら、送信の前に10秒だけ止まって、こう自問してみてください。

> この中で、答えに効くのはどのページだ?

そのページだけを渡す。全部渡すより、たぶんいい答えが返ってきます。

![](https://assets.st-note.com/img/1783852273-ZlJq2STo3e5ACj7HazvNwtiQ.png?width=1200)
