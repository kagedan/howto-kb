Claude Codeで実装が壊れまくる人の95%がやってる致命的なミスがあって、

それが、いきなり「実装して」って投げるやり方。

3段階に分けて、各段階で止めるだけで後戻りが激減する。順番はこう↓

具体的にこの3ステップ。
①「こういうツール作りたいんだけどどうやったら作れるかな。不明点は全部聞いて。コードはまだ書かないで」
→AIが詳細リサーチとブレインストーミングやってくれる。ここで方針が決まるから大きくぶれない。

②「ここまでに
　　　　　　　　　　　　　　　　　　　　　　　　　　　決まった内容を反映して仕様書を作成して。コードはまだ書かないで。」
→実装に入る前に仕様書ファイルを作成する。これやっておかないとAIは途中で細かいところ忘れちゃう。仕様書読むの面倒だけどここで意図と違う内容があったら修正しておく。ここをサボると後からもっと面倒になる。

③「仕様書に従って実装して」
→ここでようやく実装開始。いきなり実装すると大体イメージと全然違うツールができるけどこのステップを踏めば手戻りが激減する

これだけで5時間かけて全部やり直しが、30分で完成に変わる。
僕も最初は半分以上途中で「全部やり直して」って苦労が水の泡になってたけど、今はほとんどなくなった。

これCodexやCursor、ChatGPTでも全部同じ。
「いきなり実装させない」が共通ルール。
ブックマークして明日からの作業で実践してみて。
