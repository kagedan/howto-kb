## はじめに

「この不具合、@Claude 調べておいて」——Slackでメンションを飛ばすだけで、Claudeがチームの一員のように動いてくれる。そんな未来がすでに始まっています。

2026年6月にAnthropicが発表したと報じられている **Claude Tag** は、Slackのチャンネルに常駐し、`@Claude` とタグ付けするだけで仕事を任せられる「共有型のAI同僚」です。個人用のチャットボットではなく、**チーム全員で同じ1人のClaudeを共有する**というのが最大の特徴です。

この記事では、Claude Tagとは何か、Slackのコンテキストを読んで何ができるのか、ファイル操作やgit操作、そして独自MCPを繋いだ社内活用の未来像までを、公式情報をもとに整理します。

## Claude Tagとは

Claude Tagは、ひとことで言うと **「Slackに常駐する、組織共有のAIチームメイト」** です。ポイントは次のとおりです。

* **`@Claude` で呼び出すだけ**: チャンネルやスレッドで `@Claude` とメンションし、やってほしいことを書くと、Claudeがタスクを段階に分解して順に処理し、結果をスレッドに返してくれます。
* **共有アイデンティティ（マルチプレイヤーモデル）**: チャンネル内の全員が「同じ1人の `@Claude`」と会話します。誰かが始めた作業のコンテキストを、別のメンバーがそのまま引き継げます。個人ごとに別々のボットが立つのではありません。
* **管理者が設定する組織向けの仕組み**: 組織の管理者がアクセス範囲やツールを設定します。誰がどのチャンネルで、何にアクセスできるClaudeを使えるかをコントロールできます。

### 旧「Claude in Slack」からの進化

Claude Tagは、これまでの「Claude in Slack（Claude Code in Slack）」を、**組織単位で管理できる形に進化させたもの**です。

* 同じSlackアプリ上で動作するため、**再インストールは不要**（公式ドキュメントに記載）
* 既存のセットアップは移行期間中もそのまま動作する（同上）
* 従来のClaude in Slack体験がClaude Tag体験へ切り替わる時期については、**2026年8月3日が期限だと報じられています**（記事執筆時点の第三者情報のため、正確な日付は公式の案内をご確認ください）

移行の詳細は管理者向けに案内されているとされるため、既に旧統合を使っているチームは早めに公式情報を確認しておくと安心です。

## Slackのコンテキストを「読んで」何ができるのか

Claude Tagの強みは、**Slackの会話そのものを文脈として理解する**点にあります。わざわざ状況を説明し直さなくても、その場の議論を踏まえて動いてくれます。

* **スレッドから**: スレッド内で `@Claude` とメンションすると、そのスレッドの全メッセージを読み取り、会話の全体像を把握します。
* **チャンネルから**: チャンネルで直接メンションした場合は、最近のメッセージから関連するコンテキストを拾います。

このコンテキスト理解によって、Claudeは「何が問題なのか」「どのリポジトリを対象にすべきか」を判断し、適切なアプローチを選びます。

具体例を挙げると、次のような使い方が自然にできます。

* バグ報告のスレッドで「@Claude これ直しておいて」と伝えると、報告内容（再現手順やエラーログ）を文脈として調査・修正に着手する
* 仕様の議論スレッドで「@Claude この方針でドラフト書いて」と依頼すると、議論を踏まえた成果物を返す

## ファイルの書き込みやgit操作もできる

Claude Tagは「読む・返信する」だけではありません。コーディング系の依頼を検知すると、その裏で **Claude Code のセッション**を立ち上げ、実際の開発作業まで進めます。

* **コーディング意図の自動判定**: メンションの内容を分析し、コーディングタスクだと判断すると、通常のチャット応答ではなくClaude Codeのセッションにルーティングします。
* **ファイル操作とコード変更**: 接続済みのリポジトリに対して、コードの修正・リファクタ・機能実装といったファイル操作を行います。
* **git操作 / プルリクエスト作成**: 作業結果からそのまま **プルリクエストを作成**できます（現時点ではGitHubリポジトリが対象。1セッションにつき1つのPR）。
* **進捗の可視化**: 作業の進捗をSlackスレッドに随時投稿し、完了すると要約と「セッションを表示」「PRを作成」などのアクションボタンを添えて返してくれます。

つまり、**「Slackで話す → Claudeがコードを書く → PRができる」** という流れが、チャンネルから離れずに完結します。バグ調査、簡単な機能追加、レビュー指摘の反映などを、非同期で任せておける点が実務的です。

## 独自MCPを繋いで「社内サポート」に活かす未来像

Claude Tagが本当に面白くなるのは、**MCP（Model Context Protocol）で社内システムと繋いだとき**です。

MCPは、Anthropicが2024年に公開したAIと外部ツールを繋ぐ標準プロトコルで、「AI版のUSB-C」とも呼ばれます。Claude Tagでは、コネクタ（MCPサーバー）を追加することで、Claudeがアクセスできる情報源やツールを拡張できます。

* **公式コネクタ**: 報道されている情報では、Notion、Linear、GitLab、Sentry、Datadog、Stripe、Vercelなど、主要なエンタープライズサービス向けのコネクタが用意されているとされています（対応サービスは変動する可能性があるため、最新の一覧は公式情報をご確認ください）。
* **カスタム（独自）コネクタ**: 自社の社内システムやデータベースを、独自MCPサーバーとして接続できるとされています。

これにより、たとえば次のような「社内サポートのAI同僚」が構想できます。

* **社内ヘルプデスク**: 社内ナレッジベースやFAQをMCPで接続し、`@Claude` に質問すればチャンネル上で一次回答が返る
* **顧客サポートの下支え**: 社内CRMを繋ぎ、「顧客IDを伝えるだけで過去の問い合わせ・購入・対応状況を把握」して回答ドラフトを用意
* **運用監視**: 監視系コネクタ（Datadog / Sentry等）を繋ぎ、アラートのスレッドで原因の当たりをつけてもらう

さらに、報じられている情報では、管理者が **ambient（アンビエント）モード** を有効にすると、Claudeはタグ付けされていなくても接続チャンネルのメッセージを監視し、重要と判断したことをフラグしたり、停滞したスレッドをフォローアップしたりと、**先回りして動く**ようになるとされています。この挙動の詳細や名称は公式情報での確認をおすすめします。

## 組織での具体的な活用シナリオ

チーム全員が同じ `@Claude` を共有するからこそ活きる、代表的なシナリオを整理します。

| シナリオ | 使い方のイメージ |
| --- | --- |
| 社内ヘルプデスク | 総務・情シスのチャンネルに常駐させ、社内規定やFAQをMCPで接続。よくある質問に一次対応 |
| コードレビュー / バグ修正 | 開発チャンネルでバグ報告に対し `@Claude` が調査・修正し、PRまで作成 |
| ドキュメント生成 | 仕様議論のスレッドから、設計メモやドキュメントのドラフトを生成 |
| 顧客サポート支援 | 社内CRMを接続し、問い合わせ内容から過去履歴を踏まえた回答案を提示 |
| 運用・監視の初動 | アラートのスレッドで、関連ログやコネクタ情報をもとに一次切り分け |

いずれも「**その場の会話の文脈を、そのまま仕事に変換できる**」のがClaude Tagらしさです。

## セットアップ・導入イメージ

Claude Tagの導入は **管理者主導** です。報じられている情報をもとにすると、おおまかな流れは次のようになるとされています（正確で詳細な手順・画面は必ず公式ドキュメントを参照してください）。

1. **アイデンティティを作成**: 組織のオーナー / 管理者が、管理設定から組織共有の `@Claude` を用意する
2. **アクセスバンドルを付与**: そのClaudeがどのツール・コネクタ（MCP）にアクセスできるかを設定する
3. **チャンネルにスコープを設定**: どのチャンネルで使えるようにするかを絞り込む
4. **テストして運用開始**: 想定どおりに動くか確認し、チームへ展開する

利用者側は、Claudeを使いたいチャンネルに `/invite @Claude` で招待し、あとは `@Claude` とメンションして依頼するだけです。Claudeは招待されたチャンネルでのみ応答するため、**チャンネル単位でのアクセス制御**が自然に働きます。

## まとめ

Claude Tagは、Slackに常駐する **組織共有のAIチームメイト**です。要点を振り返ります。

* `@Claude` とメンションするだけで、タスクを分解して実行し、結果をスレッドに返す
* スレッドやチャンネルの**コンテキストを読み取り**、状況を踏まえて動く
* コーディング依頼では**ファイル操作やgit操作、PR作成**まで進められる
* **独自MCP / コネクタ**を繋げば、社内ヘルプデスクや顧客サポートなど幅広い業務に応用できる
* 導入は**管理者主導**で、チャンネルとツールのスコープ設計、ambientモードの可否が運用の鍵

「AIを個人のツールとして使う」段階から、「**AIをチームの一員として迎える**」段階へ。Claude Tagは、その象徴的な一歩と言えそうです。まずは信頼できるチャンネルで小さく試し、コネクタとアクセス範囲を少しずつ広げていくのがおすすめです。
