## はじめに

SNSでこんな質問を見かけました：

> 「デスクトップ版 Claude APP の Code、Remote が使えない。何度試してもまったく見当たらない」

返ってきた回答はこうでした：

* 「使えるよ、私は使ってる」
* 「Claude Code CLI じゃないと無理」
* 「Codex に乗り換えたら？」
* 「`/remote control` でいける」

全員が自分の経験から答えていて、誰も「あなたの今の状態」を確認していない。  
結果、質問者はどこにも辿り着けなかった。

この記事では、**デスクトップ版 Claude APP から スマホの Claude APP へ Remote 接続する手順**を、実際に確認した介面文字をもとに整理します。  
そして「試したのに見えない」の原因になりやすい **2つの落とし穴** も合わせて説明します。

---

## 対象読者

* Claude Desktop の Remote 機能を試したが、スマホ側で何も見えない
* 公式ドキュメントを読んだが、最初の一手がわからない
* Claude に聞いたが、抽象的な答えしか返ってこなかった

---

## 前提

* Claude Desktop（Mac / Windows）インストール済み
* スマホの Claude APP インストール済み
* 同じアカウントでログイン済み

---

## 手順

### Step 1：設定でリモートコントロールを有効にする

Claude Desktop を開き、**設定 → Claude Code** に進む。

以下のトグルをオンにする：

> **「デフォルトでリモートコントロールを有効にする」**
>
> 新しいローカルセッションをリモートコントロールに自動的に接続して、CLIまたはclaude.ai/codeから続けられるようにします。

⚠️ **落とし穴 1：このトグルがオフのままだと、CLIのみノードが生成される**

デスクトップ版 APP の Code タブではノードが生成されません。  
スマホ側には何も表示されません。  
「CLIなら使える」という回答が正しかったのは、この状態を指していた可能性があります。

---

### Step 2：Desktop で新規セッションを開き、対話を発生させる

上部タブから **Code** に切り替え、**「新規セッション」** をクリック。

プロジェクトのパスを指定する。

ここで多くの人が止まります。パスも指定した、セッションも開いた——でも左側のノード一覧は何も変わらない。**「あれ、壊れてる？」と思う、あの瞬間です。**

実はこの挙動は、Chat の対話リストと同じ論理です。  
**Chat も、何も話しかけていない状態ではリストに何も表示されません。話しかけて初めて、トピックが生まれる。**  
Remote ノードも同じです——**対話が発生して初めて、ノードが形成されます。**

#### まだ使えるプロジェクトがない場合（機能確認だけしたい）

一言送るだけで確認できます：

> 「これはRemoteのテストです」

送った瞬間、左側のサイドバーに **Remote ノードの一覧** が表示されます。  
スマホで見えることを確認したら、このセッションは閉じて構いません。

#### 実際のプロジェクトで使う場合（通常の使い方）

設定は初回のみです。2回目以降は：

1. Code タブ → 新規セッション → プロジェクトパスを指定
2. 作業内容をそのまま Claude に伝える（CLI と同じ感覚で）
3. その瞬間、左側にノードが生成される
4. スマホの Code タブを開けば、そのノードに接続できる

毎回「テストです」と言う必要はありません。**普通に作業を始めるだけで Remote は有効になっています。**

⚠️ **落とし穴 2：対話が発生しないとノードは形成されない**

デスクトップ版 APP では、セッションを開いただけではノードは生成されません。  
これは CLI 版とは異なる挙動です。  
トグルをオンにしていても、対話なしではスマホ側に何も表示されません。

---

### Step 3：スマホから接続する

スマホの Claude APP を開き、**Code タブ** に移動する。

先ほど Desktop で作成したセッションが一覧に表示されているはずです。  
タップすれば接続完了です。

以降は Desktop で新規セッションを作るたびに、スマホ側に自動で表示されます。

---

## 補足：初回のみ表示される案内について

初めてスマホの Code タブを開いたとき（リストが空の状態）、  
画面中央あたりに **「環境接続」** という案内が表示された記憶があります。

その後は一度も見ていないため、**初回のみ表示される onboarding 案内**である可能性が高いです。  
再現確認ができていないため、同じ画面が表示された方はコメントで教えていただけると助かります。

---

## まとめ：「試したのに見えない」の原因

| 状態 | 原因 | 対処 |
| --- | --- | --- |
| スマホに何も表示されない | トグルがオフ | 設定 → Claude Code → トグルをオン |
| トグルはオンだが表示されない | 対話が発生していない | セッション開始後、必ず一言送る |
| 初回のみ案内が表示された | onboarding 画面 | 案内に従って接続する |

順番が重要です：**設定（トグルオン）→ 新規セッション → 対話発生 → スマホで確認**。

---

## おわりに

この機能について SNS で見かける質問の多くは、「できる・できない」の二項対立で終わっていて、「なぜ見えないのか」の構造が共有されていません。

公式ドキュメントは機能の説明はしていますが、「最初の一手」と「なぜ失敗するのか」を丁寧に示してはいない。

**わかっていることを正確に、わからないことは正直に。**  
それだけで、同じところで詰まっている人の助けになると思い、書きました。
