Claude CodeにMCPをたくさん連携してる人で、「動作が不安定・出力が雑になる」って感じてる人、設計を逆方向に組んでる可能性があります。

私はずっとSKILL.md+CLIベースの構成にしていて、MCPは最小限しか入れてません。

その理由は、MCPが起動時にすべてのツール定義を一括ロードするプロトコル設計だからです。

実際、 MCP を入れすぎると「200k→70kにコンテキストが縮む」「JSONが76KB超で壊れる」という現象が報告されてたりするんですよ

実際にこれを設計時から避けてきて分かったのは、コンテキスト汚染を避ける設計こそが、出力品質の上限を決める、ということです。

考えてみれば当たり前のことですが、起動時に渡す定義が多いほど、AIが本題に使える残り容量は減る。

つまり、MCPを多く入れるほど強くなる、という発想は逆で、起動時に渡す定義を絞るほど精度が上がる、という話。

連携してるMCPのリストを書き出して「これが全部起動時にロードされている」と想像すると、整理する優先順位が見えてきます。
