【初心者向けClaude Codeの使い方】これ1本で全部わかる｜インストールからSkills・MCPを使った自動化までを丁寧に解説
キノコード / プログラミング学習チャンネル　#AI要約

Claude Codeで仕事を自動化する方法

🔳Claude Codeの概要
Claude Codeはチャットで答えるだけのAIではなく、パソコン内でファイル作成、整理、分析、保存まで実行できるエージェント型AIとして紹介された

🔳チャット型AIとの違い
ChatGPTやClaudeの通常チャットは手順や回答を返すヘルプデスク型であり、Claude Codeは隣の同僚のように実際の作業まで代行する存在だと説明された

🔳使い方の種類
Claude CodeにはCLI版、VS Code版、デスクトップアプリ版、ブラウザ版があり、ファイル操作やデータ分析など幅広い業務にはVS Code版が使いやすいとされた

🔳VS Code版の利点
VS Code版はAIが作成したファイルをその場で確認や編集でき、フォルダー構成や作業履歴も見やすいため、初心者が業務自動化を始める環境として勧められた

🔳初期設定の流れ
VS CodeにClaude Code拡張機能を入れてログインし、作業フォルダーを開くことで、右側のパネルからClaude Codeに指示を出せる状態になると説明された

🔳Claude MDの役割
Claude MDはAIへの自己紹介カードのようなもので、自分の役割、目標、出力ルールなどを書いておくことで、毎回説明しなくても好みや作業方針を理解してもらえるとされた

🔳Claude MDの書き方
最初から細かいルールを大量に書くのではなく、役割、目標、制約を短く書き、使いながら少しずつ育てる方がAIの判断力を活かしやすいと説明された

🔳許可モードの使い分け
確認モード、自動編集モード、計画モードがあり、最初はAIの編集前に確認できる確認モードで始め、慣れたら計画モードや自動編集モードに移るのが安全だとされた

🔳便利なコマンド
スラッシュモデルでAIモデルを切り替え、スラッシュコンパクトで長い会話を圧縮し、アカウント画面で使用量を確認することで、作業効率と利用制限を管理できると説明された

🔳スキルの仕組み
スキルはAIに特定業務の手順を教える業務マニュアルであり、議事録、レポート、メール返信などを毎回同じ品質と形式で作れるようにする機能だと紹介された

🔳スキルの呼び出し方
議事録を作ってと自然に依頼すると自動で該当スキルが使われ、確実に指定したい場合はスラッシュコマンドでスキル名を呼び出す方法があると説明された

🔳Skill MDの構造
Skill MDにはスキル名や説明を書くヘッダーと、具体的な作業手順を書く本文があり、AIは説明文を見て必要なスキルかどうかを判断するとされた

🔳スキルの保存場所
グローバルのスキルフォルダーに置けば全フォルダーで使え、作業フォルダー内に置けばそのプロジェクトだけで使えるため、個人用とチーム共有用を使い分けられると説明された

🔳Claude MDとスキルの違い
Claude MDは常に読まれる共通方針であり、スキルは必要な時だけ読まれる業務手順書なので、共通ルールは短く、個別作業の詳細はスキルに書くのがよいとされた

🔳MCPの役割
MCPはClaude Codeと外部サービスをつなぐ仕組みで、Googleカレンダー、Gmail、Canva、Slack、Notionなどと連携して、情報取得や下書き作成を任せられると説明された

🔳MCPの設定方法
Claudeの設定画面からコネクターを連携する方法と、設定ファイルに直接書く方法があり、難しい設定はClaude Codeに作成を手伝わせることもできるとされた

🔳MCPの注意点
外部サービスと連携するとAIがその情報へアクセスできるため、業務に必要なサービスだけを連携し、APIキーや機密情報を安易にAIへ渡さないことが重要だと注意された

🔳日常業務の自動化
スキル、MCP、MDファイル、自動実行を組み合わせることで、朝のタスク整理、メール返信、会議準備、スケジュール調整、プレゼン作成を自動化できると説明された

🔳集中時間の回復
短い雑務が何度も入ることで集中が分断される問題に対し、Claude Codeへ雑務を任せることで、まとまった作業時間を取り戻せると語られた

🔳AIチーム化の発想
Claude Codeでは部署ごとに役割を持たせ、PM部、マーケ部、エンジニア部、品質管理部、会計財務部のようなAIチームとして業務を分担させられると紹介された
