Fableは高くて使えないから関係ないと思ってるあなた。  
先にお伝えします。

今回の記事はFable期間が終わった後でも

**恒久的にあなたのAI事業が加速する  
"超"進化の設定です。**

プランにかかわらず  
Claude Codeをビジネスに使っている人は  
読んで損になることはない。

**というか、読まないと損しまくります。**

実際、私は  
Fableが再び公開された10日間。  
一日中Claude Codeをいじりまわしてました。

Fableでこんなアプリが！  
Fableでこんなサイトが！  
Fableでこんなゲームが！

そんな情報が飛び交う中、  
私自身は、そういったものは作りませんでした。

では何をしていたのか？

それは、徹底的に  
自分の事業プランの見直しと  
それに伴ったClaudeのアップデート。

その成果が、今のこの状態です。

## Claude Codeと一緒に作ってきたもの

* 🛠 自分専用のスキル（AIに定型作業を覚えさせる部品）＝**15個**
* 🤖 AIの分身（エージェント）＝**23体**
* 🌙 寝てる間に動く無人バッチ＝**7〜8種**
* 📱 動くアプリ＝**3個**（単語帳／学習ロードマップ／数字ダッシュボード）
* 📄 申込ページ（LP）＝**6本**
* ✍ ブログ／SEO記事＝**4本**（各1万字級）＋**10本以上を量産できる仕組み**
* 🖥 スライド＝セミナー・教材・提案あわせて**500枚超**
* 💬 集客用のLINE導線＝**4本**

ほとんどを、  
10日間で一気に作りました。

細かく数えてないけど、まあ、多いですよね。

この期間にツールや設定をガッと生み出したことで、

なーんだ、やっぱりFableがないとできないんじゃん。

**だれでも使える"超"設定**なんです。

## で、それで、何が変わったか

数字だけ見せても、「へー、すごいね」で終わっちゃうので。  
"実際に、何が勝手に回るようになったのか"を並べます。

* 🌙 **寝てる間に、AIが働いてる**：毎晩、下書き・リサーチ・検品を回してくれて、朝の私は○×をつけるだけ
* 📊 **クライアントの数字を、毎朝ひとりでに見せる**：提出→集計→本人だけが見えるページに、毎朝自動で反映
* 📨 **問い合わせが来たら、勝手に処理**：予約を検知→台帳とカレンダーに登録→返信の下書き作成
* 🧭 **散らばってた稼ぎ方が、1本道に**：AIに"詰まり"を畳ませて、止まってた判断を前に進めた
* 📚 **読んだ記事が、その日のうちに事業のノウハウに**：インプットを"型"で吸い取って、自分の仕事に落とす
* 🧩 **一度やった作業は、次から自分がやらない**：その場でスキルにして残す
* 🎬 **動画も、勝手に編集**：縦型・テロップ入りの完成品＋投稿文まで

気づいたら、作業がどんどん減って、  
手元には"判断"だけが残ってる。

この状態を、10日で作ったわけです。

同じClaude Codeを触ってても、  
半年後に"差"がつくのは、ここだと思ってます。

![](https://assets.st-note.com/img/1784224069-o9vm0RUedyJr8Ii3jFBVaKkf.png?width=1200)

作った物の量じゃなくて、  
"作れる形"にしたかどうか。

## 一番マネしやすいのをひとつだけ

これ全部の土台にある考え方、  
実はたった1つなんです。

**一番賢いAIに、全部やらせない。**

賢いモデルには「判断」だけ。  
手を動かす作業、リサーチや、  
決まったコマンドの実行、保存などは、  
枠の軽い、下のモデルに任せる。

これをAIが毎回読む設定ファイルに、1行入れておく。  
それだけで、高い頭脳が単純作業で  
すり減っていくのを止められます。

Claude Codeなら、  
プロジェクトの一番上にある「[CLAUDE.md](http://claude.md/)」に貼るだけ。  
（無ければ、作ればOK）

```
上位モデルは「判断」だけに使う。決まった作業（探索・定型コマンド・保存など）は下位モデルに任せる。ただし失敗の影響が大きい変更や、最終チェックは上位モデルに戻す。
```

これ1個だけでも、今日から効きます。

無料の記事から来てくれた人は、  
もう貼ってあると思います。

でも、正直に言いますね。  
これ、まだ"入口"なんです。

さっきの、スキル15個。  
あれがなんで、10日で15個まで増えたのか。

"本当のカラクリ"は、別にあるんです。

それが——  
やり終えた**"直後"に打つ、たった一文**。

ふつう、面倒な作業を1回やったら、それで終わりですよね。  
次に同じ場面が来たら、また一から手を動かす。

でも私は、作業が終わったその瞬間に、  
AIへ、ある一文を投げます。

呪文でもなんでもない、ふつうの日本語です。  
でも、これを打つか打たないかで、  
**1ヶ月後の"手の空き方"が、まるっきり変わる。**

打ったそばから、その作業が  
「次から自分がやらなくていい部品」に変わっていく。

やってるのは、これのくり返しだけ。  
仕組みが仕組みを生んで、10日で15個まで増えました。

---

で、ここから先。

私が"毎日そのまま打ってる型"を、ぜんぶ置いておきます。  
考え方の説明じゃなくて、**コピペして打つだけ**の形で。

* 🧩 さっきの**「作業を"部品"にする一文」**（これ1つで勝手に増えていく）
* ⚙ **CLAUDE.mdに貼る設定**（"判断は上・作業は下"の完全版）
* 🌙 **寝てる間に、仕事が半分終わってる状態**を作る無人ワーカー＋"事故らせない掟"
* 📊 **クライアントの数字を毎朝見せる**仕組み＋個人情報の穴つぶし
* 📨 問い合わせの自動さばき／📚 記事を即事業に／🎬 動画の自動編集
* ⚠ 私が溶かした**"ハマりどころ"リスト**（先に知れば、1日ぶん損しない）

ぜんぶ、あなたの環境でそのまま再現できます。

先に言っておくと、  
私はエンジニアじゃありません。  
コードも書けません。

やったのは、この"一文"を打つのを、  
10日くり返しただけです。

まずは、その"一文"から——

ここから先は、ほぼ"そのまま打てるプロンプト"です。

各仕組みは、この3点セットで置いてあります。

① それを作らせる一文  
② 中に入れる"安全の掟"  
③ 私が踏んだ地雷（ハマりどころ）

上から順番に貼っていけば、  
さっき見せた仕組みが、自分の環境で再現できます。

Claude Codeなど、AIコーディング環境を使っていることです。
