先月、Anthropicが無料の学習プラットフォーム「Anthropic Academy」を公開しているのを知りました。

![](https://assets.st-note.com/img/1773934657-nR1FlrUjMACgWmGZ2dTx9ahc.png?width=1200)

**無料で、13コースある。しかも修了証まで出る。**

「とりあえず全部受けてみよう」と思って受け始めたら、思っていたより本格的でした。動画・テキスト・クイズまで揃っていて、ちゃんと学べる内容になっている。

特にMCPのコースを受けてから、**freeeとの連携が劇的に変わりました。**「なんとなくつないでいた」状態から「仕組みを理解して使いこなせる」状態になった感覚があります。

今回は全13コースを受けた感想と、特に刺さったコースの体験談をまとめます。

## Anthropic Academyって何？

Anthropicが運営する**公式の学習プラットフォーム**です。Skilljarというシステムを使っていて、アカウントを作れば**全コース無料**で受けられます。

Claudeのアカウントは不要です。Skilljarのアカウントだけあれば始められます。

修了証はLinkedInに追加できる形式で発行されます。**Anthropic公式の証明書**なので、AI関連の仕事をしている方には**価値**があります。

「無料でここまでやってくれるんだ」と素直に驚きました😺

## 全13コースの全体マップ

13コースは大きく**3つのグループ**に分かれています。

### グループ1　AI活用の土台を作るコース（6本）

AIをどう使うか、という「考え方」を学ぶグループです。

![](https://assets.st-note.com/img/1773935342-JGSqWUg1k25DVuHrNwhXObde.png?width=1200)

**Claude 101**  
Claudeの基本操作と使い方を一通り学べる入門コースです。「プロンプトって何？」「どんなことができる？」という疑問がまとめて解消されます。すでにClaudeを使っている方でも、改めて体系的に整理されるので受ける価値があります。

**AI Fluency: Framework & Foundations**  
AIと上手に付き合うための**思考フレームワーク**を学ぶコースです。「4Dフレームワーク」という考え方を使って、AIに何を任せるか・どう指示するかを体系的に学べます。技術的な話は少なく、考え方の話がメインです。

**AI Fluency for educators・for students・for nonprofits・Teaching AI Fluency**  
特定の立場向けのコースです。教育者・学生・非営利団体向けにそれぞれカスタマイズされています。自分の立場に合うものを選んで受けるといいと思います。

### グループ2　技術・開発系コース（5本）

Claude Codeを使いこなしたり、APIやMCPを学んだりする**実践的なグループ**です。

![](https://assets.st-note.com/img/1773935438-jze7Mtb5qf83TCPvDu0Y1rRQ.png?width=1200)

**Claude Code in Action**  
Claude Codeの全体像を学ぶコースです。インストールからMCP連携・GitHub統合・HooksとSDKまで、体系的に学べます。

**Building with the Claude API**  
APIを使ってアプリを開発するコースです。**8時間以上のボリューム**があって、かなり本格的です。

**Introduction to Model Context Protocol**  
MCPの仕組みを基礎から学ぶコースです。このコースが今回のフリーランスの経理改革に**一番効きました**（後述します）。

**Model Context Protocol: Advanced Topics**  
MCPの上級編です。本番環境向けの実装パターンや、ファイルシステムへのアクセス方法などを学べます。

**Introduction to Agent Skills**  
Claude CodeのSkills機能（カスタムコマンドのこと）を体系的に学ぶコースです。

### グループ3　クラウド連携系コース（2本）

**Claude with Amazon Bedrock**  
**Claude with Google Cloud's Vertex AI**  
それぞれのクラウドサービスでClaudeを使うためのコースです。AWS・GCPを業務で使っている方向けの内容です。

## 特に刺さった5コース、正直な感想

### 1. Claude 101：「知ってるつもり」がリセットされた

正直、最初は「これは初心者向けでしょ」と思って**軽く流すつもり**でした。

でも受けてみたら、自分が「なんとなく使っていた」ことの意味が整理されました。プロンプトの書き方、Claudeが何を考えているか、どういう指示が伝わりやすいか。感覚でやっていたことが**言語化**されて、「だからこういう返し方になるんだ」という納得感がありました。

**「知ってるつもり」をリセットするのに最適なコース**です。まずここから受けるのをおすすめします**😸**

### 2. AI Fluency: Framework & Foundations：AIとの付き合い方が変わった

「**4Dフレームワーク**」という考え方が印象に残っています。AIに何かを頼むとき、Define（定義する）・Design（設計する）・Delegate（委任する）・Debrief（振り返る）という4つのステップで考えると、**指示の質が上がる**という内容です。

このフレームワークを知ってから、Claude Codeへの指示の出し方が変わりました。「なんとなく頼む」から「**何をどう頼むかを考えてから頼む**」になった。

技術的な話は一切出てこないので、コードが書けなくても理解できます。

### 3. Claude Code in Action：連載で学んでいたことの「なぜ」がわかった

この連載でClaude Codeを使ってきた方には**特におすすめしたいコース**です。

CLAUDE.mdやカスタムコマンドを「とりあえず使っていた」状態から、**「なぜこれが必要で、どういう仕組みで動いているか」**を理解できました。

特に驚いたのが**Hooksの説明**です。Hooksとは、Claude Codeが特定の行動をする前後に自動で処理を実行できる仕組みです。たとえば「ファイルを変更する前に必ずバックアップを取る」「コマンドを実行する前にログを記録する」といったことが自動化できます。

カスタムコマンドを作れるようになった後のステップとして、**ちょうどいいタイミング**で受けられました😹

### 4. Introduction to MCP：「なんとなくつないでいた」が終わった

このコースが今回**一番インパクトがありました。**

freee MCPを使ってはいたけど、「なぜこれで外のサービスとつながれるのか」を理解できていなかった。このコースで、MCPの3つの核心である**「ツール」「リソース」「プロンプト」**という概念を学んでから、一気に理解が深まりました。

![](https://assets.st-note.com/img/1773934851-eENYOcWuZrKn9vbjIU40qtSz.png?width=1200)

「ツール」はAIが実行できるアクション（freeeのデータを取得する、など）、「リソース」はAIが参照できるデータ、「プロンプト」は再利用できる指示のテンプレート。**この3つの組み合わせでMCPは動いています。**

仕組みを理解してから使うと、エラーが出たときに「なぜ動かないか」が自分で考えられるようになりました。**これが一番大きな変化です。**

### 5. Introduction to Agent Skills：カスタムコマンドが「型」になった

このコースを受ける前は、カスタムコマンドを「便利なショートカット」として使っていました。

受けてからわかったのは、**Skillsはもっと体系的に設計できる**ということです。「どのタスクに自動的に適用するか」「複数人で共有するにはどうするか」「トラブルシューティングの方法」まで学べます。

/keiri-todo や /x-post を作って使っていた経験が、このコースでより深く理解できました😺

## MCPコースを受けてfreee連携が劇変した話

MCPコースを受ける前と後で、freeeとの連携が**大きく変わりました。**

受ける前は「Claude Codeからfreeeに話しかけたら、なんかデータが返ってくる」という状態でした。動いているけど、**なぜ動いているかはよくわからない。**

受けた後は「freee MCPがどういうツールを持っていて、Claude Codeがそのツールをどう呼び出しているか」が見えるようになりました。

その結果、**できることが変わりました。**

以前は「今月の支出を見せて」という単純な取得しかしていませんでした。でも今は**複数のツールを組み合わせた複雑な分析**ができるようになっています。

たとえば /keiri-todo コマンドは、裏側でこういう処理をしています。まず今月の取引を全件取得する、次に過去3ヶ月の取引と比較する、次に固定費のパターンを分析する、最後に未計上のものをリストアップして優先度をつける。**この一連の流れが、MCPの「ツール」を連鎖的に呼び出すことで実現しています。**

「仕組みを知っている」と「仕組みを知らない」では、トラブルが起きたときの対応が全然違います。MCPコースを受けてから、エラーが出たときに自分でデバッグできるようになりました。

「なんとなく動いている」から「理解して使いこなしている」への変化は、思っていたより大きかったです😺

## 受ける順番のおすすめ

13コースを全部受けた経験から、**おすすめの順番**をまとめます。

Claude Codeをメインで使っている方（この連載の読者の多くはこちらだと思います）はこの順番がおすすめです。

まず**Claude 101**で基礎を固めます。次に**AI Fluency: Framework & Foundations**で考え方を学びます。そのあと**Claude Code in Action**で実践的な使い方を学びます。**Introduction to MCP**でMCPの仕組みを理解します。最後に**Introduction to Agent Skills**でカスタムコマンドを深めます。

APIやクラウド開発に興味がある方は**Building with the Claude API**や**Amazon Bedrock・Vertex AI**のコースも受けてみてください。

自分の立場に合うAI Fluencyのコースは、どのタイミングで受けても大丈夫です。

## まずここから始めよう

「全部受けるのはハードルが高い」という方は、まず**1コースだけ**受けてみてください。

Claude Codeをすでに使っている方には**「Introduction to MCP」**から始めることをおすすめします。今まで「なんとなく」使っていたMCPの仕組みが一気に整理されます。1時間程度で受けられます。

まだClaude Codeを使ったことがない方には**「Claude 101」**から始めるのがおすすめです。

修了証は全コース共通で、受講後のクイズに合格すると発行されます。LinkedInに追加できる形式なので、受けたら損はないです。

13コース全部受けるのに合計で**10時間くらい**かかりました。でも「**Claude Codeが10倍強くなった**」という感覚は本当にあります。

具体的には3つ変わりました。できる作業の幅が広がった、エラーが出たときに自分で対処できるようになった、指示の質が上がって一発で意図通りの結果が返ってくるようになった。どれも「仕組みを理解したから」起きた変化です。

特に**MCPコース**はおすすめです。ぜひ試してみてください😸
