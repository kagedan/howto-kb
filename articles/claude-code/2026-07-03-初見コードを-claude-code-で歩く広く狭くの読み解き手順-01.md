知らないリポジトリに放り込まれた初日。`README` を読んでも、どのファイルから手を付ければいいか分からない——この「歩き出しの迷い」を、Claude Code に道案内させて短くします。

この記事では、初見コードを**広く→狭く**の順で読み解く手順を、実際のプロンプト・`@` 参照・確認の流れつきで組み立てます。ポイントは「全部読ませる」ではなく、**読ませる範囲を制御し、触らせず、理解だけを持ち帰る**こと。最後に、得た理解を `CLAUDE.md` に固定して次セッションへ残すところまで通します。

!

**対象読者と前提**

* Claude Code をリポジトリのルートで起動できるエンジニア
* 中〜大規模の「初見コード」を担当することになった人
* AI の応答（要約・トレース結果）はすべて「想定例」です。実際の出力はリポジトリで変わります

## 結論：読み方には順番がある（広く→狭く→流れ）

初見コードをいきなり「このバグ直して」から始めると、AI も人も迷子になります。読む順番を固定します。

読み解きの3段

```
1. 広く … リポジトリ全体の地図（構成・アーキ・主要モデル）
2. 狭く … 担当機能に関係するファイル群へ絞る
3. 流れ … 1つの処理を入口から出口までトレースする
```

各段で「何を聞くか」が決まっていれば、初見でも30分で当たりがつきます。順に見ていきます。

## 0. 歩き始める前に：clone して起動するだけ

題材は手元の任意のリポジトリで構いません。ルートで `claude` を起動します。

```
$ git clone https://github.com/your-org/your-app.git
$ cd your-app
$ claude
```

この時点ではまだ何も聞きません。まず「読むだけ」を保証する設定（後述の plan モード）に切り替えてから歩き出します。

## 1. 広く：リポジトリの地図を描かせる

最初は粒度の大きい問いから。狭い質問をいきなり投げず、**全体像 → 個別**へ降りていくのがコツです。

プロンプト（広く）

```
give me an overview of this codebase
```

返ってくる概観はこんな粒度です（想定例）。

想定例（リポジトリにより変わる）

```
このリポジトリは Flask 製の REST API です。
- エントリポイント: app/main.py
- ルーティング: app/api/ 配下（users, orders）
- データモデル: app/models/（SQLAlchemy）
- 設定: app/config.py（環境変数で上書き）
- テスト: tests/（pytest）
```

地図が出たら、気になった軸を1つずつ掘ります。広い問いを起点に、アーキ・データモデル・横断機能へ降りていきます。

プロンプト（掘り下げの軸を変えていく）

```
explain the main architecture patterns used here
what are the key data models?
how is authentication handled?
```

## 2. 狭く：`@` 参照で読ませる範囲を固定する

担当機能が見えてきたら、AI に探させるのではなく、**こちらが範囲を指定**します。`@` でファイルやディレクトリを直接コンテキストに載せられます。

プロンプト（@ でファイル/ディレクトリを指定）

```
@app/api/orders.py の処理の流れを説明して
@app/models のデータ構造を要約して
```

* `@app/api/orders.py` … そのファイルの**全文**をコンテキストに含めます。
* `@app/models` … ディレクトリの**ファイル一覧**（中身ではなく構成）を渡します。

「関係ファイルを探す」段でも、ドメイン語で具体的に頼むと精度が上がります。

プロンプト（関係ファイルを探す → 関係を掴む）

```
find the files that handle order checkout
how do these checkout files work together?
```

## 3. 流れ：1つの処理を入口から出口までトレースする

機能の地図ができたら、**1本の処理を端から端まで**追わせます。「フロントの入口 → DB まで」のように、経路を明示するのがコツです。

プロンプト（トレース）

```
trace the checkout process from the API endpoint to the database
```

返答は経路の連鎖で出ます（想定例）。

想定例（トレース結果）

```
1. POST /orders/checkout  → app/api/orders.py: checkout()
2. checkout() が OrderService.place_order() を呼ぶ  → app/services/order.py
3. place_order() が在庫を確認  → app/services/inventory.py: reserve()
4. Order モデルを生成し commit  → app/models/order.py
5. 確認メールをキューに投入  → app/tasks/email.py（非同期）
```

経路が出たら、各ステップを `@` で開いて**自分の目で確認**します。AI のトレースは「読む順の地図」であって、正しさの保証ではありません。地図に沿って実ファイルを開けば、確認は速く済みます。

## 4. 触らせない：plan モードで「読むだけ」を担保する

初見コードの読み解き中に、AI が気を利かせて編集を始めると困ります。**plan モード**にすると、Claude はファイルを読んで計画を提案するだけで、**承認するまでディスクに触れません**。

起動時から plan モードにする

```
$ claude --permission-mode plan
```

セッションの途中で切り替えるなら **`Shift+Tab`** でトグルできます。読み解きフェーズは plan、直す段になって初めて編集を許可、と分けると事故が減ります。

## 5. 文脈を汚さない：探索はサブエージェントに委譲する

大きなリポジトリを読ませると、ファイル読み込みで**メインの会話のコンテキストが埋まり**ます。探索は別の文脈で走らせて、**結論だけ持ち帰る**のが効きます。

プロンプト（サブエージェントに委譲）

```
use a subagent to investigate how our auth system handles token refresh
```

サブエージェントは自分のコンテキストでファイルを読み、要約だけを返します。メインの会話は要点で埋まり、長い読み解きでも息切れしません。

なぜ委譲すると楽になるのか（補足）

初見コードの調査は「読む量」が多く、関連ファイルを次々開くほどメインの会話履歴が膨らみます。膨らんだ履歴は後半の質問の精度に響きます。サブエージェント委譲は、調査の「読み込み」を隔離し、メインには**結論（どのファイルで・どう実装されているか）だけ**を戻す分業です。複数の論点を並行調査させたいときにも向きます。

## 6. 理解を残す：`/init` で CLAUDE.md に固定する

歩いて分かったこと（ビルド手順・配置規約・主要モデル）は、セッションが終われば**揮発します**。次の自分とチームのために `CLAUDE.md` へ残します。`/init` はコードベースを分析してたたき台を作ります。

Claude Code セッション内（想定例）

```
> /init

Analyzing codebase...
Created CLAUDE.md with build commands, project structure,
and detected conventions. Review and refine.
```

生成された CLAUDE.md に、読み解きで掴んだ「地図」を**自分の言葉で追記**します。

CLAUDE.md

```
  # プロジェクト概要
  - Flask 製 REST API（app/main.py がエントリポイント）
+
+ ## 主要な処理の地図（読み解きメモ）
+ - 注文確定: api/orders.py → services/order.py → models/order.py
+ - 在庫確保は services/inventory.py: reserve() が担当
+ - 確認メールは tasks/email.py で非同期（キュー投入）
```

これで次セッションは、地図が**最初からコンテキストに載った**状態で始められます。CLAUDE.md の置き場所・読み込み順・分割設計そのものは、別記事「CLAUDE.md を読み込み順から設計する」で詳しく扱っています。

## まとめ

* 初見コードは\*\*広く（概観）→ 狭く（@で範囲指定）→ 流れ（トレース）\*\*の順で歩く
* 概観は補完が混じり得る。`@` 参照で**実ファイルに足を着けて**裏を取る
* **plan モード**（`--permission-mode plan` / `Shift+Tab`）で読み解き中は触らせない
* 長い調査は**サブエージェントに委譲**してメインの文脈を汚さない
* 掴んだ地図は **`/init` ＋追記**で CLAUDE.md に残し、次セッションへ引き継ぐ
