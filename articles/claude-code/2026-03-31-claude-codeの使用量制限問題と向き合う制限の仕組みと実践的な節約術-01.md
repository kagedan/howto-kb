Claude Codeの使用量制限について、Xで不満の声が爆発しています。

Anthropicのエンジニア Lydia Hallie氏が制限について言及したポストには大量のリプライが付き、「$20払ってるのに数時間で使えなくなる」という声が後を絶ちません。

この記事では、制限の仕組みを正しく理解した上で、半年間の運用で見つけた実践的な節約術をまとめます。

## 制限はメッセージ数じゃなくトークン数

まず最も大事なポイント。Claude Codeの使用量制限は「何回やり取りしたか」ではなく「合計で何トークン消費したか」で決まります。

つまり、1回のやり取りでも巨大なファイルを読み込めば一気に消費するし、短いやり取りを繰り返せば長持ちします。この違いを理解しているかどうかで、1日の使える量が体感で2〜3倍変わります。

## 実践的な節約術

### 1. モデルルーティングを活用する

すべてのタスクにSonnetを使う必要はありません。Claude Codeでは `/model` コマンドでモデルを切り替えられます。

単純なファイル操作やコード整形にはHaiku 4.5を使い、複雑な設計判断やデバッグにはSonnetを使う。これだけで消費トークンが大幅に減ります。Haikuの性能はSonnetの9割程度ありながら、コストは3分の1です。

### 2. /compact のタイミングを見極める

`/compact` はコンテキストウィンドウを圧縮するコマンドです。ただし、使うタイミングを間違えると逆効果になります。

おすすめは「一つの作業が完了した直後」。たとえば、機能実装が終わってコミットした後、次のタスクに移る前。作業途中で圧縮すると、直前の文脈が失われて同じ作業をやり直すことになります。

### 3. バッチ操作でまとめる

「このファイルを読んで」「次にこのファイルも読んで」「じゃあこの2つを比較して」と3回に分けるより、「AとBのファイルを比較して違いをまとめて」と1回で指示する方がトークン効率がいい。往復のオーバーヘッドが減るためです。

### 4. 不要なファイルを読ませない

Claude Codeは賢いので、関係ありそうなファイルを自動で読みにいきます。これが大量のトークンを消費する原因になることがあります。

`.claudeignore` ファイルで、node\_modules、ビルド成果物、ログファイルなどを除外しておくのが効果的です。

### 5. CLAUDE.mdを最適化する

プロジェクトのCLAUDE.mdが長すぎると、毎回のやり取りで大量のトークンを消費します。本当に必要な情報だけに絞って、詳細はサブファイルに分割するのがおすすめです。

## $20プランの現実

正直に言うと、$20のProプランでガッツリ開発するのは厳しいです。集中して使えば2〜3時間で制限に到達します。

ただし、上記の節約術を全部やると話が変わります。僕の場合、最初は2時間で制限に引っかかっていたのが、半年かけて最適化した結果、1日通して使えるようになりました。

具体的に効果が大きかったのは、モデルルーティングと /compact のタイミング最適化の2つ。この2つだけで体感3倍は持ちが変わります。

## Maxプランという選択肢

月額$100のMaxプランは制限が5倍です。仕事で毎日使うなら、正直こっちの方がストレスフリーです。1日$3ちょっとで制限を気にせず開発できるなら、十分元が取れます。

## まとめ

使用量制限は確かにストレスです。でも「仕組みを理解して、使い方を最適化する」ことで、同じ予算でも体験は大きく変わります。

制限がなくなることは当分ないでしょう。だからこそ、賢く使う方法を知っておくことが大事です。
