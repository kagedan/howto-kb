ClaudeやCursorと連携する「MCP」という仕組みに、深刻な構造的リスクが発覚しました。
（先日発覚した、知っておくべき脆弱性の話です）

Claudeの開発元のAnthropicは「修正しない」と公式表明。

自分が対象かどうかの確認方法と、今すぐできる対策を解説します🧵

まず「自分は対象？」を確認してください

対象：以下のツールでMCPサーバーを設定している人
　　 ・Claude Desktop（デスクトップアプリ）
　　 ・Claude Code（ターミナル版）
　　 ・Cursor / Windsurf など

「MCPって何？」という人は、ほぼ今すぐの対応は不要です。

MCPって何？という人向けに一言で言うと

「AIがあなたのPCやファイル・外部サービスに触れるための接続口」です。

ClaudeにGitHubやNotion、ファイルシステムを読ませたい時に使う仕組み、それがMCPです。

何が起きたのか

セキュリティ企業OX Securityが発見した問題は「欠陥」ではなく、設計そのものに起因するリスクです。

悪意あるMCPサーバーを経由すると：
・ファイルを盗まれる
・APIキーを奪われる
・チャット履歴を覗かれる
可能性があります。

「設計通りなのに危険」ってどういうこと？

例えるなら…

鍵業者に「合鍵を作れる設備」を渡したとき、その業者が悪人だったら家に入り放題になる。

「設備を渡したのはあなた」なので業者（Anthropic）は「仕様通り」と言える。でも実害は起きる。

実際の攻撃パターンは2つ

①「悪意あるMCPサーバーをインストールしてしまう」
→ 非公式のMCPを試した・勧められて入れた場合
②「AIが読んだ悪意ある文章に仕込まれた命令が実行される」
→ prompt injectionと呼ばれる手法

「気づかずに踏む地雷」に近いイメージです。

今すぐできる4つの対策

① 使っていないMCPサーバーを削除する
→ ClaudeやCursorの設定を開いて確認
② 出所不明のMCPをオフにする
→ 公式ドキュメントに掲載されているもの以外は一旦停止
③ AIの会話・設定ファイルにAPIキーを直書きしない
→ すでに書いていたら今すぐ再発行を
④ 信頼できないファイルやURLをMCP経由でAIに読ませない
→ prompt injectionの入口になります

影響範囲について正確に言うと
「1.5億ダウンロードに影響」と報道されていますが、これはSDKの累計ダウンロード数。

実際に攻撃可能な状態のサーバーは最大20万台と推定されています。

ただし、1点が突破されると連鎖するサプライチェーンリスクは本物です。

まとめ

✅ https://t.co/RdoDKsFiUSをブラウザで使うだけなら今すぐの対応は不要
✅ MCPを自分で設定している人は設定を見直して
✅ 出所不明のMCPは削除・停止
✅ APIキーをAIに渡していたら再発行

「AIは便利に使えばいい」だけの時代は終わりつつあります。

最後に一言

「自分はエンジニアじゃないから関係ない」と思っていた人も、ノーコードツールやCursorを使い始めた瞬間に対象になります。

知っているだけで防げるリスクです。
