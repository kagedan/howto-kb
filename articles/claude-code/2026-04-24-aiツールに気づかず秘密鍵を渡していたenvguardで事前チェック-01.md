## はじめに

Claude Code や Cursor を日常的に使っている方に、一度確認してほしいことがあります。

AIツールを起動する前に、こんな設定が `~/.zshrc` に残っていませんか。

```
export AWS_SECRET_ACCESS_KEY="AKIA..."
export GITHUB_TOKEN="ghp_..."
export OPENAI_API_KEY="sk-..."
```

もしそうなら、AIツールを起動するたびにこれらの値も一緒に渡しています。意図せず、毎回。

この記事では、なぜそうなるのか・どこに秘密情報が潜んでいるかを説明し、一括でチェックできるツール **[envguard](https://github.com/winky/envguard)** を紹介します。

## TL;DR

* AIコーディングツールはターミナルの**環境変数をそのまま引き継ぐ**
* `~/.zshrc` だけでなく launchctl・LaunchAgents・direnv など複数の経路がある
* `envguard` を実行すると、AIツールに渡っている可能性のある秘密情報を一括スキャンできる
* Read-only で動作し、値はマスキングして表示される（`AKIA****...****WXYZ` のように）
* 見つかったら `env -i` で隔離起動するか、aws-vault などへの移行を検討する

---

## なぜ起きるのか

AIコーディングツールは、ターミナルから起動する**ひとつのプロセス**です。Unix の仕組みとして、子プロセスは親プロセスの環境変数をそのまま受け継ぎます。

```
あなたのシェル（~/.zshrc を読み込んだ状態）
  └─ claude（子プロセス）
       └─ AWS_SECRET_ACCESS_KEY, GITHUB_TOKEN, ... が見える
```

AIツール自体が悪意を持って秘密鍵を盗む、という話ではありません。問題になるのは次のようなケースです。

* **プロンプトインジェクション**: 悪意あるコードやドキュメントを読み込んだとき、AIが環境変数の内容を外部に送信するコードを生成・実行する
* **MCP サーバー経由の漏洩**: 拡張機能やサードパーティの MCP サーバーが環境変数にアクセスできる状態になる
* **AIが生成したデバッグコード**: `console.log(process.env)` のようなコードがレビューをすり抜ける

「今のところ何も起きていない」は「リスクがない」とは異なります。特に、AIツールの利用が広がりMCPエコシステムが拡大している今は、一度立ち止まって確認する価値があります。

## 秘密情報が潜んでいる場所

環境変数は `~/.zshrc` に書かれているものだけではありません。macOS を使っている場合、気づきにくい場所に認証情報が設定されていることがあります。

| 場所 | 具体例 |
| --- | --- |
| シェル設定ファイル | `~/.zshrc`, `~/.bashrc`, `~/.profile` |
| クレデンシャルファイル | `~/.aws/credentials`, `~/.kube/config`, `~/.docker/config.json` |
| launchctl | macOS のユーザーセッション全体に設定された環境変数 |
| LaunchAgents | `~/Library/LaunchAgents/*.plist` 内の環境変数定義 |
| direnv | プロジェクトごとの `.envrc` ファイル |

特に **launchctl** 経由のものは見落としやすいです。GUI アプリのインストール時に設定されることがあり、自分では意識していなくてもターミナル全体に引き継がれます。

## envguard で確認する

これらを一括スキャンするために、**[envguard](https://github.com/winky/envguard)** を作りました。

動作の特徴は 3 つです。

* **Read-only**: ファイルの変更・削除・ネットワーク通信は一切しません
* **マスキング表示**: 発見した値は `AKIA****...****WXYZ` のように伏せて表示されます
* **リスク分類**: 変数名のパターンをもとに CRITICAL / HIGH / MEDIUM / LOW / INFO の 5 段階で評価します

### インストール

Go がある場合はこのコマンドで入ります。

```
go install github.com/winky/envguard@latest
```

バイナリをダウンロードしたい場合は以下を使ってください（macOS arm64 の例）。

```
curl -L https://github.com/winky/envguard/releases/latest/download/envguard-darwin-arm64 -o envguard
chmod +x envguard
```

### 実行してみる

引数なしで実行すると、6 つのスキャナーが順に走ります。

出力はこのようになります。リスクレベル・変数名・マスクされた値が並びます。

```
[CRITICAL] env    AWS_SECRET_ACCESS_KEY  AKIA****...****WXYZ
[HIGH]     env    GITHUB_TOKEN           ghp_****...****1234
[HIGH]     env    ANTHROPIC_API_KEY      sk-ant-****...****
[MEDIUM]   env    AWS_ACCESS_KEY_ID      AKIA****...****
[INFO]     file   ~/.aws/credentials     exists
```

CRITICAL や HIGH がいくつも並ぶようであれば、後述の対処を検討してください。

### よく使うオプション

ノイズを減らして重要なものだけ確認したいときは `--min-risk` が便利です。

```
# HIGH 以上のみ表示
envguard --min-risk high

# 件数だけ確認したいとき
envguard --summary

# 環境変数スキャナーだけ実行
envguard --only env
```

## リスクレベルの判定基準

envguard は変数名のパターンマッチングでリスクを自動判定します。

| レベル | 対象の例 |
| --- | --- |
| **CRITICAL** | `AWS_SECRET_ACCESS_KEY`, `*_PASSWORD`, `*_PRIVATE_KEY`, `DATABASE_URL` |
| **HIGH** | `GITHUB_TOKEN`, `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `*_API_KEY`, `*_TOKEN` |
| **MEDIUM** | `AWS_ACCESS_KEY_ID`, `GOOGLE_APPLICATION_CREDENTIALS`, `KUBECONFIG`, `TF_VAR_*` |
| **LOW** | `AWS_REGION`, `AWS_DEFAULT_REGION` |
| **INFO** | `~/.aws/credentials` などの認証情報ファイルの存在 |

`TF_VAR_*` 変数には特別な処理が入っています。たとえば `TF_VAR_DB_PASSWORD` は、サフィックス `DB_PASSWORD` を再評価して CRITICAL に引き上げられます。Terraform 経由で渡しているつもりでも、変数自体は環境に存在していることに注意が必要です。

## 見つかったときの対処

### 【今すぐできる】AIツールを隔離して起動する

`env -i` を使うと、最小限の環境変数だけを渡してツールを起動できます。

```
# HOME・PATH・SHELL だけ引き継いで claude を起動
env -i HOME="$HOME" PATH="$PATH" SHELL="$SHELL" claude
```

direnv を使っている場合は、プロジェクトの `.envrc` で unset する方法もあります。

```
# ~/work/my-project/.envrc
unset AWS_SECRET_ACCESS_KEY GITHUB_TOKEN
```

### 【中長期】常設の環境変数をやめる

根本的な解決は「常に環境変数にセットしておく」運用から「必要なときだけ注入する」運用へ移行することです。

| 現状 | 代替手段 |
| --- | --- |
| `~/.zshrc` に直書き | [aws-vault](https://github.com/99designs/aws-vault) でキーチェーン管理 |
| 静的なAPIキーを常設 | [1Password CLI](https://developer.1password.com/docs/cli/) でセッション都度取得 |
| launchctl に設定済み | `launchctl unsetenv 変数名` で削除 |

## まとめ

AIコーディングツールの利便性を活かしながら、秘密情報が意図せず渡っている状況を放置するのはリスクです。

`envguard` は一度実行するだけで現状を把握できます。結果がゼロなら安心して使い続けられますし、何か見つかれば対処のきっかけになります。

まず一度、試してみてください。

```
go install github.com/winky/envguard@latest && envguard
```
