📝Claude Codeのスケジュール機能が「ルーチン機能」に進化。これかなり使い勝手の幅が広がった

「プロンプト + リポジトリ + コネクタ」を一度設定するだけで、スケジュール・API・GitHubイベントの3つのトリガーで自動実行できる

しかもクラウドで動くから
PCを閉じてても、寝てても、勝手に動き続ける

スケジュールされたルーチンを使うと、Claudeに定期ルーチンを与えてそのまま放置可能。
例えばClaudeに毎晩午前2時にLinearからトップのバグを引っ張ってきて、修正を試みて、ドラフトPRを開くように指示なんかもできる

本日、全有料プランで利用可能、ウェブ上でClaude Codeを有効にした状態で使えます。

スレッドで深掘りします↓🧵

📝海外でバズってた「スケジュール機能で5時間枠を固定する」という裏技。

これやれば1日の使用量をムダなく使い切れる

✅️流れはこんな感じ　↓
1. スケジュール機能で「Hi」と自動送信させる
2. 5時間の制限枠が毎日同じ時間に固定される
3. いつ制限がかかるか読めないストレスが消える
4. 1日のトークン量を計画的に使い切れる

ただし、海外の投稿をそのまま真似すると失敗する。

Claudeの時間はUTC（協定世界時）基準だから。

日本時間（JST）で朝8時・昼13時・夕方18時・夜23時に枠を固定したいなら、9時間引いたこの設定を使う。

「0 4,9,14,23 * * *」

これをスケジュール機能のカスタムcronに入れるだけ。

ツールに振り回されるんじゃなくて、ツールの仕様を理解して自分のペースに引き込む。

こういう小さな仕組み化が、日々の生産性を地味に底上げしてくれる。
