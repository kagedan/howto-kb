2026年5月13日、VSCode v1.120がリリースされました。今回の目玉は**Agent Window**。。個人的に、この機能がVSCodeに提供されるのを待っていたので、同じような方もいるかもと思い、記事にまとめました。

VSCodeでAIエージェントを使う場合、これまではメインのエディターに統合された形でした。単一プロジェクト、単一タスクの管理には十分でしたが、複数プロジェクトを同時進行させたい、複数の並行タスクを含む複雑なタスクをエージェントに任せたい、みたいなことになると途端に使いづらくなっていたんですよね。Agent Windowはそこのフラストレーションを解決する設計になってます。

## 使い始めるには

### 最小限の準備

* VS Code v1.120 以上
* GitHub Copilot のサブスクリプション（CLI/Cloud使う場合）
* GitHub アカウント

### Agent Windowを開く

**UIから：**  
VSCode のタイトルバーにある六角形のボタンをクリック  
![](https://static.zenn.studio/user-upload/042dc069d169-20260518.png)  
（マウスカーソルをあわせると「エージェントで開く」という文言が表示されます）

**コマンドパレットから：**  
Cmd+Shift+P → 「Chat: Open Agents Window」

**ターミナルから：**

### Agent Windowの全体像

VSCodeのメインエディターとは独立した専用ウィンドウとして動作します。

### 画面構成

![](https://static.zenn.studio/user-upload/d58420a71fbe-20260518.png)  
**① 左側：セッション一覧**  
すべてのセッションがプロジェクトごとにまとまって表示されます。リネーム、完了マーク、削除とか右クリックで実行できます。

**② 中央：チャット**  
エージェントとのやり取りがここ。「こういう機能を作ってほしい」と送ったら、エージェントが「ステップ1：プロジェクト構造を確認します」みたいに進捗を教えてくれます。チャットの内容はセッションごとに保存されるので、いつでも見返せます。

**③ 右側：変更パネル**  
エージェントが加えたファイル変更がズラッと並びます。各ファイルのdiffを見ることもできるし、「こここう変えてほしい」ってコメント入れることもできます。その後エージェントが「了解、修正します」で自動実装。

**④ 左下：カスタマイズパネル**

以下の要素をカスタマイズして、エージェントの動作を調整できます。

* エージェント
* スキル
* 指示(instructionのことです)
* フック
* MCPサーバー
* プラグイン

### セッション開始の手順

1. セッション一覧パネルで「新規`⌘N`」ボタンクリック
2. フォルダ or リポジトリ選択
3. エージェント選択（Copilot CLI / Claude / Copilot Cloud）
4. 必要に応じて隔離モード選択（Git worktree or フォルダ）
5. 指示をテキスト入力して Enter

あとはエージェントに任せる。チャットで進捗確認したり、指示修正したり。

## 大きなメリット：3つの視点から

### 1. **ユーザー入力要求の通知機能**

エージェントが判断に迷ったり、追加情報が必要な場合、ユーザーの入力を要求します。このとき、OS通知を送信します。例えば**Macだったら通知センター（画面右上）に通知をとばしてくれます**。

```
【具体例】
エージェント: 「複数のテスト方針が考えられます。どちらを採用しますか？」
↓
Mac右上に通知が表示
↓
通知をクリックしてAgent Windowに戻る
↓
選択肢を選んでエージェント続行
```

**従来との比較：**

* 従来：エージェントの応答待ちに気づくまでセッションを切り替えて確認する必要がある
* 新機能：ユーザー応答が必要な瞬間に自動通知される（見落とし防止）

この仕組みは長時間実行タスクの管理に非常に有効です。エージェントが別プロジェクトで動作していても、入力が必要なら即座に通知されるため、待ち時間を有効活用できます。

### 2. **セッションの状態管理が明確**

これまでだとエージェントのチャット履歴がどんどん蓄積されて、「あ、これ完了した？」「この機能ってどこまで実装された？」みたいなのが曖昧でした。

Agent Windowではセッションに状態を付けられます。

```
作成
  ↓
進行中 ← エージェント実行中
  ↓
完了 ← コミット/マージして完了
  ↓
削除 ← 本当に不要なら削除
```

セッション一覧を見ると「あ、これはまだ途中、これは完了した」が一目瞭然。セッションの作業目的が達成できたら右クリックで「完了」マークにするだけ。完了したセッションは別セクションにまとめられるから、進行中のタスクに集中できます。

### 3. **デスクトップAIアプリの便利さをVSCodeで使える**

Claude Code(デスクトップ版)みたいなスタンドアロンのAIアプリでは、複数プロジェクト同時管理、セッション追跡、ユーザー入力要求の通知などは既に実現されていました。なので、VSCodeユーザーとしては、同じ機能がVSCodeでも使えるようになるのを待望していました。そして今や**Agent Window**により、VSCodeの豊富な拡張機能、リモート開発、デバッガ、統合ターミナル…などなどVSCodeが元々持ってる力をそのまま使いながら、エージェント管理もできるようになったわけです。

## その他細かいメリット

### ワークツリーで各タスクを隔離

Copilot CLIだと、[Git worktree](https://git-scm.com/docs/git-worktree)を使った隔離が可能。メインブランチに影響なく、複数タスクを並行実装できます。各タスクの変更は明確に分かれるから、マージもシンプル。また、セッションを「完了」にすると、自動でworktreeを削除してくれるのも地味に便利。（これまで自分で`git worktree`の手動管理をしていたときは、完了したタスクのworktreeを手動で削除しなきゃいけないのが面倒だった…）

### 統合ターミナルと統合ブラウザ

ターミナルもブラウザも、セッション内で動かせます。`npm install` したり `pytest` 実行したり、`localhost:3000` を開いて動作確認。すべてAgent Window内で完結。
