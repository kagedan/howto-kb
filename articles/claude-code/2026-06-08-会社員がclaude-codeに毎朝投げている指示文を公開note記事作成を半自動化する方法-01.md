## はじめに

「副業を始めたい」と思いながら、なかなか行動できない時期が続いていました。

やろうやろうと思いつつ、忙しさを言い訳にして後回しにしていた。そんな自分にようやく区切りをつけて、最近「AI活用ラボ」というnoteを立ち上げました。AIを使った副業や業務効率化を検証して、その結果をありのまま発信していくというコンセプトです。

まだ始めたばかりで、正直なところ手探りの状態です。ただ、動き出してすぐに気づいたことがあります。**時間がない、ということではなく、「毎回ゼロから考えること」が一番しんどい**、ということです。

本業の仕事が終わって帰宅し、子どもを寝かしつけてから作業できる時間は、平日だと1〜2時間くらい。その限られた時間の中で「今日は何を書こうか」から始めると、気づいたら記事の1文字も書けないまま0時を回っている、なんてことになりがちです。

続けるには、最初から仕組みを作っておく必要があると思い、導入したのがClaude Codeです。

---

## 記事作成で一番の課題は「時間」ではなかった

副業でnoteを続けようとすると、毎回こんな作業が発生します。

* 今週どんなネタを書くか考える
* 構成を組み立てる
* 本文を書く
* SNS（X）に投稿する文章を考える

一つひとつは大したことじゃないように見えますが、毎週これをゼロからやると消耗します。特に大変だと感じているのは「ネタ出し」と「構成作成」です。

書くことは好きなんです。でも「何を書くか」「どういう順番で伝えるか」を考えるのに時間とエネルギーを使ってしまって、肝心の本文を書くフェーズにたどり着けない。

気力がある日は書けるけど、疲れた平日夜には何も書けない。そういう状態になりそうな予感がしたので、先に手を打つことにしました。

---

## Claude Codeを導入して変わったこと

Claude Codeを使い始めて、最初に感じたのは「壁打ち相手ができた」という感覚でした。

たとえばネタ出し。「今週のAI副業の検証テーマ、何が読まれそうか」と投げると、いくつかの方向性を提案してくれます。全部採用するわけじゃないけど、ゼロから考えるより圧倒的に早い。

構成作成も同様です。「こういうテーマで、こういう読者に向けて書きたい」と伝えると、見出しの骨組みを作ってくれます。私はそれを見ながら「ここはいらない」「この順番は逆にしたい」と修正していく。最初から全部自分で考えるより、修正するほうがずっと速い。

そして感じているのは、**記事を書くことへの心理的なハードルが下がった**ことです。

「今日は何も書けなかった」という罪悪感が生まれにくくなりました。Claude Codeに作業の一部を任せることで、疲れた日でも少しだけ前に進みやすくなっています。

---

## なぜChatGPTではなくClaude Codeなのか

正直に言うと、ChatGPTも使っています。用途によって使い分けているのが現状です。

情報収集や簡単な調べ物はChatGPTのほうが手軽に使えることも多い。ただ、note記事のような長文を作成するときは、Claude Codeのほうが自分には合っていると感じています。

理由は大きく二つあります。一つは、長い指示文を渡したときの精度が安定していること。記事の文体や禁止事項など、細かい条件を複数指定しても、それをわりとちゃんと守ってくれる印象があります。もう一つは、文章のトーンが自分の好みに近いこと。ChatGPTで出力した文章はどこかビジネス文書っぽくなりがちで、noteの雰囲気にそぐわないと感じることがありました。

これはあくまで今の私の使い方における個人的な感想です。どちらが優れているかという話ではなく、記事作成という用途においてClaude Codeのほうがしっくりきた、というだけです。ツールは自分に合うものを選べばいいと思っています。

## 毎朝投げている指示文の考え方

「完璧なプロンプトを作ろう」とは思っていません。

完璧を求めるほど使わなくなるんですよね。指示文を磨くことが目的になってしまって、肝心の記事が書けなくなる。まずは動かせることを優先しています。

今私が使っている指示文のベースはこれです。

```
あなたはAI活用系メディアの編集者です。
以下の条件でnote記事を作成してください。

・記事テーマ：[テーマ]
・ターゲット：AI副業に興味がある会社員
・一人称：「私」
・文体：実体験ベース、初心者向け、親しみやすい文章
・文字数：2000〜3000文字

【禁止事項】
・誇張表現（「劇的に変わった」「圧倒的に」など）
・AIらしい一般論（「〜することが大切です」「〜を心がけましょう」など）
・失敗談・不安を省くこと
・体言止めの多用
・箇条書きだけで済ませること

まず記事構成（見出しのみ）を提案してください。
確認後に本文を作成します。
```

このプロンプトで特に意識しているのは3点あります。

**役割を明示する**ことで、「一般的な文章」ではなく「メディア編集者目線の記事」を意識した出力になります。「あなたは〜です」と最初に書くだけで、アウトプットのトーンがかなり変わりました。

**禁止事項を具体的に書く**ことが、AIっぽい文章を避けるうえで一番効いています。「自然な文体で」と書いても伝わらない。「〜することが大切です、は書かないで」と具体的な言い回しで制限をかけると、ぐっと読みやすくなります。

**二段階に分ける**こと（構成確認→本文作成）も重要です。一発で本文まで出力させると、方向性がずれたときの手戻りが大きい。構成を先に確認することで、「この見出しはいらない」「ここを先に持ってきたい」という調整が早い段階でできます。

このプロンプトは今も少しずつ調整しています。完成形ではなく、使いながら育てていくものだと思っています。

## 実際の運用フロー

平日と休日で役割を分けています。

**土日（作成フェーズ）**

休日の3〜5時間を使って、翌週分の記事ストックを作ります。Claude Codeと一緒に構成を作り、本文の骨格を整えて、自分の体験談や気づきを肉付けしていく。だいたい1〜2記事分をストックすることを目標にしています。

**平日（公開フェーズ）**

ストックしておいた記事をnoteに投稿して、X（旧Twitter）用の告知文を投稿するだけ。これだけなら平日夜の30分もあれば終わります。

まだ始めたばかりなので、この仕組みがうまく回るかどうかはこれから検証していく段階です。ただ、最初から仕組みを決めておくことで、「何から始めればいいか」で迷う時間が減った実感はあります。

## AIに全部任せない理由

「AIに書かせればいいじゃん」と思う人もいるかもしれません。実際、試したことはあります。

でも、全部AIに書かせた記事って、読んでいて何も残らないんですよね。情報としては正しいし、文章も整っている。でも「この人だから読みたい」という気持ちにならない。

**体験談は自分にしか書けない**。AI副業ラボを始めようと思った経緯、平日夜に疲れながら作業していること、うまくいかなかった検証結果。そういうリアルな話は、どんなに優秀なAIにも代替できません。

失敗談も同じです。「試してみたけど全然ダメだった」という話のほうが、読んでいる人には刺さることが多い。AIは失敗を経験しないから、失敗談は書けない。

私にとってClaude Codeは、**補助輪**のようなものです。大事なのは「続けること」であって、AIを使うかどうかはその手段に過ぎません。

## まとめ

AI活用ラボはまだ始まったばかりで、「継続できるか」も「成果が出るか」もこれから検証していく段階です。正直、うまくいくかどうかわかりません。

それでも、最初から仕組みを作っておくことで、続けやすい状態を整えることはできると思っています。Claude Codeは、そのための道具の一つです。記事作成の時間を減らすというより、**使えるエネルギーを増やす**イメージで使っています。

これからも運用しながら改善していきたいと思います。同じようにAI副業に興味がある会社員の方の、何か参考になれば嬉しいです。

## これから検証したいこと

現在は、

・週何本投稿できるか  
・記事ストック方式は続くのか  
・noteのアクセス数はどう変化するか  
・AI副業として収益化できるのか

を検証しています。

良い結果も悪い結果もそのまま公開していく予定です。

うまくいけば再現できるかもしれないし、失敗したら同じ失敗を避けられるかもしれません。

その過程も含めて記録していこうと思います。
