お疲れ様です。花冷です🌸

まだわからないことだらけのClaudeを使っていきます。

前回の記事でチュートリアルをこなし、  
どういうことがどの機能でできるのかをなんとなく把握しました。

今日はチュートリアルで扱ったScheduled Taskを自分用に作成してみます。

### 🚩スタートアップ

今回は\*\*「Scheduled（予定済み）」タスク\*\*を作成します。

やりたいことはいくつかありますが、まずは**毎日の情報収集**です。

**自分の情報収集は似たような記事が多くなりがち**という課題を感じていました。

私はシステムアーキテクトやソリューションアーキテクトを目指しているため、  
できるだけ多様な観点や技術を知りたいと思っています。

**『溢れる情報の中から、自分に合った情報を抽出する』**  
という作業をClaudeで自動化したいと考えました。

とはいえ最初から上手くできるわけではないので、  
今回はミナトAIさんの投稿を参考にしました。

※せっかくなのでFable5も使ってみました。

🌸まずは上手くできている人のやり方を模倣するのが近道だと思っています👣

> 自動化したい業務に使えるプロンプトを公開しているので、  
> 気になる方はぜひ参考にしてください！  
> ミナトAIさん、ありがとうございます！勉強になりました。

ミナトAIさんのプロンプトの対象業務部分は以下のように設定しました。

```
最新IT技術の収集（システムアーキテクト、ソリューションアーキテクトを目指しています。）
```

やってみないとわからない部分なので仕方ありませんね。

### 👀Scheduled Taskを微修正してみる

```
## わかりやすい書き方（成果物・報告）

ユーザーが読みやすい日本語で書く。専門家向けの略語だけの文は避ける。

- **専門用語・略語は初出で短く説明する。**
- **難しい言い回しより、普通の言葉を優先する。**
  - 避ける: 「〜が示唆される」「〜の文脈で」「〜を前提とした」
  - 使う: 「〜と書いてある」「〜ができるようになる」「〜のときに影響する」
- 要約内に長い括弧説明は入れない。用語の説明は下の「単語解説」へ分ける。
- **単語解説**（各エントリ任意）:
  - 要約で出てきた専門用語・略語を必要に応じて **1〜3個**、箇条書きで説明する。
  - ここは **用語の意味をわかりやすく言い換える場所**。一般的なIT用語の定義を書いてよい。
  - 難しい用語がなければ、単語解説は実施しなくてよい。
```

この内容をAGENTS.mdに追記して再実施してもらいました。

すると結果は以下のように変わりました。

![image](https://static.zenn.studio/user-upload/deployed-images/9d84c537e68933a9bf45ffff.png?sha=90b9bace4590ed8c530f2f7ac856b0e32a685d9f)

どうでしょうか？

先ほどは専門的すぎた要約が、言い回しが柔らかくなり、  
単語解説を追加したことでより親切になった印象です。

このような定常タスクは、  
**一度やってみてから少し続けてみる**ことが重要ですよね。

そこから微調整していくイメージです☁️☁️
