# はじめに

最近、ターミナル上で自律的に動くAIコーディングエージェント（Claude Code や Antigravity CLI、OpenAI Codexなど）を利用する開発者が増えています。しかし、高度な設計判断や複雑なバグ修正、多角的なリスク検証といった難問に対して、単一のモデルに「一発回答」を求めると、どうしても思考の盲点やハルシネーション（もっともらしい嘘）が発生しがちです。

こうしたエージェントの限界を超えるために、ポータブルな能力（スキル）をAIエージェントに直接追加できるオープン規格が **Agent Skills** です。

本記事では、オープンソースで提供するクロスエージェント向けのスキルコレクション **`buddypia/agent-skills`** と、その中で提供される最も強力な意思決定・自律解決スキルである **`multi-llm-recursive-meta-cognition`（再帰的メタ認知）** について徹底解説します。

# Agent Skills（agent-skills）とは？

**`buddypia/agent-skills`** は、さまざまなAIエージェント（Claude Code、OpenAI Codex、Cursor、Gemini CLI / Antigravity など）を横断して動作する、MITライセンスの共有スキルコレクションです。

個々のスキルは `SKILL.md` という標準化された仕様に準拠しており、手動でコピーするか、[`skills`](https://skills.sh) CLIを使用することで、お使いのAIエージェントにnpx経由で**一発でインストール**することができます。

```
# 共有リポジトリからお使いのエージェントにスキルを追加する
npx skills add buddypia/agent-skills
```

このコレクションに含まれる多くのスキルは、**「異なるベンダーのLLM（Gemini、Claude、GPT）をまたいで協調動作させ、お互いの成果をチェックし合う」** というアプローチを採用しています。これにより、単一モデルのバイアスや盲点を克服し、頑健な回答を導き出します。

---

コレクション内の代表的なスキルである **`multi-llm-recursive-meta-cognition`** は、難解な多段階の推論問題に対し、3つのベンダー公式CLI（`agy` / `claude` / `codex`）を駆動して、以下の5つのステージからなる「再帰的メタ認知」パイプラインを実行します。

![5段階の再帰的メタ認知パイプラインのフロー](https://static.zenn.studio/user-upload/e95e3c60fc53-20260701.png)

*図2: 5段階の再帰的メタ認知パイプラインのデータフロー*

### 5つのステージの役割と担当モデル

1. **Decompose（分解）** — `agy` (Gemini 3.5 Flash)
   * 大きな課題を、依存関係のない独立した小さなサブタスクに分解し、整理します。
2. **Solve（解決）** — `agy` (Gemini 3.5 Flash)
   * 分解されたサブタスクに対して、それぞれ個別のドラフト解答（コードの骨組みやロジック）を生成します。
3. **Verify（検証）** — `claude` (Claude Code - Opus 4.8)
   * ドラフト解答を冷静に監査し、矛盾、論理的破綻、エッジケースの漏れ、セキュリティリスクを徹底的に洗い出して指摘リストを作成します。
4. **Integrate（統合）** — `codex` (GPT 5.5)
   * ドラフト解答と検証による指摘リストを統合し、指摘された問題を解決するように全体を繋ぎ合わせて再構築します。
5. **Reflect（内省）** — `codex` (GPT 5.5)
   * 統合された解決策について、最初の問題に対する適合度や思考プロセスそのものを自己評価・修正し、最終的な品質を確定します。

推論の深さは環境変数 `MULTILLM_REASONING_EFFORT` で調整でき、デフォルトは `high` です。`xhigh` も指定できますが、5ステージを直列実行する分の時間予算を圧迫しやすいため、公式には `high`（速度優先なら `medium`）が推奨されています。

各モデルは完全に独立したコンテキストで実行され、中間の状態はすべて構造化されたJSONとしてやり取りされます。異なるベンダーで学習されたモデルは**相関の低い誤り**を犯すため、Claudeの厳しい「Verify（検証）」とGPTの「Reflect（内省）」を経ることで、より頑健な結果を得ることができます。

# 主なユースケース

このスキルは、手早い調べ物や単純なコード編集には向きません（コストと時間がかかるため）。以下のような、重要度の高い複雑なタスクで効果を発揮します。

* **複雑なアーキテクチャのセキュリティ脆弱性診断**
  + **状況**: 大規模なWebアプリケーションのソースコードから潜在的な脆弱性を探す際。
  + **効果**: Geminiが分解した各モジュールをClaudeが超厳格に検証（Verify）し、GPTがその指摘を統合して実用的な対策へ落とし込みます。
* **異なる仕様が混在する大規模レガシーコードの移行計画策定**
  + **状況**: 古いフレームワークからモダンな環境へシステムを安全にマイグレーションする際。
  + **効果**: 移行に伴う依存関係をDecomposeで整理し、移行手順をSolveで作成し、Verifyで移行時のダウンタイムやデータの整合性バグを未然にあぶり出します。
* **ビジネス・技術的な制約が競合する意思決定レビュー**
  + **状況**: セキュリティ、納期、予算、法規制などの複数の制約がある中でのアーキテクチャ選定。
  + **効果**: 5つのフェーズによる思考・監査の繰り返しにより、特定のバイアスや見落としのない高度な意思決定レポートが作成されます。

# 使い方（エージェントからの呼び出し方）

`npx skills add` でスキルをエージェントに追加した後は、ターミナルで手動コマンドを叩く必要はありません。お使いのAIエージェント（Claude Codeなど）に対して、チャットで直接実行を指示するだけです。

### エージェントへの指示例

> 「multi-llm-recursive-meta-cognitionスキルを使って、[解決したい複雑な問題] を解決して。」

これだけで、エージェントはインストールされた `SKILL.md` の指示を解釈し、裏で必要なスクリプトを自動的に呼び出して5段階のメタ認知パイプラインを実行し、その結果をまとめて回答してくれます。

### 事前準備（裏で必要なCLIの設定）

エージェントが裏でこのスキルを問題なく実行できるようにするために、初回のみ以下の環境設定を済ませておく必要があります。

1. **3つの公式CLIのインストールとログイン**:  
   ローカル環境で `agy`、`claude`、`codex` コマンドが動作し、それぞれログイン（OAuthまたはサブスクリプション認証）が完了している必要があります。
2. **依存関係の自動解決**:  
   Pythonの依存パッケージは、エージェントがスキルを実行する際に `uv` または `venv` を通じて自動的にセットアップされるため、手動でインストール作業を行う必要はありません。

# 各ステージが返すデータ：confidenceスコアと不確実性の可視化

このスキルの面白いところは、5つのステージがすべて自由記述の文章ではなく、**構造化されたJSON**でバトンを渡していく点です。実際に各ステージへ渡されるプロンプト（`assets/prompts/*.txt`）を読むと、出力スキーマが以下のように定義されています。

| ステージ | 主なJSONフィールド |
| --- | --- |
| Decompose | `subtasks`（サブタスク配列）, `assumptions`（前提）, `constraints`（制約）, `questions`（不明点）, `confidence` |
| Solve | `solutions`（サブタスクごとの解答）, `open_questions`, `risks`, `confidence` |
| Verify | `issues`（指摘事項）, `corrections`（修正提案）, `self_corrections`, `validation_notes`, `confidence` |
| Integrate | `integrated_answer`（統合後の解答）, `applied_corrections`（反映した指摘）, `confidence` |
| Reflect | `final_response`, `confidence_score`, `uncertainties`（残存する不確実性）, `self_corrections`, `reflection_notes` |

ポイントは、**全ステージが `confidence`（0.0〜1.0の自己申告スコア）を返す**ことと、最終のReflectステージでは「confidenceが0.8未満なら `uncertainties` の明示が必須」というルールがプロンプト側に明記されていることです。つまりこのスキルは単に5モデルをバケツリレーさせているのではなく、各モデルに「自分の解答にどれだけ自信があるか」「まだ何が不確かなのか」を毎回言語化させることで、最終出力が読者にとって"どこまで信じてよい回答か"を判断しやすい形にする設計になっています。

# 類似スキルとの使い分け

`buddypia/agent-skills` には、同じ「異なるベンダーCLIを協調させる」思想を持つスキルが他に2つあります。目的に応じて使い分けると効果的です。

| スキル | 進行パターン | ステージ数 | 向いている用途 |
| --- | --- | --- | --- |
| `multi-llm-debate` | proponent → opponent → moderator（討論） | 3 | 賛否が分かれる意思決定、トレードオフの比較検討 |
| `multi-llm-reflection` | generator → critic → refiner（生成・批評・改善） | 3 | 文章・設計・分析など、品質を練り上げたい成果物づくり |
| `multi-llm-recursive-meta-cognition`（本記事） | decompose → solve → verify → integrate → reflect | 5 | 複雑な多段階の問題解決。分解してから解いて検証するプロセスそのものが必要な難問 |

`multi-llm-debate` と `multi-llm-reflection` は3ステージで比較的短時間（数分程度）に結果が返るのに対し、本記事の `multi-llm-recursive-meta-cognition` は5ステージ構成で最も重く、その分「分解の妥当性」「解の検証」「統合後の再確認」まで踏み込みます。「そもそも結論を出すこと自体に価値がある単純な二択」なら `multi-llm-debate`、「一発書いた文章やコードの完成度を上げたい」なら `multi-llm-reflection`、「複雑すぎて一人（一体）では解ききれない問題を分解から始めたい」なら本スキル、という選び方になります。

# メリット・デメリット（トレードオフ）

| メリット | デメリット |
| --- | --- |
| **適材適所の協調**: 探索が得意なGemini、監査が得意なClaude、推論が得意なGPTを最適に配置できる。 | **初期セットアップの手間**: `agy`、`claude`、`codex` の3つのCLIをインストールし、それぞれログインを通す必要がある。 |
| **高い堅牢性**: Claudeによる検証フェーズを挟むため、論理破綻やエッジケースの見落としが極めて少ない。 | **実行時間の長さ**: 5つのフェーズを順次実行するため、回答が完了するまでに数分かかる。 |
| **客観性の担保**: 単一モデルのバイアスや「ハルシネーションの自己肯定」を防ぎ、多角的な視点から精査される。 | **各サブスクリプションの利用量消費**: 基本は各CLIの既存ログイン（サブスクリプション認証）で動くため追加のAPI課金は必須ではないが、その分プランのレート制限や利用枠を消費する。 |

# まとめ

`buddypia/agent-skills` は、AIエージェントの回答を単一モデルの「一発回答」から、複数のモデルが互いを補完し合う「マルチLLMによる協調解決」へとアップグレードするポータブルなスキルコレクションです。  
セットアップや実行時間のコストはかかりますが、ミッションクリティカルな要件や複雑な問題の解決においては、非常に頼もしい設計フレームワークを提供してくれます。

是非、`npx skills add buddypia/agent-skills` でお使いのエージェントに追加し、その効果を体験してみてください！
