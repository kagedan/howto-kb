ネタバレするわ。
Claude CodeのSkillsは、プロンプト集にするより作業レシピにするとかなり強いです。

これ、最初は勘違いしやすいところで、Skillsって聞くと「よく使うプロンプトを保存する場所」みたいに見えるんですよね。

もちろん、それでも便利です。
毎回同じ指示を貼らなくて済むだけでも助かる。

でも、本当においしいのはそこじゃないです。

Skillsはプロンプト集というより、AIに仕事の手順を覚えさせる作業レシピです。

たとえばX投稿を作るスキルなら、ただ「でぷれっぽく書いて」と置くのではなく、何を入力素材にするか、どの読者に向けるか、使っていい口調は何か、使わない構文は何か、最後に何を検品するか、どこに保存するかまで分けて置く。

構造としては、こんな感じです。

/投稿生成スキル
├── 目的（何を作るスキルか）
├── 入力素材（過去投稿、メモ、リサーチ、一次情報）
├── 口調（でぷれの言い回し、温度、語尾）
├── NG構文（箇条書き量産、私語り調、前は今は量産）
├── 検品条件（切り口、文字数、冒頭、重複）
└── 保存先（Obsidianで確認できる場所）

こうしておくと、AIへの頼み方が変わります。

「投稿を書いて」ではなく、「このスキルの手順で、素材を読んで、切り口を選んで、でぷれ口調で書いて、最後にNG構文を落として保存して」になる。

さらに返してほしい成果物も固定します。投稿本文だけではなく、採用した切り口、参考にした冒頭型、NG構文チェック、修正理由、保存先まで出させる。ここまで返ってくると、人間は本文だけを読んで感覚で直すのではなく、どの判断でその文章になったかを見られる。

プロンプトだけだと、その場の会話で終わります。
でもSkillsにすると、同じ作業を次回も使える形で残せる。

AI活用で反応が取れるのって、こういう地味な仕組みの話だと思ってます。

便利なプロンプトを集めるより、自分の仕事の手順をAIが読める形にする。
ここまでいくと、Claude Codeはただのチャットじゃなくて、制作ラインに近くなる。
