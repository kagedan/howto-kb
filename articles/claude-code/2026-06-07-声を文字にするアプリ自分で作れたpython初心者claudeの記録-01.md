## 「前に作ったツール、どこ行ったっけ？」から始まった

ある日、自分のデスクトップを眺めていて、ふと思った。

> 「マイクからしゃべった声を、そのまま文字にできるソフトって、自分で作れないかな？」

会議の録音、取材メモ、動画のテロップ起こし。文字起こしって地味に時間がかかる。  
そして調べてみると、世の中の文字起こしサービスの多くは**クラウドに音声を送る**仕組みだった。仕事の打ち合わせ音声を外部に送るのは、ちょっと気が引ける。

そこで方針はこうなった。

* **完全オフライン**（音声を外に出さない）
* **無料**
* **Python初心者でも作れる範囲**

結論から言うと、Claude（Claude Code）と一緒に作りきれた。  
しかも最終的には、ファイル変換・マイク録音・**話しながらのリアルタイム文字起こし**・**話者分け（誰が話したか）**・**要約**・**英語対応＆日本語翻訳**まで盛り込んだ、ちょっとした業務ツールになった。すべてオフラインで。

ただ、ここに辿り着くまでに **5回、見事に詰まった**。  
この記事は、その「詰まり方」と「抜け方」の記録だ。同じところでハマる人の役に立てばと思う。

---

## 何を選んだか：技術スタック

最初に、最終的な構成だけ載せておく。

| 役割 | 使ったもの | なぜ |
| --- | --- | --- |
| 音声→テキスト | **faster-whisper** | OpenAI Whisperの高速版。後述の理由で `torch` が要らない |
| 音声の読み込み | **PyAV** | これのおかげで**ffmpegのインストールが不要** |
| マイク録音 | **sounddevice** | シンプルにマイク入力が取れる |
| 話者分け | **sherpa-onnx** | ONNXベースで、これも**torch不要** |
| 要約・翻訳 | **Ollama**（ローカルLLM） | PC内で動くので完全オフライン |
| 画面（GUI） | **tkinter** | Python標準。追加インストール不要 |

ポイントは「**重いものを避ける**」。とくに `torch`（PyTorch）と `ffmpeg` を入れずに済ませたのが大きい。理由は次の章でわかる。

---

## つまずき①：Python 3.14が新しすぎて、定番ライブラリが入らない

最初の壁。文字起こしと言えばWhisper、Whisperと言えば普通は `openai-whisper` や `torch` を入れる。  
ところが私の環境は **Python 3.14**。出たばかりのバージョンだ。

`torch` は、新しすぎるPythonにはまだ対応（ビルド済みのファイル＝wheel）が用意されていないことが多い。案の定、すんなり入らない。

ここでClaudeの提案が効いた。

> **torchを使わない `faster-whisper` にしましょう。**

`faster-whisper` は内部で CTranslate2 という別エンジンを使っていて、`torch` に依存しない。試しに入れてみると——あっさり成功。しかも `PyAV` という音声デコードライブラリも一緒に入って、**ffmpegのインストールすら不要**になった。

```
python -m pip install faster-whisper
```

> 教訓：新しいPythonで定番ライブラリが入らないときは、「同じことをする"軽い代替"」を探す。`torch` → `faster-whisper`、`ffmpeg` → `PyAV` のように。

---

## つまずき②：バッチをダブルクリックしたら、一瞬で閉じる

最小のスクリプトができたので、初心者にやさしく「音声ファイルを **`.bat` にドラッグ&ドロップ**したら変換される」形にした。

ところが——**ウィンドウが一瞬開いて、すぐ閉じる**。何も起きない。

原因は2つあった。どちらもWindowsあるあるだった。

### 原因A：`python` がニセモノだった

Windowsには、Microsoft Storeへ誘導するための**ダミーの `python.exe`**（`WindowsApps`の中）が住んでいることがある。`.bat` の中で `python` と書くと、本物ではなく**こっちが反応して何もせず閉じる**。

→ 対策：バッチでは**本物のPythonをフルパスで指定**する。

### 原因B：`.bat` に日本語を書いたら文字化けして誤作動

`.bat` ファイルはWindowsが昔ながらの文字コード（cp932）で読む。なのに私はUTF-8で日本語のメッセージを書いていた。結果、文字化けして命令がぐちゃぐちゃに。

→ 対策：**バッチの中身は英語（ASCII）だけ**にして、日本語の案内はREADMEに逃がす。さらにエラーで即閉じしないよう、末尾に必ず `pause` を入れる。

> 教訓：`.bat` は「①本物のPythonをフルパスで」「②中身はASCIIだけ」「③末尾にpause」。この3点で“一瞬で閉じる”の9割は防げる。

---

## つまずき③：モデルのダウンロードがSSLエラーで止まる

Whisperは初回だけAIモデルを自動ダウンロードする。ところがここで——

```
SSL: CERTIFICATE_VERIFY_FAILED
```

証明書の検証に失敗してダウンロードできない。会社や家庭のセキュリティソフト・プロキシが通信を検査していると、証明書が標準のものと食い違ってこうなる。

`certifi`（証明書の束）を指定しても直らない。最終的に効いたのが **`truststore`** だった。

```
import truststore
truststore.inject_into_ssl()  # Windowsの証明書ストアを使うようにする
```

これでWindowsが持っている証明書を使うようになり、ダウンロードが通った。

> 教訓：Windowsで `CERTIFICATE_VERIFY_FAILED` が出たら、`truststore` で**OSの証明書ストアを使わせる**のが手っ取り早い。

## ここから一気に「育てた」

土台が動いたら、あとは機能を1つずつ足していった。Claudeに「次はマイク入力」「次は画面化」と頼むと、その都度ちゃんと積み上がっていく。最終的にこうなった。

* 🎵 **音声ファイルを変換**（mp3 / wav / m4a など）
* 🎤 **マイク録音 → 文字起こし**
* 🔴 **リアルタイム文字起こし**（話すと数秒遅れで文字が出続ける）
* 📝 **自動整形**（「えーと」などのフィラー削除・読みやすく改行）
* 🗣 **話者分け**（「話者1：」「話者2：」と分けて出力）
* 🧾 **要約**（要点・ToDoまで自動整理）
* 🌐 **英語対応＆日本語への翻訳**

全部、画面（GUI）のボタンから使えるようにした。

### リアルタイムは「無音で区切る」方式にした

リアルタイム文字起こしは、ただ垂れ流すと単語の途中で切れて読みにくい。  
そこで「**話の切れ目（少しの無音）を検知して、一区切りごとに文字にする**」方式にした。音の大きさ（RMS）で「話している／黙った」を判定するシンプルな仕組みだ。これだけで、ぐっと自然な文章になる。

### 話者分けは“torch不要”で実現した

「誰が話したか」で分ける機能（話者ダイアライゼーション）は、普通は `pyannote` などを使い、結局 `torch` が必要になる。3.14では避けたい。

調べると **`sherpa-onnx`** がONNXベースで、Python 3.14にも対応していた。これと文字起こしを**単語ごとの時間で突き合わせ**て、「話者1：〜」の形に組み立てた。検証では、別人の声をちゃんと2人に分け、同じ人なら1人と判定できた。

### 要約と翻訳は、手元のローカルLLM（Ollama）に任せた

PCに入れていた **Ollama**（ローカルで動くLLM）に文字起こし結果を渡して、要約と翻訳をしてもらう構成にした。これも完全オフライン。  
「英語の音声 →（Whisperで）英語テキスト →（Ollamaで）日本語テキスト」という流れで、**英語の会話を日本語のメモにする**ことまでできるようになった。

## つまずき⑤：精度を上げようとしたら、AIが“ヒントを丸暗記して吐き出した”

最後の罠が、いちばん面白かった。

固有名詞（会社名・専門用語）の精度を上げるため、Whisperに「こういう言葉が出てきます」とヒント（語彙）を渡した。最初は効果バツグンで、「4000**社**」のような同音異義語（社／車）も正しく出るようになった。

ところが——文字起こし結果の末尾に、**渡したヒントの単語がそのまま羅列で出てくる**ことがあった。

```
スクリーニング 配当 スクリーニング 配当 … 株式投資 株式投資 …
```

これはWhisperの「ハルシネーション（幻覚）」の一種で、無音や不明瞭な箇所で**ヒントをオウム返し**してしまう現象だった。

対策は2段構え。

1. ヒントの渡し方を、復唱されにくい仕組み（`hotwords`）に変更
2. それでも漏れたとき用に、**“怪しい出力”を自動で捨てるフィルタ**を追加

フィルタの判定ロジックがちょっと面白い。日本語は本来、単語の間にスペースが入らない。なのに「**スペースが多い＋ヒント語が複数混じる**」出力は、ほぼ間違いなく幻覚だ。これと「同じ単語が何度もくり返される」を組み合わせて検出した。

```
# ざっくりした考え方
# ・日本語なのにスペースだらけ ＆ ヒント語が複数 → 幻覚として捨てる
# ・同じ単語が4回以上くり返される → 幻覚として捨てる
```

英語モードのときは、ヒント語（日本語）と一致しないので誤って消えることもない。実際の漏れ文だけをピンポイントで除去できた。

> 教訓：AIに「ヒント」を与えると精度は上がるが、**そのヒント自体を出力にコピーしてしまう**ことがある。与えっぱなしにせず、「おかしな出力を弾く番人」を後ろに置くと安定する。

## まとめ：初心者がオフラインAIツールを作るときのコツ

今回の5つの壁から、再利用できる教訓だけ抜き出すと——

1. **新しすぎるPythonで定番が入らない** → 同じことをする“軽い代替”を探す（torch→faster-whisper）
2. **`.bat`が一瞬で閉じる** → 本物のPythonをフルパス指定／中身はASCII／末尾pause
3. **SSL証明書エラー** → `truststore` でOSの証明書ストアを使う
4. **重いライブラリを避ける** → torchもffmpegも使わずに、音声認識も話者分けもできる
5. **AIにヒントを渡すと、ヒントごと出力する** → “怪しい出力を弾く番人”を後ろに置く

同じように「自分の手元で動く、自分専用のAIツール」を作ってみたい人の、はじめの一歩になればうれしい。

次は、話者を実名にリネームしたり、要約をその場でリアルタイム表示したり——まだ育てる余地がある。また記録します。
