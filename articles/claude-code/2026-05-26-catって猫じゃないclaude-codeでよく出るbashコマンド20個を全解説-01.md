# 「cat」って猫じゃない。Claude Codeでよく出るBashコマンド20個を全解説

Claude Codeを使っていると、こんな会話が起きることがある。

---

Claude：「では以下のコマンドを実行します」

```
cat error.log | grep "ERROR" | tail -20
```

自分：「……猫？」

---

これは猫ではない。`cat`はファイルの中身を表示するコマンドだ。

Claude Codeは作業の中でターミナルのコマンドを使う。エンジニアじゃなくても、このコマンドが「何をしているのか」をざっくり知っておくだけで、Claudeが何をしようとしているかが読めるようになる。

ここでは、Claude Codeを使っていてよく出てくるコマンド20個を、ビジネスパーソン向けにやさしく解説する。

## まず「ターミナル」と「Bash」って何？

ターミナルとは、コンピュータに文字で命令を送る黒い画面のことだ。マウスでクリックする代わりに、文字を打つことで操作する。

BashはそのターミナルでPCに命令するための「言語」のようなもの。Claudeはこのターミナルを通じて、ファイルの読み書きやプログラムの実行をしている。

Claude Codeのターミナルで `!コマンド` と打つと、そのコマンドをその場で実行できる（例：`! ls` と打つと今いるフォルダのファイル一覧が出る）。

## ファイル・フォルダ操作系

**`ls`**  
今いるフォルダの中にあるファイルやフォルダの一覧を表示する。  
「このフォルダ、何が入ってる？」の確認に使う。

**`ls -la`**  
`ls`の強化版。隠しファイル（`.env`など）も含む詳細な一覧を表示。ファイルのサイズや更新日時も出る。

**`cat ファイル名`**  
ファイルの中身をそのまま表示する。「猫」じゃなくて「concatenate（連結する）」の略。  
Claudeが「!cat error.log」と実行するとき、エラーログの中身を読んでいる。

**`cd パス`**  
フォルダを移動する。`cd`は「change directory（ディレクトリを変える）」の略。

`cd ..` で一つ上のフォルダに戻る。`~` はホームフォルダ（ユーザーフォルダ）を指す。

**`mkdir フォルダ名`**  
新しいフォルダを作る。`mkdir`は「make directory」の略。

**`cp 元ファイル 先`**  
ファイルをコピーする。`cp`は「copy」の略。

`.env.example`（テンプレ）を`.env`（実際の設定ファイル）にコピーするのはよく使う操作。

**`rm ファイル名`**  
ファイルを削除する。⚠️ ゴミ箱に入らず即削除されるので注意。  
`rm -rf フォルダ名` はフォルダごと全削除。これが出てきたら何を消そうとしているかを必ず確認する。

## テキスト検索系

**`grep "文字列" ファイル名`**  
ファイルの中から特定の文字列を含む行を探して表示する。  
エラーログから「ERROR」の行だけ抜き出す、といった使い方。

**`find . -name "*.py"`**  
現在のフォルダ以下から条件に合うファイルを探す。  
「どこにそのファイルある？」というときに使う。`*.py`はPythonファイル全部、という意味。

## プログラム実行系

**`python3 スクリプト名.py`**  
Pythonで書いたプログラムを実行する。  
Claudeがスクリプトを作ったあと、動作確認のために実行する。

**`pip install パッケージ名`**  
Pythonの追加機能（ライブラリ）をインストールする。  
「requests」（HTTP通信）や「pandas」（データ処理）などを追加するときに使う。

**`npm install`**  
Node.js（JavaScriptの実行環境）のパッケージを一括インストールする。  
Next.jsなどのWebアプリのセットアップ時に最初に実行するもの。

**`npm run dev`**  
開発用サーバーを起動する。実行後にブラウザで `localhost:3000` を開くとアプリが表示される。

## Git（バージョン管理）系

Gitとはコードの変更履歴を管理するツール。「いつ・誰が・何を変えたか」を記録する。

**`git status`**  
現在変更されているファイルの一覧を表示する。「今何が変わってる？」の確認。

**`git diff`**  
ファイルの変更内容を Before / After で表示する。コミット前に「何を変えたか」を確認するのに使う。

**`git add .`**  
変更したファイルを全部コミット候補にする。`.`は「全部」を意味する。

**`git commit -m "メッセージ"`**  
変更を記録する（セーブポイントを作る）。メッセージに「何をしたか」を書く。

```
git commit -m "add scraper for food factory list"
```

**`git push`**  
ローカルの変更をGitHub等のサーバーに送る。VercelやNetlifyと連携していれば、pushした瞬間に本番サイトに反映される。

## ポート・プロセス確認系

**`lsof -i :3000`**  
ポート3000を使っているプロセスを確認する。  
「アプリが起動しない」というとき、すでに別のプロセスがそのポートを使っていないか確認するために使う。

**`curl URL`**  
URLにHTTPリクエストを送ってレスポンス（返ってくるデータ）を表示する。  
APIが動いているか確認したり、スクレイピング前にページの中身を確認したりするときに使う。

```
curl https://api.example.com/status
```

## コマンドがよくわからなくても大丈夫な理由

Claude Codeは自分でコマンドを考えて実行してくれる。コマンドを自分で打てなくても、「このフォルダのファイル一覧を見せて」と日本語で言えば `ls -la` を実行して結果を見せてくれる。

ただ、「今Claudeが何をしているのか」を把握する上で、コマンドの意味がわかっていると安心感が全然違う。特に `rm` 系の削除コマンドが出てきたときに「これ何を消そうとしてる？」と気づけるかどうかは重要だ。

完全に覚える必要はない。「あのコマンド、なんだっけ」と思ったらこの記事に戻ってきてほしい。
