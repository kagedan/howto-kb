最近あらためて感じているのは、**「AI利用は 波に乗り遅れないことが超重要」**ということです。  
特に 非エンジニアは、"ものづくり" の文脈では AIをまだまだ上手く活用できていないケースが大半でしょう。

プログラミングを勉強してから使う。  
もっと理解度が高まってから使う。  
綺麗な設計ができるようになってから作る……etc

……等と考えているうちに、気付けば「使えるひと」と「そうでない●●●●●ひと」の差がとんでもなく開いてしまう──今はまさに「その瀬戸際に在り」。

そこで今回（参考までに）、非エンジニアとして Claude Codeを使い、**自分だけのキュレーションアプリ**を作った流れを書き留めておきます。

  

---

## ▶ 作ったもの：Insight Picker

今回作ったのは、ざっくり言えば**「自分専用の情報収集アプリ」**です。

名付けて **"Insight Picker"**。  
目的は、「普段自分から積極的に取りに行かない情報源」から、

* **ちょっと面白い小ネタ**
* **雑学的な話題**
* **AI活用のヒント**
* **ビジネスへの示唆がありそうな記事**
* **海外コミュニティで盛り上がっているトピック**

──を拾ってくることです。  
「普段自分から積極的に取りに行かない情報源」は、主に以下👇

ポイントは、**自分が普段見ているタイムラインの外側に、あえて釣り糸を垂らす**ことです。

既存のキュレーションアプリは便利ですが、（情報源含め）自分の興味関心の内側だけを回遊してしまいがち。  
そこで、その外側にある示唆●●●●●●●を拾いに行くための道具として、自分だけのキュレーションアプリを作りました。

結論から言うと、**楽勝で作れました。**レベル感で言うと、

斯様にサクサクと進めたのは、Claude Codeが単にコードを生成するだけではなく、**こちらの状況を予測しながら 次に何をすれば良いかを案内してくれる**からです。

> 🦀「あなたの環境でPythonが使えるか確認しています」

## ▶ 実際の流れ

やったことは、大きく分けるとこんな感じです。

### 1. 作業用フォルダを作る

まず、WindowsのPowerShellで作業用フォルダを作りました（ここらへんの不明点は ChatGPTに訊きながら進めています）。

```
cd C:\Users\User
mkdir insight-picker
cd insight-picker
```

この時点では、まだ何もありません。  
ただの空フォルダです。  
後々、この場所が「自分だけのアプリ工房」になります。

\*\*\*

### 2. Claude Codeを起動する

次に、作成したフォルダ内でClaude Codeを起動します（"claude" と打ち込む）。

```
claude
```

すると、Claude Codeの画面が立ち上がります👇

![](https://assets.st-note.com/img/1779111899-HDluax5znGJmyToWt06rO1LE.png?width=1200)

ここまで来たら、（Chat GPT や Gemini を利用するのと同様に）作りたいアプリの仕様を自然文（≠プログラミング言語）で打ち込みます。

今回は、以下を伝えました👇

実際に使用したプロンプトはこちら……👇

```
自分だけが使うローカルWebアプリ「Insight Picker」を作りたいです。目的は、はてなブックマークRSSとReddit RSSを中心に、雑学的・示唆的・note記事ネタになりそうなURLと見出しを自動収集することです。

最重要条件：
```

### 3. Claude Codeがファイルを作る

Claude Codeは、必要なファイルを作っていきます。

たとえば、こんな感じ……👇

```
app.py
database.py
collector.py
scorer.py
requirements.txt
README.md
templates/
static/
```

非エンジニアから見ると、この時点では（正直）何が何だかわかりません。

ただ、これもClaude Codeに訊くと教えてくれます。  
曰く、以下のような役割とのこと👇

* app.py：アプリ本体
* database.py：保存処理
* collector.py：情報収集
* scorer.py：スコア付け
* templates/：画面のHTML
* static/：見た目のCSSや画像
* requirements.txt：必要な部品一覧

名称から想起される意味と、教えてくれた各役割との乖離がなさそうなので、理解だけしてドンドンと先に進めていきます。

\*\*\*

### 4. Pythonを入れて、必要な部品をインストールする

途中で、Pythonが使えない？問題が生じました。  
ここは少し詰まりましたが、「そもそも、Pythonが使える状態じゃないじゃん」と気付いたので、Pythonを早速インストールし、必要なライブラリを入れました。

```
python -m pip install -r requirements.txt
```

このコマンドを実行すると、Flaskやfeedparserなど、アプリを動かすための部品が入ります。  
「Successfully installed」と画面に表示されたら、基本的には成功です。

\*\*\*

### 5. アプリを起動する

その後、作業フォルダに移動した状態で、次を実行（打ち込み）します。

```
python app.py
```

すると、PowerShellに以下の表示が出ます👇

```
Running on http://127.0.0.1:xxxx
```

これで、アプリが「自分のPC上で動く」状態になります。  
任意のブラウザで、

```
http://127.0.0.1:xxxx
```

これ👆を開くと、"Insight Picker" の画面が表示されます。

これにより、単に情報を集約・通過させるだけではなく、それを次回のスコアリングに反映させながら**アプリ自体を育てていく**ことができます。

## ▶ なぜ、はてブとRedditなのか

今回、報道機関出典の硬いニュースをキュレーションの対象には選びませんでした。  
理由は、欲しかったのが "ニュース" ではなく、**"示唆"**だったからです。  
しかも、「普段は能動的に触れることのない情報源」から取りたかった──。

どちらも、**普段の私がまず触れることのない情報源**です。  
加えて、「ひとが反応している情報」を取りやすく。

報道機関が整えた綺麗な●●●情報ではなく、誰かが「これは面白い」「これは議論したい」と感じた、人肌にざらついた●●●●●●●●ものが流れてきます。  
今回の目的感には、そちらの方が合っていました。

---

## ▶ 完璧なアプリではないが、それも味

もちろん、今回作ったものは完璧ではありません。

市販アプリのように洗練されている訳ではないですし、UIも改善の余地があります。PowerShellを起動しないと使えないので、アプリを利用できる環境も限られます。

ただ、それでも充分。  
なぜなら、今回の目的は「万人向けのプロダクトを作ること」ではなく、**「非エンジニアがClaude Codeに触れながら、欲しいものを形にしてみること」**だからです。

……そして、この一歩には大きな意味があります。

---

## ▶ AI利用は「理解→使用」では遅い

AIツールは、理解を深めてから使うものではないと思います。  
むしろ、

> とりあえず触ってみる  
> 詰まったら訊く  
> 動いたら少し変える  
> 失敗したら戻す  
> 小さく作り、育てていく

……という態度の方が合っています。  
少し前までなら、まずはプログラミング言語を学び、環境構築すらも手探りで、エラーと格闘しながら牛歩的に●●●●理解を深めていかなければなりませんでした。

しかしながら、Claude Codeは 自然文で、

```
こんなものを作りたい
これは使いたくない
無料の範囲で進めたい
既存機能は壊したくない
初心者向けに説明してほしい……etc
```

……と伝えながら革新的に●●●●進められます。  
もちろん、完全に任せ切る訳にもいかないのは事実です。  
とはいえ、非エンジニアにとって「ものづくりの間口」は 飛躍的に広がっています。

## ▶ 自分専用の道具を持つ感覚

今回 特に興味深かったのは、**自分専用の道具を持つ感覚**です。  
世の中には、優れたアプリがたくさんあります。

けれども、それらは基本的に「みんなのための道具」です。  
他方、Claude Codeで作るローカルアプリは、最初から自分だけの要望に合わせて作れます。

例えば、私の場合は、

* 硬いニュースより、示唆的な話が欲しい
* はてブとRedditを中心にしたい
* note記事のネタになりそうなものを拾いたい
* 「興味なし」を次回以降の情報収集に反映させたい
* 無料で使いたい
* APIキーには手を出さなくてよろしい
* 自分のPCだけで動けばいい

これらの条件がありました。  
既製のアプリでは、「はてブとRedditを中心にしたい」がまず難しい。  
しかし、Claude Codeなら 自然言語のみでパパっと作れてしまう。  
この**「自分だけの道具を持つ感覚」が具体化されるに至る体験**は、かけがえのない価値があります。

## ▶ 結び：非エンジニアこそ、まずは触ってみる

Claude Codeは、非エンジニアにとって少し取っ付きにくいかもしれません。  
黒い画面も出るし（まあアプリを使えば良いのだけれど）、エラーも出るし、何を確認されているのか分からない場面もあります。  
しかしながら、そこで１つずつ訊きながら進めれば、非エンジニアでも意外とサクサク動かせます。
