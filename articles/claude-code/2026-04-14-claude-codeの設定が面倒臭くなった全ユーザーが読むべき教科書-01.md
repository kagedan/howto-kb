# はじめに

Claude Code を日々使っていると、プロジェクトごとに **CLAUDE.md** や **settings.json** を毎回ゼロから書くのが面倒になりませんか？

「前のプロジェクトで書いたいい感じの設定、どこいったっけ...」  
「新しいメンバーが入ったけど、設定ファイルの共有どうしよう...」

こうした課題を解決するため、**curl ワンライナーで Claude Code の設定テンプレート一式を導入できるリポジトリ** [claude-config](https://github.com/sanpicule/claude-config) を作りました。

<https://github.com/sanpicule/claude-config>

この記事では、なぜ作ったのか、何が便利なのか、そしてどう使うのかを紹介します。

# 解決したかった課題

Claude Code をチームで使い始めると、以下の課題に直面します。

| 課題 | 具体的な困りごと |
| --- | --- |
| **設定の属人化** | 人によって CLAUDE.md の書き方がバラバラ。ベストプラクティスが共有されない |
| **初期設定の手間** | 新しいプロジェクトを始めるたびに、CLAUDE.md / settings.json / .mcp.json を手作業で作成 |
| **セキュリティの抜け漏れ** | `.env` の読み取り禁止設定を入れ忘れるなど、守りの設定が漏れがち |
| **オンボーディングコスト** | 新メンバーに「まず CLAUDE.md を書いて」と言っても、何を書けばいいかわからない |

# claude-config でできること

## curl ワンライナーで一発導入

プロジェクトのルートで以下を実行するだけです。

```
cd /path/to/your-project
curl -fsSL https://raw.githubusercontent.com/sanpicule/claude-config/main/scripts/setup-project.sh | bash -s .
```

これだけで、以下のファイルが配置されます（**既存ファイルはスキップ**されるので安全です）。

```
your-project/
├── CLAUDE.md                     # プロジェクトルール
└── .claude/
    ├── settings.json             # チーム共有設定
    └── settings.local.json       # 個人ローカル設定
```

## テンプレートの中身

### CLAUDE.md ── 「何を書けばいいか」がわかるテンプレート

CLAUDE.md は Claude Code の動作に大きく影響するファイルですが、白紙から書くのは意外と大変です。

claude-config のテンプレートでは、**よく使うセクションがコメント付きで用意**されています。

```
## 概要
<!-- 例: 社内向けの在庫管理 SaaS。管理者が商品マスタと発注履歴を管理する Web アプリ。 -->

## 言語・フレームワーク
<!-- 例:
- 言語: TypeScript 5.x (strict: true)
- フロントエンド: Next.js 14 (App Router) / React 18 / Tailwind CSS
- バックエンド: Hono / Prisma
-->

## コーディング規約
<!-- 例:
- フォーマッタ: Prettier (設定は .prettierrc)
- 命名: コンポーネントは PascalCase、hooks は use から始める
- any 禁止。unknown + 型ガードを使用。
-->
```

コメントを外して自分のプロジェクトに合わせて書き換えるだけで、実用的な CLAUDE.md が完成します。

### settings.json ── 権限設定のカタログ

Claude Code の `settings.json` は、`deny` / `ask` / `allow` の 3 段階で権限を制御できます。しかし、どんな設定項目があるのかドキュメントだけでは把握しにくいのが実情です。

claude-config のテンプレートには、**よく使う権限設定がコメント付きで網羅的に列挙**されています。

```
{
  "permissions": {
    "deny": [
      "Read(./.env)",              // .env の直接読み取りを禁止
      "Read(./.env.*)",            // .env.local 等も禁止
      // "Bash(rm -rf:*)",         // rm -rf を禁止 (破壊的)
      // "Bash(git push --force:*)", // force push を禁止
      // "Bash(terraform apply:*)", // インフラ適用を禁止
    ],
    "ask": [
      // "Bash(git push:*)",       // push 前に確認
      // "Bash(npm install:*)",    // 依存追加時に確認
    ],
    "allow": [
      // "Bash(npm run:*)",        // npm scripts を許可
      // "Bash(git status:*)",     // git status を許可
    ]
  }
}
```

必要な行のコメントを外すだけで、プロジェクトに合った権限設定が完成します。**セキュリティ的に最低限守るべき `.env` の読み取り禁止はデフォルトで有効**にしてあります。

# 使い方の詳細

!

以下のコマンドは `curl | bash`（パイプ to bash）を利用しています。これは外部から取得したスクリプトを中身を確認せずに実行する形になるため、セキュリティ上の懸念があります。

本記事のスクリプトは公開リポジトリ（[sanpicule/claude-config](https://github.com/sanpicule/claude-config)）で誰でも中身を確認できますが、**不安な方は後述の「[スクリプトを確認してから実行](#%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%97%E3%83%88%E3%82%92%E7%A2%BA%E8%AA%8D%E3%81%97%E3%81%A6%E3%81%8B%E3%82%89%E5%AE%9F%E8%A1%8C)」の手順を利用してください**。チーム導入時も、必ず fork して内容をレビューした上で使うことを推奨します。

## 基本の導入（推奨）

```
# プロジェクトルートから実行
cd /path/to/project
curl -fsSL https://raw.githubusercontent.com/sanpicule/claude-config/main/scripts/setup-project.sh | bash -s .
```

## パスを指定して実行

```
# 任意のディレクトリから対象を指定
curl -fsSL https://raw.githubusercontent.com/sanpicule/claude-config/main/scripts/setup-project.sh | bash -s ~/Documents/my-app
```

## スクリプトを確認してから実行

パイプ to bash に抵抗がある方はこちら。

```
curl -fsSL https://raw.githubusercontent.com/sanpicule/claude-config/main/scripts/setup-project.sh -o /tmp/setup.sh
less /tmp/setup.sh       # 中身を確認
bash /tmp/setup.sh .     # 納得したら実行
```

## 導入後のカスタマイズ手順

1. **CLAUDE.md** を開き、各セクションのコメントを自分のプロジェクトに合わせて書き換える
2. **settings.json** を開き、有効にしたい権限設定のコメントを外す
3. **settings.local.json** に個人用の設定を書く（`.gitignore` で除外推奨）

# チームで使うための工夫

## fork して自分のチーム用にカスタマイズ

デフォルトのテンプレートを自チーム向けに作り込みたい場合は、リポジトリを fork して使えます。

```
# 環境変数で参照先を切り替え
CLAUDE_CONFIG_REPO=your-org/claude-config \
  curl -fsSL https://raw.githubusercontent.com/your-org/claude-config/main/scripts/setup-project.sh | bash -s .
```

| 環境変数 | デフォルト | 用途 |
| --- | --- | --- |
| `CLAUDE_CONFIG_REPO` | `sanpicule/claude-config` | 参照するリポジトリ |
| `CLAUDE_CONFIG_BRANCH` | `main` | 参照するブランチ |

これにより、チーム独自のルール（コーディング規約、禁止コマンド、MCP サーバー設定など）をテンプレートに焼き込み、全メンバーが同じ基盤から始められます。

## 既存ファイルは上書きしない

セットアップスクリプトは**既存ファイルをスキップ**する設計です。すでにカスタマイズ済みの CLAUDE.md がある場合も安心して実行できます。

```
$ bash setup-project.sh .
=== プロジェクト .claude セットアップ ===
スキップ（既存）: CLAUDE.md        # 既にあるファイルはスキップ
作成: .claude/settings.json       # 無いファイルだけ作成
作成: .claude/settings.local.json
セットアップ完了
```

# 設計で意識したこと

## 「そのまま使える + すぐカスタマイズできる」バランス

テンプレートは **最小構成をデフォルトで有効** にしつつ、**よく使う設定はコメントアウトで例示** する方針にしました。

* 白紙から書く苦しみがない
* かといって不要な設定が有効になっていて困ることもない
* コメントを読むだけで「こんな設定もできるのか」と学びになる

## セキュリティファースト

`.env` ファイルの読み取り禁止は**デフォルトで有効**にしています。Claude Code は非常に強力なツールですが、うっかりシークレットを読み取ってしまうリスクは見落としがちです。

# こんな人におすすめ

* Claude Code を使い始めたが、CLAUDE.md に何を書けばいいかわからない
* プロジェクトごとに毎回設定ファイルを書くのが面倒
* チームで Claude Code の設定を統一したい
* セキュリティ設定の抜け漏れを防ぎたい

# おわりに

Claude Code は設定次第で大きくパフォーマンスが変わるツールです。しかし、その設定をゼロから書くのはハードルが高い。

claude-config は「とりあえずこれを入れておけば、いい感じの設定が手に入る」という体験を目指して作りました。
