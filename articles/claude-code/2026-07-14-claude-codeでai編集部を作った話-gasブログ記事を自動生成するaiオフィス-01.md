「ネタ探し→リサーチ→下書き」って、書くこと自体より時間がかかりませんか。

私はGoogle Apps Script(GAS)やAppSheet、Looker StudioまわりのTipsをブログに書いているのですが、正直ここが一番の律速でした。書きたいテーマは頭の中にあっても、「今どんな記事が読まれているか」「まだ書かれていない切り口は何か」を毎回調べ直すのがしんどい。

そこで、Claude Codeを使って\*\*3人のAIエージェントが分業でネタ探し〜下書き作成をしてくれる「AIオフィス」\*\*を作ってみました。この記事では、その全体の仕組みと、実際に手を動かして構築するところまでを解説します。

※あくまで簡易的なAIオフィスです。（オリジナルでカスタムを楽しんでください！）

![](https://assets.st-note.com/img/1783971407-JZ3r47deAUnzNqX8IO9H2RQK.png?width=1200)

作れるAIオフィスのレイアウト

## 何を自動化して、何を自動化していないか

先にゴールをはっきりさせておきます。

投稿の自動化は意図的にやっていません。多くのブログ・記事プラットフォームには公式の投稿APIが用意されていないことが多く、それを非公式な手段(内部APIの解析やブラウザの自動操作)で突破するのは規約的にもグレーです。なので、このオフィスの仕事は「Googleドライブに完成した下書きを置くところまで」。最終確認と実際の公開は、必ず自分の目と手で行います。

## 全体のワークフロー

![](https://assets.st-note.com/img/1783936017-5cwhJpMXgea9Wb26yoAnVxPm.png?width=1200)

各担当が役割を担っていてドラフト記事をドライブへ

登場するのは3人のAIエージェントです。それぞれClaude Codeの「サブエージェント」機能で作っています。

名前役割やること**Aoi**リサーチ担当GAS関連の人気記事・最新トレンドを調査し、ブリーフを作成**Ren**執筆担当ブリーフをもとに記事の下書きを執筆**Yui**編集担当人気記事の模倣になっていないかチェックし、独自性を高めて最終化。Googleドライブへの格納まで担当

流れはシンプルです。

```
自分
  ↓ 「新しい記事のネタを探して下書きまで作って」と指示
Claude Code
  ↓
Aoi(リサーチ) → Ren(執筆) → Yui(独自性チェック + Drive格納)
  ↓
Googleドライブに「記事タイトルのフォルダ + 本文入りGoogleドキュメント」が自動生成
  ↓
自分が中身を確認して、ブログに手動で貼り付けて公開
```

さらに、localhost で開けるドット絵風の「AIオフィス」ダッシュボードを作ってあり、Aoi・Ren・Yuiが今どんな作業をしているかをリアルタイムに眺められるようにしています。エージェントがそれぞれ「稼働中」「会議中(チェック中)」「待機中」のゾーンをうろうろするだけの見た目上のおまけ機能ですが、地味に「ちゃんと動いてる感」があってモチベーションが上がります。

## 事前準備:Googleドライブと連携するためのGCP設定

![](https://assets.st-note.com/img/1783936196-mhpfnjD1ev0wMG2BW3kZocHr.png?width=1200)

このオフィスの肝は、完成した下書きを**Googleドライブに自動でフォルダ+ドキュメントとして格納する**部分です。これにはGoogle Cloud Platform(GCP)側での設定が必要になります。

最初、私は「サービスアカウント」という方式で試したのですが、これが個人のGmailアカウントだとうまく動きませんでした。サービスアカウントには個人アカウント経由だとGoogleドライブのストレージ容量が割り当てられず(0バイト)、空のフォルダは作れても、中身入りのファイルが作成できないというエラーにハマったのです。法人のGoogle Workspaceであれば「共有ドライブ」で回避できるのですが、個人のGmailでは共有ドライブ自体が使えません。

そこで\*\*OAuth2方式(自分のGoogleアカウントで都度認可する方式)\*\*に切り替えたところ、あっさり解決しました。以下、その手順です。

### 1. GCPプロジェクトを作成する

1. [Google Cloud Console](https://console.cloud.google.com/)にアクセスし、対象のGoogleドライブを持っているアカウントでログインする
2. 画面上部のプロジェクト選択メニューから「新しいプロジェクト」を選ぶ
3. プロジェクト名を入力(例: ai-office)して作成する
4. 作成後、画面上部でそのプロジェクトが選択されていることを確認する

### 2. Google Drive APIを有効化する

1. 左上のメニュー(三本線)→「APIとサービス」→「ライブラリ」
2. 検索欄に「Google Drive API」と入力して選択
3. 「有効にする」ボタンを押す

### 3. OAuth同意画面を設定する

1. 「APIとサービス」→「OAuth同意画面」(新しいUIでは「Audience」と表示される場合あり)
2. User Typeは「外部」を選ぶ
3. アプリ名(例: ai-office)、ユーザーサポートメール、デベロッパーの連絡先メールを入力して保存
4. スコープの設定画面はそのままスキップしてよい
5. **「テストユーザー」に、実際に使う自分のGoogleアカウントのメールアドレスを追加する**

ここが最大のハマりどころでした。テストユーザーへの追加を忘れると、後で認可しようとしたときに「アクセスをブロック: Googleの審査プロセスを完了していません」というエラーになります。忘れずに登録してください。

### 4. OAuthクライアントID(デスクトップアプリ)を作成する

1. 「APIとサービス」→「認証情報」→「認証情報を作成」→「OAuthクライアントID」
2. アプリケーションの種類は「デスクトップアプリ」を選ぶ
3. 名前は何でもよい(例: ai-office-cli)
4. 「作成」を押すと client\_id と client\_secret が画面に表示される

### 5. クライアント情報(JSON)をダウンロードして配置する

1. 作成したクライアントの一覧から、ダウンロードボタンを押す
2. ダウンロードされたJSONファイルを oauth-client.json にリネームする
3. プロジェクトのルート(server.js と同じ階層)に配置する

### 6. 格納先のGoogleドライブフォルダを用意する

下書きの格納先にしたいフォルダをGoogleドライブ上に作成し、そのフォルダIDを控えておきます(URLの folders/ の後ろの文字列です)。

OAuth方式では**ログインしたアカウント自身のドライブとして動作する**ため、サービスアカウント方式のときのような「フォルダを個別に共有する」作業は不要です。自分がもともとアクセスできるフォルダであれば、そのまま使えます。

### 7. 初回の認可を行う

プロジェクトのルートで下書きアップロード用のスクリプトを一度実行すると、ターミナルに認可用のURLが表示されます。それをブラウザで開き、対象のGoogleアカウントでログイン・許可すると、oauth-token.json が自動生成され、以降はこのファイルを使って自動的に認証されるようになります(トークンの有効期限が切れても refresh\_token により自動更新されます)。

「このアプリは確認されていません」という警告が出た場合は、「詳細」→「(アプリ名)に移動」→「許可」と進めれば大丈夫です(自分で作ったアプリなので問題ありません)。

ここまでがGCP側の準備です。地味な作業ですが、一度終わらせてしまえば以降はノータッチで動き続けます。

ここから先は、実際にClaude Codeでこの「AIオフィス」を構築していく手順です。プロジェクトの設定ファイル(CLAUDE.md)、3人のサブエージェントの定義、ダッシュボードのサーバー、Googleドライブへの自動アップロードスクリプトまで、**そのまま使えるファイル一式をこの記事の下に添付**しています。ダウンロードしてご自身のフォルダに置くだけで動かせる状態にしてあります。
