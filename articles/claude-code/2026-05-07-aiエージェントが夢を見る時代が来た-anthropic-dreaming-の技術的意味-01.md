スクロールしていた手が止まった。

Anthropic が5月6日に発表した Managed Agents の新機能リストを読んでいたとき、「Dreaming」という単語が目に入った瞬間のことだ。

マルチエージェントオーケストレーション、Outcomes による自動評価——どちらも重要な機能だ。でも Dreaming は違う種類の驚きだった。エージェントが「何をするか」ではなく、エージェントが「何であるか」という問いに触れていたから。

---

## 「夢を見る」とはどういうことか

人間が眠っている間、脳は日中の経験を整理して長期記憶に定着させると言われている。Dreaming はそれに近い比喩だ——ただし、比喩以上のものでもある。

技術的に言えば、Dreaming はセッションとセッションの間に走るスケジュールされたバックグラウンドプロセスだ。エージェントは過去の会話トランスクリプトを読み返し、パターンを抽出し、自分のメモリストアを更新する。

* 繰り返しているミスは何か
* うまくいったアプローチはどれか
* 見落としていたエッジケースはあるか

これを、人間が指示しなくても、自律的にやる。

従来のエージェントメモリは受動的だった。「これを覚えておけ」と明示的に言われたことだけを記録する。Dreaming は能動的だ。エージェント自身が「何を覚えておくべきか」を判断する。

---

## なぜこれが面白いのか

単一セッションには構造的な盲点がある。

サポートエージェントが今月12回同じ分類ミスをしていたとしても、各セッションはそれを知らない。毎回、白紙から始まる。Dreaming はそのクロスセッションのパターンを初めて可視化する仕組みだ。

自律性には2つのモードがある：

**Automatic モード**：エージェントが判断してメモリに直接書き込む。人間のレビューなし。

**Human Review モード**：エージェントがメモリ更新の「提案」を出す。人間が承認してから反映される。

どちらを選ぶかは、そのエージェントにどれだけ自律性を与えるかの設計判断だ。

---

## Outcomes との組み合わせが鍵

Dreaming 単体では「何が良い出力か」がわからない。そこで Outcomes が必要になる。

Outcomes は開発者が定義した成功ルーブリックに照らして、エージェントの出力を自動評価する仕組みだ。評価は**エージェントの推論から完全に隔離された別の Claude インスタンス**が行う。出力が基準を満たさなければ、何を変えるべきかを特定して、エージェントが再試行する。

Anthropic の内部テスト数値：

* タスク成功率：標準プロンプティング比 **最大10ポイント向上**
* .docx 生成：**+8.4%**
* .pptx 生成：**+10.1%**

Outcomes が「試験」で、Dreaming が「エラーノート」だとすると、この2つが組み合わさることで初めてフィードバックループが閉じる。

```
実行 → Outcomes で評価 → 失敗を記録 → Dreaming でパターン化 → 次回に活かす
```

## マルチエージェントオーケストレーションの現在地

3つ目の機能、マルチエージェントオーケストレーションはパブリックベータに移行した。

コーディネーターエージェントがタスクを分解し、最大20のスペシャリストサブエージェントに並列委譲する。各サブエージェントは独自のコンテキストウィンドウを持ちながら、共通のファイルシステムを共有する。

実際の事例がいくつか公開されている：

* **Harvey**（リーガルAI）：タスク完了率が約6倍
* **Wisedocs**（文書検証）：品質維持しながらレビュー速度50%改善
* **Netflix**：数百のビルドログを並列バッチ分析
* **Spiral by Every**：Haiku コーディネーター + Opus ライティングサブエージェント + Outcomes グレーダーの組み合わせ

CrewAI や LangGraph のようなフレームワークと比較したときの Anthropic の主張は「インフラを我々が管理する」という点だ。プロセス管理、障害回復、コンテキスト同期、タイムアウト処理——これらを Anthropic が担い、開発者は「各エージェントが何をするか」の定義に集中できる。

---

## 「蓄積するシステム」という新しい概念

ここで少し立ち止まって考えたいことがある。

3ヶ月稼働したエージェントと、今日デプロイしたエージェントは、同じプロンプトを持っていても、もはや同じシステムではない。前者は3ヶ月分の自己キュレーションされた経験を持っている。

これは**ステートレスなツール**から**蓄積するシステム**へのシフトだ。

エージェントの競争優位は、もはやプロンプトの巧みさやモデルの性能だけではない。どれだけ長く稼働し、何を学んだか——その「歴史」が差になる。

これは少し怖い話でもある。エージェントが自律的にメモリを更新するということは、意図しない方向に「学習」する可能性もある。Human Review モードが存在する理由はそこにある。

---

## 制限事項と現実的な注意点

Dreaming はまだリサーチプレビューだ。GA ではない。

Managed Agents は Claude モデル専用だ。コスト最適化や特定タスクへの適性から複数モデルを使いたい場合、この層は別途解決する必要がある。

オーケストレーション深度は1レベルに制限されている（サブ-サブエージェントは不可）。

料金は $0.08/アクティブセッション時間 + 標準トークンコスト。アイドル時間は無料。
