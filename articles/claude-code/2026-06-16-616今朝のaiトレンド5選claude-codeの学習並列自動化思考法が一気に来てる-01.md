最近のタイムライン、新機能の速報そのものより「で、これをどう日常業務に組み込むの？」って話が一気に増えてる。Claude Codeをゲーム感覚で学ぶRPG、計画をCodexに渡して並列で実装する手法、262個のスキルが詰まったOSS、ノーコードで情報収集を自動化する話、そして「AIで資料を量産できる今こそ仮説が先」っていう思考法まで。

Xのスレッドでは1トピック数行でサクッとまとめたけど、正直それじゃ伝えきれない背景や「自分ならどう使うか」がたくさんあった。この記事ではそこを一段深掘りしていくよ。気になったところだけ拾い読みでもOK。

## 1. ゲーム感覚でClaude Codeを学べる学習RPG

「Claude Code Quest」っていう学習サイトが話題になってた。一言でいうと、Claude Codeをゲーム感覚で攻略しながら覚えられる「AIディレクター育成RPG」。

中身がよくできてて、疑似ターミナルにコマンドを打ち込んで敵を倒す→ステージをクリアするとClaude Codeの操作が自然と身につく、っていう構成になってる。間違えるとAIがヒントをくれるから、詰まっても進める。Webディレクターや非エンジニアが「楽しく手を動かすうちに勘所を掴む」ことを狙った設計なんよね。

ここがポイントだと思ってて、非エンジニアにとってClaude Codeの最大の壁って「機能」じゃなくて「最初のとっつきにくさ」っていう心理面なんよね。黒い画面にコマンド打つって、それだけで身構える人がめっちゃ多い。だからこそ、ゲームっていう入口は理にかなってる。

作った人も言ってたけど、チャット型のChatGPTやGeminiと決定的に違うのは「自分のPCを実際に操作してくれる体験」。これを最初に体感できるかどうかで、その後の伸びが変わる。

**実践ポイント**：もし社内でAI研修やチーム導入をやるなら、最初のオンボーディング教材として相性が良さそう。ただし今はまだウェイトリスト段階で即利用はできないから、登録だけ済ませて待っておく感じ。「うちのチーム、Claude Codeに腰が引けてる人が多いな」って人はチェックしておくといいよ。

## 2. /handoffで計画をCodexに渡して並列実装する

これは副業で複数案件を抱えてる人ほど刺さると思う。海外のエンジニアが公開してた手法で、Claude Codeに計画を立てさせて、その計画を複数のCodexに並列で投げて同時に実装させる、っていう流れ。

カギになるのが `/handoff` っていうスキル。これは「受け手のセッションが必要とする情報を揃えた、構造化された引き継ぎドキュメント」を生成してくれるもの。要は、Claude Codeで考えた計画を、そのままCodexやCopilot、Gemini CLIが無改変で読めるバトンの形にしてくれるってこと。

具体的な流れはこんな感じ。

* まずClaude Codeで要件を深掘りして固める
* `/handoff` で実装プランを引き継ぎドキュメントとして書き出す
* git worktree（作業コピー）を案件の数だけ作る
* 各worktreeでCodexを起動して並列実装

こうすると、1人でも「複数の自分」が同時に手を動かしてる状態になる。実装の待ち時間で日が暮れる、みたいな悩みがかなり軽くなる。

ただ注意点もあって、これは何でも並列化すれば速くなる魔法じゃない。**肝は「曖昧さが少なくて、独立してて、並列化できるタスク」を見極めること**。仕様が曖昧なタスクを無理やり並列に投げると、各エージェントがバラバラの解釈で動いて逆に破綻する。

**実践ポイント**：いきなり大型案件で試すより、「お互いに干渉しない小さな修正」を2〜3個まとめて回すところから始めるのがいい。worktreeの環境構築は最初だけ少し手間だけど、一度組めば使い回せる。

## 3. 262スキル入りのCC環境OSS「ECC」

GitHubで21万スターを超えてる「ECC（Everything Claude Code）」っていうリポジトリが話題に。コーディング・リサーチ・セキュリティ・メディア向けに262個（版によっては270以上）のスキルが詰まった、エージェント環境の全部入りセットみたいな存在。Claude CodeだけじゃなくCodexやCursor、Gemini、Copilotなんかも横断で使えるのが特徴。

今回おもしろいのは「AgentShield」っていう機能。これは自分のClaude Code設定をスキャンして、脆弱性や秘密情報（APIキーとか）の漏れがないかチェックしてくれるもの。1,000件以上のテストと100超の静的解析ルールで、危険なgitコマンドのブロックや設定の改ざん防止までやってくれる。

あと地味にすごいのが「レッドチーム/ブルーチームモード」。3体のOpusがそれぞれ攻撃役・防御役・監査役になって並列で動いて、自分の設定の穴を洗い出す。

ただ、これ全部入れるのは正直おすすめしない。262個もスキルがあると管理コストが膨らむし、何が入ってるか把握しきれなくなる。

**実践ポイント**：全量導入じゃなくて「自分の業務に近いものだけ選んで入れる」のが現実的。たとえばまずはAgentShieldの秘密情報スキャンだけ試してみる、とか。`.claude` フォルダにいつの間にかAPIキーをベタ書きしてた…みたいな事故、けっこうあるあるなので、棚卸しのつもりで一回かけてみるといいよ。

ここまでの3つって、実は「Claude Codeをどう使いこなすか」で地続きなんよね。学んで（1）、並列で回して（2）、環境を標準化する（3）。次はちょっと毛色が変わって、非エンジニア寄りの自動化の話。

## 4. Google Workspace Studioで情報収集を自動化

税理士事務所向けに「税制情報チェックの自動化」を作った事例が共有されてた。使ったのはGoogle Workspace Studio。Gemini 3を搭載した、Googleのノーコード自動化ツール。

何をしたかというと、「過去24時間の税制情報を収集して要約して」ってGeminiに自然言語で頼むだけ。あとは結果をGoogle Chatに通知して、スプレッドシートに蓄積していく、っていう流れを全部自動で組んだ。

ここがWorkspace Studioの強みで、フォルダにファイルが追加されたとか、メールを受信したとかをトリガーにして、ドキュメント生成・メール送信・スプレッドシート記録・Chat通知・条件分岐まで自然言語で組める。n8nやMakeみたいな自動化ツールと違って、Googleのツール群にネイティブで統合されてるのが大きい。

考えてみると、業界ニュース・税制・競合の動きみたいな「定例の情報収集」って、毎回手でやると地味に時間を食うのに価値はそこまで高くない、自動化の費用対効果がいちばん出やすい領域なんよね。士業やバックオフィスの「定例リサーチ→通知→蓄積」はまさにハマる。

**実践ポイント**：注意点として、利用にはGoogle Workspace Business Standard以上のプランが必要。あと当然Google環境前提。逆にいうと、すでに会社でGoogle Workspaceを使ってる部署なら、わざわざ別の自動化ツールを契約するより導入が早い。「毎朝この情報だけは集めてる」って作業がある人は、まずそれ1個を自動化してみるのがおすすめ。

## 5. 仮説なき資料はハリボテ｜構想と作業を分ける

最後はツールじゃなくて思考法の話。「仮説なき資料はハリボテ」っていう投稿が伸びてた。

元コンサルの人が、前職で「いきなりPowerPointを作るな、まず仮説を作れ」って言われ続けたって話。仮説のない資料は、どんなに見た目が綺麗でも中身のないハリボテになる。だから「構想（=仮説を立てる）」と「作業（=資料に落とす）」をはっきり分けて、構想を固め切ってから作業に入る。これだけでアウトプットの生産性が劇的に上がる、と。

これ、AIの文脈で読むとめちゃくちゃ刺さる。だってAIを使えば、見た目の整った資料は一瞬で量産できるようになったから。逆に言うと、**仮説が曖昧なままAIに「いい感じの提案書作って」って投げると、綺麗だけど中身のないハリボテを高速で量産するだけ**になる。

ここまで紹介してきた1〜4が「作業を速くする」話だったのに対して、これは「作業の前の構想こそ人間の仕事」っていう、いい対比になってる。

**実践ポイント**：AIに資料を作らせる前に、一回手を止めて「で、この資料で何を主張したいんだっけ？」を自分の言葉で言語化しておく。箇条書きでもメモでもいい。その仮説をAIに渡してから作業させると、出てくるものの質が段違いになる。AIが優秀になればなるほど、この「何を作らせるか」を決める力が効いてくる。

## 今日のトレンドを俯瞰すると

今朝の5つ、並べてみると流れがはっきりしてる。

「学習（Claude Code Quest）」「並列運用（/handoff×worktree）」「環境の標準化（ECC）」「ノーコード自動化（Workspace Studio）」。新機能の速報じゃなくて、AIやエージェントを"どう日常業務に組み込むか"っていう運用レイヤーに話題が移ってきてる。

そして締めの「仮説なき資料はハリボテ」が象徴的なんよね。AIで成果物を量産できる前提に立ったうえで、改めて人間側の思考法を問う論考が伸びてる。ツールが賢くなるほど、「何をやらせるか」「どう組むか」っていう人間の設計力に価値が寄っていく。

つまり今のAI活用は「学習・並列・自動化・思考法」の4つで考えると整理しやすい。手を速くする3つと、その手前の構想力1つ。このバランスを意識しておくと、新しいツールが出てきても自分の中で位置づけしやすくなると思う。
