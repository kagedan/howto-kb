## Claude Code Agent View とは

<https://code.claude.com/docs/ja/agent-view>

Claude Code の Agent View は、複数のバックグラウンドセッションを一覧で監視・管理できる機能。

claude agents を実行すると専用の画面が開き、各セッションの状態確認、返信、アタッチ、停止、削除などをまとめて扱うことができる。

`2026年5月11日`に Research Preview として追加された。

機能については公式を見てもらえば終わる話なので、実際使ってみて自分が良いと思った箇所や、ここだけ抑えれば十分使える、といった部分だけを抽出して書こうと思う。

これから触ろうとしてる人は、まずはここだけ抑えたら良いかもしれない。

## Agent View は監視ダッシュボードとして使う

まず、起動コマンドから

以降、バックグラウンドで起動したセッションはこの一覧に追加される。セッションの状態確認、返信、アタッチ、停止、削除などをここで扱える。

このバックグラウンドというのが重要で、普通のセッションはAgentsの中に出てこないので注意。

![](https://static.zenn.studio/user-upload/9a0b1b7e6870-20260528.png)

見てもらうと分かるが、worktreeとかでの分割概念はこの中にはない。

あるのは、動いてるか、完了したか、インプット待ちかの３種類だ。

なので、あらゆるバックグラウンドエージェントがごちゃごちゃとした坩堝になっている。

しかし、安心して欲しい。

後続でそれを少しでも緩和する方法も書いた。

## 混沌としたダッシュボードに統一感を

各worktreeのセッションがごちゃまぜになったダッシュボードはカオスだ。

そこで少しでも統一感を出すために、prefixをつける。

セッション行に対してこれを押すだけでセッション名を変えることができる。

そこで、worktreeの名前をprefixとしてつけるだけでも分かりやすくなるだろう。

![](https://static.zenn.studio/user-upload/20a3838985f3-20260528.png)

さらにそこに追い討ちをかけるように、このコマンドで整理しちゃおう。

そもそもバックグラウンドでパッと行うものなのでタスクが終わったらすぐ消すのが良いだろう。

長いセッション自体は、各worktreeで常駐させるようにしてもいい。

そのための終わったバックグラウンドセッションはすぐに消すためのコマンドはこれだ。

ちなみに、1回目はエージェントを停止させるだけなので、セッションを消したい場合は2回押す必要があるので注意。

## 複数 worktree 運用

手短に説明する。

ここで注意なのは、`claude agents` を立ち上げる場所だ。

特定のプロジェクトで立ち上げると新しいセッションが立ち上がるのもその場所で立ち上がることになる。

つまり、project-1とproject-2というワークツリーがある場合、project-1でagentsを立ち上げてしまうと、新しいセッションを作るとproject-1の場所で立ち上がり、project-2で立ち上げるのが面倒になる。

そこで、worktreeがあるディレクトリの一つ親の階層からagentsを立ち上げる。

そして、このコマンドを使えば一撃だ。

```
@project-2 プロンプトプロンプトプロンプト
```

![](https://static.zenn.studio/user-upload/ef81ad0c936c-20260528.png)

これでproject-2で立ち上げることができる。

ghqとかと合わせるとかなり使い道が広がるだろう。

特定のユーザーや組織配下で立ち上げれば、そこに入ってるリポジトリ全てが対象になるのだから。

## 各セッションが何やってるのか分からない時

カオスとなってるダッシュボードで、各セッションが何をやってるか気になっちゃう人もいるだろう。

そんな時はこれだ。

![](https://static.zenn.studio/user-upload/529098bdbc7c-20260528.png)

そう、セッション行の上でspaceキーを押すだけで、画面を変えずに何やってるかを覗き見することができる。

これでわざわざセッションに入り込んで見に行く必要もない。

しかも、覗き見した後にそのまま追加プロンプトを投げ入れることができるのだ。

もはや神ダッシュボードである。

## カオスな環境にピンで終止符を打つ

当然、セッションをピン留めができる。

これで、お気に入りのセッションにピンを打ち込んで逃げられないようにしておこう。
