## 今日やったこと

Claude Codeにタイムパフォーマンス（タイパ）計算ツールを作ってもらいました。

指示はこれだけです。

```
作業時間と成果を入力したら、タイムパフォーマンス（タイパ）スコアを計算して表示するPythonツールを作ってください。複数の作業を比較できるようにしてください
```

## 実際に試した結果

**勉強（1時間10分）**

```
────────────────────────────────────────
  結果プレビュー

  作業名  : 勉強
  作業時間: 1時間10分
  成果    : 7/10
  難易度  : 4/10
  満足度  : 6/10
  タイパ  : 342.0 pt  ★★☆☆☆  ランク C
  コメント: やや低め。作業の見直しを検討しましょう
────────────────────────────────────────
```

**noteを書く（30分）**

```
────────────────────────────────────────
  結果プレビュー

  作業名  : noteを書く
  作業時間: 30分
  成果    : 8/10
  難易度  : 6/10
  満足度  : 7/10
  タイパ  : 1176.0 pt  ★★★★★  ランク S
  コメント: 最高のタイパ！時間を超えた成果です
────────────────────────────────────────
```

勉強よりnoteを書く方がタイパが高いという結果になりました。

## スコアの計算方法

```
タイパスコア = (成果 × 難易度 × 満足度) ÷ 作業時間(時間) × 100
```

短い時間で高い成果・高い満足度を得られるほどスコアが上がります。

## 結果を見て思ったこと

勉強のタイパがランクCだったのは、時間が長くて満足度が低かったからです。同じ勉強でも「集中できた30分」と「ダラダラした2時間」では全然違うスコアになるはずです。

noteはランクSでしたが、これは自分が楽しんでやっているからスコアが高く出ているだけかもしれない。満足度を高く入力すればスコアが上がる仕組みなので、自己評価の甘さがそのまま出る。

「タイパ」を数値化すると客観的に見えるけど、入力する数字が主観なので結局は自分の感覚次第だと気づきました。
