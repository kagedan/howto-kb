## 一言でいうと

2つのJSONを貼り付けるだけで、追加・削除・変更されたキーを色分けで表示するツールです。ブラウザだけで完結し、データはどこにも送信しません。

👉 <https://json-diff-tool-bvs.pages.dev>

devnestio（英語圏の開発者向け無料ツール集）の一つとして公開しています。

## 作った動機

APIのレスポンスが「昨日と今日で微妙に違う」「デプロイ前後で設定JSONが変わった気がする」──こういう場面は日常的にあります。ところが素のJSONを目視で比較するのは苦行です。テキストのdiffツールに突っ込むと、キーの並び順やインデントの違いだけで大量の差分が出てしまい、本当に知りたい「値の変化」が埋もれます。

欲しかったのは、テキストの行差分ではなく **構造としての差分** でした。キー単位で「これは追加」「これは削除」「これは値が変わった」を教えてくれるものです。

## 主な機能

左右に2つのJSONを貼り付けると、パースした上でキーをたどって比較します。ネストしたオブジェクトや配列も再帰的に潜っていき、`user.profile.name` のようなパス表記で差分を列挙します。追加は緑、削除は赤、変更は黄色でハイライトし、変更箇所は「旧値 → 新値」の形で並べて表示します。

入力が壊れたJSONの場合は、その場でパースエラーの位置を出すようにしました。差分を見る前に「そもそも貼り付けたものが正しいJSONか」で詰まることが多いためです。

## 技術的なポイント

フレームワークは使わず Vanilla JS で書いています。比較の中核は、両方のオブジェクトのキー集合の和をとり、それぞれのキーについて「片方だけに存在するか」「両方にあるが値が違うか」を判定する再帰関数です。配列は要素をインデックスで対応付けて比較しています。

値の等価判定では、オブジェクト同士は再帰、プリミティブは厳密等価（`===`）で比較します。ここを雑にやると `1` と `"1"` を同一視してしまい、型の変化という重要な差分を見逃すので気をつけました。

ホスティングは Cloudflare Pages。静的ファイルだけなのでビルドも不要で、当然データは一切サーバに送られません。JSONには機密情報が混ざりがちなので、クライアント完結は必須要件でした。

## AIエージェントの活用

このツールも Claude Code のセッションで組み立てました。「再帰diffのエッジケースを洗い出して」と頼むと、空オブジェクト・配列内オブジェクト・null と欠損キーの区別といった、後から地味にバグる箇所を先回りで指摘してくれます。特に `null` と「キー自体が存在しない」を区別する設計は、AIとの対話の中で固まった部分です。

## テスト結果

代表的なペア（設定ファイルのbefore/after、APIレスポンスのスナップショット同士）で試したところ、数百キー規模でも体感で瞬時に差分が出ました。ネスト5〜6段でも崩れず、型変化（数値→文字列など）もきちんと「変更」として拾えています。

## 今後の展望

配列の比較を、今のインデックス対応から「要素の中身ベースのマッチング」に進化させたいと考えています。要素の順序が入れ替わっただけなのに全件差分になるのは、まだ改善余地があります。あわせて、差分結果をパッチ形式（JSON Patch, RFC 6902）でエクスポートできると実用性が上がりそうです。
