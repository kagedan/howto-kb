## はじめに

**「Claude Code × 人材紹介」**連載の5本目です。  
※これまでの投稿は、下記のマガジンにまとめています。

うちの朝は、**Slackに「昨日の選考の動き」が全部並んでいるところから始まります。**  
誰がどの会社の書類を通過したか。面接はいつに確定したか。どの選考が次のステップに進んだか。夜のあいだにAIエージェントが処理して、出社する頃には、その日に見るべき選考の一覧ができあがっています。

**今日書くのは、この連載の中でも一番地味な業務です。**求人推薦でも面接対策でもなく、選考ステータスの管理。誰も「面白い」とは言ってくれない裏方ですが、  
**ここを仕組みにしたことで、「選考の全体像が毎朝はっきり見える」という、地味だけれど効く状態が手に入りました。**その話をします。

## 現場の違和感：「全員がやるべきで、誰もやりたくない」業務

選考ステータスの管理って、人材紹介の中で一番ほめられない作業だと思います。

候補者が10人いて、それぞれ3社、4社と並行で選考が進んでいる。  
書類が通った、面接が組まれた、二次に進んだ、残念ながら見送りになった——この一つひとつを、管理システムのどこかに正しく反映しておく。  
やったところで売上が増えるわけでも、候補者から直接感謝されるわけでもありません。

**しかも厄介なのは、サボっても今日は何も起きないことです。**  
更新を後回しにしても、その瞬間は誰も困らない。  
だから忙しい日は、つい一番下に沈んでいきます。面談が立て込んだ日、提出書類に追われた日、優先順位の底へスッと落ちる。

でも、これが溜まると、じわじわ効いてきます。いま誰がどの段階にいるのか、全体像がぼやけてくる。  
気づけば「あの人の二次、いつだっけ」「あの会社、まだ結果来てなかったっけ」と、頭の中で複数の選考を追いかけ続けることになる。  
**そして頭で追いかけている限り、候補者への対応は、少しずつ後手に回っていきます。**

![](https://assets.st-note.com/img/1782025013-3SXhsG5Ya6gRVEPKnQeLwMxv.png?width=1200)

## 課題の構造：選考の情報は、一か所には来ない

なぜ管理が追いつかなくなるのか。  
気合いや責任感の問題に見えて、**実は構造の問題**でした。

そもそも**選考の情報は、一か所には届きません。**  
企業から直接メールで来ることもあれば、求人シェアリングサービスを経由して通知が来ることもある。  
さらに「面接の日程はこれで」「先に条件面だけ確認させてください」といった、結果そのものではないやりとりも、同じ受信箱に混ざってくる。

**人間がこれを横断して、抜け漏れなく拾い続けるのは、構造的に無理があります。**  
経路が複数あると、人はそのうちのどれかを必ず見落とすからです。  
しかも見落としは、本人が気づけません。「見落としたこと」に気づけないからこそ、見落としなわけで。

結果として、**頭の中にある選考の全体像は、いつも少しだけ古い・少しだけ欠けた状態**になります。  
そのズレが、そのまま候補者対応の遅れになって表れる。気合いで挽回しようとしても、情報が散らばっている限り、いたちごっこです。

## AIで解けること：横断して読み、ステータスとカレンダーを揃える

いまは、Claude Codeで組んだエージェントが、こう動いています。

**サポート中の候補者全員について、AIが選考の連絡を毎日チェック**します。企業から直接届くメールも、提携サービス経由で届く通知も、横断して読む。  
そのうえで**候補者×企業ごとに、管理システムのステータスを更新**する。  
面接日が確定していれば、**カレンダーにも自動で登録**されます。

そして毎朝、**「昨日の選考の動き」をSlackに整形して共有**してくれる。  
誰がどこを通過し、どの面接が決まり、どの選考が動いたか。お見送りになった選考も、もちろんその中に並びます。  
バラバラの経路に散らばっていた情報が、毎朝ひとつのリストにまとまって、目の前に揃う。  
これが、**ずっとほしかった状態**でした。

外注のツールを買ってきたわけではなく、これもClaude Codeで自社の運用に合わせて組んでいます。  
だから「結果ではない条件確認のメールを、結果と取り違えない」とか、「同姓の別候補者を混同しない」とか、現場で実際にハマった細かい落とし穴を、その都度仕組みの側に教え込めます。  
汎用の管理ツールでは拾いきれない、うちの業務の癖の部分です。

## 人間が担うこと：「今日、誰に向き合うか」を決める

ここまでがAIの仕事で、ここから先が人間の仕事です。

AIは、選考の動きを漏れなく拾って、きれいに並べるところまでやってくれます。  
でも、**その一覧をどう使うかは人間の仕事です。**面接が近い人の対策に時間を取るのか、選考が思わしくない人の相談に乗るのか、状況が動いた人に今日のうちに連絡するのか。

私は朝のSlackを、業務の処理リストとしてだけでは見ていません。  
**「今日、誰に向き合うか」のリスト**として見ています。  
**全体像が一目で揃っているからこそ、限られた時間を、今日いちばん必要としている候補者に寄せられる。**  
情報を探すところからは始めなくていい。

**合理性はAIに、熱量は人間に。**  
選考の動きを漏れなく拾って揃えるという合理の部分はAIが引き受けてくれたから、**私は「誰に、どう向き合うか」だけを考えればよくなりました**。  
情報を追いかけることに使っていた時間が、人に向き合う時間に変わる。  
順番が、まるごと入れ替わった感覚があります。

## おわりに

選考ステータスの管理は、やってもほめられない、でも溜まると効いてくる、いわば「守りの業務」です。  
こういう、人の気合いに頼ると必ずどこかで穴が空く領域こそ、AIに渡すべきだと思っています。

派手な自動化ではありません。でも、毎朝「選考の全体像」が揃って見えるようになって、頭の中で選考を追いかける時間がなくなった。  
その分まるごと、候補者に向き合う時間に回せるようになったのは、人材紹介をやっている人間としては、地味に一番うれしい変化でした！
