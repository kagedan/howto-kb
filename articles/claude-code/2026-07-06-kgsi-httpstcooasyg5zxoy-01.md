Figmaを一切開かずに、LPを1本作りました。使ったツールは、主にClae DesignとClaude Codeだけ。制作期間は1週間、AIとのやり取りは合計300往復を超えています。

その300往復でわかったのは、作る行為はAIでずいぶん楽になった一方、**仕上がりの上限を決めるのは相変わらず人間の判断だ、ということでした。**候補を出すところまではAIが全部やってくれる。そこから狙ったトーンへ引き上げるのは、良し悪しを判断して差し戻す側の仕事でした。

作ったのは、制約・文脈・検証・評価の4層でデザインを扱うフレームワーク「[Design Harness](https://design-harness.com/)」のドキュメントサイトです。デザインを名乗るサイトである以上、表層やインタラクションの作り込みでどこまで行けるか、という裏テーマも持たせています。

どんな手順で作り、どこに300往復を費やしたのか。実際の画面キャプチャを交えて振り返ります。

# Claude DesignとClaude Codeを、どう使い分けたか

このLPは、主に2つの道具を役割で分けて作っています。[Claude Design](https://support.claude.com/ja/articles/14604416-claude-design%E3%82%92%E5%A7%8B%E3%82%81%E3%82%8B)は対話型のデザイン生成ツールで、チャットで指示すると、その場でライブHTMLのプロトタイプが出てきます。[Claude Code](https://code.claude.com/docs/ja/overview)はターミナルで動くコーディングエージェントで、コードベースを直接実装します。

使い分けは明快で、早い段階でプロトタイプを形にするのは、Claude Designが得意です。今回も、デザインの方針をまとめたドキュメントとワイヤーフレームを渡して、いくつかの質問に答えたら、最初のLPが出力できました。方向を試して壊す...という段階で、この作り方には効果があります。

一方、プロダクション品質まで持っていく作業、たとえばレスポンシブ対応やアクセシビリティのようなコードの細部は、Claude Codeで詰めていく必要があります。
**Designで意匠を決め、Codeで仕上げる。**この順番で使っています。

# 作る前に、やりたいことを全部言葉にした

このサイトを作ると決めた時点で、方向性は固まっていました。頭にあったのは「古典的な海外風のトーンに、モダンな書体や演出を足したらどうなるか」。コンセプト自体はDesign Harnessのドキュメントから引いています。

なお、画像はClaude単体では生成不可能なので、ChatGPT Images 2.0に任せました。サンプルにしたサイトイメージから別案を派生させ、作りたいサイトのムードボードを組み、背景画像も生成しています。**良しとする方向を、先に言葉と画像で全部出し切っておく。**以降の往復は、ぜんぶこれとの突き合わせでした。

# 磨き込みは何百往復も（最低）かかる

準備が済んでから、Claude Designとの往復が始まります。スクリーンショットを遡って数えたら、やり取りは合計で200回を超えていました（トータルの制作時間でいうと1週間ぐらい）。セッとの実測はこうです。
