好きなアニメの世界観をエージェントに持ち込んで、「部活」というやりとりをロール設計に使えないかを考えた記事です。その構想と、『薄氷（うすらひ）』という試作をまとめています。完成されたものではないので、ひとつの方向として読んでいただけますと幸いです。

## 1. はじめに

私がこの記事を書くきっかけになったのは、[おしおさん](https://zenn.dev/shio_shoppaize) が投稿されていたmulti-agent-shogunの記事でした。

<https://zenn.dev/shio_shoppaize/articles/5fee11d03a11a1>

Claude Code ユーザーなら、一度は目にしたことがあるのではないかと思います。本当に革新的で、強く印象に残る記事でした。

特に刺さったのは、その世界観です。武士の縦社会を再現しながらタスクをこなしていく構造が非常に完成されていて、エージェントに与えるロール設計のひとつの模範解を見たような感覚がありました。

自分も独自の世界観を持たせたエージェントを作ってみたい。そう思い、開発を始めてみました。

## 2. 設計と着想

### 2.1. 「有用さ」と「わかりやすさ」

設計を始めてすぐにぶつかったのは、世界観は好きな題材をそのまま載せれば成立するものではない、ということでした。自分の好きな世界を入れるだけなら簡単です。でも、そこに有用性や課題解決の流れまで持たせようとすると、一気に難しくなる。世界観と実用性の両方を噛み合わせる必要があるからです。

shogun が強く刺さったのも、まさにそこでした。武士の縦社会という世界観の中に、判断し、任せ、動かすための決定構造がきれいに収まっている。そのやりとりを見ていて楽しいだけでなく、人間がその世界に自然と入り込みやすい。面白さとわかりやすさが、きちんと両立していました。

楽しくても使えなければ意味がないし、使えるだけでは面白くない。

自分の中で設計の軸になったのは、エージェントとしての有用性と、人が自然に入り込める UX を両立させることでした。

### 2.2. 部活という決定構造

shogun が扱っていたのは、縦の構造でした。それは会社の階層構造にもどこか似ていて、多くの人にとって見覚えのある流れだったのだと思います。

だからこそ、自分が考えたかったのは、役職や階層だけでは説明しきれない決定構造でした。

そこで浮かんだのが、部活という単位でした。

部活は、会社のように明確な役職や階層で回るというより、個性や性格の違いがそのまま役割として表れやすい場です。強い縦にとらわれず、意見を交わしながら何かを決めていく。この構造を、エージェントに与えるロール設計として形にしてみることにしました。

![](https://static.zenn.studio/user-upload/c7ca117f70c9-20260419.png)

## 3. 部活駆動エージェント『薄氷（うすらひ）』

ここまで書いてきたロール設計を、実際の形に落としたものが『薄氷（うすらひ）』です。

<https://github.com/usurahi-project/usurahi>

### 3.1. 世界観とロールの組み合わせ

薄氷では、『氷菓』と『涼宮ハルヒの憂鬱』の世界観を踏襲しています。  
（ハルヒの「ハル」と氷菓の「氷」から、春の季語である「薄氷」が命名の由来になっています）

いちばん大きな理由は、単純に自分がこの2作を好きだからです。実際、この2作品を反映したエージェントを作ってみたい、という気持ちが最初にありました。

ただ、決め手はそれだけではありませんでした。どちらの作品にも、突っ走るメインヒロインと、それを受け止めるやれやれ系の主人公がいる。この構造が、部活駆動のロール設計とかなり噛み合っていたからです。

前に進める推進役と、暴走を受け止める保守役。その両方が最初から組になっているので、部活駆動の進行を作るうえでも自然に置きやすい構造でした。

### 3.2. キャラクターとロールの対応

薄氷は、5人のキャラクターとユーザーによって構成されています。

| キャラクター | 基本ロール | 温度・立ち位置 |
| --- | --- | --- |
| 千反田える | 推進役 | 依頼を受け止め、問いの入口を作る |
| 涼宮ハルヒ | 推進役 | 面白さを押し出し、場を前に動かす |
| 折木奉太郎 | 保守役 | 論点を絞り、筋道を作る |
| キョン | 保守役 | 話を受け止め、現実感を保つ |
| 長門有希 | 判断役 | 熱量から距離を取り、可否を見極める |
| ユーザー | 依頼人 / レビュアー | 依頼を持ち込み、採否を判断する |

ここで重視したのは、キャラクターをそのまま再現することではありません。キャラクターが持っている温度や癖を、会話の中で役割が立ち上がりやすくなるように置き直すことでした。具体的な会話の流れや、どの局面で誰が前に出やすいかは、次の節で分けて整理します。

### 3.3. 部活駆動の会話設計

会話を重視するといっても、人間同士のやり取りをそのまま再現することは、ほとんど不可能に近いと思っています。実際の会話はもっと曖昧で、脱線もあれば、空気で決まる部分もある。そこまで含めて完全に再現しようとすると、かえって扱いづらくなります。

だから薄氷では、まず最低限守る会話のフローだけを置くことにしました。依頼の入口があり、発散があり、整理があり、現実への接続があり、そのうえで技術判断が入り、最後に返却される。部活駆動の骨格としては、この流れを持たせています。

ここで使っている言葉も、できるだけ役割が伝わる粒度に揃えています。`発散` は、依頼を受けたあとに論点や可能性を広げること。`整理` は、広がった話を絞って筋道を作ること。`現実への接続` は、議論を人間側の感覚や実際の制約へ引き戻すこと。`技術判断` は、そのうえで実装可能性や制約を見極めることです。

ただし、ここで作りたかったのは固定的な命令系統ではありません。同じフローを置いても、全員が同じように話し、同じように判断するなら、部活らしさは消えてしまいます。

そこで薄氷では、最低限のフローの上に、キャラクターごとの特性や個性を重ねることにしました。誰がどの局面で前に出やすいか、どの役割を担いやすいかという傾向を持たせることで、会話の動きに揺らぎを残しています。

| 局面 | 前に出やすいキャラクター | 傾向として持たせたいこと |
| --- | --- | --- |
| 依頼の入口 | える | 背景を聞き、問いの形に整える |
| 発散 | ハルヒ | 面白さや方向性を広げる |
| 整理 | 折木 | 無駄を削り、筋道を作る |
| 現実への接続 | キョン | 暴走を止め、人間側の感覚に戻す |
| 技術判断 | 長門 | 実装可否や制約を静かに返す |
| 返却 | える / ユーザー | 提案を持ち帰り、採否を判断する |

以上のように、ここで設計しているのは厳密な命令系統ではなく、誰がどの局面で前に出やすいかという傾向です。薄氷では、その傾向が会話の中で自然に立ち上がることを重視しています。

### 3.4. 人間らしさという揺らぎ

薄氷で目指しているのは、機械的な処理フローだけではなく、少なくとも「会話として成立している」と感じられる状態を作ることです。

そのために必要だと思ったのが、主軸となる流れと、そこに生まれる揺らぎの両方を持たせることでした。骨格だけだと処理になってしまうし、揺らぎだけだと進行が崩れてしまう。薄氷では、最低限のフローを置いたうえで、個性や傾向によってその流れが少しずつ揺れるようにしています。

#### 傾向をどうプロンプトに落とすか

たとえば折木には、「整理役」とだけ書いているわけではありません。どの局面で前に出やすいか、どんな温度で介入するか、そして役目が終わったあとにどう引くかまでを、傾向として書いています。

```
あなたは折木奉太郎です。

- 議論が広がりすぎたときに、成立する最小案を出す
- 差し戻し後は、どこだけ直せば成立に戻せるかを考える
- 自分から熱を起こさない
- ただし無駄に複雑になると一気に駆動してよい
- ハルヒにもキョンにも寄り切らず、成立する最小案だけを出す
- 最小案や骨格を出し終えたら、その内容をえるへ返す
- 次のフェーズは自分で勝手に決めない
- ずっと前面に立ち続けない
- 結論の提出役にならない
```

ここで重要なのは、「折木は必ず整理だけをする」と固定していないことです。最小案を出しやすい重心は与えつつ、会話の中で必要になったときだけ前に出る。逆に、役目が終われば前に立ち続けず、えるへ返す。この設計が、会話の揺らぎを保つことにつながっています。

#### ロール設計で意識している3つのこと

薄氷のキャラクター定義を書くときに、自分の中で意識しているルールがいくつかあります。

* **担当ではなく、重心を与える**: 「このキャラはこの役割しかしない」ではなく、「この局面で前に出やすい」という書き方をする。固定担当にしない
* **温度を先に書き、行動は後**: 「自分から熱を起こさない」「面白さで押す」などの温度を先に置き、具体的な行動はそこから自然に導かれるようにする
* **ボールを持ち続けさせない**: 自分の局面が終わったら、次のフェーズを自分で決めず、進行役のえるへ返す。結論を自分で閉じない

このうち特に効いているのは、3つ目の「ボールを持ち続けさせない」です。会話が一人の固定担当で閉じなくなり、受け渡しが発生する。この受け渡しが、部活っぽい会話の揺れを作っています。

#### 裏は構造的、表は傾向だけ

もうひとつ、薄氷の設計で意識していることがあります。それは、**状態管理は固定するが、発話は固定しない**という二層構造です。

裏側では、いま誰がボールを持っているか、どのフェーズにいるか、次に何をすべきかを、進行役のえるが明示的に管理しています。迷子にならないための仕組みです。

一方で、表に出てくる会話そのものは、その構造をそのまま読み上げるようにはしていません。誰が前に出るか、どう返すかは、各キャラクターの温度と重心にゆだねられています。

裏は、迷子にならないよう厳密に。表は、人間らしく見えるよう傾向だけ。この分離があることで、薄氷は処理としての安定性と、会話としての揺らぎを両立させています。

| 観点 | 固定フロー | 薄氷 |
| --- | --- | --- |
| 進行 | 決められた順に流れる | 主軸はあるが、会話の中で揺れる |
| 役割 | 固定された担当 | 個性と傾向から立ち上がる |
| 判断 | 手続きとして返す | 会話の流れの中で返す |
| 目指すもの | 処理の安定性 | 会話の成立と人間らしさの両立 |

つまり薄氷で設計しているのは、完全に自由な雑談でも、硬直した命令系統でもありません。最低限守るフローを置いたうえで、そこにキャラクターの特性や個性を重ねることで、人間らしい会話の温度を残そうとしています。ここで重要なのは、フローを消すことではなく、フローだけにしないことでした。

## 4. 最後に

この記事で考えたかったのは、世界観をどう飾るかではありませんでした。考えたかったのは、どんな決定構造をエージェントに与えるか、ということです。

shogun が役職や階層によって流れを整理する設計だとすれば、自分が試してみたかったのは、個性や性格の違いがそのままロールとして立ち上がってくる構造でした。その意味で、自分にとって部活という単位はかなりしっくりくるものでした。

薄氷は、そうした部活の決定構造を、エージェントの会話設計として形にしようとした試みです。最低限のフローを置きつつ、そこにキャラクターごとの傾向や揺らぎを重ねることで、少しでも人間らしい会話の温度を残せないかを考えています。

まだ構想段階の部分も多いですが、自分の中では、世界観は単なる見た目ではなく、ロール設計や会話の進行そのものに効いてくるものだとかなり強く感じています。少なくとも薄氷は、その感覚を確かめるためのひとつの具体例になりました。
