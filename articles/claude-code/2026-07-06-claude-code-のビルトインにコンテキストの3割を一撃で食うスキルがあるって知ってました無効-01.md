Claude Code には、1回発動するだけで**約30万トークン**を消費するスキルが標準で入っています。現行の主要モデル(Fable 5 / Opus 4.8 / Sonnet 5)のコンテキストウィンドウは [1M トークン](https://platform.claude.com/docs/en/docs/about-claude/models/overview)ですから、**一撃で約3割**が消える計算です。

この記事では、実測の再現手順と、無効化する方法(公式設定と、公式ドキュメントに載っていない環境変数)をまとめます。

## スキルの発動は「ファイルの注入」

スキル(Agent Skills)は `/名前` で(または Claude が必要と判断したときに自動で)発動する拡張機能ですが、実体は「発動するとスキルのファイル内容が会話コンテキストに読み込まれる」仕組みです。読み込まれる量が大きいほど、コンテキストの消費も大きくなります。

Claude Code にはビルトインスキル(bundled skills)があり、公式ドキュメントは `/code-review`、`/loop`、`/claude-api` などを挙げています。このうち `/claude-api`(Claude API の網羅リファレンス)を実測します。

<https://code.claude.com/docs/en/skills>

## 実測: 発動あり/なしで約30万トークンの差

headless モード(`claude -p`)の JSON 出力で、`/claude-api` を発動した場合としない場合の入力トークン量を比較しました。

```
# 対照: スキルを発動しない
claude -p "Do nothing and reply with exactly: ok" --output-format json --max-turns 1

# /claude-api を発動する
claude -p "/claude-api" --output-format json --max-turns 1
```

出力 JSON の `usage` では入力が3つ(通常入力 `input_tokens`、キャッシュ新規書き込み `cache_creation_input_tokens`、キャッシュ再利用 `cache_read_input_tokens`)に分かれて計上されるので、合計をそのターンの入力総量(=コンテキストに載った量)として比べます。

| セッション | 入力トークン合計 |
| --- | --- |
| 対照(発動なし) | 61,367 |
| `/claude-api` 発動 | 359,843 |
| **差分(発動による注入)** | **298,476** |

対照の 61K は、システムプロンプトやスキル一覧の説明文、CLAUDE.md など、セッション開始時に常に入っているぶんです(環境で変わります)。注目は差分で、`/claude-api` と打っただけで**約30万トークンが一度に入りました**。入ったぶんは会話履歴として以後のターンにも残るので、そのセッションで使える空間が約3割減ったままになります。

実測の usage 抜粋(生データ)

対照(発動なし)

```
{
  "input_tokens": 2,
  "cache_creation_input_tokens": 61365,
  "cache_read_input_tokens": 0
}
```

/claude-api 発動

```
{
  "input_tokens": 2,
  "cache_creation_input_tokens": 336731,
  "cache_read_input_tokens": 23110
}
```

発動側の `cache_read_input_tokens` は、直前の対照実行と共通する部分がキャッシュ再利用されたぶんです。なお応答本文(`result`)には claude-api スキルの内容を受け取った旨が返っており、発動が成立していることも確認できます。

なお 200K コンテキストのモデル(Haiku 4.5 など)で発動すると "Prompt is too long" の API エラーになり、発動自体が成立しませんでした。

## 何が入っているのか

Claude Code が一時ディレクトリへ展開する claude-api の参照ドキュメント群を観察できます。私の環境(Linux)では `/tmp` 配下にありました(パスは環境やバージョンで変わる可能性があります)。

```
$ du -sh /tmp/claude-*/bundled-skills/*/*/claude-api
920K    /tmp/claude-.../claude-api

$ ls /tmp/claude-*/bundled-skills/*/*/claude-api
csharp  curl  go  java  php  python  ruby  shared  typescript
```

中身は言語別の API リファレンス一式で、合計約920KB。テキストは1トークンあたり3〜4文字程度なので、実測の約30万トークンと符合します。

やっかいなのは、注入サイズに上限をかける設定が見当たらないことです。MCP(外部ツール連携)のツール出力には `MAX_MCP_OUTPUT_TOKENS` という上限がありますが、スキル展開の同様の設定は公式の settings・skills・環境変数一覧の各ページには見当たりませんでした(2026年7月時点)。実測でも約30万トークンがそのまま入っており、できるのは次に見るスキル単位のオン/オフです。

## 対策1(documented): skillOverrides で個別に消す

[公式ドキュメント](https://code.claude.com/docs/en/skills)に載っている方法です。対話セッションで `/skills` を打つとメニューが開き、対象スキルに合わせて `Space` で切り替えると `.claude/settings.local.json` に保存されます。直接書くならこうです(全プロジェクトに効かせるなら `~/.claude/settings.json`)。

.claude/settings.json

```
{
  "skillOverrides": {
    "claude-api": "off"
  }
}
```

`"off"` にするとスキル一覧と `/` メニューの両方から消えます(手元でも claude-api が一覧から消えることを確認済み)。「一覧には残したいが、説明文ぶんのコンテキストは節約したい」なら `"name-only"` も選べます。

## 対策2(undocumented): 環境変数1つで消す

ここからが本題です。次の環境変数を設定して起動すると、**claude-api スキルだけが無効になります**。

```
export CLAUDE_CODE_DISABLE_CLAUDE_API_SKILL=1
```

この変数は公式ドキュメントの環境変数一覧・settings・skills の各ページに記載がありません(2026年7月時点)。出どころはコミュニティの非公式まとめで、"Disable built-in claude-api skill" として載っています。

<https://gist.github.com/mculp/e6a573f2a45ef7dbbf30f6a8574c7351>

対策1・3と違って公式の裏付けがない変数なので、本当に効くのかの検証手順ごと載せます。設定あり/なしの2セッションで、使えるスキル一覧を比較しました。

```
# 通常起動: 一覧に claude-api がいる
claude -p "List the names of all skills you can invoke via the Skill tool, one per line. Do not call any tools."

# 環境変数つきで起動: claude-api だけが消える
CLAUDE_CODE_DISABLE_CLAUDE_API_SKILL=1 \
claude -p "List the names of all skills you can invoke via the Skill tool, one per line. Do not call any tools."
```

私の環境(v2.1.201)での差分はこうなりました(並びを揃えて抜粋)。

```
-claude-api
 code-review
 dataviz
 deep-research
 loop
 run
 security-review
```

他のスキルに変化はなく、結果は対策1と同じです。普段づかいは公式設定で十分で、環境変数が活きるのは **settings.json を編集・配布できない場面**(CI、コンテナ、direnv でのプロジェクト別切り替え)です。env 1行で済みます。

## 対策3(documented): ビルトインスキルをまとめて消す

ビルトインスキル自体を使っていないなら、丸ごと無効化する公式設定もあります。

.claude/settings.json

```
{
  "disableBundledSkills": true
}
```

環境変数 `CLAUDE_CODE_DISABLE_BUNDLED_SKILLS=1` でも同じ効果です(公式の[環境変数一覧](https://code.claude.com/docs/en/env-vars)に記載があります)。手元の実測でも、ビルトインスキルだけが全部消え、プラグインや `.claude/skills/` の自作スキルは残りました。ただし `/code-review` など他のビルトインスキルも一緒に消えるので、claude-api だけが問題なら対策1か2が向きます。

## まとめ

* スキルの発動はファイルの注入。ビルトインの `/claude-api` は発動一発で約30万トークン(v2.1.201 実測)
* 現行モデルの 1M コンテキストの約3割が一撃で埋まる(200K のモデルでは発動自体がエラーになる)
* 個別に消すなら公式の `skillOverrides`、settings.json に触れない場面なら undocumented の `CLAUDE_CODE_DISABLE_CLAUDE_API_SKILL=1`
* ビルトインスキルを丸ごと消すなら `disableBundledSkills`(公式)

スキルの発動コストは中身のサイズ次第です。見慣れないスキルを試す前に「発動すると何がどれだけ入るのか」を意識しておくと、コンテキストが謎に減っていく事故を避けられます。
