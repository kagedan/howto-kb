Claude Codeを使い始めて最初に感じる壁がある。

「何をどう頼めばいいのかわからない」

これは慣れの問題でもあるが、いくつかのコツを知っておくと劇的に解消される。この章では、Claude Codeへの話しかけ方の基本を説明する。

## 大前提：日本語で話しかけていい

まずこれ。英語じゃなくて大丈夫。

Claude Codeは日本語を完全に理解する。「Pythonのスクリプトを作って」でも「エクセルみたいなファイルを読み込んで集計して」でも、日本語で伝えれば動く。

英語で話しかけた方が精度が上がる、というのは昔の話で、今は日本語でも問題ない。むしろ日本語で書いた方が自分の意図が正確に伝わる。

## コツ1：「ゴール」を先に言う

悪い例：「このファイルを開いて、3行目を見て、もし数字があったら……」  
良い例：「このファイルの3行目にある数字を取り出して、CSVに書き出して」

人間に頼むときと同じで、「最終的に何がほしいか」を最初に言う方がうまくいく。途中の手順をClaude Codeが自分で考えてくれる。

## コツ2：曖昧なままでも投げてみる

「うまく説明できないんだけど……」と躊躇する必要はない。

わからないまま投げると、Claudeが足りない情報を聞き返してくれる。むしろ「何がわからないかわからない」状態のときは、とりあえず思っていることを書いてみるのが正解。

例えば「なんか売上のデータをいい感じにまとめてほしい」と言えば、「どんなフォーマットで出力しますか？」「どの期間のデータですか？」と聞き返してくれる。それに答えていけば、自然と要件が固まっていく。

## コツ3：`!コマンド` でターミナルをその場で使う

Claude Codeのプロンプト（入力画面）で `!` に続けてコマンドを書くと、その場でターミナルコマンドを実行できる。

→ 今いるフォルダのファイル一覧が表示される

→ error.logの中身が表示されて、そのままClaudeに読ませられる

これの何が便利かというと、「エラーが出た、見て」と言う代わりに `! cat error.log` と実行するだけで、Claudeがログを直接読んで原因を分析してくれる。コピペの手間が省ける。

## コツ4：`@ファイル名` でファイルを直接参照させる

ファイルのパス（場所）を `@` に続けて書くと、そのファイルをClaudeが直接読んでくれる。

```
@src/app.py を見て、バグがないか確認して
```

```
@~/Documents/sales_data.csv のデータを集計して
```

チャット画面にファイルの中身をコピペする必要がなくなる。大きなファイルでも丸ごと参照できる。

## コツ5：段階的に頼む

一度に全部頼もうとすると、うまくいかないことがある。

「Webスクレイピングして、データを整形して、CSVに保存して、Googleスプレッドシートに転送して」——これを一発で頼むより、段階的に進める方が精度が上がる。

1. まず「このサイトから商品名と価格を取ってくるスクリプトを書いて」
2. 動いたら「このデータを整形して」
3. うまくいったら「CSVに保存して」

複雑なタスクほど、小さく切って確認しながら進めると失敗が少ない。

## コツ6：失敗したら `/clear` でリセットする

会話が長くなると、Claudeが文脈を誤解したまま進んでしまうことがある。「なんかおかしい方向に行ってる」と感じたら：

これで会話をリセットできる。最初からやり直す方が、修正を重ねるより早いことも多い。

## よく使うコマンド一覧

これはClaudeに「話しかける」というより、動作を制御するコマンドだ。

| コマンド | 何をするか |
| --- | --- |
| `/clear` | 会話をリセット（プロジェクト切替時に必須） |
| `/compact` | 長くなった会話を圧縮して続ける |
| `/plan` | 実装前に計画を立てて承認してから進む |
| `/cost` | 今セッションで使ったトークン（料金）を確認 |
| `Ctrl+C` | 生成を途中で止める |
| `Esc` | 一つ前の状態に巻き戻す |
| `Ctrl+B` | 重い処理をバックグラウンドで動かして別の作業をする |

## 実際の会話例

**ケース1: ファイルを分析してほしい**

```
@~/Downloads/sales_2026.csv を読んで、月別の売上合計を出して
```

**ケース2: エラーを直してほしい**

```
さっきのスクリプトを実行したらエラーが出た
! python3 scraper.py
上のエラーを見て原因を教えて、直して
```

**ケース3: よくわからないけど頼みたい**

```
毎月Excelで手作業でやってる売上集計を自動化したい。
今は売上データのCSVを開いて、月ごとに合計を出して、別のシートに転記してる。
これを自動でやる方法ってある？
```

## 「頼み方が下手でも大丈夫」が結論

Claude Codeは、あいまいな指示でも「たぶんこういうことですよね？」と補完しながら動いてくれる。

足りない情報は聞き返してくれる。間違って解釈していたら「違う、こっちの意味」と訂正すればいい。

うまく頼めなくて当然だし、最初は失敗して当然。`/clear` でリセットしながら少しずつ慣れていくのが一番の近道だ。
