# Node.jsでMCPサーバーを自作してClaude Codeと繋いだら、AIが「補完ツール」から「業務エージェント」に変わった話

## はじめに：AIが補完ツールから「エージェント」になる境界線

ちょっと聞いてみたいんですけど、Claude Code とか Cursor、毎日使っていますか。

使っている方、どんなふうに使ってます？　たぶん、コードの補完とか、エラーメッセージの説明とか、「これどう書けばいいか教えて」みたいな使い方が中心かなと思うんですよ。

それ、めちゃくちゃもったいないかもしれないんです。

誤解してほしくないのは、そういう使い方が悪いとか意味がないとか、そういう話じゃないですよ。コード補完だけでも、作業時間は普通に半分以下になりますから。ただ、そこで止まっているとすると、AIの使い方がたぶんまだ **第一段階** のままなんですよね。

**第一段階は「質問に答えてもらう」使い方**です。わからないことを聞く、コードを書いてもらう、エラーを直してもらう。完全に受け身のパターン。

**第二段階は「AIが実際の社内ツールやサービスと連携して動く」使い方**
