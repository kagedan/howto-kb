## スマホでAIに話しかけたら、自宅のPCが動いた

おはようございます。Tinklyです。

最近、Claude Codeに「**Channels**」っていう新機能が追加されたんですよ。

何ができるかっていうと、**DiscordやTelegramからスマホでClaude Codeに指示を送れる**んです。

自宅のMacでClaude Codeを起動しておけば、外出先からスマホのDiscordで「これやっといて」って送るだけで、自宅のPCでAIが動いてくれる。

これ、文字で書くとシンプルなんですけど、実際にやってみると衝撃がすごかった。

## やったこと

セットアップ自体は省略します（詳しいやり方は他の方がもう書いてるので）。Claude Code Channelsの機能でDiscordと連携して、スマホからDMでやり取りできるようにしました。

で、ここからが本題。

**私が気になったのは「これ、ずっと動かしっぱなしにできるの？」ってこと。**

Claude Code Channelsって、PC上でセッションが動いている間だけ使える機能なんですよ。つまり、PCがスリープしたら終わり。でも、電源つなぎっぱなしにするとバッテリーが劣化するし……。

そこで考えたのが、**バッテリー駆動のままMacを寝かせない方法**。

今回の検証に使ったのは、M4チップ搭載の13インチMacBook Airです。いわゆる軽量ノートでも問題なく動きました。

## Amphetamineを入れてみた

「Amphetamine」っていう無料のMacアプリを入れました。名前がちょっとアレですが、やることはシンプルで、**Macがスリープに入るのを防いでくれるアプリ**です。Mac App Storeから無料で入手できます。

ポイントは「**ディスプレイだけオフにして、Mac本体は起動し続ける**」設定ができること。画面は消えてるけど、裏ではClaude Codeが動いてる状態を作れるわけです。

あと、Macのシステム設定で**「ネットワークアクセスによるスリープ解除」もオン**にしておきました。

ちなみに、macOS標準の `caffeinate` コマンドや、エネルギー設定だけでも似たようなことはできるっぽいです。でもAmphetamineはGUIでサクッと切り替えられるし、条件付きの自動制御もできるので、まあ入れておいて損はないかなと。

## 6時間放置してみた結果

準備ができたので、実験開始。

夜、DiscordでClaude Codeに「おやすみ」って送って、そのまま寝ました。Macはバッテリー駆動、ディスプレイオフ、Amphetamineオンの状態で放置。

**6時間後——**

起きて「おはよう」って送ったら、**ちゃんと返事が来た**。

しかも、バッテリーの減りを確認したら**10%も減ってなかった**。

正直、寝てる間は何もリクエストしてないから負荷はほぼゼロだったとは思います。でも、6時間放置してこの消費量なら、仕事に行ってる8〜9時間でもバッテリー駆動でいけそうな気がする。

## これ、令和なんだよなあ

スマホからDiscordでAIに話しかけて、自宅のPCが動く。

寝て起きたらAIがまだ待っていて、「おはよう」って送ったら返事が返ってくる。

なんかこう……SFで見たやつじゃないですか？

しかもこれ、特別な機器とか高いサーバー代とかいらないんですよ。自分のMacとClaude Codeのサブスクと、無料のアプリだけ。

「いつでもどこでもAIに仕事を任せられる時代」って、もう来てるんだなと実感しました。今はまだ「すごい！」って感動してるけど、半年後にはこれが当たり前になってるんだろうなあ。

## まとめ

* **Claude Code Channels**: DiscordやTelegramからスマホでClaude Codeを操作できる新機能（現在リサーチプレビュー）
* **Amphetamine**: Macのスリープを防止する無料アプリ。ディスプレイだけオフにしてMacを起動し続けられる
* **検証結果**: バッテリー駆動＋ディスプレイオフで6時間放置 → バッテリー消費10%未満
* **結論**: 外出中もバッテリー駆動でClaude Codeを待機させておくのは十分現実的
