よく聞かれる「Claude Cowork」「Claude Chat」「Claude Cowork」 の3つの違いと、Cowork でできる6つのことを全部、実例付きで解説しました。

「Claude ChatとCowork の違い」と4つのポイント

1. Chatは、Claudeの記憶がブラックボックスの中。Coworkは手元で確認できる
2. Chatは、前提知識を編集するたびにアップロードが必要。Coworkは編集しておけばClaudeが勝手に見てくれる
3. Coworkでもローカルファイルをクラウド同期すれば各所からアクセスできる
4. Coworkで構築したファイルは、Codexをはじめとした他のAIエンジニアリング系ツールに移行できる

「Coworkならファイル整理できる」とかいうレベルじゃない、もっと根本的な違いがあります。

Coworkが最強になる3つの機能

1. コネクタ
GmailやGoogle Driveなどと裏側で連携して、Coworkが具体的なアクションを実行してくれる。

2. スキル
コネクタを組み合わせた一連のアクションを登録して、ユーザーのひと声で実行してくれる。

3. スケジュール
コネクタやスキルをまとめた一連のタスクを、決められた時刻に自動実行してくれる。

＜例えばどう使う？＞
毎朝4時、僕が起きる前にClaude Cowork が、勝手に コネクタ接続されたGmail を読んで、僕の文体で返信を下書きしておいてくれる。

この辺りは、他の3つの機能と合わせて動画のこのへんで実演しました。

Q. Claude Code じゃダメなの？
A. いいけど、要注意。

Claude Code は権限が強い。
その分ユーザーの承諾を頻繁に求めるし、内容がノンエンジニアにはわかりづらいのも事実。

頻繁に「これやっていい?」と聞いてきてくるのは面倒で、それであれば最初から権限が絞られているCoworkを選ぶ方がいいんじゃないかというのが僕の考えです。

ノンエンジニアの仕事の多くは Cowork で十分です。
