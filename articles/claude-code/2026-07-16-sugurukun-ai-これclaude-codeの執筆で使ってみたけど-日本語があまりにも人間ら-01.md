「AIの文章は正しいのに、なぜか読み返すと疲れる」の正体を、技術書の方が仕様書レベルで言語化。

読んで鳥肌立ったのがここで、
「なぜAIの文章は読んでて疲れるのか」の正体が全部言語化されてる：
ㅤ
・緩急は情報量じゃなく認知モードの切替（観察→逡巡→断定→再観察）で作る
・未回収の問いを常に1つ開いておく。全部答えた瞬間に読者は読むのをやめる
・断定だけで押し切らない。あとで裏切られる思い込みを布石として置く
・書き終えた後の機械的な点検手順まで定義。残していい文と削る駄文の判定基準つき
ㅤ
規範というより、そのままAIに渡して動く仕様書になってる...
