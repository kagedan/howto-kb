## この記事でわかること

Claude Codeで長めの作業を頼むとき、「続けて」「次は○○して」と何度も入力するのは、思ったより手間がかかる。ちょっとやるたびに止まって確認を求めてくるからだ。

Claude Code v2.1.139で追加された **/goalコマンド** を使うと、「こうなるまで続けて」という完了条件を1行書くだけで、条件を達成するまでClaude Codeが自律的に動き続ける。なお、/goalはCodex CLI v0.128.0（2026年4月30日）が先に実装し、Anthropicがその11日後にClaude Codeに追加した機能だ。

**ゴールを伝えたら、到達するまで止まらない。** もうターンごとに「続けて」と打つ必要はない。

この記事では、公式ドキュメントに基づいた正確な使い方と、効果的な条件の書き方、他の自動化手段との使い分けを解説する。

## /goalとは何か

**/goal** は、完了条件を設定すると、各ターン終了後に小型のチェック用AIが自動判定し、条件が満たされるまでClaude Codeが動作し続けるコマンドだ。

内部では「Haiku」という軽量なAIモデルが判定役を担っている。**目的地を設定すると、そこに着くまで走り続ける自動運転モード**だと思えばいい。

### 基本的な使い方

```
/goal all tests in test/auth pass and the lint step is clean
```

（訳: test/authフォルダ内の全テストが通過し、lintチェックもクリアになること）

これだけで、Claude Codeがテスト修正に着手し、各ターン終了時にHaikuが「テスト全部通った？チェッククリア？」を判定する。まだダメなら自動で次のターンに入り、OKなら自動停止する。

## auto modeとの違い

「auto mode」という別の自動化機能と混乱しやすいので整理する。

**auto mode (Shift+Tab)**

→ 1回の作業中の確認ポップアップを自動承認。Shift+Tabでpermission modeを切り替える

→ 停止条件: Claude自身が「完了」と判断

**/goal**

→ 作業を何度でも繰り返す継続

→ 停止条件: Haikuが条件達成を確認

**たとえるなら:**

**2つを合わせると完全な自律運転になる。**

## 3つの自律アプローチの使い分け

Claude Codeには「自動で動かす」方法が3種類ある。

**/goal**

→ 開始: ターン終了のたび → 停止: Haiku判定 → 向いている作業: テスト修正、整理作業、移行

**/loop**

→ 開始: 時間間隔ごと → 停止: 自分で止めるか、Claudeが完了と判断して停止。3日後に自動失効 → 向いている作業: 定期監視、ログチェック

**Stop hook（独自スクリプト）**

→ 開始: ターン終了のたび → 停止: カスタムスクリプト → 向いている作業: 独自の判定が必要な場合

## 効果的な条件の書き方（5つのコツ）

「ゴール設定のしかた」によって、うまく動くかどうかが決まる。

### 1. 測定可能な終了状態にする

```
# 良い例（判定できる）

/goal npm test exits with code 0 and npm run lint is clean

# 悪い例（判定できない）

/goal コードをきれいにして
```

「きれいにして」ではAIも判定AIも基準が分からない。「テストが全部通る」「エラーが0件」など、数えられる状態で書くのがコツだ。

### 2. チェック方法を明示する

```
/goal git status shows no modified files after running the formatter
```

判定AIは会話の内容とコマンド実行結果を見て判定する。Haikuはnpm testやtsc --noEmitなどのシェルコマンドの実行結果も読める。「何を実行した結果を見るか」まで書くと確実だ。

### 3. 変更禁止事項を入れる

```
/goal all auth tests pass without modifying any test files
```

「テストファイルは書き換えずに」という制約を入れることで、「テストの中身を変えてクリアさせる」という抜け道を防げる。

### 4. ターン上限を設定する（暴走防止）

```
/goal migrate all API calls to v2 format, or stop after 15 turns
```

「15回試してもダメなら止まる」という安全装置を入れると、AIが延々と試行を繰り返してトークンを消費し続けるリスクを防げる。トークンとはAIとの会話量の単位で、多いほど課金される。

### 5. 4,000文字以内で書く

条件は最大4,000文字。複雑なタスクは1つの明確な終了状態に集約するのがいい。
