私が、Gmail・Slack の未返信を毎朝 Slack DM に自動レコメンドする仕組みを作った記録です。Claude Code の RemoteTrigger ＋ MCP 接続で、朝起きたら今日返すべき案件一覧が DM で届く仕組みを構築しました。

---

## どんな状況だったか

毎朝、メールとSlackを開いて「あ、これ返してなかった」と気づく。あるいは「あれ、先方からまだ返事来てないな」と思いつつ、忙しくて流す。かといって毎朝GmailとSlackを全部遡って確認する時間もない。「朝起きたら、今日返すべきものが一覧で届いていたらいいのに」と思った。

**用語メモ**

## 扱ったデータ

* **データの中身**: Gmailの受信・送信メール、SlackのDM・メンション
* **ファイル形式**: API経由でリアルタイム取得（ファイル操作なし）
* **規模**: 過去5営業日のGmail、過去3日のSlack

## やったことの流れ

初回指示 → トリガー作成 → テスト実行 → フォーマット調整×8回 → 完成 → チームに共有

## 各ステップの具体的なやり方

### 最初の一言で全体像を投げる

> 「Gmail、Slackの私の未返信一覧を、毎朝自動レコメンドをする自動化してくれない？」

### 要件の追加

> 「自分が返信していないもの、相手が返信していないもの。返信不要な完了していると思われるものは除く。ニュアンスが難しければ随時育てる。端末が立ち上がっていなくても実行してほしい」

### 出力品質を60点→100点に引き上げる議論

> 「この出力を60点とします。100点とはどのようなものですか？100点にするために足りないものを列挙した後に、100点の答えを生成してください。行動につながるか？です」

### Slackフォーマットの試行錯誤（8回）

> 「フォントの斜体やめて」

「→の部分を‐＋インデントにしたら子階層に入るんじゃない？」「:exclamation: とかは？」「Gmail/Slackは分離、未返信と返信待ちは分離、重要度は何らか記号で可視化」

1. **最終形の確定**
2. **チームへの共有**

> 「このSkillファイルを、同僚あてにSlackで送って。何のためのものか、どう使うか、見れば簡単に設定と運用ができる状態にしてね。チャンネルは{社内チャンネル}」

## これで何が変わったか

* 毎朝GmailとSlackを巡回して未返信を探す作業がゼロになった
* 「返すべきもの」と「返事を待っているもの」が分離されて、頭の中が整理される
* 重要度アイコンで、朝の5秒で「今日まず何をやるか」がわかる
* PCを開く前にスマホのSlack通知で確認できる

## 時間の変化（ビフォーアフター）

**毎朝Gmail・Slackを15〜20分巡回して未返信を探す**

## なぜこの作業をAIにやらせるのか

### 人間がやると何が辛いか

* 「これは返信済みだっけ？」「これは相手待ちだっけ？」の判断が意外に脳のリソースを食う
* 忙しい日ほど確認をサボって、抜け漏れが発生する悪循環

## やってみてわかった注意点・学び

* **Slackのmrkdwn記法は罠が多い。** 太字 が斜体に見えたり、- のインデントが無視されたりする。プレーンテキスト + 絵文字コードが最も安定
* **「60点→100点」のフレームが強力。** 最初の出力を点数評価して差分を言語化させると、Claudeが自分で改善提案を出してくれる
* **フォーマットの調整が作業の7割。** ロジック（何を取得するか）はすぐ決まったが、「見やすい」「行動につながる」の調整に一番時間がかかった。ここは実機テストを繰り返すしかない
* **完了判定は「含める側に倒す」が正解。** 漏れるより多い方がマシ。使いながら除外ルールを足していくのが現実的
* **チーム共有はコピペプロンプトを添えると効果的。** 「これ貼って」で始められるプロンプト付きで共有すると、相手のハードルが一気に下がる

## サマリー

毎朝9時に、GmailとSlackの未返信をAIが自動チェックしてSlack DMで通知する仕組みを作った。「自分が返すべきもの」と「相手の返事を待っているもの」を分離し、重要度アイコン付きで一覧化。PCが起動していなくてもクラウドで動く。構築は90分。うち7割はSlackのフォーマット調整で、ロジック自体は最初の1回の指示でほぼ完成した。「60点の出力を100点にするには？」とAIに自己採点させるフレームが、アウトプット品質を引き上げるのに効いた。
