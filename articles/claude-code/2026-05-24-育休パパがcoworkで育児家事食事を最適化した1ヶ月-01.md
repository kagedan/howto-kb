## はじめに

今日は育休最終日です。明日から仕事復帰します。

育休中に自分なりに試行錯誤して最適化してきた日常の仕組みについて、書き残しておこうと思います。

育休に入って最初に感じたのは「時間があるようで、全然ない」という感覚でした。

我が家は**完全母乳**だったので、授乳は妻担当。その分、自分はおむつ替え・寝かしつけ・沐浴・家事・ご飯…と、授乳以外のことを全部やる、というスタンスで育休に臨みました。**「妻をできるだけサポートし、自分のできることを全うする」** をモットーに。

でも実際やってみると、授乳のタイミングに合わせて動く必要があったり、寝不足でぼーっとした頭で「今日何するんだっけ」を考えたりと、なかなかにカオスな日々でした。

育休期間を、ただ育児するだけでなく「AI のキャッチアップにも使いたい」と思っていました。エンジニアとして、AI ツールの進化についていくことは仕事復帰後にも直結する。でも本格的な開発をするような余裕はない。そこで目をつけたのが、日常生活そのものを AI で最適化することでした。

試し始めたのが **Cowork**（Claude のデスクトップアプリ）と、その中で自分でカスタマイズした **`start-daily` スキル** です。

この記事では、育休中という「予測不可能な毎日」でどう Cowork を活用し、何を最適化してきたか、特に `start-daily` スキルを中心に紹介します。

## Coworkとは

[Cowork](https://claude.ai/download) は、Anthropic が提供する Claude のデスクトップアプリです。

ただのチャットアプリではなく、**スキル**と呼ばれる独自のプロンプトセットを登録できるのが特徴。繰り返し行う作業をスキルとして定義しておくと、`/start-daily` のように呼び出すだけで複雑なタスクを自動実行してくれます。

Google カレンダーや Google Drive などの外部サービスとも **MCP（Model Context Protocol）** で連携できます。CLI や API を直接叩くのではなく、MCP を通じて Claude がサービスを操作してくれるイメージです。`start-daily` スキルでも、ピヨログデータの取得は Google Drive MCP 経由、カレンダー登録は Google Calendar MCP 経由で行っています。

### なぜ Claude Code ではなく Cowork を選んだか

普段の仕事では Claude Code（CLI ツール）をメインで使っています。コードを書く・リファクタする・調査するといった開発タスクは Claude Code が圧倒的に速い。

でも育休中の日常管理に Claude Code を使うのは、**ちょっと違うな**と思いました。

ターミナルを開いてプロンプトを打つ、という体験は仕事モードのスイッチが入ってしまう。育休中は「エンジニアとして考える」場面を意図的に減らして、生活に集中したかった。

Cowork はチャット感覚で話しかけるだけで動いてくれるので、仕事とプライベートの切り替えがしやすい。**同じ Claude を、ツールを変えることで使い分ける**、というのが自分にとってちょうどよかったです。

## `start-daily` スキルについて

`start-daily` は、**毎朝のデイリーノートを自動で作成するスキル**です。

チャットで「/start-daily」や「今日のスケジュール作って」と話しかけるだけで起動します。

![start-daily実行時の画面](https://static.zenn.studio/user-upload/deployed-images/609c5d957a39e3c69b83b18f.png?sha=3b2c927cca4e6cc985ef054fd1b16b07300f3d86)

大きく分けると以下の6ステップを自動でこなしてくれます。

### ① タスクを3つのソースから収集する

* `デイリータスク.md`（毎日やること）
* `未来メモ.md`（先の日付に紐づけたメモ）
* 前日のデイリーノートの「翌日への引き継ぎ」

これらを自動でかき集めて、今日のタスクリストを作ってくれます。デイリータスクはこんな内容です。

```
- [ ] おむつ替え
- [ ] 沐浴
- [ ] 料理
- [ ] 掃除
- [ ] 洗濯
- [ ] 猫遊び
- [ ] 15分読書
- [ ] ラジオ体操
```

育休中でも「育児以外にやること」はあります。自分の習慣（読書・ラジオ体操）をデイリーとして登録しておくことで、毎朝自動でリストに入ってきます。

**未来メモ**は特に活躍しました。健診・役所手続き・友人の面会・お宮参りと、育休中は初めてのイベントが次々と出てきます。「忘れないうちにとりあえずここに書く」というルールにしたら、頭の中がすっきりしました。

```
## 2026-04-27
- 📦 荷物到着後 → 区役所へ（出生届・マイナンバー手続き）
- 産後の手続き整理（健康保険加入、児童手当申請、乳幼児医療証）

## 2026-05-01
- 🦷 歯医者に電話して予約（仮歯が取れた）

## 2026-05-04
- 🚗 妻の皮膚科送迎（09:00〜）
- 📋 会社提出書類作成・提出

## 2026-05-06
- 👥 友達の面会

## 2026-05-08
- 🏥 2週間検診

## 2026-05-10
- 🎁 母の日プレゼントを買う（妻へ）

## 2026-05-19
- 午前中　友達の面会

## 2026-05-24
- ⛩️ お宮参り

## 2026-05-25
- 🏥 1ヶ月検診
```

その日の朝に `start-daily` を実行すると、該当日のメモが自動でタスクリストに反映されます。リマインダーをセットしなくていいし、「あの予定いつだっけ」と探し回る必要もない。

そして一番大事なポイントとして、**未来メモへの記入も含めて、自分でファイルを直接編集したことは一度もありません。**

「来週の火曜に2週間検診があるんだけど、未来メモに入れておいて」と Cowork に話しかけるだけ。Claude が `未来メモ.md` の該当日付に書き込んでくれます。**全部チャットで完結**しています。

### ② 今日の夕ご飯レシピを自動で取得する

あらかじめ week-1〜week-5 の5週サイクルでレシピを管理していて、今日の曜日に合ったレシピを自動で引っ張ってきます。

レシピの元ネタは **[memiの1か月食費2万円 節約ワンプレートごはん](https://www.amazon.co.jp/dp/4522438907)** という本です。節約・時短・ワンプレートという育休中にぴったりのコンセプトで、この本を参考に献立を組み立てました。

本の内容はスキャンして Obsidian にドキュメント化し、レシピ・週一の買い物リスト・仕込みリストをそれぞれノートとして整備。それをそのまま `start-daily` スキルの参照先として使っています。

![Obsidianのレシピフォルダ構成](https://static.zenn.studio/user-upload/deployed-images/41ec5fb64ed551633ba6b1cc.png?sha=351e1b4a326f6845b1044f142662350c04446b40)

食事周りはそもそも**判断する場面をとことん減らす**方針にしていました。

* **朝ごはん**：味噌汁・目玉焼き・ご飯・納豆で固定
* **昼ごはん**：前日の夜ご飯の残りを弁当にする
* **夜ごはん**：スキルが今日のレシピを自動で出してくれる

朝と昼は考えることがゼロ、夜だけスキルに聞けばいい。「今日何作ろう」という判断コストが完全になくなります。育休中は頭の中が常に赤ちゃんのことでいっぱいなので、食事の選択をこれだけシンプルにできたのは大きかったです。買い物も週一回、リスト通りに買うだけでよいので迷いがなくなりました。

### ③ ピヨログ（育児記録アプリ）のデータを集計する

前日の授乳回数・ミルク量・おしっこ・うんち・お風呂の時刻を Google Drive から取得し、さらに過去1週間の週平均も算出してくれます。

ピヨログデータの Google Drive へのアップロードは、こちらの記事を参考にしました。

<https://zenn.dev/big_tanukiudon/articles/52d5a4b653eb47>

当初は MacroDroid を使って毎晩23:55に自動エクスポートする仕組みを組んでいたのですが、3日目あたりから定期実行がうまく動かなくなってしまいました。原因の深追いよりも育児優先ということで、今はスキル実行前にピヨログアプリから手動でアップロードするようにしています。「完全自動化」とはいかなかったものの、1アクション追加するだけなので実用上は問題なしです。

![ピヨログのGoogle Driveフォルダ](https://static.zenn.studio/user-upload/deployed-images/2a867370241c7566c26f9193.png?sha=f140ced08689313c7b77b0f08548935c2229c5e6)

完母だと授乳は妻に任せるしかないのですが、赤ちゃんが1日どんなリズムで動いているかは自分でも把握しておきたかった。サマリーが毎朝自動で出てくるようになってからは、過去の授乳・おむつのタイミングをもとに「今日の午前中は10時〜12時あたりが落ち着きそう」という見通しが立つようになりました。その隙間に買い物・仕込み・作業を入れ、**赤ちゃんのリズムに合わせて自分のタスクを組み込む**感覚です。

実際のサマリーはこんな感じです。

![子育てサマリー（Obsidianでの表示）](https://static.zenn.studio/user-upload/deployed-images/25b468398fa6362dc380066e.png?sha=d5fc9be7afc86aaa6898e19d206f4e9021d867ed)

週平均との比較（↑↓→）やパパへの一言アドバイスも生成されるので、「今日は授乳が多めだったからゆっくり休んでもらおう」という判断がデータとしてできるのがよかったです。

### ④ 授乳・おむつのタイムラインを自動生成する

前日のピヨログデータをもとに、過去平均から授乳・おむつのタイミングを予測してタイムラインに並べてくれます。根拠付きで時刻が入るので、今日も同じ時間帯に動いてあげやすくなります。これが Google カレンダーにも自動登録されるので、スマホを見るだけで「次の授乳は何時ごろ」が分かります。

![Obsidianで表示したタイムライン](https://static.zenn.studio/user-upload/deployed-images/2ee5d2ebc463a9de3a6dd4a9.png?sha=09f351a527f10771aef0bb288a254ede4ca02bbc)

### ⑤ デイリーノートとして保存する

上記の内容が1つの Markdown ファイルにまとめられて保存されます。

保存先は Obsidian の Vault 内のフォルダに設定しているので、Cowork が作ったノートをそのまま Obsidian でプレビューできます。テーブル形式の育児サマリーやタイムラインもきれいにレンダリングされるので、朝イチで開いたときに「今日の全体像」がひと目で把握できます。

Obsidian のデイリーノート機能と組み合わせることで、過去のノートへのリンクも自然につながり、「先週の子どもの様子どうだったっけ」という振り返りもしやすくなります。

![Obsidianでデイリーノートをプレビューしている画面](https://static.zenn.studio/user-upload/deployed-images/068d2122d28617fd05474e4a.png?sha=990f1969e1a1df680b35a311f3b2f5d66e097000)

### ⑥ Google カレンダーに自動反映する

タイムラインのイベント（授乳目安・沐浴）とタスクが自動でカレンダーに登録されます。時間指定ありのものは時間指定イベント、タスクは終日イベントとして入ります。

![Googleカレンダーに授乳目安・沐浴が自動登録された画面](https://static.zenn.studio/user-upload/deployed-images/b51bb928ac1d4f6c55487a48.png?sha=58f31008d53d0348e232aea8b41735acc1d6a36a)

## 実際の使い方

朝、子どもが少し落ち着いたタイミング（大体授乳後）に Cowork を開いて、こう打つだけです。

あとは Claude が自動で動いて、2〜3分後にはデイリーノートが完成しています。

**作ったノートは、その日1日の行動の軸になります。**

* 朝イチで Obsidian を開き、今日のノートをプレビューして全体像を把握
* タスクが全部消化できているか、隙間時間にチェック
* 料理のときは PC を横に置いて、ノートに展開されているレシピの手順を見ながら調理

特にレシピは材料・作り方まで全部ノートに展開されているので、料理中に「次どうするんだっけ」と別のアプリを開く必要がありません。

![レシピ通りに作った夜ご飯](https://static.zenn.studio/user-upload/deployed-images/7d260e958948e697bff6b217.jpg?sha=c9e059b200014eae7c2a9e1ac02a262f3154436e)

## スキルは自分でカスタマイズできる

`start-daily` は最初から用意されているわけではなく、自分で作ったスキルです。

Cowork のスキル機能は、SKILL.md という Markdown ファイルに「こういう手順でやって」と書くだけで作れます。自分の生活に合わせて少しずつ育てていくのが楽しくて、今もちょこちょこ改良しています。

たとえば最初はレシピ機能がなく、後から「夕ご飯の判断コストを下げたい」と思って追加しました。ピヨログ連携も「データ見るのが面倒」と感じてから追加した機能です。

「不便だな」と思ったら、その日のうちにスキルをアップデートできる柔軟さが、Cowork の好きなところです。

自分が使っている SKILL.md をそのまま置いておきます。ファイルパスは各自の環境に合わせて書き換えてください。

start-daily SKILL.md（クリックで展開）

```
# start-daily スキル

## 概要
毎朝のデイリーノートを作成し、以下を自動で行う：
1. 3つのソース（デイリータスク・未来メモ・前日引き継ぎ）からタスクを収集
2. 今日のレシピを自動取得して追記
3. 前日の子育てサマリー（ピヨログ）を自動取得して追記
4. 過去1週間のピヨログデータを分析してタイムラインに追記
5. デイリーノートを保存
6. Googleカレンダーに反映

## ファイルパス（daily-life 統合ディレクトリ）
- ルート: `/Users/<username>/Documents/Claude/Projects/daily-life`
- デイリーノート保存先: `/Users/<username>/Documents/Claude/Projects/daily-life/daily/YYYY-MM-DD.md`
- デイリータスク: `/Users/<username>/Documents/Claude/Projects/daily-life/daily/デイリータスク.md`
- 未来メモ: `/Users/<username>/Documents/Claude/Projects/daily-life/daily/未来メモ.md`
- 前日ノート: `/Users/<username>/Documents/Claude/Projects/daily-life/daily/[昨日の日付].md`
- レシピ（週別）: `/Users/<username>/Documents/Claude/Projects/daily-life/recipes/week-X/[曜日].md`
- 仕込みガイド: `/Users/<username>/Documents/Claude/Projects/daily-life/recipes/week-X/prep-guide.md`

## 実行手順

### 1. 日付・曜日を取得
```bash
date +%Y-%m-%d        # 今日
date -v-1d +%Y-%m-%d  # 昨日（macOS）
date +%A              # 英語曜日
```

曜日のファイル名マッピング:
- Monday → monday.md / Tuesday → tuesday.md / Wednesday → wednesday.md
- Thursday → thursday.md / Friday → friday.md
- Saturday → saturday.md / Sunday → sunday.md

### 2. 既存ファイルチェック
今日のデイリーノートが存在する場合は `computer://` リンクを表示して「すでに作成済みです」と伝えて終了。

### 3. タスクを収集（3ソース）

**① デイリータスク**
`デイリータスク.md` の `## タスク一覧` セクションにある `- [ ]` 行をすべて読み取る。

**② 未来メモ（当日分）**
`未来メモ.md` を読み込み、`## YYYY-MM-DD`（今日の日付）の見出し配下の項目を取得する。
取得後、その見出しブロック（見出し行＋項目行）を削除する。
過去日付（今日より前）のブロックも同様に削除する。

**③ 前日の引き継ぎ**
昨日のデイリーノートが存在する場合、`## ➡️ 翌日への引き継ぎ` セクションの行のうち、空行・`- ` のみの行を除いた項目を取得する。

### 4. 今日のレシピを取得

レシピは月曜始まりの週サイクルで管理。前日ノートから週番号を読み取り、月曜日なら+1、火〜日曜はそのまま使う。week-1〜week-5の5週サイクルで折り返す。

`recipes/week-X/[今日の曜日].md` を読み込み、献立名サマリーと詳細レシピを展開する。

### 5. 子育てサマリーを取得（前日分 ＋ 過去1週間分）

Google Drive MCP で「ピヨログ」フォルダを検索し、前日分および過去7日分のファイルを取得する。

前日分から抽出する項目：母乳・授乳回数・ミルク・搾母乳・おむつ替え回数（夫/妻別）・お風呂

過去7日分で週平均を算出し、↑↓→で比較表示する。

### 6. 授乳・おむつスケジュール生成 → タイムラインに追記

前日の最後の授乳時刻から2〜3時間おきに24時間分のスケジュールを生成してタイムラインに追記する。沐浴は前日実績の時刻、なければ19:00を目安とする。

### 7. デイリーノートを作成・保存

タスク・レシピ・子育てサマリー・タイムラインを1つのMarkdownファイルにまとめて保存する。

### 8. Googleカレンダーへ自動反映

Google Calendar MCP でタイムライン行を時間指定イベント、タスクを終日イベントとして登録する。

### 9. 完了メッセージ

`computer://` リンクで今日のデイリーノートを表示し、反映内容を簡潔に伝える。

## 注意事項
- 日付・曜日は必ず動的に取得する（ハードコードしない）
- ファイルが既存の場合は上書きしない
- Google Calendar MCP が接続されていない場合はカレンダー登録をスキップ
- Google Drive MCP が接続されていない場合は子育てセクションに「データ取得不可」と記載
```

## 育休中だからこそ刺さった

育休中は「やることは多いけど、まとまった集中時間が取れない」というジレンマがあります。

そういうときに、毎朝5分未満で今日の全体像が整理される仕組みがあると、頭の中のノイズがかなり減ります。「あれどうするんだっけ」を考える時間がなくなる、という感じです。

子どもの昼寝の間など、隙間時間で作業するときも「今日のタスクリスト」がすでにある状態から始められるので、立ち上がりが速い。

育休中に限らず、「バタバタしがちな朝を整えたい」という方にはかなりおすすめです。

## まとめ

育休中の1ヶ月、Cowork と `start-daily` スキルのおかげで「考えなくていいこと」が大幅に減りました。

献立・スケジュール・育児記録の把握…どれも地味に頭を使うことばかりです。それをチャットで話しかけるだけで片付けられる環境を作れたのは、AIキャッチアップという当初の目的以上に、育休生活そのものを豊かにしてくれました。
