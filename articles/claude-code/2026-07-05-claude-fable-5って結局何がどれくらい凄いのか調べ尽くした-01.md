## 「また新モデルか」で流していないか

Claude Code や Claude API を日常的に触っていると、新モデルのリリースはもう珍しくない。Xのタイムラインには「Fable 5がヤバい」「怪物」「壊れた」といった断片的な感想が流れてくるが、正直こういう投稿を全部真に受けるのは疲れる。輸出規制で止まったとか、Opusにこっそり差し替えられていたとか、物騒な話も混ざっていて、結局何が本当で何が誇張なのか判断がつかない。

この記事では、2026年6月9日にリリースされたAnthropicの最新フラッグシップモデル「Claude Fable 5」について、公式ドキュメントと一次情報ベースの調査記事を突き合わせて整理する。数値には必ず出典を付け、「盛られた」評価と「実測に基づく」評価を分けて書く。

## TL;DR

* Fable 5はAnthropic初の「Mythos級」一般提供モデル。SWE-bench Proで80.3%、Opus 4.8を10ポイント以上引き離す（[出典](https://platform.claude.com/docs/en/about-claude/models/overview)）
* ただしGPQA Diamond（科学推論）ではGemini 3.1 Proに負けている。「全部で1位」ではない
* 価格はOpus 4.8のちょうど2倍（入力$10/出力$50、[出典](https://platform.claude.com/docs/en/about-claude/pricing)）。単価だけ見ると高いが、実効コストの話は別
* リリース3日後に輸出規制で全世界停止、19日後に復活という異例の事件があった
* 7月7日まで無料キャンペーン中。Claude Code/APIユーザーは今のうちに触っておく価値がある

## 1. Claude Fable 5とは何か

Claude Fable 5は2026年6月9日、Claude Mythos 5と同時に発表されたAnthropicの最新フラッグシップモデルだ。モデルIDは`claude-fable-5`で、Anthropicが「Mythos級（Mythos-class）」と呼ぶ新しいティアの一般提供第一号にあたる。位置づけとしてはOpus級のさらに上、という扱いになる（[Anthropic公式発表](https://www.anthropic.com/news/claude-fable-5-mythos-5)、[モデル紹介ページ](https://platform.claude.com/docs/en/about-claude/models/introducing-claude-fable-5)）。

基本スペックは以下の通りだ。

| 項目 | 値 |
| --- | --- |
| コンテキスト長 | 1Mトークン（デフォルト＝最大） |
| 最大出力 | 128K |
| 知識カットオフ | 2026年1月 |
| 入力価格 | $10/MTok |
| 出力価格 | $50/MTok |

出典: [platform.claude.com/pricing](https://platform.claude.com/docs/en/about-claude/pricing)、[モデル紹介ページ](https://platform.claude.com/docs/en/about-claude/models/introducing-claude-fable-5)

API仕様でエンジニア視点で気になるのは以下の3点だ。

1. **thinking（思考）が常時ON**。Opus 4.8までは思考のON/OFFを開発者が選べたが、Fable 5では常時ON固定で、無効化やbudget指定は400エラーになる。生の思考連鎖（raw CoT）は返却されず、`display: "summarized"`で要約だけ取得できる
2. **アシスタントprefill・temperature等のサンプリングパラメータが使えない**
3. **30日間のデータ保持が必須**。ゼロデータ保持（ZDR）契約の組織では利用できない

さらに、cybersecurity・biology・chemistry・model distillation関連の入出力を検出するsafety classifiersが搭載されていて、該当すると`stop_reason: "refusal"`（HTTP 200）で拒否される。拒否時に`fallbacks`パラメータ（オプトイン）を指定しておくと、Opus 4.8へ自動的にリトライされる仕組みもある（[出典](https://platform.claude.com/docs/en/about-claude/models/introducing-claude-fable-5)）。

なお、トークナイザーについては誤解が広がった経緯がある。「Fable 5はOpus 4.8より30%多くトークンを消費する」という記述が一部で見られたが、これは誤りだ。実際にはOpus 4.7で導入された新トークナイザーをFable 5もOpus 4.8も共通で使っており、旧世代（Opus 4.6/Sonnet 4.5以前）と比べて約1〜1.35倍という水準にとどまる（[Anthropic公式migration guide](https://platform.claude.com/docs/en/about-claude/models/migration-guide)より）。

## 2. もう一つの顔、Claude Mythos 5

Fable 5と同時に発表された「Claude Mythos 5」は、**Fable 5とベースモデル・価格・APIが完全に同一**で、違いはsafety classifiers（セーフガード）の有無だけだ。Mythos 5はセーフガードなし版で、米政府とAnthropicが協力する「Project Glasswing」経由でのみ提供される。対象は米国の防御的サイバーセキュリティ専門家や重要インフラ事業者に限られ、招待制でセルフサービス申請はできない（[出典](https://anthropic.com/glasswing)、[出典](https://www.anthropic.com/news/claude-fable-5-mythos-5)）。

Mythos 5はセキュリティ系ベンチマークで際立った数値を出している。ExploitBenchでMythos 5は78.0%、Opus 4.8は40.0%だ（[claudefa.st](https://claudefa.st/blog/models/claude-mythos)）。

各ベンチマークが測っているものをざっくり書いておくと、SWE-bench Pro / Verifiedは実際のOSSリポジトリのissue修正（＝実務コーディング）、FrontierCode Diamondは最高難度のコーディング問題、Terminal-Benchはターミナル操作の自律遂行、GPQA Diamondは大学院レベルの科学推論、Humanity's Last Exam（HLE）は専門家が作った超難問の総合試験、OSWorldはOS/GUI操作エージェントの評価だ。

ここで正直に書いておきたいのは、**Fable 5は全部のベンチマークで1位ではない**という事実だ。GPQA Diamond（大学院レベルの科学推論）ではGemini 3.1 Proが94.3%でFable 5を上回る。コーディング・エージェント系タスクでの優位性は明確だが、科学推論の一部領域では最強ではない。「万能の王者」という煽り文句には注意が必要だ。

なお、Humanity's Last Examの53%というスコアは、8〜9%のタスクでセーフガードが発動しOpus 4.8にフォールバックした状態でのスコアである点も付記しておく。ARC-AGIはFable 5のスコアが非公表（GPT-5.5が最高値を保持）で、MMLUやAIMEのような従来定番のベンチマークは「飽和」扱いとなり、公式資料でも数値が出されなくなってきている。ベンチマーク自体が世代交代している最中だと理解した方がよい。

二次ソースになるが、幻覚率を測るAA-Omniscienceベンチでは、Fable 5が36.18%で最も低く（＝幻覚が少ない）、Gemini 3.1 Proは49.87%、GPT-5.5は85.53%という報告もある（[Lushbinary](https://lushbinary.com/blog/claude-fable-5-vs-gpt-5-5-vs-gemini-3-1-pro-comparison/)）。

### ネーミングの変遷（ここは考察）

Claudeのモデル名は「Haiku（俳句）→ Sonnet（ソネット）→ Opus（大作）」という詩の形式・規模による命名から始まった。今回登場した「Fable（寓話）」「Mythos（神話）」は、いずれも「語られる物語」を意味するラテン語・ギリシャ語由来の言葉だ。

詩（形式）から物語（内容）への転換に見えるが、**この命名意図についてAnthropicの公式説明は見当たらない**。あくまで外部の考察記事の域を出ないことは明記しておく（[考察記事の一例](https://medium.com/@alexandersamuel2310/haiku-sonnet-opus-mythos-eb66cd8db53d)）。一つ言えるのは、Fable/Mythosという「関連するが違う」文学語をペアで採用したことで、同一モデルの2バリアント（セーフガード有無）を区別する命名の仕組みが生まれた、という点だ。これは後述する独創的観点でもう一段深掘りする。

## 7. 独創的な視点 — 数字の裏側にある構造の話

ここまでは公表数値の整理だ。ここからは、数字だけでは見えてこない構造的な論点を4つ掘り下げる。

### (1) 「単価2倍は高いのか」問題

Fable 5の入力$10/出力$50は、Opus 4.8のちょうど2倍だ。単価だけを見れば「高い」で終わる話だが、判断軸はもう一つ必要になる。**実効コスト**だ。

前述の通り、同一タスクをGPT-5.5の1/3のトークン消費で完了した事例が報告されている（[Lushbinary](https://lushbinary.com/blog/claude-fable-5-vs-gpt-5-5-vs-gemini-3-1-pro-comparison/)）。単価が2倍でもトークン消費が1/3で済むなら、タスク単価としては安くなる計算だ。もちろんこれは特定タスクの一事例であり、すべてのワークロードに当てはまるわけではない。

さらにもう一段、**人件費換算**という視点を加えると見え方が変わる。Stripeの5000万行Rubyコードベース移行は、手作業見積り2ヶ月以上の作業を数日で完了させた（[出典](https://tosea.ai/blog/claude-fable-5-review-developer-reactions)）。API料金がいくらであっても、エンジニアの人件費を1〜2ヶ月分節約できるなら、単価の議論はほとんど誤差の範囲になる。

つまり「単価2倍」は入口の情報にすぎない。判断すべきは「単価 × 消費トークン数 vs 人件費換算」の掛け算であって、単価の桁だけを見て高い・安いを決めるのは早計だ。

### (2) セーフガードの透明性デザイン — 「黙って差し替える」から「差し替えたと明示する」へ

前述のNathan Lambertの批判は、初期のフォールバック設計、つまりセーフガードが発動した際に「Opus 4.8に黙って差し替えて応答する」仕組みへの批判だった。ユーザーからすれば、Fable 5に頼んだつもりが実は別モデルの答えを受け取っている、という状態になる。

これに対してAnthropicは、6月30日の輸出規制解除・7月1〜2日の再展開に合わせて設計を変更した。セーフガード発動時には「Claude Opus 4.8からの応答である」ことを明示的に表示するリルーティングに変更されている（[出典](https://www.anthropic.com/news/redeploying-fable-5)）。

これはAIプロダクトのUX設計として興味深い事例だ。「能力を落とすこと」自体が問題なのではなく、「落としたことを隠すこと」が信頼を損なう、という教訓に見える。透明性を欠いたフォールバックは短期的にはシームレスに見えるが、露見した瞬間に「勝手に手を抜かれていた」という不信に転化する。今後のAIプロダクト設計において、モデル切り替え・機能制限の明示は一つのベストプラクティスになっていきそうだ。

### (3) 輸出規制による19日間の停止事件 — 「強すぎて止められた」の皮肉

Fable 5の歴史で最も劇的な出来事は、リリースからわずか3日後に起きた。

6月9日にリリースされたFable 5は、6月12日にAmazonの研究者がセーフガード回避手法を報告したことを受け、米商務省が即日で輸出規制を適用し、**全ユーザーでアクセスが停止**した。国籍確認が技術的に困難なため、特定国のユーザーだけを止めるのではなく全面停止という措置が取られた。その後、Opus 4.8・GPT-5.5・Kimi K2.7も同じ脆弱性を持つことが確認されている。6月30日に規制が解除され、7月1〜2日にかけて全世界で再展開された。再展開時には回避手法への検出確度が99%以上に改善したセーフガードが搭載されている（[Anthropic公式](https://www.anthropic.com/news/redeploying-fable-5)、[VentureBeat](https://venturebeat.com/technology/anthropic-is-bringing-back-claude-fable-5-globally-after-us-lifts-export-control-order-where-can-enterprises-access-it)、[MacDailyNews](https://macdailynews.com/2026/07/02/anthropic-brings-back-powerful-claude-fable-5-ai-model-after-u-s-export-controls-lifted/)）。

この事件には皮肉な構図がある。輸出規制の対象になるということは、裏を返せば「軍事転用や重大インフラリスクを懸念されるレベルの能力を持っている」ことの証明でもある。「規制されるほど強い」というのは名誉なような、危険なような話だ。

一方で実務側から見ると、これは笑い話では済まない。19日間、業務でFable 5に依存していた開発チームやエンタープライズは、代替モデルへの切り替えを余儀なくされたはずだ。[MarketScaleの記事](https://www.marketscale.com/industries/software-and-technology/fable-5-and-mythos-5-are-back-what-the-19-day-shutdown-taught-every-enterprise-about-ai-as-infrastructure)はこの事件を「AIをインフラとして扱うことの単一ベンダー依存リスク」の実例として論じている。国家間の規制動向一つで、契約しているモデルが丸ごと使えなくなり得る。これはAWSの特定リージョン障害とは種類の異なるリスクで、規制リスクという新しいカテゴリのSPOF（単一障害点）だと理解しておいた方がいい。

### (4) Fable/Mythosの二重リリース構造 — dual-useへの一つの回答

Fable 5とMythos 5は同一ベースモデル・同一価格・同一APIで、違いはセーフガードの有無だけだ。これは業界的にも珍しい構造だと言える。

高性能モデルには常にdual-use問題（同じ技術が有益な用途にも悪用にも使える両面性の問題）がついて回る。多くのベンダーはこれを「一つのモデルにセーフガードをかけ、必要な相手には個別に緩和する」という運用でしのいでいるが、Anthropicは踏み込んで**モデル名そのものを分けた**。セーフガードあり版が一般提供の「Fable」、セーフガードなし版が招待制・政府協力プログラム経由の「Mythos」という形で、名前とアクセス経路と信頼レベルを一致させている。

これは「同じ強さのモデルを、利用者の信頼レベルに応じて別名で出す」という一つの解法だ。今後、他ベンダーが同種のdual-use問題に直面したとき、参照される先行事例になる可能性がある。

## 8. 結局、自分は使うべきか

ここまでの内容を踏まえて、Claude Code・API利用者向けの実践的な判断材料を書く。

**今すぐ触ってみるべき理由**

* Pro/Max/Team/一部Enterpriseで、**7月7日まで**週間使用上限の最大50%までFable 5が無料で使えるキャンペーンが実施中（[出典](https://zenn.dev/acntechjp/articles/498c1dc866af19)）。単価2倍のモデルを無料枠で試せる機会は貴重だ
* 長時間の自律実行やSWE-bench系タスクで明確な優位性がある。大規模リファクタリング・移行系のタスクを控えているなら試す価値は高い

**慎重に検討すべき理由**

* 30日データ保持が必須なので、社内ポリシーでZDR契約を結んでいる組織はそもそも選択肢に入らない
* 医療・セキュリティ・化学系のドメインを扱う場合、セーフガードの誤発動リスクを考慮する必要がある。事前に`fallbacks`パラメータでOpus 4.8への自動リトライを設定しておくのが無難だ
* 通常のCRUD実装やドキュメント作成程度のタスクにはオーバースペックで、単価2倍分のコストに見合わない可能性が高い

実務的には、「すべてFable 5に寄せる」のではなく、タスクの難易度に応じてOpus 4.8・Sonnet 5・Fable 5を使い分けるのが現実的だ。この「どのタスクにどのモデルを使うか」という判断を体系化する話は、以前書いた[Claude Codeで複数モデルを使い分ける──Bloom分類によるモデルルーティング設計](https://zenn.dev/yamada_ai_dev/articles/claude-code-bloom-model-routing)で扱っている。Fable 5は「L6：前例のない設計・長時間の自律実行」のような重量級タスク専用の枠に振り分け、日常的なタスクはSonnet 5やHaiku帯に任せる、という設計が単価2倍問題への実務的な解になる。

## まとめ

1. Fable 5はAnthropic初のMythos級一般提供モデルで、コーディング・エージェント性能はOpus 4.8を大きく上回る
2. ただしGPQA DiamondではGemini 3.1 Proに劣るなど、全ベンチマークで1位というわけではない
3. 単価はOpus 4.8のちょうど2倍だが、実効トークン効率や人件費換算まで含めて考えると評価が変わる

6. 7月7日までの無料キャンペーン中に一度触ってみて、自分のワークロードに合うか判断するのが現実的だ
