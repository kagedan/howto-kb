CLAUDE.mdを書いてるのにClaude Codeがイマイチ言うことを聞かない…

それ、書き方が間違ってるかもしれない。
最強のCLAUDE.mdにする5つのルールを書いてみるから、参考にしてほしい

① 200行以内に収める

「とりあえず全部書いておこう」はNG。

長くなるほどClaudeの読み込み精度が落ちる。
大事なことが埋もれて、結局無視される。

200行を超えたら削る。
シンプルなほど、よく動く。    

② アクション指向で書く＋意図を添える

❌「丁寧に答えてください」
⭕「前置きなし。結論から答える」

❌「なるべく確認してください」
⭕「実行前の確認は禁止。即動く」

さらに強くするなら「なぜか」も書く。
「確認禁止。副業の作業時間を無駄にしないため」
理由があると守られやすくなる。      

③ リスクを下げる記述を必ず入れる

Claude Codeは優秀だからこそ
止めないとどんどん動く。

「本番ファイルは削除禁止」
「外部への送信は必ず確認を取る」
「金額が絡む処理は止まれ」

ブレーキを書いておくのが
使いこなしている人の共通点。
