# プログラミング未経験がClaude Codeだけで資格勉強アプリを作った話

## きっかけ

簿記2級の勉強を始めて3日で挫折した。

「明日からやろう」を何度繰り返しただろう。YouTubeを開くつもりがいつの間にかショート動画を1時間見ていた。意志が弱いのはわかってる。でも意志だけではどうにもならないことも、なんとなくわかってた。

そんなとき思いついたのが「サボったら自動で課金される仕組み」。損したくないという本能を使えばいいんじゃないかと。調べてみたけど、そんなアプリは存在しなかった。

**じゃあ作るか。** プログラミングの経験はほぼゼロなのに。

---

## 使った技術

正直、最初は何も知らなかった。Claude Codeに「Webアプリを作りたい」と相談したら技術スタックを提案してもらった。

* **Next.js 16**（フロントエンド・バックエンドまとめて）
* **Supabase**（データベース・認証）
* **Stripe**（決済。ここが一番の肝）
* **Vercel**（デプロイ。GitにプッシュするだけでURLが生成される）
* **Tailwind CSS**（デザイン）

全部無料枠で運用できる。これは大事だった。

---

## Claude Codeへの指示の出し方

最初は「ログイン機能を作って」みたいな雑な指示をしていた。そうするとコードは出てくるけど、意図と違うものができることが多かった。

途中から指示の出し方を変えた。

**悪い指示：**

> 「グループ機能を作って」

**良い指示：**

> 「LINEで招待リンクを送ると、そのリンクを踏んだ人が自動でグループに参加できる機能を作って。グループ内では仲間の今日の勉強時間・連続達成日数・過去7日間のカレンダーが見える」

具体的なユーザー体験から逆算して指示する。これだけで精度がまったく変わった。

あと詰まったときに「なぜそうなっているのか教えて」と聞くようにした。コードを直してもらうだけじゃなく、理由を理解すると次に同じミスをしなくなる。

---

## 詰まったところと解決方法

**① Stripeのオフセッション課金**

一番ハマった。「サボった翌日に自動課金する」という仕組みを作るには、ユーザーが画面を開いていないときでも課金できる必要がある。これがオフセッション課金というらしい。

最初はカード登録とその日の課金を混同していて、なぜかカード登録のたびに課金されるという恐ろしいバグを作った。Claude Codeと一緒にエラーログを読み解いて、SetupIntentとPaymentIntentの違いを理解してなんとか解決。

**② Supabaseの環境変数**

`.env.local`に書いた環境変数がVercelにデプロイしたら動かない。よくある話らしいけど、最初は意味がわからなかった。Vercelのダッシュボードに環境変数を別途設定する必要があった。Claude Codeがそのまま手順を教えてくれたので助かった。

**③ RLS（Row Level Security）**

SupabaseにはRLSというアクセス制限の仕組みがある。設定しないとデータが誰でも見えてしまう。最初はポリシーを何も設定してなくて、他のユーザーのデータが見えるバグを作ってしまった。SQL Editorでポリシーを設定して解決。

**④ next/ogのCSS制限**

SNSシェア時に表示されるOGP画像を`next/og`で作ったら、真っ白な画像が出てきた。`position: absolute`など一部のCSSが非対応だったことが原因。シンプルなflex構造に書き直したら解決。

## 完成したアプリの紹介

**StudyBet**（<https://studybet.vercel.app>）

主な機能はこんな感じ：

* **ペナルティシステム**：目標未達成の翌日に、自分で設定した金額が自動課金される。Apple Pay・Google Pay・コンビニ払いにも対応
* **ポモドーロタイマー**：25分集中・5分休憩のサイクル。時間は自由にカスタマイズ可能
* **お休み設定**：曜日単位・日付単位でペナルティ対象外の日を登録できる
* **グループ機能**：LINEで招待リンクを送るだけで仲間とグループを作れる。週間ランキングと過去7日のカレンダーが表示される
* **進捗メモ**：勉強終わりに「今日の進捗」と「次回はここから」を記録。次回ログイン時にバナーで表示される
* **PWA対応**：スマホのホーム画面に追加してアプリとして使える

プログラミング未経験が作ったとは思えないと言われるとうれしい（自己評価）。

## 詰まったときの気づき

Claude Codeは「魔法のツール」ではなく「頭のいいペアプログラマー」だと思うようになった。

指示が曖昧だと曖昧なコードが返ってくる。何を作りたいかを自分の頭で整理してから聞く。エラーが出たらログをそのまま貼り付けて「何が起きてるか教えて」と聞く。これだけで大体なんとかなった。

あと「完璧なものを一気に作ろうとしない」のも大事だった。小さい機能を一つずつ作って動作確認する。そのサイクルを回し続けた結果、3ヶ月でそれなりのアプリができた。
