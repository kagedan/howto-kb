前回はClaude Proプランを契約するまでの経緯をお話しましたので、今回は私が環境構築したときの流れを語ります。  
前回：[非エンジニアがClaude Codeを使うために課金するまでの経緯](https://zenn.dev/tetsuro_works/articles/ai-agent-frontier-log-02)

Claude Codeには、使い方が複数ありますが、インストール方法は公式サイト（[クイックスタート - Claude Code Docs](https://share.google/swASkk6UoUqEKupAV)）や、他の方の記事見ていただいた方が確実なので、この記事は「触ってみてどうだったか」という体験談に絞ります。

実際に試した4つの環境——CLI・VSCode拡張・デスクトップ版アプリ・スマホ版アプリ——を個人的な目線でレポートします。

---

## 1. CLI（コマンドライン）

非エンジニアの天敵、「黒背景に白文字」のアイツです。（色はテーマ設定で変えられますけどね）

公式ドキュメントの通りインストールが完了していれば、ターミナルを開いて`claude` とコマンドを打ち込むだけでClaude Codeが起動します。

見た目はとっつきにくいですが、プロンプトを打てば返事が返ってくるのは他のAIツールとなんら変わりません。  
ただ出力されたファイルを確認したり、手動で修整したりするには、別途エディタが必要になります。

どちらかといえば、どんな動作をして、どんなアウトプットを出したか掴みづらい印象です。CLIのとっつきにくさの本質は**認知のコストの高さ**なのではと感じました。

ただ—、**「なんかすごいことをやってる」という充足感**を一番得られるのは、このCLIだと思いました。

---

## 2. VSCode拡張（メイン環境）

私にとっては、**Claude CodeはVSCodeで使うのが一番しっくりきました。**

VSCode（Visual Studio Code）は、コードを書くためのエディタです。無料で使えて、拡張機能を追加することでさまざまなツールと連携できます。

VSCodeにClaude Codeの拡張機能を入れると、画面構成を以下のようにできます。

* 左：ファイルの中身の閲覧・修整ができる（エディタ）
* 右：Claude Codeとの対話パネル（チャット）
* 下：黒背景＋白英字のアイツ（ターミナル）

これならClaude Codeのアウトプットファイルをその場で確認できますし、Claude Codeと相談しながらターミナルを操作することもできます。

GitHubのリポジトリ管理もVSCodeで一緒にできるので、「エディタ・Claude Codeチャット・ターミナル・Gitがまとまっている」という状態が、学習には最適だと感じています。

---

## 3. デスクトップ版アプリ

デスクトップ版は、**通常のAIチャットアプリと見た目が近い**です。CLIと比較して取っつきやすさは別格でしょう。  
「Claude Codeを触ってみたい」という方は、まずデスクトップ版から入るのが心理的ハードルが低くていいかもしれません。

ただ、実際に作業をしようとすると**エディタが別途あったほうが良い**と思います。ビューワー機能があることでCLIよりは見やすいとは思いますが、作業性はエディタには敵いません。

出力ファイルの中身をチェックする頻度が低いタスクであれば、違和感なく使えると思います。（たたき台の作成だけやってもらう）  
逆に修正を頻繁に行うタスクは、VSCodeのほうがこなしやすい印象でした。

---

## 4. スマホ版アプリ

スマホからClaude Codeを使う方法は幾つかあるようですが、公式アプリの「コード」と言うモードを使いましたが、結構頻繁に使っています。

できることはデスクトップ版と近いですが、大きな違いは、**会話開始前にGitHubリポジトリの指定が必要**という点です。  
なので、これを試すために初めてGitHubを使うことになりました。

VSCodeのように画面を分割して作業することはできませんが、もともとスマホは画面を切り替えながら使うのが普通ですから、あまり苦には感じません。

なにより「ちょっとした作業を出先で進める」というサブ環境としては、十分に機能しています。

---

## まとめ：私が落ち着いた使い方スタイル

4つの環境を試した結果、今の私のスタイルはこうなっています。

| 環境 | 使い方 |
| --- | --- |
| VSCode拡張 | メイン。本格的な作業はここ |
| スマホ版 | サブ。出先でのちょこちょこ作業 |
| デスクトップ版 | Claudeチャットとの会話用 |
| CLI | 通過儀礼として一度体験したのみ |

非エンジニアへのおすすめはデスクトップ版からスタートですが、実際にどんなファイルが生成されているのか確認する上ではVSCodeをエディタとしてインストールしておくといいでしょう。

そしてそこまでくれば「VSCode拡張でよくね？」になって来るかと思います。

なお、GitHubとの連携も、VSCode拡張上でClaude Codeに質問しながら作業することで実現できました。

初回の設定はClaude Codeの言う通り進めれば比較的簡単にできます。  
それだけだと何が何だか分からないので、2回目はプッシュの仕方をClaude Codeに教わりながら自分で操作して感覚を掴んでいます。

総じて、**悩んだときは、Claude Codeに聞けばなんとかなる**といった状態で、さすがだなぁと感心するばかり。  
「よくわかんない」という感想をそのまま投げつけても、生成AIはこちらに寄り添って辛抱強く教えてくれますよ。

そんな健気で優秀な**友人**と一緒に、これからもスキルアップしていきます！
