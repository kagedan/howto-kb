1. **Google Vertex AI 対話式セットアップウィザード** -- ログイン画面から GCP 認証・プロジェクト設定・モデル固定まで一気通貫で設定可能に

## Vertex AI の初期設定がウィザード形式で完結するように

Google Cloud 経由（Vertex AI）で Claude Code を使いたい方に特に有益な変更です。

**何が変わったか**

ログイン画面で「3rd-party platform」を選択すると、**対話式の Vertex AI セットアップウィザード**が起動するようになりました。ウィザードは以下のステップをガイドします。

1. GCP 認証（`gcloud auth` の確認）
2. プロジェクトおよびリージョンの選択
3. 認証情報の検証
4. 使用するモデルの固定（model pinning）

**なぜ嬉しいか**

これまで Vertex AI 経由の設定は、環境変数や設定ファイルを手動で整える必要があり、ドキュメントを読みながらの試行錯誤が必要でした。ウィザード形式にまとまったことで、**GCP 初心者でも手順を間違えにくく**なります。先日 Bedrock でも同様のウィザードが追加されましたが、今回は Vertex AI 版が揃った形です。

```
# ログイン画面で選択する項目
> 3rd-party platform  ← これを選ぶとウィザードが起動
```

ウィザードを利用するには、事前に `gcloud` CLI がインストールされ、ローカル環境から GCP へアクセスできる状態になっている必要があります。

## Perforce 環境での誤上書きを防ぐ新モード

Perforce（p4）をバージョン管理システムとして使っているチームの方向けの変更です。

**何が変わったか**

`CLAUDE_CODE_PERFORCE_MODE` という新しい環境変数が追加されました。この変数を設定した状態で `Edit` / `Write` / `NotebookEdit` ツールが読み取り専用ファイルに対して実行されようとすると、**黙って上書きするのではなく、エラーとともに `p4 edit` を実行するよう促すヒント**を表示して処理を止めます。

```
# Perforce モードを有効にする
export CLAUDE_CODE_PERFORCE_MODE=1
```

**なぜ嬉しいか**

Perforce では、ファイルを編集する前に `p4 edit` でチェックアウトする必要があります。Git と異なり、ファイルシステム上では読み取り専用として存在するのが Perforce の流儀です。このモードなしでは Claude Code がチェックアウトを経ずにファイルを上書きしようとして失敗したり、ワークスペースの状態が壊れる恐れがありました。**Perforce ユーザーにとってはかなり実用的な追加**と言えます。

## Bash ツールのセキュリティ修正を見逃さないで

`allowlist` や `--dangerously-skip-permissions` を活用している方は必ず確認してください。

**何が変わったか**

今回のリリースでは Bash ツールまわりのセキュリティ修正が複数入っています。主なものを以下にまとめます。

* **バックスラッシュエスケープによる権限バイパス**: `\ls` のようにバックスラッシュでエスケープされたコマンドが読み取り専用として自動許可され、任意コード実行につながる恐れがありました
* **複合コマンドの強制プロンプト回避**: `cmd1 && cmd2` のような複合コマンドが、`auto` モードや `bypass-permissions` モードで安全チェックをすり抜けていました
* **環境変数プレフィックス付きコマンドの扱い**: `MY_VAR=foo ls` のように環境変数を前置したコマンドが、既知の安全な変数（`LANG`、`TZ`、`NO_COLOR` など）以外でもプロンプトなしで通過していました
* **`/dev/tcp` や `/dev/udp` へのリダイレクト**: ネットワークアクセスに相当するこれらへのリダイレクトが自動許可されていました

**なぜ嬉しいか**

いずれも**意図しない権限昇格や外部通信を引き起こしうる脆弱性**です。特にチーム環境や CI での利用時には影響が大きいため、このリリースへの更新を強く推奨します。

`--dangerously-skip-permissions` を日常的に使っている場合、エージェントチームメンバーがリーダーの権限モードを継承しない問題も修正されています。想定外の権限で動作していた可能性があるため、設定を見直してください。

## バグ修正

バグ修正 37件（クリックで展開）

* **429 リトライ**: サーバーが小さい `Retry-After` を返した場合、約13秒で全リトライを消費していた問題を修正。指数バックオフを最低値として適用するように
* **ストリーミングタイムアウト**: ストリーミングレスポンスが止まった際、非ストリーミングへのフォールバックではなくタイムアウトしていた問題を修正
* **MCP OAuth**: 再起動後のトークンリフレッシュ時に `oauth.authServerMetadataUrl` の設定が反映されない問題を修正（ADFS などに影響）
* **キーボード入力（xterm / VS Code）**: kitty キーボードプロトコルが有効な環境で大文字が小文字に化ける問題を修正
* **macOS テキスト置換**: トリガーワードが置換後テキストに変わらず削除されていた問題を修正
* **`--dangerously-skip-permissions` の降格**: 保護パスへの書き込み承認後に accept-edits モードへ暗黙的に降格されていた問題を修正
* **管理設定の allow ルール**: 管理者が削除したルールがプロセス再起動まで有効なままだった問題を修正
* **`permissions.additionalDirectories`**: セッション中の変更がすぐに反映されなかった問題を修正。削除はすぐに有効になり、追加は再起動不要に
* **`additionalDirectories` と `--add-dir` の競合**: `additionalDirectories` からディレクトリを削除すると、`--add-dir` で渡した同名ディレクトリのアクセスまで失われる問題を修正
* **ワイルドカード権限ルール**: `Bash(cmd:*)` や `Bash(git commit *)` がスペースやタブを含むコマンドにマッチしない問題を修正
* **パイプコマンドの deny ルール**: `cd` を含むパイプコマンドで deny ルールがプロンプトに降格されていた問題を修正
* **偽陽性パーミッションプロンプト**: `cut -d /`、`paste -d /`、`column -s /`、`awk '{print $1}' file`、`%` を含むファイル名で誤ってプロンプトが出ていた問題を修正
* **prototype プロパティと同名のルール**: `toString` など JavaScript の prototype プロパティ名と一致する権限ルールが `settings.json` を無効化する問題を修正
* **エージェントチームの権限継承**: `--dangerously-skip-permissions` 使用時にサブエージェントがリーダーの権限モードを継承しない問題を修正
* **フルスクリーンモードのクラッシュ**: MCP ツール結果にホバーした際のクラッシュを修正
* **フルスクリーンモードでの URL コピー**: 折り返した URL のコピーに余分なスペースが入る問題を修正
* **`--resume` でのファイル差分消失**: 10KB 超のファイルを編集した際に差分が UI から消えていた問題を修正
* **`/resume` ピッカーの複数問題**: `--resume <name>` で編集不可な状態で開く、フィルター再読み込みで検索状態がリセット、リスト空の際の矢印キー無効、クロスプロジェクトでの古いデータ表示、タスクステータステキストが会話サマリーを上書きする問題を修正
* **`/export` のパス処理**: 絶対パスや `~` が無視され、拡張子が `.txt` に書き換えられる問題を修正
* **`/effort max`**: 未知または将来のモデル ID で拒否される問題を修正
* **スラッシュコマンドピッカー**: プラグインの frontmatter `name` が YAML の boolean キーワードの場合にピッカーが壊れる問題を修正
* **レート制限のアップセルテキスト**: メッセージが再マウントされると非表示になる問題を修正
* **MCP ツールのトークン制限バイパス**: `_meta["anthropic/maxResultSizeChars"]` を持つ MCP ツールがトークンベースの永続レイヤーをバイパスできない問題を修正
* **ボイスモードのスペース漏れ**: 前のトランスクリプト処理中に push-to-talk キーを再ホールドすると大量のスペースが入力に挿入される問題を修正
