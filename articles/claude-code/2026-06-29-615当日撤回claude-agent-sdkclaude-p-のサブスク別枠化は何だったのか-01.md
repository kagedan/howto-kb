2026年6月15日、Anthropic は Claude のサブスクリプション利用枠を「対話（Interactive）」と「自動化（Programmatic）」に分割し、Agent SDK や `claude -p`、GitHub Actions 経由の利用を**別建ての月額クレジット**に切り出す——という大きな課金変更を予定していた。5月中旬に告知され、6月に入ってからは英語圏の個人開発者界隈でかなり騒がれていた。

ところが、**施行予定日だった 6/15 当日に、この変更そのものが撤回（保留）された**。「対話と自動化の分離」は幻に終わり、いまのところ何も変わっていない。

報道の一次まとめとしては The New Stack が早かった。

https://thenewstack.io/anthropic-pauses-claude-agent-sdk-subscription-change/

このニュース、単なる「課金改定が延期になった」では片付けられない。**そもそも何を分離しようとしていたのか**を理解しておかないと、次に来る改定で確実に足をすくわれる。`claude -p` を cron で回している人・GitHub Actions に Claude を組み込んでいる人にとっては他人事ではないので、告知内容・撤回の経緯・個人開発者目線での影響を整理しておく。

この半年の Claude 課金まわりの迷走の流れで読むと構図がきれいに見えるので、前回の緩和ニュースもあわせて読むと分かりやすい。

https://qiita.com/sakutto-panda/items/cc57421ad20761829ea7

## 何が告知されていたか — 対話枠と自動化枠の分離

告知の骨子はシンプルで、これまで1つのサブスク枠で一括管理されていた利用を、**人間が打鍵する「対話利用」**と**プログラムが自動でモデルを叩く「自動化利用」**の2つに割り、自動化側だけを専用クレジットに移すというものだった。

### 自動化枠に移る予定だったもの

新しい月額クレジットから消費される予定だったのは、以下のような「人間が逐一打鍵していない」利用方法。

- **Claude Agent SDK**（Python / TypeScript で自作したエージェント）
- **`claude -p`**（Claude Code のヘッドレス・非対話モード。スクリプトや cron で回すやつ）
- **Claude Code GitHub Actions**（CI 上で自動実行される処理）
- **Agent SDK でサブスク認証するサードパーティアプリ**

クレジットは**標準 API のトークン単価**で課金され、使い切ったら「usage credits」設定次第で追加課金されるか、次のサイクルまで停止する、という建て付けだった。

### 対話枠（変更なし）のもの

一方、以下は従来どおりサブスク枠のまま据え置きの予定だった。

- ターミナルでの**対話的な** Claude Code（人間が打って待つ使い方）
- Claude.ai の Web / デスクトップ / モバイルのチャット
- IDE 統合での対話利用
- Claude Cowork

つまり「画面に向かって自分でプロンプトを打つ」分には影響ゼロ。料金が跳ねるのは「裏でエージェントが勝手にモデルを叩き続ける」分だけ、という線引きだった。

### 同じ Claude Code が"二重人格"になる罠

一番ややこしいのがここで、**Claude Code というバイナリは同じなのに、使い方で消費される枠が変わる**点だった。

- ターミナルで対話的に使う Claude Code → **サブスク枠**
- `claude -p "タスク内容"` でヘッドレス実行する Claude Code → **自動化クレジット枠**

同じ `claude` コマンドなのに、`-p` を付けてスクリプトに組み込んだ瞬間に課金プールが変わる。「自分はサブスクで使ってるから関係ない」と思っていた人が、CI やシェルスクリプトの中で `claude -p` を呼んでいたら、知らないうちに自動化枠を消費していた——という事故が容易に起きる設計だった。ここが最大の混乱ポイントだった。

## 付与予定だったクレジット額

プランごとに、月額で以下の Agent SDK クレジットが付与される予定だった（オプトイン制）。

| プラン | 月額 | 付与予定だった自動化クレジット |
|--------|------|------------------|
| Pro | $20 | $20 |
| Max 5x | $100 | $100 |
| Max 20x | $200 | $200 |
| Team (Standard) | $25/シート | $20 |
| Team (Premium) | $125/シート | $100 |
| Enterprise | 利用ベース / シート | $20〜$200 |

右列はサブスク料金とは別枠で、API 相当額として自動化に使える枠として付与される予定だった（Pro なら $20 を払って、さらに $20 ぶんの自動化クレジットが乗る形）。

ルールも明確で、

1. **個人単位付与**：チームでもメンバー間でプール・共有は不可
2. **月次リセット**：未使用分の繰り越しなし
3. **オプトイン必須**：自分で有効化手続きが要る
4. **優先消費**：他の課金方式より先に減る
5. **超過後**：usage credits を有効化していれば追加課金、していなければ停止

要するに「Pro $20 なら毎月 $20 ぶんの API 相当を自動化に回せる。それを超えたら別途課金 or 止まる」。これまで実質サブスク内で青天井に近かった自動化が、ドル建てのメーターに乗る——という改定だった。

## なぜ分離しようとしたのか — サブスクがエージェントを補助している構造

背景は「対話と自動化ではモデルの叩き方の桁が違う」という、コスト会計上の問題だ。

人間が対話的に使う場合、1日に投げるプロンプトはせいぜい数十回。ところが自律エージェントは、テストを回し、ツールを呼び、再帰的に自分でモデルを叩いて、**1日に数千リクエスト**を平気で生成する。同じ「1サブスク」でも、消費するコンピュートは何十倍も違う。

エディタの Zed を作っている Zed Industries は、サブスクがエージェント利用を **API 価格比でおよそ15〜30倍も補助している**状態だと見積もっていた（報道ベース）。$20 のサブスクで $300〜$600 相当の API 利用が引き出せる「コンピュート・アービトラージ」が成立してしまっていた、という話だ。

この流れ自体は今回が初めてではない。

- **2026年2月**：Anthropic がこの手の自動化利用（サードパーティ経由のヘビーなエージェント運用）を一度禁止
- **2026年4月**：禁止を強化
- **2026年5月**：「分離課金（split-billing）」という形で再度開放する方針を告知 → これが今回の 6/15 施行予定だった

つまり「サブスクでエージェントを回されるとコンピュートが逆ザヤになる → 一度締める → 完全締めはコミュニティが嫌がる → クレジット枠で計量して開放する」という、コンピュート逼迫を起点にした一連の調整の最新版が、今回の Agent SDK 分離だったわけだ。

## そして施行当日、撤回された

ここからが本題で、**6/15 当日、Anthropic はこの変更を施行せず保留した**。Help Center とサブスクライバー向けの通知で、Agent SDK・`claude -p`・サードパーティアプリの別クレジット移行は「行わない」と確認された。

公式メッセージの原文はこう。

> "We're pausing the changes to Claude Agent SDK usage described below. For now, nothing has changed."
>（以下に記載した Claude Agent SDK 利用の変更を保留する。当面、何も変わらない）

サポートドキュメントも同日に更新され、Agent SDK 利用は引き続き標準のサブスク枠から消費される・受け取るべきクレジットは存在しない・サブスク上限は不変、と明記された。XDA Developers なども同日に追随報道している。

https://www.xda-developers.com/anthropics-claude-subscriptions-no-longer-include-agent-sdk-and-claude-p-usage/

撤回理由について Anthropic は、プランを **「ユーザーが Claude サブスクでどう開発しているかをより良くサポートする形に作り直している」** とだけ述べ、改定版のタイムラインは示していない。次の改定の前には事前告知をする、とも添えている。

施行を予告して開発者にオプトインの準備までさせておいて、**ローンチ当日に引っ込める**というのは、なかなかのドタバタだ。4月の「Pro プランから Claude Code をサイレント削除 → 数時間で撤回」を彷彿とさせる動き方で、課金まわりの意思決定がまだ固まりきっていない印象を受ける。

## 個人開発者目線：自分の自動化はどうなるのか

ここから個人開発者目線。`claude -p` や GitHub Actions に Claude を組み込んでいる側として、いまどう構えるべきか。

**結論：今は何も変わっていないので、据え置きでOK。ただし「いつ来てもいい準備」はしておく。**

具体的には3点。

- **自動化を本当にサブスク枠で回し続けるかは、設計として一度見直す価値がある**。今回は撤回されたが、Anthropic が「分離したい」という意思は明確に示した。改定版が来れば `claude -p` の cron 運用や GitHub Actions の自動実行は、いずれドル建てメーターに乗る前提でコストを試算しておくほうが安全
- **対話と自動化の境界を意識した使い分けを今のうちに整理しておく**。「ここは人間が見てる対話セッション」「ここは無人で回る自動化」をコード上で切り分けておけば、次の改定で自動化側だけ別 API キー / 別課金に逃がす対応が楽になる
- **`/usage` で消費の内訳を観測する癖をつける**。どのワークフローがどれだけトークンを食っているかを把握していないと、メーター制が来た瞬間に請求が読めなくなる

裏を返すと、**対話的に Claude Code を使っているだけの人は、今回もこれからも基本的に影響を受けない**。騒ぎの本体はあくまで「無人で大量にモデルを叩く自動化」の課金健全化であって、画面に向かって打鍵している分には今まで通りだ。

## この半年の Claude 課金まわりの動きと一緒に読む

今回の一件は単発で見るより、**この半年の連続したドタバタの最新エピソード**として読むと構図がはっきりする。時系列はこう。

| 時期 | 出来事 |
|------|--------|
| 2026/4/21 | Pro プランから Claude Code をサイレント削除 → 炎上 → 数時間で撤回（「2% A/B テスト」と釈明） |
| 2026/4/23 | 品質低下の公式ポストモーテム（推論最適化・キャッシュ最適化のバグが原因と認める） |
| 2026/5/6〜7 | SpaceX 300MW 提携発表＋ Claude Code 5時間制限2倍・ピーク時間撤廃 |
| 2026/5/13 | Agent SDK・`claude -p` の別クレジット化を 6/15 施行と告知 |
| 2026/6/5 | Opus 4.1 の廃止を告知（8/5 に API から削除予定、Opus 4.8 へ移行推奨） |
| 2026/6/15 | Agent SDK 分離を**施行当日に撤回** |

全部つなげると、根っこにあるのは一貫して **「コンピュートが足りない / 逆ザヤだから課金を締めたい」という供給側の事情**と、**「締めるとユーザーが逃げる」というコミュニティ側の反発**のせめぎ合いだ。SpaceX や Amazon・Google との大型コンピュート契約で供給を積み増しつつ、自動化利用の課金を健全化しようとしているが、その健全化の落とし所がまだ定まっていない——というのが今回の撤回ににじんでいる。

前回の SpaceX 緩和の記事で「Anthropic がいつまた制限を引き直すかは読めない」と書いたが、まさにその通りの展開になった。攻めに転じたと思ったら、今度は自動化課金で締めにかかり、それも当日撤回。ユーザー側としては、**良いニュースも悪いニュースも長期コミットだと思わない**くらいの距離感がちょうどいい。

## まとめ

今回のニュースを一言でまとめると、**「Claude の自動化課金を別枠に切り出す改定が予告されたが、施行当日 6/15 に撤回された」**。

- 対象は **Agent SDK・`claude -p`・GitHub Actions・Agent SDK 経由のサードパーティ**（＝自動化利用）
- 対話的な Claude Code・Claude.ai チャット・Cowork は**元から対象外**で、今後も影響なし
- 同じ Claude Code でも **`-p` を付けてヘッドレス実行すると課金プールが変わる**設計が混乱の核だった
- 背景は **サブスクがエージェント利用を API 比15〜30倍補助している**という逆ザヤ構造（2月禁止 → 4月強化 → 5月分離告知の流れ）
- だが **6/15 当日に撤回**。「当面、何も変わらない」とだけ告知され、改定版のタイムラインは未定

今は据え置きなので慌てる必要はない。ただし Anthropic の「分離したい」という意思は明確なので、`claude -p` の自動化や GitHub Actions を多用している人は、**次の改定でドル建てメーターに乗る前提**でコスト構造を整理しておくと、再告知が来たときに落ち着いて対応できる。

この半年の流れを通しで追いたい場合は、5月の SpaceX 緩和の記事から読むと、課金まわりの迷走の全体像がつかめる。
