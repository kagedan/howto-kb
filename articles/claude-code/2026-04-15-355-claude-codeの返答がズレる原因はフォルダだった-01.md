こんにちは、ちーみつです。

作業全部を一つのフォルダで管理していた友人が、  
作業場所をはっきりさせたら、AIからの返答が変わった話です。

## 欲しい返事が返ってこない

ClaudeデスクトップアプリでClaude Codeができるようになりました。  
バイブコーディングしやすくなったので、挑戦する人は増えたと思います。

私の友人はチャットを使う程度でしたが、  
デスクトップアプリでClaude Codeを使い始めました。

作業をしていてズレた返信が返ってくると相談されました。

指示を変えたりモデルを変えたりしてみたのですが、  
全く良くなりません。

その原因って、  
ツールの問題というより、  
他の文脈が混ざっていたんです。

## 作業ごとに場所を分ける

1つのフォルダでいろんな作業をしていたので、他の作業の文脈が混じっていたのが原因でした。

私は `code` という大元のフォルダの中に、  
文章を書くフォルダ、  
Webページを作るフォルダ、その中にページごとのフォルダ。  
アプリを作るフォルダ、その中にアプリごとのフォルダ。  
というふうに分けています。

![](https://assets.st-note.com/img/1776048829-qYK85Nsp3fPujD0GMoS7zQmy.png?width=1200)

こんなフォルダ分け

文章を書く場所には、  
文章を書く時に使う指示ファイルや参照ファイル、  
アーカイブなどがあります。

一方で、  
Webページやアプリの作業場所には、  
その作業に必要なものだけを置いています。

それぞれのフォルダに余計なファイルがないので、  
どこに何があるか把握しやすいんですよね。

## どのフォルダで作業するか

私の場合のフォルダ構成でいうと、  
友人はAI作業をするときにcodeフォルダを開いて作業していたんです。

私の場合は

code > Webページフォルダ > webサイトA フォルダ

を開いて作業しています。  
自分もファイルを探しやすいですし、  
他の情報が混じりようがありません。

先に作業場所を分けるだけでも  
AIが必要な情報を探しやすくなります。

## まとめ

思い返すと、私もAIで作業し始めた頃は、  
一つのフォルダで全部の作業をしていました。

フォルダ構成を整えて開く作業フォルダを変えると、  
AIが不要な情報を読まなくなります。  
欲しい結果が得やすくなりました。

フォルダが分かれていない場合は、  
一度作業ごとに整理してみるのもいいかもしれません。  
作業するときは、一番奥のフォルダを選ぶだけで変わることがあります。

みなさんは今、どのフォルダを開いて作業していますか？

最後まで読んでいただきありがとうございます。
