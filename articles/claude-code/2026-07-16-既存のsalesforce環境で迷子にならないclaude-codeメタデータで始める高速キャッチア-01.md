## はじめに

Salesforceの標準側でも開発側でも、すでにカスタマイズされ尽くした環境に新しく入り、「この機能を修正してください」と言われたとき。「最初にどこを見ればいいのか全くわからない…」という絶望感を経験したことのあるエンジニアは少なくないと思います。

最終的な目標はもちろん環境の完全な理解ですが、それには膨大な時間と調査が必要です。しかし、現場では「今すぐこの一部を修正してほしい」というスピードが求められます。そんなとき、どうすればいいでしょうか？

AI時代において、素早く検索し、目的のコードや設定に辿り着く方法があります。これを使えば、迷っている時間を減らし、良い設計や実装を考える時間に充てることができます。その方法とは、「Salesforceのメタデータ」と「ターミナルのClaude Code」の掛け合わせです。誰でも簡単に実践できます。

## 事前準備

まずはローカル環境とAIの準備を行います。

1. 環境の接続: VSCodeやCursorで Salesforce Extension Pack を導入し、調査したい組織に接続します。
2. メタデータの取得: プロジェクトを開き、組織上の検索したい自動化（フローやApexトリガーなど）のファイルをローカル（force-app/main/default 以下）にすべてダウンロードします。雲マークのついているSalesforce Org Browserで取得したいメタデータを見つけたら、下矢印のあるRetrieve Metadataボタン（雲マーク）を押します。  
   ![Salesforce Org Browserの雲マークメタデータ取得画面](https://static.zenn.studio/user-upload/058414e3b5a6-20260706.png)
3. ツールの起動: Claude Code をインストールし、VSCodeやCursorのターミナルで起動します。

## 基本的なやり方

Claude Codeが開いているターミナルに対して、ダウンロードしたメタデータファイルを対象に質問を投げます。  
💬 プロンプトの基本形: 「[ファイル名/フォルダ] を対象に、〇〇について調べてください」 「以下の質問に回答するために、関係する自動化処理（フローやトリガー）を検索してください：[質問内容]」  
![Claude Codeで質問しているターミナル画面](https://static.zenn.studio/user-upload/7578ea8359a5-20260706.png)

!

注意

* このプロセスはあくまで「第一調査」を高速化するためのものです。AIはハルシネーションを起こすこともありますし、ローカルにないメタデータは見えません。AIの回答を起点にしつつ、最終的には実際のソースコードや設定を確認してください。
* コードベースが巨大な場合は、調査対象のフォルダを絞り込んで読み込ませるのがコツです。

## 実践的なユースケース5選

具体的に、どのような場面でClaude Codeが威力を発揮するかを紹介します。

### ① フローやトリガーの素早いドキュメント化

複雑な個別のファイル名に対して、以下のように深掘りします。

* 「このフロー（またはApexクラス）の詳細を、ステップ・バイ・ステップで分かりやすく説明してください」

### ② 謎のエラーメッセージのソース特定

画面にエラーメッセージが表示されたものの、オブジェクトの「入力規則」を見てもそれらしき定義が見つからない場合、フロー内の「カスタムエラー」で制御されている可能性があります。

* 「『[表示されたエラーメッセージ]』という文言を含んでいるフローを探してください」

### ③ 目的の処理を行っている場所の検索

UI（設定画面）のフロー一覧から目視で探すよりも、テキストベースで一発検索する方が遥かに高速です。

* 「商談を作成している画面フローを探してください」
* 「取引先を削除しているレコードトリガーフローを探してください」
* 「これら2つの似た処理をやっているフローの違いを比較して説明してください」

### ④ 固有の命名規則やパターンの発見

新しい環境では、そのプロジェクト独自の命名規則や実装パターンがまだ把握できていないことが多いです。既存のスタイルに合わせないと、技術負債を生む原因になります。

* 「〇〇に関連するフローを全て探して、命名や構造の共通パターンを教えてください」

### ⑤ 権限・プロファイル分岐の特定

特定のユーザーだけでバグが起きる場合、条件分岐が怪しいです。一つ一つの分岐を手作業で追うのは時間がかかります。

* 「プロファイルや権限セット、レコードタイプによって処理を分岐しているフローやトリガーを探してください」

## まとめ

すでにカスタマイズされているSalesforce環境の調査は、どこから手をつければいいか迷いやすく、エンジニアにとって時間がかかる大変な作業です。

しかし、メタデータをローカルにダウンロードしてターミナルでClaude Codeを使うだけで、必要な場所を一瞬で見つけることができるようになります。

「何から調べればいいかわからない」という迷う時間を減らすことで、その分、良い設計や実装を考える時間を増やすことができます。新しい環境に入って調査に困ったときは、ぜひこの方法を試してみてください。
