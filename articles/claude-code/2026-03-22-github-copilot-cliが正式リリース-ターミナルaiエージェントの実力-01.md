## はじめに

2026年2月25日、GitHub Copilot CLIが全有料プランで正式リリース（GA）された。2025年9月のパブリックプレビュー開始から約5ヶ月、数百件の改善を経ての一般提供だ。

ターミナルで動くAIコーディングエージェントといえば、すでにClaude Codeが先行して存在感を示している。そこにGitHubが本格参入してきた形になる。Copilot CLIは単なるコード補完ツールではなく、計画・実行・レビューまでこなすエージェント型ツールに仕上がっている。ターミナル派の開発者にとって選択肢が増えたのは間違いない。

## Copilot CLIの主要機能

### PlanモードとAutopilotモード

Copilot CLIの特徴的な設計が、2つの動作モードの切り替えだ。

**Planモード**は、Shift+Tabで切り替えられる。タスクを指示すると、Copilotがまず要件を分析し、必要に応じて質問を投げかけ、構造化された実装計画を組み立てる。開発者が計画を確認・承認してから実行に移る流れだ。「何をするか」を事前に把握できるので、大きな変更を加えるときに安心感がある。

**Autopilotモード**は、指示を出したらあとはCopilotが自律的にタスクを完了まで進める。ツールの実行、コマンドの実行、エラー時のリトライまで自動だ。テストの修正やリファクタリングのような、ゴールが明確なタスクに向いている。

この2モードを状況に応じて使い分けられるのは実用的だと感じる。Claude Codeでも`--dangerously-skip-permissions`で自律実行に近いことはできるが、UIレベルでモードが分かれている方が直感的ではある。

### マルチモデル対応

Copilot CLIのもう一つの強みが、複数のAIモデルから選択できる点だ。現時点で利用可能なモデルは以下の通り。

* Claude Opus 4.6
* Claude Sonnet 4.6
* GPT-5.3-Codex
* Gemini 3 Pro

`/model`コマンドでモデルを切り替えられる。タスクの性質に応じてモデルを選べるのは、単一モデルに縛られるツールにはないメリットだ。コード生成の精度を重視するならClaude Opus、速度を優先するならGPT-5.3-Codexといった使い分けが考えられる。

### Fleetモード — 並列実行

GA時点で追加された目玉機能がFleetモードだ。`/fleet`コマンドで、複数のサブエージェントを並列実行できる。

たとえば、モノレポの複数パッケージで依存関係を一括アップデートするようなタスクでは、各パッケージの更新を並列で走らせて結果を集約できる。独立したタスクが多いほど効果が大きい。Claude Codeにも最大7並列のサブエージェント機能はあるが、Copilot CLIのFleetモードはユーザーが明示的に並列実行を指示できる点で、使い勝手が異なる。

### エンタープライズ向けの管理機能

GA と同時に、Enterprise AI Controls & Agent Control Planeも正式提供が始まった。組織全体のエージェントセッションを一元管理できる。どのモデルが使われているか、どのツールが呼び出されているか、プレミアムリクエストの消費量、ポリシーへの準拠状況を把握できる仕組みだ。

個人開発者にはあまり関係ないが、企業でAIコーディングツールを導入する際のガバナンス面では大きな差別化ポイントになる。

## Claude Codeとの棲み分け

ターミナルAIエージェント同士ということで比較は避けられない。ざっくり整理するとこうなる。

**Copilot CLIが強い場面:**

* GitHub Actionsやissueとの統合が必要なワークフロー
* 組織での一括導入・管理が必要なケース
* モデルを切り替えながら作業したいとき

**Claude Codeが強い場面:**

* 大規模リポジトリの深い文脈理解が必要なタスク（1Mトークンコンテキスト）
* MCP連携で外部ツールと接続するケース
* SWE-benchスコアが示すような高精度な自律タスク

実際には競合というより補完関係に近い。Claude CodeはCopilot Pro+やEnterprise内でサードパーティエージェントとしても利用できるので、両方を使い分ける開発者も多いだろう。

## 所感

正直なところ、Copilot CLIがここまで本格的なエージェントになるとは思っていなかった。プレビュー初期はシェルコマンドの提案程度の印象だったが、GAではPlan/Autopilot/Fleetと機能が揃って、Claude Codeと真正面から比較される存在になった。

自分が特に良いと思ったのはPlanモードの存在だ。エージェントに自律的に作業させたいけれど、いきなり全部任せるのは怖い、という場面は実務でよくある。計画を見てから実行を承認できるワークフローは、チームでの利用時にも安心感がある。

一方で、マルチモデル対応は嬉しいが、「結局どのモデルを使えばいいのか」という選択コストが発生する。モデルごとの得意分野を把握するまでに試行錯誤が必要で、ここは慣れるまで時間がかかりそうだ。

個人的には、普段のコーディングはClaude Codeをメインにしつつ、GitHub連携が絡む作業ではCopilot CLIを使う、という棲み分けに落ち着くのではないかと考えている。ツールが増えること自体は歓迎だ。

## まとめ

* GitHub Copilot CLIが2026年2月25日に全有料プランで正式リリース
* Plan/Autopilot/Fleetの3モードで、確認型から自律型まで柔軟に対応
* Claude Opus 4.6、GPT-5.3-Codex、Gemini 3 Proなど複数モデルを切り替え可能
* エンタープライズ向けのAgent Control Planeで組織管理にも対応
* Claude Codeとは競合より補完関係。用途に応じた使い分けが現実的
