先日、エンジニアサミットに参加してきました。会場で隣の席になった方から「資格50個持ってます」と言われた瞬間、自分の中の"当たり前"が音を立てて崩れていく感覚がありました。

本記事では、そこで受けた3つの衝撃を、AI/技術トレンドの観点から深掘りして整理します。単なる感想ではなく、各論点について「いま何が起きていて、これからどう動くべきか」を考察します。

## 衝撃①：AI活用は「差別化」ではなく「前提」になった

トップエンジニアの方々と話して最も驚いたのは、**全員が当たり前のようにAIを使いこなしていた**ことです。「使っているかどうか」ではなく「どう使いこなしているか」が問われるフェーズに、もう完全に入っています。

### 「AIを使っていない」こと自体がリスクである理由

数年前まで、ChatGPTやClaudeを使う/使わないは個人の選好の問題でした。しかし2026年現在、状況は明確に変わっています。

| 観点 | AIを使わない | AIを使いこなす |
| --- | --- | --- |
| コード生成速度 | ベースライン | 3〜10倍 |
| ドキュメント整備 | 後回しになりがち | 同時並行で生成可能 |
| 未知ドメインの学習 | 書籍/公式ドキュメント中心 | 対話で素早く全体像把握 |
| レビュー観点 | 経験に依存 | 観点漏れをAIに補完させる |

差が「努力で埋められる量」を超えつつあります。

### 使いこなし方の例：Claude Code を CLI で運用する

たとえばClaude Codeをローカルから呼び出して、定型タスクを自動化するパターンです。

```
# 未処理の投稿をAIでサニタイズ→ファイルに書き出す例
claude -p "次のテキストから企業名・固有名詞をマスキングしてください: $(cat raw.txt)" \
  > sanitized.txt
```

ただし、`claude -p` のような起動を高頻度cronに組み込むと**トークン課金が想定外に膨らむ**ので注意が必要です。決定論的に処理できる箇所（取得→ルール判定→通知）はNode/Pythonの素朴なスクリプトで十分で、LLMは「自然言語理解」「推論」「文章生成」「不規則イベント対応」に絞るべきです。

## 衝撃②：学習量の「桁」が違う

毎日深夜3時まで勉強している方がいました。重要なのは**辛そうではなく、楽しそうにやっている**こと。好奇心をエンジンにできる人は本当に強いと感じました。

### 学習の入口が広がっている

会話に出てきた「AWS Cloud Quest」のように、**ゲーム感覚で学べる**サービスが増えています。

| サービス | 特徴 | 向いている人 |
| --- | --- | --- |
| AWS Cloud Quest | 3Dゲーム内でAWSサービスを構築 | クラウド初学者 |
| Google Cloud Skills Boost | 実環境で動かすラボが豊富 | 手を動かして覚えたい人 |
| Microsoft Learn | 体系的なラーニングパス | 資格取得志向 |
| HackTheBox / TryHackMe | セキュリティ実習 | レッドチーム志向 |
| Roadmap.sh | キャリアパスの可視化 | 全体像を掴みたい人 |

「テキストを黙々と読む」だけが学習ではなく、**手と頭を同時に動かせる教材**が圧倒的に増えています。

### 学習を継続するためのTips

* **学習ログをGit管理する**：日々の学びを `learning/yyyy-mm-dd.md` に書き続けるだけで継続率が上がる
* **AIを"ソクラテス役"にする**：「この概念を5歳児に説明してみて」「次に勉強すべき周辺概念は？」と問い続ける
* **目的を一つに絞らない**：「資格のため」「業務のため」「副業のため」を重ねると挫折しにくい

```
# learning/2026-05-12.md
## やったこと
- AWS Cloud Quest「Cloud Practitioner」ステージ3クリア
## 学んだこと
- IAMロールとリソースベースポリシーの違い
## 次やること
- Cloud Questの次ステージ + Terraformで再現
```

## 衝撃③：ダブルワークが「当たり前」になっていた

本業と副業で**専門性を掛け算**して市場価値を高めている方が、想像以上に多数いらっしゃいました。1社の中だけでキャリアを作る時代は、静かに終わりを迎えつつあります。

### 副業が生む"好循環"

* 本業では触れられない技術スタック（例：Rust、Wasm、AI Agent SDK等）を副業で触る
* そこで得た知見を本業のリファクタや改善提案に還元
* 結果として本業の成果も上がり、評価される
* さらに副業の幅が広がり、収入と裁量が増える

### よくある質問（FAQ）

**Q. 副業禁止の会社にいる場合は？**  
A. 就業規則の確認が最優先です。OSSコントリビュート、技術記事執筆、登壇など「無償でも経験値になる活動」から始めるのも選択肢です。

**Q. 何から始めればいい？**  
A. クラウドソーシング（Lancers/CrowdWorks/ココナラ）で小さな案件を1件こなすのが現実的です。重要なのは「収入」より「外部の評価軸に晒される」体験です。

**Q. 本業との両立がしんどくなったら？**  
A. 撤退基準を最初に決めておくことです。「本業の評価が下がったら」「睡眠時間が6時間を切る日が週3日続いたら」など、定量的なラインを引いておくと判断が楽になります。

## 結論：スキルアップは「業務外の趣味」ではなく「生存戦略」

技術を使って商売をする会社にいる以上、自分自身のスキルアップは**生存戦略**そのものです。そして何より痛感したのは、**強いエンジニアの近くにいると基準が勝手に上がる**ということ。

環境への投資が、もしかしたら一番リターンの大きい自己投資なのかもしれません。
