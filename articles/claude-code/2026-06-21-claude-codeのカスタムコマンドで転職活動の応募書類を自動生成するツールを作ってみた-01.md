## はじめに

転職活動をしていると、応募企業ごとに志望動機・自己PR・職務要約を毎回書き直す作業が地味に重いタスクとして積み上がっていきます。

「言いたいことの骨子は同じなのに、企業に合わせて文章を組み替えるだけで30分〜1時間かかる」

この問題を、Claude Codeのカスタムスラッシュコマンド機能を使って解消できないかと考え、**「転職活動 応募書類自動生成キット（job-hunting-kit）」**を作りました。

本記事では、ツールの設計思想と使い方、そして実際にコマンドを叩いたときの挙動を技術的な視点でまとめます。

:::note info
本ツールは無料配布中です。ダウンロードリンクは記事末尾にあります。
:::

## なぜClaude Codeで作ったのか

ChatGPTやClaudeのWeb版にプロンプトをコピペする方法でも、同様のことは一応できます。しかしその方法には以下の課題がありました。

- **毎回キャリア情報を貼り直す必要がある**(チャット履歴が流れると再入力)
- **過去の出力フォーマットを覚えていない**ため、企業ごとに文体がブレる
- **ローカルにファイルとして残らない**ため、自己PRの「最新版」がどれか分からなくなる

Claude Codeであれば、キャリア情報を**プロジェクト内のMarkdownファイルとして永続化**でき、コマンド実行のたびにそのファイルを文脈として読み込ませることができます。これにより「一度登録すれば、あとは会社名を渡すだけ」という体験を作れると考えました。

## ツールの構成

このキットは2つのカスタムスラッシュコマンドで構成されています。

| コマンド | 役割 | 実行頻度 |
|---|---|---|
| `/setup` | キャリア情報をヒアリングし `my-career.md` に保存 | 初回のみ（変更時は再実行可） |
| `/apply` | 企業名・ポジションを渡して応募書類一式を生成 | 応募のたびに毎回 |

ディレクトリ構成のイメージは以下の通りです。

```
job-hunting-kit/
├── .claude/
│   └── commands/
│       ├── setup.md      # /setup コマンドの定義
│       └── apply.md      # /apply コマンドの定義
└── my-career.md           # /setup実行後に生成されるキャリア情報ファイル
```

Claude Codeの`.claude/commands/`配下にMarkdownファイルを置くことで、独自のスラッシュコマンドとして認識される仕組みを利用しています。

## 使い方

### ① キットを開く

ダウンロードした`job-hunting-kit`フォルダをターミナルで開き、Claude Codeを起動します。

```bash
cd job-hunting-kit
claude
```

### ② `/setup` でキャリア情報を登録する（初回のみ）

```
> /setup
```

実行すると、Claudeが対話形式で以下の項目を一つずつヒアリングしてくれます。

- 現在（または直近）の職種・業界・経験年数
- これまでの職歴（会社名・役職・業務内容・在籍期間）
- 仕事での成果・実績（数字があると理想的）
- 自分の強みとエピソード
- 保有スキル・資格・専門知識
- 今回の転職で実現したいこと

所要時間はだいたい5〜10分です。回答が完了すると、`my-career.md`が自動生成されます。このファイルは直接テキストエディタで編集することも、`/setup`を再実行して上書きすることも可能です。

### ③ `/apply` で応募書類を生成する

企業ごとに以下のように実行します。

```
> /apply 株式会社〇〇 バックエンドエンジニア
```

会社名・ポジション（必要であれば事業内容も）を渡すと、`my-career.md`の内容をもとに、その企業向けにカスタマイズされた**志望動機・自己PR・職務要約**が一括生成されます。

### ④ 修正・ブラッシュアップ

生成された文章はその場で追加指示を出して微調整できます。

```
> もう少し簡潔にして、成果を数字で強調して
```

Claude Codeの会話継続性を活かして、生成→指摘→再生成のループを素早く回せるのが、コピペ運用との一番の違いです。

## 設計上のポイント

### キャリア情報をローカルファイルとして永続化する設計にした理由

`my-career.md`をプロジェクト内に保存する設計にしたことで、以下のメリットがあります。

- Gitで差分管理すれば、職務経歴の変遷を履歴として残せる
- 他のツール（履歴書PDF生成スクリプトなど）からも同じファイルを参照できる
- セッションをまたいでも情報が消えない

### プライバシーについて

登録したキャリア情報（`my-career.md`）は**ローカルフォルダ内にのみ保存**され、外部サーバーへのアップロードは行われません（Claudeへの送信を除く）。機微な経歴情報を扱うツールなので、この点は設計時に意識しました。

## 今後の改善アイデア

現状はシンプルな2コマンド構成ですが、以下のような拡張を検討しています。

- 求人票のURLやテキストを渡すと自動で事業内容を要約して`/apply`に渡す機能
- 生成した応募書類を職務経歴書フォーマット（Word/PDF）に直接出力する機能
- 複数企業分の生成履歴を一覧管理するコマンド
