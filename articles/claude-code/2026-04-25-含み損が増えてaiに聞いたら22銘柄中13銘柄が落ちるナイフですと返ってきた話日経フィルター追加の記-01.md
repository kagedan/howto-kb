「自動売買ツールが動いているのに、なんでこんなに含み損が増えているんだろう」

2026年4月3日（金）の前場の途中。ツールの画面を見ながらそう思いました。RSI逆張り戦略で動かしているのに、ポジションが全然反発しない。むしろ買うたびに下がっていく感覚がありました。

この日、Claude Codeに相談してトレードログを分析させてみたら、思いがけず明確な答えが返ってきました。

---

## 何が起きていたか

4月3日は前日の米国関税ショックの余波で、日本株全体が下落基調でした。

私のツールは「RSIが≤40まで売られたら逆張りで買う」という設定で動いています。理屈の上では「売られすぎた銘柄がいずれ反発する」という戦略です。

ただ、この日は違いました。買っても買っても下がる。同じ銘柄に2回・3回とシグナルが出て追加エントリーするたびに、さらに安い価格で買わされていく。

前場の途中で、ツールを一時停止してClaude Codeに投げました。

```
含み損が多いです。
エントリーポイントが悪いでしょうか？
それとも日経平均を見て下がっていたらエントリーしないという、
別の指標を入れたほうがよいでしょうか？
まずは提案してください。
```

---

## AIがトレードログを分析した結果

Claude Codeはまず「今日のトレードデータと価格推移を整理してから提案します」と答え、trade\_log.csvを自分で読み込んで分析を始めました。

数十秒後、分析結果が出ました。

**22銘柄中13銘柄（59%）で「落ちるナイフパターン」を検出**

落ちるナイフパターンとは、2回目のエントリー価格が1回目より低かったケースのことです。つまり「買った後も価格が下がり続けていた」銘柄が22銘柄中13銘柄あったということです。

特にひどかった3銘柄の推移：

```
6580（ライトアップ）
  1187 → 1138 → 1122 → 1116 → 1113  ▼▼ 急落

5985（サンコール）
  1432 → 1401 → 1398 → 1393         ▼▼

8835（太平洋興発）
  1117 → 1116 → 1110 → 1105          ▼
```

これを見てはっきりわかりました。エントリーポイントの問題ではない。市場全体が下落トレンドに入っているときに、逆張りをしても「底打ちからの反発」ではなく「下落途中の一時的な反発」を掴んでしまうということです。

AIの診断はこうでした。

> 個別銘柄のRSIは正常に機能していたが、市場全体が下落トレンドの中で逆張りをしていたのが根本原因です。RSIが40以下 = 売られすぎ、は正しい。しかし市場全体が崩れている日は「売られすぎからさらに売られる」状態が続きます。

---

## 提案された3つの解決策

AIは具体的な改善案を3つ出してきました。

### 提案A：日経平均トレンドフィルター ★最優先推奨

日経平均が当日始値比-1%以上下落している場合、その日はエントリーを全停止するフィルターです。

### 提案B：個別株モメンタム確認

エントリー直前の直近3本の5分足がすべて陰線（下落）だった場合、そのシグナルをスキップする仕組みです。

### 提案C：複合フィルター強化

RSIの閾値を35→25に戻す、ボリンジャーバンドや移動平均乖離のフィルターをオンにするなど、そもそもスクリーニング基準を厳しくする方法です。

AIの推奨は「A + B の組み合わせで、この2つで今日の損失の70〜80%は回避できたと推定されます」というものでした。

私は迷わずA+Bを選びました。ただ1つ修正リクエストを追加して。

```
推奨：A + B の組み合わせで、
-1%から上昇してきた場合はエントリーできるような
組み合わせはできますか？
```

「-1%以下でも上昇に転じたならエントリーしたい」という意図です。

返答は「できます」。

```
それで実装してください
```
