最近、AIへの仕事の渡し方が「その都度頼む」から「定時に勝手に走らせる」に完全に切り替わってきた。

うちはClaude Code / Coworkで、定型業務をスキルにして時刻指定で自動実行している。たとえばSNSの下書きを毎営業日の決まった時間に自動生成して、承認用にSlackへ投げておく、みたいな運用をいくつも並行で回している。やってみて効いてくるのは、こっちの仕事が「作業そのもの」から「その業務をどう動かすべきかをスキルとCLAUDE.mdに書く」ことに移った点。一回うまく言語化して仕組みにしておけば、あとは毎日勝手に走るので、レバレッジが日数ぶん積み上がっていく。

面白いのは、AIに任せる単位が「一回のタスク」じゃなく「定常プロセスまるごと」に上がること。「この記事を書いて」じゃなく「毎朝この手順で素材を集めて下書きまで作っておいて」を渡せる。人間は起動と判断（GoかNoGoか）に集中して、実行の大半は寝ている間に終わっている。

ただ、勝手に走るぶん「間違った前提のまま定常で回り続ける」リスクも出てくるので、最後の承認と、ときどきの手順の見直しは人が握る。AIに任せるほど効いてくるのは"毎回の指示"じゃなく"そのプロセスをどう設計して定常化するか"のほうだなと思う。Human as Architect って、コード生成だけじゃなく業務の回し方そのものに効いてくるのかもしれない。
