「スマホでコードが書けたら最高なのに」と思ったことはありますか？

私もそう思って、先月まるまるスマホからClaude Codeを動かす実験をしていました。正直、めちゃくちゃ苦労しました。でも、その苦労のおかげで「やっていいこと・悪いこと」がはっきり見えてきたんです。

今回は、スマホでClaude Codeを使いたい人に向けて、**実際に踏んだ落とし穴と、それを回避するためのプロンプトをコピペできる形でお届けします。**

---

## そもそも、スマホからClaude Codeを動かすってどういうこと？

Claude Codeとは、Anthropicが提供するCLI（コマンドラインインターフェース）ツールです。通常はPCのターミナルから使います。

ではスマホからどうやって動かすか？主な方法はこの2つです。

どちらも「スマホそのものでClaude Codeを動かす」わけではなく、**スマホをコントローラーとして使う感覚**です。ここを誤解すると、最初の落とし穴にはまります。

---

## 落とし穴①：「スマホだけで完結する」という幻想

Terminusを使えばスマホからサーバー操作はできます。でも、VPS（仮想プライベートサーバー）かクラウド環境が別途必要です。

私が使ったのはConoHa VPS（月額880円〜）。初期設定だけはPCでやりました。「完全スマホだけ」は現時点ではかなり無理があります。

**現実的な構成はこれ：**

```
スマホ（Termius）→ VPS or クラウドサーバー → Claude Code実行
```

「初期設定はPCで30分、それ以降はスマホで操作」が現実的なラインです。

---

### 🔁 コピペで使えるプロンプト①：環境確認プロンプト

Claude Codeに接続した最初に使えるプロンプトです。

```
あなたは私のコード実行アシスタントです。
まず以下を確認してください：
1. 現在の作業ディレクトリ（pwd）
2. Pythonのバージョン（python3 --version）
3. 利用可能なメモリ（free -h）
4. 確認結果を日本語でまとめてください。
問題があれば、その場で修正提案もしてください。
```

これを毎回冒頭で使うことで、「環境が想定通りか」を30秒で確認できます。

---

## 落とし穴②：スマホの画面でコードを読むと必ずミスる

スマホの画面幅は360〜430px程度。コードのインデントや改行がずれて表示されます。私はこれでPythonのインデントエラーを3回も見逃しました。

**対策はシンプルで、Claude Codeへの指示を「短い出力」に絞ること。**

---

### 🔁 コピペで使えるプロンプト②：スマホ最適化出力プロンプト

```
以下のルールで出力してください：
- 1回の返答は100行以内に収める
- コードブロックは必要な部分だけ抜粋する
- エラーの場合は「エラー箇所」「原因」「修正コード」の3点だけ
- 専門用語を使う場合は括弧内で日本語説明を添える
- 「次に何をすべきか」を必ず最後の1行で教えてください
```

これを使うだけで、スマホ画面でも格段に読みやすくなりました。

---

## 落とし穴③：ネットが不安定な場所でClaudeを使うと地獄を見る

カフェのWi-Fiで作業中、接続が途切れてSSHセッションが切れました。途中まで実行していたスクリプトがどこまで動いたか不明になり、1時間分の作業が消滅。泣きました。

**対策は「tmux（ティーマックス）」というツールをサーバーに入れておくこと。**

tmuxを使えば、SSH接続が切れてもサーバー上の作業が継続されます。再接続すれば続きから作業できます。

---

### 🔁 コピペで使えるプロンプト③：ネット切断リスク回避プロンプト

スクリプトを実行する前にClaude Codeへ使う指示です。

```
これから実行するスクリプトは、通信が途切れた場合でも
安全に再開できるよう設計してください。
具体的には：
- 処理の途中経過を「progress.log」に書き出す
- エラーが起きた行番号と内容を記録する
- 再実行した際は、完了済みのステップをスキップする
上記3点を組み込んだ形でコードを書いてください。
```

これを習慣にしてから、途中で接続が切れても「どこまで動いたか」が一目でわかるようになりました。

## スマホから向いているタスク・向いていないタスク

実際に試してわかった分類です。

| タスク | スマホ向き | 理由 |  
|---|---|---|  
| 短いスクリプトの修正 | ⭕ | 出力が短く読みやすい |  
| ログの確認・分析 | ⭕ | 読み取りがメインなので |  
| 自動化スクリプトの実行 | ⭕ | 実行指示だけなら簡単 |  
| 大規模ファイルの編集 | ❌ | 画面で追えない |  
| 複数ファイルを跨ぐリファクタ | ❌ | コンテキストが複雑すぎる |  
| 初期環境構築 | ❌ | タイピング量が多すぎる |

スマホは「実行と確認」に使い、「設計と構築」はPCで済ませる。これが今のところの最適解です。

---

### 🔁 コピペで使えるプロンプト④：タスク難易度の事前判定プロンプト

作業を始める前に使えます。

```
私はスマホからSSH接続でこの作業をしようとしています。
【やりたいこと】
（ここに作業内容を書く）

以下を教えてください：
1. スマホ作業に向いているか（◎/○/△/×で評価）
2. 向いていない場合、どの部分だけスマホでできるか
3. 作業を小分けにするなら、どう分割すべきか
4. 予想される所要時間（スマホの場合 vs PCの場合）
```

このプロンプト、地味に便利です。「あ、これはスマホでやるべきじゃなかった」という後悔が激減しました。

## 実際にスマホだけで完結した作業例

参考までに、私がスマホ＋Claude Code（VPS経由）で実際にやった作業です。

* **RSSフィードの自動取得スクリプト**：毎朝7時に動くcronジョブの設定。作業時間25分
* **CSVデータの整形処理**：クライアントから受け取ったデータをSlackに送る形式に変換。作業時間40分
* **WordPressのプラグインエラー確認**：エラーログを読んでClaude Codeに原因特定させた。作業時間15分

どれも「スクリプトの全設計はPCで済ませ、修正・実行・確認をスマホで」というパターンです。

---

## まとめ：スマホ開発は「補助輪」として使うのが正解

スマホからClaude Codeを使うのは「できる」。でも「スマホだけで全部できる」は現時点では幻想です。

**現実的な使い方はこれ：**

1. 設計・構築 → PC
2. 実行・確認・微修正 → スマホ

この役割分担を決めてから、移動中や外出先でも作業が続けられるようになりました。「隙間時間に1タスク終わらせる」感覚で使うのが一番ストレスがないです。

今回紹介した4つのプロンプトは、全部コピペしてそのまま使えます。まずはどれか1つ、今日の作業で試してみてください。
