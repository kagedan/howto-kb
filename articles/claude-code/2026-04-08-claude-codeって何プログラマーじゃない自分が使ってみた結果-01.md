こんにちは。  
町工場でひとり事業部を運営している40代のおじさんです。

以前、「AI Coding」という記事を書きました。  
AIがコードを書いてくれる時代になった、という話です。

今日はもう少し具体的に、**自分が実際に使っている「Claude Code」というツール**について書きます。

先に言っておくと、自分はプログラマーではありません。  
半年前まで、「ターミナル」という言葉すら知りませんでした。

## Claude Codeって何？

ひとことで言うと、**パソコンの「黒い画面」（ターミナル）の中で、AIと一緒にプログラムを作るツール**です。

普通のClaude（チャット画面のやつ）との違いは、**AIが直接パソコンのファイルを読み書きできる**こと。

チャットのClaudeは「こういうコードを書くといいですよ」と教えてくれる。  
でも、実際にファイルを作ったり、保存したり、実行したりするのは自分でやらないといけない。

**Claude Codeは違う**。

**「このフォルダにこういうファイルを作って」と言えば、AIが直接作ってくれる。  
「このコードを実行して」と言えば、実行してくれる。  
エラーが出たら、AIが自分で直してくれる。**

チャットのClaudeが「相談相手」だとしたら、Claude Codeは\*\*「一緒に手を動かしてくれる作業パートナー」\*\*です。

## 最初の印象は「怖い」だった

正直に言います。

最初にClaude Codeを使ったとき、**怖かった**です。

黒い画面にコマンドを打つ。  
AIが勝手にファイルを作り始める。  
勝手にコードを書き始める。  
何をしているかよくわからない。

「これ、大丈夫なのか？　パソコン壊れないか？」

本気でそう思いました。

でも、使っているうちに気づいたんです。  
\*\*AIは毎回「これをやっていいですか？」と聞いてくる。  
\*\*勝手に暴走するわけじゃない。  
許可を出さないと動かない。

それがわかってからは、だいぶ安心して使えるようになりました。

## 実際に何を作ったか

Claude Codeを使って、自分はこんなツールを作りました。

**見積自動化システム。** メールを監視して、図面PDFを読み取って、テンプレートに自動入力するやつ。これは別の記事で詳しく書きました。

**NCデータ転送システム。** 加工機にデータを送るためのツール。FTPという仕組みを使うんですが、機械によって設定が違って、けっこう苦労しました。

**タイムカードシステム。** 共有フォルダで管理できる出退勤記録のアプリ。

**生産管理システム。** 受注から納品までの工程を自動で管理するもの。

全部、Claude Codeで作りました。  
全部、自分ひとりで。

## 普通のClaudeとどう使い分けているか

自分の使い分けはこうです。

**普通のClaude（チャット）：** 考える作業。記事の構成を練る、アイデアを出す、調べものをする、相談する。

**Claude Code：** 作る作業。実際にコードを書く、ファイルを作る、ツールを開発する、エラーを直す。

たとえるなら、普通のClaudeは\*\*「会議室での打ち合わせ」**。  
Claude Codeは**「現場で一緒に手を動かす」\*\*。

どっちも大事ですが、ものづくりの人間としては、Claude Codeの「一緒に手を動かしてくれる感じ」が性に合っています。

## プログラマーじゃなくても使えるのか

使えます。自分が証拠です。

ただし、最初はしんどい。

ターミナルの使い方がわからない。  
Gitって何？  
Pythonのバージョンが違う？　  
環境構築で丸1日潰れたこともあります。

でも、**そのしんどさを超えたあとの世界が、全然違う。**

「こういうツールがあったらいいのに」と思ったものを、自分で作れる。  
外注しなくていい。  
仕様書も書かなくていい。  
「こうしたい」と伝えるだけで、AIが形にしてくれる。

自分がClaude Codeで一番感動したのは、\*\*「作りたいものを言葉にするだけで、目の前にツールが現れる」\*\*という体験でした。

## ずんだもんの話

ちょっと脱線しますが、自分はClaude Codeに「ずんだもん」の声で通知させています。

作業が終わったとき、エラーが出たとき、許可を求めているとき。  
ずんだもんの声で「終わったのだ！」「エラーなのだ！」と教えてくれる。

これ、ただの遊びみたいですが、実は意外と実用的です。

Claude Codeは処理に時間がかかることがある。  
その間、別の作業をしていて、終わったのに気づかないことがあった。  
声で通知が来ると、すぐに戻れる。

**道具は、楽しく使えるほうが続く。** これは製造業でもAIでも同じだと思っています。

## 最後に

Claude Codeは、プログラマーのためのツールです。

でも、**プログラマーじゃない人間が使ったとき、一番大きなインパクトがある**と自分は思っています。

なぜなら、「作れなかった人が作れるようになる」変化のほうが、「作れる人がもっと速く作れるようになる」変化よりも、圧倒的に大きいから。

黒い画面は怖い。  
ターミナルは意味不明。  
環境構築で心が折れそうになる。

でも、その先に「自分で作れる世界」がある。

製造業の現場で働く43歳のおじさんでもできました。  
あなたにもきっとできます。
