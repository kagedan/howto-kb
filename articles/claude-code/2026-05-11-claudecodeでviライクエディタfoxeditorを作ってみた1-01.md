## 2. なぜ作ろうと思ったのか

きっかけは、本当に小さな苛立ちだった。

viには Normalモード（カーソル移動や削除を行う閲覧寄りのモード）と Insertモード（文字を入力する編集モード）がある。これがviの強さの源でもあるのだが、慣れていないうちは──いや、ある程度慣れてもなお──「今どっちのモードなのか」が分かりにくい。

たとえば、こんな経験はないだろうか。

`i` を押したつもりが押せておらず、入力したい文字がコマンドとして解釈されて画面が壊れる

Escを押し忘れたまま離席し、戻ってきて何かキーを叩いたら文章の真ん中に変な文字列が紛れ込んでいる

画面下の `-- INSERT --` 表示は出ているのだが、コードに集中していると視界に入ってこない

ステータス行の文字表示は、確かに正しい情報ではある。ただ、それを 無意識のうちに認識できる強さ を持っていない。集中しているときほど、文字より色や形のほうが脳に届く。

（現状、筆者は分からなくなったらEscボタンを2回押して閲覧モードにしている……）

ならば、エディタの 枠そのものの色を変えてしまえばいい。Normalモードなら緑、Insertモードなら赤。視界の端でも分かるように、画面全体に「今どっちか」を主張させる。

これが、FoxEditorのコアコンセプトになった。

## 3. 現状の仕様（v0.1）

コンセプト：モードを"色"で見える化  
FoxEditorは、ひとことで言えば

viのモードを、色で見える化したGUIエディタ

である。秀丸エディタやSakura Editorのような実用派のWindowsエディタの雰囲気を持ちつつ、操作体系はvi/Vimに寄せる。そのうえで、モード状態をテキストではなく 枠線の色 で表現する。

| モード | 枠線色 | 意味 |
| --- | --- | --- |
| Normal | 緑 `#00ff66` | 安全、閲覧、移動 |
| Insert | 赤 `#ff3030` | 編集中、入力可能、注意 |

背景は黒 `#000000`、文字はオレンジ `#ff9900`。ターミナルっぽい雰囲気と、夜間でも目に優しい配色を意識した。

左：Insert 右：Normal  
![](https://static.zenn.studio/user-upload/ee1067fd28ba-20260510.png)  
並べてみると、かなり分かりやすい。これが第一目標だったので、満足している。

技術スタック  
言語は Python 3、GUIは tkinter。Python標準ライブラリだけで完結し、追加インストール不要、1ファイルでWindowsそのまま動く。エディタは試行錯誤の塊なので、書いて即動かせる軽さを優先した。

実装した機能  
モード：Normal / Insert / Command の3つ。起動時はNormal。

Normalモードのキー

| キー | 動作 |
| --- | --- |
| `h` `j` `k` `l` | カーソル移動 |
| `i` | Insertモードへ |
| `x` | 1文字削除 |
| `:` | Commandモードへ |

Insertモード：通常の文字入力、Enterで改行、Backspaceで削除、EscでNormalへ戻る。

Commandモード：`:w`（保存）／`:q`（終了）／`:wq`（保存して終了）。

ステータスバー：画面下部に、現在モード／ファイル名／未保存変更マーク `*` ／行番号／列番号を表示する。

-- NORMAL -- | test.txt | 行:1 列:1  
-- INSERT -- | test.txt \* | 行:4 列:1

ファイル入出力：UTF-8で読み込み、失敗したらCP932で再試行。読み書き失敗時はアプリを落とさずステータスバーにエラー表示。`:q` で未保存があれば1回目は警告、2回目で強制終了。×ボタンでは保存確認ダイアログを表示する。

Vimユーザーから見れば「えっ、これだけ？」というレベルだが、自分専用の小さなエディタが、自分の手で動いている という事実が、思っていたより嬉しい。

## 4. これからの展望

v0.1はここで一旦区切りにする。ロードマップは以下の通り。

| バージョン | 追加機能 |
| --- | --- |
| v0.2 | 検索機能（`/`、`n`、`N`、ハイライト） |
| v0.3 | 行操作（`dd`、`yy`、`p`） |
| v0.4 | Undo / Redo |
| v0.5 | 行番号表示（`:set number`） |
| v0.6 | コマンド拡張（`:e`、`:q!`、`:help`） |
| v1.0 | 検索・保存・行番号・vi基本操作が揃った、Windowsで普通に使える軽量エディタ |

ロードマップは長いが、急がずに少しずつ育てていく。  
※そのうち、ローカルGemma4と接続してコード自動生成とかしたい

自作エディタは、「実用的かどうか」で測るとどうしても既製品に勝てない。VS CodeもVimも秀丸も、十分すぎるほど完成されている。それでも、自分のためだけに作った小さな道具 には、既製品にはない手触りがある。困りごとを自分で定義して、自分で設計して、自分で動かす。色ひとつ、ショートカットひとつに自分の意図が宿る。

FoxEditorは、まだ生まれたばかりだ。これからゆっくり育てていく。
