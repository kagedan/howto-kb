## CLAUDE.md とは？？

CLAUDE.md とは、Claude に対する**説明書・ルールブック**を書くファイルです。  
Claude との会話(セッション)を開始時に自動的に読み込まれ、CLAUDE.md に書かれた内容は Claude が理解してくれる特別なファイルです。

みなさんもこういった経験はありませんか？  
「コーディングをする場合はコメントルールを反映させたい」  
「語尾ににゃーをつけ続けたい」  
このような、一つの会話だけでなく、ずっと続けてほしい内容を Claude Code へ提示するファイルが**CLAUDE.md**ファイルです。

### CLAUDE.mdの種類

CLAUDE.md はいくつか置く場所によって意味合いが異なってきます。

| ファイルの場所 | 効果範囲 |
| --- | --- |
| ~/.claude/CLAUDE.md | 全プロジェクト共通のルール |
| /"プロジェクト直下"/CLAUDE.md | そのプロジェクト専用のルール |
| /"プロジェクト"/"サブフォルダ"/CLAUDE.md | そのサブフォルダ内だけのルール |
| CLAUDE.local.md | 自分だけのルール（.gitignoreに追加してGit管理外にするのが推奨） |

※複数ある場合は**より具体的・近い場所のファイルが優先**されます。

## 内容について

[Anthropic公式のベストプラクティス](https://code.claude.com/docs/en/best-practices)では、以下のように整理されています。

| ✅ 書くべき内容 | ❌書かない方がいい内容 |
| --- | --- |
| Claude がわからない Bash コマンド | コードを読めばわかること |
| デフォルトと異なるコードスタイル | Claudeがすでに知っている標準的な言語規約 |
| テスト方法・使用するテストランナー | 詳細な API ドキュメント(URLを張ればおｋ) |
| ブランチ名・PR の命名規則 | 長い説明やチュートリアル |
| プロジェクト固有のアーキテクチャ | 長い説明やチュートリアル |
| 必要な環境変数・サービスの設定 | 「きれいなコードを書く」のような自明なこと |
| はまりやすいポイント・非直感的な挙動 | ファイルごとのコードベース解説 |

つまり、「この行を削除したら、Claude がミスするか？(抽象的か)」の問いに  
→**YESなら書く**、**NOなら削除**

## CLAUDE.md を作成してみよう！！

では実際に CLAUDE.md を作成していきます。

ですが、ちょっと寄り道、  
[Claude Code のベストプラクティス](https://code.claude.com/docs/en/best-practices#explore-first-then-plan-then-code)には「**まず探索し、次に計画を立て、それからコーディングする。**」とあります。  
ここのどこに CLAUDE.md を作成する手順が入ってくるのでしょうか？？

正解は、「一番初めの探索の前、**準備段階**です。」  
全体のワークフロー的には下記のイメージです。

```
【準備】CLAUDE.md を作る
    ↓
【Step 1】探索（プランモード）
    ↓
【Step 2】計画（プランモード）
    ↓
【Step 3】実装（ノーマルモード）
    ↓
【Step 4】コミット＆PR（ノーマルモード）
```

ただし、CLAUDE.md を作る前に、プロジェクトの規模感によっては README.md などの初めのドキュメントファイルを用意しておくべきです。

CLAUDE.md を Claude Code が作ってくれるコマンドが下記です。

これを実行すると、コードベースを解析して CLAUDE.md を作成してくれます。  
作成は以下ステップで実行されます。

1. ディレクトリツリーを走査 — プロジェクト種別を特定
2. 設定ファイルを解析 — package.json, tsconfig.json, pyproject.toml などを読んでスタック・依存関係・スクリプトを抽出
3. アーキテクチャを把握 — ソースディレクトリ、コアモジュール、エントリーポイントを特定
4. コーディング規約を検出 — ESLint, Prettier, lint-staged などの設定を確認

→つまり、`/init`は「このコードベースを解析して CLAUDE.md を作成してください。」というプロンプトを打っている感覚です。

CLAUDE.md は必要最小限で！！  
AI にはコンテキストウィンドウという AI が覚えて置ける容量のようなものがあり、CLAUDE.md は常に呼び出されるためコンテキストウィンドウを圧迫しないように。  
目安は 100 行以内！！

## CLAUDE.md は定期的にメンテナンスしよう！！

CLAUDE.md は定期的に見直すようにしましょう。

Anthropic 公式では次のように述べられています。

> Treat CLAUDE.md like code: review it when things go wrong, prune it regularly, and test changes by observing whether Claude's behavior actually shifts.  
> （CLAUDE.mdをコードのように扱いましょう。  
> うまくいかないときに見直し、定期的に整理し、Claudeの挙動が実際に変わるかどうかを観察して変更をテストしましょう）

各行について「これを削除したらClaudeがミスをするか？」と自問し、そうでなければ削除することを検討してみてください。

なお、強調したいルールには「**IMPORTANT**」や「**YOU MUST**」などを付けると、遵守率が上がる傾向があるみたいです。

## CLAUDE.md を採点してみる

ここからは少し応用です。  
CLAUDE.md を作成したはいいものの、それがあっているのか心配ではありませんか？  
そんな時は、「**claude-md-improver プラグイン**」を使って採点してみよう！！

### `claude-md-improver` とは？

Anthropic が公式に提供している Claude Code プラグイン[`claude-md-management`](https://github.com/anthropics/claude-plugins-official/tree/main/plugins/claude-md-management)に含まれるスキルです。  
やってくれることは以下の3つです。

1. プロジェクト内の CLAUDE.md をすべてスキャン
2. 採点基準をもとにスコアとグレードを算出してレポートを出力
3. 不足している部分を diff 形式で改善提案（承認した場合のみ反映）

### インストール手順

ターミナルで以下を実行します。

```
npx claudepluginhub anthropics/claude-plugins-official --plugin claude-md-management
```

または、Claude Code 内から直接インストールする場合、以下を実行します。

```
/plugin install claude-md-management@claude-plugins-official
```
