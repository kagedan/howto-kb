育児系の無料ツールを作った。生後日数カウンターと離乳食カレンダーだ。生後日数カウンターは、スマホUIに特化したものが乏しく、広告だらけで見にくいためつくった。離乳食カレンダーは、記録を保存していくものは多々あるが、日々献立を考える中で、与える食材の予定を管理するツールが無かったため、作成を決意した。

<https://0-tools.com/tools/baby-day-counter/>  
<https://0-tools.com/tools/baby-food-calendar/>

開発にはClaude Codeを使った。バックエンドなし、静的HTML＋GitHub Pages構成で、1ツールあたり数時間でリリースまでこぎつけた。モデルはその時点の最新のSonnetを使っている。

## なぜ静的HTMLなのか

最初はReactで作ることも考えた。でも要件を整理すると静的HTMLで十分だった。

ツールの性質が計算・変換系なのでサーバーサイド処理が不要だ。ならばサーバーを持たなければ費用もかからないし、デプロイも`git push`一発でいい。おまけにGooglebotがJSのレンダリングを待たずにそのままクロールできる。

ユーザーデータを外部に送らないという点も気に入っている。localStorageで完結するので、プライバシー面での安心感を素直に訴求できる。なかなか理にかなった構成ができた。

## Claude Codeへの渡し方

Claude CodeはCLIツールだ。Web版のClaudeと違い、ターミナルから直接操作する。プロジェクトのディレクトリで`claude`と打つと起動し、ファイルの読み書き・作成・実行まで一気通貫でやってくれる。「このHTMLを修正して」と指示すれば、該当ファイルを開いて編集し、保存まで完結する。ブラウザとエディタを行き来する必要がない。

プロンプトの設計が開発速度に直結する。自分が使っているパターンはこれだ。

```
## 要件
- ツール名：生後日数カウンター
- URL：/tools/baby-day-counter/
- 機能：生年月日入力 → 生後日数・週数・月齢を表示
- 付加機能：お宮参り・百日祝いまでのカウントダウン

## 技術制約
- 静的HTML 1ファイル完結（CSS・JSインライン）
- サーバー送信なし・localStorage使用可
- スマホファースト・ダークモード対応

## SEO要件
- canonical・OGP・meta descriptionを含める
- 構造化データ：FAQPage + HowTo + WebApplication
- h1にメインキーワード、静的テキストコンテンツを本文に含める

## デザイン
- テーマカラー：ピンク系（育児向け）
- Noto Sans JP使用
```

このくらい渡しておくと、ほぼ完成形が出てくる。

## 実際に作ったもの

### 生後日数カウンター

生年月日を入れると生後日数・週数・月齢を即表示する。お宮参り・百日祝い・ハーフバースデーまでのカウントダウンも付いている。深夜の授乳中でも見やすいようにダークモードに対応した。

実装で気を使ったのは「初日を含める／含めない」の切り替えだ。医療現場と一般で数え方が違うので、両方に対応している。XシェアボタンでSNS流入を狙う設計にしたのと、PWAにも対応してホーム画面からリピートしてもらえるようにした。

### 離乳食カレンダー

既存の離乳食アプリは「食べた記録を振り返る」機能が中心だ。このツールの差別化は**先の予定をカレンダーに組める**点にある。「来週の火曜日に病院が開いているから、そこでえびを初めて試す」といった使い方ができる。アレルギー反応が出たときにすぐ受診できるよう、新食材は平日の午前中に予定を入れることを勧めている。

月齢自動判定は厚生労働省のガイドに準拠した。バックアップ機能（JSON export/import）はスマホ買い替えに備えてつけた。

## Claude Codeに任せてよかった部分

構造化データの生成が特に楽だった。FAQPage・HowTo・WebApplicationのschema.orgマークアップは手書きするのが面倒だが、要件を渡すだけで正確なJSONが出てくる。

```
{
  "@context": "https://schema.org",
  "@graph": [
    { "@type": "WebApplication", ... },
    { "@type": "FAQPage", ... },
    { "@type": "HowTo", ... },
    { "@type": "BreadcrumbList", ... }
  ]
}
```

Googleリッチリザルトテストでエラーなし。

SEOコンテンツの静的埋め込みもClaude Codeに任せた。ツール本体はJS動的描画だが、使い方・FAQ・関連リンクは静的HTMLとしてbodyに直書きする構成にした。こうしておくとGooglebotがJSレンダリングを待たずにコンテンツを認識できる。

## 詰まったこと

インデックスが一朝一夕には進まなかった。

技術的に問題がなくても、新しいドメインはGoogleに「信頼されるまで」時間がかかる。Search Consoleを見ると「クロール済み・インデックス未登録」が並ぶ時期がある。robots.txt・sitemap・canonicalに問題がなくても、被リンクがゼロだとGoogleは様子見するらしい。

サイトを公開しただけでは検索結果に表示されるまで時間がかかる。まずSearch Consoleを通じてURLを登録し、Googlebotに巡回してもらう必要がある。対策として有効だったのが、URL検査ツールから手動でインデックスをリクエストすることと、内部リンクを双方向に張ることだ。そして、こういう記事を書いてサイトへのリンクを張ること。今まさにやっている。また、ツール系のサイトはツール一つでは内容が乏しいと判断されるため、わざわざ育児ツール以外のツールを作成し、ツール全般を取り扱うサイトとして認識させることにした。この記事と、ツール群が功を奏するかはここからの話だ。

## まとめ

Claude Codeと静的HTMLの組み合わせは、個人開発の無料ツールサイトに向いている。

開発コストはほぼゼロ。GitHub Pages＋Claude Codeだ。運用コストもゼロ。ただ、利用制限がすぐに頭打ちになるので、サブスクのプロプランくらいはあった方がいいかも。デプロイはgit push一発。SEOは静的HTMLなのでGooglebotに優しい。

計算・変換系のツールなら同じ構成で作れる。個人開発の題材を探している人の参考になれば。
