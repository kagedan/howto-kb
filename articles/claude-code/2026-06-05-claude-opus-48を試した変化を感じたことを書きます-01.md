みゆきです。

Opus 4.7 が出たときは「temperature が使えなくなった」「budget\_tokens が廃止された」など  
APIを使っている人には「今までのコードが動かなくなる」変更が結構ありました。

「また何か大きく変わった？」と少し身構えたんですが、

## 1. Opus 4.8 で変わったこと、変わらなかったこと

### 変わらなかったこと（API互換性）

まず安心した話。

**4.7 のコードは 4.8 でそのまま動きます。**

モデルIDを `claude-opus-4-7` から `claude-opus-4-8` に変えるだけでOK。  
追加の変更は不要でした。

料金も変わっていないです。

■Opus 4.7  
 Input：$5.00 / 1M tokens  
Output：$25.00 / 1M tokens  
コンテキスト：1M tokens   
  
■Opus 4.8  
Input：$5.00 / 1M tokens  
Output：$25.00 / 1M tokens  
コンテキスト：1M tokens

4.7 で廃止された `temperature` や `budget\_tokens` は 4.8 でも使えません。  
ここは変わらず。

thinking（推論）の指定方法も同じ。`thinking: {type: "adaptive"}` です。

```
// 4.7 と 4.8 で書き方は一緒
const response = await client.messages.create({
  model: "claude-opus-4-8", // ここだけ変える
  max_tokens: 4096,
  thinking: { type: "adaptive" },
  messages: [{ role: "user", content: "..." }]
});
```

コードを触らなくて済むのは地味にありがたいです。

### 変わったこと（行動・文章）

こちらが本題です。

モデルIDを変えただけで体感できる違いがいくつかありました。

## 2. 実際に使ってみた話

### 文章が「あたたかく」なった

これ、最初はうまく言語化できなかったんですが。

4.7 のときは、Claude の返答が「正確だけど少し固い」印象があったんです。  
情報は欲しいものが全部入っているんだけど、  
どこか教科書を読んでいるような感じ。

4.8 にしてから、文章が少しやわらかくなりました。

たとえば「この設計どう思う？」と聞いたとき。

4.7:

> 提案された設計には以下の問題点があります。1. ○○のリスクが高い。2. ××の非効率が生じる…

4.8:

> うーん、気になる点が2つあります。まず○○のところ、このまま進むと後で困りそうで…

（意訳ですが）

後者の方が、「一緒に考えてる」感がある。  
個人的にはこっちの方が作業しやすいです。

### 「確認してくれる」回数が増えた

これは好みが分かれるかもしれません。

4.8 は「進める前に確認を取る」ことが増えました。

長い処理を頼んだとき、4.7 は勢いよく全部やってくれたんですが、  
4.8 は途中で「この方向で進めていいですか？」と止まることがある。

最初は「なんで止まるの？」と思いましたが、  
よく考えると間違った方向に全力で進まれるよりはいい。

コードの大きなリファクタリングを頼んだとき、  
「既存の関数を削除することになりますが、よいですか？」  
と聞いてくれたのは助かりました。

### エージェントとして使ったとき

仕事で Claude Code を使うことが多いんですが、  
4.8 は長い作業を途中で投げ出しにくくなった印象があります。

4.7 のときは、複数のファイルにまたがる変更を頼むと  
途中でぼんやりしてくることがあったんです。  
4.8 は最後まで筋が通っていることが多かった。

「長期タスクの継続性が改善された」という Anthropic の説明、  
確かにそうかもしれないです。

## 3. 気になった点

### thinking の内容が見えなくなっている

4.8 のデフォルトでは、Claude がどう考えたか（thinking の中身）が  
**表示されません**。

ストリーミングで流れてはいるんですが、テキストが空です。

見たい場合は `display: "summarized"` を追加する必要があります。

```
thinking: {
  type: "adaptive",
  display: "summarized" // これを足さないと reasoning が見えない
}
```

これは 4.7 からの変更点なんですが、  
ドキュメントを見落としていて最初ハマりました。  
同じ状況になりそうな人は覚えておいてください。

### 確認が増えた分、スピードは少し落ちる

前述の「確認してくれる」という挙動。

良い面もあるんですが、  
単純作業を大量に回したいときは少し邪魔に感じることもあります。

Claude Code のような対話形式なら自然なんですが、  
バッチ処理っぽい使い方のときは注意です。
