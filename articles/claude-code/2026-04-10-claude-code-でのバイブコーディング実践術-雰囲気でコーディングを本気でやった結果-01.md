「バイブコーディング (Vibe Coding)」という言葉が広まってから、賛否両論のまま実践例が増えている。細かい設計をせず「こんな感じで」と雰囲気だけ伝えて AI に丸投げする開発スタイルのことだ。

俺はこの3ヶ月、Claude Code で本気でバイブコーディングをやってみた。結論: 成功条件を押さえれば**個人開発の速度は3〜5倍になる**。一方で、押さえないと爆死する。この記事は成功条件と失敗パターンを、実例のプロンプト付きで公開する。

## バイブコーディングの定義

俺の定義はこうだ。

> 詳細設計・型設計・ディレクトリ構成を人間が先に決めず、自然言語で望む挙動と雰囲気を伝え、AI エージェントに実装判断を委ねる開発手法。

重要なのは「設計しない」ではなく「詳細設計を前倒しで作り込まない」という点。ここを取り違えると丸投げ失敗する。
