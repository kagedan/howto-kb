### エージェンティックAIで、これまでにないソフトウェア構築を実現

* 対象サービス：Kiro
* ポイント
  + Kiroの仕様駆動開発はコードを書く前に、要件定義、システム設計、実装すべきタスクをすべて定義する（Requirements -> Design -> Task List）
    - Kiroに作りたいものを伝えると、要件定義書(requirements.md)を作成してくれる
    - 要件定義が問題なければ、問題ないことを伝える。そうすると、設計書(design.md)を作成してくれる
    - 設計が問題なければ、問題ないことを伝える。そうすると、実装計画(tasks.md)を作成してくれる
  + プロジェクトでの規約(コーディング規約やプロジェクトの情報、テスト手法、デプロイ手順)はSteeringに書いてKiroにコンテキストとして渡す。
    - 永続的に記憶させることができるのでテスト手法やコーディングの一貫性が維持される
    - 個人的な感想：なんとなく、Claude の CLAUDE.md に近いのかな？と思いました。
  + Kiro Powers
    - MCPとhooksをパッケージ化したもの
    - 動的にコンテキスト読み込みのため、MCPの事前全読み込みと比較してコンテキストを圧迫しない
  + KiroにはWeb版もある
    - 個人的な感想：CLIとIDEだけだと思っていたので、使ってみようと思います。

最近、ソフトウェア開発にも興味があったので聴講しましたが、本当にここまでやってくれるんですね。。  
ただ、基本は疎かにしたくないので、AIを使いつつコーディングから学んでいきたいと思います。

<https://aws.amazon.com/jp/blogs/news/stop-repeating-yourself/>

<https://aws.amazon.com/jp/blogs/news/introducing-powers/>

### 生成 AI × MCP で切り拓く次世代 SRE！自律型運用への挑戦と開発者体験の進化

対象サービス：Bedrock、CloudWatch

* ポイント
  + Bedrockがアラート内容を要約して初期報告としてSlackに自動通知
    - 先日行ったDatadog Live Tokyoでも似たようなことを話していましたね。障害対応の時の一番のニーズかもしれません。
  + SlackからNew Relic AnalyzerをメンションするとMCP経由でNew Relicにアクセスし情報収集
    - チャットから自然言語で監視データを引き出せる流れで、SREの実運用にそのまま直結しそうです。

もうSRE Agentとかはこの形が最終的ですかね。。あとよく聞くのは「最近リリースされた機能に問題があった場合などに、修正コードを書いてPRしてデプロイまでする」でしょうか。
