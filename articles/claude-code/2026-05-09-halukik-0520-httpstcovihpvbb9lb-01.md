# Codex一択な理由

少し前まで僕はClaude Codeをかなり推してました。

Claude Codeもめちゃくちゃ強いです。

ただ、今デザイン制作で使うなら、僕はCodexの方がいいかもと思ってます。

理由はシンプルで、

## **Codex内でImage2.0をそのまま使えるから。**

これがマジで大きいです。

普通の画像生成だと、

・プロンプトを書く
・画像を1枚出す
・微妙なところを直す
・またプロンプトを書く
・別ツールでページ化する

みたいになりがちです。

**でもCodexだと、ちょっと体験が違う。**

「このLPのファーストビューを作りたい」
「この特典の受け取りページを作りたい」
「このセミナー告知画像を作りたい」

みたいにCodexと会話しながら、

・誰に見せるのか
・何を伝えるのか
・どんな雰囲気にするのか
・参考にするデザインは何か
・最後にどんな行動を取ってほしいのか

を先に整理できる。

そのうえで、「じゃあImage2.0で作って」

と言えば、その文脈のままビジュアルを作ってくれます。

ここがかなり良い。

Image2.0に渡すためのプロンプトを毎回ゼロから作るというより、
Codex内で方向性を擦り合わせて、
最後に一言で生成に入れる感覚です。

しかも、同時に数案出せる。

LPなら3案。
バナーなら方向性違いで3案。
スライド表紙ならトンマナ違いで数案。

最初から見比べられるので、デザインの方向性を決めるのがめちゃくちゃ早いです。

ChatGPTで1枚ずつ作っていた時より、
「比べて選ぶ」までがかなりスムーズになりました。

![](https://pbs.twimg.com/media/HH2UHp-aoAA67_j.jpg)
