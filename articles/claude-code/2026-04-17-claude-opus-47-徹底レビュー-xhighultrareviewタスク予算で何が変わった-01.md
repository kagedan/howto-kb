2026年4月16日、AnthropicがフラッグシップモデルのアップデートとなるClaude Opus 4.7をリリースしました。料金据え置きでSWE-bench Verifiedが80.8%→87.6%と大幅に伸び、GPT-5.4・Gemini 3.1 Proを上回る結果となっています。

個人的にClaude Codeを毎日使っている立場から、今回のアップデートは単なる数値改善以上のインパクトがあります。新しく追加された `xhigh` エフォートレベル、`/ultrareview` コマンド、タスク予算（Task Budgets）という3つの機能は、エージェント的な開発フローの体験を明確に変えます。この記事では、各機能の中身と実際の使い所を整理します。

## ベンチマーク概況 ── 汎用モデルとして再びトップへ

まずは数字から整理します。Opus 4.7のコーディング性能は、前世代のOpus 4.6と比較して全方位で向上しています。

| ベンチマーク | Opus 4.6 | Opus 4.7 | GPT-5.4 | Gemini 3.1 Pro |
| --- | --- | --- | --- | --- |
| SWE-bench Verified | 80.8% | **87.6%** | - | 80.6% |
| SWE-bench Pro | 53.4% | **64.3%** | 57.7% | 54.2% |
| CursorBench | 58% | **70%** | - | - |

特にSWE-bench Pro（実世界のOSSリポジトリの課題を解かせるベンチマーク）で+10.9ptの伸びは大きく、GPT-5.4を6.6pt、Gemini 3.1 Proを10.1pt引き離しています。OSWorld（PC操作）でも+5.1pt、Finance Agent（金融分析）でも+4.3ptと、エージェント用途全般が底上げされています。

なお、4月初旬に発表されたClaude Mythos Previewは、SWE-bench Verifiedで93.9%を叩き出した実験モデルですが、サイバーセキュリティ能力が高すぎて一般公開されないことが明言されています。Opus 4.7はこのMythosの技術を商用版として抑制しつつ、一般提供できるよう調整したモデルという位置づけです。

## xhighエフォートレベル ── Claude Codeのデフォルトが変わった

Opus 4.7で新設された `xhigh`（extra high）は、従来のhighとmaxの間に挟まれた新しい推論レベルです。推論の深さとレイテンシのトレードオフをより細かく制御できます。

そして重要な変化として、**Claude Codeでは全プラン共通でデフォルトが `xhigh` になりました**。これまで「難しいタスクだけmaxに切り替える」という使い方をしていた人は、デフォルトのまま使うだけで明確に賢くなったと感じるはずです。一方で、レイテンシが気になる軽作業では明示的にhighを指定する選択肢も残されています。

Anthropicの公式推奨は「コーディング・エージェント用途はxhighから始め、高い知性が必要な場合はhigh以上を最低ライン」とのことです。

## /ultrareview ── レビュー専用のサブセッション

`/ultrareview` は、Claude Codeに新設されたレビュー特化のスラッシュコマンドです。変更差分を読み込み、慎重なレビュアーが見つけるようなバグや設計問題を洗い出す独立したレビューセッションを開きます。

通常の会話でコードを見てもらうのとは何が違うのか。体感としては、以下の点で明確に出力が変わります。

* レビュー観点が網羅的（セキュリティ、パフォーマンス、設計、テスト容易性など）
* 「動くコードを書く」文脈から切り離され、指摘の粒度が細かい
* エッジケース・ヌル安全性・並行性の指摘が増える

Pro・Maxプランのユーザーは**月に3回まで無料**で `/ultrareview` が使えます。それ以降はOpus 4.7の通常トークン料金で課金される仕組みです。PR送る前の最終チェックとして、かなり使い所がありそうです。

## タスク予算（Task Budgets）── エージェント暴走への処方箋

個人的にいちばん実用性を感じたのがこのタスク予算機能です。エージェント型の開発で「気がついたらClaude Codeが数百万トークン使っていた」という経験、ある人は多いのではないでしょうか。

タスク予算は、エージェント的ループ全体（思考・ツール呼び出し・ツール結果・最終出力）で目標とするトークン量を事前に指定する仕組みです。モデル側はカウントダウンを見ながら作業の優先度をつけ、予算が尽きる前にタスクをきれいに終わらせようと振る舞います。

```
$ claude-code --task-budget 50000 "この機能のリファクタリングをお願い"
```

従来のmax\_tokens制限と違うのは、**モデルが予算を認識して段取りを変える**点です。残り少なくなれば「ここは後回しにしてコア部分だけ仕上げる」といった判断を自分でします。自動売買Botのような本番エージェント運用では、コスト上限を事前に握れるのは大きな安心材料です。

## ビジョン性能の3倍強化

画像入力の解像度上限が大幅に緩和されました。

* Opus 4.6: 最大 約1.15メガピクセル（長辺の制限あり）
* Opus 4.7: 最大 **3.75メガピクセル**（長辺 2,576px）

スクリーンショットやグラフ、UI設計のカンプなど、細かい文字が含まれる画像の読み取り精度がかなり上がりました。ビジョン系のベンチマークでは54.5%→98.5%と劇的な改善が報告されています。

## 移行時の注意点 ── 新しいトークナイザー

料金は据え置き（Input $5 / Output $25 per 1M tokens）ですが、注意点が1つあります。**Opus 4.7は新しいトークナイザーを採用しており、同じ入力でもトークン数が1.0〜1.35倍に増えます**。

つまり、見かけの単価は変わらないものの、実質的なコストは1〜3割増加するケースがあります。プロダクション環境で使う場合は、アップグレード後に実測でコスト比較をするのがおすすめです。

その他、Claude API経由で使う場合の変更点は以下の通りです。

* モデルID: `claude-opus-4-7`
* 新パラメータ: `task_budget`（タスク予算、オプション）
* 新エフォート: `effort: "xhigh"`（high/max/xhigh/minの4段階）
* Amazon BedrockでもGA済み

## 個人開発者視点で何が変わるか

ここまでの内容を、実際に日々の開発で何が変わるのかという視点でまとめます。

* **普通に使うだけで賢くなる**: Claude Codeのデフォルトがxhighになったので、明示的な設定変更なしで体感品質が上がる
* **レビューの質が上がる**: `/ultrareview` をPR前に挟むだけで、バグ検出率が明確に上がる
* **コスト事故が減る**: タスク予算でエージェント暴走を防げる。本番運用やCI組込み時の心理的ハードルが下がる
* **画像系タスクが現実的になる**: スクリーンショットベースのデバッグや、図表の読み取りがほぼ実用レベルに

一方で、新トークナイザーによる実質コスト増はプロダクション用途では確認が必要です。API料金が単価据え置きでも、月末の請求書を見て驚く可能性があります。

## おわりに

Claude Opus 4.7は、数値上の進化だけでなく「AIエージェントを安全に制御するための機能」が充実してきたアップデートだと感じます。特にタスク予算は、個人開発で本番運用しているエージェントを持つ身としては待ち望んでいた機能でした。

Claude Mythos Previewのような「公開されないが実在する」モデルが並行して存在することも含めて、この分野の進化速度は明らかに加速しています。Opus 4.7をまずはデフォルト設定のまま数日使ってみて、`/ultrareview` とタスク予算を試すところから始めるのがおすすめです。
