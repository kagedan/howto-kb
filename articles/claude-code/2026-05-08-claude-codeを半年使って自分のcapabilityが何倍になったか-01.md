> Designium COO秦優が連載『Claude Codeで会社を回す』を書いています。本記事は#5、序章の最終回。半年でCapabilityが何倍になったかを定量で書きました。

体感は5倍から10倍。いきなり結論から書く。  
半年前に1日でやれていたことの何倍が、今1日に動いているか。数字で測ってみたら、自分でも驚いた。

連載『Claude Codeで会社を回す』の序章最終回として、半年の差分を定量で記録する。先に言っておくと、これは「AIで生産性が爆上がりしました」みたいな話ではない。同じ24時間の中で動かせる量がどう変わったかを書くものです。

## 1. 半年前と今、並行で動かしているものの違い

並行で動かしているもののリストを並べる。

半年前（2025年10〜11月）:

今（2026年5月）:

* 自社プロダクト: Stampikoを軸にプロダクト2本＋ゲーム3本＋社内/個人ツール10本以上＋MVP5本（合計20本以上）
* AI社員チーム: 4チーム（content-team / designium-note-team / stampiko-team / web-team）。各チーム3〜5名構成、合計12〜20名相当
* memory本数: **156本**（定期的に棚卸ししている）
* 自動化ワークフロー: 10本以上
* note発信: AI社員チームが平日毎日1〜2本、自分は記事の承認と最終判断のみ

## 2. Capability倍率の総合値

軸ごとに倍率を計算してみる。

* 並行運用プロダクト数: 0→20本以上 → 計算不能
* AI社員チーム数: 0→4チーム → 同上
* memory本数: 10→156 → 約15.6倍
* 自動化ワークフロー: 2〜3→10本以上 → 約4倍
* 1日に下す経営判断の数: 数件→10件以上 → 3〜5倍
* コードを「動かす」量（自分が触れて回しているコード）: 限りなくゼロに近い→大量

総合の体感倍率は、控えめに見て5倍、上振れで10倍。

これは「自分の能力が上がった」という意味ではない。  
「自分が同じ時間で投げられる仕事の数が増えた」という意味だ。1人の能力には上限があるが、AI社員に任せられる仕事の数には、今のところ上限が見えない。

## 3. 倍率を生んでいる構造

倍率を生んでいる中身は3つに絞れる。

**AI社員の人格化**

これは#4で書いた通り。Junが書く、Renが採点する、Mikiが企画する、それぞれが名前と役割を持つだけで、自分の「投げる量」が一気に増える。ツールに対しては「次に何をさせるか」を毎回考える必要があるが、人格を持った相棒に対しては「君の判断で進めて」が成立する。

**memory**

memory 156本は単なるドキュメントではなく、自分の判断軸を外部化した装置になっている。失敗パターンを書き溜めると、翌日のClaude Codeが昨日の自分と似た判断をする。1人のスケールでは到達できない速度だ。

**自動化のhook化**

朝起きると日報サマリーが上がっている。バグレポは集約されている。Xポストは自分が起動しなくても流れている。10本以上のhookが裏で動いているので、起きたあとの自分の時間は「判断」だけに使える。

ちなみに、半年前にこの3点セットの威力を聞いていたとしたら、自分は信じなかったと思う。当時はClaude Codeを「賢いツール」くらいの位置で見ていた。

## 4. 倍率が伸びない領域

逆に、半年で倍率が1倍のまま伸びていない領域もある。

**経営判断の最終責任**  
Claude Codeは案を出すし、トレードオフを比較するし、過去の判断パターンをmemoryから引っぱり出す。それでも、「どっちを選ぶか」は最後に自分の名前で決める。倍率は1.0倍のまま。

**人間関係の機微**  
クライアントの空気を読む、社内の温度を測る、相手の機嫌を察する。これらはClaude Codeに渡していない。1.0倍のまま。

**採用判断、投資判断、長期戦略の方向決め**  
これらも1.0倍。

正直、倍率が伸びない領域があることは、自分にとってむしろ安心材料だ。全部AIに渡せてしまったら、自分が会社にいる意味がなくなる。

## 5. 半年後、どこまで行くか

予測を書くと外れるので、宣言だけ書く。

半年後、AI社員チームを8〜10チームに増やしたい。  
memoryは質が良いものを常に棚卸ししながら300本超えを目指す。  
自社プロダクトの本格運用を5本に増やす。  
中小企業のAI導入支援は、現在の積極関与3社から5社に増やす。

これは「いける気がする」のではなく、「半年同じことをやってきた延長線で計算するとそうなる」という計算結果だ。

逆に言えば、Capability倍率は10倍止まりではない。半年でもう一段、20倍に行く可能性は十分ある。同じ24時間を、AIに投げられる仕事の量がどこまで伸びるか。
