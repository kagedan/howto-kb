## 1. はじめに

Claude Code や Codex などの主要な AI エージェントは，コードベースを検索する際に ripgrep（`rg`）をよく使用します．

`rg` は高速で使いやすい検索ツールですが，デフォルトではドットから始まる隠しディレクトリを無視するという仕様があります．その結果，`.github/` や `.devcontainer/` といったプロジェクトの重要なディレクトリが AI エージェントの検索対象から外れてしまい，プロジェクトの実態と乖離した回答をしてしまうことがあります．

この記事では，対策として `.ignore` ファイルを使ってこれらを検索対象に含めることを提案します．

## 2. 問題点

`rg` のデフォルトでは，ドットから始まる隠しディレクトリをすべて無視します．その一方で，現代のプロジェクトでは重要な設定ファイルが隠しディレクトリに置かれていることがよくあります．AI エージェントがこれらを参照できないと，誤った回答をしてしまったり，そうでなくても無駄にトークンを消費してしまうことが多々あります．

具体的にデフォルトで無視されてしまう隠しディレクトリには，次のようなものがあります．

| ディレクトリ | 主な用途 |
| --- | --- |
| `.github/` | GitHub Actions のワークフローなど |
| `.devcontainer/` | Dev Container の設定 |
| `.vscode/` | VS Code の拡張機能やエディタの設定 |
| `.claude/` | Claude Code の設定 |
| `.codex/` | Codex の設定 |

## 3. 提案手法

### 3.1. ignore ファイル

ripgrep には ignore ファイルによって検索対象を制御する仕組みがあります．ripgrep が参照する ignore ファイルの優先順位は次のとおりです．

1. `.rgignore`：ripgrep 専用
2. `.ignore`：アプリケーション非依存
3. `.gitignore`：Git 専用

`.ignore` は `.gitignore` より優先されるため，`.gitignore` に影響を与えずに検索設定を上書きできます．`.rgignore` と `.ignore` はどちらでも構いませんが，`.ignore` の方が ripgrep ではデフォルトとされているようなので，迷ったときはひとまず `.ignore` を使っておくとよいでしょう．

### 3.2. プロジェクトルートに `.ignore` を作成する

`.ignore` ファイルは `.gitignore` と同じ書式で記述できます．プロジェクトのルートに `.ignore` ファイルを作成し，検索で表示させたい隠しディレクトリを `!` プレフィックス付きで記述します．たとえば，`.github/` と `.devcontainer/` を検索対象に含めたい場合は次のように書きます．

.ignore

```
!.github/
!.devcontainer/
```

この設定により，`rg` を使うすべてのツールでこれらのディレクトリが自動的に検索対象になります．

### 3.3. VS Code でシンタックスハイライトを有効にする

`.ignore` ファイルは VS Code の言語認識対象外のため，シンタックスハイライトが適用されません．`.vscode/settings.json` に以下を追加すると，`.gitignore` と同じハイライトで編集できるようになります．

.vscode/settings.json

```
{
    "files.associations": {
        ".ignore": "ignore"
    }
}
```

## 4. 考察

`.ignore` は隠しディレクトリを検索対象に含める以外にも使い道があります．たとえば，サイズの大きい git submodule があると検索結果が汚染されることがありますが，`.ignore` を使えばこれを除外することができます．

自分の場合は Google Fonts のリポジトリを使って機械学習の研究をしているのですが，しばしば `rg` の検索結果に Google Fonts リポジトリのファイルが混ざってしまい，AI エージェントが再検索をかけることがありました．この問題も，`.ignore` を次のように設定することで解決できました．

.ignore

```
!/.devcontainer/
!/.github/
data/google/fonts/
```

`.ignore` は言わば aiignore です．Git で `.gitignore` を使ってリポジトリの管理対象を制御するように，`.ignore` を使って AI エージェントの検索対象を制御することは自然な流れなのではないかと思います．

## 5. おわりに

AI エージェントがよく利用する `rg` では，デフォルトで `.github/` や `.devcontainer/` などの隠しディレクトリが検索対象から外れてしまうため，AI エージェントが勘違いを起こすことがあるという問題について説明しました．対策として，プロジェクトルートに `.ignore` ファイルを置き，検索対象に含めたいディレクトリを制御する方法を提案しました．同じような問題に直面している方は，ぜひ `.ignore` ファイルの活用を検討してみてください．
