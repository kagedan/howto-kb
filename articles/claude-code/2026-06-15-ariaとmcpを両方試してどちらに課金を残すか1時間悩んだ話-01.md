## 前回予告した「AriaとMCP」、実際に使い比べた

Ariaは「ChatGPTの最大級リデザイン」として報じられ、Canva・[Booking.com](http://Booking.com)・Figma・Spotifyなどが連携パートナーとして参加している。インターフェース上のボタン一つでCanvasに作業が飛び、旅行計画はそのままBooking.comの予約画面に直結する。週9億人のユーザーが使うChatGPTの画面が、そのままタスク実行の起点になるという設計だ。

一方のMCPはそもそも「アプリ内ボタン」ではない。プロトコルとして定義されており、現在公開レジストリには10,000以上のサーバーが存在する。SDK月間ダウンロードは9,700万回に達し、Linux Foundationのガバナンス下に入った。Claude、ChatGPT、Gemini、Copilot、Cursorがすべてネイティブ対応している。

実際に使い比べてわかったのは、**Ariaは「体験」を売っていて、MCPは「接続可能性」を売っている**という違いだ。Ariaのツール連携は流れが滑らかで、ChatGPTの画面を離れずにCanvaでスライドが作れる。素直に感動した。ただ、自分のプライベートなデータソース（社内Notionや手元のスプレッドシート）につなごうとすると、現時点では手が止まる。

MCPは逆で、最初の接続設定がまだ面倒だ。OAuth 2.1を通したMCPサーバーの構築はエンジニアでないと敷居が高い。でも一度つながれば、自分のファイルサーバーにも社内DBにもClaudeがアクセスできる。要するに「誰でもすぐに使えるAriaの使い勝手」と「何でもつながるMCPの汎用性」のどちらを取るかという話で、両立している部分もあれば競合している部分もある。

少なくとも自分の場合は、「プライベートデータと連携したい」という要件がある時点でMCPの軍配が上がった。ただ、設定コストを払えない人にはAriaの方が圧倒的に早く価値が届く。

**今日から試せる1アクション**：ChatGPTでAriaのCanva連携を確認（英国・EEA外は近日ロールアウト予定）し、並行してClaudeの公式MCPディレクトリで自分のニーズに合うサーバーを1つ探す。2つを並べたとき、自分のユースケースがどちらの設計に近いかが一発でわかる。

## Claude Codeにネストサブエージェントが来た──MCP自己ホストサンドボックスも公開ベータ

Claude Code周りで今週動いた2点が地味に重要だ。

**1. ネストサブエージェント対応**：Claude Codeがサブエージェントを入れ子で動かせるようになった。「タスクAを担うエージェントが、タスクBを担う別エージェントに指示を出す」構造が組める。複数のコードベースにまたがるリファクタリングや、並行テスト実行で効いてくる機能だ。

**2. MCP self-hosted sandboxの公開ベータ**：Claude Managed AgentsがMCPサーバーを自社環境のサンドボックス内で動かせるようになった。「社内データをAnthropicのサーバーに送らずに、Claude Agentを自社インフラの中で動かす」ことを意味する。クラウドへのデータ送出を禁じているセクター（金融・医療・行政）にとっては、ようやく本番利用への道が開けた感がある。エンタープライズのClaudeへの移行を阻んでいた最大の壁の一つが外れた、という見方もできる。

あわせて、MCP connector observabilityもアドミン向けに公開ベータに入り、エラー・レイテンシ・利用状況をClaudeの管理画面から確認できるようになった。

**今日から試せる1アクション**：AnthropicのリリースノートページでMCP self-hosted sandboxのベータ申請フォームを確認する。今日申請すれば数週間以内に試せる枠が残っている可能性がある。

## MetaがLlamaの次を「独自モデル」で作り始めた

これは見落としそうで、個人的にはかなりのインパクトがあった。

MetaがAlexandr Wang（元Scale AI CEO）を最高AIオフィサーに迎え、新設した「Superintelligence Labs」が開発したフラッグシップモデル「Muse Spark」を発表した。注目点は**これがオープンソースではない**という点だ。

LlamaはMetaのAI戦略の代名詞として無償公開を続けてきたが、Muse Sparkはプロプライエタリモデルとして投入された。公式によれば、Llama 4の中型バリアントと比較して計算コストの一部で、マルチモーダル知覚・推論・ヘルスケア・エージェント用途で競合する性能を達成しているという。

なぜこれが重要か。Metaのオープンソース戦略は「OpenAIやAnthropicに規模で直接対決できないなら、エコシステムを開放して開発者を取り込む」という合理的な選択だった。その路線を一部でも曲げてきたということは、Metaが「クローズドで勝てる水準に到達した」か「オープンソースのコスト構造に限界を感じた」かのどちらかだ。MetaのAI設備投資は2026年に1,150〜1,350億ドルと昨年比ほぼ倍増が予定されており、この規模の投資の先に「全部オープンで回す」未来を描いていないのは自然とも言える。

一方でLlamaはまだオープンソースとして配布が続いている。Muse SparkとLlamaを二重戦略で走らせている意図がまだ読み切れていない。

**今日から試せる1アクション**：LM ArenaなどのベンチマークサイトでMuse SparkとLlama 4の比較スコアを確認する。MetaのモデルがOpenAIやAnthropicの最新モデルとどこで拮抗し、どこで差がつくか、数字で見ておく価値がある。

## エンタープライズエージェント、「2026年末に企業アプリの40%」という予測が現実になり始めた

Gartnerが「2026年末までに、エンタープライズアプリケーションの40%がAIエージェントを内蔵する」という予測を出している。今週はAlteryx（Agent StudioとMCPサーバーを発表）、Oracle（Autonomous AI Vector Database、Private Agent Factory）などが実際の製品をリリースしており、予測が現実に追いかけられている感がある。

特にOracleのPrivate Agent Factoryは「ノーコードでデータ駆動型エージェントをデプロイできる」設計で、ITチームを介さずにビジネスアナリストがエージェントを作れる。少し前まで「エージェントは試験運用」だったものが、本番インフラの一部として語られ始めている。AlterxのAgent Studioも既存のデータワークフローとビジネスロジックを直接エージェントに変換できるというもので、「ゼロから作る」ではなく「今あるものをエージェント化する」という方向に現場は進んでいる。

## 編集後記
