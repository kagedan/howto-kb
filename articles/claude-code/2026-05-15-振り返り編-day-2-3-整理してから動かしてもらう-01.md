## 「税務効率化システム」を作っていた

Day 1 で Claude を初めて開いてから、たぶん 2 〜 3 日後のことだったと思う。

その頃の自分は、Claude を相手に「税理士の月次業務を AI が手伝うとしたら、こんな機能だろう」を片っ端から会話で作っていた。仮の名前で「税務効率化システム」と呼んでいた気がする。仕訳のチェックも、証憑との突合も、督促のメモも、思いついた機能を全部ひとつの画面に詰め込んだ。

動いてはいた。でも、開くたびに止まる時間が増えていった。

## スパゲティの気配

最初に作ったものは、同じ画面の中に「調べる」「確認する」「判断する」「記録する」が混ざっていた。仕訳の確認をしようと開いたつもりが、隣に督促メモが出てきて、その隣に証憑のステータスが並ぶ。一つ機能を足すと、隣の機能の挙動も少し変わる。

エンジニアではない自分でも、「これはスパゲティになる前の気配だ」と感じた。

## 作り直すことにした

そこで、まず作るのをやめた。Claude (ブラウザで開く対話用の方) に「今、事務所でやっていること」と「これから AI に任せたいこと」を、思いつく順に投げていった。整理して並べてもらう。

並べ終わったら、34 項目あった。それを 4 つに仕分けした。今後も残すもの、いったん凍結するもの、段階的に廃止するもの、もう捨てるもの。だいたい、残す 23・凍結 8・段階的廃止 2・捨てる 1、くらいになった。

仕分けが終わった時点で、画面の輪郭が見えてきた。会計ソフト (MoneyForward) の仕訳を取り込んだ後、「証憑が要らないもの」「自分が判断すべきもの」「証憑がまだ届いていないもの」「あとは計上を待つだけのもの」、この 4 つに分けて並べる。同じ画面に詰め込まない。

## 動かす線を引いた

このときに、もうひとつ決めたことがある。 会計ソフトに仕訳を登録するのは、こちらが「これを登録して」と渡したときだけにする、 という線を引いた。

AI に任せるのは、判断材料を横に並べてもらうところまで。「これは前にこういう仕訳だった」「この摘要なら去年はこの科目」「この明細には証憑がまだ来ていない」、そういうのを集めて、選びやすい形で隣に出してもらう。決めて押すのは、自分。意識的に「これを登録して」と渡せば、そこから先は AI が動いてくれる。

## Claude Code に渡した

整理がついた塊から、Claude Code (ターミナル上で動く実装用の方) に渡した。「この画面を、こういう構造で作って」と指示する。会計ソフトからの取り込みは Playwright で実装してもらい、画面側は仕分けに従って 4 つに分けてもらう。動くものは Claude Code が組み立てる。

整理する側と、動かしてもらう側を、別の AI に分けた。これが、今のところ続いているやり方の起点だった。
