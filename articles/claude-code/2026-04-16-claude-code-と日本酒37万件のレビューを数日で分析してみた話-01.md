先週の土曜日、日本酒のYouTube動画を観ていたら、「SAKETIMEというレビューサイトのランキング上位は、甘旨ジューシー系の酒ばかり」という話が出てきた。言われてみれば確かにそうかもしれない。でも、「日本酒は辛口に限る」と昔は言われていたはず。本当に辛口の時代は終わったのだろうか。

漠然とした違和感のまま、翌日曜日に Claude Code を開いて「SAKETIMEをスクレイピングして、上位が本当に甘口なのか調べたい」と相談してみた。そして水曜日には、5,386銘柄・約37万件のレビューを集め、国立国会図書館の出版物データとGoogle Trendsまでつなぎ合わせた、140年分のトレンド分析記事をNoteに公開していた。**着手から公開まで、実質4日**である。

私は10年以上前、研究者を目指していた頃には R を使ったデータ分析をやっていたし、簡単なスクレイピングのコードも書いたこともある。ただ、当時の自分が一人でこの作業をやろうとしていたら、そもそも途中で挫折していた可能性が高いと思うし、できたとしたらおそらく数ヶ月はかかっていた。それが数日で終わったのだから、正直かなり面食らった。

## 技術でなく、マナーまで教わった

スクレイピングを走らせていたら、途中でサーバーから403エラー（アクセス拒否）が返るようになった。同じIPから大量にアクセスしたことで、一時的にブロックされたらしい。ここで Claude Code が、印象的なことを言ってくれた。

> SAKETIMEさんにデータ利用の許可を取った方がいいかもしれません。スクレイピング自体は可能でも、運営に一言連絡を入れるのが礼儀です

ハッとして、依頼文面まで書いてもらい、実際に連絡したところ、SAKETIMEさんから丁寧なお返事をいただき、分析利用とブログ公開の許可を得ることができた。技術的なサポートだけでなく、社会的な振る舞いまで助言してくれるのは、想像以上にありがたかった。

## データが広がり、結論が揺れ動いた

SAKETIMEだけでは「今」しか見えない。時代の変化を知りたいと伝えたら、Claude Code は別のデータソースを提案してきた。さけのわAPIの日本酒フレーバーデータ、国立国会図書館のNDL Ngram Viewer（1880〜2000年の出版物の語彙頻度）、Google Trends（2004年以降の検索トレンド）。私は存在すら知らないものもあった。

分析を進めるうちに、結論は何度も書き換わった。「甘口で芳醇」→「甘口だが芳醇ではなく、むしろ軽め」→「上位と下位で別の世界（甘口率67%対28%）」→「三層構造（愛好家の上位／下位／一般消費者）」→「辛口はむしろしぶとく生き残っている」。

特に驚いたのは、愛好家レビューでは甘口・フルーティが上位なのに、Google検索では今でも「日本酒 辛口」が「日本酒 甘口」の約2.5倍で、しかも右肩上がりに増え続けているという事実だった。データを集めなければ絶対に気づけなかった。

Claude Code の良さは、この試行錯誤を全部覚えていてくれることだった。「前は違う結論でしたが、今回のデータではこう見えます」と過去の議論を踏まえて答えてくれる。一人でこねくり回していたら、たぶん最初の結論のまま記事を書いていた。

## 文脈を保ったまま、並行作業が進む驚き

今回、作業速度を一番押し上げたのは、デスクトップ版の Claude Code のGUIで、スクレイピングをバックグラウンドで回しながら、同じセッションのなかで次の作業に進められたことだった。

37万件のレビュー取得には、どうしても数時間かかる。普通なら待つしかない時間だが、その間に「先に集まった部分データで探索的分析を始めてください」「記事のドラフトを書き始めましょう」と並行してお願いできた。データが揃った時点でのWord文書の再生成、手直しの反映、グラフの差し替えまで、すべて同じ会話の流れで繋がっていく。

Claude Code を複数立ち上げて別々の作業を任せられるのは知っていた。でも今回驚いたのは、**それまでの文脈を把握している一つの Claude Code のなかで、複数の作業を同時に進められたこと**だった。「データを取る」「分析する」「書く」が別々のモードとして分断されるのではなく、ずっと一つの思考の流れとして続いていく。この感覚は、これまでの開発ツールとはかなり違っていた。

## 好奇心の射程が広がる

そうだろうとは思っていたが、**Claude Code とデータ分析の相性はとても良かった**。自然言語でスクレイピング方針を相談でき、Notebook・スクリプト・記事を横断して編集でき、対話の連続性で問いを深められ、非エンジニアでも「やってみる→詰まる→対話で解決」のサイクルが回る。

もちろん、どのデータを信じるか、どう解釈するか、どういうストーリーで書くかは、結局自分で決める必要があった。Claude Code は優秀な伴走者だが、走る方向を決めるのはこちらだ。それでも、一人では絶対に辿り着けなかった場所まで連れて行ってくれる相棒であることは間違いない。

始めた時点で私が持っていたのは、YouTube動画を観て感じたちょっとした違和感だけだった。それが4日後には独立した分析記事一本になっていた。10年前の自分なら数ヶ月かかっていたであろう作業が、数日で完了してしまう。技術の民主化、というと大げさだが、素人が気になったことを本気で掘り下げられる時代になったこと自体が面白い。

「ちょっと気になっているけど、調べるには技術が足りない」と思っていることがあるなら、試しに Claude Code に相談してみてほしい。意外と遠くまで連れて行ってくれるはずだ。
