【経営者の生産性が爆上がりした、AI分身の作り方】

経営者のAI活用は、3段階で進化します。

▶ 第一の脳：自分で考える
▶ 第二の脳：AIが記憶する（Obsidian等）
▶ 第二の体：AIが実行する（Claude × Skills）← ここまで来ると別次元

実際の設計は、この5つです👇

✅ CLAUDE.md に自分の判断軸・文体・絶対ルールを書き込む
✅ Obsidian に事業情報・案件・議事録を蓄積
✅ /稟議書、/競合分析、/議事録要約 を Skills で仕組み化
✅ Cowork で夜中にスケジュール起動 → 朝には資料が完成
✅ MCP で Slack・Drive・Gmail と接続

「第二の脳」だけでは半分。
「第二の体」まで作って、初めて経営者の"分身"は完成します。

半年後、出力の"桁"が変わります。

以下、各要素を1本ずつ深堀ります👇

①CLAUDE.md の中身を、公開します。

これが、Claudeを"自分専用化"する核です。
vaultのルートに置けば、Claudeは毎セッションの最初に必ず読みます。

【自分について】
名前 / 役職 / 事業領域 / 今期の最重要KPI

【文体ルール】
体言止めを活用 / 過剰な敬語は避ける / 具体例で語る

【判断軸】
売上 < 利益率
スピード > 完璧
自社実例 > 一般論

【絶対ルール】
◯◯は絶対にやらない
◯◯の話題は外部に出さない
◯◯のフォーマットを必ず使う

このファイル1枚で、AIが"自分そのもの"になります。
更新は週1で十分です。

②よく呼んでいる Skills、5つ公開します。

Skillsは、業務SOPをAIに移植する仕組み。
一度作れば、コマンド1発で同じ品質を毎回再現できます。

✅ /稟議書
→ 自社テンプレ＋過去の稟議の文体で生成

✅ /競合分析
→ 自社フレームワークで強み弱みを可視化

✅ /議事録要約
→ 決定事項・TODO・宿題・次回MTGに整形

✅ /採用評価
→ 求人票と照合、強み弱み・面接で聞くべき質問を出力

✅ /PRDドラフト
→ 自社のPRDテンプレでフィーチャー仕様を起こす

業務の名前を呼ぶだけで、AIが指示通り動きます。

③寝ている間に走らせている自動化、設計図を共有します。

Coworkのスケジュール機能と組み合わせれば、朝7時にはデスクに完成品が並んでいます。

🌙 03:00 ナンバーズ
昨日の売上・登録・解約・異常値を1枚に要約

🌙 04:00 朝のブリーフィング
業界ニュース・今日の会議・午前の決断を1ページに

🌙 05:00 競合監視
主要競合5社のリリース・採用・価格変更を要約

🌙 05:30 会議準備
外部MTG相手の最新発信・効く質問・想定質問

🌙 06:00 受信トレイ整理
メール4階層分類 → VIPには返信下書きまで作成

朝の脳を、雑務に渡さない。
それを支えるのが、この夜の自動化です。
