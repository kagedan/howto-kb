最近はClaude Code に作業を任せて、自分は別のタスクを進めて並行作業をすることが多いです。

しかし実際にやっていると、**自分のタスクが終わってからClaude Code に戻ったら確認待ちで止まっていた**という悲しい事件が何度か発生しました。

気づけばずっと待ち続けていたわけです。作業を任せた意味よ……。

そこで見つけたのがClassmethodさんのこの記事 。

<https://dev.classmethod.jp/articles/claudecode-hooks-notify/>

Claude Code の Hooks 機能を使って、入力待ちや完了になったら音声で教えてくれる仕組みを作る内容です。

早速マネして作ったんですが、今度は**通知が鳴りすぎる問題**が発生。カスタマイズして解決したので紹介します 🎉

## Claude Code Hooks とは？

Claude Code には **Hooks** という機能があって、特定のイベントが起きたときにシェルコマンドを実行できます。

今回使うのはこの3つ。

| イベント | 発火タイミング |
| --- | --- |
| `UserPromptSubmit` | プロンプトを送信したとき |
| `Notification` | Claude が入力待ちになったとき（アイドル60秒以上など） |
| `Stop` | Claude の応答が完了したとき |

設定は `~/.claude/settings.json` に書くだけ。再起動も不要で、保存した瞬間から反映されます。

## まず元記事の基本実装

元記事の実装はシンプルで、macOS の `say` コマンドで喋らせるだけです。

```
~/.claude/
├── hooks/
│   ├── notify-waiting.sh   # 入力待ち通知
│   └── notify-complete.sh  # 処理完了通知
└── settings.json
```

~/.claude/hooks/notify-waiting.sh

```
#!/bin/bash
PROJECT_NAME=$(basename "$PWD")
say -v Kyoko "${PROJECT_NAME}、入力待ちです"
```

~/.claude/hooks/notify-complete.sh

```
#!/bin/bash
PROJECT_NAME=$(basename "$PWD")
say -v Kyoko "${PROJECT_NAME}、処理完了しました"
```

```
chmod +x ~/.claude/hooks/notify-*.sh
```

~/.claude/settings.json

```
{
  "hooks": {
    "Notification": [
      {
        "hooks": [{ "type": "command", "command": "bash \"$HOME\"/.claude/hooks/notify-waiting.sh" }]
      }
    ],
    "Stop": [
      {
        "hooks": [{ "type": "command", "command": "bash \"$HOME\"/.claude/hooks/notify-complete.sh" }]
      }
    ]
  }
}
```

これだけで動きます。簡単！

## 使ってみたら困った！うるさすぎる

実際に試してみたら、通知過多で困りました。

具体的には以下の内容です。

### 困り①：ちょっとした作業のたびに「処理完了しました」って言ってくる

「この文言直して」みたいな一瞬で終わる軽い作業でも毎回完了通知が来ます。

### 困り②：やり取りのたびに「入力待ちです」って言ってくる

Claude が途中で確認を求めてきて、自分が答えて、また確認が来て……というやり取りをしていると、毎回「入力待ちです」が発火。ウザみを感じた。

### 困り③：タスク終わってしばらくすると「入力待ちです」って言ってくる

`Notification` イベントはアイドル状態が60秒続くと発火します。

なので Claude はタスクを完了しているのに、しばらく経ってから「入力待ちです」と言ってきます。

## カスタマイズして解決

3つのスクリプトで対処しました。

### 対策①：完了通知は長めのタスクだけに絞る（5分未満はスキップ）

`Stop` フックで、タスクの経過時間が5分未満なら完了通知をスキップするようにしました。ちょっとした作業でいちいち喋られる問題が解消。

### 対策②：テンポよくやり取りしているときは通知しない（10秒クールダウン）

`UserPromptSubmit` フックで、プロンプトを送った時刻を記録します。`Notification` が来たとき、送信から10秒以内なら無視。

Claude が質問してきてユーザーが回答した場合も `UserPromptSubmit` が発火するので、**やりとりのたびにタイマーがリセット**されます。テンポよく会話しているときは「入力待ちです」が連発されません。

### 対策③：タスク完了後は次のプロンプトを送るまで通知しない

`Stop` フックでフラグファイルを作成。`Notification` が来たときにフラグがあればスキップします。フラグは次のプロンプト送信（`UserPromptSubmit`）で削除されます。

完了の数分後にまた「入力待ちです」と言ってくる問題はこれで解消！

## 最終的なコード全部

### `~/.claude/hooks/record-start-time.sh`（新規追加）

```
#!/bin/bash
date +%s > "/tmp/claude_task_start_$PPID"
rm -f "/tmp/claude_task_stopped_$PPID"
```

### `~/.claude/hooks/notify-waiting.sh`

```
#!/bin/bash
THRESHOLD=10
TIMESTAMP_FILE="/tmp/claude_task_start_$PPID"
STOPPED_FILE="/tmp/claude_task_stopped_$PPID"

# タスク完了後はスキップ
if [ -f "$STOPPED_FILE" ]; then
    exit 0
fi

# 送信から10秒以内はスキップ
if [ -f "$TIMESTAMP_FILE" ]; then
    START=$(cat "$TIMESTAMP_FILE")
    NOW=$(date +%s)
    ELAPSED=$((NOW - START))
    if [ "$ELAPSED" -le "$THRESHOLD" ]; then
        exit 0
    fi
fi

PROJECT_NAME=$(basename "$PWD")
say -v Kyoko "${PROJECT_NAME}、入力待ちです"
```

### `~/.claude/hooks/notify-complete.sh`

```
#!/bin/bash
date +%s > "/tmp/claude_task_stopped_$PPID"

THRESHOLD=300
TIMESTAMP_FILE="/tmp/claude_task_start_$PPID"

# 5分未満のタスクは完了通知しない
if [ -f "$TIMESTAMP_FILE" ]; then
    START=$(cat "$TIMESTAMP_FILE")
    NOW=$(date +%s)
    ELAPSED=$((NOW - START))
    if [ "$ELAPSED" -le "$THRESHOLD" ]; then
        exit 0
    fi
fi

PROJECT_NAME=$(basename "$PWD")
say -v Kyoko "${PROJECT_NAME}、処理完了しました"
```

### `~/.claude/settings.json`（フック設定部分）

```
{
  "hooks": {
    "UserPromptSubmit": [
      {
        "hooks": [{ "type": "command", "command": "bash \"$HOME\"/.claude/hooks/record-start-time.sh" }]
      }
    ],
    "Notification": [
      {
        "hooks": [{ "type": "command", "command": "bash \"$HOME\"/.claude/hooks/notify-waiting.sh" }]
      }
    ],
    "Stop": [
      {
        "hooks": [{ "type": "command", "command": "bash \"$HOME\"/.claude/hooks/notify-complete.sh" }]
      }
    ]
  }
}
```

## 動作イメージ

```
プロンプト送信
  → タイムスタンプ記録・完了フラグ削除

Claude が考え中（10秒以内）
  → Notification きても無視

Claude が途中で質問（10秒超）
  → 「プロジェクト名、入力待ちです」🔔

タスク完了（5分未満）
  → 完了フラグ記録・「処理完了しました」はスキップ
  → その後の Notification も完了フラグで抑制

タスク完了（5分以上）
  → 「プロジェクト名、処理完了しました」🔔
```

## おわりに

Hooks は bash スクリプトさえ書ければ何でもできるので、音声以外にも Slack 通知とか Mac の通知センターとか、好きなものと組み合わせられます。

ぜひカスタマイズしてみてください！
