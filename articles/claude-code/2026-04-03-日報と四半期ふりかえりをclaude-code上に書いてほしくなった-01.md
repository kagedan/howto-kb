# はじめに

自分は毎日仕事の日報を書いています。内容は下記です。

1. 今日やったこと
2. 明日やること
3. ひとこと

今まで毎日自分で1から考えて書いていたのですが、少しでも日報を楽できないか？どうせなら生成AIに手伝ってもらうことはできないのか？と考えました。  
また、自分はこれまで四半期ごとのふりかえりを日報を見て思い出すことが多かったです。日報を一定期間貯めてそれを四半期ふりかえりの土台にできないかと考えました。

そして。せっかくアイデアが生まれたので素早くまずは作ってみようと考え、今回の成果物を作りました。3時間くらいでできました。

(2026/04/03現在) 直近、Claude Codeのコードリークとか色々ありましたね。情報収集は積極的に行なっていきます。

# 成果物

<https://github.com/You-saku/log-distill>

名前は log-distill です。README.mdにも記載ある通り、情報を集めて日報に昇華する。日報を集めて四半期の振り返りへ昇華する。この流れがお酒作りの蒸留に似てると感じたためこのような名前にしました。

自分がお酒好きというのもあります笑。

# 実装するにあたり考えたこと

アイデアを具体化するために最初に作る機能のスコープ決定と自分の普段の行動を深掘りました。

## いきなり大きなものを作らない

CLIツール、AI Agentを作る… etc 最初は夢が広がりました。しかし、最初から大層なものを作ろうとすると完成するまでに時間がかかる。完成までに時間がかかって途中で飽きるという危険性があります。

Claude CodeであればMarkdownで指示を与えられるので今回は **Claude Codeが特定のタスクを実施するためのワークスペースをまず作ろう** と考えました。

## 日々の仕事を自分はどこでしているのか考える

次はClaude Codeにどんな指示を出すのが良いだろう？と考えました。すると自分は特定のアプリの中に多くいることがわかりました。それは下記3箇所です。

1. GitHub
2. Slack
3. Notion

この3つならClaude CodeのPluginやMCP、専用のCLIツールが使えるなと考えていたら本当にありました。最終的に

1. GitHub → ghコマンド
2. Slack → Plugin
3. Notion → Plugin

このような対応で日報に必要な情報を集めることにしました。Claude Codeは外部リソースを使うための設定が容易なので簡単に設定が完了しました。

## コードは書かない

> 今回は **Claude Codeが特定のタスクを実施するためのワークスペースをまず作ろう** と考えました。

先述今回した通り、今回はプログラミング言語でアプリを作ることは考えていません。ひたすらにMarkdownを整え、gitignoreを整備して他の人の日報が混ざらないようにgit管理を設定するだけです。

2026年4月現在のディレクトリ構造はこちらです。

```
log-distill/
├── sources/
│   └── yyyy-mm-dd/       # 日付ごとの生データ
│       ├── github.md     # gh CLIで収集したPR・コミット情報
│       ├── notion.md     # Notion MCPで収集したページ更新情報
│       └── slack.md      # Slack MCPで収集した発言ログ
└── logs/
    ├── daily/
    │   └── yyyy-mm-dd.md # 日報
    └── quarterly/
        └── yyyy-qN.md    # 四半期報
```

# 使い心地

毎日やることは下記のフローです

1. log-distillディレクトリに移動
2. Claude Codeを起動
3. 「yyyy-mm-ddの日報を書いて」と入力
4. Claude CodeがディレクトりやMarkdownファイル作成をはじめる
5. 自分は聞かれたことに答えるだけ
6. 勝手にMarkdown形式の日報たたきが完成する

# 使ってみて感じていること

## たまにうまくいかない

なぜかMCPで情報を取得できなくてClaude CodeがPythonとか色々使って情報を取得して加工しようとする。これがわからない

## オプションでふりかえりの着眼点や文章のレイアウトや変えたい

今は出力形式をAGENTS.mdに書いてある。日報は人によってふりかえる観点が異なるはずです。最終的に出力したいフォーマットも違うはずです。

ここを対応するとなるとCLIツールのオプションみたいに引数で渡すことになりそう。CLIにしなくても対応できる方法が知りたい。

## Google Workspaceと連携したい

Google Calendarからも今日の取り組みを取得したい。Slackの発言やNotionの記述がなくても仕事をしてる日の対応が今はできない。わざわざSlackで今日やってることを報告しないと日報作成のための情報収集ができない。

何を使えば良いのか。セキュリティ観点も意識した上で選定する。googleworkspace/cliは使えないのか  
<https://github.com/googleworkspace/cli>

# 今後

まずは今の機能だけで使って四半期のふりかえりを迎えたいと思います。今は日報を書くことしかまだできていません。時々Markdownを改修したり今感じている不便さの解消を地道に続けます。
