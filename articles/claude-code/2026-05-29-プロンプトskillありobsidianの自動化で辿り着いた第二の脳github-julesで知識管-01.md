## ① 全部試したけど、結局「身についた感」が来なかった

ノートアプリを数ヶ月使い込むと、避けられない問題に直面しませんか？

> 過去のノート群が、いまの自分の考えと矛盾し始める。  
> タグの命名規則が気づけば3パターン混在している。  
> リンクが切れている。孤立ノートが大量にある。  
> しかも、ひらめいた瞬間にスマホからメモしても、結局PCで開いて分類・タグ付け・リンク張りをする「儀式」が残り続ける。

OneNote、Evernote（最近めっきり名前を聞かなくなりました）、Notion、Obsidian——順番にハマって、順番に挫折してきました。最終的にObsidianに落ち着いた方は多いと思います。Markdown、Wikiリンク、グラフビュー、コミュニティプラグインの自由度。**「Vaultが自分の脳の拡張になる」感覚**は他では味わえません。

**Vault全体を、ある時点の整合した状態に維持し続ける作業**——これがObsidianのヘビーユーザーが避けられない重さです。

そこで、ここ1年で爆発的に増えた Obsidian × AIスキル（ obsidian-skills、 claude-obsidian、 Minervia Starter Kit、 obsidian-claude-code-mcp、 obsidian-ai、 obsidian-second-brain…）を片っ端から試しました。

でも、ある段階で気づきました。

**どれもAIが整理してくれるけど、「自分のものになった感」が薄い。**

AIがリンクを張ってくれる。AIが矛盾を直してくれる。AIが新しい仮説を出してくれる。  
便利です。便利なんですが、いつのまにかVaultが「AIに整理されたデータの倉庫」になっていて、そこに自分の判断や決断が介在していない。  
**スキルを使って自動化して、気持ちよくなってるだけ**。  
これがagent skillを渡り歩く中でずっと残った違和感でした。

かといって、過去のノート群を自分で全部レビューして整合性を取り戻すのは、もう現実的じゃない。  
これだけ情報があふれた時代、自分の脳のキャパだけで全Vaultをメンテし続けるのは無理だし、記憶も劣化していく。  
**AIに全自動で任せると意味がない。自分でやり切るのはキャパが追いつかない。** 皆うすうす気づき始めている矛盾だと思います。

そして、「次のスキルが出たら解決するんじゃないか」と探し続けるループにも疲れました。**スキル探しを終わらせたい。**

## ② 解は「Vaultをリポジトリにして、julesに育てさせる」

結論を先に言うと、私がやったのはこれです。

**Vaultを「ファイルの集合」ではなく「Gitリポジトリ」として扱い、夜中にAIエージェント（jules）を走らせる。**

Obsidian Gitプラグインを使っている人なら半分はやっているはずですが、私がやったのはその先で、**Vaultをソフトウェア開発のメタファーで運用する** ことでした。

このメタファーから生まれたのが、本記事のコアコンセプトです。

> **Hint（気づき候補）とは、Assetへの昇格を待つPR(Pull Request)である。**

記事①で書いた知識の昇格フロー（Raw → Note → Hint → Asset）を、PRワークフローで表現したわけです。Hint＝提案された変更、マージ＝Asset化、クローズ＝一旦保留。

### IssueがVaultへの最短入口になる

GitHubアプリからissueにキーワードやひらめきを投げておくだけで、後でローカルのエージェントスキルが内容をヒアリングしながらメモファイルやブランチに変換してくれます。これで「スマホからの放り込み」がVaultに直結します。

さらに、issueに 「jules」 タグを付けておけば、**julesがそのIssueを拾ってメモのドラフトを書き、ブランチを作るところまで自律でやってくれます**。  
思いつきを夜に投げておけば、翌朝にはすでに整形されたブランチが上がっている。これだけでもObsidian単体ではできない体験です。

### git revertという隠れた恩恵

控えめに見えて大きい話があります。「半年前の自分のスタンスを `git revert` できる」——ノートアプリで一番できなかったことが、いちばん当たり前のジェスチャーになる。  
考えが変わったときに過去のノートを書き換えるのではなく、「あの時点ではこう考えていた」を残したまま現在を更新できる。これがリポジトリ化の隠れた恩恵でした。

### julesを「ナレッジの自律監査エージェント」に転用する

ここからが本記事の核心です。

PRワークフローを採用しただけでは、PRを書くのは結局自分です。これでは「整理コストを下げる」目的が果たせません。  
そこで、**夜中に勝手にPRを積んでくれるエージェント** を仕込みました。**jules** です。

julesはGoogleが提供するリモート実行型AIエージェントで、GitHubリポジトリに対してスケジューラ実行でタスクを走らせられます。  
本来の用途はコードリポジトリのバグ検出・パフォーマンス改善・セキュリティホール検知——改善PRを自動で積み上げるツールです。

これを今回、**「ナレッジの自律監査エージェント」として転用しました**。コードのバグ検知ロジックが、ノート間の矛盾検知に転用できる。  
リンクの整合性チェックが、ノート間のリンク監査に転用できる——構造的に同じだと気づいた瞬間、目の前が開けました。

仕込んでいるのは大きく2つです。

**どちらも既存ノートへの直接編集は絶対にしない** 設計です。julesは提案者に徹し、書き換えるのは人間（私）の責任。

### 朝のレビューはローカルAIに統合させる

朝、コーヒーを淹れる頃、GitHubには夜中にjulesが作ったブランチとPRが並んでいます。

ローカルのClaude CodeかGemini-CLIに「今朝のjulesのPRを確認して、取り込めそうなものを一つにまとめて」と一言投げると、複数のPRを横断分析し、価値のあるものを一つのローカルブランチに統合してくれます。  
あとは私がそのブランチをAIと一緒に確認し、「このリンク追加は妥当か」「このスタンス変更の根拠は何か」を聞きながら、取り込む・捨てるを一つひとつ判断していく。

体感としては「AIが仕分けてくれた提案の束を、朝の10分で承認していく」感覚です。思考が三層に分業されている運用が、やってみると相当ラクです。

## ③ 「マージのためのレビュー」が、身についた感を取り戻す

ここまで読んで気になっているかもしれません。

「結局AIに全部やらせてるだけじゃない？それで身につくの？」

ここが本記事で一番伝えたいことです。

記事①で「最後に自分で意思決定を下すことで、はじめて思考は定着する」と書きました。  
本記事は、その意思決定プロセスを **「マージ」というGit的ジェスチャー** に置き換えた話、とも言えます。

ただし誤解のないように一言。**「マージしたから身につく」のではありません。** マージするために「これは取り込む？捨てる？なぜ？」とレビューする——その思考過程が脳に残るのです。  
マージ自体はクリック一つ、その手前の数十秒の判断が定着の源泉になります。

julesが夜中に「過去のスタンスと最近のメモが矛盾していますよ」とPRを出してくる。それを朝、私がレビューする。

* 「たしかにそうだ、考えが変わった」→ マージ
* 「いや、これは古い自分の方が正しい」→ クローズ
* 「面白い視点だが、もう少し寝かせる」→ 一旦保留

このレビュー判断こそが、ノートを「自分のもの」にする瞬間です。  
**AIは構造化と提案を担当する。人間は探索と決断だけを担当する。** 役割分担が綺麗に分かれているから、自分の時間は「身につける作業」だけに集中できる。  
それ以外の整理・要約・矛盾検知は、ベネフィットが薄いのでAIに渡す。これが運用の肝でした。

そして、「次のスキルが出たら…」と探し続けるループも、ここで終わります。  
私自身、OneNote → Evernote → Notion → Obsidian と乗り換えてきて、agent skillに片っ端から飛びついて、AIモデルの選択にも疲れた人間です。そんな自分が「もうここでいい」と思える形が、このPRワークフローでした。

## でも、このセットアップは非エンジニアには無理だと気づいた

試しに、多少PCはわかるけどエンジニアではない友人に話してみたことがあります。

> 「Git？ ふーん。jules？ なにそれ。スケジューラ？ APIキー？ ……普通の人にそれは無理」

ぐうの音も出ません。だから、この体験そのものを翻訳することにしました。

**第二の脳は、自分で「作る」ものではなく、AIに整理を任せて「飼う」ものだと思っています。** GitHubもjulesも見えないところに隠して、ユーザーにはスマホから雑に放り込む体験だけを残す。

それがWebアプリ **Sink** です。

3ステップだけ。スマホで思いついたことを放り込めば、裏側で同じ思想（PR的なHint提案＋人間が承認）のエンジンが回ります。  
**「PCを開いてメモを書き、統合する儀式」は、もう要りません。**

### β版を触ってみませんか

Sinkは現在クローズドβ版を公開中です。  
**βテスト期間中の今が、いちばんユーザであるあなたの意見が反映される時期** です。  
触ってみて「ここはこう動いてほしい」「この機能が要る」と感じたことを教えていただけると、とても助かります。  
あなたの「こう動いてほしい」が、そのまま次のリリースになります。

**βテスト版は現在、全機能を無料で試せます。**  
（SINKクレジットの割り当てはありませんが、自分のAPIキーを持ち込む設定も可能です。キーはブラウザ内にのみ保持し、サーバーには一切送られない設計です）

ここから先は、実際に動いているjulesスケジューラのプロンプト、Claude Codeに仕込んでいる統合スキル、Issue→ドラフト化スキル、Obsidian Gitの運用Tipsまで、**設定の中身を全部公開** します。  
同じ構成を自分のVaultで組みたい方は、そのまま使ってください。
