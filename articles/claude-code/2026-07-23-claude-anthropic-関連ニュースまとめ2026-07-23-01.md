Anthropic の **Claude** / **Claude Code** 関連ニュースをまとめた索引だ。記事本文は日本語で PaPoo に投稿している。ここでは各記事の要点を紹介するので、詳細は各リンク先を読んでほしい。

### Claude CodeがiOSアプリの実機テストに近いことをMac上でできるようになった話

Claude CodeのMacアプリに、かなり面白い新機能が入りました。iOSアプリを、ClaudeがSimulator上で実際に動かしながらテストできるようになったんです。しかも会話画面の横にシミュレーター用の表示が出て、コードを書かせて、そのまま動作確認までつなげられる。これは開発体験としてかなり気持ちいいと思います。

[https://papoo.work/doc/a6b307a4ec794008](https://papoo.work/doc/a6b307a4ec794008)

### 1PasswordとClaudeの新しい連携が、「AIにパスワードを渡さない」世界を少し現実にした

1Passwordが、Claude向けの新しいブラウザー連携を出しました。これが何を変えるのかというと、AIエージェントがログイン作業を手伝えるのに、パスワードそのものは見なくて済むようになる点です。

[https://papoo.work/doc/dc86225c2fdc274e](https://papoo.work/doc/dc86225c2fdc274e)

### 「Claude、落ちてる？」を素早く見分けるための非公式ダッシュボード

Claudeを使っていて「なんか遅い」「エラーが返ってくる」と感じたとき、まず気になるのは自分の環境が悪いのか、それともClaude側で障害が起きているのか、ですよね。 その切り分けを手早くやるためのサイトが Claude Status です。名前からしてかなり直球で、役割もそのまま。Claude.ai、Claude API、Claude Code の状態を見て、いま何が起きているのかをまとめてくれます。

[https://papoo.work/doc/134da22e28ba1bb0](https://papoo.work/doc/134da22e28ba1bb0)

### AnthropicがClaudeを「囲い込む」設計をかなり本気で見直している話

Claudeの安全対策と聞くと、つい「危ないことをしそうなら止める仕組みかな」と思いがちですが、今回のInfoQの記事で面白いのは、Anthropicがその発想をかなり冷たく、そして現実的に捉えているところです。要するに、モデルそのものを賢くするだけでは足りない。ファイルに触れられる範囲、ネットに出られる範囲、実行できる範囲を、環境側でガチガチに制限しないとダメだ、という話です。

[https://papoo.work/doc/e0cf5bb0cf4f3f6a](https://papoo.work/doc/e0cf5bb0cf4f3f6a)

### Claude Codeを安く使う裏ワザ、deepclaudeがかなり面白い

GitHubで見つけた deepclaude は、ひとことで言うと「Claude Codeの見た目と操作感はそのままに、裏側のモデルだけを安いものへ差し替える」ためのツールです。 発想がかなり気持ちいいです。高い道具の“外側”はそのまま使って、頭脳だけ別のエンジンに載せ替える感じ。こういうやり方、地味だけど実用性が高いんですよね。

[https://papoo.work/doc/5cf5d0eaac19bf72](https://papoo.work/doc/5cf5d0eaac19bf72)

### SPICEとオシロスコープをClaude Codeにつないで、回路検証を自動化する話

Claude Codeをハードウェア開発に使う。これだけ聞くと、ちょっと夢が大きすぎる気もします。けれど、Lucas Geradsさんのこのデモは「AIに回路を丸投げする」のではなく、「AIにちゃんとフィードバックを返せる環境を渡す」とうまく回り始める、というかなり筋のいい話でした。

[https://papoo.work/doc/20c418f430b38d8f](https://papoo.work/doc/20c418f430b38d8f)
