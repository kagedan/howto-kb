■ ターミナル作業、手数が多すぎると感じたことない？

Claude Codeを毎日使っていると、「この操作、毎回やるのちょっとめんどくさいな」と感じる瞬間が少しずつ積み重なってきますよね。ツールを呼び出すたびにメニューを開いて、項目を選んで、確認して…という流れは1回1回は小さくても、何十回と繰り返すとじわじわ効いてきます。

今回紹介するClaude Code 2.1.186は、そういう「地味にしんどい手数」を減らす方向のアップデートでした。変更件数はCLIだけで33個とボリュームが多めなんですが、その中でも「これ、毎日使う人には刺さるな」と感じた2つの変更を僕なりに丁寧に整理してみます🤖

![](https://pbs.twimg.com/media/HLdF0QsaAAEM5TM.jpg)

■ 一言でいうと、何が変わったのか

今回のアップデートで特に注目したいのは、この2点です。

・ `claude mcp login` / `claude mcp logout` コマンドの追加

・ `!` シェルコマンドの実行結果に対して、Claudeが自動で即応答するように

どちらも「劇的に新しい機能が増えた」というよりは、「今まで手でやってた操作が自動化・省略できるようになった」という性質の変更です。地味に聞こえるかもしれないけど、毎日触っている人ほど「これ助かる」と感じるタイプの改善なんです。

■ そもそもMCPって何？という人へ

MCPというのは「Model Context Protocol」の略で、ClaudeにGitHubやNotion、データベースといった外部ツールやサービスを接続するための"共通規格"みたいなものです。

たとえばUSBがいろんなデバイスを共通の差し込み口でつなげるように、MCPはClaudeとさまざまな外部サービスを統一した方法でつなぐ"変換コネクタ"の役割を担っています。このMCPを使うことで、Claudeは「GitHubのPR（プルリクエスト=コードのレビュー申請）を確認して」「Notionのページを編集して」といった、外部ツールと連携した作業ができるようになります。

MCPサーバーというのは、そのコネクタの"サーバー側"の部品で、GitHub側・Notion側などそれぞれのサービスごとに存在します。そしてこのMCPサーバーを使うためには、「このClaudeのセッションはこのサービスにアクセスしてOKですよ」という認証（ログイン）が必要になります。

![](https://pbs.twimg.com/media/HLdGA9bbIAAnKP2.jpg)

■ Before/After：MCP認証がどれだけ楽になったか

今回の変更の核心は、この認証フローが大幅にシンプルになったことです。

変更前の流れはこんな感じでした。

① Claude Codeのチャット画面で `/mcp` コマンドを入力してメニューを開く

② 表示されたサーバー一覧から認証したいサーバーを選ぶ

③ 認証操作を行う

これが変更後はこうなります。

① ターミナルで `claude mcp login <サーバー名>` を叩くだけ

たったこれだけです。ログアウトも `claude mcp logout <サーバー名>` 一行で完了します。

「メニューを開く→選ぶ→操作する」という3ステップが、1コマンドに集約されました。これだけ見ると「ちょっと便利になった」程度に聞こえるかもしれないんですが、本当に効いてくるのはここからです。

![](https://pbs.twimg.com/media/HLdGNANbsAA5KxT.jpg)

■ なぜ「コマンド一発」がそんなに大事なのか

コマンドラインで完結するということは、シェルスクリプト（Shell Script＝コマンドを並べて自動実行できるプログラム）や自動化フローに組み込めるということです。

たとえばこういう使い方が考えられます。

・プロジェクトAの作業を始めるとき、GitHub MCPにログインするスクリプトを自動実行

・プロジェクトBに切り替えるとき、Notion MCPにログインし直すスクリプトを走らせる

・CI/CD（継続的インテグレーション/デリバリー＝コードをビルド・テスト・デプロイする自動化の仕組み）のパイプラインの中にMCP認証を組み込む

今まではメニュー操作が前提だったため、こういった「人の手を介さない自動化」には組み込みにくかったんです。コマンドとして呼び出せるようになったことで、初めてスクリプトに書けるようになりました。
