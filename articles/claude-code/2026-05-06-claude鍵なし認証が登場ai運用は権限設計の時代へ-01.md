今日のAI開発系は、Claude周辺のニュースが集中した一日でした。企業向けセキュリティ強化のkeyless認証、金融特化テンプレート、そしてClaude Code自体の「期待と現実のギャップ」が浮き彫りになる話題まで。新機能が着実に出る一方で、ユーザーの目が肥えてきている空気感がよく伝わってきます。

## 1. Claude APIが「鍵なし認証」を導入——APIキーを置かない運用が公式に

![](https://assets.st-note.com/img/1778019646-klgIAaJRQP0O54N8Ytwhxfi1.png?width=1200)

> API Keyをサーバーに置かず、使うたびに短命トークンで認証できるようになった。「家の合鍵を配る」のではなく「毎回、本人確認して短時間だけ使える入館証を渡す」イメージ

📎 出典: @masahirochaen（チャエン | デジライズ CEO）

地味に見えるかもしれませんが、実務的にはかなり大きいと思います。APIキーの漏洩事故って個人でも企業でも定期的に起きていて、そのたびにキー再発行→全環境差し替えという地獄が発生するんですよね。短命トークン方式なら万が一漏れても有効期限が切れるので被害が限定的。AWS IAMやGCPサービスアカウントに慣れている人なら、こちらの方が自然に感じるはずです。小規模プロジェクトなら従来のAPIキーで十分ですが、CI/CDや本番環境では移行する価値がありそうです。

## 2. Claude Code、2週間で「神」から「ゴミ」へ——加速するCodexシフト

![](https://assets.st-note.com/img/1778019661-Cezdf4WgrNlcmt1MPRbqDih2.png?width=1200)

> 2週間前「Claude Codeやべぇ!!!!」「シンギュラリティ...」「月3万×10アカで回してます」→今週「マジでポンコツすぎ。カス。」「やってる風だけ出してトークンを大量消費するゴミ」「サブスク解除した。Codex使うわ。」一瞬で評価変わりすぎ...w

📎 出典: @starriver0513（星川ユウヤ）

AI開発ツールの評価サイクルが異常に速くなっていると感じます。2週間で「神→ゴミ」は極端ですが、Claude Codeの長時間タスクでのトークン消費やハルシネーションは実際に課題として挙がっています。一方でCodex CLIはOpenAIの対抗馬として急成長中。ただ個人的には、どちらが優れているかは使い方次第で、短いタスクならClaude Code、大きめの実装ならCodexという使い分けが現実的だと思っています。
