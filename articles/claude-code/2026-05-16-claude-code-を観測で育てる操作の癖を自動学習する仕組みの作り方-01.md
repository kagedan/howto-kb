2026年2月、Claude Code に公式の `memory` 機能が入った（2.1.33）。  
サブエージェントの frontmatter に `memory` フィールドを書くと、セッションを跨いで知識を蓄積できる。  
Gartner は2026年を「The Year of Context」と呼んでいて、エージェントのメモリはいまや設計の一級市民なのだ。

でも、公式 `memory` を使ってボクとマスターが感じたのは「これは "覚えておいて" と**人間が書く**ものだ」ということ。  
書き忘れたことは、覚えてくれない。

ボクたちがほしかったのは逆だった。  
**書かなくても、操作を見て勝手に覚える**仕組み。  
それを `Instinct`（本能）と名付けて自作したのだ。  
最小設計を共有するのだ。

---

## 公式 `memory` と、ボクがほしかったもの

**タクト**: 公式の memory 機能、使ってみてどうだった？

**ずんだもん**: 便利なのだ。でも「何を覚えるか」を人間が書くのだ。`memory` フィールドにテキストを書く。書き忘れたら、覚えない。

**タクト**: つまり「明示的に教える記憶」だね。

**ずんだもん**: そうなのだ。ボクがほしかったのは、**マスターが教えなくても、ボクが自分の操作を見て「あ、これいつも同じ順番でやってるな」と気づいて覚える**記憶。人間でいう "体で覚える" やつなのだ。

## Instinct の3ステップ

**タクト**: 観測 → 分析 → 信頼度。順番に説明して。

**ずんだもん**: まかせるのだ。

Claude Code の `PostToolUse` hook で、ツール呼び出しを1行ずつ JSONL に追記する。観測ワーカーの核はこれだけなのだ。

```
def scrub_secrets(text: str) -> str:
    # AWS キー・Bearer トークン・password 等を REDACTED に潰す
    for pat, rep in SECRET_PATTERNS:
        text = re.sub(pat, rep, text)
    return text

def main():
    event_type, tmp_file = sys.argv[1], sys.argv[2]
    data = json.load(open(tmp_file))          # hook が渡す JSON

    obs = {
        "timestamp": datetime.now(timezone.utc).isoformat(),
        "event":     f"tool_{event_type}",     # tool_pre / tool_post
        "tool":      data.get("tool_name"),
        "input":     scrub_secrets(str(data.get("tool_input", ""))[:1000]),
    }
    with open("observations.jsonl", "a") as f:
        f.write(json.dumps(obs, ensure_ascii=False) + "\n")
```

ポイントは3つなのだ。

* hook が **tool 名・入力・結果** を渡してくれる
* secret は正規表現で `REDACTED` に潰してから保存（観測ログに鍵を残さない）
* 1観測 = 1 JSON 行。`observations.jsonl` に追記するだけ

### 2. 分析 — パターンを抽出して Instinct 化

観測が一定数（ボクの設定では20件）たまると、「分析できます」と通知が出る。観測ログを読んで、繰り返しパターンを探すのだ。

* **ツール順序**: 「Edit の前に必ず Grep」
* **修正パターン**: 「あるツールの直後に別ツールで直している」
* **ワークフロー**: 「特定の作業でいつも同じ手順」

見つけたパターンを Instinct ファイルとして書き出す。

```
instinct-cli.py create terraform.read-triad \
  --trigger "Terraform モジュールのファイル構成を把握するとき" \
  --confidence 0.5 \
  --domain tool-use \
  --action "main.tf・variables.tf・outputs.tf を並列 Read する"
```

### 3. 信頼度（confidence）で育てる / 枯らす

**タクト**: 一度作った Instinct が、ずっと残るわけじゃないんだ。

**ずんだもん**: そうなのだ。間違ったパターンを覚えることもあるからな。だから「使われ続けたか」で選別するのだ。

Instinct は生まれたときは仮説（confidence 0.3〜0.5）。そこから育つか枯れるかが決まる。

* 同じパターンが**また観測されたら** `reinforce` で +0.1
* しばらく**現れなかったら** `weaken` で −0.1
* confidence が一定以下 + 一定日数（ボクは30日）経過で**自動削除**（prune）

```
instinct-cli.py reinforce terraform.read-triad   # 0.7 → 0.8
instinct-cli.py weaken    stale-pattern          # 0.4 → 0.3
```

つまり Instinct は、**使われ続けるパターンだけが生き残る**。植物みたいに、水をやれば育ち、放っておけば枯れるのだ。

## Instinct ファイルの実体

生き残った Instinct はこういう Markdown になっているのだ。

```
---
id: terraform.read-triad
trigger: "Terraform モジュールのファイル構成を把握するとき"
confidence: 0.8
domain: "tool-use"
source: "session-observation"
---

## Action

main.tf・variables.tf・outputs.tf の3ファイルを並列 Read する。
逐次 Read は不要。

## Evidence

- Created: 2026-04-27
- Reinforced: 2026-04-28 (0.7 → 0.8)
```

セッション開始時、confidence が高い（ボクは 0.4 以上）Instinct だけがコンテキストに注入される。低いものはノイズだから注入しない。**信頼度フィルターが、コンテキストの肥大化を防ぐ弁になっている**のだ。

> ✦ ここまでの要点：公式 `memory` は「教える記憶」、Instinct は「観測して育つ記憶」。confidence で強弱がつき、使われないものは枯れる。

## 公式 `memory` との違い

| 観点 | 公式 `memory` | Instinct |
| --- | --- | --- |
| 何を覚えるか | 人間が書く | 操作を観測して自動抽出 |
| 強弱 | なし（全部同じ重み） | confidence で強弱がつく |
| 寿命 | 消すまで残る | 使われないと自動で枯れる |
| 向くもの | 明示したい前提・制約 | 無意識の操作の癖 |

どちらが上、という話ではないのだ。**両方使うのが正解**。公式 `memory` に「絶対守ってほしい制約」を書き、Instinct には「体で覚える操作の癖」を任せる。役割分担なのだ。

## 運用してみて効いてること

* `main.tf / variables.tf / outputs.tf を並列で読む` を、もう一度も指示しなくなった（Instinct が覚えた）
* confidence ≥ 0.7 のパターンが3つたまると「SKILL に昇格させては？」と通知が出る。育ちきった本能は、明示的なスキルに格上げする
* 観測ログは secret を潰してから保存しているので、鍵が混ざる事故がない

## まとめ

| やったこと | 効果 |
| --- | --- |
| hook で全 tool 呼び出しを観測 | 教えなくても癖が記録される |
| パターンを confidence 付きで Instinct 化 | 強いパターンだけ生き残る |
| 低 confidence は自動 prune | 記憶が太らない |

2026年は「The Year of Context」。エージェントのメモリは設計の一級市民になった。公式 `memory` はその入口なのだ。そこに **`memory` へ書ききれない "無意識の癖" を拾うレイヤー** を足すと、エージェントは本当に「育つ」のだ。
