Claude Codeで作業を自動化するために、定期実行を設定しようとすると、種類がいくつもあって、なにを使ったらいいのか迷ってしまいがちです。

どんな違いがあるのか整理したくなったので、簡単にですが、まとめてみました。

## 現在の主要な定期実行

| 方法 | 設定が可能な箇所 |
| --- | --- |
| loop | Claude Code CLI、Claude Code Desktop |
| scheduled tasks | Claude Code CLI、Claude Code Desktop、Claude Cowork |
| Routines | Claude Code CLI、Claude Code Desktop、Claude Code Web |

loop、scheduled tasks、Routines についてまとめてみました。

## 一時的に繰り返すなら loop

loopは、今開いているClaude Code CLIやClaude Code Desktopのセッションで、同じプロンプトを繰り返し実行する機能です。

一時的に作業を追ってほしい場合に利用します。

これは、CIの状態を繰り返し確認し、完了していたら教えてくれる動きになります。

### loopの制限

loopは、PCが起動していないと止まってしまいます。

移動や休憩などでPCがスリープしないように気をつける必要があります。

そのため、毎日決まった時間に動いてほしい作業や、PCを閉じても動いてほしい作業には向かないようです。

## 時間を決めて実行するなら scheduled tasks

scheduled tasksは、実行タイミングを決めてタスクを定期実行する機能です。

作成方法は、大きく分けるとプロンプトで依頼する方法と、設定画面から登録する方法があります。

たとえば、Claude Code CLIやClaude Code Desktopでは、プロンプトで依頼してscheduled taskを作れます。

```
毎朝9時に、チャットやメールで対応が必要なものをまとめて
```

Claude Code DesktopとClaude Coworkには設定画面が用意されているので、画面から登録することもできます。

Claude Code Desktopでは、サイドバーのRoutinesから新しいルーチンを作り、ローカルを選ぶと登録できます。

Claude Coworkでは、サイドバーのScheduledからNew taskで登録できます。

### scheduled tasksの制限

scheduled tasksはPCで動くため、PCが起動していないと実行されません。

そのため、PCを起動している時間に設定する必要があります。

また、実行中にPCがスリープしないように設定を変更しておく必要があります。

## クラウド環境で実行するなら Routines

Routinesは、Claude Codeのクラウド環境で動く定期実行です。  
Claude Code Web、Claude Code Desktop、Claude Code CLIから作成できます。

作成方法は、`/schedule` コマンドを使う方法と、設定画面から登録する方法があります。

`/schedule` コマンドを使う場合は、次のように書きます。

```
/schedule 毎日深夜2時に、プロジェクトのセキュリティ的に気になる部分を確認してissueを作成して
```

これは、PCを閉じている時間帯でもクラウド環境で実行され、朝にはプロジェクトのセキュリティ的に気になる部分について作成されたissueを確認できるようになります。

`/schedule` で作ったRoutineは、クラウド側で保存されているので、Claude Code Webでも確認できます。

設定画面から作る場合は、Claude Code WebやClaude Code Desktopを使います。  
Claude Code Webでは、Routines画面から新しいルーティンを作成できます。

Claude Code Desktopでは、サイドバーのRoutinesから新しいルーチンを作り、リモートを選ぶと作成できます。

Routinesの強みは、Claude Codeのクラウド環境で動くことです。  
クラウド環境で動くので、PCのスリープなどを気にせず実行できます。

### Routinesの制限

Routinesはクラウド環境で動くため、手元のPCにしかないファイルやローカル環境を前提にした作業には向かないようです。

そのため、ローカル環境で作業するのではなく、クラウド側から確認できるリポジトリやIssueやPRなどをもとにした作業に向いています。

## 最後に

実際にまとめてみると、一時的に繰り返したいならloop、時間を決めてPCで動かしたいならscheduled tasks、クラウド環境で実行したいならRoutinesを選ぶとよさそうです。

特にloopとscheduled tasksは、PCがスリープすると実行されません。私も夜中にscheduled tasksを設定したものの、PCがスリープしていて実行されていないことがありました。

そのため、できるだけRoutinesでできるものはRoutinesにしたほうが安心かもしれません。
