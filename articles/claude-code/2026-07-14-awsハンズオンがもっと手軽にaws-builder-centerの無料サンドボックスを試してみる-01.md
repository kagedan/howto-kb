## 1. はじめに

AWS（Amazon Web Services） のハンズオンを試す際、これまでは個人の AWS アカウントを用意し、クレジットカードを登録し、利用後にリソースを削除する必要がありました。

もちろん、実務で AWS を扱ううえでは、アカウント管理や課金管理、IAM 権限、リソース削除の考え方を理解することは重要です。一方で、AWS を学び始めたばかりの方や、社内勉強会・顧客向けワークショップで複数人に環境を触ってもらう場合、事前準備や削除漏れへの不安がハードルになることもあります。

そのような中、**AWS Builder Center の一部ワークショップで、無料のサンドボックス環境が利用できる**ようになりました。

AWS 公式発表によると、対象ワークショップでは、個人の AWS アカウントやクレジットカードを用意せずに、事前にプロビジョニングされた AWS アカウント上でリソースのデプロイ、コード実行、実験ができるようになります。

## 2. AWS Builder Center とは

AWS Builder Center は、AWS を学び、作り、コミュニティとつながるためのビルダー向けポータルです。

AWS Builder ID でサインインすることで、記事の閲覧、コミュニティ参加、AWS Skill Builder コースへのアクセスなど、AWS を学ぶための入口として利用できます。

また、AWS Builder Center から AWS Workshops のカタログにアクセスできるようになっており、AWS の専門家が作成したステップバイステップ形式の手順に沿って、AWS サービスの使い方を学ぶことができます。

## 3. 何が変わったのか

今回のアップデートを一言でいうと、AWS のハンズオンを始めるためのハードルが大きく下がった、ということだと思います。

これまで AWS Workshops を進める場合、多くのケースでは自分自身の AWS アカウントを使ってリソースを作成していました。

一方、**サンドボックス対応ワークショップでは、対象ワークショップから無料の AWS 環境をリクエスト**できます。これにより、以下のような準備や不安を減らせます。

| これまで | サンドボックス対応後 |
| --- | --- |
| 個人の AWS アカウントを用意する必要がある | 対象ワークショップでは不要　※ AWS Builder IDは必要 |
| クレジットカード登録が必要になる場合がある | 不要 |
| 想定外の課金を気にする必要がある | 無料・時間制限付きの環境で実施 |
| 作成したリソースを削除する必要がある | 利用後に自動クリーンアップ |
| 初学者向けに事前準備の説明が必要 | ハンズオン本体に集中しやすい |

特に、社内勉強会や顧客向けハンズオンで使いやすくなった点が大きいと感じます。

## 4. サンドボックス環境の主な特徴

AWS 公式発表で説明されているサンドボックス環境の特徴を整理すると、以下の通りです。

| 項目 | 内容 |
| --- | --- |
| 対象 | AWS Builder Center の一部対象ワークショップ |
| AWS アカウント | 個人の AWS アカウントは不要 |
| クレジットカード | 不要 |
| 利用時間 | アクティベーションから 8 時間 |
| クリーンアップ | 利用後に自動クリーンアップ |
| リクエスト頻度 | 1 ビルダーにつき週 1 回 |
| リセットタイミング | 毎週日曜日 |
| 環境準備時間 | 多くの環境は 15 分以内に準備 |
| 対象範囲 | ローンチ時点では一部ワークショップ。今後拡大予定 |

ここで重要なのは、単に無料で試せるだけではなく、後片付けを気にせずに試せることです。

AWS のハンズオンでは、S3 バケット、Lambda 関数、IAM ロール、CloudFormation スタック、Amazon Bedrock 関連の設定など、複数のリソースを作成することがあります。

初学者にとっては、作成したリソースを正しく削除できているか確認すること自体が負担になります。サンドボックス環境であれば、一定時間後に自動クリーンアップされるため、削除漏れや想定外の課金に対する不安をなくせます。

## 6. どのような用途に向いているか

サンドボックス対応ワークショップは、特に以下のような用途に向いていると感じました。

| 用途 | 向いている理由 |
| --- | --- |
| AWS 初学者向け学習 | 個人アカウントや課金を気にせず始められる |
| 社内勉強会 | 参加者ごとの AWS アカウント準備を減らせる |
| 顧客向けハンズオン | 顧客側で AWS アカウントを用意しなくても体験しやすい |
| 生成 AI / AI エージェント体験 | Bedrock、AgentCore、Kiro などを実環境で触れる |
| 短時間の技術検証 | 2〜4時間程度のワークショップであれば 8 時間枠に収めやすい |
| 非エンジニア向け体験 | Amazon Quick など、分析・自動化系の体験にも使いやすい |

個人的には、特に社内勉強会や顧客向けワークショップで有効だと感じます。

これまで、ハンズオンを開催する際には、参加者に AWS アカウントを準備してもらう、必要な権限を確認する、請求アラートやリソース削除を案内する、といった準備が必要でした。

サンドボックス環境を使えるワークショップであれば、参加者はハンズオン本体に集中しやすくなります。主催者側も、環境準備や後片付けの説明に使う時間を減らし、サービスの考え方や実務での活用ポイントの説明に時間を使いやすくなります。

## 8. 注意点

便利なアップデートですが、いくつか注意点もあります。

### すべてのワークショップが対象ではない

現時点では、サンドボックス環境が利用できるのは一部のワークショップのみです。

AWS 公式発表でも、ローンチ時点では select workshops が対象であり、今後さらに有効化されると説明されています。そのため、実施したいワークショップがサンドボックス対応しているかは、実施前に AWS Builder Center 上で確認する必要があります。

### 利用時間は 8 時間

サンドボックス環境は、アクティベーションから 8 時間利用できます。

そのため、途中まで進めて翌日に続きを実施するような使い方には向いていません。長めのワークショップや複数ワークショップを連続で試す場合は、時間配分に注意が必要です。

### 週 1 回のリクエスト制限がある

サンドボックスは、1 ビルダーにつき週 1 回リクエストでき、毎週日曜日にリセットされます。

短期間に何度も試したい場合や、複数のワークショップを検証したい場合は、この制限を踏まえて計画する必要があります。

### 永続的な検証環境ではない

サンドボックス環境は、自動クリーンアップされる一時的な環境です。

作成したリソースを後から見返したり、継続的に開発・検証したりする用途には向いていません。あくまで、対象ワークショップを安全に体験するための一時環境として捉えるのがよさそうです。

本格的な PoC や本番に近い構成検証では、自社または個人の AWS アカウントを利用する必要があります。

## 9. まとめ

AWS Builder Center の一部ワークショップで、無料サンドボックス環境が利用できるようになりました。

ポイントをまとめると、以下の通りです。

* 対象ワークショップでは、個人の AWS アカウントやクレジットカードなしでハンズオンできる
* サンドボックス環境は無料で、アクティベーションから 8 時間利用できる
* 利用後は自動クリーンアップされる
* 1 ビルダーにつき週 1 回リクエストできる
* ローンチ時点では一部ワークショップが対象で、今後拡大予定
* 2026年7月12日時点では、画面上で 8 件のサンドボックス対応ワークショップを確認できた
* 生成 AI / AI エージェント関連のワークショップも含まれており、社内勉強会や顧客向けハンズオンで使いやすそう

これまで AWS のハンズオンでは、アカウント準備や課金管理、リソース削除が最初のハードルになることがありました。

今回のサンドボックス対応により、**AWS を「まず触ってみる」までの距離がかなり短くなった**と感じます。

特に、AWS 初学者向けの勉強会や、生成 AI / AI エージェント関連の技術体験を短時間で実施したい場合には、有効な選択肢になりそうです。

今後、より多くのワークショップがサンドボックス対応していくことに期待したいです。
