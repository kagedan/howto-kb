はじめに

前回の記事では、ソースコードを中央サーバーに露出させない「ゼロ知識暗号化」と「P2P通信」によるセキュアなAI開発インフラの思想について解説しました 。今回は実践編です。

Anthropicが提唱し、今やAIネイティブ開発のデファクトスタンダードとなったMCP（Model Context Protocol）を活用し 、手元のCursorやClaude Codeに「プロジェクトの意思決定（Why）」を学習・記憶し続ける独自の長期記憶レイヤー（MCPサーバー）を組み込むハンズオンをお届けします 。 数日経つと「前に決めたアーキテクチャのルール」を綺麗さっぱり忘れてしまうAIアシスタントに 、「忘れられない記憶（Decision Genome）」を授けてみましょう 。

1. 今回構築するアーキテクチャ今回構築するシステムは、エディタ（CursorやClaude Code）の裏側で、Kawa Codeの軽量なデーモン（CLI/ローカルMCPサーバー）を起動する構成です 。 手元のPCで動くKawa CodeのMCPサーバーが、AIとあなたの会話やGitのステージング領域から「開発の意図（Intent）」を自動で抽出し 、分類・剪定した上で 、ローカルで安全に暗号化してインデックス化します 。  
   And、AIが新しいコードを生成しようとする「その瞬間」に 、関連する過去の意思決定の記憶だけをJIT（Just-In-Time）でコンテキストに再注入します 。
2. ステップ1：Kawa Code CLIのセットアップまずは、ローカル環境で動作するKawa Codeのコアモジュール（軽量なCLIツール）をインストールします 。 インストール（Mac / Linux / Windows対応）お使いの環境に応じて、以下のコマンドを実行します 。 Bash# Homebrewを使用する場合 (Mac)  
   brew install kawacode/tap/kawa

# または、cargoを使ってソースからビルドする場合

cargo install kawacode-cli  
※Kawa Codeのコアはバイナリサイズが小さく高速な Rust ベースのデーモン（CLI）として構築されているため、エディタの動作やメモリを圧迫しません 。 初期化とプロジェクトの登録開発を行っているプロジェクトのルートディレクトリに移動し、初期化を行います。Bashcd /path/to/your-project  
kawa init  
このコマンドを実行すると、ローカル環境に暗号化されたローカルインデックスが生成されます 。秘密鍵はあなたのマシンのセキュアキーチェーン内にのみ保存され、外部に送信されることはありません 。

3. ステップ2：エディタ（AIクライアント）への接続設定次に、インストールしたKawa CodeをMCPサーバーとして各AIツールに認識させます 。 🛠️ Case A: Cursorに設定する場合Cursorを開き、 Settings -> Features -> MCP の順にナビゲートします。+ Add New MCP Server をクリックし、以下のように設定します：Name: kawa-codeType: commandCommand: kawa mcp start🛠️ Case B: Claude Codeに設定する場合ターミナルで動作する claude CLI（Claude Code）をお使いの場合は、 ~/.claude.json （またはプロジェクトローカルの claude.json）の mcpServers セクションに以下の記述を追加します 。

JSON{  
"mcpServers": {  
"kawa-code": {  
"command": "kawa",  
"args": ["mcp", "start"]  
}  
}  
}

設定が完了すると、AIエディタが起動した瞬間にバックグラウンドでKawa Codeと通信を開始します。

4. ステップ3：記憶の「自動キャプチャ」を体感する設定が終わったら、通常通りAIを使ってコーディングを行ってみてください 。  
   例えば、CursorのComposer機能やClaude Codeを使って、以下のような設計変更の指示を出します 。 👤 あなた:「認証モジュールのトークン検証処理を、セキュリティ強化のためにJWTから、ローカルセッションベース（Redis管理）に変更して。それに伴う依存関係のリファクタリングもお願い」AIエディタが複数のファイルを修正し、リファクタリングを終えた裏側で、Kawa CodeのMCPサーバーは以下のシグナルを自動的にキャプチャしています ： あなたが出したプロンプト（解決したかった課題・背景） AIがそのコードを選択した推論プロセス（思考チェーン） 実際に書き換えられたコード差分のセマンティクス（変更の本質） これらは自動的に「意思決定ゲノム（Decision Genome）」として構造化され 、ローカルでクリーンに整理・暗号化された上で保存されます 。人間がわざわざ ReadMe.md や Cursorrules に手動で書き起こす必要は一切ありません 。 5. ステップ4：JIT（ジャストインタイム）発現の凄さを知る数日後、全く別の新機能を実装しているときに、Kawa Codeの真価が発揮されます 。 👤 あなた:「新しく追加するユーザープロファイル画面のAPIエンドポイント（ /api/profile ）を実装して」通常、AIアシスタントは数日前の「JWTからRedisセッションに変更した」というルールの文脈を忘れているため 、平気で古いJWTベースの検証コードを生成しようとします 。 しかし、Kawa Codeが有効な環境では、AIが profile 関連のコードに触れようとした瞬間、MCPサーバーが「インテント（作業の意図）スコープ」を検知します 。そして、過去のRedisセッション移行時の意思決定コンテキストだけをピンポイントで抽出し、AIのプロンプトの先頭に自動注入します 。 🤖 AIの内部思考（Kawa Codeによるコンテキスト注入後）:（過去のインデックスから、このプロジェクトではトークン検証にJWTではなくRedisセッションを使用する決定がなされていることを検出しました 。新しいエンドポイントにもRedisセッション検証ロジックを適用します...） 結果として、AIはあなたのチームの「最新の設計の意図」を完全に汲んだ、手戻りのない100点満点のコードを出力します 。 プロンプトに毎回長大なコンテキストをコピペしてトークン代をドブに捨てる（コストの線形増加）時代は、これで終わりです 。

おわりに：AIの記憶を、あなたのローカル環境に。Kawa Codeが提案するMCPベースの長期記憶層は、開発者の認知負荷を極限まで下げ 、AI駆動開発の安全性を高めるための新しいインフラです 。 まずは、個人開発の小さなリポジトリからでもその効果を体感できます 。  
「AIがすぐに設計ルールを忘れてしまってイライラする」「毎回同じコンテキストを説明するのが面倒」と感じている方は、ぜひこのMCP連携の威力を試してみてください ！
