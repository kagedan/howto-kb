「Claude Codeはエンジニア向けのツールでしょ？」とよく聞かれる。 でも私はUIデザイナーとして、ワイヤーフレーム生成・UIレビュー・Figmaプラグイン作成・スキル運用まで、ほぼ全部Claude Codeでやっています。

ここ1ヶ月で見えてきたのは、**「Claude Codeを使いこなせるかどうか」は、ショートカットの引き出しの多さで効率が上がる**ということ。同じコマンドを叩いていても、Esc Esc を知っているかどうかで作業の効率が3倍変わる。

この記事では、UX設計で日常的に使っている、**本当に手が動く18個**をまとめた。 2026年5月時点の最新版（Claude Code v2.1.x / Opus 4.7対応）で、新コマンドも含めて整理しています。

## 日常編：毎日触るやつ

### 1. Esc Esc（ダブルタップで巻き戻し）

**結論：Claude Codeで最初に覚えるべきショートカット**

Escキーを2回連続で押すと、巻き戻しメニューが開く。出てくる3つの選択肢が秀逸：

* 会話だけ巻き戻す
* **コードだけ巻き戻す** ← これがキラー機能
* 両方巻き戻す

「コードだけ巻き戻す」を使うと、**ファイルの変更だけ全部リセットされて、Claudeは「何を試して何が失敗したか」を覚えたままになる**。実験的なUI試作をしているときの神機能。

/rewind コマンドでも同じことができるけど、私は完全にEsc Escで叩いている。

### 2. Shift + Tab（パーミッションモード切り替え）

Claude Codeには複数の「権限モード」がある：

* **Plan**：読むだけ、編集しない（議論モード）
* **Accept Edits**：編集は自動承認
* **Auto Mode**：危険操作以外は全部自動実行
* **デフォルト**：毎回確認

これを /config から切り替えるのは面倒。**Shift + Tabで1キーで循環できる**。

私はワイヤーフレーム生成時はAuto、コード本体に手を入れるときはAccept Edits、Murakami-sanに見せる仕様書を書くときはPlanモード、と使い分けている。

### 3. /compact（コンテキスト圧縮）

セッションが長くなってきたら呼ぶ。会話履歴を要約してコンテキストウィンドウを空ける。

/clearとの違いがよく聞かれるけど、**/clearは完全リセット、/compactは要約して継続**。タスクの続きを行うなら/compact、別作業を始めるなら/clear。

経験的に、**20〜30分作業したら1回**叩くくらいがちょうどいい。長いUIレビューを回すときは特に。

### 4. @ メンション（ファイル・URL読み込み）

```
@README.md            # ファイル指定
@src/components/      # ディレクトリ全体
@https://example.com  # URL読み込み
```

**コピペするより圧倒的に高速かつ正確**。Claudeのトークン処理が良くなるし、どのファイルを参照したかが履歴に残る。

私はFigmaのShare URLや、design.mdを@で渡すことが多い。「このページのFVを参考に」ってときに、URLを@付きで貼るだけで読みに行ってくれる。

### 5. ! プレフィックス（シェル直挿し）

プロンプトの先頭に!をつけると、その後ろがシェルコマンドとして実行されて、出力が会話に含まれる。

```
!git log --oneline -5
!ls components/
!cat package.json
```

「いまどんな状態？」を説明する代わりに、コマンド結果を直接Claudeに見せられる。**新規ユーザーがほぼ知らないけど、知ったら戻れない**系の機能。

### 6. /resume と claude -r（セッション再開）

Claudeは新しく起動すると会話の記憶を持たない。続きから作業したいときに使う。

私は「昨日のワイヤーレビューの続きを今朝再開する」のがルーティンなので、ターミナル開いたら claude -r でスタートしている。

### 7. \ + Enter と Ctrl + G（長文プロンプト入力）

地味だけど、知らないとストレスが溜まるやつ。

要件を長文で渡したいときはCtrl + G一択。VS CodeやVimで書いて、保存して閉じればそのまま送信される。ワイヤー要件のような構造化された長文プロンプトを書くときは、絶対これ。

## 品質向上編：レビュー・推論強度のコントロール

### 8. /effort（推論強度の調整）

Claude Codeのコード出力の「考える深さ」を調整するコマンド。レベルは：

```
low / medium / high / xhigh / max
```

**Opus 4.7のデフォルトはxhigh**（以前の記事でhighと書かれているものは古い情報）。Claude Code開発リードのBoris Cherny自身が普段使いしているのもxhigh。

私の使い分け：

毎回maxで叩くとコストもレスポンスも重いので、**作業のフェーズに合わせて下げる**のが鉄則。

### 9. /ultrareview（マルチエージェント・クラウドレビュー）

2026年v2.1.x系で新登場。/reviewとの違い：

Pro・Maxは2026年5月5日まで月3回無料、以降はExtra Usageが必要。

ui-reviewスキルみたいな\*\*「複数の観点で並列レビューさせる」発想を、Anthropic自身が標準機能として実装した\*\*形。仕様書や重要なPRを出す前のセルフチェックに使っている。

### 10. /security-review（セキュリティ専門レビュー）

/reviewより徹底的に、**セキュリティのみに絞って検査**する。検出対象：

* インジェクション脆弱性
* 認証バイパス
* コミットに混入したシークレット
* 依存ライブラリの脆弱性

デザイナーがあまり触らない領域だけど、**Figmaプラグインを書いたとき**や、**MCP連携の認可フロー**を実装するときは1回叩くようにしている。Murakami-sanに渡す前のセルフチェック用。

### 11. /loop（自動反復改善）

新設コマンド。**「目標スコアに達するまで自動で改善ループを回す」発想がClaude Code標準機能として入った**。

私が以前作ったautouiresearch（UIスコアを32→98に自動改善するループ）と同じ思想で、/loopで「90点を超えるまで改善し続けて」と指示すると、評価→修正→再評価を自律的に繰り返してくれる。

UIの試作・ロゴ案・コピー案など、**「複数案を出して評価して選び直す」系のクリエイティブ作業と相性が良い**。

## 並列・自動化編：大量作業を捌く

### 12. /batch（並列処理）

複数ファイルのリファクタリング、ダークモード対応、コンポーネント全部にPropを追加…みたいな\*\*「同じことを何十回もやる」作業\*\*に特化したコマンド。

Gitのworktree（別々の作業ディレクトリ）を自動生成して、複数のエージェントを並列で立ち上げて処理する。

私はsophia-clientのコンポーネント30個に対して、トークンの一括差し替えをしたとき初めて使った。順次やったら1時間かかるところが10分弱で終わる。

### 13. オートランモード（/config から設定）

/config から設定できる権限モード。Claudeに権限を渡して自動実行させつつ、**危険な操作（環境変数の読み込み・本番環境へのプッシュ等）のときだけ人間に許可を求める**。

Shift + Tabで切り替えるオートモードと内容は同じだけど、**/configで永続設定にしておく**と起動時から有効になる。

### 14. fewer\_permission\_prompts

過去の許可履歴を学習して、**毎回許可している内容には自動でパーミッションを与える**設定。

「npm installは毎回許可してるよね、もう聞かなくていいよ」をClaude側に判断させる感じ。確認プロンプトが半分以下に減る。最近のバージョンでは /less-permission-prompts というコマンド名にもなっている。

### 15. cc max の活用

これは別ツールだけど、Claude Code運用の文脈では必須。**ターミナルを簡単に分割できる軽量CLIツール**。

Cursorのようなリッチエディタは重いので、cc maxでターミナルを4分割して、それぞれにClaude Codeを立ち上げる。

* 左上：sophiaのワイヤー生成
* 右上：metisのUIレビュー
* 左下：Figmaプラグイン開発
* 右下：note記事の下書き

**エディタに依存せず、複数のClaude Codeを独立稼働させられる**のが強み。同様の選択肢としてBeamやtmuxもあり。

## 隠れ便利編：知ってると差がつく

### 16. /btw（By The Way）

長い処理を待っている間に、**メインのコンテキストを汚さずにちょっとした質問**ができる。

例：「いまUIレビュー回してる最中だけど、GitHubのworkflowってどう書くんだっけ？」をメインの会話に影響させずに聞ける。

2026年3月にClaude Codeチームが追加した機能で、Twitterで2.2M viewsを記録した話題のコマンド。**「質問のために作業を中断する」が消える**のが本質。

### 17. /insights（自分の使い方分析レポート）

ユーザーの指示の出し方を分析して、**「上手くいっている点」「改善案」をレポートで出してくれる**コマンド。

自分の癖を客観的に見られるので、定期的に叩くと勉強になる。私は月1で叩いて、**プロンプトの出し方を見直す材料**にしている。デザイナーがUI/UXを評価するように、自分のプロンプティングをUI/UX評価するイメージ。

### 18. design.md ファイルの活用

これはコマンドじゃなくテクニックだけど、デザイナーには絶対外せない。

プロジェクトルートにdesign.mdを置いて、デザイントークン・スタイルガイド・参考デザイン（shadcn/ui、Linear、Resend、Vercel等）を記述しておく。

```
# Design Direction

## Color
- Primary: #009bff (天秤AI Blue)
- Reference: shadcn/ui slate scale

## Typography
- Font: Inter / Noto Sans JP
- Headings: 600 weight, tight tracking

## Style Reference
- Linear: clean, dense, monochromatic
- Resend: spacious, modern, gradient accents
```

design.mdを読み込ませた上で「このアプリのスタイルに統一して」と言うだけで、**全体のデザイン方向性を一括で揃えてくれる**。

私はCLAUDE.mdとは別にdesign.mdを必ず置いていて、**ブランドガイドラインの「実行可能版」として運用**している。

## まとめ：覚える優先順位

全部覚えるのは無理なので、優先順位をつけるとこうなる：

**今日から使うべき5つ**

1. Esc Esc（巻き戻し）
2. Shift + Tab（モード切替）
3. /compact（圧縮）
4. @ メンション
5. ! シェル直挿し

**1週間で慣れたい5つ** 6. /effort の使い分け 7. claude -r（再開） 8. Ctrl + G（外部エディタ） 9. design.md 運用 10. /ultrareview

**月単位で活用したい8つ** 残り全部。特に/batchとcc maxは、**作業量が一定量超えてから真価を発揮**する。
