## なぜ手作業でのスクリプト作成は「沼」なのか？

プログラミング初心者の頃や、新しく開発案件に入ったばかりの頃は、何かと手作業や「ググりながらスクリプトを書く」ことからスタートしますよね。

しかし、ちょっとしたデータ処理やファイル操作の自動化でも、いざコードを書こうとすると意外と時間がかかります。手作業やイチからのスクリプト作成の最大のデメリットは以下の3つです。

1. **文法やライブラリを調べるのに時間がかかる**（「Python 正規表現 抽出」などで何度も検索）
2. **ちょっとしたエラーで手が止まる**（パスの指定ミスや型エラーなど）
3. **本来の開発業務（アプリのロジック構築など）に集中できない**

特に「ちょっとしたエラーで手が止まる」のは致命的です。サクッと終わらせるつもりが、環境構築やエラー対応にハマってしまい、気づけば数時間が溶けていた…という経験がある方も多いのではないでしょうか。

ここで少し、私が参加している長期インターンでの実体験をお話しさせてください。

しかし今回は違いました。最近話題になっていた**「Claude Code」**を使ってみることにしたのです。
