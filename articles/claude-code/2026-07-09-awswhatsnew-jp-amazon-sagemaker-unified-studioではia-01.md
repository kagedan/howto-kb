Amazon SageMaker Unified Studioでは、IAMベースのドメインでカタログにカスタムアセットタイプが追加されます

Amazon SageMaker Unified Studioでは、IAMベースのドメイン向けにカスタムアセットタイプをサポートするようになった。カスタムアセットタイプにより、ドメイン管理者はAmazon S3上の医療画像ファイル、PowerBIで構築された収益ダッシュボード、またはサードパーティプラットフォームで生成されたPDF形式の研究報告書など、SageMaker Unified Studio内であらゆる形式のアセットをカタログ化できる。カスタムアセットタイプにより、基盤となる形式に関係なくすべてのアセットがSageMakerカタログに統合され、チームは別々のツールやプロセスを必要とせずにアセットを検索、発見、購読できる。

始めるには、管理者は名前、説明、およびオプションのメタデータフォーム（各アセットが持つべきフィールドを定義）でカスタムアセットタイプを作成できる。その後、そのタイプから個々のアセットを作成し、用語集項目やREADMEドキュメントで豊か化して、人間やAIエージェントにビジネスコンテキストを追加し、発見可能の状態で公開できる。公開されると、ドメイン内の誰でも名前、タイプ、または用語集項目でアセットを検索し、他のカタログアセットと同様の管理されたワークフローでサブスクリプションをリクエストできる。

IAMベースのドメイン向けカスタムアセットタイプは、Amazon SageMaker Unified Studioが提供されているすべてのAWSリージョンで利用可能である。詳細については、SageMaker Unified Studioユーザーガイドを参照してください。
