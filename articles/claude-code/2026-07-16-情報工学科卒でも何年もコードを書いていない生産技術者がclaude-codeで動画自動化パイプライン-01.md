## はじめに

情報工学科出身です。学生時代はFortranで研究をしていましたし、社会人になってからもC言語やVisual Basicを実務で書いていた時期があります。

でも今は生産技術者として働いていて、コードを書く機会はほとんどありません。何年もキーボードでプログラムを書いていない、完全な"ブランクエンジニア"です。

副業でAI生成の動画コンテンツ（Instagramで公開している職場コメディ動画シリーズ）を作っていて、セリフの書き起こしや字幕合成といった作業を手作業で繰り返すうちに「これ、仕組み化できないか」と思うようになりました。そこで触り始めたのがClaude Codeです。

この記事では、ブランクのあるエンジニアが実際にClaude Codeでどこまでの自動化を組めたか、具体的にまとめます。

## 組んだもの：動画の字幕・バナー自動合成パイプライン

生成AI（Veo/Google Flow）で作った動画素材に対して、次の流れをコマンド一つで実行できるようにしました。

1. **Whisperで音声からセリフを自動書き起こし**（タイムスタンプ付き）
2. **Claude APIで書き起こしテキストを整形**（誤字修正・改行位置調整・話者ごとの区切り）
3. **ffmpegで字幕（SRT/ASS形式）とバナー画像を動画に合成**

実行コマンドはこんなイメージです（実際に使っているフラグ構成）。

```
python main.py ep_raw.mp4 `
  --hook "うちの部長、秘密を握られてた件" `
  --caption-cta "続きは本文で↓" `
  --ending-720 ending_bumper_720.mp4 `
  --ending-1080 ending_bumper_1080.mp4
```

`--hook`でその回の引きのフレーズを指定し、`--caption-cta`で動画末尾に出すCTAテキストとその表示時間を制御、`--ending-720`/`--ending-1080`で解像度別のエンディング映像を差し込む、という設計です。

## ブランクがあっても組めた理由

C言語やVBを触っていたのはもう何年も前で、正直「今の自分に自動化スクリプトなんて組めるのか」という不安がありました。実際にやってみて分かったのは、必要だったのは「昔の文法を覚えている」ことではなく、「何を実現したいかを具体的に日本語で伝えられる」ことでした。

例えば、最初につまずいたのは日本語字幕のフォント表示でした。Windows環境だと、フォントパスを明示的に指定しないと意図しないフォントで表示されることがあります。「Meiryoで統一して表示したい」と伝えるだけで、Claude Codeがフォントパス指定込みのコードに直してくれました。文法を覚えている必要はなく、「何がおかしいか」を言語化できれば直せる、という感覚です。

## 副業のAI動画制作で学んだプロンプト設計の考え方

動画生成（Veo）のプロンプト設計でも、Claude Codeでのやり取りでも共通して効いたのは「否定形で明示的に禁止する」書き方でした。

たとえば動画生成では、スタイル指定を何もしないと絵柄がアニメ風・コミック風に勝手に混ざることがありました。プロンプトの冒頭に「cartoon/anime/comic/CGIスタイルは禁止、実写として生成する」と否定形で明記すると安定します。AIに指示を出すときは「〜してほしい」という肯定形だけでなく、「〜はしないでほしい」を明示する方が再現性が上がる、というのはコード生成でもテキスト生成でも共通する感覚でした。

## まとめ

* ブランクのあるエンジニアでも、目的を具体的に言語化できればClaude Codeで実用的な自動化は組める
* 昔の文法知識より「何を実現したいか」「何がおかしいか」を言葉にする力の方が効いた
* AIへの指示は否定形での明示がどのAIツールでも共通して効果的だった

同じように情報系出身だけど今は現場からブランクがある人、あるいはこれからClaude Codeを触ってみたい人の参考になれば幸いです。もっと詳しい制作の裏側（Veoプロンプトの具体的なルール集、字幕合成の設定詳細）は別途まとめる予定なので、興味があればフォローしていただけると励みになります。
