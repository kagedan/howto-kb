# Claude Code / Gemini / Codex を役割ごとに使い分ける AIスクラムツール「vicara」v2.0.0 を公開しました

以前、AIスクラムツール `vicara` の紹介記事を書いたのですが、今回はその続きとして、`v2.0.0` で何が変わったのかを「実際の使う流れ」ベースで紹介します。

前回はどちらかというとプロダクトの思想や背景の話が中心でした。  
今回はもう少し実用寄りに、`企画 -> バックログ化 -> スプリント計画 -> 実装 -> レビュー` までがどうつながるのかを、画面つきでまとめます。

`vicara` は、ソロ開発者が複数の AI を「チーム」として扱いながら、スクラムっぽい流れで開発を進めるためのローカルファーストなデスクトップアプリです。

![](https://static.zenn.studio/user-upload/3a7520292302-20260411.png)

🔗 **GitHubリポジトリはこちら**  
<https://github.com/ytakahashi0302-ghb/ai_scrum_tool>  
（もしよければStarをいただけると開発の励みになります！）

## 3行でいうと

* `Claude Code CLI / Gemini CLI / Codex CLI` を、ロールごとに使い分けられるようになりました
* POアシスタント側も `API / CLI` を切り替えられ、`OpenAI / Ollama` を含めた選択肢が増えました
* `設定 -> 企画 -> 実装` の流れがかなり見えやすくなり、「試してみたいけど複雑そう」が減りました

## v2.0.0 の一番大きい変化

今回の `v2.0.0` で一番大きいのは、`Claude Code CLI` 前提だった実行基盤を作り直して、よりベンダーニュートラルな構成に寄せたことです。

開発エージェント側は `Claude Code CLI / Gemini CLI / Codex CLI` に対応し、POアシスタント側は `Anthropic / Gemini / OpenAI / Ollama` まで選べるようになりました。

つまり、「全部を同じ AI に任せる」ではなく、

* 実装の主担当は Claude
* UI まわりは Gemini
* 別ロールは Codex
* POアシスタントは API で OpenAI
* ローカル検証は Ollama

のように、役割ごとに選択しやすくなっています。

## まずはセットアップで迷いにくくなった

AIツール系で最初につまずきやすいのが、「結局何が足りてないのか分からない」問題だと思っています。

`vicara v2.0.0` では、設定画面にセットアップ状況タブを追加して、Git、各 CLI、API キー、ローカル LLM の準備状況をまとめて確認できるようにしました。

![](https://static.zenn.studio/user-upload/87d75e2c1493-20260411.png)

この画面があるだけで、

* Git は入っているか
* `Claude Code CLI / Gemini CLI / Codex CLI` は検出されているか
* API キーは設定済みか
* Ollama は動いているか

を一箇所で見られるので、導入の心理的ハードルがかなり下がったと思っています。

## 5ロールに別々の CLI を割り当てられるようになった

今回かなり気に入っているのが、チーム設定まわりです。

`Lead Engineer`、`Security & System Architect`、`UI/UX Designer`、`QA Engineer`、`PMO & Document Manager` の 5 ロールを持たせつつ、それぞれに使う CLI やモデルを割り当てられるようにしました。

![](https://static.zenn.studio/user-upload/6d1933f741ab-20260411.png)

これによって、「AIを1人使う」というより、「AIチームを編成する」感覚にかなり近づきました。

例えば、

* 設計や実装整理は Claude 系
* 画面系は Gemini 系
* ドキュメントや補助的な役割は別のモデル

のような運用がしやすくなっています。

個人的には、この「役割を持たせて使い分ける」体験が、ただの AI チャットツールとの差になりやすい部分だと思っています。

## POアシスタントも API / CLI を切り替えられるようになった

POアシスタント側も、今回かなり柔軟になりました。

これまでは API ベースの色が強かったのですが、`v2.0.0` では `API / CLI` の実行方式を切り替えられるようにしています。  
さらに、同じ設定画面から `Anthropic / Gemini / OpenAI / Ollama` などの選択肢も扱えるようにしました。

![](https://static.zenn.studio/user-upload/f4fd0ab195f2-20260411.png)

これで、

* クラウド API で素早く回したい
* CLI ベースでローカル寄りに使いたい
* OpenAI を使いたい
* Ollama でローカル LLM を試したい

といった運用の幅がかなり広がっています。  
ただし、Ollamaはまだ動作確認ができておらず…（OpenAIと同じ形式の認識なので問題ないとは思っておりますが…すいません！）

## 企画段階も、だいぶ入りやすくなった

`vicara` には `Inception Deck` があり、AI と対話しながら `PRODUCT_CONTEXT.md`、`ARCHITECTURE.md`、`Rule.md` を整えていけます。

今回の改善では、ここを「専門用語を知っている人向け」から「価値を一緒に言語化する会話」に寄せました。  
つまり、いきなりスクラム用語や設計用語をきれいに答えられなくても、プロダクトの輪郭を少しずつ固められるようにしています。

![](https://static.zenn.studio/user-upload/5e0316df9c75-20260411.png)

個人開発だと、実装に入る前の「何を作るかを言語化する工程」が意外と一番つらいことがあります。  
ここを AI と会話しながら前に進められるのは、かなり相性がいいと感じています。

## そのままバックログ化して、スプリントに積める

企画した内容は、POアシスタント経由でバックログに落とし込み、そのままスプリント計画につなげられます。

![](https://static.zenn.studio/user-upload/f0f9bcea923f-20260411.png)

この流れで意識しているのは、「アイデアを思いついたあとに、結局また別ツールへ転記する」手間を減らすことです。

* まず文脈を整理する
* バックログにする
* スプリントへ積む
* 実装担当に渡す

までが同じ UI の中でつながっていると、個人開発でもかなり進めやすくなります。

## 実装中の様子も見えやすくなった

スプリントが始まると、各ロールの Dev Agent がタスクを進めていきます。  
ターミナル Dock で実行ログを見ながら、どのタスクが進行中で、どれが `Review` に来ているかを追えるようになっています。

![](https://static.zenn.studio/user-upload/f9dfb42688a4-20260411.png)

この画面では、

* 進行中のタスク
* `Review` に入ったタスク
* プレビュー起動
* 承認してマージ
* 下部ターミナルでの実行ログ確認

までがつながっています。

`Git Worktree Review` 自体は前からある機能ですが、今回のバージョンでは「設定や役割分担の整理」とつながったことで、全体の流れとしてかなり説明しやすくなったと感じています。  
※Preview機能はついていますが、イマイチ動きが良くないので今後改修予定です。

## Analytics も追加したが、現時点では API 利用時の把握が中心

使用量や概算コストを見るための `Analytics` タブもあります。  
![](https://static.zenn.studio/user-upload/8c523fbd6fee-20260411.png)

これは便利ではあるのですが、現時点では特に `API` 利用時に意味がある機能だと思っています。  
CLI 実行分はツール側の仕様上、厳密な token 数やコストを取りづらいものがあり、どうしても参考値寄りになります。

なので今のところは、

* API でどのくらい使ったかを把握する
* スプリント単位でざっくり見る

用途で使うのが現実的です。

## 2026/04/11 時点では Windows 先行です

ここは正直に書いておきたいのですが、`v2.0.0` は現時点では Windows 先行です。  
リリース周りの安定性の都合で、一時的に Windows 中心の公開にしています。

ただ、そのぶんローカルで AI チーム開発を回す体験自体はかなり形になってきました。  
「AI を1人のチャット相手として使う」のではなく、「複数の AI に役割を持たせてチームとして動かす」感覚を試したい人には、だいぶ面白くなってきたと思っています。
