# AIリテラシー教育するためのアプリをClaudeCodeさまとVibes Codingを共にした2026年のGW

## はじめに

「Swift一行も書けないWebフロントエンジニアが、ClaudeCodeさまの力でiOSアプリをリリースした話」

そんな状態から始まり、2026年にNext.js製LPとSwift製iOSアプリを同時リリースするところまでたどり着きました。

この記事、プロダクトの話はほぼしません。**Claude Codeを使った開発体験**がメインです。「AIで開発ってどんな感じなの？」という方にさっと読んでもらえたら嬉しいです。

## Vibes Codingとは何か

2026年現在、AIコーディングは「補助ツール」のフェーズを完全に卒業しました。

**Vibes Coding** とは、コードの書き方を考えるのではなく「何を作りたいか」「なぜそうするか」を言語化してAIに委ね、自分は設計と意思決定に集中するスタイルです。

GitHub CopilotやCursorでも似た体験はできますが、Claude Codeの特徴はターミナル上でファイルの読み書き・コマンド実行まで一気通貫でやってくれる点です。「指示したら終わり、あとはClaudeさんに任せた」が成立します。

## 実際に機能した5つのプラクティス

### 1. プランニングファイルを軸にした運用

Claude Codeにはコンテキストウィンドウの制限があります。会話をまたぐと文脈が飛びます。

対策はシンプルで、実装前に `PLAN.md` を作って仕様・設計・決定事項を書いておく。実装後はそのファイルを更新して仕様書にする。これだけで「さっき決めたこと忘れてる問題」がほぼなくなりました。

### 2. 自分が書いたコードもAI向けに整える

自分でCSSを書いた後でも「このクラス名、AI向けに整理して」と一声かけるだけで、将来AIが参照しやすい構造に直してくれます。人間が書いた部分とAIが書く部分の橋渡しとして使える感覚です。

### 3. テンプレートを基準にブレを防ぐ

コンポーネントやAPIルートに標準テンプレートを用意して「これに倣って」と伝えると、スタイルの揺れが大幅に減ります。何も指定しないとAIはAIなりの書き方をするので、基準を渡すのが大事です。

### 4. デバッグは「2〜3回ダメならログを仕込む」

AIに任せてもループすることはあります。2〜3回で解決しない場合はいったん止まって、ログを追加してもらい、出力を渡しながらマイクロコントロールに切り替える。この判断を早くするだけでかなりストレスが減りました。

### 5. プロンプトは箇条書き＋知ってること全部渡す

「なんかうまくいかないんですが」は最悪のプロンプトです。「①現象 ②試したこと ③関係しそうなファイル ④期待する挙動」をセットで渡すと、出力の精度が全然違います。

## 強力なチームメンバー紹介

### 1. 上流工程：設計・企画 Gemini（発想力・ユーザー目線の共感・全体俯瞰）

正解が一つではない抽象的なテーマを一緒に膨らませていくフェーズで、Geminiの寄り添う姿勢と柔軟な発想力は最高の相棒になります。

### 2. 中流工程：実装・製造 Claude（圧倒的なコード精度・ロジックの美しさ）

余計な私情を挟まず、バグの少ないコードで書き上げてくれます。また、コンテキストウィンドウが広いため、既存のコードベースの文脈を崩さずにリファクタリングする能力がズバ抜けています。

### 3. 下流工程：検証・守り ChatGPT（厳密なレビュー・網羅的な法的チェック）

バグの検出や「規約に抵触していないか」「プライバシーポリシーの文面として法律的に穴がないか」といった、ルールに則った厳密な検証（ガバナンス）において、非常にソリッドで実用的なアドバイスをくれます。

## 生産性への影響

体感では**コーディング速度10倍以上**です。でも数字より大きかったのはメンタルの変化でした。

* 知らない言語（Swift）への恐怖がほぼなくなった
* 「この実装、どうすれば…」で詰まる時間が激減した
* コーディングだけでなく、設計の提案や技術選定の相談にも真剣に付き合ってくれる

「実装の難所」がもはや怖くない、という感覚は個人開発のQOLを爆上げしてくれます。

---

## エンジニアとして何が変わったか

一番変わったのは**何に時間を使うか**という感覚です。

コーディングをAIに委ねる前提に立つと、「どう書くか」より「なぜそう設計するか」が問われます。難しい書き方より、重複があっても読みやすいコードの方が正義。ファイル構成と命名が思った以上に生産性に直結することも実感しました。

「コードを書くエンジニア」から「設計と判断をするエンジニア」へのシフト——2026年はそれが現実になった年だと感じています。

## おまけ：App Store審査との戦い

Claude Codeとの開発体験は最高でしたが、その先に**Apple様**という最終ボスが待ち構えていました。

審査リジェクト、**5回**です。

「ゲストモードは通信してません、サーバーにデータ送ってません、本当です」と説明しても聞いてもらえず、結局Geminiへのデータ送信同意モーダルの実装を余儀なくされました。AIアプリの宿命ですね、はい。

さらに審査ステータスが動くのはカリフォルニア時間の朝イチ。日本時間に換算すると**早朝4時**です。「審査通ったかも」という期待で目が覚め、リジェクトのメールを読んで二度寝する日々を繰り返しました。

Claude Codeのおかげで実装は爆速でも、リリースの最後の関門は人間の審査員でした。

ここだけはAIに任せられない。
