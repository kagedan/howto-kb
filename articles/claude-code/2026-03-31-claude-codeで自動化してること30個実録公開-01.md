**Claude Codeで自動化していること30個【実録公開】**

## はじめに

自分がClaude Codeで実際に自動化ことをいること全部書き出してみたら、30個になった。

最初は「X投稿を楽にしたい」という動機だけだった。

それが気づいたら、情報収集・画像生成・動画生成・note執筆・競合分析・インフラ管理まで、ほぼ全部が自動で回る仕組みになっていた。

今この記事を書いている時点で、1日あたり推定4〜6時間の作業が自動化されている。

月換算で120〜180時間。

人件費換算で言えば、相当な金額になる。

全部公開する。コストも、失敗談も、「これは正直微妙だった」という話も。

  

## SNS運用系（13個）

![](https://assets.st-note.com/img/1774930210-laCTVFNxGApRwcBstSyuz9d3.jpg?width=1200)

SNS運用が一番しんどかった。

毎日投稿しようとすると、ネタ探し・文章作成・画像作成・スケジュール設定・効果測定と、やることが多すぎる。

これを全部自分でやると、軽く2〜3時間は飛ぶ。

それを全部Claude Codeに任せた。

今は朝起きたら投稿が予約されていて、勝手に配信されている状態になっている。

### ① X投稿を1日3〜5本自動生成（収集→採点→執筆）

仕組みはシンプルに3ステップ。

### Step1：収集

HackerNews・Reddit・Perplexityからその日のバズ記事・トレンドを自動収集する。

収集対象は「AI・テック・ビジネス」に絞っていて、エンゲージメント数や言及数でフィルタリングする。

### Step2：採点

収集したネタをGPT-4o-miniで採点する。

採点軸は「日本語ユーザーへの関連性・拡散されやすさ・教育的価値・タイムリー性」の4軸。

100点満点でスコアを出して、70点以上のネタだけを投稿候補に残す。

### Step3：執筆

採点を通過したネタをClaude 3.5 Sonnetに渡して、投稿文を生成する。

このとき「AI臭さ除去リスト」「バズ投稿の構造テンプレート」「過去のA/Bテスト結果」を同時に渡す。

生成された投稿文は再度品質チェックにかけて、問題なければ投稿キューに追加される。

> **削減時間：**1日あたり約2時間（ネタ探し60分＋執筆60分）  
> **使用ツール：** Claude 3.5 Sonnet、GPT-4o-mini、HackerNews API、Reddit API、Perplexity API  
> **コスト：**1投稿あたり約0.5〜1.5円

失敗談を正直に言うと、最初の1ヶ月は「採点基準が甘すぎて、微妙なネタが大量に通過する」問題があった。

採点プロンプトを5回以上書き直して、今のバージョンに落ち着いている。

### ② NanaBanana Proで投稿ごとに図解・インフォグラフィック画像を自動生成

テキストだけの投稿とビジュアル付き投稿では、エンゲージメントが体感で2〜3倍違う。

だから全投稿に画像をつけることにした。

ただし「画像を作る」という作業が一番時間を食う。

Canvaで毎回デザインするのは無理がある。

NanaBanana Proはテキストを渡すと自動でインフォグラフィック・図解・比較表などを生成してくれるAPI。

投稿文が確定したら、その内容をNanaBanana Proに渡して画像を生成し、投稿データと紐付けるところまで全自動化している。

> **削減時間：** 1投稿あたり15〜20分（月換算で15〜20時間）  
> **使用ツール：**NanaBanana Pro API  
> **コスト：**1画像あたり約3〜5円  
> **注意点：**NanaBanana Proはたまに日本語テキストの配置が崩れる。

だから次の項目が必要になった。

### ③ GPT-4o Visionで画像の文字化け・品質を自動チェック

NanaBanana Proで生成した画像を、GPT-4o Visionに渡して品質チェックする。

チェック項目は以下：

**・日本語テキストが正しく表示されているか**

**・文字がはみ出していないか**

**・背景と文字のコントラストが十分か**

**・全体的なデザインバランスが取れているか**

GPT-4o Visionがチェックして「NG」と判定した画像は自動で再生成をかける。

最大3回リトライして、それでもNGなら自分にアラートが来る仕組みになっている。

> **削減時間：**1日あたり30〜40分（手動チェックの排除）  
> **使用ツール：**GPT-4o Vision API  
> **コスト：**1画像チェックあたり約0.3〜0.8円

これを入れる前は、文字化けした画像がそのまま投稿されることが週2〜3回あった。今はほぼゼロになっている。

### ④ GPT-4o-miniで投稿前の品質スコアリング・炎上リスクチェック

投稿前に必ず通すフィルター。

**品質スコアリング：**

・ 読みやすさ（句読点・改行・文字数）

・ 具体性（数字・固有名詞があるか）

・ 価値提供度（読んだ人が何を得られるか）

・ 独自ている他の人も言ってることじゃないか）

**炎上リスクチェック：**

・ 特定の人・企業・国を過度に攻撃していないか

・ 根拠のない断言をしていないか

・ センシティブなトピックへの配慮が足りていないか

・ 著作権上の問題がある引用をしていないか

品質スコアが65点以下、または炎上リスクが「HIGH」と判定されると投稿キューから除外される。

> **削減時間：** 直接的な時間削減より「炎上リスク回避」が主目的  
> **使用ツール：** GPT-4o-mini API  
> **コスト：** 1投稿あたり約0.1〜0.3円

正直に言うと、過剰に炎上リスクを検出することがある。

「AIの可能性について論じた投稿」が「センシティブ」と判定されて除外されたことが何度もある。

チューニングが一番難しいパーツ。

### ⑤ HackerNews・Redditから海外バズ記事を自動収集

毎朝6時に起動するスクリプトが、HackerNewsとRedditから直近24時間のバズ記事を収集する。

**HackerNewsの収集条件：**

- スコア100以上

- コメント数30以上

- タグ：AI・machine-learning・startups・tech

**Redditの収集条件：**

- r/artificial・r/MachineLearning・r/startups・r/programming

- アップボート数500以上

- 直近24時間以内

収集した記事はタイトル・URL・要約・スコア・コメント傾向をセットにしてデータベースに保存される。

①の「採点」フェーズで使われる素材になる。

> **削減時間：**1日あたり45〜60分（手動リサーチの排除）  
> **使用ツール：** HackerNews API、Reddit API、Claude 3.5 Haiku（要約）  
> **コスト：**1日あたり約5〜10円

### ⑥ AI臭さ除去リライト（「様々な」「〜することが重要」を自動変換）

AIが生成した文章の弱点は「AI臭さ」。

読んだ瞬間に「これ、AIが書いたやつだ」とわかってしまう投稿は、エンゲージメントが著しく下がる。

だから「AI臭いフレーズ一覧」を作って、自動で変換するリライト処理を入れた。

**変換例（一部）：**

| Before | After |

|--------|--------|

| 様々な | いろんな / 複数の |

| 〜することが重要です | 〜しないと詰む |

| 〜と言えるでしょう | 〜だと思う |

| 非常に | めちゃくちゃ |

| 〜において | 〜では |

| 活用することで | 使えば |

| 〜を実現できます | 〜ができる |

| 効果的に | うまく |

単純な置換だけじゃなく、Claude 3.5 Sonnetに「人間らしく書き直して」というプロンプトも同時にかける。

2段階リライトで自然な文体に近づけている。

> **削減時間：** 工数削減より「品質向上」が目的  
> **使用ツール：** Claude 3.5 Sonnet  
> **コスト：** 1投稿あたり約0.5〜1円

### ⑦ 海外バズ記事を日本語で引用＋感想付きで投稿（発信者への誘導リンク付き）

海外のバズ記事を日本語ユーザー向けに「翻訳＋要約＋自分の感想」のセットで投稿するコンテンツ形式。

仕組みは単純で、⑤で収集した記事の中から「日本語ユーザーに刺さりそうなもの」をさらに絞り込んで、以下のフォーマットで投稿文を生成する。

**【海外で話題】**

（記事タイトルの日本語訳）

（2〜3行の要約）

感想：（自分目線の短いコメント）

詳細はこちら→（元記事URL）

引用元のリンクを必ず入れることで、オリジナルの発信者へのリスペクトを示しつつ、情報の信頼性も担保している。

> **削減時間：** 1投稿あたり20〜30分  
> **使用ツール：** Claude 3.5 Sonnet、Perplexity API  
> **コスト：** 1投稿あたり約1〜2円

### ⑧ バズ投稿のパターンを分析して別ネタで同じ構造を再生成

自分の過去投稿や他のアカウントのバズ投稿を分析して、「なぜバズったか」の構造を抽出する。

**抽出する要素：**

・冒頭の引きの形式（数字から始まる・問いかけ・断言・逆説）

・箇条書きの数と長さ

・句読点のリズム

・末尾のCTA（行動喚起）の有無と種類

・ハッシュタグの種類・数・位置

この構造を「テンプレート」として保存して、別のネタで同じ構造の投稿を生成する。

> **削減時間：**1日あたり30分（バズ要因の手動分析の排除）  
> **使用ツール：** Claude 3.5 Sonnet、X API  
> **コスト：**週あたり約20〜30円

正直、これは「バズを再現する」というより「一定品質の投稿を維持する」効果が大きい。

### ⑩ 引用投稿案を自動生成（毎朝レポートに追加）

⑨のレポートに、「本日引用すると良さそうな投稿候補」が3〜5件追加される。

自分のフォロー中のアカウントの直近投稿をスキャンして、  
  
「これに引用リプライすると議論が生まれそう」  
  
「補足情報を加えると価値が出る」

という投稿を抽出して、引用コメント案とセットで提示する仕組み。

引用投稿はフォロワーへのリーチを拡大するのに効果的なのに、

「何を引用するか考える」のが意外と時間がかかる。そこを自動化した。

> **削減時間：** 1日あたり20〜30分  
> **使用ツール：**X API、Claude 3.5 Sonnet  
> **コスト：** 1日あたり約5〜10円

### ⑪ 投稿完了時にTelegramへリアルタイム通知

スケジュール投稿が実際に配信されたタイミングで、Telegramにリアルタイム通知が来る。

> **削減時間：**直接的な削減より「安心感の担保」が主目的  
> **使用ツール：**Telegram Bot API  
> **コスト：** ほぼ無料

### ⑫ スケジュール投稿（12:00・17:00・21:00に自動配信）

X APIとcronを組み合わせて、1日3本を定時配信。

**時間帯の根拠：**

**・12:00（昼休み）：** ビジネスパーソンがスマホを見る時間

**・17:00（退勤前後）：** 業務終了後の情報収集タイム

**・21:00（夜）：** ゆっくりSNSを見る時間帯

投稿キューに入っている投稿を、設定時刻にX APIで自動投稿するだけ。

シンプルだが、「毎日定時に投稿する」という継続性が担保されるのが大きい。

> **削減時間：**1日あたり15〜20分（手動投稿の排除）  
> **使用ツール：** X API v2、cron  
> **コスト：** X API Basic Plan：月100ドル（これが一番高い）

X APIの料金が上がり続けているのが正直しんどい。ここだけは今後も頭痛の種になりそう。

### ⑬ A/Bスタイルテスト（ミニマル白／サイバーパンク紫をランダム切替）

NanaBanana Proで生成する画像を、2つのスタイルでランダムに切り替えて、どちらのデザインがエンゲージメントを高めるかをテストしている。

**スタイルA（ミニマル白）：**白背景・黒テキスト・シンプルなグラフ

**スタイルB（サイバーパンク紫）：**ダークバックグラウンド・紫アクセント・グロウエフェクト

各投稿のスタイルを記録しておいて、週次でエンゲージメント率を比較する。

現時点では「サイバーパンク紫のほうがインプレッションが高く、ミニマル白のほうがフォロワー増加数に寄与する」という傾向が出ている。

**削減時間：** A/Bテスト設計・集計の手間を削減

**使用ツール：** NanaBanana Pro API、データベース（SQLite）

**コスト：**スタイルB（画像生成）：1画像あたり約5〜7円

### note・コンテンツ系（7個）

![](https://assets.st-note.com/img/1774931775-zoWtdhFnfr5Uqm8RuZPHSwsA.jpg?width=1200)

noteは「毎日X投稿してるなら、その内容を深掘りすればいい」という発想で自動化した。

X投稿は140文字という制約があって伝えられる情報量に限界がある。

noteではその投稿の背景・詳細・事例・考察を8,000文字以上で展開する。

素材はすでにX投稿として存在しているから、ゼロから考える必要がない。

### ⑭ 毎夜22時に投稿済みXをベースにnote記事を自動生成（8,000文字以上）

毎夜22時に起動するスクリプトが、その日の投稿済みX投稿をすべて取得して、記事のたたき台を生成する。

**プロセス：**

**1. その日の投稿3〜5本を取得**

**2. 共通テーマを抽出して「記事タイトル」を生成**

**3. 章構成を設計（序論・各投稿の深掘り・まとめ）**

**4. 各章を2,000〜3,000文字で展開**

**5. 全体を8,000〜12,000文字にまとめる**

プロンプトに

「X投稿で言及した内容を100倍詳しく説明する」

「具体的な手順・数字・事例を入れる」

「自分が実際に試した話を必ず入れる」

を入れることで、薄い記事にならないようにしている。

> **削減時間：** 1記事あたり4〜6時間（企画・執筆・編集の合計）  
> **使用ツール：**Claude 3.5 Sonnet（長文生成）  
> **コスト：**1記事あたり約15〜25円

**失敗談：**最初は「8,000文字以上」という指定をするだけだったが、Claude が冗長な文章で文字数を稼ぐことがあった。

「具体性のある情報密度の高い文章で」という制約を加えてから品質が上がった。

### ⑮ 各章ごとの画像をNanaBanana Proで自動生成（サムネ＋章画像＋まとめ）

note記事の各章に合った画像を自動生成する。

生成する画像の種類：

**・サムネイル：** 記事タイトルと視覚的インパクトのある背景

**・章画像：** 各章のキーポイントをビジュアル化（フロー図・比較表・インフォグラフィック）

**・まとめ画像：** 記事全体の要点を1枚にまとめた画像

1記事あたり平均5〜8枚の画像が自動生成される。

手動でCanvaを使っていたときと比べると、画像生成だけで2〜3時間の差がある。

> **削減時間：** 1記事あたり2〜3時間  
> **使用ツール：**NanaBanana Pro API  
> **コスト：** 1記事あたり約20〜40円

### ⑯ noteに下書きを自動保存（Selenium経由）

生成された記事と画像を、Seleniumを使ってnoteの下書きに自動保存する。

Seleniumはブラウザ操作を自動化するライブラリ。

noteのAPIが公開されていないため、実際のブラウザ操作をスクリプトで模倣して下書きを作成している。

具体的には：

1. noteにログイン

2. 新規記事作成画面を開く

3. タイトルを入力

4. 本文を貼り付け

5. 画像を各章の適切な位置に挿入

6. ハッシュタグを設定

7. 下書き保存

自分が確認して「公開ボタンを押す」だけの状態にする。

完全自動公開は「万が一の誤爆」が怖いので手動公開にしている。

画像や文章をチェックして、間違っている場合は手作業で修正もする。

> **削減時間：** 1記事あたり20〜30分（下書き作成・画像挿入）  
> **使用ツール：** Selenium、Python  
> **コスト：**ほぼ無料

**失敗談：**noteのUI更新でSeleniumのセレクタが壊れることが月1回程度ある。そのたびにスクリプトを修正する必要があるのが地味に面倒。

### ⑰ 円グラフ・フロー図・比較表などの図解を投稿内容に合わせて自動生成

記事の内容に応じて、適切な図解タイプを自動で選択・生成する。

タイプ分類の例：

・割合・構成比 → 円グラフ

・時系列の変化 → 折れ線グラフ

・複数要素の比較 → 棒グラフ・比較表

・プロセス・手順 → フロー図

・関係性 → マインドマップ

記事テキストをClaudeに渡して

「この内容を可視化するとしたら何の図解が適切か」を判断させて、

その判断結果をNanaBanana ProのAPIに渡す。

Claudeが「フロー図が適切」と判断したらフロー図を、

「比較表が適切」と判断したら比較表を生成する。

> **削減時間：** 1記事あたり40〜60分  
> **使用ツール：** Claude 3.5 Sonnet（図解タイプ判断）、NanaBanana Pro API（生成）  
> **コスト：** 1記事あたり約10〜20円

### ⑱ 記事タイトルから各章のテーマを抽出してchapterごとに専用画像を生成

記事タイトルが決まったら、そのタイトルだけをClaudeに渡して

「最適な章構成」を提案させる。

入力：「Claude Codeで自動化してること30個【実録公開】」

出力：

Chapter1：なぜClaude Codeを選んだのか

Chapter2：SNS運用系（13個）

Chapter3：note・コンテンツ系（7個）

...

この章構成から、各章のテーマに合った画像のラフ案  
（テーマ・使用する図解の種類・強調すべきキーワード）  
を生成して、NanaBanana Proに渡す。

> 削減時間：1記事あたり30〜40分  
> 使用ツール： Claude 3.5 Sonnet、NanaBanana Pro API  
> コスト： 1記事あたり約5〜10円

### ⑲ 投稿テキストをnoteの4,000〜18,000文字記事に自動拡張

これは「文字数コントロール」の機能。

noteの記事は長すぎても短すぎても読まれにくい。

4,000〜18,000文字という幅を持たせているのは、トピックの深さによって最適な文字数が変わるから。

「AI系のHow-to記事」は詳しく書くほど価値が高いので12,000〜18,000文字を狙う。

「まとめ系・リスト系」は4,000〜8,000文字でテンポよく読める形にする。

Claudeに投稿テキストを渡すとき、  
「この内容なら何文字の記事が最適か」  
を先に判断させて、その文字数を目標に執筆させる。

> **削減時間：**直接的な時間削減より「品質の安定」が目的  
> **使用ツール：**Claude 3.5 Sonnet  
> **コスト：** 文字数に応じて変動（8,000文字で約10〜15円）

### ⑳ note記事のハッシュタグを自動生成

記事が完成したら、その内容に最適なハッシュタグをClaudeが自動生成する。

**選定基準：**

・ noteで検索されやすいハッシュタグ

・ 記事の内容に本当に関連しているか

・ 特定のコミュニティ（AI・起業・副業など）のタグ

・ 投稿数が多すぎず少なすぎないタグ

最大10個まで生成して、自分が最終確認して使用するものを選ぶ（ここだけ手動）。

ハッシュタグの選定に時間をかけても読者数への影響は限定的なので、80点のものを自動で出してもらう形にしている。

**削減時間：** 1記事あたり10〜15分

**使用ツール：** Claude 3.5 Haiku（軽量タスクなので）

**コスト：** 1記事あたり約1〜2円

### ㉖ 2台のMac（サブ1・サブ2）をSSH経由でリモート管理

メインMacから2台のサブMacをSSH経由でリモート操作している。

構成：

**・メインMac：** 指示の送受信・監視・自分が直接操作する

**・サブMac1：** X投稿生成・画像生成・SNS関連の処理を担当

**・サブMac2：** note記事生成・情報収集・データ分析を担当

Claude Codeで各Macへの指示をスクリプト化しているので、

「サブ1で画像生成が終わったらサブ2に完成ファイルを転送する」

という連携も自動化できている。

**失敗談：**

「メインでは動くのにサブでは動かない」  
  
という環境差異問題が頻発した。

.envファイルとライブラリバージョンの統一が重要。

## まとめ・今日から始めるためのステップ

![](https://assets.st-note.com/img/1774936359-3yxNdCWzmtsIRofJMwLrZ8aP.jpg?width=1200)

ここまで読んでくれた人に向けて、「自分が最初からやり直すとしたら何から始めるか」を正直に書く。

**自動化で変わったこと**

**時間：**

・X投稿関連：1日2〜3時間 → 確認のみ10〜15分

・note記事：1記事5〜7時間 → 最終確認・公開のみ20〜30分

・情報収集：1日45〜60分 → レポート確認のみ5〜10分

**合計：1日4〜6時間の削減**

**品質：**

毎日投稿することで、投稿ペースが安定した。  
手動でやっていたときは「今日はネタがない」「疲れた」で更新が止まることがあった。  
今は365日安定して投稿できる。

**精神的負担：**

「今日の投稿どうしよう」「ネタがない」「また更新できなかった」  
というストレスがほぼなくなった。

これは数字では測れないが、実感としては一番大きな変化かもしれない。

### 今日から始めるとしたら？この3ステップ

**Step1：まずClaude Codeを使ってX投稿を1本生成してみる（今日）**

難しく考えなくていい。「今日気になったニュースを要約して、140文字のX投稿を作って」とClaudeに頼むだけでいい。これが自動化の入口。

### Step2：cronで毎朝自動実行する仕組みを作る（1週間以内）

Step1で作った「1投稿生成するスクリプト」を、cronで毎朝自動実行するように設定する。これだけで「毎日の投稿ネタ出し」が自動化される。

### Step3：Telegramで通知を受け取れるようにする（2週間以内）

Telegram Botを作って、自動処理の完了通知・エラー通知を受け取る。

これがあると「ちゃんと動いてるか」の確認がスマホだけでできる。  
監視コストが激減する。

### 失敗を避けるための3つの鉄則

**鉄則1：完全自動化の前に「半自動化」を挟む**

いきなり全自動投稿にするのは危険。まずは「生成だけ自動・投稿は手動」にして、品質に慣れてから自動投稿に移行する。

**鉄則2：プロンプトは固まってから自動化に組み込む**

「なんとなく動くプロンプト」で自動化すると、コストが爆発する。  
まず手動で10〜20回試して「これで行ける」という確信が持てたプロンプトを使う。

**鉄則3：コスト上限を必ず設定する**

Anthropic・OpenAIのAPIには月間コスト上限の設定機能がある。  
最初は「月5,000円上限」に設定して、使い方に慣れてから上げる。  
上限なしで使い始めると、バグで無限ループが起きたときに請求が爆発する（経験談）。

### 最後に

Claude Codeは「AIエンジニアだけのツール」じゃない。

自分もプログラミングの専門家ではない。  
「こういう自動化がしたい」というイメージをClaudeに伝えながら、一緒にコードを作っていく感覚で30個の自動化を実現できた。

大事なのは「何を自動化したいか」を明確にすること。  
「なんとなく自動化したい」ではなく  
「毎日のX投稿作成に2時間かかっていて、これを15分にしたい」という具体的なゴールを持つこと。

その具体的なゴールさえあれば、Claude Codeは思っている10倍速く解決策を出してくれる。

まずは1個から始めよう。
