まず質問です。

以下の6つを2つのグループに分けるとしたら、あなたはどう分けますか？

**ChatGPT / Claude / Gemini / Claude Code / Codex / Gemini CLI**

・・・

「チャットAI」と「ローカルエージェント」に分けられます。

- チャットAI　ChatGPT / Claude / Gemini
- ローカルエージェント　Claude Code / Codex / Gemini CLI
この2つの違いは何でしょうか？

「ファイルを直接編集できるかどうか」という違いもありますが、それは表面的な話です。

本質的な違いは別にあります。

## 

## 「Claude Code ってすごい」の正体

よくXでも Claude Code や Codex を絶賛する声をよく聞きます。

でも「何がすごいのか」を説明できる人は少ない。

「モデルが賢いから」と思っていませんか？違います。

## 

## 違いは「コンテキスト」

コンテキスト = AI が作業するときに「見ている情報」のことです。

チャットAI にコードを貼って使っていたとき、こんな経験をしました。

バグを直してもらおうとしたが、どのファイルをコピペすればいいか分からない。関係ありそうなファイルを3つ貼ったら、今度は回答がプロジェクトの他の部分と噛み合っていない。

AI が悪いわけではありませんでした。渡した情報が断片的だっただけです。

Claude Code を初めて使ったとき、最初に驚いたのはここでした。

他のファイルとの関係を踏まえた上で回答してくれている。ターミナルのエラーを自分で見て、自分で修正しようとしている。

AI が「プロジェクト全体を見ている」状態で作業している感覚でした。

## 

## Anthropic が言う「コンテキストエンジニアリング」

少し前ですが、Anthropic はこんな考え方を発表しました。

> AI が正確に仕事を進めるために必要なのは、最小限の高シグナルな情報を渡すこと

さらに「コンテキストロット」という概念も提唱しています。

コンテキストロット = コンテキストが長くなるほど、モデルの情報想起精度が下がる現象です。

つまり、情報は「多ければいい」のではなく「必要なものだけを正しいタイミングで渡す」ことが重要。

ローカルエージェントはこれを自動でやっています。必要なファイルだけ参照し、不要な情報は捨てる。コピペしなくていい理由はここにあります。

## 

## チャットAI が「使えない」わけではない

誤解しないでほしいのは、チャットAI が劣っているわけではないということです。

チャットAI でもコンテキストをうまく渡せば、精度は上がります。「どのファイルを貼るか」「何を説明するか」を工夫することで、出力の質は変わります。

「プロンプトの書き方が大事」と言われていたのは、実はコンテキストの話だったわけです。

ローカルエージェントはその手間を自動化しているだけ。

## 

## まとめ

- チャットAI ＝ あなたが手で情報を渡す
- ローカルエージェント ＝ 自分でプロジェクトを丸ごと把握して作業する
- 差はモデルの賢さではなく、コンテキストの量と質
Claude Code や Codexが「すごい」のは、モデルが賢いからではありません。正しい情報を正しいタイミングで自分で集めているから、精度が高いんです。
