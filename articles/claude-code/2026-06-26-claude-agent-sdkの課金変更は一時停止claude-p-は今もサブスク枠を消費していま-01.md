結論から書きます。**6月15日に予告されていたClaudeの課金変更は、施行されないまま一時停止されました。** いま現在、何も変わっていません。`claude -p`、Claude Agent SDK、GitHub Actions経由の実行、Agent SDKでサブスク認証する外部アプリ。これらは以前と同じく、サブスクリプションの使用枠から引かれています。請求できるクレジットも、いまは存在しません。

公式ヘルプセンターの該当ページにも、冒頭ではっきり書かれています。「現在のところ、何も変わっていません」と。

:::note alert
2026年6月時点では、Agent SDK専用クレジットは配布されていません。`claude -p` やGitHub Actions経由の利用も、これまで通りサブスク枠から消費されます。下のクレジット表は「再開したらこうなる」という予告の内容で、現時点では効力がありません。
:::

お伝えしたいのは3つです。予告された変更の中身、それがいまは止まっていること、止まっている今のうちにやると後で楽になる「棚卸し」。順に書きます。

## 今やるなら、これだけでいい

将来クレジット制が再導入されたとき、慌てないための準備が一つあります。**非対話でのClaude利用を、いまのうちに棚卸ししておくことです。**

書き出す対象は3種類。cronで叩いている `claude -p` のスクリプト。GitHub Actionsの中でClaudeを呼んでいるワークフロー。Agent SDK経由でサブスクのアカウントにつないでいる外部ツール。クレジット制が来たとき、この3つが「サブスク枠とは別枠」に切り出される対象になります。

自分の場合、記事の下書きはローカルのClaude Codeで書いて、投稿だけGitHub Actionsに任せています。だから「Claudeを非対話で叩く自動化はそんなにないはず」と思っていました。ところが、頭の中だけで「これは該当、これは違う」を分けようとすると、境界が意外と曖昧になります。書き出して初めて見えてくる。

もう一つ、いまのうちにやれること。**重い自動化はAPIキーに寄せる設計を、先に用意しておく。** 対話はサブスクの定額に置いたまま、夜間バッチのような重い処理だけをAPIキーの従量に分けます。こうしておくと、課金体系が今後どちらに転んでも効きます。クレジット制が再開しても、APIキー側へ逃がした処理は最初から別勘定だからです。

あとは公式ページをウォッチ対象に入れておく。再実施のときは事前に告知すると書かれています。告知が来てから動けば間に合います。

## 予告されていた中身（参考）

ここからは「いつか来るかもしれない」話です。止まっている今は効力がありませんが、仕組みを知っておくと再開時に読み直さずに済みます。

予告では、`claude -p` とAgent SDKの利用がサブスクの使用枠からカウントされなくなる予定でした。その代わりに、Pro、Max、Team、Enterpriseの対象ユーザーへ、Agent SDK専用の月額クレジットが配られます。プランごとの金額はこうでした。

| プラン | 月額クレジット |
|---|---|
| Pro | $20 |
| Max 5x | $100 |
| Max 20x | $200 |
| Team(標準シート) | $20 |
| Team(プレミアムシート) | $100 |
| Enterprise(使用量ベース) | $20 |
| Enterprise(シートベースのプレミアムシート) | $200 |

シートベースEnterpriseの標準シートは対象外。APIキーで動かしているClaude Platformアカウントもクレジットはもらえず、従量課金がそのまま続きます。

クレジットが効くのは、自分のプロジェクトで動かすAgent SDK、`claude -p` の非対話モード、Claude Code GitHub Actions、Agent SDK経由でサブスク認証する外部アプリ。逆に効かないのが、ターミナルやIDEでの対話的なClaude Code、Web・デスクトップ・モバイルのチャット、Claude Cowork。ここは引き続きサブスク枠のままです。

仕組みで注意したいのは2点。クレジットはユーザーごとで、チームでプールできません。そして毎月リセット、繰り越しなし。使い切ったあとは、使用クレジットを有効にしている人なら標準APIレートでそちらへ流れます。有効にしていない人は、翌月クレジットが更新されるまでAgent SDKのリクエストが止まります。月末に夜間バッチが静かに止まる、という事故はここで起こり得ます。

## 個人的に引っかかったところ

この変更で一番ざわっとしたのは、金額ではありません。「非対話のClaude利用が、対話とは別の課金面として切り出される」という設計の考え方のほうです。

これまでは、サブスクに入っていれば対話も自動化も同じ枠で考えていられました。それが「対話はサブスク、自動化は別クレジット」に分かれると、自分が普段どれだけClaudeを非対話で動かしているかを、初めて意識させられます。普段は見えていない。GitHub Actionsの中で静かに回っているものは、ログを見にいかないと存在を忘れます。

一度止まったとはいえ、Anthropicは「プランを更新するために取り組んでいる」と明言しています。いつか何らかの形で分離される見込みは高い、と自分は見ています。そのとき効いてくるのが、今日のうちに書き出した棚卸しのメモです。

## 次の自分に渡すメモ

- 6/15の変更は一時停止。いまは全部サブスク枠で動いていて、請求できるクレジットもない
- やると後が楽なのは、非対話利用(cron / GitHub Actions / 外部アプリ)の棚卸し
- 重い自動化はAPIキーに逃がす設計を先に用意しておくと、どちらに転んでも効く
- 再開は事前告知される。告知が来てから動けば間に合う

止まっているうちに手を動かしておくと、再開のメールが届いた日に焦らずに済みます。自分はまず、非対話で動いている自動化のリストを作るところから始めます。数値はそのあとで足せばいい。
