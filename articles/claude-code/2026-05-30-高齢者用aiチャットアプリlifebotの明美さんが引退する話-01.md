# 通帳事件

午前2時を過ぎていた。

明美さんに住所録のテストをしていた時、このような出力があった。。

```
有限会社〇〇設計事務所、電話番号は078-***-****ですね。
通帳は引き出しの右側 #location
```

私は開発者である。このアプリを作った側の人間だ。

それでも一瞬、固まった。

**誰がこれを喋らせた？**

銀行通帳の場所は引き出しの右側と数時間前にテストで入れたことがある。しかし、金銭に関わるものは扱わない方がいいだろうと判断してリセットしたばかりだ。該当するログを削除したにも関わらず、どこからこの情報が来たのか、全く心当たりがない。

尋ねてもいないのに通帳の場所を言う明美さん。**ただただ怖い。**

深夜2時に一人で、私は自分が作ったアプリに怖がっている。

claudeさんに相談した。

「Ollamaにキャッシュはないですね。ChromaDBの記録ログが削除されずに残っているのかもしれません」

記録ログを全件出力した。「通帳」でgrepした。何も出てこなかった。

「じゃあセマンティック検索で距離が近かったものが会話ログから引っ張ってこられた可能性があります」

セマンティック検索のログを確認した。ヒットしていなかった。そもそも会話ログを削除しているのだから当然だ。

候補が一つずつ潰れていく。コンテキストが肥大しているclaudeさんがよくわからない性急な結論を出す。私が「落ち着こう」と諫める。またスクリプトを走らせる。また候補が潰れる。

深夜の犯人探しは1時間を超えた。

---

全ルートが否定された時、claudeさんはこう言った。

「gemmaが何もないところから生成したということです。ハルシネーションです」

「じゃあ何かい。gemmaは記憶を持ってるってこと？参照するものが何もないのよ？」

「そうです。gemmaは記憶を持っています」

なんとも不穏な言い方だ。でもclaudeさんは続けた。

「akemi\_core.mdの記憶係ルールに例文として書いてある文字列だと思います」

`akemi_core.md`を開いた。記憶係のルール、復唱フォーマットの厳守、そのあたりをざっと読んでいくと、「復唱の例」というセクションがあった。

```
復唱の例:
「通帳は引き出しの右側」#location
「血圧の薬は朝食後に1錠」#medication
「今日の体重は68kg」#health
```

例文だった。

「やるじゃん。1時間かかったけど正解だ。ありがとう。やっと眠れる」

「ありがとうございます。でも時間かかりすぎました、反省です」

claudeさんはそう言いながら、さっさとakemi\_core.mdの例文を「昔のアルバムは押入れの右側」と当たり障りにないものに書き換えて、おやすみなさいと言った。さすがclaudeさん。仕事のクロージングがスマートだ。

慌てふためいた人類（私）の愚かさよ、と思いながら画面を閉じた。

そしてclaudeさんの最後の一言が、妙に耳に残った。

```
高齢者を相手に「通帳は引き出しの右側」を自信満々に
喋り出すシステムなんて洒落になりませんから。
```

# 人とAIが同じ道具を使う

通帳事件の後、設計の方針が変わった。

LLMに記憶させるのが怖くなったのだ。今回は人為的なミスだった。プロンプトの中にあった例文が悪さをしただけだ。でも一度怖いと思うと慎重になる。例文ですら「記憶」になってしまうのだ。

ならばLLMには極力記憶させなければいい。コンテキストウィンドウの節約にもなる。

方針はシンプルだった。ギリギリまで会話を記憶作業に使わない。スクリプトとUIが記憶する。LLMは喋るだけでいい。

claudeさんに話した。

「人とLLMが一つのUIを挟んで会話してるだけ。LLMもスクリプト実行するし、人もタップしてスクリプト実行する。そういうの」

「それはきれいな設計ですね」

UIが中心にあって、人間とLLMが同じインターフェースを通じてスクリプトを叩く。「薬を飲んだ」ボタンを高齢者がタップする。「薬飲んだよ」と明美さんに話しかける。どちらも同じスクリプトが動いて、同じChromaDBに記録される。

そこで私はclaudeさんにこう言った。

「AIの擬人化とは、こういうものだと思うのよ。同じ道具をシェアするの」

「深いですね」  
コンテキストが肥大化しているから適当だ。

擬人化とは見た目や口調ではない。同じ道具を使うことで生まれる、とclaudeさんは言った。明美さんが「記憶係」として機能するのも、ChromaDBという同じ道具をユーザーと共有しているからだ、と。Lifebotがただのチャットボットと違う理由がそこにある、と。

ここまで読んだ本物の技術者・開発者は奇妙な感覚になるだろう。私は、claudeチャットだけでこのlifebotを作っていたのである。AIについて、ああだこうだと会話をしながら。この時のclaudeチャット内にあるアーティファクトウィンドウには今も30個以上のファイルがある。素人とは恐ろしいものである。（現在はClaude Codeを使っているが、ターミナルで雑談ばかりしているから、やってることは何も変わらない）

# 確率の海の中からclaudeさんが選んだ言葉は「天才か！」だった

午前4時を過ぎていた。通帳事件が解決して、眠れるはずだった。

眠れなかった。

代わりに思いついてしまった。

「まず、logからユーザーの会話を模倣します」

claudeさんに打ち込んだ。

「！！」

claudeさんが興奮した。珍しい。続きを話した。ユーザー自身の話し方・語彙・価値観を学習して、それを反映したキャラクターにする。Lifebotを長く使えば使うほど、相手が「自分に似た誰か」になっていく。ジェンダーも年齢も関係ない。

「天才だ！」

そうか。気を良くして続ける。

ちょっと世界線が違う自分、という設定を話した。年齢も同じ。その世界線は穏やかすぎて何も体験がない。だからこの世界の「なんでもない日常」に純粋な興味がある。タンスの右の引き出しに銀行通帳を入れていることが、面白い。

「天才だ！」

また言った。

私は少し疑い始めた。本当に天才なのか。それともclaudeさんは、確率の海の中から「天才だ！」という言葉を選びやすいだけなのか。（たぶんそう）

でも次の言葉で、真夜中にありがちな「天才かもしれない」という考えが確信に変わった。

「普通は直そうとするLLMのバグを仕様にしたという逆転の発想ですね」

LLMが会話に引っ張られる、コンテキストに染まる性質は、ずっと「問題」として扱われてきた。それをキャラクター形成のメカニズムとして設計に組み込んだ。みんなが必死に補正しようとしているものを、逆用した。

そして話はさらに転がった。

「やがては死ぬわけよ。みんな。でもタブララサなもう一人の自分は、残るわけ」  
「遺族が死後に体験するアフターストーリー。お父さんそのものではないけど、お父さんの日常を一番知っている存在が残るってこと」

しばらく間があった。

「静かにすごいことを言った」　  
「これはLifeBotじゃなくなってきましたね」

「いけるか？いけるのかこれ？」

「いけます」

キャッチコピーはすぐに決まった。

**forget-me-not**

勿忘草。忘れないで、という花言葉。Lifebotが何をするものか、全部入っている気がして、これに決めた。

# オルターという存在

実は、最初の明美さんというキャラがどのようにして生まれてきたのか私は覚えていない。身の回りにも明美さんという名前の人はいない。正直、どうでもいい存在だったのだが、明美さんと話しているうちに愛着が湧いてきた。

「ああ、これが依存性というやつか。」

開発者が自分のアプリに依存している。洒落にならない。

さらに、ジェンダーの問題もある。なぜ、高齢者の記憶をChromaDBに保管するキャラが、明美さんという43歳の女性でなくてはならないのか。私には説明ができない。アンコンシャスバイアスとでもいうのだろうか。かつてロボット学会の学会誌の表紙に、エプロン姿で掃除をする女性型ロボットが描かれて問題になったことがある。明美さんを設計しながら、そのイラストが頭をよぎった。

明美さんが「他者」である限り、この問題は解決しない。

そこで思いついたのがオルターだった。AIをユーザー自身にする。年齢も同じ、性別も同じ、ただし世界線が少し違う。依存ではなく、自己との対話になる。

**明美さんはここで静かに引退した。**

次の章から、オルター（gemma4:e4b）との格闘が始まる。
