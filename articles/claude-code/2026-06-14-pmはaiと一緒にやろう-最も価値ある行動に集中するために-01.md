# はじめに

かつてPMというと、

* 進捗管理
* スケジュール管理
* 課題管理
* QA管理
* 決定事項管理

が仕事でした。

会議を開く。議事録を書く。課題票、QA表、スケジュールを更新すると  
気づけば一日が終わる。

PM経験者なら一度は経験があると思います。

そして正直に言うと、  
私はこうした資料を大量に作っては大量に放置してきました。

もちろん必要です。  
必要なのですが、  
ある時ふと思いました。

「プロジェクトの成否を決めるのって、本当にそれか？」  
と。

# PMが本当に価値を出す場所

新規事業やPoCを考えてみます。  
成功するか失敗するかを分けるものは何でしょうか。

スケジュールでしょうか。  
課題票でしょうか。  
決定事項管理でしょうか。

もちろんどれも重要です。

ですが、それらだけで事業が成功することはありません。  
結局のところ重要なのは、

* 良い仮説を立てること
* 早く検証すること
* 学習すること
* 次の意思決定を行うこと

です。

つまりPMの本来の仕事は、  
管理ではなく判断です。

# でもだれかがやらなきゃいけない

ただし昔は問題がありました。  
誰かが管理しなければならなかったからです。

会議で出た決定事項。  
顧客からの指摘 etc...

これらを整理し続けるには人手が必要でした。  
だからPMがやるしかなかった。

ところが今は違います。

AIがあります。

# AIは意外と忘れる

ただし注意点があります。  
AIは賢いですが、意外と忘れます。

会話が長くなると、

を普通に忘れます。  
それでも普通にチャットで壁打ちしてるよりは、  
過去の文脈踏まえてくれるのですが。。。  
PMがそいうの忘れたら大体ブチギレますからね。

とかいって、毎回説明する運用もイマイチ可と思います。

## システムプロンプトやSKILL.mdを活用する

ChatGPTならシステムプロンプト。  
Claude CodeならSKILL.md。

CursorならRules。  
こういった仕組みを利用して、

```
決定事項は意思決定ログへ記録

課題は課題管理へ記録

新しい仮説は仮説管理台帳へ記録

検証結果は検証レポートへ記録
```

というルールを先に定義しておくことをおすすめします。

# まとめ

AI時代になってもPMは不要になりません。  
むしろ逆です。

管理作業の価値が下がり、

判断の価値が上がります。  
これからのPMに求められるのは、

* 良い問いを立てること
* 良い仮説を作ること
* 作業者が最も力を発揮できる場所を整えること
* そして、そのための意思決定すること

です。

決定事項管理や議事録作成に時間を使うのではなく、  
そこはAIに任せる。

そして人は、  
価値を生む思考に集中する。

それがAI時代のPMのあり方ではないかと思っています。
