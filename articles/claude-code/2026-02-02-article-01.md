Claude Code's creator shared how his team uses it.

I broke it down and made a cheatsheet to help you:
 
1️⃣ Run multiple sessions in parallel

This is the team's #1 unlock 🔥

→ Open 3-5 terminal windows
→ Each runs its own Claude session
→ Work on different tasks simultaneously

Set up shortcuts (za, zb, zc) to hop between them instantly.

Super helpful for getting multiple versions of something all at once.

2️⃣ Start every big task in Plan Mode

Don't let Claude just start building.

Say: "Let's plan this first."

Plan right → Claude nails it in one shot.

3️⃣ Build your CLAUDE .md file

This file teaches Claude YOUR preferences.

Every time Claude makes a mistake, say:

"Update your CLAUDE .md so you don't make that mistake again."

Claude is eerily good at writing rules for itself 🤯

4️⃣ Create custom slash commands

Do something twice a day? Automate it.

Example: /techdebt scans for duplicated code every session.

Save to git → reuse across all projects, share with your team

5️⃣ Let Claude fix bugs for you

→ Connect Slack (or paste a bug report)
→ Say: "fix"
→ That's it

Zero context switching. It's pretty smart now.

6️⃣ Level up your prompting 💬

Make Claude YOUR reviewer:

"Grill me on these changes. Don't let me PR until I pass."

After a mediocre fix:

"Scrap this. Implement the elegant solution."

7️⃣ Optimize your setup

→ Use /statusline to see context usage
→ Color-code terminal tabs (one per task)
→ Use voice dictation (fn+fn on Mac)

You speak 3x faster than you type.

I personally use WhisprFlow

8️⃣ Use subagents for hard problems

Add "use subagents" to any request.

Claude spins up helper agents for different parts.

Keeps your main conversation clean and gets more effort on the problem.

9️⃣ Use Claude for data & analytics 📊

Connect Claude to your database CLI.

Ask questions in plain English.
Get answers without writing SQL.

Boris hasn't written SQL in 6+ months.

🔟 Learn while you work 🧠

In /config → enable "Learning" output style.
