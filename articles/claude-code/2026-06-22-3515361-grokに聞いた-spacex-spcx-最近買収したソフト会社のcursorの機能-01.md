Cursorの機能をもっと詳しく（実践的な使い方例中心）
前回の説明を踏まえ、特に人気の高い機能についてより具体的な動作フロー・使い方例・Tipsを追加で詳しく解説します。2026年現在のCursor 2.5/3.0系を基準としています。

1. Composer 2.5（最強の多ファイルエージェント）
•  動作の流れ（詳細）:
1.  Cmd + Shift + I でComposerを開く。
2.  自然言語で指示（例: 「Reactアプリにユーザー認証（Clerk統合）を追加して、ログイン画面も作って。テストも書いて」）。
3.  AIが計画ドラフトを作成（ステップバイステップで表示）。
4.  ユーザーが承認/修正 → AIが自動で複数ファイル編集（新規作成含む）。

5.  ターミナルでnpm testなどを自動実行 → エラーあれば修正を繰り返す。
•  先進ポイント: 並列エージェント対応（複数のサブタスクを同時に処理）。クラウドサンドボックスで長時間タスクも可能。
•  実践Tips:
•  複雑な指示は「まず計画だけ立てて」と段階的に。
•  大規模リファクタリングに最適。

2. Tab（コンテキスト賢いオートコンプリート）
•  動作の流れ:
•  コードを書いている最中に自動提案。
•  例: function calculateTotal と打ち始めると、周辺コードや最近の変更を考慮して複数行の関数実装を提案。
•  Tabキーで適用 → さらに予測を続ける。
•  先進ポイント: 専用Tab Modelが「次に何をしたいか」を予測。従来のCopilotより自然で正確。
•  実践Tips:
•  コードを書きながら「予測に任せる」スタイルで生産性爆上がり。
•  好みのスタイル（.cursorrulesで指定）でカスタマイズ。

3. Chat & Cmd+K（対話型編集）
•  Chat（Cmd + L）:
•  コードベース全体を理解した会話。
•  例: 「このモジュールのセキュリティ脆弱性を指摘して修正案を出して」→ ファイル参照（@）しながら提案。
•  画像をドラッグ&ドロップ → 「このデザインを実装して」。
•  Cmd + K（インライン）:
•  選択範囲に対して即時編集。
•  例: 関数を選択 → 「この関数を最適化してパフォーマンス2倍に」→ 即座にリライト。
•  実践Tips: Chatで全体像把握 → Cmd+Kでピンポイント修正。

4. Agent Mode / BugBot（完全自律エージェント）
•  動作の流れ:
1.  Agent Mode有効化。
2.  タスク指示 → AIが計画・コード編集・ターミナル実行・テスト・修正ループ。
3.  背景実行可能（あなたが別の作業中も進む）。
•  BugBot: PR作成時に自動レビュー。「このコードの潜在バグを指摘」→ 修正提案。
•  実践Tips:
•  長時間タスク（新機能追加）に使用。
•  クラウドエージェントでスケール（大規模プロジェクト）。

5. その他先進機能
•  .cursorrulesファイル: プロジェクトルートに置くテキストファイルで「常にTypeScript厳格モード」「React Hooksルール遵守」などAIに指示。
•  モデルスイッチャー: タスクごとにClaude 4（論理的）、GPT-5.5（高速）、xAIモデルなどを切り替え。
•  統合: GitHub PR直接レビュー、Slackで@cursor依頼、クラウド並列エージェント。
•  コンテキスト: 200k+トークンで大規模リポジトリも丸ごと理解。

SpaceX買収後の実用的影響
•  内部開発加速: SpaceXの複雑なコードベース（ロケット制御、Starlink管理、xAIモデル訓練など）でAgent Modeが大活躍すると予想。
•  カスタマイズ: SpaceX独自モデルやセキュリティ強化が追加される可能性大。
おすすめの始め方:
1.  無料/Proプランで試す。
2.  小さなタスクからComposer/Agentを試す。
3.  .cursorrulesで好みを教える。
これでかなり実践的に使えるはずです！
