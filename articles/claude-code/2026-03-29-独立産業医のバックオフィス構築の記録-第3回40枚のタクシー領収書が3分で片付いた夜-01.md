「取説」を入れたところで、実際に使ってみましょう！

経費管理フォルダで `claude` と打って起動し、`/init` と入力すると、Claude Codeが「このフォルダは空ですね。経費管理（PDFレシート→Excel整理）のプロジェクトですか？どんな構成で進めますか？」と聞いてきました。「作業前に確認してから進めて下さい」という取説の指示が、早速機能しているようです。ありがたや。

どうやって答えたらいいですか？を Claude に聞きながら  
PDFを入れるinputフォルダと、完成したExcelを出すoutputフォルダをお願いします。  
PythonとopenpyxlとPDFを読めるライブラリを使いたいです。  
カテゴリはファイル名のキーワードで自動判定してください。

すると、Claude Codeは計画書を出してきましたよ。

```  
keihikanri/  
├── CLAUDE.md  
├── main.py  
├── extract.py       # PDF→テキスト抽出  
├── categorize.py    # キーワード→カテゴリ判定  
├── export.py        # データ→Excel出力  
├── input/  
└── output/  
```

カテゴリ判定のキーワードはこちらで指定してさしあげます。taxi・train・suicaなら交通費、cafe・coffeeなら打ち合わせ、bookなら書籍代などなど。  
「Yes, auto-accept edits」を選んで実行。約3分後、全ファイルが揃ったのです！

早速試してみたいのですが、良くも悪くも確定申告の処理直後。  
その夜、手元にあったのは年が明けてからの40枚分のGOタクシーの領収書のみ。。。

ファイル名での判別キーワードは taxi しか試せないわけですが、  
とりあえず inputフォルダに40枚のPDFをドラッグ＆ドロップ。  
ターミナルに戻って `python3 main.py` と打ってみると。。。

> 完了：経費一覧\_20260310\_003352.xlsx　処理件数：40件

うっそ！3分かかっていない！！！

outputフォルダを開いてExcelを確認すれば、ほぼ完璧でした。ただ1点だけ問題があって、それは店名が全部「領収書」になっていたこと。ちょっと面白い。これは PDFの一番上に大きく「領収書」と書いてあるために、それを店名だと誤認識していた模様。Claude 氏、領収書って屋号のお店はなかなかいいセンスww。将来何か屋号つける場面あったら思い出すかも。  
さておき、Claude Codeに状況を説明してコードを修正してもらい、今度こそ。無事に「GO株式会社」が正しく入ったものが出てきました。

さらに欲を出して「月別に別シートに分けてほしい」とお願いすると、  
日付を読んで自動で振り分けてくれるように改修されました。2026年1月シート、2月シート、3月シート。それぞれきれいに並んでいます。

これはもう来年の確定申告に、焦らなくていい気がする！勝った？勝ったわよね？  
雄叫びは流石に夜中なので上げませんが、黙ってガッツポーズして寝たのでした。寝つきスッキリ！
