## 🐼 なぜ DeepSeek V4 なのか？

「安かろう悪かろう」ではありません。 DeepSeek V4（V3以降の最新モデルを想定）が選ばれるのには、明確な理由があります。

### 💰 圧倒的なコストパフォーマンス

Claude 3 Opus と比較して、桁違いに安価です。  
それでいて、 **1.6 兆パラメータ** と **100 万トークン** という広大なコンテキストウィンドウを誇ります。

これが最大のポイントです！  
Anthropic のモデルと同様に、AI が自らツール（ファイル操作やコマンド実行）を呼び出す能力に長けています。  
つまり、 **Claude Code が持っている便利な機能を、そのまま DeepSeek に肩代わりさせることができる** のです。

## 🛠️ 環境構築とセットアップ手順

ここからは、実際にローカルの IDE（ **Antigravity IDE** ）に環境を構築する手順を、ステップバイステップで解説します。

仕組みはシンプルです。 Claude API へのリクエストをプロキシサーバーで「待ち伏せ」し、裏側で DeepSeek の API へこっそり流すという手法をとります。

### Step 1: DeepSeek API キーの取得とチャージ

まずは「燃料」となる API キーを準備しましょう。

1. [platform.deepseek.com](https://platform.deepseek.com) にアクセスし、アカウントを作成します。
2. クレジットカード等で **2ドル〜5ドル程度** をチャージしてください。これだけで、個人開発なら使い切れないほどの処理が可能です。
3. メニューの **「API keys」** から、新規 API キーを発行してコピーしておきます。

### Step 2: プロキシ用リポジトリの導入（Antigravity を活用！）

手動で環境を作るのは大変ですが、 **Antigravity** の AI チャットを使えば一瞬です。

1. Claude API インターセプト用のリポジトリ URL を用意します。
2. Antigravity のチャット欄に、以下のように指示を出してください。

> 「この GitHub リポジトリ（URL）をクローンしてインストールしてください。その後、以下の DeepSeek API キーを使って接続するように設定してください： [あなたのAPIキー] 」

3. これだけで、AI が自動でクローン、依存関係のインストール、そして `.env` ファイルへの設定まで完了させてくれます。 **「AI が AI のための環境を整える」** という、最高にスマートな体験です。

### Step 3: 起動用ショートコードの作成

毎回長いコマンドを打つのは面倒ですよね。引き続き Antigravity の AI にお願いしましょう。

> 「起動用のショートコードを 2 つ作成してください。コマンド名は『DS』と『DS-force』にしたいです。 1 つは通常通りに実行するコマンド、もう 1 つは権限確認をスキップして強制実行する（dangerously skip permissions）コマンドにしてください。」

これで、ターミナルから `DS` と打つだけで DeepSeek 版 Claude Code が立ち上がるようになります。（強制実行したい時は `DS-force` と打ちます）。

### Step 4: デュアルターミナルの動作確認

設定が完了したら、 IDE のターミナルを新しく開いてみましょう。

1. 設定したコマンド（例： `DS` ）を打ち込みます。
2. 見た目はいつもの `Claude` ですが、中身は DeepSeek です。
3. 試しに **「現在使用しているモデルが DeepSeek V4 であることを確認してください」** と質問してみてください。

正しく返答があれば、 **「標準の Claude Code ターミナル」** と **「 DeepSeek 版ターミナル」** を使い分ける準備は万端です！

## 🚀 実践ワークフロー：モデルの「適材適所」

最強のデュアルターミナル環境ができたら、以下のように役割を分担させるのが **ベストプラクティス** です。

| モデル | 担当領域 | 理由 |
| --- | --- | --- |
| **Claude** | デザイン・UI/UX・初期設計 | クリエイティブな視覚的設計は圧倒的に Claude が優秀。 |
| **DeepSeek** | ロジック・計算・ユニットテスト | 複雑な計算や単純なスクリプト量産などの「力仕事」で真価を発揮。 |
| **ChatGPT** | コードレビュー・監査 | 第 3 者的な視点でのリファクタリング提案に最適。 |

「見た目に関わる部分は Claude、裏側のロジックやテストコードは DeepSeek」という使い分けをするだけで、 **開発クオリティを維持したまま API コストを劇的に抑える** ことができます。

## ⚠️ 注意点

運用にあたっては、以下の 2 点に気をつけてください。

* **機密情報の扱い**: プロキシを経由するため、企業の機密データや重要なキーをターミナルに直書きして送信することは避けましょう。
* **デザインの限界**: DeepSeek 単体でゼロから「美しい Web サイト」を作らせるのは、まだ少し苦手です。デザインは Claude に任せるか、既存のフレームワークを活用するのが正解です。
