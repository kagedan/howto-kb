## はじめに

2026年7月13日 15:59(日本時間)をもって、Claude Code の最上位モデル Claude Fable 5 が、サブスクリプションプランの週次利用枠から外れます。それまでは Pro / Max / Team / Enterprise(Premium シート限定)の対象プランで、週次上限の最大 50% まで Fable 5 を追加費用なしで使えていました。以降は、従量課金(`usage credits`)を消費しないと使えなくなります。

「限られたサブスク枠の中で使えた」状態から、「予算を消費して使う」状態への移行です。ここで大事になるのは、①どこで使うかの見極めと、②どう使うか(費用の抑え方)の2つです。

この変化に対して、各社の判断が分かれています。筆者が観測した範囲では、「これまで通り使い続ける」と表明する企業の発信も見かけるようになりました。ただ、多くの会社にとって現実的なのは、限られた予算の中で ROI(投資対効果)を見ながら、Fable 5 を要所に絞って使うことだと思います。

カンリーのエンジニア本部でも、この「どこで・どう使うか」を、現時点(2026年7月)の情報をもとに決めました。結論から言うと、**従量課金枠の運用は従来のまま、その枠の中で賢く使う**という方針です。「なんでも Fable」でも「Fable 全面禁止」でもありません。モデルの提供条件も料金も動きが速いので、今後の状況次第で見直す前提の暫定方針です。この記事では、その方針と、判断の根拠にした公式情報を共有します。同じ問題に向き合っている会社の参考になればと思います。

## 想定読者

* Fable 5 がサブスク枠から外れた後、どう付き合うか迷っている方
* 最上位モデルを予算内でどう使い分けるか考えている方
* チームや組織での AI モデル運用方針を検討している方

## 何が変わるのか

前提として、カンリーでは Team プランの Premium シートを全エンジニアに、ビジネスメンバーには Premium または Standard シートを一部メンバーに配布しています。この記事で扱うのは、このうちエンジニア本部(Premium シート)での話です。

現在は、サブスク枠の一部として Fable 5 を使えます(2026年7月13日まで / 直近の提供条件では週次上限の最大 50% まで)。切り替え後は、サブスク枠から Fable 5 が外れ、従量課金枠の予算内でのみ使えるようになります。

Anthropic の API 従量課金の単価(100万トークンあたりの入力 / 出力)を並べると、モデル間の差がはっきりします。

| モデル | 入力 / 出力(per MTok) |
| --- | --- |
| Claude Sonnet 5 | $3 / $15 |
| Claude Opus 4.8 | $5 / $25 |
| Claude Fable 5 | $10 / $50 |

Fable 5 の単価は Opus 4.8 の2倍です。サブスク枠で気軽に投げられた時と同じ感覚で全タスクを Fable 5 に回すと、従量課金の予算はすぐに尽きます。だからこそ、使いどころと使い方をあらかじめ決める必要が出てきました。

## 判断の土台①：どこで使うか(モデル選択 × Effort Level)

Anthropic の公式ブログに、モデル選択と Effort Level(推論にどれだけ手をかけるか)の考え方が整理されています。カンリーの方針もこの整理を土台にしています。

調整できるつまみは2つあり、役割が違います。

* **モデル = 「誰に頼むか」**。知識と能力は学習時点で決まっていて、頼み方を工夫しても後から賢くはなりません
* **Effort Level = 「どれだけ働いてもらうか」**。考える時間ではなく仕事量そのもの(読む資料の量、検証の徹底度、確認を挟まず自走する量)を変えます

公式ブログでは、人に仕事を頼むときの例えで説明されています。Fable 5 は「ほぼ誰も見たことのない問題を扱ってきた専門家」、Opus は「エキスパート」、Sonnet は「とても優秀なジェネラリスト」です。この例えで言うと、低 Effort の Opus は「経験豊富な専門家との5分の相談」、高 Effort の Sonnet は「優秀なジェネラリストに午後いっぱい付き合ってもらう」ようなものです。難問でない限り、5分のエキスパートより、午後まるごと付き合ってくれるジェネラリストの方が良い結果になることが多い、というのが公式の見立てです。

### うまくいかないときの見直し順序

ここが実務で一番使える部分です。結果が出ないとき、いきなりモデルを上げるのではなく、次の順序で見直します。

1. **まず自分の渡し方を疑う**。必要な資料・前提を渡し忘れていないか
2. 資料を飛ばし読みした・検証しなかった・途中で投げ出した、という「試行不足」なら → **Effort Level を上げる**
3. 文脈を全部渡して明らかに頑張ったのに、自信満々に間違える、という「知識不足」なら → **モデルを上げる**

### コストの直感に反するところ

もう1つ、公式ブログが指摘している大事な点があります。難しいタスクでは、単価の高い大きいモデルの方がタスク総額では安く済むことがある、というものです。小さいモデルは限界に向かって何度も試行するため、トークンを積み重ねた結果、総額では割高になることがあるからです。日常タスクではどのモデルでも正解できるので小さいモデルが安く付きますが、本当に難しいタスクでは逆転が起きます。

そして Fable 5 の使いどころについて、公式ブログには「テストの中で、Fable 5 は Opus や Sonnet がどの Effort Level でも到達できない仕事を完了した」という趣旨の記載があります。裏を返せば、普段はデフォルト設定の Opus / Sonnet で十分だということです。**文脈を渡し、Effort を上げてもなお自信満々に間違え続けるタスクに出会ったときが、予算を使ってでも Fable 5 に頼る合図**になります。

## 判断の土台②：どう使うか(Fable 5 を要所に絞る2つの型)

「どこで使うか」が決まっても、「どう使うか」でさらに費用は変わります。発想はシンプルで、トークンを大量に消費する作業は安いモデルに流し、Fable 5 は判断の要所だけに使う、というものです。Anthropic が2つの型を公式に紹介しています。

### 相談役(Advisor)型

実行役の Sonnet 5 が手を動かし、迷ったときだけ Fable 5 に助言を求める型です。人間で言えば、ジェネラリストが仕事を進め、難所だけスペシャリストに短時間相談するイメージです。トークン生成の大半は安い実行役(Sonnet 5)の単価で課金され、要所の判断だけ高い Fable 5 を使うので、Fable 5 単体に近い品質を、より低い総額で得られます。

公式が公開した検証(SWE-bench Pro)では、Sonnet 5 に Fable 5 を相談役として付けると、**Fable 5 単体のスコアの約92%を、約63%の費用で達成**したと報告されています。Fable 5 が呼ばれるのはタスクあたり1回程度で、大半の作業は Sonnet 5 が進めます。数値の内訳は次の公式ポストにまとまっています。

### 指揮役(Orchestrator)型

Fable 5 は計画と采配に徹し、実作業は Sonnet 5 のワーカーに分担させる型です。人間で言えば、スペシャリストは作戦立案と指示に専念し、手を動かすのはチーム、という構成です。トークンを多く消費する実作業は、安いワーカー(Sonnet 5)の単価で課金されます。

公式の検証(BrowseComp)では、Claude Managed Agents で Fable 5 を指揮役、Sonnet 5 をワーカーのサブエージェントにした構成が、**Fable 5 単体の性能の約96%を、約46%の費用で達成**しています。トークンを多く使うリサーチ作業を Sonnet 5 に委譲した結果です。検証結果の内訳はこちらの公式ポストで公開されています。

なお、公式スレッドによると、サブエージェントはそれぞれ自分のキャッシュを保持するため、同じ文脈への繰り返し呼び出しで満額を二重に払わずに済みます。

### Claude Code での実践イメージ

この2つの型は、Claude Code でもそのまま試せます。

* **相談役型**: 普段は Opus / Sonnet で進め、行き詰まった難所だけ `/model` で Fable 5 に切り替えて方針をもらい、方針が出たら元に戻す。より仕組み化するなら、`/advisor fable` で実行役の途中に Fable 5 の助言を差し込む、フロントマターで Fable 5 を指定したカスタムサブエージェントを用意する、プロンプトで「Fable に相談して」と依頼する、といった方法もあります
* **指揮役型**: Fable 5 に計画を立てさせ、その後の実作業は Dynamic Workflows などで動くサブエージェント群に委譲する。ワークフローの各エージェントはデフォルトでセッションのモデルを引き継ぐため、Sonnet 5 のワーカーで動かすにはオーケストレーションスクリプト側でモデルを明示的に指定する必要があります(指定を忘れると Fable 5 のまま大量のエージェントが走り、想定外にコストが膨らみます)

どちらも「最初から最後まで Fable 5 に任せる」以外の選択肢です。要所投入なら、品質をほぼ保ったまま、費用を単体比 6 割前後(相談役型)から半分以下(指揮役型)に抑えられます。

## カンリー エンジニア本部の使い分けフロー

①②を踏まえて、カンリーのエンジニア本部では次の順序で使い分けることにしました(運用しながら見直す前提の暫定方針です)。

基本スタンスは「モデルと Effort を賢く使い分ける段階に入った」ということです。とにかくトークンを使う時期(Token Maxxing)から ROI を見る時期へ移ってきた、というのが足元の肌感です。かつての「全部 Opus」ではなく、モデルと Effort Level の使い分けを各自が模索し始めています。

あわせて、サブスク型の使い放題が今後も続くとは限りません。GitHub Copilot は 2026年6月から従量課金制(月次のクレジット枠 + 超過分は追加購入)へ移行しました。GitHub 自身、簡単な質問も長時間のセッションも同じコストという定額モデルは、エージェント型の利用が広がるなかで持続可能ではなくなったと説明しています。サブスクに安住して使い方を見直さないままだと、従量課金に移った時にコストが青天井になりかねません。賢い使い分けを今のうちに習慣づけておく意味でも、この方針は妥当だと考えています。

足元では、Sonnet も 4.6 から 5 への進化で性能が上がり、日常タスクに通用する場面が増えてきました。枠内に収まるなら初手 Opus でも問題ありません(その場合はフローの③まで進んだ状態から始めるイメージです)。そのうえで、Fable 5 を除く最難問は Opus(高 Effort)で当たり、それでも越えられない壁だけ Fable 5 を要所に投入する。この形なら、レート制限枠内に近い形で回せそうだという手応えがあります。

なお、Opus 4.8 では、ツールの実行に失敗する・意図しない挙動が出るといったケースも社内で観測しています(Sonnet 5 では今のところ遭遇していません)。挙動が不安定だと感じたら、Sonnet 5 への切り替えも選択肢に入れています。

### 切り替えまでの残り期間の使い方

Fable 5 がまだサブスク枠にある残り期間(この記事の公開時点では 2026年7月13日まで)は、「なんでも Fable 5 に全タスクを投げてどこまでいけるか」を試すのは控えめにして、今回紹介した相談役型・指揮役型を実際に触ってみる期間にするのがよさそうです。全部任せる使い方の感触はもう掴めてきたので、残りの時間は「要所投入でどこまで品質を保てるか」を各自で確かめる方が、切り替え後に生きる学びになります。

## 社内で見えた「Fable 5 でないと越えられなかった場面」

「全面禁止にはしない」と決めた根拠は、社内の活用事例で「Opus / Sonnet では届かず、Fable 5 でようやく実用になった」場面が実際にあったからです。共有会で集まった事例を、性質ごとに整理すると次のようになります。

* **複数ステークホルダーが絡むプロジェクトの戦略立案**: 議事録・意思決定の記録・関連ドキュメントを10枚以上まとめて読ませ、そこから戦略を組み立てさせる使い方。Opus や Sonnet では難しかったものが、Fable 5 でようやく実用的になった
* **過去に作った成果物のレビュー**: 既存のスクリプトやドキュメントをレビューさせると、他モデルが見落としていた穴や誤りを発見した。修正の緊急度の判断にも使えた
* **肥大化した設定・スキルの棚卸し**: 1,000行を超えて膨らんだファイルを整理し、競合している設定を修正したり、再利用できる形への還元方法を提案させたりできた

共通するのは、**全体を俯瞰する力と初手の筋の良さ**が問われる場面です。大量の文脈を渡して全体像をつかませたり、自律的に検証を回させたりする用途で、Fable 5 は他モデルとの差がはっきり出ました。逆に言えば、こうした難所に当たらない日常タスクでは、Opus / Sonnet で十分なケースも多くあります。

## まとめ

Fable 5 がサブスク枠から外れることは、「使い続けるか、やめるか」の二択に見えがちです。ただ、実際に考えるべきは「どこで・どう使うか」だと思います。

カンリーのエンジニア本部の目下の方針は、次の3つです。

1. **予算運用は従来のまま、従量課金枠の中で賢く使う**。全体と個人の上限を決め、必要な人には個別の引き上げで応じる。一律に増やすでも禁止するでもない(個別の引き上げが常態化してきたら、全体の底上げも検討します)
2. **標準(Sonnet / Opus)で試し、渡し方 → Effort → モデルの順で見直す**。それでも越えられない壁だけ Fable 5 を使う
3. **Fable 5 は要所に絞る**。相談役型・指揮役型を使えば、品質をほぼ保ったまま費用を抑えられる

投資余力が大きければ「ふんだんに使う」選択もあります。ただ、多くの会社は限られた予算の中で使いどころを見極める運用になるはずです。方針は運用しながら見直します。実際に回して見えたことがあれば、また共有します。
