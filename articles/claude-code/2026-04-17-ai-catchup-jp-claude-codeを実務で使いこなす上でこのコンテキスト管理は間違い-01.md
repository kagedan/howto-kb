Claude Codeを実務で使いこなす上で、このコンテキスト管理は間違いなく最重要テーマですよね！有益なまとめをありがとうございます✨
引用元の「100万トークンのコンテキストは諸刃の剣」という言葉の背景と、画像内のコマンドの使い分けを少しだけ補足させていただきます👇
・なぜ諸刃の剣か？
記憶力が良すぎるため、放置すると「失敗したデバッグの試行錯誤（ノイズ）」まで引き継いでしまい、AIが混乱して精度が落ちるため。
・コマンドの超ざっくり使い分け
🔙 /rewind：AIが間違えたら「失敗の記憶」だけ消してやり直し
🗜️ /compact：履歴が長すぎたら「要約」して引き継ぐ（※詳細が消えるのでアホになりやすい）
🧹 /clear：全く別のタスクを始めるなら「完全リセット」
🤖 Subagent：大量のログが出る作業は「別枠」でやらせて、結論だけ本体に戻す
特に「Subagent」でノイズを本体のコンテキストから隔離するテクニックは、AI開発時代の必須スキルになりそうですね！👀🔥
