Web アプリ **Garage mgr(ガレージマネージャー)**   
  
驚かれるかもしれませんが、このアプリの開発者である私は、プログラミングのプロではありません。  
  
アプリ制作に関しては「初心者」であり、実際のソースコードは 1 行も書いていません。

そして、このWeb アプリの運営コストは、月 125 円(年間ドメイン代の按分のみ)です。

では、どうやってこんなアプリを作ったのか。

実は、YouTube の解説動画を見て「これならできるかも」とインスピレーションを受け、いきなり制作をスタート。  
  
何も分からない。  
でもやってみたい。  
この気持ちだけの見切り発車です。  
  
コードはすべて対話型 AI「**Claude Code(クロードコード)**」に出力させ、自分は「実際の使用感・デザイン・ロジックの辻褄が合わない部分」を厳しくチェックし、進むべき方向を決断するという役割に徹しました。  
  
というか、専門的なことは、正直何も分からないところからのスタート。  
だから、こうするしかありませんでした。笑

この記事では、初心者が AI を駆使して「月 125 円」の極限ローコストアプリを完成させたプロセスと、その設計判断のすべてを、実データとともに開示します。  
  
なんなら、この文章も思いっきりAIを駆使して作成しています。  
もちろん、文章の内容の主な部分は自分ですが、数値やエビデンスになる部分はAIを駆使して記載しています。

## 開発の舞台裏:コードを書かない「チェックと決断」のワークフロー

私の開発スタイルは、一般的なプログラミングとはまったく異なります。  
仕事の本質は「コードを書くこと」ではなく、**「チェック」と「決断」** にありました。

### 1. YouTube からの衝動的なスタート

「アプリ開発なんて、初心者には無理」という先入観がありましたが、YouTube で AI 開発の動画を見たことで、「もしかして、自分でも作れるのではないか」と思い立ち、その日のうちに制作を開始しました。

### 2. 開発はすべて Claude Code との対話

環境構築からデプロイまで、わからないことはすべて Claude Code に質問しました。

「こういうアプリを作りたい。初心者なので、最初にパソコンで何のコマンドを打てばいいか教えて」

そんな "普通の日本語" を投げるステップから、開発は始まっています。  
技術用語も、コマンドの意味も、最初は 1 つも分かりませんでした。

### 3. 人間が担った「チェック」の役割

Claude Code が吐き出したコードをそのまま貼り付けてアプリを動かし、私は「一人のユーザー」として徹底的に触り倒しました。  
というか、それ以外ができない。笑

* **使用感のチェック**:ボタンの位置は押しやすいか、画面の切り替わりにストレスがないか
* **デザインのチェック**:スマートフォンで見たときに、文字の大きさや余白のバランスが心地よいか
* **不整合(辻褄が合わない部分)の発見**:「登録した車のデータが、別の画面を開くと消えて見える」「カレンダー連携の期日の計算が 1 日ズレている」「iPhone SE(小画面)でモーダルが画面外にはみ出す」といった、仕様上のバグや矛盾を見つけ出す作業

実際に開発中に起きた「チェック」の例:

* **例 1**:iPhone SE(旧型)でオンボーディング画面を見たら、ボタンが画面外に切れて押せない → Claude Code に「小画面で全体表示させて」と依頼 → CSS の単位を `vh` から `dvh` に変更 + ボタンを固定領域に切り出す修正が入った
* **例 2**:シェアボタンのアイコンが絵文字(📤)だったが、OS によって見た目が違ってダサい → 「iOS 標準の共有アイコンっぽい SVG に置き換えて」と指示 → 統一された SVG グリフに差し替え

### 4. 修正案の提示と「決断」

見つかった違和感や問題点を、そのまま自分の言葉(自然な日本語)で Claude Code に投げかけます。

「ここを触ったときにこういう挙動になるのが不自然だから、直して」  
「このデザインはもっとシンプルに、直感的に使えるようにして」

とリクエストすると、Claude Code がすぐに修正案やコードを提示してくれます。

複数の解決策を提案された場合は、ユーザー体験(UX)や運用コストの観点から、どれを採用するかを **自分が「決断」する**。  
このキャッチボールを一つずつ泥臭く積み重ねることで、未経験からでもアプリのクオリティを限界まで引き上げることができました。

## コスト 0 円を実現した 6 つの技術決定

Claude Code との対話の中で、私が「コストを抑えつつ、最高の体験を作りたい」と決断した 6 つの設計判断です。

### 決定 1:ホスティングに Cloudflare Workers を選んだ理由

**採用しなかった候補**:AWS Lambda / Google Cloud Run / Vercel / Netlify

**選んだ理由と決断**:Claude Code から「1 日 10 万リクエストまで無料で、世界最速クラスの配信ができるインフラがある」と提案され、即決しました。  
Vercel などのように「商用利用は有料プランのみ」という規約の縛りがなく、将来的な収益化の選択肢を残せる点も、マーケター視点で大きな決め手となりました。

### 決定 2:データ保存に IndexedDB(端末内保存)を選んだ理由

**採用しなかった候補**:Firebase Firestore / Supabase / Cloudflare D1

**選んだ理由と決断**:初心者が最も頭を悩ませるのが「データベース(DB)の管理と課金」です。Claude Code から「サーバーにデータを送らず、ユーザーのスマホ(ブラウザ)の中に直接保存する仕組み(IndexedDB)にすれば、DB 代は永遠に 0 円です」と提案されました。

「データが外部に漏れないため、情報漏洩リスクが構造的にゼロになる」「オフラインでも爆速で動く」というメリットを評価し、この構成を選択しました。

### 決定 3:AI に Gemini 2.5 Flash を選んだ理由

**採用しなかった候補**:GPT-4o / Claude 3.5 Sonnet / Gemini 1.5 Pro

**選んだ理由と決断**:アプリの目玉機能である「車検証の AI 読み取り」を実現するため、各社 AI の無料枠を比較しました。  
結果、コストパフォーマンスとマルチモーダル(画像認識)の処理速度に優れた **Gemini 2.5 Flash** を採用。

* **無料枠**:1 分あたり 15 リクエスト、月間 100 万トークンまで無料
* **マルチモーダル**:画像 + テキストの複合入力を軽量に処理できる
* **コンテキストウィンドウが広い**:車検証のような高密度な情報の構造化にも耐える
* **精度**:車台番号や型式などの複雑なテキストを 95% 以上の正確さで一瞬で抽出
* **速度**:1 回の OCR に約 2〜4 秒

### 決定 4:プッシュ通知を実装せず、iOS カレンダー連携で代替した理由

**採用しなかった手段**:Web Push Notification / OneSignal / Firebase Cloud Messaging

**選んだ理由と決断**:アプリから「車検の時期です」と通知を送る機能を検討した際、通常のプッシュ通知を実装しようとすると通知管理用のサーバーが必要になり、固定費が発生することが分かりました。

そこで Claude Code に相談したところ、「**カレンダーファイル(`.ics` 形式)を動的に作って、iPhone 標準のカレンダーに予定を登録させる方法** がある」と提示されました。これなら通知は OS が勝手にやってくれるためサーバー代は 0 円。  
ユーザーも日常の予定と一緒に管理できるため、ベストな解決策だと決断しました。

### 決定 5:ユーザーアカウント登録を実装しない選択

**選んだ理由と決断**:ログイン画面やパスワード忘却対応などの機能を盛り込むと、開発コストが跳ね上がり、認証サービスの費用もかかります。  
データが端末内にしかない以上、アカウントを作る意味自体がありません。

ユーザーにとっても「開いた瞬間に使い始められる」という最高の体験に繋がると判断し、アカウント機能をバッサリ切り捨てました。

### 決定 6:ネイティブアプリではなく PWA を選んだ理由

**採用しなかった候補**:React Native / Flutter / iOS ネイティブ

**選んだ理由と決断**:App Store でアプリを公開すると、Apple に毎年 **99 ドル(約 15,000 円)** の維持費を支払い続けなければなりません。  
また、厳しい審査を待つ必要もあります。

Web サイトでありながらスマホアプリのようにホーム画面に追加して使える **PWA** という技術を採用することで、ストア維持費をゼロにし、URL 一つで瞬時にシェアできる手軽さを手に入れました。

## 「チェックと決断」で乗り越えた、端末内保存の妥協点と代替策

「データをサーバーに置かない」という決断にはデメリットもあります。  
実際の使用感をチェックする中で見つけた課題と、それを Claude Code と一緒にどう解決したかを紹介します。

### 課題 1:複数端末間のデータ同期ができない

**チェックでの気づき**:iPad で登録したデータを iPhone で見られないのは、ユーザーによっては不便かもしれないと気づきました。

**対策の決断**:自動同期を実装するとサーバー代がかかるため、設定画面からデータを「JSON ファイル」としてワンタップで書き出せる機能を Claude Code に作ってもらいました。  
このファイルをメールや AirDrop で送って読み込ませることで、手動でのデータ移行を可能にしています。

### 課題 2:スマホの紛失やキャッシュクリアでのデータ消失リスク

**チェックでの気づき**:スマホが壊れたり、ブラウザの履歴を完全に削除したりすると、大切な車のデータが消えてしまうリスクに直結します。

**対策の決断**:UI(画面)上で「データはあなたの端末にのみ保存されているため、定期的にバックアップをとってください」と目立つ注意喚起を配置。  
上記の JSON エクスポート機能を使って、ユーザー自身が iCloud や Google ドライブに安全にファイルを保存できるように設計しました。

### 課題 3:プッシュ通知が使えない

**チェックでの気づき**:「期日を忘れないためのアプリなのに、通知が飛ばないと本末転倒では?」というジレンマがありました。

**対策の決断**:Claude Code に「iOS のカレンダーに直接予定を書き込ませる `.ics` 形式のファイルを生成して、Web Share API で共有シートから登録させよう」と提案されました。  
ユーザーはボタン 1 つで OS 標準カレンダーに登録でき、通知は OS が確実に鳴らしてくれるため、独自プッシュ通知よりも高い信頼性を実現しています。

## 初心者が同じ構成でアプリを公開するためのロードステップ

私が YouTube の動画を見ながら、Claude Code を相棒にして進めた手順のタイムラインです。  
コードの書き方が分からなくても、以下のステップに沿って「指示」を出せば形になります。

### ステップ 1:ツールの準備とアカウント作成

まずは開発に必要なベースとなるアカウントを用意します。

1. **GitHub アカウントの作成**(ソースコードの保管場所)
2. **Cloudflare アカウントの作成**(アプリを世界に配信するインフラ)
3. **Google AI Studio での Gemini API キー取得**(AI 機能の心臓部)
4. **独自ドメインの購入**([お名前.com](http://xn--t8jx73hngb.com) や Cloudflare Registrar 等で、年間 1,500 円程度で好きな名前を取得)

### ステップ 2:Claude Code に「環境構築」を丸投げする

パソコンのターミナル(黒い画面)を開き、Claude Code に

> 「Vite + React + TypeScript + Tailwind CSS で PWA アプリを作りたい。初心者にわかるように、打ち込むコマンドを 1 から順番に教えて」

と指示します。言われた通りのコマンドをコピー & ペーストするだけで、最新の開発環境が手元に整います。

### ステップ 3:仕様のチェックと「辻褄合わせ」のループ

ここからが「チェックと決断」の本番です。

Claude Code に画面のデザインや機能のコードを出力させ、それをブラウザで確認します。

* 「ボタンを押したときに、この画面にジャンプするのは不自然」
* 「車検証の文字読み取り結果を表示する枠が、スマホだと画面からはみ出しているから直して」
* 「iPhone SE で見たときにモーダルが画面から切れる。全体を表示させて」

など、自分が触って感じた違和感をそのまま伝えて、コードを何度もアップデートさせます。  
**このループを何十回、何百回と回すことがクオリティを決めます。**

### ステップ 4:セキュリティと環境変数の設定

Gemini の API キーなどの重要な秘密情報は、コードに直接書いてはいけません。  
Claude Code に

> 「Cloudflare の環境変数(Secret)に安全に API キーを登録するコマンドを教えて」

と聞き、指定されたコマンド(具体的には `npx wrangler secret put GEMINI\_API\_KEY`)を使ってインフラ側に安全に保管します。

### ステップ 5:本番環境への公開とドメイン設定

アプリが完成したら、Claude Code に

> 「Cloudflare Workers にデプロイ(公開)するコマンドを教えて」

と言って実行します。  
指定される `npx wrangler deploy` を打つだけで、数秒でアプリがインターネット上に公開されます。

最後に、Cloudflare の管理画面から購入した独自ドメイン(例:garage-mgr.com)を入力して紐付ければ、世界中で使える **あなただけのアプリが誕生します**。

**トータル所要時間**:初回セットアップは 3〜5 時間程度、費用はドメイン代のみです。

## 結論:AI 時代の開発者は「最高のチェッカー」であれ

* プログラミング未経験の初心者でも、YouTube の動画をきっかけに、Claude Code を味方にすればクオリティの高い Web アプリを完成させられる
* コードを 1 行も書かなくても、**「実際の使用感」「デザインの心地よさ」「ロジックの不整合」** を人間が厳しくチェックし、進むべき方向を決断し続ければ、プロ顔負けのシステムが組み上がる
* 「サーバーにデータを置かない(IndexedDB)」などの思い切った設計判断を自分が決断することで、ユーザーのプライバシーを究極まで守りながら、**月額 125 円(ドメイン代のみ)** という驚異的なローコスト運用が実現できる

プログラミングの文法を覚える必要はありません。  
必要なのは、**アプリを自分で触り、「もっとこうすれば使いやすくなるはずだ」というユーザー目線のこだわり(チェック)と、それを形にするための選択(決断)** です。
