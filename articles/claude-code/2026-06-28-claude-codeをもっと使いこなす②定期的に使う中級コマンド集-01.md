# Claude Codeをもっと使いこなす②｜定期的に使う中級コマンド集

こんにちは、開発日和です。

Part1では毎日使う基本コマンドを紹介しました。今回は「知っていると便利だけど毎日は使わない」中級コマンドを紹介します。これらを知っておくと、特定の場面でぐっと作業が楽になります。

---

## この記事で紹介するコマンド

| コマンド | ひとことメモ |
| --- | --- |
| `/agents` | サブエージェントの管理・確認 |
| `/effort` | タスクへの力の入れ具合を指定する |
| `/export` | 会話内容をファイルに書き出す |
| `/security-review` | セキュリティ観点に特化したレビュー |
| `/simplify` | コードをシンプルにしてもらう |
| `/plan` | 実装前に計画を立ててもらう |
| `/background` | バックグラウンドで作業を走らせる |
| `/fork` | 会話を分岐させる |

---

## `/agents` ― サブエージェントの管理

### ①何をするか

Claude Codeが内部で動かしているサブエージェント（自律的に動くAIタスク）の一覧と状態を確認・管理するコマンドです。

### ②実際の挙動

![](https://static.zenn.studio/user-upload/b11ec3e3833f-20260628.png)  
現在動いているエージェントや、完了・失敗したエージェントの一覧が表示されます。不要なエージェントの停止もここから行えます。

### ③活用方法・つまずいたポイント

* **サブエージェントを直接起動するコマンドではない**。起動はプロンプトから行い、`/agents` はその確認・管理ツール
* サブエージェントの仕組みはPart4・5で詳しく解説します
* バックグラウンドで動いているエージェントが予期せず動き続けていることがあるので、定期的に確認する習慣をつけると良い

---

## `/effort` ― 力加減を指定する

### ①何をするか

Claudeがタスクにかける力の入れ具合（effort）を指定します。「軽く見てくれればいい」から「徹底的にやって」まで、状況に応じて使い分けられます。

### ②実際の挙動

![](https://static.zenn.studio/user-upload/b36ce238591c-20260628.png)  
lowからmaxまで選択できます。

```
> /review --effort low      # 軽めのレビュー（速い）
> /review --effort medium   # バランス型（デフォルト）
> /review --effort high     # 徹底的なレビュー（時間・コストかかる）
```

このように、`/effort` 単体で使うというより、他のコマンドのオプションとして使うこともできます。

### ③活用方法・つまずいたポイント

* **`high` にすれば常にいい、というわけではない**。簡単な確認作業に `high` をつけると無駄にコストがかかる
* コードレビューや設計の確認など「品質が大事な場面」で `high` を使うのがコツ
* デフォルトは `medium` なので、ほとんどの作業はオプション指定なしでOK

---

## `/export` ― 会話をファイルに保存する

### ①何をするか

現在の会話内容をMarkdownやテキスト形式でファイルに書き出します。「この会話の内容、あとで見返したい」「チームに共有したい」というときに便利です。

### ②実際の挙動

![](https://static.zenn.studio/user-upload/0bf1e95582f3-20260628.png)  
クリップボードにコピーするか、出力先のファイル名・形式を指定してエクスポートします。  
Save to fileを選択した場合  
![](https://static.zenn.studio/user-upload/3e503f6698fa-20260628.png)

```
> /export --output ./meeting-notes.md
```

### ③活用方法・つまずいたポイント

* **セッションをまたいで保存はされない**ため、長い設計会話などは `/export` で残しておくのがおすすめ
* `/clear` の前に使うのがセオリー
* エクスポートしたファイルをそのままZennのスクラップにコピーするのも使い方として便利

---

## `/security-review` ― セキュリティ観点のレビュー

### ①何をするか

`/review` のセキュリティ特化版です。OWASP Top 10などの視点で、SQLインジェクション・XSS・認証不備などの脆弱性を重点的にチェックしてくれます。

### ②実際の挙動

![](https://static.zenn.studio/user-upload/006b90750e01-20260628.png)  
現在のブランチの変更差分に対して、セキュリティ上の問題点を優先して列挙してくれます。

### ③活用方法・つまずいたポイント

* **完璧ではない**。「Claudeがチェックしたから大丈夫」は過信で、重要な機能は人間のレビューも必ずセットで
* 認証・決済・個人情報を扱う実装の前に使う習慣をつけると効果的
* 詳しい使い方はPart13で解説します

---

## `/simplify` ― コードをシンプルにしてもらう

### ①何をするか

変更したコードを対象に、リファクタリングの観点から「もっとシンプルに書けるところ」「重複している処理」「不要な複雑さ」を指摘・修正してくれます。バグ探しではなく、コードの品質改善に特化しています。

### ②実際の挙動

![](https://static.zenn.studio/user-upload/a22f9dc530cc-20260628.png)  
差分を解析して、シンプルにできる箇所を提案・実際に修正してくれます。

### ③活用方法・つまずいたポイント

* **バグ修正は対象外**。「動いているかどうか」より「きれいに書けているか」を見るコマンド
* 自動で書き換えが行われるので、実行前に `git diff` で現状を確認しておくと安心
* リファクタリングをやりすぎると可読性が下がることもあるので、提案内容はちゃんと読んでから受け入れる

---

## `/plan` ― 実装前に計画を立ててもらう

### ①何をするか

いきなりコードを書き始めるのではなく、「どう実装するか」の計画フェーズをClaudeと一緒に進めます。複雑な機能の実装前に使うと、手戻りを大幅に減らせます。

### ②実際の挙動

![](https://static.zenn.studio/user-upload/b6c4f6261d43-20260628.png)  
Claudeが実装のアプローチ・手順・考慮すべき点などを整理した計画を提示してくれます。計画に合意したら実装フェーズへ。

### ③活用方法・つまずいたポイント

* **計画に時間をかけすぎると「計画倒れ」になる**。大枠の方針が固まったら実装に入る判断も大事
* 計画の段階でいろいろな選択肢が出てくるので、自分でどの方針をとるか決めてから実装を進める
* 計画の内容は `/export` で保存しておくと後から参照できて便利

---

## `/background` ― バックグラウンドでタスクを走らせる

### ①何をするか

時間のかかるタスクをバックグラウンドで実行させ、その間も別の作業を続けられるようにします。「大量のファイルを解析してほしいけど待ちたくない」というときに使います。

### ②実際の挙動

```
> /background テストを全部走らせて結果を報告して
```

![](https://static.zenn.studio/user-upload/7a72d15c10b3-20260628.png)  
タスクがバックグラウンドに移り、完了したら通知が来ます。

### ③活用方法・つまずいたポイント

* **バックグラウンドタスクの状態は `/agents` で確認できる**
* 長時間かかるタスクは途中でセッションが切れると止まることがある
* 複数のバックグラウンドタスクを同時に走らせると、コンテキストとコストが増えるので注意

---

## `/fork` ― 会話を分岐させて試す

### ①何をするか

現在の会話の時点から「別のアプローチを試す」ための分岐を作ります。「方針Aと方針Bの両方を試してみたい」「今の状態から戻れる退路を作りつつ実験したい」というときに有効です。

### ②実際の挙動

フォーク地点から新しいセッション分岐が作られ、元の会話に影響を与えずに別の方向を試せます。

### ③活用方法・つまずいたポイント

* **Gitのブランチと概念が似ている**。「試してみたいけど今の状態を壊したくない」場面で使う
* フォーク後の作業が良ければそのまま続行、ダメなら元のセッションに戻る、という使い方が基本
* 詳しくはPart15（並行作業・セッション分岐）で解説します

## まとめ：どんな場面で使うか

| 場面 | 使うコマンド |
| --- | --- |
| 実装を始める前に計画を整理したい | `/plan` |
| レビューはざっくりでいい | `/review --effort low` |
| セキュリティが気になる実装がある | `/security-review` |
| コードをきれいにしたい | `/simplify` |
| 会話を保存したい | `/export` |
| 時間のかかる作業を並行して進めたい | `/background` |
| 方針Aと方針Bを試したい | `/fork` |
| バックグラウンドタスクの確認 | `/agents` |
