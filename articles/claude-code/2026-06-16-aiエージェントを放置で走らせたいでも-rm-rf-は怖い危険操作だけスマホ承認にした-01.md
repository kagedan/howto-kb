## エージェントは賢い。でも結局ターミナルに張り付く

Claude Code や Codex は、ルーチン作業なら放っておいても進めてくれる。問題は `rm -rf`、`git push --force`、本番デプロイみたいな「取り返しのつかない一手」だ。

これがあるせいで、結局ターミナルの前に座って「実行していいか」を見張ることになる。自律のうまみが消える。

そこで、危険な操作の手前だけスマホに飛ばして、どこからでも承認／拒否できるようにした。`Push Bridge` という。

## ルーチンは進めさせる。危険な一手だけ止める

考え方はシンプルで、

* ルーチンな作業はエージェントにそのまま進めさせる
* `rm -rf` / force-push / デプロイ等の「危険な一手」の手前で一時停止
* その瞬間だけスマホにプッシュ通知 → タップで承認／拒否
* 承認するまでエージェントは待機（危険操作の自動通過を防ぐ）

人間の関与を「常時監視」から「危険な瞬間の一タップ」に圧縮する、という発想だ。

## 処理の流れ

```
エージェントが危険操作に差し掛かる
   │  ① 検出フックが発火（rm -rf / force-push / deploy 等のパターン）
   ▼
Push Bridge が通知を送出 → エージェントは待機
   │  ② スマホにプッシュが届く
   ▼
あなたがタップ（承認 / 拒否）
   │  ③ 承認 → エージェントが次へ進む
   │     拒否 → その操作を中止
   ▼
（無反応のままなら実行せず待機 ＝ 黙って実行されない）
```

通知の受け取りはホスト型（PWA）。エージェント側に検出フックを入れ、承認のやり取りはスマホで行う。外出先からでも承認できるので、エージェントを走らせたまま席を立てられる。

## 境界をどこに引くか

全部承認制にすると遅い。全部自動だと怖い。いろいろ試して、境界は「取り返しがつくかどうか」に引くのがちょうどよかった。読み取りや一時ファイルの生成は止めない。消える・上書きする・外に出ていく操作だけ止める。

自律エージェントを任せられるかは、賢さよりも「取り返しのつかない操作を勝手に実行しない」という一点にかかっている気がする。
