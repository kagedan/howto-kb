今年の3月くらいにClaude CodeにRemote Control機能が搭載されました。使っている方も多いと思います。

夜、ベッドで横になりながら、スマホを片手にAIへ指示を出し、朝起きたら作業が終わていて、マジ神機能！天国かよと天にも昇る気分で喜んでいました。

でも気づいたら、知らず知らずのうちにベッドの上がオフィスになっていて睡眠時間が削られて生活リズムが崩れ、地獄に叩き落とされました。

**AIを便利に使っているつもりが、いつの間にか使われる側になっていました。**

以前、エンジニアのAI疲れとの付き合い方という記事を書きました。AIをどれだけ使っても作業時間は短くなっているはずなのに、精神的な消耗だけが増していくというAI疲れのパラドックスについての記事です。

## Claude CodeのRemote Controlとは

<https://code.claude.com/docs/ja/remote-control>

Remote Controlは、PCから離れた場所からでもスマホからClaude Codeを操作できる機能です。自分のPCで動いているClaude Codeのセッションにスマホからアクセスし、チャット感覚でタスクを頼めます。PCの前にいなくてもAIがコードを書き続けてくれる状態を作れます。

自分は現場を離れていても仕事が進み、しかもそのスタッフは残業代も文句も一切ないし、ソファやベッドでくつろぎながら指示して、気が向いたときに結果を確認できる機能です。

## ベッドの上が指揮台になった

Remote Controlの存在を知ったとき歓喜しました。おﾌﾄﾝに入りながらでもAIに仕事を頼めて、寝ている間に作業が進むじゃん。  
そんなことが実際にできてしまうなんてマジやばい。

夜11時にベッドで横になりながら、明日の作業で使う機能を作っておいてと指示を出し、翌朝起きたらコードができていたときの高揚感は格別でした。寝てるだけで欲しいものが手に入る神にでもなったかのような万能感！私の脳内はドーパミンでいっぱいになり脳汁ブシャー状態。子どもの頃に夢見た寝ている間に宿題が終わっているやつが、大人になってから突然実現したような気分で、これは神機能だと驚嘆しました。

そのうち就寝前にタスクリストを作っておき指示を出すのが習慣になりました。歯を磨いて、布団に入って、スマホでタスクを入力して、おやすみ、というルーティーンです。

## 夜中に目が覚めてスマホを確認するようになった

![](https://static.zenn.studio/user-upload/05bb3fcc5acd-20260530.png)

最初に異変を感じたのは、夜中に目が覚める回数が増えたことです。

目が覚めた瞬間、最初に頭に浮かぶのが「あのタスク、ちゃんとできてるかな」という考えでした。最初のうちは気にしないようにしてそのまま眠れていましたが、しばらく経つと気になって仕方なくなり、気にしないようにと再度眠ろうとしても眠れない状態が続き、なら仕方ないかとスマホを手に取るようになりました。

確認してうまくいっていれば安心してまた眠れる、という算段でした。**でも現実はそうはいかなかったです。確認しているうちにここはもう少しこうした方がいいな、次はこれも頼んでおこうという考えが浮かんでしまい、追加の指示を出してしまう。** そうして気づいたら30分が経っている、ということが続きました。

深夜にトイレに行こうと明るい照明の中へ入ったあと急に眠れなくなる感覚があるじゃないですか。あの感覚に近くて、ブルーライトの問題もありますが、**それ以前に仕事モードに入った脳をそこから眠りへ戻すのは、かなり難しい** ことです。夜中の2時や3時にスマホで確認して追加指示まで出してしまえば、当然そうなります。その後は眠れないまま横になり、うっすら眠れたとしても浅い眠りが続いて、朝には疲労感だけが残っている日が増えていきました。

## 脳が常に火照っている感覚が続いた

眠りが浅い日々が続くと、日中の集中力も落ちていきます。でも奇妙なことに、脳自体は休んでいないという感覚がありました。

次はこれをやらせよう、あのタスクはどこまで進んだか、この機能を頼んだらどのくらいかかるだろう、という考えが、起きていても横になっていても頭をぐるぐると回り続けている。熱を出したときの頭がじんわり熱い感じに似ていて、物理的に発熱しているわけではないのに、脳だけが過負荷のまま止まらない状態です。

以前のAI疲れの記事で認知負荷の増大について書きました。AIを使うことで処理すべき情報量が増え、脳が疲れるという話です。Remote Controlはその認知負荷を、仕事の時間を超えて24時間に広げてしまいました。眠っている間は休めているはずの脳が、AIが動いているかもしれないという意識のせいで完全にオフになれない状態が続いていたのだと思います。

## 家族に心配された

ある日の夕食中、妻からずっとスマホ見てない?と言われてドキッとしました

言われてみれば確かに、食事中も、リビングで家族と過ごしている時間も、気づいたらスマホでClaude Codeの状況を確認していました。急ぎの案件があるわけでも、誰かからの連絡を待っているわけでもなく、ただAIが何かやっているかもしれないからという理由だけで、条件反射的にスマホを開いていたのです。(Remote Controlにはプッシュ通知を送る機能もあるので、通知が来てないかも常に気になる)

妻からすれば、夫が家にいるのに常に何かをスマホで確認していて、声をかけてもどこか上の空、という状態が続いていたわけです。大丈夫? 何かあった?と聞かれて、AIに仕事を頼んでいてと答えたら、は?という顔をされました。それはそうです(w

子どもが話しかけて来ても、ちょっと待ってとスマホを触っている場面も増えていました。ちょっと待ってのつもりが、確認が確認を呼んで5分、10分と経ってしまう。そのうちに子どもは私のことを無視し始めている。気づいたとき、まずいなと思いました。

## 仕事のオンとオフの切り替えができなくなった

Remote Controlを使いはじめてから、仕事とプライベートの境界がほぼ消えました。

オフィスに通っていた頃は、退勤して会社を出れば物理的に仕事から離れることができました。リモートワークになってからはそれが難しくなりましたが、それでもPCを閉じるという行為が一応の区切りになっていました。Remote Controlが加わったことで、PCを閉じることが仕事の終わりを意味しなくなりました。

スマホさえあればどこでもClaude Codeに指示が出せるので、常に今この時間にタスクを進められるのではないかという思考が頭の片隅に居座るようになります。料理をしながら、電車で移動しながら、お風呂やトイレに入りながら頭の中でAIへの仕事の割り振りを考えてしまい、体はオフでも、脳だけがずっとオンの状態です。

かつて、常に仕事のことが頭から離れないと悩んでいる管理職の話を聞いて大変だなと思っていましたが、まったく同じ状態でした。AIを活用して仕事の効率を上げているつもりが、結果的には24時間仕事モードから抜け出せない人間ができあがっていたのです。AIを使えば使うほど、AIのことを考える時間が増えていきます。これほど皮肉な話もないなと思います。

## これ、ソシャゲのリソース管理と同じじゃないか

AIが便利であればあるほど、使わないと損をしているという気持ちになる。それがAI疲れの根っこにある感情だと今は思っています。

ソシャゲでスタミナが回復するたびにアプリを開いてしまう感覚、わかる方はわかると思います。**回復したのに使わないのはもったいない、溢れさせる前にもう一周回そう、という焦りがずっとつきまとう。AIガチャを24時間効率よく引き続けて生産性を向上させなければ、という強迫観念に近いものがあります。** 使えるのに使わないことへの罪悪感とでも言いましょうか。

しかもソシャゲと違い、AIは実際に仕事が進んでいきます。ガチャを引くほど成果が出るなら、引き続けたくなるのは当然です。ゲームなら飽きたらやめられますが、仕事に使っているAIはそうはいかないので、余計に離れられなくなる。

これじゃソシャゲのリソース管理と同じじゃないか。そう気づいたとき、だいぶ楽になりました。

## Remote Controlの設定をオフにした

ある日夜中の2時に目が覚めて、またいつものようにスマホを手に取りかけた瞬間に、あ、これよくないなとはっきり自覚できました。

自分の意志でやめようとするのには限界があります。習慣になってしまったものは、意識の力だけでは止められない。今日は夜中に見ないと決めても、気づいたら手が動いている。そこで、設定を無効化することにしました。

設定を変えた翌日から、夜中にスマホを開こうとしても何も確認できなくなりました。最初の数日は確認しようとしたけど繋がらないという体験を何度か繰り返しましたが、どうせ繋がらないとわかってくると、確認しようとする衝動自体がだんだん薄れていきました。

## やめてから変わったこと

Remote Controlを無効にして一番変わったのは、夜の睡眠の質です。夜中に目が覚めてもスマホを開く理由がないので、そのまま眠りに戻れるようになりました。朝に感じる疲労感も、以前と比べると明らかに軽くなっています。

仕事とプライベートの切り替えも戻ってきました。PCを閉じたら今日の仕事は終わり、という感覚が復活しています。夕食の時間に家族の話をちゃんと聞けるようになった。それだけのことですが、自分にとってはかなり大きな変化です。子どもの今日こんなことがあったという話を、スマホを触らずに聞けるようになりました。

Claude Code自体は今も毎日使っています。PCの前に座って、ターミナルを開いて使う、元の形に戻っただけです。Remote Controlがなくても困らないし、むしろPCの前に座っているときだけ使うという制約ができたことで、仕事の時間に集中する感覚が戻ってきました。制約がパフォーマンスを下げるどころか、上げることになるとは思っていませんでした。

## AIの道具を道具として使うために

改めて振り返ると、Remote Controlでうまくいかなかったのは、機能の問題ではなく自分の使い方の問題だったと思います。

便利な機能があるなら最大限に活用しないともったいないという発想で使い始めましたが、道具は必要なときに使うから道具であって、常に稼働させ続けるものではありません。自動車を持っているからといって、24時間エンジンをかけたまま走り続ける人はいません。必要なときに乗って、使わないときは駐車してエンジンを切ります。

AIも同じで、使うべき時間に使い、使わないときは本当に切り離す、という線引きが必要です。特にRemote Controlのようないつでもどこでも使える機能は諸刃の剣です。便利さ反面、意識して使わないと生活のあらゆる時間に侵食して生活を破壊します。便利さと引き換えに、自分の時間のコントロールを手放してしまうことになりかねません。

設定を一つオフにするだけで、睡眠が戻って、家族との時間が戻ったのでそれでよかったです。Remote Controlが合っている人ももちろんいると思います。ただ、夜中にスマホを確認する習慣がついてしまったり、生活リズムが崩れてきたりしているなら、一度無効にしてみるのをおすすめします。
