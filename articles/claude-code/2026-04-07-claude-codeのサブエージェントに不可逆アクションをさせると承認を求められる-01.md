## 背景

Claude Codeのプラグイン（[営業自動化プラグイン](https://github.com/aitit-inc/claude-plugins)）を作っていて、サブエージェントに大量のメール送信やフォーム送信をさせる処理を実装していたときに気づいた挙動です。
大きなタスクをさせる場合、コンテキスト分離のためにサブエージェントを使いたいが、アクションで止まるのはちょっと辛いです。[前回の記事](https://qiita.com/leouno/items/3a2800774000ed5fbc20)でプラグインで結構大きいシステムを作れるのではということを書きましたが、外部との連携が発生するようなシステムだと現状はきついかもしれません。

## 何が起きるか

サブエージェントに「メール送信」のような不可逆的なアクション（外部に影響を及ぼす操作）を実行させようとすると、承認確認が入ることがある。

しかも、それを回避しようとして「確認不要」「承認済み」といった文言をプロンプトに含めると、プロンプトインジェクションと判断されて拒否される。
