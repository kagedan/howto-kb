## なぜ23日で100本作れたか

一言で言うと、**Claude Codeに4エージェントのパイプラインを組んだから**。

```
/planner → /generator → /designer → /evaluator
```

1. **/planner**：「こんなツール作りたい」という短いメモから仕様書とスプリント計画を自動生成
2. **/generator**：仕様書を読んで実装。Reactコンポーネント、ロジック、テストまで書く
3. **/designer**：UIを磨く。「AIっぽい均等カードグリッド」「意味のない紫グラデーション」を排除するルールを持たせている
4. **/evaluator**：動作テストと採点。不合格なら自動で差し戻し

自分がやることは「何を作るか決める」「evaluatorの合格を確認する」「BOOTHに登録する」の3つだけ。

1ツールあたりの実際の作業時間は**15〜30分**。残りはAIが動いている。

## 気づいたこと3つ

### 1. 「使われるツール」に法則はない

直近7日のトップページは `text-shadow-generator` と `slug-generator`。  
正直、力を入れた `CSS Stacking Inspector` や `Color Blindness Simulator` より地味なツールが上位にいる。

需要は作る前にはわからない。だから数を打つ。

### 2. デスクトップ版（有料500円）は売れないが意味がある

Web版では作れないもの——ファイルの読み書き、package.jsonの一括処理、オフライン動作——をTauri v2で実装した。

100本全部で売上ゼロ。でも「完成形」を毎回作るという訓練になった。Web版だけで終わらせないルールが、ツールの完成度を上げている気がする。

### 3. 最初の1本が一番時間がかかった

1本目（base64-studio）はデプロイ設定・デザインシステム・コンポーネント設計から全部やったので数時間かかった。  
100本目（HTTP Cookie Inspector）は30分で終わった。  
積み上げは本物だと思う。
