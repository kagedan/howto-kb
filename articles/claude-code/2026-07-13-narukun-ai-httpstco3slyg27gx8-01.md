# 1年。

これは私がClaude Codeでアフィ記事の設計を回してきた期間です。

その中で書かせた記事構成の下書きは、数千本。
本番投入した記事は数百本。

AI×アフィで一番決定的な要素。
それは「記事の骨格そのもの」でした。

ちなみに私はAI×アフィで累計2億円以上を作ってきました。

その全部の起点にあったのが「成約する記事の設計要素」です。

構成の骨組みが1つ変わるだけで、
同じ案件の成約率は3倍にも1/3にもなる。

これを私は数字で嫌というほど体感してきました。

結論から言います。

Claude Codeは記事の下書きを一瞬で出せる。
でも、下書きをそのまま出した記事は99%成約しない。

「Claudeに任せれば秒で終わる」と思っていた
1年前の私は、正直、甘かった。

その理由を、今から全部話します。

Claude Codeが最初に吐き出す記事の9割は「情報の並び」で終わる

Claude Codeは優秀です。

指示すれば、比較記事・レビュー記事・ランキング記事の
構成を一瞬で並べてくれます。

でも、そのままだと9割は「情報の並び」で終わる。

「〇〇のメリットとデメリット」
「〇〇のおすすめランキング」

読んだ瞬間に離脱される構成。
どれだけ検索順位が上がっても成約が伸びない。

日本語は整ってる。でも、読者の指がリンクに伸びない。

数百本を検証してわかったことがあります。

成約しない記事には「読者の今の状況」と
「読み終わった後の判断」が抜けている。

成約する記事の正体

成約する記事とは何か。

数百本を検証してふるいにかけた結果、
残ったのはたった1つの事実でした。

**成約する記事は「読者が判断を委ねたくなる記事」だ**

面白い記事じゃない。
わかりやすい記事でもない。

読み終わったときに
「この記事のおすすめでいいや」
と思ってもらえる記事。

その状態を作れる記事だけが、
コンスタントに成約を積み上げます。

その状態を作るための設計要素を、7つに絞りました。

## 1. 読者の「今の位置」を1行で言い当てる

「初心者の方へ」は誰の記事でもない。

これはClaude Codeで500本以上の記事を回して
一番はっきり出た結論です。

Claude Codeに「初心者向けに書いて」と頼むと、
必ずぼやける。

でも、「31歳・共働き・世帯年収780万・子ども1歳・住宅ローン残32年」
と条件を指定した瞬間、記事の解像度が別物になった。

具体例を出します。

NG:「これから資産形成を始めたい方へ」
OK:「毎月の家計はギリギリ黒字。
　　でも子どもが生まれてから急に将来が不安になった、
　　30代前半の共働き夫婦へ」

後者は当てはまる人が最後まで読みます。
最後まで読めば、成約導線を通ってくれる確率が上がる。

そのまま使えるプロンプトを置いておきます。

```
【プロンプト】
あなたはアフィ記事の設計者です。

以下の案件の読者を、
「たった1つの生活状況」に絞り込んでください。
・案件:【案件名・内容】

年齢、家族構成、世帯年収、生活の1シーン、
今抱えている不安を具体化してください。

そのうえで
「〇〇な状況の、〇〇なあなたへ」の形式で
記事の1行目候補を10パターン出してください。

抽象的な導入は禁止です。
```

## 2. 記事の中盤で「読者の未来」を映す

読者は案件情報を読みに来てるんじゃない。
「この案件を選んだ後の自分」を確認しに来てる。

でも、Claude Codeが最初に出す構成は
「商品スペック」から入ろうとします。

ここが、成約を落とす最大のポイント。

比較記事・レビュー記事の中盤に、
必ず「読者の未来」を映す章を差し込む。

NG:「〇〇の特徴とスペック」
OK:「〇〇を使い始めて3ヶ月後の、
　　平日の朝と休日の過ごし方の変化」

スペックじゃない。
その案件を選んだ後の生活が変わっている状態を、
具体的に描写する。

そこに読者は反応します。

未来を描くプロンプトはこれです。

```
【プロンプト】
あなたはアフィ記事の設計者です。

以下の案件について、
購入後3ヶ月〜半年の読者の日常を描写してください。
・案件:【案件名・内容】

スペック説明は禁止です。

「朝の〇〇」「休日の〇〇」「家族との会話」
のような具体的な生活シーンで、
案件を選んだあとに変わっている状態を描いてください。

そのまま記事の中盤章に貼れる形で
10パターン作ってください。
```
