## はじめに

Claude Codeで動画を作成してみようと思いついたのでやってみた。  
動画を作成するにあたり思いつくものとして以下のような方法がある。

1. パワーポイントのようなスライドとナレーションだけの解説動画
2. 口だけが動いている（リップシンク）の動画
3. もう少し発展させて、パラパラ漫画風の動画
4. wan2.2やLTX2.3などで動画作成

まずは簡単なはずなので、①のスライドをナレーションだけで動画を作成してみた。

### ①スライドとナレーションだけで動画作成

テーマはなんでもよかったのですが、せっかくなのでComfyUIについて解説してもらいました。

指示について

スライドはClaude Codeが適当にいい感じに作ってくれました。指示としては５分でComfyUIを説明するスライドを作成してください。的な感じだったと思います。

ナレーションについて

ナレーションについては最初は有料の音声は使いたくなかったので、無料で探しています。  
ナレーションは最初はGoogle翻訳を使ってやってくれましたが、機械音のような感じで違和感がありました。  
変更をお願いしたところMicrosoftのEdge TTSなるものを選んでくれました。これは結構いい感じでしたが、Youtubeにあげるつもりもなかったのですが、面白半分で聞いたところライセンスがグレーのようです。  
更に聞いていくとVOICEVOXというものがあるらしく、これはよく聞くずんだもんのやつでした。  
  
せっかくなので出来上がったものをYoutubeに置いてみます。  
これでComfyUIを使えるようにはなりませんが、概要は大まかにわかるようにはなっています。

AIに解説させた方が詳しいので、記載しておきます。

  

以下AIによる解説を元に修正したものを記載しています。

---

YouTube用の解説動画を、動画編集ソフトをほとんど触らずに作れないか——。  
そう思って、AIエージェント「Claude Code」に台本づくりから動画の書き出しまでを一通り任せてみました。題材は、いま人気の画像生成ツール「ComfyUI」の5分入門動画です。

結論から言うと、完成しました。この記事では、その手順をそのまま公開します。

## できあがったもの

これらを、すべてターミナル上の指示だけで用意しました。

## 全体の流れ

動画づくりを、Claude Codeと一緒に次の5ステップに分解しました。

1. 台本を書く
2. スライドを作る
3. ナレーション音声を作る
4. スライド＋音声を動画に合成する
5. サムネ・概要欄を整える

ポイントは、「人間がやると地味に面倒な作業」を全部スクリプト化してもらったこと。  
一度スクリプトにしておけば、文言を変えて何度でも作り直せます。

## 1. 台本を書く

まず「ComfyUIをこれから始める人向けに、5分でざっくり紹介する台本を書いて」とお願い。  
~~専門用語をかみくだく、1分あたりの文字数から尺を逆算する、といった条件も伝えました。~~

できあがったのは8セクション構成の台本。各セクションに「このスライドを表示」という指定と、ナレーション本文、一拍おく「（間）」まで入った状態です。

## 2. スライドを作る

スライドは、Claude Codeが勝手にやってくれて、PowerPointではなくコードで生成しました。  
Claude Codeに~~「台本に対応する1920×1080のスライドをSVGで作って」と指示すると、~~Pythonスクリプト（make\_slides.py）ごと用意してくれます。

（色が変えられるようになっているらしいが、やってはいない）  
文言や色を変えたいときは、スクリプトの変数を1か所いじって再実行するだけ。  
紫とティールのアクセントカラーも、変数で全スライド一括変更できるようにしてもらいました。

※動画にするときはSVGをPNGに変換します（日本語フォントを埋め込んで描画するスクリプトも用意）。

## 3. ナレーション音声を作る ＝ ここが一番の山場

ナレーションはAI音声合成（TTS）を使いますが、ここで重要なのがライセンスです。

最初、手軽なTTSで仮の音声を作って編集の流れを確認しました。ただ、公開・収益化する動画では「商用利用OK」の素材しか使えません。Microsoft Edge読み上げ系のTTSは商用利用がグレーなので、確認用にとどめます。

そこで本番は VOICEVOX（四国めたん）に差し替えました。クレジット表記をすれば商用利用OKです。  
台本のナレーション本文だけを抜き出してVOICEVOXに流し込み、セクションごとの音声ファイルを自動生成するスクリプトも作ってもらいました。

→ クレジット表記は概要欄に必ず入れる、という運用までセットで固めています。

## 4. スライド＋音声を動画に合成する

動画編集ソフトは使いません。ffmpegというコマンドラインツールで合成します。

Claude Codeに書いてもらったのは、こんな処理をするシェルスクリプト（build\_video.sh）：

1. 各スライドPNGに、対応するナレーション音声を重ねて、セクションごとの短いmp4を作る
2. 8本のmp4を順番に結合して1本にする

コマンド1回で final.mp4 が完成します。  
台本や音声を直したら、また同じコマンドを叩くだけ。「作り直しが一瞬」なのが、コードで作る最大のメリットでした。

## 5. サムネ・概要欄を整える

Youtube用にと指示したら、サムネイル画像の生成スクリプトと、YouTube概要欄のテンプレ（チャプター時間・ハッシュタグ・クレジット入り）も用意。  
再度Youtubeで問題ないか確認し（VOICEVOXのクレジットは入ってる？／チャプター時間は合ってる？）、抜け漏れを防ぎました。

## やってみてわかったこと

* **「面倒な作業のスクリプト化」とAIエージェントの相性は抜群**。一度仕組みを作れば、テーマ違いの動画も同じ流れで量産できる。
* **ライセンス確認だけは人間が主導すべき**。特にTTS音声は「商用OKか」をエージェントにも何度も確認させた。
* **動画編集ソフトのスキルがなくても、解説系の動画なら作れる**。スライド＋ナレーション形式と、コマンドラインツールの組み合わせが強い。

「動画を作る」という、一見コードと関係なさそうな作業も、分解すればほとんどがテキストとコマンドの仕事でした。  
Claude Codeのようなエージェントは、こういう“地味で繰り返しが多い制作作業”をまるごと引き受けてくれます。
