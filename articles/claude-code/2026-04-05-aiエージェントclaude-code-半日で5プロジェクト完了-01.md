**■ 記録**

2026年4月4日、土曜日。午後からの時間で以下の5つの作業を完了した。

1. Gmail整理（2アカウント、ラベル再設計、メルマガ配信停止6件）
2. Macストレージ整理（195GB回収）….ちょっと大袈裟
3. Notionでの個人事業管理環境構築
4. 自分が行ったことがある場所を表示する世界地図アプリの開発・公開
5. マルチデバイス作業環境の設計

全てClaude Code（AIエージェント）との協働で実施。

ここで共有したいのは「すごい」という話ではなく、AIエージェントが実行コストを下げると個人の生産性に何が起きるか、という実体験の記録。

**■ 従来のボトルネックは「手続き」だった**

5つの作業に共通していたのは、何をすべきかはわかっていたということ。

メルマガが不要なのは知っていた。配信停止が面倒だった。 ストレージに重複があるのは薄々気づいていた。確認が面倒だった。 Notionに管理環境を作りたかった。操作方法を調べるのが面倒だった。 旅行マップを作りたかった。プログラミングができなかった。

ボトルネックは「何をするか」の意思決定ではなく、「どうやるか」の実行手続き。

**■ CoworkからCodeへ——実感した「一段上」**

Claude Coworkを触ったとき、正直「めちゃくちゃすごい」と思った。ファイル整理や定型作業を自律的にこなしてくれる体験は、それまでのAI利用とは明らかに次元が違った。

そのCoworkの上をいく実感があったのが、Claude Code。

やりたいことに対してのレスポンスが、より的確にこちらの意図を捉えてくれる。複数ステップの作業を、途中で止まらず自律的に完走してくれる感覚。

プロンプトの出し方が上達したからかもしれない。でもそれを差し引いても、Codeとのやり取りには「ちゃんと伝わっている」という手応えがあった。

ITに多少馴染みのある人間だからこそ感じる差なのか、単なるマニアの感覚なのかはわからない。ただ、今回の5プロジェクトを半日で完了できた背景に、このCodeとの協働精度があったのは確かだ。

**■ 人間とAIの役割分担**

今回の分業はシンプルだった。

Gmail整理 → 自分：残す/消すの判断 / AI：ラベル取得・配信停止手続き ストレージ整理 → 自分：削除の最終判断 / AI：ファイル比較・重複検出 Notion構築 → 自分：構造の設計 / AI：操作・ページ作成 世界地図アプリ → 自分：デザイン方針 / AI：コーディング全般 PC環境設計 → 自分：現状の共有 / AI：構造化・フレームワーク化

人間＝判断・方針・データ。AI＝調査・手続き・実装。

この分業が成立するのは、AIエージェントが「指示を理解して、複数ステップの作業を自律的に実行できる」から。単なるチャットボットとは根本的に異なる。

**■ 実行コストが下がると「着手率」が上がる**

今回の5つは、どれも数週間から数ヶ月間「やらなきゃ」リストに入っていたもの。

着手できなかった理由は、実行コスト（時間・手間・心理的負荷）が判断コストを大幅に上回っていたから。

AIエージェントは、この実行コストを劇的に下げる。すると着手の心理的ハードルが下がり、「やらなきゃ」が「やった」に変わる速度が上がる。

これは単なる効率化ではない。意思決定と実行の間にあった摩擦が消えることで、個人が動かせるプロジェクトの数自体が変わる。

**■ 個人事業への示唆**

組織なら、実行は部下やチームに委任できる。個人事業では、判断も実行も全て自分。

AIエージェントは、個人事業主にとっての「実行チーム」になりうる。

今日の体験はまさにそれ。判断だけに集中できる半日。それだけで5つのプロジェクトが動いた。

世間では、AIに任せて生産性が指数関数的に飛躍という話も出ていた。昨日の作業は決して洗練はされてないが、まさしくこのことかもしれないと思った。

**■ まとめ**

AIエージェントの本質的な価値は「実行コストの削減」。

判断は人間、手続きはAI。この分業が最も自然に機能する。

CoworkからCodeへ。ツールが進化するたびに、人間が動かせるプロジェクトの射程が広がる。

個人事業主にとって、AIエージェントは「一人チーム」を可能にする存在と実感。
