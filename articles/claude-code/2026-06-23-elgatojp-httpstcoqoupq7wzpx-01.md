Stream Deck 7.4から、AIアシスタントとのMCP連携が可能になりました。音声、テキスト、ライブイベントを介してAIがStream Deckのアクションを実行できます。NVIDIA G-AssistやClaude Desktopとの接続方法を、順を追って解説します。

## MCPとは

MCP（Model Context Protocol）は、AIツールがアプリやサービスに接続するためのオープンスタンダードです。

USB規格に例えると分かりやすいです。USB登場以前は、デバイスごとに専用ケーブルが必要でした。USBはすべてのデバイスに対応する単一の規格を生み出しました。MCPはAIツールに対して同じ役割を果たします。AIアシスタントがアプリごとに独自の接続方法を必要とする代わりに、MCPは共通のプロトコルで通信する手段を提供します。

Stream Deck 7.4で、Stream DeckはそのMCP対応デバイスに加わりました。AIが入力を担い、Stream Deckがアクションを実行します。

## MCPで出来ること

MCPは、複数のアクションをまとめて実行したいとき、または両手を塞がずに操作したいときに真価を発揮します。

## NVIDIA G-Assist（音声操作）

「配信の準備をして」と声をかけるだけで、G-AssistがOBSの起動、シーンの設定、照明の点灯、音楽の再生をまとめて実行します。キーボードやマウスから手を離す必要はなく、画面から目を離す必要もありません。

## Aitum（ライブイベント連動）

配信中のイベントが自動でトリガーになります。レイドが来ればシーン切り替え・照明の変更・ウェルカムサウンドが一斉に動き、チャンネルポイントの使用で音声エフェクトが発動し、サブスクでDiscordへのシャウトアウトが始まります。あなたが何も操作しなくても、リアルタイムで実行されます。

MCPはオープンスタンダードなので、Claude DesktopやChatGPTなどのテキストベースのAIツールも同様に接続できます。

# MCPアクションの設定

## Stream Deckでの設定

- Stream Deckアプリを開きます
- 環境設定 → 一般 タブを開きます
- MCP Deck を有効にします
有効にすると、MCP Actions という専用プロファイルが作成されます。このプロファイルに配置したアクションだけがAIツールから見えます。他のプロファイルのアクションはAIからアクセスできないため、AIが操作できる範囲を自分でコントロールできます。

注： 環境設定に「MCP Deck」が表示されない場合は、Stream Deck 7.4以降がインストールされているか確認してください。最新版はこちらからダウンロードできます。

## アクションへの説明を追加する

各アクションには、AIに「このアクションが何をするか」を伝える説明フィールドがあります。ここが設定の要です。AIはこの説明をもとに、どのアクションをトリガーするかを判断します。

アクションを選択し、AIアイコンをタップして説明を入力します。

例：

「OBS Studioを起動します。配信や録画を開始するときに使用してください」

「Elgato Key Lightを点灯します。カメラの前に出るとき、または配信を始めるときに使用してください」

「Spotifyを起動します。配信のBGMをかけるときに使用してください」

この説明があれば、「配信の準備をして」の一言だけで十分です。AIがすべての説明を読み取り、OBS・Key Light・Spotifyをまとめて起動します。組み合わせを事前に設定する必要はなく、AIが判断して処理します。

これがMCPとキーの単発操作との本質的な違いです。個々のアクションと説明を一度設定するだけで、同じアクションセットをさまざまな指示で使い回せます。

## Node.jsのインストール

Elgato MCP ServerはNode.js上で動作します。まだインストールしていない場合は、nodejs.orgからLTS版をダウンロードしてインストールしてください。インストーラーを実行して画面の指示に従うだけです。

インストール後、ターミナル（WindowsはPowerShell）で確認できます：

node -v

バージョン番号が表示されればOKです。
