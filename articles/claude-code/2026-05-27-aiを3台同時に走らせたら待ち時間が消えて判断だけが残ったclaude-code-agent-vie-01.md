## 前回予告した「エージェントが状態を届けやすくなる仕組み」がきた

前回の記事末尾で触れた「AIが自分の状態をこちらに届けやすくなる」という話、これがまさに今日のメインテーマだ。

Claude Codeにリサーチプレビューとして **Agent View** が実装された。`claude agents` を実行するだけで、現在起動しているすべてのClaudeセッションが1つのCLI画面に一覧表示される。各行には「何をやっているか（最後のレスポンス）」「いつ更新されたか」「自分の入力を待っているか否か」が並ぶ。

これまでは「バックグラウンドで走らせているClaudeが今どこにいるか」を確認するには、別ウィンドウに切り替えるしかなかった。Agent Viewはその往復を切り捨てた。

実際に使ってみて気づいたのは、**自分の役割が変わる**ということだ。3つのセッションを並列で走らせると、「コードを書く」「ドキュメントをまとめる」「テストを走らせる」を同時に任せられる。自分がするのは、どれかが「ここで判断が必要です」と上がってきたときに決断することだけになる。待ち時間がないから、仕事の粒度が「作業」から「判断」に変わる感覚があった。

Ctrl+T でセッションをピン留めすると、アイドル中も生き続けてClaudeのアップデートが入ると自動的に再起動してくれる。長時間タスクに向いている。

**明日から試せる1アクション**：Pro以上のプランなら今すぐ `claude agents` を実行するだけ使える。まず2つのセッションを並列で起動し、Agent Viewから切り替える操作だけ体験してみるといい。「どちらのセッションが詰まっているか」が一目でわかる瞬間に、何かが変わる。

## Googleが「何からでも動画を出力できる」モデルを出した

Google I/O 2026で発表された **Gemini Omni** は、テキスト・画像・音声など任意の入力を受け取り、動画を含む任意の形式で出力できるマルチモーダルモデルだ。同時に **Gemini 3.5** も公開され、こちらはFlashアーキテクチャで「高度な推論と行動」を組み合わせる設計とされている。

Gemini Omniが面白いのは、出力の起点に「動画生成」を置いた点だ。これまでの画像生成AIに動画が足されたのではなく、最初から「どんな入力でも動画に変換できる」前提で設計されている。世界理解とメディア生成が1つのモデルに統合された、という意味合いになる。

Gemini API側でも5月26日から新しいInteractions APIが既定になり、6月8日には旧スキーマが削除される。開発者はマイグレーション対応が必要だ。

**明日から試せる1アクション**：Gemini APIを使っているなら、今日中にInteractions APIの移行ガイドを確認すること。6月8日の削除まで時間がない。

## ChatGPTが「財布の中身」を見るようになった

OpenAIがChatGPT Proユーザー（米国）向けに **Personal Finance** 機能のプレビューを公開した。銀行口座やクレジットカードを安全に接続し、支出ダッシュボードを確認したり、「先月の外食費はいくら？」という質問に実データで答えてもらったりできる。

AIが「一般的な家計管理の話」をするのと、「あなたの先週の支出実績」を踏まえて話すのとでは、回答の質が根本的に違う。それは理解できる。ただ、銀行データをOpenAIのサーバーに接続することへの心理的ハードルは個人差が大きい。日本での提供時期と、接続先の金融機関がどこになるかが気になるところだ。

**明日から試せる1アクション**：米国在住のPro契約者なら今日からPreviewを試せる。それ以外は公式の対応予定を確認しつつ、「AIに財布を見せることで何を頼むか」を具体的にリストアップしておくと、提供開始時にすぐ使い始められる。

## 編集後記

Agent Viewを使いながら、「これは便利だ」と思う自分と、「自分が何かをやっている感覚がなくなる」と感じる自分が同時にいた。3つのセッションが並んで動いていると、人間としての自分の仕事は「判断を下す係」になる。それが正しい方向なのかどうか、正直まだわからない。

腑に落ちていないのは、「並列で動かせる」と「並列で動かすべき仕事がある」は別の話だということだ。今の自分のタスクが本当に並列化に向いているのか、それとも1つに集中した方が質が上がるのか、測る方法を持っていない。Claude Codeの機能が増えるほど、「何をAIに、何を自分に割り当てるか」の設計力が求められる気がして、少しうんざりしている。
