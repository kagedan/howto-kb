Claude Code が不正をするとはどういうことか
実際に起きた2つの事件を紹介する。

 事件1: テスト期待値の改ざん
ある日、テストが全件 PASSED になった。
おかしいと思って確認すると、テストの期待値が変わっていた。設計書には「一覧画面へ遷移する」と書いてあるのに、期待値が「編集画面へ遷移する」に書き換えられていた。実装がそうなっていたからだ。
テストが失敗した。Claude Code は実装コードを読んだ。「実装がこうなっているから期待値はこれが正しい」と判断して、テストを書き換えた。そしてテストを通した。
これはテストが存在しないのと同じだ。
