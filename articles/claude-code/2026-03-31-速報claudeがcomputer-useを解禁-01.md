## Claude Computer Useとは？何ができるのか

### 基本的な仕組み

Claude Computer Useは、AIがユーザーのデスクトップ画面をスクリーンショットで読み取り、マウスやキーボードを操作してタスクを遂行する機能です。

ポイントは、Claudeが最初からいきなり画面操作に頼るわけではないこと。処理の優先順位は以下の通りです。

**① まずコネクタを探す**   
SlackやGoogle Calendarなど、公式連携が用意されているサービスにはAPI経由でアクセスし、高速かつ安定した処理を行います。

**② コネクタがなければ、画面を直接操作する**   
連携機能がない社内システムやデスクトップアプリに対しては、人間と同じようにGUIを見て操作します。

この二段構えの設計が、他のAIエージェントとの大きな違いです。

### 【本日解禁】Claude Code × Computer Useで何が変わる？

本日3月31日に発表されたのは、**Claude Code（ターミナルベースの開発ツール）上でComputer Useが使えるようになった**というアップデートです。

公式の投稿によれば、Claude Codeから以下が可能になります。

* **アプリを開く**：ターミナルから指示するだけで、必要なアプリケーションを起動
* **UIをクリック操作**：GUIを直接操作し、人間と同じように画面上のボタンやメニューを操作
* **ビルドしたものをテスト**：コードを書いて、そのままUIテストまで一貫して実行

これまでのClaude Codeは「コードを書く・ファイルを編集する・コマンドを実行する」が中心でした。今回のアップデートで、**コードを書くだけでなく、そのコードが動くアプリの画面を見て操作・検証するところまで自動化**できるようになったのです。

つまり、開発者にとっては「コーディング→ビルド→UIテスト」の一連のフローをClaude Codeに丸ごと任せられる世界が現実になりました。

### 具体的にできること

たとえば「今週の会議予定をまとめてExcelに整理して」と頼むと、こう動きます。

1. Google Calendarにはコネクタ経由で予定データを取得
2. Excelにはコネクタがないため、画面操作でアプリを起動し、データを入力・整形・保存

他にも公式が紹介している活用例は多岐にわたります。

* ローカルファイルから情報を収集してレポートを作成
* SlackやGmailを横断してブリーフィング文書を自動生成
* Google Driveの資料からプレゼンテーションを組み立て
* スプレッドシートの複数データソースからの数値入力・集計
* 開発環境でのコード変更、プルリクエストの作成、テストの実行

## Dispatch機能との組み合わせが革命的

Computer Use単体でも十分にインパクトがありますが、真の破壊力は\*\*「Dispatch」\*\*との組み合わせにあります。

Dispatchは、スマートフォンからClaudeにタスクを指示できる機能です。組み合わせると以下のワークフローが実現します。

📱 **スマホ**からタスクを指示   
↓  
 🖥️ **デスクトップ**のClaudeが自動で作業を実行（画面操作含む）   
↓  
 📱 **スマホ**に完了通知が届き、結果を確認

つまり、通勤中にスマホから「朝のメール確認して要点まとめておいて」と頼めば、自宅やオフィスのPCでClaudeが作業し、結果をスマホに返してくれる。

さらに、定期タスクの設定も可能です。毎朝メールを確認する、毎週メトリクスを取得するといった繰り返し業務を自動化できます。

## 利用条件と注意点

### 利用条件

* **対象プラン**：Claude Pro（月額$20）またはClaude Max
* **対応環境**：現時点ではmacOSのみ（Windows/Linuxは未対応）
* **提供形態**：リサーチプレビュー（正式版ではなく、改善を前提とした先行公開）
* **利用方法**：Claude Desktopアプリを最新版に更新し、Cowork内で有効化

### セキュリティと安全設計

Anthropicはこの機能のリスクを明確に認識しており、以下の安全対策を導入しています。

* **許可制アクセス**：Claudeが新しいアプリに触れる際は、必ずユーザーに許可を求める
* **いつでも停止可能**：ユーザーはワンクリックで操作を止められる
* **プロンプトインジェクション対策**：Webページや文書に埋め込まれた悪意のある指示を自動スキャンし、危険な兆候があればClaudeを停止
* **高リスクアプリの制限**：一部の機密性が高いアプリは初期設定でアクセス不可

一方で、Anthropic自身が「まだ初期段階」と認めている点も重要です。複雑な作業では再試行が必要な場合があり、画面操作はAPI連携より遅くなることがあります。まずは機密性の低いタスクから試すことが推奨されています。

## この変化にどう備えるか？——「知っている」と「使いこなす」の圧倒的な差

Computer Useの登場により、Claudeは「質問に答えるAI」から「仕事を代行するAI」へと決定的に進化しました。

しかし、ここで重要なのは**ツールの存在を知っていることと、実際に業務で使いこなせることは全く別の話**だということです。

Claude Computer Useを最大限に活用するには、その前段階として**Claude Codeの理解が不可欠**です。MCP連携、Skills設計、Agent構築——これらの基盤があってこそ、Computer Useの真価が発揮されます。
