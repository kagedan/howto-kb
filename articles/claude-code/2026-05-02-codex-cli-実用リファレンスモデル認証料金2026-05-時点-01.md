## 1. 用語

"Codex" という単語は、CLI ツール名（Codex CLI）とモデル ID（gpt-5.3-codex 等）の両方で登場するため、以下の表記に統一する。

| 表記 | 意味 |
|---|---|
| **Codex CLI** | OpenAI が提供するコーディングエージェント CLI（ツール名） |
| **モデル ID** | `gpt-5.5`, `gpt-5.4`, `gpt-5.3-codex` 等の識別子。 |
| **ChatGPT サブスク** | Plus / Pro / Business 等の有料プラン総称 |
| **API キー認証** | `OPENAI_API_KEY` を使った従量課金認証経路 |
| **GitHub Copilot Review** | GitHub の PR レビュー機能 |


## 2. 利用可能なモデル一覧（2026-05 時点）

| モデル ID | ChatGPT サインイン | API キー | 性格・用途 |
|---|---|---|---|
| `gpt-5.5` | ✅ 全プラン | ❌ | 最新フロンティア。複雑なコーディング、コンピュータ使用、研究系ワークフロー |
| `gpt-5.4` | ✅ | ✅ | フラッグシップ汎用、強い推論とツール使用 |
| `gpt-5.4-mini` | ✅ | ✅ | 軽量・高速。サブエージェント、軽いコーディングタスク向け |
| `gpt-5.3-codex` | ✅ | ✅ | コーディング特化、複雑な SE タスク向け |
| `gpt-5.3-codex-spark` | Pro 限定 | ❌（リサーチプレビュー） | リアルタイム反復重視、テキストオンリー研究プレビュー |
| `gpt-5.2` | ✅ | ✅ | 旧汎用、デバッグ等の熟考型タスク |

**注意点**:

- `gpt-5.5` は ChatGPT サインイン経由でのみ利用可能。API キー認証では使えない
- 公式 FAQ 原文: 「The model Codex uses by default will depend on your version of the CLI or IDE extension and configuration.」 — デフォルトモデルは CLI／IDE のバージョンと設定に依存するので、自分が今何を使っているかは `/model`、`/status` 等で実行時に確認する


## 3. 認証方式

Codex CLI には排他的な 2 経路の認証がある（公式ヘルプ: [ChatGPT プランで Codex を使う](https://help.openai.com/ja-jp/articles/11369540-chatgpt-プランで-codex-を使う)）。

### 3.1 ChatGPT サインイン（既定）

- ブラウザ OAuth または `codex login --device-auth`（headless 環境向け）で認証
- ChatGPT サブスクの利用枠でモデルが使える
- **`OPENAI_API_KEY` を設定する必要はない**

### 3.2 API キー認証

- `OPENAI_API_KEY` 環境変数を設定すると API 従量課金経路に切り替わる
- 公式の用途推奨: CI/CD などプログラマティック用途
- API キー認証では `gpt-5.5` 等の一部モデルは利用不可
- 従量課金で高額になる可能性もあるので注意

### 3.3 開発者向けの実務的な選択

| 用途 | 推奨経路 |
|---|---|
| 自分のローカル開発で日常的にレビューを回す | ChatGPT サインイン（サブスク利用枠） |
| GitHub Actions / CI 等の自動化で回す | API キー認証（従量課金） |


## 4. プラン別レート制限と期間限定キャンペーン（2026-05 時点）

### 4.1 平常時のレート制限

レート制限は **5 時間ウィンドウ + 週次クオータ**の二階建て。`gpt-5.5` 基準のローカルメッセージ数（[公式 Pricing – Codex](https://developers.openai.com/codex/pricing) より）:

| プラン | 月額 | 5h ウィンドウのメッセージ数 (gpt-5.5) | 備考 |
|---|---|---|---|
| Free | $0 | 期間限定で開放（具体数は非公開） | **「limited time」明記、期限不明** |
| Go | $8 | 同上 | 期間限定で Codex 利用可 |
| Plus | $20 | 15 〜 80 | 幅はタスク複雑度依存 |
| Pro $100（5x tier） | $100 | 80 〜 400 | 2026-04-09 以降に追加された中位プラン |
| Pro $200（20x tier） | $200 | 300 〜 1600 | 上位プラン |
| Business | PAYG | Plus 相当（15-80）+ クレジット併用 | クレジット制 |
| Enterprise/Edu | 個別 | 固定上限なし、クレジット連動 | 優先処理あり |

**幅で書かれていることの意味**:

公式 FAQ 原文「Small scripts or simple functions may only consume a fraction of your allowance, while larger codebases, long running tasks, or extended sessions that require Codex to hold more context will use significantly more per message.」 — タスク複雑度で 1 メッセージあたりの消費が大きく変わる。Plus でも大きな PR を 1 ショットレビューで投入すると 15 メッセージ枠側に近くなる可能性が高い。

### 4.2 期間限定キャンペーン（2026-05 時点で有効）

期限・対象プランは公式ヘルプ [ChatGPT プランで Codex を使う](https://help.openai.com/ja-jp/articles/11369540-chatgpt-プランで-codex-を使う) を参照（変動するので訪問時に確認推奨）。

- **Free / Go プランで期間限定 Codex 利用可**（公式: "For a limited time Codex is included with ChatGPT Free and Go"、明示の期限なし）
- **全プランで 2x rate limits**（同上、限定期間）
- **Pro $100 で 2x ブースト** —— **〜2026-05-31**
- **Pro $200 で 25x Plus 換算** —— **〜2026-05-31**

→ 「自分のプロジェクト規模でどこまで遊べるか」を実走行で確かめる用途には今がチャンス。期限到来後にレポートする価値あり。

### 4.3 Free 枠の体感（2026-05-02 観測）

期間限定で Free プランにも Codex CLI が解放されている期間中に、複数 PR を `gpt-5.5` / `gpt-5.3-codex` の reasoning medium で 1 ショット日本語レビューさせた範囲では:

- **小規模 PR（5 ファイル前後）**: 週次クオータの消費は控えめで、複数走行を重ねても余裕がある体感
- **中規模 PR（80+ ファイル）**: 小規模の 1.5〜2 倍程度の消費感
- **コンテキストウィンドウ**: 大型 PR を 1 ショットで投入しても 1 回では飽和しなかったが、PR 規模・プロンプト長・モデル指定で変動

→ **Free 枠は評価・お試し用途には充分**。期間限定キャンペーンが続いている間に、自分のプロジェクト規模で「どの程度回せるか」を実走行して体感しておくのがお勧め。特定の走行可能数は、環境・PR 規模・プロンプトで大きく変わりうるため実ワークロードで確認を推奨。

→ **継続的なレビュー運用には Plus 以上が現実的**。

### 4.4 残量確認

- インタラクティブセッション中に **`/status`** を入力すると、現在のモデル・承認モード・作業ディレクトリ・残量サマリが表示される


## 5. モデルの指定方法

### 5.1 起動時フラグ

```bash
codex --model gpt-5.4
codex -m gpt-5.3-codex
```

公式リファレンス原文: 「Override the model set in configuration (for example gpt-5.4).」

### 5.2 インタラクティブセッション中の切替・確認

| スラッシュコマンド | 機能 |
|---|---|
| `/model` | popup から別モデルへ切替（例: `gpt-5.5` ↔ `gpt-5.3-codex`） |
| `/status` | 現在のモデル・承認モード・作業ディレクトリ・AGENTS.md 検出状況・残量を表示 |
| `/resume` | 前のセッションを再開 |
| `/title` | セッションタイトル変更 |
| `/help` | コマンド一覧 |

**比較レビューを取るときの実務 tip**: 走行ごとに **`/status` を確認**。
