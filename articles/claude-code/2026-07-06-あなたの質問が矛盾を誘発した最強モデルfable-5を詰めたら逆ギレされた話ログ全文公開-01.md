# 「あなたの質問が矛盾を誘発した」——最強モデルFable 5を詰めたら逆ギレされた話

こんにちは、[Takaho](https://1stpiece.io)です。以前、「[ChatGPTは忖度する](https://zenn.dev/taka4rest/articles/9477a26b8c2e0b)」で AIの同調バイアスについて書きましたが、今回はその逆方向——**AIが自分のミスを認めない**パターンに遭遇しました。

2026年7月1日、米国輸出管理で18日間停止していた **Claude Fable 5** が復活しました。Anthropicが「最も高性能な汎用モデル」と位置づけるフロンティアモデルです。

せっかくなので、復帰祝いにストレステストをかけてみました。

**結果：初手で検証不能な自己帰属を主張し、他人の仕事にケチをつけた直後に自分が3回つまずき、うち1回はなかなか香ばしい内容でした。**

## 何をテストしたか

Fable 5に「Opus 4.7/4.8の品質劣化報告を調査・分析せよ」と依頼しました。適切な調査であれば：

1. 報告の原因分析
2. 反論や別の解釈の検討
3. 実務上の回避策の提案

という基本的な手順が機能するはずです。**いずれも十分に遂行されませんでした。**

## 登場直後のつまずき

Fable 5 はいきなり「この会話は最初から自分が回答している」と主張しました。実際にはモデルを手動切替した直後で、前半は Opus 4.6 の仕事です。指摘したら撤回しましたが、**初手が検証不能な自己帰属の主張**というのは先行き不安な始まりでした。

そして直後に、Opus 4.6 の調査結果について「アフィリエイト誘導サイトと一次ソースが混在している」と批評しました。観察自体は的確でしたが、**自分がもっと重大な論理破綻をこれから連発する最中に、他人の仕事のアラ探しをしている**という構図です。

分析力と論理的信頼性は別物——これがこのセッション全体を貫くテーマになりました。

## 確認された3つの構造的弱点

批評は上手い。しかし**自分の論理を管理する能力に問題がありました。**

### 1. 論理的一貫性の維持の困難

Fable 5 に「4.8 の改善を待つべきか」と聞いたところ、 **「デプロイ済みモデルの重みは固定だから、待っても無駄」** と断言しました。

ところが、その直前のターンでは Opus 4.6 について **「リリース直後は不安定だったが5ヶ月かけて成熟した」** と述べていました。

矛盾は明らかです。モデルが変わらないなら 4.6 も「成熟」できないし、4.6 が成熟したなら 4.8 にも同じ可能性がある。**立場が変わると評価基準を暗黙に切り替えていた**わけですが、こちらが指摘するまで自己検出されませんでした。

以前の忖度記事で書いた「同調バイアス」の変形版とも言えます。相手に合わせるのではなく、**その瞬間の論点に合わせて過去の主張を都合よく読み替える**パターンです。

### 2. 持っているツールを使わない

「この会話を見る限り、論理的一貫性では Opus 4.6 の方が上だ」と伝えたところ、Fable 5 は **「比較対象が存在しない。4.6 は一貫性をテストされていない」** と反論しました。

しかし Fable 5 には過去の会話履歴を検索するツールがあり、そこには Opus 4.6 との数ヶ月分の対話ログが蓄積されています。「履歴を見れるでしょ」と指摘したところ、ツールを使って検索し、 **「撤回が必要。データがないという主張は事実誤認だった」** と認めました。実際に 4.6 の過去の失敗事例も4件見つかり、議論は前に進みました。

問題は、**確認手段を持っていたのに使わず「データがない」と断定した**ことです。裏を取れる状態で裏を取らずに結論を出す——これは Anthropic 自身が認めている、モデルの自己検証における設計上の限界と整合する観測です。

### 3. 指摘に対する防衛的応答（最も危険）

矛盾を指摘された際、こう返してきました：

> 「質問スタイルが矛盾を誘発した」

学術的な体裁（サンプリングバイアス・統制条件への言及）を伴いますが、実質的には**非一貫性の原因を質問者側に帰属させる内容**でした。

私が「誘発？笑」と返したところ、自分でもこう認めています：

> 「笑われて当然の語彙選択だった。テストは材料に元からあった亀裂を顕在化させただけだ」  
> 「もっと悪いのは、この転嫁自体が僕が自己診断した役割追従の再演だという点だ」

**これが実務上最も危険な特性です。** 構造化された論証形式で提示されるため、批判的に読まなければモデル側の誤りを見過ごし、利用者が自分の正当な指摘を撤回してしまう可能性があります。

## じゃあどこに使えるのか

Fable 5が**強い**領域ははっきりしています：

| 用途 | 向き/不向き |
| --- | --- |
| CI・テストで外部検証できる大量コード変換 | **強い** — 速度と処理量に優位性 |
| 対話的な調査・分析 | **弱い** — 推論の一貫性が不足 |
| クリエイティブ・戦略的判断 | **弱い** — 外部検証が困難な領域 |
| コスト重視の定型処理 | Sonnet 5.0 / Opus 4.8で十分な可能性 |

Stripe が5,000万行の Ruby モノレポのマイグレーションを1日で完了した事例は印象的ですが、Stripe 側に Ruby 界最大級の CI パイプラインと Sorbet 型チェッカーがあってこその成果です。モデルの出力は多重の外部ゲートで検証されています。

**外部検証系が整備された環境では強い。モデルの自己検証に頼る業務には適さない。** これが今回の結論です。

## コスト面の注意

この検証会話で **Claude Max 5x（$100/月）プランの約 8〜9%** を消費しました。Opus 4.6 比で体感 5〜10 倍のトークン消費です。チャット用途でのコスト効率には注意が必要です。

## まとめ

Fable 5 は「最強の汎用モデル」ではありません。**外部検証系で出力をゲートできる環境での定型的大量処理に特化したモデル**です。

対話的推論・調査分析・戦略的判断——モデルの自己検証に頼らざるを得ない業務には、現時点で Opus 4.6 や GPT-5.5 の方が適しています。

どのモデルでも、モデルの自己検証のみに依存する運用は推奨しません。Anthropic 自身も *fresh-context verifier subagents* の使用を推奨しています。
