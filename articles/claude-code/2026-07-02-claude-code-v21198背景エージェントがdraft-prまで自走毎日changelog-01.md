## 背景エージェントが draft PR まで作るようになった

:::note info
対象読者: `claude agents` で背景エージェントを並走させて開発している人
:::

**worktree での作業を終えた背景エージェントが、確認待ちで止まらなくなりました。** コードの変更が終わると、commit・push して draft PR を開くところまで自分でやります。これまでは完了時に確認を求めて止まっていた作業が、レビューできる PR の形で戻ってきます。

あわせて通知フックが増えました。入力待ちのエージェントは `agent_needs_input`、完了したエージェントは `agent_completed` で `Notification` フックを発火します。別ターミナルで走らせた背景エージェントの状態を、フック経由で拾えるようになりました。

背景エージェント周りの安定性修正も同時に入っています。完了・再開後もタスクパネルが「Running」で固まる問題、macOS で agents ビューを開いていると 52 秒ごとに「Reconnecting…」が出る問題が直りました。`claude --bg` を `--print` / `-p` と併用するとアタッチ不能なセッションが黙って作られていた不具合も、起動時に弾くようになりました。

## Explore とサブエージェントの底上げ

Explore エージェントが haiku 固定をやめ、メインセッションのモデルを継承するようになりました(opus 上限)。コードベースを横断検索する Explore が、本体と同じモデルで動くようになります。

サブエージェントとコンテキストのコンパクションも、拡張思考(extended thinking)設定を継承するようになりました。委譲したタスクや要約処理で思考の深さが揃い、出力品質が上がります。

## `/agents` ウィザードが廃止

:::note alert
`/agents` ウィザードが削除されました。サブエージェントの作成・管理は Claude に頼むか、`.claude/agents/` を直接編集する運用に変わります。
:::

ウィザードのフォーム操作でエージェントを組んでいた人は、v2.1.198 で `/agents` が使えなくなりました。`.claude/agents/` 配下の定義ファイルは従来どおり有効なので、既存のサブエージェントはそのまま動きます。
