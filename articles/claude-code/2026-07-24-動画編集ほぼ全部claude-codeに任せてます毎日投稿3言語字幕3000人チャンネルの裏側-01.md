私は動画編集ソフトをほぼ開きません。

20分の動画を収録したあと、私がやることは「文字起こしのチェック」と「完成品の確認」くらいで、無音カット・字幕・翻訳・概要欄・投稿用データの管理まで、残りはすべて手元のMacの中でコードが処理しています。

はじめまして、あるいはいつもありがとうございます。「KENさんの金銀資産防衛研究室」というYouTubeチャンネルを運営しているKENさんです。金や銀といった現物資産を軸に、資産防衛の話をしているチャンネルで、登録者は約3,000人。撮影も編集も投稿も、すべて1人でやっています。

1人でやっている、と言いましたが、正確には「1人と、コードと、AI」です。

このチャンネルの動画には、日本語・英語・中国語の3段字幕が全編に入っています。概要欄も3言語。動画ごとにX用の短縮版も作っています。これを毎回、編集者に外注したり私がタイムラインに向かって手作業でやっていたら、とても続きません。実際続けられなくなりかけたので、制作フローそのものを作り直しました。

この記事は、その制作フローの「裏側の全部」です。

### なぜこの記事を書くのか

「AI×動画」のノウハウは世の中にたくさんあります。ただ、その多くは「AIに台本を書かせ、AI音声で読ませて、動画を量産する」という方向のものですね。

私がやっているのは逆です。**動画の中身は100%自分**。自分の声で、自分の考えを話します。AIとコードに任せるのは、その後ろにある「編集・字幕・翻訳・管理」という、時間だけを溶かしていく工程です。

5年ほどチャンネルをやってきて実感しているのは、視聴者はテーマではなく「人」につくということです。AI音声のコンテンツには、はっきりと拒否反応があります。だからこそ、人がやるべき部分と、コードに任せていい部分の線引きが大事になるわけです。

### 全体像だけ、先にお見せします

私の制作フローは、大きくこの流れで動いています。

1. **収録** — カメラの前で話すだけ。ここだけが「人間の仕事」
2. **ジェットカット** — 無音部分の自動カット（コード）
3. **文字起こし** — ローカルAIで全文書き起こし（コード）
4. **目視チェック** — 誤認識の修正と、コンプライアンス上の判断（人間）
5. **3言語字幕・概要欄の生成** — 翻訳・整形・チャプター（AI＋コード）
6. **焼き付け・完成** — 字幕・ロゴ入りの完成動画を出力（コード）
7. **投稿と記録** — 投稿履歴はローカルのデータベースで一元管理（コード）

20分の動画の場合、機械が処理している時間は合計15分ほど。その間、私は放置しています。人間が手を動かすのは、実質的に4番のチェック工程だけです。

### この記事の差別化ポイントを、正直に書きます

このジャンルの記事にありがちな「理論上できるはず」の話ではありません。**いま実際に稼働していて、登録者3,000人のチャンネルの動画がこのフローから出てきている**、そのリアルの裏側です。使っているツール名、設定した数値、失敗してハマった箇所まで、そのまま書きます。

そして、もうひとつ正直に書くと——**この記事に出てくるツール名や手法は、1年後には古くなっている可能性があります。** AIの世界は動きが速いので、それは避けられません。

それでも私がこの記事に値段をつけて出すのは、ツール名の下にある**設計思想**は古くならないと考えているからです。

* **フローはコードに任せ、毎回AIに丸投げしない**こと（再現性とコストの話）
* **データをすべてローカルに置く**こと（これは資産防衛チャンネルらしく言えば、データの「現物保有」です）
* **人間のチェックポイントを、あえて消さずに設計に組み込む**こと

ツールが入れ替わっても、この3つの原則は残ります。というより、ツールが入れ替わるたびに、この原則を持っている人だけがフローを組み直せる、と言った方が正確かもしれません。

### 有料部分の内容

* 制作パイプラインの全工程（使用ツール・実際の設定値・処理時間の実測つき）
* 無音カットの具体的なパラメータ（何dB基準で、何ミリ秒の無音を切るか）
* ローカル文字起こしの構成と、実際にハマった落とし穴3つ
* AIとの「分業設計」— どこまで任せ、どこで人間が止めるか
* 字幕・翻訳を「人間のチェック後」に回す理由（順序を間違えると事故ります）
* コンプライアンスチェックを自動化しつつ「自動修正させない」設計の理由
* X用ショート版・概要欄3言語・投稿管理データベースまでの派生フロー
* データ完全ローカル主義のコスト構造（月額いくらで回っているか）
* クラウド同期（GitHub）を試して、やめた話 — 全部ローカルに置くと決めた理由
* **コードが書けない人が、これを自分の環境で再現するための具体的な手順**
* 変わるもの・変わらないもの — 設計思想のまとめ

### この記事が向いていない方

* AI音声で動画を量産して稼ぐ方法を探している方（一切書いていません）
* 「これをやれば登録者が増える」という企画・伸ばし方のノウハウを求めている方（この記事は制作の裏側であって、集客論ではありません）
* 収益の保証を求める方（何も保証できません。書いてあるのは事実だけです）

逆に、「発信は自分でやりたい。でも編集で時間が溶けるのをどうにかしたい」という方には、外注1本分よりずっと安い値段で、私が数ヶ月かけて作ったフローの設計図をそのまま持っていってもらえる内容です。

それでは、裏側に入っていきましょう。
