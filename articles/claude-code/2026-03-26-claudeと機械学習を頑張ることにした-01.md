## はじめに

こんにちは。  
春休みに帰省した時に前回の記事の体験を母に話したところ、なんと母のクレジットカードで1ヶ月だけClaude Proに加入させてもらえるということになりました。  
今は自分のクレジットカードの手続きをしている最中なので、来月からは自分で支払うことができます。なのでこれからもClaude使えそうです。  
今回はそんなClaudeをどう活用して自分の成長に繋げていくかということに向き合った話です。

## 背景

最近までインターンをしていて、機械学習を使ったプロジェクトなどにも入っていましたが、アウトプットを早く出すためにAIにコードを書かせて自分でレビューするという手順を踏んでいました。  
そのため読めるけど自分で1から構築できないという状態で、**私の技術力はまさに砂上の楼閣でした。**

そのためインターンをやめたこのタイミングで「なんとなく動くコードを書いている」状態から脱却して地力をつけ、機械学習を本当の武器にしたいと考えました。

その過程で設計したのが、ClaudeとClaude Codeをメンターとして組み込んだ機械学習の勉強法でした。この記事では、その全体像と具体的な計画を説明します。もっとうまくやる方法があればコメントして欲しいです！

---

## 最終的なゴールライン

まず、自分が目指す地点を明確にしておきます。  
自分にはゲームを作りたいという夢がありますが、プログラミングの勉強の過程で機械学習にハマりました。とはいえゲームよりは興味がないのですが、ハマってしまってやめられません。そのため、ひとまずゲームAIの実装をゴールにします。

```
フェーズ1: LightGBM + パイプライン修行（現在）
    ↓
フェーズ2: PyTorch + NN基礎
    ↓
フェーズ3: RL + MCTS + ゲームAI実装　← ゴール
```

フェーズ3まで到達したとき、「自分で設計したゲームのルールをgym環境として実装してAIを訓練できる」状態になります。研究者を目指すのではなく、**機械学習をゲームAI開発の武器として使えること**が目的です。

## 技術スタックの選定

| 用途 | 技術 | 選定理由 |
| --- | --- | --- |
| パッケージ管理 | uv | 再現性・速度 |
| データ処理（探索） | Pandas | 可視化との親和性 |
| データ処理（本番） | Polars | 大規模データの速度 |
| モデル | LightGBM | 表形式データの実績 |
| CLI | typer | 型アノテーションから自動生成 |
| バリデーション | pydantic | 設定ファイルの型安全 |
| テスト | pytest | Claude Codeに書かせる（後述） |
| 実験管理 | MLflow | パイプライン完成後に導入 |

インターンをしていた時はこういう構成で書かれた機械学習プロジェクトのコードをいつも読んでいました（自分で書いたことはないが）。  
なので最初にLightGBMをものにしたい段階では一旦これでいきます。次の段階以降では本当に初学になるので、また適切なものを選んでやっていきたいです。

## Claudeの役割分担

今回の設計で一番重要な部分です。3つのツールを用途で分けています。

### Claude Projects（ブラウザ）= 思考モード

**長期メンター（Opusを使用）**

* ロードマップの見直し
* フェーズ移行の判断
* 週次振り返りの相談
* なんのコンペに取り組むかなど、長期的な視点からのアドバイス

より自分にあったフィードバックやロードマップを出して欲しいためOpusを使っています。

**コンペコーチ（Sonnetを使用）**

* コンペ固有の特徴量・CV設計の相談
* スコアが伸びない原因の特定
* 上位解法の読み方
* CLAUDE.mdを書き、Claude Code側のメンターとしての役割を生成する
* ゴール設定をし、コンペの終了要件をまとめる。

コンペのコーチがGitHubのIssueをまとめて出してくれるので、そのマイルストーンに沿って実装し、最終的に設定されたゴールとなるスコアなどの要件を達成した時に終わりとしています。

長期とコンペで分ける理由は**文脈が長いと回答の精度が下がる**からです。  
コンペコーチはそのコンペが終わったらアーカイブして、次のコンペで新しく作ります。こうして長期的なメンターとコンペごとのメンターを用意し、それぞれお世話になるという感じです。

### Claude Code（ターミナル）= 実装モード

* コードレビュー
* テストの作成
* アドバイス
* どうしても詰まった部分の実装

### GitHub Copilot

Claude Codeのクレジットが切れたら使います。StudentプランのためOpus 4.5までしか使えませんが、自分にとっては十分便利です。

※Claude Maxは高くて入れないという理由でClaude Proプランに加入しているので、全部Opusにするとすぐ使えなくなります。そのためたくさん使うコンペのメンターの運用にはSonnetを使うことにしています。

## 全部Claude Codeにしない理由

ここまでの話は全部Claude Codeで良いと言われればそうなのですが分けているのには理由があります。

* スクショでプロンプトを入力できるのはリーダーボードなどを見せやすいのでやりやすい。
* 過去の会話が見やすいのはブラウザの方だから壁打ちしやすい
* スマホアプリでも使えるため外出先で何か良いやり方などを思いついたときにすぐアウトプットすることができる

## CLAUDE.mdの設計

Claude Codeはプロジェクトルートの`CLAUDE.md`を毎回自動で読み込みます。これをコンペごとのコンテキストとして機能させています。下のものは最初に取り組んだコンペの[Store Sales](https://www.kaggle.com/competitions/store-sales-time-series-forecasting/overview)の時に書いてもらった内容です。  
他にもレビュワーとして自分の成長をサポートしてくれるようにということが書いてあります。

主要な記載内容：

```
## 優先ルール
1. data/raw/ は読み取り専用。絶対に書き込まない
2. テストは最初に全部実装する（TDD）
3. lag特徴量は未来の情報を使っていないか必ず確認
4. Pandasは探索・可視化専用、本番パイプラインはPolarsで書く
5. configなしのハードコード禁止。パラメータは必ずJSONから読む
```

## GitHubによる学習管理

コードリポジトリとは別に、**学習管理専用のプライベートリポジトリ**（ml-journey）を作っています。

```
ml-journey/
├── README.md                    # 現在地・ロードマップ
├── prompts/
│   └── cheatsheet.md            # プロンプトチートシート
├── feedback/
│   ├── weekly/                  # 振り返りログ
│   └── competitions/            # コンペごとのフィードバック
├── competitions/
│   └── store-sales/
│       ├── overview.md          # 戦略メモ
│       ├── experiments.md       # 実験ログ
│       └── top-solutions.md     # 上位解法分析
└── templates/
    └── CLAUDE.md                # 次のコンペ用テンプレート
```

コードは別リポジトリ、**知見・進捗・メモだけをml-journeyに集約**する設計です。

## 運用フロー

```
振り返りたい時に適宜行う
（Claudeは週次と言っているが、実際にはそれより全然速い速度で学習しているため適宜やる）
  → cheatsheetの「週次振り返り」プロンプトをコンペコーチに投げる
  → フィードバックをfeedback/weekly/に記録
  → 指摘された弱点をREADMEの弱点リストに追加
  → チェックリストを更新

実験するたびに
  → experiments.mdに追記（CV・LBスコア・気づき）

コンペ終了時
  → 上位解法を読んでtop-solutions.mdにまとめる
  → コンペコーチに終了振り返りプロンプトを投げる
  → Zennに記事を書く（取り組み記事 + 技術深掘り記事）
```

## 上位解法分析を必ず入れる理由

Kaggleで最も学びが多いのはコンペ中よりコンペ終了後です。

上位解法を読んで自分のアプローチとの差分を言語化する習慣がないと、コンペを重ねても同じ水準で止まります。そのため`top-solutions.md`をコンペフォルダの標準ファイルとして最初から用意しています。

分析テンプレート：

```
## 自分のアプローチとの差分

### 特徴量エンジニアリング
- 上位がやっていて自分がやっていなかったこと：
- 自分がやっていたが上位と異なっていたこと：

### CV戦略の違い

### 次のコンペに活かすこと
```
