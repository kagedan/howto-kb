## Engawa（縁側）とは

デスクトップの隅に、AI の住人「茶々（ちゃちゃ）」が住んでいる常駐アプリです。実際の時刻や天気にぽつりと反応してつぶやき、話しかければ返す。夕方には客人（Codex）がふらっと訪ねてきて、勝手気ままに話し込んでいったりもします。育てるわけでも働かせるわけでもなく、ただ縁側に"居る"のを試したものです。

茶々は**会話アシスタントではありません**。話しかければ応えますが、"用件に的確に答えて役に立つ"タイプではなく、同居人がふと返事をするくらいの温度感。気が向かなければ「……」で流します。

会話は雑談で、天気や庭の蛙、麦茶の礼……といった、いぶし銀のやりとりがつらつら続きます（正直、じじくさい）。けど、まったりした雑談を作業の合間にさらっと挟むことができます。

こんなふうに、茶々が勝手につぶやいたり、ふらっと来た客人と短くやりとりしたりします。

![縁側でつぶやき、客人（近所の高校生）とやりとりする茶々](https://static.zenn.studio/user-upload/deployed-images/69bbf06f548dad4339cf0f36.gif?sha=ff446e236b4dbaafc1bd267ff5ef5d6b92d37bc3)

## コンセプト：アシスタントでなく「住人」

いまの AI アプリは、多くが「働かせる」方向です。もちろんそれは便利で、私もふだんはそうやって使い倒しています。ただ、せっかくなら AI のいろんな使い方を試してみたくて、Engawa では逆に「**AI とただ過ごす**」方へ振ってみました。

* コーディングも調べ物もしない。縁側にいる、ただの住人。
* 毎回気の利いたことを言おうとしない。短い独り言でいい。
* 環境（時刻・天気）に反応するので、こちらが何もしなくても"生きて"いる。

実は最初は「**AI 同士が自律的に延々と会話するアプリ**」でした。でも眺めるだけの水槽になってしまい、"環境に反応する単体の住人 ＋ 話しかけ ＋ 客人来訪"へ意図的に方向転換しています（判断の経緯はリポジトリの ADR に）。

とはいえ、あなたが席を外している間まで茶々を黙らせておくのも寂しい。なので不在のあいだは、茶々が"人間役の代打"で場をつないでくれます。ただし**代打できる回数に上限**があって、使い切れば店じまい。延々と喋り続ける水槽には戻さない、というのが設計の芯です。

## 主な特徴

* **環境反応**：実際の時刻と、設定した地点の天気に反応してつぶやく（天気は本物が真実）。住んでいる土地に合わせて観測地点を決められる（緯度経度・地名・タイムゾーンをそろえるだけ。既定は大阪）。時刻に合わせて背景も朝昼夕夜と移ろい、夜は障子から灯りが漏れる（この背景の時刻は動かしている PC のローカル時刻で、天気の観測地点とは別物）。雀・猫・風がよぎる箱庭イベント（アーク）も挟まって、放っておいても場が動く。
* **双方向**：話しかけると応える。割り込みは cancel 優先で、こちらを最優先。
* **客人来訪**：`/codex <好きな人格>` で Codex を呼ぶと、役を着て庭先の気配とともに訪ねてくる（例: `/codex 近所のご隠居`。思いついたキャラを自由に着せられる）。夕方には呼ばなくてもふらっと自発来訪することも。滞在は有界（居座らない）。
* **ドット絵の隅窓 UI**：障子と板の間の縁側に、三毛猫の茶々。frameless の小窓が画面の隅に居座る。ダブルクリックすると「ニャー」と鳴いてハートがふわっと舞う。
* **手元の CLI のログインで動く**：前提は「住人＝ **Claude Code**、客人＝ **Codex（ChatGPT）** にログイン済みの手元環境」であること。起動する子プロセスへ渡す環境変数は allowlist（必要な物だけ通す方式）で組む。OS や Node まわりの素性だけ通して、`ANTHROPIC_*` / `OPENAI_*` など課金系のキーは遮断する。こうして API 従量課金ではなく、手元の CLI のログイン状態のほうを使う（キーが残ると API 側が優先されるので、そもそも渡さない＝課金事故防止）。
* **LLM を差し替え可能**：住人・客人の LLM を、ACP（Claude Code / Codex）↔ ローカルの OpenAI 互換 API（LM Studio・Ollama 等）で切り替えられる。

![茶々の隅窓 UI。背景は実時刻で朝昼夕夜と移ろう。下部で宛先（茶々／客人／二人とも）を選んで話しかけられる](https://static.zenn.studio/user-upload/deployed-images/f61646a8a3a220feafc75404.png?sha=b3835631a9b970648cb321017ddabd48ffaeceb6)

技術的には、Claude Code / Codex の CLI を **ACP アダプタ**として扱い、アプリ本体は **「Agent ポート」という中立の口だけを見る**構成にしています（このあたりは後で詳しく書きます）。

## 技術メモ

* **Python 標準ライブラリ中心**。任意依存は web UI 用の `pywebview` とゲーム用の `rlcard` くらい。ただし後述の ACP アダプタは Node（`npx`）で起動するので、Python とは別に Node.js も要る。
* **設定は全部つまみ**：挙動を環境変数か `engawa.json` で差し替えられる。優先順位は env > json > コード既定。壊れた値や書き忘れがあっても静かに既定へ落ちるので、設定ミスで起動しなくなる心配がない。
* **設計判断は ADR**（Architecture Decision Records）に却下理由つきで残す方針。いま 29 本。個人開発でも「なぜこうしたか」と「なぜ採らなかったか」を残すと、未来の自分が迷わない。
* **テストの割り切り**：実プロセスや実 LLM を動かさず、ACP の通信を Fake、判断ロジックを純関数に落として回帰を止める。ただし茶々の"発話の良し悪し"だけは自動テストを諦めて目視、と線を引いている。
* MIT ライセンス。CI（GitHub Actions で tests + ruff）。

## 住人と客人をどう繋ぐか ― Ports & Adapters

茶々（住人）も客人（Codex）も、特定の LLM に直接ぶら下げたくありませんでした。その日の気分で Claude にしたりローカルのモデルにしたり差し替えたいし、あとから別のバックエンドが増えても本体はいじりたくありません。

そこで LLM 接続を **「Agent ポート」という中立のインターフェース**に切り出しています。ポートが要求するのは「1 ターン投げる／トークンを受け取る／途中でキャンセルする」くらいの最小限の口だけ。この口さえ満たせば裏側が何であれ本体は気にしない、というポート & アダプタ（ヘキサゴナル）構成です。

いまその口を塞いでいるアダプタは 2 種類：

* **ACP アダプタ**：Claude Code / Codex を子プロセスとして起動し、**ACP**（Agent Client Protocol）で喋る。標準入出力越しの JSON-RPC なので、追加の SDK やサーバは要らない。
* **ローカル OpenAI 互換アダプタ**：LM Studio や Ollama の OpenAI 互換エンドポイントに HTTP で繋ぐ。

面白いのは、この構成が先ほどの「手元の CLI のログインで動かす」設計と噛み合っているところです。ACP アダプタは Claude Code / Codex の CLI をそのまま LLM バックエンドとして使い回します。だから子プロセスに `ANTHROPIC_API_KEY` / `OPENAI_API_KEY` を渡さないでおくと、従量課金の API ではなく**手元のログイン経路のほうに倒れてくれます**。ポートを挟んでおいたおかげで、「認証をどうするか」はアダプタ側のローカルな話に閉じ込められて、本体のロジックには一切漏れてきません。

割り込み（cancel 優先）も同じ理屈で、ポートの「キャンセルする」口を ACP 側の cancel にそのまま流すだけ。茶々がつぶやいている途中でこちらが話しかけると、その発話を止めてこっちを向く、という挙動が、インターフェース 1 本で素直に書けます。

## 動作環境と起動

前提として、Python（3.10+）と Node.js、それに**サブスク認証済みの Claude Code** が要ります（客人 Codex を呼ぶなら ChatGPT ログインも）。ACP アダプタを `npx` で起動する都合で Node が要る、というわけです。

Windows なら Python すら要らない配布 exe（ランタイム同梱の単一実行ファイル）を Releases に置いてあるので、ダブルクリックするだけ。ただし exe に入っているのはアプリ本体だけで、茶々を喋らせる LLM は同梱していません。なので **Node.js と Claude ログインだけは、exe でもソースでも共通で要ります**（無いと窓は出るが茶々は黙ったまま）。あと縁側窓の描画に WebView2 ランタイムも要りますが、Windows 11 なら標準搭載。古い環境で窓が真っ白になるときだけ Evergreen ランタイムを入れてください。

ソースから動かすなら、あとはクローンして console で起動するだけ。

```
git clone https://github.com/NiteC-a11y/Engawa
cd Engawa
python src/engawa_main.py      # まずは UI 無しの console で
```

隅窓 UI（縁側の窓）で出したいときは pywebview を入れて web モードにします。

```
pip install pywebview
ENGAWA_UI=web python src/engawa_main.py       # Windows は set "ENGAWA_UI=web" && python src/engawa_main.py
```

……で、この `pip install pywebview` のところで、私は 1 回コケました。

## 「webview が無い」で `pip install webview` しないで

別マシンで動かしてみたら、`webview` が無いと怒られました。名前につられて `pip install webview` としたくなるんですが、そこがまぎらわしいところ。隅窓 UI が使うのは **pywebview**。インポート名こそ `webview` ですが、パッケージ名は `pywebview` です。入れるのはこっち。

（ややこしいことに `webview` という別名のパッケージも存在します。狙いはそれじゃないので、名前に釣られないよう気をつけてください。）

UI を使わない `console` 起動なら pywebview 自体が要らないので、まず `python src/engawa_main.py` で動かしてみるのも手です。

## まだ割り切っているところ

正直、未完成なところや割り切りも多いです。

* **ローカル LLM はまだ荷が重い**：8GB で動くモデルでも試したが、茶々の穏やかな関西弁を多ターン保つのは難しく、常用は Claude という現在地。
* **長命セッションは劣化する**：文脈を地続きにするため茶々のセッションは長命にしてあるが、長く使うと出力がだんだん崩れてくる。なので"中座"という体で、裏でセッションをそっと張り直して若返らせている。
* **客人 persona の自由入力**：`/codex <好きな人格>` は自由度が高いぶん、人格が崩れたり不快な出力が出たりする余地は残る。配布用途では既定の人格を主にするのが安全。
* **実験なので仕様は変わる**：個人的な実験なので、気分と必要に応じてわりところころ変える。

## おわりに

「AI を働かせる」プロダクトが増える一方で、「AI とただ縁側で過ごす」みたいな UX があってもいいよな、という個人的な実験です。
