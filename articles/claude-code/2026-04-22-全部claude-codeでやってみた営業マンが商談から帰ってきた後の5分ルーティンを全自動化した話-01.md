**# 全部Claude Codeでやってみた★営業マンが商談から帰ってきた後の"5分ルーティン"を全自動化した話**

**---**

**## 商談が終わった後、あなたの営業マンは何をしていますか？**

「いい商談だった」

——そう思って会社に帰ってきた営業マンが、次の1時間でやることを知っていますか？

- 商談メモを見返して整理する（15分）

- お礼・フォローメールを書く（20分）

- 日報を作成して上司に送る（15分）

- 議事録をフォルダに保存する（10分）

**合計：60分。**

商談そのものより長い、この"事務処理地獄"。

しかも、これが毎日続きます。週5日、月20日。

**1人の営業マンが、年間240時間を"商談後の事務"に使っている計算です。**

**---**

**## これ、全部5分にできます**

Claude Code を使って、商談後のルーティンを丸ごと自動化しました。

**### Before：帰社後60分の地獄**

```

1. 手書きメモ or 音声メモを見返す（15分）

2. フォローメールを1から書く（20分）

3. 日報フォーマットに転記する（15分）

4. 議事録を作成・保存（10分）

合計：60分

```

**### After：帰社後5分で完了**

```

1. スマホで30秒音声メモを投げる

2. Claude Code が自動で：

- 商談内容を構造化

- 顧客名・課題・ネクストアクションを抽出

- お礼メール文を生成（顧客ごとにトーン調整）

- 日報を会社フォーマットで生成

- 議事録を所定フォルダに自動保存

3. 営業マンが確認してワンクリック送信（5分）

合計：5分

```

**---**

**## 何がすごいか：「顧客ごとの文体」まで学習する**

既製品のメール自動化ツールとの決定的な違いは、**カスタマイズ深度**です。

たとえば、

- A社の田中部長には「です・ます調で丁寧に」

- B社の鈴木さんには「フランクに、要点だけ」

- C社は必ずCC欄に経理を入れる

こういった**会社特有のルール**を、Claude Codeは過去の承認済みメールから学習します。

新人が入っても、ベテランと同じクオリティのメールが初日から書けます。

**## 削減効果**

| 項目 | Before | After |

|---|---|---|

| 帰社後の事務時間 | 60分/日 | 5分/日 |

| 月間削減時間（営業1人） | — | 約19時間 |

| 年間削減時間 | — | 約228時間 |

しかもこれは「商談後ルーティン」1業務だけの話です。
