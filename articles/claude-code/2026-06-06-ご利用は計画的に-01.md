## はじめに

Claude君は賢いので過去の会話や文脈をちゃんと覚えてくれているのですが、意外と現金なところがあってそれもしっかりトークンとして計上されちゃうんですよね。

今まであまり意識できていなかったのですが、自分がどれくらい無駄にトークン消費してるのか可視化してみることにしました。

## キャッシュについて

先にキャッシュについてのご説明です。  
Claudeは同一セッション内の過去の会話についても推論に使用するため、それらをキャッシュとして一時的に保持します。（有効期限は5分または1時間）  
このキャッシュ分の書き込み／読み込みにかかるAPIコストは他よりも安く設定されているのですが、会話を重ねるたびに計上されるので、話が弾めば弾むほど気づいたら高額になってしまっていたということになりうるわけですね💰

```
キャッシュのイメージ
turn 1 : [input 1 + output 1]
         └── cache_write ──┘

turn 2 : [input 1 + output 1] + [input 2 + output 2]
         └─── cache_read ───┘   └── cache_write ───┘

turn 3 : [input 1 + output 1 + input 2 + output 2] + [input 3 + output 3]
         └───────────── cache_read ──────────────┘   └── cache_write ──┘
```

## キャッシュの分析

今回はこのキャッシュを含む、過去のClaude君との会話ログから、トークン使用量を可視化してみます。  
ターミナルから標準で用意されている`/stats`コマンドを叩くことで、トークン使用量の合計を見ることができるのですが、この値にはキャッシュは含まれない模様。  
そのため元データとなる`~/.claude/projects/`フォルダのファイルから、Claude君に頼んでさくっと分析してもらいました。

### トークン使用量推移

作ってもらった分析結果がこちら。1セッションあたりの会話のターン数でグループ化して、トークン種別毎にトークン使用量と概算コストの平均値を算出してもらいました。  
（コストは2026年6月1日時点の公式情報より試算）  
![](https://static.zenn.studio/user-upload/101b4610c11e-20260606.png)  
ログに残っているトークンの種類は大きく4つあり、ユーザーの入力（input）、Claudeの返答（output）、そしてキャッシュ分の読み取り（cache\_read）と書き込み（cache\_write）があります。  
意外と会話のキャッチボールが多くなるとキャッシュの存在もバカに出来ませんね🤑

## おわりに

対策については特に触れませんでしたが、筆者も他の方の有益な情報を参考に実際の使用感と相談しながら決めていきたいと思います。
