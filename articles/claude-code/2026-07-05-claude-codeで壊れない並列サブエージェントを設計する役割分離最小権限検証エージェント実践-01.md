Claude Code の動的ワークフロー（Dynamic Workflows）は「Claude が自分でオーケストレーションを書き、数十〜数百のサブエージェントを並列実行する」機能だ。機能としての使い方は別記事「[Claude Code Dynamic Workflows 実践ガイド](https://zenn.dev/)」で整理した。

本記事はその一段下のレイヤー、**「自分のリポジトリ専用のサブエージェント群をどう設計するか」** を扱う。素朴に「役割を分けて並列化する」だけだと、規模が増えた瞬間に壊れる。壊れない設計には3つの原則がある。

---

## 素朴な並列化が壊れる3つの理由

サブエージェントを「とりあえず並列で投げる」と、次の3つで事故る。

1. **コンテキスト汚染**：1つの巨大プロンプトに全タスクを詰めると、エージェント同士の中間出力が混ざり、後半ほど精度が落ちる。
2. **ツール過剰権限**：すべてのサブエージェントに全ツールを渡すと、調査だけさせたいエージェントが `Write` や `Bash` を実行してしまう。
3. **検証の不在**：N体が出した結果を誰も突き合わせない。1体の幻覚がそのまま最終成果物に混入する。

逆に言えば、**役割分離・最小権限・検証エージェント** の3点を設計に織り込めば、並列数を増やしても破綻しにくい。

---

## 原則1・2：サブエージェントを定義ファイルで固定する

Claude Code のサブエージェントは `.claude/agents/*.md` に宣言できる。frontmatter で「役割の説明」と「使えるツールの許可リスト」を固定するのが肝だ。`tools` を明示すると、そのエージェントは列挙したツールしか呼べない（最小権限）。

.claude/agents/security-scanner.md

```
---
name: security-scanner
description: >
  単一ディレクトリ配下のAPIエンドポイントを読み取り専用で検査し、
  認証チェック漏れの「候補」だけを構造化して返す。修正はしない。
tools: Read, Grep, Glob
model: sonnet
---

あなたは読み取り専用のセキュリティ調査員です。
- 与えられたディレクトリの *.ts / *.js だけを対象にする。
- 各ハンドラについて「認証ミドルウェアを経由しているか」を判定する。
- 出力は1件1行の JSON（path, line, reason, confidence: high|low）のみ。
- 推測が必要な場合は confidence: low を付け、断定しない。
- ファイルの変更・コマンド実行は禁止（ツール上もできない）。
```

ポイントは `tools: Read, Grep, Glob` の3つに絞っていること。`Write` も `Bash` も渡していないので、このエージェントは設計上「壊す」ことができない。description を具体的に書くほど、親エージェントからの委譲（Task ツール経由）も正確になる。

---

## 原則3：オーケストレータと検証エージェントを自前で書く

機能としての動的ワークフローに任せず、**再現可能なスクリプトとして** ファンアウトと検証を書くと、CI でもローカルでも同じ手順を回せる。ヘッドレスモード（`claude -p`）は JSON を返すので、`child_process` で並列起動して集約できる。

scripts/agents/orchestrate.mjs

```
import { execFile } from "node:child_process";
import { promisify } from "node:util";
const run = promisify(execFile);

// 1体のサブエージェントをヘッドレスで起動し、結果テキストを返す。
async function runAgent(prompt, { agent, cwd }) {
  const args = ["-p", prompt, "--output-format", "json"];
  if (agent) args.push("--agents", agent); // 使う定義を固定
  const { stdout } = await run("claude", args, {
    cwd,
    maxBuffer: 16 * 1024 * 1024,
    timeout: 10 * 60_000,
  });
  return JSON.parse(stdout).result ?? "";
}

// フェーズ1: ディレクトリごとに security-scanner を並列ファンアウト。
async function scan(dirs, cwd) {
  const settled = await Promise.allSettled(
    dirs.map((d) =>
      runAgent(`${d} 配下のエンドポイントを検査してください`, {
        agent: "security-scanner",
        cwd,
      })
    )
  );
  return settled.flatMap((r, i) =>
    r.status === "fulfilled"
      ? [{ dir: dirs[i], findings: r.value }]
      : [{ dir: dirs[i], error: String(r.reason) }]
  );
}

// フェーズ2: 別の検証エージェントに「反証」させ、高信頼の指摘だけ残す。
async function verify(rawFindings, cwd) {
  const prompt =
    "次の指摘リストから、誤検知の可能性が高いものを除外し、" +
    "残ったものだけを JSON 配列で返してください:\n" +
    JSON.stringify(rawFindings);
  return runAgent(prompt, { agent: "finding-verifier", cwd });
}

const cwd = process.cwd();
const dirs = process.argv.slice(2);
const raw = await scan(dirs, cwd);
const verified = await verify(raw, cwd);
console.log(verified);
```

調査（フェーズ1）と検証（フェーズ2）でエージェントを分けるのが要点だ。同じエージェントに「調べて、かつ自分の結果を疑え」と頼むより、**別人格に反証させる** ほうが幻覚を落としやすい。`Promise.allSettled` を使い、1体が落ちても全体を止めないのも実務上重要だ。

## 検証方針：トークンを燃やさずにオーケストレーションをテストする

LLM の出力は非決定的なので、「正しい指摘が出るか」をユニットテストにはできない。テストすべきは **オーケストレーション層の挙動**（並列起動・集約・失敗時の縮退）だ。そこで `claude` バイナリをモックに差し替える。

scripts/agents/mock-claude.sh

```
#!/usr/bin/env sh
# PATH の先頭に置いて本物の claude の代わりに使うスタブ。
# 常に固定の JSON を返すので、ネットワークもトークン消費もゼロ。
printf '%s' '{"result":"[{\"path\":\"src/a.ts\",\"line\":12,\"confidence\":\"high\"}]"}'
```

```
# 検証手順（ローカル／CIで再現可能）
chmod +x scripts/agents/mock-claude.sh
mkdir -p .tmpbin && cp scripts/agents/mock-claude.sh .tmpbin/claude
PATH="$PWD/.tmpbin:$PATH" node scripts/agents/orchestrate.mjs src/routes src/api
# 期待: 2ディレクトリ分が集約され、verify を通った JSON が1つ出力される
```

この方法なら「2件投げたら2件分が集約される」「1件が異常終了しても残りは処理される」といった**決定的な性質**を CI で固定できる。実際のエージェント品質は別途、少数の代表ケースを目視レビューで担保する——自動テストと人手レビューの責務を分けるのが現実解だ。

## 失敗パターンと回避策

* **検証エージェントに修正権限を渡してしまう**：`finding-verifier` にも `tools: Read, Grep` だけを渡す。検証役が勝手にコードを書き換えると、何を検証したのか追えなくなる。
* **並列数を最初から最大にする**：まず2〜3ディレクトリでトークン消費とレイテンシを計測し、`/workflows` 相当の感覚を掴んでから広げる。
* **タイムアウト未設定で吊られる**：`timeout` と `maxBuffer` は必ず指定。1体のハングが全体を止める。
* **定義ファイルを共有しない**：`.claude/agents/` をコミットしてチームで共有する。個人の `~/.claude/agents/` だけだと再現性がない。
