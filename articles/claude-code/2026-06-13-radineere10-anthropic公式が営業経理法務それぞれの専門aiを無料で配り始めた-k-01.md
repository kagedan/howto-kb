Anthropic公式が、営業・経理・法務それぞれの“専門AI”を無料で配り始めた。。。

「knowledge-work-plugins」がガチで本命🔥 
Claudeを職種ごとの専門家に変える11種のプラグイン集です。
汎用AIを「うちの営業」「うちの経理」に仕立てる素材が、まるごと無料で公開されました。

部門にAIを導入したい人、AIを“便利だけど中途半端”で止めている人は、絶対にブクマして。
導入の出発点が手に入ります。

押さえるべき6点👇
・11職種ぶんのプラグインが用意済み — 営業・マーケ・経理/財務・法務・カスタマーサポート・プロダクト・データ・社内検索・productivityなど。自部門のものを入れるだけで、Claudeが役割に最適化される
・スラッシュコマンドで仕事が呼び出せる — 「/sales:call-prep（商談準備）」「/finance:reconciliation（消込）」のように、定型業務をコマンド一発で。何を頼めばいいか分からない問題が消える
・既存の業務ツールに接続 — HubSpot・Slack・Notion・Figma・Snowflake・JiraなどにMCPでつながる。データを取りに行くところからAIが動く
・スキルが自動で発動 — 関連する作業のとき、その職種の専門知識がClaude側で自動的に立ち上がる。毎回プロンプトで指示し直す必要がない
・コード不要でカスタムできる — 中身はすべてMarkdownとJSON。自社の用語・プロセス・ツール構成を書き足すだけで、“自社仕様の参謀”に育てられる
・Claude Cowork / Claude Code 対応・公式・MIT級の手軽さ — 公式が出している安心感と、フォークして自社配布できる自由度を両立している

法人視点での衝撃はこうだ…

多くの会社のAI活用が“便利止まり”で停滞する理由は明確です。
Claudeは賢いけれど「あなたの会社のやり方」を知らない。
だから毎回ゼロから前提を説明し、出力も人によってバラバラになる。
結局「個人が上手く使うツール」のまま、組織の戦力にならない。

このプラグイン集は、その壁を最初から越えた状態を配っています。
職種ごとの仕事の型・使うツール・呼び出すコマンドが最初から入っているので、現場は“育てる前の参謀”をいきなり手に入れられる。
しかも全部MarkdownとJSONなので、そこに自社の用語と手順を足せば、汎用AIが「うちの会社仕様」に変わる。
重要なのは、これがAnthropic公式の構造だという点です。
職種別にAIを標準化し、部門ごとに専門家を持つ…その設計図が、無料で手に入る時代になりました。

なぜ「公式が出した」ことが大きいのか。
これまで職種別のAI活用は、各社が手探りでプロンプトを作り、個人のノウハウとして散らばっていました。
そこにAnthropic自身が「営業ならこう、経理ならこう」という型を、ツール接続もコマンドもセットで提示した。
職種別AI活用の“事実上の標準フォーマット”が無料で公開されたわけです。
本質は、ここから「自社仕様に育てる」ことにあります。
全部MarkdownとJSONなので、自社の商品名・審査基準・承認フロー・ツール構成を書き足せば、汎用の営業プラグインが「うちの営業」に変わる。
逆に言えば、型をそのまま使うだけでは“便利止まり”。
自社の知識をどう書き足し全社へ配布するか…その設計ができる会社だけが、AIを個人技から組織の仕組みへ引き上げられます。

特に以下の立場で即効果が出ます。
・経営・AI推進 — 部門別にAIを標準化し、「個人の使いこなし」を「組織の仕組み」へ引き上げる
・営業・マーケ — 商談準備・パイプライン確認・コンテンツ作成を、コマンド化して属人性を下げる
・経理・法務 — 消込・仕訳・契約レビューなど定型業務の型を、自社ルールに合わせて整える
・情シス・DX — 自社ツール構成に合わせてプラグインを改変し、全社へ配布する基盤にする

ただし、これは「入れたら完成」ではなく「自社仕様に育てる出発点」です。
真価が出るのは、自社の用語・プロセス・ツール接続を書き足してから。
逆に言えば、ここをカスタムできる体制があるかが、組織でAIが戦力になるかの分かれ目になります。

あなたの会社のAIは、“便利な個人ツール”で止まっていませんか。
それとも、部門の仕組みになっていますか。
