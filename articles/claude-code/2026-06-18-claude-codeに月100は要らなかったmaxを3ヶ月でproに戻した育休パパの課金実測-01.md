Claude（クロード）の課金、月$20のProと月$100のMax、どっちにすべきか迷っていませんか。

ぼくは「どうせやるなら一番上だろう」と、何も考えずに月$100のMaxプランに上げました。そして3ヶ月使って、**1つ下のProに戻しました**。

結論から言うと、**個人で副業をやる規模なら、ほとんどの人にMaxは要らない**。これが、実際に3ヶ月課金して出した答えです。

この記事を読むと、こんなことが分かります。

全部、自分が実際に払って持て余した実体験ベースです。Claude課金で迷っている人の、判断材料になればうれしいです。

## 副業を始めた前提（自己紹介）

簡単に。

* 30代、2児の父（上の子2歳、下の子は生後2ヶ月）
* 本職はコンサル。プログラミングは独学レベル
* 2026年4月から育児休業に入り、その時間でAI副業を実験中
* 武器は **Claude Code**。記事の下書きも小さいアプリの実装も、ほぼ全部やってもらっている

つまり「ヘビーな開発者」ではなく、**スキマ時間でAIを使う個人**の目線だと思って読んでください。

## まず、Claudeの個人プランは3つ＋API

最初に事実から。Claudeの個人向けプランは、ざっくりこうなっています（2026年6月時点・公式）。

![](https://assets.st-note.com/img/1781743070-oeV0LDfXjWi5gGZOYsKTtdSx.png?width=1200)

ポイントだけ言葉でも書いておきます。

ここで地味に大事なのが、**Maxは月額のみで、年割がない**こと。Proには年払いの割引があります。長く使うなら、この差は地味に効いてきます。

あと、使用量の制限は「1日いくつ」ではなく **約5時間ごとのローリング制**。2026年5月には、Maxのレート制限が約2倍に引き上げられました。ヘビーユーザーにはありがたい改定ですが──逆に言うと、**そこまで回す人じゃないと恩恵を感じにくい**ということでもあります。

## ぼくがMaxに上げて、戻すまで

では本題。なぜ上げて、なぜ戻したのか。ここが、ぼくの場合はちょっと特殊で、でも一番伝えたいところです。

### 上げた理由：「大量に作れば、どれか売れる」と思っていた

最初のぼくは、こう考えていました。

「Claude Codeで、いったい何が作れるんだろう」  
「アプリも記事も、とにかくたくさん作れば、どれかは当たるはずだ」

だから方針は**数で勝負**。Claude Codeに大量に指示を出して、記事もアプリも片っ端から作るつもりでした。

**大量に生成するなら、上限は高いほうがいい**。その発想で、迷わず月$100のMaxに上げたんです。

### 戻した理由：「作ること」と「売れること」は、別の話だった

ところが、実際にたくさん作ってみて、はっきり気づきました。

**むやみに作っても、売れない。**

記事を何本も量産しても、お金になったのは1本だけ。**作った数と、売上は、まったく比例しなかった**んです。作ることと、売れることは、完全に別の話でした。

しかも、ぼくの時間は限られています。子どもが寝た後の1日90分。その中で、Claude Codeに大量に指示を出して数を撃つより、**「何を作るか」をちゃんと絞って、ポイントを決めてから作るほうが、ずっと効率的**だと気づきました。

そして方針が「**大量に作る**」から「**厳選して作る**」に変わった瞬間、面白いことが起きました。

**そもそも、Maxの大容量が要らなくなったんです。**

数を撃つのをやめたら、月90分の作業はProの範囲に余裕で収まる。月¥17,500のMaxは、完全にオーバースペックになっていました。

> **「作る」と「売れる」は別物だった。数を撃つのをやめて狙って作るようにしたら、最強プランは要らなくなった。**

## じゃあ、誰にどのプランが向くのか

ここが一番伝えたいところです。プラン選びは「人による」が、その「人による」を分解すると、実はシンプルな1つの問いに行き着きます。

![](https://assets.st-note.com/img/1781743094-0cqNvjgDQ1LmEaZYtpe26ISx.png?width=1200)

判断の軸はこれだけ。

> **Proの5時間制限に、週3回以上当たるか？**

* **当たらない（月に数回）** → **Pro**で十分。年払いにすればさらに安い
* **週に何度も当たる** → **Max 5x**を検討する価値あり
* **1日中ぶん回す・本業の主戦力** → **Max 20x**
* **自動化やプログラムに組み込みたい** → **API**（使った分だけ）

ぼくみたいに「スキマ時間で記事とアプリ」という個人副業の規模なら、**ほぼ間違いなくProで足ります**。

逆に、1日中Claude Codeを動かして開発している人、チームで使っている人、本業の生産性が直接お金に変わる人は、Maxの価値は十分あります。**要は「制限に当たって作業が止まる回数」が、月¥17,500に見合うかどうか**。それだけの話です。

## 戻して気づいた3つのこと

最後に、課金を上げ下げして学んだことを3つ。

**1. 「作った数」と「売れた数」は比例しない**  
これが一番の学びでした。量産すれば当たる、は幻想。むやみに作るより、**何を作るかを絞る**ほうが、結果的に売上にも時間効率にも効きます。Maxを持て余したのは、そもそも「数で勝負」という前提が間違っていたからでした。

**2. 作る方針が変われば、必要なプランも変わる**  
「大量に生成する」前提ならMaxにも意味はあります。でも「厳選して作る」に切り替えたら、必要な容量はガクッと下がりました。**プランは、作業スタイルとセットで考える**もの。先にプランを決めるのは順番が逆でした。

**3. プランは「上限に当たってから」上げればいい**  
最初から最強を買う必要はありませんでした。一番安いプランで使ってみて、**実際に制限で困ってから**上げる。この順番が正解です。逆（とりあえず上）は、ぼくみたいにお金を持て余します。
