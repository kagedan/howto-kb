### フォルダ構造

だいたいこんな形です。  
「計算（`engine/`）」「判断（`agents/`）」「可視化（`web/`）」が、そのままフォルダで分かれているのがポイントです。

```
nba-finals-sim/
├─ engine/               # 決定論エンジン（Python・標準ライブラリのみ）
│  ├─ __main__.py        # `python -m engine`（calibrate / autoseries）
│  ├─ possession.py      # 心臓部：1ポゼッションの確率モデル
│  ├─ game.py            # 時計・スコア・クォーター・クラッチ判定
│  ├─ series.py          # 7戦4勝のシリーズ管理（4勝した方が優勝というルールのため）
│  ├─ rules.py           # NBAルール & グローバル係数（LEAGUE_DRTG ほか）
│  └─ data/
│     ├─ rosters.json    # 選手の能力値（実データ + 役割からの推定）
│     └─ teams.json      # チーム設定（守備力・ホームコートなど）
├─ agents/               # AIエージェント（プロンプト & スキーマ）
│  ├─ coach.md           # 🧠 監督：試合前 / ハーフタイム / クラッチの采配
│  ├─ player.md          # 🏀 選手：クラッチの主張 + セリフ
│  ├─ referee.md         # 🦓 審判：際どい笛の裁定（※いまは設計のみ）
│  └─ schemas/           # 返り値JSONのスキーマ
├─ orchestrator.py       # エンジンを区切って進め、合間にエージェントへ相談
├─ web/                  # 可視化（静的サイト・ビルド不要）
│  ├─ index.html
│  ├─ app.js             # series.json を読んで描画・EN/日本語切替
│  ├─ style.css
│  └─ assets/            # チームロゴ・ファビコン
├─ series.json           # エンジンの出力（= サイトの入力）
```

### 全体の構成（AIエージェントなど）

下記のような構成となります。

![](https://static.zenn.studio/user-upload/06c5a7e9fc09-20260604.png)

役割別では下記のとおりです

* **決定論エンジン**：  
  時計、ショットクロック、得点、ファウル、NBAのタイムアウトルール、7戦4勝のシリーズ管理（先に4勝したら優勝のルール）。1回の攻撃ずつ確率モデルで回す、ただの正確な計算機です。LLMは一切入りません。
* **AIエージェント**：  
  監督（試合の戦略・選手交代等）、選手（試合で誰がどう動くか等）、審判（際どい笛の裁定）。今回は点が動く高レバレッジな局面でしか呼びません（攻撃のたびに呼ぶとコストも時間も爆発するので...）。
* **オーケストレーター**：  
  エンジンを「ハーフタイムまで」「クラッチまで」と区切って進め、その合間にエージェントへ相談し、返ってきた作戦をエンジンに食わせる。最後に結果を `series.json` に書き出します。

## どうシミュレーションしたか

シミュレーションエンジンの中身は「1回の攻撃（＝1ポゼッション）」を確率で解いていく、という素朴なものです。

下記の流れを毎回サイコロを振って決めています。

これを1試合分（だいたい片チーム100回ほど）繰り返すと、1試合のスコアになります。

ただこれだとただの確率論になるので、味気ないですよね。なのでより実際のシミュレーションに類似するように下記点を工夫しました。

### こだわりポイント：選手の「心理状況」も取り入れる

各選手の特徴になるよう、数値（シュート成功率、3ポイント成功率、ミスの頻度など）を実データから採取し、重みづけしています。

それだけでなく、ここに **「心理・状況」の補正** を掛けているのがこだわりポイントです。実際の選手って、調子・プレッシャー・疲労で同じ実力でも結果が変わりますよね。そこを倍率で表現しています。

```
# possession.py より（抜粋・コメントは日本語に直しています）
base = shooter["fg3_pct"] if shot_type == "three" else shooter["fg2_pct"]
mult = MAKE_GLOBAL * shooter.get("form", 1.0)   # プレーオフの好不調

# 相手チームの守備力（守りが固いチームほどシュートが入りにくくなる）
mult *= rules.LEAGUE_DRTG / game.drtg(deff)

# ダブルチーム（守備2人がかりで1人を囲む）：囲まれたスターは確率ダウン、代わりに味方がフリーに
if def_focus and def_focus.get("scheme") == "double":
    mult *= DOUBLE_TARGET_MULT if is_target else DOUBLE_OFF_MULT

# クラッチ（残り5分・5点差以内）：クラッチ補正と、締まる守備
if game.is_clutch():
    mult *= shooter.get("clutch", 1.0)
    mult *= 0.97

prob = base * mult

# ホームコート（+2.5点ぶんくらい）、Q4のスタミナ、初戦の“試合勘の鈍り”
mult *= game.home_mult()            # ホームコートアドバンテージ
mult *= game.stamina_mult(shooter)  # 終盤（Q4）の疲労
mult *= game.rust_mult(shooter)     # 休養明けの試合勘の鈍り

# 0〜1の確率に収めて、このシュートの成功率が決まる
prob = max(0.02, min(0.99, base * mult))
```

具体的には、こんな現実の文脈を数値に落としました。

* **ホームコートアドバンテージ**：  
  地元開催の有利さ。1試合あたり約2.5点ぶん。最近のNBAは3ポイントが観客の声援に左右されにくいので、昔ほど効かない（勝率55%程度）という肌感も反映。
* **休養と疲労**：Spurs は優勝決定前の前哨戦（カンファレンス決勝）を7試合フルで戦って中3日 → 試合終盤（Q4）にスタミナ補正。Knicks は前哨戦を4連勝で楽に勝ち上がって中9日 → 逆に第1戦の立ち上がりに「試合勘が鈍る」補正。
* **クラッチ**：残り5分・5点差以内で、選手ごとの勝負強さ倍率を発動。Brunson は別格、ルーキーの Castle や Champagnie もクラッチ強め、ウェンビーも強い、という設定。
* **ケガ**：Mitchell Robinson は右小指骨折でブレースをつけながらプレー → 効率にペナルティ、など。

## 完成した結果

完成した結果をまとめた Web サイトが下記となります。（再掲）

<https://nba-finals-sim.vercel.app/>

NBA 好きな私が見ても、AI が作ったと言われなければわからないくらい、かなりリアルなシミュレーション結果となりました。特に面白かった点を2つ紹介します。

### 1: 監督エージェントのリアルさ

監督エージェントは試合の戦略・選手交代等を行うんですが、これが絶妙にリアルでした。  
NBA だけではなく、どのスポーツにおいてもチームごとの特徴ってあると思います。例えば「このチームはオフェンスが得意だけどディフェンスが苦手」とか。

今回のシミュレーションでも、このチームの癖や監督の癖がリアルに表現されており、これは**通常のシミュレーションでは出ずに今回の AI エージェントならではの結果**になったと思いました。

例えば監督がタイムアウト（試合を止めて指示を出す権利）を切ったタイミングも、ちゃんと「連続失点を止めるため」に使われており、切るタイミングがまさに本当の監督のような使い方でした。

NBA 見てないとそこが伝わりにくいかもですが、一応下記にハーフタイムやクラッチタイムでの監督の指示などが記載されていますのでご参考までに。  
![](https://static.zenn.studio/user-upload/542af52c1dd0-20260608.png)

### 2: 各選手の思想までリアル

これも先ほどと似てますが、選手エージェント（各選手）が本当の選手のように動きました。  
それは得点やリバウンドなどのスコアだけではなく、試合中の思想が本当の選手のようなもので動いていました。

例えば、**第3戦（108-107 の1点差）** の終盤、両チームの選手エージェントはこう動きました。

> **Spurs**：「Get me the ball, we runnin' this（ボールをくれ、回すのは俺だ）」と Fox に終盤を委ねる。Castle はドラッグスクリーンからのセカンドアクション、Wembyは明確なミスマッチのときだけポストアップ。

> **Knicks**：「Run it through me. Clock hits zero, we're done.（全部俺を通せ。時計が0になったら終わりだ）」と Brunson のピック＆ロール／アイソで締める。Bridges は Fox にフルコートで張り付かせる。

細かいところは割愛するんですが、選手が発した言葉や、実際に取った各選手の行動や戦略まで、完全に思想まで同じものとなり鳥肌が立ちました。  
数字ではなく「物語」が出てくる瞬間で、ここもAIを足したおもしろポイントでした。

## おわりに

面白半分で作ってみましたが、実際は普通のシミュレーションでは出せない結果となり、NBA ファンとしても技術者としても満足のいく検証となりました。

技術的に大きかったのは、最初の **「LLMに全部やらせない」** という割り切りでした。AIは万能の計算機ではなく、「人間が悩む意思決定」を任せる相手だと考えると、急に設計がスッキりします。算数はコードに、判断はAIに。この線引きは、たぶんNBA以外のシミュレーションでもそのまま効くかなと思いました。

実際の結果が出たら、本シミュレーションとどう違ったのかを比較してまた色々検証してみようと思います。
