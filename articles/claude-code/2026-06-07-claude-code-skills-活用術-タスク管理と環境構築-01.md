## はじめに

「Claude Code に話しかけるだけで、ログが記録されてバックログが更新されたら最高じゃないですか？」

そう思って試行錯誤した結果、**7 個のカスタムスキル**を自分で実装して運用するようになりました。  
この記事では、実際に動いているスキルの構成と、それを支えるツール連携を紹介します。

Claude Code のカスタムスキルは `~/.claude/skills/` 配下に `SKILL.md` を置くだけで使えます。  
自然言語でトリガーできるので、コマンドを覚えなくてもよいのが気に入っています。

---

## スキル構成の全体像

現在 7 個のスキルを運用しており、大きく以下の役割を担っています。

| カテゴリ | スキル | 一言説明 |
| --- | --- | --- |
| タスク管理 | log, implement-backlog, inbox-capture | 記録・実装・取り込みを自動化 |
| プロジェクト基盤 | setup, python-project-setup | 新規プロジェクトの雛形を即生成 |
| ドキュメント | docs-review, claude-status-article | レビューと現状把握を効率化 |

以下の紹介順は、実際のプロジェクトワークフローに対応しています。

```
① setup / python-project-setup   ← プロジェクトを立ち上げる
         ↓
② inbox-capture                  ← アイデア・タスクをバックログへ流す
         ↓
③ implement-backlog              ← バックログを実装する
         ↓
④ log                            ← 作業を記録・整理する
         ↓
⑤ docs-review                   ← 成果物をレビューする
         ↓
⑥ claude-status-article         ← 運用状況を整理・発信する
```

---

## 各スキルの紹介

### 1. setup（プロジェクト初期化）

> カレントディレクトリにプロジェクトの基本構造（ディレクトリ・設定ファイル群）を一括生成するスキル。

**トリガー例**: 「セットアップして」「初期化して」「新しいプロジェクトを始めたい」

**処理の流れ**（3 ステップ）:

* `git init` と `.claude/`・`works/` ディレクトリを作成する
* `.gitignore`・`CLAUDE.md`・`README.md`・`progress_report.md`・`project_context.md` を生成する
* 作成したファイル・ディレクトリを一覧で報告する

**こんな場面におすすめ**: 新しいリポジトリを作るたびに毎回同じファイルを手で用意していた人に最適です。「セットアップして」の一言でプロジェクト管理の土台が整います。

#### 生成されるテンプレートファイル

`setup` スキルで生成される 2 つの管理ファイルのテンプレートを紹介します。

**`progress_report.md`** — 作業ログの蓄積先。`/log` スキルが自動追記します。

```
# Progress Report

このファイルは `/log` スキルによって自動追記されます。
初期状態のままでよく、手動編集は不要です。

---

# 2026-05-21

## 10:00~10:45 feat: ユーザー離脱予測モデルの特徴量エンジニアリング
  - 関連ファイル:
    - `src/features/user_activity.py`
    - `notebooks/feature_exploration.ipynb`
  - **解決した課題:** 過去7日間のアクセス頻度が特徴量として未実装で、ベースラインのAUCが0.78に留まっていた
  - **解決方法:** user_activity.py に rolling window 集計ロジックを実装し、7日・14日・30日の頻度特徴量を追加した
  - **残課題:** XGBoostとのアンサンブル検証、および少数クラスに対するオーバーサンプリング手法の検討
```

**`project_context.md`** — プロジェクト方針とバックログの一元管理ファイル。

```
# プロジェクト方針と課題管理 (Project Context)

## 📍 現状 (Current Status)

### プロジェクトの目的
（例：ユーザーの離脱率を予測し、マーケティング施策の最適化を図る）

### 目指す状態・To-Be（評価指標と合格水準）
- 評価指標: ROC-AUC
- 合格水準: AUC 0.85以上
- ビジネス要件: 推論レイテンシ100ms以内

### 現在のベースライン・As-Is
- 最新スコア: AUC 0.78 (ベースラインモデル: LightGBM)
- 現状の課題: 少数クラスの予測精度が低い

### 制約条件・前提事項
- 使用可能な特徴量は前日までのログデータのみ

---

## ⏩ 未来 (Future & Backlog)

### 当面のネクストアクション
- 特徴量エンジニアリング：ユーザーの過去7日間のアクセス頻度を集計する処理の実装

### 残課題リスト (Backlog)

| ID  | 優先度 | 大分類  | 中分類   | 施策内容                       | 想定作業量 | 実現可能性 | ステータス  |
| :-- | :----- | :------ | :------- | :----------------------------- | :--------- | :--------- | :---------- |
| T001 | 高    | 特徴量  | 行動ログ | 過去7日間のアクセス頻度集計    | 中         | 高         | In Progress |
| T002 | 高    | モデル  | 手法     | XGBoostとのアンサンブル検証    | 小         | 高         | Not Started |
```

### 2. python-project-setup（Python 環境構築）

> `uv` を使って Python 仮想環境を構築し、`CLAUDE.md` と `README.md` に Python 関連情報を自動追記するスキル。

**トリガー例**: 「Python 環境を作って」「uv でセットアップして」「/python-project-setup」

**処理の流れ**（4 ステップ）:

* `uv init` と `uv venv` で仮想環境を構築する
* `CLAUDE.md` に Python 環境情報を追記する
* `README.md` にセットアップ手順を追記する
* 仮想環境の有効化コマンドを含めて完了を報告する

**こんな場面におすすめ**: Python プロジェクトを始めるたびに環境構築でつまずく人におすすめです。`uv` を使った高速な環境構築と、ドキュメント更新まで一度で済みます。

---

### 3. inbox-capture（Inbox タスク取り込み）

> Flow Launcher で書き溜めた `Inbox.md` のメモから指定タグの行を抽出し、`project_context.md` の残課題テーブルへ自動追加するスキル。

**トリガー例**: 「inboxを処理して」「[タグ名]のタスクを取り込んで」「/inbox-capture」

**処理の流れ**（8 ステップ）:

* タグを特定する（引数から、または大分類から推定）
* `Inbox.md` から対象タグの行を抽出する
* 既存テーブルの最大 ID を読み取り、連番を採番する
* 各タスクの優先度・中分類・施策内容を自動で決定する
* 追加予定の内容をユーザーに提示して確認を取る
* `project_context.md` の残課題テーブルに追記する
* `Inbox.md` から処理済み行を削除する
* 追加件数を報告する

**こんな場面におすすめ**: 「思いついたことをとりあえずメモして、あとでプロジェクトに整理したい」という人向けです。メモとバックログの間にある手作業を自動化してくれます。

---

### 4. implement-backlog（バックログ自動実装）

> `project_context.md` の優先度「高」・ステータス「Not Started」のタスクを自動で実装し、仕様書作成からステータス更新まで一気通貫で行うスキル。

**トリガー例**: 「残課題を実装して」「高優先度タスクを実装して」「/implement-backlog」

**処理の流れ**（6 ステップ）:

* 残課題テーブルから優先度「高」かつ `Not Started` のタスクを抽出する
* `works/YYYYMMDD_improvement_vN.md` として仕様書を作成する
* 仕様書に沿って各タスクを実装する
* ビルドシステムを検出して動作確認を行う
* `update_statuses.py` でステータスを `Completed` / `In Progress` に更新する
* 実装結果と手動作業が残るタスクをまとめて報告する

**こんな場面におすすめ**: バックログを溜め込みがちな人に特におすすめです。「残課題を実装して」と言うだけで、優先度の高いものから自律的に片付けてくれます。

---

### 5. log（作業ログ記録）

> git コミット履歴を読み取り、`progress_report.md` に構造化ログを追記しながら、`project_context.md` の残課題リストも同時に更新するスキル。

**トリガー例**: 「ログを記録して」「進捗をまとめて」「/log」

**処理の流れ**（7 ステップ）:

* git ログを収集してコミット情報を JSON で取得する
* 各コミットの「解決した課題・解決方法・残課題」を自然言語で言語化する
* 新規タスクの候補を抽出する
* エントリ内容を JSON ファイルに書き出す
* `progress_report.md` にログを追記・コミット SHA を保存する
* `project_context.md` に新規タスク行を追記する
* 残課題テーブルを優先度・ステータス順にソートする

**こんな場面におすすめ**: 毎日の作業終わりに「今日何をしたか」をまとめたいときに便利です。コミットさえしていれば、あとは Claude が自動で振り返りを書いてくれます。

---

### 6. docs-review（ドキュメントレビュー）

> 設計書・仕様書などのドキュメントを読み込み、整合性・網羅性・可読性の3観点から懸念点を洗い出してレポートを出力するスキル。元のドキュメントは一切変更しない。

**トリガー例**: 「ドキュメントをレビューして」「懸念点を洗い出して」「/docs-review」

**処理の流れ**（3 観点のチェックリスト形式）:

* 整合性・コンフリクト（ドキュメント間・内の矛盾、用語ブレ）を確認する
* 網羅性・正確性（記述の不足、未確定事項、参照エラー）を確認する
* 可読性・認知負荷（重複、抽象的表現、情報の詰め込み）を確認する

**こんな場面におすすめ**: 自分で書いた仕様書をリリース前にセルフチェックしたいときに便利です。「自分では気づきにくい矛盾や抜け漏れ」を第三者視点でピックアップしてくれます。

---

### 7. claude-status-article（実装状況記事生成）

> `~/.claude/skills/` 配下のスキル群を読み取り、現在の Claude Code カスタムスキル実装状況を Note/Qiita 向けのマークダウン記事として出力するスキル。

**トリガー例**: 「実装状況をまとめて」「Claude の実装を記事にして」「/claude-status-article」

**処理の流れ**（4 ステップ）:

* 各スキルの `SKILL.md` を読み込んで目的・ステップ数・トリガーを収集する
* 固定テンプレートにスキル情報を埋め込んで記事を生成する
* `works/YYYYMMDD_claude_article.md` に書き出す
* 対象スキル数と出力ファイルパスを報告する

**こんな場面におすすめ**: スキル構成が安定してきたタイミングで「今の運用を整理して発信したい」ときに便利です。記事の骨格を自動で作ってくれるので、あとは細部を磨くだけで済みます。

## 使用ツールの紹介

このスキル群は以下の外部ツールと連携しています。

### Flow Launcher

公式サイト: <https://www.flowlauncher.com/>

Windows 向けのオープンソースアプリランチャー。キーボードショートカット一発でどこからでも起動でき、  
カスタムプラグインを使えばメモのクイックキャプチャが可能です。

**このスキル構成での使い方**: `inbox-capture` スキルとの連携。  
Flow Launcher のプラグインで思いついたタスクを `Inbox.md` に即時書き込み、  
あとから `/inbox-capture` を呼ぶことでプロジェクトのバックログへ自動追加します。

**おすすめ度**: アイデアをその場でメモして、あとでプロジェクトに紐付けたい人に最適です。

## まとめ

Claude Code のカスタムスキルを活用すると、「話しかけるだけで定型作業が走る」体験が作れます。  
最初の一歩は `setup` スキルを試してみることをおすすめします。  
`~/.claude/skills/setup/SKILL.md` に数十行の指示を書けば、  
新規プロジェクトの雛形生成が自然言語で完結するようになります。

今後はスキル間の連携をさらに深めて、「毎朝の作業開始フロー」を自動化する予定です。  
興味があればぜひ試してみてください。
