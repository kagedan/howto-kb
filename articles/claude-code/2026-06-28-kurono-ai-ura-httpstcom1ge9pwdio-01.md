# ■ 9割の人がClaude Codeの「本当の使い方」を知らずに損してる

![](https://pbs.twimg.com/media/HL4BVm4a8AAudgD.jpg)

## ▍「プロンプトが上手い」だけじゃもう勝てない。時代は"コンテキストエンジニアリング"

「プロンプトを上手に書けば、AIの出力が良くなる」。これ、半分正解で半分古いです。

今の時代は「プロンプトエンジニアリング」から**「コンテキストエンジニアリング」**に変わっています。

何が違うのか。プロンプトエンジニアリングは「1回の指示を上手に書く技術」。チャットを開いて、その場で上手い指示を考える。毎回ゼロからやり直す前提のアプローチです。

コンテキストエンジニアリングは違います。**プロジェクト全体の「文脈」を設計して、AIが常に最適な情報を持っている状態を作る技術**です。

わかりやすく言い換えます。

- **プロンプトエンジニアリング**: 毎回「俺はこういう人で、こういうプロジェクトで、こういうルールで…」と説明する
- **コンテキストエンジニアリング**: 1回設定したら、AIが勝手に「全部わかってる状態」でスタートしてくれる
僕はClaude Code 1,500時間以上使ってきましたが、毎回プロンプトを工夫するフェーズはとっくに卒業しています。

今やってるのは「仕組みで伝わる状態」を作ること。**だからAIエージェント100体以上が、僕の指示なしで勝手に動けるんです。**

## ▍Claude Codeが持つ「5層の記憶システム」を全部使え

![](https://pbs.twimg.com/media/HL4BVm0aQAABfre.jpg)

Claude Codeには5つの「記憶の層」があります。ほとんどの人は一番下の「会話」しか使っていません。

上から順に影響力が強いです。

- **System Prompt（システムプロンプト）**: Claude Code自体が持つ基本設定。ユーザーは触れない
- **CLAUDE.md**: プロジェクトに置く「永続的な指示書」。セッション開始時に自動で読み込まれる
- **.claude/rules/**: 特定のファイルを触ったときだけ発動するルール
- **memory**: セッションを跨いでAIが学習・記憶する仕組み
- **会話（Conversation）**: 今まさにやり取りしている内容
ほとんどの人は5番目の「会話」だけで全部やろうとしている。毎回ゼロから口頭で伝えてるのと同じ。ブレて当然です。

**5層全部を使っている人と、会話だけの人では、出力品質が10倍違います。** これは体感ではなく、実際にチームで検証して出ている数字です。

## ▍CLAUDE.mdを書いてない人はAIを半分しか使えてない

CLAUDE.mdがない状態でClaude Codeを使うのは、毎回初対面のコンサルタントに最初から説明し直しているのと同じです。

CLAUDE.mdを1つ置くだけで、AIは「わかってる状態」でスタートしてくれます。

具体的にどれくらい変わるか。

- セッションごとに**トークン消費が20〜30%減る**（毎回の説明が不要になるから）
- コードレビューの指摘が**70%減った**報告
