# Claude CLI オプション完全ガイド — コマンド一覧と実践的な使い方

Claude Code（`claude` コマンド）には多数のオプションが用意されています。本記事では各コマンドの実際の使い方をコード例中心で解説します。

---

## サブコマンド

### `claude auth` — 認証管理

Anthropicアカウントの認証を管理します。

```
# 初回ログイン
claude auth login

# SSO でログイン（メールを事前入力）
claude auth login --email user@example.com --sso

# 認証状態の確認（人間向け形式）
claude auth status --text

# ログアウト
claude auth logout
```

---

### `claude agents` — エージェント一覧

設定済みのエージェントを一覧表示します。`--agent` オプションで指定できるエージェント名を確認したいときに使います。

```
claude agents
claude agents --setting-sources user   # 読み込む設定ソースを指定
```

---

### `claude mcp` — MCPサーバー管理

MCP（Model Context Protocol）サーバーの追加・削除・一覧表示を行います。Figma・Slack・データベース等の外部サービスとの連携設定をここで管理します。設定は `~/.claude/settings.json` に保存されます。

```
# サーバーを追加（対話形式）
claude mcp add

# JSON形式でサーバーを追加
claude mcp add-json '{"name": "my-server", "command": "node", "args": ["/path/to/server.js"]}'

# 設定済みサーバーの一覧を確認
claude mcp list

# 特定サーバーの設定を確認
claude mcp get my-server

# サーバーを削除
claude mcp remove my-server

# Claude Desktop の設定を移行
claude mcp add-from-claude-desktop
```

---

### `claude plugin` — プラグイン管理

Claude Code のプラグイン（スキル）をインストール・管理します。スキルとは `/commit` や `/pr-respond` 等のスラッシュコマンドを定義するパッケージです。

```
# プラグインをインストール
claude plugin install @anthropic/pr-tools

# インストール済みプラグインを確認
claude plugin list

# プラグインを一時的に無効化
claude plugin disable my-plugin

# 再度有効化
claude plugin enable my-plugin

# アップデート
claude plugin update

# マーケットプレイスを確認
claude plugin marketplace
```

---

### `claude remote-control` — リモートコントロールサーバー起動

claude.ai または Claude アプリから Claude Code をリモート操作できるサーバーとして起動します。

```
claude remote-control --name "My Project"
```

インタラクティブセッションを起動しつつリモート操作も受け付けたい場合は、オプションの `--remote-control` / `--rc` フラグを使います。

---

### `claude setup-token` — 長期認証トークン設定

ブラウザ認証を使わず、長期間有効なトークンでログインし続けるためのセットアップです。CI環境やサーバー環境で長期運用する際に便利です。

**必要条件:** Claude のサブスクリプション（Pro以上）。

---

### `claude doctor` — 自動アップデーター診断

自動アップデーターが正常に動作しているかチェックします。`claude update` が失敗する場合や、バージョン管理に問題が起きたときに使います。

---

### `claude install` — ネイティブビルドのインストール

特定バージョンのネイティブビルドをインストールします。通常の `claude update` と異なり、バージョンを明示的に指定してインストールしたい場合に使います。

```
claude install           # デフォルト（stable）
claude install stable    # 安定版
claude install latest    # 最新版
claude install 1.2.3     # バージョン指定
claude install --force   # 強制再インストール
```

---

### `claude update` — アップデート

最新バージョンを確認してインストールします。

---

## オプション

### 1. セッション管理

Claude Code はセッション（会話履歴）を保存します。中断した作業の再開や、特定の文脈を引き継いだ作業に役立ちます。

```
# 直近の会話を継続
claude -c

# 名前でセッションを保存 → 後で再開
claude -n "BIZ-37対応"
claude -r "BIZ-37対応"

# PRに紐づいたセッションを再開
claude --from-pr 10389

# 元セッションを保持しつつ分岐
claude -c --fork-session

# セッションを保存しない（-p使用時のみ）
claude -p --no-session-persistence "質問"

# 特定のセッションIDで起動
claude --session-id <uuid>
```

**便利な使い方：** タスクごとにセッション名をつけておくと、数日後でも `-r` で文脈ごと復元できます。

---

### 2. 出力・実行モード

インタラクティブな対話が不要な場面では `-p` フラグが便利です。CI/CDやシェルスクリプトとの組み合わせに適しています。

```
# レビュー結果をファイルに保存
claude -p "このPRをレビューして" > review.md

# JSON形式で出力してjqで処理
claude -p --output-format json "バグを特定して" | jq '.result'

# 最大精度モードで実行（Opus 4.6 専用）
claude --model opus --effort max "このアーキテクチャ設計をレビューして"

# コスト上限とターン数の上限を設定
claude -p --max-budget-usd 0.5 --max-turns 10 "このコードを分析して"

# 構造化出力
claude -p --json-schema '{"type":"object","properties":{"bugs":{"type":"array"}}}' "バグを列挙して"
```

**`--effort` の使い分け：**

* `low` / `medium` — 簡単な質問・コード補完など素早く済ませたい場合
* `high` / `max` — 複雑なデバッグ・設計レビューなど精度優先の場合（`max` は Opus 4.6 のみ）

---

### 3. ツール・権限制御

Claude が使えるツール（ファイル編集・コマンド実行など）を細かく制御できます。

```
# 読み取りと検索のみ許可（安全な調査用）
claude --allowedTools "Read,Grep" -p "このコードの問題を調査して"

# git コマンドのみ許可
claude --allowedTools "Bash(git *)" "コミット履歴を分析して"

# ファイル編集は自動許可でサクサク作業
claude --permission-mode acceptEdits

# CI環境など全自動で実行したい場合
claude --permission-mode dontAsk -p "テストを実行して結果を報告して"

# ツールを全て無効にして純粋な会話モード
claude --tools ""

# プランモードで権限バイパスを組み合わせる
claude --permission-mode plan --allow-dangerously-skip-permissions
```

**`--permission-mode` の種類：**

| モード | 動作 |
| --- | --- |
| `default` | 危険な操作は都度確認（デフォルト） |
| `acceptEdits` | ファイル編集は自動許可、それ以外は確認 |
| `dontAsk` | 全操作を自動許可 |
| `bypassPermissions` | 権限チェックを完全スキップ |
| `plan` | プランモード（実行せず計画のみ提示） |
| `auto` | Claude が自動判断 |

---

### 4. モデル指定

タスクの重要度やコストに応じてモデルを切り替えられます。

```
# 最高精度モデルで設計レビュー
claude --model opus "このシステム設計をレビューして"

# 高速なHaikuで簡単な質問
claude --model haiku -p "この関数の引数を説明して"

# 過負荷時はHaikuにフォールバック
claude -p --model opus --fallback-model haiku "分析して"
```

**モデルエイリアス：**

| エイリアス | 実際のモデル | 用途 |
| --- | --- | --- |
| `sonnet` | claude-sonnet-4-6 | バランス型（デフォルト） |
| `opus` | claude-opus-4-6 | 最高精度・複雑なタスク |
| `haiku` | claude-haiku-4-5 | 高速・軽量タスク |

---

### 5. プロンプト・エージェント制御

Claude の動作をカスタマイズしたり、専用エージェントを設定したりできます。

```
# 特定の役割を与えて起動
claude --system-prompt "あなたはセキュリティ専門家です。脆弱性の観点でレビューしてください。"

# ファイルからシステムプロンプトを読み込み
claude --system-prompt-file ./prompts/review.txt

# 追記形式（既存の機能を保持しつつ追加）— こちらが推奨
claude --append-system-prompt "常にTypeScriptで回答してください。"
claude --append-system-prompt-file ./style-rules.txt

# エージェントチームをtmuxで表示
claude --teammate-mode tmux

# スキル（/commit等）を無効化
claude --disable-slash-commands
```

> **使い分け：** `--append-system-prompt` が推奨。組み込み機能を保持しつつ要件を追加できます。`--system-prompt` は完全に置き換える場合のみ。

---

### 6. Web・リモート連携

claude.ai やモバイルの Claude アプリと連携してセッションを管理できます。

```
# ログインバグの修正をWebセッションとして開始
claude --remote "Fix the login bug"

# ローカルで作業しつつモバイルからも操作できるように
claude --remote-control "My Project"

# Webセッションをローカルに引き継ぐ
claude --teleport
```

---

### 7. Git Worktree 連携

現在のリポジトリから独立した作業環境（worktree）を作成できます。メインブランチを汚さずに実験的な作業ができます。

```
# 隔離環境で機能を試す
claude -w feature-experiment

# tmuxで並行作業
claude -w hotfix-v2 --tmux
```

---

### 8. MCP・設定の読み込み

MCP サーバーや追加設定を柔軟に読み込めます。

```
# 特定のMCPサーバー設定を読み込む
claude --mcp-config ./my-mcp-servers.json

# プロジェクト設定のみ読み込む
claude --setting-sources project

# 作業ディレクトリを追加
claude --add-dir /tmp/workspace "このディレクトリのファイルを分析して"

# Chrome連携でWebテスト
claude --chrome "このページのフォーム動作を確認して"
```

---

### 9. デバッグ

動作がおかしいときや、ツール呼び出しの詳細を確認したいときに使います。

```
# 全デバッグ情報を表示
claude -d

# API関連とhooksのみ表示
claude --debug "api,hooks"

# 特定カテゴリを除外（先頭に ! をつける）
claude --debug "!statsig,!file"

# ファイルに保存して後で確認
claude --debug-file /tmp/claude-debug.log
```

---

### 10. その他

```
# バージョン確認
claude -v

# 初期化フックのみ実行（CI/CDでのセットアップ用）
claude --init-only
```

## 実践パターン集

### 中断した作業を再開する

```
# 昨日の作業を名前で再開
claude -r "認証機能リファクタリング"

# 直近の会話をそのまま続ける
claude -c
```

### CI/CDに組み込む

```
# PRの差分をレビューしてコメントを作成
git diff main...HEAD | claude -p --input-format text "このdiffをレビューしてMarkdownで出力して" > pr_review.md

# テスト失敗の原因を分析
npm test 2>&1 | claude -p "このエラーの原因と修正方法を教えて"

# ターン数とコストを制限してCI実行
claude -p --max-turns 20 --max-budget-usd 1.0 "テストを実行して失敗を修正して"
```

### セキュアな読み取り専用調査

```
# 編集権限を与えずコードを調査
claude --allowedTools "Read,Grep,Glob" "この認証周りのコードにセキュリティ上の問題があるか調査して"

# 設計の把握のみ（何も変更させない）
claude --tools "" "このシステムのアーキテクチャを説明して"
```

### タスクの重要度でモデルを使い分ける

```
# 軽い質問はHaikuで素早く
claude --model haiku -p "この関数の戻り値の型は？"

# 重要な設計判断はOpusで丁寧に
claude --model opus --effort max "このデータベース設計を見直して、スケーラビリティの問題を指摘して"
```

### 実験的な作業を安全に行う

```
# worktreeで本番コードを汚さず実験
claude -w experiment-new-algo

# 確認なしで自動作業（スクリプト的に使う）
claude --permission-mode dontAsk -p "このRubyファイルのN+1クエリを全て修正して"
```

### カスタムロールで専門的なレビュー

```
# セキュリティレビュー専用ロール
claude --append-system-prompt "セキュリティの専門家として、OWASP Top 10の観点でのみ指摘してください。" \
  -p "このコードをレビューして"

# パフォーマンスレビュー専用
claude --append-system-prompt "パフォーマンスエンジニアとして、N+1クエリ・メモリリーク・不要な計算を中心にレビューしてください。" \
  "このコードを確認して"
```

### Webセッションとローカルを行き来する

```
# モバイルのClaudeアプリでもローカル作業を操作できるように
claude --remote-control "My Project"

# Webセッションで作業してからローカルターミナルに引き継ぐ
claude --teleport
```

## まとめ

| やりたいこと | 使うオプション |
| --- | --- |
| 前の会話を続けたい | `-c` / `-r` |
| スクリプトや自動化に使いたい | `-p` + `--output-format` |
| ファイルを触られたくない | `--tools ""` or `--allowedTools "Read,Grep"` |
| 確認ダイアログなしで動かしたい | `--permission-mode dontAsk` |
| 精度を最大にしたい | `--model opus --effort max` |
| 作業ブランチを隔離したい | `-w` |
| モバイルからも操作したい | `--remote-control` |
| Webセッションをローカルに引き継ぐ | `--teleport` |
| コスト・ターン数を制限したい | `--max-budget-usd` / `--max-turns` |
| デバッグしたい | `-d` / `--debug-file` |
