## 1. 導入：AIは「魔法の杖」ではなく「新人ロボット」

「AIに頼めば、何でも完璧にやってくれる」と思っていませんか？実は、最新のコーディングアシスタント（Claude Codeなど）の実体は、**「物覚えは良いけれど、まだ世間知らずな新人ロボット」** です。

このロボットをただ動かすのではなく、いかに「教育」し「管理」するか。本記事では、Anthropic社が提供する公式学習コンテンツ[「Claude Code in Action」](https://anthropic.skilljar.com/claude-code-in-action/303242)をベースに、ロボットを使いこなすための3つの極意を解説します。

## 2. 本論：ロボットを賢く動かす「3つの仕組み」

### ① 「ダメ！」を教える安全装置（Hooks）

ロボットは命令されると、一生懸命すぎて周りが見えなくなることがあります。そこで、行動の「前後」にルールを設けます。

* **やる前のルール：** 「秘密のファイルは触っちゃダメ！」と事前に設定します。ロボットが誤って手を伸ばした瞬間、システムが強制停止させます。
* **やった後のルール：** 「終わったら必ずテストを回して！」というルールです。ミスがあれば、ロボット自身に「やり直し！」と反省させることができます。

**具体的な設定例**

```
{
  "hooks": {
    "pre-command": "echo '機密ファイル（.env等）へのアクセスを制限しています...'",
    "post-command": "npm test"
  }
}
```

このように設定しておくことで、ロボットがコードを書いた後に自動でテストが走り、品質を担保できます。

### ② 脳を疲れさせない「正しいメモ」の渡し方（Context）

ロボットの脳（AI）は、一度にたくさんのことを言われるとパニックになります。情報を整理して渡すことが、精度の高い回答を引き出す鍵です。

* **プロジェクトの地図（CLAUDE.md）：**  
  この仕事で必要なことだけをまとめた1枚のメモを用意します。教科書を丸ごと渡すより、要点を絞った「地図」をプロジェクトのルート直下に置いておきましょう。

CLAUDE.mdのテンプレート例

```
# Project: [プロジェクト名]

## Build & Test Commands
- Build: `npm run build`
- Test: `npm test`
- Lint: `npm run lint`

## Technical Stack
- Language: TypeScript
- Framework: Next.js (App Router)
- UI: Tailwind CSS

## Code Style & Rules
- 変数名はキャメルケース（camelCase）を使用すること。
- コンポーネントは `src/components` 配下に機能ごとに配置。
- 非同期処理は必ず `async/await` を使用し、エラーハンドリングを徹底する。
```

* **脳の掃除：** 会話が長くなって話がゴチャゴチャしてきたら、スラッシュコマンドを使って頭をスッキリさせてあげましょう。
  + `/compact`：今までの会話の流れをギュッと要約し、メモリを節約します。
  + `/clear`：今の文脈を一度リセットし、新しい作業に集中させます。

### ③ 新しい「道具」でスキルアップ（Extensions）

最初は「読み書き」しかできなかったロボットも、後から新しい道具を握らせることで進化します。

* **ブラウザという「目」：** ネットを見る道具を持たせれば、最新のライブラリ情報を調べたり、「デザインがズレているな」と自分で気づいて直したりできるようになります。
* **無限の可能性（MCP）：** **[MCP（Model Context Protocol）](https://modelcontextprotocol.io/)** という共通規格を利用すれば、Google Drive、Slack、GitHub、あるいは独自のデータベースなどとロボットを繋ぐことができます。  
  これにより、ロボットを単なるコード書きから、社内情報まで把握した自分専用の「最強の職人」へとカスタマイズできるのです。

## 3. 結論：あなたがロボットの「監督」になろう

この資料が教えてくれる一番大切なこと。それは、**「AIのすごさは、使う側の『管理能力』で決まる」** ということです。

1. **ルール（Hooks）** で安全を守る。
2. **正しい情報（Context）** で迷わせない。
3. **新しい道具（Extensions）** で可能性を広げる。

これからのエンジニアに求められるのは、コードを書く力だけでなく、AIという相棒を上手に導く「監督（マネージャー）」としての視点なのです。
