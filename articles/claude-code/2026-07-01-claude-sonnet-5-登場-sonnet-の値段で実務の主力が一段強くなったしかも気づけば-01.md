2026 年 6 月 30 日、Anthropic が **Claude Sonnet 5** をリリースした。

ニュースとしては地味に見えるかもしれない。「Sonnet が 4.6 から 5 になった」だけ、と。けれど Claude を日常的に使っている人にとって、今回はいくつか見逃せないポイントがある。しかも、多くの人は **すでに知らないうちに Sonnet 5 を使い始めている**。順番に整理する。

なお、ネット上には早くも「SWE-bench 92.4%」「コードネーム Fennec」「2M コンテキスト」といった景気のいい数字が出回っているが、**これらは公式の一次情報では確認できない**。この記事では、Anthropic 公式で裏が取れた事実だけを扱う。

## まず、あなたはもう使っている

いちばん実用的な事実から。Claude Sonnet 5 は、claude.ai の **Free プランと Pro プランの新しいデフォルトモデル**になった。

つまり、特に何も設定を変えていなければ、6 月 30 日以降に claude.ai で会話している相手は、もう Sonnet 5 だ。「新モデルを試そう」と身構える前に、すでに手元で動いている。まずはこれを認識しておくといい。

Claude Code でも利用できる。モデル ID は日付なしの **claude-sonnet-5**(AWS Bedrock や Google Cloud でも同じ系統の ID)。指定して使う場合はこれを覚えておけばいい。

## 何が「一段強くなった」のか

今回の本質は、**Sonnet ティア(中位)の実力が、また一段上がった**ことだ。

Claude のモデルは大きく、軽量・高速の Haiku、バランスの Sonnet、その上の Opus、さらにフロンティアの Fable / Mythos という階層になっている。これまで「本気の作業は上位の Opus、日常は Sonnet」という使い分けをしていた人は多いと思う。

Sonnet 5 は、その「Sonnet で済む範囲」を押し広げてきた。価格は据え置きのバランスティアのまま、こなせる仕事の質が上がっている。**上位ティアをわざわざ呼び出さなくても、Sonnet 5 で足りる場面が増える**——これが今回いちばん効く変化だ。

Claude モデル地図（2026年7月時点）。最上位は停止中、実務の主力は Sonnet 5

具体的な仕様で、公式に確認できる範囲を挙げておく。

**1M(100 万)トークンのコンテキスト**。しかも追加課金(長文サーチャージ)なしの標準価格内。大きめのコードベースや長い資料を、まとめて文脈に入れて扱える。

**最大 128K トークンの出力**(バッチ処理では最大 300K)。長い生成物を途中で切らさず出せる。

**Adaptive thinking が常時オン**で、Claude Code や API では「努力量(effort)」が high がデフォルト。難しい問題に自動で深く取り組む構えが標準になっている。

## 価格の「見かけ」と「実際」

ここは少し注意して読んでほしい。

Sonnet 5 の **標準価格は、入力 100 万トークンあたり $3、出力 $15**。これは Sonnet 4.6 と同じ水準だ。

ただし今は **導入価格として、2026 年 8 月 31 日まで入力 $2 / 出力 $10** に下がっている。9 月 1 日から標準の $3 / $15 に戻る。つまり **7〜8 月は通常より安く使える期間**ということになる。

一方で、見落としやすい注意点がある。Sonnet 5 では **トークナイザ(文章をトークンに刻む仕組み)が新しくなり、同じ文章でもトークン数が従来より約 3 割多くカウントされる**、という公式の但し書きがある。

これが意味するのはこういうことだ。**「1 トークンあたりの単価が安く見えても、同じ作業で消費するトークンが増えるなら、実際の請求はその分だけ相殺される」**。導入価格の安さだけを見て「コストが下がった」と早合点せず、自分の実ワークロードで実際のトークン消費を一度測ってみるのが正解だ。

## 文脈 —「最強」が止まっている今だからこそ

ここで前回の記事を読んだ人向けの補足を。

6 月、Anthropic は最上位モデル **Claude Fable 5** を投入したが、その 3 日後の 6 月 12 日、米政府の輸出管理指令を受けて Fable 5(および Mythos 5)へのアクセスを全停止した。そして **本稿時点でも、その停止を解除したという公式アナウンスは確認できていない**。

つまり今は、「最強ティアが使えない」状態が続いている。その状況で、**実務で確実に・すぐ回せる主力**として Sonnet 5 が一段強くなって出てきた、という巡り合わせは大きい。派手な最上位モデルが地政学で止まっても、日々の仕事は中位ティアの底上げで着実に前に進む——今回はそれを象徴する出来事でもある。

## Claude ユーザーが今やること 3 つ

最後に実用アクションを 3 つ。

**ひとつ、デフォルトが変わったことを認識する**。claude.ai の Free / Pro はもう Sonnet 5。普段の会話の手応えが変わっているはずなので、「あれ、賢くなった?」を放置せず、意識して使い込む。

**ふたつ、Claude Code で上位ティア(Opus)の出番を見直す**。これまで重い作業を Opus に振っていた人は、まず Sonnet 5 でどこまで足りるかを試す。足りるなら、コストと速度の両方で得をする。

**みっつ、導入価格の期間(〜8/31)に、自分のトークン消費を測っておく**。安いうちに使い込みつつ、新トークナイザで実際にどれだけトークンを食うかを把握しておけば、9 月以降の標準価格に切り替わったときの月額が読める。値上げとトークナイザ変更が二重で効く前に、自分の「実コスト感」を持っておくと強い。

## 締め

Claude Sonnet 5 は、見出しこそ地味だが、「**日常の主力が、また静かに一段上がった**」という、いちばん効くタイプのアップデートだ。

最上位モデルがニュースを賑わせる裏で、実は仕事の大半を支えているのは中位ティアだったりする。その中位が強くなることは、派手さはなくても、実務には確実に効いてくる。

まずは、もう手元で動いている Sonnet 5 を、ちゃんと意識して使ってみるところから。
