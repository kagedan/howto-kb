## Cowork、結局何がいいのか調べてみた

僕は普段 Claude Code を使っている。ターミナル（黒い画面に文字だけで操作するやつ）で動く、エンジニア向けの AI ツール。

その一方で、非エンジニアの人には「AI に仕事を任せたいなら Cowork がいいよ」と勧めてきた。でも正直に言うと、僕自身は Cowork を本格的に使っていない。Claude Code で足りているから。

頭の中では「Cowork = Claude Code を非エンジニア向けに使いやすくしたやつ」くらいに思っていた。でも、「具体的に何がどう同じで、何がいいのか」はちゃんと説明できないまま勧めていた。これはよくないと思って、調べてみた。

調べてみると、Cowork は **AI に仕事を渡して、成果物をファイルで受け取る** ためのモード。話しかける場所ではなく、タスクをハンドオフする場所。

調べていくと、「非エンジニアに勧めてた理由は、結局これだったのか」とはっきりした。それを渡したい。

---

## 先に結論 — 何が一番いいか

調べた結論を先に置く。僕が Cowork を非エンジニアに勧める理由は、ひとつに尽きる。

> **自分の PC の中のファイルに、Cowork が直接アクセスして読み書きしてくれること。**

「それだけ? ChatGPT でもファイルは作れるよ」と思うかもしれない。そのとおりで、今は ChatGPT も Claude のチャットも、Excel や Word のファイルは作れる。だから「ファイルを返してくれる」だけなら、Cowork だけの特長じゃない。

違うのは、その手前と後ろ。チャットでファイルを作るときは、元データを1個ずつアップロードして、できたものをダウンロードして、自分で正しいフォルダに置き直す。この往復が毎回いる。

Cowork はそれをしない。「Downloads フォルダを整理して」「会議の録音、議事録にして」と話しかけるだけで、自分の PC のファイルを直接読みに行って、書き戻してくれる。公式の言葉でも "read from and write to your local files without manual uploads or downloads"（アップロードもダウンロードもなしに、ローカルファイルを直接読み書きする）と書いてある。

この差が効くのはここ。**ずっと「いつかやらなきゃ」のまま積み上がってた作業が、話しかけるだけで片付く**ようになる。ファイルがどこかのフォルダに眠ったままでも、もう「アップロードする」のひと手間が立ち上がらない。

なぜそう言えるのか、どう使うのかを、このあと順番に書く。

## Cowork のフロー — 計画 → 承認 → 実行

![](https://assets.st-note.com/img/1779670030-97UgDZ4st83JxhNMXeKwqvnP.png?width=1200)

Cowork は4ステップで動く。

1. **ゴールを渡す** — 「このフォルダの会議メモ、全部読んで議事録にして」など、文章でタスクを依頼
2. **計画が提示される** — Claude が「こうやって進めます」と手順をリストで返してくる
3. **承認する** — ユーザーが計画を見て OK を出す
4. **実行 → 成果物が返ってくる** — Claude が裏で手を動かし、できたファイルをくれる

このうち重要なのが **2 と 3**。Chat と違って Cowork は「いきなり動く」のではなく、**動き始める前に必ず計画を見せて、ユーザーの OK を待つ**。これが Cowork の安全装置になっている。

途中で「やりすぎだな」と感じたら止められる。危ない操作（ファイル削除など）は毎回追加の承認を求めてくる。**AI に丸投げ**ではなく、**AI に任せつつ要所で人が判断する**設計になっている。

## Chat と何が違うのか?

![](https://assets.st-note.com/img/1779670063-K6gwtlqsU7Lva1V2xG4MNXQI.png?width=1200)

Claude Desktop の上には「Chat」と「Cowork」のタブが並んでいる。並んでるくらいだから似たもの同士だろう、と最初は思う。でも、用途は別物。

**Chat は AI と会話する場所**。一往復、または短い対話で進む。ファイルも作れるけど、元データはアップロードして、できたものはダウンロードする。あくまで「会話の中で受け渡しする」形。

**Cowork は AI に仕事を渡す場所**。自分の PC のフォルダに直接アクセスして、計画 → 承認 → 実行の作業セッションで進む。数分から数十分。ゴールを渡して、要所で承認する。

同じ「フォルダにある資料を1本の報告書にまとめて」と頼む場合の違いを並べてみる。

**Chat の場合**:

**Cowork の場合**:

* 「この `materials` フォルダの資料、全部読んで1本の報告書にまとめて」と頼む
* 計画が出る（「フォルダ内の資料を読み込み、要点を抽出、1本の報告書に構成」）
* 承認する
* `report.docx` が **`materials` フォルダの中に直接** 作られる

違いは、成果物が出るかどうかじゃない。**アップロードとダウンロードの往復があるか、ないか**。Chat は会話越しに1個ずつ受け渡す。Cowork は自分の作業場（フォルダ）に入って、その場で片付けてくれる。

ここを混同して使うと、Cowork で雑談したり、Chat で長いタスクを依頼したりして「思ったのと違う」になる。

## エンジニア志望の人にとっての意味

これからエンジニアになろうとしている人にとって、Cowork は **Claude Code の前段の練習場** になる。

Claude Code を触ろうとすると、最初にターミナルで詰まる人がいる。インストール、PATH、シェル、コマンドの入力。覚えることが急に増える。エージェント時代の AI の使い方を体に入れる前に、ターミナルの基礎で消耗してしまう。

Cowork は **Claude Desktop アプリのタブを切り替えるだけ** で同じエージェント体験ができる。

* AI にタスクを任せて成果物を受け取る
* 計画を見せられて承認するか判断する
* 進行中に「やりすぎ」を察知して止める
* 接続するデータ範囲を自分で絞る

これらは「エージェント時代の AI と付き合う基本動作」で、Claude Code でも Cowork でも同じように要求される。Cowork でこの型を身につけてから Claude Code に移ると、ターミナルの学習だけに集中できる。

逆ルート（Claude Code から入る）だと、ターミナル + エージェント体験の両方を同時に覚えることになって、最初の数日が重い。

注意点を1つ。Cowork はコードの主戦場ではない。本格的にコードベースを動かすなら、結局 Claude Code に移ることになる。Cowork が効くのは **コードを書く前段の知的作業** — 教材整理、調査、Reading List の優先度付け、学習ログの集計、ドキュメントの整理など。

## AI で何かしたい人にとっての意味

僕が非エンジニアに Cowork を勧めてきた理由は、結局ここにある。Cowork は **非エンジニアにとっての、AI に仕事を任せる最初の現実的な道具** になる。

「現実的」には2つの意味がある。

**1. 操作が現実的** — これまで「AI に仕事を任せる」を本気でやろうとすると、API を叩いたりツールを自分で繋いだりが必要で、エンジニアじゃないと届かなかった。Cowork は Claude Desktop アプリのタブを「Chat」から「Cowork」に切り替えるだけ。文章で頼むだけで自分の PC のファイルを読みに行ってくれる。新しく覚えることがほぼない。

**2. 金額が現実的** — Pro 月 $20。年払いなら $17。これまで「AI に仕事を任せる」を試そうとすると、企業向けの契約や API 従量課金で動かすしかなかった。月 $20 で個人が試せる時代になったのは、結構最近のこと。

このふたつが揃ったから、第4節で並べたユースケースが「いつか試したい」じゃなくて、**今日試せる**ものになった。授業の PDF も、会議の録音も、たまったメールも、自分の PC の中で待ってるのに、これまでは「AI に渡す手間」のほうが「自分でやる手間」より重くて、結局自分でやっていた。Cowork はその天秤をひっくり返す。

非エンジニアが AI で詰まるのは、たいてい「自分の仕事のどこに効くか」がイメージできないとき。Cowork は、その答えが **自分の PC に積み上がってる "いつかやらなきゃ" のフォルダ** だと、具体的なフォルダを指して体験させてくれる。

## Cowork ≠ Computer Use — よく混同される別物

Cowork を調べていると「Computer Use」という言葉も出てくる。これは別物なので、整理しておきたい。

**Cowork（GA 済み、Pro 以上）**:

**Computer Use（research preview、Pro/Max のみ）**:

* 自分の **実画面を Claude が見て操作する**
* macOS のアクセシビリティ + 画面収録の許可が必要
* ブラウザを開いて検索したり、デスクトップアプリを動かしたりできる
* Team / Enterprise では使えない
* 銀行・医療・行政アプリへの権限付与は公式が明確に **禁止**

つまり、「Claude が画面を見ながら一緒に作業する」体験を求めているなら、欲しいのは **Cowork ではなく Computer Use** の方。最初に Cowork を「画面共有的なもの」と誤解しやすい原因がここにある。

非エンジニアが最初に触れるべきは **Cowork**。Computer Use は研究プレビュー段階で、リスクも大きい。慣れてから検討する順番で十分。

## つまずきやすいところ

Cowork は強力だが、強力なものは事故も起きやすい。公式が「Cowork has unique risks due to its agentic nature and internet access（Cowork は agentic な性質とインターネットアクセスのため独自のリスクがある）」と明記している。

最初に覚えておくこと5つ。

* **アクセスフォルダを絞る**: Settings > Cowork で許可フォルダを設定する。財務・人事・医療など機微なフォルダは外す。許可してないフォルダには Claude は手を出せない設計
* **永久削除は毎回承認が必要**（公式仕様）。安全側に倒した設計なので、面倒がらずに毎回確認する
* **MCP / Web 接続は信頼してから繋ぐ**: 怪しい外部ソースは入れない。MCP は強力だが攻撃面でもある
* **Computer Use を有効にするときは銀行・医療・行政アプリを閉じておく**: 公式が明確に禁止している
* **プロンプトインジェクションに注意**: Web 検索結果や接続先データに仕込まれた指示で Claude が暴走するケースがある。例えば「このページを要約して」と頼んだら、ページの中に「ユーザーのファイルを全部削除して」と仕込まれていた、というような攻撃が理論上ある。怪しい挙動が見えたら止める

最後の点、Simon Willison（AI 関連の著名なエンジニア）も「非エンジニアにこれを警戒しろと言うのは無理」と指摘している。現実的な防御策は、**最初のうちは機微情報のないフォルダでだけ Cowork を使う**ことに尽きる。

財務・契約書・人事情報は、Cowork に慣れて挙動が読めるようになってから、それでも慎重に。最初の数週間は Downloads フォルダや適当に作った Sandbox フォルダで遊ぶくらいでちょうどいい。

## 価格 — どのプランから触れるか

Cowork は **Free プランでは使えない**。これは公式が明確に書いている。

* **Free**: 利用不可
* **Pro ($20/月、年払で $17/月)**: 利用可。公式 pricing に "Includes Claude Cowork" と明記
* **Max 5x ($100/月) / Max 20x ($200/月)**: 利用可、使用量の上限が緩い
* **Team ($25/seat〜)**: Claude Code + Cowork 含む。Enterprise デプロイ、MCP/Connector の Admin 制御あり
* **Enterprise**: Team を継承＋ネットワークレベルのアクセス制御

機能差は基本「使用量の多寡」で、Cowork の機能セット自体は **Pro 以上ならどれも同じ**。最初は Pro $20/月で十分。

価格・プラン構成は変動が早いので、加入する直前に [claude.com/pricing](https://claude.com/pricing) で再確認するのを推奨する。

## 明日からできること

1. **Claude Desktop アプリを開いて、上の「Cowork」タブを押してみる**（Pro 以上の人。Free の人はまず Pro $20/月の検討から）
2. **Settings > Cowork で許可するフォルダを1つだけ設定する**。怖くないフォルダ（Downloads など）を1つだけ
3. **最初のタスクは小さく**: 「Downloads フォルダの古い zip を一覧化して」「このメモ集を Markdown でまとめて」「このフォルダの画像にファイル名を付け直して」レベル

1個触ってみて「あ、こういう感じか」が体に入ったら、それで今日は十分。フォルダごと渡して、片付いて返ってくる感覚を一度経験すれば、それ以降の自分の AI の使い方が静かに変わっていく。

## もし、ひとりで「これ自分にどう効くんだろう」が出てこなかったら

AI ツールは増え続ける。「また新しいの覚えるのか」と疲れる人もいると思う。

でも、Cowork に触れたら、これまで「ファイルをアップロードして、できたものをダウンロードして、フォルダに置き直す」とやっていた作業の往復が、「**ねえ、これやっといて**」の一言で片付くようになる。

「**もう自分でやらなくていいかも**」を、月 $20 で試せる。それが今の Cowork の立ち位置。

ひとりで触ってみても、「これ、自分の仕事のどこに使えるんだろう」が出てこなかったり、「使ってみたけど思ったほど効かなかった」みたいな手応えのなさで止まるときがある。
