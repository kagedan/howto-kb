## はじめに

先日Claudeから登場したデザインプラットフォーム **Claude Design** を、ここ数週間にわたって一通り使ってみました。本記事では、その中で見つけた**実践的な使い方**を紹介します。

これから Claude Designを使ってみたい方、あるいは Figma Makeなどのツールから乗り換えを検討している方の参考になれば幸いです。

## Claude Designの基本

詳しい活用方法に入る前に、まずは基本的な始め方を簡単に押さえておきます。

### アクセス方法と対応プラン

Claude Designには **`claude.ai/design`** または **Claude Desktop のサイドバー**からアクセスできます。

対応プランは **Pro / Max / Team / Enterprise** です。

![Claude Design のトップ画面](https://static.zenn.studio/user-upload/eba90794705e-20260628.png)

### 画面構成

画面は**左がチャット、右がキャンバス**というシンプルな構成です。「作りたいもの」をチャットで指示すると、Claudeが右側のキャンバス上に**実際に動作するデザイン**を生成してくれます。

![Claude Design の基本画面（左：チャット / 右：キャンバス）](https://static.zenn.studio/user-upload/e8c671ef1f7c-20260628.png)

### デザインを調整する3つの方法

生成後の調整は、目的に応じて次の3つを使い分けます。

| 方法 | 用途 |
| --- | --- |
| **チャット** | 全体的な変更・大きな方針転換 |
| **インラインコメント** | 対象部分を指定したピンポイント修正 |
| **直接編集** | 色やサイズなど、素早いビジュアル変更 |

### エクスポート・共有

完成したデザインは **.zip / PDF / PPTX / HTML** 形式でダウンロードできるほか、**Canva・Adobe・Vercel** などへの連携にも対応しています。共有リンクは「表示のみ / コメント / 編集」のアクセス権を選んで発行できます。

## 事前準備：既存プロジェクトからデザインシステムを作る

プロトタイプを作る前に、私がまず行ったのが**既存プロジェクトからデザインシステムを作成する**ことです。これは Claude Designを実務で使ううえで、個人的に最も便利だと感じた機能でした。

Claude Designは、スクリーンショットや画像、既存のアセット、さらには**コードベースや既存のデザインファイル**をアタッチして、組織のデザインシステムを取り込めます。Claude Codeを使っている場合は、**`/design-sync` コマンド**で GitHub リポジトリやローカルのコードベースから直接デザインシステムをプルできます。

![既存プロジェクトからデザインシステムを作成](https://static.zenn.studio/user-upload/5142ee35a45a-20260628.png)

### 既存デザインシステムを使うメリット

既存のデザインシステムを取り込んだうえでプロトタイプを作ると、**ゼロから真新しいデザインが出てくるのではなく、すでに出来上がっているアプリを前提にUIを生成できる**のが大きなメリットです。

* 既存アプリのトンマナ・コンポーネント・配色を踏襲できる
* 「それっぽいけれど自社プロダクトとは別物」というデザインにならない
* 生成されたデザインを、そのまま実装に近い形で扱える
* いかにも AIが作ったような“それっぽいデザイン”から脱却できる

## 具体的な活用方法は主に3つ

ここからが本題です。私が実際に使っている活用パターンは、大きく次の3つです。

1. **プロトタイプを作成する**
2. **デザインを元に仕様書を作成する**
3. **デモ動画を作成する**

それぞれ詳しく見ていきます。

## 活用方法1：プロトタイプを作成する

新しいアプリや新機能を作りたいと考えたとき、私はまず**作りたいものの概要やプロトタイプの元になる仕様書**を用意します。

その仕様書を Claude Designにそのまま貼り付けると、Claude側から「作る画面の範囲は？」「作り込みの度合いは？」といった**確認の質問**が返ってきます。プロトタイプを作るうえで必要な情報を先に擦り合わせてくれる形です。

![仕様書を貼り付けると確認の質問が返ってくる](https://static.zenn.studio/user-upload/0cfb4ee77842-20260628.png)

質問に答えて作成ボタンを押し、数分待つとデザインが出来上がります。あらかじめデザインシステムを取り込んでおけば、生成されるデザインも既存プロダクトのトンマナに沿ったものになります。

### 部分的に指定してブラッシュアップできる

生成されたデザインは、要素を部分的に指定して変更したり、指示を出しながら手修正したりできます。

「このボタンの色を変えたい」「このレイアウトをもう少し詰めたい」といった調整も、自然言語で指示するだけで反映されます。一度生成して終わりではなく、**対話しながら仕上げていける**のがポイントです。

![部分的に指定してブラッシュアップ](https://static.zenn.studio/user-upload/6cbd84755719-20260628.png)

## 活用方法2：デザインを元に仕様書を作成する

作成したデザインを実際のコードに落とし込む際、私は**デザインから仕様書を逆生成**させています。

具体的には、出来上がったデザインに対して次のような内容を、**画像と文章のセット（HTML形式など）で出力**させます。

* どのような操作があるか（ユーザーインタラクション）
* どのような表示項目があるか（表示要素・データ）
* どのような画面遷移があるか

こうして「デザイン」と「仕様」がセットになった状態を作り、最終的に Claude Codeへ渡すことで、**より精度の高い実装**が可能になります。

```
[デザイン] ──────> [仕様書（操作・表示項目・画面遷移）] ─────────┐
                                                          ├─> Claude Code に渡す ─> 高精度な実装
[デザイン画像] ─────────────────────────────────────────────┘
```

デザインだけ、あるいは仕様だけを渡すよりも、両者がセットになっていることで実装時の解釈のブレが減り、意図に沿ったコードが出てきやすくなります。

実際に試したところ、下記のような**詳細な仕様書**（20ページ程）を出力してくれました（画像は2ページ分を抜粋）。

## 活用方法3：デモ動画を作成する

社内で「どのようなものを作りたいか」を擦り合わせる際に役立つのが、**デモ動画の作成**です。

Claude Designで作ったデザインを元に、**15秒〜120秒程度のデモ動画**を作成できます。プロトタイプを参照させて動きのある動画にすると、静止画だけでは伝わらない操作感やフローを共有できます。

「言葉で説明するより動画を見せたほうが早い」という場面は多く、社内共有や認識合わせのコストを下げられます。

![デモ動画から切り出した GIF](https://static.zenn.studio/user-upload/66fedf5f04e2-20260628.gif)

## その他の活用シーン

上記3つ以外にも、Claude Designは次の用途で活用できます。

* **ワイヤーフレーム**の作成
* **スライド**の作成
* **ドキュメント**の作成

## 使ってみて感じた課題

便利な一方で、実際に使う中で「ここはまだ惜しいな」と感じた点もありました。

### デザインの分岐・方向転換がしにくい

デザイン作成時には「Tweaks」という機能があり、複数案を並べて比較すること自体は可能です。ただし Tweaks はあくまで、色味やスペースの幅を変えるといった**数パターンの微調整**を行う機能です。

難しいのは、途中でデザインの**方向性そのものを変えたい**ときです。フォークして別の方向で作り直す、といった作業がしにくいと感じました。

一度加えた指示を元に戻すことはできますが、**Git のブランチのように複数の案を分けて残せる**と嬉しいところです。

### コードベースとのシームレスな連携

共通のコンポーネントやデザインシステムでつなぐことはできますが、Claude Designで作ったものを実際のプロダクトに反映する流れは、まだスムーズとは言い難い状況です。

Claude Designからの「ハンドオフ」は、`claude_design` という MCPを介して実装に渡す形になっています。具体的には、ハンドオフ時に次のようなプロンプトが生成され、これを Claude Code側で実行することでデザインを取り込んで実装します。

```
Use the claude_design MCP (https://api.anthropic.com/v1/design/mcp, auth via /design-login) to import this project:
https://claude.ai/design/p/xxxxxxxx?file=SpeakBiz+仕様書.dc.html
Implement: SpeakBiz 仕様書.dc.html
```

単純に HTMLをそのままコードベースへ持ってくるだけならほぼ完璧に実装できますが、結局はこのMCPを通じた実装になるため、**複雑なコードベースに合わせて実装させようとすると、精度が落ちる印象**です。

また、コードベース側の差分をデザインに反映したり、デザインと実装を相互に同期した状態に保ったりするのは、現状では難しいところです。

このあたりの往復がスムーズに行き来できるようになると、デザインと実装の距離がさらに縮まりそうです。

## まとめ

本記事では、Claude Designの実践的な使い方を紹介しました。

| 活用方法 | 概要 | 効果 |
| --- | --- | --- |
| デザインシステム取り込み | `/design-sync` 等で既存プロジェクトから取り込む | 既存アプリ前提の UIを生成できる |
| プロトタイプ作成 | 仕様書を貼り付けて対話的にデザイン生成 | シンプルで綺麗な UIを素早く形にできる |
| 仕様書の逆生成 | デザインから操作・表示項目を抽出 | デザイン × 仕様セットで実装精度が上がる |
| デモ動画作成 | プロトタイプから1分程度の動画化 | 社内共有・認識合わせがスムーズになる |

仕様設計から実装までを一連の流れで扱えるのが Claude Designの魅力だと感じています。気になった方はぜひ試してみてください。
