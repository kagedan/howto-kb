## はじめに
本日Anthropic一般公開したモデル「Fable 5」を、リリース翌日から丸一日、Claude Code でのコーディング・開発作業を中心に触ってみました。本記事はその所感を述べたものです。

## モデル概要
まあ読んでいる時点で知らないことはないと思いますが前提のおさらいとして軽く

Anthropic が 2026/6/9（現地時間）に一般公開した新モデルです。

これまで一部のパートナーにしか提供されていなかったMythos級の能力を、一般ユーザー向けにセーフガードを施して開放したものとされています。

API 料金は入力 100 万トークンあたり 10ドル/ 出力 100 万トークンあたり 50ドル。
Pro / Max / Team / Enterpriseにて6/9〜6/22の間プラン内で追加料金なしで利用可能。

## 利用してみて
ここからが本題の利用してみた所感です。
使用環境はClaudeCode v2.1.170でOpenSpecを入れています。
OpenSpecのエージェントは問題なく動作しました。

### 良かった点
- **UI を仕様書に合わせ込む作業が明らかに上手くなった印象**
これまでの Claudeは、仕様書,PoC等に書いたUIを正確に落とし込むのが苦手であったがそこがうまくなった気がします。
- **バグ調査での自律性が想像を超えていた**
バグの可能性がある事象が発生した際自分でもどこが原因か分かっていなかったのだが、Fable 5 はアクティブな他人のブランチまで pull して調べ、原因を突き止めてきました。指示していない範囲まで自律的に踏み込んで調査・解決する能力は素直にすごいとおもいましたが同時に怖いです。

### 気になった点・注意点
- **とにかくトークン消費が激しい**
体感として4.8と比較してかなり消費が激しいです。公式はトークンが以前より効率的と主張している（いつもそう）が、少なくとも自分の一日の使い方では「効率的」というより「よく考え・よく動く分よく消費する」という印象でした（これもいつもそう）。
＊もともと後日上げる予定でしたが消費が激しいためMax(20x)プランに今日上げました。
- **速度は速くない**
effortがmediumでも体感では Opus 4.8 の xhigh より時間が倍以上かかる気がします。 賢くなった分、待ち時間は増えた印象です。
