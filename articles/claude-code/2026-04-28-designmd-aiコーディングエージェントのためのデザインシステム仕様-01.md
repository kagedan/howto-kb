## DESIGN.md とは

[DESIGN.md](https://github.com/google-labs-code/design.md) は、Google Labs が開発中の「AIコーディングエージェント向けの視覚的アイデンティティ記述フォーマット仕様」です。

デザインシステムを**機械可読形式**で定義することで、Claude Code や Cursor などのAIエージェントが「設計の意図を理解した上で」実装できるようにします。

## なぜ DESIGN.md が必要なのか

従来のデザインシステムには以下の課題がありました：

1. **Figma や Storybook** - 人間向けに最適化されており、AIが直接参照しにくい
2. **デザイントークン（JSON）** - 値は定義できるが「なぜこの値か」が欠落
3. **ドキュメント** - 散在しがちで、AIが全体像を把握しにくい

DESIGN.md は、**YAMLフロントマター**（トークン定義）と **Markdown本文**（設計根拠）を組み合わせることで、AIが「何を」「なぜ」使うべきかを理解できるようにします。

## 基本構造

DESIGN.md は二層構成です：

```
---
# トークン定義（YAML フロントマター）
colors:
  primary: "#B8422E"  # Boston Clay
  secondary: "#2E5CB8"
  
typography:
  heading:
    family: "Inter"
    weight: 700
  body:
    family: "Inter"
    weight: 400
    
spacing:
  unit: "4px"
  scale: [4, 8, 12, 16, 24, 32, 48, 64]

rounded:
  sm: "4px"
  md: "8px"
  lg: "16px"
---

# デザインシステム

## カラーパレット

Boston Clay (#B8422E) は相互作用の単一の駆動力として使用します。
CTAボタン、リンク、フォーカス状態など、ユーザーのアクションを促す要素に適用してください。

## タイポグラフィ

見出しは Inter Bold を使用し、視覚的な階層を明確にします...
```

この形式により、AIは単に「primary は #B8422E」という事実だけでなく、「CTAボタンやリンクに使うべき」という意図も理解できます。

## CLI ツール

DESIGN.md には便利な CLI ツールが用意されています：

### lint - 構造チェック

```
npx @google/design.md lint DESIGN.md
```

構造の正確性、壊れた参照、WCAG コントラスト比をチェックし、JSON で結果を返します。

### diff - 変更検出

2つのバージョン間でトークンの変更を検出します。デザインシステムの変更管理に便利です。

### export - フォーマット変換

```
npx @google/design.md export DESIGN.md --format tailwind
```

Tailwind CSS や W3C DTCG 形式など、既存のツールチェーンに統合できる形式でエクスポートできます。

## 活用方法

### 1. プロジェクトルートに配置

```
your-project/
├── DESIGN.md          # AIが参照するデザインシステム
├── CLAUDE.md          # Claude Code 向け指示
├── src/
│   └── ...
```

DESIGN.md をプロジェクトルートに配置することで、AIエージェントが自動的に参照します。

### 2. CLAUDE.md から参照

```
# CLAUDE.md

## デザインガイドライン

UIコンポーネントを実装する際は、必ず DESIGN.md を参照してください。
特に以下の点に注意：
- カラーはトークンを使用（ハードコード禁止）
- スペーシングは定義されたスケールに従う
```

### 3. コンポーネント実装時の活用

AIに UI 実装を依頼する際、DESIGN.md が存在すれば：

```
ユーザー: ログインフォームを作って

AI: DESIGN.md を参照します。
- primary カラー (#B8422E) をCTAボタンに使用
- spacing.scale の 16px と 24px で余白を設定
- rounded.md (8px) でボタンの角丸を設定
...
```

デザインの一貫性を保ちながら、意図に沿った実装が可能になります。

### 4. Tailwind CSS との連携

```
npx @google/design.md export DESIGN.md --format tailwind > tailwind.tokens.js
```

エクスポートした設定を `tailwind.config.js` に統合：

```
// tailwind.config.js
const tokens = require('./tailwind.tokens.js');

module.exports = {
  theme: {
    extend: {
      colors: tokens.colors,
      spacing: tokens.spacing,
      borderRadius: tokens.rounded,
    },
  },
};
```

### 5. デザインレビューの自動化

CI/CD パイプラインに lint を組み込み：

```
# .github/workflows/design-lint.yml
- name: Lint DESIGN.md
  run: npx @google/design.md lint DESIGN.md
```

コントラスト比の問題やトークンの不整合を自動検出できます。

## CLAUDE.md との使い分け

| ファイル | 目的 | 内容 |
| --- | --- | --- |
| CLAUDE.md | 開発プロセス全般 | コーディング規約、コマンド、アーキテクチャ |
| DESIGN.md | ビジュアルデザイン | カラー、タイポグラフィ、スペーシング |

両者を組み合わせることで、AIエージェントは「どう作るか」と「どう見せるか」の両方を理解した上で実装できます。

## 現在のステータス

DESIGN.md は現在 **alpha バージョン** です。仕様と CLI は開発中のため、プロダクション利用には注意が必要です。

ただし、コンセプト自体は非常に有用です。正式リリースを待たずとも、同様の構造で独自の DESIGN.md を作成し、AIエージェントに参照させることは十分可能です。

## まとめ

* **DESIGN.md** は AI コーディングエージェント向けのデザインシステム仕様
* **YAML + Markdown** の組み合わせで「何を」「なぜ」を伝える
* **CLI ツール** で lint、diff、export が可能
* **CLAUDE.md と併用** することで、開発プロセスとビジュアルデザインの両方をAIに伝達

AIと協働する開発において、デザインの意図を正確に伝えることはますます重要になっています。DESIGN.md は、その課題を解決する有力なアプローチです。
