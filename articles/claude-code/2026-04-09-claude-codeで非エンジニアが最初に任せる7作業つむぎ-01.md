Claude Codeを入れたあと、最初に困ったのは「できること」より「今の自分の仕事のどこに置けばいいか」だった。

アプリを作れる 自動化できる という説明はたくさんある。

でも、最初の1本として何を頼むと失敗しにくいのかは、意外と見えにくい。

ここが曖昧だと、触ってみても「すごそう」で終わる。

使い続けてわかったのは、最初に向いているのは次の条件がそろう仕事だった。

* すでに手元にファイルや材料がある
* ゴールがはっきりしている
* 結果を自分で確認できる

その前提で、非エンジニアでも最初から任せやすかった7作業をまとめる。

この3条件がそろうものから始めると、「すごそう」で終わりにくい。

## 1. 既存ファイルを読んで、要点を整理する

議事録、仕様書、ログ、CSV、過去の記事。こういう「もう手元にあるもの」をまとめるのはかなり向いている。

ただ要約するだけではなく、

まで整理しやすい。

自分の運用でも、ファイルが増えてくるほどここは助かった。

## 2. 定型作業をまとめて実行する

毎回同じ順番でやる作業は、Claude Codeと相性がいい。

たとえば、

* リサーチ結果を集めて保存する
* フォルダ構成を整える
* レポートを決まった形式で出す
* 古い資料を更新版に直す

みたいなものだ。

「何をやるか」が決まっている仕事ほど、会話より手順化の効果が大きい。

## 3. バグやエラーの切り分けをする

Claude Codeは、エラー文を読んで終わりではなく、関連ファイルを見に行って、原因候補を並べて、修正案まで出せる。ここが強い。

特に向いているのは、こういう場面だ。

逆に、「なんか変」で止めると弱い。ログ、再現手順、期待結果を渡せるほど精度は上がる。

## 4. 小さな修正をまとめて入れる

1ファイルだけ直すより、関連する複数ファイルをまたいで整える作業が得意だ。

たとえば、

* 文言の統一
* ボタン名や見出しの修正
* 軽いレイアウト調整
* 同じルールでのリネーム

みたいな「人がやると細かくて面倒」なものは、かなり進めやすい。

## 5. 手順書やマニュアルを更新する

これは非エンジニアにとって、かなり実用的だと思う。

古い手順書、新しい画面のスクリーンショット、変更後の仕様。この3つがあれば、更新版のたたき台を作りやすい。

チャットAIだと「どう直すか」を説明して終わることが多いけれど、Claude Codeは実ファイルを読みながら更新できる。

「マニュアルを直したいけど、どこが古いか探すのが面倒」という作業ほど向いている。

## 6. リサーチ結果から「今日の判断材料」を作る

ここは、今いちばん実感がある。

生データを集めて終わりではなく、

まで1枚に整理する作業はかなり相性がいい。

「何が集まったか」ではなく、「今日は何を書くか」が見える形に変える。ここは、Claude Codeに最初に任せる価値が大きい。

## 7. 繰り返す依頼を「型」にして残す

毎回同じ説明をしているなら、そこはもう型にした方がいい。

たとえば、

* 毎朝どの順番で見るか
* 何をログに残すか
* どんな条件なら次に進むか

を CLAUDE.md やチェックリストとして残しておく。

必要なら後で Skills にしてもいいけれど、最初から難しくしなくていい。

まずは「この順番で見て、この条件なら次に進む」を固定するだけでも、かなり使いやすくなる。

## できないこともある

ここを先に知っておくと、期待外れが減る。

Claude Codeが苦手なのは、

* ゴールが曖昧な依頼
* 確認なしで一気に全部任せる運用
* 元データが足りない状態での判断
* 人間の承認が必要な最終決定

つまり、「何でもできる」より「条件がそろうと強い」が近い。

## 最初にやるなら、この順番がわかりやすい

いきなり大きな自動化に行くより、次の順番の方が失敗しにくい。

1. 既存ファイルを1つ読ませて整理する
2. 毎週やる定型作業を1つだけ任せる
3. その結果を brief 1枚にまとめる
4. 繰り返す指示を CLAUDE.md やチェックリストに残す

Claude Codeでできること一覧を見ていると、つい大きな夢の方に目が行く。

でも実際に効いたのは、「今やっている面倒な作業を1つだけ置き換える」だった。

自分にとっては、それが朝のリサーチだった。全部を変える前に、1つだけ置き換える。そこからの方が、Claude Codeはずっと使い続けやすい。

もし「じゃあ、その1本目は何にするのがいいのか」で止まっているなら、自分は朝リサーチを先に整えるのがいちばん失敗しにくかった。

翌朝から使えるセットを出しています。付録フォルダを開くだけで動きます。
