# Claude Opus 4.8 と自律エージェントを自社で 8時間 走らせて分かったこと

黒いターミナル画面上で、Claude Code が自律的に数十のサブエージェントを立ち上げ、並列でテストコードを書き直していく。

米Anthropicが発表した新機能「Opus 4.8」の自律制御は、ただのコード生成ではない。

僕が実際に自社プロダクトのリポジトリを食わせ、エラー自己修復を監視し続けたリアルな記録を共有する。

# Claude Code のターミナルで Opus 4.8 が無数のサブエージェントを起動した瞬間

## 黒い画面の中で自律的に自問自答を繰り返すエージェントたち

深夜の静まり返ったオフィスで、MacBook Proの黒いターミナル画面を見つめていた。

Anthropic公式の Claude Opus 4.8 と、新たに提供されたコマンドラインインターフェース（CLI）ツールである Claude Code を連携させ、開発リポジトリに対する自律修正を指示した時のことだ。

僕がキーボードを叩き、一つのコマンドを実行した瞬間、画面のスクロール速度が急激に加速した。

これまで体験してきたどのAI開発ツールとも異なる、異様な挙動が始まった。

ターミナルの中では、親となるエージェントがタスクを細分化し、無数のサブエージェントを裏で自律的に起動していた。

画面上に「Analyzing repository...」「Starting sub-agent for dependency check...」といったログが、またたく間に流れていく。

それぞれのサブエージェントは独立した役割を与えられ、あるエージェントはデータベースの定義ファイルを読み込み、別のエージェントは過去のデプロイログを解析し、さらに別のエージェントは問題の発生しているAPIエンドポイントのテストコードを自律的に実行していた。

驚くべきは、これらのサブエージェント群が、僕からの追加指示を一切仰ぐことなく、互いにデータをやり取りしながら「自問自答」を繰り返していた点だ。

「サブエージェントAが発見した依存関係のエラーを、サブエージェントBが自動的に検知し、適切なバージョンへの書き換え提案を親エージェントにフィードバックする」という自律制御ループが、ターミナルの奥深くで完全に自動で回っていた。

この様子を画面越しに眺めながら、ソフトウェア開発の概念が根本から覆される予感がした。

これまでのAIチャットやコード生成ツールは、人間がプロンプトを入力し、それに対する回答を1対1で受け取るプロセスが基本だった。

しかし、今回の Claude Code と Opus 4.8 の組み合わせは違う。

人間が指示するのは「このバグを修正し、テストが通る状態にしろ」というゴールだけだ。

そのゴールに向けて、AI自らが組織（エージェントネットワーク）を形成し、役割を分担し、エラーが発生すれば自律的に原因を突き止め、解決策を試行錯誤する。

そのプロセスは、優秀なエンジニアチームがホワイトボードを前に議論を重ね、それぞれが端末に向かって黙々と作業を進める姿そのものだった。

## 実プロダクトのソースコードを読み込ませて 8時間 走らせた検証結果

僕が実際に運用している『sline-sales-system』という実プロダクトの開発リポジトリを対象に、この新しい仕組みを本格的に検証することにした。

検証の目的は、蓄積していた技術負債の解消と、複数のAPI連携部分で発生していた複雑な非同期処理のバグを完全に修正することだ。

僕が Claude Code に対して「バックエンドの非同期処理におけるメモリリークの原因を特定し、修正パッチを当てた上でテストコードを実行せよ」という大まかな指示を出し、そこから約8時間にわたりシステムを監視し続けた。

最初の1時間は、コードベース全体の静的解析に費やされた。

Opus 4.8 の圧倒的なコンテキストウィンドウと処理能力を背景に、システム内の全ファイルをスキャンし、依存関係のマップを自律的に構築していく。

2時間を経過した頃、AIは特定の非同期キュー処理にボトルネックがあることを突き止めた。

ここからが自律制御の真骨頂だった。

AIは検証用のサブエージェントを立ち上げ、ローカル環境でダミーデータを流すスクリプトを自動生成し、実際にメモリの消費量が推移する様子を計測し始めた。

4時間が経過した時点で、問題の箇所を特定したエージェントは、コードの書き換えに着手した。
