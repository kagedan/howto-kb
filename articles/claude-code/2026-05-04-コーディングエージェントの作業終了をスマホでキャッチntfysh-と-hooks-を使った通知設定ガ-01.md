こんにちは！日々の開発お疲れ様です。  
最近はコーディングエージェントに重たいリファクタリングやテストコードの生成を任せる機会も増えてきましたよね。でも、処理が終わるまでターミナルの進捗をじっと眺めているのは、ちょっと時間がもったいない気がしませんか？

「終わったらスマホにプッシュ通知が来ればいいのに」

そんな願いを叶えてくれるのが、軽量な通知サービス ntfy.sh と、エージェントの Hooks（フック） 機能の組み合わせです。今回は、エージェントの作業が終わった瞬間にスマホへ通知を飛ばす仕組みを、2000文字程度のボリュームで詳しく解説します。

今回はGitHub Copilot 設定を想定していますが、Hooks機能があるツールなら何でも応用可能です。それでは、さっそく設定していきましょう！

---

## 1. 通知の要：ntfy.sh の準備

まず、今回通知のハブとして使う **ntfy.sh** についてご紹介します。  
これ、実は「ユーザー登録不要」「ログイン不要」で、特定のURLに `curl` を投げるだけでスマホにプッシュ通知が飛ばせるというとてもお手軽なツールです。

### ステップ 1：スマホアプリをいれる

まずは通知を受け取りたいスマホにアプリをインストールしましょう。

* Android: Google Play ストアで「[ntfy](https://play.google.com/store/apps/details?id=io.heckel.ntfy&hl=ja)」を検索
* iPhone: App Store で「[ntfy](https://apps.apple.com/jp/app/ntfy/id1625396347)」を検索

### ステップ 2：秘密のトピック名（ID）を生成する

ntfy.sh は、URL（トピック名）さえ知っていれば誰でも通知を送れてしまいます。そのため、推測されにくい自分専用のIDを作ることが重要です。  
ここでは`uuidgen` と `md5sum` を組み合わせて、世界に一つだけの強力なトピック名を発行しましょう。もちろん、みんなが使いそうなピックを作成して見知らぬ人の通知を受け取るのも自由ですが、おすすめはしません。

ターミナルで以下のコマンドを実行してください。

```
# ユニークなトピック名を生成して表示
uuidgen | md5sum | cut -d' ' -f1
```

ここで出力された `a1b2c3d4...` のような長いハッシュ値を、これからの「通知先ID」として使います。

### ステップ 3：アプリで「購読」を開始

スマホの ntfy アプリを開き、「+」ボタン（Add subscription）をタップします。  
先ほど生成したIDをトピック名としてsubscriptionをしてください。

これで、スマホ側の準備は万端です！

## 2. エージェントの設定：Hooks を仕込む

いよいよ本題です。コーディングエージェントが仕事を終えた瞬間に、自動で通知コマンドを叩くように設定します。

多くのエージェントツールには、特定のイベント（開始、終了、エラーなど）に合わせてスクリプトを実行できる Hooks 機能があります。今回は GitHub Copilot を例に、エージェントが停止したときに呼ばれる `Stop` という Hook を使ってみましょう。

### Hooks設定ファイルの記述例

GitHub CopilotのHooksはいろいろな設定方法がありますが、今回はホームディレクトリの ~/.copilot/hooks/hooks.json に記載します。  
${通知先ID}は先程生成した通知先IDに置換してください。

#### ~/.copilot/hooks/hooks.json

```
{
  "version": 1,
  "hooks": {
    "Stop": [
      {
        "name": "notify-ntfy-onstop",
        "description": "Send ntfy notification when agent stops",
        "type": "command",
        "bash": "curl -H \"Title: エージェント作業完了\" -H \"Priority: urgent\" -H \"Tags: tada,robot\" -d \"GitHub Copilot エージェントが処理を終了しました。\" https://ntfy.sh/${通知先ID}",
        "windows": "powershell.exe -Command \"Invoke-RestMethod -Method Post -Uri 'https://ntfy.sh/${通知先ID}' -Body 'エージェントの作業が終了しました！' -Header @{ 'Title' = '完了通知'; 'Priority' = 'high' }\"",
        "timeoutSec": 10
      }
    ]
  }
}
```

## 3. この仕組みがもたらす「快適な開発体験」

「たかが通知でしょ？」と思うかもしれませんが、実際に導入してみると開発のリズムが「人がポーリングして確認する」状態から、「AIからのPush通知」へと劇的に変わります。  
最近エージェントは自走力が上がって、数時間の作業もこなすことができるようになりました。その間、ずっと画面を見ていることは現実的ではありませんし、人間のやるべきことではありません。定期的に確認するにしても、その間の人間のコンテキストスイッチコストは無視できるものではありません。  
通知を設定することで安心して放置することができ。通知があったらAIの作業完了時にすぐに次の作業に取り掛かることができます。

---

## 4. さらに使いこなすためのヒント

せっかくなので、通知をもっとリッチにしてみましょう。ntfy.sh は HTTP ヘッダーを追加するだけで、いろいろなカスタマイズが可能です。

* **緊急度を変える**: `-H "Priority: 5"` (または `max`) とすると、スマホによってはマナーモードを貫通して音を鳴らすこともできます（使い所には注意！）。
* **アイコンやタグ**: `-H "Tags: robot,tada"` のように指定すると、通知画面に可愛い絵文字が表示されます。
* **アクションボタン**: `-H "Click: https://github.com/..."` とすれば、通知をタップした瞬間にリポジトリのPRページを直接開くといった設定も可能です。

## まとめ

今回の手順をまとめると、以下の3ステップです。

1. 自分だけの秘密のトピックIDを作る。
2. スマホの ntfyアプリ でURLとIDを登録する。
3. エージェントの Hooks (Stop) で `curl` か `Invoke-RestMethod` を呼び出してntfyに通知する。

たったこれだけで、あなたの開発環境に「AIエージェントからの完了報告」という未来的な機能が加わります。そして、完了報告が来ることであなたとAIの共同作業がより緊密なものになるでしょう。
