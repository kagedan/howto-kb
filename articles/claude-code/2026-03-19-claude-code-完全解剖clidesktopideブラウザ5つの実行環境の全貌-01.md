## Claude Code 完全解剖——CLI・Desktop・IDE・ブラウザ、5つの実行環境の全貌

**バージョン情報：Claude Code v2.1.76、2026年3月時点の公式ドキュメントに基づく**

---

Claude Codeは単一の「ターミナルツール」ではない。ターミナルCLI、デスクトップアプリ、VS Code拡張、JetBrainsプラグイン、ブラウザ（Web）という5つの実行環境を持つ、マルチサーフェス型のコーディングエージェントだ。すべての環境が同一のClaudeエンジンを共有し、CLAUDE.mdファイル、設定（~/.claude/settings.json）、MCPサーバーの構成はどの環境でも共通して機能する。

環境間の移動も設計の中核だ。「ウェブで長時間タスクを開始して /teleport でターミナルに引き継ぐ」「ターミナルのセッションを /desktop でDesktopアプリに渡してビジュアルdiffを確認する」「Remote Controlでスマートフォンからローカルセッションを操作する」——各サーフェスは独立したツールではなく、同一ワークフローの異なる窓口として機能する。

ただし環境ごとに哲学が異なる。何を選ぶかはユースケース次第だ。

---

## 0. 横断する重要アップデート（2026年3月）

個別の環境の説明に入る前に、すべての環境に影響する最新の重要変更を押さえておきたい。

**デフォルトモデルがOpus 4.6に**（v2.1.75以降）。APIのOpus 4.1は廃止され、Claude CodeのデフォルトはOpus 4.6に完全移行した。これはBedrock、Vertex、Microsoft Foundry経由でも同様だ。

**1Mトークンコンテキストウィンドウ**（3月13日GA）。Opus 4.6とSonnet 4.6が100万トークンのコンテキストウィンドウを標準サポート。Max・Team・Enterpriseプランでは追加設定なしで自動的に1Mコンテキストが有効になる。Proプランは /extra-usage でオプトイン可能。以前は20万トークンを超えると追加課金が発生していたが、1Mトークン全域が標準料金になった。コンテキスト管理の認知コストが大幅に下がり、大規模なコードベースやエージェントの長時間ループが実用的になった。

**ボイスモード**（/voice、v2.1.76、3月時点でロールアウト中）。スペースバーを押しながら話し、離すと送信するプッシュトゥトーク方式。keybindings.json でキーをカスタマイズでき、meta+k などの組み合わせも使える。常時リスニングではなく、完全にユーザーが制御するモデルだ。

**/loop コマンド**（v2.1.63以降）。繰り返し実行のスケジューリング機能。Desktopアプリの「スケジュールタスク」に近い機能をCLIから使えるようになった。

## 1. ターミナルCLI——最も制御可能な環境

インストールは npm install -g @anthropic-ai/claude-code（あるいはHomebrew、ネイティブインストーラー、WinGet）。Claude Codeの全コマンド・フラグへのフルアクセスを持つオリジナルの形態だ。

### 料金と認証の特権

**ターミナルCLIとVS Code拡張だけが、サードパーティプロバイダーのAPIキー直接課金に対応している。** Bedrock、Vertex、Azure、その他のOpenAI互換エンドポイントを使って従量課金できる。他の環境（Desktop、ブラウザ）はAnthropicサブスクリプションが必要だ。CI/CDや大規模自動化でコストコントロールが必要な場面では、CLIが唯一の現実的な選択肢になる。

注意点として、ANTHROPIC\_API\_KEY 環境変数がセットされているとサブスクリプションではなくAPIキー課金に自動切り替わるため、意図しない課金が発生することがある。

### OSサポート

macOS（Ventura 13.0以上）とLinuxがネイティブ。WindowsはWSLが推奨だが、Git for WindowsをインストールすればPowerShellでのネイティブ動作も可能。ただしWindowsでは一部機能に制限がある（CJK文字や絵文字のクリップボード問題はv2.1.75で修正済み）。Homebrewインストールは自動更新されないため、定期的に brew upgrade claude-code が必要。

### 自動化とCIへの組み込み

シェルスクリプトと自由に組み合わせられる点がCLIの最大の強みだ。--print（非インタラクティブ）モードや --output-format json でCI/CDパイプラインへの組み込みが可能。また Ctrl+B で現在のタスクをバックグラウンドに送り、別の作業を続けることもできる。

ウェブセッションへのタスク送信は & プレフィックスで行える（& APIのリファクタリングを進めて のように）。これによりクラウドで非同期実行し、ターミナルを閉じても処理が継続する。

コンテキスト管理では /compact コマンドでその時点でのコンテキストを圧縮・要約できる。v2.1.76からは CLAUDE\_CODE\_AUTO\_COMPACT\_WINDOW 環境変数で自動圧縮のしきい値を調整可能。

### プライバシーと透明性

ファイルはローカルに留まり、モデル推論のためのAPI呼び出しのみが外部通信の対象。機密コードや規制産業での利用では最も透明性が高い。

## 2. Desktop App——視覚的ワークフローと並列実行の拠点

claude.ai/download からダウンロードする。macOS（Big Sur 11以上）とWindowsをサポート。Windowsバージョンは2026年2月10日リリースで完全なフィーチャーパリティを達成した。**Linuxは2026年3月時点で非対応。**

### Cowork——コーディング以外のナレッジワーク

DesktopアプリにはCodeタブとCoworkタブが共存する。Coworkは独立した軽量Linux VM（Appleの VZVirtualMachine フレームワーク使用）上でClaudeが動作し、コーディング以外の複雑な知識作業を自律的にこなす。具体的には、Wordドキュメント・スライド・スプレッドシート・PDFの作成と編集、ファイル整理・リネーム、GitHubやSlackとの連携などだ。

Coworkでは「Work in a Folder」でアクセスするディレクトリを明示的に指定する必要があり、指定されていないディレクトリには触れられない。ハード分離によってシステムファイルは保護される。

**プラン・ハードウェア要件の正確な内訳：** macOSでは Apple Silicon（M1以降）が必要。Intel MacではCodeタブとChatタブは正常動作するが、Coworkは非対応。Windowsでは対応ハードウェア全体でCoworkが利用可能（ただしArm64版Windowsへの対応は開発中）。プランはPro、Max、Team、Enterpriseすべての有料プランでリサーチプレビュー利用可能。

Coworkタスクはアプリが起動している間しか実行されない。スリープ中やアプリを閉じると処理が中断する。

### Gitワークツリーによる並列セッション

同一プロジェクトの複数ブランチに対して、互いに干渉しない独立したClaudeセッションを同時に実行できる。「バグ修正と機能実装と仕様書更新を並列で」というワークフローが実現する。ワークツリーは .worktreeinclude ファイルで含めるディレクトリをカスタマイズ可能で、大規模モノレポではsparsePaths設定と組み合わせて必要なディレクトリのみをチェックアウトできる（v2.1.76の worktree.sparsePaths 設定）。

セッション分離にはGitが必要。Codeタブを使う場合はGit for Windowsのインストールが前提になる。

### Desktop Extensions（.mcpbファイル）

MCPサーバーをすべての依存関係ごとパッケージングした1クリックインストール形式。仕様上はChromeの .crx やVS Codeの .vsix に相当するアーキテクチャだ。Desktop Appには内蔵Node.jsランタイムが含まれており、ユーザーはNode.jsを別途インストールする必要がない。APIキーなどの設定情報はOSのキーチェーンに安全に保存される。

Anthropicは仕様とツールチェーンをオープンソース化しており（github.com/modelcontextprotocol/mcpb）、他のAIデスクトップアプリケーションでも採用を推奨している。なお .dxt という旧拡張子から .mcpb に名称変更されており、旧ファイルも引き続き動作する。

### スケジューリングと /desktop コマンド

Desktopアプリには定期タスク機能があり、「毎朝9時にコードレビュー実行」のようなスケジュールを設定できる。ただしDesktopアプリが起動していないとスケジュールタスクは実行されない。

ターミナルCLIからは /desktop コマンドでセッションをDesktopアプリに引き渡し、ビジュアルdiffレビューができる。

## 3. VS Code拡張——最も洗練されたIDE統合

VS Codeのマーケットプレイスで「Claude Code by Anthropic」をインストール。Cursor、Windsurfといったフォーク版VS Codeでも動作する。2026年3月時点で29百万日次インストールを記録しており、実質的にはClaude Codeの主要アクセス手段の一つだ。

### IDE固有の強化機能

**チェックポイント機能**：Claudeが意図しない変更をした場合のアンドゥシステム。ファイル変更の取り消しのみ、会話のフォーク（別の方向性を試す）のみ、あるいは両方を選択できる。

**自動コンテキスト共有**：カーソル位置と現在の選択コード、リントエラー・構文エラー・コンパイルエラーといったIDE診断情報が自動的にClaudeと共有される。エラーをコピーペーストする手間がない。

**インラインdiff表示**：ネイティブGUIのサイドバイサイド比較で変更を確認し、承認/却下できる。

**@メンション**でファイルや選択範囲を参照。コマンドパレット（Cmd+Shift+P→「Open in New Tab」）で複数の並列会話が可能。

### 認証と設定

VS Code拡張は設定ファイル（~/.claude/settings.json）をCLIと共有するため、MCPサーバーなどの設定は自動的に引き継がれる。ただし initialPermissionMode などの拡張固有設定はVS Code自身の設定システムに格納される。

CLIとの重要な差異として、/compact コマンドは各環境で独立して動作する。ターミナルで圧縮してもVS Code拡張のコンテキストには影響しない。

外部ターミナルでCLIを使っている場合は /ide コマンドでVS Codeに接続できる。

### サードパーティAPIキー対応

ターミナルCLIと同様、VS Code拡張もサードパーティプロバイダーのAPIキー直接課金に対応した唯一の環境のひとつだ。

## 4. JetBrains プラグイン——ベータ版、しかし本格統合

IntelliJ IDEA、PyCharm、WebStorm、PhpStorm、GoLandなどに対応。JetBrainsマーケットプレイスで「Claude Code [Beta]」を検索してインストールする。執筆時点でベータステータスのため、ユーザーから「コンソール出力が不安定」「IDEがフォーカスを奪う」といった動作が報告されている。

### アーキテクチャの特徴

VS Code拡張とは異なり、JetBrains版は統合ターミナルでCLIを起動し、そのCLIにIDEが接続するという二層構造を取る。統合ターミナルで claude を実行することですべての統合機能が有効になる。外部ターミナルを使う場合は /ide コマンドで接続する。

ショートカットは Cmd+Option+K（Mac）または Alt+Ctrl+K（Linux/Windows）でファイル参照（@File#L1-99 形式）を挿入できる。現在の選択内容、開いているタブ、IDE診断エラーは自動共有される。

JetBrainsのリモート開発（SSH経由）を使う場合は、ローカルマシンではなくリモートホスト側にプラグインをインストールする必要がある。

設定は ~/.claude/settings.json をCLIと共有しており、認証情報は一元管理される。チェックポイント機能など一部の機能はまだ完全な実装に至っていない。

## 5. ブラウザ（Web）——クラウド実行と非同期タスク

claude.ai/code からアクセス。ローカルへのインストールが一切不要で、クラウドのサンドボックス環境でClaudeが動作する。

### クラウド実行の本質

GitHubアカウントにClaude Appをインストールし、アクセス許可したリポジトリに対してClaudeがクラウドから直接アクセスする。ローカルのファイルシステムやMCPサーバーには基本的にアクセスできない点がRemote Controlとの根本的な違いだ。

CLIから claude --remote "タスク内容" でウェブセッションを新規作成できる。また & プレフィックスを付けたプロンプトで現在のリポジトリに対してバックグラウンドのクラウドタスクを起動できる。タスクは /tasks コマンドで進捗確認可能だ。

長時間タスクを非同期に投げて「あとで確認する」用途に最も最適化されている。

### Teleport（/teleport または /tp）

ウェブ（またはiOSアプリ）で開始したセッションをターミナルに引き継ぐ機能。ブランチの確認・切り替えとコンテキストの転送を自動処理する。公式ドキュメントでは「ウェブやiOSアプリで長時間タスクを開始して、/teleport でターミナルに引き込む」という方向で説明されている。

### iOS・Androidアプリとの連携

Claude iOS/Androidアプリからも同じウェブセッションにアクセスでき、外出先での進捗確認や追加指示が可能。モバイルアプリ内で /mobile コマンドを実行するとQRコードが表示され、ダウンロードリンクを素早く共有できる。

## 6. Remote Control——環境を超えるセッション管理

2026年2月25日リリース（v2.1.51、リサーチプレビュー）。ローカルのCLIセッションと claude.ai/code・Claude iOS/Androidアプリを橋渡しする同期レイヤーだ。

### ブラウザ実行との根本的な違い

ブラウザ実行ではコードがクラウドに移動する。Remote Controlではコードは一切クラウドに移動しない。CLIセッションはローカルマシン上で走り続け、リモートデバイスはそのセッションへの窓口として機能するだけだ。ローカルのファイルシステム、環境変数、MCPサーバー、CLAUDE.md設定——すべてが維持される。

セキュリティモデルはシンプルだ。ローカルのClaude CodeセッションはAnthropicのAPIに対してアウトバウンドのHTTPS接続のみを行う。ポート開放やVPN設定は不要。転送されるのはチャットメッセージとツール実行結果のみで、すべてTLS暗号化される。

### 起動方法

```
# 専用サーバーモードで起動（インタラクティブな入力なしで待機）
claude remote-control

# または /rc コマンドを既存セッション内で実行
/rc

# --remote-control フラグ付きで通常セッションを開始
claude --remote-control
```

起動後、セッションURLとQRコードが表示される。スペースバーを押してQRコードを表示させることもできる。

### プランと制限

**プランの正確な状況：** 公式ドキュメント（code.claude.com/docs/en/remote-control）によれば、Remote ControlはPro・Max・Team・Enterpriseすべての有料プランで利用可能。ただしTeam・EnterpriseではデフォルトでOFFであり、管理者が claude.ai/admin-settings/claude-code の「Remote Control」トグルを有効化する必要がある。また「Claude Code on the web」トグルが先に有効になっていることが前提条件だ。

リサーチプレビューとして2月25日にMaxプラン向けにリリースされ、その後Proプランへの展開が進んだという経緯があり、一時的にProユーザーでも「Not yet available on your plan」エラーが出るという報告が相次いだ。現在は公式仕様上は全有料プランが対象だが、環境によっては追加手順が必要になるケースもある。

APIキーのみの認証は非対応。/login でclaude.aiアカウントとサインインする必要がある。

**現時点の制限：** 同時に使えるセッションは1つまで。--dangerously-skip-permissions フラグはRemote Controlと非互換のため、すべてのアクションで承認が必要。ネットワーク断が約10分続くとセッションが終了し、再度 claude remote-control で新しいセッションを開始する必要がある。セッションクラッシュ時に「謎のAPIエラー」が表示されることがある（v2.1.75でポーリングレートが毎1〜2秒→10分に最適化され、サーバー負荷が約300分の1に削減）。

## 環境選択の実践ガイド

どの環境を主軸に置くかは、作業内容によって決まる。

**ターミナルCLIが最適な場面：** CI/CDへの組み込み、APIキー直接課金による大規模自動化、リモートサーバー上での作業、プライバシーが最優先の環境、サードパーティプロバイダーの使用。/loopコマンドやシェルスクリプトとの組み合わせによる夜間自律タスク。

**Desktop Appが最適な場面：** Gitワークツリーを使った並列機能開発、視覚的なdiffレビュー、Coworkを活用したコーディング以外の知識作業（文書作成・ファイル整理・データ分析）、技術的な知識が少ないチームメンバーと共有する場合。

**VS Code拡張が最適な場面：** VS Code（またはCursor）をメインエディタとして使っており、チェックポイントやIDEネイティブなdiff機能をフル活用したい場合。最もユーザー数が多く、エコシステムも成熟している。

**JetBrainsプラグイン：** JetBrains IDEから離れたくないJava/Kotlin/Pythonなどの開発者向け。ベータだが実用水準には達している。

**ブラウザが最適な場面：** ローカル環境をセットアップせずに試したい場合、長時間タスクをバックグラウンドで非同期実行したい場合。

現実的な答えとして、多くのエンジニアはCLIとDesktopを軸に使いながら、Remote ControlやTeleportで環境間の境界を流動的に越えていく。Claude Codeが目指しているのは「どの環境でも同じエンジンが動く」という一貫性と「環境間をシームレスに移動できる」というモビリティの両立だ。
