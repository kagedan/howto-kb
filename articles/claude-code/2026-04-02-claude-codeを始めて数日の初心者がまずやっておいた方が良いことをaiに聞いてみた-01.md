Claude Codeを使い始めて数日が経ちました。  
カレンダー登録やオセロ・クイズアプリ作成など、いくつか試してはみたものの「自分、ちゃんと使えているのかな？」という不安がずっとありました。

そこで「初心者向けにまず何をやるべきか」を、Claude CodeをブラウザベースのUIで使えるCoworkにそのまま聞いてみました。  
今回はその回答をまとめつつ、自分なりの補足を加えながら整理しておきます。

## Coworkへ投げたプロンプト

```
Claude Codeを始めてまだ数日の初心者です
・やっておいた方が良い設定
・追加した方が良い機能
・試しておいた方が良い機能
・初心者はこういうのをやってみたらいい
みたいなのがあれば教えてほしい
できれば情報ソースは最新である方が嬉しい（最新機能をカバーできているもの）
```

「最新情報でお願いしたい」と書いたのは、Claude Codeはアップデートが頻繁なので、古い情報をもとに「なんか違う」となるのを避けたかったからです。

## 回答まとめ：まずやっておくべき設定2つ

### 1. /init コマンドを実行する

プロジェクトフォルダで最初に /init を実行するだけで、ClaudeがコードベースをはClaude が分析して CLAUDE.md を自動生成してくれます。  
ビルドコマンドやテスト手順などを自動で検出してくれるので、「まず何を書けば良いか分からない」という最初の壁を越えやすくなります。

### 2. /config で設定を確認する

ここを一度確認しておくと、「なぜか意図しない動きをする」というトラブルを事前に減らせます。

## 絶対に押さえておきたい機能

### CLAUDE.md（これが最重要）

Claudeがセッションをまたいで「覚えておく」ためのファイルです。  
毎回同じ説明を繰り返さなくて済むようになる、というのが一番の利点です。

```
このプロジェクトのルール
- ビルド: `npm run build`
- テスト: `npm test`
- インデント: 2スペース
- コミット前に必ずテストを実行
```

200行以内に収めると効果的とのこと。  
保存場所の使い分けもポイントで、

と分けて管理できます。

### Auto Memory（自動学習）

/memory コマンドでオン/オフできる機能です。  
作業中にClaudeが気づいたこと、たとえば「このプロジェクトのビルドはこのコマンド」「このコードスタイルが好み」などを自動でメモしてくれます。  
オンにしておくだけで、使うほど自分向けに育っていくイメージです。

## 試しておきたい機能（少し慣れてきたら）

### カスタムスキル（/スキル名）

繰り返し使う作業をコマンド化できる機能です。  
「コードレビュー」「デプロイ手順」「コミットメッセージ作成」など、パターン化しやすい作業に向いています。  
~/.claude/skills/ にフォルダを作って SKILL.md を書くだけで使えるようになります。

### Hooks（自動化・バリデーション）

Claudeがツールを使う前後に、自動で処理を走らせられる機能です。  
「危険なコマンドをブロックする」「変更後にテストを自動実行する」といった使い方ができます。  
まだ自分は試せていませんが、作業の安全性を上げるために使いこなしたい機能の一つです。

### MCP（外部ツール連携）

Google Drive、Slack、Jiraなどと連携できます。  
「SlackのメッセージをClaudeに読ませてコードに反映」「Jiraのチケットを更新」といったことが可能になります。  
ITコンサルの仕事と組み合わせると、情報収集や議事録整理などにも応用できそうだと感じています。

## 最初の1週間でやること（推奨順）

Coworkが提案してくれた「最初の1週間でやること」のロードマップです。

1. cd プロジェクト && claude で起動 → /init を実行
2. 生成された CLAUDE.md を自分好みに編集
3. /config を開いてモデルと権限を確認
4. Auto Memory をオンにして使い続ける
5. 繰り返し使う作業があればスキルを作る

この順番で進めると、「とりあえず動かしてみた」状態から「ちゃんと使えている」状態に近づけそうです。

## やってみての感想

実際に /init を実行して CLAUDE.md を作るところまで試してみました。

手順自体はそこまで複雑ではなく、慣れてしまえば簡単にできそうな印象です。  
ただ、画面に表示される内容が英語で書かれている部分も多く、「これはどういう意味？」「ここはどう判断すればいい？」と迷う場面が何度かありました。

そういうときに活躍したのが、Coworkへの「スクショ＋質問」です。  
画面をそのまま貼って「これどうしたらいいですか？」と聞けば、その画面に合わせて丁寧に教えてくれます。  
「パソコン教室でマンツーマン授業を受けるって、こういう感じなのかな？」と思いました（実際に通ったことはないのですが）。

ただ、迷いながら進めていたところでリミットに達してしまい、今回は CLAUDE.md を作るところまでで終了。  
続きの設定（Auto Memory、/config の確認、カスタムスキルなど）はまた改めてやってみようと思います。

「迷ったらスクショで聞く」というスタイルが意外と快適だったので、次回はもう少しサクサク進められそうな気がしています。

ただ、Mac miniで作業しながらスクショを撮っていると、どんどんデスクトップ上にファイルが溜まっていくのが地味に気になりました。  
「スクショを自動で整理する」みたいなスキルをClaude Codeで作ってみるのも、次に試してみたいことの一つです。
