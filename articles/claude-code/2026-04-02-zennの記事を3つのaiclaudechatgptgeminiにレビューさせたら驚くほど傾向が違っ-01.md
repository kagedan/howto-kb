## はじめに

AIにレビューを頼むと、もっともらしいフィードバックが返ってくる。しかし、それをそのまま全採用すると、そのモデルが考える「良い文章」の平均に引っ張られる。

AIのレビュー結果は「読むもの」ではなく「採掘するもの」だと思っている。モデルごとにバイアスが違うので、盲点の位置も違う。複数のモデルにレビューさせて、使える指摘だけを抜き、残りは捨てる。

先日、Zennに「[Claude Codeで商用プロダクトを開発して気づいた、AIが静かに壊すもの](https://zenn.dev/cognitiveosmdl/articles/7472256f950b07)」という記事を書いた。この記事の執筆過程で、Claude・ChatGPT・Geminiの3つのAIにレビューを依頼した。その結果、上記の考えを裏付ける面白い傾向が見えたので共有する。

本記事は厳密な比較検証ではない。1本の記事を3つのAIにレビューさせてみたら、こんな傾向が見えた、という実践報告である。

## やったこと

### レビュー対象

セマンティックドリフト（AIが生成する成果物の意味的なズレ）について書いた技術記事。約5000字。

### レビュー依頼

3つのモデルに以下の同一プロンプトを投げた。

```
添付したドキュメントはzenn.devに掲載予定の内容です。
「認知を1つ変えているか。3行で要点が伝わるか。読者の前提に接続しているか」
という視点でレビューしてください。
```

## 結果：3モデルの指摘を並べてみる

同一プロンプトに対して、3モデルが返してきたフィードバックの重心がはっきりと違っていた。以下、観点ごとに整理する。

### 「認知を1つ変えているか」への回答

**ChatGPT** は、記事の焦点が「現象の発見」と「ツール紹介」の2つに割れていると指摘した。そして改善案として「AIは正しく開発するのではなく、意味がズレたまま整合していく」というパンチラインを冒頭に置くことを提案した。

**Gemini** は、認知変化は十分に達成されていると評価した上で、「セマンティックドリフト」という言葉の定義をもう少し厚く書くことを提案した。

**Claude** は、ChatGPTと同じく焦点の分裂を指摘したが、改善の方向が違った。パンチラインの追加ではなく、「ツールの詳細を削って、現象の発見に重心を寄せる」ことを提案した。つまり「足す」ではなく「削る」。

### 「3行で要点が伝わるか」への回答

**ChatGPT** は、まとめの3行を「だから何？が弱い」と評価し、より挑発的な表現への書き換えを提案した。

**Gemini** は、まとめの3行は機能していると評価した上で、そのエッセンスを冒頭にも配置することを提案した。

**Claude** は、まとめではなく冒頭の弱さを指摘した。冒頭の数行に「何が起きるのか」の具体性が足りないとして、読者が追体験できる具体的なシーンから始めるリライト案を示した。

### 「読者の前提に接続しているか」への回答

ここが最も差が出た。

**ChatGPT** は、対象読者が「AI開発をしている人」と「ChatGPTでコードを書くレベルの人」に分かれるとし、冒頭に「ChatGPTやClaudeでコードを書かせていて」という一文を追加すれば全AIユーザーに広がると提案した。

**Gemini** は、Zennの読者層にはトレーサビリティマトリクスを用いた開発を日常的に行っていない人が多いと分析した。そして「GitHubのIssueとPRの紐づけ」「数日間にわたるAIとのチャットセッション」など、読者が自分の経験に置き換えるための具体的な橋渡しを提案した。

**Claude** は、他の2モデルが触れなかった点を指摘した。「自分はその原因を特定し、対策を打った側の人間である」という一文が、Zennの読者コミュニティでは「マウントを取っている」と受け取られるリスクがあるというものだ。これは表現や構成の話ではなく、**読者コミュニティの文化的規範**に関する指摘だった。

## 見えてきた各モデルの傾向

1本の記事に対するレビューなので一般化はできないが、今回の結果から見えた傾向を整理する。

### ChatGPTの傾向：表現のインパクトを重視する

ChatGPTのレビューは一貫して「刺さる一文を追加する」「パンチラインを置く」方向に向かった。問題の解決策が「足し算」に偏る傾向がある。

また、レビューの評価が肯定寄りに振れやすい。「バズ圏に入る」「Zennで上位いける」といった着地が多く、配信経路や読者規模といった記事の外側の変数は考慮されない。

指摘自体の粒度は「構成・論旨・読者接続」のレベルで、具体的な改善案は表現の差し替え提案が中心だった。

### Geminiの傾向：情報の過不足と読者像を重視する

Geminiのレビューは「読者が次に何を知りたがるか」を具体的に指摘する傾向があった。

同一プロンプトのレビューとは別に、「Zennの読者目線でレビューしてください」と依頼した際には、使用したEmbeddingモデルの具体名、CLIの出力例、CLAUDE.mdの記述例、ツールの公開状態という4つの「読者が記事を読んだ後に抱く具体的な疑問」を挙げた。

さらに「技術的主張に事実誤認がないか検証してください」と依頼した際には、Embeddingの肯定/否定逆転の死角という、他の2モデルからは出なかった技術的に正確な指摘を返した。

プロンプトで観点を明示すると、その方向に深く掘る傾向がある。

### Claudeの傾向：構造の取捨選択と文化的文脈を重視する

Claudeのレビューは「足す」より「削って尖らせる」方向に向かった。ツールの技術詳細を別記事に分離するか補足セクションに畳むことを提案し、「この記事の主役はセマンティックドリフトという現象の発見であり、ツールはその証拠にすぎない」と明言した。

また、Zennの読者コミュニティの文化的規範に関する指摘（マウントに見えるリスク）は、3モデルの中でClaudeだけが出した観点だった。

一方で、整合性の高さゆえに出力が「きれいすぎる」傾向がある。構造化が強く、論旨にブレがない代わりに、読者の感情に引っかかる「粗さ」が薄れやすい。

## 実際に何を採用したか

3モデルのレビューをすべて採用したわけではない。各指摘を以下の基準で取捨選択した。

### 採用した指摘

| 指摘元 | 指摘内容 | 採用理由 |
| --- | --- | --- |
| ChatGPT | 冒頭に読者の体験と接続する文を追加する | 他のモデルからは出なかった「初見の読者の離脱」への対策 |
| ChatGPT | まとめの3行を「現象→原因→必要なもの」の構造に整理する | 情報の並び順の改善として妥当 |
| Gemini | CLAUDE.mdの具体的な記述例を載せる | 読者の持ち帰り価値が高い |
| Gemini | ツールの公開状態について明記する | 読者の疑問を先に解消できる |
| Gemini | Embeddingの肯定/否定逆転の限界に触れる | 技術的に正確で、信頼性が上がる |
| Claude | 「対策を打った側の人間である」のトーンを調整する | Zennの読者文化への配慮として妥当 |
| 全モデル共通 | トレーサビリティを使わない読者への橋渡し | 3モデルが同じ方向を指摘しており、信頼性が高い |

### 採用しなかった指摘

| 指摘元 | 指摘内容 | 不採用理由 |
| --- | --- | --- |
| ChatGPT | パンチラインを冒頭に追加する | 記事の淡々としたトーンと合わない |
| ChatGPT | タイトルを変更する | 既知の言葉で構成された現タイトルの方がクリック率が高いと判断 |
| ChatGPT | まとめの表現をより挑発的にする | 記事の誠実さを損なう |
| Claude | 冒頭を具体的なシーン描写からリライトする | 既に修正済みの冒頭で十分に機能している |
| 全モデル共通 | ツール詳細の比重を下げる | 技術詳細は既に折りたたみ（:::details）で対応済み |

## やってみて分かったこと

### 同じプロンプトでも、モデルごとに「良いレビューとは何か」が違う

3モデルに同一のレビュー依頼を投げたが、ChatGPTは「表現を磨くこと」、Geminiは「情報を補うこと」、Claudeは「構造を絞ること」を良いレビューだと捉えていた。どれも間違いではないが、どれも偏っている。

### レビュー結果は「読むもの」ではなく「採掘するもの」

各モデルのレビューをそのまま全採用すると、それぞれのモデルが考える「良い文章」の平均に引っ張られる。レビューから使える指摘だけを抜き、残りは捨てる判断が必要だった。

### 3モデルが共通して指摘した点は信頼できる

異なるバイアスを持つ3つのモデルが同じ結論に到達した場合、その指摘はモデルのバイアスではなく、記事の実際の問題である可能性が高い。今回は「読者の前提との接続」について3モデルが同方向の指摘をしたため、優先的に対応した。

### プロンプトの設計とモデルの特性は掛け算で効く

Geminiに「事実誤認の検証」を明示的に指示したとき、Embeddingの技術的な死角という指摘が出た。同じ指示をChatGPTに入れたら同じ粒度の指摘が出るかは分からない。プロンプトは方向を指定するが、その方向でどこまで掘れるかはモデルの特性に依存する。

## まとめ

1本のZenn記事を3つのAIにレビューさせた結果、各モデルの傾向が見えてきた。

!

* ChatGPTは「表現のインパクト」を重視し、足し算で改善しようとする
* Geminiは「情報の過不足」と「読者の次のアクション」を重視する
* Claudeは「構造の取捨選択」と「文化的文脈」を重視し、引き算で改善しようとする

重要なのは、どのモデルが「正しい」かではなく、**盲点の位置がモデルごとに違う**ということである。1つのモデルだけでは見つからない問題が、別のモデルでは見つかる。

マルチモデルでレビューする意味は、ここにある。

そしてレビュー結果を全採用しないことも同じくらい重要である。各モデルのレビューには、そのモデルのバイアスが含まれている。最終的に何を採用し、何を採用しないかは、人間が判断する必要がある。

これは、先の記事で書いた「AIの出力を信頼するのではなく、検証する仕組みを作る」という考え方の、執筆版だと思っている。
