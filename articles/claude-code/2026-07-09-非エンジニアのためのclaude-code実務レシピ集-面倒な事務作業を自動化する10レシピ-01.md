## この記事は誰のためのものか

* ChatGPTやClaudeを**ブラウザで**使ったことはある
* でも「Claude Code」は名前を聞いたことがある程度、または初耳
* プログラミングはできないし、するつもりもない
* ファイル整理、集計、議事録、メール下書きみたいな「地味に時間を食う事務作業」を減らしたい

——そんな方に向けて、**コピペで動く手順**だけを10個まとめました。

ブラウザ版のAIチャットとの一番の違いはここです。ブラウザ版は「文章で答えてくれる」だけですが、Claude Codeは**あなたのパソコンの中のファイルを直接読んで、作って、変更できます**。「このフォルダのファイルを整理して」と日本語で頼むと、実際にファイル名が変わります。ファイルをひとつずつAIチャットにコピペする作業から解放されるのが最大の価値です。

* **無料パート**: Claude Codeの導入手順（Mac/Windows）＋ レシピ1「散らかったファイルの一括整理・リネーム」。ここまでで「自分のPCで動いた」という体験が完結します
* **有料パート**: レシピ2〜10（CSV集計 / 議事録整形 / PDFから一覧表 / お礼メール量産 / 画像一括リサイズ / 週次報告 / Webページの表化 / プレゼン構成 / タスク棚卸し）＋ トラブルシューティング集

まず無料パートでレシピ1まで試してみて、「これは使える」と思ったら先へ進んでください。

## Claude Codeとは（1分で説明）

**Claude Code**は、AIチャットの「Claude」を作っているAnthropic（アンソロピック）社の公式ツールです。「ターミナル」という黒い画面（後述。怖くありません）の中で動き、日本語で指示すると、あなたのPCの中で次のことができます。

「Code」という名前ですがコードが書ける必要はありません。指示はぜんぶ日本語の文章でOKです。

### 料金と必要なもの

すでにブラウザでClaude Proを使っている方は、追加料金なしでClaude Codeも使えます。

### 大事な注意（最初に読んでください）

Claude Codeは**実際にファイルを変更できる**ツールです。だからこそ便利なのですが、次の3つを守ってください。

1. **練習用フォルダで試す**。いきなり仕事の本物のフォルダでやらず、まずコピーで試す
2. **確認画面を雑に許可しない**。Claude Codeはファイルを変更したりコマンドを実行したりする前に「実行していい？」と確認してきます。何をしようとしているか一応目を通してから許可しましょう
3. **結果はそのまま信用せず目視チェック**。集計の数字やメール文面は、送る前・提出する前に必ず自分の目で確認する。AIは間違えることがあります

## 導入手順（Mac / Windows）

専門用語がひとつだけ出てきます。**ターミナル**＝キーボードで命令文（コマンド）を打ち込んでPCを操作する画面のことです。マウスの代わりに文字で操作する、と思えばOKです。以下、コマンドはすべてコピペで大丈夫です。

### Macの場合

**手順1: ターミナルを開く**

「command ⌘ + スペース」でSpotlight検索を開き、「ターミナル」と入力してEnter。白か黒の文字だけの画面が開けば成功です。

**手順2: インストールコマンドを貼り付けて実行**

次の1行をコピーして、ターミナルに貼り付けてEnterを押します。

```
curl -fsSL https://claude.ai/install.sh | bash
```

これはAnthropic公式のインストール方法です（[公式ドキュメント](https://code.claude.com/docs/en/setup)に記載のもの）。1〜2分待つとインストールが終わります。

**手順3: ターミナルを一度閉じて、開き直す**

インストール直後は `claude` コマンドが認識されないことがあるため、ターミナルのウィンドウを閉じて、もう一度開いてください。

**手順4: 起動してログイン**

```
claude
```

と入力してEnter。初回はログインを求められるので、画面の案内に従って進めるとブラウザが開きます。**普段Claudeを使っているアカウント**（Pro以上）でログインしてください。一度ログインすれば、次回からは不要です。

**手順5: 動作確認**

ログイン後、Claude Codeの入力欄が表示されたら導入完了です。試しに日本語で「あなたは何ができますか？」と打ってEnterしてみてください。返事が来たら成功です。

終了したいときは `/exit` と入力するか、「control + D」を押します。

なお、インストールできたかどうかだけを確かめたいときは、ターミナルで次を実行するとバージョン番号が表示されます。

```
claude --version
```

### Windowsの場合

> 筆者の検証環境はMacのため、Windowsの手順は[Anthropic公式ドキュメント](https://code.claude.com/docs/en/setup)（2026年7月時点）の記載に基づきます。手順の出典が公式であることは確認済みですが、筆者自身のWindows実機での通し確認は行えていません。この点は正直にお伝えしておきます。

**手順1: PowerShellを開く**

スタートメニューで「PowerShell」と検索して開きます。青い画面で、行の先頭に `PS C:...` と表示されていればPowerShellです。

**手順2: インストールコマンドを貼り付けて実行**

```
irm https://claude.ai/install.ps1 | iex
```

もし「'irm' は認識されません」というエラーが出た場合、開いているのはPowerShellではなく「コマンドプロンプト」です（先頭に `PS` が付いていない画面）。その場合は次のコマンドを使ってください。

```
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```

逆にPowerShellで「The token '&&' is not a valid statement separator」と出たら、PowerShell用の1つ目のコマンドを使ってください。

補足: 公式ドキュメントでは、Windowsでは[Git for Windows](https://git-scm.com/downloads/win)というツールを追加で入れることが推奨されています（Claude Codeが作業に使う「シェル」の選択肢が増えます）。なくても動作しますが、レシピの動きが本記事の例と多少変わる可能性があります。

**手順3以降はMacと同じ**

PowerShellを開き直し、`claude` と入力してEnter → ブラウザでログイン → 「あなたは何ができますか？」で動作確認、です。

### 導入でつまずいたら

この記事の最後の「トラブルシューティング集」に、代表的なエラーと対処をまとめてあります。また `claude doctor` というコマンドを実行すると、インストール状態を自己診断してくれます。

## レシピの使い方（共通ルール）

各レシピは同じ型で書いてあります。

共通の操作はこれだけです。

1. ターミナルを開く
2. `cd` に続けて対象フォルダの場所を指定して移動する（`cd` ＝フォルダを移動する命令。**Macならフォルダをターミナルにドラッグ＆ドロップすると場所が自動入力されて楽です**。`cd ` と打ってからドラッグしてEnter。WindowsのPowerShellでも同様にドラッグで入力できるとされていますが、こちらは筆者未検証です。うまくいかない場合はエクスプローラーのアドレスバーからパスをコピーして貼り付けてください）
3. `claude` と入力して起動する
4. レシピのプロンプトを貼り付けてEnter
5. Claude Codeが「このコマンドを実行していい？」「このファイルを変更していい？」と聞いてきたら、内容を見て許可する

なお、本記事の「期待結果の例」はすべて**架空のサンプルデータ**での実行結果です（実在の企業名・個人名は含みません）。また、AIの出力は毎回まったく同じにはなりません。表現や体裁は変わりますが、同じ種類の結果が得られることは全レシピで確認済みです。

**所要時間の目安: 5〜10分**

### ① 課題

### ② 事前準備

### ③ コピペ用プロンプト

```
このフォルダのファイルの中身を読んで、内容がわかるファイル名に変えてください。ルール: 「日付_種類_内容.txt」の形式（日付はYYYYMMDD、不明なら日付なし）。変更前と変更後の対応表も最後に見せてください。
```

拡張子（.txtの部分）は自分のファイルに合わせて読み替えてください。「PDFはそのまま.pdfで」と一言足してもOKです。

### ④ 実行手順

1. ターミナルを開き、`cd ` に続けて練習用フォルダをドラッグしてEnter
2. `claude` で起動
3. 上のプロンプトを貼り付けてEnter
4. ファイルを読む・名前を変える確認が出たら、内容を見て許可する

### ⑤ 期待結果の例

筆者が架空の書類5ファイルで実行した際は、全ファイルの中身が読まれた上でリネームされ、最後にこのような対応表が表示されました。

（変更前 → 変更後）

* aaaa.txt → 20260701\_見積書\_取引先品目A下書き.txt
* document(1).txt → 請求書\_会議室レンタル2026年5月分.txt
* メモmemo\_final\_v2最新.txt → メモ\_新商品企画.txt
* スキャン001.txt → 請求書\_オフィス備品2026年6月分.txt
* 無題のドキュメント.txt → 20260620\_議事録\_定例ミーティング.txt

気が利くのは、日付が特定できないファイルを勝手に捏造せず「日付なし」ルールに従って処理し、その判断理由まで説明してくれた点です。

### ⑥ つまずきポイント

* **いきなり数百ファイルでやらない**。まず10個以下で挙動を確かめ、命名ルールが気に入らなければプロンプトのルール部分を直して再実行
* **必ずコピーで試す**。「対応表を見せて」を必ず入れておくと、後から元に戻す手がかりになります（この一文はプロンプトに含めてあります）
* 画像だけのスキャンPDFなど「文字データが入っていないファイル」は中身を読めないことがあります。その場合Claudeは読めなかった旨を報告してくるので、そのファイルだけ手動で対応してください
