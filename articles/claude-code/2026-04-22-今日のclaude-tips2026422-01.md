## 💻 API・開発向け

Claude Codeは、ターミナルから直接Claudeにコーディング作業を任せられる公式[CLIツール](https://scrapbox.io/yougokaisetsu/CLI%E3%83%84%E3%83%BC%E3%83%AB)です。[リポジトリ](https://scrapbox.io/yougokaisetsu/%E3%83%AA%E3%83%9D%E3%82%B8%E3%83%88%E3%83%AA)内の[ファイル](https://scrapbox.io/yougokaisetsu/%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB)を読み書きしながら、バグ修正・[リファクタリング](https://scrapbox.io/yougokaisetsu/%E3%83%AA%E3%83%95%E3%82%A1%E3%82%AF%E3%82%BF%E3%83%AA%E3%83%B3%E3%82%B0)・テスト作成・新機能実装などを自然言語で指示できます。npm install -g @anthropic-ai/claude-code でインストールし、プロジェクトのルート[ディレクトリ](https://scrapbox.io/yougokaisetsu/%E3%83%87%E3%82%A3%E3%83%AC%E3%82%AF%E3%83%88%E3%83%AA)で claude と入力するだけで対話セッションが始まります。[Git](https://scrapbox.io/yougokaisetsu/Git)や[IDE](https://scrapbox.io/yougokaisetsu/IDE)、[MCP](https://scrapbox.io/yougokaisetsu/MCP)[サーバー](https://scrapbox.io/yougokaisetsu/%E3%82%B5%E3%83%BC%E3%83%90%E3%83%BC)とも連携できるため、ローカル開発フロー全体にClaudeを組み込めるのが最大の強みです。

> $ cd my-project  
> $ claude  
> > まず src/ 配下のTypeScriptファイル構成を把握してください。  
> > そのうえで、未使用の import 文をすべて削除する計画を立て、  
> > ファイルごとに diff を見せてから、私の承認を得て順に適用してください。

### 💡 ポイント

「まず計画を立てる → diffを見せる → 承認後に適用」という3段階を指示すると、意図しない変更を防ぎつつ安全に作業を任せられます。いきなり実装させるのではなく、Claudeに「思考 → 提示 → 実行」のリズムを作らせるのがコツです。
