## はじめに

私は去年の11月からClaudeを使い始めました。  
主な用途はコード生成です。

業務ではフロントエンド開発（React / TypeScript など）で使い、  
プライベートでは趣味のアプリ開発にも活用しています。  
ツールとしては Claude Code を使っています。

プランはPro Maxの$100プラン（日本円で月約16,000円）を選択しました。

ただし、利用状況にはかなりムラがあります。  
がっつり使える時期もあれば、あまり使えない時期もあり、  
「サブスクリプション、もったいないな...」と感じることが増えていきました。  
１日5時間の制限に達するくらいガッツリ使う日もあれば、  
まったく触らない日もありました。  
平均的には１日2〜3時間ほどでしょうか。

最初は「生産性が上がるのだから、これくらいの投資は当然」と思っていました。  
しかし、5ヶ月が経ち、気づけば総額で7万円以上をClaudeに費やしていました。  
毎月のクレジットカード明細を確認するたびに、貯金がみるみる減っていく実感が湧いてきます。  
このペースで使い続けると、1年で約19万円になります。

これまでにClaudeに生成してもらったコードは、  
業務・プライベート合わせて５つくらいのプロジェクトに及びます。  
それだけ頼りにしていたからこそ、なおさら「この出費は本当に続けられるのか」と  
考えるようになりました。

自分の経済状況と真剣に向き合ったとき、  
「このままでは持続可能ではない」と強く感じるようになったのです。

そこで私は、Claudeからの乗り換えを本気で検討し始めました。  
この記事では、乗り換えを決意した理由、比較検討した候補、そして最終的に選んだAIについて、  
実体験をもとに詳しく紹介します。

「AIにかけるお金が重くなってきた...」と感じている方の、  
何かしらのヒントになれば嬉しいです。

## なぜ乗り換えを決意したのか

### 料金を細かく計算してみた

月16,000円の投資は、単純計算で1日あたり約533円です。  
平均して1日2〜3時間使うとして、1時間あたり約200円前後になります。

しかし、問題は「使わない日」が存在することです。  
月に10日ほどまったく触らない日があるとすると、  
その月だけで約5,300円が「使ってもいないのに消えている」計算になります。  
月によっては、実質半分の日数しかコーディングしていないこともありました。

1時間あたりのコストで見れば安いのかもしれません。  
でも、使わない日も同じだけお金が引かれていくサブスクリプションモデルに、  
次第に違和感を覚えるようになりました。  
体調を崩したときには「サブスクがもったいない」と苦しむようになってきました。  
これはメンタルによくありませんでした。

### 競合の台頭で相対的な価値が下がった

去年の11月に使い始めたとき、コード生成の品質においてClaudeは圧倒的でした。  
当時は「これだけの性能なら月16,000円は妥当」と納得していました。  
むしろ、Claude以外に選択肢がないようにさえ思えました。  
Claude Codeが一強だったのです。

しかし、ここ数ヶ月で状況は大きく変わりました。  
DeepSeek, Kimi, Qwen, MiniMax などオープンソース系のAIが、  
自分のケースではコード生成においても十分使える品質を実現し始めています。

「Claudeでなければ得られない価値」が相対的に薄れていったのです。

### 決め手となった瞬間

ここ１か月ほど、DeepSeek APIに$50クレジットを突っ込んで試してみた結果が衝撃的でした。  
仕事でも使えるレベルだったのはもちろんのこと、なによりクレジットが全然減らないのです。

もちろん、正直に言えばClaudeの方が優れたモデルです。  
特に私はNixOSを利用しているのですが、  
複雑なバグの調査やシステム設定のデバッグにおいては、  
Claudeの方が明らかに優れていました。

それでもです。

もっと安い選択肢がある。  
それだけで十分な理由になりました。

### 乗り換えという選択

結果的に、私は「Claudeをやめる」という選択をしました。  
決してClaudeの性能に不満があったわけではありません。  
むしろ、とても満足していました。

ただ、自分にとって持続可能な選択をする。  
それだけのことです。

次の章では、実際に検討した乗り換え先の候補について詳しく見ていきます。

## 乗り換え先の候補と評価軸

乗り換え先を検討するにあたり、まずは評価軸を明確にしました。  
そして、候補となるサービスをモデルレイヤーとルーターレイヤーに分けて整理しました。

### 評価軸

私は以下の4つの軸で各候補を評価することにしました。

1. **価格（月額/従量制）**

   * 月額いくらか、またはAPIの従量課金がどの程度か
   * 自分の利用量で月あたりどのくらいのコストになるか
2. **コンテキスト長**

   * 一度に入力できるコード量（トークン数）
   * 大規模なコードベースを扱う際に重要
3. **コード生成品質**

   * 実際に使ってみた感覚ではなく、**ベンチマークスコア**で客観的に判断
   * 複数のモデルを比較する際の共通指標として活用
4. **マルチモーダル対応**

   * 画像入力ができるかどうか
   * スクリーンショットや図解をコード生成に活かせるか

### 候補の整理

検討した候補は大きく2つのレイヤーに分かれています。

**モデルレイヤー（実際のAIモデル）**

**ルーターレイヤー（複数プロバイダをまとめるサービス）**

つまり、選択肢は以下の2パターンになります。

* パターンA: DeepSeekまたはKimiの公式APIを直接契約する
* パターンB: OpenCode GoやOpenRouterなどのルーター経由で利用する

ルーターを経由するメリットは、一つのインターフェースで複数のモデルにアクセスできることです。  
モデルごとにAPIキーを管理する手間が省け、必要に応じて使い分けも可能になります。

次の章では、これらの候補を評価軸に沿って比較していきます。

## 各候補の比較検討

それでは、前章で整理した候補を評価軸に沿って比較していきます。  
評価軸は以下の4つです。

1. 価格（月額/従量制）
2. コンテキスト長
3. コード生成品質（SWE-benchで判断）
4. マルチモーダル対応

### DeepSeek（公式API）

価格  
$50クレジットを投入して1ヶ月ほど試用したところ、クレジットがほとんど減りませんでした。  
体感的には月に10〜15ドルもあれば十分で、20ドルは余る見込みです。

公式のAPI料金は以下の通りです。

* V4-Pro（1.6Tパラメータ、49B活性）：入力$0.435/1M、出力$0.87/1M
* V4-Flash（284Bパラメータ、13B活性）：入力$0.14/1M、出力$0.28/1M  
  キャッシュヒットすればさらに安くなります。

コンテキスト長  
1Mトークンです。  
大規模なコードベースを扱う際にも十分な長さです。

コード生成品質（SWE-bench）  
DeepSeek-V4-Pro-MaxはSWE-bench Verifiedで80.6%を達成しており、オープンウェイトモデルでは最高スコアです。  
クローズドモデルと比較しても、Gemini 3.1 Pro（80.6%）と同点、Claude Opus 4.6（80.8%）とほぼ互角の性能です。  
LiveCodeBenchでは93.5%という最高スコアを記録しています。

マルチモーダル  
画像には非対応です。

実際に使ってみた感想  
フロントエンド開発では問題なく使えました。  
ただし、オープンソースのChatGPT代替アプリJanでAPI利用を試みたところ、function callingがうまく動作しませんでした。  
ツール連携の面ではやや課題が残ります。

### Kimi K2.7 Code（公式API）

概要  
2026年6月12日にMoonshot AIがリリースしたコード生成に特化したモデルです。  
Kimi K2.6をベースに、コーディングタスク向けにチューニングされています。  
アーキテクチャは1.1T総パラメータのMoE（Mixture-of-Experts）で、毎トークン32Bのパラメータが活性化されます。

価格  
API価格は入力$0.95 / 1Mトークン、出力$4.00 / 1Mトークンです。  
Claude Opus 4.8（入力$5/1M、出力$25/1M）と比較すると約6倍安く、コストパフォーマンスは非常に高いです。

また、KimiはKimi Codeというコーディングエージェントも提供しています。  
プランは以下の通りです。

* Moderato：$19/月
* Allegretto：$39/月
* Allegro：$99/月
* Vivace：$199/月

コンテキスト長  
256Kトークン（262,144）です。  
DeepSeekの1Mには及びませんが、多くのコード生成タスクでは十分でしょう。

コード生成品質（SWE-bench）  
注意が必要な点があります。

Kimi K2.7 CodeはSWE-bench Verifiedのスコアを公開しておらず、独自ベンチマーク（Kimi Code Bench v2、Program Bench、MLS Bench Liteなど）のみで評価を主張しています。  
これらの独自ベンチマークではK2.6比で10〜31.5%の向上を謳っていますが、第三者による独立検証はまだありません。

参考までに、ベースモデルのKimi K2.6はSWE-bench Verifiedで80.2%を記録しており、当時はオープンソースの最強モデルでした。K2.7 Codeがこのスコアをどこまで伸ばしているかは、今後の第三者検証を待つ必要があります。

マルチモーダル  
画像入力にネイティブ対応しています。  
スクリーンショットをコード生成に活かせるのは大きなメリットです。

その他の特徴

* ライセンス：Modified MIT（オープンソース）
* 思考モードが強制（オフにできない）
* 高速版（6月15日より提供開始）：出力速度が通常版の5〜6倍

### OpenCode Go（ルーター）

特徴  
DeepSeek、Kimiなどのオープンソース系モデルを呼び出せるルーターサービスです。  
クローズドモデル（ClaudeやGPTなど）は呼び出せません。  
OpenCodeで手軽に利用できるのが特徴です。

価格  
月$10で利用可能です。  
ただし、以下の制限があります。

* 5時間の制限（$12分の利用相当）
* 週間の制限（$30分の利用相当）
* 月間の制限（$60分の利用相当）

備考  
なお、OpenCodeには従量課金版のOpenCode Zenもありますが、  
今回はOpenRouterを候補に入れているため検討対象外としました。

### OpenRouter（ルーター）

特徴  
1つのAPIでさまざまなモデルを使えるルーターサービスです。  
オープンソース系モデルだけでなく、ClaudeやGPTなどのクローズドモデルも呼び出せます。  
また、function callingの正規化なども行ってくれるとのことで、  
モデル間の互換性問題を吸収してくれる点が魅力です。

価格  
従量課金制で、モデルごとに料金が異なります。

OpenRouter経由での各モデルの価格は以下の通りです。

* DeepSeek V4-Pro：入力$0.435/1M、出力$0.87/1M
* Kimi K2.7 Code：入力$0.74/1M、出力$3.50/1M

### 検討の結果

比較検討の結果、以下の3段階で判断を下しました。

ステップ1：ルーターを使うか、公式APIを使うか

結論から言うと、ルーターを採用することにしました。

その理由は、若干の手数料がかかるものの、モデルを柔軟に切り替えられる点が魅力的だからです。  
特定のモデルにロックインされるリスクを避け、その時々のタスクに最適なモデルを選べる柔軟性を重視しました。

また、OpenRouterはクローズドモデルにも対応しているため、将来的にClaudeやGPTなどを使いたくなった場合もシームレスに切り替えられます。  
function callingの正規化機能も、モデル間の互換性問題を吸収してくれる点で魅力でした。

ステップ2：どちらのルーターにするか

結論から言うと、OpenCode Go と OpenRouter を併用することにしました。

私はClaude Codeを卒業したら、OpenCodeに移行する予定です。  
OpenCode Goは月$10と予算内で収まるため、まずはこちらをメインで使います。

ただし、OpenCode Goには月間$60分の利用制限があるため、  
制限に達した場合や、OpenCodeが対応していないモデルを使いたい場合は、  
OpenRouterを補助的に利用することにしました。

この2つを併用することで、コストを抑えつつ、柔軟性も確保できると判断しました。

ステップ3：どのモデルを主に使うか

ルーターは決まったものの、実際にどのモデルを呼び出すかは別の判断です。

私の結論はこうです。

普段はDeepSeekを使う。スクリーンショットが必要な場面ではKimi K2.7 Codeを活用する。

DeepSeekを選んだ理由は以下の通りです。

* SWE-bench 80.6% を信頼する
* コンテキスト長が1Mと圧倒的に長い
* コストが非常に安い（月$10-15あれば十分）

一方、Kimi K2.7 Codeはマルチモーダルに対応しているため、スクリーンショットをコード生成に活かしたい場面で重宝します。  
SWE-benchスコアは未公開でやや不透明ですが、実際に試してみて十分使えるレベルであれば、用途に応じて使い分けていくつもりです。

そして、これら以外のモデルも適宜試してみることにしています。  
ルーターを導入した最大のメリットは、新しいモデルが出てきたときにすぐに試せることです。

## まとめ

今回は、Claude Pro Max（$100/月）から、OpenCode GoとOpenRouterを併用し、DeepSeekをメインにKimi K2.7 Codeを補助的に使う構成に乗り換える決断をした経緯をまとめました。

乗り換えの決め手は、何よりもコストです。  
月16,000円の出費は、使わない日もあることを考えると、自分の経済状況に合わなくなっていました。  
また、DeepSeekやKimiといったオープンソース系AIへの興味があったことも大きいです。

最終的に選んだ構成は以下の通りです。

* ルーター：OpenCode Go（メイン / 月$10）+ OpenRouter（補助 / 従量課金）
* メインモデル：DeepSeek（コスパ・コンテキスト長・SWE-bench 80.6%）
* サブモデル：Kimi K2.7 Code（マルチモーダルが必要な場面で）

Claude Codeを卒業したらOpenCodeに移行する予定で、OpenCode Goの月間制限に達した場合や、対応していないモデルを使いたい場合はOpenRouterで補う計画です。
