Anthropic がすごい機能を発表したんです。

Claude の Managed Agents API に「Dreams」っていう機能が加わりました。名前は面白いですけど、これエージェント開発の根本的な問題を解決してます。

その解決した問題をまとめました↓ https://t.co/L6APSvuOfI

その問題とはセッションを重ねるごとに、メモリ腐敗が起きること。

・同じ情報が重複して書かれてる
・昔の情報と今の情報が矛盾
・もう二度と起きない事象の記録が残ってる

セッションを積み重ねると、エージェントは肥大化したメモリを毎回読み込む→トークン消費量増 → パフォーマンスが低下

ここで Dreams が登場です。

Anthropic の新機能を一言で言うと
「エージェントの睡眠時間」

非同期ジョブで動く。Claude が以下をやります。

・既存のメモリストア読み込む
・過去 100 セッションの記録を参照
・重複をマージ、矛盾を解決、腐敗を削除
・新しいメモリストア出力

名前にも深い意図がありまして。

人間の脳は睡眠中にメモリ統合します。

・経験を再生
・弱い繋がりを削除
・意味のあるパターンを強化

ClaudeのDreamsは構造的に同じ仕組みです。
エージェントメモリに対して、人間の睡眠と同じ「整理・洗練」プロセスを適用してる。

すごい発想だなと思います。

従来のエージェントは 1 つのループだけ。
●行動 → メモリ記録 → 次の行動 → メモリ記録...

でも Dreams で 2 つのループになります。
[メインループ] 行動 → 記録
      ↕
[Dream ループ] 反思 → 再構成 → 改善

メモリが「進化するナレッジベース」に変わる。

Dreams は単なる「要約」ではないんです。

実装の特徴：
パターンマイニング：セッション記録から明示されてないパターン発見
カスタム指示対応：「コーディングの好みは残す、デバッグログは削除」みたいな指示可能
ライブ観察：Dream 実行中、何が読み込まれて何が書かれるかリアルタイム観察可能

料金システムは標準の API トークン料金で課金。

Opus か Sonnet 選択可能
入力セッション数・長さに応じてスケール
小規模から試すのを推奨

「毎日 20 セッションで 1 回 Dream 実行」くらいのペースなら、月 100~500 円程度の追加コストですむはず。

小さく試す価値があります。

短期効果：

メモリストアが 30~50% 圧縮
トークン消費量が減る
レイテンシー改善

中期効果（3ヶ月以上）：

エージェントが自分の経験から学ぶ外観
ドメイン固有のパターン自動発見
回答の一貫性向上

複合効果で、エージェント性能が時間とともに自動改善。

従来の仮説：「より優れたエージェント = より大きなコンテキストウィンドウ」

これ、Dreams で否定される可能性があります。

無限のコンテキストを与えても、ノイズと矛盾が蓄積される。答えは容量ではなくキュレーション。

メモリ品質のライフサイクル管理が重要になる。

※注意事項

✅ すべきこと：
セッション増えるごとに定期実行
カスタム指示を活用
出力メモリを必ず監査

❌ すべきでないこと：
出力を盲目的に信頼
大量一括実行
メモリスキーマが甘いまま実行

Dreams は強力ですけど、初期メモリ設計とプロンプト品質が前提です。

Anthropic は段階的に AI 能力層を積み重ねてます。

音声（会話インターフェース）
脳（推論・判断）
メモリ（知識の蓄積）
睡眠（Dreams による統合・改善）←いまここ

人間の知能進化と同じ順序で、AI が多層化してる。

次は「マルチエージェント間のメモリ共有」とか出てくるかな。

要するにこういうことです。

Dreams = エージェントのメモリを自動デフラグするシステム

一度設定すれば、バックグラウンドで無限にメモリが進化し続ける。

メモリ品質が自動改善
トークン効率が上がる
エージェント性能が複合効果で向上

小規模から始める価値、すごくあります。
