## 1. はじめに

近年、Claude CodeやRoo Code (旧Cline)、GitHub Copilotなどの自律型AIエージェントが台頭し、開発の風景は劇的に変わりました。しかし、実プロジェクトで本格的に使いこなそうとすると、以下のような新たな雑務に悩まされることはないでしょうか？

* 「要件定義」「設計」「実装」のフェーズごとに、**毎回同じようなプロンプト文章**を作成するのに手間がかかる。
* プロジェクト固有のルールや前提条件をエージェントに**伝え忘れて、手戻りが発生**する。
* 複数のタスクを並行していると、**どの対話セッションがどの機能実装のものか分からなくなった**り、**セッションの情報が消えてしまった**りする。

本記事では、エージェントへの「指示出し」の認知負荷を限りなくゼロにし、純粋な設計と実装に集中するための汎用Bash(Zsh)スクリプト「Agent Dev Base」の作り方とワークフローをご紹介します。  
**記事の後半に完全なスクリプト全文を記載しています。お急ぎの方は 「5.導入方法」からお読みください。**

---

## 2. なぜこのスクリプトが必要だったのか？（開発の背景）

エージェントとの開発において、私は当初「プロンプト管理」と「ルールの学習」の2つの壁にぶつかりました。

### 課題①：定型プロンプトの管理・入力コスト

開発フェーズ（要件定義、設計、実装、テスト）ごとに、エージェントに渡すべき最適な指示（プロンプト）は異なります。これを毎回Markdownファイルに手作業でコピペし、対象ファイルのパスを書き換える作業は、せっかくAIを使っているのに非常にアナログで苦痛でした。

### 課題②：Agentの「Skills（独自ルール）」育成・管理コスト

多くのエージェントには「Memory」機能やカスタムスキル機能が備わっています。しかし、これらに頼りすぎると、プロジェクトごとのルールの切り替えが難しかったり、意図しない学習をしてしまう**ブラックボックス化のリスク**があります。

### 💡 解決策：「セッション主導開発」

そこで行き着いたのが、エージェント自体の記憶力に頼るのではなく、実装タスク（＝セッション）ごとに「文脈（Context）」と「テンプレート化された指示」を物理的なファイルとして切り出し、それを都度エージェントに読み込ませるという堅牢なアーキテクチャです。

## 3. スクリプトの全体像とディレクトリ構造

「Agent Dev Base」は、リポジトリのディレクトリ構造を汚さず、PCのどこからでもシームレスに使える軽量なCLIツールです。`git rev-parse --show-toplevel` を用いてカレントディレクトリから動的にGitルートを解決するため、ディレクトリを移動しても自動でそのリポジトリ専用の環境に切り替わります。

スクリプトを導入すると、リポジトリのルートに以下のような `.agent` ディレクトリが生成されます。

```
.agent/
 ├── templates/                # フェーズごとのプロンプト雛形を一元管理
 │    ├── cnew/                # 設計フェーズ用テンプレート配置場所
 │    └── summary/             # PR・SKILL作成依頼テンプレート配置場所
 ├── active_session            # 現在アクティブなセッション名を記録
 └── sessions/
      └── user-auth-feature/   # タスクごとの作業部屋（セッション）
           ├── context.md      # そのタスクのルール・状態のハブ
           ├── conversations/  # AIとの実際の対話指示書が蓄積される
           └── refs/           # エージェントに読ませる参考資料
```

## 4. 実際のワークフロー（使い方デモ）

このCLIツールを使うと、日々の開発ループは以下のように洗練されます。

### Step 1: タスクの開始 (`session-new`)

新しい機能を実装する際、まずはクリーンな作業部屋を作成します。

```
$ session-new user-auth-feature
📁 セッション 'user-auth-feature' を作成中...
✅ セッションを作成しました: /path/to/.agent/sessions/user-auth-feature
```

### Step 2: テンプレート呼び出しと指示出し (`cnew`)

次に、対話用のファイル（プロンプト）を作成します。`cnew -t` と打つと、`fzf`（あいまい検索）が起動し、あらかじめ用意したテンプレートから使いたいものを即座に展開できます。

```
$ cnew -t
# fzfで「design.md」「plan.md」「exec.md」などを選択
✅ 対話ファイルを作成しました: conversation_20260401_120000.md
```

ここで生成されるファイル(`conversation_<timestamp>.md`)には、エージェントへの「作業ログを `context.md` に自動記録して」というシステムプロンプトが既に埋め込まれています。  
その下に`User Request`欄及び、`Agent Work Log`欄がデフォルトで用意されます。

### Step 3: エージェントの実行と自動記録

生成されたMarkdownファイルをエージェント（Claude Code等）に読み込ませて実行します。  
エージェントは指示に従って実装を行い、完了すると`conversation`ファイルに作業ログを残し、 **その過程で得られた気付きやルールを自ら `context.md` に追記**してくれます。

### Step 4: PR作成と汎用ルールの抽出 (`session-summary`)

実装がすべて完了したら、以下のコマンドを叩きます。

`fzf`により、作業中のセッション名の選択肢が表示されるので、該当のセッションを選択します。（`session-summary user-auth-feature`でコマンド指定も可）  
これにより、蓄積された `context.md` や対話ログを解析し、「PRのサマリー（説明文）」**と**「次回以降使い回せるスキル（汎用ルール）」のプロンプトが一撃で生成されます。これらのプロンプトをエージェントに渡し、成果物を作成してもらいます。

### Step 5: セッションのアーカイブ (`session-archive`)

セッションでの作業がすべて完了したら、以下のコマンドを叩きます。

`fzf`により作業中のセッション名一覧が表示されるので、アーカイブ対象を選択します。（`session-archive user-auth-feature`でコマンド指定も可）  
`.agent/sessions/archive/`配下に指定したセッションディレクトリが移動されます。

## 5. 導入方法（Setup）

ツールの導入は非常にシンプルです。  
依存関係はインタラクティブな選択UIを提供する `fzf` のみとなります。

**1. `fzf` のインストール**

* **Mac (Homebrew):** `brew install fzf`
* **Windows (WSL2 / Ubuntu):** `sudo apt install fzf`

**2. スクリプトの配置と読み込み**  
以下のスクリプトを `~/scripts/agent-dev-base.sh` などの適当な場所に保存し、ご自身の `~/.zshrc` や `~/.bashrc` に読み込み設定（`source ~/scripts/agent-dev-base.sh`）を追記するだけで使い始められます。

スクリプトの全コード (agent-dev-base.sh)

```
#!/bin/bash
# ~/.zshrc または ~/.bashrc で source すること

unalias cnew session-summary session-new session-list session-archive 2>/dev/null
unset -f cnew session-summary session-new session-list session-archive agent-help _load_template _get_session _init_agent_env 2>/dev/null

_init_agent_env() {
    local git_root
    git_root=$(git rev-parse --show-toplevel 2>/dev/null)
    if [[ -n "$git_root" ]]; then
        _AGENT_DIR="${git_root}/.agent"
    else
        _AGENT_DIR="${PWD}/.agent"
    fi
    _SESSIONS_DIR="${_AGENT_DIR}/sessions"
}

_get_session() {
    local session="$1" usage_msg="$2"
    if [[ -z "$session" ]]; then
        if ! command -v fzf &>/dev/null; then
            echo "使用方法: $usage_msg" >&2; return 1
        fi
        [[ ! -d "$_SESSIONS_DIR" ]] && echo "エラー: $_SESSIONS_DIR が見つかりません" >&2 && return 1
        
        local sessions=($(find "$_SESSIONS_DIR" -maxdepth 1 -mindepth 1 -type d -not -name "archive" -exec basename {} \; 2>/dev/null))
        [[ ${#sessions[@]} -eq 0 ]] && echo "セッションがありません。session-new で作成してください。" >&2 && return 1
        
        session=$(printf '%s\n' "${sessions[@]}" | fzf --prompt="📁 セッション選択> " --height=40% --border --reverse --preview="cat $_SESSIONS_DIR/{}/context.md 2>/dev/null | head -20" --preview-window=right:50%)
        [[ -z "$session" ]] && return 1
    fi
    echo "$session"
}

_load_template() {
    local tpl_file="$_AGENT_DIR/templates/$1/$2.md"
    if [[ ! -f "$tpl_file" ]]; then
        echo "エラー: テンプレートが見つかりません ($tpl_file)" >&2; return 1
    fi
    sed -e "s|{{SESSION_NAME}}|$3|g" -e "s|{{SESSION_DIR}}|$4|g" "$tpl_file"
}

cnew() {
    _init_agent_env
    local session="" tpl_name="" prompt_tpl=0

    while [[ $# -gt 0 ]]; do
        case "$1" in
            -t|--template)
                if [[ -n "$2" && ! "$2" =~ ^- && -f "$_AGENT_DIR/templates/cnew/$2.md" ]]; then
                    tpl_name="$2"; shift 2
                else
                    prompt_tpl=1; shift
                fi ;;
            *) session="$1"; shift ;;
        esac
    done

    session=$(_get_session "$session" "cnew [-t [<name>]] <session-name>") || return 1
    local s_dir="$_SESSIONS_DIR/$session"

    if [[ -z "$tpl_name" && $prompt_tpl -eq 1 && -x "$(command -v fzf)" ]]; then
        local tpl_dir="$_AGENT_DIR/templates/cnew"
        local templates=($(find "$tpl_dir" -maxdepth 1 -type f -name "*.md" -exec basename {} .md \; 2>/dev/null | sort))
        if [[ ${#templates[@]} -gt 0 ]]; then
            tpl_name=$(printf '%s\n' "${templates[@]}" | fzf --prompt="📄 テンプレート選択> " --height=40% --border --reverse --preview="cat $tpl_dir/{}.md 2>/dev/null | head -30" --preview-window=right:50%)
            [[ -z "$tpl_name" ]] && return 1
        fi
    fi

    if [[ ! -d "$s_dir" ]]; then
        read -r -p "⚠️ セッション '$session' は存在しません。作成しますか? (y/n): " res
        [[ "$res" != "y" ]] && return 1
        session-new "$session"
    fi

    local ts=$(date '+%Y%m%d_%H%M%S')
    local conv_file="$s_dir/conversations/conversation_${ts}.md"
    local req_content="[ここに人間の指示・実装方針の選択肢などを記載してください]"

    mkdir -p "$s_dir/conversations"

    if [[ -n "$tpl_name" ]]; then
        req_content=$(_load_template "cnew" "$tpl_name" "$session" "$s_dir") || {
            read -r -p "テンプレートなしで続行しますか? (y/n): " res
            [[ "$res" != "y" ]] && return 1
            req_content="[ここに人間の指示・実装方針の選択肢などを記載してください]"
        }
    fi

    cat > "$conv_file" << EOF
## System: Instructions for Agent
以下の [User Request] を実行し、その結果や思考プロセスを [Agent Work Log] に直接追記してください。追記完了後、\`context.md\` を1行更新してください。

1. [User Request] を実行
2. 実行後、以下を実施:
   - このファイルの「Agent Work Log」に作業・結果・判断理由を記録
   - \`$s_dir/context.md\` に以下を追記:

     ### conversation_${ts} ($(date '+%Y-%m-%d %H:%M'))
     **Request 要約:** [要約]
     **実施内容 要約:** [要約]
     **抽出されたルール/思想:**
     - [今後の実装で再利用可能な設計判断・ルール]

---
## User Request

${req_content}

---
## Agent Work Log
EOF

    echo "$session" > "$_AGENT_DIR/active_session"
    echo "✅ 対話ファイルを作成しました: $conv_file"
    [[ -n "$tpl_name" ]] && echo "   テンプレート: $tpl_name"

    if [[ -n "$EDITOR" ]]; then
        read -r -p "エディタで開きますか? (y/n): " res
        [[ "$res" == "y" ]] && $EDITOR "$conv_file"
    fi
}

session-summary() {
    _init_agent_env
    local session=$(_get_session "$1" "session-summary <session-name>") || return 1
    local s_dir="$_SESSIONS_DIR/$session"
    [[ ! -d "$s_dir" ]] && echo "エラー: '$session' が見つかりません" >&2 && return 1

    local ts=$(date '+%Y%m%d_%H%M%S')
    local out_dir="$s_dir/session-summary"
    mkdir -p "$out_dir"

    local f_design=$(mktemp) f_plan=$(mktemp) f_conv=$(mktemp)
    find "$s_dir/refs/design" -type f 2>/dev/null | sort | sed "s|^|- \`|; s|\$|\`|" > "$f_design"
    find "$s_dir/refs/plan" -type f 2>/dev/null | sort | sed "s|^|- \`|; s|\$|\`|" > "$f_plan"
    find "$s_dir/conversations" -type f -name "*.md" 2>/dev/null | sort | sed "s|^|- \`|; s|\$|\`|" > "$f_conv"
    
    [[ ! -s "$f_design" ]] && echo "  (ファイルなし)" > "$f_design"
    [[ ! -s "$f_plan" ]] && echo "  (ファイルなし)" > "$f_plan"
    [[ ! -s "$f_conv" ]] && echo "  (ファイルなし)" > "$f_conv"

    local pr_out="$out_dir/pr_description_${ts}.md"
    local skill_out="$out_dir/rule_extraction_${ts}.md"
    local date_str=$(date '+%Y-%m-%d %H:%M:%S')

    local pr_tpl="$_AGENT_DIR/templates/summary/pr_draft.md"
    local skill_tpl="$_AGENT_DIR/templates/summary/reusable_patterns.md"

    [[ -f "$pr_tpl" ]] && sed -e "s|{{SESSION_NAME}}|${session}|g" -e "s|{{SESSION_DIR}}|${s_dir}|g" -e "s|{{DATE}}|${date_str}|g" -e "s|{{TIMESTAMP}}|${ts}|g" -e "s|{{SUMMARY_DIR}}|${out_dir}|g" "$pr_tpl" | sed -e "/{{DESIGN_LIST}}/{r $f_design" -e "d}" -e "/{{PLAN_LIST}}/{r $f_plan" -e "d}" > "$pr_out"
    [[ -f "$skill_tpl" ]] && sed -e "s|{{SESSION_NAME}}|${session}|g" -e "s|{{SESSION_DIR}}|${s_dir}|g" -e "s|{{DATE}}|${date_str}|g" -e "s|{{TIMESTAMP}}|${ts}|g" -e "s|{{SUMMARY_DIR}}|${out_dir}|g" "$skill_tpl" | sed -e "/{{CONV_LIST}}/{r $f_conv" -e "d}" > "$skill_out"

    rm -f "$f_design" "$f_plan" "$f_conv"

    echo "✅ サマリープロンプトを生成しました: $out_dir/"
    [[ -f "$pr_out" ]] && echo "   - $(basename "$pr_out")"
    [[ -f "$skill_out" ]] && echo "   - $(basename "$skill_out")"
}

session-new() {
    _init_agent_env
    local session="$1"
    [[ -z "$session" ]] && echo "使用方法: session-new <session-name>" >&2 && return 1
    local s_dir="$_SESSIONS_DIR/$session"
    [[ -d "$s_dir" ]] && echo "⚠️ '$session' は既に存在します" >&2 && return 1

    mkdir -p "$s_dir"/{conversations,refs/plan,refs/design,refs/ticket,refs/pr}
    cat > "$s_dir/context.md" << EOF
# Session: $session

## Task Overview
[このセッションで実現したいこと]

## Current Status
- [🔄] セッション開始

## Technical Constraints
[技術的な制約・使用技術]

## Decisions Made
[重要な決定事項]

## Conversation Log
## Session Info
- Created: $(date '+%Y-%m-%d %H:%M:%S')
EOF

    echo "$session" > "$_AGENT_DIR/active_session"
    echo "✅ セッションを作成しました: $session"
}

session-list() {
    _init_agent_env
    [[ ! -d "$_SESSIONS_DIR" ]] && echo "セッションがありません" >&2 && return 1
    echo "📋 アクティブセッション一覧:"
    
    local found=0
    for s_dir in "$_SESSIONS_DIR"/*; do
        [[ ! -d "$s_dir" || "$(basename "$s_dir")" == "archive" ]] && continue
        local name=$(basename "$s_dir")
        local overview=""
        [[ -f "$s_dir/context.md" ]] && overview=$(awk '/## Task Overview/{getline; sub(/^[ \t]+/, ""); print; exit}' "$s_dir/context.md")
        
        echo "  📂 $name"
        [[ -n "$overview" && "$overview" != "[このセッションで実現したいこと]" ]] && echo "     └─ $overview"
        found=1
    done
    [[ $found -eq 0 ]] && echo "  (なし)"
}

session-archive() {
    _init_agent_env
    local session=$(_get_session "$1" "session-archive <session-name>") || return 1
    local s_dir="$_SESSIONS_DIR/$session"
    [[ ! -d "$s_dir" ]] && echo "エラー: '$session' が見つかりません" >&2 && return 1

    local dest="$_SESSIONS_DIR/archive/${session}_$(date '+%Y%m%d_%H%M%S')"
    mkdir -p "$_SESSIONS_DIR/archive"
    mv "$s_dir" "$dest"
    echo "✅ アーカイブしました: $dest"
}

agent-help() {
    _init_agent_env
    cat << EOF
=====================================================
  🤖 Agent Dev Base - コマンドリファレンス
=====================================================

[現在の対象ディレクトリ]
  $_AGENT_DIR

[対話・タスクの進行]
  cnew [-t <template>] [session]  新規対話ファイル作成
                                  (-t: テンプレート指定、引数省略で一覧から選択)
  session-summary [session]       PR説明文・ルール抽出用のプロンプトを生成

[セッション（タスク）管理]
  session-new <name>              新しいセッションを作成
  session-list                    アクティブなセッション一覧を表示
  session-archive [session]       完了したセッションをアーカイブ
EOF
}
```

## 6. おわりに

AIエージェントの能力は日々進化していますが、それでも「文脈を揃える」「定型フォーマットを守らせる」といった作業は、人間が仕組みでカバーしてあげる必要があります。  
ツールに振り回されるのではなく、「いかにAIとの協業の摩擦をなくすか」を追求することで、開発の楽しさとスピードはまだまだ引き上げられると感じています。

## 7. 付録 (Appendix)

ここからは、実際に導入する際に役立つ高度な設定や、テンプレートの実例をご紹介します。`.agent/templates/` 配下に配置することで、自動的にコマンドから呼び出せるようになります。

### Appendix A: 設計用プロンプト（design.md）

`.agent/templates/cnew/design.md` に配置し、機能の要件定義から詳細設計をAIに壁打ちさせるためのテンプレートです。

詳細な設計用プロンプトの例

```
# 🎯 ミッション
あなたは優秀なアーキテクトです。現在のコードベースを読み込み、以下の「目的と要件」に従って、機能実装のための「詳細設計書（Markdown形式）」を作成してください。
具体的な実装ファイルやクラスの特定・改修方針はあなたの知識と分析に任せます。

# 📋 解決したい課題と目的
{ここに「なぜこの機能が必要か」「何を達成したいか」を記載}

# 💡 実装要件
{ここに「満たすべき具体的な機能」を箇条書きで記載}

# ⚠️ 設計の必須条件（制約事項）
{ここに「既存アーキテクチャを壊さないためのルール」などを記載}

# 設計書格納先
- {{SESSION_DIR}}/refs/design/

# 📝 出力フォーマット
以下の構成で、実装フェーズで迷わずコードを書けるレベルの具体的なMarkdown設計書を出力してください。
1. **改修の目的と全体像（ユースケース）**
2. **影響範囲（対象ファイルとクラス）**
3. **データフロー / 主要ロジックの追加設計**
4. **想定されるエッジケースとその対策**
```

### Appendix B: 実装計画用プロンプト（plan.md）

`.agent/templates/cnew/plan.md` に配置します。設計書をもとに、安全に実装するための「手順書」をAIに作らせるテンプレートです。

実装計画用プロンプトの例

```
# 手順書作成指示

`{{SESSION_DIR}}/refs/design/` にある設計書を読み、以下のタスクを実行してください。

1. `{{SESSION_DIR}}/refs/plan/` 配下に `plan_タスク名.md` を作成してください。
2. 設計書を読み解き、実装を「独立してテスト・コミット可能な最小のステップ（Step 1-1, 1-2...）」に分割してチェックボックス付きで記載してください。
3. 各ステップには「実装内容」「動作確認コマンド」「期待される結果」を明記してください。
4. 計画を作成したら、一度作業を停止し、私にレビューを求めてください。
```

### Appendix C: 実装実行用プロンプト（exec.md）

`.agent/templates/cnew/exec.md` に配置します。作成した計画書（plan.md）に沿って、AIにコードを書かせ、テストさせ、コミットさせるための厳格な実行プロンプトです。

実装実行用プロンプトの例

```
# 実装実行指示

## 🎯 ミッション

2. **確認**: 対象環境でテストを実行する。
3. **記録**: テスト結果が問題なければ、`plan.md` の該当ステップのチェックボックスを `[x]` に更新する。
4. **コミット**: 日本語1行で簡潔なコミットメッセージを作成し、git commit を実行する。
   - コミットメッセージには `Agent-Session: {{SESSION_NAME}}` が自動付与されます。
```

### Appendix F: Git Hooksを用いたコミットとセッションの自動紐付け

複数のセッションを並行開発していると、どのコミットがどのタスクに属するのか分からなくなりがちです。  
各リポジトリの `.git/hooks/commit-msg` に以下のスクリプトを追記しておくと、手動コミットでもAIによる自動コミットでも、**コミットメッセージの末尾に `Agent-Session: <task-name>` が強制的に自動付与**されるようになります。

commit-msg フックのスクリプト

```
#!/bin/bash
# .git/hooks/commit-msg
COMMIT_MSG_FILE="$1"

# ========================================================
# Agent Dev Base: Agent-Session の自動付与
# ========================================================

# カレントリポジトリのルートを動的に取得
GIT_ROOT=$(git rev-parse --show-toplevel 2>/dev/null)
ACTIVE_SESSION_FILE="${GIT_ROOT}/.agent/active_session"

# active_session ファイルが存在する場合のみ処理
if [[ -f "$ACTIVE_SESSION_FILE" ]]; then
    SESSION_NAME=$(tr -d '[:space:]' < "$ACTIVE_SESSION_FILE")
    
    if [[ -n "$SESSION_NAME" ]]; then
        # すでに Agent-Session: の記載がない場合のみ、末尾に追記
        if ! grep -q "^Agent-Session:" "$COMMIT_MSG_FILE"; then
            echo "" >> "$COMMIT_MSG_FILE"
            echo "Agent-Session: $SESSION_NAME" >> "$COMMIT_MSG_FILE"
        fi
    fi
fi
```
