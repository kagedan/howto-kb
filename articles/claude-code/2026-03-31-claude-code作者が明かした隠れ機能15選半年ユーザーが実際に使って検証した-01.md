Claude Codeの生みの親、Boris Cherny氏が「過小評価されている機能15選」をXで公開しました。1.9万いいね、3万ブックマーク。開発者コミュニティが一斉に反応した形です。

僕はClaude Codeを半年以上使っていて、スキルも50個以上作ってきました。それでも「これ知らなかった」という機能がいくつもあった。今回はその中から、実際に試して特に実用的だった6つを紹介します。

## 1. モバイルアプリからClaude Codeを操作

判定：これは便利

ClaudeのモバイルアプリからClaude Codeのセッションに接続できます。つまり、電車の中からスマホでコードを書かせて、家に着いたらPCで結果を確認する、というワークフローが可能です。

実際に試してみると、長時間かかるリファクタリングを外出前に指示して、移動中にスマホで進捗を確認する使い方がかなり良かったです。

## 2. /batch でワークツリーエージェント

判定：上級者向けだけど強力

`/batch` コマンドで、Gitのワークツリーごとに独立したエージェントを走らせられます。メインブランチで開発しつつ、別のワークツリーでテスト修正を並行して進める、といったことが可能です。

自分の場合、3つの独立した機能を同時に実装するときに使いました。ブランチの競合を気にせず並列開発できるのは快適です。ただし、セットアップに慣れが必要で、初心者にはハードルが高いかもしれません。

## 3. teleport（セッション転送）

判定：地味だけど実用的

あるマシンのClaude Codeセッションを、別のマシンにそのまま転送できる機能です。会社のPCで作業していたセッションを、自宅のPCにそのまま持ってくる。コンテキストが全部引き継がれるので、「さっきの続き」がすぐできます。

リモートワークで場所を移動する人には地味に嬉しい機能です。

## 4. hooks の高度な使い方

判定：自動化好きなら必須

hooksは「特定のタイミングで自動的にスクリプトを実行する」機能です。Boris氏が紹介していたのは、ファイル保存時に自動フォーマット、コミット前に自動テスト実行、といった使い方。

僕は以前からhooksを活用していましたが、PostToolUseフックでコード変更後に自動的にリンターを走らせる設定は、コードの品質維持に効果的でした。settings.jsonに数行書くだけで設定できます。

## 5. loop（自律ループ）

判定：使い方次第で最強

Claude Codeに「テストが全部通るまで修正を繰り返して」と指示すると、自律的にコード修正→テスト実行→結果確認→再修正のループを回してくれます。

寝る前に「このテストスイートを全部グリーンにして」と指示して、朝起きたら全部通っていた、という体験は感動ものです。ただし、制限なくループさせると大量のトークンを消費するので、上限回数を設定するのがおすすめです。

## 6. /model でのリアルタイム切り替え

判定：毎日使う基本機能

タスクの難易度に応じて、やり取りの途中でモデルを切り替えられます。設計の相談はOpusで、実装はSonnetで、単純作業はHaikuで。

これを知らずにずっとSonnetだけ使っていた人は、切り替えるだけでトークン消費が半分以下になる可能性があります。

## 試してみて分かったこと

Boris氏が挙げた15機能のうち、日常的に使えるのは今回紹介した6つが中心です。残りは特定のユースケース向けだったり、まだ実験的な機能だったりします。

半年使ってきた身として言えるのは、Claude Codeの真価は「基本機能をどれだけ深く使いこなすか」にあるということ。新機能を追いかけるより、hooksやモデルルーティングといった基盤機能を自分のワークフローに組み込む方が、圧倒的に生産性が上がります。

まだ試していない機能があれば、ぜひ一つずつ触ってみてください。
