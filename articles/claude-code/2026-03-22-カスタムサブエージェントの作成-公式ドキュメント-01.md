## 概要

Claude Code公式ドキュメント。カスタムサブエージェントの作成・設定方法を解説。サブエージェントはカスタムシステムプロンプト、特定のツールアクセス、独立した権限を備えた独自のコンテキストウィンドウで実行される。

## サブエージェントの利点

- コンテキスト保持: 探索と実装をメイン会話から分離
- 制約の強制: 使用可能なツールを制限
- 設定の再利用: ユーザーレベルで全プロジェクト共有
- コスト制御: Haikuなど安価なモデルにタスクをルーティング

## 組み込みサブエージェント

| エージェント | モデル | 用途 |
|---|---|---|
| Explore | Haiku | コードベース検索・分析（読み取り専用） |
| Plan | 継承 | 計画のためのコードベース研究 |
| General-purpose | 継承 | 複雑なマルチステップタスク |

## 作成方法

### /agents コマンド（推奨）
インタラクティブに作成・管理。Claude生成も可能。

### ファイルベース
YAML frontmatter付きMarkdownで定義:

```markdown
---
name: code-reviewer
description: Reviews code for quality and best practices
tools: Read, Glob, Grep
model: sonnet
---

システムプロンプト（本体）
```

### スコープ

| 場所 | スコープ | 優先度 |
|---|---|---|
| `--agents` CLIフラグ | セッション | 1（最高） |
| `.claude/agents/` | プロジェクト | 2 |
| `~/.claude/agents/` | 全プロジェクト | 3 |
| プラグインのagents/ | プラグイン | 4（最低） |

## 主要設定フィールド

- `name`, `description`（必須）
- `tools` / `disallowedTools`: ツールの許可/拒否リスト
- `model`: sonnet, opus, haiku, inherit
- `permissionMode`: default, acceptEdits, dontAsk, bypassPermissions, plan
- `skills`: スキルコンテンツのプリロード
- `memory`: user, project, local（永続メモリスコープ）
- `hooks`: サブエージェント固有のライフサイクルフック
- `maxTurns`: 最大ターン数
- `isolation`: worktreeで分離実行

## 永続メモリ

`memory`フィールドで会話間で知識を蓄積:
- `user`: `~/.claude/agent-memory/<name>/`（全プロジェクト共有）
- `project`: `.claude/agent-memory/<name>/`（バージョン管理で共有）
- `local`: `.claude/agent-memory-local/<name>/`（ローカルのみ）

## 実行モード

- **フォアグラウンド**: 完了までブロック、権限プロンプトあり
- **バックグラウンド**: 並行実行、事前に権限を承認、Ctrl+Bで切替可能

## 活用パターン

- 大量出力の分離（テスト実行、ログ処理）
- 並行研究（複数モジュールの同時調査）
- サブエージェントのチェーン（レビュー→最適化の順序実行）
