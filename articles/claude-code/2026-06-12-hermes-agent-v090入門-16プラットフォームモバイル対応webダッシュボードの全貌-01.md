## Hermes Agentとは

Hermes Agentは[NousResearch](https://github.com/nousresearch/hermes-agent)が開発する「成長するエージェント」をコンセプトにしたオープンソースAIエージェントフレームワークです。2026年2月25日の初回リリース以来、GitHubスターが85,300以上に達し（2026年4月15日現在）、最も成長の速いOSSエージェントフレームワークのひとつとなっています。

Hermes Agentの特徴として、永続メモリ・スキルシステム・マルチプラットフォーム対応の3本柱があります。単一のセッションで完結する従来のチャットAIとは異なり、Hermesはユーザーの行動パターンを学習しながら複数の通信チャネルにまたがって動作します。

## v0.7.0〜v0.9.0 リリース履歴

2026年4月だけで3つのメジャーアップデートがリリースされています。

| バージョン | リリース日 | コードネーム | 主要機能 |
| --- | --- | --- | --- |
| v0.7.0 | 2026-04-03 | resilience | pluggableメモリプロバイダー・Camofoxブラウザ・ゲートウェイ強化（168 PR） |
| v0.8.0 | 2026-04-08 | intelligence | ライブモデル切り替え・Google AI Studio統合・MCP OAuth 2.1・無料MiMo v2 Pro（209 PR） |
| v0.9.0 | 2026-04-13 | everywhere | Webダッシュボード・Android/Termux・16プラットフォーム・Fast Mode（269 PR） |

### v0.7.0 "resilience" の概要

v0.7.0ではメモリシステムが**プラグイン式**に刷新されました。組み込みメモリに加え、6種類のサードパーティプロバイダーから選択でき、`hermes memory setup` コマンドで切り替え可能です。また、Camofox（アンチ検出ブラウザ）統合により、ウェブスクレイピングの検出回避が強化されています。

### v0.8.0 "intelligence" の概要

v0.8.0では**セッション中のライブモデル切り替え**が実現しました。たとえばClaude Sonnet 4.6で会話を始め、複雑な問題に直面した際にGPT-5.4へ切り替えるといった運用が可能になっています。また、Google AI Studio（Gemini）がネイティブ統合され、Nous Portal経由でMiMo v2 Proが無料で使えるようになりました。MCP OAuth 2.1対応により、セキュアなMCPサーバー接続も強化されています。

## v0.9.0 "everywhere" の主要新機能

### 1. ローカルWebダッシュボード

v0.9.0最大の新機能がローカルWebダッシュボードです。以下のコマンドで起動できます。

ブラウザで `http://127.0.0.1:9119` にアクセスすると管理画面が表示されます。

ダッシュボードの主要機能は以下の通りです。

**セッション管理**: 全セッションの一覧表示。各行にセッションタイトル・送信元プラットフォームのアイコン（CLI, Telegram, Discord, Slack, cron）・モデル名・メッセージ数・ツール呼び出し数・最終アクティブ時刻が表示されます。ライブセッションは点滅バッジで識別可能です。

**設定管理**: ダッシュボードから設定を変更すると、内部的に `~/.hermes/config.yaml` が更新されます。ターミナルの `hermes config set` コマンドと同じファイルを操作するため、どちらの方法で変更しても反映されます。設定変更は次のエージェントセッションまたはゲートウェイ再起動時に有効になります。

**スキル管理**: `~/.hermes/skills/` に配置されたスキルをカテゴリ別に一覧表示・検索・有効化/無効化できます。

### 2. Android（Termux）サポート

v0.9.0では**Android端末でのネイティブ動作**が公式にサポートされました。Termux経由でインストールします。

```
# Termux初期セットアップ
pkg update && pkg upgrade
pkg install nodejs-lts git python

# Hermes Agentインストール
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Termux環境向けの最適化として、以下が実装されています。

* **インストールパスの自動調整**: Termux固有のファイルシステム構造（`/data/data/com.termux/files/home`）に対応
* **TUI最適化**: モバイル画面サイズに合わせたテキストユーザーインターフェース
* **音声バックエンド対応**: `/voice` コマンドによる音声入力
* **/imageコマンド**: Termux上での画像生成もサポート

### 3. 16メッセージングプラットフォーム対応

Hermes Agent v0.9.0では以下の16プラットフォームがサポートされています。

| カテゴリ | プラットフォーム |
| --- | --- |
| Apple系 | BlueBubbles（iMessage） |
| 中国系 | WeChat, DingTalk（钉钉）, Feishu（飞书）, WeCom（企业微信） |
| 西洋系 | Telegram, Discord, Slack, WhatsApp, Signal, Matrix, Mattermost |
| 汎用 | Email, SMS, Home Assistant, Webhooks |

**iMessage連携（新機能）**: BlueBubblesプロジェクトを介したiMessage統合が実現しました。自動Webhook登録・セットアップウィザード統合・クラッシュ耐性が実装されています。

**WeChat連携（新機能）**: WeChatアカウントをHermes Agentと接続し、WeChatから直接エージェントを操作できます。

### 4. Fast Mode（低レイテンシ優先処理）

`/fast` コマンドを使用すると、OpenAIとAnthropicの対応モデルに対して優先キューへのルーティングが有効化され、大幅なレイテンシ低下が期待できます。

対応モデル（公式ドキュメント参照）:

* **OpenAI**: GPT-5.4, Codex
* **Anthropic**: Fast Tierに対応したClaudeモデル（詳細は[公式プロバイダードキュメント](https://hermes-agent.nousresearch.com/docs/integrations/providers)を参照）

```
# Fast Modeのオン/オフ切り替え
/fast
```

### 5. バックグラウンドプロセス監視

v0.9.0では `watch_patterns` 設定により、バックグラウンドプロセスの出力をリアルタイム監視できます。パターンに一致した際に通知を受け取るため、ポーリング不要です。

```
# config.yaml の設定例
watch_patterns:
  - "ERROR"
  - "WARN"
  - "Build complete"
```

### 6. バックアップ・リストア機能

設定・セッション・スキル・メモリを丸ごとバックアップできます。

```
# バックアップ作成
hermes backup

# 別マシンにリストア
hermes import hermes-backup-2026-04-15.tar.gz
```

マシン間の移行やメジャーアップデート前のスナップショット作成に活用できます。

## インストールとクイックスタート

### 標準インストール（Linux / macOS / WSL2）

```
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

### 開発者向け手動インストール

```
git clone --recurse-submodules https://github.com/NousResearch/hermes-agent.git
cd hermes-agent
uv venv venv --python 3.11
export VIRTUAL_ENV="$(pwd)/venv"   # activate不要（公式ドキュメント仕様）
uv pip install -e ".[all]"
```

### 初回起動

```
# APIキーを設定してエージェントを起動
hermes setup        # 対話式セットアップウィザード
hermes start        # エージェント起動
hermes dashboard    # Webダッシュボード起動（別ターミナル）
```

セットアップウィザードではOpenAI / Anthropic / Nous PortalのAPIキーを設定します。Hermes Agentは64,000トークン以上のコンテキストウィンドウを持つモデルが必要です。

## v0.9.0 で何が変わったか

| 機能 | v0.8.0以前 | v0.9.0 |
| --- | --- | --- |
| 管理インターフェース | ターミナル + config.yaml のみ | Webダッシュボード（127.0.0.1:9119）追加 |
| モバイル | 非対応 | Android/Termux公式サポート |
| iMessage | 非対応 | BlueBubbles経由で対応 |
| WeChat | 非対応 | 公式対応 |
| レイテンシ最適化 | なし | Fast Mode（/fast）追加 |
| データ移行 | 手動ファイルコピー | hermes backup / import |
| プラットフォーム数 | 13 | 16 |

## まとめ

Hermes Agent v0.9.0は「どこでも動くエージェント」への大きな一歩です。Webダッシュボードの登場でターミナル操作に不慣れなユーザーでも管理しやすくなり、Android/Termuxサポートとモバイルデバイスとの組み合わせで、AIエージェントの活用シーンが大幅に広がっています。

特にiMessageとWeChatの対応は、日常使いのメッセージングアプリからエージェントを操作できることを意味し、エンジニア以外のユーザーへの普及も期待できます。

Hermes Agentの次のリリースでは、v0.7.0の永続メモリ・v0.8.0の知能強化・v0.9.0のプラットフォーム拡張をベースに、さらなる自律性の向上が期待されます。
