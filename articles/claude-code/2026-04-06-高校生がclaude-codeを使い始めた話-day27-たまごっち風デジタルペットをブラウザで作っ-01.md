## 今日やったこと

ブラウザで動くデジタルペット育成ゲームを作りました。

指示はこれだけです。

```
ブラウザで動くデジタルペットゲームをHTMLで作ってください。キャラクターに名前をつけて育てる。毎日話しかけるとご機嫌が上がる、放置するとお腹が空いたり眠くなる。空腹・元気・親密度のステータスがある。
```

その後、機能を追加し続けて今の形になりました。

## できたもの

![](https://assets.st-note.com/img/1775436125-q6Enf32LlF9ompUCIvJSM5sh.png?width=1200)

実際の画面

**ステータス（4種）**

* 空腹・元気・清潔・親密度（各0〜100）
* 放置するとどんどん下がる
* 清潔度が低いまま放置すると病気になる

**アクション**

**ミニゲーム**

* もぐら叩き（15秒・コイン獲得）
* じゃんけん3本勝負（コイン獲得）

**ショップ・カスタマイズ**

* ゴッチコインで食べ物・アクセサリー・家具を購入
* 帽子・めがね・服の着せ替え
* 部屋の背景・家具を変更

**成長システム**

* たまご→こども→おとな→伝説の4段階進化
* 進化には日数・親密度・全ステータスの複合条件が必要
* たまご→こども：3日＋親密度20以上
* こども→おとな：7日＋親密度60以上＋平均ステータス60以上
* おとな→伝説：30日＋親密度90以上＋全ステータス平均80以上＋ミニゲーム10回勝利

## 今回気づいたこと

「伝説」への進化条件を最初は親密度だけにしていました。それだと簡単に達成できてしまうので、日数・ステータス・ミニゲームの複合条件に変更しました。

ゲームバランスを考えながら指示を出す経験が、Deck Rushを作った4日間で身についていたと思います。「難しすぎず、簡単すぎず」をどう設定するかが、ゲーム作りで一番難しい部分だと気づきました。

## 遊べます

ブラウザで開くだけで遊べます。データはブラウザに保存されるので、閉じても続きから遊べます。
