## **はじめに**

**2026年4月21日午後（米国時間）**、Anthropicが[`claude.com/pricing`](http://claude.com/pricing)を**告知なく**書き換え、Claude Codeを$20 Proプランから外して$100 / $200のMaxプラン専用にしました。

数時間でSNSで広く拡散され、Anthropicが**ページを元に戻すという結果になりました。**（これも告知なし）

Head of Growthのツイートでは「**新規プロシューマー登録の約2%を対象としたテスト**」であり、「既存Pro/Maxユーザーには影響なし」と説明されています。

つまり、**現時点（2026年4月22日）では**、Pro $20プランでもClaude Codeは引き続き使えます！  
**ただしいつまた同じことが起きてもおかしくない**ので、自分のユースケースに合うプランを意識的に選んでおくべきです。

そこで本記事はClaude Codeが20$プランから消された事件の整理と、ユースケースに合わせてどのプランを選ぶべきかを解説します。

## **何が起きたのか — タイムライン**

### **4月21日 午後（米国時間）**

Anthropicが[`claude.com/pricing`](http://claude.com/pricing)を**サイレントで更新**。  
Pro列からClaude Codeのチェックマークを削除し、Maxプラン専用扱いに変更。

### **数時間後**

* Reddit、Hacker News、Xで一気に拡散。
* 「個人開発者の生命線が消える」「$20 → $100 は5倍値上げに等しい」といった反応。
* 多くの開発者が料金ページのスクリーンショットを共有。

### **Head of Growth がツイートで補足**

Anthropic Head of Growthの[Amol Avasare氏が以下を投稿](https://twitter.com/TheAmolAvasare/status/2046724659039932830)。

> "For clarity, we're running a small test on ~2% of new prosumer signups. Existing Pro and Max subscribers aren't affected." （明確にしておくと、新規プロシューマー登録の約2%を対象とした小規模テストです。既存のPro / Maxユーザーには影響しません）

ただしSimon Willison氏は記事内で「自分の周りはみんな新しいpricing gridを見ているし、Internet Archiveも更新前後を捕捉している。本当に2%なのか怪しい」と疑問を呈しています。

### **さらに数時間後 — 撤回**

[Simon Willison氏が記事](https://simonwillison.net/2026/Apr/22/claude-code-confusion/)を執筆中に、[`claude.com/pricing`](http://claude.com/pricing)が**元に戻る**。Pro列のClaude Codeチェックマークも復活。

> "In the time I was typing this blog entry Anthropic appear to have reversed course" （このブログを書いている最中に、Anthropicは方針転換したようだ）

> — Simon Willison "Is Claude Code going to cost $100/month? Probably not—it's all very confusing" (2026-04-22)

ここでも**撤回についての公式コミュニケーションはなし**。

### **競合の反応**

OpenAIのCodexエンジニアリングリード[Thibault Sottiaux氏が即座に反応](https://twitter.com/thsottiaux/status/2046740759056162816)。

> "I don't know what they are doing over there, but Codex will continue to be available both in the FREE and PLUS ($20) plans. (中略) Transparency and trust are two principles we will not break"

Codexは無料プラン・$20 Plusプラン共に維持を明言。Claude Cowork（実質的にはClaude Codeのリブランド版）はこの騒動の間も$20 Proで使えていたという皮肉も指摘されています。

## **二度あることは三度ある**

「結局元に戻ったなら良いじゃん」で済む話ではないと考えています。

**A/Bテストの対象になったという事実そのものがシグナル**です。Anthropic社内には少なくとも一度「Claude CodeをPro $20から外す」という案を実装まで持っていく判断ラインが存在した、ということが考えられます。  
テストの結果次第（あるいは経営判断次第）で**いつでも再現する可能性**があるかもしれません。

Simon Willison氏の懸念をそのまま借りると

* $20と$100の差は「特に高所得国以外」では大きい。
* Claude Code向けの教材・チュートリアルを作る側からすると、「最低$100からのツール」で教えるのは現実的ではない。
* 「いつ5倍になるかわからない」という不確実性自体が、ツール選定の信頼性を毀損する。

つまり「撤回されたから安全」とは言えません。一度、改定のテストまで踏み込んだ以上、**Anthropicが再び同じ判断を下す可能性は十分にあります**。その前提でプラン選びを再評価する価値があります。

## **現行プラン比較（2026年4月22日時点）**

[`claude.com/pricing`](http://claude.com/pricing)からの情報を整理しました。

### **個人向け**

| プラン | 料金 | Claude Code | 主な特徴 |
| --- | --- | --- | --- |
| **Free** | $0 | ❌ | Web/iOS/Android/Desktopのチャット  画像分析  Web検索など基本機能のみ |
| **Pro** | $17/月（年払い・$200一括）  $20/月（月払い） | ✅ | Claude Code  Claude Cowork  Excel/PowerPoint/Word連携  Research  無制限プロジェクト |
| **Max 5x** | $100/月〜 | ✅ | Pro比 **5倍**のusage  Higher output limits  優先アクセス  新機能の早期アクセス |
| **Max 20x** | $200/月〜 | ✅ | Pro比 **20倍**のusage  その他はMax 5xと同じ |

### **チーム / 組織向け**

| プラン | 料金 | Claude Code | 主な特徴 |
| --- | --- | --- | --- |
| **Team Standard** | $20/seat/月（年払い）  $25/seat/月（月払い） | ❌ | Pro以上のusage  5〜150seats  SSO  中央課金  コネクタ管理 |
| **Team Premium** | $100/seat/月（年払い）  $125/seat/月（月払い） | ✅ | Standard比 5倍のusage  Claude Code  Claude Cowork含む |
| **Enterprise** | $20/seat + API従量 | ✅ | SAML SSO  SCIM  監査ログ  HIPAA対応  ロールベースアクセス  IPホワイトリスト |
| **Education** | 大学向け割引 | — | 大学全体向けプラン  API credit付き |

### **Claude Codeのusageの仕組み（重要）**

* **Pro/Maxのusage limitは Claude.ai（Web/Mobile/Desktop）と Claude Code で共有。**
* つまりWebチャットを大量に使った日にClaude Codeを動かすと、両方の枠を同時に消費。
* 上限到達時の選択肢は以下の通り。
  1. リセットを待つ（rolling window）。
  2. **Extra usage**（pay-as-you-go）を有効化 → API従量課金。
  3. プランをアップグレード（Pro → Max 5x → Max 20x）。
  4. Claude Console（API）に切り替え。

### **API従量課金（参考）**

`ANTHROPIC_API_KEY`環境変数経由でClaude Codeを動かす場合の料金

| モデル | Input | Output | Prompt cache (write/read) |
| --- | --- | --- | --- |
| **Opus 4.7** | $5 / MTok | $25 / MTok | $6.25 / $0.50 |
| **Sonnet 4.6** | $3 / MTok | $15 / MTok | $3.75 / $0.30 |
| **Haiku 4.5** | $1 / MTok | $5 / MTok | $1.25 / $0.10 |

Batch processingで全モデル50%割引。

## **ユースケース別のおすすめプラン**

ここからは私見も入りますが、上記情報をベースに「自分はどれを選ぶべきか」を整理します。

### **ケースA: Claude Codeを「ちょっと触ってみたい」個人**

→ **Pro（$17〜$20/月）一択**

* Free にClaude Codeは含まれない（これは騒動前から）。
* 月数時間〜十数時間程度のコーディングなら、Proのusageで十分足ります。
* 年払いだと月$17で2ヶ月分相当が安くなる。

ただし\*\*「Pro = 永久にClaude Code付き」\*\*と仮定しないこと。今回の件で永続できるかが怪しくなりました。

### **ケースB: 業務でメインIDEとして毎日使うエンジニア**

→ **Max 5x（$100/月）が最もコスパが良いライン**

私自身の使い方を仮に当てはめると、複数リポジトリを並列で動かし、長時間セッションを回すような使い方では、Proのusageは平日のうちに枯渇します。

* Max 5x = Pro比5倍のusage。
* 月$100は確かに高いが、**1日の生産性が時間単位で測れる職種なら半日分の業務インパクトで回収可能。**
* 「Pro + Extra usage（API従量）」と比べた損益分岐は、Claude Codeを毎日3〜4時間以上ガッツリ使うかどうかが目安。

### **ケースC: Opusを多用する重い案件 / 並列worktree運用**

→ **Max 20x（$200/月）か、API従量**

* Opusは1ターンあたりSonnetの数倍トークンを消費するので、Max 5xでも消費量が大きい。
* `git worktree` + 複数Claude Codeセッション並行運用しているなら、Max 20xが安定。
* またはMax 5xを契約しつつ、ピーク時だけ`/login`でAPI keyに切り替える運用も可能。

### **ケースD: チーム開発（5人以上）**

→ **Team Premium seat（$100〜$125/seat/月）**

* Team Standardには**Claude Codeが含まれない**点に注意。
* メンバーごとに個別にProを契約させるよりも、課金・SSO・コネクタ管理を一元化できる。
* メンバー全員がClaude Code必須でなければ、**Standard と Premium をmix and match**できる（[料金プラン参考](https://claude.com/pricing)）。

### **ケースE: スポット利用 / 「サブスクは嫌」**

→ **API従量 + Claude Code**

* `claude login`でConsole API key認証。
* Sonnet 4.6で1セッション$1〜数ドル程度。
* ただし長時間セッションでcontext無圧縮のまま走らせると、あっという間に十数ドル課金。`/cost`コマンドで都度確認推奨。

### **ケースF: 報道機関・教育・予算固定組織**

→ **Educationプラン or Enterprise**

* 「予算が固定で、突然5倍値上げされたら回らない」組織は、契約として安定する形を選ぶべき。
* 今回の騒動の教訓は、**消費者向けプランは経営判断で変更され得る**ということ。

## **リスクヘッジ — 同じことが起きた時のために**

「次に5倍値上げが来ても困らない」状態を作っておく実践的な選択肢。

### **認証経路を分ける運用に慣れておく**

Claude Codeは認証によって課金経路が変わります。

* **サブスクリプション認証**: Pro/Max枠を消費する。
* **`ANTHROPIC_API_KEY`環境変数**: API従量課金が優先される（重要）。

`claude logout` → `claude login`で切り替え可能。プラン変更があっても、API keyフォールバックを準備しておけば作業の手が止まることはありません。

### **代替ツールに触れておく**

完全な代替にはならないにせよ、特性を知っておく価値はあります。

* **OpenAI Codex**: Free / $20 Plusでも利用可（4月22日時点でCodexエンジニアリングリードが明言）
* **Cursor**: $20/月、独自のagentic機能 + Claude/GPT等を選択可。
* **GitHub Copilot Workspace**: 別の方向性のagent。
* **Cline / Roo Code**: VSCode拡張、API key持ち込み型。

「Claude Codeしか知らない」状態を脱しておくと、騒動が再発した時に冷静に判断できます。

### **ワークフローを「ツール非依存」に保つ**

* CLAUDE.mdをAGENTS.mdに置き換えたり、プロンプト規約を整備し、別のagentでも動くようにしておく。
* `git worktree`でセッションを分割する運用は、どのagentでも応用可能。
* pre-commit hookやCIによる自動レビューは、agentに依存しない品質ゲート。

ベンダーロックインを意識的に薄める設計にしておくのが、今回の事件から得られる現実的な学びだと思います。

## **まとめ**

* **2026年4月21日の「Claude Code Proから削除」騒動は、テストとして実施され、数時間で撤回された。**
* **2026年4月22日現在、Pro $20でClaude Codeは引き続き使用可能**（既存ユーザーは元々影響なし）。
* ただしAnthropic側に **「やろうとした」実績が残った**ので、プラン選びは「いつ変わってもいい」前提で考えて準備しておく姿勢が大切。
* ユースケース別の現実的な落としどころ
  + 軽い個人利用: **Pro**
  + 業務メイン: **Max 5x**
  + 重い案件・並列運用: **Max 20x or API従量**
  + チーム: **Team Premium seat**
* リスクヘッジ: API key認証への切替手順を確認、代替ツールにも触れておくことで、ワークフローをツール非依存にする準備を整える。
