Claude Code を使い始めようとすると、たぶん最初につまずくのが「で、これどこから動かすの？」です。入り口がいくつもあって、正直、私も最初はどれを選べばいいのか分かりませんでした。

前回の最後に「次回はどこから使うかを整理する」と書いたので、今日はそこを片づけます。最初に言っておくと、入り口を全部覚える必要はありません。選ぶのは1つでいいんです。

> 💬 Claude Code は、どの入り口を使うにしても有料プランの契約が前提です。無料では使えません。料金とプランの選び方は、次回くわしく扱います。

## 入り口は複数あるが、エンジンは1つ

最初に1つだけ、知っておくと気が楽になることを。

Claude Code には、呼び出すための画面がいくつもあります。これが入り口です。でも、その奥で動く本体は1つだけ。だから、どの入り口から入っても、設定や、Claude への自分専用の指示書である `CLAUDE.md` は、そのまま共通で効きます。

つまり、軽く試してみて、気に入ったら本格的な場所にちゃんと移ればいい。そのとき、お試しで書いた設定はそのまま持っていけます。作業場所なんて毎日コロコロ変えるものでもないので、焦って決めなくて大丈夫です。

このあと名前がいくつか出てきますが、「まず触ってみる用」と「腰を据えて毎日使う用」の2つに分けて見れば足ります。

## まず試す入り口 ― インストールするだけ

「Claude Code ってこういう感じか」を、できるだけ少ない手間でつかむための入り口です。

ひとつめは**デスクトップアプリ**。Mac と Windows で使えます。Linux は対象外です。入れたらすぐ動く手軽さがいいところで、面倒な事前準備が要りません。公式も「ターミナルは不要」と言い切っています。ターミナルというのは、文字で命令を打つ画面のこと。あの黒い画面を触らずに済むわけです。Claude が「ここをこう変えます」と示す変更前後の差分を、目で見ながら進められるのも安心です。

もうひとつが**Web版**。何もインストールせず、ブラウザや iPhone のアプリから動かせます。ただし使うには GitHub アカウントが要ります。GitHub はプログラムを保存・共有するサービスで、このアカウント作りが地味に最初のひと手間。初めての人にはちょっとだけ重いです。すでに GitHub を使っている人なら、こっちが手軽。なお Web版は、まだお試し公開の段階です。

最初の一歩としては、私はデスクトップアプリを勧めます。アカウント連携みたいな準備をすっ飛ばして、入れたら触れるからです。

## 腰を据えて使う場所 ― 本格的に使うなら

ここからは、毎日の作業を実際に進める「本拠地」になる入り口です。頻繁に乗り換えるものではないので、自分に合うものをじっくり選んでいい場所です。

ひとつは**エディタの中で使う**方法。私のおすすめは VS Code です。無料のプログラム作成アプリで、後から機能を足す「拡張」を入れるだけで使えます。公式もこの方法を推していますし、私自身もこれです。利用者が多いぶん情報も多いので、詰まったときに解決の手がかりを見つけやすい。

もうひとつが**ターミナル**。例の、文字で命令を打つ画面から使う方法です。全OSで動いて、機能はいちばん多い。ただ初心者には、そもそもターミナルを触ること自体が最初の壁になります。機能が多い＝最初に選ぶべき、ではありません。慣れてからでも遅くないです。

> Slack 連携や GitHub Actions のような、チームで使ったり作業を自動化したりする入り口もあります。ただ一人で始めるうちは要りません。「そういうのもあるんだ」くらいで十分です。

## で、結局どれから？

向き・不向きを表にしておきます。

| 入り口 | 準備 | 向いている人 |
| --- | --- | --- |
| デスクトップアプリ | インストールのみ | 準備の手間なく、まず触りたい |
| Web版 | GitHubアカウント | すでに GitHub を使っている |
| エディタ（VS Code） | 拡張を入れる | 本格的に毎日使っていきたい |
| ターミナル | 必要 | 慣れている／いちばん強力に使いたい |

初心者なら、まずデスクトップアプリで軽く触ってみるのがいいと思います。準備がほぼ要らないので、合わなかったときの「やってしまった感」もありません。それで「これ、続けられそうだな」と思えたら、VS Code に移ればいい。先に感触をつかんでおくと、本拠地を選ぶときに外しにくくなります。

あれこれ考えるより、自分の手元で一回動かすのがいちばん早い。ここは前回から変わりません。

## 私の場合 ― 拡張からターミナルへ移った

最後に自分の話を少し。

私は最初、VS Code の拡張から入りました。普段の作業場所が VS Code だったので、そこから離れずに使えるのが良かったんです。

でも今は、同じ VS Code の中のターミナルを使っています。拡張を使っていた頃、サブエージェント、つまり Claude Code が裏で呼び出す補助の作業役を動かすと、応答をうまく受け取れずに画面が固まることがありました。当時はまだ拡張側の作り込みが追いついていなかったんだと思います。それでターミナルに切り替えました。

念のため書いておくと、これは「拡張はダメ」という話ではありません。私が使っていた頃の話なので、今は違うかもしれません。

結局、作業場所である VS Code そのものは変えませんでした。変えたのは中の使い方だけ。両方さわって、しっくりくる方に落ち着きました。

## まとめと次回

* 入り口は複数あっても、奥で動くエンジンは同じ。だから設定や `CLAUDE.md` はそのまま使い回せる。
* まずデスクトップアプリ。インストールだけで初日から触れる。
* 続けそうなら VS Code に移す。毎日の作業の本拠地はここ。
* ターミナルと Web版は後回しでいい。慣れてから、GitHub に馴染んでからで十分。

入り口が決まると、次に気になるのはやっぱり「結局いくらかかるの？」だと思います。次回は、料金プランの選び方を、損をしにくい目線で整理します。
