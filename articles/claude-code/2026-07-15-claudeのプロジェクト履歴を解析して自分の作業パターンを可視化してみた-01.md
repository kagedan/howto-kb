## はじめに

Xでこんなポストが流れてきた：

> "THERE'S A FOLDER ON YOUR MACHINE THAT KNOWS YOU BETTER THAN YOUR THERAPIST. CLAUDE JUST LEARNED TO READ IT."  
> — @0xAI42exe

「Claudeがあなたの全セッション履歴を読んで、時間の無駄や委任すべきタスクを特定する」という触れ込み。6フェーズのプロセスで、あなたのClaudeの使い方を自己分析するという。

実際にどれほど有用なのか、検証してみた。

## `~/.claude/projects` の正体

Claude Code（CLI版Claude）は、プロジェクトごとにセッションログを保存している。場所は `~/.claude/projects/`。

ディレクトリ名は絶対パスをハイフン区切りに変換したもの。中身はこんな構成：

```
-プロジェクト名/
  ├── UUID.jsonl   ← セッションログ
  └── memory/       ← プロジェクト記憶
```

各セッションは **JSONL**（1行1JSON）形式。1ファイルに全イベントが時系列で詰まっている。

### JSONLの中身

```
{"type":"user","message":{...},"timestamp":"...","entrypoint":"cli"}
{"type":"assistant","message":{"content":[{"type":"text","text":"..."},{"type":"tool_use","name":"Bash","input":{...}}]},"timestamp":"..."}
{"type":"tool_result","message":{...},"timestamp":"..."}
```

イベントタイプは主に：`user`（人間の入力）、`assistant`（Claudeの応答）、`tool_result`（ツール実行結果）、`mode`（モード変更）、`file-history-snapshot`（ファイルスナップショット）など。

## 解析スクリプトを作る

68セッション、6プロジェクト、総ファイルサイズ56MB。これを手で見るのは非現実的なので、解析スクリプトを書いた（約200行のPython）。

抽出した情報：

* セッションID, 開始/終了時刻, エントリーポイント
* ユーザー/アシスタントメッセージ数
* ツールコールの種類と頻度
* リクエスト数, 推定トークン量
* ファイルサイズ

### 無駄パターン検出

解析ロジックで「無駄っぽい」セッションを自動検出した：

**1. Read呼び出し過多（>20回）: 3セッション**  
最大209回のRead。ファイルを何度も読み直しているパターン。

**2. Bash大量実行（>50回）: 3セッション**  
最大1,709回のBash実行。コマンド実行ループに陥っている可能性。

**3. ユーザー往復過多（>10回）: 15セッション**  
最大3,447往復。人間が細かく指示を出し続けているパターン。

**4. AskUserQuestion使用: 3セッション**  
判断を人間に委ねているパターン。

## 委任提案

解析結果をLLMに渡して、自動委任可能なタスクを提案させた：

### 1️⃣ 定期的なCode Reviewの委任

Bash 59.5% + Edit 10.8% + Read 10.6% = 81%がコード操作系。変更差分のレビューをClaudeに常時委任することで、ユーザー往復の30%削減が見込める。

### 2️⃣ ファイル発見タスクの自動化

Readコール508回の多くが「ファイルを探す」行為。`find`/`grep`ラッパーを定形化してAgentタスクにすれば、Readコール50%削減も可能。

### 3️⃣ Web検索+情報収集の自動委任

WebFetch 71回 + WebSearch 52回。情報収集系は即座にサブエージェント委任すべき。

### 4️⃣ 定形コマンド実行のキャッシュ

Bash 2,840回のうち少なくとも30%は同一コマンドの再実行。結果キャッシュまたはスキル化でBashコール20%削減。

### 5️⃣ AskUserQuestionの自動判断

68回の質問のうち「どっちがいい？」系が多い。過去の判断パターンを記憶し、自動判断させれば応答待ちが解消できる。

## 検証の限界

**実際に「6フェーズ」の実装までは再現できなかった。** 元ポストの主張には誇張が含まれている：

* ❌ 「every session you have ever run」→ 実際はプロジェクトごとに分かれた約半年分
* ❌ 「セラピストよりあなたを知ってる」→ 誇張表現
* ❌ 全セッションをLLMに丸投げして意味解析 → トークン消費が大きすぎて非現実的

**わかるのは「使ったツールの傾向」や「時間の使い方の大雑把なパターン」であって、深い心理や意図までは読み取れない。**

## 結論

### 実用的な価値：ある

「自分のClaude使用パターンを客観的に見る」という行為自体は非常に有用。特に：

1. **Bashが59.5%** という事実は、コード生成AIが「書くより実行している」現状を浮き彫りにする
2. **Read過多のセッション**は、ファイル探索の効率化余地を示す
3. **ユーザー往復過多**は、バッチ処理への移行余地を示す

### おすすめの使い方

1. この記事のスクリプトをベースに、自分の `~/.claude/projects` を解析する
2. 「Bashばかり使ってる」「Readが多い」といった傾向を見つける
3. その傾向に合わせて、スキルやエイリアスを整備する
4. 3ヶ月後に再測定して改善を確認する

**「データドリブンな自己改善」の入り口としては十分価値がある。** ただし、ポストの主張をそのまま信じて「Claudeに全履歴を読ませる」のは現実的ではない（トークン消費が大きすぎる）。

まずはツール使用パターンという「外側のデータ」から自分の傾向を知り、それをもとに改善していくのが現実解だ。
