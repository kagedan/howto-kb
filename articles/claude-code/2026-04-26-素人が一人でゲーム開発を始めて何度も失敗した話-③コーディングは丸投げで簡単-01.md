これの続きです

前回、お気に入りの手順欄とお別れして
<font color = "orange">『チャットが長くなってもダメ』</font>
ということを知った私はAIに新しい方法を教えてもらいます

<font color = "orange">『引継ぎ文書を作ってもらって次のチャットに移動』</font>という運用です

<font color = "cyan">「ええー、けど情報が少なくなるんじゃないの？」
「その、これまでのやり取りとかがなくなるからニュアンス的なアレが抜けるんじゃないの？」
「今まで一緒に頑張ってきたのに少し寂しい」</font>
と思いました
<font color = "orange">「むしろ長いと精度が落ちます、要約したほうがいいですよ」</font>
とAI
<font color = "cyan">「そういうもの？」</font>と思いつつすごい長さとなっていた各チャットのお引越しを開始しました

これがここから、長く付き合っていく相棒のような運用となりました

使用量問題は見事に解決しました、手順欄と長くなったチャットが犯人でした

.

すぐには使用量が減らなくなりゲームの開発は進んでいきました
このころは<font color = "orange">「あなたの規模なら、claude codeを使う必要はない」</font>といわれ
<font color = "cyan">「そういうものなのかー」</font>とチャットで指示を貰い開発をしていました

このころは毎日4時間ほどこのプロジェクトの作業とC#とUNITYの勉強していました
（作業時間は『慣れてくるもの』なのかこの先どんどん増えていきます）

そして何も考えずに
<font color = "cyan">「こういう風にしたい」</font>と丸投げし
<font color = "cyan">「その機能は後からでいいや」</font>と先送りにして
<font color = "cyan">「ゲームデータも後で考えるから、仮のをつくって」</font>と依頼
『とりあえず動く』ものを優先して作りました

本職はグラフィックなのでゲーム内のイラストやUIなどは先に完成していました

こうしていくつもの手順をすっぽかして
<strong>アルファ版が完成しました！</strong>

スクリプトの数は8個！中身はたくさんのメソッドが詰まっています！

アルファ版で遊びながら、私は勘違いしました
<font color = "cyan">「こんなに早くAIのコーディングって終わるんだ、すごい！」
「それなら、意外と早く演出とか完成度上げに入れるかもしれないぞ」
「思ったより早く完成しそう、グラフィックにもっと時間を割いたほうがいいかな」</font>

この時は、
<font color = "red">ぐちゃぐちゃなアルファ版をこのまま完成させることは難しいこと
ゲームデータ設計が大変だということ
私が考えたゲームは思ったより複雑だということ</font>
そんなことは知らずに
<font color = "cyan">「すぐにテストプレイとバランス調整に入れるかも」</font>と
うきうきでした

<font color = "cyan">「あとは本データを作るだけだ！」</font>

ここから1か月弱
見切り発車したゲームデータの設計を行うこととなります
