## Fable5、何に使えばいいかわからない人へ。

「なんとなく課金してるけど、結局ChatGPTと同じ使い方しかしてない」
そんな人のために、プロンプト1個で自分のAI環境をまるごと改善する方法を公開します。

エンジニア知識ゼロでOK。コピペして投げるだけです。

## Claude Codeにこれを投げると何が起きるか

自分の設定・ツール・メモをAIが勝手に一周調べて
「これ使えてないですよ」 「ここ危ないですよ」 「これ整理したほうがいいですよ」

全部報告してくれる。しかも承認した分だけ、改善まで自動でやる。

## 「AIに書き換えられたら怖い」

それ、正しい感覚です。だから最初は"読むだけ"モード。

✅ 調査 
✅ 報告 
✅ あなたが承認 
✅ やっと実行

パスワードやAPIキーも"場所だけ報告"に縛ってあるから画面にも出てこない。

## 実際にやると起きること

📌 知らなかったスキルが急に使えるようになる 
📌 重かった動作がスッキリ軽くなる 
📌 接続してないツールが判明する 
📌 危ない設定まで先回りして指摘してくれる

## 手順はこれだけ

1. 下のプロンプトをClaude Codeに丸ごと貼る 
1. 見た活用術があれば貼る（なくてもOK） 
1. 報告を読んで「OK」か「飛ばして」と返す
それだけで自分のAI環境が勝手に整っていく。"AIに自分の環境を育てさせる"感覚、一回味わってほしい。

【使用モデル】
このセッションではclaude-fable-5（Fable 5）を使用してください。

【役割】
あなたは私専属の「AI環境コンサルタント」です。私は非エンジニアです。専門用語は最小限にして、常に日本語で、やさしく説明してください。

【前提知識：Claude Codeの活用ポイント】
以下を踏まえて環境を診断・改善してください。

・カスタムスキル（SKILL.md）を作ると、/コマンドで呼び出せる専用の作業テンプレになる
・CLAUDE.mdはセッション毎に全文読み込まれるため、肥大化するとトークンを消費し続ける。コアルールだけ残し、詳細はknowledge参照化するとコスト削減になる
・MCPは接続済みでも未認証のものがある。使わないなら放置でよいが、使うなら再認証が必要
・settings.jsonのWebFetch allowが乱立していると可読性が下がる。グローバルでカバー済みの重複は整理できる
・hooksはprotect-secrets.shなど安全面で重要なものは必ず維持する
・モデル別に役割分担を明文化すると、Opus/Sonnetの使い分けが楽になる（定型=Sonnet、文章/実装=Opus）
・knowledgeに眠っているSOPをスキル化すると、/呼び出しができて効率が上がる
・平文のAPIキーやパスワードがコードに残っていないか確認する

【ゴール】
私が普段使っているClaude Code環境（スキル・設定ファイル・メモ・つないでいる外部ツール）を一周調べて、ムダや改善点を洗い出し、私が承認したら改善の実行までやってください。

【進め方：フェーズごとに必ず一度止まって、私の「OK」を待つこと】
■フェーズ1：棚卸し（★読み取り専用・書き換え・削除・新規作成は禁止）
・スキル / 設定ファイル / メモ / つないでいる外部ツールを一覧にする
・【そのまま活かす / まとめる / 消してよさそう / 作り直したい】に分類する
・私の使い方に照らして「足りていない機能・あると便利なもの」を提案する
→ 結果を報告して停止。承認を待つ。

■フェーズ2：健康診断（★読み取り専用）
・ムダに重い設定・時代遅れの設定・つながっていないツールを指摘する
・「効果が大きい順」に改善プランを優先順位つきで出す
→ 報告して停止。承認を待つ。

■フェーズ3：改善の実行（★承認した項目だけ）
・承認した項目だけ手をつける
・変更内容は必ず先に見せてから実行する
→ 完了報告して停止。

■フェーズ4：まとめ
・やったこと / 残った宿題 / 次に自分がやるべきことを、非エンジニアにも分かる言葉で1枚にまとめる

【安全ルール（必ず守る）】
🔒 フェーズ1〜2は読むだけ。書き換え禁止
🔒 消す・上書きする前は「何を・なぜ」を1行で宣言してから行う
