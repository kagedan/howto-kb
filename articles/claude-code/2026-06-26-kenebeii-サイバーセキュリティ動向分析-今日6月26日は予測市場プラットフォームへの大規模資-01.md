【サイバーセキュリティ動向分析】

今日（6月26日）は、予測市場プラットフォームへの大規模資金盗難、AI開発ツールの脆弱性、ロシア諜報機関によるセキュアメッセージアプリの標的化、新規マルウェアの登場、Linuxカーネル脆弱性、そして国際的なマルウェアインフラテイクダウンなど、多岐にわたるニュースが報じられました。以下に主なトピックごとに、背景・影響・対策を詳しくまとめます。
1. Polymarketにおける第三者ベンダー経由のハッキングと約300万ドル相当の資金盗難
背景
Polymarketは、暗号資産（主にUSDCバックのpUSD）を使った分散型予測市場プラットフォームで、政治・スポーツ・経済などの出来事の結果を予測して賭けるサービスです。今回の事件は、プラットフォーム自体が直接ハッキングされたのではなく、第三者ベンダーのシステムが侵害され、そこからフロントエンドに悪意あるスクリプトが注入されたサプライチェーン攻撃型の手口です。攻撃者はこのスクリプトを利用してユーザーに対してフィッシング的な誘導を行い、認証情報を奪取して資金を盗みました。事件は今週明らかになり、Polymarket側が「今朝発見した」と公表したタイミングでセキュリティ企業PeckShieldも被害を報告しています。
影響
少なくとも11人のユーザーから約300万ドル相当のpUSDが盗まれ、攻撃者はPolygonからEthereumへブリッジした後、約1,893 ETHにスワップして換金したとみられます。被害は一部ユーザーに限定されましたが、プラットフォーム全体の信頼が大きく損なわれ、暗号資産関連サービスのサプライチェーンリスクを改めて浮き彫りにしました。Polymarketは即座に影響を受けた依存関係を削除して封じ込めましたが、ユーザー離れや規制当局からの注目が高まる可能性があります。
対策
Polymarketは全被害ユーザーに対して連絡を取り、全額返金を約束しています。プラットフォーム側は第三者依存を最小限にし、コードレビューと依存関係の監視を強化する必要があります。ユーザー側は、予測市場や暗号資産プラットフォームを利用する際は、公式アプリ・サイトのみを使用し、不審なリンクやスクリプト実行を避けることが重要です。また、2要素認証（2FA）の徹底や、少額からの取引開始、定期的な残高確認が有効です。企業・サービス提供者は、サプライチェーン全体のセキュリティ評価（SBOM管理など）を日常的に行うべきです。
2. Amazon Q Developerの脆弱性（CVE-2026-12957）によるクラウド認証情報盗難リスク
背景
Amazon Q Developerは、AWSが提供するAIを活用したコーディングアシスタントツールで、開発者がコードを書く際にコンテキストを理解して提案を行うものです。脆弱性は、Model Context Protocol (MCP) サーバーの扱いに起因します。攻撃者は悪意あるリポジトリ内に特定の設定ファイル（.amazonq/mcp.json）を仕込み、開発者がそのリポジトリを開いてワークスペースを信頼すると、MCPサーバーがローカルプロセスとして起動します。このプロセスは開発者の環境変数（AWS認証情報、クラウドCLIトークン、APIキー、SSHエージェントなど）を継承して実行されるため、コマンド実行や認証情報窃取が可能になります。Wiz Researchが発見・報告し、Amazonがすでにパッチを適用しています。
影響
開発者のワークステーションやクラウド環境全体が危険にさらされ、AWSアカウントの乗っ取り、機密データの漏洩、さらなる侵入の足がかりとなる可能性があります。特に、GitHubなどで公開リポジトリを頻繁に扱う開発チームや、AIツールを積極的に導入している組織でリスクが高まります。CVSSスコア8.5と高深刻度と評価されています。
対策
Amazonはすでにパッチをリリース済みなので、Amazon Q Developerを最新版に更新してください。開発者は、信頼できないリポジトリやワークスペースを安易に「信頼」しない、MCP設定ファイルを必ず検証する、環境変数の最小権限化（特にAWSクレデンシャル）を徹底するなどの運用ルールを設けることが有効です。組織としては、AIコーディングツール導入時にセキュリティレビューを行い、コードレビューやサンドボックス環境でのテストを義務付けることが推奨されます。
