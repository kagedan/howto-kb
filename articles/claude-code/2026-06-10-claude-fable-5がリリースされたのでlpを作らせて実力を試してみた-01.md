## はじめに

Anthropicから新しいモデル「Claude Fable 5」がリリースされました。

Claude Codeのモデル選択にもさっそく出てきました。

![Claude Codeのモデル選択画面にFable 5が表示されている](https://static.zenn.studio/user-upload/deployed-images/de61c2e937840a011d05c5d7.png?sha=caf1105808758bc198c56c268a8166b6492f2c69)

「とりあえず何か作らせてみよう」ということで、Fable 5自身を紹介するランディングページ（LP）を作らせてみました。新しいモデルに自分自身の宣伝ページを作らせる、というちょっとしたメタなお題です。

成果物はこちら。

![LPのスクリーンショット](https://static.zenn.studio/user-upload/deployed-images/9e4bb0c7691a9cfbda6bb213.png?sha=1592dc30dae36a49f0e14dbff16032d0bdd91dd4)

この記事は、その制作過程と触ってみた感想の記録です。

## Claude Fable 5とは

[公式の発表](https://www.anthropic.com/news/claude-fable-5-mythos-5)によると、今回は同じ基盤モデルが2つの提供形態でリリースされています。

* Fable 5: 一般向けに広く提供されるモデル
* Mythos 5: アクセスが制限された、より高い能力を引き出す提供形態

「Mythosクラスの知性を、手の届く価格で」というのが売りのようです。詳細なベンチマークや位置付けは公式発表に譲るとして、本記事では実際に使ってみてどうだったかに絞ります。

## 作らせたもの

お題は「Fable 5自身を紹介するLP」。投げたプロンプトはこれだけです。

```
以下のページを参照して、claude fable5の位置付けやこれまでのモデルと異なるところ、
セールスポイントをLPにまとめてください。LPはテックで先進的なイメージにして、
Javascriptなどを使用して動きのあるものにしてください。

https://www.anthropic.com/news/claude-fable-5-mythos-5
```

要件らしい要件は「テックで先進的」「動きのあるもの」くらいの、かなりふわっとした指示です。

一発目の出力もそれなりに見られるものでしたが、もう少し攻めたデザインにしたかったので、追加で一言だけ投げました。

```
フリーの画像を使用したりして、もっとサイバーパンクな感じにしてください。
```

やり取りは実質この2往復だけです。

## 出てきたもの

生成されたのは、フレームワークなしの素のHTML/CSS/JavaScript構成でした。

```
fable5-lp/
├── index.html
├── css/
│   └── style.css
└── js/
    └── main.js
```

中身を見ていくと、「サイバーパンクにして」という一言から、こういうものが入っていました。

* デジタルレイン: Canvasで描いたマトリクス風のカタカナ＋16進文字が降ってくる背景
* グリッチアニメーション: 見出しがときどきノイズで崩れる演出
* ターミナル風の演出: セクション見出しに`[SYS://BENCHMARK]`のようなタグを付けるなど、コマンドライン的な世界観で統一
* 配色とフォント: シアン×マゼンタのネオンカラーに、Orbitron＋JetBrains Monoの組み合わせ

「サイバーパンク」と言われて連想するものを一通り、それも統一感のある形で入れてきた印象です。

### コードを読んで感心したところ

デザインだけでなく、コードの細部に配慮が行き届いていたのが印象的でした。

まず、`prefers-reduced-motion`への対応が全エフェクトに入っています。

```
const prefersReducedMotion = window.matchMedia("(prefers-reduced-motion: reduce)").matches;

(function initRain() {
  const canvas = document.getElementById("rain-canvas");
  if (!canvas || prefersReducedMotion) return;
  // ...
})();
```

OS側で「視差効果を減らす」を設定しているユーザーにはアニメーションを切る、という配慮です。指示していないのに勝手に入っていました。

デジタルレインの描画もフレームレートを絞っています。

```
function step(now) {
  requestAnimationFrame(step);
  if (now - last < 50) return; // ~20fps で十分
  last = now;
  // ...
}
```

背景演出に60fpsは要らないので20fps程度に間引く、というコメント付き。派手な見た目のわりに負荷への配慮があるのは好印象でした。

ほかにも、リサイズ対応、`"use strict"`、IIFEでのスコープ分離など、雑に書こうと思えばいくらでも雑に書ける部分がきちんとしています。素のJavaScriptでLPを作らせると、フレームワークの流儀に頼れないぶん地力が出やすい気がしますが、安心して読めるコードでした。

## 感想

### 良かったところ

* 「サイバーパンクな感じ」という曖昧な一言を、配色・フォント・エフェクト・コピーの文体まで一貫した形に落とし込んでくれた
* アクセシビリティやパフォーマンスへの配慮など、指示していない品質が乗ってくるのは、これまでのモデルから一段進んだ感触
* 実質2往復で完成し、修正の往復で消耗することがほぼなかった

### 気になったところ

* 出力に時間がかかります。じっくり思考してから書き始めるためか、指示を投げてから完成までの待ち時間は体感で長めでした。品質とのトレードオフではあります
* トークンの消費が激しいです。思考分のトークンも乗ってくるので、usageの減りはこれまでのモデルより明らかに早い印象でした
* LPの内容は公式発表ページを元にしていますが、数値や表現が正確に転記されているかは自分で検証していません。実用で使うならファクトチェックは必須です
* 今回はLP一枚という小さなお題なので、大規模なコードベースでの実力はまだ分かりません

### コスト

LP構築後のusageはこんな感じです。ちなみに、私が使っているのは20ドルのPROプランです。

![LP構築後のusage画面。セッションの56%を消費](https://static.zenn.studio/user-upload/deployed-images/c5fa323c68e4050084774cf5.png?sha=949b8890bea03d359914d86a7c01ed782c0d4617)

前述のとおりトークンの消費ペースは速いものの、LP一枚を作る程度であれば許容範囲でした。日常的にヘビーに使う場合は、プランの上限と相談になりそうです。

## まとめ

Fable 5に自分自身のLPを作らせてみたところ、曖昧な指示2回でデモ公開できるレベルのものが出てきました。

正直、この手の「お試しでLPを作らせる」はこれまでのモデルでも何度かやっているのですが、デザインの引き出しとコードの細部の丁寧さは確実に良くなっていると感じます。次はもう少し複雑な、実際の開発タスクで試してみるつもりです。
