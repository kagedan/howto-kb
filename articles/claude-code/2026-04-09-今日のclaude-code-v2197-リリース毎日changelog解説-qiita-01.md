Bedrock認証の修正、パーミッション周りの堅牢化、`NO_FLICKER`モードの多数の不具合解消など、安定性と信頼性の向上に注力したリリースです。

## 今回の注目ポイント

1. **`--dangerously-skip-permissions`の意図しないダウングレード修正** -- 保護パスへの書き込み承認後にサイレントダウングレードされていた重大な不具合を修正

3. **429リトライのバックオフ改善** -- サーバーが小さい`Retry-After`を返した際にすべてのリトライが約13秒で燃え尽きる問題を修正
4. **Bash toolパーミッション強化** -- 環境変数プレフィックスやネットワークリダイレクト周りのチェックを引き締め、誤ったプロンプト表示を削減

## `--dangerously-skip-permissions`のサイレントダウングレードを修正

**パーミッション管理の信頼性に関わる重要な修正です。**

`--dangerously-skip-permissions`フラグを使って運用しているチームや自動化環境に直接影響する修正です。

### 何が変わったか

`--dangerously-skip-permissions`フラグを付けてClaude Codeを起動した際、保護されたパスへの書き込みを一度承認すると、その後フラグが **accept-editsモードへサイレントダウングレード** されてしまうバグが修正されました。

これはユーザーが気づかないまま動作モードが変化するという、予測可能性・再現性の観点から非常に問題のある挙動でした。

### なぜ嬉しいか

CI/CD環境や自動化スクリプトで`--dangerously-skip-permissions`を使っている場合、セッション中に動作が変わることで自動化が途中で止まるといったトラブルが発生していました。今回の修正でフラグの意味が一貫して保たれるようになります。

`--dangerously-skip-permissions`は強力なフラグです。本番環境での利用には十分注意してください。自動化環境でのみ利用し、権限の範囲を最小限に絞ることを推奨します。

## `Ctrl+O`でフォーカスビュー切り替えが可能に

**`NO_FLICKER`モードユーザー向けに、画面ノイズを大幅に減らせる新機能です。**

`NO_FLICKER`モード（ターミナルのちらつきを抑える表示モード）を使っている方に便利な機能追加です。

### 何が変わったか

`NO_FLICKER`モードで `Ctrl+O` を押すと、**フォーカスビュー**に切り替わるようになりました。このビューでは以下の3要素だけが表示されます。

* 現在のプロンプト
* ツール実行の1行サマリー（ファイル編集の diff 統計を含む）
* 最終応答

### なぜ嬉しいか

ロングセッションや多数のツール呼び出しが連続するときに、今何が起きているかを **シンプルに把握** できます。特に複雑なタスクを実行させながら要点だけ追いたいときに役立ちます。

```
# NO_FLICKER モードで起動
claude --no-flicker

# または設定ファイルで
{
  "noFlicker": true
}

# 起動後、Ctrl+O でフォーカスビュートグル
```

## 429エラー時のリトライ戦略を改善

**APIレート制限に引っかかったとき、短時間で全リトライが消費されてしまう問題を修正しました。**

長時間の作業中にAPIのレート制限（429エラー）に当たることがある方に関係する修正です。

### 何が変わったか

サーバーが `Retry-After` ヘッダーで短い待機時間（例：数秒）を返した場合、Claude Codeはその値をそのまま使ってリトライを繰り返していました。その結果、**全リトライ試行が約13秒で消費** されてしまい、それ以降はエラーになるという問題がありました。

修正後は、`Retry-After`の値が小さい場合でも **指数バックオフが最小値として適用** されるようになりました。

### なぜ嬉しいか

レート制限に引っかかってもリトライが適切に分散されるため、長時間のエージェントタスクが途中でクラッシュしにくくなります。特にAPIクォータを共有しているチーム環境での安定性が向上します。

バグ修正 27件（クリックで展開）

* **Bashツールパーミッション**: 環境変数プレフィックスおよびネットワークリダイレクト周りのチェックを強化し、一般的なコマンドでの誤プロンプト表示を削減
* **パーミッションルール名の衝突**: `toString`などJavaScriptプロトタイプのプロパティ名と一致するルール名が`settings.json`をサイレント無視する問題を修正
* **管理設定のallow rule**: 管理者が削除した後もプロセス再起動まで有効のままになっていた問題を修正
* **`permissions.additionalDirectories`の即時反映**: 設定変更がセッション中に適用されなかった問題を修正
* **`--add-dir`と`additionalDirectories`の競合**: ディレクトリを`additionalDirectories`から削除すると`--add-dir`で渡した同じディレクトリへのアクセスも失効する問題を修正
* **MCP HTTP/SSEのメモリリーク**: サーバー再接続時に約50 MB/hrの未解放バッファが蓄積する問題を修正
* **MCP OAuth `oauth.authServerMetadataUrl`**: 再起動後のトークンリフレッシュ時に設定が無視され、ADFSなど類似IdPで認証が失敗していた問題を修正
* **`/resume`の複数不具合**: `--resume <name>`で開いた会話が編集不可になる、Ctrl+Aによるリロードで検索がクリアされる、空リストでナビゲーションが飲み込まれる、タスクステータステキストが会話サマリーを上書きする、クロスプロジェクトの陳腐化判定など複数の問題を修正
* **`--resume`時のファイル編集diff消失**: 10KB超のファイルを編集した際にdiffが消える問題を修正
* **`--resume`キャッシュミス**: 添付メッセージがトランスクリプトに保存されずキャッシュミスが発生していた問題を修正
* **作業中の入力がトランスクリプトに保存されない**: Claudeが処理中にタイプしたメッセージが記録されなかった問題を修正
* **`Stop`/`SubagentStop`フック**: 長時間セッションで失敗する問題、APIエラー時に「JSON validation failed」と表示される問題を修正
* **サブエージェントの作業ディレクトリ漏洩**: worktreeアイソレーションや`cwd:`オーバーライドを使ったサブエージェントの作業ディレクトリが親セッションのBashツールに漏洩する問題を修正
* **コンパクション時の重複トランスクリプト**: prompt-too-longリトライ時にマルチMBのサブエージェントトランスクリプトが重複書き込みされる問題を修正
* **`claude plugin update`の誤報**: gitベースのマーケットプレースプラグインでリモートに新しいコミットがあっても「already at the latest version」と報告される問題を修正
* **スラッシュコマンドピッカーのクラッシュ**: プラグインのfrontmatter `name`がYAMLのboolean予約語の場合にピッカーが壊れる問題を修正
* **`NO_FLICKER`モードでのURL折り返し**: ラップされたURLコピー時に行頭にスペースが挿入される問題を修正
* **`NO_FLICKER`モードのzellij内スクロール**: zellij内でのスクロールレンダリングアーティファクトを修正
* **`NO_FLICKER`モードのクラッシュ**: MCPツール結果にホバーした際のクラッシュを修正
* **`NO_FLICKER`モードのメモリリーク**: APIリトライ時にストリーミング状態が残留するメモリリークを修正
* **`NO_FLICKER`モードのマウスホイール**: Windows Terminalでのスクロールが遅い問題を修正
* **`NO_FLICKER`モードのステータスライン非表示**: 24行未満のターミナルでカスタムステータスラインが表示されない問題を修正
* **Warpでのショートカット不具合**: `NO_FLICKER`モードでShift+EnterおよびAlt/Cmd+矢印キーが動作しない問題を修正
* **Windows上の文字化け**: `NO_FLICKER`モードでコピーした韓国語・日本語・Unicode文字が化ける問題を修正
* **Bedrock SigV4認証失敗**: `AWS_BEARER_TOKEN_BEDROCK`または`ANTHROPIC_BEDROCK_BASE_URL`が空文字列（GitHub Actionsの未設定inputでよく起きる）の場合に認証が失敗する問題を修正
* **レート制限アップグレード選択肢の消失**: コンテキストコンパクション後にレート制限アップグレードのオプションが消える問題を修正
