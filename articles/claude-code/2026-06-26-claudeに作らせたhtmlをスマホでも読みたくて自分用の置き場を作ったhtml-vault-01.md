## 作った経緯

最近、何かをキャッチアップするときはClaudeに「1枚のHTMLインフォグラフィックにまとめて」と頼んでいます。図解や比較表で返ってくると頭に入りやすく、気づけば手元にHTMLが増えていきました。  
ただ、できあがったHTMLをPCのローカルに置いたままだと、スマホからは見返せません。  
作ったHTMLを、PCからもスマホからもいつでも参照できる仕組みが欲しくて作ったのがhtml-vaultです。

## html-vaultでできること

ざっくり言うと、

* Claudeに作らせたHTMLを保存しておく、自分専用の置き場
* パスワードでログインして使う（共有用ではなく、自分が見るためのもの）
* 保存したHTMLを、ブラウザ上でそのまま表示する
* Dockerイメージ1枚で動く

HTMLを貯めて、一覧から開いて読む。それだけです。  
ただ、それが自分のサーバーにあってスマホからも開ける、というのが自分にはちょうどよかったと思っています。

## 使い方

普段はだいたいこんな流れです。

### 1. ClaudeにHTMLインフォグラフィックを作らせる

キャッチアップしたいテーマをClaude（Claude CodeでもClaude ChatでもOK）に渡して、HTMLのインフォグラフィックにしてもらいます。図解や比較表、フロー図の入った1枚ものが返ってきます。これだけでも、けっこう読みやすいものになります。

![Claudeに作らせたHTMLインフォグラフィックの例](https://static.zenn.studio/user-upload/e610c94a42de-20260626.png)  
*Claudeに作らせたHTMLインフォグラフィックの例*

### 2. vaultに上げて保存する

気に入ったものは、html-vaultに保存します。ここはMCPサーバ経由で、「vaultに上げて」と明示的に指示したときだけアップロードされます。勝手には上げません。手元で作って、残したいものだけ言葉で指示して保存する。この運用に落ち着きました。

![Vaultの一覧画面](https://static.zenn.studio/user-upload/c0743e348708-20260626.png)  
*Vaultの一覧画面*

### 3. スマホで開いて読む

あとはスマホのブラウザからvaultにログインして、一覧から開くだけです。通勤中や寝る前などに、前に作った図解をさっと見返せます。

![スマホで開いたところ](https://static.zenn.studio/user-upload/a02e72f4601b-20260626.png)  
*スマホで開いたところ*

PCで作って、スマホで読む。この往復ができるようになっただけですが、思っていたよりずっと快適になりました。

## Claude Codeでも Claude Chatでも上げられる

最初は、この「vaultに上げる」部分はClaude Codeからだけ使えるようにしていました。手元のMCPサーバをClaude Codeに繋いで、`upload_html`というツールを呼んでもらう仕組みです。これはこれで便利でしたが、ひとつ不満がありました。スマホのClaudeアプリで思いついて作った図解は、その場では上げられないのです。結局あとでPCを開いて、Claude Codeから上げ直すことになります。これでは「スマホで完結」が片手落ちでした。

最近のClaudeには「カスタムコネクター」という仕組みがあり、リモートのMCPサーバを繋げられます。Web版claudeでも、スマホアプリでも使えます。整理すると、2系統になりました。

* **Claude Code（手元）** → 同梱のstdio MCPサーバ経由
* **Web版claude / Claude Chat / スマホアプリ** → vault本体のリモートMCPエンドポイント経由

どちらも同じ`upload_html`ツールで、「このHTMLをvaultに上げて」と頼むと保存されます。

これで、通勤中にスマホのClaudeで「○○を図解して」と頼んで、できたものをそのまま「vaultに上げて」と言うだけで保存できるようになりました。PCを開かずに、作る・保存する・あとで読み返す　が、すべてスマホの中で完結します。

リモートMCPの登録は、Web版claudeの Customize > Connectors からURLを貼るだけです。authless＋秘匿パスで保護していて、自己ホストする人向けの設定方法はリポジトリのREADMEに書きました。

## ふだん、こう使ってる

最近の自分の使い方は、だいたいこんな感じです。

* PCで作業しているときは、**Claude Code**に「○○を図解して」と頼んで、できたHTMLをそのまま「vaultに上げて」で保存します。手を止めずに貯められるのが楽です。
* 思いついたときや移動中は、スマホの**Claudeアプリ**で同じように作って、その場で上げます。PCを開かなくて済みます。
* 貯まったものは、スマホのブラウザでvaultを開いて読み返します。

作る場所（PC / スマホ）はそのときどきです。どちらで作っても同じvaultに集まるので、「PCで作ったやつ、スマホのどこだっけ」がなくなりました。あとで読み返すためのものなので、上げるのは気に入ったものだけにしています。

## 安全性について

保存したHTMLを表示するということは、その中のスクリプトを動かすということでもあります。自分で作ったものとはいえ、無防備に実行するのは少し気持ちが悪いです。なので、プレビューは`allow-same-origin`を外したsandbox付きのiframeに隔離して、表示中のHTMLがアプリ本体に触れないようにしています。

詳しい部分はリポジトリ側に書いてます。

## 自分はWorkersで動かしている（構成）

配布用はDockerイメージですが、自分自身はCloudflare Workersに載せて動かしています。PCを落としても常時動いていてほしいのと、無料枠で収まるからです。構成はシンプルで、要は「Worker 1本 ＋ KV」。常駐サーバもDBもありません。

![Cloudflare Workersを用いたシステム構成図](https://static.zenn.studio/user-upload/6c944a7f4199-20260626.png)  
*Cloudflare Workersを用いたシステム構成図*

* **Worker 1本**がAPI・認証・フロントのHTML配信まで全部やります。落ちる常駐プロセスがないので、放っておいても動いているのが気に入っています。
* データは**Cloudflare KV**。HTML本体は`snip:<id>`、一覧のメタ情報は`index`、あとはログイン試行のレート制限カウンタくらいです。
* ログインパスワードはPBKDF2でハッシュ化してSecretに保存。セッションはHMAC署名した**ステートレスCookie**で、サーバ側にセッションを持ちません。
* ブラウザからのアクセスは前段に\*\*Cloudflare Access（メールOTP）\*\*を一次ゲートとして置き、その後ろでアプリ自身のパスワードが二次ゲートになります。

Web版claude用のリモートMCPも、この同じWorkerに`/mcp`を生やしただけで、新しくサーバを立てたわけではありません。Dockerで配っている中身と、Workersで動かしている自分の環境とで、やっていることはほぼ同じです。
