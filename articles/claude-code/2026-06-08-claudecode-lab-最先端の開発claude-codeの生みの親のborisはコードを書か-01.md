【最先端の開発】Claude Codeの生みの親のBorisはコードを書かない！

https://t.co/QoKmixJ30K

❗️Anthropicの開発現場でもloopを多用しています❗️
「今の私の仕事は"loopを書くこと"だ」
「11月にIDEをアンインストールした。ずっと開いてないなと気づいたので」

【1年前】
IDEでオートコンプリート(AI補完)しながらコードを書いていた。
それが普通のエンジニアリングだった。

【数ヶ月前】
IDE(vscodeやcursorなど)をアンインストールした。
5〜10個のClaude Codeセッションを並行して動かし、
「Claudeにプロンプトを送ること」がコーディングになった。

【今】
Claudeに直接プロンプトを送ることさえしていない。
ループを走らせ、そのloopがClaudeに指示を出す。
loopが「何を作るべきか」を判断している。

loop+Routines+スケジュールで業務自動化を！
↓他の話はリプライにまとめていきます！

Anthropic社内でClaude Codeを全面導入した結果、何が起きたか。

【数字1】エンジニア一人あたりの生産性：3倍以上
【数字2】PR数：数百パーセント増
【数字3】新入社員の教育期間：数週間 → 2日

特に教育期間の話が面白い。

以前は新入りが既存メンバーに「これどうやるの？」と聞くため、
ベテランがコードを書く時間が削られていた。
チームが大きくなると生産性が下がる、という古典的な問題だ。

今は違う。
「Claude Codeに聞けば」で解決する。

通常、チームが大きくなると生産性は落ちる。
それがAnthropicでは逆転している。
人が増えても生産性が下がらない。

Boris氏は「複利効果」という言葉を使った。

「人員を減らしてトークンにシフトする。
リソースが少ない（トークン以外は）ことで、複利効果が生まれる。
自動化することで次回はさらにうまくでき、コストも下がる」

これ、あなたの組織で再現できるか？

Boris氏のアドバイスが面白い
「4人必要なプロジェクトなら2人にして、トークンをたくさん渡して、やり遂げさせてみてください。
おそらく自動化できることが多く、効率化できるはずです」

20〜30年の経験を持つエンジニアが
Claude Codeを使い始めると何が起きるか。

Boris氏は「数ヶ月かけて、もはや通用しない古い習慣を手放すことを教えます」という。

一方で、新卒で入ってきた人がBoris氏にClaude Codeのより良い使い方を教えることもある。

「ネイティブに考えられるからです」

新卒は「コードを書く道具」という先入観がない分、
Claude Codeを「こう使うもの」という固定観念も持っていない。

「新しいモデルが出るたびに再調整して学び直す必要もある」

これは初心者だけの話ではない。
ベテランも、新しいモデルが出るたびに自分のプロンプトスタイルをアップデートしなければならない。

経験は武器だが、同時に負債にもなりうる。

Boris氏は10年のこだわりを、数ヶ月で更新した。

学ぶ速度より、手放す速度の方が大事かもしれない。
