# Day 1 — Claudeとは何か？全体概要

> 参照ドキュメント: [Welcome to Claude](https://docs.claude.com)

---

## なぜこれを学ぶのか

Claudeを使いこなすには、まず「そもそもClaudeって何者なの？」を正確に知ることが大事。なんとなく使っていると、できないことをお願いして「なんで？」ってなりがち。今日はそのモヤモヤを解消する。

---

## 今日学んだこと

### ClaudeはAnthropicが作ったAIアシスタント

**Anthropic（アンソロピック）** というAI安全研究会社が開発したAIアシスタントが「Claude」。2021年創業で、OpenAIの元研究者たちが「AIの安全性をもっと真剣に考えたい」という思いで立ち上げた会社。だから競合他社との一番の違いは、安全性・信頼性への姿勢にある。

### Claudeにアクセスする方法は複数ある

ひとくちに「Claudeを使う」と言っても、いくつかの入口がある。

| アクセス方法 | 向いている人 |
|---|---|
| **claude.ai**（Web/アプリ） | 一般ユーザー。ブラウザやスマホから普通に使う |
| **API（Application Programming Interface）** | 開発者。自分のサービスにClaudeを組み込む |
| **Claude Code** | エンジニア向けのCLI（コマンドライン）ツール |
| **Cowork** | 非エンジニア向けデスクトップアプリ。ファイル操作や自動化ができる |

※ API = プログラム同士が会話するための「窓口」のこと。

### Claudeはどうやって動いているのか

Claudeは**大規模言語モデル（LLM: Large Language Model）**というしくみで動いている。超大量のテキストを学習して、「次に来る言葉は何か」を高精度で予測する機械、というのが本質。

料理で言うと、何百万冊もレシピ本を読んだシェフみたいなもの。でも「リアルタイムで食材の値段を調べる」ことはできない。

**重要な制限：Claudeはインターネットをリアルタイム検索しない。**
学習データには**カットオフ（cutoff）**、つまり「知識の締め切り日」がある。今日のニュースや株価は、基本的に知らない。

### モデルの種類

Claude には現在（2026年6月時点）3つのラインナップがある。最新情報は必ず公式サイトで確認すること。

| モデル | 特徴 | 向いている用途 |
|--------|------|----------------|
| **Claude Haiku（ハイク）** | 最速・最軽量 | 簡単な質問、大量処理 |
| **Claude Sonnet（ソネット）** | 速度と性能のバランス | 日常的なタスク全般 |
| **Claude Opus（オーパス）** | 最高性能 | 複雑な分析、長文ドキュメント |

名前がすべて**詩の形式**なのがAnthropicらしい。

## 実際に試してみた

**実習1：Claudeに自己紹介させる**

```
あなたはどんなAIですか？Anthropicという会社について、そしてあなたの主な得意なこと3つを、初心者にもわかるように教えてください。
```

**実習2：限界を確認する**

```
今日の日付と、現在の主要ニュースを教えてください。わからないことがあれば正直に教えてください。
```

2つ目の実習が特に大事。Claudeが「わかりません」と言う場面を体験することで、正しい期待値がつかめる。「なんでも知ってる神様」ではなく、「知識の締め切り日がある専門家」として見ると付き合い方が変わる。

---

## 今日のまとめ

- Claudeを作ったAnthropicは「AI安全性」を軸に設立された会社で、それがClaudeの設計哲学にも反映されている
- 使い方はclaude.ai / API / Claude Code / Coworkと複数あり、自分の目的に合った入口を選ぶ
- 仕組みはLLM（大規模言語モデル）で、インターネットのリアルタイム検索ではなく学習済みデータを使う
- 知識にはカットオフ（締め切り日）があるため、最新情報には使えない場面もある
- モデルはHaiku・Sonnet・Opusの3段階。用途によって使い分けるのが正解
