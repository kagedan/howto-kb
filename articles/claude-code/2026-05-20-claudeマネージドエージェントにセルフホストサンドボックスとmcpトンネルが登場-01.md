# ClaudeマネージドエージェントにセルフホストサンドボックスとMCPトンネルが登場

Code with Claude Londonにて、Claudeマネージドエージェントの新機能として\*\*セルフホストサンドボックス（パブリックベータ）**と**MCPトンネル（リサーチプレビュー）\*\*が発表されました。これらの機能により、自社のセキュリティ境界内でエージェントを実行できるようになります。

---

## セルフホストサンドボックス（パブリックベータ）

セルフホストサンドボックスを使うと、**自分でコントロールする任意の環境**でエージェントを実行できます。対応する環境は以下の通りです。

* 自社インフラ（独自環境）
* **Cloudflare**
* **Daytona**
* **Modal**
* **Vercel**

エージェントの実行環境を自社インフラまたは信頼するマネージドサンドボックスプロバイダー内に閉じ込めることで、**デフォルトで自社のセキュリティコントロールが適用された状態**でエージェントを動作させることができます。

オンボーディングを支援するため、Cloudflare・Daytona・Docker・Modal・Vercel向けのクックブックも追加されています。

---

## MCPトンネル（リサーチプレビュー）

MCPトンネルを使うと、エージェントは**プライベートネットワーク内のMCPサーバーに接続**できます。MCPサーバーをパブリックインターネットに公開することなく、自社セキュリティ境界内のサービスと安全に連携できるのが特長です。

アクセスリクエストはClaudeプラットフォームから申請できます。

---

## その他の改善点

セルフホストサンドボックスとMCPトンネルに加え、Claudeマネージドエージェント全体にも以下の改善が加えられました。

1. **ライブセッション中のツール・MCPサーバー・Vault IDの動的切り替え**  
   セッションを再起動することなく、実行中のセッションのツール、MCPサーバー、またはVault IDを入れ替えることが可能になりました。
2. **大きなMCPツール出力の自動オフロード**  
   100,000トークンを超える大きなMCPツール出力は、自動的にサンドボックスファイルにオフロードされます。
