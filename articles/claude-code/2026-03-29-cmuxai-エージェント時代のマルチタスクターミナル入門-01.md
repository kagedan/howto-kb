## はじめに

Claude Code、Codex、Gemini CLI など、ターミナル上で動く AI コーディングエージェントが増えてきました。複数のエージェントを同時に走らせていると、「どのタブで何が動いてるか分からない」「エージェントが入力待ちなのに気づかなかった」みたいなことが起きがちです。

**cmux** はこの問題を解決するための macOS ネイティブターミナルです。Ghostty のレンダリングライブラリ（libghostty）で GPU アクセラレーション描画しつつ、複数エージェントの並行運用に特化した UI を持っています。

<https://cmux.com/ja>

## TL;DR

* Ghostty の libghostty ベースの macOS ネイティブターミナル
* **通知リング + サイドバーバッジ + デスクトップ通知**で、エージェントの状態変化を見逃さない
* **縦タブ（サイドバー）** に Git ブランチ・作業ディレクトリ・ポート・通知がリアルタイム表示
* **アプリ内ブラウザ** でターミナル横にブラウザを並べられる
* **Socket API + CLI** でスクリプトから操作できる

## 対象読者

* Claude Code / Codex などの AI エージェントを日常的に使っている人
* 複数プロジェクトを並行して開発している人
* ターミナル環境を見直したい macOS ユーザー
* Ghostty ユーザーで AI エージェント向けの拡張に興味がある人

## cmux の概要

cmux は [manaflow-ai](https://github.com/manaflow-ai/cmux) が開発している macOS 向けターミナルです。

<https://github.com/manaflow-ai/cmux>

### Ghostty との違い

Ghostty は高速・軽量な汎用ターミナルとして優秀ですが、cmux は AI エージェントとの協調作業向けに独自機能を載せています。

| 機能 | Ghostty | cmux |
| --- | --- | --- |
| レンダリングエンジン | libghostty | libghostty（同じ） |
| タブ | 横タブ | **縦タブ（サイドバー）** |
| サイドバー情報 | なし | **Git ブランチ、ディレクトリ、ポート、通知テキスト** |
| 分割ペイン | あり | **あり（ワークスペース単位で管理）** |
| アプリ内ブラウザ | なし | **スクリプタブルなブラウザビュー** |
| セッション永続化 | あり（macOS 限定） | **ワークスペース・スクロールバック・ブラウザ履歴の復元** |

## インストール

```
brew tap manaflow-ai/cmux
brew install --cask cmux
```

## 機能紹介

### 1. 通知システム

cmux の一番の売り。エージェントが入力待ちや処理完了になったとき、複数の手段で知らせてくれます。

**通知リング** — エージェントが入力待ちになると、該当ペインに青いリングが光る。これが地味に便利で、ちらっと画面を見るだけで状態が分かります。色は設定画面で変更できます。

**サイドバーバッジ** — どのワークスペースで通知が発生したか、サイドバーのタブにバッジが付きます。

**デスクトップ通知** — macOS の通知センターにも飛ばせる。OSC 777 にも対応。

**カスタム通知サウンド** — 設定画面から好きなものに変えられます。

**未読通知ジャンプ** — ⌘ + Shift + U で未読通知のあるワークスペースにジャンプ。複数エージェントを走らせていても追いかけられます。

![](https://static.zenn.studio/user-upload/9b310ea89bf8-20260329.png)

![](https://static.zenn.studio/user-upload/bcb3ed1e5624-20260329.png)

### 2. 縦タブ（サイドバー）

横タブだと「Tab 3」「Tab 4」みたいな表示になりがちですが、cmux のサイドバーには意味のある情報が並びます。

* **Git ブランチ名**
* **作業ディレクトリ**
* **リスニングポート番号** — クリックでブラウザ表示できます
* **最新の通知テキスト**
* **ワークスペースカラー**

タブ操作もひと通り揃っています。ドラッグ & ドロップでの並び替え、ウィンドウ間のタブ移動、ピン留め、タブ名リネームなど。  
個人的にワークスペースごとに絵文字、タブ色で視認性が高いワークスペースを作って使っています。

### 3. 分割ペイン & ワークスペース

ワークスペースごとにペインを水平・垂直に分割できます。レイアウトはワークスペース単位で保持されるので、プロジェクトごとに違うレイアウトを維持できます。tmux 互換のコマンドも使えます。  
分割レイアウトはワークスペース内でセッション永続化されます。

| 操作 | ショートカット |
| --- | --- |
| 水平分割 | ⌘ + D |
| 垂直分割 | ⌘ + Shift + D |
| ペイン間移動 | ⌥ + ⌘ + ← or → or ↑ or ↓ |
| ペイン内にタブ追加 | ⌘ + T |

例えば左右分割してAIエージェントを動かしながら分割ペインで開発サーバを動かすという感じ。

![](https://static.zenn.studio/user-upload/b40c73282dac-20260329.png)

### 4. アプリ内ブラウザ

ブラウザが内蔵されていて、ターミナル横に並べて表示できます。ドキュメントを見ながらAIを使ったりするのにウィンドウ切り替えが要りません。

設定画面にて、各ブラウザ (Chrome / Firefox / Safari) から Cookie をインポートできるので、ログイン済みのサービスにそのままアクセスできるのが嬉しい。

| 操作 | ショートカット |
| --- | --- |
| ブラウザを開く | ⌘ + Shift + L |
| 閉じたブラウザパネルを再開 | ⌘ + Shift + T |
| 検索バーにフォーカス | ⌘ + L |
| ページ内検索 | ⌘ + F |
| ズーム | ⌘ + + / ⌘ + - |

![](https://static.zenn.studio/user-upload/617a4a5c808c-20260329.png)

### 5. AI エージェント統合

Hooks の設定を自分で書かなくても、Claude Code や Codex CLI の通知・入力待ちを自動検出して通知してくれます。

### 6. セッション永続化

再起動後も分割レイアウトやワークスペース構成、作業ディレクトリ、ブラウザ履歴などは復元されます。

## Ghostty からの移行

cmux は Ghostty の設定ファイルからテーマ・フォント・色設定を読み込むので、Ghostty ユーザーなら見た目の設定はそのまま使えます。

---

## まず覚えるショートカット

全部覚える必要はなく、これだけ押さえておけば普段の操作は困りません。

| 操作 | ショートカット |
| --- | --- |
| 新規ワークスペース | ⌘ + N |
| ペイン水平分割 | ⌘ + D |
| ペイン垂直分割 | ⌘ + Shift + D |
| ペイン間移動 | ⌥ + ⌘ + ←/→/↑/↓ |
| 未読通知にジャンプ | ⌘ + Shift + U |
| ブラウザを開く | ⌘ + Shift + L |
| コマンドパレット | ⌘ + Shift + P |

## オマケ

私の cmux 設定です。  
Fira Code使う場合は `brew install --cask font-fira-code` でインストールしてください。

cmux設定

```
# --- テーマ・外観 ---
# Vercel: ダークベースにクリーンなコントラスト
theme = Vercel
# 不透明度（1.0 = 完全不透明、下げると半透明）
background-opacity = 1.0

# --- フォント ---
# Fira Code: プログラミング向けリガチャ対応等幅フォント
font-family = Fira Code
font-size = 14
font-style = Regular
# コンテキスト依存の代替字形（contextual alternates）
font-feature = +calt
# 標準リガチャ（!=, =>, -> 等を結合表示）
font-feature = +liga

# --- カーソル ---
# 視認性の高い赤系カーソル + ブリンク
cursor-color = #ff2c6d
cursor-style = block
cursor-style-blink = true

# --- ウィンドウ ---
# macOS ネイティブのタイトルバーを透過表示
window-decoration = true
macos-titlebar-style = transparent
# 内側パディングで余白を確保し、端色をコンテンツと揃える
window-padding-x = 25
window-padding-y = 25
window-padding-color = extend
# 初期サイズ（桁 x 行）
window-width = 140
window-height = 40
# セッション間でウィンドウ位置・サイズを保持
window-save-state = always
# 新規ウィンドウ/タブで現在のディレクトリを引き継ぐ
window-inherit-working-directory = true

# --- クリップボード ---
# アプリからのクリップボード読み書きを許可
clipboard-read = allow
clipboard-write = allow
# テキスト選択時に自動でクリップボードへコピー
copy-on-select = clipboard

# --- アニメーション ---
# クイックターミナル（ドロップダウン）の表示アニメーション時間
quick-terminal-animation-duration = 0.3

# --- アップデート ---
# 新バージョンを自動ダウンロード（インストールは手動）
auto-update = download

# --- キーバインド ---
# Shift+Enter でエスケープシーケンス付き改行を送信（REPLやエディタ向け）
keybind = shift+enter=text:\x1b\r
```

## まとめ

cmux は「AI エージェントを複数並行で走らせる」という使い方にピッタリのターミナルです。

Ghostty の libghostty ベースで設定互換もあるので、Ghostty ユーザーならほぼそのまま乗り換えられます。AI エージェントを日常的に使っている macOS ユーザーにはおすすめです。
