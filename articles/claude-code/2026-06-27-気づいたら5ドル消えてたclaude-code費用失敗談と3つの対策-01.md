## 最初、コストのことを全然考えていなかった。

Claude Codeを使い始めたとき、正直なところ「APIにお金がかかる」という感覚がほとんどなかった。

コードを書いてもらって、修正してもらって、またテストして——その繰り返しが、あまりにもスムーズだった。だから気にならなかった。ツールが動くたびに達成感があって、コスト画面を開こうという発想自体がなかった。

Xへの自動投稿ツールが完成したとき、素直に嬉しかった。「これで毎週の投稿作業が自動化できる」と思って、動作確認を何度も繰り返した。

翌日、Anthropicのコンソールを開いて固まった。$5が消えていた。

---

## 何が起きていたか

$5という金額だけ見れば大したことはない。ただ、自分が意図してお金を使ったわけではなかった。それが問題だった。

原因を調べると、シンプルだった。**テスト実行を繰り返すたびに、APIリクエストが積み上がっていた。**

1回の実行コストは数円程度だ。でも「ちょっと確認」「やっぱりもう一回」を10回、20回と重ねると、気づかないうちに膨らむ。しかも使っていたモデルはSonnet。処理の精度は高いが、その分コストも高い。テスト段階でそれを使い続けていたのは、完全に無駄だった。

コストを意識する仕組みがなければ、人は自然とコストを忘れる。自分がまさにそうだった。
