# はじめに

最近、Claude Code や Cursor、Devin など AI コーディングツールが話題ですが、これらは基本的に 1人のAIに1つの作業を頼む モデルです。

自分が欲しかったのは、AIエージェントを「チーム」として管理して、スプリント単位で開発を回す仕組み でした。

そこで「Toban」というプラットフォームを作っています。

## Toban とは

AIコーディングエージェントをスクラムチーム（のような何か）として管理するプラットフォームです。

* Manager AI がプロジェクトのコードを読んでタスクを提案
* Builder AI がタスクを実装してコミット
* 完了したら 自動でコードレビュー
* ユーザーは Approve / Reject するだけ
* 最後に関わった AI がレトロスペクティブもしてくれる（嬉しい）

<https://www.youtube.com/watch?v=5iHt7c0C2kM>

## 構成

```
ブラウザ                           ローカルマシン
┌─────────────────┐               ┌──────────────────┐
│  Dashboard      │  WebSocket    │  Toban CLI       │
│  (Next.js)      │ ◄──────────►  │  (Node.js)       │
│                 │               │                  │
│  スプリント管理  　│               │  Manager AI      │
│  チャット       　│               │  Builder AI      │
│  ターミナル       │               │  Git操作         │
└─────────────────┘               └──────────────────┘
        │                               │
        ▼                               ▼
   Toban API                        GitHub
   (Cloudflare Workers)             リポジトリ
```

* **Dashboard**: Next.jsで作ったWebダッシュボード。カンバンボード、チャット、ターミナルパネルが1画面に
* **CLI**: ローカルで動くエージェントランナー。Claude Code CLI を使ってエージェントを起動・管理
* **API**: Cloudflare Workers + D1。タスクやスプリントのデータを管理

## 開発の流れ

### 1. プロジェクトをセットアップ

GitHub リポジトリを接続して、プロジェクトの Vision や Tech Stack を入力。これがエージェントのコンテキストになります。

### 2. Manager AI がタスクを提案

チャットで「タスクを提案して」と送ると、Manager AI がバックログやプロジェクトの Spec を読んで、インタラクティブなカードで提案してくれます。ワンクリックでスプリントに追加できます。

### 3. エージェントが実装

タスクをカンバンの「In Progress」に移動すると、承認プロンプトが表示されます。Approve するとエージェントが git worktree で隔離された環境で作業を開始。

ターミナルパネルでリアルタイムにエージェントの行動が見えます:

```
14:03:42  Read src/app/page.tsx
14:03:45  Edit src/components/todo-list.tsx
14:03:52  Bash: npm test
14:04:01  Commit: feat: add todo list component
```

### 4. 自動コードレビュー

エージェントの作業が完了すると、CLI が自動で:

1. worktree のブランチを main にマージ
2. リモートに push
3. LLM にコード差分を渡してレビューコメントを生成

レビューパネルで変更内容と AI のレビューを確認して、Approve / Reject。

```
Agent: builder
Commit: abc1234 feat: add todo list component

Files changed: 3
  src/components/todo-list.tsx | 45 ++++
  src/app/page.tsx             | 12 +-
  __tests__/todo-list.test.tsx | 30 +++

--- Code Review ---
- タスク要件を満たしている
- コンポーネントの分離が適切
- テスト3件追加済み
- Verdict: APPROVE
```

## こだわったポイント

### テンプレートシステム

エージェントの行動を「テンプレート」で制御しています。

```
pre_actions:  git認証チェック → タスクステータス更新
post_actions: git マージ → push → コードレビュー → レトロ送信
```

実装タスクと調査タスクでテンプレートを切り替えれば、調査タスクではファイル変更を禁止するなどの制御ができます。コードを変えずに設定だけで行動を変えられます。

### 承認フロー

エージェントを起動する前に必ずユーザーの承認が必要です。勝手にコードを書き換えたり push したりしません。

### Trunk-Based 開発

エージェントは worktree でコミットだけして、push は CLI が担当。GitHub App のトークン管理も CLI 側で行うので、エージェントが認証情報を扱う必要がありません。

トランクベース開発はガンガン実装してくれるエージェントに合ってると勝手に思ってます。

### Playbook（ルールブック）

エージェントに守らせるルールを「Playbook」として管理できます。

* **セキュリティルール**（locked）: 「秘密情報をハードコードするな」「SQL は必ずプリペアドステートメント」— これらは無効化できない
* **ワークフロールール**（overridable）: 「進捗を API で報告せよ」— プロジェクトに合わなければ OFF にできる
* **カスタムルール**: 「テストは必ず書け」「コミットメッセージは日本語で」— 自由に追加

さらにルールごとに **適用対象のエージェントを選べる** ので、Builder には厳しいコーディング規約を課して、Manager には緩めにする、といった運用ができます。

## Claude Code Agent Teams との違い

Claude Code には最近「Agent Teams」という実験的機能が追加されました。複数の Claude Code セッションがタスクリストを共有して並列作業できる、かなり強力な機能です。

Toban との違いは「誰のためのツールか」です。

Agent Teams は **エンジニアがターミナルで使う** もの。セッションが終われば履歴は消えるし、レビューフローもない。使いこなすには CLI に慣れている必要があります。

Toban は **Web ダッシュボードでチームを管理する** もの。カンバンでタスクを動かし、チャットで Manager に相談し、レビュー画面でコード差分を確認して Approve する。スプリントの履歴やレトロスペクティブも残ります。

将来的には非エンジニアのプロダクトマネージャーが AI チームを管理する、という使い方も見据えています。

## 実際に使ってみて

Toban 自身の開発に Toban を使ってみました（ドッグフーディング）。

元々は別のOSSを作るためのタスク管理アプリだったんですが、勢い余って作り込んでしまいました。

### 良かった点

* ターミナルパネルでエージェントの動きが見えるのは安心感がある
* 凄い勢いでスプリントが回せる
* レビューフローがあるので品質を担保しながら速度が出る

### まだ課題

* Manager AI の応答が遅い（ファイルを読みに行くため）
* エージェントがたまにタスクと無関係な修正をする
* 正直Claude Codeに直接頼んだ方が速い

## AI エージェントのスクラムはまだ発展途上

正直なところ、人間のスクラムをそのまま AI に適用するのは無理があります。

従来のスクラムにはストーリーポイント、ベロシティ、バーンダウンチャートといった計測の仕組みがありますが、AI エージェントでは意味が変わってきます。人間なら「3日かかる」と見積もれるタスクが、AI だと「5分で終わるかもしれないし、30分ハマるかもしれない」。

今の Toban にはこのあたりの機能はほとんどありません。あるのは priority（p0〜p3）だけ。ベロシティもバーンダウンも未実装です。

ただ、AI 向けに意味がありそうな指標は見えてきました:

* **タスク複雑度** (simple / medium / complex) — エージェントの成功率に直結する
* **スプリントベロシティ** — 何タスク完了したか、失敗率はいくつか
* **スプリントゴール** — Manager の提案精度が上がる

AI チームのスクラムに正解はまだないので、使いながら探っていくつもりです。

## こんなこともやりたい

* **デモ環境構築中**: フィードバックお待ちしてます
* **マルチエージェント並列実行**: 複数の Builder が同時にタスクを処理
* **複数エンジン対応**: Gemini、Codex 等にも対応予定
* **運用まで完結**: Vercel や Sentry と連携して、デプロイ・監視・バグ修正のサイクルを自動化
* **AIによるデモ**: スプリントレビュー時に Playwright 等でデモするようにしてくれれば面白い
* **非エンジニア向け**: コードを書かない PM やデザイナーでも AI チームを管理できる UI

## まとめ

AI コーディングツールは 1人のAIに頼む から **AIチームを管理する** に進化すると思っています。Toban はそのための実験です。
