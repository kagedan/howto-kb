3ヶ月前、僕はClaude Codeを使うたびに同じことをしていました。

「口調はこれで、ターゲットはこの層で、禁止ワードはこれで……」

毎回、6つのファイルを手動で開いて、  
  
内容をコピペして、チャット欄に貼り付ける。

投稿を1本作るだけで30分かかっていました。

AIを使っているのに、作業が全然楽にならない。  
  
「これ、何のためにAI使ってるんだろう」と

本気で思っていた時期がある。

**今は、1日15分でX投稿3本が回っています。**  
3ヶ月、毎週21本の投稿を続けて、

フォロワーは328人。

**有料noteは5本出して、3冊売れました。**

値下げはしていません。

100円noteにもしていません。

全部500円です。

フォロワー1,000人いても売れない人は売れない。

でも、**フォロワー300人でも設計がしっかりしていれば売れる。**

**売上は1,500円。**

大きな数字じゃないです。  
  
でも、相互フォロー企画も使わず、

むやみにフォローもしないで、

Xで交流のない人がnoteを買ってくれた。

これは設計ファイルが動いた結果だと思っています。

変わったのは、AIの性能じゃなくて「渡し方」でした。

この記事では、僕が3ヶ月かけて気づいた「3層構造」の話をします。

  

最初に変わったきっかけは、CLAUDE.mdでした。

CLAUDE.mdは、

Claude Codeが起動するたびに自動で読み込むファイルです。

ここに「**自分のルール**」を書いておくと、

毎回コピペしなくても、

Claudeが最初からそのルールを理解した状態で動いてくれる。

僕が最初に書いたのは、こんな内容です。

```
・一人称はnoteでは「僕」、Xでは「ぼく」
・禁止ワード（「簡単に稼げます」「誰でもできます」など）
・ターゲットは25〜40代の会社員で、AIに興味はあるけど使いこなせていない人
・口調は丁寧だけど堅すぎない。データを根拠にしながら噛み砕いて伝える
```

これだけで、毎回のコピペが消えました。

「**Claudeが別人みたいになった**」と感じたのを覚えています。  
  
投稿を頼むと、ちゃんと僕の口調で、

僕のターゲットに向けた文章が返ってくる。

ただ、しばらく使っていると限界が見えてきました。

## CLAUDE.mdだけでは足りなかった理由

CLAUDE.mdには「自分のルール」を書ける。  
  
でも書ける量には限界があります。

X投稿を作るときに必要な情報と、

note記事を書くときに必要な情報は違う。

品質チェックをするときの基準と、

リプライを作るときの基準も違う。

全部CLAUDE.mdに詰め込むと、

ファイルが膨れ上がって、

逆にClaude Codeの精度が落ちる。

「何でも知ってるけど、何を優先すべきかわからない」

状態になるんです。

CLAUDE.mdは万能じゃなかった。  
  
「この作業のときだけ、この知識を使ってほしい」という

切り分けが必要でした。

## 第2層：Skills ── 「業務」を渡す

そこで出会ったのが、Skillsという仕組み。

Skillsは、

「**この作業をするときだけ読み込む専用ルール**」

を作れる機能です。

CLAUDE.mdが「**人格**」を定義するなら、

Skillsは「**業務マニュアル**」を定義する。

僕が実際に作ったSkillsの一部を紹介します。

* **x-post**：X投稿を作るときに、口調ルール・投稿テンプレート・知識ベースを自動で読み込む
* **note-checker**：note記事の品質チェック（文字数・漢字率・長文チェックなど）を自動で回す
* **sns-strategy**：設計ファイル6つを一括で読み込んで、SNS戦略の相談ができる状態を作る

「X投稿を作って」と言えば、x-postスキルが起動する。  
  
必要なファイルを全部読んだ状態で、Claude Codeが動き始める。  
  
毎回6つのファイルを手動で開いていた作業が、

一言で終わるようになりました。

でも、ここでも壁がありました。

## Skillsだけでも足りなかった理由

Skillsは「業務の自動化」です。  
  
「X投稿を作って」と言えば、

決まったファイルを読み込んで、

決まった手順で動いてくれる。

ただ、

**Skillsが読み込む「中身」**

がなければ意味がない。

たとえばx-postスキルは、

起動すると4つの知識ベースファイルを

読み込みます。

でもそのファイルに何も書いてなかったら、

Claude Codeは

「読み込むものがありません」で終わる。

Skillsは配達員みたいなもので、

届ける荷物がなければ動けない。

その「荷物」にあたるのが、第3層の設計ファイル。

## 第3層：設計ファイル ── 「知識」を渡す

設計ファイルは、

**・自分の戦略  
・知識  
・判断基準**をまとめたファイル群です。

僕が実際に使っている設計ファイルを、

カテゴリだけ紹介します。

```
【設計（戦略の土台）】
・ターゲット設計：誰に向けて発信するか、その人の痛み・欲求・お金を出すトリガー
・ブランドボイス設計：自分の口調・キャラクター・禁止ワード・世界観
・ポジショニング設計：競合との違い、自分の強み、核心メッセージ
・商品設計：無料→低単価→高単価の導線マップ
```

```
【知識ベース（投稿の素材）】
・過去に伸びた投稿のパターン集
・自分の体験談ストック
・ターゲットがよく使う言葉の記録
・投稿ネタ一覧（使用済み管理付き）
```

```
【運用ルール】
・曜日別の投稿テーマ設計
・口調・語尾の細かいルール
・投稿の構成パターンとテンプレート
```

これらのファイルを、

SkillsがClaude Codeに届ける。

Claude Codeは、

**CLAUDE.mdで「人格」**を、  
  
**Skillsで「業務手順」**を、

**設計ファイルで「知識」**を受け取る。

3つが揃って初めて、

「**自分の代わりに考えて動ける状態**」になる。  
  
これが、僕が3ヶ月かけて気づいた3層構造です。

## 3層構造で何が変わったか

数字で見せます。

**投稿作成にかかる時間：30分 → 5分  
X投稿の継続期間：15週間（毎週21本）  
有料noteの公開数：5本  
フォロワー：0人 → 328人（相互フォローなし）**

これは才能でもセンスでもないです。  
  
設計ファイルを作って、

Skillsで届けて、

CLAUDE.mdで人格を定義した。  
  
それだけです。

重要なのは、

**この仕組みは一度作ったら使い回せること。**

**・新しいSkillsを追加すれば、新しい業務にも対応できる。  
・設計ファイルを更新すれば、Claude Codeの判断精度が上がる。**

最初に時間をかけて「渡し方」を整えれば、

あとはどんどん楽になっていきます。

## 最後に

僕がこの3層構造を作るのに、3ヶ月かかりました。  
  
試行錯誤して、  
失敗して、  
「こうすればよかったのか」と気づいて、

少しずつ形になった。

この経験を、ファイルごとまとめたものを準備しています。

「スキルファイルパック」という形で、  
  
僕が実際に使っている設計ファイル・

Skills・CLAUDE.mdの設計思想  
  
をまとめて公開する予定です。

**・CLAUDE.mdの書き方と設計の考え方  
・Skillsの作り方と実例  
・設計ファイルの全セット  
（ターゲット設計・ブランドボイス・ポジショニング・商品設計など）  
・知識ベースの運用方法  
・投稿カレンダーの設計**

公開をお待ちください。
