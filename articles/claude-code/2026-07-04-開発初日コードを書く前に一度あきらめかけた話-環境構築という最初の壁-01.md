「AIでアプリを作る」と決めた初日。僕はコードを一文字も書かないうちに、いきなり詰まりました。

前回、9個の機能を1個の「縦串」に削ったところまで書きました。じゃあ次はいよいよ開発だ——と意気込んでターミナルを開き、最初のコマンドを打った瞬間、画面が真っ赤なエラーで埋まります。

**■ いきなり動かない。原因すら分からない**

Next.js というアプリの土台を作るために、ネットで見つけた「おまじない」みたいなコマンドを打ちました。動くはずでした。でも、返ってきたのはこれです。

![](https://assets.st-note.com/img/1783108723-4lUofhtYwyZ2SqRPcQ9NLibn.png?width=1200)

※実際に出たエラー（再現）。コードは、まだ1行も書いていません。

「Node.js のバージョンが古いから無理です」。……そもそも Node.js って何、という状態です。エラーを読んでも、何が悪いのか、どこを直せばいいのか、まったく見当がつきませんでした。正直に言うと、この時点で軽く心が折れました。プログラミングって、本題に入る前のこういう設定で消耗するのか、と。

**■ 原因は"コード"じゃなく"環境"だった**

ここでAI（Claude Code）に泣きつきます。エラーをそのまま貼り付けて「これ、どういう意味?」と聞くと、落ち着いて教えてくれました。要するに、こういうことでした。

・アプリを動かすには「Node.js」という土台のソフトが要る  
・僕のMacに入っていた Node が古すぎて、新しい Next.js が動かない  
・「Homebrew」という道具で Node を新しくすれば直る

言われた通り、Homebrew で node を入れ直しました。バージョンを確認すると、無事に新しくなっています。

![](https://assets.st-note.com/img/1783108781-HBuyU7VtmrcAaK9L12liGkbv.png?width=1200)

※node -v が v22 になった瞬間。これで土台が整いました。

やったことは、たった数行のコマンド。でも大きかったのは「何が原因で、なぜこれで直るのか」を説明してもらえたことです。ただ手順をコピペするのと、理由が分かって打つのとでは、安心感がまるで違いました。

**■ そして、初めて自分のアプリがブラウザに出た**

土台が整ったら、もう一度アプリを起動するコマンドを打ちます。今度はエラーが出ません。ブラウザで「localhost:3000」という、自分のパソコンの中の住所を開くと——

![](https://assets.st-note.com/img/1783108824-erDj6ioZ3lvXRpPJk0FW8Tym.png?width=1200)

※初めて自分のアプリが表示された画面。フッターの「開発1日目 🎉」が嬉しかった。

出ました。自分で作った（正確には、AIと二人で作った）アプリが、初めて画面に映った瞬間です。中身はまだ文字が並んでいるだけ。ボタンも本当には動きません。それでも「レッスンは先生が。成長はAIが。」という、自分で決めた言葉が画面に出ているのを見て、じわっときました。

**■ 今日の学び：最初の壁は「コードを書く前」にある**

非ITエンジニアがつまずくのは、多くの人が想像する「難しいコード」ではないと思います。その手前の、環境構築です。何のエラーか分からない、用語も分からない、そもそも何を調べればいいかも分からない。ここで多くの人が「やっぱり自分には無理だ」と離れてしまう気がします。

でも、今回はっきり分かったことがあります。この壁は、AIに横にいてもらえば越えられる。エラーをそのまま貼って「どういう意味?」と聞けばいい。理由まで教えてくれる相棒がいるだけで、"詰み"が"ただの手順"に変わります。

**💬 今日のひとりごと**

エラー画面の真っ赤な文字、あれ本当に心臓に悪いですよね……。昔だったらここで確実に諦めてました。でも「これどういう意味?」って気軽に聞ける相手がいるだけで、こんなに違うのかと。Nodeが何かも知らない人間がアプリ動かせてるの、冷静にすごい時代だと思います。

**■ 次回**

土台ができたので次はいよいよ本題——と言いたいところですが、このAIコーチには"最大の難所"がありました。しかも「AIの精度」ではありません。レッスンの音声を、どうやって手に入れるか。次回はその話を、正直に。
