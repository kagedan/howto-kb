【海外で大バズ】
Hermes AgentがVSCodeのStar数を超えた
→AI agent repoとして急成長
→そのまま作業OSとして見られ始めて話題😳
名前は Hermes Agent！！
中身はもう“チャットAI”じゃなくて、作業環境を丸ごと持つagentに近い。
正直、今までのAI agent運用👇
・毎回プロンプトで前提を説明
・作業ログが残らない
・skillsやmemoryが人によってバラバラ
・安全確認は手動か無し

これ、人が頭で支えてただけ。Hermesはそこをrepoとprofileで埋めにきてる。

【何が入ってるかというと】

ただのagent UIじゃない。
Hermes Agentの👇
・profile管理
・skills
・MCP連携
・subagents

これらが
→ 1つの作業OSとして連携

で、さらにさらに👇

【ここが評価されてる理由】

これは
❌ プロンプトを書き直して頑張る発想

代わりに👇
① agentの前提をprofileに持たせる
② skillsで作業手順を分ける
③ Obsidian側に判断理由を残す

この3段で
“AIを“相談相手”から“作業組織”に格上げ”する設計。

【しかも一番ヤバいのが Obsidian連携】

・sourceを残す
・判断理由を残す
・次の行動を残す
＝翌日も同じ文脈から再開できる

Hermes / Codex / Claude Codeを使うほど、前提が飛ばないことの価値が大きくなる。

【さらに地味に効くのが Cross-session memory】

毎回ゼロスタートなAI agentに対して、Obsidianはあなたの作業ログを観察して次の文脈に変えられる。
2〜3週間続けるとAIが「あなたの仕事の進め方」で動き始める。

【セットアップは小さくていい】

① sourceをObsidianに残す
② 判断理由を書く
③ 次アクションを書く
④ profile / skillsへ渡す

【結論】

Hermes Agentは「便利なrepo」じゃない。
AI作業を毎回ゼロから組み直してた
“作業チームの仕事”を、OS化し始めた塊。

本気でObsidian + agent運用を触ってる人ほど
効きが分かるレベルです。

Hermesが作業OSとして見られ始めた話を見て終わりにせず、自分のvaultで「AI agentの永続メモリ」まで作るなら、次はこの記事が自然につながります👇
