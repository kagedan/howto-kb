Claude Codeを使っていると、こんな言葉をよく見かける。

> 「Gmailに接続するにはMCPの設定が必要です」  
> 「GitHubのMCP、有効にしてますか？」  
> 「MCPサーバーを起動してください」

最初は何のことかわからない。MCP？サーバー？設定？

ここではMCPという言葉の意味と、それが何をしてくれる仕組みなのかを、できるだけシンプルに解説する。実際の接続手順は次章でやる。この章のゴールは「MCPが何なのか腑に落ちる」こと。

## MCPがない世界——コピペ地獄

想像してほしい。Claude Codeに毎日のGmailを処理させたい、とする。でもClaude Codeはあなたのメールボックスを直接見られない。だから人間がやることになる。

1. Gmailを開く
2. メールの内容をコピー
3. Claude Codeに貼り付ける
4. 「この内容まとめて」と指示する
5. 返答をもらう

メール1通でこの5ステップ。100通あれば500ステップ。気が遠くなる。

---

**自分**：「毎回コピペするのめんどくさいんだけど」

**Claude**：「それ、MCPで解決できる。設定しておけば、自分でGmailを開いて読みに行ける」

**自分**：「自分で？」

**Claude**：「『今週の請求書メール、全部抜き出して』って一言頼むだけで完結する。コピペ不要」

**自分**：「もっと早く言ってほしかった」

---

## MCPとは「Claudeと外部サービスを繋ぐ規格」

**MCP（Model Context Protocol）** は、Claude Codeを外部サービス（Gmail・GitHub・Notion・freeeなど）に安全に繋ぐための「接続規格」だ。

イメージとしてはUSBポートに近い。パソコンにUSBポートがあれば、マウスもキーボードもプリンタも繋がる。MCPも同じで、Claude Code側に「MCPを受け付ける窓口」があり、外部サービス側が「MCP対応です」と宣言していれば、その窓口経由でやり取りができる。

結果として、コピペなしにClaudeが直接「外の世界」を読み書きできるようになる。

## MCPがある世界——Claudeが手を伸ばせる範囲

具体的な例で見てみる。

**Gmail MCP**

* 「今月の請求書メール、全部抽出して」→ Claudeが検索して取得
* 「このメール、文面整えて返信案作って」→ ドラフトを準備

**GitHub MCP**

* 「最新のPRをレビューして」→ PRを読み込んでフィードバック
* 「コミットメッセージ、英語に直して」→ 修正案を提示

**freee MCP**

* 「今月の経費、勘定科目別に集計して」→ 仕訳データから集計
* 「この請求書、登録しといて」→ 自動登録

**Notion MCP**

* 「プロジェクトページの最新ステータス見せて」→ ページを取得
* 「今週のタスク、Notionに転記して」→ 書き込み

どれも「Claudeに話しかけるだけ」で完結する。これがMCPの威力だ。

## どんなMCPが存在するか

主要なMCPは大きく3系統に分かれる。

**仕事・コミュニケーション系**

* Gmail / Google Calendar / Slack / GitHub / Linear / Notion

**業務・財務系**

* freee / Stripe / HubSpot / Salesforce

**データ・開発系**

* Google Sheets / Figma / BigQuery / Postgres

Anthropic公式が出しているものもあれば、各サービスのベンダーが出しているもの、コミュニティが作っているものもある。「自分が毎日使っているサービス」を検索すれば、だいたい何かしらのMCPが見つかる。

## 「接続する」って具体的にどうやるのか

詳しい手順は次章で書くが、基本は設定ファイルに数行書くだけだ。たとえばGitHub MCPの場合、こんな感じになる。

```
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "ghp_xxxxxxxxxxxx"
      }
    }
  }
}
```

この4〜5行を `settings.json` に追記してClaude Codeを再起動すると、もうGitHubに繋がっている。あとは「自分のリポジトリ一覧出して」と言うだけで動く。

難しいのは「APIキーをどこから取ってくるか」だけ。これも次章で各サービス別に丁寧に解説する。

## MCPを使うときのセキュリティ感覚

便利だが、注意点もある。MCPはClaudeに「外部サービスへのアクセス権」を渡す仕組みだ。APIキーを設定するということは、そのキーで実行できる操作（メール送信、ファイル更新、決済操作など）をClaudeが代わりにできるようになる、という意味だ。

だから、入れるMCPは選んだほうがいい。

* 自分が毎日使うサービス → 入れる価値あり
* たまにしか使わないサービス → 不要、コピペで済ませる
* 出所がよくわからないMCP → 避ける

公式または信頼できるベンダーが出しているMCPを優先する。GitHubで★が多く、メンテナンスが続いているものを選ぶといい。

## ローカルMCPとリモートMCP

MCPには大きく2タイプある。

**ローカルMCP**  
自分のPC内で動く。APIキーは自分のマシン内に保存される。プライバシー的に安心。今あるMCPの多くはこのタイプ。

**リモートMCP**  
ベンダーがホストしているサーバーに繋ぐ。OAuthでログインするだけで設定が完了するから楽。一方で、アクセスログがベンダー側に残る点は把握しておく。

入門としては、まずローカルMCPから1つ入れてみるのがおすすめだ。

## まとめ——MCPは「Claudeに手足を生やす」仕組み

|  | MCPなし | MCPあり |
| --- | --- | --- |
| Gmail処理 | 都度コピペ | 「今週のメール抽出して」で完結 |
| GitHub操作 | ブラウザを開いて確認 | 「このPRレビューして」で完結 |
| freee確認 | freeeにログインして抽出 | 「今月の経費教えて」で完結 |

MCPは、Claude Codeを「内側だけで完結する優秀なAI」から「外の世界に手を伸ばせる本物のアシスタント」に変える仕組みだ。

中級者になるということは、この「Claudeに任せる範囲」を広げていくこと。MCPはその第一歩になる。
