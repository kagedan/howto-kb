🤖Claude CodeのCLAUDE.md運用ルール整理

「テストは必ず書くこと」「コミットはConventional Commitsに従うこと」…
丁寧に書いたのに、セッション後半になるとほぼ無視されていませんか？

🔍 原因はCLAUDE.mdの仕組みにあります。
CLAUDE.mdはSystem Promptではなく「最初のUser Message」として注入されます。
公式も「強制的な設定ではなく、コンテキストとして扱われる」と明記。会話が進むほど「だいぶ前に言ったこと」として影響力が薄れていくんです。

📊書くべきものを整理するとこうなります。
✅ CLAUDE.mdに書くべきもの
・プロジェクトの概要・モジュール構成
・セッション開始時の定型手順

❌ 書くべきでないもの
・コーディングルール・テスト方針・コミット規約

CLAUDE.mdの正しい捉え方は「Session Start Hook」。
この視点で見直すだけで、Claude Codeの使い倒し方が一段階上がります💪
