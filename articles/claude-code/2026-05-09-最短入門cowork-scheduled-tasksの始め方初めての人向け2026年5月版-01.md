[昨日の記事](https://note.com/matomarusan/n/ncd9a69603aa0)で、Anthropicが2026年5月に公開した**Claude Code Routines**を紹介しました。あちらはGitHubを使う人向けの機能で、Anthropic側のクラウドで自動実行されるのが特徴です。

今日扱う**Cowork Scheduled Tasks**（以下、Scheduled Tasks）は、その対になる機能です。実行は**自分のPC上のCoworkアプリ**で行われ、対象は**メール・カレンダー・ローカルファイル・ブラウザ・Notion・Slack**など、Coworkがすでに繋がっているサービス全般。

GitHubを使わない方でも使えるため、こちらの方が用途は広い、という整理になります。

本記事は、Coworkを触ったことがない方を対象に、設定から最初の1本を走らせるところまでを順に解説します。

---

## 1. Scheduled Tasksでできること

公式が挙げているユースケースは、以下のあたりです。

* 朝のブリーフィング（メール・カレンダー・Slackの要約）
* 週次レポート（Google Drive・スプレッドシートからの集計）
* 業界・競合の定期リサーチ
* ファイル整理（Downloadsフォルダの分類など）
* 請求書・領収書の振り分け

3つだけ、具体例を挙げておきます。

**例1：朝のブリーフィング**

平日の朝、過去24時間に届いたメール・今日のカレンダー・Slackの未読を読み、「対応が必要なメール3件」「今日の会議4本の要約」「先回りで準備すべき1件」をMarkdownにまとめてNotionに保存する、という運用。

**例2：レシート・領収書の振り分け**

毎週日曜の夜、Downloadsフォルダにたまったレシート画像・領収書PDF・メール添付を月別フォルダに振り分け、合計金額を家計簿スプレッドシートに転記する、という運用。

**例3：金曜の週次レビュー**

毎週金曜18時、今週のカレンダーで実際にあった会議・完了したNotionタスク・自分が出したSlack発言を集めて、「今週の決定事項」「来週への持ち越し」「学んだこと」の3項目にまとめてGoogle Docsに保存する、という運用。

——共通点は、「**毎週手で開いて拾い集めるのが面倒な情報集約**」を定型化しておけることです。

## 2. 始める前に確認する3点

**(1) 有料プラン**

Claude Pro／Max／Team／Enterpriseのいずれか。無料プランでは利用不可。状態は[claude.ai/settings/billing](https://claude.ai/settings/billing)で確認できます。

**(2) Claudeデスクトップアプリのインストール（Coworkタブが利用可能なもの）**

ブラウザ版では使えません。Mac版またはWindows版のClaudeデスクトップアプリを [claude.com/download](https://claude.com/download) から導入してください。Coworkは同アプリ内のタブとして提供されています。初回起動時にCowork用の仮想環境が自動でダウンロードされます。

**(3) PCを起こしておく必要がある**

これが昨日のRoutinesとの最大の違いです。Scheduled Tasksは自分のPC上で動くため、**PCがスリープしている時間、またはClaudeデスクトップアプリが閉じられている時間は走りません**。設定時刻に走れなかった場合は、**次にPCを起こした時点（またはアプリを再度開いた時点）で取り戻し実行**されます。

24時間止めずに動かしたい用途は、Routines側に任せるのが正解です。

## 3. セットアップの5ステップ

### ステップ1：Coworkタブの「Scheduled」サイドバーを開く

Claudeデスクトップアプリを起動し、上部または左サイドバーから**Cowork**タブに切り替え。Coworkの左サイドバーに「**Scheduled**」という項目があるのでクリック。

![](https://assets.st-note.com/img/1778287063-Vqn9iyZw51CMkBhu3SAt2QPx.png?width=1200)

Coworkの左サイドバーに「Scheduled」という項目がある

### ステップ2：「+ New task」をクリック

Scheduledページ右上の「**+ New task**」を押すと、タスク作成モーダルが開きます。

> 補足：チャット欄から `/schedule` と打つことでも、専用Skillが起動して同じことができます。会話的に作りたい場合はそちら。

### ステップ3：タスク内容を入力

モーダルでは以下を入力します（オプション項目以外は必須）。

最初の練習として、朝のブリーフィング用のプロンプトはたとえばこう書きます。

```
今日の予定とやることを整理してください。

1. 過去24時間に届いたメールから、自分が返信や対応する必要が
   あるものを3件まで抽出（重要度順）
2. 今日のGoogleカレンダーの会議を時系列で並べ、各会議の目的を
   1行で要約
3. Notionの「今日のタスク」のうち、優先度高のものを上から3件
4. 最後に、今日まずやるべき1つを推奨してください

出力はMarkdown形式で、Notionの「今日のページ」に保存。
```

### ステップ4：実行頻度を選ぶ

5種類のプリセットがあります。

朝のブリーフィングなら**On weekdaysの朝7時**が無難です。プリセット以外の細かい間隔（例：6時間ごと）が必要な場合は、Coworkチャット内で自然言語で頼むこともできます。

### ステップ5：保存して動作確認

保存すると、Scheduledサイドバーに新しいタスクが追加されます。動作を確認したい場合は、タスク詳細ページの「**Run now**」をクリック。

初回実行時に、メール・カレンダー・Notionなどへの**アクセス許可プロンプト**が出ます。「常に許可」（または同等の選択肢）を選んでおくと、以降の自動実行が止まらずに済みます。最初の1回はその場で立ち会って、すべての権限プロンプトを通しておくのが定石です。

## 4. 最初に試すお題3選

朝のブリーフィング以外で、初学者がうまくいきやすい3例。

**お題1：金曜の振り返り（毎週金曜18時）**

```
今週の振り返りを作成してください。

1. 今週のGoogleカレンダーの会議を全部リストアップし、
   主要な決定事項を3つまでまとめる
2. 今週Notionで「完了」にしたタスクを5件まで抜粋
3. 来週に持ち越したタスクを3件まで
4. 「今週学んだこと」を1つ、自分の発言や進捗から推察して提案

出力はGoogle Docsに「Weekly Review YYYY-MM-DD」として保存。
```

**お題2：Downloadsフォルダの月次整理（毎月1日朝9時）**

```
~/Downloads フォルダを点検し、以下を実行してください。

1. PDFファイルは ~/Documents/PDFs/ に移動
2. 画像ファイル（jpg/png）は ~/Pictures/Inbox/ に移動
3. 30日以上前に保存されたzipファイルは削除候補リストに
   追加（実際の削除はしない、リスト化のみ）
4. 削除候補リストはデスクトップに「downloads_purge_候補.txt」
   として保存

実行結果を最後に1段落でまとめてください。
```

**お題3：今週ブックマークした記事の要約（毎週日曜21時）**

```
今週ブラウザでブックマーク・お気に入り登録した記事
（過去7日間）を全部開き、各記事の要約を3行で書いてください。

それぞれに「自分の業務に活かせるポイント」を1行ずつ追記。
最終的にMarkdown形式で、Notionの「Reading Notes」データベース
に新規ページとして追加してください。
```

## 5. Q. RoutinesとScheduled Tasks、結局どう使い分けるのか

昨日も並べましたが、改めて整理しておきます。

**動く場所**

**PCを閉じても走るか**

**触れる対象**

**実行頻度プリセット**

**使えるトリガー**

**使い分けの目安**

* メール・カレンダー・ローカルファイルなど手元の環境を相手にする：Cowork Scheduled Tasks
* GitHubで完結する作業を、PCを閉じていても回したい：Claude Code Routines
* 両方やりたい：両方使える（同じClaudeアカウントで併用可）

朝のブリーフィングはCowork、夜間のGitHub PRレビューはRoutines、という役割分担が現実的です。

詳しい仕様はAnthropic公式の[Coworkスケジュールタスク解説](https://support.claude.com/en/articles/13854387-schedule-recurring-tasks-in-claude-cowork)、および昨日の[Routines記事](https://note.com/matomarusan/)を参照してください。

## 6. 運用前に押さえておく注意点3つ

**(1) PCのスリープ・アプリ終了に注意**

繰り返しになりますが、**PCがスリープしている時間とClaudeデスクトップアプリが閉じている時間**は走りません。「毎朝7時に動かしたいのにPCを閉じて寝ていた」という事故が起きやすい点に注意。対策はおおむね2つです。

**(2) 初回の権限許可を取りこぼさない**

メール・カレンダー・ファイルへのアクセスには初回に権限プロンプトが出ます。「**常に許可**」（または同等の選択肢）を選ばないと、次回以降の自動実行で停止することがあります。新しいタスクを作ったら、必ず「Run now」で1度走らせて、出てくるプロンプトをその場で全部通しておくこと。

**(3) プロンプトに成功条件と例外時の挙動を書く**

自動実行のため、人間が途中で介入できません。「何が成功か」「該当ゼロのときどう振る舞うか」をプロンプト内に明記しておくと、結果が安定します。

```
成功条件：上記3項目がNotionの今日のページに保存された状態。
該当メールが0件なら「対応が必要なメールはありません」と
だけ書き、推測を含めない。
```

## まとめ

Cowork Scheduled Tasksは、**Coworkで都度実行している作業のうち、定型化できるものを時刻トリガーで回せるようにする機能**です。設定は5分程度。最初の1本は失敗しても削除すれば消えます。

最初に試すなら、平日朝7時に「メール・カレンダー・Notionから今日の準備を整える」朝のブリーフィングが、構築コストも低く、結果も実感しやすい題材です。

役割分担は、手元のメール・カレンダー・ファイルが対象ならCowork Scheduled Tasks、GitHubで完結する作業を不在中も回したいならClaude Code Routines、と覚えておけば迷う場面はそれほど多くありません。
