Claude Code はほぼ毎日パッチが出ており、2〜3週間見ないうちにサブエージェントや権限まわりの前提が変わっていることがあります。この記事は、公式 CHANGELOG をもとに **2026年5月〜7月（v2.1.136 → v2.1.202）の差分**をテーマ別に整理したものです。

## 対象範囲

* **差分の起点**: v2.1.136（2026-05-08）
* **差分の終点**: v2.1.202（2026-07-06）
* **集計日**: 2026-07-07

つまり「2026年5月上旬から7月上旬までの約2ヶ月」で入った変更のまとめです。バージョン番号は追跡しやすいよう各項目に併記しています。

## TL;DR

* **サブエージェントが大きく変わった（v2.1.198）**: デフォルトでバックグラウンド実行になり、Explore エージェントは Haiku ではなくメインセッションのモデル（上限 Opus）を継承、extended thinking 設定も継承するようになった。
* **デフォルト権限モードが「Manual」に（v2.1.200）**: 何も設定しないと確認プロンプトが増える。あわせて hook マッチャーがハイフン付きで完全一致に変わった（v2.1.195）ので、既存の `settings.json` / hooks は点検推奨。
* **スキル機能の拡張**: スラッシュスキルのスタック起動（最大5個, v2.1.199）、ネスト `.claude/skills` の自動ロード（v2.1.178）、`disallowed-tools` frontmatter（v2.1.152）など。
* **モデル世代交代**: Opus 4.8（v2.1.154）、Fable 5（v2.1.170）、Sonnet 5 がデフォルトモデルに（v2.1.197）。
* **dynamic workflows 導入（v2.1.154）**: 数十〜数百エージェントのオーケストレーション。トリガー語は `workflow` → `ultracode` に変更（v2.1.160）。

## 1. サブエージェント / オーケストレーション

この2ヶ月で最も体感が変わった領域です。

* **サブエージェントがデフォルトでバックグラウンド実行に**（v2.1.198）。Claude はサブエージェント実行中も作業を続け、完了時に通知を受け取る形になりました。
* **Explore エージェントがメインセッションのモデルを継承**（上限 Opus）するように変更（v2.1.198）。従来は Haiku 固定でした。
* **サブエージェントとコンテキスト圧縮が extended thinking 設定を継承**するように（v2.1.198）。委譲タスクの出力品質向上が狙い。
* **`claude agents` から起動したバックグラウンドエージェントは、worktree でのコード作業完了時に commit・push・draft PR 作成まで自動で行う**ように（v2.1.198）。以前は停止して確認していました。
* **サブエージェントが自分のサブエージェントを起動可能に**（最大5階層, v2.1.172）。
* **`Tool(param:value)` 権限ルール構文の追加**（v2.1.178）。例えば `Agent(model:opus)` で Opus サブエージェントをブロックできます。`Agent(type)` の deny ルールが named spawn に効かないバグは v2.1.186 で修正。
* **エージェントチームの簡素化**（v2.1.178）: `TeamCreate` / `TeamDelete` ツールを廃止し、Agent ツールの `name` パラメータで直接 teammate を起動する形に。
* **dynamic workflows の導入**（v2.1.154）: 「ワークフローを作って」と頼むと数十〜数百のエージェントをバックグラウンドでオーケストレーションします。`/workflows` で実行状況を確認。トリガーとなるキーワードは `workflow` から **`ultracode`** に変更されました（v2.1.160）。「workflow」という単語だけでは発動しません。

## 2. 権限 / hooks / 設定

* **デフォルト権限モードが「Manual」に変更**（v2.1.200, CLI・VS Code・JetBrains 共通）。`--permission-mode manual` / `"defaultMode": "manual"` も受理。未設定だと確認プロンプトが増えるため、allow ルールの整備か `defaultMode` の明示が実質必要になりました。
* **`AskUserQuestion` ダイアログが自動継続しなくなった**（v2.1.200）。アイドルタイムアウトは `/config` でオプトイン。
* **hook マッチャーのハイフン付き識別子（`code-reviewer`, `mcp__brave-search` など）が substring 一致 → 完全一致に**（v2.1.195）。MCP サーバの全ツールに当てたい場合は `mcp__brave-search__.*` のように書く必要があります。
* **カンマ区切りマッチャー（`"Bash,PowerShell"`）が発火していなかったバグを修正**（v2.1.191）。今まで無音で無効でした。
* **`!` バッシュコマンドの出力に Claude が自動応答するように**（v2.1.186）。従来のコンテキスト注入のみに戻すには `"respondToBashCommands": false`。
* **`$HOME` 経由の参照で deny ルールを迂回できたバグを修正**（v2.1.163）。`Read(~/Desktop/**)` のような deny が `$HOME` 参照の Bash コマンドもブロックするように。
* **`WebFetch(domain:...)` の明示ルールが事前承認ドメインの自動許可より優先**に（v2.1.162）。
* **auto mode のオプトイン同意が不要に**（v2.1.152）。`autoMode.hard_deny`（allow 例外なしの無条件ブロック）も追加（v2.1.136）。

## 3. スキル

* **スラッシュスキルのスタック起動**（v2.1.199）: `/skill-a /skill-b do XYZ` のように書くと、先頭の最大5個のスキルをすべてロードするように（従来は先頭のみ）。
* **ネストした `.claude/skills` の自動ロード**（v2.1.178）: サブディレクトリのスキルも作業時に読み込まれ、名前衝突時は `<dir>:<name>` 形式で両方使えます。
* **スキル再 invocation で指示が二重追加されるバグを修正**（v2.1.202）。
* **`disallowed-tools` frontmatter の追加**（v2.1.152）: スキル/スラッシュコマンド実行中だけツールをプールから除外できます。あわせて **`/reload-skills`** でセッションを再起動せずスキルディレクトリを再スキャン可能に。
* **`disableBundledSkills` 設定 / `CLAUDE_CODE_DISABLE_BUNDLED_SKILLS` 環境変数**（v2.1.169）: バンドルスキル・組み込みスラッシュコマンドをモデルから隠せます。

## 4. verify / sandbox / セキュリティ

* **`acceptEdits` モードでも、コード実行につながるビルドツール設定ファイルの書き込みには確認が入る**ように（v2.1.160）。対象は `.npmrc`, `.yarnrc*`, `bunfig.toml`, `.bazelrc`, `.pre-commit-config.yaml`, `.devcontainer/` など。
* **シェル起動ファイル（`.zshenv` など）と `~/.config/git/` への書き込み前にプロンプト表示**（v2.1.160）。意図しないコマンド実行の防止。
* **`sandbox.credentials` 設定の追加**（v2.1.187）: sandbox 化されたコマンドから資格情報ファイル・秘密環境変数の読み取りをブロック。
* **auto mode が破壊的 git コマンドを指示なしでブロック**（v2.1.183）: `git reset --hard`, `git checkout -- .`, `git clean -fd`, `git stash drop` など。`terraform destroy` / `pulumi destroy` / `cdk destroy` も対象。

## 5. 新ツール / コマンド

* **`claude agents`（agent view, Research Preview）**（v2.1.139）: 実行中・入力待ち・完了のセッションを一覧表示。
* **`/goal`**（v2.1.139）: 完了条件を設定すると、満たすまでターンをまたいで作業を継続。
* **`/dataviz`**（v2.1.198）: チャート/ダッシュボード設計ガイダンスとカラーパレット検証。
* **`/cd`**（v2.1.169）: プロンプトキャッシュを壊さずにセッションの作業ディレクトリを移動。
* **`--safe-mode` フラグ / `CLAUDE_CODE_SAFE_MODE`**（v2.1.169）: CLAUDE.md・プラグイン・スキル・hooks・MCP をすべて無効にして起動（トラブルシュート用）。
* **レビュー系コマンドの再編**: `/simplify` → `/code-review` に改名（v2.1.147）→ `/simplify` はクリーンアップ専用として復活（v2.1.154）→ `/review <pr>` は高速単一パスに戻り、多段レビューは `/code-review <level> <pr#>` へ（v2.1.202）。

## 6. モデル

* **Opus 4.8**（v2.1.154）: デフォルト effort が high に。最難タスク向けに `/effort xhigh`。
* **Claude Fable 5**（v2.1.170）: 一般提供向けに安全化された Mythos クラスのモデル。
* **Claude Sonnet 5**（v2.1.197）: Claude Code のデフォルトモデルに。ネイティブ 1M トークンコンテキスト、8/31 まで $2/$10 per Mtok のプロモ価格。
* **`/model` の選択が新規セッションのデフォルトとして保存されるように**（v2.1.153）。現セッション限定で切り替えたい場合はピッカーで `s`。
* **lean システムプロンプトがデフォルトに**（v2.1.154, Haiku・Sonnet・Opus 4.7 以前を除く）。

## 参考: コミュニティで報告されている既知の不具合（changelog外）

* **サブエージェントのモデルルーティングが機能しない（[#43869](https://github.com/anthropics/claude-code/issues/43869), open）**: frontmatter の `model:`、`Agent(model:...)`、環境変数 `CLAUDE_CODE_SUBAGENT_MODEL`（settings.json の `env` 含む）といった、サブエージェントを別モデルにルーティングする機構がすべて無視され、常に親セッションのモデル（Opus）で動く、という報告。上記「Explore がメインモデルを継承」の変更（v2.1.198）とあわせ、サブエージェントに探索を任せる構成ではコストが跳ねやすい点に注意。
* **バックグラウンドエージェントの自動 commit/push/PR を止める設定の要望（[#73197](https://github.com/anthropics/claude-code/issues/73197), open）**: v2.1.198 の自動 commit・push・draft PR 化により、人間のレビューを挟む運用（propose-only）が崩れるとして、無効化オプションを求める feature request。
* **ドキュメントの追従漏れ**: Explore のモデル継承変更後もサブエージェントのドキュメントが「Haiku を使う」と古いまま（[#72940](https://github.com/anthropics/claude-code/issues/72940), open）、`sandbox.credentials` が設定ドキュメントに未掲載（[#70440](https://github.com/anthropics/claude-code/issues/70440), open）といった報告。

サブエージェントを別モデルで回す構成を組んでいる場合は、`/usage` のカテゴリ別内訳（v2.1.149 で追加）で実際のモデル別消費を確認するのが安全です。
