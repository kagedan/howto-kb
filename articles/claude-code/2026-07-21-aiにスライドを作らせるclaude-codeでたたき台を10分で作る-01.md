Claude Codeを使ってプレゼン資料を作るとき、こんな風景に出くわす。

と言うと、次の瞬間には「pptx生成します」「Markdown形式で」「HTMLで」と選択肢が出てくる。教科書でも `generate-slides` スキルは何度か出てきたし、Claude Codeはスライドを作れる。でも、**「どうやって作らせるのか、正解は何なのか、どれを選べばいいのか」** についてはあんまり明確に書かれていなかった。

ここで一度、整理する。スライド作成をAIに任せる流れを理解すれば、そこそこ見られる資料の下地が10分でできる。あとは人間が磨くだけだ。

## AIにスライドを作らせる仕組み

スライド作成でAIが活躍するのは、**「構成を決めて、テンプレートに詰め込む」フェーズ** だ。

手作業で考えてみてほしい。プレゼンを作る流れはこうなる。

1. テーマと目的を決める（「営業向けに四半期成果を説明」など）
2. 大まかな構成を書く（「導入→成果→今後」という3枚、みたいな）
3. 1枚1枚のタイトルと要点をリストアップ
4. 図を入れたり、デザインを整えたり、色を決めたり

AIが得意なのは、**2と3の部分** 。白紙から全部を作るより、「大体こういう流れでいきたい」と言葉で伝えて、AIに構成とテキストを返してもらう。それから人間がスライドを開いて、図を足したり色を整えたり——この流れの方が、ずっと速い。

## 作り方の選択肢

Claude Codeがスライドを生成する方法は3つ。どれを選んでもいい。

### 1. PowerPoint（.pptx）

Claude Codeのpptxスキルを使って、PowerPointファイルそのものを生成する。

| メリット | デメリット |
| --- | --- |
| オフィスの標準。そのまま編集・プレゼンできる | AIが生成したデザインはシンプル（高度な装飾は人間が後処理） |
| 図表の挿入がしやすい | ファイルサイズが重くなる傾向 |
| Wordやその他アプリと連携しやすい | 凝ったアニメーションは手動で追加が必要 |

**向いている場面：** 営業資料、社内報告、クライアント向けプレゼン。「ファイルとして納品」する場合。

### 2. Markdown（Marp）

Markdownという文字ベースのフォーマットでスライドを書き、ブラウザやHTML、PDFに変換する。

| メリット | デメリット |
| --- | --- |
| テキストベース。版管理がシンプル | デザインの自由度は低い |
| ブラウザで即座に確認できる | 図の挿入は別途作成が必要 |
| HTMLやPDFに簡単に変換できる | チームでの共編集には向かない |
| 情報量を詰め込むのに向いている | PowerPointの「ふつう」の要求に応えづらい |

**向いている場面：** 技術者向けの説明資料、ブログ記事の実装例、勉強会のスライド。「デザインより情報」を優先する場合。

### 3. HTML（生JavaScript）

Vue.jsやReveal.jsなどのライブラリで、Webブラウザで動作するスライドを作る。

| メリット | デメリット |
| --- | --- |
| 完全カスタマイズできる。アニメーションも可 | 実装コストが高い。CSSやJavaScriptの知識が必要 |
| オンラインで共有・プレゼンできる | ローカルファイルより配布が手間 |
| 凝った表現ができる | AI生成の段階では大幅な下地が必要 |

**向いている場面：** プロダクトローンチイベント、デザイナー向けポートフォリオ、技術トークの拡張版。「見た目や体験」を重視する場合。

ほとんどの場面では、**1. PowerPoint（pptx）で十分** だ。理由は単純——「オフィスの標準だから」。編集も共有も楽。

## AI に「正しく」頼む流れ

ここが重要だ。AIにスライドを作らせるときの黄金パターンがある。

まず、こう頼む。

```
以下の営業向けプレゼンのスライド構成を作ってください。

【テーマ】
四半期成果報告（Q2）

【メッセージ】
売上20%増、新規顧客30件獲得

【構成案】
- 導入（Q2の目標）
- 成果の内訳（売上増の理由、顧客の特性）
- 今後の施策

【制約】
- 5枚以内
- 1枚のテキストは50字以内
- 図は後から自分たちで入れる
```

大事なのは、**「完全に仕上がったものを期待しない」こと** だ。AIが返してくるのは「たたき台」。人間が加筆・修正する前提で頼む。

返ってきた構成を確認して、こう言う。

```
このスライド構成、2枚目の「成果の内訳」をもう少し詳しく。
売上増の要因と新規顧客の業種別内訳をそれぞれ1項目ずつ追加してください。
```

段階的に調整してもらう。これが、短時間で良い資料を作るコツだ。

## よく聞く用語ミニ辞典

* **pptx** — PowerPointの保存形式。AIはこれで生成できるし、すぐに編集できる
* **Markdown** — 見出しや箇条書きを `#` や `-` で書く文字ベースのフォーマット
* **Marp** — Markdownでスライドを書くツール。プレゼン用に拡張されている
* **Reveal.js** — HTMLベースのスライドライブラリ。ウェブブラウザで動作
* **テンプレート** — あらかじめ決めておいた雛形。AIはここに内容を詰め込む

## まとめ

| 選択肢 | 向いている場面 | 難易度 |
| --- | --- | --- |
| PowerPoint（pptx） | 営業・社内報告・クライアント向け。汎用性重視 | 低。AIに丸投げできる |
| Markdown（Marp） | 技術資料・ブログ・勉強会。情報密度重視 | 中。変換設定は必要 |
| HTML（JavaScript） | ローンチ・ポートフォリオ・トークイベント。デザイン重視 | 高。AIの下地からの手直し必須 |

ひとことで言えば——**「全部作らせるのではなく、骨組みだけAIに任せて、人間が肉付けする」**。これが、AIリソースを使って資料を速く作るコツだ。
