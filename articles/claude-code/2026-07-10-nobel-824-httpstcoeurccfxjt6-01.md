# そもそもCodexはどこで動かす？

作り方の前に、入口だけ先に片付けます。ここが分からないと、この先の「こう頼む」が全部絵に描いた餅になるからです。

Codexには2つの入口があります。手軽なのは、ChatGPTの中から使う形。ブラウザで chatgpt.com を開き、サイドバーやアプリの中から「Codex」を選べば始められます（画面や使えるプランは変わることがあるので、見当たらなければさっきの公式ページで最新を確認してください）。もうひとつが、パソコンに入れて使う「Codex CLI」です。僕も最初はChatGPT側から触りました。初めてなら、こちらで十分です。

先に正直に言っておくと、この記事の後半に出てくる AGENTS.md やコマンドは、もっと作り込みたい人向けのCLI版の話です。まずはChatGPTのCodexに「資料を作って」と日本語で頼むだけでも、ちゃんと始められます。身構えなくて大丈夫です。

「CLI＝黒い画面にコマンド」と聞くと構えるかもしれませんが、コマンドすら自分で打つ必要はありません。この記事にコマンドがいくつか出てきますが、実際には「このコマンドを実行して」とCodexにお願いすれば、Codex自身が動かしてくれます。あなたがやるのは、日本語で頼むことだけです。

# なぜ「コードを書くAI」が資料作成に強いのか

では、なぜコードを書くAIが、資料まで正確に組めるのでしょう。理由は2つあります。

答えはシンプルで、資料は「構造化された成果物」だからです。見出しがあって、階層があって、色やフォントのルールがあって、図が配置される。これはコードとまったく同じ構造をしています。だからコードを正確に組み立てられるAIは、資料も正確に組み立てられる。むしろ人間より、ルールを最後まで律儀に守ります。

そしてもう一つ大きいのが、Codexが作るのは「編集可能な資料」だという点です。
