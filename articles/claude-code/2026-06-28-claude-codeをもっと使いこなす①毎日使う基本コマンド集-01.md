# Claude Codeをもっと使いこなす①｜毎日使う基本コマンド集

こんにちは、開発日和です。

Claude Codeを使い始めると「チャットに指示するだけ」で満足しがちですが、実はスラッシュコマンドを知っているだけで作業効率がぐっと変わります。

この記事では、**毎日使うレベルの基本コマンド**をまとめました。「こんな機能あったの？」がきっといくつかあるはずです。

---

## この記事で紹介するコマンド

| コマンド | ひとことメモ |
| --- | --- |
| `/init` | プロジェクトの初期設定ファイルを作る |
| `/memory` | Claudeに覚えてもらう情報を管理する |
| `/model` | 使うClaudeのモデルを切り替える |
| `/compact` | 会話を圧縮してコンテキストを節約する |
| `/clear` | 会話履歴をリセットする |
| `/review` | コードレビューをしてもらう |
| `/permissions` | 実行を許可するコマンドを設定する |
| `/mcp` | 外部ツールとの連携を管理する |
| `/usage` | 今のセッションのコストを確認する |

---

## `/init` ― プロジェクトを「理解してもらう」第一歩

### ①何をするか

プロジェクトのルートディレクトリを解析して、`CLAUDE.md` ファイルを自動生成します。このファイルがあると、Claudeがプロジェクトの構成・技術スタック・よく使うコマンドを会話のたびに把握した状態でスタートできます。

### ②実際の挙動

実行するとClaudeがディレクトリ構成・設定ファイル・READMEなどを読み込み、`CLAUDE.md` の草案を作ってくれます。内容を確認して、必要に応じて追記・修正すればOKです。

### ③活用方法・つまずいたポイント

* **既存の `CLAUDE.md` があっても上書きせず改善提案をしてくれる**（いきなりがっつり変更されるわけではないので安心して使える）
* **空のディレクトリで実行すると内容が薄くなる**。ある程度コードが揃ってから使うほうが良い
* `CLAUDE.md` は一度作ったら終わりではなく、プロジェクトを進めつつ育てていくもの

---

## `/memory` ― Claudeに「これ覚えておいて」を伝える

### ①何をするか

Claudeがセッションをまたいで記憶しておく情報を管理するコマンドです。`~/.claude/CLAUDE.md`（ユーザーグローバル）やプロジェクトの`CLAUDE.md`に情報を追記する形で動作します。

### ②実際の挙動

![](https://static.zenn.studio/user-upload/3cf3186441a9-20260628.png)  
現在記憶している内容の確認・追加・削除ができます。  
実行すると、保存先を聞かれるので、記録させる場所を選択できます。  
たとえば「私はPHPをメイン言語として使っている」「このプロジェクトはLaravelで作られている」といった情報を覚えさせておくと、毎回説明しなくて済みます。

### ③活用方法・つまずいたポイント

* **記憶の保存先は `CLAUDE.md` ファイル**。特別な領域があるわけではなく、ファイルに書き込まれているだけ
* **なんでもかんでも覚えさせすぎると `CLAUDE.md` が肥大化**して、コンテキストを圧迫する原因になる
* 「このセッションだけ覚えておいてほしい」ことは `/memory` ではなくチャット内で直接伝えればOK

---

## `/model` ― 作業に合わせてモデルを使い分ける

### ①何をするか

使用するClaudeのモデルをその場で切り替えます。重い作業にはOpus、軽いタスクにはHaikuなど、コストと性能のバランスを調整できます。

### ②実際の挙動

![](https://static.zenn.studio/user-upload/228693e93cfb-20260628.png)  
実際の実行結果。ここから選択肢が表示されるので、使いたいモデルを選ぶだけです。切り替えは即座に有効になります。

利用可能なモデルの例：

* `opus` ― 最高性能。複雑な設計・コードレビューに
* `sonnet` ― バランス型。日常的な開発作業に
* `haiku` ― 高速・低コスト。簡単な質問・変換作業に

### ③活用方法・つまずいたポイント

* **モデルを切り替えても会話履歴は引き継がれる**（コンテキストは消えない）
* **コスト最適化の観点から、作業に応じて使い分けるのがコツ**。設計の相談はOpus、コードの修正はSonnet、など
* モデルのIDはアップデートで変わることがあるので、最新情報は `/help` や公式ドキュメントで確認を

---

## `/compact` ― コンテキストが膨らんできたら使う

### ①何をするか

会話履歴を要約・圧縮して、コンテキストウィンドウの使用量を減らします。長い作業セッションで「コンテキストがいっぱいになりそう」というときに使います。

### ②実際の挙動

Claudeがこれまでの会話を要約し、重要な情報だけを残してコンパクトにまとめます。要約後も作業は継続できます。

カスタム指示を渡すこともできます：

```
> /compact 認証機能の実装に関する部分を重点的に残して
```

### ③活用方法・つまずいたポイント

* **圧縮後は細かい経緯が消える**ので、「さっき言った〇〇の件」が通じなくなることがある。圧縮前に重要な決定事項はメモしておくと安心
* **コンテキスト使用量は `/usage` で確認できる**（後述）
* 圧縮しすぎると必要な情報も消えるので、タイミングを見極めるのがコツ

---

## `/clear` ― 気持ちよく新しいセッションを始める

### ①何をするか

現在の会話履歴をすべて消去して、まっさらな状態に戻します。別のタスクを始めるときや、Claudeが混乱してきたと感じたときに使います。

### ②実際の挙動

確認なしに即座に会話履歴がリセットされます。ただし `CLAUDE.md` の内容は消えないので、プロジェクトの基本情報はリセット後も引き継がれます。

### ③活用方法・つまずいたポイント

* **元に戻せない**ので、重要な内容が会話に残っている場合は先にメモを
* `/clear` ではなく `/compact` で節約するという選択肢もある。完全リセットが必要かどうかを考えてから使う
* セッション変数や一時的な情報もすべて消えるので、長い作業の途中で使うのは注意が必要

---

## `/review` ― コードレビューをサクッと頼む

### ①何をするか

現在のブランチの変更差分をClaudeがレビューします。「このコード大丈夫かな？」という確認を気軽に依頼できます。

### ②実際の挙動

git diffの内容やPRの内容をもとに、バグの可能性・改善点・セキュリティの懸念などを指摘してくれます。

レビューの深度を指定することもできます：

PRのレビューを依頼したい場合は、PR番号を指定することもできます：

![](https://static.zenn.studio/user-upload/54a8ebd3f6b1-20260628.png)  
こんな感じで、PRの要約と、色々な検証結果を報告してもらえるので便利です。  
添付画像は適当な内容のPRなので、あまり参考にならないかもしれないですが、実際はもっと的確に指摘してくれます。

### ③活用方法・つまずいたポイント

* **gitで管理されていないファイルは対象外**。レビューしてほしいファイルはちゃんとgit管理下に置いておく
* **差分がない（コミット済みで変更なし）と何もしない**。`git diff`で差分があることを確認してから実行をする
* `/security-review` という派生コマンドもあり、セキュリティに特化したレビューをしてくれる（Part13で詳述）

## `/permissions` ― Claudeが実行できることを管理する

### ①何をするか

Claudeが自動で実行できるコマンドやツールの許可リストを管理します。「毎回確認が来て面倒」なコマンドを事前に許可しておけます。

### ②実際の挙動

![](https://static.zenn.studio/user-upload/fcffc1ae642f-20260628.png)  
現在の許可設定の確認・追加・削除ができます。許可されていないコマンドはClaude実行前に確認ダイアログが表示されます。  
Allow, Ask, Denyなど、それぞれの項目をここで管理できます。

### ③活用方法・つまずいたポイント

* **許可しすぎると思わぬコマンドが自動実行される**リスクがある。特に `rm` や `git push` などは慎重に
* 設定は `settings.json` に保存されるので、チームで共有する場合は `.gitignore` との兼ね合いに注意
* 詳しいパーミッション設計はPart12で解説します

---

## `/mcp` ― 外部ツールとの接続を管理する

### ①何をするか

MCP（Model Context Protocol）サーバーの接続状況を確認・管理します。GitHubやSlackなどの外部サービスをClaudeに接続するための設定です。

### ②実際の挙動

![](https://static.zenn.studio/user-upload/3ebe6d465f1a-20260628.png)  
現在接続されているMCPサーバーの一覧と状態が表示されます。接続が切れているものはここで確認できます。

### ③活用方法・つまずいたポイント

* **MCPサーバー自体の設定は `settings.json`** で行う。`/mcp` は確認コマンドという位置づけ
* MCPの詳しい設定方法はPart8・9で解説します

---

## `/usage` ― 今の使用量を確認する

### ①何をするか

現在のセッションで消費したAPIトークン数とコストの概算を表示します。「気づいたら使いすぎてた」を防ぐための確認コマンドです。

### ②実際の挙動

![](https://static.zenn.studio/user-upload/ef4fc94cc647-20260628.png)  
入力・出力トークン数と、それに対応するコストの見積もりが表示されます。

### ③活用方法・つまずいたポイント

* **表示されるのはあくまで概算**。正確な請求額はAnthropicのコンソールで確認を
* コストが気になるなら、軽い作業は `/model` でHaikuに切り替えるのが効果的
* 長いセッションはコストが積み上がるので、タスクが一段落したら `/clear` や `/compact` を活用する

## まとめ

| コマンド | こんなときに使う |
| --- | --- |
| `/init` | 新プロジェクト開始時・CLAUDE.mdを作りたいとき |
| `/memory` | 繰り返し伝えている情報を覚えさせたいとき |
| `/model` | 重い作業・軽い作業でモデルを使い分けたいとき |
| `/compact` | 会話が長くなってきてコンテキストを節約したいとき |
| `/clear` | 別タスクに切り替えるとき・Claudeをリフレッシュさせたいとき |
| `/review` | コミット前にサクッとコードチェックしたいとき |
| `/permissions` | 毎回確認が来て面倒なコマンドを許可したいとき |
| `/mcp` | 外部ツールの接続状況を確認したいとき |
| `/usage` | コストが気になったとき |

次回はPart2で、定期的に使う中級コマンドを紹介します。
