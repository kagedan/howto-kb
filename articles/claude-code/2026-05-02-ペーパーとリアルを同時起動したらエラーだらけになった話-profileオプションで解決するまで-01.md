「ペーパーとリアルを同時に動かせますか？」

私もペーパートレードが安定してきたとき、真っ先にそう思いました。リアルに移りたいけれど「いきなりリアルだけ」に切り替えるのは怖い。ペーパーで動かしながらリアルの動作も確認できれば安心なのに、という気持ちです。4/13の朝、Claude Codeに依頼したその直後からエラーが出始めました。

結果として、試みた瞬間にエラーが出て、そこから3日かけてちゃんと解決しました。

---

## 何が起きたか

4/13、ターミナルを2つ開いてそれぞれで起動しようとしました。

```
# 端末1
python auto_trader.py  （ペーパー）

# 端末2
python auto_trader.py  （リアル予定）
```

端末2の起動直後からエラーが出始めました。

```
ERROR: positions.json の読み込みに失敗しました
ERROR: JSONDecodeError: Expecting value: line 1 column 1 (pos 0)
```

最初は「どちらかが書き込み中にもう一方が読もうとしているせいだ」と思いました。ただ、エラーの内容を確認していくと、問題はファイルの競合だけではありませんでした。

---

## エラーの内容を整理する

エラーが複数種類あって、最初は混乱しました。整理するとこうです。

**問題① positions.jsonの競合**

ペーパーとリアルが同じ `positions.json` を読み書きしていた。ペーパーが書いたポジション情報をリアルが読んで、存在しないリアル注文として扱おうとしていた。

**問題② trade\_log.csvの混在**

ペーパーの取引とリアルの取引が同じCSVに混ざる。後で損益を集計したときに「この取引はペーパー？リアル？」の区別がつかなくなる。

**問題③ configが1つしかない**

ペーパートレードでは資金上限を50万円に設定していましたが、リアルトレードでは初期資金を少なく設定したかった。でも [config.py](http://config.py) は1ファイルしかなく、どちらかに合わせると片方がズレる状態でした。

**問題④ ログが混在する**

`logs/2026-04-13.log` という1ファイルに両方のログが混ざった。どちらの処理によるログか追いかけるのが難しくなりました。

正直に言うと、「2窓起動するだけ」のつもりが、設計の根本的な問題に気づかされた感じでした。

---

## 原因はシンプルだった

原因を一言で言うと「全ファイルが1種類しかなかった」ことです。

開発当初はペーパートレードだけを動かすことを前提に設計していたので、config.pyも、positions.jsonも、trade\_log.csvも、ログファイルも、すべて1つだけ存在する設計でした。

2つのモードを同時起動する想定がなかったので、こうなっていたのは当然といえば当然です。
