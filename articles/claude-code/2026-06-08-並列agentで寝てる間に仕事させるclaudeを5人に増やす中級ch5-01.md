仕事を頼むとClaudeはこう返してくる。

「記事5本、書くのに30分かかります」

夜中の2時。朝の会議まであと数時間。待ってられない。

そこで効くのが **並列Agent** だ。Claude Codeは自分の「分身」を同時に何体も動かせる。1体が30分かけてやる仕事を、5体で並行して走らせれば10分弱で片付く。

## Agentとは「Claudeの部下」

Claude Codeで「Agent」と言うときは、メインのClaudeとは別に動く子Claudeのことを指す。メインClaudeがプロジェクトマネージャーなら、AgentはそのPMが立てた部下たちだ。

---

**自分**：「営業メール5社分、同時に書いて」

**Claude（メイン）**：「了解。5社それぞれにAgentを1体ずつ立てて並列で走らせる」

**5体のAgent**：（同時並行で作業中）

**Claude（メイン）**：「全員終わった。5本分まとめて出すよ」

---

メインは指揮、Agentは現場。これが基本構造だ。

## 直列と並列、何が違うか

**直列（普通の頼み方）**

> 「記事5本書いて」

1本目（6分）→ 2本目（6分）→ 3本目（6分）→ 4本目（6分）→ 5本目（6分）  
合計**30分**

**並列（Agentを使う頼み方）**

> 「記事5本、それぞれ別のAgentで同時に書いて」

Agent1（6分）／Agent2（6分）／Agent3（6分）／Agent4（6分）／Agent5（6分）が一斉に走る  
合計**6〜10分**

時間が約5分の1になる。これが並列の威力だ。

## 実用シーン3つ

### シーン1：複数社の営業メールを同時に書く

> 「営業リスト99社、3体のAgentで分担してメール文を書いて」

メインClaudeが「Agent1にA〜33社、Agent2にB〜66社、Agent3にC〜99社」と振り分ける。3体が同時並行で書き始めるから、99社を順番に書くより一気に短縮できる。

その間、自分は別の作業をしていればいい。

### シーン2：複数ファイルを同時に調査

> 「このフォルダの10個のCSVファイル、それぞれ別のAgentで解析して問題点を出して」

10ファイルを順番に読ませると重いし時間もかかる。並列なら、ファイル数ぶんのAgentが同時に読んで分析する。終わったらメインがまとめて返してくれる。

### シーン3：A/Bパターンの同時生成

> 「このLP、Agent1は『若年層カジュアル調』、Agent2は『経営者向けフォーマル調』で同時に書いて」

トーンの違う2バージョンが同時に出てくる。並べて見比べて、よかった方を採用する。1本ずつ書かせて後から見比べるより、明らかに速い。

## 並列を起こすためのコツ：独立したタスクに分ける

並列Agentが効くのは、**互いに依存していないタスク**のときだけだ。

**ダメな例**

```
Agent1：社名を確認する
   ↓ ※これが終わらないと
Agent2：その社名でメールを書く
```

Agent2はAgent1の結果を待たないと動けない。これでは並列の意味がない。

**良い例**

```
Agent1：A〜33社のメールを書く
Agent2：B〜66社のメールを書く
Agent3：C〜99社のメールを書く
```

3体とも他のAgentの結果を待たない。完全に独立しているから、本当に並列で動ける。

「分けた仕事が、それぞれ単独で完結するか？」——これを並列を頼む前にチェックする。

## バックグラウンド実行で「寝てる間」を活用する

並列Agentに頼んだら、進捗をずっと眺めている必要はない。**バックグラウンド実行**にしておけば、Claude Code自体は別の話題に進めるし、PCを閉じてもいい設定もできる。

Claude Codeには走っているAgentの一覧を確認するUIがあって、状況をいつでも見に行ける。

```
[Agent-1] 営業メール（A群）    完了
[Agent-2] 営業メール（B群）    進行中 60%
[Agent-3] 営業メール（C群）    進行中 35%
```

こんな感じで指揮塔から見渡せる。深夜に走らせて朝確認、というワークフローが現実的になる。

## 注意：コストも並列にかかる

楽になる代わりに、忘れちゃいけないのが**コスト**だ。

5体のAgentを同時に動かすと、APIの使用量も5倍になる。30分の作業を6分に短縮する代わりに、料金は5倍に増える。**「時間を金で買う」のが並列Agentの正体**だ。

判断軸はシンプル。

* 締め切りに追われていて時間が一番大事 → 並列でOK
* 急いでない、コスト重視 → 直列で十分

「全部並列で爆速だぜ」とノリで使うと月末の請求書で泣くことになる。ここぞの場面で使う、と決めておくと安全だ。

## 定期実行で「自分が動かなくても進む」を作る

並列Agentと組み合わせて効くのが、定期実行系の機能だ。Claude Codeには **`/loop`**（一定間隔で繰り返し実行）や **`/schedule`**（指定時刻に実行）といったコマンドがある（バージョンによって挙動が違うので、`/help` で自分の環境を確認してほしい）。

これを使えば、

* 5分おきに営業リストを更新する
* 毎朝9時に競合のメディアを巡回する
* 1時間ごとにエラーログをチェックする

といった「自分が指示を出さなくても勝手に進む」運用が組める。並列Agent×定期実行で、寝ている時間も働き続けるシステムができる。

## まとめ——並列Agentは「時間を圧縮する」道具

|  | 直列 | 並列Agent |
| --- | --- | --- |
| 5本の記事生成 | 約30分 | 約6〜10分 |
| 10ファイルの解析 | 約100分 | 約10〜20分 |
| コスト | 通常 | タスク数倍 |

並列Agentは魔法ではなく、**時間を金で買う仕組み**だ。日常で気軽に使うものではないが、ここぞの締め切り、夜中に山積みのタスク、明日朝までに揃えたい資料——そういう局面ではこれ以上ないくらい頼りになる。

次章は最終回、**コンテキスト管理とコスト最適化**。「並列で動かしたら月末に泣いた」を起こさないための、地味だけど最重要な話だ。
