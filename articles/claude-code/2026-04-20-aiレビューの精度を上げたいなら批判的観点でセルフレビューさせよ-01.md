## 背景

AIによるPRレビューの指摘を前にして、こんなことを感じたことはないでしょうか。

* 「この指摘、本当に対応すべきだろうか？」
* 「好みの問題では？」

## AIレビューの残課題

以前、Claude Code のカスタム Skills で PR レビュー基盤を作った話を書きました。

<https://zenn.dev/zozotech/articles/93d318a99d329e>

本記事では、1パス目で網羅的に指摘を出し、2パス目で「本当に重要な指摘か」を批判的に検証して厳選する、という2パス構成を紹介しました。  
指摘の質は明確に上がったのですが、運用を続けると別の課題が見えてきました。

AIがPRをレビューしてくれることはすごく便利なのですが、1つ1つの妥当性を人間が判断する工程が大変です。  
**この吟味コストを減らすために、批判的観点でのコメントレビューを実施させてみました。**

## 批判的観点のセルフレビュー

取り組み当初は、「レビューコメントをセルフレビューして、必要なものだけを残して」と指示してみました。  
ただし、**LLM は一度出した判断を正当化する方向** に動きやすく、期待した成果は得られませんでした。

そこで方針を変え、**セルフレビューを **「批判的観点」** で実施させ、その結果をコメントに併記させる**ようにしました。

```
[AI]
**[<severity>]** <タイトル>

<本文>

---

> **批判的観点**: <この指摘に対する自己批判>
```

## 結果

実際の出力例です。  
批判的観点が指摘の最後に入ることで、指摘の妥当性を素早く判断できるようになりました。  
プロジェクト独自の慣習に関する批判的指摘などを指摘してくれることが多く、AIレビューの確認時間がかなり削減されたことを実感しています。

![](https://static.zenn.studio/user-upload/586228e23a69-20260420.png)

## まとめ

批判的レビューのコメントを追加するだけで、AIレビューのチェック負荷が明確に下がりました  
普通のセルフレビュー指示だけでは効果が出きらないと感じている方は、是非試してみてください。
