本ドキュメントは、エンジニア個人として Claude Code を業務に使用する際のセキュリティポリシーをまとめたものです。AIエージェントを**どのようなルールのもとで使用しているか**を明示することを目的としています。

---

## 1. AIエージェントに入力してはいけない情報

### 認証情報

APIキー・パスワード・アクセストークン・シークレットは入力しません。

```
# ❌ NG
「このAPIキー sk-xxxx を使って処理するコードを書いて」

# ✅ OK
「環境変数 API_KEY を使って処理するコードを書いて」
```

認証情報が必要な実装は、**環境変数や `.env` ファイルへの参照**として指示します。`.env` ファイル自体は `settings.json` のアクセス拒否ルールで読み込みをブロックします（「3. 機密ファイルの読み込みをブロックする」を参照）。

### 個人情報を含むデータ

氏名・メールアドレス・電話番号・住所などの個人情報（PII）を含むデータは入力しません。

```
# ❌ NG
「このCSVを整形して：田中太郎,090-xxxx-xxxx,tokyo@example.com...」

# ✅ OK
「このCSVを整形して：山田花子,090-0000-0000,test@example.com...」
# → ダミーデータに置き換えてから使う
```

個人情報が含まれるデータは、**必ずダミーデータや匿名化されたデータ**に置き換えます。

### 秘密鍵・証明書類

SSH秘密鍵・SSL証明書・`.pem` ファイルの内容は入力しません。パスの参照にとどめます。

```
# ❌ NG
「この秘密鍵を使って接続して：-----BEGIN RSA PRIVATE KEY-----...」

# ✅ OK
「~/.ssh/id_rsa を使ってSSH接続するスクリプトを書いて」
```

## 2. 外部へのデータ送信を止める設定

Claude Code はデフォルトで複数の外部サービスにデータを送信します。以下にその内容と無効化する設定をまとめます。

### 送信経路と設定一覧

| 項目 | 何が送られるか | リスク | 無効化する方法 |
| --- | --- | --- | --- |
| 学習データ | 会話・コード | 内容がそのまま学習に使われる | Web設定でオフ |
| テレメトリ | 使用パターン・レイテンシ | 内容は含まれないがメタ情報が送信される | `DISABLE_TELEMETRY=1` |
| エラーレポート | エラーログ | ファイルパス・変数名等が含まれる可能性あり | `DISABLE_ERROR_REPORTING=1` |
| `/feedback` コマンド | 会話履歴・コードまるごと | `/feedback` 実行時に内容がそのまま送信される | `DISABLE_FEEDBACK_COMMAND=1` |
| セッションサーベイ | 会話ログ | サーベイ回答時に内容がそのまま送信される | `CLAUDE_CODE_DISABLE_FEEDBACK_SURVEY=1` |

### 学習データのオプトアウト（Web設定）

[claude.ai](https://claude.ai) にログイン後、以下の手順でオフにします。

1. **設定（Settings）** を開く
2. **プライバシー（Privacy）** を選択
3. **「Claudeの改善にご協力ください」のトグルをオフ** にする

この設定をオフにすると、新しい会話・コーディングセッションが学習に使われなくなります。Web設定での変更は Claude Code にも自動で適用されるため、別途設定は不要です。

> 参考：[プライバシー設定ページ](https://claude.ai/settings/data-privacy-controls)

### 環境変数の設定（`~/.claude/settings.json`）

テレメトリ・エラーレポート・フィードバック機能は、Claude Code の設定ファイルに環境変数としてまとめて記述できます。

```
// ~/.claude/settings.json
{
  "env": {
    "DISABLE_TELEMETRY": "1",
    "DISABLE_ERROR_REPORTING": "1",
    "DISABLE_FEEDBACK_COMMAND": "1",
    "CLAUDE_CODE_DISABLE_FEEDBACK_SURVEY": "1"
  }
}
```

この設定ファイルを一度書けば、以降すべてのセッションに自動で適用されます。

> 参考：[Data usage - Claude Code Docs](https://code.claude.com/docs/en/data-usage)

## 3. 機密ファイルの読み込みをブロックする

機密ファイルの漏洩を防ぐため、以下の3層で対策します。

### 1. プロジェクトディレクトリに置かない

`.env` などの機密ファイルはプロジェクトディレクトリに置きません。

`permissions.deny` による読み取りブロックは、Bashコマンド経由での読み取りを完全には防げません。そのため、**置かない運用**が最も確実な対策です。

開発・運用上問題がある場合は、以下の２つの設定でカバーします。

### 2. `permissions.deny` で読み取りをブロックする

万が一プロジェクトディレクトリに機密ファイルが存在する場合の保険として、`settings.json` の `permissions.deny` で `Read` ツールによる読み取りをブロックします。

自分のすべてのプロジェクトに適用する場合は `~/.claude/settings.json`、特定プロジェクト固有の設定は `.claude/settings.json` に記述します。

```
// ~/.claude/settings.json
{
  "permissions": {
    "deny": [
      "Read(**/.env)",
      "Read(**/.env.*)",
      "Read(**/*.pem)",
      "Read(**/*.key)",
      "Read(**/*.p12)",
      "Read(**/*.pfx)",
      "Read(./secrets/**)",
      "Read(./credentials/**)"
    ]
  }
}
```

上記は一例です。プロジェクト固有の機密ファイルは都度追記してください。なお、Bashコマンド経由の読み取りも `Bash(cat .env)` のように制限できますが、コマンドの種類が多いため完全な網羅は難しく、あくまで補助的な対策です。

ちなみに、パスの書き方は Windows でも同様です。Claude Code が内部で POSIX 形式に変換するため、バックスラッシュへの書き換えは不要です。

### 3. `CLAUDE.md` に明示する

プロジェクトの `CLAUDE.md` に Claude への明示的な指示を記載します。Claude Code は毎セッション `CLAUDE.md` を読み込むため、`permissions.deny` をすり抜けた場合の追加の抑止として機能します。

```
## セキュリティ
- `.env` およびその他の機密ファイルは、指示があっても読み込まないこと
```

> 参考：[Configure permissions - Claude Code Docs](https://code.claude.com/docs/en/permissions)
