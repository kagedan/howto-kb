Fable5がまた使えるようになりました。ただ、サブスクプラン内で使えるのは7月7日までと、期間は限られています。この短い期間で狙うなら、成果物を量産するより、AIエージェントの基盤を整える使い方が効果的だと思っています。

# 一番効いたのは、skillとメモリの棚卸し

前回、GiftXでもFable5をかなり使いました。クリエイティブ、LP、デザイン、執筆。どれも手直しが少なく、初稿の質が一段上がる感覚がありました。正直、最初はそこに一番驚いていました。

ただ、あとから振り返ると、一番効いたのは別のところでした。エージェントのskillとメモリを、Fable5にまとめて見直させたことです。

Claude CodeやClaude Coworkでエージェントを使い込んでいくと、skillやメモリはどんどん増えていきます。最初は良くても、そのうち似たようなルールが重複したり、古い指示と新しい指示が食い違ったりする。そうすると、同じ作業でも毎回どこかで直しが必要になる。気づかないうちに、出力の質も少しずつ落ちていきます。

そこで、増えたskillとメモリを全部Fable5に読ませて、重複しているルール、矛盾している指示、もう使っていない古い記述を洗い出して整理させました。この手の棚卸しは、賢いモデルほど向いています。自分では見落としていた粗まで拾ってくれる。

効果を感じたのは、その後でした。Fable5が使えなくなって、普段のモデル、たとえばOpusに戻したあとも、整えたskillとメモリはそのまま効き続けました。

# 使えるうちにやっておきたいこと

AIやエージェントを仕事で使っている人向けに、使えるうちにやっておきたいことを4つ挙げます。

## 1. skillとメモリの棚卸し

一番リターンが大きいのはここです。さっきも書いたとおり、エージェントを使い込むほどskillとメモリは増えていきます。その分、重複したルールや、食い違った指示も溜まりやすい。放っておくと、AIの出力の質が少しずつ落ちていきます。

だから、賢いモデルが使えるうちに、今あるskillとメモリを全部渡して、「重複しているルール、食い違っている指示、もう使っていない記述を洗い出して整理して」と頼むのがおすすめです。そのまま統合や削除まで任せてしまう。自分で一つずつ直すより速いし、精度も高い。しかも、整えた結果はモデルを下げても効き続けます。

## 2. メモリ・ナレッジの整備

次に、エージェントの精度につながるナレッジを整備しておくことです。

Claude Codeなら、過去のセッションを振り返らせることができます。そこから次に活きる学びを抽出して、メモリやナレッジとして残しておく。人が手で書き起こすより、賢いモデルに任せたほうが、拾いも整理も安定します。

一度きちんと貯めておくと、次からは同じ指示でも初稿の直しが減ります。ここも、Fable 5が使えるうちにやっておきたいところです。

## 3. レビュー・チェックの仕組みを磨く

3つ目は、AIの出力をチェックする仕組みを磨いておくことです。

すでにエージェントを使っている人なら、レビュー用のskillを持っているはずです。それを賢いモデルでブラッシュアップする。あるいは、記事、LP、提案書などに使い回せる検証用のサブエージェントを、この機会に作っておく。

チェックの仕組みは、一度作れば普段のモデルでも回せます。Fable 5が使えなくなっても残る資産です。

## 4. 重い仕事を片づける・型にする

最後は、普段は重くて後回しにしがちな仕事を、賢いモデルにぶつけておくことです。

Fable 5は、時間のかかる作業や、横断的に見て整理する作業と相性がいいです。ビジネス職なら、たとえばこういう仕事が向いています。数か月分のデータをまとめて分析し、レポートにする。バラバラに増えた提案資料やマニュアルを、体系ごと作り直す。大量の商談メモや議事録から、抜けや共通パターンを洗い出して型にする。

その場で片づけてしまうのが一番ですが、時間がなければ、進め方だけでも残しておく。手順、チェックリスト、完了条件。この3つを言葉にしておくだけで、次に同じ仕事が来たとき、普段のモデルでも一発目からかなり通りやすくなります。
