📝Claude Codeの「使用量が急増する問題」、Anthropicの中の人から調査結果と対策方法が出ました

結論から言うと、バグではなく「私たちの使い方」が原因だったようです

簡単に言うと、コンビニに行くのにアクセルベタ踏みで毎回フェラーリに乗って行くなってこと

非エンジニアの方でもすぐに対策できるので、自分用メモとしてまとめておきます

・原因は「1M（100万）トークン」という巨大な記憶領域を使いすぎていること
・システム側の過剰請求バグはすでに修正済み
・対策①：モデルを「Sonnet 4.6」に固定する（Opusは消費量が約2倍）
・対策②：深い推論（effort level）を下げるかオフにする
・対策③：1時間以上放置した作業は、再開せずに「新しいセッション」を立ち上げる
・対策④：環境変数で記憶領域の上限を制限する（CLAUDE_CODE_AUTO_COMPACT_WINDOW=200000）
・対策⑤：常に最新版にアップデートしておく

特に「放置した作業の再開」は、過去の記憶をすべて読み直すため一気にトークンを消費するそうです。
これを知っているだけで、使用量は劇的に抑えられます。

もしこれらを試しても「明らかにおかしい」と感じたら、作業画面で「/feedback」と入力すれば、Anthropicが個別に調査してくれます。

AIツールは進化が早い分、こういった「賢い使い方」を知っているかどうかが、そのままコストに直結する時代ですね。
