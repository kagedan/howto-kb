Claude Codeを使い始めると、まずCLAUDE.mdを作りますよね。プロジェクトの説明やコーディング規約をざっと書いて、「よし、これでOK」と。

ただ、CLAUDE.mdは書き方ひとつで応答精度がかなり変わります。公式ドキュメントを読み込むと、使いこなせていない機能がいくつも見つかるので、すぐ実践できるものを7つに絞りました。

## 1. /init で叩き台を自動生成する

白紙から書き始める必要はありません。Claude Codeのセッション内で `/init` を実行すれば、リポジトリを解析してビルドコマンド・テスト手順・規約を自動で拾い上げてくれます。

```bash
# Claude Code セッション内で
/init
```

すでにCLAUDE.mdがある場合は上書きではなく改善提案になるので、既存プロジェクトでも安全です。

:::note info
環境変数 `CLAUDE_CODE_NEW_INIT=1` を設定すると、CLAUDE.md・Skills・Hooksのどれを作るか対話形式で選べるようになります。
:::
