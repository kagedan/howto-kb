## Claude Codeに「ダイナミックワークフロー」が登場

Claude Code（リサーチプレビュー）に、複雑な大規模タスクに対応するための新機能\*\*「ダイナミックワークフロー（Dynamic Workflows）」\*\*が追加されました。

## ダイナミックワークフローとは？

通常のエージェントループでは処理しきれないほど大きなタスクに対応するための仕組みです。

* Claudeがオーケストレーションスクリプトをその場で自動生成
* 大量のサブエージェントを並列起動し、協調させながらタスクを処理
* プロンプトに **「workflow」** というワードを含めるだけで起動可能

活用シーンの例：

* サービス全体にまたがるバグ調査
* 大規模なコード移行・マイグレーション
* リファクタリング
* パフォーマンス最適化
* アーキテクチャ設計のストレステスト

## Salesforceの導入事例：231日分の作業が13日で完了

Salesforceがアジェンティックな開発フローにClaude Codeを導入した詳細なレポートを公開しており、その成果は驚くべきものでした。

| 指標 | 結果 |
| --- | --- |
| 大規模マイグレーションの所要期間 | 当初見積もり **231日 → 実績 13日** で完了 |
| 単一PRでのエンドポイント実装数 | **21エンドポイント、テストカバレッジ100%** |
| インシデント数の変化 | PR出荷数増加にもかかわらず **5%減少** |

## 「生産性 vs 品質」はトレードオフではない

AIエージェントを活用すると「スピードが上がる代わりに品質が落ちる」と思われがちですが、Salesforceのケースはそれを否定しています。

アウトプット量の増加と同時に、品質も向上。これは、**セキュリティのガードレールや品質基準をアジェンティックなワークフロー自体に組み込んだ**ことが大きな要因とされています。

## 大規模移行・リファクタリングの「先送り問題」を解決

大規模な移行作業やリファクタリングは、チームにとって重要な仕事でありながら、「エンジニアを1四半期拘束してしまう」という理由で先送りにされがちです。

ダイナミックワークフローを使えば、そうした作業を**数日〜数週間**で完了させることが可能になります。「良いタイミング」を待たずに、今すぐ技術的負債に着手できる時代になりました。
