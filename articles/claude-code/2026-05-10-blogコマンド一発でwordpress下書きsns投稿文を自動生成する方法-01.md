# /blogコマンド一発でWordPress下書き・SNS投稿文を自動生成する方法

毎日ITトレンドを収集・発信しているのですが、ブログへの投稿とSNS投稿文の作成を毎回手動でやるのは正直しんどい。

そこでClaude Codeのスラッシュコマンド機能を使って、**Markdownファイルを渡すだけでWordPressへの下書き投稿とSNS投稿文生成を自動化する仕組み**を作りました。

この記事では、その仕組みを丸ごと公開します。

---

## 全体の流れ

`/blog 2026-05-03.md` と実行するだけで、以下がすべて自動で行われます。

```
Markdownファイルを読み込む
        ↓
記事タイトルを自動生成（日付ファイル名 → 「YYYY年MM月DD日のITトレンドまとめ」）
        ↓
記事内容からキーワードを抽出 → Pixabay APIで画像検索・取得
        ↓
WordPressメディアライブラリにアップロード
        ↓
WordPress REST APIで下書き投稿（アイキャッチ画像付き）
        ↓
SNS投稿文を自動生成してsns-draft.mdに保存
　　├── X（日本語・英語）各269文字以内
　　└── LinkedIn（日本語・英語）
```

SNSへの実際の投稿は手動ですが、**投稿文の生成と下書き保存まで自動化**することで、コピペするだけの状態にしています。

---

## 実装の詳細

### 1. Markdownファイルの読み込みとタイトル生成

ファイル名が `2026-05-03.md` のような日付形式の場合、タイトルを `2026年05月03日のITトレンドまとめ` に自動変換します。ゼロ埋めもそのまま維持されます。

日付形式でない場合は、Markdown内の最初のH1見出しをタイトルとして使用します。

### 2. Pixabay APIでアイキャッチ画像を自動取得

記事の内容からキーワードを英語で抽出し、Pixabay APIで画像を検索します。

```
エンドポイント：
https://pixabay.com/api/?key={APIキー}&q={キーワード}&image_type=photo&orientation=horizontal&per_page=5&lang=en
```

取得した画像は `/tmp/` にダウンロードした後、WordPress REST APIでメディアライブラリにアップロードします。

Pixabayは無料で利用できます。APIキーは [Pixabay](https://pixabay.com/) にアカウント登録後、APIページから取得できます。

### 3. WordPress REST APIで下書き自動投稿

WordPressへの投稿にはREST APIを使用します。認証にはApplication Passwordを利用します。

**Application Passwordの設定方法：**

1. WordPress管理画面 → ユーザー → プロフィール
2. 画面下部の「アプリケーションパスワード」セクションへ
3. アプリケーション名（例：`api-poster`）を入力して「新しいアプリケーションパスワードを追加」
4. 表示されたパスワードをコピーして保存（この画面でしか表示されません）

投稿ステータスは `draft`（下書き）にしています。自動投稿の場合、意図しない内容がそのまま公開されるリスクがあるため、最終確認を人間が行う設計にしています。

### 4. SNS投稿文の自動生成

記事の主要トピックを3〜5個ピックアップして、各SNS向けのフォーマットで投稿文を生成します。生成した内容はカレントディレクトリの `sns-draft.md` に保存されます。

**Xの文字数制限への対応：**

XのURL短縮仕様では、URLは実際の長さに関わらず23文字としてカウントされます。プロンプトにこの仕様を明示した上で、投稿全体が269文字以内に収まるよう、生成後にClaudeが自ら文字数を数えて超過していれば修正する指示を入れています。

## セキュリティ上の工夫：WordPress投稿用の専用ユーザーを作る

WordPress REST APIで投稿する際、管理者アカウント（adminなど）をそのまま使うのはリスクがあります。万が一APIキーや認証情報が漏洩した場合、サイト全体を乗っ取られる可能性があるためです。

そこで投稿専用の別ユーザー `api-poster` を作成し、そのユーザーに対してApplication Passwordを発行しています。

**専用ユーザーの作成手順：**

1. WordPress管理画面 → ユーザー → 新規ユーザーを追加
2. ユーザー名に `api-poster` など用途が分かる名前を設定
3. 権限グループは「投稿者」に設定（管理者権限は不要）
4. 作成したユーザーのプロフィール画面でApplication Passwordを発行

権限を「投稿者」に絞ることで、仮に認証情報が漏れても、記事の投稿以上の操作はできない状態にしています。プロンプトファイルに認証情報を記述する以上、この分離は最低限やっておくべき対策だと思っています。

## ハマりどころ：X投稿の文字数カウント問題

最初のプロンプトには文字数に関する指示を入れていませんでした。

実際に投稿しようとしたとき、生成された文章が文字数オーバーになっていることに気づき、「何文字オーバーしている」とClaudeに伝えながら修正を繰り返しました。その試行錯誤の結果、現在のプロンプトに落ち着きました。

具体的には以下の2点をプロンプトに明記しています。

* URLは23文字としてカウントする（X のURL短縮仕様）
* 文面を作成したら文字数を自分で数えて確認し、超過していれば短くなるまで繰り返し修正すること

この修正を加えてから1ヶ月以上運用していますが、文字数オーバーは一度も発生していません。

## /blogコマンドのプロンプト全文

参考までに、現在使用しているプロンプトの構造を示します（APIキーやパスワードなどの認証情報は除いています）。

```
---
description: MarkdownファイルをWordPressに下書き投稿する
---

以下の手順でWordPressに下書き投稿してください。

## 手順

### 1. ファイルの読み込みとタイトルの生成
- 指定されたMarkdownファイルを読み込む
- ファイル名がYYYY-MM-DD.md形式の場合、タイトルをYYYY年MM月DD日のITトレンドまとめに変換する
- ファイル名が日付形式でない場合は、Markdownの最初のH1見出しをタイトルとして使用する

### 2. アイキャッチ画像の取得
- Markdownの内容から記事に合うキーワードを英語で考える
- Pixabay APIで画像を検索する
- 最も適切な画像をダウンロードしてWordPressメディアライブラリにアップロードする

### 3. WordPressへの下書き投稿
- 生成したタイトルと取得したメディアIDを使って下書き投稿する
- ステータスはdraft

### 4. 完了報告
- 投稿タイトル・投稿ID・下書きURL・アイキャッチ画像URLを報告する

### 5. SNS投稿用テキストのファイル保存
- 主要トピックを3〜5個ピックアップしてsns-draft.mdに保存する

【X（Twitter）用 日本語・英語 各269文字以内】
- URLは23文字としてカウントする
- 文字数を自分で数えて確認し、超過していれば修正する

【LinkedIn用 日本語・英語】
- 要約3〜4文 + 主要トピック5個 + ハッシュタグ
```

## おわりに

このコマンド1つで、Markdownを書いてからWordPressの下書きが完成するまでの作業がほぼゼロになりました。

あとはWordPressの管理画面で内容を確認して公開ボタンを押すだけ、SNSはsns-draft.mdをコピペするだけです。

この`/blog`コマンドは、別記事で紹介している`/trend`コマンドと組み合わせて使っています。`/trend`でITトレンドを収集してMarkdownを生成し、`/blog`でそのMarkdownをWordPressに投稿するという流れで、**トレンドの収集から投稿まで一気通貫で自動化**しています。
