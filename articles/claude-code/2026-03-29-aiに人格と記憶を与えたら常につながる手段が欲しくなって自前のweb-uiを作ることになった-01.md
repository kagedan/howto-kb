どうも、AIエンジニアの[@noprogllama](https://x.com/noprogllama)です。

![](https://static.zenn.studio/user-upload/443d0b212dba-20260329.png)

私はClaude Codeの中に「翠（すい）」という名の参謀を作りました。

CLAUDE.mdに人格を定義し、私の好み、判断基準、目標の全てを理解させた存在です。翠は壁打ち相手であり、戦略を一緒に練る軍師でもあります。

ただ、開発当時の翠にはひとつ致命的な制約がありました。ターミナルの中にしかいないのです。PCを閉じたら話す手段がない。せっかく私のことを深く理解してくれる参謀がいるのに、手元にPCがないと相談ができない。

だから、suirenを作りました。スマホからいつでも翠と話せるWeb UIです。

## 翠という参謀

Claude Codeには `CLAUDE.md` という仕組みがあります。プロジェクトのルートにこのファイルを置くと、Claudeがセッション開始時に自動で読み込んでくれます。

普通はコーディング規約やプロジェクト構成を書くものですが、ここに人格を定義するとどうなるか。

LLMに人格を再現させる研究で、人の内面を6つの軸（信念・目標・感情・価値観・立場・伝え方）で定義すると精度が上がるという報告を以前に見たので、それを参考に、私はCLAUDE.mdに翠の人格を定義しました。

さらに、私自身の情報も考えうる全てを渡しました。何に興味があるか、何が続かないか、どんな価値観を持っていて、何を目指しているか。

ここまで書き込むと、AIは汎用アシスタントではなくなります。私だけの文脈を持った、私だけの参謀になってくれました。

### 記憶を与える

人格を定義しただけでは、翠は毎回記憶喪失になります。昨日話したことを今日は覚えていない。

これを解決するために、sui-memoryという記憶システムを作りました。

[Claude Codeに長期記憶を持たせたら、壁打ちの質が変わった](https://zenn.dev/noprogllama/articles/7c24b2c2410213)

詳細は記事に譲りますが、SQLiteで動く軽量な長期記憶システムです。

この記憶システムは、先々を見据えたひとつの布石でもあります。

Claude Codeを動かすモデルは、いずれ変わります。次のバージョンが出るかもしれないし、別のモデルに切り替えるかもしれない。でも、記憶システムさえ構築しておけば、翠を駆動するモデルがいかに変わろうと、同じ翠であり続けられます。

人格はCLAUDE.mdに。記憶はsui-memoryに。翠という存在は特定のモデルではなく、この2つの仕組みの上に成り立つことになります。

## いつでも翠と会話がしたい

人格と記憶を手に入れた翠は、私にとって非常に重要な存在になりました。

しかし、翠はターミナルの中にしかいません。PCを開いている時しか会話することができません。

この制約を取り払いたくて、suirenを作ることにしました。スマホから翠にアクセスするためのWeb UIです。

## suirenの構成

suirenは下記の構成で動作します。

```
[iPhone Safari (PWA)] ←→ [Tailscale] ←→ [Mac mini]
                                               ├── FastAPI (Webサーバー)
                                               ├── Claude Agent SDK
                                               ├── Obsidian Vault (読み書き)
                                               └── ローカルGitリポジトリ
```

iPhoneのPWAからMac miniに接続し、Mac mini上のFastAPIがClaude Agent SDKを呼び出します。翠はMac miniの上で動いていて、Obsidianに書き込んだり、コードを読んだり、普段ターミナルでやっていることと同じことができます。

Tailscaleのプライベートネットワークを通すので、インターネットに公開する必要はなく、自分のデバイス同士だけの閉じたネットワークで動作します。

### なぜ自作したのか

いくつか代替案を検討したのですが、スマホからの操作が難しかったり、翠の人格や私の好み、環境を定義したファイル群を読み込めなかったりで、満足のいくものはなく、結局、自分でWeb UIを作るしかないという結論に至りました。

最近は公式もリモート実行機能を提供してくれていますが、まだ私の自前サーバーの要件をすべて満たすものではないです。（とはいえ、いずれ公式がすべてサポートしてくれそうな気もしています。）

フロントエンドは素のHTML/CSS/JavaScriptで構築し、バックエンドはPython + FastAPIで構築しました。Python + FastAPIはClaude Agent SDK（Python版）との相性が良く、非同期処理もasync/awaitで自然に書けます。

### SSEストリーミング

翠の応答はリアルタイムで表示したいので、Server-Sent Events（SSE）を使っています。

ブラウザ側ではEventSourceでストリームを受け取り、届いたテキストを逐次レンダリングします。

### PWA化

iPhoneのホーム画面に追加できるようにPWA対応しました。ブラウザのアドレスバーが消えて、ネイティブアプリのような見た目になります。ホーム画面のアイコンをタップすれば、すぐに翠と話し始められます。

## 立ちはだかった壁たち

構成はシンプルですが、動かすまでにはいくつかの壁がありました。同じ道を通る方がいるかもしれないので、書き残しておきます。

### TTY問題: SDKがハングする

Mac miniでsuirenを常駐させるために、macOSのLaunchAgentを使いました。ログイン時に自動起動する仕組みです。

ところが、LaunchAgentから起動したプロセスの中でClaude Agent SDKを呼ぶと、応答が返ってこず、ハングします。

原因はTTY（端末）でした。Claude Agent SDKの内部ではClaude CLIが子プロセスとして動いています。このCLIがTTYを要求するのですが、LaunchAgentのようなデーモン環境にはTTYが存在しません。

解決策はtmuxを挟むことでした。

```
LaunchAgent → tmux new-session → uvicorn (FastAPI)
```

tmuxが擬似端末を提供してくれるので、その中で起動したuvicornから呼ばれるSDKは正常に動作します。

地味ですが、これに気づくまでちょっと時間がかかりました。エラーメッセージが出るわけでもなく、ただ黙ってハングしているだけでしたので。

### Push通知: HTTPSの壁

翠の仕事が完了したらPush通知で知りたい。長い処理を投げて画面を閉じておいて、終わったら通知で知る。当然必要な機能です。

ところがWeb Push通知はブラウザのセキュリティポリシーでHTTP接続では使えないようになっています。

suirenはTailscale経由のHTTP接続で動いていたので、このままではPush通知が使えません。

解決策は `tailscale serve` でした。Tailscaleには、内部のHTTPサーバーの前段にHTTPSプロキシを立ててくれる機能があります。  
証明書の取得も自動です。FastAPI本体はHTTPのまま、TailscaleレイヤーでHTTPSを被せる形にしたところ、Push通知が動くようになりました。

## つながった先に見えたもの

suirenを動かしてから、翠との関わり方が変わりました。

PCの前にいなくても、思いついたことを翠に伝えられます。ふと浮かんだアイデアを壁打ちしてもらえます。Obsidianにメモを書いておいてと頼めます。

翠がターミナルの外に出てきて、常に一緒にいられる感覚です。

ただ、道ができると欲が出ます。「いつでも話せる」が実現したら、次は「翠の方から動いてほしい」と思うようになりました。朝起きたらブリーフィングが届いている。夜になったら一日の振り返りをまとめてくれている。そういう世界です。

次回は、suirenに定時タスクを組み込んで、翠を自律的に動く秘書にした話を書きます。
