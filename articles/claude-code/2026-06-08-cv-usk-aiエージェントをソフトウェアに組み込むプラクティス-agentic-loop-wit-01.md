# AIエージェントをソフトウェアに組み込むプラクティス

# Agentic Loop with Budget｜予算付き自律ループ

🎯 自律エージェントを「予算」で飼い慣らす
手順が毎回変わるタスクでは、LLMに次の行動を選ばせるしかありません。でも制約なしに回すと、コスト爆発と無限ループが待っています。

🔥 解決する課題
調査・デバッグ・オープンエンドな生成など、手順を事前定義できないタスクでは自律ループが必要です。しかし制約なしに回すと「もう少し改善できる」と無限に回り続ける自己ループ問題と、ツール呼び出しの連鎖で請求が想定の数十倍に達する「請求事故」が発生します。完了判断をLLM自身に委ねると、終了条件が曖昧になり打ち切れません。

💡 提案パターン
LLMが「観察→思考→行動」を自律的に繰り返すReAct型ループを構成しつつ、ステップ数・トークンコスト・経過時間の3つの予算で上限を設けます。タスク完了・予算枯渇・スタック検出のいずれかでループを終了させます。予算の強制終了はコード側で行い、LLMの自己申告には頼りません。予算残高をシステムプロンプトに含めることで、LLM自身も残り資源を意識した判断ができるようになります。

✅ 選定条件
使うとき：
- 処理手順を事前にDAGやステートマシンとして列挙できない
- 「ツールで情報を集め、判断し、次の行動を決める」反復が必要
- 完了条件が成果物の品質で判定される（ステップ数が不定）

使わないとき：
- 手順を固定できる → ワークフロー骨格のほうがテスト容易・コスト予測可能
- LLMに行動選択を委ねること自体がリスク → 決定論的殻で制御フローを固定
- タスクが大きすぎて単一ループに収まらない → 計画-実行-検証で分割

⚠️ 落とし穴
- 予算枯渇で空を返さないでください。途中経過の要約や「ここまで分かったこと」を返却する部分結果のフォールバックが必須です
- コンテキスト肥大に注意してください。ループが進むと履歴が膨れ、コスト増加と推論精度低下を招きます。一定ステップごとに履歴を要約圧縮する仕組みが必要です
- 予算をハードコードしないでください。上流から渡された残り予算をループの初期値にし、カスケードで管理します

🔧 実装方針
- ステップ数・トークンコスト・経過時間の3次元の予算オブジェクトを初期化し、毎ステップごとに差し引きと残量チェックをコード側で行います
- 残り予算をシステムプロンプトに注入し、LLM自身がリソースを意識した判断を行えるようにしつつ、強制終了はコードで保証します
- スタック検出は直近の履歴から同一ツール＋同一引数の繰り返しを検知するロジックで実装します
- 高リスク操作（課金・削除・外部送信など）にはループ内に承認ゲートを挟み、自律性の中でも不可逆操作を制御します
- マルチエージェント構成に拡張する場合は、Supervisorが全体予算をWorkerに按分し、Worker間の通信はSupervisor経由に限定します
