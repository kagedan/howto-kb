> 「毎回チャットの冒頭に、自分の役割と前提とお願いの仕方を、長々と貼り付けてから頼んでいる」 「同じ作業を頼むたびに、同じ手順をゼロから説明し直している」

このあたりで時間を溶かしている人は、たぶん多いと思います。AI に頼むこと自体は便利になったのに、頼むための前置きが毎回ほぼ同じで、その入力にいちばん時間がかかっている、という状態です。

ここが、設定をひとつ仕込むだけで変わります。Claude の Cowork（コワーク。デスクトップアプリの中で、許可したフォルダの作業を代わりに進めるモード）には、一度だけ書いておけば毎回読まれる「指示」と、手順をまるごと部品にする「Skill」、それらを束ねる「Plugin」という、3つの設定の階層があります。これを先に整えておくと、200 文字の前置きを毎回書く代わりに「例の件、進めといて」くらいの短い指示で、同じ前提・同じ手順で動いてくれます。

しかもこの3階層は、僕が考えた整理ではなく、Anthropic 自身が「Cowork のカスタマイズは3つの階層で行う」と公式チュートリアルで説明している枠組みです（[Customize Claude Cowork（公式）](https://claude.com/resources/tutorials/customize-claude-cowork)）。海外で出回っている設定術も、たどると結局この公式の3階層に行き着きます。だから流行りの裏ワザを追うより、公式の3階層を一度きちんと押さえるほうが早い、というのが今回の話です。

tatsuki（[@nobel_824](https://x.com/nobel_824)）と申します。中小企業向けに AI の活用サポートをしていて、Claude / Codex の業務導入を手伝いつつ、自分でも Claude Code を1日中走らせています。「導入はしたけど、結局毎回同じ説明をしていて疲れる」は、サポートの現場でいちばんよく出る相談のひとつです。この記事では、Cowork のカスタマイズを公式の3階層に沿って、何をどこに書けばいいかまで、非エンジニアの現場目線で橋渡しします。

## まず全体像：設定には「3つの階層」がある

中身に入る前に、地図だけ先に渡します。公式は Cowork のカスタマイズを、次の3つの階層に分けています（[Customize Claude Cowork（公式）](https://claude.com/resources/tutorials/customize-claude-cowork)）。

- **第1階層：あなたの文脈とツール**。毎回守ってほしい「指示（インストラクション）」と、外部サービスにつなぐ「コネクタ」
- **第2階層：手順を記録する**。決まったやり方を部品にした「Skill（スキル）」
- **第3階層：束ねて配る**。コネクタと Skill をまとめて1クリックで入れる「Plugin（プラグイン）」
ざっくり言うと、第1階層が「AI への前提と道具」、第2階層が「決まった手順」、第3階層が「それらを一式にして共有する箱」です。下の階層ほど毎回効く土台、上の階層ほど特定の作業やチーム共有のための仕組み、と捉えると迷いません。

順番にいきます。

## 第1階層：指示（インストラクション）とコネクタ

いちばん効いて、いちばん最初に手をつけるべきなのがここです。

毎回読まれる「指示」を2段階で置く

Cowork の「指示」は、AI に毎回守ってほしい立ち位置やルール（口調・出力の形・先に見るべき情報・命名規則など）を、前もって書いておくものです。公式は、この指示を2段階で設定できると説明しています（[Customize Claude Cowork（公式）](https://claude.com/resources/tutorials/customize-claude-cowork)）。

ひとつめが**グローバル指示（Global instructions）**。これはどのプロジェクトで作業していても、すべての Cowork セッションに共通で効く「憲法」です。場所は Settings → Cowork → Global instructions で、デスクトップアプリ専用です。ここには、案件をまたいで変わらない自分の前提を書きます。たとえば、

- 僕は中小企業向けに AI 活用サポートをしている個人事業主。スピードと再現性を重視する
- 出力は結論ファースト。長い前置きは要らない
- 重要な操作（削除・送信・お金が絡む作業）は、実行する前に必ず計画
