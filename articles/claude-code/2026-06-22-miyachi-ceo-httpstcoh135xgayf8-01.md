「ChatGPTはもう使っている」という事業者（個人事業主・ひとり社長・士業）は、もう珍しくありません。でも、よく聞くと中身はだいたいこうです。

- 顧客へのメールの下書きを“その都度”頼む

- 文章のトーンを“その都度”直す

- 専門的な内容を噛み砕いた説明を“その都度”作る

便利です。でも、**毎回ゼロから頼んでいる**。頼むたびにこちらの頭を使うので、「相談相手」としては優秀でも、「仕事が減った」実感は出てきません。これが“壁打ち止まり”の正体です。

足りていないのは、賢いAIではありません。**あなたの判断基準で、毎回同じように動いてくれる“仕組み”**です。私はこれを、自分の仕事を**「部署ごと」に整理してAIに渡す**、と呼んでいます。人を増やさずに、まるで業務ごとに“担当者”がいるかのように回すイメージです（実際に動くのは同じ1つのAIです。からくりは後で説明します）。

![](https://pbs.twimg.com/media/HLYjImaaoAANKNU.jpg)

# ChatGPTを毎日使っているのに、なぜ仕事が減らないのか

理由はシンプルで、**ChatGPTへの依頼は「その場限り」**だからです。今日うまくいった頼み方も、明日は自分で思い出して、また一から指示する。判断はいつもあなたの頭の中にあり、AIには毎回コピーして渡している。

これは「相談」としては正しい使い方です。けれど、**仕事そのものを肩代わりさせる**には足りません。肩代わりさせるには、あなたの判断基準を“あなたの外”に出して、AIがいつでも参照できる形にしておく必要があります。

# 「賢いAI」より「仕組み」が要る理由

ここがいちばん大事なので、先に種明かしをします。

AI（大規模言語モデル）は、確率で次の言葉を選んでいます。だから、単発のタスクではうまくいっても、毎日の業務で“同僚”として使おうとすると、こういうことが起きます。

- 入力がちょっと違うだけで、出てくる答えが変わる

- 会話が長くなると、要領を得なくなる

- 渡した資料が増えると、肝心なところを参照しなくなる

- やってほしくないことを、たまにやってしまう

つまり、**「一度うまくいった」と「毎日まかせられる」はまったくの別物**です。この差を埋めるのが「ハーネス（馬具）」という考え方です。性能の高い馬（AI）に、毎回安定して走ってもらうための装具、と捉えてください。

具体的には、4つの道具で構成します。

- **CLAUDE.md（憲法）**：全体の方針・前提・判断基準を渡す

- **Skills（業務マニュアル）**：繰り返す作業を「手順書」として固定する

- **サブエージェント（担当の分離）**：役割ごとに別の“担当者”に任せる

- **hooks（自動発動）**：特定のタイミングで必ず処理を走らせる

![](https://pbs.twimg.com/media/HLYjh3iboAAjXdn.jpg)

この記事では、いちばん効果が大きく、いちばん始めやすい **CLAUDE.md（憲法）** を主役にします。コードは一切書きません。書くのは「日本語のルール」だけです。

# 設計の核：フォルダを「部署」に、CLAUDE.mdを「判断基準」に

どんな事業でも、仕事はいくつかの“役割”に分かれています。お金まわり、顧客対応、そして本業。それぞれ、見るべきルールも、気をつける観点も違います。

AIに預けるときも、まったく同じ発想で組みます。

```
事業AI/
├── 経理/
│   └── CLAUDE.md   ← お金まわりのルール
├── 顧客対応/
│   └── CLAUDE.md   ← 窓口のルール
└── 本業/
    └── CLAUDE.md   ← 本業の判断基準
```

**フォルダ＝部署、その中の `CLAUDE.md`＝その部署の判断基準。**

![](https://pbs.twimg.com/media/HLYj4RHaUAAgK5-.jpg)

こうすると、経理の作業をしているときは経理のルールだけが、本業の作業をしているときは本業のルールだけが効きます。「一人の万能AIに全部を覚えさせる」のではなく、「役割ごとに、見るべきものを分ける」。これだけで、精度も安全性も一段上がります。

念のため、からくりを正確に書きます。**これは、AIを何体も並列に動かす話ではありません。動いているのは、いつでも同じ1つのAI**です。フォルダごとに `CLAUDE.md`（そのフォルダのルール）
