「Claude Codeを使ってみたいけど、何から始めればいいのかわからない」——そんな悩みを抱えていませんか？

ターミナルが怖い、コマンドが難しそう、料金体系がよくわからない……はじめて触るときは、不安なことだらけですよね。私も最初は「どこから手をつければいいんだ」と戸惑った記憶があります。

この記事では、**Claude Codeを使い始める前に知っておくべき基礎知識**を体系的にまとめました。インストール方法・料金・基本コマンド・CLAUDE.mdの使い方まで、一通りカバーしているので、この記事を読めば「とりあえず動かせる状態」まで到達できるはずです。

---

## 結論：Claude Codeはこの4点を押さえれば使える

まず結論から伝えます。Claude Code初心者が最初に知るべきことは、次の4つです。

| # | ポイント | 概要 |
|---|---|---|
| ① | **Claude.aiとは別物** | ブラウザで使うClaude.aiと、ターミナルで動くClaude Codeは別ツール |
| ② | **有料プランが必須** | Pro（$20/月）以上が必要。無料プランでは使えない |
| ③ | **インストールはコマンド1行** | セットアップ自体は3分あれば終わる |
| ④ | **CLAUDE.mdが品質を左右する** | プロジェクト設定を書いておくと応答の質が格段に上がる |

これだけ押さえておけば、大きく迷うことはありません。以下で順番に深掘りしていきます。

---

## なぜ「知らないと損する」のか——初心者がはまりがちな落とし穴

Claude Codeを使い始めてすぐ「思ったより難しい」「コストが怖い」と感じる人の多くは、**事前情報なしに触り始めてしまう**パターンに陥っています。

たとえば、Claude.aiとClaude Codeを同じものだと思っていると、「なぜわざわざターミナルを使うの？」と混乱します。また、料金の仕組みを理解せずに使い始めると、API従量課金とサブスクリプションのどちらを選ぶべきか迷ったまま損をしてしまうことも。

私自身、最初は「普通のチャットAIと何が違うの？」という感覚でした。でも実際に触れてみると、**コードを書くだけでなく、テスト実行・Gitコミット・複数ファイルの横断編集まで自律的にやってくれる**ことに驚きました。これはチャット型AIにはない強みです。基本をきちんと理解してから使い始めると、最初の1時間でその違いが体感できます。

---

## ① Claude.aiとClaude Codeの違いを理解する

✅ まずここだけは絶対に覚えてください。

| 項目 | Claude.ai | Claude Code |
|---|---|---|
| 使う場所 | ブラウザ（Web・スマホ） | ターミナル（CLI） |
| ファイル操作 | ❌ できない | ✅ 読み書き・削除が可能 |
| コマンド実行 | ❌ できない | ✅ npm, git, ls 等を実行 |
| Git操作 | ❌ できない | ✅ コミット・PRまで対応 |
| 複数ファイル横断 | △ 限定的 | ✅ プロジェクト全体を理解 |
| スマホ利用 | ✅ 可能 | ❌ 基本的に不可 |

Claude.aiで「TODOアプリを作って」と頼むと、コードを表示してくれます。でも自分でファイルを作る必要があります。一方、Claude Codeに同じことを頼むと、**ファイルを作成してコードを書き、テストまで走らせてくれます**。この違いは大きい。

「どのプランを選べばいいの？」は、初心者が一番悩むポイントです。Claude Codeには2つの使い方があります。

| 方式 | 料金 | 特徴 |
|---|---|---|
| **サブスクリプション（推奨）** | Pro: $20/月〜、Max: $100/月〜 | 月額定額。コスト予測が立てやすい |
| **API従量課金** | トークン数に応じた課金 | 使った分だけ払う。ヘビーユーザー向き |

◎ **最初はProプラン（$20/月）から始めるのがベスト**です。

理由は単純で、「コストが読める」から。API従量課金は使いすぎると青天井になることがあり、初心者には心理的な不安が大きくなります。Proプランで感覚をつかんでから、必要に応じて切り替えるのが王道です。

> **注意：** 無料のClaude.aiプランにはClaude Codeのアクセスは含まれていません。

## ③ インストール手順（3分で完了）

インストール自体は驚くほど簡単です。OSに合わせて以下のコマンドを1行実行するだけ。

```bash
# macOS / Linux / WSL（推奨）
curl -fsSL https://claude.ai/install.sh | bash

# macOS（Homebrew）
brew install --cask claude-code

# Windows PowerShell
irm https://claude.ai/install.ps1 | iex
```

インストール後は `claude` コマンドで起動します。初回はブラウザが開いてログイン認証を求められます。

```bash
claude
```

✅ 起動したら `claude doctor` を実行して動作確認しましょう。「All checks passed」と表示されれば成功です。

### システム要件の確認

| 項目 | 要件 |
|---|---|
| OS | macOS 10.15+ / Ubuntu 20.04+ / Windows 10+（WSL） |
| Node.js | バージョン 18 以上（npmインストールの場合） |
| ネット接続 | 必須（Anthropic APIへの接続が必要） |

## ④ 最初に覚えるべき基本コマンド一覧

Claude Codeには専用のスラッシュコマンドがあります。最初はこの7つだけ覚えれば十分です。

| コマンド | 説明 |
|---|---|
| `/init` | プロジェクト用のCLAUDE.mdを自動生成 |
| `/help` | 利用可能なコマンド一覧を表示 |
| `/clear` | コンテキストをリセット（タスクが変わったら即実行） |
| `/compact` | 会話を要約してトークンを節約 |
| `/model` | 使用するモデルを変更（OpusかSonnetか） |
| `/cost` | API利用時のトークン消費とコストを確認 |
| `/doctor` | 環境の動作チェック |

❌ よくある失敗：コンテキストをリセットせずに別のタスクを始めると、前の文脈が残って的外れな回答が増えます。タスクが変わったら `/clear` を習慣にしましょう。

## ⑤ CLAUDE.mdの使い方——これが一番重要かもしれない

**CLAUDE.mdは、Claude Codeへの「事前説明書」です。**

プロジェクトのルートに置いておくと、Claude Codeが毎回読み込んで文脈を理解してくれます。これを書くか書かないかで、応答の品質がまったく変わります。私はこの仕組みを知ってから、「なんで最初から使わなかったんだ」と思いました。

```bash
# まずはinitコマンドで自動生成してもらう
/init
```

生成されたCLAUDE.mdに、以下のような情報を追記していくのがおすすめです。

```markdown
# プロジェクト概要
ECサイトのバックエンドAPI。PythonとFastAPIを使用。

# コーディングルール
- 変数名は英語スネークケース
- コメントは日本語で書く
- テストは必ずpytestで書く

# よく使うコマンド
- テスト実行: pytest tests/
- 起動: uvicorn main:app --reload
```

✅ CLAUDE.mdは「育てるもの」という意識で、使いながら少しずつ追記していくのがコツです。

## ⑥ プロンプトの書き方で結果が変わる

Claude Codeへの指示の仕方にも、コツがあります。

❌ **悪い例（曖昧すぎる）**
```
このコードを改善して
```

✅ **良い例（具体的）**
```
このコードのパフォーマンスを改善して。
特に配列の処理部分で、現在O(n²)になっている部分をO(n)にしたい。
変更後はテストが全部通ることも確認して。
```

指示が具体的であればあるほど、無駄なトークン消費を抑えられ、期待通りの結果が返ってきます。大きなタスクを一度に渡すより、**細かく分割して指示する**ほうがトークンの節約にもなります。

## ⑦ モデルの選び方（OpusとSonnet）

Claude Codeでは `/model` コマンドでモデルを選べます。

| モデル | 特徴 | おすすめの使いどころ |
|---|---|---|
| **Opus** | 高性能・高精度 | 複雑なロジック、設計相談、大規模リファクタリング |
| **Sonnet** | 高速・コスト効率が高い | 軽めのタスク、繰り返し作業、limit警告が出たとき |

◎ Maxプランの場合は最初はOpusを選んでおいて、使用制限の警告が出たらSonnetに切り替えるという使い方が効率的です。Proプランの場合はSonnetで十分なケースも多いです。

Claude Code初心者が最初に知るべきことを整理すると、以下の通りです。

| チェック | 内容 |

| ✅ | Proプラン以上が必要だと把握した |
| ✅ | インストールコマンドを実行して起動できた |
| ✅ | 基本スラッシュコマンドを7つ覚えた |

ターミナルに `claude` と打ち込む、その一歩が、開発体験をガラッと変えてくれるはずです。
