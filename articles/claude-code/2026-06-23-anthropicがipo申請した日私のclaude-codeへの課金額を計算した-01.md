## Anthropicのほぼ1兆ドルIPO──ARR 300億ドルの実態

2026年6月1日、AnthropicはSECに機密でS-1（上場申請書）を提出した。10月のNasdaq上場を目指し、Goldman Sachs・JPMorgan・Morgan Stanleyが主幹事を担う。調達目標は600億ドル（約9兆円）以上とされる。

成長速度がおかしい。2024年末に10億ドルだったARRが、2026年4月には300億ドルに達した。18ヶ月で30倍。その中にClaude Code単体のARR 25億ドル（約3,700億円）が含まれているのは、エンジニア的には真剣に受け取るべき数字だと思う。

「AIはまだビジネスになっていない」という声を今でも聞く。この数字を見れば、その議論は終わる。コーディングツール単体でこの規模の収益になっているという事実が、AIが「実験」の段階をとっくに超えていることを示している。

IPO後がどうなるかは正直わからない。上場すれば短期的な利益を求める株主の声が大きくなる。「AI安全性」を掲げ続けるAnthropicが、収益最大化プレッシャーにどう対応するか。今のAPIとサブスクリプション価格がいつまで維持されるかも不透明だ。楽観的にはなれない。

**今日から試せること：** 企業でClaudeを活用しているなら、Claude Code ARRの推移を四半期ごとに確認する習慣をつける。競合サービスの活用判断ベンチマークにも使える数字だ。

## 前回予告：Record & Replayの「チームシェア」はこう使う

前回の記事でRecord & Replayを自分で試した感想を書いた。今日はその続きとして予告していた「チームへの展開」に踏み込む。

Codex app 26.616（6月18日リリース）のRecord & Replayが生成するスキルは、Open Agent Skills Standard準拠のファイルとして出力される。チームへの配布は、実はシンプルだ。

これのいいところは「手順書が不要になること」だ。「この操作、うまく説明しにくい」という業務を、Record → スキル化 → push の3ステップでチームに渡せる。ドキュメントを書く代わりに、実際の動作を渡す感覚に近い。

もう一点、Open Agent Skills Standard準拠なので、Codexで録画したスキルがClaude CodeにもCursorにもそのまま使える。ツールをまたいだポータビリティがある。チームでツールが混在していても、スキルは共通資産として管理できる。

正直に言うと、スキルの品質はまだムラがある。操作が複雑すぎると生成されたスキルが冗長になり、手直しが必要になることもあった。チームへの公開前に自分で1回通して動かすことは推奨する。

**今日から試せること：** 自分の反復業務を1つRecordしてスキル化してみる。`.claude/skills/`に置いてリポジトリに追加し、チームのSlackに「このスキル使える？」の一言で共有するだけで始められる。

## GPT-4.5が今週金曜日に退役する

あまり話題になっていないが、GPT-4.5が2026年6月27日にChatGPTから退役する。OpenAIはすでに6月12日にGPT-5.2シリーズを終了しており、現在の標準モデルはGPT-5.5 Instantだ。

モデルのライフサイクルが急速に短くなっている。かつてGPT-4は2年以上現役だった。今は半年で世代交代が起きている。

これが意味することは一つ。既存システムへのモデル組み込みが、定期的なバージョンアップ作業を前提にしなければならなくなっているということだ。「一度組んだら終わり」ではなく、「モデルのバージョン管理」をアーキテクチャに組み込む発想が必要になっている。GPT-4.5の退役は小さなニュースに見えて、この構造変化を示す一例だ。

**今日から試せること：** 自社サービスやワークフローでGPT-4.5を使っている箇所を今日中にリスト化する。6/27以降は自動でGPT-5.5に移行されるが、応答の変化を事前にテストしておく価値がある。
