Claude Code で新登場した 「Agent teams」 と 「SubAgent」 の使い分けが公式ドキュメントにも書いてました

Subagent は 「リサーチ・検証・レビュー」 など、焦点を絞ったワーカーが必要な場合に使う

一方、Agent teamsは 「競合仮説の検証や並列レビュー」 など、結果を共有し、議論と協力が必要な場合に使う

使い分けの目安
- 結果だけ欲しい → Subagent
- 議論・協力が必要 → Agent teams
- Subagentでコンテキスト限界 or SubAgent 同士の通信が必要に → Agent teamsへ移行
