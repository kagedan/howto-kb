Claude codetを業務でも使い始めてから、毎週・毎月どれくらい使っているのかが把握できなくなってきた。さらに Codexも併用するようになると、全体でいくら使い切れているのか？  
どのプロジェクトに集中しているのかが全くわからない。

ccusageというバチくそイケているOSSがあるが、CLIで都度確認するスタイルのため、「振り返り」や「時系列での変化」が見えない。  
<https://ccusage.com/>  
<https://github.com/ryoppippi/ccusage>  
※ 製作者のryoppippiさんもあくまでジョークツールとポストされてました

そこで、クラウドに蓄積して後から分析できるようにccreportを作った。

---

## できること

ヒートマップで使用量を可視化  
1年分の日別トークン量を表示することで、どのタイミングでトークンの消費量が多かったかが一目でわかる。  
※個人的にはGithubのUI以上に勝るものはないと思ってオマージュしました。

![](https://static.zenn.studio/user-upload/cba97731bfca-20260328.png)

Claude code + Codex を1画面で2つのツールの使用量・コスト・セッションのリセット時間をまとめて確認できるようにした。  
ただ、ccusageに対してリスペクトを持ち続けるという意図で今月 Claude Code に $89、Codex に $323分のトークンを使っているという把握をできるようにしている。

１コマンドでクラウドに同期

これだけ。  
ローカルの `~/.claude/projects/`と`~/.codex/sessions/`を読み取ってクラウドに送信する。  
※cronに登録しておけば自動で蓄積可能

---

## ccusageとの違い

|  | ccusage | ccreport |
| --- | --- | --- |
| 動作 | CLIローカル完結 | CLI + クラウドダッシュボード |
| データ保持 | 都度確認 | 時系列で蓄積 |
| Codex 対応 | なし | あり |

ccusageは早くて軽い。  
ccreportは「蓄積して振り返る」用途に向いている。

---

## 使い方

1.アカウントを作成  
![](https://static.zenn.studio/user-upload/62d2745c84c3-20260328.png)

<https://ccreport.ippei-matsuda.workers.dev> からメールアドレスとパスワードで登録する。

2.CLIをインストールして設定

Settig ページでCLIトークンを発行し、環境変数に設定する。

```
export CCREPORT_CLI_TOKEN=your_token
export CCREPORT_USER_ID=your@email.com
```

3.pushで同期

実行するとローカルのセッションデータをスキャンして送信する。以降は定期実行で自動化できる。

# cronに登録する場合（毎時）
