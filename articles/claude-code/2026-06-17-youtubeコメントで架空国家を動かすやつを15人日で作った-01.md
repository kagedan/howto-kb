## 何を作ったか

YouTube Liveのコメントが「世論」になり、架空国家の状態がリアルタイムで変化し続ける常時配信型Webアプリを作りました。

その名も **AInation**。

* 視聴者がコメントを投稿する
* AIがコメントを世論として解析する
* 経済・治安・過激化など**13の国家指標**がリアルタイムで変動する
* AIが架空ニュースを自動生成し、速報・検閲・声明などの介入も行う
* 全部常時配信でYouTube Liveに流れ続ける

コンセプトは「視聴者全員で一つの国家を動かす」です。

---

## 技術スタック

```
YouTube Live API
      ↓ (polling)
   worker
      ├── Redis（コメント短期保存）
      ├── OpenAI API（世論解析 / ニュース生成）
      ├── PostgreSQL（世界状態・履歴永続化）
      └── Redis Pub/Sub（リアルタイム配信）
            ↓
          web（Next.js）
            ├── SSE /api/stream
            └── 管理API /api/admin/*
```

* **フロントエンド**: Next.js（スマホ最適化UI / OBS Browser Source対応）
* **バックエンド**: Node.js worker
* **DB**: PostgreSQL（Prisma）+ Redis
* **AI**: OpenAI API
* **インフラ**: AWS Lightsail（Docker Compose）
* **UI**: 監視画面風ダークテーマ

管理画面（`/admin`）からYouTube Live Video IDや解析間隔、AI介入のON/OFFを設定できます。

## 一番苦労したこと：「スクロールできない」をAIに理解させる

これが地味に一番のハマりポイントでした。

通常のWebアプリなら「コンテンツが増えたらスクロールして見る」が当たり前です。でもこれはYouTube Liveに配信する画面。**OBSで固定サイズのブラウザウィンドウとして表示されるため、スクロールは一切できません。**

何度かUIの提案をAIにさせると、どうしても縦に伸びるレイアウトを出してきます。

「スクロールなしで全情報を固定画面に収めてくれ」と伝えても、微妙にはみ出るレイアウトが返ってくる。

最終的に効いた伝え方はこれでした：

> 「これはOBS Browser Sourceとして使います。画面サイズは固定で、スクロールバーは存在しません。画面外に出た要素は永遠に見えません。全ての情報を指定した解像度の中に収めてください。」

「スクロールできない」より「画面外は永遠に見えない」のほうがAIには刺さりました。
