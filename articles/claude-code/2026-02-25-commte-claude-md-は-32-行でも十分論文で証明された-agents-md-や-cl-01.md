CLAUDE .md は 32 行でも十分。論文で証明された

AGENTS .md や CLAUDE .md の効果を初めて厳密に検証した研究

◯ 論文の結論

- コンテキストファイルはタスク成功率を下げる傾向がある
- 推論コストは 20% 以上増える
- エージェントは指示に忠実すぎて、不要な要件まで律儀に守る
- スタイルガイドや「〜を使え」系の指示が、タスクを無駄に複雑にしてる

◯ じゃあ何を書けばいいのか

- ビルド/テストコマンド
- リポジトリ固有のツール（uv、pdm 等）
- エージェントがコードを読んでも分からない慣習だけ

逆に言うと、コードを読めば分かること、標準的な規約、クリーンなコードを書けみたいな自明なことは全部ノイズ

CLAUDE .md はコードと同じ。定期的に剪定して、本当に必要な行だけ残す
論文はリプ
