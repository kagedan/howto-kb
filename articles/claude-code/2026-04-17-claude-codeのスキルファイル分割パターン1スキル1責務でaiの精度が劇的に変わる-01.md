## TL;DR

CLAUDE.mdが200行を超えたら分割のサイン。ルーティングテーブルだけ本体に残し、具体的な手順は用途別スキルファイルに分離する。「1スキル=1責務」の原則を守れば、LLMの指示遵守率が体感で大幅に向上する。

## 問題：CLAUDE.mdの肥大化

Claude CodeやCursorでプロジェクトルールを書くとき、最初はすべてを1ファイルに書きがちです。

```
# CLAUDE.md（肥大化版）
- プロジェクトルール
- コーディング規約
- デプロイ手順
- 外部API連携仕様
- テスト方針
- セキュリティポリシー
... (500行超)
```

これは「動く」のですが、行数が増えるにつれて問題が顕在化します。具体的には、**指示の半分が無視されるようになる**。

## 解決：スキルファイル分割パターン

やり方はシンプルです。

**CLAUDE.md（本体）** にはルーティングテーブルだけ残す：

```
# CLAUDE.md
## スキルルーティング
| トリガー | スキルファイル |
|---|---|
| 音声合成, TTS | ~/.claude/skills_tts.md |
| 動画生成 | ~/.claude/skills_video.md |
| デプロイ | ~/.claude/skills_deploy.md |
| PDF変換 | ~/.claude/skills_pdf.md |
```

**各スキルファイル** は統一構造にする：

```
# skills_tts.md
## トリガー条件
「PPTXから音声を生成して」「ナレーション音声を作って」等

## 実行手順
1. ...
2. ...

## パラメータ
- voice: Takumi (default)
- speed: 1.0
```

## なぜ効果があるのか

分割が効く理由は、コンテキストウィンドウの「密度」にあります。

LLMは入力されたトークンすべてに対してAttentionを計算します。500行すべてを読み込ませると、今必要な手順10行に対して490行がノイズになる。SNR（Signal-to-Noise Ratio）が極めて低い状態です。

スキルファイル分割では、今必要なファイルだけを読み込むため、コンテキスト内のSNRが劇的に上がります。

| 構成 | コンテキスト | 関連情報 | SNR |
| --- | --- | --- | --- |
| 1ファイル500行 | 500行全部 | 10行 | 2% |
| 分割（必要なスキルのみ） | 50行 | 40行 | 80% |

## 実践で見えた効果

分割前後で変わったのは主に3点です。

**1. 複雑な手順の再現性**  
10ステップ以上の手順（例：PPTX→音声生成→動画変換→結合）がほぼ毎回正確に再現されるようになった。分割前は3-4ステップ目で脱線することが多かった。

**2. 条件分岐の精度**  
「Aの場合はX、Bの場合はY」という条件付きルールの遵守率が向上。1ファイルでは条件の片方が無視されることがあった。

**3. メンテナンス性**  
スキルごとに独立しているため、1つを修正しても他に影響しない。チーム開発でも分担しやすい。

## 分割のベストプラクティス

**1. 1スキル=1責務**  
「TTS音声生成」と「動画変換」は別ファイル。混ぜると肥大化の二の舞。

**2. トリガー条件を明確に**  
ルーティングテーブルのキーワードは具体的に。「動画」だけでなく「PPTXから動画」「スライドをPNGに」「クリップを結合」等。

**3. 200行ルール**  
1つのスキルファイルが200行を超えたら、さらに分割を検討する。

**4. パラメータセクションを必ず入れる**  
デフォルト値を明記しておくと、LLMが勝手に推測する余地が減る。

## 他ツールとの比較

| ツール | ルール管理 | 分割対応 |
| --- | --- | --- |
| Claude Code | CLAUDE.md + skills\_\*.md | 手動分割、ルーティングテーブル |
| Cursor | .cursorrules | 単一ファイル前提（分割は非公式） |
| GitHub Copilot | .github/copilot-instructions.md | 単一ファイル |
| Windsurf | .windsurfrules | 単一ファイル |

Claude Codeのスキルファイル分割は、現時点でこの種のツールの中で最も柔軟なルール管理が可能です。

## まとめ

CLAUDE.mdの肥大化はLLMの精度低下に直結します。「1スキル=1責務」で分割し、必要なときだけ読み込ませることで、コンテキストのSNRを上げる。これがAIコーディングツールを最大限活用するためのプラクティスです。
