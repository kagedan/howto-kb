## 5. コードの見どころ 💻

### 傾き検知とドラッグ操作の両立

スマホのジャイロセンサー（DeviceOrientation）と、  
PCでのマウスドラッグを**1つの状態（State.pourAngle）** で統一管理しています。

ドラッグ→傾き変換のコア部分

```
// Handle touch move to compute tilt angle from drag distance
handleCanvasTouchMove(e) {
  if (!this.pouchDrag.active) return;
  const touch = e.touches[0];
  const dy    = touch.clientY - this.pouchDrag.startY;
  // Map drag distance to tilt angle: 100px drag = 45° tilt
  this.pouchDrag.currentAngle = Math.max(-60, Math.min(60, -dy * 0.45));
  State.manualOverride = true; // ユーザーがタッチ→センサー上書き
  State.pourAngle = this.pouchDrag.currentAngle;
},

// PCでのマウスドラッグ（左ボタン押し続け中のみ）
handleCanvasMouseMove(e) {
  if (!this.pouchDrag.active || e.buttons !== 1) {
    this.pouchDrag.active = false;
    return;
  }
  const dx = e.clientX - this.pouchDrag.startX;
  State.manualOverride = true;
  State.pourAngle = Math.max(-60, Math.min(60, dx * 0.5)); // 120px drag = 60°
},
```

`State.manualOverride` フラグで「センサー vs タッチ操作」を切り替えるのがポイントです。

このアプリ最大のハイライトがここです。  
Geminiにもコードレビューをお願いしたところ、衝撃の事実が発覚しました。

> Gemini:高度な検知関数は用意されている — `checkWristShake`（手首の震えを検知）や `checkMouthOpen`（口が開いているか＝「アホ面」になっていないか検知）という非常に本格的な判定ロジックがしっかり書かれています。しかし、ゲーム本編では**一度も呼ばれない**。

* ARモードで `MediaPipeManager.init()` を呼んでいますが、これは**CDNからJSを読み込んで右上のバッジを「MEDIAPIPE: ACTIVE」と緑色にするだけ**
* 最も重い処理である実際のカメラ映像の解析（`new Hands(...)`など）は**起動すらしていない**
* ゲーム開始時にカメラ自体もオフにしてしまう

Geminiに「バカバカしくて良い」とコメントをもらい、**あえてこのまま**にすることにしました。

> Gemini:「詰め替え」という日常の超絶地味な家事に、謎に **「TACTICAL REFILL SYSTEM（戦術的詰め替えシステム）」** と名付けたり、無駄にサイバーなUIを出したりするスタンスが最高にバカバカしくて良い味を出していると思います。

それがClaudeにはまた受けてました😂
