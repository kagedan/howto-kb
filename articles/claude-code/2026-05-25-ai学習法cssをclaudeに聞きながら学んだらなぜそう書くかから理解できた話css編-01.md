## TL;DR

* CSS変数は「色の名前」ではなく「色の役割」で命名する
* `clamp()` を使うとメディアクエリ3つが1行になる
* ダークモード対応は CSS変数があれば `:root` を上書きするだけ
* 上の3つは全部「なぜそう書くか」を聞いたから今も覚えてる

## はじめに

前回の HTML 編では、Claude Codeを家庭教師として使いながら自己紹介ページの骨格を作るところまで書いた。

今回はその続き。CSS でデザインをつけながら、CSS変数・レスポンシブデザイン・ダークモードを実装した話を書く。

「CSS は雰囲気でなんとかなる」と思っていたが、Claude Codeに聞きながら進めると **「なぜそう書くか」まで教えてもらえた**。それが一番の収穫だった。

## CSS変数：命名は「何色か」ではなく「何のための色か」

最初に CSS変数を学んだとき、こう書いてしまった。

```
:root {
  --blue: #3b82f6;
  --gray: #6b7280;
}
```

「変数に入れたからOK」と思っていた。Claude Codeに見せたら、こう返ってきた。

> 後でブランドカラーを青から緑に変えたとき、`--blue` という名前が残ったままになります。「青いのに緑」という矛盾が起きる。変数名は「何色か」ではなく「何のための色か」で付けるのが現代の書き方です。

言われてみれば当たり前なのに、一人で本を読んでいたら気づかなかったと思う。

```
:root {
  --color-primary: #3b82f6;   /* 主要アクション・リンク色 */
  --color-text: #1f2937;      /* 本文テキスト */
  --color-bg: #ffffff;        /* 背景 */
  --color-border: #e5e7eb;    /* 区切り線 */
}
```

さらに「変数化する基準は？」と聞くと、**2回以上使う、または後で一括変更したくなりそうな値**だけ変数にすればいい、と教えてもらった。1回しか使わない値まで全部変数にするのは逆に管理が大変になる。

「なぜそう書くか」までセットで教えてもらえると、ルールが覚えやすかった。

## clamp()：メディアクエリ3つが1行になる

学習資料に `clamp()` が登場したとき、正直「これ必要？」と思っていた。メディアクエリで書けばいいじゃないか、と。

疑問をそのまま聞いた。

> clamp() は「最小値・推奨値・最大値」を1行で書ける関数です。メディアクエリ3つで書いていたものが1行になって、さらに画面幅に応じてなめらかに変化します。

「なめらかに変化する」というのがポイントで、メディアクエリだと「ある幅を超えた瞬間に切り替わる」のに対して、`clamp()` は連続的に変わる。使ってみて、確かに違うと実感した。

レスポンシブ対応でこんなコードを書いていた。

```
h1 { font-size: 1.5rem; }

@media (min-width: 48rem) {
  h1 { font-size: 2rem; }
}

@media (min-width: 64rem) {
  h1 { font-size: 2.5rem; }
}
```

3つのメディアクエリを `clamp()` 1行に置き換えられる。

```
h1 { font-size: clamp(1.5rem, 5vw, 2.5rem); }
/*                   最小     推奨   最大   */
```

動作はシンプルで、画面が狭いと最小値（1.5rem）を下回らず、広いと最大値（2.5rem）を超えない。その間は `5vw`（画面幅の5%）でなめらかに変化する。

余白にも使えるのが便利だった。

```
.container {
  padding: clamp(1rem, 5vw, 3rem);
}
```

スマホでは狭めのパディング、デスクトップでは広めのパディングが、メディアクエリなしで自動的に変わる。

## ダークモード：CSS変数があれば `:root` を上書きするだけ

「ダークモード対応って難しそう」というイメージがあった。Claude Codeに聞いてみると、

> CSS変数で設計されていれば、`:root` の変数をダークモード用に上書きするだけです。難しくはないですが、**色を直書きしているとどこを変えればいいかわからなくなる**ので、変数化が前提になります。

ここで前のセクションの話がつながった。変数で設計しておくことの意味が、ダークモードを実装した瞬間に実感できた。

ダークモードは難しそうに見えて、CSS変数で設計されていればシンプルだった。

```
/* ライトモード（デフォルト） */
:root {
  --color-bg: #ffffff;
  --color-text: #1f2937;
  --color-primary: #3b82f6;
}

/* OS設定がダークモードのとき上書き */
@media (prefers-color-scheme: dark) {
  :root {
    --color-bg: #111827;
    --color-text: #f9fafb;
    --color-primary: #60a5fa;
  }
}

body {
  background: var(--color-bg);
  color: var(--color-text);
}
```

`prefers-color-scheme: dark` でOSのダークモード設定を検知して、`:root` の変数だけ上書きする。あとは `var()` で書いておけば自動的に切り替わる。

逆に言うと、**直書きしてしまった色はダークモードで変わらない**。これでハマった。最初にボタンの色を `background: #3b82f6` と直書きしてしまっていて、ダークモードにしても変わらなかった。CSS変数経由で書くことの意味が、この瞬間に実感できた。

さらに `color-mix()` を使うと、ホバー色も自動生成できる。

```
.button:hover {
  background: color-mix(in oklch, var(--color-primary), black 15%);
}
```

`--color-primary` を変えるだけで、ホバー色も連動して変わる。色のバリエーションをすべて変数で管理しなくていい。

## ハマったこと

### transition はカンマ区切りを忘れると全部無効になる

```
/* NG：全部無効になる */
transition: background 0.2s transform 0.2s;

/* OK */
transition: background 0.2s, transform 0.2s;
```

カンマを忘れると指定全体が無効になる。「あれ、アニメーションが動かない」と30分ほどハマった。

### translate と translateY は別物

```
transform: translate(-2px);   /* 横に移動 */
transform: translateY(-2px);  /* 上に移動 */
```

ホバーで浮き上がらせようとして `translate(-2px)` と書いたら横にずれた。上に動かすには `translateY(-2px)`。

### rgba() の透明度は 0〜1 の小数

```
background: rgba(0, 0, 0, 15);   /* NG：ほぼ真っ黒になる */
background: rgba(0, 0, 0, 0.15); /* OK：15%の透明度 */
```

`15` と書いて「なんか全体が暗い」となった。第4引数は 0〜1 の小数。

## まとめ

CSS変数・`clamp()`・ダークモードの3つを組み合わせると、こういう設計になる。

* 色はすべて CSS変数経由で書く（直書きしない）
* 変数名は役割ベースにする（`--color-primary`）
* フォントサイズと余白は `clamp()` で流体化する
* ダークモードは `:root` の変数を上書きするだけ

技術の話もそうだけど、今回一番よかったのは\*\*「なぜそう書くか」を気軽に聞けた\*\*ことだった。

本や動画だと「こう書きましょう」で終わることが多い。でも Claude Code に「なぜ？」と聞くと、「こう書かないとこういう問題が起きる」という背景まで教えてもらえる。理由がわかると、ルールとして覚えるんじゃなくて意味として残る感じがした。

この記事で出てきた「`--blue` じゃなくて `--color-primary`」も、「`clamp()` がなめらかに変化する」も、「色を直書きするとダークモードで困る」も、全部「なぜ？」から入ったから今も覚えてる。
