## Sonnet 5の主要な特徴

### 1. エージェント能力の大幅な向上

![](https://static.zenn.studio/user-upload/e94904ef7268-20260701.webp)

Sonnet 5はブラウザやターミナルなどのツールを活用し、**自律的に計画→実行→検証**のサイクルを回せるよう設計されています。

従来のSonnetシリーズはコーディングとツール利用で先駆的な実績を残しましたが、直近のエージェント能力向上はOpusクラスのモデルが牽引していました。Sonnet 5はその差を大幅に縮め、**Opus 4.8に近い性能をより低いコストで提供**します。

![](https://static.zenn.studio/user-upload/30ac9170c936-20260701.webp)

早期アクセスパートナーからは次のようなフィードバックが寄せられています。

* 以前のSonnetが途中で止まっていた複雑なタスクを完遂できる
* 明示的に指示しなくても自分の出力を自己チェックする
* マルチステップのソフトウェアエンジニアリング作業での実行品質が向上した

### 2. ベンチマーク結果

![](https://static.zenn.studio/user-upload/0047b452d0e6-20260701.webp)

Sonnet 5はSonnet 4.6と比較して、以下の重要な指標で大幅な改善を達成しています。

| 評価項目 | 特徴 |
| --- | --- |
| 推論 (Reasoning) | Opus 4.8に迫るスコア |
| ツール使用 (Tool use) | 実務的なタスクでの完遂率が大幅向上 |
| コーディング (Coding) | 複雑なバグの根本原因を追跡して修正できる |
| 知識作業 (Knowledge work) | 法律調査・ビジネス分析など多様な領域で改善 |

**SWE-bench Pro** （実務レベルのコーディング能力を測るベンチマーク）では、63.2%とSonnet 4.6の58.1よりも高く、Opus 4.8の69.2の間の性能となっています。

**BrowseComp**（エージェント検索評価）と**OSWorld-Verified**（コンピュータ操作評価）では、エフォートレベルを調整することで「コスト」と「精度」のトレードオフを柔軟に選択できる設計になっています。

### 3. 具体的なユースケース（パートナー企業の事例）

* **コード修正**: バグ調査を依頼すると、再現テストの作成→修正実装→修正なしで症状が再現することの確認、までを1パスで自律的に完了
* **Salesforce自動化**: アカウント情報の更新→対象ユーザーへのアナウンス送信という2段階タスクをエンドツーエンドで完遂
* **レガシーコード対応**: race conditionや隠れたテストなど複雑な既存コードで症状ではなく根本原因を修正
* **法律リサーチ**: 価格性能比が高く、法律調査・分析で明確な品質向上

## 安全性評価

Anthropicの事前展開評価では、Sonnet 5はSonnet 4.6と比較してほぼすべての安全性指標で改善しています。

### 改善点

* 悪意のあるリクエストの拒否率向上
* プロンプトインジェクション攻撃への耐性向上
* ハルシネーション（幻覚）率の低下
* 過度な同調（sycophancy）の減少

### サイバーセキュリティの扱い

![](https://static.zenn.studio/user-upload/059da52ffe4b-20260701.webp)

Sonnet 5はサイバーセキュリティタスクのための意図的なトレーニングは行われていません。Firefoxの脆弱性エクスプロイト開発能力を測定した評価では、Sonnet 5は動作するエクスプロイトを作成できませんでした（成功率0%）。一般的な知能の改善に伴い部分的な成功率はわずかに上昇しましたが、Opus 4.8やMythos 5と比べて大幅に低い水準にとどまっています。

このため、デフォルトでリアルタイムのサイバーセーフガードが有効化されています。

### 位置づけ

![](https://static.zenn.studio/user-upload/46f7e76692cc-20260701.webp)

全体的な誤動作率ではSonnet 4.6を下回りますが（＝より安全）、Opus 4.8やClaude Mythos Previewと比較するとやや高い水準にあります。

## まとめ

Claude Sonnet 5は、Sonnet系モデルのコストメリットを維持しつつ、エージェント能力をOpusクラスに近づけた重要なアップデートです。

**こんな用途に特におすすめ**：

* マルチステップのコーディング・デバッグ作業
* ブラウザ/ターミナルを使った自律的なタスク実行
* 繰り返し発生する業務フローの自動化
* コスト意識の高い大規模なエージェント構築

Sonnet 5とOpus 4.8の間でエフォートレベルを調整しながら使い分けることで、コストと精度のバランスを柔軟にコントロールできます。エージェントAI開発の新たなスタンダードとして注目の一本です。
