「SWELLでClaude Codeって使えるの？」  
「LPを作ってもらえたら時短になりそう」  
「でも、SWELLと相性が悪かったりしない…？」

普段SWELLでサイト制作している私が、Claude Code と Claude Design を使って美容室LPを実際に作ってみました。

結論を先に3行で。  
・Claude Designでの設計図づくりは、かなり便利  
・SWELL実装は思っていたより詰まる（特に見出し・全幅・微調整）  
・成立はするが、SWELLの知識がないと正直キツい

「軽い気持ちで触ったら、想像以上に大変だった」

でも、やってよかった。そんな試行錯誤の記録を、できるだけそのまま書いてみます。

![](https://assets.st-note.com/img/1778397426-7OBl8QALdiE1IqcMzDY6gZNx.png?width=1200)

## 気になっていたこと

自分は普段SWELLでサイト制作をしているので、気になっていたのはこのあたりです。

・SWELLとClaude Codeは組み合わせて使えるのか  
・実際に時短になるのか  
・そもそも成立するのか

試してみるしかないと思い、やってみました。

## 結果：完成はしたが、想像以上に大変だった

最初に結論を書いておくと、完成はします。

ただ、想像していたよりずっと大変でした。

コードに詳しくなくても進めることはできますが、SWELLの仕組みはある程度理解していないと詰まります。

---

## まずはClaude Designで設計図を作る

![](https://assets.st-note.com/img/1778395964-pThqdu6JMPA3aUsDvcLBH0n2.png?width=1200)

最初にやったのは、Claude Designでデザインを作ることでした。

条件や構成をある程度伝えると、それっぽいデザインがすぐに出てきます。

ここは正直かなり便利でした。

### ただし、ざっくりすぎるとズレる

情報が少ないと、「なんとなく違う」というものが出てきます。

なので、

・構成を先に固める  
・参考サイトを渡す  
・イメージを言語化する

このあたりは丁寧にやったほうが結果は良くなります。

### もうひとつ注意点

人物やアイコンのイラストはうまく出ません。  
代わりに図形のようなものになります。  
そのため、画像素材は別で用意する前提になります。

## SWELLに実装したら、思っていたのと違った

デザインができた時点で、「あとは入れるだけだろう」と思っていました。

ここが甘かったです。

### HTMLをそのまま使う案

Claude Designで出てきたHTMLをそのまま使おうとしましたが、SWELLと競合してうまく表示されませんでした。

![](https://assets.st-note.com/img/1778446848-2CshMktBIcnoSdPLRfZO5EK4.png?width=1200)

どうやら、SWELL専用のブロックで作っていくことはできないのですね。  
これだとコードがわからないクライアントさんへの納品は難しいですね。。

![](https://assets.st-note.com/img/1778446926-Yl4okGucTI3DUy7FL9251v0g.png?width=1200)

### 方針を変更して作り直し

結局、

・フルワイドブロック  
・カスタムHTML  
・CSSで調整

という形で、SWELL用に作り直すことにしました。  
この判断自体は間違っていなかったのですが、ここからが本番でした。

## Claude Codeとやり取りしながら進める

ここからはひたすらやり取りです。

「ここをこうしたい」と伝えると、HTMLやCSSを作ってくれます。  
グローバルメニューが小さかったので修正してもらいました。

![](https://assets.st-note.com/img/1778447168-P04GkId1cYDs6CL2w7HvQoqT.png?width=1200)

![](https://assets.st-note.com/img/1778447208-UfroYAVnjaIRKHDCEgNkJsud.png?width=1200)

コードを書けなくても進められるのは確かに大きいです。

ただし、

「いい感じに」は通じません。

指示が曖昧だと、そのまま曖昧な結果になります。

## 実際に詰まったポイント

いくつか印象に残っているものを書きます。

### 見出しが勝手にSWELLのデザインになる

h2を使うと、SWELLのスタイルが当たってしまい、意図したデザインになりませんでした。

HTMLの構造を調整して対応しました。

### 全幅にならない

レイアウトがコンテナに収まってしまい、画面いっぱいに広がらない問題もありました。

これもCSSで調整して対応しています。

## 一番時間がかかったのは細かいズレ

大きな問題よりも、むしろこちらのほうが大変でした。

・余白が少し違う  
・フォントサイズが微妙に違う  
・色味が少しだけ違う

「大きくは間違っていないけど、なんとなく違う」  
この状態を修正し続ける作業が、一番時間を使いました。

## SWELLとClaude Codeの相性

良い点としては、

・デザインのたたき台がすぐできる  
・コードを自分で書かなくていい  
・外注よりコストを抑えられる

一方で、

・SWELLの知識がないと対応が難しい  
・指示の精度がそのまま結果に出る  
・細かい調整に時間がかかる

このあたりは事前に理解しておいたほうがいいです。

実際に出来上がったデモLPです。  
興味があれば見てみてください！  
<https://kenwebdesign.com/lp/lino-demo/>

## 最後に｜これから SWELL × Claude Code を試す人へ

claudeは「自分の知識と組み合わせて使うことで、はじめて力を発揮する」ものだと、今回はっきり感じました。

これから挑戦する人へ、私の経験から3つだけ伝えるとすれば：  
1. Claude Designは「たたき台づくり」と割り切る  
完成品ではなく、SWELLで仕上げる前提のラフ案として使う。

2. SWELLの構造を理解してから依頼する  
フルワイドブロック・カスタムHTMLブロック・h2の挙動を知っているかどうかで、詰まり方が変わります。

3. 「いい感じに」を捨てて、目的と条件を伝える  
曖昧な指示には曖昧な結果が返ってきます。

まだ試行錯誤の途中ですが、使い続ける価値はあると感じています。  
SWELLで時短したい同業の方に、この記録が一つの参考になればうれしいです。
