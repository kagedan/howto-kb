# Claude Codeで広がる可能性：プラグイン・Wiki・スキルの活用法

Claude Codeのエコシステムは急速に拡張しています。本記事では、注目の3つのトピック——フロントエンドスライドプラグイン、個人Wikiツール「Synthadoc」、そしてスキルの仕組み——をまとめて紹介します。

---

## 1. フロントエンドスライドプラグインを2コマンドでインストール

Claude Codeのプラグインマーケットプレイスから、フロントエンドスライド生成プラグインをわずか2つのコマンドで導入できます。

```
/plugin marketplace add zarazhangrui/frontend-slides
/plugin install frontend-slides@frontend-slides
```

マーケットプレイスを活用することで、Claude Codeの機能を手軽に拡張できます。コミュニティが開発したプラグインをそのまま利用できる点が魅力です。

---

## 2. Synthadoc：クラウド不要の個人Wiki生成ツール

Andrej Karpathy氏が提唱した「個人Wiki」のアイデアを実装したツール「**Synthadoc**」が登場しました。

### 特徴

* **ドキュメントをWikipedia形式に整理**：手持ちのドキュメントを自動的にWikipediaライクな構造へ変換します。
* **クラウド不要**：外部のクラウドサービスへの依存がありません。
* **ベンダーロックインなし**：特定のサービスに縛られず、データの所有権はユーザー自身にあります。
* **Claude Codeから直接利用可能**：追加のセットアップなしにClaude Code上で動作します。

自分のドキュメントを自分でコントロールしながら、ナレッジベースを構築したい方に最適なツールです。

---

## 3. スキル（Skills）とは何か？：コンテキストエンジニアリングの新視点

Claude Codeにおける「スキル」の概念は、単なるテキストファイルではありません。理解しておきたい重要なポイントが2つあります。

### ① スキルはフォルダである

スキルはテキストだけでなく、**スクリプト・アセット・データなどを含むフォルダ**として構成されます。ファイルシステム全体を「コンテキストエンジニアリングの一形態」として捉えると、スキルの可能性がより広く見えてきます。

> 「スキルはフォルダだ——ファイルシステム全体をコンテキストエンジニアリングの形式として考えよう」

### ② `description`フィールドはモデルのためにある

スキルの`description`フィールドは人間向けのドキュメントではなく、**モデルが読むためのもの**です。モデルが正しくスキルを理解・活用できるよう、丁寧かつ明確に記述することが重要です。
