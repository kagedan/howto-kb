Claudeの「スキル」を、自分は半年ずっと勘違いしていました。

チャットのスキルとClaude Codeのスキルは、名前もファイルも同じなのに、中身は別物です。

片方に登録しても、もう片方では動きません。

この記事は、スキルを使い始めたものの「結局どれがどれ?」で止まっている、かつての自分のような人に向けて書いています。

「スキルを使いたいけど、どうやって使ったらいいかわからない」  
「スキルを入れたはずなんだけど、全然起動してくれない」  
「人によって保存する場所の説明が違う」

そんなふうに思っている人必見です。場所と保存先と設定を、まるごと整理しました。

この記事で分かることは、次のとおりです。

① チャットとClaude Codeのスキルが、どこに保存され、なぜ別物なのか   
② 自分はどちらを使えばいいのか、3つの問いで分かる判定   
③ スキルを動かすために絶対に必要な、たった1つの設定   
④ コピペで使える「スキルをClaudeに作らせる指示文」

（機能名・設定名・画面は2026年6月時点のものです）

## そもそも「スキル」とは、毎回説明しなくて済む指示書です

![](https://assets.st-note.com/img/1781563411-aPsvWzuykXArV3g1w6D59hUx.png?width=1200)

スキルは、ひとことで言うと「**毎回説明しなくて済む指示書**」です。

たとえばサムネイルを作るとき、毎回こう書いていたら面倒です。

「このフォントで、この配色で、このサイズで作って、、、」

これをスキルに登録しておけば、「サムネ作って」の一言で同じ品質のものが出てきます。

## 「スキル」が分かりにくいのは、2つの場所に別々に住んでいるから

![](https://assets.st-note.com/img/1781563448-Mehbi6ISDLgoFAjaEpsqxf0d.png?width=1200)

問題はここからです。

このスキルが2つの場所に存在していて、それぞれ仕組みが違います。

この2つを同じものとして扱うから、話が分からなくなります。

### ① claude.ai（チャット）のスキル

ブラウザやデスクトップアプリでClaudeとチャットしている人は、こちらです。

場所は Chat → カスタマイズ → スキル。 → 保存先は**Anthropicのクラウド**（サーバー上）です。

登録方法は2つあります。   
→ 画面から操作して選ぶか、SKILL.mdファイルをZIPにしてアップロードするかです。

クラウドに保存されるので、別のパソコンからログインしても、同じアカウントなら使えます。

出張先のPCでも、スマホのアプリでも、自分のスキルがそこにあります。

### ② Claude Code（ターミナル）のスキル

Claude Codeで動かしている人は、こちらです。

場所は自分のPC内の ~/.claude/skills/ か、プロジェクトの .claude/skills/ フォルダ。 → 保存先は自分のパソコンの中です。

登録方法は、フォルダを作ってSKILL.mdファイルを置くだけです。

**ローカル（自分のPC）にある**ので、そのPCでしか使えません。

別のPCで使いたいときは、GitHubなどで同期する必要があります。

自分はこの違いを知らずに、チャットで作ったスキルがCode側で出てこなくて、しばらく固まっていました。

ただ住所が違っただけだったんです。

## 何が同じで、何が違うのか

![](https://assets.st-note.com/img/1781563483-IvGK2f5sUlb4mZNQeaJ7H8ho.png?width=1200)

ややこしいのは、**ファイルの書き方が完全に同じ**という点です。

どちらもSKILL.mdというMarkdownファイルに、名前と説明と指示を書きます。

このSKILL.mdは「Agent Skills」という共通の標準にもとづいています。 → Claude Code以外に、Codex CLIやCursorなど他のツールでも同じ形式が使えます。

なので、一度作ったスキルファイルは、チャットにもClaude Codeにもコピーして使えます。

ただし、置く場所が違うので、お互いに自動では発動しません。

チャットに登録したスキルは、チャットでだけ動きます。 → Claude Codeからは見えません。

Claude Codeに置いたスキルは、ターミナルでだけ動きます。 → チャットからは見えません。

同じ指示書でも、棚が別なら別の棚からは取り出せない。

ここを表で整理しておきます。

![](https://assets.st-note.com/img/1781478391-icpbD81Y3FEPOUhqVjGasXtr.png?width=1200)

## 自分はどっちを使えばいいのか、3つの問いで決まります

![](https://assets.st-note.com/img/1781563985-1kpiBoCNFaHwQLgPnSvVj8fR.png?width=1200)

迷ったら、この3つの問いで判断できます。

問いに答えるだけなので、難しく考えなくて大丈夫です。

### Q1. 普段Claudeをどこで使っていますか?

ブラウザやClaudeデスクトップアプリでチャットしているなら、チャットのスキルです。

ターミナルでClaude Codeを動かしているなら、Claude Codeのスキルです。

両方使っているなら、**両方に同じファイルを登録する必要があります。**

ここで1つ、つまずきポイントがあります。

「デスクトップアプリ」にはチャットとCode、両方あるという点です。

デスクトップ版のClaude Codeは、見た目はアプリでも中身はターミナルベースです。 → **スキルの管理はClaude Code側**になります。

ややこしいですが、一度確認してみてください。

### Q2. やりたいことは何ですか?

資料作成、記事生成、サムネイル作成、翻訳などなら、チャットのスキルが向いています。

コードの自動生成、ファイル操作、開発タスクの自動化なら、Claude Codeのスキルが向いています。

### Q3. ふだんコードを書く仕事をしていますか?

「いいえ」なら、まずはチャットのスキル一択で問題ありません。

ターミナル操作なしで使えます。

ただし、ここで脅しっぽく「Claude Codeをやらないと取り残される」とは言いません。

チャットのスキルだけでも、課金額はじゅうぶん回収できます。

慣れてきて「自動化までやりたい」と思ったときに、Claude Codeへ進めば間に合います。

## スキルの探し方は、大きく3つあります

![](https://assets.st-note.com/img/1781564348-NEPi4zTwlhGBQWs5g6nVktAy.png?width=1200)

「他の人が作ったスキルを探して入れたい」と思う人は多いはずです。

ただ、今のところclaude.aiの中に、大きなアプリストアのような検索機能はまだありません。

探し方は主に3つです。

### ① claude.aiの中で探す

Chat → カスタマイズ → スキル に行くと、Anthropicが用意した例（ビルトイン）のスキルが並んでいます。

トグルをONにするだけで使えるので、まずはここから始めるのが安全です。

![](https://assets.st-note.com/img/1781478845-G0lD5megBVht4E3PwkN8LSjW.png?width=1200)

![](https://assets.st-note.com/img/1781478855-oH0aVEOCrYGIjPu5D4AnTtJS.png?width=1200)

### ② Claudeに作ってもらう

正直、これが一番おすすめです。

自分は昔この発想がありませんでしたが、なんでも相談するのが一番。

「こういうスキルを作りたいので、質問してください」と伝えるだけで、聞き取りながらSKILL.mdを生成してくれます。

自分の業務に合ったスキルが、10分くらいで形になります。

そのまま使える指示文を置いておきます。

```
これからスキル(SKILL.md)を一緒に作りたいです。
いきなり書かず、まず自分に質問してください。
・どんな作業を毎回くり返しているか
・その作業のルールや好み(フォーマット、文体、禁止事項)
・完成イメージ
ヒアリングが終わったら、SKILL.md形式で出力してください。
```

### ③ 外部のマーケットプレイスで探す

コミュニティが急成長していて、無料でスキルを公開しているサイトがいくつかあります。

2026年6月時点で、代表的なものを挙げます。   
→ ClaudeSkills.info、SkillsMP.com、LobeHub あたりがメジャーです。

掲載数はどこも数十万〜百万件規模で、しかも日々増え続けています。 → 数字は変動が激しいので、ここでは固定しません。

これらのサイトからSKILL.mdをダウンロードして、claude.aiにアップロードすれば使えます。

1つだけ注意点があります。 → ZIPの中には、ファイル単体ではなくフォルダごと入れてください。

ここでもう1つ、大事な注意を足します。

外部マーケットのスキルは、GitHubから自動で集められた**未審査のものがほとんど**です。

ある調査では、2万件超のスキルから平均6.3件の問題が見つかったという報告もあります。

あまり知らないものには手を出さずに、スターが多くついているものをおすすめします。

## スキルが動かないときは、だいたいこの設定が原因です

![](https://assets.st-note.com/img/1781564909-bwgPSMW45yjr2XvzdG7q3lmu.png?width=1200)

1つだけ、忘れてはいけない設定があります。

設定(Settings) → 機能(Capabilities) → 「コード実行とファイル作成」をONにしてください。

これがOFFのままだと、スキルはそもそも動きません。

恥ずかしい話、自分は最初これを知らずに、スキルを登録して「何も起きない…」と1時間ほど固まってました。

スキルはコード実行の仕組みの上で動いています。

だから、このトグルがOFFだと、どんなに良いスキルを入れても無反応のままなんです。

登録してうまく動かないときは、まずここを疑ってください。

## 両方で同じスキルを使いたいときは、手で2か所に置きます

![](https://assets.st-note.com/img/1781564929-mTk0daYC6xnFv49uzyBN3WZj.png?width=1200)

チャットとClaude Codeの両方で同じスキルを使いたいときは、同じSKILL.mdを両方に置く必要があります。

チャット側は、ZIPにして Chat → カスタマイズ → スキル からアップロードします。

Claude Code側は、~/.claude/skills/スキル名/SKILL.md にファイルを配置します。

内容を更新したら、両方を手で更新することになります。

少し面倒ではあります。

ただ、フォーマットが共通なので、ファイルのコピーだけで済む点は助かります。

自分は映像制作を15年やってきましたが、素材を複数の場所で管理する感覚に近いです。

元データは1つ、置き場所は用途ごとに分ける。

そう考えると、二重管理もそこまで苦になりません。

## 今日からの3ステップ

![](https://assets.st-note.com/img/1781564955-ILJi75aTdSFbuBQ1CGecWZkH.png?width=1200)

名前は同じ「スキル」でも、住んでいる場所が違うだけです。

それを知っているだけで、もう混乱しません。

最後に、今日からできる3ステップを置いておきます。

① 設定 → 機能 で「コード実行とファイル作成」をONにする   
② Chat → カスタマイズ → スキル を開いて、例のスキルを1つだけONにして試す   
③ いつもの作業を1つ、Claudeにスキル化してもらう（上の指示文をコピペでOK）

自動化はまだ先でいい人は、①と②だけで十分です。

チャットのスキルを使いこなせるようになってから、Claude Codeに進んでも遅くありません。

同じ場所で止まっていた頃の自分に、一番伝えたい話でした。

今回の手順をまとめたプロンプトや設定例は、他の記事でも順番に公開しています。

AI初心者だった非エンジニアの自分がClaude Codeをどこまで使えるか、試行錯誤を発信中です。
