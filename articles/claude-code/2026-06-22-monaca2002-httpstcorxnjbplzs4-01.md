## 記事の概要

**Claude Codeの挙動を制御する7つの方法**について、それぞれの「いつ読み込まれるか」「コンパクション後の扱い」「コンテキストコスト」「適切なユースケース」を整理した公式ブログ記事（2026-06-18公開）です。

## 7つの方法と特性

① CLAUDE.md（ルート）

- **読み込みタイミング**：セッション開始時に常にロード、セッション全体で保持
- **コンパクション後**：Memoized（再読み込みされる）
- **コンテキストコスト**：高（関係なくてもトークンを消費）
- **用途**：ビルドコマンド、ディレクトリ構成、コーディング規約、チームノーム
② CLAUDE.md（サブディレクトリ）

- **読み込みタイミング**：そのサブディレクトリ配下のファイルをClaudeが読んだときにオンデマンドで
- **コンパクション後**：そのディレクトリに再アクセスするまで失われる
- **コンテキストコスト**：低
- **用途**：特定ディレクトリ固有の規約
③ Rules（.claude/rules/）

- **読み込みタイミング**：ユーザーレベルはセッション開始時、パスをスコープしたものは該当ファイルアクセス時のみ
- **コンパクション後**：再注入される
- **コンテキストコスト**：中（パススコープなしは常時）
- **用途**：特定の制約・規約（例：src/api/**配下のハンドラーはZodでバリデート必須）
④ Skills（.claude/skills/）

- **読み込みタイミング**：名前・説明はセッション開始時、**本体はスキル呼び出し時のみ**
- **コンパクション後**：呼び出されたスキルは共有バジェット内で再注入、古いものから削除
- **コンテキストコスト**：低（本体は呼ばれたときだけ）
- **用途**：手続き的なワークフロー（デプロイチェックリスト、リリース手順など）
⑤ Subagents（.claude/agents/）

- **読み込みタイミング**：名前・説明・ツールリストはセッション開始時、**本体は親会話には一切入らない**
- **コンパクション後**：サブエージェントの最終メッセージ（要約＋メタデータ）だけが返る
- **コンテキストコスト**：最低（独立したコンテキストウィンドウで実行）
- **用途**：並列作業、メインスレッドを汚さない側務タスク（深いWeb検索、ログ解析、依存関係監査）
⑥ Hooks（settings.jsonなどで設定）

- **読み込みタイミング**：ライフサイクルイベント（ファイル編集、ツール呼び出し、セッション開始など）で発火
- **コンパクション後**：コンパクションを完全に回避する
- **コンテキストコスト**：低（設定自体はメインコンテキスト外）
- **用途**：決定論的な自動化（リンター実行、Slack通知、コマンドブロック、PreCompactでのチャット履歴バックアップ）
⑦ Output Styles / システムプロンプト付加

- **Output Styles**：システムプロンプトに注入、コンパクション対象外、**デフォルトの指示を上書き**するので注意が必要
- **システムプロンプト付加（--append-system-promptフラグ）**：上書きではなく**追記**のみ、そのセッションにのみ適用
- **用途**：大きな役割変更（コードアシスタント→汎用アシスタント）や、トーン・フォーマット指定
## 特に重要なTips

**「CLAUDE.mdに Never do this を書く」のは間違い**。Claudeは「ほとんどの場合」従うが、長いセッションや曖昧な状況、プロンプトインジェクション下では失敗しうる。本当に防ぎたいことは**Hooks（PreToolUseフック＋exit code 2でブロック）**か**Managed Settings（管理者デプロイ、ユーザーがオーバーライド不可）**を使う。
