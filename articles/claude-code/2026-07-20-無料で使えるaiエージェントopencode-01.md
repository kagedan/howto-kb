# 無料で使えるAIエージェントOpenCode

ChatGPT のようなチャット AI ではなく、リポジトリを読んでコードを直す **コーディングエージェント** を試したい人向けに、OSS の [OpenCode](https://opencode.ai/) をまとめます。

ポイントは次の3つです。

* 無料モデルから始められる
* ターミナルや IDE から使える
* ローカルLLM（Ollama / LM Studio など）にも接続できる

## OpenCode とは

OpenCode は、オープンソースの AI コーディングエージェントです。  
会話するだけでなく、プロジェクト内のファイルを読み、編集し、開発作業を進められます。

公式が打ち出している特徴は、次のとおりです。

* ターミナル UI（TUI）が中心
* デスクトップアプリや IDE 拡張にも対応
* 多数のモデルプロバイダ（Claude / OpenAI / Gemini など）に接続可能
* ローカルモデルにも対応
* コードやコンテキストを OpenCode 側に保存しないプライバシー方針

要するに、無料・有料、クラウド・ローカルを問わず、さまざまなモデルを扱える AI エージェントです。

## 使い方

まずは入れて動かしてみます。モデルの選び方や `AGENTS.md` / Skills の詳細は、後半の「特徴」で補足します。

### Mac へのインストール

OpenCode を Mac に入れるいちばん簡単な方法は、インストールスクリプトです。

```
# インストールスクリプト
curl -fsSL https://opencode.ai/install | bash
```

Homebrew や npm でも入れられます。

```
# Homebrew
brew install anomalyco/tap/opencode
```

```
# npm
npm install -g opencode-ai
```

### Windows へのインストール

Windows では、次のいずれかの方法でインストールできます。

```
npm install -g opencode-ai
```

公式では、Windows では WSL の利用が推奨されています。

> **推奨: WSL を使用する**  
> Windows で最高のエクスペリエンスを得るには、Windows Subsystem for Linux (WSL) を使用することをお勧めします。これにより、パフォーマンスが向上し、OpenCode の機能との完全な互換性が提供されます。

### デスクトップ版のインストール

ターミナルに抵抗がある場合や、WSL を使うのが難しい場合は、デスクトップ版も使えます。  
次のページからダウンロードしてインストールできます。

[OpenCode のダウンロード](https://opencode.ai/ja/download)

![](https://static.zenn.studio/user-upload/c47a33e1e4b1-20260720.png)

### 起動と初期化

ターミナル版では、プロジェクトのディレクトリで `opencode` を起動します。

```
cd /path/to/project
opencode
```

起動後、プロジェクトを把握させたいときは次を実行します。

これで OpenCode がリポジトリを分析し、`AGENTS.md` を作ります。  
`AGENTS.md` の役割は、後述の「特徴」で説明します。

VSCodeのTerminalを開き、その中でopencodeを起動すると、AIへの指示と成果物を同時に確認できるので便利です。

![](https://static.zenn.studio/user-upload/280a6ccc576d-20260720.png)

デスクトップ版では、ターミナルでの `/init` の代わりに UI からプロジェクトを開きます。`プロジェクトの追加` から作業フォルダを選び、`新しいセッション` をクリックして開始します。

![](https://static.zenn.studio/user-upload/1582ee613c20-20260720.png)  
![](https://static.zenn.studio/user-upload/ff3d9e7bbea8-20260720.png)

### プロンプトを打って作業させる

あとはプロンプトを送って、エージェントに作業してもらうだけです。  
試しに、OpenCode のフォルダ内に「Web で動く縦スクロールシューティングゲームを作って」と指示してみます。

![](https://static.zenn.studio/user-upload/ca26417d39cb-20260720.png)

数分ほど待つと、HTML のシューティングゲームができあがりました。  
このあとも指示を重ねて、理想の成果物に近づけていくのが、コーディングエージェントの基本的な使い方です。

![](https://static.zenn.studio/user-upload/85848fab0a5a-20260720.png)

### よく使うコマンド

| 操作 | 内容 |
| --- | --- |
| `/init` | プロジェクト分析と `AGENTS.md` 作成 |
| `/connect` | モデルプロバイダ接続 |
| `/models` | モデルの切り替え |
| `/undo` | 直前の変更を戻す |
| `/redo` | 戻した変更をやり直す |
| `/share` | 会話を共有リンクにする |
| `Tab` | Plan / Build の切替 |

## 特徴

ここからは、冒頭の3ポイントを中心に、OpenCode の特徴を掘り下げます。

### 無料で始めやすい

OpenCode 自体は OSS です。  
また、いくつか無料で使えるモデルが用意されています。

![](https://static.zenn.studio/user-upload/07a068c48ff8-20260720.png)

触った時点で見えていた無料モデルの例は、次のとおりです。

| モデル | 作成者 | 速度・規模の傾向 | 向きそうな用途 |
| --- | --- | --- | --- |
| DeepSeek V4 Flash Free | DeepSeek | 速い（Flash系）。MoE で総パラメータは大きいが、起動する分は小さめ | コーディング全般・反復作業 |
| MiMo V2.5 Free | Xiaomi（小米） | Flash ほどではないが、推論寄りで腰を据えて考えるタイプ | 設計・方針検討・エージェント作業 |
| North Mini Code Free | Cohere | 小型寄り（コード特化の Mini）。軽くて速い印象 | コード編集・細かい修正 |
| Nemotron 3 Ultra Free | NVIDIA | Ultra 級の大型。重めになりやすい代わりに余力がある | レビュー・長文読解・複雑推論 |
| Hy3 Free | Tencent（騰訊） | Hunyuan 系の大規模 MoE。長コンテキスト向き | 汎用チャット・長めの対話 |
| Big Pickle | 非公開（ステルス） | 正体・詳細スペックは伏せられている | とりあえず試す汎用枠 |

「まず触ってみたい」という段階なら、API キーやサブスクリプションなしで始められる手軽さが魅力です。  
迷ったら **DeepSeek V4 Flash Free**（速さ）か **North Mini Code Free**（編集）、方針検討なら **MiMo**、重い読み込みなら **Nemotron**、という選び方で十分実用的です。

上記はいずれも執筆時点で見えていた例です。無料モデルの顔ぶれは入れ替わりが速いので、最新の一覧は `/models` で確認してください。

ただし無料モデルは、恒久提供とは限りません。  
試用・フィードバック目的の提供である可能性が高いので、常用するなら有料APIや自前プロバイダも用意しておくと安心です。

### ターミナルでも IDE・デスクトップでも使える

OpenCode の主戦場はターミナルです。  
VS Code を開き、統合ターミナルで起動する使い方が自然です。

```
VS Code
└─ ターミナル
   ├─ OpenCode
   ├─ npm test
   ├─ npm start
   └─ git diff
```

普段の開発と同じ場所で、次の流れを回しやすいです。

1. エージェントに修正を任せる
2. 差分確認
3. テスト実行
4. ブラウザでの動作確認

ターミナルが苦手な場合は、前述のデスクトップ版から入る選択肢もあります。  
ただし開発フロー全体としては、差分確認やテスト実行まで含めてターミナル／IDE と組み合わせる前提で考えると扱いやすいです。

### 好きなモデルに切り替えられる

OpenCode は、特定ベンダーのモデルに固定されません。  
無料モデルだけでなく、有料のOpenAI / Claude / Gemini などにも接続できます。

OpenAI や Claude は、API キーを環境変数に置いておくと自動的に認識され、`/models` に表示されます。  
キーは [OpenAI](https://platform.openai.com/) や [Anthropic](https://console.anthropic.com/) のコンソールから発行します。

**OpenAI（`OPENAI_API_KEY`）**

```
# Windows PowerShell
$env:OPENAI_API_KEY = "sk-..."
opencode
```

```
# macOS / Linux
export OPENAI_API_KEY="sk-..."
opencode
```

**Claude（`ANTHROPIC_API_KEY`）**

```
# Windows PowerShell
$env:ANTHROPIC_API_KEY = "sk-ant-..."
opencode
```

```
# macOS / Linux
export ANTHROPIC_API_KEY="sk-ant-..."
opencode
```

毎回設定するのが面倒な場合は、シェルの設定ファイル（`~/.zshrc` など）や Windows のユーザー環境変数に登録しておくと便利です。

環境変数を使わない場合は、TUI で `/connect` を実行し、プロバイダ（Anthropic など）を選んで API キーを貼り付ける方法もあります。

無料モデルで試し、必要になったら有料モデルへ切り替える、という段階導入がしやすいです。

### ローカルLLMも使える

クラウドのAPIを使わず、手元のマシンで動かすローカルLLMにも接続できます。  
コードやプロンプトを外部に出したくないときや、オフラインで試したいときに向いています。

よく使われるのは、次のようなランタイムです。

| ランタイム | エンドポイント例 |
| --- | --- |
| [Ollama](https://ollama.com/) | `http://localhost:11434/v1` |
| [LM Studio](https://lmstudio.ai/) | `http://127.0.0.1:1234/v1` |
| llama.cpp（llama-server） | `http://127.0.0.1:8080/v1` |

いずれも OpenAI 互換の API を公開するので、OpenCode 側では `opencode.json` にプロバイダを追加する形でつなげます。

Ollama の設定例です。

```
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "ollama": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "Ollama (local)",
      "options": {
        "baseURL": "http://localhost:11434/v1"
      },
      "models": {
        "qwen3-coder": {
          "name": "Qwen3 Coder (local)"
        }
      }
    }
  }
}
```

Ollama なら、次のコマンドで OpenCode 向けに起動する方法もあります。

このコマンドは環境変数や設定ファイルなしで OpenCode を起動できますが、Ollama v0.15 以降が必要です。また OpenCode はコンテキスト長 64k 以上のモデルを前提とします。

### `AGENTS.md` でリポジトリごとのルールを渡せる

`/init` で作られる `AGENTS.md` には、「このリポジトリではこう動いてほしい」と書けます。  
`AGENTS.md` 自体は、OpenCode 専用ではなく、Cursor や Claude Code など複数のエージェントが読む **共通フォーマット** です。仕様と事例は [agents.md](https://agents.md/) にまとまっています。

例:

```
- 実装前にコードを読む
- 大きな書き換えをしない
- 依存パッケージを勝手に追加しない
- テストを通す
- 変更内容を短く説明する
```

毎回プロンプトにルールを書かなくても、リポジトリ側に制約を置けるのが便利です。  
エージェントを「優秀な後輩」だとすると、`AGENTS.md` は社内ルールに近い存在です。

書き方の参考にしたい場合は、次のサイトも役立ちます。

* [agents.md](https://agents.md/) — フォーマットの公式サイト。OpenAI Codex や Apache Airflow など、有名リポジトリの AGENTS.md 例が載っています
* [AgentsMD Space](https://agentsmd.space/) — コミュニティが投稿した AGENTS.md / CLAUDE.md のテンプレート集

### Skills で定型作業を手順化できる

OpenCode は Skills に対応しています。  
`.opencode/skills/<name>/SKILL.md` を置くと、エージェントが必要なときに読み込めます。

たとえば、次のような Skill を置けます。

| Skill | 依頼例 |
| --- | --- |
| `start-server` | 「サーバ起動して」 |
| `run-tests` | 「テスト実行して」 |
| `commit-push` | 「コミットしてプッシュして」 |
| `create-pr` | 「PR作って」 |

`AGENTS.md` が「いつも守るルール」なら、Skills は「よくやる仕事の手順書」です。  
丸投げではなく、決まった手順で作業させたいときに効きます。

## まとめ

OpenCode は、無料で始めやすい OSS のコーディングエージェントです。

* 無料モデルから始められる
* ターミナルや IDE から使える
* ローカルLLM（Ollama / LM Studio など）にも接続できる

「AIにコードを聞く」から一歩進んで、  
「AIと一緒にリポジトリを直す」体験を試すなら、候補としてかなり有力です。
