## はじめに

友達や営業先と予定を合わせるとき、毎回「TimeTree を開いて空いてる時間を目で数えて、メッセージに書く」という作業が発生しています。  
この作業を Claude Code のカスタムスキルに切り出して、`/timetree-free-slots` コマンド一発で完了するようにしたので紹介します。

### 想定読者

* Claude Code で独自の `/コマンド` を作ってみたい人
* Playwright MCP でブラウザ操作を伴うスキルを作りたい人
* 日程調整のような地味な反復作業を自動化したい人

Claude Code・Playwright MCP・TimeTreeアカウント（ログイン済み）があれば、記事内のSKILL.mdをほぼそのまま流用できます。

## As-Is / To-Be

As-Is: TimeTree のウィークリービューを目で見て、予定の合間を計算してメッセージに書く

To-Be: `/timetree-free-slots` を実行するだけで、以下のようなテキストがそのまま出てくる

```
6/24(水) 09:00-09:30 / 11:30-12:00 / 17:00-18:00
6/25(木) 09:00-09:30 / 12:00-15:00 / 15:15-18:00
6/26(金) 09:00-09:30 / 10:30-11:00 / 15:00-18:00
6/27(土) 09:00-12:00 / 12:30-16:30
6/28(日) 09:00-18:00
6/29(月) 09:00-09:30 / 10:30-18:00
6/30(火) 09:00-09:30 / 10:45-13:00 / 13:30-14:00 / 15:00-16:30 / 17:00-18:00
7/1(水) 09:00-09:30 / 10:30-11:30 / 12:00-13:00 / 15:00-15:30 / 15:45-18:00
```

コードブロックで返るので、そのままLINEやメールにコピペできます。時間帯を省略すると `9:00-18:00`、範囲を省略すると「今日から1週間後まで」がデフォルトです。

## 仕組み

Claude Code には `SKILL.md` を1枚書くだけで `/コマンド名` を作れる仕組みがあります。手順として書いた内容を、Claude がツール（ブラウザ操作など）を実行しながらなぞってくれます。

```
/timetree-free-slots 実行
  ↓
① Playwright MCP で TimeTree Web を開く
  ↓
② ログイン画面ならユーザーに手動ログインを依頼して停止
  ↓
③ 対象週分のウィークリービューを snapshot 取得
  ↓
④ snapshot から各曜日の予定を抽出（午前/午後表記→24時間制）
  ↓
⑤ 指定時間帯から予定の区間を除外して空き時間を算出
  ↓
⑥ コピペ用コードブロックで出力
```

ブラウザ操作には Playwright MCP（`browser_navigate` / `browser_snapshot` / `browser_handle_dialog`）を使い、`https://timetreeapp.com/calendars/<calendar_id>/weekly/<YYYY-MM-DD>` にアクセスして週単位のビューを取得します。TimeTreeへのログインだけは自動化しておらず、②の時点でログイン画面が出た場合はユーザーに手動ログインを依頼していったん停止します。

## なぜこの構成にしたか

TimeTreeは[2023年12月に公式APIの提供を終了しており](https://timetreeapp.com/intl/ja/newsroom/2023-12-14/connect-app-api-202312)、今回はログイン済みのブラウザセッションをそのまま使えるPlaywright MCPでWeb画面を読む方式にしました。

ログインの自動化もあえてしていません。ID・パスワードをファイルに書きたくなかったので、セッションが切れていたら処理を止めてユーザーに再ログインを任せる設計にしています。

## 実際のSKILL.md

日程調整のたびに何度も使うことが前提なので、毎回同じ挙動になるよう確認や個人差が出る余地を減らしました。実際に動いている `SKILL.md` の全文を参考として貼ります。カレンダーIDや文言など、自分の環境に合わせて適宜調整してください。

実際のSKILL.md全文

```
---
name: timetree-free-slots
description: TimeTreeのウィークリー画面から空き時間（指定時間帯）を抽出し、「x/xx(曜日) HH:MM-HH:MM」形式で出力する。営業・友人など日程調整全般に使う。範囲は未指定なら「今日から1週間後まで」、他に「今週（カレンダー通り）」「今日からN日間」を選べる。使い方: /timetree-free-slots [時間帯（例: 9:00-18:00）] [range=week|next<N>（例: next10）]
---

## TimeTree Free Slots Skill

日程調整の際に、TimeTreeから今日以降の空き時間を抽出するスキル。
Playwright MCPでTimeTree Webにアクセス → 対象期間のウィークリービューを読み取り → 指定時間帯から既存予定を引いた空き時間を出力する、というフローを固定化する。

---

## 固定方針（ユーザーに確認しない）

- AskUserQuestion は使わない
- 引数で時間帯を受け取る。未指定なら `9:00-18:00`
- 出力はチャットのみ（ファイルは作らない）
- 本作業（ブラウザ操作・抽出）は メインスレッドで直接実行する。ログインが必要になった場合にその場で気づいて対応できるようにするため、バックグラウンド実行はしない
- 出力は コピペできるよう必ずコードブロック（```）で囲む。メールやチャットにそのまま貼れる形にする
- フォーマットは `M/D(曜) HH:MM-HH:MM / HH:MM-HH:MM` の素のテキスト（先頭の `- ` は付けない）
- 既にTimeTreeにログイン済み・ウィークリービュー表示済みの想定で動く。ログインしてなければユーザーに依頼して停止

## Workflow

### Step 1: 引数パース

- `HH:MM-HH:MM` 形式があれば時間帯としてパース。未指定なら `START=9:00, END=18:00`
- `range=week` または `range=next<N>`（Nは日数）を見る。未指定なら「今日から1週間後まで」（= `今日〜今日+7日`、8日間）とする。過去日を含む `week`（カレンダー通りの月曜始まり）は明示的に指定された時だけ使う
  - `week`: 今週の月曜〜日曜（7日）。過去の曜日も含まれるので、明示指定がない限り使わない
  - `next<N>`: 今日〜今日+(N-1)日（N日間）— 「今日からN日間」を返したい時に使う
- ユーザーが「今日から」のような自然文で頼んできた場合も、未指定時と同じデフォルト（今日〜今日+7日、8日間）を使う。「今日からN日間」と日数が明示されたら `range=next<N>`（today〜today+(N-1)日、N日間）とする

### Step 2: 対象日付の特定

```bash
# 今日
date +%Y-%m-%d
# 今週の月曜
date -v-mon +%Y-%m-%d
# 今日からN-1日後（Nは日数）
date -v+<N-1>d +%Y-%m-%d
# 今日+7日後（未指定時のデフォルトの終了日）
date -v+7d +%Y-%m-%d
```

- 未指定（デフォルト） → `TARGET_DATES = 今日〜今日+7日（8日間）`
- `range=week` → `TARGET_DATES = 今週の月曜〜日曜（7日）`
- `range=next<N>` → `TARGET_DATES = 今日〜今日+(N-1)日（N日間）`

`TARGET_DATES` がカバーするウィークリービュー（=月曜起点の週）を列挙する：
- `TARGET_DATES` の開始日が属する週の月曜から、終了日が属する週の月曜まで、1週間ずつ進めて列挙する
- 例: 今日が水曜で `next10`（終了日は今日+9日）の場合、今週・来週・再来週の3週分のビューが必要になることがある

### Step 3: TimeTreeのウィークリービューを開く

必要な週ぶん `https://timetreeapp.com/calendars/<id>/weekly/<Monday>` に順次ナビゲートして `browser_snapshot` を取る。
すでに目的の週が開いていれば再ナビゲートは不要。

ログイン画面が表示された場合のみ: snapshot がウィークリービューではなくログイン画面（メールアドレス/パスワード入力フォームなど）であれば、「TimeTreeにログインしてから再実行してください」とユーザーに伝えて処理を停止する。ウィークリービューが表示されていればこのチェックは何もしない。

「カレンダーを1つ以上選択してください」ダイアログが出たら `browser_handle_dialog accept:true` で閉じる。

### Step 4: 予定をスナップショットから抽出

snapshot内の `button "<タイトル> <開始時刻> - <終了時刻> avatar of ..."` 要素から、各曜日の予定一覧を取得。
TimeTreeは曜日ごとに `generic` ブロックが分かれているので、月→日の順に予定を集める。

時刻表記は `午前9:00 / 午後3:00` 形式なので24時間制に変換する：
- `午前X:YY` → `0X:YY`（午前12は `00:YY`）
- `午後0:YY` → `12:YY`
- `午後X:YY (X≥1)` → `(X+12):YY`

### Step 5: 各曜日の空き時間を算出

各曜日について：
1. 指定時間帯 `[START, END]` を初期空き範囲とする
2. その日の予定で `[START, END]` と重なる区間を全て除外
3. 残った連続区間が空き時間

注意点：
- 予定の終了時刻 == 開始時刻 の点状予定（例: 締切表示）は無視する
- 予定が `END` を超えても、`END` までの部分だけ考慮
- 予定が `START` 未満から始まっても、`START` 以降の部分だけ考慮
- 「終日」予定（タイトルのみで開始/終了時刻が無い予定）は空き時間の判定に使わない。完全に無視し、その日を埋まり扱いにしない

### Step 6: 出力

コピペ用にコードブロックで囲んで出力する：

````
```
M/D(月) HH:MM-HH:MM / HH:MM-HH:MM
M/D(火) HH:MM-HH:MM
M/D(水) 空きなし
...
```
````

曜日記号: 月火水木金土日（日本語1文字）
空き時間が全く無い日は `M/D(曜) 空きなし` と出す。先頭の `- ` などは付けない（メールにそのまま貼れる素のテキスト）。

## 使ってみた所感

実際の日程調整で何度か使ってみましたが、TimeTree を目視して計算する時間がゼロになったのが一番の収穫でした。

* TimeTreeのような目視系の反復作業ほど、スキル化の費用対効果が高い
