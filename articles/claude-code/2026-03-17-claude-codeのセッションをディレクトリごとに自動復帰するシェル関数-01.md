# はじめに

Bash / Zsh にシェル関数を定義して、Claude Code の「セッションがあれば復帰、なければ新規作成」を自動で切り替えるTipsです。

## 何が不便なのか

Claude Code でプロジェクトの作業をしていて、一度ターミナルを閉じたり `claude` を終了した後に「さっきの続きをやりたい」という場面はよくあります。

このとき、前回のセッションを復帰するには `--continue` オプションを使います。

しかし、カレントディレクトリにまだセッションが存在しない場合（初めてそのディレクトリで作業する場合など）、`--continue` は以下のようなエラーになります。

```
No recent conversations found in this directory. Starting a new conversation.
```

つまり、毎回「このディレクトリで前に作業したっけ？」を考えて、

* 作業したことがある → `claude --continue`
* 初めて → `claude`

と使い分ける必要があります。これが地味に面倒です。

逆に、普段から `claude --continue` を癖にしていると、新しいディレクトリで作業を始めるたびにエラーメッセージが表示されて気になります。

そこで、この切り替えを自動で行うシェル関数を紹介します。

# Claude Code のセッションとは

Claude Code は会話の履歴を**セッション**という単位で管理しています。セッションはディレクトリごとに保存されるため、プロジェクトAで作業した会話とプロジェクトBで作業した会話は別々のセッションになります。

## 保存場所

セッションは以下のディレクトリに JSONL 形式で保存されます。

```
~/.claude/projects/<ディレクトリパス>/<セッションID>.jsonl
```

たとえば `/home/yousan/my-project` で作業した場合、以下のようなパスになります。

```
~/.claude/projects/-home-yousan-my-project/ca9ee964-9a4e-4666-b242-733e0a9603e0.jsonl
```

ディレクトリのパスがそのままフォルダ名になりますが、`/` が `-` に置換されています。

## セッションファイルの中身

JSONL ファイルの各行には、ユーザーの入力やアシスタントの応答、ツールの実行結果などが JSON オブジェクトとして記録されています。主な `type` は以下の通りです。

| type | 内容 |
| --- | --- |
| `user` | ユーザーの入力メッセージ |
| `assistant` | Claude の応答 |
| `progress` | ツール実行の進捗 |
| `file-history-snapshot` | ファイル変更のスナップショット |

これにより、セッションを復帰した際に過去の会話コンテキストをすべて引き継ぐことができます。

## セッション関連のオプション

Claude Code にはセッションを扱うオプションがいくつかあります。

| オプション | 説明 |
| --- | --- |
| `--continue` (`-c`) | カレントディレクトリの**最新のセッション**を自動的に復帰する |
| `--resume` (`-r`) | セッションIDを指定して復帰する。IDを省略すると対話的にセッションを選択できる |
| `--session-id <uuid>` | 特定のセッションIDを指定して会話する |
| `--fork-session` | `--continue` や `--resume` と併用し、元のセッションを残したまま新しいセッションとして分岐する |

`--continue` は「とにかく前回の続き」、`--resume` は「特定のセッションを選んで復帰」という使い分けです。

```
# 最新のセッションを復帰
claude --continue

# 対話的にセッションを選んで復帰
claude --resume

# セッションIDを指定して復帰
claude --resume ca9ee964-9a4e-4666-b242-733e0a9603e0
```

# 関数の定義

お使いのシェルの設定ファイルに以下を追加します。

**Zsh の場合** (`~/.zshrc`)

```
claude() {
  command claude --continue "$@" 2>/dev/null \
    || command claude "$@"
}
```

**Bash の場合** (`~/.bashrc`)

```
claude() {
  command claude --continue "$@" 2>/dev/null \
    || command claude "$@"
}
```

関数の内容は同じです。追加したら `source ~/.zshrc` または `source ~/.bashrc` で反映させます。

関数名を `claude` にしていますが、関数内では `command claude` で実体を呼んでいるため再帰にはなりません。

この関数を通さずに素の `claude` コマンドを直接呼びたい場合は、以下のいずれかの方法が使えます。

```
# バックスラッシュを前置する
\claude --help

# command キーワードを使う
command claude --help

# フルパスで指定する
/home/yousan/.claude/local/claude --help  # パスは環境による
```

# 何をしているのか

この関数は2段階で動作します。

1. **まず `--continue` 付きで起動を試みる** — カレントディレクトリに過去のセッションがあれば、そのセッションを復帰して続きから作業できます
2. **失敗したら `--continue` なしで起動する** — セッションが存在しない場合は `--continue` がエラーを返すので、`||` により通常の新規セッションとして起動します

`2>/dev/null` で `--continue` 失敗時のエラーメッセージを抑制しているため、ユーザーからは常にスムーズに起動するように見えます。

# ポイント解説

## `command` キーワード

`command` は zsh/bash の組み込みキーワードで、**エイリアスをバイパスして元のコマンドを直接実行**します。

たとえば `~/.zshrc` に以下のようなエイリアスがあっても、`command claude` はエイリアスを無視してオリジナルの `claude` コマンドを呼びます。

```
alias claude='claude --dangerously-skip-permissions'
```

これにより、エイリアスと関数が循環参照してしまう問題を防いでいます。

## `"$@"` — 引数の透過的な受け渡し

`"$@"` は関数に渡されたすべての引数をそのまま `claude` コマンドに渡します。これにより、以下のように通常の `claude` コマンドと同じ感覚で使えます。

```
claude "このコードをリファクタリングして"
claude --model sonnet "テストを書いて"
```

# 使い方

```
# プロジェクトディレクトリに移動して
cd ~/my-project

# 初回: 新規セッションが開始される
claude

# 2回目以降: 前回のセッションが自動的に復帰される
claude

# 引数も渡せる
claude "前回の続きで、テストも追加して"
```

# まとめ

| 項目 | 内容 |
| --- | --- |
| 関数名 | `claude` |
| セッションあり | `--continue` で復帰 |
| セッションなし | 新規セッションを開始 |
| エラー表示 | `2>/dev/null` で抑制 |
| エイリアス衝突 | `command` で回避 |

たった3行の関数ですが、「セッションがあるかないか」を意識せずに `claude` だけで作業を開始できるのは非常に快適です。

# 応用: 権限確認をスキップする

毎回の確認プロンプトが煩わしい場合は、`--dangerously-skip-permissions` を追加することもできます。

**Zsh の場合** (`~/.zshrc`)

```
claude() {
  command claude --dangerously-skip-permissions --continue "$@" 2>/dev/null \
    || command claude --dangerously-skip-permissions "$@"
}
```

**Bash の場合** (`~/.bashrc`)

```
claude() {
  command claude --dangerously-skip-permissions --continue "$@" 2>/dev/null \
    || command claude --dangerously-skip-permissions "$@"
}
```

このオプションを使うと、Claude Code がファイル編集やコマンド実行を確認なしに行います。インターネットアクセスのないサンドボックス環境や、信頼できるプロジェクトでの利用が推奨されています。
