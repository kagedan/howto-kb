Claude Codeを使い始めて50日目。正直に言う。最初の2週間、私はこのツールをまったく使いこなせていなかった。「AIがコードを書いてくれる」と期待していたのに、なぜか作業が全然終わらない。そのギャップに気づいたのは、使い始めて15日目のことだった。

最初の頃、私のやり方はこうだった。エラーが出る→エラーメッセージをそのままコピー→「直して」と貼り付ける。これを繰り返すだけ。1日目に4時間かかった作業が、翌日も3時間半かかった。何も改善されていない。

転機は15日目だった。試しに「このエラーが起きる理由を、初心者にもわかるように説明してほしい」と聞き方を変えてみた。Claude Codeは「環境変数が設定されていないため、ファイルパスが見つかりません」と丁寧に教えてくれた。初めて「なるほど、そういう仕組みか」と腑に落ちた瞬間だった。

そこからは面白いように変わった。同じ種類のエラーに次の日も遭遇したとき、「あ、環境変数かも」と自分で気づけた。試行錯誤の回数が5回から1〜2回に減り、1作業あたりの時間が平均4時間→45分まで縮んだ。数字にすると地味だが、感覚的には別物だ。「何をやっているか理解した上で動いている」という手応えが生まれた。

副業的な成果でいうと、50日目の今週、note経由での問い合わせが初めて2件届いた。「どうやって自動投稿の仕組みを作ったんですか？」という内容だ。Claude Codeを使った仕組みを少しずつ記事で公開してきた積み重ねが、ようやく人の目に留まり始めた気がする。

AIに「やってもらう」から「一緒に理解する」へ。この思考のシフトが、副業を継続する上で一番大切なスキルかもしれない。答えだけ受け取り続けても、次のトラブルに自分では対応できない。でも仕組みがわかれば、応用できる。それが積み重なって、はじめて収益につながると実感している。

あなたはAIツールを使うとき、「答えだけ」を求めていますか？それとも「なぜそうなるか」も一緒に理解しようとしていますか？どちらのアプローチがしっくりくるか、ぜひコメントで教えてください。
