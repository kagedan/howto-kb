Claude Code、結局なに？

Claude Codeという名前は、前から見かけていました。

一緒にSNSに出てくるのは、「AI社員」「動くAI」「コードを書くAI」といった言葉。なんとなくすごそうではある。多分🤔でも、その言葉が何を言っているのかも、Claude Codeで結局何ができるのかも、全然分かりませんでした。

AI社員と言われると、勝手に仕事を見つけて、最初から最後まで全部やるものを想像してしまう。動くAIと言われても、何がどう動くのか分からない。コードを書くAIとなると、黒い画面に英語を打つ人の道具になのか？

![](https://assets.st-note.com/img/1783960958-7L9hFMVJAzW3IUnQcCojtarl.png?width=1200)

最初に頭の中で混ざっていた言葉

そこでClaude Codeの超入門を読みました。読めば分かるだろうと思ったのですが、スキル、サブエージェント、権限、差分。知らない言葉が次々に出てくる。その言葉をそのまま覚えようとしても、やっぱり頭に入りません。

なので今回は、用語を覚えるのではなく、自分が分かる作業に一つずつ直してみました。

## 「AI社員」と言われても、何をするのか分からなかった

資料を読んで見えてきたのは、もっと小さい話でした。

自分が選んだフォルダの中を読んで、頼んだ作業を進める。必要ならファイルを作ったり、文章を直したり、作ったものが動くか確認したりする。そして、何をしたのかをこちらへ返す。

自分の中では、「AI社員」より「面倒なファイル作業を頼める道具」の方が分かりやすい。

![](https://assets.st-note.com/img/1783960986-hN9TCb1Ael3sqrMwY8HQfx0O.png?width=1200)

自分なりに言い換えたClaude Code

以前、Codexで返信準備メーカーという小さいWeb道具を作った時、フォルダの中にいくつかファイルができました。Claude Codeも、会話の返事だけで終わらず、実際のファイルを扱う側の道具だと考えると、少しつながりました。

自分が毎日触っているnoteの文章やMarkdownファイルも、パソコンの中にあるファイルです。そこを読んだり整理したりするなら、自分にも関係があります。

## 「動く」を、自分の知っている作業に直してみる

「実行」と言われても、何を実行するんだよ？？？  
読んでみると、文章を直す、ファイルを作る、ちゃんと動くか確かめる。そういうところまでやる話でした。

これまでは、AIの返事を自分でコピーして、必要な場所へ貼っていました。  
Claude Codeは、その貼るところまでファイル側でやる。そこが今までと違うらしい。

![](https://assets.st-note.com/img/1783961010-p3X9LJTkjQVAfsqmiZUcECOg.png?width=1200)

普通のチャットAIとClaude Codeの違い

読んでいくと、全部を勝手に任せるわけではなく、変更してよいか最後に決めるのは自分だと分かりました。

![](https://assets.st-note.com/img/1783961035-PaWdHM02TsvjzYmlqeQCKf3F.png?width=1200)

自分がする4つの流れ

自分なら、まずこんなふうに頼みます。

「このフォルダにある文章ルールと現在地のファイルを読んで、同じ内容や似た内容を一覧にしてください。まだファイルは変更しないでください」と

![](https://assets.st-note.com/img/1783961056-jkqr43BUv9mbKApHxuMNnFQi.png?width=1200)

最初に頼む文章

これなら、何を読んでほしいかと、まだ何をしてほしくないかが自分でも分かります。

ここまで読んで、ようやく「動くって、ファイルを触ることか」となりました。

## スキルは手順書、サブエージェントは別担当

次に出てきたのが、Skills（スキル）とSubagents（サブエージェント）です。

スキルと聞くと、また何か新しい機能を覚えるのかと思いました。資料の説明を自分の作業に直すと、「何度も頼む作業の手順書」に近いものでした。

自分はnoteを書くたびに、文章ルール.mdやnote記事生成ルール.mdを読ませています。先生っぽくしない、参考記事の言い換えにしない、指定文字数を無理に伸ばさない。毎回同じ説明を打たなくていいように、先にファイルへ書いてあります。

スキルそのものはまだ使えていませんが、同じ説明を手順書へまとめる考え方なら、すでに少しやっていました。

サブエージェントは、作業を別担当に分けるものでした。

これは今の自分にはまだ早いです。

そもそも、自分が一つの作業をどこで分ければよいのか、まだうまく言えないからです。何人ものAIへ仕事を分ける前に、一つのフォルダで一つの作業を頼めるようになる方が先です。

![](https://assets.st-note.com/img/1783961082-Y1p0ovNc2I7BMhSKaGWfdsei.png?width=1200)

## ファイルを変えられるのは、やっぱり少し怖い

Claude Codeは、会話で答えるだけではなく、文章を書き換えたり、新しいファイルを作ったりします。

そこは便利そうだけど、普通に怖いです。

大事な文章を別の内容へ書き換えられたらどうするのか。必要なファイルを消されたらどうするのか。個人情報が入ったフォルダまで読まれたらどうするのか。「安全に使えます」とだけ言われても、自分が知りたいのはそこでした。

資料を読むと、変更する前に内容を確認して、進めるか止めるかは自分で決められるようです。

それでも、最初から大事な原本では試したくありません。自分なら、個人情報の入っていないコピーを作って、まずそこで試します。

![](https://assets.st-note.com/img/1783961105-KcB7AFDs82tX9TlrJNQMZELY.png?width=1200)

怖いところと最初の対策

その上で、「変更はまだしないで、最初に作業内容を見せてください」と頼む。

何を許可しているのか分からないまま進めるのは、まだ怖いです。

資料の最後には、詳しい内容を確認するための公式ページも載っていました。

![](https://assets.st-note.com/img/1783963377-vNnr4gIjFbkRKwAZVWtyQE39.png?width=1200)

## 自分なら、何を一つ頼めそうか

ここまで言葉を直してみると、Claude Codeで何を作るかより先に、自分がどの作業を面倒だと思っているかを考える話になりました。

今の自分なら、note用フォルダの整理が一番近いです。

文章ルール、現在地、素材メモ、記事の下書きが増えてくると、どこに何を書いたか分からなくなる。同じルールを別のファイルへ何度も書いていることもあります。

だから最初は、ファイルを直してもらうところまで行かなくていい。フォルダの中を読んで、何が入っているか、同じ内容がどこにあるかを一覧にしてもらう。その結果を見てから、まとめるか、そのまま残すかを自分で決める。

まだClaude Codeを使いこなせるとは言えません。実際に触れば、また迷うと思います。今の自分が最初に頼めそうなのは、note用フォルダの中身を一覧にしてもらうところまでです。
