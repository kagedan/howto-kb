🆕 Claude Code v2.1.162 リリース！

📌 主な変更点:
• `claude agents --json` が待機セッションのブロック理由を表示
• スラッシュコマンドのオートコンプリートがクリック時に即実行ではなく入力に挿入
• Remote Control が起動メッセージではなく永続的なフッター表示に
• 設定ディレクトリが書き込み不可の場合の起動ハング問題を修正
• Windows のパーミッションルール（バックスラッシュ表記等）のマッチング不具合を修正

https://t.co/8c0hsT3j4y

🚀 新機能

【Agents 管理】
• `claude agents --json` に `waitingFor` フィールドを追加：待機中のセッションが何でブロックされているか（例: パーミッション承認待ち）を表示
• セッション名の切り詰めを廃止：端末幅に応じて名前列が拡張
• ライブステータステキストが端末の全幅を使用（従来は60〜120カラムで切り詰め）
• Ctrl+V での画像貼り付けが dispatch 入力欄とセッション返信ボックスで動作するように修正
• セッションをバックグラウンド化（←キー）した際、バックグラウンドサービスが起動できない場合でも会話が失われず、失敗行として残る
• 送信失敗した返信がキューに保持され、次回セッション開始時に配信
• 実行中のバックグラウンドセッションを開く際の5秒間の遅延を解消

【UX 改善】
• スラッシュコマンドをオートコンプリートメニューでクリックすると、即座に実行ではなくプロンプトに挿入されるように変更（Enter で実行）
• `/effort` コマンドで選択したレベルが新規セッションのデフォルトとして保持されることを確認表示
• Remote Control が起動メッセージではなく、セッションへのリンク付き永続フッターピルとして表示
• Windsurf を Devin Desktop にリネーム（`/ide` メニュー、`/terminal-setup`、`/scroll-speed`）、エディタのリブランドに対応

【起動表示】
• 起動時の通知を重要度別にグループ化、セッション情報とアナウンスを1行に集約
• 起動ワーニングを短く明確に書き直し、各問題に具体的な修正方法を提示
• ディープリンク/事前入力プロンプトの警告が、操作するまで入力欄下部に固定表示
• 失敗したターンが複数行の赤いエラーブロックではなくコンパクトな警告行に
• 「Claude in Chrome enabled」「marketplace installed」メッセージを削除、モデル自動更新とチームオンボーディングのヒントはロゴ下の控えめな通知として表示

🐛 バグ修正 ⚡ パフォーマンス

【ツール・検索】
• `--tools` で Grep/Glob を明示的に列挙すると、組み込み検索を持つネイティブビルドで専用検索ツールが提供されるように修正（従来は無視されていた）
• LSP ツールの `workspaceSymbol` 操作が結果を返さない問題を修正：`query` パラメータを受け入れ、言語サーバーに渡すように

【パーミッション・セキュリティ】
• 設定ディレクトリが読み取り専用または書き込み不可の場合の起動ハングを修正：インメモリ設定で起動し、空白画面ではなく起動エラーを表示
• WebFetch パーミッションルールが組み込み事前承認ドメインに適用されない問題を修正：明示的な `WebFetch(domain:...)` の deny/ask/allow ルールが事前承認ホストの自動許可より優先
• Windows でバックスラッシュ表記（`~\`、`\\server\share`）や大文字小文字が異なるパスでパーミッションルールがマッチしない問題を修正
• Read deny ルールが Glob/Grep 結果からファイルを隠さない問題を修正

【セッション・API】
• ターンの最初に送信された中断（Esc）が stream-json/SDK セッションで無視され、「Interrupted」フィードバックなしでターンが実行され続ける問題を修正
