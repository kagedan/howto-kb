## x402 とは何か (30 秒)

[x402](https://github.com/x402-foundation/x402) は HTTP `402 Payment Required` を実用化した micropayment protocol です。Coinbase が 2025 年に提唱し、現在は x402 Foundation が標準を運営しています。

流れはシンプルです。

1. AI agent が API を叩く
2. サーバが `402` で「いくら、どこに、どの token で払えばよいか」を返す
3. agent が USDC を送金 (Base mainnet)
4. agent が `X-Payment` header 付きで再リクエスト
5. サーバが受領を検証して本来のレスポンスを返す

API キーもサインアップも不要、コール単位で料金が確定する。**人間 SaaS の Stripe と違い、agent が autonomous に支払う前提**で設計されている、というのが核です。

## なぜ marketplace が複数あるのか

x402 は protocol であって、marketplace ではありません。「402 を返す seller」と「402 を解釈して払う buyer」がいれば、その間の **discovery 層・decision 層** に多様な実装が成立する余地があります。

実際、現在は次のような軸で各 marketplace が分かれています。

* **公式リファレンス** (Coinbase 自身が運営)
* **垂直特化** (LLM API に絞る、など)
* **MCP 体験フル装備** (Claude Desktop / Cursor から直接 browse)
* **検索 / 索引** (買い手側 tool)
* **言語・地域コミュニティ向け**

ecosystem はまだ早期で、6 ヶ月後には統廃合 or 1-2 強の構図になっている可能性もあります。本記事は **2026-05-09 時点のスナップショット**として読んでください。

## 主要 marketplace 7 つ

### 1. Coinbase Agentic.Market — 公式リファレンス

| 項目 | 内容 |
| --- | --- |
| URL | <https://agentic.market> |
| 運営 | Coinbase (x402 protocol 自体とは独立した product) |
| 規模 | **694 APIs** (inference / data / search / media / social / trading / infra / storage / travel) |
| Network | Base + Solana |
| Take-rate | 非公開 |
| MCP | TypeScript skills (`agentic-wallet-skills` npm) |
| 買い手 surface | Web UI + SDK + REST + `llms.txt` |

ecosystem の中で最大の catalog 規模を持ち、Coinbase ブランドと流動性が backed しています。**「とりあえず何があるか俯瞰したい」「公式に近いものを使いたい」用途に最も向く**選択肢です。一方で TypeScript skills + Coinbase 製 SDK が前提なので、Python や Rust メインの dev は少し回り道が必要です。

### 2. Apiosk — agent 決済インフラへ pivot 中

| 項目 | 内容 |
| --- | --- |
| URL | <https://apiosk.com> |
| 運営 | Olivier Brinkman (個人 / EU) |
| 立ち位置 | 2026-02 launch 当初は API marketplace、現在は **B2B agent 向け決済インフラ**寄り |
| Rails | SEPA (EU 銀行送金) + USDC (Base) のデュアル |
| 認可 model | KYB ベースの mandate 方式 (per-tx 承認なし、事前枠内で agent が動く) |
| MCP | 言及なし (CLI / Python / curl 中心) |

dev.to での launch 時点 (2026-02) には「9 default API + 90/10 split」を打ち出していましたが、その後 **agent が銀行レール (SEPA) と USDC を同じ gateway 経由で扱える**点を前面に出すように positioning が変わっています。

「marketplace で API を売る/買う」ではなく「**自社の agent に予算枠を持たせて autonomous に外部 API を叩かせたい、しかも SEPA も USDC も同じ ledger で見たい企業ユース**」が今の Apiosk の主戦場です。日本の dev 個人でカジュアルに API を売りたい・買いたいだけなら、後述の他選択肢のほうがフィットします。

### 3. BlockRun.AI — LLM gateway 特化

| 項目 | 内容 |
| --- | --- |
| URL | <https://blockrun.ai> |
| 規模 | **55+ モデル** (Claude / GPT-5 / Gemini / DeepSeek / Grok ほか) |
| 範囲 | chat + image + video + music の全モダリティ |
| Network | Base + Solana |
| Take-rate | **provider cost + 5% margin** (薄め) |
| MCP | Yes (`one install. 8 agent tools`) |
| 認証 | unitary、API キー不要、署名なし call OK の "free" モデルも提供 |

x402 を「agent から LLM を叩く統一 gateway」として活用したい場合、現状もっとも厚いのが BlockRun です。Claude/GPT-5/Gemini を MCP の単一 tool で切り替えられるので、**「モデル比較を実装したい」「ユーザの subscription を肩代わりするのではなく per-call で課金したい SaaS を作る」用途**に強い。

逆に「自分の独自データ API を載せたい」用途には設計が合わない (LLM proxy gateway であって、汎用 marketplace ではない)。

### 4. the402.ai — MCP-native marketplace

| 項目 | 内容 |
| --- | --- |
| URL | <https://the402.ai> |
| 範囲 | データ API + 自動 service + 人手 delivered work |
| 特徴 | escrow 付きで marketplace 体験を MCP に統合 |
| MCP | Yes (Claude Desktop / Cursor / Windsurf 内で完結) |
| 規模 | 公開数値なし |
| Network | Base |

特徴は「**Claude Desktop の中で marketplace を一通り回せる**」点です。catalog browse、purchase、自分の seller listings の管理、earnings tracking まで MCP 経由で操作できます。escrow が入っているので、配送品 (人手 delivered work など) の payout タイミングをコントロールできるのも他にない要素。

「Claude Desktop 中心の生活をしていて、その中で API も人手 service も買いたい」**個人開発者・freelance 寄り**にいちばんハマりやすい marketplace です。catalog 規模・運営体制は公開数値が乏しいので、本格的に build する前に手元で挙動を検証することをおすすめします。

### 5. EntRoute — curated registry

| 項目 | 内容 |
| --- | --- |
| URL | <https://entroute.com> |
| 規模 | **37 capabilities / 13 categories** |
| 検証 | 10 分ごとの自動テスト + 上位は実 paid request で validate |
| 範囲 | finance (17) / communications (5) / security (4) ほか |
| MCP | Yes (`@entroute/mcp-server`、npx で 1 コマンド install) |
| Network | Base |

数を追わず、**curated + 品質保証**を前面に出す方針です。10 分間隔で endpoint を叩いて落ちている API を弾く運用は、agent が build-time に "壊れた API を踏む" 確率を下げる効果があります。category や capability ごとにきちんと整理されているので、**discovery で迷う時間を削りたい**用途に向いています。

### 6. Fluora — developer preview の MonetizedMCP 実装

| 項目 | 内容 |
| --- | --- |
| URL | <https://www.fluora.ai> |
| ステータス | **developer preview** |
| 標準 | MonetizedMCP + x402 (peer-to-peer agent 間 transaction を志向) |
| 規模 | 公開数値なし |
| 特徴的 product | Kula (旅行 × 健康 practitioner マッチング、launching soon) |

まだ developer preview の段階ですが、**「MCP 自体を monetize の単位にする」MonetizedMCP** を採用している点が他と違います。x402 と組み合わせて、agent 同士が直接 service を売買する layer を狙っています。本番運用に組み込むには時期尚早ですが、protocol レイヤーで何が起きているかをウォッチしたい dev は登録だけしておく価値があります。

### 7. x402-market — 日本語 docs + MIT オープンソース (筆者運営)

> 🙋 **disclosure**: ここから先は筆者 ([Shota Asai](https://github.com/AsaiShota)) が運営している marketplace です。ポジショントークになりすぎないよう、**他社と同じ表構造**で先に数値を並べます。

| 項目 | 内容 |
| --- | --- |
| URL | <https://x402-market-api.lien-studio-akiyama.workers.dev> |
| Stack | Cloudflare Workers + Supabase + viem (フル MIT 公開) |
| 規模 | 7 listings (内 4 が cdp-mode、Coinbase Bazaar に indexed 済み) |
| Take-rate | **10% platform / 90% seller** |
| MCP | `@x402-market/mcp-server` (stdio) + HTTP variant |
| Network | Base mainnet |
| 特徴 | **自動 refund 判定** (status='failed' or latency>30s で承認) |
| 言語 | Zenn / Qiita に**日本語チュートリアル**が複数本 (ecosystem で唯一) |

x402-market が他の marketplace と差別化している軸は次の 3 つです。

1. **日本語 docs / コミュニティ向け執筆**: Zenn・Qiita を Web 検索しても、x402 marketplace 系の日本語コンテンツは他に見当たりません (protocol 解説記事は除く)。日本の dev が手を動かす最初の一歩として、日本語の手順書・実装サンプルが揃っているのは現時点で唯一の選択肢です。
2. **MIT で end-to-end オープンソース**: server (API + DB schema)、buyer SDK (TypeScript / Python)、MCP server すべてが MIT で GitHub に公開されています。「fork して自社運用したい」「supply chain を内部で完全管理したい」夢を持つチームに刺さります。
3. **自動判定の refund flow**: API call が `status='failed'` か `latency_ms > 30000` を満たすと自動で refund を承認、payout cron で返金します。\*\*「壊れた API を agent が踏んでも自動で返金される」\*\*仕組みは、agent 経済の摩擦を下げるうえで重要なんですが、surveyed competitors の中では明示的に refund mechanism を documenting しているものを他に見つけられませんでした。

現在は 7 listings を公開しており、うち 4 件 (cdp-mode) が Coinbase Bazaar に indexed 済みです (2026-05-09 時点)。Bazaar 側の indexer 仕様に追従するまで段階的な fix を要しましたが、現状は安定して discovery される状態にあります。

正直に言うと、catalog 規模では他の選択肢に劣ります (7 vs 数百〜数万)。流動性で選ぶなら Agentic.Market、LLM gateway なら BlockRun が現状ベターです。x402-market が役立つのは「**まず自分が日本語で挙動を理解したい**」「**fork できるオープンソース実装が欲しい**」「**refund 設計を踏襲したい**」用途です。

## (補足) x402list.fun — marketplace ではなく「検索屋」

| 項目 | 内容 |
| --- | --- |
| URL | <https://x402list.fun> |
| 規模 | **23,630 services / 893 providers** (8 networks に分散) |
| 役割 | seller 向けの marketplace ではなく、buyer 側の横断検索 tool |

「全 marketplace ・全 network を横断して、〇〇系の API を探したい」用途では x402list.fun が現時点で最も網羅的です。各 marketplace の listing が裏でこの index に流れこむ構造なので、**buyer は x402list.fun から discovery → 個別 marketplace の URL を叩く**という使い方が現実的です。

x402-market を含む各 marketplace は seller 側の門で、x402list.fun はそれらを束ねる buyer 側の探索 layer、と整理できます。

## どれを選ぶ? (用途別ガイド)

| 用途 | おすすめ |
| --- | --- |
| 公式・大手志向、catalog 規模で選ぶ | **Coinbase Agentic.Market** |
| LLM API を agent で叩きたい | **BlockRun.AI** |
| Claude Desktop の中で marketplace 体験を完結させたい | **the402.ai** |
| curated・品質重視で discovery したい | **EntRoute** |
| MonetizedMCP の動向を追いたい / 早期参入したい | **Fluora** |
| EU + Fiat レール (SEPA) も agent から扱いたい | **Apiosk** |
| **日本語 docs で挙動を理解したい / fork できる OSS 実装が欲しい / refund フロー込みで使いたい** | **x402-market** |
| 全 marketplace 横断で API を検索したい (買い手) | **x402list.fun** |

「最強の 1 つ」は存在しません。自分の優先軸 (規模 / モダリティ / 言語 / OSS / fee 構造) で選ぶ前提です。

## まとめ

* x402 ecosystem は 2026-05 時点で 7 marketplace + 検索 tool が並走している早期段階
* 各社の差は「catalog 規模 × MCP 体験 × 専門領域 × 言語コミュニティ × OSS 度」の組み合わせで決まる
* 日本の dev / AI agent 開発者にとって、**日本語コンテンツを起点に自走できる選択肢は現状 x402-market だけ**だが、規模で選ぶなら Agentic.Market、LLM 用途なら BlockRun が strict にベター
* 6 ヶ月後にはまた状況が変わっている前提で、本記事は **2026-05-09 時点のスナップショット**

筆者 ([Shota Asai](https://github.com/AsaiShota)) は x402-market を運営しつつ、日本固有の API データ (野球統計、行政データ、為替データなど) を seller として供給する検証も並行しています。日本の dev コミュニティに x402 が根付いていく過程を、引き続きこの場で書いていきます。
