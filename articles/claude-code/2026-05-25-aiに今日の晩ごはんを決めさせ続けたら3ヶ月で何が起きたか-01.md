## 動機：最強の意思決定疲れ

エンジニアの一日を思い浮かべてほしい。

設計の意思決定、PRレビューの判断、技術選定の選択…。仕事で脳のリソースを使い果たした帰り道、「今日の晩ごはん何にしよう」で詰まる。

**もう全部AIに決めてもらえばいいのでは？**

そう思い立ち、3ヶ月間の実験を始めました。

---

## 実験のルール

- 毎日夜、AIに「今日の晩ごはんを決めてください」と投げる
- AIの提案は基本的に従う（拒否権は週1回だけ）
- プロンプトを少しずつ改善していく
- 技術的な工夫を記録する

使用モデル：Claude Sonnet（コスパと品質のバランスで選択）

---

## 第1週：素朴なプロンプトの限界

最初のプロンプトはシンプルでした。

```
今日の晩ごはんを1つ提案してください。
```

**結果：**
- 月：親子丼
- 火：カレー
- 水：パスタ
- 木：炒飯
- 金：カレー（2回目）

……カレーが2回来た。

問題は「コンテキストがない」こと。AIは前回何を食べたか、自分の好みが何か、冷蔵庫に何があるかを知らない。当然です。

---

## 第2週：プロンプトエンジニアリングで改善

制約条件を追加しました。

```
今日の晩ごはんを1つ提案してください。

制約：
- 調理時間：30分以内
- 今週すでに食べたもの：親子丼、カレー、パスタ、炒飯
- 苦手な食材：セロリ、パクチー
- 1人前
```

**結果：** かなりマシになった。バリエーションが出て、苦手食材も避けてくれる。

でも毎回「今週食べたもの」を手で入力するのが面倒くさい。

---

## 第3週：状態管理を自動化する

ここで少し技術的な話になります。

「今週食べたもの」を毎回手入力するのをやめて、ログを自動管理する仕組みを作りました。

```python
import json
from datetime import datetime, timedelta
from anthropic import Anthropic

client = Anthropic()

def load_meal_log(filepath="meals.json"):
    try:
        with open(filepath, "r") as f:
            return json.load(f)
    except FileNotFoundError:
        return []

def save_meal(meal_name, filepath="meals.json"):
    log = load_meal_log(filepath)
    log.append({
        "date": datetime.now().strftime("%Y-%m-%d"),
        "meal": meal_name
    })
    with open(filepath, "w") as f:
        json.dump(log, f, ensure_ascii=False)

def get_recent_meals(days=7):
    log = load_meal_log()
    cutoff = datetime.now() - timedelta(days=days)
    recent = [
        entry["meal"] for entry in log
        if datetime.strptime(entry["date"], "%Y-%m-%d") >= cutoff
    ]
    return recent

def suggest_dinner():
    recent = get_recent_meals()
    recent_str = "、".join(recent) if recent else "なし"

    prompt = f"""今日の晩ごはんを1つ提案してください。

制約：
- 調理時間：30分以内
- 直近7日間に食べたもの（重複を避けてください）：{recent_str}
- 苦手な食材：セロリ、パクチー
- 1人前
- 提案は料理名と簡単な理由を1〜2文で

形式：
料理名：〇〇
理由：〜〜〜"""

    response = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=256,
        messages=[{"role": "user", "content": prompt}]
    )
    return response.content[0].text

# 実行
suggestion = suggest_dinner()
print(suggestion)

# 食べた料理を記録（実際は提案内容をパースして保存）
save_meal("回鍋肉")
```

これで「今週食べたもの」の管理が自動化されました。

## 第6週：RAGで「好み」を学習させる

ここからが本番です。

毎日の食事に評価をつけて、それをもとに好みを推定する仕組みを作りました。いわゆるRAG（Retrieval-Augmented Generation）の簡易版です。

```python
def get_favorite_patterns():
    log = load_meal_log()
    # 評価4以上の料理を抽出
    favorites = [
        entry for entry in log
        if entry.get("rating", 0) >= 4
    ]
    return favorites[-10:]  # 直近10件

def suggest_dinner_with_preference():
    recent = get_recent_meals()
    favorites = get_favorite_patterns()

    fav_str = "、".join([f["meal"] for f in favorites]) if favorites else "データなし"

    prompt = f"""今日の晩ごはんを1つ提案してください。

参考にしてほしい情報：
- 直近7日間に食べたもの（重複を避けてください）：{"、".join(recent) or "なし"}
- 好んで食べている料理の傾向（評価が高かったもの）：{fav_str}
- 苦手な食材：セロリ、パクチー

好みの傾向を考慮しつつ、マンネリにならないよう提案してください。"""

    response = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=256,
        messages=[{"role": "user", "content": prompt}]
    )
    return response.content[0].text
```

**効果：** 明らかに「外れ」が減りました。自分が麺類と一汁三菜系を好むらしいことをAIが把握し始めた。

## 3ヶ月の総括

### 数字で振り返る

| 指標 | 結果 |
|---|---|
| 実験期間 | 92日間 |
| AIの提案に従った率 | 87日（94%） |
| 拒否権を使った回数 | 5回 |
| 平均評価（5段階） | 3.8点 |
| 特に良かった提案 | 鶏むね肉の梅しそ蒸し、豆腐チゲ |
| 最悪だった提案 | 酢豚（作るのが大変すぎた） |

### 技術的に学んだこと

**1. コンテキストの質が全て**
AIの提案精度は、渡す情報の質に正比例します。「苦手食材」を追加した途端に外れが減った。

**2. 状態管理は自動化すべき**
手で入力する運用は3日で破綻します。ログの自動保存と読み込みは必須。

**3. RAGは効く（小規模でも）**
「好みの傾向」を渡すだけで、推薦の質が体感できるレベルで向上しました。100件もデータがあれば十分機能します。

**4. フォーマット指定は大事**
「料理名と理由を1〜2文で」と指定することで、毎回パースしやすい形式で返ってくるようになった。
