# ■ 「2倍」は嘘。実質2.6倍払わされてる

![](https://pbs.twimg.com/media/HMim1nDbcAAc3BE.jpg)

Fable 5の公式料金は、入力$10/100万トークン、出力$50/100万トークン。Opus 4.8が入力$5、出力$25。

だから「Opus 4.8の2倍」と言われてます。

**嘘です。実質2.6倍です。**

Fable 5は新しいトークナイザーを使ってます。同じ日本語のテキストを送っても、**Fable 5は30〜35%多くトークンを消費します。**

つまり、同じプロンプトを送っても、Opus 4.8より3割多くカウントされる。

- 単価: 2倍（$5 → $10）
- トークン数: 1.3倍（同じテキストなのに3割増し）
- **実質コスト: 2.6倍**
僕もこれに最初ハマりました。「まあ2倍くらいなら許容範囲かな」と思って使い始めたら、請求額が想定の3割増し。原因を調べて初めてトークナイザーの仕様変更に気づきました。

**対策はシンプルです。** 

Token Counting APIで事前に見積もりを取ること。`model: "claude-fable-5"` を指定しないと、旧モデルのトークン数で計算されて見積もりがズレます。

ほとんどの人が知らないこの罠。**最初に知っておくだけで「思ったより高い」は消えます。**

# ■ プロンプトキャッシュで入力コスト90%カット

![](https://pbs.twimg.com/media/HMim7ZsbEAAHVFd.jpg)

ここからが本題です。Fable 5のコストを劇的に下げる最大の武器が、**プロンプトキャッシュ**です。

## ▍ 同じプロンプトに毎回フルで払ってませんか

APIを使ってると、毎回同じシステムプロンプトを送りますよね。コーディングガイドライン、会社の文書テンプレート、分析ルール。

これ、毎回$10/100万トークンを払ってます。

プロンプトキャッシュを使えば、**2回目以降は$1/100万トークン。90%オフ。**

最初の1回だけキャッシュ書き込み料金がかかります。5分間有効のキャッシュで$12.50、1時間有効で$20。でも2回目以降は$1。

同じプロンプトを10回送るなら、キャッシュなしで$100のところが、キャッシュありで**$21.50。** 5分の1以下です。

## ▍ Fable 5だけが持ってる優位性

ここが面白いところです。モデルごとのキャッシュ条件を比較してみましょう。

- Opus 4.8 / Sonnet 5: **1,024トークン以上**でないとキャッシュ不可
- Fable 5: **512トークンからキャッシュ可能**
Fable 5は最も料金が高いのに、キャッシュのハードルは最も低い。より短いプロンプトでもキャッシュの恩恵を受けられます。

「Fable 5は高い」と言われてるけど、**キャッシュ効率では全モデル中で最強。** この事実、僕の周りでも知ってる人はほとんどいません。キャッシュ前提で設計すれば、Fable 5が一番コスパがいいモデルになります。
