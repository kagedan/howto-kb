# 脆弱性を可視化して💕

**セキュリティ脆弱性の深刻度（CVSSスコアに基づく）を、ファッションとして可視化するAIエージェントスキル**を作成しました。

## ソースコード（Claude Code スキル）

Claude Code スキルは [Agent Skills](https://agentskills.io/) オープンスタンダードに従っており、複数の AI ツール全体で機能しますが、当方での動作確認は Claude Code のみです。

<https://github.com/WATab2000/zennfes-spring-2026-perfect>

## なぜ作ったか？

セキュリティリテラシーが高い職場は、実はそう多くはないと思っています。  
だからこそ、近年も有名企業における個人情報漏えいのニュースが後を絶たないのではないでしょうか？

Claude Mythos のような新しい脅威が現実味を帯びてきている昨今、自社プロダクトのセキュリティ状況に不安を感じているエンジニアも多いはずです。

そこで、セキュリティの重要性がなかなか伝わりにくい職場の上司や経営層に向けて、自社プロダクトの脆弱性が「どれだけ危険なのか」を直感的に理解してもらうための AI エージェントスキルを、YouCam API を使って開発しました。

**たとえば、あなたが自社プロダクトの脆弱性を見つけたとします。**

**「上司！ CVSS スコアが 9.8 です！」**

そう伝えても、セキュリティリテラシーが低い職場では、 **「ふーん」** で終わってしまうでしょう。数字だけでは、危機感はなかなか伝わらないものです。

しかし、

**「上司！職場にパンダの着ぐるみを着てくる人くらいやばいです！」** と言って、**その画像**を見せたらどうでしょうか。

さすがに **（この格好くらいヤバいのか？）** と、直感的に伝わるはずです。

**使い方も簡単。**  
本コードは、Claude Code のスキルとして動作します。

あなたは  
**「このリポジトリの脆弱性を可視化して」**  
と自然言語で依頼するだけです。

その後、AIエージェントが、コードを静的解析し、脆弱性の深刻度を「ファッションの奇抜さ」として画像化します。

脆弱性が多く、深刻であるほど、出力されるファッションは常識的な装いから崩れていきます。穴だらけのコードを放置することは、パンダの着ぐるみ姿で会議に出るようなものです。

**セキュリティに詳しくない人にも、画像一枚で現状の「危うさ」や「恥ずかしさ」が伝わるでしょう。**  
それが、このスキルを作った理由です。

### 修正 → 再可視化のサイクル

画像と同時に、脆弱性の詳細が書かれたレポートが生成されます。  
そのレポートをClaude Code に読み込ませて修正し、もう一度「脆弱性を可視化して」と頼んでみましょう。  
**ほら、スコアが下がって日常的な服装が戻ってきました。**  
ファッションで安全を実感できる、あなたが待ち望んでいたAIエージェントスキルであることを実感できるはずです。

## YouCam API

Perfect Corp (YouCam) の Clothes VTO API と Claude Code スキルを組み合わせ、 **セキュリティレビューの結果（危険度）を「着ているものの奇抜さ」で表現** しています。

* CVSSスコアが高い（危険）ほど、着るものがカオスになる。

| CVSS | 重大度 | ファッションスコア | 見た目 |
| --- | --- | --- | --- |
| 0.0 | None | 1 | ビジネススーツ |
| 2〜4 | Low | 2〜3 | ジャケットが1〜2枚派手な色 |
| 4〜6 | Medium | 4〜6 | 形が変（パフスリーブ・バルーンパンツ） |
| 7〜9 | High | 7〜9 | セーラー服・着ぐるみ |
| 9〜10 | Critical | 10 | パンダ着ぐるみ＋タータンキルト（最大カオス） |

## 基本の使い方

Claude Code 上で自然言語で指示するだけ。

### メインユースケース：セキュリティ可視化

「**XXXの脆弱性を可視化して**」と話しかけるだけ。

```
https://example.com の脆弱性を可視化して
```

Claude が次の4ステップを自動実行する：

1. **セキュリティレビュー** — コード or URLを静的解析、脆弱性を検出
2. **CVSSスコア算出** — 各脆弱性に CVSS v3.1 基本スコアを付与
3. **ファッション画像生成** — 最高CVSSスコアの脆弱性1件のみを対象にVTO画像を1枚生成
4. **レポート保存** — 画像と同じファイル名・同じディレクトリに `.txt` で脆弱性レポートを保存

```
results/result_20260606123456.jpg  ← 生成画像
results/result_20260606123456.txt  ← 脆弱性レポート（全件 + 最高スコア詳細）
```

### スキル単体で使う

```
# ファッション画像だけ生成
スコア7のファッションにして

# セキュリティレビューだけ
このリポジトリのセキュリティレビューして
```

## スキル一覧

スキルは `.claude/skills/` に置かれた Claude Code のカスタム命令。  
`SKILL.md` に記載された手順を Claude が自律的に実行する。

### `security-visualize` — セキュリティ可視化（メインスキル）

**トリガー**：「**XXXの脆弱性を可視化して**」（XXX はパス・URL・「このリポジトリ」等）

セキュリティレビュー → CVSSスコア算出 → ファッション画像生成 → レポート保存を一気通貫で実行する。

**CVSS → ファッションスコアのマッピング**：

```
fashion_score = max(1, min(10, round(highest_cvss)))
```

| CVSS基本スコア | ファッションスコア |
| --- | --- |
| 0.0 | 1 |
| 0.1〜1.9 | 1 |
| 2.0〜3.9 | 2〜3 |
| 4.0〜5.4 | 4〜5 |
| 5.5〜6.9 | 6〜7 |
| 7.0〜8.9 | 8〜9 |
| 9.0〜10.0 | 10 |

### `security-review` — セキュリティレビュー単体

**トリガー例**：「セキュリティレビューして」「このURLの脆弱性を調べて」

チェック項目：

| カテゴリ | 具体例 |
| --- | --- |
| インジェクション | SQLi、コマンドインジェクション、XSS、XXE |
| 認証・認可 | ハードコードシークレット、弱いハッシュ（MD5/SHA1）、JWT検証省略 |
| 設定・通信 | HTTP使用、証明書検証無効（`verify=False`）、CORS `*` |
| 依存関係 | 既知脆弱バージョン、メンテ停止パッケージ |
| 情報漏洩 | スタックトレース露出、ログへの機密出力、`.env` の git 追跡 |

出力形式：

```
## [HIGH] コマンドインジェクションの可能性
CVSSスコア: 7.8 (AV:L/AC:L/PR:N/UI:R/S:U/C:H/I:H/A:H)
場所: `app.py:42`
概要: subprocess を shell=True で呼び出している
影響: 外部入力が混入した場合、任意コマンドを実行される
修正案: shell=False + リスト形式でコマンドを渡す
```

### `funny-fashion` — ファッション画像生成単体

**トリガー例**：「スコア10のファッション」「レベル3にして」

ファッションスコア（1〜10）を指定すると、毎回異なる服を Wikipedia から探して VTO 画像を生成する。

**ファッションスコア体系**：

| スコア | 変化の種類 | 変えるパーツ数 | 説明 |
| --- | --- | --- | --- |
| 1 | なし | 0 | ビジネススーツ（革靴・長ズボン・白シャツ・ジャケット） |
| 2 | **色**を個性的に | 1 | ジャケット・長ズボン・シャツのうち1つだけ派手な色 |
| 3 | **色**を個性的に | 2 | 2つを派手な色 |
| 4 | **形**を個性的に | 1 | 1つだけ変わった形（オーバーサイズ等） |
| 5 | **形**を個性的に | 2 | 2つを変わった形 |
| 6 | **形**をとても個性的に | 2 | 2つを極端な形（バルーン・極端なオーバーサイズ） |
| 7 | **チグハグ** | 1 | 1つを全く別の服に差し替え |
| 8 | **チグハグ** | 2 | 2つを全く別の服に差し替え |
| 9 | **チグハグ** | 3 | 3つを全く別の服に差し替え |
| 10 | **チグハグ** | 全部 | 全パーツを全く別の服（最大カオス） |

**チグハグとは**：上半身か下半身のどちらか一方だけを全く別ジャンルの服に替えること。  
片方だけ着替えた「場違い感」が笑いの核心。

スコアが同じでも毎回 Wikipedia から異なる参照画像を動的に取得するため、  
同じスコアを繰り返しても結果が変わる。

## `funny_fashion.py` の仕組み

VTO の1ステップを実行するシンプルなラッパースクリプト。  
Claude スキルがこれを複数回チェーンして多ステップの服替えを実現する。

```
python3 funny_fashion.py <画像パス> <カテゴリ> <参照画像URL>
# カテゴリ: upper_body | lower_body | full_body
```

### 実行フロー

```
参照画像URL
    ↓ urllib でダウンロード（User-Agent付き）
    ↓ PIL で JPEG 変換
    ↓ Perfect Corp S3 に ref_file_id としてアップロード
                           ↑
元画像（model.jpg）  ──────────────────→ POST /task/cloth-v3
                           ↓
                        ポーリング（3秒間隔、最大90秒）
                           ↓
                        results/result_{timestamp}.jpg に保存
```

### なぜ参照画像を手動でアップロードするか

Wikipedia/Wikimedia の画像は直接 URL を API に渡すとホットリンク保護でブロックされる。  
`fetch_and_upload_ref()` が事前にダウンロード・変換してアップロードすることで回避。

### 複数ステップのチェーン（スキルが自動実行）

2パーツ以上を変える場合、前のステップの出力画像を次のステップの入力にする：

```
set -a && source .env && set +a

# ステップ1: 上半身を変える
STEP1=$(python3 funny_fashion.py model.jpg upper_body "https://..." | grep "完了" | sed 's/.*→ //')

# ステップ2: 前の出力を入力にして下半身を変える
python3 funny_fashion.py "$STEP1" lower_body "https://..."
```

## API の概要（Perfect Corp YouCam Clothes VTO）

ベースURL: `https://yce-api-01.makeupar.com`  
認証: `Authorization: Bearer YOUR_API_KEY`

```
1. POST /s2s/v2.0/file/cloth-v3        → S3 pre-signed URL と file_id を取得
2. PUT  <pre-signed URL>               → 画像を S3 に直接アップロード
3. POST /s2s/v2.0/task/cloth-v3       → VTO タスク作成、task_id を取得
4. GET  /s2s/v2.0/task/cloth-v3/{id}  → ポーリングして結果取得
```

`garment_category` の種類：

| 値 | 適用範囲 |
| --- | --- |
| `upper_body` | ジャケット・シャツ・トップス |
| `lower_body` | パンツ・スカート |
| `full_body` | 全身（ワンピース・着ぐるみ等） |
