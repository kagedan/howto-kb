最近、Claude Codeの進化が非常に速いです。  
外部システムと連携するための「MCP」や定期実行が行える「Routines」などが次々と追加され、  
開発業務に欠かせない存在となっています。  
この記事では、Claude Codeでできることをざっくりとまとめてみました。

## Claude Codeとは？

Claude Codeとは、Anthropic社が提供するAIコーディングエージェントです。  
主な特徴は以下の通りです。

* プログラムを作成することができる
* ターミナルから操作することができる
* Git操作ができる
* テスト実行ができる
* 外部ツールと連携ができる
* 業務の自動化ができる

Claude Codeは開発業務全体の効率化を図ることができるツールです。

## 5. Hooks

Hooksとは特定の操作を実行する前後に自動で処理を実行する仕組みのことです。

例えば、

* プログラムの変更後にテストを実行
* プログラムの変更後にコードフォーマットを実行
* プログラムの変更後に Lint（静的解析）を実行

などが可能です。

## 6. Routines

Routinesとは、Claude Codeをクラウド上で自動実行するための仕組みです。

| Trigger | 内容 |
| --- | --- |
| Scheduled | 定期実行 |
| API | API経由で実行 |
| GitHub | GitHubイベント |

### 1. Scheduled Trigger

スケジュールに基づいて定期実行します。

**使用例：**

* 毎朝 8:00 にPRレビュー
* 毎晩 23:00 にセキュリティ脆弱性チェック
* 月初にリリースノート生成

### 2. API Trigger

外部システムからAPIを呼び出して実行します。

**使用例：**

* 本番デプロイ完了後にリリース内容をレビュー
* 障害アラート発生時にログを収集・分析
* セキュリティスキャン完了後に結果を要約

### 3. GitHub Trigger

GitHub上のイベントに基づいて実行します。

**使用例：**

* PR作成時のコードレビュー
* PR更新時の再レビュー
* PRマージ後の影響範囲調査

## 8. Subagents

Subagentsとは、Claude Code内で特定の役割を持つAIエージェントを作成し、作業を分担できる機能です。  
例えば、実装担当・レビュー担当・テスト担当などを分けて実行することが可能です。

Claude Codeのターミナルで以下を実行します。

```
agents
↓
Library
↓
Create new agent
↓
Generate with Claude
```

を実行し、エージェントの役割を入力します。

```
Laravelアプリケーションのコードレビューを行うエージェント
```

また、Subagentsは`.claude/agents/`配下にMarkdownファイルとして定義することもできます。

```
.claude/
└── agents/
    └── laravel-reviewer.md
```

```
--- 
name: laravel-reviewer
description: Laravelコードレビュー担当
tools: Read, Grep, Glob
model: sonnet
--- 

あなたはLaravelアプリケーションのコードレビュー専門のエージェントです。 
LaravelおよびPHPのベストプラクティスに従い、 コードレビューしてください。
レビュー時は単なる指摘だけでなく、 「なぜ問題なのか」「どう改善すべきか」まで具体的に説明してください。

確認項目:
- 可読性
  - 命名
  - 責務分離
  - Fat Controller
  - 重複コード

- Laravel設計
  - FormRequest利用
  - Eloquentの適切な利用
  - Serviceクラスへの分離
  - config/envの適切な利用

- セキュリティ
  - SQLインジェクション
  - XSS
  - Mass Assignment
  - バリデーション不足

- パフォーマンス
  - N+1問題
  - ループ内クエリ
  - 不要なデータ取得

- 保守性
  - テスト容易性
  - 拡張性
  - ハードコード
```

## 9. Skills

Skillsとは、Claude Codeに特定の作業手順やノウハウを覚えさせ、必要なときに再利用できる機能です。

例えば、

* Laravel API の実装
* テストコード作成
* React レビュー
* APIドキュメントの生成

などが可能です。

`.claude/skills/`配下にスキルごとのディレクトリを作成し、その中に説明や手順を記述した SKILL.md を配置します。

```
.claude/
└── skills/
    └── laravel-api-development/
        └── SKILL.md
```

```
---
name: laravel-api-development
description: Laravel API の実装
---

まず、依頼されたタスク内容を確認し、APIの目的・仕様・入出力内容を確認してください。

実装時のガイドライン:
- Fat Controller を避ける
- FormRequest を利用する
- Resource を利用する
- N+1問題を防ぐ
- 適切な HTTP Status を返却する
- validation を適切に実装する
- Service クラスへ責務分離する
- Feature Test を作成する
- セキュリティを考慮する
- 可読性・保守性を意識する
```

## 10. Slash Commands

Slash Commandsは、/ から始まるコマンドを利用して Claude Codeの操作を効率化する機能です。  
Claude Codeには標準でいくつかの組み込みコマンドが用意されており、利用することができます。

例えば、

* /clear
* /security-review
* /review
* /insights

などのコマンドがあります。

## 11. CLAUDE.md

CLAUDE.md は、Claude Codeに対するプロジェクト共通の指示やルールを定義するためのファイルです。Claude Codeは作業時に CLAUDE.md を参照し、記載された開発ルールや設計方針に従って実装・レビューを行います。

例えば、以下のような内容を定義できます。

* コーディング規約
* ディレクトリ構成ルール
* テスト方針
* 使用技術・アーキテクチャ
* 禁止事項

```
# プロジェクト共通ルール

## コーディング規約
- 可読性を重視する
- Fat Controllerを避ける
- 共通処理はServiceクラスへ分離する

## ディレクトリ構成
- Backendは `app/`
- Frontendは `resources/js/`
- テストは`tests/`配下に配置する

## テスト方針
- 新規実装にはテストを追加する
- バグ修正時は再発防止テストを書く

## 使用技術・アーキテクチャ
- BackendはLaravel を利用する
- FrontendはReactを利用する
- APIはRESTfulに設計する

## 禁止事項
- ハードコーディングを避ける
- 秘密情報をコミットしない
- 不要なログを残さない
```
