## カスタムスキル——「一言で全自動」を実現する最強機能

カスタムスキルは、Claude Codeの中でも**最も過小評価されている機能**です。

スキルとは、あらかじめ定義した「命令テンプレート」のことです。たとえば、こんなことができます。

* **/note-post AIの最新トレンド** → テーマに沿った記事を自動生成してストック保存
* **/seo-check** → 指定したURLのSEOスコアを自動分析
* **/translate-ja** → 英語の技術記事を自然な日本語に翻訳
* **/manga AI副業** → テーマに合った4コマ漫画のプロンプトを生成

スキルの本質は**「毎回同じ説明をしなくていい」**ということです。「noteに投稿する記事を書いて。HTMLで。見出しと太字付きで。10000文字くらいで。画像プロンプトも一緒に……」と毎回説明するのは面倒ですよね。スキルに一度書いておけば、次からはコマンド一つで全部やってくれます。

### スキルの作り方

スキルは**ただのテキストファイル（.md）**です。特別なプログラミング知識は不要。日本語で「こういう時はこうして」と書くだけです。

保存場所は .claude/skills/スキル名/SKILL.md。Claude Codeが自動的に認識して、スラッシュコマンドとして使えるようになります。

> スキルは追加も削除も自由。不要になったらフォルダごと消すだけ。試行錯誤のハードルが極めて低いのも魅力です。

### スキルの活用例——実際に使われているパターン

スキルは「定型作業の自動化」に最適です。実際にどんな場面で使われているのか、いくつか例を挙げます。

**ブログ運営：**記事のテーマを入れるだけで、SEOを意識した構成・見出し・本文・画像プロンプトまで一括生成。記事ストックとして保存し、好きなタイミングで投稿。

**SNS運用：**ブログ記事のURLを渡すだけで、X（Twitter）用の宣伝ツイートを5パターン生成。ハッシュタグ付き。

**プログラミング：**「このエラーメッセージを解決して」と貼り付けるだけで、原因分析→修正コード生成→テスト実行まで自動。

**リサーチ：**競合サイトのURLを渡すだけで、記事構成・文字数・キーワード・SEOスコアを分析してレポート生成。

### スキルで失敗しないためのコツ

スキルを作る時によくある失敗は、**最初から完璧を目指しすぎること**です。「あらゆるケースに対応するスキルを作ろう」と考えると、いつまでたっても完成しません。

おすすめの進め方はこうです。

1. **まず最小限で作る**——「記事を生成してファイルに保存する」だけのスキルを作る
2. **使ってみる**——実際に何度か使って、不便な点を見つける
3. **少しずつ改善**——「画像プロンプトも一緒に生成して」「SEOタイトルも付けて」と機能を追加
4. **繰り返す**——使うたびに改善。1ヶ月後には完璧なスキルになっている

このアプローチなら、プログラミング経験がない人でもスキルを作れます。日本語で書くだけですから。

### スキルの共有と販売

作ったスキルは**他の人に共有・販売することも可能**です。スキルはただのテキストファイルなので、メールで送ったり、noteで販売したりできます。

たとえば「SEO記事自動生成スキル」「YouTube台本作成スキル」「競合分析スキル」など、需要のあるスキルを作れば、それ自体が商品になります。Claude Codeのスキルマーケットはまだ成熟していないので、今がチャンスです。

## CLAUDE.md——AIに「あなた専用のルール」を覚えさせる

CLAUDE.mdは、プロジェクトごとに設置する**「AIへの永続的な指示書」**です。

通常のチャットでは、セッションが切れるとAIは指示を忘れます。でもCLAUDE.mdに書いた内容は、**毎回のセッション開始時に自動で読み込まれる**ため、AIが「前にも言ったのに」という問題が起きません。

### CLAUDE.mdに書くべきこと

* **プロジェクトの概要**——「これはWordPressの自動投稿プラグインです」
* **コーディング規約**——「関数名はスネークケースで」「コメントは日本語で」
* **禁止事項**——「Chromeのプロファイルパスを絶対に変更しないこと」
* **よく使うコマンド**——「ビルドは npm run build で実行」
* **ファイル構成**——「src/にソース、dist/にビルド済みファイル」

CLAUDE.mdの最大のメリットは**「チーム共有」**です。チームメンバー全員が同じCLAUDE.mdを使えば、AIの挙動が統一されます。「あの人が聞いた時はうまくいくのに、自分が聞くとダメ」という問題がなくなります。

### CLAUDE.mdの具体的な書き方

CLAUDE.mdは難しく考える必要はありません。**AIに対する「お願い事リスト」**だと思ってください。具体的な記述例を紹介します。

## メモリ——AIがあなたを「覚える」仕組み

Claude Codeには**永続メモリ機能**があります。セッションをまたいで、あなたの好みや過去のやり取りを記憶します。

「前にも言ったのに」「何度も同じ説明をさせないで」——こんな不満を持ったことはありませんか？メモリ機能を使えば、この問題は解決します。

### メモリの種類

* **ユーザーメモリ**——あなたの役割、好み、スキルレベルを記憶。「この人はデータサイエンティストで、Pythonが得意」
* **フィードバックメモリ**——過去の修正指示を記憶。「この人はコードにコメントを書かないでほしい」
* **プロジェクトメモリ**——プロジェクトの進捗や決定事項を記憶。「マージフリーズは3月5日から」
* **リファレンスメモリ**——外部リソースの場所を記憶。「バグ管理はLinearのINGESTプロジェクト」

メモリは自動で蓄積されていきますが、「これを覚えて」と明示的に指示することもできます。逆に「これは忘れて」と言えば削除されます。

> メモリが蓄積されるほど、AIはあなた専用のアシスタントに進化していきます。使い始めて1ヶ月後には、説明なしで意図を汲み取ってくれるようになるでしょう。

### メモリを活用した実際の体験

メモリ機能の真価は、**使い続けることで初めてわかります**。最初の1週間はほとんど変化を感じないかもしれません。でも1ヶ月も使えば、こんな変化が起きます。

**1週間目：**「私はブログ運営をしています」「記事は日本語で書きます」——基本的な情報を覚えさせる段階。

**2週間目：**「この人はSEOを重視する」「見出しにキーワードを入れたがる」——あなたの癖や好みを学習し始める。

**1ヶ月目：**「記事を書いて」と言うだけで、あなた好みの構成・文体・長さで記事が出てくる。いちいち指示しなくても、過去のフィードバックから最適な形を推測してくれる。

**3ヶ月目：**もはや「自分の分身」。あなたの考え方、判断基準、好き嫌いまで理解した上で提案してくれる。新しいプロジェクトを始めても、過去の経験を踏まえたアドバイスが最初から出てくる。

> メモリは「AIとの信頼関係」を構築する仕組みです。人間の部下を育てるのと同じように、最初は手間がかかりますが、育てた分だけ返ってきます。

### メモリの管理と整理

メモリは放置すると古い情報が残り続けます。定期的な整理が必要です。

* **月に1回**——「メモリの一覧を見せて」と聞いて、古い情報を削除
* **プロジェクト完了時**——プロジェクト固有のメモリを整理
* **方針変更時**——「前はSEO重視だったけど、今はUX重視に変えた」と更新

メモリが整理されていると、AIの回答精度が格段に上がります。古い情報に基づいた的外れな提案がなくなるからです。

## ターミナル版——VS Codeなしでも使える

Claude CodeはVS Code版だけでなく、**ターミナル（コマンドライン）版**も用意されています。

ターミナル版のメリットは**軽さ**です。VS Codeを起動しなくていいので、メモリ消費が少なく、起動も速い。「ちょっと聞きたいだけ」という時に最適です。

### ターミナル版とVS Code版の使い分け

* **大きなプロジェクト**→ VS Code版（ファイル構造を見ながら作業）
* **ちょっとした質問**→ ターミナル版（すぐ起動してすぐ聞ける）
* **サーバー作業**→ ターミナル版（SSH接続先で使える）
* **記事作成**→ どちらでもOK（好みで選ぶ）

両方を併用するのが最も効率的です。同じアカウントで両方使えるので、追加料金はかかりません。

## セキュリティと安全性——AIに何を任せて何を任せないか

Claude Codeは強力なツールですが、**何でもかんでも任せていいわけではありません**。特にファイル操作やコマンド実行は、設定を誤ると取り返しのつかない事態になることがあります。

### 安全に使うためのルール

* **本番環境での自動実行は避ける**——テスト環境で確認してから本番に反映
* **バックアップを取る**——重要なファイルを編集する前に必ずバックアップ
* **パーミッション設定を活用**——不要な操作は許可しない
* **APIキーや認証情報を直接貼り付けない**——環境変数で管理する
* **CLAUDE.mdに禁止事項を明記**——過去のトラブルを教訓に

特に**データベースの操作**と**ブラウザの設定変更**は要注意です。「この操作は安全か？」と少しでも疑問に思ったら、AIに確認してから実行しましょう。

### プライバシーについて

Claude Codeはローカルで動作するため、**コードや作業内容がクラウドに保存されることは基本的にありません**。AIとの対話内容はAnthropicのサーバーを経由しますが、コード自体はあなたのPC上で処理されます。

企業で利用する場合は、Anthropicのプライバシーポリシーを確認した上で、機密情報の取り扱いルールを決めておくことをおすすめします。

## よくある質問

### Q. Claude Codeは無料で使えますか？

無料プランでは使えません。**Proプラン（月額$20）以上**が必要です。ただし、月額約3,000円で記事の自動生成、ファイル操作、ブラウザ自動操作がすべて使い放題なので、コスパは非常に高いです。

### Q. プログラミングができなくても使えますか？

使えます。Claude Codeの操作はすべて**日本語のチャット**で行います。「ファイルを作って」「このエラーを直して」と日本語で指示するだけ。プログラミングの知識があればより高度なことができますが、必須ではありません。

### Q. スキルやCLAUDE.mdの設定が難しそうです

最初は「敬語で対応して」「日本語で書いて」程度の簡単な設定から始めてください。Claude Code自体に「CLAUDE.mdを作って」とお願いすることもできます。AIが雛形を作ってくれるので、それをベースに調整していけばOKです。

### Q. 他のAIツール（ChatGPT, Gemini等）と何が違うのですか？

最大の違いは**「実際に作業をしてくれるかどうか」**です。ChatGPTやGeminiは「会話」はできますが、あなたのPC上でファイルを作成したり、プログラムを実行したり、ブラウザを操作したりはできません。Claude Codeは会話だけでなく、実際に手を動かしてくれる「作業するAI」です。

### Q. 月額$20と$100のプランはどう違いますか？

Proプラン（$20）はSonnetモデルが中心。Maxプラン（$100）はOpusモデル（最高性能）が使えます。日常的な記事作成やファイル操作ならProで十分。大規模なプロジェクト管理や高度な分析が必要ならMaxがおすすめです。

## 実践Tips——今日から使える設定テクニック

### 1. 最初にやるべき3つの設定

**CLAUDE.mdを作る：**プロジェクトのルートに置くだけ。最初は「敬語で対応する」「日本語でコメントを書く」程度でOK。使いながら追加していけばいい。

**よく使うスキルを1つ作る：**一番繰り返している作業をスキル化する。完璧を目指さず、まず動くものを作って改善していく。

**メモリに自己紹介を保存：**「私は○○です。△△の仕事をしています」と伝えて「覚えて」と言う。次回以降、あなたに最適化された回答が返ってくるようになる。

### 2. 中級者向け——MCPで外部連携

MCPサーバーの設定は最初は難しく感じるかもしれません。でも、公式ドキュメントにテンプレートが用意されているので、コピー＆ペーストで始められます。まずはファイルシステムMCPから始めて、慣れてきたらSlackやGitHub連携に挑戦してみてください。

### 3. 上級者向け——フックで完全自動化

フックを使いこなせば、「AIに指示する」という行為すら自動化できます。たとえば、GitHubにPRが作成されたら自動でコードレビューを開始する、といったワークフローが構築できます。

## 実際のワークフロー例——1日の作業がこう変わる

ここまで機能を個別に解説してきましたが、実際にはこれらを**組み合わせて使う**ことで真価を発揮します。ある日の作業フローを例に見てみましょう。

### ブロガーの1日（Claude Code導入前）

1. ネタ探し（30分）——Google検索、SNS、ニュースサイトを巡回
2. 構成作成（20分）——見出しを考えて並べる
3. 執筆（2時間）——本文を書く
4. 画像探し（30分）——フリー素材サイトで画像を探す
5. SEO調整（20分）——タイトル、メタディスクリプション、キーワードを調整
6. 投稿作業（10分）——WordPressやnoteにコピペ、書式設定

**合計：約4時間**

### ブロガーの1日（Claude Code導入後）

1. /note-post AIの最新トレンド と入力（10秒）
2. 生成された記事を確認・微調整（15分）
3. noteにコピペ（1分）
4. 画像は[**どこでもAIくん**](https://note.com/tothinks/n/n6babb1aed8ac)で生成（5分）

**合計：約20分**

4時間が20分になる。**12倍の効率化**です。しかも、1日に1本が限界だった記事が、3本でも5本でも作れるようになります。

### プログラマーの場合

プログラマーの場合はさらに劇的です。

* 「このバグを直して」→ AIがコードを読み、原因を特定し、修正コードを書き、テストまで実行。**数時間の作業が5分で完了。**
* 「この機能を追加して」→ 仕様を伝えるだけで、AIが必要なファイルを全部作成。人間はレビューするだけ。
* 「このコードをリファクタリングして」→ AIが最適な設計パターンを提案し、一括で書き換え。

### ビジネスパーソンの場合

プログラミングをしないビジネスパーソンでも、Claude Codeは強力な武器になります。

* **資料作成**——「先月の売上データを分析して、プレゼン用のサマリーを作って」
* **メール対応**——「このクレームメールに丁寧に返信して。お詫びと今後の対策を含めて」
* **競合分析**——「このURLの競合サイトを分析して、うちとの差別化ポイントをまとめて」
* **翻訳**——「この英語の契約書を日本語に翻訳して。法務的に重要なポイントをハイライトして」

## 他のAIツールとの併用——Claude Codeだけじゃもったいない

Claude Codeは万能ですが、**他のAIツールと組み合わせるとさらに強力**になります。

* **Claude Code × Gemini**——Claude Codeで記事構成を作り、Geminiで画像生成。テキストはClaude、ビジュアルはGeminiという使い分け。
