# Claude APIで初回トークンの待ち時間を短縮する — プロンプトキャッシュの事前ウォームアップ

## 概要

長いプロンプトを扱うClaude APIにおいて、**初回トークンが返ってくるまでの時間（Time-to-First-Token）を短縮する**便利なテクニックが公式から紹介されました。

## テクニック：プロンプトキャッシュの事前ウォームアップ

### 手順

1. **ユーザーのプロンプトより先に、システムプロンプトだけを送信する**
2. Claudeはそのシステムプロンプトをキャッシュに書き込むが、レスポンス（出力）は生成しない
3. 実際のユーザーリクエストが届いたとき、すでに**ウォームアップされたキャッシュにヒット**する

### なぜ効果があるのか

Claudeのプロンプトキャッシュ機能は、同一のプロンプト（特にシステムプロンプト）を繰り返し処理するコストと時間を削減します。事前にシステムプロンプトをキャッシュへ書き込んでおくことで、ユーザーからの本番リクエストが来た際には、システムプロンプトの処理をスキップしてすぐに応答生成へ移行できます。

### 活用シーン

* **長大なシステムプロンプト**を使うチャットボットやエージェント
* ユーザーが操作を開始する前に、バックグラウンドでキャッシュを温めておきたいアプリケーション
* レイテンシが重要なリアルタイム系のAPIインテグレーション

## まとめ

| ステップ | 内容 |
| --- | --- |
| 事前リクエスト | システムプロンプトのみ送信（出力なし） |
| キャッシュ状態 | ウォームアップ済み |
| 本番リクエスト | キャッシュヒットで高速レスポンス |

シンプルながら効果的なこのテクニックは、ユーザー体験の向上に直結します。特に長いシステムプロンプトを使用している場合は、ぜひ取り入れてみてください。
