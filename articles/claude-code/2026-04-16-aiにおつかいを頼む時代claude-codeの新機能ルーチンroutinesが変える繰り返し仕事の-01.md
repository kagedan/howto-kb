毎朝、同じことをしていませんか？

昨日の作業内容をまとめてチームに共有する。  
提出されたコードに問題がないか確認する。  
週次レポートを作ってSlackに投稿する。どれも大切な仕事です。  
でも、毎回自分の手でやるのは正直なところ、消耗します。

2026年4月14日、Anthropicはそんな"繰り返し仕事"をAIに任せるための新機能「ルーチン（Routines）」を **Claude Code** に追加しました。  
まだ試験提供（リサーチプレビュー）の段階ですが、働き方を変えるポテンシャルを持った機能です。

> **Claude Codeとは？**  
> Anthropicが提供する開発者向けのコーディングツールです。通常のClaudeチャット（claude.ai）とは別のツールで、コードを書いたり、GitHubなどの開発ツールと連携したりすることに特化しています。ルーチン機能はこのClaude Codeの機能であり、利用にはPro以上の有料プランが必要です。

---

## 「ルーチン」とは何か、一言で言うと

**AIへの指示を一度書いておくだけで、条件が揃ったら自動で動いてくれる仕組みです。**

料理に例えるなら、毎朝「今日の献立を考えて、食材を冷蔵庫から出しておいて」と頼む代わりに、「毎朝7時になったら自動でやっておいて」と一度設定するイメージです。あとは何もしなくてもAIが動いてくれます。

---

## 実際にどんなことができるの？

具体的なシーンで見てみましょう。

**シーン1：毎朝の朝会準備を自動化する**  
チームの朝会前に「昨日マージされた変更内容をまとめて、Slackの#dailyチャンネルに投稿して」と設定しておくだけ。毎朝9時になると、AIが自動でコードの変更を読み取り、わかりやすくまとめてSlackに投稿してくれます。

**シーン2：コードのチェックをAIに任せる**  
誰かが新しいコードを提出（プルリクエスト）すると、AIが自動で内容を確認し「ここにテストが足りていません」「この変数名はルールに合っていません」といったコメントをつけてくれます。レビュアーが確認する前に、基本的なチェックが済んでいる状態になります。

**シーン3：週次レポートを自動生成する**  
毎週金曜の夕方に「今週マージされたPRと解決したIssueをまとめて、進捗レポートを作成してSlackに送って」と設定しておけば、週末のレポート作成から解放されます。

---

## これまでの「自動化」と何が違うの？

「自動化なら今までもできたんじゃないの？」と思った方、鋭いです。

確かにGitHub ActionsやCronジョブといったツールを使えば、以前から自動化は可能でした。ただしそれらは「決まったスクリプトを決まった時間に実行する」だけで、AIが状況を読んで判断することはできませんでした。

ルーチンは**AIが内容を読み、考えたうえで動く**点が大きな違いです。「このPRに問題があるか確認して」と指示するだけで、AIがコードの意味を理解した上でコメントを書いてくれます。

もうひとつ大きな違いが、**PCをつけておく必要がない**こと。ルーチンはAnthropicのクラウド上で動くため、自分のパソコンを起動していなくても、深夜でも休日でも設定通りに動き続けます。

---

## 指示文はどう書けばいい？

ルーチンの設定で最も重要なのが「プロンプト（AIへの指示文）」です。普通の会話と違い、ルーチンは**誰も見ていない状態で自動実行される**ため、指示文は自己完結している必要があります。

**良い指示文の3つのポイント**

1. **何をすべきか明確に書く** ─ 曖昧な表現は避け、具体的な作業内容を記述する
2. **対象の範囲を指定する** ─ 「どのリポジトリの」「過去24時間の」など、範囲を絞る
3. **出力先を指定する** ─ 「Slackの#channelに投稿」「PRにコメント」など、結果をどこに出すかを書く

**実際の指示文の例**

スケジュールトリガーの場合：

```
毎晩2時に、Linearから優先度の高いバグを1件取得し、
修正を試みてドラフトPRを作成してください。
```

GitHubトリガー（PRが作成されたとき）の場合：

```
PRの差分を確認し、/auth-providerモジュールへの変更が
含まれている場合は、変更内容を要約して#auth-changesチャンネルに投稿してください。
```

このように、「いつ・何を・どこに」を明確に書くことで、AIが迷わず動けるようになります。

## 設定の始め方

ルーチンは以下の3つの方法で作成できます。

**Webブラウザから**  
`claude.ai/code/routines` にアクセスし、「新しいルーチン」を作成。画面の指示に従ってプロンプト・リポジトリ・コネクタ（SlackやGitHubなど）を設定します。

**CLIから**  
Claude Codeのターミナルで `/schedule` と入力すると、対話形式でルーチンを作成・管理できます。

![](https://static.zenn.studio/user-upload/98bbdf2b2708-20260416.png)

**デスクトップアプリから**  
Claude Codeデスクトップアプリの「ルーチン」メニューから「新しいルーチン（リモート）」を選択します。

## AIとの付き合い方が変わる

これまでのAIは「聞いたら答えてくれる存在」でした。  
質問すれば回答し、依頼すれば作業してくれる。でも基本的には、自分が声をかけないと動かない。

ルーチンはその関係を変えます。一度お願いしておけば、あとは自分で動いてくれる。  
まるで信頼できるアシスタントに仕事を任せるような感覚です。

繰り返しの作業をAIに任せて、自分はもっと創造的な仕事に集中する──ルーチンはそんな働き方への第一歩かもしれません。

まずは「自分の仕事の中で、毎回同じことをしているのはどこだろう？」と考えてみるところから始めてみてください。
