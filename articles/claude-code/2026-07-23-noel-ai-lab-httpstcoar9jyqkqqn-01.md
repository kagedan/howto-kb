私は普段から、文章作成やリサーチ、企画、台本作成、ツール作成、動画生成、アプリ制作などにAIをかなり使っています。

だからこそ最近感じるのが、「AIを使っている」と「AIに仕事を任せられている」は、似ているようで全然違うということなんです。

毎回ゼロから長い指示を書いて、過去の資料を探して、同じルールを説明して、出てきた文章をまた別のAIに貼り直す。

これでも確かにAIは使えています。でも、私たちが毎回つきっきりになっているなら、まだ「便利なチャット」の範囲を出ていませんよね。

そこで注目したいのが、Claude Coworkで使えるプラグインです。

プラグインを入れると、Claudeに仕事の進め方や判断基準を持たせたり、よく使う作業を決まった流れで呼び出したり、必要に応じて外部ツールとつないだりできます。

要するに、毎回その場でお願いするだけだったAIを、役割を理解して動く担当者に近づけられるんです。

今回紹介するのは、Anthropicの公式GitHubリポジトリ「knowledge-work-plugins」で公開されているものの中から、AIを学んでいる人、すでにAIを使っている人、自分でビジネスをしている非エンジニアに関係しやすい16個です。

これはコードを書く人だけの機能集ではありません。

マーケティング、商品企画、営業、顧客対応、データ分析、業務整理など、知識労働の進め方をClaudeへ追加するものです。

ここは最初にはっきり分けておきます。

Claude Codeは、公式に「agentic coding tool」と説明されている開発向けの製品です。

一方、Claude Coworkは、ローカルファイルやSlack、Google Driveなどを使い、資料作成や調査、定型業務を成果物まで進める知識労働向けの製品です。

今回のプラグイン集も、公式READMEに「Claude Cowork向けに作られ、Claude Codeにも対応」と明記されています。

つまり、非エンジニアが発信、営業、資料、顧客対応へ使うなら、基本はCoworkです。

Claude Codeでも同じプラグインを使えますが、そちらはアプリ制作やコードを含む仕事をする人向けの選択肢です。

また、この記事は私が16個すべてを長期間使った体験レビューではありません。

この中には、私が実際に入れて使っているプラグインもあります。ただし16個全体としては、2026年7月23日時点のAnthropic公式リポジトリ（ [https://github.com/anthropics/knowledge-work-plugins](https://github.com/anthropics/knowledge-work-plugins) ）、Claude Cowork公式ガイド（ [https://claude.com/blog/the-claude-cowork-product-guide](https://claude.com/blog/the-claude-cowork-product-guide) ）、プラグイン導入ドキュメント（ [https://claude.com/docs/cowork/guide/plugins](https://claude.com/docs/cowork/guide/plugins) ）、Claude Code公式概要（ [https://code.claude.com/docs/en/overview](https://code.claude.com/docs/en/overview) ）を照合し、非エンジニアの実務に置き換えて整理した調査記事です。

# 【最新情報】米国時間7月22日（日本時間7月23日）、Claude Code向け「Claude Security」プラグインがβ公開

米国時間2026年7月22日、日本時間では7月23日に、AnthropicからClaude Code向けの「Claude Security」プラグインがβ公開されました。

AIでアプリやサービスを作っている人には、かなり大きな更新です。

公式ドキュメント（ [https://code.claude.com/docs/en/claude-security](https://code.claude.com/docs/en/claude-security) ）によると、Claude Codeの中から、リポジトリ全体またはブランチ、プルリクエスト、単一コミットなどの変更範囲を対象に、脆弱性を調べられます。
