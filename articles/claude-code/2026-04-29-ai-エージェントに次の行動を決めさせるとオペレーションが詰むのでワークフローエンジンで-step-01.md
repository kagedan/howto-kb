はじめに
AI エージェントを本番に出すとき、最初に直面するのが「エージェントが次に何をするかを自分で判断する」問題です。
たとえば「Notion に 1 行要望が来たら → PRD 起こして → 実装タスクに分解して → ブランチ切って → Claude Code 実行して → PR 作って → E2E テスト走らせて → デプロイ通知」というフローを 1 つの Claude Agent SDK セッションに任せると、

「今ブランチ切るべきか PR 作るべきか」を毎回 LLM に判定させることになる
同じ入力でも次の step が時々違う(non-deterministic)
