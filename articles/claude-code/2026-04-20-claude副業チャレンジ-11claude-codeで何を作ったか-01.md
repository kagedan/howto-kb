Claude副業チャレンジ [#11](https://note.com/hashtag/11) 「Claude Codeで、何を作ったか。」  
  
📌 前回のあらすじ  
  
Claude Codeで作業が激変した。  
自分で言うのもなんだけど、有料級だと思う——と書いた。  
  
じゃあ、具体的に何を作ったのか。  
  
今回はそれを話します。  
  
―――――――――――――  
  
🛠 作ったもの① 仕入れ判断ツール  
  
ヤフオクで気になった商品を見つけたとき、  
以前はこういう手順を踏んでた。  
  
作家名をGoogle検索する  
　↓  
Catawikiの過去落札データを調べる  
　↓  
手数料・送料を引いて利益計算する  
　↓  
「買うか、やめるか」を判断する  
  
この一連の作業、慣れても15〜20分かかってた。  
  
今は、ツールに作家名を入れるだけ。  
過去の落札相場、仕入れ上限価格、予想利益が数秒で出る。  
  
判断が、速くなった。  
速くなったので、見られる商品数が増えた。  
商品数が増えたので、いい仕入れに出会う確率が上がった。  
  
これだけで、月の利益が変わり始めた。  
  
―――――――――――――  
  
🛠 作ったもの② 出品サポートツール  
  
Catawikiへの出品、実は英語で書かないといけない。  
  
作家の経歴、作品の特徴、サイズ、状態——  
これを毎回英文で書くのは、正直しんどかった。  
  
今は、商品の基本情報を入力するだけで  
英文の説明文がほぼ自動で生成される。  
  
あとは微調整するだけ。  
出品にかかる時間が、半分以下になった。  
  
―――――――――――――  
  
🛠 作ったもの③ 発送・顧客対応サポート  
  
海外のバイヤーへのメッセージも、毎回英語。  
  
「荷物が届かない」「追跡番号を教えて」  
「これは本物ですか」——  
  
こういう問い合わせへの返信も、  
今はツールのサポートでほぼ一発で出せる。  
  
対応が速くなった。  
バイヤーからの評価も上がってきた。  
  
―――――――――――――  
  
全部、Claude Codeで作った。  
  
プログラミングの知識は、ほぼゼロだった。  
でも「こういうものが欲しい」と伝えれば、  
Claudeが一緒に作ってくれる。  
  
最初の1週間は試行錯誤の連続だった。  
でも今は、副業の「インフラ」になってる。  
  
正直、紹介したのはまだ一部です。  
  
在庫管理、売上記録、ヤフオク監視——  
Claude Codeで作ったものは、まだまだある。  
  
全部話すと、それだけで本になる。  
  
―――――――――――――  
  
どうやって作ったか、は——  
  
正直、ここから先は長くなる。  
  
環境の作り方、Claudeへの指示の出し方、  
失敗したこと、うまくいったこと。  
  
全部書くと、それだけで一冊になる。  
  
次回、有料記事として出そうと思ってます。  
  
無料部分で概要はお伝えします。  
でも「実際に自分でも作りたい」という方には、  
有料部分で具体的な手順をまとめます。  
  
買うかどうかは、読んでから決めてください。  
押し売りはしません。
