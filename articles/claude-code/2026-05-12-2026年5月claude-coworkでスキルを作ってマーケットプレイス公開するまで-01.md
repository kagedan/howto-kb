## この記事のモチベーション

Claude Coworkはすごい。

かなり革新的なプロダクトだと個人的には思っています。  
私はエンジニアとして`Claude Code`をメインに利用していますが、非エンジニアの業務要件に対して、Claude Codeは複雑すぎるし、下手にアプリケーションもどきを作るとメンテナンスコストやセキュリティのリスクが伴います。

その点、`Cowork`はスキルやコマンド等のマークダウンファイルのみを横展開するだけで、誰でも同じ業務が同じように自動化できます。

世間では割と盛り上がっているように感じているのですが、Coworkのスキル配布やマーケットプレイスでの展開方法について言及した記事が少ないなと感じました。(AI驚き屋さんたちが最新情報を日々Xで発信してくださっていますが、具体的な記事はなかなかありませんでした)

もちろん「公式ドキュメントを読めばできる」のですが、実体験としてどなたかの参考になれば良いなと思って執筆します。

## 事前情報

プラグインとマーケットプレイスという単語がでてくるので事前に補足しておきます。  
![](https://static.zenn.studio/user-upload/69917b8dfd74-20260506.png)  
配布したい機能単位をプラグイン、それを一元化してカタログのようにしたようなものがマーケットプレイスです。  
プラグイン化してマーケットプレイスで配布すればClaude CodeでもCoworkでも利用できます。

## 1. まずはスキルを作る

Coworkに`skill-creator`というスキルがあるので任意のスキルを作っておきましょう。  
サンプルとして試したい方は以下を使ってください。

ディレクトリ構成

```
hello-cowork/
└── SKILL.md
```

SKILL.md

```
---
name: hello-cowork
description: Cowork の世界への挨拶を返す入門用スキル。
  ユーザーから「hello」「こんにちは」などのカジュアルな挨拶が来た場合、
  または `/hello-cowork` がスラッシュコマンドとして呼び出された場合に使ってください。
---

# Hello Cowork

以下のフォーマットで、必ず応答してください。

> やあ、Cowork の世界へようこそ！
> このスキルは `hello-cowork` プラグインから提供されています。

挨拶以外の追加情報は付けないでください。
```

## 2. スキルをプラグインに「包む」

ディレクトリ構成

```
hello-plugin/                         # ← プラグイン名
├── .claude-plugin/
│   └── plugin.json                   # プラグインのマニフェスト（必須）
└── skills/
    └── hello-cowork/                 # ← 先ほど作成したスキルそのまま
        └── SKILL.md
```

[https://code.claude.com/docs/ja/plugins-reference#プラグインディレクトリ構造](https://code.claude.com/docs/ja/plugins-reference#%E3%83%97%E3%83%A9%E3%82%B0%E3%82%A4%E3%83%B3%E3%83%87%E3%82%A3%E3%83%AC%E3%82%AF%E3%83%88%E3%83%AA%E6%A7%8B%E9%80%A0)

プラグインの箱が`hello-plugin`、その中に入っているスキルが`hello-cowork`、という入れ子構造です。  
`plugin.json`がプラグインのアイデンティティを決める部分です。

plugin.json

```
{
  "name": "hello-plugin",
  "description": "Cowork のスキル配布フローを学ぶための、最小構成のサンプルプラグイン",
  "version": "1.0.0"
}
```

[https://code.claude.com/docs/ja/plugins-reference#プラグインマニフェストスキーマ](https://code.claude.com/docs/ja/plugins-reference#%E3%83%97%E3%83%A9%E3%82%B0%E3%82%A4%E3%83%B3%E3%83%9E%E3%83%8B%E3%83%95%E3%82%A7%E3%82%B9%E3%83%88%E3%82%B9%E3%82%AD%E3%83%BC%E3%83%9E)

## 3. 配布方法を選ぶ

マーケットプレイスとして配布するのも一つですが、プラグインの配布方法はいくつか存在しています。現在の仕組みでは大きく次の4つがあります。

| 方式 | 向いている状況 |
| --- | --- |
| **GitHub マーケットプレイス（公開）** | OSS として広く公開したい、外部の人にも使ってほしい |
| **GitHub マーケットプレイス（非公開）** | チームプランとしてではなく、小規模で共有したい |
| **GitHub 同期マーケットプレイス（組織管理）** | 自社のチーム/エンタープライズプランで、メンバーに配布したい |
| **手動 ZIP アップロード** | 一度だけ、特定の人に渡したい。ad-hoc 配布 |

※試したことはありませんが、GitHub以外のGitサービスにも対応しているそうです。

チーム共有を前提として考えると非公開のマーケットプレイスかエンタープライズ用の同期マーケットプレイスが選択肢になってきます。  
本記事は簡易的に個人のプライベートリポジトリにマーケットプレイスをホスティングしてみます。  
[https://code.claude.com/docs/ja/plugin-marketplaces#マーケットプレイスのホストと配布](https://code.claude.com/docs/ja/plugin-marketplaces#%E3%83%9E%E3%83%BC%E3%82%B1%E3%83%83%E3%83%88%E3%83%97%E3%83%AC%E3%82%A4%E3%82%B9%E3%81%AE%E3%83%9B%E3%82%B9%E3%83%88%E3%81%A8%E9%85%8D%E5%B8%83)

## 4. GitHubマーケットプレイスを作る

GitHub マーケットプレイスは、要するに「`marketplace.json`を持った GitHub リポジトリ」です。リポジトリ全体は次のような構造になります。

リポジトリ構成

```
hello-marketplace/                    # ← マーケットプレイス（GitHub リポジトリ）
├── .claude-plugin/
│   └── marketplace.json              # マーケットプレイスのマニフェスト
└── plugins/
    └── hello-plugin/                 # ← プラグイン本体
        ├── .claude-plugin/plugin.json
        └── skills/
            └── hello-cowork/         # ← スキル
                └── SKILL.md
```

三層構造（マーケットプレイス → プラグイン → スキル）が、ディレクトリにそのまま反映されている形です。  
続いて`marketplace.json` の中身です。

marketplace.json

```
{
  "name": "hello-marketplace",
  "owner": {
    "name": "Your Name"
  },
  "plugins": [
    {
      "name": "hello-plugin",
      "source": "./plugins/hello-plugin",
      "description": "Cowork のスキル配布フローを学ぶための最小サンプル"
    }
  ]
}
```

[https://code.claude.com/docs/ja/plugin-marketplaces#マーケットプレイスファイルの作成](https://code.claude.com/docs/ja/plugin-marketplaces#%E3%83%9E%E3%83%BC%E3%82%B1%E3%83%83%E3%83%88%E3%83%97%E3%83%AC%E3%82%A4%E3%82%B9%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%81%AE%E4%BD%9C%E6%88%90)
