## これはかなり大きい変化だと思う

正直、このポストはただの機能追加ではないです。 が「質問に答えるAI」から、**実際にPC上で作業を進めるAI** に一段踏み込んだ、という話だからです。Cowork はもともと、 Code のエージェント的な動きをコーディング以外の仕事にも広げるための機能として 2026年1月に研究プレビュー公開されました。最初は Max 向け、数日後に Pro に拡大され、その時点では macOS 限定でした。 今回のポストで言っているのは、その Cowork や Claude Code の文脈で、 が**アプリを開く、ブラウザを操作する、表計算を埋める**みたいな「人が机の前でやっていたこと」を肩代わりし始めた、ということです。単に文章を出すだけじゃなく、画面を見て、クリックして、入力して、進める。ここが本質です。 サポートセンター+1

## どこが新しいのか。実は「2段階」で進化している

面白いのは、 がいきなり全部を雑に触り始めたわけじゃないことです。Cowork 自体は、あなたのPC上で動きつつ、作業の多くを隔離された実行環境で処理します。そのうえで、必要なときだけローカルファイルやコネクタ、さらに実アプリへアクセスしていく設計になっています。つまり、**まずは比較的コントロールしやすい箱の中で動き、必要に応じて現実のデスクトップに手を伸ばす**感じです。 サポートセンター+2Claude  
しかも現在のヘルプでは、Cowork 自体は macOS に加えて Windows x64 でも使える状態になっています。一方で、**コンピューター操作そのものは現時点では Pro / Max 向け研究プレビュー**で、デスクトップアプリを開いたまま、PCをスリープさせずに使う必要があります。ポストの「macOS only」は発表時点の温度感としては正しかったけど、2026年3月25日時点のドキュメントでは Cowork の対応範囲は少し進んでいます。 サポートセンター+2Claude サポートセンター+2

> ここで面白いのは、 が「安全な作業箱の中で動くAI」と「あなたの実デスクトップを触るAI」の両方を持ち始めたこと。  
> これ、単なるチャットUIの延長じゃなくて、仕事そのものの置き換えに近いです。

さらに 2026年3月17日には、スマホから Cowork に継続スレッドで仕事を投げられる機能も出ました。いわゆる Dispatch 的な使い方で、**外出先から指示して、実際の作業は自宅や職場のデスクトップで進める**、という流れが見えてきています。これはかなり未来感があります。 サポートセンター+1

## Claudeの類似機能と、どこが違うのか

ここは整理するとかなり見えやすいです。まず Claude in Chrome は、 がブラウザの中でページを読んだり、クリックしたり、ナビゲートしたりする機能です。かなり便利ですが、基本は**Web作業寄り**です。Cowork の computer use は、ブラウザだけでなくデスクトップ上のアプリまで対象に広がるので、守備範囲が一段広い。 次に Claude Code の Remote Control。これはローカルで走っている Claude Code のセッションを、スマホや別ブラウザから継続操作できる機能です。便利だけど本質は「既に動いている開発セッションへの遠隔窓口」です。今回の computer use は、**セッションを遠隔で見るだけでなく、アプリ操作まで任せる**ところが違います。 API Docs

そして Antigravity。ここが一番比較されやすいと思う。Antigravity は 2025年11月に出た Google のエージェント開発プラットフォームで、Gemini 3 を土台に、Editor View と Manager Surface を使い分けながら、複数エージェントを非同期で回せる設計です。エージェントは editor・terminal・browser をまたいで計画、実行、検証し、進捗を artifact で返してきます。しかも public preview は macOS / Windows / Linux で展開されています。 blog.google+3

ただ、個人的には \*\*Antigravity は「開発の司令塔」\*\*で、今回の Claude は \*\*「デスクワーク全般の代行者」\*\*に近いと思っています。Antigravity はかなり開発中心です。コードを書く、テストする、ブラウザで検証する、複数ワークスペースを束ねる。そこが強い。一方で Claude は、開発にも使えるけど、表計算、資料整理、コネクタ経由の情報収集、ローカルファイル処理まで含めて、「日常業務の手足」になろうとしている感じが強いです。 サポートセンター+3Google Developers Blog+3blog.google+3

> Antigravityは開発チームの管制塔。 はあなたの机に座る作業代行。  
> 似ているようで、狙っている主戦場が少し違うんですよね。

ちなみに他社で近いところを挙げるなら、ChatGPT agent はブラウザ操作、ファイル処理、端末実行、スケジュール実行まで含むかなり近い思想です。ただしこちらは「オンライン作業を仮想ブラウザ上で進める」色が強く、ChatGPT for macOS の Work with Apps は現状、主に IDE やターミナル、ノート系アプリとの連携が中心です。 の今回の打ち出しは、**ローカルPCを仕事場として使う感じ**がかなり前面に出ています。 ---

## AI駆動開発の視点で見ると、かなり景色が変わる

AI駆動開発って、これまで「コード生成」「レビュー補助」「テスト作成」あたりが主戦場でした。でも computer use が本格化すると、開発の前後にある細かい仕事まで AI に寄せやすくなります。たとえば管理画面で確認、Excel 更新、ブラウザ検証、資料化、報告文の下書き、軽いリサーチ。このへん、地味に時間を食う。そこを AI が持ち始めると、**実装だけ速くてもダメで、仕事全体を束ねるエージェント体験が強いツールが勝つ**流れになっていくと思います。 Claude サポートセンター+3Claude サポートセンター+3

## だから今後の比較軸は、「どのモデルが賢いか」だけじゃなくて、「どこまで触れるか」「どこまで任せられるか」「途中でどれだけ人間が介入しやすいか」に移っていくはずです。 は MCP やローカル環境との接続が強みだし、Antigravity は複数エージェントの非同期管理が強い。ChatGPT agent はオンライン作業の横断力が強い。**モデルの性能差より、作業面のOSっぽさが重要になる**、そんなフェーズに入ってきた感じがします。 Claude サポートセンター+3Claude

## 個人的には「便利」より先に、権限設計が勝負だと思う

ここは夢がある一方で、怖さもあります。 側の安全ガイドでも、Cowork や computer use には prompt injection（外部コンテンツに紛れた悪意ある指示）や、ローカルファイル・ブラウザ・接続先へのアクセスに伴うリスクがあるとかなり明確に書かれています。便利だから全部つなぐ、は普通に危ないです。 サポートセンター+2Claude サポートセンター+2

個人的には、これから大事なのは「何をAIにやらせるか」より、**どの権限で、どの境界までやらせるか**だと思っています。開発でも業務でも、最強のAIを探すより先に、権限、監視、レビュー、失敗時の戻し方を設計できる人が強くなる。ここ、地味だけど本当に重要です。 サポートセンター+1

それでも、この流れ自体はかなり本物です。AIはもう「相談相手」ではなく、「画面の向こうで仕事を進める同僚」になり始めている。今回のポストは、その変化が一気に可視化された瞬間だったんじゃないかな、と思います。 サポートセンター+2Claude サポートセンター+2
