## 今日やったこと

Claude Codeにカーチェイスゲームを作ってもらいました。

指示はこれだけです。

```
ブラウザで動くカーチェイスゲームをHTMLで作って。車を操作してパトカーから逃げる。矢印キーで操作。スコア・タイム表示。シンプルに作って。
```

その後、難易度・天気イベント・ミサイル・スマホ対応を追加しました。

## できたもの

![](https://assets.st-note.com/img/1778027435-tuzwHIvQ9JecgCsaMOo864By.png?width=1200)

![](https://assets.st-note.com/img/1778027446-kgacevyROZQNsI8BJGSzd5xi.png?width=1200)

**基本ルール：**

* 矢印キー（またはスマホタッチ）で車を操作
* パトカーから逃げ続けてスコアを稼ぐ
* ライフが0になるとゲームオーバー

**難易度3段階：**

* Easy：ライフ5・弾10発・速度×0.75
* Normal：ライフ3・弾6発・速度×1.0
* Hard：ライフ2・弾3発・速度×1.4

**特殊要素：**

* 天気イベント：雨でスリップ・霧で視界制限
* 道路イベント：一般市民の車・工事エリア・コーン障害物
* ミサイル：スペースキーで発射・パトカー撃破で+100pt
* スマホタッチ操作：左/右/上タップ・中央でミサイル発射

## 今回気づいたこと

「シンプルに作って」という指示から始めて、追加指示を重ねていくと今の形になりました。

Deck RushもRPGも同じパターンでした。最初から全部決めるより、動かしながら「これも欲しい」と追加していく方が自分のイメージに近いものができる。

ただ今回はGitの問題でファイルの更新確認に時間がかかりました。「ファイルが更新されているか確認する」という習慣をつけた方がいいと学びました。

このシリーズはプログラミング未経験の高校生がClaude Codeを毎日使い続けた記録です。
