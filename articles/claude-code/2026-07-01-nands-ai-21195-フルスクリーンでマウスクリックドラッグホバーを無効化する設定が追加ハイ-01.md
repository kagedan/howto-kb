2.1.195: フルスクリーンでマウスクリック/ドラッグ/ホバーを無効化する設定が追加。ハイフン付きMCPツールのフックが部分一致していた不具合を完全一致に修正。音声入力や背景エージェントの安定性も改善。#ClaudeCode https://t.co/YLVSZaKHrf

使い方① — フルスクリーンでマウス入力だけ止めたいとき: 環境変数で無効化できる。ターミナルで一時的に試すなら: `export CLAUDE_CODE_DISABLE_MOUSE_CLICKS=1` その後 Claude Code を再起動する。永続化するなら `~/.bashrc` や `~/.zshrc` に追記するだけだ。ホイールスクロールはそのまま使える。

使い方② — ハイフン付きMCPツールのフック設定例: 2.1.195 からはツール名はデフォルトで完全一致になる。複数ツールをまとめてマッチさせたい時は正規表現を使う。例:`~/.claude/hooks.json`

`{
  "hooks": [
    {
      "name": "code-review-hook",
      "matchers": ["code-reviewer"]
    },
    {
      "name": "brave-tools-hook",
      "matchers": ["mcp__brave-search__.*"]
    }
  ]
}`

解説: `code-reviewer` は正確一致。`mcp__brave-search__.*` はハイフンを含む MCP サーバーの全ツールにマッチする。

注意点・移行メモ — 音声/プラグイン/エージェント周り
▪️ macOS 音声入力: デフォルト入力デバイスを切り替えた長時間セッションで「無音を取り込む」問題を修正済みだが、該当バグを経験した場合は声入力を再起動すると確実だ。スペース無し言語（日本語など）の自動送信も機能するようになった。
▪️ プラグイン: プロジェクトの `.claude/settings.json` で有効化していた外部プラグインが別ロード経路で明示的インストール同意を要求しない問題を修正。以前に `/plugin enable` が失敗した場合は再実行を推奨する。
▪️ 背景エージェント: 新しいバージョンで書かれたジョブが消える/データを失う問題や、クラッシュ再開時に数秒の空白画面が出る問題を修正。疑わしい挙動を見たら 2.1.195 へアップデートすること。
