前回の記事で、Obsidian × Claude Codeの組み合わせで「AIがツールから相棒に変わった」という話をした。

今回は、その具体的なセットアップ手順を全公開する。

この記事を上から順にやれば、あなたのPCにも「自分の知識を全部理解したAI」が立ち上がる。所要時間は約30分～１時間。

---

## 前提：必要なもの

・PC（Windows / Mac / Linux どれでもOK）  
・Claude Pro または Claude Max のサブスクリプション（月額$20〜）  
・やる気（30分だけ集中）

Windowsの人は「WSL2」というLinux環境が必要になる。これも手順に含めるので安心してほしい。

---

## STEP 1：Obsidianをインストールする

まず「第二の脳」を作る。

1. <https://obsidian.md> にアクセス
2. 「Download」からインストーラーを落とす
3. インストールして起動
4. 言語を日本語に変更して「保管庫を新規作成する」を選ぶ
5. 保管庫名を決める（例：my-vault）
6. ロケーション（保存場所）を決める（どこでもOK。あとで変えられる）

これだけ。Vaultが作成されると、フォルダの中に空の場所ができる。ここに今後すべてを放り込んでいく。

**つまずきポイント：**  
最初から完璧なフォルダ構造を作ろうとしないこと。最初はメモを書き始めるだけでいい。構造は後からいくらでも変えられる。それがObsidianの強みだ。

---

## STEP 2：Obsidianの初期設定（最低限これだけ）

やることは2つ。「コミュニティプラグインの有効化」と「ブラウザ拡張の追加」。この2つは別モノなので、順番にやる。

---

**【2-A】コミュニティプラグインを有効にする**

Obsidianの設定画面（左下の歯車アイコン）を開く。

1. 左メニューから「コミュニティプラグイン」を選ぶ
2. 「コミュニティプラグインを有効化」をクリック
3. 「閲覧」をクリック（プラグインの一覧が出る）
4. 検索バーに「Iconize」と入力
5. 「Iconize」を選んで「インストール」→「有効化」

Iconizeはフォルダやファイルにアイコンをつけるプラグインだ。必須ではないが、Vaultが視覚的に見やすくなる。ここでは「コミュニティプラグインを入れる手順に慣れる」のが目的。

他に気になるプラグインがあれば、同じ手順でいくつでも追加できる。

---

**【2-B】Obsidian Web Clipperをブラウザに入れる**

ここからは**Obsidianの外**での作業。ブラウザ（Chrome / Edge / Firefox）側にインストールする拡張機能だ。

1. ブラウザで「Obsidian Web Clipper」と検索（またはChromeウェブストアで検索）
2. 「Chromeに追加」（Edgeの場合は「インストール」）をクリック
3. ブラウザのツールバーにObsidianのアイコンが追加される（拡張機能を固定：ピン留めしておくと便利）
4. 気になる記事を開いた状態でそのアイコンをクリック → Obsidianに記事が保存される

保存先のフォルダは、Web Clipperの設定から変更できる。僕は /Clippings というフォルダに保存するようにしている。

**つまずきポイント：**  
Web Clipperは「Obsidianのコミュニティプラグイン」ではなく「ブラウザの拡張機能」。Obsidian内で検索しても出てこない。必ずブラウザ側のストア（Chromeウェブストアなど）からインストールすること。

これで「知識を貯める装置」の準備は完了。

---

## STEP 3：Claude Codeをインストールする

ここからが本番。「相棒」を召喚する。

**Macの人：**

ターミナルを開いて、以下を実行。

```
npm install -g @anthropic-ai/claude-code
```

**Windowsの人：**

まずWSL2（Linux環境）を入れる必要がある。

1. PowerShellを管理者で開く
2. 以下を実行

```
wsl --install
```

1. 再起動を求められるので再起動
2. Ubuntuが立ち上がるので、ユーザー名とパスワードを設定
3. 以下を順に実行

```
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
source ~/.bashrc
nvm install --lts
npm install -g @anthropic-ai/claude-code
```

**つまずきポイント（Windows）：**  
WSL2のインストールでエラーが出る場合、BIOSの「仮想化機能（VT-x / AMD-V）」がオフになっている可能性がある。PCメーカー名 + 「BIOS 仮想化 有効」で検索すると手順が出てくる。

---

## STEP 4：Claude Codeを起動して認証する

インストールが終わったら、ターミナルで以下を実行。

```
claude
```

初回起動時に認証を求められる。ブラウザが開くので、Claude ProまたはMaxのアカウントでログインする。

認証が通ると、ターミナルにClaude Codeの画面が出る。

ここまでで「AIエンジン」の準備は完了。

---

## STEP 5：Claude CodeをObsidianのVaultにつなげる

ここが一番大事なステップ。

ターミナルで、ObsidianのVaultフォルダに移動する。

**Macの場合：**

```
cd ~/Documents/my-vault
claude
```

**Windowsの場合（WSL2から）：**

```
cd /mnt/c/Users/あなたのユーザー名/Documents/my-vault
claude
```

/mnt/c/ はWSL2からWindowsのCドライブにアクセスするパス。Obsidianの Vault がどこにあるか確認して、そのパスに合わせて変更する。

**確認方法：** Obsidianの設定 → 「保管庫について」にVaultのパスが表示される。

これでClaude Codeが、あなたのObsidian Vaultの中身を全部読める状態になった。

---

## STEP 6：最初の会話。「自分を知っているAI」を体感する

Claude Codeが立ち上がったら、まずはこれを打ってみてほしい。

```
このフォルダの中にあるファイルを一覧にして、どんな内容があるか要約して
```

Vault内のメモが少ない段階でも、Claude Codeがフォルダ構造を読み取って答えてくれる。

メモがある程度貯まっている人は、こんな質問を試してみてほしい。

```
過去1ヶ月に書いたメモの中から、共通するテーマを3つ見つけて
```

```
このVault内の情報をもとに、僕が今一番関心を持っているトピックは何だと思う？
```

返ってきた答えを見た瞬間、「こいつ、僕を知っている」と感じるはずだ。

これが前回の記事で書いた、AIが「ツール」から「相棒」に変わる瞬間。

---

## STEP 7：CLAUDE.mdを置く——「相棒」に自分を教える

ここからさらに一歩進む。

Vaultのルート（一番上のフォルダ）に CLAUDE.md というファイルを作る。

このファイルは、Claude Codeが起動するたびに最初に読む「説明書」だ。ここに自分の情報を書いておくと、毎回説明し直さなくてよくなる。

**最初はこれだけ書けばいい：**

```
# CLAUDE.md

## このVaultについて
- 所有者：[あなたの名前]
- 目的：[例：仕事の知識管理、記事執筆の素材置き場、学習ノート]

## フォルダ構造
- /Clippings — 気になった記事のクリップ
- /Notes — 日々のメモ
- /Projects — 進行中のプロジェクト

## 回答のルール
- 日本語で回答すること
- Vault内のファイルを参照して回答すること
```

これだけで、Claude Codeは「誰のVaultか」「何が入っているか」「どう振る舞うべきか」を毎回理解した状態でスタートする。

**つまずきポイント：**  
CLAUDE.mdを完璧に書こうとしないこと。最初は5行でいい。使いながら「これも教えておきたい」と思ったことを追記していけばいい。僕のCLAUDE.mdも、最初は3行だった。今は数十行ある。育てていくものだ。
