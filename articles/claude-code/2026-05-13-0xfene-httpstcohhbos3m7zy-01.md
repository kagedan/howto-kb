**Claude Codeに「〜を作って」と雑に投げて、出てきた成果物を見て「あれ、求めてたのこれじゃない、、、」ってなった経験**、ありませんか？

僕は何回もあります。

機能を作らせる、記事の構成案を考えさせる、業務オペを設計させる。雑に投げると、たいてい雑なものが返ってくる。じゃあ仕様をちゃんと書くか、、、と思ったその瞬間に、もう手が止まるんですよね。

「仕様を書く時間あったら自分で実装した方が早いやつ」

これ、Claude Code触ってる人なら、たぶん全員1回は通ってる道だと思います。

でも、この「実装前のヒアリング」を、AIに代行させるやり方があるんです。それが **Claude Code の AskUserQuestion**。かなり前から話題になっていたので、知っている&すでにやっている人もいるかと思うんですが、意外と使ってない方もいたので改めて書こうと思います！

今日は、実装着手前の「5分の取材」だけで、後工程の30時間の手戻りがまるごと消える話を、コピペで使える初動プロンプト3つと一緒にシェアしていきます。

「Claude Code触ってるけど、なぜか手戻りが多い気がする」と感じている方には、たぶん全部刺さるはずです。

# ■ そもそもAskUserQuestionって何？

これ、めちゃくちゃ良い機能なんですが、意外と「存在は知ってるけど使いこなしてない」人が多いんですよね。

Claude Code v2.0.21 で追加された、**AIから能動的に質問を投げてくる機能**です。

ただし、ただの「どうしますか？」じゃないんです。

**優秀なコンサルみたいな感じで、抽象的なアイデアをいくつかの適切な質問をもちいて、具体に落としてくれる**んですよね。

ここがポイントで、**AIが先にコードベースとか文脈を読んで、「A: 速いが脆い ／ B: 堅いが遅い ／ C: 別案」みたいに、選択肢自体に判断材料が乗ってきます。**

普通の質問だとこうなるところが、

> エラーハンドリング、どうしますか？

AskUserQuestionだとこうなります。

> エラーハンドリング、どれにしますか？

>  A) 即失敗（シンプル、デバッグ容易）

>  B) リトライ（最大3回まで自動再試行）（推奨）

>  C) カスタムハンドラ（独自ロジック実装）

もうこの差だけで全然違いません？

「考えなきゃ」じゃなくて「選べばいい」になる。これがめちゃくちゃデカいんです。

# ■ 一番ヤバいのは「ヒアリング下手な自分」を代行してくれること

ちょっと正直な話をします。

僕、ヒアリングがそんなに得意じゃないんですよ。

商談でも、社内のメンバーとの1on1でも、終わってから「あ、その論点聞き忘れた、、、」がよくある。後から「なんでそれ最初に確認しなかったんだ自分は、、、」って枕に顔を埋めるやつです（笑）

でも、AskUserQuestionを使うようになってから、ある発見があったんですね。

**聞き忘れの大半は「想像力」じゃなくて「網羅性」の問題。**

何を聞くべきかのチェックリストが頭にちゃんと入っていれば、聞き忘れは起きない。逆に、頭の中の引き出しに無い質問は、永遠に出てこないんです。

ここでAIが効くんですよ。

AIは、目的・成功条件・対象者・トレードオフ・リスク・運用・依存関係・スケジュール・既知の制約、っていう取材軸を全部もう知っている。じゃあ**、「取材」自体をAIに任せてしまえば、ヒアリングが下手な僕でも、レベルの高い要件定義ができてしまう**、と。

以前は、実装させて出てきた成果物を見て「あ、これ全然違うわ、、、」って作り直していたんですが、**AskUserQuestionで実装前にちゃんと固めてから走らせるようになってから、手戻りがバーッと減りました。**

体感ですが、取材を5分入れるだけで、後工程の30分の手戻りがまるっと消えます。
