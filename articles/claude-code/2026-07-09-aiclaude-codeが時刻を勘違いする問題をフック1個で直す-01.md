IT未経験でアプリ開発を進めているKKです。

Claude Codeを長く使っていると、地味にストレスがたまることがあります。  
それは日付や時刻の勘違いです。

実際にあったこと

* 昨日の話を「今日のこと」として扱う
* まだ朝なのに、作業の区切りで「おやすみなさい」と言ってくる
* 「24時間後に再度やってみます」と提案してくるが、何も起こらない（24時間後をカウントしていない）

Claude Code以外のAIを使っていても実際に発生したことがありました。  
最初は「AIってそういうものか」と思いつつ、わざわざ毎回今は何月何日の何時何分ですと伝えるのもめんどくさいし、直すほどでもないか。と放置していました。  
このように工数をかけてまで直す必要はないと考えている人は多いのではないでしょうか？

今回はAppleの返信待ちで時間ができたので、時刻対策に取り組んでみました。  
ぜひ気になる方は読んでみてください。

原因が分かってみると、settings.jsonにフックを1個足すだけで直りました。  
今日（2026-07-09）実際に設定して、動作も確認できたので、その手順を共有します。

## なぜ時刻を間違えるのか

調べてみると、理由はシンプルでした。

Claude Codeは、セッション開始時に「今日の日付」だけを渡され、時計（現在時刻）を持っていません。  
だから、セッションが日をまたいで長く続くと、開始時点の日付のまま話が進み、日付がズレます。  
そもそも「今が何時か」も分からないので、朝でも夜のあいさつをしてしまう、というわけです。

個人的にはチャットの最後に○時間前、○日前と表示が出るので、てっきりどのぐらい前に会話したか覚えているものだと思っていました。  
言ってしまえば、AIが現在時刻を認識すること自体、あまり価値がないと言えばそうなのかもしれません。

## 解決策：UserPromptSubmitフックで「今の日時」を毎回渡す

解決策はいくつかあるようです。  
わかりやすいのは、  
毎回会話の最初に、現在は○月○日○時○分（日本時間）といった文言を入れることです。  
これで認識してくれます。

ただ、めんどくさい笑  
会話のテンポも悪くなるのでこれは避けたいですね。

そこで使ったのは UserPromptSubmitフックです。

これは、自分がメッセージを送信するたびに、指定したコマンドを実行し、その出力をClaudeの文脈に自動で差し込む仕組みです。  
ここに「現在日時を出すコマンド」を仕込めば、毎回のメッセージに最新の日時が付いてくるので、Claudeが時刻を勘違いしなくなります。

## 設定方法

ユーザー全体の設定ファイル（全プロジェクトに効きます）に追記します。

* 場所：`C:\Users\<ユーザー名>\.claude\settings.json`

ここに、次の `hooks` を追記します。

```
{
  "hooks": {
    "UserPromptSubmit": [
      { "hooks": [ { "type": "command", "command": "date \"+現在日時: %Y-%m-%d (%a) %H:%M\"" } ] }
    ]
  }
}
```

これだけです。  
以降、メッセージを送るたびに「現在日時: 2026-07-09 (Thu) 10:30」のような1行が自動で渡されます。  
ちなみに時刻はClaude Code内部で認識しているだけで、表側には見えないようになっています。  
確かに一々チャットに現在日時、、、と表示されると見づらいですし、仕様としては良いと思いますね。

## つまずきやすいポイント

いくつか落とし穴があるので、先に共有します。

* 既存のsettings.jsonがある場合は、`"hooks"` キーを「追記」する。ファイルごと置き換えると、他の設定が消えます。
* JSONが壊れていると、フックは「黙って」無効になります。エラーが出ないので気づきにくいです。追記したら必ず構文チェックを。
* Windowsでも、Git Bashがあれば `date` コマンドのままで動きます。無い場合は、PowerShell形式に置き換えます。

```
"command": "powershell -Command \"Get-Date -Format 'yyyy-MM-dd (ddd) HH:mm'\""
```

## 検証のしかた（実際にやった手順）

私は次の順で確認しました。

1. まず、コマンド単体をターミナルで実行して、日時が出るか確認する
2. settings.jsonに追記したら、JSONの構文を検証する（`node -e` などでパースできるか）
3. 次のメッセージで、Claudeに「今何時？」と聞いて答え合わせする

実測では、設定後は再起動なしで、次のメッセージから有効になりました。

## コストはほぼ無視できるレベル

あと私が気になっていたのは  
「毎回時刻を認識させると、トークンの消費が膨大にならないか」  
です。

この記事を書いている2026年7月9日現在、Fable 5が解禁されており、めちゃくちゃトークンを消費します。  
だから少しでもトークンは節約したい。

調べてみると  
注入されるのは「現在日時: 2026-07-09 (Thu) 10:30」の1行だけです。  
1メッセージあたり約20〜30トークン（包む文言を入れても50未満）。  
1日に100回送っても数千トークン程度で、実用上は無視できる量だと言えるでしょう。

## おわりに

AIが賢くても、「今が何時か」のような当たり前の情報は、こちらから渡さないと分かりません。  
逆に言えば、必要な文脈を自動で渡す仕組み（フック）を1個用意するだけで、こういう地味な誤認はかなり防げます。

小さな設定ですが、日をまたぐ長いセッションが多い人ほど効くはずです。
