副業でいろいろと活動していると、ふと「やっぱり自分のホームページ、必要だな」と思う瞬間ってありますよね。

私の場合、KIZUNA Worksという屋号でITを使った業務効率化のお手伝いや、デジタルツールの導入サポートなど、ありがたいことに本当に幅広くいろんなことをやらせてもらっています。

そうやって活動の幅が広がってくると、名刺代わりになるような「私が何者で、どんなことができるのか」をまとめた基地みたいな場所がどうしても欲しくなってくるんです。

でも、いざ作ろうとすると「サーバー代やドメイン代で毎月のお金はかけたくない」「本業もあるし、保守やサイト更新に手間もかけたくない」というのが本音なんです。副業だからこそ、ランニングコストと運用コストは極限まで削りたい。

私も最初は「とりあえず無料でなんとかしよう」と軽い気持ちで始めたんですが、最初のやり方が思ったより手間がかかって……。エンジニアの性（さが）なのか、「この面倒な作業、もっとスマートに効率化できないかな」と感じてAIの「Claude（クロード）」に相談してみたら、あっさり解決策が見つかってなんとか形になった、というお話です。

今回は、副業エンジニアがAIに頼りながら、なるべくお金と手間をかけずに本格的なホームページを作ってみたリアルな過程を、ゆるーくお届けしますね。

ちなみにドメインは、Google Workspaceを始めた際にムームードメインで取得済みだったので、ホームページ制作にあたって追加のドメイン費用はゼロ。GitHub Pagesも無料なので実質ほぼコストなし、しかもAIのおかげで手間もかからず作れちゃいました。

  

## まず「無料でホームページを作る」を調べてみた

ホームページを作るとなると、普通はドメイン代やサーバー代がかかりますよね。

ただ私の場合は、以前副業用にGoogle Workspaceを始めた際、すでにムームードメインで自分のドメインを取得済みだったため、今回は新たなドメイン費用は発生しませんでした。

（GWS導入の経緯については、こちらの記事もよかったらどうぞ！）

👉 [【初心者向け】開業届の次に準備するものは「Google Workspace」！](https://note.com/kizuna_works/n/n4998cc8396d3)

これから新しくドメインを取ろうかなと考えている方には、管理がしやすくておすすめですよ。

👉 [ムームードメインはこちら](https://px.a8.net/svt/ejp?a8mat=4B1MHR+DIF60I+348+1BNBJM)

というわけでドメイン問題はクリアしていたので、次は「どうやってサイトを作って、どこに置くか」を調べ始めました。

まずは「無料　ホームページ作成」あたりで検索してみたんです。すると、「ペライチ」とか「Wix」「STUDIO」といったサービスがずらりと出てきました。

ちょっと触ってみたんですが、いやー、すごいですね！最近はプログラミングの知識がない非エンジニアの方でも、画面上のパーツをマウスでドラッグ＆ドロップするだけで、あっという間にオシャレなサイトが作れちゃうんです。「へえ、今はこんなに便利な時代なんだな」と素直に感心してしまいました。

ただ、ここで一つ問題が。

どのサービスも「無料で始められる」とうたってはいるものの、よくよく調べてみると、自分がすでに持っている「独自ドメイン」を使おうとすると、月額1,000円〜数千円の有料プランにアップグレードしないといけないことがわかったんです。

もちろん無料プランのままでも使えるんですが、そうするとURLがサービス側のものになったり、サイトの目立つところに広告が表示されたりといった機能制限がついてきます。

「せっかくKIZUNA Worksの独自ドメインを持っているのに、それを連携してサイトを公開するためだけに、毎月ランニング費用を追加で払い続けるのはちょっと避けたいな……」

それが、副業エンジニアとしての私の率直な判断でした。チリツモで毎月の固定費が上がっていくのは、副業にとって一番の敵ですからね。

そこでパッと思いついたのが、普段から業務で使っている「GitHub」の活用です。

GitHubには「GitHub Pages（ギットハブページズ）」という、自分で書いたコードをそのまま無料でWebサイトとして公開できる便利な機能があるんですよね。エンジニアとしてはすっかりお馴染みのサービスなので、「すでにドメインはあるし、公開場所はこれならタダでいける！」と迷わずこちらを選択することにしました。

そんな感じでコストゼロの道を選び、意気込んでサイトを作り始めました。この時点でもコードを書く作業自体はAIの「Claude Code」にお願いしていたので、最初の1、2ページを作るのは全然余裕でした。「お、AIに書いてもらえばサクサク進むじゃん。これで十分だね」なんて、余裕をかましていたんです。

ただ、この時のファイルの作り方は、いわゆる「HTMLべた書き」という、ページごとに独立したHTMLファイルを置いていく昔ながらの構成でした。

コンテンツが増えてくると、これが思わぬ罠になります。

ブログ記事を追加したり、作ったツールを紹介するページを増やしたりと、SEO（検索で上の方に表示させるための対策）も意識してページを量産し始めたんです。

すると、「全ページのヘッダーのメニューに『お問い合わせ』を1個増やしたい」と思ったときに、5ページあれば5ページ分、10ページあれば10ページ分のHTMLファイルを開いて、ページをまたいで共通パーツの更新を手作業で管理しないといけなくなってしまったんですね。

これ、実際にやってみるとわかるんですが、本当に地味で苦痛な作業なんです。「あれ、HTMLの記述自体はAIにお願いしてラクしてるはずなのに、なんで私こんなにひたすらページごとのコピペ作業してるんだろう……。これじゃ本末転倒じゃないか」と、思ってたより大変な現実に直面してしまいました。

![](https://assets.st-note.com/img/1776469442-p1mYsgPhRjvnG5qQkMVOSKl6.png?width=1200)

ページが増えるたびに更新コストが跳ね上がる、HTMLべた書き構成の落とし穴。

## Claude Codeに相談したら「Astroって知ってる？」と言われた

このままじゃコピペの繰り返しで更新が面倒すぎて、絶対にホームページの運用に挫折する……。

そう危機感を覚えた私は、日頃からプログラミングや業務効率化の壁にぶつかったときに頼りにしているAI、「Claude Code（クロードコード）」に相談してみることにしました。

「Claude Codeを使ってホームページを作っているんだけど、ヘッダーとか共通パーツを直すたびに全部のHTMLファイルを手作業で書き換えるのが本当にしんどくて。もっと自動化して効率よく更新できる、なんかいい方法ない？」

そんな風に、ちょっと愚痴をこぼすようなゆるいプロンプト（指示文）を投げてみたんです。そうしたら、「それならAstro（アストロ）というフレームワークはいかがですか？」という、なんとも魅力的な解決策を教えてもらえました。

Astroというのは、「静的サイトジェネレーター」と呼ばれるツールの一種です。ちょっと難しそうな名前ですが、要するに「Webサイトを効率よく、しかも爆速で作るための賢い仕組み」ですね。

Claudeが言うには、Astroの最大のメリットは、ヘッダーやフッターのような「全ページで共通するパーツ」を、1箇所でまとめて管理できる機能を持っていることだそうです。

つまり、ヘッダー専用のファイルを作っておけば、各ページからそれを読み込むだけで済むとのこと。

「え、それ最高じゃない？」と思い、早速Claudeのナビゲートに従ってAstroを導入してみました。

結果から言うと、もう感動の一言でした。

今までみたいに10ページ分のファイルをチマチマ開いてコピペで書き換える必要は一切なくなって、共通パーツのファイルを1つ直すだけで、全ページにパッと一瞬で変更が反映されるようになったんです。

実は私、自分で技術ブログを漁ってこのAstroにたどり着いたわけじゃないんですよね。単純に「今のやり方、しんどいんだけど助けて！」ってClaudeに丸投げしたら、「あなたの要望なら、これがおすすめですよ！」ってピンポイントで提案してくれただけなんです。

世の中には星の数ほど便利な技術がありますが、それを全部自分でゼロから調べて勉強するのは時間的に不可能です。でも、AIに自分の現状と「どうなりたいか」を聞けば、いまの自分にぴったり合った解決策がスッと出てくる。AIと一緒に作業するって、こういうことなんだなと深く実感した瞬間でしたね。

## Claude Codeで更新したら即ホームページに反映された

Astroのおかげでサイトの仕組みがスッキリ整った後、さらに私を感動させたのが「Claude Code」を使ったサイト更新の魔法のような流れでした。これが想像を絶する快適さだったんです。

「Claude Code」というのは、ブラウザでチャットするAIとは違い、黒い画面（ターミナル）上で動くAIエージェントのことです。パソコン内のファイルを直接見て、コードの編集からコマンドの実行までをこなす、エンジニア向けの強力な相棒ですね。

実際の更新フローは、こんな感じです。

まずは「VS Code」といういつも使っている編集ソフトでプロジェクトのフォルダを開き、黒い画面で「claude」とコマンドを打ってAIを起動します。

ここからが一番の見どころ。私が日本語で「新しいツールの紹介ページを追加して」と指示を出すだけで、作業は実質終わりです。

指示を受けたClaude Codeは、自動で該当ファイルを探し出していい感じにコードを書いてくれます。

しかも、事前にパスワードのようなもの（GitHubトークン）を設定してあるので、「アップロードして」と別途お願いする必要もありません。ファイルの変更を記録してネット上の保管場所へ送る作業（git add → commit → push）まで、全部自動で実行してくれるんです。

コードが送られると、今度は「GitHub Actions」という機能が裏側で動き出し、サイトの組み立てから公開（デプロイ）までを勝手にやってくれます。

つまり、「これを追加して」と指示を出したら、あとは公開まで完全な全自動。私がコーヒーを一口飲んで数分待っている間に、もう実際のホームページには新しい情報が反映されているんです。

難しいコマンドをいくつも手作業でカタカタ打ち込む……なんていうエンジニアっぽい作業はもう必要ありません。

「ちょっとここ直しといてー」と、隣にいる優秀な後輩アシスタントに頼む感覚でサイトが育っていく。指示するだけで公開まで完結するこの体験は本当にラクすぎて、もう昔のやり方には絶対に戻れませんね。

## 実際にできたホームページがこれです

そんなこんなで、最初はコピペ地獄に陥りかけながらも、AIに助けられてなんとか形になったホームページが[「kizuna-works.jp」](https://kizuna-works.jp/)です。

まだまだ作りかけの部分も多いですし、これから足していきたいアイデアもたくさんあるんですが、とりあえず「自分の基地」としてちゃんと動いてくれています。

サイト内には、日々のちょっとした業務効率化に使えるHTMLの無料ツールなんかを置いています。普段私が業務改善の現場で「こういう機能があったら絶対便利なのにな」と思って自作しているkintoneプラグインも、まだ作成中ですがゆくゆくは紹介していきたいですね。

あとは、こういった副業のリアルな体験談やITにまつわるブログ記事なんかも、これから少しずつマイペースに書き始めていくつもりです。

デザインについては、既存のテンプレートは使わず、Claudeと「こんな感じのレイアウトにしたいんだけど」と相談しながら一緒に作りました。すごく凝っているわけではないですし、プロのWebデザイナーさんが作ったようなピカピカで動きのあるサイトではありません。

でも、「自分が必要な情報を、それを必要としてくれる人にしっかりと届ける」という一番大事な目的は、十分に果たせています。

お金もドメイン代（既存のもの）以外は一切かかっていないし、日々の更新の手間もAIのおかげでほぼゼロです。

副業という限られた時間とリソースの中で運営していくには、これくらい「とりあえずちゃんと動いてるし、必要十分だよね」というゆるいスタンスが、ちょうどいいのかなと思っています。

これを読んでいるあなたも、もし「自分のサイトを持ちたいけど難しそう」と迷っているなら、「意外と自分でも作れそうだな」って思ってもらえたら嬉しいですね。

## 完璧を目指さないのが、長続きのコツかも

今回、自分のホームページをイチから作ってみて一番強く感じたのは、「最初から100点の完璧を目指さなくて本当によかった」ということです。

もし最初に「全部最新の技術でしっかりプログラミングして、デザインも完璧なサイトを作ろう！」と意気込んでいたら、HTMLのコピペ地獄の段階で、あるいはAstroの勉強の段階で、間違いなく挫折して放置していたと思います。

「なんか面倒だな」「ここから先はやり方がわからないな」と思ったタイミングで、すぐにAI（Claude Code）に相談して、Astroという便利な仕組みを教えてもらい、そのまま実際のコード更新作業まで丸投げして手伝ってもらう。

この「AIに聞きながら、走りながら少しずつ軌道修正していく」というスタイルが、本業の合間を縫って活動する副業のペースにすごく合っていました。

サーバー代も毎月無料のままで維持できて、面倒なコーディングの手間もAIが巻き取ってくれる。本当に、少しの工夫とAIの力で、無料でここまでできるすごい時代になったなと日々実感しています。

「副業用にホームページやポートフォリオが欲しいけど、技術的なハードルが高そうだし、維持費もかけたくない」と迷っている方がいたら、ぜひAIを頼ってみてください。全部ひとりで抱え込んでゼロからやろうとせず、AIを優秀なアシスタントとして活用すれば、きっとあなたの想像以上にサクッと形になりますよ。

実際のホームページはこちら（ 👉 <https://kizuna-works.jp/> ）から見られます。

また、「自分でもAstroとGitHub Pagesで作ってみたい！」という方向けに、実際のセットアップ手順をまとめた続きの記事も書いてみました。

カフェのコーヒー1杯分（300円）で読めるようにしているので、もしこれから作ってみようかなと思ったら、気が向いた時にでも覗いてみてくださいね。
