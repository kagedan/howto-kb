Claude Codeを作った本人・Boris Chernyがこんなことを言っていました。

「私はもうClaudeにプロンプトを打っていない。ループが走っていて、そのループがClaudeに指示を出している。私の仕事はループを書くことだ」

これを読んで、ちょっとびっくりしました。

Claude Codeを一番使いこなしているはずの人が、「手打ちでプロンプトを書く」フェーズをすでに卒業しているんです。

Anthropic社内の数字を見ると、変化の速さがよくわかります。

・本番コードの80%以上をClaudeが書いている
・エンジニア1人あたりの生産量が2024年比8倍
・新人のオンボーディングが数週間→2日に短縮
・チームによってはコードの90%以上がClaude製

「AIが補助している」という話じゃなくて、「AIが書いて、人間がレビューする」構造にすでになっているんです。

で、「ループを書く仕事」って何かというと、こういう仕組みです。

・TwitterやGitHub・SlackのフィードバックをClaudeに自動で収集させる
・そのデータをもとにClaudeがアイデアや改善案を出す
・それを次のアクションに繋げる
・人間はこの「仕組み全体」を設計する

ルンバを毎回手動で動かすか、スケジュールで自動化するか——そんなイメージに近くて、さらにそのルンバが「次にどこを掃除すべきか」まで判断してくれる仕組みを作っている感じです。

Borisはこれを「プログラミングの抽象化の歴史の次の段階」と表現していました。パンチカード→アセンブリ→Pythonと来て、「Claude Codeへの手打ち」が次の抽象化で置き換わっていく、という見方です。

もちろん「AIが書いたコードをちゃんとレビューできる人間」は依然として必要で、生産性が上がることと品質が保たれることは別の話なので、そこは冷静に見ておく必要があります。

そしてBorisが最後に語った言葉が印象的でした。

「最終的に人間がAIに教えられるのは、価値観だ。良い子どもに育てるように、良いモデルになる方法を教えること」

AIがコードを書き、AIがAIを作るサイクルが始まってきた今、人間に残るのは「何が大切か」を判断して渡す力なのかもしれません🤖
