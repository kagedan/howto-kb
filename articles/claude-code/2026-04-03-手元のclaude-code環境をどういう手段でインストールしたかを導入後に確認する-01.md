🤔.oO『そういえば手元のClaude Code環境、どういう経緯、手段でインストールしたっけか...』って思うことはありますか？私はつい最近ありました。でも後になってどういう手段でインストールしたのかって確認する術、あるのでしょうか。

調べて見たらありそうです。

当エントリでは、そんな時に使える確認手段を紹介します。

ちなみに動作環境のOSはMacOS前提で進めます。

```
% sw_vers
ProductName:            macOS
ProductVersion:         26.3
BuildVersion:           25D125
```

## ネイティブで導入したかを確認する方法(claude doctor または /doctor)

まずは doctorコマンドを実行して確かめてみます。Claude起動時に`claude doctor`と併せて実行するか、Claudeログイン後にスラッシュコマンドとして `/doctor`と実行。結果を確認します。

今回の場合、診断結果は`native`と出ました。どうやら過去の私はネイティブで導入していたようです。ちなみにネイティブ＝『Claude Code専用の自己完結型バイナリとして直接インストール・実行されている』という意味。

```
% claude doctor

  Diagnostics
  └ Currently running: native (2.1.91)
  └ Path: /Users/xxxxxxxxxx/.local/share/claude/versions/2.1.91
  └ Invoked: /Users/xxxxxxxxxx/.local/share/claude/versions/2.1.91
  └ Config install method: native
  └ Search: OK (bundled)
  Updates
  └ Auto-updates: enabled
  └ Auto-update channel: latest
  └ Stable version: 2.1.84
  └ Latest version: 2.1.91
  Version Locks
  └ No active version locks
  Press Enter to continue…
```

`which -a claude`で「今のシェルから見えている claude がどこにあるか」「過去に入れた別経路の claude が残っていないか」を確認することも出来ます。結果は以下の形式で出力されました。この方式だと上の行ほど優先されやすいので、「今はシェルのPATH上で `/Applications/cmux.app/...` の claude が優先され、その下に別経路でインストールされた claude が残っている可能性がある」と読むことが出来ます。

```
% which -a claude
/Applications/cmux.app/Contents/Resources/bin/claude
/Users/xxxxxxxxxx/.local/bin/claude

% ls -l "$(which claude)"
-rwxr-xr-x  1 xxxxxxxxxx  admin  4326  x月 xx xx:xx /Applications/cmux.app/Contents/Resources/bin/claude
```

`ls -l` の結果が `/Applications/cmux.app/Contents/Resources/bin/claude` を指しているのは、シェルの PATH 上でこちらが先に見つかるためです。cmux.app はターミナル管理系のアプリで、その中に claude のラッパーコマンドが含まれています。(ファイルサイズが 4326 バイトと小さいのもそのため)

一方、`claude doctor` が `native` と診断しているのは、claude バイナリ自身が「自分はどのように導入されたか」を内部で認識して返しているためです。cmux.app 経由で呼ばれても、実体は `/Users/xxxxxxxxxx/.local/share/claude/versions/2.1.91` の native バイナリが動いています。

つまり「`which claude` が cmux.app を指す ≠ cmux.app でインストールした」ということ。`doctor` の診断が `native` であれば、インストール方式は native と判断して問題ありません。

ちなみにネイティブインストール＝curlコマンドによる導入方式です。下記公式ページだと『Terminal』タブに展開されている手順です。

<https://code.claude.com/docs/en/quickstart#step-1%3A-install-claude-code>

## brewでインストールしたかどうかを確認する方法

今回はネイティブで導入していたので即一件落着、終了...！な流れでしたが、その他のパッケージ導入ツールでの導入確認コマンドも見ていきましょう。まずはbrew。brewコマンドの結果に情報が出ていれば Homebrew 管理の可能性が高いです。

私の環境ではbrewでも結果が出ましたがバージョンが古めのもの。`Not installed`とも出ています。brewにも情報は残っていたがNot installedの状態という診断ですね。そういえば以前brewでインストールしてましたが一旦取り止め、ネイティブで入れ直した気がしてきました。(←

```
% brew list --cask | grep claude-code
% brew info --cask claude-code
==> claude-code ✘: 2.1.81
https://www.anthropic.com/claude-code
Not installed
From: https://github.com/Homebrew/homebrew-cask/blob/HEAD/Casks/c/claude-code.rb
==> Name
```

## npmでインストールしたかどうかを確認する方法

そしてnpm。npm list -g に @anthropic-ai/claude-code が出れば npm グローバル導入の可能性が高いです。今回はそれらしい情報は出ませんでした。

npm系は global prefix を ~/.npm-global のように変えているケースもあるので、npm config get prefix も一緒に見ると判断しやすいです。

```
% npm list -g --depth=0 | grep @anthropic-ai/claude-code
% npm config get prefix
/opt/homebrew
```

## まとめ

という訳でClaude Code環境の導入方法確認手順の紹介でした。ちなみに現時点では冒頭紹介したネイティブ(Native Install)での導入方式が推奨されています。個々の環境や状況があるかと思いますが、差し支えなければネイティブインストールしておいた方が良さそうですね。  
![](https://static.zenn.studio/user-upload/bda24ed832e7-20260404.png)

Claude Codeは日々バージョンが新しくなり、何かしら案内があった場合は都度最新のものに入れ替えてください、または『この方式で入れ直すのがオススメです』というアナウンスもあったりします。適宜導入状況を確認し、安心安全に利用出来る(出来ている)環境を整えていくようにしましょう。
