## 背景：なぜ金融サービスにAIエージェントが必要か

金融機関のアナリスト・審査担当者が日々こなす業務の多くは、高度な判断を要する一方で、データ収集・照合・フォーマット変換といった反復処理が含まれています。

- **ピッチブック作成**：財務データの収集 → モデル構築 → 資料化に数日かかる
- **KYCスクリーニング**：エンティティ確認・書類レビュー・エスカレーション判断を手動で行う
- **月次決算**：仕訳チェック・残高照合・報告書作成が毎月の定型作業

AnthropicはClaude Opus 4.7を基盤に、これらの業務を自動化するテンプレートを「すぐにデプロイできる形」で提供します。公式発表によると、同モデルは**Vals AI Finance Agent ベンチマークで64.37%を達成**し、金融タスクにおいて現時点のSOTAを記録しています[^1]。

## 10種のエージェントテンプレート

テンプレートは2つのカテゴリに分類されています。

### カテゴリ1: 調査・顧客対応（Research and Client Coverage）

![調査・顧客対応5種エージェントテンプレート](https://raw.githubusercontent.com/kai-kou/zenn-blog-automation/main/images/claude-finance-agents-microsoft365-guide/03-agent-templates.png)

#### 1. Pitch Builder（ピッチ資料作成）

ターゲットリストを入力すると、類似企業分析（コンプス）・財務モデル・ピッチブック（PowerPoint）・カバーレター（Word）を自動生成します。

公式の説明によると、Microsoft 365連携により「モデルで始めた作業がデッキで終わる」ワークフローを実現します。

#### 2. Meeting Preparer（ミーティング準備）

顧客・カウンターパーティの情報を自動収集し、コール前のブリーフィング資料を作成します。FactSet・S&P Capital IQなどのデータソースに接続することで、直近の財務指標・ニュースを含んだ資料を生成できます。

#### 3. Earnings Reviewer（決算レビュー）

決算トランスクリプト・SEC/EDINETファイリングを読み込み、財務モデルを更新します。「投資テーゼに関連する変化」を自動フラグ立てする機能が含まれます。

#### 4. Model Builder（財務モデル構築）

ファイリングとデータフィードから財務モデルを作成・維持管理します。ExcelアドインのClaude連携により、フォーミュラのデバッグや感度分析を自然言語で指示できます。

#### 5. Market Researcher（市場調査）

セクター動向のトラッキング・ニュース合成・リサーチレポートの統合を行います。IBISWorld（業界分析）やThird Bridge（一次情報インタビュー）などのデータソースと連携します。

---

### カテゴリ2: 財務・オペレーション（Finance and Operations）

#### 6. Valuation Reviewer（バリュエーション確認）

類似企業・類似取引との比較と手法スタンダードの確認を行います。

#### 7. General Ledger Reconciler（総勘定元帳照合）

勘定照合・NAV計算を実行します。

#### 8. Month-End Closer（月次決算）

クローズチェックリストの実行・仕訳伝票の作成・各種レポート生成を自動化します。

#### 9. Statement Auditor（ステートメント監査）

財務諸表の整合性確認・監査対応準備を支援します。

#### 10. KYC Screener（KYCスクリーニング）

エンティティファイルの組み立て・書類レビュー・エスカレーション対応をパッケージ化します。Dun & Bradstreet（企業識別照合）などと連携し、コンプライアンス要件に対応した確認フローを構築します。

## デプロイ方法

テンプレートは2つの形式で提供されます[^2]：

### オプション1: Claude Cowork / Code プラグイン

アナリストのデスクトップ上でClaudeが並走する形式です。Claude CoworkまたはClaude Codeにプラグインとして追加します。

```bash
# Claude Codeへのプラグイン追加例
claude plugin add @anthropic/finance-pitch-builder
```

有料プランのClaude Cowork/Codeユーザーであれば、即座に利用を開始できます。

### オプション2: Claude Managed Agents クックブック

Claude Platform上で自律実行する形式です。監査ログ・権限管理・スケジュール実行が含まれます。

テンプレートは`anthropics/financial-services-plugins`リポジトリ（GitHub）で公開されており、各エージェントの実装パターンをカスタマイズして自社のリスクポリシー・承認フローに適合させることができます。

```python
# Claude Managed Agents クックブック実装例
import anthropic

client = anthropic.Anthropic()

# Step 1: エージェントを定義
agent = client.beta.agents.create(
    name="KYC Screener",
    model="claude-opus-4-7",
    system="You are a KYC screening specialist. Assemble entity files, review documents, and package escalations.",
    tools=[{"type": "agent_toolset_20260401"}],
)

# Step 2: 実行環境を作成
environment = client.beta.environments.create(
    name="kyc-env",
    config={"type": "cloud", "networking": {"type": "unrestricted"}},
)

# Step 3: セッションを起動してタスクを送信
session = client.beta.sessions.create(
    agent=agent.id,
    environment_id=environment.id,
)

with client.beta.sessions.events.stream(session.id) as stream:
    client.beta.sessions.events.send(
        session.id,
        events=[{
            "type": "user.message",
            "content": [{"type": "text", "text": "Screen entity: Acme Corp, jurisdiction: JP, enhanced due diligence"}],
        }],
    )
    for event in stream:
        if event.type == "agent.message":
            for block in event.content:
                print(block.text, end="")
        elif event.type == "session.status_idle":
            break
```

## Microsoft 365 完全統合

Claude for Microsoft 365は、Excel・PowerPoint・Word・Outlook（近日提供予定（coming soon））の各アドインとして提供されます。**アプリをまたいでコンテキストが自動的に引き継がれる**のが最大の特徴です。

### Excelアドイン（GA）

```
機能:
- ファイリング・データフィードから財務モデルを構築
- フォーミュラのデバッグと説明（セルレベルの参照付き）
- 感度分析の自動実行
- スクラッチからのスプレッドシート作成
```

エージェントの変更はトレースが記録され、どのセルをなぜ変更したかを自然言語で説明します。

### PowerPointアドイン（GA）

元のデータが変わると内容が自動更新されるデッキを作成します。ピッチブックや決算レビュー資料の管理に有効です。

### Wordアドイン（GA）

社内テンプレートに沿ったクレジットメモや調査レポートの編集を自然言語で指示できます。

### Outlookアドイン（近日提供予定（coming soon））

ミーティング前のブリーフィング送付・フォローアップメールの自動作成などを予定しています。

**統合の仕組み：** 各アドインは単独で動くのではなく、コンテキストを共有します。Excelで構築した財務モデルを、再度説明することなくPowerPointのピッチブックに反映できます。

## 注意点

### カスタマイズの必要性

テンプレートはそのまま動きますが、**自社のリスクポリシー・承認プロセス・モデリング規則**に合わせたカスタマイズを前提に設計されています。公式ドキュメントでは「firm's internal modeling conventions, risk policies, and approval processes」への適合を推奨しています。

### データガバナンス

外部データコネクタ（FactSet・Moody's等）は、**governed access controls**のもとで接続されます。社内データウェアハウス・調査リポジトリ・CRMとの統合も可能ですが、アクセス制御の設定は各組織が管理する必要があります。

### 金融規制への対応

エージェントが生成した調査・分析・推奨事項の最終確認は、引き続き人間のアナリスト・コンプライアンス担当者が行う責任があります。テンプレートは業務効率化を目的としており、規制上の意思決定を代替するものではありません。
