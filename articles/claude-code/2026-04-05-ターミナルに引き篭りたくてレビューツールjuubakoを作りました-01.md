# はじめに

このブログ/ツールは現時点ではmacOS限定の内容になります。

## 最近の開発

Claude Codeやcodexなど、ターミナルベースのAIエージェントを本格的に使い始めてから、タブの管理やメモリの観点から、イケてるターミナルをいい感じに整備してそこに引き篭もることが増えた。

結果として、エディタを開く回数が減っていて、ある程度開発した後にgitのdiffのページを眺めてコードの差分を確認するぐらいがエディタのメイン機能な気がしてきた。  
生のコードを書くぞという時は開く気が起きるが、コーディングエージェントと格闘しているときに開くにはエディタはデカ重すぎる。（自分は若者なので、cliエディタはどうにもなじまず...)

## どこで生成されたコードを見るといいか

選択肢としては以下ぐらい？

* git commandで頑張る
* エディタを開く
* cliエディタやツールを勉強する
* いったんpushしてweb UIで見る
* 見ない

見るためだけにエディタ開くのもなんだかな、という感じがしてきた。GUIでいい感じに楽ちんに見たいけど。なんなら気づいたタイミングでAIに文句はつけたいけど。

つまり要件はこんなもん。

> ターミナルからコマンド一発で、diffをサクッと見て、気になるところにコメントして、AIに渡す

個人的にはGitHubのPRのレビューは、レビューをつけるには結構使いやすい（慣れている）感覚があるので、cliでポン呼びできるようにしたらいいかな、と思った。

# やったこと

## juubako を作った

<https://github.com/iammytoo/juubako>

ターミナルで `jbk` と打つだけ。

ウィンドウが開いて、GitHubのpull request風のdiffビューが表示される。

![](https://static.zenn.studio/user-upload/de4e6c6a2ece-20260405.gif)

生成コードの「重箱の隅」つついていきましょう。

## 機能

* **Unified / Split表示** — GitHubのPR画面と同じ感覚で切り替え
* **ブランチdiff & 未コミットdiff** — PRを作らなくても、ローカルで即レビューできる
* **インラインコメント** — 行をクリックしてコメントを残す
* **Viewedチェック** — ファイルごとにレビュー済みをマーク。進捗が見える
* **レビュー履歴** — 過去のレビューを一覧表示。同じ差分ならコメントを復元
* **Cmd+R** でリロード、**Cmd+S** で保存

## AIにフィードバック

「Save Review」するとコメントが `.juubako/` ディレクトリにテキストファイルとして保存される。

```
# juubako review
# date: 2026-04-05 17:30:00
# mode: branch (main...feature-branch)
# hash: a1b2c3d4e5f6g7h8

## src/app.ts:42
This function should handle errors more gracefully.

## src/utils.ts:15
Consider using a streaming approach here.
```

ファイル名、行番号、コメント内容がシンプルなフォーマットで入っている。AIエージェントにこのファイルを渡せば、そのまま修正に取りかかれる。（はず）

```
.juubako/2026-04-05-17-30-00.txt を読んで、指摘を修正して
```

個人的には、こんなもんで進めたい。

## 技術的な話

選定技術

* **Rust** — git操作、diffパーサー、コメント管理。全てのgitコマンドはread-only
* **Tauri v2** — WebViewベースのネイティブmacOSウィンドウ。Electronと違ってバイナリ8MB、起動も速い
* **React + TypeScript** — diff表示UI。

macOSのフルスクリーンターミナルから起動しても同じSpaceにウィンドウが出るように、Accessoryモード + `MoveToActiveSpace` を設定している。ターミナル中心の開発フローを邪魔しない。  
gitコマンドの上に乗っかっている。その辺のアルゴは実装していない。git2-rsなるものがあるのは作ってから気が付きました。速度的には要検討。

# 使い方

## インストール

```
brew tap iammytoo/tap
brew install juubako
```

以降は任意のgitリポジトリで `jbk` を実行するだけ。

リポジトリごとにデフォルトのベースブランチを設定したい場合は `.juubako/config.toml` を置ける。

```
[diff]
base_branch = "develop"
```
