## なぜやろうと思ったか

月次レポートをAIに作らせようと思ったとき、先に気づいたことがありました。

「このまま指示しても、毎回同じ説明をしなければいけない。」

「アイ太郎のキャラクターで書いて」「スプシ用にタブ区切りで出力して」「読者は副業したい会社員で…」

AIに指示するたびに、この前置きを毎回打ち込んでいました。月次レポートを丸投げする前に、まずこの問題を解決しようと思いました。「CLAUDE.mdという設定ファイルを作れば、Claudeがプロジェクトの文脈を自動で読み込む」という話を知り、試してみることにしました。

![](https://assets.st-note.com/img/1779467435-e4CmKzl2QcL0OHNySsXbEpJq.png?width=1200)

---

## やったこと

### まず、Claude Codeの機能を棚卸しした

「12 Claude Code Features Every Engineer Should Know」という動画をきっかけに、自分が使えている機能を数えてみました。

**結果：12個中、意識して使えていたのは2〜3個でした。**

| 機能 | 状態 |  
|------|------|  
| [CLAUDE.md](http://CLAUDE.md) | ❌ 未設定 |  
| MCP（外部ツール連携） | ✅ 使用中 |  
| Memory（記憶） | ✅ 使用中 |  
| Skills（専用スキル） | △ 存在は知っていた |  
| Hooks・Plan Mode等 | ❌ 未使用 |

「Claude Codeを使っている」と思っていましたが、ポテンシャルの2〜3割しか使えていませんでした。

---

### CLAUDE.mdを設置した

プロジェクトフォルダに`CLAUDE.md`というファイルを1枚置くだけです。中身は日本語で書けます。

入れた内容：

**効果はすぐ出ました。** 翌朝「今日の投稿3本お願い」と一言打つだけで、アイ太郎のキャラクターで、スプシ用の形式で、3本が出てきました。前日まで5行の前置きが必要だったのに。

---

### 専用スキルを2本作った

`.claude/skills/`フォルダにMarkdownファイルを1枚置くと、`/スキル名`で呼び出せる専用コマンドになります。プログラミングは不要です。

作ったスキル：

**`/post`（毎日の投稿作成）**  
→ テンプレートを参照してアイ太郎キャラで3本作成、スプシ用タブ区切りで出力まで一気通貫

**`/note`（note記事作成・アイケン審査）**  
→ テンプレート参照→記事執筆→アイケンが公開前審査→スコア判定まで一気通貫

---

### 月次レポートを丸投げした

環境が整ったので、5月の月次レポートをClaude Codeに作らせてみました。

実際に手を動かしたのは、設定込みで合計3〜4時間だけでした。  
（CLAUDE.md作成：約1時間／スキル作成：約1時間／微調整・確認：約1〜2時間）

渡したのはスプシの数字だけです。売上・経費・投稿数・フォロワー・noteのビューとスキ数。それだけで、レポート骨格・note記事・アイケン審査カード・ダッシュボード（数字を自動で見やすく整理した管理画面）まで1日で揃いました。

## 気づいたこと

**「使っている」と「使いこなしている」は全然違う。**

12個の機能のうち2〜3個しか使えていなかった。この事実が一番刺さりました。同じことは、他のAIツールにも言えると思います。使えているつもりで、実は表面だけしか触っていない。

**CLAUDE.mdは「社内ルールを覚えさせる」という感覚に近い。**

新入社員に仕事を教えるイメージです。一度教えれば、毎回最初から説明しなくていい。AIの場合は、ファイル1枚で翌日から動きが変わります。

**設定した当日から効果が出る。**

難しい準備は何もありませんでした。Markdownが書ければ、今日から同じことができます。

最初の設定には数時間かかりました。でも、一度作ったルールは翌日から使い回せます。「毎回説明する時間」が消えたのが、一番大きな変化でした。

## この準備の先で

環境を整えた上で、5月の月次レポートをAIに丸投げしました。結果は実験⑧で全部公開します。

Claude Codeにまだ使えていない機能が9個あります。「人間がトリガーを引かなくていい」状態にどこまで近づけるか、引き続き記録していきます。

まだ答えは出ていません。

AIに"社内ルール"を渡したこと、ありますか？
