* AIエージェントって、実質なんなんだろう？ただプロンプトを呼ぶのと何が違うのかな
* MastraやLangChainっていうフレームワークの存在は知っているけど、何ができるかわからない
* Claude CodeやCodexは使っているが、中身がどうなっているかまったく想像できない
* なんかLLM使った開発要件をアサインされたが、AIエージェントとして解くべきかわからない

といった方がAIエージェントのコア概念を掴めるような100行足らずのコードを考えてみました！

個人的に2025年4月頃から実務と個人開発でAIエージェントを開発している中で、社内外に対してAIエージェントとは何かを説明する機会が多かったのですが、それらの経験と、自分が最初からこの概念を掴んでいたらもっと速く開発できたかも、という思いを込めて書いてみました。

本記事では上記Gistの簡単な解説を述べます。さっさと理解したい方は、上記のGistをClaude Codeにでも渡して解説してもらうと良いと思います。

## エージェントは、LLM自身が終了して良いと判断するまで、ツールの呼び出しを続ける

AIエージェントが実現する要件で、個人的にもっとも革新的だと思うところがこれです。  
以下のコードにあるように、AIエージェントは、コードの終了条件を原則としてLLMが判断します。

```
for (let step = 0; step < 6; step++) {
  const res = await client.chat.completions.create({
    model,
    temperature: 0,
    response_format: { type: "json_object" },
    messages,
  });

  const text = res.choices[0].message.content ?? "{}";
  const action = Action.parse(JSON.parse(text));

  messages.push({ role: "assistant", content: JSON.stringify(action) });

  if (action.type === "end") {
    console.log("\n[final]");
    console.log(action.answer);
    process.exit(0);
  }
```

サンプルコードではエージェントに4つのツールが登録されています。  
通常のプログラミングでは、4つあるツールのうちいくつ呼ばれるかは決定的です。この入力に対してはこのツールが呼ばれる、呼ばれないが必ず一意に定まります。  
しかしエージェントでは、LLMが、もう十分に役目を果たせたと判断したら専用のフラグを出力し、そのフラグを以てループを抜け出す形で処理が終了します。こうすることで、自律的にアプリケーションを動かすという要件が実現できるのです。

本記事の要点としては、LLMが自律的に終了を判断するというのは、意外と上記のように原始的な機構でも実現できるんだということです。ブラックボックスに見えるClaude CodeやMastraやLangChainといったエージェントの見え方が、少し変わるのではないでしょうか。

## 本質的には、AIのAPI経由での呼び出しを大量に行っているだけであり、”魔法”ではない

ここまでの内容にあるように、エージェントは魔法ではありません。見慣れたAIのAPI呼び出しを、分岐やループの中でLLM自身に判断させながら繰り返しているだけです。

この構造を理解していると、AIエージェントのフレームワークを使うときに自分が設定した値がどう作用するかイメージしやすくなります。コンテキストエンジニアリングのような話題も、具体的な仕組みと結びつけて考えられるようになるはずです。ぜひ、これを意識しながらエージェント開発に取り組んでみてください。

もう一つ伝えたいのは、何でもエージェントで解こうとしないことの重要性です。AIのAPIを数回呼び出すだけで済む要件をエージェントにすると、前述のとおり終了条件が非決定的になる分だけ、むしろ欠点が増えます。今回のコードを踏まえて、自分が解こうとしている問題が本当にエージェンティックに解くべきなのかを見極めてみてください。
