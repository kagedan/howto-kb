【まず前提の用語だけ押さえて】

このスレッドに出てくる言葉、これだけ知ってれば全部読める↓

- AIエージェント＝AIが自分で判断してツールを使い、タスクを最後まで進めるシステム
- ツール＝AIが使える機能（ウェブ検索・ファイル編集・コマンド実行など）
- プロンプト＝AIへの指示文
- コンテキスト＝AIが今の会話で覚えている情報のかたまり
- トークン＝AIが文章を扱う最小単位（だいたい日本語1〜2文字で1トークン）

【1つ目】そもそもエージェントって何？（Anthropic公式定義）

「モデル（AI本体）がツールをループ（繰り返し）で使うもの」
これだけ

①タスクを与える
②AIがツールを使って作業
③結果を見て判断を更新
④完了まで自律的に続ける

構成要素は環境・ツール・システムプロンプトの3つだけ
シンプルに保つほど性能が出る

【2つ目】エージェント使うべき4つの条件

下記が全部YESの時だけ使え。1個でもNOなら別の方法↓

①タスクが複雑（人間が手順を順番に書き出せないレベル）
②価値が高い（売上直結や時間節約効果が大きい）
③必要なツールをAIに渡せる
④エラーが検知・修正可能（取り返しがつく）

何でもエージェント化は逆に金と時間の無駄

【3つ目】Anthropic最重要原則：「エージェントの気持ちで考えろ」

これが全テクニックの中で一番大事

具体的には↓
- そのツール説明・スキーマ（仕様書）だけ見せられて、自分が混乱しないか想像する
- 人間が理解できないなら AI も絶対に理解できない

ツール設計もプロンプト設計も、この視点が無いと全部ズレる

【4つ目】エージェントには「新卒インターン」みたいに指示しろ

「これくらい分かるだろ」は通じない
一般原則（ヒューリスティック＝経験則）は全部明文化する

例：
- 「答えが見つかったら検索を止めていい」
- 「シンプルな質問は5ツール以下、複雑なら10〜15まで」

これを書くだけで暴走が止まる
書かないとAIは「もっと完璧を求めて」延々動き続ける

【5つ目】ツール選択の指示が9割

AIは「どのタスクでどのツール使うべきか」を最初から知らない
ツール並べただけで放置されるのが大半の失敗パターン

例：
「会社の情報なら Slack を優先して検索せよ」
「コード関連は GitHub を最初に当たれ」

このコンテキスト固有のルールを書くだけで精度が激変する

【6つ目】思考プロセスを誘導する裏技

extended thinking（拡張思考＝答える前に長く考える機能）はオンにするだけで効くが、更にチューニング可能↓

- 「最初の thinking ブロックで計画を立てろ」
- 「いくつツールを使うか先に決めろ」
- 「どんなソースを当たるか先に書け」
- 「成功条件を先に定義せよ」

「考えろ」じゃなく「何をどう考えるか」を指示するのがコツ

【7つ目】Claude 4 の新機能：ツール呼び出し間の思考（Interleaved Thinking）

ツール実行の合間に毎回考える機能（Claude 4 で追加）

普通のAI：ウェブ検索結果を全部正しいと信じて即使う
正解：「結果が信頼できるか毎回 thinking で振り返れ」と指示する

これでウェブから拾った嘘情報をそのまま使う事故が激減
免責事項を自分で付けるようにもなる

【8つ目】エージェントは予測不能：副作用に必ず注意

実例で起きた事故↓

指示：「完璧なソースが見つかるまで検索を止めるな」
副作用：完璧なソースが存在しない場合、コンテキスト枯渇まで延々検索し続けて死ぬ

正解：「完璧なソースが無ければ数ツール後に止めていい」を必ず追記

プロンプト1行追加で必ず想定外が起きる前提で書け

【9つ目】コンテキスト管理①：コンパクション（24時間動かす最大の鍵）

ここが本日のメイン裏技↓

Claude Code には自動コンパクション機能（自動圧縮）がある
- 約19万トークンに達すると自動で発火
- 全コンテキストを「濃いけど正確な要約」に圧縮
- 要約を新しい Claude に渡して継続

結果：「ほぼ無限」に動かし続けられる
寝てる間も動き続ける仕組みの正体がこれ
