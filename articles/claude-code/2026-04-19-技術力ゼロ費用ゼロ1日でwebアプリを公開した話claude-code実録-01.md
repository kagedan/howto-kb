ここでは正直に書こうと思った。

かっこいい話じゃない。  
「AIで稼いだ」でも「天才的な開発をした」でもない。

「何が起きてるか、全部はわからなかった。」

それでも動いた。公開できた。

18年のSE経験を持つ自分が、はじめてAIに開発を任せた日の記録を残しておく。

## 第1章「完成した。でこれ何に使うんだ？笑」

アプリが公開された瞬間、最初に出た言葉がこれだった。

達成感がないわけじゃない。  
でも正直に言うと、何が起きたか全部は理解できていなかった。

エラーは12回出た。  
デプロイスクリプトは5回走らせた。  
「セキュリティ大丈夫か？」と、たぶん10回以上確認した。

それでも完成した。

「わからなくても、進めた。」

これがすべてだった。

## 第2章　何を作ったのか

作ったのは「[CLAUDE.md](http://CLAUDE.md) ジェネレーター」という無料ツールだ。

Claude Code（AIを使った開発ツール）でプロジェクトを始めるとき、CLAUDE.mdというファイルを置くと、AIがセキュリティルールを守って動いてくれる。

でも、このファイルを自分で書くのは難しい。  
何を書けばいいかわからない。設定ミスが怖い。

だから作った。  
「4つの質問に答えるだけで、自動生成してくれるツール」を。

費用：0円。  
データ送信：なし。  
サーバー：なし。

ブラウザの中だけで動く、完全無料のツールだ。

<https://ch-ragge.github.io/claudemd-generator/>

## 第3章｜実録：12回エラーが出ても進めた理由

ここが本題だ。

ログに記録が残っている。事実だけ書く。

デプロイ試行回数　：5回  
エラー発生件数　　：12件  
　内訳：認証失敗 ×6、誤検知エラー ×6  
開始から成功まで　：約13分

08:09に始めて、08:22に成功した。  
13分の間に、12回エラーが出た。

でも焦らなかった。

エラーが出るたびに、AIが動いた。

１．ログファイルに自動保存  
２．エラーの原因を推定  
３．対処案を3つ提示  
４．「どれを実行するか」を確認

自分はYes/Noを判断するだけでよかった。

これが「わからなくても進められた」理由だ。

## 第4章｜セキュリティが不安で、何度も止まった

作業中、一番繰り返したのがセキュリティの確認だった。

「このAPI、課金されないよね？」  
「トークンって漏れてないよね？」  
「このコード、本当に安全？」

家族に迷惑をかけるわけにはいかない。  
技術の知識が薄い分、不安が先に来る。

AIに何度聞いても、毎回答えてくれた。

今回の設計はこうなっている。

【課金暴走】→ AWS・外部APIは一切使わない  
【データ漏洩】→ サーバーなし・DBなし・ブラウザ完結  
【トークン漏洩】→ 使用後に自動削除・メモリクリア  
【予期しない出費】→ GitHub Pagesは完全無料

今日の支出：0円。

確認することで、不安は消えた。  
「わからないまま進む」のではなく、「確認して納得してから進む」。

それだけでよかった。

## 第5章｜「わからない」は止まる理由じゃなかった

今日気づいた。

経験年数は、「踏み出す勇気」とは別の話だ。

わからなくても進んだ。  
エラーが出ても止まらなかった。  
不安があっても確認しながら前に進んだ。

それだけで、世界に公開できた。

AIは怒らない。何度聞いても答えてくれる。  
判断だけ、自分がすればいい。
