* `SessionEnd` フックから `claude` を起動すると、それ自体が新しいセッションになり、終了時にまた `SessionEnd` を呼ぶ無限ループになる。
* 終了系フックは同期で動いて完了まで待つので、重い処理を書くと `/clear` のたびに固まる。
* ガードは「成功すれば記録が残る」式の出口頼みではなく、`session_id` を見る入口側で掛けるのが安全。

## 終了フックから claude を呼ぶと無限ループになる

Claude Code には、セッションの開始や終了といった節目で自動的にスクリプトを走らせる「フック」というしくみがあります。  
このうち `SessionEnd` フックは、セッションが終了するときに走ります。  
`SessionEnd` は、`/clear`・セッションの再開・ログアウトなど、セッションが終わるさまざまな場面で発火します。  
このことは Anthropic Docs > Hooks reference にまとまっています。  
ここで終了フックの中に `claude` そのものを起動する処理を書くと、起動された `claude` はそれ自体が新しいセッションになります。  
その新しいセッションも、終わるときにまた `SessionEnd` を呼びます。  
こうして「フックが `claude` を呼び、その `claude` の終了がまた同じフックを呼ぶ」連鎖が止まらなくなり、暴走が始まります。  
フックは入力で `session_id` を受け取ります。  
発火するたびにこの値が違っていれば、別々のセッションが次々に生まれている再帰の証拠だと読み取れます。

実際にあった私のやらかし事故を紹介します。  
私は終了フックの中に、セッションの振り返りを忘れたときだけ `claude` を裏で起動して後追いの記録を作る、という設定を入れていました。  
意図していなかった再帰の始まりです。  
ログを見ると、発火のたびに `session_id` が違っていて、まさにこれが起きていると確認できました。  
普段は一日に数回しか発火しないフックが、その日は数百回発火し、数分でセッションの5時間ごとの利用上限に達してしまいました。  
私は再帰を止めるためのガードを一つだけ入れていたつもりでした。  
ところが、そのガードが頼っていた処理を、別の作業のついでに私自身が消してしまっていて、ガードは一度も発動しませんでした。  
終了フックは同期で動き、Claude Code はその処理が終わるまで待ちます。  
だから本来すぐ終わる `/clear` が、長いときは二分ほど固まり続けました。  
「ん?遅くね?」と思ったときに、内部で何が起こっているのか調べるべきでした。

## 暴走しない終了系フックの設計

終了系フックには、無限ループとは別にもう一つの罠があります。  
フックはデフォルトで同期で動き、Claude Code は処理が終わるまで待ちます。  
そのため終了フックに重い処理や別プロセスの起動を書くと、`/clear` やセッションの終了のたびに、完了まで固まってしまいます。  
これを避けるには、フックを背景で動かす `async` 設定を使う方法があります。  
あるいは、フックでは記録を書き残すだけにとどめ、重い処理は別のプロセスに切り出します。

ガードの掛け方も大切です。  
「処理が成功すれば目印のファイルが残る」という出口側のガードは、その目印を作る手順が変わったり消えたりすると、ガードごと効かなくなります。  
代わりに、入口で止めるのが安全です。  
たとえばフックが `claude` を起動するときに、目印となる環境変数を一緒に渡しておきます。  
入口でその環境変数があれば「自分の中から自分が起動された」と判断し、何もしないようにします。  
こうすれば再帰そのものを止められます。  
ポリシーを強制したいフックは `exit code 2` で止めます。  
ほとんどのフックでは `exit code 2` だけがその処理をブロックします。  
それ以外の終了コードでは Claude Code は処理を続けてしまうので注意が必要です。  
危険なコマンドを実行する直前で止めたいなら、ツールの実行前に走る `PreToolUse` フックを使い、deny を返して関所のように止める方法もあります。

フックを足す前には、次の点を確認しておくと安全です。

* どのイベントで何回発火するか
* 同期か非同期か、最大でどれくらいブロックするか
* 中で起動するものが、同じフックをもう一度呼び出さないか
* ガードは入口にあるか、それとも出口の副産物頼みか
* 暴走したときの被害の上限と、止める手段が用意されているか
