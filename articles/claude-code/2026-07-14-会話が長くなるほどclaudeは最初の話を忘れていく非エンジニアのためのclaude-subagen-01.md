## はじめに

こんにちは。株式会社MOCHITECでCTOをしている上田（@ryo\_u27）です。

Claudeとの会話が長くなると、最初のほうに伝えたことをだんだん取りこぼす。そんな覚えはないでしょうか。

原因はシンプルです。Claudeが一度に見ておける量には、限りがあります。1つの会話に調べ物や資料をため込むほど、注意が薄まり、前の話が抜けていきます。

そこで、重い作業だけ別のClaudeに預けて、結論だけ受け取る。それが「SubAgent（サブエージェント）」です。前回の[Skillの記事](https://zenn.dev/mochitec_tech/articles/b2014086b31acf)の続編として、今回はこれを紹介します。

※SubAgentはCoworkかClaude Codeの機能です。

## SubAgentは、もう一人のClaude

一言でいうと、SubAgentは **別の頭で考えてくれる、もう一人のClaude** です。

あなたが話す相手は、メインのClaude。重い作業が来たら、それをもう一人のClaudeにまるごと渡します。

その担当は、自分の頭のなかだけで作業を進めます。調べる・下書きする・直す、といった手を動かす部分は、ぜんぶその担当のなか。あなたに返ってくるのは、結論だけです。だからメインの頭はふさがらず、前の話も取りこぼしません。

この「頭」が、さきほどの「一度に見ておける量」、少し難しい言葉で「コンテキスト」です。もう一人のClaudeは、これを別に持っています。

## なぜ使うのか

もう一人のClaudeに任せると、うれしいことが3つあります。

### 1. メインのClaudeが、本題に集中できる

重い作業を切り出せば、メインの会話は軽いまま。答えの精度が保てます。これが一番の値打ちです。

### 2. 何人にも、同時に頼める

別々のもう一人のClaudeが、同時に動きます（これを「並列」と呼びます）。10社を1社ずつ待たず、一気に。

![1本ずつ順番だと時間がかかる。もう一人ずつに分ければ、同時に進む](https://static.zenn.studio/user-upload/5d1da56a389a-20260714.png)

*図2：1本ずつ順番だと時間がかかる。もう一人ずつに分ければ、同時に進む。*

### 3. 決めたやり方を、毎回まもってくれる

担当には、仕事のやり方を先に決めて持たせられます。「競合リサーチ担当」なら「資料だけをもとに、中立の目線で調べる」といった型です。一度きめておけば、いつ呼んでも同じやり方で返ってきます。Claudeの知識が増えるわけではありません。あなたの決めた手順どおりに動くので、結果がそろいます。

## Skillと何が違う？

前回のSkillと、何が違うのか。ひとことで言うと、**Skillは「手順書」、SubAgentは「担当者」** です。

|  | Skill | SubAgent |
| --- | --- | --- |
| 正体 | やり方の手順書 | もう一人のClaude |
| すること | 同じClaudeに型を教える | 別のClaudeに作業を預ける |
| たとえ | 新人に渡すマニュアル | 別の頭で考える担当者 |
| いつ | 仕上がりの型をそろえたい | 重い作業を結論だけ受け取りたい |

2つは組み合わせられます。もう一人のClaude（SubAgent）に、やり方メモ（Skill）を持たせる。これが一番強い形です。

また、「毎回この方針で」のように覚えさせたいだけなら、Skillやプロジェクト設定（CLAUDE.md）が本命です。SubAgentは、重い作業を切り出すほうの道具です。

## どこで効く？ 割に合わない場面も

とくに効くのは「**重い・切り離せる・結論だけ欲しい**」がそろうときです。エンジニアと、それ以外で1つずつ挙げます。

### エンジニアなら、コードのレビュー

書いたコードや変更点を、もう一人のClaudeに「隅々まで見て、気になる点だけ挙げて」と任せます。返ってくるのは「バグの恐れ・分かりにくい書き方・直し方の案」をまとめたリスト。何十ファイル分を読むのは相手のなか。こちらに届くのは指摘だけです。

しかも、もう一人は前の会話を引き継ぎません。「こう書いたつもり」に引きずられない、まっさらな目です。自分では気づけない抜けが見つかります。

### エンジニア以外なら、打ち合わせの準備

取引先フォルダの議事録・見積・メールを、まるごと読ませます。返ってくるのは「進行中の案件・経緯・未対応・提案の余地」をまとめた1枚。半年分を読むのは相手のなか。こちらに届くのは会議用のまとめだけです。

どちらも、形は同じ。**重い仕事を別のClaudeに出して、結論だけ受け取る**。

逆に、割に合わない場面もあります。

| 効くとき | 割に合わないとき |
| --- | --- |
| 重い作業・大量の読み込み | 短い一問一答、すぐ終わること |
| 他と切り離して任せられる | 前の結果を見て次を決める作業 |
| ほしいのは結論だけ | 経過を見ながら直したい作業 |

もう一人のClaudeを立てると、そのぶんトークン（AIの利用量）がかかります。軽い用事なら、わざわざ立てるより自分で聞くほうが早いこともあります。迷ったら、上の左の3つがそろうかで判断してください。

ここに挙げたのは一例です。調べ物にかぎらず、資料の整理・下書き・チェックなど、「重くてまとまった仕事」ならいろいろ任せられます。

## 実際に任せてみる

難しそうに見えて、やさしいほうから始められます。

### 1. まずは、頼むだけ

メインの会話で、こう頼むだけです。

```
生成AIの主要サービスを代表的なものから5つ選んで、
それぞれ調べて「サービス名 / 特徴 / 気になる点」の表にまとめて。
どれを選ぶかも含めて、並列でSubAgentに任せて、結果の表だけ見せて。
```

あとは、Claudeにおまかせ。重いと判断すれば、複数の担当に分けて並列で動かし、結果だけ返します。分けるかどうかもClaudeが決めるので、名指しは要りません。この自動の振り分けは、CoworkとClaude Codeで同じです。分かれたときは、進行状況の画面に担当がいくつも並びます。

実際に送ると、こんなふうに動きました。

![Coworkで、5つのSubAgentが並列に動いて調べているところ](https://static.zenn.studio/user-upload/f1c14b9d1452-20260714.png)  
*図3：実際の実行画面。5つの担当が並列で「ウェブを検索しています」。これが「同時に頼めている」証拠です。*

### 2. よく使う役割は「担当」に登録する

同じ役割を何度も頼むなら、「担当」として登録しておけます（少しだけ準備の要る、一歩進んだ使い方です）。やり方は、次の文をそのまま送るだけ。コードは書きません。

```
「競合リサーチ担当」という担当（サブエージェント）を作って。
・できるのは、資料の読み取りと検索だけ（書き換えはしない）
・会社名を渡したら「会社名 / 主力サービス / 気になる点」の表にして返す
設定はあなた（Claude）が書いてください。
```

すると、Claudeが設定ファイルを書いてくれます。中身はコードでなく、こんな短いメモです。

```
name: 競合リサーチ担当
役割: 指定された会社を調べ、要点を表にして返す
使える道具: 資料の読み取りと検索だけ（書き換えはしない）
```

あとは、この設定を担当フォルダ（`.claude/agents/`）に置けば完成です。Coworkならフォルダを接続すればClaudeがそのまま置いてくれます。次からは名前で呼び出せて、中身を開いて直せます。

## 任せる前に、ひとつだけ

!

任せる前に、2つだけ気をつけてください。

* **まずは少しから**。同時に何人も頼めますが、そのぶんトークン（AIの利用量）を多く使います（5人なら約5倍）。1つか2つで試しましょう。
* **信頼できるものから**。他人が作ったSubAgentは、公式か自作のものを。悪意ある指示が混ざることもあります。

## まとめ

SubAgentは、絶対に必要なものではありません。たいていの仕事は、メインのClaude1つでもこなせます。でも、重い仕事を切り出して任せるほど、いいことが起きます。会話は散らからず、同時に進み、任せられる範囲も決めておける。使うほどに、安心して頼れる幅が広がります。それがSubAgentです。

| 登場人物 | ひとことで | 使いどころ |
| --- | --- | --- |
| Skill | やり方の手順書 | 型を毎回そろえたい |
| SubAgent | もう一人のClaude | 重い作業を結論だけ受け取りたい |
| 合わせ技 | 担当に手順書を持たせる | 型のまま重い作業も任せたい |

短い作業や、経過を見たいことには向きません。でも「この重い仕事、毎回しんどい」が1つでもあれば、そこが任せどきです。

まずはその1つを、もう一人のClaudeに頼んでみてください。
