## ──で、自分はどうするか

今日の5本、ぶっちゃけ**2本目（PwC 3万人認定）と4本目（OpenAI DeployCo）が自分文脈に直撃**するんですよね。私自身、毎日Claude Codeを 「**ノート管理 / メモ管理**」に活用してるなんで、「3万人認定」も「FDE常駐」も**個人レベルでやってる話の企業版**として読めるんです。

* **2本目「PwC 3万人認定」に対して**: 私の「**ノート管理 / メモ管理**」法は個人版の「Claude Code 全社展開」みたいな状態で、CLAUDE.mdに「セッション冒頭で多階層メモリを読む」を絶対ルールとして書いて、/kpi-collect /re-daily /sync-reminders のような25本超のスキルで毎日の運用を自動化してます。PwCがやってる3万人認定は要するに「**型化と再現性**」の話で、ここは個人運用でも全く同じ。**昨日の インプレッション数 29,289 + フォロワー6日累計+35人で 300到達**したのも、リプ運用の型を CLAUDE.md にコピペ可能な形で残してきた結果なので、PwC の動きは「やっと企業側が個人運用に追いついてきた」感覚で読んでます
* **4本目「OpenAI DeployCo 40億ドル」に対して**: 「PoC止まりの企業AI」を商業モデルで解決しに行く動きですが、個人運用でも全く同じ詰まりがあります。私の場合、/kpi-collect で**手動スクショ10-15分 → MCP経由JS抽出30秒（20-30倍高速化）**にしたのが効いてて、要は「**PoC（手動運用）→ 本番（自動化）**」をスキル単位で進めてるんですよね。DeployCo の Forward Deployed Engineer は「常駐して業務ごと AI化」する人なんで、**個人版なら自分が自分の FDE になるしかない**。今週は /sync-reminders バグ修正＋Apple Reminders 161件全削除みたいな保守作業も走らせたんで、これも一種の FDE 仕事です
* **5本目「Gemini Intelligence + Googlebooks」に対して**: 5/19の I/O 本番で Gemini 4 が出るまで判断保留。ただ「**AI搭載PC新カテゴリ**」は、今後 Mac mini / MacBook が「Claude Code 標準搭載」に進む可能性も視野に入れて観察します

今週は**6日累計でフォロワー +35人（先週合計+9の3.89倍）**まで来てて、ツール運用の型がデータで証明された週。今日の5本はそのトリガーじゃなくて、**「企業も同じ方向に動き出した」**という追い風として読みます。来週からはテンプレ運用フェーズに入ります。

## 1. VerizonがProject Glasswingに参加、通信インフラへAI脆弱性診断

### ClaudeがVerizonの通信網を診断する。27年前のOpenBSD脆弱性をAIが見つける時代へ

### 要約

Anthropicは5月15日、通信大手VerizonがProject Glasswingに参加したと発表しました。**Verizonは参加19社の中で唯一の通信会社**で、Claude Mythos Previewをネットワークインフラの**サイバーセキュリティ診断**に活用します。Mythos PreviewはOpenBSDの27年前の脆弱性やFFmpegの16年前の欠陥を自動発見した実績があり、**人間の専門家でも見逃してきたゼロデイを大量検出**できる点が通信事業者を引きつけました。

### 実務で何が変わるか

通信事業者がAIに自社インフラを診断させるという動きは、中小企業のシステム運用にも**1〜2年後に降りてくる**現実的な未来です。「AIに脆弱性スキャンを任せる」が標準オペレーションになると、社内エンジニアが「**コード書く時間より、AIが出した報告書を裁く時間**」のほうが長くなる可能性があります。レポートの優先度判定のフレームを先に作っておくと、AIが出してきた数百件の指摘に振り回されずに済みます。

### Verizon × Project Glasswingのポイント

* **Project Glasswing**はクリティカルインフラのセキュリティ強化を目的にした招待制コンソーシアム（4月7日発表）
* Verizonは参加19社（AWS、Apple、Cisco、Google、Microsoftら）の中で**唯一の通信会社**
* Claude Mythos Previewは**OpenBSD 27年前の脆弱性とFFmpeg 16年前の欠陥**を自動発見した実績あり
* Mythos Previewは**$25/$125（入出力100万トークン）**でAPIアクセス可能（Project Glasswing参加企業のみ）
* AI主導のゼロデイ発見は、ペネトレーションテストやセキュリティ監査の工数・頻度を根本から変えうる

## 2. PwCがClaude Codeを全社展開・3万人認定 ⭐

### コンサル最大手が「AIを提案する側」から「AIを動かす側」に変わった日

### 要約

### 実務で何が変わるか

コンサルが「AIの使い方を教える側」から「**AIを自分で動かす側**」に変わると、クライアント企業に降ってくる成果物の質と速度が一気に変わります。中小企業の経営者から見ると、「**コンサルに頼んだら3日で出てくるはずだった資料が、AIで30分で出る**」状態が日常になる。逆に言えば、コンサルが**AIを使えない側**にいるなら、その仕事は1〜2年で価格破壊されます。

今すぐやることチェックリスト

* 自社が外部コンサルに頼んでいる業務（資料作成・分析レポート・市場調査）を**棚卸し**

* 来期のコンサル発注計画を「**AI併用前提**」で再設計

### PwC × Anthropic 提携拡大のポイント

* PwCは**Claude CodeとAIエージェントを技術開発・M&Aデューデリ・財務・HR・サプライチェーン**全域に展開

## 3. 米国に1,208本のAI法案、評価基準ゼロ問題

### 法案が乱立するほどAIは安全になるか。米国1,208本の現実をFortune分析

### 要約

Fortuneは5月15日、米国のAI規制が「**1,208本もの法案が乱立しながら、まともな評価基準がひとつもない**」状態であると指摘する論考を公開しました（Sonnenfeld・Marcus・Henriques執筆）。2025年だけで全米50州にAI関連法案が1,208本提出され**145本が成立**した一方、連邦政府はトランプ政権が州法への対抗姿勢を見せ混乱が深まっています。**法案の多さと規制の実効性は全く別問題**であることを明確に示した論考として反響を呼びました。

### 実務で何が変わるか

日本の中小企業が米国向けにAIを使ったサービスを出す場合、「**どの州で展開するか**」でコンプライアンスコストが変わる状況に入りました。コネチカット州SB5、ニューヨーク州RAISE法、テキサス州TRAIGAなど州独自法が乱立しているので、**AI関連の海外展開は「州単位で評価する」運用**を先に作っておくと、後で焦らずに済みます。日本国内の規制も近い将来同じ展開になる可能性があり、**規制ウォッチを月1回ルーチン化**するのがおすすめ。

### 米国AI規制1,208本のポイント

* 2025年に全米50州で提出されたAI関連法案は**1,208本、うち145本が成立**
* **評価基準の欠落**: 安全性・説明責任・適応性・執行力のいずれも標準化されていない
* トランプ政権12月11日大統領令は州AI法に対抗する姿勢を示したが、NDAは翌日に**先取条項を排除**
* **コネチカット州SB5、ニューヨーク州RAISE法、テキサス州TRAIGA**など州独自法が続々前進
* 企業にとっては「**どの州でサービスを展開するか**」でAIコンプライアンスコストが大きく変わる

## 4. OpenAIが企業AI展開専門会社DeployCo設立 ⭐

### OpenAIが「AI導入を手伝う会社」を作った。40億ドル・19社連合で「PoC止まり」を解決

### 要約

OpenAIは5月11日、企業のAI導入・展開を専門支援する「**OpenAI Deployment Company（DeployCo）**」の設立を発表しました。TPGが主導し**Bain Capital、Brookfield、Goldman Sachs、SoftBank、McKinsey、Capgemini**など19社が参画、初期投資**40億ドル超**。約**150名のForward Deployed Engineer**をクライアントに常駐させ、PoC段階で止まりがちな企業AIを「**本番稼働する業務インフラ**」に変えることを目的とします。AI研究から商業実装への橋渡しが、独立した事業として明確化されました。

### 実務で何が変わるか

「**ChatGPTライセンスを買ったが、業務にハマらず止まっている**」企業がDeployCoの主戦場です。中小企業レベルでは40億ドルの恩恵は直接ないですが、**「常駐エンジニアがいないと AI は本番稼働しない」**という商業的な認識が確立されると、社内に1人「**AI 担当**」を置く動きが日本でも加速します。すでに「**AIに詳しい総務**」みたいな職種が動き始めているので、自社の中で誰がその役割になるか、来期の組織設計で決めておくのが良いです。

コピペ用「自社のFDE候補」棚卸しプロンプト

```
うちの会社で「Forward Deployed Engineer」のような役割を担えそうな人を棚卸ししたい。

【条件】
- AI/プログラミングへの抵抗が低い
- 業務全体を俯瞰できる立場（兼業可・コードを書く必要はない）
- 来期から月10時間以上を社内AI推進に投下できる

各部門から候補者を1-2名挙げて、その人に最初に渡すべき業務（30日以内に完了するPoC）を3つ提案して。
```

### OpenAI DeployCoのポイント

* 設立日: **2026年5月11日**。OpenAIが過半数所有、投資ファンド・コンサル・SI 19社が参画
* **Forward Deployed Engineer (FDE)** がクライアントに常駐して「業務のAI化」を末端まで実装する体制
* **Tomoro社を買収**し約150名の経験豊富なFDEを即日確保
* 企業にとっては「**ChatGPTライセンスを買う**」から「**専門チームが業務ごと再設計してくれる**」に変わる
* AI導入の詰まり（PoC止まり、担当者不足、業務統合の難しさ）を**商業モデルで解決**しにいく初例

## 5. Google I/O直前: Gemini IntelligenceとGooglebooks発表

### GoogleがAI搭載PCの新カテゴリを作った。I/O本番は5月19日、何が来るか

### 要約

Googleは5月12日のAndroid Show（I/O直前特番）で、Androidに深く統合された新AI機能「**Gemini Intelligence**」と、Geminiを中核に設計した**新カテゴリのラップトップ「Googlebooks」**を発表しました。Googlebooks（AcerやASUS、Lenovo製）は**今秋発売予定**で、Androidとのシームレス連携や**Magic Pointer**、**AI Widget**などの機能を持ちます。本番のGoogle I/O 2026は**5月19〜20日**に開催予定で、**Gemini 4やAndroid 17**のさらなる発表が見込まれています。

### 実務で何が変わるか

**「AI搭載PC」が PCの新カテゴリとして確立**すると、中小企業の社内PC更新時に「Mac / Windows / Googlebooks のどれを選ぶか」という選択肢が増えます。**Googlebooksは Android アプリの資産が活きる**ため、Androidスマホ運用が標準の業界（営業・現場系）にとっては、PC・スマホ両方をGoogle側に寄せる動きが加速する可能性。**今秋までに各社のレビューを集めておく**と、来期の社内PC更新計画に役立ちます。

### Google I/O直前発表のポイント

* **Gemini Intelligence**は各デバイス上で**アプリ横断的に動くエージェント型AIアシスタント**機能
* **Googlebooks**: Gemini搭載を前提に設計された**新カテゴリラップトップ**。今秋発売
* **Android 17、Android XR**（ARグラス）のプレビューもI/O 2026（5月19〜20日）で予定

* Google I/O本番は**5月19日10時PT開始**。**Gemini 4モデル**の詳細が明らかになる予定
