私、しばらく気づかないフリをしていたことがあります。

Claudeに何か頼むたびに、毎回おなじ前置きを打ち直していたんです。  
  
**「あなたは私のアシスタントで、口調はこうで、出力はこの形式で、専門用語は使わないで」。**

それを、来る日も来る日も、コピペしたり打ち直したり。

ある日ふと、自分が何回これを打ってきたかを数えそうになって、やめました。

こわかったので。

これ、私だけじゃないと思います。

AIは便利になったのに、「便利に使うための前置き」を毎回人間が手で運んでいる。賢い道具を、わざわざ不便に使わされている状態です。  
  
煽るわけじゃなく、これは普通に時間と気力の漏れです。

で、ここからが本題。

その「毎回打つ前置き」は、Claudeに一度覚えさせて、名前で呼び出せます。これを**スキル**と言います。

むずかしく考えないでください。

スキルというのは、要するに**「決まった作業のショートカット」**です。

よく使う指示をテキスト1枚にまとめて置いておくと、次からは長い説明をぜんぶ省いて、ひとことで呼べるようになる。

それだけです。

しかも、ここが大事なところなんですが——**作るのにプログラミングはいりません。**

スキルの正体は、ただのテキストファイルです。

「これはこういう作業をするもの」という説明と、「こう動いてほしい」という中身を書くだけ。コードは1行も出てきません。

書くのが面倒なら、その作文すらClaude本人にやらせられます（やり方は後で書きます）。

なのに、なぜみんな作らないのか。理由はシンプルで、**「自分が毎回打っているアレが、そのまま仕組みにできる」と誰も教えてくれないから**です。

機能の名前は聞いたことがあっても、「あなたのあの手間が消える」という話とつながっていない。

この記事では、そこをつなぎます。

ここまでで、あなたはもう「スキル＝自分の手間を1回で固定できるショートカット」「作るのにコードはいらない」という全体像を持ちました。

ここから先は、**実際に手を動かして、今日あなたのスキルを1個完成させる**パートです。

私が実際に使っている穴埋めテンプレ（コピペして埋めるだけ）と、完成例、そして初心者が必ず引っかかる3つのつまずきと回避をぜんぶ置いておきます。

読み終わったときには、もう「毎回おなじ説明」をしなくて済む体になっています。

  

## 本編後半（実践：ここから手を動かす）

**全体の流れ（これだけ）**  
スキル作りは、たった4ステップです。

1. **何を省きたいか決める**（毎回打っているアレを1つ選ぶ）
2. **中身を書く**（穴埋めテンプレを埋める／面倒ならClaudeに書かせる）
3. **置く**（決められた場所にテキストを1枚置く）
4. **呼ぶ**（/名前 で呼び出す）

順番にいきます。

### ステップ1：何を省きたいか、1つだけ決める

最初から欲張らないでください。「あれもこれも自動化」は挫折の入口です。 **あなたが今週いちばん多く打った指示**を1つ思い出してください。それが最初のスキルです。  
例：

* 毎回「私の口調（やわらかめ・敬語ひかえめ）でメールの下書きにして」と打っている
* 毎回「この文章、専門用語を中学生にわかる言葉に直して」と打っている
* 毎回「この箇条書きを、お客様向けの丁寧な文章にして」と打っている

どれでもいいです。  
1つ選んでください。  
これを仮に「メール下書き」スキルとします。

### ステップ2：穴埋めテンプレを埋める

スキルの中身（ファイル名は SKILL.md）は、この形です。**コピペして、かっこの中だけ埋めてください。**

```
---
name: mail-draft
description: 〔いつ使うかを一言で。例：メールの下書きをあさみの口調で作るとき〕
---

# 〔スキルのタイトル。例：メール下書き（私の口調）〕

## やること
〔Claudeに毎回させたいことを、普通の言葉で。例：要件を渡すと、やわらかめ・敬語ひかえめのメール下書きにする〕

## 守ってほしいこと
- 〔ルール1。例：絵文字は使わない〕
- 〔ルール2。例：本文は300字以内〕
- 〔ルール3。例：最後に「ご確認ください」で締める〕

## 例
入力：〔よくある依頼の例〕
出力：〔こう返してほしい、という見本〕
```

ポイントは2つだけ。

書くのが面倒なら、**この作文ごとClaudeにやらせられます。** Claudeにこう言ってください：

> 「メール下書きというスキルを作りたい。私が毎回頼んでいるのは〔さっき選んだ内容〕。上のSKILL.mdの形で、説明文と中身を書いて」

入力例

これで下書きが出てきます。中身を読んで、違うところだけ直せばOKです。 （※対話で作ってくれる skill-creator という補助機能を使う手もあります。）

### ステップ3：置く

できたテキストを、スキル置き場に1枚置きます。置き場所は、あなたの使っているClaudeによって少し違います。

ここがいちばん「？」になりやすいので、迷ったらClaudeに

**「作ったこのスキル、どこに置けばいい？私は〔Claude Code／アプリ〕を使ってる」**

と聞いてください。

あなたの環境に合わせて手順を出してくれます。

### ステップ4：呼ぶ

置けたら、/メール下書き のように名前で呼ぶか、ふつうに「メール下書きして、要件はこれ」と話しかけるだけ。  
もう前置きはいりません。

**一度だけ書いた説明を、Claudeがずっと覚えています。**

## 初心者が必ず引っかかる3ポイント

![](https://assets.st-note.com/img/1782260896-Usd0j7DCIM6FAwkeRZ2orlpE.png?width=1200)

私もここで転びました。  
あなたが転ぶ前に置いておきます。

1. **置き場所を間違える** → スキルは「決まったフォルダ」に入っていないと認識されません。デスクトップに置いても動きません。  
   **ステップ3の場所に置くこと**。  
   動かないと思ったら、まずここを疑う。
2. **description（説明文）が雑** → ここが「いつ使うか」になっていないと、Claudeはスキルがあることに気づかず、結局あなたが毎回手で呼ぶことになります。  
   「〜するとき」「〜したい場合」  
   という形で書くと発火率が上がります。  
   ここだけは手を抜かないでください。
3. **最初から完璧を狙う** → 1個目は雑でいいです。  
     
   使いながら「ここ毎回直してるな」という部分をSKILL.mdに足していく。スキルは育てるものです。  
   完璧な1枚を作ろうとして作らないより、雑な1枚を今日置くほうが100倍効果ある。

なんか、簡単そうかも？と思ったら今すぐトライしてみましょ🎵

## まとめ＋次の一手

スキルは、難しい機能ではありません。  
  
**「あなたが毎回打っている前置きを、1回だけ書いて固定する」**

——ただそれだけのことです。

コードもいらない。今日1個でいい。  
次の一手は決まっています。  
**さっき選んだ「毎回打っている指示」を、いま1個だけスキルにする。**

それが終わったら、来週いちばん多く打った指示で2個目を作る。

気づいたら、あなたはもう同じ説明を二度としなくなっています。  
道具に合わせて自分が動くのは、もう終わりにしましょう。
