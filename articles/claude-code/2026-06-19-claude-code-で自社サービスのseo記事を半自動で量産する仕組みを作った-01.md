個人開発で AI スキル・プロンプトの売買マーケット [equaliA（イコリア）](https://equalia.jp) を運営しています。ローンチ初期の最大の課題は「集客」でした。広告予算は限られ、SNS の手運用も続かない。そこで **Claude Code をマーケティングの実行エンジンにして、SEO 記事の執筆からデプロイ・検索エンジンへの登録までを半自動で回す仕組み**を作りました。この記事ではその構成と、実装で気をつけた点を共有します。

## 全体像

やりたかったのはシンプルで、「ネタを渡したら、記事が公開されて検索インデックスに載るところまで通す」こと。Claude Code の **スキル（Skill）機能**で 1 本のワークフローにまとめました。

1. キーワード/テーマを受け取る
2. 記事本文（Markdown）を執筆
3. 記事メタデータを型付き設定に登録
4. ビルドして push（Vercel が自動デプロイ）
5. 本番 URL をポーリングして公開を確認
6. Google Search Console にサイトマップ再送 + URL 検査

## 「全自動」にしない設計

最初は全部を無人で回そうとしましたが、**外向きのアクション（検索エンジンへのインデックス登録リクエストや記事の公開フラグなど）は人が最後に確認して操作する**形に落ち着きました。

* 自動化部分：記事生成・ビルド・デプロイ・サイトマップ送信
* 人手ゲート：インデックス登録リクエストの実行、記事公開フラグの確定

理由は、API 規約・スパム判定・誤公開のリスク管理です。文面と対象は機械が用意し、最後のトリガーだけ人が引く。この「下書きをキューに積む」設計にしてから、運用がぐっと安定しました。

## コンテンツ設計（カニバリ回避）

量産すると必ず起きるのが、自サイト内で記事同士が検索順位を食い合う **キーワードカニバリゼーション**です。対策として、

* ビッグワード単体（「AI」「プロンプト」）は狙わず **複合ワードを面で取る**
* 記事 → 関連ランディングページ → 商品一覧、への **内部リンク導線**を全記事に持たせる
* 既存記事の薄い部分は新規量産より **加筆で深掘り**を優先

Claude Code はコードを書くだけのツールではなく、「定型の運用フローを実行するエンジン」として使うとかなり強力でした。同じ発想は SEO に限らず、定期レポート・データ整形・お知らせ配信などにも応用できます。
