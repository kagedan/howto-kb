AIで自動化したい人ほど、最初にプロンプトを書き始めます。

でも、本当に先にやるべきことはプロンプト作成ではありません。仕事の分類です。

Claude CodeやCodexのようなAIエージェントは、単発の作業代行にも使えます。ただし、それだけだと毎回同じ説明を繰り返します。少し便利なチャットで止まります。

自動化を本気で進めるなら、仕事を次の6つに分けます。

![](https://assets.st-note.com/img/1780962642-IgGnvLirXHsecmfAOQ512UVT.png?width=1200)

![](https://assets.st-note.com/img/1780962663-epfwozXFbOalv9UMhQ78uWtP.png?width=1200)

この記事では、この分類を「**AI自動化OS**」と呼びます。

ただ、重要なのは名前そのものではありません。

仕事をどの粒度でAIに渡すかです。

## AI自動化はプロンプトではなくOS設計で決まる

AI活用が伸びない原因は、プロンプト不足ではありません。

![](https://assets.st-note.com/img/1780962682-0f3XFdKhQeq9cLb1rlCTURgH.png?width=1200)

多くの場合、仕事の渡し方が曖昧です。

たとえば、次の依頼は全部違います。

* **毎日同じCSVを整形してほしい**
* **note記事を書く時の判断基準を毎回守ってほしい**
* **ブラウザを開いて競合LPを調べてほしい**
* **セミナー企画から告知文まで全部作ってほしい**
* **ファイル編集後に必ず検証してほしい**

これを全部「**AIにお願いする**」でまとめると、運用が崩れます。

本来は、仕事の種類ごとに置き場所を変えます。

> 同じ作業なら `loop`。判断基準なら `skill`。専門担当なら `agent`。成果達成なら `goal`。強制実行なら `hook`。外部接続なら `MCP`。

この分け方があるだけで、AI活用はかなり安定します。

## まず仕事を6種類に分ける

![](https://assets.st-note.com/img/1780962747-wFmuU4fYrAJzPGZbVQoeR05d.png?width=1200)

最初に作るべきものは、プロンプト集ではありません。

業務分類表です。

以下の表をそのまま使ってください。

![](https://assets.st-note.com/img/1780962848-3kacRYEDWxQZiTHXVtwSbL48.png?width=1200)

ここで大事なのは、いきなり全部を自動化しないことです。

まず10個だけ業務を書き出します。

例です。

* **AIニュース収集**
* **X投稿案作成**
* **note記事作成**
* **サムネイルプロンプト作成**
* **競合LP調査**
* **Slack返信下書き**
* **請求書作成**
* **画像ファイル整理**
* **アプリ修正**
* **公開前チェック**

この10個を6分類に振り分けます。

振り分けできない仕事は、まだ自動化対象として荒いです。作業単位を小さくします。

## loopは同じ作業を回す

![](https://assets.st-note.com/img/1780962765-3JA6Vkl0X1Q4anFzRxu9PCjp.png?width=1200)

`loop` は、同じ入力、同じ手順、同じ完了条件で回る作業に使います。

人間が毎回判断しなくてよい仕事です。

向いている作業はこれです。

* **毎朝のニュース収集**
* **CSVの列整形**
* **note一覧のURL同期**
* **画像のリサイズ**
* **リンク切れチェック**
* **定期レポート作成**

`loop` 化の判断基準は簡単です。

> **「前回と同じ手順でいいか」**

答えがYESなら、AIに毎回考えさせるより、スクリプト化した方が速いです。

Claude Codeに任せる場合も、毎回長い指示を出す必要はありません。

以下のように設計します。

```
目的:
- 毎朝、AIニュース候補を収集する

入力:
- 指定RSS
- 既存CSV
- 重複除外リスト

処理:
- 新着取得
- 重複除外
- 重要度分類
- CSV追記

完了条件:
- CSVに候補10件
- URL重複なし
- 出典リンクあり
```

この形になれば、あとはClaude Codeにスクリプト化させます。

`loop` はAI自動化の入口です。ただし、ここだけで止まると単なる作業削減です。

## skillは判断基準を固定する

![](https://assets.st-note.com/img/1780962791-uWeFqAgxBXJm3yl9nM8tVGOS.png?width=1200)

AI自動化で一番重要なのは `skill` です。

理由は、仕事の品質を決めるのは実行速度ではなく判断基準だからです。

たとえば、note記事を書く時に毎回こう説明しているなら、skill化すべきです。

* **誰向けに書くか**
* **冒頭で何を約束するか**
* **タイトルで何を避けるか**
* **出典確認をどうするか**
* **CTAをどこへ置くか**
* **保存先をどう決めるか**

これは単なるプロンプトではありません。

AIに渡す「**仕事のルール**」です。

`skill` 化すると、毎回の依頼は短くなります。

```
AI自動化OSをテーマにnoteを書いて。
copy_note skillを使って。
```

この短い依頼でも、裏側でルール、保存先、構成、チェックが動きます。

つまり `skill` は判断自動化です。

`loop` が手を動かす自動化なら、`skill` は頭の使い方を固定する自動化です。

ここを作らないままAI活用を増やすと、毎回品質が変わります。

## agentは専門担当を分ける

![](https://assets.st-note.com/img/1780962874-KqlZ3RGNurEyDhcStH1Oxn4k.png?width=1200)

`agent` は、作業を専門担当に分ける時に使います。

Claude Codeの文脈では、subagentは別コンテキストで特定タスクを担当できます。

向いている作業はこれです。

* **コードレビュー担当**
* **調査担当**
* **テスト担当**
* **ブラウザ操作担当**
* **セキュリティ確認担当**
* **UI品質確認担当**

ポイントは「別人格」ではなく「**別責任**」です。

悪い分け方はこれです。

* **すごいAI担当**
* **なんでも調べる担当**
* **雰囲気でレビューする担当**

良い分け方はこれです。

* **変更差分だけを読み、バグと未検証リスクを指摘する担当**
* **公式ドキュメントだけを確認し、仕様変更を要約する担当**
* **ブラウザでUIを操作し、再現手順つきで問題を報告する担当**

agentは、責任範囲が狭いほど強くなります。

人間の組織と同じです。全員が何でも屋だと品質が落ちます。

AI自動化でも、専門担当を分けた方が安定します。

## goalは成果まで束ねる

`goal` は、複数作業を束ねて成果まで進める単位です。

これは `loop` より大きいです。

たとえば、以下はgoalです。

* セミナーを企画し、告知文、スライド構成、LP導線まで作る
* note記事を作り、X投稿、サムネイル、公開前チェックまで進める
* アプリの不具合を調査し、修正し、テストし、リリース準備する

goalには、必ず完了条件が必要です。

```
目的:
- AI自動化OSの記事を公開準備状態にする

成果物:
- note下書き
- X投稿
- サムネイルプロンプト
- 公開前チェック結果

完了条件:
- note本文が保存済み
- X投稿が保存済み
- 出典が確認済み
- CTAが入っている
```

goalは「全部いい感じにやって」ではありません。

成果物、順序、検証、完了条件を持った仕事の束です。

ここまで設計すると、AIはかなり自律的に動けます。

## 最初に作るべきAI自動化OSテンプレ

![](https://assets.st-note.com/img/1780962917-WpBZmwQ0JeIaxT1GzPV3Ycyf.png?width=1200)

最後に、**最初の1枚として使えるテンプレ**を置きます。

まずこれを埋めてください。

```
# AI自動化OS 設計テンプレ

## 1. 自動化したい業務
- 

## 2. 業務分類
- loop:
- skill:
- agent:
- goal:
- hook:
- MCP:

## 3. 入力
- ファイル:
- URL:
- 既存ルール:
- 参考資料:

## 4. 判断基準
- 成功条件:
- 禁止事項:
- 品質基準:
- 出典確認:

## 5. 成果物
- 

## 6. 完了条件
- 

## 7. 次に自動化する作業
-
```

AI自動化を極める最短ルートは、いきなり複雑なエージェントを作ることではありません。

まず、自分の仕事を分類することです。

* **`loop` = 作業自動化**
* **`skill` = 判断自動化**
* **`agent` = 専門分業**
* **`goal` = 成果自動化**
* **`hook` = 強制実行と安全制御**
* **`MCP` = 外部接続**

この6つに分けるだけで、AIに任せる仕事と、人間が設計すべき仕事が見えます。

Claude Codeを本気で使うなら、プロンプトを増やすより先に、自分のAI自動化OSを作るべきです。
