コードは書く。ChatGPTもGitHub Copilotも使う。でも「**Claude Code（クロードコード）**」はまだ——そんなエンジニアは多いはずです。

Claude Codeは、ただのコード補完ではありません。**日本語で指示すれば、自分でファイルを読み書きし、コマンドを実行し、gitまで操作してくれる「コーディング・エージェント」**。コピペの往復は、もう要らなくなります。

この記事では、「普段コードは書くけれど、Claude Codeはまだ」という方に向けて、**何が違うのか → インストール → ミニアプリを1本つくる → 実務での使いどころ**までを、コピペできるコマンド付きで一気に解説します。読み終わるころには、自分のリポジトリでClaude Codeを動かし始められるはずです。

## Claude Codeとは？ ChatGPT・コード補完と何が違うのか

![](https://assets.st-note.com/img/1781507867-VdnZGW5kIKD1oBLta8TfFslC.png?width=1200)

チャットは相談相手、Claude Codeは実行者。ターミナルの中で動くコーディング・エージェント

ひとことで言えば、\*\*ターミナルの中で動く「コーディング・エージェント」\*\*です。日本語で指示すれば、プロジェクトのファイルを読み書きし、コマンドを実行し、gitも操作します。

普段の「Claudeチャット」との違いはシンプルです。

最近よく聞く「**AIエージェント**」——自分で計画を立て、手を動かし、頼まれた仕事を最後までやり切るAI——の、**開発特化版**だと思ってください。

## Copilot・Cursorとの違い（どこで効くか）

![](https://assets.st-note.com/img/1781507988-hfXuA4Z36m0UiPIdweVgKvx5.png?width=1200)

Copilot・Cursorはエディタの中、Claude Codeはターミナルでリポジトリ全体を横断。役割で使い分け

エンジニアなら「Copilot や Cursor と、結局何が違うの？」が一番気になるところでしょう。役割で整理すると分かりやすくなります。

最近はどのツールもエージェント機能を持ち始めていて、境界は重なってきました。だから「どれが一番」ではなく、**補完でサッと速くするCopilot系と、タスクごと任せるClaude Code を、役割で使い分ける**のが現実的です。

## 始め方｜インストールと初回起動（数分）

![](https://assets.st-note.com/img/1781509453-bDnaLFsjZg4SydlvTBCHxOwN.png?width=1200)

インストールは1行、起動は claude だけ。数分で初回起動まで

必要なのは、**ターミナルと、Claudeの有料プラン（Pro / Max など）のアカウント**だけ。

### インストール（OS別・1行）

公式が推奨するのは「**ネイティブインストーラー**」です。お使いのOSに合わせて、ターミナルで次の1行を実行します。

**macOS / Linux / WSL：**

```
curl -fsSL https://claude.ai/install.sh | bash
```

**Windows（PowerShell）：**

```
irm https://claude.ai/install.ps1 | iex
```

ポイントは2つ。

1. **以前は Node.js が前提でしたが、今はネイティブインストーラーが推奨になり、Node.js は不要**になりました（旧来の `npm install -g @anthropic-ai/claude-code` はレガシー扱い）。
2. **ネイティブ版は自動でアップデート**してくれるので、入れたら基本ほったらかしでOK。

Homebrew や WinGet を使い慣れている方は、こちらでも入れられます（こちらは自動更新ではないので、ときどき手動アップデートが必要です）。

```
# macOS（Homebrew）
brew install --cask claude-code
```

```
# Windows（WinGet）
winget install Anthropic.ClaudeCode
```

> Windowsでネイティブに使う場合は、[Git for Windows](https://git-scm.com/downloads/win) を入れておくと Bash ツールが使えて快適です（無ければ PowerShell で動作します）。WSL では不要です。

<https://code.claude.com/docs/ja/quickstart>

### 初回起動

インストールできたら、**使いたいプロジェクトのフォルダに移動して `claude` と打つだけ**です。

```
cd /path/to/your/project
claude
```

初回だけブラウザでログインを求められます。認証すれば、そのターミナルがそのまま“AIエージェントとの作業場”になります。VS Codeの拡張機能もありますが、まずは**いつものVS Codeのターミナルで `claude`** を実行するのがおすすめです。

### うまくいかないときは

途中でエラーが出ても慌てなくて大丈夫。そんなときこそ、さきほど“相談相手”と呼んだ\*\*普段のClaude（チャット版）\*\*の出番です。**エラーの文章をそのままコピーして貼り付けて聞けば**、たいてい原因と直し方まで教えてくれます。

## 最初の成功体験｜ミニアプリを“言葉だけ”で作る

![](https://assets.st-note.com/img/1781509586-5rCGTBMYcsk729uj0bhIwSgV.png?width=1200)

“言葉だけ”でアプリができる流れ：計画→承認→実装→ブラウザで動いた

「動いた！」を最速で体験するために、**ポモドーロ・タイマー**（25分集中して5分休む、を繰り返す定番タイマー）をゼロから作ってみましょう。

おすすめは、いきなり作らせず、まず\*\*「プランモード」で段取りを立ててもらう\*\*進め方です。プランモードは、コードを書く前に「どう作るか」の計画だけを先に出させるモード。小さなアプリでも、先に地図ができると手戻りが減ります。

`Shift+Tab` でモードを切り替え（パーミッションモードが順番に切り替わります）、プランモードにしたうえで、こう依頼します。

```
ポモドーロタイマーのWebアプリを作りたい。
25分の作業と5分の休憩を自動で切り替えて、開始・一時停止・リセット、
残り時間の大きな表示、完了したセット数のカウントも。
HTML・CSS・JavaScript の各1ファイルで、ブラウザで開くだけで動くように。
まずは計画を立てて。
```

すると Claude Code は、\*\*コードを書く前に「開発プラン」\*\*を提示してきます——どんなファイルを、どんな構成で、どの順番で作るか。中身を読んで、直したいところがあればチャットで修正を頼みます。

プランがよければ、`Shift+Tab` で通常モードに戻して「**この計画で進めて**」と承認するだけ。実装は \*\*diff（差分）\*\*で提案されるので、いきなり書き換わることはありません。**内容を確認してから反映**できます。

あとは、できあがった `index.html` をブラウザで開くだけ。スタートを押せば、ちゃんとカウントダウンが始まります。自分で書くと地味に手間のかかる“UI付きアプリ”が、「**計画 → 承認 → 実装**」の流れで数分で完成します。

## 覚えておく基本操作（最小限でOK）

操作で覚えることはわずかです。困ったら Claude Code 自身に「セッションを終了するコマンドを教えて」と聞けば教えてくれます。

* `claude` … 対話セッションを開始
* `claude -c` … 直前の会話を続きから再開（`-c` は continue）
* `claude -r` … 過去のセッションを選んで再開
* `/clear` … 会話の文脈を一度リセット
* `/help` … 使えるコマンド一覧を表示
* `exit`（または Ctrl+D）… セッションを終了
* `Shift+Tab` … パーミッションモードの切り替え（毎回確認 → 編集を自動承認 → プラン、と順番に）

慣れてきたら、編集を自動で承認するモードや、確認を求めずに自律で進めるモードも使えます。**何を許可しているかを分かったうえで**使うのが鉄則です。

## 実務で効く使い方①｜既存コードを「説明・修正・レビュー」させる

Claude Code の真価は、ゼロから作ること以上に、**“今あるコード”を扱うとき**に出ます。さきほど作ったポモドーロアプリを題材に、実務さながらに育ててみましょう。

**1. 説明させる**

数日経つと、自分の書いたコードでも中身を忘れます。そんなときは——

```
このポモドーロアプリの全体像と、タイマー処理の流れを、要点だけ説明して。
```

自分のコードでも、引き継いだ他人のコードでも、これで最速で頭に入ります。

**2. 直す・育てる**

本物のポモドーロには“長い休憩”がありますよね。

```
4セットごとに、5分の休憩のかわりに15分の「長い休憩」を入れて。
```

Claude Code が既存コードを読み、**必要な箇所だけを書き換えて**機能を足してくれます。ゼロから作り直すのではなく、“今あるもの”にちゃんと差し込んでくれるのがポイントです。

**3. レビューさせる**

仕上げは `/code-review`。いま入れた変更を、AIが“バグ・改善点”の観点で見てくれる組み込みコマンドです。AIに書かせて、AIにレビューもさせる。**ただし、最後にOKを出すのは人間**——この「読む・直す・レビュー」の往復が、ターミナルの中だけで完結します。

## 実務で効く使い方②｜[CLAUDE.md](http://CLAUDE.md)・MCP・git で“自分仕様”にする

知ると一気に“自分仕様”になる3つです。

### CLAUDE.md＝プロジェクトの約束事

![](https://assets.st-note.com/img/1781509684-d1ZyP7HVDiorbYGLaNmwg0l3.png?width=1200)

CLAUDE.mdはプロジェクトの約束事。使う言語・命名規則・やってほしくないことを /init で自動生成

`CLAUDE.md` は、使う言語やフレームワーク、命名規則、やってほしくないこと——といった**プロジェクトの約束事**を書いておくファイルです。書いておけば、毎回同じ説明をしなくても、あなたの流儀で動いてくれます。

作り方は簡単。プロジェクトで `/init` と打つだけ。Claude Code がコードベースを読んで、`CLAUDE.md` のたたき台を自動で書いてくれます。あとは自分のルールを足していくだけです。

### MCP＝外部ツールをつなぐ

![](https://assets.st-note.com/img/1781510016-VMkBYNI8PoswjfmRx5gT4tzc.png?width=1200)

MCPは外部ツールをつなぐ仕組み。最新ドキュメント・ブラウザ・データベース…つなぐほどできることが広がる

`MCP`（Model Context Protocol）は、外部のツールやデータを“つなぐ”仕組みです。最新ドキュメントの取得やブラウザ操作など、つなぐほどできることが広がります。

### git との相性

![](https://assets.st-note.com/img/1781510075-yw2kGcQsYxrN4zIXFaepqS6t.png?width=1200)

gitとの相性。意味のあるまとまりでコミットし、分かりやすいコミットメッセージまで任せられる

普段の git 操作を、そのまま言葉で頼めます。

```
変更を、意味のあるまとまりで、わかりやすいコミットメッセージを付けてコミットして。
```

ブランチ作成や、環境が整っていればプルリクエストの作成まで任せられます。もちろん、**最終承認は人間**です。

## つまずき＆安全に使うコツ

![](https://assets.st-note.com/img/1781510877-lSjfptI6sAGJyxE8MFz5bPw4.png?width=1200)

安全に使う3つのコツ：①権限は確認してから ②コスト感は小さく試す ③最終チェックは人間

便利な分、付き合い方のコツも3つ。

* **権限**：初期設定では、あなたの承認なしにファイルを書き換えることはありません。自動承認やバイパス系のモードは、**何を許可しているか分かったうえで**使いましょう。
* **コスト感**：大きなリポジトリを丸ごと読ませれば、その分の使用量がかさみます。最初は小さく試して感覚をつかむのがおすすめ。`/usage` で、いまどれくらい使ったかをプランの使用量として確認できます。
* **AIは間違える**：それっぽいけれど正しくないコードを出すこともあります。だから**最終的なチェックとレビューは必ず人間が**。「超優秀だけど、たまに勘違いする同僚」——その前提でレビュー文化を崩さないのが、いちばんうまくいきます。

## よくある質問（FAQ）

**Q. 無料で使えますか？**  
A. Claude Code 自体のツールは無料で導入できますが、利用には Claude の有料プラン（Pro / Max など）のアカウントが必要です。プラン名・上限・料金は変わることがあるので、最新は[公式の料金ページ](https://claude.com/pricing)でご確認ください。

**Q. Windowsでも使えますか？**  
A. 使えます。Windows・macOS・Linux に対応し、WSL でも動きます。Windowsネイティブでは Git for Windows を入れておくと快適です。

**Q. Node.js は必要ですか？**  
A. ネイティブインストーラー（推奨）を使うなら不要です。以前は Node.js が前提でしたが、今は1行のコマンドで完結します。

**Q. Copilot や Cursor から乗り換えるべき？**  
A. 「乗り換え」よりも「併用」が現実的です。エディタ内の補完はCopilot系、タスクごと任せたいときはClaude Code、と役割で使い分けるのがおすすめです。

**Q. 日本語で指示できますか？**  
A. できます。この記事のプロンプト例のように、日本語の自然な文章でそのまま依頼できます。
