「月次の数字は事務長が出してくれるけど、エクセルが何枚もあって全体が見えない」「経営判断のために必要な数字が、毎月違うファイルから手作業で集計されている」「来月の経営会議までに、また同じ集計を一から作り直さないといけない」——こうした声を、訪問診療クリニックの院長先生からよく耳にします。

訪問診療クリニックを経営する院長先生へ。

**経営判断の精度は、見ている数字の精度で決まります。**そして、見ている数字の精度は、毎月「同じフォーマットで」「同じスピードで」出てくるかどうかで決まる。月によって集計方法が違ったり、出てくるタイミングが遅れたりすると、判断は「勘」に戻ります。

多くのクリニックで、月次の数字管理はエクセルが何枚もある状態です。患者数・単価・重症割合はレセコンから、紹介元別の件数は連携管理ツール（CrossLog）から、医師1人あたり患者数は人事台帳と突き合わせる——情報がバラバラに散らばっている。毎月、事務長が一枚一枚集めて、また別のエクセルに転記する。これでは経営判断の前に、**集計疲れで終わってしまう**。

ここで効くのが、**Claude Codeで「月次経営ダッシュボード」を自作する仕組み**です。レセコンとCrossLog reportのCSVを読み込ませて、A4一枚で「今月の経営状態」がわかる1枚レポートを毎月自動生成する。当院でこの仕組みを動かし始めて約半年、経営会議の準備時間が10分の1になり、何より「判断のスピード」が変わりました。

この記事では、Claude Codeで月次経営ダッシュボードを自作する設計と、ダッシュボードに載せるべきKPI、そして実装の3ステップを整理します。

![](https://assets.st-note.com/img/1779497631-u4z6NCnoTgeb8pdywYs0PfUt.jpg?width=1200)

### なぜ「ダッシュボード」が必要なのか

訪問診療クリニックの経営判断は、外来クリニックや病院と違って、見るべき数字の幅が広いという特徴があります。

**第一に、患者数の動き。**総患者数の純増減、新規患者数、医師1人あたりの患者数——「規模」を測る指標。

**第二に、紹介元の動き。**ケアマネ・病院・地域包括・施設など、紹介元カテゴリ別の構成比とその月次推移。どの紹介元が止まっていて、どこから新しい紹介が来ているか。

**第三に、算定・収益の状況。**重症患者割合（2割要件）、患者1人あたりの月間点数。診療報酬の構造的な質を測る指標。

**第四に、訪問オペレーションの状況。**緊急往診件数、看取り件数、1日訪問件数——現場の負荷と機能を測る指標。

この4領域を「別々のファイルで」見ていると、それぞれは正確でも、全体像が描けません。**経営の本質は「数字と数字の関係性」を見ること**。例えば「新規患者が増えたのに単価が落ちている」なら、軽症患者の比率が上がっているか、重症割合が要件を割りそうなサイン。「総患者数は横ばいなのに緊急往診が増えた」なら、患者層が重症化しているか、夜間体制を見直すタイミング。

ダッシュボードは「関係性を見せる」ためのツールです。バラバラの数字を1枚に並べた瞬間、見えなかった因果が浮かび上がる。

### ダッシュボードに載せるべき9つのKPI

当院のダッシュボードに載せている指標は、最終的に9つに絞り込みました。**多すぎても見ない、少なすぎても判断できない**——この間のバランスです。

**指標1：総患者数（純増減）**  
その月末時点の在宅患者総数と、純増減（新規−中止・看取り・転居）。前月比・前年同月比を併記する。**純減が2か月続いたら、入口（紹介）か出口（看取り・中止）のいずれかに構造変化が起きているサイン**。データソース：レセコン。

**指標2：新規患者数**  
その月に新たに受け入れた在宅患者数。指標1の「純増減」と分けて見ることで、**「新規は来ているのに総数が減っている」＝離脱が増えている**、という診断ができる。データソース：レセコン。

**指標3：紹介元カテゴリ別構成比＋月次紹介総数**  
ケアマネ・病院連携室・地域包括・施設・その他のカテゴリ別構成比と、月次の紹介総数。直近3か月並べて表示。「先月までケアマネ比率が6割だったのに今月急に4割」を即座に検知できる。データソース：CrossLog reportを2種類組み合わせて作成。

**指標4：重症患者割合（2割要件）**  
別表第八に該当する重症患者の在宅患者全体に占める割合。**在宅医療充実体制加算等の施設基準で2割要件を満たしているかを毎月モニタリング**する。1割台に落ちそうな月は要警戒。データソース：レセコン。

**指標5：緊急往診件数**  
その月の緊急往診件数。在宅療養支援診療所の機能要件・施設基準にも直結する指標。月によるブレは大きいので、3か月移動平均で見る。データソース：レセコン。

**指標6：看取り件数**  
その月の在宅看取り件数。施設基準（在宅看取り実績）に直結すると同時に、地域内でのクリニックの機能を示す指標でもある。データソース：レセコン。

**指標7：患者単価（1人月間点数）**  
患者1人あたりの月間総点数。単価が下がっている場合、軽症患者の比率が上がっているか、算定漏れがあるかのいずれか。**重症患者割合（指標4）とセットで見ると診断精度が上がる**。データソース：レセコン。

**指標8：医師1人あたり患者数（100人以下要件）**  
常勤医師1人あたりの在宅患者数。**在宅医療充実体制加算で「医師1人当たり100人以下」の要件があり、これを超えると算定不可になる**。常勤医師数の設定値と、レセコンから出る在宅患者数を突き合わせて算出する。データソース：レセコン＋設定値（常勤医師数）。

**指標9：訪問効率（1日訪問件数）**  
1日あたりの平均訪問件数。医師・看護師ごとに算出し、訪問効率と負荷バランスを見る。極端に少ない日が続けば移動・ルート設計の問題、多すぎる日が続けば燃え尽きリスク。データソース：レセコン。

この9つを1枚にまとめて、グラフ・前月比・要件アラートと一緒に毎月出力する。これが当院の月次経営ダッシュボードです。

### Claude Codeで自作する3つのステップ

ここからが実装パートです。当院がClaude Codeで実装した手順は、大きく3ステップでした。

**ステップ1：データソースを整理し、CSVに統一する**  
最初にやるのは「何のデータを、どこから、どんな形式で取れるか」の棚卸しです。指標1・2・4・5・6・7・8・9はレセコンから月次CSVで出力できる。指標3はCrossLog reportを2種類（紹介元一覧と紹介履歴）組み合わせる。指標8の分母にあたる「常勤医師数」だけは設定値として手元のYAMLやJSONに書いておく。

まず、それぞれのデータを「同じディレクトリに、月別フォルダで保存する」というルールを決めます。「2026\_05」というフォルダの中に、receconn\_patients.csv、receconn\_kasan.csv、crosslog\_referrals.csv、crosslog\_history.csv、config.yaml（医師数）を置く——この程度のシンプルさで十分です。

Claude Codeに「このディレクトリ構造のCSVを読み込んで集計する」と依頼するだけで、**データ統合の8割は終わります**。重要なのは「人間が読みやすいファイル名」にしておくこと。AIが文脈を理解しやすくなります。

**ステップ2：Claude Codeに「集計ロジック」を書かせる**  
次に、ダッシュボードに載せる9つの指標を、それぞれどう計算するかをClaude Codeに伝えます。「receconn\_patients.csvから月末時点の在宅患者数と純増減を出して、前月比をつけて」「crosslog\_referrals.csvとhistory.csvを突き合わせて、紹介元カテゴリ別構成比を直近3か月並べて」——**日本語でそのまま依頼するだけ**です。

Pythonでpandasを使った集計スクリプトが数分でできあがります。一度作ってしまえば、翌月以降は「同じスクリプトに、新しい月のCSVを食わせるだけ」で済む。事務長は集計作業から解放され、Claude Codeが毎月同じ品質で数字を出してくれます。

ここでのコツは**「いきなり全部の指標を一気に作らない」**こと。1指標ずつ作って動作確認して、次に進む。当院では最初は3指標——総患者数・新規患者数・紹介元構成比——から始めて、半年かけて9指標まで増やしました。

**ステップ3：HTML出力で「1枚のレポート」にまとめる**  
集計結果を最終的に「A4一枚で見られる形」にします。Claude CodeにHTMLレポートを生成させる依頼を出すと、グラフ・表・前月比のアイコン付きの1枚レポートを作ってくれます。

特に、**施設基準に直結する指標4（重症割合2割）と指標8（医師1人100人以下）は、要件ラインを赤い破線で表示するように指示しておく**。要件を割りそうな月は赤色でハイライトされる仕様にすると、見た瞬間に判断できます。

ブラウザで開いてPDFに変換すれば、経営会議の資料として配布できる形になります。スタッフ・幹部・銀行——共有先に応じて見せる指標を切り替えるバージョンも、テンプレートを複数用意すれば簡単です。

月初に事務長がCSVを所定のフォルダに置き、Claude Codeのコマンドを1つ叩く。**10分後にはダッシュボードPDFが完成している**——これが当院の現在のワークフローです。

### 自院で実践して見えたこと

最初の失敗は、**「最初から完璧なダッシュボードを作ろうとした」**ことです。9つの指標を一気に実装しようとして、データソースの整理段階で頓挫しかけました。CSVのフォーマットが月によって微妙に違っていたり、列名が日本語と英語で混在していたり、想定外の前処理が必要だったり。最初は3指標——総患者数・新規患者数・紹介元構成比——だけで作り、動かしながら指標を増やしていく方が確実です。

さらに、ダッシュボードがあると**「異常検知の速さ」が劇的に変わりました**。これまでは「何かおかしい」と気づくのが3〜6か月遅れていたのが、月次でほぼリアルタイムに気づける。先月、特定の施設からの紹介が突然ゼロになっているのに紹介元構成比のグラフで気づき、すぐに連携相談員に連絡したところ、担当者の異動で連携が一時的に途切れていたことがわかりました。ダッシュボードがなければ、3か月後に「あの施設、最近紹介が来てないな」と気づいて、すでに別のクリニックに紹介ルートが切り替わっていた——そんな事態になっていたはずです。

**施設基準系の指標（重症割合・医師1人あたり患者数）が常に視界に入る効果**も大きい。要件を割りそうな月は、新規受け入れの優先順位を「重症患者」に振り替える、医師の採用を前倒しする——こうした判断が、年度末に「気づいたら要件を割っていた」となる前に打てるようになりました。

### ダッシュボードは「経営判断の速度」を変える

月次経営ダッシュボードは、特別なSaaSツールを買う必要はありません。**Claude CodeとCSVファイルだけで、十分な品質のものが自作できる時代**です。

9つのKPI（総患者数・新規患者数・紹介元構成比・重症割合・緊急往診・看取り・患者単価・医師1人あたり患者数・1日訪問件数）を1枚に並べる。それをClaude Codeで毎月自動生成する。事務長は集計作業から解放され、院長は「判断」に集中できる。

今日から始められるのは、**自院のレセコンとCrossLog reportから、上記9指標のうちまず3つを出してみること**です。総患者数・新規患者数・紹介元構成比——この3つだけでも、毎月同じフォーマットで並べて見るだけで景色が変わります。

戦わない在宅医療——それは、勘ではなく数字で経営すること、そして数字を「集計」するのではなく「判断」に使うこと。ダッシュボードは、その入り口です。
