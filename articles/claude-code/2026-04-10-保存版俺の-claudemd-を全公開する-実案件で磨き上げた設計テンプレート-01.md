Claude Code を本気で使い倒すなら、`CLAUDE.md` の設計がすべてを決める。雑に書かれた CLAUDE.md の下で動く Claude は、雑な成果物しか返さない。逆に、人間エンジニアへの入社初日オンボーディング資料レベルで書き込んだ CLAUDE.md を置くと、同じモデルとは思えないほど挙動が引き締まる。

この記事では、半年以上の実案件運用で磨き込んだ俺の CLAUDE.md を、セクションごとに全公開する。コピペで即戦力になる完成版を末尾に置いた。

## なぜ CLAUDE.md が必要か

Claude Code は起動時に作業ディレクトリの `CLAUDE.md` と、ホームの `~/.claude/CLAUDE.md` を自動で読み込む。つまりここに書いたことは毎回のシステムプロンプトに注入されると思っていい。毎回同じ注意を口頭で伝える必要がなくなり、禁止事項の破りもほぼ消える。

## セクション1: セッション開始時の動作

```
### セッション開始

作業再開時、以下のファイルが存在すれば読むこと:
- `tasks/todo.md` — 未着手タスクと進捗
- `tasks/lessons.md` — 過去の失敗と学び

作業完了時、変更があれば更新する。存在しなければ必要になった時点で作成。
```

なぜ必要か。複数セッションをまたぐと Claude は前回の文脈を覚えていない。ファイルに状態を書き出させる癖をつけることで、擬似的な長期記憶が生まれる。

## セクション2: 禁止事項

```
### 禁止事項

- 明示的な指示なしにコミットするな
- console.log / print デバッグをコミットに残すな
- 変更対象外のコードにリファクタ・コメント追加・型注釈追加をするな
- 聞かれていない改善提案を長々と書くな
- any 型を使うな（TypeScript の場合）
- お世辞、水増し、冗長な説明は禁止
```

禁止事項セクションは CLAUDE.md の中で一番効く。「〇〇してくれ」より「〇〇するな」のほうが守られやすいのは、生成モデルの特性として知っておくと得。特に `any` 禁止と「変更対象外をいじるな」は、レビューのストレスを激減させる。

## セクション3: 並列作業ルール

```
### 並列作業

独立したタスクが複数あるとき、サブエージェントを最大3つ使って並列実行すること。
依存関係があるタスクは直列で実行する。
並列実行時はユーザーに告知すること（許可は不要）。
```

Claude Code はサブエージェントを並列で起動できる。放っておくと全部自分でやろうとするので、明示的に「並列化しろ」と書く必要がある。3つまでに制限するのは、APIレート制限とコンテキスト管理の現実的な落とし所。

## セクション4: コミット前チェック

```
### コミット前チェック

Hook で型チェック・ビルド・テストを強制実行済み。
Hook 未設定の環境では、コミット前に以下を手動実行:
1. `npx tsc --noEmit`
2. `npm run build`
3. `npm test`
```

Hooks でガチガチに縛っているプロジェクトと、そうでないプロジェクトが混在するので、フォールバックを書いておく。これがないと、Hooks がない環境で壊れたコードを平気でコミットしてくる。

## セクション5: 回答スタイル

```
### 回答スタイル

- 回答言語: 日本語
- 文体: 簡潔・断定的。前置きと後書き禁止
- コードブロックには言語指定を必ず付ける
- 「いかがでしたか？」などの定型句は禁止
```

文体指定は地味だが効果絶大。特に「前置き禁止」は、毎回 "本記事では〜について解説します" みたいな無駄を消してくれる。

## 完成版テンプレート

以下を `CLAUDE.md` にコピペして、プロジェクトに合わせて数行変えるだけで使える。

```
# プロジェクト指示

### セッション開始
作業再開時、`tasks/todo.md` と `tasks/lessons.md` が存在すれば読むこと。
作業完了時、変更があれば更新する。

### 並列作業
独立したタスクはサブエージェント最大3つで並列実行。
依存関係があるものは直列。並列時はユーザーに告知（許可不要）。

### 禁止事項
- 明示的な指示なしにコミットするな
- console.log / print デバッグをコミットに残すな
- 変更対象外のコードを勝手にリファクタするな
- any 型を使うな
- 聞かれていない改善提案を長々と書くな
- お世辞・水増し・冗長な説明禁止

### コミット前チェック
1. `npx tsc --noEmit`
2. `npm run build`
3. `npm test`
全て通ってからコミット。

### 回答スタイル
- 日本語、断定的、前置き・後書き禁止
- コードブロックには言語指定
- 定型句禁止（「いかがでしたか」「ぜひ試してください」など）

### ツール選好
- ファイル検索: Glob（find 禁止）
- 文字列検索: Grep（grep/rg 禁止）
- ファイル読み: Read（cat/head/tail 禁止）
- ファイル編集: Edit（sed/awk 禁止）
```

## 要点

CLAUDE.md は「毎回同じ注意を書かない仕組み」だ。禁止事項、並列ルール、コミット前チェック、回答スタイル、この4本柱を押さえれば、出力品質は別物になる。プロジェクト固有のドメインルールはこれに足していくだけでいい。
