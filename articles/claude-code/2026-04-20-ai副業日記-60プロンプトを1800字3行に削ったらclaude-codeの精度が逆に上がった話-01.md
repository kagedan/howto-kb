「良いプロンプト＝長いプロンプト」だと、ずっと信じていた。Claude Codeに1,800字の指示を書き、5回やり直しても満足いく記事が出てこない。ある日、プロンプトを半分に削ってみたら…2回目で通った。所要時間は4時間→45分に。

先週までの私は、とにかくプロンプトを"盛る"タイプだった。「SEOを意識して、読者の悩みに寄り添い、PREP法で書き、H2見出しを5つ、専門用語には必ず注釈を入れて…」Claude Codeへの依頼は毎回1,800字超え。"これだけ書けば完璧だろう"と思い込んでいた。

結果はこうだった。出力の主旨がどこかズレている。指示の半分くらいが反映されていない。手直し5回目で心が折れる——。ある火曜の夜、記事1本に4時間かけた時点で気づいた。「この量、人間でも覚えられないのでは？」

試しにプロンプトを半分に削り、「本当に伝えたい条件3つ」だけに絞った。①読者ターゲット ②悩み ③PREP法。たった3行だ。出力を見て、思わず声が出た。1回目でほぼ完成形。直したのは数字1箇所だけ。トータル45分。 指示量を「3分の1以下」に削っただけで、結果が別物になった瞬間だった。

Before：プロンプト1,800字・修正5回・4時間  
After ：プロンプト3行・修正1回・45分

AIは"全部伝える"相手ではなく、"本質だけ渡す"相手だった。指示が多いほど、AIは優先順位を見失う。制約を減らすほど、AI側が"考えるスペース"を得られる。これは、人に仕事を頼むときも同じだと気づかされた1日だった。今週からは、プロンプトを書く前に「本当に必要な条件は何か？」を30秒だけ考える時間を作った。それだけで、記事1本あたりの修正回数が平均5回→1.2回に減った。
