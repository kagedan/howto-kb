ここまでくるのに、さまざまなものを試してきた。  
  
Dify+Mem0  
Dify+Googleカレンダー  
Dify+Discord  
n8n  
Open WebUI  
ngrok

記事にはしていないが、なぜかDifyからDiscordには繋がらないし、簡単だと言われたGoogleカレンダー連携も6時間格闘してもダメだった。  
  
Dockerなどのローカルホストを使うことでセキュリティに引っ掛かっているのではないかとは思うが、非エンジニアのわたしにはもうちょっとお手上げだ。

しかも、ツールが増えれば増えるほど、メモリを圧迫する。  
必然的に小さいモデルを使わなければいけなくなる。

その辺りの事情を考慮して、最終的にはObsidianに思考メモを貯めるのが一番理にかなっているという結論に至った。  
  
ファイルがそのまま残る。  
AIが解釈して変換することがないから、メダカが錦鯉になったりしない。

Discordなどを窓口としてObsidianと連携する場合、ひとつ問題がある。

iPhoneから気軽にメモしたいとき、毎回YAMLフロントマターを手書きするのは続かない。

```
---
date: 2026-05-01
type: ライフログ
tags: [犬, 病院]
---
```

これを毎回打つのは無理。絶対やめる。  
後からAIに成形させればいいんじゃない？  
  
なら、最初から

**「CoworkのDispatchを使えばいいんじゃないか」**とひらめいた。

## Claudeと一緒にライフログの設計をした

まずチャットのClaudeに相談しながら、ライフログのタグ設計から始めた。

何を記録したいか整理すると、だいたいこういうカテゴリに落ち着いた。

* **犬**：散歩、病院、体調
* **外出**：食事、買い物、コンサート
* **生き物**：メダカ（水替え、産卵）
* **人**：友達、家族
* **体調**：病院、薬
* **家**：掃除、修理、家電
* **お金**：出費、確定申告
* **記録**：読書、映画、ゲーム
* **ガーデニング**：種まき、収穫、水やり

ガーデニングは庭で野菜を作っているので追加した。  
種まいた日、収穫日、失敗した品種。  
後から絶対見返すやつだ。

YAMLはシンプルにこれだけにした。

```
---
date: 2026-05-01
type: ライフログ
tags: [生き物, メダカ, 水替え]
---
```

typeは「ライフログ／アイデア／仕事の案件」の3択。  
多すぎると続かないから。

「Coworkでライフログ管理する人、いるのかな」と聞いたら、「いないと思う笑」と即答された。

そうなの？

## Coworkに「Dispatch」を教えてあげた

フローも考えてタグも決めた。  
あとはiPhoneから実際に送るようにするだけだ。

Coworkに「dispatchでできるかな？」と聞いたら、こう返ってきた。

> 「dispatch」というのがCoworkのどの機能を指しているか、正直なところ私には把握できていません。

**Coworkが、Coworkの機能を知らなかった。**

3月にリリースされたばかりの新機能だから仕方ない。（このやりとりは4月下旬）  
わたしがサポートページのURLを貼って教えてあげた。

「なるほど！それは知りませんでした。読んでみます。」とThinkingが走って、ちゃんと理解してくれた。

Dispatchとは何かというと、スマホのClaudeアプリとMacのCoworkをQRコードでペアリングして、iPhoneからMacのCoworkに指示を送れる機能だ。  
3月17日にリリースされたばかり。

```
iPhone（Claudeアプリ）
  ↓ メモを送る
Mac（Coworkが受け取って処理）
  ↓ YAML付きMDファイルを生成
cowork-note/memo/
  ↓ ObsidianのVaultで検索できる
```

「メダカの水替えをした」とiPhoneから投げるだけで、日付とタグがついた状態でObsidianに保存される。  
これが作れれば、脳内の「あれいつだっけ」問題がだいぶ解消できる。

## 設計が進んだところで壁が出てきた

Coworkへの指示テンプレートも一緒に作った。

> 私からメモが届いたら、以下のルールでMarkdownファイルを作成し、Google DriveのMy Drive/cowork/memo/に保存してください。YAMLフロントマターにdate・type・tagsを付与し、ファイル名はYYYY-MM-DD\_内容一言.mdで。

いい感じだ。

あとはDispatchをペアリングするだけ——と思ったら、ひとつ気になることが出てきた。

Dispatchの説明文にこう書いてあった。

> Claudeは、スマートフォンから送信されたタスクを完了するために、デスクトップ（ファイル、アプリ、ブラウザ）にアクセスします。

**指定したフォルダだけじゃなく、Mac全体にアクセスできる。**

「毎回細かく指示しないとダメなんじゃない？」という疑問も出てきた。

Dispatchは「毎回の指示をMacに届ける橋」であって、ルールを覚えてくれる場所じゃない。

「メダカ水替えした」だけ投げても、ルールがなければただのチャットになる。

Cowork側のグローバル指示を探したけど、それらしき設定欄が見当たらない。

「スキルに登録して、毎回スキルを呼び出す」という方法なら行けそう——というところまで話が進んだ。

## 「なかったことにした」のではなく、「より良い候補が来た」

今回の流れを振り返ると、こんな感じだ。

1. Dify+Mem0が錦鯉を生み出したので断念
2. ObsidianにライフログをDispatch経由で保存する設計を考えた
3. Coworkに新機能Dispatchを教えてあげた（逆転）
4. 設計はほぼできたが、Hermesの登場で優先度が下がった

Dispatchの設計は無駄じゃない。  
ObsidianとiPhoneを繋ぐ手段としては今でも有効だ。  
ただHermesの方がデジタルクローン育成と併用できる。

Coworkは「窓口を一つにしたい」という用途には向いていると思う。  
仕事のタスク自動化なら文句なく動く。  
スケジュール生成ファイルの整理も、cowork-noteフォルダの再編も、Coworkにお願いしたら一発で終わった（それはまた別の話）。

問題は、ライフログにCoworkが最適解なのか？ということだ。

まあ、やってみようとしたことは記録しておく。  
CoworkやClaude codeを使えば、おそらくできないことはない。
