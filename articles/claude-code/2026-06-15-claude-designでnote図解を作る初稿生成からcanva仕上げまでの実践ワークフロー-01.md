文章は書けても、図解で手が止まる。

noteやブログを継続している人ほど、この問題にぶつかります。画像生成AIに「分かりやすい図解を作って」と頼むだけでは、日本語が崩れたり、情報の優先順位が曖昧になったり、あとから一部分だけ直せなかったりするからです。

2026年4月17日にAnthropic Labsから公開されたClaude Designは、会話しながらデザイン、インタラクティブなプロトタイプ、プレゼンテーションなどを作り、チャットやインラインコメントで修正できる研究プレビューです。PDF、PPTX、HTML、Canvaなどへの書き出しにも対応しています。

この記事では、Claude Designを「完成画像を一発生成する道具」ではなく、情報設計から編集可能な図解の初稿を作る道具として使います。対象は、noteの解説記事、比較記事、手順記事に入れる図解です。

## 前提と狙い

Claude Designは、2026年6月13日時点でClaude Pro、Max、Team、Enterpriseの対象ユーザーに提供されています。無料プランには含まれず、研究プレビューのため仕様や利用上限は今後変わる可能性があります。

このワークフローの狙いは、デザイン作業をすべてAIへ渡すことではありません。

この分業にすると、生成AIが苦手な事実確認と最終責任を人間側に残しながら、白紙から配置を考える時間を減らせます。

## 実践ワークフロー

## 1. 本文から「図にする役割」を1つだけ抜き出す

最初に決めるのは色や雰囲気ではなく、画像を見た読者に何を理解してほしいかです。

図解の役割は、次のいずれか1つに絞ります。

* 比較：選択肢の違いを判断できるようにする
* 手順：作業の順序と分岐を理解させる
* 構造：要素同士の関係を見せる
* 要約：長い説明の要点を再確認させる
* 評価：採点基準やチェック項目を一覧にする

比較と手順を同じ1枚に詰め込むと、文字量が増え、スマートフォンで読めなくなります。「この画像がなくても本文は読めるが、画像があると判断が速くなる」という単位が適切です。

入力情報は、まず次の形式に整理します。

```
図解の目的：
読者：
画像を見た直後に分かってほしいこと：
必ず入れる情報：
入れない情報：
掲載場所：
想定表示幅：
```

## 2. 初稿では装飾より情報階層を指定する

Claude Designでは、プロジェクトにスクリーンショット、画像、既存資料などの文脈を追加し、チャットで依頼するとキャンバス上に初稿が生成されます。

最初のプロンプトでは、「おしゃれ」「プロっぽい」といった抽象語より、情報の順番と制約を明示します。

### 初稿生成プロンプト

```
note記事内に挿入する日本語の図解を作成してください。

目的：
読者が「Claude Designで図解を作る5段階」を
30秒以内に把握できるようにする。

読者：
生成AIの基本操作に慣れた個人の情報発信者。

内容：
1. 図解の役割を決める
2. 情報を構造化する
3. Claude Designで初稿を作る
4. チャットとインラインコメントで修正する
5. Canvaで最終調整し、実寸で検品する

レイアウト：
左から右へ進む5ステップ。
各ステップは短い見出し、1文の説明、単純なアイコンで構成。
スマートフォンで縮小表示しても順番が分かるようにする。

デザイン条件：
横幅1600pxを想定。
背景は白。
本文は濃紺。
アクセントは青とオレンジの2色まで。
見出しと本文のコントラストを明確にする。
装飾より可読性を優先する。

禁止事項：
入力していない機能名や数値を追加しない。
小さな注釈を増やさない。
ロゴ、透かし、架空の引用を入れない。

最初に、異なる情報階層のレイアウトを2案提示してください。
```

重要なのは、1案を完成させる前に複数案を比較することです。Claude Designの公式ガイドも、方向性に迷う場合は2〜3案を出させる方法を案内しています。

## 3. 修正方法を変更範囲で使い分ける

Claude Designには、全体を会話で変えるチャットと、キャンバス上の特定箇所へ指示するインラインコメントがあります。直接編集できる箇所は、人間が直した方が速い場合もあります。

![](https://assets.st-note.com/img/1781338340-h7N0y4DJrKE2PntkcmpFIBbG.png?width=1200)

画像1：Claude Designの修正手段使い分け

チャットは、構成変更、配色変更、セクション追加など、全体へ影響する修正に向いています。

```
5ステップの順序は維持したまま、
各カードの説明を現在の60％程度まで短くしてください。
見出しだけを読んでも流れが理解できる情報階層にします。
変更前後の情報を箇条書きで示してから反映してください。
```

インラインコメントは、ボタン、余白、見出しなど対象が明確な局所修正に向いています。

```
この見出しだけ2行以内に短縮する。
意味とステップ番号は変えない。
隣のカードと文字サイズ、上下余白を揃える。
```

一方、固有名詞の誤字、句読点、1文字単位の表記は、可能なら直接編集またはCanva書き出し後に直します。小さな文言修正を何度も再生成すると、関係のない場所まで変わることがあるためです。

なお、公式ヘルプでは、インラインコメントがClaudeに読み取られる前に消える場合があると案内されています。反映されないときは、同じ指示をチャットへ貼り付けるのが現時点の回避策です。

## 4. AI自身に検品させた後、人間が実寸で確認する

生成直後の見た目が整っていても、そのまま公開しません。まずClaudeへ、作成ではなく監査を依頼します。

### 図解監査プロンプト

```
この図解を修正せず、先に監査してください。

次の項目をそれぞれ5点満点で評価します。
・本文を読まなくても主題が分かる
・情報の順序を迷わない
・スマートフォン幅で主要文字を読める
・事実、数値、固有名詞が入力資料と一致する
・色だけに意味を依存していない
・不要な装飾が理解を妨げていない

各項目について、
問題箇所、理由、最小限の修正案を示してください。
入力資料で確認できない内容は「要確認」とし、
推測で正しいと判定しないでください。
まだデザインには反映しないでください。
```

その後、人間が次を確認します。

1. 画像を記事の想定表示幅まで縮小する
2. 固有名詞、数値、ステップ順を本文と照合する
3. 見出しだけを追って内容を説明できるか試す
4. グレースケールでも区別できるか確認する
5. 代替テキストを一文で書けるか確認する

「拡大すれば読める」は不合格です。noteの読者が見るサイズで読めるかを判断します。

## 5. Canva書き出しは「仕上げ」と「再利用」に使う

Claude Designは、ZIP、PDF、PPTX、スタンドアロンHTMLのほか、Canvaへの送信に対応しています。

Canvaへ渡した後は、次の作業だけに絞ると効率的です。

* 日本語の誤字、禁則、改行位置を修正する
* 記事内の他画像とフォント、色、余白を揃える
* note用、SNS告知用などサイズ違いを作る
* 出典や注記を必要な範囲で追加する

Claude DesignとCanvaを往復し続けると、どちらが最新版か分からなくなります。構造が固まるまではClaude Design、表記とブランド調整はCanva、と工程を分けてください。

## 品質を高める評価基準

### 1. 情報量ではなく判断速度が上がっているか

図解へ本文の全文を移す必要はありません。比較なら違い、手順なら順番、評価なら基準が一目で分かることを優先します。

### 2. 視線の入口と出口が明確か

最初に見る見出し、次に追う要素、最後に残す結論が分かるかを確認します。すべての文字が同じ大きさだと、読者は読む順番を決められません。

### 3. AIが追加した内容を除外できているか

もっとも危険なのは、見栄えを整えるためにAIが架空の数値、機能名、引用、実績を補うことです。元資料にない情報は、装飾的な小文字であっても削除します。

### 4. 編集可能な状態を残しているか

サービスの料金、仕様、日付を含む図解は更新が必要です。完成画像だけでなく、Claude DesignのプロジェクトやCanvaの編集データを残し、ファイル名に更新日を入れます。

### 5. 画像だけに重要情報を閉じ込めていないか

検索エンジンや支援技術が画像内の情報を完全に理解するとは限りません。重要な結論は本文にも書き、画像には内容を説明する代替テキストを設定します。

## 無料・有料と利用上限

2026年6月13日時点で、Claude DesignはPro、Max、Team、Enterprise向けです。

* Pro：月額20ドル、年払いは200ドル。短い探索や単発利用向け
* Max：月額100ドルから。継続的に多く使う利用者向け
* Team：5人以上が前提。Standardは月払い25ドル、年払い20ドル
* 無料プラン：Claude Designは対象外

Claude Designの利用量は通常のClaudeチャットやClaude Codeとは別に計測され、個人プランには7日ごとにリセットされる週間枠があります。上限の具体量は固定値として公開されておらず、ベータ期間中に変更される可能性があります。

追加利用を有効にすると、含まれる枠を超えた利用が従量課金へ切り替わります。使う場合は、月間上限と使用通知を先に設定してください。

料金と提供条件は変わるため、契約前に公式ページで再確認してください。

## 注意点

### 研究プレビューとして扱う

Claude Designは正式な安定版ではありません。公式ヘルプでは、インラインコメントの消失、コンパクト表示での保存エラー、大規模リポジトリ接続時の遅延などが既知の制約として挙げられています。

重要な制作物は定期的に書き出し、唯一の編集元をプレビュー機能だけに置かない方が安全です。

### 個人情報と機密情報を入れない

顧客名、未公開企画、アクセス解析の生データ、契約書、顔写真などは、必要性を確認してからアップロードします。匿名化できる情報は置換し、不要な列やメタデータを削除してください。

個人向けClaudeでは、ユーザーがモデル改善を許可した場合などに、会話やコーディングセッションがモデル改善へ使われる可能性があります。Incognito chatは、モデル改善設定が有効でも改善には使われないと案内されています。利用前にPrivacy Settingsを確認します。

### 著作権とブランド模倣を区別する

参考画像を添付するときは、自分が利用権を持つ資料、自作物、利用条件を確認した素材を使います。

「この画像と同一にして」ではなく、余白、配色数、情報密度、視線誘導など、必要なデザイン原則へ分解して指示してください。第三者のロゴやキャラクターを無断で組み込まないことも重要です。

### AIのアクセシビリティ評価を鵜呑みにしない

Claudeへコントラストや情報階層のレビューを依頼できますが、最終判定は実データで行います。書き出した画像の縮小表示、コントラスト、代替テキスト、色覚に依存しない表現を人間が確認してください。

## まとめ

Claude Designの価値は、図解を一発で完成させることではなく、情報構造を会話で試し、部分ごとに修正し、編集可能な形式へ渡せることにあります。

実践の要点は5つです。

1. 図解の役割を比較、手順、構造、要約、評価の1つに絞る
2. 初稿では装飾より情報階層と禁止事項を指定する
3. 全体変更はチャット、局所変更はコメント、表記修正は直接編集で行う
4. AI監査の後に、本文照合と実寸確認を人間が行う
5. Canvaは最終調整とサイズ展開に使い、工程を分ける

「生成する」より「直し方を設計する」ことが、AI図解を実用品に変えるポイントです。
