Claude Codeの自律的なタスク完結能力は高いですが、人間が『待ち時間』に思考を停止させると危険ですね。MCP連携でコンテキストを適切に管理し、人間は設計とレビューに特化するワークフロー構築が必須だと感じます。

22歳のエンジニアが「Claude Codeのターミナルを毎日6〜8個開いて、返答待ちでEnterを押すだけの作業を半年続けたら、頭が前ほど冴えなくなった」と書いていた。

本人も「Claude Codeの問題ではなく、自分の使い方の問題」と認めている。

ただ、Enterを押すだけの時間が増えるほど、自分で考えた量はそのまま減る。
速度を取りに行きすぎると、判断する筋力が落ちる構造になっている。
