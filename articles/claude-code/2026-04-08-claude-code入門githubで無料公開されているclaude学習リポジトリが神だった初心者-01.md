## はじめに

こんにちは！エンジニアのたまごのKAIです。

普段からClaude Codeを開発業務でバリバリ活用しているのですが、最初に「どうやって使いこなせばいいの？」と途方に暮れた経験があります。

そんなときに出会ったのが、GitHubで無料公開されているClaude Code学習リポジトリ **「claude-howto」** です。私自身このリポジトリで勉強したこともあり、Claude初心者には自信を持っておすすめできます。

---

## claude-howtoとは？

**GitHubスター5,900以上**を獲得しているコミュニティ主導のClaude Codeチュートリアル集です。

公式ドキュメントは「各機能を個別に説明するだけ」で、「実際どう使えばいいの？」という疑問が残りがちです。このリポジトリは**複数の機能を組み合わせた実践的なワークフロー**を、豊富な図解とコピペで使えるテンプレートとともに解説してくれています。

---

## 3段階のレベル別カリキュラム

初心者から上級者まで段階的に学べる構成になっています。

レベル対象者学習時間の目安BeginnerClaude Codeを使い始めた人約2.5〜3時間IntermediateCLAUDE.mdを知っている人約3.5〜5時間AdvancedMCPやHooksを試したい人約5時間

合計11〜13時間ほどのボリュームがあり、読み応え抜群です。

## 収録されている主なコンテンツ（全10モジュール）

### 1. Slash Commands（スラッシュコマンド）

Claude Codeで使える55以上のコマンドと、オリジナルのカスタムコマンドの作り方を解説。ここを押さえるだけで作業効率が大きく変わります。

### 2. Memory（メモリ）

CLAUDE.mdを使ってClaudeに情報を記憶させる方法。プロジェクトのルールや自分の好みを覚えさせて、毎回説明する手間をなくせます。

### 3. Skills（スキル）

コードレビュー・ドキュメント生成・ブログ下書きなど、再利用できるスキルのテンプレートが28ファイル収録されています。

### 4. Subagents（サブエージェント）

コードレビュー・テスト・セキュリティ監査など専門特化型のAIアシスタントの定義方法。複数のエージェントを使い分けるワークフローが学べます。

### 5. MCP（Model Context Protocol）

GitHubやSlack、Google Docsなど外部ツールとClaudeを繋ぐ方法。エンタープライズ向けの応用的な内容です。

### 6. Hooks（フック）

コードの保存時に自動フォーマット、ツール実行前にセキュリティスキャンなど、イベント駆動型の自動化を実現する方法。

### 7. Plugins（プラグイン）

PR自動レビュー、Kubernetesデプロイ自動化、APIドキュメント自動生成など、実務投入できる完成品プラグインが3種類収録されています。

### 8〜10. Advanced Features・Checkpoints・CLI

拡張思考モード・バックグラウンドタスク・セッションの保存と巻き戻しなど、上級者向けの機能も網羅しています。

## このリポジトリが特にすごいポイント

### コピペで即使えるテンプレートが100以上

「こういう使い方をしたい」という目的に合わせてテンプレートをそのまま使えます。ゼロから設定を考えなくていいのがありがたいです。

### 図解が豊富でわかりやすい

Mermaidダイアグラムを使った視覚的な説明が多く、「なんとなく使えてるけど仕組みがわかってない」という状態から抜け出せます。

### 自己評価機能付き

/self-assessment と /lesson-quiz というコマンドで、自分の習熟度をチェックできます。「どこまで理解できているか」が可視化できるのは珍しいです。

最初はBeginnerのモジュールから順番に読み進めました。「ふむふむ」と読むだけでなく、**実際にClaude Codeで試しながら進める**のがおすすめです。

「なんとなくClaudeに話しかけてるだけ」という段階から「ワークフローとして組み込んで使う」という段階に進めたのは、このリポジトリのおかげです。
