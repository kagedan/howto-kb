AIツールの進化によって、個人でも瞬時にアプリやウェブサイトの画面を生成できるようになりました。しかし、AIに画面を作らせる中で **「画面ごとにデザインがバラバラになる」「細部を修正させようとして指示を繰り返すと泥沼化する」** といった壁にぶつかったことはありませんか？

今回は **Claude Design** の具体的な使い方や、AI共通規格の仕様書 **`design.md`** を活用した実践的な手順を、今一度整理してまとめてみました。

## 1. AI時代におけるワークフローの「劇的な変化」

従来のプロダクト開発と、AIを活用する現代の開発では、プロトタイプ（試作品）の位置づけが真逆になっています。

### 従来の開発フロー

1. 戦略を固める
2. ワイヤーフレーム（画面の骨組み・ラフ）をFigma等で作成する
3. 詳細なビジュアルデザインを作り込む
4. エンジニアがコードを書いて本実装する

### 現代のAI開発フロー（サイクル型）

1. 戦略を固める
2. **いきなり動くプロトタイプ（画面の外側）を作る**
3. 実際に触りながら、高速で改善サイクルを回してコンセプトを煮詰める

現代において、プロトタイプは単なる「完成品のシミュレーション」ではなく、**「手で動かしながら考えるための道具」** へと進化しました。プロダクトマネージャーもエンジニアも、AIを使ってコストをかけずに即座にプロトタイプを出せるようになったため、このやり方が圧倒的に最速となっています。

## 2. 早く作れるからこそ陥る「AIデザインの落とし穴」

AIを使った高速プロトタイピングには、強力なメリットの裏に**重大な落とし穴**があります。それは、**「早く作れる分、画面がバラバラになるのも早い」** という点です。

AIに画面を1枚ずつ作らせると、AIはその画面単体での最適化を試みます。その結果、以下のような現象が発生します。

* 1枚目の画面と3枚目の画面で、使われている「青色」のカラーコードが微妙に違う
* ボタンの角丸（Radius）や余白の取り方が画面ごとにズレている

1枚ずつ見ると綺麗なのに、並べた瞬間に一貫性がなくなり、気持ち悪さや使いにくさを与えてしまう。これこそが、AIデザインにおける「なんか違う」の正体です。

### 解決策：「デザインシステム」を先に敷く

このバラバラ問題を解決するために、プロトタイプを作る**前**に用意するのが **「デザインシステム」** です。  
デザインシステムとは、専門的な難しいものではなく、 **「1度決めたら2度と蒸し返さない決定の束（パターン）」** のことです。

デザインの世界で最も重要視される「一貫性（Consistency）」を保つことには、以下の3つの重要なメリットがあります。

1. **見た目の美しさ**: 要素が揃っているだけで整って見える。
2. **ブランドと信頼性**: トーンが統一されていることでユーザーの記憶に残り、プロダクトへの信頼感が生まれる。
3. **使いやすさの担保**: いつも同じ場所に同じ見た目のボタンがあることで、ユーザーは次に何が起こるか予測でき、脳に負担をかけず「本能的（反射的）」に操作できるようになる。

## 3. なぜ本実装の前に「Claude Design」を使うのか？

「プロトタイプを作るなら、v0やClaude Codeでいきなりロジックも含めてコードを書いて実装すればいいのでは？」と思うかもしれません。しかし、プロの現場では必ず**本実装の前にデザインツール（Claude Design等）で画面の外側（UI）を固めます**。

理由は、**その方が圧倒的にコスト（時間・お金・AIのトークン消費量）が安いから**です。  
バックエンドのロジックまで作り込んだ本実装コードをAIに何度も書き直させると、手戻りのコストが非常に高くなります。まずは「表面的な動きと見た目（UI・インタラクション）」だけをデザインツール上で100%決め打ちし、その後にコーディングエージェントやエンジニアに渡すのが、最も安上がりで確実な方法です。

そのための、デザインシステム構築とUIデザインに特化した最新のエージェントが **「Claude Design」** です。

## 4. 【実践】Claude Designを使ったデザインシステム構築＆プロトタイプ作成手順

ここからは、動画内で行われた「**浮世絵・かわら版風のユニークなヘルスダッシュボード**」という実演を例に、具体的な手順をステップ・バイ・ステップで解説します。

### ステップ1：ブランドのインスピレーション（色・雰囲気）を準備する

まずは作りたいプロダクトのトーン＆マナーの参考となる画像（今回の場合は浮世絵の画像など）を用意します。既存のカラーパレットジェネレーターでは表現しにくい独特な世界観も、AIを活用すれば画像から直接カラーコードを抽出できます。

### ステップ2：Claudeで共通のAIデザイン仕様書`design.md`を生成する

用意した参考画像を通常のClaude（Claude 3.5 Sonnetなど）にアップロードし、デザインシステムのスペック（仕様書）である **`design.md`** を作成してもらいます。

Claudeへの指示（プロンプト）の例：

> 「この浮世絵の画像をインスピレーションにして、基盤となる和紙のようなテクスチャをベースにした、日本の伝統色を用いたヘルスダッシュボードのデザインシステム仕様書（design.md）を作成してください。数字は木版画や漢字のニュアンスを持たせ、完全に日本語対応にしてください」

出力される`design.md`には、デザインの基本要素（アトム）が厳密に定義されます。

* **Colors（カラーパレット）**: 和紙のベース色、墨色、アクセントとなる伝統色（藍色や臙脂色など）。アクセシビリティ（コントラスト比）の自動検証も含まれます。
* **Typography（フォント・文字組）**: 見出し、本文、データ表示用のフォントサイズやウェイト。
* **Spacing（余白ルール）**: UIを整える上で最も重要な要素。要素間の余白ルール（4の倍数や偶数ルール）が自動適用されます。
* **Motion（アニメーション）**: 画面遷移やボタンホバー時の速度やイージングカーブの定義。

### ステップ3：Claude Designに`design.md`を投入してセットアップする

1. **Claude Design**の管理画面を開きます。
2. 「Setup Design System」を選択し、カンパニー名やシステム名（例：「ジ和風河ら板システム」）を入力します。
3. 先ほど通常のClaudeで生成した`design.md`のコードを、アセット/メモ欄（Add On Logos, Assets）にペーストします。
4. 追加の補足事項（例：「レイアウトはガーミンコネクトやNikeのアプリのようなダッシュボードスタイルを基準にして」など）を入力し、生成（Continue to creation）を開始します。

### ステップ4：生成されたデザインシステムをレビューする

Claude Designがベースとなるコンポーネント（部品）のレビュー画面を自動生成してくれます。

* ベースカラー（和紙の色や墨の色）が意図通りか確認。
* テキストのアクセシビリティ（背景色に対する文字の読みやすさ）が基準をクリアしているかの自動チェック画面を確認します。
* 定義したフォント（明朝体や特殊なフォントなど）の統一感を確認します。

### ステップ5：チャットと視覚的なコントロールによる微調整（Tweak）

Claude Designの最も強力な特徴は、出力されたUIを直感的に、かつ一気通貫で微調整できる点です。

* **指示による修正と伝播（カスケード）**: 「枠線（キーライン）を少し太くして、黒を強めてメリハリを出して」のようにチャットでリクエストを送ると、システム全体のコンポーネントに一瞬でその変更が適用されます。アトム（原子＝枠線の太さ）を変えることで、自動的にすべての分子（ボタンやテキストボックス）のデザインが一貫性を保ったまま変化します。
* **スライダーやつまみ（Tweak機能）での直感的調整**: デザインバリエーションを実験するためのコントロールが自動生成されます。例えば、データを「漢字表記」にするか「アラビア数字」にするかをトグルで切り替えたり、和紙のテクスチャの劣化度合い（テクスチャの濃さ）をスライダーで「70%」にする、といった微調整がコードを触らずに可能です。
* **マークアップ（注釈）ツールでのピンポイント指示**: UI上の変更したい部分をドラッグで囲み、「セグメントの代わりに適度な日本語にして」などと直接画面上にコメントを書き込んで部分修正させることができます。

### ステップ6：UIの組み立てと、実装フェーズへのエクスポート

デザインシステム（ルール）が固まったら、実際に「走行データをトラックするダッシュボード画面」などのデスクトップ用WebアプリUIをAIに組み立てさせます。見た目や使い心地、世界観に合わせた日本語ラベルの表記（「ライド」ではなく「牢益」に変更するなど）を徹底的にこのツール上で納得がいくまで作り込みます。

完璧に一貫したUIプロトタイプ（外側）が完成したら、いよいよ実装フェーズへの移行です。  
Claude Designの画面から「Share」>「Send to」を選択し、**Claude Codeへそのままプロジェクトを送信する**か、または「Project Archive（プロジェクトアーカイブ）」として`design.md`やスペック一式をダウンロードします。これをコーディングAIや開発エンジニアに渡すことで、手戻りのない、完璧に一貫した本実装がスタートします。

## 5. まとめ：変わり続けるAIツールと、変わらない「唯一の真理」

AI開発ツールの進化スピードは凄まじく、Claude Designやv0、その他のエージェントなど、来年にはまた新しいツールが登場したり、名前が変わったりしているかもしれません。

しかし、どれだけツールが変わろうとも、プロダクト開発における **「唯一変わらない真理」** があります。

> **「実際にコードを実装する前に、デザインシステムを構築し、プロトタイプでUIを固める」**

この順序を間違えて、いきなりAIに本実装コードを書かせて画面を量産しようとすると、どれだけ優れたAIツールを使っても「なんかバラバラで使いにくい、違うもの」しか生まれません。

AIによって誰もが1人でプロダクトを作れるようになった現代だからこそ、**デザインシステムは一部の専門デザイナーだけのものではなく、モノづくりに携わるすべての人（エンジニア, PM, インディーズ開発者）の「必須の基礎教養」** になりつつあります。

ぜひ、今回ご紹介した「デザインシステム ➔ プロトタイプ ➔ 本実装」の黄金のワークフローを、あなたの開発にも取り入れてみてください。
