Claude Codeである作業をやっていた時に『その作業、時間的にやっていたのはいつ頃だったか』を知りたいってことはありますか？ありますよね。

Claude Codeである作業をやっていた時に、『その作業、地球が何回回った時？』って聞かれることはありますか？

いや無ぇよ。

まぁ、ここはあるってことにしておいてください。聞かれるってことは無いだろうけどふと自問することはもしかしたらあるかもしれません。

## きっかけ

やってみよう。と思い立ったのは [@yancya](https://x.com/yancya) さんの下記X投稿です。こちらの投稿で『地球が何回回った時』という、幼少期に誰もが通るであろうロマン溢れるこのコメントに心打たれました。

<https://x.com/yancya/status/2037445802294223314>

上記投稿では、大枠の仕組みとしてHookを活用した旨言及しています。@yanchaさんの投稿では `PreToolUse` という設定＝Claudeがツール(Bash、Edit、Write、MultiEditなど)を実行する直前に発火するフックイベントで実現した形ですね。

<https://code.claude.com/docs/ja/hooks>

## 私がやろうとしたこと＆実践結果

私はこれを受けて『ユーザーがClaude Codeに対して会話を投げたタイミングでその時の日時が分かる』『更にはその時に地球が何回回っていたか』を分かるようにしたいなと思いました。

前者は `UserPromptSubmit` という設定を活用することにしました。この設定はユーザーがプロンプトを送信するときに実行されるときに利用可能なもので、Claude がそれを処理する前にプロンプト/会話に基づいて追加コンテキストを追加したり、プロンプトを検証したり、特定のタイプのプロンプトをブロックしたりできます。

後者については処理が実現可能だとしてもワンライナーで実現するには骨が折れそうな記載になりそうだなと判断、『前者＋後者の情報を文字列として出力するシェルスクリプトを作成し、それを設定に盛り込む』形としました。

まずは前者、トリガーの設定から。`~/.claude/settings.json`に`hooks`パートを設け、下記のような設定を追加します。上述の様に実行内容自体はシェルスクリプトに任せる形としています。

~/.claude/settings.json

```
{
    :
    :
    "hooks": {
      "UserPromptSubmit": [
        {
          "hooks": [
            {
              "type": "command",
              "command": "~/.claude/hooks/user-prompt-submit.sh"
            }
          ]
        }
      ]
    },
    ;
    ;
  }
```

そして実際の処理を行っているシェルスクリプトの内容について、スクリプト自体は、Claudeに『こういうことしたいんだけど出来る？』ってのを聞いて、幾らかの会話のキャッチボールで出来上がったものを採用しました。途中Claudeが『笑』の文字列をぶっこんで来たときは『お、こいつこういうこともやるんだ』となりました。

![](https://static.zenn.studio/user-upload/b493ae982f06-20260328.png)

やり取りの結果作成されたシェルスクリプトがこちら。コメント部分も『今のやり取りを踏まえて、コメントを見た人が"くっだらねぇなぁと思うような(でも褒めてる)"書き味でよろしく』と伝えて仕上げてもらった内容となっています。

user-prompt-submit.sh

```
#!/bin/bash
  # UserPromptSubmit hook: JST timestamp + 地球自転回数(地球誕生起算)
  #
  # 【このスクリプトがやっていること】
  # Claude Codeにプロンプトを投げるたびに、「今この瞬間は地球が誕生してから
  # 何回目の自転か」をシステムメッセージとして表示する。
  # 実用性は皆無。ロマンのためだけに存在する。
  #
  # 【計算の根拠(という名の雑な概算)】
  #
  # ■ 地球の年齢
  #   現在の科学的コンセンサスによれば、地球の年齢は約45億4000万年。
  #   ただし誤差は±5000万年ほどあるので、下1〜2桁は最初から信頼できない。
  #   そのあたりは見なかったことにする。
  #
  # ■ EARTH_BASE_DAYS の算出
  #   4,540,000,000年 × 365.25日/年 = 1,658,235,000,000日
  #   (365.25はうるう年を考慮した平均。太古の地球の自転速度とか、
  #    月との距離が違うとか、そういう話は今日のところは忘れる。)
  #
  # ■ 今日までの日数(Unix epoch分)
  #   1970年1月1日 00:00:00 UTC からの経過秒数を86400で割る。
  #   要するに「Unixエポック起算の日数」を足すことで今日まで繋げている。
  #
  # ■ 合計
  #   EARTH_BASE_DAYS + EPOCH_DAYS = 地球誕生からの総自転回数(のようなもの)
  #
  # ■ 表示フォーマット
  #   桁が大きすぎて読めないので、兆・億・万・以下4桁に分けて表示。
  #   「1兆6582億3502万0176回目」みたいな感じ。
  #
  # ■ 免責事項
  #   この数値の下10桁くらいは誤差の中に埋まっているので、
  #   精度を求めるような場面での使用は推奨しない。
  #   というか、そんな場面でこのスクリプトを使おうとする人間がいたら
  #   そちらの方が問題である。

  JST_DATE=$(TZ=Asia/Tokyo LC_TIME=ja_JP.UTF-8 date '+%Y/%m/%d(%a) %H:%M:%S JST')
  EARTH_BASE_DAYS=1658235000000
  EPOCH_DAYS=$(( $(date +%s) / 86400 ))
  TOTAL_DAYS=$(( EARTH_BASE_DAYS + EPOCH_DAYS ))

  CHO=$(( TOTAL_DAYS / 1000000000000 ))
  OKU=$(( (TOTAL_DAYS % 1000000000000) / 100000000 ))
  MAN=$(( (TOTAL_DAYS % 100000000) / 10000 ))
  NOKORI=$(( TOTAL_DAYS % 10000 ))
  FORMATTED="${CHO}兆$(printf '%04d' $OKU)億$(printf '%04d' $MAN)万$(printf '%04d' $NOKORI)"

  echo "{\"systemMessage\": \"${JST_DATE} | 🌏️♻️ ${FORMATTED}回目\"}"
```

Claude Codeを再起動し、表示内容を確認します。  
はい、まさしくこれが私の実現したかったものでした！

上記手順ではプロンプト実行結果でのみその情報が分かる形となっていますが、『結果としてログ・ファイルにも残しておきたい』のであれば下記のように別途ログファイルを吐き出すスクリプトを定義し、活用してみても良いかもですね。

user-prompt-submit-v2.sh

```
#!/bin/bash
# UserPromptSubmit hook: JST timestamp + 地球自転回数(地球誕生起算) + ログ出力
#
# （コメント省略）

# ★ これが必要。stdinは1回しか読めないので最初に変数へ保存する
INPUT=$(cat)

JST_DATE=$(TZ=Asia/Tokyo LC_TIME=ja_JP.UTF-8 date '+%Y/%m/%d(%a) %H:%M:%S JST')
EARTH_BASE_DAYS=1658235000000
EPOCH_DAYS=$(( $(date +%s) / 86400 ))
TOTAL_DAYS=$(( EARTH_BASE_DAYS + EPOCH_DAYS ))

CHO=$(( TOTAL_DAYS / 1000000000000 ))
OKU=$(( (TOTAL_DAYS % 1000000000000) / 100000000 ))
MAN=$(( (TOTAL_DAYS % 100000000) / 10000 ))
NOKORI=$(( TOTAL_DAYS % 10000 ))
FORMATTED="${CHO}兆$(printf '%04d' $OKU)億$(printf '%04d' $MAN)万$(printf '%04d' $NOKORI)"

# ★ 変数に入れておく（ログでも使うため）
SYSTEM_MESSAGE="${JST_DATE} | 🌏️♻️ ${FORMATTED}回目"

echo "{\"systemMessage\": \"${SYSTEM_MESSAGE}\"}"

# --- ログ出力 ---
LOG_DIR="$HOME/.claude/logs"
mkdir -p "$LOG_DIR"

jq -n -c \
  --arg ts "$(date -u +%Y-%m-%dT%H:%M:%SZ)" \
  --arg jst "$JST_DATE" \
  --arg rotation "$FORMATTED" \
  --arg prompt "$(echo "$INPUT" | jq -r '.prompt // empty')" \
  --arg session "$(echo "$INPUT" | jq -r '.session_id // empty')" \
  --arg cwd "$(echo "$INPUT" | jq -r '.cwd // empty')" \
  --arg hook_output "$SYSTEM_MESSAGE" \
  '{ts: $ts, jst: $jst, earth_rotation: $rotation, prompt: $prompt, session_id: $session, cwd: $cwd, hook_output: $hook_output}' \
  >> "$LOG_DIR/user-prompts.jsonl"
```

## まとめ

という訳で、幼少期のロマン(？)をClaude Codeのhooksの仕組みを使って実現する手順の紹介でした。いざという時に情報を返せるように、または純粋に実行ログとして残しておけるように、興味のあるかたは是非設定を検討してみてください。(※日時表示は普通に有用だと思います)
