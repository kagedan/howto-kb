## TL;DR

あまり長い記事ではないのですが、どういう使い分け型があるのかだけ知りたい方のためにどういったケースでどの方式で利用の決め方があるのかだけをまとめておきます。

## Claude Codeを使うには

組織でClaude Codeを使うには大きく3つの利用経路からどれかを使って利用します。

### 1. AWS Bedrock / Google Vertex 等（API）

AWSやGoogle CloudのAPIを経由してClaudeモデルを呼び出す方式です。すでにAWSやGoogle Cloudをインフラとして使っているチームであれば、既存のIAMやSSOの仕組みをそのままユーザー管理に活用できます。従量課金なので少人数・低頻度の利用ではコストを抑えやすい反面、ヘビーに使い始めると青天井になるため、利用量が増えてくると管理が必要になります。Claude WebやCoworkといったチーム向け機能は利用できません。

### 2. Claude Developer Platform

Anthropicが提供するAPIを直接利用する方式です。最新モデルへのアクセスや細かなAPIパラメータの制御が必要な場合に向いています。料金体系はAWS Bedrock等と同様にトークン従量課金ですが、請求はAnthropicへの直接支払いになります。こちらもClaude WebやCoworkは利用できません。

<https://claude.com/ja-jp/pricing#api>

### 3. Claude Team plan

Claudeのチーム向けサブスクリプションプランです。1ユーザーあたりの月額定額制で、Standard / Premiumのシート種別があります。Claude WebやCoworkをはじめとしたTeam向け機能が利用でき、エンジニア以外のメンバーも含めたチーム全体でClaudeを活用したい場合に適しています。なお、今回は取り上げていませんがより大規模な組織向けにEnterprise planも用意されています。

<https://claude.com/ja-jp/pricing#team-&-enterprise>

## これまでツクリンクでAWS Bedrock経由を使っていた理由

ツクリンクでは一番初めにPoC目的でコーディングエージェントの利用を開始しました。（その時はClineを使ってました。懐かしい。。）  
PoC目的でかつ生成AIの成長速度が凄まじかったため、素早く導入し検証する必要があったことから新規にClaudeを契約せずに社内でも利用しているAWS Bedrock経由で利用を開始しました。  
AWS SSOを構築していたこともあり、ユーザー管理基盤も整っており、IAMポリシーでAWS Bedrockへのアクセス制御を設定すれば、新しいアカウント管理基盤を別途用意せずにそのまま使える手軽さがありました。  
PoCの後、エンジニア全体展開する時も全員がどのくらい活用できるか未知数だったことと、当時はCowork等の機能がなかったことからAWS Bedrock経由のまま展開しました。

また、この時は社内でもCursorやGitHub Copilot等他のAIツールを利用していた人もおり、Claude Codeの利用が少額だったこともあり、AWS Organizations配下でまとめて請求が来るのでクラウドインフラ全体の請求と一緒に管理できるというメリットもありました。

## Team planに移行した理由

### 1人あたりの利用コストがTeam planを使った場合を上回った

AWS Bedrock経由での利用コストを改めて計算してみると、月換算で$150程度かかっていました。Team planはPremiumシートでも月払いだと1ユーザー$125、年払いだと1ユーザー$100です。利用の少ないメンバーはStandardシートにすることで$30程度になります。さらに生成AIの活用を推進していくことを考えるとAWS Bedrockの従量課金で使い続けるよりもTeam planを利用してコントロールした方が良いのではと思ってきました。

コスト面での優位性がなくなったことで、他のメリットを比較して判断できるようになりました。

### チャット機能、Cowork機能のニーズが社内から出てきた

Cowork機能が登場したことや非エンジニアの関心が高まったことで社内でコード機能以外のClaudeの機能を利用したいというニーズが出てきました。なかでもCoworkは複数人でClaudeとリアルタイムに共同作業できる機能で、チームでの活用の幅が広がりそうだったのが移行を後押ししました。  
これらはAWS等のAPIだけしか利用できない経路だと使うことができないのでTeam planを利用する必要があります。

### プロダクトのコストと分けて管理したくなった

Claude CodeへAIコーディングツールを統一したことでコストもAWSに集中して大きくなりました。AWSには他にもプロダクトのインフラコストもまとまっています。Claude Codeの利用コストもAWSの請求に混在すると、どこまでがプロダクトのコストでどこまでがClaude Codeのコストなのかが少し見えにくくなります。Anthropicへの直接支払いに切り出すことで、Claude Codeの利用コストを独立して把握しやすくなります。

## Team planを使う上での注意点

Team planにすることで利用できる機能は増えますが、一方でいくつか事前に確認しておくべきこともあります。

### 上限超過時の仕組み

Team planには月ごとの利用上限が設定されています。上限に達した場合、そのままではClaude Codeが使えなくなります。ただし、事前にクレジットを購入しておくことで上限到達後もリセットまで引き続き利用できます。  
クレジットを消費して継続利用させるかどうかはオーナー権限のアカウントへの申請制になるので勝手に誰かが使い切ったみたいなことにはなりません。  
このクレジットはデポジット形式で、一定額を事前にまとめて購入する仕組みです。チームの利用量に合わせて余裕を持ったクレジットを確保しておくのがおすすめです。（今回取り上げてないEnterprise planだと超過分は使用した分だけの後払いの従量課金方式のようでした。）

### 年払いと月払いの検討

年払いだと月払いに比べて1ユーザーの毎月あたりPremiumシートで$25、Standardシートでも$5安くなります。ただ、AIの進化が速いことを考えると1年縛りは少し慎重に考えたほうが良いと思っています。  
今後さらに良いサービスや仕組みが出てきたときに移行するハードルが上がるので、今は月払いのまま様子を見つつ、状況に応じて判断していくつもりです。

## 移行してみて

移行自体はスムーズで、ユーザーをTeam planに招待してClaude Codeの接続先をAWS BedrockからAnthropicに切り替えるだけです。AWSのSSO経由での管理はなくなりましたが、Anthropicの管理画面でのユーザー管理はシンプルで特に困っていません。現時点で使用が少ないメンバーやちょっと試してみたいというメンバーはStandardシートで登録することでコストを圧縮し、さらなる拡充にも対応できています。  
コスト面での差分がない状況では、機能面のメリットが大きいTeam planを選ぶのが現時点では得策かなと思います。Coworkをはじめとした機能は活用し始めたばかりなので、さらなる効率化のため引き続き試行錯誤していこうと思っています。  
AIツールの利用環境は変化が速いので、コストや機能を定期的に見直しながら最適な構成を維持していきたいと思います。

一方で、最近Claude Codeで障害が発生し、しばらく使いづらい期間がありました。Team planへの統一でメリットは大きくなりましたが、単一ツールへの依存が高まった分、障害時のリスクも増してるかなとも思っています。バックアップとなる手段をどう確保するかは今後の課題として意識しておこうと思っています。
