Claude Code v2.1.101は、セキュリティ修正・セッション管理の改善・バグ修正を中心に、エンタープライズ環境対応や開発者体験の向上に注力した大規模リリースです。

## 今回の注目ポイント

1. **`/team-onboarding` コマンドの追加** -- ローカルの使用状況からチームメンバー向けのオンボーディングガイドを自動生成
2. **OS の CA 証明書ストアをデフォルト信頼** -- エンタープライズの TLS プロキシ環境でも追加設定なしで動作するように
3. **`permissions.deny` が PreToolUse フックに確実に優先されるよう修正** -- セキュリティポリシーの抜け道をふさぐ重要な修正

## チームのオンボーディングを自動化する `/team-onboarding`

**ローカルの Claude Code 使用実績をもとに、チームメンバー向けのランプアップガイドを生成するコマンドが追加されました。**

新しくチームに加わるメンバーへの Claude Code 導入を担当している方、またはチーム内での使い方を標準化したい方に特に役立ちます。

### 何が変わったか

`/team-onboarding` コマンドを実行すると、自分がローカルで Claude Code をどのように使ってきたか（よく使うコマンド、ワークフロー、設定など）の情報をもとに、チームメンバー向けのオンボーディングドキュメントを自動生成します。

### なぜ嬉しいか

これまで「うちのチームではこういう使い方をしている」という知識は、口頭での引き継ぎや手書きのドキュメントに頼るしかありませんでした。自分の実際の使用パターンが反映されたガイドが自動生成されることで、**ベテランメンバーの暗黙知をドキュメント化**するコストが大きく下がります。

## エンタープライズ環境での TLS プロキシ設定が不要に

**OS の CA 証明書ストアをデフォルトで信頼するようになったため、社内のカスタム TLS プロキシがそのまま動作します。**

企業のセキュリティポリシーで通信が TLS プロキシを経由するよう設定されている環境を使っている方に朗報です。

### 何が変わったか

これまでは、企業の TLS インターセプトプロキシ（自己署名証明書や社内 CA が使われているケース）を使う環境では、証明書エラーを回避するために追加の設定が必要でした。今回のアップデートで、**OS の CA 証明書ストアがデフォルトで信頼される**ようになり、OS 側に正しく CA が登録されていれば追加作業なしで動作します。

### 設定の切り替え

従来どおり Claude Code にバンドルされた CA のみを使いたい場合は、環境変数で切り替えられます。

```
export CLAUDE_CODE_CERT_STORE=bundled
```

バンドル CA のみを使う設定に戻すと、OS に追加されたカスタム CA が無視されます。エンタープライズ環境では `CLAUDE_CODE_CERT_STORE=bundled` を設定しないことが推奨です。

## `permissions.deny` の抜け穴をふさぐセキュリティ修正

**`permissions.deny` で設定した拒否ルールが、PreToolUse フックによって「確認プロンプト」に降格されてしまうバグが修正されました。**

`permissions.deny` でツール使用を厳密に禁止している環境では、このバグがセキュリティポリシーの意図しない迂回につながる可能性がありました。アップデートを強く推奨します。

### 何が変わったか

`settings.json` の `permissions.deny` にツールの使用禁止ルールを設定していても、PreToolUse フックが `permissionDecision: "ask"` を返した場合に、拒否が「ユーザーへの確認」に格下げされてしまうバグがありました。今回の修正で、`permissions.deny` のルールが常に最優先されるようになります。

```
{
  "permissions": {
    "deny": ["Bash(rm -rf *)", "WebFetch"]
  }
}
```

上記のような設定は、フックの応答に関わらず確実に適用されます。

### なぜ嬉しいか

CI 環境や共有の開発環境など、**ツール使用を明示的に制限したい場面**では、ポリシーが確実に守られることが重要です。フックの実装の仕方によらずポリシーが機能する安心感は、チームでの運用において大きな意味を持ちます。

## バグ修正

バグ修正 36件（クリックで展開）

* **セキュリティ**: LSP バイナリ検出に使用していた POSIX `which` フォールバックにコマンドインジェクション脆弱性が存在していたのを修正
* **メモリリーク**: 長いセッションで仮想スクローラーがメッセージリストの過去コピーを数十件保持し続けるメモリリークを修正
* **セッション再開**: `--resume`/`--continue` で大きなセッションを再開する際、デッドエンドブランチにアンカーされて会話のコンテキストが失われるバグを修正
* **セッション再開**: `--resume` のチェーン復元がサブエージェントのメッセージ付近の書き込みギャップを原因として無関係なサブエージェント会話にブリッジされるバグを修正
* **セッション再開**: `file_path` が欠損した Edit/Write ツール結果が永続化されている場合に `--resume` でクラッシュするバグを修正
* **タイムアウト**: ハードコードされていた5分のリクエストタイムアウトを修正。`API_TIMEOUT_MS` の設定に関係なくローカル LLM・extended thinking・低速ゲートウェイが中断されていた
* **設定管理**: `--setting-sources` に `user` が含まれない場合にバックグラウンドクリーンアップが `cleanupPeriodDays` を無視して30日より古い会話履歴を削除するバグを修正
* **Bedrock**: `ANTHROPIC_AUTH_TOKEN`、`apiKeyHelper`、または `ANTHROPIC_CUSTOM_HEADERS` が `Authorization` ヘッダーを設定している場合に Bedrock の SigV4 認証が 403 で失敗するバグを修正
* **ワークツリー**: `claude -w <name>` が前回セッションのクリーンアップ後にステールなディレクトリが残り "already exists" エラーで失敗するバグを修正
* **MCP**: サブエージェントが動的に注入されたサーバーの MCP ツールを継承しないバグを修正
* **ワークツリー**: 分離されたワークツリー内で実行されるサブエージェントが自身のワークツリー内のファイルへの Read/Edit アクセスを拒否されるバグを修正
* **Bash**: 起動直後に `mktemp: No such file or directory` でサンドボックス Bash コマンドが失敗するバグを修正
* **MCP**: `outputSchema` を検証する MCP クライアントで `claude mcp serve` のツール呼び出しが "Tool execution failed" で失敗するバグを修正
* **RemoteTrigger**: `RemoteTrigger` ツールの `run` アクションが空のボディを送信してサーバーに拒否されるバグを修正
* **セッションピッカー**: `/resume` ピッカーの複数の問題を修正（デフォルトの狭いビューで他プロジェクトのセッションが隠れる、Windows Terminal でプレビューに到達できない、ワークツリーでの cwd が不正、セッション未検出エラーが stderr に出ない、ターミナルタイトルが設定されない、再開ヒントがプロンプト入力と重なる）
* **Grep**: VS Code 拡張の自動更新や macOS App Translocation によって組み込みの ripgrep バイナリパスが古くなった場合の ENOENT エラーを修正。システムの `rg` にフォールバックしてセッション中に自己修復するように
* **`/btw`**: 使用するたびに会話全体のコピーをディスクに書き込んでいたバグを修正
* **`/context`**: Free space と Messages の内訳がヘッダーのパーセンテージと一致しない表示の不整合を修正
* **プラグイン**: 複数の問題を修正（`name:` フロントマターが重複しているとスラッシュコマンドが誤ったプラグインに解決される、`/plugin update` が `ENAMETOOLONG` で失敗する、Discover にすでにインストール済みのプラグインが表示される、ディレクトリソースプラグインが古いバージョンキャッシュから読み込まれる、`context: fork` と `agent` フロントマターフィールドがスキルに反映されない）
* **MCP メニュー**: `headersHelper` で設定された MCP サーバーに OAuth 固有のアクションが表示されていた問題を修正。代わりにヘルパースクリプトを再実行する Reconnect を表示するように
* **キーバインド**: `ctrl+]`、`ctrl+\`、`ctrl+^` が raw C0 コントロールバイトを送信するターミナル（Terminal.app、デフォルト設定の iTerm2、xterm）で動作しないバグを修正
* **`/login`**: OAuth URL のレンダリングに余分なパディングがつきマウスで選択しづらい問題を修正
* **レンダリング**: 非フルスクリーンモードでのちらつき、長いセッションで非フルスクリーンモードのターミナルスクロールバックが消去される問題、マウススクロールのエスケープシーケンスが稀にプロンプトにテキストとして漏れ込む問題を修正
* **設定**: `settings.json` の env 値に文字列ではなく数値が入っているとクラッシュするバグを修正
* **設定の同期**: `/add-dir --remember`、`/config` などのアプリ内設定の書き込みがインメモリのスナップショットを更新せず、削除したディレクトリがセッション中に取り消されないバグを修正
* **キーバインド**: Bedrock、Vertex、その他サードパーティプロバイダーでカスタムキーバインド（`~/.claude/keybindings.json`）が読み込まれないバグを修正
* **セッション継続**: `claude --continue -p` が `-p` や SDK で作成されたセッションを正しく継続できないバグを修正
* **Remote Control**: ワークツリーがセッションクラッシュ時に削
