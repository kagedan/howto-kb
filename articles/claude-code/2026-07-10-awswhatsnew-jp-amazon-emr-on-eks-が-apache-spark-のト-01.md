Amazon EMR on EKS が Apache Spark のトラブルシューティングエージェントをサポートするようになりました

Amazon EMR on EKS が Apache Spark のトラブルシューティングエージェントをサポートするようになりました。データエンジニアは、自然言語で EMR on EKS のジョブ失敗を診断でき、分散ログや Spark History Server のデータを手動で確認することなく、自動化された根本原因の分析と PySpark コードの推奨事項を受け取ることができます。このエージェントは、Spark History Server のデータ、分散エグゼキュータログ、およびクラスター設定を分析し、メモリエラー、データスケワリング、リソース競合、接続障害などの問題を特定します。このリリースにより、Spark トラブルシューティングエージェントは EMR on EC2、EMR Serverless、EMR on EKS のすべてのデプロイオプションをカバーするようになりました。このエージェントは、失敗したジョブの「AI でトラブルシューティング」オプションから EMR on EKS コンソールで直接アクセスできます。さらに、このエージェントは、Kiro、Claude Code、Cursor など、互換性のある AI コーディングエージェントを使用して MCP（Model Context Protocol）を介して利用できます。すべての操作は読み取り専用で、IAM ロールで認証され、AWS CloudTrail にログが記録されます。Amazon EMR on EKS 用の Spark トラブルシューティングエージェントは、SageMaker Unified Studio が利用可能なすべての AWS リージョンで利用できます。始めるには、EMR on EKS コンソールにアクセスするか、お好みの AI コーディングエージェントで MCP サーバーを設定してください。詳細なガイダンスについては、EMR トラブルシューティングエージェントのドキュメントを参照してください。
