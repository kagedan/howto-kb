Anthropicが提供するエージェント型CLIツール「Claude Code」を、現場で迷わず使い倒すためのリファレンスを網羅しました。基本操作から、初期設定、コスト管理、隠れたショートカットまで、エンジニアが知っておくべき全情報を結合しています。

## 1. CLIコマンド（ターミナル実行）

ターミナルから直接、Claude Codeの起動やプロジェクト操作を行うコマンドです。

| コマンド | 分類 | 機能・用途 |
| --- | --- | --- |
| `claude` | 基本 | Claude Code エージェントを対話モードで**起動** |
| `claude auth login` | 設定 | Anthropicアカウントへの**ログイン認証** |
| `claude auth logout` | 設定 | **ログアウト**：現在のセッションを終了し、認証情報を破棄 |
| `claude auth status` | 設定 | **ステータス表示**：ログイン中のメールアドレスや組織を表示 |
| `claude update` | 管理 | Claude Codeを**最新バージョンに更新** |
| `claude commit` | Git | 変更内容からメッセージを自動生成し**コミット** |
| `claude test` | 検証 | プロジェクトのテストを自動認識して**実行** |
| `claude mcp list` | 管理 | MCP一覧表示 |

## 2. スラッシュコマンド（対話セッション内）

`claude` 起動後、AIエージェントへの指示として `/` に続けて入力します。

### 🛠️ 開発・アクション系

| コマンド | 機能・用途 |
| --- | --- |
| `/edit <指示>` | **[中核]** ファイルの作成・修正を具体的に指示 |
| `/init` | プロジェクト初期化。`CLAUDE.md` を生成し**最適化** |
| `/ask <質問>` | コードベースに対する質問。解析のみで**編集はしない** |
| `/search <用語>` | プロジェクト全体から文字列やシンボルを**全文検索** |
| `/explain` | コードやプロジェクト構造の意図を**詳細解説** |
| `/lint` | Linterを実行し、静的解析エラーを**特定・修正** |
| `/v` | **外部検索**。ブラウザで最新ドキュメントや解決策を調査 |
| `/pr` | 現在のブランチの変更に基づき **Pull Request を作成** |
| `/<command>` | **シェルコマンド実行**。/ls -la のように直接コマンドを叩く |

### 📂 コンテキスト（認識範囲）管理

| コマンド | 機能・用途 |
| --- | --- |
| `/ls` | 現在AIが「読み込んでいる」**ファイル一覧を表示** |
| `/add <path>` | 特定のファイルをコンテキストに**明示的に追加** |
| `/drop <path>` | 不要なファイルをコンテキストから**除外**（トークン節約） |

### 🔄 セッション・コスト管理

| コマンド | 機能・用途 |
| --- | --- |
| `/usage` | **トークン使用量・制限の確認**。現在の消費状況を詳細表示 |
| `/cost` | セッション中に発生している**APIコストをリアルタイム表示** |
| `/compact` | 会話履歴を要約・圧縮し**トークン消費を抑制** |
| `/continue` | **前回のセッションを再開**し、文脈を維持する |
| `/resume` | **過去の会話セッションを再開**する機能 |
| `/clear` | 会話履歴を完全に消去し、**コンテキストをリセット** |

### ℹ️ ヘルプ・設定

| コマンド | 機能・用途 |
| --- | --- |
| `/help` | 利用可能な全コマンドのリストを表示 |
| `/config` | 動作設定（エディタ連携、権限等）を表示・変更 |
| `/bug` | Claude Code 自体の不具合を報告 |
| `/exit` | セッションを終了して通常のシェルに戻る |

## 3. 操作ショートカット（キーバインド）

入力の手間を省き、AIとの同期速度を上げるためのショートカットです。

| キー操作 | アクション | 用途 |
| --- | --- | --- |
| **`Ctrl + R`** | **コマンド履歴検索** | 過去に入力した指示を高速に再呼び出し |
| **`Esc` × 2回** | **メッセージ履歴表示** | これまでのAIとのやり取りを一覧表示 |
| **`Ctrl + V`** | **画像貼り付け** | UIデザイン図やエラー画面をコンテキストに送信 |
| **`Ctrl + C`** | **中断 / 終了** | 実行中のタスク停止、またはエージェントの終了 |

## 4. 設定とデータの保存先（重要）

トラブルシューティングや、複数端末での環境共有時に必要となるディレクトリパスです。

| 項目 | 保存先パス | 備考 |
| --- | --- | --- |
| **グローバル設定** | `~/.claude/` | 認証、ログ、セッション履歴の保存場所 |
| **プロジェクト指針** | `[Project Root]/CLAUDE.md` | /init で生成。コーディングルール等を記述 |
| **プロジェクト設定** | `[Project Root]/.claude.json` | プロジェクトごとの動作オプション設定 |
| **認証情報** | `~/.claude/logs/` | API利用のためのセッショントークン |

## ４. 運用 Tips

Claude Codeは単なるチャットではなく「実行権限を持つエージェント」です。

* 初期化: 新規プロジェクトではまず `/init` を行い、エージェントにプロジェクトの輪郭を掴ませること。
* コンテキスト最適化: `/ls` で現在の認識状況を確認し、不要なファイルは `/drop`、必要なファイルは `/add` することで推論精度が劇的に向上します。
* 最新性の維持: `claude update` を習慣化し、常に最新のモデル最適化とMCP機能の恩恵を受けること。
* 長いリファクタリングでは `/compact` を適宜行い、推論精度の低下を防ぐこと。
* 予算管理のために `/cost` を定期的に確認すること。

特に、新規プロジェクトにおける `/init` → `/ls` → `/edit` → `/test` → `/commit` という流れるようなワークフローの構築が、生産性を劇的に変える鍵となります。

## まとめ

ターミナルを離れることなく、設計からデプロイ準備までをAIと共に完結させる。これこそが現代のエンジニアに求められるDX（開発者体験）です。このコマンド集を武器に、開発スピードを極限まで高めてください。
