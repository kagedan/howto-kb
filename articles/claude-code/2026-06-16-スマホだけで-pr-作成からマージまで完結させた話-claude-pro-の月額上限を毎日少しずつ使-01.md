## はじめに

自分が使いたい家計簿アプリを個人で作っています。副業ということではなく、純粋に「ほしいものを自分で作っている」という感じです。

ただ、日々それなりに忙しいので、土日に予定が入ると「先週 Claude をほとんど使えなかった」という週がちょくちょく出てきます。月額で Claude Pro を契約しているのに、使わない日が続くとなんとなくもったいない。

もうひとつ、自分の性格的な問題として、**やらない期間が長くなると飽きてしまいます**。数日触れないと「あれ、どこまでやってたっけ」となって、再起動コストが上がる。毎日少しでもいいから触れていたい。

そういう背景から、**スマホからでも Mac と同じように Claude に作業させられないか**を試してみました。

もともと Mac では「設計書を見て気になることや追加したい仕様をコメントとして投稿する → Claude に実装させる → Claude にPR を作らせる」という流れで開発を進めていました。その仕組みについては以前記事にまとめているので、詳細はそちらをご覧ください。

<https://zenn.dev/tukaponx/articles/2e15f2b9dcbda6>

今回はその延長として、**同じことをスマホからできないか**を試した記録です。

---

## やりたかったこと

* 設計書に気づきをメモしたい → スマホのブラウザで設計書を開いてコメントを書く
* そのコメントを Claude に実装させたい → スマホの Claude アプリから起動する
* レビューして問題なければマージしたい → GitHub アプリで完結させる

**Mac を開かなくていい。** これが目標でした。

---

## 全体の仕組み

```
[スマホ] 設計書を開いてコメントを書く（Firestore に保存）
         ↓
[スマホ] Claude アプリでクラウドセッションを起動
         「CLOUD_WORKFLOW.md を読んで、open コメントを1件実装して PR を作って」
         ↓
[Claude クラウド] Firestore からコメント取得
               → 設計書 HTML と Swift コードを修正
               → GitHub に PR を作成
         ↓
[スマホ] GitHub アプリで PR を確認・レビューコメントを追加
         ↓
[スマホ] Claude アプリで再度クラウドセッションを起動
         「PR #2 のレビューコメントに対応して」
         ↓
[Claude クラウド] GitHub API でレビューコメントを取得 → 修正 → push
         ↓
[スマホ] GitHub アプリで Merge
```

## ポイントごとの解説

### 設計書コメントが「作業キュー」になる

設計書は Firebase Hosting で公開している HTML で、テキストを選択するとコメントを追加できる機能を自作しています。コメントは **Firestore にリアルタイム保存**されるので、スマホからでも PC からでも同じ設計書に書き込めます。

```
[スマホのブラウザで設計書を開く]
  → テキストを選択 → 「＋コメント」
  → Firestore に status: "open" で保存
```

この `status: "open"` のコメントが、Claude への作業指示キューになります。気づいたときにコメントを書き溜めておけば、あとで Claude がまとめて処理してくれます。

---

### クラウドセッションは月額プランで動く

ここが一番大事なポイントです。

スマホの Claude アプリからクラウドセッションを起動すると、**月額の Claude Pro プランの枠内で動作します**。従量課金の API クレジットは不要。

Mac を起動していなくても、スマホから起動してリポジトリをクローン → コードを修正 → PR を作成まで全部やってくれます。

なんでも上手いことやろうとすると、従量課金の API クレジットが必要ということで断念してしまうことが多かったので非常に嬉しいポイントでした。

---

### CLOUD\_WORKFLOW.md がスマホ向けの「手順書」

Mac の Claude Code ではスキル（`.claude/skills/`）という仕組みが使えますが、クラウドセッションはリポジトリをクローンするところから始まるのでスキルが読めません。

そのため、リポジトリのルートに **`CLOUD_WORKFLOW.md`** という手順書を置きました。Claude にこのファイルを読ませると、以降は自律的に動いてくれます。

```
CLOUD_WORKFLOW.md（抜粋）

Step 1: Firestore から open コメントを取得（認証不要・公開 URL）
Step 2: リポジトリをクローンしてブランチ作成
Step 3: 設計書 HTML と Swift の両方を修正
Step 4: コミット・push・PR 作成
```

スマホからは以下を貼り付けるだけで動きます（メモアプリにテンプレを保存しておく）：

```
CLOUD_WORKFLOW.md を読んで、
open の設計コメントを1件設計・実装修正して PR を作成してください。
```

---

### Mac からはコマンド一発

PC を開けるときは、ローカルスキルが使えてさらに簡単です：

```
/comment-fix     # Firestore の open コメントをまとめて PR 化
/pr-fix #2       # PR #2 のレビューコメントに一発対応
```

ローカルでは `gh` コマンドが認証済みなのでトークン入力も不要。

## やってみた感想

**「毎日少しでも触れる」が実現できました。**

通勤中に設計書を読んでコメントを書いて、昼休みに Claude アプリでセッションを起動して PR を作らせて、夜に GitHub アプリで確認してマージする——という流れが、Mac なしでできるようになりました。

1日の中でまとまった時間が取れなくても、スキマ時間を組み合わせると意外と前に進みます。やらない期間が続いて飽きる、というサイクルからも抜け出せた気がしています。

---

## 次回：レビュー体制を作りたい

現状の課題は「**コードの初回レビューを自分で実施している**」点です。Claude が作った PR を自分がレビューするわけですが、AIの対応の方が明らかに早いということだけでなく、コードを一切自分で触れていないのと、iOSアプリ開発は初心者のためSwiftのコードの理解スピードが遅いんです。

次回は、PR を作ったら **別の Claude セッションが自動でレビューコメントを投稿する** 仕組みを作ってみようと思っています。

* SwiftUI・SwiftData 固有のアンチパターンを拾うペルソナ設計
* セキュリティ観点・UX 観点など複数の視点でレビューする仕組み
* 指摘があれば自動修正してから人間がマージする体制

個人開発でも品質を担保しながら省力化を続けたい、その試行錯誤の記録です。
