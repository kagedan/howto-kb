Claude Codeで複数プロジェクトを並行開発していると、Claudeからの承認待ちや定型的な指示入力が頻繁に発生します。左手デバイス「XPPen ACK05」のボタンに定型文入力を割り当てて、ワンボタンで応答できるようにしました。

EnterやDeleteといった単純なキー操作はXPPenドライバ側で直接割り当てられます。本記事では、ドライバだけでは実現できない「定型文の入力」にフォーカスします。

## 全体の構成

```
ACK05のボタン
  → キーボードショートカットを発火
    → Automatorクイックアクションが実行
      → AppleScriptでテキスト入力を送信
```

ACK05のボタンにはキーボードショートカットを割り当てられるので、そのショートカットをトリガーにしてMac側でAutomatorの処理を実行する2段構成です。

## セットアップ

### 1. Automatorでクイックアクションを作成する

macOS標準のAutomatorを開き、「クイックアクション」を新規作成。「AppleScriptを実行」アクションを追加します。

#### 定型文を入力する

```
on run {input, parameters}
    set the clipboard to "ここに定型文を入れる"
    tell application "System Events"
        keystroke "v" using command down
    end tell
end run
```

クリップボードに定型文をセットし、`⌘V` でペーストする仕組みです。Claude Codeへの指示で頻繁に使うフレーズを登録しておくと便利です。

### 2. アクセシビリティ権限を設定する

AppleScriptでキー操作を送信するには権限が必要です。初回実行時に「Automatorにはキー操作の送信は許可されません」というエラーが出た場合は、以下を設定してください。

1. **システム設定** → **プライバシーとセキュリティ** → **アクセシビリティ**
2. **Automator** をオンにする（一覧になければ「+」から追加）
3. 必要に応じて **System Events** も追加

設定後、Automatorを再起動してください。

### 3. ショートカットキーを割り当てる

1. 各クイックアクションに名前を付けて保存（例：「CC\_定型文1」「CC\_定型文2」など）
2. **システム設定** → **キーボード** → **キーボードショートカット** → **サービス** → **一般** で、各アクションにショートカットを割り当てる（例：`⌃⇧F1`、`⌃⇧F2` など）
3. **XPPenドライバ**で、ACK05のボタンに同じショートカットを設定

## おまけ：ACK05のホイールでウィンドウ切り替え

ACK05にはダイヤル（ホイール）が付いています。ここにターミナルのウィンドウ切り替えのショートカットを割り当てると、ホイールを回すだけでプロジェクト間を移動できるようになります。

自分はGhosttyを使っているので、`goto_window:next` / `goto_window:previous` のキーバインドをホイールの左右回転に割り当てています。Ghosttyでのウィンドウ切り替え設定については別記事にまとめました。
