## 今日やったこと

ずっと予告していた「自分でコードを書く」をやりました。

書いたコードはこれだけです。

python

```
name = input("名前を入力してください: ")
print("こんにちは、" + name + "さん！")
```

2行です。

## 実際にやってみた

最初はターミナルに直接貼り付けようとしてエラーが出ました。

```
zsh: unknown file attribute
```

ターミナルはコードを実行する場所であって、コードを書く場所ではない。当たり前のことだけど、最初はわかっていませんでした。

次にnano（テキストエディタ）でファイルを作ろうとしたら、保存するためのcontrolキーが使えない状態になってしまいました。

結局、ターミナルから直接ファイルに書き込む方法で解決しました。

## 実行結果

※ここにスクリーンショットを貼る

```
名前を入力してください: えぬべえ
こんにちは、えぬべえさん！
```

動きました。

## 2行書いてわかったこと

Claude Codeが書くコードと、自分が書いたコードの違いがわかりました。

Claude Codeは1つの指示で100行以上書いてくれる。自分は2行書くのに10分以上かかって、途中でエラーも出た。

ただ、この2行は「なぜ動くのか」がわかります。input()で入力を受け取って、print()で出力する。Claude Codeが書いた200行のコードより、自分で書いた2行の方が理解できている。

**「使える量」と「わかる量」は全然違う。** 8日間ずっとそれを感じていたけど、今日初めて「わかる量」が少し増えた気がします。

このシリーズはプログラミング未経験の高校生がClaude Codeを毎日使い続けた記録です。

**次回：** 8日間の振り返りをします。
