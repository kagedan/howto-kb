Claude Coworkの正式リリース以降、「Claude Code触ってみたけど挫折した」って人の救出ルートが完全に変わった。

業務中枢にClaude Codeを置いてる俺が言うと、変な話に聞こえるかもしれない。

ただ、パソコン苦手な人ほど、Claude Codeじゃなくこっちから入った方がいい。
30分で経費1ヶ月分の集計が、口頭指示だけで終わる世界。

俺の手元の使い分けと合わせて、リプに整理する。

Claude Coworkの正体は、非エンジニア版Claude Code。

ターミナルもファイル操作も無し。
ブラウザを開いてプロジェクトを作って、口で指示するだけ。

プラグインの数が異常に多い。
Google Drive、Notion、Slack、Excel、ファイナンス系のデータ取得。標準で接続できる範囲がClaude Codeより一段広い。

Claude Codeで「ターミナル開いて、ファイル指定して」のステップで詰まる人が、Claude Coworkだと詰まらずに最後まで通る。

入口の段差を、Anthropicが意図的に下げてる。

実際の業務でどこまで動くか。

Google Driveに10月の請求書PDFを399件置いておく。
Claude Coworkに「このフォルダのPDFを全部読んで、経費一覧表をスプレッドシートで作って」と口で指示する。

Coworkがフォルダを開いて、399ファイルを順番に読んで、合計金額まで出して、Google Sheetsに整形してくれる。

ここまで30分。

人間でやると2日かかる作業。

経理のスタッフがいるなら、ここを丸ごとCowork側に振って、人間は「数字の意味の判断」だけに時間を回す。

業務の棚卸し方が変わる。

俺が一番ROIあると思ってるのが、スケジュールスキルとの合わせ技。

経費集計のフローを組んだあと、「毎月5日にこれを自動でやって」と追加で指示する。

Coworkが内蔵のスケジュールスキルを呼び出して、毎月5日のタスクとして登録する。

翌月5日、俺が何もしなくても、経費一覧表が自動で更新されてSlackに通知が飛んでくる。

「自動化を組み込んだ」ではなくて、「自動化を口頭で指示した」だけ。

人を雇うコストと比較する必要がもう無い。

非エンジニアの受講生にCoworkを最初に渡すと、3日で景色が変わる。

俺の周りで使い始めた人の事例だと、

副業会社員→Xの過去投稿を全部読み込ませて、自分の文体で投稿案を毎日自動生成。
個人事業主→ファイナンスプラグインで売上データを取得して、月次レポートを自動作成。
社員数3人の会社→Google Driveの請求書PDFから経費集計を毎月自動化、経理工数を月20時間圧縮。

Claude Codeの「学習コスト」で止まってた人が、Coworkで一気に動き始める。

まとめ。

Claude Cowork × Claude Code の使い分けは3軸。

第1。ブラウザ完結で動かしたい業務は Cowork。プラグイン経由でGoogle Drive・Notion・Excel・ファイナンスに接続。

第2。コーディング・Skill構築・ターミナル系の重い作業は Claude Code 側に残す。業務中枢は変わらず Claude Code。

第3。非エンジニアの社員・受講生・パートナーに最初に渡すのは Cowork。挫折ポイントが激減する。

Claude Codeで詰まってる人ほど、今日 Cowork を開いて、Google Drive 連携で1個タスクを動かすところから。

口頭指示で経費集計が終わる感覚を一度体験すると、業務の組み方が完全に変わる。
