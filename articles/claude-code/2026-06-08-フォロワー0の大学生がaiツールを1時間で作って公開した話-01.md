毎回、大学の授業でコメントペーパーを書かされるんですが、正直しんどい。  
「ChatGPTに貼ればよくない？」と思いつつ、毎回プロンプトを考えるのも面倒だし、出てくる文章がなんか外国語っぽい論理展開になりがちで使いにくかった。  
だったら自分で作ればいい、ということで作りました。

ChatGPTとの差別化で考えたこと  
最初に「ChatGPTに貼るだけじゃダメなの？」という問いに向き合いました。  
差別化として意識したのはこの2点です。

日本の大学っぽい文体に最適化する（要約→考察→疑問の順、です・ます調）  
PDFを貼るだけで動く（プロンプトを考えなくていい）

ChatGPTは汎用なので、日本のコメントペーパーの「型」を知りません。そこを最初から組み込むのが差別化になると考えました。

技術構成  
シンプルにしました。

HTML / CSS / Vanilla JS（フレームワークなし）  
Anthropic API（Claude Sonnet）  
Vercel Edge Function（APIキー管理）  
pdfjs-dist（CDNでPDF読み込み）  
Vercel（デプロイ）

サーバーなし、DBなし、ログインなし。静的サイトとして動いています。

一番大変だったこと  
技術的な詰まりはほぼなかったのですが、自分が作りたいものとAIが生成するコードのイメージのすり合わせが一番時間を使ったと思います。  
Claude Codeに指示を出すとき、「なんとなくこういう感じ」だと伝わらない。機能・デザイン・挙動を言語化して渡す必要があって、そこが一番頭を使いました。  
逆に言うと、言語化さえできれば実装はほぼ任せられます。実働1時間で公開まで行けたのはそのおかげです。

作ったもの  
レポート・コメントペーパー生成ツール  
<https://report-generator-git-master-mine-s-projects1.vercel.app/>  
機能一覧：

PDFアップロード（ドラッグ＆ドロップ対応）  
形式選択（コメントペーパー・小レポート・自由入力）  
字数選択（200・400・800字・自由入力）  
生成履歴（直近3件をブラウザ保存）  
レスポンシブ対応・スマホ動作確認済み

無料・ログイン不要で使えます。
