## 今北産業

2. それぞれのworktreeでClaudeを立ち上げて一次レビューさせる

## このやり方の何が良いか

### 同じ手順で無限に並列レビューを行える

wt pr > claude で一次レビュー (auto mode) > wt pr > ...

この手順自体はほぼ同じなので、無心で並列にAIレビュー依頼ができます。  
あとは順番に一次レビュー終わったPRからAIに内容聞きながら壁打ちレビューしていく感じです。  
ちなみに、レビューするだけなのでauto modeもそんなにリスク無いはず。

### PR紐づけが勝手に効く

worktreeのブランチがPR元のブランチそのままなので、Claude Codeを起動した時点で **自動で現在のブランチからPRを逆引き** してくれます。こちらが「PR #1234を」と指定しなくても、「レビューして」だけでそのPRのレビューしてくれるので便利。

かつ、Claude Codeのフッターには **自動で現在のブランチに紐づくPR番号とURLが表示される** のも地味に便利です。  
並列でPRレビューしまくってると「このPR何の機能だっけ。。」となるときも珍しくありませんし、コード上で対象のdiff見たい時にも便利です。  
その時もこのフッターリンクからPRに飛べるので地味に効きます。

![](https://static.zenn.studio/user-upload/6026f5656e43-20260423.png)

### 並列レビューしててもコンフリしない

worktreeなので、当たり前ですが PR Aを見ながらPR BでAIと壁打ちしていても互いに干渉しません。`wt go` でブランチ間の移動もfzfで一瞬。  
もちろん必要であればそのまま対象PRを修正してpushすることも可能です。

### ローカルでレビューしたい理由

もちろんGitHub上でのAIレビューも並行して回してます。  
ただ、一発目のAIレビューはそこそこの確率で判断を外すので、ローカルで何度もやり取りして「本当にその指摘は妥当か」「他の箇所への影響は」など壁打ちしながら詰めていきます。またその中で、PRの内容も人間がきっちり理解しながら進めるためにも壁打ち大事だと思います。
