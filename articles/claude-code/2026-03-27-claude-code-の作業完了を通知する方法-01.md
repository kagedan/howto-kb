## はじめに

Claude Code に長めのタスクを任せた後、こんな経験はありませんか？

* 「どうせ時間かかるだろう」と別の作業をしていたら、とっくに終わっていた
* ブラウザを見ていたら「確認が必要です」という入力待ちになっていて、数分ロスした
* 複数の Worktree を並列で動かしているとき、どれが終わったのか把握できない

Claude Code の Hooks 機能を使えば、これらのシチュエーションで **macOS の通知センターに通知を飛ばす** ことができます。

※ 最初は [cmux](https://cmux.com/ja) を使っていましたが、VSCodeで開いた Claude Code の通知はできないのでこの機能を作りました。

---

## 仕組みの概要

Claude Code には設定ファイル（`~/.claude/settings.json`）に **Hooks** を定義できる機能があります。  
Hooks は「特定のイベントが発生したときにシェルコマンドを実行する」という仕組みです。

今回利用するイベントは以下の 4 つです。

| イベント | タイミング |
| --- | --- |
| `Stop` | Claude が応答を完了したとき |
| `PreToolUse` (`AskUserQuestion` / `ExitPlanMode`) | Claude が確認入力待ちになるとき |
| `PermissionRequest` | 権限確認ダイアログが表示されるとき |
| `SessionEnd` | セッションが終了するとき |

これらのイベントをトリガーに通知スクリプトを呼び出します。

```
Claude Code でタスク実行中
    ↓  Stop イベント発火
~/scripts/notify_from_agent.sh '✅ 応答完了' が実行される
    ↓
terminal-notifier または osascript で macOS 通知を送信
    ↓
通知をクリックするとターミナルが前面に出る
```

## 通知スクリプトの実装

`~/scripts/notify_from_agent.sh` を作成します。

```
#!/bin/bash

# Claude からリポジトリ情報を取得
REPO_PATH="${PWD}"
REPO_NAME=$(basename "${REPO_PATH}")

# Git リポジトリの場合はより詳細な情報を取得
if git rev-parse --git-dir > /dev/null 2>&1; then
  REPO_ROOT=$(git rev-parse --show-toplevel)
  REPO_NAME=$(basename "${REPO_ROOT}")
  BRANCH=$(git branch --show-current 2>/dev/null || echo "unknown")
else
  REPO_ROOT="${REPO_PATH}"
  BRANCH=""
fi

# hook から渡される引数を取得
MESSAGE="${1:-タスクが完了しました}"

# 通知タイトルを構築（ブランチ名を含める）
NOTIFICATION_TITLE="${REPO_NAME}"
if [ -n "${BRANCH}" ]; then
  NOTIFICATION_TITLE="${REPO_NAME} (${BRANCH})"
fi

# TERM_PROGRAM でアクティベート先を決定
# case に任意のターミナルやIDEを登録する
case "${TERM_PROGRAM}" in
  vscode)       ACTIVATE_APP="com.microsoft.VSCode" ;;
  WarpTerminal) ACTIVATE_APP="dev.warp.Warp-Stable" ;;
  *)            ACTIVATE_APP="dev.warp.Warp-Stable" ;;
esac

# macOS の通知を送信
if command -v terminal-notifier > /dev/null 2>&1; then
  # terminal-notifier がある場合：クリックでターミナルを前面に出せる
  terminal-notifier \
    -title "${NOTIFICATION_TITLE}" \
    -message "${MESSAGE}" \
    -activate "${ACTIVATE_APP}" \
    -sound default
else
  # フォールバック：osascript
  osascript -e "display notification \"${MESSAGE}\" with title \"${NOTIFICATION_TITLE}\" sound name \"Glass\""
fi

# ログを記録
echo "[$(date '+%Y-%m-%d %H:%M:%S')] Notification sent: ${REPO_NAME} - ${MESSAGE}" >> "${HOME}/.claude_notifications.log"
```

作成後、実行権限を付与します。

```
chmod +x ~/scripts/notify_from_agent.sh
```

### ポイント

* **リポジトリ名 + ブランチ名がタイトルに入る**  
  `git (main)` のように表示されるため、複数の Worktree を並列実行していても「どれの通知か」が一目でわかります。
* **terminal-notifier があればクリックでターミナルを前面に出せる**  
  通知をクリックすると Warp（または VS Code）がアクティブになります。  
  `brew install terminal-notifier` でインストールできます。
* **ログが残る**  
  `~/.claude_notifications.log` に記録が残るので、後から確認できます。

## Hooks の設定

`~/.claude/settings.json` の `hooks` セクションに以下を追加します。

```
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "tool:AskUserQuestion|ExitPlanMode",
        "hooks": [
          {
            "type": "command",
            "command": "~/scripts/notify_from_agent.sh '⚠️ 確認が必要です'"
          }
        ]
      }
    ],
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "~/scripts/notify_from_agent.sh '✅ 応答完了'"
          }
        ]
      }
    ],
    "SessionEnd": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "~/scripts/notify_from_agent.sh '💤 セッション終了'"
          }
        ]
      }
    ],
    "PermissionRequest": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "~/scripts/notify_from_agent.sh '🔐 権限の確認が必要です'"
          }
        ]
      }
    ]
  }
}
```

### 各イベントの使いどころ

**`Stop`（応答完了）**  
最も使うイベントです。Claude が一通りの応答を終えたタイミングで通知が届きます。  
長めの実装タスクを頼んで他の作業をしているときに重宝します。

**`PreToolUse`（確認待ち）**  
Claude が `AskUserQuestion` ツール（ユーザーへの確認）や `ExitPlanMode`（プランの承認待ち）を呼ぼうとしたタイミングで通知されます。  
「あ、入力待ちになってた」という状況を防げます。

**`PermissionRequest`（権限確認）**  
Claude が権限確認ダイアログを表示したタイミングで通知されます。  
これも見逃すとタスクが止まるので、早めに気づけると便利です。

**`SessionEnd`（セッション終了）**  
セッションが完全に終了したときの通知です。  
バックグラウンドで動かしていたエージェントが終了したことを検知できます。

## 動作確認

スクリプトを直接呼び出してテストできます。

```
~/scripts/notify_from_agent.sh テスト通知
```

通知センターに表示されれば成功です。

## 並列 Worktree との相性が抜群

[複数のリポジトリを跨いでAIに作業させる方法](https://zenn.dev/articles/ee6bd254e8de32/edit) を使い複数の Worktree を並列で動かすと特に効果的です。

複数のタスクを並列で Claude Code に任せているとき、それぞれの Worktree から通知が届きます。  
通知タイトルに `frontend (feature/20)` `backend (fix/18)` のようにリポジトリ名とブランチ名が表示されるため、どの作業が終わったのかを即座に判断できます。

```
[通知センター]
frontend (feature/20)
✅ 応答完了

backend (fix/18)
⚠️ 確認が必要です
```

## まとめ

Claude Code の Hooks 機能を使った通知の仕組みをまとめます。

| 設定箇所 | 内容 |
| --- | --- |
| `~/scripts/notify_from_agent.sh` | 通知スクリプト（リポジトリ名・ブランチ名付き） |
| `~/.claude/settings.json` | Hooks 定義（Stop / PreToolUse / PermissionRequest / SessionEnd） |
| `terminal-notifier`（任意） | クリックでターミナルを前面に出す |

「タスクを頼んだら他の作業をする」スタイルで Claude Code を使っている人には特におすすめです。  
通知が来るまで別の作業に集中できるようになります。
