## きっかけ：社内向けに作ったら、想像以上に刺さった

きっかけは、自分の発表資料を作るのが面倒だったことです。

毎週ではないのですが、月に 1〜2 回、必要なタイミングでスライドを作ります。そのとき、内容を考える時間より「体裁を整える時間」のほうが長くなりがちでした。色を会社のブランドカラーに合わせ、ロゴを置き、フォントを揃え……と、本質的でない作業に時間を取られます。

そこで「Claude Code に資料ごと作らせればいいのでは」と考えて作ったのが、最初のバージョンでした。社内で使ってもらったところ、職種を問わず「使いたい」という反応が多く、フィードバックをもらいながら何度か直すことになりました。

![brand-pptxを使って作成したPPTXの例](https://static.zenn.studio/user-upload/80a6788eed44-20260630.png)

▲ brand-pptxを使って作成したPPTXの例（Microsoftの無料テンプレートを利用したサンプル）

とはいえ社内版は、会社のテンプレートやロゴが前提になっています。そこを **テンプレートと設定ファイル（`theme.json`）だけ差し替えれば動く**形に組み直したのが、今回公開する汎用版です。ねらいはシンプルで、**どの会社でも、自社のテンプレート PPTX に差し替えるだけで、そのブランドに沿ったスライドを生成できる**——そういう状態にすることです。

## 課題：AIにスライドを作らせると「それっぽいけど自社の資料じゃない」

生成 AI でスライドを作る手段はすでにいくつもあります。実際、社内でも「AI スライド生成サービスに課金してみた」という人がいました。

ただ、共通して聞こえてくるのは次のような声です。

* **テンプレートから外れる** — 指示しても、出てくるのは「それっぽいけど自社のフォーマットではない」スライド
* **ブランドがブレる** — 色やフォントが毎回微妙に違い、複数人で作ると統一感がなくなる
* **指示が効きづらい** — 「弊社のブランドカラーで」と書いても、思った通りの色・レイアウトにならない

要するに、**「内容は AI に任せたいが、見た目は会社のルールから外れてほしくない」** というニーズに、汎用ツールはあまり応えてくれませんでした。

そして自分で作り始めて、最初の方針でいきなり壁にぶつかりました。

## 解決策：テンプレートを"同梱"して、そこから離れられないようにする

最初は、プロンプトに「ブランドカラーは #〇〇、フォントは〇〇、レイアウトは〇〇」と細かく書けば制御できると思っていました。

**結論から言うと、テキスト指示だけでブランドを固定するのは無理でした。**

色は近い値になっても微妙にずれ、レイアウトは毎回変わり、回数を重ねるほどブレていきます。プロンプトをどれだけ厳密にしても、生成のたびに「だいたい合っているけど、ちょっと違う」スライドが出てくるのです。

そこで方針を変えました。**色やレイアウトを「言葉で指示する」のをやめて、「データとして固定する」** ことにしたのです。

具体的には、

1. レイアウト（カバー・セクション・コンテンツ等）を作り込んだ **テンプレート PPTX を用意し**、
2. 生成時はそれを **複製してレイアウトを選び**、
3. **プレースホルダーの中身だけを差し替える**

という流れにしました。装飾・配色・ロゴはテンプレート側にすべて入っているので、AI は「中身を入れる」ことに専念でき、見た目はテンプレートから離れようがありません。

### 仕組み

スライド生成は [python-pptx](https://python-pptx.readthedocs.io/) で行います。流れはこれだけです。

```
from pptx import Presentation

prs = Presentation("template.pptx")   # ① テンプレートを複製
delete_all_slides(prs)                # ② サンプルスライドを消す（定義は SKILL.md 参照）

slide = prs.slides.add_slide(prs.slide_layouts[0])  # ③ レイアウトを index で選ぶ（cover）
# ④ 中身を差し替える（実際は idx でプレースホルダーを引くヘルパーを使う）
slide.placeholders[0].text = "プレゼンテーションタイトル"

prs.save("output/presentation.pptx")
```

ポイントは ③ の `slide_layouts[0]` です。テンプレートに作り込まれたレイアウトを **index で選ぶ** だけで、そのレイアウトの装飾・配色・ロゴがそのまま付いてきます。AI がやるのは「どのレイアウトを使い、どんなテキスト・数値を入れるか」を決めることだけです。

### 使い方

Claude Code で、ふつうに日本語で頼むだけです。

トピック・対象者・スライド枚数などを対話で確認しながら、スライドを生成してくれます。

なお、スキル本体は `npx skills add` で入りますが、スライド生成には Python パッケージが必要です。`pip install python-pptx "markitdown[pptx]"` も入れておきましょう（スキルのインストールと Python 依存の導入は別、という点だけ注意です）。

ちなみに、生成された `.pptx` は **Google スライドで「ファイル」→「開く」→「アップロード」** すると Google Workspace 上でも開けます。PowerPoint を持っていなくても使えるので、地味に便利です。

## スキルの設計：ブランドを"データ"として固定する

ここからは、スキルをどう設計したかの話です。Claude Code のスキルは `SKILL.md` というファイルにプロンプト（指示）を書いて作りますが、今回いちばん意識したのは **「色やレイアウトを SKILL.md に直書きしない」** ことでした。

### 1. 見た目はテンプレートに、設定は `theme.json` に

ブランドの見た目（配色・装飾・ロゴ）は **テンプレート PPTX** が持ち、生成コードは触りません。  
そのうえで、**どのレイアウトをどの役割に使うか**や、**コードが描く部分（KPI カードなど）の色**といった設定だけを `theme.json` の 1 ファイルにまとめました。

```
{
  "template":  { "path": "assets/template.pptx" },                       // 使うテンプレート
  "layoutMap": { "cover": 0, "section": 2, "content": 1,
                 "contentVisual": 5, "ending": 7 },                      // 役割 → レイアウト index
  "colors":    { "accent": "5B6CFF", "bg": "F4F5FA", "dark": "131722" }, // コードが描くパーツの色
  "fonts":     { "headingCJK": "Noto Sans JP" },
  "typeScale": { "coverTitle": 44, "title": 26, "kpiValue": 40 }
}
```

こうしておくと、**誰が・何度頼んでも同じトンマナのスライド**が出てきます。そして他社が使うときは「テンプレートを差し替えて `layoutMap` を合わせるだけ」で済みます。

> これは、HTML 変換スキルを作ったときに CSS 変数（`--color-primary` など）でブランドを切り替えられるようにしたのと同じ発想です。「見た目を決めるものを 1 か所に寄せる」と、再利用がぐっと楽になります。

### 2. 既定テンプレートで動き、自社テンプレートで化ける

スキルには **クリーンな既定テンプレート**を同梱しています。`npx skills add` した直後、何も設定しなくても、ちゃんとデザインされたスライドが出てきます。

![Claude Codeでテンプレートを指定して作成してもらう例](https://static.zenn.studio/user-upload/624038a4244d-20260630.png)

そして本命は自社テンプレートへの差し替えです。カバー・セクション・コンテンツなどのレイアウトを作り込んだ **自社の PPTX** を渡せば、そのレイアウトの装飾・ロゴごとブランドを高精度に再現します。社内版はこの形で、会社の公式テンプレートを丸ごと活かしていました。

しかもこの差し替えは手作業でなくてもよくて、スキルが初回に **「自社のテンプレートはありますか？」** と一度だけ聞きます。テンプレートを渡せば、レイアウトを自動で読み取って `layoutMap` を設定し、以後そのブランドで生成します。**使う側は PPTX を渡すだけ**です。

### 3. 視覚パーツを自動で組む

「テキストだけのスライド」を防ぐために、よく使う視覚パーツをヘルパー関数として用意しました。

* **KPI カード** — 大きな数字 + ラベル（角丸カード）
* **進捗バー** — 達成率などを `0.0〜1.0` で
* **ステップフロー** — 番号付き円 + 矢印
* **テーブル** — ヘッダー付きの表

数値は KPI カードで、手順はステップフローで……と、内容に合った見せ方を選ぶと、ただテキストを流し込むより圧倒的に伝わりやすくなります。色はもちろん `theme.json` から取るので、パーツもブランドから外れません。

## 工夫したポイント

### テキストだけの制御は諦めた

これがいちばんの学びでした。「プロンプトを工夫すればなんとかなる」と最初は思っていたのですが、**ブランドの一貫性のような"見た目の正確さ"は、自然言語の指示と相性が悪い**です。

「だいたい正しい」では、資料としては使えません。なので、正確さが要るところ（配色・レイアウト）はデータとテンプレートで固定し、AI には「内容を考える」「適切なレイアウトを選ぶ」という、AI が得意な判断のほうを任せる——という役割分担にしました。

### 重なり・はみ出しを座標計算で防ぐ

図形を自動配置すると、要素が重なったりテキストがカードからはみ出したりしがちです。ここは地道に、

* テキストフレームには必ずマージンを設定する
* 図形内テキストは「左右中央 + 上下中央 + 折り返し」を 3 点セットで指定する
* 次の要素の位置は、前の要素の下端 + 余白から計算する（座標をハードコードで並べない）

といったルールを `SKILL.md` に明記しました。これだけで、生成物の崩れがかなり減ります。

### QA を必須手順にする

最後に、**「生成して終わり」にしない**ようにしました。生成した PPTX を [markitdown](https://github.com/microsoft/markitdown) でテキストに戻して中身を確認し、抜け・誤字・プレースホルダーの消し忘れがないかをチェックする——という QA サイクルを必須手順として組み込んでいます。

## 他社でも使えるように、ブランドを抜いて公開しました

社内版は会社のテンプレート・ロゴ・ブランドカラーが前提でした。公開にあたっては、それらの **ブランド固有の資産をすべて抜き**、中立の既定テンプレートだけを残した汎用版にしています。

### 自社ブランドにするには

いちばん簡単なのは、**Claude に自社テンプレート（.pptx）を渡すこと**です。前述のとおり初回に一度聞かれるので、渡せばレイアウトを自動で読み取って設定してくれます。

手動でやる場合も、触るのは基本 2 つだけです。

**① テンプレートを差し込む**

自社の `.pptx` を `~/.config/brand-pptx/` に置き、レイアウト index を確認して、同じ場所の `theme.json` の `layoutMap` に書き写します。

```
python3 tools/inspect_template.py ~/.config/brand-pptx/company.pptx
# [ 0] Title Slide
# [ 1] Title and Content
# [ 2] Section Header
#  ... のように index とレイアウト名が並びます
```

```
// ~/.config/brand-pptx/theme.json
{
  "template":  { "path": "company.pptx" },
  "layoutMap": { "cover": 0, "section": 4, "content": 7, "contentVisual": 8, "ending": 1 }
}
```

> 設定を `~/.config/brand-pptx/`（スキル本体の外）に置くので、スキルを更新（`npx skills add` の再実行など）しても自社テンプレートは消えません。設定しなければ同梱の既定テンプレートで動きます。

**② 描画パーツの色を合わせる（任意）**

KPI カードや進捗バーなど、コードが描く部分の色を `theme.json` の `colors` で調整します（`#` を付けない 6 桁 hex）。表紙やセクションの色はテンプレート側が持つので、ここでは触りません。ロゴはテンプレートに入っていれば、それで完了です。

## おわりに

正直に言うと、このスキルがやっていること自体は、技術的にはたいしたことではありません。「テンプレートを複製して、レイアウトを選んで、プレースホルダーに中身を入れる」——ほぼそれだけで、すごいモデルも凝ったアルゴリズムも使っていません。

それでも、作って公開するまでには学びがいくつもありました。

* **テキスト指示だけで"見た目"は固定できない**。正確さが要る部分（配色・レイアウト）はデータとテンプレートに任せ、判断が要る部分（何を・どう載せるか）を AI に任せる、という役割分担に落ち着きました。これはスライド以外にも応用が効きそうです。
* **社内ツールを「誰でも使える形」にするのは、機能を作るのとは別の仕事**でした。ブランド資産を抜く、テンプレートを持っていない人向けの既定を用意する、設定をスキルの外に逃がして更新で消えないようにする——そうした「配布物として成立させるための地味な配慮」のほうに、むしろ時間がかかりました。

「AI に資料を作らせる」と聞くと、つい「すごいデザインを一発で」を期待しがちですが、実際に効いたのは **"自社のフォーマットから外れないことを保証する"** という、ずっと地味な部分でした。その地味さを誰でも再現できる形にできたのが、今回いちばんの収穫だった気がします。

スキル本文（`SKILL.md`）は下のリポジトリにすべて入っています。`theme.json` と自社テンプレートを差し替えれば、あなたの会社のブランドにも対応できるはずです。ご自身のニーズに合わせて自由にカスタマイズしてお使いください。
