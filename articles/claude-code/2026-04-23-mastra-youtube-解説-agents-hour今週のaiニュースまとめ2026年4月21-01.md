## 詳細

### Jensen Huang ポッドキャスト：「Nvidiaは車ではない」

Jensen HuangがDarreshのポッドキャストに出演し、激しい議論を交わした。主な論点は中国へのチップ販売の是非だ。Jensenは「AIエコシステムには5つのレイヤーがあり、最も重要なのはAIアプリケーションだ」と語り、「Nvidiaは必要なだけ多く、しかし可能な限り少なくやるべきだ」という印象的な発言を残した。「我々は車ではない」という発言はSemi Analysisのツイートでバズった。

25歳のDarreshがNvidiaのCEOと対等に議論を交わしたこと自体も話題になっている。

### Allbirds：靴からAIへのピボット

靴ブランドのAllbirdsがAIピボットを発表し、株価が350%上昇した。ホストたちは「完全なハイプ」「どんなインフラやエンジニアがいるのか」と懐疑的だが、「今の世界ではみんながAI企業になる」と総括している。

### Vercelセキュリティインシデント：サプライチェーン攻撃の教訓

Vercelの従業員が、使用していたAIプラットフォーム顧客のcontext.aiの侵害を通じて被害を受けた。なお、このcontext.aiはOpenAIに買収された同名企業（Context AI）とは別の会社で、元の創業者がドメインを売却した先の企業だ。

> （訳註）本インシデントの概要は以下の通りです：まず、攻撃者がAIプラットフォーム「context.ai」を攻撃、内部の認証トークンなどの資格情報を窃取しました（この原因として社員PCのマルウェア感染（Lumma Stealer）などが指摘されています）。Vercelのある従業員は、このサービスにGoogle Workspaceアカウントでログインし、しかもOAuthでかなり強い権限（"Allow All"）を付与していました。攻撃者は盗んだトークンを使い、その従業員のGoogle Workspaceアカウントを乗っ取りました。その上で攻撃者は、その従業員のアカウントを足掛かりにVercelの社内システムへ侵入し、権限を拡大しながらVercel内部情報へアクセス、一部の環境変数や社員関連データを外部へ持ち出しました。

ゲストのRenee（Casco社、常時稼働型セキュリティエンジニアリングサービス）は、Vercelの透明性ある対応を高く評価した上で、以下の具体的な防御策を提示した：

* **Google Workspace管理者向け対策**: `admin.google.com/ac/owl/settings` にアクセスし、未構成サードパーティアプリの設定を「基本情報のみ要求するアプリを許可」に変更する
* これにより、名前・メール・プロフィール写真以上のスコープを要求するアプリは管理者の承認が必要になる
* Google Drive全ファイルのダウンロード・削除を要求するような怪しいスコープを管理者が個別に審査できる

これはアプリケーションセキュリティの問題が組織セキュリティの問題に波及し、さらに大きな攻撃経路を形成するという典型的な事例だ。**多層防御（Defense in Depth）** の実践が強調された。

### Lovableのデータ公開問題

Lovableで2025年11月以前に作成されたプロジェクトのソースコード、データベース認証情報、チャット履歴、顧客データが無料アカウントから閲覧可能だったと報告された。Nvidia、Microsoft、Uber、Spotifyの従業員もアカウントを持っているという。バグは48日前に報告されたが「重複」としてマークされ放置された。

Lovableの回答は「Public設定の文書化が不明確だった」「データ侵害ではない」というもので、Vercelの率直で技術的な対応とは対照的にリーガル寄りの印象だった。Reneeは「ユーザーの期待として、Google Workspaceアカウントでログインしたら少なくとも組織にスコープされていると想定する」と指摘し、急成長する企業がエンタープライズの期待値を学ぶ必要性を語った。

### AIとセキュリティの新時代：10xセキュリティエンジニア

AIがセキュリティの攻撃側・防御側の両方を10倍にする時代が到来している。生成AIによるバグバウンティのノイズ増大で、curlがバグバウンティプログラムを閉鎖するなどの動きが出ている。一方で正当な脆弱性を報告しても、相手側のAIが十分なコンテキストを持たず「正当でない」と判定されるケースもある。

Vercelの件では、セキュリティチームを通さず個々の従業員にメールで開示する者がおり、「暗号通貨での支払いを要求し、バグバウンティプログラムを迂回するのは恐喝だ」と厳しく批判された。正当な研究者は公式チャンネルを通じてすべての詳細を提供すべきだ。

### Claude Opus 4.7：期待と現実のギャップ

数日間メインで使用したホストの率直な評価は「4.6が好調だった時と比べて大きな改善は感じない」だ。2週間前の4.6が後退していたのに比べれば改善だが、ポイントリリースの差すら感じないという。

**トークン量の増加**が具体的に数値化されている：

* Simon Willsonの測定：同一タスクで4.6比**1.46倍**のトークン使用量
* Abishek Rayの測定：技術ドキュメントで1.47倍、claude MDファイルで1.45倍
* 画像では約**3倍**のコスト
* Anthropic公式ドキュメントでは1〜1.35倍としており、実測値を過小評価している可能性

Adam CADのZack Diveは「専用ハーネスを構築して長時間実行させるのがベスト」と提案しているが、ホストは「一つのモデルのためだけにハーネスを作るのは変だ」と疑問視した。Borisがツイートしたティップス：

* **オートモード**：許可プロンプトなし
* **/recaps**：エージェントの行動と次のステップの要約
* **フォーカスモード**：中間作業を隠して最終結果のみ表示
* **エフォートレベル設定**
* **自己検証の仕組み**を与える

また、Simon Willsonの比較で**Qwen 3.6 35BがラップトップでOpus 4.7よりも良いペリカンの絵を描いた**（SVG生成）という話題も。Opus 4.7が生成した自転車はフレームが逆で「物理的に走れない」もので、フラミンゴの一輪車もまともに描けなかった。strawberryの「P」の数も間違えており、「4.7は確実にまだAGIではない」と結論付けた。

### Theoのrant：Claudeデスクトップアプリの品質

Theo（人気テック系YouTuber）がClaudeデスクトップアプリを「根本的なレベルで壊れている」と痛烈に批判し、「1時間に40以上のバグを発見した」「4000億ドル企業がこれを出荷するのはばかげている」と述べた。

ホストの分析は興味深い：「人々がバグに目をつぶる理由は、Claudeがまだ最高のモデルだという認識があるからだ。Co-workで4分でPowerPointを作ってくれるのは魔法のように感じる。魔法のためなら摩擦は我慢する。」

Anthropicのプレイブックについて：「90%良いものを出す→モデルが魔法を感じさせる→使っている人がいたらバグを素早く修正→初日に試さなかった人はバグの存在すら知らない。」完璧を求めず速く出荷し続けるという戦略だ。

### Claude Design：Figmaに脅威

Claude Designが発表され、5800万ビューを記録した。Opus 4.7搭載で、プロトタイプ、スライド、ワンページャーをClaudeとの会話で作成できる。HTMLベースなので事後編集も可能。

ホストの友人が小規模ビジネスのチラシをワンショットで作成し「手動で作るよりはるかに良い」と感動した。一方でFigmaの株価は過去5日で3%、過去1ヶ月で約20%下落。Mike Krieger（元Instagram CPO）がこの発表の直前にFigmaの取締役会を辞任していたことも話題になった。

### Claude Code：ルーティンとリデザイン

Claude Codeにルーティン（Routines）が搭載された。一度設定すればスケジュールで自動実行できる。OpenClaudeのハートビート（Heartbeats）に似たスケジュールタスク機能だが、よりコンテキスト認識型。

また、Claude Code on Desktopがリデザインされ、複数セッションを並べてサイドバーで管理できるようになった。バグの報告もあるが、Anthropicの「素早くリリースして素早く修正する」プレイブックの一環だろう。

### OpenAI の動き：Agents SDK・Codex・GPT Rosalind

**OpenAI Agents SDK**の大規模アップデートでは、コンピュータ操作、スキル、メモリ、コンパクション（Compaction）など、モダンエージェントのすべてが統合された。自前でハーネスを構築せずにCodex風のエージェントを本番スケール可能にするという位置付けだ。

**Codex**のアップデートでは「コンピュータ操作（Computer Use）」機能が追加された。毎週300万人以上が利用するCodexが、ユーザーのPC上で日常的に使うツールやアプリと連携可能になった。

**GPT Rosalind**は生物学、創薬、トランスレーショナルメディシン向けのフロンティア推論モデルだ。Anthropicも最近バイオ分野の人材を買収しており、OpenAI対Anthropicの全面競争（「Microsoft対Apple」に例えられた）がバイオ分野にも拡大している。

### クイックヒット

### Qwen 3.6 35B A3B：小型オープンモデルの台頭

Qwen 3.6 35B A3Bがオープンソース公開された。総パラメータ350億、アクティブ30億のスパースモデル。Apache 2ライセンスで、エージェンティックコーディングではアクティブサイズの10倍のモデルと同等性能を持ち、マルチモーダルのthinking・non-thinkingモードに対応する。

ローカルモデル実行に興味がある人向けに **runthislm.com** が紹介された。ハードウェアを入力するとどのモデルが動くかを教えてくれるサービスだ。Code Rabbitによると推論の40%以上がオープンソースで実行されており、推論コストを電力・ハードウェアコストにトレードオフするのが合理的なケースも多い。IoTや定義の明確な生産性タスクには小型モデルで十分な場合がある。

### 原始人トークン削減法（Caveman）

Claude Codeのスキルとして話題になっている「原始人のような簡略英語でプロンプトを書く」アプローチが紹介された。「Why use mini token when few token do trick?（少ないトークンで済むなら、なぜ多くのトークンを使う？）」のように冗長な文法を省略し、**トークン使用量を65%削減**するというものだ。

Mastra Codeのオブザベーショナルメモリ（Observational Memory）で実験中で、「今のところ目に見える劣化はなく、ただ動く」と報告された。英語のコンテキスト手がかりに優れたLLMは、文法が崩れていても周囲の単語から正確に意味を推測できるため、人間でも理解可能な程度の省略であればモデルの性能に影響しないという主張だ。
