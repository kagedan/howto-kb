Claude Codeには、複数の役割を持つAIを協調させて動かす仕組みがあります。

「サブエージェント」と呼ばれる機能です。

この仕組みを使うと、1人のAIに全部やらせるより精度の高い出力が得られます。

---

## エージェントとは何か

「エージェント」は、特定の役割と指示書を持ったAIの単位です。

Claude Codeでは、.claude/agents/ フォルダにテキストファイルを置くことでエージェントを定義できます。

ファイルの中に「このエージェントは何をする役割か」「何をしてはいけないか」「どう判断するか」を書きます。

---

## サブエージェントとして呼び出す

CLAUDE.mdや別のエージェントから、「〇〇エージェントを呼び出して」という指示を出せます。

たとえばこういう流れです。

1. メインのAI（CLAUDE.md）が「記事を作る」という指示を受ける
2. plannerエージェントを呼び出してテーマ選定を依頼する
3. plannerの結果をoutlinerエージェントに渡して構成設計を依頼する
4. outlinerの結果をwriterエージェントに渡して本文執筆を依頼する
5. writerの結果をreviewerエージェントに渡してチェックを依頼する
6. 合格が出たらWordPressに保存する

この連鎖が「新規記事の作成をお願いします」という一言で自動的に動きます。

---

## ファイルの作り方

エージェントのファイルはテキストファイルです。

冒頭に簡単な設定（名前・説明・使うモデル）を書いて、その下に指示内容を日本語で書くだけです。

```
---
name: writer
description: 記事の本文を執筆する
---
```

## 役割

outlinerが作成した構成書をもとに、本文を執筆します。  
構成の変更・テーマ変更は行いません。

## 禁止事項

* Markdownのテーブル記法を使わない
* 個人体験を捏造しない  
  …

プログラミングは不要です。日本語の指示書です。

---

## 役割を分けることで何が変わるか

1人のAIに全部やらせると、「構成を考えながら文章も書く」という状態になります。

サブエージェントで分担すると、writerは「書くこと」だけに集中できます。

構成の良し悪しはoutlinerが責任を持つ。文章の品質はwriterが責任を持つ。ルール違反のチェックはreviewerが責任を持つ。

責任の所在が明確になるほど、それぞれの精度が上がります。

---

## 何エージェントが必要か

最低限の構成は4つです。

planner・outliner・writer・reviewerで、テーマ選定から品質チェックまでをカバーできます。

必要に応じて追加もできます。

「リライト専用のwriter」を別に用意する。「カニバリゼーションチェック専用のエージェント」を用意する。

ただし増やすほど管理が複雑になります。最初は4エージェントで運用を安定させてから拡張する方が現実的です。
