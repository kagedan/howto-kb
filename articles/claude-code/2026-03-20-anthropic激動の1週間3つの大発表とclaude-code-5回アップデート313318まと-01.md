はじめに

2026年3月13日〜18日の1週間、Anthropicが立て続けに動きました。

企業としての大型発表が3本。Claude Codeのアップデートが5回。正直、追いきれないくらいのペースです。

僕自身もClaude Codeを毎日使っていて、「あれ、昨日と挙動が違う？」と思ったらバージョンが上がっていた、ということが何度かありました。

全部を細かく追う必要はないですが、「これだけは知っておいた方がいい」 というポイントを整理しました。

大発表 その1：81,000人のAIユーザー調査（3/18）

![](https://assets.st-note.com/img/1773967333-YLBsKoCRwEQA2HPvNfFdSJaW.png?width=1200)

Anthropicが、Claude.aiのユーザー81,000人に「AIに何を求めているか」を聞いた調査結果を公開しました。

ポイント:  
・159カ国、70言語で実施された、おそらく過去最大規模のAIユーザー調査  
・面白いのは調査方法。アンケート用紙ではなく、AIが聞き手になって対話形式でインタビューした  
・つまりClaude自身がインタビュアーになって、ユーザーの本音を引き出した

これ、ただの「満足度調査」ではなくて、「人々がAIに何を夢見ていて、何を恐れているか」 という深い内容です。

僕が気になったのは、この調査でAIが聞き手を務めたこと自体。「AIについての調査をAIがやる」って、ちょっとメタですよね。でもこの方法だからこそ、81,000人という規模で「深い対話」ができた。従来のアンケートでは不可能だったはずです。

大発表 その2：Claude Partner Network に1億ドル投資（3/12）

![](https://assets.st-note.com/img/1773967342-J8CPoafBx4dkVHwliLhKTFZM.png?width=1200)

Anthropicが、企業向けにClaudeの導入を支援するパートナー企業ネットワークに1億ドル（約150億円）を投資すると発表しました。

ポイント:  
・Accenture、Deloitte、Cognizant、Infosysなど大手コンサル・IT企業が参加  
・パートナー向けに技術研修、営業支援、共同マーケティングを提供  
・「Claude Certified Architect」 という初の技術認定資格を開始  
・Code Modernization（古いプログラムを新しい技術で書き直す支援ツール）キットをパートナーに提供  
・ネットワークへの参加は無料

これは僕たちのような個人ユーザーには直接関係なさそうに見えますが、実は重要です。Claudeが企業に本格導入される → ユーザー数が増える → 開発リソースが増える → 機能が良くなる という好循環に入る兆し。

特に「Claude認定資格」の登場は、Claudeのスキルが「履歴書に書ける」時代が来る ことを意味しています。

大発表 その3：Anthropic Institute 設立（3/11）

が社会に与える影響を研究する専門研究機関「Anthropic Institute」 を設立しました。

ポイント:  
・共同創業者のJack Clarkが「Head of Public Benefit（公益責任者）」として統括  
・3つのチームで構成：  
・Frontier Red Team — AIの限界を見つけるストレステスト  
・Societal Impacts — AIの実際の使われ方を研究  
・Economic Research — AIが雇用・経済に与える影響を追跡  
・Google DeepMindの元研究ディレクター Matt Botvinick氏が参画

「AIを作る会社が、AIのリスクも自分で研究する」という姿勢。これはAnthropicならではだと思います。

Claude Code 5回アップデート（v2.1.75 → 2.1.79）

ここからは僕たちユーザーに直接関係する話。たった6日間で5バージョンのアップデートが出ました。

・変更: 1Mコンテキストがデフォルトに ／ バージョン: v2.1.75（3/13） ／ なぜ重要？: Opus 4.6で100万トークン（本1冊以上）を標準で保持できるように  
・変更: 出力トークン上限が64k→128kに ／ バージョン: v2.1.77（3/17） ／ なぜ重要？: Claudeが1回で書ける文章量が倍に  
・変更: /effortコマンド追加 ／ バージョン: v2.1.76（3/14） ／ なぜ重要？: 思考の深さをlow/medium/high/maxで切り替え可能に  
・変更: VS Codeから/remote-control ／ バージョン: v2.1.79（3/18） ／ なぜ重要？: VS Codeのセッションをclaude.ai/codeに橋渡しできるように  
・変更: 起動メモリ18MB削減 ／ バージョン: v2.1.77-79 ／ なぜ重要？: 体感できるレベルで起動が軽くなった

個別の注目アップデート

v2.1.75（3/13）— 1Mコンテキストのデフォルト化

これが一番大きい。Max/Team/Enterpriseプランのユーザーは、Opus 4.6で自動的に100万トークンのコンテキストが使えるようになりました。

以前は200Kトークンが上限で、長い作業をするとClaude Codeが過去の会話を忘れていく問題がありました。100万トークンあれば、かなり長いセッションでも文脈を保てます。

v2.1.76（3/14）— /effortコマンド

effort low で軽く、/effort high で深く考えさせる。前回の記事でも紹介しましたが、このバージョンでスラッシュコマンドとして正式に使えるようになりました。

v2.1.77（3/17）— 出力トークン上限128k

Opus 4.6のデフォルト出力トークンが64kに。上限は128k。つまり1回の応答で書ける文章量が大幅に増えた。長いコードや記事を一度に生成するときに効いてきます。

v2.1.78（3/17）— セキュリティ修正

サンドボックスが無効になっている場合の警告が見えるように修正。地味ですがセキュリティ的に重要。.gitフォルダへの書き込み権限の問題も修正されました。

v2.1.79（3/18）— VS Code連携強化

remote-control でVS CodeのセッションをClaude.ai/codeにブリッジできるように。AI生成のセッションタイトルも追加。

結局、僕たちは何をすればいい？

![](https://assets.st-note.com/img/1773967368-9AzYgHxoQBpWUb8ZSywFa1Nu.png?width=1200)

すぐやること:  
1. Claude Codeのバージョンを確認（claude --version）。2.1.79になっていなければ更新  
2. /effort コマンドを試す。普段はmedium、複雑な作業はhighに

知っておくこと:  
・1MコンテキストはMax/Team/Enterprise限定。Proプランの場合は追加料金が必要  
・出力128kは設定不要で自動適用。長文生成が以前より途切れにくくなっている  
・Claude認定資格が始まった。将来のキャリアに「Claudeが使える」は武器になる可能性あり

気にしなくていいこと:  
・Anthropic Instituteの研究論文（まだ出ていない）  
・Partner Networkの詳細（企業向け。個人は直接関係なし）
