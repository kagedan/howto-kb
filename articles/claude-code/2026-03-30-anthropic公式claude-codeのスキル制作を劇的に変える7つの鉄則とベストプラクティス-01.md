# Claude Codeのスキル制作が劇的に変わる！AgentSkills公式の7つの鉄則を解説

## はじめに

AIエージェント（Claude Codeなど）を日常業務や開発に組み込む際、独自の「スキル」をどのように設計・記述するかが、エージェントの精度を大きく左右します。

本記事では、AgentSkills.ioおよびAnthropic公式ドキュメントで推奨されている「スキル制作のベストプラクティス」について、実践しやすい形に落とし込んでまとめました。

AIエージェントの挙動が安定しない、あるいはより高度な自動化ツールを作りたいと考えている方の参考になれば幸いです。

![Claude API Documentation](https://raw.githubusercontent.com/Inosuke0718/Zenn/main/images/claude-api-docs.png)

---

## 1. 汎用知識ではなく「ドメイン知識（専門知識）」から始める

AIエージェントに曖昧で抽象的な指示を出すのはNGです。  
「適切にエラー処理をして」といった汎用的なプロンプトではなく、 **具体的な専門用語やプロジェクト固有のルール** を教え込むことが重要です。

* **具体例:** UI生成を依頼する際、単に「いい感じの画面を作って」ではなく、「モーダル」「サイドバー」といった具体的なコンポーネント名を定義しておく。
* **メリット:** エージェントが連想するイメージの解像度が上がり、出力のブレや迷いがなくなります。

## 2. 実際のタスクから抽出し、実行して磨き上げる

スキルは「一度書いて終わり」ではありません。日常的に使っているプロンプト、社内ドキュメント、あるいは過去のセッションログから共通パターンを見つけ出してスキル化するのが効果的です。

また、作成後は **「実行トレース」** を必ず確認しましょう。

* AIが意図した通りにツールコールを行っているか？
* 余計な出力をしていないか？

期待通りに動くまで、テストとプロンプトの洗練を繰り返す泥臭い作り込みが、最終的な精度を決定づけます。

## 3. コンテキストを賢く使う（段階的開示）

スキルの説明文（`skill.md`など）に、初めから全ての情報を詰め込むのは悪手です。AIが情報を処理しきれず、迷いを生む原因になります。

ここで重要なのが **「プログレッシブ・ディスクロージャー（段階的開示）」** の考え方です。

* **skill.md** の本体はシンプルに保つ（5000トークン以下推奨）。
* 詳細な仕様、リファレンス、実行スクリプトなどは別のファイル（アセット）に切り出す。
* 「必要になったタイミングで○○のファイルを読み込むこと」と指示しておく。

これにより、無駄なトークン消費を抑えつつ、AIのパフォーマンスを最大化できます。

## 4. 適切な「関数スコープ」で設計する

AIのスキルは、プログラムの「関数」と同じように一貫性のある適切な粒度で設計しましょう。

* ❌ **広すぎるスコープ:** 「何でもできる文章生成スキル」などは、AIが何をすべきか迷ってしまう。
* ❌ **狭すぎるスコープ:** 単一の簡単なコマンドを叩くだけのスキルは実用性に欠ける。
* ⭕ **適切なスコープ:** 「UIデザイン生成」「サムネイル画像生成」「特定形式のスライド生成」など、目的が明確な単位で切り分ける。（`skill.md`は500行以内が目安）

## 5. 脆弱性（リスク）に比例したコントロールを行う

AIにどこまで自由度を与えるかは、そのタスクの「失敗時のリスク」に比例させるべきです。

* **リスクが高い・確実にこなしてほしいタスク:**  
  自由度を与えず、スクリプト化して **決定論的に（必ず手順通りに）** 実行させる。バリデーション（検証）プロセスを組み込む。
* **柔軟な視点が必要なタスク（コードレビューなど）:**  
  チェックすべき観点だけを与え、順番や具体的なアプローチはAIの自己判断に委ねる。

何でもかんでも自由にするのではなく、タスクの性質に応じた「手綱の引き方」が求められます。

## 6. 効果的な指示パターン（フォーマットや手順）を活用する

単に「こうしてほしい」と宣言するのではなく、AIが確実に実行できる「手順」や「型」を用意してあげることが重要です。

* **落とし穴セクションの設置:** スキル内でAIが引っかかりやすいミスを事前に明記し、禁止する。
* **出力フォーマットの固定:** JSON、Markdown、YAMLなどの明確なテンプレートを用意して出力形式を縛る。
* **検証ループの構築:** 「計画作成 → 検証 → 実行 → 再検証」といったサイクルを回すよう指示する。

## 7. 仕様への最適化（メタデータの重要性）

CLIツールでスキルを定義する場合、`skill.md`上部のフロントマター（YAML形式のメタデータ）に記述する `Name` と `Description` が極めて重要です。

AIエージェントは、この `Description` を読んで「いつ、どのような目的でこのスキルを呼び出すべきか」を判断します。ここが曖昧だと、せっかく作ったスキルが適切なタイミングで発動しません。 **「どういうタイミングで使うのか」を具体的かつ簡潔に記述** しましょう。

## まとめ

優れたAIエージェントのスキル開発は、単なるプロンプトエンジニアリングの枠を超え、 **システム設計やワークフロー構築** に近いアプローチが必要になります。

1. ドメイン知識の注入
2. スコープの最適化
3. 情報の段階的開示
4. リスクに応じた制御

これらを意識してスキルを記述・改善していくことで、Claude CodeをはじめとするAIエージェントは、より強力で信頼できる「開発パートナー」へと進化します。ぜひご自身のエージェント開発に取り入れてみてください。
