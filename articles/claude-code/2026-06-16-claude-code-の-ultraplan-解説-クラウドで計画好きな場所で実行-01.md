## はじめに：プランモードの壁

Claude Code の**プランモード**を使ったことがある人は多いでしょう。実装前にファイル構成や方針を整理し、承認してから着手する。合理的なワークフローです。でも、こんな場面でひっかかりませんか。

> 計画に 10 分かかるのに、その間ターミナルが使えない。Web の UI でコメントを付けたり、チームメンバーに見せたりできたらいいのに。

ローカルのプランモードは「計画」と「実行」が同じターミナルに閉じています。計画中は別の作業ができず、レビューもテキストベースのやりとりに限られます。この壁を取り払うのが **Ultraplan** です。

一言でまとめると、Ultraplan とはこういう機能です。

> Ultraplan は、Claude Code の計画フェーズをクラウドに移し、リッチな Web エディタでレビュー・編集したうえで、クラウドかローカルか好きな場所で実行できる機能です。

ローカルのプランモードが「**速い・手軽・ターミナル占有**」なのに対し、Ultraplan は「**深い・視覚的・ターミナル解放**」という位置づけです。

## 核心は「計画と実行の分離」

このセクションでは、ローカルプランモードのどこに限界があり、Ultraplan がそれをどう解決するかを整理します。

ローカルのプランモードでは、計画の作成・レビュー・実行がすべて同じターミナルセッションで完結します。手軽ですが、計画が大きくなるほど以下の問題が出てきます。

1. **ターミナル占有**: 計画を練っている間、他の作業ができない
2. **レビュー体験の限界**: テキストベースのやりとりでは、特定のセクションへの部分的なフィードバックがしにくい
3. **共有の壁**: チームメンバーに計画を見せて意見をもらう手段がない

Ultraplan はこれを**クラウドで計画 → Web でレビュー → 実行場所を選択**という 3 ステップで解決します。

### 仕組み：クラウド計画 → Web レビュー → 実行

`/ultraplan` を実行すると、以下のフローがクラウド上で動きます。

```
CLI からプロンプトを送信
  ↓
【クラウド計画フェーズ】
  クラウドの専用セッションで
  リポジトリを分析し、実装計画をドラフト
  ↓
【Web レビューフェーズ】
  ブラウザで計画を確認。インラインコメント・
  絵文字リアクション・アウトラインサイドバーで編集
  ↓
【実行オプション選択】
  ├─ クラウドで実行 → PR を作成
  ├─ ターミナルにテレポート → ローカルで実行
  └─ キャンセル → ファイルに保存して後で使用
```

## ローカルプランモードとの比較

「いつものプランモードで十分では？」と思うかもしれません。判断基準は変更の規模とレビューの必要性です。

|  | ローカルプランモード | Ultraplan |
| --- | --- | --- |
| 計算場所 | ローカルのターミナル | Anthropic のクラウド |
| ターミナル | 計画中は占有される | 解放される |
| レビュー体験 | テキストベースの対話 | Web エディタ（コメント・リアクション） |
| 修正指示 | プラン全体への返信 | 特定セクションへのインラインコメント |
| 実行場所 | ローカルのみ | クラウド or ローカルを選択可能 |
| 共有 | セッション URL の共有のみ | Web UI でチームレビュー |
| 計算リソース | ローカルマシン | クラウドの専用セッション |

**ローカルプランモードが向く場面**: 小〜中規模の変更、すぐ実装に入りたいとき、オフラインで作業したいとき。

**Ultraplan が向く場面**: 大規模リファクタの方針決め、複数アプローチの比較検討、チームメンバーに計画を見せてフィードバックをもらいたいとき。

## 使い方

ここでは起動からレビュー・実行までの一連の操作を 4 ステップで説明します。

### Step 1: Ultraplan を起動する

起動方法は 3 つあります。

| 方法 | 例 |
| --- | --- |
| スラッシュコマンド | `/ultraplan 認証モジュールをリファクタして` |
| キーワード | `ultraplan 認証モジュールをリファクタして` |
| ローカルプランからの昇格 | ローカルで作成済みのプランをクラウドに送って深掘り |

スラッシュコマンドでもキーワードでも動作は同じです。プロンプトに「ultraplan」と含めるだけでクラウドセッションが開始されます。

### Step 2: ステータスを確認する

クラウドで計画が進行中のあいだ、CLI にはステータスが表示されます。

| ステータス | 意味 |
| --- | --- |
| `◇ ultraplan` | リサーチ・計画をドラフト中 |
| `◇ ultraplan needs your input` | 質問あり。入力待ち |
| `◆ ultraplan ready` | 計画完了。レビュー可能 |

完了すると通知が届くので、セッション URL を開いてレビューに進みます。

### Step 3: Web でレビュー・コメントする

ブラウザでセッション URL を開くと、計画が Web エディタに表示されます。ローカルのテキストベースのやりとりとは異なり、以下の操作が可能です。

* **インラインコメント**: 計画の特定セクションに直接コメントを付けてピンポイントで修正指示
* **絵文字リアクション**: 各セクションに承認・懸念をシグナル
* **アウトラインサイドバー**: 計画全体の構造をナビゲーション

### Step 4: 実行オプションを選択する

計画に納得したら、まずクラウドで実行するかローカルに戻すかを選びます。

| 選択 | 説明 | 向いているケース |
| --- | --- | --- |
| クラウドで実行 | クラウドセッションでそのまま実装し、PR を作成 | GitHub 上で完結する変更 |
| ターミナルにテレポート | 計画をローカルに送り返す | ローカル環境への依存がある変更 |

「テレポート」を選ぶと、ターミナル側でさらに 3 つの選択肢が表示されます。

| サブ選択肢 | 説明 |
| --- | --- |
| Implement here | 現在のセッションで計画を実行 |
| Start new session | 新しいセッションで計画を実行 |
| Cancel | 計画をファイルに保存して後で使用 |

## 要件と制約

導入前に確認すべきバージョン・認証・環境の条件をまとめます。Ultraplan は Ultrareview と同じく「Claude Code on the web」のインフラ上で動くため、環境に制約があります。

**バージョン要件**

| 条件 | 最低バージョン |
| --- | --- |
| 基本動作 | v2.1.91 |
| クラウド環境の自動作成 | v2.1.101（推奨） |

**利用可能な環境**

| 環境 | 利用可否 |
| --- | --- |
| Claude.ai 直接認証（Pro / Max） | 利用可能 |
| Amazon Bedrock | 利用不可 |
| Google Cloud Vertex AI | 利用不可 |
| Microsoft Foundry | 利用不可 |
| Zero Data Retention 有効な組織 | 利用不可 |

**その他の要件**

* GitHub でホストされたリポジトリが必要
* Claude Code on the web のアカウントが必要

## 「ultra」ファミリーの全体像

Ultraplan 以外にも「ultra」を冠する機能が 2 つあります。どれをいつ使うかの判断基準を整理します。

| 機能 | 目的 | フェーズ | 実行場所 |
| --- | --- | --- | --- |
| **ultraplan** | 設計・計画 | 実装着手前 | クラウド |
| **ultracode** | マルチエージェント実行 | 実装中 | ローカル |
| **ultrareview** | ディープコードレビュー | マージ前 | クラウド |

### ultracode との違い

ultracode は Dynamic Workflow を使い、最大 1,000 エージェントを実行する機能です（同時並列は最大 16）。大規模な実装タスク向けで、計画ではなく実行を担います。

「ultraplan で設計 → ultracode で大規模実装 → ultrareview で最終確認」という流れが、ultra ファミリーの想定する開発サイクルです。

## 実践 Tips

Ultraplan を効果的に使うための具体的なパターンを紹介します。

**大規模リファクタの方針決め**

複数のアプローチ（段階的移行 vs. 一括置換、ライブラリ A vs. B）を比較したいとき、Ultraplan で各案の長所・短所を並べて計画し、Web UI でチームメンバーにコメントしてもらうと、方針決定が格段にスムーズになります。

**ローカルプランからの昇格**

最初はローカルのプランモードで軽く骨格を作り、「これはもっと深掘りが要る」と判断したらクラウドに昇格する、という段階的なアプローチも有効です。

**計画の保存と再利用**

キャンセルを選ぶと計画がファイルに保存されます。後日のセッションでそのファイルを参照しながら実装に着手したり、別の文脈で計画を再利用したりできます。

## トラブルシュート

Ultraplan 利用時によくある問題と対処法をまとめます。

| 症状 | 原因 | 対処 |
| --- | --- | --- |
| `/ultraplan` が `Unknown command` になる | Claude Code のバージョンが古い、または Vertex AI / Bedrock 経由で利用している | `claude --version` で v2.1.91 以上を確認。クラウドプロバイダー経由の場合は Ultraplan 自体が無効化されるため、Claude.ai 直接認証に切り替える |
| クラウドセッションが開始しない | Claude Code on the web のアカウントが未設定、または GitHub リポジトリでない | Claude.ai アカウントにログインしているか確認（`claude` コマンドで認証状態を確認可能）。ローカル専用リポジトリは GitHub にプッシュしてから実行 |
| 計画が途中で止まる（`needs your input` のまま） | Claude が実装方針について判断できず質問している | セッション URL をブラウザで開き、質問に回答する |
| テレポート後にローカルで計画が反映されない | CLI セッションが終了している | テレポートは実行中の CLI セッションに計画を送り返す。セッションを閉じていた場合は、保存された計画ファイルを新しいセッションで読み込む |

## まとめ

Ultraplan は、Claude Code の計画フェーズをクラウドに移し、リッチな Web エディタでレビューしてから好きな場所で実行できる機能です。

ローカルのプランモードとの位置づけを整理すると：

* **ローカルプランモード**: 手軽で速い。小〜中規模の変更向き
* **Ultraplan**: 深くて視覚的。大規模な設計・チームレビュー向き

ultra ファミリー全体での開発フローは以下のサイクルです。

```
設計 → ultraplan で計画をレビュー
  ↓
実装 → ultracode で大規模並列実行（or 通常の実装）
  ↓
確認 → ultrareview でマージ前の深いレビュー
```

リサーチプレビュー段階ですが、Pro / Max ユーザーは今すぐ試せます。大きな変更の前に `/ultraplan` で計画を立ててみてください。
