## 自己紹介

数年前にカナダのカレッジで1年間、モバイルアプリ開発コースを修了しました。アルゴリズム、C言語、Swift、Kotlin をひと通り（という名のマッハで）学び、卒業課題ではモバイルではなく React + Firebase で Web アプリも作りました。今は日本の（IT系ではない）会社でデータ分析の仕事をしていて、業務では GAS を楽しく使っている程度です。

自己評価としては、**素人に毛が生えたくらい**だと思っています。現役エンジニアではないし、ゴリゴリ書けるわけでもない。

そんな自分が、**Claude Code（AIコーディングアシスタント）と二人三脚で**、iOSアプリをゼロから作り始めました。  
この日誌は、その記録です。記事を出したからには、アプリを完成させなあかん。お尻に火をつけます🔥🔥🔥🔥

## なぜ作ろうと思ったか

カナダ留学中、毎日英字新聞を読んでは知らない単語を調べていました。翌週、また同じ単語を調べている。**「調べる → 忘れる → また調べる」の無限ループ**です。

Anki は試しました。アルゴリズム的には最強なんですが、カードを自分で作る作業が続かない。Google翻訳も DeepL も、調べたらそこで終わり。帰国後も英語メディアは読み続けていますが、正直語彙力がまったく上がっていないです。

**そして何より、決まった単語帳をペラペラめくるのが嫌い。** 自分が出会った単語を覚えたいんです。

なので作ることにしました。ズボラでもなんとかなるやつを。

```
【いまの流れ】
単語に出会う → 調べる → 忘れる → また調べる（無限ループ）

【理想の流れ】
単語に出会う → 調べる → 自動でストック → 最適なタイミングで復習 → 長期記憶へ
```

「調べる」は今も毎日やっている。それ以降を全部自動にすればいい。

## 作っているアプリ：Drey

**Drey（ドレイ）** という名前にしました。  
開発終わってもいないのに名前だけ先に考えて浮かれてるやつですね。恥ずかしい。

リスが木の実を貯蔵する巣を意味する英単語です。私がリスが好きなので、リスに関わる言葉でなんか候補出して、で決まりました。  
「単語を蓄える場所」というコンセプトにぴったりですね。

### 使い方はこれだけ

1. Safari やニュースアプリで気になった英単語（または文）を選択
2. iOS の「共有」ボタンを押して「Drey」を選ぶ
3. それだけ

アプリが自動で辞書を引いて意味をつけ、忘れそうなタイミングで復習を促してくれます。

### ポイント：「調べる」ときの文脈ごと保存

たとえば、"The protagonist faces an impossible choice." みたいな文章を選択してアプリに共有すると、単語がチップ状に並んだ画面が開きます。覚えたい単語をタップするだけで、辞書が引かれ、**文章ごと一緒に登録**されます。

復習するとき、「あの記事で出てきた単語だ」という手がかりがあると記憶が引き出しやすい。それを狙っています。

### こんな人に使ってほしい

* 英字メディア（BBC・Reuters・The Economist）を日常的に読んでいる
* 留学・海外駐在中でまさにスマホで調べてるみなさん！
* 留学・海外駐在を経て、身につけた英語力を維持したい
* Anki を試したが、カード作りが面倒で挫折した
* いわゆる単語帳をペラペラめくって覚えるのが嫌い・苦手な人

どちらかというと、英語を「勉強」ではなく「使う」意識がある中上級者向けです。

## Claude Code との開発体制

正直に書きます。**バイブコーディングです。**

カレッジで1年勉強したくらい、かつ業務でも使わなかったら、そりゃ作れるようにはならないよね！

アーキテクチャの設計も実装も Claude Code が担っています。自分がやっているのは、「何を作るか・どう動いてほしいか」の要件と仕様の判断、動作確認、フィードバックです。設計については提案をもらい、雰囲気の部分はあれど「なんとなく納得できるか」を判断しながら対話して進めています。

「バグっぽい」「仕様はこうあるべきでは」という勘所は自分の中にあって、そこは役に立っている感覚があります。カレッジでの学習や、今もGASいじったりしてる部分が土台になっているんだと思います。

特に **Share Extension**（iOS のシェアシートに Drey を追加する機能）は、App Group の設定や OS の仕組みへの理解が必要な部分で、Claude Code なしでは相当時間がかかっていたと思います。「なぜこうなるのか」を聞きながら進められるのが、二人三脚の強みです。

## 現在地（開発日誌 #0 時点）

開発を始めてざっくり4日目くらい、現在実装が完了しているのは以下です。**これがバイブコーディングの現実です。**

**実装済み：**

* 英日辞書（約17万語）＋英英辞書による意味検索
* "running" を調べても "run" で引けるよう自動変換
* Share Extension 経由の単語登録（文章ごと保存）
* 分散学習（SRS）による復習スケジューリング
* 4択テスト＋フラッシュカードの2モード
* iCloud 自動同期（マルチデバイス対応）
* 日英UI対応の準備

**次にやること：**

* 復習リマインダー通知
* UI のポリッシュ
* TestFlight でのベータ配布

ちなみに早々にClaude Proの上限が来たので、Maxに切り替えました。  
Pro でcode使えなくなる直前のタイミング？で滑り込めたの本当によかった。。。
