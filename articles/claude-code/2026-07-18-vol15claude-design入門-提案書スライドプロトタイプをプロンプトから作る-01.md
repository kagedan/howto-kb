## はじめに

こんにちは。株式会社リアルインベントCTOです。

今回は、Anthropicが2026年4月に公開したビジュアル生成ツール「Claude Design」を取り上げます。

弊社にはデザイン部門があり、数名のデザイナーが在籍しています。なので正直に言うと、最初にこのツールを見たときの関心事は「これはデザイナーの仕事を奪うのか？」でした。1ヶ月ほど触ってみた結論を先に書くと、**答えはノー**です。Claude Designが変えるのは「何をデザインするか」ではなく、**「デザイナーへのイメージ共有をどこまで短い時間で行えるか」という課題に向き合うか**です。

本記事では、Claude Designの基本的な使い方と、デザイン部門がある会社での「AIとデザイナーの分業」という使いこなし方を、弊社の実例ベースで紹介します。

## 1. Claude Designとは

Claude Designは、Anthropicの実験部門「Anthropic Labs」が2026年4月17日にリサーチプレビューとして公開した、AIビジュアル生成ツールです。テキストで「こういう資料が欲しい」と伝えるだけで、次のようなものを生成できます。

* **スライドデッキ**（提案書・ピッチデック・研修資料）
* **プロトタイプ**（ボタンを押すと画面遷移する、動くモックアップ）
* **ワンページャー・LP**（1枚企画書、ランディングページ）
* **マーケティング素材**

ポイントは、**Claudeの有料プラン（Pro/Max/Team/Enterprise）であれば追加料金なしで使える**ことです（専用の週次使用量が別枠で管理され、上限超過時のみ従量課金）。claude.aiのナビゲーションから起動できます。

従来のデザインツール（PowerPoint、Canva、Figma）との最大の違いは、「テンプレートを選んで自分で組み立てる」のではなく、**作りたいものを言葉で説明すると、構成・文章・配色・レイアウトまで含めた完成形が返ってくる**ことです。

## 2. 実際の生成フロー：3ステップ

「SES企業向けAI研修の提案書」を題材に、実際の流れを紹介します。

### ステップ① ざっくりしたプロンプトを投げる

まず作りたいものを日本語で伝えます。この時点で細かい指定は不要です。

```
中小SES企業の経営層向けに、社内AI研修プログラムを
提案するスライドを作ってください。
```

### ステップ② 深掘り質問に答える

ここがClaude Designの気が利いているところで、いきなり生成せず、**精度を上げるための追加質問**が返ってきます。「対象者は誰か」「枚数はどのくらいか」「トーンはフォーマルかカジュアルか」といった質問に、選択肢や自由記述で答えて進めます。

プロンプトを書き込む自信がなくても、この質問フォームに答えるだけで要件が固まる設計です。**要件定義をAI側がリードしてくれる**感覚に近く、非エンジニアの営業メンバーでも使えます。

### ステップ③ 生成 → Tweaksで調整

数分待つと、表紙から各章、まとめ、さらにスピーカーノート（発表台本）まで含んだ一式が生成されます。

生成後の調整手段が豊富なのも特徴です。

* **チャットで指示**：「3ページ目の図をもっとシンプルに」のような自然言語での修正
* **Tweaks（スライダー）**：配色の濃淡やレイアウト密度などを、Claudeが自動生成したスライダーでワンクリック調整
* **Editモード**：スライド上のテキストをPowerPoint感覚で直接打ち換え
* **インラインコメント**：要素に直接コメントを付けて修正依頼

「AIの一発生成で終わり」ではなく、**初稿を出発点に反復して仕上げる**ワークフローとして設計されています。

## 3. エクスポート：作った後どうするか

完成した成果物は複数の形式で書き出せます。用途別の使い分けはこうです。

| 形式 | 向いている用途 |
| --- | --- |
| PPTX | 社内で最終微調整して客先提出（いちばん使う） |
| PDF | 印刷・そのまま配布 |
| Canva連携/Figma連携 | デザイン部門がブランドに合わせて作り込む場合 |
| HTML | そのままWeb掲載（LPなど） |
| 共有URL | 社内レビュー・コメント収集 |

弊社の実務では「Claude Designで8割 → PPTXまたはCanvaに出して、デザイン部門が会社のフォーマット（表紙・フッター・トーン&マナー）に整えて提出」という流れが定着しそうです。**AIの初稿をデザイナーに渡すための出口**として、Canva/Figma連携とPPTX出力は特に重要です。

さらに開発会社として見逃せないのが**Claude Codeへのハンドオフ**です。プロトタイプを作った後、ボタン一つでClaude Codeに引き渡して実装フェーズに進めます。「デザインと実装の分断」を最小化する思想で、企画→デザイン→実装が1つのエコシステムで繋がります。

## 4. SES企業での使いどころ — デザイン部門との分業

1ヶ月ほど触った時点での、弊社での活用シーンです。前提として、弊社にはデザイン部門があります。それでもClaude Designが効くのは、**デザイナーの時間の使い方が変わる**からです。

**① 営業提案書のたたき台**。従来、提案書はゼロからの初稿作成に半日〜1日かかり、その多くをデザイン部門か営業自身が担っていました。Claude Designなら、営業やエンジニアが自分で15〜30分で「議論できるレベルのたたき台」を出せます。デザイナーは**ゼロから作る仕事から解放され、ブランド調整と磨き込みに集中**できます。

**② 開発前のプロトタイプ**。「こういう画面イメージです」を口頭やワイヤーフレームで伝える代わりに、**動くプロトタイプ**を見せられます。受託開発の要件定義で「作ってから認識違いに気づく」リスクを減らせるのは、開発会社として実利が大きい。エンジニアが作ったプロトタイプを起点に、UI設計はデザイナーが引き取る、という受け渡しもスムーズです。

**③ 社内提案・報告資料**。経営会議向けの資料などは、内容が命でデザインに時間をかけたくない典型です。こうした「社外に出ない資料」までデザイン部門に依頼するのは気が引けるもの。ここをAIに任せれば、**デザイナーの工数を顧客向け・採用向けの本当に品質が問われる仕事に温存**できます。

**④ 採用ピッチ資料**。会社説明資料のリニューアルでは、Claude Designで構成案とビジュアルの初稿を複数パターン出し、その中からデザイン部門が方向性を選んで作り込む、という使い方を試しています。**「どの方向性で行くか」の検討段階を高速化**できるのは、デザイナー側からも好評でした。

## 5. 現時点の限界と注意点

正直な評価として、まだ癖もあります。

* **リサーチプレビュー段階**であり、挙動や提供条件が変わり得る
* **日本語フォントやレイアウトの細部**は崩れることがあり、編集可能形式での書き出しは調整が必要な場合がある。最終品質が必要ならPPTX出力後にデザイナーが仕上げる前提で
* **週次使用量は別枠管理**とはいえ無限ではない。大量生成する日はタイミングの分散を
* **機密情報の扱い**は通常のAI利用ポリシーと同じ注意を。顧客名や未公開情報を含む提案書は、社内ルールに沿って抽象化してから

念のため強調しておくと、Claude Designは「デザイナーの代わり」にはなりません（一部は担えるかもしれませんが、、、）。  
ブランドの一貫性、細部の品質、顧客の温度感に合わせた表現——こうした部分は引き続きデザイナーの領域です。現時点の等身大の評価は、**ゼロ→デザイナーへの共有の工程を誰でも短い期間でこなせるようにし、デザイナーの初稿→完成品の磨き込みに振り向けられるツール**です。代替ではなく、分業の再設計です。

## 6. まとめ

1. Claude Designは**プロンプトからスライド・プロトタイプ・LPを生成**するAnthropic Labsの新ツール。有料プランなら追加料金なし
2. **深掘り質問→生成→Tweaks調整**の流れで、プロンプトに自信がなくても使える
3. **PPTX/PDF/Canva/HTML**へ書き出せて、Claude Codeへのハンドオフで実装まで繋がる
4. SES企業の使いどころは**提案書のたたき台・要件定義用プロトタイプ・社内資料・採用資料**
5. 位置づけは「デザイナーの代替」ではなく**たたき台はAI、磨き込みはデザイナーという分業の再設計**。初稿づくりを民主化し、デザイナーは品質が問われる仕事に集中できる
