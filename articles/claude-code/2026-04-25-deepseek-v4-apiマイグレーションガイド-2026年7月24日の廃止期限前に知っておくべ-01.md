DeepSeek APIを使っている方は知っておくべき期限があります：**2026年7月24日**に、レガシーモデル名 `deepseek-chat` と `deepseek-reasoner` が廃止されます。

DeepSeek V4は2026年4月24日にリリースされました。1Mコンテキスト、新しいアテンションアーキテクチャ、エージェンティックコーディングのオープンソースSOTA。本記事では、移行に必要なすべての情報を解説します。

## 何が変わったか

DeepSeek V4は2種類のバリアントがあります：

| モデル | 総パラメータ数 | アクティブパラメータ | コンテキスト | 用途 |
| --- | --- | --- | --- | --- |
| `deepseek-v4-pro` | 1.6T | 49B (MoE) | 100万トークン | 複雑な推論、エージェント、コーディング |
| `deepseek-v4-flash` | 284B | 13B (MoE) | 100万トークン | 高速・低コスト処理 |

## レガシーモデルのルーティング

2026年4月24日から：

* `deepseek-chat` → `deepseek-v4-flash`（通常モード）にルーティング
* `deepseek-reasoner` → `deepseek-v4-flash`（思考モード）にルーティング

**両方のレガシー名は2026年7月24日に廃止されます。** それ以降、旧名称でのリクエストは失敗します。

## 移行方法（1行の変更）

```
# 移行前 — 2026年7月24日以降は動作しない
client.chat.completions.create(
    model="deepseek-chat",
    messages=[...]
)

# 移行後 — V4-Flash（同等コストティア）
client.chat.completions.create(
    model="deepseek-v4-flash",
    messages=[...]
)

# または重タスク向けV4-Pro
client.chat.completions.create(
    model="deepseek-v4-pro",
    messages=[...]
)
```

ベースURLとAPIキーは変わりません。本当に1行の変更です。

## V4-Pro vs V4-Flash：使い分けの判断基準

**V4-Proを選ぶとき：**

* 複数ステップのエージェントタスク（コーディング、リサーチ、分析）
* 複雑な推論チェーンが必要
* 100万トークンのフルコンテキストが必要（大規模コードベース等）

**V4-Flashを選ぶとき：**

* 大量処理・低コスト重視
* シンプルな検索・分類・要約
* レイテンシ優先
* 旧 `deepseek-chat` のドロップイン置換

## エージェント開発者に重要な新機能

### 100万トークンコンテキスト

ほとんどのプロダクションエージェントは、コンテキストウィンドウが小さすぎるため外部メモリ（Mem0、Zep、RAG）を使っていました。100万トークンあれば、大規模コードベース全体や長文ドキュメントをチャンキングなしで1コンテキストに収められます。

### リクエストごとの思考モード切替

V4は同一モデルで通常モードと拡張推論モードの両方をサポート。シンプルなステップは高速モード、複雑な計画立案は思考モード、という使い分けが1モデルで可能。

### オープンソースでのエージェンティックコーディングSOTA

DeepSeekはV4-Proがエージェンティックコーディングベンチマークでオープンソース最高性能と主張。OpenAI互換APIでClaude CodeやOpenCodeからも利用可能。
