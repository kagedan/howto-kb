THIS GUY FOUND A FREE LOCAL DASHBOARD FOR CLAUDE CODE TOKEN BURN

Most Claude Code users only notice tokens when the limit warning shows up.

CodeBurn is useful because it shows the burn pattern before that moment.

It is a free, open-source, local dashboard for AI coding usage.

The video shows it breaking spend down by:

> project
> model
> activity type
> sessions
> shell commands

That matters because “Claude feels expensive today” is not an engineering signal.

“This project is burning tokens on exploration” is.

“This session kept retrying the same failure” is.

“This model was overkill for the task” is.

The best metric in the demo is the one-shot rate.

Simple idea:

Did Claude solve the task on the first attempt, or did it spiral into retries, debugging, rewriting, and tool calls?

A low one-shot rate is not just a cost problem.

It usually points to a workflow problem:

bad spec
unclear context
missing tests
too much repo noise
wrong model for the job
blind retries instead of narrowing the task

That is the real reason this kind of dashboard matters.

It does not magically make Claude cheaper.

It gives you the receipt.

You still have to change the behavior: smaller tasks, cleaner context, better prompts, better specs, cheaper models when possible, and fewer “try again” loops.

But if Claude Code is part of your daily dev loop, that receipt is valuable.

The next layer for AI coding is not only smarter agents.

It is observability for agents.

Once the assistant becomes part of the engineering workflow, token spend stops being just a billing line.

It becomes a debugging signal.
