## 前回までのあらすじ

これまで、Gemini無料プランやClineを駆使して「一日ひとこと日記アプリ」を作ってきた。  
Antigravityの一発生成 → 設計・環境構築 → コンポーネント実装と、紆余曲折ありつつ一応「粗方やりたいと思っていたこと」は実現できた。

ただ、初学者の私は完成したコードの是非や良し悪しが全く理解できていない。  
そこで今度は、Claude Codeに課金してSonnet 4.6の力を借り、コードレビューをしてもらいながら中身の理解を深めてみようと思う。

今回のレビュー対象である日記アプリはこちら⇨[ClaudeとGeminiを行き来して日記アプリに再挑戦してみた：実装編](https://zenn.dev/whipea/articles/7ab8d7a32c0fee)

## コードレビューを依頼してみた

早速ClineのモデルをSonnet 4.6に切り替えて、前回のプロジェクトを保存しているフォルダパスと共に以下の簡単なプロンプトを投げてみた。

**投げたプロンプト**

```
ここに、日記アプリのプロジェクトがあります。コードレビューをお願いします。
また、ベテランの教育担当エンジニアとして、初学者に向けて各ファイルおよびコンポーネントの内容の解説と実装する上での注意点について教えてください。
```

返ってきたのがこちら。  
![](https://static.zenn.studio/user-upload/6a9fb1b08c06-20260331.png)  
![](https://static.zenn.studio/user-upload/eb02bb586abf-20260331.png)

かなり簡単な指示だったにも関わらず、かなりしっかりまとめてくれている。特にコンポーネントの解説については、1つ1つに対して**解説・良い点・改善提案・初学者へのアドバイス**を盛り込んでくれた。

## Notionに転記してもらう

見返しやすいようにNotionに転記してもらおうと思い、Notion APIを使ってClineに直接転記を依頼した（この手順についてもやり取りの中で聞きながら進めた）。本当になんでも教えてくれるな……。

![](https://static.zenn.studio/user-upload/1a2fb7c7eea4-20260331.png)

接続が完了しいざ転記するよう指示したら、なぜかさっきの出力ではなく、端折った内容（概要＋詳細はClineとのチャットを参照してね、という感じ）にされてしまった。

前からなんとなく感じていたのだが、**「詳細に！」と言わないとAIは概要止まりになりがち**なんだよな。  
「AIでなんでもできる」って言われるけど、結局はちゃんと具体的な指示に落とし込む力が必要なんじゃないか？  
みんなデフォルトでそんなスキル持ってるんですか？ すげえな……

「もっと詳細に、コンポーネントごとに解説を追記して」と言い直した結果、いい感じに以下を盛り込んだページが完成した。

* 各コンポーネントの役割
* 重要な関数やロジックのポイント
* 具体的な改善提案

さらに「改善提案ページ」も別で作ってくれた。状態管理、テストコード、アクセシビリティ……本業で聞いたことのある言葉ばかりだが、ゼロから自分で考慮に入れるのはかなり慣れが必要そう。「色々考えなきゃいけないことがあるんだなあ……」と改めて実感した。

考えるべきことを網羅するには、人間もある程度レベルが高くないと難しいのかも？  
とはいえ「そこも全部AIに聞けばいいじゃん」と言われればその通りではあるが、やはり**何がしたいかを明確に言語化する力**が鍵になりそう。

![](https://static.zenn.studio/user-upload/a898b50c30e6-20260331.png)

## 謎のLintエラーも解析してもらう

前回から気になっていた「動くのにLintエラーが消えない」問題も聞いてみた。  
最初は「2箇所エラーがありそう」とだけ伝えたら、思ってたのと違う場所のリファクタリングを提案されてしまった（指示の具体性大事・n回目）。

そこで「何行目のこの関数で、このエラーメッセージです」と具体的に伝え直したところ、

* エラーが出ている原因
* なぜそのコードで警告が出るのか
* 修正提案

を丁寧に説明してくれた。  
内容が難しかったので「もっと噛み砕いて、1行ずつ教えて」とお願いしたら、関数の中身を本当に1行ずつ解説してくれた……ありがたい。

対人間だとなかなか質問攻めにするのも憚られる部分もあると思うので、AI課金して「専属の先生」として使い放題にするの、かなりアリなのでは？

## 振り返り

というわけで、日記アプリ作成プロジェクトはこれにて一旦ひと段落です。

無料プランだけでもそこそこには作れたけど、コードの品質を上げたり、自分で理解を深めたりするには、やはり課金した高性能モデルが強いことを実感した。特に「レビューしてもらう」「解説してもらう」用途では、Sonnet 4.6の丁寧さと深さが段違いだったなあ。

ただし、結局のところ「**良い指示を出す力**」が一番大事だということも再確認。  
課金したので、他にも色々遊んでみようと思います。  
では、また次の学習記録でお会いしましょう〜！
