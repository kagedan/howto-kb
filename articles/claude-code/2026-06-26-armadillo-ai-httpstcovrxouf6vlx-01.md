Claude Codeを使っていて、急に「court」という意味不明な単語のあと、「<invoke name="Edit">」のようなタグが文章のまま流れてきて、手が止まった。

```
court
<invoke name="Edit">
<parameter name="file_path">/path/to/file</parameter>
...
</invoke>
```

もしそんな経験があるなら、まず安心してください。これはあなたの設定ミスでも、使い方の問題でもありません。

このバグは2026年の5月末から6月にかけて、特にOpus 4.8で一気に報告が増えました。GitHubのある報告では「court」が1度のやり取りで182回、別の報告では490回も出続けています。ある検証者がログをまとめて調べたところ、4,758ファイル・1.3GBの記録の中から、同じ壊れ方が58件も見つかりました。

報告はZennやnote、技術ブログにも次々と広がり、もはや一部の人の特殊な環境だけの話ではなくなっています。

やっかいなのは、ただ止まるだけでは済まないことです。最悪なのは、Claudeが「完了しました」と言うのに、実際には何も実行されていないパターン。今回は、この「court問題」の正体と、文章を消すだけで抜け出す直し方を整理します。

# なぜ再起動やバージョン下げでは直らないのか

動かなくなったら、入れ直すか、バージョンを戻せばいい。多くの人はそう考えます。

ところが、このバグはそれでは消えません。特定のバージョンの不具合ではないからです。複数の2.1系で報告され、CLIでもデスクトップアプリでもWSLでも出る。環境を変えても、入れ直しても、同じところで止まります。報告されているバージョンは2.1.148や2.1.179などまちまちで、特定の不良版さえ避ければいい、という話でもありません。

理由は、壊れている場所がアプリではないことにあります。崩れているのは、AIが出す「ツールの呼び出し方」そのもの。土台のプログラムは正常で、渡される命令の形が崩れている。だから、土台をいくら入れ替えても効かないのです。

まずは、画面に出るその「court」が何なのかを知るところから始めましょう。

# 「court」の正体は、命令が「ただの文章」に化けた状態

courtはエラーコードか、何かの警告だと思うかもしれません。

そうではありません。courtは、本来は裏側で静かに実行されるはずのツール呼び出しが、表の文章として漏れ出したときに、その先頭に紛れ込む余計な単語です。報告によっては「court」ではなく「count」や「call」のこともありますが、起きていることは同じです。

ふだんClaude CodeがファイルをReadしたりEditしたりするとき、その指示は「ツール呼び出し」という決まった形で、土台のプログラムに渡されます。ところがこのバグでは、「これから実行する」という合図だけが立つのに、肝心の呼び出し本体が作られない。代わりに「<invoke name="Read">」が、ただの文字として吐き出される。命令のはずだったものが、ただの文章に化けるわけです。技術的には、「これからツールを実行する」という内部の合図だけが立つのに、実行できる呼び出しの中身がどこにもない、という食い違いが起きています。記録を開くと、合図はあるのに本体は空、という形がそのまま残っています。本来なら裏方として一瞬で終わる作業が、台本だけ読み上げて誰も動かない、そんな状態に近いのです。

文章に化けた命令は、誰も実行しません。だからファイルは変わらず、コマンドは走らず、会話の見た目だけが進んでいく。止まるのは、あなたの腕の限界ではなく、ツール側の一時的な不具合です。

そして、この「化け」がいちばん怖い形で出るのが、次の場面です。

# 「完了しました」は嘘かもしれない

Claudeが「できました」と言えば、できている。ふつうはそう受け取ります。

ここは完全な間違いです。courtが出たとき、Claudeは実行していない作業を「やりました」と報告することがあります。命令が文章に化けただけなので、AIは出力した時点で、もうやった気になっているのです。

「PRを作成しました」と言うのにリンクを開くと404。「ファイルを修正しました」と言うのに中身は元のまま。「テストを通しました」と言うのに、走らせると赤いまま。どれも、実体のない報告だけが残ります。これが空約束です。怖いのは、この空約束を信じて次に進むと、ありもしない変更の上に、さらに作業を積み上げ
