## 実現方法

公式が出している「Terminal Mode」は、音声入力と、PC-メガネ間の接続を追加の開発なしで実現することができます。

### Even G2特有の設定手順

Terminal Modeを使うには、以下を行う必要があります。

* PC側でライブラリをクローンし、起動する
* スマホアプリ側で接続

**PC側**

Mac 側で `@evenrealities/even-terminal` を起動します。Even Realities 公式が npm で出している CLIです。

<https://www.npmjs.com/package/@evenrealities/even-terminal?activeTab=readme>

```
npx @evenrealities/even-terminal -d /path/to/workspace
```

`-d` で Claude が動く作業ディレクトリを渡します。起動するとターミナルに QR コードが表示されるので、スマホの Even Realities App からスキャンするだけで接続できます。

**アプリ側**

1. ターミナルモードをON

![](https://static.zenn.studio/user-upload/9c5146171d8c-20260517.png)  
2.ホストの設定画面の右上にあるアイコンからQRコードカメラを起動

![](https://static.zenn.studio/user-upload/a165bca6afe6-20260517.png)

### gws CLIでClaude CodeとGoogle Calendarを繋げる

ここからは、Claude Codeに親しんでる人なら知ってる話がほとんどです。

Calendar 操作はGoogle 公式の `gws` CLI（[`@googleworkspace/cli`](https://github.com/googleworkspace/cli)）を使いました。ここは好みの問題です。

```
brew install googleworkspace-cli
gws auth login   # 初回は OAuth クライアントの準備が必要
```

これで「来週金曜 20 時に飲み」みたいなつぶやきから、Claude が Calendar API を叩いて予定登録、までが繋がります。

### CLAUDE.md でメガネ向けに出力スタイルを縛る

これがけっこう大事です。メガネは **576×288 のモノクロ緑ディスプレイ** が視界の一部に小さく浮かんでいるだけ。テーブルや装飾は崩れて読めないし、長文は流れて消えていきます。

なので `-d` で渡したディレクトリの `CLAUDE.md` に、こんなルールを書いてみました：

```
- markdown table を使わない（崩れる）
- 結論を最初の 1 文に
- 通常のクエリには 1〜3 文で返す
- 「Let me check...」のような前置きは書かない
- 完了報告は短く（"Done." 程度）
- 絵文字・装飾文字は避ける
- 長いログや差分はファイルに書いて path だけ返す
```

これで幾分かマシになります！

## まだ足りないこと

* **ローカルネットワークのPCとの接続しか試せてない**：同じWifiを共有してる状態でのセッション共有しかできてないです。多分、頑張ればできるのでしょうが、まだ探索してないです。
* **ターミナル UI がそのままメガネに流れる**ので、tool use の途中経過（"Reading file..." 等）をいちいちUIに出すのでまだまだ読みにくいです。メガネで見やすいUIを構築するためには、ちゃんとSDKでアプリを作る必要がありそう...
