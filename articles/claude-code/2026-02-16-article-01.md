Claude Code の /handover っていうカスタムコマンド、やってみた

セッション終わりに打つだけで、今回やったこと、バグの修正履歴、設計判断の理由、次やるべきこと、全部まとめた HANDOVER .md（引き継ぎノート）を自動生成してくれる

リプで出てた「pre auto compact hook に仕込む」ってアイデアも良くて、ちょい試したらコンテキストが溢れそうになったタイミングで HANDOVER .md が自動生成された

次のセッションではこれを読み込ませるだけ。Claude が「あ、前回ここまでやったんやね」って完全に文脈を理解した状態で再開してくれる

何がいいかって、CLAUDE .md とは役割が違うところかな。CLAUDE .md はプロジェクト全体のルール。HANDOVER .md は「今日の作業の引き継ぎ」。この2つを分けるだけで、セッション跨ぎの精度が変わる

スクショは一部しかのせてない。整理して後ほど完全版を公開する

This prevents amnesia and preserves institutional knowledge
