# Claude Code 環境変数リファレンス完全ガイド

Claude Code の環境変数はご存知の通り大量にあり、全体を把握するために体系的にまとめてみました。シェルで起動前に設定するか、`settings.json` の `env` キーに書いておくことで、全セッションまたはチーム全体に適用することができます。ご参考にどうぞ。

> 参考: [code.claude.com/docs/ja/env-vars](https://code.claude.com/docs/ja/env-vars)

---

## 設定方法

### 方法1: シェルで一時的に設定する

```
export ANTHROPIC_API_KEY="your-api-key"
claude
```

### 方法2: settings.json で永続化する

```
{
  "env": {
    "ANTHROPIC_MODEL": "claude-sonnet-4-6",
    "CLAUDE_CODE_ENABLE_TELEMETRY": "1"
  }
}
```

`settings.json` のスコープ（ユーザー / プロジェクト / Managed）に応じて、個人・チーム・企業全体への展開が可能です。

## カテゴリ別 環境変数一覧

勝手にカテゴリを分けまとめてみました

### 1. 認証・API接続

Claude API への認証やエンドポイントの設定に使用する

| 変数名 | 説明 |
| --- | --- |
| `ANTHROPIC_API_KEY` | Anthropic API の主要 API キー |
| `ANTHROPIC_AUTH_TOKEN` | Anthropic サービス用の代替認証トークン |
| `ANTHROPIC_BASE_URL` | カスタム API エンドポイント URL |
| `ANTHROPIC_BETAS` | カンマ区切りのベータ機能リスト |
| `ANTHROPIC_CUSTOM_HEADERS` | Anthropic API リクエストへのカスタム HTTP ヘッダー（JSON 形式） |
| `API_TIMEOUT_MS` | API リクエストのタイムアウト（ミリ秒） |
| `CLAUDE_CODE_OAUTH_TOKEN` | OAuth 認証トークン |
| `CLAUDE_CODE_MAX_RETRIES` | リクエスト失敗時の最大リトライ回数 |
| `CLAUDE_CODE_CLIENT_CERT` | TLS 認証用クライアント証明書のパス |
| `CLAUDE_CODE_CLIENT_KEY` | TLS 認証用クライアント秘密鍵のパス |
| `CLAUDE_CODE_CLIENT_KEY_PASSPHRASE` | 暗号化された秘密鍵のパスフレーズ |

### 2. モデル設定

使用するモデルやエイリアスを制御する

| 変数名 | 説明 |
| --- | --- |
| `ANTHROPIC_MODEL` | デフォルトで使用する Claude モデル |
| `ANTHROPIC_SMALL_FAST_MODEL` | 高速処理用の小型モデル |
| `CLAUDE_CODE_SUBAGENT_MODEL` | サブエージェント処理に使用するモデル |
| `ANTHROPIC_DEFAULT_OPUS_MODEL` | `opus` エイリアスが指すモデル ID（バージョン固定に使用） |
| `ANTHROPIC_DEFAULT_SONNET_MODEL` | `sonnet` エイリアスが指すモデル ID |
| `ANTHROPIC_DEFAULT_HAIKU_MODEL` | `haiku` エイリアスが指すモデル ID |
| `ANTHROPIC_CUSTOM_MODEL_OPTION` | モデルセレクターに追加するカスタムモデル ID |
| `ANTHROPIC_CUSTOM_MODEL_OPTION_NAME` | カスタムモデルの表示名 |
| `ANTHROPIC_CUSTOM_MODEL_OPTION_DESCRIPTION` | カスタムモデルの説明文 |

**モデルのバージョン固定の例（Bedrock 利用時）:**

```
export ANTHROPIC_DEFAULT_OPUS_MODEL='us.anthropic.claude-opus-4-6-v1'
export ANTHROPIC_DEFAULT_SONNET_MODEL='us.anthropic.claude-sonnet-4-6'
export ANTHROPIC_DEFAULT_HAIKU_MODEL='us.anthropic.claude-haiku-4-5-20251001-v1:0'
```

### 3. 動作・UX 制御

Claude Code のインタラクションや出力形式を制御する

| 変数名 | 説明 |
| --- | --- |
| `CLAUDE_CODE_ACTION` | 操作の権限モード。`acceptEdits`（ファイル変更前に確認）、`plan`（分析のみ）、`bypassPermissions`（全操作を自動承認）、`default` |
| `CLAUDE_CODE_MAX_OUTPUT_TOKENS` | レスポンスの最大出力トークン数 |
| `MAX_THINKING_TOKENS` | モデルの思考ステップの最大トークン数 |
| `CLAUDE_CODE_DISABLE_TERMINAL_TITLE` | ターミナルウィンドウタイトルの自動更新を無効化 |
| `CLAUDE_CODE_DISABLE_FINE_GRAINED_TOOL_STREAMING` | ツール実行の詳細ストリーミングを無効化 |
| `DISABLE_INTERLEAVED_THINKING` | インターリーブ思考モードを無効化 |
| `DISABLE_MICROCOMPACT` | マイクロコンパクト出力フォーマットを無効化 |
| `FORCE_CODE_TERMINAL` | GUI 環境でも強制的に CLI モードで起動 |
| `CLAUDE_CODE_BRIEF` | コンパクト出力のブリーフモードを有効化 |

### 4. Bash・ツール実行

コマンド実行の挙動を制御する

| 変数名 | 説明 |
| --- | --- |
| `BASH_DEFAULT_TIMEOUT_MS` | Bash コマンドのデフォルトタイムアウト（ミリ秒） |
| `BASH_MAX_TIMEOUT_MS` | Bash コマンドの最大許容タイムアウト（ミリ秒） |
| `BASH_MAX_OUTPUT_LENGTH` | Bash コマンド出力の最大長 |
| `CLAUDE_BASH_MAINTAIN_PROJECT_WORKING_DIR` | Bash コマンドの作業ディレクトリをプロジェクトディレクトリに維持（`1`/`true`） |
| `CLAUDE_CODE_DISABLE_COMMAND_INJECTION_CHECK` | コマンドインジェクションのセキュリティチェックを無効化 |
| `USE_BUILTIN_RIPGREP` | バンドルされた ripgrep を使用（`true`）/ システムの ripgrep を使用（`false`） |

### 5. コンテキスト・パフォーマンス最適化

コンテキストウィンドウやキャッシュの設定する

| 変数名 | 説明 |
| --- | --- |
| `CLAUDE_CODE_AUTO_COMPACT_WINDOW` | 自動圧縮のコンテキストウィンドウサイズをオーバーライド |
| `DISABLE_PROMPT_CACHING` | プロンプトキャッシュの最適化を無効化 |
| `USE_API_CONTEXT_MANAGEMENT` | API コンテキスト管理を有効化（メモリとコンテキスト最適化） |
| `CLAUDE_CODE_DISABLE_1M_CONTEXT` | 100 万トークンコンテキストウィンドウを無効化（`1` を設定） |
| `CLAUDE_CODE_DISABLE_ADAPTIVE_THINKING` | Opus 4.6 / Sonnet 4.6 のアダプティブ推論を無効化し固定予算に戻す（`1` を設定） |
| `CLAUDE_CODE_EFFORT_LEVEL` | 推論の努力レベル。`low`、`medium`、`high`、`max`、`auto` |

### 6. MCP（Model Context Protocol）

MCP サーバーの接続と動作を制御する

| 変数名 | 説明 |
| --- | --- |
| `MCP_TIMEOUT` | MCP 操作の全般タイムアウト（ミリ秒） |
| `MCP_TOOL_TIMEOUT` | MCP ツール実行の特定タイムアウト |
| `MAX_MCP_OUTPUT_TOKENS` | MCP サーバー出力の最大トークン数（デフォルト: 25,000） |
| `MCP_SERVER_CONNECTION_BATCH_SIZE` | MCP サーバー接続のバッチサイズ |
| `MCP_OAUTH_CALLBACK_PORT` | MCP 認証の OAuth コールバックポート |

**MCP 出力トークンを増やす例:**

```
export MAX_MCP_OUTPUT_TOKENS=50000
```

| `OTEL_METRICS_EXPORTER` | OpenTelemetry メトリクスエクスポーター設定（# オプション: otlp, prometheus, console) |
| `OTEL_LOGS_EXPORTER` | OpenTelemetry ログエクスポーター設定 (# オプション: otlp, console) |
| `OTEL_EXPORTER_OTLP_ENDPOINT` | OTLP エクスポーターのエンドポイント |
| `OTEL_EXPORTER_OTLP_HEADERS` | OTLP エクスポーターのヘッダー |
| `OTEL_EXPORTER_OTLP_PROTOCOL` | OTLP プロトコル |
| `OTEL_LOG_USER_PROMPTS` | テレメトリにユーザープロンプトのログを含める |
| `OTEL_METRIC_EXPORT_INTERVAL` | メトリクスのエクスポート間隔 |
| `OTEL_LOGS_EXPORT_INTERVAL` | ログのエクスポート間隔 |
| `OTEL_EXPORTER_PROMETHEUS_HOST` | Prometheus エクスポーターのホスト |
| `OTEL_EXPORTER_PROMETHEUS_PORT` | Prometheus エクスポーターのポート |
| `CLAUDE_CODE_OTEL_SHUTDOWN_TIMEOUT_MS` | OpenTelemetry シャットダウンのタイムアウト |
| `CLAUDE_CODE_ACCOUNT_TAGGED_ID` | OTEL メトリクス用のタグ付きアカウント ID |

**テレメトリを有効化する settings.json の例:**

```

}
```

### 8. AWS Bedrock 連携

AWS Bedrock を通じて Claude を利用する場合の設定

* `AWS_REGION`は必須の環境変数
* Claude Codeはこの設定について、`.aws`設定ファイルから読み込まない
* Bedrockを使用する場合、`/login` および `/logout` コマンドは無効になる

| 変数名 | 説明 |
| --- | --- |
| `CLAUDE_CODE_USE_BEDROCK` | AWS Bedrock を使用（`1`） |
| `AWS_REGION` | Bedrock サービスの AWS リージョン（必須） |
| `AWS_ACCESS_KEY_ID` | Bedrock 認証用 AWS アクセスキー |
| `AWS_SECRET_ACCESS_KEY` | Bedrock 認証用 AWS シークレットキー |
| `AWS_SESSION_TOKEN` | 一時的な AWS 認証情報のセッショントークン |
| `AWS_PROFILE` | 認証情報選択用 AWS プロファイル名 |
| `AWS_DEFAULT_REGION` | AWS サービスのデフォルトリージョン |
| `AWS_BEARER_TOKEN_BEDROCK` | Bedrock 認証用 Bearer トークン |
| `BEDROCK_BASE_URL` | AWS Bedrock API のカスタムベース URL |
| `ANTHROPIC_SMALL_FAST_MODEL_AWS_REGION` | Bedrock 利用時の小型高速モデル用 AWS リージョン |
| `CLAUDE_CODE_SKIP_BEDROCK_AUTH` | AWS Bedrock 認証をスキップ |

```
# 推論プロファイル ID を使用
export ANTHROPIC_MODEL='global.anthropic.claude-sonnet-4-6'
export ANTHROPIC_SMALL_FAST_MODEL='us.anthropic.claude-haiku-4-5-20251001-v1:0'

# アプリケーション推論プロファイル ARN を使用
export ANTHROPIC_MODEL='arn:aws:bedrock:us-east-2:your-account-id:application-inference-profile/your-model-id'

# オプション：必要に応じてプロンプトキャッシングを無効にする
export DISABLE_PROMPT_CACHING=1
```

<https://code.claude.com/docs/ja/amazon-bedrock>

### 9. Google Vertex AI 連携

Google Cloud の Vertex AI を通じて Claude を利用する場合の設定

* 請求が有効になっているGCPプロジェクト
* Vertex AI APIが有効になっているGCPプロジェクト
* 目的のClaudeモデルへのアクセス
* Google Cloud SDK (gcloud)がインストールされ、設定されていること
* 目的のGCPリージョンに割り当てられたクォータ

| 変数名 | 説明 |
| --- | --- |
| `CLAUDE_CODE_USE_VERTEX` | Google Vertex AI を使用（`1`） |
| `ANTHROPIC_VERTEX_PROJECT_ID` | Vertex AI 連携用 Google Cloud プロジェクト ID |
| `VERTEX_BASE_URL` | Google Vertex AI API のカスタムベース URL |
| `GOOGLE_APPLICATION_CREDENTIALS` | Google Cloud サービスアカウント JSON のパス |
| `GOOGLE_CLOUD_PROJECT` | Google Cloud プロジェクト識別子 |
| `CLOUD_ML_REGION` | Google Cloud ML リージョン設定 |
| `CLAUDE_CODE_SKIP_VERTEX_AUTH` | Google Vertex AI 認証をスキップ |
| `VERTEX_REGION_CLAUDE_4_0_OPUS` | Claude 4 Opus 用 GCP リージョン |
| `VERTEX_REGION_CLAUDE_4_0_SONNET` | Claude 4 Sonnet 用 GCP リージョン |
| `VERTEX_REGION_CLAUDE_3_5_HAIKU` | Claude 3.5 Haiku 用 GCP リージョン |

## チートシート

### 個人開発：APIキーを設定する

```
# ~/.zshrc または ~/.bashrc に追記
export ANTHROPIC_API_KEY="your-api-key-here"
```

### チーム展開：settings.json で共通設定を管理する

```
{
  "env": {
    "ANTHROPIC_MODEL": "claude-sonnet-4-6",
    "BASH_DEFAULT_TIMEOUT_MS": "30000",
    "CLAUDE_CODE_ENABLE_TELEMETRY": "1",
    "OTEL_METRICS_EXPORTER": "otlp",
    "OTEL_EXPORTER_OTLP_ENDPOINT": "https://your-otel-endpoint"
  }
}
```

### AWS Bedrock 経由で利用する

```
export CLAUDE_CODE_USE_BEDROCK=1
export AWS_REGION=us-east-1
export AWS_PROFILE=my-bedrock-profile

# モデルバージョンを固定する（推奨）
export ANTHROPIC_DEFAULT_SONNET_MODEL='us.anthropic.claude-sonnet-4-6'
```

### Google Vertex AI 経由で利用する

```
export CLAUDE_CODE_USE_VERTEX=1
export ANTHROPIC_VERTEX_PROJECT_ID=my-gcp-project
export CLOUD_ML_REGION=us-east5
export GOOGLE_APPLICATION_CREDENTIALS=/path/to/service-account.json
```

### CI 環境で自動承認モードで動かす

```
export CLAUDE_CODE_ACTION=bypassPermissions
export ANTHROPIC_API_KEY="$CI_ANTHROPIC_KEY"
```

### プロキシ経由でアクセスする

```
export HTTPS_PROXY=http://proxy.example.com:8080
export NO_PROXY=localhost,127.0.0.1
```

### Docker / WSL でファイル監視を安定させる

```
export CHOKIDAR_USEPOLLING=true
export CHOKIDAR_INTERVAL=500
```

## 設定の優先順位

Claude Code の設定は以下の優先順位で適用されます（上が優先度高）：

1. **Managed（企業管理）** — MDM や managed-settings.json による展開
2. **コマンドライン引数** — 起動時の `--model` などのフラグ
3. **プロジェクト設定** — `.claude/settings.json`
4. **ユーザー設定** — `~/.claude/settings.json`
