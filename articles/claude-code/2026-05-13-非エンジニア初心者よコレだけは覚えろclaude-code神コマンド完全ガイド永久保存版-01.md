### 1. はじめに：なぜ「コマンド」があなたの開発を劇的に変えるのか？

「チャットでお願いする（プロンプト）だけで十分じゃないの？」と思うかもしれません。でも、コマンド（/から始まる命令）には決定的な違いがあるんです。

![](https://assets.st-note.com/img/1778632044-5kPJLy4ZgCYNEdAcOp6uRatW.png?width=1200)

* **「お願い」ではなく「機能のスイッチ」：** プロンプトがClaudeへの「相談」なら、コマンドはツールの「機能そのものを直接操作する物理スイッチ」です。脳の賢さを切り替えたり、パンパンになった記憶を整理したり、ツールのコンディションを整えるために不可欠な操作なんです。
* **「黒い画面の相棒（コマンドライン版）」の真価：** デスクトップアプリ版も手軽ですが、このコマンドライン版でしか発動できない強力な機能が山ほどあります。
* **賢さを維持するメンテナンス：** Claudeは会話が長引くと、人間と同じように「記憶（コンテキスト）」が混乱してきます。コマンドを使えば、常にClaudeをキレッキレの状態でキープできる。これが、質の高い成果物を生むための最大の秘訣なんです！

![](https://assets.st-note.com/img/1778632277-OMuW2P1xkbS80HXaB4dYNC9E.png?width=1200)

### 2. 理想的なワークフロー：セットアップからリリースまでの全体像

非エンジニアの方が「何から手をつければいい？」と迷わないよう、プロジェクトを進める黄金のステップを用意しました。

![](https://assets.st-note.com/img/1778632312-TPvVbHEtwiYLxoUXdZ9eK0jq.png?width=1200)

**① プロジェクトの開始（健康診断と自己紹介）** まずは /init でフォルダを分析し、指示書（CLAUDE.md）を作成。さらに /memory を使って、「自分好みのルール」を Claude に教えていきます。これはまさに「秘伝のタレ」を育てるような楽しい工程です！

**② タスクの実行（計画とチューニング）** /plan を入力して実装の作戦会議を開始。次に /model や /effort を使い、その時のタスクに最適な「脳の賢さ」と「思考の深さ」をセットします。

**③ 仕上げと確認（記憶の整理）** /diff で変更箇所を確認。会話が長くなって Claude が疲れてきたら、/compact で記憶をギュッと圧縮して賢さを取り戻します。

**④ 終了と再開（未来の自分へパス）** 作業が終われば /exit で安全に終了。後日、/resume を使えば、前回中断した場所に一瞬でワープして再開できます！

### 3. これを知らないと始まらない！必須の基礎コマンド8選

まずは土台となる基本中の基本。ここからマスターしましょう！

* **/init：プロジェクトの健康診断** フォルダの中身を全部チェックして、構成やルールをまとめた指示書「CLAUDE.md」を自動で作ってくれます。「今、自分は何を作ろうとしているのか」をClaudeが完璧に理解するための自己紹介タイムです！
* **/plan：実装計画（プラン）** 実際の作業（コードの変更やコンテンツの生成など）に入る前に、Claudeと対話しながら「実装計画（プラン）」を立てるためのコマンドです。いきなり作業をさせて「やっぱり違う」となるのを防ぎ、ピンポイントで目的のものを作れるため、結果的に会話の数や無駄な時間を削減できます。

![](https://assets.st-note.com/img/1778632434-f2L8GXBMI0mJFlwP39KVdRZ6.png?width=1200)

* **/effort：思考の粘り強さ（5段階設定）** Claudeがどれだけ深く考えるかを「Low / Medium / High / XHigh / Max」の5段階で指定。エラーが解決しない時は「Max」にして、Claudeの本気を見せてもらいましょう！
* **/model：脳の切り替え（Opus 4.7 & Sonnet 4.6）** AIのモデルを切り替えます。最新の「Opus 4.7」や「Sonnet 4.6」が選択可能。特におすすめは、**「Opus Planモード」**！「計画は天才のOpus、実行は快速のSonnet」という風に、質とコストのいいとこ取りができる神機能です。

![](https://assets.st-note.com/img/1778632829-2sHd8PqAhovmRarg4N7jX1YS.png?width=1200)

* **/clear：記憶のリセットボタン** これまでの会話を一度きれいに掃除します。会話が長くなりすぎてClaudeが混乱したとき、真っさらな状態から考え直してもらうことで、正答率がグンと上がります！
* **/compact：記憶の圧縮（サマライズ）** 会話の内容を要約して、脳の空き容量を増やします。実はこれ、「エラーの経緯は残して！」といった指示もできるんです。賢さを保ったまま長時間作業を続けるための秘策です。
* **/exit：セッションの終了** 「今日はお疲れ様！」の合図。安全に作業を閉じます。履歴は保存されているので、いつでも続きから始められますよ。
* **/help：困った時の説明書** 「あの呪文、なんだっけ？」と思ったらこれ。使えるコマンドの一覧をサッと表示してくれます。

### 4. 生産性が爆上がりする！日常使いの効率化コマンド4選

「同じことを何度も言わせない」「失敗をなかったことにする」ための、実戦で超役立つコマンドたちです。

![](https://assets.st-note.com/img/1778633139-P8RkaOytoz59MuNibcAHlxYs.png?width=1200)

![](https://assets.st-note.com/img/1778633433-SiHPpNeVbQIcvxOrYnE2zmUM.png?width=1200)

* **/agents：小さなClaude Code（専門のサブエージェント）」を作成・管理** 作業を細分化して各サブエージェントに任せる（委譲する）ことで、効率的にタスクを進めることができます。サブエージェントごとに使用するAIモデル（Opus、Sonnet、Haikuなど）を指定できます。複雑な判断が必要ない単純作業（文字起こしの整理など）には、素早く処理できるモデル（Sonnetなど）を割り当てることで、無駄なコストを抑えつつ全体の効率を上げることができます。
* **/rewind：魔法のやり直しボタン** 「指示をミスしてコードがめちゃくちゃになった！」という時に、特定のメッセージの時点まで時間を巻き戻せます。まさに魔法のタイムマシン！
* **/rename：セッションにラベル貼り** 「画像生成用の部屋」「バグ修正用の部屋」など、セッションに名前をつけて整理。後で再開する時に「どれだっけ？」と迷うことがなくなります。

Claude Codeを「ただのチャットAI」以上の存在にする、ワクワクする機能です！

![](https://assets.st-note.com/img/1778633189-yprRS9hBYLxjOIf0C7cMEoNv.png?width=1200)

* **/remote-control：未来の操作感（ブラウザ・スマホ同期）** PCで動かしているセッションを、ブラウザ版（claude.ai/code）やスマホアプリと同期できます。自宅のMac miniで起動しっぱなしにして、外出先からスマホでポチポチ開発……なんて未来っぽい使い方も可能です！
* **/mcp：外部ツールとの合体** SlackやNotion、Googleカレンダーなど、外部サービスと連携させるための入り口です。Claudeをあなたの秘書として最強にパワーアップさせましょう。
* **/insights：自分だけの分析レポート** 過去1ヶ月の使い方のクセを分析し、HTML形式でオシャレな改善レポートを出してくれます。自分の「使いこなしレベル」が見える化されて楽しいですよ！
