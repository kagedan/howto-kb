## はじめに

AIと会話していると、すごくいいアイデアが出たり、深い内省ができたりすることがある。でも、チャットが終わるとその内容は流れていく。「あの時Claudeと話した内容、なんだったっけ…」を何度か繰り返して、ようやく気づいた、考えた跡を残す仕組みの必要性。

この記事では、Claude（claude.ai）とObsidianをMCP経由で接続し、会話から生まれたアイデアやジャーナリングの記録を構造化して保存する仕組みの構築過程を紹介する。  
最終的にはClaude.aiのカスタムスキルとして `/obsidian ideas` と打つだけで保存できるところまで。

## この記事で扱うこと

* ObsidianとClaudeをMCPで接続し、Claudeから直接ノートを作成できるようにする
* 運用ガイド（`Obsidian-Claude.md`）をvault内に設置し、テンプレートとルールを一元管理する
* Claude.aiのスキル機能を使って、`/obsidian フォルダ名` で保存できるワークフローを構築する
* ジャーナリング用プロジェクト（Lifewell）のレポート構造に合わせてテンプレートを最適化する

## 背景: なぜObsidianなのか

NotionでもGoogle Docsでもなく、Obsidianを選んだ理由はシンプルだ。

**ローカルのMarkdownファイル**であること。つまり、データは自分のものであり、フォーマットは標準的で、どんなツールとも組み合わせられる。そしてObsidianの`[[wikilink]]`によるノート間のリンクは、「点と点を繋げて思考の地図を広げる」のに最適だった。

ただし、Obsidianは書くためのツールであって、「AIとの会話内容を自動で構造化して保存する」機能は持っていないし、wikilinkを自分で関連記事探してつけていくのも大変だしイケてない。  
Claudeとの連携を行うことで、ここの自動化ができ、勝手にリンクが広がっていくはず。

## Phase 1: ObsidianとClaudeの接続

### Obsidian MCPプラグインの導入

#### Obsidian側

#### Claude側

* `/Library/Application Support/Claude/claude_desktop_config.json`にローカルmcpサーバー登録を行う

```
"mcpServers": {
    "obsidian": {
      "command": "npx",
      "args": [
        "-y",
        "@huangyihe/obsidian-mcp",
        "--vault-path",
        "/Users/username/Library/Mobile Documents/iCloud~md~obsidian/Documents/My Memos",
        "--api-token",
        "MY API KEY",
        "--api-url",
        "https://localhost:27124"
      ]
    }
  },
```

これで以下のツールがClaude側から使えるようになる。

* `obsidian:create_note` — ノートの新規作成
* `obsidian:read_note` — 既存ノートの読み込み
* `obsidian:list_notes` — ノート一覧の取得
* `obsidian:delete_note` — ノートの削除
* `obsidian:move_note` — ノートの移動

接続後、Claudeに「Obsidianのノート一覧を見せて」と言えば、vault内のファイル構成が返ってくる。

```
Obsidian-Claude.md
💡ideas/外見と中身で意外性を持たせる.md
💡ideas/裏キャラAI - Agent Teams プロジェクト計画書.md
📓journal/2026-03-22 愛のかたちについて考えた日.md
📥inbox/サービスをユーザー自身の手で育てて行く必要あり.md
...
```

これだけで、Claudeが「Obsidianの中に何があるか」を把握し、関連ノートへのリンクを貼ったり、既存ノートとの重複を避けたりできるようになる。

### 実際に感じた注意点

運用する中でいくつかハマったポイントがあった。

`obsidian:move_note`はセッション中にサイレントに失敗することがある。ファイル名の変更は「削除→再作成」が確実だった。また、`tool_search`でObsidianツールを再検索すると、接続が切れてその後のセッションで使えなくなることがある。この場合は新しいチャットを開くのが手っ取り早い。ファイル作成時にパスの`.md`が二重に付くケースもあったので、`list_notes`で作成後に確認する癖をつけた方がいい。

## Phase 2: 運用ガイドの設計

MCPで繋がったのはいいが、毎回「こういうテンプレートで、このフォルダに、このルールで保存して」と指示するのは面倒だ。そこで、**vault内に運用ガイドとなるMarkdownファイルを置く**ことにした。

### `Obsidian-Claude.md` の役割

vault直下に `Obsidian-Claude.md` を作成し、以下を定義した。

**フォルダ構成とその用途:**

| フォルダ | 用途 | いつ使う |
| --- | --- | --- |
| `📥inbox/` | 思いつき・気づき・未整理メモ | まだ形になっていない段階 |
| `💡ideas/` | 練られたアイデア・プロジェクト計画 | アイデアの方向性が固まったら |
| `📓journal/` | ジャーナリングの内容と結果 | 振り返り・記録を残したいとき |

**フォルダごとのテンプレート:**

ideas用にはコンセプト・背景・技術スタック・課題・次のアクションといった構造化テンプレを、journal用にはテーマ・内容・分析・感情の可視化・アクション・次回への問いといったLifewell対応テンプレを、inbox用にはシンプルな自由記述テンプレをそれぞれ定義した。

**リンクのルール:**

ノート作成時に `list_notes` で既存ノートを確認し、関連があれば `[[wikilink]]` でリンクを貼るルールにした。これにより、ノート同士のつながりが自動的に編まれていく。

### なぜvault内にガイドを置くのか

「テンプレートをClaudeのプロンプトやスキルに直書きする」という選択肢もあった。でも、vault内に置くメリットは大きい。

まず、テンプレートを変えたいときにObsidianで直接編集すれば即反映される。スキルの再パッケージやシステムプロンプトの変更は不要だ。次に、Obsidian上でこのガイド自体を閲覧・管理できるため、例えclaudeじゃないAIツールを用いるにしても同じ型でメモを生成できる。  
そして何より、ガイドそのものがナレッジの一部になるので、vault内で一覧したときに「自分がどういうルールで知識を整理しているか」が残る。

## Phase 3: ジャーナリングとの統合

ここからが、この仕組みの真価が発揮されるところだ。

### Lifewellプロジェクト

Claude.aiのプロジェクト機能を使って、「My Journaling」というプロジェクトを運用している。このプロジェクトにはLifewellというウェルビーイング専門のジャーナリングコンパニオンとしての設定が入っていて、CBTやマインドフルネスをベースにした対話ができる。

Lifewellのレポートは6段階の構造を持っている。エンパシー（共感）、記録（日付/感情/トリガー/洞察の構造化）、分析（パターン抽出）、可視化（感情チャート）、アクション（SMART目標）、次回への問い、という流れだ。

### テンプレートとの整合性チェック

最初のjournalテンプレートは、Lifewellの出力構造を十分にカバーできていなかった。具体的には3つの問題があった。

**1. 感情データの構造化**

気分スコア（3.5/10）や感情ラベル（悲しみ 8/10、不安 7/10）が「内容」セクションに埋もれてしまっていた。これではObsidianのDataviewプラグイン等で感情のトレンドを追えない。

解決策として、YAML frontmatterに `mood` と `emotions` を追加した。

```
---
date: 2026-03-22
mood: 3.5
emotions:
  - 悲しみ・寂しさ: 8
  - 自責: 7
  - 不安: 7
tags:
  - journaling
  - 恋愛
source: claude-conversation
---
```

これにより、Dataviewクエリで「過去1ヶ月の気分スコアの推移」や「特定の感情が強い日のパターン」を抽出できるようになった。

**2. 感情の可視化セクションの欠落**

Lifewellがセッション中に出したテキストチャートやグラフの要約を残す場所がなかった。「感情の可視化」セクションを新設した。

```
## 感情の可視化
| 感情 | 強度 |
|------|------|
| 悲しみ・寂しさ | ████████░░ 8/10 |
| 自責 | ███████░░░ 7/10 |
| 不安 | ███████░░░ 7/10 |
```

**3. 次回への問いの独立**

Lifewellの6番目のステップ「次回への問い」は、次のセッションの起点になる大事な要素だ。これがアクションに混ざっていたので、独立セクションにした。

```
## 次回への問い
今週、〜以外で愛情を感じた瞬間はあった？
```

## Phase 4: スキルの構築

ここまでの仕組みを毎回手動で指示するのは非現実的だ。Claude.aiの**スキル機能**を使って、ワンコマンドで動くようにした。

### `/obsidian` スキルの設計

設計で最も意識したのは**使いやすさ**だ。

**呼び出し方をシンプルに:**

```
/obsidian ideas    → 💡ideas に保存
/obsidian journal  → 📓journal に保存
/obsidian inbox    → 📥inbox に保存
```

毎回絵文字を入力するのは面倒なので、エイリアスマッピングを設けた。`ideas` → `💡ideas`、`journal` → `📓journal`、`inbox` → `📥inbox`。`idea`（単数形）でも`💡ideas`に入る。

**テンプレートはスキルに埋め込まない:**

スキルのワークフローは以下のように設計した。

1. **毎回 `Obsidian-Claude.md` を読み込む** — テンプレートの最新版を常に参照
2. エイリアスでフォルダを解決
3. 会話の内容から保存すべき情報を判断
4. タイトルを自動生成（journalは日付プレフィックス付き）
5. `list_notes` で既存ノートを確認し、関連リンクを付与
6. テンプレートに沿ってノートを作成
7. 簡潔に完了報告

テンプレートをスキル側に持たせなかったことで、テンプレートの変更はvault内の `Obsidian-Claude.md` を編集するだけで済む。スキルの再パッケージは不要だ。この「関心の分離」が効いている。

### パッケージングとインストール

スキルはSKILL.mdを含むフォルダをzip化し、拡張子を `.skill` にしたものだ。Claude.aiの設定 → プロフィール → スキルからアップロードすればインストール完了。

## 実際の使い方

### アイデアの保存

💡ideaスレッドでアプリのコンセプトを壁打ちした後:

Claudeが会話内容を分析し、ideasテンプレに沿ってコンセプト・背景・技術スタック・課題・次のアクションを整理してノートを作成する。既存ノートとの関連があれば `[[wikilink]]` も自動で付与される。

### ジャーナリングの記録

My Journalingスレッドでの内省セッション後:

Lifewellのレポート構造に沿って、frontmatterに感情データが構造化され、分析・可視化・アクション・次回への問いが整理されて保存される。

### ちょっとしたメモ

会話中にふと出た気づき:

シンプルな自由記述で📥inboxに保存。後から💡ideasに昇格させるかどうかは自分で判断する。

## 運用してみての所感

### うまくいっていること

**AIとの対話内容を自動で残せるようになった**  
以前はAIと良い対話ができても、スレッドを閉じたらそれきりだった。今は `/obsidian` と打つだけで構造化された記録が残る。AIとの対話は、それだけでインプットの宝庫なので、それを気軽に残せるようになったのは大きい。

**ノート間のリンクが育っていき、新しいアイデアに繋がる**  
Claudeが `list_notes` で既存ノートを確認してリンクを貼るので、意図しなかったノート同士のつながりが見えてくることがあり、これが例えば新しいアプリケーションのアイデアとか、SNS投稿のネタ発掘とかに繋がったりした。これがObsidianの真骨頂。

**感情データの蓄積でメンタルが少し安定してきた**  
journalのfrontmatterに `mood` と `emotions` を入れたことで、「あの週はなぜ調子が悪かったのか」を後から振り返れるようになり、特にネガティブな感情との向き合い方が前とは変わってきたように感じる。  
元々ジャーナリングは、メンタルを安定させ自分軸をぶらさないようにする目的で始めたし、効果は出ていると思う。  
まだDataviewでの可視化は組んでいないが、データが溜まってきたら分析の幅が広がるはず。

### 改善の余地

**Obsidian MCPの接続安定性**  
セッション中に接続が切れることがたまにある。新しいチャットを開けば復帰するが、長い対話の途中だと困る。

**テンプレートの柔軟性**  
今はフォルダごとに1つのテンプレートだが、同じ💡ideas内でも「アプリ企画」と「技術調査メモ」では最適な構造が違う。テンプレートの分岐ロジックを入れるかどうかは検討中。

**wikilinkのメンテナンス**  
ノートが増えていく度にリンクはアップデートさせて育てて行きたい。obsidianをセカンドブレインにする、arscontextaも試していきたいところ。

## まとめ: 思考のインフラを作る

この仕組みで作ったのは、単なる「メモを保存する機能」ではない。

**AIとの対話 → 構造化 → 蓄積 → 接続**

というパイプラインだ。Claudeと深い対話をして、その結果を構造化し、Obsidianに蓄積し、ノート同士をリンクで接続していく。これを繰り返すことで、自分の思考の地図が少しずつ広がっていく。

心を整えるジャーナリングも、アイデアを練る壁打ちも、その成果が流れていかずに積み上がる仕組みがあるだけで、AIとの対話の価値は何倍にもなるなと思うし、継続できそうな気がする。（実際、ジャーナリングを始めたのは3月。継続中である。）
