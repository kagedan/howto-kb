2026年5月27日（日本時間）、Anthropic公式の開発者アカウント@ClaudeDevsが「security-guidance」プラグインを発表した。

結論から言うと、**Claude Codeでコードを書くだけで、脆弱性の検出と修正提案がリアルタイムで行われるようになった。**

「セキュリティは後回し」が許されない時代に、これは全Claude Codeユーザーが知っておくべきアップデートだ。

---

## 何が変わったのか

これまでClaude Codeでセキュリティチェックをするには、自分でプロンプトを書いて「このコードに脆弱性はないか」と聞く必要があった。

security-guidanceプラグインを入れると、**Hooks（フック機能）を使って自動的にコードをレビューする**仕組みになる。手動で聞く必要がない。

**具体的な動作タイミング**

Anthropicの内部テストでは、**プルリクエストでのセキュリティ関連コメントが30〜40%減少した**という結果が出ている。

---

## インストール方法

Claude Codeのターミナルで以下を実行するだけだ。

**手順1**: Claude Codeを開く

**手順2**: /plugins と入力してプラグインマーケットプレイスを開く

**手順3**: 「security-guidance」を検索してインストール

これだけで有効化される。追加の設定は不要だ。

---

## 何を検出してくれるのか

公開情報によると、25種類の危険なパターンを検出する。代表的なものを挙げる。

**eval() / new Function()** → 任意コード実行につながる危険な関数の使用を警告

**os.system() / child\_process.exec()** → コマンドインジェクション（外部入力がOSコマンドに混入する）リスクを警告

**pickle デシリアライゼーション** → Pythonで信頼できないデータを復元すると任意コード実行される危険性を警告

**dangerouslySetInnerHTML / innerHTML** → XSS（クロスサイトスクリプティング）のベクタとなるDOM操作を警告

**GitHub Actions workflowのコマンドインジェクション** → CI/CDパイプラインでの外部入力混入を警告

**ハードコードされた秘密情報** → APIキーやパスワードがコード内に直書きされている場合に警告

---

## 組織独自のルールも設定できる

プロジェクトのルートに claude-security-guidance.md ファイルを作ると、組織固有のセキュリティルールを追加できる。

たとえば「社内APIには必ずOAuth 2.0を使用すること」「ログに個人情報を出力しないこと」といったカスタムルールを書いておけば、プラグインがそのルールも含めてチェックしてくれる。

---

## 2段階レビューの仕組み

このプラグインは**パターンマッチ（第1段階）**と**モデルレビュー（第2段階）**の2段構成で動作する。

**第1段階: パターンマッチ（モデル呼び出しなし・課金ゼロ）**

ファイルを編集するたびに、25種類の危険なパターンを高速な文字列マッチングで検出する。AIモデルは使わないため、追加コストは一切かからない。

**第2段階: モデルレビュー（Claude Opus 4.7）**

end-of-turnレビュー（ターン終了時）とコミットレビュー（git commit時）では、Claude Opus 4.7がコード全体の文脈を踏まえて脆弱性を分析する。パターンマッチでは見落とすロジック上の問題も検出できる。

## 誰が使うべきか

**個人開発者** → 「セキュリティは詳しくないけど最低限は押さえたい」人に最適。無料でプロレベルのレビューが受けられる

**チーム開発** → PRレビューの負担を減らしたい。特にセキュリティ専任者がいない少人数チームに有効

**学習者** → セキュリティの基礎を実践的に学べる。脆弱性を指摘されるたびに知識が増える

---

## 注意点

**万能ではない。** AIによるセキュリティチェックは、専門のセキュリティ監査を完全に置き換えるものではない。あくまで「最初の防御線」として考えるべきだ。

**フォールスポジティブ（誤検知）はある。** 安全なコードに対して警告が出ることもある。その場合はスキップすればいい。

**本番環境のセキュリティ監査は別途必要。** プロダクションに出す前には、依然として専門ツール（Snyk、SonarQubeなど）や手動レビューを組み合わせることを推奨する。
