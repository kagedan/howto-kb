「この作業、毎回いちいち指示するの、しんどくない？」

Claude Codeをしばらく使うと、だいたいここに行き着きます。

一つ指示して、結果を見て、また指示して。 気づいたら、AIじゃなくて自分が働いている。

でも、仕事が速い人の使い方は、もう違います。

**指示を出すのをやめて、"ゴール"だけ渡している。**

やることを1個ずつ言わない。 「どうなったら完了か」を先に渡して、あとは走らせる。

プロンプトが上手い、とかじゃないです。 渡し方を変えるだけ。今日から真似できます。

保存して、上から順にどうぞ。

# なぜ"指示"だと遅いのか

指示は、1往復で1歩です。

AIが一つ終わらせるたびに、こっちが次を考えて、また打つ。 速いAIを、人間の入力速度に合わせて止めている。

考える。打つ。待つ。また考える。

AIを使っているようで、AIに待たされている。

# ①「完了の形」を先に渡す

いちばん効くのが、これ。

「◯◯して」じゃなく、「◯◯が△△になったら完了」で渡す。

✕ このデータを集計して ◯ このデータを月別で集計して、先月比が一目でわかる表になったら完了

完了の形があると、AIは自分で判断できます。 まだ足りないのか、もう終わりなのか。

こっちが逐一チェックしなくても、勝手に着地する。

逆にこれがないと、AIは猛スピードで、見当違いの方向に走り続けます。 速さは、行き先を決めてはじめて武器になる。

# ②同じルールを、二度と打たない

「日本語で」「この書式で」「このフォルダに」。

毎回、打っていませんか。

プロジェクトに CLAUDE.md を1枚置きます。 そこにルールを書いておくと、Claude Codeは動く前に毎回それを読む。

一度書けば、同じ指示はもう要りません。

# ③手が空いたら、勝手に動かす

Claude Codeには hooks という仕組みがあります。

「保存したら、テストを走らせる」 「完了したら、通知する」

こういう"お決まりの流れ"を登録しておくと、あなたが言わなくても回る。

指示のうち、機械でいい部分を、機械に返す。

# ④いつもの道具と繋ぐ

Claude Code単体だと、AIは自分の手元しか触れません。

MCPという規格で繋ぐと、普段使っているツールを直接読み書きできる。 Claude CodeでもCodexでもCursorでも、MCP対応ならなんでも。

AIが動ける範囲が、一気に広がります。

# ⑤ゴールを渡して、立ち去る

海外では最近、こう言われ始めています。

**「目標を渡して、立ち去れ（give an objective and walk away）」。**

①〜④が揃うと、これができる。

ゴールを渡す。 コーヒーを淹れて戻る。 終わっている。

手を動かすのはAI。人間がやるのは、行き先を決めることだけ。

仕事なら、資料も集計も調べ物もこの回り方になる。 個人なら、副業や資格勉強の段取りを丸ごと任せられる。

# 1個だけ、穴があります

この使い方、弱点がひとつ。

渡した「ゴール」と「今どこまで来たか」を、AIは忘れます。 チャットを閉じれば流れる。次の日、また一から説明。

せっかくゴールを渡す運用にしても、毎朝リセットされたら意味がない。

なので僕らは、ゴールと進み具合だけを置いておく場所を作りました。

Addnessという無料のツールです。 ここに目標を置いてClaude Codeと繋ぐと、渡したゴールと「完了の形」がずっと残る。 片付けた分は自動で記録されて、次に呼ぶときの続きになる。

👉 hp.addness.com

# まとめ

天才は、指示していません。 ゴールと、完了の形を、先に渡しているだけです。

②ルールは CLAUDE.md に置く 
③hooksで自動で動かす 
④MCPでいつもの道具と繋ぐ 
⑤ゴールを渡して、立ち去る

保存して、明日、①から一つ試してみてください。
