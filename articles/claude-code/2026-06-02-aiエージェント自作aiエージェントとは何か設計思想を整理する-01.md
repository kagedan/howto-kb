## AIエージェントとは何か

「AIエージェント」という言葉は広く使われていますが、ここでは以下のように定義します。

> **自律的にツールを使いながら、目標を達成するまでループし続けるLLMの応用**

通常のチャットとエージェントの違いを図にするとこうなります。

```
通常のチャット
  ユーザー入力 → LLM → 回答（終わり）

エージェント
  ユーザー入力 → LLM → ツール呼び出し
                  ↑         ↓
                  └── 結果を受け取ってループ → 最終回答
```

ポイントは**ループ**と**ツール**の2つです。LLMが自分でツールを呼び出し、その結果を見てまた考え、目標が達成されるまで繰り返します。

## 設計のポイント

### 1. LLMを抽象化する

OllamaとClaude APIを切り替えられる設計にします。抽象基底クラスを定義し、どちらのLLMも同じ `chat()` メソッドで呼び出せるようにします。

src/llm/base.py

```
from abc import ABC, abstractmethod

class LLMClientBase(ABC):

    @abstractmethod
    def chat(
        self,
        messages: list[dict],
        tools: list[dict] | None = None,
        system: str | None = None,
    ) -> dict:
        """
        Returns:
            {
                "content": str | None,
                "tool_calls": list | None,
                "stop_reason": str    # "end_turn" | "tool_use"
            }
        """
        pass
```

クライアントの生成はファクトリ関数に集約し、起動引数で切り替えられるようにします。

src/llm/\_\_init\_\_.py

```
from .base import LLMClientBase
from .ollama_client import OllamaClient
from .claude_client import ClaudeClient

def create_client(llm: str = "ollama", model: str | None = None) -> LLMClientBase:
    """LLMクライアントのファクトリ関数。"""
    if llm == "ollama":
        return OllamaClient(model=model or "qwen2.5-coder:7b")
    elif llm == "claude":
        return ClaudeClient(model=model or "claude-sonnet-4-20250514")
    else:
        raise ValueError(f"未対応のLLM: {llm}。'ollama' または 'claude' を指定してください。")
```

起動時に `--llm` 引数を渡すだけでLLMが切り替わります。

```
python src/agent.py                         # Ollama（デフォルト）
python src/agent.py --llm claude            # Claude APIに切り替え
```

ローカルLLMで開発・検証し、必要に応じてClaude APIに切り替えることで、コストを抑えながら品質を確認できます。

### 2. 人間の確認を挟む

ファイルの書き込みやコマンド実行は、必ずユーザーの承認を得てから実行します。「自律的に動く」と「人間がコントロールする」のバランスがエージェント設計の核心です。

```
書き込み系ツール（write_file / patch_file / run_command）
  → 実行前に差分・コマンドを表示して確認を取る
```

### 3. 段階的に実装する

一気に作らず、13のステップに分けて実装します。各ステップで動作確認をしながら進めます。

**Phase A：土台（API呼び出し・REPL）**

## 技術スタック

| 項目 | 内容 |
| --- | --- |
| 言語 | Python 3.11以上 |
| LLM（メイン） | Ollama ローカルLLM（既定：qwen2.5-coder:7b） |
| LLM（サブ） | Claude API（claude-sonnet-4-20250514） |
| LLM切り替え | 起動時引数 `--llm ollama` / `--llm claude` |
| 動作形態 | ターミナルREPL |
| 依存ライブラリ | `openai`（Ollama互換接続）、`anthropic`、`python-dotenv`、`difflib`、`subprocess` |
| リポジトリ | <https://github.com/pekopagu/agent01> |

### OllamaをメインにしてClaude APIをサブにする理由

* **コスト**：ローカルLLMは無料で何度でも試せる
* **速度**：ネットワーク遅延がない
* **学習効果**：ローカルで動かすことでLLMへの理解が深まる
* **比較**：同じコードで2つのモデルの挙動を比較できる
