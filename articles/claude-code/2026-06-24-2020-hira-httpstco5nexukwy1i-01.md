2026 年 6 月 23 日、Anthropic が **Claude Tag** を発表した。Slack のチャンネルに @Claude をタグ付けしてタスクを委任する、という製品である。

一見すると「Slack で Claude と話せる」既存の体験の延長に見えるが、中身は別物だ。旧来の Claude in Slack を置き換えるものであり、位置づけとしては「Claude Code の進化版」とされている。本記事では、何が新しいのか、裏側はどうなっているのか、導入するならどう設計するか、を一次情報ベースで整理する。

## Claude Tag とは何か

Claude Tag は、Slack のチャンネルに参加した Claude に対して、誰でも @Claude でタスクを投げられる仕組みである。タグ付けすると、Claude はタスクをステージに分解し、与えられたツールを使って順に処理し、完了したらスレッドに結果を返す。進行中はチェックリストをスレッドに出すため、何をやっているかがチャンネル全体から見える。

Anthropic 社内ではすでに主要な業務手段になっており、プロダクトチームのコードの 65% が社内版 Claude Tag で生成されている、とされている。用途はエンジニアリングに限らず、プロダクト指標の追跡、サポートチケットの処理、バグの根本原因調査まで広がっている。

提供範囲は現時点で Claude Enterprise / Team のベータ。既存の Claude in Slack アプリを置き換えるもので、管理者は 30 日以内にオプトインで移行できる。なお旧 Claude in Slack は 2026 年 8 月 3 日に新しい Claude Tag 体験へ切り替わる予定とされている。

## 3 つのアクセス方法と課金の違い

Claude Tag にはアクセス方法が 3 つある。チャンネルでの @Claude タグ付け、DM、そして Slack の AI アシスタントパネルだ。ここで重要なのは、**チャンネルと DM で動く「中身」が違う**点である。

- **チャンネルのタグ付け** — 組織のアイデンティティで動く。管理者がそのチャンネル向けに設定したツール・リポジトリ(Access bundle)を使い、**課金は組織の利用残高**から引かれる。
- **DM / アシスタントパネル** — あなた自身の claude.ai アカウントで動く。あなたが有効にしている Web 検索や接続ツールを使い、**課金は個人アカウント**。組織のスペンド上限の対象外。
この違いは後述の「見分け方」でも効いてくる。DM で @Claude が動いても、それは組織の Claude Tag が有効である証拠にはならない。判別は必ずチャンネル内で行う必要がある。
