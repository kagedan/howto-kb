## 1-1. Claude Code（部下AI、中心）

![](https://assets.st-note.com/img/1779321755-EO0mKMlPk4j5vWnyrspT1oIQ.png?width=1200)

## 1-3. GitHub（ネット上の金庫）

![](https://assets.st-note.com/img/1779321828-rJkNpLMCTRjV2Y8PBoHfwvK9.png?width=1200)

PC のフォルダの中身を、**ネット上にコピーで保管しておく「金庫」**。

### GitHub の最大の価値：失敗のコストをゼロにする

GitHub があると、「うっかり編集を間違えた」「ファイルを消した」「サイトを壊した」── そういう事故が起きても、**過去のどの時点の状態にも即座に戻せます**。

私が KAGURA を開発している3ヶ月の間に、何度も「サイトが動かなくなった」「設定をミスった」事故が起きました。すべて、GitHub で **5分以内に元の状態に戻し** ました。

「失敗できる」という心理的安全性が、上位20%の挑戦量を支えています。

![](https://assets.st-note.com/img/1779321834-m76hxatqHbDdoynLr0EZSpQV.png?width=1200)

## 1-4. Cloudflare（サイトを保管・公開・管理する場所）

![](https://assets.st-note.com/img/1779321839-Egj2wMQtev0YThnmxbBkVZW6.png?width=1200)

GitHub に保管した中身を、**そのまま世界に公開して、1ヶ所で管理できる場所**。

ドメイン名（example.com みたいなもの）を取れば、5分後にはあなたの「お店」が世界中の人に見える状態になります。

「サーバー契約」「インフラ管理」「複数社のサービスを組み合わせる」── そういう面倒さは、Cloudflare ではすべて消えます。

![](https://assets.st-note.com/img/1779321870-HCyaxUYu6Xjv8Vd0p7oR4Lrz.png?width=1200)

* **サーバー**：サイトやアプリを動かす本体（Pages / Workers）
* **配達網**：世界中の倉庫から、近い人へ高速配信（CDN）
* **ドメイン管理**：example.com の名前を住所に変換（DNS）
* **セキュリティ**：攻撃や不正アクセスから守る（WAF / Bot Management）
* **データ置き場**：会員・予約・画像・動画を保管（D1 / R2 / KV）

**1社で全部** カバー、**個人なら、ほぼ無料**。

私の場合、3つの整体院サイト・KAGURA・Mai POST・note 用画像置き場、合計10個以上のシステムを Cloudflare で動かしていて、月額は数百円〜数千円です。サーバーコストで赤字、みたいな話は Cloudflare では起こりません。

### Cloudflare の料金感

* **無料プラン**：個人ホームページ、小規模 SaaS、月数万アクセスまでなら完全無料
* **Pro プラン**（月20ドル）：商用サイト、CDN高速化、攻撃対策強化
* **Workers Paid**（月5ドル + 従量）：API バックエンドを本格運用する人向け

ほとんどの個人は無料プランで十分です。

ブラウザでクリックする作業を、**そのまま AI に渡せる Chrome 拡張**。

「ブラウザでクリックする作業」を AI に丸投げできます。

### Claude in Chrome の用途

* 競合5社の料金ページを開いて、料金プランを表にまとめる
* 予約サイトの空き時間を一覧化
* フォームの下書き入力（住所・電話番号・名前を、コピペじゃなくて意味を理解して入れる）

### 注意：使ってはいけない場所

* **銀行・証券のサイト**
* **クレジットカード情報を入れるフォーム**

これらは、人間が直接操作するルールにしておきましょう。

AI に間違って情報を渡すと、不正アクセスのリスクが上がります。

### Claude in Chrome のインストール

1. Chrome で Chrome ウェブストアを開き、「Claude」で検索
2. Anthropic 公式拡張を選んで「Chrome に追加」
3. ツールバーにピン留めし、拡張アイコンでログイン
4. 適当なページで「このページを読んで要約して」と頼んで動作確認

これだけ。所要時間 5分。

## MCP接続 ＝ AI 用の「コンセント規格」

![](https://assets.st-note.com/img/1779321881-eMKfHzjpmEYWkqniJy5L68Ic.png?width=1200)

コンセントの形が世界共通だから、どこの家電も差せる。MCP という規格があるから、**Claude Code は Notion でも Gmail でもカレンダーでも、ぜんぶ繋いで使える**。

### MCP で繋がる代表サービス（私が実際に繋いでいるもの）

* **Notion** ── 会話ログ、note販売記事管理、KAGURA 関連メモ、タスク管理
* **Google Workspace（Gmail / Calendar / Sheets / Drive）** ── メール、予定、表計算、ファイル
* **GitHub** ── コードと原稿の保管庫

PC の中の「黒い画面」に、文字で命令を送る入口。**Claude Code も Codex も、この黒い窓から呼ぶ** バージョンがあります。

## API接続 ＝ ソフト同士の「連絡通路」

自分のアプリやサイトの中から、AI を呼び出す「窓口」。**個人事業主こそ API**。月数千円で、人件費10万円分の作業が自動化できます。

### API の代表的な活用シーン

* **決済データの取り込み**（Square、Stripe、PayPay、SBPS）── 売上の自動集計
* **予約管理の自動化**（Calendly、ホットペッパーサロンボードなど）
* **顧客通知**（LINE、SMS、メール）── リマインダー、クーポン配信
* **会員管理の同期**（freee、マネーフォワード）── 経理の自動化
* **自分のサイトに AI を埋め込む**（Anthropic API、OpenAI API）── チャットボット

API は「最初の繋ぎ込み」だけが少し難しい。でも、Claude Code に頼めば30分で終わります（特典1で具体手順を公開）。

## 1行で違いを覚える

* **MCP** ＝ AI ←→ 外のサービスを繋ぐ"コンセント"。AI が他の道具を呼ぶときの規格
* **CLI** ＝ あなた ←→ PC を繋ぐ"黒い窓"。あなたが PC に命令して、PC 上の AI が動く
* **API** ＝ アプリ ←→ アプリを繋ぐ"連絡通路"。ソフト同士が情報をやりとりする

3つは排他ではありません。**1つのプロダクトで全部使う** のが普通です。

私の KAGURA も、MCP で Notion / Cloudflare、API で Square / LINE、CLI で開発作業、と3つ全部使っています。
