## はじめに

クラシルでSREをしています、moikeiです。

Claude Codeは多機能でアップデートも多いですが、もはやよく使う機能のみで「なんとなく使っている」状態の方も多いと思います。私もそうでした。  
v2.1.90で、Claude Codeの主要機能を体験できる公式チュートリアル **`/powerup`** コマンドが追加されました。  
機能を「読んで理解する」より「動かして理解する」方が定着しやすいため、基礎からざっと体系的に把握できて便利でしたので紹介します。

## 前提

* **Claude Code**: v2.1.90以降（本記事はv2.1.112で確認）
* **チュートリアルの言語**: 英語固定（切り替え不可）

## 使い方

Claude Codeのセッションを起動し、`/powerup` を実行するだけです。

実行するとターミナル上にインタラクティブなメニューが表示されます。矢印キーでレッスンを選択し、Enterで開始します。

各レッスンは**テキストによる機能説明と、操作例を示す小さなアニメーションデモ**で構成されています。実際に手を動かして試すハンズオン課題はなく、読み終わったら `Enter` で完了マーク、`Esc` で一覧に戻る、という感じです。  
![](https://static.zenn.studio/user-upload/eebce2032d82-20260417.png)

思ったよりも最小限にとどめており軽量で、全体的にサクサク進められます。

## 各レッスンの日本語訳

レッスンは全て英語表記のため、全10レッスンを日本語訳で整理しました。

### 1. Talk to your codebase

プロンプトのどこにでも `@` を入力すると、ファジー検索でファイルを添付できます。Claudeが回答前にそのファイルを読み込むため、コードを貼り付ける必要はありません。

* `src/app.ts:42` のように行番号まで指定すれば、Claudeが該当箇所にピンポイントで飛びます。双方向で機能し、Claudeが引用したファイルをクリックすればエディタで開けます。
* `@folder/` でディレクトリツリー全体を添付できます。

### 2. Steer with modes

`Shift + Tab` で権限モードを切り替えます。モードごとに、Claudeが行動前に確認する量が変わります。

* **default**: 編集のたびに確認
* **accept edits**: 編集は自由、コマンドは確認
* **plan**: 調査と提案のみ、ファイルには触らない
* **auto**: Claudeが安全と判断した操作は自動実行

大きなリファクタは `plan`、長時間の無人実行は `auto`、という使い分けです。`/permissions` で特定コマンドを事前許可しておけば、Claudeの確認を減らせます。

### 3. Undo anything

Claudeは編集のたびに自動でチェックポイントを作成しています。`Esc` を2回押すと `/rewind` が開き、コード・会話・両方を任意の時点まで巻き戻せます。

* 間違った方向に進んでしまった時も、分岐点まで戻って別のプロンプトを試せます。git履歴は汚れません。
* `/clear` は会話だけリセットしてファイルは残す。`/branch` は会話をフォークして2つのアプローチを並行試行できます。

### 4. Teach Claude your rules

リポジトリに `CLAUDE.md` を置くと、Claudeはセッション開始時に毎回読み込みます。テストコマンド、スタイルルール、触ってほしくないディレクトリなどをここに書きます。

* `/init` でコードベースから初期版のCLAUDE.mdを生成
* `/memory` でインライン編集
* リポジトリ / ホームディレクトリ（全プロジェクト共通） / サブディレクトリ上書き の3階層で動作します

### 5. Automate your workflow

`.claude/skills/deploy/SKILL.md` にプロンプトを保存すると、`/deploy` というスラッシュコマンドになります。`/skills` で登録済みの一覧を確認できます。

Hooksは、ツール実行前・応答後・セッション開始時などのイベントで独自スクリプトを動かす仕組みです。ルール強制、ログ取り、コンテキスト注入に使えます。`/hooks` でどのイベントに何が紐づいているかを確認できます。

`/install-github-app` を実行すると、Claudeをメンション付きPRレビュワーとして呼び出せるようになります。

MCPサーバーはClaudeに新しいツールを与えます。Slackを読む、DBをクエリする、ブラウザを操作する、といった拡張が可能です。`/mcp` で利用可能なサーバーを一覧・接続できます。

* 接続すると自動でツールが使える状態になり、「カレンダー確認して」「Notion検索して」と指示するだけで動作します
* シェルから `claude mcp add my-server -- npx some-mcp-pkg` でターミナルを離れずに接続可能

### 7. Run in the background

長時間のビルドやテストスイートは、ブロッキングする必要がありません。bashコマンドの末尾に `&` を付けるとバックグラウンド実行となり、その間も対話を続けられます。完了時にClaudeが通知してくれます。

`/tasks` で実行中のタスク一覧を確認でき、Claudeは途中経過を読んで失敗時に自動でリアクションします。サブエージェントやワークフローも、このタスクキューに乗ります。

### 8. Multiply yourself

Claudeは自分自身のコピーを生成して並列動作できます。「これら5つのディレクトリをサブエージェントで調べて」と指示するだけで、ファンアウトする様子を確認できます。

* `.claude/agents/` に専門エージェント（テストランナー、コードレビュワー、ドキュメントライター等）を定義し、それぞれに専用のツールと指示を持たせられます
* `/agents` でエージェントを管理
* サブエージェントは独立したコンテキストで動作します。ブランチを分けた本当の並列セッションが必要な場合は `claude --worktree` で起動できます

### 9. Code from anywhere

`/remote-control` を実行すると、このターミナルがスマホと `claude.ai/code` から見えるようになります。出力の確認、プロンプト送信、ツール呼び出しの承認を、別デバイスから行えます。元のセッションは動き続けます。

* Web上で始めたセッションをこのターミナルに引き込むには `/teleport`。履歴を丸ごと持ってこれます
* 長時間タスクを投げてラップトップを閉じ、スマホから進捗を確認する、という使い方ができます

### 10. Dial the model

`/model` でモデルを切り替えます。難しい問題は Opus、大半の作業は Sonnet、手早い質問は Haiku。速度と深さのトレードオフです。

* `/effort` は Claude が回答前にどれだけ考えるかを制御します。難しいバグは `high`、手早い編集は `low`
* `/fast` は同じモデルのまま出力を速くするトグル

## 実際にどうだったか

軽量なのでサクサク進められました。使ったことがないコマンドは、もう一つターミナルを開いて実際に動かしながらやっていくのがおすすめです。

特に **Undo anything（`/rewind`、`/branch`）** のレッスンについて、これまでClaudeが意図しない変更をしたときは、新しい指示でリカバリしたりと、雰囲気で対処していましたが`/rewind` / `/branch` コマンドを活用できそうです。

基礎からざっとコマンドを総覧したい方は、`/powerup` コマンドで把握してみると良いかと思います。
