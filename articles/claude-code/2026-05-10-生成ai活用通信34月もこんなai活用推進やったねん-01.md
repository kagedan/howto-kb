# 活動報告

## 1. Claude Desktop / Web 利用ガイドラインの整備

[前回の活動報告](https://zenn.dev/tsukulink/articles/3655fbe7c94b26#1.-claude-code-%E3%81%AE%E5%88%A9%E7%94%A8%E5%9F%BA%E7%9B%A4%E3%82%92-aws-bedrock-%E3%81%8B%E3%82%89-claude-team-%E3%83%97%E3%83%A9%E3%83%B3%E3%81%B8%E7%A7%BB%E8%A1%8C)でも触れていた、AWS BedrockからClaude Teamプランへ移行したことにより、Claudeのデスクトップアプリを利用できるようになりました。これに伴い、社員が安全にAIツールを活用できるよう利用ガイドラインの整備を実施。  
社内のエンジニア以外の人たちもAIツールをかなり活用し始めているため、委員会としてガードレールとなるような仕組み作りに取り組んでいます。  
ガイドラインには以下のようなことを定義しています（一部抜粋・not原文ママ）

* 漏洩やプロンプトインジェクションなど、AIツールを利用する上での汎用的な注意事項（エンジニア以外の社員も利用するため、改めて注意喚起）
* アーティファクトの利用について
  + 基本的に利用は可だが、Anthropic社公式か自作のものを利用
  + その他のアーティストについてはAIラボのメンバーへ相談をしてもらう
* Coworkの利用について
  + 意図せぬフォルダを参照しないように作業フォルダを選択してからプロンプトを実行すること
  + 標準のチャットに比べて多くの使用量を消費します。単純なタスクには標準チャットを利用を推奨

などなど

## 2. 各種SKILLSのアップデート

弊社のエンジニア組織は複数のチームに分かれて一つのアプリ開発を行っています。各チームごとに必要なスキルが違ってきたりする場合もありますが、共通して利用できるスキルの更新などを順次行っています。  
具体的には、プルリク作成用のスキルやコードレビュー用のスキルなどです。  
地味に時間のかかるプルリク作成時間の削減や、一定以上・均一なコード品質保持など、人的・時間的コストパフォーマンスアップに役立てています。

SKILLSの管理方法については随時検討中ですが、特定のチームやメンバーだけで利用するスキルなどにはプレフィクスなどをつけて管理・検索しやすくしています。

e.g.`.claude/skills/${team_name}-${skill}/SKILL.md`  
これにより、可読性やスラッシュコマンドでの検索性を上げています

## 3. Claude Managed settingsの設定を実施

組織全体のClaude利用環境最適化のために、Claude Managed settingsの設定を行いました。  
Managed settingsは、ユーザ設定・プロジェクト設定などの設定よりも最優先される設定で、組織単位での設定を管理できるものです。  
これにより、秘匿な環境変数の参照や実行権限周りなどを厳格に守ることができます。

<https://code.claude.com/docs/ja/server-managed-settings>

envファイルの読み書きや`--dangerously-skip-permissions`での実行を拒否するなどの設定を行いました。

## 4. Claude Codeサンドボックス設定の導入

Bash系ツールをファイルシステム・ネットワークの両面からOSレベルで隔離して実行する「サンドボックスモード」の利用設定を行いました。  
AIエージェントが意図しないファイル操作や外部通信を行うリスクを低減しています。  
具体的には以下のような設定です

* `allowUnsandboxedCommands`をfalseにする
* `excludedCommands`でdockerとgitを除外する（ただしdockerとか全て許可すると実質sandboxの無効化になるのでもっとコマンドを絞って除外する）

## 5. 各種新規機能の検証

毎日といっても過言ではないほど日々更新される新規機能などを、他の社員より先んじてAIラボのメンバーで検証を行っています。この検証で使いどころ・リスク・導入などの検討を行っています。  
`Claude in Chrome`、`Claude Computer Use`、今回採用に至った`Managed settings`やClaudeサンドボックス環境の検証などを行いました。

## 6. AIツール利用社員のサポート

技術職以外の社員もAIツールを利用しているなかで、「こういった使い方できるのかな」とか「どうやって使うのかな」などの疑問を持っている人も多く、AIラボのSlackチャンネルに書き込まれる相談事のサポートなども行っています。  
もともと当該委員会のパーパスが「生成AI活用推進」なので、社内全体でのAI活用が広がっていくことは嬉しいことです。

## 7. 新規アップデート情報のキャッチアップ・共有

毎日、AIラボのSlackチャンネルに新しいTipsや新規機能の紹介、テックブログやXのポストなどがインプットとして投稿されます。それを蓄積して、AIラボの週次MTGで読み合わせを行っています。その中で便利なTipsから深刻な脆弱性の問題まで様々な動向を知ることができています。

### セキュリティ系

特定のソフトウェアなどにおける脆弱性の問題や、プロンプトインジェクションによるリスクや被害などの動向を追い、日々注意喚起を実施しています。  
サプライチェーン攻撃に関するセキュリティニュースや、Chrome拡張機能における脆弱性の報告など。個々人では拾いきれない情報も、AIラボで拾い、精査し共有まで行っています。

* **Chromeなどブラウザの拡張機能における脆弱性の注意喚起**

### Tips系

* Claudeのこんな機能・モデルがリリースされています
  + **Claudeのセッション保持期間（cleanupPeriodDays）の設定について**  
    セッション保持期間はデフォルトは30日だが、cleanupPeriodDaysをsettings.jsonに設定することで、チャットセッションの保持期間をコントロールできます
  + **Claude Codeからの提案コマンドの編集が可能に**  
    Claude Codeが提案する実行コマンドに対し、手動で微修正を加えてから実行できるようになりました。（それまではYes/Noの選択のみだった）
  + **opusplan モデル設定**  
    `/model opusplan`コマンドを実行することでPlanモードはOpus、作業の実行はSonnetといった使い分けを自動的に行えます。各領域に適したモデルを使うことでトークン消費量の削減や実行時間の短縮が見込めますの共有  
    [https://code.claude.com/docs/ja/model-config#opusplan-モデル設定](https://code.claude.com/docs/ja/model-config#opusplan-%E3%83%A2%E3%83%87%E3%83%AB%E8%A8%AD%E5%AE%9A)
  + **Claudeのeffort-level（思考レベル）について**  
    現在、effort-levelがデフォルトがmediumになっており、`/effort`で現在のレベルを確認できる。思考のレベルを上げたい、もしくはトークンの消費を抑えたいので下げたいとかあれば `/effort [low|medium|high|max|auto]`みたいに変更できまっせ
  + **Claude CodeのAdvisor toolについて**  
    Advisor toolとは、メインの軽量モデル（Sonnet等）が通常作業を担いつつ、難所だけ高性能モデル（Opus等）に相談できる仕組みです。  
    実行するとadvisorとして使うモデル（Opus等）を選択して有効化でき、必要に応じて高性能モデルに相談してフィードバックを得ることができるようになります。  
    デフォルトではOFFになっていますが、/advisor コマンドを実行するとadvisorとして使うモデル（Opus等）を選択して有効化でき、必要に応じて高性能モデルに相談してフィードバックを得ることができるようになります。  
    <https://platform.claude.com/docs/en/agents-and-tools/tool-use/advisor-tool>

などなど
