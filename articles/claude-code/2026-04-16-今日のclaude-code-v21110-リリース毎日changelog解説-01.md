v2.1.110では、ターミナル体験を大きく変える `/tui` コマンドの追加と、IDE連携・プラグイン管理周りの改善が中心です。

**今回の注目ポイント**

1. **`/tui fullscreen` でフリッカーフリーなフルスクリーンモード** — ちらつきのない没入型UIが同一会話のまま利用可能に
2. **Write toolがIDE diff編集をモデルに通知** — ユーザーが提案内容を書き換えた事実をAIが認識できるように
3. **`--resume`/`--continue` がスケジュールタスクを復活** — セッション再開時にcron・loopの未完タスクが自動復帰

---

## `/tui fullscreen` でフルスクリーンモードが同一会話のまま切り替え可能に

ターミナルでClaude Codeを日常的に使っている方向け

これまでもフルスクリーン表示は試験的にありましたが、今回 `/tui` コマンドとして正式に追加されました。会話の途中でも `/tui fullscreen` と打つだけで、フリッカーのないフルスクリーンレンダリングに切り替わります。

あわせて関連する変更もいくつか入っています:

* **`autoScrollEnabled` 設定が追加。** フルスクリーンモードで自動スクロールをOFFにできます
* **`Ctrl+O` の挙動が変更。** 通常/詳細トランスクリプトの切り替え専用になりました。フォーカスビューの切り替えは新しい `/focus` コマンドに分離されています
* フルスクリーンでテキスト選択中にツール実行するとCPU使用率が跳ね上がるバグも修正済み

フルスクリーンで集中して作業したい人にはかなり嬉しいアップデートです。

---

## Write toolがIDE上でのdiff編集をモデルに通知するように

VS CodeやJetBrainsなどのIDE拡張でClaude Codeを使っている方向け

地味だけど効く改善です。Claude CodeがWrite toolでファイルを書き込もうとしたとき、IDEのdiffビューでユーザーが内容を手動で書き換えてから承認するケースってありますよね。

これまではモデル側がユーザーの編集を認識できず、「自分の提案がそのまま通った」と思い込んでいました。v2.1.110からは、diffで編集された事実がモデルにフィードバックされるようになります。

これにより、後続のコード生成でユーザーの修正意図が反映されやすくなります。IDE連携で「直したのにまた同じコード出してくる...」という体験が減るはずです。

---

## `--resume`/`--continue` がスケジュール済みタスクを復活

`/loop` やcronスケジュールを活用している方向け

`--resume` や `--continue` でセッションを再開したとき、期限切れになっていないスケジュール済みタスクが自動的に復活するようになりました。

これまではセッションを閉じて再開すると、`/loop` で設定した定期実行などが消えてしまっていました。それがちゃんと引き継がれるようになります。長時間のモニタリングやポーリング系のワークフローを組んでいる人には朗報です。

## まとめ

v2.1.110は `/tui fullscreen` の追加でターミナル体験の選択肢が広がったのが一番大きな変化です。Write toolのdiff編集通知やセッション再開時のタスク復活など、日常的に使い込んでいる人ほど恩恵を感じる改善が多く入っています。バグ修正はMCPやフルスクリーン周りが手厚く、安定性が着実に上がっている印象です。
