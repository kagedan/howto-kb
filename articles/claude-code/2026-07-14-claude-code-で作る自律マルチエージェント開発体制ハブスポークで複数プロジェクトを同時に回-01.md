# Claude Code で作る自律マルチエージェント開発体制(ハブ&スポークで複数プロジェクトを同時に回す)

Claude Code の単一セッション運用は、複数プロジェクトを何ヶ月も並行で回そうとした瞬間に破綻する。本書はその診断から、監督役(ハブ)と実働役(スポーク)に分けて自律稼働させる体制を、実際に数ヶ月運用している設定ファイル・運用ルール・失敗談を一次情報として解説する。tmux での並列起動そのものより、状態をファイルに整理して落ちても即復旧する設計・人間の承認ゲート・モデル階層によるコスト配分・実際の障害対応まで踏み込む、長期運用に耐える「体制の作り方」の本。
