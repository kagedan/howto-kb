## 0. そもそも「Claude」って何？
 
「もう知ってるよ!」という方は[1章](#1-claude-opus-47を一言でいうと)まで飛ばしてOKです。
 
### Claude = Anthropic社が作っているAIアシスタント
 
ChatGPTのライバル、と思えばだいたい正解です。テキスト生成・コーディング・画像理解などができる **大規模言語モデル(LLM)** で、Anthropic社が開発しています。
 
### モデルの命名ルール
 
Claudeには**サイズ違い**のモデルがあります。
 
| モデル名 | ポジション |
|---------|-----------|
| **Opus** | 最強・最賢い・一番高い |
| **Sonnet** | バランス型・主力 |
| **Haiku** | 軽量・高速・安い |
 
これに **バージョン番号(4.5、4.6、4.7...)** が付きます。
今回登場した **Claude Opus 4.7** は、「一番賢いやつの最新版」というポジションです。
 
### 知っておくと話が早い用語
 
これから出てくる単語をサクッと整理しておきます。
 
- **トークン**: AIが文章を処理するときの最小単位。日本語だと「1文字 ≒ 1〜2トークン」くらいの感覚。
- **コンテキストウィンドウ**: 一度にAIに渡せる文章の最大量。「短期記憶の容量」みたいなもの。
- **エージェント / エージェンティック**: AIが自分で考えて、複数ステップのタスクを自走で進めること。「指示待ち」じゃなく「自分で動く」AI。
- **マルチモーダル**: テキスト以外(画像など)も扱える能力のこと。
- **Thinking(推論)モード**: 答える前にAIが内部で「考える」フェーズを挟むモード。じっくり考える代わりに時間とコストがかかる。
これだけ押さえておけば大丈夫です!
 
 
## 1. Claude Opus 4.7を一言でいうと?
 
> **「Opus 4.6を、より粘り強く・より目が良くなるように鍛え直したマイナーアップデート版」**
 
派手な新機能というより、「実務で使ったときの完成度を上げてきた」アップデートです。
 
公式が推している主な改善ポイントを5行で:
 
1. 🧑‍💻 **コーディング性能が13%向上**(社内93タスクベンチマーク)
2. 👁️ **画像認識の解像度が約3倍**に拡大
3. 📚 **1Mトークン(100万)のコンテキスト**が標準価格で使える
4. 🤖 **長時間タスクで「自分の出力を自分で検証する」能力が向上**
5. 🛡️ **サイバーセキュリティ系の悪用を防ぐリアルタイム防御を新搭載**
ここから一個ずつ噛み砕いていきます。
 

 
## 2. アップデート内容を噛み砕いて解説
 
### 2-1. コーディング性能の向上
 
社内ベンチマークで **Opus 4.6比 +13%** の改善。Opus 4.6でも、Sonnet 4.6でも解けなかった4つのタスクが新たに解けるようになったとのことです。
 
特に「**長時間にわたる複雑な作業**」での改善が大きく、これまで人間が常に張り付いて監督しないとダメだったような難しい開発タスクも、安心して任せやすくなったとAnthropicは述べています。
 
#### 具体的に何が嬉しいのか
 
- ❌ 旧: 「だいたい動くけどよく見ると微妙にバグってる」コードを出してくる
- ✅ 新: **計画段階で論理的な穴を自分で見つけて潰す**
実装を始める前にAIが自分で「あれ、ここの仕様だと矛盾するな」と気づいてくれる、というイメージです。
 
### 2-2. 画像認識の解像度が約3倍に
 
これは地味に**革命的**なアップデートです。
 
| | 旧モデルまで | Opus 4.7 |
|---|---|---|
| 最大解像度 | 1568px / 1.15MP | **2576px / 3.75MP** |
 
#### なんで嬉しいの?
 
- 📸 **スクショの細かい文字が読める**: 設定画面の小さい文字、エラーメッセージの末尾までちゃんと認識
- 🖱️ **コンピュータ操作系のエージェントで超有利**: 画面上の座標とピクセルが1:1で対応するようになり、これまで必要だったスケール計算がいらなくなった
- 📊 **図表のピクセル単位の数値転記**ができる: グラフから数値を読み取らせる用途で精度UP
- 📍 **物体検出の枠(バウンディングボックス)が正確に**
「Claude in Chrome」みたいなブラウザ操作エージェントを作っている人には恩恵がデカいです。
 
:::note warn
解像度が上がる = **画像トークン消費量も増えます**。
細かい認識が要らない用途なら、画像を縮小してから送ったほうがコスト効率が良いです。
:::
 
### 2-3. 1Mトークンのコンテキストウィンドウが標準で使える
 
ここ、見落とされがちですが**かなり重要**です。
 
#### 1Mトークンってどれくらい?
 
ざっくり言うと:
 
- 日本語の小説で **約50万字** ≒ 文庫本5〜7冊分
- ソースコードで **数十万行**
これを **一気にAIに食わせて、全部を踏まえた回答をさせる** ことができます。
 
#### 何が嬉しいか
 
- 🗂️ **巨大なコードベース全体を一発で読ませる**ことが可能(普通のリポジトリなら丸ごと入る)
- 📖 **長い設計書 + 関連コード + 過去ログ** を全部渡して質問できる
- 💰 **長文プレミアム価格が無い**(これが嬉しい)
「1Mコンテキスト」自体はOpus 4.5あたりからベータで存在していましたが、Opus 4.7では **標準のAPI価格で使える** ことが明記されています。
 
### 2-4. 新しい "xhigh" エフォートレベル
 
Claudeには「どれだけ深く考えるか」を指定する **effort(エフォート)** という設定があります。
 
これまで:
```
low → medium → high → max
```
 
Opus 4.7から:
```
low → medium → high → xhigh → max
                       ↑NEW!
```
 
`high` と `max` の間に **xhigh** が追加されました。
 
「`high`じゃちょっと足りない、でも`max`ほどお金と時間はかけたくない」という痒いところに手が届くオプションです。
 
Claude Codeでは `/effort` コマンドで切り替えられます。コーディングやエージェント用途では **high か xhigh から始めるのが推奨** とのこと。
 
### 2-5. Claude Codeのアップデート
 
ターミナル上で動くAIコーディングツール **Claude Code** にも便利機能が追加されました。
 
#### `/ultrareview` コマンド
 
「シニアエンジニアによるコードレビュー」をシミュレートするコマンドです。
 
普通のコードレビューが「シンタックスエラー探し」レベルだとすると、`/ultrareview` は:
 
- 設計上の微妙な欠陥
- ロジックの抜け漏れ
- ありがちな落とし穴
をベテラン視点で指摘してくれます。リリース前の最終チェックに便利。
 
#### Auto modeがMaxプランで利用可能に
 
**Auto mode** は、「いちいち許可を求めずにClaudeが自走で作業を進める」モード。これが従来は限定提供だったのが、**Maxプラン契約者なら使える** ようになりました。
 
「ファイル編集していい?」「コマンド実行していい?」と毎回聞かれて作業が止まる、という地味な摩擦が消えます。
 
### 2-6. リアルタイムのサイバーセキュリティ防御
 
ここはちょっと毛色が違う話。
 
Anthropicは別途 **Mythos** という、Opus 4.7よりさらに強力なモデルを開発しているのですが、これがあまりに強力すぎて「ソフトウェアの脆弱性を熟練ハッカー並みに発見・悪用できてしまう」という安全上の課題が判明したそうです(Project Glasswing)。
 
そこでOpus 4.7では、
 
- 🔒 **学習段階でサイバー攻撃系の能力を意図的に弱める**
- 🚨 **悪用が疑われるリクエストをリアルタイムで検知・ブロック**
という二段構えの安全策が入っています。
 
セキュリティ研究者・ペンテスターの方は、正規の用途であれば **Cyber Verification Program** に申請可能です。
 

 
## 3. 動作の「クセ」が変わった点(プロンプト要調整)
 
ここはエンジニア視点で**地味に重要**なやつです。
 
Opus 4.7は「能力アップ」だけでなく、「**応答スタイル**」もチューニングされました。Opus 4.6時代に最適化したプロンプトは、調整が要る可能性があります。
 
| 項目 | 変化 |
|------|------|
| 指示の解釈 | より**文字通り**に従う(勝手に解釈を広げない) |
| 応答の長さ | タスクの複雑さに応じて**動的に調整**(固定の冗長さがなくなった) |
| トーン | より**直接的・断定的**に(検証フォワードな表現や絵文字が減った) |
| ツール呼び出し | デフォルトでは**少なめ**、推論を優先 |
| 進捗報告 | 長時間タスクで**こまめに途中経過を出す** |
| サブエージェント | デフォルトでは**少なめ**(プロンプトで制御可能) |
 
### 特に注意したいポイント
 
**「指示してないことは勝手にやらない」** 性質が強まりました。
 
- 旧: 「リスト1件目のフォーマットに揃えて」と言うと、暗黙的に他の項目もやってくれてた
- 新: 言われた1件目しかやらない、他もやってほしいなら明示が必要
逆に言えば「**プロンプトに書いた通りに動く**」ので、 制御性は上がっています。
 
:::note info
「Opus 4.7にしたら何かおかしい…」と感じたら、まずプロンプトの**明示性**を見直すと良いです。
:::
 

 
## 4. その他、開発者が知っておくべき技術的トピック
 
### 新しいトークナイザー
 
トークナイザーが更新され、テキスト処理の効率が改善されました。
ただし入力によっては**トークン数が1.0〜1.35倍に増える**ことがあるとのこと。
 
→ コストの見積もりは要再計算。`max_tokens` は余裕をもって設定推奨。
 
### ファイルベースのメモリ機能が強化
 
Claudeに「スクラッチパッド(メモ用ファイル)」を持たせて、長時間タスク中に自分でメモを取り、後でそれを参照する、という使い方が向上しています。
 
エージェント設計が捗ります。
 
### 価格
 
| | Opus 4.6 | Opus 4.7 |
|---|---|---|
| 入力 | $5 / 1Mトークン | **$5 / 1Mトークン**(据え置き) |

性能向上 + 価格据え置き = 実質値下げ、と捉えてOKです。

## 5. まとめ:結局どう使えばいい?
 
最後に「**じゃあ自分は何すればいいの?**」を立場別に整理します。
 
### 一般ユーザー(Claude.aiを使う人)
 
何もしなくてOK。**自動でOpus 4.7が使われる**ように切り替わります。応答がより簡潔・直接的になっているのに気づくかも。
 
### APIユーザー / 開発者
 
1. モデルIDを `claude-opus-4-7` に切り替えてみる
2. 既存プロンプトを流して**挙動の差分をチェック**(特に「暗黙の解釈」に頼っているプロンプト)
3. トークン数のコスト試算を**再計算**

1. `/effort` で xhigh を試す
2. リリース前のレビューに `/ultrareview` を使ってみる
3. Maxプランなら **Auto mode** を有効化
