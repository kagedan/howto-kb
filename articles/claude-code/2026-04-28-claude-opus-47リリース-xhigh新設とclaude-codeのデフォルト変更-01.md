## はじめに

Anthropicが2026年4月16日にClaude Opus 4.7をリリースした。コーディング性能の素直な底上げに加えて、エフォートレベルに「xhigh」が新設され、Claude Codeのデフォルトもこれに変わった。日々Claude Codeを触っている身としてはデフォルト変更のインパクトが大きいので、リリース内容と実務目線での影響をまとめておく。

## 本題

### コーディング性能はどれくらい伸びたか

Opus 4.7の数字をOpus 4.6と並べてみると、地味に大きい伸びが出ている。

* SWE-bench Pro: 64.3%（+10.9ポイント）
* SWE-bench Verified: 87.6%（+6.8ポイント）
* CursorBench: 70%（+12ポイント）
* 視覚推論 CharXiv: 82.1%（+13.0ポイント）
* コンピュータ操作 OSWorld: 78.0%（+5.3ポイント）

ベンチマークなので額面通りに受け取るのは禁物だが、SWE-bench Verifiedは現実のIssue修正に近い設定で、ここで6.8ポイント伸びるのは体感に効きやすい数字だ。視覚推論の+13.0は群を抜いて大きく、スクリーンショットや図表をエージェントに渡して読ませる用途では相応の差を感じるだろう。

### xhighエフォートレベルが新設

これまでのエフォートレベルはlow / medium / high / maxの4段階だったが、Opus 4.7では「high」と「max」の間に\*\*xhigh（extra high）\*\*が追加された。推論トークン上限の目安は約100kで、Opus 4.6のmax（上限200k）よりも少ないトークンで高いスコアを出すという位置付けだ。実測の例ではxhighが約71%のスコアを出している、という。

要するに「max未満のトークン消費でmax相当以上の質」を狙えるレベルが用意された、と理解しておけばよい。

### Claude Codeのデフォルトがxhighに

Claude Code側では、全プランでデフォルトのエフォートレベルがxhighに変更された。Opus 4.6時代に「max相当が欲しいときは明示的に指定」していた人にとっては、何もしなくても以前のmax以上の出力が返ってくる、という挙動になる。

プロプランの利用者は要注意で、何も意識しないとトークン消費が増える可能性がある。逆に、品質を上げたいときに毎回設定を切り替えていた人は手間が一段減る。

合わせて`/ultrareview`コマンドも追加されている。Pro/Maxプランで月3回無料で使えるレビュー特化機能で、慎重にレビューしたいPRに対して使える枠が用意された格好だ。

### ビジョン性能の引き上げ

地味だが重要なのが、画像入力の対応解像度の引き上げ。これまで約1.15メガピクセルだった上限が、最大3.75メガピクセルまで広がった。長辺は2,576ピクセルまで対応する。

スクリーンショットのリサイズで情報が潰れて読み取れない、という現象に何度か遭遇したことがあるが、3.75MPあれば一般的なノートPCのフルスクリーンショットなら大体そのまま渡せる。

### 料金は据え置き、ただしトークナイザー更新に注意

入力$5/百万、出力$25/百万トークンの価格はOpus 4.6から据え置きだ。ただし**トークナイザーが更新されており、同一テキストでも最大1.35倍程度トークン数が増えるケースがある**。特にJSONなど構造化データで影響が大きいとされている。

価格が同じでも、実質コストは入力データの性質次第で1割〜3割上振れする可能性がある、という点は見落としやすい。請求書を見て驚かないために、移行直後はコスト推移をモニタしておくのが無難だろう。

合わせて「タスク予算ベータ」も導入され、セッション全体のトークン支出を管理できるようになった。長時間走るエージェント運用では、これでブレーキが踏みやすくなる。

## 所感

Opus 4.7はベンチの数字も派手だが、自分にとって一番影響が大きいのは「Claude Codeのデフォルトがxhighになった」ことだ。

普段Claude Codeで雑にコードを書かせるとき、「max指定のために手間をかけるほどでもないがhighだと物足りない」というモヤッとしたゾーンが地味にあった。xhighがデフォルトに入ったことで、ここの選択肢を考える時間が消える。これは思っていた以上に効く。

一方で、トークナイザー更新の話は油断ならない。手元のプロジェクトはJSONを長く渡す系のスクリプトが多いので、たぶん体感1〜2割は消費が増える気がしている。Opus 4.6 maxを使い倒していた人ほど、同じ感覚で書いていると月末に「あれ?」となるはずだ。タスク予算ベータでセッション単位のキャップを掛けておくのは早めに試したい。

ベンチの+13ポイントとかを見るとつい「もう人間いらない」みたいな話に走りたくなるが、現場で効くのはむしろ視覚推論の伸びとビジョン解像度の引き上げのほうだと思っている。エラー画面のスクショをそのまま渡して読ませる、というワークフローが現実的に機能するレベルに来ている。

## まとめ

* Opus 4.7は2026年4月16日リリース。SWE-bench Pro 64.3%、Verified 87.6%とコーディング性能が底上げされた
* エフォートレベルに「xhigh」が新設され、Claude Codeの全プランでデフォルトに採用された
* ビジョンは最大3.75MPまで対応、料金は据え置きだがトークナイザー更新で実質コストが上振れする可能性あり
* 移行直後はトークン消費とタスク予算をモニタしながら使うのが安全
