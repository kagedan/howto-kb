> **筆者の財務の利害の開示**: 本記事の末尾で、 同主題を深く扱う有料の本 (19米ドル) と無料の試し読みを紹介します。 本文の中の3つの確認の手順だけで、 多くの利用者の判定は完結します。
2026年6月15日にAnthropicのClaude Codeの課金の経路の分割 (Pool 2の新規) が発火します。 本日2026年5月26日の時点で残り20日。 業界の独立な分析の集まりで、 既存の自動化の利用者にとって「軽い作業の負荷で12倍、 重いSonnetの自動の作業で150倍以上」 の価格の倍率の合図 ([codersera.comの深い解析](https://codersera.com/blog/anthropic-june-2026-billing-change-claude-code/)と[buildthisnow.comの手引き](https://www.buildthisnow.com/blog/guide/mechanics/claude-billing-change-june-2026))。
ただし、 業界の整理で意外と扱いが薄い項目が1つあります。 それは「受け取りのメール」 の存在です。
Anthropicは2026年6月15日の前 (報告では2026年6月8日頃から14日の週) に、 各利用者のClaudeのアカウントに「クレジットを一度だけ受け取る」 ためのメールを送信する予定。 メールを受け取った後、 Claudeのアカウントの設定でクレジットを一度だけ「受け取る」 操作が必要です。
> **受け取らない場合の警告**: 業界の解析の中で、 受け取らないと自動化の作業の枠を完全に失う、 の文言が確認されています。
つまり、 2026年6月8日から14日の週に届くメールを見逃すと、 2026年6月15日の後で`claude -p`やAgent SDKやGitHub Actionsの経路が完全に動作不能になる経路の合図。 本記事は、 20日後の前に5分で取得できる3つの確認の手順を整理します。
## 確認の手順1: 自分のAnthropicのアカウントのメールの受信の経路の点検

AnthropicからのメールはClaudeのアカウントの設定で登録されているメールの住所に届きます。 直近のAnthropicからの通知 (例: 更新の通知、 新規の機能の発表) を受信したか確認します。

```bash
# Gmailの場合: Anthropicからの直近のメールを検索の枠で確認
# 検索の文字列: from:anthropic.com

# 受信していない場合は、 迷惑メールの枠も確認
# 検索の文字列: from:anthropic.com label:spam
```

直近のAnthropicからのメールを1件も受信していない場合、 以下のいずれかの場合:
- アカウントの登録のメールの住所が古い (登録のメールを最新の住所に更新が必要)
- メールの受信の経路の問題 (受信の絞り込みの整理が必要)
- Anthropicの通知の購読が無効の状態 (Claudeのアカウントの設定で有効に変更が必要)
各場合の対処の経路は、 Anthropicの公式のサポートセンター (https://support.anthropic.com) を参照。
## 確認の手順2: カレンダーの想起の登録 (2026年6月8日から14日の窓)

メールの到達の窓は「2026年6月15日の前」 (公式) で、 業界の報告では「2026年6月8日頃から14日の週」 の見込み。 2026年5月26日から6月15日までの20日の間隔で、 カレンダーの想起の登録を行います。

```bash
# 想起の登録を3件行う
# 1. 2026-06-08 09:00 JST: 受け取りのメールの到達の窓の開始
# 2. 2026-06-11 09:00 JST: 受け取りのメールの点検 (中間の時点)
# 3. 2026-06-14 09:00 JST: 受け取りのメールの到達の窓の終了の点検
```

3件の想起の経路で、 到達の窓の中で1件でもメールを見逃した場合の予防の経路。

加えて、 自動の経路としてcronの想起の登録も整備可能:

```bash
# 2026年6月8日から14日の毎日の朝9時に、 自分のメールを末尾で確認する想起の発火
# crontab -e で以下を追加
0 9 8-14 6 * notify-send "Anthropicの受け取りのメールの確認"
```
## 確認の手順3: Pool 2の月次のクレジットの自分の状況の整理

クレジットを受け取った後の月次の金額の整理:

```
| 料金体系                  | 月次のクレジット   |
|---------------------------|--------------------|
| Pro                       | 20米ドル           |
| Max 5x                    | 100米ドル          |
| Max 20x                   | 200米ドル          |
| Team Standard             | 20米ドル / 席      |
| Team Premium              | 100米ドル / 席     |
| Enterprise (使用量に応じた)| 20米ドル           |
| Enterprise Premium        | 200米ドル / 席     |
```
繰越し不可 — 使い切るか失うかの様式。

## 受け取りのメールの到達の後の判定の入力

メールを受け取り、 クレジットを「一度だけ受け取る」 操作の後の判定:
```
| 月次の見込み              | 推奨の対応                     | 必要な作業    |
|---------------------------|--------------------------------|---------------|
| 0米ドル (利用なし)         | 対応の必要なし                  | 0時間         |
| 枠の中                     | 自動化を継続、 月の費用を許容    | 0から1時間    |
| 枠の超過                   | 超過の切り替えの手動の有効化     | 1から3時間    |
| 高額の超過                 | 自動化の経路の最適化、 代替の併用 | 3から40時間   |
```
超過の切り替えの経路の整理 (業界の解析より):
- 有効の場合: 月次のクレジットの超過の後、 APIの標準の料率で課金の継続 (稼働の保護)
- 無効の場合: 月次のクレジットの超過の後、 要求が拒否 (費用の固定、 次の課金の周期まで利用不可)
判定の基準は「作業の負荷の特性」 で選択。 本番の連携の経路は有効の推奨、 趣味の作業は無効の推奨。

## 開発の集まりの利用の場合の追加の整理

開発の集まりでClaude CodeをCIに組み込んでいる場合の追加の整理:
- クレジットはアカウントの間で共有不可
- CIの担当者の通知の必須性 (連携の経路の停止の予防)
- 各席の独立な受け取りの経路
開発の集まりの通知の素案:
```
件名: 【行動要請】2026年6月15日のClaude Codeの課金変更の対応 (期日: 2026年6月14日)
本文:
2026年6月15日にAnthropicがClaude Codeの課金の経路を分割します。 我々のCI/CDで claude -p または Agent SDK を利用している場合、 2026年6月15日の後で動作不能になる可能性があります。
集まりの一員は次の3点を実行してください:
1. 自分のAnthropicのアカウントに届く受け取りのメールを2026年6月8日から14日の週に確認してください
2. クレジットをClaudeのアカウントで受け取ってください (一度だけ)
3. 必要に応じて超過の切り替えの判定を行ってください
```

## 直接のAPIの鍵の利用者の整理 (誤った対応の予防)

直接Anthropicに API の鍵で課金されている利用者は、 2026年6月15日の変更の影響を受けません。
確認の経路は、 https://console.anthropic.com の課金の頁で、 個人の購読の状態か、 直接のAPIの鍵の課金の状態の確認。 後者の場合、 本記事の確認の手順1から3は不要です。

## 本記事の枠組みで判定が完結しない場合

月額100米ドル以上の自動化の利用者で、 次の項目の全件を1つの場所で整理した文書が必要な場合のみ、 有料の本が役に立ちます。
- Pool 2の対象の経路の棚卸しの体系 (cron、 CI、 SDK、 第三者の連携の整理の表)
- 2026年6月15日の後の月額の試算の枠組み (Pool 1とPool 2の振り分けの経路)
- 留まる、 切り替える、 混在の3経路の判定の枠組みの整理
- 実機の検証のhookの例 (cost-guard、 cache_creationの監視、 sub-agentの境界のhook)
これらが既に整理済の利用者、 月額100米ドル以下の利用者、 自動化の経路がない利用者は、 本の購入は不要。

## 20日の残り

2026年5月26日時点のAnthropicの公式の予定では、 2026年6月15日に課金の経路の分割が発火します。 公式の運用の変更で日付が変わる可能性は完全には排除できませんが、 2026年5月26日時点では確定の予定。
本記事の3つの確認の手順 (合計5分) と受け取りのメールの到達の窓のカレンダーの想起の登録で、 多くの利用者は受け取りのメールを見逃す経路を予防できます。 受け取らないと自動化の枠を完全に失う場合の予防の経路として、 20日後の前に5分の時間を割り当てる価値があります。
