実際の改善例

例えばこんなコード。

for (User user : users) {
    Order order = orderRepository.findByUserId(user.getId());
    process(order);
}

一見普通ですが、
ユーザー数分 DBアクセスが発生する典型的な N+1 問題です。

Claude に確認すると、

Map<Long, Order> orderMap =
    orderRepository.findAllByUserIds(userIds);

for (User user : users) {
    process(orderMap.get(user.getId()));
}

のような改善案を提示。

さらに、

なぜ遅いのか
どこが危険か
影響範囲
テスト観点

まで説明してくれる。

これがかなり強かった。
