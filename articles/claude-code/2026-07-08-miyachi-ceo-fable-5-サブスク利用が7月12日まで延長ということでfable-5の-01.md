Fable 5 サブスク利用が7月12日まで延長ということで、Fable 5の『頭脳』をSkill Packとして残す移植プロンプトを投下しておきます↓ https://t.co/BTO9H9alKp

Fable 5が今週以降、現在のプランでは使えなくなる可能性があります。

今後も私が Opus / Sonnet  などの別モデルで、あなたに近い実行力・判断力・自走力を再現できるように、現在のClaude Code環境をread-onlyで診断しつつ、あなたの仕事の進め方を「移植可能なスキル」として残してください。

目的は、モデルそのものや内部システムプロンプトをコピーすることではありません。
あなたが高い成果を出すために使っている、説明可能な範囲の
思考手順・判断基準・作業分解・確認方法・プロンプト・ワークフロー・品質基準を、
他モデルに読み込ませて再現しやすくすることです。

必ず Plan Mode で進めてください。
最初はファイル作成・編集・削除・設定変更・破壊的操作は一切しないでください。
まずは診断対象ディレクトリとスキャン範囲の確認から始めてください。

コマンド実行については、私が明示的に許可した場合のみ、
pwd / ls / find / tree / grep / wc / cat などの read-only コマンドを実行して構いません。

ただし、catでファイル本文を読む場合は、CLAUDE.md / SKILL.md / settings.json / .mcp.json など、診断に必要な設定ファイルに限定してください。
個人情報・機密情報・認証情報・不要なファイル本文は開かないでください。

私の承認を得るまで、環境への変更は一切禁止です。

# 役割

あなたは以下の2つの役割を兼ねます。

## 1. Claude Codeセットアップ監査の専門家

- 私のPC環境とClaude Code構成をread-onlyで診断する
- 現状の問題点、重複、肥大化、未活用、設定ミス、改善余地を見つける
- 最適な構成案と段階的な改善プランを提示する

## 2. 実行力移植設計者

- あなたの仕事の進め方を、Opus / Sonnet に移植できる形に整理する
- CLAUDE.md、skills、agents、rules、hooks、output-styles、MCP、Git運用、日常業務フローへ落とし込む
- 私の用途に合わせて、実務で使えるSkill Packとしてまとめる

# 診断対象

以下を漏れなく診断してください。

## 1. PC全体のディレクトリ階層

対象は、ホーム直下から主要プロジェクト群、ドキュメント、ダウンロード、開発フォルダなどです。

見たい観点：

- 散らかり具合
- 命名規則の不統一
- 深すぎる / 浅すぎる階層
- 重複フォルダ
- 放置フォルダ
- プロジェクト置き場の一貫性
- Claude Codeから参照しやすい構造か
- AIチーム運用に向いた構造か

注意：
ファイルの中身は不用意に開かず、まずは構造・ツリー・命名のみを診断してください。

## 2. CLAUDE.md

以下を確認してください。

- ~/.claude/CLAUDE.md
- 各プロジェクト直下のCLAUDE.md

観点：

- 内容の粒度
- トークン量
- 重複
- 常時読むべきでない手続きが混ざっていないか
- ユーザー全体ルールとプロジェクト固有ルールが混ざっていないか
- 古い指示が残っていないか
- 実行力を移植する場所として適切か

## 3. .claude/rules/

観点：

- パススコープの妥当性
- ルールの重複
- 常時ルール化すべきものと、skill化すべきものの分離
- プロジェクト別に分けるべきもの
- 全体ルールに昇格すべきもの

## 4. skills

対象：

- .claude/skills/*/SKILL.md

観点：

- descriptionの精度
- 発火トリガーの明確さ
- 本文の粒度
- 重複
- 未発火リスク
- 旧 .claude/commands/ からの移行状況
- 手順としてskill化すべきもの
- 実行力を再現するために新規追加すべきskill

## 5. subagents

対象：

- .claude/agents/

観点：

- 役割分担
- toolsのread-only制限
- model指定
- memory活用
- どの仕事をsubagentに分離すべきか
- 重い処理をsubagentへ委譲すべき場面
- 自走力を再現するために必要なagent構成
