## きっかけ

`anthropics/claude-code-action` のバージョンアップに**TOCTOU対策**（v1.0.45）という見慣れない単語があった。

調べてみたら意外と身近で怖い脆弱性だったので、整理してみる。

## TOCTOU（Time-of-Check to Time-of-Use）とは

**チェックした時点と、実際に使う時点の間にデータが変わってしまう脆弱性。**

名前の通り、Check と Use のタイミングのズレを突く攻撃手法で、レースコンディション（競合状態）の一種。

### 日常の例え

1. 警備員が部屋を覗き、誰もいないことを確認する（**Check**）
2. 警備員がドアを閉めに向かう（**Gap**）
3. その隙に侵入者が部屋に入る
4. 警備員は「確認済み」としてドアを施錠する（**Use**）

ポイントは、チェック自体は正しく行われていること、そして**第三者が意図的に隙を突いて状態を変える**こと。単なる「情報が古くなる」話ではなく、攻撃者の介在を前提にしたセキュリティ上の問題である点が重要。

## claude-code-action での具体例

claude-code-action は、Issue や PR で `@claude` とメンションされると起動し、その本文を読んで作業を行う。誰でもトリガできては危ないので、「呼べるのは書き込み権限を持つメンテナだけ」というガードはちゃんとある。

ところが、**ガードしているのは「誰が呼んだか」だけで、「呼ばれた後に読む本文の中身」はノーガード**だった。ここに TOCTOU の隙が生まれていた。

### 図式して見る

公開リポジトリで、外部コントリビューター（攻撃者）が以下を仕掛けるケース。

### 何が起きていたか

* **Check（権限ガード）**: メンテナが `@claude` を呼んだ。ここは正しく検証されている
* **Use（本文の使用）**: 数秒後に取得した本文は、攻撃者が書き換えた後のもの
* 「**誰が**言ったか」は守れていても、「**何を**言ったか」が後から差し替え可能だった

ログには「メンテナがレビューを依頼した、Claude が応答した」としか残らない。事後の調査でも気づきにくい。これが TOCTOU の厄介なところ。

### 修正の中身

問題は、action が起動後に**もう一度**本文を取りに行っていたこと。 webhook ペイロードはイベント発火後に変更されない。

v1.0.45 の修正は単純で、**body だけは GraphQL の結果を使わず、webhook ペイロードのものをそのまま採用する**ようにした。

## なぜ検出が難しいか

* **ログに痕跡が残りにくい** — チェック時点では実際に安全だった
* **再現が困難** — タイミング依存なので、同じ手順を踏んでも毎回発生するわけではない
* **既存のテストで見つからない** — 単体テストやE2Eテストは基本的に逐次実行を前提としている

## 一般的な対策

| 対策 | 説明 | 具体例 |
| --- | --- | --- |
| **原子性の確保** | チェックと使用を分離せず、一つの操作として実行する | ファイル: `open()` で取得したfdに対して操作する／DB: `SELECT ... FOR UPDATE` で取得と同時にロックする |
| **ロック** | チェックから使用まで対象リソースをロックする | OSのファイルロック、DBの行ロック、分散ロック（Redis等） |
| **スナップショット固定** | Check 時点で取得した参照（fd、バージョン番号、ペイロードなど）を Use まで使い回す／Use 時に「Check したときと同じ状態か」を原子的に確認する | コミットSHAを固定してチェックアウトする、webhook ペイロードを再取得せず使う、楽観的ロック（バージョン番号での再確認） |

先述の claude-code-action v1.0.45 の修正（`fix: use original body from webhook payload for TOCTOU hardening`）は、まさにこの「スナップショット固定」にあたる。詳細は [リリースノート](https://github.com/anthropics/claude-code-action/releases/tag/v1.0.45) を参照のこと。

## まとめ

* **TOCTOU** は「確認」と「使用」のタイムギャップを突く脆弱性
* 見た目上は正しく動いているように見えるので検出が難しい
* 対策は**原子性の確保**・**ロック**・**スナップショット固定**
* 依存パッケージのアップデートを放置すると、こうしたセキュリティ修正を取りこぼす

バージョンアップのchangelogを読むだけでも、セキュリティの学びがある。定期的なアップデート、大事。
