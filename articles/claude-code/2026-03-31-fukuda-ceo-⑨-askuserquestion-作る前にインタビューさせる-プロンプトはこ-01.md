Claude Code、半年ほぼデフォルトのまま使ってたんよ。気づいたのは子供が寝た後の深夜で、しかもXで偶然だった。誰も教えてくれへんやつ、損した気分そのままに10個まとめて置いとく。 保存して、1個ずつ試してみて。 https://t.co/Jtj2N6L6Pt

① /rewind
やらかした瞬間の命綱

ESCを2回押すだけ。
変更前の状態に即座に戻れる。

Gitを触らずにチェックポイントまで一発。

② CLAUDE.md
毎回の説明をゼロにする

プロジェクトのルートに置くだけで、
Claudeが毎回そこを読む。

コーディングスタイル、
禁止事項、
使うコマンド。 

全部ここに書けば、
毎セッション説明しなくていい。

まず /init を打つと自動生成してくれる。

③ Hooks
お願いじゃなく強制にする

ファイルを編集したら自動でlintを走らせる。
特定フォルダへの書き込みをブロックする。

CLAUDE.mdは「お願い」
Hooksは「強制」

この違いを知ってから、
ミスの数が体感で半分になった。

Claudeに「Hookを書いて」って
頼むだけで作ってくれる。 

/hooks で確認、
.claude/settings.json で直接編集もできる。

④ Git
ワークツリー＋並列セッション

claude --worktree feature-auth で一発。 

ブランチごとに別ディレクトリで
Claudeを同時起動できる。

フロントのバグ修正と、
バックエンドの新機能を
同時進行。 

Gitの操作は
全部Claudeがやってくれる。

3〜5個のワークツリーを
同時に走らせてる人もいる。 

上限はマシンのスペック次第。

⑤ Ctrl+S　
プロンプトの下書き保存

長いプロンプトを書いてる途中で、
先に確認したいことができた。 

Ctrl+Sで下書きを保存、
別の質問を先に投げる。 

終わったら自動で下書きが戻ってくる。

⑥ Ctrl+B　
バックグラウンド実行

テストやビルドが
走り出したらCtrl+B。 

待ち時間がそのまま
別の作業時間に変わる。

⑦ /branch　
会話を分岐させる

今の会話をコピーして、
別方向で試せる。 

「Aの実装」
「Bの実装」

を同じ文脈から
同時に試したい時。 

本流を汚さずに実験できる。

⑧ /color と /rename　
セッションを見分ける

/rename auth-refactor で
セッションに名前をつける。

 /color red で
プロンプトバーの色を変える。

並列で動かしてる時、
どれがどれかわからなくなる。 

それを一瞬で解決する機能やのに、
ほぼ誰も使ってない。

⑨ AskUserQuestion　
作る前にインタビューさせる

プロンプトはこれだけ。

「AskUserQuestionツールを使ってボクにインタビューして。 技術的な実装・UI/UX・エッジケース・トレードオフを掘り下げて。 終わったらSPEC.mdに仕様書を書いて」

⑩ /clear　
これだけで精度が上がる

新しいタスクに入る前に、
必ず /clear。 

古い履歴がトークンを食って、
パフォーマンスが落ちてた。

シンプルすぎるけど、
ボクが一番サボってた習慣やった。

全10個まとめると、

 /rewind か /clear から。 

ハードル低いのが
いい人は /clear 一択。

あったら教えてほしい。
