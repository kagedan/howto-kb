ショート動画の運用で一番よく聞くのが「最初の1〜2週間は順調だったのに、月の後半で失速した」という話です。最初はストックしていたアイデアで回せるのですが、それが尽きた瞬間、「今日は何を撮ろう…」から毎回始まることになります。

毎日0から企画を考えるのは消耗します。しかも「とりあえず撮れそうなもの」を投稿しても、ジャンルが散らかってフォロワーに何の人か伝わらない。気づいたら投稿頻度が落ちて、フィードから消えていきます。

**Claude Codeに「ジャンル・ターゲット・動画の型」を渡しておくと、来月分のショート動画30本の企画カレンダーを1時間で出せます**。タイトル・フック・動画の流れ・投稿日の割り振りまで一気に出るので、月初に1回作業するだけで、あとは「今日はこれを撮る」だけの状態になります。

## このワークフローで何ができるか

**ビフォー: 投稿するたびにネタを考える→月後半に失速→頻度が落ちてアルゴリズムに埋もれる**  
**アフター: 月初に30本の企画カレンダーを確定→毎日「今日撮ること」だけ考える状態へ**

しかも、企画がランダムにならず「このアカウントが扱うテーマ」に沿ったネタだけが並ぶので、ジャンルの一貫性が保てます。「何を発信しているのか分かるアカウント」が積み上がる土台になります。

## 手順

### ステップ1: 「何をするアカウント」かを一度だけ決める

ネタ切れの根本原因は「何でも投稿する」ことにあります。先にジャンルを絞ります。

```
【アカウント設計の基本（架空例）】
アカウント名: ○○（架空）
ジャンル: 飲食店の個人事業主向け・SNSマーケティングTips
ターゲット: 独立3年以内・SNSを試しているが成果が出ていない飲食オーナー
投稿目的: フォロワー獲得 → 月1本の有料相談につなげる
投稿頻度の目標: 週5本（平日毎日）＝月20本
扱わないネタ: 政治・炎上リスクのあるトレンド・他業種の事例
```

「何でも」ではなく「誰の・どんな悩みに答えるか」を1行で言えるまで絞ります。ここが曖昧なままカレンダーを作ると、ネタの散らかりが解決しません。

### ステップ2: CLAUDE.mdに「ショート動画の型」を5〜6種類固定する

毎月テンプレを変えると一貫性が崩れます。型を最初に決めてしまいます。

```
あなたは私のショート動画企画担当です。
渡したジャンル・ターゲット・月の情報をもとに、来月の投稿カレンダーを作成してください。

【アカウント設計】（架空）
- ジャンル: 飲食店オーナー向けSNSマーケTips
- ターゲット: 独立3年以内の飲食オーナー
- 目的: フォロワー獲得 → 有料相談誘導

【動画の型（この6種類を繰り返す）】
① ○○をやってみた（検証・比較）
② ○○の間違い3選（あるある・訂正型）
③ ○○を1分で解説（ノウハウ要約）
④ Before/Afterの変化を見せる（結果重視）
⑤ Q&A（よく聞かれる質問に答える）
⑥ ○○の裏側を公開（舞台裏・ルーティン）

【出力フォーマット（1本あたり）】
- 企画タイトル（フック入り）
- 動画の流れ（①フック→②本題→③オチ/CTA の3段）
- 型の種類（①〜⑥）
- 推奨投稿日（来月1日〜末日）
```

6種類の型を繰り返すことで、30本が単調にならず、かつジャンルがぶれません。

### ステップ3: 「月のネタのヒント」を渡して30本を一気に生成する

月初にその月のトレンドや季節要因を渡し、カレンダーを生成します。

```
来月（7月）のショート動画企画カレンダー30本を作成してください。
CLAUDE.mdの6種類の型を満遍なく使い、投稿日を7月1日〜31日で割り振ってください。

【7月のネタのヒント】
- 季節: 夏本番。飲食店は客単価が下がりやすい時期（ドリンク系が伸びる）
- 業界イベント: 土用の丑（7/19）、海の日連休（7/21）
- このアカウントで反応が良かったネタ: 「SNS広告を使わずに満席にした方法」系
- 今月やってみたこと: Instagramのストーリーを週3本追加した効果を検証中

以上を踏まえ、7月のカレンダーを出してください。
```

「反応が良かったネタ」と「今月試したこと」を毎月追記するのがコツです。月を追うごとに企画の精度が上がります。

### ステップ4: カレンダーを確認し「撮れないネタ」を差し替える

AIが出した30本はすべて採用するのではなく、実行可能性でふるいにかけます。

```
確認ポイント:
1. 「Before/After」「舞台裏」など、素材の撮影が難しいネタはないか
2. 競合・他者への比較・批判がないか（炎上リスクを確認）
3. 同じ型が連続していないか（同じ型が3本以上連続はNG）
4. 法的にグレーな主張・誇張した成果表現がないか（「必ず集客できる」等は削除）

差し替え指示の例:
「No.15の企画はBefore/Afterの素材が撮れないので、同じ日程で⑤Q&A型に変えてください」
```

ここが人のやる仕事です。残り20〜25本が使える企画になれば、月間の投稿素材は確保できます。

### ステップ5: スプレッドシートに落として運用ループにする

確定したカレンダーをスプレッドシートに転記し、毎月末に振り返ります。

```
運用ループ:
1. 月初: 30本の企画カレンダーを生成→人が差し替え→スプレッドシートに入力
2. 毎週: その週の撮影・編集・投稿を実行
3. 月末: 反応が良かった型・タイトル・テーマをCLAUDE.mdの「効いたネタ」に追記
4. 翌月: 「効いたネタ」を踏まえて再生成（精度が毎月上がる）
```

月初の1時間で全体の企画を固め、あとは「今日の分を撮る」だけにする——これが失速を防ぐ一番の仕組みです。

## 応用と注意点

**応用1: 複数プラットフォームへの横展開**  
TikTok向けに作ったカレンダーを「同じ企画をリール・YouTubeショート向けにフック文だけ調整して」と渡すと、3媒体の展開案が一気に出ます。撮影は1回、編集・投稿を媒体ごとに最適化する運用につながります。

**応用2: 季節商材・キャンペーンとの連動**  
新商品ローンチや期間限定メニューがある月は、その情報を「月のネタのヒント」に追加するだけで、自然に商品訴求の企画が混ざり込みます。プロモーションを「売り込み感なく組み込む」のに使えます。

**注意点1: 動画の型は変えすぎない**  
CLAUDE.mdの型を毎月変えたくなりますが、変えるほどカレンダーの一貫性が崩れます。半年に1回、反応データを見ながら見直す程度に抑えてください。

**注意点2: トレンド音源・楽曲の著作権**  
ショート動画のBGMや流行り音源を使う際は、各プラットフォームの著作権ポリシーを必ず確認してください。商用利用に対応していない楽曲は収益化・広告出稿で問題が起きることがあります。

**注意点3: 「必ず伸びる」「フォロワーが爆増」等の断定表現を混ぜない**  
AI生成の企画タイトルに成果を誇張する表現が入っていることがあります。投稿前に「これは約束しているように見えないか」を必ず確認してください。ショート動画はプラットフォームのアルゴリズム次第であり、特定の成果を保証することはできません。

## まとめ

* ショート動画の失速の原因は「毎回0から企画すること」。Claude Codeにジャンル・型・月の情報を渡せば30本が1時間で出る
* CLAUDE.mdに6種類の動画の型を固定し、型を繰り返すことでジャンルの一貫性を保つ
* 月末に「効いたネタ」をCLAUDE.mdに追記して再生成——月を重ねるほど精度が上がるループを作る
* 人の仕事は「撮れないネタの差し替え」と「著作権・誇張表現の確認」だけ

月の後半にネタが尽きて失速する、そのサイクルを断ち切れます。月初の1時間の投資で、あとは「今日の分を撮るだけ」の状態を作ってみてください。
