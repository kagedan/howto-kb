AnthropicがClaude in Slackの上位互換Claude Tagリリース🔥1日公式Doc読んでまとめた、これはAIを組織利用する上で重要なリリースなのでいつもよりメモ多め📝

ただ、モデルはOpus4.8固定（Sonnet4.6には変更できない）のでお高めなのと、
ClaudeTagでSlack1回のメンション指示では、セッションは「1セッション＝1リポ＝1ブランチ＝1PR」を前提なので、複数リポを修正してまとめて複数PRはできないので、今後のアップデートに期待🎵

【Anthropic新発表「Claude Tag」を機能ごとに全部メモシェア🎵SlackにClaudeがチームの一員として常駐する時代きたぞと👀】

🤖 Claude Tag：チームのSlackで働くClaude、機能フル解説

■ 1. 基本コンセプト 🚀
 ▸ 1.1 一人で使うチャットから「チームにいるAI」へ
   • チャンネルに@Claudeを招待しメンションするだけで始動
   • タスクを渡して自分は別作業へ＝同僚に委任する感覚
   • Anthropic製品チームのコードの65%が社内版Claude Tag製
 ▸ 1.2 Claude Code / Cowork との使い分け
   • Claude Tag：Slackチャンネル＝チームで見て操舵する共有作業
   • Cowork：https://t.co/zWdv4c4E6jチャット＝自分の個人リサーチや下書き
   • Claude Code：ターミナル/IDE＝手元リポジトリでのコーディング

■ 2. マルチプレイヤー性 👥
 ▸ 2.1 一つのチャンネルに一つのClaude、全員で共有
   • 誰が何を依頼し、どこまで進んだか全員に見える
   • 起動者でなくても会話を続きから引き継げる
 ▸ 2.2 スレッド内なら再メンション不要で操舵できる
   • 走行中のセッションに別の人が返信して指示を追加
   • 「やっぱりステージ環境を対象に」と途中で方針転換もOK
   • 編集・削除は遡及しない＝やり直しは新しい返信で伝える

■ 3. メモリ（記憶）機能 🧠
 ▸ 3.1 チャンネルを追いながら文脈を蓄積
   • 毎回ゼロから説明し直す必要がなくなる
 ▸ 3.2 公開チャンネルの記憶はワークスペース全体で共有
   • #launch-weekでの決定が#gtm-westでも参照可能
   • プライベートチャンネルは自分専用の記憶として隔離
 ▸ 3.3 記憶は確認・修正できる
   • 「このチャンネルについて何を覚えてる?」で中身を確認
   • 古い項目を名指しで「忘れて」と修正可能

■ 4. 自律性・先回り（Ambient） 🔔
 ▸ 4.1 有効化すると自分から動く
   • 必要そうな情報を横断的にフラグ立て
   • 止まったまま未解決のスレッドやタスクを追跡
 ▸ 4.2 非同期で長時間タスクを推進
   • 渡せば数時間〜数日かけて自律的にプロジェクト遂行
   • 自分でタスクをスケジュールして定期実行

■ 5. スケジュール実行・ルーティン ⏰
 ▸ 5.1 定期ジョブを言葉で設定
   • 「毎週金曜15時に今週のリクエストを要約して投稿」
   • リポジトリのイベントやチャンネル監視でも発火
 ▸ 5.2 ルーティンは誰でも監査・停止できる
   • 「このチャンネルのルーティン一覧は?」で重複を防ぐ
   • 「日次ダイジェストを止めて」と名指しで無効化

■ 6. 仕事の進み方（セッションの仕組み） 📋
 ▸ 6.1 1スレッド＝1サンドボックスで実作業
   • Claude Code on the webと同じ実行環境でコード実行・PR作成
   • 進捗はチェックリストをその場で書き換えて表示
 ▸ 6.2 静かなスレッド＝詰まりでなく作業中
   • Slackは編集を通知しないので止まって見えるだけ
   • 行き詰まった時だけ沈黙でなく返信で知らせる
 ▸ 6.3 成果物は形を選んで投稿
   • 短い答えは返信、データは画像/チャート、コードはドラフトPR
   • スレッドとPR/ブランチは残るがサンドボックス内ファイルは消える

■ 7. アクセス制御・権限 🔐
 ▸ 7.1 アクセスは「人」でなく「チャンネル」基準
   • 営業用と開発用で記憶もツールも完全分離
   • 「このチャンネルから何にアクセスできる?」で確認
 ▸ 7.2 サービスアカウントで動作・行動を追跡可能
   • Slackは Claudeアプリ、コードは Claude GitHub App経由
   • チャンネルでの
