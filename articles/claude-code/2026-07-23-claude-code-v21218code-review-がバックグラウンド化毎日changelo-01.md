:::note info
Claude Codeの[公式Changelog](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md)をリリースから最速で翻訳・解説しています。
:::

v2.1.218 で `/code-review` がバックグラウンドのサブエージェントとして走るようになりました。レビューの過程が会話コンテキストを埋めないので、レビュー後もそのまま作業を続けられる。`context: fork` のスキルもデフォルトでバックグラウンド実行になり、「本体の会話を汚さない」方向の変更がそろった版です。

## 今回の注目ポイント

1. **`/code-review` がバックグラウンドサブエージェント化** - レビュー作業が会話コンテキストを消費しなくなった (v2.1.218)
2. **`context: fork` スキルもデフォルトでバックグラウンド実行** - スキル単位の `background: false` で従来挙動に戻せる (v2.1.218)
3. **auto モードの許可ダイアログを削減** - 危険な `rm` などの静的チェックを分類器の判定に置き換え (v2.1.218)
4. **左矢印キーで会話が消える事故に確認が入った** - 編集直後の押下は確認、Esc で元の会話に復帰できる (v2.1.218)
5. **MCP 接続失敗時に HTTP ステータスとエラー本文を表示** - `claude mcp list` と `/mcp` で原因を追える (v2.1.218)
6. **Windows パスの `\u` 文字化けを修正** - `C:\Users\unicorn` のようなパスが CJK 文字に化けていた (v2.1.218)

## /code-review が会話の外で走るようになった

:::note info
対象読者: `/code-review` を日常的に回している人、長丁場のセッションでコンテキスト残量が気になる人
:::

**レビューがバックグラウンドのサブエージェントに移り、本体の会話にコンテキストを残さなくなりました (v2.1.218)。** これまでは `/code-review` を打つと、レビューで読んだ差分や検証の過程が会話に積まれ、後続の実装に使える残量が目減りしていた。今回からはレビューが裏で走り、会話側は結果を受け取るだけ。`/code-review` の後ろに積んだスラッシュコマンドをレビュー対象として巻き込むこともなくなっています。

スキル側も足並みがそろいました。`context: fork` を指定したスキルはデフォルトでバックグラウンド実行に変わっています。前面で走らせたいスキルだけ、frontmatter で個別にオプトアウトする形。

```yaml
---
context: fork
background: false
---
```

非対話セッションで `/code-review ultra` が黙ってローカルレビューに落ちていた問題はクラウドレビュー起動に修正。`/ultrareview` に「review my auth changes」のような自由文を渡すと失敗していた挙動も、現在のブランチのレビューに注記として反映される形に変わりました。

## 左矢印キーで会話がまるごと消えていた

入力を編集した直後に左矢印キーを押すと、会話が破棄されて元に戻せない問題がありました。アンドゥは無し。長いセッションほど失うものが大きい操作事故です。

v2.1.218 からは、編集直後の押下で確認を挟むようになりました。エージェントビューで Esc を押した場合も、バックグラウンドに回した元の会話へ戻れます。

## auto モードの許可ダイアログが減る

:::note warn
auto モードで、危険な `rm`・バックグラウンド実行の `&`・不審な Windows パスの各チェックが許可ダイアログを開かなくなりました (v2.1.218)。代わりに auto モードの分類器が可否を判定します。plan モード + auto の組み合わせも同様で、静的解析で読み取り専用と証明できない Bash コマンドは分類器の判定に変わりました。ダイアログで都度止める前提の運用なら、更新後の挙動を一度確認してください。
:::

## まとめ

v2.1.218 の軸はバックグラウンド化。`/code-review` と `context: fork` スキルが、レビューや作業のコンテキスト消費を本体の会話から切り離しました。auto モードのダイアログ削減と合わせて、長いセッションを止めずに回す狙いが一貫しています。
