Claude codeで非稼ぐ低単価ローンチしたら売上500万出たので、やったこと公開します↓

①情報を全部揃える

ジャンルのドメイン知識、競合のLP、ターゲットの悩み、参考になる配信文。これを全部1つのフォルダに集める。

ローンチで一番外しちゃいけないのが「情報の解像度」。ここが薄いとAIに何書かせても薄い。

僕はYouTubeの文字起こし、競合LPのPDF、過去のオプチャ配信ログまで全部Claude Codeに食わせた。情報量で勝負が決まる。（続く）

②必要なことを洗い出す

情報が揃ったら、Claude Codeに「ローンチに必要なものを全部リストアップして」と投げる。

出てくるのはLP構成、教育要素、配信スケジュール、セールスメール、サポート文、特典設計。全部出してもらってチェックリスト化する。

これやらないと作業漏れが出てローンチ中にバタバタする。先にタスクを全部見える化しておくのが大事。（続く）

③本文系はClaude codeにやらせる

LP本文、配信文、セールスメール、投稿文。全部Claude Codeに書かせる。

ポイントはナレッジの設計。教育要素フレームワーク、価格正当化の型、参考LP、過去のバズ投稿を全部ナレッジに入れる。

ナレッジが揃ってれば、出てくる文章のクオリティは外注以上。1日100本でも書ける。（続く）

④Codexで全体チェック

Claude Codeで書いた本文は「Codex Plugin for Claude Code」を使ってCodexから全体チェック。

Claude Codeは自分の出力に甘い。論理破綻、訴求の弱さ、誤字脱字を別AIに見つけてもらう。

このダブルチェックで品質が一段上がる。これやってる人ほぼいない。（続く）

⑤LP作成

LPの構成はClaude Codeに作らせて、デザインはノーコードツールに流し込むだけ。

構成テンプレートさえあれば、テキスト差し替えで何ジャンルでも作れる。

僕は1ローンチあたりLP作成は半日で終わる。デザインの細かいところだけ外注に投げる。（続く）

⑥オプチャローンチ

オープンチャットでのローンチ配信文も全部Claude Codeで量産。

ティザー、教育、価格発表、限定オファー、クロージング。フェーズごとの配信文をスケジュールに合わせて流す。

配信文は毎回切り口を変える。同じ訴求の繰り返しは飽きを生む。ここはナレッジで管理。（続く）

⑦サポート + まとめ

購入者対応の定型文はClaude Code、個別対応は外注に投げる。自分は判断だけ。
