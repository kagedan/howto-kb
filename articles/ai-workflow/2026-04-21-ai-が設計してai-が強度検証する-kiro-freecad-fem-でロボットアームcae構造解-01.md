## はじめに

本記事は **Kiro × 設計開発シリーズ**の 3 本目です。

本シリーズでは、AWS の AI コーディングアシスタント「**[Kiro](https://kiro.dev/)**」を使って、自然言語の指示だけで製造業向けの設計・解析タスクを進めています。

FreeCADで製図して、編集しました。次はKiro への自然言語指示だけで **FreeCAD FEM による構造解析（CAE）** を実行し、ロボットアーム全体に荷重をかけたときの応力分布と変位を定量的に評価します。

> 注記：本記事では、Kiro CLI の基盤モデルとして Claude Opus 4.6 を使用しています。モデルのバージョンによって生成されるコードの品質や挙動が異なる場合があります。

![](https://static.zenn.studio/user-upload/c2e73793f80e-20260420.png)

### 今回の流れ

```
Step 1: [CAD 編集編]で生成した STEP モデルを読み込み
Step 2: Gmsh でメッシュ生成（有限要素に分割）
Step 3: 材料・境界条件・荷重を設定
Step 4: CalculiX ソルバーで構造解析を実行
Step 5: 応力分布・変位分布を可視化
```

なお、FreeCAD FEM をヘッドレスで動かすには試行錯誤が必要でした。その経緯は本記事末尾の[補足セクション](#%E8%A3%9C%E8%B6%B3-%E6%A7%8B%E9%80%A0%E8%A7%A3%E6%9E%90%E3%82%92%E5%8B%95%E3%81%8B%E3%81%99%E3%81%BE%E3%81%A7%E3%81%AE%E8%A9%A6%E8%A1%8C%E9%8C%AF%E8%AA%A4)にまとめたので、興味のある方はそちらを参照してください。本編では結果の話から始めます。

### 構造解析（FEA）とは

構造解析は「この形状にこの力をかけたら、どこにどれだけ応力がかかるか、どれだけ変形するか」を計算するシミュレーションです。形状を小さな要素（有限要素）に分割し、各要素の弾性的な挙動を行列で表現してから全体を 1 つの連立方程式（剛性方程式）として解き、各節点の変位を求めます。応力・ひずみは変位から計算されます。

ロボットアームの設計では標準的な工程で、以下を検証します:

* 先端に荷物を持たせたとき、アームが壊れないか（応力が材料の降伏応力を超えないか）
* 荷重でアームがどれだけたわむか（位置精度に直結）
* どの部位に応力が集中するか（補強が必要な箇所の特定）

## 使用ツール

本記事はすべて**ヘッドレス**（GUI なし、コマンドラインとスクリプトだけ）で実行しています。Kiro は GUI アプリを人間のように操作することはできませんが、CLI コマンドと Python スクリプトであれば自律的に組み立てて走らせることができるため、一貫してヘッドレス前提で進めます。

| ツール | バージョン | 用途 |
| --- | --- | --- |
| FreeCAD | 1.1.0 | STEP 読み込み、brep エクスポート |
| Gmsh | 4.15.0 | メッシュ生成（FreeCAD 内蔵） |
| CalculiX | 2.23 | FEA ソルバー（FreeCAD 内蔵） |
| Python | 3.14.1 | スクリプト実行環境 |
| matplotlib | 3.10.8 | 結果の可視化 |

* **[Gmsh](https://gmsh.info/)**: オープンソースの 3D 有限要素メッシュ生成ツール。STEP や brep などの CAD 形状を入力として受け取り、四面体要素・六面体要素などのメッシュを生成します
* **[CalculiX](https://calculix.de/)**: オープンソースの 3D 構造 FEA **ソルバー**（連立方程式を解いて応力や変位を計算する計算エンジン）。線形・非線形、静的・動的・熱解析に対応。商用ソフト Abaqus と同じ `.inp` キーワード構文を採用しているため、Abaqus ユーザーにも馴染みやすいのが特徴です。計算本体は `ccx` というコマンドで実行します（`CalculiX Crunchier` の略）

## 解析対象

[CAD 編集編](https://zenn.dev/aws_japan/articles/45fb5d3130035d)で生成した **6軸多関節ロボットアーム（STEP）** 全体を解析対象とします。10パーツ（base, swivel, shoulder\_joint, upper\_arm, elbow\_joint, forearm, wrist\_j4/j5/j6, flange）を Boolean Fuse で結合し、単一のソリッドとして解析します。

* **姿勢**: J2=-30°, J3=60°, J5=-45°（作業姿勢）
  + J1=ベース旋回、J2=ショルダー前後傾動、J3=エルボー上下傾動、J4-J6=手首 3 軸。つまり肘を曲げて腕を斜め上前方に伸ばした、実作業でよくあるポーズです
* **材料**: アルミ合金 6061-T6
  + 産業用ロボットのフレーム材料として最も一般的な選択肢の 1 つで、世界のロボットフレーム用途の約 7 割を占めるとされます。強度/重量比が良く、機械加工性が高く、安価なのが理由です
* **境界条件**: ベース底面を固定、フランジ（先端）に 100N（≒10kg）の下向き荷重
  + 実際のロボットアームは床や架台にボルト固定されるため、底面完全固定はロボット FEA の標準的な境界条件です。先端集中荷重は「根元固定の片持ちはり」として捉える最も単純化したモデルで、10kg は中型産業用ロボット（リーチ 1m 級）のペイロード帯域に収まる典型値です
  + なお、実運用では重力トルク、運動慣性、オフセット荷重（フランジから離れた重心でモーメントアームが伸びる）などでより厳しくなります。本記事は静的・最も単純なケースとしての検証です

### 材料特性（アルミ合金 6061-T6）

| 特性 | 値 |
| --- | --- |
| ヤング率 | 69 GPa |
| ポアソン比 | 0.33 |
| 降伏応力 | 276 MPa |

安全率 = 降伏応力 ÷ 最大応力。延性材料の一般的な機械部品では、安全率 1.5〜2.0 以上を目安とすることが多いとされます。

### 本記事の位置づけ（モデルについての注記）

今回解析する STEP モデルは、[CAD 編集編](https://zenn.dev/aws_japan/articles/45fb5d3130035d) で教材として CadQuery で作った **中身まで材料で詰まった「無垢（ソリッド）モデル」** です。実物の産業用ロボットアームは、上腕・前腕リンクを肉厚 2〜5mm 程度のアルミ箱・鋳造中空パイプにして、軽量化とコスト削減を両立させています。無垢モデルは実物より断面が桁違いに太く、応力・変位の絶対値は実物より 1 桁以上小さく出ます。

本記事の主目的は「AI（Kiro）への自然言語指示だけで FEM ワークフロー（メッシュ → 材料 → 境界条件 → ソルバー → 結果可視化）を完走できる」ことの実証であり、実物ロボットの強度証明ではありません。ただし、手順・ツールチェーンはそのまま実物に適用でき、薄肉中空の STEP を入力として差し替えれば、同じスクリプトで実物に近い数値が得られます。

## 解析結果

メッシュ生成から CalculiX 実行、結果の可視化まで、すべて Kiro がヘッドレスで実行しました。FreeCAD の GUI は開かず、Python から `ccx` を直接叩き、結果の `.frd` ファイルを読み取ってまとめています。FEM のヘッドレス実行は一筋縄ではいかず、3 回方式を乗り換えてようやく完走させています（技術的な経緯は本記事末尾の[補足セクション](#%E8%A3%9C%E8%B6%B3-%E6%A7%8B%E9%80%A0%E8%A7%A3%E6%9E%90%E3%82%92%E5%8B%95%E3%81%8B%E3%81%99%E3%81%BE%E3%81%A7%E3%81%AE%E8%A9%A6%E8%A1%8C%E9%8C%AF%E8%AA%A4)にまとめました）。

以下の結果画像は、同じ `.frd` ファイルを FreeCAD の FEM ワークベンチで開いて取得したものです。本記事では GUI を使わずに解析を走らせていますが、結果ファイルは GUI でも同様に読めるので、カラーマップを切り替えたりモデルを回転させたりして対話的に確認するのに便利です。

### 応力分布（フォン・ミーゼス応力）

フォン・ミーゼス応力は、3 次元の複雑な応力状態（引張・圧縮・せん断の 6 成分）を、引張試験で測る降伏応力（1 方向の単軸応力）と直接比較できるよう 1 つのスカラー値に換算した指標です。金属のような延性材料の降伏判定に最も一般的に使われ、`σ_vm < 降伏応力` であれば永久変形しないと判定できます。今回のアルミ 6061-T6 なら 276 MPa が基準です。

![](https://static.zenn.studio/user-upload/9172c6a8a8c9-20260420.png)

* **最大フォン・ミーゼス応力: 0.13 MPa**
* **安全率: 2,144**（降伏応力 276 MPa ÷ 最大応力 0.13 MPa）

応力はエルボー関節付近と上腕リンクの根元に集中しています。ベースや手首付近は応力が低い。ロボットアームの構造として妥当な結果です — 長いリンクの付け根に曲げ応力が集中します。

### 変位分布

応力が「壊れるか」を見る指標なら、変位は「使えるか」を見る別の評価軸です。ロボットアームでは位置決め精度が重要で、荷重でアームがたわんで先端が教示した座標からずれると、壊れていなくても作業に使えません。

![](https://static.zenn.studio/user-upload/00a980510e7d-20260420.png)

* **最大変位: 5.76 μm（0.00576 mm）**

変位はベース（固定端）からフランジ（先端）に向かって大きくなっています。100N の荷重に対して最大変位 5.76μm は、産業用ロボットの位置精度（一般的に ±0.05mm 程度）に対して十分小さい値です。

100N（≒10kg）の荷重に対して安全率 2,000 以上、変位も位置決め精度の 1 桁下という過剰な余裕は、**今回のモデルが中身まで材料で詰まった「アルミ無垢モデル」** であることに起因します。実物の産業用ロボットアームは、上腕・前腕リンクを肉厚数 mm の薄肉中空構造（アルミ箱・鋳造パイプ）にしており、断面積は無垢モデルの 1/5〜1/10 程度しかありません。

## 補足: 構造解析を動かすまでの試行錯誤

[CAD 編集編](https://zenn.dev/aws_japan/articles/45fb5d3130035d) の CAD 編集と比べて、今回の構造解析はすんなりいきませんでした。ただ結論から言えば、**Kiro がエラーメッセージを起点に原因を切り分け、3 回方式を乗り換えて最終的に解析を完走させた**のも今回の記事のポイントです。

前提として、FreeCAD FEM で構造解析を実行するには **（1）形状のメッシュ分割 → （2）CalculiX 用の入力ファイル（`.inp`）生成 → （3）ソルバー `ccx` 実行 → （4）結果の読み取り** という 4 ステップを繋ぐ必要があります。どのステップを FreeCAD に任せ、どこを自前でやるかで方式が分かれます。

### 試行 1: FreeCAD FEM の Python API にすべてを任せる → 失敗

最もシンプルな方式は、FreeCAD FEM が提供する Python API（`ObjectsFem`）で解析オブジェクト・ソルバー・材料・境界条件・メッシュをすべて設定し、FreeCAD 自身に inp を書かせて、`FemToolsCcx` で ccx を起動してもらう、という方法です。GUI で手動操作するのと同じ手順を、スクリプトで自動化するイメージです。

問題が 2 つ発生しました。

**問題 1: API 名がバージョン間で変わっており、既存のサンプルコードがそのまま動かない**

FreeCAD FEM モジュールは現在も活発にリファクタリングされており、関数名やプロパティ名がバージョン間で変わります（例: `makeSolverCalculixCcxTools` → FreeCAD 1.1.0 では `makeSolverCalculiXCcxTools` と `X` が大文字に。`mesh_obj.Part` → `mesh_obj.Shape`）。

[FreeCAD の公式 Wiki](https://wiki.freecad.org/FEM_Module) には FEM ワークベンチの GUI 手順は丁寧に載っているのですが、`ObjectsFem.makeSolverCalculiXCcxTools` のような内部 Python API のシグネチャを体系的に網羅したリファレンスは存在しません。公式に用意されているのは GUI チュートリアルとサンプルスクリプトまでで、残りはソースコード（`src/Mod/Fem/femobjects/` 配下）とフォーラムの断片的な投稿を読んで使い方を推測することになります。Kiro も当初は検索でヒットするサンプル（古いバージョン向け）を試して `AttributeError` を踏み、そこから最新のソースコードを直接読みに行って修正する、という手順を踏みました。

**問題 2: `ConstraintForce` の荷重方向設定で `NodeLoadTable` エラー**

FreeCAD の `ConstraintForce`（力の境界条件）は、荷重の方向を **「どの面やエッジに対して垂直か／沿っているか」** という形状参照で決める仕組みです。GUI なら面をクリックして方向を指定できますが、ヘッドレスのスクリプト実行では、方向の基準となる面やエッジを API で明示的に渡す必要があります。

この「方向参照」を設定せずに inp の書き出しを呼ぶと、FreeCAD は内部で荷重をどの向きに分配すべきか計算できず、各ノードに割り付ける荷重テーブル（`NodeLoadTable`）の構築に失敗して `KeyError: 'NodeLoadTable'` が出ます。ヘッドレス環境で面やエッジの参照を API だけで正確に渡すのは煩雑で、ここで試行 1 を打ち切りました。これは AI 固有の難しさというより、**FreeCAD FEM のヘッドレス API が GUI 操作を前提に設計されている**ことに起因する制約です。

### 試行 2: inp を自前で書く → 別のエラーに当たる

試行 1 の `NodeLoadTable` エラーを回避するため、**FreeCAD には「形状を読み込んでメッシュを作る」ところまでを任せ、inp ファイルは自分で書いて ccx を直接起動する**方式に切り替えました。inp の中身（ノード座標、要素の接続情報、材料、境界条件、荷重）はプレーンテキストなので、メッシュ情報さえ取れれば Python で書き下せます。

ここで別のエラーに当たりました。

**問題 3: `*ERROR in e_c3d: nonpositive jacobian determinant`**

ccx を実行すると、ほぼすべての要素で「ヤコビアン行列式が非正」というエラーが出て計算が止まりました。四面体要素（C3D4）は 4 つの節点を決まった順番で並べることが期待されており、順序が反対向きになっていると、要素が「裏返った」状態と解釈されてヤコビアンが負になります（[CalculiX コミュニティでも頻出のトラブル](https://calculix.discourse.group/t/help-error-in-e-c3d-nonpositive-jacobian/1320)）。

FreeCAD が内部で保持しているメッシュ形式（`.unv`）と、CalculiX が期待する inp 形式の間でノード順序の規約が違うようで、FreeCAD 経由で取り出したメッシュをそのまま inp に書くとこのエラーが頻発しました。10 パーツを Boolean Fuse した全体形状でも、上腕リンク単体でも同じでした。

### 試行 3: Gmsh を直接呼ぶ → 成功

最終的に、**FreeCAD のメッシュ機能は使わず、Gmsh をコマンドラインから直接呼び出し、CalculiX 互換の inp を生成してもらう**方式でうまくいきました。

```
# FreeCAD で STEP → brep エクスポート（パーツ単位）
# Gmsh で brep → inp（CalculiX 形式）直接出力
gmsh upper_arm.brep -3 -clmax 12 -clmin 3 -order 1 -format inp -o mesh.inp
```

**なぜこれで解決したか**: Gmsh には CalculiX 互換の inp 形式で直接出力するモード（`-format inp`）があり、このとき C3D4 要素のノード順序は ccx が期待する向きで書き出されます。試行 2 でぶつかっていた「FreeCAD の unv 形式 → inp 変換の間でノード順序の規約がズレる」問題は、Gmsh が直接 inp を書くことで丸ごとスキップできる、というのがポイントです。

**なぜ最初からこの方式にしなかったか**: 一般論として、あるソフトウェアを使い始めるときは「公式に統合された GUI / API」から入るのが自然な順番です。FreeCAD FEM も公式ドキュメントは「FEM ワークベンチで GUI 操作」「Python API で同等の手順を自動化」という流れで整理されており、「内部のメッシュ機能をバイパスして Gmsh を直接呼べ」と書いてあるわけではありません。今回の Gmsh 直呼びは「FreeCAD の統合機能自体が inp 書き出し時点で問題を持っている」という診断を経て初めて選べる方式で、試行 1-2 で 2 種類のエラーを踏むまでは思いつきにくい発想でした。後から振り返れば、**FreeCAD をヘッドレスで FEM に使う場合は、メッシュ生成とソルバー起動は内部機能に頼らず Gmsh と ccx を直接叩くのが最も堅牢**と言えます。

Gmsh が出力する inp にはエッジ要素（T3D2）や面要素（CPS3）などボリューム以外の要素も含まれるので、Python で C3D4 ボリューム要素だけを抽出し、材料・境界条件・荷重を追加して完全な inp を組み立てました。`ccx` を直接実行して、正常に解析が完走しました。

## まとめ

### 設計 → 編集 → 解析の一気通貫

3 本の記事を通じて、以下のパイプラインが自然言語指示だけで実現できることを示しました:

今回扱ったロボットアームのようなシンプルな静的構造解析の範囲であれば、**設計の初期段階で「こういう形状にしたい」→「この形状で強度と変形は許容範囲か？」という一連のサイクルを、コードを 1 行も書かずに回せました。**
