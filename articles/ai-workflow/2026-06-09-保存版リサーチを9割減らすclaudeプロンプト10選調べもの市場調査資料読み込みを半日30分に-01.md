このページは、動画で紹介した「リサーチを9割減らすプロンプト10本」を、そのままコピーして使える形でまとめたものです。

動画の主張をひと言で言うと——  
リサーチは「設計 → 収集 → 検証 → まとめ」の4つの工程に分け、プロンプトを順番に重ねる。丸投げしないことで、半日の調べ物が30分になります。

リサーチ 半日（約4時間）→ 30分。月のリサーチ 約32時間 → 約4時間（年で約340時間、ほぼ4日分の削減）。

【大事な前提】Claudeは「知らないこと」をそれっぽく作ってしまうことがあります。事実・数字・出典は、必ず自分の目で裏を取ってください（詳しくは最後の「注意点」へ）。

◆ 全体像：リサーチ4工程 ＝ プロンプト10本

● 工程1 設計：何を・どの順で調べるか決める → ①リサーチ設計 ②問い分解 ③検索キーワード  
● 工程2 収集・読解：集めて、読んで、要点を抜く → ④長文要約 ⑤かみ砕き ⑥横断要約  
● 工程3 比較・検証：並べて、正しさを確かめる → ⑦比較表 ⑧競合・市場の下調べ ⑨反論チェック  
● 工程4 まとめ：提案書・記事の形にする → ⑩アウトプット化

ポイントは「同じチャットで続けて頼む」こと。一度テーマや資料を渡せば、Claudeは前の文脈を覚えているので、工程をまたいでも説明し直す必要がありません。文中の { } は、自分の状況に合わせて差し替えてください。

◆ 工程1：設計（プロンプト1〜3）

やみくもに検索する前に、まず「地図・ゴール・入り口」を手に入れます。ここまで5分かかりません。

【①リサーチ設計プロンプト】  
━━━━━━━━━━━━━━━━  
これから「{調べたいテーマ}」について調べます。  
初心者がまず押さえるべき論点を、体系的に整理してください。  
・大きな論点を5〜7個に分け、それぞれ「なぜ重要か」を一言添える  
・調べる順番（おすすめのリサーチ手順）も提示する  
・全体像が一目で分かるように、見出し＋箇条書きで  
━━━━━━━━━━━━━━━━  
→ やみくもに検索する前に「地図」を手に入れる。これだけで迷う時間が消えます。

【②問い分解プロンプト】  
━━━━━━━━━━━━━━━━  
この調べ物を、最終的に答えるべき「具体的な質問」のリストに分解してください。  
・各質問は、調べれば事実で答えられるサイズまで小さくする  
・「何が分かればこのリサーチは完了か」が分かる粒度で  
・まず答えるべき順に優先度を付ける  
━━━━━━━━━━━━━━━━  
→ 「何を調べれば終わりか」が見えると、リサーチは一気に速くなります。

【③検索キーワードプロンプト】  
━━━━━━━━━━━━━━━━  
「{テーマ}」を調べるための検索キーワードを、3段階で出してください。  
・初級（一般的な言葉）／中級（やや専門的）／専門家レベル（業界用語・英語含む）  
・各レベル5つずつ  
・それぞれ「このキーワードで何が見つかるか」を一言添える  
━━━━━━━━━━━━━━━━  
→ 自分では思いつかない専門用語・切り口が出てきて、いい情報に早くたどり着けます。

◆ 工程2：収集・読解（プロンプト4〜6）

集めた資料をClaudeに読ませて、要点を抜きます。ここで「読む時間」がごっそり消えます。

【④長文要約プロンプト】  
━━━━━━━━━━━━━━━━  
この資料を、次の型で要約してください。  
・【結論】最も重要な主張を3行で  
・【根拠】結論を支える事実・データを箇条書きで  
・【具体例】理解を助ける例を2〜3個  
※重要な数字・固有名詞はそのまま残す／ビジネスの提案に使えるトーンで  
（↓ここに記事・PDFの本文を貼る）  
━━━━━━━━━━━━━━━━  
→ ただ「要約して」だと平べったいまとめに。型を指定するだけで、そのまま提案に使える要約になります。

【⑤かみ砕きプロンプト】  
━━━━━━━━━━━━━━━━  
この専門的な内容を、前提知識のない人にも分かるように説明してください。  
・専門用語が出たら、その都度かんたんな言葉で言い換える  
・身近なたとえ（料理・スポーツ・日常）を使う  
・最後に「つまり何が言いたいのか」を1行でまとめる  
（↓ここに難しい資料・論文を貼る）  
━━━━━━━━━━━━━━━━  
→ 読むだけで30分かかる専門記事が、1分で頭に入る形になります。

【⑥横断要約プロンプト】  
━━━━━━━━━━━━━━━━  
これから複数の資料を貼ります。次を整理してください。  
・【共通している主張】どの資料も言っていること  
・【食い違っている点】資料によって意見が割れていること（誰がどう言っているか）  
・【取るべき結論】上記を踏まえた中立的なまとめ  
※どの主張がどの資料由来か、出典を併記する  
（↓ここに複数の資料を続けて貼る）  
━━━━━━━━━━━━━━━━  
→ 何本もの記事を1本ずつ読む必要がなくなります。「割れている論点」がひと目で分かるのが収集のヤマです。

◆ 工程3：比較・検証（プロンプト7〜9）

集めた情報を「信頼できる結論」に変える工程です。

【⑦比較表プロンプト】  
━━━━━━━━━━━━━━━━  
これらの選択肢を、比較表にしてください。  
・軸：価格／主な機能／向いている人／注意点  
・表のあとに「私（{自分の状況}）への推奨」を1つ、理由つきで  
（↓ここに候補の情報を貼る、または候補名を列挙）  
━━━━━━━━━━━━━━━━  
→ ツール選び・サービス選び・外注先選び。バラバラの情報が1枚の表になります。

【⑧競合・市場の下調べプロンプト（Web検索機能をON）】  
━━━━━━━━━━━━━━━━  
※Web検索機能をオンにして実行してください  
「{業界・市場名}」の主要なプレイヤーを調べて、整理してください。  
・主要プレイヤーを5〜8社  
・各社の「強み・弱み・ターゲット顧客・価格帯」  
・出典リンクを必ず付ける  
・最後に「この市場の空白地帯（狙い目）」を考察する  
━━━━━━━━━━━━━━━━  
→ 新しい市場に参入するとき、「誰がいて、何で戦っているか」の地図が数分で手に入ります。ただし、出た情報は必ず裏取りを（→注意点へ）。

【⑨反論チェックプロンプト】  
━━━━━━━━━━━━━━━━  
次は私が調べてまとめた結論です。これに対する「反論」と「私が見落としている観点」を挙げてください。  
・反対意見とその根拠を3つ以上  
・私の結論の弱い部分・前提の甘さを指摘  
・「この観点も調べるべき」という追加リサーチ項目も  
（↓ここに自分の結論を貼る）  
━━━━━━━━━━━━━━━━  
→ プロが必ずやる工程。自分の結論をわざと攻撃させると、抜けていた視点が出てきてリサーチの穴が埋まります。ひとりだと「都合のいい情報」だけを集めがち。それを防ぐのがこの9本目です。

◆ 工程4：まとめ（プロンプト10）

【⑩アウトプット化プロンプト】  
━━━━━━━━━━━━━━━━  
ここまでの調査をもとに、{提案書／記事／報告メモ}の構成案を作ってください。  
・結論ファースト（最初に要点）  
・見出し＋各セクションで何を書くかのメモ  
・想定読者：{読者}  
・最後に「この構成で説得力が弱い箇所」も指摘する  
━━━━━━━━━━━━━━━━  
→ 用途を「記事の見出し構成」「報告メモ」などに変えるだけで、調べたことがそのまま成果物の下書きになります。

◆ 10本を使いこなす3つのルール

1. 丸投げしない：「全部調べて」ではなく、4工程を順番に通す。プロンプトを重ねるほど精度が上がります。
2. 2. 1つのチャットで続ける：同じチャット内で工程を進めると、Claudeが前の文脈を覚えていて、毎回説明し直す手間が消えます。
3. 3. 出典は自分で開く：これが一番大事。Claudeが挙げた事実・数字は、必ず元のページを開いて自分の目で確かめます。

→ 設計して、重ねて、裏を取る。この3つで、リサーチは速く、そして信頼できるものになります。

◆ 実演：新規参入市場のミニ調査レポートを1本

【手作業の場合：4時間（半日）】  
・検索しながら情報を集める … 2時間  
・見つけた資料を読み込む … 1時間  
・比べて・まとめて・レポートにする … 1時間

【Claudeの場合：30分】  
・設計3本（地図・問い・検索KW）… 3分  
・検索機能＋要約で集めて要点を抜く … 15分  
・比較表＋反論チェックで整理・穴埋め … 7分  
・アウトプット化＋出典の確認 … 5分

4時間が、30分。最後に残るのは「確認する」という一番大事な仕事だけ。下ごしらえは全部Claudeが終わらせてくれます。

◆ 1ヶ月の試算  
・1回あたり：4時間 → 30分  
・週（リサーチ2回）：8時間 → 1時間  
・月：約32時間 → 約4時間  
・年：約380時間 → 約48時間  
月およそ28時間、年で約340時間の削減。戻ってくるのは、検索をさまよったり長い資料を読んだりという、一番すり減る時間です。

◆ 注意点（使う前に必ず確認）

1. 事実・数字は、うのみにしない
2. Claudeは知らないことをそれっぽく作ってしまうことがあります（ハルシネーション）。統計・固有名詞・日付の3つは、必ず一次情報で裏を取ってください。

1. 出典リンクは、自分で開く
2. Web検索機能が示したページも、中身が要約とずれていることがあります。引用する前に、必ず元のページを自分の目で確認します。

1. 機密情報・著作権
2. 社外秘の資料や有料記事の全文を貼るときは、利用プランのデータ取り扱い方針と、各サービスの利用規約を確認。心配な場合は固有名詞・数字を伏せてから貼る。

→ 調べるのはClaude、確かめるのは自分。この線引きさえ守れば、リサーチは速く、そして安全になります。

◆ まとめ

リサーチは「設計→収集→検証→まとめ」の4工程。10本のプロンプトを順番に重ねれば、半日の調べ物が30分になります。「すごい」ではなく「何時間削減できたか」。提案・企画・営業の前段がある人ほど効く仕組みです。  
次の調べ物が始まったら、まず①の設計プロンプトから。そこから始めてみてください。
