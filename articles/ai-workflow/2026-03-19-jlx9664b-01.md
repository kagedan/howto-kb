## JLX9664B グラフィックLCDモジュール 仕様書

aitendo取扱品（取扱終了品）。Siemens携帯電話用LCDを96x65ドットのグラフィック液晶モジュールとしてベース基板に実装したもの。

![](https://assets.st-note.com/img/1773885689-sPO7yH3naElcQqAJIidZFMbR.png?width=1200)

## 基本仕様

| 項目 | 値 |  
|------|-----|  
| コントローラ | PCF8812 (Philips/NXP) |  
| 総解像度 | 96 x 65 ドット |  
| アイコン行 | 1行（Y=0、96セグメント） |  
| ドットマトリクス | 96 x 64 ドット（Y=1〜64） |  
| 表示色 | 黄緑（FSTN） |  
| バックライト | LED（黄緑） |  
| インターフェース | 4線式SPI（シリアル） |  
| ロジック電圧 | 2.5〜5.5V |  
| バックライト電圧 | 3〜3.3V / 10〜20mA |  
| 液晶外形寸法 | 58 x 43mm |  
| 表示領域 | 52 x 30.5mm |  
| ベース基板寸法 | 88 x 60mm |  
| コネクタ | 2.54mmピッチ 8ピン スルーホール |

## ピン配置（シルク印刷、上から順）

| ピン番号 | 信号名 | 方向 | 説明 |  
|----------|--------|------|------|  
| 1 | RESET | 入力 | アクティブLOWリセット |  
| 2 | VSS | 電源 | GND |  
| 3 | SCLK | 入力 | SPIクロック |  
| 4 | SCE | 入力 | チップセレクト（アクティブLOW） |  
| 5 | D/C | 入力 | データ/コマンド選択（LOW=コマンド、HIGH=データ） |  
| 6 | SDIN | 入力 | SPIデータ入力（MOSI） |  
| 7 | VDD | 電源 | 3.3V |  
| 8 | LED+ | 電源 | バックライトアノード（100Ω経由で3.3Vに接続） |

## SPIインターフェース

### タイミング

```
SDIN  ──╲___X___╱──  データセット
SCLK  ____╱╲____╱╲  立ち上がりでラッチ
SCE   ╲____________╱  LOW中に転送
D/C   ===（コマンド時LOW、データ時HIGH）===
```

## 初期化シーケンス

```
// ハードウェアリセット
RESET = HIGH → 10ms → LOW → 100ms → HIGH → 100ms

// 拡張命令モード (H=1)
0x21        // Function Set: 拡張命令有効
0x09        // 内部HV-gen x3（チャージポンプ3倍、必須）
0x80|0x54   // Vop = 0x54（コントラスト、個体差あり）
0x16        // バイアス n=2
0x06        // 温度係数 2

// 通常命令モード (H=0)
0x20        // Function Set: 通常命令
0x0C        // ディスプレイON（ノーマルモード）
```

### コントラスト（Vop）について

### HV-gen（高電圧ジェネレータ）について

* `0x09`コマンドでx3モードを有効にする（**必須**）
* これがないとLCD駆動電圧が生成されず、一切表示されない
* U8glibの標準PCF8812ドライバにはこのコマンドが含まれていないため、JLX9664Bでは動作しない

## 画面座標系

### LCDハードウェアの物理配置（通常と逆）

```
LCD page 8 = 画面最上部（アイコン行）
LCD page 7 = アイコンの直下
  ...
LCD page 1 = 画面下部付近
LCD page 0 = 画面最下部

LCD column 0 = 画面右端
LCD column 95 = 画面左端
```

### 座標変換（bufSetPixel内で実装）

画面座標(sx, sy)からバッファ座標への変換:

```
// X反転: 画面左(0) → バッファ右(95)
int16_t bx = LCD_WIDTH - 1 - sx;

// Y反転: 画面上(0) → バッファ上位ページ(64)
int16_t by = LCD_HEIGHT - 1 - sy;

// バッファインデックス
uint16_t idx = bx + (by / 8) * LCD_WIDTH;

// ページ内ビット反転
uint8_t bit = 1 << (7 - (by & 7));
```

### 画面レイアウト

```
Y=0:      アイコンセグメント行（96ビット）
Y=1〜64:  ドットマトリクス描画領域（64行 × 96列）
```

## 描画性能

| 項目 | 値 |  
|------|-----|  
| SPI転送速度 | ビットバングで約1MHz相当（48MHz MCU、digitalWrite使用） |  
| フレームバッファサイズ | 864バイト（96 × 9ページ） |  
| 全画面転送時間 | 約29ms（理論値、コード上のFPS≒34） |  
| **実用フレームレート** | **5〜10FPS**（FSTN応答速度制限） |

### FSTN液晶の応答速度制限（実測）

コード上は約34FPSで描画・転送が可能だが、FSTN液晶パネルのピクセル応答速度が遅いため、実用フレームレートは大幅に制限される。

| FPS | 見え方 | 評価 |  
|-----|--------|------|  
| 5 | くっきり表示、残像なし | 最適 |  
| 8〜10 | 許容範囲、わずかに残像あり | 実用上限 |  
| 12 | ドットが飛ぶ（一部ピクセルの描画が追いつかない） | やや厳しい |  
| 15以上 | 滲んで視認困難 | 非実用的 |

アニメーション表示を行う場合は**8〜10FPS以下**にフレームレートを制限すること。静的な表示（テキスト、アイコン等）であればFPS制限は不要。

## トラブルシューティング

### 画面に何も表示されない

1. **Vopが低すぎる** — 0x40未満では視認不可。コントラストスイープで最適値を探す
2. **HV-gen未設定** — `0x09`コマンドが抜けている。これがないとLCD駆動電圧が生成されない
3. **配線の確認** — 特にRESET, SCE, D/Cの3本。テスターで各ピンの3.3V出力を確認

### 表示が上下左右反転する

このLCDモジュールはページ0=画面下、カラム0=画面右という配置のため、bufSetPixelで座標変換（X反転 + Y反転 + ページ内ビット反転）が必要。

### U8g2/U8glibで動作しない

U8g2のPCF8812ドライバには`0x09`（HV-gen x3）コマンドが含まれていないため、JLX9664Bでは動作しない。また座標系も逆のため、カスタムドライバが必要。

### 「ふわっと暗くなってすぐ消える」

Vop値が視認閾値ギリギリの場合に発生する。Vopを上げて安定した表示を得ること。

## 注意事項

* 本モジュールはaitendo取扱終了品であり、入手困難
* PCF8812の一般的な初期化シーケンス（U8glib等）ではHV-genコマンドが不足しているため動作しない
* アイコンセグメントのビット配置はSiemens携帯の仕様に依存しており、公式ドキュメントは存在しない
* 未使用ビット（06〜13, 15〜17等）は別バリエーションの携帯で使われていたアイコンの名残と推測される
