## 起：きっかけはClaudeへの何気ない質問だった

ここ2年ほど、ピアノよりプログラミングに時間を使っていた。「ABC（Always Be Coding）」をモットーに日々コードを書いているうちに、気づけばピアノから遠ざかっていた。久しぶりに練習を再開したいとは思うものの、なかなか踏み出せずにいた。

ある日、いつものようにClaudeにプロンプトを投げた。「Taskerと連携して何か面白いことできないか？」という軽い質問だった。返ってきた回答の中に、ふと気になる一文があった。

**「.midファイルをClaudeに読み込ませると演奏データを分析できます」**

え、ClaudeってMIDIファイルを認識できるの？  
ならピアノを弾いて読み込ませたら、どんなフィードバックが返ってくるんだろう？

思い立ったが吉日。必要なものを調べると「MIDI端子のある楽器、MIDIインターフェース、DAW」と、全部押し入れに眠っているではないか。よし、やってみよう。

---

## 承：3冊の説明書をClaudeに丸投げした

引っ張り出したのは3つの機器だ。

* **Nord Electro 3**（電子ピアノ）：高機能すぎて使っていない機能だらけ
* **TASCAM iU2**（MIDIインターフェース）：長らく眠っていた
* **Band-in-a-Box**（DAW）：友人の結婚式の余興で伴奏に使ったっきり

「簡単につながるだろう」と高をくくっていたが、演奏すれどPCは無反応。設定画面を開いても何が何やら分からない。

そこで思いついた。**説明書をClaudeに投げてしまおう。**

各メーカーのサイトからPDFマニュアルを落としてアップロードすると、Claudeは3冊分の機器の仕様を丁寧に解説してくれた。

* 「Nord Electro 3のUSBポートはMIDI通信に非対応で、5ピンDIN端子を使う必要がある」
* 「iU2のMIDI端子は3.5mmミニジャックなので変換ケーブルが必要」
* 「Band-in-a-BoxのMIDI録音は[Record MIDI]ボタンから行う」

自分では読み解くのに何時間もかかったであろう情報が、5分で整理された。すごいぞ、Claude。それでもMIDIでの録音はかなわなかった。

---

## 転：原因は「設定1つ」と「機器の寿命」だった

ここからが本番の詰まりポイントだ。

PowerShellでデバイスを確認すると、iU2はWindowsに認識されていた。

```
Get-PnpDevice | Where-Object { $_.FriendlyName -like "*iU2*" }

Status  FriendlyName
------  ------------
OK      iU2
OK      スピーカー (iU2)
OK      Microphone (iU2)
```

物理的な接続も正しい。それでも信号が来ない。Claudeと一緒にトラブルシューティングを進めた結果、Nord Electro 3のMIDIメニューに「ローカルコントロール：Off」という設定が潜んでいた。

**これがオフの状態だと、鍵盤を弾いても外部へMIDI信号が出力されない。**

設定をOnに変えても、まだ動かない。今度はiU2のMIDI回路自体が長年の眠りで機能しなくなっていた可能性が浮上した。Claudeに後継機種の選定を相談し、届いたU2MIDI Proを繋いだら──**一発でつながった。**

詰まっていた時間、約3日。原因は「設定1つ」と「古い機器」という、なんともシンプルな結末だった。

---

## 結：Claudeに演奏を聴いてもらったら、練習したくなった

録音にはSekaijuとBand-in-a-Boxの2種類を試した。同じ月光ソナタを2回録音して.midファイルをClaudeに渡すと、こんな比較が返ってきた。

### DAWごとの録音データ比較

| 項目 | Band-in-a-Box | Sekaiju |
| --- | --- | --- |
| ファイル形式 | Type 1 | Type 1 |
| テンポ記録 | 85 BPM | 120 BPM（デフォルト） |
| トラック数 | 8本 | 17本 |
| 余分な内容 | BiaB伴奏込み | 空チャンネルのみ |
| 総音数 | 454音 | 466音 |
| Claude分析への適性 | ◎ | ◎ |

### 演奏の強弱（ベロシティ）比較

|  | Band-in-a-Box版 | Sekaiju版 |
| --- | --- | --- |
| 平均ベロシティ | 71.9（やや強め） | 51.3（自然な強さ） |
| 最大ベロシティ | 111 | 92 |

Claudeからのコメント：

> 「Sekaiju版の方がベロシティが自然です。BiaBのメトロノームや伴奏に合わせて少し力が入ったのかもしれません。月光ソナタはpppの曲なので、Sekaiju版（平均51）の方が正しい方向に近いです」

MIDIデータには音の高さだけでなく、打鍵の強さ（ベロシティ）・音の長さ・タイミングのズレまで記録されている。Claudeはそれを解析して「全体的に強すぎます。月光ソナタはpppの指定です」「三連符のパターンは安定しています」と冷静に返してきた。

**褒められたことで、次も弾きたくなった。**

これが思わぬ収穫だった。演奏を聴いてくれる人がいるだけでモチベーションは上がる。Claudeはその代わりになれる。ただ、細かいミスまで見えてしまうので、今後は譜読みの教本をClaudeと一緒に読み進めながら、基礎から丁寧に練習していこうと思っている。

### 結論：SekaijuがClaude連携に最適

```
✅ シンプルで録音が簡単
✅ 演奏データだけが綺麗に保存される
✅ 余分な伴奏が混入しない
✅ Claude分析に最適
```

今日の教訓：**押し入れの機材と、AIと、20年のブランクがあれば、ピアノ練習は再開できる。**
