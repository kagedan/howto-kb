# そもそも「ストリーミング」って何？

```
通常：        送信 → 待つ → 全文一気に表示
ストリーミング：送信 → 少しずつリアルタイムで表示
```

ChatGPTやClaudeで文字がタイピングされるように表示されるあれです。

---

# なぜストリーミングが必要か

長い返答の場合、通常だと全部生成し終わるまで何も表示されません。

```
通常：　　　　10秒待つ → ドン！と全文表示
ストリーミング：すぐ表示開始 → 徐々に表示 → 完了
```

ユーザー体験が大きく変わります。

---

# なぜ `with` 文を使うのか

コードの前に、`with` 文について理解しておきましょう。

ストリーミングは**通信を張り続ける**処理です。

```
通常：        リクエスト → レスポンス → 接続終了（一瞬）
ストリーミング：リクエスト → 接続を保ちながら少しずつ受信 → 接続終了
```

接続を「開いて・使って・閉じる」という管理が必要になります。

**`with` なしだとどうなるか**

```
# ❌ with なし
stream = client.messages.stream(...)
for text in stream.text_stream:
    print(text, end="", flush=True)
# ← ここで接続を閉じるのを忘れがち！
#   エラーが起きても閉じられずリソースが残り続ける
```

**`with` ありだと**

```
# ✅ with あり
with client.messages.stream(...) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)
# ← ブロックを抜けた瞬間に自動で接続が閉じられる
#   エラーが起きても確実に閉じられる
```

`with` 文はブロックを抜けると**正常終了でもエラーでも必ず後片付けしてくれる**仕組みです。

---

# コードで見てみる

```
from dotenv import load_dotenv
import anthropic

load_dotenv()
client = anthropic.Anthropic()

# ストリーミングで受け取る
with client.messages.stream(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[
        {"role": "user", "content": "LLMとは何ですか？詳しく説明してください。"}
    ]
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)  # 届いた断片をすぐ表示

print()  # 最後に改行
```

通常の `messages.create` の代わりに `messages.stream` を使うだけです。

# 通常との比較

**通常（全文待つ）**

```
response = client.messages.create(...)
print(response.content[0].text)  # 全部来てから表示
```

**ストリーミング（少しずつ）**

```
with client.messages.stream(...) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)  # 断片が届くたびに表示
```

---

# `end=""` と `flush=True` って何？

```
print(text, end="", flush=True)
```

| 引数 | 意味 |
| --- | --- |
| `end=""` | 改行しない（デフォルトは改行あり） |
| `flush=True` | バッファを即時出力する（溜め込まず即表示） |

---

# ストリーミング完了後に全文も取れる

```
with client.messages.stream(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[
        {"role": "user", "content": "LLMとは何ですか？詳しく説明してください。"}
    ]
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)

    # ストリーミング完了後に全文取得も可能
    final = stream.get_final_message()
    print(f"\n\n使用トークン：{final.usage.input_tokens} / {final.usage.output_tokens}")
```

---

# まとめ

* `messages.stream` を使うだけでストリーミングになる
* ストリーミングは接続を保ち続けるため `with` 文で確実にクローズする
* `for text in stream.text_stream` で断片を受け取る
* `end=""` と `flush=True` は必須
* 完了後に `get_final_message()` でトークン数も取れる
