ルートマップができたので、次はAIが旅のルートを提案してくれる機能を作ります。行き先と日数を入れると、AIがスポットを提案して地図に表示してくれる /plan ページです。  
※このAIの機能は現在Webページから削除しています。

### 罠①「Claude Proに入ってるからAPIも使えると思っていた」

まずAPIキーを取得しようと、console.anthropic.com にアクセスしました。

ここで最初の詰まりです。「Claude.aiのProプランに入っているんだから、APIも使えるんじゃないか」と思っていたのですが、Claudeに確認したところ、こんな返答が来ました。

> 「Claude.aiのProプランとAnthropicのAPIは別物です。APIを使うにはConsoleにクレジットを追加する必要があります。最小$5から追加できます。」

チャット用のサブスクと、開発者向けのAPIは課金体系が違うとのことでした。そういうものか、と理解して進めました。

「ユーザーがたくさん使ったら費用はどんどん増えるのか」も気になって確認しました。Claudeに聞くと、1回のルート生成あたり約0.5〜0.8円で、月1,000回使われても600円程度との回答でした。この金額なら問題ないと判断して、$5追加して進めることにしました。

### 罠②「モデル名を間違えるとエラーになる」

APIキーを取得して実装を進めると、最初に書いたコードでこんなエラーが出ました。

```
not_found_error
```

原因はモデル名の指定ミスでした。　※claude-sonnet-4-5（現時点のモデル）

```
誤: claude-sonnet-4-20250514
正: claude-sonnet-4-5
```

モデル名は正確に指定しないとエラーになります。Claudeに確認したら正しいモデル名を教えてもらえて、すぐに解決しました。

### 罠③「JSONが途中で切れてパースエラーになる」

モデル名を直したら動き始めたのですが、今度は別のエラーが出ました。

```
Expected ',' or ']' after array element in JSON at position 1212
```

AIが返すJSONが途中で切れていてパースできない、というエラーです。

原因は max\_tokens: 1024 という設定でした。AIが返せるテキストの長さに上限を設けていたのですが、スポット数が多い場合にその上限に引っかかってJSONが途中で切れてしまっていました。

```
変更前: max_tokens: 1024
変更後: max_tokens: 2048
```

これを2048に増やしたら、JSONが最後まで返ってくるようになりました。

### 動いた

3つの罠を乗り越えて、/planページが動き始めました。

行き先・日数・テーマを入力してボタンを押すと、AIがスポットを提案して地図にピンと曲線で表示されます。

ただ、最初はスポットの順番がバラバラで地図がごちゃごちゃになっていました。AIが効率的な順番で並べてくれないことがあるためです。近いスポットを順番に繋ぐ形に修正してもらいましたが、正直まだ見にくい部分があります。ルートの表示については、引き続き改善が必要だと思っています。

### 気づいたこと

今回は「詰まった原因が全部事前に調べれば分かること」でした。

モデル名の確認、max\_tokensの設定、ProとAPIの違い、どれもドキュメントを読めば分かる話です。ただ初めて実装する時は「何を確認すれば良いか」自体が分からないので、エラーが出てから初めて気づく形になりました。

Claudeに「このエラーは何ですか?」と聞けばすぐ答えが返ってくるので、エラーが出たら即座に貼り付けて確認するのが一番早いと感じました。
