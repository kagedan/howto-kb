エンジニアとして、会議の音声・動画から日本語の議事録（決定事項・アクションアイテム付き）を自動生成するサービスを開発しています。文字起こしに OpenAI Whisper、議事録生成に Anthropic Claude を使った構成と、実装で詰まった点を共有します。

## 全体構成

フロント: Next.js (App Router) / バックエンド: FastAPI (Python)

文字起こし: OpenAI Whisper API / 議事録生成: Anthropic Claude / 保管: AWS東京リージョン

流れ: アップロード →（動画なら）ffmpegで音声抽出 → Whisperで文字起こし → Claudeで構造化議事録 → 保存・表示

## 詰まり1: 動画から音声を抽出

ユーザーはMP4などの動画をそのまま上げてくるので、Whisperに渡す前に音声だけ抜きます。

```
ffmpeg -i input.mp4 -vn -ar 16000 -ac 1 -c:a libmp3lame output.mp3
```

16kHz/モノラルに落とすとサイズが減り、精度もほぼ落ちません。

## 詰まり2: 長い音声のチャンク分割

Whisper APIにはファイルサイズの上限があるため、1時間を超える会議は分割が必要です。無音区切りで分割すると単語が切れにくく、タイムスタンプを保持して結合順を担保します。

## 詰まり3: Claudeのプロンプト設計（構造化が肝）

「要約して」だけだと議事録になりません。出力をJSONで固定し、決定事項／アクションアイテム（担当・期限）／論点を分けて出させます。\*\*「原文にない情報は推測しない」\*\*を入れないと、それっぽい担当者をハルシネーションで作ります。

## 詰まり4: コスト最適化（モデルの使い分け）

全部を高精度モデルでやると原価が合いません。翻訳（多言語→日本語）は安価なモデル、議事録の構造化要約は精度重視のモデル、と役割で分けます。無料枠を出す以上、1ユーザーあたりのトークン原価は死活問題なので、処理前にトークン量を見積もってガードを入れています。

## データの扱い

日本の会議は機密情報が多いので、音声・議事録の保管はAWS東京リージョン。音声は文字起こし後すぐ削除し、テキストだけ残す設計にしました。

## おわりに

Whisper + Claude の組み合わせは、日本語議事録の自動化に十分実用的でした。特に効いたのは **長音声のチャンク分割** と、プロンプトでの **「原文にない情報は推測しない」** 指定です。同じ構成を作る人の参考になれば。フィードバックもらえると嬉しいです。
