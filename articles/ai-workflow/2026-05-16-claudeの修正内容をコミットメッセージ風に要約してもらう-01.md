## git commit は自分で実行したい

AIに修正してもらった後で修正内容をコミットする際、個人開発の場合はAIにそのままコミットしてもらっていますが、  
仕事の場合は一応自分でadd/commitを実行したいと思っています。

その際にコミットメッセージの検討が少し面倒なため、  
AIに「修正+コミットメッセージの提案」をしてもらいました。

## 設定方法

Claudeは指示を与えると以降もそのルールを守るようにメモリに記憶してくれます。  
適当なClaudeのセッションで以下のような指示を与えるとmemoryディレクトリに設定を保存してくれます。

> 修正のたびに、コミットメッセージとして使える1行を表示して。プレフィックス付きで。

すると以下のようなファイルが作成されます。

`~/.claude/projects/<プロジェクトのパス>/memory/feedback_commit_message.md`

```
---
name: コミットメッセージの表示
description: 修正を行うたびに、コミットメッセージとして使える1行の内容を表示する
type: feedback
---

修正を行うたびに、コミットメッセージとして使える1行の内容を表示する。

**Why:** ユーザーが毎回コミットメッセージを考える手間を省くため。
**How to apply:** コード修正を完了したら、その変更内容を要約した1行のコミットメッセージ（`fix:` / `chore:` 等のプレフィックス付き）を表示する。
```

## 出力例

AIにコード修正をしてもらった際に以下のように修正内容が1行で出力されるようになります。

```
fix: 月をまたぐ期間の日数計算が1日ずれる不具合を修正

## 修正内容

- `src/lib/date.ts` の `diffDays` 関数で、終了日を含めずに計算していたため
  月をまたぐケースで結果が1日少なくなっていた
- 終了日を含める仕様に統一し、関連する単体テストを追加
- 呼び出し側のシグネチャ変更はなし
...
```

## おわりに

AI修正後に内容をコミットメッセージ風に要約してもらうことで  
コミットメッセージを考えるコストが軽減され、また、修正内容を把握する助けになりました。  
ちょっとしたことですが個人的に作業が結構楽になりました。
