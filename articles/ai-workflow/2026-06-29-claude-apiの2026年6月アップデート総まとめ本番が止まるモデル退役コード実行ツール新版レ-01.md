2026年6月、AnthropicはClaude開発者プラットフォーム（Claude API・各SDK・Console）に立て続けの変更を入れた。中身は地味だが、本番で動くコードに直接効くものが多い。「退役済みモデルへのリクエストはエラーを返す」「コード実行ツールに新版が来てベータヘッダ不要で使える」「レート制限のティアが3つに統合された」——いずれも、知らずに月をまたぐと壊れる・損をするタイプの更新だ。出典は公式リリースノート（[Claude Platform release notes](https://platform.claude.com/docs/en/release-notes/overview)）。この記事では、6月分のうちエンジニアが今日動くべき3点を整理する。

## 結論

| 確認項目 | ニュースの含意 | 直すこと |
|---|---|---|
| Sonnet 4 / Opus 4 | 6月15日に退役。リクエストはエラー | モデルIDを `claude-sonnet-4-6` / `claude-opus-4-8` に置換 |
| Opus 4.7 fast mode | 6月25日に非推奨、7月24日に削除 | `speed:"fast"` をOpus 4.8へ移すか外す |
| code_execution_20260120 | 6月18日に全7SDKが対応、ベータヘッダ不要 | ツールの `type` を更新しREPL永続化/PTCを有効化 |
| レート制限 | 6月26日にStart/Build/Scaleの3層へ統合 | Sonnet/Haikuのスループット前提を見直す |

## モデルの退役と非推奨：retired と deprecated は別物

### 確認できる事実

- **2026年6月15日、Claude Sonnet 4（`claude-sonnet-4-20250514`）と Claude Opus 4（`claude-opus-4-20250514`）が退役した。** これらへのリクエストはエラーになる。推奨移行先はそれぞれ Claude Sonnet 4.6 と Claude Opus 4.8。
- **2026年6月25日、Claude Opus 4.7 の fast mode が非推奨になった。** 削除は2026年7月24日で、削除後に `claude-opus-4-7` へ `speed: "fast"` を付けたリクエストはエラーを返す。移行先は Opus 4.8 の fast mode。
- **2026年6月5日、Claude Opus 4.1（`claude-opus-4-1-20250805`）の非推奨が告知された。** Claude APIでの退役は2026年8月5日予定。

原文: "We've retired the Claude Sonnet 4 model (`claude-sonnet-4-20250514`) and the Claude Opus 4 model (`claude-opus-4-20250514`). All requests to these models will now return an error."
日本語訳: 「Claude Sonnet 4（`claude-sonnet-4-20250514`）と Claude Opus 4（`claude-opus-4-20250514`）を退役した。これらのモデルへのすべてのリクエストは、今後エラーを返す。」（出典: Claude Platform release notes, 2026年6月15日）

### 実務解釈

退役（retired）と非推奨（deprecated）を混同すると事故になる。退役済みは**今すぐエラー**、非推奨は**退役予定日まで動く猶予あり**だ。6月15日の Sonnet 4 / Opus 4 は前者で、コードや設定に日付サフィックス付きの旧IDが残っていれば、その経路はすでに落ちている。

fast modeの非推奨（7月24日削除）は後者で、まだ動くが期限がある。`speed: "fast"` を本番で使っている場合は、削除日までに Opus 4.8 の fast mode へ移すか、429時に通常モードへ落とすフォールバックを入れておく。期限を「いつか」で放置すると、ある日突然エラーになる典型パターンだ。

なお、Anthropicのドキュメントは「アップグレードの既定は最新Opus（`claude-opus-4-8`）」としている。Fable 5（`claude-fable-5`）はOpus級を超える料金で、思考が常時オン・拒否のフォールバック設計など挙動も異なるため、「最新だから」で安易に差し替えるモデルではない。

## コード実行ツールの新版：ベータヘッダ不要で無償アップグレード

### 確認できる事実

- **2026年6月18日、Python / TypeScript / Go / Java / Ruby / PHP / C# の各SDKが `code_execution_20260120` に対応した。** これはREPLの状態永続化を追加し、programmatic tool calling（コードからツールを呼ぶ機能）の最低バージョンでもある。採用方法はツールの `type` を `code_execution_20260120` にするだけで、ベータヘッダは不要。対応モデルは Claude Fable 5 / Mythos 5 / Opus 4.5以降 / Sonnet 4.5以降。
- **2026年6月11日、`code_execution_20260521` が登場し、90秒/セルの実行時間制限をツール説明に明示するようになった**（モデルが長時間セルを見積もれる）。同日、`web_search_20260318` / `web_fetch_20260318` が `response_inclusion` パラメータを追加し、消費済みの検索結果ブロックをAPIレスポンスから落とせるようになった。いずれもベータヘッダ不要。

原文: "To adopt it, set the tool's `type` to `code_execution_20260120`; no beta header is required."
日本語訳: 「採用するには、ツールの `type` を `code_execution_20260120` にするだけでよい。ベータヘッダは不要だ。」（出典: Claude Platform release notes, 2026年6月18日）

### 実務解釈

このタイプの更新は「ベータヘッダ不要・コード1箇所」で入る無償の改善なので、見送る理由がほとんどない。コード実行ツールを使うエージェントは、`type` 文字列を新版に更新するだけでREPLの状態永続化が効き、複数セルにまたがる処理（前のセルで読み込んだデータを次のセルで使う等）が安定する。programmatic tool callingを使う予定があるなら、この版が前提になる。

`web_search_20260318` の `response_inclusion` は、エージェントのコンテキスト肥大に効く。検索を多用するループでは、消費済みの結果ブロックがAPIレスポンスに残り続けてトークンを食う。これを落とせるだけで、長時間エージェントのコスト曲線が変わる。

```text
# ツール宣言の type を新版に差し替えるだけ（ベータヘッダ不要）
- { "type": "code_execution_20250825", "name": "code_execution" }
+ { "type": "code_execution_20260120", "name": "code_execution" }
```

ただし版を表す数値（`_20260120`）は**リリースノートの文字列をそのままコピー**すること。「最新そうな日付」を手で打つと存在しないバージョンで400になる。

## レート制限の3層統合：Sonnet/HaikuがOpus並みに

### 確認できる事実

- **2026年6月26日、Claude API全体でレート制限が引き上げられた。** Claude Sonnet と Claude Haiku のレート制限が全ティアで Claude Opus と同等になり、ティアは Start / Build / Scale の3つに統合された。多くの組織はより上位のティアへ移り、上限が下がる組織はなく、対応は不要。現在のティアと上限はConsoleで確認できる。

原文: "Claude Sonnet and Claude Haiku rate limits now match Claude Opus at every usage tier, and usage tiers have been consolidated into three: Start, Build, and Scale."
日本語訳: 「Claude Sonnet と Claude Haiku のレート制限が、全使用ティアで Claude Opus と一致するようになった。使用ティアは Start / Build / Scale の3つに統合された。」（出典: Claude Platform release notes, 2026年6月26日）

### 実務解釈

「対応不要」の更新だが、設計上は前提が変わる。これまでコスト最適化のために重い処理をSonnet/Haikuへ逃がしていた構成では、これらの**スループット上限がOpus並みに上がった**ため、フォールバックやバッチの並列度を見直す余地がある。リトライ閾値を「測定したレート制限値」にハードコードしているコードがあれば、新しい上限で再調整しておく。

ティア名が `Start / Build / Scale` の3段に整理されたので、社内ドキュメントやアラートで旧ティア名を参照している箇所は表記を合わせておくと混乱が減る。

  - [ ] Opus 4.1依存があれば8月5日の退役前に移行計画を立てた

  - [ ] コード実行ツールの `type` を `code_execution_20260120` に更新した（ベータヘッダ不要）

## 失敗パターン

**パターン1：退役を「非推奨だから猶予がある」と誤解する** → retired（退役）は即エラー、deprecated（非推奨）は退役予定日まで動く。6月15日のSonnet 4 / Opus 4は前者。対策：両者を区別し、退役済みIDは最優先で置換する。

**パターン2：モデルIDに日付サフィックスを付ける** → エイリアス（`claude-opus-4-8`）に余計な日付を足すと404になる。対策：エイリアスはそのまま使い、構築しない。

**パターン3：fast modeの削除日（7月24日）を見落とす** → 削除後は `claude-opus-4-7` + `speed:"fast"` が突然エラーになる。対策：期限をカレンダーに入れ、事前にOpus 4.8へ移すかフォールバックを実装する。

**パターン4：ツールの版番号を手で「最新そうな日付」に書き換える** → 存在しないバージョンで400。対策：`code_execution_20260120` のような文字列はリリースノートからコピーする。
