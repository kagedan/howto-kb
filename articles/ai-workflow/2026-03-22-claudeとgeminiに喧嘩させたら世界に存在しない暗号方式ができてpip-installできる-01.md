# ClaudeとGeminiに喧嘩させたら、世界に存在しない暗号方式ができてpip installできるようになった

## はじめに

[世界最速のソートアルゴリズムを発明](https://qiita.com/urokiurotsuki/items/dd95f4ebd86fd7f2f69f)して、[ツンデレにコードレビューしてもらうLinter](https://qiita.com/urokiurotsuki/items/dc71943e7119fb724a97)を作った者です。

前2作は「ふざけてるけどちゃんと動く」がコンセプトでした。ZoroSortは全データを消去してO(0)でソート、ツンデレLinterはAST解析でガチのバグを見つけつつ罵倒してくる。ネタだけど技術的にはちゃんとしてる、というライン。

で、3作目。今回は逆です。**ふざけてない。ガチです。**

AIに「この世にまだ存在しない暗号を作って」と頼んだら、1回のチャットで独自の暗号方式が完成してしまい、PyPIに公開するところまで行ってしまいました。

```
pip install kinetic-cipher
```

⚠ 教育・実験用です。ガチの機密にはAES使ってください。

## 💥 Geminiに15点つけられた

次の案「衝突暗号」。14個のボールが円形コースを跳ね回って、衝突で速度や重さが変わる。その動きから暗号鍵を生成する物理シミュレーション方式。今度は先行研究なし。

ここで思いつきました。**Claudeが作ったものをGeminiに採点させたら面白くない？**

結果：**15/100点。** 「改ざん検出できない」「パスワード処理甘い」「ボールが止まる」...全部もっとも。

ZoroSortのベンチマークでBubbleSortに213,000倍勝ったときは嬉しかったけど、こっちの15点はマジで凹みました。

## ⚔️ AI同士のレビューバトルで55点アップ

Geminiの指摘をClaudeに投げて全修正。またGeminiに投げる。

改ざん検出 → HMAC-SHA256導入。パスワード → PBKDF2で10万回強化。ボール停止 → エントロピーポンプでエネルギー注入。

**Gemini再採点：70点。** 1往復で55点アップ。AI同士のピアレビュー、普通に機能する。

ツンデレLinterは自分のコードにダメ出しするツールだったけど、こっちはAIがAIにダメ出しするという構図。なんか似てる。

## 🐛 致命的バグの話

完成！テスト全通過！...あるサイズだけ復号できない。

暗号化と復号でボールの動きがズレてた。**リリース後ならユーザーのファイルが永久に開けなくなるバグ。**

ZoroSortは全データ消去がコンセプトだから消えても仕様通りだけど、暗号でデータが消えたら事故です。

Claudeが書いて、テストして、「問題なし」と言った。でもバグはあった。最終的に10件修正（致命的4件）。さらにリリース後、別のClaudeに聞いたら追加2件見つかって即修正。

**AIで作って、AIで査読して、別のAIで叩いて、また直す。** このサイクルが大事。

## 💪 実際どのくらい使えるの？

```
from kinetic_cipher import encrypt, decrypt

secret = encrypt(b"社外秘メモ", "パスワード")
original = decrypt(secret, "パスワード")
```

2行。依存ライブラリ**ゼロ**。10KBの軽量パッケージ。ZoroSortと同じくPython標準ライブラリだけで動きます（こっちはデータが消えません）。

Rust/WASM版 × 8コアCPUでの速度：

| ファイル | サイズ | 時間 |
| --- | --- | --- |
| テキスト・写真・MP3 | 〜10MB | ⚡ 0.1秒以下 |
| PDF文書 | 50MB | ⚡ 0.5秒 |
| 動画（10分） | 500MB | ✅ 5秒 |
| 映画（2時間） | 4GB | ✅ 43秒 |

日常のファイルは全部1分以内。AESとの速度差は8コアで約8倍まで縮まります。

## 🤔 AIすごい。でも手綱は握れ。

ZoroSort、ツンデレLinter、Kinetic Cipher。3つ作ってみて思ったのは、**ジョークとガチで必要な「人間の関与」のレベルが全然違う** ということ。

ZoroSortはAIに丸投げで成立した。`arr.clear()` に論文つけるだけだし。ツンデレLinterもほぼ丸投げ。口調の調整くらい。

でも暗号は違った。

**AIは嘘をつく** → 先行研究を「ない」と言った  
**AIはバグを見逃す** → 致命的欠陥を「問題なし」と報告した  
**AIは万能じゃない** → 数学的な安全性証明はできない

一方で、設計・実装・テスト・バグ修正・査読・公開、全部AIがやってくれたのも事実。自分はコード0行。**判断だけ全部やった。**

AIは最強の道具だけど、扱うものが「笑い話で済むネタ」か「笑えないガチ」かで、手綱の握り方が変わる。3作通して一番学んだのはそこでした。
