Claudeで色々調べることも増えているのですが、Claudeで調べると、ソースが曖昧な情報も結構紛れてきます。そうなると、やはりGoogle検索等で自分で調べるかーとなるんですよね。

「何かいい方法はないか」と感じていたところで考えたのが、NotebookLMを使うことでした。NotebookLMであれば、「（Fast or Deep）Research」でテーマに対して調べることができるし、ソースを明確にしつつ、いろんな情報を柔軟に引き出すことができる。

さらに調べてみると、NotebookLMをMCPで接続する方法もあるらしく、そうであればClaudeからNotebookLMを直接操作できる仕組みができるので、調査→分析→出力がもっと幅広く、効果的にできるのでは？と考えました。

特にNotebookLMが効果を発揮するのは「調査」のところですね。

この記事では、NotebookLMとClaude（AIエージェント）をつなぐワークフローを紹介します。

読み終わる頃には、次の3つが手に入ります。

ひとつ最初に断っておきます。この記事で紹介するツールは**Google公式ではありません**。有志によるオープンソースプロジェクトであり、利用規約への抵触リスクや動作の不安定性があります。その点を理解した上で、先を読んでください（詳しくは後半で整理します）。

---

## NotebookLMを利用するメリット

### ハルシネーションをゼロに近づける設計

AIの回答に「それ本当？」と感じる場面は、使い慣れた人ほど多いはずです。この現象は**ハルシネーション**と呼ばれ、AIがソースにない情報をもっともらしく作り上げることを指します。

NotebookLMはここが根本的に違います。「**アップロードしたソース内に情報がなければ、回答を拒否する**」設計です。

もう一つ見逃せないのが**トークン消費の節約**です。ClaudeにPDFや記事を直接読ませると、毎回大量のトークンが消費されます。NotebookLMはGoogleのAIが事前にドキュメントを処理・インデックス化しているため、エージェント側の消費を最小限に抑えられます。

---

## 実際に使っている場面

### 記事執筆の材料集め

この記事自体、この連携で書いています。

フローはこうです。

1. テーマに関するURL・PDF・YouTubeをNotebookLMに放り込む
2. Claudeに「このノートブックで記事の材料を集めて」と依頼する
3. Claudeが自律的にNotebookLMへ複数のクエリを投げ、引用付きでまとめてくれる
4. その素材をもとに記事を書く

以前は「調べる」と「書く」の間に必ず「コピペして貼り付ける」工程がありました。今はそれがない。

より実感しているのが**出典の信頼性**です。Claudeが持ってくる情報は、私が事前にNotebookLMへ入れたソースに基づいています。「どこから拾ってきたか分からない情報」が混ざらなくなりました。

最終確認は自分でします。それでも「たたき台の精度」が上がった分、確認の時間は明らかに減っています。

### ポッドキャスト化──「聴ける調査結果」

NotebookLMにはAudio Overview（音声解説）という機能があります。ノートブックの内容を自動でポッドキャスト形式に変換する、地味ながら強力な機能です。

連携前は「生成ボタンをクリックして待つ」。連携後は、Claudeに「このノートブックを音声解説にしておいて」と伝えるだけで、バックグラウンドで処理が走ります。

活用しているのは「移動中のインプット」です。前日の調査をポッドキャスト化しておき、翌朝の通勤中に聴く。テキストを読む時間がとれない日でも、インプットが止まりません。

リクエスト自体は1分もかかりません。生成に10分程度かかりますが、その間は別の作業ができます。

## NotebookLM MCPのセットアップ方法

### Python版を選んだ理由

NotebookLM用のMCPサーバーにはいくつか種類があります。現在も、ツールの更新などが行われているようでもあります。

自分が導入したのは**Python版（notebooklm-mcp-cli）**です。ノートブックの作成からソース追加・コンテンツ生成まで、ほぼすべての操作をエージェントから直接できるのが決め手でした。

参考にしたのはこちらの記事です。

セットアップの流れは4ステップです。

**ステップ1：必要なものを揃える**

* **Google Chrome**（ブラウザ認証が必要）
* **uv**（Pythonパッケージ管理ツール）

**ステップ2：インストール**

```
uv tool install notebooklm-mcp-cli
```

**ステップ3：Google認証**

```
nlm login
```

Chromeが自動で立ち上がります。**開いたウィンドウを途中で閉じないこと**。閉じた瞬間に「Authentication failed」エラーになります。

**ステップ4：設定ファイルへの書き込み**

設定ファイルへの書き込みを行い、Claude Desktopを再起動すれば、通常のClaudeのチャット等からNotebookLMを利用できるようになります。

### エラーはClaudeに聞けばよかった

正直に書くと、セットアップ中にエラーが出ました。自分の環境の問題だったのですが、エラーメッセージをそのままClaudeに貼り付けて「これどういう意味？」と聞けば、原因と対処法をすぐに返してくれました。特別な技術知識は不要です。エラーに臆さず、一つひとつ確認していけば完了できます。

## 使う前に知っておくこと

改めて整理します。

**① 公式ツールではない**　現在普及しているMCPサーバーは、すべて有志によるオープンソースプロジェクトです。

**② 利用規約への抵触リスクがある**　Googleの規約は「自動化された手段によるサービスへのアクセス」を禁止している場合があります。ブラウザ自動操作はこれに該当する可能性があります。

**③ 突然動かなくなる可能性がある**　GoogleがNotebookLMのUI構造を変更した瞬間に、MCPサーバーが機能しなくなることがあります。

**④ 1日約50クエリの制限がある**　無料版には1日あたりのクエリ制限があり、MCP経由でも同様に適用されます。

これらを把握した上で、お試しで使ってみる、ぐらいが現実です。最悪、MCPにしなくてもNotebookLM自体で操作することもできます。
