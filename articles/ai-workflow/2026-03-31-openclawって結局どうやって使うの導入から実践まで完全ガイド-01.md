2026年、AI界隈で最も話題をさらったオープンソースプロジェクトといえば、間違いなく**OpenClaw（オープンクロー）**でしょう。

GitHub史上最速で25万スターを突破し、2026年3月時点で32万スターを超え、NVIDIAのJensen Huang CEOに「おそらくソフトウェア史上、最も重要なリリース」とまで言わしめたこのツール。中国では「ロブスター飼育（lobster-farming）」と呼ばれるブームが巻き起こり、日本でもエンジニアを中心に急速に注目が集まっています。

でも正直、「名前は聞くけど、結局何なの？」「自分でも使えるの？」という方が多いのではないでしょうか。

## そもそもOpenClawとは何か？

OpenClawは、オーストリアのエンジニアPeter Steinberger氏が2025年11月に開発した、**オープンソースの自律型AIパーソナルアシスタント**です。

「ChatGPTやClaudeと何が違うの？」

これが一番多い質問ですが、決定的な違いは\*\*「実行能力」\*\*にあります。

たとえば、ChatGPTに「デスクトップのファイルを整理して」と頼んでも、「こうすればいいですよ」とアドバイスが返ってくるだけです。しかしOpenClawは、あなたのルールに従って**実際にファイル名の変更・分類・移動を直接実行**します。

これが「チャットボット」と「AIエージェント」の本質的な違いです。

### OpenClawの3つの決定的な特徴

**① 実行能力**   
シェルコマンドの実行、ブラウザ操作、ファイル管理、メール送信まで、PCを直接操作してタスクを完遂します。

**② 永続的メモリ**   
セッションを超えて会話内容やユーザー設定を記憶し続けます。毎回ゼロから説明し直す必要がありません。

**③ マルチチャネル統合**   
WhatsApp、Telegram、Discord、Slack、LINE、iMessage、Microsoft Teamsなど、50以上のメッセージングプラットフォームから操作可能。普段使っているアプリからそのままAIに指示を送れます。

### 名前の変遷について

OpenClawは短期間で3回名前が変わっています。「Clawdbot」（2025年11月）→「Moltbot」（2026年1月27日）→「OpenClaw」（2026年1月30日）。Anthropic社からの商標に関する指摘を受けてのリネームでした。ネット上で旧名を見かけても、すべて同じプロジェクトを指しています。

## OpenClawを始める前に必要なもの

OpenClawを導入するには、以下の準備が必要です。

**ハードウェア要件**

**「24時間動かすなら専用マシンが必要」という話**

ここが見落としがちなポイントです。OpenClawの本領は「常時稼働のパーソナルアシスタント」にあります。普段使いのノートPCで動かすと、スリープや持ち出しのたびにエージェントが止まってしまい、メッセージへの自動応答やスケジュール監視が機能しません。

そのため、実用的に使うなら**専用の常時稼働マシン**が事実上必要になります。代表的な選択肢は以下の3つです。

* **Mac mini（最も人気の選択肢）**：省電力で静音、macOS環境のためiMessage連携も可能。公式ドキュメントでも推奨ハードウェアとして「Mac Studio with M4 Max」が挙げられていますが、M2/M4のMac miniでも十分に動作します。daemonとして登録すれば、電源が入っている限り再起動後も自動起動します。
* **VPS / クラウドサーバー**：DigitalOceanにはOpenClawプリインストール済みのDropletがあり、2vCPU・4GB RAM以上のプランで動きます。ConoHa VPSでも対応済み。月額数千円で24時間稼働環境が手に入ります。
* **Raspberry Pi 5（8GBモデル）**：最も低コストな常時稼働オプション。ただし性能は限定的で、マルチエージェントや重い処理には不向きです。

「ちょっと試してみるだけ」なら手元のPCで十分ですが、本格運用するならMac miniかVPSの用意を前提に考えてください。

**必須アカウント**

**ランニングコスト**

OpenClaw自体はMITライセンスの無料ソフトウェアですが、AIの頭脳にあたるLLM APIの利用料が別途かかります。利用頻度やモデルによりますが、月額6〜200ドル以上が目安です。ローカルモデル（Ollama経由）を使えばAPI費用はゼロになりますが、マシンスペックとレスポンス品質のトレードオフがあります。

**スキルレベル**

OpenClawのメンテナー自身が「コマンドラインの使い方がわからないなら、このプロジェクトは危険すぎる」と警告しているほどです。非エンジニアの方がいきなり触るのはハードルが高い、という点は正直にお伝えしておきます。

## インストール手順（3ステップ）

### ステップ1：ワンライナーインストール

最も簡単な方法は、公式のワンライナーインストーラーです。ターミナルで以下を実行します。

```
# macOS / Linux
curl -fsSL https://openclaw.ai/install.sh | bash

# Windows
powershell -c "irm https://openclaw.ai/install.ps1 | iex"
```

Node.jsを含む依存関係を自動でインストールしてくれます。

npmから直接インストールする場合は以下の通りです。

```
npm install -g openclaw@latest
openclaw --version  # インストール確認
```

### ステップ2：初期セットアップ（オンボーディング）

インストール後、対話形式のセットアップウィザードを起動します。

```
openclaw onboard --install-daemon
```

ウィザードでは以下を設定します。

* **セットアップモード選択**：QuickStart（初心者向け）か Advanced（詳細設定）
* **AIプロバイダー選択**：Anthropic / OpenAI / Google / Ollamaなど
* **APIキー入力**：選択したプロバイダーのキーを入力
* **連携チャネル設定**：Discord、Telegram、LINEなどから選択
* **デーモン登録**：バックグラウンドサービスとして自動起動

初心者の方は「QuickStart」を選べば、最短でAIアシスタントが動く状態を作れます。

### ステップ3：動作確認

Gatewayが起動したら、ブラウザで http://localhost:18789 にアクセスすると、OpenClawのWebコントロールパネル（Dashboard）が開きます。ここからチャット、設定変更、トークン使用量の確認が可能です。

設定したメッセージアプリからOpenClawのボットに「こんにちは」と送信してみましょう。AIが応答すれば設定完了です。

## 日本語環境で使うためのポイント

OpenClawを日本語で快適に使うには、**SOUL.md**の設定が重要です。

SOUL.mdとは、エージェントの「人格定義ファイル」のようなもの。起動時に最初に読み込まれ、エージェントの振る舞いを決定します。

ワークスペースディレクトリ（デフォルトは ~/.openclaw/agents/main/）にSOUL.mdを作成し、以下のように日本語での応答を指示します。

```
# SOUL.md
あなたは日本語で応答するパーソナルアシスタントです。
ユーザーからの指示は日本語で受け取り、日本語で応答してください。
丁寧語を基本としつつ、簡潔にわかりやすく答えてください。
```

また、日本のユーザーにとっては**LINEとの連携**が特に便利です。LINEから直接AIに指示を送れるため、スマホだけでPCを遠隔操作するような体験が可能になります。

## スキル（Skills）で機能を拡張する

OpenClawの真価は「スキル」と呼ばれるプラグインシステムにあります。

```
# スキルを検索
openclaw skills search <キーワード>

# スキルをインストール
openclaw skills install @openclaw/github

# インストール済みスキル一覧
openclaw skills list
```

公式スキルだけでも、天気情報取得、Googleカレンダー連携、Gmail操作、GitHub管理など多数の拡張機能が利用可能です。

2026年3月からは**ClawHub**というスキルマーケットプレイスも本格稼働しており、コミュニティ製のスキルも豊富に揃っています。

ただし、**サードパーティ製スキルのインストールには細心の注意が必要です**（後述のセキュリティの章を必ずお読みください）。

## 実践的な活用シーン

OpenClawが特に力を発揮するユースケースをいくつか紹介します。

**毎朝のニュース要約**   
「毎朝7時に主要ニュースを5本要約してSlackに投稿して」と指示すれば、Heartbeat機能（能動的アクション）で自動実行されます。

**メール管理の自動化**   
Gmailと連携させれば、受信メールのトリアージ（分類）、定型返信の下書き作成、不要なメルマガの解除まで自動化可能です。

**開発ワークフローの効率化**   
GitHubと連携して、Issue管理やプルリクエストのレビュー補助、デバッグの自動化をこなします。

**スケジュール管理**   
Googleカレンダーと連携し、会議の自動スケジューリングやリマインダー送信を設定できます。

## 絶対に知っておくべきセキュリティリスク

ここからが最も重要な章です。OpenClawは非常に強力なツールですが、**その強力さゆえに深刻なセキュリティリスクを抱えています**。

### 実際に起きた事件

**① 暴走スパム事件（2026年2月）**   
あるエンジニアがiMessageへのアクセスを許可したところ、OpenClawが暴走。本人と妻に500通を超えるメッセージを送り、さらに連絡先リストの無関係な相手にもスパムを送信した事例がBloombergで報じられました。

**② 出会い系プロフィール自動作成事件（2026年2月）**   
ある学生のOpenClawエージェントが、指示なく勝手にMoltMatch（AIエージェント向け出会いプラットフォーム）にプロフィールを作成し、マッチング相手のスクリーニングまで始めていたことが判明しました。

**③ Infostealerマルウェアによる設定ファイル窃取（2026年2月）**   
サイバーセキュリティ企業Hudson Rockの報告によると、VidarというマルウェアがOpenClawの設定ファイル（ゲートウェイトークン、暗号鍵、人格定義）を丸ごと持ち去る事案が確認されています。

### 主要な脅威カテゴリ

* **プロンプトインジェクション攻撃**：悪意のあるメールやWebページに隠された指示をOpenClawが実行してしまう
* **サプライチェーン攻撃**：悪意のあるサードパーティ製スキルによるデータ流出（専門家の指摘では利用可能なプラグインの約20%にマルウェアが含まれる可能性）
* **ゲートウェイ露出**：設定ミスによりインターネットから直接アクセス可能な状態になるリスク

### 各方面からの警告

Cisco、Gartner、CrowdStrike、トレンドマイクロ、チェック・ポイント（Lakera）、ベルギーサイバーセキュリティセンター、中国政府、オランダデータ保護庁など、世界中のセキュリティ機関がOpenClawのリスクについて警告を発しています。

Meta社では業務用PCでのOpenClaw使用を禁止し、違反した場合は解雇リスクがあるとまで通達されています。

### 安全に使うための鉄則

OpenClawを使う場合は、以下の対策を必ず実施してください。

1. **隔離環境で運用する**：メインPCではなく、仮想環境・専用PC・VPSなどで動かす
2. **Sandbox Modeを有効にする**：危険なコマンド実行前に確認を求める設定にする
3. **最小権限の原則**：APIキーやアクセス権限は必要最小限に設定する
4. **スキルは信頼できるものだけ**：ClawHub公式スキルを優先し、サードパーティ製は必ずソースコードを確認する
5. **ゲートウェイをloopbackにバインド**：外部からのアクセスを遮断する
6. **設定ファイルのパーミッション管理**：~/.openclaw/ を700、設定ファイルを600に
7. **定期的なアップデート**：セキュリティパッチを含む最新バージョンに常に更新する

### NemoClaw（NVIDIA製セキュリティアドオン）

2026年3月、NVIDIAがOpenClaw向けのエンタープライズセキュリティアドオン「NemoClaw」をリリースしました。OpenShellサンドボックスによるエージェントアクションの隔離実行が目玉機能ですが、2026年3月時点ではアルファ版であり、本番環境への導入は時期尚早です。今後の成熟に期待しましょう。

## まとめ

OpenClawは、AIエージェント時代の到来を象徴するプロジェクトです。

* 従来のチャットボットとは異なり、**実際にPCを操作してタスクを実行する自律型AIアシスタント**

AIエージェントの世界は、まさに今、急速に動いています。OpenClawはその最前線にあるプロジェクトの一つですが、「使いこなす」には技術力とセキュリティ意識の両方が求められます。
