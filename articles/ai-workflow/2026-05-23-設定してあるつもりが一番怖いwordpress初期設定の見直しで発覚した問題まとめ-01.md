## 「設定した」と「動いている」は別物だった

リライト記事を投稿し始めた3月末、GA4を見たら楽天アフィリエイトのボタンが132回クリックされていた。

でも、GA4のCTAクリック計測には何も表示されていなかった。

以前にWPCodeでクリック計測のJavaScriptを入れたはずだった。「設定した」と思っていた。でも動いていなかった。

原因を調べたら、コードに指定したCSSクラス名が実際のボタンのクラス名と違っていた。コードは正しく書かれていた。でも対象のボタンを見つけられていなかったから、何も起きていなかった。

この一件で「設定した≠動いている」を痛感した。

## この記事で書いていること

## 😱 CTAクリック計測が動いていなかった

**原因：CSSクラス名が違った**

WPCodeに入れていたコードはこうだった。

javascript

```
document.querySelectorAll('a.btn-glow.btn.btn-shine')
```

でも実際のSWELLボタンのHTMLクラスは swell-block-button\_\_link だった。

コードは毎回「btn-glow.btn.btn-shineというボタンを探す」→「見つからない」→「何もしない」を繰り返していた。

**修正はシンプルだった。**

WPCodeのスニペットを開いて、1行だけ変えた。

javascript

```
// 修正前
document.querySelectorAll('a.btn-glow.btn.btn-shine')

// 修正後
document.querySelectorAll('a.swell-block-button__link')
```

修正後にスマートフォンのモバイル回線でボタンをクリックしたら、GA4のリアルタイムで cta\_click が表示された。動いた。

**GA4での確認場所：**

```
GA4管理画面
→ レポート → リアルタイム
→「過去30分のイベント」欄に「cta_click」が表示されればOK
```

動作確認はモバイル回線（Wi-Fiオフ）で行う。自宅のIPがGA4から除外設定されている場合、Wi-Fi経由だと自分のクリックが計測されない。

**同じテーマ（SWELL）を使っている人へ：** コードに書くクラス名は必ずデベロッパーツール（F12→「検証」）で実際のボタンのHTMLを確認してから指定する。思い込みで書くと、動いていないのに動いていると思い込む。

## 😱 ソフト404が23件溜まっていた

サーチコンソールで「ソフト404：23件」という警告が出ていた。

原因は記事を削除したときに残ったタグページだった。記事をリライトで削除すると、その記事についていたタグだけが残る。タグページに記事が0件になると、Googleが「存在しないページ」と判定してソフト404になる。

対処法はカウントが0のタグを削除するだけだ。

```
WordPress管理画面
→ 投稿 → タグ
→「カウント」列で昇順ソート
→ 0のタグをチェック → 一括操作 → 削除
```

ただし前の記事でも書いたが、**予約投稿中の記事のタグもカウントが0になる。**

全部消してしまうと、予約投稿の記事からタグが消える。削除前に予約投稿の記事を確認して、そのタグは残しておく必要がある。

自分はこれを確認せずに全部消してしまった。

**定期的に空タグを削除する習慣をつける。** リライトが進むほど古いタグが残り続けるので、月1回くらい確認するといい。

## 🤝 図解プロンプトをYAML風から自然文に変えた

リライト記事に入れている図解は、Geminiに生成させている。でもプロンプトをYAML風の入れ子構造で書いていたため、Geminiが誤解釈してレイアウトが崩れることが多かった。

**変更前（YAML風）：**

```
全体デザイン設定:
  トーン: "極小, 透明, 純粋, 繊細, モダン。"
  ビジュアル・アイデンティティ:
    背景色: "#FFFFFF (Pure White)"
```

**変更後（自然文＋箇条書き）：**

```
デザインは以下の通りにしてください。
- 背景は白（#FFFFFF）
- 文字は黒（#000000）
- アクセントカラーは○○（#******）
- 線のみで構成し、塗りつぶしは使わない
- 余白を広く取りシンプルに仕上げる
```

変更後、図解の生成精度が上がった。

Geminiへのプロンプトは人間が読むような自然な文章で書いた方が伝わりやすい。YAML風の構造は人間には読みやすいが、Geminiにとっては「フォーマットの解読」と「内容の理解」を同時にやらなければならない余計な負荷になる。

## 💡 今の段階でGA4に見るべきデータは3つだけ

GA4を入れたはいいが「何を見ればいいかわからない」という状態が続いていた。

Claudeに聞いたら「今の段階で見るべきデータは3つだけ」と言われた。

```
① どの記事が読まれているか（人気記事の把握）
  → Site Kitダッシュボード → Top content

② どこから来ているか（検索流入かどうか）
  → Site Kitダッシュボード → Traffic sources

③ CTAがクリックされているか（収益への導線確認）
  → GA4 → レポート → エンゲージメント → イベント → cta_click
```

まだアクセスがほぼゼロの段階では、毎日GA4を見ても意味がない。週1回、上の3つだけ確認すれば十分だと言われた。

数字が少ないうちは、数字が増えたときにすぐ気づける状態にしておくことが大事。

## 💡 AI運用で一番怖いのはエラーじゃなかった

この時期に気づいた一番大事なことを書いておく。

**AI運用で怖いのはエラーじゃない。「動いていると思い込むこと」だった。**

エラーが出れば気づける。でも「設定した、動いているはず」と思い込んでいると、何週間も何ヶ月も、実は動いていない状態が続く。CTAクリックが計測されていなかったのも、1ヶ月近く気づかなかった。

設定したら必ず動作確認する。定期的に「本当に動いているか」を見直す。これはAIに任せる部分が増えるほど、人間がやらなければいけないことだと思った。
