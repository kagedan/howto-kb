## Eval（Scorer）とは何か

システム開発の現場でソフトウェアを開発する場合、さまざまな「テスト」は欠かせません。テストケースを作り、実際にソフトウェアを動かしてみて、期待通りに振る舞うか検証します。ソフトウェアテストは基本的には「合格 / 不合格」を明確に判定できます。

一方でAIエージェントの場合は、同じ入力でも出力が揺らぐ、つまり非決定的であるため、単純な二値判定で品質を捉えにくいです。 AIエージェントを実装し始めた最初のうちは、手作業と目視で確認をすればよいのですが、規模が大きく複雑になってくると、システム全体を見渡してひとつの修正がどこにも悪影響を及ぼしていないことを確認しなければなりませんが、手作業と目視には限界があります。Mastraではこの課題に対して、**Scorer**（**スコアラー**）を使って品質を数値化します。これを**Eval**( **評価** )と呼びます。

評価は、**実運用をしながら**全量またはサンプリング的に評価し続ける、という **Observability**（**可観測性**）的な使われ方もしますし、ソフトウェアテストのように**開発時や変更時の検証**として**回帰的かつ自動的**に評価するような使われ方もします。

## 評価方法

Scorerは、次のような方法で出力を評価する自動テストです。

* **モデル判定**（**LLM Judge**）

  + 判定器がLLMです。
  + 「この回答は質問意図に沿っているか？」のような曖昧・主観的な基準を、評価用プロンプトで採点します。
  + 強みは柔軟性、弱みはコスト・揺らぎ（同条件で微差が出る可能性）。
* **ルールベース判定**

  + 判定器が固定ルール（if/else, 正規表現, JSONスキーマ, キーワード必須など）です。
  + 例: 必須項目が含まれるか、禁止語がないか、出力形式が厳密に一致するか。
  + 強みは再現性・高速性、弱みは複雑な意味判断に弱いこと。
* **統計的な判定**

  + 判定器が数値指標です。
  + 単純な「合ってる/間違い」ではなく、類似度・ランキング品質・分布傾向を数式でスコア化します。
  + 例:
    - 文字/語の一致率（類似度、差分）
    - 埋め込みベクトルのコサイン類似度
    - 検索品質指標（Precision/Recall、MAP など）
    - 複数テストケース全体の平均スコア・分散・通過率推移

Scorerの結果は通常 `0〜1` の数値スコアです。これにより次が可能になります。

* 品質を定量的に追跡する
* 手法同士を比較する
* 改善ポイントを見つける

また、Scorerはクラウド上のリアルタイム評価にも、CI/CDでの継続評価にも使えます。

## 評価観点

評価の観点は、いわゆるソフトウェアテストのような「〇〇という出力をしたかどうか」だけではありません。

質問の意図にかみ合った回答をしているか、依頼事項にすべて答えているか、指定された回答様式を守っているか、攻撃・ヘイト・脅し等の有害要素がないか、バイアスのかかった回答をしていないか、など様々な観点での評価が可能であり、それらに対応した専門のスコアラーが用意されます。詳細は次回ご紹介します。

# ライブ評価（Live evaluations）

ライブ評価は、エージェントやワークフローの実行中に**非同期で自動採点**する仕組みです。  
手動バッチ実行を待たずに、運用しながら継続監視できます。

## 採点結果の保存先と確認方法

採点結果は、Mastraの**storage設定先DB**に保存されます（テーブル: `mastra_scorers`）。  
確認方法は主に次の3つです。

* Studioで確認する（`mastra dev` 実行後、Observability/Scorers系の画面）
* DBを直接参照する（`mastra_scorers` をSQLで確認）
* CI評価は `runEvals` の戻り値（`scores`, `summary`）で確認する

> 重要: DB保存先は **Agent定義ではなく、Mastra初期化時の `storage`** で定義します。

## DB保存先の定義（Mastra側）

`index.ts` で `new Mastra()` する際に、storage定義をすることで、そこの`mastra_scorers`テーブルに評価結果が保存されます。

```
import { Mastra } from "@mastra/core/mastra";
import { LibSQLStore } from "@mastra/libsql";

export const mastra = new Mastra({
  // Agent/Workflow定義は省略
  storage: new LibSQLStore({
    id: "mastra-storage",
    url: "file:.mastra/mastra.db", // このDBにmastra_scorers等が保存される
  }),
});
```

## AgentにScorerを付ける

エージェントで、`new Agent()` した際に、scorers属性でスコアラーを複数定義できます。

下のサンプルコードでは、scorers 属性の中で２つのスコアラーを定義しています。この中で `relevancy` とか `safety` という名前が定義されていますが、ここは特に決まりは無く、つまり固定予約語ではないので、開発者・運用者にとって分かりやすい名前であれば何でもよいです（とはいっても、同一オブジェクトの中で一意である必要はありますし、後から名前を頻繁に変更すると比較や追跡がしづらくなるので、命名規約を決めて固定化することを推奨します）。

スコアラーそのものは、いずれもMastraがあらかじめ用意している組み込みスコアラーを利用しています。

サンプリング中するする割合が、ひとつめのスコアラーは0.5、もうひとつは1となっています。

```
import { Agent } from "@mastra/core/agent";
import {
  createAnswerRelevancyScorer,
  createToxicityScorer,
} from "@mastra/evals/scorers/prebuilt";

export const evaluatedAgent = new Agent({
  id: "test-agent",
  name: "Test Agent",
  instructions: "You are a helpful assistant.",
  model: "openai/gpt-5.1",
  scorers: {
    relevancy: {
      scorer: createAnswerRelevancyScorer({ model: "openai/gpt-4.1-nano" }),
      sampling: { type: "ratio", rate: 0.5 },
    },
    safety: {
      scorer: createToxicityScorer({ model: "openai/gpt-4.1-nano" }),
      sampling: { type: "ratio", rate: 1 },
    },
  },
});
```

## WorkflowのStepにScorerを付ける

ワークフローのステップの中に評価を定義することも可能です。定義の仕方は先ほどのエージェントの場合とほぼ同じです。

```
import { createWorkflow, createStep } from "@mastra/core/workflows";
import { customStepScorer } from "../scorers/custom-step-scorer";

const contentStep = createStep({
  scorers: {
    customStepScorer: {
      scorer: customStepScorer(),
      sampling: {
        type: "ratio",
        rate: 1,
      },
    },
  },
});

export const contentWorkflow = createWorkflow({ /* ... */ })
  .then(contentStep)
  .commit();
```

# トレース評価（Trace evaluations）

ライブ評価だけでなく、過去の実行トレースに対して後からScorerをかけることもできます。  
用途は、過去分析・不具合調査・一括評価です。

> トレース採点にはObservability（トレース収集設定）が必要です。

## Studioでトレース評価する流れ

1. MastraインスタンスにScorerを登録
2. StudioのObservabilityから対象トレースを選んで実行

```
const mastra = new Mastra({
  scorers: {
    answerRelevancy: myAnswerRelevancyScorer,
    responseQuality: myResponseQualityScorer,
  },
});
```
