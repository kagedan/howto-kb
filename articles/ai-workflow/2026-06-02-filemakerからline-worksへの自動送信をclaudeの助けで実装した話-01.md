今回は、FileMakerからLINE WORKSのグループチャネルにメッセージや画像を自動送信できるようにした話を書きます。タイトルにある通り、実装の心強い相棒はClaudeでした。

---

## LINE Notifyが終了して、手動コピペ運用が始まった

もともとチームへの通知には LINE Notify を使っていました。FileMakerで作った受注データや日次レポートを、LINE Notifyを経由して自動送信する仕組みです。それなりに便利に使っていたのですが、2025年3月にサービスが終了してしまいました。

代替手段として最初に試したのは、LINE WORKSのBot APIです。これを使えばプログラムから直接送れる、というのはわかりました。ただ、設定しようとすると意味がわからなすぎて全く進みません。当時はChatGPTを使いながら試行錯誤してみたのですが、それでも解決できずに断念しました。

結局、「FileMakerのテキストをコピーしてLINE WORKSに手動で貼り付けて送る」という運用に落ち着きました。毎回FileMakerからコピーして、LINEを開いて、貼り付けて送信する。単純作業なのに地味に時間を取られますし、送り忘れも起きます。自動化したいとは思いつつ、一度諦めた状態が続いていました。

---

## ChatGPTからClaudeへ。きっかけはこの案件だった

当時、AI活用といえば上記の通りChatGPTを使っていました。ただ、ちょうどそのころClaudeの評判をよく見かけるようになって、「試してみようか」という気持ちが芽生えていました。

そこでClaudeに相談してみたのが、この案件です。「FileMakerからLINE WORKSにメッセージを送りたい。Bot APIを使いたいがAPIの仕組みがよくわからない」という状況をそのまま伝えました。

返ってきた回答が具体的で、実際に動くコードを出してくれたんです。JWT認証の実装から、Pythonスクリプトの全体像まで。本件の解決がきっかけでChatGPTからClaudeに移行（課金）することにしました（現在は両方課金していますが）。

---

## 仕組みの全体像

完成したシステムの構成はシンプルです。

* FileMaker Pro のボタンをクリックする
* FileMaker スクリプトが Python スクリプトを呼び出す
* Python スクリプトが LINE WORKS Bot API を叩いてメッセージ・画像を送信する

FileMaker には外部プログラムを呼び出す「イベント送信」という機能があり、これを使ってコマンドラインからPythonを実行しています。

なお、「なぜPythonを経由するのか」と思われるかもしれません。FileMakerには直接HTTP通信できる機能もあるのですが、LINE WORKSのBot APIで必要なJWT認証の生成がFileMaker単体では難しく、Pythonを噛ませるのが現実的な選択でした。

![](https://assets.st-note.com/img/1780360974-Ar6NsQqJicV8Rj419GtEY0zd.png?width=1200)

認証には **JWT（JSON Web Token）方式** を使っています。LINE WORKSのDeveloper ConsoleでService Accountを作成し、その秘密鍵を使ってトークンを生成、APIを呼び出す流れです。この部分の実装もClaudeに任せたので、仕組みはあまり理解していません笑。コードは丸ごと書いてもらいました。

## ハマり続けた「400エラー」

テキストメッセージの送信は比較的早い段階でうまくいきました。問題は**画像送信**です。

断続的に格闘が続きました。エラーはこれです。

```
ERROR: 400 - {"code":"INVALID_PARAMETER","description":"Request body cannot be parsed."}
```

試したことを並べると、HTTPメソッドをPUTにしたりPOSTにしたり、Content-Typeをあれこれ変えたり、エンドポイントを複数試したり、パラメータ名を推測で変えたり……。試した組み合わせは10パターン以上になりました。全部ダメでした。

解決したのは、**LINE WORKSの公式ドキュメントをちゃんと読んだとき**です。「ファイルアップロード / ダウンロード」という項目に、画像送信には2ステップが必要なことが明記されていました。

さらに、フォームのフィールド名が Filedata でないといけないという仕様まで決まっていました。この1点が全ての答えでした。

ドキュメントを読めば書いてあったのですが、「どのセクションを読めばよいか」がわからなかったんですよね。ここもClaudeに「どのドキュメントを確認すればいいか」と聞きながら進めていきました。

## FileMakerスクリプトの実装例

実際にFileMaker側でどう呼び出しているかというと、こんなイメージです（パス等は伏せています）。

```
変数を設定 [ $channelId ; 値: "グループチャネルのID" ]
変数を設定 [ $message ; 値: "【受注報告】" & ¶ & "新規受注が発生しました" ]
変数を設定 [ $command ; 値:
  "cmd /c py \"スクリプトのパス\" " &
  Quote($channelId) & " channel " &
  Quote($message)
]
イベント送信 [ [aevt] ; [odoc] ; $command ]
```

FileMakerの改行記号 ¶ はそのままではLINE WORKSで改行として認識されないので、Pythonスクリプト側で \n に自動変換する処理も入れています。これもハマりポイントのひとつでした。

## 使ってみて

今は受注データが更新されたタイミングでボタンを押すだけで、テキストと画像がLINE WORKSのグループチャネルに飛んでいきます。貼り付け作業がなくなりました。

地味な変化ではありますが、毎日の積み重ねを考えると効いています。送り忘れもなくなりましたし、画像も一緒に送れるので情報量も上がりました。

ひとつ正直に書くと、稀に画像送信が失敗するケースがあります。これはPythonを経由することの課題で、Python側の処理が遅延してFileMaker側が先に動いてしまうタイミングの問題かと思っています。FileMaker単体でJWT認証を生成する手段がないためPythonを噛ませているのですが、その構造上避けにくい部分でもあります。ただ、画像はファイルサーバーに自動保存される仕組みにしているので、そのときは手動で送るだけです。頻度は低いですし、フォールバックがあるので実用上は困っていません。
