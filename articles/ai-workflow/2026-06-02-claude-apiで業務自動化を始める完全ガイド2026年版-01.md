### レシピ① メール文面の自動生成

毎日同じようなメールを書いていませんか？テンプレート＋AIで一瞬に。

```
import anthropic

client = anthropic.Anthropic()

def generate_email(purpose: str, recipient: str, key_points: list[str]) -> str:
    points_text = "\n".join(f"- {p}" for p in key_points)
    
    message = client.messages.create(
        model="claude-opus-4-8",
        max_tokens=500,
        messages=[{
            "role": "user",
            "content": f"""以下の情報をもとに、ビジネスメールを作成してください。

目的: {purpose}
宛先: {recipient}
伝えたいポイント:
{points_text}

条件:
- 丁寧だが冗長にならない
- 件名も含める
- 200字以内"""
        }]
    )
    return message.content[0].text

# 使用例
email = generate_email(
    purpose="会議の日程調整",
    recipient="営業部 田中様",
    key_points=["来週中に30分", "オンライン希望", "Zoom URLは後送"]
)
print(email)
```

**出力例:**

```
件名：打ち合わせ日程のご相談

田中様

お世話になっております。
来週中に30分ほどお時間をいただけますでしょうか。
オンライン（Zoom）でのご面談を希望しております。
URLは日程確定後にお送りいたします。

ご都合の良い日時をお知らせいただけますと幸いです。
よろしくお願いいたします。
```

### レシピ② 議事録の自動整理

音声文字起こし（WhisperやNotta等）のテキストをそのままClaude APIに投げるだけ。

```
def summarize_meeting(transcript: str) -> dict:
    message = client.messages.create(
        model="claude-opus-4-8",
        max_tokens=800,
        messages=[{
            "role": "user",
            "content": f"""以下の会議文字起こしを整理してください。

---
{transcript}
---

以下の形式でJSON出力してください：
{{
  "summary": "会議の概要（3行以内）",
  "decisions": ["決定事項1", "決定事項2"],
  "actions": [
    {{"task": "タスク内容", "owner": "担当者", "deadline": "期限"}}
  ],
  "next_meeting": "次回会議の予定（不明な場合はnull）"
}}"""
        }]
    )
    
    import json
    return json.loads(message.content[0].text)

# 使用例
transcript = """
田中：では始めましょう。今日は新機能のリリース日程について。
佐藤：開発は来週末に終わる予定です。
田中：じゃあテストを翌週月曜から木曜でお願いできますか？
佐藤：了解です。
田中：リリースは再来週の月曜日で。告知は田中が金曜までに作成します。
"""

result = summarize_meeting(transcript)
print(result)
```

### レシピ③ 長文ドキュメントの要約（プロンプトキャッシュ活用）

PDFや長いドキュメントを処理するときはプロンプトキャッシュを使うとコストが最大90%削減できます。

```
def summarize_document(document: str, questions: list[str]) -> list[str]:
    """同じドキュメントに複数の質問をする場合、キャッシュが効いてコスト削減"""
    answers = []
    
    for question in questions:
        message = client.messages.create(
            model="claude-opus-4-8",
            max_tokens=500,
            system=[
                {
                    "type": "text",
                    "text": "あなたはドキュメント分析の専門家です。",
                },
                {
                    "type": "text",
                    "text": document,
                    "cache_control": {"type": "ephemeral"}  # ここがキャッシュの肝
                }
            ],
            messages=[{
                "role": "user",
                "content": question
            }]
        )
        answers.append(message.content[0].text)
    
    return answers

# 100ページのPDFテキストに5つの質問をする場合、
# 2問目以降のドキュメント読み込みコストが90%削減される
```

### レシピ⑤ Slack連携で自動通知ボット

定期的にデータを集計してSlackに報告するボットをGitHub Actionsで動かす構成。

```python
# bot.py
import anthropic
import requests
import os

client = anthropic.Anthropic()

def generate_daily_report(data: dict) -> str:
    message = client.messages.create(
        model="claude-haiku-4-5-20251001",  # 定型レポートはHaikuで十分＆安い
        max_tokens=300,
        messages=[{
            "role": "user",
            "content": f"""以下のデータから日次レポートを作成してください（Slack向け絵文字OK）。

{data}

形式：
- 1行目：タイトル
- 箇条書き3〜5個
- 最後に一言コメント"""
        }]
    )
    return message.content[0].text

def post_to_slack(message: str, webhook_url: str):
    requests.post(webhook_url, json={"text": message})

# GitHub Actionsのcronで毎朝9時に実行
if __name__ == "__main__":
    data = {
        "昨日の売上": "¥450,000",
        "新規問い合わせ": 12,
        "未対応チケット": 3,
        "目標達成率": "87%"
    }
    
    report = generate_daily_report(data)
    post_to_slack(report, os.environ["SLACK_WEBHOOK_URL"])
```

```
# .github/workflows/daily-report.yml
name: Daily Report
on:
  schedule:
    - cron: '0 0 * * 1-5'  # 平日9時（UTC 0時 = JST 9時）
jobs:
  report:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'
      - run: pip install anthropic requests
      - run: python bot.py
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          SLACK_WEBHOOK_URL: ${{ secrets.SLACK_WEBHOOK_URL }}
```

## コストを抑える3つのコツ

| コツ | 効果 | 実装方法 |
| --- | --- | --- |
| モデルを使い分ける | 最大10分の1のコスト | 定型タスクはHaiku、創造的タスクはOpus |
| プロンプトキャッシュ | 最大90%削減 | `cache_control: ephemeral` を設定 |
| max\_tokensを適切に設定 | 無駄なトークン削減 | 想定出力の1.2倍程度に設定 |

## エラー処理の実装

本番運用では必ずリトライを実装します。

```
import time
import anthropic
from anthropic import RateLimitError, APIStatusError

client = anthropic.Anthropic()

def call_with_retry(messages: list, max_retries: int = 3) -> str:
    for attempt in range(max_retries):
        try:
            message = client.messages.create(
                model="claude-opus-4-8",
                max_tokens=1024,
                messages=messages
            )
            return message.content[0].text
            
        except RateLimitError:
            wait_time = 2 ** attempt  # 指数バックオフ: 1秒, 2秒, 4秒
            print(f"レートリミット。{wait_time}秒後にリトライ...")
            time.sleep(wait_time)
            
        except APIStatusError as e:
            if e.status_code >= 500:  # サーバーエラーはリトライ
                time.sleep(2 ** attempt)
            else:
                raise  # 4xxエラーはリトライしない
    
    raise Exception(f"{max_retries}回リトライしましたが失敗しました")
```

## まとめ

Claude APIで自動化できる業務の共通パターンは：

1. **繰り返し書く文章** → テンプレート＋API生成
2. **読んで整理する作業** → 要約・構造化API
3. **データを解釈する作業** → 分析レポートAPI
4. **定期的な報告** → GitHub Actions＋API

最初の一歩は小さく。まず手元の「毎日5分かかる作業」を1つ自動化してみてください。
