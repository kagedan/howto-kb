## 4. AGENTS.mdの作成

Gemini CLIと同様に仮想環境の扱いやMAPDLインスタンスの取り扱いで苦労していたので、次から迷わないようにAGENTS.mdを作成しておきます。以下のようなプロンプトで作ってくれます。

```
このセッションで得られた教訓から、プロジェクトに関する追加の指示と背景情報をAGENTS.mdとして提供したい。内容を考えて。
```

Codex回答：

```
AGENTS.md を作成しました。

内容は、このセッションで分かった以下を中心にまとめています。

.venv を必ず使うこと
.venv の Python がサンドボックス内で起動できない場合の扱い
PyMAPDL / MAPDL の既定ポート 50052 と残プロセス対策
MAPDL VERIFICATION RUN ONLY や PyVista warning の報告方針
解析結果として何を確認・報告するべきか
mapdl_work/3d_plate_thermal.py に関する既知情報
```

作られた内容を見ると個別の解析に踏み込んでいるようだったので、システム関連に絞るように依頼。また、MAPDLインスタンスは先に立ち上げておいて、Python実行時は接続するだけという運用が可能なので、それも記載してもらいます。

```
今後、このプロジェクトでは様々な解析を行う予定であるため、システム関連（venvの使い方やMAPDLインスタンスの取り扱い）に内容を絞って。
PyMAPDLのドキュメントを見るとMAPDLのインスタンスを先に起動しておき、Python実行時にはそのインスタンスに接続するという使用方法があるので、確認して記載すること。接続時に内容をクリアしてから実行するように。
```

出来上がったAGENTS.md：

> # AGENTS.md
>
> このプロジェクトでは、さまざまな MAPDL 解析を行う。
>
> ## Python 仮想環境
>
> * Python 実行には、必ずプロジェクト直下の `.venv` を使う。
> * Windows では `.venv\Scripts\python.exe` を明示して実行する。
> * `.venv` の Python がサンドボックス内から起動できない場合がある。これは仮想環境>がユーザー環境側のベース Python を参照しているためで、必要ならユーザー承認を得て>サンドボックス外で実行する。
> * 依存関係を勝手にインストール、削除、アップグレードしない。必要な場合は、目的と影>響を説明してから行う。
>
> ## 推奨する MAPDL 運用
>
> * 可能なら、MAPDL インスタンスは Python 実行前にあらかじめ起動しておき、Python >側はその既存インスタンスへ接続する。
> * PyMAPDL 公式ドキュメントでは、MAPDL を gRPC モードで起動してから接続する運用が>案内されている。
> * Windows の起動例:
>
> ```
> "C:\Program Files\ANSYS Inc\v241\ansys\bin\winx64\ANSYS241.exe" -grpc
> ```
>
> ```
> "C:\Program Files\ANSYS Inc\v241\ansys\bin\winx64\ANSYS241.exe" -port >50052 -grpc
> ```
>
> * Python 側では、既存インスタンスへ接続するために `start_instance=False` を使>う。
> * 接続時は、前回の解析状態が残らないよう `clear_on_connect=True` を明示する。
>
> ```
> from ansys.mapdl.core import launch_mapdl
>
> mapdl = launch_mapdl(
>    start_instance=False,
>    ip="127.0.0.1",
>    port=50052,
>    clear_on_connect=True,
> )
> ```
>
> * スクリプト内でも、解析開始前に明示的に状態をクリアする。
>
> ```
> mapdl.finish()
> mapdl.clear()
> ```
>
> ## ポートとプロセス管理
>
> * PyMAPDL / MAPDL gRPC の既定ポートは `50052`。
> * `Port 50052 is already in use` が出た場合は、既存 MAPDL インスタンスが残って>いる可能性が高い。
> * 別の解析やユーザー作業中の MAPDL を誤って止めない。プロセス終了が必要な場合は、>対象プロセスの開始時刻や今回の作業で起動したものかを確認する。
> * 確認対象になりやすいプロセス名は `ANSYS`, `ANSYS261`, `ansyscl` など。
> * 別ポートで起動・接続する場合は、MAPDL 起動時と Python 接続時のポートを必ず一致>させる。
>
> ## Python から MAPDL を起動する場合
>
> * 既存インスタンス接続が使えない場合のみ、Python から `launch_mapdl()` で >MAPDL を起動する。
> * Python から起動した場合は、正常終了時に `mapdl.exit()` を呼ぶ。
> * 失敗、例外、タイムアウトが起きた場合は、MAPDL プロセスが残っていないか確認す>る。
> * 長い解析や起動待ちが必要な解析では、コマンドのタイムアウトを十分長く設定する。
>
