いつも自分は複数処理を同時に走らせる場合、ターミナルを複数起動して  
claudecodeを起動、もしくはcmuxを利用していた。

でもこれ毎回毎回ポチポチするのめんどくさいし、そういえばバックグラウンドで走らせること  
できるよなって思ったので、やり方を備忘録として残す。

## エージェントを複数起動させ並行処理するシチュエーション

これは自分の日々のタスクにおけるケースとする。

* Fan-out型（同じ処理を対象違いで一斉に行う）
  + 複数サイトのスクレイピングを同時で行いたい時。
  + 複数KWのSerpsを調査したいとき
* 探索型(1つのことに対し、異なる戦略・プロンプトのエージェントを走らせてベストを採る)
  + 同じリサーチ問いを Grok / DeepSeek / Claude に投げて突き合わせ
  + トレード戦略のバックテストを、パラメータ違い（T/SL/TPのグリッド）で並列探索
* パイプライン型（役割分担して流す）
  + コード生成エージェントとレビューエージェントを分けて、コード生成中に生成済みコードのレビューをさせる
* Interrupt/割り込み型（監督者＋実行者の分離）
  + 学習計画に基づき、学習を進めていたが、途中で変更が生じるとき。（学習はしつつ、将来の設計部分を変更しておきたい）

## ClaudeCodeでエージェントをバックグラウンドで複数走らせる方法

方法としては「バックグラウンド Bash型」「並列サブエージェント」の2つ。  
それぞれ詳細を説明する。

Bashにするか、サブエージェントを起動するかの判断方法は、  
「実行するだけか / 考えさせたいか」で選べばOK。以下に違いを記載する。

|  | バックグラウンド Bash | 並列サブエージェント |
| --- | --- | --- |
| 動くもの | 1個のコマンド | 複数のミニClaude（AI） |
| 頭を使うか | ❌ 使わない（機械的に実行するだけ） | ✅ 使う（考えて・調べて・判断する） |
| 何個か | 基本1本の処理 | 何個も同時に |
| たとえ | コピー機を回しておく | 部下を複数人、別々の仕事に送り出す |
| 使う場面 | dev server起動・テスト・ビルド・スクレイピング | 複数社の調査比較・記事の並列ファクトチェック |
| 一言で | 決まった処理を裏で走らせる | 考える作業を複数人に同時に任せる |

### バックグラウンド Bash (1本の長い処理に使う)

ビルド・テスト・dev server・スクレイピングなど、時間のかかるコマンドを裏で走らせます。

* 複数を同時起動でき、それぞれ終了したタイミングで通知が届く
* ポーリング不要 ：終わると自動でその結果を報告できる

バックグラウンド Bash ＝ ただの「コマンドを裏で走らせる」  
npm run dev とか npm test みたいな、決まった処理を実行して結果を吐くだけ。  
考えたり判断したりはしません。ボタンを押したら動く機械。

【具体例】  
・dev server を起動しておく  
・テストを回す  
・ビルドする  
・スクレイピングスクリプトを走らせる  
→ 決まったコマンドを流すだけ。判断は不要。

```
頼み方の例：
「npm run dev をバックグラウンドで起動して」
「テストを裏で回して、終わったら教えて」
```

### 並列サブエージェント(複数処理を同時に)

「複数処理を並列で」の本命。独立した作業を1メッセージでまとめて投げると並列で走り、  
各エージェントが終わると結果だけ返ります。

並列サブエージェント ＝ 「Claudeを複数人、同時に働かせる」  
各自が自分でファイルを読んで、考えて、ツールを使って、判断して作業する。  
あなたの代わりに動く"アシスタントが3人いる"イメージ。

【具体例】  
・A社・B社・C社をそれぞれ調べて比較して  
・3つの記事を並列でファクトチェックして  
→ 各自が読んで考えて成果物を作る。だから並列で人数を増やすと速い。

```
頼み方の例：
「A・B・C の3ファイルをそれぞれ別エージェントで並列に調査して」
```

## FleetViewの見方

バックグラウンド・タスクの管制塔。裏に流した処理が、状態ごとに溜まっていく。  
claudecode起動後に、キーボードで「←キー」を押すと表示される。

![](https://static.zenn.studio/user-upload/56ad3ee7710d-20260707.png)

| セクション | 意味 |
| --- | --- |
| Needs input | 裏の処理があなたの返事待ちで止まっている（承認・選択・確認が必要）。行の右に「いま何を聞かれているか」が出る |
| Completed | 完了済み。右側に成果の要約。開いて結果を確認できる |
| Running | 実行中のものがあればここに出る |

### Completedからセッションを削除する方法

削除したいセッションにカーソルを合わせる、その後入力欄の下に  
コマンドがヒント表示されるので、ヒントに合わせて入力。

windowsの場合は「ctrl + x」を2回押すことで削除が可能。  
まぁ削除しなくてもいいらしい。トークンが食うことはないそうだ。

むしろNeedsInputで止まっているものを消化したほうがいいらしい。

## Workflow の使い方

複数エージェントを決定論的（ループ・分岐・fan-out）にオーケストレーションする仕組み。JavaScript でスクリプトを書き、裏で実行され、完了時に通知が返ります。

### いつ使う？

* 大量ファイルの一括処理・移行・監査
* 「調査 → 設計 → レビュー」の段階的パイプライン
* 数観点で並列に調べて統合したいとき

### 実行の流れ

1. 「ワークフローで◯◯して」と依頼する
2. Claudeがスクリプトを組んで起動
3. 即タスクIDを返し裏で走る
4. /workflowsでライブ進捗を確認
5. 完了時に通知が届き、結果を報告
