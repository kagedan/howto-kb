「この数字、何を意味しているんだろう…」

Excelの画面を眺めながら、  
そう思ったことはありませんか？

売上の推移、コストの変化、  
アクセス数の増減。

数字は揃っている。  
でも、そこから何を読み取ればいいか  
わからない。

「次の会議で何を報告すれば…」  
「上司に提案できるネタが見つからない」  
「分析に時間がかかって本業が進まない」

こうした悩みを抱えながら、  
毎週レポートを作り続けている人は  
決して少なくありません。

分析が苦手な理由は、  
センスでも経験でも能力でもありません。

**正しい問いの立て方を知らないだけです。**

---

✅ **時間を大幅に短縮できる**  
　→レポート作成が半分以下の時間で  
　　終わるようになる

✅ **見落としのリスクを減らせる**  
　→AIが客観的な視点で  
　　重要なトレンドを拾ってくれる

✅ **会議での発言力が上がる**  
　→「次の一手」まで示せる提案が  
　　自然とできるようになる

---

AIは「データの翻訳者」です。

数字の羅列を、  
人間が判断できる言葉に変えてくれる。

今日から、あなたのレポート作成は  
根本から変わります。

## 基本のプロンプト

まず、例え話からはじめます。

AIにデータ分析をさせるとは、  
どういうことか。

---

**お天気予報に例えてみます。**

気象予報士は、  
大量の温度・湿度・風速データを見て  
「明日は雨が降りそう」と教えてくれます。

私たちは数字の意味を理解しなくても、  
「傘を持っていこう」と  
行動を決められますよね。

AIによるデータ分析も  
それと同じです。

数字を渡すと、  
「このトレンドは注意が必要です」  
「ここにチャンスがあります」と  
言葉で教えてくれる。

だから私たちは、  
「次に何をすべきか」に  
集中できるのです。

### ◆データ傾向を言語化するプロンプト

**✅目的：**  
　数字の羅列から  
　重要なトレンドと課題を  
　自動で言葉にする。

**✅いつ使うか：**  
　週次・月次レポートを  
　作る前の「整理ステップ」として。

```
# ロール設定
あなたはビジネスデータの分析専門家です。
提供されたデータから
重要なトレンド・課題・機会を
わかりやすく言語化してください。

# 背景・目的・対象
・データの種類：[例：月別売上データ]
・分析の目的：[例：次月の施策を検討したい]
・報告相手：[例：上司・経営会議]
・業界・事業内容：[例：BtoBのSaaS企業]

# 指示内容
以下の4点を整理してください。

1. 全体の傾向を3点で要約（ポジティブ・ネガティブ両面）
2. 特に注目すべき変化とその理由（仮説でよい）
3. このデータが示す「次に打つべき一手」（2〜3案）
4. 追加で確認すべきデータがあれば指摘する

# 制約条件
・専門用語は使わず、平易な日本語で書く
・数値は必ず引用して根拠を示す
・「おそらく」「可能性がある」など
　不確かな部分は明示する

# 出力形式
見出しつきの箇条書き。
最後に「1行の結論サマリー」を追加する。

# 入力データ
データ内容：[ここに数字・表・テキストを貼り付ける]
```

**✅具体的な使用方法：**  
　「入力データ」の欄に  
　ExcelやGoogleスプレッドシートの  
　データをコピペするだけ。

　表形式でも、  
　数字の羅列でも対応できます。

**✅読者への一言：**  
　難しい分析の前に、  
　まずこのプロンプトで  
　「何が起きているか」を把握する。

　それだけで、  
　レポートの質が一段上がります。

### ◆Excel作業を丸ごと依頼するプロンプト

**✅目的：**  
　「どんな関数を使えばいいか」  
　「グラフはどう設定するか」を  
　プロンプト一つで解決する。

**✅いつ使うか：**  
　Excelで「やりたいことはあるけど  
　やり方がわからない」と  
　詰まったとき、すぐに使う。

```
# ロール設定
あなたはExcel・スプレッドシートの
エキスパートです。
初心者でも手順通りに操作できるよう、
具体的なステップで教えてください。

# 背景・目的・対象
・使用ツール：[例：Excel / Googleスプレッドシート]
・やりたいこと：[例：月別売上を自動集計して
　前月比を出したい]
・現在のデータ構成：
　[例：A列に日付、B列に売上金額が入っている]
・スキルレベル：[例：関数はSUMくらいしか知らない]

# 指示内容
1. 目的を達成するための
　具体的な操作手順をステップ形式で
2. 使う関数・機能の名前と
　かんたんな説明
3. コピペできるサンプル数式（あれば）
4. よくあるミスと対処法

# 制約条件
・手順は1ステップずつ番号で書く
・専門用語には
　必ずかっこで補足説明を入れる
・初心者がつまずくポイントを
　先回りして補足する

# 出力形式
ステップ形式＋サンプル数式をコードブロックで。

# 入力データ
やりたいこと詳細：[ここに具体的に入力]
データの構成：[ここに列の説明などを入力]
```

**✅具体的な使用方法：**  
　「やりたいこと」と  
　「今のデータの構成」を入力するだけ。

　関数の名前を知らなくても、  
　「こんな計算がしたい」と書けば  
　AIが適切な方法を選んでくれます。

**✅読者への一言：**  
　「Excelが苦手」という人は、  
　実は「Excelの聞き方」を  
　知らないだけかもしれません。

　このプロンプトで、  
　その壁を超えましょう。

**同じデータを渡しても、  
プロンプトの作り方で

```
この売上データを分析してください。
```

**AIの回答（例）**：

> 先月は全体的に売上が増加しています。  
> 特定の商品が好調のようです。  
> 引き続き注視していくことをお勧めします。

---

👉 **どこが足りないか？**

---

### After（本記事のプロンプトを使用）

```
# ロール設定
あなたはビジネスデータの分析専門家です。

# 背景・目的・対象
・データ：3月の商品別売上（全5品目）
・目的：月次会議でのアクション提案
・報告相手：営業部マネージャー
・事業内容：食品メーカーの直販EC

# 指示内容
重要なトレンドと次の一手を整理して
```

**AIの回答（例）**：

> **【全体トレンド】**  
>  ・商品Aが前月比+32%（3月に突出）  
>  ・商品Cが2ヶ月連続で-15%の下落傾向  
>  ・全体売上は+8%だが、  
> 　A依存のリスクあり  
>   
> **【次に打つべき一手（案）】**  
>  ①商品Aの在庫確保と販促の継続  
>  ②商品Cの価格見直しまたは訴求変更  
>  ③商品AとCのセット割引でリカバリー  
>   
> **【結論サマリー】**  
>  好調なAを活かしつつ、  
>  Cのテコ入れが今月の最優先課題。

「ロール設定」で

情報を渡すほど、  
AIの精度は上がります。

## 実践プロンプト集

## 1. 複数期間を比較して変化を可視化する

データ分析で一番難しいのは、  
「どこを比べればいいか」を  
決めることです。

今月だけ見ても意味がない。  
でも、どの期間と比べるべきか。

AIに「比較の切り口」ごと  
考えてもらいましょう。

---

### ◆多角的比較分析プロンプト

**✅目的：**  
　複数の期間・カテゴリーを  
　比較して、意味のある変化を  
　自動で見つける。

**✅いつ使うか：**  
　月次・四半期レポートを  
　作る前の「比較設計」ステップとして。

```
# ロール設定
あなたはビジネスの多角的比較分析を
専門とするデータアナリストです。
比較の切り口を複数持ち、
重要な変化を見落とさないよう
整理してください。

# 背景・目的・対象
・データの内容：[例：過去1年間の月別売上]
・業種・商材：[例：アパレルのEC]
・知りたいこと：
　[例：季節性の影響と、
　それ以外の要因を分けて理解したい]

# 指示内容
以下の比較を行い、結果を整理してください。

1. 前月比（直近の変化を把握）
2. 前年同月比（季節性を除外した比較）
3. 目標値との乖離（あれば）
4. 上記から読み取れる
　「構造的な課題」と「一時的な変動」の区別

# 制約条件
・数値は必ず引用して根拠を示す
・季節要因・外部要因は仮説として明示
・判断のために追加で
　必要なデータがあれば指摘する

# 出力形式
比較ごとに見出しをつけ、
最後に「1行の総括」を加える。

# 入力データ
データ：[ここに数値・表を貼り付け]
目標値（わかれば）：[ここに入力]
```

**✅具体的な使用方法：**  
　「前月比」と「前年同月比」を  
　両方出すことで、  
　「今月だけ下がった一時的な落ち込み」と  
　「ずっと下がっている構造的な問題」が  
　区別できるようになります。

　この区別ができると、  
　打つべき手が変わります。

**✅読者への一言：**  
　数字は「何と比べるか」で  
　意味が変わります。

　比較の設計まで  
　AIに手伝ってもらいましょう。

## 2. 原因を深堀りして対策を立てる

「売上が落ちた」という事実はわかった。

でも、「なぜ落ちたか」を  
答えられなければ、  
報告は終わっても  
問題は解決しません。

AIに「原因の仮説」を  
一緒に考えてもらいましょう。

---

### ◆原因仮説と対策立案プロンプト

**✅目的：**  
　数値変化の「なぜ」を深堀りし、  
　根本原因と対策を  
　セットで整理する。

**✅いつ使うか：**  
　「想定外の数値が出た」と気づいたとき。

```
# ロール設定
あなたはロジカルシンキング
（論理的思考）と
問題解決を専門とするコンサルタントです。
表面的な数値の変化から
根本原因を特定し、
実行可能な対策を提示してください。

# 背景・目的・対象
・起きていること：
　[例：先月のリピート購入率が
　前月比で-20%落ちた]
・業種・商材：[例：化粧品のD2C]
・すでに把握している要因（あれば）：
　[例：新商品を出した月だった]
・解決したいゴール：
　[例：リピート率を元の水準に戻したい]

# 指示内容
1. 考えられる原因を
　「内部要因」「外部要因」に分けて各3つ

2. 最も可能性が高い原因を
　理由とともに上位2つ絞り込む

3. 各原因に対する
　具体的な対策（短期・中長期それぞれ）

4. 対策を優先順位つきで並べ直す

# 制約条件
・推測・仮説は「仮説：」と明記する
・データで確認すべき点は
　「要確認：」と示す
・対策はすぐ動ける粒度で書く

# 出力形式
原因分析→仮説絞り込み→対策リスト
の順番で、見出しつきで整理。

# 入力データ
状況詳細・データ：[ここに入力]
```

**✅具体的な使用方法：**  
　「すでに把握している要因」の欄に  
　思い当たることを書いておくと、  
　AIが「それは原因ではなく  
　背景です」と区別してくれます。

　知っている情報ほど多く渡すほど、  
　精度が上がります。

**✅読者への一言：**  
　「何が起きたか」を報告するのは誰でもできる。

　「なぜ起きたか・どうすべきか」まで  
　言えると、あなたの価値が変わります。

## 3. 報告書の構成を一発で作る

データの分析ができた。

でも、どう伝えるかで  
相手の理解は大きく変わります。

「伝わらない報告書」は、  
正しいデータを持っていても  
意味がありません。

相手に刺さる構成を  
AIに設計してもらいましょう。

---

### ◆報告書・提案資料の構成設計プロンプト

**✅目的：**  
　分析結果を  
　相手に伝わる報告書の構成に  
　整理する。

**✅いつ使うか：**  
　会議・プレゼン・上司への報告資料を  
　作る前のステップとして。

```
# ロール設定
あなたはビジネス文書と
プレゼンテーション設計の専門家です。
読み手が「で、何が言いたいの？」と
思わない構成を作ってください。

# 背景・目的・対象
・報告・提案の内容：
　[例：3月の売上分析と
　4月の施策提案]
・報告相手：
　[例：営業部長（数字は見るが
　詳細より結論を好む）]
・使用媒体：
　[例：PowerPoint / Googleスライド]
・持ち時間・ページ数：
　[例：10分 / スライド8枚以内]

# 指示内容
1. 報告書・スライドの推奨構成
　（タイトル・各スライドの内容・順番）

2. 各スライドで「必ず入れるべき要素」

3. 「この相手・この目的」に最適な
　伝え方のポイント3つ

4. 避けるべき表現・構成のNG例

# 制約条件
・結論を最初に持ってくる
　「PREP法（結論→理由→根拠→再結論）」
　を基本にする
　※PREP法：
　　話の説得力を高める
　　伝え方のフレームワーク
・相手の「判断・行動」を
　引き出す構成にする
・情報を詰め込みすぎない

# 出力形式
スライド構成案をリスト形式で。
各スライドに「目的」を一言添える。

# 入力データ
分析内容・伝えたいこと：
[ここに箇条書きで入力]
```

**✅具体的な使用方法：**  
　「報告相手」の欄を丁寧に書くのが  
　最大のポイントです。

　「数字重視の上司」か  
　「ストーリー重視の経営者」かで、  
　最適な構成は変わります。

**✅読者への一言：**  
　「良い分析」と  
　「伝わる報告書」は別物です。

　このプロンプトで、  
　その橋渡しをしましょう。

## 4. 根拠ある予測で意思決定を助ける

「来月の売上はどうなる？」

この問いに答えられないと、  
計画が立てられない。

でも、でたらめな予測は  
かえって判断を誤らせます。

AIに「根拠ある仮説」を  
立ててもらいましょう。

---

### ◆客観的予測と意思決定支援プロンプト

**✅目的：**  
　過去データと外部要因をもとに、  
　根拠のある予測と  
　意思決定の材料を作る。

**✅いつ使うか：**  
　計画策定・予算会議・  
　次期施策の検討時に。

```
# ロール設定
あなたはデータに基づいた
ビジネス意思決定の支援専門家です。
感覚や希望的観測ではなく、
根拠に基づいた予測と判断材料を
提示してください。

# 背景・目的・対象
・対象の数値・指標：
　[例：次月の新規顧客獲得数]
・過去データの概要：
　[例：過去6ヶ月の月次推移]
・既知の外部要因：
　[例：来月に新競合サービスが参入予定]
・意思決定したいこと：
　[例：広告予算を増やすか据え置くか]

# 指示内容
1. 過去データから読み取れる
　トレンドと傾向

2. 楽観・中立・悲観の
　3シナリオでの予測範囲
　（数値の幅で示す）

3. 各シナリオで
　取るべき行動の違い

4. 判断を左右する
　「最も重要な確認事項」1〜2点

# 制約条件
・予測は必ず「幅」で示す
　（断言しない）
・「希望」と「根拠ある予測」を
　明確に区別する
・AIが判断できない部分は
　「要人間判断」と明示する

# 出力形式
シナリオ別の表＋アクション推奨。

# 入力データ
過去データ：[ここに入力]
検討中の意思決定：[ここに入力]
```

**✅具体的な使用方法：**  
　「意思決定したいこと」を  
　具体的に書くのがポイントです。

　「分析してほしい」より  
　「AとBどちらを選ぶか判断したい」と  
　書くと、AIの回答が格段に実用的になります。

**✅読者への一言：**  
　「予測は外れることがある」  
　それは当然です。

　でも、根拠のある予測と  
　でたらめな予測では  
　「外れたときの学び」の深さが違います。

　根拠から始める習慣を  
　AIと一緒に作りましょう。

## 5. 自分のバイアスをAIに指摘してもらう

「このデータは自分の仮説を  
支持しているから正しいはず…」

こういう思い込みを  
「確証バイアス」と言います。  
※確証バイアス：  
　自分の信念に合う情報ばかりを  
　集めてしまう心理的傾向

データ分析の最大の敵は、  
実は「自分の思い込み」です。

AIに「悪魔の代弁者」を  
やってもらいましょう。

### ◆分析のバイアスチェックプロンプト

**✅目的：**  
　自分の分析・解釈に含まれる  
　思い込みや見落としを  
　AIに指摘してもらう。

**✅いつ使うか：**  
　「この分析、合ってるかな？」と  
　自信がないとき。

　または重要な意思決定の前に  
　必ずセルフチェックとして使う。

```
# ロール設定
あなたは「批判的レビュアー」です。
私の分析や結論に対して、
反論・見落とし・思い込みを
遠慮なく指摘してください。

「良い点を見つける」のではなく、
「穴を見つける」のが
あなたの仕事です。

# 背景・目的・対象
・私の分析・結論：
　[例：来月は広告を増やすべきだ。
　なぜなら先月の広告費増加と
　売上増加が連動していたから]
・使ったデータ：
　[例：先月の広告費と売上の推移]
・この結論に基づいて
　取ろうとしているアクション：
　[例：広告予算を2倍に増やす]

# 指示内容
1. この分析の「論理的な穴」を3つ指摘

2. 見落としている可能性がある
　「別の解釈」を2つ提示

3. 結論を信頼するために
　「追加で確認すべきこと」を3点

4. 「このまま進んでよい条件」と
　「立ち止まるべき条件」を整理

# 制約条件
・批判は具体的に、理由とともに
・「批判のための批判」ではなく
　より良い判断のための指摘にする
・全否定ではなく
　改善提案もセットで

# 出力形式
番号付きリストで、
最後に「総合判断コメント」を一言。

# 入力データ
分析内容・結論：[ここに入力]
```

**✅具体的な使用方法：**  
　「私の分析・結論」に  
　自分が考えたことを  
　そのまま書いて渡すだけ。

　AIが第三者の目で  
　問題点を洗い出してくれます。

　重要な意思決定の前に  
　必ず一度使ってみてください。

**✅読者への一言：**  
　自分の分析を誰かに  
　「ツッコんでもらえる環境」は  
　貴重です。

　AIなら何度でも  
　遠慮なく突っ込んでもらえます。

　この習慣が、  
　判断ミスを防ぐ最強の盾になります。

## まとめ

データ分析をAIに任せることは、  
「自分で考えることをやめる」のではなく、  
「自分の思考をより深く、速く進める」ことです。

トレンドを言葉にし、原因を深掘りし、  
構成を設計し、バイアスを除き、  
根拠ある予測で判断を下す。

そのすべてのステップを  
AIと一緒に行えば、  
あなたのレポートはもう  
「数字の羅列の報告書」では  
なくなります。

今日からプロンプトを  
一つ試してみてください。  
「分析は得意じゃない」が、  
過去のセルフイメージになるはずです。

参考になったら、  
ぜひ**スキ・フォロー**をいただけると  
とても励みになります。

次の記事でもお役に立てるよう、  
精一杯書き続けます。
