# 宅建AIアシスタント開発で学んだ：Claude APIの5つのアンチパターン

宅建試験対策アプリ [takkenai.jp](https://takkenai.jp) を個人開発する中で、Claude APIの使い方について多くの失敗と学びがありました。実際に踏んだアンチパターン5つをTypeScriptのコード例とともに解説します。

## アンチパターン1：プロンプトに全部詰め込む

```
// ❌ 悪い例：全部ひとつのメッセージに詰め込む
const response = await anthropic.messages.create({
  model: 'claude-3-5-sonnet-20241022',
  max_tokens: 800,
  messages: [{
    role: 'user',
    content: `あなたは宅建試験の専門講師です。【結論】【根拠】【覚え方】で解説してください。
問題：${questionText}
選択肢：${choices.join('\n')}
正解：${answer}番`,
  }],
});
```

```
// ✅ 正しい例：システムプロンプトとユーザープロンプトを分離
const SYSTEM_PROMPT = `
あなたは宅建試験の専門講師です。
【結論】1〜2文で正誤と要点を明示
【根拠】法令根拠と理由
【覚え方】試験本番で使えるキーワード
`.trim();

const response = await anthropic.messages.create({
  model: 'claude-3-5-sonnet-20241022',
  max_tokens: 800,
  system: SYSTEM_PROMPT,  // ← systemとして分離
  messages: [{
    role: 'user',
    content: `問題：${questionText}\n選択肢：${choices.join('\n')}\n正解：${answer}番`,
  }],
});
```

**効果：** システムプロンプトのキャッシュ効率が上がり、コスト約20%削減。

## アンチパターン2：JSONレスポンスを信頼しすぎる

```
// ❌ 悪い例：バリデーションなしでいきなりパース
const parsed = JSON.parse(response.content[0].text); // 💥 SyntaxError頻発
```

LLMは以下のような形式で返すことがあります：

* JSONの前後に説明文を付ける（```json...）
* 数値の代わりに文字列（`"answer": "1"` vs `"answer": 1`）
* max\_tokensに引っかかりJSONが途中で切れる

```
// ✅ 正しい例：Zodでバリデーション
import { z } from 'zod';

const QuestionSchema = z.object({
  text: z.string().min(10),
  choices: z.array(z.string()).length(4),
  answer: z.number().int().min(1).max(4),
});

function extractJson(rawText: string): unknown {
  const match = rawText.match(/```(?:json)?\n?([\s\S]*?)\n?```/) 
    || rawText.match(/(\{[\s\S]*\})/);
  return JSON.parse(match ? match[1].trim() : rawText.trim());
}

export async function parseQuestion(text: string) {
  try {
    return QuestionSchema.parse(extractJson(text));
  } catch {
    return null; // バリデーション失敗はnullで返す
  }
}
```

## アンチパターン3：エラーハンドリングを後回し

```
// ✅ 指数バックオフ付きリトライ
async function withRetry<T>(fn: () => Promise<T>, maxRetries = 3): Promise<T> {
  let lastError: Error;
  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      return await fn();
    } catch (error) {
      lastError = error as Error;
      if (error instanceof Anthropic.APIError && [400, 401].includes(error.status)) {
        throw error; // リトライ不要なエラーは即座に再スロー
      }
      if (attempt < maxRetries) {
        await new Promise(r => setTimeout(r, Math.min(1000 * 2 ** attempt, 10000)));
      }
    }
  }
  throw lastError!;
}
```

リトライが尽きたときのFallbackも必須です：

```
export async function generateExplanation(question: Question) {
  try {
    return { text: await withRetry(() => callClaude(question)), isFallback: false };
  } catch {
    return { text: 'AI解説を現在生成できません。しばらく後にお試しください。', isFallback: true };
  }
}
```

## アンチパターン4：コストを意識しない

1250問×キャッシュなし＝毎回API呼び出し。月次コストが青天井になります。

```
// ✅ 多層キャッシュ戦略
const memCache = new Map<string, string>();

export async function getExplanation(questionId: string, question: Question) {
  const key = `exp:${questionId}`;
  
  // L1: メモリキャッシュ
  if (memCache.has(key)) return memCache.get(key)!;
  
  // L2: Redisキャッシュ
  const cached = await redis.get<string>(key);
  if (cached) { memCache.set(key, cached); return cached; }
  
  // L3: DB（事前生成済み解説）
  const dbRow = await prisma.question.findUnique({ where: { id: questionId }, select: { explanation: true } });
  if (dbRow?.explanation) {
    await redis.setex(key, 86400 * 30, dbRow.explanation);
    memCache.set(key, dbRow.explanation);
    return dbRow.explanation;
  }
  
  // L4: Claude API（最終手段）
  const text = await withRetry(() => callClaude(question));
  await Promise.all([
    prisma.question.update({ where: { id: questionId }, data: { explanation: text } }),
    redis.setex(key, 86400 * 30, text),
  ]);
  memCache.set(key, text);
  return text;
}
```

**効果：** 月次APIコストを開発初期比で約75%削減。
