「何を学べばいいか、分からない…」

そう感じながら、  
何ヶ月も止まったままに  
なっていませんか？

本を買っても途中で挫折する。  
動画を見ても身につかない。  
時間はかけているのに、  
スキルになっている実感がない。

そんな悩みを抱えているのは、  
あなただけではありません。

---

学習がうまくいかない理由は、  
「頭が悪い」でも  
「意志が弱い」でもありません。

**自分に合った学び方を  
知らなかっただけです。**

---

この記事を読むと、

✅ **時間を大幅に短縮できる**  
　→ 1年かかる学習が3ヶ月に。  
　　AIが最短ルートを  
　　設計してくれます。

✅ **何を学ぶか迷わなくなる**  
　→ キャリアに直結する  
　　「稼げるスキル」を  
　　AIが特定してくれます。

✅ **インプットが本物のスキルになる**  
　→ AIを練習相手に使うことで、  
　　知識が確実に定着します。

---

AIは「賢い家庭教師」です。

あなただけのカリキュラムを作り、  
分からない箇所をすぐに解説し、  
アウトプットの練習にも付き合ってくれる。

今日から学び方が、  
根本から変わります。

## 基本のプロンプト

まず、例え話からはじめます。

AIを学習に使うとは、  
どういうことか。

---

**カーナビに例えてみます。**

昔は、知らない場所に行くとき  
地図を広げて  
ルートを自分で考えていました。

でも今は、目的地を入力するだけで  
最短ルートを教えてくれますよね。

AIを使った学習も、  
それと同じです。

「何を学びたいか」を入力すれば、  
AIがあなた専用の  
最短ルートを設計してくれます。

渋滞（つまずき）があれば  
迂回路（別の説明方法）も  
提案してくれます。

地図を広げて迷う時間は、  
もう必要ありません。

### ◆あなた専用カリキュラム作成プロンプト

**✅目的：**  
　ゼロから学習計画を立てる。

**✅いつ使うか：**  
　「何かを学ぼう」と  
　思い立ったその日に。

```
# ロール設定
あなたは学習コーチの専門家です。
初心者でも無理なく続けられる
パーソナライズされた学習計画を
設計してください。

# 背景・目的・対象
・学びたいスキル：[例：Pythonプログラミング]
・現在のレベル：[例：完全初心者]
・目標：[例：3ヶ月後に業務自動化ツールを作りたい]
・使える時間：[例：平日30分、休日2時間]

# 指示内容
以下を設計してください。

1. 最終ゴールまでのロードマップ（週単位）
2. 最初の1週間でやること（具体的に）
3. つまずきやすいポイントと
　その乗り越え方
4. 無料で使えるおすすめ教材（3つ）

# 制約条件
・初心者が挫折しないペースにする
・「1回30分以内」に収まる学習単位で設計
・専門用語には必ず補足説明を入れる

# 出力形式
ロードマップは週ごとの表形式で。
その他は見出しつき箇条書きで。

# 入力データ
学習テーマ：[ここに具体的に入力]
```

**✅具体的な使用方法：**  
　「学びたいスキル」と「目標」を  
　具体的に書くのがポイントです。

　「なんとなく英語」より  
　「TOEIC700点を取って転職したい」の方が、  
　AIの回答は格段に良くなります。

**✅読者への一言：**  
　計画がないまま走り出すと、  
　すぐ迷子になります。

　まずここから始めてみてください。

### ◆分からない箇所を即解決するプロンプト

**✅目的：**  
　学習中のつまずきを、  
　その場で解消する。

**✅いつ使うか：**  
　「ここが分からない」と  
　思ったそのタイミングで。

```
# ロール設定
あなたは10歳の子どもにも
説明できる、やさしいプロ講師です。
難しい概念を、
具体的な例え話を使って
分かりやすく教えてください。

# 背景・目的・対象
・学習中のテーマ：[例：Pythonの「関数」]
・今の理解度：
　[例：「関数を作れ」と言われたが
　何のためにあるか分からない]
・これまでに試したこと：
　[例：教材を読んだが
　ピンとこなかった]

# 指示内容
1. 概念を10歳にも分かる
　例え話で説明する
2. 具体的なコード例（または事例）を
　1つ見せる
3. よくある誤解・間違いを1つ教える
4. 理解を確認するための
　簡単なクイズを1問出す

# 制約条件
・専門用語を使う場合は
　必ず日常語で補足する
・難しい説明より
　分かりやすさを優先する

# 出力形式
各項目に見出しをつけて
読みやすく整理。

# 入力データ
分からないこと：[ここに具体的に入力]
```

**✅具体的な使用方法：**  
　「分からないこと」を  
　正直に書くのがコツです。

　「なんとなく分からない」でも大丈夫。  
　AIは親切に付き合ってくれます。

**✅読者への一言：**  
　「こんな初歩的なこと聞いていいのか」  
　と遠慮する必要はありません。

　AIは何度聞いても怒りません。  
　これが最大の強みです。

## ビフォー・アフター比較

**同じ悩みでも、プロンプトの作り方で  
結果は大きく変わります。**

想定シナリオ：  
「副業に使えるスキルを  
　AIで学びたい社会人」

---

### Before（簡易プロンプト）

```
副業に使えるスキルを
学ぶ計画を立ててください。
```

**AIの回答（例）**：

> プログラミング、デザイン、  
> 動画編集などが人気です。  
> 本や動画で学ぶと良いでしょう。

---

👉 **どこが足りないか？**

* 一般論すぎて  
  　自分に当てはまらない
* 「今日から何をするか」が分からない
* 自分の状況が全く考慮されていない

---

### After（本記事のプロンプトを使用）

```
# ロール設定
あなたは学習コーチの専門家です。

# 背景・目的・対象
・現在の仕事：会社員（経理担当・34歳）
・使える時間：平日30分、休日2時間
・副業で目指したいこと：
　月3万円を6ヶ月以内に稼ぎたい
・得意なこと：
　Excelは使える、文章は苦手
・学習予算：月2000円まで
```

**AIの回答（例）**：

>  【おすすめスキル】  
>  Excelの強みを活かした  
>  データ整理・集計代行  
>   
>  【ロードマップ（6ヶ月）】  
>  1〜2ヶ月目：クラウドソーシングの  
>  登録と案件リサーチ  
>  3〜4ヶ月目：初案件に挑戦・実績作り  
>  5〜6ヶ月目：単価アップ交渉・月3万達成  
>   
>  【最初の1週間でやること】  
>  ・ランサーズ・クラウドワークスに登録  
>  ・プロフィール文をAIで作成  
>  ・競合5名のプロフィールを分析

---

👉 **何が変わったか？**

「ロール設定」で  
AIを専門の学習コーチとして起動し、

「背景指定」で  
自分の状況・時間・目標・得意不得意を  
渡したことで、

**今日から動ける具体的な計画が  
出てきました。**

情報を渡すほど、  
AIの回答は自分ごとになります。

これがプロンプトの核心です。

## 実践プロンプト集

## 1. 稼げるスキルを特定する

「何を学べばいいか分からない」

これが、学習を始める前の  
最大の壁です。

やみくもに学んでも、  
市場で評価されないスキルでは  
意味がありません。

AIに「今あなたが学ぶべきスキル」を  
特定してもらいましょう。

---

### ◆稼げるスキル特定プロンプト

**✅目的：**  
　自分のキャリア・状況に合わせた  
　「今学ぶべきスキル」を特定する。

**✅いつ使うか：**  
　学習を始める前、  
　または「このまま続けていいか」  
　迷ったとき。

```
# ロール設定
あなたはキャリアコンサルタントと
学習アドバイザーを兼ねる
専門家です。
依頼者の状況を分析し、
費用対効果が最も高いスキルを
特定してください。

# 背景・目的・対象
・現在の仕事・職種：[例：営業職・28歳]
・年収・収入の目標：
　[例：今より月3万円増やしたい]
・現在持っているスキル・経験：
　[例：Excel基本操作、
　営業トーク5年分]
・使える時間：
　[例：平日1時間]
・学習に使える予算：
　[例：月3000円まで]

# 指示内容
1. 今の状況から最も稼げる
　スキルを3つ特定する
2. 各スキルの
　「習得期間の目安」
　「平均収入への影響」
　「難易度」を整理する
3. 一番おすすめの1つを選び、
　その理由を説明する
4. 今週から始められる
　最初の一歩を具体的に示す

# 制約条件
・夢物語ではなく現実的な提案にする
・半年以内に効果が出るものを優先
・すでに持っているスキルを
　活かせるものを重視する

# 出力形式
比較表＋おすすめ理由＋今週の行動リスト

# 入力データ
自分の状況：[ここに詳しく入力]
```

**✅具体的な使用方法：**  
　「現在持っているスキル・経験」を  
　できるだけ詳しく書くのがポイント。

　「普通のサラリーマン」ではなく、  
　「5年間の営業経験・  
　　月100件の電話対応経験あり」  
　のように書くと精度が上がります。

**✅読者への一言：**  
　「自分には何もない」と  
　思いがちですが、  
　それは間違いです。

　あなたの経験の中に、  
　必ずお金になるスキルがあります。

　AIに探してもらいましょう。

## 2. 記憶に定着させる反復練習

「覚えたはずなのに  
　すぐ忘れてしまう」

これは意志の問題ではありません。

人間の脳は、1回読んだだけでは  
定着しない構造になっています。

鍵は「アウトプット」です。

自転車の乗り方を  
本で読むだけでは  
乗れるようにならないのと同じ。

AIを練習相手にして、  
知識を本物のスキルに  
変えましょう。

---

### ◆反復アウトプット練習プロンプト

**✅目的：**  
　学んだ内容を  
　問題演習・質問応答を通じて  
　定着させる。

**✅いつ使うか：**  
　テキストを一通り読んだ後、  
　または「理解したつもり」の  
　内容を確かめたいとき。

```
# ロール設定
あなたは厳しくも
丁寧な試験官です。
私が学んだ内容を
正しく理解しているか
テストしてください。

# 背景・目的・対象
・学習した内容：
　[例：Pythonのリスト操作]
・理解度の自己評価：
　[例：70%くらい分かった気がする]
・苦手だと感じる部分：
　[例：forループとの組み合わせが曖昧]

# 指示内容
1. この内容に関する
　確認問題を5問出す
　（難易度：易2問・中2問・難1問）
2. 私が回答したら、
　正誤と解説をする
3. 間違えた問題は
　別の視点から
　もう一度説明し直す
4. 最後に「理解度チェック表」を
　作って弱点を整理する

# 制約条件
・問題は実際の業務や
　日常に近い場面で出す
・「正解できた＝理解できた」
　ではなく、
　応用できるかも確認する

# 出力形式
問題→回答欄の形式で
1問ずつ出題してください。

# 入力データ
学習テーマ・内容：[ここに入力]
```

**✅具体的な使用方法：**  
　AIが問題を出してきたら、  
　実際に答えを書いて送ります。

　「なんとなく分かる気がする」と  
　「実際に答えられる」の間には  
　大きな差があります。

　その差を埋めるのが  
　このプロンプトです。

**✅読者への一言：**  
　インプットだけでは  
　スキルになりません。

　アウトプットの回数が、  
　成長のスピードを決めます。

## 3. つまずきを即解消する壁打ち

学習を続ける中で  
一番の敵は「挫折」です。

挫折の多くは  
「分からないことが解決できずに  
　放置される」ことから生まれます。

教室なら先生に聞けます。  
でも独学では聞く相手がいない。

だからAIを  
「いつでも答えてくれる先生」  
として使いましょう。

---

### ◆壁打ち深掘り解説プロンプト

**✅目的：**  
　「なんとなく分かる」を  
　「ちゃんと説明できる」  
　レベルまで引き上げる。

**✅いつ使うか：**  
　学習中に「これってどういうこと？」  
　と思った瞬間すべてに。

```
# ロール設定
あなたはソクラテス式問答法を使う
教育の専門家です。
答えをすぐ教えるのではなく、
私が自分で気づけるよう
質問しながら導いてください。

# 背景・目的・対象
・理解しようとしている概念：
　[例：「変数」とは何か]
・今の自分の理解：
　[例：数値を入れる箱みたいなもの？]
・何が腑に落ちていないか：
　[例：なぜわざわざ変数にする必要があるか]

# 指示内容
①私の理解が正しいかチェックし、
　ズレがあれば指摘する

②「なぜこれが必要か」を
　現実の例え（料理・スポーツ・日常）
　に置き換えて説明する

③私の理解を確認する
　一問一答を3問行う

④「自分の言葉で説明するとしたら？」
　と聞いて、
　説明させてみる

# 制約条件
・答えをすぐ言わず、
　気づきを引き出すよう促す
・褒めながら正しい方向へ導く

# 出力形式
会話形式で進める。

# 入力データ
理解したい概念・状況：[ここに入力]
```

**✅具体的な使用方法：**  
　「今の自分の理解」を  
　正直に書くのがポイントです。

　「間違っていたら恥ずかしい」  
　と思わなくて大丈夫。

　AIは絶対に馬鹿にしません。

**✅読者への一言：**  
　「自分の言葉で説明できる」が、  
　本当に理解した証拠です。

　誰かに説明するように  
　AIに話してみてください。

## 4. モチベーションを管理する

「最初は頑張れるのに、  
　3週間で続かなくなる」

これはあなたの性格の問題ではなく、  
学習設計の問題です。

人間のやる気には波があります。

波に乗るのではなく、  
波に左右されない仕組みを  
作りましょう。

AIがあなた専用の  
やる気管理システムを  
設計してくれます。

---

### ◆学習継続システム設計プロンプト

**✅目的：**  
　途中で挫折しない  
　学習の仕組みを作る。

**✅いつ使うか：**  
　「また続かなかった」と  
　落ち込んだとき。  
　または学習を始める前の設計段階で。

```
# ロール設定
あなたは行動科学と習慣化の
専門家です。
人間の心理特性を踏まえた、
続けられる学習システムを
設計してください。

# 背景・目的・対象
・過去に挫折した学習と
　その理由：
　[例：英語学習・忙しくなると
　真っ先に削った]
・自分のモチベーションの特徴：
　[例：結果が出ると続くが、
　成果が見えないと辞める]
・生活リズム：
　[例：朝7時起き・21時帰宅]
・達成したい目標と期限：
　[例：3ヶ月でTOEIC600点]

# 指示内容
1. 挫折パターンを分析し、
　対策を3つ提案する

2. 「最低限これだけ」という
　ミニマム習慣を設計する
　（忙しい日でも5分でできるもの）

3. 成果を実感しやすい
　「小さな指標」を3つ作る
　（モチベーション維持用）

4. 1ヶ月後・3ヶ月後の
　自分へのご褒美プランを提案する

# 制約条件
・「根性論」は使わない
・科学的な根拠のある方法を優先
・完璧主義にならない設計にする

# 出力形式
挫折対策・ミニマム習慣・指標・
ご褒美の4つを
見出しつきで整理。

# 入力データ
自分の状況・挫折歴：[ここに詳しく入力]
```

**✅具体的な使用方法：**  
　「過去に挫折した理由」を  
　できるだけ正直に書いてください。

　「なんとなくダレた」より  
　「夜遅く帰ると疲れて  
　　何もやる気が起きなかった」  
　の方が、的確な対策が出ます。

**✅読者への一言：**  
　続かないのは、  
　あなたが弱いからではありません。

　仕組みが足りなかっただけです。

　AIと一緒に、  
　続く仕組みを作りましょう。

## 5. 学んだことをすぐ収入につなげる

スキルを学んでも、  
「実際にお金になるまで」に  
時間がかかりすぎて  
モチベーションが落ちる。

そんな経験はありませんか？

学習と収益化を  
同時に進める方法があります。

「学びながら稼ぐ」を  
AIと一緒に設計しましょう。

---

### ◆学習×収益化同時設計プロンプト

**✅目的：**  
　スキル習得と収  
　入につながる実践を同時進行させる。

**✅いつ使うか：**  
　学習計画を立てるとき、  
　または「いつになったら稼げるか」  
　と思い始めたとき。

```
# ロール設定
あなたは副業・フリーランス支援の
専門家です。
学習とマネタイズ（収益化）を
並行して進める戦略を
設計してください。

# 背景・目的・対象
・学習中または学ぼうとしているスキル：
　[例：Webデザイン]
```
