## 自分を解剖したあと、他人もやってみたくなった

前記事で自分の31秒スピーチを AmiVoice ESAS × Claude にかけて解剖し、「結論先出し型のつもりが middle 型だった」「『元気なイメージ』と言った瞬間に stress=46 + energy=3 で身体が真逆だった」など、自覚と数値のズレが出てきました。次に湧いた疑問はシンプルで、「**比較対象として参照されやすい著名スピーチを通したら何が出るんだろう?**」でした。

そこで、公開されている4本の著名スピーチを各 **1分ずつ** 切り出して同じパイプライン（AmiVoice ESAS × Claude 三層解析）に通しました。結果として、4人それぞれの「結論位置」「フィラーの個性」「身体と言葉のラグ」が、明らかに違う形で出てきました。先に断っておくと、「うまい人」を解剖するつもりが、結果的に4人それぞれの弱点を可視化する記事になっています。すいません、データがそう言うので、そう書きます。

## 解析対象（各1分、YouTube公開動画から `yt-dlp` で切り出し）

| 動画 | 切り出し範囲 | アップロード元 |
| --- | --- | --- |
| **Steve Jobs — Stanford Commencement Address (2005)** | 14:00–15:00（"Stay hungry, stay foolish" 部分） | Stanford 公式チャンネル |
| **孫正義 — 新30年ビジョン (2010) 第10章「情報革命で人々を幸せに」** | 0:00–1:00（冒頭セクション） | SoftBank Group 公式チャンネル |
| **西野亮廣 — 講演会 in ホーチミン** | 0:00–1:00（冒頭セクション） | 西野亮廣 公式チャンネル |
| **落合陽一 — TEDxTokyo 2014「Physicalization of Computer Graphics」** | 0:00–1:00（冒頭セクション） | TEDx Talks 公式チャンネル |

音声ファイル自体は本ツールリポジトリにも本記事にも含めません（YouTube ToS・著作権の都合で、抽出した一部分の再配布が曖昧なため）。本記事には **解析結果から抽出した観察と知見のみ** を掲載します。

Jobs 終盤と落合冒頭はどちらも日英混在で、AmiVoice の `-a-general`（日本語エンジン）だと認識テキストは誤認識まみれになります。**それでも ESAS の値はちゃんと出ます**。前記事でも触れた「言葉が分からなくても声の癖は分かる」性質が、ここで再度効きます。

## 比較表（要点）

| 指標 | Jobs終盤 | 孫正義冒頭 | 西野冒頭 | 落合冒頭 |
| --- | --- | --- | --- | --- |
| **energy** peak | 21 | **44** | 32 | 39 |
| **concentration** peak | 100 | 100 | 90 | 100 |
| **anticipation** peak | 68 | **100** | 35 | 91 |
| **emo\_cog** peak | 207 | 136 | **276** | 244 |
| **passionate** peak | 6 | 4 | **11** | 8 |
| **結論位置** | back | **front** | middle | back |
| **フィラー** | "ね"×1 | "ね"×1 | **"ちょっと"×5 + 認識上の token "なと"×3** | "あの"×1 |
| **認識精度** | conf 0.74–0.89（日英混在） | **0.99–1.00**（完璧） | 0.93–1.00（完璧） | 0.70–0.82（日英混在） |

`emo_cog`（Emotional-Balanced-Logical）の値域は公式の `result-parameters.json` を叩くと 1–500 と返ってきます。他のパラメータ（0–100 や 0–30）と同列で読むと「異常値だ」と誤判定するので、解析プロンプト側に値域を明記しています（[前記事の値域分類表](https://zenn.dev/kenimo49/articles/amivoice-esas-claude-self-speech-analysis)を参照）。

## 各話者の核となる癖

### Steve Jobs（Stanford 終盤・"Stay hungry, stay foolish"）

* **`passionate=平均0.7、peak=6`** — 「熱い」スピーチの代表として引用されがちですが、AmiVoice の `passionate` 指標は **ほぼ反応しません**。代わりに `concentration peak=100` が出ていて、「**静かに語る確信スタイル**」が音響的に観察されます
* 結論ゾーン（56–59s）で `confidence=27 → 残存`、`passionate=6 → 0` — 着地に向けて熱量と自信が同時に低下している。"Stay hungry, stay foolish" は **最後にもう一発上げて締める**のではなく、**じわじわ収束させる**スタイルだったとデータで分かります
* 認識テキストは英語×日本語エンジンで誤認識まみれですが、ESAS は完全に機能。「ステイフーリッシュ」だけは部分的にカタカナで残りました（耳で聞いて分かる単語と認識エンジンが拾える単語は別物）

### 孫正義（新30年ビジョン 冒頭・「情報革命で人々を幸せに」）

* **結論を冒頭に置く**典型 — `1.8s で concentration=100 / 14.7s で anticipation=100 / 16.3s で energy=44 スパイク` と、開始20秒以内に主要指標がすべてピークアウト
* 中盤の「30年後」反復で `energy=1〜2` まで低迷。**リフレインしながら声量が落ちる**癖が音響的にはっきり出ています
* 終端は「思うかもしれません、あるいは思わないかもしれません」で両論留保 → 冒頭で宣言した強い主張が終端で失圧（一介の OSS 開発者が孫さんの話し方に物言いをつけるのはさすがに恐れ多いのですが、ESAS の値はそう言っています）
* 認識精度は `0.99–1.00` で完璧。マイクが良いプレゼン環境 + 訓練された発話のコンボで、AmiVoice の精度が天井に張り付きます

### 西野亮廣（ホーチミン講演 冒頭）

* **フィラー "ちょっと"×5、"なと"×3** — 他3人が "ね/あの" 1回程度なのに対し、**定量的に多い**。これが「西野節」として観察可能な癖
* `hesitation=12–21` が全25サンプルで持続 → **躊躇が語彙レベル（フィラー）と音声レベル（hesitation）で二重固定**。「ちょっと」と言う瞬間に hesitation も上がる、ではなく、**ずっと両方上がっている**
* 終幕の無音/余韻区間（45.4s〜）で `energy=32、emo_cog=276、confidence=30` と全スピーチ最高値が残る、珍しいパターン。話している区間ではなく余韻側に高値が出る現象自体が他3人と差があり、話法上はこれを「西野節」の一部として読めそうです（区間切り出しのアーティファクトで説明できる余地もあるので、強い意図推定までは踏み込まずに留めておきます）

### 落合陽一（TEDxTokyo 2014 冒頭）

* 39.9s で **`concentration=100, stress=35, confidence=2` が同時発生** — 「**全力で考えているが自分の言葉に確信が持てない**」音響的矛盾
* 39.1s で `anticipation=91` のスパイク、同区間で認識テキストが日英混在に崩壊 → **身体の高揚が言葉の構造を追い越して空回り**している
* 早口で英語混じりの話し方がデータに反映され、認識精度 `conf 0.70–0.82`。ESAS は無問題に機能するので、「**英語混じりプレゼンの音響特徴を測りたい**」用途にも使える可能性（「身体が言葉を追い越す」と書かれても本人が「そうだろ、それが面白いんだろ」と返してきそうな空気は感じます）

## 横断的に見えた知見

### 1. 「身体と言葉のラグ」は4人ともどこかで観測される

`energy / concentration / anticipation` の peak と、テキスト層が捉えた主張位置が完全には一致しない場面が **4本すべてで観察**されました。たとえば Jobs は結論ゾーンで passionate=6→0 と熱量が逆に落ち、落合は身体高揚 (anticipation=91) の直後に言葉が日英混在で崩れる、孫正義は冒頭主張と主要ピークが近いものの中盤リフレインで energy が 1〜2 まで落ちる、といった形で各自違うラグが出ます。これは ESAS 単体でも Claude 単体でも見えず、**両方をプロンプト境界の中で交差させたクロス層**だけが可視化する現象です。

### 2. 構成パターンは3型に分かれる

* **front 型（孫正義）**: 結論先出し → 詳細 → 余韻
* **middle 型（西野）**: 前置き → 依頼/主張 → フェード
* **back 型（Jobs / 落合）**: 詳細展開 → 終盤に主張

判定は主にテキスト層の `conclusion_position.zone`（front / middle / back）で行い、その zone 内に `concentration peak` の時刻が落ちているかを補助条件として見る、というシンプルなロジックで4本すべて分類できました。「あなたは何型か」を可視化できるのは、自己診断のフックとして地味に強い。

![結論位置の3型: TYPE A FRONT (孫正義) / TYPE B MIDDLE (西野亮廣) / TYPE C BACK (Jobs/落合陽一)、それぞれタイムライン上の主張位置とパターンを図示](https://static.zenn.studio/user-upload/deployed-images/770f7449e6425002a72ea8d6.png?sha=067d0900cef2889a701b621f15af039947e4d87f)

### 3. フィラーの個性が定量化される

他3人が "ね/あの" 1回程度に対して、西野だけ "ちょっと" 5回 + "なと" 3回。**フィラーの「種類」と「頻度」を見るだけで話者を識別できそうな**密度差が出ました。実用的には、コーチング目的で「あなたの口癖は X です」を数字付きで返せる、ということ。

### 4. `passionate` は全員低い（peak 4–11）

「熱い」と感じる4人全員の `passionate peak` が 4–11 に収まりました。**「観察者の主観的熱量」と「音響パラメータの passionate」は別物**として扱う必要があります。AmiVoice の `passionate` は別の音響特徴を測っている可能性が高く、「passionate=高 = 情熱的」と単純に読まないほうがいい指標です。

### 5. 認識テキストが誤認識でも ESAS は機能する

Jobs（英語×日本語エンジン）、落合（日英混在）でも ESAS パラメータは意味のある値を出しました。これは前記事でも触れた点ですが、**4人比較で同じ現象が再現された**ことで確度が上がりました。「言葉が分からなくても、声の癖は分かる」性質はこのツールの汎用性を支える柱です。

## このケーススタディが、ツールに何を還元したか

4本を通したことで、speech-habit-lens の `v0.1.2` 改修につながった発見が4つありました。

| 発見 | 反映先 |
| --- | --- |
| `emo_cog` の値域が他パラメータと違う（公式 doc では 0–300 と書かれていたが、後日 `result-parameters.json` で 1–500 と確定） | `prompts/acoustic_layer.md` に値域明記、Claude が「異常値だ」と誤判定しないよう改修 |
| フィラー検出が日本語の終助詞「ね」を拾う | `prompts/text_layer.md` でフィラー定義を厳密化、終助詞「ね」は機械的には除外、文脈で再判定 |
| 並列実行時に `/tmp/shl-{layer}.txt` が上書きされる | `analyze.py` で per-process ディレクトリ（pid + timestamp）に変更 |
| `text_layer` が確率的に JSON パース失敗 | `analyze.py` で `stop_reason=end_turn` 時に1回自動 retry、それでもダメなら `raw_decode` で末尾余分を捨てる |

**自分1人のスピーチを解剖していたら絶対に見つからなかった bug が、ケーススタディで4倍のサンプルを通すと出てくる**、という当たり前のことが実感できました。Dogfooding はサンプル数1で終わらせると盲点が残る、ということでもあります。

## まとめ

* 4人の名スピーチを通すと、**結論位置・フィラー・身体と言葉のラグ・passionate の解釈** がそれぞれ違う形で定量化されました
* 「Jobs は熱量で押し切る」「孫正義は結論先出し」「西野はフィラーが多い」「落合は身体が言葉を追い越す」といった **印象論を、再現可能な数値とテキスト引用で言語化**できます
* ツール側の盲点（`emo_cog` 値域、フィラー誤検出、並列実行時の I/O 衝突、JSON パース失敗）も4本サンプルで初めて顕在化しました。**Dogfooding はサンプル数を増やすほど bug が出る**、を素直に体感できる検証パイプラインでもあります

ツールは [`kenimo49/speech-habit-lens`](https://github.com/kenimo49/speech-habit-lens)（MIT）で公開しています。AmiVoice API key と Anthropic API key を `.env` に入れて `shl serve` で動きます。

```
git clone https://github.com/kenimo49/speech-habit-lens
cd speech-habit-lens
python3 -m venv .venv && source .venv/bin/activate
pip install -e .
cp .env.example .env  # API key を記入
shl serve             # http://localhost:8501
```

録音タブを使う場合は `ffmpeg` が PATH に必要です（既存 WAV のアップロード解析だけなら不要）。
