# 導入｜AIは本当に投資判断できるのか？

LLMは「物事を理解しているAI」ではなく、「次に来る言葉を予測するAI」です。

それでも、ニュースを読むことができ、企業情報を整理した上で、それっぽい投資判断を出すことはできます。

しかしここで1つ疑問があります。

LLMが出力したその判断は、本当に正しいのでしょうか？

株式投資では、判断を間違えればそのまま損失につながります。「それっぽい判断」では通用しない世界です。

一方でLLMは、過去のデータからそれらしい答えを生成しているに過ぎません。

それが正しいかどうかは、別の話です。

本記事では、シンプルな投資エージェントを構築し、AIがどのように投資判断を誤るのかを検証していきます。

# 結論

単一エージェントでは、ほぼ確実に誤った投資判断を下します。

理由はシンプルで、投資は1つの情報だけで決まるものではないからです。

経済、業界、企業の実情、決算、技術、社会情勢。  
複数の要素が絡み合った結果として、最終的な価格が決まります。

このような前提に対して、単一のコンテキストを与えて一度で判断させるのは、構造的に無理があります。

次に、思考の問題です。

単一エージェントは、基本的に「1回の推論」で結論を出します。しかし投資判断は、本来1回で終わるものではありません。

情報を集め、仮説を立て、別の視点から見直し、必要であれば前提を修正する。このプロセスを繰り返すことで、ようやく判断の精度が上がります。

事前にプロンプトで判断基準をすべて定義すれば、ある程度の精度は出せるかもしれません。ただしそれは、AIが考えているのではなく、人間が考えたものをなぞっているだけです。

最後に、評価の問題があります。

投資は、すぐに正解が出るものではありません。短期トレードであれば比較的早く結果が出ますが、それでもノイズは多く、単純な評価は難しいです。

長期投資であれば、評価には数年かかることもあります。

つまり、間違っているかどうかをすぐに検証できない。

これは、エージェントを改善していく上で致命的な問題になります。

以上から、単一エージェントで投資判断を行い、精度を高めていくのはかなり難しいと言えます。

# 実験設計｜最初のAIエージェントを作る

最終的には複雑な構成になることが想定されますが、まずはシンプルな投資エージェントから始めます。

今回作るエージェントは、以下のような最小構成です。

* 入力：銘柄（例：トヨタ）
* 情報取得：関連ニュースを取得
* 分析：LLMがニュース内容をもとに判断
* 出力：「買い」または「見送り」とその理由

ユーザーが銘柄を入力すると、関連するニュースを取得し、その内容をもとにLLMが投資判断を行います。

いわゆる「ニュースを読んで判断するだけ」のエージェントです。

この時点で、良い投資判断ができるとは思っていません。  
むしろ、どのように間違えるのかを確認するための構成です。

まずはこのシンプルなエージェントをベースに、どこに問題があるのかを見ていきます。

# 実行｜実際に動かしてみる

実際の出力ログ

```
投資判断: 見送り
理由: 提供されたニュースはトヨタ固有の業績・戦略・バリュエーションに関する情報がなく、半導体/AI関連や米株動向、最低投資額低下といった市場環境の一般論に留まるため、長期投資の判断材料としては不十分。
日本株の投資ハードル低下は需給面で追い風となる可能性はあるものの、トヨタの長期価値創造（電動化・ソフトウェア戦略、収益性、為替耐性等）への影響は本ニュース群からは評価できない。
不確実性が高いため見送り。
```

Tavilyから取得したデータをもとに、投資判断自体は行えていることが確認できました。

一方で、この出力にはいくつかの問題が見えます。

まず、取得されている情報が「市場全体の一般論」に偏っており、トヨタ固有の情報がほとんど含まれていません。

その結果、LLMは以下のような挙動を取っています。

* 個別企業ではなく、市場全体の不確実性を理由に判断している
* 情報不足を「リスク」として扱い、見送りを選択している

これは一見合理的な判断に見えますが、裏を返すと「十分な情報がなくても、それっぽい理由で結論を出してしまう」ことを意味しています。

つまりこの時点で、以下の問題が顕在化しています。

* 情報の問題：企業固有の情報が取得できていない
* 思考の問題：情報の質を評価せずに結論を出している

前章で述べた「単一エージェントの限界」が、すでにこの段階で現れていることがわかります。

ただし、ニュース情報の質を上げることで、判断の精度自体は一定程度改善される可能性はあります。

しかし、それはあくまで「入力の質を上げている」だけであり、単一エージェントの構造的な問題を解決しているわけではありません。

# 問題｜なぜこのAIは間違えるのか？

一見すると、このエージェントは合理的に見えます。

最新のニュースを取得し、その情報をもとに投資判断を行っているためです。

しかしここに、大きな前提のズレがあります。

このエージェントが見ているのは「ニュース」という一部の情報に過ぎません。

長期投資の判断を行うためには、本来以下のような情報が必要になります。

* 有価証券報告書
* ファンダメンタルズ分析（売上・利益・成長性）
* テクニカル分析（需給・トレンド）
* 業界構造や競争環境

つまり、投資判断とは「複数の異なる種類の情報を統合する行為」です。

それに対して、このエージェントは「ニュースだけ」で結論を出しています。

この時点で、判断の前提が大きく欠けている状態です。

さらに問題なのは、情報の質ではなく扱い方です。

LLMは、与えられた情報の正しさを検証することなく、それを前提として推論を行います。

そのため、もし取得したニュースの中に誤情報や偏りが含まれていた場合、そのまま判断に反映されてしまいます。

これは、人間であれば「本当に正しいのか？」と疑う部分です。

しかし単一エージェントでは、そのような再検証のプロセスが存在しません。

つまりこのエージェントは、

という2つの問題を同時に抱えています。

これが、今回のAIが間違える本質的な理由です。

# 仮説｜何を改善すればいいのか？

このエージェントの問題は、単純な精度の問題ではありません。

投資判断のプロセス自体が再現できていないことにあります。

現状のエージェントは、「ニュースを取得して、そのまま判断する」という構成になっています。

しかし実際の投資では、このような判断は行いません。

通常は、

* ファンダメンタルズで企業の価値を把握する
* テクニカルで需給やトレンドを確認する
* 必要に応じてニュースで変化の要因を確認する

というように、複数の観点を使い分けて判断を行います。

つまり、ニュースは「常に使う情報」ではなく、「必要なタイミングで参照する情報」です。

それに対して今回のエージェントは、ニュースだけを常に見て判断しているため、判断の前提がズレています。

また、取得している情報の質にも問題があります。

Tavilyから取得されたデータには、企業ページに近い内容が含まれており、投資判断に直接使える情報とは言えません。

そのため、単に検索精度を上げるだけでなく、

を設計する必要があります。

さらに、現状のエージェントは1回の推論で結論を出していますが、実際の投資判断は1回で完結するものではありません。

仮説を立て、情報を集め、見直すというプロセスを繰り返します。

以上より、このエージェントを改善するためには、

* 投資判断のプロセスを分解する
* 情報の役割とタイミングを定義する
* 複数回の思考を前提とした構造にする

といった設計が必要になります。

# まとめ｜投資AIの本質は「設計」にある

今回の検証を通して明らかになったのは、投資AIの問題は単純な精度の問題ではないということです。

ニュースを取得し、LLMに判断させるだけでは、それっぽい結論は出せても、実際に使える意思決定にはなりません。

その理由は、投資判断が「複数の情報を使い分けながら、何度も見直すプロセス」であるためです。

つまり重要なのは、

* どの情報を使うのか
* どのタイミングで使うのか
* どのように検証するのか

といった「設計」です。

単一のエージェントにすべてを任せるのではなく、役割ごとに分解し、必要な情報を適切なタイミングで扱う構造にする必要があります。

今回のシンプルなエージェントは、その限界を確認するためのものでした。

ここから先は、この意思決定プロセスをどのように分解し、エージェントとして実装していくかが次のテーマになります。

AIエージェントの本質は、モデルの性能ではなく、「どのように考えさせるか」という設計にあります。
