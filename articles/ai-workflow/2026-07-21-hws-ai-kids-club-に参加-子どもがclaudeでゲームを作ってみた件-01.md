## 全社でAIを使い倒す、という方針

Headwatersでは「FY2027に生産性2倍」を目指して、全社でAIを使い倒す方針を掲げています。  
私が所属する技術戦略推進事業部では、このAI推進を担当していて、開発チームだけでなくミドルオフィス・バックオフィスまで含めて、全社に広げる活動をしています。  
合言葉はシンプルな3つ。

外部AIとしてMicrosoft Copilot、GitHub Copilot、Claudeなどを併用しながら、社内では「使う人を選ばない」AI活用文化づくりを進めています。

……といっても、いくら方針を掲げても「配ったから2倍になる」わけじゃないのが難しいところ。

今回は少し番外編として、その文化を社員の子どもたちにも体験してもらったときの様子をご紹介します。

## HWS AI Kids Club 第2回開催に参加

社員の子ども向けに不定期で開催している「HWS AI Kids Club」。第2回が2026年7月18日に開催されたので参加しました。

私は小学3年生 8歳 の次女と参加しました。本社で開催され10組以上の親子が参加。久しぶりに会うメンバーもいて最初からテンションUp。  
私達親子は「ボー人間ドッジボール」――棒人間たちが体育館で5対5のドッジボール対決をする、というゲームを考えました。

## 8歳のアイデアを、プロンプトにする

子どもが口で話す「こんなゲームにしたい」を、そのままAIに投げてとりあえず実行！  
Claudeが整理し、詳しい仕様を一緒に考えてくれました。

* ボールは1個だけ、当たったらアウト
* ワンバウンドしたボールはセーフで、拾い直せる
* 当たる瞬間にクリックすると「ナイスキャッチ」でボールを奪える
* 最初にボールを持つチームはじゃんけんで決める
* 制限時間は60秒。時間切れは残り人数が多いほうの勝ち
* 難易度（かんたん／普通／難しい）を選べる

HTML・CSS・JavaScript一式のブラウザゲームが一発で返ってきました。当たり判定や、矢印キーを同時押しすると斜めに投げられる角度計算まで、動くコードとしてその場で出てくる様子を見て、誰より子ども自身が驚いていました。

```
// 当たり判定のイメージ(Claudeが生成したロジックを簡略化)
function checkHit(ball, player) {
  const dx = ball.x - player.x;
  const dy = ball.y - player.y;
  const dist = Math.sqrt(dx * dx + dy * dy);
  return dist < HIT_RADIUS && !ball.isBounced;
}
```

## 遊んでは直す、直しては遊ぶ

コードが動いて終わり、ではありません。実際にプレイしながら、

* 「ボールが速すぎる」
* 「キャッチできる時間をもう少し長くしたい」

といった感想がそのまま次の要望になり、また子どもの言葉をプロンプトに直して反映していきます。仕様変更のたびに数十行のコードが書き直され、その場ですぐに試せる。「思いついたことがすぐ形になる」という体験を、子どもは何度も味わっていました。

## 子どもも「いち早くキャッチ・適材適所・全社で実践」していた

「いち早くキャッチ・適材適所・全社で実践」――正直、自分たちで作った方針スライドを見返しても「まあそうだよね」くらいにしか思っていませんでした。でも8歳児は、説明されるでもなく、ドッジボールを作りながら勝手にそれをやっていたんです。

SE出身の私としては、正直「まず仕様をちゃんと固めてから作ろうよ」と言いたくなる場面が何度もありましたが「とりあえず実行」して、動いたものを見ながら直していくほうが、結局早いし楽しい！

AIは「難しい人だけの道具」じゃない。使う人を選ばず、思いついたことをすぐ試せる状態にしておくこと。それが私の目指す全社AI活用の姿で、今回のHWS AI Kids Clubはその縮図でした。

次回のHWS AI Kids Clubでも、子どもたちがどんなアイデアをAIと形にしていくのか楽しみです。
