ぶっちゃけ最初、全然使いこなせてなかった。

「要約して」「翻訳して」── それだけ。なんか高性能なコピペツールみたいな使い方してた。で、「AIってそんなすごくなくない？」ってちょっとがっかりしてた時期がある。

あれから3ヶ月、**毎日触り続けた結果。**

仕事のやり方が、めちゃくちゃ変わった。

---

## なぜプロンプトの質が「全て」を決めるのか

AIって、聞き方次第で返ってくるものが全然違う。

「メールを書いて」と言えば、なんかフワッとした文章が返ってくる。  
でも「取引先への納期延期のお詫びメール。相手は50代の部長。丁寧だけど言い訳がましくならないように」と言うと、そのまま使えるレベルのものが出てくる。

この差、わかる？

**AIはあなたの「意図」を読めない。だから、背景・目的・条件をセットで伝える必要がある。**

最初にこれを知っておくだけで、使える人と使えない人に分かれる。

そして今回は、私が3ヶ月で実際に試した中から「本当に使えた」プロンプトだけを15個、厳選してまとめた。

全部コピペできる形式で書いてあるので、そのまま使ってほしい。

---

## 無料で読める：まず1個だけ試してほしいプロンプト

### 🔥 企画書の構成を30秒で出す

**こう入力する：**

```
あなたはベテランの企画書ライターです。
以下の条件で企画書の構成（章立て）だけを作ってください。

・サービス名：[サービス名]
・ターゲット：[例：30代の働く女性]
・解決したい課題：[例：忙しくて料理する時間がない]
・提案したいこと：[例：週3回届く時短ミールキット]

構成は5〜7項目で、各項目に2行以内の説明をつけてください。
```

**実際に使ってみたら：**

30秒で構成が出てきた。しかも「競合との差別化」「導入コスト試算」みたいな、自分じゃ後回しにしがちな項目まで入ってた。

叩き台ができると、あとは「ここ違う」「もっとこうして」って会話しながら仕上げられる。一人でぼーっと悩む時間が、対話する時間に変わった。**これがでかい。**

## 【有料級】本当に使えたプロンプト15選

### ── BLOCK 01：仕事効率化（5個）──

---

### ① 企画書の構成を30秒で出す（再掲・完全版）

```
あなたはベテランの企画書ライターです。
以下の条件で企画書の構成（章立て）だけを作ってください。

・サービス名：[サービス名]
・ターゲット：[例：30代の働く女性]
・解決したい課題：[例：忙しくて料理する時間がない]
・提案したいこと：[例：週3回届く時短ミールキット]

構成は5〜7項目で、各項目に2行以内の説明をつけてください。
```

**ポイント：** 最初から完璧を求めない。まず構成だけ出して、そこから肉付けしていくのが一番速い。

---

### ② 議事録を3行に要約する

```
以下の議事録を読んで、3行で要約してください。

条件：
・決定事項だけに絞る
・誰が何をいつまでにやるかを明確にする
・ビジネス文書として使えるような文体で

【議事録】
[ここに貼り付け]
```

**ポイント：** 「要約して」だけじゃなく「何を残すか」まで指定すると、使える要約になる。

---

### ③ メールの返信を3パターン作る

```
以下のメールに対する返信を3パターン作ってください。

パターンA：丁寧・かしこまった文体
パターンB：やや柔らかく、親しみやすい文体
パターンC：簡潔・要点だけ伝えるビジネス文体

【受け取ったメール】
[ここに貼り付け]

【自分のスタンス】
[例：今週中の対応は難しいが、来週には対応できる]
```

**ポイント：** 3パターン出してもらうと、相手や状況に合わせて選べる。ゼロから書くより圧倒的に速い。

---

### ④ 報告資料のタイトルを10案出す

```
以下の報告内容に合う、資料タイトルを10案考えてください。

条件：
・上司や役員に見せる資料を想定
・「読みたい」と思わせるタイトルにする
・数字や実績が入るとなおよい

【報告内容の概要】
[例：Q3の営業成績。目標比110%達成。新規顧客獲得が主な要因]
```

**ポイント：** タイトルって地味に時間かかる。10案出してもらって、いいとこどりするのが楽。

---

### ⑤ 自分の強みを言語化してもらう

```
以下の経験・実績をもとに、私の「強み」を3つ言語化してください。

条件：
・採用面接や自己PRで使えるレベルにする
・抽象的すぎず、具体的すぎない表現で
・各強みに、一言の補足説明をつける

【私の経験・実績】
[例：営業5年。新規開拓メイン。チームリーダー経験あり。MVP受賞2回]
```

**ポイント：** 自分のことって自分で一番わからなかったりする。第三者視点で整理してもらうと気づきがある。

---

### ── BLOCK 02：副業・フリーランス向け（5個）──

---

### ⑥ 提案書のたたき台を作る

```
以下の条件でクライアントへの提案書のたたき台を作ってください。

・クライアントの課題：[例：SNSのフォロワーが増えない]
・私が提供できること：[例：月4本のInstagram運用代行]
・金額感：[例：月3万円〜]
・クライアントの業種：[例：地方の飲食店]

構成は「課題の整理→提案内容→期待できる効果→料金→進め方」でお願いします。
```

**ポイント：** 提案書って最初の一文が一番難しい。たたき台があるだけで作業時間が半分以下になる。

---

### ⑦ SNS投稿を1記事から10本展開する

```
以下のブログ記事・note記事をもとに、SNS投稿を10本作ってください。

条件：
・Twitter/X向け（140文字以内）
・それぞれ切り口を変える（驚き・共感・疑問・Tips・体験談など）
・ハッシュタグは各投稿の最後に2〜3個つける

【元記事】
[ここに貼り付け]
```

**ポイント：** 1記事書いたら10本のツイートに変換できる。発信のコスパが一気に上がる。

---

### ⑧ 単価交渉のメール文を作る

```
以下の状況で、クライアントへ単価交渉するメール文を作ってください。

・現在の単価：[例：月2万円]
・希望単価：[例：月3万円]
・交渉の理由：[例：稼働時間が当初より増えた。実績もある]
・関係性：[例：取引6ヶ月。関係は良好]

条件：
・嫌われない、丁寧な文体で
・「値上げ要求」ではなく「再確認のお願い」という雰囲気で
・断られた場合の代替案も一文添える
```

**ポイント：** 単価交渉って感情的になりやすい。AIに客観的な文章を作ってもらうと、冷静に送れる。

---

### ⑨ ポートフォリオの自己紹介文を書く

```
以下の情報をもとに、ポートフォリオに載せる自己紹介文を書いてください。

条件：
・300文字程度
・「依頼してみたい」と思わせる文体
・スキルの羅列ではなく、どんな人かが伝わるように

【プロフィール情報】
・職種：[例：Webデザイナー]
・経験年数：[例：3年]
・得意なこと：[例：LP制作・バナーデザイン・Figma]
・大切にしていること：[例：クライアントの意図を汲み取ること]
```

**ポイント：** 自己紹介文って、自分で書くと謙遜しすぎるか自慢になりすぎるかのどちらかになる。AIに書いてもらうとちょうどいいバランスになる。

---

### ⑩ クライアントへのヒアリング質問を設計する

```
以下の職種・案件に対して、初回ヒアリングで使える質問リストを作ってください。

・私の職種：[例：Webライター]
・案件内容：[例：企業ブログの記事制作]
・ヒアリングの目的：[例：方向性のズレをなくす・手戻りを防ぐ]

質問は10〜15個。「絶対聞くべきもの」と「できれば聞きたいもの」に分けて出してください。
```

**ポイント：** ヒアリング不足が手戻りの9割の原因。聞くべきことを事前に整理しておくだけで、仕事の質が上がる。

### ── BLOCK 03：note継続・発信向け（5個）──

---

### ⑪ ネタ切れしない記事テーマを毎週出す

```
私はnoteで[テーマ：例：AIと仕事術]について発信しています。

以下の条件で、来月分の記事テーマを8本出してください。

条件：
・読者は[例：副業・フリーランスに興味がある20〜30代]
・「スキ」されやすいタイトルにする
・体験談・ノウハウ・失敗談をバランスよく混ぜる
・有料記事に発展できそうなテーマを2本含める
```

**ポイント：** ネタ出しって意外と時間かかる。月1回これをやっておくと、「今週何書こう」って悩まなくなる。

---

### ⑫ 書いた記事のタイトルを10案出す

```
以下の記事内容に合う、noteのタイトルを10案出してください。

条件：
・「読みたい」と思わせるタイトル
・数字・体験・意外性のどれかを入れる
・長すぎず、30文字以内が理想
・バズりやすいパターン（「〇〇したら△△だった」「〇〇の人が知らない△△」など）も含める

【記事の概要】
[例：AIツールを使って仕事の効率が上がった体験談。具体的なプロンプトも紹介]
```

**ポイント：** タイトルで読まれる数が3倍変わる。10案出して一番ピンとくるやつを選ぶだけでいい。

---

### ⑬ 読者が「スキ」を押したくなる締め方を作る

```
以下の記事の締めくくり（最後の200文字程度）を3パターン書いてください。

パターンA：読者に問いかけるタイプ
パターンB：次回への期待感を持たせるタイプ
パターンC：共感・感情に訴えるタイプ

【記事テーマ】
[例：AIを使って仕事が変わった体験談]

【記事で伝えたかったこと】
[例：AIは怖くない。使い方次第で武器になる]
```

**ポイント：** 締め方がぼんやりしてる記事は「スキ」されにくい。感情が動く終わり方にするだけで変わる。

---

### ⑭ プロフィール文を刷新する

```
noteのプロフィール文を書き直してください。

条件：
・150〜200文字
・「この人の記事を読みたい」と思わせる文体
・何を発信しているかが一目でわかる
・親しみやすさと信頼感を両立させる

【現在のプロフィール or 自己紹介情報】
[ここに貼り付け or 箇条書きで入力]
```

**ポイント：** プロフィールはフォロワー増加に直結する。でも自分で書くと客観視できない。定期的にAIに書き直してもらうのがおすすめ。

---

### ⑮ コメント返信を自然な口調で作る

```
以下のコメントへの返信文を書いてください。

条件：
・フランクすぎず、でも堅くもない自然な口調
・コメントの内容に具体的に言及する
・次の記事も読みたくなるような一言を添える
・100文字以内

【もらったコメント】
[ここに貼り付け]

【私のnoteのトーン】
[例：フランク・体験談重視・AI×仕事術がテーマ]
```

**ポイント：** コメント返信って地味に時間かかる。でも返信するかしないかで、読者との関係が全然変わる。これで一気に楽になった。

## まとめ：プロンプトは「会話」だと思え

結局、AIをうまく使えてる人と使えてない人の差は一つだけ。

**「命令してるか、会話してるか。」**

「要約して」じゃなくて「私が明日の会議で使うために、この文章の要点を3つ教えて」って言う。

背景・目的・条件をセットで伝えると、返ってくるものが全然違う。

最初はめんどくさく感じるかもしれない。でも一度コツをつかむと、もう戻れなくなる。
