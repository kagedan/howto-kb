**この記事を読むと、「MCPって最近よく聞くけど何？」が解消される。コードは一切出てこない。AIをちょっと触ったことがある人なら誰でも理解できる内容にした。**

## 3行まとめ

* MCPは「AIに手足を生やす仕組み」
* これがあると、AIがGmail・カレンダー・ファイル管理などを**直接操作**できるようになる
* 2026年4月現在、IT業界で最もホットな技術の一つ

## そもそもAIの「困りごと」

ChatGPTやClaudeを使ったことがある人は、こんな経験がないだろうか。

> 「来週の会議の予定をGoogleカレンダーに入れて」  
> → AI「申し訳ありませんが、Googleカレンダーに直接アクセスすることはできません」

> 「このExcelファイルの売上データをグラフにして」  
> → AI「ファイルを直接開くことができないため、データを貼り付けてください」

これがAIの最大の弱点だった。**AIは「考える」のは得意だけど、「動く」ことができない。**

頭はめちゃくちゃ良いのに、手も足も出ない。すごく優秀な人が、椅子に縛り付けられた状態で仕事しているようなもの。

## MCPは「AIの手足」

MCP（Model Context Protocol）は、この問題を解決するための仕組み。

ひとことで言うと：

> **AIが外のアプリやサービスを「直接」操作できるようにする、共通のルール（プロトコル）**

MCPがあると、こうなる：

| MCPなし | MCPあり |
| --- | --- |
| 「カレンダーに予定入れて」→「できません」 | 「カレンダーに予定入れて」→ **実際に予定が入る** |
| 「メール確認して」→「アクセスできません」 | 「メール確認して」→ **未読メール3件あります** |
| 「ファイル作って」→「コードだけ出力します」 | 「ファイル作って」→ **デスクトップにファイルが出現** |

つまり、AIが「考えるだけの存在」から「実際に仕事をしてくれるアシスタント」に進化する。

## 身近なモノで例えると

### スマホの「USB-C」と同じ発想

MCPはよく\*\*「AIのUSB-C」\*\*と呼ばれる。

USB-Cが登場する前を思い出してほしい。iPhoneの充電ケーブル、Androidの充電ケーブル、カメラのケーブル、全部バラバラだった。旅行にケーブルを3本持ち歩いていた人もいるはず。

USB-Cはそれを**1本に統一**した。

MCPも同じ。AIとアプリをつなぐ方法がバラバラだったのを、**1つの共通ルール**で統一した。

```
【USB-Cの世界】
USB-C ─── スマホ
       ├── パソコン
       ├── イヤホン
       └── モニター

【MCPの世界】
MCP ─── Gmail
    ├── Googleカレンダー
    ├── Slack
    ├── GitHub
    └── ファイル管理
```

### もう一つの例え：「通訳」

AIとアプリは、そのままでは言葉が通じない。

* AIは「日本語（自然言語）」で考える
* アプリは「API」という独自の言語で動く

MCPは、この2つの間に立つ**通訳**。AIが「メール送って」と言ったら、MCPがそれをGmailのAPIに翻訳して実行してくれる。

## 具体的に何ができるの？

MCPを使うと、AIにこんなことを頼めるようになる。

### 仕事の場面

* 「今日の会議予定を教えて」→ Googleカレンダーを確認して回答
* 「田中さんにメール書いて」→ 下書きを作成してGmailに保存
* 「先週のSlackで出た課題をまとめて」→ Slackのメッセージを検索して要約
* 「このバグの詳細をGitHubのIssueに登録して」→ GitHub上にIssueを作成

### プライベートの場面

* 「今週の天気を調べてメモに保存して」→ 天気を検索してメモアプリに書き込み
* 「明日の出張ルートを調べて」→ 地図サービスで検索

### ポイント：AIに「やって」と言うだけ

従来は「AIに聞く → 回答をコピー → 自分でアプリを開いて操作」という手順が必要だった。MCPがあると「AIに言うだけ」で完結する。この差はでかい。

## MCPの登場人物は3人だけ

MCPの仕組みはシンプルで、登場人物は3人。

```
┌──────────┐    ┌──────────────┐    ┌──────────┐
│  あなた   │───→│  AIアプリ     │───→│  外部サービス │
│          │    │（ホスト）     │    │           │
│ 「メール  │    │  Claude等が   │    │  Gmail     │
│  送って」 │    │  MCPで接続    │    │  Slack     │
│          │    │              │    │  カレンダー  │
└──────────┘    └──────────────┘    └──────────┘
                  ↑ここにMCPの仕組みが入っている
```

| 登場人物 | 役割 | 具体例 |
| --- | --- | --- |
| **ホスト** | AIが動くアプリ本体 | Claude Desktop、ChatGPT |
| **MCPクライアント** | ホストの中にいる「接続係」 | （ユーザーからは見えない） |
| **MCPサーバー** | 外部サービスとの「窓口」 | Gmail用、Slack用、GitHub用など |

「MCPサーバー」という名前がちょっと紛らわしいけど、\*\*「特定のアプリとAIを繋ぐアダプター」\*\*と思えばOK。Gmail用のアダプター、Slack用のアダプター、GitHub用のアダプター……という感じで、サービスごとにアダプターが存在する。

## どれくらい広まってるの？

数字で見ると、MCPの勢いがわかる。

* **2024年11月**: Anthropic（Claudeの開発会社）がMCPを公開
* **2025年3月**: OpenAI（ChatGPTの開発会社）がMCP対応を発表
* **2025年12月**: Linux Foundationに寄贈。業界の共有財産になった
* **2026年3月**: 月間**9,700万回**ダウンロード、公開サーバー**13,000個以上**
* **対応企業**: OpenAI、Google、Microsoft、Amazon、Cloudflareなど

要するに、AI業界の主要プレイヤーが全員乗っかっている。「一時的な流行」ではなく、**AIの基盤技術として定着する流れ**。

## MCPを体験してみよう（コード不要）

「理屈はわかった、実際に触りたい」という人向け。プログラミング不要で体験できる方法がある。

### 方法1: Claude Desktopを使う（最も簡単）

1. [Claude Desktop](https://claude.ai/download)をインストール
2. 設定画面からMCPサーバーを追加（Filesystem等）
3. 「デスクトップにあるファイルの一覧を見せて」と話しかける

これだけで、AIがパソコンのファイルを読んでくれる体験ができる。

### 方法2: Claude.aiのWeb版を使う

Claude.aiのWeb版にも、Google Drive・Gmail・Googleカレンダー・Notion・Slack等のMCP連携が組み込まれている。アカウント連携するだけで使える。

### 方法3: ChatGPTを使う

ChatGPTもMCP対応を進めている。「GPTs」や「Actions」の機能が近い体験を提供する。

## 「で、自分の仕事にどう関係あるの？」

### エンジニアじゃない人へ

MCPを直接触る必要はない。ただ、**今後使うAIツールの裏側でMCPが動いている**ことが増える。

例えば：

* 会社が導入したAIアシスタントが、社内のSlackやGoogleカレンダーと連携している → 裏でMCPが動いている可能性が高い
* ChatGPTやClaudeで「外部サービスと接続」できる機能 → MCPベース

「MCPって聞いたことある」というだけで、AI関連の話についていきやすくなる。

### エンジニアの人へ

MCPサーバーの自作は驚くほど簡単。Pythonなら `pip install fastmcp` して、関数にデコレータをつけるだけで動く。自分の社内ツールをAIに繋ぎたいなら、MCPが最短ルート。

## よくある疑問

### Q: セキュリティは大丈夫？

MCPはユーザーの許可なくアプリを操作しない設計になっている。AIが「メールを送りたい」と判断しても、ユーザーが「OK」と言わないと実行されない。2026年のロードマップではOAuth 2.1認証や監査ログなど、企業向けのセキュリティ機能がさらに強化される予定。

### Q: 無料で使える？

MCPの仕様自体はオープンソースで無料。ただし、AIサービス（Claude、ChatGPT等）の利用料は別途かかる場合がある。

### Q: ChatGPTでもClaudeでも使えるの？

はい。MCPは特定のAIに縛られない「共通規格」。USB-Cがどのメーカーのスマホでも使えるように、MCPもどのAIでも使える。
