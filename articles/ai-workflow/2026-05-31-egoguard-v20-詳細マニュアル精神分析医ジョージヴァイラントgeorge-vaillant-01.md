## EgoGuard v2.0 詳細マニュアル

## （DSQ-40 15問・AIナラティブ対応システム仕様＆運用ガイド）

## 1. システム概要とアーキテクチャ

### 1.1 プロダクトの本質

EgoGuard v2.0 は、精神分析医ジョージ・ヴァイラント（George Vaillant）の防衛機制階層説、およびHayashiら（2004）による「DSQ-40（Defense Style Questionnaire）日本語版」の知見をベースに構築された、モダンな自己理解・心理アセスメント・Webアプリケーションです。

ユーザーが無意識に発動させている「防衛機制（精神の自己防衛システム）」を15の質問から1.0〜5.0の生平均値として算出し、自己適応の現状を可視化します。さらに、ユーザーが任意入力した「現在の状況（テキストデータ）」と「計算された防衛スコア」を掛け合わせ、LLM（Claude-Sonnet）を介して超パーソナライズされた動的ナラティブ（分析文）を出力するシステム構造を持っています。

### 1.2 技術スタックと依存関係

* **フロントエンド：** バニラJavaScript（ES6+） / HTML5 / CSS3（CSSカスタムプロパティによる動的テーマ制御、ダークモード対応）
* **データ可視化エンジン：** Chart.js v4.4.1 (レーダーチャート生成に使用)
* **外部API連携：** Anthropic Claude API（claude-sonnet-4-20250514 搭載）
* **フォント・アイコン：** 外部CSS経由で Tabler Icons (ti-) を一部参照

### 1.3 カラーパレットとUIシステム（デザインシステムプロトコル）

:root 変数によって管理されるセマンティック・カラーシステムは、ライトモードとダークモード（@media (prefers-color-scheme: dark)）の双方に最適化されています。ユーザーの視覚的疲労を軽減しつつ、データの心理的属性（成熟、神経症、未熟）を直感的に識別できるよう設計されています。

* **基本テーマ・アクセント：**
* **防衛機制の「適応階層」別テーマカラー：**

  + **成熟（Mature）：** --teal: [#1D9E75](https://note.com/hashtag/1D9E75) ; / 背景：--teal-light。安全・成長・適応を象徴。
  + **神経症的（Neurotic）：** --amber: [#BA7517](https://note.com/hashtag/BA7517) ; / 背景：--amber-light。注意・自己正当化・葛藤を象徴。
  + **未熟（Immature）：** --warm: [#D85A30](https://note.com/hashtag/D85A30) ; / 背景：--warm-light。警告・原始的防御・ストレス高負荷を象徴。

### 2.1 成熟した防衛機制（Mature Factors / 5種）

高いスコアであるほど現実への適応度が高いことを示します。

1. **昇華（Sublimation）**
2. **ユーモア（Humor）**
3. **利他主義（Altruism）**
4. **抑制（Suppression）**
5. **予期（Anticipation）**

### 2.2 神経症的防衛機制（Neurotic Factors / 5種）

自我と超自我（道徳心）の葛藤。過剰発動は認知の歪みを引き起こします。

6. **合理化（Rationalization）**

1. **知性化（Intellectualization）**
2. **抑圧（Repression）**
3. **置き換え（Displacement）**
4. **反動形成（Reaction Formation）**

### 2.3 未熟な防衛機制（Immature Factors / 5種）

現実を著しく歪曲する原始的な防衛。精神的エネルギーの消耗が最も激しい階層です。

11. **否認（Denial）**

\* ID / 理論適応度：denial / 20

\* 挙動定義：目の前にある圧倒的な現実、苦痛な事実、脅威そのものを「存在しない」として認識することを拒否する原始的な機制。

12. **投影（Projection）**

\* ID / 理論適応度：projection / 18

\* 挙動定義：自分自身の中に存在する、受け入れがたい感情（敵意、嫉妬、悪意など）を、自分が持っているのではなく「相手が自分に向けている」と思い込む機制。

13. **分裂（Splitting）**

\* ID / 理論適応度：splitting / 15

\* 挙動定義：他者や自己、または世界を「完全に良いもの」と「完全に悪いもの」のどちらか一方のみで評価し、中間（灰色）の複雑さを統合できない二項対立の機制。

14. **退行（Regression）**

\* ID / 理論適応度：regression / 22

\* 挙動定義：実許容困難なストレスに直面した際、心理的発達段階を以前のレベル（幼児期や子供っぽい振る舞い、癇癪、過度の依存、あるいはドカ食い・過眠など）に巻き戻す機制。

15. **受動的攻撃（Passive Aggression）**

\* ID / 理論適応度：passive\_aggression / 12

\* 挙動定義：相手に対する怒りや敵意を言語で直接表現せず、遅刻、忘れ物、わざと仕事を遅らせる、皮肉、無視といった間接的な「不作為・非協力的行動」によって表現する機制。

## 3. スコアリングアルゴリズムと数理ロジック

本システムは、Hayashiら（2004）に準拠した数理処理を実行します。

### 3.1 生平均値（Raw Average Scores）の算出

ユーザーが質問1〜15に対し、ライカート尺度（1 = 全く違う 〜 5 = 非常に当てはまる）で回答したデータ（ans 配列）をスキャンします。各防衛機制IDに紐づく質問項目の合計値をその有効数（n=1）で除算し、**1.0から5.0の生平均値**を保持します。

### 3.2 因子パーセンテージ（Factor Percentages）の正規化

レーダーチャートおよびバー表示用に、1.0〜5.0の範囲をとる因子平均値を「0〜100%」のレンジへ線形正規化します。

数式は以下の通りです。

* mPct（成熟因子%）：mAvg（成熟5種の生平均）から変換
* nPct（神経症因子%）：nAvg（神経症5種の生平均）から変換
* iPct（未熟因子%）：iAvg（未熟5種の生平均）から変換

### 3.3 全体成熟度（ODF: Overall Defensive Functioning）の重み付け計算

全体適応度を示すODFスコアは、ヴァイラントの階層臨床評価に準拠し、成熟因子が高いほど高得点、神経症的および未熟因子が「低い（発動していない）」ほど高得点になるよう、以下の数式で加重平均を算出します（0〜100点満点）。

## 5. 外部API（AIナラティブ）連携プロトコル

結果画面内の「AIパーソナライズド分析を生成」ボタン（#ai-btn）がクリックされると、非同期通信（async function runAI()）が走り、プロフェッショナルな臨床プロトコルに基づいた分析が生成されます。

### 5.1 APIリクエスト仕様

* **エンドポイント：** POST https://api.anthropic.com/v1/messages
* **使用モデル：** claude-sonnet-4-20250514
* **ヘッダー：** 'Content-Type': 'application/json'（運用時は個別にAPI認証用のトークンヘッダーが必要）

### 5.2 プロンプトインジェクション防御とシステムプロンプト（System Prompt）

LLMの出力を厳格に制御し、ハルシネーションを防ぐため、バックエンド（コード内）で以下のシステム指示文が固定インジェクションされます。

> **【システムプロンプト要件】**

精神分析ベースの心理的洞察を提供する専門アシスタントとして振る舞うこと。ユーザーの状況とスコアリングを統合し、Hayashi et al. (2004)のDSQ-40の文脈で解釈すること。温かく、非判断的（ノン・ジャッジメンタル）で、科学的根拠に基づくこと。必ず以下の「厳格な3段落構成」で出力すること：**第1段落：** 現在の心理状態の簡潔な言語化（必ず「あなたは今〜」で開始、1〜2文）。**第2段落：** 最も活性化している防衛機制の意味と、それが守ろうとしているもの（不安や自我の痛み）の解釈（2〜3文）。**第3段落：** 成熟型へのシフトのための具体的かつ実践可能な1ステップ（必ず「今日できることとして〜」で開始、1〜2文）。最終行に「本ツールは補助ツールである」旨の免責を1行で添えること。

### 5.3 ユーザーデータ（ペイロード）の結合形式

通信時、textarea#sit に入力された現実のコンテキストと、JavaScriptのスコアリング関数が弾き出した数理データが以下のようにプレーンテキストとして整形され、LLMへ送信されます。

Plaintext

```
状況：[ユーザーが入力した文字列、空欄の場合は（記入なし）]
成熟因子平均：[mAvg], 神経症的因子平均：[nAvg], 未熟因子平均：[iAvg]
ODF相当スコア：[odf]/100
推定上位防衛機制：[上位3つの機制名、英語名、Rawスコアの羅列]
```

### 5.4 UIの例外処理・ローディング制御

* 通信開始時、div#aibox に .loading クラスが付与され、CSSアニメーション（@keyframes spin）によって右上にインジケータ（回転する円）が描画され、二重リクエストを防ぐためにボタンがディセーブル（disabled=true）化される。
* エラー発生時（ネットワーク切断、API制限等）は、catch(e) ブロックが作動し、警告カラー（--warm = オレンジレッド）で「AIとの接続でエラーが発生しました」というエラーメッセージを安全に出力し、システム全体のフリーズを回避する。

## 6. 運用のためのチェックリスト・トラブルシューティング

保守担当者およびサービス運用者が、本システムを安全にデプロイ・稼働させるためのチェックプロトコルです。

### 6.1 デプロイ前確認チェックリスト

* [ ] **Chart.js のCDN生存確認：**

  1. https://cdnjs.cloudflare.com/ajax/libs/Chart.js/4.4.1/chart.umd.js が正常にロードできているか。これが遮断されている場合、結果画面のレンダリング全体がストップします。
* [ ] **APIキーの秘匿性（最重要）：**

  1. 現在のコードはフロントエンドから fetch を直接呼び出す構造（モック・ローカル検証用仕様）になっています。商用環境へ移行する際は、クライアントサイドにAPIキーが露出するのを防ぐため、必ず自社のサーバーサイド（Node.js, Python等）にエンドポイントを中継（プロキシ化）させてください。
* [ ] **レスポンシブ・表示領域の確認：**

  1. 最小ハイト（min-height: 620px;）が確保されているか、スマートフォン表示時にレーダーチャートのCanvasサイズ（width="260" height="200"）がはみ出していないか確認してください。

### 6.2 トラブルシューティング（想定されるバグと対処プロトコル）

* **現象：質問への回答ボタン（1〜5）を押しても「次へ →」ボタンが活性化しない**
* **現象：AI分析を生成しようとすると、文字が真っ赤（またはオレンジ）になり「エラーが発生しました」と出る**

  + 原因： AnthropicのAPIキーが未設定であるか、CORS（Cross-Origin Resource Sharing）制限によってブラウザがリクエストをブロックしています。
  + 対処： サーバーサイドプロキシを構築し、APIリクエストをバックエンド経由に修正してください。また、システムプロンプト内のモデル名（claude-sonnet-4-20250514）が最新の有効なモデルIDであるか確認してください。
* **現象：結果画面で、レーダーチャートが重なって描画される、または更新されない**
