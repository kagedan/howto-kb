AIに文章を書かせると、どこかぎこちない仕上がりになる。「まず〜、次に〜、最後に〜」「〜かもしれません」「参考になれば幸いです」──読んだ瞬間、人間が書いた文章じゃないとわかってしまう。

OpenClawでX投稿・note記事・Slack通知まで全業務をエージェントに任せて3ヶ月。この記事では、その過程で気づいた「AIっぽさの正体」と、プロンプト設計で根本から解決する具体的な方法を全部書きます。文体を変えるより先にやるべきことがあります。

## 「AIっぽい」と感じる瞬間はどこか ─ 違和感のポイントを分解する

AIが書いた文章を読んでいて「なんか違うな」と感じる瞬間、具体的にどこで感じているか。

実際に観察してきたパターンを整理すると、だいたい3箇所に集中しています。

### ① 冒頭の問いかけ「〜していませんか？」

「毎日のタスク管理に悩んでいませんか？」

「AIを使いこなせていますか？」

この形が出てきた瞬間、読むのをやめたくなります。AIに「記事を書いて」と投げると、なぜかこの冒頭が量産される。

### ② 均等な3段構成

「まず〜、次に〜、最後に〜」

「1点目は〜、2点目は〜、3点目は〜」

内容の重さに関係なく、すべての段落が同じ分量・同じ重さで並ぶ。人間が書く文章には波がありますが、AIの出力には波がない。

### ③ 締めの抽象化

「ぜひ活用してみてください」

「一歩踏み出してみましょう」

「この知識が役に立てば幸いです」

具体的な話をしてきたのに、最後だけ急に抽象的になる。これがもっとも「AIっぽさ」を感じさせるポイントだと思っています。

## 文体の問題ではなかった ─ 「AIっぽさ」の正体はプロンプトの構造にある

最初は文体の問題だと思っていました。「もっと自然な口調で」「ですます調で」と指示を重ねても、根本的には変わらない。

気づいたのは、問題が「何を書くか」ではなく「どういう型で考えるか」にあることでした。

AIは明示的な指示がないとき、「記事らしい記事の型」に引っ張られます。学習データの中で最頻出のパターン──冒頭の問いかけ・均等な3段構成・締めの抽象化──がデフォルトになっている。

つまり、文体をどう指示しても、「考える構造」を変えないと同じパターンが出続けます。

## 全業務をAIエージェントで回して気づいた3つのパターン

OpenClawでエージェントを動かし続けて、「AIっぽさ」が出やすい状況には共通点があることがわかりました。

### パターン1: ゴールが曖昧なプロンプト

「SNS投稿を書いて」だと、AIは「SNS投稿らしい文章」の型を参照します。

「Xでエンジニア向けに、具体的な数字を入れた280文字の投稿を書いて」まで絞ると、型ではなくゴールから考え始める。

ゴールの解像度が上がるほど、AIは型から離れます。

### パターン2: 役割が設定されていない

指示だけあって「誰として書くか」がないとき、AIは無人格な文体になります。

後述するSOUL.mdに「一人称：僕、口調：ですます+断言ミックス」と書いてから、出力がまったく変わりました。

人格の設定は、文体指示よりはるかに効く。

### パターン3: 禁止事項が設定されていない

「自然に書いて」と言っても、AIにとって「自然」の基準がありません。

「参考になれば幸いです・〜してみてはいかがでしょうか・〜かもしれません」という禁止ワードリストを渡すほうが、はるかに効きます。

「何をするか」より「何をしないか」の制約のほうがAIには伝わりやすい。

## OpenClawで「AIっぽさ」を消す実装例 ─ SOUL.mdとbrand\_voice.mdの設計術

OpenClawはワークスペースのMarkdownファイルをエージェントに自動的に読み込ませる仕組みを持っています。

僕の構成はこうなっています。

```
workspace/
├── SOUL.md                   ← エージェントの人格・口調・価値観
└── sns-strategy/
    └── brand_voice.md        ← 禁止ワード・文体サンプル・世界観
```

### SOUL.mdに書いていること

```
## 口調
- 一人称：僕
- 読者への呼びかけ：みんな
- 文体：ですます基調 + 断言ミックス（「〜できました。正直、戻る気がしない。」）
- 熱量：抑えめ。テンション高め・感情的な表現は使わない
- 証拠：数字・スクショ・コードを積極的に出す。語りだけで終わらせない
```

### brand\_voice.mdの禁止ワードリスト（抜粋）

```
## 禁止ワード
- 「〜かもしれません」「〜だと思います」「参考になれば幸いです」
- 「まず〜、次に〜、最後に〜」（型が先行している感）
- 「一歩踏み出す勇気を」「可能性は無限大です」「人生が変わりました」
```

これをシステムプロンプトに毎回埋め込むのではなく、ファイルとして管理してエージェントに読み込ませています。更新すれば全ジョブに即反映されます。

## Before/After：改善前と改善後の出力を比較する

\*\*改善前（SOUL.mdなし）:\*\*

> AIツールを活用することで、業務効率が向上する可能性があります。まず現状の課題を整理し、次に適切なツールを選択し、最後に運用フローを構築することが重要です。ぜひ参考にしてみてください。

\*\*改善後（SOUL.md + brand\_voice.md読み込み済み）:\*\*

> NoteFlowを動かし始めて3ヶ月が経ちました。今は1日2本のnote記事が手を動かさずに出ています。リサーチ・執筆・投稿の3ステップをすべてエージェントに委ねた結果、コードの量はゼロ。プロンプトの設計だけで動いています。

同じ「AI自動化の成果」を書いても、出力のトーンがまったく変わります。変えたのは文体指示ではなく、読み込ませるファイルの内容です。

## まとめ ─ AIらしさを消すのではなく、人らしさを設計する

AIっぽさの原因は文体ではなく、プロンプトの構造にあります。「ゴールを具体化する・役割を明示する・禁止ワードを設定する」この3点を押さえるだけで出力が変わります。OpenClawのSOUL.mdとbrand\_voice.mdをファイルで管理することで、全エージェントに一括適用できます。「もっと自然に」と言い続けるより、型を渡したほうが早い。
