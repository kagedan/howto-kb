こんばんわ🌙

自分のための備忘も兼ねてまとめてみました(*´∀｀*)

GPTで画像生成していると、たまに

- なんかガビガビする
- ザラザラする
- 変なキラキラが増える
- 細かい模様が勝手に出る
みたいなことありませんか？

![](https://pbs.twimg.com/media/HIg4u1zbcAAThb6.jpg)

これ、内部すかしが原因というより、
**AIが「描き込みを頑張りすぎている」**可能性が高そうです。

特に、プロンプトに

・高精細
・緻密な描き込み
・複雑な質感
・キラキラ
・リアルな肌
・細かい髪の毛
・豪華な背景
・光の粒
・テクスチャ多め

みたいな言葉が入っていると、AIは
**「もっと細かく描けばいいんだ！」**
と解釈しやすいです。

一般的な画像生成AIだとよく入れるキーワードですよね。

でも、その細かさが行きすぎると、
肌、髪、服、背景、影の部分に、
ザラザラした粒や、謎のキラキラ、細かすぎる線が増えてしまうことがあります。

これが、いわゆる
**ガビガビ感**
につながっているのかなと思っています。

特にアニメ調やフラットなイラストは、この影響がかなり目立ちます。

アニメ絵って、本来は

・太めの輪郭線
・なめらかな面
・整理された影
・読みやすいシルエット

が大事です。

そこにAIが勝手に
「質感！粒！光！細部！情報量！」
を足しすぎると、絵が急に汚れて見えます。

つまり、ガビガビを減らしたい時は、
**描き込みを増やすより、情報量を整理する**
のが大事です。

ちなみに、C2PAなどの来歴情報、いわゆるメタデータが原因で、画像がガビガビになる可能性は低そうです。

これは画像の出所を示すための情報で、基本的には
**見た目にザラザラを焼き込むものではなく、ファイルに付く情報**
と考えた方が近いです。

なので疑うべきは、

・プロンプトで質感を盛りすぎていないか
・細かい装飾を入れすぎていないか
・参考画像がすでに荒れていないか
・何度も編集してノイズを引き継いでいないか
・JPEG保存やSNS投稿で圧縮されていないか

このあたりです。

私が今いちばん効くと思っているのは、
AIに対して

**「そこまで頑張って描き込まなくていいよ」**
**「なめらかで見やすくして」**
と明確に伝えることです。

たとえば、いつものプロンプトの最後にこれを足します。

## 【軽めの改善プロンプト】

```
複雑な質感や細かすぎる描き込みは抑え、なめらかでクリーンな質感を重視してください。
粒状ノイズ、ザラつき、不要なキラキラ表現は避け、輪郭線は太くはっきり描いてください。
```

もう少ししっかり抑えたい時はこちら。

## 

## 【標準版】

```
高密度なテクスチャ表現や微細な装飾は控えめにし、なめらかで整理された質感を優先してください。
粒状ノイズ、ザラつき、砂目、過剰な光沢、不要なキラキラやちらつくハイライトは避けてください。
輪郭線は太くクリーンにし、キャラクターや物体のシルエットが一目でわかるようにしてください。
影やハイライトはシンプルにまとめ、フラットで見やすい仕上がりにしてください。
```

かなりガビガビしやすい時は、さらに強めに。

## 

## 【強め版】

```
この画像は、質感の多さよりも形の読みやすさを優先してください。
複雑な質感、細かすぎる描き込み、粒状ノイズ、ザラつき、過剰な光沢、微細なキラキラ、ランダムな模様の増殖は抑えてください。
面はなめらかでクリーンに、輪郭線は太く安定した線で描いてください。影は少なめで単純化し、フラット寄りの塗りにしてください。
背景も情報量を増やしすぎず、整理された構成にしてください。
全体として、ノイズの少ない、すっきりした、SNS投稿後も荒れにくい仕上がりにしてください。
```

## 

## 【むぎがよく使うもの】

私が実際によく使うのは以下のキーワードです。

```
仕上げはフラットでクリーンに。
複雑な微細テクスチャ、過剰な細密描写、粒状ノイズ、ザラつき、斜めグリッド状の模様、タイル状の繰り返し、泥っぽい汚れのオーバーレイ、不要なキラキラ表現は抑えてください。
「intricate textures」の要素は弱め、なめらかで均一な面、太めで明確な輪郭線、読みやすいシルエットを優先してください。
細部を増やすよりも、形の明快さ、余白、構図の整理、清潔な画面を重視してください。
```

例えば、テーマを適当ですが「ゴスロリ、女の子、ドラグーンランス、ダイナミック、背景格
