Gartnerは米国に本拠を置く世界最大級のITリサーチ＆アドバイザリ企業であり、企業のIT投資判断・技術採用ロードマップに最も強い影響力を持つ存在だ。

ガバナンスとリスク管理の欠如により、運用を停止または縮小せざるを得なくなる。**」と驚きの発表をした。

## AIエージェントが停止・縮小に追い込まれる理由

この予測の背景には、企業が直面する 5つの構造的リスク がある。

**１．権限管理の不備かつエージェントがやりすぎる問題**

AIエージェントは、メール送信・ファイル操作・システム更新など、従来は人間が行っていた業務を自律的に実行する。

しかし多くの企業では、権限の粒度・範囲・監査ログが整備されていない。

その結果、  
意図しないデータアクセス  
不適切なファイル削除  
誤った意思決定の自動実行  
といった事故が発生し、運用停止に追い込まれる。

**２．スキル過剰付与：万能化させすぎて制御不能に**

最新研究では、「エージェントにスキルを与えすぎると、逆に成功率が下がる」とが示されている。

つまりスキルが多いほど  
どのスキルを使うべきかの判断が難しくなる  
意図しない行動パスが増える  
テスト範囲が爆発的に拡大する  
結果として、企業は「安全に動かせない」という理由で縮小を余儀なくされる。

**３．データガバナンスの欠如：内部情報の“自動漏洩”リスク**

エージェントは社内データを横断的に扱うため、アクセス制御・分類・保持ルールが未整備だと重大な情報漏洩につながる。

特に問題となるのは、  
個人情報の自動処理  
社外送信の自律実行  
機密文書の誤った共有

など、従来のDLPでは想定していなかった**自律行動型リスク**である。

**４．説明責任の欠如：事故時に「誰が責任を取るのか」問題**

AIエージェントは人間の指示なしに行動するため、事故発生時に 責任の所在が曖昧 になりやすい。  
エージェントの判断か  
モデルのバイアスか  
スキル設計の問題か  
データの誤りか  
この曖昧さが、法務・監査部門の強い反対を招き、結果として運用停止につながる。

**５．評価・監査の仕組み不足：動作保証ができない**

エージェントは複雑なタスクを実行するため、従来の「単体テスト」「結合テスト」では品質保証ができない。

Gartnerは**Adaptive Evaluation（適応型評価）** が必須になると指摘しているが、多くの企業はまだ導入できていない。

## 停止・縮小を避けるための対策

以下は Gartnerの推奨とMicrosoft の最新アーキテクチャ（Agent 365 / Entra ID / Foundry）を踏まえた実践的な対策である。

**（A）エージェントのID化と権限の最小化**

Microsoft の Agent 365 はこの領域を強くカバーする。

**(B）スキル設計のガイドライン化**  
スキルは 少数・高精度 に絞る  
スキル間の依存関係を明確化  
万能エージェントを作らない  
スキル追加時は必ずリスク評価を実施  
これにはMicrosoftが公開したスキル設計フレームワークを活用することが推奨される。

**(C）評価・監査の自動化：Adaptive Evaluation の導入**

エージェントは実行環境やデータに応じて挙動が変わるため、  
静的テストでは安全性を保証できない。

必要なのは、  
実行ログの自動解析  
リスク行動の検知  
失敗パターンの学習  
継続的な改善ループ  
Microsoft Foundry の Adaptive Evaluations はこの領域をカバーする。

## M365 Copilotの視点

M365 Copilotは、Gartnerが警告する**ガバナンス不在による40%停止リスク**を回避するための、最も現実的な安全地帯となるだろう。

理由はCopilotが Microsoft 365の権限モデル・Entra ID・監査ログと完全統合され、企業の既存ガバナンスをそのまま継承できる点にある。

さらにCopilotは自律エージェントではなく**制御されたアシスタント**として設計されており、勝手に行動せず、すべての操作がユーザー主体で記録される。

この構造が、エージェント導入で最も問題となる「権限過剰」「説明責任の欠如」「データアクセスの暴走」を根本から抑制する。

2028年に向け、企業が段階的にエージェントへ移行する際、M365 Copilot は安全な入口として機能するだろう。

## まとめ

Gartnerの**40%が停止・縮小**という予測は、単なる悲観論ではなく、企業がエージェント導入で直面する現実的なリスクの指摘である。

そしてこの予測は、Agent 365、Entra ID、Foundry、MAI モデル といった Microsoft の最新アーキテクチャが**なぜ今必要なのか**を裏付ける強力な根拠となります。
