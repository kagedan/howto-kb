4章かけてバラバラに作ってきたパーツを、全部つないだ。

![チャット応答デモ](https://static.zenn.studio/user-upload/deployed-images/0e1c2aed4408d2ea6f279609.gif?sha=74f0a0c75868ebaf675897966e70f8542e5f15a3)  
*テキストを送ると、1〜2秒でアバターが喋り始める。口が動いて、体が揺れて、表情が変わる。*

この記事は「結局どうなったの？」に答える回です。個々の技術の詳細は第1〜4章に書いたので、ここではシステム全体を俯瞰します。

## 全体像

ユーザーが「おはよう」と送ると、裏で4つの処理が順番に走ります。

構造としてはWebのBFFパターンに近いです。APIを直列に叩いて、レスポンスをまとめてフロントに返す。

### どこに時間がかかっているか

ローカルの処理は全部合わせて300ms以下。ボトルネックはClaude APIのネットワーク往復です。

## サーバー構成

Pythonサーバー2つと、HTMLを返すだけの静的サーバー1つです。GPUのVRAMは合計約8.6GB。

## 各パーツの役割

### Claude API — 会話と感情

`claude-haiku-4-5`にJSON出力を強制して、返答テキストと「感情ラベル」を同時に取得します。

```
{"text": "おはよう！今日は何するの？", "emotion": "happy", "emotion_intensity": 0.7}
```

この感情ラベルが、後でアバターの表情に反映されます。

### GPT-SoVITS — 声

友達に8分間喋ってもらった音声でファインチューンしたモデルです。テキストを入れると、**その友達の声で**音声が出てきます。完全ローカル、API代ゼロ。

HTTPのPOSTで叩くだけなので、普通のREST APIと同じ感覚で使えます。

### LAM-ja — 口の動き

音声を入れると、毎秒30回、口のパーツ52箇所の「開き具合」が0〜1の数値で出てきます。

52箇所はAppleのFace Tracking規格（ARKit）です。これを「あいうえお」5母音に変換してVRMに反映します。

```
aa = clamp(jawOpen * 1.3 + smile * 0.2)   // あ — あごが開いたら
ih = clamp(stretch * 0.8 + smile * 0.3)    // い — 口が横に伸びたら
ou = clamp(max(mouthPucker, mouthFunnel))   // う — 口がすぼまったら
```

### BEAT2 — 体のジェスチャー

[第3章](https://zenn.dev/reiyaitatani/articles/ai-avatar-gesture-generation)でAIにジェスチャーを生成させる試みは3手法とも失敗しました。代わりに、モーションキャプチャのデータから4秒のクリップをランダムに選んで連結しています。

つなぎ目はCSS `transition`的な滑らかブレンドで処理。AI推論ゼロ、2ms。

### 感情 → 表情

Claudeの感情ラベルをアバターの表情に反映します。いきなり切り替えず、約100msかけて「じわっと」変わります。

## 会話していない間の「生きてる感」

![アイドルアニメーション](https://static.zenn.studio/user-upload/deployed-images/736f2524b044de947d67a10d.gif?sha=dede837013f7b3f4b7a33eedfc49123c83f3a9e7)

実はここが一番時間かかりました。**黙っているときの自然な動き**をどう作るか。

やっていることは、6種類のノイズを重ね合わせているだけです。

ポイントは、**全ての周期を互いに割り切れない数にすること**。こうすると波形が永遠にループしません。いつ見ても違う動きになります。

これに加えて、8〜23秒ごとにランダムな「癖」も入ります。

| 癖 | 確率 | 見た目 |
| --- | --- | --- |
| 首傾げ | 35% | くいっと首を傾ける。微かに笑顔 |
| ため息 | 15% | 深く吸い込む。少し悲しい顔 |
| 表情変化 | 50% | 微笑み / 口すぼめ / 眉上げ / 驚き / 考え事 |

まばたきは3〜7秒間隔。視線はマウスカーソルを追いかけるモードと正面を見るモードを交互に切り替えています。

## 音声と口の同期

サーバーから音声・口の動き・ジェスチャーを**1パケットで**受け取り、ブラウザで同期再生します。

`AudioContext.currentTime`（音声ハードウェアの時計）を使うことで、`requestAnimationFrame`の時計より正確に音声と口を同期できます。

## 起動

```
bash start.sh
# → http://localhost:8080/viewer/chat.html
```

3サーバーが立ち上がります。Ctrl+Cで全部止まります。

## ここまでの振り返り

第1章で描いた技術マップがどこまで埋まったか。

| 要素 | 第1章 | 今 | 技術 |
| --- | --- | --- | --- |
| 口の動き | ★ | ✅ | LAM-ja |
| 表情 | 未 | ✅ | Claude感情ラベル |
| 視線・まばたき | 未 | ✅ | プロシージャル |
| ジェスチャー | 未 | ✅ | BEAT2クリップ |
| 声 | 未 | ✅ | GPT-SoVITS |
| アイドル | 未 | ✅ | 6層ノイズ |
| 全部つなげる | — | ✅ | WebSocket + AudioContext |

パイプラインは一通り動くようになりました。

まだ荒い部分はあります。ジェスチャーは感情に関係なくランダム選択だし、音声データも8分しかないので声質の再現度にムラがある。でも「テキストを送ったらアバターが喋る」という体験は、思ったより自然でした。

コードは整理でき次第GitHubで公開予定です。
