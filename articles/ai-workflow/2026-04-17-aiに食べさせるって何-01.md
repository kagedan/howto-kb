## AIに資料を食べさせると、何が起きるのか

前回、NotebookLMで「自分の資料を入れる」という話をしました。

「でも実際に何ができるの？」

わからなくて当然です。 だってその悩み、AIに聞けばいいんですから。

今回はその答えを、具体的にお見せします。

## 「食べさせる」って何？

AIに資料を渡すことを、私は「食べさせる」と表現しています。

難しく考える必要はありません。

```
PDFを渡す　→　AIが読む
Wordを渡す　→　AIが読む
写真を渡す　→　AIが読む
```

渡した瞬間から、**AIはその資料の専門家になります。**

「この資料について何でも聞いてください」という状態になるイメージです。

## 1つの資料を食べさせる

まずは1つの資料を渡すだけでできることから見てみましょう。

### STEP1｜議事録を渡して要点をまとめる

```
渡すもの：会議の議事録
聞くこと：「決定事項と次のアクションをまとめて」
結果　　：整理されたTo-Doリストが返ってくる
```

### STEP2｜契約書を渡して注意点を抽出する

```
渡すもの：契約書のPDF
聞くこと：「気をつけるべき条項はどこ？」
結果　　：専門用語を噛み砕いて説明してくれる
```

### STEP3｜マニュアルを渡して新人向けに噛み砕く

```
渡すもの：社内マニュアル
聞くこと：「新入社員が最初に覚えるべきことは？」
結果　　：初心者向けにわかりやすく整理してくれる
```

## 2つの資料を統合・分析する

ここからが本番です。

**2つの資料を同時に食べさせると、AIが比較・統合・分析までやってくれます。**

### STEP1｜2つの見積もりを比較分析する

```
渡すもの：A社とB社の見積書
聞くこと：「どちらがコスパがいい？違いを教えて」
結果　　：項目ごとの比較表が返ってくる
```

### STEP2｜先月と今月の報告書を比べる

```
渡すもの：先月・今月の業務報告書
聞くこと：「何が改善されて何が課題として残っている？」
結果　　：変化点と課題が整理されて返ってくる
```

### STEP3｜バラバラな議事録を一つにまとめる

```
渡すもの：複数回の会議議事録
聞くこと：「プロジェクトの進捗を時系列でまとめて」
結果　　：一本のプロジェクト報告書が完成する
```

## 分析結果をそのまま資料化する

分析が終わったら、そのまま資料にできます。

```
「この分析結果を報告書の形にして」
「表にまとめて」
「箇条書きにして」
「上司に説明するための資料にして」
```

一言添えるだけで、**分析→資料化まで一気に完結します。**

## 実際の事例

**事例①：社内制度の資料**

あるクライアントが、頭の中にあった構想をAIに話しかけながら整理して、本格的な社内制度の資料を作り上げました。

「見てください！」と送ってくださったとき、正直感動しました。このnoteの記事を読んで実践してくださった結果が、ここまでの資料になるんだと。

**事例②：VideoBase調査報告書（第4話）**

展示会で見たサービスの情報と、既存ツールの情報をAIに食べさせて比較分析。その結果を調査報告書として資料化しました。

月10万円の問題が2,000円で解決できたのも、この「食べさせる→分析→資料化」の流れがあったからです。

## まとめ

* AIに資料を「食べさせる」＝渡すだけでOK
* 1つの資料なら要約・抽出・噛み砕きができる
* 2つ以上の資料なら比較・統合・分析までできる
* 分析結果はそのまま資料化できる

## まず今日やってみること

手元にある資料を2つ選んでAIに渡してみてください。

> 「この2つを比べて、違いをまとめて」

それだけでOKです。
