## はじめに

「スーパーで3000円使った」

これだけ言えば、家計簿がつけられたら最高じゃないか。

そう思って作ったのが **家計簿MCP** だ。  
UIも、アプリも、フォームも、ボタンも一切ない。  
Claude（AI）とチャットするだけで、複式簿記の家計簿がつけられる。

---

## デモを先に見せる

こんな感じで使える。

```
私: 「今日スーパーで3000円使った」

Claude: ✅ 仕訳を記録しました
         借方: 食費   3,000円
         貸方: 現金   3,000円
```

```
私: 「今月の収支見せて」

Claude: ## 2026年6月 収支

        収入合計  300,000円
        支出合計   10,200円
        差引     289,800円（黒字）

        費用内訳
        ・食費    9,000円
        ・交通費  1,200円
```

```
私: 「残高試算表を見せて」

Claude: 【残高試算表】2026-06

        ■ 資産
          現金          残高:  290,800円
          普通預金      残高:  300,000円

        ■ 収益
          給与収入      残高:  300,000円

        ■ 費用
          食費          残高:    9,000円
          交通費        残高:    1,200円
```

これが全部、チャットだけで動く。

## なぜ複式簿記なのか

一般的な家計簿アプリは**単式簿記**だ。

これだけ記録する。シンプルでわかりやすい。

でも複式簿記はこうなる。

```
借方: 食費  3,000円
貸方: 現金  3,000円
```

「食費が増えた」と「現金が減った」を同時に記録する。

なぜこれが良いか？**資産全体の動きが見えるから**だ。

* 現金がいくらあるか
* 銀行残高がいくらか
* クレジットカードの未払いがいくらか
* 全部まとめた純資産はいくらか

これが正確に把握できる。単式の家計簿アプリでは、真の意味での「資産管理」は難しい。

ただし複式簿記には問題がある。**難しい**のだ。

「借方」「貸方」「仕訳」という概念を理解しないと入力できない。だから一般には普及しない。

---

## AIがその問題を解決する

「スーパーで3000円使った」

これを聞いたAIは自動的に考える。

* 支出だから費用が増える
* 費用は借方で増える → 借方: 食費
* 現金払いなら現金が減る
* 資産は貸方で減る → 貸方: 現金

ユーザーは複式簿記を知らなくていい。**普通に話すだけで正確な仕訳が記録される。**

これが家計簿MCPの核心だ。

---

## MCPとは何か

MCP（Model Context Protocol）は、AIにツールを持たせる仕組みだ。

普通のChatGPTやClaudeは「話すだけ」のAIだ。情報を与えたり、アドバイスをもらったりできる。でも何かを「実行」はできない。

MCPを使うと、AIが外部のツールやシステムを直接操作できるようになる。

```
ユーザー → Claude → MCPサーバー → データ保存・読み込み
```

家計簿MCPの場合：

1. ユーザーが「スーパーで3000円」とチャット
2. Claudeが「これは食費の仕訳だ」と判断
3. MCPサーバーの「仕訳記録」ツールを呼び出す
4. JSONファイルに保存される

## 作ったもの

技術スタックはシンプルだ。

* **言語**: Python
* **MCPライブラリ**: `mcp`（Anthropic公式）
* **データ保存**: JSON（サーバー不要）
* **行数**: 約300行

```
kakeibo_mcp/
  server.py      ← MCPサーバー（ツール定義）
  kakeibo.py     ← 複式簿記エンジン
  data/
    kakeibo.json ← データファイル
```

### 用意したツール

server.py

```
@mcp.tool()
def 仕訳記録(日付, 借方科目, 貸方科目, 金額, 摘要):
    """仕訳を1件記録する"""

@mcp.tool()
def 仕訳一覧(年月):
    """月別仕訳を表示する"""

@mcp.tool()
def 残高試算表(年月):
    """残高試算表を表示する"""

@mcp.tool()
def 月次収支(年月):
    """月の収支サマリーを返す"""

@mcp.tool()
def 科目追加(科目名, 種別):
    """新しい勘定科目を追加する"""
```

たったこれだけ。あとはAIが全部やってくれる。

## 使ってみた感想

**驚いたのは、入力が本当に楽なこと。**

「今日コンビニで800円使った」と言えばいい。  
「先月末の電気代22,000円引き落とし」と言えばいい。  
「給与が振り込まれた」と言えばいい。

Claudeが文脈を読んで、適切な科目を選んで記録してくれる。

間違えても「さっきの仕訳を消して」で削除できる。  
「電気代の科目名は光熱費にして」と言えばその通りになる。

## どのLLMでも使えるはず

今回はClaudeで試したが、**MCPに対応しているLLMであれば理論上は何でも使える**。

| LLM | 公式MCP発表 | 動作確認 |
| --- | --- | --- |
| Claude（Anthropic） | ✅ あり | ✅ 確認済み |
| ChatGPT（OpenAI） | ✅ あり | ⚠️ 未確認 |
| Gemini（Google） | ✅ あり | ⚠️ 未確認 |

家計簿MCPサーバー自体はただのPythonスクリプトなので、LLM側を変えても動くはず。
