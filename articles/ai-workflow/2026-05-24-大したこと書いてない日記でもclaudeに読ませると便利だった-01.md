家庭菜園の記録をどこに書くか、ずっと迷っていた。

最初は手帳に書いていた。次にTimeTreeやカレンダーアプリを試した。入力を忘れることが多く、気づくと「いつ植え付けたっけ」と記憶を頼りに適当にやっていた。日記も試したが、続いたことがない。1日で終わることもあった。

Obsidianに落ち着いたのは、書いたものが蓄積されていくのが見えるからだと思う。アプリを開くと過去のノートが全部残っていて、日が経つにつれて増えていく。管理しようと気負わなくていい感じが、自分には合っていた。iCloudでMacとiPhoneを同期しているので、畑でiPhoneを開いてさっとメモして、家に帰ってMacで見る、という流れも自然にできた。

書いている内容は大したことではない。

```
[[トマト]]12株植え付け
[[ナス]]3、[[ピーマン]]2、[[シシトウ]]1も同日
```

こんなもんだ。

## ClaudeをObsidianに繋いだ

Claude DesktopからObsidianを読めるようにしてみた。MCPという仕組みを使うと、Claude DesktopからローカルのObsidian Vaultを読めるようになる。

設定自体はそこまで難しくなかった。HomebrewとNode.jsを入れて、Claude Desktopの設定ファイルにVaultのパスを書くだけだった。

```
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
brew install node
```

設定ファイル（~/Library/Application Support/Claude/claude\_desktop\_config.json）に以下を保存。

```
{
  "mcpServers": {
    "obsidian": {
      "command": "/opt/homebrew/bin/npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/your/vault"]
    }
  }
}
```

/path/to/your/vault は自分のVaultのパスに書き換える。npxはフルパスで書かないとClaude Desktopが見つけられずにエラーになる。これだけ注意すれば、あとはClaude Desktopを再起動するだけで繋がった。

## 何ができるようになったか

「トマトについて書いてあるノートを探して」と聞くと、トマト.mdの内容を返してくれる。

さらに「デイリーノートの方にも書いてるかも」と言うと、デイリーノートを日付指定で読みに行って、5月5日に12株植えたこと、同じ日にナスやピーマンも植えたことまで拾ってくれた。デイリーノートを横断してキーワード検索もできるので、「トマトが出てくる日を全部探して」という使い方もできる。

読むだけでなく、ノートへの追記もできる。デイリーノートから拾った情報を「トマト.mdにまとめて」と頼むと、整理してファイルに追記してくれる。株数を訂正しながら確認したら、その場で反映もしてくれた。

## 正直なところ、思った以上に便利だった。

見返すのが面倒で放置していた記録が、ちゃんと使えるものになった。その場では大したことを書いているつもりはない。でも、「いつ植えたっけ」と記憶を頼るよりは、雑でも残っている方がずっと良かった。

いつ植えたか、何株だったか、どのタイミングで伸び始めたか。前なら記憶を頼りに適当にやっていたことが、あとから普通に引き出せるようになった。

秋にサツマイモを掘るころには、また雑なメモが少し増えていると思う。
