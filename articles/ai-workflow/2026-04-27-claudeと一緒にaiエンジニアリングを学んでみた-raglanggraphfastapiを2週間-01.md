## はじめに

AIエンジニアリングを学びたいと思いつつ、「何から始めればいいかわからない」「学習コストが高そう」と感じていました。

そこで試してみたのが、**Claudeを学習パートナーとして使う**という方法です。ただ質問するだけでなく、一緒にコードを書きながら「なぜこの設計なのか」まで深掘りしていく学び方です。

結果として2週間ほどで以下が実装できるようになりました。

- RAGチャットボット（ChromaDB + HuggingFace Embedding）
- LangGraphを使ったマルチステップエージェント
- FastAPIでのAPIサーバー構築
- SQL生成AIツール

この記事では、**Claudeとの具体的な学び方・活用法**をまとめます。

## なぜClaudeを学習パートナーにしたのか

最初は書籍やUdemyで学ぼうとしていましたが、2つの問題がありました。

**① 詰まったときの解消が遅い**
エラーが出るたびにGoogle検索 → Stack Overflow → ドキュメントを行き来するのが時間のロスでした。

**② 「なぜそうするのか」がわからない**
コードのコピペはできても、設計の意図や他の選択肢との比較が学べませんでした。

Claudeと一緒に学ぶと、この2つが解決されます。エラーはそのまま貼れば即解決、「なぜ？」という疑問にもその場で答えてもらえます。

---

## 実際の学び方：5つのポイント

### 1. エラーはそのまま全文貼る

詰まったときは、エラーメッセージを**全文そのまま**貼ります。

```
# こういうエラーが出ました
anthropic.BadRequestError: Error code: 400 - {'type': 'error', 
'error': {'type': 'invalid_request_error', 
'message': 'messages.0.content: Field required'}}
```

「エラーが出ました」だけより、全文貼った方が原因を正確に特定してもらえます。今回は `content` が `context` になっていたタイポでした。

### 2. 実装の前に「なぜ」を聞く

コードを書く前に必ず概念を確認します。

```
「LangGraphとLangChainの違いは何ですか？
なぜLangGraphが必要なんですか？」
```

これをやると「なんとなく動いた」ではなく「意図して選んだ」状態になります。副業案件の面談で「なぜこの技術を選んだか」を聞かれたときに答えられるようになります。

### 3. 疑問はその場で全部聞く

コードを読んでいて「これは何をしているの？」という疑問が出たらすぐ聞きます。

たとえばこのコード：

```python
is_sufficient = "不十分" not in response.content
```

「`not in` は何をしている？」とその場で聞けば、次のステップに進む前に理解できます。わからないまま進むと後で詰まります。

### 4. 設計判断ログを残す

実装のたびに「なぜこの技術を選んだか」をREADMEに記録しています。

```markdown
### なぜHuggingFaceのEmbeddingを使ったか
- OpenAIのAPI quotaが不足していたため
- multilingual-e5-smallは日本語対応・コスト0
- トレードオフ：初回モデルダウンロードが必要
```

これが副業案件の面談で話せるエピソードになります。「動くものを作った」だけでなく「意思決定ができるエンジニア」として見てもらえます。

### 5. 学習ログをREADMEに追記する

毎回の学習後にREADMEに以下を書いています。

```markdown
#### やったこと
#### 学んだこと
#### 疑問と回答
#### 設計判断
#### 次にやること
```

これをやると3つの効果があります。

- **定着する**：書くことで理解が深まる
- **振り返れる**：「あのときどう実装したっけ」がすぐわかる
- **ポートフォリオになる**：GitHubのREADMEが学習の証跡になる

## 2週間で作ったもの

### 1. 社内FAQ AIアシスタント

RAG + LangGraph + FastAPIで作った社内ドキュメント検索ツールです。

```
質問 → RAGで関連ドキュメントを検索 → 回答生成 → 評価 → 不十分なら再生成
```

ポイントは**LangGraphで評価・リトライのループ**を実装したことです。「回答が十分か」をLLMに評価させて、不十分なら再生成します。

### 2. SQL生成AIツール

自然言語の質問からSQLを生成・実行するAPIツールです。

```
「売上が一番高い商品は？」
→ SELECT p.name FROM products JOIN sales ...
→ 「ぶどうです」
```

SQLエラーが出たときはエラーメッセージをプロンプトに含めてリトライするので、**LLMが自己修正**できます。

詳細はこちらの記事でまとめてます。

## Claudeと学ぶときの注意点

### モデルはHaikuで十分

学習中のコード実行にはclaude-haiku-4-5で十分です。Opusは高いので必要なときだけ使いましょう。

### APIクレジットは先に買っておく

学習中にクレジット切れになると中断します。事前に購入しておくとスムーズです。（自分は途中で止まりました）

### 「なぜ」を聞き続ける

Claudeは聞けば何でも答えてくれますが、聞かないと教えてくれません。コードを見て「わかった気」になるより、疑問を全部吐き出す方が定着します。
