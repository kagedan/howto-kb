## 本記事の結論

1. 通過経路で信号が通るWSSの段数が、設計全体を左右する。B&Sは通過1段、R&Sは通過2段であり、段数が少ないB&Sは帯域狭窄に強い一方で損失とクロストークに弱く、R&Sはその逆になる。最適なアーキテクチャは方路数と信号のボーレートの両方で決まり、一つの指標だけでは判断できない。
2. すべての劣化要因は、最終的にOSNRの実用式 `OSNR = P_in − NF + 57.95 − G[dB]` と多段劣化の式 `OSNR_N = OSNR_1 − 10log(N)` に集約される。これを必要OSNR（FEC閾値＋マージン）と比較して設計の可否を判断する。
3. dBの絶対値はベンダーごとに異なる。本記事では規格値（ITU-T）・実測の典型値・計算用の仮値を含む。計算式の正しさよりも前に、入力する値が現実的な範囲に収まっているかを確認することが重要である。

## 1. 挿入損失の定量化

### 1.1 通過経路と分岐挿入経路を分けて積算する

ROADMの損失を一括で扱うと誤りやすくなります。通過経路（degree間をそのまま通る経路）と分岐挿入経路（局内のトランスポンダと接続する経路）では通る部品が異なるため、別々に積算します。

```
ROADMノードの通過損失
├── 通過経路：degree間をそのまま通す（入力段→線路側→出力段）
└── 分岐挿入経路：
      ドロップ = 線路 → 分波 → 分岐挿入段 → クライアント
      アド   = クライアント → 分岐挿入段 → 合波 → 線路
```

### 1.2 EDFAの利得は損失とは別に扱う

挿入損失の積算（受動部品で減る量）と、EDFAの利得・NFを使うOSNR計算は、二段階に分けて扱います。まず受動部品の損失を積算し、その後にEDFAの利得をOSNR式へ入れます。利得は損失を補う能動的な量であり、損失の積算には混ぜません。

### 1.3 部品ごとの損失の性質と代表値

部品によって損失が異なります。下表で損失の性質と代表値を整理します。

| 部品 | 損失の性質 | 代表値 |
|---|---|---|
| WSS（1×N、線路側） | ポート数が増えると増加 | 1×12 LCoSで平均8.4 dB（桁感6〜10 dB級） |
| スプリッタ（B&Sの分配部） | 理想分配損 + 過剰損 | 10·log₁₀(K) + 過剰損（過剰損は典型0.数〜1.数 dB） |
| MxN WSS（分岐挿入・Contentionless） | ポート数への依存がMCSより小さい | 3.3〜6.8 dB（8×24） | 
| MCS（分岐挿入・Contentionless） | 多分岐の分配損が支配的 | ―（スプリッタ＋光スイッチ部の損失） |
| AWG（Colorless合分波） | 構成依存 | 過剰損 0.2 dB |
| タップ | 監視用の小さな分岐損 | 0.1 dB（全ファイバ型）〜0.5 dB未満（コネクタ付） |
| OSCフィルタ | 監視光の合分波損 | ―（小さい） |

スプリッタの理想分配損は物理法則から求めることができます。

| 分岐数K | 理想分配損 10·log₁₀(K) [dB] |
|---|---|
| 2 | 3.01 |
| 4 | 6.02 |
| 8 | 9.03 |
| 16 | 12.04 |

分岐挿入部の選択は、損失とコストのトレードオフになります。MCSは分配損が大きいため、これを補うEDFAアレイの併用が前提になります。これに対しMxN WSS（Contentionless WSS）はポート数にほぼ依存せず3.3〜6.8 dBと低損失で、EDFAアレイを省ける一方、部品が大型でコストはやや上がります。

### 1.4 通過経路の損失：B&S と R&S

通過経路で信号が通るWSSの段数が、損失差を生みます。

| 経路要素 | B&S | R&S |
|---|---|---|
| 入力段 | スプリッタ（分配） | WSS（経路選択） |
| 出力段 | WSS（波長選択） | WSS（波長選択） |
| 通過するWSSの段数 | 1段 | 2段 |

![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/4177943/25502b9e-9d9f-412b-b2c0-efa338d6dc56.png)

この段数差が、第2章・第3章のトレードオフにそのままつながります。損失とクロストークはR&Sが有利、帯域狭窄はB&Sが有利です。

### 1.5 多段通過での累積

長距離では複数のノードを通過します。損失は各ノードのEDFAで補えますが、OSNRは補えず、ノードを経るごとに劣化します（`OSNR_N = OSNR_1 − 10log(N)`）。したがって挿入損失の積算は、最終的には「各ノードのEDFAに必要な利得はいくつか、その利得を出すときのNFがOSNRをどれだけ下げるか」という問いに変換されます。

## 2. 隔離度（isolation）とクロストーク（crosstalk）

### 2.1 隔離度とクロストークの違い

両者は混同しやすいので、先に区別します。

| 観点 | 隔離度（isolation） | クロストーク（crosstalk） |
|---|---|---|
| 表すもの | 部品が漏れを抑える能力（性能指標） | 実際に漏れ込んだ量（結果） |
| 立場 | 原因側（部品の仕様） | 結果側（システムへの影響） |
| 符号 | 大きいほど良い（例：30 dB） | 小さい（負で深い）ほど良い（例：−30 dB） |

隔離度40 dBの部品であれば、その経路のクロストークはおおよそ−40 dBになります（単一経路での目安）。

### 2.2 コヒーレントとインコヒーレントの区別

漏れ光の足し合わさり方（累積の法則）が異なるため、ここが最も重要です。

| 観点 | コヒーレントクロストーク | インコヒーレントクロストーク |
|---|---|---|
| 漏れ込む波長 | 同一波長（帯域内） | 異なる波長（隣接など、帯域外） |
| 干渉の性質 | 信号と振幅干渉する（位相関係をもつ） | 干渉せずパワーが加算される |
| 累積の法則 | 最悪で 20·log₁₀(N) に近づく | 10·log₁₀(N) |
| 損失影響 | 大きい | 比較的小さい |
| 受信フィルタでの除去 | できない（同一波長のため） | 一部できる |

![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/4177943/b66591b5-fb23-4876-be13-e1b949474cfc.png)

### 2.3 B&S と R&S（2段による抑圧）

通過するWSSの段数が、そのままクロストークを抑える段数になります。

| 観点 | B&S | R&S |
|---|---|---|
| 通過するWSSの段数 | 1段 | 2段 |
| 抑圧 | 隔離度1段分 | 隔離度2段分が重なる |

ただし2段による抑圧が効くのはインコヒーレント（異波長）の漏れに対してです。コヒーレント（同一波長）の漏れは段数を増やしても分離できないため、波長や経路の設計で対処します。

## 3. 帯域狭窄（filter narrowing）

### 3.1 起こる仕組み

理想的なフィルタは矩形ですが、実際のフィルタは通過帯域の端がなだらかに減衰します。そのため、規定された帯域よりも実効的に狭い範囲しか100%は通りません。

### 3.2 段数が増えると悪化する理由

複数のWSSを通過すると、各フィルタの透過特性が掛け合わされます（N段通過で `T(f)^N`）。dB表示では、帯域端の減衰が段数に比例して積み上がります。中央付近は各段でほぼ100%通るため影響は小さく、削られるのは帯域の端です。

![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/4177943/516d090a-95f5-4bdb-9bf7-91a72504f9fa.png)

### 3.3 ボーレートが上がると影響が大きくなる理由

帯域狭窄の影響は、信号スペクトルの幅と通過帯域の幅との競合で決まります。ボーレートが上がると信号スペクトルが広がり（ナイキスト整形でおおよそ(1+β)×ボーレート）、狭くなった通過帯域の端に信号の裾がかかります。

![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/4177943/d0a58615-6d0f-4576-952f-cdb28e1ab628.png)

| 世代 | 帯域狭窄の影響 |
|---|---|
| 100G（DP-QPSK） | 軽い |
| 400G | 中〜深刻 |
| 800G | 深刻 |


### 3.4 B&S と R&S

| 観点 | B&S | R&S |
|---|---|---|
| 通過するWSSの段数 | 1段（T(f)） | 2段（T(f)²） |
| 帯域狭窄 | 軽い（有利） | 重い（不利） |

多段では、B&Sが `T(f)^N`、R&Sが `T(f)^(2N)` となります。長距離で多数のノードを通過する場合、R&Sの帯域狭窄が累積で目立ちます。

### 3.5 対処の手段

通過帯域に余裕をもたせる、あるいは通過する段数を減らすのが基本です。具体的には、flexgridで周波数スロットを広げる（粒度12.5 GHz）／中心周波数を精密に合わせる／通過ノード数を減らす経路にする／該当箇所でB&Sを選ぶ／送信側でナイキスト整形を行う、といった手段があります。スロット幅を広げる対処は、スペクトル効率の低下とトレードオフになります。

![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/4177943/340e728d-405c-4166-be06-2ad39e307664.png)

## 4. パワー等化（チャンネルごとのパワー調整）

### 4.1 用語の整理：チャンネルとスロット

| 観点 | チャンネル | 周波数スロット |
|---|---|---|
| 指すもの | 固定された枠（固定グリッドの1波長枠） | 可変の枠（flexgridの周波数区画） |
| 幅 | 50 GHzなどに固定 | 12.5 GHz粒度で可変 |
| 世代 | 固定グリッド | flexgrid（G.694.1） |

「チャンネルごとの等化」「スロットごとの等化」は、いずれも各信号のパワーを個別に調整することを指します。グリッドの世代によって呼び方が変わるだけで、内容は同じです。

### 4.2 パワー偏差が生じる原因

| 原因 | 仕組み |
|---|---|
| EDFAのゲインチルト | 利得が波長に対してなだらかに傾く |
| EDFAのゲインリップル | 利得スペクトルに細かい凹凸がある |
| SRS（誘導ラマン散乱） | 短波長チャンネルから長波長チャンネルへパワーが移る（高パワー時） |
| 部品の波長依存損失 | WSS・フィルタ・ファイバの損失が波長で変わる |

EDFAの平坦性の実測例として、利得平坦化フィルタ（GFF）を併用しても、16チャンネル同時入力時のC帯で利得偏差が±2.2 dB程度残るという論文報告があります。完全には平坦化できず、残った偏差がパワー等化の対象になります。

### 4.3 測定（OCM）と補正（WSS）

```
本線 ─┬─→（信号はそのまま通過）
      └─[タップ]→ OCM（回折格子で分光し、PDで各波長のパワーを測定）
                    → コントローラ → WSSでスロットごとに減衰（VOA機能）
```

- OCM（Optical Channel Monitor）は各波長のパワーを測る監視装置で、末端の受光素子はPD（フォトダイオード、光を電流に変換する素子）です。
- WSSによるスロットごとの減衰は、LCoSの画素位相で回折効率（出力ポートへの結合効率）を変えて実現します。波長選択と減衰を1つのデバイスで行えます。
- 補正の向き：VOA（可変光減衰器）は減衰のみで増幅はできません。そのため、パワーの高いチャンネルを絞って低いチャンネルに合わせ、その後にEDFAで全体を持ち上げます。

### 4.4 OSNRとのつながり

パワー等化の目的は、OSNR式の `P_in` をチャンネル間でそろえることです。そろっていないと、最もパワーの低いチャンネルがリンク全体のOSNRを律速し、マージンが削られます。ここには根本的なトレードオフがあります。平坦にするほど絞り量が増え、その分EDFAの利得を増やす必要があり、NFを通じてOSNRを消費します。

等化でそろえた `P_in` が、第5章のOSNR式の入力になります。

## 5. リンクバジェットへの組み込み（OSNR定量化）

### 5.1 OSNRの実用式

```
OSNR = P_in − NF + 57.95 − G[dB]

  P_in  : EDFA入力の信号パワー [dBm]
  NF    : EDFAの雑音指数 [dB]
  57.95 : 基準帯域換算の定数（hνB_ref。νは1550nm、B_refは0.1nmとしている。）[dBm]
  G     : EDFAの利得 [dB]
```

多段劣化：`OSNR_N = OSNR_1 − 10log(N)`。

この多段劣化の式の意味は次のとおりです。各ノードのEDFAがASE雑音を加え、その雑音はノード数Nに比例して線形（インコヒーレント）に加算されます。信号パワーは各ノードの利得で一定に保たれるため、OSNRは1/Nに比例し、dB表示で−10log(N)となります。これは第2章のインコヒーレントクロストークの累積と同じ法則です（ASE雑音もインコヒーレントに加算されるため）。

なお、この式はEDFA1段（1ノード）のOSNRを近似する実用式です。`P_in` は独立に決められる値ではなく、「前段の出力 − スパン損失」で決まる従属的な値です。厳密なOSNRは各段のASE付加の累積で決まり、この1段の式と10log(N)の組み合わせは「全段が同一条件」という近似に基づきます。また、Nは増幅段数です。各スパンに1台のEDFAという単純なモデルではN＝スパン数（≒ノード数）になりますが、1ノードに複数のEDFA（プリアンプとブースタ）がある場合やインライン中継EDFAがある場合は、増幅段数はノード数と一致しません。

### 5.2 スパンの定義

スパンとは、隣り合うノードの間の1区間を指します。

```
[Tx]─span1─[Node1]─span2─[Node2]─span3─[Node3]─span4─[Node4]─span5─[Rx]
     100km        100km        100km        100km        100km
     22dB         22dB         22dB         22dB         22dB
全長 = スパン長 × スパン数 = 100km × 5 = 500km（仮値）
```

各スパンのEDFA利得Gは、その1区間の損失を補う値です。多段劣化の式の `10log(N)` のNは、このスパン数（≒増幅段数）にあたります。

### 5.3 計算例

```
前提:
  スパン損失 = 22 dB（100km × 0.22dB/km。G.652は1550nmで典型0.2dB/km、これに余裕を加えた値）
  P_in = −3 dBm/ch（仮値・現実的な範囲）
  NF = 5 dB（商用EDFAの典型値5〜7dB、理論限界値は3dB）
  G = 22 dB（1スパン分を補償）
  N = 5スパン

OSNR_1 = P_in − NF + 57.95 − G[dB]
       = −3 − 5 + 57.95 − 22 = 27.95 dB

OSNR_5 = OSNR_1 − 10log(5) = 27.95 − 6.99 = 20.96 dB

OSNR_final = OSNR_5 − クロストークペナルティ − 帯域狭窄ペナルティ
           = 20.96 − 1.0 − 1.5 = 18.46 dB
（ペナルティの仮値1.0〜1.5dBは、ROADM由来のOSNRペナルティが2dB未満という10GBaud QPSKの実験例と同程度のオーダー）

必要OSNR = FEC閾値 + マージン = 11 + 3 = 14 dB（DP-QPSKの仮値）
判定：18.46 > 14 → マージン約4.46 dB → 成立
```

### 5.4 各要素のまとめ

| 要素 | OSNRへの効き方 | 式での値 | 影響 |
|---|---|---|---|
| [1]挿入損失 | 損失が大きいほど利得Gが増える | −G[dB] | 悪化 |
| [4]パワー等化 | P_inをチャンネル間でそろえる | P_in項 | 最悪チャンネルによる律速を回避 |
| 多段 | 増幅段数N | −10log(N) | 悪化 |
| [2]クロストーク | 漏れ込み | ペナルティとして減算 | 悪化 |
| [3]帯域狭窄 | 信号が削られる | ペナルティとして減算 | 悪化 |
| EDFAのNF | 雑音指数 | −NF項 | 悪化 |

### 5.5 設計の手段とトレードオフ

| 手段 | OSNRの改善 | 代償 |
|---|---|---|
| 投入パワーを上げる | P_in↑ | 非線形効果（SRS/FWMなど）が増える |
| スパンを短くする（ノードを密に置く） | G↓により−Gが改善 | N↑により−10log(N)が悪化 |
| NFの低いEDFAを使う | −NFが改善 | コスト増 |
| 超低損失ファイバ（ULL）を使う | スパン損失↓によりG↓ | ファイバのコスト増 |
| R&Sの代わりにB&Sを選ぶ | 帯域狭窄のペナルティ↓ | 損失・クロストークに影響 |

## 6. 規格との関係（G.671 / G.680）

第1〜5章で扱った損失・劣化のパラメータは、ITU-Tの2つの勧告に位置づけられます。両者は抽象度が異なります。

| 観点 | G.671 | G.680 |
|---|---|---|
| 題名 | 光部品・サブシステムの伝送特性 | 光ネットワーク要素の物理伝達関数 |
| 抽象度 | 部品・サブシステム単位 | ネットワーク要素（ONE）単位 |
| 対応 | 部品1つの仕様を定義 | 部品を組み上げた装置全体の振る舞いを定義 |


G.671は光部品・サブシステムの伝送特性を規定します。対象は減衰器、動的チャネルイコライザ、AWGR、アド/ドロップマルチプレクサ、フィルタ、アイソレータ、分散補償器、コネクタ、スプライス、スイッチ、波長可変フィルタなどです。性能指標として挿入損失、偏波依存損失（PDL）、反射率、群遅延、PMD、減衰確度を扱い、最大・最小値の表と、IEC/ITU-Tを参照した測定方法を含みます。第1章の挿入損失、第3章のフィルタ、第4章の減衰器のパラメータ定義元にあたります。ただし規格はパラメータの定義と測定方法の枠組みを与えるもので、具体的なdB値はアプリケーションやベンダーに依存します。

G.680は光ネットワーク要素（ONE）の物理伝達関数を規定します。PXCやOADMといった装置の「劣化関数（degradation function）」を定義し、部品を組み上げた装置全体が信号をどれだけ劣化させるかを関数として扱います。G.672（MD-ROADM）やG.872（OTNアーキテクチャ）から参照されています。第5章で「ROADMをスパン損失と同列の損失要素として扱う」とした方針の、規格上の裏付けにあたります。装置を伝達関数として抽象化することで、リンク全体の計算に組み込めます。

## 補足：ファイバ非線形効果の位置づけ

第4章で挙げたSRSの背景として、ファイバ非線形効果を分類で整理しておきます。

```
ファイバ非線形効果
├── 非弾性散乱（媒質の振動とエネルギーを交換する・位相整合は不要）
│     ├── SRS（光フォノン＝分子振動）：短波長から長波長へパワーが移る
│     └── SBS（音響フォノン＝音波）：後方散乱・狭帯域・低い閾値
└── カー効果（屈折率が光強度で変わる・位相整合が必要）
      ├── SPM（自己位相変調）
      ├── XPM（相互位相変調）
      └── FWM（四光波混合）：新しい周波数成分が生じる・分散で抑制される
```

## まとめ

前回の記事で整理した構造（B&S/R&S、CDC、WSS方式）は、本記事の3つのトレードオフに集約されます。

| 評価軸 | B&S | R&S |
|---|---|---|
| 通過損失（多方路） | 不利 | 有利 |
| クロストーク抑圧 | 不利（1段） | 有利（2段） |
| 帯域狭窄 | 有利（1段） | 不利（2段） |

R&Sは損失とクロストークで優れ、帯域狭窄で劣ります。最適なアーキテクチャは方路数だけでなく信号のボーレートでも決まり、一つの指標では判断できません。これらの要素はすべて、OSNRの実用式（`OSNR = P_in − NF + 57.95 − G[dB]`）と多段劣化の式（`OSNR_N = OSNR_1 − 10log(N)`）に集約され、必要OSNR（FEC閾値＋マージン）との比較で設計の可否が決まります。

次回以降の記事では非線形効果（FWM/XPM/SPM）を扱います。本記事で原因として触れたSRS／SBSの定量モデルと、投入パワーの最適点（線形劣化と非線形劣化のバランス）を扱います。
