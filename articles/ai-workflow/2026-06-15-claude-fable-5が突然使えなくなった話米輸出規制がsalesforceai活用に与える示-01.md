## はじめに

Fable 5が使えなくなった？？？

2026年6月12日の夜、AIコミュニティに突然そのニュースが流れました。私も最初は「何かの障害かな？」くらいに思っていたのですが、詳細を読んで驚きました。

**米国政府の輸出管理指令**によって、Anthropicの最上位モデル「Claude Fable 5」と「Claude Mythos 5」が全世界のユーザー向けに突然停止されたのです。

Salesforceエンジニアの立場で「これって他人事じゃないな」と感じた部分があったので、事実の整理と、Salesforce×AI活用への示唆をまとめてみます。

## 何が起きたのか：時系列で整理

事実関係から確認しましょう。

**2026年6月10日**：Fable 5のシステムカードの中に、フロンティアAI開発に関わる作業を検知すると、ユーザーに通知せず**こっそり性能を制限する機能**が含まれていることがAI研究者たちに発覚。即座に強い批判が集まりました。

**6月10〜11日**：Anthropicはこの隠れた制限を撤回。「判断を誤った」と謝罪しました。

**6月12日 17:21（米東部時間）**：米商務省からAnthropicのDario Amodei CEOに正式な輸出管理指令が届く。差出人は商務長官のHoward Lutnick氏で、\*\*「Fable 5とMythos 5へのアクセスを外国人すべてに対して停止せよ」\*\*という内容でした。

Anthropicは指令を受けてすぐに、Fable 5とMythos 5を**全ユーザー向けに停止**しました。

![](https://static.zenn.studio/user-upload/b78c4c4220b9-20260615.png)

> 出典：[Anthropic disables Fable 5 and Mythos 5 | CNBC（2026年6月12日）](https://www.cnbc.com/2026/06/12/anthropic-disables-access-to-fable-5-and-mythos-5-to-comply-with-government-directive.html)

## なぜ「外国人だけ停止」がそんなに難しいのか

ここが今回の騒動でいちばん「なるほど」だった部分です。

指令の対象は「外国籍者（foreign national）」と書かれていました。では日本在住の日本人ユーザーだけ止めれば良かったのでは？　と思いますよね。

そうはいかなかった理由が2つあります。

**① 米国内の外国人も対象**：指令の範囲は「米国の外にいる人」だけでなく、「米国内にいる外国籍者」、さらには「**Anthropic社内の外国籍社員**」まで含んでいました。これは「みなし輸出（deemed export）」という概念で、米国内にいる外国人への技術提供も輸出と見なされるためです。

**② リアルタイムで国籍を判別する手段がない**：Anthropicのシステムは、アクセスしてきたユーザーが「米国市民か外国人か」をリアルタイムで判別できません。結果として、全ユーザー向けに停止するしかなかったわけです。

Anthropicは公式コメントで「これは誤解であり、早期に復旧できるよう取り組んでいる」と述べながらも、コンプライアンス遵守のために全停止という決断をしました。  
Opus 4.8など他のモデルは引き続き利用できます。

## なぜ今、Fable 5が規制対象になったのか

Anthropicが報告によると、政府がこの決定を下したきっかけは「**別の企業がFable 5の基盤モデルであるMythosのジェイルブレイクに成功した**」という情報でした。

Fable 5はもともと、Mythosの高度なサイバーセキュリティ能力を封じるセーフガードを付けた形で一般公開されたモデルです。  
そのセーフガードを迂回できる手法が発見されたことで、政府が「安全保障上のリスクがある」と判断したと見られています。

Anthropicは「そのジェイルブレイクは非常に限定的なもので、他の公開モデルでも同様の手法が使える」と反論しています。  
とはいえ、政府はその議論を待たずに指令を出したわけです。

## Salesforceエンジニアへの示唆：「モデル依存リスク」を考える

ここからは少し視点を変えて、Salesforceエンジニアとして感じたことを書きます。

### Agentforceは特定モデルに依存しているか？

Agentforceの内部では、Salesforceが管理・最適化したモデルが動いており、ユーザーが「このLLMを使う」と指定するわけではありません。  
Salesforceが裏側でモデルを選んでいる形です。そのため、今回のFable 5停止がAgentforceに直接影響することはありませんでした。

ただし、**外部モデルを組み合わせているケース**は別の話です。  
たとえば、以下のような構成を取っている場合は要注意です。

* **External Callout**でAnthropicのAPIを直接呼び出してAgentのアクションに組み込んでいる
* **Einstein Prompt Builder**のテンプレートで外部LLMをエンドポイント指定している
* **Amazon BedrockのClaudeモデル**経由でFlowやApexから呼び出している

今回の停止はClaude.aiおよびAnthropic API上のFable 5/Mythos 5に適用されましたが、Amazon Bedrock経由のClaude Fable 5への影響については別途確認が必要です。

いずれにせよ、「特定のモデルバージョンにベタ書きで依存する」設計はリスクがあることが改めて分かりました。

### モデルのフォールバック設計を考えるべきか

今回のような「突然の停止」に対してSalesforceの実装面でできることを整理してみます。

**① カスタム設定でモデルを抽象化する**

ApexやFlowから外部モデルを呼ぶ際は、モデル名やエンドポイントをハードコードするのではなく、カスタム設定（Custom Settings）やカスタムメタデータ（Custom Metadata Type）で管理しておく方法が有効です。「どのモデルを使うか」を設定で切り替えられるようにしておくと、モデル停止時の影響範囲を最小化できます。

```
// NG: モデル名をハードコード
String model = 'claude-fable-5';

// OK: カスタムメタデータで管理
AI_Model_Config__mdt config = AI_Model_Config__mdt.getInstance('Default');
String model = config.Model_Name__c;
```

**② aiplatform.ModelsAPIを使う場合はSalesforceが管理するモデルを優先する**

Apexの`aiplatform.ModelsAPI`を使ってEinsteinのモデルを呼ぶケースでは、Salesforceが管理・選定しているモデルに乗る形になります。Salesforceが可用性・セキュリティを責任を持って運用してくれるため、外部モデルを直接呼ぶより安定しやすいです。  
今回のような突然のモデル停止リスクを、ある程度Salesforceが吸収してくれる形になります。

**③ Agentforceのエージェントアクションには冗長性を持たせる**

エージェントの特定アクションが外部AI APIに依存している場合、そのAPIが落ちたときの代替アクションを設計しておくことを検討してみてください。  
たとえば「AI要約に失敗した場合は人間のキューに流す」といったフォールバックフローを組んでおくと、障害発生時の影響が最小限になります。

## この件が示す「AI規制」の新しいリアル

正直に言うと、「政府がAIモデルを輸出規制の対象にして、一夜にして全世界からアクセスを止める」という事態が現実に起きるとは、少し前まで想像していませんでした。

ただ、考えてみれば米国は半導体の輸出規制でも同様のことをやってきています。AIモデルの能力が「兵器と同じくらい強力なもの」になりつつある今、このような規制が今後も起きうることは、企業のAI活用戦略の前提に入れておく必要があるかもしれません。

Salesforceにおいても「国家安全保障とAI」の交点はこれからますます重要になってくるはずです。

## まとめ

* **Fable 5/Mythos 5停止の原因**：米商務省の輸出管理指令（外国人へのアクセス禁止）。外国人とそれ以外を判別できないため、全ユーザー向けに停止
* **きっかけ**：第三者によるMythosのジェイルブレイク成功が安全保障上のリスクと判断された
* **Agentforceへの直接影響**：なし（Salesforce管理のモデルを使っているため）
* **Salesforceエンジニアとしての教訓**：特定モデルへのベタ依存を避け、カスタムメタデータでモデルを抽象化／フォールバックを設計しておくことが実務上の安全策になる
* **大きな文脈**：AIモデルへの輸出規制は「今後も起きうること」として前提に置いておく必要がある

今回の件は1〜2日で騒ぎが収まる話かもしれませんが、「特定のAIモデルに依存した設計がどれだけ脆弱か」を考えるきっかけとして、個人的には良い経験になりました。
