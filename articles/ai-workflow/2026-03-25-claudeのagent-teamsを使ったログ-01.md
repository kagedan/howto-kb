## やったこと：

プロダクト企画・開発を支援する6つの役職エージェント（PdM・UXリサーチャー・UXデザイナー・UIデザイナー・エンジニア・テスター）を構築。

企画書やPRDを送ると、テンプレートと照合して不足項目を自動検知し、担当役職付きで次のアクションを提案。プロダクトデザインの専門知識や8つの開発フェーズのプロセス定義を組み込んだ。

1. ドキュメントの種類を判定
2. テンプレートと照合して不足項目を一覧表示
3. トークン使用量と必要なエージェントを提示してユーザーの許可を要求。必要であればユーザーにデータを要求する。
4. 補完案を作成。テンプレートに従って成果物を出す。

Claude.ai（ブラウザ）とClaude Code（ターミナル）の両方で使えるよう、使い方もドキュメント化した。

知識はこれまでNotionとかに蓄えてた資料。出典も明記する仕様にする。

## つまずいたこと：

エンジニアなら容易に突破できる部分だと思うが、GitHubの設定でつまずいた。ただ、Claudeのチャットに全部やり方を聞いて全部解決した。

GitHub Appのアクセス許可、リポジトリが空だとナレッジ追加時にエラーが出た→README.mdを作成して解決。agents/ にファイルを置いてもClaude Codeが認識しなかった（正しくは .claude/agents/ に配置＋YAMLフロントマター必要）…など。

セキュリティ関係だけ、疑って調査をした。質問をし直したら、別のやり方を提案してきたり。

他の件と比較してかなり簡単だった。問題はチームで使ってみて、どれだけ精度を上げていけるか？「何食ったら、そんな発想できるん？」という定型文があるが、AIに何を食わせるかが問題である。
