## 作ったもの

「ゴリラ語変換」というWebツールを作って公開しました。

**何ができるか：** テキストを入力すると、こうなります。

```
こんにちは
↓
ウッホホ ウホホホ ハッホホッ ウホホホ フホホ ンッホホ ウホホホ ンッホ ウホ ウッホ ウホ ンッホホ ウッホ
```

ウホ。

そして、このゴリラ語を入力すると元の日本語に戻ります。完全に可逆です。役には立ちません。

👉 [ゴリラ語変換はこちら](https://take-shiro.github.io/gorilla-encoder/)

## なぜ作ったか

個人開発、いつも途中でやめてしまっていました。

「もっと便利なものを作ろう」「でも似たものがすでにある」「スコープが広がってきた」「なんか飽きた」——このループで何度もお蔵入りにしてきました。

そこで発想を変えました。  
**くだらないものを作ろう**、と。

くだらないものには競合がいません。  
誰も「ゴリラ語変換なら既にいいのがある」とは言いません。  
スコープも自然と小さくなります。  
そして何より、作っていて楽しい。

## 試行錯誤(させた話)

1時間で作成した(させた)ツールですが、詰まった部分があります。

最初のアイデアは「LLMにゴリラ語に翻訳させる」でしたが、それだと復元できません。  
なので**文字コードベースのエンコード**にしました。

最初は16進数で実装しました。

```
「あ」→ U+3042 → 3 0 4 2 → ウホッ ウホ ウッホホ ウホホ
```

でも1文字あたり4トークンで長い。そこで**64進数**に変更しました。

64種類のゴリラ語を用意すれば、1文字を3トークンで表現できます。

```
「あ」→ 64進数3桁 → ウホ ウホ アホホ
```

ゴリラ語64種は、ウ系・ハ系・フ系・ン系・ア系・ホ系などに分けて自然な語感になるよう考えました。「グホ」「ゴホ」はゴリラっぽくないのでボツにしました（個人の感想です）。

この変更で**約35%の文字数削減**を達成。誰も求めていない最適化です。
