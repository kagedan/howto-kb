メール処理は **AIが最も得意な仕事** です。理由は3つ。

1. **入力と出力が明確**：受信文 → 返信文、という変換作業
2. **パターンが繰り返される**：お礼・日程調整・お断り・確認、の4種でほぼ9割

つまりメールは「**AIに8割やらせて、人間が2割直す**」が最もハマる領域なんです。

## ステップ①：コピペ運用（最速・5分）

### やり方

いちばん簡単な方法は、**メール本文をClaudeに貼って返信を作らせる**だけ。

1. Gmailで返信したいメールを開く
2. 本文をコピー
3. ClaudeのProjects（前回の記事で作ったもの）に貼り付け
4. 下記のプロンプトで返信を生成

### コピペで効く返信プロンプト

```
あなたは私のメール返信アシスタントです。
以下のメールに対する返信文を作ってください。

条件：
・敬語、ただし固すぎない
・3〜5文で簡潔に
・相手の要件に1つずつ答える
・最後に次のアクションを1行で添える

メール本文：
【ここに貼る】
```

これだけで、返信の「叩き台」が3秒で出ます。あとは固有名詞や日付だけ直せばOK。

### 効果

* 1通あたりの作成時間が **5分→1分**
* 「書き出しが思いつかない」のストレスがゼロに
* 文面のトーンが安定する（雑な返信が減る）

## ステップ②：Claudeのブラウザ拡張で「開いたまま」処理

### こんな人向け

「コピペの往復が面倒」という人は、**ブラウザ拡張機能** が次の一手です。

Claude（Claude in Chrome など）の拡張を使うと、Gmailを開いたまま、画面のメールをそのままClaudeに読ませて返信を作れます。コピペの往復が消えます。

### 設定の流れ

1. Claudeのブラウザ拡張をインストール
2. Gmailを開いてログイン
3. 返信したいメールを表示
4. 拡張のチャット欄で「このメールに返信を作って」と指示

### 効果と使いどころ

* **タブを行き来しない**ので、受信トレイの処理が一気に進む
* 「未読30通を上から順に処理」が現実的なテンポになる
* スレッドの流れも読ませれば、文脈を踏まえた返信になる

### 運用のコツ

返信の「型」を最初に拡張へ伝えておくと精度が上がります。

```
今日のメール返信ルール：
・お礼メール → 感謝＋次回への一言
・日程調整 → 候補日を3つ提示する形に整える
・お断り → 角を立てない断り方＋代替案
この3パターンで返信を作って。
```

## ステップ③：API自動化（上級・仕組み化）

### 何ができるか

Gmail APIとClaude APIをApps Scriptでつなぐと、**「受信→自動分類→下書き作成」までを全自動**にできます。

朝起きたら、受信トレイの各メールに「返信下書き」がすでに用意されている状態が作れます。

### 自動化できる処理

### Apps Scriptの骨組み（イメージ）

```
function autoDraftReplies() {
  const threads = GmailApp.search('is:unread label:要返信', 0, 10);
  threads.forEach(thread => {
    const msg = thread.getMessages().pop();
    const body = msg.getPlainBody();
    const reply = callClaude(body);
    thread.createDraftReply(reply);
  });
}
```

callClaude() は、前に書いた「スプレッドシート自動化」記事のAPI呼び出しと同じ要領で組めます。

## 実務に効く返信テンプレ3本

ここからは、コピペでそのまま使える返信パターンを3つ。

### テンプレA：日程調整メール

```
以下のメールに対し、日程調整の返信を作って。
・こちらの候補日を3つ提示する形に
・「ご都合いかがでしょうか」で締める
・候補日は来週の平日午後から選ぶ前提で空欄にしておく

メール本文：【貼る】
```

### テンプレB：お礼メール

```
以下のメールにお礼の返信を作って。
・感謝を具体的に（何に対してかを明示）
・次回につながる一言を添える
・3文以内で簡潔に

メール本文：【貼る】
```

### テンプレC：やわらかいお断り

```
以下の依頼を断る返信を作って。
・感謝→理由→代替案、の順
・角が立たない表現で
・相手との関係を壊さないトーンで

メール本文：【貼る】
```

この3本を Projects に登録しておけば、メールの9割はカバーできます。

## 受信トレイをゼロにする1日のワークフロー

テンプレが揃ったら、次は「1日の回し方」です。私が実際にやっている手順を時系列で置いておきます。

### 朝（10分）

1. 受信トレイを開き、**全メールをざっと3分類**する：「要返信」「あとで読む」「不要」
2. 「要返信」だけをClaudeに順番に渡し、下書きを一気に作る
3. 下書きを上から確認し、固有名詞・日付だけ直して送信

ポイントは **「考える」と「書く」を分けること**。分類のときは返信文を考えない。下書きを作るときは分類しない。脳の切り替えコストが消えて、一気に速くなります。

### 昼・夕方（各3分）

### 夜（5分）

* 「あとで読む」を消化
* 明日に持ち越すものだけ「要返信」ラベルを付け直す

この回し方にすると、**受信トレイが毎日ゼロに戻る**ようになります。「未読100通」の山に怯える生活から抜けられます。

## 返信精度を上げる「自分の文体」の覚えさせ方

AIの返信が「自分っぽくない」と感じる最大の原因は、**Claudeがあなたの過去の返信を知らない**ことです。

解決策はシンプル。Projects の knowledge に、**自分が過去に送った「いい感じの返信」を5〜10通アップ**しておく。すると、Claudeはあなたの口癖・締め方・距離感を学習して、驚くほど「自分が書いた風」の返信を出すようになります。

具体的には、こんな指示を Custom instructions に足します。

```
添付した私の過去メールを参考に、私の文体で返信を作って。
守ること：
・書き出しは「お世話になっております」固定
・一文は長くしすぎない
・お礼は具体的に（何に対してかを明示）
・締めは「よろしくお願いいたします」
```

この設定をしてから、返信の「直す量」が体感で半分になりました。

## つまずきやすい3つのポイント

### 1. AIの返信がそのまま使えない

→ 最初の数回は「私ならこう直す」を伝えると学習する。Projects に自分の過去の返信を5本入れておくと精度が跳ね上がる。

### 2. トーンが毎回バラバラ

→ Custom instructions に「文末は『です・ます』、絵文字なし、署名は固定」と書いておく。

### 3. 機密メールを貼ってしまいそうで怖い

→ ステップ①では「テスト用の文面」で練習。本番は固有名詞をマスクする癖をつける。慣れたらステップ③で「自分の環境内だけで完結する」API運用に移行。

## 応用：メール以外にも効く「返信の型」

この「読ませて・分類して・下書きさせる」の流れは、メール以外にもそのまま応用できます。

* **チャット返信**（Slack / Chatwork）：長い相談スレッドを要約させて、返信案を3パターン出させる
* **問い合わせフォーム対応**：よくある質問をProjectsに入れておき、回答下書きを自動生成
* **SNSのDM**：営業DMへの丁寧な断り、コラボ依頼への返信などを型化
* **クチコミ・レビュー返信**：店舗やサービスのレビューへの返信を、トーンを揃えて量産

共通しているのは「**相手の文章を読んで、こちらの文章を返す**」という構造。この構造のタスクは、すべて今日のメール術がそのまま使えます。

「メールが速くなった」で終わらせず、**自分の1日の中で『読んで返す』作業を全部リストアップ**してみてください。そのすべてが、半自動化の対象になります。
