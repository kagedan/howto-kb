## はじめに

「AIがニュースを読む仮想スタジオを作りたい」という妄想を、実際に動くシステムとして完成させました。

**yabou01** という自作プロジェクトで、毎朝こんなことが自動で起きています：

```
6:30 AM
  ↓ ニュース自動収集（Web スクレイピング）
  ↓ Claude で台本を生成
  ↓ VOICEVOX で音声合成
  ↓ Unity で VRM アバター「クロエちゃん」が口パクしながら録画
  ↓ BGM ミックス + サムネイル自動生成
  ↓ YouTube に自動アップロード（非公開）
```

すべて手動ゼロ。毎朝 Telegram に「アップロード完了しました」と通知が来るだけです。

この記事では、最終フェーズである **Unity スタジオ背景の設定** と **全パイプラインの結合** を中心に紹介します。

## 今回のメイン：背景画像をスタジオに設置する

### 背景画像の生成

Nano Banana Pro2（画像生成 AI）で以下のプロンプトを使ってニューススタジオ風の背景を生成しました：

```
A modern TV news studio background, dark navy blue gradient,
glass panels with soft blue ambient lighting, subtle geometric patterns,
professional broadcast studio set, no people, no text, 16:9
```

生成した画像を `NewsStudio.png`（1920×1080）として保存。

### Unity に背景を設置する — MCP 経由

Unity Editor を手動で操作する代わりに、C# スクリプトをエディタ拡張として作成し、Python から MCP 経由で実行します。

**`Assets/Editor/Yabou01BackgroundSetup.cs`**

```
using UnityEngine;
using UnityEditor;

public class Yabou01BackgroundSetup : MonoBehaviour
{
    [MenuItem("yabou01/Setup Background")]
    public static void SetupBackground()
    {
        // テクスチャを読み込む
        string texPath = "Assets/Textures/NewsStudio.png";
        Texture2D tex = AssetDatabase.LoadAssetAtPath<Texture2D>(texPath);
        if (tex == null)
        {
            Debug.LogError($"[BackgroundSetup] テクスチャが見つかりません: {texPath}");
            return;
        }

        // Quad を作成して背景に配置
        GameObject bg = GameObject.Find("Background") ?? GameObject.CreatePrimitive(PrimitiveType.Quad);
        bg.name = "Background";
        bg.transform.position = new Vector3(0f, 1.5f, 6f);
        bg.transform.rotation = Quaternion.identity;
        bg.transform.localScale = new Vector3(16f, 9f, 1f);

        // マテリアルを設定（Unlit/Texture でライティング無視）
        Material mat = new Material(Shader.Find("Unlit/Texture"));
        mat.mainTexture = tex;
        bg.GetComponent<Renderer>().material = mat;

        Debug.Log("[BackgroundSetup] 背景を設定しました");

        // シーンを保存
        UnityEditor.SceneManagement.EditorSceneManager.SaveOpenScenes();
    }
}
```

**Python からの実行（`scripts/setup_background.py`）:**

```
import shutil
import subprocess
import os

# 画像を Unity の Assets フォルダにコピー
src = "/mnt/c/work/GDrive/NewsStudio.png"

# MCP 経由でメニューを実行

execute_menu_item("yabou01/Setup Background")
```

**ポイント：**

* シェーダーは `Unlit/Texture` を使う → ライティングの影響を受けず、常に正しい色で表示される
* Quad の Z 座標をクロエちゃん（Z≈1.75）より奥の `6` に設定することで自然な奥行きが出る
* スケールを `(16, 9, 1)` にして 16:9 アスペクト比を維持

## VRM リップシンクの実装

詳細は別記事（[VRMアバターにVOICEVOX音声で口パクさせる](./unity_vrm_lipsync_voicevox)）にまとめていますが、重要なポイントだけ再掲します。

**UniVRM 1.0 では `SetBlendShapeWeight` が使えない：**

VRM 1.0 の Expression（口パク含む）は内部で毎フレーム上書きされるため、`SkinnedMeshRenderer.SetBlendShapeWeight()` で直接変更してもすぐ 0 に戻ってしまいます。

**解決策：`Vrm10RuntimeExpression.SetWeight()` を `LateUpdate` で呼ぶ**

```
using UniVRM10;

public class Yabou01AmplitudeLipSync : MonoBehaviour
{
    private AudioSource _audioSource;
    private Vrm10RuntimeExpression _expression;
    private float[] _samples = new float[256];

    void Start()
    {
        _audioSource = GetComponent<AudioSource>();
        var vrm10 = GetComponentInParent<Vrm10Instance>();
        _expression = vrm10.Runtime.Expression;
    }

    void LateUpdate()  // ← LateUpdate が必須！
    {
        if (!_audioSource.isPlaying) return;

        _audioSource.GetOutputData(_samples, 0);
        float rms = 0f;
        foreach (var s in _samples) rms += s * s;
        rms = Mathf.Sqrt(rms / _samples.Length);

        float weight = Mathf.Clamp01((rms - 0.005f) / 0.295f);
        _expression.SetWeight(ExpressionKey.Aa, weight);
    }
}
```

## 全パイプラインの結合

`pipeline_all.py` で Phase 1〜5 を一括実行します。

```
# pipeline_all.py（抜粋）

import subprocess
import glob
import os
import wave

def get_audio_duration(wav_path):
    """WAV ファイルの長さを秒で返す"""
    with wave.open(wav_path, 'r') as f:
        return f.getnframes() / f.getframerate()

def run_pipeline(unity=False, privacy="private"):
    # Phase 1: ニュース収集・台本生成
    run_phase("python scripts/pipeline_phase1.py")

    # Phase 2: 音声合成
    run_phase("python scripts/tts.py")

    # 音声ファイルの長さを取得
    wav_files = sorted(glob.glob("output/audio/*.wav"))
    audio_duration = int(get_audio_duration(wav_files[-1])) + 2  # 2秒バッファ

    if unity:
        # Phase 3U: Unity 録画（音声の長さに合わせる）
        run_phase(f"python scripts/unity_recorder.py record --duration {audio_duration}")

        # 最新の Unity 録画ファイルを特定
        unity_videos = sorted(glob.glob("/mnt/c/work/GDrive/videos/unity/yabou01_*.mp4"))
        unity_video = unity_videos[-1] if unity_videos else None

        # Phase 4: BGM ミックス + Unity 映像合成
        cmd = "python scripts/pipeline_phase4.py"
        if unity_video:
            cmd += f" --unity-video {unity_video}"
        run_phase(cmd)
    else:
        run_phase("python scripts/pipeline_phase4.py")

    # Phase 5: YouTube アップロード
    run_phase(f"python scripts/pipeline_phase5.py --privacy {privacy}")
```

**実行コマンド：**

```
python scripts/pipeline_all.py --unity --privacy private
```

## ハマりポイント まとめ

| 問題 | 原因 | 解決策 |
| --- | --- | --- |
| Unity 背景が真っ暗 | Standard シェーダーでライトがない | `Unlit/Texture` に変更 |
| 口パクが毎フレームリセットされる | VRM 1.0 の Expression が内部管理 | `Vrm10RuntimeExpression.SetWeight()` を `LateUpdate` で呼ぶ |
| 録画時間が短い | `--duration 30` でハードコードしていた | TTS 音声の長さを `wave` モジュールで自動取得 |
| Unity Recorder が Play モード外で失敗 | `PrepareRecording` は Play モード必須 | Python 側で `manage_editor("play")` を先に送る |
