## はじめに

自分の運営するサッカーチーム(とランニングコミュニティ)の公式サイトを作りたかった。メンバー紹介、試合スケジュール、ユニフォーム紹介、ギャラリー——やりたいことは明確にある。でも、デザインができる人もいなければ、コードを書ける人もいない。

外注するには小さすぎる規模で、かといってWordPressをゼロから触るには時間が足りない。そんな状況で試したのが **Claude Design** だ。

Anthropic Labsが2026年4月17日にリリースしたばかりのプロダクトで、「チャットしながらビジュアルを作る」という新しいスタイルのデザインツール。使ってみたら、予想をはるかに超えるものが出てきた。

## 全体の流れ

```
要件をチャットで伝える（約10分）
      ↓
初稿が自動生成される（約3分）
      ↓
会話で修正・調整する（約20分）
      ↓
HTMLファイルでダウンロード・デプロイ（約5分）
━━━━━━━━━━━━━━━━━━
合計：約40分
```

---

## 1. 要件を整理する ── 会話だけで仕様書ができた

Claude Designを開くと、まずチャットパネルが現れる。ここに「何を作りたいか」を日本語で書くだけでいい。  
プロンプトを作るために数回Claudeとチャットをして、生成されたものを利用した。

```
社会人ランニングコミュニティの公式サイトを作りたい。

以下のセクションを作ってほしい:
- Hero（メインビジュアル）
- チーム紹介・理念
- 活動情報と練習場所
- メンバー紹介
- ユニフォーム紹介
- ギャラリー
- スケジュール
- フッター
```

すると、**Claude側から逆に質問が来た**。デザインの方向性を決めるための選択式フォームが自動で生成されたのだ。

* カラーの系統は？（ダーク系・ライト系・カラフルなど）
* フォントの雰囲気は？（モダン・クラシック・スポーティなど）
* アニメーションの量は？（控えめ・標準・リッチ）
* 初稿のアプローチは？（「1案を作り込む」か「複数案を並べる」か）

「モダンで斬新に」という抽象的な言葉を具体的な選択肢に落とし込んでくれるので、自分の好みを整理する作業にもなった。

選択を送信すると、左にチャット・右にライブキャンバスという画面構成に切り替わり、リアルタイムでデザインが生成されていく。

## 2. 生成されたもの ── 全8セクション、1ファイル

初稿が完成するまで約3分。出来上がったのは以下の構成をすべて含む **1つのHTMLファイル** だった。  
Opus 4.8なのでかなり時間がかかったが、モデルを変更すれば時間も減らせるのではないかと思う。

| セクション | 内容 |
| --- | --- |
| **Hero** | フルスクリーン・パララックス背景 |
| **About / 活動理念** | エディトリアルレイアウト（雑誌風の段組み） |
| **Activity & Routes** | Google Maps埋め込み・練習ルート表示 |
| **Members** | 横スクロールカルーセル |
| **Uniform** | 実物写真を自動配置するグリッド |
| **Gallery** | グリッド＋ライトボックス拡大 |
| **Schedule** | 今後/過去タブ・日付で自動振り分け |
| **Footer** | SNSリンク・コピーライト |

ブラウザで開いた瞬間、「これ、外注品じゃないか」と一瞬思った。

## 3. Claude Designが特にすごかった点

### 3-1. デザインシステムを自律的に定義した

色・フォント・余白・モーションが**一貫したシステム**として設計されていた。

* カラー：漆黒 `#0D0D0D` × ライムグリーン `#C8FF00`
* フォント：見出しに `Zen Kaku Gothic New`、数字・英字に `Saira Condensed`
* モーション：スクロール量に応じて変わるイージング設定

「モダンで斬新に」という一言を、ここまで具体的な造形に翻訳してくれた。デザイナーがトーン＆マナーを定義するあの作業が、数分で完了していた。  
![](https://static.zenn.studio/user-upload/dff4a670b0a5-20260629.png)

### 3-2. インタラクションまで込みで出てくる

コードを一行も書いていないのに、以下がすべて動いていた。

* **スクロール連動リビールアニメーション**（要素が下からフェードイン）
* **スクロール追従ナビゲーション**（スクロールするとスリム化＆背景ブラー）
* **パララックス効果**（ヒーロー背景が視差でずれる）
* **ホバーエフェクト**（カードにカーソルを当てると浮き上がる）
* **キーボード対応ライトボックス**（矢印キーで写真を送れる）
* **日付自動振り分けスケジュール**（今日より前の試合は自動で「過去」タブへ）

静的なデザインではなく、**動くプロトタイプ**が出てくる点がClaude Designの核心だと感じた。

### 3-3. 写真をドラッグ＆ドロップで差し替えられる

画像はすべてプレースホルダーとして配置されていて、実際の写真をそのままドロップすれば差し替えられる。撮影前でも「雰囲気の確認」ができるし、写真が揃ったらそのまま本番に近い状態にできる。

### 3-4. 修正が会話でできる

生成後の修正もすべてチャットで行った。

```
「荒川グラウンド→代々木公園に変えて」
「練習ルートは2本に絞って、地図も更新して」
「Heroのコピーをもっと短くして。1行に収めたい」
```

自然言語で指示すると、**変更箇所だけを外科的に編集**してくれる。他のセクションが崩れない。「ここだけ直して他はそのまま」という感覚で使えるのが、実務上かなり大きい。

## 4. 現状の機能における限界

良い点だけ書いても参考にならないので、詰まったところも記録しておく。

### Proプランだと一瞬でトークンを使い果たす

一度のやり取りで消費されるトークン数も膨大になりがちです。複雑なコード生成や長文の処理などを何度も繰り返していると、あっという間に上限に達してしまうことも珍しくありません。

【対策：事前にプランを練る】  
限られたリソースを最大限に活かすためには、AIに指示を出す前の準備が鍵を握ります。

目的の明確化: 最終的に何を出力させたいのか、ゴールを具体的に設定する。

プロンプトの推敲: チャット上で試行錯誤するのではなく、事前にテキストエディタ等で要件や条件を整理し、少ない回数で精度の高い指示を出す。

タスクの分割: 一度にすべてを求めるのではなく、段階を追って処理させることで、軌道修正にかかる無駄なトークン消費を減らす。

### アニメーション作成は限界あり

テキストや画像からアニメーションを生成する機能は画期的ですが、現段階では実用においていくつかの壁があります。(あとはこちらもトークンをかなり消費します)

現時点では、本格的な映像制作というよりも「アイデア出し」や「プロトタイプの作成」といった補助的な用途として割り切るのが無難です。しかし、この分野の進化スピードは凄まじいため、より長時間の生成や、カメラワーク・モーションの細かな指定が可能になるなど、今後のアップデートによって一気に化ける可能性を秘めています。

## 5. 所要時間と工数感

| 作業 | 時間 |
| --- | --- |
| 要件整理・質問回答 | 約10分 |
| 初稿生成 | 約3分 |
| 内容の修正（数回） | 約20分 |
| ダウンロード・デプロイ | 約5分 |
| **合計** | **約40分** |

外注すれば数週間・数十万円と発生するであろうプロセスが、40分で完了した。もちろん品質の差はあるが、「チームの公式サイトとして十分機能する」レベルには間違いなく達していた。

## 6. まとめ

**Claude Designで変わること**:

* デザイナー・エンジニアがいなくても、プロ品質に近いサイトが作れる
* 「作る」より「伝える」にコストをかけるフェーズに移行できる
* 写真と実データを差し込めば、翌日から公開できる状態で出てくる

**向いている用途**:

* チーム・サークル・店舗などのコミュニティサイト
* スタートアップのランディングページ
* ピッチデック・提案資料（PDF/PPTXエクスポートも対応）
* デザイナーが「まず方向性を見せる」ためのプロトタイプ

**まだ難しいこと**:

* Figmaレベルのピクセル精度の調整
* 複雑なバックエンド連携（フォーム送信、DB接続など）
* 大規模なデザインシステムの管理
* 動画やアニメーション作成
