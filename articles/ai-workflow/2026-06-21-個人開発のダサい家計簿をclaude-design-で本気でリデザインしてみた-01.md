## 先に宣伝

家計簿アプリを作成しています。  
複式簿記の形式で作成をしていて、資産の一覧や、口座内のお金にタグをつけて「お金に色を付けて管理」できるようにしています。  
デザインは少し下にあるので軽くでも見てもらえると嬉しいです。

**HP(LP)**

**アプリ**

## デザインがずっとコンプレックスだった

個人で家計簿SaaSを作っています。React + AWS で、複式簿記ベースで貸借対照表まで出せる…と機能は盛ったものの、**見た目がずっと素人くさい**のが気がかりでした。配色は何となく選んだゴールド、余白もバラバラ。「中身はいいのに」と自分で言い訳していたやつです。

そこで Claude Design（claude.ai/design）を使って、いっそ全面的に作り直してみました。アプリ13画面とランディングページ、まるごと。やってみたら想像以上に学びがあったので、つまずきも含めて残します。

ちなみに前提として、自分は**デザインの専門家ではない**し、**機能は1つも減らしたくない**（既存ユーザーがいる）。この2つの縛りの中での話です。

まず結論から。これがビフォーアフターです（左＝旧、右＝刷新後）。

![ダッシュボードのBeforeAfter](https://static.zenn.studio/user-upload/813675644cd6-20260622.png)

## まず「今のダサい画面」を見せるところから

Claude Design に「今こうなってます」を理解してもらうため、現状の全画面スクショを用意しました。ただ、空っぽの画面を見せても伝わらないので、ゲストモードにダミーデータを流し込んでから撮りました。

```
// Playwrightでゲストにサンプルデータを入れてから撮影
await ctx.addInitScript((seed) => {
  localStorage.setItem('kk4_guest', seed);
  localStorage.setItem('kk_guest', '1');
  localStorage.setItem('kk_onboarded', '1'); // オンボ抑制
}, JSON.stringify(sampleData));
```

ダッシュボードに残高やグラフが入った“それっぽい”状態で撮れたので、これを設計のたたき台にしました。

## 生成はサクサク。でもエクスポートで手が止まった

Claude Design 側でデザインを作るのは速かったです。配色も構図も一気に複数パターン出てくる。テンション上がってエクスポート…したら、出てきたHTMLを開いて「**え、これどう使うの？**」と固まりました。

中を覗くと、ふつうのHTMLじゃなくて独自のパッケージ形式（bundler形式）でした。

```
<script type="__bundler/manifest"> {"<uuid>":{"compressed":true,"data":"<base64(gzip)>"}} </script>
<script type="__bundler/template"> "<!DOCTYPE html>... <x-dc>...</x-dc> ..." </script>
```

最初「ランタイムっぽい巨大JS」を読んで「デザインどこ…？」と迷子になったんですが、よく見ると **見た目の本体は `template` の方** にインラインCSSで入っていました。`manifest` はランタイムとフォント、`template` が実デザイン。

分かってしまえば抽出は一瞬で、Nodeでデコードするだけ。

```
const tpl = JSON.parse(
  html.match(/<script type="__bundler\/template">([\s\S]*?)<\/script>/)[1].trim()
);
fs.writeFileSync('design.html', tpl); // 中の <x-dc> を読む
```

ここでようやく、欲しかった“デザインの素”が全部見えました。

* 配色：アクセント = ティール `#0d9488`、背景 `#e8eaed`、文字 `#14201f`…
* フォント：`Manrope` + `Noto Sans JP`
* 意匠：角丸のソフトカード、ピル型ナビ、純資産を大きく見せる“ヒーロー”、ライト/ダーク両対応

`<x-dc>` のマークアップはテンプレ記法（`{{ }}`）なのでコピペでは使えませんが、**インラインstyleを読めば全部分かる**。ここからは手作業の世界です。

## 既存アプリへの“移植”が本番だった

正直、ここからが本題でした。生成したデザインを、**動いている既存アプリにどう載せるか**。

### 救われたのは「CSS変数で書いてた」こと

過去の自分を褒めたいのですが、配色を全部 CSS 変数にしていました。おかげで `variables.css` を差し替えるだけで全画面に新色が回ります。

```
/* before(ゴールド) → after(ティール) */
[data-theme="light"]{
  --bg0:#e8eaed; --bg1:#fff; --bd:#e6ebec;
  --tx:#23262d; --tx2:#71767f;
  --ac:#0d9488; --ac2:#0b7d72; --acb:rgba(13,148,136,.10);
}
```

「トークン駆動にしといて良かった…」と心から思いました。リデザインのコストが段違いです。

### でも「色を替えただけ」では、自分でもしょぼいと思った

最初はトークンだけ替えて「お、それっぽい」と一瞬満足したんですが、改めて見ると **レイアウトは前のまま**。色が変わっただけ。Claude Design が提案していたのは、もっと構造から違うものでした。

そこでダッシュボードは**構造ごと作り直し**。

* 一番上に **純資産ヒーロー**（ティールのグラデ、純資産をドンと大きく、横に収入/支出/収支）
* 総資産/負債のカード、内訳ドーナツ、セグメント型の期間切替

データを取る部分や計算ロジックはそのまま、**見せ方だけ差し替える**。これが既存アプリをリデザインするときの一番のコツでした。

### モックは“簡易版”。機能を削らない縛りとの戦い

ここが一番悩みました。Claude Design のモックは、仕訳入力もクレジットも「読み取り専用の簡易画面」になりがちです。これをそのまま再現すると、複数行仕訳・分割・税みたいな実機能が消えてしまう。

なので自分の中でルールを決めました。

> **モックの“簡易フォーム”は採用しない。刷新の“視覚言語”だけを、既存の機能UIに被せる。**

たとえば仕訳入力は、Claude Design の「借方＝ティール／貸方＝赤の2カラム構造化フォーム」という見た目は採用しつつ、既存の一行クイック入力や高度な入力モーダルはそのまま残しました。見た目は新しく、できることは減らない。

![仕訳入力のBefore/After](https://static.zenn.studio/user-upload/db72f847bd17-20260622.png)

レポート系（貸借対照表など）も同じ視覚言語で統一しました。

![貸借対照表のBefore/After](https://static.zenn.studio/user-upload/9a17e7a08f5b-20260622.png)

## 巨大な数字で全部崩れて焦った

ひと通りできて満足していたら、テストで大きい金額を入れた瞬間にモバイルでレイアウトが崩壊。原因は `1fr`（＝`minmax(auto,1fr)`）で、中身が縮まずに横へあふれてページごと広がるやつでした。

```
// 金額を ×137 して数千万〜億で崩れないか毎回チェック
for (const j of ds.journals) for (const l of j.lines) l.amount *= 137;
```

`minmax(0,1fr)` に直し、保険で `body{overflow-x:hidden}`、明細テーブルだけ個別に横スクロール。**“雰囲気で完成”にしないために、スクショ検証を自動化**しておいて本当に良かったです（ライト/ダーク/モバイル/巨大数字を毎回撮る）。

![モバイルのBefore/After](https://static.zenn.studio/user-upload/c1ca9d863af3-20260622.png)

## LP もアイコンも同じ要領で

ランディングページ（静的HTML）も同じ流れで teal/Manrope に。代表アイコン（favicon＋左上ロゴ）は、上昇カーブ＋↗矢印を **自分でSVGを書いて** 用意しました。小さいものは生成よりベタ書きが早い。

## やってみての感想

* **Claude Design は“方向性”を出すのがめちゃくちゃ速い**。素人が0から悩むより、まず叩き台が出るのが精神的に大きい。
* ただし **実プロダクトへの落とし込みは結局、人間（自分）の判断**。機能との整合、状態、崩れ対策…ここは誰も代わってくれない。
* **トークン駆動・スクショ自動検証** をやっておくと、リデザインが「怖くない作業」になる。
* 何より、**長年のコンプレックスだった見た目に、ようやく胸を張れた**のが一番うれしかったです。
