### この記事を読むとわかること

* Claudeの新機能「Channels」と「Scheduled Tasks」で何が変わったのか

---

### Claudeの新機能、まだあるの？

Dispatchだけでもすごかったのに、調べてみたら  
こんなものまで出てきていました。

「**Channels（チャネルズ）**」は、TelegramやDiscordからClaude Codeに直接指示を送れる機能です。スマホどころか、普段使ってるチャットアプリから「あれやっておいて」ができるようになりました。

「**Scheduled Tasks（定期タスク）**」は、「毎朝9時にこれチェックして」みたいな繰り返しタスクをAIに任せられる機能です。

ここでピンときた人もいるかもしれません。

---

### Dispatch（Cowork）の弱点が、一気に埋まった

前回の記事で紹介したDispatch。  
あれは「Claude Cowork」というPC上で動くAIアシスタントです。

めちゃくちゃ便利なんですけど、弱点がひとつあったんですよね。

見れるフォルダが制限されるんです。

**ぼくの作業フォルダはDドライブにあるんですけど、Coworkからは触れない。これが地味にキツかった。**

ところがChannelsは違います。

Channelsは、スマホのチャットアプリからPC上のClaude Code本体を動かせます。Claude Codeなら、Dドライブだろうがどこだろうが触れる。

つまり、Coworkの「フォルダ制限」という弱点を、Channelsが補完してきたんです。

---

### さらに。GitHubに置いておけば……

もともとClaude Codeは、GitHubのリポジトリに対して作業できます。

ということは、Dドライブの作業フォルダをGitHubにも置いておけば。

**Channelsでスマホから指示** → PCのClaude Codeが動く → Dドライブ触れる。  
**GitHub経由なら** → PC閉じてても、クラウド上でファイル操作できる。  
**Scheduled Tasksなら** → 定期実行で自動処理までできる。

どこからでも、いつでも、自分のファイルにAIが触れる環境になるんですよね。

正直、これを見たとき思いました。

「**もうOpenClawいらないんじゃない？**」

---

### Mac miniを買うか悩んでる最中だから余計に

いま、ぼくはMac mini（32GB）を買うかMacBookにするか、かなり悩んでいます。

OpenClawを常時動かすなら、Mac miniを自宅サーバーとして置きっぱなしにする選択肢がある。でも、もしClaude側の新機能で全部まかなえるなら、サーバー用のMacを買わなくて済む。

その差額、けっこうデカいんです。

「**ここで済ませられるなら……！**」

という気持ちが、めちゃくちゃ大きいです～～～！

### Claude君に聞いたら、今回も空気が変わった

ここまでの期待を込めて、Claude君に正面からぶつけてみました。

ぼく「Dispatch + Channels + GitHub + 定期タスク。全部組み合わせたらOpenClawの代わりになる？」

最初、Claude君はこう答えました。

「やっぱりOpenClawが要るのか……」

と思いかけたんですけど。

ぼくがいくつか質問を返しているうちに、状況がどんどん変わっていきました。

---

### 「それ、GitHub Actionsでできますよ」

ぼくが「**DドライブのファイルをGitHubに置いて、push/pullで同期したらどうなる？**」と聞いたら、Claude君が「あ、それ今まさにやってることですね」と。

そうなんですよ。ぼくのHQ自体がGitHubリポジトリなんです。

そして**GitHub Actions**を調べてもらったら、想像以上にいろいろできました。

定期実行は、cronで最短5分おきに動かせます。外部からの割り込みも、「repository\_dispatch」という仕組みで、どんなサービスからでもトリガーできます。

Claude Code用の公式Actionもあるし、ワークフローを連鎖させることもできます。通知はSlack、Discord、メールに送れて、公開リポジトリなら無料です。

整理するとこうなります。

* 定期実行 → できる（cron 5分間隔から）
* PC閉じても動く → できる
* ファイル操作 → できる（push/pullで同期）
* 外部からの割り込み → できる（repository\_dispatch）
* ワークフロー連鎖 → できる
* ルール・記憶の維持 → できる（[CLAUDE.md](http://CLAUDE.md) / Memory）
* Slack/Discord通知 → できる

……あれ？

**OpenClawでやりたかったこと、ほぼ全部できるじゃないですか。**

---

### 残る差は、ひとつだけ

Claude君と一緒に整理した結果、本当に埋まらない差は「**AIモデルの選択**」だけでした。

Claude Codeは当然、Claudeしか使えません。

OpenClawなら、タスクに合わせてモデルを使い分けられます。簡単な仕事はGeminiやDeepSeekの安いモデルに回して、大事な判断だけClaudeに任せる、みたいなことができる。

コストを最適化したい場面で、選択肢が1つしかないのは、たしかに制約ではあります。

でも、それだけのために、Mac mini買ってサーバー立てますか？っていう話になった。

---

### ここで冷静に、OpenClawの現実を見てみた

「モデル選択のためにOpenClaw」と考える前に、ひとつ確認しておきたいことがありました。

OpenClawって、本当に順調に使えるものなのか。

Claude君に調べてもらった結果が、なかなかキツかったです。

---

**まず、セットアップで撃沈する人が多い。**

macOSでインストールが通らないケースがあります。設定ファイル1つ間違えるとゲートウェイが起動しない。公式ドキュメントのコマンドが間違っているケースまであるそうです。

Redditにこんな声がありました。

> 「3日かけてOpenClawを設定して、$50溶かした。Claude Codeに切り替えたら1時間で終わった」

---

**次に、アップデートのたびに壊れる。**

設定のキー名が変わったり、サンドボックスの権限が勝手にリセットされたりする。「昨日まで動いてたのに」が日常茶飯事なんだそうです。

---

**そしてコストが爆発するリスクがある。**

エージェントがループに入ると、一晩で数万円溶けます。月$10の予定が$3,600になった事例も報告されています。支出制限が組み込みじゃないから、気づいた時にはもう遅い。

---

**セキュリティも怖いです。**

2ヶ月で9件の脆弱性（CVE）が見つかっています。公開インスタンスの63%がリモートコード実行可能な状態だったという報告もありました。マーケットプレイスには800以上の悪意あるプラグインが。

---

**GitHubのスターは25万。でも実際に動いているのは2〜4万。**

スターの8〜16%しか稼働していないんですよね。ほとんどの人が「すげー！」で星をつけて、導入で止まっているか、撃沈しているようです。

---

### OpenClawで成功してるのは、こういう人

フルタイムのAI/MLエンジニアで、壊れたら自分で直せる人。予算に余裕があって、月$100〜の予期せぬ出費に耐えられる人。セキュリティリスクを理解して受け入れられる人。「壊れたら直す」を楽しめる人。

つまり、腹をくくった人か、経営者か、そもそもお金に余裕がある人なんですよね。

前回ぼくが「**まずはDispatchで十分**」と書いたのは、「そうだろうなぁ」と思ったからです。

---

### 前回の自分に言いたいこと

2日前のぼくは「OpenClawの90%をDispatchで！」と書きました。

今日の結論は、もっと踏み込んでいます。

**Claude Code + GitHub Actionsで、95%いけます。**

残りの5%は「モデルを自由に選べること」。そしてその5%を取りに行くには、Mac mini買って、OpenClawセットアップして、アプデで壊れるたびに直して、コスト爆発に怯えながら、セキュリティリスクを自分で管理する。それだけの覚悟が要ります。

ほとんどの人にとっては、Claude側で十分だと思います。

---

### でも正直、Claude側もめんどくさい

ここまで読んで「じゃあClaude Code + GitHub Actionsでいこう！」と思った人に、ひとつ言っておきたいことがあります。

**設定、けっこう手間です。**

Claude Codeをインストールして、GitHubアカウント作って、リポジトリ作って、push/pullを理解して、GitHub ActionsのYAMLを書いて、Channelsの設定をして、Scheduled Tasksも設定して。

それぞれ別のシステムを、自分で繋がないといけません。

OpenClawは、インストールして権限を渡したら、あとは話しかけるだけ。PC権限もアカウントも財布も預けたら、ほぼ人間と同じように動いてくれる。

Claude Code + GitHub Actionsは「優秀なフリーランスを5人集めて、自分でプロジェクト管理する」ようなもの。一人ひとりは優秀だけど、つなぎは自分でやらないといけない。

---

### なぜ大手は「全部やるエージェント」を出さないのか

ここが面白いところで。

OpenClawに全権委任すると、何でもやってくれます。でも全部パーになるかもしれない。コスト爆発も、セキュリティ事故も、全部自分の責任です。

大手は、うかつにそういうサービスを出せないんですよね。APIやツールは提供しても、「つなぐのは個人の責任」にしておかないと、やらかした時の責任がサービス側にかかってくる。

Anthropicが「バラバラのパーツ」で出しているのは、臆病なんじゃなくて、責任の線引きなんです。

**ところがアジアの企業は違います。**

**MetaはManusを20億ドルで買収**して、PCアプリとしてワンストップで出してきました。  
ローカルファイル操作、アプリ制御、常時起動まで揃っています。  
  
中国ではKimiが100エージェント同時並列、MiniMaxは自己進化型AIを自社サービスに統合。  
**OpenClawのハッカソンは学生参加OKで大盛況ｗ**

欧米勢も、のんびりしてられない空気が出てきています。

OpenAIはOpenClawの開発者を雇い入れました。  
Googleはワークスペース全体にGeminiを急いで統合中。

OpenClawの「残り5%」は、Claudeが将来マルチモデル対応したら消える可能性すらあります。（裏ワザが・・・）
