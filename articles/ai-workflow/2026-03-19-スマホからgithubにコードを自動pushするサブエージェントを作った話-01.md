# スマホからGitHubにコードを自動pushするサブエージェントを作った話

## はじめに

[前回の記事](https://zenn.dev/acropapa330/articles/zenn_article3)では、自作スケジューラと Brave Search API を使って毎朝 AI テックニュースを自動配信する仕組みを構築しました。

今回は、**Telegram から `/project` コマンドを送るだけで、コードを書いて GitHub に自動 push するサブエージェント構成**を実装しました。

```
/project myapp FastAPIのTodoアプリを作って
    ↓
🆕 新規プロジェクト
🔗 GitHub リポジトリを自動作成
📌 requirements.txt を作成しました（進捗通知）
📌 main.py を作成しました（進捗通知）
💬 コミットメッセージを生成中...
📝 feat: add FastAPI todo application with CRUD endpoints
✅ GitHub に push しました！
```

---

## サブエージェントとは

Claude Agent SDK の `cwd`（カレントディレクトリ）オプションを使うと、指定したフォルダを作業ディレクトリとして Claude を実行できます。

```
from claude_agent_sdk import query, ClaudeAgentOptions

async for message in query(
    prompt="FastAPIのTodoアプリを作って",
    options=ClaudeAgentOptions(
        allowed_tools=["Read", "Write", "Edit", "Glob", "Bash(python *)"],
        permission_mode="acceptEdits",
        cwd="c:\\work\\projects\\myapp",  # ← ここがポイント
    ),
):
    ...
```

これにより Claude は `c:\work\projects\myapp` をカレントディレクトリとして認識し、ファイルをそこに作成します。

---

## 実装のポイント

`Bash` ツールをそのまま許可すると `rm -rf` などの危険なコマンドも実行できてしまいます。`Bash(コマンド *)` の形式で許可するコマンドを絞ります。

```
allowed_tools=[
    "Read", "Write", "Edit", "Glob",
    "Bash(python *)",   # Python 実行
    "Bash(pip *)",      # パッケージインストール
    "Bash(mkdir *)",    # フォルダ作成
    "Bash(git *)",      # Git 操作
    "Bash(echo *)",     # テキスト出力
    "Bash(dir *)",      # ファイル一覧
    "Bash(type *)",     # ファイル内容表示
]
```

削除系コマンド（`del`, `rd`, `rmdir`）は入れないのがポイントです。

### CLAUDE.md で文脈を引き継ぐ

Claude Agent SDK はセッション間で記憶を持ちません。プロジェクトフォルダに `CLAUDE.md` を置いておくと、次回起動時に Claude が自動で読み込んで前回の作業内容を把握します。

```
# myapp プロジェクト

## 目的
FastAPIでTodoアプリのAPIを作る

## 技術スタック
- Python 3.14
- FastAPI 0.115
- SQLAlchemy 2.0（SQLite）

## ファイル構成
myapp/
├── main.py
├── database.py
├── models.py
├── schemas.py
├── routers/todos.py
└── requirements.txt

## 作業履歴
- 2026-03-18 18:10 初回作成：FastAPIでTodoアプリのAPIを作る
- 2026-03-18 18:30 テストコードを追加
```

2回目以降の `/project myapp テストコードを追加して` では、この CLAUDE.md を読んでから作業するため、前回の構成を把握した上で正しくテストコードを追加してくれます。

### notify.py でリアルタイム進捗通知

サブエージェントが作業中に Telegram へ進捗を通知できるよう、専用スクリプトを用意しました。

```
# notify.py
import asyncio, os, sys
from pathlib import Path
from dotenv import load_dotenv
from telegram import Bot

load_dotenv(Path(__file__).parent / ".env")

async def main():
    message = " ".join(sys.argv[1:])
    await Bot(token=os.environ["BOT_TOKEN"]).send_message(
        chat_id=int(os.environ["MY_USER_ID"]),
        text=f"📌 {message}",
    )

asyncio.run(main())
```

CLAUDE.md の「進捗通知ルール」にこのスクリプトの使い方を記載しておくと、Claude が作業の節目で自動的に実行してくれます。

```
## 進捗通知ルール
作業の節目ごとに以下のコマンドで通知すること：
  python "c:\\work\\ClaudeAgentBot\\notify.py" 通知メッセージ
```

### タイムアウトで bot.py の停止を防ぐ

`pip install` などの長時間処理を指示すると bot.py が詰まってしまいます。`asyncio.wait_for` で 5 分のタイムアウトを設けました。

```
async def run_query():
    async for message in query(prompt=prompt, options=options):
        ...

try:
    await asyncio.wait_for(run_query(), timeout=300)
except asyncio.TimeoutError:
    await update.message.reply_text("⚠️ タイムアウトしました（5分）")
```

## GitHub 自動連携

### GitHub API でリポジトリを自動作成

Personal Access Token（PAT）を取得して `.env` に設定しておくと、初回の `/project` 時に自動でリポジトリを作成します。

```
import json, urllib.request

def github_create_repo(repo_name):
    payload = json.dumps({
        "name": repo_name,
        "private": True,   # プライベートリポジトリ
        "auto_init": False,
    }).encode("utf-8")
    req = urllib.request.Request(
        "https://api.github.com/user/repos",
        data=payload,
        headers={
            "Authorization": f"token {GITHUB_TOKEN}",
            "Accept": "application/vnd.github.v3+json",
            "Content-Type": "application/json",
        },
        method="POST"
    )
    with urllib.request.urlopen(req, timeout=10) as resp:
        data = json.loads(resp.read().decode())
        return data.get("html_url")
```

### Claude がコミットメッセージを生成する

作業完了後に Claude に conventional commits 形式のコミットメッセージを生成させます。

```
async def generate_commit_message(instruction, result):
    cm_result = ""
    async for message in query(
        prompt=(
            "以下の作業内容に対して、conventional commits 形式の英語コミットメッセージを"
            "1行だけ生成してください。説明や改行は不要です。\n\n"
            f"作業内容：{instruction}\n"
            f"作業結果：{result[:300]}"
        ),
        options=ClaudeAgentOptions(allowed_tools=[]),
    ):
        if hasattr(message, "result") and message.result:
            cm_result = message.result.strip().splitlines()[0]
    return cm_result or f"feat: {instruction[:70]}"
```

生成されるコミットメッセージの例：

```
feat: add FastAPI todo application with SQLite persistence
fix: update button element with onclick alert handler
test: add comprehensive test suite for todo CRUD endpoints
```

### git push の実装

```
def git_setup_and_push(project_dir, repo_name, commit_message):
    remote_url = f"https://{GITHUB_USERNAME}:{GITHUB_TOKEN}@github.com/{GITHUB_USERNAME}/{repo_name}.git"

    def run(cmd):
        return subprocess.run(
            cmd, cwd=str(project_dir),
            capture_output=True, text=True,
            encoding="utf-8", errors="replace"
        )

    if not (project_dir / ".git").exists():
        run(["git", "init"])
        run(["git", "branch", "-M", "main"])

    run(["git", "remote", "add", "origin", remote_url])
    run(["git", "add", "."])
    run(["git", "commit", "-m", commit_message])
    run(["git", "push", "-u", "origin", "main", "--force"])
```

## 詰まったポイント

### プロンプトに絶対パスを含めると変なフォルダが作られる

プロンプトに `c:\work\projects\myapp` のようなパスを含めると、バックスラッシュが除去されて `cworkprojectsmyapp` というフォルダが作られてしまいました。

```
# NG
prompt=f"c:\\work\\projects\\myapp で作業してください。{instruction}"

# OK
prompt=f"作業ディレクトリはカレントディレクトリです。{instruction}"
```

`cwd` でカレントディレクトリを指定しているので、プロンプトには絶対パスを含めないのが正解です。

### permission\_mode の選択

| モード | 動作 |
| --- | --- |
| `bypassPermissions` | 全ての権限確認をスキップ（危険） |
| `acceptEdits` | ファイル編集は自動承認、危険なコマンドは確認 |

`bypassPermissions` は便利ですが、`rm -rf` なども実行できてしまいます。`acceptEdits` + `allowed_tools` で特定コマンドのみに絞るのがバランスが取れています。

## まとめ

* `cwd` オプションでサブエージェントの作業ディレクトリを指定できる
* `CLAUDE.md` をプロジェクトに置くと文脈を引き継げる
* `notify.py` でサブエージェントから Telegram へ進捗通知できる
* `asyncio.wait_for` でタイムアウトを設けると bot.py が詰まらない
* Claude に conventional commits 形式のコミットメッセージを生成させられる
* プロンプトに絶対パスを含めると意図しないフォルダが作られるので注意
