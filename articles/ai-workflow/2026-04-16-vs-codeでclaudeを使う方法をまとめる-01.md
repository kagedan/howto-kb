## VS CodeでClaudeを使う方法をまとめる

こんにちは、情報工学系男子です。今回は初心者向けに、**VS CodeでClaudeを使う方法**を、できるだけシンプルにまとめます。

Claude はブラウザだけで使うものだと思っていたのですが、今は **VS Code上でも、CUI（ターミナル）上でも使えます**。  
コードの説明をさせたり、エラー原因を聞いたり、修正案を出してもらったりできるので、かなり便利です。

---

## 1. まず知っておくこと

Claude を開発で使う方法は、大きく2つあります。

* **VS Code で使う方法**  
  画面上で差分を見ながら使いやすい
* **CUI で使う方法**  
  ターミナルから素早く使いやすい

初心者にはまず **VS Code版** が分かりやすいです。  
一方で、普段から Git やコマンド操作に慣れているなら、**CUI版** もかなり便利です。

---

## 2. 事前に必要なもの

最初に、以下を用意しておきます。

* VS Code
* Claude を利用できるアカウント
* 作業したいプロジェクトフォルダ

コードが1つもない状態でも触れますが、実際には何かしらのプロジェクトを開いた状態の方が使いやすいです。

---

## 3. VS Codeで使う方法

### 拡張機能を入れる

VS Code を開いて、拡張機能タブを開きます。

検索欄で **Claude Code** を探して、インストールします。

### Claudeを開く

インストール後は、コマンドパレットから Claude を開けます。

その後、`Claude Code` と入力して開きます。

### 何ができるの？

VS Code 版では、たとえば次のようなことができます。

* 今開いているコードの説明
* エラー原因の調査
* 修正案の提案
* 複数ファイルにまたがる変更の補助
* コミットメッセージのたたき作成

変更を伴う場合は、**差分を見ながら確認しやすい** のが強みです。  
そのため、初心者が最初に使うならこちらの方が安心です。

---

## 4. CUIで使う方法

Claude は、ターミナルから直接使うこともできます。

### インストールする

環境に応じて、以下のようにインストールします。

```
# macOS / Linux / WSL
curl -fsSL https://claude.ai/install.sh | bash
```

```
# Windows PowerShell
irm https://claude.ai/install.ps1 | iex
```

### 起動する

プロジェクトのフォルダに移動して、Claude を起動します。

初回はログインを求められるので、そのまま進めれば大丈夫です。  
困ったら、次のコマンドでヘルプを見られます。

### どう使うの？

起動したら、そのまま自然文で指示します。

```
変更内容を要約して、コミットメッセージも考えてください
```

---

## 5. CUIで使うメリット

### ① とにかく速い

エディタ上でパネルを開くより、**そのままターミナルで呼び出せる** ので速いです。

### ② Gitと相性がよい

普段から `git status` や `git diff` を使っている人にはかなり自然です。  
開発の流れを止めずに、そのまま相談できます。

### ③ 小さい依頼を投げやすい

「このエラー文を読んで」「このディレクトリ構成を説明して」程度なら、  
CUI の方がむしろ気軽です。

### ④ 自動化っぽい使い方にもつなげやすい

ターミナル中心なので、ログを見せたり、他のコマンドと組み合わせたりしやすいです。  
少し慣れると、VS Code版とは別の便利さがあります。

---

## 6. VS Code版とCUI版の使い分け

ざっくり言うと、次のように考えると分かりやすいです。

* **VS Code版**  
  差分を見ながら安全に編集したい時向け
* **CUI版**  
  素早く質問したい時、Git やコマンド操作と一緒に使いたい時向け

最初は **VS Code版から入る** のがおすすめです。  
その後、慣れてきたら **軽い確認や素早いやり取りは CUI** と使い分けるとかなり便利です。

---

## 7. 使うときの注意

Claude は便利ですが、出てきた提案を**そのまま無条件で信じるのは危険**です。

特に、次の2つは意識した方がよいです。

* 提案されたコードは自分でも読む
* 実行するコマンドの意味が分からない時は通さない

便利な補助役ではありますが、最終的に責任を持つのは自分です。  
「全部任せる」というより、**説明係・相談役・下書き係として使う** のが安全です。

---

## 8. まとめ

VS CodeでClaudeを使う流れはかなり単純です。

1. 拡張機能を入れる
2. Claude を開く
3. コードについて質問する
4. 提案や差分を見て、自分で採用するか決める

また、ターミナルから使う CUI 版もあります。

* **VS Code版** は見やすくて初心者向け
* **CUI版** は速くて Git と相性がよい

個人的には、最初は VS Code で使い始めて、  
慣れてきたら CUI も併用するのが一番自然だと思います。
