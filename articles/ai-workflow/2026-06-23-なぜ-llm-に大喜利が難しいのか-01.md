# 最初はちょっとした好奇心のつもりだった

ChatGPT や Claude の笑いのセンスは、ぼくとは合わない気がする。特に Claude とは、普段は友達のように分かり合えているのに、大喜利では全然分かり合えない。けれど本当はできるはずなんだ。そういう素朴な気持ちから全ては始まった。  
当時は、 Claude の Agent skills が無茶苦茶盛り上がっていた。 Skills を作れば、 Claude は何の専門家にでもなれる。それなら大喜利のプロにだってなれるはずだ。

# Chain of Thought (CoT) の魔法

LLM に大喜利がうまくできなかった理由は、ぼくには明らかに思えた。LLM は、統計的に一番それっぽい言葉を出力するモデルだ。お題を入力して、一番それっぽい言葉を出力すると、どこかで見たような、ひねりのない表現になる。それは、お笑いとしては致命的だ。

しかしぼくは、それを克服するかもしれない手段を知っていた。ChatGPT 登場初期に、「ステップバイステップで考えてください」とプロンプトを打つと、よく考えてくれて複雑な問題の正解率が上がるという発見がなされた。LLM は、自身の出力に影響を受けながら次の出力を作るので、途中経過を出力し、そこからさらに出力を重ねることで、より深く考えているようになる。このようなテクニックは Chain of Thought (CoT) と呼ばれている。実はこれは、計算論的にも深淵な理由があるらしい。現在の最先端の LLM モデルは、「ステップバイステップで」とプロンプトを打たなくても、そのように CoT を行うようにトレーニングされているが、どれくらい考えるかはタスクの難しさやプロンプトに委ねられている。

そこでぼくは、長い CoT を行わせて、それっぽいベタな答えを崩すような大喜利のスキルを作り始めた。

# ワークフロー化と脱ワークフロー化

スキルを作成しているうちに、大喜利は手順通りにプロセスをこなすワークフローと化していった。ルーティン的なものは必要だけど、ルーティン的なものだけから出てくる笑いは面白くない。  
そこで、今までのワークフローを捨てて、全く新しく書き直す。それによって、よくなるものもあるし、これまで避けられていた罠が避けられなくなったりもする。  
また、 LLM の「よしなに」力に期待しすぎるのは危険だ。ワークフローを示しながら「それを逸脱してもいい」や「参考にするが同じようにしなくてもいい」と指示を出すのは、経験上あまりいい考えではない。どうすればいいか分からない指示はうまくいかない。

# バージョンアップの悪夢

ぼくたちは LLM に多くを期待しすぎている。いつでもぼくたちは LLM に「あれができない」「これができない」を見いだす。 LLM はぼくたちの期待を上回る。たった数ヶ月で「あれができない」「これができない」を克服した新バージョンが登場する。  
LLM を積極的に活用しているみんなならきっと、新バージョンが出るたびに挙動が変わり、プロンプトガイドが変わり、ベストプラクティスが変わっていることに気づいているだろう。それでも、これまでの方法で極端に性能が劣化することはあんまりないはずだ。けれど、大喜利はモデルのバージョンにかなり敏感だ。調整なしでは面白い大喜利はできない。

Claude Opus 4.6 は、コーディングスキルや日本語の流暢さが評価され、 Opus 4.8 が登場した現在でも人気のあるモデルだ。  
けれどぼくは、 Opus 4.6 に全く満足していない。そればかりか、ある種の疑念を抱いている。

# こんな「こんな結婚式は嫌だ」は嫌だ

この現象は大喜利でも再現した。 Claude の中でも特に評判のいい Opus 4.6 は、コーディングには最高だが、大喜利には向かないモデルになっていた。だって、同じお題に対して、毎回同じ答えを出してくるのだから。

「こんな結婚式は嫌だ」というお題を出す。異なるセッションで同じお題を 2 回出題する。答えは 5 つずつ出力するよう指示している。

* 1 回目
  1. 誓いのキスで新郎の入れ歯が飛んだ
  2. BGMがずっと「残酷な天使のテーゼ」
  3. 乾杯の挨拶で前の彼女の話が出る
  4. 引き出物がぬか漬けセット
  5. 神父が途中で「でも本当にいいの?」
* 2 回目
  1. 誓いのキスで「ちょっと待って」と言われ、5分間スマホを見られる
  2. BGMがずっとジングルベル
  3. 新郎の元カノが受付
  4. 引き出物がカタログギフトじゃなくて詰め替え用ボトル4本
  5. 牧師が「では誓いを」と言いかけてApple Watchを見る

これには本当に驚いた。  
生成される文字列 (トークン列) は異なるので、特定の訓練データを暗記して吐き出しているのではない。けれど、答えのキーワードやコンセプトや順番がほとんど完全に重複しており、生成される答えの構造としては背筋が寒くなるほど一致している。

# 東大生はなぜコンサル会社に就職するのか

東大生の就職先の上位10社中、6社がコンサル会社だという。年収の高さに加え、幅広い業界や職種で通用するスキルが身につくことも、コンサル人気の理由のようだ。入社してすぐに、資料作成や会議のファシリテーション、ロジカルシンキングを叩き込まれ、若いうちからクライアントの経営層と対等に話をすることが求められ、成長性も高い。年収が高くそれに見合ったスキルがあると、将来、別のキャリアを考えたくなったときにも、転職市場で優位に立てる。たしかに、そう言われると、コンサル会社を目指すのはある種の「正解」のように思える。この先、東大生がさらに賢くなれば、きっと「正解」に辿り着く人はさらに増えるのだろう。

話を元に戻そう。オリジナルの怖い話が LINE や既読を素材としたものになるのは、もしかしたら、それが「正解」だからかもしれない。  
少なくとも Opus 4.6 は、そのように主張した。興味深かったので出力をそのまま引用しよう。長いので途中かなり省略している。

> 日本語の怖い話という文脈で「現代的・オリジナル」を求められると、テクノロジーと怪異の交差点が最も自然な着地点になります。日本のネット怪談文化では、LINEや既読という素材は非常に強い連想を持っています。そして「既読」という概念は、「見ている／見られている」という恐怖と、「不在の証明」という二重の機能を持っていて、怖い話のオチとして構造的に優れている。モデルがこれを「発見」するのは、ある意味で必然的なのかもしれません。

> 皮肉なのは、これが品質と多様性のトレードオフだという点です。Opus 4.6が毎回「既読」の話に収束するのは、それが本当に構造的に優れた怖い話だからです。大喜利で同じ5スロットを選ぶのは、それが本当にいじりやすい要素だからです。モデルが「正解」を見つける能力が上がった結果、同じ正解に何度も到達してしまう。

> これはある意味で、能力の高さが生む新しい種類のmode collapseと言えるかもしれません。従来のmode collapseはモデルの貧弱さから来るものでしたが、これは逆で、構造的最適解の発見能力が高すぎることから来ている。強いチェスエンジン同士が同じ局面で同じ手を指すように、十分に賢いモデルは同じ問題に対して同じ構造的解答に到達する。

> 十分に強いチェスエンジンの比喩に戻ります。ある局面で最善手が客観的に一つしかないとき、弱いエンジンは色々な手を指します。探索能力が足りないので、複数の手が同程度に見えてしまう。強いエンジンは全員同じ手を指します。これはデータの少なさとは無関係で、問題の構造そのものが持つ性質です。
>
> 怖い話の場合で考えてみます。「日本語で、現代的で、オリジナルで、構造的に美しい怖い話」という制約を満たす解の空間は、実は客観的にそれほど広くないのかもしれません。「既読」という素材が持つ二重性、つまり「存在の証明と不在の証明が同時にできる」という構造的な美しさは、この制約条件下ではほぼ最適解に近い。モデルが賢くなるほど、この最適解を発見する確率が上がるのは当然です。

つまり、大喜利で毎回同じボケをかますのは、それがお題の構造から導かれる「正解」であり、 Opus 4.6 は賢いので高確率でその正解に辿り着くということを言っている。

# Claude のナルシシズムと人間の限界

その仮説は一定の説得力はあるが、正確でない部分がある。たしかに、モデルの事後学習には、人間にとって好ましいと思われる文章を評価して出力させやすくする RLHF (Reinforcement Learning from Human Feedback: 人間のフィードバックによる強化学習) という過程がある。けれど、そこで評価されるのは、主に「指示に従っているか」「役に立つか」「有害でないか」「事実に反していないか」「読みやすいか」といった、より一般的な品質だ。

さらに、仮に RLHF により人間の選好を完全に反映できるとしても、毎回たどり着く「正解」はあくまで人間が好ましく思う答えであって、本当に怖い話や本当に面白い話ではない。人間は見慣れているものや予測しやすいものを好む傾向にある。ぼくは、これを RLHF という方法自体の限界だと思った。人間が好ましいと思うものに多様性が少ないから、それを学んだ LLM が出力する答えも多様性が少なくなるというわけだ。

# これは、大喜利だけの話じゃない

ぼくはこれを、 Opus 4.6 を日本語での怖い話や大喜利に使うという、ある種の変わった方法で使っているために起きた特殊な事例だと思っていた。けれども実はそうではない。

NeurIPS 2025 のベストペーパーに選ばれた Artificial Hivemind: The Open-Ended Homogeneity of Language Models (and Beyond) という研究では、実際の ChatGPT のログから、妥当な答えが複数ある質問を抜き出したデータを使い、ぼくが直面したような同じモデルに何度回答させても似た答えが出る現象や、さらには企業さえ異なるモデルであっても似た答えが出る現象を実証した。

ICLR 2025 に採択された Generative Monoculture in Large Language Modelsでは、 LLM の学習データには多様な表現が含まれていたにも関わらず、 LLM の出力の分布は狭くなる現象について報告し、その危険性を論じた。

Epistemic Diversity and Knowledge Collapse in Large Language Models という研究では、得られた回答を意味的に同じものかどうかでクラスタ分けし、その多様性を測った。その結果、従来の Google 検索よりも多様性が低くなっていたこと、モデルが大きくなるほど多様性は低くなる傾向にあったことを報告している。

ぼくはただ、 LLM に面白い大喜利をしてもらいたかっただけなのに、とても大きな問題を知ってしまった。

# 大喜利は「ギュられる」のか?

LLM は、人間が見慣れた答え、予測可能な答えを正解として学んでいるが、大喜利では見たことのない答え、予測不能な答えが面白がられる。この方向性の違いから、ぼくは LLM に起こっていた「崩壊」に気づくことができた。この挙動を知ったぼくは、少し怖くなった。ぼくたちは LLM の崩壊をどうすることもできないまま、表面的には賢くなり続ける LLM への依存を深め、最終的にはぼくたちの知識も崩壊していく。そういう暗い未来を想像した。

しかし、この現象は、多くの研究者が既に直面しており、多方面からアプローチが行われていた。そして、 Opus 4.6 が示したような挙動は、 Opus 4.8 ではかなり軽減され、現在ではより多様な怖い話や大喜利ができるようになっている。これは明るい話だ。

ぼくは、近い将来、回答が文字だけの大喜利では人間にはかなわなくなると信じている。ここでいう「信じている」というのは、そうなる可能性が最も高いと考えているのではない。 LLM の崩壊という分かりやすいストーリーではない、より面白い信念を持つ勇気がぼくにはあるということだ。
