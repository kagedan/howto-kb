「**自分のPCでAIを動かしたい**」と思ったことはありませんか？

ClaudeやChatGPTはクラウド上で動いているため、会話の内容がサーバーに送られます。機密情報を扱う場合や、純粋に「自分のマシンで完結させたい」という場合には、ローカルLLMという選択肢があります。

そこで実際にローカルLLMを構築し動かしてみたところある疑問が浮かびました。  
**普段使っているClaude Opus 4.6と同等の精度をローカルで出すにはどのくらいのスペックが必要なのか？**

本記事ではまずPowerShell上でローカルLLMを動かしてチャットするところまでを試し、その結果からClaude Opus 4.6と同等のAIをローカルで動かすにはいくらかかるかについて調べてみます。

## ローカルLLMとは

そもそもローカルLLMとは、自分のPC上で動かす大規模言語モデルのことです。推論内容はサーバーに送信されず、ネットワークから切り離されている状態のPCでも会話が可能です。

ローカルLLMで使用可能なモデルについてですが、ClaudeやGPTはAnthropicやOpenAIがモデルの重みを非公開にしているクローズドソースモデルなので動かすことはできません。オープンウェイトモデルと呼ばれる、MetaのLlamaやAlibabaのQwenなどの重みが公開されているモデルを利用します。

**ローカルLLMのメリット：**

* 完全オフラインで動作する
* 会話内容が外部に送られない
* 機密情報を扱える
* API費用がかからない
* 必要なのは電気代だけ

**デメリット：**

* モデルサイズに上限がある（VRAMに依存）
* 一般的にはクラウドLLMより精度が落ちる
* 環境構築が必要

現状では、かなりの高スペックPCでない限りはクラウドLLMより推論速度や精度が落ちます。詳しくは後述。

## 環境・スペック

今回使用した環境は以下の通りです。

| 項目 | 内容 |
| --- | --- |
| OS | Windows 10 Home 64bit |
| CPU | AMD Ryzen 7 5800X |
| RAM | 16GB |
| GPU | NVIDIA GeForce RTX 3070 (VRAM 8GB) |

## モデル選定の理由

VRAMとモデルサイズの関係は以下の通りです。

| VRAM | 動かせるモデル規模 |
| --- | --- |
| 4GB | 〜3Bモデル |
| 8GB | 〜7Bモデル |
| 16GB | 〜13Bモデル |
| 24GB | 〜30Bモデル |

今回はRTX 3070（VRAM 8GB）で動かすので、**qwen2.5:7b**を選びました。QwenシリーズはAlibabaが開発したモデルで、同規模のモデルの中では日本語の精度が比較的高いことが選定理由です。

## 構築手順

では実際に構築していきましょう。

### 1. Ollamaをインストール

Ollamaは最も手軽にローカルLLMを動かせるランタイムです。  
<https://ollama.com> にアクセスして「Download for Windows」からインストーラーをダウンロード。

![](https://static.zenn.studio/user-upload/6a8ec8095f95-20260324.png)

.exeファイルを実行します。

![](https://static.zenn.studio/user-upload/54d423c2fc97-20260324.png)

### 2. モデルをダウンロードして起動

PowerShellを開いて以下を実行します。

![](https://static.zenn.studio/user-upload/5347e3301398-20260324.png)

初回はモデルファイル（約5GB）を自動ダウンロードします。回線速度にもよりますが10〜20分程度かかります。

![](https://static.zenn.studio/user-upload/57c447315084-20260324.png)

ダウンロード完了後、そのままターミナル上でチャットが始まります。

![](https://static.zenn.studio/user-upload/5076eb2baf57-20260324.png)

> 実用的に使うにはOpen WebUIを使ったUI整備やRAGの導入なども必要になりますが、それはまた別の記事で取り上げたいと思います。

### 3. モデルの停止

使い終わったらVRAMを解放できます。

モデルファイル自体はローカルに残るため、次回は再ダウンロード不要です。

日本語で流暢に回答してくれましたが、よく見ると「千種駅」は名古屋の駅で、紹介された店名も正しくはありません。一風堂は北1条にはありませんし、大勝軒は札幌にありません。名古屋の南1条にも存在しませんでした。これはハルシネーション（幻覚）と呼ばれる現象で、LLMが存在しない情報を自信満々に生成してしまうことがあります。

## ハルシネーションの考察

2回の質問でいずれも架空の店名・駅名が出力されました。ハルシネーションを起こしています。

ハルシネーションは、生成AIが事実とは異なる情報や存在しない情報を、もっともらしく生成するといった問題です。LLMは「次のトークンとして確率的に最もありそうな文字列を生成する」仕組みのため、実在しない情報も自信満々に出力することがあります。

ハルシネーションの発生原因はモデルサイズだけでなく、学習データの偏りや誤り、専門知識の不足、論理推論能力の欠如など複合的な要因によるものです。**大規模モデルほど知識量は増えますが、同時に「もっともらしい嘘」をつく能力も巧妙になる**ため、モデルを大きくすれば解決するわけでもありません。

今回のケースに限って言えば、**7Bという小規模モデルがローカルな店舗情報という学習データが少ない専門領域で質問されたこと**が主な原因と考えられます。

VRAMが少ないと大きなモデルを動かせないため、結果的にハルシネーションが起きやすくなります。「メモリが足りないから精度が低い」という理解は厳密には正確ではなく、直接の原因はモデルサイズ（知識量・推論能力）にあります。

ここで自然と湧いてくる疑問があります。「では、クラウドLLMと同等の精度をローカルで出すにはどのくらいのスペックが必要なのか？」

## Claude Opus 4.6と同等の精度をローカルで出すには？

Claude Opus 4.6の正確なパラメータ数はAnthropicが非公開しています。ただしX（旧Twitter）で公開された興味深い検証データがあります。

<https://x.com/N8Programs/status/2025785575236624516?s=20>

@N8Programs氏の検証によると、「思考なし（Reasoning-Free）」の正答率とパラメータ数には対数線形の相関（R²=0.877）があることが示されています。

> acc = 18.5 × log₁₀(p) + 11.5

この回帰式に各モデルのスコアを当てはめると、以下のように推定されます。

| モデル | 推定パラメータ数 |
| --- | --- |
| Claude Opus 4.6 | 約5,800B |
| Claude Sonnet 4.6 | 約961B |
| GPT-5.2 | 約900B |
| Claude Haiku 4.5 | 約114B |

なお、別の情報源ではGPT-5を約1,800Bと推定するものもあり、推定値は情報源によって大きく異なります。いずれも非公式な推定であり、2倍程度の誤差は覚悟しておく必要があります。

Claude Opus 4.6が約5,800Bとすると、fp16で動かすだけで必要なVRAMは以下の計算になります。

```
5,800B × 2バイト（fp16）≈ 約11.6TB
```

**約11.6TBのVRAMが必要**です。個人がローカルで同等のモデルを動かすのがいかに非現実的かを実感できます。

11.6TBのVRAMを構築するには…？

| 構成 | VRAM合計 | 金額目安 |
| --- | --- | --- |
| RTX 5090（32GB）× 372枚 | 11.6TB (11,904GB) | 約1億4,880万円 |
| Mac Studio M3 Ultra（256GB）× 47台 | 11.8TB (12,032GB) | 約4,835万円 |

**RTX 5090が372枚**、**Mac Studio M3 Ultraが47台**...もはや個人では現実的ではありません

ちなみに、Anthropic Proプランを$20(3,000円)/月で使い続けた場合と比較すると…

```
4,835万円 ÷ 3,000円/月 = 16,117ヶ月 ≈ 約1,343年分
```

飛鳥時代から現在までの期間に相当します。ロマンはありますが、現時点では料金面だけを見るとクラウドLLMに分があります。

ロマン計算をしてみましたが、技術は日々進歩しています。この記事を執筆している前日には「LLM in a Flash」という手法でiPhoneで397Bモデルを動かすことに成功した事例も登場しており、5〜10年後にはローカルLLMの常識が大きく変わっている可能性もあります。

<https://gigazine.net/news/20260324-iphone-397b-ai/>

## まとめ

ローカルLLMを構築し最低限の推論精度を求める場合、ある程度のメモリが必要ということがわかりました。正確な情報が必要なタスクにはRAGを組み合わせるかクラウドLLMを使うのが現実的ですが、ローカルLLMも以下の用途では実用的です。

* 社内文書を参照させるRAG構成（機密情報を外部に出したくない場合）
* 災害時などネットワークから遮断された場合の利用
* 文章の要約・校正
* アイデア出し・雑談

ローカルLLMに一番求める点としては、情報を外部に持ち出さない安全性ではないでしょうか。私自身の経験則としても、機密情報を提供しないように気を付けていますし、何らかの方法でマスクして渡すこともあります。そのストレスから解放されるのはローカルLLMの唯一無二の魅力です。

今回はRTX 3070 + Ollamaの組み合わせで30分以内にローカルLLMを構築してチャットを行い、完全オフラインで動作することも確認できました。

ハルシネーションの課題はありますが、用途を絞れば災害時などには実用的だと感じました。数年後にはMacBookの吊るしモデルでRAM 32GBとかになっていることを想像していますが、モデルがより軽量に動くような進歩も願います。
