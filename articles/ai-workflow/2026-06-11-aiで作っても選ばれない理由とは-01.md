AIでサムネ、LP、UI、資料を作れる人が増えました。  
その結果、**「作れること」自体の価値は下がっています**。

問題は、AIの性能不足ではありません。  
作る前に、**誰へ何を届けるかを決めていないこと**です。

読み終わると、**AIに依頼する前に何を決めるべきか、出てきた案を何で判断すべきかが分かります**。

## AIで作れる人が増えるほど、設計不足が見える

![](https://assets.st-note.com/img/1781137501-1ewUEYDV2Z9l4x0MsSgi7rpd.png?width=1200)

AI時代に差がつくのは、作業速度ではありません。  
差がつくのは、何を作るべきかを決める判断です。

> 画像、文章、UI、コード、動画、音楽

以前は専門職や制作チームが必要だったものを、今は一人でも形にできます。

これは便利です。

ただし、便利になった分だけ、別の問題が見えます。

作れる人が増えるほど、次のような制作物も増えます。

* **きれいだが、誰向けか分からない**
* **速く作ったが、使う理由が弱い**
* **情報量は多いが、刺さる感情がない**
* **AIっぽい完成度はあるが、記憶に残らない**
* **修正するたびに、方向性が変わる**

原因は、ツール不足ではありません。  
制作前の**設計不足**です。

AIは、出力を速くします。

しかし、誰を動かすか、どこまで作れば価値になるか、何を捨てるかは自動で決めてくれません。

ここを人間が決めないままAIを使うと、速く凡庸なものが増えます。  
逆に、先に判断軸を決めると、AIは制作チームになります。

## 最初の問いは「誰のために作るか」

![](https://assets.st-note.com/img/1781137521-BQV2NI0bOnZivAzUEc7X41aj.png?width=1200)

制作前に最初に決めるべきことは、ターゲットではありません。  
「**誰が、どんな反応をしたら成功か**」です。

よくある失敗は、対象を広げすぎることです。

* **初心者にも分かる**
* **経営者にも刺さる**
* **若手にも使える**
* **既存顧客にも新規顧客にも届く**
* **とにかく多くの人に見てほしい**

この方向に進むほど、制作物は弱くなります。  
対象が広いほど、一人の欲望への解像度が下がるからです。

「みんなに使ってほしい」は、制作前の願望です。  
設計ではありません。

設計として書くなら、次の形にします。

```
誰が:
AIで制作時間を短縮したいが、安っぽいAI感を避けたいデザイナー

どんな状態で:
クライアント提案前に、サムネやLPの方向性で迷っている

どう反応したら成功か:
「この方向なら提案できる」と判断し、1案を選べる
```

ここまで書くと、デザインの判断が変わります。

たとえば、同じAIサムネでも、初心者向けなら安心感が必要です。  
プロ向けなら、説明の丁寧さより、切れ味や編集感が必要です。

同じ「分かりやすい」でも、相手によって意味が変わります。  
初心者には迷わないこと。  
プロには意図が一瞬で伝わること。

だから、最初に決めるのは属性ではありません。  
欲望と反応です。

## 次に決めるのは「どこまでやれば価値か」

![](https://assets.st-note.com/img/1781137548-aJxY9q7fcVjHt1ndKQg3lZum.png?width=1200)

誰に向けるかを決めたら、次は**ゴールライン**を決めます。  
ここがない制作は、終わりません。

AIを使うと、案はいくらでも出せます。  
だからこそ、次の罠に入りやすくなります。

* **もう1案だけ出す**
* **もう少し高級感を足す**
* **もう少し情報を入れる**
* **競合っぽい表現も試す**
* **念のため別方向も残す**

これを続けると、制作物は磨かれるのではなく、ぼやけます。

必要なのは、未来の反応を先に書くことです。  
完成後に言われたい言葉を、制作前に決めます。

たとえば、noteサムネならこうです。

```
完成後に読者が言うこと:
「自分がAIで作っても選ばれない理由はこれかもしれない」

行動:
記事を開き、制作前チェックリストを保存する

最低限満たす品質:
note一覧で主コピーが読める
AI制作の話だと分かる
ツール紹介ではなく判断軸の話だと伝わる
```

この未来の反応が、ゴールラインになります。

重要なのは、完成物の量ではありません。  
読者の反応が変わるところまで作ることです。

**サムネ**なら、読まれるきっかけを作るところまで。  
**LP**なら、次の行動を選べるところまで。  
**UI**なら、迷わず使えるところまで。  
**資料**なら、相手が判断できるところまで。

価値は、作り手の作業量では決まりません。  
受け手の反応が変わるかで決まります。

## 最後に決めるのは「何を優先するか」

![](https://assets.st-note.com/img/1781137572-DZBHIutoCpTUs0lhAPKzvmw9.png?width=1200)

制作物が弱くなる原因は、意見が多いことではありません。  
優先順位がないことです。

制作中は、必ず横から意見が入ります。

* **もっとシンプルにしたい**
* **もっと情報を入れたい**
* **もっと目立たせたい**
* **もっと高級感を出したい**
* **もっと親しみやすくしたい**
* **もっと早く出したい**

どれも間違いではありません。  
ただし、全部を同時には満たせません。

だから、制作前に原則を決めます。  
これはスローガンではなく、迷った時に使う判断ルールです。

弱い原則はこうです。

```
ユーザーを大切にする
分かりやすくする
品質を高める
ブランドらしくする
```

これでは判断できません。  
誰も反対しない言葉だからです。

使える原則は、優先順位が書かれています。

```
速さより、初見の理解を優先する
情報量より、主張の強さを優先する
万人受けより、対象読者の納得を優先する
装飾より、スマホ一覧での可読性を優先する
AIっぽい派手さより、提案に使える信頼感を優先する
```

ここまで書くと、修正判断が速くなります。

> **「この案はかっこいいが、スマホ一覧で読めない」  
> 「この案は情報が多いが、主張が弱い」  
> 「この案は目立つが、対象読者の信頼を落とす」**

判断が感覚ではなくなります。  
制作前に決めた原則へ戻るだけです。

AI時代の制作では、この原則が特に重要です。  
AIは複数案をすぐ出します。  
しかし、どれを採用するかは人間の責任です。

## AIへ渡す前にこの設計シートを埋める

![](https://assets.st-note.com/img/1781137632-NfJbnB38Ir2A4vsc9HREhSyO.png?width=1200)

AIへ依頼する前に、1枚だけ設計シートを作ります。  
これだけで、出力の質と修正速度が変わります。

使う項目は7つです。

* **作るもの**
* **誰のため**
* **今の困りごと**
* **満たす欲望**
* **成功時の反応**
* **価値のゴールライン**
* **優先すること、捨てること**

そのまま使える形にすると、こうです。

```
作るもの:
note記事のサムネイル

誰のため:
AIで制作時間を短縮したいが、安っぽいAI感を避けたいデザイナー

今の困りごと:
AIで案は出せるが、どれを採用すべきか判断できない

満たす欲望:
選ばれる制作物を作りたい
クライアントに説明できる判断軸がほしい
AI時代でも価値ある制作者でいたい

成功時の反応:
「自分に必要なのはツール追加ではなく設計だった」と気づく

価値のゴールライン:
制作前に3つの問いを埋められる
AIへの依頼文へ判断軸を入れられる
出力案を感覚ではなく原則で選べる

優先すること:
初見の分かりやすさ
スマホでの可読性
実務で使える信頼感

捨てること:
万人受け
AIっぽい派手さ
細かい装飾
情報の詰め込み
```

このシートを埋めたあと、AIへ依頼します。

プロンプト例です。

```
以下の設計シートをもとに、note記事サムネイル案を5案出してください。

条件:
- 誰向けかが一瞬で伝わる
- 画像内文字は最大2要素
- スマホ一覧で読める
- AIっぽい派手な背景は禁止
- 公式ロゴや実在企業ロゴは禁止
- 5案それぞれ、狙う読者感情と採用理由も書く

設計シート:
{ここに設計シートを貼る}
```

AIに「いい感じにして」と頼むほど、出力は曖昧になります。  
AIに判断軸を渡すほど、制作物は使いやすくなります。

## やってはいけない失敗パターン

![](https://assets.st-note.com/img/1781137651-ji9ZclT4kYJdMnA1Bto0SQu5.png?width=1200)

AI時代の制作で避けるべき失敗は、ツール選びに逃げることです。  
出力が弱い時、原因は新しいAIが足りないことではない場合が多いです。

よくある失敗は5つあります。

* **ターゲットを広げすぎる**
* **欲望ではなく属性だけを見る**
* **完成後の反応を書かない**
* **優先順位を決めない**
* **AI出力を雰囲気で選ぶ**

特に危ないのは、属性だけで読者を見たつもりになることです。

「20代女性」「経営者」「会社員」「デザイナー」だけでは、設計できません。

必要なのは、その人が何を避けたいか、何を得たいか、何を言える状態になりたいかです。

たとえば、同じデザイナーでも欲望は違います。

* **制作時間を減らしたい**
* **AIっぽさを消したい**
* **クライアントへ説明したい**
* **修正回数を減らしたい**
* **単価を落としたくない**
* **自分の価値を言語化したい**

どの欲望を狙うかで、デザインもコピーも変わります。

もう一つ危ないのは、優先順位のない修正です。  
意見を全部足すと、無難になります。

無難な制作物は、嫌われにくいです。  
しかし、選ばれにくいです。

AIで平均点の出力が増える時代に、無難さは強みになりません。  
必要なのは、誰に強く届かせるかを決めることです。

## 今日使う制作前チェックリスト

![](https://assets.st-note.com/img/1781137671-RF6p3UOvnY7Ve5LMlcSuZT4m.png?width=1200)

最後に、次の制作前に使うチェックリストを置きます。  
AIを開く前に、ここだけ埋めてください。

```
□ 誰のために作るかを1人まで絞ったか
□ その人の困りごとを行動レベルで書いたか
□ その人の欲望を言葉にしたか
□ 成功時に言ってほしい一言を書いたか
□ どこまで作れば価値かを決めたか
□ 何を優先し、何を捨てるかを書いたか
□ AIへ渡す前に、判断軸をプロンプトへ入れたか
□ 出力案を雰囲気ではなく原則で比較したか
```

AI時代に、手を動かす価値がなくなるわけではありません。  
ただし、手を動かす前の判断の価値が上がります。

> **何を作るか。  
> 誰に届けるか。  
> どこまで作れば価値になるか。  
> 何を優先し、何を捨てるか。**

この4つを決められる人は、AIを使っても制作物がぶれません。  
逆に、ここを決められない人は、AIを使うほど迷います。

ツールを増やす前に、設計シートを作る。  
これが、AI時代に選ばれる制作の最初の一歩です。
