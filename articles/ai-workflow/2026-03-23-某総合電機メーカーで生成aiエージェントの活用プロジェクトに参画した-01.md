# はじめに

こんにちは。  
ネイバーズ株式会社の河口です。  
2025年6月に入社し、現在は主にPMO業務などに従事しています。

今回、日本でも有数の総合電機メーカーにおいて、**生成AIエージェントの活用プロジェクト**に参加する機会をいただきました。

# 対象読者

* 生成AIエージェントの活用事例に興味がある方
* 生成AIを用いた企業の取り組みに興味がある方

# プロジェクト概要

本プロジェクトでは、生成AIエージェントを活用したチャットボットの構築に取り組みました。

* チャットボット
* 生成AIエージェント（Microsoft Copilot）使用
* ナレッジ（知識）がSharePointやTeams主体で管理されており、Copilotと相性が良い
* 数年に一度、似たような作業工程の発生が予想される業務

これらの課題に対し、チャットボットによる業務支援・代替を目指しました。

# 方法

以下のステップで構築を進めました。

* 過去のプロジェクト（PJ）に関するExcelファイルやTeams（Microsoft社製チャットツール）の履歴などをナレッジ（知識）としてCopilot Studioに読み込ませる
* ナレッジをもとに、質問に対して回答を生成するプロンプトを作成する  
  　（例：このPJの要件定義プロセスに必要な期間を概算してください）
* プロンプトを改良し、出力を改善させる（プロンプトエンジニアリング）

# 結果

* チャットボットの正答率を50％程度→**80％以上**に向上させることに成功  
  実務利用に耐えうるレベルまで精度を改善させることができました。
* プロンプト設計の**ノウハウを他のプロジェクトに展開**することで、**さらなる利便性の向上**に貢献  
  将来的には、他業務にエージェントを応用することで、より発展的な活用を見込んでいます。

# 気づき・学び

ここからは、本プロジェクトで得られた学びをまとめます。

## 車の両輪…「ナレッジ」と「プロンプト」

**生成AIエージェントの基礎**は、良質な「ナレッジ」（知識）と「プロンプト」（指示）です。  
この**どちらが欠けても、実用的なエージェントにはなりえません。**  
これらのうち、ナレッジの蓄積には時間がかかりますが、**プロンプトの改善にはすぐに着手できます。**  
実際のプロジェクトでも、プロンプト改善の試行錯誤に時間を割き、より良いチャットボットを目指しました。

## 言語化の難しさ

特に難しかったのが、「AIエージェントにどう動いてほしいか」を**言語化**することでした。  
プロンプトに曖昧な指示文を書いても、適切な回答文を得ることはできません。  
一見問題なく動いているように見えても、**回答の精度が不十分**だったり、間違った答えを生成してしまったり（**ハルシネーション**）することがありました。  
そのためにも、社員の方々と綿密にコミュニケーションを図り、  
「チャットボットの出力を改善するために、プロンプトの方向性をどうすべきか」  
「より正確な回答を出力するため、新たに用意すべきナレッジ資料はあるか」  
といった内容の打ち合わせを日々積み重ね、チャットボットに関する**認識のすり合わせ**を行いました。

## 継続的な改善・チューニングの大切さ

AIエージェントは「作って終わり」ではありません。  
ナレッジの追加等の要因によって、「最適」なプロンプトもまた移り変わっていきます。  
**日々改良を重ね、より良い成果物を目指してノウハウを積んでいく**ことの大切さを実感しました。

# おわりに

今回のプロジェクトは、私のキャリアの中でも特にインパクトの大きいものの一つとなりました。  
生成AIエージェントを用いたチャットボットが実務で使用されるにはまだ課題がありますが、いずれは確実に実用化が進んでいくと感じています。  
今後もこの経験を活かし、より実務に即した生成AI活用を推進していきたいと考えています。
