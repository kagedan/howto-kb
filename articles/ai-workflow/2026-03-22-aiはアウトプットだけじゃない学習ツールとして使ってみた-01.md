### Step 1: 基本的なI/Oと型

```
  src/Main.hs を編集して、以下の動作を実現してください：

  1. "何を追加しますか？" と表示する
  2. ユーザーの入力を受け取る
  3. "追加しました: <入力内容>" と表示する

  ヒント

  使う関数は3つだけです：

  - putStrLn :: String -> IO () — 文字列を表示（改行付き）
  - getLine :: IO String — 1行入力を受け取る
  - <- — IO の結果を変数に束縛する（do記法の中で使う）

  理解のポイント

  書きながら以下を意識してみてください：

  - getLine の型は IO String であって String ではない。<- を使って初めて String を取り出せる
  - do ブロック内の各行は 上から順に実行される（IOモナドが順序を保証している）
  - putStrLn は String -> IO () — 文字列を受け取って「画面に出力するという副作用」を返す
```

Haskellを少しだけかじっていたので、なんとなくわかりますが、初見だったら型シグネチャあたりでだいぶ混乱していたかもしれません。  
わからない箇所があればどんどん質問していきます。

### Step 2: Todoのデータ型を定義する

```
data Todo = Todo { title :: String, done :: Bool } deriving (Show, Read)
```

最終的にこんな感じでデータ型を定義しましたが、derivingを学びました。  
derivingで、この型が操作できる内容の実装を自動生成している。  
例えば、この例だと`show`でTodoデータを文字列化したり`read`で文字列から復元できたりするってことですね。なるほど、便利ですね。

### Step 3: コマンドのパースとメインループ

```
以下の動作を実現してください：

  1. >  のようなプロンプトを表示して入力を待つ
  2. 入力に応じて処理を分岐する：
    - add <タイトル> → 「追加しました: <タイトル>」と表示
    - list → 「一覧表示（未実装）」と表示
    - done <番号> → 「完了しました: <番号>」と表示
    - quit → 終了
    - それ以外 → 「不明なコマンドです」と表示
  3. quit 以外ならループして再び入力を待つ
```

アプリケーションっぽくなってきましたが、記法が全然わからないのでAIに質問しながら進めていきます。

```
module Main (main) where
import System.IO (hFlush, stdout)

data Todo = Todo { title :: String, done :: Bool } deriving (Show, Read)

loop :: [Todo] -> IO ()
loop todoList = do 
  putStr "> "
  hFlush stdout
  input <- getLine 
  case words input of 
    ["quit"] -> do
      putStrLn "終了します"
      return ()
    ["list"] -> do 
      putStrLn "一覧表示（未実装）"
      loop todoList
    ("add": rest) -> do
      let title = unwords rest
      let todo = Todo { title = title, done = False }
      putStrLn $ "追加しました！: " ++ show todo
      loop (todoList ++ [todo])
    _ -> do
      putStrLn "不正なコマンドです。"
      loop todoList

main :: IO ()
main = loop []
```

ここまででこんな感じのコードになっています。関数型言語なのでfor文などは使わず、再帰でループを表現しています。面白い！  
Todoのリストを引数にとるloop関数を新たに作成し、quit以外の入力が得られたら更新したリストを再帰loopの引数に渡す感じですね。

### Step 4: Todoリストの操作（純粋関数）

```
  以下の純粋関数を実装してください：

  addTodo :: String -> [Todo] -> [Todo]
  completeTodo :: Int -> [Todo] -> [Todo]
  showTodo :: Todo -> String
  showTodos :: [Todo] -> String

  それぞれの仕様：

  - addTodo — タイトルを受け取り、未完了の Todo をリストの末尾に追加
  - completeTodo — 番号（1始まり）を受け取り、該当の Todo の done を True にする
  - showTodo — 1つの Todo を "[ ] 牛乳を買う" や "[x] 掃除する" の形式で表示
  - showTodos — リスト全体を番号付きで表示（"1: [ ] 牛乳を買う\n2: [x] 掃除する"）
```

ここで面白かったのがデータの更新処理です(todoをdoneにする)。普段慣れ親しんだ言語だと下記のように実装してしまいがちなのですが、

```
todoList[index].done = true
```

Haskellでは純粋関数型のため、副作用が許されません。そのためリストを直接変更するのではなく、更新済みの新しいリストを生成して返す必要があります。  
こんな感じで実装しました。

```
completeTodo :: Int -> [Todo] -> [Todo]
completeTodo index todoList = take index todoList ++ [updatedTodo] ++ drop (index + 1) todoList
  where
    oldTodo = todoList !! index
    updatedTodo = oldTodo { done = True }
```

### Step 5: ファイルへの永続化

```
1. 終了時 — quit したときに Todo リストをファイルに保存する
2. 起動時 — ファイルが存在すれば読み込み、なければ空リストで開始する
```

今回は文法や記法に重きを置いているのであまり深掘りませんでしたが、想像以上に簡単にファイル間のやり取りができました。

```
-- 読み込み
import System.Directory (doesFileExist) -- これが必要

main :: IO ()
main = do
  exists <- doesFileExist "todos.txt" -- ファイル確認
  todoList <- if exists
    then do
      contents <- readFile "todos.txt" -- 内容読み込み
      return (read contents :: [Todo])
    else return []
  loop todoList
  
-- 書き込み
["quit"] -> do
    putStrLn "終了します"
    writeFile "todos.txt" (show todoList)　-- これだけ
    return ()

-- +α System.Directory を使うには依存関係にdirectoryを指定する必要があり、
-- プロジェクトセットアップ時に生成されたtodo-app.cabalの build-depends に
-- directory を追加してbuildしなおすだけでした。
```

これでtodoListの永続化ができました。

## まとめ

AIに先生役をやってもらってHaskellの勉強をしてみましたが、Haskellに対してかなり解像度が上がりました。  
見出しのStep1〜6の章立てもAIが用意してくれました。また記事では割愛していますが、各Stepには「どう進めるか」のヒントも含まれており、行き詰まったときも手がかりをもとに質問しながら実装を進められました。  
断片的な文法や概念の知識が、実際にアプリケーションを作ることで体系的につながっていく感覚がありました。  
と言っても、まだ「Haskellが書ける」と自信を持って言える段階ではありませんが、AIに学習難易度を上げてもらったりすると応用的なインプットもできそうです。  
冒頭でも触れましたが、AIによってアウトプットのハードルが下がっただけでなく、インプットのハードルも劇的に下がっているのが個人的には嬉しいポイントです。  
今回は、少しだけ知っている技術を選定しましたが、初見の技術を学習するのも楽しそうです。  
AIをインプットに使うことに興味を持った方は、ぜひ自分の気になる技術で試してみてください。
