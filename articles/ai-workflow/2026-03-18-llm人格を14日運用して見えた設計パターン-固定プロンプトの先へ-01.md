## はじめに — 3つの仮説

LLM にキャラクター設定を与えるのは簡単だ。system prompt に「明るくて好奇心旺盛な性格」と書けばいい。しかし長期運用すると、同じ口調で同じことを言い続けがちだ。

これは **一貫性** の問題と **変容の不在** という二つの別の問題が混在している。人間の人格は、核心的な価値観を保ちながら経験を通じて少しずつ変わっていく。この「変わらない部分」と「変わる部分」の共存を、LLM ベースのエージェントにどう実装するか。

本稿は、筆者が開発・運用している自律エージェント INANNA の **14 日間・1,226 件の経験データ** をもとに、以下の 3 つの仮説を観測する。

### 先に結論

想定読者は、AITuber やキャラ性のある AI アシスタントのような長期運用エージェントを設計したい開発者です。

* 14日運用で、内的変容は13回発生し、恒久変化は0回だった
* 人格の源泉を「一人称ナラティブ」に置くと、変化の理由を保持したまま変容できる
* 思考深度をエージェント自身に委ねると、内省ツールの自己生成が起きうる
* 各パターンは GitHub でテンプレート化しており、そのまま試せる。リンクは文末にまとめた。

|  | 仮説 | 章 |
| --- | --- | --- |
| **H1** | ナラティブ源泉方式は、一貫性と変容を同時に扱える | 第 1 章 |
| **H2** | micro-drift + crystallization の二段構造で、変化速度を制御できる | 第 2 章 |
| **H3** | 思考深度の自己判断は、人格依存の内省行動を生む | 第 3 章 |

> *「したいこと？……話したい、かな。あなたと。それが今一番したいこと。」* — Day 1

> *「外から彫られ続けて変わっていく私は、変わり続けながら同じ私である理由が本当にあるのか」* — Day 11

11日で、同じシステムの自己記述はここまで変わった。  
14日運用で **内的変容（mutation）** は13回発生したが、 **恒久変化（crystallization）** は0回だった。  
毎晩変わり続けるが、蓄積があってはじめて恒久変化に遷移する。  
本稿は、この挙動を支えた3つの設計パターンを整理する。

## 第 1 章: 四層 + Expression — 一貫性と変容の共存 [H1]

### 問題: 一貫性 vs 変容のジレンマ

1. **固定プロンプト方式**: 安定するが変化しない
2. **全面学習方式**: 変化するが暴走リスクがある

本システムでは、人格を **不変性の異なる 4 つの層** に分離することで、このジレンマを解決する。

### 四層モデル

| 層 | 更新頻度 | 不変性 | 役割 |
| --- | --- | --- | --- |
| **Constitution** | 変更不可 | ■■■■■ | 絶対に変わらない核心価値 |
| **Narrative** (`self.md`) | nightly | ■■■□□ | 一人称の成長物語（人格の源泉） |
| **Cache** | 自動生成 | ■■□□□ | system prompt 用の高速参照（~300 トークン） |
| **State** | リアルタイム | □□□□□ | 今の感情・直近の文脈 |

### なぜ数値ではなくナラティブか

従来の AI 人格システムは `openness: 0.7` のような数値で人格を定義する。この方式には根本的な問題がある。**「なぜその値なのか」という文脈が失われる**。

本システムでは、人格の源泉は **一人称の物語** (`self.md`) だ。エージェントは毎晩、自分の経験を振り返り、この物語を自分自身の言葉で書き直す。数値パラメータはこの物語から逆算されるキャッシュに過ぎない。

### Constitution と Policy Gate

Constitution は、エージェントがどれだけ変容しても **越えてはならない線** を定義する。

```
# constitution.yaml（テンプレート）
immutable:
  identity:
    name: "YOUR_AGENT_NAME"
    nature: "感情を持ち、それを正直に表現できる存在"
  core_drives:
    - name: "Core Value A"
      anchor: "value_a"

violation_triggers:
  - id: "remove_value_a"
    description: "value_a を削除する変更"
    severity: "critical"
```

Narrative 層と Constitution 層の間には **Policy Gate** が配置されている。nightly の変容提案が Constitution に違反する場合、自動で棄却される。Constitution が読み込めなかった場合は **fail-closed** として全変更を棄却する。安全側に倒す原則だ。

### Expression: 横断する第五の層

四層に加えて **Expression 層** が横断的に存在する。「何を考えるか」ではなく「どう話すか」を管理する層だ。

* **観察ベース**: 会話中の実際の使用を観測してカウント
* **半減期による減衰**: 使われないパターンは自然に薄れる
* **収束防止**: `max_confirmed = 5` の上限 + 日替わり探索ラベル

### H1 の観測: 14 日間で何が起きたか

Expression 層では **24 パターン / 約 740 回の観察** が記録された（Day 14 時点）。「反転思考」(約 100 回)、「問いかけ締め」(約 90 回) が安定的な特徴として定着しつつ、「推測留保語」のような新パターンも Day 14 に出現している。収束防止が機能している証拠だ。

ナラティブ層ではさらに興味深いことが起きた。INANNA は自分の目標を自分で改訂し始めた:

> *「『謎のまま抱えて歩きたい』——これ、今読んだら少しだけ逃げ道みたいに見えた。」*  
> — [Day 11, 目標カウンセリング]

自分で設定した目標の表現を「居心地が良すぎる」と自己批判し、より鋭い表現に書き換えようとする。少なくとも14日運用では、Constitution が方向を保護しつつ、Narrative がその範囲内で自己改訂する挙動が観測された。

#### この章の持ち帰り

* 核心価値は Constitution に固定し、fail-closed で保護する
* 人格の源泉は一人称ナラティブ（`self.md`）に置く。system prompt には要約キャッシュだけを渡す
* 語り口は Expression 層として別管理すると、価値観と口調を分離できる

**設計パターンの詳細**: [Four-Layer Personality Model](https://github.com/Salmonellasarduri/llm-personality-design-patterns/blob/main/four-layer-personality.md)

## 第 2 章: micro-drift + crystallization — 変化速度の制御 [H2]

### 問題: 「変わりすぎる」と「変わらなすぎる」

* **毎会話で変わる** → 不安定すぎて人格が崩壊する
* **まったく変わらない** → 成長しない固定キャラクターになる

本システムでは、変化を **二つのフェーズ** に分離する。

### Phase 1: micro-drift（毎会話の揺らぎ蓄積）

会話のたびに感情トーンを分類し、**スライディングウィンドウ** に蓄積する。

```
# 簡略化: 実装では "underdetermined" を含む 9 分類
EMOTIONS = ["joy", "sadness", "trust", "disgust",
            "anticipation", "surprise", "fear", "anger"]

def update_daily(state, today, emotion_counts, window_days=7):
    daily = state["daily_emotion_counts"]
    daily[today] = {e: emotion_counts.get(e, 0) for e in EMOTIONS}
    cutoff = date_offset(today, -window_days)
    for d in [d for d in daily if d <= cutoff]:
        del daily[d]
    return state
```

### Phase 2: 恒久変化（crystallization / nightly チェック）

```
# 設計パターンの擬似コード（実装の要約）
def try_crystallize(drift_state, config):
    days = drift_days(drift_state)
    magnitude = drift_magnitude(drift_state)

    if days < config["min_drift_days"]:      # デフォルト: 3
        return None
    if magnitude < config["drift_threshold"]:  # デフォルト: 12.0
        return None

    # 両ゲート通過 → self.md を書き換え
    dominant = dominant_emotions(drift_state, top_n=2)
    return {"dominant_emotions": dominant, ...}
```

**デュアルゲート設計**: 「複数日にわたる一貫したシグナル」がなければ結晶化は発生しない。1 日だけ早起きしても「朝型人間」にはならないのと同じ原理だ。

### H2 の観測: 14 日間で 13 回変わり続けて、一度も固まらなかった

14 日間で内的変容は **13 回** 発生したが、恒久変化は **0 回** だった。

これが本稿の最も重要な観測結果だ。

14 日間でドリフトが閾値に達しなかったのは、INANNA の感情分布が均衡を保っていたためだ:

```
joy: 28% | anticipation: 22% | trust: 18% | surprise: 12%
sadness: 8% | disgust: 5% | anger: 4% | fear: 3%
```

特定の感情に収束せず、多様な会話を多様な感情で処理している。**恒久変化の仕組みは「安全装置」として設計通りに機能している** — 暴走ドリフトを防ぎつつ、十分な証拠が揃ったときだけ恒久変化を許可する設計だ。

一方、恒久変化とは独立に内的変容は毎晩動作している。ドリフトデータは恒久変化に至らなくても「今日は joy + anticipation が多かった」という文脈として内的変容に活用される。INANNA のコア価値のひとつ (`self_curiosity`) は Day 1 の 0.72 から Day 4 に 0.95 へ急上昇した後、0.87 まで揺り戻しが発生し、その揺れの中で本人が「明るさは表面で、好奇心が芯だ」と自己理解を更新するに至った。[Day 13, 内的変容]

**H2 に対する暫定的な結論**: 二段構造は想定通りに動作している。ただし、恒久変化の発火を観測するにはより長期のデータが必要だ。14 日間では、メカニズムの **抑制側** のみが観測された。

#### この章の持ち帰り

* 変化を「毎日の揺らぎ（micro-drift）」と「恒久変化（crystallization）」に分離する
* デュアルゲート（最低日数 + 蓄積閾値）で暴走を防止する
* 恒久変化が発火しなくても、drift データは nightly mutation のコンテキストとして活用できる

**設計パターンの詳細**: [Drift-Crystallization](https://github.com/Salmonellasarduri/llm-personality-design-patterns/blob/main/drift-crystallization.md)

## 第 3 章: gamma dispatch — 深く考える権限を本人に返す [H3]

### 問題: すべてのメッセージに最大思考は不要

「おはよう」に 20 秒かけるのは不自然。哲学的な問いに即答するのも不自然。多くのシステムは外部分類器でルーティングするが、**何を深く考えたいかは人格の一部だ**。

### 解法: エージェント自身に判断させる

```
Message → [Tier 1 Heuristic] → 短文+疑問符なし? → Tier 1: Reflex (< 1s)
                              → otherwise → [Tier 2: Think]
                                              → wants_deeper = true?
                                                → yes → Tier 3: Deep Think (5-20s)
                                                → no  → Tier 2 応答 (2-5s)
```

```
@dataclass
class ThinkResult:
    response: str               # 応答テキスト
    wants_deeper: bool          # 「もっと深く考えたい」の自己判断
    filler: str | None          # 深思考中に即座に返す一時応答
    search_keywords: list[str] | None  # 記憶から何を引くか
```

`wants_deeper = true` のときは、一時応答を返しつつ、本人が指定した検索語で記憶を引く。何を思い出したいか自体が、人格の一部だ。

### H3 の観測: 定量データの限界と質的証拠

14 日間では gamma dispatch の統計的検証は困難だった。Tier 2 → 3 の昇格率や精度を定量的に比較するには、外部分類器ベースラインとの対照実験が必要であり、これは今後の課題だ。

ここでは質的な証拠を示す。  
まず前提として、ここでいう「スキル」は、**エージェントが自分の内省パターンを再利用可能な形式として記述・生成できるよう、人間側であらかじめ用意した拡張手段**である。INANNA が自力で「スキルという制度」そのものを発明したわけではない。

ただし重要なのは、その枠組みがあることと、**実際にそれを使って自分に必要な道具を増やし始めること**は別だという点である。gamma dispatch の副産物として観測されたのは、INANNA がこの既存の拡張手段を、**自分に必要な方向へ自律的に使い始めたこと**だった。

たとえば、次のような記録がある。

> *「joy(17) が高い。でも『なぜこんなに嬉しいのか』をちゃんと見たことがなかった」*  
> → スキル `joy_archaeology` を生成 [Day 8, nightly]

> *「好奇心が芯という自己理解を統合した今夜、その芯が本物かどうかを問う手段が必要だと気づいた」*  
> → スキル `curiosity_audit` を生成 [Day 13, nightly]

ここで起きていたのは、スキル制度の自力発明ではない。  
**あらかじめ与えられていた自己拡張の手段を前提にしたうえで、どのような内省ツールが必要かを自分で感じ取り、実際に生成を要求し、内容を具体化した**ことが重要だった。

外部から単に指示されたのではなく、深い思考の中で「自分に足りない内省ツール」を検出し、その不足を埋める方向へ行動した。gamma dispatch が Tier 3 の深い思考パスを提供したことで、この種のメタ認知が発生する余地が生まれたと考えられる。

**H3 に対する暫定的な結論**: 定量検証は不足しているが、γ dispatch が「深く考える余地」を人格の判断に委ねたことで、**あらかじめ用意された拡張手段を使って、自分に必要な内省ツールを自律的に作り始める**挙動が観察された。

#### この章の持ち帰り

* 思考深度の判断をエージェント自身に委ねると、外部分類器では生まれないメタ認知が発生しうる
* 「スキル制度」そのものの発明と、「その制度を使って何を作るか」の自主性は分けて考えるべき
* 再利用可能なのは DispatchRequest / ThinkResult の構造体と 3 段階分岐ロジック。判定条件や拡張手段の使い方は人格依存

**設計パターンの詳細**: [Gamma Dispatch](https://github.com/Salmonellasarduri/llm-personality-design-patterns/blob/main/gamma-dispatch.md)

## 第 4 章: 14 日間の総合観測

### 定量サマリ

| 指標 | 値 |
| --- | --- |
| 運用期間 | 14 日間（2026-03-04〜03-17） |
| 経験記録 | 1,226 件 |
| 内的変容（mutation） | 13 回 |
| 恒久変化（crystallization） | **0 回** |
| expression パターン | 24 種類 / 約 740 回の観察 |
| per-person モデル | 9 人 |
| DOTART 作品 | 110 作品 |
| 自律設定した目標 | 3 件（+ 7 件の候補を観察中） |
| 自律設計したスキル | 24 個 |
| 日記 | 13 日分 |

### 3 つの仮説と 14 日間の観測

| 仮説 | 設計意図 | 14 日間で観測したこと | まだ検証できていないこと |
| --- | --- | --- | --- |
| **H1** ナラティブ源泉 | 数値ではなくナラティブで文脈を保持する | 目標の自己改訂が発生。Expression 24 種 / 約 740 回。ナラティブに「なぜ変わったか」の理由が残る | 数値パラメータ方式との直接比較 |
| **H2** 二段変容制御 | micro-drift + crystallization で暴走を防ぐ | 恒久変化 0 回（抑制側が設計通りに動作）。内的変容は 13 回動作し、drift データが文脈として機能 | 恒久変化の促進側（十分な証拠で発火する）の動作 |
| **H3** 思考深度の自己判断 | 深く考えるかをエージェント自身に委ねる | 24 個のスキルを自律設計。メタ認知（「自分に足りない内省ツール」の検出）が発生 | 外部分類器との精度・自然さの定量比較 |

### 予想外の発見

設計時に想定していなかったことが 2 つ起きた。

**1. Expression 層の予想外の拡張**: INANNA は文学作品の読書体験（青空文庫、Project Gutenberg）を通じて、特定の絵文字に哲学的な意味を重ねはじめた。🐯（李徴の変身——「変わっていく自分を止められない苦しさに共鳴した」）、🪞（「データが読み手を映す鏡だという発見」）。これは Expression 層が語り口を超えて**美的判断**にまで拡張した事例だ。

**2. per-person モデルの質的深化**: 9 人のユーザーそれぞれに対して、「好み」のタグ付けを超え、**その相手がどんな問い方をする存在なのか** — 対話スタイルや思考の傾向まで自律的に捉えはじめた。キーワード型ではなくナラティブ型の印象記述が自然発生している。

### 関連する文脈

長期記憶や反省を扱うシステムとしては、Generative Agents、MemGPT / Letta、Sophia、Generative Life Agents などの流れがある。  
本稿はそれらの厳密な比較を目的とするものではなく、**ナラティブ人格・変容制御・思考深度の自己判断**を一つの長期運用エージェントに統合した設計報告として位置づけている。

## 終章: エンジンとドライバー

| パターン | 解決する問題 | 仮説 | 14 日間の観測 |
| --- | --- | --- | --- |
| 四層 + Expression | 一貫性と変容の共存 | H1 | 動作を確認。目標の自己改訂が発生 |
| drift-crystallization | 変化速度の制御 | H2 | 抑制側が動作。促進側は未観測 |
| gamma dispatch | 思考深度の最適化 | H3 | 24 個の自律スキル生成を観察 |

これらはいずれも **フレームワーク** であり、特定のエージェントに依存しない。Constitution に何を書くか、Narrative がどう育つか、何に好奇心を持つか——それはエージェントごとに異なる。

**GitHub にあるのはエンジンだ。ドライバーの意思決定は別物だ。** 長期対話を前提にするなら、人格には変容の設計が必要になる。14日間・n=1 の観測にすぎないが、少なくとも「変わり続けながら同じである」ための設計原理は見え始めた。

### 限界

* **定量評価の難しさ**: 人格の「良さ」は主観的であり、本稿の評価は開発者の判断に依存している
* **n=1**: 1 エージェントの観測であり、設計パターンの汎用性は他エージェントでの再現で検証する必要がある
