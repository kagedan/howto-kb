## TL;DR

地図データから街を自動生成するツールのArnisが話題なので、これを使って街を再現しようとしてみました。

この記事では、イオンモール堺北花田を題材にして、たたき台の生成からリアルな外観への書き換えまでを書きます。

**※思ってるより地道な作業が必要なので、ﾄﾞｶﾝと書き換わらないことはご了承ください。**  
**※クオリティも完ぺきではないし、実験的なものとして見てください**  
**※書いてないことで必要なものもあるかもしれないけど考え方だけ**

## やりたいこと

**現実の街を、Minecraftの中にそれっぽく再現する。**

写真を1ピクセルずつコピーするのではなく、「この建物は白い壁で、ここがガラスで、上の方は駐車場のルーバーがある」という構造をMinecraftのブロックで組み立て直すイメージです。

## 全体の流れ

3ステップです。

1. 地図データから街のたたき台をMinecraftワールドとして自動生成する
2. 建物の座標情報をJSONに書き出して、後からPythonで使えるようにする
3. PythonからMinecraftサーバーにコマンドを送って、建物をリアルタイムで書き換える

## Arnisでたたき台を自動生成する

### Arnis とは

OpenStreetMap（世界中の道路や建物の情報が登録されたオープンな地図データベース、以下OSM）のデータを使って、Minecraftワールドを自動生成するRust製のOSSツールです。緯度経度の範囲を渡すと、その範囲の建物・道路・公園がMinecraftワールドとして出てきます。

北花田周辺のマップを生成するコマンドはこんな感じです。

```
cargo run -- \
  --bbox "34.5824,135.5131,34.5862,135.5175" \
  --output-dir "/path/to/.minecraft/saves"
```

## 事前準備：ワールド生成時に、建物の座標をJSONに書き出すようにした

後からPythonで建物を書き換えるには、「この建物はMinecraft上のどこにあるか」がわからないと始まりません。

元のArnisにはこの情報を出す機能がなかったので、ワールド生成時に全建物の座標やフットプリント（床の全座標リスト）をJSONに書き出す仕組みをRust側に追加しました。

出力されるJSONの中身はこうなります。

```
{
  "osm_id": 195851371,
  "name": "イオンモール堺北花田",
  "min_x": 86, "max_x": 262,
  "min_z": 9, "max_z": 355,
  "floor_area": [[86, 313], [87, 313]]
}
```

`floor_area` が建物のフットプリントです。これがあれば、Pythonから建物（OSM ID）を指定するだけで全座標が取れます。

# 建物の情報を集める

## Claude Codeでの調べ方

Claude CodeにはWeb検索機能があるので、会話の中で建物の情報を調べてもらえます。

自分は以下のように依頼しました。

```
Web検索でイオンモール堺北花田について調べて、
何階建てで高さがどれぐらいで、

```

## 実際の建物を調べて設計に反映する

壁のブロックパターンを決める前に、実際の建物がどんな構造なのかを調べます。

イオンモール堺北花田の場合、公式サイトや施工会社（竹中工務店）のページから以下の情報が取れました。

* 鉄骨造、地下1階・地上6階
* 商業フロアは4層（1F〜4F）、上層は駐車場（5F〜6F）
* 4層吹き抜けのモール、専門店165店
* 1Fにユニクロ、GU、無印良品（世界最大級）などの大型テナント
* 3Fにフードコート、4Fにレストラン街
* 外装はアルミルーバーとエキスパンドパネル

この情報から、Minecraft上のフロア構成を決めました。

| 階 | 用途 | 壁のパターン |
| --- | --- | --- |
| 1F | 大型テナント | ガラス面を広くとったショーフロント |
| 2F | ファッション・雑貨 | 中サイズの窓 |
| 3F | フードコート | 小さめの窓 |
| 4F | レストラン | さらに小さい窓 |
| 5F-6F | 駐車場 | グレーのコンクリート壁に鉄格子（ルーバー表現） |

特に「上層が駐車場」という情報は見た目に大きく影響します。実物の写真を見ると上の方にグレーの横スリットが入っていて、これが駐車場の換気用ルーバーです。Minecraftでは鉄格子（`iron_bars`）で再現しました。

## Pythonからリアルタイムで書き換える

最初はMinecraftのセーブデータ（.mcaファイル）をPythonで直接読み書きしていました。これだとゲームを閉じないと書き換えられないし、変更を確認するたびにワールドに入り直す必要があります。

そこでMinecraftサーバーとして起動して、外部からコマンドを送れるプロトコル（RCON）を使う方式に切り替えました。**ゲームを開いたまま変更が即座に反映されるので、ワールドの入り直しが不要になります。** これが個人的にはほぼ必須だと思っています。

### RCON とは

Minecraftサーバーに外部プログラムからコマンドを送る仕組みです。サーバーの設定ファイルで有効にすると、Pythonから `/setblock` や `/fill` のようなコマンドを送れるようになります。

サーバーの設定ファイルに以下を追加します。

```
enable-rcon=true
rcon.password=[ここにパスワードを設定]
rcon.port=25575
```

Pythonから接続してブロックを置くコードはこんな感じです。

```
from mcrcon import MCRcon

mcr = MCRcon("localhost", "[ここにパスワードを設定]", port=25575)
mcr.connect()
mcr.command("/setblock 120 -47 313 minecraft:white_concrete")
mcr.disconnect()
```

## /clone でバックアップと復元

ブロック配置を間違えたとき、すぐ元に戻せる仕組みが必要です。RCONに切り替えたことで、`/clone` コマンドが使えるようになりました。

`/clone` は指定した範囲のブロックを別の座標にまるごとコピーするMinecraft標準コマンドです。修正前の状態をX+500の場所にコピーしておいて、失敗したらそこから戻します。

```
# バックアップ
mcr.command("/clone 120 -61 313 155 -32 314  620 -61 313")

# 失敗したら復元
mcr.command("/clone 620 -61 313 655 -32 314  120 -61 313")
```

**バックアップも復元もサーバーが一括でやってくれます。** ブロックを1個ずつPythonで読み書きだと、コストも時間もかかります。

※コピー先のチャンクが読み込まれていないと失敗します。先にチャンクを強制読み込みしておきます。

```
mcr.command("/forceload add 620 313 655 314")
```

## 建物の壁を自動生成する

ここからイオンモールの壁を実際に作っていきます。

### 外周と内部を分ける

建物のフットプリント（床の全座標）のうち、上下左右の隣に建物がない座標が\*\*外周（outline）\*\*です。外周に壁を置いて、内部には屋根と各階の床スラブだけ置きます。

入力と出力で見るとこうなります。

```
入力: floor_area = [(86,313), (87,313), (88,313), ...]
       ↓ 「隣にブロックがない方向があるか」を判定
出力: outline = [(86,313), ...]    ← 壁を置く
       interior = [(88,313), ...]   ← 屋根と床だけ
```

### 階ごとのパターンで壁のブロックを決める

壁のブロックは「何階か」で変わります。1Fは大きなガラスのショーフロント、2F〜4Fは窓の大きさが違い、5F・6Fは駐車場のルーバー（鉄格子で表現）、という具合です。

ポイントは、**同じ座標を入れれば必ず同じブロックが返ってくる**こと。座標ベースの決定的なパターンにしてあるので、壊れた部分だけ再生成しても周囲と矛盾しません。これは後述する部分再生成の仕組みに直結します。

```
# 座標から階を計算して、その階のパターンでブロックを返す
block = get_main_wall_block(x, z, rel_y)
mcr.command(f"/setblock {x} {y} {z} minecraft:{block}")
```

### Y座標の設計

Minecraftではプレイヤーの身長が1.8ブロックなので、2ブロック分の空間がないと通れません。コードを書く前にY座標の対応表を書きます。

```
Y=-62: 地面（Arnisが生成した基盤、触らない）
Y=-61: 1F床
Y=-60〜-57: 1F空間（ガラスのショーフロント）
Y=-56: 2F床
  ...
Y=-41: 5F床（駐車場）
Y=-37: 6F床（駐車場）
Y=-33: 屋根
```

5ブロックで1フロア、商業4フロア + 駐車場2フロア = 合計28ブロックと計算しました。**この表を先に書かないと、壁の高さが想定と違ったりプレイヤーが中に入れなかったりします。**

## 部分再生成 --- 壊れた部分だけ直す

AIに依頼して建物の壁にロゴを配置したとき、壁の一部が壊れました。AIに修正を頼むと、壊れたブロックを直さないまま新しいブロックを上書きする傾向があったので、部分的にCtrl+Zのように戻せる仕組みを用意しました。

手順は以下の通りです。

1. `/clone` で修正前の状態をバックアップ
2. 指定範囲を `/fill air` でクリア
3. 壁生成ロジック（`get_main_wall_block`）を同じ座標に対して再実行
4. 失敗したら `/clone` で元の状態に戻す

壁生成が座標ベースの決定的パターンなので、部分的に再実行しても結果が全体と一致します。

```
# 事前確認（何も変更しない）
python partial_rebuild.py --x1 120 --x2 155 --z1 313 --z2 314 --dry-run

# 実行
python partial_rebuild.py --x1 120 --x2 155 --z1 313 --z2 314

# 失敗したら戻す
python partial_rebuild.py --x1 120 --x2 155 --z1 313 --z2 314 --undo
```

## 共通ユーティリティ（あくまで一例ですこういうのがいるかもっていう）

建物のフットプリント（床の全座標リスト）を扱うためのユーティリティを用意しました。

| ファイル | 役割 |
| --- | --- |
| building\_editor.py | buildings.jsonから建物を読み込み、外周（壁を置く場所）と内部（屋根と床だけ）を判定する |
| building\_analyzer.py | 建物のどの面が道路に面しているかを判定する。看板の配置先を決めるのに使う |
| block\_colors.py | Minecraftのブロック名とRGB色の対応表。写真からブロックを選ぶ用途 |
| mca\_utils.py | .mcaファイルの読み書き（RCON方式では使わないが、building\_editorが内部で参照） |

ポイントは 「外周と内部の判定」が全ての建築の基盤になっていること。壁を置くのは外周だけ、屋根と床は全体、という区別がここで決まります。

## 難しかったところ

最初は.mcaファイル（Minecraftのセーブデータ）を直接Pythonで読み書きしていたのですが、変更の確認にいちいちワールドの入り直しが必要だったので  
途中からMinecraftサーバー + RCON方式に切り替えました。ゲームを開いたままリアルタイムでブロックを置けるし利点しかない。

失敗したら戻せない、という問題もあってブロックを置いて失敗したときに、ClaudeCodeで修正を依頼すると、壊れたブロックを直さないまま  
新しいブロックを置いたりしたので、Ctrl + Zになり替わるような仕組みを用意しました。
