# はじめに

私はプライベートではClaude Max（Max 5x: $100/月）を、業務ではGoogle AI Pro（2,900円/月）を使用しています。

生成AIをものすごく使い込んでいるわけではないのですが、両方を日常的に使う中で、素人なりにそれぞれの良い点・悪い点が見えてきたので、まとめてみました。

本記事では「会話（チャット）」と「CLI（コードエージェント）」の2つの観点から比較しています。

# 会話編

## Claude Max

### 良い点

#### 応答速度が速い

簡単な会話なら、質問した1秒後くらいから回答が返ってきます。リサーチ機能を使用しても、よほど複雑な指示を出さなければ数十秒で返答が得られるので、テンポよく作業を進められます。

#### デフォルトで会話履歴のフォルダ分け機能（プロジェクト）が付いている

フォルダ分け機能のおかげで、「この会話ってどこでしたっけ？」ということになりにくいです。Claudeに指示する際も、「このプロジェクト内で会話した内容を要約して」というようにプロジェクト単位で指示を出すことができます。

Geminiでも拡張機能をインストールすればフォルダ分け機能を使用できるそうですが、業務利用となると外部ツールのインストールにはセキュリティ的なハードルもあります。デフォルトで提供されている点が良いと感じました。

### 悪い点

#### すぐに成果物を作りたがる

気軽に質問しただけなのに、「〇〇用のプロンプトを作りましょうか？」や「上記の内容をExcelで出力することも可能です」のように、すぐにアウトプットを作ろうとしてきます。

「まだ作らなくて結構です」と言えば解決する話ではあるのですが、毎回その返答をするのが若干のストレスです。

#### 期待とズレた推論をされることがある

質問した内容について、Claudeが自分の期待していない方向に推論を始めてしまい、「〇〇について教えてください」というような逆質問を受けることがあります。

必要な情報を逆質問してくれるのはとても助かるのですが、自分が期待していない方向に推論が始まったとき、軌道修正をするのが大変に感じることがあります。

## Google AI Pro

### 良い点

#### 文章の校正レベルが高い

自分は誤字脱字がひどいので、お客さんに長文を送るときはGeminiに文章の校正をしてもらうことが多いです。校正してくれた文章の内容がとても自然で、毎度感動しています。

Claudeでも文章の校正はやってくれますが、Geminiの方がより自然な日本語に感じました。

#### 異なるスレッドで話した会話内容を覚えてくれる

Claudeだと会話が長くなって別スレッドを開くと、再度前提条件などをインプットさせる必要があります。一方Geminiは、他スレッドで会話した内容も自動で覚えてくれるので、再度情報をインプットしなくて済みます。

### 悪い点

#### レスポンスが遅い

月額100ドルのClaude Maxと月額2,900円のGoogle AI Proでは価格差があるので仕方ないことなのですが、質問してから返答が返ってくるまでに時間がかかります。

簡単な質問でも10秒弱はかかりますし、若干複雑な指示だとレスポンスが返ってくるのに数十秒かかることもざらにあります。

#### 異なるスレッドの会話内容を覚えてしまう（メリットの裏返し）

会話内容を覚えてくれることはメリットでもありますが、過去の会話履歴のせいでGeminiが余計な類推をしてしまうこともよくあります。

一応、会話履歴を覚えさせない設定にすることもできますが、それだと本当に過去の会話履歴を覚えておいてほしいときにClaudeと同じように再度インプットが必要になります。

個人的には、Geminiにもデフォルトでフォルダ分け機能を追加して、フォルダごとに会話内容を覚えておいてくれると嬉しいなと思いました。

# CLI編

## Claude Code

### 良い点

#### 実装レベルが高い

実装スピードが本当に速い。そして精度も高い。いつもClaude Code先生には実装面でお世話になっています笑。

ファイル単位ではなく、フォルダ単位で複数ファイルの調査を依頼しても、高精度の調査結果を出してくれます。（精度が高いのは個人開発でソースの量が少ないからという可能性はありますが。）

#### 有志作成ツールの数が多い

昨今では「コードエージェントといえばClaude Code」という風潮もある気がします（個人的な感想ですが）。その勢いもあってか、多数のプラグインが存在しています。

<https://buildwithclaude.com/>

自分は生成AIツール大好き人間ではないので、何十種類もツール使ったわけではないのですが、いくつか試してみたところ、どれもとても優れており、面白い機能ばかりでした。

ただ、Gemini CLIにも拡張機能ディレクトリがあるので、今後Geminiも盛り上がってくるかもしれません。

<https://www.geminicliextensions.com/>

#### ドキュメントが充実している

Claude Code Docsのドキュメントはとても充実しています。他の生成AIと比べても圧倒的な情報量です。

<https://code.claude.com/docs/ja/overview>

Claude Codeに関しては、参考書を買わなくても、Claude Code Docsと似たようなユースケースの実装を紹介しているブログをいくつか見れば、大体のことは解決します。

### 悪い点

#### 特にない

正直、思いつきません。強いて言えば料金が高いくらいでしょうか（Claude Proだとすぐにリミットが来てしまうので、実用するにはMaxが必要です）。

## Gemini CLI

### 良い点

#### GCPとの親和性が高い（らしい）

作成元の会社が同じなので当たり前と言えば当たり前ですが、GeminiとGCPは親和性が高く、BigQuery・Cloud SQLなどのオープンソース拡張機能が公式に提供されています。そのため、SQLが書けない人でもデータ探索ができたり、より高度な分析予測ができるようになります。

また、Vertex AIのバックエンドとして、ADC（Application Default Credentials）、サービスアカウントJSONキー、Google Cloud APIキーの3つの認証方式をサポートしているそうです（これは初めて知りました）。

普段から業務でGCPを使用している方にとっては、Gemini CLIは相性の良い選択肢になると思います。Gemini CLIとGCPの連携についてもっと詳しく知りたい方は、公式ドキュメントを参照してみてください。

### 悪い点

#### 実装の精度が低い（個人的に）

特に複数ファイルが絡む実装の精度がとても悪いと感じました。的外れな回答をされることが高頻度でありました。

メソッドの説明やコードのリファクタなどは任せられますが、Gemini CLIにがっつりコードを書いてもらうのは心配が残ります。

また、コードに関する質問をGemini CLIに聞くと的外れな回答をされるのに、チャットで聞くとすぐに欲しい回答が得られたという経験も度々ありました。

# まとめ

会話編・CLI編を通して感じたのは、どちらのサービスも明確な強みを持っており、ユースケースに応じて、生成AIを使い分ける必要があると感じています。

| 観点 | Claude Max | Google AI Pro |
| --- | --- | --- |
| 応答速度 | 速い | やや遅い |
| 会話の整理 | プロジェクト機能あり | 拡張機能が必要 |
| 文章校正 | 良い | とても自然 |
| 会話の記憶 | スレッド内のみ | スレッド横断で記憶 |
| CLI実装精度 | 高い | やや低い |
| CLIエコシステム | 充実 | 成長中 |
| GCP連携 | - | 親和性が高い |
| 料金 | $100/月 ※1 | 2,900円/月 ※2 |

※1 Claude Maxには Max 5x（$100/月）と Max 20x（$200/月）の2プランあり  
※2 Gemini CLIは無料で利用可能。Google AI Proはチャット利用の料金

会話面では、スピード重視ならClaude Max、日本語の文章校正や会話の継続性を重視するならGoogle AI Proという印象です。CLI面では、コード実装の精度やエコシステムの充実度ではClaude Codeが一歩リードしていますが、GCPを使用している環境ではGemini CLIにも強みがあります。

自分のように、プライベートと業務で使い分けるのもひとつの手だと思います。どちらかに絞る必要はないので、それぞれの良いところを活かして使っていくのがおすすめです。

生成AI業界は進化のスピードがとても速いので、今回紹介したメリット・デメリットも数ヶ月後には変わっている可能性があります。今後も両方のAIツールの動向を追いつつ、新たな発見があればまた記事にしたいと思います。
