「今月の広告費、結局いくら使ったの？」  
「このキャンペーン、費用対効果はどうだった？」

マーケティング部門なら、こんな質問に即答できなくて冷や汗をかいた経験があるのではないでしょうか。

freeeで経費管理はしているものの、広告宣伝費の分析となると、毎回CSVをダウンロードしてExcelで集計して……という手作業に追われていませんか？

実は、**freee MCP（Model Context Protocol）** を使えば、Claude（AI）が直接freeeのデータにアクセスし、広告宣伝費の集計・分析・レポート作成を自然言語で指示するだけで自動化できます。

この記事では、マーケティング部門がfreee MCPを活用して**広告宣伝費の効果測定**を行う具体的な方法を、実践的なプロンプト例付きで解説します。

先日、Xで「Claude Code × freee MCPで家計簿・財務分析アプリを約1時間で自作した」という事例が話題になっていました。

この投稿を見て、マーケ担当の方から「銀行APIの複雑さをfreeeが吸収してくれて、Claude Codeでカスタム分析ができるなら、**広告費の可視化にも応用できそう**」という反応がありました。

まさにそうなんです。

freeeで取引先別（Google/Meta/Yahoo等）に広告費を集計し、Claude Codeで月次推移やROASを自動算出する——そんな流れがすぐ組めます。タグ機能を使えばキャンペーン単位の分析までいけるので、マーケ実務との親和性は非常に高いと思います。

既存のSaaS（マネーフォワードやfreee単体のダッシュボード）では表示項目が固定されていて、「広告費だけ月次で追いたい」「特定のキャンペーンだけ抜き出して見たい」といった柔軟な分析は難しい。でもClaude Code × freee MCPなら、「ここにこのグラフを追加して」と言えば即反映。**UIを完全に自分仕様にできる**のが最大の強みです。

freee側のデータ設計をちゃんとやれば、Claude Codeとの接続はかなりスムーズにいきます。この記事では、そのための具体的な設計方法と活用シナリオをまとめました。

## そもそもfreee MCPとは？

freee MCPは、AIアシスタント「Claude」からfreeeのAPIを直接操作できる仕組みです。

従来のAPI連携との違いは、**プログラミングが不要**であること。Claudeに日本語で「先月の広告宣伝費を集計して」と話しかけるだけで、裏側でfreee APIが呼び出され、結果が返ってきます。

銀行APIやカード会社との連携は個別にやると非常に手間がかかりますが、freeeのMCPサーバー経由ならカード明細・銀行残高をリアルタイム取得できます。つまり、**銀行APIの複雑さをfreeeが吸収し、マーケターはClaude Codeでカスタム分析に集中できる**という構図です。

マーケターが自分の言葉で経理データを分析できる。しかも既存SaaSのように表示が固定ではなく、自由に分析軸を変えられる。これがfreee MCPの真価です。

## マーケティング部門で使える5つの活用シナリオ

### シナリオ1：月次広告宣伝費レポートの自動生成

**課題**: 毎月末に広告宣伝費の実績を集計してレポートにまとめる作業に半日かかっている

**freee MCPでの解決方法**:

Claudeに以下のように指示するだけです：

> 「freeeから今月の広告宣伝費の取引をすべて取得して、取引先別・媒体別に集計してください。前月比も出してください。」

裏側では、freeeの取引API（`/api/1/deals`）から広告宣伝費の勘定科目に紐づく支出取引が抽出され、部門やタグ情報をもとに自動集計されます。

**ポイント**: freeeの取引登録時に、摘要欄に媒体名を入れたり、メモタグで「#Google広告」「#SNS広告」などを付けておくと、より精度の高い分析が可能になります。

---

### シナリオ2：キャンペーン別の費用対効果（ROI/ROAS）分析

**課題**: 複数のキャンペーンを同時進行していて、どのキャンペーンに予算を寄せるべきか判断できない

**freee MCPでの解決方法**:

freeeの**セグメントタグ**機能を活用します。セグメントタグは最大3次元まで設定でき、例えば以下のように設計します：

| セグメント | 用途 | 設定例 |  
|-----------|------|--------|  
| セグメント1 | キャンペーン名 | Spring2026、Summer2026 |  
| セグメント2 | 広告媒体 | Google広告、Meta広告、YouTube |  
| セグメント3 | ターゲット地域 | 関東、関西、全国 |

各取引にこれらのタグを付与しておけば、Claudeにこう聞くだけです：

> 「今四半期のキャンペーン別の広告費を集計して、各キャンペーンの支出推移とROASをまとめてください。」

試算表API（`/api/1/reports/trial\_bs`）の内訳表示機能（`breakdown\_display\_type=segment\_1\_tag`）を使えば、キャンペーン別の費用集計が一発で取得できます。

売上データと突き合わせれば、キャンペーンごとのROAS（広告費用対効果）算出も可能です。

タグ機能でキャンペーン単位の分析までいけるのは、マーケ実務では本当に熱い機能です。既存SaaSでは「広告費全体」しか見えなかったものが、キャンペーン・媒体・地域という3軸で自由に切れるようになります。

---

### シナリオ3：広告媒体別コスト分析と最適配分

**課題**: Google広告、Meta広告、YouTube広告など複数媒体に出稿しているが、どの媒体のコストパフォーマンスが良いか把握できていない

**freee MCPでの解決方法**:

freeeで取引先をGoogle/Meta/Yahoo等の媒体別に登録しておけば、取引先別に広告費を集計できます。さらにメモタグで広告媒体を分類し、月次で媒体別のコスト推移を追跡します。

> 「過去6ヶ月間の広告宣伝費を媒体別（Google広告・Meta広告・YouTube広告）に月次推移で出してください。」

さらに一歩進めて：

> 「媒体別の月次コストと、各媒体経由の売上データを突き合わせて、CPA（顧客獲得単価）の推移を算出してください。」

Claude Codeなら「ここにグラフを追加して」と言えば即反映されるので、媒体別の推移グラフやROAS比較チャートも自在に作れます。マネーフォワードやfreee単体のダッシュボードでは表示項目が固定されていてこうした柔軟な可視化は難しいですが、**Claude Code × freee MCPなら完全に自分仕様の分析画面が作れる**のが大きな差です。

**実務のコツ**: freeeの取引登録時に以下のルールを徹底するだけで、分析精度が格段に上がります。

---

### シナリオ4：予算消化率のリアルタイム監視

**課題**: 月末になってから「予算オーバーしていた」と気づくことが多い

**freee MCPでの解決方法**:

月初に予算を設定し、定期的にClaudeに確認するだけです：

> 「今月の広告宣伝費の予算は500万円です。現時点での消化率を教えてください。残りの営業日数から日割りペースも計算してください。」

freeeから当月の広告宣伝費の取引を即座に集計し、予算との差分を算出します。freee MCPサーバー経由でカード明細・銀行残高がリアルタイムで取得できるので、「今この瞬間の消化状況」が分かるのがポイントです。

さらに、Claude Codeのスケジュール機能を使えば、毎週月曜日に自動で予算消化率レポートを生成することも可能です。

---

### シナリオ5：決算期の広告費分析レポート作成

**課題**: 決算期に経理部門から「広告宣伝費の内訳を出してほしい」と言われるたびに大仕事になる

**freee MCPでの解決方法**:

> 「今期（2025年4月〜2026年3月）の広告宣伝費について、四半期ごと・取引先別・媒体別の内訳をまとめてください。前期比較も付けてください。」

試算表APIと総勘定元帳API（`/api/1/reports/general\_ledgers`）を組み合わせることで、経理部門が求めるフォーマットでの集計が可能です。

## 成功のカギは「freee側のデータ設計」

ここまで5つのシナリオを紹介してきましたが、すべてに共通する大前提があります。

**freee側のデータ設計をちゃんとやること。これが9割です。**

どんなに優秀なAIでも、データが正しく構造化されて入っていなければ分析はできません。逆に言えば、freee側の設計さえしっかりしていれば、Claude Codeとの接続はかなりスムーズにいきます。

---

## 導入のための3ステップ

### Step 1：freeeの勘定科目とタグを整備する

まず以下を整備しましょう：

* **勘定科目**: 「広告宣伝費」を基本に、必要に応じて「販売促進費」「PR費」などと使い分けるルールを決める
* **取引先**: Google/Meta/Yahoo等、広告媒体ごとの取引先を登録しておく
* **メモタグ**: 広告媒体名（#Google広告、#Meta広告など）を統一ルールで設定
* **セグメントタグ**: キャンペーン名・媒体・地域など多次元分析用に設計（法人アドバンスプラン以上）
* **部門**: マーケティング部門配下に「Web広告」「オフライン広告」「イベント」など細分化

### Step 2：freee MCPをセットアップする

Claude DesktopまたはClaude Codeでfreee MCPサーバーを設定します。初回の認証を済ませれば、以降はClaudeから直接freeeのデータにアクセスできるようになります。

### Step 3：分析プロンプトのテンプレートを作る

チーム内で頻繁に使うレポートのプロンプトをテンプレート化しておくと、誰でも同じ品質のレポートが出せるようになります。

**テンプレート例：月次広告レポート**

```
以下の条件でfreeeから広告宣伝費のデータを取得し、月次レポートを作成してください。

- 対象期間：{YYYY年MM月}
- 勘定科目：広告宣伝費
- 集計軸：
  - 媒体別（メモタグ）
  - キャンペーン別（セグメント1）
  - 取引先別（Google/Meta/Yahoo等）
- 比較：前月比・前年同月比
- 出力形式：表形式でサマリー＋主要KPIのハイライト＋ROAS推移グラフ
```

## 注意点とベストプラクティス

### 知っておくべき制限事項

* **セグメントタグ**は法人アドバンスプラン以上で利用可能です。スタンダードプランの場合はメモタグと部門で代替します
* 部門・セグメントの登録数が5,000件を超えると、試算表の内訳表示でエラーが出る場合があります
* 総勘定元帳APIは現在β版のため、大量データの取得時にタイムアウトが発生する可能性があります

### 運用を成功させるコツ

1. **入力ルールの徹底が9割**: 経費精算時のタグ付けルールをチーム全員で共有しましょう。ここが崩れるとすべての分析が成り立ちません
2. **小さく始める**: まずは「月次の広告費総額を自動取得する」だけから始めて、徐々に分析軸を増やしていくのがおすすめです
3. **定型レポートから自動化**: 毎月同じフォーマットで作っているレポートこそ、freee MCP自動化の最適な候補です
4. **UIは後から自由に変えられる**: Claude Codeなら「グラフの色を変えて」「比較軸を追加して」が一言で反映されます。最初から完璧を目指す必要はありません

## まとめ：経理データをマーケティングの武器にする

freee MCPの本質は、「経理データへのアクセス障壁をゼロにする」ことです。

これまでは、広告宣伝費のデータを分析しようとすると：

1. freeeにログインしてCSVをエクスポート
2. Excelで集計用のピボットテーブルを作成
3. 前月・前年のデータも同様に取得して比較

という作業が必要でした。

freee MCPを使えば、これが「Claudeに質問する」という1ステップに変わります。しかも既存SaaSのように表示が固定ではなく、自分の見たい切り口で自由にカスタム分析ができる。

マーケティング部門にとって、広告費の効果測定は「やらなきゃいけないけど面倒な作業」から、「AIに任せて瞬時に洞察を得られる戦略的活動」に進化します。

まずは今月の広告宣伝費の集計から、freee MCPを試してみてください。
