Microsoft のAIエージェント製品が Copilot Agent Builder / Copilot Studio / Microsoft Foundry と3つに分かれている。

その理由は利用者の技術レベル・求める制御度・業務の複雑さが大きく異なるためであり、1つの製品で個人の作業代行から企業の基幹業務判断まで全てを満たすことは難しく、用途ごとに最適化されたレイヤーを提供している。

Copilot Agent Builder は「個人の作業を任せる」ための最軽量レイヤー、Copilot Studio は「組織共通のエージェントをローコードで作る」ための中間レイヤー、Foundry は「モデル選択・RAG・評価・ガバナンスまで制御する」ための最上位レイヤーである。

この3つが揃うことで、企業は小さな自動化から高度なAI組み込みまで一貫して構築できることを説明したい。  
![](https://static.zenn.studio/user-upload/2ad7ec2707a8-20260701.png)

## 各製品の特徴と役割

**● Copilot Agent Builder**  
Microsoft 365 Copilot の拡張として、個人の作業代行を最も簡単に自動化できる。メール整理、議事録生成、簡易問い合わせなど、担当者自身の作業を効率化する用途に向く。  
M365の文脈で動くため導入コストが低く学習もほぼ不要です。

**● Copilot Studio**  
組織が共通エージェントをローコードで内製するための基盤。Power Platform と同じ感覚で扱え、市民開発者でも構築可能。PDFにもある通り「インフラはMicrosoftにお任せ」で、ガバナンスやセキュリティは既定で安全。FAQ、申請受付、部署横断の問い合わせ窓口など、社内事務処理の多くはStudioで十分でしょう。

**● Microsoft Foundry**  
開発者向けのフルカスタム環境。  
・11,000以上のモデル選択  
・ファインチューニング  
・マルチエージェント  
・Foundry IQ（高度RAG）  
・評価・ガードレールの集中管理  
・Agent 365 によるガバナンス  
など、企業レベルの高度要件に対応する唯一のレイヤーである。  
審査・与信・機微データ・製品内AI機能など、厳密な判断や大量データ処理が必要な領域で力を発揮します。

## 市民開発ならどれが向いているか

結論は明確で Copilot Studioが最適です。  
理由はPDFの「内製・市民開発向け」という記述に加え、以下の点がある。  
・ローコードで扱える  
・コネクタ中心で学習コストが低い  
・Power Automate と連携しやすい  
・インフラ管理不要  
・ガバナンスが既定で安全  
Agent Builder は「個人の作業代行」なので、組織共通の仕組みを作るには機能が不足します。

## 社内事務処理ならどれが良いか

社内事務処理は「複雑さ」で選ぶのが最も合理的である。

**● 単純な事務処理（FAQ、申請受付、定型フロー）**  
→ Copilot Studio  
トピック型ロジックで十分。  
Power Automate と組み合わせれば多くの事務処理を自動化できる。

**● 中程度の処理（部署横断、データ量が多い、構造化出力が必要）**  
→ Copilot Studio + Foundry の共通エージェント  
StudioからFoundryの頭脳を呼び出す構成が最適。

**● 高度な処理（審査・スコアリング・機微データ・厳密な判断）**  
→ Microsoft Foundry  
決定的オーケストレーション、閉域処理、評価・ガードレールなど、Studioでは対応できない領域。

## 導入ハードル（コスト・運用）

AIエージェントの導入ハードルは、コストと運用負荷の観点で3製品それぞれに大きな違いがある。

まず Copilot Agent Builder は、Microsoft 365 Copilot の延長として利用できるため、初期コストはほぼゼロに近い。  
追加のインフラ構築も不要で、運用は既存のM365環境に完全に乗る、また個人の作業代行が中心であるため、ガバナンスや監査の設定も既定のCopilotに準拠し、導入障壁は最も低い。

一方で Copilot Studio は、組織共通エージェントを構築するための専用ライセンスが必要になるため、初期コストは中程度となるが、インフラはMicrosoftが管理するマネージド環境であり、運用負荷は非常に低い。  
コネクタ中心の設計で保守が容易なうえ、市民開発者が主体となって改善を続けられるため、長期的な運用コストは抑えやすく、社内事務処理の自動化を広く展開する場合、最も費用対効果が高いレイヤーと言える。

Microsoft Foundry は、3製品の中で最も導入ハードルが高い。  
モデル利用料やAzureリソースの消費に加え、開発者によるフルカスタム構築が前提となるため、初期コストは大きくなる、また評価・ガードレール・ログ管理・CI/CD・バージョン管理など、運用面でも専門的な体制が必要になる。  
ただし、FoundryはPDFでも強調されているように「再実装地獄を避ける共通化」を実現でき、複雑な業務判断や機微データ処理を一度Foundryで共通エンジン化すれば、Copilot Studioや既存アプリから再利用できるため、長期的には大幅なコスト削減につながる。  
短期的には高コストだが、企業の基幹業務をAI化する場合には最も合理的な投資となるだろう。

## まとめ：Copilot Agent Builder を触った人への次のステップ

Agent Builder を触った経験がある人は、「個人 → 組織 → 高度業務」へと階段を上がるイメージで理解するとよい。

・まず Copilot Studio で組織向けエージェントを作る  
・必要に応じて Foundry の共通エージェントを呼び出す構成へ拡張  
・高度化したい場合は Foundry でフルカスタム開発

3製品は競合ではなく、役割分担されたレイヤー構造である。  
この構造を理解することで、企業は小さな自動化から高度なAI組み込みまで一貫して構築できるでしょう。
