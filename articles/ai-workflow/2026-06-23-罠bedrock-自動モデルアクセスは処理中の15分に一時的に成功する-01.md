ディップ株式会社スポットインテリジェンス開発部の町永です。

## この記事で学べること

* Amazon Bedrock で新しいサードパーティモデルを使うときは、AWS Marketplace の権限を持つ担当者が事前にモデルを呼び出しておく必要がある(Bedrock 自動モデルアクセス)
* 権限のないIAMで事前アクセスを行っても、呼び出しに成功する事がある(!?)

## 起きたこと

* 本番アプリケーションで LLM モデルを Claude Sonnet 4.6 にアップデート
* 初回呼び出しは成功。二回目以降の呼び出しの大半が失敗(`AccessDeniedException`)
* CloudWatch の `AWS/Bedrock` で `InvocationClientErrors` を観測
  + `InvocationServerErrors` と `InvocationThrottles` は 0
  + クォータ制限も余裕あり

## 調査

呼び出し成功の実績もあるので、最初はLLM側の一時的な不具合かと思ったのですが、SCPが悪さをしているという疑惑が登場しました。

アプリケーションはモデルを `us.anthropic.claude-sonnet-4-6` のように `us.` で始まる **クロスリージョン推論プロファイル (CRIS)** で指定しています。  
このプロファイルを `us-west-2` から呼んだ場合、Bedrock は実行を `us-east-1`、`us-east-2`、`us-west-2` のいずれかに容量に応じて振り分けます。  
このうち特定のリージョンでの実行がSCP で制限されているのでは？という推測です。これであれば 100% の失敗ではなく、部分的に成功する状況とも辻褄が合います。

上記の見解をもとに、弊社のプラットフォームチームに伝え、確認を依頼しました。

しばらくして、プラットフォームチームから次の回答を受領しました。

* SCPの設定は問題なく、制限はされていない。
* こちらでもsonnet-4.6を呼び出してみたところ同様にエラーになったが、しばらくして解消した。一時的な障害だったのでは？

しかし、この状況をAIに伝えたところ、AIの見解は違いました。

```
🎯 原因確定 — モデルアクセス（AWS Marketplace サブスクリプション）未完了でした。 SCP でも quota でもコード不具合でもありません。
```

**！？**

AWS Bedrock のドキュメント「自動モデルアクセスについて」に該当の記述がありました。

[モデルへのアクセスをリクエストする - Amazon Bedrock](https://docs.aws.amazon.com/ja_jp/bedrock/latest/userguide/model-access.html)

つまり、Amazon Bedrock で新しいサードパーティモデルを使うときは、AWS Marketplace の権限を持つIAMで事前にモデルを呼び出しておく必要があるという案内なのですが、注意すべきは次の一文です。

> アカウントで初めてサードパーティーモデルを呼び出すと、Amazon Bedrock はサブスクリプションプロセスをバックグラウンドで自動的に開始します。この設定期間中 (最大 15 分）、サブスクリプションの確定中に API コールが一時的に成功することがあります。

ここでの呼び出し時に、Marketplaceの権限は関係なく、とにかく呼び出しを行われるとサブスクリプション処理を開始し、その期間はAPIコールが成功する事がある。ということらしいです。

## おわりに

事象の解釈について、サポートにも確認をおこない次のように整理ができました。

* リリース後、本番アプリのロールからsonnet4.6が初めて呼ばれ、サブスクリプションプロセスがバックグラウンドで開始された
* 本番アプリのロールは Marketplace アクセス許可を持っていなかったため、サブスクリプションは前提条件不足で最終的に失敗確定
* ただし、処理中の最大 15 分間は API コールが一時的に通る仕様のため、初回呼び出しは成功し、その後も間欠的に成功した
* プラットフォームチームがsonnet4.6を呼び出したタイミングで、その担当者は Marketplace 権限を持っていたため、アカウント全体のサブスクリプションが完了し、以降の `AccessDeniedException` が解消した

事象をわかりづらくしていたのは、権限を持たないユーザーが初回アクセスしても、最大 15 分のサブスクリプション処理中は一時的に成功してしまう、というこの仕様でした。

権限を持たないのであれば一度も成功しないでくれ...
