## はじめに

NotebookLMのスライド生成機能は、プレゼンテーション資料を手軽に自動生成できる便利な機能です。しかし、出力形式にはひとつ大きな制約があります。

NotebookLMのスライド生成機能はPDFやPowerPoint (.pptx) 形式でダウンロードできるものの、スライドの中身は1枚丸ごと「**画像**」として埋め込まれた状態で生成されます。

そのため、PowerPointでテキストボックスや図形を個別に選択して編集するといった操作が一切できません。NotebookLMが出力したスライドに対して軽微な修正を加えたい場合や、自社のデザインテンプレートに合わせて整えたい場合には、この制約が大きな障壁となります。

この問題へのアプローチとして、以下の記事ではNotebookLMのスライドの内容をYAML形式で出力し、それをGeminiに渡してスライドを**再作成する**という方法が紹介されています。

<https://zenn.dev/hirokita117/articles/24c42f407dc9b9>

ただし、このアプローチはあくまでGeminiでゼロからスライドを生成し直すものであり、NotebookLMが作成したもとのデザインや構成を忠実に引き継ぐわけではありません。

そこで本記事では、もとのスライドのデザインをできる限り保持しながら、編集可能な形式に変換できないかという観点で、いくつかの方法を実際に試した結果を共有します。

## 結論

先に結論をお伝えすると、以下が現実的です。

* **NotebookLMが編集可能な形式でスライドを出力できるようになるのを待つ**
* **NotebookLMで出力されたスライドを参考資料として使いつつ、自分で一から作り直す**

さまざまな方法を試してみましたが、現時点では「NotebookLMが生成したスライドをそのまま加工して編集可能にする」ことは難しく、もとのスライドを参照しながら自分の手で作り直した方が、品質・工数の両面で効率的という結論に至りました。

とはいえ、各アプローチでどこまで通用するのかを知っておくことは有用です。よって、以下に検証内容を詳しく紹介します。

## 検証

### 1. Adobe AcrobatのWebツールでPDF→PPT変換

#### 方法

Adobe AcrobatのオンラインツールでPDFをPowerPoint形式に変換します。無料で利用できるため、手軽に試せる選択肢のひとつです。

<https://www.adobe.com/jp/acrobat/online/pdf-to-ppt.html>

#### 結果

**日本語が文字化けしてしまい、実用になりませんでした。**

そもそもNotebookLMが出力したPDFにはテキストデータとしての文字情報が含まれておらず、スライド全体が1枚の画像として埋め込まれているだけです。

そのため、Adobe Acrobatのような変換ツールがテキストや図形を認識して構造を読み取ることができず、文字化けや変換エラーが生じてしまいます。変換ツール側の問題というよりも、もとデータの構造上の制約が原因です。

---

### 2. PDFをCopilot（有料版）に投入して変換

#### 方法

MicrosoftのOfficeアプリケーション群と親和性が高い有料版のCopilotを活用します。  
PDFをそのままCopilotに渡し「編集可能なPowerPointスライドを作成してください」と指示するシンプルな方法です。

#### 結果

**スライドは生成されましたが、もとの内容から大きく改変されていました。**

出力されたスライドは、Copilotが自動生成するモダンなデザインで見栄えはよいのですが、もとのPDFの内容とはかなり乖離しています。

Copilotが独自に内容を解釈してしまうため、もとデータの忠実な再現という観点では不十分です。「もとのスライドを変換する」というよりも「もとのスライドを参考にした新しいスライドを生成する」に近い動作でした。

ちなみに「編集可能な」を指定しないと、画像をそのまま貼り付けただけのスライドを出力することがあります（NotebookLMがPowerPoint形式で出力するのと同じ）。

---

### 3. プロンプトを詳細に指定して再挑戦（2の改善版）

#### 方法

2の結果が不十分だったため、プロンプトをより詳細に記述し「もとのPDFの内容と完全に一致するように、テキストや構成を変えずにスライドを再現してください」という趣旨の厳密な指示を加えました。

```
添付したPDFファイルをもとに、PowerPointファイル（PPTX）を作成してください。
以下の要件を厳守してください。

【基本方針】
- PDFの内容を一字一句変えず、完全に忠実に再現すること
- テキストの追加・削除・言い換え・要約は一切行わないこと
- PDFの各ページを、対応するスライド1枚に再現すること

【テキストについて】
- 文字の内容・順序・改行位置をPDFと完全に一致させること
- フォントの種類・サイズ・太字・斜体・下線などの書式を可能な限り再現すること
- 文字色・強調表現もPDFに合わせること

【レイアウト・構成について】
- 見出し・本文・箇条書き・表・図などの配置をPDFに忠実に再現すること
- 余白・インデント・行間もできる限りもとのPDFに合わせること
- 段組みがある場合はその構造を維持すること

【画像・図表について】
- 図・画像・グラフ・表はPDFから抽出し、もとの位置・サイズに配置すること
- 表はテキストボックスではなく、PowerPointの表機能を使って再現すること

【スライド設定】
- スライドサイズはPDFのページサイズに合わせること（16:9）
- 背景色・テーマもPDFの見た目に合わせること

出力はPPTXファイルとしてください。
```

#### 結果

**内容（テキスト）の再現度は向上したものの、デザイン面での再現は困難でした。**

プロンプトを厳密に指示することで、テキストの内容はある程度もとのスライドに近づけることができました。ただし、フォント・色・レイアウト・図形の配置といったビジュアル面は依然として大きく崩れており、「もとのデザインを保ったまま編集可能にする」という当初の目標には届きませんでした。

Copilotの努力は感じられますが、画像データからデザイン情報を復元することには根本的な限界があるようです。

### 4. YAMLを使用してCopilotでスライドを再作成

#### 方法

この記事の冒頭で紹介したGeminiでスライドを再作成する方法と同じく、Copilotで**構造的な情報をできる限り引き継いだうえでスライドを再生成する**アプローチを試みます。

NotebookLMのチャット欄からスライドの内容をYAML形式で出力し、そのYAMLデータをCopilotに渡してPowerPointスライドを生成させます。YAMLにはテキストだけでなく図や要素間の関係性も含まれるため、もとのスライドの構成に近い形で再現できることを期待しました。

具体的な手順は以下の通りです。

1. NotebookLMのチャット欄で「スライドの内容をYAML形式で出力してください」と指示し、構造化されたデータを取得する
2. 取得したYAMLデータをCopilotに貼り付け、「このYAMLの内容をもとにPowerPointスライドを作成してください」と指示する

#### 結果

**スライド全体に統一感はあるものの、デザインは無難でシンプルな仕上がりになりました。**

YAML経由でコンテンツの構造は引き継げているため、内容の乖離は小さくなっています。しかし、ビジュアルの豊かさや、もとスライドの雰囲気を再現するには至りませんでした。  
また、同様のYAMLを使ってGeminiでスライドを作成した場合と比較するとデザインのクオリティはGeminiの方が高い印象があります。Copilotをより深く考える（`Think Deeper`）ようにしても、結果に大きな改善は見られませんでした。

### 5. python-pptxからスライドを再作成

**この記事の検証の本題です。**

PDFを画像に変換し、LLM（Claude）を用いて画像ファイルからpython-pptxのコードを出力します。もとのスライドにできる限り近づけるという観点ではもっとも近い結果が得られています。ただし、画像やデザインは抽出できないため、諦めざるをえません。

#### 方法

1. PDFファイルを1ページずつ画像ファイルに変換する

pdf\_to\_image.py

```
import argparse
import os
from pathlib import Path
from pdf2image import convert_from_path

def pdf_to_images(pdf_path: str, output_dir: str) -> list[str]:
    """
    PDFを画像に変換して保存する

    Args:
        pdf_path: PDFファイルパス
        output_dir: 出力ディレクトリパス

    Returns:
        保存した画像パスのリスト
    """
    # 出力ディレクトリの作成
    output_path = Path(output_dir)
    output_path.mkdir(parents=True, exist_ok=True)

    # PDF→画像変換（300DPI）
    images = convert_from_path(pdf_path, dpi=300)

    # 画像の保存
    image_paths = []
    for i, image in enumerate(images, start=1):
        image_file = output_path / f"page_{i:03d}.png"
        image.save(image_file, "PNG")
        image_paths.append(str(image_file))
        print(f"保存完了: {image_file}")

    print(f"\n合計 {len(image_paths)} ページを変換しました")
    return image_paths

def main():
    parser = argparse.ArgumentParser(description="PDFを画像に変換する")
    parser.add_argument("pdf_path", help="PDFファイルパス")
    parser.add_argument("output_dir", help="出力ディレクトリパス")
    args = parser.parse_args()

    if not os.path.exists(args.pdf_path):
        raise FileNotFoundError(f"PDFファイルが見つかりません: {args.pdf_path}")

    pdf_to_images(args.pdf_path, args.output_dir)

if __name__ == "__main__":
    main()
```

2. Claudeを使用して画像ファイルからpython-pptxのコードを出力する

   * 最初のスキャンでスライド全体の背景色やフォントなど、デザインガイドラインを抽出して統一する
   * スライドを1枚ずつスキャンし、詳細に分析を行う

script\_generator.py

```
import argparse
import base64
import boto3
import json
import os
from botocore.config import Config
from dotenv import load_dotenv
from pathlib import Path

# .envファイルは相対パスで、../env/.env にあります
load_dotenv(Path(__file__).parent.parent / "env" / ".env")

def get_bedrock_client():
    """Bedrockランタイムクライアントを生成する"""
    config = Config(
        read_timeout=300,
        connect_timeout=60,
        retries={'max_attempts': 3}
    )

    return boto3.client(
        "bedrock-runtime",
        aws_access_key_id=os.getenv("AWS_ACCESS_KEY_ID"),
        aws_secret_access_key=os.getenv("AWS_SECRET_ACCESS_KEY"),
        region_name=os.getenv("AWS_BEDROCK_REGION"),
        config=config
    )

def encode_image(image_path: str) -> str:
    """画像をBase64エンコードする"""
    with open(image_path, "rb") as f:
        return base64.b64encode(f.read()).decode("utf-8")

def extract_design_guidelines(client, image_paths: list[str]) -> dict:
    """
    全画像を分析してデザインガイドラインを抽出する

    Args:
        client: Bedrockクライアント
        image_paths: 画像ファイルパスのリスト

    Returns:
        デザインガイドライン辞書
    """
    prompt = """これらの画像はPDFの全ページをスキャンしたものです。
全体を見て、PowerPointのデザインガイドラインをJSON形式で抽出してください。

抽出する情報:
1. 背景色（RGB値）
2. フォントサイズの基準（タイトル、見出し、本文、キャプションの4段階）
3. 繰り返し出現する要素（ヘッダー、フッター、ページ番号など）の位置と内容
4. 全体的な配色パターン
5. テキストの配置傾向（中央揃え、左揃えなど）

出力形式:
{
    "background_color": {"r": 255, "g": 255, "b": 255},
    "font_sizes": {
        "title": 44,
        "heading": 32,
        "body": 18,
        "caption": 14
    },
    "repeated_elements": [
        {
            "type": "footer",
            "position": {"x": 0.1, "y": 0.95},
            "text": "会社名"
        }
    ],
    "text_alignment_default": "left"
}

注意事項:
- 出力はJSON形式のみで、他の説明は不要です
- 繰り返し要素は除外対象として識別してください"""

    content = []
    for image_path in image_paths:
        image_base64 = encode_image(image_path)
        content.append({
            "type": "image",
            "source": {
                "type": "base64",
                "media_type": "image/png",
                "data": image_base64
            }
        })

    content.append({
        "type": "text",
        "text": prompt
    })

    request_body = {
        "anthropic_version": "bedrock-2023-05-31",
        "max_tokens": 4000,
        "messages": [
            {
                "role": "user",
                "content": content
            }
        ]
    }

    print(f"全{len(image_paths)}枚の画像からデザインガイドラインを抽出中...")
    response = client.invoke_model(
        modelId="jp.anthropic.claude-sonnet-4-6",
        body=json.dumps(request_body)
    )

    response_body = json.loads(response["body"].read())
    content_text = response_body["content"][0]["text"]

    if "```json" in content_text:
        json_start = content_text.find("```json") + 7
        json_end = content_text.find("```", json_start)
        content_text = content_text[json_start:json_end].strip()
    elif "```" in content_text:
        json_start = content_text.find("```") + 3
        json_end = content_text.find("```", json_start)
        content_text = content_text[json_start:json_end].strip()

    try:
        guidelines = json.loads(content_text)
        print(f"  背景色: RGB({guidelines['background_color']['r']}, {guidelines['background_color']['g']}, {guidelines['background_color']['b']})")
        print(f"  フォントサイズ基準: {guidelines['font_sizes']}")
        print(f"  繰り返し要素数: {len(guidelines.get('repeated_elements', []))}")
        return guidelines
    except json.JSONDecodeError as e:
        print(f"  警告: JSON解析エラー - {e}")
        print(f"  レスポンス: {content_text[:200]}")
        return {
            "background_color": {"r": 255, "g": 255, "b": 255},
            "font_sizes": {"title": 44, "heading": 32, "body": 18, "caption": 14},
            "repeated_elements": [],
            "text_alignment_default": "left"
        }

def generate_slide_code(client, image_path: str, page_number: int, guidelines: dict) -> str:
    """
    単一画像からpython-pptxのスライド生成コードを直接生成する

    Args:
        client: Bedrockクライアント
        image_path: 画像ファイルパス
        page_number: ページ番号
        guidelines: デザインガイドライン辞書

    Returns:
        スライド生成コード（文字列）
    """
    bg = guidelines['background_color']
    fonts = guidelines['font_sizes']
    repeated = guidelines.get('repeated_elements', [])

    repeated_text = ""
    if repeated:
        repeated_text = "\n除外する要素（繰り返し要素）:\n"
        for elem in repeated:
            repeated_text += f"- {elem.get('type', 'unknown')}: {elem.get('text', '')}\n"

    prompt = f"""この画像はPDFの1ページをスキャンしたものです。
この画像を見て、PowerPointのスライドとして再現するpython-pptxのコードを生成してください。

デザインガイドライン（全体で統一）:
- 背景色: RGB({bg['r']}, {bg['g']}, {bg['b']})
- フォントサイズ基準: タイトル={fonts['title']}pt, 見出し={fonts['heading']}pt, 本文={fonts['body']}pt, キャプション={fonts['caption']}pt
- デフォルトの配置: {guidelines['text_alignment_default']}{repeated_text}

重要な要件:
- スライドサイズは16:9（10インチ×5.625インチ）で固定です。これは絶対に変更しないでください
- この画像のアスペクト比を正確に確認してください
- 画像が16:9と異なるアスペクト比（例: A4縦サイズの7:10）の場合、以下のように処理してください：

座標変換ルール（重要）:
1. 画像のアスペクト比を確認（幅/高さ）
   - 16:9の場合: aspect = 1.778
   - A4縦の場合: aspect = 0.707 (210/297)
   - A4横の場合: aspect = 1.414 (297/210)

2. 画像を16:9スライドにフィットさせる方法:
   - 画像がスライドより縦長（aspect < 1.778）の場合:
     * 高さ基準でフィット: 有効高さ = 5.625インチ
     * 有効幅 = 5.625 * 画像aspect
     * 左右に余白: offset_x = (10 - 有効幅) / 2
     * 座標変換: x = offset_x + (画像内x座標 * 有効幅), y = 画像内y座標 * 5.625

   - 画像がスライドより横長（aspect > 1.778）の場合:
     * 幅基準でフィット: 有効幅 = 10インチ
     * 有効高さ = 10 / 画像aspect
     * 上下に余白: offset_y = (5.625 - 有効高さ) / 2
     * 座標変換: x = 画像内x座標 * 10, y = offset_y + (画像内y座標 * 有効高さ)

3. 必ず画像全体が見えるように配置してください（文字が切れないように）

スライド境界チェック（絶対必須・全オブジェクトに適用）:
- **すべてのテキストボックス、すべての図形、すべてのオブジェクト**に対して境界チェックを実装してください
- 例外は一切認められません。1つでもはみ出すオブジェクトがあってはいけません
- 座標の制約:
  * left >= 0
  * top >= 0
  * left + width <= 10 (スライド幅)
  * top + height <= 5.625 (スライド高さ)

必須の実装パターン（全オブジェクトで使用）:
```python
    # 座標計算
    left_val = 計算値
    top_val = 計算値
    width_val = 計算値
    height_val = 計算値

    # 境界チェック（必須）
    if left_val < 0:
        left_val = 0
    if top_val < 0:
        top_val = 0
    if left_val + width_val > 10:
        width_val = 10 - left_val
    if top_val + height_val > 5.625:
        height_val = 5.625 - top_val

    # Inches変換
    left = Inches(left_val)
    top = Inches(top_val)
    width = Inches(width_val)
    height = Inches(height_val)
    ```

- このパターンを**全ページの全オブジェクト**に適用してください
- 直接Inches()を使わず、必ず上記のパターンを使用してください
- 調整後もテキストが読めるように、極端に小さくならないよう注意してください

テキストの配置（重要）:
- 各テキストブロックの配置（左寄せ、中央寄せ、右寄せ）を画像から正確に判断してください
- テキストがボックスの左端にある → PP_ALIGN.LEFT
- テキストがボックスの中央にある → PP_ALIGN.CENTER
- テキストがボックスの右端にある → PP_ALIGN.RIGHT
- デフォルト値を使わず、必ず画像を見て判断してください

その他の要件:
- テキストの内容、位置、サイズ、スタイル（太字など）を可能な限りもとの画像に忠実に再現してください
- フォントサイズは上記のガイドラインに従ってください
- 背景色は上記のRGB値を使用してください
- 除外する要素（ヘッダー・フッター・ページ番号など）は出力しないでください

出力形式:
- 以下の変数がすでに存在すると仮定してください: prs (Presentationオブジェクト)
- 1枚のスライドを追加するコードのみを出力してください
- コメントで「# ページ {page_number}」を先頭に含めてください
- コードブロック（```python）は不要で、そのままコードを出力してください
- 説明文は不要です

必須インポート（スクリプト先頭で既にインポート済み）:
- from pptx import Presentation
- from pptx.util import Inches, Pt
- from pptx.enum.text import PP_ALIGN
- from pptx.dml.color import RGBColor
- from pptx.enum.shapes import MSO_SHAPE

使用可能なAPI（これ以外は使用しないでください）:
- slide.shapes.add_textbox(left, top, width, height) - テキストボックス追加
- slide.shapes.add_shape(MSO_SHAPE.RECTANGLE, left, top, width, height) - 長方形追加
- slide.shapes.add_shape(MSO_SHAPE.OVAL, left, top, width, height) - 楕円追加
- text_frame.paragraphs[0] - 段落取得
- p.text = "テキスト" - テキスト設定
- p.font.size = Pt(size) - フォントサイズ設定
- p.font.bold = True/False - 太字設定
- p.font.color.rgb = RGBColor(r, g, b) - フォント色設定
- p.alignment = PP_ALIGN.LEFT/CENTER/RIGHT - 配置設定

def generate_script(image_paths: list[str], output_path: str, pptx_output_path: str = None):
    """
    画像からpython-pptxスクリプトを生成してファイルに保存する（2パス処理）

    Args:
        image_paths: 画像ファイルパスのリスト
        output_path: 出力スクリプトファイルパス
        pptx_output_path: PowerPoint出力パス（Noneの場合は'output.pptx'）
    """
    client = get_bedrock_client()

    for image_path in image_paths:
        image_file = Path(image_path)
        if not image_file.exists():
            raise FileNotFoundError(f"画像ファイルが見つかりません: {image_path}")

    print(f"\n=== パス1: デザインガイドライン抽出 ===")
    guidelines = extract_design_guidelines(client, image_paths)

    print(f"\n=== パス2: 各ページのコード生成 ===")
    slide_codes = []
    for i, image_path in enumerate(image_paths, start=1):
        code = generate_slide_code(client, image_path, i, guidelines)
        slide_codes.append(code)

    print(f"\n=== 完全なスクリプトを構築 ===")
    script_lines = [
        "from pptx import Presentation",
        "from pptx.util import Inches, Pt",
        "from pptx.enum.text import PP_ALIGN",
        "from pptx.dml.color import RGBColor",
        "from pptx.enum.shapes import MSO_SHAPE",
        "",
        "prs = Presentation()",
        "prs.slide_width = Inches(10)",
        "prs.slide_height = Inches(5.625)",
        "",
    ]

    for code in slide_codes:
        script_lines.append(code)
        script_lines.append("")

    # 出力パスの設定
    if pptx_output_path:
        pptx_path = pptx_output_path
    else:
        pptx_path = "output.pptx"

    script_lines.append("from pathlib import Path")
    script_lines.append(f"output_path = Path(r'{pptx_path}')")
    script_lines.append("output_path.parent.mkdir(parents=True, exist_ok=True)")
    script_lines.append("prs.save(str(output_path))")
    script_lines.append("print(f'PowerPointファイルを生成しました: {output_path}')")

    script = "\n".join(script_lines)

    output_file = Path(output_path)
    output_file.parent.mkdir(parents=True, exist_ok=True)
    with open(output_file, "w", encoding="utf-8") as f:
        f.write(script)

    print(f"\nスクリプトを保存しました: {output_file}")
    print(f"スクリプトの行数: {len(script.splitlines())}")

def main():
    parser = argparse.ArgumentParser(description="画像からpython-pptxスクリプトを生成する")
    parser.add_argument("image_dir", help="画像ディレクトリパス")
    parser.add_argument("output_script", help="出力スクリプトファイルパス")
    args = parser.parse_args()

    image_dir = Path(args.image_dir)
    if not image_dir.exists():
        raise FileNotFoundError(f"ディレクトリが見つかりません: {image_dir}")

    image_paths = sorted(image_dir.glob("page_*.png"))
    if not image_paths:
        raise FileNotFoundError(f"画像ファイルが見つかりません: {image_dir}")

    print(f"=== python-pptxスクリプト生成 ===")
    print(f"画像数: {len(image_paths)}")

    generate_script([str(p) for p in image_paths], args.output_script)

if __name__ == "__main__":
    main()

## まとめ

現時点では、NotebookLMが生成したスライドを「そのまま編集可能な形に変換する」完全な方法は存在しません。

編集が必要な場面では、NotebookLMの出力を下書きや参考資料として活用しながら、自分の手でスライドを作り直すのが最も現実的な選択肢です。  
他には、SharePointのCopilotに自社テンプレートを読み込ませた状態でスライドの設計情報を流し込み、気に入らない箇所をpython-pptxで生成したスライドからコピペで修正していくという方法が無難です。

今後、NotebookLMが直接編集可能なPowerPointを出力できるようになることに期待しつつ、本記事が同様の課題に直面している方々の参考になれば幸いです。
