## 作ったもの

**[公文書AI検索](https://kobunsho-search.vercel.app)**

国立国会図書館デジタルコレクションの著作権切れ資料（28万冊以上）を、自然言語で検索してAIが要約してくれるWebアプリです。

例えば「戦後の農地改革はどのように行われたか」と聞くと、関連する資料を自動で探し出し、OCRテキストを読み取って回答を生成します。

![検索画面のスクリーンショット](https://res.cloudinary.com/zenn/image/fetch/s--ckHkAC7u--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_1200/kobunsho-search-screenshot.png?_a=BACAGSGT)

## なぜ作ったか

きっかけは「[AIを使って国立国会図書館デジタルコレクションを大量に読んでいる](https://cocolog-nifty.hatenablog.com/entry/2026/03/26/164700)」というはてブ記事（406users）。

国会図書館には膨大な資料がデジタル化されているのに、検索がキーワードベースで使いにくい。研究者やライターが「あの時代のあの話題について書かれた資料」を探すのに、何度もキーワードを変えて試行錯誤している現状がある。

**「自然言語で質問したら、関連資料を見つけてAIが要約してくれたら便利じゃない？」** というシンプルな動機です。

## 技術スタック

| 層 | 技術 |
| --- | --- |
| フロントエンド | Next.js 16 (App Router) + Tailwind CSS |
| AI | Claude API (Haiku: キーワード抽出, Sonnet: 回答生成) |
| データソース | NDL次世代デジタルライブラリーAPI |
| ホスティング | Vercel |

## アーキテクチャ

```
ユーザーの質問
  ↓
Claude Haiku でキーワード抽出
  例: "戦後の農地改革" → ["農地改革", "戦後", "地主", "小作農"]
  ↓
NDL APIで書籍検索（各キーワードで並列検索）
  ↓
上位5件のOCRテキストを取得
  ↓
Claude Sonnet が資料を分析・ストリーミング回答
  ↓
引用元（PID + ページ番号）付きで表示
```

ポイントは **2段階のClaude利用**。Haikuで安くキーワードを抽出し、Sonnetで本格的な回答を生成します。1回の検索コストは約5〜10円です。

## NDL APIの使い方

### 書籍検索

次世代デジタルライブラリーのAPIが非常に優秀です。

```
const res = await fetch(
  `https://lab.ndl.go.jp/dl/api/book/search?keyword=${encodeURIComponent("農地改革")}&size=5`
);
const data = await res.json();
// data.list に検索結果、data.hit に総件数
```

レスポンスにはOCRテキストのハイライト（`highlights`）も含まれるので、検索精度の確認もできます。

### OCR全文テキスト取得

```
const res = await fetch(
  `https://lab.ndl.go.jp/dl/api/book/fulltext-json/${pid}`
);
const data = await res.json();
// data.list[].contents に各ページのOCRテキスト
```

1冊あたり数百KBのJSONが返ってきます。座標情報（`coordjson`）も含まれるので、文字の位置まで分かります。

### 注意点

* **OCR品質はまちまち**。特に古い資料は誤読が多い。Claudeの文脈理解力でカバーしています
* **営利目的かつ継続的な利用は事前申請が必要**。無料公開なら申請不要です
* 対象は著作権保護期間満了資料（PDM）のみ

## Claudeの使い方の工夫

### キーワード抽出（Haiku）

```
const response = await client.messages.create({
  model: "claude-haiku-4-5-20251001",
  max_tokens: 200,
  system: "ユーザーの質問から検索キーワードを2〜5個抽出。JSON配列で返す。",
  messages: [{ role: "user", content: query }],
});
```

Haikuは高速＆安価。キーワード抽出のような軽いタスクに最適です。

### 回答生成（Sonnet）ストリーミング

```
const stream = await client.messages.stream({
  model: "claude-sonnet-4-5-20250929",
  max_tokens: 2048,
  system: "国会図書館の資料を分析する専門アシスタント。引用元を明示。",
  messages: [{ role: "user", content: userMessage }],
});

// Server-Sent Events で逐次送信
for await (const event of stream) {
  if (event.type === "content_block_delta") {
    controller.enqueue(encoder.encode(`data: ${JSON.stringify({ text: event.delta.text })}\n\n`));
  }
}
```

ストリーミングにすることで、体感の待ち時間を大幅に短縮できます。

## ハマったポイント

### 1. Turbopackが日本語でpanic

Next.js 16のTurbopackは、テンプレートリテラル内の日本語マルチバイト文字で内部エラーを起こします。

```
thread panicked at highlight.rs:1011:45:
end byte index 113 is not a char boundary; it is inside '検'
```

**対策**: 日本語のプロンプトを別ファイルに切り出し、`join("\n")`の配列形式で定義。

```
// NG: テンプレートリテラル内の日本語
const prompt = `ユーザーの質問から検索キーワードを抽出してください。`;

// OK: 配列でjoin
export const PROMPT = [
  "ユーザーの質問から検索キーワードを抽出してください。",
].join("\n");
```

### 2. NDL OpenSearch APIのクエリエンコーディング

`q=`パラメータはShift-JISエンコーディングで送るとゴミ結果が返ります。`title=`パラメータをUTF-8で使うのが正解。

```
# NG: 10億件ヒットしてゴミが返る
https://iss.ndl.go.jp/api/opensearch?q=農地改革

# OK: 2500件の正確な結果
https://iss.ndl.go.jp/api/opensearch?title=農地改革
```

ただし、次世代デジタルライブラリーAPI（`lab.ndl.go.jp`）を使えばこの問題は起きません。

### 3. IIIF Manifestのアクセス制限

`dl.ndl.go.jp/api/iiif/{pid}/manifest.json` はPDM（パブリック・ドメイン・マーク）の資料のみアクセス可能。制限のある資料は`{"checkResult": "NG"}`が返ります。
