確定申告の季節になると、毎年繰り返す同じ光景。

レシートの山。スキャンしたPDFのフォルダ。そしてExcelに手入力する夜。独立してバックオフィスを全部自分でやっている身には、これが地味にきつい。医師免許を持って産業医をやっている人間が、タクシーの領収書を1枚ずつ転記しているというのは、人的資源の使い方としてちょっとどうかな？と思いながら、結局毎年同じことをしている。。。

それで、ここにClaude Codeを本格的に使い始めるモチベーションが生まれたわけですよ。

もちろん最初は他のいろいろな人の導入編を見ながら手探りでやるわけですね。で、最初の壁は、「プロジェクトフォルダって何？」というところ。「1つの仕事のファイルをまとめておく入れ物」という説明でようやく腹落ちしたけれども、デスクトップにファイルが散らばりがちな自分には、そもそもその発想が薄かったのでした。。。そこからかい！

```
mkdir -p ~/claude-projects/keihikanri
mkdir -p ~/claude-projects/french
```

この2行を打って、2つのフォルダを作るところから始めます。

次にやったのが、`CLAUDE.md` というファイルを作ること。これがいわば「私の取説」なわけですね。Claude Codeは毎回ゼロリセットで起動します。このチャット画面（[Claude.ai](http://Claude.ai)）と違い、会話を覚えていてくれない。だから自分のことをまとめたファイルを置いておくと、起動するたびに読み込んで「この人はこういう人だ」と把握した状態で仕事を始めてくれるという仕組み。毎度自己紹介をさせられるというのは相当かったるいので、これは大変助かります。

何を書くか。最初は職業と使用ツールだけでしたが、折角なので人柄的なところも入れてみることにしました。Twitterの書き込みをいくつか見せて、Claudeに「この人はこういう傾向がある」と分析してもらい、そのまま取説に変換したら、

出てきたのはこんな記述。

```
# 思考・コミュニケーションのスタイル
- 結論や感情を直接言わず、構造や比喩で語る傾向がある
- 短くて切れ味のある文章を好む。冗長な説明は不要
- 不条理・理不尽には静かに、でも明確に反応する
- ユーモアは皮肉寄り
```

……当たっている。当たっているのが少し悔しい。

取説の中には「大丈夫ですか？系の過保護な確認は不要。へり下りも禁止。対等に話すのを歓迎」という項目も足しました。AIに敬語で話しかけられ続けるのは、やっぱりちょっと居心地が悪い。。

こうして「取説」入りのClaude Codeが完成したところで、次はいよいよ実際に使ってみます！

次回は、40枚のタクシー領収書が3分で片付いた夜の話。
