# AIエージェントによる解析計画立案

構造解析（強度解析）では解析内容により材料を線形にしたり、非線形（降伏後）まで取り扱うなど、解析の計画が変化することがあります。そのような工学的判断をAIがやってくれるのか検証してみます。

## 使用するもの

* Codex App + GPT 5.5
* Ansys MAPDL
* PyMAPDL
* PyDPF Core
* PyDPF Post

## 形状データ

3D形状はあらかじめMAPDLで手作業で作成したものを利用します。  
![MAPDLで作成した3Dジオメトリ](https://static.zenn.studio/user-upload/0a7828081b66-20260425.png)

## 構造解析（弾性範囲のみ）

材料を降伏点以下で使う前提の解析を行ってみます。材料が降伏したら結局アウトなので、解析では特段非線形の設定や降伏後の材料特性を定義する必要はないはずなので、そのあたりが注目点です。

以下のようなプロンプトをCodexのPlanモードで実行し、計画を作ってもらいます。

```
以下の解析のプランを作成し、プランの根拠と狙いを説明する。

# 解析目的
PyMAPDLで以下の解析を実施し、この部品が安全であるかどうか確かめる。
材料は弾性範囲で使う前提で、安全率1.5以上が必要。

# 形状データ
./mapdl_work/bar/bar_body.cdb
長さの単位系はmm

# 境界条件
X=0の面を固定、X=100の面に+X方向に40000N負荷

# 材料
SS400とする。材料物性はWebで調べてユーザーに確認すること。

# 留意事項
ポスト処理にはPyDPF Postを利用すること。
```

### Codexからの確認事項

材料物性はWebで調べるように依頼していましたが、いくつか候補があるようなので確認してくれました。こういう地味に面倒な作業をやってくれるのはありがたい。  
![](https://static.zenn.studio/user-upload/901cd8870325-20260426.png)

![](https://static.zenn.studio/user-upload/313262c234ab-20260426.png)

### Codex作成プラン

結構しっかりプランを考えてくれました。  
![](https://static.zenn.studio/user-upload/0eb963284a68-20260426.png)  
単位系はOKですね。長さがmmなので応力やヤング率の単位はMPaにするのが定石です。材料物性の出典も記載してあるのは素晴らしい。

![](https://static.zenn.studio/user-upload/b3a86e3f57e0-20260426.png)  
荷重を等価面荷重に変えているのも妥当。RBE3とかで節点荷重に分配する方法もありますが、このケースでは面荷重のほうが簡単です。静解析で大変形OFFもこの解析内容であれば妥当。

![](https://static.zenn.studio/user-upload/5c82808135dd-20260426.png)  
Test PlanはだいたいOKですが特異点も考慮したほうが良いかも。Mises応力を利用するのは、この材料であれば妥当です。

### プランを実行

プランを実行すると試行錯誤しながら、最終的にレポートにまとめてくれました。  
![](https://static.zenn.studio/user-upload/eccba2d0bce6-20260426.png)  
![](https://static.zenn.studio/user-upload/9658a347b82b-20260426.png)  
![](https://static.zenn.studio/user-upload/a8e585491e4f-20260426.png)  
ちょっと内容が煩雑ですが、反力チェックもできておりなかなかの内容。  
ただメッシュがどうなっているのかとか、最大応力が中央部で出ているのか（特異点を拾っていないか）が気になるところ。荷重のかけ方もよくない。（端面の全節点に同じ荷重をかけると、エッジ部分に過大な荷重がかかってしまう）

### 手動で結果ファイルを確認してみる

手動で確認してみると不安的中。メッシュが粗いし、端部にMAX応力が出てしまっています。AIのレポートは一見よさそうに見えても、3Dの図を見て判断していないので、こういう問題が起きるということがわかりました。  
![](https://static.zenn.studio/user-upload/e0947e81cc99-20260426.png)

# AIエージェントによる解析結果レビュー

人がチェックすれば解析のおかしな点が把握できますが、AIエージェントがどこまで自律的に問題点を把握して修正できるか試してみます。

## Codexにレビューさせてみる

Codexにレビューを依頼します。

```
今回の解析の妥当性をチェックし、問題点があれば報告すること。
```

するとさっそく問題点を見つけてくれました。  
![](https://static.zenn.studio/user-upload/0dc480d0aab8-20260426.png)  
荷重と丸棒の面積から公称応力を算出して解析結果と比較し、AIが「数値の矛盾（面積×圧力と全荷重の不一致）」に気づき解析結果が妥当でないことを発見しており**工学的な判断ができています**。荷重のかけ方に問題があることや、メッシュの比較がされていないことにも気づいてくれました。素晴らしい。

## レビュー内容を実行させてみる

Codexがレビューで出した推奨対応を実行させてみると、荷重を面荷重に変更してメッシュサイズを複数切り替えて解析してくれました。  
![](https://static.zenn.studio/user-upload/809463149c03-20260426.png)  
![](https://static.zenn.studio/user-upload/19da7e122d8d-20260426.png)  
単にメッシュサイズを変えただけでなく、最大応力が収束傾向にあるかもチェックしています。最大応力が収束すれば妥当なメッシュサイズというのは本に書いてある**知識**としてAIが持っているとは思いますが、それを実際にコードを書いて動かして確認し**工学的判断ができている**のは良いですね。

そして注目したいのが作業時間。メッシュサイズ変更・再解析・結果まとめまで10分程度でやってのけました。このTRY＆ERRORが速いというのがAIエージェントのアドバンテージの1つです。

実際に結果ファイルを手動で開いてみると、妥当な結果になっていると推察できる内容でした。  
![](https://static.zenn.studio/user-upload/94a951dcdf0c-20260426.png)

# 解析結果レポート作成

最後にレポートを作成してもらいます。報告する相手によって、レポート内容を分けることも試してみます。

## 報告相手を指定しないレポート

プロンプト：

```
この解析結果のレポートをMarkdownで作成。応力MAX位置がわかる分布図やメッシュ図も含めるように。
```

特に報告相手を指定しない場合は、解析の結果と妥当性を中心に述べていて、どちらかというとCAE技術者向けの内容になっています。

# まとめ

AIエージェントに解析計画立案/実行/レビュー/レポート作成まで一通りやらせてみました。最初の解析はおかしなところもありましたが、AI自身にレビューさせることで問題を解決し妥当な解析結果までもっていくことができました。  
今回のは非常に単純な形状とはいえ、GPTのような言語モデルが**3Dの物理現象を理解して工学的な判断までやってくれた**のには少々驚きました。思っていたよりもよくできたという印象です。  
そして、**資料作成は明らかに人間よりも上手**です。

## 自律型CAE AIエージェントにおける人間の役割は？

一連の作業を振り返ってみると、少々プロンプトを打っただけであとはAIエージェントがほぼ自律的に作業をしていました。人が介在しないといけない部分は何だろうかと考えてみました。

### 形状と環境の準備は必要

まず形状の準備は必要です。最近はCAD AIエージェントもいろいろ出てきているので将来的にはAIにお任せになるかもしれませんが、現時点では人がやったほうが早いと思います。CADのように完成形が明確な場合は、AIのように確率的ゆらぎのある出力をされると困るという事情もあります。

### あとはタスクの設定と承認作業だけ

どんな条件でどんな結果を見たいのかを指示すれば、あとはAIエージェントが勝手に回してくれたという印象でした。  
これまでCAE技術者がどちらかというと中心的に時間を割いてきた解析テクニック的なところはAIで大部分カバーできそうに感じます。特にAIエージェントが優れているのは**TRY＆ERRORが速い**という点で、CAE技術者がAIの力を使うと仕事の内容が変わってくるように思います。

### おかしなことに気付けるスキルは必要かも

今回はAI自身にレビューさせて問題点を発見・修正できましたが、人が結果をチェックしておかしな部分をあらかじめ把握していたので安心して任せられたという側面もあります。「なんかここおかしいな」と気付けるスキルは今後も必要かもしれません。
