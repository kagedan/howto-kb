## はじめに

1 + 1 = 2。これは揺るがない。

では、∞ + 1 は？ 答えは ∞ だ。∞ + ∞ も ∞。じゃあ、∞ と ∞ + 1 はどちらが大きい？ ……ここで、普段頼りにしている「論理」がぐらつき始める。

これは数学パズルの話ではない。Amanda Askellというひとりの哲学者が、この問題に真正面から取り組んでいる。彼女の問いはこうだ。

「無限が絡む状況で、私たちはどうやって"より良い選択"を判断すればいいのか？」

Askellは現在、Anthropic（Claudeの開発元）でAIの振る舞いを設計するチームを率いている。「無限の倫理」を研究していた哲学者が、なぜAI企業にいるのか。それ自体が、この研究の射程の広さを物語っている。

本稿では、Askellの無限倫理（Infinite Ethics）の考え方を、できるだけ平易に解説する。

## 1. そもそも「倫理」はどうやって"良い選択"を決めるのか

Askellの研究に入る前に、倫理学の基本的な枠組みを押さえておく。

### 功利主義（Utilitarianism）

「最大多数の最大幸福」。全員の幸福の総和が最も大きくなる選択が正しい、という考え方。

```
選択肢A: 100人が幸福度 +10 → 総幸福 = 1,000
選択肢B:  80人が幸福度 +15 → 総幸福 = 1,200

功利主義の答え: B
```

シンプルで強力なフレームワークだ。数値化して比較すればいい。エンジニアリングの意思決定にも通じる合理性がある。

### パレート原則（Pareto Principle）

「誰かの状況を改善し、誰の状況も悪化させないなら、その変更は"良い"」。

```
選択肢A: アリスの幸福 10、ボブの幸福 5
選択肢B: アリスの幸福 12、ボブの幸福 5

パレート原則の答え: B（アリスが改善、ボブは同じ → 文句なしにBが良い）
```

これも直感的に納得できる。少なくとも有限の世界では。

## 2. 「無限」が登場すると、すべてが壊れる

ここからが本題だ。上の枠組みに「無限」を持ち込むと何が起きるか。

### 無限ホテルのパラドクス ― 直感を壊すウォームアップ

数学者ヒルベルトが考えた有名な思考実験がある。

> 無限の部屋があるホテルが満室だとする。新しい客が1人来た。フロント係は全員に「隣の部屋に移ってください」と言う。すると部屋1が空き、新しい客が入れる。満室なのに新しい客を受け入れられる。

無限を扱うと、有限の世界では当たり前の「満室なら入れない」というルールが通用しなくなる。同じことが倫理的判断でも起きる。

### 功利計算の崩壊

こんな状況を考えてみよう。

```
世界A: 無限人の住民がいて、全員の幸福度が 10
世界B: 無限人の住民がいて、全員の幸福度が 20

直感: Bの方が良いに決まっている

功利主義の計算:
  世界Aの総幸福 = 10 × ∞ = ∞
  世界Bの総幸福 = 20 × ∞ = ∞

結果: ∞ = ∞ → 「同じ」と判定される
```

全員の幸福が2倍なのに「同じ」。これは明らかにおかしい。功利主義の「総和を比較する」という方法が、無限の前では機能しなくなる。

### もっと厄介なケース

```
世界C: 住民1は幸福度1、住民2は幸福度2、住民3は幸福度3 ... （永遠に続く）
世界D: 住民1は幸福度2、住民2は幸福度3、住民3は幸福度4 ... （永遠に続く）
```

世界Dは世界Cの「全員が+1されたバージョン」だ。直感的にはDの方が良い。だが総和はどちらも無限大で、差を計算しても ∞ - ∞ = 不定形。論理的な比較ができない。

ここで起きていることを整理すると：計算（論理）は「どちらも無限だから同じ」と言う。しかし直感（倫理的判断）は「全員がより幸福なDの方が良いに決まっている」と言う。論理と直感が食い違っている。

## 3. Askellのアプローチ ― パレート原則を無限に拡張する

Amanda Askellの博士論文『Pareto Principles in Infinite Ethics』（2018年）は、この問題に対して一つの回答を示した。

### 基本的なアイデア

功利主義の「総和を比較する」方法が壊れるなら、別の基準を使えばいい。Askellが着目したのがパレート原則だ。

> 世界Dは世界Cと比べて、すべての住民の幸福が+1されている。  
> 誰かが改善され、誰も悪化していない。  
> → パレート原則により、Dの方が良い。

総和を計算する必要がない。一人一人を比較するだけでいい。無限 × 無限の計算を避けて、「個人レベルの比較」に落とし込む。

### ただし、パレート原則だけでは足りない

パレート原則で解決できるケースは限定的だ。

```
世界E: 偶数番目の住民が幸福度 10、奇数番目が幸福度 0
世界F: 偶数番目の住民が幸福度 0、奇数番目が幸福度 10
```

世界Eと世界F、どちらが良い？ 半数が改善され、半数が悪化している。パレート原則では判定できない。

Askellはここで、パレート原則を拡張したいくつかの原則を検討している。

* Weak Pareto: 全員が改善されるなら、その選択は良い（上の世界C vs Dのケース）
* Strong Pareto: 少なくとも1人が改善され、誰も悪化しないなら良い
* さらに強い原則の探索: 「無限のトレードオフ」をどこまで合理的に判定できるか

重要なのは、Askellが「完璧な答えを出した」わけではないということだ。むしろ彼女の貢献は、「功利計算が壊れた後にも、倫理的判断の道具はある」ことを示し、その道具の限界を正直に整理した点にある。

## 4. 「無限」は本当に現実の問題なのか？

「面白い思考実験だけど、現実には無限の人間はいないでしょ」と思うかもしれない。しかし、無限に近い状況は意外と身近だ。

### 未来世代の問題

気候変動の議論を考えてみよう。今の行動が影響を与える未来世代の数は、事実上無限に近い。「今の世代の利益を100万損なうことで、今後1000世代にわたって各世代が1万の利益を得る」場合、功利計算をどう行うべきか。将来の世代をどこまで割り引くか（割引率の問題）は、無限倫理の議論と直結する。

### パスカルの賭け的な状況

確率は極めて低いが、起きた場合の影響が極めて大きい（事実上無限の）リスク。AI安全性の議論で頻繁に登場する「存在リスク（existential risk）」はまさにこの構造を持つ。

```
期待値 = 確率 × 影響度

通常のリスク:   0.01 × 1,000 = 10（計算可能）
存在リスク:     0.0001 × ∞ = ?（計算不能）
```

確率がどんなに低くても、影響度が無限なら期待値は無限になる。あるいは不定になる。「期待値を最大化せよ」という功利主義のルールが、ここでも壊れる。

### AIの判断回数

大規模なAIシステムは1日に何十億回もの判断を行う。推薦アルゴリズム、コンテンツモデレーション、自動運転の判断。個々の判断は小さくても、積み重なると影響は莫大になる。Askell自身が指摘しているように、「一つ一つは小さな判断でも、無限に近い回数繰り返されるなら、その倫理的重みは無限に近づく」。

## 5. Askellがなぜ Anthropicにいるのか

ここで、Amanda Askellという人物について簡単に紹介しておく。

イギリス出身の哲学者。ニューヨーク大学（NYU）で博士号を取得し、博士論文のテーマがまさに本稿で紹介した無限倫理だった。卒業後、AI安全性の研究機関を経て2021年にAnthropicに参画。現在はClaudeの振る舞い（性格や応答の原則）を設計するチームを率いている。「哲学者がAIの"人格"を作っている」という、一見奇妙に見えるキャリアの背景には、ここまで見てきた研究の流れがある。

### AIアライメントとは「倫理で判断する仕組み」を作ること

AIアライメント（人間の意図にAIを合わせること）の核心は、AIに「良い判断」をさせることだ。しかし、何が「良い」かを純粋に論理だけで定義することはできない。Askellの無限倫理の研究が示したのは、まさにその限界だった。

Askellは現在、Anthropicでいくつかの領域に関わっている。

#### Claudeの「性格」の設計

Claudeのシステムプロンプトや振る舞いの指針（Claude's Character）の策定にAskellは深く関わっている。「ユーザーに同意しすぎない」「知的に誠実であれ」といった原則は、功利的な最適化（ユーザー満足度を最大化する → 迎合する）とは異なる倫理的判断を組み込んだものだ。

#### 「論理が割れたとき」のAIの振る舞い

LLMは日常的に「論理だけでは決められない問い」に直面する。

```
ユーザー: 「この薬はがんに効きますか？」

論理A: 一部の予備的研究では効果が示唆されている → 「効く可能性があります」
論理B: 大規模臨床試験では有意差なし → 「効果は確認されていません」
```

どちらも論理的には正しい。しかし回答の仕方によって、ユーザーの健康に関わる判断が変わる。ここで必要なのは、「どちらの情報をどう伝えるべきか」という倫理的判断だ。

#### AIの判断が「無限に近い回数」行われる現実

Claudeは日々、膨大な数のユーザーと対話している。一つ一つの回答の倫理的影響は小さくても、それが何百万回と繰り返されれば、社会全体への影響は無視できない。Askellが研究してきた「無限に近いスケールでの倫理的判断」は、そのままAIの運用課題になっている。

Askellの経歴は、哲学の抽象的な議論とAI開発の現場がいかに地続きかを示している。「無限で功利計算が壊れる」という純粋哲学の問題は、「AIに何を最適化させるか」という工学の問題と、根っこでつながっている。

## 6. エンジニアへの示唆 ― 論理が割れたら何で決める？

Askellの研究から、エンジニアが持ち帰れる考え方を整理する。

### 6.1 「最適化」の限界を知る

エンジニアリングは最適化の学問だ。しかし、何を最適化するかは最適化では決められない。

```
推薦システム: エンゲージメントを最適化 → エコーチェンバーが発生
SNS: 滞在時間を最適化 → メンタルヘルスへの悪影響
LLM: ユーザー満足度を最適化 → sycophancy（迎合）が発生
```

これらはすべて「論理的に正しい最適化が、倫理的に問題のある結果を生む」ケースだ。最適化関数の選択そのものが倫理的判断であることを、Askellの研究は示唆している。

### 6.2 「パレート的に考える」習慣

完璧な解がない状況でも、パレート原則的な思考は使える。

> 「この変更で、誰かの状況が改善し、誰の状況も悪化しないか？」

もしYesなら、その変更は論理が割れていても正当化できる。Noなら、トレードオフを明示して判断を仰ぐ。

```
例: パフォーマンス改善でコードの可読性が下がる

パレート的に考える:
  - パフォーマンス: 改善 ✓
  - 可読性: 悪化 ✗
  → パレート改善ではない → トレードオフを明示して議論が必要
```

### 6.3 「倫理的デフォルト」を設計に組み込む

論理だけでは判断できないケースに備えて、あらかじめ「迷ったらこちらを選ぶ」というデフォルトを設計に組み込む。

Anthropicが Claudeに対して行っているのがまさにこれだ。

```
迷ったら...
- 安全側に倒す（不確かな情報は不確かだと伝える）
- ユーザーの自律性を尊重する（判断を代行しない）
- 透明性を優先する（何が分からないかを正直に言う）
```

これらは「功利的に最適」とは限らないが、「倫理的に防御可能」な選択だ。

## 7. おわりに ― 「正しい答えがない」ことの正しさ

Askellの無限倫理が教えてくれる最も重要なことは、逆説的だが「論理には限界がある」という事実を論理的に示せるということだ。

私たちは日常的に、論理だけでは決まらない判断を下している。技術選定、プロダクトの方向性、AIの振る舞い。データを集め、ロジックを組み立て、それでも答えが一つに定まらないとき、最後に頼るのは「何が正しいか」という倫理的直感だ。

それは弱さではない。Askellの研究が示すように、論理が限界に達した先にこそ、倫理の出番がある。

そして、この問題はもはや哲学者だけのものではない。AIが日々何十億もの判断を下す時代に、「論理が割れたとき何で決めるか」は、コードを書くすべての人に関わる問いだ。
