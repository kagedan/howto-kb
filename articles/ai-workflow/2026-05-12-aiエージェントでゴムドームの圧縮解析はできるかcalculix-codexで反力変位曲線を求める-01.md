# AIエージェントで構造非線形解析のモデル化

今回はゴムドームの圧縮（押し込み）解析を試します。所定量押し込んだときの押し込み量に対する反力曲線を得ることを目的とします。  
この解析は材料非線形（超弾性材料）＋大変形＋接触という構造非線形を代表する3つの非線形性が同時に発生するという解析難易度が高い題材で、適切なモデル化を行わないとうまくいかないと予想されます。  
AIはおそらく様々な解析テクニックは知識として持っていると思われますが、それを実際の物理問題に対して適用し、CAEモデル化の相談相手になりうるのかに注目します。

## 使用するもの

* Codex App + GPT5.5 （AIエージェント）
* CalculiX （CAEソルバー）
* Gmsh （メッシュ作成用）
* meshio （メッシュ変換用）
* build123 （ジオメトリ作成用）
* FreeCAD （手作業によるジオメトリ作成用）

プロジェクトディレクトリにて`AGENTS.md`と[CAEワークフロー](https://zenn.dev/ms_ai/articles/6ed4f9e640ed26)も設定しています。

### 境界条件

ゴムドームを平面に置いて、上側から平面で10mm圧縮します。ただし平面はモデル化していないので、新たに作成する必要があります。

### 求めたいもの

圧縮したときの荷重変位曲線を求めます。

### 材料データ

適当に作った応力vsひずみ線図をcsvで渡してみます。超弾性材料として定義したい場合は、ここから材料構成則にあわせたフィッティングが必要です。  
なおCalculiXでは完全非圧縮として取り扱えないため、少し工夫が必要とみられます。


## CAEモデル化の考え方

このような解析では、よく知られた解析テクニックや知見がいくつか存在します。以下にそれらを記載しますがCodexには教えていません。Codexが自動的に発見/適用できるのかが気になるところです。

### 1.ゴム部品だけでは解析にならない

今回用意した形状データはゴム部品だけですので、上下の平面を作成する必要があります。下のほうはUZを変位拘束するという手もあります（面から離れないという仮定）。これらは通常剛体壁として取り扱います。

### 2.荷重設定

10mm押し込むと指定しているので、変位制御で押し込む形になるはずです。このような接触解析では力制御よりも変位制御のほうが安定します。  
押し込む剛体壁の反力を取得することで、荷重変位曲線が得られます。

### 3.使用要素

形状は3Dで用意しましたが、実際には2D軸対称要素を使うほうが解析的には安定し計算コストも低くなります。ただしCalculiXの2D軸対称要素ではY軸を対称軸としないといけないので、今回の形状データは剛体回転させてあげないと使えません。そのあたりをどう判断するかが気になるところです。

### 4.接触解析

接触解析には様々な手法がありますが、CalculiXではペナルティ法が使えるようです。この場合、ペナルティ剛性をどう扱うかがポイントになります。低すぎると食い込みすぎて失敗、高すぎると発散してしまいます。

### 作成された解析プラン

作成されたプランにはCAEとして必要なモデル化の判断がいくつか含まれていました。  
Codexの解析プランで注目した点は以下の通り。

#### 1. 3D STEPから2D軸対称モデルへ落とし込む判断

3D STEP形状をそのまま3D解析するのではなく、子午断面を抽出して2D軸対称モデルを作る方針になっていました。今回のゴムドームは形状も荷重も軸対称とみなせるため妥当な判断です。まず軽量な2D軸対称モデルで挙動を確認する、というCAEらしい進め方になっています。  
ただし、Z軸を対称軸としているので、このままだとうまくいかないように思われます。どう取り扱うか気になるところです。

#### 2. ゴム単体ではなく、上下の剛体平面を追加する判断

ゴム部品だけでは圧縮解析にならないため、上下に剛体平面を追加する計画になっていました。下側平面を固定し、上側平面に10 mmの強制変位を与え、その反力から荷重変位曲線を求める設定です。実際の圧縮試験における押し治具と受け治具をCAEモデルに反映する考え方です。

#### 3. 荷重制御ではなく、変位制御で反力を読む設定

もともとの解析条件として変位量を指定していたため、変位制御で接触させ、その反力を読むプランになっています。接触解析として妥当な考え方です。

#### 4. 材料CSVから超弾性材料モデルへフィットする流れ

CSVで与えた単軸公称応力—公称ひずみデータを読み込み、超弾性材料の一種であるMooney-Rivlin 2定数モデルへフィットする計画になっていました。線形材料で妥協せずに超弾性材料を利用しているのは良い判断だと思います。

#### 5. スモーク解析・メッシュ依存・接触状態確認まで含めた検証計画

非線形接触解析では、計算が最後まで流れたことと、結果が妥当であることは別問題です。Codexの解析プランでは、その妥当性確認まで含めて進めようとしている点が好印象です。

#### 6. 人間側で確認すべき制限事項

全体としてかなり緻密な計画を立てていると感じました。よく使われる解析テクニックもだいたい網羅されています。  
ただし接触解析は接触剛性やメッシュの状態によっても影響を受けるので、最終的な接触妥当性は人間が確認する必要があると考えます。

### 解析プラン（全文）

> # ゴムドーム 10 mm 圧縮解析計画
>
> ## Summary
>
> `cases/rubber_dome_3d` に解析ケースを整備し、STEP 形状から軸対称 2D FEM モデルを作成して、剛体平面で上側から 10 mm 圧縮したときの荷重変位曲線を求める。接触は上下とも摩擦なし、材料 CSV は単軸公称応力-公称ひずみデータとして扱う。
>
> ## Key Changes
>
> * `analysis.md` と `checklist.md` を作成し、目的、単位系、軸対称化、材料フィット、接触、メッシュ、検証項目を記録する。
> * STEP の子午断面を Gmsh OCC で抽出し、半径方向 `R >= 0`、高さ方向 `Z` の 2D 軸対称メッシュを生成する。
> * CalculiX 入力は `input/main.inp` と `mesh/materials/sections/bcs/loads/step/output.inc` に分割する。
> * 要素は軸対称 2D 要素を第一候補にし、実装時に CalculiX 2.23 の小規模スモーク解析で要素・超弾性・接触の組み合わせを確認する。第一候補が不適合なら、同じ軸対称モデル内で対応要素へ切り替え、変更理由を `analysis.md` に記録する。
> * 上下平面は剛体または実質剛体の接触面として表現し、下側は固定、上側は `Z=-10 mm` の強制変位を与える。荷重は上側平面の `Z` 反力から算出する。
> * `material.csv` の単軸データから Mooney-Rivlin 2 定数を最小二乗フィットし、近非圧縮性として体積剛性を設定する。フィット結果、元データ比較図、係数を保存する。
> * 変位 0-10 mm の荷重変位曲線を `results/load_displacement.csv` と図に出力する。
>
> ## Result Review And Report
>
> * 収束ログ、接触状態、反力バランス、変形形状、底面/上面接触の妥当性を確認してから報告書を書く。
> * メッシュ依存を確認するため、少なくとも粗/標準/細の 3 メッシュで荷重変位曲線を比較する。標準結果を主結果、差分を制限事項として記録する。
> * 日本語 Markdown 報告書 `report/report.md` を作成し、形状概要、材料フィット、メッシュ、境界条件、接触、荷重変位曲線、変形図、接触/反力レビュー、制限事項を含める。
> * 図は `report/figures/` に保存する。
>
> ## Test Plan
>
> * Python スクリプトの構文確認を行う。
> * メッシュ生成後、断面寸法が STEP の概略寸法、外径約 110 mm・高さ約 38 mm と一致することを確認する。
> * CalculiX は環境 BAT を呼んだ上で `ccx jobname` 形式で実行する。
> * 解析完了後、`.dat/.sta` の重大警告、非線形収束、上側反力の符号と単調性、10 mm 時の最終反力を確認する。
> * 荷重変位 CSV に `displacement_mm` と `force_N` を含め、図と数値が一致することを確認する。
>
> ## Assumptions
>
> * 形状と荷重は完全に軸対称で、偏心や傾きは考慮しない。
> * 上下平面は摩擦なし接触とする。
> * 材料 CSV は単軸の公称ひずみ[-]・公称応力[MPa]であり、ゴムは等方・超弾性・近非圧縮性として扱う。
> * 粘弾性、永久変形、摩擦発熱、座屈後の非軸対称変形は今回の範囲外とする。

### 確認すべき点

CAE技術者として見ると確認すべき点も残っています。

* 10 mm圧縮時の反力が0.09 N程度とかなり小さく、**物理的・工学的な直観に反する**。
* 全体的にメッシュが粗いためか、細メッシュで反力が大きく変わっている
* 下側支持が接触ではなくUZ拘束で近似されている　（→ただし本当に接触にすると解析が安定しないので初期検討としてはこれでもよいと思います）

「解析を形にする」ところまではできていますがモデル化やメッシュの妥当性という点では、気になるところがあります。

#### （補足）2D軸対称要素について

CalculiXでは2D軸対称要素はY軸を対称軸にとる必要があります。レポートではZ軸と書かれていたのでcgxで調べてみると、ちゃんとY軸をとっていました。  
つまりレポートではR-Z座標系に書き換えて出しているだけで、解析はXY平面で行っていたということになります。AIエージェントの器用さに驚くばかりです。

#### CalculiXの接触解析の難しさが見えてきた

その後、いろいろ調整した結果、AIエージェントではなくソルバー性能の問題であろうという考えに至りました。効果があったのは接触剛性を1.5~2.0ぐらいまで減らすという対策でした。ゴム材料の物性からすると50,100といった数値は高すぎておかしな接触になっていたようです。  
商用ソルバーの場合は自動的に接触剛性を調整してくれるので、あまり問題にならないのですが、CalculiXにはそのような手厚い機能がないので調整しないとうまくいかないようです。

### AIエージェントはCAE作業を支援できるが人の判断も重要

今回の解析では、AIエージェントが形状処理、材料フィット、CalculiX入力作成、実行、後処理、感度解析、レポート作成まで広範囲に自動化できることが分かりました。

一方で、非線形接触解析では計算が収束しただけでは不十分で、反力のオーダー、軸対称出力の解釈、接触剛性、メッシュ、変形形状、接触状態を確認しながら、解析結果を吟味する必要があります。

AIエージェントはCAE作業を強力に支援できることはわかりましたが、接触解析のような難しい領域では人が**物理的/工学的直観も利用して**違和感を持ち、検証を追加し、限界を判断することが重要です。
