## はじめに

前回の【要件定義編】では、機能要件・データ設計・技術スタック（Streamlit + Gemini API）を確定しました。

今回は「何を作るか」から\*\*「どう作るか」\*\*に落とし込む基本設計フェーズです。実装に入る前の設計図を作ります。今回もClaude・ChatGPT両方に相談しながら進めました。

---

## 基本設計で決めたこと

### ① システム構成

全体の処理の流れはシンプルにこの形にしました。

```
ユーザー → Streamlitアプリ → JSON読み込み → 回答 → Gemini API → 解説表示
```

| レイヤー | 内容 |
| --- | --- |
| フロントエンド | Streamlit |
| データ | JSONファイル |
| AI解説 | Gemini API |

---

### ② 画面構成（1画面完結）

Streamlitの特性を活かして、1画面で完結する構成にしました。

* タイトル（AWS SAA 問題集）
* カテゴリ表示（IAM）
* 問題番号（例：1 / 5 + 進捗バー）
* 問題文
* 選択肢（ラジオボタン）
* 回答ボタン
* 正解・不正解表示
* ローディング表示（解説生成中）
* AI解説表示
* 次の問題へボタン

---

### ③ 画面動作フロー

1. アプリ起動
2. JSONから問題を読み込む
3. 問題を表示する
4. ユーザーが選択肢を選ぶ
5. 回答ボタンを押す
6. 正解・不正解を表示
7. 「Geminiで解説を生成中...」を表示
8. Gemini APIを呼び出す
9. 解説を表示
10. 次の問題へ進む

---

### ④ ファイル構成

```
📁 aws-saa-quiz-app
├── 📄 app.py                      ← Streamlitメイン
├── 📁 data
│   └── 📄 questions.json          ← 問題データ
├── 📁 services
│   └── 📄 gemini_client.py        ← Gemini API処理
├── 📁 prompts
│   └── 📄 base_prompt.md          ← ベースプロンプト
├── 📄 .env                        ← APIキー（GitHubに上げない）
├── 📄 .gitignore
├── 📄 requirements.txt            ← 必要なライブラリ一覧
└── 📄 README.md
```

`app.py` とGemini API処理を `services/gemini_client.py` に分けることで、UI処理とAPI処理が分離されて管理しやすくなります。

---

### ⑤ モジュール設計

**app.py**

* 画面表示
* ユーザー入力処理
* 正誤判定
* gemini\_client.py の呼び出し

**services/gemini\_client.py**

* Gemini APIとの通信処理
* プロンプト生成
* 解説テキストの取得

---

### ⑥ Gemini API連携設計

| 項目 | 内容 |
| --- | --- |
| 呼び出しタイミング | 回答ボタン押下後 |
| 渡す情報 | 問題文・選択肢・正解・ユーザーの回答 |
| プロンプト内容 | 初心者向けに正解の理由と他選択肢が違う理由を説明する |
| 出力 | 初心者向け解説文 |

---

### ⑦ UI/UX設計

**ローディング表示**

**エラー時**

* API失敗：「解説の取得に失敗しました。再試行してください。」を表示
* JSON読み込み失敗：エラーメッセージを表示してアプリを停止

---

### ⑧ 状態管理（Streamlit）

Streamlitは画面操作のたびにコードが再実行される仕組みです。そのため以下の状態を `st.session_state` で管理します。

* 現在の問題番号（index）
* 選択された回答
* 回答済みフラグ

これを知らずに実装すると「回答したのにリセットされる」という問題が起きやすいので、最初から意識しておくのが重要です。

## AIに相談した感想

今回も同じ質問をClaude・ChatGPT両方に投げました。

**Claudeの視点**：画面イメージの具体化とGemini APIの詳細設計（入力・指示内容）まで踏み込んでくれた。プロンプト資産管理の観点で `prompts/` フォルダをファイル構成に入れる提案が良かったです。

**ChatGPTの視点**：状態管理（`st.session_state`）という**Streamlit特有の重要な概念**を指摘してくれた。実装時に詰まりやすいポイントを先に教えてくれたのは助かりました。

両方に投げることで、**実装時に詰まりやすいポイントを事前に把握**できたと感じています。
