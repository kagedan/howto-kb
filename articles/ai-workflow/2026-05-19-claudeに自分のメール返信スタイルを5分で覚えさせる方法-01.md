結論から言うと、メール返信は「自分の文体サンプル」を先に渡すと、かなり使いやすくなります。

Claudeにいきなり「このメールに返信して」と頼むより、過去の返信文を3つ見せてから頼む方が、自然な文章になりやすいです。

私の場合、メール返信の下書きにかかる時間が、1通あたり10分から3分くらいまで短くなることがあります。

この記事では、Claudeに自分のメール返信スタイルを覚えさせる手順を紹介します。

## 毎日のメール返信に何時間かけていますか?

メール返信で時間がかかる理由は、文章そのものよりも「ちょうどいい温度感」を探すことです。

丁寧すぎると距離があります。  
短すぎると冷たく見えます。

相手との関係、用件の重さ、こちらの立場。  
そのあたりを考えているうちに、1通の返信で意外と時間を使います。

ここをClaudeに手伝ってもらうと、かなり楽になります。

ただし、最初から完璧な返信文を期待しない方がいいです。

大事なのは、Claudeに「私ならこう書く」というサンプルを渡すことです。

## Claudeに自分のスタイルを覚えさせる「3つのサンプル提示法」

結論から言うと、過去の返信メールを3つ渡すだけで、出力の精度はかなり変わります。

もちろん、個人情報や社名はそのまま入れません。  
相手名や会社名は、A社、担当者Aのように置き換えます。

使うプロンプトは次の形です。

```
以下は、私が過去に送った業務メールのサンプルです。
この文体・温度感・文章量を参考にして、今後の返信文を作ってください。

条件:
- ですます調
- 丁寧だが、かしこまりすぎない
- 1文を長くしすぎない
- 相手を急かす表現は避ける
- 不明点は勝手に補わず、確認事項として残す

サンプル1:
ここに過去の返信文を貼る

サンプル2:
ここに過去の返信文を貼る

サンプル3:
ここに過去の返信文を貼る

上記をもとに、私のメール返信スタイルを3点で要約してください。
```

先にスタイルを要約させるのがポイントです。

いきなり返信文を作らせるのではなく、Claudeがこちらの文体をどう理解したかを確認します。

たとえば、次のように返ってきます。

* 冒頭で相手へのお礼を入れる
* 本文は短めで、要点を先に書く
* 依頼や確認は柔らかい表現にする

この要約が自分の感覚と合っていれば、そのまま返信作成に進めます。

違う場合は、「もう少し簡潔に」「もっと事務的に」「やわらかさを減らして」などと調整します。

## すぐ使えるテンプレ コピペOK

ここからは、実際に返信文を作るテンプレです。

```
あなたは私の業務メール返信を手伝うアシスタントです。

前提:
- 私の文体は、丁寧だが堅すぎない
- 返信は短め
- 結論を先に書く
- 相手への配慮は入れる
- 不明点は勝手に補わない

依頼:
以下の受信メールに対する返信文を作ってください。

条件:
- 件名は不要
- 本文のみ
- 300字以内
- 必要なら確認事項を最後に1つだけ入れる

受信メール:
ここに相手からのメールを貼る
```

このテンプレは、日常的な業務メールに使いやすいです。

たとえば、日程調整、資料送付、見積もり確認、軽いお礼などです。

もう少し慎重なメールの場合は、条件を足します。

```
このメールは重要な取引先への返信です。
失礼がないように、少し丁寧めにしてください。
ただし、過剰にへりくだる表現は避けてください。
```

逆に、社内向けならこうです。

```
このメールは社内向けです。
簡潔で、要点がすぐわかる文章にしてください。
必要以上に丁寧にしなくて大丈夫です。
```

同じメール返信でも、相手によって温度感は変わります。

Claudeには、相手との関係を一言添えるだけで、かなり出力が変わります。

## うまく動かないときのチェックリスト

結論から言うと、うまくいかないときは「情報不足」か「条件の出しすぎ」が原因になりやすいです。

私がよく見るポイントは次の4つです。

### 1. サンプルが短すぎないか

「了解しました。よろしくお願いします。」だけでは、文体の特徴がほとんど伝わりません。

できれば、3〜8行くらいの返信文を3つ渡すと使いやすいです。

短文しかない場合は、サンプル数を5つくらいに増やすのもありです。

### 2. 相手との関係を書いているか

メール返信は、相手との距離感でかなり変わります。

取引先、既存顧客、初回問い合わせ、社内メンバー。  
このあたりを最初に書くだけで、文章のズレが減ります。

### 3. 何を伝えたいかを先に決めているか

Claudeは文章を整えるのは得意ですが、こちらの判断までは決められません。

たとえば、断るのか、保留するのか、承諾するのか。  
ここは人間側で決めてから渡した方がいいです。

### 4. 一度で完成させようとしていないか

最初の出力をそのまま使うより、1回だけ直す方が自然になります。

たとえば、次のように頼みます。

```
少し丁寧すぎるので、もう少し自然な業務メールにしてください。
文章量は今の8割くらいにしてください。
```

この一往復を入れるだけで、かなり使える文章になります。

## メール返信を任せたら次にやること

メール返信の下書きが速くなると、次にやるべきことは「判断の時間を守ること」です。

AIに任せたいのは、文章の整形や言い回しです。

一方で、次のような判断は自分で持っておいた方がいいです。

* 受けるか断るか
* いつまでに対応できるか
* どこまで約束してよいか
* 金額や条件に触れるか
* 相手に確認すべきことは何か

ここまでClaudeに丸投げすると、あとで自分が困ることがあります。

私の場合は、先に箇条書きで判断だけ書きます。

```
返信方針:
- 依頼は受ける
- 納期は来週金曜まで
- 追加費用が発生する可能性は伝える
- 詳細資料を送ってほしい
```

この方針を渡してから返信文にしてもらうと、かなりブレにくいです。

## まとめ

Claudeにメール返信を任せるなら、最初に自分の文体サンプルを渡すのがおすすめです。

今回のポイントは次の通りです。

* 過去の返信文を3つ渡す
* 個人情報や社名は置き換える
* 先に文体を要約させる
* 相手との関係を書く
* 判断は自分で決めてから、文章化を任せる

メール返信は、毎日少しずつ時間を持っていく仕事です。

1通あたり数分でも短くなると、1週間ではかなり違います。

派手な使い方ではありませんが、こういう小さな時短の積み重ねが、いちばん効く感じです。
