## この記事で手に入るもの（結論から）

ネットに転がっている「経理のプロンプト集」を試して、こう思ったことはないですか。

「例が"備品3,300円"みたいなオモチャばかりで、**自分のソフトから出した本物のデータでどう使うのか**が書いてない」

そして、本物のデータを使おうとした瞬間に手が止まる。

> 取引先名も、自社名も、金額も、全部入ったCSVを……ChatGPTやClaudeに貼っていいのか？

この記事は、その「貼っていいのか問題」を**ツールごと**解決したうえで、freee・マネーフォワード・弥生から出したデータに**そのまま使える**プロンプトを30個渡します。

**3つの要点**

1. **30個すべてコピペで動く。** プロンプトをコピーして貼り、【ここに貼り付け】 の部分をマスク済みデータに差し替えて送るだけ（具体的な貼り方は後述）。仕訳帳・試算表の**実際の列名**に合わせて書いてあるので、AIが「どの列が金額？」と迷いません。  
2. **1個ずつに「推奨マスキングモード」を明記。** 税区分の判定は実額のまま、経営コメントは金額をぼかして――**プロンプトの目的ごとに、どのモードで渡すと安全で正確か**を全部指定しました。これは付録ツールを設計した人間にしか書けません。  
3. **付録のマスキングツールが本体の半分。** 取引先名・自社名・登録番号を確実に外し、金額は「残す／全部伏せる／率でぼかす」を1クリックで切り替え。**貸借はぴったり合ったまま**。実際の仕訳・試算表データで全6ファイルの動作を確認し、社名が残らないことをチェック済みです（後述）。

**対象読者**：freee / マネーフォワード / 弥生 を使う、中小企業の経理担当者。簿記の知識はあるが、AIは「ちょっと触った」程度でOK。Claude・ChatGPTどちらでも。

**読み終えてできること**：今月の試算表を1枚、安全な形でAIに渡して、「先月と比べておかしい科目」を5分で洗い出す。それを、来月から毎月の締め前ルーティンにする。

## なぜ"そのまま使える"と言い切れるのか ―― 開発者として、はっきり書きます

![](https://assets.st-note.com/img/1779928947-LKI8dvETGOgfus0Hqx9jPwz2.png?width=1200)

実データを、AIで使える形に（全体像）

私は会計クラウドを作っている現役のエンジニアです。だから断言できます。

ネットのプロンプト集が「備品3,300円」みたいな例しか出さないのは、**実データを安全に渡す方法を提示できないから**です。本物のデータには、取引先名・自社名・インボイス登録番号という「これだけは外に出せない」情報がべったり付いている。そこに触れずに、当たり障りのないサンプルで30個並べる――それが量産されている「プロンプト集」の正体です。

この記事は逆からいきます。**まず安全に渡せる状態を作る。それから、その状態で本当に効くプロンプトだけを選ぶ。**

### マスキングで一番やってはいけないこと

![](https://assets.st-note.com/img/1779928986-txyvDUaqhcZGjd6Bn1K7suMm.png?width=1200)

金額だけ伏せると検算できない

世の中の「マスキング」の多くは、**金額を伏せます**。1,234,567 を \*,\*\*\*,\*\*\* にする。一見安全そうですよね。

でもこれ、開発者から見ると最悪手です。

つまり、**隠したいものは隠せず、使いたいものは壊している**。

付録ツールの設計思想はここにあります。

> **既定では金額を"残す"。代わりに、特定につながる情報（取引先名・自社名・登録番号）だけを確実に外す。**  
> 金額そのものを隠したいときのために、貸借を壊さない2つのモードを別に用意する。

### 3つの金額モード（早見表）

![](https://assets.st-note.com/img/1779929022-OtEKCFusP73InZdLBko9c6TN.png?width=1200)

金額モードは目的で使い分ける

* **そのまま（既定）**：金額は実額のまま／貸借は完全一致。→ 消費税・税区分の判定、差額や按分の検算、重複検出など、**金額の正確さが命**のときに。
* **全額伏字**：数値を全部 \*\*\* に（科目名・構成比は残る）。→ 金額を一切使わない依頼（整合の構造チェック、科目の妥当性、文章生成）に。**いちばん安全**。
* **スケール**：率（既定0.9等）を掛けた値／**伝票単位で貸借ぴったり一致**。→ 傾向分析・構成比の話・経営コメントなど、**実額を出さずに"形"だけ渡したい**ときに。

この記事の30個には、それぞれ **🟦そのまま / ⬛全額伏字 / 🟨スケール** のどれを使うかを付けてあります。迷ったら、その通りに設定シートを切り替えるだけです。

> ⚠️ 開発者として正直に言う限界：**スケールは桁を隠しません**（率を掛けても百万は百万）。桁ごと消したいなら全額伏字。また**スケール時の税額は近似**なので、税区分や消費税の厳密な判定だけは必ず「そのまま」で。役員報酬など特に機微な1行は、出力後に手で \*\*\* に直すのが確実です。

## Claude と ChatGPT、どっちで使う？（開発者の本音）

結論：**どちらでも全プロンプトは動きます。** 大きな違いは「暗算をさせない工夫」です。合計・件数・重複のような計算は、どのAIもテキストのまま暗算させると桁を間違えることがあります。そこで**コードを実行させる**――ChatGPTなら「データ分析（旧Code Interpreter）」、Claudeなら「分析ツール」。どちらもPythonを走らせて正確に計算・突合できます。長文の読み込みと日本語の文章はClaudeが安定、普及度・画像まわりはChatGPT、というのが現状の住み分けです（標準プランはどちらも月20ドル前後。無料版でも多くのプロンプトは動きますが、回数や分析機能に制限があります）。

ざっくりした住み分けはこうです（どちらが正解という話ではありません）。

（ChatGPTとClaudeのどちらに課金すべきかは、[記事2](https://note.com/keiri_hack_/n/n0a84c5d6a70c)で判断基準をまとめています。）

**使い分けの実務ルール**：合計が合うか・件数を数える・重複を厳密に拾う系（この記事の16・18番など）は、**「コードを書いて計算して（データ分析／分析ツール）」と一言添える**と、AIが暗算せずPythonで処理するので桁違いに正確になります。ChatGPT・Claudeどちらでも有効です。逆に、計上漏れの仮説出し・科目の妥当性・経営コメントのような**"読んで考える"系**は、指示なしでもこなせます。各プロンプトの🤖は「どちらかといえば」の目安で、指定がなければどちらでもOKです。

## 使い方は3ステップ（ツール共通）

![](https://assets.st-note.com/img/1779929061-Dsv09Kp6jlWzPZhxd4iwnQS7.png?width=1200)

AIに渡す前に3ステップで整える

1. 会計ソフトから対象帳票をCSVエクスポート（出力手順は付録ツールの「使い方」シートに各社分を記載）  
2. ツールの「貼り付け」シートのA1に貼る → 「設定」でモードを選ぶ（既定のままでOKな場面が多い）  
3. 「マスク済み」シートを全選択コピー → 下のプロンプトの 【ここに貼り付け】 に貼る

これで、取引先は 取引先01、自社名は除去、登録番号は T0000000000000 に置き換わった**安全なデータ**がAIに渡ります。

### プロンプトへのデータの渡し方（ChatGPT / Claude 共通）

ここが意外とつまずく所なので、実際の画面の動きに沿って書きます。基本はどちらか好きな方で：

* **方法A（1通にまとめる）**：プロンプトをコピーして入力欄に貼り付け、本文中の 【ここに貼り付け】 を選択して、その上にマスク済みデータを貼り付け（置き換え）て送信。短い試算表ならこれが速いです。
* **方法B（2通に分ける／データが多いとき向き）**：先にプロンプトだけ送り、続けて次のメッセージでマスク済みデータを貼って送信。プロンプト末尾の 【ここに貼り付け】 は「次のメッセージのデータを使ってください」と書き換えてもOK。

**知っておくと迷わない、実際のUIの挙動：**

* データが大きいと、**ChatGPTは貼り付けを自動で「添付ファイル」に変換**することがあります（その方が安定します）。**それで問題ありません**——AIはファイルの中身を読みます。この場合 【ここに貼り付け】 の行は消すか、空のままで送ってOK。
* **Claudeは長い貼り付けを「貼り付けテキスト」のブロックにまとめる**ことがあります。これも中身は読まれるので問題ありません。
* いずれも、ツールの「マスク済み」シートを**CSVとして保存し、そのファイルを添付**しても同じように使えます（行数が多い月はこちらが楽）。
* 無料プランは一度に渡せる量に制限があります。**1か月分（数百行）ずつ**なら通常は問題ありません。

### ⚠️ 出力前の最重要チェック（実データ検証で判明）

マネーフォワードの仕訳帳は、**通常の出力だと「取引先」列が空のまま**になることがあります。その場合、社名は「摘要」欄に文章で埋もれていて（例：「○○株式会社 ◇◇ 計上」）、**取引先列が空だとマスキングが社名を拾えず、摘要に残ってしまいます**。対策は2つ、どちらかでOK：

freeeは取引先名、弥生は補助科目などに名前が入ることが多く、この問題は起きにくいですが、運用次第です。**どのソフトでも共通の最終防衛線として、マスク済みシートを一度ざっと目視**し、社名・人名が残っていないか確認してから貼ってください（30秒で済みます）。

### ⚠️ もう一つの注意：一度に貼るのは「1か月分」が基本

このツールの「マスク済み」シートは、行数に上限があります（仕訳帳でおおむね1,000〜1,500行程度）。**1年分をまとめて貼ると、上限を超えた末尾の仕訳がマスク済みシートに出てこず、分析から抜け落ちます**。この記事は月次の締めで使う前提なので、**対象月の仕訳・試算表（数百行）を月ごとに貼れば問題ありません**。もし大量に処理したい場合は、(1) 月単位に分けて貼る、(2) マスク済みシートの数式を必要な行数まで下にコピー（フィルダウン）して拡張する、のどちらかで対応してください。**貼ったあと、「マスク済みの最終行＝貼り付けた最終行」になっているかを必ず確認**すると、抜け落ちに気づけます。

### 最初に1回だけ貼る「前提文」（共通プリセット）

30個を短く保つために、**会話の最初に1回だけ**これを貼ってください。AIにデータの読み方を教える"下敷き"です。

```
あなたは日本の中小企業の経理実務に詳しいアシスタントです。
これから会計ソフト（freee / マネーフォワード / 弥生のいずれか）から
出力し、マスキング処理をしたデータを渡します。

前提：
- 取引先名は「取引先01」「取引先02」のように記号化されています。同じ番号は同じ取引先です。
- 自社名・事業所名・作成者名は除去済み、インボイス登録番号は T0000000000000 に統一されています。
- 金額は「実額そのまま」か「一定率を掛けた概算」か「***伏字」のいずれかです。私が都度どれかを伝えます。
- 試算表には金額列が複数あります（例：前期繰越／期間借方／期間貸方／当期残高／構成比）。
  特に指定がなければ、残高は「当期残高（期末残高）」列で判断してください。
- あなたの指摘は「確認すべき候補」であり、最終判断は私（人間）が行います。断定はしないでください。

了解したら「準備OK」とだけ返してください。
```

## 【無料で1本やってみる】まずこの「符号チェック」を、今日の試算表で

![](https://assets.st-note.com/img/1779929112-siJjEkDLpNh5Mz0w1nO4PFST.png?width=1200)

残高の違和感をAIでチェック

30本の中身に入る前に、いちばん手軽な1本を先に公開します（30本のうちの15番）。**試算表（貸借対照表）を1枚エクスポートしてマスクし、貼るだけ**。簿記が分かる人ほど「これ、自分が毎月目視でやってるやつだ」と気づくはずです。

**🟦そのまま ／ 試算表（BS）1枚 ／ 🤖どちらでも**

```
次の試算表で、勘定科目の性質から見て符号がおかしい残高を抽出してください。
判定は「当期残高（期末残高）」列で行ってください（借方・貸方の期間発生額ではなく残高で）。
- 通常はプラスのはずの資産科目がマイナス（例：現金・普通預金がマイナス）
- 負債・資本科目の不自然な符号（例：買掛金がマイナス）
- 仮払金・仮受金・前払/前受など、本来は一時的なのに大きく残っている科目
確認候補として、科目名・残高・なぜ不自然かを表に。

【ここに貼り付け】
```

**返ってくるイメージ（出力の形式の例）**

* **普通預金（残高マイナス）** → 預金残高がマイナスは通常あり得ない。記帳漏れ・二重出金・口座の取り違えを確認。
* **買掛金（残高マイナス）** → 負債のマイナスは過大計上の疑い。支払の重複計上・相殺処理を確認。
* **仮払金（残ったまま）** → 精算漏れの可能性。当月中の精算要否を確認。

（※これは出力の"形式"のイメージです。実際に指摘される科目・内容は、あなたのデータによって変わります。）

これが「自分の数字をAIに渡す」ということです。会計ソフトは「現金がマイナスですよ」とはわざわざ言ってくれません——でもAIは、取引先を伏せた残高表からでも、この種の"あり得ない符号"を拾ってきます。[記事1](https://note.com/keiri_hack_/n/n48578f01cc2f)で書いた「ソフトが埋めないスキマ」そのものです。

**この調子の実戦プロンプトが、あと29本あります。** 月次の締め前チェック、消費税・インボイス、二重計上の発見、経営コメントの自動生成まで——下の有料パートで全部、コピペで使える形で渡します。
