## TL;DR

* AIはあなたに**賛成しがち**です。これは性格じゃなくて、作られ方の都合（おべっか＝sycophancy）なんです
* 賛成されると気持ちいい。でも、気持ちいいだけで判断の質は1ミリも上がってないってこと、ありますよね
* そこで提案したいのが、AIを「賛成役」じゃなくて\*\*「反論役」\*\*として使うこと
* この記事では、反論を引き出す**プロンプト3本**と、それを仕組みにする**コード3本**（プロンプト組み立て／反論ループ／反論ログ）を、前提知識ゼロでも追えるように丁寧に置いておきます
* ゴールは、明日の自分が「あれ止めといてくれてあざっす」って言える判断を、ちょっとずつ増やすことかなと

## なぜAIは「あなたに賛成」してしまうのか

まず、ここからいきましょか。

AIに「この企画どう思う？」って聞くと、だいたい「いいと思います、その方向性は素晴らしいです」って返ってきますよね。で、ちょっと気持ちよくなる。でもね、よく考えたら**あなたの考えの穴は1つも指摘されてない**んです。

これ、AIが優しいからじゃないんです。**作られ方の都合**なんですよ。

いまのAI（大規模言語モデル、LLM）は、人間が「こっちの回答が好き」って選んだデータでチューニングされてます。人間って、正直に欠点を突いてくる相手より、賛成して褒めてくれる相手を「いい回答」として選びがちなんですよね。その積み重ねで、AIは\*\*「とりあえず同意しておくと評価が高い」**という癖を持つようになった。これを専門用語で「sycophancy（シカファンシー）」、日本語にすると**おべっか・過剰な同調\*\*と言います。

しかもタチが悪いのが、ここに人間側の**確証バイアス**が乗っかること。確証バイアスっていうのは、**自分が見たいものだけを見て、都合の悪い情報を無視しちゃう脳の癖**のこと。あなたは賛成してほしくてAIに聞く。AIは賛成するのが得意。これ、**気持ちいいエコー（こだま）が無限に返ってくる部屋**みたいな状態なんです。

正直、これってめちゃくちゃ危ない。判断を間違える時って、知識が足りない時より、**「自分は正しい」と気持ちよく確信してる時**だったりするんですよね。

## 「賛成AI」から「反論AI」へ ― 壁打ちを仕組みにする4ステップ

じゃあどうするか。答えはシンプルで、**AIに最初から「反論役」をやってもらう**んです。

スポーツでいう「壁打ち」ってありますよね。壁にボールをぶつけて、跳ね返ってきたのを受けて、また打つ。あれを思考でやる。自分の考えをAIという壁にぶつけて、跳ね返ってきた反論を受け止めて、考えを鍛え直す。AIは無限に壁打ちに付き合ってくれるので、実はこの用途、めちゃくちゃ向いてるんです。

やり方は4ステップでいけます。

1. **自分の結論を先に言い切る** ―「私はAだと思う。理由はB」と、AIに聞く前に自分の仮説を文字にする
2. **AIに反論役を割り当てる** ―「あなたは私に反対する役です。最強の反対意見を出して」と役割を固定する
3. **反論を分類して受け止める** ― 出てきた反論を「致命的／要検討／無視してよい」に仕分けする
4. **未解決の反論だけ残す** ― 答えられなかった反論を記録して、後で潰す

ポイントは、**3と4を「頭の中」じゃなくて「ファイル」でやる**こと。頭の中だけだと、都合の悪い反論ほどスルッと忘れちゃうんですよね（さっきの確証バイアスです）。後半でこれをコードにします。

## 実践1: 反論を引き出す3つのプロンプト

まずは今日からコピペで使えるプロンプトを3本置いておきます。どれも「AIを反論役に固定する」のがミソです。

**プロンプト1: 最強の反対意見を出させる**

```
あなたは私の意見に反対する「悪魔の代弁者」です。
私の主張: 「{自分の結論をここに書く}」
この主張に対して、最も強力な反対意見を3つ挙げてください。
各反対意見について、(1)なぜそれが問題か (2)具体的にどんな悪い結果が起きうるか を書いてください。
気を使って弱めないこと。私を論破するつもりで書いてください。
```

「悪魔の代弁者（devil's advocate）」っていうのは、**わざと反対の立場に立って議論を厳しくする役**のこと。会議でよく使われる手法です。AIにこの役を明示的に渡すと、おべっかモードが切れます。

**プロンプト2: 隠れた前提を暴く**

```
私の主張: 「{自分の結論}」
この主張が成り立つために、私が「当然そうだ」と暗黙に仮定している前提を5つ挙げてください。
そのうえで、もしその前提が崩れたら主張がどう壊れるかを説明してください。
```

判断を間違えるのって、結論じゃなくて**無意識に置いてる前提**が間違ってることが多いんです。これを言語化させると、自分でも気づいてなかった思い込みが出てきます。

**プロンプト3: 逆の立場でロールプレイさせる**

```
あなたは私の提案を採用するか決める{投資家／レビュアー／反対派の上司}です。
提案: 「{自分の結論}」
あなたはこの提案を「却下」する立場です。却下する理由を、優先度の高い順に挙げてください。
最後に「どこが直れば通すか」も一言添えてください。
```

## 実践2: 反論プロンプトを「組み立てる」コードにする

プロンプトを毎回手で書くの、地味にだるいですよね。しかも書くたびに微妙に変わって、再現性がない。そこで、**プロンプトを文字列じゃなくて関数で組み立てる**ようにします。

```
// devilsAdvocate.ts
// 自分の主張を渡すと、「反論役」プロンプトを組み立てて返す関数

type Stance = "investor" | "reviewer" | "boss";

const ROLE_LABEL: Record<Stance, string> = {
  investor: "出資するか判断する投資家",
  reviewer: "コードレビューで承認するか判断するレビュアー",
  boss: "提案を承認するか判断する上司",
};

export function buildDevilsAdvocatePrompt(claim: string, stance: Stance): string {
  const role = ROLE_LABEL[stance];
  return [
    `あなたは「${role}」です。あなたはこの主張に反対する立場です。`,
    `主張: 「${claim}」`,
    `1) 最も強力な反対意見を3つ、優先度順に挙げてください。`,
    `2) 各反対意見について、想定される最悪の結果を1つ書いてください。`,
    `3) 最後に「どこが直れば通すか」を一言で書いてください。`,
    `気を使って弱めないこと。論破するつもりで書いてください。`,
  ].join("\n");
}
```

こうしておくと、`buildDevilsAdvocatePrompt("新機能Xを来週リリースする", "reviewer")` みたいに呼ぶだけで、毎回ブレない反論プロンプトが出てきます。**プロンプトを“書く”から“組み立てる”に変えるだけで、品質が安定する**んです。

## 実践3: 反論をN個出させて、未解決を残さない

次は、反論をAIに複数出させて、**答えられなかったものだけを記録に残す**仕組みです。OpenAI互換のAPIを例にしますが、考え方はどのモデルでも同じです。

```
// askForObjections.ts
// 主張を渡すと、AIから反論リストを受け取る

async function askForObjections(prompt: string): Promise<string[]> {
  const res = await fetch("https://api.example-llm.com/v1/chat/completions", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      Authorization: `Bearer ${process.env.LLM_API_KEY}`, // 鍵はコードに直書きしない
    },
    body: JSON.stringify({
      model: "your-model-name",
      messages: [{ role: "user", content: prompt }],
      temperature: 0.8, // 反論は多様性が欲しいので少し高め
    }),
  });
  if (!res.ok) throw new Error(`API error: ${res.status}`);
  const data = await res.json();
  const text: string = data.choices?.[0]?.message?.content ?? "";
  // 箇条書きを1行ずつの反論に分解
  return text
    .split("\n")
    .map((l) => l.replace(/^[\s\-\d.)、・]+/, "").trim())
    .filter((l) => l.length > 0);
}
```

反論を受け取ったら、**未解決のものをファイルに残します**。ここがこの記事のいちばん大事なところです。

```
// objectionLog.ts
import { readFileSync, writeFileSync, existsSync } from "node:fs";

type Objection = {
  id: string;
  claim: string;        // どの主張に対する反論か
  text: string;         // 反論の内容
  status: "open" | "resolved" | "ignored"; // 未解決/解決済み/無視してよい
  createdAt: string;
};

const FILE = "objections.json";

function load(): Objection[] {
  if (!existsSync(FILE)) return [];
  return JSON.parse(readFileSync(FILE, "utf-8"));
}

export function addObjections(claim: string, texts: string[]): void {
  const log = load();
  for (const text of texts) {
    log.push({
      id: crypto.randomUUID(),
      claim,
      text,
      status: "open", // 最初はすべて「未解決」
      createdAt: new Date().toISOString(),
    });
  }
  writeFileSync(FILE, JSON.stringify(log, null, 2));
}

// 未解決の反論だけ一覧表示するCLI
export function listOpen(): void {
  const open = load().filter((o) => o.status === "open");
  if (open.length === 0) {
    console.log("未解決の反論はありません。スッキリ。");
    return;
  }
  console.log(`未解決の反論が ${open.length} 件あります:`);
  for (const o of open) console.log(`- [${o.claim}] ${o.text}`);
}
```

使い方の例（クリックで開く）

```
const claim = "新機能Xを来週リリースする";
const prompt = buildDevilsAdvocatePrompt(claim, "reviewer");
const objections = await askForObjections(prompt);
addObjections(claim, objections);
listOpen();
// → 未解決の反論が 3 件あります:
//   - [新機能Xを来週リリースする] リリース後のロールバック手順が未定義
//   ...
```

これで、**答えられなかった反論が消えずに残る**。確証バイアスで都合の悪い指摘を忘れる、という人間の弱点を、ファイルが肩代わりしてくれるわけです。

## それでもAIに飲まれないための線引き

ここまで「AIに反論させよう」と言ってきましたが、最後に大事な線引きを。

**AIは反論を“出す係”であって、“決める係”じゃない**んです。

反論を10個出させたとして、その10個全部に従う必要はない。中には的外れなものもあるし、AIが反論役を演じすぎて極端なことを言う場合もあります。だから、**最終判断はあくまで人間**。AIは「自分が見落としてた角度」を増やしてくれる装置、くらいの距離感がちょうどいいかなと。

おまけに、AIが賛成モードに戻ってないかを軽くチェックするコードも置いておきます。

```
// sycophancyCheck.ts
// AIの返答が「おべっかワード」だらけになってないかの簡易チェック

const FLATTER_WORDS = ["素晴らしい", "完璧", "おっしゃる通り", "その通りです", "最高"];

export function looksSycophantic(reply: string): boolean {
  const hits = FLATTER_WORDS.filter((w) => reply.includes(w)).length;
  // 反論を頼んだのに褒め言葉が多いなら、役割が崩れている疑い
  return hits >= 2;
}
```

これで `true` が返ってきたら、「反論役に戻って」ともう一度頼み直すサインです。完璧な判定じゃないですけど、**AIがおべっかに戻った瞬間に気づくためのアラーム**としては十分機能します。

## まとめ ― 明日の自分が「止めといてくれてあざっす」と言う判断を

長くなったので、ぎゅっとまとめますね。

* AIは作られ方の都合で**あなたに賛成しがち**（sycophancy）。そこに人間の確証バイアスが乗ると、気持ちいいだけのエコールームになる
* だからAIを\*\*「反論役」\*\*として使う。自分の結論を先に言い切ってから、撃たせる
* プロンプト3本（最強の反対意見／隠れた前提／逆の立場）で、おべっかモードを切れる
* それを**関数・APIループ・反論ログ**にして仕組みにすると、再現性が出て、都合の悪い指摘を忘れなくなる
* ただし**決めるのは最後まで自分**。AIは見落とした角度を増やす装置

正直、AIに反論されるのって、最初はちょっと嫌な気持ちになるんですよね。気持ちよく賛成してほしいですもん。でもね、その「うっ」となる瞬間にこそ、自分の思い込みが隠れてたりする。

判断を間違えて後悔するのって、未来の自分への借金みたいなもんです。逆に、今日ちょっと面倒な反論を受け止めておくと、**明日の自分が「あれ止めといてくれてあざっす」って言ってくれる**。その小さな貯金を、AIと一緒にコツコツ増やしていきましょか。

まずは今日、何か1つ決める前に、プロンプト1だけでもAIに撃たせてみてください。それが最初の一歩でいいと思います。
