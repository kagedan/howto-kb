私たちは大企業向けにAIの定着化支援を行っている株式会社 Low Code です。

今回、[Microsoft Agent Hackathon 2026](https://zenn.dev/hackathons/microsoft-agent-hackathon-2026) に参加するにあたり、現場で積み上げてきた知見を公開します。

「Copilot Studio でうまくいかない」という声を、現場で何度も聞いてきました。多くのケースで、原因は技術的な限界ではなく**設計判断の問題**でした。

この記事では、その核心にある **2つの判断ポイント** をお伝えします。

## 大前提：Copilot Studio とは

![Copilot Studio のホーム画面](https://res.cloudinary.com/zenn/image/fetch/s--trq2WDWk--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_1200/https://learn.microsoft.com/microsoft-copilot-studio/media/fundamentals-what-is-copilot-studio/home-page.png?_a=BACAGSGT)

Copilot Studio は、**ノーコード・ローコードで AI エージェントを作れる Microsoft のクラウドサービス**です。

プログラミングの専門知識がなくても、チャットで社内の質問に答えてくれるAIや、申請フローを自動化するエージェントを誰でも作れます。

詳しくは公式ドキュメント「[Microsoft Copilot Studio の概要](https://learn.microsoft.com/ja-jp/microsoft-copilot-studio/fundamentals-what-is-copilot-studio)」をご覧ください。

## Copilot Studio の会話には「2つの作り方」がある

まず前提として、Copilot Studio には会話を設計する方法が2種類あります。

### トピック（Topic） ── 流れを自分で組む

> 「こう聞かれたら、こう答える」を自分で組み立てる。フローチャートを作る感覚。

| 特徴 | 内容 |
| --- | --- |
| 会話制御 | 「金額は？」「日付は？」と順番に確実に聞ける |
| 予測可能性 | 答えが予測しやすく、制御しやすい |
| **向く業務** | 経費精算の申請、上長への承認依頼、ステップを踏む業務処理 |

### インストラクション（Instructions） ── AIに自由に答えさせる

> 「資料を読んでおいてくれ、あとは自由に答えて」をAIに任せる。ChatGPTに近い感覚。

| 特徴 | 内容 |
| --- | --- |
| 柔軟性 | 言い回しが違っても、想定外の質問にも対応 |
| 予測可能性 | 答えは予測しにくい（それがトレードオフ） |
| **向く業務** | 社内ルールへの自由質問、FAQ、なんでも調べたい時 |

## うまくいかないパターン① ── 「よしなにやって」では動かない

### ありがちな指示の書き方

一見まとまっているように見えますが、これだと3つの問題が起きます。

### ✅ 解決策：指示を「6つの役割」に分けて書く

| # | 役割 | 書く内容 | 例 |
| --- | --- | --- | --- |
| ① | ペルソナ | AIの立場・役割の定義 | 「あなたは社内ヘルプデスクです」 |
| ② | やってほしいこと | AIにやってほしいことのリスト | 「ナレッジ **/社内規定・申請手順PDF**について回答する」「ツール **/メール送信** でメールを送信する」 |
| ③ | やってはいけないこと | やってはいけないことのリスト | 「個人情報を含む回答は絶対にしない」 |
| ④ | 優先順位 | 複数制約がある時の優先度 | 「安全性 ＞ 正確性 ＞ 丁寧さ」 |
| ⑤ | 答え方のフォーマット | 出力の形式・構造 | 「結論→理由→参照先の順で答える」 |
| ⑥ | 例外処理 | 想定外の質問への対応方針 | 「範囲外は『担当部署にご確認を』と返す」 |

**1段落でまとめず、6つの見出しに分けて書くだけで、AIの安定度が大きく上がります。**

## うまくいかないパターン② ── 「全部AIに任せる」と現場がまわらない

AIが自由に答えてくれる便利さゆえに、申請・承認フローまで全部任せてしまうケースが多発しています。

### 会話の「種類」で作り方を分ける

| 会話の種類 | 使う機能 | 代表的な業務 | 理由 |
| --- | --- | --- | --- |
| 決まった流れ | **トピック** | 経費精算・承認・エスカレーション | 必要な情報を順番に確実に取得したい |
| 自由な対話 | **インストラクション** | 社内FAQ・ルール質問 | 想定外の言い回しにも対応したい |
| 両方の組み合わせ | **トピック + インストラクション** | 申請しながら制度を質問 | 両方を1つの会話で完結させたい |
