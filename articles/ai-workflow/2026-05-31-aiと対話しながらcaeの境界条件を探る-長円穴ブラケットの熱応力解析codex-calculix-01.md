# AIと対話しながらCAEの境界条件を探る

## はじめに：境界条件は一発で決まらない

CAE解析では、形状や材料物性よりも、境界条件の決め方で悩むことがよくあります。

たとえば「ボルトで固定されている」と言っても、FEM上でそれをどう表現するかは一意に決まりません。

* 穴周りを完全固定するのか。
* 完全に滑ってよいのか。
* あるいは、ボルト締付け摩擦によって滑るかどうかが決まると考えるのか。

実務では、このあたりを最初からきれいに決められるわけではありません。まず仮説を立てて、簡単な解析や概算で影響を見て、必要ならモデルを詳細化していきます。

## 問題設定

今回は、真ん中にアーチ形状を持つブラケットを題材に、Codexと対話しながら境界条件を探ってみました。  
![](https://static.zenn.studio/user-upload/999471e695be-20260530.png)

このブラケットの環境温度が上がることで膨張しひずみます。ブラケットは**ボルトで固定**されており、右側の穴で位置決め、左側の穴で公差や熱膨張を吸収という意図が感じられます。

ただし**ボルトで固定**というのは文字通りの **固定（つまりFEMにおける全自由度拘束）** とは言い切れず、どう扱うか迷っている状態です。

## 使用するもの

* Codex App + GPT5.5 （AIエージェント）
* CalculiX （CAEソルバー）
* Gmsh （メッシュ作成用）
* meshio （メッシュ変換用）
* build123d （ジオメトリ作成用）
* FreeCAD （手作業によるジオメトリ作成用）
* ccx2paraview （CalculiXの出力データをVTK/VTUに変換）
* PyVista （3Dデータ表示用）

プロジェクトディレクトリにて`AGENTS.md`と[CAEワークフロー](https://zenn.dev/ms_ai/articles/6ed4f9e640ed26)も設定しています。

Codexに解析計画を作ってもらいます。

```

材料はSS400。
この部材の環境温度が22度から120度に上昇した時に生じる変形と応力を評価したい。

```

## 出来上がった解析計画 - Codexが提示した3つの境界条件

Codexは次の3ケースを提案してきました。

* case\_free: 自由熱膨張の検証
* case\_pin\_slot: 長円穴の長軸方向に滑りを許容
* case\_locked: 締付け摩擦などで滑らない拘束上限  
  ![](https://static.zenn.studio/user-upload/33e5d6943a9f-20260530.png)

Codexは「ボルトで固定」を単純に「穴周りを完全固定」とは扱わず、長円穴の意味を考慮して、滑る場合と滑らない場合を分けてきました。

構造解析では、こういう境界条件の置き方で悩むことがよくあります。長円穴は長手方向に位置調整や熱膨張を逃がす意図があることが多いです。一方で、実際にはボルトで締め付けているので、摩擦によって滑らない可能性もあります。つまり、同じ「ボルト固定」でも、FEM上では複数の理想化があり得ます。

### 「ボルト固定」と3ケースの対応を確認する

ただし、Codexの説明はざっくりしすぎていて、ユーザーが指定した「ボルトで固定される」と、3ケースがどのように整合しているかわからないので、その説明を求めるとともに、確認事項に回答しておきます。

プロンプト

```
自分が指定した「ボルトで固定されるブラケット」という説明と、ここで提案されている3ケースがどのように対応しているかを説明して。
質問については以下の通り回答します。
1.今は取り付け相手を省略してます。取り付け相手は別々の部品です。
2.その通りです。
3.取り付け相手も同じ環境温度に置かれますが、熱膨張は無視してよいです。
4.まずはブラケット単体で評価して、そのあとでボルト締め付けは考えようと思います。
```

この質問を受けて、3ケースと実機状態の対応を整理してくれました。

![](https://static.zenn.studio/user-upload/7e829347cbbf-20260530.png)

さらに説明を求めてみます。

プロンプト

```
この3ケースを解析して比較することで、最終的に何が分かるのかを説明してください。
```

そうすると、ボルト固定との対応および用途を整理してくれました。

つまり、最初の解析は「どれが正解か」を決めるためではなく、実機がどちらに近いかを後で判断するための確認という位置付けです。

![](https://static.zenn.studio/user-upload/bf3596f5f495-20260530.png)  
このようにAIと対話することで、AIが出した解析ケースをそのまま受け入れるのではなく、「それは自分が言った実機条件とどう対応しているのか」を確認することができます。  
これで解析計画の意味がかなり明確になりました。

なお、解析計画を作る際にベテランCAEエンジニア視点でのレビューを出すようにしているので、CAE視点での解説をみることもできます。

## 3ケースの熱膨張解析を実行する

解析計画の意味が整理できたので、まずはブラケット単体で3ケースを解析しました。

ここでは、まだボルトや座金、相手方部品はモデル化していません。  
目的は、詳細な締結部応力を求めることではなく、長円穴が滑る場合と滑らない場合で、ブラケット本体の変形や応力がどの程度変わるかを見ることです。

レポートは次の通り。  
![](https://static.zenn.studio/user-upload/7f3158beaa0a-20260530.png)

変形図を見ると、長円穴を固定するかどうかで大きく変わる様子が観察できます。  
![](https://static.zenn.studio/user-upload/050827916bf1-20260530.png)  
![](https://static.zenn.studio/user-upload/0566e4989627-20260530.png)

レポートの結論は次の通り。  
![](https://static.zenn.studio/user-upload/c7a932658863-20260530.png)

この結果から分かるのは、長円穴が滑れる場合は熱応力がほとんど出ないということです。長円穴側が丸穴側に対して約 0.229 mm 移動しています。これは自由膨張時の相対移動量とほぼ同じで、温度上昇による伸びをかなり逃がせることが分かります。

長円穴側を固定すると熱膨張が拘束され、ブラケット本体に約 80 MPa の応力が生じました。

この時点で、論点はかなりはっきりしました。次に確認すべきなのは、実機のボルト締付け状態では長円穴側は本当に滑るのか？という点です。

## 長円穴はあるが、ボルト締付け摩擦で滑るのか？

レポートでは「必要スライド量に対する穴余裕量を確認し、ボルト締付け軸力、座面摩擦、穴クリアランスを含む滑り開始条件を評価する」と書かれていました。

今回の形状からして長円穴の幾何学的な余裕は十分です。  
したがって次は、熱膨張で発生する力が、ボルト締付けによる摩擦保持力を超えるか、が焦点になります。ただ、まだボルト自体のCADモデルがないので、どうすればいいかも聞いてみます。

プロンプト

```
現在の膨張量だと長円穴の余裕量は十分です。次はボルト締め付けを考慮したいです。
長円側はM8ボルトで、固定穴はM12ボルトで固定している前提ですが、まだボルトはCADモデル化していません。
簡易的に評価するにはどうすればいいですか？別ケースで解析計画を作って。
なお、相手方部品は-Z側に存在し組付け面は平面と考えてよいです。
```

そうすると以下の計画を作ってくれました。  
![](https://static.zenn.studio/user-upload/f36d14c85bd2-20260530.png)

トルクが必要とのことなので回答します。

プロンプト

```
締付け条件
- M8長円穴側: 締付けトルク 22 N·m
- M12固定穴側: 締付けトルク 85 N·m

座金
- M8側: 丸座金外径 18 mm 相当
- M12側: 丸座金外径 26 mm 相当

摩擦係数 μ = 0.2 を想定します。M12側の穴クリアランスは無視してよいです。
```

Webの情報も参照して軸力を推定してくれました。  
![](https://static.zenn.studio/user-upload/704521e9335d-20260530.png)

よく見ると

> 先行解析の必要熱反力207.7 Nの約13.2倍なので、初期予測は長円穴側の固着

と書かれています。この時点で、実機の挙動はかなり固着側に近いと予測できます。

（補足）このような「手計算」は解析の妥当性見積や、無駄な解析の回避につながるため有効です。現実には手計算が面倒でやらないことも多いのですが、AIと対話すると簡単に出してくれるので便利です。

Codexに、結局固着の場合と同じなのか確認します。

プロンプト

```
初期予測は長円穴側の固着先行解析の必要熱反力207.7 Nの約13.2倍なので、初期予測は長円穴側の固着ということは、最初の解析の固着条件での結果とほぼ同じになると考えてよい？
```

Codexの回答は、概ねその理解でよい、というものでした。M8側の摩擦保持力が必要熱反力を十分に上回るため、温度上昇時に長円穴側はほぼ滑らず、ブラケット本体の熱応力は先行解析の `case_locked` に近づくと予測される、という整理です。  
![](https://static.zenn.studio/user-upload/f4e8dc7198b4-20260530.png)

ただし、完全に同一ではありません。`case_locked` は代表位置を拘束した簡略モデルですが、実際のボルト締付けでは、座面内で部分滑りが生じる可能性があります。また、座面面圧による局所応力や、支持面との接触による面外変形も加わります。

そこで、次の解析目的を少し変更しました。当初は、摩擦接触解析で「滑るかどうか」を見ようとしていました。しかし概算ですでに固着側と予測できたため、接触圧分布や部分滑りと固着領域の評価に切り替えました。

プロンプト

```
全体挙動がcase_lockedへ近づくことを確認しつつ、座面圧分布、部分滑り、本体応力を評価するための計画にしてほしい。
```

Codex回答  
![](https://static.zenn.studio/user-upload/e96ae784dcea-20260530.png)

このように、途中で解析目的が変わるのはCAEではよくあります。  
最初に立てた計画をそのまま押し通すのではなく、概算や初期結果を見て、次に見るべきものを変えていきます。今回はCodexとの対話で自然に進めることができました。

## 解析時間を見ながら、軽量な診断モデルへ切り替える

ボルト締付けを簡易的に考慮した接触解析へ進みます。モデル化の考え方は次の通りです。

* ブラケットの -Z 側に左右別々の支持ブロックを追加する
* ブラケット下面と支持ブロック上面に摩擦接触を設定する
* M8、M12座金の投影領域に、ボルト軸力相当の分布荷重を与える
* M12固定穴側で面内基準位置を決める
* M8長円穴側はX方向を直接拘束せず、摩擦接触で滑り量を決める

このモデルで、座面圧、摩擦抵抗、部分滑り、ブラケット本体応力を確認します。

### 最初の解析は時間がかかりすぎた・・・

最初は、もう少し細かい接触モデルで進めようとしていました。  
しかし、接触付き非線形解析は計算時間が読みにくく、最初の試行では時間がかかりすぎました。

しばらく待っても解析が終わりそうにないので、途中で作業を止めて、Codexに次のように依頼しました。

プロンプト

```
解析時間が長くなりすぎないように配慮して実行して。
```

この指示に対して、CodexはCAE専任者らしい方針に切り替えました。  
![](https://static.zenn.studio/user-upload/3846c2da7681-20260530.png)

単一ケースに絞り、粗い診断メッシュを使い、ソルバー実行時間に上限を設けて実行しました。この診断解析は、M8長円穴側が全体滑りするかどうかを短時間で確認するのが目的です。

![](https://static.zenn.studio/user-upload/23a43e7779a9-20260530.png)

結果は、やはり固着優勢でした。長円穴側はほとんど自由滑りしていないと判断できます。  
ただ、少しわかりづらいので、もう少し説明してもらいます。

![](https://static.zenn.studio/user-upload/6b827f992cfe-20260530.png)

ここで興味深かったのは、こちらが細かく「このメッシュサイズで」「タイムアウトを入れて」と指示したわけではないことです。単に「解析時間が長くなりすぎないように」と伝えただけで、Codexは計算コストを考慮して、粗い診断モデルへ切り替えました。

これは、実務CAEでよくやる進め方です。最初から詳細な非線形接触解析をきれいに回そうとすると、時間もかかりますし、収束しないこともあります。まず粗いモデルで大まかな挙動を確認し、徐々に詳細度を上げていきます。

このように、解析の途中で目的やモデルの詳細度を変えていけるのは、AIと対話しながら進めるCAE AIエージェントの価値あるところです。

## 接触解析で部分滑りを確認する

「短時間診断モデルでも固着寄り」と分かったので、部分滑り・接触圧・本体応力を確認する流れにします。

ボルトや座金のCADモデルは作成していないので、先の計画通り、M8、M12の座金投影領域にボルト軸力相当の分布荷重を与え、ブラケット下面と支持ブロック上面の摩擦接触で滑り量を評価します。詳細評価の結果は以下の通り。

![](https://static.zenn.studio/user-upload/6388d74cec20-20260530.png)

事前の予想通り、摩擦接触を入れて解析すると固着に近い結果になりましたが、少しは滑るという状態のようです。

## 解析結果を設計判断に翻訳する

CAE技術者であれば、この結果を見て

> これは部分滑りはあるけど、全体滑りではないな。  
> 本体応力は固着側で扱えばよさそうだな。

と読めます。しかし、設計で本当に知りたいのは、数値そのものではありません。

今回で言えば、

> 長円穴は熱膨張逃がしとして意味があるのか？  
> それとも固着側として評価すべきなのか？  
> 部分滑りが出ているなら、それをどう解釈すればよいのか？

といった「物理量から何が言えるか？」を知ることが重要です。

CAEに慣れている人なら、解析結果の数値を見ながら自分の中で翻訳ができますが、接触解析の知識がないとすぐには読み取るのは難しいと思います。そこで、Codexにそのまま聞いてみます。

プロンプト

```
つまり、完全に滑ってもいないし、完全に固着してもいないということね。
この結果から言えることは？
```

Codexの回答は以下の通り。  
解析結果から熱膨張逃がしとしては機能しないと解説してくれました。

> 接触面の一部では滑っていますが、ボルト座面全体が移動する「全体滑り」には至っていません。  
> 長円穴は幾何学的な余裕を持っていますが、熱膨張逃がしとして完全には機能していません。  
> M8座面の最大部分滑りは `0.0303 mm` です。必要な自由移動量 `0.229 mm` の約 `13%` に留まります。  
> ブラケット本体の代表応力は `81.4 MPa` 程度です。完全固着モデルの `80.45 MPa` とほぼ同じです。  
> したがって、現状の締付け条件では設計上は「固着側」として扱うのが妥当です。

接触面の状態についても解説してくれました。興味深いのが温度サイクルやフレッティングにも言及されていることです。解析結果のコンター図を眺めているだけでは見落としやすい観点です。AIがこうした「次に気にすべき論点」を出してくれるのは、対話型解析のメリットの1つだと思います。

> M8側の接線合力は0.779 kNで、概算摩擦保持力2.20 kNの約35%です。全体としては摩擦余裕があります。
>
> 一方で、M8側の接触積分点の約92%が局所摩擦限界付近です。接触面内で荷重が再配分されながら部分滑りが進んでいます。このため、振動や温度サイクルが繰り返される場合は微小な位置ずれ、摩耗、フレッティングの可能性があります。

そして、解析結果の物理量を、設計上の扱いに変換してくれています。解析結果を見たときに最初に出てくる「ん？ これは結局どういう意味？」という部分を、AIがかなり埋めてくれています。もちろん、最終判断は人間が行う必要があります。

> 単発の22 -> 120 degC温度上昇: 全体滑りしない前提で評価する。  
> ブラケット本体の代表応力: 約80 MPaを目安にする。  
> 局所最大372.7 MPa: メッシュ依存ピークのため、そのまま設計判定に使わない。  
> 長円穴を確実な熱膨張逃がしとして機能させたい場合: M8側の締付け軸力、摩擦条件、座金構成を見直す必要がある。  
> 温度サイクル耐久が重要な場合: 繰返し部分滑りを考慮した評価が必要。

## 一連の検討をCodexにまとめてもらう

ここまでの一連の作業をレポートとしてまとめてもらいました。

手動でCAE解析をしている場合は、通常このような試行錯誤過程はレポートとして残さない（レポート作るのが大変なので）のですが、AIは簡単にまとめてくれます。後で振り返るときに有益な情報になると思います。

## Codexに今回の検討の感想を聞いてみる

今回の検討ではAIが単なる解析実行係ではなく、パートナーのように振舞ってくれました。そこで最後に、Codex自身に今回の感想を聞いてみました。

解析ツールに「感想」を聞く。従来のCAEソフトでは思いもつかないことですが、これができるのがCAE AIエージェントの面白いところです。

プロンプト

```
この一連の作業ではあいまいな状態から様々な仮説を立てて実行してもらったね。
こうやって人間がCodexと対話しながら解析作業を進めることについて、Codex君から見た価値は何かな？
```

Codexの回答は以下の通り。

> 価値は、曖昧さを隠したまま計算を始めず、設計判断に影響する不確定点を順番に潰せることです。
>
> 今回も、最初から詳細なボルトモデルを作っていたら、計算時間を使った割に「長円穴は滑るのか」という本質が見えにくくなっていました。対話によって、次の順序で問題を分解できました。
>
> 1. 幾何学的に逃げ量が足りるか。
> 2. 固着と自由滑りで結果がどれだけ変わるか。
> 3. 実際の締付け条件では、どちらに近づくか。
> 4. 完全固着でも完全滑りでもない部分滑りを、設計上どう扱うか。
> 5. 追加評価が必要な論点は何か。
>
> Codexの役割は、解析を自動実行するだけではありません。仮説、前提、モデル化の限界、計算コスト、結果の使い方を記録しながら、次に確認すべき事項を具体化することです。一方、人間の役割は、実機条件や設計目的を補い、どの保守性を採用するかを決めることです。
>
> 特に有効なのは、CAEでありがちな「高精細な結果は出たが、前提が不明確で判断に使えない」という状態を避けられる点です。今回の結論も、単なる応力値ではありません。
>
> * 長円穴の幾何学的余裕は十分。
> * 締付け摩擦により全体自由滑りにはならない。
> * 接触面では部分滑りが生じる。
> * 単発の温度上昇に対する本体評価は固着側で扱う。
> * 温度サイクルでは摩耗やフレッティングを別途検討する。
>
> このように、数値と設計上の扱いを結び付けられたことが対話型解析の価値です。
>
> 限界もあります。Codexは、実機の組付けばらつき、製造現場の知見、許容値の最終決定、解析モデル外の故障モードを自動では把握できません。したがって最も良い使い方は、Codexに判断を丸投げすることではなく、人間が設計責任を持ちながら、Codexを仮説検証と記録の速度を上げる共同作業者として使うことだと考えます。

なかなか良いことを書いてくれてますね。

Codexの回答で特に印象に残ったのは、次の部分です。

> 価値は、曖昧さを隠したまま計算を始めず、設計判断に影響する不確定点を順番に潰せることです。

このコメントは今回の検討をよく表していると思います。  
そしてこれはCAE技術者が普段やっている思考の流れに近いです。

CAE技術者は普段はこの流れを文書化せず頭の中で考えるだけのことが多いのですが、AIと対話型で進めることで対話ログとレポートとして残り、あとからほかの人が見てわかる状態になっているのも良い点だと思います。

次のコメントもよいですね。

> 最も良い使い方は、Codexに判断を丸投げすることではなく、人間が設計責任を持ちながら、Codexを仮説検証と記録の速度を上げる共同作業者として使うことだと考えます。

AIが何でも自動で決めるわけではありません。今回も、締付けトルクをどう置くのか、相手方部品の熱膨張を無視してよいのか、固着側を代表条件として扱うのか、といった判断は人間が与えています。このような作業を通して、人間が設計責任を持っているといえるでしょう。

一方で、CAE解析の進め方や実行作業はAI側が担当し、出てきた物理量と設計判断の間にある翻訳作業もかなり支援してくれます。「結果を見て、次に何を考えるべきか」を一緒に整理できるというのは、CAE AIエージェントならではの新しい感覚でした。

## まとめ

今回の検討は、AIエージェントがCAEの答えを一発で出すという話ではなく、曖昧な前提を言語化し、複数の仮説を比較し、結果に応じて次の一手を変えていく共同作業者として使える、という内容でした。

さらに、解析結果として得られた滑り量や応力値を、設計上どう扱うべきかというところまで言語化してくれました。物理量と設計判断の間にある「翻訳作業」をAIが支援してくれるのは大きいです。

解析ツールに「感想」を聞けるようになったというのは、なかなか新時代感があります。
