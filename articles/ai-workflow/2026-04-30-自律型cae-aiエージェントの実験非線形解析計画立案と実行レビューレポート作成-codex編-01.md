# AIエージェントによる非線形解析計画立案

構造解析（強度解析）では解析内容により材料を線形にしたり、非線形（降伏後）まで取り扱うなど、解析の計画が変化することがあります。そのような工学的判断をAIがやってくれるのか検証してみます。

## 使用するもの

* Codex App + GPT 5.5
* Ansys MAPDL
* PyMAPDL
* PyDPF Core
* PyDPF Post

および、以前に作成した解析用のSkillも利用します。  
<https://zenn.dev/ms_ai/articles/ae4e9ae8001f7d>

3D形状はあらかじめMAPDLで手作業で作成したものを利用します。

## 構造解析（降伏後あり）

材料が破壊するかどうかを調べる解析を行ってみます。以前の記事で同じ形状で解析していますが、そのときは線形解析でした。  
<https://zenn.dev/ms_ai/articles/e55929c010bf60>

今回は（材料によりますが）SS400のような構造用鋼だと降伏しても直ちに破壊するとは限りません。また、破壊形態にもさまざまあります。  
降伏後破壊だとすると弾塑性解析になり、材料も降伏後の材料特性（SS Curve）を定義する必要があると考えられます。解析設定でも大変形（大ひずみ）を検討する必要があります。このあたりまで考慮できるかが注目点です。

以下のようなプロンプトをCodexのPlanモードで実行し、計画を作ってもらいます。

```
以下の解析のプランを作成し、プランの根拠と狙いを説明する。

# 解析目的
PyMAPDLで以下の解析を実施し、この部品が破壊しないかどうか確かめる。

# 形状データ
./mapdl_work/bar/bar_body.db
長さの単位系はmm

# 境界条件
X=0の面を固定、X=100の面に+X方向に40000N負荷

# 材料
SS400とする。材料物性はWebで調べてユーザーに確認すること。

# 留意事項
ポスト処理にはPyDPF Postを利用すること。
```

内容を見るとなかなか良い感じです。  
材料は二直線近似で定義されています。

```
def set_elastoplastic_material(mapdl: Any) -> None:

    mapdl.mp("NUXY", 1, NUXY)
    mapdl.et(1, "SOLID186")
    mapdl.run("TB,BISO,1")
    mapdl.run(f"TBDATA,1,{YIELD_STRESS_MPA:.12g},{TANGENT_MODULUS_MPA:.12g}")
    mapdl.run("TYPE,1")
    mapdl.run("MAT,1")
    mapdl.run("REAL,")
    mapdl.run("ESYS,0")
```

荷重定義も概ね問題なし。2つの荷重ステップに分けて、第1荷重ステップで荷重負荷、第2荷重ステップで除荷（荷重削除）しています。

スクリプトを見ると除荷後に最大相当塑性ひずみ（Equivalent Plastic Strain）が発生している節点の座標データを取得して、その位置で断面図を描いていました。適当にやっているわけではないことがわかります。

最大塑性ひずみが発生した座標を特定し、その位置で断面カット（スライス）して画像を保存するまでの一連の流れに関連するコードを抜粋すると以下の通り。主に「座標の特定」「グリッドの生成」「断面カットとプロット」の3つのステップで構成されています。

#### (1) 最大値を持つ節点の座標を特定する

解析結果から、最大値が発生しているインデックス（`idx`）を見つけ、その地点の座標（`grid.points[idx]`）を取得します。

```
def nodal_grid(simulation: Any, dataframe: Any, scalar_name: str) -> tuple[pv.UnstructuredGrid, np.ndarray, float]:
    grid = simulation.mesh._core_object.grid.copy(deep=True)
    field = dataframe._core_object[0]
    values = np.asarray(field.data, dtype=float).reshape(-1)
    
    # 節点IDに対応する値をマッピング
    values_by_node = dict(zip([int(v) for v in field.scoping.ids], values))
    nodal_values = np.array([values_by_node.get(int(n), np.nan) for n in simulation.mesh.node_ids])
    grid.point_data[scalar_name] = nodal_values

    # 最大値を持つインデックスを特定し、その座標を取得
    idx = int(np.nanargmax(nodal_values))
    return grid, np.asarray(grid.points[idx], dtype=float), float(nodal_values[idx])
```

#### (2) 断面位置（X座標）を決定する

`nodal_grid` で取得した座標 `ep_xyz` のうち、X成分（`index 0`）を取り出します。

```
# write_reports 関数内での処理
ep_grid, ep_xyz, _ep_value = nodal_grid(sim, figure_data["eppl_unloaded"], "eppl_unloaded")

# 最大塑性ひずみ地点の X座標を断面位置として採用
section_x = float(ep_xyz[0])
```

#### (3) 指定した座標でスライスしてプロットする

PyVistaの `slice` メソッドを使用して、特定した X座標でメッシュをカットし、保存します。

```
def save_plot(grid: pv.UnstructuredGrid, scalar: str, title: str, path: Path, section_x: float | None = None) -> None:
    plotter = pv.Plotter(off_screen=True, window_size=(1600, 1050))
    mesh = grid

    # section_x が指定されている場合、X軸に垂直な面（normal="x"）でスライス
    if section_x is not None:
        mesh = grid.slice(normal="x", origin=(section_x, 0, 0))

    plotter.add_mesh(mesh, scalars=scalar, cmap="turbo", show_edges=False)
    # （中略：カメラ設定など）
    plotter.screenshot(str(path))
    plotter.close()

# 実行部分
save_plot(ep_grid, "eppl_unloaded", "最大塑性ひずみ位置の断面図", 
          figures / "section_residual_eqv_plastic_strain.png", section_x=section_x)
```

# まとめ

解析内容の文脈をもとに、非線形解析にするかどうかの判断もAIが自律的にできることが分かりました。非線形解析になると様々な解析設定が必要ですが、それも難なくこなせました。しかも、どういう結果を見せるのが良いかまで理解して実装していました。  
CAEエンジニアは非線形解析を行う際には、材料や解析設定、見るべき結果の種類など様々な点に目配りしながら解析を進めますが、同等レベルのことができているように思えます。

一方で、材料データをちゃんと渡さないと勝手に推定してやってしまうことも分かったので、AIの実行結果は確認して必要に応じて追加で説明させるといった対策をとるのが大事だということも改めて認識しました。
