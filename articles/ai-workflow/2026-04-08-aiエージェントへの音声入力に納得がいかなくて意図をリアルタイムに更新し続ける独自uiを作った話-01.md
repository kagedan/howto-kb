## Voxclaw

PWAによる独自UIを持ったAIエージェント、Voxclawを今回作成した。最大の特徴は音声からリアルタイムに意図推定して、意図推定結果をエージェントに実行させることができること。意図推定体験を向上させるためには独自UIにならざるを得なかった。

<https://github.com/Nishina-N/voxclaw>

### Voxclaw UI

![Voxclaw interface](https://static.zenn.studio/user-upload/5edf3b110eba-20260408.png)  
UIは一番左のログインを除き、５画面構成とした。メインはchatへの意図推定入力だが、settingsで[Keybinder](https://zenn.dev/nishina__n/articles/2fd0f90086841b)に登録するAPIキーを設定できたり、文字サイズの変更があるなど、使いやすさにこだわっている。今後も使いにくい部分は変えていくと思う。

### 動作イメージ

![Voxclaw action](https://static.zenn.studio/user-upload/98d912813d94-20260408.png)  
左端のマイクボタンを押して、音声入力を始めると、ひたすら意図推定結果を入力欄に出してくれる。  
この入力でいいなと思ったら、右の送信ボタンでエージェントに投げられる。マイクボタンをずっとONにしておけば意図推定は常にされるので、喋りながらいいタイミングで送信ボタンを押すと、エージェントへ継続して指令を出せる。

## gemiclaw との違い

過去に作った[gemiclaw](https://github.com/Nishina-N/gemiclaw)はDiscordをUIとして使うエージェントで、Voxclawはその後継にあたる。細かい説明に入る前に違いを述べておく。主な変化点は以下の通り。

|  | gemiclaw | Voxclaw |
| --- | --- | --- |
| UI | Discord | 専用PWA |
| 音声入力 | Discordボイスチャット | Gemini Live APIによるリアルタイム意図推定 |
| 操作 | 複数回の確認フロー | 意図推定＋1アクションで実行 |
| 外部公開 | DiscordのIDが必要 | JWT認証＋Cloudflare Tunnel |

Discordをやめたのは音声入力に対する制約が大きすぎたからで、UIを自作し、意図推定体験をコントロールできるようにした。もちろん[gemiclaw](https://github.com/Nishina-N/gemiclaw)が持っていた軽量、セキュア、自己拡張ができるという思想と、スキルの仕組みや[Keybinder](https://zenn.dev/nishina__n/articles/2fd0f90086841b)連携はそのまま引き継いでいる。

## なぜ作ることになったのか

3週間ほど前だろうか。[gemiclaw](https://github.com/Nishina-N/gemiclaw)に音声入力を実装していた。

discordでボイスチャットを送信し、その音声を分析したエージェントが「〇〇ということですか？」と聞いてきて、オッケーを出せば実行してくれる機能である。

音声入力は声が出せる環境では非常に便利なのだか、DiscordのUIに依存している限りは１つの指令を出すまでにどうしても複数回のやりとりが発生してしまう。

原因は音声を文章に変換する時の誤変換である。これは外部の環境音や、発話の明瞭度、語彙、イントネーションや訛りに由来する。しかも省略表現まで絡んでくると、誤変換や省略表現を文章に変換した後の文章から意図推定することは困難になる。

誤変換から発生した誤推定は専門用語を使ったりするようなエッジケースまで考えると、どうやっても排除できない。従って複数回の確認フローが必要になる。この複数回の確認フローがDiscord のUIであるボイスチャットを使っている限りは回避できない。

![](https://static.zenn.studio/user-upload/29753bf39d1c-20260409.png)

## 意図推定UI

こうなってくると、音声入力に関してはチャット方式から離脱するしかない。今回目指した姿はマイクオンにしておけば、発話から意図を推定し続け、常にチャットで送信予定のメッセージを更新し続ける形である。そうやって意図推定が修正され続け、意図と合えば送信ボタンを押して、エージェントに実行させれば良い。これは最初に示した動作イメージの図そのものである。

もちろん、意図推定結果のテキストはユーザーが手入力で変更することができる。これは今後ユーザーの変更結果を学習して個人最適化させることを見越している。

また、このUIアプリはユーザーの意図推定を実施し、AIエージェントに渡すだけでなく、エージェント応答の表示や、各種設定を行うインターフェースでないと使いにくい。そのため現在は５つのタブを設定し、初期の立ち上げの負荷を減らすために[Keybinder](https://zenn.dev/nishina__n/articles/2fd0f90086841b)に渡すAPIキーもアプリから入力可能としている。

| タブ | 説明 |
| --- | --- |
| **Chat** | メイン画面 — 音声またはテキスト入力、スキルの実行結果を表示 |
| **Skills** | 利用可能なスキルの一覧と説明 |
| **Cron** | スキルの定時実行スケジュールを設定（時刻・曜日・送信先） |
| **Task** | ローカルタスク管理 — 追加・完了・編集・期限設定。音声入力にも対応 |
| **Settings** | APIキー（Brave Search、Mapbox）と Google 認証の管理、表示言語および入力言語の変更、文字サイズの変更 |

ちなみにskillsのところは[aa\_taka](https://zenn.dev/aa_taka)さんがgemiclawの成長が見えると良いというコメントをくださり、それを参考にまずは個数を見える化させていただきました。ありがとうございました。

## データフロー

![](https://static.zenn.studio/user-upload/3d11094879f9-20260409.png)  
データフローに関してはこの図の通り、PWAによるアプリ部分のフロントエンド及びバックエンドにVoxclaw（元gemiclaw)が接続される形となっている。  
ユーザーの音声はAPPのフロントエンドからバックエンドに渡されて、gemini Live APIで意図抽出された後、一度フロントエンドに戻り、ユーザーの確認を経て、AIエージェント側に渡される構成になっている。AIエージェントであるvoxclawはfunctionとskillsを使ってユーザー意図の実現に尽力するが、外部とのやり取りでAPIキーが必要なものは[Keybinder](https://zenn.dev/nishina__n/articles/2fd0f90086841b)経由でやり取りすることにより、voxclawからはAPIキーが見えない構成となっている。

## アーキテクチャ

![](https://static.zenn.studio/user-upload/6c03da0bc0cf-20260409.png)  
データフローでも触れたが、意図推定を重視した入力方式を作るために、AIエージェントとは別にUIアプリ部分フロントエンドとバックエンドが必要となった。  
各モジュール単位で機能をまとめることで、アプリ部分、AIエージェント部分、[Keybinder](https://zenn.dev/nishina__n/articles/2fd0f90086841b)部分は素結合となる構成にしており、アプリ部分やエージェント部分も内部で機能の階層化を意識した作りになっている。  
オプションとしては、DISCORD への接続も残してはあるものの、現在私はほとんど使っていない。

また、[gemiclaw](https://github.com/Nishina-N/gemiclaw)も今回のVoxclawも自分のPCやサーバーで動かすことを前提としているが、そのサーバーや、自分のPCに外部ネットワークからアクセスする必要が出てくる。外部からのアクセス方法にはTail ScaleとCloudflare Tunnelを想定している。Cloudflare Tunnelを使う場合はドメインが必要なので無料では使いづらいが、Tail Scaleであれば無料で自分のPC上で動作するVoxClawにアクセスすることができる。なお、Cloudflare Tunnelを使う場合は様々な人がアクセスできてしまうため、JWT認証によるパスワード設定を行った方が良い。（2026/4/10 TailScale部分の記載追加修正）

## 始め方

ローカルで試してみるのは簡単。PWAはブラウザベースのアプリなので、PCのマイクでも動作する。  
必要なものはGEMINI\_API\_KEYと自分で決めるパスワード（PWA\_PASSWORD）だけ。

```
git clone https://github.com/Nishina-N/voxclaw.git
cd voxclaw
cp .env.example .env  # GEMINI_API_KEY と PWA_PASSWORD を記入
docker compose up -d
# http://localhost:3000 にアクセス
```

BraveSearchやMapboxなどのAPIキーはSettingsタブからあとで追加できる。  
外出先からもアクセスしたい場合は、TailScaleとCloudflare TunnelのセットアップについてREADMEの[セットアップ](https://github.com/Nishina-N/voxclaw/blob/main/docs/setup.md)に記載している。

スマホなど外部からのアクセスにはTailscaleを使うのが非常に便利です。[こちらの記事](https://zenn.dev/nishina__n/articles/8a9a201b8f629e)でTailscaleを用いた接続方法について解説しています。

## まとめと今後

Voxclawは「音声で意図を推定し、確認してから実行する」というフローを専用UIで実現したAIエージェント。チャット形式では避けられなかった複数回の確認フローを、リアルタイムの意図推定とテキスト編集の組み合わせで1アクションにできたところが最大の差別化である。

今後やりたいこととしては、ユーザーが意図推定結果を手で書き直した内容を学習して、個人ごとの語彙や言い回しに最適化していくことを考えている。また現在はGeminiで動いているエージェント部分をClaudeに切り替えられるようにすることも視野に入れている。

AIに何をさせるかを常にユーザーがコントロールできることが設計の根幹です。OSSとして公開しているので、試してみていただけると幸いです。フィードバックや改善提案も歓迎しています。
