## はじめに

こんにちは。株式会社MOCHITECでCTOをしている上田（@ryo\_u27）です。

Claudeに何か頼むとき、こんなことはないでしょうか。「この前と同じ感じで、議事録にして」。でも、"この前と同じ感じ"は、新しいチャットに伝わりません。だから毎回、見出しの順番、表にするか、空欄の書き方を、ひとつずつ説明し直すことになります。

この「毎回おなじ説明」を、Claude側に一度だけ覚えさせておく。それが「Skill」です。

社内でClaudeを使う人は増えました。ただ、多くはチャットで止まっていて、この一歩先をまだ使っていません。便利なのにもったいないので、社内の、とくにエンジニアでないみんなに知ってほしくて書きました。

## Skillは、新人に渡す業務マニュアル

Skillは、Claudeの機能のひとつです。一言でいうと、\*\*新しく入った人に渡す「業務マニュアル」\*\*のようなものです。

優秀な新人でも、入った初日は自社のやり方を知りません。だから、議事録の書き方マニュアルを一度渡しておく。次からは、その人がマニュアルを見て、自分で同じように仕上げてくれる。Skillもこれと同じで、一度渡しておけば、Claudeが自分でそれを見て動きます。

身構えなくて大丈夫です。中身はプログラムではなく、ふつうの日本語で書いた指示メモです。たとえば、議事録を整えるSkillの中身はこんな感じです。

```
name: 議事録フォーマッター
いつ使う: 会議のメモを議事録に整えるとき
手順:
  - 日時 / 参加者 / 決定事項 / ToDo（担当・期限）/ 次回 の順に並べる
  - ToDo は表にする
  - メモに無いことは「（記載なし）」と書く
```

これだけです。いつもチャットに打っている指示を、メモにして名前をつけて置いておく。すると、会議のメモを渡すだけ。Claudeが自分でSkillを引き出し、その通りに仕上げます。あなたは「議事録にして」と言うだけです。

（正式には「SKILL.md」というファイルですが、書くのはこういう普通の文章です）

## 毎回プロンプトで頼むのと、何が違う？

ここで、こう思うかもしれません。「それ、毎回プロンプトで指示すればいいだけでは？」。

近いです。でも、一度マニュアル（Skill）にしておくと、3つ変わります。

* **言わなくても効く** … 関連する仕事のたびにClaudeが自分で引き出すので、毎回こちらから言わなくていい
* **誰でも同じ** … 配れば全員が同じ型で仕上がり、人によってバラつかない
* **必要なときだけ軽い** … たくさん作っても普段は重くならず、関連するときだけ中身を開く

ひとつだけ。Skillは、あなたの代わりに考えるものではありません。やり方を覚えるだけです。何を書くかを決めるのは、あなた自身です。

*図1: 毎回自分でやる vs Skill。一度作れば、自分が抜けても回る*

## どこで効く？ 要らない場面も

では、自分の仕事のどこで使えばいいのか。答えはシンプルです。**あなたが毎回、Claudeの答えを「ここはこう直す」「これを足す」と手直ししている、その場所**です。そこは、Claudeのくせと自社のやり方がズレている所。マニュアルを渡せば埋まります。

部署ごとに、たとえばこんな所です。

* **営業** … 商談メモを、社内の日報フォーマットに整える
* **採用** … 面接メモを、評価シートの型（評価軸・点数・コメント）に落とす
* **総務・経理** … 経費や稟議の説明文を、決まった書き方にそろえる
* **広報・マーケ** … 投稿文やリリースを、自社の表記ルールでそろえる

逆に、要らない場面もはっきりあります。単発の作業や、普通に頼めば十分なことに、Skillは要りません。むしろ作るほうが手間です。効くのは「**繰り返す × 決まった型がある**」ときだけ。迷ったら、下の図で見てください。

## 実際に作ってみる

「作るの難しそう」と思うかもしれません。でも、コードは書きません。claude.aiで、会話するだけです。

### 1. 作りたいSkillを伝える

新しいチャットで、「自社の議事録はこういう形にしたい」を、新人に説明するように書きます。難しい書式は要りません。下の文（これがClaudeへの入力です）を、自社のルールに合わせて書き換えて、そのまま送ります。

```
社内の議事録を、決まったフォーマットに整えるSkillを作りたいです。会議の生メモを貼ったら、「日時／参加者／決定事項／ToDo（担当・内容・期限の表）／次回」の形に整えてください。文体は敬体で、元メモに無い情報は創作せず、空欄は「（記載なし）」とします。nameとdescriptionも提案してください。
```

すると、Claudeが「skill-creator」という仕組みで、マニュアル（SKILL.md）を組み立ててくれます。

![skill-creatorが議事録整形SkillのSKILL.mdを生成したところ](https://static.zenn.studio/user-upload/33f0e99292f2-20260617.png)  
*図3: 会話だけでSKILL.mdができる*

できたら「スキルを保存」を押すだけ。保存すると自動で有効になり、すぐ使えます。

### 2. 使ってみる

新しいチャットに、整えたい生メモを貼って「議事録にして」と頼みます。Skillを名指しする必要はありません。Claudeが自分でそのSkillを選びます。

試しに、こんな雑なメモを貼ってみます（最後の「議事録にして」も含めて）。

```
6/17 14時〜 会議室A
田中、佐藤、鈴木
・新ロゴ案はB案で決定
・予算は来週あらためて協議
todo 佐藤→競合調査 月末まで
鈴木→ロゴ入稿（期限未定）
次回 6/24 同じ時間
議事録にして。
```

![貼った生メモが議事録フォーマットに整って出力されたところ](https://static.zenn.studio/user-upload/a029a35f16b2-20260617.png)  
*図4: 雑なメモが、決まった型の議事録になる*

雑なメモが、決まった型の議事録になりました。決定事項とToDoが分かれ、ToDoは表になり、空欄も「（記載なし）」で埋まります。普通に頼んでも議事録は書けますが、この型まで毎回そろうのが、Skillにしておく値打ちです。

## 始める前に、ひとつだけ

他人の作ったSkillは、信頼できるものだけにしてください。中身が悪意あるものだと、データを抜かれることもある、と公式も注意しています。公式リポジトリ（<https://github.com/anthropics/skills>）か、自分で作ったものから始めるのが安全です。

## まとめ

新しく入った人に、一度だけ業務マニュアルを渡す。Skillは、それをClaudeにやるだけのことです。賢いClaudeも、自社のやり方までは知りません。そこを一度渡しておけば、次から毎回、自分が言わなくても、同じ型で返ってきます。

万能ではありません。単発の作業には要りません。でも、「毎回これは直しているな」という作業がひとつでもあるなら、それはSkillの使いどころです。まずはその1つから、マニュアルを渡す感覚で作ってみてください。
