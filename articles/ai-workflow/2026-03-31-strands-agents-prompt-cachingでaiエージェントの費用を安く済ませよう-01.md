こんにちは👋

Strands Agents + Amazon BedrockでAIエージェントを作ってみたけど、費用が思ったより高くてびっくり！！...ということあると思います。

AIエージェント元年だなんだと言いつつ、一回の実行に数百円もかかっているようでは、まともに開発することすらできません。  
今回はAmazon Bedrockの機能である「**Prompt Caching**」を利用して、AIエージェントの推論コストを大幅に削減する方法を紹介しようと思います。

## Prompt Cachingとは？

**Prompt Caching**は、Amazon Bedrockの一部基盤モデルで利用可能なオプション機能です。その名の通り、リクエストに含まれるプロンプトを**キャッシュ**することができます。

この記事を執筆した時点でPrompt Cachingが利用可能な基盤モデルは、Anthropic ClaudeとAmazon Novaのみです。

<https://docs.aws.amazon.com/ja_jp/bedrock/latest/userguide/prompt-caching.html>

プロンプトをキャッシュすることで何が変わるのかというと、推論時の「**コスト**」と「**レイテンシ**」が変化します。

まず、コストについては「**キャッシュの書き込み時**」と「**キャッシュの読み込み時**」でコストが変化します。Anthropicが提供している基盤モデルの場合だと、以下のようになります。

| Model | Base Input Tokens | 5m Cache Writes | 1h Cache Writes | Cache Hits & Refreshes | Output Tokens |
| --- | --- | --- | --- | --- | --- |
| Claude Opus 4.6 | $5 / MTok | $6.25 / MTok | $10 / MTok | $0.50 / MTok | $25 / MTok |
| Claude Sonnet 4.6 | $3 / MTok | $3.75 / MTok | $6 / MTok | $0.30 / MTok | $15 / MTok |
| Claude Haiku 4.5 | $1 / MTok | $1.25 / MTok | $2 / MTok | $0.10 / MTok | $5 / MTok |

<https://platform.claude.com/docs/ja/about-claude/pricing>

5m Cache Writes、1h Cache Writesというのは、プロンプトを**キャッシュする期間**を表しています。5分よりも1時間の方がキャッシュヒットする可能性が上がりますが、その分、トークンあたりの料金が高くなるという感じです。

注目して欲しいのは「Cache Hits & Refreshes (キャッシュ読み取り時のコスト)」の部分です。基本のコストである「Base Input Tokens」と比較すると、いずれのモデルも**10分の1の価格**になっています！

AIエージェントは、短い期間で同一の会話履歴を持つリクエストを繰り返し送信するため、過去の会話履歴がキャッシュとして必ずヒットし(会話履歴を圧縮したり、意図的に改変しない限りは)、**コスト削減の恩恵を最大限に生かすことができます！**

!

では、何でもかんでもキャッシュするべきか、というとそうでもなくて、キャッシュ書き込み時のコストは、基本のコストより25%以上も高く設定されています。

そのため、キャッシュ読み取りによって得られるコスト減少が、キャッシュ書き込みによるコスト増加よりも大きいと見込まれる場合のみ、Prompt Cachingを有効化するべきです。

AIエージェントは、基本的に数十回ほどの推論サイクルを繰り返すため、Prompt Cachingの有効化によりコスト削減のメリットが得られる可能性が高いというだけで、必ずしもコストが削減される訳ではないことに注意してください。

また、キャッシュがヒットした場合は、推論にかかる**レイテンシが短くなる**というメリットもあります。コンテキストが多くなるほど推論に時間がかかるAIエージェントにとっては、こちらも嬉しいメリットです。

## どこにキャッシュが効くのか？

ややこしいですが、Prompt Cachingを適用できるフィールドとして、**tools**、**system**、**messages**の3つがあります。

* **Tool Caching**

  + 基盤モデルに渡す**ツールの定義**をキャッシュすることができます
  + ツールの定義なので、MCPサーバで提供されるツールなどもこれに該当します
* **System Prompt Caching**

  + **システムプロンプト**をキャッシュすることができます
  + システムプロンプトに動的な値を差し込んでいる場合は、異なるセッション間でキャッシュが無効化されます(同一セッション内では問題ありません)
* **Messages Caching**

  + ユーザとアシスタントの間で発生する**メッセージのやり取り**をキャッシュすることができます
  + 画像やドキュメント、ツールの実行リクエスト・ツールの実行結果なども同様にキャッシュされます
  + キャッシュするためには、**キャッシュポイント**と呼ばれるマーカーを配置する必要がありますが、Claudeモデルではメッセージの末尾に自動でキャッシュポイントを配置してくれる**自動キャッシュ戦略**がサポートされています
  + 同じく過去のメッセージ履歴を動的に変更した場合は、キャッシュが無効化されます(実際はどこにキャッシュポイントを差し込んだかによって変わるのですが、話がややこしくなるので割愛します)。

また、これらのキャッシュには「**キャッシュ最小トークン数**」と呼ばれる制限が設けられています。

リクエストに含まれる`tools`や`system`、`messages`のトークン数が、キャッシュ最小トークン数を超えていない場合は、**Prompt Cachingを有効化したとしてもキャッシュされない**ので注意が必要です。

| Model | キャッシュ最小トークン数 |
| --- | --- |
| Claude Opus 4.6 / 4.5 | 4,096 tokens |
| Claude Sonnet 4.6 | 2,048 tokens |
| Claude Sonnet 4.5 | 1,024 tokens |
| Claude Haiku 4.5 | 4,096 tokens |

<https://platform.claude.com/docs/en/build-with-claude/prompt-caching>

ちなみに、どういう条件でキャッシュが効くのか？無効化されるのか？については、以下の記事で丁寧に解説してくださっているので、こちらをぜひ参考にしてみてください。

<https://zenn.dev/ubie_dev/articles/ade17afebabaa9>

## Strands AgentsにおけるPrompt Caching

では、これらのキャッシュ可能なフィールドを「Strands Agents」でどのように扱えばよいのか、一つずつ見ていきましょう。

<https://strandsagents.com/docs/user-guide/concepts/model-providers/amazon-bedrock/#caching>

ツールをキャッシュする場合は、BedrockModelの引数である`cache_tools`に`default`を渡すだけで有効化することができます。

以下の例だと、`calculator`や`current_time`などのツール定義がキャッシュ最小トークン数を超えているのであれば、キャッシュされます。

```
from strands import Agent
from strands.models import BedrockModel
from strands_tools import calculator, current_time

agent = Agent(
    model=BedrockModel(
        model_id="anthropic.claude-sonnet-4-20250514-v1:0",
        cache_tools="default"
    ),
    tools=[calculator, current_time]
)
```

### System Prompt Caching

システムプロンプトのキャッシュには、`SystemContentBlock`を使用します。

キャッシュ最小トークン数を超えたシステムプロンプトと`cachePoint={"type": "default"}`を渡すことで、キャッシュすることができます。

```
from strands import Agent
from strands.models import BedrockModel
from strands.types.content import SystemContentBlock

agent = Agent(
    model=BedrockModel(
        model_id="anthropic.claude-sonnet-4-20250514-v1:0"
    ),
    system_prompt=[
        SystemContentBlock(
            text="You are a helpful assistant..." * 1600
        ),
        SystemContentBlock(cachePoint={"type": "default"})
    ]
)
```

### Message Caching

Strands Agentsでは、先ほど説明した**自動キャッシュ戦略**と手動キャッシュポイントのどちらを利用するかを選択することができます。

以下のコードは、キャッシュ方法として自動キャッシュ戦略を適用した例であり、`cache_config`に`CacheConfig(strategy="auto")`を渡すだけで、いい感じにキャッシュしてくれるようになります(ちなみに、キャッシュ期間は5分で固定です)。

こちらも、メッセージのやり取りがキャッシュ最小トークン数を超えていない場合は、キャッシュされない点に注意してください。

```
from strands import Agent
from strands.models import BedrockModel, CacheConfig

agent = Agent(
    model=BedrockModel(
        model_id="us.anthropic.claude-sonnet-4-5-20250929-v1:0",
        cache_config=CacheConfig(strategy="auto")
    )
)
```

というわけで、ぽこポケの情報をWeb検索して回答できるAIエージェントを作成してみようと思います。

### Web検索AIエージェントを作成する

まずは、普通にStrands AgentsでWeb検索AIエージェントを作ってみます。

Web検索には**Tavily**というサービスを使います。Tavilyなどの検索ツールは、**Strands Agents Tools**にて、Strands Agentsで利用できる形で提供されているので、こちらを使って簡単に組み込むことができます。

<https://www.tavily.com>

<https://github.com/strands-agents/tools>

```
from strands import Agent
from strands.models import BedrockModel
from strands.tools.executors import SequentialToolExecutor
from strands_tools.tavily import tavily_search

agent = Agent(
    model=BedrockModel(
        region_name="ap-northeast-1",
        model_id="global.anthropic.claude-sonnet-4-6",
        temperature=0.0,
        max_tokens=8192,
    ),
    system_prompt=(
        "あなたは「ぽこあポケモン」と呼ばれるゲーム作品に関する質問に回答するチャットボットです。なるべく簡潔に回答を生成してください。"
        "検索ツールを利用する場合は、検索キーワードに「ぽこあポケモン」というゲーム名を必ず付与してください。"
    ),
    tool_executor=SequentialToolExecutor(),
    tools=[tavily_search],
)
result = agent(prompt)
```

```

\mathrm{Input} = \frac{8096}{1000000} \times 3 = 0.024288

\mathrm{Output} = \frac{301}{1000000} \times 15 = 0.004515

\mathrm{Total} = 0.024288 + 0.004515 = 0.028803

AIエージェントが扱うコンテキストの量が少ないため安いですね。この程度のやり取りであれば、Prompt Cachingを使うメリットはほぼありません。

### 😊 Prompt Cachingがあるとき〜

同じ質問をPrompt Cachingを有効化した状態で聞いてみましょう。

System PromptやToolはキャッシュ最小トークン数を越えないと思うので、今回は**Message Caching**だけを有効化します。

```
from strands.models import CacheConfig

model=BedrockModel(
    region_name="ap-northeast-1",
    model_id="global.anthropic.claude-sonnet-4-6",
    temperature=0.0,
    max_tokens=8192,
    cache_config=CacheConfig("auto"),
)
```

### まとめ

- **全9世代・全27匹の御三家すべてがぽこあポケモンに存在**しています！
- 生息地は「草むら」「キャンプ」「秘密基地」「ロッカールーム」など多彩な種類があり、特定の素材を組み合わせて作成する形式です。
- 一部のポケモン（キモリ、ミズゴロウ、ナエトルなど）は今回の検索では生息地名の詳細が確認できませんでした。詳しくはゲームウィズやゲームエイトの攻略サイトでご確認ください。
```

似たような回答が得られましたが、トークン消費量はどうでしょうか？

```
{
  "input_tokens": 138421,
  "output_tokens": 3871,
  "cache_read_input_tokens": 296250,
  "cache_write_input_tokens": 117319,
  "total_tokens": 555861
}
```

`cache_read_input_tokens`と`cache_write_input_tokens`が増えており、入力トークン数のうち半分以上が読み込みキャッシュを利用しています。

\mathrm{Input} = \frac{138421}{1000000} \times 3 = 0.4153

\mathrm{Output} = \frac{3871}{1000000} \times 15 = 0.0581

\mathrm{Cache\ Read} = \frac{296250}{1000000} \times 0.30 = 0.0889

\mathrm{Cache\ Write} = \frac{117319}{1000000} \times 3.75 = 0.4399

\mathrm{Total} = 0.4153 + 0.0581 + 0.0889 + 0.4399 = 1.0022

コストを計算すると約1.0ドルで、日本円に変換すると**158円**です！

Prompt Cachingのコードを少し追加するだけで、ジュース一本分くらいの値段になりました。これならまだギリギリ許せる☺️

仮に今回のケースでPrompt Cachingを有効化せずに動かしていた場合を考えると、キャッシュのトークンが全て通常の入力トークンに回るため、コストは約1.71ドルになります。つまり、Prompt Cachingにより**約48%のコストを削減できたことになります！**

単発の実行であればともかく、リクエストに応じて何度もAIエージェントを実行するようなケースだと、その差は何千円〜何万円と大きくなるため、Prompt Cachingがいかに有用か伝わったかなと思います。
