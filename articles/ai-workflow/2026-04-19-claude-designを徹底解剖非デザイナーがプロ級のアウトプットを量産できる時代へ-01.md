## はじめに

「資料のデザインがどうしても垢抜けない」「ランディングページのワイヤーフレームを描きたいけれど、Figmaは敷居が高い」「デザイナーに発注するほどでもないけれど、それなりの見た目にはしたい」。こうした悩みは、エンジニアや経営企画、マーケティング担当者、スタートアップの代表など、デザイナーではない多くのビジネスパーソンが抱えるものです。

Anthropic Labsから2026年4月に発表された **Claude Design** は、まさにこの「デザイナーではない人たちが、デザイナーに近いアウトプットを出すための橋渡し」を狙ったプロダクトです。会話ベースで指示を出すだけで、スライド、ワイヤーフレーム、ランディングページ、プロトタイプ、さらには動画のようなアニメーション付き表現までを一気通貫で生成できます。

本記事では、Claude Designの基本操作から、押さえておくべき7つの強力な機能、そしてAnthropicがなぜこのタイミングでこのプロダクトを投入してきたのかという戦略的背景まで、実際に触った体感ベースで掘り下げて解説していきます。想定読者は、Claudeの有料プランを契約していて、デザイン業務やアウトプット作成を効率化したいエンジニアや企画担当者です。

## Claude Designとは何か

Claude Designは、Anthropic Labsが2026年4月17日にリサーチプレビューとして公開した、会話型のビジュアル制作サービスです。搭載モデルは同日リリースされた **Claude Opus 4.7**（ビジョン対応モデル）で、Pro、Max、Team、Enterpriseの各有料プラン契約者が利用できます。

位置付けとしては、Figma、Canva、Adobe系ツールといった既存のデザインツールを「完全に置き換える」ものではありません。むしろ、着想から初稿までの最も時間がかかる部分を会話で一気に埋め、仕上げは既存ツールに渡せるようエクスポート先を充実させる、というスタンスをとっています。

### どんなアウトプットが作れるのか

Claude Designで現時点（2026年4月）に作成できるアウトプットは、ざっと挙げると以下の通りです。

* スライド・ピッチデック・1ページ資料
* ランディングページやウェブサイトのワイヤーフレーム
* インタラクティブなプロトタイプ（HTMLベースで動く）
* マーケティング用のバナーや一枚絵
* アプリのUIモックアップ（オンボーディングフロー、フォームなど）
* SVG形式のアイコンやイラスト
* HTMLベースのアニメーション付き「動画風」コンテンツ

面白いのは、「動画っぽいもの」もHTML/CSSアニメーションで作り込めてしまう点です。本物のMP4を吐き出すわけではありませんが、プレゼンテーション内で見せる紹介動画の雛形としては十分に使えます。

### 利用できるプランと枠

重要な点として、Claude Designの使用量はチャットやClaude Codeの利用枠とは**別枠**でカウントされます。The Register の記事によれば、各プランの週次リミットとは独立して、Claude Design専用の枠が割り当てられる設計です。

このため「チャットの残枠を気にせず試せる」というメリットがある一方、集中的に使うとあっという間にリサーチプレビューの枠を使い切ってしまうという体験をしているユーザーも少なくありません。筆者自身、ProとMaxの両方で枠を使い切り、数日待ちを経験しました。

## まずは基本操作から押さえる

Claude Designは `claude.ai/design` のパスから、あるいはClaude.aiの左サイドバーにあるパレット型のアイコンから開始できます。画面構成は非常にシンプルで、

* **左側**：チャットインターフェース（ここで指示を出す）
* **右側**：キャンバス（生成結果がリアルタイムで描画される）

という二分割レイアウトです。Claude CodeやCowork を触ったことがある方なら、構造としてはほぼ同じと考えて差し支えありません。

### 新規プロジェクトの作成フロー

プロジェクトを開始するには、左サイドバーからメニューを開き、作りたいアウトプットの種類（スライド、ワイヤーフレーム、プロトタイプなど）を選んでから「Create」を押します。たとえば「オンラインイベントのアンケート結果を紹介するページ」というような指示を投げると、Claude Designはいきなり画像を生成するのではなく、まず質問を投げ返してきます。

* ターゲットは社内向けか、一般公開向けか
* トーンは明るく親しみやすい感じか、フォーマルか
* 用意すべきワイヤーフレームのパターンはいくつか

こうした確認項目を複数のチェックボックスや選択肢として並べてくれるので、ユーザーは「選ぶだけ」で意図を明確化できます。選び終えて「Continue」ボタンを押すと、そこから初回のデザインが一気に生成されます。

### デザインシステムという独自概念

Claude Designの根幹を成すのが「デザインシステム」という仕組みです。これは自分の組織やプロジェクト固有の、色・フォント・ロゴ・UIパーツなどの**ビジュアル言語**をClaudeに事前登録しておく機能です。

登録方法は柔軟で、以下のいずれかから取り込めます。

* 自社のウェブサイトのスクリーンショット
* 会社ロゴの画像ファイル
* 既存のスライド資料（DOCX、PPTX、XLSXに対応）
* GitHubのコードリポジトリ

特にコードリポジトリをリンクする方法は強力で、既存のコンポーネントやスタイリングパターンを読み取り、自動的にデザイン規約を抽出してくれます。抽出後は、Claude自身がブランドパターン集のようなページを生成し、色味・タイポグラフィ・余白・ボタンのバリエーションなどを一覧化してくれます。

このデザインシステムは一度作ってしまえば、それ以降のすべてのプロジェクトに自動適用されるため、アウトプットのトーン＆マナーがブレなくなるという大きな利点があります。

## Claude Designの強み①：デザインシステムによる「ブランド統一」

非デザイナーが自分の思いつきで資料やウェブページを作ると、フォントや配色がプロジェクトごとにバラバラになりがちです。いわゆる「素人っぽさ」の正体は、色の選び方や余白の取り方以前に、この**トーン＆マナーの不統一**にあります。

Claude Designはまず、ここをシステムで解決しようとします。最初のプロジェクトでデザインシステムを構築してしまえば、次のスライドを作るときも、ランディングページを作るときも、同じ色・同じフォント・同じコンポーネント群が自動適用されます。

| 機能 | 従来のAIによるデザイン生成 | Claude Design |
| --- | --- | --- |
| ブランド統一 | プロンプトで毎回指示が必要 | デザインシステムで自動適用 |
| 色・フォントの一貫性 | 生成ごとにブレる | プロジェクト間で固定 |
| 参照元 | 都度指定 | コードベース・既存資料から自動抽出 |
| 複数ブランド管理 | 困難 | 複数のデザインシステムを切り替え可能 |

組織単位で複数のデザインシステムを維持できるため、自社ブランドと、受託案件のクライアントブランドを併存させるような使い方もできます。

## Claude Designの強み②：対話による「意図の言語化」支援

非デザイナーがデザイン制作で詰まる最大の理由は、「自分が何を作りたいのかを言語化できない」ことにあります。「かっこよくして」「シンプルにして」と言ったところで、何を以てかっこよく、何を以てシンプルとするかが人によって異なるからです。

Claude Designは、このボトルネックをディレクターのような役回りで埋めてきます。たとえば「AIエージェントの仕組みを1枚の資料にまとめたい」という、相当ざっくりした依頼を投げてみると、次のような質問が返ってきます。

* この資料は誰向けか（経営層、エンジニア、一般社員、一般公開）
* 用途は何か（研修、提案、社内共有、プレゼン）
* レイアウトの方向性（情報密度高め、ビジュアル重視、テキスト重視）
* 含めたい要素（従来のチャットとの違い、構成要素、動作フロー）
* ビジュアルの方向性（落ち着いた雰囲気、モダン、テクノロジー寄り）

こうした質問が、単なる自由記述ではなく「選択肢つきの質問」として提示される点がポイントです。選ぶだけで意図が明確になっていくため、非デザイナーが陥りがちな「いきなり変な方向に走り出す」事故を防げます。

これは、Claudeがこれまで膨大なユーザーとのやり取りを通じて蓄積した「デザインアウトプットを作るときにユーザーが暗黙に持っている意図」を、構造化された質問に落とし込んでくれているとも言えます。結果として、優秀なディレクターや営業担当者が横についてくれるのに近い体験になります。

## Claude Designの強み③：バリエーションの一括生成

デザインの難しさは「パッと見ないとどれがいいか判断できない」ことにあります。言葉でどれだけ意図を詰めても、最終的には並べて比較しないと選べない。

この課題に対してClaude Designは、**最初から複数のバリエーションを一気に並べて見せてくる**というアプローチを採用しています。

たとえば既存のアンケート結果ページをリニューアルしたいと依頼すると、以下のような複数パターンを一度に提示してくれます。

1. エディトリアル型（雑誌風の読ませるレイアウト）
2. ヒーロークォート＋ダッシュボード型
3. ストーリータブ型
4. ストーリー縦型（手動スクロール）
5. カード中心のグリッド型
6. ミニマル1カラム型

この段階ではまだ細部は詰まっていませんが、大枠の構造が6種類並ぶので、「3番をベースに、1番のヒーロー部分のテイストを混ぜたい」といった判断が非デザイナーでもできるようになります。選んだベースに対して、さらに細部を詰めていくフェーズに進む、という二段階のアプローチは、従来のAIデザインツールには少ない設計です。

初稿が気に入っても、現実にはそのまま使えることはほぼありません。「ここのフォントをもう少し大きく」「このカード、もっと詰めて」といった微調整の往復が発生します。

Claude Designには、こうした微調整を効率化するための仕組みとして **Tweaks**（ツイクス）と **Inline Comment**（インラインコメント）が用意されています。

### Tweaksによるスライダー調整

Tweaksは、Claude自身が「このプロジェクトで調整したくなりそうなポイント」を先回りして抽出し、スライダーやトグルとしてUIに用意してくれる機能です。スライドプロジェクトなら、

* フォントサイズ
* 配色のプリセット
* 情報密度（スカスカか、びっしりか）

ウェブページプロジェクトなら、

* カードの密度
* グラフをカード内に埋め込むかどうか
* フィルターUIのスタイル（チップ形式 vs ドロップダウン）
* セクションの順序（導入→詳細→CTAの並び順）

といった具合に、**プロジェクトの種類ごとに動的に違うTweaksが用意される**のが特徴的です。これは前述の「意図確認の質問」の段階で、「この資料では何を可変にしたいか」も同時にClaudeが把握しているから可能になっています。

キャンバス上の任意の要素をクリックしてコメントを書き込むと、その要素に対する指示としてClaudeが解釈してくれます。「この部分のアイコンを削除」「このカードだけ背景色を変えて」といった局所的な修正を、チャットで全体を再生成することなく実現できます。

| 調整方法 | 得意な場面 | 注意点 |
| --- | --- | --- |
| チャット | 全体のトーン変更、構造の入れ替え | 局所修正には向かない |
| Tweaks | パラメトリックな調整（色、余白、密度） | 用意された軸しかいじれない |
| Inline Comment | 特定要素のピンポイント修正 | ときどきコメントが消える事象あり |
| Direct Edit | テキストの直接書き換え | レイアウト崩れに注意 |

公式サポートページにも記載されていますが、リサーチプレビュー段階ではインラインコメントがClaudeに読まれる前に消えてしまうことがまれにあります。消えた場合はコメント内容をコピーしてチャットに貼り直す、という回避策が案内されています。

## Claude Designの強み⑤：アウトプットのクオリティ

ここまで仕組み面を中心に説明してきましたが、実際の出力品質そのものもかなり高い水準にあります。Claude Opus 4.7 は、Anthropicが一般提供しているモデルの中で最も性能の高いビジョン対応モデルと位置付けられており、Claude 4系から定評のあった「ウェブサイトやUIデザインの生成品質」がさらに強化されています。

実際に使ってみて感じる品質上のポイントは以下の通りです。

* **配色バランスが破綻しない**：非デザイナーがやりがちな「色を使いすぎてうるさくなる」状態を自動で避けてくれる
* **余白の取り方が詰まりすぎない**：行間や余白が詰まりすぎず、読みやすさを担保している
* **タイポグラフィの階層が明確**：見出し・小見出し・本文のコントラストが自然
* **アイコンや装飾が用途に合っている**：コンサル資料なら落ち着いた線画アイコン、マーケ資料ならカラフルなイラスト、など使い分けがうまい

もちろん、ハイエンドのデザイナーが作るような「息を呑むほど美しい」アウトプットには及びません。しかし、「社内会議で出して恥ずかしくない水準」「外部の顧客に送れる水準」は十分に満たしています。ここをAIだけで通過できるのは、非デザイナーにとって大きな変化です。

## Claude Designの強み⑥：対応する制作物のレンジの広さ

Claude Designは、一つのツールで対応できる制作物のレンジが非常に広いのも特徴です。

* プレゼンテーション・スライド
* ランディングページ・ウェブページのワイヤーフレーム
* インタラクティブプロトタイプ（クリック可能、フォーム動作あり）
* マーケティング素材（ヒーロービジュアル、バナー）
* アプリケーションUI（モバイル、デスクトップ両対応）
* SVGアイコン・イラスト
* HTMLアニメーションによる動画風コンテンツ
* フォーム・ダッシュボード・内部ツールのモック

特筆すべきは、**HTMLベースで「動くもの」をそのまま出せる**点です。静的なモックアップではなく、クリックするとページが遷移したり、フォームを入力するとバリデーションが効いたり、といった振る舞いまでそのまま動きます。これは、デザイナーにとっても、ユーザーテストのためにわざわざ別途プロトタイピングツールに起こす手間が省けるという実益があります。

## Claude Designの強み⑦：「次の工程」への接続設計

Claude Designが他のAIデザインツールと決定的に違うのは、**作ったあとの接続**を真剣に設計している点です。

### エクスポート先の広さ

作成したアウトプットは、右上のExportボタンから以下の形式・サービスに書き出せます。

| エクスポート先 | 主な用途 |
| --- | --- |
| ZIPファイル | ローカルでの保管・他ツールへの受け渡し |
| PDF | 共有・印刷・ステークホルダーレビュー |
| PPTX（PowerPoint） | 社内プレゼン、既存のスライドテンプレートへの組み込み |
| Canva | ノンデザイナーによる仕上げ編集 |
| HTML（単一ファイル） | ウェブ公開、ホスティング |
| Claude Codeハンドオフ | 本番実装へのバトンタッチ |

なお、PPTXエクスポートについては日本語を含むスライドで一部、編集不可の画像貼り付け形式に落ちてしまう事象が筆者の環境では確認されています。リサーチプレビュー段階のため、このあたりは今後の改善に期待したいポイントです。

### Claude Codeへのハンドオフ

最も強力なエクスポート先が、**Claude Codeへのハンドオフ**です。デザインが固まったら、Claude Designはプロトタイプをコンポーネント、デザイントークン、コピー、インタラクション情報を含む「実装バンドル」としてパッケージ化し、Claude Codeに一発で引き渡せます。

これにより、以下のような流れが実現できます。

1. Claude Designで、ワイヤーフレームから見た目まで固める
2. バリエーションを比較しながらベースを確定
3. Tweaksとインラインコメントで細部を詰める
4. Claude Codeハンドオフを使って、実装バンドルを生成
5. 手元のリポジトリでClaude Codeを起動し、バンドルを渡して実装させる
6. Next.js、React、Vueなど、フレームワークは指定可能

筆者も実際に、既存のイベント案内ページをClaude Designで作り直し、そのバンドルをClaude Codeに渡してリニューアルまで1日で完了させた経験があります。従来であればワイヤーフレーム作成だけで数日かかっていた作業が、数時間に圧縮される感覚です。

## 実践：Claude Designを業務に取り入れる際の手順

ここからは、実際にClaude Designを業務フローに組み込む際の具体的な手順を紹介します。筆者自身の実践から得られた、つまずきやすいポイントと合わせて解説します。

### ステップ1：デザインシステムを最初に構築する

いきなりスライドやページを作り始めず、まず自分の組織・プロジェクトのデザインシステムを作ります。以下のいずれかの方法で素材を用意しておきましょう。

```
必要な素材（いずれか）
・会社ウェブサイトのURL、もしくはスクリーンショット（PC版とモバイル版両方あると理想）
・会社ロゴ（SVGまたは高解像度PNG）
・既存のブランドガイドライン資料（PDF、PPTX）
・GitHubのフロントエンドリポジトリURL
```

Claude Designの初期画面で「Design System」のメニューを選び、これらの素材をアップロードまたはリンク指定します。生成されたデザインシステムは、あとで色やフォントの差し替えも可能なので、完璧を目指さず「7割の精度」で先に作ってしまうのがおすすめです。

### ステップ2：最初のプロジェクトはスライドから始める

いきなりウェブサイトを作ろうとすると、構造が複雑すぎてTweaksでどこを調整していいか判断しづらくなります。最初はスライドや1ページ資料のような、**一次元的な構造のプロジェクト**から始めるのがおすすめです。

以下は、スライド生成時のプロンプトテンプレート例です。

```
目的：[何のための資料か]
読者：[誰に見せるか、専門性レベル]
長さ：[スライド枚数の目安]
含めたい要素：
  ・[要素1]
  ・[要素2]
  ・[要素3]
トーン：[フォーマル/カジュアル/テクニカルなど]
デザインシステム：[プロジェクトで設定済みのシステム名]
```

このテンプレートに沿って依頼すると、確認質問も少なめで済み、初稿が意図から大きく外れにくくなります。

### ステップ3：バリエーションを比較し、ベースを決める

初稿が複数並んで出てきたら、一つを選ぶ前に以下を確認します。

* **情報階層**：見出し→本文→補足の流れが追いやすいか
* **視線誘導**：重要な情報が最初に目に入る配置になっているか
* **余白のバランス**：詰まりすぎ・空きすぎになっていないか
* **ブランド整合性**：デザインシステムの色・フォントが適用されているか

選んだベースに対して、「このパターンをベースに、2番目のヒーローセクションの雰囲気を混ぜて」のように、他のバリエーションのいいとこどり指示を出すと、さらに良い方向にブラッシュアップできます。

### ステップ4：Tweaksとインラインコメントで詰める

ベースが固まったら、まずTweaksを一通り触って大枠の調整を終えます。Tweaksで対応できないピンポイント修正は、インラインコメントに切り替えます。

ここで重要なのは、**一度に複数のコメントを投げすぎない**ことです。Claude Designはコメントを一つずつ処理しますが、レイアウトに影響する修正を同時に複数投げると、意図しない副作用が発生することがあります。1つのコメントごとに結果を確認しながら進めるほうが、手戻りが少なくなります。

### ステップ5：エクスポートと次工程への接続

アウトプットが固まったら、用途に応じてエクスポート先を選びます。

| 用途 | 推奨エクスポート先 |
| --- | --- |
| 社内共有（確認してほしい） | URL共有、PDF |
| プレゼン（当日投影する） | PPTX、PDF |
| 外部納品（編集してもらう） | Canva、PPTX |
| ウェブ公開 | HTML |
| 本番実装 | Claude Codeハンドオフ |

Claude Codeハンドオフを使う場合は、エクスポート時に生成される指示テキストをそのままコピーし、手元のClaude Codeセッションに貼り付けるだけで、実装ワークが始まります。

```
# Claude Code でハンドオフバンドルを受け取る例
# ハンドオフ指示テキストをコピーした状態で、プロジェクトディレクトリへ
cd ~/projects/my-app

# Claude Code を起動し、クリップボードの内容を指示として渡す
claude "<ここにClaude Designからコピーしたハンドオフ指示を貼り付け>"
```

## つまずきポイントとトラブルシューティング

リサーチプレビュー段階のため、現時点で以下のような事象に遭遇することがあります。

| 事象 | 対処法 |
| --- | --- |
| インラインコメントがClaudeに読まれる前に消える | コメント内容をチャットに貼り直す |
| コンパクトビューで保存エラーが出る | フルビューに切り替えてから再試行 |
| 大きなリポジトリをリンクすると動作が重い | サブディレクトリだけをリンクする |
| PPTXエクスポートで日本語が画像化される | 画像化前提で受け取り、Canvaで仕上げに回す |
| Claude Design専用枠を使い切る | 週次リセットを待つ、もしくはTeam/Enterpriseプランを検討 |
| Tweaksが期待どおりに効かない | チャットで「この要素を変更して」と指示に切り替える |

公式サポートページでも、大きなモノレポを丸ごとリンクするのではなく、関連するサブディレクトリだけを指定することが推奨されています。

## Anthropicの戦略：Claude Designが示す「AIによる仕事OS化」

ここからは、Claude Designを単体のプロダクトとしてではなく、**Anthropicの事業戦略の文脈**で捉え直してみます。

### Anthropic Labsという実験場

Claude Designは、Anthropic LabsというAnthropic内の実験部門から出てきたプロダクトです。Anthropic Labsは、Claudeの基盤モデル能力を実際のユースケースで試すためのプロダクトを量産している部門で、過去には以下のようなプロダクトを生み出してきました。

* **Claude Code**：ターミナル上で動くエージェントコーディングツール。現在は独立したプロダクトとして急成長中
* **Claude Cowork**：ホワイトカラー向けの汎用エージェント。ファイル管理やタスク自動化を担う
* **Claude for Chrome**：ブラウザ操作を自動化するエージェント
* **Claude for Excel / PowerPoint**：MicrosoftのOffice系ツール上で動くエージェント

いずれも最初はリサーチプレビューでローンチし、ユーザーの反応を見ながら本プロダクト化していくという流れを取っています。Claude Codeが成功事例の代表で、エンジニアの業務に深く入り込んだ結果、Anthropicの収益を大きく押し上げる柱になりました。

### Claude Designが狙うポジション

Claude Code がエンジニア、Claude Cowork がリサーチャーやオペレーター、そして Claude Design は **デザイナー相当の役割**を担う、という構図が見えてきます。

| プロダクト | 代替する職能 | 主な出力物 |
| --- | --- | --- |
| Claude Code | エンジニア | 動くコード、デプロイ可能なアプリ |
| Claude Cowork | リサーチャー・オペレーター | 調査結果、業務自動化、タスク実行 |
| Claude Design | デザイナー・ディレクター | ビジュアルアウトプット、プロトタイプ |

これら3つを組み合わせると、「調査→アウトプットデザイン→実装」という、ホワイトカラー業務の主要な流れをAIだけで完結できるようになります。Anthropicは単発のAIツールを作っているのではなく、**仕事のオペレーティングシステム**を構築しようとしている、と読むのが自然でしょう。

### Figma・Canvaとの関係

Claude Designのリリースに合わせて、Figmaの株価が一時的に下落するという動きがありました。Mike Krieger氏（Anthropic CPO、元Instagram共同創業者・CTO）が、同時期にFigmaの取締役を退任したことも話題になっています。

ただし、Anthropic自身は「Figma/Canvaの完全な代替を目指しているわけではない」というスタンスを取っています。実際、Claude DesignのエクスポートメニューにはCanva、PPTX、HTMLといった多様な出力先が用意されており、特にCanvaに関してはAnthropicとCanvaが共同で連携を深めていることがCanva側からも発信されています。

つまりAnthropicの構想は、

* 「入り口から初稿までの、最も時間のかかる部分」をAIで圧縮する
* 「最終仕上げや、マルチプレイヤー編集など、既存ツールが強い領域」は連携する

という棲み分けで、デザインワークフロー全体のエントリーポイントを押さえにいく、というものだと考えられます。

### 現時点でのClaude Designの限界

一方で、現時点のClaude Designには明確な限界もあります。

* **マルチプレイヤー編集がない**：Figmaのように複数人が同時に同じファイルを編集することはできない。TeamプランではワークスペースレベルのシェアリングはあるがリアルタイムCo-editingはサポートされていない
* **公開APIがない**：ローンチ時点ではパブリックAPIがなく、デザイン生成の自動化や他ツールへの埋め込みはできない
* **ベクターツールとしての成熟度**：細かいパスの編集、カスタムイラストレーションといった領域は、既存のベクターツールに及ばない
* **リサーチプレビューゆえの不安定さ**：機能や制限が頻繁に変わる可能性があり、商用のクリティカルな用途には向かない

これらは、あくまで初稿生成〜初期プロトタイプというポジションでClaude Designを使い、仕上げや運用は別ツールに任せる、という使い分けで対応できます。

## まとめ：使い始めるべき理由と、そのタイミング

Claude Designは、非デザイナーがデザイナーに近いアウトプットを量産できる世界を現実のものにしつつあります。特に以下のような状況にある人は、いますぐ触ってみる価値があります。

* Claudeの有料プラン（Pro、Max、Team、Enterprise）をすでに契約している
* スライド資料、提案書、ランディングページなどを作る機会が多い
* Figmaなどの専門ツールを覚える時間はないが、見た目は整えたい
* Claude Codeとの連携で、プロトタイプから実装まで一気に流したい
* 自分一人、または小規模チームでプロダクトを作っている

一方で、以下のような方は、現時点では無理に飛びつく必要はないかもしれません。

* 複数人で同時編集する大規模なデザインチームを運用している
* APIやスクリプト経由でのデザイン自動化が必須
* 細密なイラストレーションやブランドデザインを業務の中心にしている
