【Claude Opus 4.7 が GA になって、4/23 から API と Enterprise のデフォルトも 4.7 に切り替わる。モデル選定が「使う側が乗り換えを決める」から「提供側がデフォルトを差し替える」方向に寄ってきたのが、ここにきて鮮明な気がする】

料金は Opus 4.6 と据え置きで、コーディング・長時間タスク・画像の高解像度読み取りが強化された。うちの案件では仕様書→実装の流れで使うので、特に仕様からまとまった実装を生成させる場面で効いてくる想定。

モデルの切り替わり方が、半年単位の乗り換え判断から、数週間単位で気付いたらデフォルトが変わっているリズムに寄ってきた。運用側は特定モデルにチューニングし切らず、切り替えに耐える仕様とプロンプトの書き方を先に設計しておく方が、中期では楽な気がしている。
