「営業リスト、どこから買ってますか？」

この質問を受けるたびに、答えに困る。

うちは買っていない。**全部、自分で作っている。**

そう答えると、「えっ、どうやって？」と必ず聞き返される。

この記事はその全手順を公開する。3部作の第2部。第1部で書いた「3社分業モデル」の最重要パーツ、**営業リスト作成**の中身を全部見せる。

※社名は前回同様、A社（本部・私の会社）・B社（商材元）・C社（実働送信部隊）で表記する。

### 営業リストを買おうとした日

3社分業モデルを動かすには、毎月数百〜数千社分の営業リストが必要だ。

最初は素直にリスト業者に見積もりを取った。3社に問い合わせた。

返ってきた金額：

* A業者：1,000社で10万円
* B業者：1,000社で15万円
* C業者：1,000社で30万円

「高い」と思った。でも、それより気になったのは中身だった。

「公式サイトのURLは入ってますか？」と聞くと、「半分くらいです」と返ってくる。  
「問い合わせフォームのURLは？」と聞くと、「データにはありません」と返ってくる。  
「データの鮮度は？」と聞くと、「2年前のデータが多いです」と返ってくる。

10万円払って、半分が空欄で、2年前のデータを買う。

**これは投資じゃない。浪費だ。**

### 「自分で作るしかない」

リスト業者から離れて考えた。

営業リストの元データは、どこから来ているのか。

調べたら、答えは1つだった。**国税庁の法人番号公表サイト**だ。

日本の全法人が登録されている。会社名、本店所在地、法人番号、設立日、廃止情報。全部入っている。完全に無料で公開されている。

「これを自分でダウンロードして整形すれば、買わなくて済む」

そう思って、Claudeに話しかけた。

「国税庁の法人番号サイトから、47都道府県の全法人を自動でDLしてDB化したい」

ここから営業リスト事業の構築が始まった。

### Step 1：国税庁データを8都県分取り込んだ

Claudeに頼んでPythonスクリプトを書いてもらった。

db/download\_houjin.py というファイルを作って、都道府県コードを指定するとCSVを自動でDLする仕組みにした。

```
python -m db.download_houjin 13   # 東京都
```

python -m db.download\_houjin 23 # 愛知県

python -m db.download\_houjin 27 # 大阪府

たったこれだけで、都道府県別の全法人CSVが手元に来る。

最初に始めたのは愛知県。**24万社のデータが1日で揃った**。

そこから8都県に拡大した。

**都道府県法人数**東京都708,757愛知県240,865神奈川県約20万埼玉県約16万千葉県約14万静岡県約11万岐阜県約8万三重県約7万**合計約192万社**

これが「営業リストの素材」になった。

### Step 2：廃止・解散・吸収合併を自動除外

CSVをそのまま使うと、危険な落とし穴がある。

**「過去に存在したけど今はない会社」が大量に含まれている**ことだ。

廃止、解散、吸収合併、承継。これらをそのまま営業リストに使うと、「もうやってませんよ」と言われる電話を毎日かけることになる。

国税庁CSVには「処理区分」という列がある。21=廃止、22=解散、71=承継。これを使って自動除外した。

**項目件数**元CSV（愛知）285,333行廃止・解散・承継44,468社（除外）**取込済み（現役のみ）240,865社**

データの鮮度は国税庁が毎月更新しているので、常に最新だ。

### Step 3：商号から業種を自動分類

ここで一番悩んだ。

**国税庁のデータには業種が入っていない。** 会社名と住所と法人番号だけ。業種別の営業リストを作りたいのに、業種がわからない。

Claudeに相談したら、こう返ってきた。

「商号（会社名）から業種を推定できます」

なるほど、と思った。

〇〇建設 〇〇工務店 〇〇工業 〇〇土木 〇〇設備 〇〇内装 などの語が含まれていれば建設業の確率が高い。〇〇商事 〇〇貿易 なら卸売。〇〇クリニック なら医療。

13業種のキーワードリストを作って、商号にどれが含まれているかで自動分類した。

愛知県24万社の業種分布はこうなった。

**業種件数比率建設業10,3204.3%**製造業9,6344.0%卸売8,8093.7%小売7,2003.0%飲食5,4002.2%医療3,2001.3%

8都県全体だと、建設業だけで**約78,000社**。これだけで営業リストの素材として十分すぎる。

### Step 4：gBizINFO APIで信頼性データを乗せた

会社名と住所だけでは、まだ営業に使えない。

「この会社、規模はどれくらい？」「いつ設立？」「資本金は？」がわからないと、提案が刺さらない。従業員5人の会社と500人の会社では、同じ商材を持っていっても伝え方が違う。

そこで**gBizINFO API**を叩いた。経済産業省が無料で提供している企業情報APIだ。

```
# db/gbiz_enrich.py
```

法人番号を渡すと以下が返ってくる：

- 従業員数

- 資本金

- 設立年

- 業種コード（公式分類）

- 事業内容

これで「会社名・住所・業種・規模・設立年」が揃った営業リストになった。

### Step 5：エンリッチメント（公式サイト・電話・代表者名）

最後のステップ。

営業するには、公式サイトのURL・電話番号・代表者名が要る。問い合わせフォームを使うか、代表者宛に郵送するか、電話するか、判断できる。

これも自動化した。

会社名+住所でGoogle検索 → 公式サイト特定 → サイト内のお問い合わせページから情報抽出。

検証として30社で試したら、29社で成功した。**成功率97%**。

**項目取得成功**公式サイト30/30（100%）電話番号28/30（93%）代表者名27/30（90%）**総合（全項目取得）29/30（97%）**

### Step 6：A社のターゲット条件で100社に絞り込む

ここまでで「会社名・住所・業種・規模・設立年・公式サイト・電話・代表者名」が揃った192万社のDBが完成した。

これを、A社（本部）の今のターゲットでフィルタする。

**ターゲット条件：**

* エリア：愛知県
* 業種：建設業
* 規模：従業員11〜50人
* 公式HP：あり
* 問い合わせフォーム：あり
* 設立：5年以上

このフィルターを通すと、こうなった。

```
愛知県全法人 240,865社
```

↓ 業種フィルタ：建設業

建設業 10,320社

↓ 規模フィルタ：11〜50人

中規模建設業 約2,000社

↓ 公式HP有り

HP保有 約1,200社

↓ 問合せフォーム有り

フォーム有 約400社

↓ 設立5年以上

ターゲット 約300社

↓ 最終手作業チェック

最終リスト 100社

24万社から100社まで絞り込んだ。

これがA社の最初の営業リストになる。

### この仕組みのすごいところ

**① コストがほぼゼロ**

リスト業者なら10万円かかるリストが、Pythonスクリプトを動かすだけで完成する。サーバー代も無料（自分のPC）。実質0円。

**② データの鮮度が常に最新**

国税庁が毎月CSVを更新する。月1回スクリプトを走らせれば、常に最新の現役企業リストが手元にある。「2年前のデータ」を買うことがない。

**③ 条件を変えればいくらでも作れる**

「東京の製造業100人以下」「大阪の小売10人以下」など、条件を変えるだけで違うリストが瞬時に出る。1リスト10万円じゃなく、1リスト数秒。

**④ 営業リスト自体が事業になる**

A社の中だけで使うのはもったいない。同じリストを**ココナラなどで販売**できる。「〇〇県の〇〇業100社リスト 5,000円」みたいなサービスだ。

### 副次的に立ち上がった「営業リスト販売事業」

リーパ販売戦略のためだけに作ったDBだったが、気づいたら**ココナラで販売できる商品**になっていた。

ココナラには「営業リスト作成代行」のカテゴリがある。3万円〜10万円の価格帯で売られている。

A社のDBがあれば、注文を受けてから30分でリストを納品できる。利益率はほぼ100%。

これが「リスト事業の単独収益化」の正体だ。

A社は今、\*\*「リーパ販売の最上流」と「ココナラでのリスト販売」\*\*の両方からリストを使っている。同じインフラを2つの収益源に使う、典型的なプラットフォーム戦略になった。

### Step 7：A社のリストをC社に渡す（次回予告）

ここまででリストが完成した。

次のフェーズが、第3部のテーマだ。

**「100社のリストを、どうやって送信するか」**

A社（本部）が作った100社リストを、C社（実働部隊）に渡す。C社が問い合わせフォームから1社ずつ送信する。反応があった企業を記録する。それをB社（商材元）に引き渡す。

このフローを支える**フォーム送信システム**の中身を、第3部で全公開する。

* 1日1000社送信を可能にした仕組み
* メール文面テンプレートの管理方法
* 送信ログとダッシュボード（port 8505）
* 反応率の可視化

これが3部作の最後の山だ。

### 中小企業の社長に伝えたいこと

「営業リストを買う」が当たり前になっている人へ。

**買うのをやめてほしい。**

国税庁の公開データから、無料で、最新の、自社専用リストが作れる。Claudeに話しかけてスクリプトを書いてもらえば、コードゼロで完成する。

リストを「買うもの」じゃなく「作るもの」にした瞬間、営業の景色が変わる。

そして、自社で作ったリストはそのまま新規事業（リスト販売）にもなる。1つのインフラを複数の収益源に使える。
