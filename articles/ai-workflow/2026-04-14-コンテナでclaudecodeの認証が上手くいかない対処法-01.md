## DevcontainerでのClaudeCode認証

先日、DevcontainerでClaudeCodeをホストと隔離して運用していると記事を書きました。

が、あの構成ではClaudeCodeの認証が実は上手くいっておらず毎回リビルドのたびにトークンをコピペしていました...（本来正しくマウントできていれば一度の認証で認証情報がホスト側に書き込まれる）

それで認証は出来ているし、個人用途なのでまぁ良いかと思っていたのですがClaudeCodeでトークンのコピペが上手くいかない不具合が発生したためこの方法ではClaudeCodeが使えなくなってしまいました。

対処法の備忘録として残しておきます。

### 新しい構成

```
{
  "name": "project-name",
  "build": {
    "dockerfile": "../../.devcontainer-image/Dockerfile"
  },
  "features": {
    "ghcr.io/devcontainers/features/node:1": {
      "version": "20"
    }
  },
  "postCreateCommand": "npm install -g @anthropic-ai/claude-code",
  "workspaceFolder": "/workspace",
  "mounts": [
    "source=${localWorkspaceFolder},target=/workspace,type=bind,consistency=cached",
    "source=${localWorkspaceFolder}/../.claude,target=/home/vscode/.claude,type=bind,consistency=cached",
    "source=${localWorkspaceFolder}/../.claude.json,target=/home/vscode/.claude.json,type=bind,consistency=cached"
  ],
  "remoteUser": "vscode"
}
```

前回記事からの変更点は

```
"source=${localWorkspaceFolder}/../.claude.json,target=/home/vscode/.claude.json,type=bind,consistency=cached"
```

を追記したのみ。

### 何が問題だったか

ClaudeCodeは認証に必要なファイルが2箇所に分散しており、片方しかマウント出来ていなかったこと。~/.claude.jsonがマウント出来ていないためホストに認証情報を書き込めておらず、コンテナをリビルドするたびにオンボーディングが発生していた。

本来は正しくマウントすれば解決すると思うのですが、不具合が解決するまでの対処法として

1. ホストに一時的にClaudeCodeをインストール

```
npm install -g @anthropic-ai/claude-code
```

2. 長期トークン生成

ブラウザが開くので認証を完了する。ターミナルに sk-ant-oat01-... から始まるトークンが表示される。

3. credentials.jsonに保存

```
cat > ~/dev/.claude/.credentials.json << EOF
{"claudeAiOauth":{"accessToken":"ここにトークン","refreshToken":"","expiresAt":$(date -v+1y +%s)000,"scopes":["user:inference","user:profile"]}}
EOF
```

4. claude.jsonを作成

```
echo '{"hasCompletedOnboarding":true}' > ~/dev/.claude.json
```

これがなかったので毎回オンボーディング発生していました

5. Claude Codeをアンインストール

```
npm uninstall -g @anthropic-ai/claude-code
```

ホストで動かしたくないため

トークンの期限は1年程度なので、切れたらまた同じ手順が必要だと思います。
