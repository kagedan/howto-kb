訪問介護の記録業務、1日30分〜1時間かかっていませんか。

私も現場にいた頃は、1件の訪問が終わるごとにパソコンの前で手が止まっていました。

この記事では、看護師8年・介護福祉士7年の現場経験を経て医療ライターになった私が、実際にClaudeを使って記録時間を半分にできた具体的なプロンプトを5つご紹介します。

大切な前提を1つだけ先にお伝えすると、AIは「記録を肩代わりする道具」ではなく「書き出しを助けてくれる相棒」です。最後は必ず人の目で確認する──この一点だけは、どのプロンプトにも共通する鉄則です。

## はじめに:AIに入れていいこと・ダメなこと

具体的なプロンプトをお伝えする前に、3つだけ前提を共有させてください。現場のリスクに直結する話なので、ここは省略できません。

### 1. 個人情報は絶対に入れない

利用者様のお名前・住所・病名・家族構成など、個人を特定できる情報はAIに入れない。これは例外なしです。

Claude を含め多くの生成AIは、入力内容をサービス改善のために参照する可能性があります(有料プランでは制限できる場合もありますが、事業所として利用する場合は、必ず利用規約の確認を)。

現場で使うときは、以下のように抽象化してから入れます。

* 利用者A様(80代女性、要介護3)
* 家族構成:同居のご長男
* 疾患:ある慢性疾患

### 2. 出力は「下書き」、最終確認は必ず人の目で

AIは、もっともらしい嘘(ハルシネーション)を自信満々に書くことがあります。

「日付を間違えて出力した」  
「ありもしない診断名を付け加えた」

こうした例は、実際に私も経験しました。AIの出力はあくまで下書き、最終的な記載は必ずご自身の目で確認してください。

### 3. 事業所のルールを確認する

「AI を業務で使っていいか」は事業所によって方針が分かれます。個人判断で使い始めず、まずは管理者・責任者に相談するのが望ましいといえます。

前提は以上の3点です。ここからは実際に使えるプロンプトを1つずつご紹介します。

## プロンプト1|ヘルパーメモをケアマネ向け報告文に整形

訪問介護のヘルパーさんから上がってくるメモ。箇条書きや短い文で、事実の羅列になっていることが多いですよね。

それをケアマネジャー向けの報告文として整える作業、意外と時間がかかります。私がサービス提供責任者として現場にいた頃は、1件あたり10〜15分かかっていました。

Claude を使うと、このくらいの時間配分に変わります。

### 元になるヘルパーメモの例

```
利用者A様(80代女性、要介護3) 訪問日:4/20
・体温 36.5℃、血圧 128/78
・朝食7割摂取、水分500ml
・トイレ誘導3回、失禁なし
・ご家族から「最近便秘気味」と相談あり
・下剤の使用について次回ケアマネに相談予定
```

### プロンプト(このままコピペで使えます)

```
以下は訪問介護ヘルパーの訪問記録メモです。
ケアマネジャー向けの報告文として、
以下の条件で整理してください。

・敬体(です・ます調)
・200字以内
・結論を最初に書く
・事実と相談事項を分けて記載
・ヘルパー目線ではなくサービス提供責任者の立場で

【メモ】
(ここに上記メモを貼り付け)
```

### Claudeの出力例

```
A様の訪問記録をご報告いたします。

体温・血圧ともに安定されており、食事摂取量も良好です。
トイレ誘導も3回実施し、失禁は見られませんでした。

ご家族より「最近便秘気味」とのご相談をいただいております。
次回、下剤の使用方法について
ケアマネジャー様にご相談させていただきたく存じます。
```

### 時短効果

手作業で15分かかっていた報告文が、約3分に。特に「結論を最初に」という構造の指示は、AI が苦手な「優先順位付け」を肩代わりしてくれます。

### 使うときのコツ

プロンプトの「以下の条件で」の部分を事業所のルールに合わせてカスタマイズすると、精度が一段上がります。

たとえば、医療依存度が高い事業所なら「バイタル変動は冒頭に明記」を条件に加えるなど。

## プロンプト2|利用者ご家族への連絡文を丁寧語で作成

ご家族への連絡文、「言い回しに悩む」ことありませんか。

* 近況報告の連絡
* 軽微な体調変化のお知らせ
* サービス内容の相談事項

現場の事実をそのまま書くと事務的になりますし、気を遣いすぎると回りくどくなる。そのバランスが、意外に難しいと感じていました。

### 元になるメモの例

```
A様(80代女性)、本日訪問
・体温 36.8℃(平熱内だがやや高め)
・朝食の進みがいつもより遅い
・「昨晩あまり眠れなかった」と発言あり
・次回訪問まで様子観察の方針
```

### プロンプト

```
以下は訪問介護の記録メモです。
ご家族様にお送りする連絡文として、次の条件で整理してください。

・敬体、300字以内
・心配しすぎない、落ち着いたトーン
・事実→現状の判断→次回の対応、の順
・専門用語は使わない
・末尾に「何かご不明点があればお気軽に」の一文

【メモ】
(ここに貼り付け)
```

### Claudeの出力例

```
A様のご家族様へ

本日の訪問の様子をお伝えいたします。

朝食の進みがいつもより少しゆっくりで、
「昨晩あまり眠れなかった」とのお話がございました。
体温は36.8℃と平熱の範囲内です。

今のところ様子観察で対応しておりますが、
次回訪問時にも引き続き状態を確認させていただきます。

何かご不明点があれば、お気軽にご連絡ください。
```

### 時短効果

1通あたり15〜20分かかっていた連絡文が、約5分に。トーン調整で迷う時間がほぼゼロになります。

### 使うときのコツ

事業所として伝える姿勢(「心配させすぎない」か「事実を詳しく」か)をプロンプトに明記しておくと、文章の空気感が安定します。

## プロンプト3|サービス担当者会議の議事メモを要約

サービス担当者会議の議事録、作成するまでが仕事ですよね。

録音から起こしたメモ、あるいは手書きの走り書き。これを会議後の関係者配布用に整えるのが、地味ですが時間を食う作業です。

### 元になる議事メモの例

```
サ担会議(A様、居宅支援、訪介、訪看、デイ参加)
・現状:ADL(日常生活動作)やや低下傾向
・課題:トイレの失敗が月3回程度に増加
・デイ:入浴時の見守り継続
・訪看:服薬状況の確認強化
・次回モニタリング:5月中旬
```

### プロンプト

```
以下はサービス担当者会議の議事メモです。
関係者配布用の議事録として整理してください。

・敬体、400字以内
・参加事業所/現状認識/今後の方針/次回予定の4項目
・箇条書きと文章を適切に混ぜる
・専門用語は初出でカッコ展開(ADL→ADL(日常生活動作))

【メモ】
(ここに貼り付け)
```

### Claudeの出力例

```
A様 サービス担当者会議 議事録

【参加事業所】
居宅介護支援事業所、訪問介護、訪問看護、通所介護

【現状認識】
ADL(日常生活動作)にやや低下傾向が見られ、
トイレの失敗が月3回程度に増加している状況です。

【今後の方針】
- 通所介護:入浴時の見守りを継続
- 訪問看護:服薬状況の確認を強化
- 訪問介護:トイレ誘導の回数見直しを検討

【次回予定】
5月中旬にモニタリングを実施予定です。
```

### 時短効果

手作業で30分かかっていた議事録整理が、約8分に。

### 使うときのコツ

参加事業所の略称(訪介=訪問介護、訪看=訪問看護 等)をプロンプトに含めておくと、初出展開の精度が上がります。

## プロンプト4|月次モニタリング報告書の下書き生成

月次モニタリング報告、ケアマネジャーさんにとっては毎月の定番業務。1件あたり30分〜1時間かかる、重い作業です。

AIに全部任せるのは危険ですが、「下書きを作らせて、そこから自分が整える」という使い方なら大きな時短になります。

### 元になる情報の例

```
A様 2026年4月分モニタリング素材
・訪問介護:週3回(月・水・金)、身体介護中心
・利用頻度の変化:なし
・ADL:食事自立、排泄一部介助、入浴全介助(前月と同じ)
・新規の課題:夜間の不眠の訴えが増えている
・家族からの要望:デイの日数を週2→週3に増やせないか
・次月の目標候補:夜間対応の検討、デイ追加の調整
```

### プロンプト

```
以下は訪問介護事業所の月次モニタリング素材です。
ケアマネジャー向けの報告書下書きとして整理してください。

・敬体、600字以内
・以下の4項目立て
  1. 利用状況
  2. ADLと変化
  3. 新規の課題とご家族要望
  4. 次月の目標案
・判断・評価はサービス提供責任者の所見として書く
・「〜と考えられます」「〜が望ましいといえます」のトーン

【素材】
(ここに貼り付け)
```

### 出力例(抜粋)

```
A様 2026年4月 モニタリング報告

【利用状況】
週3回(月・水・金)の訪問介護を、前月から変更なく
継続して利用いただいております。身体介護を中心とした
サービス提供となっております。

【ADLと変化】
食事は自立、排泄は一部介助、入浴は全介助と、
前月との状況に大きな変化はございません。

【新規の課題とご家族要望】
夜間に不眠の訴えが増えていることが気になる点です。
ご家族様より、通所介護の日数を週2回から週3回へ
増やすことのご相談をいただいております。

【次月の目標案】
夜間対応の方針を関係事業所と検討することと、
通所介護の日数追加に向けたケアプラン調整が
望ましいといえます。
```

### 時短効果

60分かかっていた報告書が、AIの下書き+仕上げで20分程度に。

### 使うときのコツ

「判断・評価は〜の所見として」という立場の指定が重要です。これを省くと、AIが中立的な文章を書きすぎて「誰の目で見たのか」が曖昧になります。

## プロンプト5|事業所内研修資料の骨子作成

事業所の月次勉強会や新人研修。テーマは決まっていても、資料の骨子作りから始めると気が重くなる作業です。

全部書かせるのではなく、「骨子だけ作ってもらい、現場の実例は自分で埋める」という使い方が安全です。

### プロンプト

```
訪問介護事業所の月次勉強会の資料を作成します。
以下のテーマで、45分の研修資料の骨子を作ってください。

テーマ:「認知症の利用者様への声かけ、NGと代替案」

・対象:訪問介護のヘルパー、経験1〜3年
・45分(導入5分/本編35分/まとめ5分)
・全体構成:H2レベルの見出し5〜7個
・各見出しの下に、話す内容のポイントを箇条書きで3つずつ
・具体的な事例が入る箇所には【事例】とだけ書き、
 本文は空欄にしておく(私が埋めます)
```

### Claudeの出力例

```
# 認知症の利用者様への声かけ、NGと代替案

## 導入:なぜこのテーマが大切か
- 不適切な声かけは BPSD(行動・心理症状)を悪化させる
- 声かけ一つで、1日の過ごし方が変わる
- 【事例】

## 1. 否定・叱責の言葉を避ける
- 「さっき言ったでしょ」「違うでしょ」は禁句
- 本人の世界に一度寄り添うことの意味
- 【事例】

## 2. 質問より肯定形の声かけ
- 「何してるの?」より「一緒にやりましょうか」
- 選択肢を2つに絞る
- 【事例】

(…以下、本編の見出しが続く…)

## まとめ:今日から使える3つの心得
- 否定しない
- 急かさない
- 本人の時間軸に合わせる
- 【明日からの実践ポイント】
```

### 時短効果

資料の骨子作成が、60分→10分に短縮。「事例は自分で埋める」ことで、現場のリアリティも担保できます。

### 使うときのコツ

「【事例】は空欄のまま」と明示することで、AIが作った偽の事例(ハルシネーション)を資料に混ぜないで済みます。

## 失敗しないための3つのコツ

最後に、私が現場で AI を使いながら失敗して学んだ3つのコツをお伝えします。どのプロンプトを使うときも、この3つだけは忘れないでください。

### 1. 匿名化の徹底──「AIに入れる前に1拍置く」

最初の頃、私は利用者様のお名前をそのまま入力して文章を作らせていました。

後から「これ、クラウド上にログが残る可能性がある」と知って、背筋が寒くなった経験があります。

いまは、AIに貼り付ける前に以下の置き換えを必ず行っています。

* 氏名 → A様、B様
* 住所 → (市区町村名は省略、地域名のみ)
* 病名 → 「ある慢性疾患」等に抽象化

「1拍置いてから貼る」。これだけでリスクはかなり減ります。

### 2. ファクトチェック──AIは自信満々に嘘をつく

AIは、ありもしない加算名を書いたり、制度改正の年を間違えて出力することがあります。

私が経験したのは、「介護保険の◯◯加算」という、存在しない加算名を堂々と書かれたケースです。

数字・固有名詞・制度名・日付は、必ず自分の目と一次情報で確認する。これは毎回、例外なくです。

### 3. 責任は人にある──AIは道具、判断は自分

「AIが書いたから」は、現場では言い訳になりません。

利用者様に届く文書、ケアマネジャーさんに届く報告書、その内容の責任は、それを使う自分にあります。

AIは下書きを作ってくれる優秀な相棒ですが、最後の判断と責任は、変わらず人の側にあります。この前提を持っていれば、AIは怖くない道具になります。

## さいごに:AIは道具、現場が主役

ここまで読んでくださって、ありがとうございます。

今回ご紹介した5つのプロンプトは、看護師として病棟で8年、介護福祉士として訪問介護・施設で7年、そのあと医療ライターとして医療・介護メディアで記事を書いてきた自分自身が、実際に現場で使ってきた型です。

AIは、現場のしんどさを理解してくれる道具ではありません。でも、現場を理解している人が使えば、大きな時間の余裕を生んでくれる道具になります。

その余裕を、利用者様と向き合う時間に回せたら、私たち現場の人間にとって、これ以上ありがたい変化はないのではないでしょうか。
