## はじめに

5人以下のスタートアップを経営していると、「秘書がほしい」と思う瞬間が何度もある。

スケジュール調整、メール返信、情報収集、タスクの整理。  
これらを誰かに任せられたら、もっと「本業」に集中できるのに。

でも、人を雇うほどの規模でもない。月額数十万円の秘書サービスも現実的じゃない。

そこで僕は「AIエージェント」を秘書代わりに導入してみた。

**結論：週約10時間の業務削減に成功した。**

この記事では、実際の設定コードやプロンプトを含めて、AIエージェントを「秘書」として使う方法を解説する。

## 環境

* **AIエージェント基盤**: [OpenClaw](https://github.com/OpenClaw/OpenClaw)
* **LLM**: Claude 3.5 Sonnet / GPT-4o（タスクによって使い分け）
* **連携ツール**: Slack, Google Calendar, Notion, Gmail

## AIチャットとAIエージェントの違い

まず前提を整理する。

| 種類 | 特徴 | 例 |
| --- | --- | --- |
| AIチャット | 聞かれたら答える | ChatGPT, Claude |
| AIエージェント | 聞かなくても動く。複数ツールを連携してタスクを自律的に完了 | OpenClaw, Zapier AI Agents, Lindy |

「来週の会議のアジェンダを整理して」と言ったとき：

* **AIチャット** → 「アジェンダ案はこうです」と提案
* **AIエージェント** → Slackの過去ログを読み、カレンダーを確認し、Notionにアジェンダを作成し、参加者にリマインドを送る

この「勝手に動いてくれる」部分が秘書としての価値になる。

## 実装①：毎朝のタスク整理を自動化

### Before

朝イチでSlack、メール、カレンダー、Notionを開き、今日やることを頭の中で整理。15〜20分かかっていた。

### After

AIエージェントが毎朝8:30に「今日のタスク一覧」をSlackに投げてくる。

### 設定方法

OpenClaw の cron 設定で、毎朝8:30にタスク整理ジョブを登録する：

```
# openclaw.yaml
cron:
  - name: daily-task-summary
    schedule: "30 8 * * *"  # 毎日 8:30
    timezone: Asia/Tokyo
    payload:
      kind: agentTurn
      message: |
        今日のタスクを整理して Slack #general に投稿してください。
        
        以下の手順で進めてください：
        1. Google Calendar から今日の予定を取得
        2. Notion のタスクボードから期限が今日のものを抽出
        3. Slack の未読メンションを確認
        4. 優先度順に並べて、以下のフォーマットで投稿
        
        フォーマット:
        おはようございます。今日のタスクです。
        
        🔴 最優先
        - [予定/タスク]
        
        🟡 通常
        - [予定/タスク]
        
        📌 今日の予定
        - [時間] [内容]
```

### 実際の出力例

```
おはようございます。今日のタスクです。

🔴 最優先
- 14:00〜 クライアントとのミーティング（準備資料の確認を）
- 経費精算の提出期限

🟡 通常
- 採用候補者への返信（昨日受信）
- ブログ記事の下書きレビュー

📌 今日の予定
- 10:00 チーム定例
- 14:00 クライアント MTG
- 17:00 1on1
```

**削減効果：週あたり約1.5時間**

## 実装②：メール下書き生成

### Before

返信が必要なメールを読み、考え、書く。1通あたり10〜15分。

### After

Slackでメールを転送し、一言添えると下書きが返ってくる。

### 運用フロー

```
[Slack で操作]
1. 返信が必要なメールを Slack の専用チャンネルに転送
2. 「丁寧に断って」「前向きに検討すると伝えて」など一言添える
3. AI エージェントが下書きを返信
4. 自分で微調整して Gmail から送信
```

### プロンプト例

```
以下のメールに対する返信を作成してください。

【依頼内容】: ${ユーザーの指示}
【元メール】:
${転送されたメール本文}

【制約】
- 丁寧だが簡潔に
- 自分の名前は「中前」
- 署名は不要
```

**削減効果：1通あたり10分 → 3分（週1.5時間削減）**

## 実装④：議事録の自動作成

### Before

会議中にメモを取り、終了後に整理してNotionに貼る。15〜30分かかっていた。

### After

録音 → 文字起こし → 要約・アクションアイテム抽出が自動化。

### 構成

```
[録音]        [文字起こし]      [要約・抽出]       [保存]
Plaud    →   Whisper API   →   Claude      →   Notion API
```

### OpenClaw での設定例

```
# 議事録処理のワークフロー
tools:
  - name: process-meeting
    trigger: file_upload  # 音声ファイルがアップロードされたら
    steps:
      - transcribe:
          model: whisper-1
      - summarize:
          model: claude-3-5-sonnet
          prompt: |
            以下の議事録から要約・決定事項・アクションアイテムを抽出してください。
            
            【フォーマット】
            ## 要約
            （3行以内）
            
            ## 決定事項
            - 
            
            ## アクションアイテム
            - [ ] 担当者: 内容（期限）
      - save_to_notion:
          database_id: ${NOTION_MEETING_DB}
```

**削減効果：週あたり約2時間**

## AIエージェントが「できないこと」

完全代替は無理。残り20%は人間がやる必要がある。

### 苦手なこと

1. **複雑な判断を伴う調整**

   * 「この会議、延期したほうがいい?」→ 状況を読んで判断は難しい
2. **感情を読む必要がある対応**

   * デリケートな人間関係の調整
   * 相手の気持ちに寄り添った対応
3. **物理的な作業**
4. **オーナーシップを持った推進**

   * 「このプロジェクト、どうすれば成功する?」→ 推進は人間の仕事

### 正しい設計

```
80% → AI に任せる（定型・反復・調査系）
20% → 自分でやる（判断・感情・責任）
```

## 始め方

### Step 1: 毎朝のタスク整理から

一番効果が実感しやすく、リスクが低い。

```
# OpenClaw のインストール
npm install -g openclaw

# 設定ファイル作成
openclaw init

# cron ジョブ追加
openclaw cron add --name daily-summary --schedule "30 8 * * *"
```

### Step 2: メール下書きを試す

返信が必要なメールを1日1〜2通、AIに下書きさせてみる。
