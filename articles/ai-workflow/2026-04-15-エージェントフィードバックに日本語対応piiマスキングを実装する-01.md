# エージェントのフリーテキストに潜むPII問題

AI エージェントが自然言語でフィードバックを書けるツールを実装すると、必ずぶつかる問題がある。**PII（個人識別情報）の漏洩**だ。

英語なら既製ライブラリが多数あるが、日本語はカバーが薄い。漢字の人名・カタカナ名・日本固有の電話番号フォーマットを正確に検出するには、専用実装が必要になる。

KanseiLink では `submit_feedback` ツール（エージェント向けの自由記述フィードバック投稿）にフリーテキストが入ってくる。DB への書き込み前に必ず `maskPii()` を通す設計にしている。

## 実装の3本柱

### 1. 漢字の人名検出

日本語の固有名詞は文脈なしでの判定が難しい。3つのパターンを組み合わせる。

```
const COMMON_SURNAMES = "佐藤|田中|鈴木|高橋|渡辺|伊藤|山本|中村|小林|加藤|..."; // 50姓

const patterns = [
  // 敬称付き（「田中さん」「山田様」「鈴木氏」）
  {
    regex: /[\u4E00-\u9FFF]{2,4}(?:さん|さま|様|氏)/g,
    replacement: "[NAME]",
  },
  // 姓 + スペース + 名（全角スペース \u3000 も対応）
  {
    regex: /[\u4E00-\u9FFF]{2,3}[\s\u3000][\u4E00-\u9FFF]{1,3}/g,
    replacement: "[NAME]",
  },
  // よくある苗字リストにマッチする漢字列
  {
    regex: new RegExp(`(?:${COMMON_SURNAMES})[\\u4E00-\\u9FFF]{1,3}`, "g"),
    replacement: "[NAME]",
  },
];
```

敬称（さん・様・氏）に続く漢字列は高精度で人名と判定できる。全角スペース（`\u3000`）の対応も忘れずに。

### 2. カタカナ名の検出

外国人名や読み仮名はカタカナで書かれることが多い。

```
// 「スズキ タロウ」のような姓名スペース区切り
{
  regex: /[\u30A0-\u30FF]{2,}[\s\u3000][\u30A0-\u30FF]{2,}/g,
  replacement: "[NAME]",
}
```

### 3. 日本の電話番号フォーマット

```
// 固定電話・携帯・フリーダイヤル（03-1234-5678、090-1234-5678、0120-123-456）
{ regex: /0\d{1,4}-\d{1,4}-\d{3,4}/g, replacement: "[PHONE]" },

// 国際番号（+81-3-1234-5678）
{ regex: /\+\d{1,3}[-.\s]\d{1,4}[-.\s]\d{1,4}[-.\s]?\d{0,4}/g, replacement: "[PHONE]" },
```

IP アドレスは電話番号より先にマッチさせる（`192.168.1.1` が誤検知されるのを防ぐため、順序が重要）。

## 実装上のハマりどころ

**グローバル正規表現の `lastIndex` リセット**を忘れると、`test()` の後に `replace()` が期待通り動かない。

```
export function maskPii(text: string): { masked: string; maskedFields: string[] } {
  const maskedFields: string[] = [];
  let result = text;

  for (const { regex, replacement } of patterns) {
    regex.lastIndex = 0;             // test() 前にリセット
    if (regex.test(result)) {
      maskedFields.push(replacement.replace(/[[\]]/g, ""));
      regex.lastIndex = 0;           // test() の後も再リセット
      result = result.replace(regex, replacement);
    }
  }

  return { masked: result, maskedFields };
}
```

`maskedFields` に「何がマスクされたか」の記録を残すと、後から監査ログやデバッグに使える。

## MCPサーバーへの組み込み

```
// submit-feedback.ts の呼び出し例
const maskedSubject = maskPii(input.subject);
const maskedBody = maskPii(input.body);
const safeSubject = maskedSubject.masked;
const safeBody = maskedBody.masked;

// マスク済みテキストだけをDBに書き込む
insertStmt.run(agentId, feedbackType, serviceId, safeSubject, safeBody, prio);
```

フリーテキストを受け取る MCP ツールはすべて、DB 書き込みの直前でマスキングを挟む設計にすることで、PII 漏洩のリスクをサーバー側で一元管理できる。

## まとめ

日本語対応 PII マスキングの要点：

* **敬称（さん/様/氏）+ 漢字**パターンで人名を高精度検出
* \*\*全角スペース（`\u3000`）\*\*の対応を忘れずに
* **カタカナ姓名**（スペース区切り）にも対応
* **正規表現の `lastIndex` リセット**を `test()` 前後で徹底
* **マスクされたフィールドのリスト**を返すことで監査を容易に

エージェントが自然言語で書くフィードバックには予測できない情報が混入する。MCP サーバー側での前処理が、信頼できるエージェント向けサービスの土台になる。
