「AIツールを使い始めたけど、なんか思ったような回答が返ってこない…」そんな経験はありませんか？

実は、ChatGPTやClaudeを使いこなせるかどうかは、**プロンプトの書き方で8割が決まる**と私は感じています。ツール自体の性能ではなく、「どう伝えるか」が鍵なんです。

この記事では、現役エンジニアである私が実務で実際に使っているプロンプトを、カテゴリ別にまとめました。コピーしてすぐ使えるものばかりなので、ぜひ手元に保存しておいてください。

---

## 結論：プロンプトに「役割・背景・制約」を入れるだけで精度が激変する

AIに「なんとなく質問」していませんか？

たとえば「バグを直して」と投げるのと、「TypeScriptのNext.js 14プロジェクトで、useEffectの依存配列に起因するメモリリークが発生しています。原因と修正方法を教えてください」と投げるのでは、返ってくる回答の質が全然違います。

**良いプロンプトの三要素**はこれだけです。

| 要素 | 内容 | 例 |
| --- | --- | --- |
| 役割（Role） | AIにどんな専門家として振る舞ってほしいか | 「あなたはシニアエンジニアです」 |
| 背景（Context） | 状況・技術スタック・前提条件 | 「Next.js 14、TypeScript、App Router使用」 |
| 制約（Constraint） | 出力形式・文字数・禁止事項 | 「箇条書きで、コード例も含めて回答して」 |

この三要素を意識するだけで、AIの回答精度は目に見えて上がります。

---

## なぜプロンプトの質がこれほど重要なのか

AIは「賢い同僚」ではなく、「情報が多ければ多いほど精度が上がる検索エンジンの進化版」に近い存在です。

曖昧な質問をすれば曖昧な回答が返り、具体的な質問をすれば具体的な回答が返る。シンプルなようで、これを意識できているエンジニアは意外と少ないと感じます。

私自身、最初の頃はChatGPTに「このコード直して」と丸投げして「なんか使えないな」と思っていました。でも、背景情報や制約を加えるようになってから、回答の活用率が劇的に上がりました。プロンプトはスキルです。練習すれば必ず上手くなります。

## 【実務で使えるプロンプト集】カテゴリ別に紹介

### 🔍 コードレビュー・バグ修正

コードレビューはAIが最も得意とする領域のひとつです。レビュアーが忙しいときや、深夜に一人でコードを書いているときに重宝します。

**▼ コードレビュー依頼プロンプト**

```
あなたはシニアのバックエンドエンジニアです。
以下のPythonコードをレビューしてください。

【前提】
- FastAPI + SQLAlchemyを使用したREST API
- 本番環境へのデプロイを想定

【確認してほしいポイント】
1. セキュリティ上の問題（SQLインジェクション、認証漏れ等）
2. パフォーマンス上の問題（N+1クエリ等）
3. 可読性・保守性の改善点

【コード】
（ここにコードを貼り付ける）

問題点は重要度（高・中・低）と一緒に箇条書きで教えてください。
```

**▼ バグ原因調査プロンプト**

```
以下のエラーログと状況を元に、バグの原因を特定して修正方法を提案してください。

【環境】
- Node.js v20 / Express 4 / TypeScript 5
- Docker上で動作

【エラーログ】
（エラーをここに貼る）

【発生タイミング】
ユーザーがログイン後、初回のAPIコールで発生。2回目以降は発生しない。

原因の仮説を3つ挙げたうえで、最も可能性が高いものから順に修正コードを示してください。
```

### 📝 技術ドキュメント・仕様書作成

ドキュメント作成はエンジニアが最も後回しにしがちな作業ですよね。AIに任せることで、作業時間を大幅に削減できます。

**▼ README自動生成プロンプト**

```
以下のリポジトリ情報をもとに、GitHubのREADMEを作成してください。

【プロジェクト概要】
タスク管理WebアプリのバックエンドAPI

【技術スタック】
- Go 1.22 / Gin / GORM
- PostgreSQL 16
- Docker / Docker Compose
- JWT認証

【主な機能】
- ユーザー登録・ログイン
- タスクのCRUD操作
- タスクへのタグ付け・検索

Markdown形式で、バッジも含めてわかりやすく作成してください。
```

**▼ API仕様書プロンプト**

```
以下のエンドポイント情報をもとに、OpenAPI（Swagger）形式の仕様書用コメントを書いてください。

エンドポイント：POST /api/v1/tasks
機能：タスクの新規作成
認証：JWTトークン必須
リクエストボディ：title（必須）, description（任意）, due_date（任意, ISO8601形式）
レスポンス：作成したタスクオブジェクト（id, title, description, due_date, created_at）

バリデーションエラーのケースも含めて記載してください。
```

### 🏗️ 設計・アーキテクチャ相談

設計の相談相手がいないときに、Claudeはとくに頼りになります。「こういう設計でいいか？」と聞くと、意外な観点を指摘してくれることが多いです。

**▼ DB設計レビュープロンプト**

```
以下のER図（テキスト形式）をレビューしてください。

【システム概要】
EC サイトの注文管理システム

【テーブル定義】
- users（id, email, name, created_at）
- products（id, name, price, stock, category_id）
- categories（id, name, parent_id）
- orders（id, user_id, total_price, status, created_at）
- order_items（id, order_id, product_id, quantity, price）

【確認してほしい点】
1. 正規化の問題はないか
2. インデックスを張るべきカラムはどこか
3. 将来的な機能追加（クーポン、ポイント機能）を見据えた拡張性

表形式でまとめたうえで、改善案も提示してください。
```

### ✍️ コミットメッセージ・PR説明文

地味だけど、毎日発生する作業こそAIに任せると積み重ねで大きな時短になります。

**▼ コミットメッセージ生成プロンプト**

```
以下の変更内容をもとに、Conventional Commits形式のコミットメッセージを3パターン提案してください。

【変更内容】
- ユーザー一覧取得APIのレスポンスタイムが遅かったため、N+1クエリを解消
- eager loadingに変更し、SQLクエリ数を約30件→1件に削減
- 合わせてキャッシュ（Redis）を導入し、TTL=5分で設定

短すぎず長すぎない、チームで使いやすいものにしてください。
```

| 変更の種類 | prefix | 例 |
| --- | --- | --- |
| 新機能追加 | feat | feat: ユーザー検索機能を追加 |
| バグ修正 | fix | fix: ログイン時のセッション切れを修正 |
| リファクタリング | refactor | refactor: N+1クエリをeager loadingで解消 |
| パフォーマンス改善 | perf | perf: Redis導入によりAPIレスポンスを改善 |
| ドキュメント | docs | docs: README にセットアップ手順を追記 |

### 💬 Claude vs ChatGPT、どちらを使うべきか

「結局どっちがいいの？」とよく聞かれます。私の感覚では、**用途によって使い分けるのがベスト**です。

| 用途 | おすすめ | 理由 |
| --- | --- | --- |
| 長文ドキュメント生成・要約 | Claude | 長いコンテキストを正確に保持しやすい |
| コードデバッグ・補完 | どちらも◎ | 最新モデルは甲乙つけがたい |
| 複雑なロジック設計の壁打ち | Claude | 論理展開が丁寧で見落としを指摘してくれやすい |
| ブラウザ検索との連携 | ChatGPT | Web検索との統合が成熟している |
| APIコストを抑えたい | Claude Haiku / GPT-4o mini | 軽量モデルが優秀 |

## プロンプト精度をさらに上げる5つのテクニック

### 1. 出力形式を明示する

「箇条書きで」「表で」「コードブロックで」と言うだけで、読みやすさが段違いになります。

### 2. 「ステップバイステップで考えて」を入れる

複雑な問題には有効です。AIが思考プロセスを可視化してくれるため、間違いに気づきやすくなります。

### 3. 否定形より肯定形で伝える

「〜しないで」より「〜してください」の方が意図が伝わりやすいです。

### 4. 具体的な数字・固有名詞を使う

「最近のバージョン」より「React 19」、「大きなファイル」より「10MB以上のファイル」の方が精度が上がります。

### 5. 期待するアウトプットのサンプルを添える

「こういう形式で出してほしい」と例を見せると、AIが出力のフォーマットを合わせてくれます。

## まとめ

* プロンプトに**役割・背景・制約**を入れると精度が上がる
* コードレビュー・ドキュメント作成・設計相談など、実務のあらゆる場面で活用できる
* ClaudeとChatGPTは用途によって使い分けるのがベスト
* プロンプトはスキル。意識して書き続けるほど上達する

AIツールは「使えない」のではなく、「使い方を知らない」だけなことがほとんどです。今日からプロンプトを少し変えるだけで、毎日の開発体験がぐっと楽になるはずです。
