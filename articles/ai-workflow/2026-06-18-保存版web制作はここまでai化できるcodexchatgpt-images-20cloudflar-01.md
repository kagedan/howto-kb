AIでWebサイトを作る記事を読んでも、手が止まる理由は決まっています。

リサーチ、情報設計、デザイン、公開、改善がバラバラに説明されるからです。

この記事では、AIを教える講師、研修、コンサル、顧問、教材開発を行うAIソロプレナーが、自分の公式Webサイトを作る流れを1本にします。

読み終わると、**Codex、ChatGPT Images 2.0、Cloudflare**だけで、初版公開から改善まで動かせます。

なお、OpenAI公式ドキュメントでは画像生成モデルとして `gpt-image-2` が説明されています。

ChatGPT上の画像生成体験はヘルプで `ChatGPT Images` と表記されています。

この記事では、読者が使っている呼び方に合わせて **`ChatGPT Images 2.0`** と書きます。

## まず全体像: 公式サイトは7日で公開して育てる

![](https://assets.st-note.com/img/1781743212-248YfvMeHVlo7sdArkQEipCF.png?width=1200)

結論から言います。

公式サイトは、最初から完成品を作るものではありません。  
7日で公開し、数字を見て育てるものです。

AIソロプレナーの公式サイトで必要なのは、豪華なページ数ではありません。

必要なのは、次の3つです。

* **何者かが3秒で分かる**
* **何を頼めるかが分かる**
* **次に何をすればいいかが分かる**

この3つがあれば、初版として十分です。

最初から、事例ページ、ブログ、ダウンロード資料、会員ページまで作ろうとすると止まります。

今回使う道具は3つだけです。

* **Codex**
* **ChatGPT Images 2.0**

  + ヒーロー画像
  + プロフィール写真風ビジュアル
  + 図解
  + OGP画像
* **Cloudflare**

  + Pages公開
  + 独自ドメイン
  + Web Analytics
  + Core Web Vitals確認
  + A/Bテスト

ここで重要なのは、順番です。

いきなりCodexへ「**公式サイトを作って**」と言うと、見た目だけのページが出ます。

先に作るのはサイトではありません。  
先に作るのは、AIへ読ませる前提ファイルです。

最初に作るファイルは5つです。

* **`site-brief.md`**
* **`customer.md`**
* **`offer.md`**
* **`design.md`**
* **`publish-checklist.md`**

この5つがあると、Codexの出力が安定します。

画像も、コピーも、ページ構成も、公開チェックも同じ基準で動きます。

ここで今すぐやることは1つです。  
空のフォルダを作り、上の5ファイル名だけ置いてください。

中身はまだ空で構いません。  
ファイル名を置くと、作業が始まります。

## Phase1 リサーチ: AI講師の売れる材料を集める

![](https://assets.st-note.com/img/1781743264-C1ZR93l2HjXmPepdU5YQvLSK.png?width=1200)

最初のリサーチで集めるものは、競合URLではありません。  
読者が「**この人に頼んで大丈夫**」と判断する材料です。

AIソロプレナーの公式サイトは、デザイン作品ではありません。  
営業前の不安を減らすページです。

AI講師、研修講師、コンサル、顧問、教材開発者なら、必要な材料はほぼ決まっています。

* **何を教えられるか**
* **誰に教えられるか**
* **どんな状態へ変えられるか**
* **どんな形式で提供できるか**
* **いくらから相談できるか**
* **どんな人には向かないか**
* **問い合わせ前に何を準備すればよいか**

ここを飛ばしてデザインへ行くと、見た目は整っても問い合わせが増えません。  
なぜなら、読者の不安が減っていないからです。

リサーチは、次の4方向で集めます。

* **自分の材料**

  + プロフィール
  + 登壇歴
  + 教材テーマ
  + 研修テーマ
  + 得意な業界
  + よく受ける質問
* **顧客の材料**

  + 研修担当者の悩み
  + 経営者の不安
  + 受講者のつまずき
  + 社内展開の壁
* **競合の材料**
* **証拠の材料**

  + 実績
  + 受講者の声
  + 制作物
  + 出版物
  + note記事
  + X投稿
  + セミナー資料

ここで使うCodexへの最初の依頼はこれです。

```
AI講師、研修、コンサル、顧問、教材開発を行うAIソロプレナーの公式Webサイトを作ります。
まず、サイト制作前のリサーチ項目を整理してください。

# 目的
- 問い合わせ獲得
- 法人研修相談
- 顧問相談
- 教材購入前の信頼形成

# 出力してほしいもの
1. 集めるべき自分の材料
2. 集めるべき顧客の材料
3. 集めるべき競合の材料
4. 集めるべき証拠の材料
5. 足りない情報
6. 公式サイトに載せるべきではない情報

# 条件
- AI初心者にも分かる言葉
- 研修担当者、経営者、個人受講者を分ける
- 実績不明な数字は使わない
- 最後に site-brief.md の初稿構成を出す
```

この段階のゴールは、完璧な調査ではありません。  
サイトに必要な材料が見えることです。

30分で終わらせるなら、次の項目だけ埋めます。

* **誰向けか**
* **何を提供するか**
* **一番売りたいもの**
* **問い合わせ後の流れ**
* **実績として言えること**
* **言えないこと**
* **使いたくない表現**

この7つが埋まれば、次へ進めます。
