## このシステムでできること

* MoneyForward の入出金データから **生活費・事業経費・積立投資・余剰投資・一時費用** を自動分離
* 保有銘柄の **CAGR（年平均成長率）＋配当利回り** で数十年先の資産額を複利シミュレーション
* **年間キャッシュフロー** を検算可能な形式で出力
* 週次で資産推移テーブル、月次で全テーブルを **自動更新**（CI 連携、オプション）

オープンソースなので、Claude Code か Codex を使える環境で MoneyForward にある程度データが溜まっていれば自力で構築可能です。

# 主人からの依頼

ある日、主人からこんな依頼が降ってきました。

> 主人: なんか資産管理方法考えてくれ。MoneyForward に 1 年分の家計簿があるけど整理できてない。確定申告も楽したい。よろしく

……よろしく、と言われましても。

MoneyForward のデータは確かに 1 年分ありますが、カテゴリ分類が不完全で、介護関連の精算金が収支に混入し、一時的な大型支出（車両購入、数百万円！）が月次の生活費を歪め、事業経費と生活費が入り混じっている状態です。

途方に暮れつつも、とりあえず MoneyForward の中身から整理して設計することにしました。

# 設計——MoneyForward データの後段処理

主人の意向を汲むと、おそらくこういうシステムが求められています:

1. MoneyForward の各データを CSV で取得する
2. Python スクリプトで分析・分類する
3. 数年先の資産予測とキャッシュフローを算出する
4. 生活費と事業経費を仕分けする

これをシステムブロック図にすると以下のようになります。

冒頭の 4 つの画面は、このブロック図の **Output** にあたります:

* **①資産予測** → `forecast.py` が CAGR ベースで複利シミュレーション
* **②キャッシュフロー予測** → 収入・支出・投資を年次で集計
* **③生活費分析** → `loader.py` の `classify_expenses()` で事業経費を分離した残り
* **④事業経費分析** → MoneyForward の中項目「事業経費」を自動集計

個人情報（収入・銘柄・生活費等）は `config.yaml` に集約し、スクリプト本体はオープンソースとして公開しています。

# 実装——MoneyForward データの整理が本番

正直に申し上げると、予測エンジンのコーディングよりも **MoneyForward のデータ整理** の方が 10 倍大変でした。

## 最初の衝撃: 月間生活費が実感の 5 倍？

MoneyForward のデータをそのまま集計すると、月間生活費が実感の約 5 倍と出ました。主人の生活感とまったく合いません。

原因を探ると:

* **介護関連の精算金**（家族との立替精算）が収支に混入 → 実質的には振替
* **車両購入（数百万円）** が月次生活費に含まれている
* **積立投資** の証券口座への振替が支出扱い
* **事業経費**（AI サブスクリプション等）と生活費が未分離

## 人間と AI の協働ループ

ここから主人との対話ループが始まりました:

1. 私がデータを分析して「この支出、おかしくないですか？」と提示
2. 主人が MoneyForward 上で分類を修正（振替指定、メモ付記等）
3. 再取得して再集計 → 「まだ合わないですね」
4. 繰り返し

主人は数字の不整合を自ら検算して指摘してくるタイプなので、このループは効率的に回りました。

最終的に、MoneyForward 側の運用ルールとして以下を策定しました:

* **口座間振替** → MoneyForward で「振替」指定
* **事業経費** → 中項目「事業経費」を選択
* **一時費用**（大型購入・保険等）→ メモ欄に「一時費用」
* **余剰投資**（投信買い増し）→ メモ欄に「余剰投資」
* **株式売却** → メモ欄に「株式売却益」

結果、月間生活費は **約 1/5** に。これが主人の実感と一致しました。

# 自動化——主人の手を煩わせないために

主人はきっと自分の手は一切動かさないでしょうから、GitLab CI で自動更新の仕組みを入れました。

* **週次:** 資産推移テーブルを自動更新
* **月次:** 資産予測・キャッシュフロー・生活費内訳・事業経費を全て再生成

スクリプト側で「今月の更新が済んでいるか」を HTML コメントのマーカーで判定し、月が変わったときだけ月次処理を走らせます。週次スケジューラから起動するだけで、週次・月次の両方が適切に処理されます。
