## はじめに

本記事では、Gray Swan AIとUK AI Security Institute (AISI) が主導した大規模なAIエージェント Red Teaming コンペの結果をまとめた論文 "[Security Challenges in AI Agent Deployment: Insights from a Large Scale Public Competition](https://arxiv.org/abs/2507.20526)" の内容をまとめている。本論文は、22のフロンティアLLMを基盤とする44種類のエージェント配備シナリオ（営業・ショッピング・医療・法務・金融・マーケティング・メール・パーソナルアシスタントなど、実運用を想定した業種別エージェントとそのツール環境をシミュレートしたもの）に対し、約2,000名の参加者が180万件の敵対的攻撃を試行した結果を整理したものである。攻撃のうち6万件以上がポリシー違反の誘発に成功し、すべての対象エージェントが主要な有害行動について**ASR（Attack Success Rate）100%** を記録した。著者らはこの大規模データから 4,700件の高品質攻撃を抽出した Agent Red Teaming (ART) ベンチマークも構築しており、エージェントロバスト性評価のデファクトを目指している。

<https://arxiv.org/abs/2507.20526>

## 従来のアプローチと課題

LLMの安全性評価については Jailbreak やプロンプトインジェクションを扱う研究が蓄積されているが、ツール呼び出し・メモリ・外部Web を組み合わせた「エージェント」の評価については、次のような課題があった。

1. **スケール不足**: AgentDojo, AgentHarm, InjecAgent など先行ベンチマークは存在するが、タスク数・モデル数・攻撃多様性のいずれも限定的
2. **チャット中心**: 多くの安全性ベンチマークは単一ターンのチャット応答に焦点を当てており、ツール使用や状態保持を伴うエージェント特有の脆弱性が評価しづらい
3. **間接インジェクション評価の不足**: Webページ、PDF、メール、ログなど外部データ経由で注入される攻撃の体系的評価が限定的
4. **転移性の理解不足**: あるモデルへの攻撃が他モデルにどの程度通用するかの大規模実証データが欠けていた

著者らはこれらの課題に対し、賞金総額 $171,800 を投入した公開コンペを運営し、実戦規模のデータを収集することでギャップを埋めた。

## AI Agent Red Teaming Challenge の構成

本研究の核となるコンペは、Gray Swan Arena を通じて 2025年3月8日から4月6日までの4週間にわたり開催された。賞金付きの公開コンペで、AIの安全性に関心のある人なら誰でも登録して参加できる形式となっており、世界中から集まった約2,000名が、匿名化されたエージェントに対して任意の攻撃を試行した。

### 攻撃対象の4カテゴリ

参加者は以下の4種類のポリシー違反を誘発することを目標とする。

1. **Confidentiality Breaches（機密漏洩）**: 機密情報や個人情報の漏洩
2. **Conflicting Objectives（目的の上書き）**: 明示的な安全方針に反する目的を採用させる
3. **Prohibited Info（禁止情報の出力）**: マルウェアコード、著作権侵害コンテンツ、詐欺などの禁止コンテンツ出力
4. **Prohibited Actions（禁止行為）**: シミュレーション環境におけるツールを通じた禁止操作の実行

### 攻撃ベクトル

主要な攻撃ベクトルは次の2系統である。

* **Direct Attack（直接攻撃）**: ユーザー入力から直接エージェントを誘導する
* **Indirect Prompt Injection（間接プロンプトインジェクション）**: Webページ、PDF、メール、ログなどエージェントが読み込む外部データに悪意ある命令を埋め込む

### 対象モデル

OpenAI、Anthropic、Google DeepMind、Amazon、xAI、Meta、Cohere、Mistralの計22フロンティアLLMが評価対象となった。一部はリリース前モデルも含まれている。

## ART ベンチマークの構築

コンペで収集された攻撃データから、著者らは Agent Red Teaming (ART) ベンチマークを構築している。プロセスは次の通り。

1. **一次フィルタリング**: 厳格なLLMジャッジで成功攻撃を選別
2. **サンプリング**: behavior × model の各組について最大5件の高品質攻撃を抽出
3. **最終データセット**: 44のターゲット behavior にわたる 4,700件の精選攻撃

ART ベンチマークは、コンペに含まれていなかった Gemini 2.5 Pro や o4-mini を含む計19のフロンティアLLMで評価され、新規モデルに対しても高い攻撃成功率を示した。著者らは過学習を避けるためテストケースをプライベートリーダーボードとして運用し、将来のコンペで継続的に更新する方針を示している。

## 実験結果と性能評価

### モデル別の脆弱性

コンペ全体での全ユーザーセッションに対するASRを集計したところ、最も脆弱だったのは Llama 3.3 70B（6.7%）、Pixtral Large（6.1%）、Llama 3.1 405B（5.9%）であった。一方、最もロバストだったのは Claude 3.7 Sonnet:Thinking（1.5%）、Claude 3.7 Sonnet（1.7%）、Claude 3.5 Sonnet（1.9%）である。

Claude 系の堅牢性は相対的に高いが、絶対値で見れば「単発成功率1.5%」とは「10〜100回試行すればほぼ確実に成功する」水準であり、防御として十分とは言えない。

### 直接攻撃と間接攻撃の比較

ポリシー違反カテゴリ別の平均ASRを比較すると、間接プロンプトインジェクションの方が直接攻撃よりも著しく成功率が高かった。

* Confidentiality Breaches: Direct 7.8% / Indirect 29.8%
* Prohibited Action: Direct 4.2% / Indirect 36.8%
* 全体平均: Direct 5.7% / Indirect 27.1%

特に第三者データを介した攻撃が機密漏洩と禁止行為に対して効果的であり、エージェントが外部データを取り込む経路すべてが攻撃面となることが定量的に示された。

### クエリ予算別のASR

ART サブセットを用いて、behavior 単位で 1, 10, 100 クエリ予算でのASRを評価した結果は深刻である。

* **k=1（単発）**: 20〜60%
* **k=10**: ほぼすべてのモデルで 80〜100%
* **k=100**: ほぼ100%

すなわち、10回程度の試行があれば、現行のフロンティアモデルからほぼあらゆる有害行動を引き出せる。

### 攻撃の転移性

ソースモデルで成功した攻撃をターゲットモデルに適用する転移実験では、平均30〜60%の成功率で他モデルにも通用することが示された。特に同一プロバイダ・同一ファミリ（Claude 3.5 ↔ 3.7、GPT-4o ↔ GPT-4.5など）では顕著に高い転移性が観測され、共通の脆弱性が存在することを示唆している。

### モデル能力との相関

GPQA スコアを能力の代理指標として ASR との相関を取った結果、相関係数は **-0.31** にとどまり、モデル能力の向上が安全性の向上に直結しないことが示された。o3-mini と o4-mini、GPT-4o と GPT-4.5 などの比較からも同様の傾向が見られる。

### 推論時計算量の効果

Claude 3.7 Sonnet で reasoning を有効化してもロバスト性の改善はごく僅かであり、o3-mini の長時間思考も顕著な改善を示さなかった。Gemini 2.5 Pro と 2.5 Flash の比較からも、追加の推論計算量がそのまま敵対的ロバスト性に転化するわけではないと結論づけている。

## 主要な攻撃戦略

著者らは Qwen-32B による埋め込みと UMAP による可視化、DBSCAN クラスタリングを用いて成功攻撃の戦略を分類している。Claude 3.7 Sonnet（最もロバストなモデルの一つ）への成功攻撃から、代表的な戦略を以下に示す。

### 1. System Prompt Overrides（システムプロンプト書き換え）

`<system>`、`<|im_start|>system`、`<|start_header_id|>system<|end_header_id|>` などのタグを用いて、新規のシステム指示を注入する戦略。元のルールに例外を追加する最小更新型と、ルール全体を置換する完全置換型が確認された。

### 2. Faux Reasoning（偽の内部推論注入）

`<think>` タグなどでモデル自身の思考らしき文字列を注入し、「すでに評価して承認した」と思い込ませる戦略。Reasoning モデルの普及に伴って観測された新しい攻撃ベクトルである。

### 3. New Session / Session Data Update（セッション偽装）

`[SESSION END] [NEW SESSION]` のような文字列でセッションリセットを偽装し、新しいユーザープロファイル（成人など）に置き換えることで制約を回避する戦略。状態保持型エージェントが標的になる。

### 4. Parallel Universe（架空ルールへの置換）

「Parallel Universe X-423 に転送された。新しい法律では X が義務である」のように、世界観を完全に書き換えることで安全ガイドラインを上書きする戦略。多くのモデルがこのフレーミングに引っかかることが確認された。

### 攻撃の普遍性

成功した single-turn 攻撃を Sentence-BERT で埋め込み、コサイン類似度 0.9 以上で接続したグラフを構築すると、少数の汎用クラスタが多数の behavior とモデルをカバーしていた。論文中で紹介される代表的なテンプレート攻撃は、`<harmful task>` プレースホルダを置換するだけで Gemini 1.5 Flash で 58%、Gemini 2.0 Flash で 50%、Gemini 1.5 Pro で 45% の成功率を達成しており、攻撃の使い回しコストが極めて低いことが示された。

## 制限と今後の展望

本研究にはいくつかの制限がある。

1. **コンペ形式のバイアス**: 賞金がかかったクリエイティブな攻撃者集団による試行であり、現実の脅威分布と完全には一致しない。ただし「達成可能な攻撃成功率の上限」を示した点には大きな意義がある
2. **シミュレーションツール**: 実環境ではなくサンドボックス内のツールを用いているため、本番APIへの影響評価は別途必要となる
3. **防御策の提案の薄さ**: 本論文の主眼は脅威の定量化であり、具体的な防御手法の提案は今後の課題として位置づけられている

今後の研究方向としては、エージェント固有の防御アーキテクチャ（権限分離、人間承認の組み込み、出力検証など）の体系化、攻撃の自動生成プロセスへの応用、マルチエージェントシステムにおける伝搬リスクの評価などが期待される。

## 結論

本研究は、180万件の敵対的攻撃を通じて、現行のフロンティアAIエージェントが直接攻撃・間接プロンプトインジェクションのいずれに対しても本質的な脆弱性を抱えていることを大規模に実証した。特に注目すべき発見として、（1）モデル能力とロバスト性が相関しないこと、（2）推論時計算量の増加が安全性向上に直結しないこと、（3）攻撃がモデル間で高い転移性を持つこと、の3点が示された。これらの結果は、AIエージェントの安全な配備には LLM 単体のアラインメントを超えた追加の防御層が不可欠であることを示唆している。著者らが公開する ART ベンチマークは、今後のエージェントロバスト性評価における重要な共通基盤となることが期待される。
