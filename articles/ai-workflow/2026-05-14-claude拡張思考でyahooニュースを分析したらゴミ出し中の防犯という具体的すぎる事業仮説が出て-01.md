はじめに 🧠  
2026年現在、Claudeの\*\*Extended Thinking（拡張思考）\*\*は、長文の構造化分析タスクで実用レベルに達しています。今回、Yahoo!ニュースのトピック一覧をスクレイピングし、Claude APIに「ビジネスチャンス抽出」を依頼するパイプラインを組みました。本記事ではその技術的な工夫と、AIが出した予想外の結果を共有します。

アーキテクチャ概要 ⚙️

```
Yahoo!ニュース → BeautifulSoup → 構造化JSON

↓

Claude API (extended thinking: enabled)

↓

Markdown分析レポート
```

ポイントは、ニュースを **「番号付きリスト」** にしてClaudeに渡すこと。後段で「根拠ニュース番号」を参照させるためのトレーサビリティ確保です。

プロンプト設計の肝 ✍️  
Extended Thinkingを活かすには、**「思考の枠組み」と「出力フォーマット」を分離**するのが効果的でした。

```
system_prompt = """

あなたはビジネスアナリスト。以下の順で思考せよ:

1. ニュース横断のトレンドを3つ抽出（複数ニュースの統合を必須）

2. 各トレンドから派生するビジネスチャンスを5つ

3. それぞれに「根拠ニュース番号」「ターゲット顧客」「実現難易度」を付与

4. 最後に、24時間以内に実行可能なアクションを1つだけ提示

"""
```

注目すべきは **「24時間以内に実行可能なアクションを1つだけ」** という制約。これがないと、AIは抽象的な「市場調査せよ」を返しがちです。**具体性は制約から生まれる**、というのが今回最大の学びでした。

```
response = client.messages.create(

model="claude-sonnet-4-5",

max_tokens=8000,

thinking={"type": "enabled", "budget_tokens": 5000},

system=system_prompt,

messages=[{"role": "user", "content": news_text}],

)
```

`budget_tokens=5000` がスイートスポット。これ以下だとトレンドの統合が雑になり、これ以上だと冗長になります。

AIが出した分析で面白かった点 🎯

### 1. 「ゴミ出し中の数分間」という解像度

強盗ニュース2本から、Claudeは「**ゴミ出し・玄関開放時の数分間に特化した短時間警戒モード**」というサービス仕様を提案。報道テキストには「ゴミ出し」という単語が直接登場しないケースもあったのですが、拡張思考のログを見ると「被害発生時間帯」と「住宅構造」を組み合わせて推論していました。**人間が見落とす"時間軸の盲点"を埋めるのがAIの強み**だと再確認しました。

### 2. 大谷翔平 → マンダラチャート教材化

スポーツニュース3本から「**二刀流育成スクール**」を提案し、教材として大谷選手が高校時代に使った目標設定フレームワークまで言及。ニュース本文には含まれない一般知識との接続が自然に行われていました。

### 3. リスクの自己批判

驚いたのは、**「大谷個人への依存度が高く、ブームが沈静化すると訴求力低下」** と自分の提案を自己批判してきたこと。Extended Thinkingでは「反対意見を内部で検討する」ステップが効いているようで、出力の信頼度が一段上がります。

ハマりどころ 🪤

* **思考トークンは課金対象**。dry runでbudgetを調整推奨
* **ニュース番号の対応付け**は、出力後にregexで検証必須（稀にズレる）
* 時事性が高い話題は**1日でモデル知識と乖離**するため、Web検索ツールとの併用が望ましい

まとめ  
Extended Thinkingは「思考プロセスの分離」と「具体性を強制する制約」を組み合わせると真価を発揮します。次は同じパイプラインを業界別ニュースに適用し、ドメイン特化型の仮説生成器を作る予定です。
