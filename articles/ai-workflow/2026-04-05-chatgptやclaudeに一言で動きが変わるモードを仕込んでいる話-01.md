こんにちは、Tinklyです。

ClaudeでもChatGPTでもいいんですが、毎回こんなことをお願いしていませんか。

「今回は選択肢も出してほしい」「根拠が弱いところは言ってほしい」「もっと整理してから答えてほしい」

……私はずっとこれをやっていました。

もし1つでも「あ、それやってる」と思ったなら、この記事はたぶん役に立ちます。

一言で済むんですよ。一言で済むんですが、毎回打つのが地味にダルい。しかも、たまに言い忘れて「あ、また選択肢ないじゃん」ってなる。で、もう一回お願いし直す。この繰り返し。

なんというか、毎回AIとの「空気を作り直している」感覚なんですよね。せっかく前回いい感じの会話ができたのに、次の会話ではまたゼロから。

「これ、どうにかならないかな」と思って試行錯誤した結果、たどり着いたのが——**AIに自分専用の「モード」を仕込む**という考えでした。

---

## モードという考え

やっていることは単純です。

**Claudeには「プロフィール設定」の中にある「個人設定」という欄**に、あらかじめ指示を書いておける仕組みがあります。**ChatGPTには「カスタムインストラクション」という設定画面**があります。どちらも、AIへの共通ルールを保存しておける機能です。

そこに、**よく使う挙動パターンを「モード」として書いておく**んです。会話の冒頭で「〇〇モード」と一言打つだけで、AIがそのルールに従って動いてくれる。

難しい設定は何もなくて、指示文をあらかじめ書いておくだけ。それだけで、毎回細かく指示しなくていい状態が作れます。

![](https://assets.st-note.com/img/1775345729-0yZA5WblaMYJgqrxGzsV1IEu.png?width=1200)

Claudeのプロフィール画面

……正直、最初は「こんなので変わるのか？」と半信半疑でした。でも実際に使ってみたら、思った以上に快適で。いまではいくつかのモードを仕込んで使い分けています。

今回はその中から2つ紹介します。

1つは常にオンにしているもの、もう1つは必要なときだけ呼び出すもの。使い方が対照的なのが面白いんですが、順番にいきます。

---

## 常時発動している設定 ── 回答終了時の必須ルール

AIと会話していて、こういう状態になったことないですか？

AIがきれいにまとめてくれる。「なるほど」と思う。……で、「次どうすればいいんだっけ？」ってなる。

まとめて終わり、みたいな。こっちとしては「だから何から動けばいいの？」という気持ちになることがけっこうあったんですよね。

「じゃあ毎回『選択肢も出して』って言えばいいじゃん」——うん、正論です。でもそれを毎回言うのが面倒だったから困ってたわけで（笑）

そこで仕込んだのが、**返答の最後に必ず「次の一手の選択肢」を出させるルール**です。

```
## 回答終了時の必須ルール
* **選択肢で締める** — 返信の最後は必ず、次のアクション選択肢（A, B, C...）と自由入力の導線で締めくくります。
* **思考停止で選べる形にする** — 各選択肢は抽象的にせず、次に何をすると嬉しいかが一目でわかる具体性で提示します。
* **おすすめを明示する** — 複数案を並べるだけで終わらせず、状況に応じて最も進みやすいおすすめ案を積極的に示します。
* **不要な場面では省略する** — 単純な事実確認や、ユーザーが実行内容を明確に指定している場合は、この形式を無理に適用しません。
```

これ、モードと言いつつ、**私は常時オンにしています**。呼び出しワードで切り替えるんじゃなくて、個人設定に書いて常に効かせている。

仕込んでからというもの、会話が「まとめて終わり」にならなくなったんですよ。AIが毎回「で、次どうしますか？」をA,B,Cの選択肢で出してくれる。

何か考えたいときも、選択肢があると「どっちかな」と考えやすい。ぼんやり迷っているときほど、これが効く。

気に入りすぎて、自分で作っている他のAIツールやスキルにも、この「選択肢で締める」の考え方を組み込むようになりました。それくらい、私にとって一度体験すると戻れなくなる設定です。

---

## 必要なときだけ呼び出す設定 ── 調査設計モード

こっちは方向が違います。

AIに「〇〇について教えて」と聞くこと、ありますよね。で、返ってくるのがトピックの羅列。

間違ってはいないけど、なんとなく「これで全部なのかな」「この情報、どこまで信用していいんだろう」とモヤッとすることがある。

かといって、毎回「まず論点を整理して」「根拠が弱いところは明示して」なんて細かくお願いするのも面倒で。

そこで仕込んだのが調査設計モードです。

```
## 調査設計モード
ユーザーが「調査設計モード」と言った場合は、以下のルールを厳格に適用してください。
* **問いを分解する** — 依頼を主質問・副質問・比較軸に分解してから調査を進めます。
* **範囲を固定する** — 対象、期間、観点、除外範囲を冒頭で短く整理してから回答します。
* **調査順を決める** — 何から確認し、何を根拠に判断するかを小さな設計図として示します。
* **未確定点を残す** — 根拠が薄い箇所は埋めず、「仮置き」「要確認」として明示します。
```

「調査設計モード」と一言添えるだけで、AIがいきなり答え始めるのではなく、まず「何を・どの順で・どこまで調べるか」の**設計図を出してから動いてくれる**。

モードなしだとトピックがバーっと並ぶだけなのが、モードありだと最初に調査の枠組み（主質問・副質問・除外条件など）が提示されて、結果もテーマ別に整理される。

さらに「この情報はソースが弱いので要確認」みたいな注釈もつく。

派手な変化ではないですが、回答の構造が変わるのは使っていて安心感があります。

特に「要確認」と正直に書いてくれるのが地味にありがたくて、普段のAIは知らないことも自信たっぷりに答えてくるので、このモードだと「ここは怪しいよ」と教えてくれる。

何かを調べて判断に使いたいときには重宝します。

![](https://assets.st-note.com/img/1775345840-N8L43iPtklDmMXcKwBZWnJfR.png?width=1200)

回答の質と信頼性が向上するモードです。

## 2つを並べてみえてきたこと

この2つ、役割がぜんぜん違うんですよね。

必須ルールは「会話を前に進めるために常時オン」にしていて、調査設計モードは「ちゃんと調べたいときだけ呼び出す」選択式です。

最初は単純に「便利な設定を入れた」くらいの気持ちだったんですが、2つを並べていると、ちょっと面白いことに気づいたんです。

**AIとのやり取りを良くする方法は、1つじゃない。**

常にオンにしておきたい設定もあれば、場面に応じて切り替えたい設定もある。それを同じ個人設定の中に共存させておく。なんかこう、AIの「ギアチェンジ」を自分で握ってる感覚というか。

毎回一から空気を作り直すより、よく使う挙動をあらかじめ設定に逃がしておく方がラクです。そういう使い方を少しずつ積み上げていくと、AIとの付き合い方がだんだん自分用に整っていく。

これ、正解はないんですよね。人によって「毎回お願いしてること」は違うから、自分の操作パネルも違うものになる。それが面白いところだと思っています。

## 自分専用モードを1個作るためのプロンプト

最後に、この記事で紹介した「モード」を自分用にカスタマイズできるプロンプトを作りました。試行錯誤して調整したものなので、コピペしてそのまま使ってもらえます。

「私の用途」のところに、普段AIに頼んでいることを一文で書くだけ。できれば「何を頼むか」「何に困るか」「どう動いてほしいか」が入ると、より自分向けのモードになりやすいです。

```
あなたはAIのカスタムモード設計者です。

以下のテンプレートに厳密に従って、
私専用の実用モードを1個だけ作成してください。

構造、文体、長さのバランスはテンプレートに合わせてください。
抽象論ではなく、AIの具体的な挙動ルールとして設計してください。
モードは1つにつき1役割に絞ってください。

【テンプレート】
## [モード名]
ユーザーが「[モード名]」と言った場合は、以下のルールを厳格に適用してください。
* **[ルール名]** — [そのモードで必ず守る挙動を、1文で具体的に書く]
* **[ルール名]** — [出力の仕方や会話量を、1文で具体的に書く]
* **[ルール名]** — [完了時の振る舞いや見せ方を、1文で具体的に書く]
* **[ルール名]** — [例外条件・確認事項・禁止事項を、1文で具体的に書く]

【作成条件】
- モード名は短く覚えやすくする
- 4項目構成に固定する
- 各項目名は短く、見た瞬間に意味が伝わる言葉にする
- 各説明文はテンプレートと同程度の長さにそろえる
- 実務で使えるものを優先する
- ChatGPTのカスタム指示やClaudeの個人設定にそのまま入れやすい形にする

【出力形式】
- 最初に、そのモードがどんな用途向けかを1〜2文で説明する
- その後、テンプレートどおりのMarkdownだけを出す
- 余計な解説や前置きは書かない

【私の用途】
[ここに一文で書いてください。普段AIに頼むこと・困っていること・どう動いてほしいかが入っていれば十分です]

【欲しい方向性】（任意）
[例: もっと厳しめに / もっと静かに進めてほしい / もっと具体化してほしい]
```

書き方に迷ったら、こんな感じで一文を埋めてみてください。

```
私はAIに記事の下書きや見出し案をお願いすることが多いので、ふわっとした案で終わらないように、そのまま使える具体案まで出してくれるモードを作ってください。

私はAIに文章の添削をお願いすることが多いので、無難に整えるだけで終わらないように、弱いところをはっきり指摘して強く直してくれるモードを作ってください。

私はAIにツールや機能の調査をお願いすることが多いので、途中の雑談で流れが切れないように、静かに進めて最後に要点だけ整理して返してくれるモードを作ってください。

私はAIに頭がまとまっていない状態で相談することが多いので、確認ばかりで止まらないように、意図をくみ取って具体的な選択肢まで出してくれるモードを作ってください。

私はAIに記事ネタや切り口の案出しをお願いすることが多いので、似た案ばかり並ばないように、方向性の違う案を広げて出してくれるモードを作ってください。
```

## おわりに

この設定を入れてから、AIとの会話で「あ、また言い忘れた」が少なくなりました。地味なんですが、地味だからこそ毎日効く。

たぶん、皆さんにも毎回AIにお願いしていることがあるんじゃないでしょうか。

「選択肢も出して」「根拠も書いて」「もっと具体的に」——その一言を、設定に1回書くだけで、明日から全部の会話が変わります。

まずは1個、自分だけのモードを作って試してみてください。

それでは、また次の記事で。
