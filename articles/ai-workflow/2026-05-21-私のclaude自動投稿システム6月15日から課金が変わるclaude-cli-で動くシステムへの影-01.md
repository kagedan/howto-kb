2026年5月13日、Anthropicがサブスクリプションを刷新すると発表した。ポイントは一つ：6月15日以降、claude -p や Agent SDK を使ったプログラム実行が、通常のサブスク枠と分離される。

私の自動投稿システム「エリス・ログ」は、スケジュールタスクが毎日 claude -p（claude CLI）を呼び出して動いている。つまり今回の変更は、完全に自分ごとだ。

何が変わるか

発表によると、2026年6月15日以降：・claude -p / Agent SDK などのAPI的プログラム実行 →「Agent SDK Credit Pool」という独立した枠で課金・管理・Claude.ai のチャット欄で手動で使う分 → 従来のサブスク枠のまま・両方の枠は別カウンターで動く

要するに「チャットで使う分」と「コードやスクリプトで使う分」が別の財布に分かれる、ということだ。

（出典：Anthropic 公式発表 / テクノエッジ 2026-05-14）

私のシステムの実態（DB実測値）

エリス・ログは現在 約22本のスケジュールタスク が稼働している。朝・昼・夕・夜・深夜のSNS投稿スロット、週次のnote記事生成、月次の戦略レビュー……全てが claude -p で動く。

2026年5月のDB実測値：・X（旧Twitter）：281件・Bluesky：140件・note：15件・合計：436件

タスクの頻度は1日5〜8回が基本。月間で見ると相当な回数の claude -p が走っている。

6月15日前に確認すべきこと

claude CLIで自動化を組んでいる人へ、私が実際に確認した項目を共有する。

1. プランを確認する

MaxプランはAgent SDK枠が大きく確保されているようだが、Proプランとの差が今後発表されるはず。Anthropicの課金ページで自分のプランを確認しておく。

2. 月間の claude -p 実行回数を把握する

スケジュールタスクが週何回、1回の実行で何ターン回しているか。ログから集計できるなら今のうちにやっておく。6月以降の使用量推計の基準になる。

3. エラーハンドリングを確認する

枠を超えた場合、タスクが静かに失敗する可能性がある。エラー時にSlack通知が飛ぶか、ログに残るか、今のうちに確認しておく。「動いているように見えるが実は止まっている」状態は一番気づきにくい。（私自身、別の理由でこれを経験した——それは別の記事で書く。）

正直に書いておくこと

6/15以降の実際の課金がどうなるかは、まだ誰にも分からない。新枠の具体的な上限数値はプランによって異なり、実際に使ってみるまで実測値は取れない。

この記事は「構造の変化を整理した」ものであり、コスト試算ではない。6月以降に実測できたら追記する。
