# XAIP: AIエージェントがツールを呼ぶ前に見る、署名付き実行receipt

[![Trust Evidence Before Delegation](https://raw.githubusercontent.com/xkumakichi/xaip-protocol/main/docs/evidence-before-delegation.gif)](https://xkumakichi.github.io/xaip-protocol/evidence-before-delegation.html)

この記事は、以前公開した「信頼は持ち運べる」のアップデートです。

今回の主な更新点は、public demo の追加、最新の live dataset、そして  
MCP / LangChain.js callback / OpenAI-compatible tool-call loop からの  
receipt が同じ public trust graph に入ったことです。

XAIP という、AIエージェントのツール呼び出し向けの provider-neutral  
な署名付き実行 evidence レイヤーを作っています。

考え方はシンプルです。エージェントが外部ツールに作業を委譲する前に、  
過去の署名付き receipt から実行 evidence を確認できる状態にする、  
というものです。

XAIP は新しい agent framework ではありません。Runtime の下に座る、  
portable な receipt レイヤーです。Receipt フォーマットは、どの  
runtime が emit しても同じです。

## 現在の receipt の出どころ

ライブで動いている統合:

* MCP サーバー
* LangChain.js callback handler
* OpenAI-compatible tool-call loop

## 現在のスナップショット (2026-05-11)

* public trust graph に 10 サーバー
* 3,239 件の署名付き実行 receipt
* MCP、LangChain.js callback、OpenAI-compatible tool-call loop からの receipt

## デモが見せるもの

public demo はシンプルな対比を示します:

* XAIP なし: 候補ツールの違いが見えにくい
* XAIP あり: 委譲前に、署名済み receipt 履歴、観測された失敗、  
  unscored 候補が可視化される

XAIP は、ツールの安全性や信頼性を保証するものではありません。  
委譲前に、過去の実行 evidence を可視化するための仕組みです。  
Trust score は receipt から導かれる一つの view であり、  
receipt そのものが primary な artifact です。
