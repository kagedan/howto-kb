## はじめに

**日報、正直めんどくさくないですか？**

一日の終わりに、疲れた頭で今日やったことを思い出しながら、毎回同じようなフォーマットに書き込む。内容もだいたい同じ。「今日も商談2件、資料作成、メール対応」——それを丁寧な文章に整えて、提出する。

毎日たった5分かもしれません。でも、**毎日続く5分**は、じわじわとストレスになります。

しかも、日報で一番消耗するのは「書く時間」より「思い出す時間」だったりします。

「あれ、午前中って何してたっけ」「あの商談、今日だったっけ先週だったっけ」——Googleカレンダーを見返して、記憶をかき集めて、ようやく書き始める。

**その作業、全部Claudeに任せられます。**

Googleカレンダーの予定をそのままClaudeに渡すと、日報として整形された文章が返ってきます。あとは確認して送るだけ。所要時間は**30秒**です。

この記事では、その仕組みの作り方を、非エンジニアの方でもできるようにステップで解説します。難しいことは何もありません。必要なのはGoogleカレンダーとClaudeのアカウントだけです。

## 仕組みの全体像

**難しそうに見えて、やることは3ステップだけです。**

まず全体の流れを見てください。

```
① Googleカレンダーで今日の予定を確認する
　　↓
② 予定をコピーしてClaudeに貼り付ける
　　↓
③ 日報が完成する
```

以上です。プログラミングも、特別なツールも、APIの設定も不要です。Googleカレンダーを開いて、予定をコピーして、Claudeに貼るだけ。**スマホでもできます。**

### 必要なものは2つだけ

必要なもの費用備考Googleカレンダー無料すでに使っている方がほとんどClaudeのアカウント無料〜無料プランでも十分動く

### なぜGoogleカレンダーなのか

日報を自動化しようとすると、「今日何をしたか」という情報が必要です。その情報が一番きれいに揃っているのが、**カレンダーの予定**です。

会議・商談・作業時間——普段カレンダーに入れている予定が、そのまま日報の素材になります。逆に言うと、**カレンダーをきちんと使っている人ほど、この自動化の恩恵が大きい**です。

「カレンダーにあまり予定を入れていない」という方も心配不要です。後半で、最低限の入力で最大限の日報を作るコツもお伝えします。

### Claudeは何をしてくれるのか

Claudeの役割は、**「予定の羅列」を「読める日報」に変換すること**です。

たとえば、こんな予定の羅列が：

```
10:00 株式会社A社 定例MTG
13:00 提案資料作成
15:00 △△社 商談
17:00 週報作成・メール対応
```

こんな日報に変わります：

```
【本日の業務報告】

午前中は株式会社A社との定例MTGに参加し、
今月の進捗確認と来月の方針について協議しました。

午後は△△社への提案資料を作成後、商談に臨みました。
提案内容はおおむね好評で、来週中に正式な返答をいただく予定です。

夕方は週報の作成とメール対応を行い、本日の業務を完了しました。
```

予定の羅列が、文脈のある文章になっています。**しかも、毎回ゼロから書かなくていい。**

### 一つ補足：「完全自動」ではありません

正直にお伝えすると、この方法は「ボタン一つで全自動」ではありません。カレンダーをコピーしてClaudeに貼る、という**30秒の作業**は毎日発生します。

ただ、「思い出す・考える・書く」という消耗する作業がなくなります。確認して送るだけになります。それだけで、日報に対するストレスはほぼゼロになります。

## 実際にできあがる日報のサンプル

**百聞は一見にしかず。実際に比べてみましょう。**

### 想定する一日のカレンダー

```
09:30 朝会（チーム全体・15分）
10:00 株式会社A社 訪問商談
13:00 提案書作成（新規案件B社向け）
15:30 社内レビュー（B社提案書）
17:00 メール対応・事務処理
18:00 退社
```

### Before｜手書きの日報

```
【本日の業務報告】

お疲れ様です。本日の業務報告です。

午前中はA社へ訪問し、商談を行いました。
午後はB社向けの提案書作成と社内レビューを実施しました。
その後、メール対応等を行い業務を終了しました。

以上、よろしくお願いいたします。
```

所要時間：約5〜10分（思い出す時間含む）

### After｜Claude自動生成の日報

```
【本日の業務報告】

本日はまず朝会でチーム全体の進捗を確認し、一日の方針を共有しました。

午前中は株式会社A社へ訪問し、商談を実施しました。先方のニーズを丁寧に
ヒアリングし、次回提案に向けた具体的な課題を整理することができました。

午後は新規案件であるB社向けの提案書作成に注力しました。作成後は社内
レビューを実施し、内容のブラッシュアップを行いました。提案書は明日中に
最終仕上げを予定しています。

夕方はメール対応および事務処理を行い、本日の業務を完了しました。
明日はB社提案書の仕上げと、A社へのフォローアップを優先して進めます。

以上、よろしくお願いいたします。
```

所要時間：約30秒（コピー＆ペーストのみ）

### 比較まとめ

Before（手書き）After（Claude生成）所要時間5〜10分30秒文章の質箇条書き寄り・簡素文脈があり読みやすい翌日の予定書き忘れがち自動で追記される精神的負荷高い（思い出す作業）ほぼゼロ毎日の継続しんどい苦にならない

Claude生成の日報は、手書きより**むしろ質が高い**です。毎日5分かけて書いていた日報より、30秒で作った日報のほうが上司に刺さる。これが、自動化の本当の価値です。
