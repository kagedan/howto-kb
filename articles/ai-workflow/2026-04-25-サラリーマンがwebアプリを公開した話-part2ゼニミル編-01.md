## 壁にぶつかった：`window.storage`問題

最初、アプリはClaudeのアーティファクト（プレビュー環境）で動いていました。

データの保存に`window.storage`というAPIを使っていたのですが、これはClaude専用のAPIで**一般のブラウザでは動きません**。

```
// これはClaudeアーティファクト専用。普通のブラウザでは動かない
await window.storage.set(key, JSON.stringify(data), true);
```

デプロイするには、ここを**Firebase Firestore**に置き換える必要がありました。

## 構成：Firebase Firestore + Vercel

| 役割 | サービス | 料金 |
| --- | --- | --- |
| データ保存・同期 | Firebase Firestore | 無料（Sparkプラン） |
| ホスティング | Vercel | 無料（Hobbyプラン） |

**重要：** 両サービスとも、上限を超えても**課金されません**（止まるだけ）。クレジットカードも不要。

## ストレージをFirestoreに置き換え

`window.storage`を以下に置き換えました。

```
// firebase.js
import { initializeApp } from 'firebase/app';
import { getFirestore, doc, setDoc, getDoc } from 'firebase/firestore';

const app = initializeApp({
  apiKey: import.meta.env.VITE_FIREBASE_API_KEY,
  // ... 環境変数から読み込む
});
export const db = getFirestore(app);

export async function saveRoom(roomId, data) {
  await setDoc(doc(db, 'rooms', roomId), data);
  return true;
}

export async function loadRoom(roomId) {
  const snap = await getDoc(doc(db, 'rooms', roomId));
  return snap.exists() ? snap.data() : null;
}
```

ルームIDをURLパラメータ（`?room=zeni-abc123`）で管理しているので、同じURLを開けばパートナーと自動同期されます。

## デプロイ手順

### 1. Firebaseプロジェクト作成

1. [Firebase Console](https://console.firebase.google.com) でプロジェクト作成
2. **Firestore Database** を有効化（テストモードでOK）
3. セキュリティルールを設定

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /rooms/{roomId} {
      allow read, write: if true;
    }
  }
}
```

4. プロジェクト設定からWebアプリを追加して設定値をコピー

### 2. 環境変数の設定

`.env.local`に以下を記入：

```
VITE_FIREBASE_API_KEY=xxx
VITE_FIREBASE_AUTH_DOMAIN=xxx
VITE_FIREBASE_PROJECT_ID=xxx
VITE_FIREBASE_STORAGE_BUCKET=xxx
VITE_FIREBASE_MESSAGING_SENDER_ID=xxx
VITE_FIREBASE_APP_ID=xxx
```

### 3. Vercelにデプロイ

コマンド一発でデプロイできます。その後、Vercelダッシュボードの **Environment Variables** に同じ環境変数を登録してリデプロイ。

## 公開して気づいたこと

**良かった点**

* Firebase + Vercelは本当に無料。維持費ゼロ
* URLを送るだけで共有できるのが思った以上に便利
* Claudeがコードをほぼ全部書いてくれた

**反省点**

* `window.storage`の問題は最初から気づくべきだった
* 環境変数の設定がCLIだと少し手間がかかった（ダッシュボードからやるのが楽）
