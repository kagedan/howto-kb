# 概要

GIS(地理情報システム)ソフトは難しい。

まず、ボタンがたくさんあってどれを押せば良いのかわからない。

![](https://static.zenn.studio/user-upload/6ef290a86cfe-20260627.png)  
*QGIS*

知っておかねばならない前提知識も難しい。

指定した地点の半径5kmにある喫茶店の数を数えたいだけなのに「このデータの単位は度数なのでメートル単位の座標系に変換してから5kmのバッファを作る必要が…」とか訳のわからないことを言われる。

とても辛い。

これこそAIの力を使うべきなのでは？と思ってQGISをAIに操作させるMCPやプラグインを試してみたけれど、そもそもQGISが多機能すぎてこちらのやりたいことに対してAIが複雑すぎる手順を選択してしまったり、プロンプトを深読みしすぎて余計なことをやったりする。

そこで、もうちょっとカジュアルな感じで使えるAIエージェントをUIとしたWebGISを作ってみた。

# WebGIS AI Agent

チャット欄でAIに指示を出すだけで基本的な地理データの操作をAIが実行してくれるWebGIS。

<https://shimizu.github.io/web-gis-ai-agent/>

![](https://static.zenn.studio/user-upload/abcb2488a607-20260627.png)  
*スクリーショット*

※ 利用にはClaude Message APIのAPIキーが必用です。キーはブラウザに保存されClaude Message APIへのリクエストのみに使われます。従量課金制になるので利用の際はかならず上限の設定を。

NotebookLM生成の解説スライド

できることはベクターデータの操作のみだけれど、わりとシンプルで使いやすいものになったと思う。

データの抽出、フィルタリング、結合、差分、バッファの作成、面積や距離の計測、コロプレス図にして表示など基本的なことはAIに任せればよしなにやってくれる。

AIエージェントはフィードバックループを備えているので、データや処理になんらかの問題があったときも自律的に解決策を考えて動作し、どうしてもブラウザ上の仕組みだけで解決が出来ない場合は、ユーザーに解決策を提案してくれたりする。  
とても楽ちんである。

claude APIのキーが必要で従量課金なので使えば使うほど金がかかるのが悩みだけれど、GISソフトの複雑さをお金の力で解決できる。すばらしい。

# アーキテクチャの解説

興味ある人向け。

基本は vite,react,DuckDB-WASMでClaude Messages APIを利用したサーバーレスな構成。

AI エージェントは、ブラウザ上で Claude Messages API の tool use ループを回すことで実装した。DuckDBののクエリを実行するツールを実装し、それらの使い方を記載したskillをClaude Messages APIのsystemフィールドに付加して送ることでツール／スキルの仕組みを作っている。

ユーザーがプロンプトを送信すると、モデルが `run_spatial_sql` や `describe_layer` などのツール呼び出しを返し、クライアント側のランタイムがそれを順番に実行し、結果を `tool_result` として会話履歴へ戻す。コンテキストはブラウザ側のメモリに保存される。  
モデルが指示を完了したと判断するか、設定されているループ上限(30回)を超えるまでループが回りツールを用いた処理が行われる仕組みになっている。

エージェントによる GIS 操作は、DuckDB-WASM と [Spatial](https://duckdb.org/docs/lts/core_extensions/spatial/overview) 拡張を使ったブラウザ内の空間 SQL 処理。アプリでは「1 レイヤー = DuckDB の 1 テーブル」として扱い、ジオメトリ列を `geom` に正規化し、レイヤー ID、表示名、ジオメトリ型、地物数、生成元 SQL などをレジストリに保存している。たとえばバッファや空間結合を行う場合、エージェントはまず `list_layers` や `describe_layer` で実在するレイヤーと列を確認し、DuckDB Spatial の SQL を組み立て、`run_spatial_sql` に `layer_name` を付けて実行する。  
SQL の `SELECT` 結果は新しい DuckDB テーブルとして materialize され、GeoJSON へ変換された描画オブジェクトがレイヤーストアへ追加されることで地図に表示される。

なお、空間結合や範囲検索をブラウザ内で現実的な速度に保つため、アップロードされたデータが大きい時にはテーブルに [R-tree indexs](https://duckdb.org/docs/lts/core_extensions/spatial/r-tree_indexes) を設定してクエリの処理効率を上げている。

その他に、publicフォルダにNutral Earthなどの境界ポリゴンをプリセットとして配置、これらの知識もシステムプロンプトとしてリクエストのsystemフィールドに含めて送信するようになってる。
