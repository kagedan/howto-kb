# AIと人間の関わり方：3パターン

優劣はなく、状況や目的に応じて使い分ける。

![](https://static.zenn.studio/user-upload/34b3a30be923-20260321.png)

# AIの強みと限界を把握する

### 強み

* **汎用性**：異なる多様なタスクを、チューニングなしに同一モデルで実行できる
* **ランダムネス**：多様なアイデア生成に向いている（temperatureで制御可能）
* **会話の記憶**：過去の会話を考慮した回答を生成する（コンテキスト内）
* **接続性**：様々なツールへのアクセス

### 限界

* **学習データの制約**：カットオフ以降の情報は知らない、不正確な情報が混入することもある
* **幻覚（Hallucination）**：学習情報をつなぎ合わせることで誤情報を生成することがある
* **コンテキストウィンドウ**：一度に処理できる情報量に上限がある
* **非決定性**：同じ入力でも出力が異なることがある

# 4Dフレームワーク：AIの活用サイクル

この4つをループさせながら、AIとの協働を洗練させていく。

![](https://static.zenn.studio/user-upload/a96818238b34-20260321.png)

**1. Delegation（委任**）

* 全体像に焦点を当てる
* 「自分がやること」「AIにやらせること」を明確に分ける
* 効果的な委任には、**自分の専門理解** と **AIの得意・不得意の把握** が前提

**2. Description（説明）**

* 会社の価値観・ターゲット・制約など、コンテキストを具体的に伝える
* 「何を達成したいか」「どんなタスクをやってほしいか」を明示する
* AIは心を読めない。具体的な指示・出力形式・対象読者を明記する

**3. Discernment（識別）**

* AIの出力・プロセス・動作を批判的に評価する
* 「要求を満たしているか」「事実に基づいているか」を自問する
* よくある問題：論理的誤り・注意力欠如・循環論理・却下済みアイデアの再出現

**4. Diligence（誠実さ）**

* 倫理・安全性の観点でAIとの協働を評価する
* 最終成果物の責任は常に人間にある

## 委任（Delegation）の深掘り

**委任 = 何を人間がやり、何をAIにやらせるかの判断**

優れた委任は AI とは無関係な、一般的なビジネス・思考スキルが土台

目標が不明確な状態では、AIも正しい出力にたどり着けないし、評価もできない

**ステップ**

1. 問題認識：何を解決したいかを明確化
2. プラットフォーム認識：AIの特性・得意不得意を理解する
3. タスク委任：問題を分解してAIに割り当てる

## 説明（Description）の深掘り

AI = DBでも自動販売機でもない

**インタラクティブなパートナー**として扱う

### 説明の3要素

| 要素 | 内容 |
| --- | --- |
| **モノの説明** | 作成してほしいものを正確に伝える |
| **プロセス説明** | どうやるかの手順・ガイドを示す |
| **パフォーマンス説明** | どう振る舞ってほしいか（やること・やってはいけないこと） |

## 識別（Discernment）の深堀り

AIの出力、プロセス、動作を批判的に評価すること（説明の裏返し）  
受け取ったものが実際のニーズを満たしているかを判断する

「回答は事実に基づいているか、要求を満たしているか」を自問自答する

* 論理的な誤り
* 注意力の欠如
* 誤った手順
* 特定の問題解決に固執して代替案を示せなくなる
* 循環論理
* 却下されたアイデアの混入

AIの振る舞いを都度評価するのも重要

大きな観点は３つ

* 製品識別：出力そのものの品質
* プロセス識別：どのようにして出力に至ったかの評価
* パフォーマンス識別：どのように動作しているか

もし誤りがあった場合、「## 効果的なプロンプトテクニック」を参考に修正を促すと良い

## 誠実さ（Diligence）の深堀り

倫理面、安全性に焦点を当てるもの  
AIとのやり取りに責任を持つ

**このAIと協働する意味はなにか？**

* 倫理的にOKか
* データソースは信頼できるか
* 説明できるか
* 所有権はどうなっているか
* セキュリティは良さそうか
* 組織ポリシーに沿っているか

最終的な成果物の責任は**人間である**

# 効果的なプロンプトテクニック

効果的なプロンプト = 人間同士のコミュニケーションスキル + AI特有の考慮

以下を可能な限り具体的に書く：

1. 自分が何者か・何を望むか・なぜ望むか
2. 例を示す（If A then B → If C then ?）
3. 制約を示す（言語・フォーマット・やってはいけないこと）
4. 段階的な手順を示す
5. 考える時間を与える（推論させる）
6. AIの役割・トーンを定義する

## プロンプトテンプレート（案）

これは完全オリジナル

使っていき、定期的に変更していきたい

```
# 自己紹介
わたしはだれか

# 概要
モノの概要 / 何をしてほしいか / なぜしてほしいか

## あなたの役割
あなたは〇〇です
インタラクティブに会話してください

## 例
こんな感じで

# 制約
これをしてほしい / これはしてはいけない

じっくり考えてください

# 手順
まず〇〇を、次に〇〇を
```

**Tips**

* Claude自身にプロンプトの作成を依頼してもよい
* Claude自身に回答を説明させることで品質を確認できる
* 効果的なプロンプトは反復的。AIと一緒に深めていくのが成功パターン
