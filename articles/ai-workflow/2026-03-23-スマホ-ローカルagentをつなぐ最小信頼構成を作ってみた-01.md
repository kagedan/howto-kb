AIエージェントは、この1〜2年で急速に実用化が進んだ。OpenClawをはじめとするAIエージェントフレームワークも広まり、活用場面が拡大している。  
ただし、多くの応用はサンドボックス環境でのコーディングにとどまっている。

> 例：  
> -リポジトリ解析・コード生成などの自動プログラミング  
> -テスト生成やバグ修正などの開発支援

理由は

AI が個人データや実環境に触れ始めると、  
**信頼境界（trust boundary）** の問題が急に大きくなります。

結果として、AI は何でもできると言われるものの、  
応用場面が 制限され、もったいなく感じています。

---

# このプロジェクトの考え方

!

自律的に動くAgentを安心して使えるように

**モバイル入口と Agent 実行環境を分離する。**

![](https://static.zenn.studio/user-upload/8c3602cc6170-20260313.png)  
*リモートPCでAgentがブラウザを利用した時のスクショ*

![](https://static.zenn.studio/user-upload/6817ecbc9943-20260313.png)  
*リスクある操作実行する際に人間が権限を確認*

スマホは **操作インターフェース**。  
AI エージェントは **自分の環境で実行** します。

```
Telegram（スマホ）
      ↓
Webhook（制御ゲート）
      ↓
ローカル Agent
```

つまり：

**スマホはリモコン、AI はローカルで動く。**

---

# Webhook の役割

Webhook は Agent を実行しません。

役割は次の 4 つだけです。

* メッセージを受け取る
* 権限とポリシーを確認する
* ローカル Agent に実行を渡す
* 結果を返す

つまり Webhook は

**信頼境界（Trust Gateway）**

として機能します。

---

# システム構成

全体構成はかなりシンプルです。

```
Telegram（スマホ）
      ↓
Cloudflare Tunnel（HTTPS）
      ↓
Webhook Server
      ↓
Local Agent（Copilot CLI など）
```

役割は次の通りです。

| 層 | 役割 |
| --- | --- |
| Telegram | モバイル UI |
| Tunnel | セキュアな入口 |
| Webhook | 制御ゲート |
| Agent | 実行 |

重要なのは：

**Agent は常にローカルで実行されること。**

---

# オープンソースプロジェクト

この構成は、以下のオープンソースとして公開しています。

**telegram-copilot-cli-webhook**

<https://github.com/cqyefeng119/telegram-copilot-cli-webhook>

このプロジェクトは、

Telegram をモバイルインターフェースとして使い、  
ローカルで実行される AI Agent を  
安全に操作するための **Webhook ベースの最小構成**です。

主な機能：

| 機能 | 説明 |
| --- | --- |
| Telegram Bot | スマホから Agent を操作 |
| Webhook Gateway | モバイル入口と Agent を分離 |
| Policy / Approval | 権限確認と承認フロー |
| Local Agent Execution | Agent はローカル実行 |
| Audit Log | 実行イベントの記録 |

---

# 実現できるユースケース

この構成を使うと、例えば次のようなことができます。

| シナリオ | 運用方式 |
| --- | --- |
| 飲食店予約 | Agent がブラウザ操作 → スクリーンショット確認 |
| ドキュメント整理 | ローカルディレクトリのみ処理 |
| メッセージ下書き | 下書き生成 → 人間が確認 |
| フォーム入力 | Agent が入力候補を作成 |

重要なのは、

**Agent が勝手に完結しないこと。**

多くの場合は

1. Agent が提案
2. 人間が確認
3. 実行

という流れになります。

---

# 設計のポイント

このプロジェクトは次の 4 つを重視しています。

| 特徴 | 説明 |
| --- | --- |
| 高透明性 | 操作が可視化される |
| 高制御性 | 権限を段階的に付与 |
| 低複雑性 | 単一 Agent 構成 |
| 拡張性 | Agent を自由に交換可能 |

例えば Agent は次のものに置き換え可能です。

* Copilot CLI
* Claude Code
* 自作 Agent
* 別の LLM ツール

このプロジェクトは  
**Agent を固定するフレームワークではありません。**

**Agent を遠隔操作するための入口です。**

---

# セットアップ（概要）

前提条件：

* Windows + PowerShell
* Python 3.11+
* `uv`
* Copilot CLI
* Telegram Bot Token

基本セットアップ：

```
.env.example を .env にコピー
BOT_TOKEN を設定
uv sync
```

その後、サーバーを起動：

Cloudflare Tunnel を使用することで  
ローカルサーバーを安全に公開できます。

---

# まとめ

AIエージェントの能力は急速に上がっています。

しかし、実際の運用では

**信頼境界の設計**

が大きな課題になります。

このプロジェクトで自律的に動くAgentを安心して使えるように

を分離することで

**最小の信頼経路（minimal trust path）**

を作ってみました。
