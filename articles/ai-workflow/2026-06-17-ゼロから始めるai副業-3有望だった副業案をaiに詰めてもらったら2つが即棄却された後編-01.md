## 案B：LINEボット月額サブスク → 棄却

LINEボットのサブスクは技術的には難しくありませんでした。

既存のスキル（LINE Messaging API・FastAPI・Claude API・Supabase）が全部使えるし、月額1,500〜3,000円で5〜10件集めれば月1〜3万円になる計算でした。

ところが調べてみると、**LINE公式が2025年11月にAIチャットボット（β）を月額3,000円で純正リリース済み**でした。

個人開発者が価格・信頼性・サポートの面でLINE公式に太刀打ちできません。さらに保守工数が月4〜8時間かかる見込みで、「週1〜2時間」という制約にも引っかかりました。

棄却です。

## 案C：RakuRaku予約の他業種横展開 → 棄却

こちらも理屈の上ではよさそうな案でした。

開発中の脱毛サロン向け予約SaaSをネイルサロンや整体院に転用するだけ。追加開発は最小限で、月額3,000〜5,000円 × 3〜4社で月1〜2万円が見込めました。

ただ、予約システム市場はすでに成熟しきっていました。

**STORESやairREGIなど、無料で使える競合が29種類以上**存在しています。「月3,000円の個人製品」と「無料の大手サービス」を比べたとき、よほど強い差別化がないと選ばれません。

差別化ポイントを考え始めたら、開発工数が一気に膨らみそうでした。

棄却です。

## 最終的に残った案：Python × Claude API 業務自動化

2案が消えた後、Claudeが提案してきたのが「**Python × Claude API 業務自動化スクリプトの受託 → 月額保守モデル**」です。

具体的にはこんなイメージです。

* ExcelリストをClaude APIで処理 → 商品説明文を一括生成 → 月額3,000円
* 採用担当の箇条書き → 求人票を自動生成 → 月額5,000円
* 請求書CSV → 摘要を整形 → 会計ソフト出力 → 月額3,000円

月1万円の達成に必要な顧客は**2〜4社**。不特定多数を相手にするSaaSではなく、少数の固定客に月額保守で入ってもらうモデルです。

## なぜこの案が残ったのか

理由は4つあります。

**① 工数が制御できる** 顧客数を2〜4社に絞れば、週1〜2時間の制約に収まる。

**② SNS不要** クラウドワークスへの出品か、知人紹介で完結できる。マーケティングが苦手な自分には助かります。

**③ 差別化できる** Python単体のフリーランサーにClaude APIを組み合わせると、自動化の幅が一段上がる。同じ価格帯の競合と比べてできることが多い。

**④ 競合がまだ薄い** Claude APIを業務自動化に使いこなせる個人開発者は2026年時点でまだ少数です。

## 「調べてから参入する」が一番の節約

今回のリサーチで実感したのは、「簡単そうに見える副業ほど、すでに競合が強い」という現実です。

LINEボットも予約システムも、アイデア段階では「需要がありそう・スキルが合っている」と見えていました。でも調べると、すでに大手が無料でやっていました。

逆に言えば、「参入前に調べる」こと自体に大きな価値があります。

Claudeに「この副業の競合を批判的に調べて」と頼むだけで、1〜2時間かかる調査が数分で終わります。参入後に「やっぱり無理だった」と気づくより、参入前に「やめておこう」と判断できる方がずっといい。

副業を考えている人にはまず、「すでに誰かが無料でやっていないか」を確認することを勧めます。
