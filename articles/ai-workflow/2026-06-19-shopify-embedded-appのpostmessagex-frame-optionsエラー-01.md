開発の中で、一番長く苦しんだバグの話をします。

「仕入れFX」のコア機能(商品原価のCRUD、為替アラート、ダッシュボード)は、わりとあっさり動きました。MVPとしては完成していたんです。

ところが、たった一つ、どうしても消えない問題が残りました。

**ホーム画面(/app)を開いたときだけ、コンソールにエラーが出て、ときどき画面が真っ白になる。** しかもサブページ(商品原価ページ、アラートページ)は完璧に動く。ホームだけがおかしい。さらに毎回ではなく「時々」起きる。

この回は、その犯人にたどり着くまでの記録です。先に言っておくと、犯人は一人ではありませんでした。同じ「真っ白」という症状の裏に、性質の違う原因が3つ重なっていた。だから一つ直しても症状が残り、何日もループしました。同じエラーで検索してここに来た人のために、まず症状を正確に書きます。

## 症状:2つのコンソールエラーと、真っ白な画面

ホーム画面で出ていたエラーは、この2つでした。

```
1. Failed to execute 'postMessage': target origin
   (アプリのURL) does not match
   recipient window (https://admin.shopify.com)

2. Refused to display 'https://accounts.shopify.com/'
   in a frame because it set 'X-Frame-Options' to 'deny'
```

Shopifyアプリを作ったことがある人なら、これを見た瞬間に胃が痛くなるはずです。ざっくり訳すとこうです。

Shopifyアプリは管理画面の中にiframe(埋め込み枠)として表示されます。その枠の中で、よりによってShopifyのログイン画面を開こうとして拒否されている——これが画面が真っ白になる正体でした。

## なぜ後回しにできなかったか

最初、私はこのエラーを軽く見られそうになりました。見た目に影響しないときもあったからです。あるときAIに「コンソールのエラーは表示には影響していません。後で対処しましょう」と言われました。

でも私は引っかかりました。私はAIにこう返しています。

> 「UXに影響ないかもだけど、絶対審査通らないよ。どうにか解消しないとでしょ?」

これは正しい直感でした。Shopifyの審査では、コンソールエラーゼロが事実上の必須条件です。とくにX-Frame-Options関連のエラーは、埋め込みアプリとして根本的に何かを間違えている証拠なので、確実に落ちます。「動いているからいい」ではなく「エラーが出ている時点でアウト」。後回しにできない、リリースを左右する問題でした。

## 謎:なぜ「ホーム画面だけ」「時々」なのか

ここが地獄の入り口でした。

* ホーム画面(/app):エラーが出て、時々真っ白
* サブページ:完璧に動く

同じアプリ、同じ認証の仕組みなのに、なぜ入り口のホームだけが、しかも毎回ではなく時々落ちるのか。この「時々」が、犯人を見えなくしました。再現性の低いバグは、原因の切り分けが極端に難しい。エラーが出たり出なかったりするので、「直った」と思っても、本当に直ったのか、たまたま今回は出なかっただけなのか、判別がつかないんです。

そして実際、私とAIはここから何日も格闘しました。何度も「直った」と思っては裏切られた。理由は後から分かります——犯人が3人いて、しかも全員が「画面が真っ白になる」という同じ結末に流れ着いていたからです。

ここから先は、その3人の犯人の正体と、誤った仮説をどう潰し、最終的にどう直したかの全記録です。同じエラーで検索してここに来た人へ:この領域は日本語の一次情報がほとんどありません。私が溶かした数日を、ショートカットしてください。
