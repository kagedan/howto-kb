## はじめに

本連載はスターウォーズの作品時系列をClaudeアーティファクトで作成した際にアーティファクトの可能性を感じたことから他に何が出来るかをClaudeに聞いていたら地球儀アプリに発展した、という記事の連載となります。

一連の連載では該当アプリの紹介と共にどのようなこと考えながらアプリ作成を進めていったのかをざっくばらんに記載していきます。

![](https://assets.st-note.com/img/1781480215-bFC70I9LznGUfZV2hkuORt3v.jpg?width=1200)

WEBアプリに接続すると表示されるメイン画面  
世界各国の都市や自国のタイムゾーンへのジャンプやその地域の時刻で進む早送り機能、  
季節における太陽直下点の位置などを表現

## 基本情報

**この辺りは毎回、記載していますのですっ飛ばしていただいても問題有りません**

* アプリ名称：**sphere** (プロジェクトコード：E4R7H)
* 公開URL(上記): [**https://sphere.e4r7h.app**](https://sphere.e4r7h.app/)
* 位置づけ: 個人制作・非商用。
* 学習/鑑賞用のインタラクティブ地球儀&天体ビジュアライザ
* PCでもMacでもスマホ(推奨)のブラウザでも参照可能
* 制作期間：2週間、実施10日程度(Max5xでトークン回復待ち期間有り)  
  ※公開してからは不具合やちょっとした部分の修正などで更新しています

## アーキテクチャ

**この辺りもすっ飛ばしてください**

* 全コードが単一ファイル src/App.jsx(約15,400行)に収まる単一JSXアプリ
* 描画は **Canvas 2D API**(3Dライブラリ不使用、依存最小)
* ビルド: **Vite + Tailwind v3 + React 19**(Node 24)、**Vercel** で静的配信
* 設計上は Claude API 連携可能　　　　　　　　　　　　　　　　claude.ai アーティファクト=直接 / 外部サーバ=薄いプロキシ、をホスト名で自動判定。公開版は課金回避の為、無効化
* **現在は API 不使用の静的サイト**　　　　　　　　　　　　　　　　　　ニュースは静的ダミー、サーバ処理なし
* Claude Artifact 上で開発 → Vite/Vercel へ移植

## ざっくりした機能一覧

**すっ飛ばしてください**

* **通常モード(地球儀)** — 経緯線つき地球儀を回す基本ビュー。都市の色分け、ニュース点表示、日食/月食の自動演出に対応
* **マルチビュー(公転追尾・3画面連動)** — 地球の公転を追い、メイン+情報スフィア(昼夜/月相/季節)+ミニ俯瞰を連動表示
* **ソーラーモード(太陽系)** — 太陽中心に水星〜海王星をリング配置。鑑賞用に地球儀の重ね表示も実装
* **ギャラクシーモード(天の川銀河)** — 真上から見た天の川をドラッグで自由回転。鑑賞用に地球儀重ね表示も実装
* **ナイトモード** — 夜側の街明かりを暖色の点で表現
* **海溝モード** — マリアナ・日本海溝・トンガ等の主要海溝を線でオーバーレイ
* **パンゲアモード** — 約2.5億年前のパンゲア→現代の大陸移動を時代スライダーで再生、データが不足していて日本列島は最初から日本列島．．．

![](https://assets.st-note.com/img/1781480948-i1BbWgLZJSKzQmXA0HPtoeEh.jpg?width=1200)

## 伝わらない要望と修正されない不具合の対処としてスクショ/ペン入れメソッドに移行

以前からchatGPTやClaudeでスクリーンショットやなんらかの画像をアップロードしたりすることはあったのですが、本アプリ作成を進めるにあたって自然言語による指示について早い段階で機能しない＆限界を感じた為、というよりテキストで説明することが面倒になった為、スクリーンショットを貼り始めたところ案外、機能することが判明。

![](https://assets.st-note.com/img/1781481519-lwNXUcvnpkqPIoYaCLEWeK0z.png?width=1200)

地表塗りつぶしを依頼して試行錯誤するも満足いく形にならなかった為、  
断念して現在の形に．．

---

Claude側は実際に動作している画面が見えている風を装うこともあるが、基本はコードレベルでの理解や内部的なエラーの有無(普通に開くとエラーになることもある)による思考動作のような実際のアプリ画面を貼ってあげるとご認識することもあるが、作成しているアプリの仕様と照らし合わせて配置やその後にどうすればいいかを自立して進められることが出来る。**もちろん、30ラリーくらい全く進まないこともある。**

![](https://assets.st-note.com/img/1781482438-tVLkaZp394nCFr2P5MxbyvHB.jpg?width=1200)

修正依頼した画面のボタン配置やデザインについてスクリーンショットを共有したことに対するClaudeの応答

## 赤ペン先生

新しい機能のボタン配置を自然言語(テキスト)で依頼しても気を利かせて一発で通ることもあるものの大抵は画面が見えていない感が丸出しな箇所にプロットすることもある為、スクリーンショットした画像に赤ペンを入れて再依頼したところ、ちゃんと理解してくれる。**勿論、20ラリーくらいすることもある。**

ただ、この辺りからClaudeからは「スクショをお願いします」「赤入れ待っています」がデフォルト要求となりますw

![](https://assets.st-note.com/img/1781483185-wvlRSkbz1WA74DYrQfdhTtmj.png?width=1200)

スライド設置を依頼するもの極小スライドになってしまったのでスクショ + 赤入れで攻める
