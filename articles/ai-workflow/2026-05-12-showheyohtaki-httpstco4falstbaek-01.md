OpenAI Codex CLIの「/goal」が話題になっています。
「数時間動き続けて、複雑なゲームを丸ごと作った」
「ゴールを渡すだけで、完了するまで自律的に動く」
こういうポストを見て「すごそう」とは思うものの、肝心の使い方がわからない。そんな人、多いんじゃないでしょうか。

/goal はデフォルトでは使えません。
なのに有効化の手順も、良いゴールの書き方も、まとまった情報がほとんどない。

そこで、この記事で全部まとめました。
有効化の手順からゴール設計まで、読み終わったらすぐ実践できる内容です。

━━━━━━━━━━━━━━

## /goal とは

Codex CLIに持続的な目標を設定する機能です。

通常のチャットは「1回の指示→1回の回答」で終わりますが、/goal を設定すると Codex が「計画→実行→テスト→修正」のループを自律的に回し続けます。

つまり「**ゴールに到達するまで、勝手に考えて、勝手に直して、勝手に完成させる**」モードです。

![](https://pbs.twimg.com/media/HIIVlqbaMAAj8H2.jpg)

━━━━━━━━━━━━━━

## STEP 1: 有効化する

/goal はデフォルトでオフになっています。

ターミナルで以下のコマンドを実行してください。Codex を起動する前に打ちます。

```plaintext
codex features enable goals 
```

これで完了です。

**有効化できたか確認する**

```plaintext
codex features list
```

一覧の中で goals が true になっていればOKです。

**その起動だけ一時的に有効にしたい場合**

```plaintext
codex --enable goals
```

恒久的に設定を変えたくないときに便利です。

> **注意**: バージョン 0.128 以降が必要です。
使えない場合は codex --version を確認し、codex update などで最新版にしてください。

> CLIで goals を有効化すると、同じ設定を使うCodexデスクトップアプリでもGoal機能が使えるようになります。
ただしデスクトップアプリでは /goal コマンドとして入力するより、「Goalを『◯◯』にして」と自然文で依頼する形になります。

![](https://pbs.twimg.com/media/HIIcvSsaUAExaCh.jpg)

## STEP 2: 基本の使い方

有効化したら、すぐ使えます。

**ゴールを設定する**

```plaintext
/goal サイト全ページのmeta情報とtitleタグを最適化する 
```

これだけです。Codex がゴールを解釈して、計画を立てて、自律的にコードを書き始めます。

**状態を確認する**

```plaintext
/goal 
```

引数なしで実行すると、今のゴールと進捗が表示されます。

**一時停止・再開・解除**

```plaintext
/goal pause    ← 一時停止 
/goal resume   ← 再開 
/goal clear    ← ゴールを解除 
```

**1つ注意点があります。** ゴールは同じスレッド内で変更できません。別のゴールを設定したい場合は、新しいセッションを開始してください。

━━━━━━━━━━━━━━

## STEP 3: 良いゴールの書き方

ここが一番重要なところです。

**曖昧なゴールが壊れる2つのパターン**

「いい感じに直して」「サイトを改善して」のような曖昧なゴールを渡すと、Codex は2つの壊れ方をします。

**パターンA: 数分で諦める
**ゴールの達成基準が曖昧すぎて、「もうできたかも？」と早期判断してしまう。実際には何も改善していないのに、ループを抜けてしまいます。

**パターンB: 永遠に止まらない
**逆に「まだ改善できるかも」と延々コードを変え続けるケース。達成の判定基準がないので、いつまでも完了しません。

どちらも原因は同じです。「何をもって完了とするか」が定義されていない。

この問題を防ぐために、良いゴールには4つの要素を入れます。

1. **何を達成するか（ゴール）
**「何をしてほしいか」を具体的に。
1. **何を変えないか（制約）
**既存の機能やデザインを壊さないための安全弁。
1. *
