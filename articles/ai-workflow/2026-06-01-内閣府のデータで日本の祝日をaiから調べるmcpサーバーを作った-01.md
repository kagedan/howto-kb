## 作ったもの

Claude Desktopに「今日は祝日ですか？」と聞くと、内閣府の公式データをリアルタイムで調べて答えてくれるMCPサーバーを作りました。

![](https://static.zenn.studio/user-upload/adcce4f3a93f-20260601.png)

「japan-holiday-mcp連携を使用しました」という表示が出て、MCPサーバーが実際に呼ばれているのがわかります。

使えるツールは4つです。

| ツール名 | 何ができるか |
| --- | --- |
| `ping` | サーバーが動いているか確認 |
| `is_holiday` | 指定した日が祝日かどうか判定 |
| `get_holidays_in_month` | 指定した年月の祝日一覧を取得 |
| `get_next_holidays` | 次のN件の祝日と「あと何日か」を取得 |

![](https://static.zenn.studio/user-upload/77de448bc285-20260601.png)

GitHubとSmitheryで公開しています。

## なぜ作ったか

### データの信頼性が決め手だった

祝日データを扱うサービスはいくつかありますが、「内閣府が公式に配布しているCSV」を直接使っているMCPはほぼ見当たりませんでした。

内閣府のデータは法律に基づいて管理されており、振替休日や国民の休日も正確に反映されています。AIに祝日を聞くなら、一次情報源を使うのが一番信頼できると判断しました。

### Japan Data MCPシリーズの1本として

私はノンエンジニアとして、日本の公官庁データをAIから使えるMCPシリーズを作っています。

| シリーズ | データ元 |
| --- | --- |
| japan-holiday-mcp（本記事） | 内閣府 |
| japan-weather-mcp | 気象庁 |
| japan-realestate-mcp | 国土交通省 |

すべてCloudflare Workersで動いており、無料枠で運用しています。

## 技術的なポイント

### Shift-JIS変換

内閣府が配布している祝日CSVは「Shift-JIS」という古い文字コードで書かれています。現代のプログラムが標準で使う「UTF-8」とは異なるため、読み込み時に変換処理が必要でした。

```
// Shift-JISのCSVをUTF-8に変換
const buffer = await response.arrayBuffer();
const decoder = new TextDecoder('shift-jis');
const text = decoder.decode(buffer);
```

### メモリキャッシュで高速化

毎回内閣府のサーバーにアクセスすると遅くなるため、一度取得したデータをメモリ上にキャッシュする仕組みを入れました。2回目以降は即座に返せます。

### Cloudflare Workersで無料運用

サーバーはCloudflare Workersにデプロイしています。1日10万リクエストまで無料で、世界中のエッジサーバーから低遅延で応答します。

## つまずいたポイント（正直に書きます）

### `new_classes` → `new_sqlite_classes` の罠

デプロイ時に一番ハマったのがこれです。

```
# ❌ これで本番デプロイが失敗した
[[migrations]]
tag = "v1"
new_classes = ["HolidayAgent"]

# ✅ 正解はこちら
[[migrations]]
tag = "v1"
new_sqlite_classes = ["HolidayAgent"]
```

Cloudflare Workersの無料プランでは `new_classes` は使えず、`new_sqlite_classes` と書かなければなりません。公式ドキュメントに記載はあるのですが、ネットで見かけるサンプルコードが古い書き方のままになっていることが多く、気づくのに時間がかかりました。

### PowerShellのcurlは使えない

ローカルで動作確認しようとしてPowerShellで `curl` を叩いたら、SSEハンドシェイクエラーが出て応答しませんでした。

MCPサーバーの動作確認は**Claude Desktopで直接試す**のが正解です。ターミナルからのcurlテストにこだわって時間を無駄にしないように。

## Smitheryへの登録

Smitheryは、MCPサーバーを公開・発見できるプラットフォームです。登録することでAIエージェントのユーザーに自然に発見してもらえます。

登録後のスコアは **55/100** でした。

改善の余地はありますが、まず公開することを優先しました。スコアを上げるには**説明文・タグ・アイコンの整備**が効果的です。今後アイコンを設定して再チャレンジ予定です。

登録で必要なのは**稼働中のサーバーURL**です。GitHubのURLではなく、Cloudflare WorkersのURLを入力します。

```
https://japan-holiday-mcp.tsukuras-jp.workers.dev/mcp
```

## 使い方

### Claude Desktopに設定する

`claude_desktop_config.json` に以下を追加します。

```
{
  "mcpServers": {
    "japan-holiday-mcp": {
      "command": "npx",
      "args": [
        "-y",
        "@smithery/cli@latest",
        "run",
        "kakar-satoshi/japan-holiday-mcp",
        "--key",
        "your-smithery-api-key"
      ]
    }
  }
}
```

設定後にClaude Desktopを再起動すると使えるようになります。
