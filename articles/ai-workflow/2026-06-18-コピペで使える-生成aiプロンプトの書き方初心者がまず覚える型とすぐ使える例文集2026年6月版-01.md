生成AIを使ってみたけれど、思ったような答えが返ってこない。そう感じている初心者向けに、プロンプト（AIへの指示文）の書き方を、コピペできる例文つきでまとめます。

先に言ってしまうと、返ってくる答えの質は、AIの性能より、指示の書き方で大きく変わります。同じ ChatGPT や Claude でも、頼み方ひとつで結果がまるで違う。自分は仕事で毎日 AI にコードや文章を書かせていますが、うまくいくときといかないときの差は、たいてい指示の側にありました。

この記事は、初心者がまず覚える「型」を一つ示して、そのあと、そのまま使える例文をたくさん並べます。難しい用語は使いません。長めなので、目次から必要なところだけ拾って、例文をコピペしてもらえれば十分です。保存しておいて、使うときに開く、という読み方を想定しています。

:::note info
この記事は ChatGPT・Claude・Gemini など、チャット形式の生成AI全般で使える内容です。特定のサービスに依存しません。例文はコピペして、カッコの中を自分の内容に置き換えれば、すぐ使えます。
:::

## なぜ、頼み方で結果が変わるのか

AIは、こちらが言ったことだけを手がかりに答えます。人間の同僚なら「いつもの感じで」で通じますが、AIには、その「いつもの」が分かりません。だから、ざっくり頼むと、ざっくりした答えが返ってきます。

逆に言うと、何を、誰向けに、どんな形で欲しいかを伝えるほど、答えは狙いに近づきます。プロンプトを書くというのは、難しい呪文を覚えることではなく、頼みごとを具体的にすることです。

ここを勘違いして、特殊な言い回しや裏技を探しに行く人が多いです。でも、最初に効くのは、テクニックより具体性でした。誰に何をどう、を言葉にするだけで、返ってくるものは見違えます。この記事の例文も、特別な呪文は使っていません。全部、頼みごとを具体的にしているだけです。

## まず覚える型：目的・相手・形式・制約

初心者が最初に覚えるなら、この4つを意識するだけで十分です。

- 目的 … 何をしてほしいか
- 相手 … 誰が読む・使うのか
- 形式 … どんな形で欲しいか（箇条書き、表、文章、コードなど)
- 制約 … 文字数、トーン、条件など

全部を毎回きっちり書く必要はありません。ただ、答えがイマイチなときは、この4つのどれかが抜けていることが多いです。困ったら、この4つを順番に確かめると、原因が見つかります。

たとえば「会議の議事録をまとめて」だけだと弱い。これを4つで補うと、こうなります。

```
次の会議メモを、議事録にまとめてください。
- 相手：会議に出ていない上司が読む
- 形式：決定事項・宿題・次回予定の3項目に分けて、箇条書き
- 制約：全体で400字以内、です・ます調

会議メモ：
（ここにメモを貼る)
```

頼みごとが具体的になっただけで、返ってくるものが安定します。やっていることは、特別なことではありません。人にものを頼むとき、自然と添えている情報を、AIにも省かずに言っているだけです。

## 弱い頼み方と、強い頼み方

同じ用件でも、書き方でこれだけ変わる、という例を並べます。前半が弱い例、後半が、型で補った強い例です。

文章を書くとき。弱いのは「商品の紹介文を書いて」。強いのは「30代向けの腕時計の紹介文を、オンラインストアの商品ページ用に、200字で、落ち着いた大人っぽいトーンで書いて」。

要約するとき。弱いのは「これ要約して」。強いのは「これを、内容を知らない人向けに、箇条書きで5点に、専門用語は避けて要約して」。

調べるとき。弱いのは「Reactについて教えて」。強いのは「プログラミング未経験の人に、Reactが何のための道具かを、たとえ話を交えて、300字くらいで説明して」。

メールのとき。弱いのは「お詫びのメール書いて」。強いのは「納期が3日遅れることのお詫びメールを、取引先の担当者向けに、言い訳がましくならないトーンで、代わりの納期を入れて書いて」。

差は、才能でも呪文でもありません。誰に・何を・どんな形で・どれだけ、を足したかどうか、だけです。この差を、以降の例文で具体的に見ていきます。

## そのまま使える例文集：文章まわり

カッコの中を自分の内容に置き換えれば、すぐ使えます。よく出る場面から並べます。

### 文章を直してもらう

```
次の文章を、もっと読みやすく直してください。
- 相手：（社外のお客様)
- トーン：（ていねいだが、固すぎない)
- 制約：意味は変えず、長くしない

文章：
（ここに文章を貼る)
```

「読みやすく」だけだと方向が定まらないので、相手とトーンを足すと、直し方が安定します。

### 文章をチェックしてもらう

```
次の文章を、誤字脱字と、分かりにくい箇所だけ指摘してください。
- 形式：指摘の箇所と、直し方を、対で
- 制約：文章全体を書き換えず、直す場所だけ示す

文章：
（ここに貼る)
```

「全部書き換えて」ではなく「直す場所だけ示して」と頼むと、自分の文章を保ったまま直せます。

### 長い文章を要約する

```
次の文章を要約してください。
- 相手：（内容を知らない人)
- 形式：箇条書きで5点まで
- 制約：専門用語は、かんたんな言葉に言い換える

文章：
（ここに貼る)
```

要約は「何点まで」「誰向け」を指定すると、ちょうどいい粒度になります。

### 文章のトーンを変える

```
次の文章を、（カジュアルな)トーンに書き換えてください。
- 相手：（SNSのフォロワー)
- 制約：内容は変えず、固い言い回しだけやわらかくする

文章：
（ここに貼る)
```

同じ内容でも、相手に合わせてトーンを変えたいときに使えます。

### 箇条書きを文章にする（その逆も)

```
次の箇条書きを、つながった文章にしてください。
- 相手：（報告書を読む上司)
- 制約：です・ます調、3段落くらいにまとめる

箇条書き：
（ここに貼る)
```

メモを清書するときに便利です。逆に「この文章を箇条書きにして」も、同じ要領で頼めます。

## そのまま使える例文集：仕事の連絡

### メールの下書きを作る

```
次の用件で、メールの下書きを作ってください。
- 相手：（取引先の担当者)
- 目的：（来週の打ち合わせの日程調整)
- トーン：ていねい、簡潔
- 制約：候補日を3つ入れる

用件：
（ここに要点を書く)
```

### お詫び・お断りの文面を作る

```
次の状況で、（お断り)の文面を作ってください。
- 相手：（依頼をくれた取引先)
- トーン：申し訳なさは伝えつつ、きっぱり
- 制約：理由は簡潔に、代わりの案を1つ添える

状況：
（ここに事情を書く)
```

言いにくい連絡ほど、トーンの指定が効きます。

### 返信の下書きを作る

```
次のメールへの返信を、下書きしてください。
- 相手：（社外の方)
- 方針：（日程は了承、場所だけこちらから提案)
- トーン：ていねい

受け取ったメール：
（ここに貼る)
```

受け取った文面を貼って、方針だけ伝えると、返信が速く作れます。

### チャットの短い連絡を整える

```
次の内容を、（社内チャット)用の短い連絡文にしてください。
- 制約：3行以内、結論から、絵文字なし

内容：
（ここに貼る)
```

## そのまま使える例文集：考えをまとめる

### アイデアを出してもらう

```
（新しいブログ記事)のアイデアを10個出してください。
- 相手：（WordPressを使う個人開発者)
- 形式：タイトル案だけ、箇条書き
- 制約：初心者でも読めるテーマにする
```

アイデア出しは、数を指定すると量が安定し、相手を指定すると的が絞れます。

### 表で整理してもらう

```
次の内容を、表にまとめてください。
- 形式：縦に項目、横に（比較する対象)
- 制約：1マスは短く、一目で分かるように

内容：
（ここに貼る)
```

### 賛成と反対の両方を出してもらう

```
（この案)について、賛成の理由と反対の理由を、両方挙げてください。
- 形式：賛成3つ、反対3つ、箇条書き
- 制約：どちらも同じくらいの強さで

案：
（ここに書く)
```

一方に偏らせず、両論を見たいときに使えます。判断の材料を集めるのに向いています。

### 段取りを組んでもらう

```
（やりたいこと)の段取りを、順番に並べてください。
- 形式：ステップを番号つきで
- 制約：初心者がつまずきそうな所に、ひとこと注意を添える

やりたいこと：
（ここに書く)
```

### 質問で考えを引き出してもらう

```
（決めたいこと)について、私が考えを整理できるように、質問を5つ投げてください。
- 制約：一度に全部ではなく、1問ずつ

決めたいこと：
（ここに書く)
```

AIに答えさせるのではなく、自分に問いを投げてもらう使い方です。考えがまとまらないときに効きます。

## そのまま使える例文集：学ぶ・調べる

### かんたんに説明してもらう

```
（知りたいこと)を、初心者向けに説明してください。
- 相手：（その分野をまったく知らない人)
- 形式：たとえ話を1つ入れて、300字くらい
- 制約：専門用語を使うときは、必ず言い換えを添える

知りたいこと：
（ここに書く)
```

### 違いを教えてもらう

```
（AとB)の違いを教えてください。
- 形式：表で、観点ごとに左右に並べる
- 制約：初心者がどちらを選ぶべきかの目安も、最後に一言

比べたいもの：
（ここに書く)
```

似た2つの違いは、表で頼むと頭に入りやすいです。

### 翻訳してもらう

```
次の文章を、（英語)に翻訳してください。
- 相手：（海外のお客様)
- トーン：（ていねいなビジネス文面)
- 制約：直訳ではなく、自然な言い回しにする

文章：
（ここに貼る)
```

翻訳も、相手とトーンを足すと、固すぎたり砕けすぎたりを防げます。

### コードについて聞く（少し慣れてきたら)

```
次のコードが何をしているか、初心者向けに説明してください。
- 形式：1行ずつではなく、全体の流れを段落で
- 制約：専門用語には、かんたんな補足をつける

コード：
（ここに貼る)
```

## 用途別の早見表

どの指定を足すと効くか、場面から逆引きできるようにしておきます。

| やりたいこと | 特に効く指定 |
|---|---|
| 文章を書く・直す | 相手とトーン |
| 文章をチェックする | 直す場所だけ、の制約 |
| 要約する | 何点まで、誰向け |
| トーンを変える | 変える前と後のトーン |
| メールを書く | 相手、目的、条件 |
| お詫び・お断り | トーン、代わりの案 |
| アイデアを出す | 数、相手 |
| 段取りを組む | 番号つき、注意点 |
| 説明してもらう | 相手のレベル、たとえ話 |
| 違いを知る | 表、選ぶ目安 |
| 翻訳する | 相手、トーン、自然さ |

迷ったら、この表の「特に効く指定」を足してから頼むと、外しにくくなります。

## もう一歩、質を上げる小さな工夫

型に慣れてきたら、次の工夫を足すと、さらに安定します。

ひとつ。役割を与える。冒頭に「あなたは（編集者)です」と立場を指定すると、答えのトーンが寄ります。校正なら編集者、設計の相談ならエンジニア、というように、立場を一言足すだけで、返ってくる視点が変わります。

ふたつ。例を見せる。「こんな感じで」と完成例を1つ渡すと、AIはそれに合わせてきます。言葉で説明するより、例を見せるほうが速いことが多いです。自分も、欲しい形が言葉にしづらいときは、近い見本を1つ貼ってしまうほうが、結局そのほうが早いと感じます。

みっつ。一度で完璧を狙わない。返ってきた答えに「もっと短く」「3つ目だけ詳しく」と追加で頼む。会話を重ねて寄せていくほうが、最初の一発に全部を詰め込むより、うまくいきます。AIとのやり取りは、一往復で終わらせる必要はありません。

よっつ。出力の形を、先に見本で示す。「次の形式で出して」と、空欄の雛形を渡すと、その通りに埋めてくれます。表や定型の文書を作るときに効きます。

いつつ。だめな例も伝える。「ただし、（こういう言い回し)は避けて」と、避けたいものを名指しすると、外れが減ります。欲しいものだけでなく、欲しくないものを伝えるのも効きます。

## トーンの指定、よく使う言葉

トーンは、ふわっとしがちなので、よく使う言い方を並べておきます。そのまま頼みに混ぜて使えます。

- ていねいだが、固すぎない
- 親しみやすく、くだけすぎない
- 簡潔に、結論から
- やわらかく、角が立たないように
- きっぱり、でも失礼にならないように
- 落ち着いた、大人っぽい
- 明るく、前向きに

「ちゃんとした感じで」より、こういう具体的な言葉のほうが、狙ったトーンに寄ります。

## 長い文章を渡すときの工夫

長い資料や、複数の文章をまとめて渡すときは、貼り方を少し工夫すると、扱いが安定します。

ひとつ。どこからどこまでが資料かを、はっきりさせる。「ここから資料」「ここまで資料」と区切りを書くと、指示と資料が混ざりません。

ふたつ。先に頼みごと、後ろに資料。長い資料を上に貼ると、肝心の頼みごとが埋もれます。指示を先に書いて、資料を最後に貼るほうが、伝わりやすいです。

みっつ。一度に欲張らない。長い資料に対して、要約も翻訳も分類も、と一度に頼むと、どれかが雑になります。まず要約、次にその要約を翻訳、と分けるほうが、結局きれいに進みます。

## 自分のプロンプトを直してみる

実際に、弱いプロンプトを、型で直す練習をしてみます。

最初のプロンプトが、これだったとします。

```
このデータをいい感じにまとめて
```

これだと、AIは「いい感じ」が何か分かりません。誰向けで、どんな形で、何を強調したいのか、手がかりがない。型で補うと、こうなります。

```
次のアンケート結果を、まとめてください。
- 相手：数字が苦手な上司
- 形式：要点を箇条書きで3点、そのあとに短い表
- 制約：専門用語は使わず、増えた・減ったで表現する

データ：
（ここに貼る)
```

直したのは、難しいことではありません。「いい感じ」を、相手・形式・制約の言葉に置き換えただけです。プロンプトの上達は、この置き換えに慣れていくこと、とも言えます。

もう一つ、別の例も見てみます。弱い版が、これ。

```
プログラミングについて教えて
```

範囲が広すぎて、AIも何から話せばいいか分かりません。絞ると、こうなります。

```
プログラミングを独学で始めたい人に、最初の一歩を説明してください。
- 相手：何の言語も触ったことがない社会人
- 形式：何を、どの順番でやるか、ステップで
- 制約：おすすめの言語は1つに絞って、理由を一言
```

「教えて」を、誰に・何を・どの形で、に置き換える。やっていることは、さっきと同じです。

## やりがちな失敗

初心者がつまずきやすい点を、先回りで挙げます。

- 指示がざっくりすぎる。「いい感じにして」では、AIは何を「いい」とするか分かりません
- 一度の指示に、要望を詰め込みすぎる。多すぎると、どれかが無視されます。大きい用件は分けて頼む
- 返ってきた答えを、そのまま信じる。とくに事実や数字は、必ず自分で確かめてください
- 個人情報や社外秘を、そのまま貼る。無料サービスでは、入力内容の扱いに注意が必要です
- 一発で諦める。一度の答えがイマイチでも、追加で頼めば寄っていきます。会話の途中で投げ出すのは、もったいないです
- 専門用語で固めようとする。難しい言葉を使うより、ふつうの日本語で具体的に書くほうが、ずっと通じます

## よくある質問

初心者からよく出る疑問を、短く答えておきます。

長いプロンプトほど良いのですか。長さは関係ありません。必要な情報が入っているかどうかです。短くても、相手と形式がはっきりしていれば、十分良い答えが返ります。

毎回こんなに書くのは大変では。よく使う頼み方は、メモに保存しておいて、使い回せばいいです。一度作れば、カッコの中を変えるだけで済みます。この記事の例文を、自分用にコピーしておくのもおすすめです。

英語で書いたほうが良いですか。日本語で問題ありません。日本語で具体的に書くほうが、無理な英語で曖昧に書くより、ずっと良い答えになります。

AIによって書き方を変えるべきですか。最初は同じで大丈夫です。この型は、ChatGPT でも Claude でも Gemini でも効きます。慣れてきたら、各AIの個性に合わせて調整すれば十分です。

思った答えと違うときは。AIを責める前に、頼み方を見直してください。たいていは、相手か形式か制約のどれかが抜けています。足りない指定を足して、もう一度頼めば、たいてい寄ります。

無料版でもこの書き方は効きますか。効きます。プロンプトの書き方は、有料・無料に関係なく効きます。むしろ、無料版で性能に限りがあるときほど、頼み方の差が出ます。

## 次の自分に渡すメモ

うまくいかないときは、AIを変える前に、頼み方を見直す。目的・相手・形式・制約のどれかが抜けていないか確かめる。例を見せると速い。一度で完璧を狙わず、会話で寄せる。トーンは具体的な言葉で指定する。長い資料は、区切りをつけて、指示を先に。そして、返ってきたものは鵜呑みにせず、自分で確かめる。プロンプトは呪文ではなく、具体的な頼みごとです。それだけ覚えておけば、最初は十分です。
