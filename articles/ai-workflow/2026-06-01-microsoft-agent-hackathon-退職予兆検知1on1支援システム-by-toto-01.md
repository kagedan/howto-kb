# Azure AI Foundry × Azure Functionsで構築する、退職予兆検知＆1on1支援システムの裏側

> ハッカソン: Microsoft Agent Hackathon 2026  
> チーム: totonaude  
> メンバー: 久野 宏輔、岡 真史、桑原 亮、尾﨑 洋仁

<https://youtu.be/Y0VWErSsupI>

## 1. プロジェクト概要

### 対象ユーザー

* **HR（人事担当者・管理者）**：全社の離職リスクの傾向を俯瞰し、高リスクな社員へのフォロー状況やマネージャーの介入状態を定量的に管理したいユーザー。
* **現場のマネージャー（管理職）**：自身のチームメンバーのコンディション変化に早期に気づき、効果的な1on1を実践したいユーザー。

### 解決する課題

優秀な人材の離職を防ぐにあたり、マネジメント層が抱える課題をAIの力で解決します。

**Before**

* 退職の予兆となるデータ（勤怠、サーベイ、カレンダー、会議発言など）が各システムに散在しており、人間では総合的なリスクに気づけない。
* マネージャーの経験や勘に依存するため、1on1の実施頻度や対話の質にばらつきがある。
* 退職リスクという機微な情報を、現場とHRでどう安全に共有すべきかの基準がない。
* 1on1を実施しても議事録やネクストアクションが蓄積されず、やりっぱなしになってしまう。

**After**

* 複数の専門AIエージェントがデータを統合分析し、早期に退職予兆スコアを算出・検知する。
* AIが対象者にパーソナライズされた1on1アジェンダを提案し、属人化を防ぎ対話の質を底上げする。
* 権限分離の徹底により、HRと現場マネージャーそれぞれに適切な粒度の情報のみを提供する。
* 1on1の議事録からAIがネクストアクションと振り返りを抽出し、次の検知サイクルへと繋げる。

### ソリューションの特徴

**1. マルチエージェントによる退職予兆分析**  
勤怠・カレンダー・サーベイなどの専門AIエージェントがそれぞれの視点でシグナルを抽出し、スコア統合AIが最終的な退職予兆を統合算出します。

**2. 社内データの自動コンテキスト統合**  
分散している勤怠データ、非公開予定の件数、ESサーベイのスコアなどを自動で把握・連結。HRやマネージャーはダッシュボードを開くだけです。

**3. AIによる1on1振り返り＆アクション抽出**  
1on1の対話メモや文字起こしを入力するだけで、AIが内容を分析し、ネクストアクションとマネージャー自身の振り返りを構造化して抽出します。

**4. 権限分離によるセキュアな情報提供**  
個人の退職スコアなど機微な情報はHRのみに限定表示。マネージャーには「気づき（事実）」だけを提示し、先入観によるマネジメント悪化を防ぎます。

**5. パーソナライズされたアジェンダ自動生成**  
AIが検知したシグナルや過去の未完了タスクに基づき、該当社員のコンテキストに合わせた最適な1on1アジェンダをAIが動的に生成します。

## 2. システムアーキテクチャ

![](https://static.zenn.studio/user-upload/58ae58a163b4-20260601.png)

システム全体はフルマネージドなクラウドサービスと生成AIを組み合わせたサーバーレスアーキテクチャを採用しています。

* **フロントエンド**: Azure Static Web Apps (Standard)
* **バックエンド**: Azure Functions (Python 3.11)
* **データベース**: Azure SQL Database (`taishoku-db`: 6テーブル)
* **ファイルストレージ**: Azure Blob Storage (月次CSVやテキストの格納)
* **AIエージェント基盤**: Azure AI Foundry

バックエンドのAzure Functionsは、フロントエンドとDB・AI基盤を繋ぐハブとして機能します。AIのオーケストレーションだけでなく、ユーザー権限に応じたSQLクエリの動的生成や、監査ログの自動記録など、セキュアなデータアクセス層として構築されています。

---

## 3. AIエージェントが実現する「退職予兆の検知と提案」

システム側から能動的に推論を行い、アクションを提案するフェーズです。  
日次バッチ処理、またはHRからの手動実行を契機に、バックエンドは「1従業員＝1起動」のワークフローを開始し、Azure AI Foundry上のエージェント群を直列（Chain of Agents）で呼び出します。

1. `sensor-agent`: 会議の発言頻度などの生データからコミュニケーション減退シグナルを抽出。
2. `attendance-agent`: 勤怠データから休暇や遅刻、残業の増減シグナルを抽出。
3. `calendar-agent`: Blob上のCSV（非公開予定の件数等）からシグナルを抽出。
4. `survey-agent`: ESサーベイ結果をRAGで読み込み、上司との関係性などを分析。
5. `attrition-score-agent`: 上流の全シグナルを統合し、予兆スコア、リスクレベル、根拠（XAI）を算出。
6. `action-proposal-agent`: リスク情報に基づき、推奨アクションと介入タイミングを提案。

最終的に算出されたデータは、バックエンドのトランザクション処理により、Azure SQL Databaseのテーブルへ一括保存されます。

---

## 4. AIエージェントが実現する「1on1の振り返りと採点」

マネージャーが実施した1on1の内容に対し、AIが事後フィードバック・評価を行うフェーズです。

* **1on1実施前のアジェンダ生成**  
  マネージャーが1on1の実施を決めた際、バックエンドはDBから過去の未完了タスクやAIが検知したシグナルを取得し、`agenda-agent` に渡します。エージェントはこれらを元に、該当社員専用の「1on1アジェンダ」をJSONで動的生成し、有意義な対話の土台を作ります。
* **1on1実施後の議事録採点**  
  1on1終了後、マネージャーが入力した対話メモやデータを `minutes-agent` が解析します。対話から「具体的なアクションアイテム」や「マネージャーの振り返り」を構造化して抽出します。  
  抽出後、バックエンドは `1on1Interactions` テーブルを更新し、ステータスを `COMPLETED` に書き換えることで、やりっぱなしを防ぎます。

## 5. その他の機能

* **AIのブラックボックス化を防ぐ説明可能性（XAI）**  
  「なぜそのスコアになったのか」が分からないと現場はアクションを起こせません。本システムでは、スコア算出時に必ずその根拠となるサマリ（XaiSummary）を文章としてAIに生成・保存させています。
* **包摂性除外（ExclusionType）の組み込み**  
  社員マスタの `ExclusionType` を判定し、「育休復帰直後」や「時短勤務」などの社員に対しては、スコアリング処理をスキップ、あるいは閾値を調整することで、機械的な分析による冷酷な誤検知を防ぎます。

---

## 6. 開発で工夫した点

システム全体を貫く最大の工夫は、**「生成AIの柔軟性」と「基幹システム並みの堅牢性」をバックエンド（Azure Functions）で統合設計した点**です。

退職予兆スコアという極めて機微な情報を扱うため、フロントエンドからAIエージェントを直接呼び出すような簡易的な構成は避けました。すべてのリクエストはFunctionsを経由させ、以下のアーキテクチャをシステム全体に強制しています。

* **API層での徹底したデータブロック**：マネージャーからのAPIリクエスト時には、該当メンバーの退職スコア（RiskAnalyses）をSQLクエリの段階で結合させず、ロジック層でも強制的に除外しています。これにより、意図せぬ情報漏洩を構造的に防いでいます。
* **監査ログの完全義務化**：データの更新（POST）だけでなく、単なる画面閲覧（GET）であっても、APIが叩かれるたびに「誰が・いつ・どの情報を見たか」を `AuditLogs` テーブルへ1回1レコードとして必ず記録（INSERT）するインターセプターを実装しました。

単にAIでデータを分析するだけでなく、「現場とHRが安心して利用できるセキュリティと監査可能性」を土台から作り込んだことが、本システムの実用性を支えています。

---

## 7. 今後の展望

現在はモックデータおよび手動連携のCSVを利用していますが、今後はセキュアな閉域網（Azure VNet / ExpressRoute等）を経由して、社内システムからBlobストレージへ各データを自動同期するパイプラインを構築する予定です。  
また、`minutes-agent` が抽出した「マネージャーの振り返り」をエージェントの長期記憶としてベクトル化し、次回の検知やアジェンダ生成のコンテキストに組み込むことで、AIの推論精度を継続的に高めていきます。

---

## おわりに

本プロジェクトでは、散在するデータを「退職予兆の検知（提案）」として昇華させるだけでなく、現場の「1on1による介入（採点・蓄積）」までを一つのサイクルとして完結させるアーキテクチャを構築しました。

機微なデータを扱うからこそ、実カレンダー連携のようなリスクを伴う機能は開発初期にスコープから除外し、その分「権限分離」や「全操作の監査ログ記録」といった実運用に不可欠な堅牢性の確保に開発リソースを集中させました。この「引き算の設計」と「クローズドループの構築」が、一人でも多くの優秀な社員が定着する組織づくりに貢献できると確信しています。
