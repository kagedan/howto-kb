Python + Claude API + Google Forms APIで英単語学習を自動化した話をする。

英語学習でいちばん効率が悪いのは、全部を同時にやろうとすることかもしれない。  
文法、リスニング、スピーキング、読解——どれも大事だとわかってはいる。でも結局、広く浅くなって何も身につかないまま終わる、というパターンを何度も繰り返してきた。

そこで一度、思い切って範囲を絞ることにした。対象は「単語」だけ。他は全部いったん捨てる。

単語学習にもやり方は無数にある。教材の選び方、記憶の定着方法、復習のタイミング、文脈との紐づけ方……。その一つひとつを検証しながら、自分なりに最適解を探していく。これはそのプロセスの記録です。

---

## なぜこの方法に辿り着いたか

最初はスマホアプリで完結させようとしていた。手軽だし、隙間時間に使えると思っていたからだ。  
ただ実際にやってみると、アプリを開いても別の通知が来る、そのままSNSに流れる、気づけば学習が止まっている——そのパターンが繰り返された。スマホは便利すぎるがゆえに、集中の敵でもある。

そこで「紙」に切り替えた。プリントアウトした単語リストが机の上にあれば、それ自体が学習のトリガーになる。スマホを開く必要がないから、余計なものに引っ張られない。アナログをあえて挟むことで、集中できる環境を物理的に作る、という発想だ。

## システム全体の流れ

```
英語記事 (Markdown)
    ↓
[Step 1] LLMで約50語をピックアップ → YAML生成
    ↓
[Step 2] Google Formで「わからない単語」を自分で選択
    ↓
[Step 3] 選んだ単語でPDF単語リストを生成
    ↓
[Step 4] TTS音声ファイル(MP3)を自動生成
    ↓
[Step 5] Google Formクイズを自動作成 + QRコード付きPDFを印刷
    ↓
紙で学習 → クイズ → [Step 6] 間違えた単語だけ復習PDF
```

## Step 1: LLMで単語をピックアップする

`scripts/generate_vocabulary_yaml.py` が担当する。英語記事のMarkdownを受け取り、Claude APIを使って単語リストとクイズをYAML形式で出力する。

```
def generate_vocabulary_yaml(en_path: str, yaml_path: str, words: list[str] | None = None) -> None:
    with open(en_path, encoding="utf-8") as f:
        en_content = f.read()

    if words:
        # 単語リストが指定されている場合はそれを使う
        prompt = USER_PROMPT_WITH_WORDS.format(
            word_list=word_list,
            en_content=en_content,
            output_path=pdf_output_path,
        )
    else:
        # 指定がなければClaudeが記事から抽出する
        prompt = USER_PROMPT_AUTO.format(
            en_content=en_content,
            output_path=pdf_output_path,
        )

    client = anthropic.Anthropic()
    message = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=8192,
        system=SYSTEM_PROMPT,
        messages=[{"role": "user", "content": prompt}],
    )
```

プロンプトには明確な制約を設けている。クイズ問題は「記事を読まなくても解ける」ことが必須条件だ。記事の文脈に依存した問題では、紙で学習するときに意味をなさないからだ。

```
CRITICAL: Questions must be fully standalone — never add "as used in the article",
"in this context", "in the passage", or any phrase that requires reading the article to answer.
Use only general dictionary-style definitions.
```

出力されるYAMLの構造はこうなる。

```
output: "vocabulary/2026-06/article_name.pdf"
title: "Article Title"
date: "2026-06-01"

vocabulary:
  - word: "nudging"
    definition: "the practice of gently steering someone toward a desired behavior"
    pos: "noun"
    ja: "行動を促すこと"

quiz:
  - question: "What does 'nudging' mean?"
    choices:
      - "A. gently steering someone toward a desired behavior"
      - "B. forcefully directing someone"
      - "C. ignoring someone's actions"
      - "D. criticizing someone's choices"
    answer: "A"
    explanation: "Correct: A. nudging (gently steering someone). B. forcefully directing, C. ignoring, D. criticizing."

quiz_ja:
  - question: "「nudging」の日本語の意味として最も適切なものを選んでください。"
    choices:
      - "A. 行動を促すこと（そっと誘導する）"
      - "B. 強制すること"
      - "C. 無視すること"
      - "D. 批判すること"
    answer: "A"
    explanation: "「nudging」は行動経済学の文脈でよく使われ、強制せずに行動を促すことを意味します。"
```

英語クイズと英和クイズの2種類を同時に生成している点がポイントだ。単語の意味を問う問題と、日本語訳を選ぶ問題——それぞれ異なる方向から定着を確認できる。

## Step 2: TTS音声ファイルを自動生成する

`scripts/generate_vocabulary_audio.py` が担当する。YAMLを読み込み、Google TTS (gTTS) で単語と定義を読み上げたMP3を生成する。

```
def _tts_segment(text: str) -> AudioSegment:
    tts = gTTS(text=text, lang="en", slow=False)
    buf = io.BytesIO()
    tts.write_to_fp(buf)
    buf.seek(0)
    return AudioSegment.from_mp3(buf)

def generate_audio(yaml_path: str, output_path: str) -> None:
    vocabulary = _load_vocabulary(yaml_path)

    gap_word_def = AudioSegment.silent(duration=200)   # 単語と定義の間: 0.2秒
    gap_next_word = AudioSegment.silent(duration=300)  # 単語間: 0.3秒

    combined = AudioSegment.empty()
    for i, item in enumerate(vocabulary):
        word = item.get("word", "").strip()
        definition = item.get("definition", "").strip()

        combined += _tts_segment(word) + gap_word_def + _tts_segment(definition)

        if i < len(vocabulary) - 1:
            combined += gap_next_word

    combined.export(output_path, format="mp3")
```

音声の構造は `[単語] → 0.2秒 → [定義読み上げ] → 0.3秒 → [次の単語]` という繰り返しだ。この音声ファイルはGoogle Driveに保存し、QRコード経由でPDFからアクセスできるようにする。

## Step 3: Google Formクイズを自動作成する

`scripts/create_quiz_form.py` が担当する。YAMLのクイズ問題をGoogle Forms APIでフォームに変換し、自動採点まで設定する。

```
def create_quiz_form(yaml_path: str, meta_out: str | None = None, quiz_type: str = "en") -> str:
    with open(yaml_path, encoding="utf-8") as f:
        data = yaml.safe_load(f)

    section_key = "quiz_ja" if quiz_type == "ja" else "quiz"
    quiz = data.get(section_key, [])

    forms_service, drive_service = _build_services()

    form = forms_service.forms().create(body={
        "info": {"title": form_title}
    }).execute()
    form_id = form["formId"]

    requests = [
        {
            "updateSettings": {
                "settings": {"quizSettings": {"isQuiz": True}},
                "updateMask": "quizSettings",
            }
        }
    ]
    for i, q in enumerate(quiz):
        correct_value = next(
            (c for c in q["choices"] if c.startswith(q["answer"] + ".")), q["answer"]
        )
        requests.append({
            "createItem": {
                "item": {
                    "title": f"Q{i+1}. {q['question']}",
                    "questionItem": {
                        "question": {
                            "required": True,
                            "grading": {
                                "pointValue": 1,
                                "correctAnswers": {"answers": [{"value": correct_value}]},
                            },
                            "choiceQuestion": {
                                "type": "RADIO",
                                "options": [{"value": c} for c in q["choices"]],
                                "shuffle": True,
                            },
                        }
                    }
                },
                "location": {"index": i}
            }
        })

    forms_service.forms().batchUpdate(formId=form_id, body={"requests": requests}).execute()

    drive_service.permissions().create(
        fileId=form_id, body={"type": "anyone", "role": "reader"}
    ).execute()
```

英語クイズ (`quiz_type="en"`) と英和クイズ (`quiz_type="ja"`) を別々のフォームとして作成する。選択肢はシャッフル設定済み (`"shuffle": True`) なので、答えの位置を覚えることができない。

フォームのURLとメタデータはJSONに保存し、後のPDF生成とレビュー処理で参照される。

```
{
  "form_id": "1BxiMV...",
  "form_url": "https://docs.google.com/forms/d/1BxiMV.../viewform",
  "responses_url": "https://docs.google.com/forms/d/1BxiMV.../responses",
  "quiz_count": 10
}
```

## Step 4: QRコード付きPDFを生成する

`pdf_generator/services/pdf_service.py` が担当する。ReportLabでA4 PDFを構築し、QRコードを埋め込む。

```
def build_vocabulary(self, doc: VocabularyDocument) -> str:
    pdf_doc = SimpleDocTemplate(
        output, pagesize=A4,
        leftMargin=15 * mm, rightMargin=15 * mm,
        topMargin=15 * mm, bottomMargin=15 * mm,
    )

    story = []
    # 1ページ目: 英語定義テーブル + 音声QRコード
    story.extend(_vocab_en_page(doc.vocabulary, audio_url=doc.audio_url or ""))
    story.append(PageBreak())
    # 2ページ目: 英和対応テーブル
    story.extend(_vocab_ja_page(doc.vocabulary))

    if doc.form_url and doc.form_url_ja:
        story.append(PageBreak())
        # 3ページ目: 英語クイズQR + 英和クイズQRを横並びで表示
        story.extend(_dual_qr_page(doc.form_url, doc.form_url_ja, pdf_url=doc.pdf_url or ""))
```

QRコードの生成は `_make_qr_image()` が担当する。

```
def _make_qr_image(url: str, size_mm: float) -> Image:
    import qrcode

    qr = qrcode.QRCode(error_correction=qrcode.constants.ERROR_CORRECT_M, box_size=8, border=2)
    qr.add_data(url)
    qr.make(fit=True)
    img = qr.make_image(fill_color="black", back_color="white")
    buf = io.BytesIO()
    img.save(buf, format="PNG")
    buf.seek(0)
    s = size_mm * mm
    return Image(buf, width=s, height=s)
```

生成されるPDFは3ページ構成だ。

* **1ページ目**: 単語・定義・品詞のテーブル（英語）＋音声QRコード
* **2ページ目**: 単語・日本語訳のテーブル
* **3ページ目**: 英語クイズQR ／ 英和クイズQR（横並び）

コンビニのプリンターはフォント埋め込みで文字化けが発生することがある。そのため `rasterize()` でPDFを画像化した印刷用PDFも生成する。

```
def rasterize(self, src_path: str, dpi: int = 200) -> str:
    import fitz  # PyMuPDF

    A4_W, A4_H = 595.28, 841.89
    dst_path = src_path.replace(".pdf", "_print.pdf")
    mat = fitz.Matrix(dpi / 72, dpi / 72)

    src = fitz.open(src_path)
    dst = fitz.open()
    for page in src:
        pix = page.get_pixmap(matrix=mat, alpha=False)
        pix.set_dpi(dpi, dpi)
        img_page = dst.new_page(width=A4_W, height=A4_H)
        img_page.insert_image(img_page.rect, pixmap=pix)

    dst.save(dst_path, deflate=True)
```

## Step 5: 間違えた単語の復習PDFを生成する

`scripts/generate_review_pdf.py` が担当する。Google Forms APIでクイズの回答を取得し、誰がどの問題を間違えたかを集計して復習用PDFを生成する。

まず回答を取得し、問題ごとの誤答数を集計する。

```
def _analyze_responses(form_schema: dict, responses: list[dict], quiz: list[dict]) -> dict:
    items = form_schema.get("items", [])
    item_id_to_idx = {}
    for item in items:
        item_id = item.get("itemId")
        title = item.get("title", "")
        m = re.match(r"Q(\d+)\.", title)
        if m:
            item_id_to_idx[item_id] = int(m.group(1)) - 1

    wrong_counts: dict[int, int] = {}
    for response in responses:
        answers = response.get("answers", {})
        for item_id, answer_obj in answers.items():
            idx = item_id_to_idx.get(item_id)
            if idx is None or idx >= len(quiz):
                continue
            q = quiz[idx]
            given = answer_obj.get("textAnswers", {}).get("answers", [{}])[0].get("value", "")
            if not given.startswith(q["answer"] + "."):
                wrong_counts[idx] = wrong_counts.get(idx, 0) + 1

    wrong_questions = [
        {**quiz[idx], "wrong_count": cnt, "question_num": idx + 1}
        for idx, cnt in sorted(wrong_counts.items(), key=lambda x: -x[1])
    ]
    return {"wrong_questions": wrong_questions, "total_responses": len(responses)}
```

間違えた問題に登場する単語をYAMLから抽出し、練習リストを作る。

```
def _find_practice_words(wrong_questions: list[dict], vocabulary: list[dict]) -> list[dict]:
    practice = []
    seen = set()
    for q in wrong_questions:
        text = q.get("question", "") + " ".join(q.get("choices", []))
        for vocab in vocabulary:
            word = vocab.get("word", "").lower()
            if word and word not in seen and re.search(r'\b' + re.escape(word) + r'\b', text, re.IGNORECASE):
                practice.append(vocab)
                seen.add(word)
    return practice
```

出力されるPDFは2部構成だ。

* **1ページ目**: 誤答した問題ごとに正解・解説を表示。`[N/M人が誤答]` の表示で自分の理解度がわかる
* **2ページ目**: 誤答に関連する単語だけを抜き出した練習リスト（Word / Definition / POS / 日本語）

```
story.append(Paragraph(
    f"Q{wq['question_num']}. {wq['question']} "
    f"<font color='#c0392b'>[{wq['wrong_count']}/{total_responses}人が誤答]</font>",
    _WRONG,
))
correct_text = next(
    (c for c in wq.get("choices", []) if c.startswith(wq["answer"] + ".")), wq["answer"]
)
story.append(Paragraph(f"正解: {correct_text}", _CORRECT))
story.append(Paragraph(wq.get("explanation", ""), _EXPLAIN))
```

翌日に再クイズを受け、満点が取れたら次のリストへ進む。この「間違えた単語だけに絞る」サイクルが、このシステムの復習設計の核心だ。

## 学習フローのまとめ

印刷したPDFは3役を担う。

1. **単語リスト**: 机の上に置くだけで学習のトリガーになる
2. **QRコード (音声)**: 発音が気になったときだけスキャンしてMP3を聴く
3. **QRコード (クイズ)**: 学習後にスマホでスキャンしてGoogle Formクイズを受ける

スマホをメインにしないことで集中を守りつつ、必要なときだけ音声・クイズにアクセスできる設計だ。
