## はじめに

2026年4月30日、**Claude Sonnet 4.5** および **Claude Sonnet 4** 向けの1Mトークンコンテキストベータが廃止されます。

対象の [ベータヘッダー `context-1m-2025-08-07`](https://platform.claude.com/docs/en/release-notes/overview) を使用しているアプリケーションは、**4月30日以降に200kトークンを超えるリクエストがエラーを返す**ようになります（サイレント切り捨てではなくエラー返却）。

この記事では、移行が必要かどうかの確認方法から、移行手順・料金変化・新機能まで整理します。

### この記事で解決できること

* 自分のコードが廃止の影響を受けるかどうかを確認できる
* Claude Sonnet 4.6 / Opus 4.6 への移行手順を把握できる
* 移行に伴う料金変化を理解できる
* 移行後に使える新機能（output-300k）を把握できる

### 対象読者

* Claude API を使って1Mトークンコンテキストを利用しているエンジニア
* `context-1m-2025-08-07` ベータヘッダーを使用しているアプリケーションの開発・運用担当者

## TL;DR

* **廃止日**: 2026年4月30日
* **影響**: `context-1m-2025-08-07` ヘッダー + Sonnet 4.5 / Sonnet 4 の組み合わせ
* **移行先**: `claude-sonnet-4-6` または `claude-opus-4-6`（1M標準搭載・追加料金なし）
* **手順**: モデル名変更 + ベータヘッダー削除の2ステップのみ
* **ボーナス**: 移行後は Message Batches API で max\_tokens 300k が利用可能

---

## 背景：1Mコンテキストベータとは

Claude Sonnet 4 / 4.5 は標準で200kトークンのコンテキストウィンドウを持ちます。2025年8月に導入された `context-1m-2025-08-07` ベータヘッダーを付与することで、200kを超える最大1Mトークンのコンテキストを利用できるようになりました。

ただしこのベータ期間中は、**200kトークンを超えるリクエストに long context pricing（追加料金）が適用**されていました。

2026年3月13日、Claude Sonnet 4.6 および Opus 4.6 が GA（一般公開）され、これらのモデルは**ベータヘッダーなしで1Mトークンコンテキストを標準サポート**し、さらに長文コンテキスト追加料金も廃止されました。これにより旧ベータの役割は終わり、2026年4月30日をもって廃止されます。

## 影響確認：自分のコードは対象か？

以下の**両方の条件を満たす**コードが影響を受けます。

**条件1: 使用モデルが以下のいずれか**

* `claude-sonnet-4-5`
* `claude-sonnet-4-20250514`（Sonnet 4.5 の日付指定エイリアス）
* `claude-sonnet-4`
* `claude-sonnet-4-0`（または `claude-sonnet-4-20250514`：Sonnet 4 の日付指定エイリアス）

**条件2: ベータヘッダーを付与している**

```
anthropic-beta: context-1m-2025-08-07
```

### コード内の確認方法

**Python（anthropic SDK）の場合:**

```
import anthropic

client = anthropic.Anthropic()

# 旧コード（対象）
response = client.messages.create(
    model="claude-sonnet-4-5",  # ← 対象モデル
    max_tokens=4096,
    extra_headers={
        "anthropic-beta": "context-1m-2025-08-07"  # ← 廃止されるヘッダー
    },
    messages=[{"role": "user", "content": "..."}]
)
```

**grep でコードベース全体を確認:**

```
# ベータヘッダーの使用箇所を検索
grep -r "context-1m-2025-08-07" ./

# 対象モデルの使用箇所を検索
grep -rE "claude-sonnet-4-5|claude-sonnet-4[^.]" ./
```

## 移行手順

移行は**2ステップ**で完了します。

### ステップ1: モデル名の変更

```
# Before（廃止対象）
model = "claude-sonnet-4-5"

# After（移行先：Sonnet 4.6 を推奨）
model = "claude-sonnet-4-6"

# または Opus 4.6（より高性能・高コスト）
model = "claude-opus-4-6"
```

### ステップ2: ベータヘッダーの削除

`context-1m-2025-08-07` ヘッダーは**不要になるため削除**します。

```
# Before（廃止対象）
import anthropic

client = anthropic.Anthropic()
response = client.messages.create(
    model="claude-sonnet-4-5",
    max_tokens=4096,
    extra_headers={
        "anthropic-beta": "context-1m-2025-08-07"  # 削除する
    },
    messages=[{"role": "user", "content": long_document}]
)

# After（移行後）
import anthropic

client = anthropic.Anthropic()
response = client.messages.create(
    model="claude-sonnet-4-6",  # モデル名を変更
    max_tokens=4096,
    # extra_headers 不要（ヘッダーごと削除）
    messages=[{"role": "user", "content": long_document}]
)
```

**TypeScript（@anthropic-ai/sdk）の場合:**

```
// Before
const response = await anthropic.messages.create({
  model: "claude-sonnet-4-5",
  max_tokens: 4096,
  extra_headers: {
    "anthropic-beta": "context-1m-2025-08-07",
  },
  messages: [{ role: "user", content: longDocument }],
});

// After
const response = await anthropic.messages.create({
  model: "claude-sonnet-4-6",
  max_tokens: 4096,
  messages: [{ role: "user", content: longDocument }],
});
```

**curl / 直接 API 呼び出しの場合:**

```
# Before
curl https://api.anthropic.com/v1/messages \
  -H "anthropic-version: 2023-06-01" \
  -H "anthropic-beta: context-1m-2025-08-07" \
  -H "x-api-key: $ANTHROPIC_API_KEY" \
  -H "content-type: application/json" \
  -d '{
    "model": "claude-sonnet-4-5",
    "max_tokens": 4096,
    "messages": [{"role": "user", "content": "..."}]
  }'

# After（anthropic-beta ヘッダーを削除、モデル名変更）
curl https://api.anthropic.com/v1/messages \
  -H "anthropic-version: 2023-06-01" \
  -H "x-api-key: $ANTHROPIC_API_KEY" \
  -H "content-type: application/json" \
  -d '{
    "model": "claude-sonnet-4-6",
    "max_tokens": 4096,
    "messages": [{"role": "user", "content": "..."}]
  }'
```

## 料金への影響

<!-- IMAGE\_SLOT: pricing-comparison

-->

移行によって料金体系が変わります。[公式料金ページ](https://platform.claude.com/docs/en/about-claude/pricing) によると：

|  | Claude Sonnet 4.5（旧） | Claude Sonnet 4.6（新） |
| --- | --- | --- |
| 入力トークン | $3.00 / MTok | $3.00 / MTok |
| 出力トークン | $15.00 / MTok | $15.00 / MTok |
| 長文コンテキスト追加料金 | あり（200k超） | **なし** |
| ベータヘッダー | 必要 | **不要** |

|  | Claude Opus 4.6 |
| --- | --- |
| 入力トークン | $5.00 / MTok |
| 出力トークン | $25.00 / MTok |
| 長文コンテキスト追加料金 | なし |

**ポイント:** Sonnet 4.6 への移行では、トークン単価は変わりませんが、200kトークン超の長文コンテキスト使用時の追加料金がなくなります。長文処理を多用しているアプリケーションでは**コスト削減**になる可能性があります。

## 移行後に使える新機能：output-300k

移行先の Sonnet 4.6 / Opus 4.6 では、**Message Batches API** で出力トークン数の上限を 300,000 まで引き上げる新しいベータ機能が利用できます。[公式リリースノート](https://platform.claude.com/docs/en/release-notes/overview) (2026-03-30付) で発表されました。

```
import anthropic

client = anthropic.Anthropic()

# Message Batches API で 300k 出力を有効化
from anthropic.types.beta.message_create_params import MessageCreateParamsNonStreaming
from anthropic.types.beta.messages.batch_create_params import Request

message_batch = client.beta.messages.batches.create(
    betas=["output-300k-2026-03-24"],  # betas パラメータで指定
    requests=[
        Request(
            custom_id="long-form-request",
            params=MessageCreateParamsNonStreaming(
                model="claude-sonnet-4-6",
                max_tokens=300000,  # 最大 300k トークンまで指定可能
                messages=[
                    {
                        "role": "user",
                        "content": "以下の大規模コードベースを分析してください..."
                    }
                ]
            )
        )
    ]
)
```

**主な用途：**

* **ロングフォームコンテンツ生成**: 書籍の章、詳細な技術仕様書
* **大規模コード生成**: プロジェクト全体のスキャフォールディング
* **構造化データ抽出**: 大量ドキュメントからの一括データ抽出

## 注意点

### 廃止後のエラー動作

廃止後に旧モデル＋旧ベータヘッダーで200kトークン超のリクエストを送ると、**サイレントな切り捨てではなくエラーが返却**されます。事前に十分な余裕を持って移行を完了させることを推奨します。

### AWS Bedrock / Google Vertex AI について

今回の廃止アナウンスは Claude API（Anthropic 直接）向けのものです。AWS Bedrock や Google Vertex AI 経由での利用については、各クラウドプロバイダーの公式ドキュメントを確認してください。

### 長文コンテキストの活用

Sonnet 4.6 / Opus 4.6 では1Mトークンが標準機能として利用できますが、必要以上に大きなコンテキストを送ることはコストとレイテンシに影響します。実際に必要なコンテキスト長を見直すことも合わせて推奨します。

| 移行先 | `claude-sonnet-4-6` または `claude-opus-4-6` |

4月30日の期限が近づいています。早めに移行を完了させることを推奨します。
