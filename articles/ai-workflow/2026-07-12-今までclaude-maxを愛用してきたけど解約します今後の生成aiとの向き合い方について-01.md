# はじめに

私は去年の8月から現在までClaude Maxを契約しており、個人開発ではコーディングからドキュメント、スライド生成まで、ほぼ全てをClaudeにやらせてきました。AIは日に日に性能が上がっており、今ではClaudeなしで実装やシステム設計をするのは考えられないくらい、ほぼ毎日Claude Maxを利用しています。

今までClaude Maxを使ってきて特に不満はなかったのですが、現在私がアサインされているプロジェクトで利用している生成AIは主に**Gemini**と**GitHub Copilot**で、Claudeではありません。プロジェクト内でも本格的に生成AIを活用した業務改善を推進していこうという動きが出ており、プロジェクト内で今以上にバリューを出そうと思ったら、今後はClaude以外の生成AIツールの知識が必須になりそうです。

個人では今までずっとClaudeオンリーで開発してきた私ですが、個人学習で身につけた生成AIスキルを最大限活かすなら、Claudeだけではなく他の生成AIも使っていくべきではないか？と考えるようになりました。

そんな経緯で、今回Claude Maxを解約することにしたので、その理由と今後の生成AIとの向き合い方について書いていきます。

# 結論：Claude Maxを解約した

結論から言うと、Claude Maxを解約し、浮いたお金を別の生成AIサービスの利用に割り当てることにしました。

新しく契約する（している）サービスは以下の通りです。

| サービス | 料金 |
| --- | --- |
| Claude Pro | $220/年（税込・≒ $19/月） |
| GitHub Copilot Pro | $10/月 |
| Google AI Pro | 2,900円/月 |
| Google Skills Pro（生成AIではない） | $29/月 |

Claude Maxが$110/月だったので、これらを合計しても月々の料金で考えると**5,000~6,000円くらい安く**なりました。

# なぜそうしたのか？

理由は大きく分けて以下の3つです。

* いろんな生成AIに広く触れて、Claudeに依存しない生成AIの活用方法を身につけたい
* Claude Maxが高すぎる
* Claude Maxのポテンシャルを扱いきれていない

それぞれ詳しく書いていきます。

## いろんな生成AIに広く触れて、Claudeに依存しない生成AIの活用方法を身につけたい

正直、個人開発をする分には、今まで通りClaude Maxだけを使っていても何も問題ないと思います。それくらい今のClaudeは優秀です。なんとなくフワッとした指示を出しただけでも、よしなにいい感じに意味を捉えてくれて、自分が期待した（時には期待以上の）成果物を作ってくれます。

ただ、優秀すぎるが故に、Claudeの優秀さに甘んじて、自分自身が生成AIをうまく制御できていない気がしています。直近は「ハーネスエンジニアリング」というワードがホットですが、個人開発規模でアプリを作っているだけなら、ハーネスエンジニアリングなんかしなくても、プロンプトをいい感じにするだけで十分なものを作成してくれるじゃん、というのが正直なところです。

しかし個人ではなくプロジェクトレベルになると話が変わってきます。プロジェクトで扱う情報量は個人開発で扱う情報量とは次元が違う量になりますし、情報のフォーマットもExcel、Jira、コンフル、チャット、Gitと多岐に渡ります。  
また、私の身の回りだと、プロジェクトで使用している生成AIツールはClaudeではなく、それ以外の生成AI（GitHub CopilotやGeminiなど）を使っているプロジェクトが多数派です。

そういった業務運用的な制約の中で生成AIを最大限活用するには、さまざまな生成AIに触れて、業務内容に応じて適切なユースケースを導く必要があるなーと最近思っています。

## Claude Maxが高すぎる

ずっと思っていたんですけど、しがないエンジニアの私にとっては$110/月は流石に高いです…。もちろん$110/月の価値はあるんですけどね。

個人だけではなく企業レベルでも、Claudeを導入していない理由を聞くと「値段が高いから」という話をかなり聞きます。

利用する生成AIがClaude Maxだけならまだいいのですが、これからいろんな生成AIに触れていきたい私としては、Claude Maxにプラスで他サービスを契約するのは、お財布的にかなりキツいです。

Xや技術系の記事を見ていると、複数の生成AIサービスを最上位プランで契約していそうな方々をたまに見かけるのですが、やっぱり最前線でバリバリ活躍しているエンジニアの方々は、生成AIの利用料金なんて屁じゃないくらい高給取りなんでしょうか…？

## Claude Maxのポテンシャルを扱いきれていない

そもそも、Claude Maxで利用できるトークンを使いきれていません。

先ほどから個人開発をしているとは言いましたが、所詮は趣味程度でやっています。一日中コーディングしていても苦にならないつよつよエンジニアや、一日中生成AIに何かさせないと気が済まない生産性の鬼みたいなエンジニアではありません。

じゃあ、なぜ今までClaude Maxを契約していたかと言えば、趣味程度と言ってもやっぱり使う時は一日中使うからです。そういう時にClaude Proだとすぐにトークンの利用上限が来るので、そのストレスがないようにMaxを契約していました。

しかし今後、私はいろんな生成AIを使っていく方針に切り替える予定なので、特定の生成AIだけを酷使して頻繁に利用上限に達するということはなくなるはずです。それならMaxじゃなくてもいいかなと思い、Claude Proにグレードダウンしました。

# 「今後どう生成AIを使っていくの？」の具体

ここからは、今後実際にどう生成AIを使い分けていきたいかを書いていきます。

## 実装：GitHub CopilotとClaude Proの並行利用

実装については、現在私のプロジェクトで利用しているGitHub CopilotとClaude Proを並行して使っていきたいなと考えています。

具体的には、2つの生成AIを使って以下のようなことをしてみたいです（どれもよくXのスレッドや記事で見かけるような内容ですが…）。

* 同じ機能をそれぞれのAIに実装させたら、どれくらい成果物の質に差が出るか？
* 同じコンテキストを読み込ませて実装させたら、AIごとで実装の精度は変わるのか？
* 異なるAI同士で実装内容をお互いにレビューさせたら、実装の質は上がるのか？

## 学習：Google Skills Proでインフラをハンズオン学習

あと、生成AIとはあまり関係ないのですが、**Google SkillsのProプラン**を契約してみようかと考えています。

最近、IT技術の学習（主にGoogle製品）にGoogle Skillsがとても有効なことに気が付きました。特にGCPやTerraformなどのインフラ系の学習を、文字としての知識だけではなく、実際に手を動かしながらハンズオン形式で学べる点にすごい魅力を感じています。

インフラ周りは中途半端な知識で実物を使って勉強するのに抵抗感があるというか、何かあった時を考えると怖いので…笑。

# まとめ

今回は、長年愛用してきたClaude Maxを解約して、複数の生成AIを使い分ける方針に切り替えた話を書きました。改めて整理すると、以下のような理由と方針です。

* **理由**：Claudeに依存しない活用方法を身につけたい／Claude Maxが高すぎる／ポテンシャルを扱いきれていない
* **今後の方針**：Claude ProとGitHub Copilotで実装を、NotebookLMでドキュメント生成を、Google Skillsでインフラ学習を進める

念のため補足しておくと、これは「Claudeが良くない」という話では全くありません。むしろClaudeは今でも文句なしに優秀で、個人開発だけを考えるならClaude Maxを使い続けるのが一番ラクだと今でも思っています。

それでも解約に踏み切ったのは、**優秀な1つのツールに依存しきってしまう前に、いろんな生成AIを触って、自分の中の引き出しを増やしておきたい**と考えたからです。生成AIの進化スピードを考えると、「特定のツールを深く使いこなす力」と同じくらい、「ツールに依存せず、目的に応じて適切な生成AIを選び取る力」がこれから重要になっていく気がしています。

しばらくこの体制で運用してみて、実際どうだったかはまた別の記事で振り返れたらと思います。同じように複数の生成AIの使い分けで悩んでいる方の参考になれば嬉しいです。
