## 大学時代の研究である数学問題推薦システム

「二次関数の問題を解きたい」

学習者がそう検索したとき、どんな問題を推薦するのが正解なのだろうか。  
私は大学時代、この問いに向き合う数学問題推薦システムを研究していた。

最初は作問学習で作成された大量の問題を有効活用したいというモチベーションで研究を行った。学習者が自然言語で問題を検索し、その要望にあった問題を作問学習で作られた問題のデータベースから推薦しようというシステムである。

技術的にはBERTなどの自然言語をコンピュータが理解できるようにする埋め込みモデルを利用し、学習者の要望クエリと問題文を比較し、意味が近しいものを推薦するという形だった。

もちろん推薦アルゴリズムを技術的に進歩させるのが最初の興味だった。しかし当初思いも寄らなかったところで研究がつまずいた。

## 「いい推薦」とは何か？

数学問題推薦システムを構築し、自然言語を用いて検索してみた。

```
検索例1:
2次関数の最大最小を求める問題を解きたい
```

```
推薦結果:
A:f(x)=x²-4x+3 の最小値を求めよ
B:放物線 y=x²+2x-1 の頂点を求めよ
C:xの範囲が制限されたときの最大値を求めよ
```

```
推薦結果:
A:サイコロを1回振るとき偶数が出る確率
B:コインを2回投げるとき表が出る回数
C:トランプから1枚引く確率
```

このような結果が出た。

**この推薦は良い推薦なのだろうか？何を持って精度を測定すれば良いのだろうか？**

研究において優位性を示すための精度の測定方法がどうしてもわからなかった。既存研究では学習者からのフィードバックや数学教師がクエリに対して適切な問題の組み合わせを作りそれを元に精度を測定していた。

確かにこの2つの方法はこの研究の精度測定として良い方法である。しかしこんな疑問を私は持った。

* **良い推薦結果は学習者の状況や目標によって違うのでは？**
* **学習者自身が自分の状況や目標を正しく理解し、フィードバックすることは可能なのか？**
* **良い推薦とは学習者が欲しい問題を推薦することなのか、それとも学習者に必要な問題を推薦することなのか？**

である。ここから私は「良い推薦とは何か」を考えるようになった。

## 推薦システムは何かを最適化している

良い推薦のヒントを得るために世の中で成功している既存の推薦システムを考えてみることにした。YoutubeやNetflixなどのおすすめ機能、TikTokなどショート動画の動画推薦機能、ECサイトなどで見られるおすすめ商品などたくさんある。

これらの推薦システムにおいては何が「良い推薦」と定義されているのだろうか。この問いは比較的簡単に想像できる。YoutubeやNetflixなどでは視聴時間、TikTokなどではエンゲージメント、ECサイトでは購入率などが想像される。またこれらの推薦システムにおける精度は加点方式であると感じた。

仮に全ての推薦がヒットしなくても推薦として成り立つ。しかし精度が上がれば上がるほど良い推薦になる。（現在は推薦のハードルがあがりユーザによって良い推薦ができないと離脱する可能性がかなり高いかもしれないが）

## 教育では目的関数が難しい

教育ではどうだろうか？

今回の数学問題推薦システムでの目的関数の候補は以下のようなものが考えられるだろう

* 類似度最大化
* 正答率最大化
* 学習効果最大化
* 知識ギャップ解消
* 適切な難易度への最適化
* 学習時間最小化
* モチベーション維持
* 長期的な数学力最大化

このようにたくさん考えることができる。より良い推薦システムにするためにこれらを全部盛り込んだらよいとも考えられる。しかし、これらの目的関数は時に衝突する。

例えば「類似度最大化」と「学習効果最大化」においてはどうだろうか。

```
類似度最大化を目的関数とした場合

二次関数の最大値・最小値の問題
↓
二次関数の最大値・最小値の問題
↓
二次関数の最大値・最小値の問題
```

```
学習効果最大化を目的函数とした場合
平方完成の問題
↓
頂点形式の問題
↓
最大最小の問題
```

などこのような推薦が考えられる。どちらかの目的関数を優先するともう片方の目的関数を疎かにしてしまう。そうした状況になってしまう。

また、どちらの目的関数を優先するかが良いのかは学習者の状況や目標によって異なる。目的関数の多さやどの状況でどの目的関数を優先するかを加味すると推薦が複雑になる。

以上のことから、教育においては「精度をどう測るか」以前に、「何を良い推薦とするか」を決めること自体が難しいことが分かる。

私は次第に、推薦アルゴリズムそのものよりも、この問いに興味を持つようになった。

## 推薦システムからAIエージェントへ

これまでは数学における問題推薦システムを考えてきた。しかし、LLMの普及が加速しAIエージェント時代に突入している今、教育分野でもAIエージェントの議論が加速している。

問題推薦システムでは考慮する必要がなかった学習計画や進捗管理、他の科目も含めて横断的に学習を最適化する必要がある。これほど複雑なAIエージェントは一体何を最適化する目的関数を持つべきなのだろうか。

このような教育における目的の課題は、AI導入以前から存在していた。

私がアルバイトで行っていた小学生向けのプログラミングスクールを例に出す。

スクールでは１コマ90分という形式で個別のカリキュラムでプログラミングを教えていた。授業終了間際、ある生徒はプログラミングが楽しくてもっとやっていたいと言う。しかし授業時間は決まっており次の生徒の授業が始まってしまうため、その生徒にはプログラミングをやめてもらい授業が終わった。

ここでは目的の衝突が起こっている。その子の好奇心を拡大させるという目的においてはそのままプログラミングを続けてもらう方が良いかもしれない。スクールの運営目線で言うと授業時間に合わせてプログラミングを終えてもらう方が良い。また、単に運営上の都合だけではない。時間になったら区切りをつけることや、他の生徒との約束を守ることも教育の一部だと考えることができる。

このように1場面を切り取っても複雑に目的が衝突している。目的の衝突は教育そのものの課題なのかもしれない。

## 私の問い

教育の推薦やエージェントシステムにおける目的関数設定の難しさや既存教育の課題をこれまで考えてきた。

これまでの教育では先生の感覚に頼ったり、目的を固定化したりしてこの難しさに対応していた。これはこの複雑な目的関数設定に対して人間の知能では限界があったからだと思う。

しかし、現在人間を超えうる知能が生まれている。その知能を活用すれば教育がより最適なものになり人間の能力や豊かさを向上させられると考える。その可能性に気づき私は教育AIエージェントに興味を持った。

では、AIが教育を最適化できるとして、どのような設計や目的関数を持つべきなのだろうか。

そして、その最適化の方向は誰が決めるのだろうか。
