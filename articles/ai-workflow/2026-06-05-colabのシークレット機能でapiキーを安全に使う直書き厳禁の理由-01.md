前回の記事「[Google Colabに3行コピペするだけでClaude（AI）を動かす最短手順](https://zenn.dev/randy_epiccat/articles/455f8a8e733ad3)」で動かしたコードでは、APIキーを `userdata` 経由で読み込みました。今回はその土台になる「Colabシークレット機能」で、APIキーを安全に扱う方法を解説します。

なぜ直書きが危険か

APIキーをコードに直接書くと、ノートを共有・GitHubに上げた瞬間に第三者が高額請求を発生させられるおそれがあります。キーは「クレジットカード番号」と同じ扱いをするのが鉄則です。

安全な保存（Colabシークレット）

1. Colab左の🔑アイコン →「新しいシークレット」
2. 名前 `ANTHROPIC_API_KEY`／値に `sk-ant-...` を貼る／ノートブックアクセスをON
3. コードからは下記で読む（直書きしない）

```
from google.colab import userdata
import os
os.environ["ANTHROPIC_API_KEY"] = userdata.get("ANTHROPIC_API_KEY")
# 確認（APIは呼ばないので課金されない）
k = userdata.get("ANTHROPIC_API_KEY")
print("OK" if k and k.startswith("sk-ant-") else "NG: シークレットを確認")
```

このひと手間で、ノートを共有しても鍵が漏れません。
