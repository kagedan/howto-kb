# はじめに

AIコーディングツールの比較記事は世に溢れていますが、その多くは「数日使ってみた」程度のレベルに留まっています。

私は主にモバイル／フロントエンド領域の開発に従事しており、過去1年以上にわたり**Cursor、Claude Code、ChatGPT/Codex、Gemini**の4つのサブスクを同時に契約し、毎日3〜5時間、実際のプロジェクトで徹底的に使い比べてきました。

この記事では**ベンチマーク（スコア）の比較はしません。あくまで現場での「体感」を語ります。** 単刀直入に結論を出します。

# 次元その1：総合的なコーディング能力（モデル ＋ ツール）

コーディング能力 ≠ モデルのベンチマークスコアです。優れたツールと優れたモデルの組み合わせこそが、現場での真の生産性をもたらします。

| **順位** | **組み合わせ** | **評価** | **説明** |

| 🥇 | Claude CLI ＋ Opus 4.6 | S | ターミナルネイティブ。エージェント（Agent）モードの完成度が最も高く、複雑なリファクタリングで無双。 |

# 次元その2：利用可能枠（リミット）—— 同じ20ドルでどれだけ働けるか？

**ここが開発体験に最も直結する次元**であり、同時に皆さんが一番見落としがちな部分です。

分かりやすく比較するため、「**中規模タスク**」という単位を定義します。

中規模タスクの具体例：

* 既存のページにバリデーション付きのCRUD処理を追加する
* 3〜5つのファイルにまたがるバグを修正する
* 200行程度のコンポーネントをリファクタリングする

通常、AIと3〜5回やり取りし、15分程度かかる作業を指します。

## Cursor Pro —— 完全に置いていかれた ⭐⭐

**約50回の中規模タスク / 月**（20ドル API Pricing モード）

先月実際に計測したところ、わずか2日（8時間の稼働）で制限に達しました。

同じ20ドル払って、ChatGPTなら176個のタスクをこなせるのに、Cursorは当初の10分の1である50個しかできません。「少し劣る」というレベルではなく、もはや使う気を無くすレベルです。  
![](https://static.zenn.studio/user-upload/fdcbaf45f69a-20260406.png)

## Gemini Pro —— コーディングに関しては発展途上 ⭐⭐

Gemini Web版の調査能力は強力ですが、複雑なコーディングタスクの完了品質や効率においては、Claude CodeやCodexにはまだ明確な差があります。現段階では、コーディング目的だけでGeminiをサブスクするのはおすすめしません。

# 私の推奨構成

## 💰 ライト〜ミドルユーザー（1日2〜3時間）

**「ChatGPT Plus（月額20ドル）」1つで十分。**

Codexでコーディングし、Web版で調べ物やドキュメント作成をする。これで制限枠は余裕で足ります。

## 🔥 ヘビーユーザー（1日4時間以上）

**「ChatGPT Plus ＋ Claude Pro ＝ 月額40ドル」の2刀流。**

* 日常の量産タスク：Codex ＋ GPT-5.3 Codex（容量たっぷり、気にせずガンガン使える）
* 複雑なリファクタやアーキテクチャ設計：Claude CLI ＋ Opus 4.6（品質最高）
* 2つのプラットフォームで制限枠が独立しているため、同時に上限に達することはほぼありません。

# おわりに

AI2024年はCursorの独壇場でしたが、2025年にClaude Codeが台頭し、2026年にはCodexが再び業界をかき回しています。この分野の変化は本当に目まぐるしいです。

この記事の結論も2ヶ月後には古くなっているかもしれませんが、選ぶ際の判断基準は変わりません。**ベンチマークスコアに踊らされず、「20ドルでどれだけの作業ができるか」を見極め、流行を追うのではなく、自分の作業リズムに最もフィットするものを選びましょう。**
