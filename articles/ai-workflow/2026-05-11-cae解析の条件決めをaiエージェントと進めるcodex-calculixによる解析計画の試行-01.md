# AIエージェントとの対話による解析計画の立案

CAE解析を行う際には必要な情報が全部そろってから始められると理想的ですが、現実問題として不明・未確定な情報が存在するのはよくあることです。材料物性、荷重条件、拘束条件、評価基準、実機の使われ方など、最初の段階では未確定な情報が多くあります。むしろ、それらを確認しながら解析条件に落とし込んでいくこと自体が、CAE業務の重要な一部です。  
今回は、この「未確定な情報を整理して解析計画に落とし込む」プロセスを、AIエージェントと対話しながら進めてみました。  
題材として、SS400の丸棒に軸方向荷重と横荷重を与え、疲労破壊しないかを評価するケースを扱います。

## 使用するもの

* Codex App + GPT5.5 （AIエージェント）
* CalculiX （CAEソルバー）
* Gmsh （メッシュ作成用）
* meshio （メッシュ変換用）
* build123 （ジオメトリ作成用）
* FreeCAD （手作業によるジオメトリ作成用）

### AGENTS.md

プロジェクトディレクトリの構成や実行方法を教えておかないと無駄なトライ＆エラーが多発してトークンを無駄にしてしまうため`AGENTS.md`に記載しておきます。  
CAEワークフローのルールの設定もあわせて行っています。

## 今回の題材：情報が不足した状態から始める

以下の情報は既知とします。この時点では、疲労荷重の履歴、拘束条件、材料物性、疲労判定基準などは未確定です。  
今回は、この不足情報をAIエージェントとの対話で洗い出し、解析可能な条件に変換していきます。

FreeCADで作成したジオメトリ（ソリッド）。  
![](https://static.zenn.studio/user-upload/df0e1f14faf0-20260509.png)

### 材料

SS400ですが詳しい材料物性は未調査。

### 境界条件

軸方向に20kN負荷。軸と垂直方向に5kN負荷。

### 解析目的

疲労破壊しないかどうか検証する。

上記の情報だけではいろいろ不足しているので、AIエージェントと相談しながら詰めます。

cases\bar_tensile1\geometry\tensile_bar-Body.step

## Codexが作成した解析プラン

Codexが作成した解析計画は以下の通りです。

> # SS400 丸棒の複合荷重疲労評価計画
>
> ## Summary
>
> * 対象は [tensile\_bar-Body.step] の丸棒形状。STEP読み取り上、単位は mm、全長約200 mm、端部直径40 mm、中央最小直径30 mm。
> * `x=0` 端面を固定し、`x=200` 端面へ軸方向20 kNと横方向5 kNを同時に載荷する。
> * 荷重履歴は片振り `0 -> 最大荷重`、疲労判定はSS400の推定疲労限度を使う無限寿命Goodman評価とする。
> * 解析はまず線形弾性静解析で最大荷重時応力を求め、そこから平均応力・応力振幅を算出して疲労安全率を評価する。
>
> ## Key Changes
>
> * `cases/bar_tensile1/analysis.md` を作成し、目的、単位系、形状解釈、境界条件、荷重、材料、疲労評価式、限界事項を記録する。
> * ケース構成を標準レイアウトに揃える: `scripts/`, `input/`, `results/`, `report/figures/`。
> * GmshでSTEPを読み込み、3D二次四面体ソリッドメッシュを生成する。CalculiX側は `C3D10` を第一候補とし、中央絞り部と端面近傍を細かくする。
> * CalculiX入力は `main.inp` から `mesh.inc`, `materials.inc`, `sections.inc`, `bcs.inc`, `loads.inc`, `step.inc`, `output.inc` を include する構成にする。
> * 端面荷重は端面ノードセットに対する等価分布荷重、または端面参照点への剛体/分布結合で入力し、反力合計が `Fx=20 kN`, 横方向 `5 kN` と釣り合うことを確認する。
> * 材料初期値は SS400 のスクリーニング用として `E=206000 MPa`, `nu=0.30`, `Sy=245 MPa`, `Su=400 MPa`, 補正済み疲労限度 `Se=100 MPa` を使う。正式な材料証明・表面状態・仕上げ条件がある場合は置き換える。
> * 疲労評価は最大荷重時の相当応力を `sigma_max` とし、片振りなので `sigma_a=sigma_max/2`, `sigma_m=sigma_max/2` として Goodman:  
>   `damage = sigma_a/Se + sigma_m/Su`, `n_goodman = 1/damage`。
> * 静強度は最大von Mises応力と `Sy` の比で降伏安全率を併記する。
> * 日本語Markdownレポートを作成し、形状、メッシュ、境界条件/荷重、変形、von Mises応力、中央絞り部拡大、反力バランス、Goodman線図または疲労安全率表を含める。
>
> ## Test Plan
>
> * Python環境確認: `.\.venv\Scripts\python.exe` が起動できない場合は、まずvenvの状態を確認し、スクリプト実行可能な状態にする。
> * メッシュ確認: STEP読み込み、物理グループ、端面セット、要素タイプ、メッシュ品質、中央絞り部の要素密度を確認する。
> * Solver確認: CalculiX 2.23環境を初期化して `ccx jobname` で実行し、`.inp` 拡張子付きでは呼ばない。
> * 結果確認: 反力合計、変形方向、最大応力位置、中央絞り部の応力集中、異常な拘束応力の有無を確認する。
> * 収束性確認: 少なくとも粗・標準・細の3メッシュで最大応力とGoodman安全率の変化を比較する。
> * 受入基準: `n_goodman >= 1.0` なら無限寿命Goodman基準では疲労破壊しない判定、`n_goodman < 1.5` は余裕小として注意記載。降伏安全率も `>=1.0` を最低条件とする。
>
> ## Assumptions
>
> * 座標系はSTEP形状の長手方向を `x` 軸とする。
> * 横荷重5 kNの方向は仮に `+Y` とする。
> * 端面は理想的に固定・載荷される。実治具の局所接触、把持滑り、ボルト/ピン、溶接などはモデル化しない。
> * 疲労荷重は20 kN軸力と5 kN横荷重が同位相で `0 -> 最大` を繰り返す。
> * SS400の疲労限度は材料証明がないため保守的なスクリーニング値を使う。最終設計判定には実材料、表面粗さ、寸法効果、残留応力、腐食環境を反映した値が必要。

### 解析プランを見て気づいたこと

Codexが作成した解析プランを見ると、単にCalculiXの入力ファイルを作る方針を書いているだけではなく、解析の前提条件を文章として整理しています。

#### 興味深い点

今回、最初のプロンプトでは形状の寸法や単位系を明示していませんでしたが、Codexは提供したSTEPデータから形状と単位系を読み取り、それをもとに固定端と荷重端を設定していました。AIエージェントがジオメトリ情報を解析条件へ変換できるという点で興味深いです。

解析タイプとしては「疲労で壊れないかどうか」という解析目的に照らして、線形弾性静解析で最大荷重時の応力を求め、片振り荷重として平均応力と応力振幅を算出し、Goodman評価につなげる方針になっていました。初期検討・スクリーニング評価として自然な流れです。

メッシュはCalculiXのC3D10（四面体2次要素）を第一候補とし、中央絞り部と端面近傍を細かくする方針で、解析上の着目点がある程度反映されています。

プランでは確認項目も組み込まれています。反力合計、変形方向、最大応力位置、中央絞り部の応力集中、異常な拘束応力の有無を確認し、さらに粗・標準・細の3メッシュで最大応力とGoodman安全率の変化を比較する、という方針です。

#### 注意すべき点

このプランでは仮定をいくつかおいています。x=0端面を完全固定し、x=200端面に軸方向20kNと横方向5kNを同時に載荷する条件は解析モデルとしては明快ですが、実機の治具や把持条件をそのまま表しているとは限りません。AIエージェントは、不明な情報がある場合でも、それらしい仮定を置いて前に進めてくれますが実機条件として妥当か最終的には人間が判断する必要があります。

材料物性はSS400の代表値として弾性係数、降伏応力、引張強さ、疲労限度が設定されていましたが根拠が十分に明示されていないので、後で材料データの出典を明記するように追加で依頼が必要です。

#### 材料データの出典を要求

材料データの根拠が不明だったため、すぐに計画を承認せずに修正を依頼してみます。

すると、Webから情報を探索して出典を表示してくれました。  
通常は材料選定の段階でこれらの情報は何かしらあるとは思いますが、もしなければこのようにWebで探した情報をもとに実施するのも一案だと思います。

> ## Material Data And Sources

この解析は方針さえ決まれば簡単な内容なので、このプランで実行してもらいます。

### AIエージェントで解析計画を作れるが人間のレビューは必要

Codexは未確定な情報を放置せず、暫定条件として整理してくれました。これは非常に便利です。  
一方で、その暫定条件をそのまま受け入れてよいかどうかは、実際の材料や実機の状況などによります。  
AIエージェントは仮定を置いて前に進めてくれる力はありますが、その仮定が設計判断に使えるものかは人間が確認する必要があります。

## 今回見えた、AIエージェントの使いどころ

今回の試行で、AIエージェントは以下の用途では有効そうだと感じました。

* 不足情報の洗い出し
* 仮定条件の明文化
* 解析計画のドラフト作成
* 材料物性や疲労評価式の候補提示、Web調査

一方で、以下は人間側の判断が必要です。

* 実機の拘束条件や荷重条件に照らして、仮定条件を受け入れてよいかの判断
* AIが示した材料物性や疲労限度を設計判断に使ってよいかの判断

## まとめ

今回の試行では、未確定な情報が多い状態から、AIエージェントと対話しながら不足情報を洗い出し、仮定を置き、解析計画として整理することができました。CAE解析では、ソルバーを実行する前の「条件決め」が重要です。AIエージェントは、この条件決めのプロセスを支援し、人が確認すべき論点を見える化する役割を持てると思われます。

一方で、AIが置いた仮定は実機の状態や実際の材料と照らして妥当かどうか人間で確認・判断する必要があります。AIエージェントは未確定情報を構造化し、確認すべき論点を見える化するための支援役として使うのが現実的だと思います。
