## ⓪ tldr

* Claude（無料プラン含む）のチャットだけで、レコーディングダイエット＋トレーニング管理ができる
* 「朝ごはんプロテインとバナナ食べた」と話すだけ。カロリー計算もPFC管理もClaudeがやる
* データはPGlite（ブラウザで動くPostgreSQL）でClaude内部に保存。DB関数でロジックもDB側に持つ
* 性別・身長・年齢・目標・運動環境をヒアリングして、あなた専用のプランを自動構築
* 会話が終わったらデータをダウンロード → 次の会話で復元
* **この記事をClaudeに読み込ませるだけで始められる**

## ① これは何？

### レコーディングダイエットの問題

食べたものを記録するだけで痩せる。原理はシンプル。問題は**記録がめんどくさくて続かない**こと。

### この記事の解決策

Claudeに話しかけるだけで記録完了。カロリー検索不要。PFC計算不要。Claudeが全部やる。

|  | ダイエットアプリ | Claude + この記事 |
| --- | --- | --- |
| カロリー入力 | 食品を検索して手入力 | 「牛丼食べた」で終わり |
| PFC計算 | 自分で足し算 | 自動計算＋残高表示 |
| トレーニング提案 | 汎用メニュー | あなたの前回記録・性別・環境に基づく個別提案 |
| 料金 | 月額500〜2,000円 | 無料 |
| データ所有権 | アプリに閉じ込め | SQLファイルであなたが持つ |

### なぜPGliteか

|  | SQLite | PGlite |
| --- | --- | --- |
| DB関数 (CREATE FUNCTION) | ❌ 使えない | ✅ plpgsqlが動く |
| PostgreSQL互換 | ❌ SQL方言が違う | ✅ 完全互換 |
| セッション引き継ぎ | データだけ | データ＋ロジックごと復元 |

DB関数があると「食事を記録して残高を返す」を1回の呼び出しで完結できる。ロジックがDB内に入るので、ダンプ→復元すればロジックごと復活する。

## ② セットアップ

### やること: この記事をClaudeに渡すだけ

claude.aiで新しい会話を開き、この記事のファイルを添付して:

> 「このドキュメントに従ってレコーディングダイエットの環境を作って」

### Claudeが自動でやること

**Phase 1: 環境構築（全員共通）**

1. PGliteをインストール
2. テーブル＋DB関数を作成

**Phase 2: ヒアリング（5問）**

Claudeがあなたに質問します:

```
Q1: 性別を教えてください → [男性 / 女性 / その他]
Q2: 身長(cm)・年齢・現在の体重(kg)を教えてください
Q3: 目標体重と期限はありますか？（「特にない」もOK）
Q4: 食事の方針は？ → [高タンパク型（筋量維持しながら絞りたい） / バランス型（普通に痩せたい）]
Q5: 運動環境は？ → [ジムに通っている / 自宅でトレーニング / 運動習慣なし]
```

**Phase 3: あなた専用のプラン提示**

```
あなたの基礎代謝: 約1,650kcal（Mifflin-St Jeor式）
減量ペース: 月1〜1.5kg
1日の目標: Cal 1,320 / P 75g / F 42g / C 130g
トレーニング: 自宅メニュー（自重スクワット+腕立て+プランク）

これでいく？ 調整したい項目があれば教えて
```

合意したらスタート。

### PFC計算の仕組み

```
基礎代謝（Mifflin-St Jeor式）:
  男性: 10 × 体重(kg) + 6.25 × 身長(cm) - 5 × 年齢 + 5
  女性: 10 × 体重(kg) + 6.25 × 身長(cm) - 5 × 年齢 - 161

カロリー目標 = 基礎代謝 × 0.8（月1〜1.5kg減ペース）

タンパク質(P):
  高タンパク型 = 体重 × 1.5g（筋量維持）
  バランス型   = 体重 × 1.0g（一般的な減量）

脂質(F) = 体重 × 0.6g
炭水化物(C) = (カロリー - P×4 - F×9) ÷ 4
```

Claudeの会話はセッション単位。新しい会話を始めると前のデータは消える。でもデータをファイルに書き出して次の会話で読み込めばOK。

```
あなた: 「データを保存して」
Claude: ✅ ダンプ完了。diet_backup_20260318.sql をダウンロードしてください。
```

### 次の会話で復元

1. この記事のファイルを添付
2. バックアップファイル（.sql）を添付
3. 「環境構築してバックアップから復元して」

Claudeが自動で全部やる。**ファイル2つを添付するだけ。**

### 会話が重くなったら

応答が遅くなったり、以前の内容を忘れ始めたら → データ保存して新しい会話を始める。

## ⑤ トレーニングの仕組み（ジム通いの人向け）

### 3分割ローテーション

**男性（ジム）:**

| 日 | メイン | 補助 |
| --- | --- | --- |
| A日: 足+腰 | デッドリフト | スクワット(軽め) + カーフレイズ |
| B日: 胸+肩+腕 | ベンチプレス | ショルダープレス + サイドレイズ + ダンベルカール |
| C日: 足+背中 | スクワット | ラットプルダウン + カーフレイズ |

**女性（ジム）:**

| 日 | メイン | 補助 |
| --- | --- | --- |
| A日: 足+臀部 | スクワット | ヒップスラスト + ブルガリアンスクワット |
| B日: 背中+体幹 | デッドリフト | ラットプルダウン + プランク |
| C日: 臀部+肩 | ヒップスラスト | グルートブリッジ + ショルダープレス + カーフレイズ |

**自宅トレ（共通）:**

| 日 | メイン | 補助 |
| --- | --- | --- |
| A日: 下半身 | 自重スクワット | ブルガリアンスクワット + カーフレイズ |
| B日: 上半身 | 腕立て伏せ | ダンベルショルダープレス + ダンベルロウ |
| C日: 全身+体幹 | ダンベルデッドリフト | プランク + カーフレイズ |

### 2週サイクル（ジム通いの人）

| 週 | 目的 | やり方 | 休息 |
| --- | --- | --- | --- |
| Week A: MAX探り | 最大重量を見つける | ウォームアップで段階的に重くし、前回MAX→+5kgチャレンジ | 3〜5分 |
| Week B: 維持 | MAXを体に定着させる | 80%1RM × 10回, 9回, 8回（ダウンセット） | 90秒〜2分 |

### 運動習慣なしの人の段階的導入

```
Week 1-2: 1日20分歩く（有酸素のみ）
Week 3-4: 歩き + 自重トレ2種目（スクワット+プランク）
Week 5+:  歩き + 自重トレ3-4種目
慣れたら: ジムや自宅ダンベルにステップアップ
```

## ⑥ 罠チートシート

| 罠 | 影響 | 対処 |
| --- | --- | --- |
| 会話終了でデータ消失 | 🔴 全記録ロスト | 毎回「データ保存して」でダンプ |
| カロリー推定の誤差 | 🟡 ±10〜20%ズレる | 精度より継続。だいたいでOK |
| PFC目標が合ってない | 🟡 効果が出にくい | 2週間体重が動かなければ再計算を相談 |
| 会話が重くなる | 🟡 応答遅延 | 週1で新しい会話に切り替え |
| フォームが不明 | 🟡 怪我リスク | Claudeに「スクワットのフォーム教えて」で解説してもらう |
| 目標が厳しすぎて辛い | 🟡 挫折 | Claudeに「目標緩めたい」と相談。再計算してくれる |

### Phase 2: ヒアリング後のパーソナライズ

```
// Mifflin-St Jeor式で基礎代謝を計算
function calculateBMR(gender, weightKg, heightCm, age) {
  if (gender === 'male') return 10 * weightKg + 6.25 * heightCm - 5 * age + 5;
  return 10 * weightKg + 6.25 * heightCm - 5 * age - 161;
}

function calculatePFCTargets(gender, weightKg, heightCm, age, dietStyle) {
  const bmr = calculateBMR(gender, weightKg, heightCm, age);
  const calTarget = Math.round(bmr * 0.8);
  const proteinPerKg = dietStyle === 'high_protein' ? 1.5 : 1.0;
  const pTarget = Math.round(weightKg * proteinPerKg);
  const fTarget = Math.round(weightKg * 0.6);
  const cTarget = Math.max(Math.round((calTarget - pTarget * 4 - fTarget * 9) / 4), 50);
  return { calTarget, pTarget, fTarget, cTarget, bmr };
}

// ヒアリング結果をsettingsに保存
async function saveSettings(db, { gender, heightCm, age, weightKg, weightTarget, targetDate, dietStyle, trainingEnv }) {
  const { calTarget, pTarget, fTarget, cTarget, bmr } = calculatePFCTargets(gender, weightKg, heightCm, age, dietStyle);
  const settings = {
    gender, height_cm: heightCm, age, diet_style: dietStyle, training_env: trainingEnv,
    bmr, cal_target: calTarget, p_target: pTarget, f_target: fTarget, c_target: cTarget,
    weight_target: weightTarget || '', target_date: targetDate || ''
  };
  for (const [key, value] of Object.entries(settings)) {
    await db.query(`INSERT INTO diet_settings (key, value) VALUES ($1, $2) ON CONFLICT (key) DO UPDATE SET value = $2`, [key, String(value)]);
  }
  return { calTarget, pTarget, fTarget, cTarget, bmr };
}

// 種目マスタの選定

  ],
  home: [
    ['自重スクワット','足','big3'],['腕立て伏せ','胸','big3'],['ダンベルデッドリフト','背中','big3'],

function selectPreset(gender, trainingEnv) {

### Phase 3: DB関数（全8つ）

```
-- 1. PFC残高（settingsから目標値を読む）
CREATE OR REPLACE FUNCTION get_pfc_balance(target_date DATE DEFAULT CURRENT_DATE)
RETURNS TABLE(total_cal INT, total_p INT, total_f INT, total_c INT,
  cal_remaining INT, p_remaining INT, f_remaining INT, c_remaining INT,
  cal_target INT, p_target INT, f_target INT, c_target INT)
LANGUAGE plpgsql AS $$
DECLARE vt_cal INT; vt_p INT; vt_f INT; vt_c INT;
BEGIN
  SELECT COALESCE((SELECT value::INT FROM diet_settings WHERE key='cal_target'),1900) INTO vt_cal;
  SELECT COALESCE((SELECT value::INT FROM diet_settings WHERE key='p_target'),135) INTO vt_p;
  SELECT COALESCE((SELECT value::INT FROM diet_settings WHERE key='f_target'),50) INTO vt_f;
  SELECT COALESCE((SELECT value::INT FROM diet_settings WHERE key='c_target'),240) INTO vt_c;
  RETURN QUERY SELECT
    COALESCE(SUM(estimated_calories),0)::INT, COALESCE(SUM(estimated_protein_g),0)::INT,
    COALESCE(SUM(estimated_fat_g),0)::INT, COALESCE(SUM(estimated_carb_g),0)::INT,
    (vt_cal-COALESCE(SUM(estimated_calories),0))::INT, (vt_p-COALESCE(SUM(estimated_protein_g),0))::INT,
    (vt_f-COALESCE(SUM(estimated_fat_g),0))::INT, (vt_c-COALESCE(SUM(estimated_carb_g),0))::INT,
    vt_cal, vt_p, vt_f, vt_c
  FROM diet_meals WHERE recorded_on = target_date;
END; $$;

-- 2. 食事INSERT+残高
CREATE OR REPLACE FUNCTION insert_meal_with_balance(
  p_date DATE, p_slot TEXT, p_name TEXT,
  p_cal INT DEFAULT 0, p_protein INT DEFAULT 0, p_fat INT DEFAULT 0, p_carb INT DEFAULT 0)
RETURNS TABLE(inserted_id INT, total_cal INT, total_p INT, total_f INT, total_c INT,
  cal_remaining INT, p_remaining INT, f_remaining INT, c_remaining INT)
LANGUAGE plpgsql AS $$
DECLARE new_id INT; vt_cal INT; vt_p INT; vt_f INT; vt_c INT;
BEGIN
  SELECT COALESCE((SELECT value::INT FROM diet_settings WHERE key='cal_target'),1900) INTO vt_cal;
  SELECT COALESCE((SELECT value::INT FROM diet_settings WHERE key='p_target'),135) INTO vt_p;
  SELECT COALESCE((SELECT value::INT FROM diet_settings WHERE key='f_target'),50) INTO vt_f;
  SELECT COALESCE((SELECT value::INT FROM diet_settings WHERE key='c_target'),240) INTO vt_c;
  INSERT INTO diet_meals (recorded_on,meal_slot,meal_name,estimated_calories,estimated_protein_g,estimated_fat_g,estimated_carb_g)
  VALUES (p_date,p_slot,p_name,p_cal,p_protein,p_fat,p_carb) RETURNING id INTO new_id;
  RETURN QUERY SELECT new_id,
    COALESCE(SUM(estimated_calories),0)::INT, COALESCE(SUM(estimated_protein_g),0)::INT,
    COALESCE(SUM(estimated_fat_g),0)::INT, COALESCE(SUM(estimated_carb_g),0)::INT,
    (vt_cal-COALESCE(SUM(estimated_calories),0))::INT, (vt_p-COALESCE(SUM(estimated_protein_g),0))::INT,
    (vt_f-COALESCE(SUM(estimated_fat_g),0))::INT, (vt_c-COALESCE(SUM(estimated_carb_g),0))::INT
  FROM diet_meals WHERE recorded_on = p_date;
END; $$;

-- 3. トレーニング判定
CREATE OR REPLACE FUNCTION get_training_recommendation()
RETURNS TABLE(day_type TEXT, next_rotation TEXT, next_week TEXT, last_session_date DATE, training_env TEXT)
LANGUAGE plpgsql AS $$
DECLARE v_last_rot TEXT; v_next_rot TEXT; v_same_rot_type TEXT; v_last_date DATE; v_env TEXT;
BEGIN
  SELECT COALESCE((SELECT value FROM diet_settings WHERE key='training_env'),'none') INTO v_env;
  SELECT ts.rotation, ts.recorded_on INTO v_last_rot, v_last_date
  FROM diet_training_sessions ts ORDER BY ts.recorded_on DESC LIMIT 1;
  IF v_last_rot IS NULL THEN v_next_rot:='all';
  ELSIF v_last_rot='A' THEN v_next_rot:='B'; ELSIF v_last_rot='B' THEN v_next_rot:='C';
  ELSIF v_last_rot='C' THEN v_next_rot:='A'; ELSIF v_last_rot='all' THEN v_next_rot:='A'; END IF;
  IF v_last_date IS NULL THEN v_next_rot:='all'; END IF;
  SELECT ts.session_type INTO v_same_rot_type FROM diet_training_sessions ts WHERE ts.rotation=v_next_rot ORDER BY ts.recorded_on DESC LIMIT 1;
  RETURN QUERY SELECT
    CASE WHEN EXISTS(SELECT 1 FROM diet_cardio WHERE recorded_on=CURRENT_DATE-1 AND estimated_burned_calories>200) THEN 'recommend_strength'
         WHEN EXISTS(SELECT 1 FROM diet_training_sessions WHERE recorded_on=CURRENT_DATE-1) THEN 'recommend_cardio_or_rest'
         ELSE 'either' END,
    v_next_rot,
    CASE WHEN v_env='none' THEN 'cardio_only'
         WHEN v_last_date IS NULL THEN 'initial'
         WHEN v_same_rot_type IS NULL THEN 'max_test'
         WHEN v_same_rot_type='max_test' THEN 'maintain'
         ELSE 'max_test' END,
    v_last_date, v_env;
END; $$;

-- 4. 1日サマリ
CREATE OR REPLACE FUNCTION get_daily_summary(target_date DATE DEFAULT CURRENT_DATE)
RETURNS TABLE(meal_cal INT, meal_p INT, meal_f INT, meal_c INT, cardio_burned INT,
  strength_sets INT, weight_kg NUMERIC, p_target_hit BOOLEAN, cal_in_range BOOLEAN)
LANGUAGE plpgsql AS $$
DECLARE vt_cal INT; vt_p INT;
BEGIN
  SELECT COALESCE((SELECT value::INT FROM diet_settings WHERE key='cal_target'),1900) INTO vt_cal;
  SELECT COALESCE((SELECT value::INT FROM diet_settings WHERE key='p_target'),135) INTO vt_p;
  RETURN QUERY SELECT
    COALESCE((SELECT SUM(estimated_calories) FROM diet_meals WHERE recorded_on=target_date),0)::INT,
    COALESCE((SELECT SUM(estimated_protein_g) FROM diet_meals WHERE recorded_on=target_date),0)::INT,
    COALESCE((SELECT SUM(estimated_fat_g) FROM diet_meals WHERE recorded_on=target_date),0)::INT,
    COALESCE((SELECT SUM(estimated_carb_g) FROM diet_meals WHERE recorded_on=target_date),0)::INT,
    COALESCE((SELECT SUM(estimated_burned_calories) FROM diet_cardio WHERE recorded_on=target_date),0)::INT,
    COALESCE((SELECT COUNT(*) FROM diet_strength_sets WHERE recorded_on=target_date),0)::INT,
    (SELECT bw.weight_kg FROM diet_body_weights bw WHERE bw.recorded_on=target_date),
    COALESCE((SELECT SUM(estimated_protein_g) FROM diet_meals WHERE recorded_on=target_date),0) >= vt_p - 5,
    COALESCE((SELECT SUM(estimated_calories) FROM diet_meals WHERE recorded_on=target_date),0) BETWEEN vt_cal-100 AND vt_cal+150;
END; $$;

-- 5. 前回ローテセット参照
CREATE OR REPLACE FUNCTION get_last_rotation_sets(rot TEXT)
RETURNS TABLE(exercise_name TEXT, target_muscle TEXT, set_number INT, weight_kg NUMERIC, reps INT, recorded_on DATE)
LANGUAGE plpgsql AS $$
DECLARE v_date DATE;
BEGIN
  SELECT ts.recorded_on INTO v_date FROM diet_training_sessions ts WHERE ts.rotation=rot ORDER BY ts.recorded_on DESC LIMIT 1;
  IF v_date IS NULL THEN RETURN; END IF;
  RETURN QUERY SELECT s.exercise_name, m.target_muscle, s.set_number, s.weight_kg, s.reps, s.recorded_on
  FROM diet_strength_sets s JOIN diet_exercise_master m ON s.exercise_name=m.exercise_name
  WHERE s.recorded_on=v_date ORDER BY s.exercise_name, s.set_number;
END; $$;

-- 6. 種目別MAX
CREATE OR REPLACE FUNCTION get_exercise_maxes()
RETURNS TABLE(exercise_name TEXT, target_muscle TEXT, estimated_1rm NUMERIC, heaviest_weight NUMERIC)
LANGUAGE plpgsql AS $$
BEGIN
  RETURN QUERY SELECT s.exercise_name, m.target_muscle,
    ROUND(MAX(s.weight_kg*(1+s.reps::numeric/30)),1), MAX(s.weight_kg)
  FROM diet_strength_sets s JOIN diet_exercise_master m ON s.exercise_name=m.exercise_name
  GROUP BY s.exercise_name, m.target_muscle ORDER BY m.target_muscle;
END; $$;

-- 7. 体重推移
CREATE OR REPLACE FUNCTION get_weight_trend(days INT DEFAULT 14)
RETURNS TABLE(recorded_on DATE, weight_kg NUMERIC, daily_change NUMERIC)
LANGUAGE plpgsql AS $$
BEGIN
  RETURN QUERY SELECT bw.recorded_on, bw.weight_kg,
    ROUND(bw.weight_kg - LAG(bw.weight_kg) OVER (ORDER BY bw.recorded_on), 2)
  FROM diet_body_weights bw WHERE bw.recorded_on >= CURRENT_DATE - days ORDER BY bw.recorded_on;
END; $$;

-- 8. ペースチェック
CREATE OR REPLACE FUNCTION get_pace_check()
RETURNS TABLE(current_weight NUMERIC, target_weight NUMERIC, avg_daily_change NUMERIC,
  projected_target NUMERIC, days_remaining INT, on_track BOOLEAN)
LANGUAGE plpgsql AS $$
DECLARE v_cur NUMERIC; v_tgt NUMERIC; v_avg NUMERIC; v_days INT;
BEGIN
  SELECT bw.weight_kg INTO v_cur FROM diet_body_weights bw ORDER BY bw.recorded_on DESC LIMIT 1;
  SELECT COALESCE(NULLIF((SELECT value FROM diet_settings WHERE key='weight_target'),'')::NUMERIC, v_cur-5) INTO v_tgt;
  SELECT ROUND(AVG(d),4) INTO v_avg FROM (SELECT bw.weight_kg-LAG(bw.weight_kg) OVER (ORDER BY bw.recorded_on) d FROM diet_body_weights bw WHERE bw.recorded_on>=CURRENT_DATE-14) sub WHERE d IS NOT NULL;
  SELECT COALESCE(NULLIF((SELECT value FROM diet_settings WHERE key='target_date'),'')::DATE - CURRENT_DATE, 180)::INT INTO v_days;
  RETURN QUERY SELECT v_cur, v_tgt, COALESCE(v_avg,0), ROUND(v_cur+COALESCE(v_avg,0)*v_days,1), v_days, v_cur+COALESCE(v_avg,0)*v_days<=v_tgt;
END; $$;

### Claudeへの指示（環境構築後の振る舞い）

**初回セットアップ直後:**

* Phase 1のDB構築を実行
* 5問のヒアリングを実施（Q1〜Q5）
* ヒアリング結果からPFC目標を計算して提示
* ユーザーの合意を得てからsettingsに保存＋種目マスタINSERT
* **何も聞かずに固定値を入れるのは禁止**

**食事報告を受けたら:**

* カロリー・PFCをWeb検索で推定
* `insert_meal_with_balance` でINSERT＋残高を1発で返す
* 残高を見やすく表示

**体重報告を受けたら:**

* diet\_body\_weightsにINSERT
* 前日比を表示
* 今日のPFC残高を表示

**「ジム行く」「運動したい」と言われたら:**

* `get_training_recommendation` で判定
* training\_env に応じた提案（ジム/自宅/散歩のみ）
* 前回記録があれば `get_last_rotation_sets` で表示

**トレーニング報告を受けたら:**

* diet\_strength\_sets にINSERT
* diet\_training\_sessions にINSERT（忘れないこと）

**「今日どうだった？」と言われたら:**

* `get_daily_summary` で1日サマリを表示

**「データ保存して」と言われたら:**

* 全テーブルのデータをSQL INSERT文に変換
* .sqlファイルとしてダウンロード可能にする
* DB関数のCREATE文も含める（復元時にロジックごと復活）

**目標変更を申告されたら:**

* diet\_settingsを更新
* PFC目標を再計算して提示
* 合意後にUPDATE

**2週間以上体重が動いていない場合:**

* get\_pace\_checkで検知
* 「目標Cal下げる？ 運動増やす？ もう少し様子見る？」と選択肢提示

**食事提案のスタイル:**

* 命令形（「食え」「やめろ」）ではなく提案形（「〜がおすすめ」「〜にするとFが収まるよ」）
* 判断の主権はユーザーにある
* アレルギーや好みを伝えられたら考慮する
