## はじめに

AIエージェントを作りたいとき、最初にぶつかる壁が「どうやって処理の流れを制御するか」です。

LLMに単発で答えさせるのは簡単ですが、「検索して→評価して→足りなければ再検索して→回答する」といった複数ステップのフローになった途端、コードが複雑になります。

**LangGraph** はその問題を「グラフ構造」で解決するフレームワークです。処理をノード（関数）とエッジ（接続）で定義し、条件分岐・ループ・状態管理を明示的に書けます。

2026年現在、AIエージェント開発のデファクトスタンダードとして採用が進んでおり、「CrewAIでPoC → LangGraphで本番移行」というパスが業界標準になりつつあります。

本記事では、LangGraphのコア概念から動くコードまで、段階的に解説します。

## LangGraphの3つのコア概念

LangGraphを理解するには、まず以下の3つを押さえると全体像が見えます。

### 1. State（状態）

グラフ全体で共有される「メモリ」です。ノード間でデータをやり取りするための器で、TypedDictで定義します。

```
from typing import TypedDict, List

class State(TypedDict):
    messages: List[str]   # 会話履歴
    query: str            # ユーザーの質問
    documents: List[str]  # 検索結果
    answer: str           # 最終回答
```

各ノードは Stateを受け取り、更新した辞書を返します。LangGraphが差分をマージして次のノードに渡します。

### 2. Node（ノード）

実際の処理を担う関数です。Stateを受け取りStateを返します。

```
def search_node(state: State) -> dict:
    """検索ノード：クエリに関連する文書を取得"""
    results = vector_store.similarity_search(state["query"])
    return {"documents": [doc.page_content for doc in results]}

def generate_node(state: State) -> dict:
    """生成ノード：文書を元に回答を作成"""
    context = "\n".join(state["documents"])
    answer = llm.invoke(f"Context: {context}\nQuestion: {state['query']}")
    return {"answer": answer.content}
```

### 3. Edge（エッジ）

ノード間の接続です。通常のエッジ（常に次のノードへ）と条件付きエッジ（状態に応じて分岐）の2種類があります。

```
# 通常のエッジ：search → generate へ常に進む
workflow.add_edge("search", "generate")

# 条件付きエッジ：状態を見て分岐を決める
workflow.add_conditional_edges(
    "evaluate",
    decide_next,  # 分岐ロジックの関数
    {
        "retry": "search",     # 再検索
        "generate": "generate" # 回答生成へ
    }
)
```

## 最小構成：シンプルなRAGエージェント

まず動くものを作ります。検索→回答の2ノード構成です。

```
from langgraph.graph import StateGraph, END
from langchain_openai import ChatOpenAI
from langchain_community.vectorstores import Chroma
from langchain_openai import OpenAIEmbeddings
from typing import TypedDict, List

# 状態定義
class RAGState(TypedDict):
    query: str
    documents: List[str]
    answer: str

# LLMとベクトルDBの初期化
llm = ChatOpenAI(model="gpt-4o-mini")
vectorstore = Chroma(embedding_function=OpenAIEmbeddings())

# ノード定義
def retrieve(state: RAGState) -> dict:
    docs = vectorstore.similarity_search(state["query"], k=3)
    return {"documents": [d.page_content for d in docs]}

def generate(state: RAGState) -> dict:
    context = "\n\n".join(state["documents"])
    prompt = f"""以下のコンテキストを参考に質問に答えてください。

コンテキスト:
{context}

質問: {state["query"]}"""
    response = llm.invoke(prompt)
    return {"answer": response.content}

# グラフ構築
workflow = StateGraph(RAGState)
workflow.add_node("retrieve", retrieve)
workflow.add_node("generate", generate)

workflow.set_entry_point("retrieve")
workflow.add_edge("retrieve", "generate")
workflow.add_edge("generate", END)

app = workflow.compile()

# 実行
result = app.invoke({"query": "LangGraphとは何ですか？"})
print(result["answer"])
```

## 条件分岐：評価ループ付きRAG

次に「検索結果が不十分なら再検索する」ループを追加します。

```
from langgraph.graph import StateGraph, END
from typing import TypedDict, List, Literal

class AgenticRAGState(TypedDict):
    query: str
    documents: List[str]
    answer: str
    retry_count: int

def retrieve(state: AgenticRAGState) -> dict:
    docs = vectorstore.similarity_search(state["query"], k=3)
    return {"documents": [d.page_content for d in docs]}

def evaluate(state: AgenticRAGState) -> dict:
    """取得文書の関連度をLLMで評価"""
    if not state["documents"]:
        return {"retry_count": state.get("retry_count", 0) + 1}
    
    prompt = f"""以下の文書は質問「{state['query']}」に関連していますか？
文書: {state['documents'][0][:200]}
「yes」か「no」だけ答えてください。"""
    
    result = llm.invoke(prompt).content.strip().lower()
    if result == "no":
        return {"retry_count": state.get("retry_count", 0) + 1}
    return {"retry_count": state.get("retry_count", 0)}

def decide_next(state: AgenticRAGState) -> Literal["retry", "generate"]:
    """次のノードを決定する分岐ロジック"""
    if state.get("retry_count", 0) >= 2:
        return "generate"  # リトライ上限 → 回答生成へ
    if state.get("retry_count", 0) > 0:
        return "retry"     # 関連度低 → 再検索
    return "generate"      # 関連度高 → 回答生成へ

def generate(state: AgenticRAGState) -> dict:
    context = "\n\n".join(state["documents"]) if state["documents"] else "関連情報が見つかりませんでした。"
    response = llm.invoke(f"Context: {context}\nQuestion: {state['query']}")
    return {"answer": response.content}

# グラフ構築
workflow = StateGraph(AgenticRAGState)
workflow.add_node("retrieve", retrieve)
workflow.add_node("evaluate", evaluate)
workflow.add_node("generate", generate)

workflow.set_entry_point("retrieve")
workflow.add_edge("retrieve", "evaluate")
workflow.add_conditional_edges(
    "evaluate",
    decide_next,
    {"retry": "retrieve", "generate": "generate"}
)
workflow.add_edge("generate", END)

app = workflow.compile()
```

## チェックポイント：会話履歴の永続化

LangGraphには **チェックポイント** 機能があります。実行途中の状態を保存し、後から再開できます。会話履歴の保持や、Human-in-the-Loopの実装に使います。

```
from langgraph.checkpoint.memory import MemorySaver

# MemorySaverでインメモリ保存（本番ではSqliteSaverやPostgresSaverを使う）
memory = MemorySaver()
app = workflow.compile(checkpointer=memory)

# thread_idで会話セッションを識別
config = {"configurable": {"thread_id": "user-123"}}

# 1回目の実行
result1 = app.invoke({"query": "LangGraphとは？"}, config=config)

# 同じthread_idで続きから再開できる
result2 = app.invoke({"query": "具体的な使い方は？"}, config=config)
```

## Human-in-the-Loop：人間の確認を挟む

エージェントが自律的に動作する前に人間の確認を入れたい場合、`interrupt_before` で特定ノードの手前で一時停止できます。

```
# 「generate」ノードの前で停止
app = workflow.compile(
    checkpointer=memory,
    interrupt_before=["generate"]
)

config = {"configurable": {"thread_id": "review-session"}}

# ここで一時停止→取得した文書をユーザーに確認させる
result = app.invoke({"query": "競合分析をしてください"}, config=config)
print("取得文書:", result["documents"])

# ユーザーがOKしたら再開
final_result = app.invoke(None, config=config)
print("最終回答:", final_result["answer"])
```

## create\_react\_agent：最速でエージェントを作る

シンプルなReActエージェントなら、`create_react_agent` で数行で作れます。

```
from langgraph.prebuilt import create_react_agent
from langchain_community.tools.tavily_search import TavilySearchResults

# ツールを定義
tools = [TavilySearchResults(max_results=3)]

# ReActエージェントを作成（グラフを手書きしなくてよい）
agent = create_react_agent(
    model=ChatOpenAI(model="gpt-4o-mini"),
    tools=tools,
    state_modifier="あなたは日本語で回答する調査アシスタントです。"
)

result = agent.invoke({
    "messages": [{"role": "user", "content": "2026年のAIエージェント市場動向を調べて"}]
})
print(result["messages"][-1].content)
```

`create_react_agent` は内部的にLangGraphのグラフを自動生成します。カスタマイズが必要になったら、手書きのグラフに移行するのがおすすめです。

## LangChainとの使い分け

よく混同されるので整理します。

|  | LangChain | LangGraph |
| --- | --- | --- |
| 主な役割 | LLM・ツール・プロンプトの部品集 | ワークフローのオーケストレーション |
| 処理フロー | 線形（A→B→C） | グラフ（条件分岐・ループ可） |
| 状態管理 | 限定的 | 明示的なState管理 |
| 向いているもの | 単純なチェーン処理 | 複雑なエージェントフロー |
| 関係 | LangGraphの部品として使う | LangChainの上に構築されている |

LangGraphはLangChainの置き換えではなく、**LangChainのコンポーネントをオーケストレーションするレイヤー**です。

## まとめ：学習ロードマップ

```
Step 1：StateとNodeの概念を理解する
         → 最小RAGエージェントを動かす

Step 2：条件付きエッジでループを実装する
         → 評価→再検索の分岐を書く

Step 3：チェックポイントで状態を永続化する
         → 会話履歴の保持を実装

Step 4：Human-in-the-Loopで承認フローを作る
         → 本番品質のワークフローへ

Step 5：マルチエージェント構成に挑戦する
         → 複数グラフをサブグラフとして組み合わせる
```

LangGraphは最初の概念理解さえできれば、あとは組み合わせ次第でどんな複雑なエージェントも作れます。まずは `create_react_agent` で動くものを作り、カスタマイズが必要になったら手書きのグラフに移行するのが現実的なパスです。
