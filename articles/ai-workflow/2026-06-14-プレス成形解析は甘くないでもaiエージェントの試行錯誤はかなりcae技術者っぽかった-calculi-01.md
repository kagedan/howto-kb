# プレス成形解析は甘くない。でもAIエージェントの試行錯誤はかなりCAE技術者っぽかった

CalculiXとAIエージェントを使って、2Dプレス成形解析に挑戦しました。

先に結論を書くと、解析はうまくいきませんでした。  
鋼板を金型で押し込み、下死点での応力・ひずみを見て、さらに型から取り出した後のスプリングバック形状を得る。言葉で書くとシンプルですが、普通に静的構造解析で計算すると大変形、弾塑性、接触、摩擦、除荷が絡むため、かなり難しい非線形解析です。  
さらにソルバーがOSSのCalculiXであるため接触解析には課題があると最初から予想しており、解析がうまくいかないのは想定の範囲内でした。

ただダメだとわかるまでのAIエージェントの挙動が興味深いものだったので、動作状況を残しておきます。

## 解析計画を作る

まず解析計画を作ってもらいます。

プロンプト

```
鋼板をプレス成形したときの応力・ひずみと、成形後のスプリングバックしたときの形状をみたい。解析計画を作成して。
ジオメトリデータは projects\Stamping\geometry\assy.step です。Plateというパーツが鋼板でDieTop（上型）とDieBottom（下型）は金型です。金型は剛体として扱って構いません。

下死点は上型が-Y方向に20mm移動したところです。
```

![](https://static.zenn.studio/user-upload/e99a43caccc1-20260614.png)

いくつか不明点が残っているようなので解説してもらいます。

Codexの回答：

> 不明点は、解析の「細部」ではなく、結果の意味を左右する条件です。特にスプリングバックはこれらに強く依存します。
>
> **1. 2D plane strainでよいか**
>
> 今回のSTEPは、見た限りZ方向の幅を持つ3D板ではなく、XY断面形状として定義されています。そのため初期案では「幅方向に十分長い板を、中央断面で見る」2D plane strain解析にしています。
>
> この仮定が妥当なのは、板幅が成形断面寸法に対して十分広く、幅方向端部の影響を無視できる場合です。  
> 逆に、実際の板幅が狭い、幅方向のそり・ねじれ・端部拘束を見たい、実部品形状としてのスプリングバックを評価したい場合は、3Dシェルまたは3Dソリッド解析が必要です。
>
> **2. 板端条件**
>
> 板の左右端が自由に引き込まれるのか、把持されるのか、ブランクホルダーで押さえられるのかで、ひずみ分布と残留応力が大きく変わります。
>
> 初期計画では、板端は基本自由、剛体運動を防ぐ最小拘束だけにしています。これは「金型で押して自然に曲がる傾向を見る」診断モデルです。  
> 実際に板押さえがある場合は、押さえ力、押さえ位置、摩擦、または引込み抵抗を追加しないと、実機に近い応力・ひずみにはなりません。
>
> **3. 摩擦係数**
>
> 金型と鋼板の摩擦は、材料がどれだけ滑るかを決めます。滑りやすいと板は金型になじみやすく、摩擦が高いと引張・圧縮ひずみや残留応力が変わります。
>
> 未指定なら初回は無摩擦で解くのが安定です。ただし実用評価では、例えば `μ=0.05`、`0.10`、`0.15` のような感度解析を行うべきです。スプリングバック量が摩擦で大きく変わるなら、摩擦条件を詰めないと結論は出せません。
>
> **4. スプリングバックの評価状態**
>
> 「成型後のスプリングバック形状」といっても、少なくとも2通りあります。
>
> 1つ目は、上型・下型から完全に取り出した自由状態の形状です。これは部品単体の最終形状に近い評価です。  
> 2つ目は、上型だけを離して下型上に残した状態です。これは金型内での戻りや、取り出し前の状態を見る評価です。
>
> 初期計画では、金型を退避させた自由スプリングバック形状を第一案にしています。最終製品形状を見たいならこの解釈でよいです。
>
> **5. 成形方向と初期位置**
>
> ユーザー指定では「上型が-Y方向に20 mm移動したところが下死点」です。STEP座標から見るとこの解釈で進められますが、念のため、CAD上の+Y/-Yと実際の上下方向が一致しているかが重要です。
>
> もしSTEP上で上型・下型の位置関係が逆、または座標の上下定義が違う場合、接触の向きや移動方向が逆になります。

内容を理解したので、次のように回答しておきます。

プロンプト

```
1: 板幅は整形断面に対して十分広いとみなしてよいです。
2: 仮押さえはありません。
3: 摩擦係数は0.2としてください。
4: 型から取り出した状態の形状を見たいです。
5: CAD上の+Y/-Yと実際の上下方向は一致しているはずです。
```

### 解析1回目ー解析中のCodex応答（抜粋）

作業途中のCodexの応答を見てみると、いくつか興味深い点があります。

> 接触は計算自体は進みましたが、ステップ1の約4.7%で接触・摩擦の非線形が強くなり、最小増分まで落ちて停止しました。解析時間を伸ばすより、まず接触ペナルティと摩擦のstick剛性を下げて収束性を優先します。摩擦係数0.2は維持します。
>
> ペナルティ接触を柔らかくした入力を再生成して再実行します。これは診断ケースなので、接触貫通量は結果レビューで必ず制限事項として確認します。

予想通り収束性が悪いので、ペナルティ法の接触解析の定石である接触剛性を下げることを試してますね。手っ取り早く試すのには良い方法です。

> 接触面を絞っても、成形途中の接触切替で静解析が同じ領域で破綻します。これ以上接触設定だけで粘ると時間が伸びるため、計画からの逸脱として「接触診断入力は残しつつ、短時間で結果を出すキネマティック成形近似ケース」を追加します。  
> 接触なしのキネマティック近似では、下型の成形面プロファイルへ板を強制的に沿わせ、その後拘束を外して弾性回復させます。摩擦接触の定量性はありませんが、短時間で応力・塑性ひずみ・スプリングバック形状を確認できます。

解析の収束性が難しいので、接触解析をあきらめて、いったん強制変位で動かして開放するという近似ケースを作成しています。定量評価には使いにくい方法ですが、スプリングバック形状を簡易的にみるという意味ではなるほどと思うところもあります。

> 近似モデルは成形ステップは完了しましたが、拘束解放ステップで急激なスプリングバックが数値的に不安定になり停止しました。ここで無理に長時間回すのは避け、現時点の成果と失敗理由を記録し、次に必要な安定化方針を明確にします。

ところが近似モデルもうまくいかなかったので、いったん作業を止めています。CAEで一番危ないのは、収束した風の怪しい結果をそれっぽく見せることなので、むしろ途中で止めているのは評価できます。

## 解析を収束させる方法を聞いてみる

解析がうまくいかないのは想定の範囲内だったので、どうやったらうまくいくのか聞いてみます。

プロンプト

```
接触成型モデルを収束させ、自由スプリングバック形状を得るにはどうすればよいか？
```

Codexの回答：

> 接触成形モデルを収束させるには、いきなり「摩擦あり・20 mm成形・自由解放」まで解かず、解析を段階化するのが現実的です。今の失敗原因は主に、接触のオンオフと摩擦滑りが同時に強く切り替わり、さらに解放時に `*BOUNDARY, OP=NEW` で拘束を急に外して剛体運動に近い不安定状態を作っていることです。
>
> **推奨手順**
>
> 1. **接触前処理を先に確認する**
>
> * 初期ギャップ、法線方向、接触面の範囲を図化する。
> * DieTop/DieBottom の接触面は、成形に使う曲面だけに限定する。
> * 角・端部・垂直壁など、板が触れてほしくない面は接触面から外す。
> * 下死点までの各中間位置、例えば 0, 5, 10, 15, 20 mm で、工具と板の相対位置を確認する。
>
> 2. **まず摩擦なしで成形を収束させる**
>
> * `mu=0.2` は最終条件だが、最初の収束確認では摩擦なし、または `mu=0.02 -> 0.05 -> 0.1 -> 0.2` の順に段階化する。
> * 摩擦あり接触は、接触状態が安定してから入れる。
> * 摩擦なしで下死点まで行けない場合、摩擦係数を調整しても根本解決しない。
>
> 3. **20 mm移動を複数ステップに分ける**
>
> * 例:
>   + Step 1: 初期接触確立、DieTop -1 mm
>   + Step 2: -5 mm
>   + Step 3: -10 mm
>   + Step 4: -15 mm
>   + Step 5: -20 mm
> * 各ステップで十分小さい初期増分を使う。
> * 失敗している step time 0.256 付近は、接触状態が大きく変わる箇所なので、そこだけ増分をさらに細かくする。
>
> 4. **板の剛体運動だけを弱く抑える**
>
> * 仮押さえはないので、板端を物理的に固定してはいけない。
> * ただし数値的な剛体運動防止として、最小限の `UX` 拘束、または非常に弱いばねを使う。
> * その拘束反力が成形反力に対して無視できることを確認する。
> * 強い端部拘束を入れると、スプリングバック形状が不自然になる。
>
> 5. **メッシュを三角形CPE3から改善する**
>
> * CPE3は曲げ・接触・塑性曲げには硬く出やすく、収束にも不利です。
> * 可能なら CPE4/CPE8 系の四角形主体にする。
> * 板厚1 mmなら厚さ方向に最低4要素、できれば6から8要素。
> * 工具R部、接触開始部、曲げ戻し部は局所的に細かくする。
> * まず粗い四角形メッシュで収束ルートを作り、その後だけ細分化する。
>
> 6. **工具は剛体相当でも、`*RIGID BODY` は避ける**
>
> * すでに `*RIGID BODY` と接触MPCの相性で警告が出ているため、当面は工具節点に直接変位を与える方式がよいです。
> * 工具自体の応力は評価しないので、工具を非常に剛な弾性体または prescribed node motion として扱う。
> * 工具節点を一括変位させ、回転自由度や剛体参照点MPCを接触と競合させない。
>
> 7. **スプリングバックは一括解放しない**
>
> * 今の `*BOUNDARY, OP=NEW` による急解除は不安定です。
> * 下死点保持後、次のように段階解除するのがよいです。
>   + Step A: DieTop変位を -20 mm で保持し、接触を落ち着かせる
>   + Step B: DieTopを -20 -> -15 mm に戻す
>   + Step C: -15 -> -10 mm
>   + Step D: -10 -> -5 mm
>   + Step E: 工具を十分離す
>   + Step F: 数値安定化用の最小拘束だけ残して自由形状を得る
> * 接触を急に無効化するより、工具を幾何的に離して自然に接触を外す方が安定します。
>
> 8. **必要なら準静的動解析へ切り替える**
>
> * 静的接触でどうしても接触切替が激しい場合は、準静的な動解析として解く方が安定する可能性があります。
> * ただし慣性が結果を支配しないよう、運動エネルギーが内部エネルギーに対して十分小さいことを確認する必要があります。
> * これは最終手段寄りですが、プレス成形の大変形接触では有効な逃げ道です。
>
> **このケースでの最短ルート**
>
> まずは次の順で進めるのがよいです。
>
> 1. 接触モデルを摩擦なし、粗い四角形主体メッシュで下死点まで収束させる。
> 2. 同じモデルで工具を段階的に戻し、自由スプリングバック形状まで収束させる。
> 3. 摩擦係数を `0.05 -> 0.1 -> 0.2` と上げる。
> 4. 収束後にメッシュを細かくし、下死点応力・塑性ひずみ・スプリングバック量のメッシュ依存性を見る。
>
> 現時点のモデルから一番効く修正は、`20 mm一括成形` と `OP=NEW一括解放` をやめることです。接触成立、成形、保持、工具戻し、自由化を別ステップに分けるのが、自由スプリングバック形状を得るための本筋です。

6はソルバー依存のところがあるかと思います。8は好みの分かれるところです。それ以外は、たぶん自分だったらこう考えるだろうなという内容で、CAE技術者っぽい対策をあげてくれたといえます。

### 解析2回目ー解析中のCodex応答（抜粋）

上記の対策を実施してもらいます。  
途中、解析モデルを見ると、いつの間にか上型が板にちょうど接する位置まで移動していることに気付きました。  
実はこれがこの手の解析の定石です。上型が板に接触するまでは空打ちになってしまうのと、解析を安定させるための工夫です。最初わざと上型を離しておいたのですが、それに気づいて修正したようです。  
![](https://static.zenn.studio/user-upload/597df88f08e0-20260614.png)

解析を試してもらいましたが、うまくいかず。

Codexの作業途中の応答：

> 接触面分離後も、-7 mm付近の同じ段階で止まりました。出力は軽くなったので、失敗位置の確認と、次の対策として「端部拘束の影響」か「端部ノード接触」を切り分けます。
>
> 収束ログの支配ノードが板端・角ノードに出ています。板幅は十分広い前提なので、端部は実成形断面の評価対象外として、接触候補から端部数mmを除外します。これで端部角が金型接触に巻き込まれる数値不安定を避けます。

偽接触がありそうなことを指摘してますね。これもわざとそういうモデル化をしておいたのですが気づいてくれました。

> 静的接触では、-7 mm付近で接触切替により同じ不安定が残ります。次の有効な逃げ道として、同じ段階ルートを準静的な陰解法動解析に切り替えます。密度を追加し、慣性で接触切替を通過できるかを確認します。
>
> 準静的動解析は初期接触の衝撃判定で即座にカットバック過多になり、今回の設定では静解析より悪化しました。ここまでで「静解析は-7 mm付近まで、動解析は初期接触で不可」という切り分けができたので、結果を記録して次の実務的な打ち手を明確にします。

動解析もうまくいかなかったので、ここでいったんまとめています。

Codex回答：

> これは厳密なFEMスプリングバック解析ではなく、**形状比較用の近似計算**です。
>
> やっていることは大きく3段階です。
>
> 1. **下死点形状を作る**
>
> * `DieBottom` の成形プロファイルを抽出します。
> * 板の中心線をそのプロファイルへ写し込みます。
> * これを「下死点で型形状が転写された状態」とみなしています。
> * 接触圧や摩擦で押し込んだ結果ではありません。
>
> 2. **曲率から弾性回復を見積もる**
>
> * 下死点形状の中心線曲率を計算します。
> * SPCCの応力ひずみ曲線を使い、板厚方向の曲げひずみ分布から断面曲げモーメントを概算します。
> * その曲げモーメントが除荷時に戻る量として、曲率を少し戻します。
> * これが「スプリングバック後の推定曲率」です。
>
> 3. **形状を再構成して比較する**
>
> * 戻した曲率から中心線形状を再構成します。
> * 自由スプリングバックでは剛体移動・剛体回転は任意なので、下死点形状と最小二乗で位置合わせします。
> * そのうえで、下死点形状との差をCSVと図にしています。
>
> つまり、今回のアウトプットは、
>
> > 型に沿った形を作る  
> > その曲げ曲率が弾性分だけ戻ると仮定する  
> > 戻った後の中心線との差を見る
>
> という計算です。
>
> **できていること**
>
> * 下死点形状とスプリングバック後形状の傾向比較
> * どのあたりで戻り量が大きそうかの確認
> * 最大差・平均差の概算
>
> **できていないこと**
>
> * 接触成型の収束解
> * 摩擦係数0.2の影響評価
> * 実際の残留応力分布
> * 板厚方向の正確な塑性履歴
> * 型抜き後の厳密な自由スプリングバック形状
>
> なので、これは「CAE解析結果」というより、**型転写形状からの一次的なスプリングバック見積もり**です。実設計判断に使うなら、次はこの形状差を目安にして、接触成型を最後まで収束させるモデルへ戻す必要があります。

プロンプト

```
形状差を目安にして、接触成型を最後まで収束させるモデルへ戻すとはどういうこと？
```

> 「近似で出た形状差を、次の本解析モデルのデバッグ指標として使う」という意味です。
>
> 今の近似結果は、例えば「端部やR部で数 mm 程度戻りそう」という形状傾向だけを教えてくれます。これを使って、接触解析側では次のように戻します。
>
> 1. **まず接触解析を -20 mm まで通す**
>
> * 現在の最良接触モデルは -18 mm付近まで進みました。
> * 近似結果から、下死点で重要なのは中央の底部と左右R部、端部の形状差です。
> * だから次は -18 ～ -20 mm だけに集中して、接触条件・ステップ・メッシュを調整します。
>
> 2. **下死点形状を近似形状と比較する**
>
> * 接触解析で -20 mmまで到達したら、その板形状を今回の「型転写形状」と重ねます。
> * 大きく違うなら、接触がまだ正しく型に沿っていない、または接触範囲・剛性・メッシュが不適切という判断になります。
> * 近似形状は正解ではありませんが、少なくとも「型に転写されたらこの程度の形になるはず」という基準になります。
>
> 3. **自由スプリングバック解析を実行する**
>
> * -20 mmの接触成型状態から、上型・下型を段階的に退避させます。
> * その後、剛体移動防止だけを残して自由形状を得ます。
> * このとき得た形状差を、今回の近似スプリングバック量と比べます。
>
> 4. **結果の妥当性を見る**
>
> * 接触解析のスプリングバックが近似結果と同じ桁、同じ傾向なら、まず形状としては妥当そうと判断できます。
> * 逆に、近似では最大3 mm程度なのに接触解析で20 mm戻る、あるいはほぼ0 mmしか戻らないなら、接触解除、拘束条件、材料モデル、剛体モード処理のどこかを疑います。
>
> 要するに、近似形状差は「最終答え」ではなく、接触解析を最後まで通したときに結果が変な方向へ行っていないかを見るための物差しです。
>
> 今回のケースなら、次にやるべき本解析側の具体策はこれです。
>
> * 現在の最良モデル、つまり `1 mm構造化CPE4板 + 要素辺接触 + 摩擦なし` を再開点にする
> * -18 mmまでは既に通るので、-18 ～ -20 mmだけを細かく制御する
> * 金型R部の接触面メッシュを細かくする
> * 接触剛性は安易に下げず、`100` 付近を基準にする
> * -20 mm到達後、上型を段階退避、下型も段階退避
> * 得られた自由形状を今回の近似スプリングバック形状と比較する
>
> つまり「近似で形を見る」段階から、「接触・残留応力・型抜きを含む解析で同じような形状傾向が出るかを確認する」段階へ戻す、ということです。

つまりFEM解析ではないものの、「近似で出た形状差を、次の本解析モデルのデバッグ指標として使う」という割り切った結果とのことでした。

## まとめ

今回は、CalculiXによる2Dプレス成形解析としては成功しませんでしたが、AIエージェントの動きは興味深いものでした。

CAE AIエージェントは、難しい解析を魔法のように成功させるものではなく、手作業でも難しい解析はAIにとっても難しいです。ただし、失敗原因を切り分ける、次の打ち手を考える、作業ログを残す、という用途では有用性を感じました。  
なお、最後に得た形状差は厳密なFEM結果ではなく、本解析を再構築するときのデバッグ指標として扱うべきものです。

CAE技術者の代わりに正解を出すというより、CAE技術者の試行錯誤を加速し、記録し、整理する相棒としてAIを使う。今回の結果は、その可能性を感じさせるものでした。
