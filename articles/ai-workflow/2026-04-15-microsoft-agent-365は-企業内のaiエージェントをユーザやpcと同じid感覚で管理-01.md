AIエージェントは、Copilot Studio・SaaS・外部ベンダー・自社スクリプトなど、あらゆる場所で急速に増殖しています。

ただ多くの企業では誰が作ったか？ どのデータに触れているか？ 暴走したらどう止めるか・・などガバナンスとして把握できず、かってのShadow ITと同じ構造的リスクを抱えていないでしょうか。

Microsoft Agent 365は、この企業内で把握できないAIエージェントを可視化＆統制し、保護するためのコントロールプレーンです。

特に重要なのは、AIエージェントを人間ユーザーと同じID管理体系に統合できる点であり、企業のゼロトラスト戦略を人＋デバイスから人＋デバイス＋AIへ拡張する基盤となるでしょう。

今回は Agent 365の価値、社外エージェント管理の仕組み、Entra IDとの一貫性など、企業がAIエージェントを安全に使いこなす未来を描いてみたいと思います。

![](https://static.zenn.studio/user-upload/fcda6fe0acc8-20260415.png)

## AIエージェントをユーザーのやPCと同じように管理できる

Agent 365の最大の価値は、Entra ID(旧ActiveDirectory)を利用して、AIエージェントに Entra Agent IDを付与し、ユーザーと同じゼロトラスト管理を適用できる点にある。

これはMicrosoft公式でも「エージェントは人間と同様の権限・認証・ロール・コンプライアンス機能を持つ」と明記されており、従来ブラックボックスだったAIの行動を、企業内のユーザID＆PCと同じ粒度で追跡できる。

| 管理項目 | 人間ユーザー（Entra ID） | AIエージェント（Entra Agent ID） |
| --- | --- | --- |
| 固有ID | 〇 | 〇 |
| 認証（OAuth） | 〇 | 〇 |
| 最小権限アクセス | 〇 | 〇 |
| 条件付きアクセス | 〇 | 〇 |
| 監査ログ | 〇 | 〇（行動理由まで記録） |
| アクセスレビュー | 〇 | 〇 |

つまりAIが、いち社員やPCと同じ透明性を持つ管理が実現できます。

## 社外で作られたAIエージェントもノー改造で管理できる原理

Agent 365 の素晴らしいところは、Microsoft製だけでなくサードパーティ製・自社開発・OSSエージェントも管理対象である。

これは Agent 365がエージェントの内部ロジックではなく外側(ID・権限・行動)を管理する仕組を持っているいるからであり、Microsoftは「どの種類のエージェントもユーザーのように制御できる」と明言している。

具体的に管理可能になる条件はとてもシンプル  
・外部から識別できる（API/Webhook/MCP対応）  
・Agent 365 のレジストリに登録できる  
の2点だけです。

逆にいえば上記に当てはまらいAIエージェントは、統制管理配下に入れなくても企業ガバナンスとして影響ない、簡易な個人利用AIエージェントになるだろう。

| エージェント種別 | 改造の必要性 | 理由 |
| --- | --- | --- |
| SaaSのAI Bot（GitHub / Confluence など） | 不要 | MCPやAPIで外部から識別可能で、Agent 365がそのまま管理できる |
| Teams / Slack Bot | 不要 | Agent 365 が自動検出し、レジストリに登録できる |
| 自社スクリプト（Python / Node / RPA連携など） | ほぼ不要 | エンドポイントURLとメタデータ登録のみで管理対象にできる |
| 完全ローカルの閉域AI | 必要 | 外部から識別できず、ID付与や監査ができないため薄いラッパーが必要 |

既存のAIエージェントを改修せずに統合管理できることが、企業にとって最大の導入メリットとなります。

## 統合ガバナンス：レジストリ・可視化・制御

Agent 365は企業内のすべてのエージェントを一元管理するレジストリを提供し、Microsoft自身も50万超のエージェントを管理していること公表している。

レジストリは、エージェントの種類・作成者・権限・利用状況を一覧化し、シャドウAIを検出する、さらにDefender・Purview と統合し、データアクセスや異常行動をリアルタイムで監視できます。

| 機能 | 内容 |
| --- | --- |
| レジストリ | 全エージェント（Microsoft製・サードパーティ製・自社開発）の台帳化。シャドウAIも検出し登録可能。 |
| 可視化 | 行動ログ、データアクセス、エージェント間の関係性マップをリアルタイムで可視化。 |
| 制御 | 条件付きアクセス、最小権限、権限剥奪、エージェントの無効化などの統制機能を提供。 |
| 監査 | Unified Audit Log に統合され、AIエージェントの行動理由まで含めて監査可能。 |

企業は人・デバイス・AIを同じガバナンス体系で扱えるようになります。

## Entra ID と同じ感覚で扱えるエージェントID管理の価値

Entra Agent ID は、エージェントに対して「社員番号」を与えるようなものです、これによりAIエージェントは人間ユーザーと同じID体系に統合され、ゼロトラストの原則（最小権限・継続的評価・明示的検証）がそのまま適用されれます。

Microsoftは「エージェントは人間と同様の権限・認証・ロールを持つ」と明記しており、これは企業のセキュリティ部門にとって極めて説明しやすいでしょう。

| Entra ID が管理するもの | Agent 365 が管理するもの |
| --- | --- |
| 人間ユーザー（社員ID） | AIエージェント（Entra Agent ID） |
| PC / デバイス（Intune） | エージェントの行動・権限・実行環境 |
| 認証（OAuth / MFA） | 認証（OAuth、エージェント専用制御） |
| 認可（RBAC / 最小権限） | 認可（RBAC / 最小権限 / JIT権限） |
| 条件付きアクセス | 条件付きアクセス（リスクベース） |
| 監査ログ（Unified Audit Log） | 監査ログ（行動理由・意図レベルまで記録） |

## 導入コスト：月額$15の投資でエージェント乱立リスクを解消

Agent 365はスタンドアロンで月額$15/ユーザーと発表されている。

また Microsoft 365 E7(Frontier Suite)では、E5＋Copilot＋Agent 365＋Entra Suiteとして統合され、、$99/月で提供される。

E7は個別購入より約15%安く、AIエージェントを全社展開する企業にとって最もコスト効率が高くなる。

| ライセンス | 価格（月額） | 内容 |
| --- | --- | --- |
| Agent 365（単体） | $15 / ユーザー | AIエージェントのレジストリ・可視化・制御・監査を提供する管理基盤 |
| Microsoft 365 E7（Frontier Suite） | $99 / ユーザー | E5 + Copilot + Agent 365 + Entra Suite を統合した最上位プラン |
| Copilot for Microsoft 365 | 別途 | Copilot本体。Agent 365と組み合わせることでエージェント運用が安全に拡張 |

管理されていないAIエージェント触れるリスクを考えれば、$15は極めて小さな投資であると筆者は思う。

＊筆者は決してMicrosoftの営業ではありません(^^)
