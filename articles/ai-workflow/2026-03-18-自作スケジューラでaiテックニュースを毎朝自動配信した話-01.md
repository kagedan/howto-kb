# 自作スケジューラでAIテックニュースを毎朝自動配信した話

## はじめに

[前回の記事](https://zenn.dev/acropapa330/articles/zenn_article2)では、Telegram Bot から Google Drive の PPTX にスピーカーノートを自動注入する機能を実装しました。

今回は、**毎朝 6 時に AI テックニュースを自動収集して Telegram に配信する仕組み**を構築しました。

実装した機能は 2 つです。

1. **自作スケジューラ**（`scheduler.py`）— cron 式でジョブを管理するバックグラウンドプロセス
2. **ニュースパイプライン**（`tech_news_pipeline.py`）— Brave Search API でニュースを収集して Claude が YouTube スクリプトを生成

---

## 自作スケジューラの設計

### なぜ自作したのか

タスクスケジューラでも定期実行はできますが、以下の理由で自作しました。

* Telegram から `/scheduler` コマンドで状態確認・起動できるようにしたかった
* `schedule.json` を編集するだけでジョブを追加・変更できるようにしたかった
* ホットリロード（ファイル変更を再起動なしに反映）が欲しかった

### schedule.json の構造

```
{
  "jobs": [
    {
      "id": "tech_news_pipeline",
      "enabled": true,
      "cron": "0 6 * * *",
      "command": "c:\\work\\ClaudeAgentBot\\venv\\Scripts\\python.exe c:\\work\\ClaudeAgentBot\\tech_news_pipeline.py",
      "description": "毎朝6時にテックニュースパイプラインを実行"
    },
    {
      "id": "ping",
      "enabled": true,
      "cron": "*/5 * * * *",
      "command": "echo scheduler heartbeat",
      "description": "5分ごとの死活確認"
    }
  ]
}
```

### cron 判定のバグ

`croniter` を使って cron 式を評価する際に詰まりました。

最初の実装では `get_prev()` を使っていましたが、これは常に「現在時刻の1つ前の実行時刻」を返すため、実行タイミングと一致しませんでした。

```
# NG: get_prev は base より前の時刻を返すので常に不一致になる
prev = croniter(cron_expr, base).get_prev(datetime)
return prev == base  # 常に False
```

`get_next()` を使う方式に変えて解決しました。

```
# OK: 1分前を起点にして get_next で次の実行時刻を取得
from datetime import timedelta

def _should_run(cron_expr, now):
    base = now.replace(second=0, microsecond=0)
    prev_base = base - timedelta(minutes=1)
    next_run = croniter(cron_expr, prev_base).get_next(datetime)
    return next_run.replace(second=0, microsecond=0) == base
```

### Windows の文字コード問題

`subprocess.run` でジョブを実行すると `UnicodeDecodeError: 'cp932'` が出ました。Windows のデフォルト文字コードが Shift-JIS（cp932）なのが原因です。

```
result = subprocess.run(
    command,
    shell=True,
    capture_output=True,
    text=True,
    encoding="utf-8",   # ← 追加
    errors="replace",   # ← 追加（読めない文字は ? に置換）
    timeout=600,
)
```

`encoding="utf-8"` と `errors="replace"` を追加するだけで解決しました。

## ニュースパイプラインの設計

### 全体の流れ

```
Brave Search API でニュース検索（5クエリ）
    ↓
重複チェック（MD5ハッシュで既配信を除外）
    ↓
各記事の本文を取得（urllib で簡易スクレイピング）
    ↓
Claude Agent SDK で YouTube スクリプト生成
    ↓
Markdown ファイルに保存
    ↓
Telegram に通知
```

### Brave Search API の使い方

```
import urllib.request, urllib.parse, json

def search_brave(query, count=5):
    params = {
        "q": query,
        "count": count,
        "freshness": "pd",  # 過去24時間
    }
    url = f"https://api.search.brave.com/res/v1/web/search?{urllib.parse.urlencode(params)}"
    headers = {
        "Accept": "application/json",
        "Accept-Encoding": "identity",
        "X-Subscription-Token": BRAVE_API_KEY,
    }
    req = urllib.request.Request(url, headers=headers)
    with urllib.request.urlopen(req, timeout=30) as resp:
        data = json.loads(resp.read().decode("utf-8"))
    return data.get("web", {}).get("results", [])
```

`freshness: "pd"` で過去 24 時間のニュースに絞れます。無料プランで月 2,000 リクエストまで使えます。

### Claude Agent SDK でスクリプト生成

Brave Search で収集したニュースを Claude に渡して YouTube スクリプトを生成します。

```
from claude_agent_sdk import query, ClaudeAgentOptions

async def generate_script(news_items):
    news_text = ""
    for i, item in enumerate(news_items[:7], 1):
        news_text += f"{i}. {item['title']}\n   {item['snippet']}\n   {item['url']}\n\n"

    result = ""
    async for message in query(
        prompt=f"以下のテックニュースからYouTube動画向けスクリプトを生成してください。\n\n{news_text}",
        options=ClaudeAgentOptions(allowed_tools=[]),
    ):
        if hasattr(message, "result") and message.result:
            result = message.result
    return result
```

Claude Agent SDK はサブスクリプション（Pro/Max）で API キー不要で使えます。

## Telegram からスケジューラを操作する

`bot.py` 起動時にスケジューラが動いていなければ自動起動します。

```
def ensure_scheduler_running():
    if is_scheduler_running():
        return
    start_scheduler()

ensure_scheduler_running()  # bot.py の起動時に呼ぶ
```

手動での操作は `/scheduler` コマンドで行えます。

| コマンド | 機能 |
| --- | --- |
| `/scheduler` | 状態確認・停止中なら起動 |
| `/status` | Bot 全体の状態（スケジューラの状態も表示） |

## まとめ

* cron 式の評価には `get_prev` ではなく `get_next` を使う
* Windows の `subprocess.run` には `encoding="utf-8", errors="replace"` を忘れずに
* Brave Search API の `freshness: "pd"` で最新ニュースに絞れる
* `schedule.json` を編集するだけでジョブを追加できるのが便利
