# UIより先に型とロジック
まずドメイン（型）と勝率計算ロジックだけを純粋関数で書き、テストで固める。 理由は単純で、ロジックがネットワークやDBを知らなければ、テストが書きやすく壊れにくいから

```
src/
  ggst_tracker.gleam          … gleam run 用のデモ
  ggst_tracker/
    character.gleam           … 全キャラを直和型で定義
    match.gleam               … 1試合の記録
    stats.gleam               … 勝率・マッチアップ集計（心臓部）
test/
  ggst_tracker_test.gleam     … stats のテスト
```

# 型設計
#### キャラを文字列ではなく型で管理
```gleam
pub type Character {
  Sol
  Ky
  May
  // ... 
}
```
`"Sol"` のような文字列で扱うと typo で `"sol"` と書いてもコンパイルが通ってしまう。

直和型にすれば**存在しないキャラはそもそも書けない**。
さらに表示名へ変換する関数を `case` で書くと
```gleam
pub fn to_string(character: Character) -> String {
  case character {
    Sol -> "Sol Badguy"
    Ky -> "Ky Kiske"
    // ...
  }
  ```
  追加キャラを `Character` に足したのに `to_string` に足し忘れると、
**コンパイラが「網羅できてない」と教えてくれる**
キャラは継続的に増えるので、追加漏れを見張らずに済む。

#### 勝敗もBoolにしない
```gleam
pub type MatchResult {
  Win
  Loss
}
```
`true/false` だと「どっちが勝ち？」が読む人に伝わらない。専用型なら意味が明確で、取り違えが起きづらいかと

#### 試合の記録
```gleam
pub type Match {
  Match(
    id: Int,
    my_character: Character,
    opponent: Character,
    result: MatchResult,
    note: Option(String),  // 試合メモ（任意）
  )
}
```

メモはあればなので `Option(String)`。null ではなく `Option` で**無いことを型で表す**。

# ロジック
#### 勝率と一番苦手なキャラ
集計の最小単位として勝率の型を作り、ゼロ除算を型で回避する
```gleam
pub type WinRate {
  WinRate(wins: Int, losses: Int)
}

pub fn percentage(rate: WinRate) -> Float {
  case rate.wins + rate.losses {
    0 -> 0.0  // 0試合なら0%（ゼロ除算しない）
    n -> int.to_float(rate.wins) /. int.to_float(n) *. 100.0
  }
}
```

相手キャラごとの勝率は `Dict` に畳み込む
```gleam
pub fn by_opponent(matches: List(Match)) -> Dict(Character, WinRate) {
  list.fold(matches, dict.new(), fn(acc, m) {
    let current = case dict.get(acc, m.opponent) {
      Ok(rate) -> rate
      Error(_) -> empty()
    }
    dict.insert(acc, m.opponent, add(current, m.result))
  })
}
```

そして一番苦手な相手を返す関数
```gleam
|> list.fold(None, fn(worst, candidate) {
  case worst {
    Some(current) ->
      case percentage(candidate.1) <. percentage(current.1) { ... }
    // ...
  }
})
```

上記だとコンパイルエラーを起こすので次のようにアキュムレータに型注釈を追加
```gleam
let initial: Option(#(Character, WinRate)) = None
// ...
|> list.fold(initial, fn(worst, candidate) { ... })
```
:::note info
 強い型推論を持つ言語でも、空のコレクション/None起点だと注釈が要る
:::
