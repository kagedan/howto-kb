## 背景：ナナカファームに農作業小屋が必要だった

こんにちは、熊本で米とクレソンを栽培しているナナカファームです。  
以前から農機具・資材を収納する専用の農作業小屋が欲しいと思っていました。

参考にしたのが学研ムック DIYシリーズ **『屋根付きウッドデッキの作り方』**（Gakken、2014年）。このムックの三面図を見て、「片流れ屋根、南高・北低、木造軸組」という基本設計が頭に浮かびました。

* 目標：**木造軸組・片流れ屋根・30㎡（6000×5000mm）**
* ただし建設前の練習として\*\*縮小版（約3畳 1820×2730mm）\*\*も欲しい
* 木造が目標だが、まず**単管パイプ版**で経験を積みたい

「この4種類を全部設計図にしたい」——そう思ったとき、AI × Blender MCPという選択肢が頭に浮かびました。

## 技術的な工夫と教訓

技術的な工夫と教訓

### ① `transform_apply` を使わないパネル配置

最大のハマりポイントでした。Blenderで壁パネルを作ろうとすると：

```
# ❌ ダメなパターン：壁が空中に散らばる
bpy.ops.mesh.primitive_cube_add(size=1, location=(cx, cy, cz))
obj.scale = (sx, sy, sz)
bpy.ops.object.transform_apply(scale=True)  # ← これで座標がリセットされる
obj.location = (cx, cy, cz)  # 2倍ずれる！
```

```
# ✅ 正しいパターン：transform_apply を使わない
bpy.ops.mesh.primitive_cube_add(size=1, location=(cx, cy, cz))
obj.scale = (sx, sy, sz)
# スケールをそのまま残す → ワールド座標 = location ± scale/2 で正確
```

### ② 2点間接続法によるパイプ配置

単管パイプを任意方向に正確に配置するための汎用関数：

```
def pipe_between(name, p1, p2, radius, mat, col):
    cx = (p1[0]+p2[0])/2
    cy = (p1[1]+p2[1])/2
    cz = (p1[2]+p2[2])/2
    dx, dy, dz = p2[0]-p1[0], p2[1]-p1[1], p2[2]-p1[2]
    L  = math.sqrt(dx**2 + dy**2 + dz**2)
    
    bpy.ops.mesh.primitive_cylinder_add(
        vertices=12, radius=radius, depth=L, location=(cx,cy,cz))
    obj = bpy.context.active_object
    obj.name = name
    
    # ベクトルの向きに合わせて回転
    z_ax  = mathutils.Vector((0,0,1))
    direc = mathutils.Vector((dx,dy,dz)).normalized()
    obj.rotation_euler = z_ax.rotation_difference(direc).to_euler()
    
    obj.data.materials.append(mat)
    add_to(obj, col)
```

これを使うことで、傾斜した桁・筋交い・スパイラル杭を全て同じ関数で配置できました。

### ③ 傾斜角の計算と符号

片流れ屋根で必ず引っかかるのが「傾斜方向の符号」：

```
slope = math.atan2(Hs - Hn, D)   # 南高・北低: 正の値

# ポリカ板の回転
polica.rotation_euler[0] = -slope  # ✅ 南が高い正しい方向
# polica.rotation_euler[0] = +slope  # ❌ 南が低くなってしまう
```

### ④ ワールド座標検証ルーティン

毎STEPの最後に必ず実行：

```
for obj_name in ["Wall_S_L", "Wall_N", "Wall_W", "Wall_E"]:
    obj = bpy.data.objects.get(obj_name)
    verts = [obj.matrix_world @ v.co for v in obj.data.vertices]
    xs = [round(v.x) for v in verts]
    ys = [round(v.y) for v in verts]
    zs = [round(v.z) for v in verts]
    print(f"{obj_name}: X={min(xs)}〜{max(xs)}, Y={min(ys)}〜{max(ys)}, Z={min(zs)}〜{max(zs)}")
```

「壁が空中に浮いている」「座標が2倍になっている」などの問題をこれで即発見できます。

* `stroke-dasharray` による軸線・引き出し線の使い分け
* `viewBox` と縮尺計算（1:20 / 1:50）による実寸対応

## 農研機構マニュアルとの連携

単管パイプ版は、農研機構（NARO）西日本農業研究センター「**建設足場資材利用園芸ハウス施工マニュアル**」（2017年）の仕様に準拠しています。

Claudeがウェブ検索でこのマニュアルの内容を取得し、以下の仕様をBlenderモデルに反映しました：

* スパイラル杭工法（φ48.6mm × 地中900mm埋設）
* 鋼板製十字金具（スーパークロス 48.6×48.6）
* 筋交い（ブレース）を各面2本以上配置
* 横垂木ピッチ @950mm（ポリカ波板に最適な間隔）

## 本番 vs 練習版のスケールダウン設計

本番（30㎡）と練習版（約3畳）で**部材断面はそのまま**にしてサイズだけ縮小したのがポイントです。

| 項目 | 本番 30㎡ | 練習版 約3畳 | 備考 |
| --- | --- | --- | --- |
| 幅 | 6,000mm | 1,820mm | 1間（半間ピッチ） |
| 奥行 | 5,000mm | 2,730mm | 半間×2スパン |
| 柱断面 | 105×105mm | **105×105mm** | 規格品そのまま |
| 桁断面 | 105×150mm | **105×150mm** | 規格品そのまま |
| 垂木断面 | 45×105mm | **45×105mm** | 規格品そのまま |
| 柱ピッチ | 1,500mm | **910mm**（半間） | 半間ピッチに変換 |

**「練習版で施工感覚を掴んでから本番へ」**というアプローチが実現でき、さらに**練習版の部材を本番でそのまま流用できる**という設計的合理性もあります。
