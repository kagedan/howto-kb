# CAE AIエージェントに3D形状を理解させるには？

CAEでは、境界条件をどの面に与えるかが非常に重要です。

* どの面を固定するか
* どの面に荷重を与えるか
* どの面を接触面とするか
* どの面に熱流束を与えるか

人間のCAE技術者なら、CAD画面を見ながら「この面ですね」と判断できます。  
しかし、AIエージェントに同じことをさせるには工夫が必要です。

## ジオメトリ準備（FreeCAD）

今回はFreeCADで作成した形状に対して、CAEエージェントに特定の面を認識させる方法を2通り試しました。

1つ目は、FreeCAD上で対象面に色を付け、その色情報を含めてSTEPとして書き出す方法です。

2つ目は、FreeCAD上で対象面を選択してハイライト表示し、そのスクリーンショットをCodexに貼り付けて「この面を対象にして」と指示する方法です。

人間に説明するなら、スクリーンショットで面をハイライトして「ここ」と示す方が自然です。しかし、AIエージェントに解析処理まで任せる場合、画像上の「ここ」をSTEPデータ内の具体的なFaceへ対応付ける必要があります。その対応付けは、人間が画面を見て判断するほど簡単ではありません。

### 荷重面に色を付ける

荷重面は色を付けて識別させることにしました。  
FreeCADでは面を選択して、メニューの「表示」＞「面ごとの外観」と進み、色を付けたい面を選択します。  
![](https://static.zenn.studio/user-upload/3aa353087078-20260522.png)  
面の外観はマテリアル名のプリセットから選ぶ形式のようなので、ここでは Aluminum を選んでおきます。  
![](https://static.zenn.studio/user-upload/fe9c2bcd410c-20260522.png)  
濃いグレー色になりました。  
![](https://static.zenn.studio/user-upload/8f1681331d31-20260522.png)

#### STEPファイルの中身を見てみる

このデータをSTEP形式で保存します。STEPデータでは面の色情報はどのように扱われているのでしょうか？保存したSTEPデータをテキストエディタで開いて確認してみます。

cover-Body.step（抜粋）

```
#3033 = FILL_AREA_STYLE('',(#3034));
#3034 = FILL_AREA_STYLE_COLOUR('',#3035);
#3035 = COLOUR_RGB('',0.678431390124,0.709803936702,0.741176476869);
#3036 = CURVE_STYLE('',#3037,POSITIVE_LENGTH_MEASURE(0.1),#3038);
#3037 = DRAUGHTING_PRE_DEFINED_CURVE_FONT('continuous');
#3038 = COLOUR_RGB('',9.803921802644E-02,9.803921802644E-02,
  9.803921802644E-02);
#3039 = OVER_RIDING_STYLED_ITEM('overriding color',(#3040),#1314,#3028);
#3040 = PRESENTATION_STYLE_ASSIGNMENT((#3041));
#3041 = SURFACE_STYLE_USAGE(.BOTH.,#3042);
#3042 = SURFACE_SIDE_STYLE('',(#3043));
#3043 = SURFACE_STYLE_FILL_AREA(#3044);
#3044 = FILL_AREA_STYLE('',(#3045));
#3045 = FILL_AREA_STYLE_COLOUR('',#3046);
#3046 = COLOUR_RGB('',0.300000011149,0.300000011149,0.300000011149);
```

確認するとSTEP内では、`COLOUR_RGB`で色が定義され、`PRESENTATION_STYLE_ASSIGNMENT`や`OVER_RIDING_STYLED_ITEM`を介して形状要素にスタイルが割り当てられているように見えます。少なくとも、単なるスクリーンショット上の色ではなく、STEPファイル内のテキスト情報として色情報が残っていることは確認できます。

この状態であれば **文字（コード）** ですので、LLMに「この面が荷重面ですよ」という情報を伝えられそうです。

### 固定面の指示は画像のみ

一方、固定面の指示は画像のみとしました。FreeCADで面をハイライト表示してスクリーンショットをとり、Codexに貼り付けて「水色の部分が固定面」と伝えるだけにします。この方法ではSTEPデータには何も情報が反映されていません。  
![](https://static.zenn.studio/user-upload/fb3554479d9a-20260522.png)

このジオメトリデータをもとに、Codexで解析させてみます。

### スクショは人間向け、面色付きSTEPはエージェント向け

面色付きSTEPを渡した場合はエージェントが色情報を手がかりに対象面を認識できましたが、ハイライト画像を貼り付けて指示した場合は失敗しました。画像上では対象面が分かるように見えても、それを3D形状上の特定の面に対応付けるところでうまくいきませんでした。

「画像認識が使えない」というより、画像上の2D情報をSTEP内の3Dトポロジー、つまり具体的なFaceに対応付けるという作業が難しかったようです。

一方で、色の指示はきちんとしたRGBコードではなく「濃いグレー」のようなざっくりした指示でも対応できるようです。（状況によりうまくいかないこともあるかもしれません）

今回の違いをまとめると、次のようになります。

| 方法 | 情報の持ち方 | 必要な対応付け | 結果 |
| --- | --- | --- | --- |
| スクリーンショット | 2D画像上のハイライト | 画像上の領域 → STEP内のFace | 不安定 |
| 面色付きSTEP | 3D Faceに紐づいた色情報 | 色 → STEP内のFace | 認識しやすい |

### 固定面を修正

やはり面に色情報をつけておくのが間違いないようなので、FreeCADで固定面にも色を付けてSTEPデータを修正し、再度解析計画を作ってもらいます。


## まとめ

LLMと3D CAEの世界の間には「橋渡し」が必要です。

CAEエージェントにCAE作業を任せる場合、人間と同じUI操作をそのまま真似させるだけでは不十分です。重要なのは、エージェントが読み取りやすい形で、設計意図や解析意図を形状データに埋め込むことです。

LLM系エージェントは自然言語やコードから意図を読み取るのは得意ですが、CAEで必要なのは、その意図を最終的にSTEP内のFace、メッシュの節点集合・要素集合、ソルバー入力の境界条件へ落とし込むことです。

今回はSTEPデータに色情報を埋め込むことで、3D形状の情報をLLMに理解させることができました。色を単なる見た目ではなく、CAEエージェントにとってのセマンティックな注釈として機能させるのがポイントだったといえます。
