## はじめに

こんにちは！ブロックチェーン×AI Agentで自律経済圏を創るKomlock labでエンジニアをしている小原（[@brto\_0224](https://x.com/brto_0224)）です。

この1年半ほど、AIコーディングツールを乗り換えまくっていました。

「これが最強や！」と言った翌月には別のツールを使っている。振り返ってみると、AIコーディングの世界全体が「補完」から「自律的に働くエージェント」へと動いていた1年半でした。

補完する → 指示を実行する → 外部ツールと繋がる → 常駐して動く → 人間なしで動き続ける——この1年半のツールの変遷は、そのままAIができることの拡張の歴史でした。

![くるくるくる](https://static.zenn.studio/user-upload/deployed-images/53fa2f2d3f431d434e6d5671.jpg?sha=51c4a9a344987d910d1cd2a9cde4ad752921030c)

### 想定読者

* AIコーディングツールの変化についていくのが大変だと感じているエンジニア
* Claude Code、Codex、MCPなどを聞いたことはあるが整理できていない人
* エージェント経済って最近聞くけどピンとこない、という人

## 1. GitHub Copilot

最初にAIコーディングで衝撃を受けたのはCopilotでした。

それまでは「コード補完ってよく使う関数をサジェストしてくれるやつでしょ」くらいの認識だったのが、Copilotは文脈を読んでその先を書いてくれる。「そうそう、それが書きたかった」という体験が初めてできたツールでした。

2023〜2024年頃はVSCode + Copilot、ChatGPT、Geminiがあれば十分な時代だったと思います。CursorやClineの存在は知っていたし少し触れてもいたものの、「VSCode + Copilotで別にええやろ」という気持ちが勝っていました。Cursorを本格的に使い始めたのは2025年秋ごろ、仕事で使う機会が出てきてからです。

この頃のAIはあくまで「提案する」存在でした。主役は人間で、AIはサポートでした。

<https://github.com/features/copilot>  
<https://www.cursor.com>  
<https://cline.bot>

## 2. Devinの登場

Devinが発表されたとき、かなり衝撃でした。

複数のリポジトリを横断して読み込み、自律的にPRを出して、レビューまでこなす。「これが実用化されたらエンジニアの仕事どうなるんや」と本当に怖くなったのを覚えています。自律的に動くエージェントという概念が初めてリアルになった瞬間でした。

AIが「提案する」から「実行する」に変わった瞬間でした。

<https://devin.ai>

## 3. AIが外部ツールと繋がり始めた頃

Devinのように「AIが実行する」方向が見えてきたあと、MCPのような仕組みも注目されるようになっていきました。

この時期はGitHub MCPやPlaywright MCPをよく使っていました。PRを自分で書かずにAIに書いてもらえるようになったのは感動でした。AIがコードを補完するだけでなく、外のツールに触れながら作業できるようになってきた感覚がありました。

<https://modelcontextprotocol.io>

## 4. Claude Code

MCPで外部と繋ぐ面白さを知ったあと、今度は「もっとローカルに、もっと直接、AIに作業を任せたい」という感覚が強くなってきました。そんな頃に出てきたのがClaude Codeでした。

最初は「ターミナルから直接操作して何がいいんや」と思っていました。

でも実際に使ってみると、IDEに依存せずプロジェクトのコンテキストをまるごと渡してタスクを任せられる感覚が好きになりました。「IDEより全然いいじゃん」と考えが変わったのを今でも覚えています。

さらにSkillsやHooksといった仕組みも出てきて、MarkdownファイルにAIへの指示を書くだけでエージェントの動きをカスタマイズできるようになりました。CLIとSkillsを組み合わせることで、MCPをそこまで使わなくても十分になった感覚があります。進化が止まらないなと感じた時期です。

<https://claude.ai/code>  
<https://code.claude.com/docs/ja/skills>

## 5. OpenAI Codex

OpenAIからエージェントが出るんだから熱いに決まってる、と思って色々使ってみました。

同じくCLIで動かせるエージェントで、精度も高い。ただ個人的には出力が少し硬くて、何を言っているかわかりづらいと感じることが多かったです。今は実装のレビューにたまに使う程度に落ち着いています。Claude CodeがいいかCodexがいいかという論争はずっと続いていますが、自分の中では自然と使い分けが決まった感じです。

<https://openai.com/codex>

## 6. OpenClaw

そして2026年2月、OpenClawに出会いました。

OpenClawは、LLMを使ったパーソナルAIエージェントをDiscord・Telegram・Slackなどのメッセージングプラットフォーム上で動かせるフレームワークです。

Claude CodeやCodexが「頼んだら動く」ツールだとすれば、OpenClawは「常時起動していて、自分から動いてくれる」エージェントです。Discordで話しかければ応答するし、cronで定期的にタスクを実行する。ヘルスチェックや情報収集もスケジュールで回せる。

「AIに仕事を頼む」から「AIが一緒に働いている」感覚に変わったのはこの辺りです。今もメインで使っています。

<https://docs.openclaw.ai>

## この1年半を振り返ると

* GitHub Copilot → AIが補完・提案する
* Devin → AIが自律的に実行する
* MCP → AIが外部ツールと繋がる
* Claude Code + Skills/Hooks → CLIベースのエージェント作業へ移行
* OpenAI Codex → Claude Codeとの使い分けを模索
* OpenClaw → AIが常駐して動き続ける

正直、ここに書いた以外にもWindsurf、Kiro、Figma MCP、Stitch、Gemini CLIなど、ちょろっと触ったものはたくさんあります。追いかけるだけで精一杯な1年半でした。

この流れを並べると、補完 → 指示実行 → ツール連携 → 常駐 → 経済活動、という能力の拡張が見えてきます。「AIが提案する」から「AIが動く」へ、そして「AIが常に動き続ける」へ。次のステップは「AIが自律的に経済活動をする」です。

ちなみに今は OpenClaw + Claude Code をメインに、コードを確認したいときは Cursor、レビューに Codex を挟む、という使い分けに落ち着いています。くるくるしながらも、なんとなく定まってきた感じがあります。

## AIエージェントが経済活動をする時代へ

AIが「提案する」から「実行する」へ、「常駐して動き続ける」へ。この流れで行くと、次は「AIが自律的に経済活動をする」あたりが来そうだなと思っています。

エージェントがサービスを使うために課金する、エージェントがエージェントに仕事を依頼する、エージェントが稼ぐ——そういうインフラが整いつつあるのを見ていると、そんなに遠い話でもない気がします。このあたりの構造は以前整理した記事があるので、興味ある方はぜひ。

<https://zenn.dev/komlock_lab/articles/agent-payments-stack-2026>

1年前にCopilotで補完していた自分は、まさかこんな場所に辿り着くとは思っていませんでした。
