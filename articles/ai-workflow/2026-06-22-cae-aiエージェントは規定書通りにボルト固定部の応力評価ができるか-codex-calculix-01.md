## CAE AIエージェントは、ボルト固定部の応力評価を規定書通りに実装できるか？

現場ではCAE解析において、メッシュ作成方法や評価方法が規定されていることがあります。CAE AIエージェントで解析を自動化できるとしても、それだけでは現場では使えません。実際に業務で利用するには、部署や製品ごとの規定として決まっているCAE手順を守る必要があります。解析結果が出せることと、その現場の決まりに従って評価できることは別物です。

そこで今回は、架空の応力評価規定書を用意し、それをAIエージェントに読ませて、ボルト固定部の応力評価を実行させてみました。規定に従って解析計画を立て、メッシュを作成し、CalculiXで解析し、指定された位置の応力を抽出するという一連の作業をAIエージェントがどこまで対応できるのかが気になるところです。

この記事では、Codex（GPT5.5）、CalculiXなどを使って、CAE AIエージェントがボルト固定部の応力評価をどこまで規定書通りに実行できたのかを紹介します。

結論から言うと、解析自体にはCalculiXの制約による妥協点もあった一方で、メッシュ生成については「おっと、そこまでできるのか」と思う内容でした。

## どうやってメッシュを作成したのか？

気になるのが、どうやってメッシュを作成したのかという点です。

今回使用したSTEPファイルでは、あえてボルト座面相当のフェース分割を入れていません。穴は空いていますが、φ19の座面領域を表す円形フェースや、その外側の分割線は存在しない状態です。

通常のCAE作業であれば、ここでCAD側に座面径の円を追加してフェースを分割するか、メッシュツール側で仮想的なパーティションを作成することを考えます。つまり、人間のCAE技術者は「まず形状を分割してからメッシュを切る」という発想になりやすいです。

ところが、今回AIエージェントはCADフェースを追加で分割するのではなく、応力評価手順書に必要なメッシュトポロジーをPythonスクリプトで直接生成していました。言い換えると、穴まわり専用の小さなメッシュジェネレータを作っていた、という方が実態に近いです。

### 要素を1枚ずつ張っていく地道な作業

スクリプトの中では、まず `Builder` という小さなクラスが定義されています。このクラスは、節点座標を登録し、4つの点から四角形要素を1個作る役割を持っています。

```
def quad(self, points):
    tag = self.next_element
    self.next_element += 1
    self.linear_quads[tag] = [self.node(point) for point in points]
    return tag
```

ここでやっていることはかなり素朴で、四角形要素の4隅の座標を渡すと、節点番号を割り当て、要素番号とコネクティビティを保存します。つまり、汎用メッシャーに任せるのではなく、節点座標を計算し、その4点を使って四角形要素を1枚ずつ張っていくという地道な作業です。

人間が手作業でこれをやることはまずありませんが、スクリプトであればむしろ自然です。特に、評価手順でメッシュの分割数や評価位置が厳密に決まっている場合、この方法は非常に相性が良いです。

### 穴まわりは極座標でO-gridを作る

穴まわりでは、穴中心を基準にした極座標を使っています。

```
def polar_xyz(radius, theta):
    return center_xyz \
        + radius * math.cos(theta) * es3 \
        + radius * math.sin(theta) * ev3
```

`radius` が穴中心からの半径、`theta` が周方向角度です。`es3` は板の長手方向、`ev3` は板幅方向を表しています。これにより、穴中心まわりの円周上の節点座標を機械的に作ることができます。

応力評価手順書では、穴全周を24分割することになっています。そのため、スクリプトでは周方向分割数を次のように設定しています。

さらに、穴半径から座面外半径までを径方向に分割します。

```
radial_values = np.linspace(
    hole_radius,
    seat_radius,
    radial_divisions + 1
)
```

今回の主メッシュでは、穴径φ11、座面相当径φ19として扱っています。したがって、穴半径は5.5 mm、座面外半径は9.5 mmです。径方向を4分割すると、1要素あたりの径方向幅は1.0 mmになります。

```
穴半径     : 5.5 mm
座面外半径 : 9.5 mm
径方向分割 : 4分割
径方向幅   : 1.0 mm
```

この条件で、周方向24分割、径方向4分割のリング状メッシュを作ります。

```
for i in range(ntheta):
    t0 = 2.0 * math.pi * i / ntheta
    t1 = 2.0 * math.pi * (i + 1) / ntheta

    for j in range(radial_divisions):
        builder.quad([
            polar_xyz(radial_values[j],     t0),
            polar_xyz(radial_values[j + 1], t0),
            polar_xyz(radial_values[j + 1], t1),
            polar_xyz(radial_values[j],     t1),
        ])
```

この部分が、穴エッジから座面外径までのO-gridです。

1つの四角形要素は、内側半径・外側半径・開始角度・終了角度の組み合わせで決まります。これを周方向と径方向に繰り返すことで、穴まわりに整列した四角形メッシュが作られます。

### 円形O-gridから正方形境界へ遷移させる

次に、座面外周の円形メッシュを、そのまま外側へ広げるのではなく、穴中心まわりの30 mm角の正方形境界へ遷移させています。

```
half = 15.0

radius = half / max(abs(math.cos(theta)), abs(math.sin(theta)))
```

この式は、角度 `theta` の方向に伸ばした直線が、中心から半幅15 mmの正方形境界と交わる位置を求めています。つまり、円形の座面外周から、正方形ブロックの外周へ向かうための距離を計算しています。

イメージとしては、次のようになります。

```
φ11穴
  ↓
φ19座面相当のO-grid
  ↓
30 mm角の正方形境界へ遷移
  ↓
外側は通常の矩形パッチで埋める
```

この遷移領域も、やはり四角形要素を1枚ずつ張って作っています。

```
builder.quad([
    polar_xyz(r00, t0),
    polar_xyz(r10, t0),
    polar_xyz(r11, t1),
    polar_xyz(r01, t1),
])
```

ここで `r00`, `r10`, `r11`, `r01` は、座面外周から正方形境界までの途中半径です。周方向の隣り合う2本の放射線と、遷移方向の2つの半径位置から、四角形を作っています。

この実装により、円形のO-gridと、外側の直交的な矩形メッシュを無理なくつなぐことができます。

### 正方形の外側は矩形パッチで埋める

正方形ブロックの外側は、通常の矩形パッチとして処理されています。

```
builder.rectangular_patch(u_values, v_values, p0_xyz, es3, ev3)
```

`rectangular_patch()` は、u方向とv方向の分割位置を受け取り、それらの格子点から四角形要素を連続的に作る関数です。

穴まわりだけは円形O-gridで丁寧に作り、それ以外の平坦部は矩形パッチで埋める。かなり合理的な構成です。

### 最後に4節点クワッドをS8Rへ変換する

ここまでの処理では、まず4隅だけを持つ四角形要素としてメッシュを作っています。その後、各辺の中点に節点を追加して、8節点の2次シェル要素に変換しています。

```
for a, b in zip(corners, corners[1:] + corners[:1]):
    edge = tuple(sorted((a, b)))
    edge_nodes[edge] = self.node(0.5 * (self.coords[a] + self.coords[b]))

result[eid] = corners + mids
```

これにより、最終的にはCalculiXのS8R要素として出力できるメッシュになります。

この処理によって、穴内周にも中間節点が作られます。穴全周24分割なので、角節点24個と中間節点24個、合計48節点が穴内周に存在することになります。

今回のRBE3相当結合では、この48節点をボルト穴内周の結合対象として使っています。一方で、座面領域の節点はRBEに含めず、シェルとして自由に変形できるようにしています。

### この実装で何がうれしいのか

このメッシュ作成方法では、評価手順書で必要になる領域を、メッシュ生成時点で明確に分離できているというのがメリットです。

```
穴内周エッジ節点
  → RBE3相当結合に使用する

穴縁から1要素内側のリング節点
  → 応力評価に使用する

座面領域の節点
  → RBEに含めず、自由変形させる
```

特に、応力評価位置が「穴縁から1要素内側」と指定されている場合、メッシュの切り方そのものが評価方法の一部になります。フリーメッシュで穴まわりの要素形状や節点位置が揺れると、評価値も揺れやすくなります。

今回のように、穴まわりを決定論的なO-gridとして作っておけば、評価リングの半径も節点数も明確になります。主メッシュでは、穴半径5.5 mm、座面外半径9.5 mm、径方向4分割なので、評価リングは半径6.5 mmになります。

```
評価半径 = 5.5 + (9.5 - 5.5) / 4 = 6.5 mm
```

この評価リングをスクリプト側で自動的にノードセット化できるため、解析後に人間が目視で節点を探す必要がありません。

### CAE AIエージェントらしいポイント

興味深いのは、AIエージェントがCADのフェース構成に引きずられなかった点です。

今回のSTEPファイルには、座面相当領域のフェース分割は入れていませんでした。それでもAIエージェントは、応力評価手順書に書かれた「φ19座面相当領域」「穴全周24分割」「径方向3〜5分割」「穴縁から1要素内側で評価」という条件を読み取り、CAD上には存在しない評価用の仮想領域をスクリプトで作成しました。

これは、単なる自動メッシュ生成というより、評価手順から逆算して専用メッシュ生成ロジックを組み立てたものです。

人間のCAE技術者であれば、まずCAD分割やメッシュツールの機能を考えるところです。しかし今回のAIエージェントは、節点と要素を直接生成することで、手順書が要求するメッシュトポロジーを再現しました。

このあたりは、CAE AIエージェントの可能性を感じた部分です。人間が思いつかないというより、人間なら手間が大きすぎて普通は選ばない実装を、AIエージェントがスクリプトとして淡々と作ってしまった、という印象です。

この方法は万能ではありませんが、「応力評価手順書に従って、ボルト穴まわりの評価応力を安定して抽出できるか」を確認するという目的に対して、AIなりの解釈と実装がされている点で可能性を感じる内容でした。

## まとめ

今回は、架空の応力評価規定書をCAE AIエージェントに読ませ、ボルト固定部の応力評価をどこまで規定書通りに実行できるかを試しました。

結果として、単にCalculiXを実行するだけでなく、規定に従った解析計画の作成、穴まわりのメッシュ方針、RBE3相当結合、評価リングでの応力抽出まで、一連の作業をかなり具体的に進めることができました。

特に印象的だったのは、STEPファイル側に座面相当領域のフェース分割を入れていなかったにもかかわらず、AIエージェントが規定書に書かれた条件を満たすために、穴まわり専用のメッシュ生成ロジックを実装していた点です。これは、AIエージェントが単にツールを操作したというより、評価手順から逆算して必要なモデル化を組み立てた例だと感じました。

今回の結果から感じたのは、CAE AIエージェントの価値は「解析を自動で流せること」だけではないということです。現場でありがちな様々な規定や手順に従って作業できる可能性は十分に感じられました。

人間によるレビューは必要ですが、規定書を前提にCAE作業を進められるAIエージェントは実務的な方向に発展していくのではないかと思います。

> # 【社内技術標準】シェル要素を用いたボルト固定部の応力評価規定
>
> ## 1. 目的
>
> 本規定は、シェル要素を用いた有限要素解析（FEA）において、ボルト締結部周辺に発生する局> 所応力を安定した精度で抽出し、構造の静的強度および疲労強度を統一の基準で評価することを> 目的とする。
>
> ---
>
> ## 2. メッシュモデリング仕様（必須要件）
>
> ボルト座面および穴周辺のメッシュは、応力勾配を正しく捉え、かつ要素の歪みによる異常高応> 力を防ぐため、以下の構成でマップドメッシュ（整列格子）を作成しなければならない。
>
> * **要素タイプ：** 原則として**1次または2次の四角形（クワッド）シェル要素**とする。三> 角形（トリア）要素の混入は禁止する。
> * **周方向の分割数：** 穴の全周（360^\circ）を **24分割**（15^\circ ピッチ）と> する。
> * **径方向の分割（座面領域）：** ボルト座面相当エリアを径方向に**3～5分割**とする。
> * **メッシュサイズ（目安）：** ボルト径 M に対して、座面領域の1要素の辺長が「板厚 t の 0.5倍〜1.0倍」の範囲に収まるよう調整する。
> * **ボルト穴の締結方法：** 中央にボルト軸を模したBeam要素を配置し、ボルト穴の内周節点とRBE3要素で放射状に結合する。座面領域の節点はRBEに含めず、自由に変形できる状態とすること
>
> ---
>
> ## 3. 応力抽出仕様
>
> 評価に使用する応力値は、ソルバーの生データではなく、以下の指定に従って抽出する。
>
> * **使用する応力：**
> * 延性材料（鋼・アルミ等）：**ミーゼス応力（von Mises Stress）**
> * 脆性材料（鋳鉄等）または溶接部近傍：**最大主応力（Max Principal Stress）**
> * **抽出位置（断面方向）：** シェル要素の **Top面（表面）および Bottom面（裏面）** > の両方を確認し、値が大きい方を採用する（Mid面の中立軸応力は使用不可）。
> * **抽出方法：** 要素中心（Element Center）値ではなく、節点共有化（アベレージング）> された節点応力値（Nodal Stress）**を採用する。ただし、特異点による異常値を除外するた> め、最急勾配となる「穴の最内周エッジの節点」は避け、**「座面領域内（エッジから1要素入> った位置）の最大値」をピーク値として採用する。
>
> ---
>
> ## 4. 強度評価基準（OK/NG判定）
>
> 抽出したピーク応力（\sigma\_{peak}）に対し、荷重の種類（静的/変動）に応じて以下の安> 全率を満たすこと。
>
> ### 4.1 静的強度評価（一発破壊の防止）
>
> 常用最大荷重（設計荷重）が作用した際の判定基準。
>
> \sigma\_{peak} \le \frac{\sigma\_y}{S\_s}
>
> * \sigma\_y ：材料の降伏応力（または0.2%耐力）
> * S\_s ：**静的基準安全率 ＝ 1.2** （局所的なピークのため、通常の構造評価より緩めに> 設定）
>
> ### 4.2 疲労強度評価（繰り返し荷重の防止）
>
> 振動や作動圧など、繰り返し荷重（変動応力振幅 \sigma\_a）が作用した際の判定基準。
>
> \sigma\_a \le \frac{\sigma\_w}{S\_f}
>
> * \sigma\_w ：材料の疲労限度（未加工材の疲労限度に、表面粗さや切欠き係数を考慮した> 値）
> * S\_f ：**疲労基準安全率 ＝ 1.5 〜 2.0**（製品の重要度、実験データとの相関により決> 定）
>
> ---
>
> ## 5. 免責・特記事項（例外規定）
>
> * 上記基準で安全率が不足（NG）となった場合でも、応力集中が極めて局所的であり、塑性崩壊> （全体がグニャリと曲がる）に至らないことが「応力線形化（膜応力評価）」または「弾塑性解> 析」によって証明された場合は、上長承認のもとで合格と判定できる。
