同じYAMLプロンプト・同じ入力情報を「Notion AI（Claude Opus4.6搭載）」と「素のClaude Opus4.6」に投げたら、出力が明確に違った。

モデルの性能差ではなく、モデルに渡されるコンテキストの違いが原因だった、という話。

## やったこと

架空のカンファレンス協賛提案書を生成するYAMLプロンプトを作り、同一の入力情報とセットで以下の2環境に投げた。

* **Notion AI**：ワークスペース内のページ・会話履歴・ユーザー設定にアクセスできる状態
* **素のClaude**：YAMLプロンプトとユーザー入力のみ

## 出力の比較

| 観点 | Notion AI | 素のClaude |
| --- | --- | --- |
| 情報量 | 多い |  |
| ワークスペース内の関連情報を引っ張ってくる | 入力された情報だけで完結 |  |
| フレーミング | 情報の網羅性で勝負 | 説得のロジック構成で勝負 |
| 費用の扱い | 協賛費のみ記載 | 別途コスト（ノベルティ・設営費）を含めた合計概算を算出 |
| リスクの質 | 網羅的だが粒度にバラつき | 実務的に刺さるリスクを厳選 |
| QAスコア | 7.3/10 | 6.8/10 |

## なぜ違いが出るのか

|  | Notion AI | 素のClaude |
| --- | --- | --- |
| 思考モデル | Claude Opus | Claude Opus（同一） |
| 渡される情報 | YAMLプロンプト + ワークスペース内のページ + ユーザー設定 + 会話履歴 | YAMLプロンプト + ユーザー入力のみ |
| システム指示 | ツール固有の振る舞い指示あり | デフォルトのシステムプロンプト |

比喩で言えば、「社内のSlackも議事録も全部読んでいる担当者」と「提案書の情報だけ渡された外部コンサル」が同じテーマで書いたら、違う切り口になるのと同じ構造。

## 具体的にどこで差が出たか

### サマリの書き方

* **Notion AI**：評価スコアや経営レイヤーの優先指定を根拠に押す。情報量で説得するスタイル
* **素のClaude**：「作る人全員に届く場」「多職種連携の親和性」といったキャッチーなフレーズで感情に訴える。サマリとしてのキャッチ力が高い

### 「実績がない」の扱い方

* **Notion AI**：近接する別イベントの実績で補完しようとする
* **素のClaude**：「当てる」より「計測する」ことに意義がある協賛、と再定義。実績なしの弱点を正当化するフレーミングとして秀逸

### 費用の出し方

* **Notion AI**：協賛費60万円のみ
* **素のClaude**：別途コスト（ノベルティ・設営費）を加算し、合計概算75〜80万円と算出。承認者にとって「本当にかかる金額」が分かる

## ここから言えること

Notion AIの強みは「ワークスペースの情報を活用できること」。素のClaudeの強みは「入力だけで完結する提案書を書く力が鍛えられること」。

どちらが優れているかではなく、用途によって使い分けるのが正解。

* ドラフトの骨格やフレーミングは素のClaudeに作らせる
* 社内コンテキストの肉付けはNotion AIに任せる
* 最終的に両方のいいとこ取りをマージするのが最も完成度が高い

## 補足：QAサマリ（自己採点）の仕組み

YAMLプロンプトに `rubric` セクションで評価基準を定義し、出力に自己採点を含める設計にした。

```
rubric:
  - name: 判断可能性
    weight: 30
    description: 読んだ人が「承認/却下」の判断を出せる情報が揃っているか
  - name: 根拠の具体性
    weight: 25
    description: 数値・実績・比較など具体的な根拠が含まれているか
  - name: 構造の明快さ
    weight: 25
    description: 見出しだけ追えば全体像が掴めるか
  - name: コスト対効果の提示
    weight: 20
    description: 投資額に対して何が得られるか明示されているか
```

これにより：

* 出力の弱点が事前に分かる。上司に持っていく前に「ここが弱い」が自分で確認できる
* 情報不足なのか、構造の問題なのかを切り分けられる
* 素のClaudeは「根拠の具体性が5/10だが、これは書き方の問題ではなく情報不足によるもの」と弱点の原因まで自己分析してくれた

rubricの設計は、対象タスクの「承認者が頭の中でやっている評価プロセス」を項目化するのがコツ。
