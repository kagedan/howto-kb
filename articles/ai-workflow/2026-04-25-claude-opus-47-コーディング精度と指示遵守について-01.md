## この記事で分かること
- Claude Opus 4.7でコーディング能力がどう変わったか
- 指示遵守（instruction following）の改善点と注意点
- 画像認識解像度が3.3倍になった意味
- 既存プロンプトの再調整が必要なケースと対処法
- 利用できるプラットフォームと料金の変化

## 背景
Anthropicは2026年4月16日、大規模言語モデルの最新版「Claude Opus 4.7」を一般公開した。前バージョンOpus 4.6が1Mトークンコンテキストを引っさげて登場してから数ヶ月、今回は「精度」と「従順さ」に焦点を当てたアップグレードだ。

コーディング系のベンチマークで顕著なスコア改善が見られ、画像認識の解像度も従来比約3.3倍に拡大。開発者から「別物になった」という声が上がっている。

## 解説

### コーディング能力の変化
Opus 4.7のコーディング評価スコアは、前モデルと比べて明確に向上している。

特に変わったのはこのあたりだ。

- 複雑なリファクタリングタスクの完了率が上昇
- エラーハンドリングの提案精度が改善
- 長大なコードベースを読んだ上での変更提案の一貫性が増した

「努力レベル（effort level）」と「タスク予算（task budgets）」という新機能も追加されており、トークン消費量と出力品質のバランスを開発者側で調整できるようになった。

### 指示遵守の厳密化
使い始めてすぐ気づくのは、プロンプトの解釈が以前より厳密になっている点だ。

Anthropicは「より正確に指示を解釈する」と表現しているが、裏を返せば既存プロンプトが意図通りに動かなくなるケースが出てくる。特に曖昧な表現や暗黙の前提に頼っていたプロンプトは要注意だ。

- 曖昧な指示 → モデルが「最も字義通り」の解釈を取る
- 複合的な指示 → 優先順位が明示されていないと意図と異なる動作をする

### 画像認識の改善
解像度が3.3倍になったことで、細かい図表や手書きのホワイトボード写真、スクリーンショットのUI要素の読み取り精度が大幅に向上した。コードレビューで設計図を添付するユースケースが一気に現実的になる。

## 実務への落とし込み

AI駆動開発の観点で、すぐ試せる手順はこれだ。

**1. 既存プロンプトの棚卸し**

    # チェックポイント
    # - 暗黙の前提を文字に起こす
    # - 「〜してください」→「〜してください。それ以外はしないでください」のように明示化
    # - Few-shotの例を更新して期待動作を再定義

**2. task budgets の活用（ベータ機能）**

    {
      "model": "claude-opus-4-7",
      "max_tokens": 16000,
      "task_budget": {
        "token_limit": 8000
      },
      "messages": [...]
    }

**3. 画像添付の活用**

設計ドキュメントや要件定義書の画像をそのまま添付し、「このUIを実装するコードを書いて」というプロンプトが実用レベルで動くようになった。試す価値がある。

## 注意点 / 限界 / 誤解されやすい点

- **レート制限の緩和は条件付き**：思考トークンが増えるため全プランでレート制限を緩和しているが、effort levelの調整やプロンプトの簡素化が必要になる場面もある
- **プロンプトの再調整は避けられない**：既存環境をそのまま移行すると動作が変わる可能性がある。ステージング環境での検証を挟むこと
- **「賢くなった」＝「何でも通じる」ではない**：指示が厳密に解釈されるため、雑なプロンプトは逆に扱いにくくなる
- **task budgetsはまだベータ**：本番環境への全面適用は様子見が賢明
