## **なぜ今、Codex/ClaudeCodeとXをつなぐと強いのか**

これまでXの運用は、ほとんどが手作業でした。

![](https://pbs.twimg.com/media/HMFLEV4aoAA3uvw.jpg)

ネタを考えて、文章を書いて、画面を開いて、貼り付けて、出す。出したあとは、伸びたかどうかを自分で見にいく。

ひとつひとつは小さな手間です。でも毎日積み重なると、地味に時間を食っていました。

ところが、ここが大きく変わりました。**Codex/ClaudeCode**のようなAIエージェントが、Xを直接さわれるようになったんです。

**Codex/ClaudeCode**というのは、指示を出すと自分で考えて作業を進めてくれる、開発用のAIです。文章を書くだけでなく、自分で道具を呼んで動くのが特徴です。

これがXとつながると、人間の代わりに投稿を作ったり、データを取ってきたり、反応を集めたりできます。

きっかけは、Xが**X MCP**という公式の接続口を出したことでした。

公式アカウントの言葉を借りると、これで「AIエージェントが、世界で最良のリアルタイム情報源に、設定なしでつながる」ようになった、とのこと。

これまでは、自分でコードを書いて橋を架ける必要がありました。その橋を、Xが用意してくれた。だから手間がぐっと減りました。

つまり、こういうことです。

書く・出す・分析する・返信する。これまでバラバラだった作業が、一本の流れにつながり始めました。

## **つないだ後にできることの全体像**

細かい話に入る前に、地図を持っておきましょう。

Xが出した接続口は、実は2種類あります。役割が違うので、ここで分けて押さえます。

![](https://pbs.twimg.com/media/HMFLYa-aYAAP20s.jpg)

- ひとつ目が**X API MCP**。Xを操作するための口です。投稿を出す、検索する、自分の成績を取る、記事の下書きを作る。Xに「やってほしいこと」を投げる窓口です。
- ふたつ目が**X Docs MCP**。Xの説明書を引くための口です。Xを外から動かすための取扱説明を、AIが自分で調べられます。コードを書くときに役立ちます。
- この2つはセットで使えます。説明書を引きながら、実際の操作も走らせる。AIが「調べて、動かす」を一人で完結できる形です。
操作する口の**X API MCP**には、たくさんの道具がそろっています。

公式によると、その数は**200**を超えます。投稿の取得や検索、トレンドの取得、ブックマークの管理、そして記事の下書き作成まで。
