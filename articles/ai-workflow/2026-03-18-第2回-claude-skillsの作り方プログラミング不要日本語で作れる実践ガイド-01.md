**サブタイトル：Anthropic公式の登録手順と、私が3日間ハマった落とし穴**

---

## はじめに——「できた」は「使える」ではなかった

正直に白状します。

私はこの記事を書くにあたって、まず自分でClaudeとSkillの作り方を試しました。  
SNS投稿文を自動生成してObsidianに保存するワークフローを、SkillとしてClaudeに覚えさせようとしたんです。

結果は惨敗でした。

「登録できました！」  
「ファイルに書き込みました！」  
——Claudeはそう何度も言ってくれました。  
だが翌日チャットを開いてみると、何も残っていない。  
何も動かない。  
この繰り返しを3日間、何十回ものやり取りの末に続けました。

あとでわかったのは、私が3つのものを混同していたということです。

1つ目は、**Claude.ai 上で使うcustom Skills**。   
2つ目は、**Claude Codeで使うローカルのskills**。   
3つ目は、**Claude DesktopやMCPの設定**。

この3つは見た目が似ていますが、導入方法も保存場所も役割もまったく違います。

この記事では、その混同をどうほどいたか、そして**Claude.aiでSkillを正しく作る方法**を、失敗談込みで整理していきます。

---

## そもそもClaude Skillsとは何か（第1回の復習）

第1回でも触れましたが、改めて整理します。

**Skill（スキル）とは、Claudeが特定のタスクを処理する際に、必要に応じて読み込む手順書や補助リソースのまとまりです。**

Anthropicの説明では、SkillsはInstructions・Scripts・Resourcesなどをまとめたフォルダで、特殊な作業を繰り返し実行しやすくするための仕組みとされています。

ポイントは3つあります。

* **Claudeが必要だと判断したときに読み込まれる**   
  毎回すべてを常時読み込むのではなく、関連する作業で必要になったときに使われます。
* **中身は人間が読める形で書ける**   
  中心になるのは SKILL.md というMarkdownファイルで、手順やルールを自然言語で記述できます。
* **一度追加すれば継続して使える**   
  Claude.aiのcustom Skillsは、設定画面からZIPをアップロードして追加すると、そのアカウントのSkills一覧に保存され、有効化して再利用できます。

「Claudeに毎回同じ説明をするのが面倒」  
「自分の仕事の型を再利用したい」  
——そういうときに役立つのがSkillです。

## まず整理したい——似ているけれど、別物である

![](https://assets.st-note.com/img/1773627248-0ExAaZMdhSlmY8VGHDgqoC1B.png?width=1200)

3つの『Skill』の違い

ここが一番混乱しやすい部分です。

### 1. Claude.aiのcustom Skills

ClaudeのWeb版・デスクトップ版で使うスキルです。   
**ZIPファイルをアップロードして登録する**のが正式な手順で、設定画面の  
カスタマイズ > スキル から追加します。

### 2. Claude Codeのskills

Claude Codeで使うローカルの仕組みです。   
こちらは ~/.claude/skills/ やプロジェクト配下の .claude/skills/ に SKILL.md を置いて使います。

### 3. Claude Desktop / MCPの設定

これはSkillそのものではありません。   
ClaudeとDifyのような外部ツールやサーバーをつなぐための設定です。  
MCPについては第4回で詳しく扱います。

私が3日ハマったのは、**この3つを全部同じものだと思っていた**からでした。

## Skillの構造 ——まずは SKILL.md が中心と覚える

Skillは複数ファイルを持てますが、最初に覚えることは単純です。

**必須なのは SKILL.md だけです。**

よくある構成例はこうなります。

```
my-skill/
├── SKILL.md         ← 必須。これだけでも動く
├── scripts/         ← 補助スクリプト（任意）
├── examples/        ← サンプル（任意）
├── template.md      ← テンプレート（任意）
└── reference.md     ← 参考資料（任意）
```

大事なのは、**フォルダ名そのものより、Claudeに読ませたい中核手順が SKILL.md にあること**です。

---

## SKILL.md の基本形 ——最重要なのは description

SKILL.md の冒頭には、YAMLフロントマターを書きます。

```
---
name: sns-post-generator
description: |
  SNS投稿文の作成を支援するスキル。
  X、Threads、LinkedIn向けの投稿文作成や叩き台作成が必要なときに使う。
---

# SNS投稿文生成スキル

## 役割
SNS投稿文の叩き台を作る。

## 手順
1. ユーザーが伝えたテーマを整理する
2. 配信先ごとに文体を切り替える
3. 複数案を提示する
```

この中で特に重要なのが description です。

Anthropicの公式でも、Claudeが自動でSkillを使うかどうかは、**そのSkillがいつ役立つかを description でどれだけ明確に書けているか**が重要だと案内されています。  
うまく使われない場合は description を見直すよう明示されています。

つまり、こう考えるとわかりやすいです。

この役割分担が曖昧だと、Skillは発動しにくくなります。

## 私が3日間ハマった罠 ——「チャット内で作れた」≠「正式に登録された」

![](https://assets.st-note.com/img/1773627295-oOrlgdeP41MVn7kiIamCShAq.png?width=1200)

チャットでの保存に関する誤解

ここからが本題の失敗談です。

私が最初にやっていたのは、Claudeとのチャットの中で /mnt/skills/user/ というフォルダに SKILL.md を書き込ませることでした。  
Claudeはチャット内でファイルを作成する機能を持っているので、実際にファイルは「作れた」ように見えました。

その場では、それらしく動きます。Claudeも「登録できました！」と言ってくれる。

だが翌日になると使えない。セッションをまたぐと残っていない。

**ここでようやく理解しました。  
チャット環境はセッションが終わるとリセットされます。  
その中で書いたファイルも一緒に消えてしまうのです。**

つまり、私の失敗の本質はこうでした。

* セッション内で一時的に存在していたファイル
* アカウントに保存されるcustom Skill

この2つを同じものだと思っていました。  
「できた」と思っていたのは、正確には「**その会話の流れの中では、できたように見えた**」に過ぎなかったのです。

**Claude.aiのcustom Skillsは、チャット内でファイルを書いただけでは正式登録になりません。  
正式な追加方法はZIPアップロードです。**

これは初心者ほどハマりやすい。  
なぜなら、Claudeがその場では実際に賢く振る舞うからです。  
でも正式登録は別工程、ここを取り違えなければ、あとはシンプルです。

## 本当に正しいSkillの登録方法

では、Claude.aiでcustom Skillsはどう登録すればいいのか。  
Anthropicの公式手順に沿って説明します。

### Step 1：フォルダを作り、SKILL.md を用意する

まずPC上でフォルダを作ります。

```
my-skill/
└── SKILL.md
```

テキストエディタで十分です。  
最初は SKILL.md だけでOKです。

### Step 2：フォルダごとZIPに圧縮する

![](https://assets.st-note.com/img/1773627348-CsbptPlDAzYowVcXSTeFqWhQ.png?width=1200)

正しいZIP階層

そのフォルダをそのままZIPにします。

```
my-skill.zip
```

ここで初心者がやりがちなミスが1つあります。  
圧縮の仕方によっては、解凍したときにこうなってしまうケースです。

```
my-skill/
└── my-skill/        ← 二重になってしまう
    └── SKILL.md
```

この二重階層だと意図通りに認識されない原因になります。  
**解凍したときにフォルダ直下に SKILL.md がある状態**を意識してください。

### Step 3：Claudeでアップロードする

![](https://assets.st-note.com/img/1773627425-z3Z5ahKF8IH40YNEtGvpgU2o.png?width=1200)

Claudeでアップロードする！

Claudeの設定画面から追加します。

1. Claudeを開く
2. カスタマイズ > スキル に進む
3. + からcustom skillを追加する
4. ZIPファイルをアップロードする

これだけです。  
アップロードが完了すると、そのアカウントでClaudeを使うどのデバイスからでも、そのSkillが使えるようになります。

## Skillの書き方——日本語で作る実践例

では実際に書いてみましょう。  
「SNS投稿文の叩き台を作るSkill」を例にします。

### ダメな例（発動しにくい）

```
---
name: sns-helper
description: SNS投稿を手伝うSkill
---
SNS投稿文を作成する。
```

これでも意味は通じますが、Claudeが「いつ使うべきか」を判断する材料としては弱すぎます。

### 良い例（確実に発動する）

```
---
name: sns-post-generator
description: |
  SNS投稿文の作成を支援するスキル。
  X、Threads、LinkedIn向けの投稿文案、下書き、言い換えが必要なときに使う。
  「SNS投稿文ネタ：」というフレーズが入力されたときも必ず使うこと。
  SNS投稿ネタ、投稿文作成、発信文の叩き台づくりに関連する依頼で役立つ。
---

# SNS投稿文生成スキル

## 役割
ユーザーが与えたテーマをもとに、SNS投稿文の下書きを作る。

## 手順
1. テーマの要点を1〜3個に整理する
2. 誰向けの発信かを推定または確認する
3. プラットフォームごとに文体を変えて案を作る

## 出力ルール

### X
- 短く、フックを強めにする
- 1投稿で読める密度にする
- 改行を使って読みやすくする

### Threads
- 少し会話調にする
- 体験や気づきを入れる
- 共感されやすい流れにする

### LinkedIn
- 学び・実務・再現性を重視する
- 専門性を少し強めに出す
- ビジネス読者向けにまとめる

## 出力形式
X・Threads・LinkedInごとに2案ずつ提示する。
```

ポイントは2つです。

**① descriptionに「いつ使うか」を具体的に列挙する**   
Claudeは発動すべきか迷います。  
迷わせないために、トリガーとなるフレーズのバリエーションを書いておきましょう。

**② 本文に「どうやるか」を具体的に書く**   
曖昧な指示より具体的なルールの方が品質が上がります。  
文字数、フォーマット、出力形式まで書き込むのがコツです。

## Anthropicの設計思想から学べること ——全部を詰め込まない

Skill設計で重要なのは、**全部を一枚岩で書かないこと**です。

Anthropicの公式ドキュメントでも、Skillは必要なときだけ読み込まれる仕組みなので、詳細な手順や参照資料は SKILL.md 本文や補助ファイルに分けるのが合理的とされています。

実務的には、次のように考えると整理しやすいです。

**description** ：いつ使うかを短く明確に書きます。

**SKILL.md 本文** ：実行手順・ルール・出力形式を書きます。

**補助ファイル** ：テンプレート・サンプル・長文の参照資料を置きます。

最初から大きな万能Skillを作ろうとすると、たいてい失敗します。  
**役割を絞った小さめのSkill**の方が作りやすく、使われやすいです。

## MCPとSkillの組み合わせ ——ここは別問題として切り分ける

もう一つ、私が混乱していたのがMCPとSkillの関係です。

ここははっきり書きます。**SkillとMCPは別物です。**

私自身、DifyのようなツールをMCP経由で連携しようとして、タイムアウトや設定が消えてしまうといった問題に遭遇しました。  
ただしこれらはSkillの作り方の問題ではなく、Claude DesktopとMCPの設定に関する別の話です。

このため、この記事では次のように整理します。

MCPとの連携・活用については第4回で詳しく扱う予定です。

## プロジェクト機能との使い分け

![](https://assets.st-note.com/img/1773627482-mql8Wo0C1vsGNQfJg7zpdIAD.png?width=1200)

プロジェクトとの使い分け

Skillに似た機能として、Claudeにはプロジェクト機能とカスタム指示があります。  
混乱しやすいので整理します。

**Skill**   
複数の場面で再利用したい業務の型に向いています。  
一度登録するとアカウント単位で保存され、どのチャットからでも使えます。

**プロジェクトの指示**   
その案件だけの前提や進め方を書いておく場所です。  
プロジェクト単位で保存されます。

**通常の会話**   
一回限りの依頼に使います。  
セッションが終わると消えます。

私の感覚での使い分けはこうです。

**Skillに向いているもの**

* 提案書の型
* SNS投稿文の型
* ヒアリング項目の型
* 分析レポートの型

**プロジェクト指示に向いているもの**

* この案件の背景
* このクライアントの事情
* 今回だけの制約条件

つまり、**横断的に再利用したい「仕事の型」はSkill、その案件だけの文脈はプロジェクト**と考えるとわかりやすいです。

## まとめ——Skill作成の正しいステップ

最後に、Claude.aiでcustom Skillsを作る最短ルートを整理します。

```
STEP 1：何をSkillにするか決める
         ↓
STEP 2：PCでフォルダを作り、SKILL.mdを書く
         ・descriptionに「いつ使うか」を書く
         ・本文に「どうやるか」を書く
         ↓
STEP 3：フォルダごとZIPに圧縮する（二重階層に注意）
         ↓
STEP 4：Claudeの Customize > Skills からアップロードする
         ↓
STEP 5：新しいチャットで実際に試す
         ↓
STEP 6：使われにくければdescriptionを見直す
```

一番やりがちなミスは、**チャットの中でファイルを書いただけで登録されたと思ってしまうこと**です。  
正式登録はあくまでZIPアップロードです。  
ここを取り違えなければ、Skill作成は思っているよりずっとシンプルです。

## 次回予告

第3回では、**今日からそのままコピペで使えるClaude Skills 20選**を公開します。

文章作成・業務整理・マーケティング・分析の4カテゴリに分けて、SKILL.mdのテンプレートごと紹介します。  
「どんなSkillを作ればいいかわからない」という方は、まずコピペして使ってみてください。

ただし、20個の選定基準は厳しく設けています。

* 繰り返し使える
* 型として安定している
* Skillにしないと毎回説明が必要になる

この3条件を満たすものだけを厳選しました。  
「Skillを作ってみたけど意味がなかった」という二度手間を避けるために、選定の根拠も含めて解説します。  
次の記事を読み終わったら、すぐに1つ作れる状態になっているはずです。
