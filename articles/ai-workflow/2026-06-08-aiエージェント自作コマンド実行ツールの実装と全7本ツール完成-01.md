## run\_command の実装

src/tools/exec\_tools.py

```
import subprocess
from pathlib import Path

TIMEOUT_SECONDS = 30

def run_command(command: str, cwd: str = ".") -> str:
    for dangerous in DANGEROUS_COMMANDS:
        if dangerous.lower() in command.lower():
            return f"エラー：危険なコマンドのため実行を拒否しました: {command}"

    cwd_path = Path(cwd).resolve()
    if not cwd_path.exists():
        return f"エラー：作業ディレクトリが見つかりません: {cwd}"

    try:
        result = subprocess.run(
            command,
            shell=True,
            cwd=str(cwd_path),
            capture_output=True,
            text=True,
            encoding="utf-8",
            errors="replace",
            timeout=TIMEOUT_SECONDS,
        )

        output_parts = []
        if result.stdout:
            output_parts.append(f"[stdout]\n{result.stdout.rstrip()}")
        if result.stderr:
            output_parts.append(f"[stderr]\n{result.stderr.rstrip()}")
        output_parts.append(f"[returncode] {result.returncode}")

        status = "✅ 成功" if result.returncode == 0 else "❌ 失敗"
        output_parts.insert(0, f"[実行結果] {status}")

        return "\n".join(output_parts)

    except subprocess.TimeoutExpired:
        return f"エラー：タイムアウト（{TIMEOUT_SECONDS}秒）: {command}"
    except FileNotFoundError:
        return f"エラー：コマンドが見つかりません: {command}"
    except Exception as e:
        return f"エラー：コマンド実行に失敗しました: {command} ({e})"
```

`subprocess.run` の主な引数を解説します。

| 引数 | 値 | 理由 |
| --- | --- | --- |
| `shell=True` | True | Windowsのビルトインコマンド（`dir`など）を動かすため |
| `capture_output` | True | stdout/stderrを文字列として取得するため |
| `text=True` | True | バイト列ではなく文字列として受け取るため |
| `encoding="utf-8"` | utf-8 | Pythonスクリプトの出力はUTF-8 |
| `errors="replace"` | replace | cp932出力を壊さずにUTF-8に変換するため |
| `timeout` | 30秒 | 長時間コマンドで永遠に待たないため |

戻り値は `stdout` / `stderr` / `returncode` をまとめた文字列です。LLMはこの文字列を読んで「成功したか」「エラーメッセージは何か」を判断します。

src/tools/exec\_tools.py

```
DANGEROUS_COMMANDS = [

```

実行前に `DANGEROUS_COMMANDS` の各キーワードが含まれていないかをチェックします。

これは**完全なサンドボックスではありません**。キーワードを変形すれば回避できます。ただし今回のエージェントは「人間確認フローで補完する」設計です。コマンドを実行する前に必ず `[コマンド実行確認]` で人間が確認するので、危険なコマンドが表示された時点でキャンセルできます。

簡易チェックは「うっかり呼び出してしまった」ケースへの最低限の防壁であり、主な安全装置は人間確認フローという考え方です。

## Windowsでのcp932問題

Windowsの `dir` コマンドなど、システム標準のコマンドの出力はcp932（Shift-JIS）エンコーディングで返ってきます。これをUTF-8として読もうとすると文字化けします。

`encoding="utf-8", errors="replace"` の組み合わせが対策です。デコードできない文字は `?` に置き換えて処理を続行します。

```
[テスト2] dir src（Windowsコマンド）
[実行結果] ✅ 成功
[stdout]
 ?????̓??????????????  Windows

 C ?????̃h???C?u?ɂ???{?[???̓??x?? ...
[returncode] 0
```

見た目は文字化けしていますが `returncode=0` で正常終了しています。LLMに返す情報としては「コマンドが成功した」という事実が重要なので、日本語の表示が崩れても機能上の問題はありません。

## 動作確認の様子

`step10_command.py` で単体テストを行いました。

**正常なコマンドの実行**

```
result = run_command("python --version")
```

```
[実行結果] ✅ 成功
[stdout]
Python 3.13.0
[returncode] 0
```

**存在しないコマンドのエラーハンドリング**

```
result = run_command("nonexistent_command_xyz")
```

```
[実行結果] ❌ 失敗
[stderr]
'nonexistent_command_xyz' は、内部コマンドまたは外部コマンド、
操作可能なプログラムまたはバッチ ファイルとして認識されていません。
[returncode] 1
```

`shell=True` でcmd.exe経由で実行されるため、存在しないコマンドは `FileNotFoundError` ではなく `returncode=1` で返ってきます。エラーメッセージはstderrに含まれているので、LLMはそれを読んで「コマンドが見つからなかった」と判断できます。

## 躓いた点・気づき

**`shell=True` の意味をWindowsで理解する**

Linuxでは `shell=True` は `/bin/sh -c <command>` として実行することを意味します。Windowsでは `cmd.exe /c <command>` として実行されます。

`shell=True` がないと `python` のようなPATHに登録されたコマンドしか呼べません。`dir` / `type` / `copy` のようなWindowsビルトインコマンドはcmd.exeの組み込み機能なので、`shell=True` なしでは `FileNotFoundError` になります。

**cp932出力は `errors="replace"` で許容する**

`encoding="utf-8"` を指定しても、cp932のバイト列をUTF-8として解釈しようとすると `UnicodeDecodeError` になります。`errors="replace"` を加えることでデコードエラーを例外ではなく `?` 文字への置換で処理します。

完全に正確な出力が得られないのは許容範囲です。エージェントが `dir` の出力の日本語ファイル名を正確に読む必要がある場面では `list_files` ツールを使うべきで、`run_command` はコマンドの成否とstdoutの主要な情報を取得するために使うものです。

**存在しないコマンドが `FileNotFoundError` にならない**

`subprocess.run("nonexistent", shell=True)` はcmd.exeを経由するため、cmd.exeは起動できます。「nonexistentというコマンドは知らない」というエラーをcmd.exe自身がstderrに出力し、returncode=1で終了します。結果として `FileNotFoundError` は発生せず、`returncode=1` のエラーとして正常に捌けます。

`shell=False` の場合は `subprocess.run(["nonexistent"])` で直接起動しようとして `FileNotFoundError` になります。
