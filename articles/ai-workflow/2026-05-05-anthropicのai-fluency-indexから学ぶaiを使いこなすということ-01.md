Anthropicが2026年に公開した[AI Fluency Index](https://www.anthropic.com/research/AI-fluency-index)というレポートを読み解く記事です。AIエージェントを日々使うエンジニアと、組織でAI活用を進める立場の人に向けて、この調査が示した知見を立場別に整理していきます。

## 調査の目的

Anthropicの問題意識はとてもシンプルです。AIの利用率は世界中で伸びているけれど、「人々は本当にAIをうまく使えるようになっているのか」は誰も測っていない、というものです。その問いに答えるために行われたのがこの調査です。

ここで言う「うまく使えている」状態のことを、レポートでは **AI流暢性（AI Fluency）** と呼んでいます。語学の流暢性が単語量や文法知識ではなく場面に応じて自然に使える状態を指すように、AI流暢性も知識の量ではなく対話と判断の習慣として現れるものです。だからこそ今回の調査は、会話ログという「行動」を分析対象にしています。

ベースになっているのはRick DakanとJoseph Fellerが提唱した **4D AI Fluency Framework**。次の4つの軸で構成されています。

* Delegation（委任）：何をAIに任せるかを判断する
* Description（記述）：意図と文脈を明確に伝える
* Discernment（識別）：出力の品質を見抜く
* Diligence（誠実さ）：責任を持って使う

このフレームワークが定義する24の行動のうち、Claude.aiの会話ログから観察可能な11個を分析対象にしました。残り13個は会話の外で起こる行動なので、今回の調査では追跡されていません。

### 4Dフレームワークの12コンピテンシー

各次元はさらに3つの下位コンピテンシーに分かれています。Anthropic公式の[AI Fluency: Key Terminology Cheat Sheet](https://www-cdn.anthropic.com/4396730ed190e691a3712cf2fd6bfe35509deca2.pdf)で定義されている12項目は次の通りです。

| 次元 | 下位コンピテンシー | 説明 | 原文表記 |
| --- | --- | --- | --- |
| 委任 | 課題の把握 | AIを使う前にゴールと作業の性質を明確に理解する | Delegation/Problem Awareness |
| 委任 | プラットフォーム理解 | 各AIシステムの能力と限界を理解する | Delegation/Platform Awareness |
| 委任 | タスク委任 | 人間とAIの強みを活かして仕事を分配する | Delegation/Task Delegation |
| 記述 | 成果物の記述 | アウトプット、形式、対象、スタイルを定義する | Description/Product Description |
| 記述 | プロセスの記述 | AIがリクエストにどう取り組むかを定義する | Description/Process Description |
| 記述 | 振る舞いの記述 | 協働中のAIの振る舞い（簡潔／詳細など）を定義する | Description/Performance Description |
| 識別 | 成果物の評価 | AIが生み出したものの品質（精度・整合性など）を評価する | Discernment/Product Discernment |
| 識別 | プロセスの評価 | AIがどう結論に至ったかの論理を評価する | Discernment/Process Discernment |
| 識別 | 振る舞いの評価 | AIのコミュニケーションスタイルが適切かを評価する | Discernment/Performance Discernment |
| 誠実さ | 創造への誠実さ | どのAIをどう使うかを慎重に選ぶ | Diligence/Creation Diligence |
| 誠実さ | 透明性への誠実さ | AIの関与を関係者に正直に開示する | Diligence/Transparency Diligence |
| 誠実さ | 配備への誠実さ | アウトプットの検証と保証に責任を持つ | Diligence/Deployment Diligence |

24個の行動はこの12コンピテンシーをさらに細かく展開したもので、フル一覧はAnthropicの公式資料には掲載されていません。

### 実際に測定された11の行動

今回の調査で会話ログから観測された11行動を、出現率の高い順に並べたのが次の表です。

| 順位 | 日本語訳 | 出現率 | 行動（英語） | 関連次元 |
| --- | --- | --- | --- | --- |
| 1 | 反復・改善 | 85.7％ | Iterative improvement | Description |
| 2 | 目標の明確化 | 51.1％ | Clarifying goals | Delegation |
| 3 | 良い例の提示 | 41.1％ | Providing examples | Description |
| 4 | フォーマットの指定 | 30％ | Specifying format | Description |
| 5 | 協働の作法の指示 | 30％ | Setting interaction parameters | Description |
| 6 | トーン・スタイルの選好表明 | 22.7％ | Tone/style preference | Description |
| 7 | 欠落文脈の指摘 | 20.3％ | Identifying missing context | Discernment |
| 8 | 対象オーディエンスの定義 | 17.6％ | Defining target audience | Description |
| 9 | 推論への疑問 | 15.8％ | Questioning reasoning | Discernment |
| 10 | 実行前のアプローチ相談 | 10.1％ | Consulting before execution | Delegation |
| 11 | 重要事実の検証 | 非公開 | Verifying important facts | Discernment |

注目すべきは、11行動のうち過半数が **Description（記述）** に属している点です。AIに対して「何を、どう、どんな振る舞いで」を伝える行動が、流暢性の中心になっていることがわかります。

## 調査の結果

調査の骨格は次の通りです。

| 項目 | 内容 |
| --- | --- |
| 対象期間 | 2026年1月の7日間 |
| サンプル数 | 複数ターンを含む9,830件の会話 |
| 分析手法 | プライバシー保護分析ツールCLIOによる11行動のバイナリ分類 |
| 対応言語 | 英語、フランス語、スペイン語、中国語、日本語、ドイツ語 |

ここから浮かび上がった主要な発見は2つです。

### 反復こそが流暢性の鍵

会話の **85.7％** で「反復・洗練」の行動が見られました。そして反復している会話は、他の流暢性行動が出る確率が桁違いに高いことがわかっています。

* Claudeの推論に疑問を呈する確率：**5.6倍**
* 欠けている文脈を指摘する確率：**4倍**
* 平均的な流暢性行動の数：2.67個（反復なしの会話では1.33個）

「会話を続ける人ほど、AIをうまく使うあらゆる行動を取っている」という強い相関が見えています。

### 完成された出力ほど検証が甘くなる

会話の12.3％はコードやドキュメントなどの成果物作成を含んでいました。その会話では、指示は丁寧になる一方で、評価行動が逆に減るという矛盾が観測されています。

| 行動カテゴリ | 成果物作成時の変化 | 原文表記 |
| --- | --- | --- |
| 目標の明確化 | ＋14.7ポイント | Clarifying goals |
| 形式の指定 | ＋14.5ポイント | Specifying format |
| 例の提供 | ＋13.4ポイント | Providing examples |
| 欠落文脈の指摘 | −5.2ポイント | Identifying missing context |
| 事実確認 | −3.7ポイント | Checking facts |
| 推論への疑問 | −3.1ポイント | Questioning reasoning |

> 完成度の高い出力は、追加の検証が不要に見えてしまう。

レポートはこの傾向に警鐘を鳴らしています。見栄えのよさと正しさは別物、ということです。

## AIエージェントを使う人にとって大事なこと

レポートがユーザーに向けて示している提言は次の3つです。

1. **会話を続ける**：最初の応答はゴールではなくスタート地点と考える
2. **完成された出力こそ疑う**：見栄えがいいほど検証を怠りやすいという人間の癖を自覚する
3. **協働の条件を最初に提示する**：「違う前提があれば反論してほしい」と冒頭で伝える

特に重要なのは1番目です。9,830件のデータから浮かび上がったのは、AIをうまく使う人は「一発で正解を引き出すプロンプトの達人」ではなく、「Claudeとやりとりを重ねる人」だという事実です。プロンプトエンジニアリングという言葉が指し示す技能の重心は、すでに「最初の質問」から「対話の継続」へと移ってきています。

## シニアエンジニアに向けて

シニアの強みは、コードや文書の品質を瞬時に見抜く目です。普段ならClaudeの出力に「ここの例外処理が雑」「この設計は将来詰む」とすぐ気づけるはずです。

しかし、調査が指摘する「成果物が出ると検証が甘くなる」傾向は、熟練度と無関係に効きます。むしろ忙しいシニアほど、深夜や締め切り前に「動いているからOK」と通してしまうリスクが高いと思います。

意識すべきことは次の2つです。

* **後輩のPRをレビューする基準を、Claudeの出力にも適用する**
* **チームのAI利用規範を作る役割を引き受ける**：「会話を続ける」「成果物には別の評価ターンを設ける」などをドキュメント化する

> The role of senior engineers shifts from being a great user to designing how the team uses AI well.

シニアの責任は「自分がうまく使う」だけでなく、「組織としてうまく使う仕組みを作る」ことに移っています。

## ジュニアエンジニアに向けて

ジュニアにとってこの調査の最大のメッセージは、「経験不足は反復で補える」ということです。

レポートは反復している人ほどあらゆる流暢性行動が伸びると示しました。これを裏返すと、**反復さえできれば経験の差は縮まる**ということでもあります。

実践してほしいのは次のような行動です。

* 最初の回答を読んだら、必ず「なぜそう判断したのか」を聞く
* コードが動いたら「他にどんな書き方があるか」を聞く
* 自分がわからなかった部分を「もっと噛み砕いて説明して」と頼む

これらは経験を補うだけでなく、Claudeとの会話そのものが学びの場になります。シニアの隣に座って質問できない環境でも、Claudeには無限に質問できる。これは過去のジュニアにはなかった大きな機会です。最初の答えを鵜呑みにする癖さえ捨てれば、AIはあなたの最高の家庭教師になります。

## 人事や経営者に向けて

経営の視点でこの調査を読むと、「AI導入率」というKPIの限界が見えてきます。

ライセンスを配って利用率が上がっても、調査が示すような「流暢な使い方」をしていなければ、生産性も品質も上がりません。むしろ「成果物の検証を怠る組織」が増えれば、品質事故の温床になります。

考えるべきことは次の通りです。

| 領域 | やるべきこと |
| --- | --- |
| トレーニング設計 | プロンプトの書き方ではなく、対話の継続と出力の検証を教える |
| 評価指標 | 利用回数ではなく、利用の深さ（反復ターン数、検証行動の有無）を見る |
| ガバナンス | 成果物がそのまま本番に流れない、ヒューマンレビューを残す仕組みを作る |

レポートは「相関であって因果ではない」と限界を明記していますが、「反復する組織は質の高いアウトプットを出す」という相関は十分に示唆的です。研修プログラムを再設計するときの指針になると思います。

## まとめ

* [AnthropicのAI Fluency Index](https://www.anthropic.com/research/AI-fluency-index)は、9,830件の会話から「AIをうまく使う人の行動」を可視化したレポートです
* 鍵は「反復」と「完成された出力への懐疑」の2つです
* シニアは仕組み作り、ジュニアは反復の習慣化、経営層は評価指標の刷新が課題になります
* AIをどう使うかは、もはや個人の生産性の問題ではなく、組織のケイパビリティの問題になってきています
