月6万円。

これがうちのSNS自動投稿システムが、私の気づかないうちに毎月吸い上げていたAPI使用料だった。

きっかけは、5月29日の朝。スマホにこんな通知が届いた。「クレジット残高不足のため、X（旧Twitter）への投稿に失敗しました」。週3本が、空に消えていた。

正直、最初は意味がわからなかった。「クレジット？うち、月額のサブスク払ってるだけのつもりだったけど」。コーヒーを淹れ直して、請求履歴を開いた。そこに並んでいたのは、月3万円超×2アカウント。合計、月6万円。

書いた分だけ勝手に送ってくれる便利な箱だと思っていた。実態は、送るたびに小銭が落ちる課金システムだった。

## 「動いてるから問題ない」が、いちばん危ない

うちには、SNS投稿を回している自社のAIエージェントがある（社内では「SNS投稿担当」と呼んでいる）。役割はシンプル。記事公開や日々の気づきを、X と Threads に1日3本、週21本、自動で散らす。

導入したときの私は、たぶん多くの中小事業の経営者と同じ顔をしていたと思う。「これでSNSの手間がゼロになる、よかった」。それ以上、中を覗かなかった。動いているし、毎週ちゃんとフォロワーは少しずつ増えていく。問題はないように見えた。

問題は、見える場所に来るまで時間がかかる、ということだった。

X の API には、月額の基本料金とは別に「投稿1本ごと」に消費されるクレジットがある。週21本を3か月続けると、クレジットが先に底をつく。クレジット切れの瞬間まで、UI 上は静かに動いている。請求書を開かないと気づかない。

これは、X だけの話じゃない。ChatGPT の API も、画像生成も、SaaS の「使った分だけ」プランも、全部同じ構造をしている。

「動いているから問題ない」と「動いているから気づけない」は、ほぼ同じ意味だ。

## 週21本を、週7本まで削った

5月29日の夜、いったん全自動投稿を止めた。サウナに行った。整いながら考えた。たぶん10分くらい。

戻ってきて、設計を全部書き直した。考え方はシンプルだった。

X にだけ「お金がかかる」。Threads は無料。だったら、X に流すのは、Threads で反応が良かった投稿の「上澄み」だけにすればいい。

具体的にはこうした。

まず、投稿スロットを「コア」と「サブ」に分けた。コアは X と Threads 両方に投稿する枠。サブは Threads だけに投稿する枠。月〜金で、コアは週7本、サブは週14本。合計の投稿数は週21本のまま、X への投稿だけが3分の1に減る。

次に、曜日ルールを入れた。水曜は X 完全スキップ（コア枠でも Threads のみ）。エンゲージメントが構造的に弱い曜日は、X に投げない。金曜だけは朝と夜の2本、コア枠を確保した。週末前後はリーチが伸びやすいので、ここはお金を使う。

ここで、ひとつ気をつけたことがある。「投稿が減った」だけだと、3か月後にまた忘れる。だから、投稿ログに「X投稿スキップ（コスト最適化）」という1行が必ず残るようにした。スキップした理由が、後から自分でも追えるようにしておく。

結果、月6万円が月2万円弱まで下がった。露出の体感は、1〜2割減くらい。Threads のフォロワー伸びは、むしろ良くなった。たぶん、伸びる時間帯にだけ X に出すようにしたからだ。

## あなたの会社にも、同じ落とし穴がある

正直に書く。これは「うちが SNS で失敗した笑い話」じゃない。

中小事業で、なにかしら自動化を回している人にとっては、ぜんぶ自分の話だと思って読んでほしい。

会計ソフト、議事録ツール、顧客対応のチャットボット、AI 翻訳、画像生成、メール配信、フォーム送信通知。月額3,000円で導入したつもりが、内部で「1回あたり」「1通あたり」「1文字あたり」が動いていることがある。

最悪なのは、それが赤字を直接生まない金額で動いているケースだ。月6万円は、うちの規模だと「経費の中で目立つ違和感」になる。月8,000円だったら、私はたぶん半年気づかなかった。

ここまでで気付いた人もいると思う。SNS自動化が高かったんじゃない。「自動化されたものを、その後だれも見ていなかった」のが高かったんだ。

## 明日の朝5分で、自社の「従量課金リスト」を作る

ここからが本題。

サウナで考えた話を、あなたの会社でも回せる手順に落とす。所要時間は、朝のコーヒー1杯。

### 手順1：今月の請求書を、3つの箱に仕分ける

請求書（クレジットカード明細でも口座引き落としでも）を開いて、SaaS・ツール系の支払いだけを抜き出す。それを次の3つに仕分ける。

1. 定額（月額・年額が変わらない）
2. 従量（使った分だけ請求が変動）
3. 混合（定額＋使った分の上乗せ）

混合と従量に入ったものが、今日のリスク銘柄だ。

### 手順2：従量・混合のツールに、ChatGPT または Claude に質問する

ここでAIを1本使う。プロンプトはこれだけでいい。

```
あなたはSaaS導入支援の専門家です。
以下のサービスについて、料金体系を「定額の部分」「従量の部分」「上限の有無」「課金が発生する操作」の4項目で簡潔にまとめてください。

サービス名：（ここに1つだけ書く）
私が現在契約しているプラン：（わかれば書く）
```

返ってきた答えは100%は信じない（料金体系は変わる）。けど、自社の請求書とAIの答えを並べると、「これは投稿1本ごとに課金されるのか」「APIコール1回ごとに課金されるのか」が見える。

### 手順3：「上限がない」サービスを、上限がある状態に変える

ここがいちばん大事。

従量課金のサービスには、たいてい「予算上限」「クレジット上限」「月次キャップ」みたいな設定がある。

X の API なら、開発者ポータル（[developer.x.com](https://developer.x.com/)）でクレジット残高と消費アラートを設定できる。ChatGPT の API なら、OpenAI のダッシュボードで月額の使用上限と、80%通知のメールを設定できる。

これ、ほぼ全社が「使えるけど設定していない」状態になっている。

### 手順4：「使う側」を、ルールで間引く

これは SNS で私がやったのと同じことを、別のサービスでもやる、という話。

たとえば AI AI 翻訳（1文字あたり課金）なら、「英文の社外メールだけ翻訳、社内Slackは原文のまま」に変える。

ぜんぶの自動化を止める必要はない。「効果が高いところに、お金を集中させる」だけでいい。

### 手順5：月次で「スキップしたログ」を見る

これが意外と効く。

うちの SNS 投稿エージェントは、今は「X投稿スキップ（コスト最適化）」というログを、投稿のたびに残す。これを月末に AI に読ませて、「今月いくら節約できた見込みか、削った投稿のうち反応が良かった時間帯はないか」を要約してもらう。

これをやらないと、「節約しすぎてリーチが落ちた」「実は水曜の方が反応よかった」みたいなズレに気づけない。節約は、放っておくと過剰になる。

## ぜんぶの自動化に、これが効く

整理しよう。

自動化は便利だ。便利すぎる。だから、その下で何が課金されているのかを、私たち経営者は見なくなる。月6万円で済んだのは、たまたまうちが小規模だったからで、もう少し大きい組織だったら、月数十万円の請求書がしれっと届き続けていただろう。

「動いているから問題ない」を、「動いているから、月1回中を見る」に変える。

これだけで、ぜんぶの自動化が、経営者にとって安全な道具になる。

明日の朝、5分でいい。請求書を1枚開いて、3つの箱に仕分けるところから始めてほしい。

きっと、1個か2個、見覚えのない名前が混ざっている。
