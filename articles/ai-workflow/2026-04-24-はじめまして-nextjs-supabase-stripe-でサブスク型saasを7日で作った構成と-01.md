# 作ったもの

「PRGym」という「コードレビュー力を鍛える」サービスです。  
着想からリリースまで約1週間で開発しました。

<https://prgym.dev>

「PRGym」は、エンジニアの方ならおなじみの「PR = Pull Request」のレビュー力を鍛えるというイメージで命名しました。  
GitHub の PR レビューのようにコードの行を選んでレビューコメントを投稿すると、AIがフィードバックを返します。（レビューコメントのレビューをしてくれます）  
答えを直接教えるのではなく、ヒントと問いかけで自力発見を促す設計にしています。

![](https://static.zenn.studio/user-upload/d57c0a401c10-20260424.png)

料金体系は Free 無料（週2問） / Lite ¥980/月（毎日1問） / Standard ¥1,980/月（無制限）の3プランです。

# 技術スタック

| 領域 | 技術 |
| --- | --- |
| フロントエンド / バックエンド | Next.js (App Router) |
| 認証 / DB | Supabase (Auth + PostgreSQL） |
| 決済 | Stripe (Subscription) |
| AI | Anthropic Claude |
| ホスティング | Vercel |

すべてフリープランまたは従量課金で始められるサービスを選びました。初期コストを極力抑え、ユーザーが増えてからコストがスケールする構成にしました。

# 選定理由

全体的に、開発スピードと初期コストを考慮して選定しました。

## Next.js (App Router)

フロントとAPI（バックエンド）を同一プロジェクトで管理できるのと、個人的に最近一番よく触っているフレームワークなので選定しました。  
Server Components で初期ロードを軽くしつつ、レビューエディタのようなインタラクティブな部分は Client Components で実装しています。

API Routes (Route Handlers) で Stripe Webhook や AI 関連のエンドポイントを実装しました。

## Supabase

認証と DB をワンストップで提供してくれる点が個人開発に最適でした。特に認証まわりは、 @supabase/ssr を使えば、 Next.js の Server Components / Route Handlers / Middleware のどこからでも一貫した方法でセッション管理ができます。

Google OAuth も Supabase 側で Provider を有効にするだけで、フロント側は

```
signInWithOAuth({ provider: 'google' })
```

の1行のみで済みました。

## Stripe

サブスクリプション管理に Stripe を採用しました。 Checkout Session でプラン選択 → 決済 → Webhook で DB 更新、という流れです。  
設計上のポイントとしては、Webhook でプランを判定する際に Price ID で分岐させています。最初はメタデータだけで判定していましたが、プランのアップグレード・ダウングレード時には `customer.subscription.updated` イベントから Price ID を見る必要がありました。

## Anthropic Claude API

レビューのフィードバック生成に Claude を使っています。プロンプト設計で意識したのは「答えを教えない」こと。ユーザーが問題を指摘したら、その正確性を評価しつつ、見逃している問題にはヒントだけを出す設計にしています。

最終評価では、問題発見力、指摘の正確性、コメントの質、カバレッジの4軸で100点満点のスコアを算出します。

# 設計で工夫した点

## インラインレビューUI

GitHub の PR レビューと同じ感覚でレビューできる UI を目指しました。 shiki でシンタックスハイライトされたコードの各行をクリックすると、コメント入力欄が展開されます。  
実装のポイントとしては、 shiki が返す HTML を行ごとに分割してレンダリングし、各行に onClick ハンドラを付けています。インデントが shiki の出力で失われるケースがあったので、元のコードと比較して補完する処理も入れています。

## プラン制限の設計

「ワークアウト（問題）を聞いただけ」でカウントするのか、「レビューを送信したとき」にカウントするのかで、ユーザー体験が大きく変わります。 PRGym では後者を採用し、実際にレビューを送信した回数で制限をかけています。  
これにより、問題を見て考えている段階では制限を消費しないので、ユーザーがプレッシャーなく問題を選べます。

## 問題の自動生成

ワークアウトの問題は AI で自動生成しています。言語・フレームワーク・難易度・カテゴリを指定すると、意図的にバグが仕込まれたコードや改善点があるコードとその解説が生成されます。  
精度を上げるために、フレームワーク未指定の場合は標準ライブラリのみを使うように指示したり、カテゴリごとの詳細な定義をプロンプトに含めたりしています。
