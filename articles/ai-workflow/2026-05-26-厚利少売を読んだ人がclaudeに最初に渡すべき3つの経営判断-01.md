すがけんです。

厚利少売を読むと、たぶん最初に考えることはこうです。

誰に売るか。  
誰に売らないか。  
何を安く受けないか。  
どの単価で成立させるか。  
どうやって、少ない数で利益を残すか。

これは、考え方としてはわかる。

でも、実務に戻ると止まる人が多いです。

なぜなら、経営判断は一度決めて終わりではないからです。

今日の商談で迷う。  
来週の見積もりで迷う。  
新しい問い合わせで迷う。  
採用で迷う。  
商品を増やすか減らすかで迷う。

厚利少売は、判断基準です。

でも、判断基準は、日々の仕事の中で回して初めて効きます。

その時に、Claudeはかなり相性がいいです。

今日は、厚利少売を読んだ人が、Claudeに最初に渡すべき3つの経営判断について書きます。

## Claudeは、答えを出す道具ではなく、判断を回す道具

Claudeを使う時、多くの人はこう聞きます。

この文章を直して。  
このメールを作って。  
この議事録を要約して。  
この企画を考えて。

もちろん便利です。

でも、厚利少売を実装するなら、もう少し使い方を変えた方がいいです。

Claudeに作業を頼む前に、判断を渡す。

これが大事です。

判断を渡さずに作業だけ頼むと、Claudeはそれっぽいものをたくさん作ってくれます。

でも、それが自分の事業にとって正しいかは別です。

たとえば、誰にでも売る前提でClaudeに営業文面を書かせると、誰にでも刺さりそうな文章が出ます。

安く受ける前提でClaudeに提案書を書かせると、安い仕事を取りにいく提案書が出ます。

商品を増やす前提でClaudeに企画を出させると、商品がどんどん増えます。

つまり、Claudeは前提を増幅します。

だから、最初に渡すべきなのはプロンプトのテクニックではありません。

自分の経営判断です。

## 最初に渡すべき判断1: 断るべき顧客条件

最初にClaudeへ渡すべきなのは、断るべき顧客条件です。

厚利少売で大事なのは、たくさん売ることではありません。

利益が残る相手に、ちゃんと価値を届けることです。

そのためには、誰に売らないかを決める必要があります。

Claudeには、こう聞きます。

```
私は次の事業をしています。
[事業内容]

厚利少売を実践する前提で、断るべき顧客条件を10個出してください。
それぞれについて、なぜ断るべきか、断らない場合に起きるリスク、やわらかく断る表現をセットで出してください。
```

これをやると、ただの顧客リストではなく、判断基準ができます。

断るべき顧客が見えると、営業文面も変わります。

問い合わせフォームも変わります。

無料相談の聞き方も変わります。

提案書で強調する場所も変わります。

Claudeは、ここを一緒に整理する相手として使えます。

## 最初に渡すべき判断2: 守るべき利益構造

次に渡すのは、守るべき利益構造です。

厚利少売は、単価を上げればいいという話ではありません。

利益が残る構造を作る話です。

だから、Claudeには売上だけで考えさせない方がいいです。

こう聞きます。

```
私の事業について、売上ではなく利益が残る構造を作るために確認すべき数字を整理してください。

事業内容:
[事業内容]

現在わかっている数字:
[単価、粗利、工数、リピート率、問い合わせ数など]

出してほしいもの:
1. 守るべき数字
2. 悪化すると危険な数字
3. 増やしてはいけない売上
4. 値上げより先に見直すべきこと
```

ポイントは、「売上を増やす方法」ではなく「増やしてはいけない売上」まで聞くことです。

ここが厚利少売っぽいところです。

売上は増えているのに、現場が苦しくなる。

忙しいのに、利益が残らない。

受注はあるのに、社長がずっと疲れている。

こういう時は、増やしてはいけない売上が混ざっています。

Claudeにその前提を渡すと、単なる売上アップ案ではなく、利益構造の壁打ちになります。

## 最初に渡すべき判断3: 今月決めるべき経営判断

3つ目は、今月決めるべき経営判断です。

厚利少売は、読むだけでは変わりません。

何かを決める必要があります。

値上げするのか。  
断る条件を作るのか。  
商品を減らすのか。  
顧客層を絞るのか。  
営業の入口を変えるのか。  
安く受けている仕事をやめるのか。

Claudeには、こう聞きます。

```
厚利少売を実践する前提で、今月決めるべき経営判断を5つ出してください。

条件:
- すぐ売上を増やすためではなく、利益が残る構造に近づけるための判断にする
- 各判断について、やる場合/やらない場合のリスクを出す
- 最初の1週間で確認する数字も出す
```

これをやると、厚利少売が「いい考え方」から「今月の経営判断」に変わります。

経営は、抽象論のままだと動きません。

いつ、何を、どの順番で決めるか。

ここまで落とす必要があります。

Claudeは、この「落とす」作業に向いています。

## Claudeに渡す前に、雑でもいいから前提を書く

Claudeをうまく使う人は、きれいなプロンプトを書いている人ではありません。

前提を渡している人です。

事業内容。  
単価。  
粗利。  
誰に売っているか。  
本当は売りたくない相手。  
現場で詰まっていること。  
今月決めたいこと。

このあたりを雑でもいいから渡す。

すると、Claudeの返答は変わります。

逆に、前提を渡さずに「どうすれば売上が上がりますか」と聞くと、一般論が返ってきます。

一般論は間違っていない。

でも、経営判断には弱い。

厚利少売を実装するなら、一般論ではなく、自社の前提に合わせて考える必要があります。

## 今日やるなら、この3行でいい

今日Claudeを開くなら、まずこの3行だけでいいです。

```
厚利少売を実践する前提で、うちが断るべき顧客条件を10個出してください。
```

```
うちの事業で、増やしてはいけない売上を分類してください。
```

```
利益が残る構造に近づけるために、今月決めるべき経営判断を5つ出してください。
```

これだけでも、Claudeの使い方が変わります。

メール作成や議事録要約ではなく、経営判断の壁打ちになります。

## 厚利少売とClaudeは、別物ではない

厚利少売は、判断基準です。

Claudeは、その判断基準を毎週の仕事に落とす実装環境です。

どちらか片方でも役に立ちます。

でも、組み合わせると見えるものが変わります。

厚利少売で、何を大事にするかを決める。  
Claudeで、それを毎週の判断に落とす。  
また厚利少売に戻って、判断基準を磨く。

この往復ができると、経営の考え方が少しずつ変わります。

AIを使う目的は、作業を速くすることだけではありません。

経営者が、同じ迷いを何度も繰り返さないようにすることです。

厚利少売を読んだ人ほど、Claudeには作業ではなく判断を渡してください。

そこから、AIの使い方が変わります。

---

厚利少売を読んで「次に何を決めればいいか」で止まっている方へ。

Claude AI転換では、Claudeを経営判断に使うための考え方と実装手順をまとめています。
