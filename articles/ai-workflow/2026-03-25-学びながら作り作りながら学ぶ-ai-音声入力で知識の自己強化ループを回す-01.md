## 😵‍💫 触ってみた → 次が出た → 前のやつ忘れた

新しいツールが出たから触ってみる、本を読んでみる、作ってみる。

でも翌週にはまた新しいものが出ていて、先週やったことはもう記憶の彼方です。  
AIの動向が速すぎて、学んだそばから忘れていきます。

ただ、新機能を触るだけでは身につくものも身につきません。  
どういう原理で動いているのか、原理原則を深く理解しないと、次の新機能が出たときにまたゼロからやり直しになります。

かといって原理をインプットするだけだと心が折れます。  
分厚い本を前に「これ全部やるのか…」となって手が止まります。

これはAIに限った話じゃなくて、例えばマネジメントや経営、資産運用なんかでも同じでした。

どうすればモチベーションを維持しながら、原理原則を深く学べるか

少しずつ作る、少しずつ成果が見える、それがまた次の学びにつながる。  
学びから学びを分析し、次の学びに活かす。  
そういうフローを色々と模索・試行錯誤しながら、今回紹介するループの仕組みにたどり着きました。

**これはあくまで自分にフィットしたループです。**  
人それぞれもっと良いやり方やフィットするものがあると思うので、参考程度に読んでもらえれば🙏

## 👥 エージェントチームが並列で動く

このループの中核を担っているのが、[Claude Code](https://docs.anthropic.com/en/docs/agents-and-tools/claude-code/overview)の[Agent Teams](https://docs.anthropic.com/en/docs/claude-code/agent-teams)です。  
複数のエージェントをチーム構成で並列に動かしています。

なぜ役割を分けているかというと、**リサーチしながら実装しながらナレッジを書くのは1つのエージェントでは難しい**からです。  
人間のチームと同じで、調査担当・実装担当・ドキュメント担当に分けた方が、それぞれのアウトプットの品質が上がりますし、何より並列で走るので速いです。

1つのトピックを投げると、5つの役割が同時に走ります。

**🎙️ インプット** → **Claude Code（リーダー）** が5人のチームに振り分け → **統合・commit + push**

> 🔍 **リサーチ** — 最新動向をWebで調査 → 2026年の最新手法・ツール
>
> 📝 **ナレッジ** — 知識を体系化して書き出し → `knowledge/` にMarkdown
>
> 🛠️ **実装** — コンポーネントの改善・追加 → 各リポジトリにコード
>
> 💼 **業務活用** — 本業・副業にどう活かせるか → 業務プロジェクトへの提案
>
> ✍️ **アウトプット** — ブログ・ポートフォリオへの発信 → Zenn記事、ポートフォリオ更新

**5つの仕事が並列で走ります。**  
1人で順番にやったら30分以上かかるところが、数分で終わります ⚡

スキルやナレッジは[Codex](https://openai.com/index/codex-cli/)からも読み込めるので、役割を分散できる場合はCodexにもチームに参加してもらっています。  
[Claude Code](https://docs.anthropic.com/en/docs/agents-and-tools/claude-code/overview)だけに閉じず、**適材適所でAIエージェントを使い分ける**のがポイントです。

## 🚀 投げるだけで全自動

では、実際にどうやってチームにトピックを投げているのか。

入力方法は色々あります。  
[Aqua Voice](https://www.aqua-voice.com/)で音声入力したり、Claude Codeの[`/voice`モード](https://docs.anthropic.com/en/docs/claude-code/voice)で直接話しかけたり、参考記事のURLや図解の画像をそのまま投げたり。

**📖 本を読む** → **🎙️ 音声・テキスト・画像で投げかける** → **🤖 あとは全自動**

投げたら、先ほどのチームが以下を自動実行します。

| ステップ | やること | 例 |
| --- | --- | --- |
| 💡 解説 | 自分の実装に例えて説明 | 「Self-Attentionはあなたのキャッシュ設計と繋がる」 |
| 🔍 リサーチ | 2026年の最新動向を調査 | FlashAttention-3、GQA、MoEの現状 |
| 📝 ナレッジ化 | Markdownで書き出し | `knowledge/` に自動追加 |
| 🛠️ 実装 | コードに落とす | ガードレール、キャッシュ、トレーシング等 |
| 💾 保存 | commit + push | 全リポジトリで自動実行 |

**指示していません。**  
[`CLAUDE.md`](https://docs.anthropic.com/en/docs/agents-and-tools/claude-code/memory#claude-md)にワークフローを書いただけです。

最近はClaude Codeの[Remote Control](https://docs.anthropic.com/en/docs/claude-code/remote-control)機能でスマホからも操作できるようになったので、お風呂に入りながらでもこのループを回せます 🛁

湯船で本を読んで、思いついたことをスマホから投げる。  
上がったらナレッジと実装ができています。

ただし、お風呂でスマホに話しかけていると子供にめちゃくちゃ不審がられます笑

## 🌍 AIだけじゃない

ここまでAIの話が中心でしたが、この仕組みはAI専用ではありません。  
同じサイクルを複数のドメインで回していて、それぞれ目に見えるアウトプットが出ています。

| ドメイン | インプット | アウトプット | 本業への還元 |
| --- | --- | --- | --- |
| 🤖 AI | 文献・記事・論文等 | MCPクライアント/サーバー、FTの仕組み、ゲートウェイ、キャッシュ機構、ガードレール、RAG、各種エージェント等 | プロダクト開発、自動化・効率化、開発の効果性を高める取り組み等 |
| 👥 マネジメント | 文献・記事・実務経験等 | 知識ベース + Webアプリ等 | チーム運営・1on1・評価に活用等 |
| 🔒 セキュリティ | 文献・記事・実務経験等 | 資格取得、ナレッジ整理等 | BCP策定、セキュリティリスクへの対策等 |
| 🏢 経営・組織論 | 文献・記事・実務経験等 | フレームワーク整理、意思決定モデル、組織設計パターン等 | 経営視点に立った開発体制の構築、事業戦略と技術戦略の接続等 |
| 💰 資産運用 | 文献・記事・市況等 | ポートフォリオ戦略等 | 本業・副業には活かせないけど、資産は増やしたい笑 |

ドメインが変わっても仕組みは同じです。  
音声で投げる → ナレッジ化 → 体系化。

本業に活かせるものはそのまま業務に還元できますし、直接関係ないドメインでも知識の体系化自体に価値があります。

## 🏗️ 学ぶたびにシステムが育つ

このループを回していくと、学んだ知識がそのままシステムのコンポーネントとして積み上がっていきます。

その全体を管理するために、Kubernetesの考え方にインスパイアされて、リポジトリを **コントロールプレーン** と **ワークロード** に分けています。

### 🎛️ コントロールプレーン (Private)

ナレッジ・ワークフロー定義・学習戦略を管理する **司令塔** です。  
学習途中の恥ずかしいメモや試行錯誤も全部入っているので、ここはプライベートにしています🙈

| 役割 | 内容 |
| --- | --- |
| `knowledge/` | 文献・記事をトピック別に蓄積 |
| `CLAUDE.md` | ワークフロー定義（全自動化の設定） |
| `.claude/skills/` | スキル定義（繰り返し使うパターンの定型化） |
| `.claude/agents/` | サブエージェント・エージェントチームの構成 |
| `drafts/` | ブログ・アウトプットのドラフト |

### ⚙️ ワークロード (Public)

コントロールプレーンの知識をもとに、実際に動くシステム群です。  
こちらはなるべく公開しています。

モノレポにまとめることも考えましたが、ワークロードはリポジトリを分けています。  
理由は2つで、**トピックを限定したかった**のと、**どんどん公開していきたかった**から。  
1リポジトリ1テーマにすることで、それぞれ独立してPublicにできますし、コンテキストも絞れます。

*※ 以下は執筆時点（2026年3月）の状態です。日々アップデートされています。*

| リポジトリ | 学んだこと | 作ったもの（現時点） |
| --- | --- | --- |
| [`mcp/`](https://github.com/hiro8ma/mcp) | プロンプト設計、ガードレール、キャッシュ等 | MCPクライアント/サーバー、MCP間連携、ゲートウェイ、キャッシュ機構、ガードレール、トレーシング等 |
| [`agent/`](https://github.com/hiro8ma/agent) | RAG、エージェント設計等 | Go Genkitベクトル検索、Google ADK ReActループ、各種エージェントパターン等 |
| [`ft/`](https://github.com/hiro8ma/ft) | ファインチューニング等 | MLX LM + LoRA FT、データ生成パイプライン、学習曲線分析、モデル評価等 |
| [`claude/`](https://github.com/hiro8ma/claude) | Claude Code設計等 | 設定テンプレート、スキル、ワークフロー、エージェント定義等 |
| [`portfolio/`](https://hiro8ma.github.io/portfolio/) | アウトプット等 | ポートフォリオサイト（Next.js）等 |
| ... |  | 今後も増えていきます 📈 |

**📖 学ぶ** → **📝 コントロールプレーンにナレッジ追加** → **🛠️ ワークロードに実装反映** → **📖 また学ぶ** → ...

最初から設計したわけではありません。  
学ぶたびにワークロードのどこかが1つ良くなっていった結果です。  
新しいドメインを学べば、ワークロードのリポジトリもどんどん増えていきます。

## 🔁 自己強化ループ

ここまでの仕組みだけでも十分便利ですが、さらに面白いのは、蓄積したナレッジがモデルの学習データにもなることです。

**📖 学ぶ** → **📝 ナレッジ化** → **🤖 Q&Aデータ自動生成** → **🧠 LoRA FT** → **🚀 MCPサーバーで推論**

学べば学ぶほど、手元のモデルが賢くなります🧠✨

## 🧪 実際に回してみた例

ここまで仕組みの説明が続いたので、実際にトピックを1つ投げたときの、チームの動きと成果物をお見せします。

### 1. 投げた内容

> 「Transformerの内部処理について学んだ。Tokenize、Embedding、Self-Attention、Feed Forwardの4ステップ...」

### 2. チームの動き

| エージェント | やったこと | 所要時間 |
| --- | --- | --- |
| 🔍 リサーチ | Embedding選定基準、FlashAttention-3の性能、日本語Token効率の実測比較を調査 | 約2分 |
| 📝 ナレッジ | 4ステップの解説 + 自分の実装との接続をMarkdownに書き出し | 約1分 |
| 💼 業務活用 | Embedding次元削減でRunbook検索コスト削減、予測確率で信頼度スコア設計 | 約30秒 |
| ✍️ アウトプット | Zenn記事ネタ3本提案 + この記事自体の改善点4つ | 約30秒 |
| 🛠️ 実装 | トークン統計をトレーシングに追加する改善を提案 | 約1分 |

### 3. 生成された成果物

| 種類 | 内容 |
| --- | --- |

1トピック投げただけで、ナレッジ2ファイル + ブログネタ複数 + 業務提案が生成されました🎉

## おわりに

最初は「忘れたくない」から始めた仕組みが、気づいたら学習と構築が同じサイクルで回るようになっていました。

完璧な仕組みではありません。  
エージェントの権限問題で手動介入が必要なこともありますし、リサーチ結果の精度にムラもあります。

でも、**昨日より今日の方がナレッジが増えていて、システムのどこかが1つ良くなっている**。  
その積み重ねが、振り返ると大きな差になっています。

冒頭に書いた通り、これは自分にフィットしたループにすぎません。

ただ、もし同じように「学んだそばから忘れていく」と感じている方がいたら、何かのヒントになればうれしいです🙏
