# 元花屋が独学でプログラミングを覚えて作ったバイク検索サービスが3ヶ月でDAU420人になるまでの全記録

## はじめに

20年間、花屋をやっていました。

最初の頃は仕事終わりにコードを書く時間がありましたが、役職に就いてからは忙しくなり、まとまった開発はほとんどできなくなりました。それでも細々と学習は続けました。2025年にIT企業に転職。エンジニア職ではありません。

平日は夜だけ、あとは週末。この限られた時間で、バイクの中古・新車一括検索プラットフォーム「[MotoHub](https://motohub.jp)」を一人で開発・運営しています。

開発開始から約3ヶ月。現在のDAU（日間アクティブユーザー）は約420人。Google検索からの流入が毎日50〜110人。掲載台数は13万台超。

**本業ありの個人開発で、どうやってゼロからDAU420まで持っていったか。** 技術選定・実装・SEO・グロースの全部を、リアルな数字と一緒に書きます。

## MotoHubとは

全部一人でやっています。元花屋が、本業の後の夜と週末だけで。

## 「元花屋・本業あり」の開発リアル

最初に正直に書いておきます。

20年間花屋で働いていて、独学でプログラミングを始めました。CS学位もなければ、情報系の学校も出ていません。花屋の仕事が終わった後にPHPを書く生活を18年間続けて、2025年にIT企業へ転職しました（非エンジニア職）。**平日は仕事が終わった21時〜25時くらい。週末は土日のどちらか、または両方。**

月に使える開発時間は多くて60〜80時間。フルタイムで個人開発している人の半分以下です。

だからこそ、**開発効率を最大化する技術選定**が生命線でした。後述しますが、Claude Code（AI コーディングツール）の導入が決定的に大きかった。花屋時代から独学でやってきた自分にとって、AIの力を借りられる今の時代は本当にありがたい。これがなかったら3ヶ月でここまで来られなかったと断言できます。

## Phase 1: 「まず作る」— 1ヶ月目（DAU 0 → 10）

### 最初の2週間でMVPを公開

最初に作ったのは**中古バイクの検索機能だけ**。

バイク販売サイトの情報を集約して、MotoHub上で横断検索できるようにしました。MySQLの`LIKE`検索で十分だった時代です（データが少なかったので）。

この時点でのユーザーは自分と友人数人。DAU 0〜3くらい。

**学び**: 本業ありだからこそ、完璧を目指さない。まず「検索して結果が出る」だけで公開する。週末2回で公開まで持っていった。

### Meilisearch導入で検索を爆速化

データが10万件を超えたあたりで`LIKE`検索が限界に。キーワード入力のたびに2〜3秒待たされるようになりました。

Meilisearch（全文検索エンジン）を導入した結果：

|  | Before (MySQL LIKE) | After (Meilisearch) |
| --- | --- | --- |
| 検索速度 | 2〜3秒 | 50ms以下 |
| タイポ耐性 | なし | あり |
| ファセット検索 | SQL手書き | 標準搭載 |
| メモリ消費 | — | 約800MB |

さくらVPS 4GBでDocker上に動かしていますが、12万件でもメモリ800MBで収まっています。Elasticsearchだと4GB以上必要なので、個人開発にはMeilisearchが正解でした。

## Phase 2: 「見つけてもらう」— 2ヶ月目（DAU 10 → 100）

作っただけでは誰も来ない。本業ありだと広告費もかけにくい。**SEOとコンテンツで勝負する**しかありません。

### SEO：ページ数で殴る戦略

MotoHubの最大の武器は**ページ数**です。

* 車種詳細ページ: 約2,500ページ（全車種）
* 車両詳細ページ: 13万ページ超（全在庫）
* 駐車場ページ: 約8,000ページ
* ブログ記事: 増加中

合計で**14万ページ以上**がGoogleにインデックスされています。

ただし、大量ページをインデックスさせるにはサイトマップの整備が必須でした。

**踏んだ罠: サイトマップの分割が必要だった**

駐車場データ（約8,000件）のサイトマップを1ファイルにまとめたら、Google Search Consoleで「取得できません」エラー。ファイルサイズが大きすぎてGoogleがタイムアウトしていました。

10,000件ごとに分割生成することで解決。

```
// sitemap-parking の分割生成
$parkings = Parking::select('id', 'updated_at')->get();
$chunks = $parkings->chunk(10000);

foreach ($chunks as $index => $chunk) {
    $filename = "sitemap-parking-" . ($index + 1) . ".xml";
    // ... 各ファイル生成
}
```

### Zenn / Qiitaからのバックリンク戦略

**平日夜に技術記事を書いて、記事内にMotoHubへのリンクを自然に含める。** これが一番効果のあったSEO施策です。

現在のバックリンク: **Zenn・Qiitaから合計約10本**

書いた記事のテーマ：

* Laravel + Meilisearchで全文検索
* WebARでバイク駐車場を可視化
* バイクの車種判定AIをLaravelで実装
* Cloudflareのbot対策
* ブログ機能のMarkdownハイブリッドレンダリング

**ポイント**: 「MotoHubの宣伝記事」ではなく「技術記事の中で自然にMotoHubに言及する」。Zennの読者はエンジニアなので、宣伝臭がすると読まれない。開発で得た実践的な知見を書くのが大事。

### Cloudflare WAFでbot遮断

Google Search Consoleを見ていたら、アクティブユーザー数の割にPVが異常に多い時期がありました。

調べるとYandexBot、SemrushBot、AhrefsBotなどのクローラーが大量にアクセスしていました。

```
// Cloudflare WAFカスタムルール
(http.user_agent contains "YandexBot") or
(http.user_agent contains "SemrushBot") or
(http.user_agent contains "AhrefsBot") or
(http.user_agent contains "DotBot")
→ Block
```

ブロック後、Googleの順位への影響はゼロ。サーバー負荷だけが軽くなりました。

### パーツ価格比較

楽天・Yahoo Shopping・Amazonから横断検索。JAN コード（商品の固有番号）やメーカー品番を抽出して高精度マッチング。

[![image.png](https://qiita-user-contents.imgix.net/https%3A%2F%2Fqiita-image-store.s3.ap-northeast-1.amazonaws.com%2F0%2F3669654%2F1371fef0-6554-4277-9d16-7bbc28691bb0.png?ixlib=rb-4.0.0&auto=format&gif-q=60&q=75&s=d5c25f7add49d5d6d2f84c5abf0d77c2)](https://qiita-user-contents.imgix.net/https%3A%2F%2Fqiita-image-store.s3.ap-northeast-1.amazonaws.com%2F0%2F3669654%2F1371fef0-6554-4277-9d16-7bbc28691bb0.png?ixlib=rb-4.0.0&auto=format&gif-q=60&q=75&s=d5c25f7add49d5d6d2f84c5abf0d77c2)

* 楽天: `itemCaption` からJANコードを正規表現で抽出（約60%の商品に含まれる）
* Yahoo: `janCode` フィールドを直接利用（ほぼ100%）
* Amazon: PA-API利用（要売上3件）→ 未達成のため検索リンク方式でフォールバック

初期表示が空のページだったのが「寂しい」というフィードバックを受けて、人気カテゴリ（マフラー・タイヤ・チェーン）のプリロード商品を表示するように改善。Rakuten APIの`genreId`でバイク用品ジャンルに絞り込むことで、関係ない商品が表示される問題も解決しました。

### WebAR駐車場ファインダー

Three.js + WebRTC + Geolocation + DeviceOrientationを組み合わせて、スマホのカメラ越しに近くのバイク駐車場をAR表示する機能。

技術的にはかなり苦労しました（磁気センサーのキャリブレーション、Safari対応、WMM磁気モデルデータの期限切れ等）。

### 値下げ・新着通知（Web Push）

気になる車種を登録すると、毎朝8:30に値下げ・新着情報をプッシュ通知で受け取れる機能。リテンション施策です。

### AI相性診断

5つの質問に答えると、MotoHubのDBからあなたにぴったりのバイクを提案。結果をXでシェアできるようにして、ソーシャルからの流入も生まれました。

## 開発効率を支える技術

### Claude Code — これなしでは3ヶ月で作れなかった

本業ありの個人開発で最も生産性に貢献しているのが**Claude Code**（Anthropicの CLI型AIコーディングツール）です。

ワークフロー：

1. **claude.ai（チャット）で設計・アーキテクチャ決定** → 大きな方針を議論
2. **Claude Code（CLI）で実装** → 具体的なコード生成・ファイル作成・テストを自動実行

実例を出すと、つい昨日（2026/3/29）、1日で以下の改善を全てデプロイしました：

* 診断ページに戻るボタン＋進捗バー修正
* ヘッダー検索バーをアイコン展開式にリデザイン
* パーツ検索に人気カテゴリチップ＋プリロード商品表示
* 買取相場の車種プルダウンを軽量API化（1秒→0.1秒）
* ブログ見出しの不要な`#`記号を修正
* LINE通知バナーに閉じるボタン＋7日間非表示機能
* 通知ボタンに説明文追加
* 埋め込みウィジェットにユースケース説明＋デモプレビュー
* 比較ページに使い方ガイド追加
* 比較・お気に入りボタンのラベル統一
* DB文字化け修正

**平日夜の3〜4時間で11個の改善。** Claude Codeなしでは1〜2個が限界だったと思います。

## うまくいかなかったこと

失敗も正直に書きます。

### Amazon PA-API の売上条件

Amazon PA-APIを使うには「過去30日間に3件以上の売上」が必要。MotoHubはまだアフィリエイト売上が少なく、API未開放のまま。検索リンクでフォールバック対応中。

### WebARの磁気センサー問題

WMM（World Magnetic Model）の磁気モデルデータは5年ごとに更新。MotoHubで使っていたWMM 2020データは2025年末に期限切れ。更新が必要なことに気づくのが遅れました。

### EUC-JP文字化けの残骸

外部からインポートしたバイクスペックデータの一部にEUC-JP→UTF-8変換の失敗が混入。エンジン種類カラムにキリル文字のような文字化けが表示される車種がありました。ユーザーからの指摘で発覚。

**教訓**: データインポート時にエンコーディング検証は必須。

### 夜中の判断ミス

本業の後に疲れた状態で開発すると、判断ミスが起きます。実際に、本番DBのマイグレーションでIDをハードコードしてしまい、本番と開発でIDが一致しないことに気づかず適用してしまったことがありました。

**教訓**: 疲れてる時にDBマイグレーションを本番実行しない。名前ベースで対象レコードを特定する。
