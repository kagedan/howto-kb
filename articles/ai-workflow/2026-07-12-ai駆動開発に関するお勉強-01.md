### 開発環境、どうしよう状態
- Geminiに聞いてみた
    - https://share.gemini.google/nesPsn1N2Sik
- 上記、ざっくり概要
    - 最適な環境構成: 個人での学習および「型の確立」には、「VS Code ＋ Cline（拡張機能） ＋ Anthropic API（Claude）」の組み合わせがベスト。
    - コスト面（圧倒的な安さと安全性）:
        - 月額サブスク（$20〜）と違い、完全従量課金（使わない月は0円）のため、個人開発レベルなら月数百円〜で収まるケースが多い。
        - Anthropicのコンソールで「月の上限（例：$10）」を設定できるため、会社の共通クレジットのようなバースト（過剰請求）の心配が100%ない
        - 新世代のClaude（4系・5系）になっても公式のトークン単価（卸値）は据え置きであるため、常に最新の頭脳を最安値で使える。
    - 性能面（実務に直結する「型」の習得）:
        - 世界最高水準のベンチマーク（SWE-bench等）で実証されている「Claude」の自律開発力（エージェント機能）をフルに体験できる。
        - 単なるコード補完ではなく、「仕様策定 ➔ タスク分解 ➔ 1タスクずつAIが実行し、人間が承認する」という、これからの標準となる開発フローが身につく。
    - 将来性と横展開:
        - プラットフォームが「VS Code」のままなので、ここで掴んだ「AIへの指示出しや手戻り防止のテンプレ」は、会社のVS Code ＋ Copilot（Claude/GPT切り替え）環境へそのまま横展開（移植）が可能。
- **結論：vscode + cline + claude でやってみよう**

## まずは環境構築
1. Claude ConsoleでAPIキー発行
    - クレジットはチキンなのでいったん10$で様子見
    - 自動リロードはOFFのまま（SKIP）
    - キー名は適当に「vscode-claude」で
    - 有効期限はひとまずで30日を設定
    - 「請求」から「月間支出上限」を設定
        - これまたチキンなので20$に設定
1. VSCodeにclineを入れる
    - 上記で発行したキーを使うので「Bring my own API key」を指定
    - Configure画面が出るので以下を設定
        - API Provider: Anthropic
        - Anthropic API Key: 前の手順で作成したもの
        - Model: Claude-sonnet-5
        - Adaptive Thinking: 適応型思考、ひとまずNoneで様子見
        - Use different models for Plan and Act modes: Plan/Actを分ける予定は今のところないのでOFF
1. 動作確認で適当にフォルダ切らせてみる（プロジェクト名も考えさせつつ）
