## はじめに ─ 初級と中級の境界線

初級編では「とにかく使ってみる」ことを目標にしました。

中級編のテーマは **「意図した通りに動かす」** こと。

Claudeはプロンプトの書き方ひとつで、回答の質が劇的に変わります。この記事では、プロの現場でも使われる設計の考え方を、できるだけわかりやすく解説します。

## 第1章：プロンプト設計の5つの技術

![](https://assets.st-note.com/img/1778892204-ky8ca5PIOluV27JEvgeLZxzQ.png?width=1200)

### 1-1. 技術① 構造化プロンプト

情報を「見出し」で整理して渡すと、Claudeの理解精度が上がります。

❌ 整理されていない例

```
田中さんというお客様がいて先月から問い合わせが多くて
クレームっぽいんですけど返信メールを書いてほしいです
```

✅ 構造化した例

```
## 役割
あなたはカスタマーサポートの専門家です。

## 状況
- お客様：田中様（先月から複数回お問い合わせ）
- 内容：商品の配送遅延に対するクレーム
- 現状：配送会社の遅延が原因、翌日発送予定

## 依頼
以下の条件で返信メールを作成してください。
- 文体：丁寧・誠実・謝罪しすぎない
- 長さ：200文字以内
- 件名も含めること
```

**ポイント：** ## や箇条書きで情報を分けるだけで、Claudeが「何を求められているか」を正確に把握できます。

---

### 1-2. 技術② ネガティブ指示（やらないことを伝える）

「〜してください」と同じくらい、「〜しないでください」が効きます。

```
商品紹介文を書いてください。

【やらないこと】
- 「最高」「業界No.1」などの誇張表現は使わない
- 競合他社との比較は書かない
- 3文以上の長い文章にしない

【商品情報】
オーガニック素材の保湿クリーム、敏感肌向け、無香料
```

---

### 1-3. 技術③ 出力例を見せる（Few-shot）

「こういう形式で出力してほしい」という例を1〜2個見せると、精度が大幅に上がります。

```
以下のフォーマットで、商品レビューの要約を作ってください。

【出力例】
商品名：〇〇クリーム
評価：★4.2
一言：敏感肌でも使いやすい保湿クリーム。香りがないのがいい。
改善点：容量がもう少し多いと嬉しい。

【レビュー本文】
（ここにレビューを貼り付け）
```

---

### 1-4. 技術④ ステップ思考（Chain of Thought）

複雑な問題は「考える手順」を指示することで、精度が上がります。

```
以下の問題を解いてください。
答えを出す前に、必ず考え方のステップを書いてから結論を出してください。

【問題】
月商300万円のECサイトで、広告費を20%削減した場合、
売上への影響はどう考えられますか？
```

「ステップを書いてから結論を」という一言だけで、論理的な分析が返ってきます。

---

### 1-5. 技術⑤ 反復改善（イテレーション）

1回のプロンプトで完璧を目指さず、会話を重ねながら精度を上げる方法です。

```
【ステップ1】
まず構成案だけ作ってください（本文はまだ不要）

↓ 構成を確認してOKなら

【ステップ2】
第1章の本文を書いてください

↓ 内容を確認して修正指示を出す

【ステップ3】
第2段落をもう少し具体例を増やして書き直してください
```

## 第2章：Projectsで「記憶」を作る

![](https://assets.st-note.com/img/1778892273-rWcB6aPiC4LKxjAghkOfND8z.png?width=1200)

### 2-1. Projectsとは何か？

通常のChaudeは、**チャットを閉じると会話内容を忘れます。**

Projects（プロジェクト）を使うと、Claudeに**長期的な記憶と指示**を持たせることができます。

### 2-2. Projectsでできること

機能 説明 カスタム指示 「常にタメ語で返す」「私の会社名はXXX」など固定の指示を設定 ファイルの保存 資料・マニュアル・データをプロジェクト内に保存 会話の継続 同じプロジェクト内なら文脈を引き継いで会話できる チーム共有 Teamプランではメンバーと共有可能

### 2-3. Projectsの設定例

**ライティング業務用プロジェクトの設定例：**

```
【カスタム指示】
あなたは私の専属コピーライターです。

以下を常に守ってください：
- 文体：カジュアルすぎず、固すぎない「ていねい語」
- ターゲット：30〜40代の働く女性
- NG表現：「革命的」「唯一無二」などの大げさな表現
- 文章の長さ：1文を60字以内に

【私のブランド情報】
ブランド名：〇〇
コンセプト：〇〇
メインカラー：〇〇
```

このカスタム指示を設定しておくと、毎回「〜という設定で」と書く必要がなくなります。

### 2-4. メモリ機能（自動記憶）

設定でメモリ機能をオンにすると、Claudeが重要な情報を自動で記憶します。

```
設定 → メモリ → オンにする
```

例えば「私はフリーランスのデザイナーです」と伝えると、次回以降の会話でもその情報を踏まえた回答をしてくれます。

## 第3章：Artifactsで成果物を直接出力する

![](https://assets.st-note.com/img/1778892320-XOIRylmrWGjpdB5eoga2w6VM.png?width=1200)

### 3-1. Artifactsとは？

Claudeが**ファイルや成果物を直接作成する機能**です。

チャット画面の右側にプレビューが表示され、そのままコピー・ダウンロードできます。

### 3-2. Artifactsで作れるもの

種類 具体例 Markdown文書 ブログ記事・議事録・仕様書 HTMLページ ランディングページ・メール本文 Reactコンポーネント インタラクティブなUI部品 SVG図解 フローチャート・構成図 コード Python・JavaScript・SQLなど 表・データ CSV形式のデータ整理

### 3-3. Artifactsを呼び出すプロンプト

```
以下の内容で、すぐに使えるHTMLメールテンプレートを
Artifactとして作成してください。

【条件】
- 件名：〇〇キャンペーンのお知らせ
- 構成：ヘッダー画像エリア・本文・CTAボタン・フッター
- デザイン：シンプル・白ベース・ボタンは青
```

## 第4章：ウェブ検索・ファイル読み込みの組み合わせ

### 4-1. ウェブ検索機能（Pro以上）

Claudeは基本的に2025年8月頃までの情報しか知りません。

ウェブ検索をオンにすると、**リアルタイムの情報**を取得できます。

**活用例：**

```
【ウェブ検索オン】
2025年の最新のSEOトレンドを調べて、
私のブログ記事の改善点を3つ提案してください。

【ブログ記事URL または本文】
〇〇〇〇
```

### 4-2. ファイルのアップロード

PDFや画像・ExcelなどをClaudeに読み込ませることができます。

**PDFを要約する：**

```
添付のPDFを読んで、以下の形式でまとめてください。

① 全体の要旨（3行）
② 重要なデータ・数字（箇条書き）
③ 私が取るべきアクション（箇条書き）
```

**画像を分析する：**

```
この画像を見て、以下を教えてください。
- グラフが示している傾向
- 注目すべきポイント
- 改善できる点
```

### 4-3. 組み合わせ技：ウェブ検索＋ファイル

```
添付の競合分析レポートを読んだうえで、
ウェブ検索で最新の業界動向も調べてください。
その両方を踏まえて、来期の差別化戦略を提案してください。
```

自分のデータ＋最新情報の組み合わせが、Claudeの真価を引き出します。

## 第5章：中級者のための実践プロンプト集

### 5-1. 長文コンテンツ作成（構造化）

```
## 役割
あなたはSEOに詳しいコンテンツライターです。

## 記事の概要
タイトル：「フリーランスが最初の1年で失敗しないための5つの心得」
ターゲット：会社員から独立を考えている20〜30代
文字数：2,000文字
トーン：親しみやすく、経験談っぽく

## 構成（この順番で書くこと）
1. 導入（読者の悩みに共感）
2. 心得①〜⑤（それぞれ見出し＋200文字の解説）
3. まとめ（行動を促す締め）

## 禁止事項
- 「〜でしょう」「〜かもしれません」などの弱い表現は使わない
- 実例を最低1つ各項目に入れること
```

### 5-2. データ分析・考察

```
以下の売上データを分析してください。

【手順】
① まずデータの全体傾向を1段落で説明する
② 特徴的な数値を3つピックアップして理由を考察する
③ 次のアクションとして考えられることを優先度つきで3つ提案する

【データ】
（Excelやテキストデータを貼り付け）
```

### 5-3. フィードバック・添削

```
以下の文章を厳しめに添削してください。

【評価基準】
A. 論理の一貫性（1〜5点）
B. 読みやすさ（1〜5点）
C. 説得力（1〜5点）

【出力形式】
- 各項目の点数と理由
- 具体的な改善箇所（引用して指摘）
- 修正後の文章例

【文章】
（ここに貼り付け）
```
