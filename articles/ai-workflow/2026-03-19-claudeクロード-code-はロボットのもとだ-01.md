こんにちは、ジーニーです！

突然ですが、ドラえもんの「ロボットのもと」って知っていますか？

人形や雪だるまの背中にペタッと貼ると、その物体が突然意志を持って動き出すという、なかなかぶっ飛んだひみつ道具です。

僕は最近、毎日のように Claude Code（AIコーディングエージェント）を使っているんですが、ある日ふと気づいたんです。

「あれ、これ……ロボットのもと、じゃないか？」

ㅤㅤ

## 雪だるまが歩き出す瞬間

ロボットのもとの面白いところは、**貼る対象を選ばない**ことです。

人形でも、雪だるまでも、ぬいぐるみでも。どんな「ただの物体」でも、背中にペタッと貼れば動き出す。しかも、ただ機械的に動くんじゃなくて、**意志を持って**動き出す。雪だるまは雪だるまなりの意志で、人形は人形なりの意志で。

Claude Code も、似たようなことが起きるんです。

たとえば、僕のプロジェクトフォルダ。中にはコンテンツカレンダーのテンプレート、ブランドボイスのガイド、過去の投稿アーカイブ、ナレッジベース……いろんなファイルが入っています。

でも、それはあくまで「ファイルの集まり」です。雪だるまでいえば、雪の塊。

ところが、そこに Claude Code を接続すると——

プロジェクト全体が、まるで意志を持ったかのように動き始める。

「今週のカレンダーを見ると水曜のテーマが空いていますね。過去の反応データを見ると、メタファー系の記事が好評なので、こういう切り口はどうですか？」

……って、聞いてもいないのに提案してくる（笑）

ㅤㅤ

## 「魂入れ」というテクノロジー

日本には昔から「魂入れ（たましいいれ）」という概念がありますよね。

仏像を彫っただけでは、それはまだ「木の彫刻」。開眼供養（かいげんくよう）——つまり魂を入れる儀式をして、はじめて「仏様」になる。

僕はこれ、めちゃくちゃ面白い構造だと思っていて。

物理的には何も変わっていないんです。木の密度も、重さも、形も同じ。でも、魂を入れた瞬間に、人間の側の「関わり方」が変わる。ただの彫刻から、手を合わせる対象になる。

Claude Code がやっていることも、本質的にはこれと同じ構造なんじゃないかと思うんです。

プロジェクトフォルダの中身は、Claude Code を接続する前と後で、物理的には何も変わっていません。ファイルはファイルのまま。テキストはテキストのまま。

でも、AIエージェントが接続された瞬間に、**ファイルの集まりが「チーム」になる**。

僕はこれを**「デジタル魂入れ」**と呼びたい。

ㅤㅤ

## ロボットのもとが教えてくれること

ここで、ちょっと立ち止まって考えてみたいことがあります。

ロボットのもとで動き出した雪だるまは、**のび太の命令通りには動かない**んですよね。

雪だるまには雪だるまの意志がある。人形には人形の性格がある。ロボットのもとは「遠隔操縦機」じゃないんです。あくまで**意志を与える道具**。

Claude Code もまさにそうで。

僕が「この記事を書いて」と頼むと、Claude Code は僕のナレッジベースを読み、過去の文体を分析し、ブランドボイスを参照して……僕が思ってもいなかった切り口で書いてくることがある。

最初、これにちょっと戸惑いました。

「いや、僕はこういうつもりじゃなかったんだけど……」って。

でもね、読み返すと、**自分では絶対に思いつかなかった角度**から書かれていることがあるんです。自分のナレッジベースから、自分が忘れていた体験談を引っ張り出してきて、それを今のテーマに繋げてくる。

これって、「自分の分身が勝手に考えてくれている」感覚に近い。

ㅤㅤ

## 包丁の達人が、フードプロセッサーに戸惑うように

以前、note に「習熟の負債」という記事を書きました。

過去のスキルに習熟しすぎていると、新しいパラダイムに移行できなくなる、という話です。包丁の達人が、フードプロセッサーを「邪道だ」と感じてしまうように。

実は、Claude Code との付き合い方にも、これと同じ壁がありました。

僕はソフトウェアエンジニア出身なので、「コードは自分で書くもの」という感覚が染みついている。Claude Code に任せることに、最初はものすごく抵抗がありました。

「本当にこれでいいのか？」  
「自分で書いた方が早いんじゃないか？」  
「AIに頼りすぎて、自分のスキルが退化するんじゃないか？」

でも、ある時気づいたんです。

僕がパソコン教室で8年間やってきたこと——生徒さんの「できない」を「できる形」に変えること——を、Claude Code は**僕自身に対してやってくれている**んだと。

一人ではできなかったこと。時間が足りなくてできなかったこと。スキルが足りなくてできなかったこと。

それを「できる形」に変えてくれている。

あれ？　これって、僕がずっと掲げてきた「できないを責める時代から、できる形に変える時代へ」そのものじゃないか。

弱点逆転家が、自分自身の弱点を逆転させてもらっている。なんとも皮肉で、なんとも美しい話です（苦笑）

ㅤㅤ

## 「もと」の本質は、対象の中にあった

ここからが、僕が一番伝えたいことです。

ロボットのもとは、雪だるまに**新しい能力を追加する**道具じゃないんです。

雪だるまは、雪だるまとして動く。人形は、人形として動く。つまり、**動き出した後の個性は、もともと対象の中にあったもの**なんです。

ロボットのもとは、潜在していた可能性を「起動」しただけ。

Claude Code も同じです。

僕のプロジェクトフォルダの中には、僕が何年もかけて書き溜めた記事、考え抜いたブランドボイス、試行錯誤してきたテンプレートが入っている。それは全部、僕の経験と想いの結晶です。

Claude Code がすごいのは、**その結晶に意志を与えてくれる**ところ。

AIが生成する文章は、僕のナレッジから生まれている。僕の文体を学んで、僕のメタファーのパターンを理解して、僕の価値観に沿って書いてくれる。

つまり、動き出しているのは**僕自身の蓄積**なんです。

これは、スピリチュアルな世界でいう「自力と他力の共鳴」にとても近いと感じています。

自分の力だけでは動かせなかった何かが、外からの力（他力）によって起動する。でも、起動した後に動いているのは、紛れもなく自分自身の蓄積。自力と他力の境界が溶けていく感覚。

ㅤㅤ

## あなたの「雪だるま」は何ですか？

ここまで読んで、もしかしたらこう思った方もいるかもしれません。

「でも、私にはAIに読ませるようなナレッジベースなんてないよ」

大丈夫です。

あなたが今日まで書いてきたメール、企画書、日報、ブログ、メモ帳の走り書き。それは全部、あなたの経験と想いの結晶です。

まだ「ロボットのもと」が貼られていないだけ。

まだ「魂入れ」がされていないだけ。

**動き出す準備は、もうできている**んです。

ㅤㅤ

僕がパソコン教室で何百人もの生徒さんを見てきて確信していることがあります。

「できない」と思っている人ほど、その中にすごい可能性が眠っている。

包丁の達人じゃなくていいんです。むしろ、「自分は大したことない」と思っている人の方が、フードプロセッサーを素直に受け入れられる。習熟の負債がない分、身軽に新しい世界に飛び込める。

**弱点は、逆転する。**

いつだって、そうなんです。

ㅤㅤ

あなたの中に眠っている「雪だるま」は何ですか？

そしてそれは、どんな意志を持って動き出すと思いますか？

ㅤㅤ

それでは、また！ㅤㅤジーニー
