週次報告の作成、毎回30分以上かかっていませんか？

「今週やったことをまとめなきゃ」と思いながら、何から書けばいいか悩む。結局、締切直前に慌ててまとめる。そんな繰り返しが続いていませんか。

AIに今週の業務を箇条書きで投げるだけで、読みやすい報告書の下書きが10分以内に揃います。この記事では、週次報告×AIの使い方をプロンプトつきで解説します。

### 週次報告が「後回し」になる3つの理由

週次報告は重要だとわかっていても、毎回後回しになりやすい理由があります。

**理由1：何から書くか毎回一から考えるから**

「成果」「来週の予定」「課題」など、毎回同じ構成で書くのに、なぜか毎回最初から考えてしまいます。フォーマットが決まっていないと、テンプレートを作る手間から始まります。

**理由2：成果をうまく言語化できないから**

「〇〇を進めた」「△△の会議に出た」という記録はあっても、それを「成果」として言葉にするのが難しいと感じることがあります。やったことと、それが持つ意味は別物です。

**理由3：「重要な話」と「細かい話」の区別がつかないから**

今週あったことを全部書くと長くなる。でも短くすると「伝わっているか」が不安になる。この判断に時間がかかります。

AIを使うと、この3つをまとめて解決できます。

### ステップ1：今週の業務をAIに渡して下書きを作る

最初のステップは、今週の業務を箇条書きでAIに渡すだけです。

```
以下は今週の業務記録です。週次報告書の形式に整えてください。

【宛先】：上司・マネージャー
【フォーマット】：
・今週の成果（3〜5点）
・来週の予定（3〜5点）
・相談・共有事項（あれば）

今週の業務（箇条書きで入力する）：
【ここに入力する】
```

「今週やったこと」をメモ程度に箇条書きしたものを貼り付けるだけでOKです。

AIが構成を整えて、報告書らしい文章に変換してくれます。「やったこと」を「成果」の言葉に置き換えてくれるのが一番の時短ポイントです。

**なぜ効くのか**：「書く作業」と「整理する作業」を分けることで、ゼロから考える必要がなくなります。まずメモを書いて、AIに整形させる。このステップを分けるだけで、作業時間が大幅に変わります。

### ステップ2：成果を「読みやすい表現」に変換する

下書きができたら、次は成果の部分を磨きます。

```
以下の週次報告の「今週の成果」の部分を、より具体的・定量的な表現に変えてください。
数字が入っていない項目には「約〇件」「〇割削減」など、補完した提案も入れてください。

現在の成果セクション：
【コピーして貼り付ける】
```

「営業活動を行った」→「新規顧客への提案を4件実施、うち1件が次回商談へ進展」のように、具体性が上がります。

**なぜ効くのか**：上司が週次報告に求めているのは「何が起きているか」の把握です。抽象的な表現より、数字や具体的な動きが見えた方が読みやすくなります。AIは「より具体的に」という変換が得意です。

### ステップ3：上司の視点で「伝わるか」を確認する

報告書を仕上げたら、提出前にAIに読んでもらいます。

```
以下の週次報告書を、多忙な上司の視点でレビューしてください。

・30秒で「何が起きているか」が把握できますか？
・「次のアクション」が明確ですか？
・見づらい・わかりにくい部分があれば指摘してください。

報告書：
【コピーして貼り付ける】
```

「この文は2つの内容が混ざっています」「来週の予定がないと、何を求めているかわかりません」のようなフィードバックが返ってきます。

**なぜ効くのか**：自分で書いた文章は自分では気づきにくい。AIを「第一読者」として使うことで、提出前に穴を見つけられます。

### ステップ4：毎週使えるテンプレートをAIに作ってもらう

毎回の入力を減らすために、自分専用のテンプレートを作ります。

```
私は【あなたの職種・役割を入れる】です。毎週の週次報告書テンプレートを作ってください。

・読み手：【例：直属の上司、プロジェクトリーダー】
・重視するポイント：【例：進捗と課題を中心に報告したい】
・分量の目安：【例：A4半ページ以内】

繰り返し使えるテンプレートを出力してください。
```

テンプレートを一度作ると、毎週はそこに書き込むだけになります。「何を書くか」ではなく「どれだけ進んだか」だけを考えればよくなります。

**なぜ効くのか**：週次報告に毎回時間がかかる最大の理由は「フォーマットを考える手間」です。テンプレートを一度作ると、次週から入力作業のみになります。

### ステップ5：英語・他言語の報告書にも対応する

外資系や海外部門がある職場では、英語での報告が必要な場合があります。

```
以下の週次報告書を英語に翻訳してください。
ビジネスレポートとして自然な表現にしてください。

【日本語報告書をここに貼る】
```

日本語で書いた報告書を貼り付けるだけで、英語版が出力されます。細かい表現の確認は必要ですが、ゼロから英語で書くよりはるかに速くなります。

**なぜ効くのか**：英語報告書は「英語で書く作業」と「内容を考える作業」が混ざって時間がかかります。日本語で内容を作ってからAIで翻訳する順序に変えるだけで、全体の作業時間が下がります。

### よくある失敗例

**失敗1：AIに「今週やったこと全部」を雑然と渡す**

「会議出た、資料作った、メール返した」のような雑な入力では、雑な出力しか返ってきません。

5〜10行の箇条書きに整理してから渡すのが基本です。入力を整えると、出力の質が上がります。

**失敗2：AIの出力をそのまま提出する**

AIは自分の業務の文脈を知りません。プロジェクト名の誤記・数字の補完ミス・部署固有の言葉の使い方などは、自分で必ず確認してから提出しましょう。

あくまで「下書き生成ツール」として使い、最終チェックは自分で行うのが正しい使い方です。

### よくある質問

**Q：ChatGPTとClaude、週次報告に向いているのはどちらですか？**

どちらも同程度に使えます。長い業務記録を渡すときはClaudeの方が整理が丁寧な印象です。使い慣れている方を選んで問題ありません。

**Q：毎週同じプロンプトで飽きてきたときはどうすれば？**

「先週と比較して今週の変化を強調して」「課題の部分を箇条書きではなく説明文にして」など、毎週の状況に合わせて一言変えるだけでアウトプットが変わります。

**Q：上司から「もっと具体的に」と言われることが多いのですが改善できますか？**

ステップ2の「定量的な表現に変換する」プロンプトが効果的です。「具体的に」という指摘は多くの場合「数字がない」「動きが見えない」という意味です。AIで数字の補完・具体的な表現への変換を試してみてください。

## まとめ

週次報告が後回しになるのは、「毎回ゼロから考えているから」です。

AIを使った5つのステップのまとめです。

1. **下書き作成**：業務の箇条書きを渡して報告書形式に整えてもらう
2. **成果の言語化**：定量的・具体的な表現に変換してもらう
3. **第一読者レビュー**：上司視点で伝わるかを確認してもらう
4. **テンプレート作成**：一度作れば毎週の入力だけで済む
5. **英語対応**：日本語で作ってから翻訳する順序に変える

まず今日、来週の週次報告の前日にステップ1のプロンプトを試してみてください。
