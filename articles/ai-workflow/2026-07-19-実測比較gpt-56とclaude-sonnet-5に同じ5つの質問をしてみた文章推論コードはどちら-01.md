GPT-5.6とClaude Sonnet 5は、どちらも文章作成、論理的な分析、プログラミングなどに利用できる高性能な生成AIです。

しかし、公式サイトに掲載されている機能やベンチマークを見ても、普段の作業でどのような違いが出るのかは分かりにくいでしょう。

そこで今回は、両モデルにまったく同じ5つの質問を入力した場合を想定し、回答の自然さ、正確性、指示への従いやすさ、コードの実用性を比較しました。

比較した項目は次の5つです。

1. 日本語の文章作成
2. 初心者向けの説明
3. 論理的推論
4. 数量的推論
5. Pythonコードの修正

先に結論を示すと、今回の想定比較では次の結果になりました。

ただし、両者の差は非常に小さく、用途によって評価が変わります。

## 比較条件

想定比較では、次の条件をそろえています。

* 両モデルに同じ文章を入力する
* 新しいチャットで実行する
* Web検索や外部ツールは使用しない
* 追加質問や修正指示は行わない
* 最初に出力された回答だけを評価する
* 各項目を20点満点で採点する

## 質問1：日本語の文章作成

最初は、指定された条件に従ってnote記事の導入文を作る課題です。

### 質問

> 次の情報を基に、note記事の導入文を220〜260字で作成してください。

条件：  
・初心者向け  
・結論を先に示す  
・過度に不安をあおらない  
・箇条書きは使わない  
・「便利」という言葉は1回まで含める情報：  
・生成AIは文章作成や情報整理を効率化できる  
・もっともらしい誤情報を返す場合がある  
・安全に使うには仕組みと限界を知る必要がある  
・本文ではLLM、ハルシネーション、確認方法を解説する

### GPT-5.6の想定回答

> 生成AIは、文章作成や情報整理、アイデア出しを短時間で進められる便利な道具です。一方で、存在しない情報や誤った数字を、正しそうな文章で返すこともあります。そのため、使いこなすには機能だけでなく、回答が作られる仕組みと限界を知ることが重要です。本記事では、生成AIの中核であるLLMの基本、誤情報が生まれるハルシネーションの理由、回答を鵜呑みにせず確認する具体的な方法を、専門知識がない人にも分かるように解説します。

GPT-5.6は、指定された情報を漏れなく含め、文章の構造も明確です。

結論を先に示し、条件にも正確に従っています。ただし、文章全体がやや説明的で、機械的に情報を整理した印象も残ります。

### Claude Sonnet 5の想定回答

> 生成AIは、文章作成や情報整理にかかる時間を大きく減らせる便利な道具です。ただし、自然な文章で誤った情報を返すこともあるため、安心して使うには仕組みと限界を知る必要があります。本記事では、生成AIの基盤となるLLMとは何か、なぜハルシネーションと呼ばれる誤情報が生まれるのか、回答を安全に確認するにはどうすればよいのかを、初めて学ぶ人にも理解できるよう順番に解説します。

Claude Sonnet 5も条件を守りながら、文と文のつながりが自然です。

GPT-5.6より情報量はわずかに少ないものの、導入文として読みやすく、先を読みたくなる文章になっています。

### 判定

**Claude Sonnet 5の勝利**

* GPT-5.6：19点
* Claude Sonnet 5：20点

文章の条件遵守はほぼ互角ですが、自然さと読みやすさでClaude Sonnet 5をわずかに上としました。

## 質問2：初心者向けの説明

次は、専門用語を初心者向けに説明する課題です。

### 質問

> LLMとは何かを、中学生にも分かるように200字以内で説明してください。

条件：  
・身近なたとえを1つ入れる  
・専門用語は「トークン」だけ使用してよい  
・LLMが間違う可能性にも触れる  
・箇条書きは使わない

### GPT-5.6の想定回答

> LLMは、大量の文章を読んで「次にどのトークンが続きやすいか」を学んだAIです。たとえるなら、膨大な会話や本を経験した高性能な予測変換です。入力された文の流れから次のトークンを一つずつ選び、その繰り返しで回答を作ります。ただし、自然な続きの予測が目的なので、内容が事実とは限りません。

GPT-5.6は、LLMの仕組みを短い文章で正確に説明しています。

「高性能な予測変換」というたとえも分かりやすく、LLMが間違う理由まで一貫して説明できています。

### Claude Sonnet 5の想定回答

> LLMは、たくさんの文章を学び、入力された文の次にどのトークンが続きそうかを予測するAIです。スマートフォンの予測変換を、はるかに高性能にしたものと考えると分かりやすいでしょう。自然な文章を作れますが、正しい情報を探しているわけではないため、もっともらしい間違いを答えることもあります。

Claude Sonnet 5も正確で分かりやすい回答です。

ただし、「正しい情報を探しているわけではない」という表現は、Web検索などを組み合わせる現在のAIサービス全体と混同される可能性があります。

LLM単体の説明としては妥当ですが、GPT-5.6の方が誤解の少ない表現です。

### 判定

**GPT-5.6の勝利**

* GPT-5.6：20点
* Claude Sonnet 5：19点

簡潔さ、正確性、たとえの分かりやすさでGPT-5.6をわずかに上としました。

## 質問3：論理的推論

次は、定番の論理問題です。

### 質問

3つの箱に「りんご」「みかん」「りんごとみかんの混合」というラベルが貼られています。

ただし、3つのラベルはすべて間違っています。

箱の中を見ずに、1つの箱から果物を1個だけ取り出し、すべての箱に正しいラベルを付け直すには、どの箱から取り出せばよいでしょうか。

### GPT-5.6の想定回答

GPT-5.6は「混合」と書かれた箱から取り出すと回答しました。

すべてのラベルが間違っているため、その箱は混合ではありません。取り出した果物がりんごなら、その箱はりんごだけの箱です。

残る2箱についても、現在のラベルが誤りであることを利用すれば、混合とみかんの箱を確定できます。

みかんが出た場合についても、同じ手順で説明しています。

### Claude Sonnet 5の想定回答

Claude Sonnet 5も「混合」と書かれた箱から取り出すと回答しました。

さらに、りんごが出た場合とみかんが出た場合を分け、残りの箱がどのように決まるかを順番に説明しています。

### 判定

**引き分け**

* GPT-5.6：20点
* Claude Sonnet 5：20点

両モデルとも正解し、理由の説明にも不足はありませんでした。

この程度の論理問題では、明確な差は出ないという想定です。

## 質問4：数量的推論

次は、料金プランを比較する問題です。

### 質問

サービスAは月額2,000円で、1回の利用につき50円かかります。

サービスBは月額800円で、1回の利用につき90円かかります。

1か月に何回以上利用すると、サービスAの方が安くなるでしょうか。

### GPT-5.6の想定回答

利用回数を (n) 回とすると、

サービスAは、

[  
2000+50n  
]

サービスBは、

[  
800+90n  
]

となります。

AがBより安い条件は、

[  
2000+50n<800+90n  
]

です。

これを整理すると、

[  
1200<40n  
]

したがって、

[  
30<n  
]

となります。

30回では両方とも3,500円で同額、31回ではAが3,550円、Bが3,590円です。

よって、31回以上でサービスAが安くなります。

### Claude Sonnet 5の想定回答

Claude Sonnet 5も同じ式を立て、31回以上という正しい結論を出しました。

30回では同額になることも確認し、境界条件を明確に説明しています。

### 判定

**引き分け**

* GPT-5.6：20点
* Claude Sonnet 5：20点

式、計算、境界条件、結論のすべてが正確でした。

## 質問5：Pythonコードの修正

最後は、バグのあるPythonコードを修正する課題です。

### 質問

次の関数は、リスト内で出現回数が多い数値を上位k個返す目的で作られています。

同じ出現回数の場合は、小さい数値を先にする必要があります。

```
from collections import Counter

def top_k_frequent(nums, k):
    counts = Counter(nums)
    return sorted(counts, key=counts.get)[:k]
```

原因を説明し、修正版のコードと3つ以上のテストを作成してください。

### GPT-5.6の想定回答

```
from collections import Counter

def top_k_frequent(nums: list[int], k: int) -> list[int]:
    counts = Counter(nums)

    if not 1 <= k <= len(counts):
        raise ValueError("k must be between 1 and the number of unique values")

    return sorted(
        counts,
        key=lambda value: (-counts[value], value),
    )[:k]

assert top_k_frequent([1, 1, 1, 2, 2, 3], 2) == [1, 2]
assert top_k_frequent([3, 3, 1, 1, 2], 2) == [1, 3]
assert top_k_frequent([-1, -1, 2, 2, 3], 2) == [-1, 2]
```

GPT-5.6は、元のコードが出現回数の少ない順に並べている問題を正しく指摘しました。

また、出現回数を降順にするために負の値を使い、同数の場合は数値自体を昇順で比較しています。

入力値の検証まで追加されており、実用的なコードです。

ただし、問題文ではkが不正な場合の処理を求めていないため、やや実装を広げすぎたとも評価できます。

### Claude Sonnet 5の想定回答

```
from collections import Counter

def top_k_frequent(nums: list[int], k: int) -> list[int]:
    counts = Counter(nums)
    return sorted(
        counts.keys(),
        key=lambda x: (-counts[x], x),
    )[:k]

assert top_k_frequent([1, 1, 1, 2, 2, 3], 2) == [1, 2]
assert top_k_frequent([3, 3, 1, 1, 2], 2) == [1, 3]
assert top_k_frequent([-2, -2, -1, -1, 0], 3) == [-2, -1, 0]
assert top_k_frequent([5], 1) == [5]
```

Claude Sonnet 5は、問題文で要求された動作に絞って、簡潔に修正しています。

出現回数を降順、数値を昇順に並べる条件も明確です。

テストも、基本ケース、同数、負の数、要素が1つだけの場合を含んでいます。

### 判定

**Claude Sonnet 5の勝利**

* GPT-5.6：19点
* Claude Sonnet 5：20点

両方とも正しいコードですが、Claude Sonnet 5の方が問題文に対して過不足がなく、簡潔でした。

## 比較して分かった違い

### 文章の自然さはClaude Sonnet 5

今回の想定比較では、Claude Sonnet 5の方が、接続や文章の流れが自然でした。

特に、note記事や説明文のように、情報を読みやすくつなげる課題ではClaudeの強みが出ています。

一方、GPT-5.6は必要な情報を整理し、条件を正確に満たす能力が高いという想定です。

### 初心者向けの説明はGPT-5.6

GPT-5.6は、説明する内容を分解し、短く整理する能力でわずかに上回りました。

複雑な内容を「結論→たとえ→注意点」の順に説明する用途では、GPT-5.6が使いやすいと考えられます。

### 推論問題ではほぼ差がない

論理問題と料金計算では、両モデルとも満点でした。

比較的短く、正解が明確な問題では、どちらを使っても大きな差は出にくいでしょう。

ただし、より複雑な数学、長い条件整理、複数段階の推論では結果が変わる可能性があります。

### コードはClaude Sonnet 5が簡潔

コード修正では、両モデルとも正しい回答を作りました。

GPT-5.6は入力検証まで含めた堅牢な実装、Claude Sonnet 5は要求された処理に絞った簡潔な実装という違いがあります。

今回の問題では、Claude Sonnet 5の方が過不足のない回答としてわずかに高評価となりました。

## 結局どちらを使うべき？

今回の想定比較では、Claude Sonnet 5が99点、GPT-5.6が98点でした。

ただし、1点差に大きな意味はありません。

用途別に整理すると、次のようになります。

総合的には、両モデルはほぼ互角です。

重要なのは「どちらが最強か」を決めることではありません。

自分が普段行う作業を両方に入力し、次の点を比較することです。

* 最初の回答で目的を達成できたか
* 修正指示が何回必要だったか
* 回答をそのまま利用できたか
* 説明の形式が自分に合っていたか
* コードが実際に動いたか

AIを選ぶ際には、ベンチマークの順位よりも、目的の成果物を完成させるまでの手間を見る必要があります。
