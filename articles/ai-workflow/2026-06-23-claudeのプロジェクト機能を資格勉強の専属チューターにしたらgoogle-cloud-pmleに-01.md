## はじめに

Claude は普段、単発のチャットをその都度開いて使っていて、それで事足りているような気がしていました。だからプロジェクト機能も、便利なんだろうなとは思いつつ長いこと開かずじまい…とはいえ、せっかく用意されているのに使わないのはもったいないと思い、自分に合う使い道はないものかと模索していました。

そんなとき、ちょうど Google Cloud の認定資格である Professional Machine Learning Engineer（PMLE）を受けることにしたので、勉強のパートナーとして使ってみることにしました。

結論から言うと、あまり凝ったことはしていないにもかかわらず、合格まで十分に伴走してくれたと思います。

## Claude の「プロジェクト機能」とは

ざっくり言うと、専用のメモリ・手順（カスタムインストラクション）・ファイル（ナレッジ）を持った作業空間です。

![プロジェクトのトップ画面。右側にメモリ・手順・ファイルのパネルがある](https://static.zenn.studio/user-upload/deployed-images/df7a6d174503e25632517dee.png?sha=6007e313f6fa53ab652c65061cd59cdc5e09d7d3)

単発のチャットと一番違うのは、会話やファイルの文脈がプロジェクトに溜まっていく点です。

## 実際にやった手順

### ① プロジェクトの初期設定

最初にカスタムインストラクション（手順）を設定します。私が入れたのは以下の3行です。

```
・推測で回答せず、不明事項があればWeb検索を行った後に回答してください。
・Google Cloud公式ドキュメントの情報を正とし、それ以外のブログなど個人発信情報は参考程度としてください。
・必要があれば図などを適宜用いてわかりやすく説明してください。
```

![カスタムインストラクションの設定画面](https://static.zenn.studio/user-upload/deployed-images/6d8b2bf681e1544a3279eb01.png?sha=89f8280269bce248b9815bd0aa48b07323b29ebd)

勉強のパートナーとして使っていくにあたって私が懸念していたのは、AI がもっともらしい嘘（ハルシネーション）を本当のことのように教えてこないかということです。そこで「推測で答えず、まずウェブ検索」「公式ドキュメントを正とする」を最初に明示しました。あわせて、プロジェクトのツール設定でウェブ検索をオンにしておきました。

![ツール設定でウェブ検索をオンにする](https://static.zenn.studio/user-upload/deployed-images/a490e53b2231781386c88088.png?sha=fe0b63530f7e6fb25a35feec0be78527ba93f683)

ナレッジ（ファイル）には、公式が一般公開している試験ガイド PDF を入れておきました。こうしたことで、質問したときに「これは試験ガイドのこの範囲にあるので重要です」といった形で、出題範囲と結びつけて教えてくれました。

[PMLE 公式試験ガイド PDF（日本語）](https://services.google.com/fh/files/misc/professional_machine_learning_engineer_exam_guide_japanese.pdf?hl=ja)

セットアップが終わったら、いきなり問題演習に入るのではなく、まず「どのあたりが重要になりそうか」を整理させるところから始めました。配点の重みや何が問われるのかの当たりを最初につけておくと、この後の勉強の際に優先順位を決めやすくなります。

![試験ガイドを分析して、重要になりそうな範囲を整理してくれた様子](https://static.zenn.studio/user-upload/deployed-images/336247a0c227926fe9b70891.png?sha=cccc4bf919a09130870bf998180b375947c17990)

### ② わからないところはその場ですぐ聞く

あとはひたすら勉強を進めていき、つまずいたらその場でプロジェクトに聞くだけです。「この用語なんだっけ？」「このサービスとあのサービスの違いは何？」となったら気軽に聞いていました。

初期設定がうまく効いているようで、質問すると推測で即答するのではなく、公式情報を確認するためにまずウェブ検索を挟んでから答えてくれました。

![質問すると、公式情報を確認するためにウェブ検索してから答えてくれる](https://static.zenn.studio/user-upload/deployed-images/e2c3f2d92ca5ebb55514478e.png?sha=e062d49f0f6daf09670aeef1e6154e948a84a878)

また、カスタムインストラクションに「必要なら図で説明して」と入れてあることで、以下のような図を添えて説明してくれることも多く、視覚的なイメージとしても頭に残りやすかったです。

![Vertex AI の全体像を図で説明してくれた例](https://static.zenn.studio/user-upload/deployed-images/18367417b2c22555bab548a9.png?sha=b1e3d0ced1394bcd981fa0aa1e951198a91b05e3)

![混同行列と評価指標の整理](https://static.zenn.studio/user-upload/deployed-images/3c6d87527ca65ccfdb14c940.png?sha=3fdc6811e0c14c3262d8a3e8bc15d144f688f8e1)

![ランダムフォレストの仕組み](https://static.zenn.studio/user-upload/deployed-images/f687ab29056593138031c33e.png?sha=40538bac2a06bb5d9fe21a7aa102d4bec21cba52)

ここでとにかく疑問点を質問することで、やり取りがプロジェクト内に履歴やメモリとして蓄積されていきます。

### ③ 弱点中心の直前マニュアルを作らせる

試験直前、ここまでやってきた内容を踏まえて、自分専用の直前対策マニュアルを作ってもらいました。

> 試験が近いためこれまでの質問を分析し、私の弱点を重点的にした直前確認などの用途となる試験対策マニュアルをmd形式で作成してほしいです。網羅性を優先し、多少長くなっても構いません。

続けて、私の場合はモデル選択の問題が苦手だったので、追加でこうお願いしました。

> 各推論モデルや分析モデルについても一覧でまとめてほしい

プロジェクトに質問履歴が溜まっているので、それを分析させたような形です。

すると、横断的な判断原則・弱点トップ・セクション別解説・即答パターン集まで揃った、かなりのボリュームの md ファイルを2つ作成してくれました。中身の一部は以下の通りです。

![生成された直前対策マニュアルの一部（ドリフト判別の項）](https://static.zenn.studio/user-upload/deployed-images/68043cec7ade7da7feadff0d.png?sha=3223bdd91b4c47e9d9c1e1f203a18afb686ddabb)

「気づいていなかった弱点が発掘された」というよりは「まあここは押さえておくところだよね」という、ある程度内容としては予想できたものでした。しかし、市販の教材や試験解説サイトと違うのは、自分が勉強中に「ここ間違えたな」と引っかかった箇所とリンクした内容になっていることでした。一般的なまとめよりも自分の弱点に寄せてある分、直前の総まとめとしては明確に頭の中を整理できる内容だったと思います。

これはプロジェクトに文脈が溜まっていたからこそで、単発のチャットでは作れないものです。

### 補足：モデルの使い分けについて

モデルは作業の重さで使い分けていました。試験ガイドの重点整理（①）や直前マニュアルの生成（③）のような重い分析は Opus、日々の細かい質問（②）は応答の速い Sonnet です。重い処理だけ Opus に寄せることで、勉強のテンポを落とさないようにしつつ、コストとのバランスも取れました。

## つまずいた点

ある程度順調に進んでいたのですが、途中で一つ困ったことがありました。

たとえば、ある監視設定の値について「これくらいで十分」といった回答が返ってきたとき、これが機械学習の一般論なのか、Google Cloud の公式仕様なのか、それとも Claude の推測（＝ハルシネーション）なのかが区別できませんでした。

そこで「一般論と推測は切り分けて答えて」と指摘したところ、今後の回答方針として次のように整理してくれました。

> * Google Cloud の公式仕様 → 公式情報として説明
> * 一般的な機械学習の知識 → 「一般論として」と明示
> * Claude による推測・補足 → 「推測ですが」または言及しない

そして、この方針がプロジェクトのメモリに記憶されました。

> 回答の根拠を必ず以下の3つに切り分けて明示する：①一般論、②Google Cloud公式仕様、③推測。ハルシネーション防止のため、根拠のない補足説明は行わない。

これ以降は、回答や例示の頭にラベルが付くようになりました。

![「①一般論：」のラベル付きで答えてくれた例](https://static.zenn.studio/user-upload/deployed-images/3fd4345ac4a2e98264aa91e8.png?sha=c19fee877b88ec9e244f68e63dc484e44e96d5fd)

![「②Google Cloud 公式仕様：」のラベル付きで答えてくれた例](https://static.zenn.studio/user-upload/deployed-images/11a97693147e357fd7df1e03.png?sha=dbe9bc48e2d48a283c75de96201b64f15e606134)

この切り分けは直前マニュアルにもそのまま効いていて、生成されたマニュアルには以下のような凡例が付いていました。

```
【公式】     Google Cloud 公式ドキュメントで確認済みの仕様
【ガイド】   アップロードされた試験ガイドPDFに明記されている範囲
【判別】     過去に混同しやすかった識別ポイント
【一般/推測】 公式の明記がなく、一般論・推測として扱うべき箇所
```

各項目がどの根拠に基づくのかが一目でわかるので、「ここは公式準拠だから安心」「ここは推測だから自分でも裏を取ろう」という読み方ができます。

反省点として、今回は途中からメモリに追加する形になりました。次に実施するときは、この根拠3分類を最初からカスタムインストラクションに入れておこうと思います。

## まとめ

やったことを振り返ると、本当にこれだけです。

1. 初期設定：推測で答えない・公式を正とする・図で説明、の3行＋ウェブ検索オン＋試験ガイド PDF をナレッジに入れる
2. 即質問：勉強中につまずいたらその場で聞く（文脈が溜まっていく）
3. 直前マニュアル：溜まった履歴を分析させ、弱点を重点化した対策マニュアルを作らせる

この進め方で、実際に PMLE に合格することができました！

![Professional Machine Learning Engineer 認定バッジ](https://static.zenn.studio/user-upload/deployed-images/52c53717f066ae884e157b92.png?sha=4f35977570d1f920889698e7d50afeac0402d16f)

「前提とルールを与える → 普段は気軽に聞いて文脈を溜める → 仕上げに溜めた文脈を分析させる」という型は、プロジェクト機能の良い使いどころとして自分の中に定着しました。これを機に他の使いどころも積極的に模索し、活用の幅を広げていきたいと思います。

## 余談

気になった点として、PMLE の対象である機械学習という分野が生成 AI とそもそも相性が良く、Claude にとって答えやすい内容だったのではないかということです。うまくいったのは題材の性質によるところも大きく、別の分野で流用する場合はカスタムインストラクションやナレッジの追加設定が必要になる可能性もある、という点はあらかじめご留意いただければと思います。
