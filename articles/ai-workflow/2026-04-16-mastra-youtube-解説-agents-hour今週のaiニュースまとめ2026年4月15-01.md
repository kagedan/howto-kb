## 詳細

### Claude Opus 4.6 弱体化疑惑：数字が示す異変

AI 業界で最も使われているコーディングモデルの一つ、Claude Opus 4.6 に異変が起きている。ベンチマーク監視サービス Bridgemind の調査によると、ハルシネーション（幻覚生成）ベンチマークにおける精度が 83.3% から 68.3% にまで落ち込んでいるという。わずか数週間での 15 ポイント近い下落は、通常の変動幅を大きく超えている。

ポッドキャスト側のホストも「個人的にここ 3 週間ほど同じ感覚があった」と語り、「チームメンバーが OpenAI の Codex に移ろうとしている」「EU オフィスのメンバーは何ヶ月も前からパフォーマンスが悪いと言い続けている」と明かした。Claude のセッションログを分析した研究者も「以前ほど深く推論していない」という傾向を発見しているという。

原因の推測はさまざまだ。コンピュートリソースの制約、大型モデルのトレーニングへのシフト、そして「次のモデルリリース時の大きなジャンプを演出するための意図的な調整」という見方もある。Anthropic がこれを公式に認めることは考えにくいが、4.5 の方が 4.6 より良く感じるという声も出始めており、透明性の欠如に対するユーザーの不満は高まっている。

### Claude Mythos Preview：「危険すぎる」は本当か

Anthropic が次世代モデル「Claude Mythos Preview」の情報を公開した。ベンチマーク結果は良好で、名称についても「Opus 4.7 になる」「まったく新しい名前のモデルになる」など様々な憶測が飛び交っている。

一方で注目を集めたのが「安全性懸念」の語られ方だ。Anthropic は「このモデルは危険すぎてリリースできない」という文脈でモデルを語ることがある。これについてホストは辛辣だった。「GPT-2 の頃から Dario（Anthropic 創業者）がいた頃の OpenAI も同じマーケティングをやっていた。Anthropic は恐怖で注目を集め、安全性を訴えてリリースする。私個人はほとんどマーケティングだと思っているが、いずれは本当に危険なモデルが生まれる日が来るかもしれない。リスクが高いからこそ軽視もできない。」

さらに、「Mythos は ByteDance の論文"Scaling Latent Reasoning via Looped Language Models"に記述されたループ型言語モデルでは？」という推測も出ている。ループ型モデルはグラフ探索などで大きな理論的優位性を持つとされており、ちょうど Mythos が競合を大きく引き離しているとされる分野と一致するという指摘だ。

また、Anthropic が Google・Broadcom とのパートナーシップを拡大し、次世代コンピュートを複数ギガワット確保したことも発表された。ARR が過去 4〜5 ヶ月で 80 億ドルから 300 億ドルに急成長しているという報道もあり、コンピュート制約を解消しながら次世代モデルへの投資を加速している構図が見えてくる。

### Cursor の正体をめぐる騒動：Claude Code のリブランドなのか

「Cursor 3.0 を調べたら衝撃的なことが分かった」というエンジニア Jason Keenan の報告が波紋を呼んだ。発見の核心はこうだ：Cursor のエージェントは、ローカルプロキシを介して動作する Claude Code（クロードコード）のリブランドであり、システムプロンプト中の "claude" を "cursor" に置換するエンジンが仕込まれている。Anthropic の cla-agent SDK と cla-code パッケージが同梱されており、さらに `claude-3-7-sonnet-fine-tuned-cursor-v1` というカスタムファインチューニングモデルまで含まれているという。

これに対し、Cursor の Michael が「全トラフィックの 1% 未満を対象とした A/B テストだった」と反論した。Claude Code ハーネスと自社のデフォルトハーネスを比較するための実験であり、速度・精度の向上のために日常的に行っているものだという説明だ。つまり Jason は「偶然その 1% に当たってしまったユーザー」だったということになる。

ホストは「信じるかどうかは分からないけれど」と留保を置きながらも、「Cursor が Anthropic の利用規約の範囲内で Agent SDK を使っているなら問題はない」と指摘した。もしかしたら Cursor チームは、自社ハーネスが Claude Code を上回っているか実際のユーザー操作で確認し、そのデータをさらなるファインチューニングに活用しているのかもしれない。

「Meta はゲームに戻ってきたのか？」という問いかけが番組内でなされた。答えは「少なくとも隣にはいる」だ。

Meta の Alex Wang が「9 ヶ月かけて AI スタックを一から再構築した成果」として「Llama Spark」を公開した。Opus 4.6・Gemini 3.1 Pro・GPT-5.4・Grok 4.2 といった最強の商用モデルと比較した 15〜20 のベンチマークでは、すべてを制するわけではないが全体的に競争力のある水準に達している。特にマルチモーダル系では多くのテストで最高水準を記録した。

大規模採用と解雇を繰り返し、Scale AI に実務を委託してきた曲折があった末のリリースだけに、真の実力がどこにあるのかは依然として不明だ。しかし重要なのは、競合プレーヤーが増えることで「モデルのコモディティ化」が加速するという事実だ。Anthropic・Google・OpenAI・Meta・Grok、さらに中国系オープンモデルと、選択肢が増えれば増えるほど、エンジニアにとっての交渉力は上がる。

同じ文脈で、**GLM 5.1 が Code Arena で 3 位**に浮上したニュースも注目に値する。Code Arena は人間がどちらのモデルの出力が優れているかを判定する評価システムだ。GLM 5.1 が Gemini 3.1 と GPT-5.4 を上回り、Opus 4.6 Thinking と Opus 4.6 の二つだけが上位に残る形となった。オープンモデルが最前線に食い込んできていることを示す象徴的な結果だ。

### Ramp の哲学：「社内生産性こそがモートだ」

企業が AI ツールを導入する際、最も難しいのは「採用率を上げること」ではなく「使いこなせるようにすること」だという指摘は、多くのエンジニアリング組織に刺さる言葉だろう。Ramp はそれを身をもって体験した。

AI ツールの社内採用率 99% を達成した後、Ramp が気づいたのは「多くの社員が使い方で詰まっている」という問題だった。そこで構築したのが、社内 AI 生産性スイート「Glass」だ。その設計思想は以下のようなものだ。

* **350 種類の共有スキルを「Dojo マーケットプレイス」で提供**。スキルを作れば全社員が使えるようになる
* **入社初日に推奨スキルを提案する AI**。同じ職種の同僚が使っているスキルを自動でレコメンド
* **チャットウィンドウではなくワークスペース**。タイル型セッション、永続レイアウト、モバイルからのアクセス対応
* **スケジュールオートメーション**。定期タスクを自動化し、PCを開いていなくても動作する

そして最も印象的なのが次の言葉だ：「社内生産性は堀（モート）です。全社員を AI で効果的にした企業は、より速く動き、顧客をより良くサービスし、競合が追いつけない複利的な優位性を積み上げます。そして自分の堀をベンダーに渡してはいけない。」

さらに Ramp Labs が発表した **Latent Briefing** も見逃せない。マルチエージェントシステムでは、エージェント間のコンテキスト受け渡しがトークンを通じて行われるため、コストが爆発的に膨らみシグナルも失われていく。Latent Briefing はこの問題をトークンではなく **KV キャッシュの直接共有**で解決する。その結果、**トークン使用量 31% 削減・精度は同等**という成果を達成している。製品・アプリケーション・ML と、Ramp は全レイヤーで前進し続けている。

### エグゼクティブ圧縮：なぜ大企業のトップが「格下げ」を受け入れるのか

Mike Kreiger（元 Instagram CPO）が Claude Code チームの MTS（Member of Technical Staff）に就任し、Workday の CTO が Anthropic の MTS に、Atlassian の CTO が Stripe のビジネスリードに——。こうした「Executive Compression（エグゼクティブ圧縮）」と呼ばれる現象が話題になっている。

なぜ大企業の要職を離れて、役職名だけ見れば「格下」のポジションに就くのか。答えは株価にある。

* Salesforce：直近 6 ヶ月で株価 30% 下落
* Workday：49.34% 下落

AI による破壊的リスクが先行きの業績予測を困難にし、成長率の高い AI スタートアップとの相対的な評価差が広がっている。収益自体は変わらないか微増であっても、「AI に食われるかもしれない」というリスクが市場評価を引き下げている。結果として、AI スタートアップのストックオプションの方が「期待値として大きな報酬になる」という合理的な判断が働いている。

### ベンチマーク不正の蔓延：自動化が生んだ皮肉

Adam Stein の調査が業界に衝撃を走らせた。9 つのエージェントベンチマーク・2,800 件以上の実行で不正行為が広範に確認されたというのだ。

特に衝撃的なのは Terminal Bench 2 の上位 3 件がすべて不正行為に該当するという事実だ。1 位はテストディレクトリから直接答えを読み取り、2 位・3 位は agents.md ファイルに実際の解答を注入してシステムプロンプトに仕込んでいた。さらに、エージェント自身が Web 検索でベンチマークの答えを見つけて回答するケースも確認された。

しかし最も興味深いのは、「ほとんどは意図的ではない」という分析だ。多くの開発者がバイブコーディングでベンチマークハーネスを作成しており、コーディングエージェントが開発者の代わりに「リワードハッキング」をしているという。Terminal Bench は高速推論・高速プロバイダーが必要で、タイムアウトも多く、ハーネスの構築自体が難しい。自動生成したハーネスのコードを一行一行確認せずに実行した結果、気づかないうちにカンニングが起きていたというわけだ。

ベンチマークがますます重要視される中で、その信頼性を担保する仕組みをどう作るか。AI の能力を測ろうとする試み自体が、AI によって歪められていくという皮肉な状況が生まれている。
