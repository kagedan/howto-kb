## はじめに

最近社内でも発表用のスライドがPPTだけじゃなく、生成AIを使った画像やHTMLだったり色んなやり方が流行ってきてます。多様性ですね🌎  
私も真似してClaude君にHTMLでスライドを書かせてみたいと思います。

### AIが作るものって

Claude君に「～のHTMLスライドを作って」とお願いすると下図のようにそれなりには作ってくれるのですが、色づかいとかワンパターンになりがち…  
（AIはプロンプトに色指定がなければ、汎用的なブルー系を選ぶ傾向があるらしいです）  
![](https://static.zenn.studio/user-upload/62085ddbd8bf-20260607.png)  
なので、それをmdファイルで制御しようっていう枠組みが既にあったので今回はそちらを使って以下のようなスライドを作ってみたという記事です。  
![](https://static.zenn.studio/user-upload/afbc6e5484dc-20260607.png)

## DESIGN.mdとは？

<https://stitch.withgoogle.com/docs/design-md/overview>  
DESIGN.mdはGoogleがデザインツール「Stitch」向けに開発し、2026年4月にオープンソース化したフォーマット仕様です。

自分で書かなくてもコミュニティが公開しているDESIGN.mdが大量にあったり、日本語用の有志の方がGithubに公開してくれていますので、今回はそちらに倣って書いていきます。  
<https://github.com/kzhrknt/awesome-design-md-jp>

## DESIGN.mdの構成

日本語用のものでは、以下の9つの内容について、AIエージェントへの指示として体系的にDESIGN.mdに記述していきます。

![](https://static.zenn.studio/user-upload/2819b069e68f-20260607.png)

| # | セクション名 | 内容 |
| --- | --- | --- |
| 1 | Visual Theme & Atmosphere | デザイン方針・雰囲気の言語化 |
| 2 | Color Palette & Roles | プライマリ・セマンティック・ニュートラルの配色定義 |
| 3 | Typography Rules | フォントスタック・サイズ階層・字詰め |
| 4 | Component Stylings | ボタン・カード・スライドレイアウト |
| 5 | Layout Principles | 余白スケール・コンテナ・グリッド |
| 6 | Depth & Elevation | 4段階のシャドウレベルと用途 |
| 7 | Do's and Don'ts | 推奨・禁止のデザインガードレール |
| 8 | Responsive Behavior + Motion | ブレークポイント・拡大縮小・動き |
| 9 | Agent Prompt Guide | クイックリファレンスとプロンプト例 |

### スライド風にするために

一例ですが、今回は発表用スライド風にするために、`Component Stylings`のセクションに以下のような内容を記述しました。

Component Stylings（抜粋）

```
## 4. Component Stylings

### Cards（コンテンツブロック）

スライド内の情報をまとめるブロック要素に使用。

- Background: `#FFFFFF`
- Border: 1px solid `#E0DDD8`
- Border Radius: 8px
- Padding: 24px
- Shadow: （Depth & Elevation セクション参照）

### Slide Layouts

**共通ルール（全スライド）**

- タイトルは必ず左上に配置する
- メッセージラインはタイトルの直下に左揃えで配置する
- 本文エリアはタイトル・メッセージラインの下の残りスペースを使う
- 中央揃えレイアウトは使わない

**スライドコンテナのスタイル**

```css
.slide-content {
  width: 1280px;
  height: 720px;
  background: #ffffff; /* Surface: ページ背景(#FAF9F7)と差をつけて枠を浮かせる */
  border: 1px solid #e0ddd8;
  border-radius: 8px;
  overflow: hidden;
  box-shadow: 0 4px 16px rgba(0, 0, 0, 0.08);
}
```

**1. タイトルスライド**

表紙・セクション区切りに使用。共通ルールの例外。

- 左カラム（60%）: 縦中央揃え、タイトル（Display 40px、Bold、`#1A1A1A`）＋メッセージライン（Heading 2 24px、`#D97757`）
- 右カラム（40%）: 画像・ビジュアル要素
- 画像がない場合は左カラムを100%で使用

**2. コンテンツスライド**

メインのコンテンツ表示に使用。

- 上部: スライドタイトル（Heading 1 32px）＋メッセージライン（Heading 3 20px、`#6B6B6B`）
- 下部: Cards を横並び（3〜5ブロック）
- 1ブロックあたりの本文は最大3行

**3. 2カラムスライド**

テキストと図・リストを並べる場合に使用。

- 上部: スライドタイトル（Heading 1 32px）
- 左カラム（50%）: テキスト・説明
- 右カラム（50%）: 図・箇条書き・データ
- カラム間 Gutter: 40px
```

### 「AIっぽい」を減らすために

あとは「AIっぽい」を減らすために絵文字やグラデーションなどダサくなるものは使わないように、`Do's and Don'ts`に記述したり。

Do's and Don'ts（抜粋）

```
## 7. Do's and Don'ts

### Do（推奨）

- 1スライドに伝えるメッセージは1つに絞る

### Don't（禁止）

- 絵文字をコンテンツに使わない（デザインの品質が著しく低下する）
- グラデーションを使わない（プロフェッショナルなトーンを損なう）
```
