「AIを使ってみたけど、なんか微妙な回答しか返ってこない......」

そう感じたことはありませんか？ ChatGPTやClaudeなどの生成AIは、今や誰でも使える時代になりました。しかし、**同じAIを使っているのに、人によって得られる結果に大きな差がある**のが現実です。

その差を生んでいるのが「プロンプト」——つまり、**AIへの指示の出し方**です。

本記事では、プログラミング知識ゼロでも実践できる「プロンプトの極意」を、業務シーン別の具体例とともに解説します。

**この記事を読むと...**

* AIから「期待通りの回答」を引き出すための4つの型がわかる
* 議事録・メール・企画書など、業務ですぐ使えるプロンプトテンプレートが手に入る
* 明日から実践できるチェックリストで、AI活用の質が変わる

## なぜプロンプトが重要なのか

AIは非常に賢い存在ですが、**あなたの頭の中までは読めません**。

たとえば、新しく入った部下に「いい感じの資料作っておいて」と言ったらどうなるでしょうか。おそらく、あなたの期待とはかけ離れたものが出てくるはずです。

AIもまったく同じです。曖昧な指示には曖昧な回答が返ってきます。逆に、**的確な指示を出せば、驚くほど的確な回答が返ってくる**。これがプロンプトの本質です。

プロンプトを工夫するのに、プログラミングの知識は一切不要です。必要なのは「**相手にわかりやすく伝える力**」——つまり、日常のコミュニケーション力の延長線上にあるスキルです。

## 非エンジニアが陥りがちな3つの罠

AIをうまく使えない人には、共通するパターンがあります。

### 罠1：指示が曖昧すぎる

これでは範囲が広すぎて、AIは教科書的な一般論を返すしかありません。

**良い例**: 「BtoB SaaS企業のマーケティング担当者です。月額5万円の予算でリード獲得を増やすためのSNS施策を3つ提案してください」

### 罠2：一発で完璧を求める

AIとの会話は、一問一答のテストではありません。**対話を通じて、答えを一緒に磨き上げていくプロセス**です。最初の回答が60点でも、そこから修正を重ねて100点に近づけるのが正しい使い方です。

### 罠3：背景情報を伝えない

あなたが知っていることを、AIは知りません。「誰に向けて」「どんな目的で」「どんな制約があるか」——こうした\*\*コンテキスト（背景情報）\*\*を伝えるだけで、回答の質は格段に上がります。

## 4つの極意 — 全体像

まずは全体像を把握しましょう。この4つを押さえれば、プロンプトの質は格段に上がります。

## 極意①：「役割」を与える

最初のテクニックは、AIに\*\*役割（ペルソナ）\*\*を設定することです。

人間の部下でも「あなたはベテランのコピーライターです」と言われれば、それに応じた振る舞いをしますよね。AIも同じです。

### Before / After で比較

**Before**（役割なし）:

```
キャッチコピーを考えてください。新商品のハンドクリームです。
```

→ 無難で平凡なコピーが返ってくる

**After**（役割あり）:

```
あなたは大手化粧品ブランドで10年の経験を持つコピーライターです。
20代〜30代の働く女性向けに、新商品のハンドクリーム
「シルクヴェール」のキャッチコピーを5案考えてください。
商品特徴：ベタつかず、オフィスでも使える速乾タイプ。
```

→ ターゲットと商品特性を踏まえた、プロ品質のコピーが返ってくる

### よく使う役割の例

| 業務 | 役割の設定例 |
| --- | --- |
| メール作成 | 「あなたはビジネスマナーに精通した秘書です」 |
| 企画書作成 | 「あなたは大手広告代理店のプランナーです」 |
| データ分析 | 「あなたはデータアナリストです。非専門家にもわかりやすく説明してください」 |
| 議事録整理 | 「あなたは正確さを重視するプロの議事録作成者です」 |

## 極意②：「具体的なゴール」を示す

「いい感じに」「わかりやすく」——こうした抽象的な指示では、AIは何を目指せばいいのか判断できません。

**出力のゴールを具体的に定義する**ことで、回答の精度が劇的に変わります。

### 具体化の3つの軸

**1. フォーマットを指定する**

```
以下の内容を、箇条書き5項目で要約してください。
各項目は30文字以内にしてください。
```

**2. 条件・制約を明示する**

```
以下の条件で提案してください：
- 予算：月10万円以内
- 期間：3ヶ月
- ターゲット：30代男性会社員
- トーン：カジュアルだが信頼感のある文体
```

**3. 良い例・悪い例を示す**

```
以下のような文体で書いてください：
良い例：「たった3ステップで完了します」
悪い例：「所定の手続きに従い、段階的にプロセスを遂行してください」
```

## 極意③：「段階的に聞く」

複雑なタスクを一度にお願いすると、AIも混乱します。**ステップに分けて、対話しながら進める**のがコツです。

### 実践例：企画書を作る場合

**ステップ1**: 方向性を相談する

```
社内の業務効率化ツール導入の企画書を作りたいです。
まず、企画書に入れるべき項目を箇条書きで提案してください。
```

**ステップ2**: 構成にフィードバックする

```
ありがとうございます。「費用対効果」と「導入スケジュール」の
項目を追加してください。また、「背景」の部分をもう少し詳しく
構成してください。
```

**ステップ3**: 本文を依頼する

```
この構成で企画書の本文を書いてください。
読み手は部長クラスの管理職です。
A4で3ページ程度に収まる分量でお願いします。
```

このように、**AIと"壁打ち"しながら完成度を上げていく**のが最も効率的です。

## 極意④：「テンプレート化」する

うまくいったプロンプトは、**テンプレートとして保存**しましょう。毎回ゼロから考える必要がなくなり、業務の標準化にもつながります。

### 万能プロンプトテンプレート

```
# 役割
あなたは[役割]です。

# 背景
[状況・目的を説明]

# タスク
以下の条件で[成果物]を作成してください。

# 条件
- [条件1]
- [条件2]
- [条件3]

# 出力フォーマット
[形式を指定：箇条書き、表、文章など]
```

このテンプレートの `[ ]` 部分を埋めるだけで、どんな業務にも応用できます。次のセクションで、具体的な業務シーンに当てはめた実践例を見ていきましょう。

## 実践：業務シーン別プロンプト例

ここまでの極意を組み合わせた、すぐコピペで使える実践例を紹介します。

### シーン1：会議の議事録を要約する

```
# 役割
あなたは正確さを重視するプロの議事録作成者です。

# タスク
以下の会議の書き起こしを整理してください。

# 出力フォーマット
1. 会議の概要（3行以内）
2. 決定事項（箇条書き）
3. 各メンバーのTODOと期限
4. 次回までに確認が必要な事項

# 条件
- 発言者の名前は残してください
- 曖昧な表現は「要確認」と注記してください

---
（書き起こしテキストをここに貼り付け）
---
```

### シーン2：お断りメールを書く

```
# 役割
あなたはビジネスマナーに精通した秘書です。

# タスク
取引先からの会食のお誘いを丁寧にお断りするメールを作成してください。

# 条件
- 相手：株式会社〇〇の田中部長（長年のお付き合い）
- 理由：出張と日程が重なるため
- トーン：感謝を伝えつつ、次の機会につなげる
- 代替案：来月以降で改めてこちらから日程を提案したい
- 文字数：200〜300文字程度
```

### シーン3：プレゼン資料の骨子を作る

```
# 役割
あなたは大手コンサルティング会社のプレゼン資料作成のプロです。

# 背景
経営層向けに「DX推進プロジェクトの中間報告」を行います。

# タスク
プレゼン資料の構成案を作ってください。

# 条件
- スライド枚数：10枚程度
- 1スライドごとにタイトルと要点を記載
- データや図表の提案があれば「【図表案】」と注記

# 出力フォーマット
結論を最初に持ってくるピラミッド構造で
```

### シーン4：Excelデータの傾向を分析する

```
# 役割
あなたはデータアナリストです。非専門家にもわかりやすく説明してください。

# タスク
以下の売上データから、次のことを分析してください：
1. 月別の売上トレンド
2. 前年同月比での変化が大きい月とその要因仮説
3. 来期の売上予測（楽観・標準・悲観の3シナリオ）

# 出力フォーマット
- 各項目を見出し付きで整理
- 数値には必ず単位をつける
- 重要な発見は太字で強調

---
（Excelからコピーしたデータをここに貼り付け）
---
```

## まとめ：明日から使えるチェックリスト

プロンプトを書くときは、以下の4つを意識するだけで回答の質が大きく変わります。

| # | チェック項目 | 確認ポイント |
| --- | --- | --- |
| 1 | **役割を与えたか？** | 「あなたは〇〇です」と専門性を設定する |
| 2 | **ゴールは具体的か？** | フォーマット・条件・文字数などを明示する |
| 3 | **一度に詰め込みすぎていないか？** | 複雑なタスクはステップに分けて依頼する |
| 4 | **背景情報を伝えたか？** | 誰に向けて、何の目的で、どんな制約があるかを書く |

**プロンプトは「AIへの指示書」ではなく「AIとの対話の入り口」です。**

完璧なプロンプトを一発で書く必要はありません。まずは今日紹介したテンプレートを使って試してみて、AIとの対話の中で修正していく——それが、AIを"優秀な部下"に育てる一番の近道です。
