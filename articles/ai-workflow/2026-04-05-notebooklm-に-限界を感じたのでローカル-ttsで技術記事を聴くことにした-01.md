## 積読が解消しない

気になる技術記事。「あとで読もう」が溜まっていくばかり...  
電車時間やランニング、お風呂の時間など手を動かせない時間でなんとかキャッチアップできないだろうか。

ということで、URL を渡したら音声ファイルにしてくれるパイプラインを作りました。

## NotebookLM で感じた限界

最初に手を出したのは NotebookLM です。URL やテキストを投げると、AIホストがポッドキャスト風に要約してくれるものです。

試してみたものの、使い続けるうちに以下を感じるようになりました。

|  | 事例 |
| --- | --- |
| **冗長** | 1-2分で読める記事が5分のポッドキャストに膨張。本質的でない会話が追加されがち |
| **カスタマイズ不可** | 声、長さ、トーンの調整に限界がある。パターンがないので飽きやすい。 |

## ローカルTTS選定

NotebookLM の代わりに、ローカルで動く日本語 TTS モデルを探しました。ChatGPT や Claude の読み上げ機能も試しましたが、機械的な音声で実用的ではありませんでした。

いくつか試した中で **Tsukasa\_Speech** を使うことにしました。選んだ理由は、単純に聞きやすかったからです。VOICEVOX や AivisSpeech も使ってみましたが、Tsukasa\_Speech が一番自然に聞くことができました。

### ローカル TTS ならではのカスタマイズ

ローカル TTS なら自分で調整できます。  
プロンプトでランダムな値を指定し、毎回違う音声で出力することで飽きがこないように工夫しています。

| パラメータ | 内容 |
| --- | --- |
| **声** | リファレンス音声から選択したり、自分の好みの声を指定できる |
| **速度** | `rate_of_speech` で自由に調整 |

## パイプラインの作成

TTS モデルが決まったので、要約と音声生成を行うパイプラインを組みます。

### 処理フロー

ポイントは **LLM による「読み上げ用ブリーフィング生成」** です。

TTS に渡すテキストは、元テキストをそのまま流し込むのではなく、LLM で「聴いて分かる形」に再構成しています。マークダウン記法の除去、箇条書きの文章化、技術用語のカタカナ置換。この前処理が音声の聞きやすさにかなり効きます。

エンジンに Claude Code を選んだのは、**ローカルLLMの処理性能だと、要約とブリーフィングの出力精度に課題を感じた**ことと、**MCP 経由でデータを直接取得できる**ためです。

### 技術用語のカタカナ変換

Tsukasa\_Speech は日本語専用モデルなので、英語をそのまま渡すとアルファベットを1文字ずつ読み上げてしまいます。「API」が「エー。ピー。アイ。」になるイメージです。

これを防ぐために、パイプライン側に変換辞書を持たせました。

```
TECH_TERM_MAP = {
    "Python": "パイソン",
    "JavaScript": "ジャバスクリプト",
    "React": "リアクト",
    "Docker": "ドッカー",
    "API": "エーピーアイ",
    "AWS": "エーダブリューエス",
    "CI/CD": "シーアイシーディー",
    "GitHub": "ギットハブ",
    "Notion": "ノーション",
    # ... 100語以上登録
}

def preprocess_tech_terms(text):
    """英語の技術用語をカタカナ読みに置換する"""
    # 長いキーから順にマッチ（"Next.js" が "Next" に先に引っかからないように）
    sorted_terms = sorted(TECH_TERM_MAP.keys(), key=len, reverse=True)
    for term in sorted_terms:
        if term.isupper() or "/" in term:
            # 略語は大文字小文字を区別しない
            pattern = re.compile(re.escape(term), re.IGNORECASE)
        else:
            # 固有名詞はそのままマッチ
            pattern = re.compile(re.escape(term))
        text = pattern.sub(TECH_TERM_MAP[term], text)
    return text
```

加えて、ブリーフィング生成時にも LLM に「技術用語はカタカナで書くこと」と指示しています。辞書と LLM の二重変換で、読み上げ精度を上げました。

電車時間や散歩時間をより効率的なインプット時間に充てることができるようになりました。
