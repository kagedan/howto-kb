# Claude API のコストを50%削減する5つのテクニック

Claude API を使い始めると、気づけば月の請求が想定以上になっていた…という経験はありませんか？

本記事では、**実際に API コストを50%以上削減した具体的な5つのテクニック**を紹介します。すべて Python のコード例付きです。

## 前提: Claude API の料金を正しく理解する

まず押さえるべき事実があります。

| モデル | 入力 ($/1M tokens) | 出力 ($/1M tokens) | 出力/入力 比 |
| --- | --- | --- | --- |
| Claude Opus 4 | $15 | $75 | **5倍** |
| Claude Sonnet 4 | $3 | $15 | **5倍** |
| Claude Haiku 3.5 | $0.80 | $4 | **5倍** |

**出力トークンは入力の5倍高い。** つまりコスト最適化の最大のレバーは「出力を短くする」ことです。

## テクニック1: モデルの使い分け（最大80%削減）

すべてのタスクに Sonnet を使う必要はありません。

```
def select_model(task_type: str) -> str:
    """タスクの種類に応じて最適なモデルを選択"""
    # Haiku で十分なタスク（Sonnet の約1/4のコスト）
    haiku_tasks = {"classification", "extraction", "formatting", "sentiment"}

    # Opus が必要なタスク
    opus_tasks = {"complex_reasoning", "creative_writing", "code_architecture"}

    if task_type in haiku_tasks:
        return "claude-haiku-4-5-20251001"
    elif task_type in opus_tasks:
        return "claude-opus-4-20250514"
    else:
        return "claude-sonnet-4-20250514"
```

**実際のコスト比較（月間10万リクエスト、平均500入力+200出力トークン）：**

| 戦略 | 月額コスト |
| --- | --- |
| 全部 Sonnet | $450 |
| 使い分け（Haiku 60% / Sonnet 30% / Opus 10%） | **$207** |

**54%削減！**

## テクニック2: プロンプトキャッシュ（最大90%削減）

同じ system プロンプトやコンテキストを繰り返し使う場合、**キャッシュされた部分は90%割引**になります。

```
import anthropic

client = anthropic.Anthropic()

# 長いコンテキスト（マニュアル、仕様書など）
long_context = open("company_manual.txt").read()  # 10,000 tokens

# cache_control を指定するだけ！
message = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    system=[
        {
            "type": "text",
            "text": long_context,
            "cache_control": {"type": "ephemeral"},  # ← これだけ
        }
    ],
    messages=[{"role": "user", "content": "この文書の要約をしてください"}],
)

# 2回目以降は cache_read_input_tokens として90%割引で計算される
print(f"キャッシュ読取: {message.usage.cache_read_input_tokens}")
```

**注意:** キャッシュは最低1,024トークン以上のブロックに適用。5分間の TTL があります。

## テクニック3: Batch API で50%割引

即時レスポンスが不要なら、**Message Batches API** で50%オフ。

```
import anthropic

client = anthropic.Anthropic()

# バッチリクエストを構築
requests = [
    {
        "custom_id": f"req-{i}",
        "params": {
            "model": "claude-sonnet-4-20250514",
            "max_tokens": 1024,
            "messages": [{"role": "user", "content": prompt}],
        },
    }
    for i, prompt in enumerate(prompts_list)
]

# 送信 — 通常料金の50%！
batch = client.messages.batches.create(requests=requests)
print(f"バッチ ID: {batch.id}")
```

**適するケース:** 夜間バッチ処理、大量ドキュメント分析、テストデータ生成

## テクニック4: 出力トークンを制御する

プロンプトの書き方ひとつで、出力トークン（= 最もコストの高い部分）を大幅に削減できます。

```
# ❌ 出力が冗長になりがち
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=4096,  # 上限が大きすぎる
    messages=[{
        "role": "user",
        "content": "Pythonのデコレータについて説明してください"
    }],
)
# → 出力: ~500 tokens = $0.0075

# ✅ 出力を明示的に制限
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=256,  # 必要十分な上限
    messages=[{
        "role": "user",
        "content": "Pythonのデコレータを3文以内で説明してください。箇条書き不要。"
    }],
)
# → 出力: ~50 tokens = $0.00075（10分の1！）
```

**ポイント:**

* `max_tokens` を必要最小限に設定
* プロンプトで出力形式・長さを明示的に指示
* JSON 出力を指定すると、余計な説明文が省ける

## テクニック5: トークン数の事前計算で予算管理

```
import anthropic

client = anthropic.Anthropic()

def check_before_send(messages: list[dict], budget_usd: float = 0.10) -> bool:
    """送信前にコストを見積もってチェック"""
    result = client.messages.count_tokens(
        model="claude-sonnet-4-20250514",
        messages=messages,
    )

    estimated_cost = (
        result.input_tokens * 3 / 1_000_000
        + 500 * 15 / 1_000_000  # 出力は500トークンと仮定
    )

    if estimated_cost > budget_usd:
        print(f"⚠️ 推定コスト ${estimated_cost:.4f} が予算 ${budget_usd} を超過")
        return False

    return True
```
