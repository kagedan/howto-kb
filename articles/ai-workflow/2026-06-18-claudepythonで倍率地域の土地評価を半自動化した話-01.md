財産評価の現場でかならず発生する作業があります。固定資産税の課税明細書を見ながら、土地評価用のエビデンスを集めてくる——この作業を自動化したくて、いくつか設計上の判断をしたのでまとめておきます。

---

## 背景：倍率地域の評価って地味に手間がかかる

相続税における土地評価の方法は、大きく「路線価方式」と「倍率方式」の2つです。

倍率方式（倍率地域）の計算自体はシンプルで、

```
相続税評価額 ＝ 固定資産税評価額 × 倍率
```

だけです。ただ、筆数が多い案件では次の作業が繰り返し発生します。

1. 課税明細書から固定資産税評価額を拾う
2. 国税庁の財産評価基準書（評価倍率表）でその地域の倍率を調べる
3. 計算して明細書に転記する

特に2の「倍率を調べる」は市区町村ごとに国税庁サイトへアクセスする必要があり、案件によっては複数の市区町村をまたぐこともあります。

## 設計上の判断①：Claude APIは使わない

最初の設計では、AnthropicのAPIをコードから直接呼んで課税明細書のOCRを行っていました。PDFをAPIに投げてCSVに変換する、という処理です。

ただ、Claude ChatにPDFをアップロードして変換すれば、コードを書かずに同じことができます。変換したCSVをスクリプトに渡すだけにしました。

APIを使う＝課金・APIキー管理・ネットワークエラーのハンドリングが必要になります。「同じ結果が得られるなら手動でいい」という判断で、コード内のAnthropicへの依存をゼロにしました。

---

## 設計上の判断②：倍率の自動読み取りも省いた

当初は国税庁サイトをスクレイピングしてHTMLテーブルから倍率を取得する設計にしていました。

ところが実際に確認すると、**評価倍率表はHTMLではなく画像PDFのiframeで表示**されていることがわかりました。テキスト抽出が一切できないフォーマットです。

PDFから倍率を読み取るには、

* OCR処理（→ Claude APIに逆戻り）
* または手動入力

のどちらかしかありません。OCR精度の問題もあるため、ここは「PDFをダウンロードするだけ」に割り切りました。倍率の確認と計算は別途行う想定です。

## 実際の使い方

**CSVフォーマット**（先頭列に年分を指定）

```
年分,所在地,地目,地積,固定資産税評価額,課税標準額_固定資産税,課税標準額_都市計画税
r07,東京都西多摩郡奥多摩町〇〇番地,宅地,150.00,3600000,3600000,
r07,埼玉県秩父郡小鹿野町〇〇番地,田,400.00,800000,800000,
```

課税明細書PDFをClaude Chatに投げて「このフォーマットでCSVにして」と依頼するだけで変換できます。

**実行**

```
python pipeline.py kazeimeisaisho.csv
```

**出力されるもの**

倍率表PDFはスクリプト実行時に市区町村単位で自動ダウンロードされるため、国税庁サイトを手動で何度も開く手間がなくなります。
