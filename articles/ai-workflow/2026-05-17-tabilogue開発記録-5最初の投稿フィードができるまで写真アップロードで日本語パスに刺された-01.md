データベースの準備が終わったので、次は実際に投稿できる画面を作っていきます。

投稿フォーム・フィード表示・写真アップロードと、アプリの「顔」になる部分です。

### 投稿フォームとフィードを作る

まず投稿フォームを作りました。Claudeにコードを書いてもらって、コピペで実装。フィードにカード形式で表示されるようにしました。ここまでは比較的スムーズでした。

ただ、できあがった投稿画面を見てみると、写真をアップロードする場所がありません。「写真の投稿はどこからやるんですか?」と確認したところ、実装がまだだったことが分かりました。

追加してもらってから、今度は「各ルートに写真を貼るにはどうするんですか?」と聞きました。ルート欄にカンマ区切りでスポット名を入力すると、スポットごとに写真アップロード欄が現れる仕組みでした。

```
難波, 道頓堀, 通天閣
```

のように入力すると、3つのスポットそれぞれに写真を紐づけられます。仕組みが分かれば直感的でした。

### 写真をアップロードしたら表示されない

写真アップロードが実装できたので、さっそく試してみました。

投稿フォームで写真を選んで「投稿する」を押しましたが、フィードには写真が表示されず、グラデーションの背景のままです。

「ストレージに入っていないんじゃないか」と思ってSupabaseのStorageを開いてみたら、バケット(ファイルの保存場所)の中身が空でした。案の定、アップロード自体が失敗していた状態でした。

### No rows returned の発見

コンソール(エラーログが見える画面)を確認すると、こんなエラーが出ていました。

```
StorageApiError: Invalid key: .../0_難波.jpeg
```

ファイルパスに「難波」という日本語が含まれていたのが原因でした。

Supabase StorageはファイルパスにASCII文字(英数字や記号)しか使えません。日本語のスポット名をそのままファイル名に使おうとしていたので、アップロードがエラーになっていたのです。

念のためSQL Editorで post\_images テーブルの中身も確認してみたら、

```
No rows returned
```

やはり1件もデータが入っていませんでした。アップロードが失敗していたので当然の結果でした。

### ファイルパスからスポット名を外す

解決策はシンプルでした。ファイルパスからスポット名を取り除いて、インデックス番号だけにする方法です。

```
変更前: {user_id}/{post_id}/0_難波.jpeg
変更後: {user_id}/{post_id}/0.jpeg
```

たった1行の修正でした。これで再投稿してみたら、写真がカードに表示されました。

スポット名は post\_images テーブルの spot\_name カラムに別途保存しているので、表示には影響ありません。ファイル名とスポット名は分けて管理するのが正解でした。

### 気づいたこと

今回初めてコンソールでエラーを確認しました。画面上は何も起きていないように見えても、裏側では失敗していることがある。コンソールを見れば原因がすぐ分かる、というのはここで学んだことです。
