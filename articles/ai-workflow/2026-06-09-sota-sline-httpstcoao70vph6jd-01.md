# 商談資料を8時間で作り直して分かった、読まれる資料と刺さる資料を分ける構造設計

提案資料の表紙をめくった瞬間、相手の視線がどこで止まり、どこで離脱するか。

商談で資料を配った直後の3秒、相手はまず価格表でも会社概要でもなく「自分の課題が正しく書かれているか」を探す。

ところが多くの提案書は最初のページで自社紹介を始める。

僕が半年、商談資料を作り直し続けて気づいたのは、読まれる資料と刺さる資料は情報量ではなく順番と根拠の見せ方で決定的に分かれるという事実だった。

事例の置き場所、価格の根拠の提示、競合比較の書き方。

この3点を組み替えるだけで成約率は変わる。

# 提案資料の1ページ目で相手が探しているもの

![](https://pbs.twimg.com/media/HKVpIicaQAAxiV9.jpg)

商談で資料を配った瞬間、相手は何を見ているのか。

僕はこの半年、商談のたびに相手の視線を追い続けた。

資料を渡してから最初の5秒、目線がどこで止まり、どこで滑っていくのか。

スマホのインカメラを薄く回しながら、自分が話していない時間の相手の顔だけを後で見返す。

気持ち悪い作業だけど、これをやると分かることがあった。

1ページ目で自社紹介を始める資料は、ほぼ例外なく3秒で離脱される。

「株式会社○○は2023年設立の」——この一文が出た瞬間、相手の目線は資料から外れて、自分の手元のスマホかコーヒーカップに移る。

表情筋が緩む。

要するに、聞いていない。

## 相手は「自分の話」しか探していない

理由はシンプルで、商談の最初の数秒で相手が探しているのは「これは自分の話か、他人の話か」の一点だけだからだ。

自社紹介は完全に「他人の話」だ。

相手にとって、あなたの会社がいつ設立されたかも、何人いるかも、どんな理念かも、その瞬間にはどうでもいい。

後で必要になるかもしれないが、1ページ目に置く情報ではない。

僕が観察し続けて確信したのは、1ページ目に置くべきは相手の課題の言語化だということ。

それも、ふわっとした「御社の集客課題」みたいな抽象語ではダメだった。

相手が商談中に実際に口にした言葉、その温度のまま資料に載っていると、相手の目線が止まる。

止まるどころか、前のめりになる。

「あ、これ俺がさっき言ったやつだ」という反応。

これが出た商談は、体感で成約率がまるで違う。

具体的には、相手の課題が解像度高く言語化されている資料は、その後の価格の話で値引き交渉に入りにくくなる。

なぜなら相手の中で「この人は分かってくれている」という前提ができているから、価格は「理解してくれた対価」に変わる。

逆に1ページ目で自社紹介から入ると、その後どれだけ良い提案をしても「で、いくら?」の防御姿勢が崩れない。

最初の3秒で「他人の話」判定されると、相手はずっと値踏みする側に立ち続ける。

## 議事録から課題ページを再構成する手順

問題は、相手が言った言葉をその温度のまま資料化するのが、手作業だと間に合わないことだった。

商談は次々入る。

前回の商談で相手が何と言ったかを記憶だけで再現すると、どうしても自分の解釈が混ざる。

「コストを下げたい」と言ったのか「無駄な工数を減らしたい」と言ったのか。

この2つは似ているようで、刺さるページの作り方が全然違う。

そこで半年前から、商談の議事録を Claude Opus 4.8 に通して課題抽出を回すようにした。

手順はこうだ。

まず商談はすべて録音する(相手の許可を取った上で)。

Slack に貼った文字起こしを、Claude Opus 4.8 に投げる。

このときプロンプトで指定するのは「相手が自分の課題について語った発言だけを、相手の言葉のまま抽出しろ。

要約も言い換えもするな」という1点だけ。

要約させないのが肝心だった。

AI に課題を要約させると、きれいな経営用語に丸められてしまう。

「業務効率化」「コスト最適化」みたいな、誰の商談でも使える無味乾燥な言葉になる。

これだと1ページ目に載せても相手の目線は止まらない。

止まるのは、相手自身が使った生々しい言葉の方だ。

```

# Claude Opus 4.8 に投げるプロンプト構造
- input: 商談文字起こし(全文)
- task: 相手が「困っている/やりたい/避けたい」と語った発言を原文ママで抽出
- 制約: 要約禁止・言い換え禁止・経営用語への翻訳禁止
- output: 発言を発言時刻順に並べ、課題の優先度を相手の語気から推定

```
