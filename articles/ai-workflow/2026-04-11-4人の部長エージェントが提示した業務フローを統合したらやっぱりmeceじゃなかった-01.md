## 部長エージェント4人に業務フロー整理を一斉指示

製造する商品を「バタークッキー スタンダード」に決めたところで、次は業務フローを整理していきます。  
Claude Codeに以下の指示を出しました。

私のプロンプト

```
▶ まずはスタンダードに、バタークッキーの製造から始めましょう。
▶ この決定を各部長に伝えた上で、部門ごとに実際の業務フローを整理して、提出させてください。
```

これにより、Claude Codeは4つの部長エージェントを**並行起動**して、各部長に担当業務フローの整理を指示しました。  
数分ほど待つと、4部門から報告が上がってきます。

Claude Codeの返答：各部門の業務フロー概要

**商品開発部（中村部長）**

* 企画 → 試作 → 評価 → 量産判定 の流れ
* 試作は複数回実施して品質を確認してからリリース判定

**製造部（山田部長）**

* 原材料受入 → 計量 → 生地作成 → 成形 → 焼成 → 冷却 → 検査 → 包装 の8工程
* ロット番号で製品の追跡ができるようにする

**品質保証部（田中部長）**

* 最終検査・不良品管理・クレーム対応・社内監査の4本を担当
* 「検査記録のない製品は出荷しない」を原則とする

**営業・購買部（佐藤部長）**

* 問い合わせ → 見積 → 受注確定 → 納品 の受注フロー
* 仕入先は審査済みの業者に限定し、受入時に品質チェックを行う

いいですね、それぞれの立場に合わせたそれっぽい業務フローが揃ってきました。  
本物の人間を相手にしていたら、これが出てくるまで一体何週間待つことになるのか・・・。このスピードは、やはり驚異的です。

## 横断レビューを依頼する

各部門のフローが揃ったところで、次は「4つが全体として繋がっているか」を確認します。  
縦割りで作った個別フローを横に繋ぎ、全体最適を進めていきます。

私のプロンプト

```
▶ 各部門から提出された業務フローを横断的にレビューして、抜け漏れやリスクがありそうなところを教えてください。
```

Claude Codeから返ってきたのは、10個の課題リストでした。

Claude Codeの返答：横断レビューで発見された10の問題

**🔴 重大な抜け漏れ（すぐ対処が必要）**

* 最終検査担当の二重定義：製造部・品質保証部の両方が最終検査を実施すると記載
* クレーム対応が宙ぶらりん：クレームの受付は営業だが、対応部署が決まっていない
* マネジメントレビューも宙ぶらりん：会社全体の品質活動を振り返る場が存在しない

**🟡 中程度のリスク（設計段階で決めておくべき）**

* 設備構成プロセスの欠落：製造設備の精度チェックをどの部門がやるか決まっていない
* 原材料企画書の作成者が曖昧：原材料の規格書を誰が作るか、2部門で話が食い違っている
* 設計変更、製造条件変更フローの欠落：製造にかかわる条件を変更したときの手続きが決まっていない
* 内部監査員の独立性問題：社内監査で「自分の部門を自分で審査する」構造になっている

**🟢 軽微なリスク（将来対応でよいもの）**

* 教育訓練フローの具体化不足：新人教育の手順が各部門でバラバラ
* 回収・リコール判断フローの欠落：万が一の回収・リコール手順がない
* 記録管理の媒体が未統一：記録の書式が部門ごとに統一されていない

いいですね～、**それっぽい課題**が沢山でてきましたね！

やはり縦割りで作業をさせると、MECE（漏れなく、ダブりなく）じゃなくなりやすいですね。業務の二重定義や、逆にやる人が誰もいない宙ぶらりんのタスクが発生するのは、現実でもよく起きる課題ではないでしょうか。

実際にこれらの課題を解決する場合、担当を決めるのはいいが現場のリソースをどうやって確保するか？という新たな課題が生まれることが多々あります。  
ですが、今は全体設計のフェーズですので、まずは**理想の姿を描くことを優先**します。リソース不足は、あとで何とかしましょう。

## 優先度をつけて対応する

まずは緊急度の高い「🔴 重大な抜け漏れ」から対応していきましょう。  
方針を決めて、各部長に対応を指示していきます。

私のプロンプト

```
▶ 発見された課題について、以下の方針で対処します。各部長に伝えてください。
▶ 🔴 最終検査：品質保証部が単独で担当。製造部との役割をきっちり分ける。
▶ 🔴 クレーム対応：どの部門が受け付けても、品質委員会で対応方針を決議する。
▶ 🔴 品質委員会を設置する。私（代表取締役）が議長、事務局は田中部長。
▶ 🔴 全社振り返りの場は、文書が整ってから設計する。
▶ 🟡 以下は各部長に対応を一任する。
```

方針を伝えると、**各部長のエージェントファイルが更新**され、それぞれの業務担当範囲が再定義されました。

ただ、エージェントファイルが更新されただけでは、各部長の「自覚」が変化しただけに過ぎません。  
ほかのエージェントがどんな役割を担っているかは、全エージェントが参照できるドキュメントにしておいた方が良いでしょうね。

エージェントファイルの更新に加えて、以下2つのドキュメントも提出されてきました。

滑り出しとしては十分ですね。これから残タスクを処理しながら、株式会社TetsuroWorksのQMS体系を充実させていきます。

最後に、各部長エージェントにねぎらいの言葉をかけて、今回の作業は終了です。

この作業スピードはとにかく驚異的。Claude Codeを使い始めた初日にここまで進めることができていますからね。  
Claude CodeのProプランを契約する前にやりたいことを整理していたのが良かったです。

今回の検証から、実際の企業の業務文書を参照させた上で横断的なレビューを行わせれば、QMS上の課題を洗い出す上で十分役立つだろうと考えています。

その際のネックは、**企業の業務文書がMarkdownのようなAIリーダブルな状態になっていないこと**が考えられますが、これすらもClaude Codeに作成を依頼すればあっという間にできてしまうんだろうなぁと思います。
