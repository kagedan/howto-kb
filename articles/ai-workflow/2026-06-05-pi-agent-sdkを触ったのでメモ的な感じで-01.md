# Pi SDK を触ってみたメモ

どうも。  
前から「AIエージェント作りたいなー」と思っていて、最近作り始めました。  
その際に一旦 Pi をコアに作ろうと思い、SDK を触ってみていたので、どんな感じだったかのメモを書いておきます。  
公式ドキュメントがあるので、基本的にはそこを見れば解決する話ではあります。  
<https://pi.dev/docs/latest/sdk>

## インストール

まずは `@earendil-works/pi-coding-agent` を import します。

## ResourceLoader

最初に `ResourceLoader` が必要です。  
この中にエクステンションやプロンプトが入ります。  
個人的には `DefaultResourceLoader` をそのまま使うのが良い感じだと思います。  
最初はよく分からずに `ResourceLoader` を継承して自作クラスを作っていましたが、微妙だった気がします。  
プロンプトをコード側から触りたい場合は、`systemPromptOverride` を使って書き換える感じでやっていました。

### 勝手に追加される Current working directory を消す

プロンプトの最後の方にディレクトリパスが自動で追加されるのですが、消したかったのでエクステンションで無理やり削除していました。

```
pi.on("before_agent_start", (event) => {
  return {
    ...event,
    systemPrompt: event.systemPrompt
      .replace(/\n?Current working directory: .*$/m, "")
      .trimEnd(),
  };
});
```

これは Codex に書いてもらいました。

### Role を system に変更する

`developer` ロールになるのもなんとなく嫌だったので、下のコードも入れていました。

```
pi.on("before_provider_request", ({ payload }) => {
  (payload as { messages: { role: string }[] }).messages[0].role = "system";
  return payload;
});
```

## セッション作成

できた `resourceLoader` を `reload()` します。  
その後、`createAgentSession()` でセッションを作成します。  
自作ツールは `customTools` で指定します。  
使いたいツールは、自作・組み込み問わず `tools` に書いておきます。  
作成した `session` に対して `prompt()` を呼べば会話できます。

## LM Studio を使う

自分はプロバイダとして LM Studio を使っていました。  
Pi 単体では用意されていなかったので、自前でプロバイダを実装しました。  
コードはパッケージカタログにある LM Studio 関連のコードを参考にしながら直接書いています。  
<https://pi.dev/packages>

```
export const DEFAULT_LM_STUDIO_URL = "http://127.0.0.1:1234";

const response = await fetch(`${baseURL}/api/v1/models`);

// response を ProviderModelConfig に変換
// model.type === "llm" のみ採用

const providerConfig: ProviderConfig = {
  name: "LM Studio",
  baseUrl: `${baseURL}/v1`,
  api: "openai-completions",
  apiKey,
  models,
};
```

## 感想

こんな感じでしょうか。  
今は Pi から OpenAI Agents SDK に移行したので使っていません。  
ただ、大体のエージェントは Pi を使えば作れる気がします。

* bash
* edit
* write
* grep
* find
* ls  
  エージェントをサクッと作りたいなら、Pi はかなり良い選択肢だと思います。
