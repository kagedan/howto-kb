## 仕組み：Skills × ローカルMCP で Girls Talk を自動生成

```

　　　↓
```

### Skills（スキルズ）って何？

Claude Desktop には **Skills** という仕組みがあって「特定のタスクを実行するための手順書」みたいなものです。

今回作ったのは、こんなスキルです：

* **girlstalk-podcast** … お題を受け取って、複数キャラのガールズトーク台本を生成 → 音声合成 → Notionに投稿

# GirlsTalk Podcast Generator — ガールズトーク風ポッドキャスト自動生成 & Notion登録

お題を受け取り → Web調査（必要時） → キャラクター対話台本作成 → MP3音声合成 → Notionの「ポッドキャスト」データベースに登録する統合ワークフロー。

## 全体フロー

1. テーマの確認と調査
2. キャラクター選定と対話台本作成
3. 音声合成（synthesize マルチセグメントモード → MP3ファイル生成）
4. Notion投稿（notion\_post\_podcast → ページ作成 & MP3アップロード）
5. 完了報告

---

## Step 1: テーマの確認と調査

ユーザーが指定したテーマ（お題）を確認する。

* **Web調査が必要な場合**: 最新ニュースや具体的な情報が必要なテーマは WebSearch で調査する。検索は日本語・英語両方で行い、年月を含めて鮮度を確保。最低3回は検索する。
* **不要な場合**: 一般的な話題、意見ベース、日常系テーマならスキップ。
* ユーザーが情報を提供済みの場合もスキップ。

### キャラクター選定の指針

テーマに合わせて**3人**を選ぶ（ガールズトーク感を出すため）。

* **日常・エンタメ・恋愛**: るな（MC）＋ ひなた（リアクション）＋ 椿（ツッコミ）
* **技術・学習**: 凛（解説）＋ るな（MC・盛り上げ）＋ ひなた（質問・リアクション）
* **真剣・社会テーマ**: ゆい（分析）＋ しおん（本音ズバリ）＋ 凛（まとめ）
* **トレンド・流行**: るな（MC・詳しい）＋ 椿（ツンデレリアクション）＋ しおん（辛口コメント）

ユーザーがキャラ指名した場合はそちらを優先。

### 台本作成のルール

1. **キャラクター性の徹底**: 各キャラの一人称・口調・性格を完全に守る。
2. **ガールズトーク感**: 友達同士のおしゃべり感。「ねえねえ聞いて！」「それわかるー！」「えーウソでしょ！？」のような自然な掛け合い。
3. **構成**:
   * **オープニング（1〜2セグメント）**: MC役がテーマを振る
   * **本編（5〜6セグメント）**: テーマの深掘り。共感・対立・新発見の流れ
   * **まとめ（1セグメント）**: MC役がまとめるか、全員で一言
4. **音声合成を意識した書き方**:
   * マークダウン記法、URL、特殊記号は絶対に含めない
   * 英語の固有名詞はカタカナ表記
   * 1セグメントは200〜350文字を目安にする（短すぎると会話が薄くなるので、しっかり語らせる）
   * 「…」で間を表現してOK
5. **感情パラメータの動的調整**: 興奮シーンはemotionを上げ、真剣シーンは下げる。
6. **セグメント数は最大9個**: レート制限対策。短い相槌は前後のセリフに統合する。

### 台本の長さ目安

**約8〜10分の音声**を目標にする。ガールズトーク形式なので、各セグメントでキャラクターがしっかり自分の意見や具体例を語るようにする。

* 各セグメント: 200〜350文字（キャラの個性を出し、具体例や体験談、深掘りした意見を盛り込む。一言二言で終わらせない）
* 合計: 約2000〜3000文字（9セグメント × 平均250〜300文字）
* 各キャラが1回の発言でしっかり語り切ることを重視する。相槌だけのセグメントは作らず、必ず内容のある発言にする。

## Step 3: 音声合成（MP3生成）

### synthesize マルチセグメントモードで一括合成

`mcp__aivis-tts__synthesize` の `segments` 配列を使い、**1回のMCP呼び出しで全セグメントをまとめて合成**する。ツール側で各セグメントを個別合成→ffmpegで結合→1本のMP3ファイルとして返却してくれる。

**呼び出し方:**

```
mcp__aivis-tts__synthesize(
  output_path: "/tmp/{YYYY-MM-DD}-girlstalk-podcast.mp3",
  bitrate: "192k",
  segments: [
    {
      "text": "セグメント1のテキスト",
      "model_uuid": "キャラ1のmodel_uuid",
      "speaking_rate": 1.3,
      "volume": 1.0,
      "emotional_intensity": 1.8,
      "pause_after": 0.5
    },
    {
      "text": "セグメント2のテキスト",
      "model_uuid": "キャラ2のmodel_uuid",
      "speaking_rate": 1.25,
      "volume": 1.0,
      "emotional_intensity": 1.5,
      "pause_after": 0.5
    },
    ...最後のセグメントは pause_after: 0.3
  ]
)
```

**レスポンス:**

```
{
  "success": true,
  "file_path": "/tmp/2026-03-31-girlstalk-podcast.mp3",
  "file_size": 2606854,
  "format": "mp3",
  "segments_count": 9
}
```

レスポンスの `file_path` を次のStep 4でそのまま使う。

### レート制限対策

Aivis Cloud TTS APIは**1分間に10リクエスト**のレート制限がある。synthesizeのマルチセグメントモードは内部で複数APIコールを行うため、9セグメントでも制限に引っかかる場合がある。

**429エラーが出た場合**: 60秒待ってからリトライする（最大1回）。

## Step 4: Notion投稿

### notion\_post\_podcast ツールで一括投稿

`mcp__notion__notion_post_podcast` を呼び出すだけで、ページ作成・MP3アップロード・音声ブロック埋め込みが一括で行われる。

**呼び出し方:**

```
mcp__notion__notion_post_podcast(
  file_path: "/tmp/{YYYY-MM-DD}-girlstalk-podcast.mp3",
  title: "GirlsTalk: {テーマ名} {YYYY/MM/DD}",
  description: "テーマの簡単な説明。出演キャラ名、議論のポイントなどを含める。"
)
```

**レスポンス例:**

```
{
  "title": "GirlsTalk: 保育園のお弁当派か給食派か 2026/03/31",
  "page_id": "334fe545-ead6-8155-8b92-eff349bc0b0f",
  "page_url": "https://www.notion.so/GirlsTalk-2026-03-31-334fe545ead681558b92eff349bc0b0f",
  "file": "2026-03-31-girlstalk-podcast.mp3",
  "file_upload_id": "334fe545-ead6-81ae-a132-00b23754c745"
}
```

## Step 5: 完了報告

全処理が完了したら、以下を出力する:

```
✅ GirlsTalk Podcast 完了！
 
テーマ: {テーマ名}
出演: {キャラ名1}、{キャラ名2}、{キャラ名3}
セグメント数: {N}
ファイルサイズ: {N} KB
Notion: {page_url}
```

加えて、対話のテキスト版もチャットに出力する:

```
**GirlsTalk: 「テーマ名」**
出演: るな、ひなた、椿
 
---
 
**るな**: はい！というわけで今日のテーマは〜
**ひなた**: わー！それ気になってた！〜
**椿**: …べ、別に興味ないけど。〜
```

## エラーハンドリング

* **音声合成失敗（429以外）**: エラー内容を伝え、テキスト台本だけ提供する。リトライは行わない。
* **音声合成失敗（429）**: 60秒待ってから1回だけリトライ。それでもダメならテキスト台本を提供。
* **Notion投稿失敗**: エラーを出力し、MP3ファイルはoutputsディレクトリにコピーして提供する。

## 注意事項

* **volume**: synthesize時は `1.0` を使う。
* **キャラブレ禁止**: 各キャラの口調・一人称を絶対に守る。
* **output\_path**: `/tmp/` 配下に出力する（`/home/claude/` はMCPから書き込めない場合がある）。
* **セグメントの pause\_after**: 基本 `0.5`、最後のセグメントのみ `0.3`。

### ローカルMCP × AivisSpeech

音声生成には **AivisSpeech** というローカルTTSエンジンを使っています。これを **MCP（Model Context Protocol）** 経由で Claude Desktop から呼び出します。

ポイントは、**キャラクターごとに違う音声モデルを使う**ということ。

例えば：

* **ルナ** 🌙 … クールなお姉さん声
* **ゆい** 🌸 … 元気で明るい声
* **りん** 🔔 … 落ち着いた知的な声

それぞれの声で台本を読み上げて、結合すれば…

**はい、Girls Talk の完成です！！**
