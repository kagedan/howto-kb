## はじめに ― なぜ試したのか

フリーランスとして開業準備を進める中、複式簿記にまだ慣れていない状態で MoneyForward クラウド会計を触り始めた。個人の生活費を事業口座から払ったときは `事業主借`、カード決済した経費は `未払金` で仕訳する必要があるのだが、これが最初はなかなかしんどい。「この支払いはどの科目で、借方と貸方はどっちに何を入れるんだっけ」と毎回調べながら入力する作業が地味に時間を食っていた。

そんなタイミングで、2026年3月26日から利用可能になった [MoneyForward MCP](https://biz.moneyforward.com/support/account/guide/others/ot10.html) の存在を知った。MCP（Model Context Protocol）とは、AI モデルが外部サービスと連携するための標準プロトコルだ。Claude に接続することで、会話するだけで会計データを読み書きできるようになる。

「プロンプトで仕訳が作れるなら、経費入力が圧倒的に楽になるのでは？」という期待で試してみた。

---

## MoneyForward MCP とは

### MCP の概要

MCP は Anthropic が策定したオープンな仕様で、AI モデルと外部ツールをつなぐための共通インターフェースを定義している。従来のツール連携は API クライアントをアプリ側が実装する必要があったが、MCP では「MCP サーバー」という形でツール側が仕様を公開し、対応クライアント（Claude など）から直接呼び出せる。

### MoneyForward Cloud 会計との連携

MoneyForward が提供する MCP サーバーは、同社の会計 API（MFC Accounting API）をラップしたものだ。大きく「認証」「マスタ参照」「仕訳・明細操作」「レポート」の4カテゴリに分類できる。

#### 認証系

Claude との接続を確立するための OAuth2 フローを担う。認可 URL の生成と、認可コードをアクセストークンへ交換する2ステップで構成されている。

#### マスタ参照系

事業者情報・会計期間、勘定科目一覧（BS/PL 全科目）、補助科目、部門一覧、連携済みの銀行口座・カードを取得できる。レスポンスには科目が BS か PL かの区分も含まれており、科目の性質をプログラムで判別できる。

#### 仕訳・明細操作系

仕訳の新規作成・更新・一覧取得、連携口座への明細作成ができる。借方・貸方のペアを複数行まとめて登録でき、補助科目・部門・取引先・インボイス区分も指定できる。実務で必要な項目はひととおり揃っている印象だ。

注意点として、**削除操作には対応していない**。誤った仕訳は上書き更新するか、MoneyForward の画面から手動削除するしかない。

#### レポート系

残高試算表（損益計算書・貸借対照表）を取得できる。「今月の売上と経費の合計を教えて」といった問いに対して、Claude がレポートを呼び出して集計・解釈してくれる。

## セットアップ手順

### 前提条件

* MoneyForward クラウド会計のアカウント（有料プラン推奨）
* Claude.ai のアカウント（MCP コネクタが使えるプラン）

### 手順

**1. アプリポータルで権限設定を行う**

MCPサーバーの利用には、まず [マネーフォワード クラウド アプリポータル](https://app-portal.moneyforward.com/) での事前準備が必要だ。管理コンソールの「全権管理」権限を持つユーザーがアプリポータルを有効化し、連携操作を行うユーザーに「アプリ連携」「クラウド会計・確定申告」の権限を付与する。

**2. Claude.ai にカスタムコネクタを追加する**

Claude.ai の設定画面から「カスタムコネクタ」を開き、以下の MCP サーバーアドレスを入力して追加する（[公式ガイド](https://biz.moneyforward.com/support/account/guide/others/ot10.html) 参照）。

```
https://alpha.mcp.developers.biz.moneyforward.com/mcp/ca/v3
```

**3. 認可 URL をブラウザで開く**

Claude に「MoneyForward MCP の認可 URL を発行して」と伝えると URL が生成される。  
![](https://static.zenn.studio/user-upload/86b9f125044b-20260331.png)それをブラウザで開くと、MoneyForward クラウドのアカウント選択画面に遷移するので、使用するアカウントをクリックする。

**4. 権限を確認して許可する**

「アプリとの連携を許可しますか？」という確認画面が表示される。アプリ名は「MFC MCP server (alpha)」で、以下の権限が一覧で確認できる。

* 勘定科目・補助科目の参照
* 連携サービスの参照
* 部門の参照
* 仕訳の参照・登録・更新・削除
* 事業者の参照
* 帳票・税の参照
* 取引先の参照・登録
* 明細・証憑の作成・更新

内容を確認して「許可」をクリックすると次の画面に進む。

**5. 認可コードを Claude に貼り付ける**

許可後の画面に認可コードが表示される。コードをコピーして Claude のチャットに貼り付けると接続完了だ。  
![](https://static.zenn.studio/user-upload/f7fab4b828c8-20260331.png)

この画面には「**データ収集・学習機能のオプトアウト設定を事前に行ってください**」という注意書きもある。Claude.ai の場合は設定から学習利用をオフにできるので、業務データを扱う前に確認しておきたい。

トークンの有効期限は1時間で、期限切れ後は同じ手順で再認証が必要になる。

## 実際に使ってみた ― ユースケース別デモ

筆者の環境（個人事業主、2026年会計期間）で実際に動作を確認した。

### ユースケース1：領収書を渡して仕訳を登録する

打ち合わせの手土産として購入した菓子の領収書（PDF）を Claude に渡して「仕訳を作って」と伝えてみた。

Claude は領収書から以下を読み取って仕訳を組み立ててくれた。

* 購入日：2026-03-25
* 金額：¥2,480（軽減税率8%、内消費税¥183）
* 支払い方法：クレジットカード → 貸方は**未払金**
* 用途：翌日の顧客候補先との打ち合わせで手土産として渡した → 借方は**接待交際費**
* 摘要：顧客候補先への手土産 <お店の名前>

複式簿記に慣れていない段階だと「これは接待交際費？それとも会議費？」「クレジット払いだから未払金になるんだっけ？」という判断に毎回迷う。Claude に用途を自然言語で説明するだけで科目の判断まで任せられるのは助かった。

MoneyForward の仕訳画面で確認したところ、「外部連携（API）」タグが付いた状態できちんと登録されていた。  
![](https://static.zenn.studio/user-upload/db447b3b3493-20260331.png)

### ユースケース2：クレジットカード明細の突合と未払金管理

フリーランスにとって地味に手間なのが、**カード引き落とし後の未払金仕訳へのメモ・タグ付け**だ。

典型的なフローはこうなる。

1. カードで経費を支払う → `（費用科目）/ 未払金` で仕訳を作成
2. 翌月、口座から引き落とされる → `未払金 / 普通預金` で消込仕訳を作成

問題は「2. の消込仕訳が、1. のどの支払いに対応するか」を管理しきれなくなることだ。明細が増えてくると、後から見返したときに何の引き落としか分からなくなる。

これを Claude に解決させたのが以下のフローだ。

**① カードの利用明細（PDF やスクリーンショット）を Claude に渡す**

```
「この明細を読んで、各明細行を科目・金額・用途に分類して」
```

Claude が明細を解析し、「〇〇 3,800円 → 通信費（AI費用）」のように勘定科目の候補を提示してくれる。

**② 突合が取れた仕訳に、まとめてメモとタグを書き込む**

Claude に「この明細と対応する仕訳にメモとタグを付けて」と伝えると、MoneyForward の仕訳を一括で更新してくれる。メモには引き落とし月や締め日を、タグには「カード精算済み」のような管理用ラベルを付与するイメージだ。

タグを統一しておくことで、MoneyForward の画面上でタグ検索するだけで「処理済み」「未処理」の仕訳を即座に絞り込めるようになる。

手動でやると明細1件ずつ仕訳を開いてメモを入力する必要があるが、Claude に明細ごとまとめて渡せば一括処理できる。月次締め作業の時間が体感で半分以下になった。

**⚠️ ハマりどころ：一部の仕訳が更新できないことがある**

実際に試したところ、6件中4件は正常に更新できたが、残り2件でエラーが発生した。エラーになった仕訳に共通していたのは、MoneyForward 側の登録経路（手入力・連携自動取込など）が他と異なっていた点だ。登録経路によって内部的な仕訳の形式が違うらしく、Claude 経由の更新操作をすべての仕訳が受け付けるわけではないようだった。

| # | 摘要 | 金額 | 結果 |
| --- | --- | --- | --- |
| #20 | クラウドサービス利用料 12/31 | ¥8,300 | ✅ 更新成功 |
| #15 | 会議費（外部打合せ） | ¥5,700 | ✅ 更新成功 |
| #18 | 回線費用（事業用） | ¥5,602 | ✅ 更新成功 |
| #19 | 携帯電話料金 | ¥3,457 | ✅ 更新成功 |
| #17 | AI ツール利用料 | ¥3,785 | ⚠️ API エラー |
| #9 | 回線工事費（按分50%） | ¥7,654 | ⚠️ API エラー |

エラーになった仕訳は Claude が「MoneyForward 上で直接メモ欄を編集してください」と案内してくれたので、手作業でフォローした。完全自動化ではなく「大半は自動、残りは手動」という運用になることは念頭に置いておきたい。

## できること・できないこと

### できること

* 自然言語での仕訳作成・更新
* 勘定科目ツリーの参照と解釈
* 事業者情報・会計期間の確認
* Claude による科目提案（「この支出はどの科目？」）
* 複数の操作を連鎖させたワークフロー

### できないこと（現時点での制約）

* **仕訳の削除**：削除操作に対応していない。誤った仕訳は MoneyForward の画面から手動削除が必要
* **帳票の出力**：PDF や CSV へのエクスポートは対応外
* **確定申告書の生成**：あくまで会計データ層の操作に留まる
* **トークン有効期限**：1時間でリフレッシュが必要。長時間のセッションには不便

### セキュリティ面の考慮点

* トークンはチャット履歴に残る。共有アカウントでの利用は避ける
* **認可コードが AI ツールの学習に使用されないよう、データ収集を許可しない設定で利用する**（公式ガイドでも明記されている）
* アプリポータル側で付与した権限はクラウド会計側の権限設定に関わらず有効になる。権限付与対象ユーザーは慎重に選ぶこと
* Claude.ai のデータ利用ポリシーを確認した上で業務利用する

## まとめ・所感

正直、触ってみてやばいと思った。

特に未払金の管理フローが一変した。カードの引き落としが来たら明細を Claude に渡して、引き落とし済みの仕訳にタグを付けてもらう。それだけで「タグのついていない未払金仕訳＝まだ引き落とされていない支払い」の一覧が即座に出来上がる。月次の締め作業がほぼ一瞬で終わってしまった。

領収書を渡して仕訳を作るフローも想像以上だった。複式簿記に慣れていない段階では「これは接待交際費か会議費か」「クレジット払いだから貸方は未払金か」という判断に毎回迷う。それを自然言語で用途を説明するだけで Claude が解決してくれる。

現状の制約として、トークンが1時間で切れることと、一部の仕訳がエラーによって更新できないケースがある点は把握しておきたい。完全自動化というより「判断と入力を Claude に任せて、自分は確認するだけ」という分担が現実的だ。

開業準備中でまだ会計に慣れていないフリーランスにとって、こういう使い方は特に効くと思う。法人化で仕訳がさらに増えるタイミングで、改めて本格的に活用していきたい。
