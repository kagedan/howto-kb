## はじめに

同居人との共有生活費の管理が面倒で、レシート管理アプリの開発を始めました。

スーパーで自分のお菓子や化粧品と一緒に、共有で使うトイレットペーパーを買うとき、  
自分の買い物が大半だから、自分のカードで会計します。  
後から「あのトイレットペーパー400円、共有口座から振り替えないと…」と思い出すのが面倒で、結局振り替えないまま放置してしまう。  
そこで、レシート画像をアップロードし、商品ごとに「共有 / 自分」を振り分けて、  
「共有口座から自分の口座にいくら振り替えるべきか」を自動計算するアプリを開発することにしました。  
今回はその第一歩として、レシート全体をOCRで読み取り、集計するMVPを実装しました。

## 最終的に作りたいもの

以下の機能を持つアプリを構想中：

1. レシート画像をアップロード
2. 商品ごとに読み取って、各商品を「共有 / 自分」に分類
3. 月ごとに集計して、「共有口座から自分の口座にいくら振り替えるべきか」を自動計算

## 今回実装したところ（MVP）

今回は以下を実装：

* ✅ レシート画像をアップロード
* ✅ OCRで店名・日付・合計金額を自動抽出（商品別はまだ）
* ✅ レシート全体を「共有 / 自分」に分類
* ✅ 月別の合計金額を算出

商品別の読み取りと精算計算は次回実装予定。

### 使用技術

* **S3**: 画像保存
* **Lambda**: 画像処理・OCR・API
* **DynamoDB**: レシートデータ保存
* **API Gateway**: HTTPエンドポイント
* **Claude API（Anthropic直API）**: OCR（画像からテキスト抽出）  
  OCRはAmazon BedrockではなくAnthropic社の直APIを使用。Lambda関数から <https://api.anthropic.com> に直接リクエストを送る構成です。

すべてサーバーレス構成で、インフラ管理不要。

## Claude APIを選んだ理由

AWS Textractではなく、Claude APIを選んだ理由：

1. **実装がシンプル** - 画像を送って「このレシートの店名・日付・金額を抽出して」と聞くだけ
2. **レシート以外にも応用できる** - 将来的に領収書や納品書にも対応しやすい
3. **コストが安い** - 月100枚で約$0.60（Textractは$1.50）

切り替え可能な設計にしてあるので、運用してみて不満が出たらTextractに変更できる。

## 実装の詳細

### 1. API設計

3つのエンドポイント：

* `POST /upload` - 画像アップロード＋OCR＋DB保存
* `PATCH /update` - カテゴリ更新（共有 / 自分）
* `GET /monthly?year_month=202605` - 月別集計

### 2. Claude APIで画像からテキスト抽出（Lambda関数のコード）

Python

```
def extract_text_claude(image_base64):
    """Claude APIで画像からテキスト抽出"""
    data = {
        "model": "claude-haiku-4-5-20251001",
        "max_tokens": 1024,
        "messages": [{
            "role": "user",
            "content": [
                {
                    "type": "image",
                    "source": {
                        "type": "base64",
                        "media_type": "image/jpeg",
                        "data": image_base64
                    }
                },
                {
                    "type": "text",
                    "text": "このレシートから以下の情報を抽出してJSON形式で返してください：店名(store)、日付(date)、合計金額(amount)。金額は数字のみ。"
                }
            ]
        }]
    }
    
    # APIリクエスト送信
    req = urllib.request.Request(
        "https://api.anthropic.com/v1/messages",
        data=json.dumps(data).encode('utf-8'),
        headers={
            "x-api-key": ANTHROPIC_API_KEY,
            "anthropic-version": "2023-06-01",
            "content-type": "application/json"
        }
    )
    
    with urllib.request.urlopen(req) as response:
        result = json.loads(response.read().decode('utf-8'))
        text_content = result['content'][0]['text']
        
        # JSONブロックを抽出
        if '```json' in text_content:
            text_content = text_content.split('```json')[1].split('```')[0].strip()
        
        return json.loads(text_content)
```

**ポイント：**

* プロンプトで「JSON形式で返して」と明示することで、構造化データを取得
* Claudeが`json`で囲んで返すので、それを除去してパース
* **モデルはコスト重視でHaikuを選択**（月100枚で約$0.60）。将来的に商品別OCRなど複雑な読み取りが必要になった場合は、Sonnet（`claude-sonnet-4-6`）への切り替えを検討。
* 本番運用では、`json.loads()` が失敗する可能性があるため、try-except でのエラーハンドリングやリトライ処理を追加すべきです。今回はMVPとして最小限の実装にしています。

### 3. DynamoDBのデータ構造

パーティションキー（PK）  
・receipt\_id: String - ユニークID（例：receipt\_20260503\_125844）

属性  
・upload\_date: String - アップロード日時（YYYYMMDD\_HHMMSS形式）  
・image\_url: String - S3のパス  
・store: String - 店名  
・date: String - レシート日付（OCRで抽出した生データ）  
・amount: Number - 合計金額  
・category: String - 分類（共有 / 自分 / 未分類）
