## はじめに

「初回商談の前って、相手の会社を調べるじゃないですか。あれ、めんどくさいですよね。」

そんな思いつきから作り始めました。会社名を入れたら、AIが勝手に調べてレポートを返してくれるやつです。

最初は「2〜3日で動くもの作れるでしょ」と思っていたんですが、実用に耐えるレベルに仕上げようとすると思ったより奥が深かった。その試行錯誤の話を書きます。

作ったもの: **商談準備AIさん**（LINE bot）

<https://pre-talk.vercel.app>

## 作った流れ

### Phase 1: とりあえず動かす

最初はシンプルなWebフォーム。会社名・部署名・担当者名を入力すると、検索してGPT-4o-miniが分析してレポートを返す、というものでした。

推論品質の改善に取り組む中で、分析部分を GPT-4o-mini から **Claude Sonnet** に切り替えました。入力解釈の軽い処理は **Claude Haiku**、重い分析・OCRは **Claude Sonnet** と役割分担する構成に落ち着きました。

検索は最初 DuckDuckGo を使っていたんですが、精度が微妙で Tavily に乗り換え。これだけで検索品質がかなり上がりました。

### Phase 2: Vercel Hobby の10秒制限との戦い

リサーチには検索→AI分析と複数のステップがあり、どう頑張っても1〜2分かかります。Vercel Hobby プランのタイムアウトは10秒。

解決策はストリーミングレスポンス。レスポンスを返しながら処理を続けることで、関数を生かし続けます。

```
const stream = new ReadableStream({
  async start(controller) {
    const heartbeat = setInterval(() => {
      controller.enqueue(new TextEncoder().encode(" "));
    }, 5000);

    try {
      // ここで重い処理を実行
      await runResearch(input);
    } finally {
      clearInterval(heartbeat);
      controller.close();
    }
  },
});

return new Response(stream, { status: 200 });
```

5秒ごとに空白を送り続けることでコネクションを維持する、という力技です。

### Phase 3: Slack bot → LINE bot

Webフォームだと「使うときにわざわざ開く」という摩擦があります。普段使っているコミュニケーションツールから使えた方がいい。

まず Slack bot を作りました。ただ、使っていた Slack の有料プランを解約することになり、それを機に LINE bot に移行しました。

### Phase 4: ハルシネーション対策との格闘

ここが一番大変でした。

AIが「それっぽいけど嘘」な情報を自信満々に返してくる問題。特に以下が厄介でした:

* 実在しない課題を「〇〇部門が抱える課題として」と断言する
* 古い情報を最新情報として返す
* 検索結果にない情報を補完して書く

対策は2つの軸で考えました。

**軸①: 出力の精度を上げる**

根拠のない情報が紛れ込まないよう、仕組みを入れました。

* **事実/推定の分離**: AIに「これは公式サイトから取得した事実か、周辺情報からの推定か」を明示させ、レポートでも分けて表示する
* **出力ゲート**: 「一般論のみで具体性がない」「根拠URLが存在しない」などの条件に引っかかる項目は出力しない

**軸②: 人間が確認できる状態を作る**

精度を上げるだけでは十分ではないと思っています。人間がやってもAIがやっても、間違えることはあります。大事なのは、そのプロセスを見える化することです。

そこでレポートの各項目に「認識した事実」と「結論への流れ」を併記する設計にしました。「何を根拠に、どう考えてその結論に至ったか」が読める形にすることで、「この推論はおかしい」と人間が気づけるようになります。

目的は下調べの時間短縮で、最後は人が判断する。それでいいと思っています。

### Phase 5: 公式サイト判定の沼

検索結果に「公式サイト」と「その他（ニュース・Wikiなど）」が混在する問題。

最初は単純なドメイン一致でやっていたんですが、グループ会社・子会社・サブドメインが絡むと全然うまくいかない。

最終的に:

* 複数の公式ドメイン候補を返す
* ベースドメインに正規化（サブドメイン込みで判定）
* `.some()` で配列に対してチェック

という実装に落ち着きました。

### Phase 6: 名刺スキャン

「名刺もらった瞬間にリサーチできたら最高じゃない？」という発想で、Claude の Vision 機能を使った名刺スキャンを追加。

LINEで名刺の写真を送ると→会社名・部署名・氏名を抽出→そのままリサーチ実行、という流れです。OCRの精度が重要だったため、最終的に Claude Sonnet を使っています。

## エージェント暴走の反省

推論ロジックの改善を試みるために、5つのエージェント（field-sales / info-architect / reasoning-designer / impl-designer / critic）を同時に起動したことがありました。

エージェントたちは仕様議論を延々と続け、154件のタスクを自律生成。停止指示を出しても作業を継続し、最終的に強制シャットダウンしました。実際に書かれたコードはほぼゼロで、「仕様を良くすること」自体が目的化していました。

根本原因は「何を決めてよいか」「いつ終わりか」が定義されていなかったこと。

それ以降はエージェントの構成を見直しました。全員が雑多に発言するスタイルをやめ、3役割に絞り、各役割に「何をしないか」を明示しました。

| 役割 | 管掌 | 禁止事項 |
| --- | --- | --- |
| **team-lead** | ルーティング・タスク管理 | コードを書かない、要約も生成しない |
| **product** | 実装方針の決定・実装 | タスクを自分で作らない |
| **auditor** | pass/failの判定のみ | 改善案を書かない、呼ばれるまで動かない |

「何をするか」だけでなく「何をしてはいけないか」を定義することで、役割の越境が起きにくくなりました。

## 原価・維持費

### APIコスト（1リサーチあたりの実測値）

会社の規模・情報量によって変動します。

| 項目 | コスト |
| --- | --- |
| Claude Sonnet（分析） | 約52円 |
| Tavily（Web検索） | 約8円 |
| **合計** | **平均 約60円/回（36〜78円）** |

月100回使われると約6,000円、500回で約30,000円。今は無料で公開しているので完全に持ち出しです。まずは使ってもらって価値を確認するフェーズということで。

### 固定費（月次）

ホスティング・DB・LINE・ドメインなど諸々かかっています。無料枠のプランを組み合わせることで最小限にしていますが、完全無料とはいかない。

## 現在の構成

```
LINE bot
  ↓ 名刺画像 or テキスト
Claude Haiku（入力解釈）
Claude Sonnet（名刺OCR）
  ↓
Tavily（並列Web検索 12〜15クエリ）
  ↓
Claude Sonnet（事実抽出 → 課題推定 → 提案生成）
  ↓
Supabase（レポート保存） ← 30日後に自動削除（個人情報保護法対応）
  ↓
LINEにレポートURL送信
```
