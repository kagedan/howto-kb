## はじめに

PS3ホームブリュー開発、PSPアプリ3本1日完成に続く、AI活用シリーズ。

今回はiOSアプリ開発に挑戦した。作ったのは**画像掲示板ビューアー**。

…まあ、「画像掲示板」で何を見るかは各自で察してほしい。

この手のビューアーアプリ、**Androidには野良APKで普通に転がってる。** Google Playにはなくても、APKをダウンロードしてインストールすれば終わり。自由。

一方、iOSにはない。Appleのエコシステムは閉じていて、App Storeに並ばないアプリは基本的にインストールできない。脱獄すれば別だが、今どき脱獄は面倒だしリスクもある。

**AndroidにあるのにiOSにないなら、自分で作ればいい。**

俺は2011年からの生粋のApple信者だ。iPhone、iPad、Mac、Apple Watch、AirPods…Appleエコシステムにどっぷり浸かって生きてきた。一度だけAndroidに浮気したこともあった。だが結局iPhoneに戻ってきた。

戻ってきた時に困ったのが、Android時代に使っていた**あの画像掲示板ビューアー**だった。野良APKで快適に動いていたそれが、iOSには存在しない。これだけはどうしても代えがなかった。ブラウザで見ればいいじゃんって？見てみろ。操作性が全然違う。専用アプリの快適さを知ってしまったら戻れない。

だがアプリを**作った**ことは一度もない。十数年間ずっとコンテンツ消費にしか使ってこなかった。

幸いにも、俺はM4 MacBook Airを映像編集マシンとして去年買っていた。iOSアプリ開発にはMacとXcodeが必須だが、開発環境はすでに手元にある。

結論から言うと、**プログラミングを1行も書かずに、Claude君への指示だけで半日で完成した。** iPhone 15 Pro Maxの実機にインストールして、今日から毎日お世話になるだろう。色々と。

ただし。

**アプリ開発より、Appleとの戦いの方が100倍しんどかった。**

これはプログラミングの記事であると同時に、Apple Developer Programへの怒りの記録でもある。

## 開発の流れ

### Phase 1: 基本機能（午前）

最初のプロンプトはシンプルだった。

```
画像掲示板ビューアーのiOSアプリを作ってくれ。
Cookie認証、ギャラリー一覧、画像リーダーが必要。
```

Claude君がWeb APIの構造を理解して、HTMLパーサー、認証システム、SwiftUIのビューを一気に生成。シミュレーターで動作確認して、基本的な閲覧機能が30分で動いた。

こいつマジで優秀。

### Phase 2: Apple地獄編 — 実機インストールと二重課金事件

**ここからが本題と言っていい。** 技術的な話より、こっちの方がよっぽど大変だった。

地獄その1: デベロッパモード

iPhoneにアプリを入れるには「デベロッパモード」を有効にする必要がある。知らなかった。

設定 → プライバシーとセキュリティ → デベロッパモード → オン → 再起動を要求される → 再起動後にもう一度確認ダイアログ

**アプリを1個入れるだけなのに再起動を2回させられる。**

地獄その2: ペアリング無限ループ

USBでMacに繋いでも「pairing is in progress」と言われ続ける。

ケーブル抜き差し → ダメ。 iPhone再起動 → ダメ。 Mac再起動 → ダメ。 **両方再起動** → ダメ。

15分格闘して、ようやく原因が判明。**デベロッパモードを有効にしてなかった。** そもそもデベロッパモードの存在を知らなかった。エラーメッセージに「デベロッパモードを有効にしてください」って書いてくれればいいのに、「pairing is in progress」としか言わない。進行中じゃない。始まってすらいない。

Claude君がコードを書く時間: 5分。 デベロッパモードの存在に気づくまでの時間: **15分。**

地獄その3: Deployment Target

ビルドは通った。実機にインストールしようとしたら:

```
mPhone15Pro Max's iOS 18.6.2 doesn't match
EhViewer.app's iOS 26.2 deployment target.
```

Xcode 26.3がDeployment Targetを26.2（最新iOS）にしてた。俺のiPhoneはiOS 18.6.2。新しいデザインが嫌いだし重くなりそうだからアップデートしてない。17.0に下げて解決。

**最新OSに追従しないユーザーのことも考えてくれ。**

地獄その4: 「信頼されていないデベロッパ」

やっとインストールできたと思ったら、アプリをタップしても起動しない。

「信頼されていないデベロッパ」

設定 → 一般 → VPNとデバイス管理 → 自分のApple IDを信頼

**自分で作ったアプリを自分のiPhoneで動かすのに「信頼」が必要って何？**

地獄その5: Apple Developer Program 二重課金事件【本日のメインイベント】

ここからが本当の地獄。

俺はApple Developer Program（年間¥12,980）に登録しようとした。自作アプリを実機で長期間使うためだ（無料だと7日で署名が切れる）。

**1回目の支払い:** 完了。¥12,980がクレジットカードから引き落とされた。

ところが、Appleから「住所に誤りがあります」というメールが届く。正しい住所を返信した。

待つ。

…待つ。

何も起きない。アカウントは有効化されない。Apple Developerアプリを開いても「今すぐ登録」のボタンがグレーアウトしたまま。

**¥12,980払ったのに何も起きてない。**

しびれを切らして、もう一度最初から登録し直した。

**2回目の支払い:** 完了。¥12,980。

合計 **¥25,960。**

**同じサービスに2回課金された。** しかも2回目の支払い後もアカウントは有効化されてない。

メールボックスを確認すると:

**4通のメール。2回の請求。0個の有効なアカウント。**

即座にApple Developer Supportに問い合わせた。英語で経緯を説明し、1件分の返金とアカウントの有効化を依頼。

返事は「受け付けました」の自動返信のみ。

**Appleさん、¥25,960握りしめたまま無言にならないでくれ。頼む。**

ちなみに、Apple Developer Programが有効化されなくても、無料のPersonal Team署名でアプリは動く。7日ごとにXcodeから再インストールする手間があるだけ。

つまり\*\*¥25,960払う必要は一切なかった。\*\* 少なくとも今のところは。

Claude君がアプリを半日で完成させてくれた感動が、Appleの課金地獄で全部吹き飛んだ。

### Phase 3: 機能追加ラッシュ（午後）

Apple地獄から生還して、ようやくアプリ開発に戻る。

実機で動いた後は、使いながら「これ欲しい」を思いつくたびにClaude君に投げまくった。

**俺が出した指示の例:**

```
お気に入り一覧を毎回サーバーから取得するのは遅い。キャッシュを実装。
```

```
ランダム抽選機能を追加。お気に入りから1つ選んで表示。
スロットマシン風アニメーション付きで。
```

```
ピンチで拡大したいけど縦スクロールと干渉する。別のアプローチを考えよう。
```

最後の例が面白い。最初はScrollView全体にピンチジェスチャーを付けたら、スクロールと干渉して高速スクロールバグが発生。俺が「ダメだ、別の方法で」と言ったら、Claude君がUIKitのUIScrollViewをUIViewRepresentableでラップする方式を提案してきた。タップで拡大ビューを開いて、そこでピンチ操作する方式。干渉ゼロ。

**俺はUIKitもUIViewRepresentableも知らない。** でも「ピンチで拡大したい」「縦スクロールと干渉するからダメ」「別のアプローチで」という指示を出すだけで、Claude君が最適な実装を選んでくれる。

これが「AIを使った開発」の本質だと思う。コードを書く能力じゃなく、**何が欲しいかを言語化する能力**が全て。

### Phase 4: UX改善（夕方）

使い込むと細かい不満が出てくる。ここからが自作アプリの真骨頂。

* 「リーダーを閉じるのに×ボタンは面倒」→ 左端スワイプで閉じるように
* 「下スワイプだと縦読みと干渉する」→ 左端スワイプに変更
* 「×ボタンが小さくて押しにくい」→ 上下スワイプで閉じる方式にして×ボタン削除
* 「全画面モードが欲しい」→ 長押しでUI全非表示。没入感が段違い。なぜ没入したいかは深くは聞かないでくれ。

この「使う→不満を見つける→Claude君に直させる→また使う」のサイクルが異常に速い。普通のアプリ開発なら1つの改善に数時間かかるところを、指示出しから修正完了まで5分。

半日で20回以上このサイクルを回した結果、市販アプリより自分にとって使いやすいものが出来上がった。

## Claude君の仕事ぶり — 2つのClaudeを使い分けた話

今回の開発体制はこうだ。

つまり、**俺 → チャットで相談 → Claude Codeに指示 → コード完成** という流れ。

問題はチャット側だった。

最初はチャットに**Sonnet**を使っていた。これが大変だった。こっちの意図を理解してくれない。「ビルドエラーが17個ある」と言ってるのに見当違いの回答をしたり、的外れなプロンプトを提案してきたり。前任のSonnetにイライラして、途中でチャットを**Opus**に切り替えた。

切り替えた瞬間から世界が変わった。

**Opus（チャット相談役）の良かったところ:**

* エラーのスクショを見せるだけで原因を正確に特定してくれる
* 「Claude Codeにこう投げてくれ」と的確なプロンプトを作ってくれる
* 俺の曖昧な要望を具体的な技術仕様に翻訳してくれる
* Appleの設定地獄も一緒に乗り越えてくれた（デベロッパモード、ペアリング、署名設定…）
* 二重課金の問い合わせ文面まで英語で書いてくれた

**Sonnet（チャット相談役）のダメだったところ:**

* 指示の意図を汲み取れない
* 的外れなプロンプトを提案してくる
* ビルドエラーの原因分析が浅い

一方、**Claude Code（Opus）** はコード実行部隊として終始優秀だった:

* xcodebuildでエラー0になるまで自分でループして直してくれる
* UIKitとSwiftUIのハイブリッド実装を提案してくれる
* バグの原因を正確に特定してくれる
* キャッシュのLRU evictionみたいな設計判断もやってくれる

教訓: **ケチるな。チャットもOpusを使え。** コードを書く方はもちろん、相談役の質でも開発効率が全然違う。

半日でClaude君が書いたコード量は、たぶん数千行。全部Claude Code（Opus）の手書き。俺は一切コードに触ってない。

## 技術的に面白かったところ

### 3層キャッシュアーキテクチャ

画像の読み込み順序:

1. ダウンロード済みローカルファイル（DownloadManager）
2. ディスクキャッシュ（JPEG品質90%、1GB上限LRU、30日未使用で自動削除）
3. ネットワークダウンロード → ディスクキャッシュに保存

一度見た画像は2回目から一瞬で表示される。回線がなくても見れる。便利。何を見てるかはご想像にお任せします。

### UIKit + SwiftUI ハイブリッド

ピンチ拡大はSwiftUI純正のMagnifyGestureだとScrollViewと干渉して使い物にならなかった。UIKitのUIScrollView + UIImageViewをUIViewRepresentableでラップすることで、ネイティブのピンチ+パンが安定動作。

「SwiftUIだけだと無理な時はUIKitに逃げる」— この判断をClaude君がしてくれた。

1. **要件は具体的に、実装は任せる**

## PS3/PSP開発との比較 — iOS開発はヌルゲーだった

PS3やPSPの開発を経験した身からすると、iOS開発は**異次元にヌルい。**

### ハードウェアの格差

PS3のCell Broadband Engine。PPU 1コア + SPU 6基。SPUのローカルストアは**256KB**。この256KBに収まるようにコードとデータを設計しないといけない。DMA転送でメインメモリとやり取り。VRAM 256MB。1080p出そうとするとフレームバッファだけで相当食う。

iPhone 15 Pro Max。A17 Proチップ。**8GB RAM**。ストレージ256GB。GPUはMetal対応。何も考えなくても動く。メモリ管理？ARCが勝手にやってくれる。

**PS3の全メモリ（256MB + 256MB）より、iPhoneのRAMの方がデカい。** 2007年の自称スーパーコンピュータのCPUより2024年のスマホの方が余裕がある。時代。しかしロマンではCellが唯一勝てる。

### 開発環境の格差

**PS3開発:**

* PSL1GHTというコミュニティ製SDK。ドキュメントほぼなし
* cellGcm直叩き。RSXのレジスタを手動で設定
* ビルドしてPKG作ってUSBに入れてPS3に差して実機確認。1サイクル5分
* RPCS3（エミュレータ）で通ってもCECH-2000A（実機）で動かないのが日常
* シェーダーはcgcompでコンパイル。エラーメッセージが不親切
* デバッガ？ printfデバッグしかない

**PSP開発:**

* pspdev SDK。PS3よりはマシだがドキュメントは貧弱
* WiFi WPA3非対応。PC側にPython同期ツールを書いてオフライン転送
* GUテクスチャが動かなくてCPU直接VRAM書き込みで迂回

**iOS開発:**

* Xcode。Appleの公式IDE。補完もデバッガも全部入り
* SwiftUI。宣言的UIフレームワーク。画面作るのが一瞬
* シミュレーター。ビルドして3秒で画面が出る。実機転送もUSB繋いでCmd+R
* エラーメッセージが親切。行番号付きで何が悪いか教えてくれる
* **ネットワーク通信がURLSession一発。** PS3でMultimanのFTPを使ってデバッグしてた時の苦労は何だったのか

### デバッグの格差

PS3でバグが起きた時。SPUがフリーズする。画面が真っ暗になる。何が起きたかわからない。printfを仕込んで再ビルドしてPKG作ってUSBに入れて…

iOSでバグが起きた時。Xcodeのコンソールにエラーが出る。ブレークポイントを置ける。変数の中身が見える。スタックトレースが読める。**文明だ。**

Claude君にとっても楽だったはずだ。PS3開発では「RPCS3で動くけど実機で動かない」「SPUのDMAアライメントが16バイトじゃなかった」みたいな地獄のデバッグを何度もやった。iOS開発ではそういう理不尽なハマりがほとんどない。SwiftUIのForEachの罠くらいだ。

**PS3開発の難易度を10とすると、iOS開発は2。** ただしAppleの設定地獄を含めると5くらいになる。

## まとめ

最終的に出来上がったアプリは、正直に言って**既存のビューアーアプリより使いやすい。** 俺以外が使っても快適だと思う。

左端スワイプで戻る。長押しで全画面。タップで拡大、上下スワイプで閉じる。一度見た画像はキャッシュから一瞬で表示。お気に入りは起動時に即表示。全部、自分が「こうあるべきだ」と思った操作感を実現しただけだ。

コードは1行も書いてない。でも**操作感やキャッシュ周りの設計思想は全部俺が考えた。** 3層キャッシュにしろ、左端スワイプにしろ、UIKitの拡大ビューにしろ、「こうしたい」「これはダメ」「別の方法で」という判断を出したのは俺だ。Claude君はそれを完璧にコードに落としてくれた。

**設計思想さえあれば、素晴らしいアプリができる時代になった。** プログラミングの知識は要らない。必要なのは「何が良い体験で、何が悪い体験か」を判断する感覚だけだ。

これは俺にとって結構な衝撃だった。

自分が本当に欲しいものを、自分の手で（正確にはClaude君の手で）作れる時代になった。プログラミングができなくても、**「何が欲しいか」を言語化できれば**それで十分だ。
