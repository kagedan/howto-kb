## はじめに

去年、本業のバックエンドエンジニア（Go/Python）として働きながら、副業を本気で試してみることにした。きっかけは単純で「技術スキルを収益に変えられるか」という実験的な興味だ。

ブログでよく見る「副業15選！」的な記事は多いが、実際にエンジニアが試した結果を具体的な数字・仕組みとともに書いた記事はほとんどない。この記事では、自分が実際に手を動かして試した15種の副業を、**月収・工数・再現性**の観点で評価する。

先に結論を言うと、**本当に稼げたのは15種中3種だけ**だった。

## 本当に稼げた3手法の詳細

### 1. 技術顧問契約（時給換算最高）

**月収20万円 / 月20時間 → 時給1万円**

これが最も効率が良かった。知人のスタートアップCTOから「週1でアーキテクチャレビューしてほしい」と声がかかったのがきっかけ。

#### 具体的な業務内容

* 週1回2時間のSlackレビュー＋月1回のオンラインMTG
* PRレビュー、インフラ設計相談、採用技術選定のアドバイス
* ドキュメント作成は最小限

#### どうやって案件を取るか

技術顧問はクラウドソーシングには少ない。以下の経路が現実的だ：

1. **Twitter/X での技術発信** → DMで相談が来るようになる
2. **勉強会・カンファレンス登壇** → 名刺交換から繋がる
3. **知人・元同僚からの紹介** → 最も多い経路

```

### 2. Udemyコース制作（初期工数は重いが長期収益）

**月収12万円 / 初期80時間 → その後はほぼ不労所得**

「Pythonで作るAPIサーバー入門」というコースを作成した。初期投資は時間だけで、機材は5,000円のマイクと既存のMacのみ。

#### コース設計の考え方

Udemyで稼ぐためには「検索ボリュームがあるが競合が薄いニッチ」を狙う必要がある。自分が調査に使ったPythonスクリプト：

```
import requests
from bs4 import BeautifulSoup
import time

def search_udemy_courses(keyword: str) -> dict:
    """
    Udemyの検索結果からコース数と評価数を取得する
    ※スクレイピングは利用規約を確認の上実施すること
    """
    url = f"https://www.udemy.com/courses/search/?q={keyword}&lang=ja"
    headers = {"User-Agent": "Mozilla/5.0"}
    
    response = requests.get(url, headers=headers)
    soup = BeautifulSoup(response.text, "html.parser")
    
    courses = soup.find_all("div", class_="course-card--container")
    
    result = {
        "keyword": keyword,
        "course_count": len(courses),
        "avg_rating": 0.0
    }
    
    ratings = []
    for course in courses:
        rating_el = course.find("span", class_="star-rating--rating-number")
        if rating_el:
            try:
                ratings.append(float(rating_el.text))
            except ValueError:
                pass
    
    if ratings:
        result["avg_rating"] = round(sum(ratings) / len(ratings), 2)
    
    time.sleep(1)  # レート制限対策
    return result

# 調査したいキーワードリスト
keywords = [
    "FastAPI 入門",
    "Go言語 API",
    "Terraform 実践",
    "Docker Kubernetes",
    "Python 自動化"
]

for kw in keywords:
    result = search_udemy_courses(kw)
    print(f"{result['keyword']}: {result['course_count']}件 / 平均評価{result['avg_rating']}")
```

#### 収益最大化のポイント

* **セール時（90%OFF）でも収益は出る**：Udemyの収益配分は複雑だが、自己集客なら97%、Udemy経由なら37%が目安
* **字幕は必須**：自動字幕でもあるとないでは完走率が大きく違う
* **Q&Aレスポンス速度**がレビュー評価に直結する

```

### 3. SaaS開発（最も高リターンだが最もリスクも高い）

**月収15万円（MRR） / 初期100時間**

「エンジニア向けの日報自動生成ツール」をSaaSとして開発・販売した。GitHubのコミットログからSlack投稿を自動生成するシンプルなツールだ。

#### アーキテクチャ

```
[GitHub Webhook]
      ↓
[FastAPI サーバー / Fly.io]
      ↓
[OpenAI API（要約・整形）]
      ↓
[Slack API（投稿）]
      ↓
[Stripe（課金管理）]
```

#### コアロジックの実装例

```
from fastapi import FastAPI, Request, HTTPException
from openai import OpenAI
import httpx
import os

app = FastAPI()
client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

@app.post("/webhook/github")
async def github_webhook(request: Request):
    payload = await request.json()
    
    # pushイベントのみ処理
    if "commits" not in payload:
        return {"status": "skipped"}
    
    commits = payload["commits"]
    commit_messages = [c["message"] for c in commits]
    
    # OpenAIで日報文章を生成
    summary = await generate_daily_report(commit_messages)
    
    # Slackに投稿
    await post_to_slack(summary, payload["repository"]["name"])
    
    return {"status": "ok", "summary": summary}

async def generate_daily_report(commits: list[str]) -> str:
    prompt = f"""
以下のコミットメッセージから、チームへの日報文章を生成してください。
- 箇条書きで3〜5項目
- 技術的すぎず、非エンジニアにも伝わる表現で
- 明日の予定も1行添える

コミット:
{chr(10).join(commits)}
"""
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}],
        max_tokens=300
    )
    return response.choices[0].message.content

async def post_to_slack(message: str, repo_name: str):
    slack_token = os.environ["SLACK_BOT_TOKEN"]
    channel = os.environ["SLACK_CHANNEL_ID"]
    
    async with httpx.AsyncClient() as client:
        await client.post(
            "https://slack.com/api/chat.postMessage",
            headers={"Authorization": f"Bearer {slack_token}"},
            json={
                "channel": channel,
                "text": f"📝 *{repo_name}* の本日の作業報告\n\n{message}"
            }
        )
```

#### 課金設計

```
# Stripeのwebhookで課金状態を管理
@app.post("/webhook/stripe")
async def stripe_webhook(request: Request):
    payload = await request.body()
    sig_header = request.headers.get("stripe-signature")
    
    try:
        import stripe
        event = stripe.Webhook.construct_event(
            payload, sig_header, os.environ["STRIPE_WEBHOOK_SECRET"]
        )
    except ValueError:
        raise HTTPException(status_code=400)
    
    if event["type"] == "customer.subscription.created":
        customer_id = event["data"]["object"]["customer"]
        # DBにアクティブユーザーとして登録
        await activate_user(customer_id)
    
    elif event["type"] == "customer.subscription.deleted":
        customer_id = event["data"]["object"]["customer"]
        # アクセスを無効化
        await deactivate_user(customer_id)
    
    return {"status": "ok"}
```

価格は月額980円に設定。153ユーザーで約15万MRRとなった。

## ハマったポイント・やってはいけないこと

### 1. 時間単価を計算せずに始めた

最初はクラウドソーシングに飛びついたが、**時給換算すると1,200円**しかなかった。本業の生産性を下げてまでやる価値はない。必ず事前に「月収目標 ÷ 投入工数」で時給計算をすること。

```
def calculate_hourly_rate(monthly_revenue: float, monthly_hours: float) -> dict:
    hourly_rate = monthly_revenue / monthly_hours
    annual_revenue = monthly_revenue * 12
    
    return {
        "時給": f"{hourly_rate:,.0f}円",
        "年収換算": f"{annual_revenue:,.0f}円",
        "判定": "✅ やる価値あり" if hourly_rate >= 3000 else "❌ 再考推奨"
    }

print(calculate_hourly_rate(80000, 40))   # クラウドソーシング → 2000円
print(calculate_hourly_rate(200000, 20))  # 技術顧問 → 10000円
print(calculate_hourly_rate(150000, 10))  # SaaS安定後 → 15000円
```

### 2. SaaSのインフラコストを見誤った

初期はHerokuで動かしていたが、ユーザーが増えるにつれコストが跳ね上がった。Fly.ioに移行してコストを1/3に削減。

```
# fly.toml の最適化例
[http_service]
  internal_port = 8080
  force_https = true
  auto_stop_machines = true   # アイドル時に自動停止
  auto_start_machines = true
  min_machines_running = 0    # 最小0台でコスト削減

[[vm]]
  memory = "256mb"  # 512MBから削減
  cpu_kind = "shared"
  cpus = 1
```

## 稼げなかった副業に共通した理由

* **YouTubeとブログ**：SEO/アルゴリズムの恩恵を受けるまでに6ヶ月以上かかる。継続が前提でエンジニアには向かない
* **ハッカソン賞金**：勝率が低く、再現性がない
* **テンプレート販売**：単価が低すぎてスケールしない
* **OSS有料サポート**：需要はあるが、マーケティングが難しい

## まとめ・今後の展望

15種を試した結論：

**エンジニアが副業で稼ぐための3原則**

1. **時給3,000円以下の案件は断る** → 本業の成長機会コストを考えると割に合わない
2. **初期投資型（Udemy・SaaS）に早めに着手する** → 時間が経つほど資産化する
3. **技術顧問はネットワーク次第** → 日頃からの発信・登壇が重要

今後は「SaaSのMRRを50万円まで伸ばす」ことを目標に、プロダクトのマーケティング自動化に取り組んでいる。副業はスキルアップとビジネス感覚を同時に鍛えられる最良の実験場だと感じている。
