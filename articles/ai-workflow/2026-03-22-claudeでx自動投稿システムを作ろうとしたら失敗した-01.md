今日はClaudeでうまくいかなかった話を紹介します。  
こんな時間になってしまいました…

  

## はじめに

「AIの力を借りれば、X（旧Twitter）の自動投稿なんて簡単にできるでしょ？」

そう思ってClaudeに相談したら、想像以上に道のりが長かった。でも、その「うまくいかない過程」こそがリアルで、同じことをやりたい人の役に立つと思ったので、包み隠さず書きます。

結論から言うと、**仕組みは完成した。でも、まだ1回も自動投稿できていない。**

---

## この記事で分かること

---

## きっかけ：「自動投稿の計画を立てて」

私はAI系の情報を発信しているXアカウント（@Bit\_AIagent）を運営しています。以前Claudeと一緒にX収益化のロードマップを作っていて、その流れで「自動投稿の仕組みも作ろう」という話になりました。

Claudeはすぐにロードマップに沿った自動投稿計画書をドキュメントにまとめてくれました。フェーズ1〜3の段階的な計画、投稿テンプレート、KPI設定まで。

…正直、この時点で**ちょっと付いていけなくなった。**

---

## 「付いていけない」と正直に言ったら変わった

Claudeが最初に出してきたのは、GitHub ActionsやCron式、tweepyといった技術用語が並ぶ計画書。理解が追いつかず、「付いていけていない。分かりやすく教えて」と正直に伝えました。

するとClaudeは、一気に説明のトーンを変えてくれた。

**「自動投稿というのは、要するに『自分がXを開かなくても、決めた時間に勝手にツイートしてくれる仕組み』のことです」**

ここからClaudeは、専門用語を使わず、1ステップずつ確認しながら進めてくれるようになりました。「ここまでできた？」「スクリーンショット見せて」「OK、次はこれ」という感じ。

**AIに「分からない」と伝えることって、めちゃくちゃ大事。** 分かったフリをして先に進むと、あとで取り返しがつかなくなる。

---

## ゼロから始めた4つのステップ

Claudeが整理してくれた全体像はこうでした。

**① X開発者アカウントを作る** → 済み（これは以前やっていた） **② GitHubアカウントを作る** → ここからスタート **③ 投稿スクリプトを作る** → Claudeが全部書いてくれる **④ GitHub Actionsを設定する** → コピペで完了

「GitHubって何？」レベルの私でも、Claudeの指示通りにやれば進められる…はずだった。

---

## GitHubアカウント作成 → リポジトリ作成

GitHubのアカウント作成自体は5分で終わりました。メールアドレスとパスワードを入れるだけ。

次に「リポジトリ」を作ります。Claudeいわく「プロジェクトのフォルダみたいなもの」。

名前を「x-auto-post」、公開設定を「Private（非公開）」にして作成。ここは問題なし。

…と思いきや、**「Add file」ボタンが見つからない。**

Claudeに「どこ？」とスクリーンショットを貼ったら、「Go to fileの右横の＋ボタンです」と教えてくれた。

**画面の見え方って、知ってる人と知らない人で全然違う。** AIに画面を見せながら進められるのは本当に助かった。

---

## 3つのファイルをコピペで設置

リポジトリに入れたファイルは3つ。

1. **scripts/post.py** — Xに投稿するプログラム本体
2. **content/queue/next.txt** — 投稿する文章を入れておくファイル
3. **.github/workflows/daily-post.yml** — 毎日17時に自動実行する設定

全部Claudeが書いてくれたコードをコピペしただけ。自分では1行もコードを書いていない。

投稿文の中身について、最初Claudeはハッシュタグ（#AI [#生成AI](https://note.com/hashtag/%E7%94%9F%E6%88%90AI) ）付きの短い投稿を提案してくれたけど、「ハッシュタグは付けたくない」「具体的な操作手順も入れたい」「文字数は気にせず読みやすさ重視で」と伝えたら、こんな感じの投稿文になった。

==========  
🛠️ Coworkって知ってる？  
Claudeがパソコンのファイル操作を自動でやってくれる神ツール💡  
たとえば「Downloads内のPDFを月別フォルダに整理して」と指示するだけで、Claudeが勝手にフォルダ作成→ファイル移動してくれる  
やり方👇  
① claude.aiの左メニューから「Cowork」を開く  
② 操作したいフォルダを選ぶ  
③ やりたいことを日本語で指示する これだけ  
30分かかってた整理が3分で終わる  
==========

**自分の意見を言えば、AIはちゃんと反映してくれる。** 「お任せ」にしないで、「こうしたい」を伝えるのが大事だと学んだ。

---

## ワークフロー設定で出てきた謎の画面

3つ目のファイル（.yml）を作っているとき、画面の右側に見慣れない「Marketplace」という画面が出てきた。

焦ってClaudeに「右側のが今までと違うけど平気？」と聞いたら、

**「GitHubが.ymlファイル編集時に自動で表示するおすすめ機能の一覧です。無視して大丈夫です」**

とのこと。初心者は画面に知らないものが出てくるだけで不安になる。こういう一言があると安心して進められる。

---

## いよいよテスト実行 → 失敗

全ファイルの設置が終わり、GitHub Actionsの「Run workflow」ボタンを押してテスト。

30秒後…

**赤いバツ。失敗。**

エラーを見ると 503 Service Unavailable。Claudeの説明によると、「X側のサーバーが一時的に応答しなかった」ということらしい。

---

## リトライ機能を追加 → また失敗

Claudeが「失敗したら30秒待って再試行する機能」をスクリプトに追加してくれた。

再実行。

**サーバーエラー（1回目）→ 30秒後にリトライ → サーバーエラー（2回目）→ 30秒後にリトライ → サーバーエラー（3回目）→ 3回失敗しました。**

全滅。

---

## リトライをさらに強化 → それでも失敗

Claudeが待ち時間を長く、回数も5回に増やしてくれた。60秒→120秒→180秒→240秒と徐々に間隔を空けるバージョン。

再実行。

**5回全部失敗。**

さすがにここで「APIキーが間違ってるのでは？」と疑い始めた。

---

## APIキーは合っていた

Claudeに聞いたら、こう整理してくれた。

* APIキーが間違い → **401エラー**が出る
* 権限が不足 → **403エラー**が出る
* X側の一時的な問題 → **503エラー**が出る

今回は503なので、APIキーは正しく認識されている。念のためX Developer Portalで権限を確認したら「Read and write」で問題なし。

**つまり、こちら側には何の問題もなかった。** X API Freeプランのサーバーが不安定すぎるだけ。

---

## 方針転換：別の方法を探す

何度やっても503エラーが出るので、Claudeに「他の方法はないか」と聞いた。

出てきた選択肢は4つ。

1. **X公式の予約投稿** — 手動だけど確実
2. **Googleスプレッドシート＋GAS** — 無料で自動化できる
3. **SocialDog** — 日本語対応の管理ツール
4. **GitHub Actionsのまま様子見** — APIが安定すれば成功する

①の予約投稿は確実だけど、毎回手作業でコピペが必要。それだと「自動投稿」じゃない。

結局、②の**Googleスプレッドシート＋GAS**を次に試すことにした。同じX API Freeプランを使うけど、GAS版のツールには503エラー対策が組み込まれているものもあるらしい。

---

## 今日の成果まとめ

**できたこと：**

* GitHubアカウント作成（ゼロから）
* x-auto-postリポジトリ作成（Private）
* X APIキー4つをSecretsに安全に保存
* 投稿スクリプト（リトライ機能付き）設置
* GitHub Actions（毎日17時JST自動実行）設定
* X Developer Portalの権限確認（Read and write）

**できなかったこと：**

**学んだこと：**

* AIに「分からない」と正直に言うと、説明レベルを合わせてくれる
* スクリーンショットを見せながら進めると、画面の「どこ？」問題が解決する
* 投稿文は「お任せ」にせず、自分の意見を伝えた方がいいものができる
* X API Freeプランは503エラーが頻発する（これは広く知られている問題）
* うまくいかなくても、別のアプローチは必ずある

---

## 今後の予定

Googleスプレッドシート＋GASで自動投稿のセットアップに挑戦します。

GitHub Actionsの仕組みはそのまま残してあるので、X APIが安定したらそっちも動き出すかもしれない。二段構えで進めます。

**「自動投稿くらい簡単でしょ」と思って始めたけど、全然そんなことなかった。** でも、AIと一緒に1つずつ解決していく過程は、思った以上に楽しかった。

次こそ、自動投稿を成功させたい！
