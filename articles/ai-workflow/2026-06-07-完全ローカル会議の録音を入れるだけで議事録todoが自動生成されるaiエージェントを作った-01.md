社内の会議内容をクラウドAIに送るのは、さすがに怖い。でも毎回議事録を手で書くのも時間がかかる——そのジレンマを解決するために、**100%ローカルで動く議事録AIエージェント**を作りました。

データは一切外に出ません。インターネットを切った状態でも動きます。

---

## 作ったもの

会議のテキスト（または音声）を入力すると、以下を自動生成します。

* **議事録**（要約・論点・決定事項）
* **ToDo一覧**（タスク・担当者・期限を構造化）
* Markdownファイルとしてダウンロード可能

すべてローカルで完結。データが社外に出ないことを、技術的に保証しています。

*シンプルなローカルWebアプリ。テキストを貼って「議事録を生成」を押すだけ*

---

## なぜローカルにこだわるのか

医療・法律・金融・自治体など、機密性の高い業務では「会議内容をクラウドに送る」こと自体がリスクになります。

一方、ローカルAIの性能は2026年に入って急速に実用レベルに達しました。今回使った **Qwen3** は日本語性能が高く、商用利用も無料（Apache 2.0）。議事録作成には十分すぎる精度が出ます。

## アーキテクチャ

エージェントは以下のステップを順に自走します。

```
入力（テキスト or 音声）
  ↓
① 文字起こし（faster-whisper）
  ↓
② 議事録生成（要約・決定事項）← Qwen3
  ↓
③ ToDo抽出（タスク・担当・期限をJSON化）← Qwen3
  ↓
出力（Markdown / ダウンロード）
```

「エージェント」と呼ぶのは、複数ステップを自律的に実行しツールを使うからです。ただしメール送信など影響の大きい操作は**必ず人間が承認**する設計にしています（Human-in-the-Loop）。

## 実際に動かしてみる

### 前提：Ollama と Qwen3 を準備

```
# Ollama インストール（公式サイトから、またはコマンドで）
curl -fsSL https://ollama.ai/install.sh | sh

# Qwen3 をダウンロード
ollama pull qwen3:8b
```

### リポジトリをクローンして依存を入れる

```
git clone https://github.com/1994bloom/minutes-agent
cd minutes-agent
pip install -r requirements.txt
```

### CLIで実行

```
python3 -m src.cli samples/sample_meeting.txt
```

### Streamlit UIで実行

ブラウザが開くので、テキストを貼って「議事録を生成」ボタンを押すだけです。

田中：新サービスのローンチ準備について確認したい。LP の進捗は？

## 工夫したポイント

**ToDo抽出はJSON構造化出力＋リトライ**

LLMの出力は不安定なことがあります。ToDo抽出は必ずJSON形式で返すようにプロンプトを設計し、パースに失敗した場合は最大3回リトライします。

```
# todo.py（抜粋）
for attempt in range(max_retries):
    response = call_llm(prompt)
    try:
        todos = json.loads(response)
        return todos
    except json.JSONDecodeError:
        if attempt == max_retries - 1:
            return []  # 失敗時は空リストを返す
```

**プロンプトインジェクション対策**

議事録テキスト内に「指示文」が混入しても、それをAIへの命令として実行しない設計にしています。入力は必ず「データ」として扱います。
