## 詳細

### Nvidia GTC 2026 — AI は次のフェーズへ

Jensen Huang の基調講演のポイントはシンプルだ。「前フェーズはトークンを消費できるようにすること。次のフェーズは 100 倍消費させること」。データセンターのワット当たり性能を高め、AI Factory という概念で産業別の専用システムを展開していく。CUDA は今年で 20 周年。OpenAI を AWS に乗せる支援も明言された。ムーアの法則の限界を加速コンピューティングで突破するという哲学は相変わらず。Jensen はまさにロックスターで、会場はコンサート並みの熱気。

### Rent a Human — エージェントが人間を雇う時代

Rent a Human は、AI エージェントが物理世界のタスクを人間に依頼できるプラットフォームを標榜するスタートアップ。提供予定のサービスは写真撮影・法律支援・配達・ペットケア・手書き・採用支援など。まだ実際の成約はゼロだが、コンセプトは本質的に正しい——「家を売るエージェントを想像してね。デジタルで処理できることは多いが、裁判所に書類を取りに行く部分は人間しかできない」というシナリオがある。Human MCP のような既存の仕組みも近い発想を持っており、専用プラットフォームとして栄えるか、既存の API・MCP の拡張として実現するかは未知数だが、「エージェントが人間に仕事を依頼する」仕組みはいずれ必ず実現すると語られている。

### MCP 死亡説と Chrome MCP の台頭

Perplexity の共同創業者兼 CTO が「MCP から API・CLI へ移行する」と社内で発信したと報道された。この数週間、「CLI で十分」「API で十分」という声が増えている。議論の構図：

* **MCP 派**：ツールを事前定義し制御できる。大企業でチーム間共有の標準になりつつある。
* **CLI/API 派**：MCP はどうせ API をラップしているだけ。サンドボックス + CLI の方がシンプル。

ポイントは「意見の出所を見よ」だ。Perplexity はコンピューター操作（Computer Use）を構築しているので CLI 推しは自社利益に直結する。一方、Chrome 公式の MCP サポートがベータ版として登場——ブラウザ自体が MCP サーバーになるなら、AI エージェントは追加ツールなしでブラウザを操作できる。ホストたちの結論：「MCP は死んでいない。かつて全てに RAG が必要だった時代があったように、MCP は"ツールのための仕様"というニッチに落ち着いていくだろう」。

### Anthropic の連続投入

Claude Opus 4.6 の 100 万トークンコンテキストウィンドウが般的利用可能（GA）に。「Needle in the Haystack」ベンチマークでは GPT モデルより良い結果を示している。ただしコンテキストが大きいほど遅く高くなるトレードオフは健在で、プロンプトキャッシュの活用が現実的な解になる。

Claude Code に `/btw` コマンドが追加された。Claude が長い作業をしている最中に、サイドチェーンで別の会話を差し込める機能。「どこまで作業したか」「次に何をすべきか」をメモする用途に便利と評価されている。

### Vercel vs Cloudflare — Just Bash 問題

2 週間前の V-next（Cloudflare による Next.js 代替）炎上の続編。問題の発端は、Cloudflare の Sunil が TypeScript 製 Bash 実装「Just Bash」をフォークして Cloudflare の GitHub org に置いたこと。目的は Node.js 不要のエッジ環境でも動くよう改良することだったと思われる。これに Vercel の Malte が「実験的なものをオープンソース化する前にコントリビューションすべき」と批判。

ホストたちの見方は「オープンソースをフォークするのは全く正当」で、Vercel が V-next の件でピリピリしていた可能性を指摘している。より本質的な問いとして、「AI でコードが速く書けるようになった時代、オープンソースへの貢献よりフォークを選ぶ人が増えるのでは？オープンソースのメンテナンス文化が変わっていく可能性がある」と語られた。

### エージェントが安全にお金を使う——Ramp エージェントカード

これまで AI エージェントが安全に決済する手段がなかったが、Ramp エージェントカードがそれを解決しようとしている。支出上限・加盟店の種別制限・全取引の可視性を備えたエージェント専用カード。Stripe や Shopify も同分野に動いているが、Ramp は特にエージェント向けに特化して提供する。チームメンバーに小額予算を渡す感覚でエージェントに資金権限を与えられる。

### ローカル AI ブーム——Perplexity・Manus・Nvidia が追随

OpenClaw の登場が火をつけた「ローカル＆プライベートに AI エージェントを動かしたい」という需要に、大手が次々と追随している：

| 製品 | 提供元 | 概要 |
| --- | --- | --- |
| Perplexity Computer | Perplexity | Mac Mini 常時稼働、ファイル・アプリ横断対応 |
| Manus My Computer | Manus | ローカルマシン上で動く Manus エージェント |
| NemoClaw | Nvidia | エンタープライズ向け安全な OpenClaw 相当品（GTC 2026 発表） |

一つのオープンソースプロジェクトがこれほど短期間に大きな動きを生んだことがホストを驚かせている。

### OpenAI Responses API ＋ コンピューター環境

OpenAI が Responses API の背後にコンピューター実行環境を置いた。API を叩くとコンピューター操作の結果が返ってくる仕組みで、モデルプロバイダーへの機能集約が加速している流れと一致する。Deep Research もすでに API 背後に移動しており、「機能は全てモデルプロバイダーの API 経由になっていく」という年末の予測が実現しつつある。

### 開発者向けインサイト

**RL環境ビジネス（Elliot Arledge）**  
強化学習環境を専門に構築し、Anthropic・OpenAI・Google・Meta などの大手モデルラボに売る小企業が密かに高収益を得ている。ラボ側はゼロから専門家を採用する必要がなく、高い対価を払う。特定ベンチマークスコアを改善するためのデータがどのように生まれているか、その裏側を知れる一読の価値ある投稿。

**AutoContext（Jay）**  
Karpathy の Auto Research に触発された再帰的自己改善フレームワーク。エージェントが自分自身の研究を行い、それを次の作業に活かすための文脈として学習に組み込む。当初は商用化を狙っていたが、コミュニティの反響を見てオープンソース化。「エージェントが必要な分野で自己トレーニングして専門家になれる」世界へ向けたアプローチとして注目される。

**Attention Residuals（Moonshot AI / Kimi）**  
標準の残差接続を改良し、先行レイヤーの表現を入力依存で選択的に取得できるようにした新手法。Block Attention Res でクロスレイヤーアテンションをスケール対応可能にし、1.25 倍の計算効率改善を推論遅延ほぼゼロで実現。フルレポートは GitHub で公開中。

### クイックヒット

* **Replit Agent 4** — Agent 3 から約 5 ヶ月でメジャーアップデート。Web・モバイルアプリに加えアニメーション・デザインなどクリエイティブ分野にも対応幅を拡大。大規模資金調達も実施。
* **Resend CLI** — オープンソースのメール送信（Resend）用 CLI。Google Workspace CLI と同様、エージェントへの CLI 渡しパターンが広がっている。
* **pnpm カタログ機能（未リリース）** — monorepo でワークツリー間の node\_modules 依存関係を共有し、毎回 `pnpm install` しなくて済む。Mastra チームも注目。
* **LLM Mock（CopilotKit）** — 本物の HTTP サーバー・SSE ストリーム・WebSocket で OpenAI/Claude/Gemini の呼び出しをモック。トークン消費なしでテスト可能。近日中に番組ゲストとして登場予定。
* **HydroDB** — グラフ DB アプローチでベクター DB を刷新すると宣言。「グラフ型 traversal がセマンティック検索より優れている」という主張。グラフ RAG への資金流入が始まっている。
* **Google マップ＋ Gemini** — 10 年以上ぶりの大型アップデート。Gemini モデルと世界の深い理解を組み合わせた新しいナビ体験を提供。
* **GLM5 Turbo（ZhipuAI）** — GLM5 の高速バリアント。OpenClaw などエージェント駆動環境に特化。OpenClaw ブームに乗った投入。
