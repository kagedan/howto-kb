**はじめに：現在のAIエージェントが抱える2つの「死」**  
ローカルLLMを用いた自律型AIエージェントの開発において、私たちは常に2つの残酷な壁に直面します。

1. **内なる死（ハルシネーションと迎合）**: LLMは本質的に確率的なトークン予測マシンであるため、知らないことでも「知ったかぶり」をして事実を捏造します（Sycophancy / 迎合性）。
2. **外なる死（WAFによるBot検知）**: エージェントが外部のWebから情報を取得しようとすると、Cloudflare Turnstile等のBotGuardに「振る舞い（マウス軌跡やタイピング）」の異常を検知され、即座にブロックされます。

今回、私が開発しているローカルAIエージェントIDE「Verantyx (VerAgent)」のv0.4.0アップデートにて、この2つの壁を\*\*「プロンプトではなく物理的・アーキテクチャ的なハック」**によって完全に破壊するパイプラインを構築しました。  
本記事では、特定のモデル（Qwen3.6-27BやGemma4-26Bなど）に依存せず、ローカル環境で完結する**「Visual Anchor」と「ハイブリッド生体偽装パイプライン」\*\*の全貌を解説します。

#### 1. [Visual Anchor] 視覚的ショック療法によるSycophancyの物理的破壊

LLMに「わからない時は検索しろ」とテキストで指示しても、確率の海に溺れたモデルは高頻度でそれを無視します。これを防ぐために実装したのが、推論直前における「Visual Anchor（視覚的アンカー）」の強制注入です。

**モダリティ・シフト（Modality Shift）による認知のハック**  
モデルがテキストの自由連想に入る直前、コンテキストに**動的に生成された強烈な警告画像**を強制的に注入します。具体的には、真っ黄色な背景に `[ SEARCH REQUIRED ] KNOWLEDGE BOUNDARY DETECTED` というテキストを印字し、さらにVision Encoderを意図的に混乱させるための「ノイズ線（Visual Attack）」を無数に走らせた画像です。

これにより、LLMのAttention（注意機構）に物理的な変化が起きます。

* テキストのみの「マルコフ連鎖的な迎合状態」が破壊される。
* 視覚情報（異なるモダリティ）を処理することで、強制的に「客観的なステート」へと状態遷移する。

結果として、未知の情報について問われた際、モデルは嘘をつくことをやめ、自律的に `[SEARCH_MULTI: query]` という検索ツール呼び出しを生成するようになります。これはプロンプトエンジニアリングの限界を超えた、アーキテクチャレベルでの「State（状態）の制御」です。

#### 2. [ハイブリッド生体偽装] WAFを貫通する「エントロピーのクローニング」

LLMが検索を決意したあと、PuppeteerやSeleniumでブラウザを操作すると `isTrusted: false` のフラグや直線的なマウスの動きによって一瞬でBotとして弾かれます。  
そこで、Verantyxでは\*\*「ユーザー自身の生体ノイズ（エントロピー）」を収穫し、それをOSレベルで再生する\*\*というアプローチをとりました。

**ステップ1：Human Puzzle による「生きた」エントロピーの直前キャプチャ**  
LLMが検索を発火させた瞬間、システムは即座にブラウザを起動するのではなく、IDE上に一瞬「Human Puzzle」というUIを表示してユーザーの操作を要求します。

```
// AgentLoop.swift: 検索直前にユーザーの生体エントロピーを強制的に収穫する
if gateDecision.searchType == .web {
    // UIにHuman Puzzleを表示し、ユーザーの手の動きを待機
    await MainActor.run { AppState.shared?.requiresHumanPuzzle = true }
    
    var waitingForPuzzle = await MainActor.run { AppState.shared?.requiresHumanPuzzle == true }
    while waitingForPuzzle {
        try? await Task.sleep(nanoseconds: 200_000_000)
        waitingForPuzzle = await MainActor.run { AppState.shared?.requiresHumanPuzzle == true }
    }
    
    // 収穫された新鮮なマウス軌跡（手の震えや減速カーブ）を抽出
    let cgPoints = await MainActor.run { AppState.shared?.lastEntropy }
    if let points = cgPoints {
        let mapped = points.map { [Double($0.x), Double($0.y)] }
        // OS注入用に最大100サンプルの座標配列に圧縮してRustへ渡す
        entropyPoints = mapped.count > 100 ? 
            stride(from: 0, to: mapped.count, by: mapped.count/100).map { mapped[$0] } : mapped
    }
}
```

ユーザーがマウスを動かした瞬間の「目標へ向かう軌跡、手の震え（ジッター）、減速のカーブ」をリアルタイムでキャプチャし、このデータをRust側へ引き渡します。

**ステップ2：OSとDOMの「ハイブリッド注入」（stealth\_bridge.rs）**  
新鮮なエントロピーを確保すると、Rust製のカスタムブラウザ（vx-browser）が起動し、防御網を突破します。

```
// stealth_bridge.rs: タイピング時のOS・DOMハイブリッド注入パイプライン
for c in chars {
    let mut buf = [0; 4];
    let s = c.encode_utf8(&mut buf);
    
    // 【OSレベル】コアフレームワーク(CGEvent)を叩き、物理キーボードのダミーログを生成
    if let Ok(event) = CGEvent::new_keyboard_event(source.clone(), 0, true) {
        event.set_string(s);
        event.post(core_graphics::event::CGEventTapLocation::Session); // WAFには物理操作に見える
    }
    
    // キャプチャしたユーザーの打鍵間隔（ホールド時間）を再現
    thread::sleep(Duration::from_secs_f64(hold_duration));
    
    // 【DOMレベル】確実な入力と画面描画のためにJSイベントを直接ディスパッチ
    let js_val = serde_json::to_string(&s).unwrap();
    let js = format!(
        "var el = document.activeElement; if (el) {{ el.value += {}; el.dispatchEvent(new Event('input', {{ bubbles: true }})); }}",
        js_val
    );
    proxy_clone.send_event(BridgeCommand { cmd: "eval_js".into(), text: Some(js), ... });
}
```

1. **OSレベルの物理注入**: macOSの `CGEvent` を直接叩き、先ほど収穫した軌跡や揺らぎをダミーイベントとしてOSに送信します。WAFから見れば「完全に物理的なUSBデバイスから送信された人間の操作」にしか見えません。
2. **DOMレベルでの確実な入力**: 裏で別作業をしている場合など、OSレベルのキー送信だけではフォーカス外れで「空振り」するリスクがあります。そのため、同時にJavaScriptで `document.activeElement` に対して文字を流し込みます。

「OSには人間の痕跡を残しつつ、実際のテキスト入力と画面描画はDOMで確実に行う」というこのハイブリッド構造により、最新の振る舞い検知（Behavioral Biometrics）を完全にすり抜けます。

#### 3. 並列スクレイピングから「JCross Cortex（長期記憶）」への統合

WAFを突破した後の動作も、ローカル完結型として最適化されています。

1. **並列読み込みの実行**: 検索結果一覧と上位2つのリンクを裏側で並列して同時にスクレイピングします。
2. **モデル非依存の推論**: 収集したHTMLをマークダウンに変換し、コンテキストとしてLLMに渡します。特定のAPIに依存せず、gemma4:26b や qwen3.6:27b などでシームレスに動作します。
3. **空間メモリへの定着**: 検索して得た事実と要約は、Verantyxの長期記憶エンジン（JCross Cortex）の「nearゾーン」に自動保存され、次回以降のセッションでは検索を介さずに高速な引き出しが可能になります。

**おわりに：API依存からの脱却と、真のローカル自律エージェント**  
現在主流となっている「クラウドの巨大モデルAPIを叩き、既存ツールでWebをスクレイピングする」アーキテクチャは、ベンダーの規約やAPIコスト、WAFの進化によって常に首根っこを掴まれています。

* LLMの脳内を **Visual Anchor** で制御し、論理の暴走を防ぐ。
* 外部との関わりを **OSレベルの生体エントロピー注入** で偽装し、ネットワークの検閲をすり抜ける。

M1 Maxとローカルモデルだけで構成されたこのシステムは、誰にも検閲されず、コストもかからない「究極のローカル自律エージェント」の一つの完成形だと考えています。
