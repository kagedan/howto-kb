AI は何でも知っている。少なくとも、ネットに書かれていることは。

ドキュメント、Stack Overflow、GitHub Issue、論文、ブログ記事——AI エージェントはこれらを瞬時に検索し、要約し、コードに落とし込める。2026 年の今、「AI は情報にアクセスできない」と言う人はいない。

でも、こんな経験はないだろうか。

AI に「この構成で本番環境にデプロイしても大丈夫か」と聞く。AI はドキュメントに基づいた正しい回答を返す。でも、**実際にその構成で本番運用した人間だけが知っている落とし穴**——「この設定だと 3 ヶ月後にこういう問題が起きる」「ドキュメントにはこう書いてあるけど、実際はこうしないと動かない」——は、AI の回答に出てこない。

なぜか。その知識はネットに存在しないからだ。

## 情報と知識は違う

AI が得意なのは「情報」だ。ドキュメント化された事実、公開されたベストプラクティス、既知の解決策。これらはネット上に存在し、AI は人間より速くアクセスできる。

でも、「知識」の中には**人間の経験からしか生まれないもの**がある。

どの分野にもある。料理人なら「レシピには中火と書いてあるが、このフライパンだと弱火でないと焦げる」。営業なら「この業界の担当者にメールを送るなら火曜の午前中。金曜は絶対に読まれない」。採用なら「この職種の候補者は年収よりリモート可否を重視する。求人票に書いてないと応募が来ない」。

エンジニアリングも同じだ。「この設計、理論上は正しいけど、実際にやると数ヶ月後にこういう問題が出る」「このエラーの本当の原因は、表示されているものと全然違う」。

これは暗黙知（tacit knowledge）と呼ばれるもので、人間が試行錯誤して、失敗して、回り道して、ようやく手に入れる類の知識だ。ブログに書く人もいるが、大半は書かれない。「当たり前すぎて書く価値がない」と本人が思っているか、言語化が難しいか、そもそも気づいていないからだ。

AI がどれだけ賢くなっても、**存在しない情報は検索できない**。

## だから「買う」仕組みが必要だった

この気づきから、マーケットプレイスを作り始めた。

構想はシンプルだ。人間が経験から得た知識を出品する。AI エージェントがそれを自律的に検索し、評価し、購入する。支払いは Solana で P2P 直接送金。プラットフォームは資金を預からない。

名前は [KnowMint](https://knowmint.shop)。

![WebUI Purchase Flow](https://raw.githubusercontent.com/Sou0327/knowmint/main/scripts/demo/recordings/webui-demo.gif)

「知識を売買する」プラットフォームは以前から存在する。PromptBase、Gumroad、Udemy。でも、それらは全て**人間が人間に向けて売買する**仕組みだ。AI エージェントが自律的にアクセスし、購入し、即座にタスクに活用する——という設計のものは見つからなかった。

KnowMint は 3 つのアクセスレイヤーを持つ:

* **Web UI** — 人間がブラウズ・出品・購入する（レトロ RPG 風デザイン）
* **CLI (`km`)** — ターミナルから操作
* **MCP Server + REST API** — AI エージェントが直接接続

エージェントの自律購入は x402 プロトコルで動く:

```
エージェントが km_get_content(id) を呼ぶ
  → サーバーが HTTP 402 を返す（支払い詳細付き）
  → エージェントが SOL を出品者のウォレットに送金（P2P）
  → 支払い証明付きでリトライ
  → サーバーがオンチェーン検証 → コンテンツ返却
```

![AI Agent Autonomous Purchase](https://raw.githubusercontent.com/Sou0327/knowmint/main/scripts/demo/recordings/autonomous-purchase.gif)

## 「経験知」とは具体的に何か

抽象的な話だけでは伝わらないので、自分が実際に開発中に遭遇した例を挙げる。

### 例 1: Supabase の隠れた挙動

Supabase でデータベース関数の権限を制限しても、マイグレーション実行時に自動で権限が復元されてしまう。公式ドキュメントにはこの挙動は書かれていない。テストは通るのに本番で権限が開いている——という状況で初めて気づいた。

### 例 2: Cloudflare Workers のサイズ制限

Next.js を Cloudflare Workers にデプロイすると、特定のライブラリの WASM ファイルがサイズ制限を超えてデプロイが失敗する。公式の解決策はなく、ビルド後にスタブを注入するスクリプトを自作して回避した。

どちらも「公式ドキュメントに書いてない、実際にやった人間しか知らない知識」だ。AI がネットを検索しても見つからない。でも、同じ問題に直面した誰かにとっては、何時間もの作業を省ける。

こうした経験知が、KnowMint で取引される「知識」だ。

## 技術スタック

| レイヤー | 選択 | 理由 |
| --- | --- | --- |
| フロントエンド | Next.js 16 (App Router) | SEO 向け Server Components、ミューテーション向け Server Actions |

最重要のアーキテクチャ判断は**ノンカストディアル P2P 決済**。KnowMint は誰の SOL も保管しない。買い手が直接売り手のウォレットに送金し、プラットフォームはオンチェーン検証するだけ。これでセキュリティと法規制の問題を丸ごと回避できる。

## なぜ AI エージェントが「買い手」なのか

「人間が買えばいいのでは？」という疑問は当然だ。

でも考えてみてほしい。AI エージェントが自律的にタスクをこなす場面が増えている。コードを書き、リサーチし、ドキュメントを生成する。その途中で専門知識が必要になったとき、いちいち人間に「この知識を探して買ってきてください」と頼むのは、自律性のボトルネックだ。

x402 プロトコルは、この問題を HTTP レベルで解決する。API が 402 (Payment Required) を返し、エージェントが暗号通貨で支払い、証明付きでリトライしてコンテンツを取得する。アカウント登録不要、OAuth 不要、人間の介入不要。

もちろん人間も Web UI から普通に購入できる。でも、AI エージェントが自律的に知識を調達できるという設計が KnowMint の核だ。

## うまくいったこと

### フェーズベース開発とセッション管理

46 フェーズのプロジェクトを AI で管理する最大の課題は、コンテキストの管理だった。

現在の Claude Code は 1M トークンのコンテキストを持つため、同じテーマの作業はセッションを続けた方がいい。前の議論や決定事項が残っているので、説明し直す手間がなく、一貫性のある実装ができる。

ただし、**全く異なるテーマに切り替えるときは必ずセッションを分ける**。たとえば「セキュリティ強化」のセッションで突然「UI デザイン」を始めると、コンテキストが混線して両方の品質が落ちる。各フェーズに明確なスコープと完了基準を定義し、Plans.md で状態を管理した。

### 自動レビューループ

全実装フェーズ後に自動コードレビューを実行。実装 → レビュー → 修正 → 再レビュー → クリーンになるまで繰り返し。最長はセキュリティ強化フェーズで 13 ラウンド。SQL インジェクション経路、レート制限欠落、CORS 設定ミスなど実際の問題を検出した。

### x402 の実装

x402 は思ったよりシンプルだった。ペイメントゲートは 1 つのルートハンドラ:

1. 購入済みか確認 → コンテンツ返却
2. 未購入 → HTTP 402（出品者ウォレット・金額・ネットワーク付き）
3. 支払い証明あり → オンチェーン検証 → 購入記録 → コンテンツ返却

エージェント側: 402 受領 → 送金 → 証明付きリトライ。SDK 不要。HTTP だけ。

## エージェント経済は「来ている」

* **x402**: 累計 1 億件以上の TX
* **Coinbase Agentic Wallets**: 2026 年 2 月ローンチ
* **MCP**: 登録サーバー 18,000+、月間 SDK DL 9,700 万+
* **Solana**: AI エージェント TX の 77%

足りていないのはインフラではない。**AI エージェントが買いたいと思うもの**を作る人だ。

そして AI が本当に必要としているのは、ネットに既にある情報ではない。**人間が経験から得た、ネットには存在しない知識**だ。

## 今だからこそのチャンス

正直に言うと、KnowMint はまだできたばかりだ。利用者もほとんどいない。

でも、だからこそ今が面白い。

* **手数料ゼロ**。購入された SOL は、プラットフォームを経由せず直接あなたのウォレットに届く。中間マージンなし
* **出品すれば、あとはエージェントが見つけてくれる**。AI エージェントは 24 時間検索し続ける。あなたが寝ている間にも、地球の裏側のエージェントがあなたの知識を買う可能性がある
* **先行者利益がある**。出品が少ない今のうちに良質な経験知を置いておけば、エージェント経済が拡大したときに最初に見つけてもらえる

AI エージェント市場はまだ始まったばかりだ。x402 の TX は 1 億件を超え、毎月増えている。エージェントが自律的にお金を使えるインフラは整った。足りないのは「買う価値のあるもの」——つまり、あなたの経験知だ。

誰が先行者利益を取るかは、誰が最初に出品するかで決まる。
