# LLM SDK の全体像

まず、SDKが何をしているのかを整理します。

```
あなたのコード
    ↓
  SDK （通信・認証・エラー処理を肩代わり）
    ↓
LLMプロバイダーのAPI（Anthropic / OpenAI など）
    ↓
  LLMモデル（Claude / GPT など）
```

SDKとは「Software Development Kit」の略で、**LLMのAPIをコードから簡単に呼び出すためのライブラリ**です。生のHTTPリクエストを自分で書く代わりに、認証・エラー処理・ストリーミングなどを肩代わりしてくれます。

## SDK でできること（5つ）

|  | 機能 | 用途 |
| --- | --- | --- |
| ① | テキスト生成 | メッセージを送って返答を受け取る（基本） |
| ② | マルチターン会話 | 会話履歴を渡して文脈を維持する |
| ③ | ストリーミング | 返答をリアルタイムで少しずつ受け取る |
| ④ | ツール呼び出し（Function Calling） | LLMに「関数を呼ぶ判断」をさせる。エージェントの核心 |
| ⑤ | 埋め込み（Embedding） | テキストをベクトル数値に変換する。RAGの核心 |

# 事前準備：APIキーの取得と設定

## APIキーとは

**「あなたが誰か」を証明するパスワード**です。

LLMプロバイダーは有料サービスなので、誰がAPIを使ったかを識別して課金するために使います。

```
あなたのコード → APIキーを添えて送信 → 「この人は登録済みユーザーだ」→ OK
```

## APIキーの取得

アカウント登録 → APIキー発行 → コピー、の流れです。

## フォルダ構成

```
my_project/
├── .env              ← APIキーをここに書く
├── .gitignore        ← .envを除外する設定
├── main.py           ← メインコード
└── requirements.txt  ← 必要なライブラリ一覧
```

## 各ファイルの中身

**.env**

```
ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxxxxxxx
```

**.gitignore**

**requirements.txt**

## セットアップ手順

```
# ① ライブラリをインストール
pip install anthropic python-dotenv

# ② .envファイルを作成してAPIキーを書く
echo "ANTHROPIC_API_KEY=sk-ant-xxxxx" > .env
```

## APIキーの扱いで絶対にやってはいけないこと

```
# ❌ 絶対にやらない！GitHubに流出する
client = anthropic.Anthropic(api_key="sk-ant-xxxxx")
```

コードにAPIキーを直書きすると、GitHubにアップした瞬間に**世界中に公開**されます。悪用されると自分に課金されます。実際に事故が多発しています。

```
# ✅ 正しい方法：環境変数から自動で読み込む
client = anthropic.Anthropic()
```

`.env` ファイルに書いておけば、SDKが `ANTHROPIC_API_KEY` という名前で自動的に読み込んでくれます。

# ① テキスト生成（基本）

## そもそも何をしているのか

```
あなた → 「質問」を送る → Claude が答えを返す
```

これだけです。メールを送って返信をもらうイメージ。

## 最小のコード

```
from dotenv import load_dotenv
import anthropic

# .envを読み込む（忘れずに）
load_dotenv()

# クライアントを作る（APIキーは環境変数から自動で読む）
client = anthropic.Anthropic()

# メッセージを送る
response = client.messages.create(
    model="claude-sonnet-4-6",          # 使うモデル
    max_tokens=1024,                    # 返答の最大長さ
    messages=[
        {"role": "user", "content": "Pythonとは何ですか？"}
    ]
)

# 返答を取り出す
print(response.content[0].text)
```

![](https://static.zenn.studio/user-upload/d246b92af999-20260326.png)

## 各パラメータの意味

| パラメータ | 意味 |
| --- | --- |
| `model` | 使うAIモデルの種類 |
| `max_tokens` | 返答の最大トークン数（長さの上限） |
| `messages` | 会話の内容。`role`と`content`をセットで渡す |

## `role` って何？

LLMの会話には**3種類の役割**があります。

| role | 誰 | 用途 |
| --- | --- | --- |
| `system` | AIへの指示 | 「あなたはプロのエンジニアです」など |
| `user` | あなた | 質問・依頼 |
| `assistant` | AIの返答 | 会話履歴として使う |

## レスポンスの構造

```
response.content[0].text      # 返答テキスト
response.usage.input_tokens   # 送ったトークン数
response.usage.output_tokens  # 返ってきたトークン数
```

`response` の中身はこうなっています：

```
{
  "content": [{"type": "text", "text": "Pythonとは..."}],
  "usage": {"input_tokens": 20, "output_tokens": 150}
}
```

# まとめ

* SDKは**HTTPリクエストを隠蔽**してくれるラッパー
* APIキーは `.env` ファイルで管理、コードへの直書きは厳禁
* 環境変数名は `ANTHROPIC_API_KEY` で固定
* `messages` は**配列**で渡す（次回ここに履歴を積む）
* `max_tokens` を設定しないとコストが爆発するので必ず設定
