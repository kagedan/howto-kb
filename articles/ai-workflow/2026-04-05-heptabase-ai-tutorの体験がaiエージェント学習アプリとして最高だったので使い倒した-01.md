## はじめに

LLMによってコーディング速度がブーストされた現代において、エンジニアとしてはインプット能力の高さや正確さがより求められるようになっています。

インプット補助のやり方として、LLMにプロンプトを与えて学習ガイドのチューターになってもらう方法もあるのですが、そこで立ちはだかる壁として、  
「AIに質問されても、自分にとって何がわかってないのかが言語化できていない・・・」  
「出力されたテキストを読んで理解した気になったが、いざ業務の場面ではどうすべきかがわからない」  
「全体像が分かっていない中、AIの質問に対してひたすら思考のタイピングが求められている（そして飽きる）」

という場面が自分の中で数多くあり、またそれを解決してくれる手法はまだ確立されていない、と考えています。

## AI Tutor

そして2026年3月31日、[ノートアプリHeptabaseにAI Tutor機能が追加されました](https://wiki.heptabase.com/changelog#:~:text=AI%20Tutor%20mode.-,v1.88.0,-March%2031%2C%202026)。  
Heptabaseとは、一言でいうと「ホワイトボードでノート間のつながりを表現可能なNotion」という感じのアプリです。精力的なアップデートが続いている点や操作性の良さには魅力を感じていたものの、効果的な使い方が見つからず、ホワイトボードで考えを整理する際に細々と使うだけのツールに収まっており、正直このまま使わなくなるかもしれない、とも思っていました。

そんな中、このAI Tutor機能が追加されていたので、少し触ってみました。  
このAI Tutor機能、思った以上に完成度が高く、睡眠時間を削るほど没頭し、**この3日間で$71.99/月のPremium+プランに加入して約100ドル相当のクレジットを消化してしまいました**。

より良いインプット方法を求めている人にとっては、「これこそが求めていたものかもしれん！！！」となり得るプロダクトなので、是非紹介したいと思いこの記事を書いています。

## AI Tutorはユーザ体験として何が良いか？

個人的に以下の三点が非常にマッチしており、学習アプリとして出来がとても良いです。冒頭で挙げた辛みをすべて払拭してくれています。

**「AIに質問されても、自分にとって何がわかってないのかが言語化できていない・・・」**  
→ システム側の問いかけやソース解析機能によって質の高いロードマップや理解度チェックを実行できる設計がされており、確実な全体像をとらえた上でクオリティが担保されたインプットができる

**「出力されたテキストを読んで理解した気になったが、いざ業務の場面ではどうすべきかがわからない」**  
→ 具体的なユースケースを踏まえた問題に答え、自分の理解を深めるパートがある

**「全体像が分かっていない中、AIの質問に対してひたすら思考のタイピングが求められている（そして飽きる）」**  
→ できるだけユーザのクリック回数が少なくなるようなUXになっている

さらに、ソース読み込みの選択肢の多さも素晴らしいです。  
英語のドキュメントも日本語教材に変換可能な上に、PDFやYoutubeなどもソースにすることができ、トークン使用料が抑えられるように分割読み込み機能が完備されています。

## AI Tutor機能を使ってみる

### 機能概要

フローを整理すると以下のような流れでループします。  
最初に達成目標の「ゴール」と、ゴール到達に向けたレッスン手順をまとめた「シラバス」を作成します。  
その後LLMが「レッスン」としてドキュメント生成・理解度チェック・まとめノート作成を実施する、というフローです。

```
[ゴール設定] ──→ [シラバス作成] ──→ [シラバス確定]
      │                                    │
      │                                    ▼
      │                           ┌──────────────────┐
      │                           │  セッションループ  │
      │                           ├──────────────────┤
      │                           │ 1. スコープ確認    │
      │                           │ 2. レッスンフロー  │
      │                           │    ノート作成      │
      │                           │ 3. レッスン実施    │
      │                           │ 4. レビュー        │
      │                           │ 5. まとめノート    │
      │                           │ 6. フィードバック  │
      │                           │ 7. 次回予告・終了  │
      │                           └───────┬──────────┘
      │                                   │
      │         ┌─────────────────────────┘
      │         ▼
      │    [フィードバック反映]
      │    ├─ ゴールノート更新
      │    └─ シラバス更新
      │         │
      └─────────┘（必要に応じてゴール再設定）
```

```
# ゴール概要

TypeScript・モジュールシステム・モノレポ構成・コンテナデプロイなど、モダンフロントエンドの基盤技術を体系的に習得する。単なる知識習得にとどまらず、実務での設計判断・バージョンアップ作業・チーム標準の策定に自信を持って臨めることを目指す。

## 学習者プロフィール

- フルスタックエンジニアとして実務経験あり。フロントエンド比率は低めだが、実践的に深めたい
- 現在の技術スタック: Next.js（App Router / Pages Router）、Vite、pnpm workspaces / Turborepo によるモノレポ、TypeScript、Docker / コンテナデプロイ
- tsconfig の細かい設定はこれまで深く追っていなかった
- コンテナデプロイにおけるフロントエンド特有のはまりポイントに自信がない
- 学習スタイル: 概念と実例をセットで理解したい
    

## コース構成

- **フロントエンド基盤の設計原則**（現在進行中）
— ESM/CJS・モジュール解決・tsconfig と各ツールの相互関係・pnpm workspaces / Turborepo / Nx によるモノレポ設計
- 今後の方向性（第1コース修了後に調整）:
    - Next.js / Vite バージョンアップ戦略
    - コンテナデプロイとフロントエンドテスト基盤（Medium/Large テスト構築）
```

### 理解を切り出す

ここからレッスンノートをリファレンスに、自分なりに概念の関連性や順序関係を再構成していきます。  
再構成の目的としては、`自分の言葉で説明するとどこに重点を置くべきか、何から説明するか？`、`まだ理解できていないことやさらに追及すべき箇所はどこか？`といった自己理解の全体像を頭の中で表現できるようにするためです。

まずはホワイトボードを開き、自分用のサブホワイトボードとして「OAuth」を作ります。

目次案:

Chapter 1：OAuthについての整理
