## はじめに

### 経緯

* 現在とあるベンダー資格の取得に向けて勉強中。
* そこまで知名度が高くないのでUdemy等に模擬試験がない。
* 仕方がないのでGeminiに模擬試験を作成してもらって、それを解いていた。
  + 基本的にベンダーが公開している公式ドキュメントの内容をもとに問題を作ってくれてるっぽい。とはいえ稀にハルシネーションが発生することもあるのでドキュメントもちゃんとチェックしよう。
  + 表形式で問題を出力させて、タイマーアプリで時間をはかりつつ、スプレッドシートに回答をメモして、解き終わったら回答をコピペしてGeminiに投げる…。みたいなことをしていたが、若干非効率的なのでは？と思い始める。
* 問題文、選択肢、回答、解説、参考URLをCSV形式でGeminiに出力してもらって、CSVをアップロードするとデータベースに記録され、そのデータをもとにして4択クイズを出題するようなWebアプリをAntigravityで作成すれば良いのでは？
* Webアプリを作成するついでに今週の記事のネタにしてしまおう！一石二鳥！

## 大まかな実装内容

* `Supabase Auth (Email/Password)`と`Middleware`による堅牢なアクセス制御。
* 指定フォーマットのCSVをアップロードするだけで、即座にクイズを生成。小文字やスペースを許容する柔軟なバリデーション付き。
* 4択形式のクイズ、シャッフル出題、回答後の詳細解説表示、参考URL連携。
* 学習の成果を可視化するダッシュボードと、間違えた問題だけを復習できる「ニガテ克服モード」。
* `shadcn/ui`をベースにした、モダンで使い心地の良いデザイン。
* クイズ開始前に「問題数」と「制限時間（分）」をカード内で指定できる。
* クイズ中にリアルタイムでカウントダウンを表示し、タイムアップ時にはその時点の成績で自動終了する仕組みを導入。

## Vercelにデプロイする

アプリを利用する度に毎回`npm run dev`するのも面倒なので、Vercelにデプロイすることにした。

### Vercelとは

<https://vercel.com/>

* Vercel Inc. が提供する高速で安全なクラウドホスティングプラットフォームである。
* Vercel Inc. はNext.jsの開発元でもある。
* GitHubやGitLabと連携し、プッシュ毎に自動で本番環境とプレビューURLを作成できる。

### Githubにpush

1. Gitの`user.name`および`user.email`の設定を行う。

ターミナル

```
git config --global user.name your-name
git config --global user.email your-email@example.com
```

2. `Commit`をクリックする。その後、Githubへの認証を済ませてから`Push`をクリックする。  
   ![](https://static.zenn.studio/user-upload/b0e2340704be-20260328.png)

### デプロイ手順

1. GitHubアカウントでVercelのアカウントを作成する。
2. `Add New…`から`Project`を選択する。  
   ![](https://static.zenn.studio/user-upload/3a53c847fa81-20260328.png)
3. `Import Git Repository`からGithubリポジトリのインポートを行う。  
   ![](https://static.zenn.studio/user-upload/b867850587b4-20260328.png)
4. `Import .env`をクリックし、予め作成しておいた`.env.local`ファイルを選択してから`Deploy`をクリックする。
5. 下記のような画面に遷移したらデプロイ成功である。  
   ![](https://static.zenn.studio/user-upload/94c433e7f5a4-20260328.png)

## ハマったポイント

* Prismaを利用しようとしたものの、`DATABASE_URL`に正しい値を入れているはずなのにSupabaseへの認証が通らない。
  + AIに聞いても原因不明だったので、今回は諦めてPrismaを利用しない方向に切り替えた。
* Geminiが`shadcn/ui`の仕様を完全には理解できていなさそう。これに関しては何度か修正点を指摘したり、`className`を自分で調整したりすることでなんとかなった。
* 何故かページ全体の再描画が走った際に問題リストが毎回新しくシャッフルされていた。これに関しても指摘を入れて、シャッフル処理を「セッション開始時の1回のみ」に固定させた。

## Quiz Appの要検討事項

* JSONでのインポートに対応してもいいかも？
* アップロード可能なCSVの制約がキツすぎる気がする。
  + しかし、下手に制約を緩くすると普通にバグりそう。今のところ解決策は思い浮かばない。
* スパム対策やリソース保護のためデータ登録を1日200件までとしているが、もう少し上限を上げるべきかは悩みどころ。
* アプリからGemini APIを叩いてデータベースに保存させることも考えたが、APIの乱用対策も考えると1ユーザーあたりの1日に作成可能な問題数をかなり絞らないといけないので、UX的にイマイチかも。あと、問題の重複防止が難しそう。

## おわりに

### 今回得られた知見

* ある程度複雑な処理が必要なWebアプリだと、AIエージェントに1回指示しただけでは想定通りのものは出力されない。なので、何度か修正指示を入れる必要がある。
* Vercelへのデプロイは思ったより簡単だった。リポジトリにPushしたら自動的に反映してくれるのも非常にありがたい。
* AIとの協働には前提知識が不可欠だと実感した。完全に任せるのではなく、違和感に気づいて軌道修正できる程度の理解はやはり必要。

### まとめ

* 自分の課題を解決できるWebアプリを、土日でサクッと作れる。
* 特に土台となる実装やアイデア検証のスピードは圧倒的に向上しており、開発体験そのものを大きく変えるポテンシャルを感じた。
* 一方で、UIの詳細部分や状態管理のような細かい挙動は、人間側の意図をしっかり伝えないとズレが生じやすいので要注意。
* 今後は、より複雑な要件や実運用を見据えた設計にもチャレンジしつつ、AIとの付き合い方も含めてブラッシュアップしていきたい。
