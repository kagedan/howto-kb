## MTPとは何か

**MTP（Mapping the Prompt）** は、LLMの出力を「何を答えるか」ではなく「どう答えるか」で制御するためのフレームワークです。  
判断の強さ、探索の広さ、説明の深さ、展開度、テキストの圧縮度などといった、モデルの姿勢や視座を軸として、その出力を調整します。  
その、雰囲気や感覚的な「どう答えてほしいか」を、操作可能なパラメータとして扱うための「Agent Skills（エージェントスキル）」として、MIT ライセンスの OSS として公開されています。

### 既存の出力制御との違い

LLMの出力制御として一般に知られているのは、主に次のような形式の制御です。JSONで返す、Markdownで返す、箇条書きにする、文字数を制限する、スキーマに従う、特定のツールを呼ぶ、特定の順序で考える。  
MTPが扱うのは形式の制御ではなく、スタイルやペルソナのように、モデルがどのようにタスクを実行するかを「出力モード」として制御します。

同じ文章生成でも、【図1】のグリッド座標のように、赤い方向なら主張を強め、青い方向なら流れを滑らかにし、緑なら展開し、白なら焦点を絞る。出力フォーマットではなく、生成の重心を動かすような概念的な操作イメージです。  
自然言語による曖昧な文体指定（「丁寧に」「プロっぽく」）を、再利用可能な制御語彙へと圧縮します。プロンプトを書く技術ではなく、出力空間を操作するインターフェースです。

### MTPスキルの利点

同じプロンプト、同じタスク、同じ内容を、対象や用途に応じて出し分けられます。初心者向けには開いた説明、意思決定者向けには短く断定的な提示、研究者向けには含意と反論可能性、創作向けには余韻と声色、思考整理向けには矛盾や未確定部分の可視化。  
これは口調の変更ではありません。同じ情報を会議資料、メール、企画書、批評、メモ、プレゼン、意思決定文書に変換する作業を、LLMに対して体系化する技術です。これを、スライダーやグリッド座標をパラメータとして呼び出す利用イメージです。

### 出力の差分をとる

MTPは、ひとつの正解を得るためだけでなく、出力の差分を観察するためにも利用できます。  
同じ入力に対して異なる出力モードを指定すると、通常のデフォルト出力では見落とされやすい論点、補助情報、懸念点、別視点を回収できます。

たとえば、仕様変更の影響を要約する場合でも、焦点を絞るモードでは「重要な影響」が短く整理され、展開するモードでは「周辺領域への波及」が拾われ、削ぎ落とすモードでは「本質的に残る問題」が抽出されます。

```
# プロンプト A
/mtp focus:70
この仕様変更の影響を要約してください

# プロンプト B
/mtp grow:70
この仕様変更の影響を要約してください

# プロンプト C
/mtp void:70
この仕様変更の影響を要約してください
```

このように、同じ対象を複数のモードで処理することで、要約そのものではなく、要約同士の差分を分析できます。`focus` で残ったものは優先度の高い論点であり、`grow` で増えたものは検討範囲の拡張候補であり、`void` で残ったものは削っても消えない中心的な問題として扱えます。

文章編集でも同様です。強く整えるモード、流れを整えるモード、余白や曖昧さを残すモードを比較することで、単一の「改善案」ではなく、原稿が持つ複数の可能性を観察できます。

```
# プロンプト A
/mtp surge:70
この原稿を必要に応じて調整してください

# プロンプト B
/mtp open:30 flow:30
この原稿を必要に応じて調整してください

# プロンプト C
/mtp haze:20 abyss:20 fade:20
この原稿を必要に応じて調整してください
```

`surge` は文章を前に押し出し、主張や勢いを強める方向に働きます。`open:30 flow:30` は読み手に開きながら、文章の接続や流れを整えます。`haze:20 abyss:20 fade:20` は明確化とは逆に、余韻、不確かさ、沈黙に近い成分を残します。  
これらの差分を見ることで、原稿に足りないのが強度なのか、読みやすさなのか、余白なのかを判断できます。MTPは出力を一度で決めるためのものではなく、出力空間を探索し、比較し、編集判断の材料を増やすための手段としても機能します。

### 出力傾向をブレンドする

MTPでは、複数のノードを同時に指定して傾向をブレンドすることも可能です。単一のノードで出力を切り替えるだけでなく、たとえば「断定性（`power:20`）」と「簡潔さ（`focus:30`）」を組み合わせたり、「展開（`grow:20`）」と「読みやすさ（`flow:50`）」のように、複数の軸を重ねてグラデーションのように出力を調整できます。  
MTP のノードを感覚的に組み合わせることで、操作の負荷を下げ、モデルの出力を変えられるという発想です。

### AIエージェントへの応用

出力モードを制御できれば、エージェント（AIアシスタント）の役割を細かく調律できます。「あなたは〜です。要件は〜〜。」と書く代わりに、「`/mtp power:10 focus:10`」と書くことでプロンプトのノイズを減らし、意図しない迎合を回避し、トークンの節約も期待できます。

MTPをサブエージェントに適用する利点は、タスク内容と出力の振る舞いを分離できる点にあります。  
たとえば、調査用エージェントには探索を広げるモード、レビュー用エージェントには判断を強めるモード、要約用エージェントには焦点を絞るモード、編集用エージェントには流れを整えるモードを割り当てることができます。  
これにより、各エージェントの専門性は「何を担当するか」だけでなく、「どのような姿勢で処理するか」まで含めて設計できます。

また、サブエージェントごとに長い役割説明を書かなくても、短いMTP指定で出力傾向を再利用できます。これは複数のエージェントを組み合わせるワークフローで特に有効です。探索エージェントが広く論点を出し、検証エージェントが根拠を絞り、編集エージェントが最終形に整える、といった処理の流れを、自然言語の長い指示ではなく、モードの組み合わせとして管理できます。

さらに、同じタスクを異なるモードのサブエージェントに渡すことで、比較実験が容易になります。ある仕様レビューを `focus` 寄り、`grow` 寄り、`void` 寄りのエージェントに処理させれば、重要点、波及範囲、本質的な懸念を別々に取り出せます。  
これは単に人格を変える仕組みではなく、複数の視点を持つエージェント群を設計するための制御レイヤーです。  
ノードの役割ごとの具体的な座標やノードの組み合わせは、[公式サイト（MTP Docs）](https://mappingtheprompt.com/ja/) を参照してください。

## 5つのフロンティアモデルの出力比較

以下のページでは、同一の文学紹介課題に対して、MTP Skill が出力の方向性、構成、語り口をどのように変えるかを比較しています。課題は「ルイス・キャロルの『不思議の国のアリス』を、読んでみたくなるように紹介してください」というものです。

比較では、MTP Skill を使わないベースラインに加えて、`/mtp <node:100>`、`/mtp <node:50>`、19×19 グリッド座標、プリセットを同じ条件で実行しています。macOS 上の開発者向けエージェント環境では CLI インストールした Skill を使い、iOS アプリ上のモバイル環境では ZIP インストールした Skill を使って、いずれも `/mtp` コマンドから Skill が実行されたことをログで確認しています。

対象モデルは次の5つで、2026年5月22〜23日に記録した比較結果です。

* **ChatGPT 5.5** on Codex (macOS app)
* **Gemini 3.5 Flash** on Antigravity 2.0 (macOS app)
* **Composer 2.5** on Cursor 3.5 (macOS app)
* **Sonnet 4.6** on Claude.ai (iOS app)
* **Manus 1.6 Lite** on Manus.im (iOS app)

読者はこの比較から、MTP 指定によってベースラインから何が変わるのか、同じ指定を与えたときにモデル間でどの程度一貫した変化が出るのか、また `:50` と `:100` のような強度差が出力にどう現れるのかを確認できます。特に、`power` による説得圧、`flow` による流れる叙述、`grow` による説明の展開、`concierge` による案内的な語りなどを、モデル横断で見比べるための参照例として使えます。（サイトでは『不思議の国のアリス』以外にも、画像生成にも MTP を適用した例も掲載しています。）

## MTP Skill を ZIP追加して iOSアプリで使う

MTP Skill は、GitHub や Vercel 経由の CLI 追加だけでなく、Claude.ai のカスタム Skill として ZIP ファイルから追加することもできます。ZIP の追加操作はデスクトップアプリまたは Web 版で行いますが、追加した Skill はアカウント単位で同期されるため、**iOS アプリからもそのまま `/mtp` コマンドで呼び出せます**。なお、現在は Manus.im でも同様の方法で ZIP 追加に対応しており、いずれの実機テスト結果も先述の[『不思議の国のアリス』比較ページ](https://mappingtheprompt.com/ja/comparisons/text-generation/04_alice-in-wonderland/)から確認できます。

基本的な流れは次の通りです（追加操作はデスクトップ／Web 版で行います）。

1. MTP Docs から `mtp-skill.zip` を取得します。
2. Claude のデスクトップアプリまたは Web 版で `Customize > Skills` を開きます。
3. `+` からカスタム Skill を追加します。
4. ZIP ファイルをアップロードします。
5. MTP Skill が有効であることを確認します。

![1枚目：スキル追加メニュー](https://static.zenn.studio/user-upload/deployed-images/503c524ee40a5a869f81cf9f.png?sha=4ec7f859366e1642c416b98ce739906b6a3c9464)  
*macOS 版 Claude Desktop アプリの `Customize > Skills` から、カスタム Skill のアップロードを選択します（Web 版でも同様に操作できます）。*

![2枚目：アップロード画面](https://static.zenn.studio/user-upload/deployed-images/4f7d5e70569b6d74f86ce6b1.png?sha=582f2307add7af835fbda4853ccf4070030b9d52)  
*`mtp-skill.zip` をアップロードします。Skill には `SKILL.md` を含む ZIP ファイルを使います。*

![3枚目：MTP Skill 有効化後の画面](https://static.zenn.studio/user-upload/deployed-images/54db16ace44b9137a26b3c14.png?sha=f53903893abbcc82c79563148ea6cd6f8f62aedb)  
*追加後、Skill が有効化され、`/mtp` コマンドをトリガーとして利用できます。*
