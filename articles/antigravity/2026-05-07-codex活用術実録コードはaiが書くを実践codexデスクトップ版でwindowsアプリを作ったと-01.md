## はじめに

今回は、Codex GPT-5.5にWindowsアプリを作成してもらったときの話になります。

いままでAIエディター（Kiro,Antigravityなと）を使っていたのですが、Codexデスクトップ版を使ってみました。

AIエディターと違って、**「丸投げ感がすごい」** が...あと対話が無いので...**ペアプロ感も無し**  
...面白いやりとりもないですｗ

まずどうやって動くものを作成しているのかの流れの説明になります。

1. AIにアプリの仕様書を作成してもらう
2. Codex GPT-5.5 デスクトップ版を使ってアプリを作成
3. 起動して動作確認
4. 作成されたものをベースに修正　←　ここは今回パスします。Codexデスクトップ版でもいいですが...Kiro,Antigravity,Cursorを使ったほうがいいかも

以降、実際の流れになります。

## 1.作りたいアプリの仕様を作成

Geminiに仕様をまとめてもらいました。

#### 🧑‍💻 ユーザー

タイムタイマーって言って通じますか?

あのタイムタイマーをWindowsのデスクトップアプリとして作成したいです。  
外観はあのいろんな色に変更できるようにしたいです。  
タイムタイマーの機能はひととおり実装してほしいこんな感じで...えっとAIエージェントに渡すアプリの仕様を考えてくれませんか

### 🤖 Gemini

Windowsのデスクトップアプリとして「タイムタイマー」を作成するプロジェクト、非常に面白そうですね！

AIエージェント（CursorやGitHub Copilotなど）に渡す際に、「一発で意図が伝わり、手戻りが少ない」レベルまで落とし込んだ仕様書案を作成しました。

特に外観（カラーカスタマイズ）と、Windowsネイティブな使い勝手を両立させる点にフォーカスしています。

以下が作成した仕様書になります。

```
## AIエージェント用：アプリ開発仕様書案
### 1. プロジェクト概要
- アプリ名: Visual Progress Timer (仮)

- 目的: タイムタイマーの「残り時間が色面で減っていく」視覚的効果を再現したWindowsデスクトップアプリ。

- 技術スタック: 指定なし（推奨：Flutter, React + Tauri, または C# / WPF）

※軽量かつデスクトップでの描画パフォーマンスを重視。

### 2. コア機能 (Core Features)
#### タイマー設定:

- 円形（または扇形）のゲージをドラッグして直感的に時間を設定。

- 最大60分（設定で最大時間の変更も可能にする）。

#### カウントダウン制御:

- スタート / ストップ / リセット機能。

- 残り時間に合わせて、色面が時計回りに消えていく（または反時計回りに減る）アニメーション。

#### 通知機能:

- タイムアップ時にシステム通知およびアラーム音の再生。

### 3. 外観・カスタマイズ (Appearance & UI)
#### 色変更機能 (Color Themes):

- ゲージの色: ユーザーがカラーピッカー、またはプリセット（赤、青、緑、オレンジ）から選択可能。

- 背景色: ダークモード / ライトモード対応。

#### ウィンドウ制御:

- 最前面表示 (Always on Top): 他の作業中も常に視界に入るように切り替え可能。

- 透過設定: ウィンドウ自体の不透明度を調整し、作業を邪魔しないようにする。

- 枠なしモード: 最小限の円形タイマーだけを表示するフローティングモード。

### 4. UI/UX 仕様
レスポンシブ: ウィンドウサイズに合わせてタイマーの円がリサイズされること。

操作性: マウスホイールでの時間微調整機能。

### 5. 実装ステップの指示（AIへのプロンプト例）
「上記仕様に基づき、まずはメインの円形タイマーを描画し、カウントダウンに合わせて色面が減っていく基本ロジックを実装してください。その後、カラーピッカーによる色変更機能と、Windowsの『最前面表示』機能を順次追加してください。」
```

### 🤖 Codex

では `dotnet run` で起動確認します。WPFなのでコマンドがプロセスを保持する可能性があり、短時間で起動できるかを見て必要なら止めます。

`dotnet run` はビルドエラーなしで起動フェーズに入りましたが、WPFアプリのウィンドウ待機でコマンドが終了しないため、ツール側の15秒タイムアウトに当たりました。念のためプロセスが残っているか確認します。

起動確認だけでなく、実際に触れる状態で立ち上げ直します。`dotnet run` を別プロセスで開始して、こちらのコマンドはすぐ戻る形にします。
