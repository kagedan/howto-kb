## はじめに

正直に言うと、1年前の自分は「AIツールはすごいけど、日常には馴染まない」と思っていた。ChatGPTやClaude、GitHub Copilotを試してみたものの、「使える場面で使う」という受動的なスタンスだった。

それが変わったきっかけは、ある朝のスタンドアップミーティングで「昨日何やってたっけ？」と詰まった瞬間だった。作業ログもなく、Gitのコミットを遡って昨日の自分の行動を再現するという、なんとも情けない経験をした。

その日から「AIを受動的に使うのをやめて、一日のルーティンそのものをAIと共に設計し直す」という実験を始めた。本記事は、その実践記録だ。

---

## 全体の仕組み（アーキテクチャ概要）

現在の自分の一日は大きく以下の3フェーズに分かれており、それぞれにAIと自動化スクリプトが絡んでいる。

```
[朝] デイリーブリーフィング生成
  └─ Notion API + OpenAI API → 今日のタスク要約・優先度付け

[日中] コーディング支援＋作業ログの自動記録
  └─ GitHub Copilot + VSCode拡張 + cronジョブ

[夜] ふりかえり自動化
  └─ Gitログ + 作業メモ → Claude API → 日報Markdown生成
```

これらはすべてローカルで動くPythonスクリプト群と、いくつかのクラウドAPIで構成されている。インフラは最小限で、月のAPI費用は1,000〜1,500円程度に収まっている。

---

## 実装手順

### Step 1：朝のデイリーブリーフィングを自動生成する

毎朝9時になると、スクリプトが自動実行されてターミナルに「今日のブリーフィング」が流れるようにした。

**必要なもの：**

* Notion APIキー（タスク管理をNotionでやっている場合）
* OpenAI APIキー
* Python 3.10以上

```
# daily_brief.py
import os
import requests
from openai import OpenAI
from datetime import date

NOTION_TOKEN = os.environ["NOTION_TOKEN"]
NOTION_DB_ID = os.environ["NOTION_DATABASE_ID"]
OPENAI_API_KEY = os.environ["OPENAI_API_KEY"]

client = OpenAI(api_key=OPENAI_API_KEY)

def fetch_today_tasks():
    url = f"https://api.notion.com/v1/databases/{NOTION_DB_ID}/query"
    headers = {
        "Authorization": f"Bearer {NOTION_TOKEN}",
        "Notion-Version": "2022-06-28",
        "Content-Type": "application/json",
    }
    payload = {
        "filter": {
            "property": "Status",
            "select": {"equals": "In Progress"}
        }
    }
    res = requests.post(url, headers=headers, json=payload)
    results = res.json().get("results", [])
    tasks = []
    for r in results:
        title = r["properties"]["Name"]["title"]
        if title:
            tasks.append(title[0]["plain_text"])
    return tasks

def generate_brief(tasks: list[str]) -> str:
    task_list = "\n".join(f"- {t}" for t in tasks)
    prompt = f"""
今日（{date.today()}）の進行中タスク一覧です：
{task_list}

これらを踏まえて、以下を日本語で出力してください：
1. 今日最も優先すべきタスクとその理由（1〜2行）
2. 集中すべき時間帯の提案（午前・午後の配分）
3. 注意すべきリスクや依存関係（あれば）

シンプルに、エンジニアが朝イチで読む前提で書いてください。
"""
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}],
        max_tokens=400,
    )
    return response.choices[0].message.content

if __name__ == "__main__":
    tasks = fetch_today_tasks()
    if not tasks:
        print("今日のタスクが見つかりませんでした。Notionを確認してください。")
    else:
        brief = generate_brief(tasks)
        print("=" * 40)
        print("📋 TODAY'S BRIEF")
        print("=" * 40)
        print(brief)
```

これをcronで毎朝9時に実行する。

```
# crontabに追加
# crontab -e で編集
0 9 * * 1-5 /usr/bin/python3 /path/to/daily_brief.py >> /tmp/brief.log 2>&1
```

**なぜgpt-4o-miniを選んだか：** ブリーフィング生成は精度より速度とコストが重要。4oや4-turboだとトークン単価が高く、毎日実行するには痛い。miniで十分な品質が出ている。

### Step 2：作業ログを15分ごとに自動記録する

「今日何やってたっけ問題」の根本原因は、作業ログがなかったことだ。そこでVSCodeのアクティブファイルと直近のGitコミットを15分おきに記録するスクリプトを仕込んだ。

```
# worklog.py
import subprocess
import json
from datetime import datetime
from pathlib import Path

LOG_PATH = Path.home() / ".worklog" / f"{datetime.now().strftime('%Y-%m-%d')}.jsonl"
LOG_PATH.parent.mkdir(exist_ok=True)

def get_recent_commits(n=3):
    try:
        result = subprocess.run(
            ["git", "log", f"-{n}", "--oneline", "--no-walk"],
            capture_output=True, text=True, cwd=Path.home() / "dev"
        )
        return result.stdout.strip().split("\n")
    except Exception:
        return []

def get_active_window_title():
    # macOS向け (windowserverを使わずosascriptで取得)
    try:
        result = subprocess.run(
            ["osascript", "-e",
             'tell application "System Events" to get name of first process whose frontmost is true'],
            capture_output=True, text=True
        )
        return result.stdout.strip()
    except Exception:
        return "unknown"

def log_activity():
    entry = {
        "timestamp": datetime.now().isoformat(),
        "active_app": get_active_window_title(),
        "recent_commits": get_recent_commits(),
    }
    with open(LOG_PATH, "a") as f:
        f.write(json.dumps(entry, ensure_ascii=False) + "\n")

if __name__ == "__main__":
    log_activity()
```

```
# 15分ごとに記録
*/15 * * * * /usr/bin/python3 /path/to/worklog.py
```

このログがあとの「夜のふりかえり」に大活躍する。

### Step 3：夜のふりかえりを自動生成する

終業後（18時以降に手動実行、または21時に自動実行）に、その日のWorklogとGitログをまとめてClaudeに投げ、Markdown形式の日報を生成する。

```
# evening_review.py
import json
import subprocess
from datetime import date
from pathlib import Path
import anthropic

client = anthropic.Anthropic()

def load_worklog(target_date: str) -> list[dict]:
    log_path = Path.home() / ".worklog" / f"{target_date}.jsonl"
    if not log_path.exists():
        return []
    with open(log_path) as f:
        return [json.loads(line) for line in f if line.strip()]

def get_git_log_today() -> str:
    today = date.today().isoformat()
    try:
        result = subprocess.run(
            ["git", "log", f"--after={today} 00:00",
             "--oneline", "--all"],
            capture_output=True, text=True, cwd=Path.home() / "dev"
        )
        return result.stdout.strip()
    except Exception:
        return ""

def generate_review(worklog: list, git_log: str) -> str:
    # ワークログを要約（全件は長すぎるので間引き）
    sampled = worklog[::4]  # 1時間ごとのサンプル
    log_summary = json.dumps(sampled, ensure_ascii=False, indent=2)

    prompt = f"""
以下は今日の作業データです。

## 作業ログ（サンプリング）
{log_summary}

## Gitコミット
{git_log if git_log else '（コミットなし）'}

これを元に、以下のMarkdown形式で日報を作成してください：

# 日報 {date.today()}

## 今日やったこと
- （箇条書き）

## 学んだこと・気づき
- （箇条書き）

## 明日やること
- （箇条書き）

## 一言コメント
（50文字以内で今日を振り返る）
"""
    message = client.messages.create(
        model="claude-3-5-haiku-20241022",
        max_tokens=600,
        messages=[{"role": "user", "content": prompt}]
    )
    return message.content[0].text

if __name__ == "__main__":
    today = date.today().isoformat()
    worklog = load_worklog(today)
    git_log = get_git_log_today()
    review = generate_review(worklog, git_log)

    output_path = Path.home() / "reports" / f"{today}.md"
    output_path.parent.mkdir(exist_ok=True)
    output_path.write_text(review, encoding="utf-8")
    print(f"日報を生成しました: {output_path}")
    print(review)
```

**なぜClaudeを使うか：** ふりかえりには「文脈を読んだ要約能力」が必要で、Claudeはこの用途での出力品質がOpenAIよりわずかに自分の好みに合っていると感じた。モデルはhaiku（最速・最安）で十分。

## ハマったポイント・注意点

### 1. Notion APIのフィルタ条件が思い通りに動かない

Notionのデータベースフィルタは、プロパティ型によって指定方法が異なる。`select`型と`status`型を混同して半日溶かした。Notion APIドキュメントは必ず最新版を参照すること。

```
# NG: selectプロパティに statusフィルタを使う
"filter": {"property": "Status", "status": {"equals": "In Progress"}}

# OK: selectプロパティにはselectフィルタ
"filter": {"property": "Status", "select": {"equals": "In Progress"}}
```

### 2. cronの環境変数が通らない

ローカルのシェルで動いてもcronだと動かない、という典型的な罠にはまった。cronの実行環境には`~/.zshrc`や`~/.bash_profile`が読み込まれないため、環境変数は別途設定が必要。

```
# crontabの先頭に環境変数を直書きする
OPENAI_API_KEY=sk-xxxx
NOTION_TOKEN=secret_xxxx
NOTION_DATABASE_ID=xxxxxxxx

0 9 * * 1-5 /usr/bin/python3 /path/to/daily_brief.py
```

もしくは`.env`ファイルを読む形にして、スクリプト内で`python-dotenv`を使う方が管理しやすい。

### 3. APIコストが予想外に膨らむことがある

最初は`gpt-4o`を全部に使っていた。ブリーフィング・ログ記録・日報をすべて4oで回すと、1ヶ月で3,000〜5,000円になる。用途ごとにモデルを使い分けることで1,500円以下に抑えられた。

| 用途 | モデル | 理由 |
| --- | --- | --- |
| 朝のブリーフィング | gpt-4o-mini | 速度・コスト優先 |
| 日報生成 | claude-3-5-haiku | 文章品質・コスト |
| コードレビュー補助 | gpt-4o | 精度最優先 |

## まとめ・今後の展望

この仕組みを導入して約3ヶ月が経った。変化を正直に書くと：

**良くなったこと：**

* 朝の「何から手をつけるか」問題がほぼなくなった
* 日報を書く時間が10分→2分になった
* 週次ふりかえりで「先週何やってたか」がログから即座にわかる

**思ったより変わらなかったこと：**

* 集中力そのものはAIに改善できない（当たり前だが）
* タスクの優先度付けは、AIの提案をそのまま使うより自分で最終判断する方が良い結果になることが多い

**今後やりたいこと：**

* Slackの発言ログも取り込んでコミュニケーションコストを可視化する
* 週次サマリーを自動生成して、月次の自己評価に使う
* ローカルLLM（Ollama + Llama3）での完全オフライン化の検討

ツールに振り回されず、「自分の一日の構造を理解する補助装置」としてAIを使う。そのマインドセットが、結局一番長続きするワークフローを作ると実感している。
